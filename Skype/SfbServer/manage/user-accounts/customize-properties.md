# <a name="customize-user-account-properties-for-skype-for-business-server"></a>Настройка свойств учетной записи пользователя для Скайп для Business Server
 
Для изменения свойств учетной записи отдельного пользователя, можно использовать процедуры, описанные в этом разделе.
  
Существует две основные операции, которые можно выполнить на уровне отдельных пользователей.
  
- [Настройка параметров телефонии для конкретной учетной записи пользователя](customize-properties.md#Tel_Op)
    
- [Перемещение пользователей в другой пул](customize-properties.md#Move_Users)
    
## <a name="configure-telephony-options-for-a-specific-user-account"></a>Настройка параметров телефонии для конкретной учетной записи пользователя
<a name="Tel_Op"> </a>

Можно настроить параметры телефонии для определенного пользователя (до тех пор отдельного пользователя включена поддержка Скайп для Business Server и телефонной связи поддерживает организации).
  
Скайп для параметры телефонии пользователей Business Server относятся следующие:
  
- **Этот параметр отключен аудио и видео** Пользователь не может выполнять вызовы с аудио- и видеоконференций.
    
- **ПК только** Пользователь может выполнять только на ПК голосовые вызовы и видеовызовы.
    
- **Корпоративная голосовая связь** Пользователя можно использовать Скайп для инфраструктуры Business Server для маршрутизации входящих и исходящих вызовов. Пользователь также может выполнять вызовы ПК-ПК.
    
- **Удаленное управление звонками** Пользователь может использовать Скайп для Business Server для управления настольным телефоном и можно также выполнять вызовы ПК-ПК.
    
Для получения дополнительных сведений о настройке телефонии для организации содержатся в документации по развертыванию [Включить пользователей для корпоративной голосовой связи в Скайп для Business Server](../../deploy/deploy-enterprise-voice/enable-users-for-enterprise-voice.md) и [Развертывание корпоративной голосовой связи в Скайп для Business Server](../../deploy/deploy-enterprise-voice/deploy-enterprise-voice.md) .
  
1. Войдите на любой компьютер во внутреннем развертывании с использованием учетной записи пользователя, назначенной роли CsUserAdministrator или CsAdministrator.
    
2. Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 
    
3. На левой панели навигации щелкните **Пользователи**.
    
4. В поле **поиска пользователей** , введите или первой части отображаемого имени, имени, последнего имя, имя учетной записи диспетчера учетных записей (SAM), SIP-адрес или строки универсальный код ресурса (URI) учетной записи пользователя и нажмите кнопку **поиска **.
    
5. В таблице щелкните учетную запись пользователя, который требуется изменить.
    
6. В меню **Правка** щелкните **Изменить**.
    
7. В разделе **Телефония**выполните следующие действия.
    
   - Чтобы отключить аудио- и видеоконференций звонков для пользователя, щелкните **аудио/видео отключено**.
    
   - Чтобы включить для ПК звука коммуникаций для пользователя, но не удаленного управления звонками или корпоративной голосовой связи, щелкните **ПК-ПК только**. Укажите значение для **URI линии** для телефона, который пользователь использует для коммуникаций звука на ПК.
    
   - Чтобы перенаправлять пользователя телефонные звонки с помощью Скайп для бизнес-инфраструктура в соответствии с классом политики службы, включая аудиосвязь ПК-ПК, щелкните **Корпоративной голосовой связи**. В **Строке URI**укажите номер телефона для корпоративной голосовой связи. В **Политика абонентской группы** и **политики голосовой связи**укажите соответствующие политики для пользователя. Чтобы задать правила нормализации для преобразования номеров телефонов набранный пользователем в формат E.164, выберите профиль соответствующее место в **политику расположения**.
    
   - Для включения удаленного вызова элемента управления, который позволяет пользователям управлять их настольных систем телефонной линии из Скайп для сервера для облегчения вызовы ПК-ПК и PC-to-phone вызовов, нажмите кнопку **удаленного управления звонками**. В **Строке URI**укажите номер телефона для удаленного управления звонками. Пользователь должен иметь стационарного телефона и относиться УАТС подключения для маршрутизации вызовов.
    
## <a name="move-users-to-another-pool"></a>Перемещение пользователей в другой пул
<a name="Move_Users"> </a>

Назначение пользователей для определенного сервера или пула, можно использовать Скайп для панели управления Business Server.
  
> [!TIP]
> Перемещение всех существующих пользователей из пула источник, на котором работает Lync Server 2010 и более ранних версий Скайп для назначения пула Business Server сложных сред с Active Directory может привести к более медленных репликации Active Directory. Чтобы избежать этого, фильтры поиска можно использовать для перемещения пользователей из пулов, работающих под управлением Lync Server 2010 или более ранних версий отдельно или Скайп для консоли Business Server можно использовать для переноса данных пользователей с помощью командлетов. Кроме того функциональные возможности фильтрации работает с Скайп для пользователей Business Server. 
  
### <a name="to-move-selected-users-to-a-different-server-or-pool"></a>Чтобы переместить выбранных пользователей в другой сервер или в другой пул

1. Войдите на любой компьютер во внутреннем развертывании с использованием учетной записи пользователя, назначенной роли CsUserAdministrator или CsAdministrator.
    
2. Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server.  
    
3. На левой панели навигации щелкните **Пользователи**.
    
4. В поле **поиска пользователей** , введите или первой части отображаемого имени, имени, последнего имя, имя учетной записи диспетчера учетных записей (SAM), SIP-адрес или строки универсальный код ресурса (URI) учетной записи пользователя и нажмите кнопку **поиска **. 
    
5. В таблице выберите одного или нескольких пользователей в списке. 
    
6. В меню **Действие** выберите команду **переместить выбранных пользователей в пул**.
    
7. В **Переместить пользователей**выберите пул, который необходимо переместить пользователей в **пул регистратора назначения**.
    
8. (Необязательно) Если конечный сервер или пул недоступен, установите флажок **Force** .
    
    > [!CAUTION]
    > Если выбран параметр **Force**, при перемещении учетной записи пользователя, но удаляется все данные пользователя (например, конференций, в которых пользователь запланировано). Если он не установлен, перемещаются учетную запись и связанных данных. 
  
### <a name="to-move-all-users-from-one-server-or-pool-to-a-different-server-or-pool"></a>Для перемещения всех пользователей из одного сервера или пула на другой сервер или пул

1. Войдите на любой компьютер во внутреннем развертывании с использованием учетной записи пользователя, назначенной роли CsUserAdministrator или CsAdministrator.
    
2. Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server.  
    
3. На левой панели навигации щелкните **Пользователи**.
    
4. В меню **Действие** выберите команду **переместить всех пользователей в пул**.
    
5. **Переместить пользователей**выберите пул, содержащий учетные записи пользователей, которые необходимо переместить в **исходный пул регистратора**.
    
6. В **конечном пуле регистратора**выберите пул, который требуется переместить пользователей.
    
7. (Необязательно) Если конечный сервер или пул недоступен, установите флажок **Force** .
    
    > [!CAUTION]
    > Если выбран параметр **Force**, при перемещении учетной записи пользователя, но удаляется все данные пользователя (например, конференций, в которых пользователь запланировано). Если он не установлен, перемещаются учетную запись и связанных данных. 
  
### <a name="to-move-users-from-one-pool-to-a-different-pool-by-using-a-filter"></a>Для перемещения пользователей из одного пула в другой пул с использованием фильтра

1. Войдите на любой компьютер во внутреннем развертывании с использованием учетной записи пользователя, назначенной роли CsUserAdministrator или CsAdministrator.
    
2. Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 
    
3. На левой панели навигации щелкните **Пользователи**.
    
4. В **Поиск пользователей**нажмите кнопку **поиска**и нажмите кнопку **Добавить фильтр**.
    
5. В области условий поиска выберите **Пул регистратора**, выберите **равно**, выберите **Текущее полное доменное имя пула**и нажмите кнопку **Найти**.
    
6. В меню **Действие** выберите команду **переместить всех пользователей в пул**.
    
    > [!NOTE]
    > При применении фильтра на существующем наборе пользователей, параметр **переместить всех пользователей в пул** , находящийся в контексте отфильтрованное подмножество пользователей, не **все** возможные пользователями.
  
7. **Переместить пользователей**выберите пул, содержащий учетные записи пользователей, которые необходимо переместить в **исходный пул регистратора**.
    
8. В **конечный пул регистратора**выберите пул, где необходимо переместить пользователей.
    
9. (Необязательно) Если конечный сервер или пул недоступен, установите флажок **Force** .
    
    > [!CAUTION]
    > Если выбран параметр **Force**, при перемещении учетной записи пользователя, но все данные пользователя удаляется (например, конференций, в которых пользователь запланированных и контакты). Если он не установлен, перемещаются учетную запись и связанных данных. 
  
### <a name="to-move-users-from-one-pool-to-another-using-windows-powershell-cmdlets"></a>Для перемещения пользователей из одного пула в другой с помощью командлетов Windows Powershell

1. В зависимости от того, как вы выполняете команды Windows PowerShell (то есть локально или удаленно) необходимо выполнить вход в качестве члена правильных Скайп для административных ролей сервера Business следующим образом:
    
   a) Если вы выполняете команды на локальном компьютере (например, входите непосредственно на сервер переднего плана): Войдите на компьютер, где установлена Скайп для консоли Business Server как член группы RTCUniversalServerAdmins или с помощью необходимых права пользователя, как описано в **Делегирование разрешений на установку**.
    
   б) Если вы выполняете команды удаленно на другом компьютере (например, входе на свой компьютер и выполняете команды удаленно на сервере переднего плана выпуска Standard): С учетной записью пользователя, назначенной роли CsUserAdministrator или CsAdministrator роль, войдите на любой компьютер во внутреннем развертывании.
    
2. Запустите Скайп для консоли Business Server: нажмите кнопку **Пуск**, последовательно выберите пункты **Все программы**, щелкните **Скайп для бизнеса**и нажмите кнопку **Скайп для консоли Business Server**.
    
3. Для перемещения отдельных пользователей, используйте командлет Move-CsUser следующим образом:
    
    ```
    Move-CsUser -Identity "Pilar Ackerman" -Target "pool01.contoso.net"
    ```

    Где пользователей для перемещения пользователя Пилар Аккерман и пользователь перемещается из их назначенного в данный момент домашнего пула в пул pool01.contoso.net.
    
4. Для перемещения большого числа пользователей используйте фильтры с помощью командлета **Get-CsUser** и передавайте полученный набор пользователей **Move-CsUser**:
    
    ```
    Get-CsUser -Filter {RegistrarPool -eq "CurrentPoolFqdn"} | Move-CsUser -Target "TargetPoolFQDN"
    ```

    Совмещение команд **Get-CsUser** и **Move-CsUser** может дать следующий результат:
    
    ```
    Get-CsUser -Filter {RegistrarPool -eq "pool02.contoso.net"} | Move-CsUser -Target "pool01.contoso.net"
    ```
