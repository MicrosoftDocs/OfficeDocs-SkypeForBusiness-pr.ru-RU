# <a name="manage-enhanced-9-1-1-and-the-location-service-in-skype-for-busines-server"></a>Manage enhanced 9-1-1 and the Location service in Skype for Busines Server

Skype для бизнеса Server поддерживает вызовы Enhanced 9-1-1 (E9-1-1) из клиентов Skype для бизнеса. When you configure Skype for Business Server for E9-1-1, emergency calls placed from Skype for Business include Emergency Response Location (ERL) information from the Location Information service database. Используйте процедуры, которые данная статья посвящена управлению политикой расположения.

> [!Note]
> Дополнительные сведения о развертывании расширенных Корпоративная голосовая связь, таких как E9-1-1 и [](../deploy/deploy-enterprise-voice/deploy-advanced-enterprise-voice-features.md)служба сведений о расположении, см. в Корпоративная голосовая связь.

В Skype для бизнеса Server можно использовать политику расположения для применения параметров, которые относятся к функциям Enhanced 9-1-1 (E9-1-1), а также к настройкам расположения для пользователей или контактов. Политика расположения определяет, активирована ли для пользователя функция E9-1-1 и, если да, то в каком режиме выполняются экстренные вызовы. Например, можно использовать политику расположения, чтобы определить номер экстренного вызова (911 в США), а также должен ли автоматически оповещаться отдел корпоративной безопасности и как маршрутизировать вызов.

Политики расположения можно настроить в группе **конфигурации** сети на панели управления Skype для бизнеса Server. На панели управления Skype для бизнеса Server можно просматривать, создавать, изменять или удалять политики расположения. Используйте следующую процедуру для просмотра сведений о политиках расположения. 


## <a name="view-location-policy-information"></a>Просмотр сведений о политике расположения 

1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server. 

3.  В левой панели навигации щелкните **"Конфигурация** сети" и выберите **"Политика расположения".**

4.  На странице **Политика расположения** выберите политику расположения, которую хотите изменить.

5.  В меню **Правка** щелкните **Подробнее**.
 
    > [!NOTE]  
    > Одновременно можно просматривать сведения только об одной политике расположения.

По умолчанию существует одна политика с названием "Глобальная", которая не может быть удалена или переименована. Однако вы можете изменить глобальную политику. Эта политика будет применяться для всех пользователей и контактов, если вы не создаете политики сайта или политики на пользователя. Политики на пользователя должны применяться к определенным пользователям.


## <a name="create-or-modify-a-location-policy"></a>Создание или изменение политики расположения 

В Skype для бизнеса Server можно переопредить время по умолчанию между запросами клиента на обновление местоположения из службы сведений о расположении. Значение по умолчанию: 4 часа. Для переопределения значения по умолчанию используется командлет **Set-CsLocationPolicy** с параметром LocationRefreshInterval.


### <a name="to-create-a-new-location-policy-in-the-skype-for-business-server-control-panel"></a>Создание новой политики расположения на панели управления Skype для бизнеса Server

1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server. 

3.  В левой панели навигации щелкните **"Конфигурация** сети" и выберите **"Политика расположения".**

4.  На странице **Политика расположения** щелкните **Создать**, а затем выберите тип политики, которую хотите создать:
    
      - Чтобы создать политику уровня сайта, щелкните **Политика уровня сайта**. В окне **Выбор сайта** выберите сайт, к которому должна применяться политика, и нажмите кнопку **ОК**. На странице **Новая политика расположения** в поле **Область действия** содержится значение **Сайт**, и в поле **Имя** содержится имя выбранного сайта. Оба этих поля недоступны для изменения. Политика уровня сайта автоматически применяется ко всем пользователям на указанном сайте и переопределяет глобальную политику для этих пользователей.
    
      - Чтобы создать **Политику уровня пользователя**, щелкните **Политика уровня пользователя**. В окне **Новая политика расположения** в поле **Область действия** содержится значение **Пользователь**. Это значение недоступно для изменения. В поле **Имя** введите имя, которое хотите присвоить этой политике. Политика уровня пользователя не применяется автоматически к каким-либо пользователям. После создания политики уровня пользователя необходимо вручную установить политику для пользователей или сетевых сайтов, к которым хотите применить политику.

5.  Заполните остальные поля следующим образом:
    
      - **Включить расширенные экстренные службы**   Чтобы включить пользователей, связанных с этой политикой, для E9-1-1, выберите этот контроль. Если экстренные службы включены, клиенты Skype для бизнеса Server будут получать сведения о расположении при регистрации и включать их при вызове экстренных служб.
    
      - **Расположение**   Укажите одно из следующих значений:
        
          - **Обязательное**   При регистрации клиента в новом расположении пользователю будет предложено ввести сведения о расположении. Пользователь может отвергнуть приглашение, не вводя никаких сведений. Если сведения введены, на экстренный вызов сначала ответит поставщик экстренных служб, чтобы проверить расположение, прежде чем звонок будет направлен оператору пункта общественной безопасности (PSAP) (то есть, оператору службы 911).
        
          - **Не требуется**   Пользователь не получит запрос на расположение. Когда выполняется вызов без сведений о расположении, поставщик экстренных служб отвечает на звонок и запрашивает место нахождения.
        
          - **Заявление об отказе**   Этот параметр является тем же, что и **required,** за исключением того, что пользователь не может отклонять запрос без ввода сведений о расположении. Пользователь по-прежнему может выполнить экстренный вызов, но никакие другие вызовы не могут быть выполнены без ввода информации. Кроме того, пользователю показывается текст отказа, который может предупреждать о последствиях отказа от ввода информации о местонахождении. Чтобы настроить текст заявления об отказе, необходимо использовать оболочку управления Skype для бизнеса Server для запуска команды **Set-CsLocationPolicy** или **New-CsLocationPolicy** с параметром EnhancedEmergencyServiceDisclaimer. Подробные сведения см. в [настройках Set-CsLocationPolicy](https://docs.microsoft.com/powershell/module/skype/Set-CsLocationPolicy) или [New-CsLocationPolicy.](https://docs.microsoft.com/powershell/module/skype/New-CsLocationPolicy)
          
    
      - **Использование расположения только для экстренных служб** Skype для бизнеса может использовать сведения о расположении по различным причинам (например, для уведомления партнеров по команде о текущем расположении). Чтобы сведения о расположении были доступны только для использования с экстренным вызовом, выберите этот контроль.
    
      - **Использование STN**   Режим работы с телефонной сетью общего пользования (PSTN), который будет использоваться для определения маршрута голосовой связи для маршрутов экстренных вызовов от клиентов, использующих этот профиль. Маршрут, связанный с таким применением, должен указывать на SIP-магистраль, выделенную для экстренных вызовов, или на шлюз идентификационного номера экстренного местоположения (ELIN), который направляет экстренные вызовы в ближайший пункт общественной безопасности (PSAP).
    
      - **Номер экстренного номера**   Номер, набираемый для экстренного обслуживания. В США это значение 911. Строка должна состоять из цифр от 0 до 9; длина строки от 1 до 10 цифр.
    
      - **Маска набора номера экстренной службы**   Номер, который вы хотите преобразовыть в значение номера экстренного номера при наборе номера. Например, если ввести значение 212 в это поле, а поле номера экстренной службы имеет значение 911, то при наборе пользователем 212 вызов направляется на номер 911. Это позволяет набирать альтернативные номера экстренной службы, и по-прежнему вызов будет поступать в экстренные службы (например, если кто-нибудь из страны или области с другим номером экстренной службы пытается набрать номер экстренной службы этой страны или области вместо номера экстренной службы страны или области, в которой находится в текущий момент). Можно определить несколько масок набора номера экстренных служб, разделяя значения точками с запятыми. Например, 212;414. Максимальная длина строки равна 100 знакам. Каждый знак должен быть равен цифре от 0 до 9.
      

        > [!IMPORTANT]  
        > Проверьте, чтобы указанное значение маски набора номера не совпадало с номером в диапазоне приостановки звонков. Маршрутизация парковки вызовов имеет приоритет над преобразованием строки набора номера экстренных служб. Чтобы просмотреть существующие диапазоны орбит парковки вызовов, щелкните **Функции голосовой связи** на левой панели навигации, а затем щелкните **Парковка вызовов**. 

    
      - **URI уведомления**   Один или несколько УНИФИЦИРОВАННЫХ кодов ресурсов SIP, которые необходимо уведомить при вызове экстренных служб. Например, служба безопасности компании может получать мгновенные сообщения при выполнении экстренных вызовов. Если сведения о расположении звонящего доступны, они будут включаться в уведомление. Несколько URI-адресов SIP могут включаться в список с запятыми-разделителями. Например, "sip:security@litwareinc.com","sip:kmyer@litwareinc.com". Поддерживаются списки разделения. Строка может иметь длину от 1 до 256 знаков, и должна начинаться с префикса "sip:". До щелчка в поле URI-адреса уведомления в нем отображается пример значения.
    
      - **URI конференции**   URI SIP, в данном случае номер телефона третьей стороны, которая будет конференц-связи с любыми экстренные вызовы. Например, служба охраны компании могла бы получать вызов, когда выполняется вызов экстренных служб, и прослушивать или участвовать в этом вызове (в зависимости от значения, указываемого в поле **Режим конференц-связи**). Строка должна иметь длину от 1 до 256 знаков и должна начинаться с префикса "sip:". Пример набираемой строки отображается в поле до тех пор, пока не щелкнут внутри данного поля.
    
      - **Режим конференции**   Если указать значение в поле **URI** конференции, режим конференции определяет, может ли третья сторона участвовать в вызове или только прослушивать его.  Укажите один из следующих вариантов:
        
          - **Односторонний**   Третья сторона может только прослушивать разговор между вызывающим абонентом и оператором пункта общественной безопасности (PSAP).
        
          - **Двухсторонний**   Третья сторона может прослушивать и участвовать в вызове между абонентом и оператором пункта общественной безопасности (PSAP).

6.  Нажмите кнопку **Зафиксировать**.


    > [!IMPORTANT]  
    > Когда создают политику на уровне пользователя, первоначально эта политика не применяется к каким-либо пользователям или сетевым сайтам. Чтобы применить политику к пользователю, щелкните **Пользователи** на левой панели навигации. Найдите пользователя, к которому хотите применить политику. В меню **Правка** щелкните **Подробнее**. На странице **"Изменение пользователя сервера"** выберите  новую политику расположения в выпадаемом списке "Политика расположения" и нажмите кнопку **"Зафиксировать".**<BR>Чтобы применить политику к сайту сети, щелкните **Конфигурация сети** на левой панели навигации, а затем щелкните **Сайт**. Найдите сайт сети, к которому хотите применить политику. В меню **Правка** щелкните **Подробнее**. В окне **Изменить сайт** выберите новую политику расположения в раскрывающемся списке **Политика расположения**, а затем щелкните **Зафиксировать**.


### <a name="to-modify-a-location-policy-in-the-skype-for-business-server-control-panel"></a>Изменение политики расположения на панели управления Skype для бизнеса Server

1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server. 

3.  В левой панели навигации щелкните **"Конфигурация** сети" и выберите **"Политика расположения".**

4.  На странице **Политика расположения** выберите политику расположения, которую хотите изменить.

5.  В меню **Правка** щелкните **Подробнее**.

6.  На странице **Изменение политики расположения** измените значения полей требуемым образом (подробности см. выше в этом разделе в описании шага 5 процедуры "Чтобы создать новую политику расположения").

7.  Нажмите кнопку **Зафиксировать**.

        
## <a name="delete-a-location-policy"></a>Удаление политики расположения


1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server. 

3.  В левой панели навигации щелкните **"Конфигурация** сети" и выберите **"Политика расположения".**

4.  На странице **Политика расположения** щелкните политику расположения, которую нужно удалить.
   
    > [!NOTE]  
    > За один раз вы можете удалить несколько политик расположения. Для этого нажмите клавишу CTRL и, удерживая ее, выберите несколько политик. Чтобы выбрать все политики, щелкните пункт **Выбрать** в меню **Правка**.


5.  В меню **Изменить** нажмите кнопку **Удалить**.

6.  Нажмите кнопку **ОК**.

    > [!IMPORTANT]  
    > Вы не можете удалить глобальную политику расположения. Если попытаться удалить глобальную политику, отображается предупреждение, а для политики будут восстановлены значения по умолчанию.
