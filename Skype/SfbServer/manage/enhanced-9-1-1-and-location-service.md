# <a name="manage-enhanced-9-1-1-and-the-location-service-in-skype-for-busines-server"></a>Управление enhanced 9-1-1 и службой расположения в Скайп для бизнес-сервера

Скайп для Business Server поддерживает Enhanced 9-1-1 (E9-1-1) вызов из Скайп пользователей. При настройке Скайп для Business Server для E9-1-1, экстренных вызовов, поступающих из Скайп для бизнеса включают расположение экстренного ответа (ERL) данные из базы данных службы сведения о расположении. Используйте процедуры, описанные в этой статье для управления политикой расположения.

> [!Note]
> Для получения дополнительных сведений о развертывании расширенных функций корпоративной голосовой связи, например E9-1-1 и службе информирования о местонахождении см [дополнительными функциями корпоративной голосовой связи](../deploy/deploy-enterprise-voice/deploy-advanced-enterprise-voice-features.md).

В Скайп Business Server можно использовать политики расположения для применения параметры, относящиеся к функциям Enhanced 9-1-1 (E9-1-1) и параметры местоположения для пользователей или контактов. Политика расположения определяет пользователя включена поддержка E9-1-1 и если это так что поведение экстренных вызовов. Например можно использовать политики расположения для определения составляет какой номер экстренного вызова (например, 911 в Соединенных Штатах Америки), ли корпоративной безопасности должны быть автоматически получать уведомления и маршрутизацией вызова.

Можно настроить политики расположения в группе **Конфигурация сети** в Скайп для панели управления Business Server. Из Скайп для панели управления Business Server можно просмотреть, создание, изменение или удаление политик расположения. Используйте следующую процедуру для просмотра сведений о политик расположения. 


## <a name="view-location-policy-information"></a>Просмотр сведений о политике расположения 

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и нажмите кнопку **Политика расположения**.

4.  На странице **Политика расположения** выберите политику расположения, который требуется изменить.

5.  В меню **Правка** щелкните **Подробнее**.
 
    > [!NOTE]  
    > Можно просматривать только сведения об одной политике расположения за раз.

Одна политика, называется Global, существует по умолчанию и его нельзя удалить или переименовать. Тем не менее можно изменить глобальную политику. Эта политика будет применяться для всех пользователей и контактов, если не создавать сайт политик или политик на уровне пользователя. Политики уровня пользователя должны применяться к определенным пользователям.


## <a name="create-or-modify-a-location-policy"></a>Создание или изменение политики местонахождения 

Скайп для Business Server может переопределять значение по умолчанию промежуток времени между запросы клиентов для обновления расположение из службы сведения о расположении. Значение по умолчанию — 4 часа. Командлет **Set-CsLocationPolicy** с параметром LocationRefreshInterval переопределить значение по умолчанию.


### <a name="to-create-a-new-location-policy-in-the-skype-for-business-server-control-panel"></a>Чтобы создать новую политику расположения в Скайп для панели управления Business Server

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и нажмите кнопку **Политика расположения**.

4.  На странице **Политика расположения** щелкните **Создать** и затем выберите тип политики, которую вы хотите создать:
    
      - Чтобы создать политику сайта, щелкните **Политика сайта**. В **разделе Выбор сайта**выберите сайт, который будет применен политику и нажмите **кнопку ОК**. На странице **Новую политику расположения** в поле **область** содержит значение **узла**и содержит имя сайта, выбранное в поле **имя** . Нельзя изменять любой из этих полей. Политики сайта автоматически применяется ко всем пользователям, включенным указанного сайта и переопределяет глобальную политику для этих пользователей.
    
      - Чтобы создать **политику пользователя**, щелкните элемент **Политика пользователей**. В **Новую политику расположения**в поле **область** содержит значение **пользователя**. Это значение нельзя изменить. В поле **имя** введите имя, которые нужно предоставить эту политику. Политика пользователя не применяется автоматически для всех пользователей. После создания политики пользователя, необходимо вручную предоставить пользователям или сетевых узлов, которым требуется политики для применения политики.

5.  Заполните остальные поля следующим образом:
    
      - **Enable enhanced emergency services**   установите этот флажок, чтобы пользователи, связанные с этой политикой для E9-1-1. При включении экстренные службы Скайп для клиентов, Business Server получить сведения о расположении по регистрации и включить эту информацию в экстренных вызов.
    
      - **Расположение**   укажите одно из следующих значений:
        
          - **Требуется**   пользователь будет предложено ввести сведения о расположении при регистрации клиента в новое расположение. Пользователь может закрыть окно строки без ввода какие-либо сведения. Если информация введена, экстренных вызовов будет сначала ответы поставщиком экстренные службы для проверки расположение перед поступает в оператор общедоступных точка ответы на безопасности (PSAP) (то есть, оператор 911).
        
          - **Не требуется**   пользователя не будут запрашиваться расположения. При вызове с не сведения о расположении поставщика экстренные службы будет ответить на звонок и запрос для расположения.
        
          - **Заявление об отказе**   этот параметр используется, то же, что **требуется** за исключением того, что пользователь не может закрыть строке без ввода сведения о расположении. Пользователь по-прежнему могут выполнять экстренных вызовов, но другие вызовы не могут выполняться без ввода данных. Кроме того текст заявления об отказе отображается для пользователя, который может оповещать их последствия отклонения приглашения для ввода сведений о местоположении. Чтобы задать текст заявления об отказе, Скайп для консоли Business Server необходимо использовать для запуска командлета **Set-CsLocationPolicy** или командлет **New-CsLocationPolicy** с параметром EnhancedEmergencyServiceDisclaimer. Дополнительные сведения см [Командлет Set-CsLocationPolicy](https://docs.microsoft.com/powershell/module/skype/Set-CsLocationPolicy) или [New-CsLocationPolicy](https://docs.microsoft.com/powershell/module/skype/New-CsLocationPolicy).
          
    
      - **Использование расположения для экстренных служб** Скайп для бизнеса можно использовать сведения о расположении по различным причинам (например, для уведомления коллег текущего расположения). Установите этот флажок, чтобы убедиться, что сведения о расположении доступен только для использования с экстренных вызовов.
    
      - **Использование ТСОП**   общего пользования использование телефонной сети (общего пользования PSTN), который будет использоваться для определения, какой маршрут голосовых вызовов будет использоваться для маршрутизации экстренные вызовы от клиентов, использующих данный профиль. Маршрут, связанный с этой использования должны указывать на канале SIP, выделенная для экстренных вызовов или шлюз экстренных расположение идентификационный номер (ELIN), который перенаправляет экстренные вызовы для ближайшего общей безопасности ответы на пункт (PSAP).
    
      - **Номер экстренного вызова**   номер, набираемый для связи с аварийными службами. В США это значение — 911. Строка должна состоять из цифр от 0 до 9 и может быть от 1 до 10 цифр.
    
      - **Маска экстренного вызова**   , которую требуется перевести в значение номер экстренного вызова при его Набираемый номер. Например если ввести значение 212 в этом поле и поле номер экстренного вызова имеет значение 911, если пользователь выполняет 212 вызова станут 911. Это позволяет альтернативного аварийного номера набираемых и по-прежнему иметь связи с аварийными службами вызова (например, в том случае, если кто-либо из страны или региона с другой номер экстренного вызова пытается наберите номер, а не на основе номер для страны или региона Страна или регион, в которых они находятся в настоящее время). Вы можете определить несколько масок экстренного вызова значения, разделив их точкой с запятой. Например 212; 414. Максимальная длина строки составляет 100 символов. Каждый символ должен быть цифры от 0 до 9.
      

        > [!IMPORTANT]  
        > Убедитесь, что значение указанной абонентской группе маски не то же, что число в диапазоне приостановки звонков. Маршрутизация парковки вызовов будет иметь приоритет над преобразование строк экстренного вызова. Для просмотра существующих парковки вызовов диапазонов орбит, выберите **Компоненты голосовой связи** в левой панели навигации и нажмите кнопку **Парковки вызовов**. 

    
      - **Notification URI**   одного или нескольких SIP универсальных кодов ресурса (URI) уведомлений при экстренных вызовов. Например office безопасности компании получать уведомления через мгновенные сообщения, при каждом экстренных вызовов. Если у вызывающего абонента она доступна эту папку будут включены в уведомлении. Можно указать как разделенный запятыми список нескольких URI SIP. Например «sip:security@litwareinc.com», «sip:kmyer@litwareinc.com». Списки рассылки, поддерживаются. Строка должна быть от 1 до 256 символов и должна начинаться с префикса «sip:». Прежде чем нажать кнопку в поле URI уведомлений отображается пример.
    
      - **URI конференции**   SIP URI, в данном случае номер телефона, который будет включаться в для экстренных вызовов, которые выполняются третьей стороны. Например office безопасности компании может прием звонка при экстренные вызовы и прослушивать или участвовать в этого вызова (в зависимости от значения, указанного в поле **режим конференции** ). Строка должна быть от 1 до 256 символов и должна начинаться с префикса sip:. Пример отображается, пока не щелкните внутри этого поля.
    
      - **Режим конференции**   задано значение в поле **URI конференции** в **режим конференции** определяет третья сторона может принимать участие в звонке или может только прослушивать. Укажите один из следующих вариантов:
        
          - **Односторонняя**   третья сторона может только прослушивать беседу между абонентом и оператором PSAP.
        
          - **Двусторонняя**   третья сторона может прослушивать и участвовать в вызове между абонентом и оператором PSAP.

6.  Нажмите **Исполнить**.


    > [!IMPORTANT]  
    > При создании политики пользователя, изначально эта политика не применяется для пользователей и сетевых узлов. Чтобы применить политику для пользователя, нажмите кнопку **Пользователи** в левой панели навигации. Найти пользователя, которого вы хотите применить политику. В меню **Правка** щелкните **Подробнее**. На странице " **Изменение пользователя Server** " выберите новую политику расположения в раскрывающемся списке **Политика расположения** и нажмите кнопку **Сохранить**.<BR>Чтобы применить политику с сетевым узлом, щелкните **Конфигурация сети** в левой панели навигации и нажмите кнопку **сайта**. Поиск сетевого узла, к которому требуется применить политику. В меню **Правка** щелкните **Подробнее**. **Изменение сайта**выберите новую политику расположения в раскрывающемся списке **Политика расположения** и нажмите кнопку **Сохранить**.


### <a name="to-modify-a-location-policy-in-the-skype-for-business-server-control-panel"></a>Чтобы изменить политику расположения в Скайп для панели управления Business Server

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и нажмите кнопку **Политика расположения**.

4.  На странице **Политика расположения** выберите политику расположения, который требуется изменить.

5.  В меню **Правка** щелкните **Подробнее**.

6.  На странице " **Изменение политики расположения** " Изменение полей при необходимости (для получения дополнительных сведений, шаг 5 см в разделе «Создание новой политики расположения» процедур, описанных ранее в этом разделе).

7.  Нажмите **Исполнить**.

        
## <a name="delete-a-location-policy"></a>Удаление политики местонахождения


1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и нажмите кнопку **Политика расположения**.

4.  На странице **Политика расположения** выберите политику расположения, которую требуется удалить.
   
    > [!NOTE]  
    > Одновременно можно удалить несколько политик расположения. Для этого нажмите клавишу CTRL и выберите несколько политик, удерживая нажатой клавишу CTRL. Или, чтобы выбрать все политики, нажмите кнопку **Выбрать все** в меню **Правка** .


5.  В меню **Правка** выберите пункт **Удалить**.

6.  Нажмите **ОК**.

    > [!IMPORTANT]  
    > Не удается удалить глобальную политику расположения. Если попытаться удалить глобальную политику вы получите предупреждение и политики восстановит значения по умолчанию.
