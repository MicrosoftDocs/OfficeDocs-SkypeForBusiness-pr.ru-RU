# <a name="assign-a-per-user-pin-policy-in-skype-for-business-server"></a>Назначение политики ПИН-кода для каждого пользователя в Skype для бизнеса Server

**Сводка:** Подготовка сертификатов AV и OAuth для Skype для бизнеса Server.

Политика личных идентификационных номеров (ПИН-код) конференц-связи с телефонным подключением — это один из индивидуальных параметров учетной записи пользователя, которую можно настроить в Skype для бизнеса Server панель управления.

Развертывание одной или нескольких политик ПИН-кода для каждого пользователя является необязательным. Вы также можете развернуть только политику ПИН-кода глобального уровня или ПОЛИТИКУ ПИН-кода на уровне сайта. При развертывании политик для отдельных пользователей необходимо явным образом назначить их пользователям, группам или контактному объекту. Права пользователей и разрешения, касающиеся использования ПИН-кодов для конференц-связи с телефонным подключением, автоматически по умолчанию устанавливаются в политике ПИН-кодов глобального уровня, если не назначена определенная политика уровня сайта или для каждого пользователя.

После создания по крайней мере одной политики ПИН-кода для каждого пользователя используйте процедуры, описанные в этом разделе, чтобы назначить политику, которая определяет ограничения, которые сервер должен наложить на ПИН-коды, созданные и используемые определенным пользователем.

## <a name="to-assign-a-per-user-pin-policy"></a>Назначение политики ПИН-кода для каждого пользователя

1. Войдите на любой компьютер во внутреннем развертывании с использованием учетной записи пользователя, назначенной роли CsUserAdministrator или CsAdministrator.

2. Откройте окно браузера и введите URL-Администратор, чтобы открыть Skype для бизнеса Server панель управления.

3. В левой панели навигации щелкните **Пользователи**.

4. Используйте один из следующих методов для поиска пользователя:

   - В поле **Поиск пользователей** введите полное отображаемое имя или его часть, имя, фамилию, имя учетной записи диспетчер учетных записей безопасности (SAM), SIP-адрес или URI учетной записи пользователя, а затем нажмите кнопку **Найти**.

   - Если вы сохранили запрос, щелкните значок **Открыть запрос**, используйте диалоговое окно **Открыть**, чтобы получить запрос (USF-файл), а затем нажмите кнопку **Найти**.

5. (Необязательно) Укажите дополнительные критерии поиска, чтобы сузить результаты:

   а. Нажмите кнопку **Добавить фильтр**.

   б. Введите свойство пользователя, набрав его или нажав кнопку стрелки в раскрывающемся списке, чтобы выбрать свойство.

   c. В раскрывающемся списке **Равно** щелкните оператор (например, **Равно** или **Не равно**).

   г) В зависимости от выбранного свойства пользователя введите критерии, которые необходимо использовать для фильтрации результатов поиска; для это введите его или щелкните стрелку в раскрывающемся списке.

    > [!TIP]
    > Чтобы добавить дополнительные пункты в запрос поиска, нажмите кнопку **Добавить фильтр**.

   д) Нажмите кнопку **Найти**.

6. Выберите пользователя в списке результатов поиска, щелкните **Действие**, а затем щелкните **Назначить политики**.

    > [!TIP]
    > Если вы хотите, чтобы к нескольким пользователям применялись одинаковые политики ПИН-кодов для отдельных пользователей, выберите несколько пользователей в результатах поиска, нажмите кнопку "Действия" и нажмите кнопку **"Назначить политики"**.

7. В **разделе "Назначение политик**" в **разделе "Политика ПИН-кода**" выполните одно из следующих действий:

    > [!NOTE]
    > Так как существует несколько политик, которые можно настроить с помощью диалогового окна  "Назначение политик", **\<Keep as is\>** по умолчанию выбирается для каждой политики в диалоговом окне. Чтобы продолжить использование политики, назначенной пользователю ранее, не вносите никаких изменений в эти настройки.

   - Разрешите Skype для бизнеса Server автоматически выбрать политику глобального уровня или, если она определена, политику уровня сайта.

   - Щелкните имя политики ПИН-кода для каждого пользователя, которую вы ранее определили на странице **политики ПИН-кода** .

     > [!TIP]
     > Чтобы с легкостью определить политику, которую необходимо назначить, после выбора названия политики щелкните **Просмотр** для просмотра прав и разрешений пользователя, заданных в политике.

8. По завершении нажмите кнопку **ОК**.

## <a name="assigning-a-per-user-pin-policy-by-using-windows-powershell-cmdlets"></a>Назначение политики Per-User ПИН-кода с помощью Windows PowerShell командлетов

Политики ПИН-кода для каждого пользователя можно назначить с помощью Windows PowerShell и **командлета Grant-CsPinPolicy**. Вы можете запустить этот командлет из командной консоли Skype для бизнеса Server или из удаленного сеанса Windows PowerShell. Дополнительные сведения об использовании удаленного Windows PowerShell для подключения к Skype для бизнеса Server см. в статье ["Удаленное администрирование PowerShell в Microsoft Lync"](https://blog.insideo365.com/2011/08/remote-lync-powershell-administration/). Этот процесс в Skype для бизнеса Server.

### <a name="to-assign-a-per-user-pin-policy-to-a-single-user"></a>Назначение политики ПИН-кодов для отдельных пользователей одному пользователю

- Приведенная ниже команда назначает политику ПИН-кода для каждого пользователя RedmondPinPolicy пользователю Ken Myer.

  ```PowerShell
  Grant-CsPinPolicy -Identity "Ken Myer" -PolicyName "RedmondPinPolicy"
  ```

### <a name="to-assign-a-per-user-pin-policy-to-multiple-users"></a>Назначение политики ПИН-кода для отдельных пользователей нескольким пользователям

- Приведенная ниже команда назначает политику ПИН-кода для каждого пользователя RedmondUsersPinPolicy всем пользователям, которые работают в городе Редмонд. Дополнительные сведения о параметре LdapFilter, используемом в этой команде, см. в [разделе Get-CsUser](/powershell/module/skype/get-csuser).

  ```PowerShell
  Get-CsUser -LdapFilter "l=Redmond" | Grant-CsPinPolicy -PolicyName "RedmondUsersPinPolicy"
  ```

### <a name="to-unassign-a-per-user-pin-policy"></a>Отмена назначения политики ПИН-кода для каждого пользователя

- Приведенная ниже команда отменит назначение любой политики ПИН-кода для каждого пользователя, ранее назначенной Пользователю Ken Myer. После отмены назначения политики пользователь Ken Myer будет автоматически управляться глобальной политикой или, если такая существует, локальной политикой сайта. Политика сайта имеет более высокий приоритет, чем глобальная политика.

  ```PowerShell
  Grant-CsPinPolicy -Identity "Ken Myer" -PolicyName $Null
  ```

Дополнительные сведения см [. в разделе Grant-CsPinPolicy](/powershell/module/skype/grant-cspinpolicy).
