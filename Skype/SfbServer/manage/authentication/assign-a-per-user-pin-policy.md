# <a name="assign-a-per-user-pin-policy-in-skype-for-business-server"></a>Назначение политики ПИН-кодов на пользователя в Skype для бизнеса Server

**Сводка:** Stage AV and OAuth certificates for Skype for Business Server.
  
Политика персонального идентификационного номера (ПИН-кода) для телефонной сети — это один из индивидуальных параметров учетной записи пользователя, которую можно настроить на панели управления Skype для бизнеса Server.
  
Развертывание одной или более политик ПИН-кодов на пользователя является необязательным. Вы также можете развернуть только политику ПИН-кодов глобального уровня или политику ПИН-кодов на уровне сайта. При развертывании политик для отдельных пользователей необходимо явным образом назначить их пользователям, группам или контактному объекту. Права и разрешения пользователей, касающиеся использования ПИН-кодов для телефонной связи, автоматически по умолчанию имеют значения, определенные в политике ПИН-кодов глобального уровня, если не назначена определенная политика уровня сайта или пользователя.
  
После создания по крайней мере одной политики ПИН-кодов для отдельных пользователей используйте процедуры, указанные в этом разделе, чтобы назначить политику, которая определяет ограничения, которые сервер должен налагать на ПИН-коды, созданные и используемые определенным пользователем.
  
### <a name="to-assign-a-per-user-pin-policy"></a>Назначение политики ПИН-кодов на пользователя

1. Войдите на любой компьютер во внутреннем развертывании с использованием учетной записи пользователя, назначенной роли CsUserAdministrator или CsAdministrator.
    
2. Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server.  
    
3. В левой панели навигации щелкните **Пользователи**.
    
4. Используйте один из следующих методов для поиска пользователя:
    
   - В поле **Поиск пользователей** введите полное отображаемое имя или его часть, имя, фамилию, имя учетной записи диспетчер учетных записей безопасности (SAM), SIP-адрес или URI учетной записи пользователя, а затем нажмите кнопку **Найти**.
    
   - Если вы сохранили запрос, щелкните значок **Открыть запрос**, используйте диалоговое окно **Открыть**, чтобы получить запрос (USF-файл), а затем нажмите кнопку **Найти**.
    
5. (Необязательно) Укажите дополнительные критерии поиска, чтобы сузить результаты:
    
   а. Нажмите кнопку **Добавить фильтр**.
    
   б. Введите свойство пользователя, набрав его или нажав кнопку стрелки в раскрывающемся списке, чтобы выбрать свойство.
    
   c. В раскрывающемся списке **Равно** щелкните оператор (например, **Равно** или **Не равно**).
    
   г. В зависимости от выбранного свойства пользователя введите критерии, которые необходимо использовать для фильтрации результатов поиска; для это введите его или щелкните стрелку в раскрывающемся списке.
    
    > [!TIP]
    > Чтобы добавить дополнительные пункты в запрос поиска, нажмите кнопку **Добавить фильтр**. 
  
   д. Нажмите кнопку **Найти**.
    
6. Выберите пользователя в списке результатов поиска, щелкните **Действие**, а затем щелкните **Назначить политики**.
    
    > [!TIP]
    > Если вы хотите, чтобы к нескольким пользователям применялись одинаковые политики ПИН-кодов на уровне пользователей, выберите несколько пользователей в результатах поиска, затем щелкните "Действия" и выберите "Назначить **политики".**  
  
7. В **области "Назначить политики"** в **политике ПИН-кодов** сделайте одно из следующих:
    
    > [!NOTE]
    > Поскольку существует несколько политик, которые можно настроить  с помощью диалоговых окна "Назначение политик", по умолчанию выбраны для каждой политики в диалоговом **\<Keep as is\>** окне. Чтобы продолжить использование политики, назначенной пользователю ранее, не вносите никаких изменений в эти настройки.
  
   - Разрешить Skype для бизнеса Server автоматически выбирать политику глобального уровня или, если она определена, политику уровня сайта.
    
   - Щелкните имя политики ПИН-кодов на пользователя, ранее определенной на странице **"Политика ПИН-кодов".**
    
     > [!TIP]
     > Чтобы с легкостью определить политику, которую необходимо назначить, после выбора названия политики щелкните **Просмотр** для просмотра прав и разрешений пользователя, заданных в политике.
  
8. По завершении нажмите кнопку **ОК**.
    
## <a name="assigning-a-per-user-pin-policy-by-using-windows-powershell-cmdlets"></a>Назначение политики ПИН-Per-User с помощью Windows PowerShell

Политики ПИН-кодов для каждого пользователя можно назначить с Windows PowerShell и с помощью Windows PowerShell **Grant-CsPinPolicy.** Вы можете запустить этот командлет из командной консоли Skype для бизнеса Server или из удаленного сеанса Windows PowerShell. Подробные сведения об использовании удаленного Windows PowerShell для подключения к Skype для бизнеса Server см. в статье [блога "Краткое руководство. Управление Microsoft Lync Server 2010 с помощью удаленной службы PowerShell".](https://go.microsoft.com/fwlink/p/?linkId=255876) В Skype для бизнеса Server этот процесс тот же.
  
### <a name="to-assign-a-per-user-pin-policy-to-a-single-user"></a>Назначение политики ПИН-кодов на пользователя одному пользователю

- Следующая команда назначает политику ПИН-кодов на пользователя RedmondPinPolicy пользователю Ken Myer.
    
  ```PowerShell
  Grant-CsPinPolicy -Identity "Ken Myer" -PolicyName "RedmondPinPolicy"
  ```

### <a name="to-assign-a-per-user-pin-policy-to-multiple-users"></a>Назначение политики ПИН-кодов на пользователя нескольким пользователям

- Следующая команда назначает политику ПИН-кодов на пользователя RedmondUsersPinPolicy всем пользователям, которые работают в городе Редмонд. Подробные сведения о параметре LdapFilter, используемом в этой команде, см. в командной [команде Get-CsUser.](https://docs.microsoft.com/powershell/module/skype/get-csuser?view=skype-ps)
    
  ```PowerShell
  Get-CsUser -LdapFilter "l=Redmond" | Grant-CsPinPolicy -PolicyName "RedmondUsersPinPolicy"
  ```

### <a name="to-unassign-a-per-user-pin-policy"></a>Чтобы отоименовать политику ПИН-кодов на пользователя

- Следующая команда открепит любую политику ПИН-кодов на пользователя, ранее назначенную пользователю Ken Myer. После отмены назначения политики пользователь Ken Myer будет автоматически управляться глобальной политикой или, если такая существует, локальной политикой сайта. Политика сайта имеет более высокий приоритет, чем глобальная политика.
    
  ```PowerShell
  Grant-CsPinPolicy -Identity "Ken Myer" -PolicyName $Null
  ```

Подробные сведения [см. в поле Grant-CsPinPolicy.](https://docs.microsoft.com/powershell/module/skype/grant-cspinpolicy?view=skype-ps)
