# <a name="manage-server-to-server-authentication-oauth-and-partner-applications-in-skype-for-business-server"></a><span data-ttu-id="c27f9-103">Управление проверкой подлинности серверов (OAuth) и партнерских приложений в Skype для бизнеса Server</span><span class="sxs-lookup"><span data-stu-id="c27f9-103">Manage server-to-server authentication (OAuth) and partner applications in Skype for Business Server</span></span>
 
<span data-ttu-id="c27f9-104">**Сводка:** Управляйте приложениями OAuth и партнерским приложением в Skype для бизнеса Server.</span><span class="sxs-lookup"><span data-stu-id="c27f9-104">**Summary:** Manage OAuth and partner applications in Skype for Business Server.</span></span>
  
<span data-ttu-id="c27f9-105">Skype для бизнеса Server должен обеспечивать надежную и полную связь с другими приложениями и серверными продуктами.</span><span class="sxs-lookup"><span data-stu-id="c27f9-105">Skype for Business Server must be able to securely, and seamlessly, communicate with other applications and server products.</span></span> <span data-ttu-id="c27f9-106">Например, вы можете настроить Skype для бизнеса Server так, чтобы контактные данные и (или) данные архивации хранились на сервере Microsoft Exchange Server 2013; Тем не менее, это можно сделать только в том случае, если Skype для бизнеса Server и Exchange смогут безопасно общаться друг с другом.</span><span class="sxs-lookup"><span data-stu-id="c27f9-106">For example, you can configure Skype for Business Server so that contact data and/or archiving data is stored in Microsoft Exchange Server 2013; however, this can only be done if Skype for Business Server and Exchange are able to securely communicate with one another.</span></span> <span data-ttu-id="c27f9-107">Кроме того, вы можете запланировать конференцию Skype для бизнеса Server в Office Web Apps Server. Это можно сделать только в том случае, если два сервера (SharePoint и Skype для бизнеса Server) доверяют друг другу.</span><span class="sxs-lookup"><span data-stu-id="c27f9-107">Likewise, you can schedule a Skype for Business Server conference from within Office Web Apps Server; again, this can only be done if the two servers (SharePoint and Skype for Business Server) trust one another.</span></span> <span data-ttu-id="c27f9-108">Несмотря на то, что вы можете использовать один механизм проверки подлинности для обмена информацией между Skype для бизнеса Server и Exchange, но отдельный механизм для обмена данными между Skype для бизнеса и SharePoint, более эффективный подход — использование стандартизированный метод для проверки подлинности и авторизации сервера и сервера.</span><span class="sxs-lookup"><span data-stu-id="c27f9-108">Although it's possible to use one authentication mechanism for communication between Skype for Business Server and Exchange but a separate mechanism for Skype for Business Server and SharePoint communication, a better and more efficient approach is to use a standardized method for all server-to-server authentication and authorization.</span></span>
  
<span data-ttu-id="c27f9-109">Использование единого стандартного метода для проверки подлинности между сервером — это подход, выполняемый в Skype для бизнеса Server.</span><span class="sxs-lookup"><span data-stu-id="c27f9-109">Using a single, standardized method for server-to-server authentication is the approach taken by Skype for Business Server.</span></span> <span data-ttu-id="c27f9-110">Работа с серверами Office 2013 Release, Skype для бизнеса Server (а также другие серверные продукты Майкрософт, в том числе Exchange Server и SharePoint Server), поддерживающие протокол OAuth (открытая авторизация) для проверки подлинности серверов и сервера прав.</span><span class="sxs-lookup"><span data-stu-id="c27f9-110">Started with Office Servers 2013 release, Skype for Business Server (as well as other Microsoft Server products, including Exchange Server and SharePoint Server) supported the OAuth (Open Authorization) protocol for server-to-server authentication and authorization.</span></span> <span data-ttu-id="c27f9-111">С помощью OAuth стандартным протоколом авторизации, используемым для ряда основных сайтов, учетные данные пользователя и пароли не передаются с одного компьютера на другой.</span><span class="sxs-lookup"><span data-stu-id="c27f9-111">With OAuth, a standard authorization protocol used by a number of major websites, user credentials and passwords are not passed from one computer to another.</span></span> <span data-ttu-id="c27f9-112">Вместо этого проверка подлинности и авторизация основаны на обмене маркерами безопасности, которые предоставляют доступ к определенному набору ресурсов в течение определенного промежутка времени.</span><span class="sxs-lookup"><span data-stu-id="c27f9-112">Instead, authentication and authorization is based on the exchange of security tokens; these tokens grant access to a specific set of resources for a specific amount of time.</span></span>
  
<span data-ttu-id="c27f9-113">Как правило, проверка подлинность OAuth включает три компонента: отдельный сервер авторизации и две области, которые должны взаимодействовать друг с другом.</span><span class="sxs-lookup"><span data-stu-id="c27f9-113">OAuth authentication typically involves three parties: a single authorization server and the two realms that need to communicate with one another.</span></span> <span data-ttu-id="c27f9-114">(Кроме того, можно выполнить проверку подлинности "сервер-сервер" без использования сервера авторизации — процесс, который будет обсуждаться ниже в этом документе.) Маркеры безопасности выдаются сервером авторизации (также называемым сервером маркеров безопасности) в две сферы, которые должны поддерживать связь; Эти маркеры проверки того, что связь, исходящие из одной сферы, должна быть доверенной для другой сферы.</span><span class="sxs-lookup"><span data-stu-id="c27f9-114">(You can also do server-to-server authentication without using an authorization server, a process that will be discussed later in this document.) Security tokens are issued by the authorization server (also known as a security token server) to the two realms that need to communicate; these tokens verify that communications originating from one realm should be trusted by the other realm.</span></span> <span data-ttu-id="c27f9-115">Например, сервер авторизации может выдавать маркеры, которые подтверждают, что пользователи из конкретной сферы Skype для бизнеса Server могут получить доступ к указанной области Exchange и наоборот.</span><span class="sxs-lookup"><span data-stu-id="c27f9-115">For example, the authorization server might issue tokens that verify that users from a specific Skype for Business Server realm are able to access a specified Exchange realm, and vice-versa.</span></span>
  
> [!NOTE]
> <span data-ttu-id="c27f9-116">Область представляет собой просто контейнер безопасности.</span><span class="sxs-lookup"><span data-stu-id="c27f9-116">A realm is simply a security container.</span></span> <span data-ttu-id="c27f9-117">По умолчанию в Skype для бизнеса Server используется домен SIP по умолчанию в качестве сферы OAuth.</span><span class="sxs-lookup"><span data-stu-id="c27f9-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="c27f9-118">В список "Альтернативное имя субъекта" в сертификате OAuth добавляются дополнительные пространства имен SIP.</span><span class="sxs-lookup"><span data-stu-id="c27f9-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span> 
  
<span data-ttu-id="c27f9-119">Skype для бизнеса Server поддерживает три сценария проверки подлинности между серверами.</span><span class="sxs-lookup"><span data-stu-id="c27f9-119">Skype for Business Server supports three server-to-server authentication scenarios.</span></span> <span data-ttu-id="c27f9-120">С помощью Skype для бизнеса Server вы можете выполнять следующие действия:</span><span class="sxs-lookup"><span data-stu-id="c27f9-120">With Skype for Business Server, you can:</span></span>
  
- <span data-ttu-id="c27f9-121">Настройка проверки подлинности серверов (сервер-сервер) между локальной установкой Skype для бизнеса Server и локальной установкой Exchange и сервера SharePoint.</span><span class="sxs-lookup"><span data-stu-id="c27f9-121">Configure server-to-server authentication between an on-premises installation of Skype for Business Server and an on-premises installation of Exchange and/or SharePoint Server.</span></span>
    
- <span data-ttu-id="c27f9-122">Настройка проверки подлинности серверов между парой компонентов Office 365 (например, с сервера Microsoft Exchange Server и Skype для бизнеса Server или между Skype для бизнеса Server и SharePoint).</span><span class="sxs-lookup"><span data-stu-id="c27f9-122">Configure server-to-server authentication between a pair of Office 365 components (for example, between Microsoft Exchange Server and Skype for Business Server, or between Skype for Business Server and SharePoint).</span></span>
    
- <span data-ttu-id="c27f9-123">Настройка проверки подлинности серверов в разных локальных средах (то есть для проверки подлинности серверов между локальным сервером и компонентом Office 365).</span><span class="sxs-lookup"><span data-stu-id="c27f9-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and an Office 365 component).</span></span>
    
<span data-ttu-id="c27f9-124">Обратите внимание, что в данный момент времени только Exchange 2013, SharePoint Server, Lync Server 2013, Skype для бизнеса Server 2015 и Skype для бизнеса 2019 поддерживают проверку подлинности серверов и серверов; Если вы не используете один из этих серверов, вы не сможете полностью реализовать проверку подлинности OAuth.</span><span class="sxs-lookup"><span data-stu-id="c27f9-124">Note that, at this point in time, only Exchange 2013, SharePoint Server, Lync Server 2013, Skype for Business Server 2015, and Skype for Business 2019 support server-to-server authentication; if you are not running one of these servers, you will not be able to fully implement OAuth authentication.</span></span>
  
<span data-ttu-id="c27f9-125">Кроме того, проверка подлинности "сервер-сервер" не является обязательной. Если в Skype для бизнеса Server не требуется взаимодействовать с другими серверами (например, Exchange), проверка подлинности сервера и сервера может быть проигнорирована.</span><span class="sxs-lookup"><span data-stu-id="c27f9-125">It should also be pointed out that server-to-server authentication is optional: If Skype for Business Server does not need to communicate with other servers (such as Exchange) then server-to-server authentication can be skipped altogether.</span></span> <span data-ttu-id="c27f9-126">Если проверка подлинности "сервер-сервер" уже настроена для Lync Server 2013 и других приложений, вам не нужно повторно выполнять ее для Skype для бизнеса Server.</span><span class="sxs-lookup"><span data-stu-id="c27f9-126">If server-to-server authentication is already configured for Lync Server 2013 and other applications, there's no need to re-do it for Skype for Business Server.</span></span> 
  
<span data-ttu-id="c27f9-127">Однако проверка подлинности "сервер-сервер" необходима для использования некоторых функций в Skype для бизнеса Server, например "единое хранилище контактов".</span><span class="sxs-lookup"><span data-stu-id="c27f9-127">However, server-to-server authentication is required if you want to use some of the features in Skype for Business Server, such as the "unified contact store."</span></span> <span data-ttu-id="c27f9-128">В едином магазине контактов контактные данные Skype для бизнеса Server хранятся в Exchange, а не в Skype для бизнеса Server. Это позволяет пользователям использовать один набор контактов, доступ к которому можно получить в Skype для бизнеса, Outlook или Outlook Web Access.</span><span class="sxs-lookup"><span data-stu-id="c27f9-128">With unified contact store, Skype for Business Server contact information is stored in Exchange instead of in Skype for Business Server; this enables users to have a single set of contacts that is readily accessible from within Skype for Business, Outlook, or Outlook Web Access.</span></span> <span data-ttu-id="c27f9-129">Так как единое хранилище контактов требует от Skype для бизнеса Server предоставления доступа к данным в Exchange, необходимо использовать проверку подлинности серверов для сервера, чтобы развернуть ее.</span><span class="sxs-lookup"><span data-stu-id="c27f9-129">Because the unified contact store requires Skype for Business Server to share information with Exchange, you must use server-to-server authentication in order to deploy the feature.</span></span> <span data-ttu-id="c27f9-130">Проверка подлинности "сервер-сервер" также необходима, если вы решили использовать архивацию Exchange, в которой записи в сеансах обмена мгновенными сообщениями сохраняются как сообщения электронной почты Exchange, а не в виде отдельных записей базы данных.</span><span class="sxs-lookup"><span data-stu-id="c27f9-130">Server-to-server authentication is also required if you choose to use Exchange archiving, in which the transcripts of instant messaging sessions are saved as Exchange emails rather than as individual database records.</span></span>
  
<span data-ttu-id="c27f9-131">В версии Office 365 для Skype для бизнеса Server для связи с аналогом Exchange Server необходимо сначала получить маркер безопасности от сервера авторизации.</span><span class="sxs-lookup"><span data-stu-id="c27f9-131">For the Office 365 version of Skype for Business Server to communicate with its Exchange counterpart, Skype for Business Server must first obtain a security token from the authorization server.</span></span> <span data-ttu-id="c27f9-132">Skype для бизнеса Server затем использует этот маркер безопасности для идентификации себя в Exchange.</span><span class="sxs-lookup"><span data-stu-id="c27f9-132">Skype for Business Server then uses that security token to identify itself to Exchange.</span></span> <span data-ttu-id="c27f9-133">Для связи с Skype для бизнеса Server версия Exchange для Office 365 должна пройти один и тот же процесс.</span><span class="sxs-lookup"><span data-stu-id="c27f9-133">The Office 365 version of Exchange must go through the same process in order to communicate with Skype for Business Server.</span></span>
  
<span data-ttu-id="c27f9-134">Однако для локальной межсерверной проверки подлинности, осуществляемой между двумя серверами Майкрософт, использование стороннего сервера маркеров.</span><span class="sxs-lookup"><span data-stu-id="c27f9-134">However, for on-premises server-to-server authentication between two Microsoft servers there is no need to use a third-party token server.</span></span> <span data-ttu-id="c27f9-135">Серверные продукты, такие как Skype для бизнеса Server и Exchange, имеют встроенный сервер маркеров, который можно использовать для проверки подлинности на других серверах Майкрософт (например, SharePoint Server), поддерживающих проверку подлинности сервера и сервера.</span><span class="sxs-lookup"><span data-stu-id="c27f9-135">Server products such as Skype for Business Server and Exchange have a built-in token server that can be used for authentication purposes with other Microsoft servers (such as SharePoint Server) that support server-to-server authentication.</span></span> <span data-ttu-id="c27f9-136">Например, Skype для бизнеса Server может выпустить и подписать маркер безопасности самостоятельно, а затем использовать этот маркер для связи с Exchange.</span><span class="sxs-lookup"><span data-stu-id="c27f9-136">For example, Skype for Business Server can issue and sign a security token by itself, then use that token to communicate with Exchange.</span></span> <span data-ttu-id="c27f9-137">В таком случае использование стороннего сервера маркеров не требуется.</span><span class="sxs-lookup"><span data-stu-id="c27f9-137">In a case like this, there is no need for a third-party token server.</span></span>
  
<span data-ttu-id="c27f9-138">Чтобы настроить проверку подлинности серверов для локальной реализации Skype для бизнеса Server, необходимо выполнить два действия.</span><span class="sxs-lookup"><span data-stu-id="c27f9-138">In order to configure server-to-server authentication for an on-premises implementation of Skype for Business Server, you must do two things:</span></span>
  
- <span data-ttu-id="c27f9-139">Назначьте сертификат встроенному издателю маркеров Skype для бизнеса Server.</span><span class="sxs-lookup"><span data-stu-id="c27f9-139">Assign a certificate to the built-in Skype for Business Server token issuer.</span></span>
    
- <span data-ttu-id="c27f9-140">Настройте сервер, с которым будет взаимодействовать Skype для бизнеса Server, как "партнерское приложение".</span><span class="sxs-lookup"><span data-stu-id="c27f9-140">Configure the server that Skype for Business Server will communicate with to be a "partner application."</span></span> <span data-ttu-id="c27f9-141">Например, если Skype для бизнеса Server должен взаимодействовать с Exchange, вам потребуется настроить Exchange для работы в качестве партнерского приложения.</span><span class="sxs-lookup"><span data-stu-id="c27f9-141">For example, if Skype for Business Server needs to communicate with Exchange, you will need to configure Exchange to be a partner application.</span></span>
