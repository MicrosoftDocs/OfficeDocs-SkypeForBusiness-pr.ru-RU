# <a name="manage-server-to-server-authentication-oauth-and-partner-applications-in-skype-for-business-server"></a>Управление проверкой подлинности "сервер-сервер" (OAuth) и партнерскими приложениями в Skype для бизнеса Server
 
**Сводка:** Управление OAuth и партнерскими приложениями в Skype для бизнеса Server.
  
Skype для бизнеса Server должен быть способен безопасно и легко взаимодействовать с другими приложениями и серверными продуктами. Например, можно настроить Skype для бизнеса Server таким образом, чтобы контактные данные и/или архивные данные хранились в Microsoft Exchange Server 2013; Однако это можно сделать, только если Skype для бизнеса Server и Exchange могут безопасно взаимодействовать друг с другом. Аналогичным образом можно запланировать конференцию Skype для бизнеса Server на сервере Office Web Apps; Опять же, это можно сделать, только если два сервера (SharePoint и Skype для бизнеса Server) доверяют друг другу. Хотя можно использовать один механизм проверки подлинности для связи между Skype для бизнеса Server и Exchange, но отдельный механизм для связи Skype для бизнеса Server и SharePoint, лучше и эффективнее использовать стандартизированный метод для проверки подлинности и авторизации между серверами.
  
Использование единого стандартизированного метода проверки подлинности между серверами — это подход, принятый Skype для бизнеса Server. На момент выпуска Office Server 2013 Skype для бизнеса Server (а также другие продукты Microsoft Server, в том числе Exchange Server и SharePoint Server) поддерживал протокол OAuth (Open Authorization) для проверки подлинности и авторизации между серверами. С помощью OAuth стандартный протокол авторизации, используемый рядом крупных веб-сайтов, учетные данные и пароли пользователей не передаются с одного компьютера на другой. Вместо этого проверка подлинности и авторизация основаны на обмене маркерами безопасности, которые предоставляют доступ к определенному набору ресурсов в течение определенного промежутка времени.
  
Как правило, проверка подлинность OAuth включает три компонента: отдельный сервер авторизации и две области, которые должны взаимодействовать друг с другом. (Вы также можете использовать проверку подлинности "сервер-сервер" без использования сервера авторизации, который будет рассмотрен далее в этом документе.) Маркеры безопасности выдаются сервером авторизации (также известным как сервер маркеров безопасности) двум областям, которые должны взаимодействовать; эти маркеры проверяют, что связь из одной области должна быть доверяема другой области. Например, сервер авторизации может выдавать маркеры, которые проверяют, могут ли пользователи из определенной области Skype для бизнеса Server получить доступ к указанной области Exchange и наоборот.
  
> [!NOTE]
> Область представляет собой просто контейнер безопасности. По умолчанию Skype для бизнеса Server использует домен SIP по умолчанию в качестве области OAuth. Дополнительные пространства имен SIP добавляются в список альтернативных имен субъектов в сертификате OAuth. 
  
Skype для бизнеса Server поддерживает три сценария проверки подлинности "сервер-сервер". С помощью Skype для бизнеса Server вы можете:
  
- Настройка проверки подлинности "сервер-сервер" между локальной установкой Skype для бизнеса Server и локальной установкой Exchange и/или SharePoint Server.
    
- Настройте проверку подлинности "сервер-сервер" между парой компонентов Microsoft 365 или Office 365 (например, между Microsoft Exchange Server и Skype для бизнеса Server или между Skype для бизнеса Server и SharePoint).
    
- Настройка проверки подлинности "сервер-сервер" в локальной среде (то есть меж серверной проверки подлинности между локального сервера и компонентом Microsoft 365 или Office 365).
    
Обратите внимание, что на данный момент только Exchange 2013, SharePoint Server, Lync Server 2013, Skype для бизнеса Server 2015 и Skype для бизнеса 2019 поддерживают проверку подлинности "сервер-сервер"; Если вы не работаете на одном из этих серверов, вы не сможете полностью реализовать проверку подлинности OAuth.
  
Следует также у отметить, что проверка подлинности "сервер-сервер" является необязательной: если Skype для бизнеса Server не требуется взаимодействовать с другими серверами (например, Exchange), то проверку подлинности "сервер-сервер" можно полностью пропустить. Если проверка подлинности "сервер-сервер" уже настроена для Lync Server 2013 и других приложений, нет необходимости ее повторно делать для Skype для бизнеса Server. 
  
Однако проверка подлинности "сервер-сервер" необходима, если вы хотите использовать некоторые функции Skype для бизнеса Server, такие как "единое хранилище контактов". При едином хранилище контактов контактные данные Skype для бизнеса Server хранятся в Exchange, а не в Skype для бизнеса Server; Это позволяет пользователям иметь один набор контактов, доступный из Skype для бизнеса, Outlook или Outlook Web Access. Поскольку единое хранилище контактов требует, чтобы Skype для бизнеса Server обменивался информацией с Exchange, для развертывания этой функции необходимо использовать проверку подлинности "сервер-сервер". Проверка подлинности "сервер-сервер" также требуется, если вы решили использовать архивацию Exchange, в которой записи сеансов обмена мгновенными сообщениями сохраняются как сообщения электронной почты Exchange, а не как отдельные записи базы данных.
  
Чтобы skype для бизнеса Server версии Microsoft 365 или Office 365 взаимодействовал с сервером Exchange, Skype для бизнеса Server должен сначала получить маркер безопасности от сервера авторизации. Затем Skype для бизнеса Server использует этот маркер безопасности для идентификации себя в Exchange. Для связи со Skype для бизнеса Server версии Exchange в Microsoft 365 или Office 365 должны пройти тот же процесс.
  
Однако для локальной межсерверной проверки подлинности, осуществляемой между двумя серверами Майкрософт, использование стороннего сервера маркеров. Серверные продукты, такие как Skype для бизнеса Server и Exchange, имеют встроенный сервер маркеров, который можно использовать для проверки подлинности с другими серверами Майкрософт (например, SharePoint Server), которые поддерживают проверку подлинности "сервер-сервер". Например, Skype для бизнеса Server может самостоятельно выдавать и подписывать маркер безопасности, а затем использовать этот маркер для связи с Exchange. В таком случае использовать сторонний сервер токенов не нужно.
  
Чтобы настроить проверку подлинности "сервер-сервер" для локальной реализации Skype для бизнеса Server, необходимо сделать два вещи:
  
- Назначьте сертификат встроенной проблеме с маркерами Skype для бизнеса Server.
    
- Настройте сервер, с помощью который Skype для бизнеса Server будет взаимодействовать, в качестве "партнерского приложения". Например, если Skype для бизнеса Server должен взаимодействовать с Exchange, необходимо настроить Exchange в качестве партнерского приложения.
    
> [!NOTE]
> "Партнерская программа" — это любое приложение, с помощью которое Skype для бизнеса Server может напрямую обмениваться маркерами безопасности без необходимости проходить через сторонний сервер маркеров безопасности. 
  
Обратите внимание, что OAuth является основной частью продукта и не может быть отключен или удален.
