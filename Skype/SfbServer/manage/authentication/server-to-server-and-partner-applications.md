# <a name="manage-server-to-server-authentication-oauth-and-partner-applications-in-skype-for-business-server"></a>Управление проверкой подлинности между серверами (OAuth) и партнерских приложений в Skype для бизнеса Server
 
**Сводка:** Управление приложениями OAuth и партнерским приложением в Skype для бизнеса Server.
  
Skype для бизнеса Server должен иметь возможность безопасно и легко взаимодействовать с другими приложениями и серверными продуктами. Например, вы можете настроить Skype для бизнеса Server так, чтобы данные контактов и/или архивные данные хранились в Microsoft Exchange Server 2013; Тем не менее, это можно сделать только в том случае, если Skype для бизнеса Server и Exchange могут безопасно общаться друг с другом. Аналогичным образом можно запланировать конференцию Skype для бизнеса Server в сервер Office Web Apps. Опять же, это можно сделать только в том случае, если два сервера (SharePoint и Skype для бизнеса Server) доверяют друг другу. Несмотря на то, что можно использовать один механизм проверки подлинности для обмена данными между Skype для бизнеса Server и Exchange, но отдельный механизм для обмена данными между Skype для бизнеса Server и SharePoint, лучший и эффективный подход — использовать стандартизированный метод для всех проверок подлинности и авторизации "сервер-сервер".
  
Один стандартизированный метод проверки подлинности "сервер-сервер" является подходом, выполняемым Skype для бизнеса Server. Начало работы с серверами Office 2013, Skype для бизнеса Server (а также другими серверными продуктами Майкрософт, в том числе Exchange Server и SharePoint Server), поддерживающих протокол OAuth (открытая авторизация) для проверки подлинности и авторизации "сервер-сервер". При использовании OAuth стандартным протоколом авторизации, используемым для ряда основных сайтов, учетные данные пользователя и пароли не передаются с одного компьютера на другой. Вместо этого проверка подлинности и авторизация основаны на обмене маркерами безопасности, которые предоставляют доступ к определенному набору ресурсов в течение определенного промежутка времени.
  
Как правило, проверка подлинность OAuth включает три компонента: отдельный сервер авторизации и две области, которые должны взаимодействовать друг с другом. (Вы также можете выполнять проверку подлинности "сервер-сервер", не используя сервер авторизации, процесс, который будет обсуждаться далее в этом документе.) Токены безопасности выдаются сервером авторизации (также называется сервером маркеров безопасности) в две сферы, которые необходимы для обмена сообщениями; Эти маркеры проверки того, что связь, поступающие из одной области, должна быть доверенной для другой области. Например, сервер авторизации может выдать маркеры, которые проверяют, что пользователи из определенной сферы Skype для бизнеса Server могут получить доступ к определенной области Exchange и наоборот.
  
> [!NOTE]
> Область представляет собой просто контейнер безопасности. По умолчанию Skype для бизнеса Server использует домен SIP по умолчанию в качестве области OAuth. Дополнительные пространства имен SIP добавляются в список альтернативных имен субъектов в сертификате OAuth. 
  
Skype для бизнеса Server поддерживает три сценария проверки подлинности между серверами. С помощью Skype для бизнеса Server вы можете:
  
- Настройка межсерверной проверки подлинности между локальной установкой Skype для бизнеса Server и локальной установкой Exchange и/или SharePoint Server.
    
- Настройка межсерверной проверки подлинности между двумя компонентами Microsoft 365 или Office 365 (например, между Microsoft Exchange Server и Skype для бизнеса Server или между Skype для бизнеса Server и SharePoint).
    
- Настройка межсерверной проверки подлинности в распределенной среде (то есть межсерверной проверкой подлинности между локальным сервером и компонентом Microsoft 365 или Office 365).
    
Обратите внимание, что в данный момент времени только Exchange 2013, SharePoint Server, Lync Server 2013, Skype для бизнеса Server 2015 и Skype для бизнеса 2019 поддерживают проверку подлинности между серверами; Если вы не используете один из этих серверов, вы не сможете полностью реализовать проверку подлинности OAuth.
  
Кроме того, следует обратить внимание на то, что проверка подлинности "сервер-сервер" необязательна: если Skype для бизнеса Server не требуется связываться с другими серверами (например, Exchange), то проверка подлинности "сервер-сервер" может быть полностью пропущена. Если проверка подлинности "сервер-сервер" уже настроена для Lync Server 2013 и других приложений, нет необходимости повторно делать это для Skype для бизнеса Server. 
  
Однако проверка подлинности "сервер-сервер" необходима, если вы хотите использовать некоторые функции Skype для бизнеса Server, например "единое хранилище контактов". В едином хранилище контактов контактные данные Skype для бизнеса Server хранятся в Exchange, а не в Skype для бизнеса Server; Это позволяет пользователям иметь единый набор контактов, доступных в Skype для бизнеса, Outlook или Outlook Web Access. Так как единое хранилище контактов требует от Skype для бизнеса Server совместно обмениваться информацией с Exchange, для развертывания этой функции необходимо использовать проверку подлинности "сервер-сервер". Проверка подлинности "сервер-сервер" также необходима, если вы решили использовать архивацию Exchange, в которой записи сеансов обмена мгновенными сообщениями сохраняются как сообщения Exchange, а не как отдельные записи базы данных.
  
Для обмена данными с аналогом Exchange для Microsoft 365 или Office 365 для Skype для бизнеса Server необходимо сначала получить маркер безопасности от сервера авторизации с помощью Skype для бизнеса Server. После этого Skype для бизнеса Server использует этот маркер безопасности для идентификации в Exchange. Для взаимодействия с Skype для бизнеса Server Microsoft 365 или Office 365 версии Exchange должны проходить аналогичный процесс.
  
Однако для локальной межсерверной проверки подлинности, осуществляемой между двумя серверами Майкрософт, использование стороннего сервера маркеров. Серверные продукты, такие как Skype для бизнеса Server и Exchange, имеют встроенный сервер маркеров, который можно использовать для проверки подлинности с другими серверами Майкрософт (например, SharePoint Server), поддерживающими проверку подлинности "сервер-сервер". Например, Skype для бизнеса Server может самостоятельно выдать и подписать маркер безопасности, а затем использовать этот маркер для обмена данными с Exchange. В таком случае использовать сторонний сервер токенов не нужно.
  
Чтобы настроить межсерверную проверку подлинности для локальной реализации Skype для бизнеса Server, необходимо выполнить два действия:
  
- Назначьте сертификат встроенному поставщику маркеров Skype для бизнеса Server.
    
- Настройте сервер, с которым будет общаться Skype для бизнеса Server, как "партнерское приложение". Например, если Skype для бизнеса Server должен обмениваться данными с Exchange, необходимо настроить Exchange в качестве партнерского приложения.
    
> [!NOTE]
> "Партнерское приложение" — это любое приложение, с помощью которого Skype для бизнеса Server может напрямую обмениваться маркерами безопасности с, без необходимости проходить от стороннего сервера маркеров безопасности. 
  
Обратите внимание, что OAuth является основной частью продукта и не может быть отключен или удален.
