# <a name="manage-server-to-server-authentication-oauth-and-partner-applications-in-skype-for-business-server"></a>Управление проверкой подлинности серверов (OAuth) и партнерских приложений в Skype для бизнеса Server
 
**Сводка:** Управляйте приложениями OAuth и партнерским приложением в Skype для бизнеса Server.
  
Skype для бизнеса Server должен обеспечивать надежную и полную связь с другими приложениями и серверными продуктами. Например, вы можете настроить Skype для бизнеса Server так, чтобы контактные данные и (или) данные архивации хранились на сервере Microsoft Exchange Server 2013; Тем не менее, это можно сделать только в том случае, если Skype для бизнеса Server и Exchange смогут безопасно общаться друг с другом. Кроме того, вы можете запланировать конференцию Skype для бизнеса Server в Office Web Apps Server. Это можно сделать только в том случае, если два сервера (SharePoint и Skype для бизнеса Server) доверяют друг другу. Несмотря на то, что вы можете использовать один механизм проверки подлинности для обмена информацией между Skype для бизнеса Server и Exchange, но отдельный механизм для обмена данными между Skype для бизнеса и SharePoint, более эффективный подход — использование стандартизированный метод для проверки подлинности и авторизации сервера и сервера.
  
Использование единого стандартного метода для проверки подлинности между сервером — это подход, выполняемый в Skype для бизнеса Server. Работа с серверами Office 2013 Release, Skype для бизнеса Server (а также другие серверные продукты Майкрософт, в том числе Exchange Server и SharePoint Server), поддерживающие протокол OAuth (открытая авторизация) для проверки подлинности серверов и сервера прав. С помощью OAuth стандартным протоколом авторизации, используемым для ряда основных сайтов, учетные данные пользователя и пароли не передаются с одного компьютера на другой. Вместо этого проверка подлинности и авторизация основаны на обмене маркерами безопасности, которые предоставляют доступ к определенному набору ресурсов в течение определенного промежутка времени.
  
Как правило, проверка подлинность OAuth включает три компонента: отдельный сервер авторизации и две области, которые должны взаимодействовать друг с другом. (Кроме того, можно выполнить проверку подлинности "сервер-сервер" без использования сервера авторизации — процесс, который будет обсуждаться ниже в этом документе.) Маркеры безопасности выдаются сервером авторизации (также называемым сервером маркеров безопасности) в две сферы, которые должны поддерживать связь; Эти маркеры проверки того, что связь, исходящие из одной сферы, должна быть доверенной для другой сферы. Например, сервер авторизации может выдавать маркеры, которые подтверждают, что пользователи из конкретной сферы Skype для бизнеса Server могут получить доступ к указанной области Exchange и наоборот.
  
> [!NOTE]
> Область представляет собой просто контейнер безопасности. По умолчанию в Skype для бизнеса Server используется домен SIP по умолчанию в качестве сферы OAuth. В список "Альтернативное имя субъекта" в сертификате OAuth добавляются дополнительные пространства имен SIP. 
  
Skype для бизнеса Server поддерживает три сценария проверки подлинности между серверами. С помощью Skype для бизнеса Server вы можете выполнять следующие действия:
  
- Настройка проверки подлинности серверов (сервер-сервер) между локальной установкой Skype для бизнеса Server и локальной установкой Exchange и сервера SharePoint.
    
- Настройка проверки подлинности серверов между парой компонентов Office 365 (например, с сервера Microsoft Exchange Server и Skype для бизнеса Server или между Skype для бизнеса Server и SharePoint).
    
- Настройка проверки подлинности серверов в разных локальных средах (то есть для проверки подлинности серверов между локальным сервером и компонентом Office 365).
    
Обратите внимание, что в данный момент времени только Exchange 2013, SharePoint Server, Lync Server 2013, Skype для бизнеса Server 2015 и Skype для бизнеса 2019 поддерживают проверку подлинности серверов и серверов; Если вы не используете один из этих серверов, вы не сможете полностью реализовать проверку подлинности OAuth.
  
Кроме того, проверка подлинности "сервер-сервер" не является обязательной. Если в Skype для бизнеса Server не требуется взаимодействовать с другими серверами (например, Exchange), проверка подлинности сервера и сервера может быть проигнорирована. Если проверка подлинности "сервер-сервер" уже настроена для Lync Server 2013 и других приложений, вам не нужно повторно выполнять ее для Skype для бизнеса Server. 
  
Однако проверка подлинности "сервер-сервер" необходима для использования некоторых функций в Skype для бизнеса Server, например "единое хранилище контактов". В едином магазине контактов контактные данные Skype для бизнеса Server хранятся в Exchange, а не в Skype для бизнеса Server. Это позволяет пользователям использовать один набор контактов, доступ к которому можно получить в Skype для бизнеса, Outlook или Outlook Web Access. Так как единое хранилище контактов требует от Skype для бизнеса Server предоставления доступа к данным в Exchange, необходимо использовать проверку подлинности серверов для сервера, чтобы развернуть ее. Проверка подлинности "сервер-сервер" также необходима, если вы решили использовать архивацию Exchange, в которой записи в сеансах обмена мгновенными сообщениями сохраняются как сообщения электронной почты Exchange, а не в виде отдельных записей базы данных.
  
В версии Office 365 для Skype для бизнеса Server для связи с аналогом Exchange Server необходимо сначала получить маркер безопасности от сервера авторизации. Skype для бизнеса Server затем использует этот маркер безопасности для идентификации себя в Exchange. Для связи с Skype для бизнеса Server версия Exchange для Office 365 должна пройти один и тот же процесс.
  
Однако для локальной межсерверной проверки подлинности, осуществляемой между двумя серверами Майкрософт, использование стороннего сервера маркеров. Серверные продукты, такие как Skype для бизнеса Server и Exchange, имеют встроенный сервер маркеров, который можно использовать для проверки подлинности на других серверах Майкрософт (например, SharePoint Server), поддерживающих проверку подлинности сервера и сервера. Например, Skype для бизнеса Server может выпустить и подписать маркер безопасности самостоятельно, а затем использовать этот маркер для связи с Exchange. В таком случае использование стороннего сервера маркеров не требуется.
  
Чтобы настроить проверку подлинности серверов для локальной реализации Skype для бизнеса Server, необходимо выполнить два действия.
  
- Назначьте сертификат встроенному издателю маркеров Skype для бизнеса Server.
    
- Настройте сервер, с которым будет взаимодействовать Skype для бизнеса Server, как "партнерское приложение". Например, если Skype для бизнеса Server должен взаимодействовать с Exchange, вам потребуется настроить Exchange для работы в качестве партнерского приложения.
    
> [!NOTE]
> "Приложение-партнер" — это любое приложение, с помощью которого Skype для бизнеса Server может напрямую обмениваться маркерами безопасности, не пробегая от стороннего сервера маркеров безопасности. 
  
Обратите внимание, что OAuth является основной частью продукта. Его нельзя ни отключить, ни удалить.
