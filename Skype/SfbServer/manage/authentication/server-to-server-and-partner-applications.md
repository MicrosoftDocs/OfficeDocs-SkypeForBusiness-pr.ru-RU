# <a name="manage-server-to-server-authentication-oauth-and-partner-applications-in-skype-for-business-server"></a><span data-ttu-id="55a45-103">Управление проверкой подлинности между серверами (OAuth) и партнерских приложений в Skype для бизнеса Server</span><span class="sxs-lookup"><span data-stu-id="55a45-103">Manage server-to-server authentication (OAuth) and partner applications in Skype for Business Server</span></span>
 
<span data-ttu-id="55a45-104">**Сводка:** Управление приложениями OAuth и партнерским приложением в Skype для бизнеса Server.</span><span class="sxs-lookup"><span data-stu-id="55a45-104">**Summary:** Manage OAuth and partner applications in Skype for Business Server.</span></span>
  
<span data-ttu-id="55a45-105">Skype для бизнеса Server должен иметь возможность безопасно и легко взаимодействовать с другими приложениями и серверными продуктами.</span><span class="sxs-lookup"><span data-stu-id="55a45-105">Skype for Business Server must be able to securely, and seamlessly, communicate with other applications and server products.</span></span> <span data-ttu-id="55a45-106">Например, вы можете настроить Skype для бизнеса Server так, чтобы данные контактов и/или архивные данные хранились в Microsoft Exchange Server 2013; Тем не менее, это можно сделать только в том случае, если Skype для бизнеса Server и Exchange могут безопасно общаться друг с другом.</span><span class="sxs-lookup"><span data-stu-id="55a45-106">For example, you can configure Skype for Business Server so that contact data and/or archiving data is stored in Microsoft Exchange Server 2013; however, this can only be done if Skype for Business Server and Exchange are able to securely communicate with one another.</span></span> <span data-ttu-id="55a45-107">Аналогичным образом можно запланировать конференцию Skype для бизнеса Server в сервер Office Web Apps. Опять же, это можно сделать только в том случае, если два сервера (SharePoint и Skype для бизнеса Server) доверяют друг другу.</span><span class="sxs-lookup"><span data-stu-id="55a45-107">Likewise, you can schedule a Skype for Business Server conference from within Office Web Apps Server; again, this can only be done if the two servers (SharePoint and Skype for Business Server) trust one another.</span></span> <span data-ttu-id="55a45-108">Несмотря на то, что можно использовать один механизм проверки подлинности для обмена данными между Skype для бизнеса Server и Exchange, но отдельный механизм для обмена данными между Skype для бизнеса Server и SharePoint, лучший и эффективный подход — использовать стандартизированный метод для всех проверок подлинности и авторизации "сервер-сервер".</span><span class="sxs-lookup"><span data-stu-id="55a45-108">Although it's possible to use one authentication mechanism for communication between Skype for Business Server and Exchange but a separate mechanism for Skype for Business Server and SharePoint communication, a better and more efficient approach is to use a standardized method for all server-to-server authentication and authorization.</span></span>
  
<span data-ttu-id="55a45-109">Один стандартизированный метод проверки подлинности "сервер-сервер" является подходом, выполняемым Skype для бизнеса Server.</span><span class="sxs-lookup"><span data-stu-id="55a45-109">Using a single, standardized method for server-to-server authentication is the approach taken by Skype for Business Server.</span></span> <span data-ttu-id="55a45-110">Начало работы с серверами Office 2013, Skype для бизнеса Server (а также другими серверными продуктами Майкрософт, в том числе Exchange Server и SharePoint Server), поддерживающих протокол OAuth (открытая авторизация) для проверки подлинности и авторизации "сервер-сервер".</span><span class="sxs-lookup"><span data-stu-id="55a45-110">Started with Office Servers 2013 release, Skype for Business Server (as well as other Microsoft Server products, including Exchange Server and SharePoint Server) supported the OAuth (Open Authorization) protocol for server-to-server authentication and authorization.</span></span> <span data-ttu-id="55a45-111">При использовании OAuth стандартным протоколом авторизации, используемым для ряда основных сайтов, учетные данные пользователя и пароли не передаются с одного компьютера на другой.</span><span class="sxs-lookup"><span data-stu-id="55a45-111">With OAuth, a standard authorization protocol used by a number of major websites, user credentials and passwords are not passed from one computer to another.</span></span> <span data-ttu-id="55a45-112">Вместо этого проверка подлинности и авторизация основаны на обмене маркерами безопасности, которые предоставляют доступ к определенному набору ресурсов в течение определенного промежутка времени.</span><span class="sxs-lookup"><span data-stu-id="55a45-112">Instead, authentication and authorization is based on the exchange of security tokens; these tokens grant access to a specific set of resources for a specific amount of time.</span></span>
  
<span data-ttu-id="55a45-113">Как правило, проверка подлинность OAuth включает три компонента: отдельный сервер авторизации и две области, которые должны взаимодействовать друг с другом.</span><span class="sxs-lookup"><span data-stu-id="55a45-113">OAuth authentication typically involves three parties: a single authorization server and the two realms that need to communicate with one another.</span></span> <span data-ttu-id="55a45-114">(Вы также можете выполнять проверку подлинности "сервер-сервер", не используя сервер авторизации, процесс, который будет обсуждаться далее в этом документе.) Токены безопасности выдаются сервером авторизации (также называется сервером маркеров безопасности) в две сферы, которые необходимы для обмена сообщениями; Эти маркеры проверки того, что связь, поступающие из одной области, должна быть доверенной для другой области.</span><span class="sxs-lookup"><span data-stu-id="55a45-114">(You can also do server-to-server authentication without using an authorization server, a process that will be discussed later in this document.) > [!NOTE]
> <span data-ttu-id="55a45-116">Область представляет собой просто контейнер безопасности.</span><span class="sxs-lookup"><span data-stu-id="55a45-116">A realm is simply a security container.</span></span> <span data-ttu-id="55a45-117">По умолчанию Skype для бизнеса Server использует домен SIP по умолчанию в качестве области OAuth.</span><span class="sxs-lookup"><span data-stu-id="55a45-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="55a45-118">Дополнительные пространства имен SIP добавляются в список альтернативных имен субъектов в сертификате OAuth.</span><span class="sxs-lookup"><span data-stu-id="55a45-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span> 
  
<span data-ttu-id="55a45-119">Skype для бизнеса Server поддерживает три сценария проверки подлинности между серверами.</span><span class="sxs-lookup"><span data-stu-id="55a45-119">Skype for Business Server supports three server-to-server authentication scenarios.</span></span> <span data-ttu-id="55a45-120">С помощью Skype для бизнеса Server вы можете:</span><span class="sxs-lookup"><span data-stu-id="55a45-120">With Skype for Business Server, you can:</span></span>
  
- <span data-ttu-id="55a45-121">Настройка межсерверной проверки подлинности между локальной установкой Skype для бизнеса Server и локальной установкой Exchange и/или SharePoint Server.</span><span class="sxs-lookup"><span data-stu-id="55a45-121">Configure server-to-server authentication between an on-premises installation of Skype for Business Server and an on-premises installation of Exchange and/or SharePoint Server.</span></span>
    
- <span data-ttu-id="55a45-122">Настройка межсерверной проверки подлинности между двумя компонентами Microsoft 365 или Office 365 (например, между Microsoft Exchange Server и Skype для бизнеса Server или между Skype для бизнеса Server и SharePoint).</span><span class="sxs-lookup"><span data-stu-id="55a45-122">Configure server-to-server authentication between a pair of Microsoft 365 or Office 365 components (for example, between Microsoft Exchange Server and Skype for Business Server, or between Skype for Business Server and SharePoint).</span></span>
    
- <span data-ttu-id="55a45-123">Настройка межсерверной проверки подлинности в распределенной среде (то есть межсерверной проверкой подлинности между локальным сервером и компонентом Microsoft 365 или Office 365).</span><span class="sxs-lookup"><span data-stu-id="55a45-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and a Microsoft 365 or Office 365 component).</span></span>
    
<span data-ttu-id="55a45-124">Обратите внимание, что в данный момент времени только Exchange 2013, SharePoint Server, Lync Server 2013, Skype для бизнеса Server 2015 и Skype для бизнеса 2019 поддерживают проверку подлинности между серверами; Если вы не используете один из этих серверов, вы не сможете полностью реализовать проверку подлинности OAuth.</span><span class="sxs-lookup"><span data-stu-id="55a45-124">Note that, at this point in time, only Exchange 2013, SharePoint Server, Lync Server 2013, Skype for Business Server 2015, and Skype for Business 2019 support server-to-server authentication; if you are not running one of these servers, you will not be able to fully implement OAuth authentication.</span></span>
  
<span data-ttu-id="55a45-125">Кроме того, следует обратить внимание на то, что проверка подлинности "сервер-сервер" необязательна: если Skype для бизнеса Server не требуется связываться с другими серверами (например, Exchange), то проверка подлинности "сервер-сервер" может быть полностью пропущена.</span><span class="sxs-lookup"><span data-stu-id="55a45-125">It should also be pointed out that server-to-server authentication is optional: If Skype for Business Server does not need to communicate with other servers (such as Exchange) then server-to-server authentication can be skipped altogether.</span></span> <span data-ttu-id="55a45-126">Если проверка подлинности "сервер-сервер" уже настроена для Lync Server 2013 и других приложений, нет необходимости повторно делать это для Skype для бизнеса Server.</span><span class="sxs-lookup"><span data-stu-id="55a45-126">If server-to-server authentication is already configured for Lync Server 2013 and other applications, there's no need to re-do it for Skype for Business Server.</span></span> 
  
<span data-ttu-id="55a45-127">Однако проверка подлинности "сервер-сервер" необходима, если вы хотите использовать некоторые функции Skype для бизнеса Server, например "единое хранилище контактов".</span><span class="sxs-lookup"><span data-stu-id="55a45-127">However, server-to-server authentication is required if you want to use some of the features in Skype for Business Server, such as the "unified contact store."</span></span> <span data-ttu-id="55a45-128">В едином хранилище контактов контактные данные Skype для бизнеса Server хранятся в Exchange, а не в Skype для бизнеса Server; <span data-ttu-id="55a45-131">Для обмена данными с аналогом Exchange для Microsoft 365 или Office 365 для Skype для бизнеса Server необходимо сначала получить маркер безопасности от сервера авторизации с помощью Skype для бизнеса Server.</span><span class="sxs-lookup"><span data-stu-id="55a45-131">For the Microsoft 365 or Office 365 version of Skype for Business Server to communicate with its Exchange counterpart, Skype for Business Server must first obtain a security token from the authorization server.</span></span> <span data-ttu-id="55a45-132">После этого Skype для бизнеса Server использует этот маркер безопасности для идентификации в Exchange.</span><span class="sxs-lookup"><span data-stu-id="55a45-132">Skype for Business Server then uses that security token to identify itself to Exchange.</span></span> <span data-ttu-id="55a45-133">Для взаимодействия с Skype для бизнеса Server Microsoft 365 или Office 365 версии Exchange должны проходить аналогичный процесс.</span><span class="sxs-lookup"><span data-stu-id="55a45-133">The Microsoft 365 or Office 365 versions of Exchange must go through the same process in order to communicate with Skype for Business Server.</span></span>
  
<span data-ttu-id="55a45-134">Однако для локальной межсерверной проверки подлинности, осуществляемой между двумя серверами Майкрософт, использование стороннего сервера маркеров.</span><span class="sxs-lookup"><span data-stu-id="55a45-134">However, for on-premises server-to-server authentication between two Microsoft servers there is no need to use a third-party token server.</span></span> <span data-ttu-id="55a45-135">Серверные продукты, такие как Skype для бизнеса Server и Exchange, имеют встроенный сервер маркеров, который можно использовать для проверки подлинности с другими серверами Майкрософт (например, SharePoint Server), поддерживающими проверку подлинности "сервер-сервер".</span><span class="sxs-lookup"><span data-stu-id="55a45-135">Server products such as Skype for Business Server and Exchange have a built-in token server that can be used for authentication purposes with other Microsoft servers (such as SharePoint Server) that support server-to-server authentication.</span></span> <span data-ttu-id="55a45-136">Например, Skype для бизнеса Server может самостоятельно выдать и подписать маркер безопасности, а затем использовать этот маркер для обмена данными с Exchange.</span><span class="sxs-lookup"><span data-stu-id="55a45-136">For example, Skype for Business Server can issue and sign a security token by itself, then use that token to communicate with Exchange.</span></span> <span data-ttu-id="55a45-137">В таком случае использовать сторонний сервер токенов не нужно.</span><span class="sxs-lookup"><span data-stu-id="55a45-137">In a case like this, there is no need for a third-party token server.</span></span>
  
<span data-ttu-id="55a45-138">Чтобы настроить межсерверную проверку подлинности для локальной реализации Skype для бизнеса Server, необходимо выполнить два действия:</span><span class="sxs-lookup"><span data-stu-id="55a45-138">In order to configure server-to-server authentication for an on-premises implementation of Skype for Business Server, you must do two things:</span></span>
