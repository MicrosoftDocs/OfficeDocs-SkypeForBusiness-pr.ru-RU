# <a name="managing-network-subnets-in-skype-for-business-server"></a>Управление подсетями в Skype для бизнеса Server

Можно использовать либо Скайп для панели управления Business Server или Скайп для консоли Business Server для управления подсетей. В большинстве развертываний Скайп для Business Server, где реализуется контроля допуска звонков (CAC) как правило будет большое число подсети. Таким образом чаще всего наиболее Настройка подсетей с Скайп для консоли Business Server.

Используйте разделы в этой статье для просмотра сведений о подсети или создание, изменение и удаление подсетей. 

## <a name="view-network-subnet-information"></a>Просмотр сведений о подсети 

Можно использовать следующую процедуру, чтобы просмотреть подсеть. Из Скайп для панели управления Business Server создание, изменение или удаление подсети. 

### <a name="to-view-a-network-subnet"></a>Чтобы просмотреть подсеть

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и выберите команду **подсети**.

4.  На странице **подсеть** щелкните подсеть, который требуется просмотреть.
 
    > [!NOTE]  
    > Можно просматривать только одну подсеть.

5.  В меню **Правка** щелкните **Подробнее**.

### <a name="view-network-subnet-configuration-information-by-using-windows-powershell-cmdlets"></a>Просмотр сведений о конфигурации подсети с командлетов с помощью Windows PowerShell

Можно просматривать сведения о подсети сети с помощью командлета Get-CsNetworkSubnet и Windows PowerShell. Этот командлет можно запустить из Скайп для консоли Business Server или из удаленного сеанса Windows PowerShell. 

### <a name="to-view-network-subnet-information"></a>Для просмотра сведений о подсети

  - Чтобы просмотреть сведения обо всех подсетях, введите следующую команду в Скайп для консоли Business Server и нажмите клавишу ВВОД:
    
        Get-CsNetworkSubnet
    
    Команда возвращает примерно следующую информацию:
    
        Identity      : 172.11.15.0
        MaskBits      : 28
        Description   :
        NetworkSiteID : Redmond
        SubnetID      : 172.11.15.0


Для получения дополнительных сведений см раздел справки для командлета [Get-CsNetworkSubnet](https://docs.microsoft.com/powershell/module/skype/Get-CsNetworkSubnet) .


## <a name="create-or-modify-network-subnets"></a>Создание или изменение подсетей 

Подсети должен быть связан с сетевым узлом в целях определения географическое положение узла, относящегося к данной подсети. Скайп для панели управления Business Server можно использовать для настройки подсети. Из Скайп для панели управления Business Server создание, изменение или удаление подсети. 

В большинстве развертываний Скайп для Business Server, где реализуется контроля допуска звонков (CAC) как правило будет большое число подсети. Таким образом чаще всего наиболее Настройка подсетей с Скайп для консоли Business Server. Отсюда можно вызвать **New-CsNetworkSubnet** в сочетании с помощью командлета Windows PowerShell **Import-CSV**. При совместном использовании эти командлеты можно читать в параметрах подсети из файла с разделителями-запятыми (CSV) и создавать несколько подсетей в то же время. Примеры того, как создание подсети из CSV-файла в разделе [New-CsNetworkSubnet](https://docs.microsoft.com/powershell/module/skype/New-CsNetworkSubnet).


### <a name="to-create-a-network-subnet"></a>Создание подсети

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и выберите команду **подсети**.

4.  На странице **подсеть** щелкните **Создать**.

5.  В **Новой подсети**введите значение в поле **Subnet ID** . Это должен быть IP-адрес (например, 174.11.12.0), и он должен быть первый адрес в диапазон IP-адресов, определенный в подсети.

6.  В поле **Mask** введите числовое значение от 1 до 32.

    > [!NOTE]  
    > Это значение — Битовая маска, применяемая к создаваемой подсети.

7.  В **ИД сетевого узла**выберите сайт, к которому относится данная подсеть.

8.  (Необязательно) Введите значение в поле **Описание** , чтобы предоставить дополнительную информацию о данной подсети, которые не могут быть выражены одним именем.

9.  Нажмите **Исполнить**.


### <a name="to-modify-a-network-subnet"></a>Изменение подсети

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и выберите команду **подсети**.

4.  На странице **подсеть** щелкните подсеть, которую требуется изменить.

5.  В меню **Правка** щелкните **Подробнее**.

6.  На странице " **Изменение подсети** " можно изменить Битовая маска, связанные с ними сетевой узел или описание. При изменении Битовая маска, которая помните, что идентификатор подсети по-прежнему должен быть первый адрес в диапазон IP-адресов, определенный в подсети.

7.  Нажмите **Исполнить**.

## <a name="delete-network-subnets"></a>Удаление подсетей

Можно использовать следующую процедуру для удаления подсети. Из Скайп для панели управления Business Server создание, изменение или удаление подсети. 

В большинстве развертываний Скайп для Business Server, где реализуется контроля допуска звонков (CAC) как правило будет большое число подсети. Таким образом чаще всего наиболее Настройка подсетей с Скайп для консоли Business Server. Отсюда можно вызвать **New-CsNetworkSubnet** в сочетании с помощью командлета Windows PowerShell **Import-CSV**. При совместном использовании эти командлеты можно читать в параметрах подсети из файла с разделителями-запятыми (CSV) и создавать несколько подсетей в то же время. Примеры того, как создание подсети из CSV-файла в разделе [New-CsNetworkSubnet](https://docs.microsoft.com/powershell/module/skype/New-CsNetworkSubnet).


### <a name="to-delete-a-network-subnet"></a>Удаление подсети

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и выберите команду **подсети**.

4.  На странице **подсеть** щелкните подсеть, которую требуется удалить.
 
    > [!NOTE]  
    > Одновременно можно удалить несколько подсетей. Для этого нажмите клавишу CTRL и выберите несколько подсетей, удерживая нажатой клавишу CTRL. Или, чтобы выбрать все подсети, нажмите кнопку **Выбрать все** в меню **Правка** .

5.  В меню **Правка** выберите пункт **Удалить**.

6.  Нажмите кнопку **ОК**.
