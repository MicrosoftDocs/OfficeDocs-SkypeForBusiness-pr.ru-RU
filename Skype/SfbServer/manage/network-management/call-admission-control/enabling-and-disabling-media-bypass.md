# <a name="enabling-and-disabling-media-bypass-in-skype-for-business-server"></a>Включение и отключение обхода сервера-посредника в Skype для бизнеса Server

Используйте процедуры, описанные в этой статье для включения или отключения сервера-посредника с помощью Скайп для панели управления Business Server.

## <a name="enable-network-media-bypass"></a>Включение сетевого сервера-посредника 

Параметры сервера-посредника мультимедиа применяются глобально через Скайп для развертывания Business Server. Обход сервера-посредника разрешает вызовы сервера-посредника. Для получения дополнительных сведений об использовании мультимедиа обходить, ознакомьтесь со статьей [Plan для мультимедиа обходить](../../../plan-your-deployment/enterprise-voice-solution/media-bypass.md).

Можно включить и настроить обход сервера-посредника из Скайп для панели управления Business Server.


### <a name="to-enable-and-configure-media-bypass"></a>Чтобы включить и настроить обход сервера-посредника

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и щелкните **Глобальная**.

4.  На странице **глобально** выберите **глобальную** конфигурацию. Всегда существует только одна конфигурация, и он всегда носит Global.

5.  В меню **Правка** щелкните **Показать подробности**.

6.  На странице **Изменение глобального параметра** установите флажок **Включить обход сервера-посредника** .

7.  Выберите один из следующих вариантов:
    
      - **Всегда пропускать**   установите этот флажок, чтобы попытаться мультимедиа обхода для всех звонков. Этот параметр будет недоступен, если этот параметр включен контроль допуска звонков (CAC). Если CAC не включен, выберите этот параметр, в следующих ситуациях:
        
          - Нет необходимости для управления пропускной способности.
        
          - Нет не требуется выполнять для когда должен происходить обход.
        
          - Есть подключение между клиентами и шлюзами.
    
      - **Использовать конфигурацию узлов и областей**   Если контроль допуска звонков включен, этот параметр установлен по умолчанию и не может быть изменено. Если выбран этот параметр, сетевой сайты и регионы конфигурации будет использоваться для определения возможности выполнения обхода мультимедиа. Если выбран этот параметр, можно включить обход сервера-посредника для сайтов, не сопоставлены. Установите флажок **Включить обход сервера-посредника для сайтов, не являющиеся сопоставляются** только в том случае, если у вас есть один или несколько крупных узлы, связанные с той же области, у которых нет ограничений пропускной способности (например, для крупных центрального сайта), а также некоторые сайты филиалов, связанных с же регионе, есть ли у ограничений пропускной способности. При включении обхода для сайтов не сопоставлены, конфигурации упрощается, так как указать только подсети, связанные с сайтами филиалов, а не требуется указать все подсети связанные со всеми сайтами. Мы рекомендуем не установите флажок **Включить обход сервера-посредника для сайтов не сопоставлены** , если этот параметр включен контроль допуска звонков.

8.  Нажмите кнопку **Сохранить** для сохранения изменений.


## <a name="disable-network-media-bypass"></a>Отключение сервера-посредника сети

Параметры сервера-посредника мультимедиа применяются глобально через Скайп для развертывания Business Server. Обход сервера-посредника разрешает вызовы сервера-посредника. Для получения дополнительных сведений об использовании мультимедиа обходить, ознакомьтесь со статьей [Plan для мультимедиа обходить](../../../plan-your-deployment/enterprise-voice-solution/media-bypass.md). Обход сервера-посредника из Скайп можно отключить для панели управления Business Server. 


### <a name="to-disable-media-bypass"></a>Чтобы отключить сервера-посредника

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и щелкните **Глобальная**.

4.  На странице **глобально** выберите **глобальную** конфигурацию. Всегда существует только одна конфигурация, и он всегда носит Global.

5.  В меню **Правка** щелкните **Показать подробности**.

6.  На странице **Изменение глобальных параметров** снимите флажок **Включить обход сервера-посредника** .

7.  Нажмите кнопку **Сохранить** для сохранения изменений.
