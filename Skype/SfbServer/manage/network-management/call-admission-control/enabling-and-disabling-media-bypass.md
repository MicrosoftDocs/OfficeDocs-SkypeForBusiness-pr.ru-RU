# <a name="enabling-and-disabling-media-bypass-in-skype-for-business-server"></a>Включение и отключение обхода сервера-посредника в Skype для бизнеса Server

Используйте процедуры, описанные в этой статье, чтобы включить или отключить обход сервера-посредника с помощью Skype для бизнеса Server панель управления.

## <a name="enable-network-media-bypass"></a>Включение обхода сетевого носителя 

Параметры обхода сервера-посредника применяются глобально в Skype для бизнеса Server развертывания. Обход сервера-посредника позволяет звонкам обходить сервер-посредник. Дополнительные сведения об использовании обхода сервера-посредника см. в разделе ["Планирование обхода сервера-посредника"](../../../plan-your-deployment/enterprise-voice-solution/media-bypass.md).

Вы можете включить и настроить обход сервера-посредника из Skype для бизнеса Server панель управления.


### <a name="to-enable-and-configure-media-bypass"></a>Включение и настройка обхода сервера-посредника

1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или которой назначена роль CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-Администратор, чтобы открыть Skype для бизнеса Server панель управления. 

3.  На панели навигации слева щелкните " **Конфигурация сети**" и выберите пункт "Глобальная **"**.

4.  На странице **Глобальная** выберите конфигурацию **Глобальная**. На этой странице всегда приводится только одна конфигурация, и она всегда имеет название "Глобальная".

5.  В меню **Правка** выберите команду **Показать сведения**.

6.  На странице **Изменение глобального параметра** установите флажок **Включить обход сервера-посредника**.

7.  Выберите один из следующих вариантов.
    
      - **Всегда обходить** — выберите этот вариант, чтобы попытка обхода сервера-посредника выполнялась для каждого звонка. Этот вариант недоступен, если включен контроль допуска звонков. Если контроль допуска звонков отключен, этот вариант следует выбирать в следующих ситуациях:
        
          - не требуется контроль за пропускной способностью канала;
        
          - не требуется выполнять сложный контроль, когда должен происходить обход;
        
          - между клиентами и шлюзами есть подключение.
    
      - **Использовать конфигурацию узлов и областей** — если контроль допуска звонков включен, этот вариант выбран по умолчанию и его нельзя изменить. В этом случае для определения того, когда возможен обход сервера-посредника, используются узлы и области конфигурации сети. Если вы выбрали этот вариант, вы можете включить обход для несопоставленных узлов. Флажок **Включить обход для несопоставленных узлов** следует устанавливать только в том случае, если имеется один или несколько крупных узлов, сопоставленных с одной областью, для которых отсутствуют ограничения по пропускной способности (например, крупный центральный узел), но также имеется несколько узлов филиалов, сопоставленных с той же областью, для которых ограничения по пропускной способности присутствуют. Когда вы разрешаете обход для несопоставленных узлов, конфигурация упрощается, так как вы указываете только подсети, сопоставленные с узлами филиалов, а указывать все подсети, сопоставленные со всеми узлами, не требуется. Мы рекомендуем не устанавливать флажок **Включить обход для несопоставленных узлов**, если включен контроль допуска звонков.

8.  Чтобы сохранить изменения, нажмите кнопку **Сохранить**.


## <a name="disable-network-media-bypass"></a>Отключение обхода сетевого носителя

Параметры обхода сервера-посредника применяются глобально в Skype для бизнеса Server развертывания. Обход сервера-посредника позволяет звонкам обходить сервер-посредник. Дополнительные сведения об использовании обхода сервера-посредника см. в разделе ["Планирование обхода сервера-посредника"](../../../plan-your-deployment/enterprise-voice-solution/media-bypass.md). Обход сервера-посредника можно отключить из Skype для бизнеса Server панель управления. 


### <a name="to-disable-media-bypass"></a>Отключение обхода медиаданных

1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или которой назначена роль CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-Администратор, чтобы открыть Skype для бизнеса Server панель управления. 

3.  На панели навигации слева щелкните " **Конфигурация сети**" и выберите пункт "Глобальная **"**.

4.  На странице **Глобальная** выберите конфигурацию **Глобальная**. На этой странице всегда приводится только одна конфигурация, и она всегда имеет название "Глобальная".

5.  В меню **Изменить** щелкните **Просмотреть сведения**.

6.  На странице **Изменить глобальную настройку** снимите флажок **Разрешить обход мультимедиа**.

7.  Щелкните **Сохранить**, чтобы сохранить изменения.
