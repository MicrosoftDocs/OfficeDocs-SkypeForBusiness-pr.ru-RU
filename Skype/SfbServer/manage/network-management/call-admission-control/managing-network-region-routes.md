# <a name="managing-network-region-routes-in-skype-for-business-server"></a>Управление маршрутов между сетевыми областями в Скайп для Business Server

*Маршрут к региону сети* определяет маршрут между парой областей сети. Каждой парой областей сети в вашего развертывания контроля допуска звонков необходимо маршрута области сети. Это позволяет каждому региону сети в рамках развертывания осуществлять доступ к любому другому региону. Используйте процедуры, описанные в этом artilce для просмотра, создания, изменения или удаления маршрутов между сетевыми областями.

## <a name="view-network-region-route-information"></a>Просмотр информации о маршруте между областями сети 

Каждого региона в конфигурации контроля допуска допуска звонков необходимо определить, какой метод для доступа к любая область. Хотя связей между областями значение ограничения пропускной способности для подключений между областями, а также представляют физические ссылки, маршрут определяют путь, который должны пройти подключения от одного региона в другой. Используйте следующие процедуры для просмотра существующих маршрутов областей сети в Скайп для панели управления Business Server или Скайп для консоли Business Server. 

### <a name="to-view-network-region-route-information-in-skype-for-business-server-control-panel"></a>Для просмотра информации о маршруте между областями сети в Скайп для панели управления Business Server

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и затем щелкните **Маршрут региона**.

4.  На странице **Маршрут к региону** щелкните маршрут к региону, который требуется просмотреть.


    > [!NOTE]  
    > Одновременно можно просматривать только один маршрут.


5.  В меню **Правка** щелкните **Подробнее**.


### <a name="viewing-network-region-route-information-by-using-windows-powershell-cmdlets"></a>Просмотр сведений о маршруте между областями сети с помощью командлетов Windows PowerShell

Информации о маршруте между областями сети можно просматривать с помощью командлета Get-CsNetworkInterRegionRoute и Windows PowerShell. Этот командлет можно запустить из Скайп для консоли Business Server или из удаленного сеанса Windows PowerShell. 

### <a name="to-view-network-region-route-information"></a>Для просмотра информации о маршруте между областями сети

  - Чтобы просмотреть сведения обо всех маршрутах между областями сети, введите следующую команду в Скайп для консоли Business Server и нажмите клавишу ВВОД:
    
        Get-CsNetworkInterRegionRoute
    
    Команда возвращает примерно следующую информацию:
    
        Identity                  : TransAmericaRoute
        NetworkRegionLinks        : {NorthwestToNortheast}
        InterNetworkRegionRouteID : TransAmericaRoute
        NetworkRegionID1          : Pacific Northwest
        NetworkRegionID2          : Northeast

Для получения дополнительных сведений см раздел справки для командлета [Get-CsNetworkInterRegionRoute](https://docs.microsoft.com/powershell/module/skype/Get-CsNetworkInterRegionRoute) .


## <a name="create-or-modify-network-region-routes"></a>Создание или изменение маршрутов между сетевыми областями

Каждого региона в конфигурации контроля допуска допуска звонков необходимо определить, какой метод для доступа к любая область. Хотя связей между областями значение ограничения пропускной способности для подключений между областями, а также представляют физические ссылки, маршрут определяют путь, который должны пройти подключения от одного региона в другой. Скайп для панели управления Business Server можно использовать для настройки маршрутов между сетевыми областями. Из Скайп для панели управления Business Server создание, изменение или удаление маршрута области сети. Используйте этот раздел для создания или изменения маршрута области сети. 

### <a name="to-create-a-network-region-route"></a>Создание маршрута области сети

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и затем щелкните **Маршрут региона**.

4.  На странице **Маршрут** к региону нажмите кнопку **Создать**.

5.  В области **новый маршрут к региону**введите значение в поле **имя** .
   
    > [!NOTE]  
    > Это значение должно быть уникальным в рамках вашей Скайп для развертывания Business Server.

6.  Из **области сети \#1** раскрывающегося списка, выберите один из двух регионов для подключения по этому маршруту.

7.  Из **области сети \#2** раскрывающегося списка выберите другие области для маршрута. Эта область должно отличаться от региона, выбранного для области сети \#1.

8.  Используйте список **связей между областями** для добавления связей между областями маршруту. Нажмите кнопку **Добавить** , чтобы отобразить страницу **Связь между областями** . Щелкните связь между областями, чтобы добавить этот маршрут и нажмите кнопку **ОК**.
    
    > [!NOTE]  
    > Продолжайте, нажмите кнопку **Добавить** , чтобы добавить дополнительные связи, или выберите связь и нажмите кнопку **Удалить** для удаления соединения.

9.  Нажмите **Исполнить**.


### <a name="to-modify-a-network-region-route"></a>Изменение маршрута области сети

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и затем щелкните **Маршрут региона**.

4.  На странице **Маршрут к региону** щелкните маршрут к региону, который требуется изменить.

5.  В меню **Правка** щелкните **Подробнее**.

6.  В области **изменить маршрут к региону**можно изменить регионы, соединенные этим маршрутом и связи с регионами маршрут.

7.  Нажмите **Исполнить**.


## <a name="delete-existing-network-region-routes"></a>Удаление существующих маршрутов областей сети

Каждого региона в конфигурации контроля допуска допуска звонков необходимо определить, какой метод для доступа к любая область. Хотя связей между областями значение ограничения пропускной способности для подключений между областями, а также представляют физические ссылки, маршрут определяют путь, который должны пройти подключения от одного региона в другой. Скайп для панели управления Business Server можно использовать для настройки маршрутов между сетевыми областями. Из Скайп для панели управления Business Server создание, изменение или удаление маршрута области сети. Используйте этот раздел для удаления существующих маршрутов областей сети. 

### <a name="to-delete-a-network-region-route"></a>Удаление маршрута области сети

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и затем щелкните **Маршрут региона**.

4.  На странице **Маршрут к региону** щелкните маршрут к региону, который требуется удалить.

    > [!NOTE]  
    > Можно удалить более чем один маршрут за раз. Для этого нажмите клавишу CTRL и выберите несколько маршрутов между областями, удерживая нажатой клавишу CTRL. Или, чтобы выбрать все маршруты области, нажмите кнопку **Выбрать все** в меню **Правка** .

5.  В меню **Правка** выберите пункт **Удалить**.

6.  Нажмите **OK**.
