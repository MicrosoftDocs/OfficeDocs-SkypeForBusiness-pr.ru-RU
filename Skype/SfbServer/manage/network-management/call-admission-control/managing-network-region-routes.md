# <a name="managing-network-region-routes-in-skype-for-business-server"></a>Управление маршрутами областей сети в Skype для бизнеса Server

*Маршрут между областями сети* определяет путь между парой сетевых областей. Для каждой пары сетевых областей в развертывании контроля допуска звонков необходим маршрут между областями сети. Благодаря этому каждая область сети в рамках развертывания может получить доступ к каждой другой области. Используйте процедуры в этой области для просмотра, создания, изменения или удаления маршрутов сетевого региона.

## <a name="view-network-region-route-information"></a>Просмотр сведений о маршруте сетевого региона 

Каждому региону в конфигурации контроля допуска звонков (CAC) необходимо предоставить возможность доступа к каждой другой области. Связи между регионами накладывают определенные ограничения на пропускную способность, доступную подключениям между областями, и также представляют собой физические соединения; маршруты же определяют путь, который должны пройти подключения от одной области до другой. Используйте следующие процедуры, чтобы просмотреть существующие маршруты сетевых регионов в Skype для бизнеса Server панели управления или Skype для бизнеса Server управленческой оболочки. 

### <a name="to-view-network-region-route-information-in-skype-for-business-server-control-panel"></a>Просмотр сведений о маршруте сетевого региона в Skype для бизнеса Server панели управления

1.  С учетной записи пользователя, которая входит в группу RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления. 

3.  В левой панели навигации нажмите **кнопку Конфигурация** сети, а затем щелкните **Маршрут региона**.

4.  На странице **Маршрут региона** выберите маршрут области, который хотите просмотреть.


    > [!NOTE]
    > Можно просматривать только один маршрут за один раз.


5.  В меню **Изменить** щелкните **Подробнее**.


### <a name="viewing-network-region-route-information-by-using-windows-powershell-cmdlets"></a>Просмотр сведений о маршруте сетевого региона с помощью Windows PowerShell cmdlets

Сведения о маршруте сетевого региона можно просмотреть с помощью Windows PowerShell и Get-CsNetworkInterRegionRoute. Этот комлет можно запускать из Skype для бизнеса Server или удаленного сеанса Windows PowerShell. 

### <a name="to-view-network-region-route-information"></a>Просмотр сведений о маршруте сетевого региона

  - Чтобы просмотреть сведения обо всех маршрутах сетевого региона, введите следующую команду в командной Skype для бизнеса Server, а затем нажмите кнопку ENTER:
    
    **Get-CsNetworkInterRegionRoute**
    
    Это приведет к возврату приблизительно такой информации:
    
    Identity: TransAmericaRoute<br/>
    NetworkRegionLinks: {NorthwestToNortheast}<br/>
    InterNetworkRegionRouteID : TransAmericaRoute<br/>
    NetworkRegionID1 : Северо-запад Тихого океана<br/>
    NetworkRegionID2 : Северо-восток<br/>

Дополнительные сведения см. в разделе справки по командлету [Get-CsNetworkInterRegionRoute](/powershell/module/skype/Get-CsNetworkInterRegionRoute).


## <a name="create-or-modify-network-region-routes"></a>Создание или изменение маршрутов сетевого региона

Каждому региону в конфигурации контроля допуска звонков (CAC) необходимо предоставить возможность доступа к каждой другой области. Связи между регионами накладывают определенные ограничения на пропускную способность, доступную подключениям между областями, и также представляют собой физические соединения; маршруты же определяют путь, который должны пройти подключения от одной области до другой. Вы можете использовать панель Skype для бизнеса Server для настройки маршрутов сетевых регионов. С панели Skype для бизнеса Server можно создать, изменить или удалить маршрут сетевого региона. Используйте этот раздел для создания или изменения маршрута маршрут между областями сети. 

### <a name="to-create-a-network-region-route"></a>Создание маршрута к регионами сети

1.  С учетной записи пользователя, которая входит в группу RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления. 

3.  В левой панели навигации нажмите **кнопку Конфигурация** сети, а затем щелкните **Маршрут региона**.

4.  На странице **Маршрут к региону** щелкните **Создать**.

5.  В области **Новый маршрут к региону** введите значение в поле **Имя**.
   
    > [!NOTE]  
    > Это значение должно быть уникальным для Skype для бизнеса Server развертывания.

6.  Из выпадаемого списка "Область сети **\# 1"** выберите один из двух областей, подключенных по этому маршруту.

7.  Из выпадаемого списка "Область сети **\# 2"** выберите другой регион для этого маршрута. Этот регион должен быть отличается от региона, выбранного для области Сети \# 1.

8.  Используйте поле со списком **Связи регионов сети**, чтобы добавить в маршрут связи с регионами. Щелкните кнопку **Добавить** для отображения страницы **Связь с регионом**. Щелкните связь с регионом, чтобы добавить ее в маршрут, и щелкните **ОК**.
    
    > [!NOTE]  
    > Нажимайте кнопку **Добавить**, чтобы добавить дополнительные связи, или выберите связь и щелкните **Удалить** для удаления соединения.

9.  Щелкните **Исполнить**.


### <a name="to-modify-a-network-region-route"></a>Изменение маршрута к региону сети

1.  С учетной записи пользователя, которая входит в группу RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления. 

3.  В левой панели навигации нажмите **кнопку Конфигурация** сети, а затем щелкните **Маршрут региона**.

4.  На странице **Маршрут к региону** щелкните маршрут к региону, который необходимо изменить.

5.  В меню **Правка** щелкните **Показать подробности**.

6.  В области **Изменить маршрут к региону** можно изменить регионы, соединенные этим маршрутом, и связи с регионами, назначенные маршруту.

7.  Щелкните **Исполнить**.


## <a name="delete-existing-network-region-routes"></a>Удаление существующих маршрутов сетевого региона

Каждому региону в конфигурации контроля допуска звонков (CAC) необходимо предоставить возможность доступа к каждой другой области. Связи между регионами накладывают определенные ограничения на пропускную способность, доступную подключениям между областями, и также представляют собой физические соединения; маршруты же определяют путь, который должны пройти подключения от одной области до другой. Вы можете использовать панель Skype для бизнеса Server для настройки маршрутов сетевых регионов. С панели Skype для бизнеса Server можно создать, изменить или удалить маршрут сетевого региона. С помощью инструкций, приведенных в этом разделе, можно удалить существующие маршруты сетевых регионов. 

### <a name="to-delete-a-network-region-route"></a>Чтобы удалить маршрут сетевого региона

1.  С учетной записи пользователя, которая входит в группу RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления. 

3.  В левой панели навигации нажмите **кнопку Конфигурация** сети, а затем щелкните **Маршрут региона**.

4.  На странице **Маршрут региона** щелкните маршрут, который следует удалить.

    > [!NOTE]  
    > Одновременно можно удалить сразу несколько маршрутов регионов. Для этого нажмите и удерживайте клавишу CTRL, одновременно выбирая несколько маршрутов регионов. Или щелкните пункт **Выделить все** в меню **Изменить** для выбора сразу нескольких маршрутов.

5.  В меню **Изменить** щелкните **Удалить**.

6.  Нажмите кнопку **ОК**.
