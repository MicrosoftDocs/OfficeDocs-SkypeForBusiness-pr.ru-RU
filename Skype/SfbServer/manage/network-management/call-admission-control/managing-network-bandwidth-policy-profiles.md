# <a name="managing-network-bandwidth-policy-profiles-in-skype-for-business-server"></a>Управление профилями политики пропускной способности сети в Skype для бизнеса Server

Используйте процедуры, описанные в этой статье для просмотра, создания, изменения или удаления профилей политики пропускной способности сети.

## <a name="view-network-bandwidth-policy-profile-information"></a>Просмотр сведений о профиле политики пропускной способности сети

Как часть контроля допуска звонков (CAC) политика пропускной способности используется для определения ограничения пропускной способности для определенных модальностей. В Скайп для Business Server только аудио- и видеоконференций модальностей может быть назначен ограничения пропускной способности. Можно задать общие ограничения пропускной способности и ограничения сеанса. Создание, изменение или удаление профиля контейнер для этих политик можно использовать Скайп для панели управления Business Server. Каждый профиль политики пропускной способности может быть связан с одной или нескольких сетевых узлов. Используйте следующие процедуры для просмотра профиля политики пропускной способности. 

### <a name="to-view-a-bandwidth-policy-profile"></a>Просмотр профиля политики пропускной способности

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети** и нажмите кнопку **Политика пропускной способности**.

4.  На странице **Политика пропускной способности** щелкните профиль политики пропускной способности, который требуется просмотреть.

5.  В меню **Правка** щелкните **Подробнее**.


### <a name="viewing-network-bandwidth-policy-profile-information-by-using-windows-powershell-cmdlets"></a>Просмотр сведений о профиле политики пропускной способности сети с помощью командлетов Windows PowerShell

Профили пропускной способности сети можно просматривать с помощью командлета Get-CsNetworkBandwidthPolicyProfile и Windows PowerShell. Этот командлет можно запустить из Скайп для консоли Business Server или из удаленного сеанса Windows PowerShell. 


### <a name="to-view-network-bandwidth-policy-profile-information"></a>Для просмотра сведений о профиле политики пропускной способности сети

  - Чтобы просмотреть сведения обо всех вашей сети профилей политики пропускной способности, введите следующую команду в Скайп для консоли Business Server и нажмите клавишу ВВОД:
    
        Get-CsNetworkBandwidthPolicyProfile
    
    Команда возвращает примерно следующую информацию:
    
        Identity          : RedmondBandwidthPolicy
        BWPolicy          : {BWLimit=200;BWSessionLimit=200;
                            BWPolicyModality=Audio, 
                            BWLimit=1400;BWSessionLimit=500;
                            BWPolicyModality=Video}
        BWPolicyProfileID : RedmondBandwidthPolicy
        Description       :


Для получения дополнительных сведений см раздел справки для командлета [Get-CsNetworkBandwidthPolicyProfile](https://docs.microsoft.com/powershell/module/skype/Get-CsNetworkBandwidthPolicyProfile) .


## <a name="create-or-modify-bandwidth-policy-profiles"></a>Создание или изменение профилей политики пропускной способности

Как часть контроля допуска звонков (CAC) политика пропускной способности используется для определения ограничения пропускной способности для определенных модальностей. В Скайп для Business Server только аудио- и видеоконференций модальностей может быть назначен ограничения пропускной способности. Можно задать общие ограничения пропускной способности и ограничения сеанса. Создание, изменение или удаление профиля контейнер для этих политик можно использовать Скайп для панели управления Business Server. Каждый профиль политики пропускной способности может быть связан с одной или нескольких сетевых узлов. Используйте следующие процедуры для создания или изменения профиля политики пропускной способности. 

### <a name="to-create-a-new-bandwidth-policy-profile"></a>Чтобы создать новый профиль политики пропускной способности

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и щелкните **Политика полосы пропускания**.

4.  На странице **Политика пропускной способности** нажмите кнопку **Создать**.

5.  В **Новый профиль политики пропускной способности**введите имя в поле **имя** . Это имя должно быть уникальным во всех профилей политики пропускной способности.

6.  В поле **ограничение звук** введите числовое значение. Это значение — максимальный объем пропускной способности для выделения для всех подключений звука, выраженная в Кбит/с.

7.  Введите числовое значение в поле **ограничения сеанса звука** . Это значение — максимальный объем пропускной способности для выделения для отдельного звукового подключения, выраженная в Кбит/с. Это значение должно быть 40 или выше.

8.  Введите числовое значение в поле **ограничение видео** . Это значение — максимальный объем пропускной способности для размещения для всех подключений с видеоконференциями, выраженная в Кбит/с.

9.  Введите числовое значение в поле **ограничения сеанса видеосвязи** . Это значение — максимальный объем пропускной способности для выделения для отдельного видео подключения, выраженная в Кбит/с. Это значение должно быть 100 или более поздней версии.

10. (Необязательно) Введите значение в поле **Описание** введите дополнительные сведения об этом профиле политики пропускной способности, которые не могут быть выражены одним именем.

11. Нажмите **Исполнить**.

    > [!NOTE]  
    > Создание нового профиля политики пропускной способности не требует автоматически принудительного ограничения пропускной способности. Профиль политики сначала необходимо связать с сайтом. 


### <a name="to-modify-a-bandwidth-policy-profile"></a>Изменение профиля политики пропускной способности

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и щелкните **Политика полосы пропускания**.

4.  На странице **Политика пропускной способности** щелкните профиль политики пропускной способности, который требуется изменить.

5.  В меню **Правка** щелкните **Подробнее**.

6.  На странице " **Изменение профиля политики пропускной способности** " Изменение полей при необходимости (для получения дополнительных сведений, содержатся в разделе [Создание нового профиля политики пропускной способности](#to-create-a-new-bandwidth-policy-profile)).

7.  Нажмите **Исполнить**.

    > [!NOTE]  
    > При изменении профиля политики пропускной способности он немедленно обновляет ограничения пропускной способности все сетевые узлы, связанные с этой профиль политики пропускной способности.

  
## <a name="delete-network-bandwidth-policy-profiles"></a>Удаление профилей политик пропускной способности сети

Как часть контроля допуска звонков (CAC) политика пропускной способности используется для определения ограничения пропускной способности для определенных модальностей. В Скайп для Business Server только аудио- и видеоконференций модальностей может быть назначен ограничения пропускной способности. Можно задать общие ограничения пропускной способности и ограничения сеанса. Создание, изменение или удаление профиля контейнер для этих политик можно использовать Скайп для панели управления Business Server. Используйте следующие процедуры для удаления профилей политики пропускной способности сети. 

### <a name="to-delete-a-bandwidth-policy-profile"></a>Чтобы удалить профиль политики пропускной способности

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и щелкните **Политика полосы пропускания**.

4.  На странице **Политика пропускной способности** щелкните профиль политики пропускной способности, который требуется удалить.

    > [!NOTE]  
    > Одновременно можно удалить несколько профилей. Для этого нажмите клавишу CTRL и выберите несколько профилей, удерживая нажатой клавишу CTRL. Или, чтобы выбрать все профили, нажмите кнопку **Выбрать все** в меню **Правка** .

5.  В меню **Правка** выберите пункт **Удалить**.
   

    > [!WARNING]  
    > Не удается удалить профиль политики пропускной способности, которая связана с сетевым узлом. Можно удалить профиль, удалите связь с сетевым узлом.
