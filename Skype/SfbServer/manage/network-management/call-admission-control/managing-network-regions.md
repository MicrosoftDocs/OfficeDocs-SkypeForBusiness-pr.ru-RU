# <a name="managing-network-regions-in-skype-for-business-server"></a>Управление областями сети в Skype для бизнеса Server

*Области сети* представляют собой сетевые концентраторы или магистрали, используемые в конфигурации контроля допуска звонков, E9-1-1 и обхода сервера-посредника. Следующие процедуры используются для просмотра, создания или изменения областей сети. Например, если области сети уже созданы для одного из компонентов голосовой связи, новые области сети создавать не требуется; остальные расширенные компоненты корпоративной голосовой связи используют те же области сети. Тем не менее, возможно, потребуется изменить существующее определение области сети, чтобы применить специальные настройки для конкретных компонентов. Например, когда после создания областей сети для системы E9-1-1 (которой не требуется связанный центральный сайт) выполняется развертывание системы контроля допуска звонков, потребуется изменить определения области сети для указания центрального сайта. 

Используйте процедуры в этой статье для просмотра сведений о сетевом регионе или создания, изменения или удаления сетевых регионов. 

## <a name="view-network-region-information"></a>Просмотр сведений о сетевом регионе 


Область сети связывает части сети, расположенные в различных географических районах. Каждый регион сети должен быть связан с центральным сайтом. Центральный сайт — это сайт центра обработки данных, на котором выполняется служба политики пропускной способности для контроля допуска звонков. Панель управления Skype для бизнеса Server для просмотра сетевых регионов. Области сети включают параметры, которые определяют, разрешены ли альтернативные пути через Интернет для аудио- и видеоподключений. Используйте эту тему для просмотра существующих сетевых регионов. 

### <a name="to-view-information-about-a-network-region-with-skype-for-business-server-control-panel"></a>Просмотр сведений о сетевом регионе с Skype для бизнеса Server панели управления

1.  С учетной записи пользователя, которая входит в группу RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления. 

3.  В левой панели навигации щелкните **Конфигурация сети**, а затем щелкните **Регион**.

4.  На странице **Регион** щелкните регион, который следует просмотреть.
  
    > [!NOTE]  
    > одновременно можно просматривать только один регион.

5.  В меню **Изменить** щелкните **Показать сведения**.


### <a name="viewing-network-region-information-by-using-windows-powershell-cmdlets"></a>Просмотр сведений о сетевом регионе с помощью Windows PowerShell-кодлетов

Сведения о сетевом регионе можно просматривать с помощью Windows PowerShell **и командлета Get-CsNetworkRegion**. Этот комлет можно выполнить либо из Skype для бизнеса Server, либо из удаленного сеанса Windows PowerShell. 

### <a name="to-view-network-region-information"></a>Просмотр сведений о сетевом регионе

  - Чтобы просмотреть сведения обо всех регионах сети, введите следующую команду в командной Skype для бизнеса Server и нажмите кнопку ENTER:
    
    **Get-CsNetworkRegion**
    
    Это приведет к возврату приблизительно такой информации:
    
    Identity: Pacific Northwest<br/>
    Описание:<br/>
    BypassID: 3b232b84-2c1d-4da2-8181-e9330bafebe9<br/>
    CentralSite : Site:Redmond1<br/>
    BWAlternatePaths: {BWPolicyModality=Audio; AlternatePath=True, <br/>
                       BWPolicyModality=Video; AlternatePath=True}<br/>
    NetworkRegionID : Северо-запад Тихого океана<br/>

Дополнительные сведения см. в разделе справки по командлету [Get-CsNetworkRegion](/powershell/module/skype/Get-CsNetworkRegionLink).


## <a name="create-or-modify-network-regions"></a>Создание или изменение сетевых регионов 

Область сети связывает части сети, расположенные в различных географических районах. Каждый регион сети должен быть связан с центральным сайтом. Центральный сайт — это сайт центра обработки данных, на котором выполняется служба политики пропускной способности для контроля допуска звонков. Панель управления Skype для бизнеса Server для настройки сетевых регионов. Области сети включают параметры, которые определяют, разрешены ли альтернативные пути через Интернет для аудио- и видеоподключений. С панели Skype для бизнеса Server можно создать, изменить или удалить сетевой регион. Используйте этот раздел для создания и изменения областей сети. 

### <a name="to-create-a-network-region"></a>Создание области сети

1.  С учетной записи пользователя, которая входит в группу RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления. 

3.  В левой панели навигации щелкните **Конфигурация сети**, а затем щелкните **Регион**.

4.  На странице **Область**  щелкните **Создать**.

5.  На странице **Создание области** введите имя области сети в поле **Имя**. Это значение должно быть уникальным для Skype для бизнеса Server развертывания.

6.  В раскрывающемся списке **Центральный сайт** выберите центральный сайт для этой области сети.

7.  Флажок **Включить альтернативный путь для звонков** установлен по умолчанию. Этот параметр указывает, будут ли звонки перенаправляться по альтернативному пути, если пропускной способности основного пути недостаточно. Этот флажок нужно снимать только в том случае, если требуется отключить разгрузку в Интернет. При наличии звонков по Интернету этот флажок должен быть установлен независимо от параметров пропускной способности.

8.  Флажок **Включить альтернативный путь видеозвонков** установлен по умолчанию. Этот параметр указывает, будут ли видеозвонки перенаправляться по альтернативному пути, если пропускной способности основного пути недостаточно. Этот флажок нужно снимать только в том случае, если требуется отключить разгрузку в Интернет. При наличии звонков по Интернету этот флажок должен быть установлен независимо от параметров пропускной способности.

9.  Введите дополнительные сведения об области сети в поле **Описание** (необязательно).

10. Щелкните **Зафиксировать**.

При создании области сети таблица **Связанные сайты** не используется. Вы можете связать сайт с областью при создании или изменении сайта. Подробные сведения см. [в материале Managing call admission control for sites](managing-call-admission-control-for-sites.md).

### <a name="to-modify-a-network-region"></a>Изменение области сети

1.  С учетной записи пользователя, которая входит в группу RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления. 

3.  В левой панели навигации щелкните **Конфигурация сети**, а затем щелкните **Регион**.

4.  На странице **Область** щелкните область, которую необходимо изменить.

5.  В меню **Правка** выберите команду **Показать сведения**.

6.  На странице **Изменение области** можно изменить параметры включения и отключения альтернативных путей для звонков и видеозвонков, а также изменить описание (дополнительные сведения см. в подразделе "Создание области сети" настоящего раздела).

7.  Щелкните **Зафиксировать**.

На этой странице нельзя изменить **Связанные сайты**. В этом списке указаны сайты, на которые может повлиять изменение параметров области.


## <a name="delete-existing-network-regions"></a>Удаление существующих сетевых регионов 

Область сети связывает части сети, расположенные в различных географических районах. Каждый регион сети должен быть связан с центральным сайтом. Центральный сайт — это сайт центра обработки данных, на котором выполняется служба политики пропускной способности для контроля допуска звонков. Панель управления Skype для бизнеса Server для настройки сетевых регионов. Области сети включают параметры, которые определяют, разрешены ли альтернативные пути через Интернет для аудио- и видеоподключений. С панели Skype для бизнеса Server можно создать, изменить или удалить сетевой регион. Используйте этот раздел для удаления существующих областей сети. 

### <a name="to-delete-a-network-region"></a>Удаление области сети

1.  С учетной записи пользователя, которая входит в группу RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления. 

3.  В левой панели навигации щелкните **Конфигурация сети**, а затем щелкните **Регион**.

4.  На странице **областей** щелкните область, которую следует удалить.
  
    > [!NOTE]  
    > Можно одновременно удалять несколько областей. Для этого нажмите клавишу CTRL и выберите несколько областей, удерживая клавишу CTRL нажатой. Можно также выбрать все области, нажав пункт **Выбрать все** в меню **Правка**.

5.  В меню **Правка** выберите команду **Удалить**.

6.  Нажмите кнопку **ОК**.


    > [!WARNING]  
    > Нельзя удалить область сети, если она связана с сетевым сайтом. При попытке удалить область, связанную с сайтом, вы получите сообщение об ошибке. Чтобы увидеть, связана ли область с какими-либо сайтами, выделите эту область и выберите команду **Показать подробности** в меню **Правка**.
