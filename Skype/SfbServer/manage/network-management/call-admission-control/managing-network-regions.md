# <a name="managing-network-regions-in-skype-for-business-server"></a>Управление областей сети в Скайп для Business Server

*Области сети* являются сетевых концентраторов и магистралей, используемые в конфигурации обход элемента управления, E9-1-1 и мультимедиа допуска звонков. Используйте следующие процедуры для просмотра, создания или изменения областей сети. Например если вы уже создали областей сети для одного компонента голосовой связи, не нужно создать новые области сети; другие расширенные функции корпоративной голосовой связи будет использовать те же области сети. Однако вам может потребоваться изменить существующее определение области сети для применения параметров, относящихся к функции. Например, если вы создали области сети для службы E9-1-1, для которой не требуется связанный центральный сайт, и затем развернули службу контроля допуска звонков, вам потребуется изменить определения областей сети, чтобы указать центральный сайт. 

Используйте процедуры, описанные в этой статье для просмотра сведений о сетевом регионе или создание, изменение и удаление областей сети. 

## <a name="view-network-region-information"></a>Просмотр сведений о сетевом регионе 


Область сети соединения различные части сети через несколько географических областей. Каждой области сети должен быть связан с центральным узлом. Центральный узел является сайт центра данных, на котором запущена служба политики пропускной способности контроля допуска допуска звонков. Скайп для панели управления Business Server можно использовать для просмотра областей сети. Области сети включают параметры, определяющие, разрешены ли альтернативные маршруты через Интернет для подключения к аудио- и видеоконференций. Используйте этот раздел для просмотра существующих областей сети. 

### <a name="to-view-information-about-a-network-region-with-skype-for-business-server-control-panel"></a>Чтобы просмотреть сведения об области сети с Скайп для панели управления Business Server

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и затем выберите **Регион**.

4.  На странице **область** щелкните область, которую необходимо просмотреть.
  
    > [!NOTE]  
    > Можно просматривать только один регион.

5.  В меню **Правка** щелкните **Подробнее**.


### <a name="viewing-network-region-information-by-using-windows-powershell-cmdlets"></a>Просмотр сведений о сетевом регионе с помощью командлетов Windows PowerShell

Сведений о сетевом регионе можно просмотреть с помощью Windows PowerShell и командлета **Get-CsNetworkRegion** . Можно выполнить этот командлет из Скайп для консоли Business Server или из удаленного сеанса Windows PowerShell. 

### <a name="to-view-network-region-information"></a>Для просмотра сведений о сетевом регионе

  - Чтобы просмотреть сведения обо всех сетевых регионах, введите следующую команду в Скайп для консоли Business Server и нажмите клавишу ВВОД:
    
        Get-CsNetworkRegion
    
    Команда возвращает примерно следующую информацию:
    
        Identity         : Pacific Northwest
        Description      :
        BypassID         : 3b232b84-2c1d-4da2-8181-e9330bafebe9
        CentralSite      : Site:Redmond1
        BWAlternatePaths : {BWPolicyModality=Audio;AlternatePath=True, 
                           BWPolicyModality=Video;AlternatePath=True}
        NetworkRegionID  : Pacific Northwest

Для получения дополнительных сведений см раздел справки для командлета [Get-CsNetworkRegion](https://docs.microsoft.com/powershell/module/skype/Get-CsNetworkRegionLink) .


## <a name="create-or-modify-network-regions"></a>Создание или изменение областей сети 

Область сети соединения различные части сети через несколько географических областей. Каждой области сети должен быть связан с центральным узлом. Центральный узел является сайт центра данных, на котором запущена служба политики пропускной способности контроля допуска допуска звонков. Настройка сетевых областей можно использовать Скайп для панели управления Business Server. Области сети включают параметры, определяющие, разрешены ли альтернативные маршруты через Интернет для подключения к аудио- и видеоконференций. Из Скайп для панели управления Business Server создание, изменение или удаление области сети. Используйте этот раздел для создания и изменения областей сети. 

### <a name="to-create-a-network-region"></a>Создание области сети

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и затем выберите **Регион**.

4.  На странице **область** щелкните **Создать**.

5.  На странице " **Создание области** " введите значение в поле **имя** . Это значение должно быть уникальным в рамках вашей Скайп для развертывания Business Server.

6.  В раскрывающемся списке **центральный сайт** выберите центральный сайт для этой области сети.

7.  По умолчанию установлен флажок **Включение звука альтернативный путь** . В этом поле определяет ли голосовые вызовы будут перенаправляться по альтернативному пути, если не пропускной способности основного пути. Снимите этот флажок только в том случае, если требуется отключить разгрузки в Интернет. Если какие-либо из ваших вызовов будет звонком через Интернет, этот флажок, должен быть установлен, вне зависимости от параметров пропускной способности.

8.  По умолчанию установлен флажок **Включить видео альтернативный путь** . В этом поле определяет, будет ли видео звонки будут перенаправляться по альтернативному пути, если не пропускной способности основного пути. Снимите этот флажок только в том случае, если требуется отключить разгрузки в Интернет. Если какие-либо из ваших вызовов будет звонком через Интернет, этот флажок, должен быть установлен, вне зависимости от параметров пропускной способности.

9.  (Необязательно) Введите значение в поле **Описание** введите дополнительные сведения об области, которые не могут быть выражены одним именем.

10. Нажмите **Исполнить**.

В таблице **связанные сайты** не используется для создания области сети. Сайт связать с областью, при создании или изменении сайта. Дополнительные сведения см [Управление контроля допуска звонков для сайтов](managing-call-admission-control-for-sites.md).

### <a name="to-modify-a-network-region"></a>Изменение области сети

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и затем выберите **Регион**.

4.  На странице **область** щелкните область, которую требуется изменить.

5.  В меню **Правка** щелкните **Подробнее**.

6.  На странице **Изменить область** можно изменить параметры для Включение и отключение звука и видео альтернативных путей и изменить описание (для получения дополнительных сведений, см раздел «Создание области сети» ранее в этом разделе.

7.  Нажмите **Исполнить**.

**Связанные сайты** на этой странице изменить нельзя. Список связанных сайтов предоставляется для ссылки, чтобы знать о том, какие сайты будут затронуты при изменении параметров области.


## <a name="delete-existing-network-regions"></a>Удаление существующих областей сети 

Область сети соединения различные части сети через несколько географических областей. Каждой области сети должен быть связан с центральным узлом. Центральный узел является сайт центра данных, на котором запущена служба политики пропускной способности контроля допуска допуска звонков. Настройка сетевых областей можно использовать Скайп для панели управления Business Server. Области сети включают параметры, определяющие, разрешены ли альтернативные маршруты через Интернет для подключения к аудио- и видеоконференций. Из Скайп для панели управления Business Server создание, изменение или удаление области сети. Используйте этот раздел для удаления существующих областей сети. 

### <a name="to-delete-a-network-region"></a>Чтобы удалить область сети

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и затем выберите **Регион**.

4.  На странице **область** щелкните область, которую вы хотите удалить.
  
    > [!NOTE]  
    > Одновременно можно удалить более одного региона. Для этого нажмите клавишу CTRL и выберите несколько областей, удерживая нажатой клавишу CTRL. Или, чтобы выбрать все области, нажмите кнопку **Выбрать все** в меню **Правка** .

5.  В меню **Правка** выберите пункт **Удалить**.

6.  Нажмите **OK**.


    > [!WARNING]  
    > Область сети не может быть удалена, если она связана с сетевым узлом. Если попытаться удалить области, связанные с сайтом, вы получите сообщение об ошибке. Чтобы просмотреть, если область связана с веб-сайты, выберите область и нажмите кнопку **Показать подробности** в меню **Правка** .
