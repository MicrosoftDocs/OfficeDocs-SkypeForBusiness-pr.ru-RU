# <a name="managing-call-admission-control-for-sites-in-skype-for-business-server"></a>Управление контролем допуска звонков для сайтов в Skype для бизнеса Server

Сайты сети — это офисы или местоположения в каждом регионе сети развертывания контроля допуска вызовов (CAC), E9-1-1 и обхода мультимедиа. Используйте процедуры, которые данная статья использует для настройки контроля допуска вызовов для сетевых сайтов.

## <a name="configure-network-site-links"></a>Настройка связей между сетевыми сайтами

В конфигурации контроля допуска звонков можно создать сетевые межузловые политики, которые определяют ограничения пропускной способности между связанными напрямую узлами. Когда сетевые узлы совместно используют прямую связь, ограничения для аудио- и видеосвязи можно задавать между этими двумя узлами. Для настройки политик сетевых сайтов нельзя использовать панель управления Skype для бизнеса Server. Это можно сделать только с помощью cmdlets из оболочки управления Skype для бизнеса Server. Вы можете создать, изменить и удалить связь сетевого сайта (также называется сетевой межсайтовой политикой) в оболочке управления Skype для бизнеса Server.

### <a name="to-create-a-network-site-link"></a>Создание связи между сетевыми узлами

1.  Войдите на компьютер, на котором установлена оболочка управления Skype для бизнеса Server, в качестве члена группы RTCUniversalServerAdmins или с необходимыми правами пользователя.

2.  Запустите оболочку управления Skype для бизнеса Server: нажмите кнопку "Начните", выберите "Все программы", "Skype для бизнеса **Server",** а затем щелкните "Skype для бизнеса Server Management **Shell".**

3.  В Введите в командной строке следующую команду, подставив значения, подходящие для вашей конфигурации:
    
        New-CsNetworkInterSitePolicy -Identity Reno_Portland -NetworkSiteID1 Reno -NetworkSiteID2 Portland -BWPolicyProfileID LowBWLimits
    
    В этом примере создается новая связь с сетевым сайтом Reno Portland, которая задает ограничения пропускной способности между сетевыми узлами \_ Reno и Portland. Перед выполнением этой команды уже должны существовать эти сетевые узлы и профиль политики пропускной способности.

Подробные описания параметров см. в описании [new-CsNetworkInterSitePolicy.](https://docs.microsoft.com/powershell/module/skype/New-CsNetworkInterSitePolicy) Чтобы получить список профилей политик пропускной способности, которые можно применять к связи между сетевыми узлами, вызовите командлет **Get-CsNetworkBandwidthPolicyProfile**. Подробные сведения см. в [командлете Get-CsNetworkBandwidthPolicyProfile.](https://docs.microsoft.com/powershell/module/skype/Get-CsNetworkBandwidthPolicyProfile)

### <a name="to-modify-a-network-site-link"></a>Изменение связи между сетевыми узлами

1.  Войдите на компьютер, на котором установлена оболочка управления Skype для бизнеса Server, в качестве члена группы RTCUniversalServerAdmins или с необходимыми правами пользователя.

2.  Запустите оболочку управления Skype для бизнеса Server: нажмите кнопку "Начните", выберите "Все программы", "Skype для бизнеса **Server",** а затем щелкните "Skype для бизнеса Server Management **Shell".**

3.  Для изменения свойств конкретной связи между сетевыми узлами используется командлет **Set-CsNetworkInterSitePolicy**. Можно изменять каждый из связанных узлов (или оба этих узла), а также профиль политики пропускной способности, привязанный к этой связи. Вот пример изменения профиля политики пропускной способности связи сайта с именем Reno \_ Portland:
    
        Set-CsNetworkInterSitePolicy -Identity Reno_Portland -BWPolicyProfileID HighBWLimits

Подробные описания параметров см. в [описании set-CsNetworkInterSitePolicy.](https://docs.microsoft.com/powershell/module/skype/Set-CsNetworkInterSitePolicy)


### <a name="to-delete-a-network-site-link"></a>Удаление связи между сетевыми узлами

1.  Войдите на компьютер, на котором установлена оболочка управления Skype для бизнеса Server, в качестве члена группы RTCUniversalServerAdmins или с необходимыми правами пользователя.

2.  Запустите оболочку управления Skype для бизнеса Server: нажмите кнопку "Начните", выберите "Все программы", "Skype для бизнеса **Server",** а затем щелкните "Skype для бизнеса Server Management **Shell".**

3.  Для удаления связи между сетевыми узлами используется командлет **Remove-CsNetworkInterSitePolicy**. В следующем примере удаляется связь с сетевым сайтом Reno \_ Portland:
    
        Remove-CsNetworkInterSitePolicy -Identity Reno_Portland

Подробные описания параметров см. в описании [remove-CsNetworkInterSitePolicy.](https://docs.microsoft.com/powershell/module/skype/Remove-CsNetworkInterSitePolicy)


## <a name="view-network-site-information"></a>Просмотр сведений о сетевом сайте

Сетевые узлы — это офисы или места, настроенные в каждой области развертывания контроля допуска звонков (CAC) или Enhanced 9-1-1. Сведения о сетевом сайте можно просмотреть в панели управления Skype для бизнеса Server или в оболочке управления Skype для бизнеса Server. 

### <a name="to-view-network-site-information-in-skype-for-business-server-control-panel"></a>Просмотр сведений о сетевом сайте в панели управления Skype для бизнеса Server

1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server. 

3.  В левой панели навигации щелкните **"Конфигурация сети"** и выберите **"Сайт".**

4.  На странице **Сайт** выберите сайт, который хотите просмотреть.
 
    > [!NOTE]  
    > Допускается одновременный просмотр данных только для одного сайта.

5.  В меню **Правка** щелкните **Показать подробности**.


### <a name="viewing-network-site-information-by-using-windows-powershell-cmdlets"></a>Просмотр сведений о сетевом сайте с помощью Windows PowerShell управления

Просмотреть сведения о сетевом сайте можно с помощью Windows PowerShell и Get-CsNetworkSite управления. Этот cmdlet можно запустить в оболочке управления Skype для бизнеса Server или в удаленном сеансе Windows PowerShell. 

### <a name="to-view-network-site-information"></a>Просмотр сведений о сетевом сайте

  - Чтобы просмотреть сведения обо всех сетевых сайтах, введите следующую команду в командной оболочке Skype для бизнеса Server и нажмите ввод:
    
        Get-CsNetworkSite
    
    Это приведет к возврату приблизительно такой информации:
    
        Identity          : Redmond
        NetworkSiteID     : Redmond
        Description       :
        NetworkRegionID   : Pacific Northwest
        BypassID          : 3b232b84-2c1d-4da2-8181-e9330bafebe9
        BWPolicyProfileID :
        LocationPolicy    :

Дополнительные сведения см. в разделе справки для командлета [Get-CsNetworkSite](https://docs.microsoft.com/powershell/module/skype/Get-CsNetworkSite).


## <a name="create-or-modify-network-sites"></a>Создание или изменение сетевых сайтов 

Сетевые узлы — это офисы или места, настроенные в каждой области развертывания контроля допуска звонков (CAC) или Enhanced 9-1-1. Панель управления Skype для бизнеса Server можно использовать для настройки сайтов и связываия их с регионами. Например, сетевую область для Северной Америки можно связать с такими сетевыми узлами, как Чикаго, Редмонд и Ванкувер. Сетевой узел CAC необходимо создать для каждого узла в организации, даже если у этого узла нет ограничений по пропускной способности. На панели управления Skype для бизнеса Server можно создавать, изменять и удалять сетевые сайты. Ниже описаны процедуры для создания или изменения сетевого узла. 

### <a name="to-create-a-network-site"></a>Создание сетевого узла

1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server. 

3.  В левой панели навигации щелкните **"Конфигурация сети"** и выберите **"Сайт".**

4.  На странице **Узел** нажмите кнопку **Создать**.

5.  В диалоговом окне **Создание узла** введите имя узла в поле **Имя**.

    > [!NOTE]  
    > Имена сайтов должны быть уникальными в развертывании Skype для бизнеса Server.

6.  В раскрывающемся списке **Область** выберите сетевую область, которую необходимо сопоставить с этим узлом.

7.  (Необязательно) Если нужно наложить ограничения полосы пропускания на аудио- или видеовызовы с данного узла, выберите профиль политики с соответствующими параметрами в раскрывающемся списке **Политика полосы пропускания**.
 
    > [!NOTE]  
    > Вы можете просмотреть сведения о доступных профилях политики пропускной способности или создать новый профиль политики пропускной способности на странице "Профиль **политики"** группы **конфигурации** сети. Подробные сведения см. в управлении [профилями политики пропускной способности сети.](managing-network-bandwidth-policy-profiles.md)

8.  (Необязательно) Если требуется предоставить параметры местоположения для данного узла, выберите политику расположения в раскрывающемся списке **Политика расположения**.

    > [!NOTE]  
    > Политика расположения устанавливает определенные параметры системы Enhanced 9-1-1 (E9-1-1) и клиентского расположения для узла. Вы можете просмотреть сведения о доступных политиках или создать новую на странице **Политика расположения** группы **Конфигурация сети**. 

9.  (Необязательно) Введите значение в поле **Описание**, чтобы предоставить дополнительную информацию об этом узле, которую нельзя выразить в имени.

10. Нажмите кнопку **Сохранить**.

    > [!NOTE]  
    > Таблица **Связанные подсети** не используется при создании нового сетевого узла. Необходимо связать подсеть с узлом при создании или изменении подсети. Подробные сведения см. [в подсетях управления.](managing-network-subnets.md)

### <a name="to-modify-a-network-site"></a>Изменение сетевого узла

1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server. 

3.  В левой панели навигации щелкните **"Конфигурация сети"** и выберите **"Сайт".**

4.  На странице **Узел** щелкните узел, который требуется изменить.

5.  В меню **Правка** выберите команду **Показать подробности**.

6.  На странице **Редактирование узла** вы можете изменить описание, область, профиль политики полосы пропускания и политику расположения, связанную с узлом. Дополнительные сведения см. в [подсети "Создание сетевого сайта"](#to-create-a-network-site) выше.

7.  Нажмите кнопку **Сохранить**.

Вы не можете изменить таблицу **Связанные подсети** на этой странице. Список связанных подсетей предоставлен для справки, чтобы вы могли узнать, на что повлияет изменение параметров узла.


## <a name="delete-an-existing-network-site"></a>Удаление существующего сетевого сайта

Сетевые узлы — это офисы или места, настроенные в каждой области развертывания контроля допуска звонков (CAC) или Enhanced 9-1-1. Панель управления Skype для бизнеса Server можно использовать для настройки сайтов и связываия их с регионами. Например, сетевую область для Северной Америки можно связать с такими сетевыми узлами, как Чикаго, Редмонд и Ванкувер. Сетевой узел CAC необходимо создать для каждого узла в организации, даже если у этого узла нет ограничений по пропускной способности. На панели управления Skype для бизнеса Server можно создавать, изменять и удалять сетевые сайты. Используйте следующую процедуру для удаления существующего сетевого сайта. Дополнительные сведения о создании или изменении сетевых сайтов см. в управлении контролем допуска [вызовов для сайтов.](managing-call-admission-control-for-sites.md)


### <a name="to-delete-a-network-site"></a>Удаление сетевого узла

1.  Из учетной записи пользователя, которая является членом группы RTCUniversalServerAdmins (или имеет эквивалентные права пользователя) или назначена роли CsAdministrator, войдите на любой компьютер во внутреннем развертывании.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server. 

3.  В левой панели навигации щелкните **"Конфигурация сети"** и выберите **"Сайт".**

4.  На странице **Узел** щелкните узел, который требуется удалить.

    > [!NOTE]  
    > Вы можете удалить несколько узлов за раз. Для этого нажмите клавишу CTRL и выберите несколько узлов, удерживая клавишу CTRL нажатой. Или, чтобы выбрать все узлы, щелкните **Выбрать все** в меню **Правка**.

5.  В меню **Правка** выберите команду **Удалить**.

6.  Нажмите кнопку **ОК**.
    

    > [!WARNING]  
    > Вы не можете удалить сетевой узел, если он связан с подсетью. При попытке удаления такого узла отображается сообщение об ошибке. Чтобы увидеть, связал ли узел с какой-либо подсетью, щелкните узел и выберите команду **Показать подробности** в меню **Правка**.
