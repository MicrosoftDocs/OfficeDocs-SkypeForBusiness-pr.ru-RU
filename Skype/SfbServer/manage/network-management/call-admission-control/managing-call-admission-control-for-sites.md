# <a name="managing-call-admission-control-for-sites-in-skype-for-business-server"></a>Управление контролем допуска звонков для сайтов в Skype для бизнеса Server

Сетевые узлы не офисы или расположения в каждом регионе сети для контроля допуска звонков (CAC), E9-1-1 и развертываний обход мультимедиа. Используйте процедуры, описанные в этой статье Настройка контроль допуска звонков для сетевых узлов.

## <a name="configure-network-site-links"></a>Настройка ссылок на сайты сети

В конфигурации контроля допуска допуска звонков можно создать сети между сайтами политик, которые определяют ограничения пропускной способности между сайтами, напрямую связанными. Если сетевые узлы имеют прямая ссылка, может быть определен ограничения пропускной способности для подключения к аудио- и видеоконференций между этих двух сайтов. Скайп для панели управления Business Server нельзя использовать для настройки сетевые политики сайта, это можно сделать только с помощью командлетов из Скайп для консоли Business Server. Создавать, изменять и удалять связи между сетевыми узлами (также известной как сетевая межсайтовая политика) из Скайп для консоли Business Server.

### <a name="to-create-a-network-site-link"></a>Создание связи между сетевыми узлами

1.  Войдите на компьютер, где установлена Скайп для консоли Business Server как член группы RTCUniversalServerAdmins или с необходимые права пользователя.

2.  Запустите Скайп для консоли Business Server: нажмите кнопку **Пуск**, последовательно выберите пункты **Все программы**, щелкните **Скайп для Business Server**и нажмите кнопку **Скайп для консоли Business Server**.

3.  В командной строке введите следующую команду, подставив значения, подходящие для вашей конфигурации:
    
        New-CsNetworkInterSitePolicy -Identity Reno_Portland -NetworkSiteID1 Reno -NetworkSiteID2 Portland -BWPolicyProfileID LowBWLimits
    
    В этом примере создается новый связи между сетевыми узлами с именем Reno\_Портленд, который задает ограничения пропускной способности между сетевыми узлами Reno и Портленд. Сетевые узлы и профиль политики пропускной способности должны уже существовать перед выполнением этой команды.

Параметр подробные описания в разделе [New-CsNetworkInterSitePolicy](https://docs.microsoft.com/powershell/module/skype/New-CsNetworkInterSitePolicy). Чтобы получить список профилей политики пропускной способности, которые могут быть применены к связи между сетевыми узлами, вызовите командлет **Get-CsNetworkBandwidthPolicyProfile** . Дополнительные сведения см [Get-CsNetworkBandwidthPolicyProfile](https://docs.microsoft.com/powershell/module/skype/Get-CsNetworkBandwidthPolicyProfile).

### <a name="to-modify-a-network-site-link"></a>Изменение связи между сетевыми узлами

1.  Войдите на компьютер, где установлена Скайп для консоли Business Server как член группы RTCUniversalServerAdmins или с необходимые права пользователя.

2.  Запустите Скайп для консоли Business Server: нажмите кнопку **Пуск**, последовательно выберите пункты **Все программы**, щелкните **Скайп для Business Server**и нажмите кнопку **Скайп для консоли Business Server**.

3.  Командлет **Set-CsNetworkInterSitePolicy** для изменения свойств связи между сетевыми данного сайта. Можно изменить один (или оба) или связи сайтов и можно изменить профиль политики пропускной способности, связанные со ссылкой. Ниже приведен пример изменения профиля политики пропускной способности связи сайтов с именем Reno\_Портленд:
    
        Set-CsNetworkInterSitePolicy -Identity Reno_Portland -BWPolicyProfileID HighBWLimits

Параметр подробные описания в разделе [Set-CsNetworkInterSitePolicy](https://docs.microsoft.com/powershell/module/skype/Set-CsNetworkInterSitePolicy).


### <a name="to-delete-a-network-site-link"></a>Удаление связи между сетевыми узлами

1.  Войдите на компьютер, где установлена Скайп для консоли Business Server как член группы RTCUniversalServerAdmins или с необходимые права пользователя.

2.  Запустите Скайп для консоли Business Server: нажмите кнопку **Пуск**, последовательно выберите пункты **Все программы**, щелкните **Скайп для Business Server**и нажмите кнопку **Скайп для консоли Business Server**.

3.  Командлет **Remove-CsNetworkInterSitePolicy** используется для удаления связи между сетевыми узлами. В следующем примере удаляется Reno\_Портленд связи между сетевыми узлами:
    
        Remove-CsNetworkInterSitePolicy -Identity Reno_Portland

Параметр подробные описания в разделе [Remove-CsNetworkInterSitePolicy](https://docs.microsoft.com/powershell/module/skype/Remove-CsNetworkInterSitePolicy).


## <a name="view-network-site-information"></a>Просмотр сведений о сетевом сайте

Сетевые узлы являются офисы или филиалы, настроенных в каждом регионе контроля допуска звонков (CAC) или развертывания Enhanced 9-1-1. Можно просмотреть сведений о сетевых сайтах в любом Скайп для панели управления Business Server или Скайп для консоли Business Server. 

### <a name="to-view-network-site-information-in-skype-for-business-server-control-panel"></a>Для просмотра сведений о сетевых сайтах в Скайп для панели управления Business Server

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и нажмите кнопку **сайта**.

4.  На странице **сайта** щелкните сайт, который нужно просмотреть.
 
    > [!NOTE]  
    > Можно просматривать только сведения для одного сайта за раз.

5.  В меню **Правка** щелкните **Подробнее**.


### <a name="viewing-network-site-information-by-using-windows-powershell-cmdlets"></a>Просмотр сведений о сетевом сайте с помощью командлетов Windows PowerShell

Сведений о сетевом сайте можно просмотреть с помощью Windows PowerShell и командлет Get-CsNetworkSite. Этот командлет можно запустить из Скайп для консоли Business Server или из удаленного сеанса Windows PowerShell. 

### <a name="to-view-network-site-information"></a>Для просмотра сведений о сетевом сайте

  - Чтобы просмотреть сведения обо всех сетевых узлов, введите следующую команду в Скайп для консоли Business Server и нажмите клавишу ВВОД:
    
        Get-CsNetworkSite
    
    Команда возвращает примерно следующую информацию:
    
        Identity          : Redmond
        NetworkSiteID     : Redmond
        Description       :
        NetworkRegionID   : Pacific Northwest
        BypassID          : 3b232b84-2c1d-4da2-8181-e9330bafebe9
        BWPolicyProfileID :
        LocationPolicy    :

Для получения дополнительных сведений см раздел справки для командлета [Get-CsNetworkSite](https://docs.microsoft.com/powershell/module/skype/Get-CsNetworkSite) .


## <a name="create-or-modify-network-sites"></a>Создание или изменение сетевых узлов 

Сетевые узлы являются офисы или филиалы, настроенных в каждом регионе контроля допуска звонков (CAC) или развертывания Enhanced 9-1-1. Скайп для панели управления Business Server можно использовать для настройки сайтов и свяжите их с областями. К примеру области сети в Северной Америке могут быть связаны с сайтами сетей, таких как Чикаго, Redmond и Vancouver. С сетевым узлом контроля допуска звонков необходимо создать для каждого сайта в пределах организации, даже в том случае, если этот сайт имеет без ограничений пропускной способности. Из Скайп для панели управления Business Server можно создавать, изменять и удалять сетевых узлов. Используйте следующие процедуры для создания или изменения с сетевым узлом. 

### <a name="to-create-a-network-site"></a>Создание сетевого узла

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и нажмите кнопку **сайта**.

4.  На странице " **сайт** " нажмите кнопку **Создать**.

5.  **Новый узел**введите имя для этого сайта в поле **имя** .

    > [!NOTE]  
    > Имена узлов должны быть уникальным в рамках Скайп для развертывания Business Server.

6.  В раскрывающемся списке **область** выберите область сети для связи с этого сайта.

7.  (Необязательно) Если вы хотите наложить ограничения полосы пропускания на аудио- или видеозвонка звонков на этот сайт, выберите профиль политики пропускной способности с соответствующими параметрами из раскрывающегося списка **Политика пропускной способности** .
 
    > [!NOTE]  
    > Можно просмотреть сведения об профилей политики пропускной или создать новый профиль политики пропускной способности, на странице " **Профили политики** " группе **Конфигурация сети** . Дополнительные сведения см [Управление профилей политики пропускной способности сети](managing-network-bandwidth-policy-profiles.md).

8.  (Необязательно) Если вы хотите предоставить параметры местоположения для данного узла, выберите политику расположения в раскрывающемся списке **Политика расположения** .

    > [!NOTE]  
    > Политика расположения назначает конкретных Enhanced 9-1-1 (E9-1-1) и клиентских параметры расположения сайта. Можно просмотреть сведения об политик расположения недоступны или создать новую политику расположения, на странице **Политика расположения** группе **Конфигурация сети** . 

9.  (Необязательно) Введите значение в поле **Описание** введите дополнительные сведения об этом сайте, которые не могут быть выражены одним именем.

10. Нажмите **Исполнить**.

    > [!NOTE]  
    > В таблице **Связанные подсети** не используется для создания нового сетевого узла. Связывание подсети с сайтом при создании или изменении подсеть. Дополнительные сведения см [Управление подсетей](managing-network-subnets.md).

### <a name="to-modify-a-network-site"></a>Изменение сетевого узла

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и нажмите кнопку **сайта**.

4.  На странице **сайта** щелкните сайт, который требуется изменить.

5.  В меню **Правка** щелкните **Подробнее**.

6.  На странице **Изменение узла** можно изменить описание, региона, профиль политики пропускной способности и политику расположения, связанного с сайтом. Дополнительные сведения см [в Создание сетевого узла](#to-create-a-network-site) выше.

7.  Нажмите **Исполнить**.

В таблице **Связанные подсети** на этой странице изменить нельзя. Список связанных подсетей предоставляется для ссылки, чтобы знать о том, какие подсети затрагиваются при изменении параметров сайта.


## <a name="delete-an-existing-network-site"></a>Удаление существующего сетевого узла

Сетевые узлы являются офисы или филиалы, настроенных в каждом регионе контроля допуска звонков (CAC) или развертывания Enhanced 9-1-1. Скайп для панели управления Business Server можно использовать для настройки сайтов и свяжите их с областями. К примеру области сети в Северной Америке могут быть связаны с сайтами сетей, таких как Чикаго, Redmond и Vancouver. С сетевым узлом контроля допуска звонков необходимо создать для каждого сайта в пределах организации, даже в том случае, если этот сайт имеет без ограничений пропускной способности. Из Скайп для панели управления Business Server можно создавать, изменять и удалять сетевых узлов. Используйте следующую процедуру для удаления существующего сетевого узла. Для получения дополнительных сведений о создании или изменение сетевых узлов видеть [Управление контроля допуска звонков для сайтов](managing-call-admission-control-for-sites.md).


### <a name="to-delete-a-network-site"></a>Удаление сетевого узла

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Конфигурация сети**и нажмите кнопку **сайта**.

4.  На странице **сайта** щелкните сайт, который требуется удалить.

    > [!NOTE]  
    > Одновременно можно удалить несколько узлов. Для этого нажмите клавишу CTRL и выберите несколько сайтов, удерживая нажатой клавишу CTRL. Или, чтобы выбрать все сайты, нажмите кнопку **Выбрать все** в меню **Правка** .

5.  В меню **Правка** выберите пункт **Удалить**.

6.  Нажмите кнопку **ОК**.
    

    > [!WARNING]  
    > Вы не можете удалить с сетевым узлом, если он связан с подсетью сети. Если попытаться удалить сайт, связанный с подсети вы получите сообщение об ошибке. Чтобы просмотреть, если сайт связан с любой подсети, щелкните сайт и нажмите кнопку **Показать подробности** в меню **Правка** .
