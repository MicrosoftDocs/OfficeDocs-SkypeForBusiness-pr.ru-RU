# <a name="configuring-port-ranges-and-a-quality-of-service-policy-for-your-conferencing-application-and-mediation-servers"></a>Настройка диапазонов портов и политики качества обслуживания для серверов конференц-связи, приложений и посредника

В этой статье описывается настройка диапазонов портов и политики качества обслуживания для серверов конференц-связи, приложений и посредника.

## <a name="configure-port-ranges"></a>Настройка диапазонов портов

Для реализации качества обслуживания, следует настроить диапазоны портов идентичными для аудио-, видео и совместный доступ к данным на серверах конференц-связи, приложений и посредник; Кроме того эти диапазоны портов не должен перекрывать никаких уведомлений. Чтобы использовать простой пример, предположим, что используют порты 10000 через 10999 для видео на серверах конференц-связи. Это означает, что необходимо зарезервировать порты 10000 через 10999 для видео на серверах приложений и посредника. Если вы не задан, QoS будет работать неправильно.

Аналогично Предположим, что зарезервировать порты 10000 через 10999 для видео, но резерв порты 10500 через 11999 для аудио. Это может вызвать проблемы для качества обслуживания, так как номер порта для диапазона перекрытия. С помощью QoS на каждой модальности должны иметь уникальный набор порты: Если используют порты 10000 через 10999 для видео, то необходимо использовать другой диапазон (например, 11000 через 11999 для звук).

По умолчанию диапазоны портов аудио- и видеоконференций не пересекаются в Скайп для Business Server; Тем не менее диапазоны портов, назначенный приложению, общий доступ к перекрытия с диапазоны портов аудио- и видеоконференций. (Что, в свою очередь, означает, что ни один из этих диапазонов не уникальный.) Вы можете проверить существующих диапазонов портов для серверов конференц-связи, приложений и посредником, выполнив следующие три команды из в пределах Скайп для консоли Business Server:

    Get-CsService -ConferencingServer | Select-Object Identity, AudioPortStart, AudioPortCount, VideoPortStart, VideoPortCount, AppSharingPortStart, AppSharingPortCount
    
    Get-CsService -ApplicationServer | Select-Object Identity, AudioPortStart, AudioPortCount
    
    Get-CsService -MediationServer | Select-Object Identity, AudioPortStart, AudioPortCount

> [!WARNING]  
> Как видно в предыдущем команды, каждый тип порта — аудио-, видео и общий доступ к приложениям — назначается два значения отдельных свойств: количество портов и запуск порта. Начальный порт указывает первый порт, используемый для этого модальности; к примеру Если Пуск портов для передачи аудио равен 50000, которое означает, что первый порт, используемый для трафика аудио — порт 50000. Если число портов для передачи аудио, 2 (который не является допустимым значением, но здесь используется для иллюстрации), это означает, что только два порта выделенная для аудиокомпонентов. Если первый порт — порт 50000 и в общей сложности состоит из двух портов, это означает, что второй порт должен быть порта 50001 (порта, который диапазонов должны быть непрерывными). В результате диапазон портов для звук будет порты 50000 через 50001 включительно.<BR><br>Обратите внимание, что сервер приложений и сервер-посредник поддерживают только качества обслуживания для аудио; необходимо изменить видео или портов в приложении серверов или серверов-посредников общего доступа к приложениям.

Как указывалось ранее, при настройке Скайп Business Server портов для QoS, следует убедиться, что: 1) портов для передачи аудио параметры идентичны между вашими конференц-связи, приложений и посредника серверов; и, 2) диапазоны портов не перекрывались. Если внимательно изучите приведенной выше таблице, будет доступно, что диапазоны портов идентичны для всех типов три сервера. Например начальный порт звука задано порту 49152 на каждый тип сервера, а также идентичен общее число портов зарезервировано для звука на каждом сервере: 8348. Тем не менее, порт для диапазона перекрытие: порты начать с использованием порта 49152, но таким образом выполните те порты, которые выделяемых для общего доступа к приложениям. Чтобы обеспечить оптимальное использование качества обслуживания, общий доступ к приложениям допускать диапазон порта. Например можно настроить общий доступ на порт 40803 и используйте 8348 портов. (Почему 8348 порты? Если вы добавляете эти значения друг с другом--40803 + 8348--, который означает, что общий доступ к приложениям будут использовать порты 40803 через порт 49150. Так как порты не начинается до порт 49152, вы больше не будут иметь любое перекрывающиеся диапазонов портов.)

После выбора нового диапазон портов для общего доступа к приложениям, можно внести изменения с помощью командлета Set-CsConferencingServer. Это изменение не должны быть выполнены на серверах приложений или на серверах посредника, так как эти серверы не обрабатывают трафик общего доступа приложения. Необходимо изменить значения портов на эти серверы, если требуется переназначить портов, используемых для трафика аудио.

Чтобы изменить значения портов для совместного использования на одном сервере конференц-связи, выполните команду, аналогичную ниже, из внутри Скайп для консоли Business Server:

    Set-CsConferenceServer -Identity ConferencingServer:atl-cs-001.litwareinc.com -AppSharingPortStart 40803 -AppSharingPortCount 8348

Если вы хотите внести эти изменения на всех серверах конференций, можно вместо этого выполните следующую команду:

    Get-CsService -ConferencingServer | ForEach-Object {Set-CsConferenceServer -Identity $_.Identity -AppSharingPortStart 40803 -AppSharingPortCount 8348}

После изменения параметров портов, необходимо остановить и перезапустите каждой службы, затрагиваемых этими изменениями.

Не является обязательным, серверы, серверы приложений и серверов-посредников общий доступ к точной тот же диапазон портов; только значение true, требование заключается в том, что вы выделяемых диапазонов порта на всех серверах. Тем не менее Администрирование, обычно считаются более удобен, если вы используете один и тот же набор портов на всех серверах.

## <a name="configure-a-quality-of-service-policy-in-skype-for-business-server-for-your-conferencing-application-and-mediation-servers"></a>Настройка политики качества обслуживания в Скайп для Business Server для серверов конференц-связи, приложений и посредника

Настройка диапазонов портов упрощает использование режима качества обслуживания, чтобы убедиться, что все указанного типа (например, аудио трафик) проходит через один и тот же набор портов. Это упрощает для системы, чтобы определить и отметить указанного пакета: Если порт 49152 зарезервирован для трафика аудио, а затем все пакеты, которые отправляются через порт 49152 можно пометить кодом DSCP, который указывает, что пакет звука. В свою очередь это позволяет маршрутизаторы определение пакет как пакет звуковых и задайте для него высокий приоритет, чем помечены пакетов (например, пакетов, которые используются для копирования файла с одного сервера на другой).

Тем не менее просто ограничение набор портов определенному типу трафика не приводят пакетов, которые отправляются через порты отмеченные с надлежащего кода DSCP. В дополнение к определению диапазоны портов, необходимо также создать политики качества обслуживания, которые задают DSCP код, чтобы быть связан с каждой диапазон портов. Для Скайп для Business Server, который обычно означает, что создание две политики: для аудио и видео.

Политик качества обслуживания, проще создавать и управлять, с помощью групповой политики. (Эти же политики могут также создаваться с помощью локальной политики безопасности. Тем не менее, который требует повторите ту же процедуру на каждый компьютер.) Вашей начальный набор политик качества обслуживания (для аудио) и одна для видео должны применяться только к Скайп для компьютеров Business Server, с сервера конференций, сервер приложений и/или служб Mediation server. Если все эти три компьютера, находятся в одном Подразделении Active Directory, можно просто назначить нового объекта групповой политики (GPO), Подразделение. Кроме того можно выполнить другие шаги для определения целевого новую политику для указанных компьютеров; к примеру можно поместить соответствующие компьютеры в группу безопасности, а затем используйте фильтры параметров безопасности групповой политики для применения объекта групповой Политики только к этой группе безопасности.

Чтобы создать политику качества обслуживания для управления аудио, войдите на компьютер, где была установлена консоль Управление групповой политикой. Откройте консоль Управление групповой политикой (нажмите кнопку **Пуск**, выберите пункт **Администрирование**и выберите **Управление групповой политикой**) и затем выполните следующие действия:

1.  В консоли Управление групповой политикой Найдите контейнер, где должен быть создан новой политики. К примеру Если все Скайп для компьютеров Business Server, находятся в Подразделении с именем Скайп для Business Server, затем новой политики следует создавать в Скайп для Подразделения Business Server.

2.  Щелкните правой кнопкой мыши соответствующий контейнер и нажмите кнопку **создать объект GPO в этом домене и связать его**.

3.  В диалоговом окне **Новый объект групповой Политики** введите имя для нового объекта групповой политики в поле **имя** (например, **Скайп для Business Server QoS**) и нажмите кнопку **ОК**.

4.  Щелкните правой кнопкой мыши созданную политику и нажмите кнопку **Изменить**.

5.  В редакторе управления групповой политики разверните узлы **Конфигурация компьютера**, затем узел **политики**, параметры **Windows**, щелкните правой кнопкой мыши **QoS на основе политики**и выберите команду **Создать новую политику**.

6.  В диалоговом окне **QoS на основе политики** на первой странице введите имя новой политики (например, **Скайп для Business Server QoS**) в поле **имя** . Выберите **Укажите значение DSCP** и задайте значение **46**. Оставьте в поле **Укажите исходящих частоту передачи** не выбран и нажмите кнопку **Далее**.

7.  На следующей странице убедитесь в том, что выбран пункт **все приложения** и нажмите кнопку **Далее**. Это просто гарантирует, что все приложения будет соответствовать пакеты из указанного диапазона с указанным кодом DSCP.

8.  На странице "третий" Убедитесь, что выбраны оба **любой исходный IP-адрес и любой конечный IP-адрес** и нажмите кнопку **Далее**. Эти два параметра убедитесь, что пакеты будут управляться независимо от того, какой компьютер пакеты отправляемых (IP-адрес) и (IP-адрес) компьютер, который будет получать эти пакеты.

9.  На странице четырех выберите в раскрывающемся списке **выберите протокол, к которому применяется данная политика качества обслуживания для** **TCP и UDP-ПОРТ** . TCP (Transmission Control Protocol) и UDP-ПОРТ (протокол) — это два сетевых протоколов, чаще всего используемые Скайп Business Server и его клиентских приложений.

10. В разделе **Укажите номер порта источника**выберите выберите **этот исходный порт или диапазон**. В соответствующем текстовом поле введите диапазон портов зарезервированы для передачи звука. Например, если зарезервированные порты 49152 через порты 57500 для трафика аудио, введите диапазон портов, используя следующий формат: **49152:57500**. Нажмите **Готово**.

> [!NOTE]  
> Значение DSCP 46 степени произвольно: несмотря на то, что часто используется в DSCP 46 Пометка аудиопакетов, нет необходимости использовать DSCP 46 для звуковых коммуникаций. Если уже был реализован качества обслуживания и другой код DSCP используется для аудио (например, 40 DSCP), необходимо настроить политики качества обслуживания для использования этого же кода (например, 40 звук). Если вы только что теперь внедряются качества обслуживания, а затем рекомендуется использовать DSCP 46 для работы со звуком просто так, как это значение обычно используется для пометки аудиопакетов.

После создания политики качества обслуживания для трафика аудио следует создайте второй политики для видео трафика (и, возможно, третий политику для управления общего доступа трафика приложения). Создание политики для видео, выполните ту же основную процедуру, были выполнены при создании политики звука, что следующие изменения:

  - Используйте другое (и уникальное) политики имя (например, **Скайп для видео Business Server**).

  - Установите значение DSCP **34** вместо 46. (Обратите внимание на то, что нет необходимости использовать значение DSCP 34. Единственное требование — это использовать разные значения DSCP видео, не используемых для совершения.)

  - Используйте диапазон портов ранее настроенные для видео-трафика. Например, если зарезервированные порты 57501 до 65 535 для видео, необходимо задать диапазон портов на этот: **57501:65535**.

Если вы решите создать политику для управления трафика общего доступа приложения, необходимо создать третью политику, внеся следующие изменения:

  - Используйте другое (и уникальное) политики имя (например, **Скайп для Business Server общий доступ к приложениям**).

  - Задайте значение DSCP **24** вместо 46. (Еще раз, вам не нужно использовать значение DSCP 24. Единственное требование — это использовать разные значения DSCP для общего доступа к приложениям, не используется для аудио или видео.)

  - Используйте диапазон портов ранее настроенные для видео-трафика. Например, если зарезервированные порты 40803 до 49151 для общего доступа к приложениям, необходимо задать диапазон портов на этот: **40803:49151**.

Новые политики, созданные вами, не вступят в силу только после обновления групповой политики на вашей Скайп для компьютеров Business Server. Хотя групповая политика периодически обновляется сама, вы можете обновить ее принудительно, запустив на каждом нужном компьютере следующую команду.

    Gpupdate.exe /force

Эту команду можно выполнить из внутри Скайп для консоли Business Server или из любого окно командной строки, на котором выполняется с использованием учетных данных администратора. Чтобы открыть такое окно с правами администратора, в меню **Пуск** правой кнопкой мыши щелкните **Командная строка** и выберите пункт **Запуск от имени администратора**.

Чтобы убедиться, что были применены новые политики QoS, выполните следующие действия.

1.  На Скайп для сервера нажмите кнопку **Пуск**и выберите команду **выполнить**.

2.  В диалоговом окне **выполнить** введите **regedit**и нажмите клавишу ВВОД.

3.  В редакторе реестра разверните узлы **компьютера**, **HKEY\_ЛОКАЛЬНОГО\_МАШИНЫ**, разверните узел **программного обеспечения**, затем узел **политики**, разверните **Microsoft**, разверните узел **Windows**и нажмите кнопку **качества обслуживания**. В разделе **QoS** вы должны увидеть разделы реестра для каждой политики QoS, которую вы только что создали. Например при создании две новые политики (один именованный Скайп для Business Server звук QoS) и именованные Скайп для QoS видео Business Server, вы увидите записей реестра для Скайп QoS звук Business Server и Скайп для QoS видео Business Server.

Чтобы обеспечить, что сетевых пакетов соответствующим значением DSCP, следует также создать новый раздел реестра на каждом компьютере, выполнив следующую процедуру:

1.  В меню **Пуск** выберите пункт **Выполнить**.

2.  В диалоговом окне **выполнить** введите **regedit**и нажмите клавишу ВВОД.

3.  В редакторе реестра разверните **HKEY\_ЛОКАЛЬНОГО\_МАШИНЫ**, разверните узлы **системы**, разверните **CurrentControlSet**, разверните узел **службы**и затем **Tcpip**.

4.  Щелкните правой кнопкой мыши **Tcpip**, выберите команду **Создать**и затем щелкните **раздел**. После создания нового раздела реестра введите **качества обслуживания**и нажмите клавишу ВВОД, чтобы переименовать раздел.

5.  Щелкните правой кнопкой мыши **QoS**, выберите команду **Создать**и затем **Строковый параметр**. После создания нового значения реестра введите **не используйте NLA**и нажмите клавишу ВВОД, чтобы переименовать значение.

6.  Дважды щелкните **не использовать NLA**. В диалоговом окне **Изменение строкового параметра** в поле **значение** введите **1** и нажмите кнопку **ОК**.

7.  Закройте редактор реестра и перезагрузите компьютер.
