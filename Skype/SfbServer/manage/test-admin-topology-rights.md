## <a name="description"></a>Описание

По умолчанию только администраторы домена могут включать топологию сервера Skype для бизнеса и вносить значительные изменения в инфраструктуру Skype для бизнеса Server. Это не является проблемой, поскольку администраторы домена и администраторы вашего сервера Skype для бизнеса должны быть и теми же. Во многих организациях администраторы сервера Skype для бизнеса не сохраняют права администратора для всего домена. По умолчанию, это означает, что администраторы (определенные как члены группы Рткуниверсалсерверадминс) не могут вносить изменения в топологию сервера Skype для бизнеса Server. Чтобы предоставить участникам группы Рткуниверсалсерверадминс право вносить изменения в топологию, необходимо назначить необходимые разрешения Active Directory с помощью командлета [Grant-кссетуппермиссион](https://docs.microsoft.com/en-us/powershell/module/skype/Grant-CsSetupPermission) .
 
Командлет Test-Кссетуппермиссион удостоверяется в том, что необходимые разрешения, необходимые для установки сервера Skype для бизнеса Server или одного из его компонентов, настроены в указанном контейнере Active Directory. Если разрешения не назначены, можно запустить командлет Grant-Кссетуппермиссион, чтобы предоставить участникам группы Рткуниверсалсерверадминс право устанавливать и включать Skype для бизнеса Server.

## <a name="running-the-test"></a>Выполнение теста

Чтобы определить, назначены ли разрешения на установку для контейнера Active Directory, вызовите командлет Test-Кссетуппермиссион. Укажите различающееся имя контейнера, который нужно проверить. Например, эта команда проверяет разрешения на установку в контейнере OU = Кссерверс, DC = плана litwareinc, DC = com:

`Test-CsSetupPermission -ComputerOU "ou=CsServers,dc=litwareinc,dc=com"`

Дополнительные сведения можно найти в разделе справки по командлету [Test-кссетуппермиссион](https://docs.microsoft.com/en-us/powershell/module/skype/Test-CsSetupPermission) .

## <a name="determining-success-or-failure"></a>Определение успеха или сбоя

Если в параметре test-Кссетуппермиссион определено, что необходимые разрешения уже установлены для контейнера Active Directory, командлет вернет значение true:

True 

Если разрешения не заданы, Test-Кссетуппермиссион вернет значение false. Обратите внимание, что это значение обычно заключается на множество предупреждающих сообщений. Например:

Предупреждение: элемент управления доступом (ACE) atl-cs-001\RTCUniversalServerAdmins; Пуск Екстендедригхт; Ничего Ничего 1131f6aa-9c07-11d1-f79f-00c04fc2dcd2 

Предупреждение: элементы управления доступом (ACE) для объекта "CN = Computers, DC = плана litwareinc, DC = com" не готовы. 

False 

Предупреждение: обработка "Test-Кссетуппермиссион" завершена с предупреждениями. во время этого запуска записано предупреждение "2". 

Предупреждение: подробные результаты можно найти по адресу "C:\Users\Admin\AppData\Local\Temp\Test-CsSetupPermission-1da99ba6-abe2-45e4-8b16-dfd244763118.html". 

## <a name="reasons-why-the-test-might-have-failed"></a>Причины, по которым может произойти сбой теста

Несмотря на то, что при выполнении теста-Кссетуппермиссион возникает редкие исключения, обычно это означает, что для определенного контейнера Active Directory разрешения на установку не назначены. Эти разрешения можно назначить с помощью командлета Grant-Кссетуппермиссион. Например, эта команда предоставляет разрешения на установку контейнера Computers в litwareinc.com домена.

`Grant-CsSetupPermission -ComputerOU "cn=Computers,dc=litwareinc,dc=com"`

Дополнительные сведения можно найти в разделе справки по командлету [Test-кссетуппермиссион](https://docs.microsoft.com/en-us/powershell/module/skype/Test-CsSetupPermission) .
