# <a name="manage-windows-updates"></a>Управление обновлениями Windows

Комнаты Microsoft Teams работают в Windows 10 Корпоративная IoT или Windows 10 Enterprise (Корпоративная лицензия) и получают одинаковые обновления Windows и операционные системы для обычных рабочих столов.

Для управления обновлениями Windows можно использовать несколько различных способов:

## <a name="hands-off-approach"></a>Подход к практическим занятиям 
- Обновления загружаются непосредственно из обновлений Windows автоматически и устанавливаются в нерабочее время. Если в конфигурации не вносятся изменения, это состояние по умолчанию.
- Обновления, не связанные с деферрабле, устанавливают день – один из выпускаемых автоматически. 
- Обновления качества и драйверы загружают и устанавливают один и тот же день, автоматически. 
- Обновления компонентов. Ознакомьтесь с дополнительными заметками ниже. 

## <a name="windows-updates-for-businesshttpsdocsmicrosoftcomwindowsdeploymentupdatewaas-manage-updates-wufb-gpo-or-intune"></a>[Обновления Windows для бизнеса](https://docs.microsoft.com/windows/deployment/update/waas-manage-updates-wufb) (GPO или Intune)   
- Обновления загружаются из WU или WSUS, но с настроенными задержками после первоначальной даты выпуска базы знаний. 
- В сочетании с несколькими политиками организационных подразделений и фильтрацией это позволяет создавать "кольца", где администраторы могут указать, какие устройства будут устанавливать обновления качества, а какие — устанавливать позже. Это обеспечивает надежность и тестирование производительности в подмножестве систем перед развертыванием обновлений в рамках всего развертывания без дополнительных издержек, таких как управление обновлениями Windows в SCCM.
- Обновления WSUS и Windows для бизнеса можно [настроить одновременно,](https://docs.microsoft.com/windows/deployment/update/waas-integrate-wufb) если вам требуется как управление пропускной способностью, так и контроль обновлений Windows для бизнеса.
- Обновления компонентов. Ознакомьтесь с дополнительными заметками ниже.

## <a name="wsussccmhttpsdocsmicrosoftcomwindowsdeploymentupdatewaas-manage-updates-configuration-manager"></a>[WSUS/SCCM](https://docs.microsoft.com/windows/deployment/update/waas-manage-updates-configuration-manager)
- Очень похоже на обновление Windows для бизнеса, но с дополнительным вариантом выбора конкретных KБ в каждом из них, а также всего развертывания. Каждое обновление можно разворачивать и тестировать отдельно, а не полагаться на задержку. 
- Обновления компонентов. Ознакомьтесь с дополнительными заметками ниже.


### <a name="feature-updates"></a>Обновления компонентов

В отличие от качества и неотложенных обновлений, Windows 10 "обновления компонентов" (основные выпуски ОС) устанавливаются только после Майкрософт и проверяются на наличие заданных функций обновления с помощью комнат Microsoft Teams. Несмотря на то, что она выпускается на полугодовой канал (или предназначена для тестирования на канале, а также в том случае, если вы установили для него системы) или задали их вручную с помощью собственных попыток или конфигураций, она не будет разрешена до тех пор, пока вы не удалите блок на своем конце.

Комната Microsoft Teams "за пределами", при использовании подхода "Выкл." не будет устанавливать обновление для Windows или перезагружать устройства автоматически из-за обновления Windows. Однако система может загрузить обновление и подождать следующей перезагрузки, чтобы установить его. Если кто-то не перезагружает его вручную, установка должна выполняться автоматически ночью. Обновления Windows должны быть прозрачными в комнате, поэтому пользовательский интерфейс не должен прерываться обновлениями Windows.

Если вы хотите присоединиться к домену, используйте SCCM или WSUS и запомните особое внимание на политиках или действиях, которые могут привести к установке обновления или принудительной перезагрузке в рабочее время. Если в вашей среде развертывания есть системы, перезагружаемые во время использования или уведомляющие об обновлениях Windows по пользовательскому интерфейсу, вам нужно будет найти конфигурацию.
