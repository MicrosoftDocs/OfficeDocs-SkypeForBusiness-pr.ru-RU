# <a name="multiple-trunk-support-in-skype-for-business-server"></a>Поддержка нескольких магистральных Skype для бизнеса Server

Skype для бизнеса Server поддерживает несколько ассоциаций между шлюзами и серверами-посредниками. Эти ассоциации сделаны путем определения магистрали, что является логической связью между пулом серверов-посредников и шлюзом открытой телефонной сети (PSTN), диспетчером пограничной связи сеанса (SBC) или IP-PBX. Используйте топологию Builder для связывать шлюзы с серверами-посредниками (то есть магистральными).

- Чтобы назначить или удалить магистраль в Skype для бизнеса Server, сначала необходимо определить магистраль в Topology Builder. Магистраль состоит из следующей ассоциации: Mediation Server, полностью квалифицированного доменного имени (FQDN), порта прослушивания сервера-посредника, шлюза FQDN и порта прослушивания шлюза.
- Чтобы настроить несколько магистральных серверов, можно создать несколько ассоциаций между тем же шлюзом и сервером-посредником. Это обеспечивает дополнительную устойчивость к Корпоративная голосовая связь инфраструктуре, которая особенно полезна в сценариях межоперации частного филиала exchange (PBX). 

После определения магистрали ее необходимо связать с маршрутом. Чтобы связать магистраль с маршрутом, необходимо определить простое имя магистрали в Topology Builder. Это простое имя используется в качестве имени магистрали в панели Skype для бизнеса Server, где магистрали могут быть связаны с маршрутами. Простое имя магистрали используется в качестве имени шлюза из Skype для бизнеса Server Management Shell.

`New-CsVoiceRoute -Identity <RouteId> -NumberPattern <String> -PstnUsages @{add="<UsageString>"} -PstnGatewayList @{add="<TrunkSimpleName>"}`

Администратор должен выбрать магистраль по умолчанию, связанный с сервером-посредником. В конструкторе топологии щелкните правой кнопкой мыши связанный сервер-посредник и нажмите кнопку **Свойства**. Укажите шлюз по умолчанию для сервера-посредника. 

На следующей схеме показано несколько магистральных серверов, определенных для каждого сервера-посредника и шлюза.
