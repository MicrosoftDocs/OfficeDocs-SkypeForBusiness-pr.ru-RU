# <a name="configure-a-trunk-without-media-bypass-in-skype-for-business-server"></a>Настройка магистрали без обхода мультимедиа в Skype для бизнеса Server

Если вы хотите настроить магистраль с обходом сервера-посредника, выполните эти действия. Если вы хотите настроить магистраль с включенным обходом мультимедиа, см. в этой ссылке Настройка магистрали с обходом мультимедиа [в Skype для бизнеса Server](configure-a-trunk-with-media-bypass.md).

Конфигурация магистрали представляет собой набор параметров, применяемых к магистрали, которой назначается эта конфигурация. Определенная конфигурация магистрали может применяться глобально (ко всем магистралям, у которых нет особых настроек сайта или пула), к сайту или к пулу. Конфигурация магистрали уровня пула позволяет применить особую конфигурацию к отдельной магистрали.

**Настройка магистрали без обхода сервера-посредника**

1. Откройте Skype панели управления Busines Server.
3. В левой панели навигации щелкните **Маршрутизация голосовой связи**, а затем выберите **Настройка линии связи**.
4. На странице **Конфигурация магистрали** используйте один из следующих методов для настройки магистрали:
    - Дважды щелкните существующую магистраль (например, в **глобальную** магистраль), чтобы открыть диалоговое окно **Изменение конфигурации магистрали**.
    - Нажмите кнопку **Создать**, а затем выберите область для конфигурации новой магистрали:
        - **Магистраль сайта:** Выберите сайт для этой конфигурации магистрали в **Выберите сайт,** а затем нажмите **кнопку ОК**. Обратите внимание, что сайты, для которых уже создана конфигурация магистрали, не отображаются в разделе **Выбор сайта**. Эта конфигурация будет применяться ко всем магистралям на сайте.
        - **Магистраль** пула. Выберите имя магистрали, к которое применяется эта конфигурация магистрали в **Выберите** службу и нажмите **кнопку ОК.** Этот магистраль может быть корневым стволом или любыми дополнительными магистральми, определенными в Topology Builder. Обратите внимание, что магистрали, для которых уже создана конфигурация, не отображаются в разделе **Выбор службы**.
    > [!Note] 
    > Выбранную область действия конфигурации магистрали уже нельзя изменить. В поле **Имя** указывается имя связанного с конфигурацией магистрали сайта или службы магистрали, это имя нельзя изменить. 

5. Выберите один из следующих параметров в поле **Уровень поддержки шифрования**:
    - **Требуется**. Необходимо использовать шифрование SRTP для защиты трафика между сервером-посредником и шлюзом или УАТС.
    - **Необязательный**: шифрование SRTP будет использоваться, если поставщик услуг или производитель оборудования поддерживает его.
    - **Не поддерживается**: шифрование SRTP не поддерживается поставщиком услуг или производителем оборудования, поэтому оно не будет использоваться.
6. Убедитесь, что флажок **Разрешить обход мультимедиа** снят.
7. Установите флажок **Централизованная обработка мультимедиа**, если существует известная точка терминирования медиаданных (например, шлюз телефонной сети общего пользования (ТСОП), где у точки терминирования такой же IP-адрес, как и у точки терминирования сигналов). Снимите этот флажок, если в магистрали нет известной точки терминирования медиаданных.
8. Если одноранговая магистраль поддерживает получение запросов SIP REFER от сервера-посредника, выберите ссылку Включить отправку на **шлюзовую** шкатулку.
9. (Необязательно) Чтобы включить маршрутизацию между магистральными линиями связи, сопоставьте и настройте записи режима работы с ТСОП для этой конфигурации магистральной линии связи. Использование PSTN, связанное с этой конфигурацией магистрали, будет применяться для всех входящих вызовов через магистраль, которая не Skype для бизнеса Server конечной точке. Чтобы управлять записями режима работы с ТСОП, связанными с конфигурацией магистральной линии связи, используйте один из приведенных ниже методов.
    - Чтобы выбрать одну или несколько записей из списка всех записей использования PSTN, доступных в Корпоративная голосовая связь развертывания, щелкните **Выберите**. Выделите записи, которые следует связать с этой конфигурацией магистральной линии связи, и нажмите кнопку **ОК**.
    - Чтобы удалить запись режима работы с ТСОП из этой конфигурации магистральной линии связи, выберите эту запись и щелкните **Удалить**.
    - Чтобы определить новую запись режима работы с ТСОП и сопоставить ее с этой конфигурацией магистральной линии связи, выполните следующие действия.
        1. Щелкните **Создать**.
        2. В поле **Имя** укажите уникальное описательное имя для записи.
            > [!Note] 
            > Имя записи режима работы с ТСОП должно быть уникальным в рамках развертывания корпоративной голосовой связи. После сохранения записи поле **Имя** изменить невозможно. 

        3. Используйте один из следующих методов для сопоставления и настройки маршрутов для этой записи режима работы с ТСОП:
            - Чтобы выбрать один или несколько маршрутов из списка всех доступных маршрутов в развертывании Корпоративная голосовая связь, щелкните **Выберите**. Выделите маршруты, которые хотите сопоставить с этой записью режима использования ТСОП, и нажмите кнопку **ОК**. 
            - Чтобы удалить маршрут из записи режима работы с ТСОП, выберите этот маршрут и щелкните **Удалить**.
            - Чтобы определить новый маршрут и сопоставить его с этой записью режима работы с ТСОП, щелкните **Создать**. 
            - Чтобы изменить маршрут, сопоставленный с этой записью режима работы с ТСОП, выберите маршрут и щелкните **Подробнее**. 
        4. Нажмите кнопку **ОК**.
    - Чтобы изменить запись режима работы с ТСОП, уже сопоставленную с этой конфигурацией магистральной линии связи, выполните следующие действия:
        1. Выберите запись режима работы с ТСОП, которую требуется изменить, и щелкните **Подробнее**.
        2. Используйте один из следующих методов для сопоставления и настройки маршрутов для этой записи режима работы с ТСОП:
            - Чтобы выбрать один или несколько маршрутов из списка всех доступных маршрутов в развертывании Корпоративная голосовая связь, щелкните **Выберите**. Выделите маршруты, которые хотите сопоставить с этой записью режима использования ТСОП, и нажмите кнопку **ОК**. 
            - Чтобы удалить маршрут из записи режима работы с ТСОП, выберите этот маршрут и щелкните **Удалить**.
            - Чтобы определить новый маршрут и сопоставить его с этой записью режима работы с ТСОП, щелкните **Создать**. 
            - Чтобы изменить маршрут, сопоставленный с этой записью режима работы с ТСОП, выберите маршрут и щелкните **Подробнее**. 
        3. Нажмите кнопку **ОК**.

    > [!Important] 
    > Важно связать записи использования PSTN в соответствии с одноранговой записью сервера-посредника, связанной с настраиваемой магистралью. Если одноранговой сервер-посредник является шлюзом PSTN или пограничным контроллером сеанса (SBC), настоятельно рекомендуется, чтобы конфигурация магистрали не была связана с записью использования PSTN, которая передается в пункт назначения PSTN или в другие нижестояльные системы, подключенные через Skype для бизнеса Server. 

10. Измените расположение записей режима работы с ТСОП, чтобы обеспечить оптимальную производительность. Чтобы изменить расположение записи в списке, выберите соответствующую запись режима работы с ТСОП и щелкните стрелку вверх или стрелку вниз.
    > [!Important] 
    > Порядок расположения записей режима работы с ТСОП в списке является важным. Skype для бизнеса Server пересекает список сверху вниз. 

11. Установите флажок **Разрешить блокирование RTP**, чтобы разрешить обход сервера-посредника для клиентов, находящихся за устройством NAT (преобразование сетевых адресов) или брандмауэром, и пограничного контроллера сеансов, поддерживающего блокирование.
12. **Необходимо выбрать историю вызовов,** чтобы включить отправку сведений об истории вызовов в одноранговой шлюз сервера-посредника.
13. Необходимо выбрать предварительные данные **P-Asserted-Identity,** чтобы в настоящее время информация об источнике вызовов PAI была переадреждена между стороной сервера-посредника и стороной шлюза (и наоборот).
14. Установите флажок **Включить таймер отработки отказа внешней маршрутизации**, чтобы включить быструю отработку отказа. Шлюз, сопоставленный с этой магистралью, может в течение 10 секунд выдать уведомление о том, что он обрабатывает исходящий вызов. Если это уведомление не получено сервером-посредником, будет происходить перенанаружение в другой магистраль. В сетях, где задержка может увеличить время ответа или где на ответ шлюзу требуется более 10 секунд, быструю отработку отказа следует отключить.
15. (Необязательный) Связывать **и настраивать правила перевода номеров вызовов** для магистрали. Эти правила перевода применяются к номеру вызова для исходящие вызовы.
    - Чтобы выбрать одно или несколько правил из списка всех правил перевода, доступных в Корпоративная голосовая связь развертывания, нажмите **кнопку Выберите**. В окне **Выбор правил трансляции** выберите правила, которые требуется сопоставить с магистральной линией связи, а затем нажмите кнопку **ОК**.
    - Чтобы определить новое правило преобразования и сопоставить его с магистралью, нажмите кнопку **Создать**. Подробные сведения о определении нового правила см. в материале Определение правил перевода [в Skype для бизнеса Server.](defining-translation-rules.md)
    - Чтобы изменить правило преобразования, которое уже сопоставлено с магистралью, щелкните имя правила, затем нажмите кнопку **Подробнее**. Подробные сведения см. [в материале Определение](defining-translation-rules.md)правил перевода в Skype для бизнеса Server .
    - Чтобы скопировать существующее правило преобразования и использовать его в качестве отправной точки для определения нового правила, щелкните имя правила и нажмите кнопку **Копировать**, а затем кнопку **Вставить**. Подробные сведения см. [в материале Определение](defining-translation-rules.md)правил перевода в Skype для бизнеса Server .
    - Чтобы удалить правило преобразования из магистральной линии связи, выделите имя этого правила, а затем нажмите кнопку **Удалить**.

    > [!Warning]
    > Не сопоставляйте правила преобразования с магистралью, если вы настроили правила преобразования на сопоставленном одноранговом узле, поскольку два правила могут конфликтовать. 

16. (Необязательно) Сопоставьте и настройте **правила трансляции набранного номера** для магистральной линии связи. Эти правила применяются к вызываемым номерам для исходящих вызовов.
    - Чтобы выбрать одно или несколько правил из списка всех правил перевода, доступных в Корпоративная голосовая связь развертывания, нажмите **кнопку Выберите**. В правилах выбора перевода щелкните правила, которые необходимо связать с магистралью, а затем нажмите **кнопку ОК.**
    - Чтобы определить новое правило преобразования и сопоставить его с магистралью, нажмите кнопку **Создать**. Подробные сведения о определении нового правила см. в материале Определение правил перевода [в Skype для бизнеса Server.](defining-translation-rules.md)
    - Чтобы изменить правило преобразования, которое уже сопоставлено с магистралью, щелкните имя правила, затем нажмите кнопку **Подробнее**. Подробные сведения см. [в материале Определение](defining-translation-rules.md)правил перевода в Skype для бизнеса Server .
    - Чтобы скопировать существующее правило преобразования и использовать его в качестве отправной точки для определения нового правила, щелкните имя правила и нажмите кнопку **Копировать**, а затем кнопку **Вставить**. Подробные сведения см. [в материале Определение](defining-translation-rules.md)правил перевода в Skype для бизнеса Server .
    - Чтобы удалить правило преобразования из магистральной линии связи, выделите имя этого правила, а затем нажмите кнопку **Удалить**.

    > [!Caution] 
    > Не сопоставляйте правила преобразования с магистральной линией связи, если вы уже настроили их для связанной одноранговой магистральной линии связи, поскольку эти два правила могут конфликтовать друг с другом. 

17. Убедитесь, что правила перевода магистрали расположены в правильном порядке. Чтобы изменить положение правила в списке, выделите имя правила и нажмите кнопку со стрелкой вверх или вниз.

    > [!Important] 
    > Skype для бизнеса Server перебирает список правил перевода сверху вниз и использует первое правило, которое соответствует набраному номеру. Если вы настраиваете магистраль таким образом, чтобы набираемый номер мог соответствовать нескольким правилам преобразования, убедитесь, что более строгие правила расположены над менее строгими. Например, если вы включаете правило преобразования, соответствующее любому 11-значному номеру, и правило преобразования, которое соответствует только 11-значным номерам, начинающимся с +1425, убедитесь, что правило, соответствующее любому 11-значному номеру, располагается ниже более строгого правила. 

18. Закончив настройку магистрали, нажмите кнопку **ОК**.
19. На странице **Конфигурация магистрали** нажмите кнопку **Передать**, а затем щелкните **Передать все**. 

    > [!Note]
    > При каждом создании или изменении конфигурации магистрали необходимо выполнить команду **Сохранить все**, чтобы опубликовать изменение конфигурации. Подробные сведения см. в материале Публикация ожидающих изменений конфигурации маршрутирования голосовой почты в Lync Server 2013 в документации по операциям.
