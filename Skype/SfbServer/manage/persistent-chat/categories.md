# <a name="manage-categories-in-persistent-chat-server-in-skype-for-business-server-2015"></a>Управление категориями на сервере сохраняемой беседы в Skype для бизнеса Server 2015
 
**Сводка:** Узнайте, как управлять категориями сервера сохраняемой беседы в Skype для бизнеса Server 2015.
  
Категория — это логическая структура для организации комнат чата. Категория определяет набор списков управления доступом (ALS) по умолчанию для управления пользователями и группами пользователей, которые могут создавать комнаты чата или присоединяться к ним. Категории комнат чата содержат комнаты чата, но не другие категории. Каждая категория описывает свое содержимое с помощью метаданных, таких как Имя и Описание. Категория имеет свойства, которые можно настроить для управления поведением комнат чата, принадлежащих к ней; например, разрешает ли комнаты чата приглашения или отправки файлов, или содержит историю чата. 
  
Создание комнат чата и управление им намного проще при правильном использовании категорий. Администратор сервера сохраняемого чата может определить allowedMembers и Creators для каждой категории, а также определить параметры и режимы комнаты чата по умолчанию, которые будут применяться во всех комнатах чата, созданных в этой категории. Например, если для allowedMembers категории установлено contoso.com, можно добавить любую группу или пользователя в Contoso в качестве участника в комнаты чата в этой категории. Если для категории "Разрешенные участники" установлено "Продажи", в качестве участников в комнаты чата в этой категории можно добавлять только группы и пользователей из этого списка рассылки. Свойство Creators позволяет контролировать, кто может создавать комнаты чата в этой категории. После создания комнаты чата любой член группы AllowedMembers может быть назначен менеджером для текущих операций управления комнатами (например, для изменения членства и утверждения).
  
Указание параметров AllowedMembers и Creators для категории имеет следующие преимущества.
  
- Все комнаты чата в этой категории будут подчиняться ограничениям, установленным на уровне категории. Это можно использовать для разделения комнат чата на основе бизнес-требований и политик доступа.
    
- Пользователь, входящий в список создателей (Creators), может создавать новые комнаты чата в этой категории. Если вы хотите реализовать систему, в которой ограниченное количество сотрудников в организации может создавать комнаты чата, этот контроль можно использовать для удовлетворения этих требований. 
    
Пользователи, подразделения и группы пользователей, которые определены как создатели категории, являются единственными пользователями и группами, которые могут создавать комнаты в этой категории. После создания категории можно выбрать пользователей, организации и группы пользователей из списка AllowedMembers категории в качестве руководителей и участников комнат чата для управления комнатой чата и участия в ней. 
  
Перед настройкой категорий обязательно прочитайте категории сохраняемой беседы, комнаты чата и роли пользователей [в Skype для бизнеса Server 2015.](../../plan-your-deployment/persistent-chat-server/categories-chat-rooms-and-user-roles.md)
  
Вы можете настраивать категории и управлять ими с помощью панели управления или с помощью Windows PowerShell управления.

> [!NOTE]
> Сохраняемая беседа доступна в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения [см. в сведениях о том,](/microsoftteams/upgrade-start-here)как начать обновление Microsoft Teams. Если необходимо использовать сохраняемого чата, вы можете либо перенести пользователей, которым требуются эти функции, в Teams, либо продолжить использование Skype для бизнеса Server 2015. 
  
## <a name="configure-categories-by-using-the-control-panel"></a>Настройка категорий с помощью панели управления

1. Войдите на любой компьютер во внутреннем развертывании с использованием учетной записи пользователя, назначенной роли CsPersistentChatAdministrator или CsAdministrator.
    
2. В меню **"Пуск"** выберите панель управления Skype для бизнеса Server или откройте окно браузера и введите URL-адрес администратора.
    
3. В левой панели навигации щелкните **Сохраняемый чат** и выберите пункт **Категория**.
    
    Для нескольких развертываний пула серверов сохраняемого чата выберите соответствующий пул в выпадаемом списке.
    
4. На странице **Категория** нажмите кнопку **Создать** или **Изменить**.
    
5. В **области "Выбор службы"** выберите службу, соответствующую пулу серверов сохраняемой беседы, в котором необходимо создать категорию. Служба — это пул серверов сохраняемого чата, который клиент сохраняемого чата использует для определения пула, к которому относится категория. Категория может принадлежать только одному пулу серверов сохраняемого чата и не может быть перемещена в другой пул или в общий доступ к ней.
    
6. В окне **Создание категории** выполните следующие действия.
    
   - В поле **Имя** введите имя новой категории комнаты.
    
   - В поле **Описание** введите подробное описание категории комнаты (например, категория комнаты для Contoso).
    
   - Чтобы определить, можно ли включить приглашения для комнат чата, принадлежащих  данной категории, выберите или с помощью этого окне сключим. Если этот выбор выбран, комнаты в этой категории могут иметь приглашения включаемой или выключенной; Если он очищен, комнаты в этой категории не могут иметь приглашения. Если в комнате есть приглашения, при добавлении нового участника в комнату он получает уведомление о новой комнате в клиенте сохраняемого чата.
    
   - Чтобы контролировать загрузку файлов в комнатах чата этой категории, установите или снимите флажок **Разрешить загрузку файлов**. Если этот флажок установлен, комнаты этой категории могут включать или отключать загрузку файлов. Если он снят, загрузка файлов для этой категории отключена.
    
   - Чтобы управлять историями чатов, выберите или с помощью этого окне уберем.  Если этот выбор выбран, комнатные чаты становятся постоянными; в противном случае сообщения чата не сохраняются. Если включено соответствие требованиям, чаты комнат будут сохранены в соответствии с соответствием, но пользователи не смогут получить доступ к старым сообщениям. Этот параметр можно использовать для комнат, предназначенных для специальной совместной работы в режиме реального времени, для сохраняемой истории чата.
    
7. В области **Изменение категории** выполните следующие действия:
    
   - В разделе "Членство" **в** разделе "Разрешенные участники" добавьте или удалите пользователей и других участников доменных служб Active Directory (пользователей, групп рассылки, подразделений и так далее), которые могут быть добавлены в качестве членов комнат чата, принадлежащих этой категории.  Разрешенные в категории субъекты могут выполнять поиск комнат в этой категории (если только комната не является скрытой — такую комнату могут искать в каталоге только ее участники).
    
   - В **разделе**"Отказано **в** членстве" добавьте или удалите пользователей и других участников Active Directory, связанных с участниками, отказано в доступе к комнате.
    
   - В **разделе**"Членство" в разделе **"Авторы"** добавьте или удалите пользователей и других участников Active Directory, связанных с создателями категории. Автор — это пользователь, обладающий разрешениями на создание комнат чата и назначение их диспетчеров и участников.
    
8. Щелкните **Исполнить**.
    
## <a name="configure-categories-by-using-windows-powershell"></a>Настройка категорий с помощью Windows PowerShell

Категории можно настроить с помощью следующих Windows PowerShell:
  

|**Командлет**|**Описание**|
|:-----|:-----|
|New-CsPersistentChatCategory  <br/> |Создание новой категории  <br/> |
|Set-CsPersistentChatCategory  <br/> |Настройка параметров для существующей категории  <br/> |
|Get-CsPersistentChatCategory  <br/> |Извлечение сведений о категориях  <br/> |
|Remove-CsPersistentChatCategory  <br/> |Удаление категории  <br/> |
   
Для категорий можно настроить следующие параметры:
  
- EnableFileUpload. Разрешает отправку файлов в комнаты чата в категории.
    
- EnableInvitations. Включает приглашения для категории. Пользователи в списке AllowedMembers автоматически получат приглашение присоединиться к новой комнате чата во время создания новой комнаты.
    
- ChatHistory. Включает или отключает функцию истории чата.
    
- Creators. Указывает пользователей, которым разрешено создавать комнаты чата в этой категории.
    
- AllowedMembers. Указывает пользователей, которым разрешен доступ к комнатам чата в этой категории.
    
- DeniedMembers. Перечисляет пользователей, которым не разрешен доступ к комнатам разговоров в этой категории.
    
Полные сведения о синтаксисах, включая все параметры, см. в руководстве По skype [для бизнеса Server 2015.](../management-shell.md)
  
### <a name="create-a-new-category"></a>Создание новой категории

Новую категорию можно создать с помощью **cmdlet New-CsPersistentChatCategory.** Например, следующая команда создает новую категорию с именем HelpDesk в пуле atl-cs-001.contoso.com. В этом примере отправка файлов включена:
  
```PowerShell
New-CsPersistentChatCategory -Name "HelpDesk" -PersistentChatPoolFqdn "atl-cs-001.contoso.com" -EnableFileUpload 
```

### <a name="configure-an-existing-category"></a>Настройка существующей категории

Существующую категорию можно настроить с помощью **cmdlet Set-CsPersistentCategory.**
  
Например, следующая команда указывает, что user1 является AllowedMember и Creator, в то время как пользователю user2 запрещен доступ к комнатам в категории:
  
```PowerShell
Set-CsPersistentChatCategory -Identity testCat -AllowedMembers @{Add="sip:user1@contoso.com", "CN=container,DC=contoso,DC=com"}  -DeniedMembers @{Add="sip:user2@contoso.com"}
Set-CsPersistentChatCategory -Identity testCat -Creators @{Add="sip:user1@contoso.com"}
```

### <a name="get-information-about-categories"></a>Получить сведения о категориях

Сведения о категориях можно получить с помощью **cmdlet Get-CsPersistentChatCategory.** Например, следующая команда возвращает сведения для всех категорий Persistent Chat в организации:
  
```PowerShell
Get-CsPersistentChatCategory
```

### <a name="remove-a-category"></a>Удаление категории

Удалить категорию можно с помощью **cmdlet Remove-CsPersistentChatCategory.** Перед удалением категории необходимо либо удалить все комнаты чата под ней, либо переместить их в новую категорию. Например, следующая команда удаляет категорию с идентификатором "atl-cs-001.contoso.com\helpdesk":
  
```PowerShell
Remove-CsPersistentChatCategory -Identity "atl-cs-001.contoso.com\helpdesk"
```
