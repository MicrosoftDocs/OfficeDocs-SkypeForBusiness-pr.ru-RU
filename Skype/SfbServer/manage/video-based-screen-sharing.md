# <a name="video-based-screen-sharing-for-skype-for-business-server"></a>Общий доступ к экрану на основе видео для Skype для бизнеса Server 
 
Видеодемонстрация (VbSS) в Skype для бизнеса Server 2015 теперь доступна для загрузки: [накопительный пакет обновления KB3061064 для Skype для бизнеса server 2015](https://www.microsoft.com/download/details.aspx?id=47690). VbSS входит в состав Skype для бизнеса Server 2019.
  
Видеообмен на основе видео или VbSS выросло из общего доступа к экрану Lync. Разница между VbSS и традиционным общим доступом к экрану состоит в том, чтобы использовать нижележащие протоколы и то, что в них находится Excel. Совместное использование экрана использует протокол удаленного рабочего стола (RDP), который отлично подходит для создания тысяч сеансов с 1 по 1 между компьютерами пользователей. Более новая технология, VbSS, будет использовать протокол UDP.
  
Skype для бизнеса Server хотел бы увеличить число пользователей от 1 до 1 и их беседы с одной стороны на несколько (с несколькими участниками) и с опытом собраний. VbSS использует платформу мультимедиа (которая основывается на протоколе UDP в качестве основного протокола), что позволяет улучшить время запуска видео, а также качество просмотра (особенно в том случае, если вы наблюдаете быстрее), а также в целом надежность.
  
Часть цели по повышению общего доступа к экрану состоит в том, что переход между VbSS и RDP должен быть как можно более простым при их возникновении. Так как VbSS — это обновление базовой технологии, которое используется для демонстрации экрана в Skype для бизнеса Server, может быть трудно определить, какую технологию вы используете, если вы не видите сведения об SIP-адресах в сетевом трафике, или вы используете общий доступ к содержимому, — Это быстрое перемещение или 3-D. Если, например, на рабочем месте работает большое количество устаревших клиентов, протокол RDP по-прежнему будет доступен в качестве отказоустойчивого для собраний и бесед. Skype для бизнеса Server использует внутреннюю логику, чтобы определить, какой из двух методов (VbSS или традиционный общий доступ к экрану) применять при подключении клиентов. RDP может и будет подставляться для VbSS, когда она вызывается из-за этой ситуации, поэтому ваше взаимодействие с просмотром не будет прервано.
  
## <a name="planning"></a>Планирование

### <a name="vbss-pros-and-cons"></a>VbSSные и отрицательные достоинства

Переключение на VbSS предназначено для внесения трех ключевых улучшений:
  
1. Предоставление общего доступа к экрану (до 5%) более надежная по сравнению с RDP.

2. Ускорение настройки сеанса и работы с видео по сравнению с RDP (установка в течение половины времени с улучшением 6:1 в кадрах в секунду).

3. Работает значительно лучше, чем RDP, в условиях низкой пропускной способности, даже при совместном использовании высокопроизводительного содержимого, такого как трехмерная графика.
    
Имейте в виду, что эти номера основываются на работоспособности и правильной настройке производительности сети, а также могут включать сети, внешние по отношению к собственным, если клиенты находятся на мобильных устройствах.
  
Кроме того, следует помнить о том, что для обеспечения надежности, скорости и эффективности совместного использования контента была обеспечена определенная точность и четкость. В большинстве случаев это не будет заметно для пользователей.

### <a name="capacity-planning"></a>Планирование ресурсов

На каждом сервере переднего плана, на котором выполняется накопительный пакет обновления 2 (CU2) для Skype для бизнеса Server 2015 или более поздней версии, поддерживается до 375 участников для общего доступа к экрану с использованием протокола удаленного рабочего стола (хотя только 250 Эта емкость не меняет POST-CU3, когда VbSS появился и используется.
  
С другой стороны, мы проверили производительность и нагрузочное тестирование в лаборатории, а также должны учитываться следующие измерения относительно собственного развертывания (в зависимости от использования).
  
Стандартное
  
- Вы используете Skype для бизнеса Server 2015 CU2 или более поздней версии в развертывании.
    
- Все пользователи в среде Skype для бизнеса Server обладают разрешениями экрана, превышающими 1920x1080.
    
На полной мощности (как указано выше, составляет 375 участников совместного использования экрана для каждого сервера переднего плана, хотя только 250 на собрание), сервер переднего плана может использовать ~ 89% от сетевой платы Gigabit (1). Это связано с тем, что существующая технология совместного использования экрана в Skype для бизнеса Server CU2 (RDP) передает содержимое на экране в собственном разрешении компьютера докладчика. Таким образом, при увеличении разрешений на экране могут возникнуть узкие места в сети для общего доступа к экрану с помощью Skype для бизнеса Server 2015 CU2.
  
Чтобы устранить эту угрозу, могут быть полезны следующие параметры:
  
- Обновите сервер переднего плана с 1 гигабитового сетевого адаптера до 10 гигабитной платы Ethernet.

## <a name="clients-and-servers-support"></a>Поддержка клиентов и серверов

Для демонстрации экрана с видео требуется Skype для бизнеса Server 2015 CU3 или более поздней версии, а также текущая версия поддерживающих клиентов, перечисленных в разделе [Сравнение функций мобильных клиентов для поддержки Skype для бизнеса](../plan-your-deployment/clients-and-devices/mobile-feature-comparison.md) и [собраний](../plan-your-deployment/clients-and-devices/desktop-feature-comparison.md#BKMK_Conferencing). 
  
Существуют ситуации, в которых общий доступ к экрану будет переходить на RDP, как показано ниже.
  
- Если ваша учетная запись размещена в среде, в которой ASMCU не соответствует минимальной сборке, поддерживающей VbSS.
- Если пользователь, использующий старую версию клиента Skype для бизнеса, присоединяется к вашему сеансу, например любой пользователь, использующий любую версию клиента Windows, которая ниже 16.0.6330.1000, Skype для бизнеса для мобильных устройств или мобильных приложений Skype для бизнеса. 
- Если пользователь совместно используется в веб-приложении Skype для бизнеса.
- Если кто-то использует Skype для бизнеса на Mac и не размещен в Skype для бизнеса Online или Skype для бизнеса Server 2015 с Июльом, 2018 (или более поздней версии).
- Если кто-то запускает программу или общий доступ к Windows.
- Если кто-то начинает записывать сеанс.
- Если кто-то вызывает удаленный экранный элемент управления во время сеанса. 
- Собраний с более чем 250 участниками (где VbSS в настоящее время не поддерживается).

Имейте в виду, что когда сеанс переходит в RDP, он не возвращается в VbSS. Опять же, переход от VbSS, как правило, не будет легко обнаруживать в большинстве ситуаций.
    
> [!NOTE]
> Он не может блокировать или пытаться блокировать переход с VbSS на RDP в режиме совместного использования экрана Skype для бизнеса. 
  
## <a name="enabling-disabling-and-configuring-vbss"></a>Включение, отключение и настройка VbSS

Отличная проблема состоит в том, что после установки накопительного пакета обновления 3 (CU3) для Skype для бизнеса Server 2015 или более поздней версии все ваши пользователи будут включены по умолчанию для всех пользователей и VbSS с несколькими участниками. Это может вызвать проблемы, если вы не включили эту функцию для всех пользователей. В этом случае вы можете использовать эти действия, чтобы отключить пользователей (дальнейшие действия по включению могут выполняться для пользователей):
  
### <a name="how-to-disable-users-from-using-vbss"></a>Отключение пользователей от использования VbSS

- Вы можете назначить пользовательскую политику, которая не разрешает VbSS для пользователей, которые не должны использовать VbSS, выполнив этот командлет в консоли управления Skype для бизнеса (замените [PolicyName] на политику, для которой вы делаете это):
    
  ```PowerShell
  Set-CsConferencingPolicy -Identity [PolicyName] -ApplicationSharingMode RDP
  ```

- Вы также можете обновить глобальную политику конференц-связи, которая повлияет на всех пользователей без назначенной политики:
    
  ```PowerShell
  Set-CsConferencingPolicy -ApplicationSharingMode RDP
  ```

    Дополнительные сведения об этой команде можно найти в статье [Set – CsConferencingPolicy](https://docs.microsoft.com/powershell/module/skype/set-csconferencingpolicy?view=skype-ps).
    
- Если вам нужно полностью отключить VbSS, можно выполнить следующую команду:
    
  ```PowerShell
  Set-CsMediaConfiguration -EnableVideoBasedSharing $false
  ```

    Дополнительные сведения об этой команде можно найти в статье [Set – CsMediaConfiguration](https://docs.microsoft.com/powershell/module/skype/set-csmediaconfiguration?view=skype-ps).
    
> [!NOTE]
> В рамках многосторонних собраний Skype для бизнеса все конечные точки клиентов будут иметь параметр политики для организатора собрания. 
  
### <a name="how-to-enable-users-to-use-vbss"></a>Как разрешить пользователям использовать VbSS

- Можно назначить определенную политику пользователей, которая разрешает VbSS всем пользователям, которым необходимо использовать VbSS, выполнив этот командлет в консоли управления Skype для бизнеса (замените [PolicyName] на политику, для которой вы делаете это):
    
  ```PowerShell
  Set-CsConferencingPolicy -Identity [PolicyName] -ApplicationSharingMode VideoWithFallback
  ```

- Вы также можете обновить глобальную политику конференц-связи, которая повлияет на всех пользователей без назначенной политики:
    
  ```PowerShell
  Set-CsConferencingPolicy -ApplicationSharingMode VideoWithFallback
  ```

    Дополнительные сведения об этой команде можно найти в статье [Set – CsConferencingPolicy](https://docs.microsoft.com/powershell/module/skype/set-csconferencingpolicy?view=skype-ps).
    
- Если вам нужно снова включить VbSS после отключения (по умолчанию он включен), можно выполнить следующую команду:
    
  ```PowerShell
  Set-CsMediaConfiguration -EnableVideoBasedSharing $true
  ```

    Дополнительные сведения об этой команде можно найти в статье [Set – CsMediaConfiguration](https://docs.microsoft.com/powershell/module/skype/set-csmediaconfiguration?view=skype-ps).
    
> [!NOTE]
> В собрании на нескольких Сторонах Skype для бизнеса все конечные точки клиента будут иметь параметр политики для организатора собрания.
