# <a name="video-based-screen-sharing-for-skype-for-business-server"></a>Видеодемонстрация экрана в Skype для бизнеса Server 
 
Видео-файл обмена экранами (VbSS) в Skype Для бизнеса Server 2015 теперь доступен для скачивания: [Skype для бизнеса Server 2015 Накопительное обновление KB3061064](https://www.microsoft.com/download/details.aspx?id=47690). VbSS включен в Skype для бизнеса Server 2019 г.
  
Video-based Screen Sharing, or VbSS, grew out of Lync screen-sharing. Разница между VbSS и традиционным обменом экранами имеет значение для используемых протоколов и их отличий. При совместном использовании экрана используется протокол удаленного рабочего стола (RDP), который отлично подходит для создания тысяч сеансов от 1 до 1 между компьютерами людей. Более новая технология , VbSS, будет использовать протокол пользовательских datagram (UDP).
  
Skype для бизнеса Server людей с 1 по 1, а также их многостолевую (многопартийную) беседу и общение. VbSS использует медиаплатформу (которая опирается на UDP в качестве основном протоколе), с целью улучшения времени запуска видео, качества просмотра того, что вы смотрите (особенно если то, что вы смотрите, быстро перемещается) и надежности в целом.
  
Часть цели улучшения обмена экранами заключается в том, чтобы переходы между VbSS и RDP были максимально бесшовными, когда они происходят. Так как VbSS — это обновление для технологии, используемой в совместном доступе к экранам для Skype для бизнеса Server, может быть сложно определить, какую технологию вы используете, если только вы не ищете сведения о SIP в сетевом трафике или вы обменивается содержимым, быстро перемещаемым или 3-D. Если, например, на рабочем месте много устаревших клиентов, RDP по-прежнему будет доступен в качестве сбоя для собраний и бесед. Skype для бизнеса Server использует внутреннюю логику, чтобы решить, какие из двух методов (VbSS или традиционный обмен экранами) применяются при подключении клиентов. RDP можно и будет заменять VbSS при вызове ситуации, чтобы не прерывать просмотр.
  
## <a name="planning"></a>Планирование

### <a name="vbss-pros-and-cons"></a>Плюсы и минусы VbSS

Переход на VbSS направлен на три основных улучшения:
  
1. Сделайте совместное использование экрана (до 5%) более надежным по сравнению с одним только RDP.

2. Сделайте настройку сеанса и видеосъемки быстрее по сравнению с одним только RDP (установка в половине времени, с улучшением 6:1 в кадрах в секунду).

3. Работает намного лучше, чем RDP в условиях низкой пропускной способности, даже при совместном использовании контента с высоким движением, например 3-D-графики.
    
Обратите внимание, что эти цифры зависят от работоспособности и правильной настройки производительности сети и могут включать внешние сети, если клиенты находятся на мобильных устройствах.
  
Вы также должны знать, что некоторые верность и четкость общего контента были обменялись надежностью, скоростью и эффективностью. В большинстве случаев это не будет легко видно пользователям.

### <a name="capacity-planning"></a>Планирование мощности

Каждый сервер переднего Skype для бизнеса Server 2015 накопительного обновления 2 (CU2) или более поздней версии поддерживает до 375 участников для совместного использования экрана с помощью RDP (хотя только 250 на собрание). Эта емкость не меняется после cu3, когда VbSS внедряется и используется.
  
При этом мы сделали тестирование производительности и стресса в нашей лаборатории, и следующие измерения также следует учитывать в отношении собственного развертывания (в зависимости от использования, конечно).
  
Предполагая:
  
- При развертывании используется Skype для бизнеса Server 2015 cu2 или более поздний.
    
- Все пользователи в Skype для бизнеса Server имеют разрешение экрана выше 1920x1080.
    
При полной емкости (которая, как уже отмечалось выше, составляет 375 участников совместного доступа к экранам на сервере переднего конечного сервера в общей сложности, хотя только 250 на собрание), на переднем конечном сервере может использоваться ~89% от 1 гигабит сетевой карты. Это происходит из-за того, что существующая технология обмена экранами в Skype для бизнеса Server CU2 (RDP) передает содержимое на экране в разрешении на родном компьютере. Таким образом, если учитываться более высокое разрешение экрана, вы уже можете испытывать узкие места в сети для общего доступа к экранам с Skype для бизнеса Server 2015 CU2.
  
Для смягчения этого может оказаться полезным один или несколько следующих вариантов:
  
- Обновление переднего сервера с 1 гигабитной сетевой карты до 10-гигабитной ethernet-карты.

- Увеличение количества серверов переднего конечного сервера для трафика балансизагрузки нагрузки.

- Ограничить пропускную способность (битрат), используемую для VbSS и RDP, поставив ограничение максимальной пропускной способности, используемой для обоих каналов.

## <a name="clients-and-servers-support"></a>Поддержка клиентов и серверов

Для обмена экранами на основе видео требуется Skype для бизнеса Server 2015 cu3 или более [](../plan-your-deployment/clients-and-devices/mobile-feature-comparison.md) поздней версии, а также текущая версия поддерживающих клиентов, перечисленных в сопоставлении функций мобильных клиентов для Skype для бизнеса и поддержки собраний. [](../plan-your-deployment/clients-and-devices/desktop-feature-comparison.md#BKMK_Conferencing) 
  
Существуют ситуации, в которых обмен экранами переходит на RDP, например:
  
- Если учетная запись находится в среде, в которой ASMCU не соответствует минимальной сборке, поддерживающей VbSS.
- Если к сеансу присоединяется кто-либо, использующий более старую версию клиента Skype для бизнеса, например любой, кто использует любую клиентскую версию Windows, которая ниже 16.0.6330.1000, Skype для бизнеса устройства системы номеров или Skype для бизнеса Мобильные приложения. 
- Если пользователь делится с Skype для бизнеса Web App.
- Если кто-то Skype для бизнеса на Mac и не находится в Skype для бизнеса Online или Skype для бизнеса Server 2015 г. с накопительным обновлением за июль 2018 г. (или позже).
- Если кто-то запускает программу/Windows общий доступ.
- Если кто-то начинает запись сеанса.
- Если во время сеанса кто-то вызывает управление удаленным экраном. 
- Собрания с более чем 250 участниками (в настоящее время VbSS не поддерживается).

Следует помнить, что после перехода сеанса на RDP он не будет возвращаться к VbSS. Опять же, переход от VbSS должен быть плавным и, с надеждой, будет нелегко обнаружить в большинстве ситуаций.
    
> [!NOTE]
> Не поддерживается блокировка или попытка блокировки перехода с VbSS на RDP в Skype для бизнеса совместном использовании экрана. 
  
## <a name="enabling-disabling-and-configuring-vbss"></a>Включение, отключение и настройка VbSS

Самое замечательное, что после установки накопительного обновления 3 Skype для бизнеса Server 2015 года (CU3) или более поздней версии все пользователи будут включены для 1-1 и многопартийных VbSS по умолчанию. Это может быть проблематичным для вас, если у вас есть причина, чтобы не включить эту функциональность для всех пользователей. В этом случае вы можете использовать эти действия для отключения пользователей (после этого будут выполняться действия, которые будут выполняться пользователями):
  
### <a name="how-to-disable-users-from-using-vbss"></a>Отключение пользователей от использования VbSS

- Вы можете назначить политику пользователя, которая не позволяет VbSS пользователям, которые не должны использовать VbSS, запуская этот комдлет в консоли управления Skype для бизнеса (замените [PolicyName] политикой, для которых вы это делаете):
    
  ```PowerShell
  Set-CsConferencingPolicy -Identity [PolicyName] -ApplicationSharingMode RDP
  ```

- Вы также можете обновить глобальную политику конференциалов, которая затронет всех пользователей без назначенной политики:
    
  ```PowerShell
  Set-CsConferencingPolicy -ApplicationSharingMode RDP
  ```

    Дополнительные сведения об этой команде см. [в сайте Set-CsConferencingPolicy.](/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
    
- Если требуется полностью отключить VbSS, можно выполнить эту команду:
    
  ```PowerShell
  Set-CsMediaConfiguration -EnableVideoBasedSharing $false
  ```

    Дополнительные сведения об этой команде см. [в ссылке Set-CsMediaConfiguration.](/powershell/module/skype/set-csmediaconfiguration?view=skype-ps)
    
> [!NOTE]
> В многопартийном Skype для бизнеса все конечные точки клиента будут соблюдать параметр политики для организатора собрания. 
  
### <a name="how-to-enable-users-to-use-vbss"></a>Как позволить пользователям использовать VbSS

- Вы можете назначить определенную политику пользователя, которая позволяет VbSS любым пользователям, которым необходимо использовать VbSS, запуская этот кодлет в консоли управления Skype для бизнеса (замените [PolicyName] политикой, для которых вы это делаете:
    
  ```PowerShell
  Set-CsConferencingPolicy -Identity [PolicyName] -ApplicationSharingMode VideoWithFallback
  ```

- Вы также можете обновить глобальную политику конференциалов, которая затронет всех пользователей без назначенной политики:
    
  ```PowerShell
  Set-CsConferencingPolicy -ApplicationSharingMode VideoWithFallback
  ```

    Дополнительные сведения об этой команде см. [в сайте Set-CsConferencingPolicy.](/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
    
- Если необходимо включить VbSS после выключения (он по умолчанию), можно выполнить эту команду:
    
  ```PowerShell
  Set-CsMediaConfiguration -EnableVideoBasedSharing $true
  ```

    Дополнительные сведения об этой команде см. [в ссылке Set-CsMediaConfiguration.](/powershell/module/skype/set-csmediaconfiguration?view=skype-ps)
    
> [!NOTE]
> На многопартийном собрании Skype для бизнеса клиентские конечные точки будут соблюдать параметр политики организатора собрания.
