# <a name="video-based-screen-sharing-for-skype-for-business-server"></a>Видеодемонстрация экрана в Skype для бизнеса Server 
 
Теперь можно скачать видеосвязь с экрана (VbSS) в Skype для бизнеса Server 2015: накопительный обновления [KB3061064](https://www.microsoft.com/download/details.aspx?id=47690)для Skype для бизнеса Server 2015. VbSS входит в skype для бизнеса Server 2019.
  
Общий доступ к экрану на основе видео или VbSS выходит из общего доступа к экранам Lync. Разница между VbSS и традиционным совместным использованием экрана заключается в используемых протоколах и их excel. При совместном использовании экрана используется протокол удаленного рабочего стола (RDP), который отлично подходит для создания тысяч сеансов по 1-1 между компьютерами пользователей. Более новая технология, VbSS, будет использовать протокол UDP.
  
Skype для бизнеса Server хочет улучшить "один к 1" и "один-многие" (многостолбные) беседы и собрания. VbSS использует медиаплатформу (которая использует UDP в качестве используемого протокола), чтобы улучшить время начала видео, качество просмотра просматриваемой вами информации (особенно если то, что вы смотрите, быстро перемещается) и общую надежность.
  
Одной из целей улучшения совместного использования экрана является то, что переходы между VbSS и RDP будут максимально плавными при их использовании. Так как VbSS — это обновление для используемой технологии, используемой при совместном использовании экрана для Skype для бизнеса Server, может быть трудно определить используемую технологию, если вы не ищете сведения о SIP в сетевом трафике или не делите быстро перемещающийся или объемный контент. Например, если на рабочем месте много устаревших клиентов, RDP по-прежнему будет доступен в качестве сбоя для собраний и бесед. Skype для бизнеса Server использует внутреннюю логику, чтобы решить, какой из двух методов (VbSS или традиционный общий доступ к экрану) применяется при подключении клиентов. RDP может заменяться VbSS при вызове ситуации, чтобы ваш просмотр не прерывался.
  
## <a name="planning"></a>Планирование

### <a name="vbss-pros-and-cons"></a>Преимущества и недостатки VbSS

Переход на VbSS направлен на три основных улучшения:
  
1. Совместное использование экрана (до 5 %) надежнее по сравнению с RDP.

2. Сделайте настройку сеанса и видеосвязь быстрее по сравнению с RDP (настройка в два раза быстрее с 6:1 кадрами в секунду).

3. Работает намного лучше, чем RDP в условиях низкой пропускной способности, даже при совместном использовании высокоскоростного содержимого, например 3D-графики.
    
Помните, что эти номера зависят от работоспособности и правильной настройки производительности сети и могут включать внешние сети, если ваши клиенты находятся на мобильных устройствах.
  
Кроме того, следует помнить, что некоторые надежность и четкость общего содержимого были проименовыы для надежности, скорости и эффективности. В большинстве случаев это будет не сразу видно пользователям.

### <a name="capacity-planning"></a>Планирование мощности

Каждый сервер переднего сервера со Skype для бизнеса Server 2015 с накопительным обновлением 2 (CU2) или более поздней версии поддерживает до 375 участников для совместного использования экрана с помощью RDP (хотя только 250 на собрание). Эта емкость не меняется после cu3, когда вводится и используется VbSS.
  
Тем не менее, в нашей лабораторной работе мы сделали тестирование производительности и нагрузки, и следующие измерения также следует учитывать в отношении вашего собственного развертывания (в зависимости от использования, конечно).
  
Предполагается, что:
  
- В развертывании используется Skype для бизнеса Server 2015 CU2 или более поздней.
    
- Разрешение экрана всех пользователей в среде Skype для бизнеса Server превышает 1920x1080.
    
При полной емкости (как было отмечено выше, 375 участников общего доступа к экранам на сервер переднего сервера, хотя только 250 на собрание) сервер переднего сервера может использовать около 89 % от 1 Гбит/с сетевой карты. Это происходит потому, что существующая технология совместного доступа к экранам в Skype для бизнеса Server CU2 (RDP) передает содержимое на экране в разрешении компьютера, на который передается автор. So with higher screen resolutions factored in, you may already be experiencing network bottlenecks for screen sharing with Skype for Business Server 2015 CU2.
  
Чтобы устранить эту проблему, может оказаться полезным один или несколько из следующих вариантов:
  
- Обновите сервер переднего сервера с 1 гигабитной сетевой карты до 10 гигабитной ethernet-карты.

- Увеличение числа серверов переднего сервера для балансивной нагрузки трафика.

- Ограничите пропускную способность (скорость), используемую для VbSS и RDP, ограничив максимальную полосу пропускания, используемую обоими каналами.
    
На числа в этой таблице влияют отдельные сети и общий доступ к содержимому. Протестировать, чтобы установить базовые показатели для вашей сети или сети.

## <a name="clients-and-servers-support"></a>Поддержка клиентов и серверов

Для совместного использования экрана на основе видео требуется Skype для бизнеса Server 2015 CU3 или более поздней версии, а также текущая версия поддерживаемых клиентов, перечисленных в сравнении функций мобильных клиентов для [поддержки Skype](../plan-your-deployment/clients-and-devices/mobile-feature-comparison.md) для бизнеса и собраний. [](../plan-your-deployment/clients-and-devices/desktop-feature-comparison.md#BKMK_Conferencing) 
  
Существуют ситуации, в которых при совместном использовании экрана будет переход на RDP, например:
  
- Если ваша учетная запись находится в среде, где ASMCU не соответствует минимальной сборке, поддерживающей VbSS.
- Если к сеансу присоединяется кто-то, кто использует более старую версию клиента Skype для бизнеса, например любой, кто использует любую версию клиента Windows, которая меньше 16.0.6330.1000, устройства системы комнат Skype для бизнеса или мобильные приложения Skype для бизнеса. 
- Если пользователь совместно использует Skype для бизнеса Web App.
- Если кто-то использует Skype для бизнеса на Mac и не находится в Skype для бизнеса Online или Skype для бизнеса Server 2015 с накопительным обновлением за июль 2018 г. (или более поздней версии).
- Если кто-то запускает программу или общий доступ к Windows.
- Если кто-то начинает запись сеанса.
- Если кто-то вызывает удаленное управление экраном во время сеанса. 
- Собрания с более чем 250 участниками (где VbSS в настоящее время не поддерживается).

Следует помнить, что после перехода на RDP сеанс не будет возвращаться к VbSS. В большинстве ситуаций переход с VbSS должен быть простым и, надеемся, будет непростой.
    
> [!NOTE]
> Не поддерживается блокировка и попытка блокировки перехода с VbSS на RDP в Skype для бизнеса при совместном использовании экрана. 
  
## <a name="enabling-disabling-and-configuring-vbss"></a>Включение, отключение и настройка VbSS

Самое главное— после установки накопительного обновления 3 (CU3) для Skype для бизнеса Server 2015 или более поздней версии для всех пользователей по умолчанию будет включена 1-1- и многостолбная VbSS. Это может быть проблемой, если есть причина не включить эту функцию для всех пользователей. В этом случае можно использовать следующие действия для отключения пользователей (действия, которые будут выполняться для пользователей):
  
### <a name="how-to-disable-users-from-using-vbss"></a>Отключение использования VbSS для пользователей

- Можно назначить политику пользователей, которая не разрешает VbSS пользователям, которые не должны использовать VbSS, запустив этот cmdlet в консоли управления Skype для бизнеса (замените [PolicyName] политикой, для которую вы делаете это):
    
  ```PowerShell
  Set-CsConferencingPolicy -Identity [PolicyName] -ApplicationSharingMode RDP
  ```

- Вы также можете обновить глобальную политику, которая затронет всех пользователей без назначенной политики:
    
  ```PowerShell
  Set-CsConferencingPolicy -ApplicationSharingMode RDP
  ```

    Дополнительные сведения об этой команде см. в командной команде [Set-CsConferencingPolicy.](https://docs.microsoft.com/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
    
- Если требуется полностью отключить VbSS, можно выполнить команду:
    
  ```PowerShell
  Set-CsMediaConfiguration -EnableVideoBasedSharing $false
  ```

    Дополнительные сведения об этой команде см. в командной ссылке [Set-CsMediaConfiguration.](https://docs.microsoft.com/powershell/module/skype/set-csmediaconfiguration?view=skype-ps)
    
> [!NOTE]
> В многостолбном собрании Skype для бизнеса все клиентские конечные точки будут соблюдать параметр политики для организатора собрания. 
  
### <a name="how-to-enable-users-to-use-vbss"></a>Как позволить пользователям использовать VbSS

- Можно назначить определенную политику пользователей, которая позволяет использовать VbSS любым пользователям, которым требуется использовать VbSS, запустив этот cmdlet в консоли управления Skype для бизнеса (замените [PolicyName] политикой, для которую вы это делаете):
    
  ```PowerShell
  Set-CsConferencingPolicy -Identity [PolicyName] -ApplicationSharingMode VideoWithFallback
  ```

- Вы также можете обновить глобальную политику, которая затронет всех пользователей без назначенной политики:
    
  ```PowerShell
  Set-CsConferencingPolicy -ApplicationSharingMode VideoWithFallback
  ```

    Дополнительные сведения об этой команде см. в командной команде [Set-CsConferencingPolicy.](https://docs.microsoft.com/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
    
- Если вам нужно снова включить VbSS после ее выключения (по умолчанию она отключена), можно выполнить команду:
    
  ```PowerShell
  Set-CsMediaConfiguration -EnableVideoBasedSharing $true
  ```

    Дополнительные сведения об этой команде см. в командной ссылке [Set-CsMediaConfiguration.](https://docs.microsoft.com/powershell/module/skype/set-csmediaconfiguration?view=skype-ps)
    
> [!NOTE]
> Во время многостолбного собрания Skype для бизнеса все клиентские конечные точки будут соблюдать параметр политики для организатора собрания.
