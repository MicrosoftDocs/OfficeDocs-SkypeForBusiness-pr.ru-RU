# <a name="configure-the-meeting-join-page-in-skype-for-business-server"></a>Настройка страницы присоединить к собранию в Skype для бизнеса Server
 
**Сводка:** Узнайте, как настроить страницу присоединить к собранию в Skype для бизнеса Server.
  
Когда пользователь щелкает ссылку на собрание в запросе на собрание, страница присоединиться к собранию определяет, установлен ли клиент Skype для бизнеса на компьютере пользователя. Если клиент уже установлен, он открывается и присоединяется к собранию. Если клиент не установлен, по умолчанию открывается клиент Skype для бизнеса. 
  
## <a name="configure-the-meeting-join-page"></a>Конфигурация страницы присоединения к собранию

Вы можете изменить поведение страницы присоединить к собранию, если хотите разрешить пользователям присоединяться к собраниям с другими версиями клиента. Эти параметры конфигурации были удалены из панели управления Skype для бизнеса Server, но их можно настроить с помощью Set-CsWebServiceConfiguration управления.
  
**Параметры параметров Set-CsWebServiceConfiguration присоединить к собранию**

|**Параметр Set-CsWebServiceConfiguration**|**Описание**|
|:-----|:-----|
|ShowJoinUsingLegacyClientLink  <br/> |Этот параметр больше не используется в локальной версии Skype для бизнеса Server.  <br/> Если задано true, пользователи, присоединяясь к собранию с помощью клиентского приложения, кроме Skype для бизнеса, будут иметь возможность присоединиться к собранию с помощью текущего клиентского приложения. Значение по умолчанию — False.  <br/> |
|ShowAlternateJoinOptionsExpanded  <br/> |Этот параметр больше не используется в локальной версии Skype для бизнеса Server.  <br/>  Если установлено true, альтернативные варианты присоединения к конференции по сети автоматически расширяются и показываются пользователям. Если установлено значение False (значение по умолчанию), эти параметры будут доступны, но пользователю придется отобразить список параметров для себя.  <br/> |
