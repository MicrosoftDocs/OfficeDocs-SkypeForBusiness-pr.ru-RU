# <a name="configure-the-meeting-join-page-in-skype-for-business-server"></a>Настройка страницы присоединиться к собранию в Skype для бизнеса Server
 
**Сводка:** Узнайте, как настроить страницу присоединиться к собранию в Skype для бизнеса Server.
  
Когда пользователь щелкает ссылку на собрание в запросе собрания, страница соединения собрания определяет, установлен ли клиент Skype для бизнеса на компьютере пользователя. Если клиент уже установлен, он открывается и присоединяется к собранию. Если клиент не установлен, по умолчанию открывается Skype для бизнеса клиент. 
  
## <a name="configure-the-meeting-join-page"></a>Конфигурация страницы присоединения к собранию

Вы можете изменить поведение страницы присоединиться к собранию, если вы хотите разрешить пользователям присоединяться к собраниям с другими версиями клиента. Эти параметры конфигурации были удалены из панели управления Skype для бизнеса Server, но их можно настроить с помощью Set-CsWebServiceConfiguration.
  
**Параметры собрания Set-CsWebServiceConfiguration страницы**

|**Параметр Set-CsWebServiceConfiguration**|**Описание**|
|:-----|:-----|
|ShowJoinUsingLegacyClientLink  <br/> |Этот параметр не используется в локальной версии Skype для бизнеса Server.  <br/> Если установлено true, пользователи присоединяются к собранию с помощью клиентского приложения, кроме Skype для бизнеса, будут иметь возможность присоединиться к собранию с помощью текущего клиентского приложения. Значение по умолчанию — False.  <br/> |
|ShowAlternateJoinOptionsExpanded  <br/> |Этот параметр не используется в локальной версии Skype для бизнеса Server.  <br/>  Если установлен параметр True, автоматически расширяются и показано пользователям альтернативные варианты присоединения к онлайн-конференции. Если установлено значение False (значение по умолчанию), эти параметры будут доступны, но пользователю придется отобразить список параметров для себя.  <br/> |
