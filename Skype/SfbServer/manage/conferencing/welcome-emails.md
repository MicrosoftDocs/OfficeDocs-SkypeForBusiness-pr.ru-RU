# <a name="send-welcome-email-to-dial-in-users-in-skype-for-business-server"></a>Отправка приветствуемой электронной почты пользователям в Skype для бизнеса Server
 
**Сводка:** Узнайте, как приветствовать пользователей на телефонных встречах в Skype для бизнеса Server.
  
После настройки телефонных конференциалов и тестирования, чтобы убедиться, что он функционирует должным образом, необходимо установить начальные персональные идентификационные номера (ПИН-коды) для пользователей и уведомить пользователей о доступности этой функции. Вы можете включить вводные инструкции, например начальный ПИН-код и ссылку на веб-страницу с Параметры телефонной связью. 
  
Как правило, для сброса ПИН-кодов используется комлет **Set-CsClientPin** , но вы можете использовать процедуру в этом разделе, если вы хотите отправить приветствие с информацией ПИН-кода. Если вы не хотите отправлять электронную почту, вместо этого можно использовать **Set-CsClientPin** .
  
Вы можете использовать **скрипт Set-CsPinSendCAWelcomeMail, чтобы установить ПИН-код** и отправить приветственный адрес электронной почты одному пользователю. По умолчанию скрипт не сбрасывает ПИН-код, если он уже задан, но для принудительного сброса ПИН-кода можно использовать параметр Force. Сообщение электронной почты отправляется с помощью простого протокола передачи почты (SMTP).
  
Можно создать сценарий, который запускает сценарий **Set-CsPinSendCAWelcomeMail** итеративным образом для набора ПИН-адресов и отправки электронной почты группе пользователей. Вы можете изменить шаблон электронной почты (то есть файл CAWelcomeEmailTemplate.html), чтобы добавить дополнительные ссылки на страницы интрасети или изменить текст электронной почты.
  


## <a name="set-an-initial-pin-and-send-welcome-email"></a>Установите начальный ПИН-код и отправьте приветственный адрес электронной почты

1. Войдите в группу RTCUniversalServerAdmins.
    
2. Запустите Skype для бизнеса Server: нажмите кнопку **Начните, нажмите** кнопку Все **программы, нажмите** кнопку Skype для бизнеса **2015**, а затем нажмите кнопку **Skype для бизнеса Server управленческой оболочки**.
    
3. Выполните в командной строке следующую команду:
    
   ```PowerShell
   Set-CsPinSendCAWelcomeMail -UserUri <user identifier>
   -From <email address of sender> [-Subject <subject for email message>]
   [-UserEmailAddress <destination email address>]
   [-Cc <email address of recipients who receive copy of email>]
   [-Bcc <email address of recipients who receive blind copies>]
   [-TemplatePath <path for email template>]
   [-SmtpServer] <SMTP server name>]
   [-BodyAsPlainText] [-UseSsl]
   [-Pin <new numeric PIN>] [-Force] `
   [-Credential <SMTP server credentials used to send email with the specified From address>]
   ```

**SmtpServer** По умолчанию сценарий использует значение переменной **зарезервированной среды $PSEmailServer** для этого параметра. Если **переменная $PSEmailServer** не установлена, необходимо указать этот параметр.
    
**Учетные данные** По умолчанию скрипт использует учетные данные текущего пользователя. Если у текущего пользователя нет разрешения на отправку электронной почты от имени указанного Адреса, необходимо указать этот параметр. Как правило, укажите этот параметр, если не указываете адрес электронной почты как адрес From.
    
В следующем примере создается новый ПИН-код, а затем отправляется приветствие от Марко бобу. Он использует текст электронной почты из шаблона по умолчанию и создает сообщение электронной почты в формате HTML. Тема по умолчанию : "Добро пожаловать на прием в конференцию":
  
```PowerShell
Set-CsPinSendCAWelcomeMail -UserUri "bob@contoso.com"
-From "marco@contoso.com"
```

В следующем примере новый ПИН-код с значением "383042650" для Боба, даже если у Боба был существующий ПИН-код, а затем отправляет приветствие от Марко Бобу. Так как параметр Credential задан, человеку, который работает с командой, будет предложено ввести пароль. Электронная почта отправляется с помощью слоя Безопасные sockets (SSL):
  
```PowerShell
Set-CsPinSendCAWelcomeMail -UserUri "bob@contoso.com"
-From "marco@contoso.com" -Subject "Your new dial-in conferencing PIN"
-Pin "383042650" -Force
-Credential Admin@contoso.com -UseSsl
```
