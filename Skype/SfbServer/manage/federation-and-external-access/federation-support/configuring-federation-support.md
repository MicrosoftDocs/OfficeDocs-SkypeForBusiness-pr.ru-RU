# <a name="configuring-federation-support-for-a-skype-for-business-online-customer-in-skype-for-business-server"></a>Настройка поддержки федерации для клиента Skype для бизнеса Online в Skype для бизнеса Server 

Вы можете предоставить доступ к службам связи для пользователей в вашей организации одним из указанных ниже способов.

  - Развертывание Skype для бизнеса Server в вашей организации (называемом *локальными службами*) и Настройка учетных записей пользователей Skype для бизнеса в Организации.
  - Настройка учетной записи клиента Microsoft Skype для бизнеса Online с поставщиком услуг размещения и Настройка учетных записей пользователей с помощью поставщика услуг размещения (называемых *веб-службами*).

Если вы развертываете Skype для бизнеса в своей организации, вы можете использовать федерацию с доменами одного или нескольких клиентов Skype для бизнеса Online. Чтобы включить федерацию между пользователями локального развертывания Skype для бизнеса и пользователей Skype для бизнеса Online, необходимо настроить поддержку для домена и пользователей клиента Skype для бизнеса Online.

> [!NOTE]  
> В этой статье описаны действия, которые необходимо выполнить, чтобы настроить поддержку Федерации с клиентом Skype для бизнеса Online. В этой документации не описаны процедуры для настройки клиента Skype для бизнеса Online в целях поддержки Федерации. 

## <a name="prerequisites-for-federating-with-a-skype-for-business-online-customer"></a>Необходимые условия для Федерации с клиентом Skype для бизнеса Online

Чтобы создать федерацию с клиентом Skype для бизнеса Online, вы должны уже выполнить начальное развертывание и настройку сервера Skype для бизнеса Server в Организации. Доступны следующие политики:

  - Развертывание по крайней мере одного стандартного сервера выпуска или одного пула переднего плана Enterprise Edition в Организации. 
  - Включение внутренних учетных записей пользователей в Skype для бизнеса Server. 
  - Развертывание по крайней мере одного пограничного сервера и других компонентов, необходимых для поддержки внешнего доступа пользователей. Подробности можно найти [в разделе Управление интеграцией и внешним доступом к Skype для бизнеса Server](../managing-federation-and-external-access.md).
  - Включение поддержки Федерации в Организации и настройка соответствующего метода для управления доступом с помощью федеративных доменов. Дополнительные сведения можно найти в разделе [Включение и отключение удаленного доступа пользователей](../access-edge/enable-or-disable-remote-user-access.md) и [Управление поставщиками SIP Федерации для вашей организации](../sip-providers/manage-sip-federated-providers-for-your-organization.md).
  - Разрешение доступа внешних пользователей для пользователей в вашей организации. Дополнительные сведения можно найти [в разделе Назначение внешней политики доступа для пользователей, поддерживающих Skype для бизнеса](../external-access-policies/assign-an-external-user-access-policy.md).



## <a name="configure-federation-support-for-a-skype-for-business-online-domain"></a>Настройка поддержки федерации для домена Skype для бизнеса Online

Для Федерации с клиентом Skype для бизнеса Online необходимо выполнить следующие действия:

  - Настройка поддержки домена для клиента Skype для бизнеса Online 2010 (например, contoso.onmicrosoft.com). В соответствии [с требованиями для интеграции с клиентом Skype для бизнеса Online](#prerequisites-for-federating-with-a-skype-for-business-online-customer)вы должны уже включить федерацию для своей организации. Для включения Федерации необходимо указать метод, который будет использоваться для управления доступом федеративных доменов. Если ваша организация настроена на использование обнаружения, Добавление домена в список разрешенных организаций не является обязательным. Если вы не включите функцию обнаружения доменов, вы должны добавить доменное имя клиента Skype для бизнеса Online в список разрешенных доменов. Вы можете добавить доменное имя либо с помощью панели управления Skype для бизнеса Server, либо путем запуска командлета **New-ксалловеддомаин** . Подробнее об использовании панели управления Skype для бизнеса Server, в том числе для включения обнаружения доменов, можно узнать в статьях [Управление поставщиками SIP-Федерации для вашей организации в Skype для бизнеса Server](../sip-providers/manage-sip-federated-providers-for-your-organization.md). Подробнее о том, как использовать командлет **New-ксалловеддомаин** для добавления домена, можно узнать в статьях [New-ксалловеддомаин](https://docs.microsoft.com/en-us/powershell/module/skype/New-CsAllowedDomain).

    > [!NOTE]  
    > У клиента Skype для бизнеса Online может быть несколько доменов. Если вы хотите сделать Федерацию более чем одним доменом, необходимо настроить поддержку для каждого домена, с которым вы хотите поддерживать Федерацию, и администратором клиента Skype для бизнеса Online необходимо включить федерацию для каждого из доменов. быть федеративным.

  - Настройка поддержки поставщика услуг размещения домена клиентов Skype для бизнеса Online, с которым вы хотите добавить Федерацию. С помощью описанной в этом разделе процедуры можно настроить поддержку для поставщика услуг размещения.

    > [!NOTE]  
    > Этот этап необходим только для Федерации с доменом клиента Skype для бизнеса Online, но не для Федерации с доменом, который развернут локально в расположении федеративного партнера.


### <a name="to-configure-support-for-a-hosting-provider"></a>Настройка поддержки поставщика услуг размещения

1.  На сервере переднего плана запустите командную консоль управления Skype для бизнеса Server: нажмите кнопку **Пуск**, выберите **все программы**, а затем — **Skype для бизнеса Server**, а затем — **Командная консоль управления Skype для бизнеса Server**.

2.  Запустите командлет **New-кшостингпровидер** для создания и настройки поставщика услуг размещения. Например, выполните следующую команду:
    
        New-CsHostingProvider -Identity LyncOnline -ProxyFqdn "sipfed.online.lync.com" -VerificationLevel UseSourceVerification -Enabled $True -EnabledSharedAddressSpace $False -HostsOCSUsers $False -IsLocal $False
    
    Команда в предыдущем примере задает следующие параметры:
    
      - **Identity** — уникальный идентификатор строкового значения для поставщика услуг размещения, который вы создаете. Обратите внимание, что если уже существует поставщик с таким удостоверением, то команда завершится неудачно.
    
      - Параметр **ProxyFQDN** определяет полное доменное имя прокси-сервера, используемого поставщиком услуг размещения. Это значение нельзя изменить. Если поставщик услуг размещения изменяет свой прокси-сервер, то придется удалить и заново создать запись для этого поставщика.
    
      - **Верификатионлевел** определяет способ проверки подлинности сообщений, отправленных поставщиком услуг размещения, чтобы убедиться, что они были отправлены этим поставщиком.
    
      - Параметр **Enabled** указывает, разрешено ли сетевое подключение между вашим доменом и поставщиком услуг размещения. Две организации не смогут обмениваться сообщениями, пока этот параметр не будет установлен в значение **True**.
    
      - Параметр **EnabledSharedAddressSpace** указывает, используется ли этот поставщик услуг размещения в сценарии общего адресного пространства SIP (разделенного домена).
    
      - **Хостсоксусерс** указывает, используется ли поставщик услуг размещения для размещения учетных записей Skype для бизнеса Server. Если **False**, то поставщик размещает другие типы учетных записей, такие как учетные записи Microsoft Exchange.
    
      - "WebProxy **" показывает,** является ли прокси-сервер, используемый поставщиком услуг размещения, включен в топологию сервера Skype для бизнеса.
    
    Подробнее об использовании этого командлета можно узнать в статьях [New-кшостингпровидер](https://docs.microsoft.com/en-us/powershell/module/skype/New-CsHostingProvider).

## <a name="configure-user-access-for-federation-with-a-skype-for-business-online-customer"></a>Настройка доступа пользователей для Федерации с помощью клиента Skype для бизнеса Online 

Вы должны настроить учетные записи пользователей для всех пользователей в Организации, чтобы они могли общаться с федеративными партнерами. Эта конфигурация применяется для всех федеративных партнеров, включая любые домены клиентов Microsoft Skype для бизнеса Online, с которыми вы поддерживаете Федерацию. Подробнее о настройке поддержки федерации для учетных записей пользователей можно узнать в разделе [Настройка политик для управления доступом к федеративным пользователям](../external-access-policies/configure-policies-to-control-federated-user-access.md) и [назначение внешней политики доступа для пользователя, поддерживающего Skype для бизнеса](../external-access-policies/assign-an-external-user-access-policy.md).

## <a name="verify-communications-with-a-skype-for-business-online-customer-in-skype-for-business-server"></a>Проверка связи с клиентом Skype для бизнеса Online в Skype для бизнеса Server

Чтобы пользователи Skype для бизнеса в организации могли общаться с пользователями Skype для бизнеса Online, необходимо выполнить указанные ниже действия.

  - Выполнены все необходимые условия. Сюда входит развертывание внутренних и пограничных серверов, включение поддержки федерации для Организации и Настройка учетных записей пользователей. Дополнительные сведения можно найти в разделе [Предварительные требования для Федерации с клиентом Skype для бизнеса Online](#prerequisites-for-federating-with-a-skype-for-business-online-customer).
  - Настройка поддержки доступа к доменам во внутренней среде. Сюда входит создание записи поставщика узла и Настройка развертывания для разрешения доступа из домена клиента Skype для бизнеса Online. Подробности можно найти [в разделе Настройка поддержки федерации для домена Skype для бизнеса Online](#configure-federation-support-for-a-skype-for-business-online-domain).
  - Настройка учетных записей пользователей для поддержки Федерации. Подробности можно найти в разделе [Настройка доступа пользователей для Федерации с помощью клиента Skype для бизнеса Online](#configure-user-access-for-federation-with-a-skype-for-business-online-customer).

После выполнения всех этих действий и администратором клиента Skype для бизнеса Online завершается вся настройка интернет-служб для поддержки Федерации с вашей организацией, проверка связи путем тестирования связи между внутренний пользователь в Организации и пользователь клиента Skype для бизнеса Online. Если связь не была успешной, для устранения проблемы используйте средство ведения журналов из пограничного сервера для захвата файлов журнала и трассировки.
