# <a name="configuring-federation-support-for-a-skype-for-business-online-customer-in-skype-for-business-server"></a>Настройка поддержки федерации для клиента Skype для бизнеса Online в Skype для бизнеса Server

Вы можете предоставить службы мгновенных сообщений пользователям свой организации любым из следующих способов:

- Развертывание Skype для бизнеса Server в организации (называемые локальными службами *) и* настройка Skype для бизнеса учетных записей пользователей в организации.
- Настройка учетной записи клиента Microsoft Skype для бизнеса Online с поставщиком услуг размещения и настройка учетных записей пользователей с поставщиком *услуг размещения* (известным как веб-службы).

При развертывании Skype для бизнеса в организации можно создать федерацию с доменами одного или нескольких Skype для бизнеса Online. Чтобы включить федерацию между пользователями локального развертывания Skype для бизнеса и пользователями клиента Skype для бизнеса Online, необходимо настроить поддержку для домена и пользователей клиента Skype для бизнеса Online.

[!INCLUDE [sfbo-retirement-skype](../../../../Hub/includes/sfbo-retirement.md)]

> [!NOTE]  
> В этой документации описаны только процедуры настройки организации для поддержки федерации с Skype для бизнеса Online. В этой документации не описаны процедуры настройки Skype для бизнеса Online для поддержки федерации.

## <a name="prerequisites-for-federating-with-a-skype-for-business-online-customer"></a>Необходимые условия для федерации с клиентом Skype для бизнеса Online

Чтобы создать федерацию с клиентом Skype для бизнеса Online, вы уже завершили начальное развертывание и настройку Skype для бизнеса Server в вашей организации. В том числе:

- Развертывание по крайней мере выпуск Standard сервера или выпуск Enterprise интерфейсного пула в организации.
- Включение внутренних учетных записей пользователей для Skype для бизнеса Server.
- Развертывание по крайней мере одного пограничного сервера и других компонентов, необходимых для поддержки доступа внешних пользователей. Дополнительные сведения см. в разделе ["Управление федерацией и внешним доступом к Skype для бизнеса Server](../managing-federation-and-external-access.md)".
- Активация поддержки федераций в организации и настройка надлежащего метода управлением доступа к федеративным доменам. Дополнительные сведения см. в разделе ["Включение или отключение](../access-edge/enable-or-disable-remote-user-access.md) удаленного доступа пользователей и управление федеративными поставщиками [SIP для вашей организации"](../sip-providers/manage-sip-federated-providers-for-your-organization.md).
- Активация внешнего доступа для пользователей в организации. Дополнительные сведения см. в статье "Назначение политики доступа внешних пользователей Skype для бизнеса [включенной учетной записи пользователя"](../external-access-policies/assign-an-external-user-access-policy.md).

## <a name="configure-federation-support-for-a-skype-for-business-online-domain"></a>Настройка поддержки федерации для домена Skype для бизнеса Online

Для федерации с Skype для бизнеса Online необходимо выполнить следующие действия:

- Настройте поддержку домена клиента Skype для бизнеса Online 2010 (например, contoso.onmicrosoft.com). Как указано в предварительных требованиях для федерации с клиентом [Skype для бизнеса Online](#prerequisites-for-federating-with-a-skype-for-business-online-customer), необходимо включить федерацию для вашей организации. Для включения федерации необходимо указать метод, используемый для управления доступом федеративных доменов. Если вы настроите в организации использование обнаружения, добавление домена в список разрешенных для вашей организации необязательный параметр. Если обнаружение домена не включено, необходимо добавить доменное имя клиента Skype для бизнеса Online в список разрешенных доменов. Доменное имя можно добавить с помощью Skype для бизнеса Server панель управления или **командлета New-CSAllowedDomain**. Дополнительные сведения об использовании Skype для бизнеса Server панель управления, включая включение обнаружения доменов, см. в разделе "Управление федеративными поставщиками [SIP для вашей организации" Skype для бизнеса Server](../sip-providers/manage-sip-federated-providers-for-your-organization.md). Дополнительные сведения об использовании **командлета New-CSAllowedDomain** для добавления домена см. в разделе [New-CsAllowedDomain](/powershell/module/skype/New-CsAllowedDomain).

  > [!NOTE]  
  > Клиент Skype для бизнеса Online может иметь несколько доменов. Если вы хотите создать федерацию с несколькими доменами, необходимо настроить поддержку для каждого отдельного домена, с которым вы хотите поддерживать федерацию, а администратор клиента Skype для бизнеса Online должен включить федерацию для каждого домена для федерации.

- Настройте поддержку поставщика услуг размещения домена Skype для бизнеса Online, с которым вы хотите создать федерацию. Используйте процедуру, описанную в этом разделе, чтобы настроить поддержку поставщика услуг размещения.

  > [!NOTE]  
  > Этот шаг требуется только для федерации с доменом клиента Skype для бизнеса Online, а не для федерации с любым доменом, который развертывается локально в расположении федеративного партнера.

### <a name="to-configure-support-for-a-hosting-provider"></a>Настройка поддержки поставщика услуг размещения

1. На сервере переднего плана запустите командную консоль Skype для бизнеса Server: нажмите кнопку "Пуск **", выберите** пункт "**Все** программы **", Skype для бизнеса Server** и Skype для бизнеса Server **командной консоли**.

2. Выполните **командлет New-CsHostingProvider** , чтобы создать и настроить поставщик услуг размещения. Например, выполните следующую команду:

    ```powershell
    New-CsHostingProvider -Identity LyncOnline -ProxyFqdn "sipfed.online.lync.com" -VerificationLevel UseSourceVerification -Enabled $True -EnabledSharedAddressSpace $False -HostsOCSUsers $False -IsLocal $False
    ```

    Команда в предыдущем примере задает следующие параметры:

    - **Удостоверение** указывает уникальный идентификатор строкового значения для создаемого поставщика услуг размещения. Обратите внимание, что если уже существует поставщик с таким удостоверением, то команда завершится неудачно.

    - Параметр **ProxyFQDN** определяет полное доменное имя прокси-сервера, используемого поставщиком услуг размещения. Это значение нельзя изменить. Если поставщик услуг размещения изменяет свой прокси-сервер, то придется удалить и заново создать запись для этого поставщика.

    - **VerificationLevel** указывает, как (или если) сообщения, отправленные от поставщика услуг размещения, проверяются, чтобы убедиться, что они были отправлены от этого поставщика.

    - Параметр **Enabled** указывает, разрешено ли сетевое подключение между вашим доменом и поставщиком услуг размещения. Две организации не смогут обмениваться сообщениями, пока этот параметр не будет установлен в значение **True**.

    - Параметр **EnabledSharedAddressSpace** указывает, используется ли этот поставщик услуг размещения в сценарии общего адресного пространства SIP (разделенного домена).

    - **HostsOCSUsers указывает**, используется ли поставщик услуг размещения для размещения Skype для бизнеса Server учетных записей. Если установлено значение **False**, то поставщик размещает другие типы учетных записей, такие как учетные записи Microsoft Exchange.

    - **IsLocal** указывает, содержится ли прокси-сервер, используемый поставщиком услуг размещения, в топологии Skype для бизнеса Server данных.

    Дополнительные сведения об использовании этого командлета см. в [разделе New-CsHostingProvider](/powershell/module/skype/New-CsHostingProvider).

## <a name="configure-user-access-for-federation-with-a-skype-for-business-online-customer"></a>Настройка доступа пользователей для федерации с клиентом Skype для бизнеса Online

Вам нужно настроить учетные записи всех пользователей организации, которым должно быть разрешено взаимодействие с федеративными партнерами. Эта конфигурация применяется ко всем федеративным партнерам, включая все домены клиентов Microsoft Skype для бизнеса Online, с которыми вы поддерживаете федерацию. Дополнительные сведения о настройке поддержки федерации для учетных записей [](../external-access-policies/configure-policies-to-control-federated-user-access.md) пользователей см. в разделе ["Настройка](../external-access-policies/assign-an-external-user-access-policy.md) политик для управления федеративным доступом пользователей и назначение политики доступа внешних пользователей пользователю с поддержкой Skype для бизнеса пользователей".

## <a name="verify-communications-with-a-skype-for-business-online-customer-in-skype-for-business-server"></a>Проверка связи с клиентом Skype для бизнеса Online в Skype для бизнеса Server

Чтобы пользователи Skype для бизнеса организации могли взаимодействовать с пользователями клиента Skype для бизнеса Online, необходимо выполнить следующие действия:

- Соблюсти все обязательные условия. К ним относятся развертывание внутренних и пограничных серверов, включение поддержки федерации для организации и настройка пользовательских учетных записей. Дополнительные сведения см. [в разделе "Предварительные требования для федерации с клиентом Skype для бизнеса Online"](#prerequisites-for-federating-with-a-skype-for-business-online-customer).
- Настроить поддержку доступа к домену во внутреннем развертывании. Это включает в себя создание записи поставщика узла и настройку развертывания, чтобы разрешить доступ Skype для бизнеса из домена клиента Skype для бизнеса Online. Дополнительные сведения см. в [разделе "Настройка поддержки федерации для домена Skype для бизнеса Online"](#configure-federation-support-for-a-skype-for-business-online-domain).
- Настроить пользовательские учетные записи для поддержки федерации. Дополнительные сведения см. в разделе "Настройка доступа пользователей [для федерации с Skype для бизнеса Online"](#configure-user-access-for-federation-with-a-skype-for-business-online-customer).

После выполнения всех этих действий администратор клиента Skype для бизнеса Online завершит настройку своего веб-службы для поддержки федерации с вашей организацией, проверит связь, проверив связь между внутренним пользователем в организации и пользователем клиента Skype для бизнеса Online. Если обмен данными не был успешным, используйте средство ведения журнала с пограничного сервера для записи файлов журнала и трассировки, чтобы устранить проблему.
