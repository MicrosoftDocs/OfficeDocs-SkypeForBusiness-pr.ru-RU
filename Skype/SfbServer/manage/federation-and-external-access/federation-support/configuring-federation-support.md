# <a name="configuring-federation-support-for-a-skype-for-business-online-customer-in-skype-for-business-server"></a>Настройка поддержки федерации для клиента Skype для бизнеса Online в Skype для бизнеса Server 

Вы можете предоставить службы мгновенных сообщений пользователям свой организации любым из следующих способов:

  - Развертывание Skype для бизнеса Server в организации (известные как локальное *обслуживание)* и настройка учетных записей пользователей Skype для бизнеса в организации.
  - Настройка учетной записи клиента Microsoft Skype для бизнеса Online у поставщика услуг размещения и настройка учетных записей пользователей у поставщика услуг размещения (также известных как *веб-службы).*

Если вы развертываете Skype для бизнеса в своей организации, вы можете вести федератерию с доменами одного или более клиентов Skype для бизнеса Online. Чтобы включить федерацию между пользователями локального развертывания Skype для бизнеса и пользователями клиента Skype для бизнеса Online, необходимо настроить поддержку домена и пользователей клиента Skype для бизнеса Online.

> [!NOTE]  
> В этой документации описываются только процедуры настройки организации для поддержки федерации с клиентом Skype для бизнеса Online. В этой документации не описываются процедуры настройки поддержки федерации клиентом Skype для бизнеса Online. 

## <a name="prerequisites-for-federating-with-a-skype-for-business-online-customer"></a>Необходимые условия для федерации с клиентом Skype для бизнеса Online

Для федерации с клиентом Skype для бизнеса Online необходимо завершить начальное развертывание и настройку Skype для бизнеса Server в организации. Эта кнопка предоставляет доступ ко следующим командам:

  - Развертывание по крайней мере одного сервера Standard Edition или одного пула переднего сервера Enterprise Edition в организации. 
  - Включение внутренних учетных записей пользователей для Skype для бизнеса Server. 
  - Развертывание по крайней мере одного сервера и других компонентов, необходимых для поддержки доступа внешних пользователей. Подробные сведения см. в управлении [федерацией и внешним доступом к Skype для бизнеса Server.](../managing-federation-and-external-access.md)
  - Активация поддержки федераций в организации и настройка надлежащего метода управлением доступа к федеративным доменам. Для получения подробных сведений см. сведения о включите [или отключите](../access-edge/enable-or-disable-remote-user-access.md) удаленный доступ пользователей и управляйте федератерными [поставщиками SIP для вашей организации.](../sip-providers/manage-sip-federated-providers-for-your-organization.md)
  - Активация внешнего доступа для пользователей в организации. Дополнительные сведения см. в сведениях о назначении политики доступа внешних пользователей [пользователю, включенного в Skype для бизнеса.](../external-access-policies/assign-an-external-user-access-policy.md)



## <a name="configure-federation-support-for-a-skype-for-business-online-domain"></a>Настройка поддержки федерации для домена Skype для бизнеса Online

Для федерации с клиентом Skype для бизнеса Online необходимо выполнить следующие действия:

  - Настройте поддержку домена клиента Skype для бизнеса Online 2010 (например, contoso.onmicrosoft.com). Как указано в [предварительных условия для](#prerequisites-for-federating-with-a-skype-for-business-online-customer)федерации с клиентом Skype для бизнеса Online, федерация для вашей организации уже должна быть включена. Для включения федерации необходимо указать метод, используемый для управления доступом федератных доменов. Если в организации настроено использование функции обнаружения, добавление домена в список разрешенных не является обязательным. Если обнаружение доменов не было разрешено, необходимо добавить доменное имя клиента Skype для бизнеса Online в список разрешенных доменов. Вы можете добавить доменное имя с помощью панели управления Skype для бизнеса Server или с помощью **cmdlet New-CSAllowedDomain.** For details about using Skype for Business Server Control Panel, including enabling discovery of [domains, see Manage SIP federated providers for your organization in Skype for Business Server.](../sip-providers/manage-sip-federated-providers-for-your-organization.md) Дополнительные сведения об использовании cmdlet **New-CSAllowedDomain** для добавления домена см. в подстройки [New-CsAllowedDomain.](https://docs.microsoft.com/powershell/module/skype/New-CsAllowedDomain)

    > [!NOTE]  
    > Клиент Skype для бизнеса Online может иметь несколько доменов. Если вы хотите установить федерацию с более чем одним из доменов, необходимо настроить поддержку для каждого отдельного домена, с которым вы хотите поддерживать федерацию, а администратор клиента Skype для бизнеса Online должен включить федерацию для каждого из доменов для федерации.

  - Настройте поддержку поставщика услуг размещения домена клиента Skype для бизнеса Online, с которым вы хотите установить федератуату. Используйте процедуру, данная в этом разделе, для настройки поддержки поставщика услуг размещения.

    > [!NOTE]  
    > Этот шаг необходим только для федерации с доменом клиента Skype для бизнеса Online, а не для федерации с доменом, развернутым локально в расположении федератного партнера.


### <a name="to-configure-support-for-a-hosting-provider"></a>Настройка поддержки поставщика услуг размещения

1.  From a Front End **Server,** Start the Skype for Business Server Management Shell: Click **Start,** click **All Programs,** click Skype for Business Server , and then click **Skype for Business Server Management Shell**.

2.  Запустите **cmdlet New-CsHostingProvider,** чтобы создать и настроить поставщика услуг размещения. Например, выполните следующую команду:
    
        New-CsHostingProvider -Identity LyncOnline -ProxyFqdn "sipfed.online.lync.com" -VerificationLevel UseSourceVerification -Enabled $True -EnabledSharedAddressSpace $False -HostsOCSUsers $False -IsLocal $False
    
    Команда в предыдущем примере задает следующие параметры:
    
      - **Идентификатор** указывает уникальный идентификатор строки для создающегося поставщика услуг размещения. Обратите внимание, что если уже существует поставщик с таким удостоверением, то команда завершится неудачно.
    
      - Параметр **ProxyFQDN** определяет полное доменное имя прокси-сервера, используемого поставщиком услуг размещения. Это значение нельзя изменить. Если поставщик услуг размещения изменяет свой прокси-сервер, то придется удалить и заново создать запись для этого поставщика.
    
      - **VerificationLevel** указывает, как (или если) сообщения, отправленные от поставщика услуг размещения, проверяются, чтобы убедиться, что они были отправлены от этого поставщика.
    
      - Параметр **Enabled** указывает, разрешено ли сетевое подключение между вашим доменом и поставщиком услуг размещения. Две организации не смогут обмениваться сообщениями, пока этот параметр не будет установлен в значение **True**.
    
      - Параметр **EnabledSharedAddressSpace** указывает, используется ли этот поставщик услуг размещения в сценарии общего адресного пространства SIP (разделенного домена).
    
      - **HostsOCSUsers** указывает, используется ли поставщик услуг размещения для размещения учетных записей Skype для бизнеса Server. Если установлено значение **False**, то поставщик размещает другие типы учетных записей, такие как учетные записи Microsoft Exchange.
    
      - **IsLocal** указывает, содержится ли прокси-сервер, используемый поставщиком услуг размещения, в топологии Skype для бизнеса Server.
    
    Подробные сведения об использовании этого cmdlet см. в подпрограмме [New-CsHostingProvider.](https://docs.microsoft.com/powershell/module/skype/New-CsHostingProvider)

## <a name="configure-user-access-for-federation-with-a-skype-for-business-online-customer"></a>Настройка доступа пользователей для федерации с клиентом Skype для бизнеса Online 

Вам нужно настроить учетные записи всех пользователей организации, которым должно быть разрешено взаимодействие с федеративными партнерами. Эта конфигурация применяется для всех федераированных партнеров, включая любые домены клиентов Microsoft Skype для бизнеса Online, с которыми вы поддерживаете федерацию. Дополнительные сведения о настройке поддержки федерации [](../external-access-policies/configure-policies-to-control-federated-user-access.md) для учетных записей пользователей см. в настройках политик для управления доступом федераированных пользователей и назначении политики доступа внешних пользователей пользователю с включенной поддержкой Skype для [бизнеса.](../external-access-policies/assign-an-external-user-access-policy.md)

## <a name="verify-communications-with-a-skype-for-business-online-customer-in-skype-for-business-server"></a>Проверка связи с клиентом Skype для бизнеса Online в Skype для бизнеса Server

Чтобы позволить пользователям Skype для бизнеса в организации взаимодействовать с пользователями клиента Skype для бизнеса Online, необходимо сделать следующее:

  - Соблюсти все обязательные условия. К ним относятся развертывание внутренних и пограничных серверов, включение поддержки федерации для организации и настройка пользовательских учетных записей. Подробные сведения см. в предварительных условия для федерации с [клиентом Skype для бизнеса Online.](#prerequisites-for-federating-with-a-skype-for-business-online-customer)
  - Настроить поддержку доступа к домену во внутреннем развертывании. Это включает создание записи поставщика услуг размещения и настройку развертывания, чтобы разрешить доступ из домена клиента Skype для бизнеса Online. For details, see [Configure federation support for a Skype for Business Online domain](#configure-federation-support-for-a-skype-for-business-online-domain).
  - Настроить пользовательские учетные записи для поддержки федерации. Подробные сведения см. в подстроке "Настройка доступа [пользователей для федерации с клиентом Skype для бизнеса Online".](#configure-user-access-for-federation-with-a-skype-for-business-online-customer)

После выполнения всех этих действий и завершения администратором клиента Skype для бизнеса Online всех настроек своих веб-служб для поддержки федерации с вашей организацией проверьте связь, проверив связь между внутренним пользователем в организации и пользователем клиента Skype для бизнеса Online. Если связь не удалась, используйте средство ведения журнала с вашего сервера для записи файлов журнала и трассировки, чтобы устранить проблему.
