# <a name="configuring-federation-support-for-a-skype-for-business-online-customer-in-skype-for-business-server"></a>Настройка поддержки федерации для Скайп для бизнеса в Интернет клиента в Скайп Business Server 

Можно предоставить службы мгновенных сообщений для пользователей в вашей организации в любом из следующих способов:

  - Развертывание Скайп для Business Server в организации (известную как *локальные службы*) и настройке Скайп для бизнеса учетных записей пользователей в организации.
  - Настройка Microsoft Скайп для бизнеса в Интернет учетной записи клиента с поставщиком услуг размещения и учетных записей пользователей с этим поставщиком услуг размещения (известную как *Интернет-служб*).

При развертывании Скайп для бизнеса в вашей организации, с которыми возможна федерация доменов Скайп один или несколько клиентов бизнеса через Интернет. Чтобы включить федерацию между пользователей из вашей локальной Скайп по развертыванию и Скайп для бизнеса в Интернет клиента, необходимо настроить поддержку доменов и пользователей Скайп для бизнеса в Интернет клиента.

> [!NOTE]  
> В этой документации описываются только процедуры для настройки вашей организации для поддержки федерации с Скайп для бизнеса в Интернет клиента. В этой документации описаны процедуры для настройки Скайп для бизнеса в Интернет клиента для поддержки федерации. 

## <a name="prerequisites-for-federating-with-a-skype-for-business-online-customer"></a>Необходимые условия для федерации с Скайп для бизнеса в Интернет клиента

В федерацию с Скайп для бизнеса в Интернет клиента должны уже выполнена конфигурации Скайп для Business Server и необходимые для развертывания в организации. Доступны следующие политики:

  - Развертывание по крайней мере один сервер Standard Edition или один пул переднего плана Enterprise Edition в вашей организации. 
  - Включение внутренних учетных записей для Скайп для Business Server. 
  - Развертывание по крайней мере один пограничного сервера и других компонентов, необходимых для поддержки доступа внешних пользователей. Дополнительные сведения см [Управление Федерация и внешний доступ к Скайп для Business Server](../managing-federation-and-external-access.md).
  - Включение поддержки федерации в пределах организации и настройка соответствующий метод для управления доступом с федеративных доменов. Дополнительные сведения см [включения или отключения удаленного доступа пользователей](../access-edge/enable-or-disable-remote-user-access.md) и [управлять SIP федеративных поставщиков для вашей организации](../sip-providers/manage-sip-federated-providers-for-your-organization.md).
  - Включение доступа внешних пользователей для пользователей в вашей организации. Дополнительные сведения см в [назначении политики доступа внешних пользователей к Скайп для пользователя включена бизнеса](../external-access-policies/assign-an-external-user-access-policy.md).



## <a name="configure-federation-support-for-a-skype-for-business-online-domain"></a>Настройка поддержки федерации для Скайп для бизнеса в Интернет-домена

Федеративные отношения с Скайп для бизнеса в Интернет клиента необходимо выполнить следующие действия:

  - Настройка поддержки для домена Скайп для бизнеса Online 2010 customer (например, contoso.onmicrosoft.com). Как указано в [Необходимые условия для федерации с Скайп для бизнеса в Интернет клиента](#prerequisites-for-federating-with-a-skype-for-business-online-customer)должна уже включена федерации для вашей организации. Включение федерации требуется указать метод, используемый для управления доступом федеративных доменов. Если вы настроили организации к использованию обнаружения, Добавление домена в список разрешенных вашей организации является необязательным. Если не было включить обнаружение доменов, необходимо добавить имя домена Скайп для бизнеса в Интернет клиента в список разрешенных доменов. Можно добавить имя домена, с помощью Скайп для панели управления Business Server или с помощью командлета **New-CSAllowedDomain** . Для получения дополнительных сведений об использовании Скайп для панели управления Business Server, включая Включение обнаружения доменов видеть [Управление SIP федеративных поставщиков для вашей организации в Скайп для Business Server](../sip-providers/manage-sip-federated-providers-for-your-organization.md). Для получения дополнительных сведений об использовании командлета **New-CSAllowedDomain** для добавления домена видеть [New-CsAllowedDomain](https://docs.microsoft.com/en-us/powershell/module/skype/New-CsAllowedDomain).

    > [!NOTE]  
    > Скайп для бизнеса в Интернет клиента могут иметь несколько доменов. Если вы хотите создать федерацию с несколько доменов, необходимо настроить поддержку каждого отдельного домена, с которой вы хотите обеспечить поддержку федерации, а администратор Скайп для бизнеса в Интернет клиента необходимо включить федерацию для каждого из доменов для федеративные.

  - Настройка поддержки для поставщика услуг размещения Скайп для бизнеса в Интернет домена клиента, с которым вы хотите добавить в отношение федерации. Настройка поддержки для поставщика услуг размещения, используйте процедуру в этом разделе.

    > [!NOTE]  
    > Этот шаг является обязательным только при использовании федерации с доменом Скайп для бизнеса в Интернет клиента, а не с любым доменом, который является развертываются локально в расположении федеративного партнера.


### <a name="to-configure-support-for-a-hosting-provider"></a>Настройка поддержки для поставщика услуг размещения

1.  С сервера переднего плана, запустите Скайп для консоли Business Server: нажмите кнопку **Пуск**, последовательно выберите пункты **Все программы**, щелкните **Скайп для Business Server**и нажмите кнопку **Скайп для консоли Business Server**.

2.  Выполните командлет **New-CsHostingProvider** для создания и настройки поставщика услуг размещения. Например, выполните следующую команду:
    
        New-CsHostingProvider -Identity LyncOnline -ProxyFqdn "sipfed.online.lync.com" -VerificationLevel UseSourceVerification -Enabled $True -EnabledSharedAddressSpace $False -HostsOCSUsers $False -IsLocal $False
    
    Команда в предыдущем примере задает следующие параметры:
    
      - **Identity** определяет строковое значение уникального идентификатора для поставщика услуг размещения, которую вы создаете. Обратите внимание, что если уже существует поставщик с таким удостоверением, то команда завершится неудачно.
    
      - Параметр **ProxyFQDN** определяет полное доменное имя прокси-сервера, используемого поставщиком услуг размещения. Это значение нельзя изменить. Если поставщик услуг размещения изменяет свой прокси-сервер, то придется удалить и заново создать запись для этого поставщика.
    
      - Указывает, **VerificationLevel** как (или, если) будут проверены сообщений, отправляемых с поставщиком услуг размещения, чтобы убедиться, что они были отправлены.
    
      - Параметр **Enabled** указывает, разрешено ли сетевое подключение между вашим доменом и поставщиком услуг размещения. Две организации не смогут обмениваться сообщениями, пока этот параметр не будет установлен в значение **True**.
    
      - Параметр **EnabledSharedAddressSpace** указывает, используется ли этот поставщик услуг размещения в сценарии общего адресного пространства SIP (разделенного домена).
    
      - **HostsOCSUsers** указывает, используется ли поставщик услуг размещения для размещения Скайп для учетных записей Business Server. Если **False**, то поставщик размещает другие типы учетных записей, такие как учетные записи Microsoft Exchange.
    
      - **IsLocal** указывает, содержится ли прокси-сервер, используемый поставщиком услуг размещения в вашей Скайп для топологии Business Server.
    
    Сведения об использовании этого командлета [New-CsHostingProvider](https://docs.microsoft.com/en-us/powershell/module/skype/New-CsHostingProvider)см.

## <a name="configure-user-access-for-federation-with-a-skype-for-business-online-customer"></a>Настройка доступа пользователей для федерации с Скайп для бизнеса в Интернет клиента 

Пользователю необходимо настроить учетные записи всех пользователей в вашей организации в порядке их разрешено для взаимодействия с федеративных партнеров. Эта конфигурация применяется для всех федеративных партнеров, включая любые Скайп Майкрософт для бизнеса в Интернет доменов клиента, с которыми поддержки федерации. Для получения дополнительных сведений о настройке поддержки федерации для учетных записей пользователей видеть [федеративных Настройка политик для управления доступом пользователей](../external-access-policies/configure-policies-to-control-federated-user-access.md) и [назначить политику доступа внешних пользователей к Скайп для пользователя включена бизнеса](../external-access-policies/assign-an-external-user-access-policy.md).

## <a name="verify-communications-with-a-skype-for-business-online-customer-in-skype-for-business-server"></a>Проверка соединений с Скайп для бизнеса в Интернет клиента в Скайп для Business Server

Чтобы включить Скайп для бизнес-пользователей в вашей организации для взаимодействия с пользователями Скайп для бизнеса в Интернет клиента, необходимо выполнить следующие действия:

  - Если выполнены все необходимые компоненты. Этот компонент включает развертывание вашей внутренней и поддерживать пограничных серверов, включение федерации для вашей организации и учетных записей пользователей. Дополнительные сведения см [Необходимые условия для федерации с Скайп для бизнеса в Интернет клиента](#prerequisites-for-federating-with-a-skype-for-business-online-customer).
  - Настроить поддержку доступа домена во внутреннем развертывании. Этот компонент включает создания операции поставщика узла и настройка развертывания на разрешение доступа из Скайп для бизнеса в Интернет клиента домена. Дополнительные сведения см [федерации настроить поддержку Скайп для бизнеса в Интернет-домена](#configure-federation-support-for-a-skype-for-business-online-domain).
  - Настроить учетные записи пользователей для поддержки федерации. Дополнительные сведения см [настройки доступа пользователей для федерации с Скайп для бизнеса в Интернет клиента](#configure-user-access-for-federation-with-a-skype-for-business-online-customer).

Завершив все эти действия и администратора Скайп for Business Online клиент выполняет все конфигурации Интернет-служб для поддержки федерации с вашей организацией, проверьте связь с проверка связи между Внутренний пользователей в вашей организации и пользователей из Скайп для бизнеса в Интернет клиента. Если связи не удается, используйте средство ведения журнала из пограничного сервера для записи файлов журнала и трассировки для устранения неполадок.
