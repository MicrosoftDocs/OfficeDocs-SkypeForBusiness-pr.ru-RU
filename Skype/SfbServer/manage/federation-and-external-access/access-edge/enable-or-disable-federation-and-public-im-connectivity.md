# <a name="enable-or-disable-federation-and-public-im-connectivity-in-skype-for-business-server"></a>Включение и отключение федерации и подключения к общедоступным обмена мгновенными Сообщениями в Скайп для Business Server

Поддержка федерации необходим, чтобы разрешить пользователям, имеющим учетную запись с надежных организацией клиентов и партнеров, включая доменов партнеров и пользователей общедоступного, обмен мгновенными сообщениями пользователи поставщика обмена Мгновенными сообщениями, которые поддерживают, совместно работать с пользователями в вашей организация. Федерации также требуется использовать поставщик размещенной службы Exchange для предоставления голосовой почты для пользователей корпоративной голосовой связи, почтовые ящики которых располагаются в размещаемой службе Exchange, например, Microsoft Exchange Online. При установке отношения доверия с помощью этих внешних доменов можно авторизации пользователей в эти домены, чтобы получить доступ к развертыванию и участвовать в Скайп для коммуникаций Business Server. Это отношение доверия, называется федерацией и не относится к, или зависят от отношение доверия Active Directory.

Чтобы обеспечить поддержку доступа пользователей из федеративных доменов, необходимо включить федерации. При включении федерации для вашей организации, необходимо также указать, следует ли реализовать следующие параметры:

  - **Включить обнаружение доменов партнеров**   при включении этого параметра Скайп для Business Server использует для попробуйте для обнаружения домены, не указанные в списке разрешенных доменов обнаруженные автоматически оценке входящий трафик от записи доменных имен (DNS) федеративные партнеры и ограничения или блокировки, трафик, в зависимости от уровня доверия, объем трафика и параметры администрирования. Если этот параметр не выбран, доступ федеративных пользователей будет доступна только для пользователей в доменах, которые можно включить в список разрешенных доменов. Ли этот параметр, можно указать, он доменов, заблокированных или разрешенных, включая ограничения доступа для конкретных серверах под управлением службы пограничного сервера доступа в федеративного домена. Для получения дополнительных сведений об управлении доступом федеративных доменов в статье [Configure поддержки для разрешенных внешних доменов](../sip-domains/manage-sip-federated-domains-for-your-organization.md#configure-support-for-allowed-external-domains-in-skype-for-business-server).

  - **Отправить отказа от архивирования федеративным партнерам**     федеративных партнеров, архивацию в развертывании будет готова отправляется уведомление об отказе. Если вы поддерживаете архивацией внешних коммуникаций с доменами федеративного партнера, следует включить архивации уведомлений заявление об отказе для предупреждения партнеры архивирования сообщения, помещенные.

Если позднее вы хотите временно или окончательно запретить доступ пользователей из федеративных доменов, можно отключить федерации для вашей организации. Используйте процедуру в этом разделе, чтобы включить или отключить доступ федеративных пользователей для организации, в том числе Указание параметров соответствующие федерации для поддержки для вашей организации.

> [!NOTE]  
> Только включение федерации для вашей организации указывает, что серверы с пограничной службы доступа поддерживает маршрутизацию федеративных доменов. Пользователей в федеративных доменов не могут участвовать в обмена мгновенными Сообщениями и конференции в вашей организации, пока также настроить по крайней мере одна политика для поддержки доступа федеративных пользователей. Пользователи общедоступных поставщиков услуг обмена мгновенными Сообщениями не могут участвовать в обмена мгновенными Сообщениями и конференции в вашей организации, пока также настроить по крайней мере одна политика для поддержки общедоступной службы обмена Мгновенными сообщениями. Скайп для Business Server нельзя использовать в размещенной службе Exchange для предоставления автоответчика, голосового доступа к Outlook (в том числе голосовой почты), или автосекретаря служб для пользователей, почтовые ящики которых располагаются в размещаемой службе Exchange только после настройки размещенной голосовой политика почты, который предоставляет сведения о маршрутизации. Для получения дополнительных сведений о настройке политик для обмена данными с пользователями из федеративных доменов из других организаций видеть [Управление SIP федеративных доменов для вашей организации](../sip-domains/manage-sip-federated-domains-for-your-organization.md). Кроме того Если вы хотите поддерживать связь с пользователями поставщиков услуг обмена мгновенными Сообщениями, необходимо настроить политики, чтобы поддерживать его и также настроить поддержку поставщиков служб, которые необходимо поддерживать. Дополнительные сведения см [Управление SIP федеративных поставщиков для вашей организации](../sip-providers/manage-sip-federated-providers-for-your-organization.md).


## <a name="to-enable-or-disable-federated-user-access-for-your-organization"></a>Чтобы включить или отключить доступ федеративных пользователей для вашей организации

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 

3.  В левой панели навигации щелкните **Доступ для внешних пользователей**и нажмите кнопку **Настройка пограничного доступа**.

4.  На странице **Настройка пограничного доступа** щелкните **Глобальная**, нажмите кнопку **Изменить**и нажмите кнопку **Показать подробности**.

5.  В **Изменить настройку пограничного доступа**выполните одно из следующих действий.
    
      - Чтобы включить доступ федеративных пользователей для своей организации, установите флажок **Разрешить взаимодействие с федеративными пользователями** .
    
      - Чтобы отключить доступ федеративных пользователей для своей организации, снимите флажок **Разрешить взаимодействие с федеративными пользователями** .

6.  Если установлен флажок **Разрешить взаимодействие с федеративными пользователями** , выполните следующие действия.
    
    1.  Если необходимо обеспечить поддержку автоматического обнаружения доменов партнеров, установите флажок **Включить обнаружение доменов партнеров** .
    
    2.  Если ваша организация поддерживает архивацию внешних коммуникаций, установите флажок **отправлять уведомление об федеративным партнерам архивации** .

7.  Нажмите **Исполнить**.

Чтобы разрешить федеративным пользователям совместно работать с пользователями в вашей Скайп для развертывания Business Server, необходимо также настроить по крайней мере одной политики внешнего доступа для поддержки доступа федеративных пользователей. Дополнительные сведения см [федеративных Настройка политик для управления доступом пользователей](../external-access-policies/configure-policies-to-control-federated-user-access.md). Для управления доступом для определенных федеративных доменов, содержатся в разделе [Настройка поддержки для разрешенных внешних доменов](../sip-domains/manage-sip-federated-domains-for-your-organization.md#configure-support-for-allowed-external-domains-in-skype-for-business-server).


## <a name="enabling-or-disabling-federation-and-public-im-connectivity-by-using-windows-powershell-cmdlets"></a>Включение и отключение федерации и общедоступной службы обмена Мгновенными сообщениями с помощью командлетов Windows PowerShell

Федерация и общедоступной службы обмена Мгновенными сообщениями также может осуществляться с помощью командлета Set-CsAccessEdgeConfiguration и Windows PowerShell. Этот командлет можно запустить из Скайп для консоли Business Server или из удаленного сеанса Windows PowerShell. 

## <a name="to-enable-federation-and-public-im-connectivity"></a>Чтобы включить федерацию и общедоступные службы обмена Мгновенными сообщениями

  - Чтобы включить федерацию и общедоступные службы обмена Мгновенными сообщениями, задайте значение свойству **AllowFederatedUsers** значение True ($True):
    
        Set-CsAccessEdgeConfiguration -AllowFederatedUsers $True



## <a name="to-disable-federation-and-public-im-connectivity"></a>Чтобы отключить федерацию и общедоступные службы обмена Мгновенными сообщениями

  - Чтобы отключить федерацию и общедоступные службы обмена Мгновенными сообщениями, задайте значение свойству **AllowFederatedUsers** значение False ($False):
    
        Set-CsAccessEdgeConfiguration -AllowFederatedUsers $False
