# <a name="manage-sip-federated-domains-for-your-organization-in-skype-for-business-server"></a>Управление федеративных доменов SIP для вашей организации в Скайп для Business Server


Для управления и настройки доменами SIP, с которыми возможна федерация, выполните следующие действия.

  - Создавать или изменять список разрешенных федеративных доменов SIP партнеров.

  - Создание или редактирование список заблокированных федеративных доменов SIP.

## <a name="configure-support-for-allowed-external-domains-in-skype-for-business-server"></a>Настройка поддержки для разрешенных внешних доменов в Скайп для Business Server

Настроена поддержка федеративных партнеров, могут управлять какие определенных доменов можно создать федерацию с вашей организации. Настройка одного или нескольких конкретных внешних доменов как разрешенных федеративных доменов. Для этого добавьте каждый домен в список разрешенных доменов. Даже в том случае, если этот параметр включен обнаружения партнера для вашей организации, для этого домена является федеративного партнера, который может потребоваться общаться с более чем 1 000 пользователей или может потребоваться отправить с более чем 20 сообщений в секунду. Если обнаружение партнера для вашей организации не включена, только пользователей из внешних доменов, которые можно добавить в список разрешенных доменов могут участвовать в обмена мгновенными Сообщениями и конференц-связи с пользователями в вашей организации. Если требуется ограничить доступ для федеративного домена на определенном сервере запущена служба пограничного сервера доступа федеративного партнера, можно указать доменное имя сервера, на которых запущена служба пограничного сервера доступа для каждого домена в список разрешенных доменов.

> [!NOTE]  
> Эта процедура описывается настройка поддержки для определенных доменов, но также для реализации поддержки федеративных пользователей необходимо включить поддержку для федеративных пользователей для своей организации и настройка и применение политик управления пользователей, которые могут Совместная работа с федеративными пользователями. Для получения дополнительных сведений о включении поддержки федеративных пользователей видеть [включения или отключения удаленного доступа пользователей](../access-edge/enable-or-disable-remote-user-access.md). Для получения дополнительных сведений о настройке политик для управления федерации видеть [федеративных Настройка политик для управления доступом пользователей](../external-access-policies/configure-policies-to-control-federated-user-access.md).

### <a name="to-add-an-external-domain-to-the-list-of-allowed-domains"></a>Чтобы добавить внешний домен в список разрешенных доменов

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.
2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 
3.  В левой панели навигации щелкните **Доступ для внешних пользователей**и нажмите кнопку **Федеративные домены**.
4.  На странице **Федеративные домены** нажмите кнопку **Создать**и затем щелкните **разрешенный домен**.
5.  В **Новые федеративные домены**выполните следующие действия.
    
      - В поле **имя домена (или полное доменное имя)** введите имя домена федеративного партнера.       

        > [!NOTE]  
        > Это имя должно быть уникальным и не может уже существовать в качестве разрешенных доменов для этого сервера, на которых выполняется служба пограничного сервера доступа. Имя не может превышать 256 символов.<BR><br>Поиск по имени домена федеративного партнера выполняется сопоставление суффикс. Например при вводе **contoso.com**поиска также возвращать домена **it.contoso.com**.<BR><br>Домена федеративного партнера не может одновременно заблокированные и разрешенные. Скайп для Business Server не позволяет это сведений о событиях, чтобы у вас нет синхронизации копирование списков.
    
      - Чтобы ограничить доступ для этого федеративного домена для пользователей работает служба пограничного сервера доступа, **Пограничная служба доступа (FQDN)** определенного сервера введите полное доменное имя сервера федеративного домена со службой пограничного доступа.    
      - Если вы хотите предоставить дополнительную информацию, в поле **комментарий**введите сведения, которые следует предоставить другим системным администраторам о данной конфигурации.

6.  Нажмите **Исполнить**.
7.  Повторите шаги с 4 по 6 для каждого домена федеративного партнера, который следует разрешить.

Чтобы включить доступ федеративных пользователей, необходимо также включить поддержку для доступа федеративных пользователей в вашей организации. Дополнительные сведения см [включения или отключения удаленного доступа пользователей](../access-edge/enable-or-disable-remote-user-access.md).

Кроме того необходимо настроить и применить политику для пользователей, которые необходимо иметь возможность совместной работы с федеративными пользователями. Дополнительные сведения см [федеративных Настройка политик для управления доступом пользователей](../external-access-policies/configure-policies-to-control-federated-user-access.md).

## <a name="configure-support-for-blocked-external-domains-in-skype-for-business-server"></a>Настройка поддержки для заблокированных внешних доменов в Скайп для Business Server 

Настроена поддержка федеративных партнеров можно управлять, какие домены будут блокироваться из федерации с вашей организации. Список заблокированных доменов будет действовать как черного списка (список явные записей, которые не разрешено) и будет применяться в обнаружения федеративного домена, если этот параметр включен. Дополнительные сведения см [Включить или отключить обнаружение федеративных партнеров](../access-edge/enable-or-disable-discovery-of-federation-partners.md).

Заблокируйте один или несколько внешних доменов из подключенных к организации. Чтобы сделать это, добавьте домен в список заблокированных доменов.


### <a name="to-add-an-external-domain-to-the-list-of-blocked-domains"></a>Чтобы добавить внешний домен в список заблокированных доменов

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.
2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server. 
3.  В левой панели навигации щелкните **Доступ для внешних пользователей**.
4.  **Федеративные**домены, нажмите кнопку **Создать**и затем щелкните **заблокированный домен**.
5.  В **Новые федеративные домены**выполните следующие действия.
    
      - В **имени домена (или полное доменное имя)** введите имя домена федеративного партнера, которую требуется заблокировать.

        > [!NOTE]  
        > Имя не может превышать 256 символов.<BR><br>Поиск по имени домена федеративного партнера выполняется сопоставление суффикс. Например при вводе **contoso.com**поиска также возвращать домена **it.contoso.com**.<BR><br>Домена федеративного партнера не может одновременно заблокированные и разрешенные. Скайп для Business Server не позволяет это сведений о событиях, чтобы у вас нет синхронизации копирование списков.
   
      - (Необязательно) В поле **Примечание**введите сведения, которые следует предоставить другим системным администраторам о данной конфигурации.

6.  Нажмите **Исполнить**.
7.  Повторите шаги с 4 по 6 для каждого федеративного партнера, которую требуется заблокировать.

Чтобы включить доступ федеративных пользователей, необходимо также включить поддержку для доступа федеративных пользователей в вашей организации. Дополнительные сведения см [включения или отключения удаленного доступа пользователей](../access-edge/enable-or-disable-remote-user-access.md).

Кроме того необходимо настроить и применить политику для пользователей, которые необходимо иметь возможность совместной работы с федеративными пользователями. Дополнительные сведения см [федеративных Настройка политик для управления доступом пользователей](../external-access-policies/configure-policies-to-control-federated-user-access.md).
