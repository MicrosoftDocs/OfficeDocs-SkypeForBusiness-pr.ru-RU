# <a name="top-failures-report-in-skype-for-business-server"></a>TOP Failures Report в Скайп для Business Server
 
**Сводка:** Узнайте о отчета по основным сбоям в Скайп для Business Server.
  
Отчет по основным сбоям позволяет узнать о наиболее частых сбоях и характере их изменения с течением времени. Сбои основаны на сочетании двух следующих метрик:
  
- **Код диагностики**. Уникальный идентификатор (в виде заголовка ms-diagnostics), прикрепленный к сообщению SIP. Коды диагностики предоставляют информацию, полезную при устранении проблем со звонками.
    
- **Код ответа**. Коды ответа используются в сеансах связи SIP для ответы на запросы SIP. Предположим, что пользователь Ken отправляет запрос INVITE пользователю Pilar Ackerman (например, пользователь Ken Myer звонит пользователю Pilar Ackerman). Если Pilar отвечает, ее телефон отправляет код ответа 200 (OK), позволяя телефону Ken узнать об ответе Pilar. Отчетом по основным сбоям включает только коды ответа, которые были отправлены в ответ на звонок сбоя; Скайп для Business Server не хранить список всех кодов ответа, выдается во время звонка.
    
Приводится информация не только об общем числе сеансов, в которых возник сбой, но и об общем числе пользователей, столкнувшихся с этим сбоем.
  
## <a name="accessing-the-top-failures-report"></a>Доступ к отчету по основным сбоям

Доступ к отчету по основным сбоям можно получить с домашней страницы отчетов по мониторингу. Щелкните показатель сеансов учтено вы перейдете к [Отчету распределения сбоев в Скайп для Business Server](failure-distribution-report.md).
  
## <a name="making-the-best-use-of-the-top-failures-report"></a>Рекомендации по использованию отчета по основным сбоям

Отчет по основным сбоям имеет одну необычную особенность: он позволяет одновременно выполнять фильтрацию по 5 кодам диагностики. (Обычно вы может фильтровать только один элемент — например, адрес SIP один пользователь - за раз.) Чтобы отфильтровать несколько кодов диагностики, просто введите каждый идентификатор в окне диагностические идентификаторы, отделяя идентификаторы разделяются запятыми. (Если вы хотите, можно оставить пробел после каждого их запятыми.) Например:
  
1011, 2412, 1033, 52116, 1008
  
Сделайте это, и будут отображаться только неудачные звонки, в которых возник один из данных кодов диагностики.
  
Если навести указатель на код диагностики, отображается подсказка с описание его значения. Например, если навести указатель на код ответа 486, отображается следующее сообщение:
  
Busy Here (Занято).
  
## <a name="filters"></a>Фильтры

Фильтры позволяют получать более адресный набор данных или просматривать полученные данные разными способами. Например, в отчете по основным сбоям можно фильтровать возвращенные данные по типу действия (одноранговый сеанс или сеанс конференц-связи) или по коду ответа SIP, выданному для неудачного сеанса. Кроме того, можно выбрать способ группирования данных. В данном случае вызовы использования по часу, дню, неделе или месяцу.
  
В следующей таблице приведены фильтры, которые можно использовать с отчетом по основным сбоям.
  
**Фильтры отчета по основным сбоям**

|**Имя**.|**Описание**|
|:-----|:-----|
|**С** <br/> |Начальные дата/время для временного диапазона. Чтобы просмотреть данные по часам, введите начальные дату и время следующим образом:  <br/> 7/7/2015 13:00  <br/> Если вы не вводите начальное время, отчет автоматически начинается с 12:00 AM указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Чтобы просмотреть данные за неделю или месяц, введите дату, выпадающую на любое время в рамках недели или месяца, которые вы хотите просмотреть (вам не требуется вводить первый день недели или месяца):  <br/> 7/3/2015  <br/> Недели всегда отсчитываются с воскресенья по субботу.  <br/> |
|**По** <br/> |Конечные дата/время для временного диапазона. Чтобы просмотреть данные по часам, введите конечные дату и время следующим образом:  <br/> 7/7/2015 13:00  <br/> Если вы не вводите конечное время, отчет автоматически заканчивается в 12:00 AM указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Чтобы просмотреть данные за неделю или месяц, введите дату, выпадающую на любое время в рамках недели или месяца, которые вы хотите просмотреть (вам не требуется вводить первый день недели или месяца):  <br/> 7/3/2015  <br/> Недели всегда отсчитываются с воскресенья по субботу.  <br/> |

|**Modality (Модальность)** <br/> |В настоящее время доступен только параметр **[All]** (Все).  <br/> |
|**Pool (Пул)** <br/> |Полное доменное имя пула регистратора или пограничного сервера. Можно выбрать отдельный пул или нажать **[Все]**, чтобы просмотреть данные для всех пулов. Этот раскрывающийся список автоматически заполняется на основе записей в базе данных.<br/> |
|**Категория** <br/> | Тип возникшего сбоя. Выберите одно из следующих действий: <br/>  Как ожидаемый сбой, так и непредвиденный сбой <br/>  Unexpected failure (Неожиданный сбой) <br/>  «Ожидаемый сбой» – это ошибка, возникновение которой было предсказуемым. Например, вызов не должен прерываться, если вызывающий абонент помещается на удержание. В этом случае это считается непредвиденным сбоем. <br/> |

Заголовки диагностики не являются обязательными (можно иметь сеансы SIP без таких заголовков), а идентификаторы диагностики указываются только для сеансов, в которых возникли какие-либо проблемы.  <br/> |
