# <a name="top-failures-report-in-skype-for-business-server"></a>Отчет о главных сбоях в Skype для бизнеса Server
 
**Сводка:** Узнайте о отчете о главных сбоях в Skype для бизнеса Server.
  
Отчет по основным сбоям позволяет узнать о наиболее частых сбоях и характере их изменения с течением времени. Сбои основаны на сочетании двух следующих метрик:
  
- **Код диагностики**. Уникальный идентификатор (в виде заголовка ms-diagnostics), прикрепленный к сообщению SIP. Коды диагностики предоставляют информацию, полезную при устранении проблем со звонками.
    
- **Код ответа**. Коды ответа используются в сеансах связи SIP для ответы на запросы SIP. Предположим, что пользователь Ken отправляет запрос INVITE пользователю Pilar Ackerman (например, пользователь Ken Myer звонит пользователю Pilar Ackerman). Если Pilar отвечает, ее телефон отправляет код ответа 200 (OK), позволяя телефону Ken узнать об ответе Pilar. Отчет о главных сбоях включает только коды ответа, отправленные в ответ на сбой вызова; Skype для бизнеса Server отслеживает не все коды ответа, выданные во время вызова.
    
Приводится информация не только об общем числе сеансов, в которых возник сбой, но и об общем числе пользователей, столкнувшихся с этим сбоем.
  
## <a name="accessing-the-top-failures-report"></a>Доступ к отчету по основным сбоям

Доступ к отчету по основным сбоям можно получить с домашней страницы отчетов по мониторингу. Щелкнув метрику Отчет о сеансах, вы сможете найти отчет о рассылке [отказов в Skype для бизнеса Server](failure-distribution-report.md).
  
## <a name="making-the-best-use-of-the-top-failures-report"></a>Рекомендации по использованию отчета по основным сбоям

Отчет по основным сбоям имеет одну необычную особенность: он позволяет одновременно выполнять фильтрацию по 5 кодам диагностики. (Как правило, одновременно можно фильтровать только один элемент , например один SIP-адрес пользователя.) Чтобы фильтровать несколько диагностических ИД, просто введите каждый ID в поле Диагностические ID, разделив их с помощью запятых. (Если вы хотите, вы можете оставить пустое пространство после каждой запятой.) Например:
  
1011, 2412, 1033, 52116, 1008
  
Сделайте это, и будут отображаться только неудачные звонки, в которых возник один из данных кодов диагностики.
  
Если навести указатель на код диагностики, отображается подсказка с описание его значения. Например, если навести указатель на код ответа 486, отображается следующее сообщение:
  
Busy Here (Занято).
  
## <a name="filters"></a>Фильтры

Фильтры позволяют получать более адресный набор данных или просматривать полученные данные разными способами. Например, в отчете по основным сбоям можно фильтровать возвращенные данные по типу действия (одноранговый сеанс или сеанс конференц-связи) или по коду ответа SIP, выданному для неудачного сеанса. Кроме того, можно выбрать способ группирования данных. В данном случае вызовы использования по часу, дню, неделе или месяцу.
  
В следующей таблице приведены фильтры, которые можно использовать с отчетом по основным сбоям.
  
**Фильтры отчета по основным сбоям**

|**Название**|**Описание**|
|:-----|:-----|
|**From** <br/> |Дата и время начала диапазона. Чтобы просмотреть данные по часам, введите дату и время начала в следующем формате:  <br/> 7/7/2015 13:00  <br/> Если не указать время начала, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Неделя всегда начинается с воскресения и заканчивается субботой.  <br/> |
|**To** <br/> |Дата и время окончания диапазона. Чтобы просмотреть данные по часам, введите дату и время окончания в следующем формате:  <br/> 7/7/2015 13:00  <br/> Если не указать время окончания, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Неделя всегда начинается в воскресенье и заканчивается в субботу.  <br/> |
|**Activity type (Тип действия)** <br/> | Тип действия. Выберите одно из следующих действий: <br/>  [Все] <br/>  "Peer-to-peer" (Одноранговый сеанс); <br/>  Conference <br/> |
|**Модальность** <br/> |В настоящее время доступен только параметр **[All]** (Все).  <br/> |
|**Pool** <br/> |Полное доменное имя пула регистратора или пограничного сервера. Можно выбрать отдельный пул или нажать **[All] ([Все])**, чтобы просмотреть данные для всех пулов. Этот раскрывающийся список автоматически заполняется на основе записей в базе данных.<br/> |
|**Категория** <br/> | Тип возникшего сбоя. Выберите одно из следующих действий: <br/>  Как ожидаемый сбой, так и непредвиденный сбой <br/>  Непредвиденный сбой <br/>  "Ожидаемый сбой" — это ошибка, возникновение которой было предсказуемым. Например, если пользователь установил статус "Не беспокоить", то ожидаемое, что все вызовы этого пользователя завершатся неудачно. "Непредвиденный сбой" — это ошибка, которая происходит в работоспособной системе. Например, вызов не должен прерываться, если вызывающий абонент помещается на удержание. В этом случае это считается непредвиденным сбоем. <br/> |
|**Response code** (Код ответа) <br/> |Код ответа SIP, отправленный при сбое конференции. Введите полный код ответа, например:  <br/> 400  <br/> |
|**Diagnostic ID** (ИД диагностики) <br/> |Уникальный идентификатор (в форме заголовка ms-diagnostics), присоединенный к сообщению SIP, который часто содержит сведения, помогающий устранять ошибки. Диагностические заголовки являются необязательными (можно иметь сеансы SIP, не включающие эти заголовки), и диагностические идентификаторы сообщаются только для тех сеансов, которые испытывали какие-либо проблемы.  <br/> |
