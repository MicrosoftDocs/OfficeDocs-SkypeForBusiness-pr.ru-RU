# <a name="server-performance-report-in-skype-for-business-server"></a>Отчет о производительности сервера в Skype для бизнеса Server
 
**Сводка:** Узнайте о отчете о производительности сервера в Skype для бизнеса Server.
  
Отчет о производительности сервера содержит список Skype для бизнеса Server серверов, на которые был высок процент ненадежаний звонков. В отчете сервера распределены по типам, для каждого из них дается статистика следующих видов:
  
- Сервер-посредник
    
- A/V Conferencing Server (Сервер аудио- и видеоконференций);
    
- A/V Edge Server (Пограничный сервер аудио- и видеоданных);
    
- Шлюз (сервер-посредник)
    
- Шлюз (в обход сервера-посредника)
    
- Видео (включая показатели видео для серверов аудио и видеоконференций и соответствующих пограничных серверов)
    
- Общий доступ к приложениям (включая показатели общего доступа для серверов аудио и видеоконференций и соответствующих пограничных серверов)
    
Важно отметить, что рейтинг, показанный в этом отчете в качестве относительного ранжирования. Например, предположим что вашем наименее производительном сервере бывает один плохой звонок из 1 000. Это более чем приемлемый показатель в 0,1%. Но данный сервер (при учете того, что у других серверов этот процент еще меньше) все равно будет приведет в отчет о производительности серверов.
  
## <a name="accessing-the-server-performance-report"></a>Получение доступа к отчету о производительности серверов

Доступ к отчету можно получить с главный страницы отчетов по мониторингу. Вы можете сверлить до отчета списка вызовов в Skype для бизнеса Server, [щелкнув](call-list-report-0.md) любой из следующих метрик:
  
- Объем звонков
    
- Процент плохих звонков
    
Также можно детализировать отчет о тренде качестве среды передачи данных сервера, щелкнув один из следующих показателей:
  
- ТЕНДЕНЦИЯ
    
## <a name="making-the-best-use-of-the-server-performance-report"></a>Наилучшее использование отчета о производительности сервера

В отчете есть несколько фильтров данных. Например, можно фильтровать по типу сети (звонки по проводным и беспроводным соединениям) и типу доступа (звонки из-за пределов и изнутри брандмауэра). Рекомендуется использовать данные фильтры при просмотре отчета. Например, предположим что имеется сервер-посредник с процентом плохих звонков 3.24%. Если рассмотреть процент плохих беспроводных звонков, то процент может достигать значения 20%. Это значит, что на данном сервере возникают проблемы при обслуживании беспроводных звонков, эти сведения были скрыты, так как проблем при обслуживании проводных соединений не было.

В данном случае данные группируются по часам, дню, неделе и месяцу.
