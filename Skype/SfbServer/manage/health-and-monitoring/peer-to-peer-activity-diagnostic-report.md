# <a name="peer-to-peer-activity-diagnostic-report-in-skype-for-business-server"></a>Диагностический отчет по одноранговой активности в Skype для бизнеса Server
 
**Сводка:** Узнайте о диагностическом отчете по одноранговой активности в Skype для бизнеса Server.
  
Диагностический отчет об одноранговом обмене данными предоставляет сведения об успешных и неудавшихся сеансах однорангового обмена данными. Обратите внимание, что Skype для бизнеса Server различает различные типы сбоев:
  
- **Ожидаемый отказ**. Под ожидаемым отказом обычно понимается отказ в сугубо техническом смысле. Например, предположим, что вы звоните кому-то, но этого человека нет на месте, и он не может ответить на звонок. Поскольку на звонок не был получен ответ, такая ситуация технически считается отказом. С другой стороны, это был ожидаемый сбой: Skype для бизнеса Server не ожидает, что вы ответите на звонок, если недоступны для ответа на звонок. Аналогичным образом, ожидаемый отказ произойдет в том случае, если вы попытаетесь отправить мгновенное сообщение пользователю, которого нет в сети или который работает с телефона, не поддерживающего обмен мгновенными сообщениями.
    
- **Неожиданный сбой.** Непредвиденное сообщение об ошибке — это именно то, что подразумевает название: ошибка, которая в зависимости от обстоятельств не предполагается. Например, предположим, что вы звоните кому-то и можете ответить на него; Однако, когда Skype для бизнеса Server пытается перенаправить вызов на голосовую почту, вызов не удастся из-за потери подключения к единой системы обмена сообщениями Exchange. Это непредвиденное сообщение об ошибке: предполагается, что вызовы всегда могут быть перена отправлены в голосовую почту. Как правило, неожиданные сбои — это истинные сбои: это проблемы, которые, скорее всего, не могут быть улажены с помощью обучения пользователей или аналогичных мер.
    
Обратите внимание на то, что число успешных сеансов, ожидаемых отказов и непредвиденных отказов может быть не равно в сумме общему числу сеансов. Например, имеются следующие значения.
  
|**Успешные операции**|**Число ожидаемых сбоев**|**Число неожиданных сбоев**|**Всего сеансов**|
|:-----|:-----|:-----|:-----|
|2024  <br/> |469  <br/> |16   <br/> |2521  <br/> |
   
Если сложить 2024 + 469 + 16, то получится 2509 сеансов, но в столбце "Всего сеансов" показано 2521 сеансов. "Недостающие" 12 сеансов — это сеансы, которые система не смогла определить как успешные или неудавшиеся. Иногда это происходит, когда сторонний продукт вводит новый диагностический код, незнакомый Skype для бизнеса Server. В этом случае звонки, совершенные с помощью этого продукта и зарегистрированные с помощью его кода диагностики, не всегда могут быть отнесены к успешным, ожидаемым отказам или непредвиденным отказам.
  
## <a name="accessing-the-peer-to-peer-activity-diagnostic-report"></a>Доступ к диагностическому отчету об одноранговом обмене данными

Доступ к диагностическому отчету об одноранговом обмене данными можно получить с домашней страницы "Отчеты наблюдения". Вы можете получить доступ к [отчету распределения](failure-distribution-report.md) сбоев в Skype для бизнеса Server, щелкнув один из следующих показателей:
  
- Unexpected failure volume (Неожиданный сбой, объем)
    
- Количество ожидаемых отказов
    
## <a name="making-the-best-use-of-the-peer-to-peer-activity-diagnostic-report"></a>Рекомендации по использованию диагностического отчета об одноранговом обмене данными

Имеется ряд способов фильтрации диагностического отчета об одноранговом обмене данными, но по умолчанию соответствующие параметры скрыты. Чтобы просмотреть доступные параметры фильтрации, нажмите кнопку "Показать или скрыть параметры" в правом верхнем углу окна отчета. Параметры фильтрации станут доступны для использования.
  
## <a name="filters"></a>Фильтры

Фильтры позволяют получить более конкретные наборы возвращаемых данных. Например, диагностический отчет об одноранговом обмене данными позволяет фильтровать данные по таким признакам, как модальность сеанса (например, обмен мгновенными сообщениями, передача файлов или общий доступ к приложениям). Вы также можете выбрать тип группировки данных. В этом случае звонки группируются по часам, дням или неделям.
  
В следующем списке перечислены фильтры, которые можно использовать с диагностическим отчетом об одноранговом обмене данными.
  
**Фильтры диагностического отчета об одноранговом обмене данными**

|**Название**|**Описание**|
|:-----|:-----|
|**From** <br/> |Дата и время начала диапазона. Чтобы просмотреть данные по часам, введите дату и время начала в следующем формате:  <br/> 7.07.2015 13:00  <br/> Если не указать время начала, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Неделя всегда начинается с воскресения и заканчивается субботой.  <br/> |
|**Для** <br/> |Дата и время окончания диапазона. Чтобы просмотреть данные по часам, введите дату и время окончания в следующем формате:  <br/> 7.07.2015 13:00  <br/> Если не указать время окончания, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Неделя всегда начинается с воскресения и заканчивается субботой.  <br/> |
|**Interval** <br/> | Временной интервал. Выберите один из следующих вариантов: <br/>  Ежечасно (может отображаться до 25 часов) <br/>  Ежедневно (может отображаться до 31 дня) <br/>  Еженедельно (может отображаться до 12 недель) <br/>  Ежемесячно (может отображаться до 12 месяцев) <br/>  Если указаны такие даты начала и конца, что превышается максимально разрешенное количество значений для выбранного интервала, то отображается только максимально допустимое количество значений (начиная с даты начала). Например, при выборе интервала "Ежедневно" с датой начала 7.07.2015 и датой окончания 28.02.2015 отображаются данные за 07.08.2015 с 12:00 до 07.09.2015 12:00 (то есть 31 день). <br/> |
|**Pool** <br/> |Полное доменное имя пула регистратора или пограничного сервера. Вы можете выбрать отдельный пул или щелкнуть **[Все]**, чтобы просмотреть данные по всем пулам. Этот раскрывающийся список заполняется автоматически в соответствии с записями в базе данных.<br/> |
|**Модальность** <br/> | Указывает на тип взаимодействия. Выберите один из следующих вариантов: <br/>  [Все] <br/>  Обмен мгновенными сообщениями <br/>  Передача файлов <br/>  Совместное использование приложений <br/>  "Audio" (Аудио); <br/>  "Video" (Видео); <br/> |
   
## <a name="metrics-per-modality"></a>Показатели (для каждой модальности)

В следующей таблице показаны сведения, содержащиеся в диагностическом отчете об одноранговом обмене данными для каждой модальности.
  
**Показатели (для каждой модальности)**

|**Название**|**Возможность сортировки по этому показателю**|**Описание**|
|:-----|:-----|:-----|
|**Success volume** (Успех, объем) <br/> |Нет  <br/> |Общее количество успешных одноранговых сеансов.  <br/> |
|**Процент успешных сеансов** <br/> |Нет  <br/> |Процент одноранговых сеансов, которые завершились без серьезных проблем. Вычисляется путем деления количества успешных сеансов на общее их количество.  <br/> |
|**Количество ожидаемых отказов** <br/> |Нет  <br/> |Общее число сеансов, в рамках которых произошли ожидаемые отказы.  <br/> Ожидаемый отказ — это отказ, который можно спрогнозировать. Например, если пользователь установил статус "Не беспокоить", любой звонок этому пользователю должен завершаться отказом.  <br/> |
|**Expected failure percentage** (Ожидаемый сбой, процент) <br/> |Нет  <br/> |Процент одноранговых сеансов, в рамках которых произошли ожидаемые отказы. Вычисляется путем деления количества ожидаемых отказов на общее количество сеансов.  <br/> |
|**Количество непредвиденных отказов** <br/> |Нет  <br/> |Общее число сеансов, в рамках которых произошли непредвиденные отказы.  <br/> Непредвиденный отказ — это отказ, который происходит в ситуации, в которой он происходить не должен. Например, звонок не должен прерываться, если он ставится на удержание. Такое событие помечается как непредвиденный отказ.  <br/> |
|**Unexpected failure percentage** (Неожиданный сбой, процент) <br/> |Нет  <br/> |Процент одноранговых сеансов, в рамках которых произошли непредвиденные отказы. Вычисляется путем деления количества непредвиденных отказов на общее количество сеансов.  <br/> |
|**Всего сеансов** <br/> |Нет  <br/> |Общее число сеансов, включая успешные сеансы, неудавшиеся сеансы (вследствие как ожидаемых, так и непредвиденных отказов) и сеансы, не отнесенные к определенной категории.  <br/> |
