# <a name="monitor-for-server-memory-capacity-limits-in-skype-for-business-server"></a>Мониторинг ограничений емкости памяти сервера в Skype для бизнеса Server
 
**Сводка:** Узнайте, как отслеживать ограничения емкости памяти сервера в Skype для бизнеса Server.
  
> [!CAUTION]
> Сведения, относящиеся к планированию емкости, относятся только к клиентам Lync 2010 Mobile и службе мобильности (Mcx). Планирование емкости для веб-API единой связи (UCWA), используемого мобильными клиентами Lync 2013, обеспечивается средством планирования Lync Server 2013. 

> [!NOTE]
> Поддержка mcX (Mobility Service) для устаревших мобильных клиентов больше недоступна в Skype для бизнеса Server 2019 г. Все современные Skype для бизнеса мобильные клиенты уже используют веб-API единой связи (UCWA) для поддержки обмена мгновенными сообщениями,присутствия и контактов. Пользователям с устаревшими клиентами, использующими MCX, потребуется обновиться до текущего клиента.
  
Два счетчика производительности мобильности помогут вам определить текущее использование и спланировать емкость для службы мобильности Skype для бизнеса Server (Mcx), а также отслеживать использование памяти для UCWA. Для UCWA встречная категория **— LS:WEB — UCWA**. Для службы мобильности (Mcx) счетчики находятся под категорией **LS:WEB — служба мобильной связи**. Счетчики для мониторинга:
  
- В настоящее время active **Session Count** с подписками на активное присутствие, которое является текущим числом конечных точек, зарегистрированных через UCWA или Службу мобильности (Mcx), которые имеют подписки на активное присутствие (число всегда подключенных мобильных пользователей)
    
- **В настоящее время** количество активных сеансов , которое является текущим числом конечных точек, зарегистрированных через UCWA или службу мобильности
    
Если разница между текущим количеством активных сеансов с подписками  на активное присутствие и текущим количеством активных сеансов со временем невелика, это означает, что большинство пользователей мобильных устройств имеют всегда подключенное устройство, например мобильное устройство Android или Nokia (только для Mcx). Всегда подключенные устройства UCWA включают устройства Apple и Android с мобильными клиентами Lync 2013. Если  в настоящее время количество активных сеансов значительно превышает количество активных сеансов с подписками на активное **присутствие, это** указывает на то, что все больше пользователей используют фоновое конечное устройство, например устройство Apple iOS или Windows Phone под Mcx. (Windows Phone является единственным мобильным клиентом Lync 2013, который будет регистрироваться в этом качестве).
  
Необходимо установить ограничение для текущего подсчета активных сеансов с подписками на активное  присутствие и счетчиков производительности в настоящее время на основе ожидаемого использования, результатов планирования емкости и текущего мониторинга службы мобильности и других счетчиков переднего конечного сервера. Установленные ограничения должны позволить вам оценить емкость сервера и повысить уровень оповещений при превышении емкости.
  
Чтобы определить соответствующие ограничения, сначала необходимо определить, сколько памяти доступно на переднем сервере службы мобильности. Отслеживайте счетчики, чтобы определить, когда необходимо планировать дополнительную емкость по следующей формуле:
  
Общая память, используемая службой мобильности Mcx (MB) = 164 + (400 + 134) / 1024 *** В** настоящее время активное количество сеансов с подписками на активное присутствие + 400 / 1024 *** (** -  В настоящее время активная сессия **CountCurrently Active Session Count with Active Presence Subscriptions**)
  
> [!IMPORTANT]
> Калькулятор емкости Microsoft Lync Server 2010 — это таблица, которая предварительно заселяется всеми формулами, которые позволяют планировщику определять требования к Skype для бизнеса серверам, включая ЦП, память и жесткий диск. Вы можете [скачать таблицу и связанный документ](https://go.microsoft.com/fwlink/p/?LinkID=212657). 
  
Переднему серверу требуется достаточное количество доступной памяти для поддержки службы мобильности в ситуациях с сбойом. Вы можете отслеживать текущую доступную память на переднем конечном сервере с помощью счетчика **Memory\Available Mbytes** или с помощью уравнения, упомянутого ранее, чтобы спланировать объем памяти, который, как ожидается, будет использовать служба мобильности.
  
Если объем памяти на переднем конечном сервере меньше 1500 МБ при планировании ожидаемого числа пользователей мобильности, необходимо добавить больше оборудования для поддержки службы мобильности. Дополнительные сведения см. в материале [Monitor mobility for performance in Skype для бизнеса Server](monitor-mobility-performance.md) документации по операциям.
