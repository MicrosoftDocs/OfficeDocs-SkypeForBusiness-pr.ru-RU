# <a name="media-quality-comparison-report-in-skype-for-business-server"></a>Отчет о сравнении качества мультимедиа в Skype для бизнеса Server
 
**Сводка:** Узнайте о отчете о сравнении качества мультимедиа в Skype для бизнеса Server.
  
Сравнительный отчет качества мультимедиа-данных позволяет сравнить значения качества звонков для различных типов аудиозвонков (например, звонков, выполненных через беспроводные сети и звонков, выполненных через проводное соединение).
  
## <a name="accessing-the-media-quality-comparison-report"></a>Доступ к сравнительному отчету качества мультимедиа-данных

Сравнительный отчет качества мультимедиа-данных доступен на домашней странице отчетов мониторинга. 
  
## <a name="filters"></a>Фильтры

Фильтры предоставляют способ получения более точно настроенного набора данных или просмотра возвращенных данных различными способами. В следующей таблице перечислены фильтры, которые можно использовать в сравнительном отчете качества мультимедиа-данных.
  
**Фильтры сравнительного отчета качества мультимедиа-данных**

|**Название**|**Описание**|
|:-----|:-----|
|**From** <br/> |Дата и время начала диапазона. Чтобы просмотреть данные по часам, введите дату и время начала в следующем формате:  <br/> 7/7/2015 13:00  <br/> Если не указать время начала, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Неделя всегда начинается с воскресения и заканчивается субботой.  <br/> |
|**To** <br/> |Дата и время окончания диапазона. Чтобы просмотреть данные по часам, введите дату и время окончания в следующем формате:  <br/> 7/7/2015 13:00  <br/> Если не указать время окончания, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Недели всегда начинаются с Воскресенья и заканчиваются в Субботу.  <br/> |
|**Звонки** <br/> | Тип звонка, который будет использоваться в качестве основного элемента для сравнения. Допускаются следующие значения: <br/>  [Все] <br/>  Внешний <br/>  Внутренний <br/>  VPN <br/>  Non-VPN <br/>  Wired <br/>  Беспроводная связь <br/>  Внешние и проводные <br/>  Внешние и беспроводные <br/>  Внешние и VPN <br/>  Внешние и не VPN <br/>  Внутренние и проводные <br/>  Внутренние и беспроводные <br/> |
|**Сравнить со звонками** <br/> | Тип звонка, который будет использоваться в качестве второго элемента для сравнения. Допускаются следующие значения: <br/>  [Все] <br/>  Внешний <br/>  Внутренний <br/>  VPN <br/>  Non-VPN <br/>  Wired <br/>  Беспроводная связь <br/>  Внешние и проводные <br/>  Внешние и беспроводные <br/>  Внешние и VPN <br/>  Внешние и не VPN <br/>  Внутренние и проводные <br/>  Внутренние и беспроводные <br/> |
|**Interval** <br/> | Временной интервал. Выберите один из следующих вариантов: <br/>  Ежечасно (может отображаться до 25 часов) <br/>  Ежедневно (может отображаться до 31 дня) <br/>  Еженедельно (может отображаться до 12 недель) <br/>  Если указаны такие даты начала и конца, что превышается максимально разрешенное количество значений для выбранного интервала, то отображается только максимально допустимое количество значений (начиная с даты начала). Например, если вы выбираете дневной интервал с датой начала 7/7/2015 и датой окончания 2/28/2015, данные отображаются за 8/7/7/2015 с 12:00 до 09:00 12:00 (то есть в общей сложности 31 день). <br/> |
   
## <a name="metrics"></a>Метрики

В следующей таблице приведены сведения, предоставляемые сравнительным отчетом качества мультимедиа-данных
  
**Метрики сравнительного отчета качества мультимедиа-данных**

|**Название**|**Возможность сортировки по этому показателю**|**Описание**|
|:-----|:-----|:-----|
|**Call volume (Интенсивность вызовов)** <br/> |Нет  <br/> |Общее количество вызовов.  <br/> |
|**Degradation (MOS) (Снижение (MOS))** <br/> |Нет  <br/> |Среднее количество деградации MOS (средняя оценка мнений), которая наблюдается во время вызова. Значение снижения может быть в диапазоне от 0.0 до 5.0; значение не выше 0.5 является приемлемым уровнем снижения. Исторические показатели оценки мнений вычислялись путем оценки пользователями качества вызова по шкале от 1 до 5. Skype для бизнеса Server использует набор алгоритмов для прогнозирования того, как пользователи оценили бы вызов.  <br/> Высокие значения снижения могут быть вызваны перегруженностью, нехваткой пропускной способности, перегруженностью или помехами в беспроводной сети, а также перегруженностью сервера мультимедиа или конечной точки. Высокие значения могут привести к искажению или потере звука.  <br/> |
|**Poor call percentage (Процент звонков низкого качества)** <br/> |Нет  <br/> |Общее количество звонков, классифицированных как звонки низкого качества. Звонок низкого качества — это любой звонок, хотя бы одна метрика которого превышает допустимое значение (например, звонок с чрезмерным дрожанием).  <br/> |
|**Round trip (ms) (Круговой путь (мс))** <br/> |Нет  <br/> |Среднее время (в миллисекундах), требуемое для переноса пакета протокола RTP на другую конечную точку и затем обратно. Время кругового пути в 200 миллисекунд или меньше считается приемлемым качеством.  <br/> Высокие значения времени кругового пути могут быть вызваны международной маршрутизацией звонков; неправильной настройкой кругового пути; или перегрузкой сервера мультимедиа. Высокие значения времени кругового пути приводят к сложностям в двухсторонних аудиобеседах в реальном времени.  <br/> |
|**Packet loss (Потеря пакетов)** <br/> |Нет  <br/> |Средняя норма потери пакетов транспортного протокола в режиме реального времени (пакетов протокола RTP). (Потеря пакетов происходит, когда пакеты протокола RTP, который используется для передачи звука и видео в Интернете, не достигают места своего назначения.) Высокие уровни потерь обычно бывают следствием перегруженности, нехватки пропускной способности, перегруженности или помех в беспроводной сети, а также перегруженности сервера мультимедиа. Потеря пакетов обычно приводит к искажению или потере звука..  <br/> |
|**Дрожание (мс)** <br/> |Нет  <br/> |Средний уровень дрожания, определенный между прибытием пакетов протокола RTP. (Дрожание — это показатель "дрожания" звонка.) Высокие значения дрожания обычно вызываются перегруженностью или высокой загрузкой сервера мультимедиа и приводят к искажению или потере звука.  <br/> |
|**Healer concealed ratio (Коэффициент скрытых сэмплов)** <br/> |Нет  <br/> |Среднее отношение скрытых звуковых сэмплов к общему количеству сэмплов. (Скрытые звуковые сэмплы — это техника, которая используется для сглаживания резких переходов, которые обычно вызываются потерей сетевых пакетов.) Высокие значения указывают на значительные уровни применения скрытия потерь, вызванных потерей пакетов или дрожанием, что может привести к искажению или потере звука.  <br/> |
|**Healer stretched ratio (Коэффициент растянутых сэмплов)** <br/> |Нет  <br/> |Среднее отношение растянутых звуковых сэмплов к общему количеству сэмплов. (Растянутые звуковые сэмплы — это звук, который растягивается для поддержания качества звонка, когда обнаруживается потерянный сетевой пакет.) Высокие значения указывают на значительные уровни растягивания сэмплов, вызванного дрожанием, что может привести к искажению или потере звука.  <br/> |
|**Healer compressed ratio (Коэффициент сжатых сэмплов)** <br/> |Нет  <br/> |Среднее отношение сжатых звуковых сэмплов к общему количеству сэмплов. (Сжатые звуковые сэмплы — это звук, который сжимается для поддержания качества звонка, когда обнаруживается потерянный сетевой пакет.) Высокие значения указывают на значительные уровни сжатия сэмплов, вызванного дрожанием, что может привести к искажению или потере звука.  <br/> |
