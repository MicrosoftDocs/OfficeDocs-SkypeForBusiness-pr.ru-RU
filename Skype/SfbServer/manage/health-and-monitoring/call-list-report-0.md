# <a name="call-list-report-in-skype-for-business-server"></a>Отчет о списке вызовов в Skype для бизнеса Server
 
**Сводка:** Узнайте о отчете о списке вызовов, используемом в Skype для бизнеса Server.
  
Отчет по списку обзвона предоставляет параметры качества взаимодействия (QoE) для отдельных вызовов, выполненных и принятых сотрудниками организации. Обратите внимание, что действительные параметры, содержащиеся в отчете, зависят от способа доступа к отчету по списку обзвона. Например, если открыть отчет из отчета устройства в [Skype для бизнеса Server, вы](device-report.md) увидите такие показатели, как следующие метрики, которые также сообщаются в отчете о устройстве:
  
- Микрофон вызывающего абонента
    
- Динамик вызывающего абонента
    
- Микрофон вызываемого абонента
    
- Динамик вызываемого абонента
    
- Соотношение времени голосовой коммутации 
    
Однако, если открыть отчет списка вызовов из отчета о расположении в Skype для бизнеса Server, вы не увидите ни [одного](location-report.md) из этих показателей; вместо этого вы увидите такие показатели:
  
- Round trip (ms)
(Круговой путь (мс))
    
- Degradation (MOS) (Снижение (MOS))
    
- Packet loss (Потеря пакетов)
    
- Колебание (мс)
    
Это показатели, содержащиеся в отчете о местоположении. Однако вы всегда можете щелкнуть «Подробные показатели» в отчете по списку обзвона, чтобы получить полную информацию QoE для любого вызова.
  
## <a name="accessing-the-call-list-report"></a>Доступ к отчету по списку обзвона

Доступ к отчету по списку обзвона можно получить из любого из следующих отчетов:
  
- Отчет [о расположении в Skype для бизнеса Server](location-report.md) (щелкнув метрию "Объем вызовов" или "Плохой процент зова").
    
- Отчет [устройства в Skype для бизнеса Server](device-report.md) (щелкнув метрию "Объем вызовов" или "Плохой процент вызовов").
    
- [Сводный отчет](summary.md) о качестве мультимедиа в Skype для бизнеса Server (щелкнув метрию "Объем зова" или "Плохой процент вызовов").
    
- Отчет [о производительности сервера в Skype для бизнеса Server](server-performance.md) (щелкнув метрию "Объем вызовов" или "Плохой процент вызовов").
    
В отчете о списке вызовов вы можете получить доступ к отчету о подробностях вызовов в Skype для бизнеса Server, [нажав](call-detail-report.md) метрику Detail.
  
## <a name="making-the-best-use-of-the-call-list-report"></a>Эффективное использование отчета по списку обзвона

Если вы не помните, для чего именно используются некоторые показатели отчета по списку обзвона (например, «Соотношение времени голосовой коммутации»), наведите указатель мыши на метку показателя; отобразится всплывающая подсказка, содержащая краткое описание показателя.
  
## <a name="filters"></a>Фильтры

Нет. Фильтрация отчета по списку обзвона невозможна.

**Показатели в отчете по списку обзвона**

|:-----|:-----|:-----|

Приемлемым считается время двусторонней передачи, равное 200 мс (или менее).  <br/> Высокие значения кругового пути могут быть следствием маршрутизации международных вызовов, неправильной настройки маршрутизации или перегруженности сервера мультимедиа. |**Деградация (MOS)** <br/> |Да  <br/> |Средний объем деградации в соответствии со средней экспертной оценкой разборчивости речи (MOS), наблюдаемый в ходе вызова. Значение снижения может быть в диапазоне от 0.0 до 5.0. Значение не выше 0.5 является приемлемым уровнем снижения. Традиционно средние экспертные оценки разборчивости речи вычислялись по оценкам пользователей качества вызова по шкале от 1 до 5. В Skype для бизнеса Server набор алгоритмов предсказывает, как пользователи оценили бы вызов.  <br/> Высокие значения снижения могут быть вызваны перегруженностью, нехваткой пропускной способности, перегруженность или помехами в беспроводной сети, а также перегруженностью сервера мультимедиа или конечной точки. Высокие значения могут привести к искажению или потере звука.  <br/> |

(Потеря пакетов происходит, когда пакеты RTP (протокол, используемый для передачи видео- и аудиокомпонентов через Интернет) не достигают места назначения.) Высокие значения потери, как правило, обусловлены перегрузкой, недостаточной пропускной способностью, помехами или перегрузкой беспроводной сети либо перегрузкой сервера-посредника. Потеря пакетов обычно приводит к искажению или потере звук.  <br/> |

|**Соотношение скрытых фрагментов в процессе восстановления** <br/> |Да  <br/> |Среднее отношение количества скрытых аудиофрагментов к общему количеству фрагментов. Высокие значения указывают на высокие уровни скрытия потерь, которые были обусловлены потерей пакетов или колебаниями. Эти высокие значения приводят к искажению звука или потере аудиосигналов.  <br/> |
|**Соотношение растянутых фрагментов в процессе восстановления** <br/> |Да  <br/> |Среднее отношение количества растянутых аудиофрагментов к общему количеству фрагментов. (Растянутые аудиофрагменты — фрагменты, которые были увеличены для поддержания качества вызова при обнаружении пропуска сетевого пакета.) Высокие значения указывают на значительные уровни растягивания фрагментов, которые были обусловлены колебаниями. (Сжатые аудиофрагменты — фрагменты, которые были сжаты для поддержания качества вызова при обнаружении пропуска сетевого пакета.) Высокие значения указывают на значительные уровни сжатия фрагментов, которые были обусловлены колебаниями. Эти высокие значения приводят к искажению звука или к созданию эффекта ускоренного звучания.  <br/> |
