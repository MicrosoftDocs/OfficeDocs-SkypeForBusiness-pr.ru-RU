# <a name="mobility-performance-counters-in-skype-for-business-server"></a>Счетчики производительности мобильности в Скайп для Business Server
 
**Сводка:** Сведения о счетчиках производительности, которые можно использовать для наблюдения за серверами под управлением Unified Communications Web API (UCWA) и Скайп для службы Mobility Mcx Business Server.
  
В следующих таблицах перечислены имена и описания счетчиков производительности, которые можно использовать для наблюдения за серверами под управлением Unified Communications Web API (UCWA) и Скайп для службы Mobility Mcx Business Server. 
  
Имя категории счетчиков в таблице UCWA — **LS:WEB - UCWA**.
  
Имя категории для счетчиков в таблице службы Mcx Mobility Service — **LS:WEB — Mobile Communication Service**.

> [!NOTE]
> Поддержка MCX для устаревших мобильных клиентов больше не доступен в Скайп для Business Server 2019. Пользователям необходимо обновить до текущего клиента.
  
## <a name="performance-counters-for-ucwa"></a>Счетчики производительности для UCWA

|Счетчик|Описание|
|:-----|:-----|
|Active Application Count (Число активных приложений)  <br/> |Текущее число приложений  <br/> |
|Active Application Sharing Modality Count (Число активных модальностей общего доступа к приложениям)  <br/> |Текущее число модальностей общего доступа к приложениям  <br/> |
|Active Audio Modality Count (Число активных модальностей звука)  <br/> |Текущее число модальностей звука  <br/> |
|Active Data Collaboration Modality Count (Число активных модальностей совместной работы с данными)  <br/> |Текущее число модальностей совместной работы с данными  <br/> |
|Active Directory Photo Get Latency (ms) (Задержка получения фотографий из Active Directory (мс))  <br/> |Этот счетчик показывает среднее время (в миллисекундах), необходимое для получения фотографии из каталога Аctive Directory.  <br/> |
|Active Messaging Modality Count (Число активных модальностей обмена сообщениями)  <br/> |Текущее число модальностей обмена сообщениями  <br/> |
|Active Panoramic Video Modality Count (Число активных модальностей панорамного видео)  <br/> |Текущее число модальностей панорамного видео  <br/> |
|Active Pending Get Count (Число активных операций ожидания получения)  <br/> |Число активных операций ожидания получения; долгосрочные подключения к серверу  <br/> |
|Active Session Count (Число активных сеансов)  <br/> |Текущее число конечных точек, зарегистрированных в интерфейсе UCWA, на приложение и общее число  <br/> |
|Active User Instance Count (Число активных экземпляров пользователей)  <br/> |Текущее число экземпляров пользователей  <br/> |

|Active Video Modality Count (Число активных модальностей видео)  <br/> |Текущее число модальностей видео  <br/> |
|Application Creation Requests Received/Second (Получено запросов на создание приложения/с)  <br/> |Частота полученных запросов (в секунду) на создание приложения  <br/> |
|AS MCU Join Failures (Сбои присоединения MCU AS)  <br/> |Число неудачных попыток присоединения блоков MCU AS  <br/> |
|AV MCU Join Failures (Сбои присоединения MCU AV)  <br/> |Число неудачных попыток присоединения блоков MCU (AV)  <br/> |
|Average Application Startup Time (ms) (Среднее время загрузки приложения (мс))  <br/> |Среднее время загрузки приложения в миллисекундах  <br/> |
|Average Lifetime for Session (ms) (Среднее время существования сеанса (мс))  <br/> |Среднее время существования сеанса в миллисекундах (мс)  <br/> |
|Data MCU Join Failures (Сбои присоединения MCU данных)  <br/> |Число неудачных попыток присоединения блоков MCU данных  <br/> |
|Exchange Contact Search Latency (ms) (Задержка поиска контакта в Exchange (мс))  <br/> |Этот счетчик показывает среднее время (в миллисекундах), затрачиваемое на поиск контакта Exchange.  <br/> |
|Задержка получения фотографий в формате HD из Exchange (мс)  <br/> |Этот счетчик показывает среднее время (в миллисекундах), затрачиваемое на получение фотографии с сервера Exchange.  <br/> |
|Ответов HTTP 4xx/с  <br/> |Частота ответов с кодом HTTP 4xx (в секунду)  <br/> |
|Ответов HTTP 5xx/с  <br/> |Частота ответов с кодом HTTP 5xx (в секунду)  <br/> |
|IM MCU Join Failures (Сбои присоединения MCU IM)  <br/> |Число неудачных попыток присоединения блоков MCU IM  <br/> |
|Number of Active Directory Photo Get Failures (Число сбоев получения фотографий из Active Directory)  <br/> |Общее число неудачных попыток получения фотографий из Active Directory  <br/> |
|Number of Contact Search failures (Число сбоев поиска контактов)  <br/> |Общее число неудачных попыток поиска контактов в Exchange  <br/> |

|Over The Maximum Subscriptions Per Application (Превышение максимального числа подписок на приложение)  <br/> |Число запросов на подписку, превышающих максимальное значение для приложения  <br/> |
|Over The Maximum Subscriptions Per Batch (Превышение максимального числа подписок на пакет)  <br/> |Число запросов на подписку, превышающих максимальное значение для пакета  <br/> |
|Presence Subscription Failures (Сбои подписок отслеживания присутствия)  <br/> |Число неудачных попыток подписаться на сведения о присутствии  <br/> |
|Registering Endpoint Failures (Сбои регистрации конечных точек)  <br/> |Число сбоев регистрации конечных точек  <br/> |
|Requests Received/Second (Полученных запросов/с)  <br/> |Частота полученных запросов (в секунду)  <br/> |
|Requests Succeeded/Second (Успешных запросов/с)  <br/> |Частота успешных запросов (в секунду) (коды ответов HTTP 2xx/3xx)  <br/> |
|Succeeded Create Application Requests/Second (Успешных запросов на создание приложений/с)  <br/> |Частота успешных запросов (в секунду) на создание приложения  <br/> |
|Timed Out Pending Get Count (Число ожидающих операций получения с истекшим временем ожидания)  <br/> |Число ожидающих операций получения с истекшим временем ожидания  <br/> |
|Total Application Creation Requests Received (Всего полученных запросов на создание приложений)  <br/> |Общее число запросов на создание приложений, полученных с момента запуска службы  <br/> |
|Всего ответов HTTP 4xx  <br/> |Общее число ответов HTTP 4xx  <br/> |
|Всего отчетов HTTP 5xx  <br/> |Общее число ответов HTTP 5xx  <br/> |
|Total Requests received on the Command Channel (Всего запросов, полученных по каналу управления)  <br/> |Общее число запросов, полученных по каналу управления  <br/> |
|Total Requests Succeeded (Всего успешных запросов)  <br/> |Общее число успешных запросов  <br/> |
|Total Sessions Initiated (Всего запущенных сеансов)  <br/> |Общее число сеансов, запущенных с момента запуска службы  <br/> |
|Total Sessions Terminated Because of User Idle Timeout (Всего сеансов, завершенных из-за таймаута бездействия)  <br/> |Общее число сеансов, которые были завершены из-за истечения времени ожидания в режиме бездействия пользователя  <br/> |
|Total Throttled Applications (Всего приложений, к которым применяется регулирование)  <br/> |Число приложений, к которым применяется регулирование  <br/> |

|**Счетчик**|**Описание**|

|Average Lifetime for a Session in Milliseconds (Среднее время существования сеанса в миллисекундах)  <br/> |Среднее время существования сеанса в миллисекундах (мс)  <br/> |

|Requests Failed/Second (Неудачных запросов/с)  <br/> |Частота запросов, завершившихся ошибками (в секунду)  <br/> |
|Requests Received/Second (Полученных запросов/с)  <br/> |Частота полученных запросов (в секунду)  <br/> |
|Requests Rejected/Second (Отклоненных запросов/с)  <br/> |Частота отклоненных запросов (в секунду)  <br/> |
|Requests Succeeded/Second (Успешных запросов/с)  <br/> |Частота успешных запросов (в секунду)  <br/> |

Запуск сеанса, как правило, означает вход пользователя, с момента выхода которого из сеанса прошло довольно много времени.  <br/> |

|Total Push Notification Requests (Всего запросов на push-уведомления)  <br/> |Общее число запросов на push-уведомления  <br/> |
|Total Push Notification Requests Failed (Всего неудачных запросов на push-уведомления)  <br/> |Общее число запросов на push-уведомления, завершившихся сбоем  <br/> |
|Total Push Notification Requests Succeeded (Всего успешных запросов на push-уведомления)  <br/> |Общее число успешно выполненных запросов на push-уведомления  <br/> |
