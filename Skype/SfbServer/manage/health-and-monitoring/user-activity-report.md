# <a name="user-activity-report-in-skype-for-business-server"></a>Отчет об активности пользователей в Skype для бизнеса Server

**Сводка:** Узнайте о отчете об активности пользователей в Skype для бизнеса Server.

Отчет об активности пользователей предоставляет подробный список одноранговых сеансов и сеансов конференц-связи, проведенных вашими пользователями за определенный период времени. В отличие от многих отчетов мониторинга отчет об активности пользователей связывает каждый звонок с отдельными пользователями. Например, одноранговые сеансы указывают универсальные коды ресурса (URI) SIP человека, который инициировал звонок (пользователь-отправитель), и человека, которому звонок был выполнен (пользователь-получатель). Если развернуть информацию по конференции вы увидите список всех участников конференции с указанием их роли.

Отчет об активности пользователей иногда называют отчетом «службы технической поддержки». Это вызвано тем, что данный отчет часто используется сотрудниками службы технической поддержки для получения информации о сеансе для определенного пользователя. Вы можете отфильтровывать входящие и исходящие звонки для отдельного пользователя, просто введя URI SIP в поле «User URI prefix» (Префикс URI пользователя).

В этом случае отчет о действии пользователя возвращает сведения для любого пользователя, у которого SIP URI начинается с указанной строки. Например, если введите **кен** в поле URI, в отчете об активности пользователей будет **находиться ken.Myer@litwareinc.com**. Тем не менее, он также будет находить этих пользователей:

- **Кен** azi@litwareinc.com

- **Кен burg@litwareinc.com**

- **Ken.Sanchez@litwareinc.com**

- **Кен** nedy@litwareinc.com

Чтобы убедиться, что возвращается информация только для Кена Myer, введите его полное URI (Ken.Myer@litwareinc.com) в поле поиска или по крайней мере достаточно типа URI Кена, чтобы уникально отличить его от других пользователей в вашей организации. Например:

Ken.my

## <a name="to-access-the-user-activity-report"></a>Чтобы получить доступ к отчету о действиях пользователей, выполните следующие действия

Доступ к отчету об активности пользователей осуществляется с домашней страницы отчетов мониторинга. Вы также можете достичь отчета о действии пользователя, щелкнув метрику URI пользователя в отчете о Телефон инвентаризации в [Skype для бизнеса Server](ip-phone-inventory-report.md). Щелкнув элемент «Conference URI» (URI конференции) в отчете об активности пользователей вы откроете подробный отчет о конференции. Аналогичным образом, щелкнув метрику Detail для одноранговых зовов, вы можете получить отчет о подробностях одноранговой сессии в [Skype для бизнеса Server](peer-to-peer-session-detail-report.md).

## <a name="making-the-best-use-of-the-user-activity-report"></a>Наилучшее использование отчета о действиях пользователя

Хотя отчет об активности пользователей содержит много полезной информации, ее бывает сложно найти. Например, все действия пользователей, которые проходят в организации в течение указанного периода, включаются в отчет о действии пользователя; это означает, что в отчете зарыты сведения о том, какие пользователи на самом деле Skype для бизнеса Server каким-либо образом.

> [!NOTE]
> Технически не исключено, что некоторые действия пользователей могут не быть записаны: хотя Skype для бизнеса Server стремится хранить сведения обо всех телефонных звонках, вполне возможно, что звонок мог быть выполнен без записи этого звонка в базу данных. Skype для бизнеса Server предназначена для того, чтобы дать очень точный, но не обязательно идеальный взгляд на то, как Skype для бизнеса Server используется. (Тот факт, что нет гарантии записи 100% всех вызовов, объясняет, почему мониторинг Skype для бизнеса Server не следует использовать в качестве системы выписывки.) Во-вторых, отчет мониторинга может отображать не более 1000 записей. Таким образом, в зависимости от уровня активности пользователей и обрабатываемого периода запрос может возвращать не все данные, находящиеся в базе данных. 

- Какие именно пользователи использовали систему в определенный период?

- Какие именно пользователи были наиболее активными в определенный период?

- Являются ли пользователи, выполнившие наиболее количество звонков, так же и пользователями, принявшими участие в большинстве сеансов обмена мгновенными сообщениями?

Если вам нужно ответить на такие вопросы, вы можете экспортировать данные, полученные отчетами мониторинга, в Excel таблицу. Затем этот файл таблиц и/или файл значений, разделенных запятой, используется для анализа данных таким образом, как отчет о действии пользователя. Например, предположим, что вы экспортировали данные отчета в Excel, а затем в файл значений, разделенных запятой. На этом этапе можно импортировать данные из файла .CSV в Windows PowerShell с помощью команды, аналогичной этой:

```PowerShell
$x = Import-Csv -Path "C:\Data\User_Activity_Report.csv"
```

После импорта данных можно использовать простые Windows PowerShell для ответа на ваши вопросы. Например, эта команда возвращает список уникальных пользователей, являвшихся пользователем-отправителем хотя бы в одном сеансе:

```PowerShell
$x | Group-Object "From user" | Select Name | Sort-Object Name
```

Другими словами:

<pre>
Name
----
David.Ahs@litwareinc.com
Gilead.Amosnino@litwareinc.com
Henrik.Jensen@litwareinc.com
Ken.Myer@litwareinc.com
Pilar.Ackerman@litwareinc.com
</pre>

Эта команда выводит список уникальных пользователей (на основании общего числа сеансов, в которых они приняли участие:

```PowerShell
$x | Group-Object "From user" | Select Count, Name | Sort-Object Count -Descending
```

При этом возвращаются данные, аналогичные следующим:

<pre>
Count    Name
-----    ----
  523    Ken.Myer@litwareinc.com
   63    David.Ahs@litwareinc.com
   29    Pilar.Ackerman@litwareinc.com
   17    Gilead.Amosnino@litwareinc.com
   10    Henrik.Jensen@litwareinc.com
</pre>

Эта команда ограничивает область поиска только сеансами, имевшими звуковую модальность:

```PowerShell
$x | Where-Object {$_.Modalities -match "audio"} | Group-Object "From user" | Select Count, Name | Sort-Object Count -Descending
```

При наведении указателя на любой идентификатор диагностики в отчете отображается подсказка с описанием этого идентификатора.

Например, отчет об активности пользователей позволяет вам фильтровать возвращенные данные, например, по типу активности (одноранговые сеансы или сеансы конференц-связи) или по SIP-адресу пользователя (позволяя вам просматривать активность одного пользователя). Вы также можете определить группировку данных. В данном случае сведения об использовании группируются по часу, дню, неделе или месяцу.

## <a name="metrics-for-peer-to-peer-sessions"></a>Метрики для одноранговых сеансов

В следующей таблице приведены сведения из отчета об активности пользователей для одноранговых сеансов (то есть сеансов, включающих всего двух участников).

**Метрики для одноранговых сеансов**

|**Имя**|**Возможность сортировки по этому показателю**|**Описание**|
|:-----|:-----|:-----|
|**Detail** <br/> |Нет  <br/> |При щелчке данного элемента отчет отображает для выбранного сеанса подробный отчет об одноранговом сеансе.  <br/> |
|**From user** (Пользователь-отправитель) <br/> |Да  <br/> |SIP-адрес пользователя, который инициировал данный одноранговый сеанс.  <br/> |
|**To user** (Пользователь-получатель) <br/> |Да  <br/> |SIP-адрес пользователя, который присоединился к данному одноранговому сеансу.  <br/> |
|**Модальности** <br/> |Да  <br/> |Тип коммуникации, использованный в сеансе. Например, мгновенные сообщения, звук или передача файлов.  <br/> |
|**Invite time** (Время приглашения) <br/> |Да  <br/> |Дата и время отправки начального приглашения присоединиться к одноранговому сеансу.  <br/> |
|**Response time** (Время ответа) <br/> |Да  <br/> |Дата и время, когда пользователь-получатель принял приглашение на сеанс.  <br/> |

|**Diagnostic ID** (Идентификатор диагностики) <br/> |Да  <br/> |Прикрепленный к SIP-сообщению уникальный идентификатор (в форме заголовка ms-diagnostics), который часто содержит информацию, полезную при поиске и устранении ошибок. Заголовки диагностики не являются обязательными (можно иметь сеансы SIP без таких заголовков), а идентификаторы диагностики указываются только для сеансов, в которых возникли какие-либо проблемы.  <br/> |
