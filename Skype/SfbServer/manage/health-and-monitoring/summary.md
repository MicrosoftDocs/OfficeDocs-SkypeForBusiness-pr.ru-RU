# <a name="media-quality-summary-report-in-skype-for-business-server"></a>Сводный отчет о качестве мультимедиа в Skype для бизнеса Server
 
**Сводка:** Сведения о сводке качества мультимедиа в Skype для бизнеса Server.
  
Сводный отчет по качеству мультимедиа, возможно, является лучшим предложением для анализа качества звонков в организации: он предоставляет подробные метрики качества взаимодействия при вызовах, распределенные по следующим категориям:
  
- UC Peer to Peer Calls (such as a Skype для бизнеса to Skype для бизнеса call)
    
- UC Conference Sessions (сеансы конференций в объединенных коммуникациях);
    
- PSTN Conference Sessions (сеансы конференций в ТСОП);
    
- PSTN Calls: Media Bypass (вызовы в ТСОП: обход сервера-посредника);
    
- PSTN Calls (Non-Bypass): UC Leg (вызовы в ТСОП (без обхода): участок объединенных коммуникаций);
    
- PSTN Calls (Non-Bypass): Gateway Leg (вызовы в ТСОП (без обхода): участок шлюза);
    
- другие типы вызовов.
    
При первом открытии отчета отображается сводка по каждой из этих категорий. Не выходя из отчета, можно расширить каждую категорию, чтобы посмотреть на подкатегории, такие как вызовы, Office Communicator 2007 R2 до Skype для бизнеса. Затем можно также детализировать эти подкатегории, чтобы просмотреть сведения о каждом вызове, сделанном в этой подкатегории.
  
В Skype для бизнеса Server сводный отчет о качестве мультимедиа дополнительно разбивает данные на три типа вызовов: звуковые вызовы, видеозвонков и вызовы общего доступа к приложениям. Для каждого типа вызова в отчете имеется свой собственный раздел и свой собственный ряд метрик вызова.
  
Кроме того, сводный отчет по качеству мультимедиа позволяет применять фильтры, чтобы сравнивать качество проводных и беспроводных вызовов, внутренних и внешних вызовов, а также вызовов с использованием VPN и вызовов без VPN.
  
## <a name="accessing-the-media-quality-summary-report"></a>Доступ к сводному отчету по качеству мультимедиа

Сводный отчет по качеству мультимедиа можно вызвать на домашней странице отчетов наблюдения. Вы можете сверлить до отчета [списка](call-list-report-0.md) вызовов в Skype для бизнеса Server, щелкнув любой из следующих метрик:
  
- Объем звонков
    
- Poor call percentage (Процент звонков низкого качества)
    
Кроме того, можно вызвать отчет по распределению метрик качества мультимедиа, щелкнув какую-либо из метрик звонка:
  
- Round trip (ms)
(Круговой путь (мс))
    
- Degradation (MOS) (Снижение (MOS))
    
- Packet loss (Потеря пакетов)
    
- Jitter (ms) (Дрожание (мс))
    
- Healer concealed ratio (Коэффициент скрытых сэмплов)
    
- Healer stretched ratio (Коэффициент растянутых сэмплов)
    
- Healer compressed ratio (Коэффициент сжатых сэмплов)
    
## <a name="filters"></a>Фильтры

Фильтры позволяют получать более адресный набор данных или просматривать полученные данные разными способами. Например, в сводном отчете по качеству мультимедиа можно фильтровать данные по типу доступа (т.е. по внутреннему или внешнему доступу), а также по проводному или беспроводному типу подключения. Кроме того, можно выбрать способ группирования данных. В данном случае вызовы группируются по часу, дню, неделе или месяцу.

Звонок низкого качества — это любой звонок, хотя бы одна метрика которого превышает допустимое значение (например, звонок с чрезмерным дрожанием).  <br/> |
|**Call volume (wireless call) (Интенсивность вызовов (беспроводной вызов))** <br/> |Нет  <br/> |Общее количество вызовов, использовавших беспроводное подключение.  <br/> |
|**Call volume (VPN call) (Интенсивность вызовов (через VPN))** <br/> |Нет  <br/> |Общее количество вызовов, использовавших подключение через VPN.  <br/> |
|**Call volume (external call) (Интенсивность вызовов (внешние вызовы))** <br/> |Нет  <br/> |Количество вызовов, которые использовали внешнее подключение (т.е. подключение за пределами внутренней сети).  <br/> |
|**Дрожание (мс)** <br/> |Нет  <br/> |Средний уровень дрожания, определенный между прибытием пакетов протокола RTP. (Дрожание — это показатель "дрожания" звонка.) Высокие значения дрожания обычно вызываются перегруженностью или высокой загрузкой сервера мультимедиа и приводят к искажению или потере звука.  <br/> |
|**Средняя относительная задержка в одну сторону** <br/> |Нет  <br/> |Средняя относительная задержка в одну сторону между двумя конечными точками медиаданных. Это мера односкачковой задержки.  <br/> |
|**Средняя задержка обработки элемента RDP** <br/> |Нет  <br/> |Среднее значение задержки обработки элементов RDP на сервере конференц-связи общего доступа к приложениям во время сеанса просмотра. Высокий средний показатель отражает более длинную задержку в просмотре и включает задержку в сети. На перегруженном сервере конференц-связи могут происходить в среднем более длительные задержки.  <br/> |
|**Total spoiled tile % (Итоговый % испорченных плиток)** <br/> |Нет  <br/> |Итоговый процент испорченных плиток протокола RDP.  <br/> |
