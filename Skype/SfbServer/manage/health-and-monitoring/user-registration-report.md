# <a name="user-registration-report-in-skype-for-business-server"></a>Отчет о регистрации пользователя в Skype для бизнеса Server
 
**Сводка:** Ознакомьтесь с отчетом о регистрации пользователей в Skype для бизнеса Server.
  
Отчет о регистрации пользователей содержит общие сведения о действиях входа в систему, наиболее часто заданных о количестве пользователей, которые вошли на сервер Skype для бизнеса в течение определенного периода времени (ежечасно, ежедневно, еженедельно, ежемесячно). Следует иметь в виду, что в отчете лишь сообщается, сколько людей входило в систему. В нем не указывается, какие люди входили в систему. Отчеты мониторинга не предоставляют сведения о том, какие пользователи используют Skype для бизнеса Server (и какие из них не являются). Тем не менее с помощью отчета об активности пользователей можно получить грубую оценку пользовательской информации.
  
При предоставлении сведений о входах пользователей в систему в отчете о регистрации пользователей делается два важных разграничения. Во-первых, в отчете входы в систему разбиваются на две основные категории: внутренние и внешние входы в систему. Внутренние входы представляют пользователей, вошедших в систему с внутренней стороны брандмауэра организации (то есть, с компьютера, подключенного к корпоративной сети). Внешние входы — это пользователи, которые вошли в систему за пределами брандмауэра через пограничный сервер (например, пользователь, выполнивший вход из Интернет-кафе, считает внешним входом). Если необходимо узнать, сколько пользователей входит в систему с внешней стороны брандмауэра, эти сведения можно найти в отчете о регистрации пользователей.
  
Кроме того, в отчете о регистрации пользователей отмечается, сколько активных пользователей присутствовало в течение заданного периода времени. Активный пользователь — это пользователь, который потратил участие в сеансе обмена мгновенными сообщениями, принял участие в собрании Skype для бизнеса Server, сделал или получил телефонный звонок, или иным образом использовал Skype для бизнеса Server в течение этого периода времени. Такой пользователь отличается от пользователя, вошедшего в систему, но не проявлявшего никакой активности в системе.
  
## <a name="accessing-the-user-registration-report"></a>Доступ к отчету о регистрации пользователей

Доступ к отчету о регистрации пользователей предоставляется только на домашней странице отчетов мониторинга. Отчет о регистрации пользователей не связан с какими-либо другими отчетами.
  
## <a name="making-the-best-use-of-the-user-registration-report"></a>Оптимальное использование отчета о регистрации пользователей

После развертывания Skype для бизнеса Server один из часто задаваемых вопросов: как узнать, действительно ли мои пользователи используют эту новую технологию? Хотя в этом отношении есть определенные ограничения, отчет о регистрации пользователей позволяет ответить на этот вопрос. Чтобы определить, работают ли пользователи в Skype для бизнеса Server, необходимо выполнить два действия. Во-первых, нужно получить значение показателя уникальных входов пользователей из отчета о регистрации пользователей. В этом параметре указывается количество отдельных пользователей, выполнивших вход в Skype для бизнеса Server.
  
По сравнению с функцией общего числа входов в систему показано общее количество пользователей, которые вошли в Skype для бизнеса Server. Например, предположим, что Кен мер вошел в Skype для бизнеса Server пятью в разных случаях за один день. В этом случае для Кен Майера было бы подсчитано пять отдельных сеансов входа в показателе общего числа входов, но лишь один зарегистрированный пользователь в показателе уникальных зарегистрированных пользователей. Аналогично, пользователи нередко входят в систему с нескольких устройств или из нескольких мест. Например, пользователь может войти в систему с помощью настольного компьютера, ноутбука и компьютера с IP-телефонным подключением, который автоматически входит в Skype для бизнеса Server. В этом примере имеется один уникальный пользователь с тремя входами в систему.
  
Для дальнейшего пояснения разницы между общим количеством входов и уникальными входами рассмотрим входы в систему для заданного периода времени, приведенные в следующей таблице.
  
|**User**|**Время входа**|
|:-----|:-----|
|Ken Myer  <br/> |07.07.2015 8:45  <br/> |
|Ken Myer  <br/> |07.07.2015 8:46  <br/> |
|Pilar Ackerman  <br/> |07.07.2015 9:17  <br/> |
|Кен Майер  <br/> |07.07.2015 9:22  <br/> |
|Пилар Аккерман  <br/> |07.07.2015 9:31  <br/> |
   
Обратите внимание, что общее количество входов в систему равно пяти; однако существует только два уникальных регистрировавшихся пользователя: Кен Майер (входивший в систему три раза) и Пилар Аккерман (заходивший в систему дважды). В этом разница между входами в систему и уникальными регистрировавшимися пользователями.
  
Помимо числа уникальных входов в систему, необходимо знать общее количество пользователей, которые были включены в Skype для бизнеса Server. Это значение можно получить, открыв командную консоль управления Skype для бизнеса Server и выполнив следующую команду Windows PowerShell:
  
```PowerShell
(Get-CsUser).Count
```

Если предыдущая команда возвращает значение 1 236 и уникальный вход Users, функция метрика Возвращает среднее значение 667, что означает, что в течение более половины пользователей Skype для бизнеса будут входить в систему каждый день (то есть 667 на 1 236, то есть около 54%).
  
> [!CAUTION]
> Следует иметь в виду, что показателями входов учитываются пользователи, которые на самом деле входили в систему в указанный период времени. Они не учитывают пользователей, которые уже были зарегистрированы в системе. Например, если показатель количества уникальных регистрировавшихся пользователей равен 667 и имеется 1236 пользователей, это свидетельствует о том, что около половины пользователей входят в систему. Однако предположим, что 300 пользователей уже находились в системе на момент, когда началась проверка данных о регистрации. Это означает, что у тебя есть почти 1 000 пользователей, выполнивших вход в Skype для бизнеса Server, что означает, что ближе к 80% ваших пользователей. 
  
Следует также сравнить значение показателя количества уникальных регистрировавшихся пользователей со значением показателя количества уникальных активных пользователей. Метрика "уникальные активные пользователи" показывает, сколько уникальных пользователей действительно использовали Skype для бизнеса Server: они были присоединены к собранию Skype для бизнеса Server или участвуют в сеансе обмена мгновенными сообщениями. Это полезные сведения, так как в Skype для бизнеса Server можно настроить автоматический запуск Windows при каждом запуске пользователя. Из-за этого у вас может быть большое количество пользователей, которые автоматически подключаются к Skype для бизнеса при каждом входе в Windows каждый день, но в этом случае никогда не используете Skype для бизнеса Server в течение этого периода.
  
Метрика уникальных активных пользователей также предоставляет более осмысленные данные в Организации, где пользователи обычно не завершают работу с Windows в конце дня. Вместо этого они просто заблокировали свои компьютеры, оставляя Windows и Skype для бизнеса запущенными. В подобной ситуации каждый день может происходить всего несколько входов в систему, так как пользователи зарегистрировались несколько дней назад и с тех пор не выходили из системы. Однако уникальные активные пользователи сообщают, активно ли пользователи в Skype для бизнеса или в другом клиенте Skype для бизнеса Server.
  
## <a name="filters"></a>Фильтры

Фильтры позволяют возвращать более точный набор данных или различными способами просматривать возвращенные данные. Например, с помощью отчета регистрация пользователей можно просматривать данные всех пулов регистраторов и пограничных серверов, а также просматривать данные для отдельного пула. Можно также выбрать порядок группирования данных. В этом случае регистрации группируются по времени, дням, неделям или месяцам.
  
В следующей таблице приведены фильтры, которые можно использовать с отчетом о регистрации пользователей.

|:-----|:-----|

Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Чтобы просмотреть данные за неделю или месяц, введите дату, выпадающую на любое время в рамках недели или месяца, которые вы хотите просмотреть (вам не требуется вводить первый день недели или месяца):  <br/> 7/3/2015  <br/> Недели всегда отсчитываются с воскресенья по субботу.  <br/> |

Например, если вы выбрали интервал "Ежедневно" с начальной датой 07.07.2015 и конечной датой 28.02.2012, будут показаны данные для периода с 07.08.2015 24:00 до 07.09.2015 24:00 (то есть для 31 дня). <br/> |
