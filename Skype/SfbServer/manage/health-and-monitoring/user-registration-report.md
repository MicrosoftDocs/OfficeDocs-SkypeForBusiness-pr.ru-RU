# <a name="user-registration-report-in-skype-for-business-server"></a>Отчет о регистрации пользователей в Скайп для Business Server
 
**Сводка:** Сведения об отчете регистрации пользователей в Скайп Business Server.
  
Отчет о регистрации пользователей предоставляет обзор действий входа в систему пользователя, особенно сведения о количество пользователей, вход в систему на Скайп для Business Server за указанный период (ежечасно, ежедневно, еженедельно, ежемесячно). Следует иметь в виду, что в отчете лишь сообщается, сколько людей входило в систему. В нем не указывается, какие люди входили в систему. Отчетов мониторинга не предоставляют сведения о конкретных пользователей, которые используют Скайп для Business Server (и какие из них не). Тем не менее с помощью отчета об активности пользователей можно получить грубую оценку пользовательской информации.
  
При предоставлении сведений о входах пользователей в систему в отчете о регистрации пользователей делается два важных разграничения. Во-первых, в отчете входы в систему разбиваются на две основные категории: внутренние и внешние входы в систему. Внутренние входы представляют пользователей, вошедших в систему с внутренней стороны брандмауэра организации (то есть, с компьютера, подключенного к корпоративной сети). Количество внешних входов представляют пользователей, которые вошел в систему из за пределами брандмауэра через пограничный сервер (например, пользователь, который вошел в систему из Интернета, кафе считается внешним входа). Если необходимо узнать, сколько пользователей входит в систему с внешней стороны брандмауэра, эти сведения можно найти в отчете о регистрации пользователей.
  
Кроме того, в отчете о регистрации пользователей отмечается, сколько активных пользователей присутствовало в течение заданного периода времени. Активных пользователей — пользователя, принявшего часть в сеанс обмена Мгновенными сообщениями, обмена мгновенными сообщениями участие в Скайп для Business Server собрания, выполненных или полученных телефонного звонка или в противном случае используется Скайп для Business Server за этот период времени. Такой пользователь отличается от пользователя, вошедшего в систему, но не проявлявшего никакой активности в системе.
  
## <a name="accessing-the-user-registration-report"></a>Доступ к отчету о регистрации пользователей

Доступ к отчету о регистрации пользователей предоставляется только на домашней странице отчетов мониторинга. Отчет о регистрации пользователей не связан с какими-либо другими отчетами.
  
## <a name="making-the-best-use-of-the-user-registration-report"></a>Оптимальное использование отчета о регистрации пользователей

Это после развертывания Скайп для Business Server один часто задаваемые вопрос: как узнать, если Мои пользователи фактически с помощью этой новой технологии? Хотя в этом отношении есть определенные ограничения, отчет о регистрации пользователей позволяет ответить на этот вопрос. Чтобы определить ли пользователи используют Скайп для Business Server, необходимо выполнить два действия. Во-первых, нужно получить значение показателя уникальных входов пользователей из отчета о регистрации пользователей. Это значение указывает, сколько отдельных лиц, вход в систему на Скайп для Business Server.
  
Для сравнения метрику общее количество входов показывает общее количество всем пользователям вход в систему на Скайп для Business Server. Например предположим, что Кен Майер вход в систему на Скайп для Business Server пять разных раз в один день. В этом случае для Кен Майера было бы подсчитано пять отдельных сеансов входа в показателе общего числа входов, но лишь один зарегистрированный пользователь в показателе уникальных зарегистрированных пользователей. Аналогично, пользователи нередко входят в систему с нескольких устройств или из нескольких мест. Например пользователь может войти в систему с помощью установил настольном ПК, установил ноутбук и она может иметь IP-телефона, который автоматически входит Скайп для Business Server. В этом примере имеется один уникальный пользователь с тремя входами в систему.
  
Для дальнейшего пояснения разницы между общим количеством входов и уникальными входами рассмотрим входы в систему для заданного периода времени, приведенные в следующей таблице.
  
|**User**|**Время входа**|
|:-----|:-----|
|Ken Myer  <br/> |07.07.2015 8:45  <br/> |
|Ken Myer  <br/> |07.07.2015 8:46  <br/> |
|Pilar Ackerman  <br/> |07.07.2015 9:17  <br/> |
|Кен Майер  <br/> |07.07.2015 9:22  <br/> |
|Пилар Аккерман  <br/> |07.07.2015 9:31  <br/> |
   
Обратите внимание, что общее количество входов в систему равно пяти; однако существует только два уникальных регистрировавшихся пользователя: Кен Майер (входивший в систему три раза) и Пилар Аккерман (заходивший в систему дважды). В этом разница между входами в систему и уникальными регистрировавшимися пользователями.
  
В дополнение к знание количество уникальных входов в систему, необходимо знать общее количество пользователей, которым был разрешен доступ к Скайп для Business Server. Это значение можно получить, открыв Скайп для консоли Business Server и выполнив следующую команду Windows PowerShell:
  
```
(Get-CsUser).Count
```

Предыдущая команда возвращает значение 1,236 и количество уникальных входов пользователей показатель возвращает среднее значение 667, который предполагает, что немного более чем вдвое пользователей включить для Скайп для бизнеса фактически вход в систему каждый день (например 667, разделенная 1,236, являющийся приблизительно 54%).
  
> [!CAUTION]
> Следует иметь в виду, что показателями входов учитываются пользователи, которые на самом деле входили в систему в указанный период времени. Они не учитывают пользователей, которые уже были зарегистрированы в системе. Например, если показатель количества уникальных регистрировавшихся пользователей равен 667 и имеется 1236 пользователей, это свидетельствует о том, что около половины пользователей входят в систему. Однако предположим, что 300 пользователей уже находились в системе на момент, когда началась проверка данных о регистрации. Который будет означать пришлось около 1 000 пользователей, вход в систему на Скайп для Business Server, который означает, что, ближе к 80% пользователей вошел в систему. 
  
Следует также сравнить значение показателя количества уникальных регистрировавшихся пользователей со значением показателя количества уникальных активных пользователей. Метрики количество активных пользователей указывает, сколько уникальных пользователи использовали Скайп для Business Server: вносил телефонный звонок, Скайп они присоединен к собранию Business Server или они приняли в сеанс обмена мгновенными Сообщениями. Это полезную информацию, так как Скайп для Business Server можно настроить на автоматический запуск при каждом запуске Windows. Из-за, могут иметь большое число пользователей, которые автоматически входить на Скайп для бизнеса при их входе в Windows каждый день, но при использовании фактически Скайп для Business Server во время этого периода времени.
  
Количество активных пользователей показатель также предоставляет более удобной для восприятия данных в организации, где пользователи обычно не завершите сеанс Windows в конце дня. Вместо этого они просто заблокировать своих компьютеров и оставьте Windows и Скайп для бизнеса. В подобной ситуации каждый день может происходить всего несколько входов в систему, так как пользователи зарегистрировались несколько дней назад и с тех пор не выходили из системы. Тем не менее количество активных пользователей о том, является ли пользователей, активно использующих Скайп для бизнеса или другой Скайп для клиента Business Server.
  
## <a name="filters"></a>Фильтры

Фильтры позволяют возвращать более точный набор данных или различными способами просматривать возвращенные данные. Например отчет о регистрации пользователей позволяет для просмотра данных для пула регистратора и пограничных серверов или для просмотра данных для отдельных пула. Можно также выбрать порядок группирования данных. В этом случае регистрации группируются по времени, дням, неделям или месяцам.

|:-----|:-----|

Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Чтобы просмотреть данные за неделю или месяц, введите дату, выпадающую на любое время в рамках недели или месяца, которые вы хотите просмотреть (вам не требуется вводить первый день недели или месяца):  <br/> 7/3/2015  <br/> Недели всегда отсчитываются с воскресенья по субботу.  <br/> |

Например, если вы выбрали интервал "Ежедневно" с начальной датой 07.07.2015 и конечной датой 28.02.2012, будут показаны данные для периода с 07.08.2015 24:00 до 07.09.2015 24:00 (то есть для 31 дня). <br/> |
