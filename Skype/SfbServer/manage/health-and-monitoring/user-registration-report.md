# <a name="user-registration-report-in-skype-for-business-server"></a>Отчет о регистрации пользователей в Skype для бизнеса Server
 
**Сводка:** Узнайте о отчете о регистрации пользователей в Skype для бизнеса Server.
  
Отчет о регистрации пользователей содержит обзор активности логоса пользователя, в частности сведения о количестве пользователей, которые вошли в систему Skype для бизнеса Server за указанный период времени (почасовая, ежедневная, еженедельная, ежемесячная). Следует иметь в виду, что в отчете лишь сообщается, сколько людей входило в систему. В нем не указывается, какие люди входили в систему. Отчеты мониторинга не предоставляют сведений о том, какие конкретные пользователи используют Skype для бизнеса Server (а какие нет). Тем не менее с помощью отчета об активности пользователей можно получить грубую оценку пользовательской информации.
  
При предоставлении сведений о входах пользователей в систему в отчете о регистрации пользователей делается два важных разграничения. Во-первых, в отчете входы в систему разбиваются на две основные категории: внутренние и внешние входы в систему. Внутренние входы представляют пользователей, вошедших в систему с внутренней стороны брандмауэра организации (то есть, с компьютера, подключенного к корпоративной сети). Внешние логотипы представляют пользователей, входивших из-за брандмауэра через edge Server (например, пользователь, вошел в интернет-кафе, считается внешним логотипом). Если необходимо узнать, сколько пользователей входит в систему с внешней стороны брандмауэра, эти сведения можно найти в отчете о регистрации пользователей.
  
Кроме того, в отчете о регистрации пользователей отмечается, сколько активных пользователей присутствовало в течение заданного периода времени. Активный пользователь — это пользователь, который принимал участие в сеансе обмена мгновенными сообщениями, участвовал в собрании Skype для бизнеса Server, сделал или получил телефонный звонок или использовал Skype для бизнеса Server в течение этого периода времени. Такой пользователь отличается от пользователя, вошедшего в систему, но не проявлявшего никакой активности в системе.
  
## <a name="accessing-the-user-registration-report"></a>Доступ к отчету о регистрации пользователей

Доступ к отчету о регистрации пользователей предоставляется только на домашней странице отчетов мониторинга. Отчет о регистрации пользователей не связан с какими-либо другими отчетами.
  
## <a name="making-the-best-use-of-the-user-registration-report"></a>Оптимальное использование отчета о регистрации пользователей

После развертывания Skype для бизнеса Server часто заданный вопрос: как узнать, действительно ли пользователи используют эту новую технологию? Хотя в этом отношении есть определенные ограничения, отчет о регистрации пользователей позволяет ответить на этот вопрос. Чтобы определить, используют ли пользователи Skype для бизнеса Server, необходимо сделать две вещи. Во-первых, нужно получить значение показателя уникальных входов пользователей из отчета о регистрации пользователей. Это значение указывает количество отдельных пользователей, во время входа в Skype для бизнеса Server.
  
Для сравнения, в метрике Total logons показано, сколько всего раз кто-либо входил в Skype для бизнеса Server. Например, предположим, что Ken Myer вошел в систему Skype для бизнеса Server пять раз за один день. В этом случае для Кен Майера было бы подсчитано пять отдельных сеансов входа в показателе общего числа входов, но лишь один зарегистрированный пользователь в показателе уникальных зарегистрированных пользователей. Аналогично, пользователи нередко входят в систему с нескольких устройств или из нескольких мест. Например, пользователь может войти в систему с помощью настольного компьютера, ноутбука, и у нее может быть IP-телефон, который автоматически входит в Skype для бизнеса Server. В этом примере имеется один уникальный пользователь с тремя входами в систему.
  
Для дальнейшего пояснения разницы между общим количеством входов и уникальными входами рассмотрим входы в систему для заданного периода времени, приведенные в следующей таблице.
  
|**Пользователь**|**Время входа**|
|:-----|:-----|
|Кен Майер  <br/> |7.07.2015 08.45  <br/> |
|Кен Майер  <br/> |7/7/2015 08:46  <br/> |
|Пилар Аккерман  <br/> |7.07.2015 09.17  <br/> |
|Кен Майер  <br/> |07.07.2015 09:22  <br/> |
|Пилар Аккерман  <br/> |7.07.2015 09.31  <br/> |
   
Обратите внимание, что общее количество входов в систему равно пяти; однако существует только два уникальных регистрировавшихся пользователя: Кен Майер (входивший в систему три раза) и Пилар Аккерман (заходивший в систему дважды). В этом разница между входами в систему и уникальными регистрировавшимися пользователями.
  
Помимо зная количество уникальных логотипов, необходимо знать общее число пользователей, которые были включены для Skype для бизнеса Server. Это значение можно получить, открыв команду Skype для бизнеса Server management Shell и задав следующую команду Windows PowerShell:
  
```PowerShell
(Get-CsUser).Count
```

Если в предыдущей команде возвращается значение 1236, а для пользователей с уникальным логотипом возвращается среднее значение в 667, это позволяет предположить, что чуть более половины пользователей в течение Skype для бизнеса фактически вошел в систему каждый день (т. е. 667, разделенных на 1236, что составляет примерно 54%).
  
> [!CAUTION]
> Следует иметь в виду, что показателями входов учитываются пользователи, которые на самом деле входили в систему в указанный период времени. Они не учитывают пользователей, которые уже были зарегистрированы в системе. Например, если показатель количества уникальных регистрировавшихся пользователей равен 667 и имеется 1236 пользователей, это свидетельствует о том, что около половины пользователей входят в систему. Однако предположим, что 300 пользователей уже находились в системе на момент, когда началась проверка данных о регистрации. Это означает, что у вас на самом деле было около 1000 пользователей, зарегистрированных в Skype для бизнеса Server, что означает, что ближе к 80% пользователей были внесены в систему. 
  
Следует также сравнить значение показателя количества уникальных регистрировавшихся пользователей со значением показателя количества уникальных активных пользователей. Метрика Уникальные активные пользователи рассказывает, сколько уникальных пользователей на самом деле Skype для бизнеса Server: они сделали телефонный звонок, они присоединились к собранию Skype для бизнеса Server или участвовали в сеансе чата. Это полезная информация, так как Skype для бизнеса Server можно настроить для автоматического запуска при каждом запуске пользователя Windows. Из-за этого у вас может быть большое количество пользователей, которые автоматически включались в Skype для бизнеса при входе в Windows каждый день, но в этот период Skype для бизнеса Server не использовались.
  
Метрика Уникальные активные пользователи также предоставляет более значимые данные в организации, где пользователи обычно не Windows в конце дня. Вместо этого они просто заблокировать свои компьютеры и оставить Windows и Skype для бизнеса работает. В подобной ситуации каждый день может происходить всего несколько входов в систему, так как пользователи зарегистрировались несколько дней назад и с тех пор не выходили из системы. Однако уникальные активные пользователи подсказывает, активно ли пользователи используют Skype для бизнеса или другого Skype для бизнеса Server клиента.
  
## <a name="filters"></a>Фильтры

Фильтры позволяют получить более конкретные наборы возвращаемых данных. Например, отчет о регистрации пользователей позволяет просматривать данные для всего пула регистраторов и edge Servers или просматривать данные для отдельного пула. Можно также выбрать порядок группирования данных. В этом случае регистрации группируются по времени, дням, неделям или месяцам.
  
В следующей таблице приведены фильтры, которые можно использовать с отчетом о регистрации пользователей.
  
**Фильтры отчета о регистрации пользователей**

|**Имя**|**Описание**|
|:-----|:-----|
|**From** <br/> |Начальные дата и время временного диапазона. Чтобы просмотреть данные по часам, введите начальные дату и время следующим образом:  <br/> 7/7/2015 13:00  <br/> Если не указать время начала, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Неделя всегда начинается с воскресения и заканчивается субботой.  <br/> |
|**To** <br/> |Конечные дата и время временного диапазона. Чтобы просмотреть данные по часам, введите конечные дату и время следующим образом:  <br/> 7/7/2015 13:00  <br/> Если не указать время окончания, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Неделя всегда начинается с воскресения и заканчивается субботой.  <br/> |
|**Interval** <br/> | Временной интервал. Выберите один из следующих вариантов: <br/>  Ежечасно (может отображаться до 25 часов) <br/>  Ежедневно (может отображаться до 31 дня) <br/>  Еженедельно (может отображаться до 12 недель) <br/>  Ежемесячно (может отображаться до 12 месяцев) <br/>  Если даты начала и окончания превышают максимальное допустимые значения для выбранного интервала, отображается только максимальное число значений (от даты начала). Например, если вы выбираете дневной интервал с датой начала 7/7/2015 и датой окончания 2/28/2015, данные отображаются за 8/7/7/2015 с 12:00 до 09:00 12:00 (то есть в общей сложности 31 день). <br/> |
|**Pool** <br/> |Полное доменное имя пула регистратора или пограничного сервера. Можно выбрать отдельный пул или выбрать **[Все]**, чтобы просмотреть данные для всех пулов. Данный раскрывающийся список заполняется автоматически на основе записей, имеющихся в базе данных. <br/> |
