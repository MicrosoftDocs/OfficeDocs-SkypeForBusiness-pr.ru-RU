# <a name="user-registration-report-in-skype-for-business-server"></a>Отчет о регистрации пользователей в Skype для бизнеса Server
 
**Сводка:** Узнайте об отчете о регистрации пользователей в Skype для бизнеса Server.
  
Отчет о регистрации пользователей содержит обзор действий по входу пользователей, в частности сведения о количестве пользователей, которые вошли в Skype для бизнеса Server в течение указанного периода времени (ежечасно, ежедневно, еженедельно, ежемесячно). Следует иметь в виду, что в отчете лишь сообщается, сколько людей входило в систему. В нем не указывается, какие люди входили в систему. Отчеты мониторинга не предоставляют сведения о том, какие конкретные пользователи используют Skype для бизнеса Server (а какие нет). Тем не менее с помощью отчета об активности пользователей можно получить грубую оценку пользовательской информации.
  
При предоставлении сведений о входах пользователей в систему в отчете о регистрации пользователей делается два важных разграничения. Во-первых, в отчете входы в систему разбиваются на две основные категории: внутренние и внешние входы в систему. Внутренние входы представляют пользователей, вошедших в систему с внутренней стороны брандмауэра организации (то есть, с компьютера, подключенного к корпоративной сети). Внешние входы представляют пользователей, которые вошли в систему за пределами брандмауэра через edge Server (например, пользователь, выбравший вход из интернет-кафе, считается внешним входом). Если необходимо узнать, сколько пользователей входит в систему с внешней стороны брандмауэра, эти сведения можно найти в отчете о регистрации пользователей.
  
Кроме того, в отчете о регистрации пользователей отмечается, сколько активных пользователей присутствовало в течение заданного периода времени. Активный пользователь — это пользователь, который принимал участие в сеансе обмена мгновенными сообщениями, принимал участие в собрании Skype для бизнеса Server, сделал или получил телефонный звонок или использовал Skype для бизнеса Server в течение этого периода времени. Такой пользователь отличается от пользователя, вошедшего в систему, но не проявлявшего никакой активности в системе.
  
## <a name="accessing-the-user-registration-report"></a>Доступ к отчету о регистрации пользователей

Доступ к отчету о регистрации пользователей предоставляется только на домашней странице отчетов мониторинга. Отчет о регистрации пользователей не связан с какими-либо другими отчетами.
  
## <a name="making-the-best-use-of-the-user-registration-report"></a>Оптимальное использование отчета о регистрации пользователей

После развертывания Skype для бизнеса Server часто заданный вопрос: как узнать, действительно ли пользователи используют эту новую технологию? Хотя в этом отношении есть определенные ограничения, отчет о регистрации пользователей позволяет ответить на этот вопрос. Чтобы определить, используют ли пользователи Skype для бизнеса Server, необходимо сделать две вещи. Во-первых, нужно получить значение показателя уникальных входов пользователей из отчета о регистрации пользователей. Это значение указывает, сколько отдельных пользователей вошли в Skype для бизнеса Server.
  
Для сравнения, показатель общего числа входов в систему показывает, сколько раз кто-либо вошел в Skype для бизнеса Server. Например, предположим, что Кен Myer вошел в Skype для бизнеса Server пять раз в один день. В этом случае для Кен Майера было бы подсчитано пять отдельных сеансов входа в показателе общего числа входов, но лишь один зарегистрированный пользователь в показателе уникальных зарегистрированных пользователей. Аналогично, пользователи нередко входят в систему с нескольких устройств или из нескольких мест. Например, пользователь может войти в систему на настольном компьютере, ноутбуке и на IP-телефоне, который автоматически входит в Skype для бизнеса Server. В этом примере имеется один уникальный пользователь с тремя входами в систему.
  
Для дальнейшего пояснения разницы между общим количеством входов и уникальными входами рассмотрим входы в систему для заданного периода времени, приведенные в следующей таблице.
  
|**Пользователь**|**Время входа**|
|:-----|:-----|
|Кен Майер  <br/> |07.07.2015 8:45  <br/> |
|Кен Майер  <br/> |07.07.2015 8:46  <br/> |
|Пилар Аккерман  <br/> |07.07.2015 9:17  <br/> |
|Кен Майер  <br/> |07.07.2015 9:22  <br/> |
|Пилар Аккерман  <br/> |07.07.2015 9:31  <br/> |
   
Обратите внимание, что общее количество входов в систему равно пяти; однако существует только два уникальных регистрировавшихся пользователя: Кен Майер (входивший в систему три раза) и Пилар Аккерман (заходивший в систему дважды). В этом разница между входами в систему и уникальными регистрировавшимися пользователями.
  
Помимо количества уникальных входа в учетную запись, необходимо знать общее количество пользователей, которым была включена возможность работы со Skype для бизнеса Server. Это значение можно получить, открыв командную командную Windows PowerShell Skype для бизнеса Server:
  
```PowerShell
(Get-CsUser).Count
```

Если при предыдущей команде возвращается значение 1236, а показатель уникальных пользователей для входа возвращает среднее значение 667, это указывает на то, что чуть более половины пользователей, в которых работает Skype для бизнеса, фактически ежедневно вошел в систему (т. е. 667 разделено на 1236, что примерно на 54 %).
  
> [!CAUTION]
> Следует иметь в виду, что показателями входов учитываются пользователи, которые на самом деле входили в систему в указанный период времени. Они не учитывают пользователей, которые уже были зарегистрированы в системе. Например, если показатель количества уникальных регистрировавшихся пользователей равен 667 и имеется 1236 пользователей, это свидетельствует о том, что около половины пользователей входят в систему. Однако предположим, что 300 пользователей уже находились в системе на момент, когда началась проверка данных о регистрации. Это означает, что на самом деле в Skype для бизнеса Server вошел почти 1000 пользователей, что означает, что почти 80 % пользователей вошли в систему. 
  
Следует также сравнить значение показателя количества уникальных регистрировавшихся пользователей со значением показателя количества уникальных активных пользователей. Показатель "Уникальные активные пользователи" указывает, сколько уникальных пользователей фактически использовали Skype для бизнеса Server: они сделали телефонный звонок, присоединились к собранию Skype для бизнеса Server или участвовали в сеансе im. Это полезная информация, так как Skype для бизнеса Server можно настроить на автоматический запуск при каждом запуске Windows пользователем. По этой причине у вас может быть большое количество пользователей, которые автоматически входить в Skype для бизнеса при каждом входе в Windows, но затем никогда не используют Skype для бизнеса Server в течение этого периода времени.
  
Показатель "Уникальные активные пользователи" также предоставляет более значимые данные в организации, где пользователи обычно не выходят из Windows в конце дня. Вместо этого он просто блокирует свои компьютеры и оставляет Windows и Skype для бизнеса запущенным. В подобной ситуации каждый день может происходить всего несколько входов в систему, так как пользователи зарегистрировались несколько дней назад и с тех пор не выходили из системы. Однако уникальные активные пользователи сообщает, активно ли пользователи используют Skype для бизнеса или другой клиент Skype для бизнеса Server.
  
## <a name="filters"></a>Фильтры

Фильтры позволяют получить более конкретные наборы возвращаемых данных. Например, отчет о регистрации пользователей позволяет просматривать данные для всех пулов регистраторов и серверов, а также просматривать данные для отдельного пула. Можно также выбрать порядок группирования данных. В этом случае регистрации группируются по времени, дням, неделям или месяцам.
  
В следующей таблице приведены фильтры, которые можно использовать с отчетом о регистрации пользователей.
  
**Фильтры отчета о регистрации пользователей**

|**Название**|**Описание**|
|:-----|:-----|
|**From** <br/> |Начальные дата и время временного диапазона. Чтобы просмотреть данные по часам, введите начальные дату и время следующим образом:  <br/> 7.07.2015 13:00  <br/> Если не указать время начала, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Неделя всегда начинается с воскресения и заканчивается субботой.  <br/> |
|**Для** <br/> |Конечные дата и время временного диапазона. Чтобы просмотреть данные по часам, введите конечные дату и время следующим образом:  <br/> 7.07.2015 13:00  <br/> Если не указать время окончания, автоматически будет выбрано время 00:00 указанного дня. Чтобы просмотреть данные по дням, просто введите дату:  <br/> 7/7/2015  <br/> Для просмотра по неделям или по месяцам введите дату, выпадающую на любой день недели или месяца, который вы хотите просмотреть (это необязательно должен быть первый день недели или месяца):  <br/> 7/3/2015  <br/> Неделя всегда начинается с воскресения и заканчивается субботой.  <br/> |
|**Interval** <br/> | Временной интервал. Выберите один из следующих вариантов: <br/>  Ежечасно (может отображаться до 25 часов) <br/>  Ежедневно (может отображаться до 31 дня) <br/>  Еженедельно (может отображаться до 12 недель) <br/>  Ежемесячно (может отображаться до 12 месяцев) <br/>  Если даты начала и окончания превышают максимальное допустимые значения для выбранного интервала, отображается только максимальное число значений (от даты начала). Например, при выборе интервала "Ежедневно" с датой начала 7.07.2015 и датой окончания 28.02.2015 отображаются данные за 07.08.2015 с 12:00 до 07.09.2015 12:00 (то есть 31 день). <br/> |
|**Pool** <br/> |Полное доменное имя пула регистратора или пограничного сервера. Можно выбрать отдельный пул или выбрать **[Все]**, чтобы просмотреть данные для всех пулов. Данный раскрывающийся список заполняется автоматически на основе записей, имеющихся в базе данных. <br/> |
