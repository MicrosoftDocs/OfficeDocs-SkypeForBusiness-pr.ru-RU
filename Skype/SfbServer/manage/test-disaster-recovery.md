# <a name="disaster-recovery-testing-in-skype-for-business-server"></a>Тестирование аварийного восстановления в Skype для бизнеса Server

Выполните восстановление системы для Skype для бизнеса Server пула, чтобы проверить документированный процесс аварийного восстановления. Этот тест смоделирует полный сбой оборудования для одного сервера и поможет гарантировать, что ресурсы, планы и данные доступны для восстановления. Попробуйте каждый месяц вращать фокус теста, чтобы организация каждый раз тестирует сбой другого сервера или другого оборудования. 

Обратите внимание, что расписание, в котором организации выполняют тестирование аварийного восстановления, будет отличаться. Очень важно, чтобы тестирование аварийного восстановления не игнорировали и не пренебрегали. 

Экспорт Skype для бизнеса Server топологии, политик и параметров конфигурации в файл. Помимо прочего, этот файл может быть использован для восстановления этих сведений в центральном хранилище управления после обновления, сбоя оборудования или некоторых других проблем, которые привели к потере данных.

Импорт Skype для бизнеса Server топологии, политик и параметров конфигурации в центральный магазин управления или на локальный компьютер, как показано в следующих командах: 

`Import-CsConfiguration -ByteInput <Byte[]> [-Force <SwitchParameter>] [-LocalStore <SwitchParameter>]`

`Import-CsConfiguration -FileName <String> [-Force <SwitchParameter>] [-LocalStore <SwitchParameter>]` 

Для подстановки производственных данных:

- Резервное копирование баз данных RTC и LCSLog с помощью SQL Server резервного копирования для сброса базы данных в устройство сброса файлов или ленты.
- Использование сторонних приложений резервного копирования для резервного копирования данных для файла или ленты.
- Используйте Export-CsUserData для создания XML-экспорта всей базы данных RTC.
- Используйте резервное копирование файловой системы или стороннее резервное копирование для резервного копирования контента собраний и журналов соответствия требованиям.
- Используйте средство Export-CsConfiguration командной строки для Skype для бизнеса Server параметров.

Первый шаг в процедуре сбой включает принудительное перемещение пользователей из производственного пула в пул аварийного восстановления. Это будет вынужденное перемещение, так как пул производства не будет доступен для переноса пользователя.

Процесс Skype для бизнеса Server фактически является изменением атрибута объекта учетной записи пользователя в дополнение к обновлению записи в базе данных RTC SQL. Эти данные можно восстановить с исходного устройства резервного копирования из производственного SQL Server с помощью стандартного процесса восстановления SQL Server или с помощью сторонних утилит резервного копирования и восстановления.

После восстановления этих данных пользователи могут эффективно подключаться к пулу аварийного восстановления и работать в обычном режиме. Чтобы пользователи могли подключаться к пулу аварийного восстановления, потребуется изменить запись DNS.

Производственный Skype для бизнеса будет ссылаться на клиентов с помощью автоконгурации и записей DNS SRV:

- SRV: _sip._tls.\<domain> /CNAME: SIP.\<domain>
- CNAME: SIP.\<domain> /cvc-pool-1.\<domain>

Чтобы облегчить сбой, эта запись CNAME должна быть обновлена для ссылки на drOCSPool FQDN:

- CNAME: SIP.<domain> /DROCSPool.\<domain>
- SIP.\<domain>
- AV.\<domain>
- webconf.\<domain>
