# <a name="add-av-mcu-pool"></a>Добавление пула A/V MCU
 
Все серверы переднего плана Enterprise Edition на центральном сайте, не поддерживающим Объединенную службу конференц-связи, могут использовать один и тот же автономный пул конференций/V. Для каждого пула конференций (/V) необходимо указать полное доменное имя (FQDN) для пула, а также будет ли он иметь только один сервер для конференц-связи A/V или несколько серверов для конференц-связи с балансировкой нагрузки.
  
> [!IMPORTANT]
> Невозможно преобразовать односерверный пул в пул с несколькими серверами. Если позже вы решите, что вашей организации требуется многосерверный пул, необходимо удалить пул с одним сервером, а затем добавить пул с несколькими серверами. 
  
> [!TIP]
> Если вы планируете реализовать пул конференц-связи A/V в будущем, выберите **несколько пулов компьютеров**. Хотя пул определяется как несколько компьютеров с балансировкой нагрузки, можно создать пул из одного компьютера и присвоить этому компьютеру полное доменное имя пула. Когда вы будете готовы добавить другие компьютеры в пул позже, необходимо снова запустить Topology Builder, чтобы определить нового участника пула, опубликовать новую топологию и настроить новый участник пула конференц-связи с помощью мастера развертывания Skype для бизнеса Server. Пулы серверов конференций/V являются уникальными, так как они не требуют подсистемы балансировки нагрузки для создания пула. Подсистема балансировки нагрузки для пула конференц-связи с внутренними устройствами и не нуждается в дополнительном оборудовании.
