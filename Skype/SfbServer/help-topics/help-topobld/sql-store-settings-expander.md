# <a name="sql-store-settings-expander"></a>Расширитель параметров хранилища SQL
 
Чтобы изменить свойства базы данных SQL Server, необходимо изменить экземпляр SQL Server базы данных. Вы не можете использовать диалоговое окно **Изменение свойств** для выполнения таких задач, как перемещение базы данных сервера архивации с одного компьютера на другой. Кроме того, диалоговое окно **Edit Properties** нельзя использовать для изменения экземпляра SQL Server, в котором расположен центральный магазин управления.
  
## <a name="editing-the-properties-of-a-sql-server-database"></a>Редактирование свойств SQL Server базы данных

Чтобы изменить экземпляр SQL Server, используемый любой базой данных, кроме центра управления, выполните следующую процедуру в Topology Builder:
  
### <a name="to-modify-an-instance-of-sql-server"></a>Изменение экземпляра SQL Server

1. Выберите соответствующую базу данных в узле **Хранилища SQL**, а затем щелкните элемент **Изменить свойства**.
    
2. В диалоговом окне **Изменение свойств** выполните одно из следующих действий:
    
   - Чтобы использовать экземпляр по SQL Server по умолчанию, выберите экземпляр **по** умолчанию и нажмите **кнопку ОК**.
    
   - Чтобы использовать именованный экземпляр базы данных, выберите элемент **Именованный экземпляр**, введите имя экземпляра в текстовом поле и нажмите кнопку **ОК**. Следует ввести только имя экземпляра (например, АрхивированиеInstance), а не весь SQL Server пути.
    
Когда вы работаете в диалоговом окне **Edit Properties,** Topology Builder не будет проверять, является ли экземпляр базы данных допустимым. Например, если вы непреднамеренно введите имя экземпляраArchivingInstanec, а затем нажмите **кнопку ОК,** то топология Builder примет этот недопустимый экземпляр. Прежде чем опубликовать эту топологию, необходимо исправить эту ошибку: если экземпляр SQL Server не удается найти, то topology Builder не создаст этот экземпляр для вас.
