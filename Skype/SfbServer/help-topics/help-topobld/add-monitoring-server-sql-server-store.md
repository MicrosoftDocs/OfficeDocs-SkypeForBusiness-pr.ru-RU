# <a name="add-monitoring-server-sql-server-store"></a>Добавление хранилища SQL Server для сервера мониторинга

Для хранения данных мониторинга сервер мониторинга должен иметь поддерживаемый 64 разрядный выпуск базы данных SQL Server. Вы можете выбрать ранее определенную базу данных SQL Server для мониторинга или определить новую базу данных SQL Server, указав полное доменное имя (FQDN) сервера, на котором будет размещаться база данных SQL Server, а также экземпляр SQL Server, который необходимо использовать для новой базы данных SQL Server (который может быть экземпляром по умолчанию). или именованный экземпляр, который вы указали.

Подробные сведения о поддержке SQL Server можно найти в статье по поддержке [баз данных и кластеризации](https://technet.microsoft.com/library/e05d0032-bbea-4e61-987d-d07b1c045fd5.aspx) в документации по поддержке. Подробные сведения о базе данных мониторинга, включая совмещенное расположение базы данных мониторинга, можно найти в статье [Поддерживаемые серверы](https://technet.microsoft.com/library/3be990a1-5485-4b83-b73f-947ac97821f9.aspx) в документации по поддержке,[планированию мониторинга отслеживания](https://technet.microsoft.com/library/26cead5a-183c-42f1-a4b0-0e8d61c6159d.aspx) в документации по планированию и [размещении данных SQL Server и файла журнала](https://technet.microsoft.com/library/67aa525b-8aa3-474f-827e-8e1d4697f30f.aspx) в документации по развертыванию.

> [!NOTE]
> Если учетная запись, использованная для публикации топологии, обладает достаточными правами и разрешениями, при публикации топологии вы можете создать базу данных мониторинга. Эту базу данных можно создать и позднее — в рамках процедуры установки. > для установки и развертывания баз данных на сервере SQL Server для мониторинга необходимо быть членом группы администраторов SQL Server для сервера SQL Server, на котором устанавливаются файлы базы данных. Если вы не являетесь участником группы администраторов SQL Server, необходимо добавить запрос в группу, пока не будут развернуты файлы базы данных. Если вы не можете быть участником группы администраторов, необходимо предоставить администратору базы данных SQL Server скрипт для настройки и развертывания баз данных. Дополнительные сведения о правах и разрешениях, требуемых для выполнения описанных процедур, см. в разделе [Deployment Permissions for SQL Server](https://technet.microsoft.com/library/56ea0c02-bcf5-4d45-aa13-570531c29074.aspx) документации по развертыванию.
