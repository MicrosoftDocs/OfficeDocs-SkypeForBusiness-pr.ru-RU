# <a name="persistent-chat-general-settings-expander"></a>Расширитель общих параметров сохраняемого чата
 
Вы редактируете общие параметры для пула стойких чат-серверов или настойчивых серверов чата, настроив или определив эти свойства: 
  
 **Общие**
  
- **FQDN:** Изменить этот параметр, чтобы определить полностью квалифицированное доменное имя вашего сохраняемого сервера чата или пула постоянных чат-серверов
    
- **Отображение имени пула сохраняемого** чата. Определите этот параметр, чтобы обеспечить удобное для пользователя и читаемое для пользователя параметр для сервера или пула. Этот параметр облегчит пользователям связывать данный постоянный чат-сервер или пул сохраняемого сервера чата на основе имени отображения, а не более сложно понять полностью квалифицированное доменное имя.
    
- **Порт непрерывных чатов**. Укажите порт, который используется для непрерывных чатов.
    
Вы редактируете **параметры Associations** для настойчивых чат-серверов или пула настойчивых чат-серверов, настраивая или определяя эти свойства:
  
 **Ассоциации**
  
- **SQL Server магазине:** выберите SQL Server и необязательный экземпляр с именем из списка.
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр.
    
- Чтобы включить **зеркальное SQL Server** для основного магазина, выберите SQL Server для хранения зеркального SQL Server.
    
    Если вы решили включить SQL Server хранения зеркального хранения, выберите магазин и экземпляр из магазина зеркального **SQL Server списка**.
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр.
    
- Выберите свидетель **SQL Server** использования, чтобы включить автоматическую проверку сбой при автоматическом сбойе основного SQL Server магазина.
    
    Если вы решили включить SQL Server зеркального свидетеля для автоматического сбой, выберите магазин и экземпляр из списка.
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр для хранилища-свидетеля.
    
- Выберите хранилища **резервного копирования SQL Server,** чтобы включить поле аварийного восстановления, если вы хотите включить SQL Server аварийного восстановления
    
    Если включена возможность аварийного восстановления, выберите хранилище и экземпляр в списке **Резервное хранилище SQL Server**.
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр.
    
- Если **вы** хотите включить зеркальное SQL Server для резервного копирования, выберите SQL Server SQL Server хранения зеркального хранения.
    
    Если вы решили включить резервное копирование SQL Server хранения зеркального хранения, выберите хранилище и экземпляр из зеркала резервного копирования SQL Server **хранения.**
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр.
    
- Выберите свидетель **SQL Server** использования, чтобы включить автоматическую проверку сбой, если необходимо автоматическое сбой резервного копирования SQL Server хранилище.
    
    Если вы решили включить SQL Server зеркального свидетеля для автоматического сбой, выберите магазин и экземпляр из списка.
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр для хранилища-свидетеля.
    
- Установите флажок **Включить соответствие**, если необходимо включить использование базы данных соответствия.
    
    Если включено соответствие требованиям, выберите хранилище и экземпляр в списке **Совместимое хранилище SQL Server**.
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр.
    
- Если вы **хотите** включить зеркальное SQL Server для хранения зеркального зеркального окна, выберите SQL Server хранения.
    
    Если вы решили включить зеркальное SQL Server хранения, выберите хранилище и экземпляр из зеркала SQL Server **хранения.**
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр.
    
- Выберите свидетель **SQL Server** использования, чтобы включить автоматическую проверку сбойной работы, если необходимо автоматическое сбой SQL Server хранения.
    
    Если вы решили включить SQL Server зеркального свидетеля для автоматического сбой, выберите магазин и экземпляр из списка.
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр для хранилища-свидетеля.
    
- Если включено соответствие требованиям, выберите хранилище и экземпляр в списке **Зеркало резервного совместимого хранилища SQL Server**.
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр.
    
- Если вы **хотите** включить зеркальное SQL Server для хранения зеркального зеркального окна, выберите SQL Server хранения.
    
    Если вы решили включить зеркальное SQL Server хранения, выберите хранилище и экземпляр из списка SQL Server **зеркала хранения.**
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр.
    
- Выберите свидетель **SQL Server** для автоматического сбойного окна, если необходимо автоматическое сбой резервного копирования SQL Server хранилище.
    
    Если вы решили включить SQL Server зеркального свидетеля для автоматического сбой, выберите магазин и экземпляр из списка.
    
    Щелкните элемент **Создать**, чтобы определить новое хранилище SQL Server и необязательный экземпляр для хранилища-свидетеля.
    
- **Хранилище файлов** Выберите расположение магазина файлов из списка или нажмите **кнопку New,** чтобы создать новый файловый магазин.
    
  При нажатии кнопки **ОК** выполняется принятие и сохранение изменений в этом диалоговом окне.
  
  При нажатии кнопки **Отмена** изменения отменяются, а диалоговое окно закрывается.
  
  При нажатии кнопки **Справка** отображается данный экран справки.
