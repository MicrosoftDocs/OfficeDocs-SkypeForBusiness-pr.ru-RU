# <a name="publish-topology-create-mirror-database-page"></a>Страница публикации топологии для создания зеркальной базы данных
 
Это предварительная документация, которая может быть измениться. Пустые разделы включены в качестве заполнителей.
  
Построитель топологий проансирует все параметры конфигурации в определенном документе топологии. Если вы настроили зеркальные SQL Server баз данных, отображается экран, на который будут отображаться зеркальное хранилище, основное хранилище, путь к файловому файлу и пути к базе данных. Чтобы создать зеркальные базы данных и путь к файловому ресурсу общего доступа, установите флажок, соответствующий хранилищу зеркальных баз данных. Если не требуется создавать и развертывать базу данных в определенном SQL Server, необходимо сместить этот SQL Server.
  
 Если вы решили в данный момент не устанавливать одну или все базы данных, снимите флажки для тех баз данных, которые устанавливать не нужно. Позднее вы решите использовать  установку базы данных, щелкнув правой кнопкой мыши большинство элементов узла в построителье топологий.
