# <a name="publish-topology-create-mirror-database-page"></a>Страница публикации топологии для создания зеркальной базы данных
 
Это Предварительная документация, которая может быть изменена. Пустые разделы включены в качестве заполнителей.
  
Построитель топологий анализирует все параметры конфигурации в определенном документе топологии. Если вы настроили зеркальные базы данных SQL Server, вы увидите экран, на котором отображается зеркальное хранилище, первичное хранилище, путь к общему файловому ресурсу и пути к базам данных. Чтобы создать зеркальные базы данных и путь к файловому ресурсу общего доступа, установите флажок, соответствующий хранилищу зеркальных баз данных. Если вы не хотите создавать и развертывать базу данных на определенном сервере SQL Server, снимите флажок для зеркального отображения базы данных.
  
 Если вы решили в данный момент не устанавливать одну или все базы данных, снимите флажки для тех баз данных, которые устанавливать не нужно. Позже вы решили использовать **базу данных установки** , щелкнув правой кнопкой мыши большинство элементов узла в построителе топологий.
