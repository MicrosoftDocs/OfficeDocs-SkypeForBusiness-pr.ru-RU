# <a name="publish-topology-create-mirror-database-page"></a>Страница публикации топологии для создания зеркальной базы данных
 
Это предварительная редакция документации и она может меняться. Пустые разделы добавлены в качестве заполнителей.
  
Topology Builder анализирует все параметры конфигурации в определенном документе топологии. Если вы настроили зеркальные базы данных SQL Server, вы увидите экран, на котором отображаются зеркальное хранилище, первичное хранилище, путь к общей папке и пути к базам данных. Чтобы создать зеркальную базу данных и путь к общей папке, установите флажок, связанный с хранилищем зеркальной базы данных. Если вы не хотите создавать и развертывать базу данных на определенном сервере SQL Server, вы очищаете флажок для зеркальной базы данных.
  
 Если вы выберете вариант не устанавливать одну или все из этих баз данных сейчас, вы очищаете базы данных, которые не будут установлены. Позже вы решите использовать **базу данных для установки** , щелкнув правой кнопкой мыши большинство элементов узла в построителе топологии.
