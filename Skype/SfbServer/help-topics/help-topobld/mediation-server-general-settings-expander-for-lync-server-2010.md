# <a name="mediation-server-general-settings-expander-for-lync-server-2010"></a>Расширитель общих настроек сервера-посредника для Lync Server 2010

В этом диалоговом окне вы изменяете свойства серверов-посредников. По левой стороне содержится набор быстрых ссылок на общие параметры, параметры узла следующего перехода и параметры шлюза в ТСОП. В каждом разделе содержатся приведенные ниже параметры.

 **Общие**сведения:

- **Полное доменное**имя: вы редактируете полное доменное имя сервера-посредника

- **Связи**: установите флажок **сопоставить пограничный пул (для компонентов мультимедиа)** и выберите Пограничный сервер или пограничный пул, который будет использоваться сервером-посредником в качестве пути к носителю для внешнего доступа.

  **Узел следующего перехода**.

- **Выбор следующего прыжка**: выберите из списка сервер переднего плана или интерфейсный пул, который будет использоваться в качестве пути к серверу-посреднику для связи с развертыванием.

  **Шлюз ТСОП**.

  **Шлюз ТСОП сервера-посредника**.

- **Порты прослушивания**: Определите порты, которые сервер-посредник должен прослушать. Можно определить порт для протокола TLS или протокола **TLS** **или протокола**управления транспортом. Чтобы запись порта TCP была доступна, необходимо установить флажок **включить порт TCP**.

    > [!IMPORTANT]
    > Обратитесь к документации и параметрам конфигурации своего шлюза ТСОП, чтобы определить, требуется ли включить и задать значения портов для TLS, для TCP или для обоих этих протоколов. TLS — это более безопасный протокол, использующий сертификаты для шифрования трафика между сервером-посредником и шлюзом PSTN. TLS поддерживают не все шлюзы ТСОП.

- Отображается список магистралей и шлюзов SIP, определенных и настроенных в развертывании. Если записи отсутствуют, магистрали SIP или шлюзы в ТСОП в данном развертывании отсутствуют. Вы определяете и настраиваете магистральы и шлюзы в разделе **Общие компоненты** в построителе топологий.
