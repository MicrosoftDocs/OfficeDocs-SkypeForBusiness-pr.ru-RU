# <a name="mediation-server-general-settings-expander-for-lync-server-2010"></a>Расширитель общих параметров сервера-посредника для Lync Server 2010

В этом диалоговом оке можно изменить свойства серверов-посредников. По левой стороне содержится набор быстрых ссылок на общие параметры, параметры узла следующего перехода и параметры шлюза в ТСОП. В каждом разделе содержатся приведенные ниже параметры.

 **Общие:**

- **Полное доменное имя:** полное доменное имя сервера-посредника

- **Связи:** выберите для сервера-посредника в качестве пути к мультимедиа для внешнего доступа, чтобы использовать его в качестве пути к носители для внешнего доступа, выберите "Связывающие" (для компонентов **мультимедиа)** и "Сервер-посредник" или "Пул".

  **Узел следующего перехода**.

- **Выбор следующего прыжка:** выберите в списке сервер переднего сервера или пул переднего сервера, который будет использовать в качестве пути для сервера-посредника для связи с развертыванием.

  **Шлюз ТСОП**.

  **Шлюз ТСОП сервера-посредника**.

- **Порты прослушивания:** определите порты, которые будет прослушивать сервер-посредник. Можно определить порт для **протокола TLS** или транспортного уровня, **TCP** или протокола управления транспортировкой. Чтобы запись порта для TCP была доступна, необходимо выбрать пункт "Включить **TCP-порт".**

    > [!IMPORTANT]
    > Обратитесь к документации и параметрам конфигурации своего шлюза ТСОП, чтобы определить, требуется ли включить и задать значения портов для TLS, для TCP или для обоих этих протоколов. TLS — это более безопасный протокол, использующий сертификаты для шифрования трафика между сервером-посредником и шлюзом STN. TLS поддерживают не все шлюзы ТСОП.

- Отображается список магистралей и шлюзов SIP, определенных и настроенных в развертывании. Если записи отсутствуют, магистрали SIP или шлюзы в ТСОП в данном развертывании отсутствуют. Магистрали и шлюзы определяются и настраиваются в области общих **компонентов** в построителье топологий.
