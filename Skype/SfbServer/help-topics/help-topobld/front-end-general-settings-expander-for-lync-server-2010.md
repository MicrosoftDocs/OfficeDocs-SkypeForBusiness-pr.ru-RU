# <a name="front-end-general-settings-expander-for-lync-server-2010"></a>Расширитель общих параметров сервера переднего плана для Lync Server 2010

Вы изменяете свойства внешнего сервера или пула переднего плана, изменяя и настраивая следующие атрибуты. Страница конфигурации разделяется на следующие разделы:

 **Общие**

- **FQDN**: полное доменное имя сервера переднего плана или пула переднего плана.

- Выберите параметр **использовать все настроенные IP-адреса** , чтобы использовать все адреса, настроенные на сервере переднего плана или в пуле внешних интерфейсов.

    > [!IMPORTANT]
    > Не выбирайте этот параметр, если вы выберие сервер по распечатку на сервере переднего плана или в пуле внешних интерфейсов. Серверам обновлений и внешним серверам необходимы выделенные IP-адреса для связи.

- Выберите пункт **ограничить использование службы до выбранных IP-адресов** и введите IP-адрес **основного IP-адреса** сервера переднего плана или связи с пулом с помощью других возможностей развертывания. Тип в **IP-адресе PSTN** IP-адрес, связанный с сервером-посредником.

    **Функции и возможности**

- **Конференции**: установите флажок, если вы хотите, чтобы в вашем развертывании были доступны функции для проведения конференций. Конференции включают в себя звук, видео, общий доступ к приложениям, общий доступ к рабочему столу и веб-конференции. Вам потребуется создать и связать сервер Office Web Apps для веб-конференций (определенный позже на этой странице свойств).

- Если вы выбрали Конференц **-связь с телефонным подключением (PSTN)** , можно выбрать один из них. Установите флажок, чтобы включить функции конференц-связи с телефонным подключением.

- Установите флажок **корпоративный голос** , если вы хотите развернуть функции, чтобы включить Lync Server 2013 в качестве телефонной голосовой системы с помощью технологий передачи голоса по протоколу IP (VoIP). в соответствии с разработкой и требованиями, в том числе для развертывания трубок с телефонным подключением, магистральных каналов SIP или подключения по коммутируемой телефонной сети с помощью сервера исправлений, шлюзов PSTN и IP-УАТС. Дополнительные сведения о корпоративной голосовой связи можно найти в разделе [корпоративный голос](https://technet.microsoft.com/library/c9da8099-6f4f-4346-ac67-f041bb96072c.aspx) и [планирование корпоративной голосовой связи в Skype для бизнеса Server 2015](../../plan-your-deployment/enterprise-voice-solution/enterprise-voice.md)

    **Связи**

- **Хранилище SQL Server**: полное доменное имя SQL Server (и, при необходимости, именованного экземпляра), связанное с внешним сервером или пулом переднего плана. Вы выбираете хранилище SQL Server из списка или создаете новое хранилище SQL Server, нажав кнопку **создать** .

- **Хранилище файлов**: вы выбираете полное доменное имя сервера и общий доступ (в формате `\\<FQDN of server>\<share name>`), который будет использоваться в качестве хранилища файлов для общих файлов, которые создаются и используются для репликации, каталогов конференций и других целей в Lync Server 2013. Для выбора хранилища файлов из списка или создания нового хранилища файлов нажмите кнопку **создать**.

- Установите флажок **сопоставить сервер архивации** , чтобы включить сервер архивации для этого сервер переднего плана или пул переднего плана. После установки флажка вы можете выбрать существующий сервер архивирования из списка или нажать кнопку **создать** , чтобы создать определения для нового сервера архивирования.

- Установите флажок **сопоставить сервер мониторинга** , чтобы включить сервер мониторинга для этого сервера переднего плана или пула переднего плана. После установки флажка вы можете выбрать существующий сервер мониторинга из списка или нажать кнопку **создать** , чтобы создать определения для нового сервера мониторинга.

- Установите флажок **пул граничного пула (для компонентов мультимедиа** ), чтобы включить пограничный сервер для этого сервера переднего плана или пула переднего интерфейса. После установки флажка вы выбираете существующий пограничный сервер или пул из списка или нажмите кнопку **создать** , чтобы создать определения для нового сервера пограничного или пула.

  **Устойчивость**

- Установите флажок **сопоставленный пул регистратора резервных копий** , чтобы выбрать из списка сервер переднего плана или интерфейс переднего плана, который будет являться регистратором резервных копий (то есть сервер переднего плана или пул переднего плана, назначенный дополнительным регистратором в случае сбоя основного сервера).

- Если вы выбрали сопоставленный пул регистратора архивации и выбрали регистратор архивации, вы можете установить флажок для **автоматического перехода на другой ресурс и восстановления размещения для голоса**. Теперь вы можете определить числовые свойства для внутренних (в секундах) и **интервалов (в секундах)** для **обнаружения отказов голосовой связи** . Подробные сведения можно найти в разделе [Планирование обеспечения устойчивости для корпоративной голосовой связи](https://technet.microsoft.com/library/ca116700-1055-4ca5-9b87-4c7f380c3655.aspx)

  **Веб-службы**

- Чтобы настроить **внутренние веб-службы**, вы определяете **порты для прослушивания** **http** и **HTTPS**. По умолчанию это TCP порт 80 и TCP + порт 443 соответственно. Кроме того, вы можете настроить **опубликованные порты** для **http** и **HTTPS**. По умолчанию это TCP порт 80 и TCP + порт 443 соответственно. С учетом внутренней конфигурации веб-служб и использования подсистем балансировки нагрузки (балансировки нагрузки для оборудования и балансирования нагрузки DNS) настройте значения портов, чтобы определить прослушивающий и опубликованные порты.

    > [!IMPORTANT]
    > Внутренние веб-службы и определенные прослушиваемые и опубликованные порты предназначены для внутренних клиентов и устройств. Внешние клиенты и устройства используют внешние веб-службы с прослушиванием и опубликованными портами, а также определенные полные доменные имена внешних веб-служб (FQDN).

- Чтобы настроить **внешние веб-службы**, вы определяете **порты для прослушивания** **http** и **HTTPS**. По умолчанию это TCP порт 80 и TCP + порт 443 соответственно. Кроме того, вы можете настроить **опубликованные порты** для **http** и **HTTPS**. По умолчанию это TCP порт 80 и TCP + порт 443 соответственно. С учетом внутренней конфигурации веб-служб и использования подсистем балансировки нагрузки (балансировки нагрузки для оборудования и балансирования нагрузки DNS) настройте значения портов, чтобы определить прослушивающий и опубликованные порты.

    > [!IMPORTANT]
    > Внешние веб-службы и определенные прослушиваемые и опубликованные порты предназначены для внешних клиентов и устройств. Внешние клиенты и устройства используют внешние веб-службы, прослушивающие и опубликованные порты, обычно определяемые обратным прокси-сервером, а также определенными внешними доменными именами (FQDN) внешних веб-служб. Отношение полного доменного имени внешней веб-службы и простых адресов URL определяет URL-адреса, с помощью которых внешние клиенты будут получать доступ к службам, доступным для внешних пользователей и устройств. Дополнительные сведения об простых URL-адресах можно найти в разделе [планирование простых URL-адресов](https://technet.microsoft.com/library/20e4f4b6-b7ff-4297-b00d-d1211ee800ac.aspx).

  **Сервер-посредник**

- Чтобы настроить свойства **сервера-исправления** для размещенного сервера-посредника (то есть сервер-посредник, развернутый на сервере переднего плана или в пуле переднего плана), выберите вариант сервер размещенных **исправлений включен**.

- Чтобы определить **порты прослушивания** для размещенного сервера-посредника, введите значение **TLS** и **TCP** -порт, на котором размещается прослушиваемый сервер. По умолчанию протокол TLS определен как TCP порт 5067.

- Чтобы определить TCP-порт для сервера-посредника, установите флажок **включить TCP-порт** . По умолчанию сервер-посредник использует протокол TLS (Transport Layer Security) для протокола TCP. Порты TCP доступны только в том случае, если включен параметр включить порт TCP.

    > [!NOTE]
    > Это необязательный параметр, и вы должны обратиться к требованиям шлюза или КТСОП, чтобы определить, требуется ли это. Значение порта TCP по умолчанию: 5068.

- Вы определяете магистральные канала, связанные с размещенным сервером исправлений. Если вы уже задали магистральные линии связи, их можно будет сопоставить с сервером-посредником.

    Если у вас есть несколько шлюзов, связанных с сервером-участником, вы можете указать шлюз по умолчанию, выбрав шлюз, который вы хотите использовать по умолчанию, и нажать кнопку **использовать по умолчанию**. Если вы решите удалить текущий шлюз по умолчанию, выберите шлюз и нажмите кнопку " **отменить выбор по умолчанию**".

> [!IMPORTANT]
> Если вы вносите изменения в свойства в этом диалоговом окне, необходимо опубликовать топологию и запустить мастер развертывания Skype для бизнеса Server на всех задействованных серверах. После публикации новой топологии на экране "Сводка успешно опубликованных данных о публикации топологии" для вас будет указан список серверов, на которых должен быть запущен мастер развертывания Skype для бизнеса Server. Подробнее о публикации обновленной топологии см. в разделе [Публикация топологии](https://technet.microsoft.com/library/3b5a744b-b3a8-4538-a55e-e2e4f72dff47.aspx). Подробные сведения о мастере развертывания Skype для бизнеса Server можно найти в разделе [средства администрирования Lync Server](https://technet.microsoft.com/library/9b006f93-4f3d-461d-89b8-e80a34fdb3c5.aspx).

Нажмите кнопку **ОК** , чтобы сохранить изменения и внести их в документ Topology.

Нажмите кнопку **Отмена** , чтобы отменить изменения и закрыть **свойства изменения** для внешнего сервера или пула переднего плана.

Нажмите кнопку **Справка** , чтобы прочитать этот раздел справки.
