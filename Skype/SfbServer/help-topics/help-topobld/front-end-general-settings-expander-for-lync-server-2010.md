# <a name="front-end-general-settings-expander-for-lync-server-2010"></a>Расширитель общих параметров сервера переднего плана для Lync Server 2010

Свойства сервера переднего сервера или пула переднего сервера можно изменять, изменив или настроив следующие атрибуты. Страница конфигурации разделена на следующие разделы.

 **Общие**

- **Полное доменное имя**: полное доменное имя сервера переднего сервера или пула переднего сервера.

- Выберите **"Использовать все настроенные IP-адреса** для использования всех адресов, настроенных на сервере переднего сервера или в пуле переднего сервера".

    > [!IMPORTANT]
    > Этот параметр не следует выбирать, если сервер-посредник вы выполнив в качестве сервера-посредника на сервере переднего сервера или в пуле переднего сервера. Для связи серверам-посредникам и серверам переднего сервера требуются выделенные IP-адреса.

- Выберите **ограничение использования** службы выбранными IP-адресами и введите IP-адрес основного **IP-адреса** для связи сервера переднего сервера или пула переднего сервера с остальной частью развертывания. Введите **IP-адрес PSTN,** связанный с сервером-посредником.

    **Компоненты и функции**

- **Conferencing**: Select the check box if you want conferencing features in your deployment. Аудио- и видеоконференцию, общий доступ к приложениям, общий доступ к рабочему столу и веб-аудиоконференцию. Вам потребуется создать и связать сервер Office Web Apps для веб-conferencing (определяется далее на этой странице свойств).

- Если выбрана конференц-связь, можно включить **Конференц-связь с телефонным подключением (ТСОП)**. Установите этот флажок, чтобы включить компоненты конференц-связи для использования телефонных подключений.

- Если вы собираетесь развернуть **lync** Server 2013 в качестве телефонной голосовой системы с использованием технологий VoIP, включите возможность развертывания телефонных телефонов, магистральных телефонов SIP или телефонных сетей общего законодательства с помощью сервера-посредника, шлюзов PSTN и IP-PBX в комбинации или отдельно в зависимости от конструкции и требований. Корпоративная голосовая связь For detail on Корпоративная голосовая связь, see [Корпоративная голосовая связь](https://technet.microsoft.com/library/c9da8099-6f4f-4346-ac67-f041bb96072c.aspx) and [Plan for Корпоративная голосовая связь in Skype for Business Server 2015](../../plan-your-deployment/enterprise-voice-solution/enterprise-voice.md)

    **Связи**

- **SQL Server хранения**: FQDN сервера SQL Server (и, при желании, именоваемого экземпляра), связанного с сервером переднего сервера или пулом переднего сервера. Выберите хранилище SQL Server в списке или создайте новое хранилище SQL Server, нажав кнопку **"Создать"**

- Хранилище **файлов:** вы выбираете FQDN сервера и общую папку (в формате), которая будет выступать в качестве расположения для общих файлов, которые Lync Server 2013 создает и использует для репликации, каталогов конференций и других `\\<FQDN of server>\<share name>` целей. Вы выбираете хранилище файлов из списка или создаете новое хранилище файлов, нажав кнопку **"Создать".**

- Выберите **"Связать сервер архива",** чтобы включить сервер архива для этого сервера переднего сервера или пула переднего сервера. После этого в списке необходимо выбрать существующий сервер архива  или нажать кнопку "Создать", чтобы создать определения для нового сервера архива.

- Чтобы включить **сервер** мониторинга для этого сервера переднего сервера или пула переднего сервера, выберите "Связать сервер мониторинга". После этого в списке выберите существующий сервер мониторинга  или нажмите кнопку "Создать", чтобы создать определения для нового сервера мониторинга.

- Выберите **"Связать пул" (для** компонентов мультимедиа, чтобы включить для этого сервера переднего сервера или пула переднего ящика) поименуйте его. После этого в списке необходимо выбрать существующий сервер или  пул или нажать кнопку "Создать", чтобы создать определения для нового сервера или пула.

  **Устойчивость**

- Выберите  в списке связанный резервный пул регистратора, чтобы выбрать в списке сервер переднего сервера или пул переднего сервера, который будет резервным регистратором (то есть сервер переднего или переднего плановых серверов, назначенный в качестве дополнительного регистратора в случае сбойного основного сервера).

- Если выбрать пункт "Связанный пул резервного регистратора" и указать резервный регистратор, можно установить флажок **Автоматическая обработка отказов и восстановление после отказа для голосовой связи**. После этого можно определить числовые свойства **Интервал обнаружения отработки отказа голосовой связи (сек)** и **Интервал восстановления после сбоя голосовой связи (сек)**. Дополнительные сведения см. в разделе [Planning for Enterprise Voice Resiliency](https://technet.microsoft.com/library/ca116700-1055-4ca5-9b87-4c7f380c3655.aspx)

  **Веб-службы**

- Чтобы настроить **внутренние веб-службы**, необходимо определить **Порты прослушивания****HTTP** и **HTTPS**. По умолчанию используются значения TCP 80 и, соответственно, TCP 443. Кроме того, следует настроить **Опубликованные порты** для **HTTP** и **HTTPS**. По умолчанию используются порты TCP 80 и, соответственно, TCP 443. На основе конфигурации внутренних веб-служб и использования балансировщиков нагрузки (аппаратных балансировщиков и балансировки нагрузки DNS) измените значения портов для определения портов прослушивания и опубликованных портов.

    > [!IMPORTANT]
    > Внутренние веб-службы и определенные порты прослушивания и опубликованные порты предназначены для внутренних клиентов и устройств. Внешние клиенты и устройства используют порты прослушивания и опубликованные порты внешних веб-служб вместе определенным полным доменным именем внешних веб-служб.

- Чтобы настроить **внешние веб-службы**, необходимо определить **Порты прослушивания****HTTP** и **HTTPS**. По умолчанию используются значения TCP 80 и, соответственно, TCP 443. Кроме того, следует настроить **Опубликованные порты** для **HTTP** и **HTTPS**. По умолчанию используются порты TCP 80 и, соответственно, TCP 443. На основе конфигурации внутренних веб-служб и использования балансировщиков нагрузки (аппаратных балансировщиков и балансировки нагрузки DNS) измените значения портов для определения портов прослушивания и опубликованных портов.

    > [!IMPORTANT]
    > Внешние веб-службы и определенные порты прослушивания и опубликованные порты предназначены для внешних клиентов и устройств. Внешние клиенты и устройства используют порты прослушивания и опубликованные порты внешних веб-служб, как правило определенные обратным прокси-сервером, вместе определенным полным доменным именем внешних веб-служб. Отношение полного доменного имени внешних веб-служб и простых URL-адресов определяют URL-адреса, которые будут использоваться внешними клиентами для открытия служб, доступных внешним пользователям и устройствам. Дополнительные сведения о простых URL-адресах см. в разделе [Planning for Simple URLs](https://technet.microsoft.com/library/20e4f4b6-b7ff-4297-b00d-d1211ee800ac.aspx).

  **Сервер-посредник**

- Чтобы настроить  свойства сервера-посредника для совместимого сервера-посредника (то есть сервера-посредника, развернутого на сервере переднего сервера или в пуле переднего сервера), выберите сервер-посредник с совместимым **размещением.**

- Чтобы определить **порты прослушивания** для сервера-посредника, размещенного с выполняемой сервером-посредником, введите значение **TLS** и **TCP-порта,** которые прослушивает сервер-посредник. По умолчанию TLS определен как TCP-порт 5067.

- Чтобы определить значение порта TCP для сервера-посредника, необходимо выбрать значение для порта **enable TCP.** По умолчанию сервер-посредник использует протокол TLS по протоколу TCP. Порты TCP доступны, только если включен выбор порта TCP.

    > [!NOTE]
    > Это дополнительный параметр, поэтому чтобы определить необходимость его использования, ознакомьтесь с требованиями к используемому шлюзу или ТСОП. По умолчанию используется порт TCP 5068.

- Следует определить магистральные линии, которые сопоставлены с совмещенным сервером-посредником. Если магистральные линии связи уже заданы, их можно будет сопоставить с сервером-посредником.

    Если с сервером-посредником связано несколько шлюзов, можно указать шлюз по умолчанию, выбрав шлюз, который нужно сделать шлюзом по умолчанию, и нажав кнопку **"По умолчанию".** Если вы решите удалить текущий шлюз по умолчанию, выберите шлюз и нажмите кнопку **"Отоимите по умолчанию".**

> [!IMPORTANT]
> При внесении изменений в свойства этого диалоговых окно необходимо опубликовать топологию и запустить мастер развертывания Skype для бизнеса Server на всех затронутых серверах. После публикации новой топологии список затронутых серверов, на которых необходимо запустить мастер развертывания Skype для бизнеса Server, предоставляется в качестве ссылки на экране сводки успешной публикации топологии. Сведения о публикации обновленной топологии см. в разделе [Publish the Topology](https://technet.microsoft.com/library/3b5a744b-b3a8-4538-a55e-e2e4f72dff47.aspx). Подробные сведения о мастере развертывания Skype для бизнеса Server см. в [подпункте "Администрирование Lync Server".](https://technet.microsoft.com/library/9b006f93-4f3d-461d-89b8-e80a34fdb3c5.aspx)

Нажмите кнопку **ОК**, чтобы принять и сохранить изменения в документе топологии.

Нажмите **кнопку "Отмена",** чтобы отменить изменения, и закройте изменения свойств для сервера переднего сервера или пула переднего сервера. 

Нажмите кнопку **Справка**, чтобы ознакомиться с данным разделом справки.
