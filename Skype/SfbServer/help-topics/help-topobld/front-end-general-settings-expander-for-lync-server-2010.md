# <a name="front-end-general-settings-expander-for-lync-server-2010"></a>Расширитель общих параметров сервера переднего плана для Lync Server 2010

Вы редактируете свойства переднего конечного сервера или переднего конечного пула путем редактирования или настройки следующих атрибутов. Страница конфигурации разделена на следующие разделы.

 **Общие**

- **FQDN:** полное доменное имя переднего конечного сервера или пула переднего конечного входа.

- Выберите **Использование всех настроенных IP-адресов** для использования всех адресов, настроенных на переднем сервере или в пуле переднего конца.

    > [!IMPORTANT]
    > Этот параметр не следует выбирать при коллокации сервера-посредника на переднем сервере или в пуле переднего конца. Серверам-посредникам и передним конечным серверам необходимы выделенные IP-адреса, с которыми можно общаться.

- Выберите **ограничение использования** службы для выбранных IP-адресов и введите IP-адрес основного **IP-адреса** для связи переднего конечного сервера или переднего пула с остальной частью развертывания. Введите **IP-адрес PSTN с** IP-адресом, связанным с сервером-посредником.

    **Компоненты и функции**

- **Conferencing:** Выберите контрольный ящик, если вы хотите функции конференциации в развертывании. Конференцию включают аудио, видео, совместное использование приложений, общий доступ к настольным компьютерам и веб-конференцию. Вам потребуется создать и связать сервер веб Office приложений для веб-конференций (определенный ниже на этой странице Свойства).

- Если выбрана конференц-связь, можно включить **Конференц-связь с телефонным подключением (ТСОП)**. Установите этот флажок, чтобы включить компоненты конференц-связи для использования телефонных подключений.

- Выберите контрольный Корпоративная голосовая связь, если вы собираетесь **развернуть** функции, позволяющие Lync Server 2013 действовать в качестве телефонной голосовой системы с использованием технологий голосовой связи через IP (VoIP), включая возможность развертывания телефонных телефонов, магистральных магистральных SIP или подключения к сети общедоступных телефонов с помощью сервера-посредника, шлюзов PSTN и IP-PBX в сочетании или в одиночку,  в зависимости от дизайна и требований. Подробные Корпоративная голосовая связь см. в [Корпоративная голосовая связь](/previous-versions/office/lync-server-2013/lync-server-2013-enterprise-voice) и [plan for Корпоративная голосовая связь в Skype для бизнеса Server 2015 г.](../../plan-your-deployment/enterprise-voice-solution/enterprise-voice.md)

    **Ассоциации**

- **SQL Server:** FQDN SQL Server (и необязательно именуемого экземпляра), связанного с передним конечным сервером или пулом переднего конца. Вы выбираете SQL Server из списка или создаете новый SQL Server, щелкнув **New**

- **Хранилище** файлов. Вы выбираете FQDN сервера и долю (в формате), которая будет выступать в качестве расположения магазина файлов для общих файлов, которые  `\\<FQDN of server>\<share name>` Lync Server 2013 создает и использует для репликации, каталогов конференций и других целей. Вы выбираете файловый магазин из списка или создаете новый файловый магазин, нажав **кнопку New**.

- Выберите **почтовый ящик Associate Archiving Server,** чтобы включить сервер архива для этого переднего или переднего конечных пулов. После выбора контрольного окна выберите существующий сервер архива из списка или нажмите кнопку **New,** чтобы создать определения для нового сервера архива.

- Выберите **почтовый ящик Associate Monitoring Server,** чтобы включить сервер мониторинга для этого пула переднего или переднего конечных серверов. После выбора контрольного окна выберите существующий сервер мониторинга из списка или нажмите кнопку **New,** чтобы создать определения для нового сервера мониторинга.

- Выберите **пул дополнительных элементов (для** почтового ящика компонентов мультимедиа, чтобы включить edge Server для этого переднего или переднего конечных пулов). После выбора контрольного окна выберите существующий edge Server или пул из списка или нажмите кнопку **New,** чтобы создать определения для нового edge Server или пула.

  **Устойчивость**

- Выберите  контрольный окне Связанного резервного регистратора для выбора из списка пула переднего или переднего конечных серверов, который будет резервным регистратором (то есть переднего сервера или пула переднего входа, назначенного в качестве дополнительного регистратора в случае сбой первичного)

- Если выбрать пункт "Связанный пул резервного регистратора" и указать резервный регистратор, можно установить флажок **Автоматическая обработка отказов и восстановление после отказа для голосовой связи**. После этого можно определить числовые свойства **Интервал обнаружения отработки отказа голосовой связи (сек)** и **Интервал восстановления после сбоя голосовой связи (сек)**. Дополнительные сведения см. в разделе [Planning for Enterprise Voice Resiliency](/previous-versions/office/lync-server-2013/lync-server-2013-planning-for-enterprise-voice-resiliency)

  **Веб-службы**

- Чтобы настроить **внутренние веб-службы**, необходимо определить **Порты прослушивания****HTTP** и **HTTPS**. По умолчанию используются значения TCP 80 и, соответственно, TCP 443. Кроме того, следует настроить **Опубликованные порты** для **HTTP** и **HTTPS**. По умолчанию используются порты TCP 80 и, соответственно, TCP 443. На основе конфигурации внутренних веб-служб и использования балансировщиков нагрузки (аппаратных балансировщиков и балансировки нагрузки DNS) измените значения портов для определения портов прослушивания и опубликованных портов.

    > [!IMPORTANT]
    > Внутренние веб-службы и определенные порты прослушивания и опубликованные порты предназначены для внутренних клиентов и устройств. Внешние клиенты и устройства используют порты прослушивания и опубликованные порты внешних веб-служб вместе определенным полным доменным именем внешних веб-служб.

- Чтобы настроить **внешние веб-службы**, необходимо определить **Порты прослушивания****HTTP** и **HTTPS**. По умолчанию используются значения TCP 80 и, соответственно, TCP 443. Кроме того, следует настроить **Опубликованные порты** для **HTTP** и **HTTPS**. По умолчанию используются порты TCP 80 и, соответственно, TCP 443. На основе конфигурации внутренних веб-служб и использования балансировщиков нагрузки (аппаратных балансировщиков и балансировки нагрузки DNS) измените значения портов для определения портов прослушивания и опубликованных портов.

    > [!IMPORTANT]
    > Внешние веб-службы и определенные порты прослушивания и опубликованные порты предназначены для внешних клиентов и устройств. Внешние клиенты и устройства используют порты прослушивания и опубликованные порты внешних веб-служб, как правило определенные обратным прокси-сервером, вместе определенным полным доменным именем внешних веб-служб. Отношение полного доменного имени внешних веб-служб и простых URL-адресов определяют URL-адреса, которые будут использоваться внешними клиентами для открытия служб, доступных внешним пользователям и устройствам. Дополнительные сведения о простых URL-адресах см. в разделе [Planning for Simple URLs](/previous-versions/office/lync-server-2013/lync-server-2013-planning-for-simple-urls).

  **Сервер-посредник**

- Чтобы настроить  свойства сервера-посредника для collocated Mediation Server (то есть сервера-посредника, развернутого на переднем сервере или в пуле переднего конца), выберите **сервер-посредник collocated.**

- Чтобы определить **порты прослушивания** для collocated Mediation Server, введите значение **порта TLS** и **TCP,** на которое прослушивается сервер-посредник. По умолчанию TLS определяется как TCP port 5067.

- Чтобы определить значение порта TCP для сервера-посредника, необходимо выбрать контрольный ящик **Включить порт TCP.** По умолчанию сервер-посредник использует безопасность транспортного слоя (TLS) по протоколу TCP. Порты TCP доступны только при включении выбора порта включить TCP.

    > [!NOTE]
    > Это дополнительный параметр, поэтому чтобы определить необходимость его использования, ознакомьтесь с требованиями к используемому шлюзу или ТСОП. По умолчанию используется порт TCP 5068.

- Следует определить магистральные линии, которые сопоставлены с совмещенным сервером-посредником. Если магистральные линии связи уже заданы, их можно будет сопоставить с сервером-посредником.

    Если у вас есть несколько шлюзов, связанных с сервером-посредником, вы можете указать шлюз по умолчанию, выбрав шлюз, который необходимо сделать по умолчанию, и нажав **кнопку Make Default.** Если вы решите удалить текущий шлюз по умолчанию, выберите шлюз и нажмите **кнопку Unmake Default**.

> [!IMPORTANT]
> Если вы внося изменения в свойства в этом диалоговом окантове, необходимо опубликовать топологию и запустить мастер развертывания Skype для бизнеса Server на всех затронутых серверах. После публикации новой топологии для вас в качестве ссылки на экране успешной публикации топологии предоставляется список затронутых серверов, на которых необходимо запустить мастер развертывания Skype для бизнеса Server развертывания. Сведения о публикации обновленной топологии см. в разделе [Publish the Topology](/previous-versions/office/lync-server-2013/lync-server-2013-publish-the-topology). Сведения о мастере Skype для бизнеса Server развертывания см. в материале [Lync Server Administrative Tools.](/previous-versions/office/lync-server-2013/lync-server-2013-lync-server-administrative-tools)

Нажмите кнопку **ОК**, чтобы принять и сохранить изменения в документе топологии.

Щелкните **Отмена,** чтобы отменить изменения и закрыть **свойства редактирования** для переднего сервера или пула переднего конца.

Нажмите кнопку **Справка**, чтобы ознакомиться с данным разделом справки.
