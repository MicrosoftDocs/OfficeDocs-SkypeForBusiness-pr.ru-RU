# <a name="front-end-general-settings-expander-for-lync-server-2010"></a>Расширитель интерфейсных общих параметров для Lync Server 2010

Вы изменяете свойства сервера переднего плана или интерфейсного пула, редактируя или настраивая следующие атрибуты. Страница конфигурации разделена на следующие разделы.

 **Общие сведения**

- **Полное доменное имя: полное**доменное имя сервера переднего плана или интерфейсного пула.

- Выберите **использовать все настроенные IP-адреса** , чтобы использовать все адреса, настроенные на сервере переднего плана или интерфейсном пуле.

    > [!IMPORTANT]
    > Этот параметр не следует выбирать при размещении сервера-посредника на сервере переднего плана или интерфейсном пуле. Серверам-посредникам и серверам переднего плана требуются выделенные IP-адреса для обмена сообщениями.

- Выберите **ограничить использование службы выбранными IP-** адресами и введите IP-адрес **основного IP-адреса** сервера переднего плана или интерфейсного пула для связи с остальной частью развертывания. Тип в **IP-адресе PSTN** IP-адрес, связанный с сервером-посредником.

    **Компоненты и функции**

- **Конференц**-связь: Установите этот флажок, если вы хотите использовать функции конференц-связи в развертывании. Конференц-связь включает звук, видео, общий доступ к приложениям, общий доступ к рабочему столу и веб-конференции. Вам потребуется создать и связать сервер Office Web Apps для веб-конференций (задается позже на этой странице свойств).

- Если выбрана конференц-связь, можно включить **Конференц-связь с телефонным подключением (ТСОП)**. Установите этот флажок, чтобы включить компоненты конференц-связи для использования телефонных подключений.

- Установите флажок **Корпоративная голосовая связь** , если планируется развернуть компоненты, позволяющие Lync Server 2013 действовать в качестве телефонной системы голосовой связи с помощью технологий VoIP. в том числе: возможность развертывания телефонов для телефонов с телефонным подключением, магистральных каналов SIP или подключения к телефонной сети общего пользования с помощью сервера-посредника, шлюзов PSTN и IP-УАТС, в зависимости от того, как они разрабатываются и удовлетворяют требованиям. Подробнее о корпоративной голосовой связи можно узнать в статье [корпоративный голос](https://technet.microsoft.com/library/c9da8099-6f4f-4346-ac67-f041bb96072c.aspx) и [спланировать для корпоративной голосовой связи в Skype для бизнеса Server 2015](../../plan-your-deployment/enterprise-voice-solution/enterprise-voice.md)

    **Связанных**

- **Хранилище SQL Server**: полное доменное имя SQL Server (и, при необходимости, именованный экземпляр), связанное с сервером переднего плана или интерфейсным пулом. Выберите хранилище SQL Server из списка или создайте новое хранилище SQL Server, нажав кнопку **создать** .

- **Хранилище файлов**: вы выбираете полное доменное имя сервера и общий ресурс (в формате  `\\<FQDN of server>\<share name>` ), который будет использоваться в качестве расположения хранилища файлов для общих файлов, создаваемых Lync Server 2013 и используемых для репликации, каталогов конференций и других целей. Вы выбираете хранилище файлов из списка или создаете новое хранилище файлов, нажав кнопку **создать**.

- Установите флажок **сопоставить сервер архивации** , чтобы включить сервер архивации для этого сервера переднего плана или интерфейсного пула. После установки этого флажка выберите существующий сервер архивации в списке или нажмите кнопку **создать** , чтобы создать определения для нового сервера архивации.

- Установите флажок **сопоставить сервер мониторинга** , чтобы включить сервер мониторинга для этого сервера переднего плана или интерфейсного пула. После установки этого флажка выберите существующий сервер мониторинга в списке или нажмите кнопку **создать** , чтобы создать определения для нового сервера мониторинга.

- Установите флажок **связать пограничный пул (для компонентов мультимедиа** ), чтобы включить пограничный сервер для этого сервера переднего плана или интерфейсного пула. После установки этого флажка выберите существующий пограничный сервер или пул из списка или нажмите кнопку **создать** , чтобы создать определения для нового пограничного сервера или пула.

  **Устойчивости**

- Установите флажок **связанный резервный пул регистратора** , чтобы выбрать из списка сервер переднего плана или интерфейсный пул, который будет служить резервным регистратором (то есть, сервер переднего плана или интерфейсный пул, назначенный дополнительным регистратором в случае сбоя основного сервера).

- Если выбрать пункт "Связанный пул резервного регистратора" и указать резервный регистратор, можно установить флажок **Автоматическая обработка отказов и восстановление после отказа для голосовой связи**. После этого можно определить числовые свойства **Интервал обнаружения отработки отказа голосовой связи (сек)** и **Интервал восстановления после сбоя голосовой связи (сек)**. Дополнительные сведения см. в разделе [Planning for Enterprise Voice Resiliency](https://technet.microsoft.com/library/ca116700-1055-4ca5-9b87-4c7f380c3655.aspx)

  **Веб-службы**

- Чтобы настроить **внутренние веб-службы**, необходимо определить **Порты прослушивания****HTTP** и **HTTPS**. По умолчанию используются значения TCP 80 и, соответственно, TCP 443. Кроме того, следует настроить **Опубликованные порты** для **HTTP** и **HTTPS**. По умолчанию используются порты TCP 80 и, соответственно, TCP 443. На основе конфигурации внутренних веб-служб и использования балансировщиков нагрузки (аппаратных балансировщиков и балансировки нагрузки DNS) измените значения портов для определения портов прослушивания и опубликованных портов.

    > [!IMPORTANT]
    > Внутренние веб-службы и определенные порты прослушивания и опубликованные порты предназначены для внутренних клиентов и устройств. Внешние клиенты и устройства используют порты прослушивания и опубликованные порты внешних веб-служб вместе определенным полным доменным именем внешних веб-служб.

- Чтобы настроить **внешние веб-службы**, необходимо определить **Порты прослушивания****HTTP** и **HTTPS**. По умолчанию используются значения TCP 80 и, соответственно, TCP 443. Кроме того, следует настроить **Опубликованные порты** для **HTTP** и **HTTPS**. По умолчанию используются порты TCP 80 и, соответственно, TCP 443. На основе конфигурации внутренних веб-служб и использования балансировщиков нагрузки (аппаратных балансировщиков и балансировки нагрузки DNS) измените значения портов для определения портов прослушивания и опубликованных портов.

    > [!IMPORTANT]
    > Внешние веб-службы и определенные порты прослушивания и опубликованные порты предназначены для внешних клиентов и устройств. Внешние клиенты и устройства используют порты прослушивания и опубликованные порты внешних веб-служб, как правило определенные обратным прокси-сервером, вместе определенным полным доменным именем внешних веб-служб. Отношение полного доменного имени внешних веб-служб и простых URL-адресов определяют URL-адреса, которые будут использоваться внешними клиентами для открытия служб, доступных внешним пользователям и устройствам. Дополнительные сведения о простых URL-адресах см. в разделе [Planning for Simple URLs](https://technet.microsoft.com/library/20e4f4b6-b7ff-4297-b00d-d1211ee800ac.aspx).

  **Сервер-посредник**

- Чтобы настроить свойства **сервера-посредника** для совмещенного сервера-посредника (то есть сервера-посредника, развернутого на сервере переднего плана или интерфейсном пуле), выберите параметр Совмещенный **сервер-посредник включен**.

- Чтобы определить **порты прослушивания** для совмещенного сервера-посредника, введите значение **TLS** и **TCP** -порт, который прослушивает совмещенный сервер-посредник. По умолчанию протокол TLS определен как TCP-порт 5067.

- Чтобы определить значение порта TCP для сервера-посредника, установите флажок **включить порт TCP** . По умолчанию сервер-посредник использует протокол TLS (Transport Layer Security) по протоколу TCP. TCP-порты доступны только в том случае, если включен параметр включить TCP-порт.

    > [!NOTE]
    > Это дополнительный параметр, поэтому чтобы определить необходимость его использования, ознакомьтесь с требованиями к используемому шлюзу или ТСОП. По умолчанию используется порт TCP 5068.

- Следует определить магистральные линии, которые сопоставлены с совмещенным сервером-посредником. Если магистральные линии связи уже заданы, их можно будет сопоставить с сервером-посредником.

    При наличии нескольких шлюзов, связанных с сервером-посредником, можно указать шлюз по умолчанию, выбрав шлюз, который будет использоваться по умолчанию, и нажать кнопку **по умолчанию**. Если выбрано удаление текущего шлюза по умолчанию, выберите шлюз и нажмите отменить **по умолчанию**.

> [!IMPORTANT]
> При внесении изменений в свойства в этом диалоговом окне необходимо опубликовать топологию и запустить мастер развертывания Skype для бизнеса Server на всех серверах, на которых она работает. После публикации новой топологии список затронутых серверов, на которых необходимо запустить мастер развертывания Skype для бизнеса Server, предоставляется в качестве ссылки на экране Сводка по успешной публикации топологии. Сведения о публикации обновленной топологии см. в разделе [Publish the Topology](https://technet.microsoft.com/library/3b5a744b-b3a8-4538-a55e-e2e4f72dff47.aspx). Подробнее о мастере развертывания Skype для бизнеса Server можно узнать в статье администрирование [Lync Server](https://technet.microsoft.com/library/9b006f93-4f3d-461d-89b8-e80a34fdb3c5.aspx).

Нажмите кнопку **ОК**, чтобы принять и сохранить изменения в документе топологии.

Нажмите кнопку **Отмена** , чтобы отменить изменения и закрыть **Свойства изменить** для сервера переднего плана или интерфейсного пула.

Нажмите кнопку **Справка**, чтобы ознакомиться с данным разделом справки.
