# <a name="move-file-store-data-to-a-new-file-store-in-skype-for-business-server-2015"></a>Перемещение данных в новое хранилище файлов в Skype для бизнеса Server 2015
 
Если необходимо удалить файлового сервера, который в данный момент действует как хранилище файлов для вашего Скайп для развертывания Business Server 2015 или если требуется внести другие изменения, которые сделают текущий файл хранилище недоступен, необходимо сначала создать новую общую папку. Затем потребуется выполнить следующую процедуру.
  
1. Завершение работы Скайп для служб Business Server 2015, использующих хранилище файлов, которое планируется удалить.
    
2. Определите хранилище файлов в построителе топологий и публикация изменений становится доступной для развертывания нового файла хранения.
    
3. Переместите данные в новое хранилище файлов.
    
4. Перезапустите серверы или службы.
    
5. При необходимости удалите старую локальную папку с файлами и общую папку.
    
### <a name="to-move-file-store-data-from-one-file-store-to-a-new-file-store"></a>Перемещение данных из старого хранилища файлов в новое

1. Войдите на компьютер как член группы RTCUniversersalServerAdmins или CsServerAdministrator установленными Скайп для Business Server 2015, средства администрирования.
    
2.  Откройте окно браузера и введите URL-адрес администрирования, чтобы открыть Скайп для панели управления Business Server.  
    
3. В левой панели навигации щелкните **Топология**, а затем **Состояние**.  
    
4. Для каждого пула директоров, директора, сервера Standard Edition и пула переднего плана, который использует хранилище файлов, который планируется удалить выберите сервера или пула, щелкните **Действие**и нажмите кнопку **Остановить все службы**. 
    
5. Войдите на компьютер, где установлен построитель топологий, с использованием учетной записи, входящей в группу администраторов домена и группу RTCUniversalServerAdmins.
    
6. Запустите построитель топологий: Нажмите кнопку **Пуск**, последовательно выберите пункты **Все программы**, щелкните **Скайп для Business Server 2015**и нажмите кнопку **Скайп для построителя 2015Topology Business Server**.
    
7. Выберите сервер или пул, использующий хранилище файлов, и выполните следующие действия.
    
1. Щелкните сервер или пул правой кнопкой мыши и выберите команду **Изменить свойства**.  
    
2. В окне **Изменить свойства** в разделе **Связи**, **Хранилище файлов** нажмите кнопку **Далее**.
    
3. В разделе **Определение нового хранилища файлов** в поле **Полное доменное имя файлового сервера** введите полное доменное имя файлового сервера. В поле **Общая папка** введите имя новой общей папки и нажмите кнопку **ОК**.
    
    > [!IMPORTANT]
    > Это действие задает новое хранилище файлов для использования в построителе топологий. Можно определить его только один раз, не для каждого сервера. Перед публикацией топологии необходимо создать определенный файловый ресурс на сервере определенный файл. Дополнительные сведения см [определить хранилище файлов для сервера переднего плана](http://technet.microsoft.com/library/90994400-c4e5-4509-af41-121ac716fbca.aspx). 
  
8. Для каждого сервера или пула, использующего хранилище файлов, выполните следующие действия.
    
1. Щелкните сервер или пул правой кнопкой мыши и выберите команду **Изменить свойства**.
    
2. В окне **Изменить свойства** в разделе **Связи**, **Хранилище файлов** выберите новую общую папку, а затем нажмите кнопку **ОК**.
    
9. Опубликуйте топологию, проверьте состояние репликации и запустите Скайп для мастера развертывания сервера Business при необходимости. Для получения дополнительных сведений просмотрите [Распространенные процедур для удаления Lync серверов и компонентов](http://technet.microsoft.com/library/5438ce1e-57fa-4031-8bdb-3af6581d901b.aspx).
    
10. Откройте окно командной строки: нажмите кнопку **Пуск**, выберите команду **выполнить**и введите cmd.exe.
    
11. В командной строке введите следующую команду:
    
  ```
  Robocopy \\<OldFileServer>\<OldShare> \\<NewFileServer>\<NewShare> /S /R:10 /W:10 /XF Meeting.Active /MT /LOG:<directory path\logname>
  ```

    > [!TIP]
    > Параметр /S копирует файлы, каталоги и подкаталоги. Параметр /XF пропускает файлы с именем Meeting.Active. Текущие версии файла robocopy.exe с параметром /MT существенно повышают скорость копирования благодаря использованию нескольких потоков. Для параметра/log используйте имя каталога журнала и путь файла в виде C:\Logfiles\log.txt. Этот параметр создает файл журнала операций в именованном расположении. 
  
12. По завершении копирования данных, в панели управления Lync Server, щелкните **Топология**и нажмите кнопку **состояние**.
    
13. Для каждого сервера или пула, для которого вы остановили службы, выберите сервер или пул, щелкните **Действие**, а затем **Запустить все службы**. 
    
14. Удалите старое хранилище файлов из топологии, а затем опубликуйте топологию. Дополнительные сведения см [удаления хранилища файлов](http://technet.microsoft.com/library/1ba7eb15-5c87-4357-b4d8-f59409ac7f71.aspx).
    
15. (Необязательно) Войдите на компьютер, на котором содержится удаленное хранилище файлов, с учетной записью, входящей в группу локальных администраторов или администраторов домена, а затем удалите старую общую папку и каталог.
