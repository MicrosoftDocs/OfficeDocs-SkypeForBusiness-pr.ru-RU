# <a name="add-front-end-monitoring-store-page"></a>Страница добавления хранилища мониторинга переднего плана
 
**Определение хранилища SQL Server для мониторинга** выполняется путем настройки следующих свойств:
  
- **Мониторинг SQL Server:** выберите SQL Server доменное имя (и, необязательно экземпляр) из списка.
    
    Нажмите **кнопку** New для создания SQL Server определения FQDN и дополнительного имени экземпляра для магазина Monitoring Server.
    
- Если вы **хотите добавить зеркальное** зеркальное отражение базы данных для сервера мониторинга, выберите SQL Server для хранения.
    
    Выберите в списке существующее **Зеркало хранилища данных наблюдения SQL Server**.
    
    **Щелкните Новое**, чтобы создать SQL Server определение FQDN и необязательно имя экземпляра для зеркального магазина.
    
- Если вы выбрали **включить зеркальное SQL Server** хранения, необязательно выберите **свидетель SQL Server** зеркального зеркала, чтобы включить автоматическую сбойную передачу, чтобы выбрать хранилище SQL Server зеркального свидетеля из списка.
    
    Нажмите **кнопку** New, чтобы создать новое определение SQL Server FQDN и необязательно имя экземпляра для зеркального хранения свидетелей.
    
Нажмите кнопку **Назад**, чтобы вернуться в предыдущее диалоговое окно определения пула.
  
После завершения ввода значений в этом диалоговом окне нажмите кнопку **Далее**, чтобы продолжить настройку.
  
Нажмите кнопку **Отмена**, чтобы отменить все изменения и завершить работу мастера.
  
Нажмите кнопку **Справка**, чтобы открыть контекстно-зависимую справку, аналогичную данной странице.
