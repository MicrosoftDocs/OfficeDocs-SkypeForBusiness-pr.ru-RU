# <a name="add-director-monitoring"></a>Добавление мониторинга директора
 
Вы можете выполнить **Определение хранилища SQL Server для мониторинга**, настроив следующие свойства:
  
- **Хранилище SQL Server** мониторинга: выберите SQL Server полное доменное имя (FQDN) (и, при желании, именуемой SQL Server экземпляр) в списке.
    
    Нажмите **кнопку** "Создать", чтобы SQL Server определения FQDN, а при желании — имя экземпляра для хранения сервера мониторинга.
    
- Если вы хотите **добавить зеркальное отражение** базы данных для сервера мониторинга, выберите SQL Server включить зеркальное отражение для хранения данных.
    
    Выберите в списке существующее **Зеркало хранилища данных наблюдения SQL Server**.
    
    Нажмите **кнопку** "Создать", чтобы SQL Server определения FQDN, а при желании — имя экземпляра для зеркального хранения.
    
- Если выбран вариант **"Включить зеркальное SQL Server** хранения", при желании выберите "Использовать свидетель зеркального SQL Server", чтобы включить автоматическую отключку, чтобы выбрать в списке SQL Server зеркального зеркального хранения. 
    
    Нажмите **кнопку** "Создать", чтобы создать новое SQL Server FQDN и при желании имя экземпляра для зеркального хранения свидетеля.
    
Нажмите кнопку **Назад**, чтобы вернуться в предыдущее диалоговое окно определения пула.
  
После завершения ввода значений в этом диалоговом окне нажмите кнопку **Далее**, чтобы продолжить настройку.
  
Нажмите кнопку **Отмена**, чтобы отменить все изменения и завершить работу мастера.
  
Нажмите кнопку **Справка**, чтобы открыть контекстно-зависимую справку, аналогичную данной странице.
