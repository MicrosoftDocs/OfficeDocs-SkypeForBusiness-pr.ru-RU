# <a name="add-edge-server"></a>Добавление пограничного сервера

Чтобы внедрить пограничный сервер или пул пограничных серверов в структуру топологии, необходимо указать полное доменное имя сервера, на котором требуется развернуть пограничный сервер или пул пограничных серверов. Перед публикацией топологии, которая включает в себя пул Edge Server или Edge Server и установку Skype для бизнеса Server, необходимо завершить все необходимые условия для развертывания внешнего доступа пользователей. Дополнительные сведения об этих требованиях см. в разделе [Preparing for Installation of Servers in the Perimeter Network](/previous-versions/office/lync-server-2013/lync-server-2013-preparing-for-installation-of-servers-in-the-perimeter-network) документации по развертыванию.

> [!IMPORTANT]
> Указываемое вами имя должно в точности совпадать с настроенным на сервере именем компьютера. По умолчанию имя компьютера, который не присоединен к домену, является коротким, а не полным доменным именем. Построитель топологии использует полные доменные имена, а не короткие имена. Следует настроить суффикс службы доменных имен (DNS) для имени компьютера, который планируется развернуть в качестве пограничного сервера или пула пограничных серверов и который не присоединен к домену.

> [!TIP]
> Если вы планируете в будущем внедрить пул пограничных серверов, выберите **Пул из нескольких компьютеров**. Хотя пул — это два и более компьютера с балансировкой нагрузки, вы можете создать пул из одного компьютера и создать для этого отдельного компьютера полное доменное имя пула. Когда вы будете готовы добавить больше компьютеров в пул позже, необходимо снова определить нового члена пула, опубликовать новую топологию, а затем настроить новый член пула edge Server через мастер развертывания Skype для бизнеса Server. Следует также добавить новый член пула в соответствующие подсистемы балансировки нагрузки для этого пула, систему балансировки нагрузки DNS или устройства балансировки нагрузки. Внутренний и внешний интерфейсы пограничного сервера должны использовать одинаковый тип балансировки нагрузки. Вы не можете использовать балансировку нагрузки DNS для одного интерфейса пограничного сервера и устройства балансировки нагрузки для другого. Убедитесь, что добавляете новый член в правильную подсистему балансировки нагрузки.

Поддержку внешнего доступа пользователей можно добавить при развертывании начальной топологии или позднее. Дополнительные сведения о добавлении пограничных серверов или пула пограничных серверов в существующую топологию см. в разделе [Define Your Edge Topology](/previous-versions/office/lync-server-2013/lync-server-2013-define-your-edge-topology) в документации по развертыванию пограничного сервера.
