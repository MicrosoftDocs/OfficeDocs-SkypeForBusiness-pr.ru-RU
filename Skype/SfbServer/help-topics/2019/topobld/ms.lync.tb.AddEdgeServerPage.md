# <a name="add-edge-server"></a>Добавление пограничного сервера

Чтобы внедрить пограничный сервер или пул пограничных серверов в структуру топологии, необходимо указать полное доменное имя сервера, на котором требуется развернуть пограничный сервер или пул пограничных серверов. Перед публикацией топологии, которая включает в себя пул серверов или серверов и установкой Skype для бизнеса Server, необходимо завершить все необходимые условия для развертывания доступа внешних пользователей. Дополнительные сведения об этих требованиях см. в разделе [Preparing for Installation of Servers in the Perimeter Network](https://technet.microsoft.com/library/5e6c457a-f964-4ef7-a709-97abda9c673a.aspx) документации по развертыванию.

> [!IMPORTANT]
> Указываемое вами имя должно в точности совпадать с настроенным на сервере именем компьютера. По умолчанию имя компьютера, который не присоединен к домену, является коротким, а не полным доменным именем. Построитель топологии использует полные доменные имена, а не короткие имена. Следует настроить суффикс службы доменных имен (DNS) для имени компьютера, который планируется развернуть в качестве пограничного сервера или пула пограничных серверов и который не присоединен к домену.

> [!TIP]
> Если вы планируете в будущем внедрить пул пограничных серверов, выберите **Пул из нескольких компьютеров**. Хотя пул — это два и более компьютера с балансировкой нагрузки, вы можете создать пул из одного компьютера и создать для этого отдельного компьютера полное доменное имя пула. Когда вы будете готовы добавить дополнительные компьютеры в пул позже, необходимо снова определить нового участника пула, опубликовать новую топологию, а затем настроить новый член пула с помощью мастера развертывания Skype для бизнеса Server. Следует также добавить новый член пула в соответствующие подсистемы балансировки нагрузки для этого пула, систему балансировки нагрузки DNS или устройства балансировки нагрузки. Внутренний и внешний интерфейсы пограничного сервера должны использовать одинаковый тип балансировки нагрузки. Вы не можете использовать балансировку нагрузки DNS для одного интерфейса пограничного сервера и устройства балансировки нагрузки для другого. Убедитесь, что добавляете новый член в правильную подсистему балансировки нагрузки.

Поддержку внешнего доступа пользователей можно добавить при развертывании начальной топологии или позднее. Дополнительные сведения о добавлении пограничных серверов или пула пограничных серверов в существующую топологию см. в разделе [Define Your Edge Topology](https://technet.microsoft.com/library/787b23f1-8fa0-4c37-abf2-c516c5dd66f0.aspx) в документации по развертыванию пограничного сервера.
