# <a name="sql-store-settings-expander"></a>Расширитель параметров хранилища SQL
 
Чтобы изменить свойства базы данных SQL Server, необходимо изменить экземпляр базы данных SQL Server. Диалоговое окно **Изменение свойств** нельзя использовать для выполнения задач, таких как база данных сервера архивации перемещается с одного компьютера на другой. Кроме того нельзя использовать диалоговое окно **Изменение свойств** для изменения экземпляра SQL Server, в котором размещается центральное хранилище управления.
  
## <a name="editing-the-properties-of-a-sql-server-database"></a>Изменение свойств базы данных SQL Server

Чтобы изменить экземпляр SQL Server, используемый любой базы данных, отличный от центрального хранилища управления, выполните следующую процедуру в построителе топологии:
  
### <a name="to-modify-an-instance-of-sql-server"></a>Изменение экземпляра SQL Server

1. Выберите нужную базу данных в узле **хранит SQL** и выберите команду **Изменить свойства**.
    
2. В диалоговом окне **Изменение свойств** выполните одно из следующих действий.
    
  - Чтобы использовать экземпляр SQL Server по умолчанию, выберите **Экземпляр по умолчанию** и нажмите кнопку **ОК**.
    
  - Чтобы использовать именованный экземпляр базы данных, выберите **Именованный экземпляр**, введите имя экземпляра в текстовом поле и нажмите кнопку **ОК**. Необходимо ввести только имя экземпляра (например, ArchivingInstance), а не всю путь SQL Server.
    
При работе в диалоговом окне **Изменение свойств** Topology Builder будет не убедитесь, что экземпляр базы данных, который был введен допустимый экземпляр. Например если вы случайно typeArchivingInstanec как имя экземпляра и затем нажмите **кнопку ОК**, Topology Builder будет принимать этот недопустимый экземпляр. Перед публикацией данной топологии необходимо исправить этой ошибки: Если не удается найти экземпляр SQL Server, Topology Builder не создаст этот экземпляр для вас.
