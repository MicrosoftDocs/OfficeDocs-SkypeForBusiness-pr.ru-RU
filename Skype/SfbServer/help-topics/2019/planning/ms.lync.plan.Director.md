# <a name="director-planning-tool"></a>Director (Средство планирования)
 
Director — это сервер, Skype для бизнеса Server средства связи, которые могут проверить подлинность запросов пользователей, но не являются домашними учетными записями пользователей. 
  
Эта роль необязательна, вы можете развернуть директора в следующих двух сценариях:
  
- Если вы включаете доступ внешних пользователей, развертывая edge Servers, необходимо также развернуть director. В этом сценарии Директор проходит проверку подлинности внешних пользователей, а затем передает их трафик на внутренние серверы. Когда директор используется для проверки подлинности внешних пользователей, он освобождает серверы пула передней части от накладных расходов на выполнение проверки подлинности этих пользователей. Он также помогает изолировать внутренние пулы переднего конечного входа от вредоносных трафика, таких как атаки с отказом в обслуживании. Если во время такой атаки сеть заполняется неподходящим внешним трафиком, этот трафик останавливается на директоре.
    
- Если развернуть несколько пулов переднего конца на центральном сайте, добавив на этот сайт директора, можно оптимизировать запросы на проверку подлинности и повысить производительность. В этом сценарии все запросы сначала идут к Директору, который затем передает их в правильный пул переднего конца.
