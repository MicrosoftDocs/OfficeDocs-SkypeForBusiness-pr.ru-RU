# <a name="client-version-policy-create-new-or-edit-existing"></a>Политика версий клиентов: создание новой или редактирование существующей

Вы может указать версию клиентов, поддерживаемых в вашей среде. При взаимодействии двух клиентов с разными версиями возможности, доступные обоим клиентам, могут быть ограничены возможностями другого клиента. Чтобы максимально использовать функции, включенные в Skype для бизнеса Server, и улучшить общий пользовательский интерфейс, можно использовать фильтр версий клиентов, чтобы ограничить версии клиентов, используемые в вашей среде. Используя этот фильтр, вы также можете сократить расходы, связанные с поддержкой нескольких версий клиентов.

> [!IMPORTANT]
> Фильтры указаны в порядке приоритета. Например, если вы используете фильтр, который разрешает клиента использовать для подключения версию 1.5, за которым идет фильтр, блокирующий клиенты с версией ниже 2.0, первый фильтр имеет более высокий приоритет, поэтому клиентам с версией 1.5 разрешено осуществлять подключение.

## <a name="tasks-you-can-perform"></a>Задачи, которые вы можете выполнить

На странице **Создание новой политики версий клиента** или **Изменение политики версий клиента** можно выполнить следующие задачи:

- добавить или изменить имя или описание политики версий клиентов;

- добавить или изменить правила версий клиентов для политики версий клиентов.

## <a name="ui-reference"></a>Ссылка на пользовательский интерфейс

В следующих списках описываются меню, команды, поля и свойства на этой странице.

- **Область действия** Определяет область (сайт, пул или пользователь) политики версий клиентов.

- **Имя** Можно добавить или изменить имя политики версий клиентов.

- **Описание** Вы можете добавить описание, чтобы определить политику в списке на странице "Политика версий клиентов".

- **Новый** В политику можно добавить новое правило версий клиентов.

- **Показать подробности** Этот параметр открывает диалоговое окно, в котором можно изменить параметры правила версии клиента.

- **Удаление** Этот параметр удаляет выбранное правило версий клиентов из политики.

- **Стрелки вверх и вниз** Этот параметр перемещает выбранное правило версий клиентов вверх или вниз по приоритету. Правила обрабатываются в порядке, который указан в списке.

Подробные сведения о взаимосвязи между клиентами и версиями клиентов см. в этой [теме.](https://technet.microsoft.com/library/0f126571-91a2-45d5-855c-1e4ddb45fc04.aspx) Дополнительные сведения о работе с политиками версий клиентов см. в разделе [Specify the Client Versions Supported in Your Organization](https://technet.microsoft.com/library/d256a581-9a48-4d1a-82cc-2e1f520d7d2e.aspx) в документации по применению.
