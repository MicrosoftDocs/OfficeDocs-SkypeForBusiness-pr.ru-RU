# <a name="client-version-rule"></a>Правило версий клиентов

Политика версий клиентов состоит из набора правил версий клиентов. Эти правила определяют действия, которые следует предпринять при попытке пользователей выполнить вход с использованием определенных клиентов и их версий.

## <a name="tasks-you-can-perform"></a>Задачи, которые вы можете выполнить

Вы можете выполнить следующие задачи на странице **Новая настройка версии клиента** или **Изменить настройку версии клиента**:

- добавить новые правила в политику версий клиентов;

- изменить правила в существующей политике версий клиентов.

- **Номер версии** Вы можете указать номера версий для следующих полей или указать номер версии клиента с помощью подмайных обозначений.

  - **Основная версия** Указывает номер, соответствующий основной версии клиента.

  - **Minor version** Указывает номер, соответствующий незначительному выпуску клиента.

  - **Сборка** Указывает номер сборки, соответствующий основной и второстепенной версии клиента.

  - **Обновление** Указывает номер, соответствующий обновленной версии клиента.

- **Операция сравнения** Можно указать операцию совпадения для версии клиента, указанной на предыдущих шагах. Доступны следующие операции:

  - **Совпадает с**

  - **Не является**

  - **Новее**

  - **Новее или совпадает с**

  - **Старее**

  - **Старее или совпадает с**

- **Действие** Вы можете указать действие, выполняемые при выполнении критериев, которые были выполнены на предыдущих шагах. Доступны следующие действия:

  - **Разрешить** Позволяет клиенту войти в систему.

  - **Разрешение и обновление** Позволяет клиенту входить в систему и получать обновления из службы Windows Server Update Service или Microsoft Update. Это действие доступно, только если выбрано приложение **OC** агента пользователя.

    > [!NOTE]
    > При выборе этого действия при следующем входе пользователей в Skype для бизнеса появляется уведомление. В уведомлении говорится, что обновление доступно, даже если обновления для службы обновления Windows Server или Обновления Майкрософт еще не выпущены. Чтобы избежать путаницы, это действие следует выбирать только после того, как обновления станут доступны.

  - **Разрешить с URL-адресом** Позволяет клиенту войти в систему и отображает сообщение о том, где скачать другую версию клиента. Соответствующий URL-адрес указывается в поле **URL-адрес**.

  - **Block** Не позволяет клиенту войти в систему.

  - **Блокировка и обновление** Предотвращает вход в систему и позволяет клиенту получать обновления из службы обновления Windows Server или Обновления Майкрософт. Это действие доступно, только если выбрано приложение **OC** агента пользователя.

  - **Заблокировать с URL-адресом** — запрещает клиенту выполнить вход и отображает сообщение о том, где можно загрузить другую версию клиента. Соответствующий URL-адрес указывается в поле **URL-адрес**.
