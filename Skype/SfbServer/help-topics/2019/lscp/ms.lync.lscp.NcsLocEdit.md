# <a name="location-policy-create-new-or-edit-existing"></a>Политика определения местонахождения: создание новой или редактирование существующей

Вы можете настроить политики расположения, чтобы определить, включена ли enhanced 9-1-1 (E9-1-1) и как она используется, а также как информация о расположении используется для пользователей и контактов.

## <a name="ui-reference"></a>Ссылка на пользовательский интерфейс

В следующем списке описываются поля на странице.

- **Область действия** Определяет область создаемой или изменяемой политики расположения: глобальную, на уровне сайта или на уровне пользователя.

- **Имя** Для каждой политики расположения требуется имя. Глобальные политики и политики расположения сайтов именуются по умолчанию, и их нельзя изменить. Для политик расположения пользователей используйте описательное имя, идентифицирующие пользователя или группу пользователей.

    > [!NOTE]
    > После сохранения политики расположения имя уже нельзя изменить.

- **Включить Enhanced 9-1-1 (E9-1-1)** Чтобы включить E9-1-1 для пользователей, которым назначена эта политика расположения, выберите этот контроль.

- **Расположение** Укажите, будут ли пользователям предложены сведения о расположении:

  - **Обязательное** Выберите этот параметр, если при регистрации клиента в новом расположении пользователям будет предложено ввести сведения о расположении. Пользователи могут отклонять запрос, не вводя сведения о расположении.

  - **Не требуется** Выберите этот параметр, если пользователям не будет предложено у вас получить сведения о расположении.

  - **Заявление об отказе** Выберите этот параметр, если пользователям будет предложено ввести сведения о расположении, но они увидят сообщение об отказе, если отклосети запрос без ввода сведений. Пользователи могут выполнить экстренный вызов, но никакие другие вызовы, пока не введите сведения о расположении.

- **Использование расположения только для E9-1-1** Если сведения о расположении будут использоваться только для экстренных вызовов, выберите этот контроль.

- **Использование STN** Выберите режим работы с телефонной сетью общего пользования (PSTN), который будет использоваться для определения маршрута голосовой связи, который будет использоваться для маршрутки экстренных вызовов от клиентов, которые используют этот профиль. Маршрут, связанный с таким применением, должен указывать на SIP-магистраль, выделенную для экстренных вызовов, или на шлюз идентификационного номера экстренного местоположения (ELIN), который направляет экстренные вызовы в ближайший пункт общественной безопасности (PSAP). Возможные варианты: **"Внутренний",** **"Локальный"** или **"Междугородный".**

- **Номер набора E9-1-1** Укажите номер, набираемого для экстренного обслуживания.

- **Маска набора E9-1-1** Укажите номер, который набирает пользователь, который затем преобразуется в номер экстренного номера. Например, введите значение 212 в это поле, чтобы пользователь может набрать номер 212, чтобы связаться со службами экстренных служб. Это позволяет набирать альтернативные номера экстренных служб и по-прежнему звонить в экстренные службы (например, если кто-то из страны или региона с другим номером экстренной службы пытается набрать номер этой страны или региона, а не номер страны или региона, в который он сейчас находится). Можно определить несколько масок набора номера экстренных служб, разделяя значения точками с запятыми. Например, 212;414. Максимальная длина строки — 100 символов. Все символы должны быть цифрами от 0 до 9.

    > [!IMPORTANT]
    > Убедитесь, что маска номера отличается от номера в диапазоне номеров парковки вызовов, так как маршрутизация парковки вызовов имеет приоритет над преобразованием строки набора номера экстренного вызова. Чтобы увидеть диапазоны номеров парковки вызовов, щелкните **"Функции** голосовой почты" в левой панели навигации, а затем выберите **"Парковка вызовов".**

- **URI уведомления** Укажите один или несколько URIS SIP, которые будут уведомлены при вызове экстренных служб. Например, введите SIP-URI офиса безопасности компании, чтобы уведомить сотрудников службы безопасности с помощью мгновенного сообщения при вызове экстренных служб. Если местоположение вызываемой нахождения доступно, это расположение включается в уведомление. Можно указать несколько SIP URIs в качестве списка, разделенного запятой. Например, "sip:security@litwareinc.com","sip:kmyer@litwareinc.com". Строка может иметь длину от 1 до 256 знаков, и должна начинаться с префикса "sip:". Можно также указать списки рассылки.

- **URI конференции** Укажите URI SIP (в данном случае номер телефона) для сторонних конференций в экстренных вызовах. Например, введите номер телефона офиса безопасности компании, чтобы он получал вызов при экстренном вызове. Параметр режима **конференции** определяет, может ли третья сторона участвовать в вызове или просто прослушивать его. Длина строки должна быть от 1 до 256 символов, строка должна начинаться с префикса sip:.

- **Режим конференции** Если вы указали значение **для URI конференции,** установите для этого поля одно из следующих значений:

  - **В одну сторону** Указывает, что третья сторона может прослушивать только вызов между вызываемой стороной и оператором PSAP.

  - **Двунавлая** Указывает, что третья сторона может участвовать в звонке между вызываемой стороной и оператором PSAP.
