# <a name="unassigned-phone-number-create-new-or-edit-existing"></a>Неназначенный номер телефона: создание нового или редактирование существующего

Неназначенные номера — это телефонные номера, допустимые для вашей организации, но не назначенные пользователю или телефону. В таблице неназначенных номеров указывается, как обрабатываются такие номера.

> [!IMPORTANT]
> После того как вы завершите создание нового неназначенного диапазона номеров или редактируете существующий, нажмите кнопку **ОК** , чтобы вернуться на страницу неназначенный **номер** , на которой перечислены все диапазоны номеров. Изменения, внесенные на странице **"новый диапазон неназначенных номеров** " или " **изменить неназначенный номер** ", не фиксируются в базе данных, пока вы не нажимайте кнопку " **сохранить все** " на странице "неназначенные **номера** ".

## <a name="ui-reference"></a>Ссылка на пользовательский интерфейс

В следующем списке описываются поля на странице.

- **Name (имя** ) Введите описательное имя, определяющее неназначенный диапазон номеров. После сохранения диапазона это имя изменить нельзя.

- **Диапазон номеров** В первом поле введите начальный номер неназначенного диапазона номеров. Во втором поле Введите конечный номер диапазона.

  - Первый номер диапазона должен быть меньше или равен последнему номеру диапазона.

  - Если первый номер или последний номер диапазона содержит добавочный номер, то оба номера (первый и последний) должны содержать добавочный номер. Кроме того, добавочный номер должен быть одинаковым для первого и последнего номера диапазона.

  - Номер должен соответствовать регулярному выражению (Tel:) _Км_ (\+) _км_ [1-9] \d{0,17}(; ext = [1-9] \d{0,9}) _км_. Это означает, что номер может начинаться со строки Tel: (если вы не укажете, что она будет автоматически добавлена для вас), знак "плюс" (+) и цифра 1 – 9. Номер телефона может быть длиной до 17 цифр, и в конце может быть указан добавочный номер в формате ";ext=добавочный_номер".

- **Служба объявлений** Нажмите **** кнопку извещение, чтобы приложение извещения обрабатывало входящий звонок или **UM Exchange** , чтобы автосекретарь Exchange UM обрабатывал входящий звонок.

- Если вы выбрали службу **извещения** для **объявлений**, выполните указанные ниже действия.

  - **Полное доменное имя конечного сервера** Выберите идентификатор службы приложения, который запускает приложение объявлений, которое будет обрабатывать входящие звонки в этот диапазон неназначенных номеров.

  - **Объявление** Выберите извещение для воспроизведения по этому диапазону неназначенных номеров.

- Если вы выбрали службу **сообщений Exchange** для **службы объявлений**, сделайте следующее:

  - **Номер телефона** для автосекретаря Выберите номер телефона автоматического секретаря UM Exchange.

Дополнительные сведения о функциях и возможностях объявлений вы увидите [в разделе Планирование приложения для объявлений в Skype для бизнеса 2015](../../plan-your-deployment/enterprise-voice-solution/announcement.md) в документации по планированию.
