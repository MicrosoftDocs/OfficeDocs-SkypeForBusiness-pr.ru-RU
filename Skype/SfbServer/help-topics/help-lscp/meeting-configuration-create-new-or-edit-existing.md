# <a name="meeting-configuration-create-new-or-edit-existing"></a>Конфигурация собрания: создание новой или редактирование существующей

Параметры конфигурации собраний определяют взаимодействие с пользователем при присоединении к конференциям, запланированным пользователями. Эти параметры распространяются только на запланированные собрания. Они не применяются к непъемным собраниям, созданным с помощью параметра **"Выполнить сейчас"** в клиенте.

## <a name="ui-reference"></a>Ссылка на пользовательский интерфейс

В следующем списке описываются поля на странице.

- **Область действия** Определяет область создаемой или изменяемой конфигурации собрания: глобальную, на уровне сайта или пул.

- **Имя** Конфигурации собраний именуются по умолчанию и не могут быть изменены.

- **Вызыватели PSTN обходят "lobby"** Чтобы автоматически допустить пользователей, которые звонят в конференцию по телефонной линии телефонной сети общего звонков (PSTN), выберите этот этот переключатель. С этого момента можно отозвать звониющих по STN в "зал залов" конференции, где они находятся на удержании до тех пор, пока его не предосмотрит его.

- **Назначить в качестве presenter** Выберите категорию пользователей (помимо организатора собрания), которые автоматически назначаются в качестве участников конференции. Независимо от этого параметра, вы можете явным образом назначать в качестве presenters при запланированной конференции или явным образом назначать их в качестве presenter во время конференции. Доступны следующие варианты:

  - **Нет** Выберите этот параметр, если никто, кроме организатора, не назначается автоматически в качестве организатора.

  - **Компания** Выберите этот параметр, чтобы автоматически назначать в качестве участников только участников организации.

  - **Все** Выберите этот параметр, чтобы автоматически назначить кого-либо в качестве presenter.

- **Тип назначенной конференции по умолчанию** Этот параметр управляет тем, будет ли надстройка конференц-связи Outlook всегда планировать конференции с помощью назначенной организатором конференции, что означает, что запланированные конференции всегда имеют одинаковый URL-адрес join и аудиоконференции. Чтобы запланированные конференции всегда использовали один и тот же URL-адрес для пользования. Чтобы использовать для каждой конференции разные URL-адреса пользования, с помощью этого этого окна можно использовать разные URL-адреса пользования.

- **Допуск анонимных пользователей по умолчанию** Если анонимным пользователям (то есть неавтоматическим) разрешено по умолчанию участвовать в конференциях, этот этот ящик будет разрешен. Если анонимным пользователям по умолчанию запрещено участвовать в конференциях, с помощью этого данной сети можно ото всех пользователей.

- **URL-адрес логотипа** Введите URL-адрес с изображением, используемым для настраиваемого приглашения на конференцию.

- **URL-адрес справки** Введите URL-адрес веб-сайта, на котором пользователи могут получить помощь по присоединению к конференции.

- **URL-адрес юридического текста** Введите URL-адрес веб-сайта с юридическими сведениями и заявлением об отказе от ответственности для конференции.

- **Текст пользовательского footer** Введите текст, который будет использоваться в настраиваемом приглашении на конференцию.

Дополнительные сведения о работе с конфигурациями собраний см. в разделе [Create a or modify a Collection of Meeting Configuration Settings](https://technet.microsoft.com/library/ce6773c1-a0d5-4405-8e32-33a6f3a46a1a.aspx) документации по использованию. Дополнительные сведения о настройке конфигураций собраний для крупных собраний см. в разделе [Setting Up Support for Large Meetings](https://technet.microsoft.com/library/8e22d34b-b395-408d-9d48-8f2a3abe9513.aspx) документации по планированию.
