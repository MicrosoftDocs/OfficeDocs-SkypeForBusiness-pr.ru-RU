# <a name="location-policy-create-new-or-edit-existing"></a>Политика определения местонахождения: создание новой или редактирование существующей

Вы можете настроить политики расположения, чтобы определить, включена ли расширенная 9-1-1 (E9-1-1) и как она используется, а также как сведения о расположении используются для пользователей и контактов.

## <a name="ui-reference"></a>Ссылка на пользовательский интерфейс

В следующем списке описываются поля на странице.

- **Область** Определяет область политики расположения, которую вы создаете или модифицируют: глобальную, сайт или пользователя.

- **Имя** Для каждой политики расположения требуется имя. По умолчанию называются политики глобального и расположения сайтов, и имя не может быть изменено. Для политик расположения пользователей используйте описательное имя, которое идентифицирует пользователя или группу пользователей.

    > [!NOTE]
    > После сохранения политики расположения имя уже нельзя изменить.

- **Включить расширенный 9-1-1 (E9-1-1)** Выберите этот контрольный ящик, чтобы включить E9-1-1 для пользователей, которым назначена эта политика расположения.

- **Расположение** Укажите, запросили ли пользователи сведения о расположении:

  - **Обязательно** Выберите этот параметр, если пользователям будет предложено получить сведения о расположении при регистрации клиента в новом расположении. Пользователи могут отклонять запрос без ввода сведений о расположении.

  - **Не требуется** Выберите этот параметр, если пользователям не будут предложены сведения о расположении.

  - **Отказ от ответственности** Выберите этот параметр, если пользователям будет предложено получить сведения о расположении, но при отклонении запроса без ввода сведений увидите сообщение об отказе. Пользователи могут выполнить экстренный вызов, но без других вызовов, пока не введите сведения о расположении.

- **Используйте расположение только для E9-1-1** Выберите этот контрольный ящик, если сведения о расположении будут использоваться только для экстренных вызовов.

- **Использование PSTN** Выберите использование общедоступных переключеных телефонных сетей (PSTN), которое будет использоваться для определения маршрута голосовой связи для маршрутизов экстренных вызовов от клиентов, которые используют этот профиль. Маршрут, связанный с таким применением, должен указывать на SIP-магистраль, выделенную для экстренных вызовов, или на шлюз идентификационного номера экстренного местоположения (ELIN), который направляет экстренные вызовы в ближайший пункт общественной безопасности (PSAP). Параметры являются **внутренними,** **локальными** или **междугородние.**

- **Номер телефона E9-1-1** Укажите номер, который набирается для достижения экстренных служб.

- **Маска набора E9-1-1** Укажите номер, который набирает пользователь, который затем будет переведен на номер экстренного набора. Например, введите значение 212 в этом поле, чтобы пользователь может набрать 212, чтобы добраться до экстренных служб. Это позволяет вызывать альтернативные номера экстренных служб и по-прежнему вызывать экстренные службы (например, если кто-то из страны или региона с другим номером экстренной помощи пытается набрать номер этой страны или региона, а не номер для страны или региона, в который они сейчас находятся). Можно определить несколько масок набора номера экстренных служб, разделяя значения точками с запятыми. Например, 212;414. Максимальная длина строки — 100 символов. Все символы должны быть цифрами от 0 до 9.

    > [!IMPORTANT]
    > Убедитесь, что маска номера отличается от номера в диапазоне номеров парковки вызовов, так как маршрутизация парковки вызовов имеет приоритет над преобразованием строки набора номера экстренного вызова. Чтобы увидеть диапазоны номеров Парка вызовов, щелкните **Функции голосовой** почты в левой панели навигации и нажмите **кнопку Call Park**.

- **URI уведомлений** Укажите один или несколько URL-адресов SIP, которые будут уведомлены при экстренном вызове. Например, введите SIP URI службы безопасности компании, чтобы уведомить сотрудников службы безопасности мгновенным сообщением при экстренном вызове. Если доступно расположение вызываемой, расположение включено в уведомление. В списке, разделенных запятой, можно указать несколько URL-адресов SIP. Например, "sip:security@litwareinc.com","sip:kmyer@litwareinc.com". Строка может иметь длину от 1 до 256 знаков, и должна начинаться с префикса "sip:". Также можно указать списки рассылки.

- **URI конференции** Укажите SIP URI (телефонный номер, в данном случае) для третьей стороны, которая будет конференц-связи на экстренные вызовы. Например, введите номер телефона службы безопасности компании, чтобы они получали вызов при экстренном вызове. Параметр для режима **конференции** определяет, может ли сторонная сторона участвовать или просто прослушивать вызов. Длина строки должна быть от 1 до 256 символов, строка должна начинаться с префикса sip:.

- **Режим конференции** Если вы указали значение **для URI конференции,** установите это поле к одному из следующих значений:

  - **В одну сторону** Указывает, что третья сторона может прослушивать только вызов между вызываемой стороной и оператором PSAP.

  - **Двунабная** Указывает, что третья сторона может участвовать в вызове между вызываемой стороной и оператором PSAP.
