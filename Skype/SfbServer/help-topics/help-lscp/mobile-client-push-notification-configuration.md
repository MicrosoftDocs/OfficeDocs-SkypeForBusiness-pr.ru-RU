# <a name="mobile-client-push-notification-configuration"></a>Мобильный клиент: конфигурация push-уведомлений
 
Чтобы настроить **push-уведомления Майкрософт** и **push-уведомления Apple**, необходимо создать политику, определяющую типы требуемых push-уведомлений.
  
На экране основной конфигурации нажмите кнопку **Обновить** , чтобы обновить и повторного заполнения списка политик. Поле поиска обеспечивает сужение списка отображаемых политик. При вводе имя, которое выполняется поиск списка политик ограничивает автоматически.
  
> [!IMPORTANT]
> Параметры политики, которые применяются на уровне одной политики, могут переопределять параметры, применяемые на уровне другой политики. Приоритет политик: политика пользователя (самое большое влияние) переопределяет политику сайта, а политика сайта — глобальную политику (минимальное влияние). То есть, чем ближе параметр политики к объекту, на который она влияет, тем больше влияния она оказывает на объект. 
  
Для создания и редактирования политик доступны две команды:
  
1. **New**: щелкните, чтобы создать новую политику. Необходимо предоставить сайта для политики для применения к. Настройте параметры для push-уведомлений. Для **Настройка Push-уведомлений**можно только создать политик для сайтов, которые уже созданы.
    
2. **Изменение**: выберите политику и нажмите кнопку Изменить, чтобы выберите нужное действие из раскрывающегося списка. Можно изменять только создать или изменить глобальную политику сайтов:
    
   - **Показать подробности**: отображаются сведения о политике выделенный в текущий момент. Вы сможете вносить изменения в существующую политику.
    
   - **Выберите все**: Если количество политик и выберите все политики, нажмите кнопку Выбрать все
    
   - **Удаление**: приведет к удалению выбранной политики. С помощью **Выбрать все** и **удалите** будут удалены все политики
    
     > [!NOTE]
     > Не удается удалить **глобальную** политику по умолчанию. При попытке удалить ее, вы получите уведомление, что глобальную политику возвращенных значений по умолчанию (то есть, все параметры очищаются), но политика не может быть удалена.
  
Создание новой политики или редактирование существующей политики связано с двумя действиями:
  
- **Фиксация** Зафиксировать действие фиксации создает или обновляет политику и сохраняет изменения
    
- **Отмена** Действие "Отмена" отменяет все изменения, внесенные с момента последнего действия зафиксировать. Если вы отменить все изменения, внесенные будут потеряны.
    
Возможны два параметра для **Настройка Push-уведомлений**. Настройки, связанные с служб push-уведомлений для Microsoft и Apple. Включение push-уведомлений для любого из службы, установив флажок рядом с именем службы. Флажок снят, выбрав его снимите его. После выбора вы завершении или Отмена. Нажмите кнопку зафиксировать будет сохранить изменения в политике.
