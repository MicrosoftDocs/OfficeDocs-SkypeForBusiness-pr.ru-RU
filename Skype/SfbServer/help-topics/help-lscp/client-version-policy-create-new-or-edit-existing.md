# <a name="client-version-policy-create-new-or-edit-existing"></a>Политика версии клиента: создание новой или редактирование существующей

Вы может указать версию клиентов, поддерживаемых в вашей среде. При взаимодействии двух клиентов с разными версиями возможности, доступные обоим клиентам, могут быть ограничены возможностями другого клиента. Чтобы обеспечить максимально эффективное использование функций, включенных в Skype для бизнеса Server 2015, и улучшить общее взаимодействие с пользователем, вы можете использовать фильтр клиентской версии для ограничения версий клиентов, используемых в среде. Используя этот фильтр, вы также можете сократить расходы, связанные с поддержкой нескольких версий клиентов.

> [!IMPORTANT]
> Фильтры перечислены в списке в порядке приоритета. Например, если есть фильтр, который разрешает подключение клиентов с версией 1.5, за которым следует фильтр, блокирующий клиенты, версии которых предшествуют 2.0, преимуществом будет пользоваться первый фильтр, поэтому клиентам версии 1.5 будет разрешено выполнять подключение.

## <a name="tasks-you-can-perform"></a>Задачи, которые вы можете выполнить

На странице **Создание новой политики версий клиента** или **Изменение политики версий клиента** можно выполнить следующие задачи:

- добавить или изменить имя или описание политики версий клиентов;

- добавить или изменить правила версий клиентов для политики версий клиентов.

## <a name="ui-reference"></a>Ссылка на пользовательский интерфейс

В следующих списках описываются меню, команды, поля и свойства на этой странице.

- **Область действия** Определяет область (сайт, пул или пользователя) политики версии клиента.

- **Name (имя** ) Вы можете добавить или изменить имя политики версии клиента.

- **Description (описание** ) Вы можете добавить описание, которое поможет идентифицировать политику в списке на странице "Политика клиентской версии".

- **Новые** Вы можете добавить новое правило клиентской версии для политики.

- **Показать подробности** Этот параметр открывает диалоговое окно, в котором можно изменить параметры правила клиентской версии.

- **Удалить** Этот параметр удаляет выбранное правило клиентской версии из политики.

- **Стрелки вверх и вниз** Этот параметр Перемещает выбранное правило клиентской версии вверх или вниз по приоритету. Правила обрабатываются в указанном порядке.
