# <a name="client-version-policy-create-new-or-edit-existing"></a>Политика версии клиента: создание новой или редактирование существующей

Вы может указать версию клиентов, поддерживаемых в вашей среде. При взаимодействии двух клиентов с разными версиями возможности, доступные обоим клиентам, могут быть ограничены возможностями другого клиента. Важнейшим использования компонентов, содержащихся в Скайп для Business Server 2015 и улучшить работу пользователей в целом, можно использовать фильтр версий клиентов для ограничения версий клиентов, которые используются в вашей среде. Используя этот фильтр, вы также можете сократить расходы, связанные с поддержкой нескольких версий клиентов.

> [!IMPORTANT]
> Фильтры перечислены в списке в порядке приоритета. Например, если есть фильтр, который разрешает подключение клиентов с версией 1.5, за которым следует фильтр, блокирующий клиенты, версии которых предшествуют 2.0, преимуществом будет пользоваться первый фильтр, поэтому клиентам версии 1.5 будет разрешено выполнять подключение.

## <a name="tasks-you-can-perform"></a>Задачи, которые вы можете выполнить

На странице **Создание новой политики версий клиента** или **Изменение политики версий клиента** можно выполнить следующие задачи:

- добавить или изменить имя или описание политики версий клиентов;

- добавить или изменить правила версий клиентов для политики версий клиентов.

## <a name="ui-reference"></a>Ссылка на пользовательский интерфейс

В следующих списках описываются меню, команды, поля и свойства на этой странице.

- **Область** Определяет область (узел, пул или пользователей) политики версий клиентов.

- **Имя** Вы можете добавить или изменить имя политики версий клиентов.

- **Описание** Можно добавить описание, помогающее идентифицировать политику в списке на странице политика версий клиентов.

- **Новые** Можно добавить новое правило версий клиентов в политике.

- **Показать подробности** Этот параметр открывает диалоговое окно, в котором можно изменить параметры для правила версий клиентов.

- **Удаление** Этот параметр удаляет правило версий выбранного клиентов из политики.

- **Стрелки вверх и вниз** Этот параметр перемещает правило версий клиентов выбранного вверх или вниз в приоритет. Правила обрабатываются в указанном порядке.
