# <a name="plan-phone-system-in-office-365-with-on-premises-pstn-connectivity-in-skype-for-business-server"></a>Планирование телефонной системы в Office 365 с использованием локальной сети PSTN в Skype для бизнеса Server

Сведения о планировании телефонной системы в Office 365 (облачная УАТС) с локальной связью по протоколу КТСОП.

Это содержимое имеет смысл, если у вас уже есть Skype для бизнеса Server или Lync Server 2013, развернутый в локальной среде. Другие сценарии можно найти в разделе [решения Microsoft для телефонной связи](https://docs.microsoft.com/en-us/SkypeForBusiness/hybrid/msft-telephony-solutions).

 Телефонная система в Office 365 с локальным подключением PSTN позволяет использовать возможности телефонной системы (облачной УАТС) для пользователей. Это удобно в следующих сценариях.

- В Skype для бизнеса Online есть некоторые пользователи Skype для бизнеса, размещенные локально и другие. Теперь вы можете включить телефонную систему в возможности Office 365 для ваших пользователей в Skype для бизнеса Online, но продолжать использовать локальную сеть PSTN.

- У вас есть локальное развертывание и вы хотите переместить некоторых или всех пользователей в Skype для бизнеса Online, но продолжать использовать локальную сеть PSTN.

    > [!IMPORTANT]
    > Чтобы успешно включить пользователей для телефонной системы в Office 365 с локальной связью по протоколу PSTN, адрес SIP должен находиться в собственном домене. Использование домена по умолчанию для Office 365, onmicrosoft.com, не поддерживается. 

Дополнительные сведения о телефонной системе в Office 365, включая лицензирование и планы, можно найти в статье [планы звонков по коммутируемой телефонной связи для Skype для бизнеса](https://support.office.com/article/PSTN-Calling-plans-for-Skype-for-Business-f47c6a97-bc8b-42e6-b5d4-ce6b41ed1918).

Облачная УАТС с локальной связью по протоколу PSTN не предлагает один и тот же набор функций в рамках локального корпоративного голосового решения. Чтобы помочь вам решить, будет ли служба облачной АТС со встроенным подключением по протоколу КТСОП предоставлять вам подходящий компонент для вашей организации, ознакомьтесь [со статьей о том, что вы получаете в облачной УАТС](https://go.microsoft.com/fwlink/?LinkId=715517).

## <a name="benefits-and-planning-considerations"></a>Преимущества и рекомендации по планированию

Для устранения этой проблемы потребуется переместить пользователей обратно в локальную среду, чтобы обновить телефоны до минимальной версии встроенного ПО. Если аппаратный сброс выполняется на устройстве с устаревшей версией встроенного ПО, по умолчанию используется проверка подлинности PIN-кода, которая не поддерживается в Skype для бизнеса Online. Дополнительные сведения можно найти в статье [Знакомство с телефонами Skype для бизнеса Online](https://support.office.com/en-us/article/Getting-phones-for-Skype-for-Business-Online-91f2d947-45fc-4fab-bd8b-2e313531c477?ui=en-US&amp;rs=en-US&amp;ad=US).

Разворачивая телефонную систему в Office 365 с помощью локальной сети PSTN, вы можете переносить пользователей в облако через Skype для бизнеса Online на свой собственный темп, сохранив при этом локальную сеть PSTN. При наличии УАТС связь через ТСОП останется доступной для пользователей, перемещенных в облачную среду. Когда пользователь перемещается в Skype для бизнеса Online и телефонная система в Office 365, старый телефон УАТС больше не работает, но их номера будут направлены на любые клиенты Skype для бизнеса для ПК или смартфона, а также как Skype для бизнеса, совместимый с телефонным подключением. месяца. После переноса на телефонную систему в Office 365 пользователи и традиционные АТС смогут звонить друг другу, а также совершать и принимать звонки PSTN с помощью обычного номера телефона.

Возможно, у вас есть настраиваемая функция или основная надстройка для вашей старой АТС, например центр обработки звонков. Если настраиваемая функция в настоящее время недоступна в телефонной системе в Office 365, вы должны оставить пользователей, которым требуется настроить локальную функцию, с помощью старой АТС и просто перенести пользователей, которые не нуждаются в доступе к настраиваемым функциям в телефонной системе в Office 365. в локальной сети PSTN.

Список устаревших АТС, взаимодействующих непосредственно с Skype для Business Server 2015, см. [в разделе инфраструктура, определенная для Microsoft Lync](https://docs.microsoft.com/SkypeForBusiness/lync-cert/qualified-ip-pbx-gateway). Если вашей АТС нет в этом списке, вы можете использовать сеансовый контроллер, чтобы подключить УАТС к телефонной линии в Office 365 в Skype для бизнеса Online.

### <a name="network-considerations-for-quality-and-performance"></a>Характеристики сети для отслеживания качества и производительности

При развертывании облачной службы, такой как телефонная система в Office 365 с использованием локальной сети PSTN, необходимо учитывать указанные ниже моменты. При развертывании корпоративной голосовой связи в Skype для бизнеса Server 2015 вся инфраструктура и клиенты находятся в корпоративной сети. Показатели качества и производительности этой сети, которые являются критичными для высококачественного звука и видео, контролируются непосредственно работниками предприятия. Телефонная система в Office 365 со встроенным подключением по протоколу PSTN состоит из трех сетей, двух из которых отвечает сотрудник, но только один из которых имеет прямой контроль над:

- **Глобальная сеть доставки мультимедиа Майкрософт** Глобальная облачная сеть и инфраструктура Microsoft. Office 365 и телефонная система на серверах и трафике Office 365 проходят эту сеть.

- **Корпоративное и облачное коммутируемое соединение** Это сеть, соединяющая корпоративный сервер с облаком Office 365. Необязательно совпадает с общим подключением к Интернету.

- **Собственная сеть вашей организации** Качество мультимедиа в режиме реального времени сильно зависит от вашей сети: особенно сети Wi-Fi, а также качества межсетевого соединения, используемого для доступа к облаку Office 365.

> [!NOTE]
> Дополнительные сведения о настройке производительности в Skype для бизнеса Online можно найти в разделе [Настройка производительности Skype для бизнеса Online](https://support.office.com/en-us/article/Tune-Skype-for-Business-Online-performance-beec23c2-c5d6-4e84-a8af-e82aefca7802?ui=en-US&amp;rs=en-US&amp;ad=US).
