# <a name="plan-phone-system-with-on-premises-pstn-connectivity-in-skype-for-business-server"></a>Планирование телефонной системы с соединением на локальной ТСОП в Skype для бизнеса Server

> [!Important]
> Skype для бизнеса Online будет прекращен до 31 июля, 2021 после чего служба станет недоступна.  Кроме того, связь по протоколу PSTN между локальной средой и в Skype для бизнеса Server или Cloud Connector Edition и Skype для бизнеса Online больше не будет поддерживаться.  Узнайте, как подключить локальную телефонную сеть к Teams с помощью [прямой маршрутизации](https://docs.microsoft.com/MicrosoftTeams/direct-routing-landing-page).

Сведения о планировании телефонной системы (облачной УАТС) с подключением локальной сети PSTN.

Это содержимое уместно, если вы уже развернули локальное Skype для бизнеса Server или Lync Server 2013. Для других сценариев можно ознакомиться с [разрешениями Microsoft для телефонии](https://docs.microsoft.com/SkypeForBusiness/hybrid/msft-telephony-solutions).

 Телефонная система с локальной связью PSTN позволяет использовать возможности телефонной системы (облачной УАТС) для пользователей. Это может помочь в следующих сценариях:

- У вас есть некоторые пользователи Skype для бизнеса, размещенные в локальной среде, а другие — в Skype для бизнеса Online. Теперь вы можете включить функции и функции телефонной системы для пользователей, размещенных в Skype для бизнеса Online, но продолжать использовать локальную сеть PSTN.

- У вас есть локальное развертывание, и вы хотите переместить некоторых или всех пользователей в Skype для бизнеса Online, но продолжать использовать локальную сеть PSTN.

    > [!IMPORTANT]
    > Для успешного включения телефонной системы для пользователей с локальной сетью PSTN адрес SIP должен находиться в собственном домене. Использование домена по умолчанию для Microsoft 365 или Office 365, onmicrosoft.com не поддерживается. 

Дополнительные сведения о телефонной системе, в том числе о лицензировании и планах, приведены в статье [планы звонков PSTN для Skype для бизнеса](https://support.office.com/article/PSTN-Calling-plans-for-Skype-for-Business-f47c6a97-bc8b-42e6-b5d4-ce6b41ed1918).

## <a name="feature-comparison"></a>Сравнение функций

Облачная УАТС с подключением по локальной сети PSTN не предоставляет тот же набор функций, что и полностью локальное Корпоративная голосовая связь. Чтобы принять решение о том, что облачная УАТС с локальной связью PSTN предоставит вам подходящий набор функций для вашей организации, ознакомьтесь [с возможностями облачной УАТС](https://go.microsoft.com/fwlink/?LinkId=715517).

## <a name="benefits-and-planning-considerations"></a>Преимущества и рекомендации по планированию

> [!CAUTION]
> Устройства Lync Phone Edition должны быть обновлены до минимального требуемого встроенного по в локальной среде до перехода на Skype для бизнеса Online.
Если вы перемещаете пользователей из локальной среды в Интернет до обновления встроенного по, пользователи не смогут подключаться с помощью своих телефонов. Чтобы устранить эту проблему, пользователи должны быть возвращены в локальную среду, чтобы их телефоны обновлялись до минимальной версии встроенного по. НЕ ПЫТАЙТЕСЬ ВЫПОЛНИТЬ ОБНОВЛЕНИЕ ДО МИНИМАЛЬНОЙ ВЕРСИИ ВСТРОЕННОГО ПО ИЛИ АППАРАТНЫЙ СБРОС ТЕЛЕФОНА ПЕРЕД ПЕРЕМЕЩЕНИЕМ ПОЛЬЗОВАТЕЛЯ В ЛОКАЛЬНУЮ СРЕДУ.
Если при выполнении аппаратной перегрузки не указана минимальная версия встроенного по, по умолчанию будет использоваться проверка подлинности ПИН-кодов, которая не поддерживается в Skype для бизнеса Online. Для получения дополнительных сведений обратитесь к разделу доступ к [телефонам Skype для бизнеса Online](https://support.office.com/article/Getting-phones-for-Skype-for-Business-Online-91f2d947-45fc-4fab-bd8b-2e313531c477?ui=en-US&amp;rs=en-US&amp;ad=US).

При развертывании телефонной системы с локальной сетью PSTN вы можете переместить пользователей в облако с помощью Skype для бизнеса Online в удобном для вас темпе, сохранив их в локальной сети PSTN. Если у вас есть УАТС, вы продолжаете использовать ее для предоставления подключений PSTN для пользователей, которые вы перемещаете в облако. Когда пользователь перемещается в Skype для бизнеса Online и телефонная система, его устаревший телефон УАТС перестанет работать, но их номер будет направляться на любой клиент Skype для бизнеса для ПК или смартфонов, а также для мобильных телефонов, совместимых с Skype для бизнеса. После переноса пользователи телефонной системы и устаревшие пользователи УАТС могут звонить друг другу, а также совершать и принимать звонки PSTN с использованием обычного номера телефона.

У вас может быть настраиваемая функция или основная надстройка для вашей устаревшей УАТС, например, центр обработки звонков. Если настраиваемый компонент в настоящее время недоступен в телефонной системе, следует оставить пользователей, которым требуется специальная функция, в локальной системе УАТС и просто перенести пользователей, которым не требуется доступ к пользовательскому компоненту, в телефонную систему с локальной сетью PSTN.

Список устаревших УАТС, которые взаимодействуют непосредственно со Skype для бизнеса Server 2015, можно найти в  [статье инфраструктура, квалифицированная для Microsoft Lync](https://docs.microsoft.com/SkypeForBusiness/lync-cert/qualified-ip-pbx-gateway). Если УАТС нет в этом списке, вы можете подключить УАТС к телефонной линии в Skype для бизнеса Online с помощью пограничного контроллера сеансов.

### <a name="network-considerations-for-quality-and-performance"></a>Особенности сети для обеспечения качества и производительности

При развертывании облачной службы, такой как телефонная система с локальной связью PSTN, необходимо учитывать следующие моменты. В чистом локальном развертывании Skype для бизнеса Server 2015 Enterprise Voice вся инфраструктура и клиенты находятся в собственной сети предприятия. Качество и производительность этой сети, которая очень важна для высококачественного звука и видео, находится за прямым управлением корпоративными сотрудниками. В телефонной системе с подключением по локальной сети PSTN существует три сети, два из которых являются ответственными за непосредственный контроль за одним из сотрудников предприятия.

- **Глобальная сеть доставки мультимедиа Майкрософт** Глобальная облачная сеть и инфраструктура Майкрософт. Серверы и трафик телефонной системы просматривают эту сеть.

- **Межкорпоративный и облачный соединения PSTN** Это сеть, в которой предприятие подключается к облаку. Это не обязательно то же самое, что общий Интернет-подключение.

- **Собственная сеть вашей организации** Качество мультимедиа в режиме реального времени сильно зависит от вашей собственной сети: особенно сети Wi-Fi и качества соединения, используемого для достижения облака.

> [!NOTE]
> Дополнительные сведения о настройке производительности в Skype для бизнеса Online можно найти в статье [Настройка производительности Skype для бизнеса Online](https://support.office.com/article/Tune-Skype-for-Business-Online-performance-beec23c2-c5d6-4e84-a8af-e82aefca7802?ui=en-US&amp;rs=en-US&amp;ad=US). 

## <a name="prerequisites-for-using-phone-system-with-on-premises-pstn-connectivity"></a>Необходимые условия для использования телефонной системы с локальной связью PSTN

Перед настройкой телефонной системы с локальной связью PSTN и перемещению пользователей в Skype для бизнеса Online необходимо убедиться в наличии следующих необходимых компонентов.

 **Локальные версии сервера.** Версии серверов в локальном развертывании должны быть указаны в следующей таблице для поддержки телефонной системы в локальной сети PSTN.


| **Роль сервера**                                       | **Поддерживаемые версии\\**\*                                                                                         |
|:------------------------------------------------------|:-------------------------------------------------------------------------------------------------------------------|
| Край Федерации\*\*  <br/>                            | Skype для бизнеса Server 2015  <br/>                                                                              |
| Внутренний сервер внутреннего пула маршрутизации Федерации следующего прыжка  <br/> | Skype для бизнеса Server 2015, 2016 марта, накопительный пакет обновления 6.0.9319.235 или более поздней версии (внешний интерфейс или директор)  <br/> |
| Сервер переднего плана пользователя  <br/>                          | Skype для бизнеса Server 2015  <br/> Lync Server 2013  <br/>                                                      |
| Пограничный сервер  <br/>                                    | Skype для бизнеса Server 2015  <br/>                                                                              |
| Mediation Server (Сервер-посредник);  <br/>                               | Skype для бизнеса Server 2015  <br/> Lync Server 2013  <br/>                                                      |

\*Минимальные поддерживаемые версии:

- Skype для бизнеса Server 2015, марта 2016 (накопительный пакет обновления 6.0.9319.235)

- Lync Server 2013 июль 2015 накопительный пакет обновления 5.0.8308.920

\*\*Маршрут Федерации для всех поддерживаемых доменов SIP должен маршрутизироваться через пограничный сервер Skype для бизнеса Server 2015, на котором установлен накопительный пакет обновления от 2016 марта или более поздней версии. Если пул пользователей находится на Lync Server 2013, трафик внешнего пула остается на пограничном сервере Lync Server 2013. 

Кроме того, необходимо убедиться в следующем.

- **Локальная Корпоративная голосовая связь настраивается и проверяется для локальных пользователей** Сюда входят компоненты подключения PSTN. Дополнительные сведения см. в следующих разделах, если вы используете Skype для бизнеса Server 2015, ознакомьтесь со статьей [планирование корпоративной голосовой связи в Skype для бизнеса server 2015](../../plan-your-deployment/enterprise-voice-solution/enterprise-voice.md) и [Развертывание корпоративной голосовой связи в Skype для бизнеса Server 2015](../../deploy/deploy-enterprise-voice/deploy-enterprise-voice.md).

- **Синхронизация Active Directory** Необходимо настроить синхронизацию Active Directory с помощью Azure AD Connect. Дополнительные сведения см. в статье [Управление Azure AD Connect](https://azure.microsoft.com/documentation/articles/active-directory-aadconnect-whats-next/).

    > [!NOTE]
    > Необходимо использовать версию подключения AAD версии 1.0.9125.0 или более поздней. Если вы используете более раннюю версию средств AAD Connect или DirSync, обновите ее до поддерживаемой версии. Вы можете обновить текущую установку и сохранить все настраиваемые правила, определенные в вашей среде. 

- **Настройка гибридного развертывания** Независимо от того, все ли пользователи Skype для бизнеса размещены в сети или в локальной среде, или если у вас в данный момент есть смесь, необходимо выполнить действия по настройке гибридного развертывания Skype для бизнеса Server или Lync Server 2013, как описано в статье [развертывание гибридных подключений между Skype для бизнеса Server и Office 365](../../skype-for-business-hybrid-solutions/deploy-hybrid-connectivity/deploy-hybrid-connectivity.md). Для получения дополнительных сведений ознакомьтесь со статьей [службы федерации Active Directory (AD FS)](https://technet.microsoft.com/library/cc736690%28v=ws.10%29.aspx).
