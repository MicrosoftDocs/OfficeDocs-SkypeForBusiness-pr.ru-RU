# <a name="plan-phone-system-in-office-365-with-on-premises-pstn-connectivity-in-skype-for-business-server"></a>Планирование телефонной системой в Office 365 с помощью локального подключения к ТСОП в Скайп Business Server
 
Сведения о вопросы планирования для телефонной системой в Office 365 (облако УАТС), а также подключение к ТСОП в локальной.
  
Этот контент относится, если вы уже Скайп Business Server или Lync Server 2013, развернутые локально. Остальные сценарии в разделе [Планирование телефонной системой в решение Office 365 (облако УАТС)](plan-your-phone-system-cloud-pbx-solution.md).
  
 Система телефона в Office 365 с помощью подключения к ТСОП в локальной позволяет использовать возможности телефонной системой (облако УАТС) для пользователей. Это удобно в следующих сценариях.
  
- У вас есть некоторые из вашего Скайп для бизнеса пользователи, размещенные локально и другим пользователям, размещенными в Скайп для бизнеса в Интернет. Теперь можно включить телефонной системой в возможностях Office 365 и компонентов для пользователей, размещенных в Скайп для бизнеса в Интернет, но продолжить использовать подключение к ТСОП в локальной.
    
- У вас есть локального развертывания и вы хотите переместить некоторые или все пользователи Скайп для бизнеса в Интернет, но продолжить использовать подключение к ТСОП в локальной.
    
    > [!IMPORTANT]
    > Чтобы успешно пользователей для телефонной системой в Office 365 с помощью подключения к ТСОП в локальной, SIP-адрес значения в собственном домене. Использование домена по умолчанию для Office 365, onmicrosoft.com, не поддерживается. 
  
Для получения дополнительных сведений о телефонной системой, в том числе лицензирования и планы Office 365, видеть [вызов ТСОП планов для Скайп для бизнеса](https://support.office.com/article/PSTN-Calling-plans-for-Skype-for-Business-f47c6a97-bc8b-42e6-b5d4-ce6b41ed1918).
  
## <a name="feature-comparison"></a>Сравнение функций

Облако УАТС с подключение к ТСОП в локальной не предоставляет набор как полностью локальной корпоративной голосовой связи решение же компонентов. Чтобы помочь вам решить, обеспечивают ли облачных УАТС с локального подключения к ТСОП правом набор функций для вашей организации, см [вот вы получаете с УАТС облака](https://go.microsoft.com/fwlink/?LinkId=715517).
  
## <a name="benefits-and-planning-considerations"></a>Преимущества и рекомендации по планированию

> [!CAUTION]
> Устройства Lync Phone Edition ДОЛЖНЫ быть обновлены до минимальной версии требуемого встроенного ПО в локальной среде, ДО перехода на Skype для бизнеса Online.
Если перенести пользователей из локальной среды в сетевую версию до обновления встроенного ПО, пользователи не смогут подключаться с помощью телефонов. Для устранения этой проблемы потребуется переместить пользователей обратно в локальную среду, чтобы обновить телефоны до минимальной версии встроенного ПО. НЕ ПЫТАЙТЕСЬ ВЫПОЛНИТЬ ОБНОВЛЕНИЕ ДО МИНИМАЛЬНОЙ ВЕРСИИ ВСТРОЕННОГО ПО ИЛИ АППАРАТНЫЙ СБРОС НА ТЕЛЕФОНЕ ДО ПЕРЕМЕЩЕНИЯ ПОЛЬЗОВАТЕЛЯ ОБРАТНО В ЛОКАЛЬНУЮ СРЕДУ.
Если аппаратный сброс выполняется на устройстве с устаревшей версией встроенного ПО, по умолчанию используется проверка подлинности PIN-кода, которая не поддерживается в Skype для бизнеса Online. Для получения дополнительных сведений обратитесь за помощью к [телефонах начало для Скайп для бизнеса в Интернет](https://support.office.com/en-us/article/Getting-phones-for-Skype-for-Business-Online-91f2d947-45fc-4fab-bd8b-2e313531c477?ui=en-US&amp;rs=en-US&amp;ad=US).
  
Развертывание телефонной системой в Office 365 с помощью подключения к ТСОП в локальной, можно переместить вашей пользователей в облако с помощью Скайп для бизнеса в Интернет на рабочем месте сохраняя их подключение к ТСОП в локальной. При наличии УАТС связь через ТСОП останется доступной для пользователей, перемещенных в облачную среду. После перемещения пользователя Скайп для бизнеса в Интернет и телефонной системой в Office 365 их прежних версий телефон УАТС больше не будет работать, но телефонный номер перейдете к любому Скайп пользователей для ПК и смартфоны, а также Скайп для бизнес требованиям стационарный телефон s. После переноса, телефонной системой в Office 365 и устаревшие УАТС пользователей можно вызвать друг с другом, а также сделать и получения звонки по ТСОП с помощью обычный телефонный номер.
  
Может потребоваться пользовательского компонента или основных дополнительный компонент традиционной УАТС, такие как центр обработки вызовов. Если настраиваемый компонент в настоящее время недоступна в системе телефона в Office 365, следует оставить этих пользователей, которым требуется, настраиваемый компонент локальную систему с традиционной УАТС и порт только пользователей, которым не требуется доступ к пользовательской функции для телефонной системой в Office 365 с помощью локального подключения к ТСОП.
  
Полный список устаревшие УАТС, взаимодействующих непосредственно с Скайп для Business Server 2015 представлен [Полный инфраструктуры для Microsoft Lync](https://technet.microsoft.com/en-us/office/dn788945.aspx). Если эту УАТС нет в списке, можно использовать пограничный контроллер сеансов для подключения к АТС с телефонной системой в Office 365 в Скайп для бизнеса в Интернет.
  
### <a name="network-considerations-for-quality-and-performance"></a>Характеристики сети для отслеживания качества и производительности

При развертывании службы размещаемых в облаке как телефонной системой в Office 365 с помощью подключения к ТСОП в локальной, необходимо помнить о следующих. В Скайп исключительно локального развертывания Business Server 2015 корпоративной голосовой связи инфраструктуры и клиентов, в сети предприятия собственные. Показатели качества и производительности этой сети, которые являются критичными для высококачественного звука и видео, контролируются непосредственно работниками предприятия. С телефонной системой в Office 365 с помощью подключения к ТСОП в локальной существует три по сети, два из которых несет ответственность за клиента, но только один из которых сотрудников предприятия имеет прямой доступ через:
  
- **Сети доставки Global Media корпорации Майкрософт** Облако глобальной сети и инфраструктуры корпорации Майкрософт. Office 365 и телефонной системой в Office 365 серверов и трафика проходят через сеть.
    
- **Interconnect Enterprise/облачных ТСОП** Это сеть с выходом предприятия в облако Office 365. Необязательно совпадает с общим подключением к Интернету.
    
- **В собственной сети предприятия** Качество мультимедиа в режиме реального времени, сильно зависит от собственной сети: особенно сети WiFi и качества interconnect, включающий в облако Office 365.
    
> [!NOTE]
> Дополнительные сведения о настройке производительности в Скайп для бизнеса в Интернет содержатся в разделе [Настройка параметров Скайп для бизнеса в Интернет производительности](https://support.office.com/en-us/article/Tune-Skype-for-Business-Online-performance-beec23c2-c5d6-4e84-a8af-e82aefca7802?ui=en-US&amp;rs=en-US&amp;ad=US). 
  
## <a name="prerequisites-for-using-phone-system-in-office-365-with-on-premises-pstn-connectivity"></a>Необходимые условия для использования телефонной системой в Office 365 с помощью локального подключения к ТСОП

Перед настройкой телефонной системой в Office 365 с помощью подключения к ТСОП в локальной и перемещение пользователей к Скайп для бизнеса в Интернет, необходимо подтвердить, что имеются следующие необходимые компоненты на месте:
  
 **Версии локального сервера.** Версии серверов в локальном развертывании должны быть перечислены в следующей таблице для поддержки телефонной системой в Office 365 с помощью локального подключения к ТСОП.
  
|**Роль сервера**|**Поддерживаемые версии\***|
|:-----|:-----|
|Пограничного сервера федерации\*\*  <br/> |Skype для бизнеса Server 2015  <br/> |
|Внутренний сервер пула маршрута федерации следующего прыжка  <br/> |Skype для бизнеса Server 2015, март 2016 г., накопительный пакет обновления 6.0.9319.235 или более поздней версии (внешний интерфейс или директор)  <br/> |
|Сервер пользователя переднего плана  <br/> |Skype для бизнеса Server 2015  <br/> Lync Server 2013  <br/> |
|Пограничный сервер  <br/> |Skype для бизнеса Server 2015  <br/> |
|Сервер-посредник  <br/> |Skype для бизнеса Server 2015  <br/> Lync Server 2013  <br/> |
   
\*Приведены минимальные поддерживаемых версий.
  
- Skype для бизнеса Server 2015, накопительное обновление за март 2016 г. 6.0.9319.235
    
- Lync Server 2013, накопительное обновление за июль 2015 г. 5.0.8308.920
    
\*\*Маршрут федерации для всех поддерживаемых доменов SIP должны направляться через Скайп для выполнения пограничного сервера Business Server 2015 марта 2016 или более поздней версии накопительного обновления. Если пул пользователей находится на Lync Server 2013, трафик этого внешнего пула остается на пограничном сервере Lync Server 2013. 
  
В дополнение к этому необходимо проверить следующее:
  
- **В локальной корпоративной голосовой связи, настроена и проверен для локальных пользователей** Этот компонент включает компоненты подключения к ТСОП. Дополнительные сведения содержатся в следующих разделах, если вы используете Скайп для Business Server 2015, см [Планирование корпоративной голосовой связи в Скайп для Business Server 2015](../../plan-your-deployment/enterprise-voice-solution/enterprise-voice.md) и [Развертывание корпоративной голосовой связи в Скайп для Business Server 2015](../../deploy/deploy-enterprise-voice/deploy-enterprise-voice.md).

- **Синхронизация Active Directory** Необходимо настроить синхронизации Active Directory с помощью Azure AD подключение. Для получения дополнительных сведений см [Управление Azure AD](https://azure.microsoft.com/en-us/documentation/articles/active-directory-aadconnect-whats-next/).

- **Настройка гибридного развертывания** Ли все вашей Скайп для бизнес-пользователей в настоящий момент, размещенных либо через Интернет или локальной, или при наличии в настоящее время сочетание разных типов, как описано в [развертывания гибридного необходимо выполнить действий по настройке гибридного развертывания Скайп Business Server и Lync Server 2013 подключение между Скайп для Business Server и Скайп для бизнеса в Интернет](../../skype-for-business-hybrid-solutions/deploy-hybrid-connectivity/deploy-hybrid-connectivity.md).     При использовании Lync Server 2013 видеть [гибридной среды Lync Server 2013](https://technet.microsoft.com/EN-US/library/jj204805%28v=ocs.15%29.aspx).
