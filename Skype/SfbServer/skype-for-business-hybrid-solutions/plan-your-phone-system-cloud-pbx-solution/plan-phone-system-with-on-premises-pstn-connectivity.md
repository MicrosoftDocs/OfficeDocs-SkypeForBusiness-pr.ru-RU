# <a name="plan-phone-system-with-on-premises-pstn-connectivity-in-skype-for-business-server"></a>Планирование телефонной системы с соединением по локальной ТСОП в Skype для бизнеса Server

> [!Important]
> Skype для бизнеса Online будет отменен 31 июля 2021 г., после чего служба будет больше недоступна.  Кроме того, подключение PSTN между локальной средой, будь то Skype для бизнеса Server или Cloud Connector Edition и Skype для бизнеса Online, больше не будет поддерживаться.  Узнайте, как подключить сеть локальной телефонии к Teams с помощью [прямого маршрутного маршрутинга.](/MicrosoftTeams/direct-routing-landing-page)

Узнайте о соображениях планирования телефонной системы (Cloud PBX) с локальной подключением PSTN.

Этот контент актуален, если на локальном сервере уже развернуты Skype для бизнеса Server или Lync Server 2013. В других сценариях см. [решения для телефонии Майкрософт.](/microsoftteams/cloud-voice-landing-page)

 Телефонная система с локальной подключением PSTN позволяет использовать возможности Phone System (Cloud PBX) для пользователей. Это может помочь в следующих сценариях:

- Некоторые пользователи Skype для бизнеса имеют локальное жилье, а другие — в Skype для бизнеса Online. Теперь вы можете включить возможности и функции телефонной системы для пользователей, обустроив их в Skype для бизнеса Online, но продолжить использовать локальное подключение к PSTN.

- У вас есть локальное развертывание, и вы хотите переместить некоторых или всех пользователей в Skype для бизнеса Online, но продолжать использовать локальное подключение PSTN.

    > [!IMPORTANT]
    > Чтобы успешно включить пользователей для телефонной системы с локальной подключением PSTN, их SIP-адрес должен быть в вашем домене. Использование домена по умолчанию для Microsoft 365 или Office 365 onmicrosoft.com не поддерживается. 

Дополнительные информацию о телефонной системе, включая лицензирование и планы, см. в приложениях [PSTN Calling plans for Skype for Business.](https://support.office.com/article/PSTN-Calling-plans-for-Skype-for-Business-f47c6a97-bc8b-42e6-b5d4-ce6b41ed1918)

## <a name="benefits-and-planning-considerations"></a>Преимущества и соображения планирования

> [!CAUTION]
> Устройства Lync Phone Edition должны быть обновлены до минимально необходимого прошивки в локальной среде перед переходом в Skype для бизнеса Online.
Если перед обновлением прошивки переместить пользователей из локального в интернет, пользователи не смогут подключиться с помощью своих телефонов. Чтобы устранить эту проблему, необходимо перенаправить пользователей в локальное пространство, чтобы их телефоны обновились до минимального уровня прошивки. НЕ ПЫТАЙТЕСЬ ОБНОВИТЬ ДО МИНИМАЛЬНОГО ПРОШИВКИ ИЛИ ЖЕСТКОГО СБРОСА ТЕЛЕФОНА ДО ПЕРЕМЕЩЕНИЯ ПОЛЬЗОВАТЕЛЯ ОБРАТНО В ЛОКАЛЬНОЙ СРЕДЕ.
Если жесткий сброс выполняется, пока устройство не имеет минимального прошивки, оно по умолчанию будет использовать ПИН-код проверки подлинности, которая не поддерживается в Skype для бизнеса Online. Дополнительные сведения можно получить в ссылке [Получение телефонов для Skype для бизнеса Online](https://support.office.com/article/Getting-phones-for-Skype-for-Business-Online-91f2d947-45fc-4fab-bd8b-2e313531c477?ui=en-US&amp;rs=en-US&amp;ad=US).

Развертывая телефонную систему с локальной возможностью подключения к PSTN, вы можете перемещать пользователей в облако через Skype для бизнеса Online в своем собственном темпе, сохраняя при этом локальное подключение к PSTN. Если у вас есть PBX, вы продолжаете использовать его для обеспечения подключения PSTN для пользователей, которые перемещаются в облако. После того как пользователь будет перемещен в Skype для бизнеса Online и Телефонная система, его старый PBX-телефон больше не будет работать, но их номер телефона будет маршрут к любому из клиентов Skype для бизнеса для компьютеров или смартфонов, а также Skype для бизнес-совместимых настольных телефонов. После переноса пользователи телефонной системы и устаревшие пользователи PBX могут звонить друг другу нормально, а также делать и принимать вызовы PSTN с помощью обычного номера телефона.

Возможно, у вас есть настраиваемая функция или основная надстройка к устаревшему PBX, например центр обработки вызовов. Если настраиваемая функция в настоящее время недоступна в телефонной системе, следует оставить тех пользователей, которые требуют, чтобы эта настраиваемая функция была локальной, с устаревшим PBX, и просто портить пользователей, которым не нужно получать доступ к настраиваемой функции в телефонной системе с локальной подключением PSTN.

Список устаревших PBX, которые напрямую связаны с Skype для бизнеса Server 2015, см. в этой [ссылке: Infrastructure Qualified for Microsoft Lync.](../../../SfbPartnerCertification/lync-cert/qualified-ip-pbx-gateway.md) Если вашего PBX нет в этом списке, вы можете использовать пограничный контроллер сеанса для подключения PBX с телефонной системой в Skype для бизнеса Online.

### <a name="network-considerations-for-quality-and-performance"></a>Сетевые соображения по качеству и производительности

При развертывании облачной службы, такой как Phone System с локальной возможностью подключения к PSTN, необходимо помнить следующее. В чисто локальном развертывании Skype for Business Server 2015 Корпоративная голосовая связь, вся инфраструктура и клиенты находятся в собственной сети предприятия. Качество и производительность этой сети, которая имеет решающее значение для высококачественных аудио- и видеоматериалов, находятся под непосредственным контролем сотрудников предприятия. С помощью телефонной системы с локальной связью PSTN участвуют три сети, за две из которых отвечает клиент, но только одна из которых непосредственно контролируется сотрудниками предприятия:

- **Глобальная сеть доставки** мультимедиа Майкрософт Глобальная облачная сеть и инфраструктура Майкрософт. Серверы телефонной системы и трафик пересекают эту сеть.

- **Корпоративный и облачный PSTN Interconnect** Это сеть, которая соединяет ваше предприятие с облаком. Это не обязательно совпадает с общим подключением к Интернету.

- **Собственная сеть предприятия** Качество мультимедиа в режиме реального времени в значительной степени зависит от вашей сети: особенно от сети WiFi и от качества интерконнекта, используемого для достижения облака.

> [!NOTE]
> Дополнительные сведения о настройке производительности в Skype для бизнеса Online см. в дополнительных сведениях о производительности [Tune Skype для бизнеса Online.](https://support.office.com/article/Tune-Skype-for-Business-Online-performance-beec23c2-c5d6-4e84-a8af-e82aefca7802?ui=en-US&amp;rs=en-US&amp;ad=US)
