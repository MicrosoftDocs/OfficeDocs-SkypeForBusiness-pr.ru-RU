# <a name="enable-the-users-for-enterprise-voice-on-premises"></a>Включение пользователей для корпоративной голосовой связи локально
 
Пользователи могут с помощью телефонной системой в Office 365 (облако УАТС) необходимо включить их для корпоративной голосовой связи и назначить их номер телефона. Это можно сделать с помощью локального развертывания, пока пользователь по-прежнему размещенный в локальном развертывании.
  
### <a name="to-enable-a-user-for-enterprise-voice-on-premises-and-assign-a-phone-number"></a>Чтобы разрешить пользователям работу с корпоративной голосовой связи на локальном и назначить номер телефона

1. Войдите на любой компьютер во внутреннем развертывании с использованием учетной записи пользователя, назначенной роли CsUserAdministrator или CsAdministrator.
    
2. Откройте панель управления Skype для бизнеса Server, используя меню "Пуск" или ярлык на рабочем столе.
    
    Кроме того, можно открыть окно браузера и ввести URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server.
    
3. На панели навигации слева щелкните **Пользователи**.
    
4. В поле **Поиск пользователей** введите отображаемое имя или его начальный фрагмент, имя, фамилию, имя учетной записи (диспетчера учетных записей безопасности (SAM), SIP-адрес или строку универсального кода ресурса (URI) из учетной записи пользователя, которому требуется предоставить доступ, затем нажмите кнопку **Найти**.
    
5. В таблице щелкните Скайп для бизнеса в Интернет учетная запись пользователя, необходимо включить для корпоративной голосовой связи.
    
6. В меню **Правка** выберите пункт **Показать подробности**.
    
7. В разделе **Телефония** щелкните **Корпоративная голосовая связь**.
    
8. Щелкните **URI строки** и введите уникальный нормализованный номер телефона (например, tel:+14255550200). Затем нажмите кнопку **Сохранить**.
    
## <a name="special-considerations-when-enabling-users-for-enterprise-voice-on-premises"></a>Что необходимо учитывать при включении пользователей корпоративной голосовой связи локально

В некоторых случаях может потребоваться изменить способ предоставления пользователям доступа к корпоративной голосовой связи, чтобы они могли успешно звонить и принимать вызовы. Если у пользователей в развертывании, которые соответствуют условиям, выполните действия, включенные для включения пользователя для корпоративной голосовой связи.
  
- Если пользователь будет создан в своей локальной AD и синхронизации с Скайп для бизнеса в Интернет без включаемого для Скайп для бизнеса или для корпоративной голосовой связи и нет LineURI значение, выполните следующие командлеты для каждого соответствующего пользователя, заменив значения в < C0 настроек <b1></b1> с фактических значений для вашей среды:
    
  ```
  Enable-CsUser $username -HostingProvider sipfed.online.lync.com -SipAddress sip:<UserName>@<SIP Domain>
  ```

  ```
  Set-CsUser $username -EnterpriseVoiceEnabled $true -LineUri "tel:+<Telephone Number>"
  ```

- Если пользователь уже включен для Скайп для бизнеса локально, но не был включен для корпоративной голосовой связи или назначенных LineURI перед перемещается в Скайп для бизнеса в Интернет, выполните следующий командлет для каждого пользователя:
    
  ```
  Set-CsUser $username -EnterpriseVoiceEnabled $true -LineUri "tel:+<Telephone Number>"
  ```

- Если пользователь уже включен в Скайп для бизнеса локально, но не включен для корпоративной голосовой связи, даже в том случае, если они уже назначены LineURI, выполните следующий командлет для каждого пользователя:
    
  ```
  Set-CsUser $username -EnterpriseVoiceEnabled $true
  ```
