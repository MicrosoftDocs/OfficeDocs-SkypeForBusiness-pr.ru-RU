# <a name="plan-to-upgrade-to-skype-for-business-server-2015"></a>Планирование обновления до Skype для бизнеса Server 2015
 
Сводка. сведения о том, что следует учитывать при планировании обновления до Skype для бизнеса Server 2015. Скачайте бесплатную пробную версию Skype для бизнеса Server 2015 в центре оценки Майкрософт по адресу [https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server](https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server):.
  
В рамках вашего плана по обновлению до Skype для бизнеса Server 2015 используйте этот раздел, чтобы узнать о рекомендуемых путях обновления для Skype для бизнеса Server 2015, как работает обновление на месте, какие поддерживаемые сценарии сосуществования и как процесс обновления выглядит следующим образом.

> [!NOTE]
> Обновление на месте было доступно в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Параллельное сосуществование поддерживается, ознакомьтесь со статьей [Миграция в Skype для бизнеса Server 2019](../../SfBServer2019/migration/migration-to-skype-for-business-server-2019.md) для получения дополнительных сведений.
  
## <a name="recommended-upgrade-paths-to-skype-for-business-server-2015"></a>Рекомендуемые варианты обновления для Skype для бизнеса Server 2015

 Чтобы выполнить обновление с Lync Server 2013, Lync Server 2010 или Office Communications Server 2007 R2 до Skype для бизнеса Server 2015, используйте следующие варианты обновления:
  
> [!CAUTION]
> При обновлении на месте автоматически перемещаются каталоги конференций из Lync Server 2013 в Skype для бизнеса Server 2015. Тем не менее, если вы планируете переместить каталоги конференций вручную, очень важно использовать командную консоль Skype для бизнеса Server 2015. Если вы попытаетесь использовать командную консоль Lync Server 2013 для перемещения каталогов конференций из Lync Server 2013 в Skype для бизнеса Server 2015, может произойти потеря данных. Как правило, при работе с Skype для бизнеса Server 2015 в любой емкости необходимо использовать набор средств Skype для бизнеса Server 2015.  
  
|**Версия**|**рекомендации**|
|:-----|:-----|
|Lync Server 2013  <br/> | Для обновления используйте построитель топологий Skype для бизнеса Server и новую функцию обновления на месте на каждом сервере, связанном с пулом. Ознакомьтесь со статьей [Планирование обновления с Lync server 2013 до Skype для бизнеса server 2015](upgrade.md#BKMK_PlanUpgradeFromLync2013) и [обновление до Skype для бизнеса Server 2015](../deploy/upgrade-to-skype-for-business-server.md) для получения подробных инструкций. <br/> |
|Lync Server 2010 + Lync Server 2013 (двойной режим)  <br/> |Сначала выполните обновление до Lync Server 2013, а затем выполните обновление до Skype для бизнеса Server 2015, используя новую функцию обновления на месте. Тем не менее, если топология является первичным сервером Lync Server 2010, можно также выполнить откат компонентов Lync Server 2013 до Lync Server 2010, а затем выполнить обновление непосредственно до Skype для бизнеса Server 2015. В этом случае вам не удастся воспользоваться обновлением на месте и использовать прямую совместную работу между Lync Server 2010 и Skype для бизнеса Server 2015. Тройной доступ не поддерживается, но поддерживается совместная работа.  <br/> |
|Lync Server 2010  <br/> |Выведите новый пул Skype для бизнеса Server 2015 и выполните миграцию пользователей в этот новый пул. Затем можно списать старый пул Lync Server 2010. Обновление с Lync Server 2010 to Skype для бизнеса Server 2015 аналогично обновлению с Lync Server 2010 до Lync Server 2013. Ознакомьтесь [с Разпереходом с Lync server 2010 на Lync server 2013](https://go.microsoft.com/fwlink/p/?LinkId=526615).  <br/> |
|Office Communications Server 2007 R2  <br/> | Выберите один из двух вариантов: <br/>  Настройте новую среду Skype для бизнеса Server 2015. <br/>  Или, если оборудование и программное обеспечение соответствует требованиям для Skype для бизнеса Server 2015, выполните обновление до Lync Server 2013, а затем выполните обновление до Skype для бизнеса Server 2015, используя новую функцию обновления на месте. Дополнительные сведения см. в статье [требования к серверу для Skype для бизнеса server 2015](requirements-for-your-environment/server-requirements.md) и [Миграция с Office Communications Server 2007 R2 на Lync Server 2013](https://go.microsoft.com/fwlink/p/?LinkId=526616).  <br/> |
   
> [!NOTE]
> SQL Server 2014 поддерживается в Skype для бизнеса Server 2015, но не поддерживается в Lync Server 2013. Если вы хотите обновить SQL Server 2012 до SQL Server 2014, необходимо сначала обновить пул до Skype для бизнеса Server 2015 с помощью метода обновления на месте, как описано в этом документе. После этого вы можете выполнить обновление с SQL Server 2012 до SQL Server 2014 в статье [Upgrade to SQL server 2014](https://msdn.microsoft.com/library/bb677622%28v=sql.120%29.aspx). Дополнительные сведения о требованиях к базе данных можно найти в статье [требования к серверу для Skype для бизнеса server 2015](requirements-for-your-environment/server-requirements.md). 
  
## <a name="plan-to-upgrade-from-lync-server-2013-to-skype-for-business-server-2015"></a>Планирование обновления с Lync Server 2013 до Skype для бизнеса Server 2015
<a name="BKMK_PlanUpgradeFromLync2013"> </a>

Вы можете обновить системы Lync Server 2013 до Skype для бизнеса Server 2015 с помощью новой функции обновления на месте. Обновление на месте предоставляет решение одним щелчком, которое создает резервные копии сертификатов, удаляет компоненты сервера, обновляет локальные базы данных и устанавливает роли Skype для бизнеса Server 2015. Обновление на месте позволяет сохранить существующие инвестиции в оборудование и серверы, уменьшая общую стоимость развертывания Skype для бизнеса Server 2015.
  
> [!NOTE]
> Обновление на месте позволяет использовать одно и то же оборудование при обновлении до Skype для бизнеса Server. Однако повторное использование одного и того же оборудования не приводит к одинаковой емкости производительности. Не следует ожидать, что нагрузка на производительность для Lync Server 2013 и Skype для бизнеса Server 2015 идентична. 
  
> [!NOTE]
> Обновление на месте не поддерживает высокую доступность или аварийное восстановление в Skype для бизнеса Server. 
  
Обновление на месте включает в себя перевод пула Lync Server 2013 в автономный режим и его обновление до пула Skype для бизнеса Server 2015. 
  
### <a name="create-an-in-place-upgrade-plan"></a>Создание плана обновления на месте

Создайте план, включающий:
  
1. Сведения о текущей топологии.
    
    > [!NOTE]
    > Обязательно удалите средство администрирования LRS для Lync Server 2013 перед выполнением обновления на месте. Средство администрирования LRS для Lync Server 2013 не может совместно использоваться с Skype для бизнеса Server 2015. После выполнения обновления на месте установите новое средство администрирования LRS. Для получения дополнительных сведений посетите [веб-портал администрирования системы комнат Microsoft Lync для Skype для бизнеса Server 2015](https://go.microsoft.com/fwlink/?LinkID=544807) .
  
2. Основной пул для обновления.
    
3. Следует ли обновить базы данных архивации и мониторинга или создать новые.
    
4. Метод обновления на месте, который вы будете использовать: offline или Move Users. В процессе перемещения пользователей также потребуется перенести глобальные каталоги конференций, связанные с основным пулом. 
    
5. План взаимодействия для затронутых пользователей.
    
6. План резервного копирования в случае сбоя обновления.
    
Все пользователи, которые находятся в основном пуле во время обновления, не смогут использовать эти службы до завершения обновления. Если у вас есть дополнительный пул, можно не повлиять на работу пользователей, переместив их в дополнительный пул перед обновлением. После обновления переместите пользователей обратно в основной пул.
  
### <a name="in-place-upgrade-methods"></a>Методы обновления на месте

Существует два сценария обновления на месте. 
  
- Метод Move User, который не требует простоя для пользователей. 
    
- Метод Offline, который требует простоя.
    
Рекомендуется запланировать обновление автономного метода во время периода обслуживания, и пользователи получают уведомление о простоях.
  
> [!NOTE]
> При обновлении связанного пула в Lync Server 2013 и необходимости обновления обоих пулов до Skype для бизнеса Server 2015. Обязательно обновите второй пул сразу же после обновления первого пула. Если один пул работает под управлением Lync Server 2013, а второй пул работает под управлением Skype для бизнеса Server 2015, параметры аварийного восстановления минимизированы. Например, если один пул работает под управлением 2013, а второй — 2015, а затем это может привести к потере данных из-за того, что отказоустойчивый пул не поддерживается в аварийном режиме, если связанные пулы не имеют одинаковую версию. 
  
#### <a name="in-place-upgrade-offline-method"></a>Автономный метод обновления на месте

Используйте этот метод, если вы не хотите перемещать пользователей между пулами пользователей. Во время обновления пользователи не смогут использовать службы Lync или Skype для бизнеса.

Поэтому необходимо переместить каталоги конференций, если вы все еще хотите, чтобы Конференции PSTN, запланированные в пуле, были доступны во время обновления.

### <a name="in-place-upgrade-process"></a>Процесс обновления на месте

 Выполните обновление с Lync Server 2013 на Skype для бизнеса Server 2015, выполнив следующие действия:
  
1. Перед обновлением создайте резервные копии всех баз данных.
    
2. Убедитесь, что все службы, которые требуется обновить, находятся в состоянии "работает".
    
3. Обновите и опубликуйте файл топологии с помощью построителя топологий.
    
4. Остановите все службы на всех серверах переднего плана.
    
5. Установите новые необходимые компоненты, необходимые для Skype для бизнеса Server.
    
6. На каждом сервере переднего плана запустите обновление на месте.
    
7. После завершения обновления перезапустите все службы.
    
   - Для пула переднего плана перезапустите службы с помощью команды Start — CsPool.
    
   - Для серверов, не являющихся серверами переднего плана, используйте Start – CSWindowsService.
    
> [!NOTE]
>  Если вы не хотите обновлять существующие базы данных архивации и мониторинга, удалите зависимость перед обновлением топологии. Если вы хотите создать новые базы данных архивации и мониторинга, то во время обновления можно создать новое хранилище SQL и связать его с пулом. Действия, описанные в этой статье, можно найти в статье[обновление до Skype для бизнеса Server 2015](../deploy/upgrade-to-skype-for-business-server.md). > обновление на месте не поддерживает высокую доступность или аварийное восстановление в Skype для бизнеса Server. Чтобы избежать прерывания обслуживания пользователей, используйте [метод Move Users (без простоя пользователей)](upgrade.md#bkmk_MoveUsersMethod) для обновления. > в процессе обновления XDS-реплика помещается в локальную общую папку на жестком диске с наибольшим объемом свободного места. Если этот диск позже удаляется, вы можете выполнять такие проблемы, как службы, которые не запускаются.
  
### <a name="upgrade-order"></a>Порядок обновления

Обновление топологии с внутренней до наружной. Сначала обновите все пулы, затем пограничные серверы и, наконец, пул центрального хранилища управления (CMS). 
  
### <a name="kerberos-authentication-considerations"></a>Рекомендации по проверке подлинности Kerberos

Если вы используете проверку подлинности Kerberos для веб-служб, необходимо переназначить учетные записи Kerberos и сбросить пароль после завершения обновления на месте. Чтобы узнать, как это сделать, ознакомьтесь со статьей [Настройка проверки подлинности Kerberos](https://go.microsoft.com/fwlink/p/?LinkId=530342).
  
## <a name="support-for-coexistence-with-lync-server-2013-and-lync-server-2010"></a>Поддержка совместной работы с Lync Server 2013 и Lync Server 2010
<a name="BKMK_PlanUpgradeFromLync2013"> </a>

Skype для бизнеса Server 2015 можно запускать в той же топологии, что и Lync Server 2013 или Lync Server 2010, но не все три в одной и той же топологии.
  
Если у вас есть совместная работа Lync Server 2010 и Lync Server 2013, рекомендуется обновить всю топологию до Lync Server 2013, а затем выполнить обновление до Skype для бизнеса Server 2015 с использованием обновления на месте. Для получения дополнительных сведений обратитесь [к разделу миграция с Lync server 2010 на Lync server 2013](https://go.microsoft.com/fwlink/p/?LinkId=526615).
  
Если ваша топология в основном является Lync Server 2010, выполните откат компонентов Lync Server 2013 до Lync Server 2010, прежде чем приступать к обновлению топологии до Skype для бизнеса Server 2015. В этом случае вы теряете преимущества обновления на месте и имеет топологию совместного существования между Lync Server 2010 и Skype для бизнеса Server 2015.
  
На следующей схеме показана поддержка совместной работы Skype для бизнеса Server 2015 с Lync Server 2013 и Lync Server 2010.
  
![Схема, на которой показана поддержка сосуществования Skype для бизнеса Server 2015 с Lync Server 2013 или Lync Server 2010.](../media/b5eb4f8c-577a-4a45-9a66-9531ce2c5dc1.png)
  
## <a name="upgrade-process-with-existing-survivable-branch-appliance-and-server"></a>Процесс обновления с использованием существующего устройства и сервера для обеспечения связи в филиалах
<a name="BKMK_PlanUpgradeFromLync2013"> </a>

Skype для бизнеса Server 2015 не поддерживает обновление на месте для устройства для обеспечения связи в филиалах (SBA) или сервера для обеспечения связи в филиалах (SBS).
  
Тем не менее мы поддерживаем сосуществование Skype для бизнеса Server datacenters с Lync Server 2010 или Lync Server 2013 SBA/SBS. 
  
При планировании обновления на месте для пула переднего плана Lync Server 2013 (FE) с помощью связанной ветви вы можете оставить существующих пользователей на сервере Lync Server 2013 SBA/SBS. Во время обновления пользователи SBA/SBS будут переведены в режим устойчивости и вернутся к нормальным функциям после завершения обновления. Дополнительные сведения о работе пользователей в режиме устойчивости можно найти в статье поддержка [устойчивости сайтов филиалов в Lync Server 2013](https://technet.microsoft.com/library/gg398715.aspx).
  
При переносе топологии Lync Server 2010 в Skype для бизнеса Server 2015 SBA/SBS необходимо повторно добавить в топологию, аналогично миграции на Lync Server 2013. Для выполнения необходимых действий необходимо прочитать [Подключение устройства для обеспечения связи в филиалах к интерфейсному пулу Lync Server 2013](https://technet.microsoft.com/library/jj688026.aspx).
  
Для топологий совместного использования Lync Server 2010 и Lync Server 2013 сначала выровняйте рекомендации, приведенные в разделе "Поддержка сосуществования с Lync Server 2013 и Lync Server 2010".
