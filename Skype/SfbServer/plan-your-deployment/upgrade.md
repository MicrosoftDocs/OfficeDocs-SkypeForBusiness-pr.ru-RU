## <a name="recommended-upgrade-paths-to-skype-for-business-server-2015"></a>Рекомендуемые пути обновления до Skype для бизнеса Server 2015

 Чтобы обновить Lync Server 2013, Lync Server 2010 или Office Communications Server 2007 R2 до Skype для бизнеса Server 2015, используйте следующие пути обновления:
  
> [!CAUTION]
> In-Place автоматически перемещает каталоги конференций из Lync Server 2013 в Skype для бизнеса Server 2015. Однако если планируется переместить каталоги конференций вручную, очень важно использовать оболочку управления Skype для бизнеса Server 2015. Если попытаться использовать Lync Server 2013 Management Shell для перемещения каталогов конференций из Lync Server 2013 в Skype для бизнеса Server 2015, может произойти потеря данных. Как правило, при работе со Skype для бизнеса Server 2015 в любой емкости следует использовать набор средств Skype для бизнеса Server 2015.  
  
|**Версия**|**рекомендации**|
|:-----|:-----|
|Lync Server 2013  <br/> | To upgrade, use the Skype for Business Server Topology Builder and the new In-Place Upgrade feature on each of the servers associated with the pool. Подробные инструкции см. в описании планов обновления [Lync Server 2013](upgrade.md#BKMK_PlanUpgradeFromLync2013) до Skype для бизнеса Server 2015 и обновления до Skype для бизнеса [Server 2015.](../deploy/upgrade-to-skype-for-business-server.md) <br/> |
|Lync Server 2010 + Lync Server 2013 (двойной режим)  <br/> |Сначала обновим Lync Server 2013, а затем до Skype для бизнеса Server 2015 с помощью новой функции In-Place обновления. Однако если топология является основной Lync Server 2010, можно также откатить компоненты Lync Server 2013 до Lync Server 2010, а затем обновить их непосредственно до Skype для бизнеса Server 2015. В этом случае вы не сможете воспользоваться преимуществами In-Place обновления и использовать прямое сосуществование между Lync Server 2010 и Skype для бизнеса Server 2015. Tri-existence is not supported but co-existence is supported.  <br/> |
|Lync Server 2010  <br/> |Вывести новый пул Skype для бизнеса Server 2015, а затем перенести пользователей в этот новый пул. Затем можно списание старого пула Lync Server 2010. Обновление Lync Server 2010 до Skype для бизнеса Server 2015 аналогично обновлению с Lync Server 2010 до Lync Server 2013. См. [переход с Lync Server 2010 на Lync Server 2013.](https://go.microsoft.com/fwlink/p/?LinkId=526615)  <br/> |
|Office Communications Server 2007 R2  <br/> | Выберите один из двух вариантов: <br/>  Настройка новой среды Skype для бизнеса Server 2015. <br/>  Или если оборудование и программное обеспечение соответствуют требованиям Skype для бизнеса Server 2015, обновите до Lync Server 2013, а затем обновите skype для бизнеса Server 2015 с помощью новой функции In-Place Upgrade. Дополнительные сведения см. в требованиях к серверу для Skype для бизнеса [Server 2015](requirements-for-your-environment/server-requirements.md) и миграции с [Office Communications Server 2007 R2 на Lync Server 2013.](https://go.microsoft.com/fwlink/p/?LinkId=526616)  <br/> |
   
> [!NOTE]
> SQL Server 2014 поддерживается в Skype для бизнеса Server 2015, но не поддерживается в Lync Server 2013. Если требуется обновить SQL Server 2012 до SQL Server 2014, сначала необходимо обновить пул до Skype для бизнеса Server 2015 с помощью метода In-Place Upgrade, как описано в этом документе. Затем можно обновить SQL Server 2012 до SQL Server 2014, см. обновление до [SQL Server 2014.](https://msdn.microsoft.com/library/bb677622%28v=sql.120%29.aspx) Дополнительные информацию о требованиях к базам данных см. в этой [теме.](requirements-for-your-environment/server-requirements.md) 
  
## <a name="plan-to-upgrade-from-lync-server-2013-to-skype-for-business-server-2015"></a>Планирование обновления Lync Server 2013 до Skype для бизнеса Server 2015
<a name="BKMK_PlanUpgradeFromLync2013"> </a>

Вы можете обновить системы Lync Server 2013 до Skype для бизнеса Server 2015 с помощью новой In-Place обновления. Обновление на месте предоставляет решение одним щелчком мыши, которое создает для сертификатов поддержку, устрашает серверные компоненты, обновляет локальные базы данных и устанавливает роли Skype для бизнеса Server 2015. При обновлении на месте необходимо сохранить существующие инвестиции в оборудование и сервер, сокращая общие затраты на развертывание Skype для бизнеса Server 2015.
  
> [!NOTE]
> In-Place обновления позволяет использовать то же оборудование при обновлении до Skype для бизнеса Server. Однако повторное оборудование не преобразуется в ту же производительность. Не следует ожидать, что нагрузки на производительность Lync Server 2013 и Skype для бизнеса Server 2015 будут идентичными. 
  
> [!NOTE]
> In-Place не поддерживает высокую доступность или аварийное восстановление skype для бизнеса Server. 
  
Обновление на месте подразумевает отключение пула Lync Server 2013 и его обновление до пула Skype для бизнеса Server 2015. 
  
### <a name="create-an-in-place-upgrade-plan"></a>Создание плана In-Place обновления

Сделайте план, который включает:
  
1. Понимание текущей топологии.
    
    > [!NOTE]
    > Не забудьте удалить средство администрирования LRS для Lync Server 2013 перед запуском In-Place обновления. Средство администрирования LRS для Lync Server 2013 не может сосуществовать со Skype для бизнеса Server 2015. После запуска In-Place установите новое средство администрирования LRS. Дополнительные сведения см. в веб-портале администрирования системы комнат Microsoft Lync для Skype для бизнеса [Server 2015.](https://go.microsoft.com/fwlink/?LinkID=544807)
  
2. Основной пул для обновления.
    
3. Будут ли обновлены базы данных архива и мониторинга или создаются новые.
    
4. Метод In-Place обновления: автономный режим или перемещение пользователей. В рамках перемещения пользователей вам также потребуется перенести глобальные каталоги конференций, связанные с основным пулом. 
    
5. План взаимодействия для пользователей, на которые они влияют.
    
6. План резервного копирования на случай сбой обновления.
    
Все пользователи, которые находятся в основном пуле во время обновления, не смогут использовать службы до завершения обновления. При работе дополнительного пула можно избежать влияния на пользователей, перемещая их во дополнительный пул перед обновлением. После обновления переместим пользователей обратно в основной пул.
  
### <a name="in-place-upgrade-methods"></a>Методы обновления на месте

Существует два сценария обновления In-Place обновления: 
  
- Метод Move User, который не требует простоя для пользователей. 
    
- Автономный метод, который требует простоя.
    
Рекомендуется, чтобы обновление автономного метода было запланировано в период обслуживания, и пользователи были уведомлены о простоях.
  
> [!NOTE]
> При обновлении парного пула в Lync Server 2013 необходимо обновить оба пула до Skype для бизнеса Server 2015. Обязательно обновим второй пул сразу после обновления первого пула. When one pool is running Lync Server 2013 and the second pool is running Skype for Business Server 2015 then disaster recovery options are minimized. Например, если один пул работает под управлением 2013, а второй — 2015 и имеется авария, то может произойть потеря данных, так как отбой пула не поддерживается в аварийном режиме, когда сопряженные пулы имеют разные версии. 
  
#### <a name="in-place-upgrade-offline-method"></a>Автономный метод обновления на месте

Используйте этот метод, если вы не хотите перемещать пользователей между пулами пользователей. Во время обновления пользователи не смогут использовать службы Lync или Skype для бизнеса.

После начала обновления пула серверов необходимо завершить обновление всего пула. Skype для бизнеса Server не поддерживает обновление только части пула. 
  
#### <a name="move-users-method-no-user-downtime"></a>Метод Move Users (без простоя пользователей)
<a name="bkmk_MoveUsersMethod"> </a>

Чтобы использовать этот метод, необходимо переместить пользователей в другой пул перед началом обновления. Во время обновления пользователи могут использовать службы Lync. После того как они будут перемещены в обновленный пул, они смогут использовать Skype для бизнеса. На следующей схеме представлен обзор этого процесса.
  
> [!IMPORTANT]
> В рамках перемещения пользователей вам также потребуется перенести глобальные каталоги конференций, связанные с основным пулом. Конференц-конференция с конференц-залом с конференц-залом с комконференции по-прежнему разрешит ConferenceID в обновляемом пуле, а не в сопряженном пуле. Поэтому необходимо переместить каталоги конференций, если требуется, чтобы конференции STN, запланированные в пуле, были доступны во время обновления.

### <a name="in-place-upgrade-process"></a>Процесс обновления на месте

 Обновив Lync Server 2013 до Skype для бизнеса Server 2015, с помощью следующих действий:
  
1. Перед обновлением сполна сполна все базы данных.
    
2. Убедитесь, что все обновляемые службы находятся в состоянии работы.
    
3. Обновите и опубликуем файл топологии с помощью построщика топологий.
    
4. Остановите все службы на всех серверах переднего сервера.
    
5. Установите новые необходимые условия для Skype для бизнеса Server.
    
6. На каждом сервере переднего In-Place обновления.
    
7. После завершения обновления перезапустите все службы.
    
   - Для пула переднего входа перезапустите службы с помощью команды Start-CsPool.
    
   - Для серверов без переднего сервера используйте start-CSWindowsService.
    
> [!NOTE]
>  Если вы не хотите обновлять существующие базы данных архивации и мониторинга, удалите зависимость перед обновлением топологии. Если вы хотите создать новые базы данных архива и мониторинга, во время обновления можно создать новое хранилище SQL и связать его с пулом. Действия, необходимые для этого, можно найти в разделе["Обновление до Skype для бизнеса Server 2015".](../deploy/upgrade-to-skype-for-business-server.md) > обновления на месте не поддерживает высокую доступность или аварийное восстановление для Skype для бизнеса Server. Чтобы избежать прерывания работы служб пользователей, используйте метод [Move Users (без](upgrade.md#bkmk_MoveUsersMethod) простоя пользователей) для обновления.> В процессе обновления реплика XDS помещается в локализованную общую папку на диске с самым большим количеством свободного места. Если позднее этот диск будет удален, могут быть проблемы, например, не запускаются службы.
  
### <a name="upgrade-order"></a>Порядок обновления

Обновите топологию изнутри наружу. Сначала обновите все пулы, а затем на edge-серверы и, наконец, пул центрального банка управления (CMS). 
  
### <a name="kerberos-authentication-considerations"></a>Вопросы проверки подлинности Kerberos

При использовании проверки подлинности Kerberos для веб-служб необходимо перенаправить учетные записи Kerberos и сбросить пароль после In-Place обновления. Чтобы узнать, как это сделать, см. настройку проверки [подлинности Kerberos.](https://go.microsoft.com/fwlink/p/?LinkId=530342)
  
## <a name="support-for-coexistence-with-lync-server-2013-and-lync-server-2010"></a>Поддержка сосуществования с Lync Server 2013 и Lync Server 2010
<a name="BKMK_PlanUpgradeFromLync2013"> </a>

Skype для бизнеса Server 2015 можно запустить в той же топологии, что и Lync Server 2013 или Lync Server 2010, но не все три из них могут быть в одной топологии.
  
При совместном использовании Lync Server 2010 и Lync Server 2013 рекомендуется обновить всю топологию до Lync Server 2013, а затем обновить ее до Skype для бизнеса Server 2015 с помощью обновления In-Place. Дополнительные сведения см. в сведениях о миграции [с Lync Server 2010 на Lync Server 2013.](https://go.microsoft.com/fwlink/p/?LinkId=526615)
  
Если топология в основном Lync Server 2010, откатите компоненты Lync Server 2013 до Lync Server 2010 перед обновлением топологии до Skype для бизнеса Server 2015. В этом случае вы теряете преимущества обновления In-Place и имеете топологию сосуществования между Lync Server 2010 и Skype для бизнеса Server 2015.
  
На следующей схеме показана поддержка сосуществования Skype для бизнеса Server 2015 с Lync Server 2013 и Lync Server 2010.
  
![Схема, на которой показана поддержка сосуществования Skype для бизнеса Server 2015 с Lync Server 2013 или Lync Server 2010.](../media/b5eb4f8c-577a-4a45-9a66-9531ce2c5dc1.png)
  
## <a name="upgrade-process-with-existing-survivable-branch-appliance-and-server"></a>Процесс обновления с помощью существующего устройства и сервера для обеспечения связи в филиалах
<a name="BKMK_PlanUpgradeFromLync2013"> </a>

Skype для бизнеса Server 2015 не поддерживает обновление In-Place устройства для обеспечения связи в филиалах (SBA) или сервера обеспечения связи в филиалах (SBS).
  
Однако мы поддерживаем сосуществование центра обработки данных Skype для бизнеса Server с Lync Server 2010 или Lync Server 2013 SBA/SBS. 
  
При планировании обновления In-Place пула переднего плана Lync Server 2013 с связанной ветвью можно оставить существующих пользователей в SBA/SBS Lync Server 2013. Во время обновления пользователи SBA/SBS будут переходить в режим устойчивости и после завершения обновления будут работать в обычном режиме. Дополнительные сведения о режиме устойчивости пользователей см. в [lync Server 2013.](https://technet.microsoft.com/library/gg398715.aspx)
  
When migrating a Lync Server 2010 topology to Skype for Business Server 2015, the SBA/SBS must re-added to the topology, similar to the migration to Lync Server 2013. Необходимые действия можно найти в подключении устройства для связи в филиалах к пулу переднего сервера [Lync Server 2013.](https://technet.microsoft.com/library/jj688026.aspx)
  
Для совместной работы Lync Server 2010 и Lync Server 2013 сначала согласуются с рекомендациями, которые приведены в разделе "Поддержка сосуществования с Lync Server 2013 и Lync Server 2010".
