# <a name="plan-to-upgrade-to-skype-for-business-server-2015"></a>Планирование обновления до Skype для бизнеса Server 2015
 
Сводка. Сведения о том, что следует учитывать при планировании обновления до Skype для бизнеса Server 2015.
  
В рамках плана обновления до Skype для бизнеса Server 2015 используйте этот раздел, чтобы понять рекомендуемые пути обновления до Skype для бизнеса Server 2015, принцип работы In-Place Upgrade, поддерживаемые сценарии сосуществования и как выглядит процесс обновления.

> [!NOTE]
> Обновления на месте были доступны в Skype для бизнеса Server 2015, но больше не поддерживаются Skype для бизнеса Server 2019. Параллельное сосуществование поддерживается. Дополнительные сведения см. в Skype для бизнеса Server [2019](../../SfBServer2019/migration/migration-to-skype-for-business-server-2019.md).
  
## <a name="recommended-upgrade-paths-to-skype-for-business-server-2015"></a>Рекомендуемые пути обновления до Skype для бизнеса Server 2015

 Чтобы выполнить обновление с Lync Server 2013, Lync Server 2010 или Office Communications Server 2007 R2 до Skype для бизнеса Server 2015, используйте следующие пути обновления:
  
> [!CAUTION]
> In-Place обновление автоматически перемещает каталоги конференций с Lync Server 2013 на Skype для бизнеса Server 2015. Однако если вы планируете вручную переместить каталоги конференций, очень важно использовать командную консоль Skype для бизнеса Server 2015. При попытке использовать командную консоль Lync Server 2013 для перемещения каталогов конференций с Lync Server 2013 на Skype для бизнеса Server 2015 может произойти потеря данных. Как правило, при работе с Skype для бизнеса Server 2015 в любой емкости следует использовать набор средств Skype для бизнеса Server 2015.  
  
|**Version**|**Рекомендации**|
|:-----|:-----|
|Lync Server 2013  <br/> | Для обновления используйте Skype для бизнеса Server построитель топологий и новую In-Place обновления на каждом сервере, связанном с пулом. Подробные инструкции см. в статье "Планирование обновления [с Lync Server 2013 до Skype для бизнеса Server 2015](upgrade.md#BKMK_PlanUpgradeFromLync2013) и обновление до [Skype для бизнеса Server 2015](../deploy/upgrade-to-skype-for-business-server.md)". <br/> |
|Lync Server 2010 + Lync Server 2013 (двойной режим)  <br/> |Сначала обновите Lync Server 2013, а затем Skype для бизнеса Server 2015 с помощью новой функции In-Place обновления. Однако если топология является основной версией Lync Server 2010, вы также можете выполнить откат компонентов Lync Server 2013 до Lync Server 2010, а затем выполнить обновление непосредственно до Skype для бизнеса Server 2015. В этом случае вы не сможете воспользоваться преимуществами In-Place Upgrade и будет использовать прямое сосуществование между Lync Server 2010 и Skype для бизнеса Server 2015. Три существования не поддерживается, но поддерживается сосуществование.  <br/> |
|Lync Server 2010  <br/> |Создайте новый пул Skype для бизнеса Server 2015, а затем перенесите пользователей в этот новый пул. Затем можно вывести старый пул Lync Server 2010 из эксплуатации. Обновление Lync Server 2010 до Skype для бизнеса Server 2015 аналогично обновлению с Lync Server 2010 до Lync Server 2013. См [. сведения о миграции с Lync Server 2010 на Lync Server 2013](/previous-versions/office/lync-server-2013/migration-from-lync-server-2010-to-lync-server-2013).  <br/> |
|Office Communications Server 2007 R2  <br/> | Выберите один из двух вариантов: <br/>  Настройка новой среды Skype для бизнеса Server 2015. <br/>  Или если оборудование и программное обеспечение соответствуют требованиям Skype для бизнеса Server 2015, обновите Lync Server 2013, а затем выполните обновление до Skype для бизнеса Server 2015 с помощью новой функции In-Place Upgrade. Дополнительные сведения см. в требованиях к серверу [для Skype для бизнеса Server 2015](requirements-for-your-environment/server-requirements.md) и миграции с [Office Communications Server 2007 R2 на Lync Server 2013](/previous-versions/office/lync-server-2013/migration-from-office-communications-server-2007-r2-to-lync-server-2013).  <br/> |
   
> [!NOTE]
> SQL Server 2014 поддерживается в Skype для бизнеса Server 2015, но не поддерживается в Lync Server 2013. Если вы хотите обновить пул с SQL Server 2012 до SQL Server 2014, сначала необходимо обновить пул до Skype для бизнеса Server 2015 с помощью метода In-Place Upgrade, как описано в этом документе. Затем вы можете выполнить обновление с SQL Server 2012 до SQL Server 2014, см. обновление до [SQL Server 2014](/sql/database-engine/install-windows/upgrade-sql-server?viewFallbackFrom=sql-server-2014). Дополнительные сведения о требованиях к базе данных см. в разделе "Требования к Skype для бизнеса Server [2015"](requirements-for-your-environment/server-requirements.md). 
  
## <a name="plan-to-upgrade-from-lync-server-2013-to-skype-for-business-server-2015"></a>Планирование обновления с Lync Server 2013 до Skype для бизнеса Server 2015
<a name="BKMK_PlanUpgradeFromLync2013"> </a>

Вы можете обновить системы Lync Server 2013 до Skype для бизнеса Server 2015 с помощью новой In-Place обновления. Обновление на месте предоставляет решение одним щелчком мыши, которое выполняет резервное копирование сертификатов, удаляет компоненты сервера, обновляет локальные базы данных и устанавливает роли Skype для бизнеса Server 2015. Обновление на месте позволяет сохранить существующие инвестиции в оборудование и сервер, уменьшая общие затраты на развертывание Skype для бизнеса Server 2015 г.
  
> [!NOTE]
> In-Place обновления позволяет использовать то же оборудование при обновлении до Skype для бизнеса Server. Однако повторное использование того же оборудования не преобразуется в ту же производительность. Не следует ожидать, что загрузка производительности для Lync Server 2013 и Skype для бизнеса Server 2015 будет идентичной. 
  
> [!NOTE]
> In-Place обновления не поддерживает высокий уровень доступности или аварийное восстановление для Skype для бизнеса Server. 
  
Обновление на месте включает перевод пула Lync Server 2013 в автономный режим и его обновление до Skype для бизнеса Server 2015. 
  
### <a name="create-an-in-place-upgrade-plan"></a>Создание плана In-Place обновления

Создание плана, который включает в себя:
  
1. Понимание текущей топологии.
    
    > [!NOTE]
    > Не забудьте удалить средство LRS Администратор для Lync Server 2013 перед запуском In-Place обновления. Средство LRS Администратор для Lync Server 2013 не может сосуществовать с Skype для бизнеса Server 2015. После запуска In-Place обновления установите новое средство LRS Администратор. Дополнительные сведения см. на веб-портале администрирования системы комнат Microsoft [Lync Skype для бизнеса Server 2015](https://go.microsoft.com/fwlink/?LinkID=544807).
  
2. Основной пул для обновления.
    
3. Обновите ли вы базы данных архивации и мониторинга или создадите новые.
    
4. Метод In-Place обновления, который вы будете использовать: "Автономный" или "Переместить пользователей". В рамках перемещения пользователей также потребуется перенести глобальные каталоги конференций, связанные с основным пулом. 
    
5. План взаимодействия для затронутых пользователей.
    
6. План резервного копирования на случай сбоя обновлений.
    
Все пользователи, которые находятся в основном пуле во время обновления, не смогут использовать службы до завершения обновления. Если у вас есть работающий вторичный пул, вы можете избежать влияния на пользователей, переместив их во вторичный пул перед обновлением. После обновления переместите пользователей обратно в основной пул.
  
### <a name="in-place-upgrade-methods"></a>Методы обновления на месте

Существует два сценария для In-Place обновления: 
  
- Метод Move User, который не требует простоя для пользователей. 
    
- Автономный метод, требуемый для простоя.
    
Мы рекомендуем запланировать обновление автономного метода во время периода обслуживания, и пользователи будут получать уведомления о простое.
  
> [!NOTE]
> При обновлении парного пула в Lync Server 2013 необходимо обновить оба пула до Skype для бизнеса Server 2015. Обязательно обновите второй пул сразу после обновления первого пула. Если один пул работает под управлением Lync Server 2013, а второй — Skype для бизнеса Server 2015, параметры аварийного восстановления свернуты. Например, если один пул работает под управлением 2013, а второй — 2015 и возникла авария, то может произойти потеря данных, так как отработка отказа пула не поддерживается в аварийном режиме, если парные пулы имеют разную версию. 
  
#### <a name="in-place-upgrade-offline-method"></a>Автономный метод обновления на месте

Используйте этот метод, если вы не хотите перемещать пользователей между пулами пользователей. Во время обновления пользователи не смогут использовать Lync или Skype для бизнеса службы.

После обновления пула серверов необходимо завершить обновление всего пула. Skype для бизнеса Server не поддерживает обновление только части пула. 
  
#### <a name="move-users-method-no-user-downtime"></a>Метод Move Users (без простоя пользователя)
<a name="bkmk_MoveUsersMethod"> </a>

Чтобы использовать этот метод, необходимо переместить пользователей в другой пул перед началом обновления. Во время обновления пользователи могут использовать службы Lync. После перемещения в обновленный пул они могут использовать Skype для бизнеса. На следующей схеме показан обзор этого процесса.
  
> [!IMPORTANT]
> В рамках перемещения пользователей также потребуется перенести глобальные каталоги конференций, связанные с основным пулом. Конференц-связь с телефонным подключением по ТСОП по-прежнему разрешает ConferenceID в обновляемом пуле, а не в парном пуле. Поэтому необходимо переместить каталоги конференций, если вы по-прежнему хотите, чтобы конференции ТСОП, запланированные в пуле, были доступны во время обновления.

### <a name="in-place-upgrade-process"></a>Процесс обновления на месте

 Обновите Lync Server 2013 до Skype для бизнеса Server 2015, выполнив следующие действия:
  
1. Создайте резервную копию всех баз данных перед обновлением.
    
2. Убедитесь, что все обновляемые службы находятся в состоянии выполнения.
    
3. Обновите и опубликуйте файл топологии с помощью построителя топологий.
    
4. Остановите все службы на всех серверах переднего плана.
    
5. Установите новые необходимые компоненты, необходимые для Skype для бизнеса Server.
    
6. На каждом сервере переднего плана запустите In-Place обновления.
    
7. После завершения обновления перезапустите все службы.
    
   - Для пула переднего плана перезапустите службы с помощью команды Start-CsPool.
    
   - Для серверов, отличных от серверов переднего плана, используйте Start-CSWindowsService.
    
> [!NOTE]
>  Если вы не хотите обновлять существующие базы данных архивации и мониторинга, удалите зависимость перед обновлением топологии. Если вы хотите создать базы данных архивации и мониторинга, во время обновления можно создать новое хранилище SQL и связать его с пулом. Инструкции по этому действию см. в разделе "Обновление до [Skype для бизнеса Server 2015"](../deploy/upgrade-to-skype-for-business-server.md). > на месте не поддерживает высокий уровень доступности или аварийное восстановление для Skype для бизнеса Server. Чтобы избежать прерывания работы служб пользователей, используйте метод [Move Users (](upgrade.md#bkmk_MoveUsersMethod) без простоя пользователя) для обновления.> В процессе обновления реплика XDS помещается в локальную общую папку на диске с самым бесплатным пространством. Если позже этот диск будет удален, могут быть проблемы, например, если службы не запускаются.
  
### <a name="upgrade-order"></a>Порядок обновления

Обновите топологию изнутри на внешний. Сначала обновите все пулы, затем пограничные серверы и, наконец, пул Центрального хранилища управления (CMS). 
  
### <a name="kerberos-authentication-considerations"></a>Рекомендации по проверке подлинности Kerberos

При использовании проверки подлинности Kerberos для веб-служб необходимо переназначить учетные записи Kerberos и сбросить пароль после In-Place обновления. Сведения о том, как это сделать, см. в разделе ["Настройка проверки подлинности Kerberos"](/previous-versions/office/lync-server-2013/lync-server-2013-setting-up-kerberos-authentication).
  
## <a name="support-for-coexistence-with-lync-server-2013-and-lync-server-2010"></a>Поддержка сосуществования с Lync Server 2013 и Lync Server 2010
<a name="BKMK_PlanUpgradeFromLync2013"> </a>

Вы можете запустить Skype для бизнеса Server 2015 в той же топологии, что и Lync Server 2013 или Lync Server 2010, но не все три в одной топологии.
  
Если между Lync Server 2010 и Lync Server 2013 существует сосуществование, рекомендуется обновить всю топологию до Lync Server 2013, а затем выполнить обновление до Skype для бизнеса Server 2015 с помощью In-Place Upgrade. Дополнительные сведения см. в статье о миграции [с Lync Server 2010 на Lync Server 2013](/previous-versions/office/lync-server-2013/migration-from-lync-server-2010-to-lync-server-2013).
  
Если топология в основном Lync Server 2010, откатите компоненты Lync Server 2013 до Lync Server 2010 перед обновлением топологии до Skype для бизнеса Server 2015. В этом случае вы теряете преимущество обновления In-Place и имеете топологию сосуществования между Lync Server 2010 и Skype для бизнеса Server 2015.
  
На следующей схеме показана поддержка сосуществования Skype для бизнеса Server 2015 с Lync Server 2013 и Lync Server 2010.
  
![Схема, показывающая поддержку сосуществования Skype для бизнеса Server 2015 с Lync Server 2013 или Lync Server 2010.](../media/b5eb4f8c-577a-4a45-9a66-9531ce2c5dc1.png)
  
## <a name="upgrade-process-with-existing-survivable-branch-appliance-and-server"></a>Процесс обновления с использованием существующего устройства и сервера для обеспечения связи филиалов
<a name="BKMK_PlanUpgradeFromLync2013"> </a>

Skype для бизнеса Server 2015 не поддерживает обновление In-Place для устройства для обеспечения связи в филиалах (SBA) или сервера филиала для обеспечения связи (SBS).
  
Однако мы поддерживаем сосуществование центров обработки Skype для бизнеса Server с Lync Server 2010 или Lync Server 2013 SBA/SBS. 
  
При планировании In-Place обновления пула переднего плана (FE) Lync Server 2013 со связанной ветвью можно оставить существующих пользователей в SBA или SBS Lync Server 2013. Во время обновления пользователи SBA/SBS переходют в режим устойчивости и возвращаются к нормальной функциональности после завершения обновления. Дополнительные сведения о работе пользователей в режиме устойчивости см. в функциях устойчивости сайта филиала [в Lync Server 2013](/previous-versions/office/lync-server-2013/lync-server-2013-branch-site-resiliency-features).
  
При переносе топологии Lync Server 2010 в Skype для бизнеса Server 2015 необходимо добавить SBA/SBS в топологию, как при миграции на Lync Server 2013. Необходимые действия см. в статье "Подключение устройства обеспечения связи филиалов к пулу переднего плана [Lync Server 2013"](/previous-versions/office/lync-server-2013/lync-server-2013-connecting-survivable-branch-appliance-to-lync-server-2013-front-end-pool).
  
Для топологий сосуществования Lync Server 2010 и Lync Server 2013 сначала выполните рекомендации, приведенные в разделе "Поддержка сосуществования с Lync Server 2013 и Lync Server 2010".
