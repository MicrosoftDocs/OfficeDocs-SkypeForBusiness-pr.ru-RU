### <a name="os-for-domain-controllers"></a>ОС для контроллеров домена

Итак, какую ОС контроллера домена можно использовать? У нас есть следующий список:

- Windows Server 2019 (необходимо иметь накопительный обновления 5 для Skype для бизнеса Server 2015 или более поздней версии)
  
- Windows Server 2016
    
- Windows Server 2012 R2
    
- Windows Server 2012
    
- Windows Server 2008 R2
    
- Windows Server 2008
    
Теперь функциональный уровень домена любого домена, в который развертывается Skype для бизнеса Server 2015, и функциональный уровень леса любого леса, в который развертывается Skype для бизнеса Server 2015, должен быть одним из следующих:

- Windows Server 2019 (необходимо иметь накопительный обновления 5 для Skype для бизнеса Server 2015 или более поздней версии)
  
- Windows Server 2016
    
- Windows Server 2012 R2
    
- Windows Server 2012
    
- Windows Server 2008 R2
    
- Windows Server 2008
    
- Windows Server 2003
    
Можно ли использовать в этих средах контроллеры домена только для чтения? Конечно, при наличии доступных для написи контроллеров домена на том же сайте, что и Skype для бизнеса Server.
  
Теперь важно знать, что Skype для бизнеса Server 2015 не поддерживает домены с одним меткой. Что это такое? Если у вас есть корневой домен с меткой contoso.local, это будет нормально. Если у вас есть корневой домен с именем local, он не будет работать и в результате не будет поддерживаться. Немного больше об этом было написано в этой статье [базы знаний.](https://support.microsoft.com/kb/300684/en-us)
  
Skype для бизнеса Server 2015 также не поддерживает переименование доменов. If you've really got to do that, then you'll need to uninstall Skype for Business Server 2015, do the domain rename, and then reinstall Skype for Business Server 2015.
  
Наконец, вы можете работать с доменом с заблокированной средой AD DS, и все в порядке. Дополнительные сведения о развертывании Skype для бизнеса Server 2015 в такой среде можно найти в документы по развертыванию.
  
### <a name="ad-topologies"></a>AD Topologies

Поддерживаемые skype для бизнеса Server 2015 topologies:
  
- один лес с одним доменом;
    
- один лес с одним деревом и несколькими доменами;
    
- один лес с несколькими деревьями и несвязанными пространствами имен;
    
- несколько лесов в топологии с центральным лесом;
    
- несколько лесов в топологии с лесом ресурсов.
    
- Несколько лесов в топологии леса ресурсов Skype для бизнеса с Exchange Online
    
- Несколько лесов в топологии леса ресурсов со Skype для бизнеса Online и Azure Active Directory Connect

На этой схеме снова показан один лес, но он также имеет один или несколько дополнительных доменов (в данном примере три). Таким образом, домен, в который создаются пользователи, может быть не таким, как домен, в который развернут Skype для бизнеса Server 2015. Зачем об этом беспокоиться? Важно помнить, что при развертывании пула переднего сервера Skype для бизнеса Server все серверы в этом пуле должны быть в одном домене. Вы можете использовать меж доменное администрирование с помощью поддержки универсальных групп администраторов Windows в Skype для бизнеса Server.
  
На схеме выше видно, что пользователи из одного домена могут получать доступ к пулам Skype для бизнеса Server из одного домена или из разных доменов, даже если эти пользователи находятся в другом домене.

Возможно, у вас есть топология, аналогичная этой схеме, где имеется один лес, но внутри этого леса несколько доменов с отдельными пространствами имен AD. В этом случае эта схема хорошо проиллюстрирована, так как пользователи в трех разных доменах имеют доступ к Skype для бизнеса Server 2015. Сплошные линии указывают на то, что они имеют доступ к пулу Skype для бизнеса Server в собственном домене, а пунктирная линия указывает, что они собираются в пул в другом дереве.
  
Как видно, пользователи одного домена, одного дерева или даже другого дерева могут успешно получать доступ к пулам.
  
#### <a name="multiple-forests-in-a-central-forest-topology"></a>несколько лесов в топологии с центральным лесом;

![Схема топологии с несколькими лесами в центральном лесу](../../media/fec40746-4254-4c84-86b9-aad4a616ea2f.png)
  
Skype для бизнеса Server 2015 поддерживает несколько лесов, настроенных в топологии центрального леса. Если вы не уверены в том, что это так, в центральном лесу в топологии используются объекты для представления пользователей в других лесах и размещены учетные записи пользователей для всех пользователей в лесу.
  
Как это работает? Продукт синхронизации каталогов (например, Forefront Identity Manager или FIM) управляет учетной записью пользователя организации на протяжении всего их существования. Когда учетная запись создается или удаляется из леса, это изменение синхронизируется с соответствующим контактом в центральном лесу.
  
Очевидно, что если ваша инфраструктура AD находится на месте, переход к этой топологии может быть непростой задачей, но если вы уже там или планируете использовать инфраструктуру леса, это может быть хорошим выбором. Развертывание Skype для бизнеса Server 2015 можно централизовать в одном лесу, а пользователи могут искать, общаться и просматривать присутствие других пользователей в любом лесу. Все обновления контактов пользователей автоматически обрабатываются с помощью программного обеспечения синхронизации.
  
#### <a name="multiple-forests-in-a-skype-for-business-resource-forest-topology"></a>Несколько лесов в топологии леса ресурсов Skype для бизнеса
<a name="BKMK_multipleforestopology"> </a>

![Схема топологии с несколькими лесами в лесу ресурсов](../../media/41efa3b6-d9e6-47df-992b-fefcfc39a80d.png)
  
Также поддерживается топология леса ресурсов; именно здесь лес выделен для запуска серверных приложений, таких как Microsoft Exchange Server и Skype для бизнеса Server 2015. В этом лесу ресурсов также размещено синхронизированное представление объектов активных пользователей, но учетные записи пользователей с поддержкой входа не включены. Таким образом, лес ресурсов — это среда общих служб для других лесов, в которых находятся объекты пользователей, и они имеют отношение доверия уровня леса с лесом ресурсов.
  
Обратите внимание Exchange Server можно развернуть в том же лесу ресурсов, что и Skype для бизнеса Server, или в другом лесу.
  
Чтобы развернуть Skype для бизнеса Server 2015 в такой топологии, необходимо создать один отключенный объект пользователя в лесу ресурсов для каждой учетной записи пользователя в лесах пользователей (если Microsoft Exchange Server уже находится в среде, это можно сделать за вас). Затем вам потребуется средство синхронизации службы каталогов (например, Forefront Identity Manager или FIM) для управления учетной записью пользователей на протяжении их жизненного цикла.

В этой топологии существует один или несколько лесов пользователей, а Skype для бизнеса Server развертывается в выделенном лесу ресурсов. Exchange Server могут быть развернуты локально в том же лесу ресурсов или другом лесу и настроены для гибридного развертывания с Exchange Online, или службы электронной почты могут предоставляться исключительно exchange Online для локальной учетной записи. Для этой топологии схема недоступна.

 Организация также использует Microsoft 365 или Office 365 и использует [Azure Active Directory Connect](https://go.microsoft.com/fwlink/p/?LinkId=614836) для синхронизации своих учетных записей в локальной организации с Microsoft 365 или Office 365. ## <a name="domain-name-system-dns"></a>Система доменных имен (DNS)
<a name="DNS"> </a>

Skype для бизнеса Server 2015 требует DNS по следующим причинам:
  
- Служба DNS позволяет Skype для бизнеса Server 2015 обнаруживая внутренние серверы или пулы, что обеспечивает взаимодействие между серверами.
    
- DNS позволяет клиентские компьютеры обнаруживать пул переднего сервера или сервер Standard Edition, используемый для транзакций SIP.
    
- Он связывает простые URL-адреса для конференций с серверами, на которые размещены эти конференции.
    
- Служба DNS позволяет внешним пользователям и клиентские компьютеры подключаться к вашим серверам или обратному прокси-серверу HTTP для обмена мгновенными сообщениями или для обмена мгновенными сообщениями.
    
- Это позволяет устройствам объединенных коммуникаций, которые не вошли в систему, обнаруживая пул переднего сервера или сервер Standard Edition, на которых запущена веб-служба обновления устройств, получать обновления и отправлять журналы.
    
- Использование DNS позволяет мобильным клиентам автоматически обнаруживать ресурсы веб-служб без необходимости ввода URL-адресов вручную в параметрах устройства.
    
- И используется в балансировку нагрузки на DNS.
    
Важно отметить, что Skype для бизнеса Server 2015 не поддерживает международные доменные имена (IDNs).
  
Кроме того, очень важно помнить, что любое имя в DNS совпадает с именем компьютера, настроенным на любом сервере, используемом Skype для бизнеса Server 2015. В частности, у нас не может быть коротких имен в среде и должны быть FQDNs для построщика топологий.
  
Это кажется логическим для любого компьютера, уже присоединив его к домену, но если у вас есть сервер, который не присоединялся к вашему домену, он может иметь короткое имя по умолчанию без суффикса домена. Убедитесь, что это не так, в DNS или на edge Server, или на любом сервере или пуле Skype для бизнеса Server 2015.
  
И не используйте символы Юникода и подчеркиваия. Стандартные символы (A-Z, a-z, 0-9 и дефис) — это символы, которые будут поддерживаться внешними DNS и общедоступными службами сертификации (необходимо назначить FQDN имени SN в сертификате, не забудьте), поэтому вы пощадите себя, если у вас есть это имя.
  
Дополнительные статьи о требованиях к DNS для сети можно узнать в разделе ["Сеть"](../../plan-your-deployment/network-requirements/network-requirements.md) документации по планированию.
  
## <a name="certificates"></a>Сертификаты
<a name="Certs"> </a>

Одно из самых важных вещей, которые можно сделать перед развертыванием, — убедиться, что у вас есть сертификаты в порядке. Skype для бизнеса Server 2015 нуждается в инфраструктуре открытых ключей (PKI) для подключений TLS и MTLS. По сути, для безопасного обмена данными стандартизированным способом Skype для бизнеса Server использует сертификаты, выданные центром сертификации (ЦС).
  
Вот некоторые из того, для чего Skype для бизнеса Server 2015 использует сертификаты:
  
- Подключения TLS между клиентами и серверами
    
- для подключений MTLS между серверами;
    
- для федерации с использованием автоматического обнаружения DNS партнеров;
    
- для доступа удаленных пользователей к обмену мгновенными сообщениями;
    
- Доступ внешних пользователей к сеансам аудио- и видеосвязи, общий доступ к приложениям и аудиоконференциям
    
- Беседа с веб-приложениями и Outlook Web Access (OWA)
    
Поэтому планирование сертификатов является обязательным. Теперь рассмотрим список некоторых моментов, которые необходимо помнить при запросе сертификатов:
  
- Все сертификаты серверов должны поддерживать авторизацию сервера (EKU сервера).
    
- Все сертификаты серверов должны содержать точку распространения списка отзыва сертификатов (CDP).
    
- Все сертификаты должны быть подписаны с помощью алгоритма подписи, поддерживаемой операционной системой. Skype для бизнеса Server 2015 поддерживает наборы дайджест-размеров SHA-1 и SHA-2 (224, 256, 384 и 512 бит) и соответствует требованиям операционной системы или превышает их.
    
- Автоматическая регистрация поддерживается для внутренних серверов со Skype для бизнеса Server 2015.
    
- Автоматическая регистрация не поддерживается для серверов Skype для бизнеса Server 2015.
    
- Запрос на сертификат через Интернет в центр сертификации Windows Server 2003 необходимо подавать с компьютера, работающего под управлением либо Windows Server 2003 с пакетом обновления 2 (SP2), либо Windows XP.
    
> [!NOTE]
> Хотя KB922706 обеспечивает поддержку устранения проблем с регистрацией веб-сертификатов в веб-регистрации служб сертификатов Windows Server 2003, она не позволяет использовать Windows Server 2008, Windows Vista или Windows 7 для запроса сертификата из ЦС Windows Server 2003. 
  
> [!NOTE]
> Использование алгоритма подписи RSASSA-PSS неподтвершено и, среди прочего, может привести к ошибкам при входе в систему и переададации вызовов. 

> [!NOTE]
> Skype для бизнеса Server 2015 не поддерживает сертификаты CNG.
  
- Поддерживаются ключи шифрования длиной 1024, 2048 и 4096. Рекомендуется использовать ключи длиной 2048 и больше.
    
- По умолчанию дайджест-алгоритмом или подписью с hash является RSA. Также ECDH_P256, ECDH_P384 и ECDH_P521 алгоритмы.
    
Таким образом, это очень много думайте, и, определенно, существует множество уровней удобства при запросе сертификатов из ЦС. Ниже приведены некоторые дополнительные рекомендации по максимальной безболезненной подготовке к планированию.
  
### <a name="certificates-for-your-internal-servers"></a>Сертификаты для внутренних серверов

Вам потребуется сертификаты для большинства внутренних серверов, и, скорее всего, вы получите их из внутреннего ЦС (который находится в вашем домене). При желании вы можете запросить эти сертификаты у внешнего ЦС (расположенного в Интернете). Если вы хотите узнать, в какой общедоступный ЦС следует перейти, ознакомьтесь со списком партнеров по сертификатам [unified Communications.](/SkypeForBusiness/certification/services-ssl)
  
Кроме того, вам потребуется сертификат, когда Skype для бизнеса Server 2015 взаимодействует с другими приложениями и серверами, такими как Microsoft Exchange Server. Очевидно, что это должен быть сертификат, который эти другие приложения и серверы могут использовать поддерживаемый способ. Skype для бизнеса Server 2015 и другие продукты Майкрософт поддерживают протокол Open Authorization (OAuth) для проверки подлинности и авторизации между серверами. Если вы заинтересованы в этом, у нас есть дополнительная статья по планированию OAuth и Skype для бизнеса Server 2015.
  
Skype для бизнеса Server 2015 также включает поддержку (без необходимости) сертификатов, подписанных с помощью функции шифрования SHA-256. Для поддержки внешнего доступа с помощью SHA-256 внешний сертификат должен быть выдан общедоступным ЦС с помощью SHA-256.

### <a name="certificates-for-your-persistent-chat-server"></a>Сертификаты для сервера сохраняемой беседы

При установке сервера сохраняемой беседы вам потребуется сертификат, выданный тем же ЦС, что и для внутренних серверов Skype для бизнеса Server 2015. Это необходимо сделать для каждого сервера с веб-службами сохраняемой беседы для отправки и загрузки файлов. Настоятельно рекомендуется иметь необходимые сертификаты перед началом установки сохраняемой беседы, и если ваш ЦС является внешним, еще больше (на выдачу этих данных может потребоваться некоторое время).
  
### <a name="certificates-for-external-user-access-edge"></a>Сертификаты для доступа внешних пользователей (Edge)

Skype для бизнеса Server 2015 поддерживает  использование единого открытого сертификата для внешних интерфейсов доступа и веб-аудиоконференций, а также службу проверки подлинности аудио- и видеоконференций, которая предоставляется через серверы. Внутренний интерфейс Edge обычно использует частный сертификат, выданный внутренним ЦС, но при этом можно также использовать общедоступный сертификат, если он выдан доверенным ЦС.
  
Обратный прокси-сервер (RP) также будет использовать общедоступный сертификат и шифрует связь от RP к клиентам, а RP — к внутренним серверам по протоколу HTTP (или, точнее, по протоколу TLS через HTTP).
  
### <a name="certificates-for-mobility"></a>Сертификаты для мобильности

Если вы развертываете мобильность и поддерживаете автоматическое обнаружение для мобильных клиентов, вам потребуется включить в сертификаты дополнительные записи альтернативных имен субъектов для поддержки безопасных подключений от мобильных клиентов.
  
Какие сертификаты? Имена SAN необходимы для автоматического обнаружения сертификатов здесь:
  
- Пул директоров
    
- Интерфейсный пул
    
- Обратный прокси-сервер
    
Мы перечислим особенности в каждой таблице ниже.
  
В этом случае предварительное планирование хорошо подходит, но иногда вы развернули Skype для бизнеса Server 2015, не планируя развертывание мобильных устройств, и это происходит после того, как в вашей среде уже есть сертификаты. Повторное выдача их через внутренний ЦС обычно довольно проста, но с общедоступными сертификатами из общего ЦС это может быть немного дороже.
  
Если это то, что вы ищете, и если у вас много доменов SIP (что сделает добавление SANS более дорогим), вы можете настроить обратный прокси-сервер для использования HTTP для исходного запроса службы автообнаружния, а не с помощью HTTPS (конфигурация по умолчанию). Дополнительные сведения об этом можно получить в разделе "Планирование мобильности".
  
Требования к сертификатам пула директоров и пула переднего входа:
  
|**Описание**|**Запись SAN**|
|:-----|:-----|
|URL-адрес внутренней службы автонаружия  <br/> |SAN=lyncdiscoverinternal.\<sipdomain\>  <br/> |
|URL-адрес внешней службы автооружия  <br/> |SAN=lyncdiscover.\<sipdomain\>  <br/> |
   
Можно также использовать SAN= \* .\<sipdomain\>
  
Требования к сертификатам обратного прокси-сервера (public CA):
  
|**Описание**|**Запись SAN**|
|:-----|:-----|
|URL-адрес внешней службы автооружия  <br/> |SAN=lyncdiscover.\<sipdomain\>  <br/> |

> [!NOTE]
> Прослушиватель обратного прокси-сервера будет иметь SANs для URL-адресов внешних веб-служб. Some examples would be SAN=skypewebextpool01.contoso.com and dirwebexternal.contoso.com, if you've deployed the Director, (which is optional). 
  
## <a name="file-share"></a>Файловый ресурс
<a name="Fileshare"> </a>

Skype для бизнеса Server 2015 может использовать один и тот же файл для всех хранилищ файлов. Необходимо иметь в виду следующее:
  
- Файловый файл должен быть либо в непосредственно подключенном хранилище (DAS), либо в сети хранения данных (SAN), в том числе в распределенной файловой системе (DFS), а также в избыточном массиве независимых дисков (RAID) для хранилищ файлов. Для дальнейшего чтения на DFS для Windows Server 2012, ознакомьтесь с [этой страницей DFS](https://technet.microsoft.com/library/jj127250.aspx).
    
- Мы рекомендуем общий кластер для общей папки. Если вы используете его, следует кластерить Windows Server 2012 или Windows Server 2012 R2. Windows Server 2008 R2 также приемлем. Почему последняя версия Windows? Более старые версии могут не иметь нужных разрешений для всех функций. Для создания файловой папки можно использовать [](https://support.microsoft.com/help/224967/how-to-create-file-shares-on-a-cluster) администратора кластера. Эти сведения помогут вам в создании файловой папки в статье о кластере.
    
> [!CAUTION] 
> Следует знать, что использование хранилища NAS в качестве файлового хранилища не поддерживается, поэтому используйте один из указанных выше вариантов.
