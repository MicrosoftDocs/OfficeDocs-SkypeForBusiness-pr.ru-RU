### <a name="microsoft-exchange-storage"></a>Хранилище Microsoft Exchange
Файлы содержимого собраний, например презентаций PowerPoint, архивируются как вложения. Если вы хотите сохранить Скайп для бизнес-данных архива с данные соответствия Exchange, необходимо использовать Exchange для развертывания Exchange и убедитесь, что максимальный размер хранилища поддерживает хранение файлов контента собрания. Необходимо развернуть Exchange перед развертыванием и Включение архивации с помощью параметра интеграции с Microsoft Exchange. 
    
    If you choose to use Exchange storage, you do not need to deploy separate SQL Server databases for archiving, unless you have Skype for Business users who are not homed on your Exchange servers. If you deploy archiving using the Microsoft Exchange integration option, Skype for Business archive data is stored with Exchange compliance data only for the users who are homed on your Exchange servers. 
  
## <a name="hardware-and-software-requirements-for-archiving-in-skype-for-business-server-2015"></a>Требования к программному обеспечению и оборудованию для архивации в Skype для бизнеса Server 2015
  
Архивация не определенных серверных ролей, необходимо установить на отдельный сервер для архивации. Объединенные агенты коллекции данных являются устанавливается и активируется автоматически на каждый пул переднего плана Enterprise Edition и каждого сервера Standard Edition. Вам потребуется активировать и опубликовать вашу топологию архивации с помощью построителя топологий.
    
Архивация использует Скайп для хранения файлов Business Server для временного хранения файлы содержимого собраний, поэтому вам не нужно настраивать отдельное файловое хранилище для архивации.
    
Очереди сообщений не требуется.
    
Необходимо настроить инфраструктуру для хранилища архивации. Этот компонент включает выбор любого из Exchange или архивации хранилища, используемого с помощью SQL Server.   Скайп для архивации сервера Business требования к инфраструктуре совпадают с требованиями для развертывания Скайп для Business Server. Дополнительные сведения см [требования к вашей Скайп для бизнес-среде](../../plan-your-deployment/requirements-for-your-environment/requirements-for-your-environment.md). 
  
> [!NOTE]
> Для поддержки пользователей, которые не размещаются на серверах Exchange, или если вы не хотите использовать параметра интеграции с Microsoft Exchange, необходимо развернуть архивации хранилища, используемого с помощью базы данных SQL Server 64-разрядная версия. 
    
Необходимо настроить платформах SQL Server перед развертыванием и включение архивирования. Если учетная запись, которую планируется использовать для публикации топологии, имеет соответствующие права и разрешения администратора, базу данных архивации (LcsLog) можно создать при публикации топологии. Базу данных можно создать и позднее в ходе процедуры установки. Для получения дополнительных сведений о SQL Server обратитесь к [документации по SQL Server](https://go.microsoft.com/fwlink/p/?linkID=129045).
    
Нагрузка компонента архивации может оказаться значительной. Таким образом необходимо убедиться, что места на диске подходит для серверов переднего плана, на которых включен архивации.

```

Однако в автономном режиме необходимо обеспечить доступность исходных файлов для команды. Дополнительные сведения об использовании PowerShell к установке ролей и компонентов, просмотрите [установки или удаления роли, службы ролей или компонентов](https://technet.microsoft.com/en-us/library/hh831809.aspx) не забудьте центра обновления Windows снова запустите после установки необходимых компонентов, даже при использовании команды PowerShell.
