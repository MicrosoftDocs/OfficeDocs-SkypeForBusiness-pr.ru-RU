## <a name="hardware-for-skype-for-business-server-2015"></a>Оборудование для Skype для бизнеса Server 2015
<a name="Hardware"> </a>

Теперь, когда ваша топология отключена (а вы не можете ознакомиться со статьей [основы топологии для Skype для бизнеса Server 2015](../../plan-your-deployment/topology-basics/topology-basics.md) ), то пришло время думать о серверах. Серверам Skype для бизнеса Server 2015 потребуется 64-разрядное оборудование. Рекомендации по оборудованию приведены ниже. Они не являются обязательными, но они отражают требования, необходимые для достижения оптимальной производительности. Документация по планированию емкости поможет вам определить, нужно ли вам больше, в зависимости от обстоятельств.
  
Рекомендуемое оборудование для серверов переднего плана, внутренних серверов, серверов Standard Edition и серверов сохраняемого чата:

|:-----|:-----|
|ЦП  <br/> |64-разрядный двухъядерный процессор с тактовой частотой в шестнадцатеричном формате, 2,26 ГГц или выше.  <br/> Процессоры Intel Itanium не поддерживаются для ролей Skype для бизнеса Server 2015.  <br/> |
|Память  <br/> |32 гигабайт (ГБ).  <br/> |
|Диск  <br/> |ОБОИХ  <br/> • 8 или более 10000 жестких дисков с частотой об/мин, на которых не менее 72 ГБ свободного места (два диска с RAID 1 и 6 с RAID 10).  <br/> ИЛИ  <br/> • Твердотельные накопители (SSDs), обеспечивающие одинаковый объем свободного места и быстродействие до 8 10000 механических дисков с частотой до об/мин.  <br/> |
|Сеть  <br/> |1 сетевой адаптер с двумя портами, 1 Гбит/с или выше (можно использовать 2 сетевых адаптера, но им необходимо присвоить один MAC-адрес и один IP-адрес).  <br/> Конфигурации с двумя или несколькими сетевыми конфигурациями **не** поддерживаются для серверов переднего плана, серверов переднего плана, серверов Standard Edition и серверов сохраняемого чата. <br/> Пока они не видны операционной системе и используются для мониторинга и управления аппаратным обеспечением сервера, вы можете использовать системы управления аппаратным стандартом, такие как DRAC или ILO. Этот сценарий не является многосетевым сервером и поддерживается.  <br/> |

### <a name="microsoft-exchange-storage"></a>Хранилище Microsoft Exchange
Файлы содержимого собраний, например презентации PowerPoint, архивируются как вложения. Если вы хотите хранить архивные данные Skype для бизнеса с использованием данных о соответствии требованиям Exchange, необходимо использовать Exchange для развертывания Exchange и обеспечить поддержку хранения файлов контента собраний в максимальном объеме хранилища. Перед развертыванием и включением архивации с помощью функции интеграции с Microsoft Exchange необходимо развернуть Exchange. 
    
    If you choose to use Exchange storage, you do not need to deploy separate SQL Server databases for archiving, unless you have Skype for Business users who are not homed on your Exchange servers. If you deploy archiving using the Microsoft Exchange integration option, Skype for Business archive data is stored with Exchange compliance data only for the users who are homed on your Exchange servers. 
  
## <a name="hardware-and-software-requirements-for-archiving-in-skype-for-business-server-2015"></a>Требования к оборудованию и программному обеспечению для архивации в Skype для бизнеса Server 2015
  
Архивация не является определенной ролью сервера, для архивации не требуется устанавливать отдельный сервер. Единые агенты сбора данных устанавливаются и активируются автоматически в каждом интерфейсном пуле Enterprise Edition и на каждом сервере Standard Edition. Вам потребуется включить и опубликовать топологию архивации с помощью построителя топологий.
    
При архивации используется хранилище файлов Skype для бизнеса Server для временного хранения файлов контента собраний, поэтому не нужно настраивать отдельное хранилище файлов для архивации.
    
Служба очередей сообщений Майкрософт не является обязательной.
    
Вам потребуется настроить инфраструктуру для хранения архивов. Это включает в себя выбор Exchange или архивации хранилища с помощью SQL Server.   Требования к инфраструктуре архивации в Skype для бизнеса Server совпадают с требованиями к развертыванию Skype для бизнеса Server. Дополнительные сведения см [в разделе Требования к среде Skype для бизнеса](../../plan-your-deployment/requirements-for-your-environment/requirements-for-your-environment.md). 
  
> [!NOTE]
> Для поддержки пользователей, не размещенных на серверах Exchange Server, или если вы не хотите использовать функцию интеграции с Microsoft Exchange, необходимо развернуть хранилище архивации с помощью базы данных SQL Server с 64 бит. 
    
Перед развертыванием и включением архивации необходимо настроить платформы SQL Server. Если у учетной записи, которая будет использоваться для публикации топологии, есть необходимые права и разрешения администратора, вы можете создать базу данных архивации (LcsLog) при публикации топологии. Кроме того, вы можете создать базу данных позже, включенную в процедуру установки. Подробные сведения о SQL Server можно найти в [документации по SQL Server](https://go.microsoft.com/fwlink/p/?linkID=129045).
    
Увеличение нагрузки для архивации может быть существенным. Поэтому необходимо убедиться, что дисковое пространство достаточно для серверов переднего плана, на которых включена архивация.

### <a name="sql-mirroring-sql-clustering-and-sql-always-on"></a>Зеркальное отображение SQL, кластеризация SQL и SQL Always on

Вы можете использовать зеркалирование SQL или кластеризацию SQL в Skype для бизнеса Server 2015, который поддерживается. Зеркальное отображение SQL настраивается с помощью построителя топологий Skype для бизнеса Server. Если вы планируете настроить кластеризацию SQL, это выполняется в SQL Server.
  
Убедитесь, что у вас есть активная/пассивная конфигурация для кластеризации SQL, так как это вхат'с поддерживается. Не предоставляйте пассивный узел к любому другому экземпляру SQL.
  
Для отказоустойчивой кластеризации можно выполнить указанные ниже действия.
  
Два узла:
  
- Microsoft SQL Server 2019 Standard (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.

- Microsoft SQL Server 2017 Standard (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.

- Microsoft SQL Server 2016 Standard (64-разрядный выпуск) с пакетом обновления 1 (SP1) или более поздней версии. Мы рекомендуем работать с последним пакетом обновления.

- Microsoft SQL Server 2014 Standard (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.
    
-  Microsoft SQL Server 2012 Standard (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.

16 узел:

- Microsoft SQL Server 2019 Enterprise (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.

- Microsoft SQL Server 2017 Enterprise (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.

- Microsoft SQL Server 2016 корпоративный (64-разрядный выпуск) с пакетом обновления 1 (SP1) или более поздней версии. Мы рекомендуем работать с последним пакетом обновления.
  
- Microsoft SQL Server 2014 Enterprise (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.
    
- Microsoft SQL Server 2012 Enterprise (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.

> [!IMPORTANT]
> Для обновления необходимо убедиться, что на серверах переднего плана установлен по крайней мере SQL Server 2012 с пакетом обновления 1 (SP1) до обновления. [Вот ссылка](https://www.microsoft.com/download/details.aspx?id=35575) на пакет обновления 1 (SP1), если вы хотите скачать его сразу же.
  
Если вам нужно больше узнать о зеркальном отображении SQL, у нас есть высокий уровень доступности внутреннего сервера в статье Skype для бизнеса Server 2015. Настройка кластеризации SQL Server для Skype для бизнеса Server 2015 содержит инструкции по обеспечению готовности кластера к работе. Кроме того, существуют дополнительные ссылки на отказоустойчивую кластеризацию для SQL, [2014](https://technet.microsoft.com/library/hh231721.aspx), [2012](https://technet.microsoft.com/library/hh231721%28v=sql.110%29.aspx)и [2008](https://technet.microsoft.com/library/ms189134%28v=sql.105%29.aspx).
  
> [!NOTE]
> Новые возможности выпуска 2015 поддерживают SQL Always on. Он поддерживается, и вы можете ознакомиться с дополнительными сведениями в разделе [Высокая доступность внутреннего сервера в разделе Skype для бизнеса server 2015](../../plan-your-deployment/high-availability-and-disaster-recovery/back-end-server.md) .

> [!NOTE]
> Зеркальное отображение SQL доступно в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Группы доступности AlwaysOn, экземпляры отказоустойчивого кластера AlwaysOn (FCI) и методы отказоустойчивой кластеризации SQL предпочтительнее Skype для бизнеса Server 2019.  

## <a name="software-that-should-be-installed-before-a-skype-for-business-server-2015-deployment"></a>Программное обеспечение, которое следует установить перед развертыванием Skype для бизнеса Server 2015
<a name="Software"> </a>

Необходимо выполнить установку или настройку для любого сервера, на котором работает Skype для бизнеса Server 2015, и указанные ниже. После этого дополнительные требования к определенным ролям сервера.
  
> [NOTE!] Skype для бизнеса Server 2015 не поддерживает .NET Framework 4,8.
  
 **Все серверы:**
  
|**Программное обеспечение/роль**|**Details**|
|:-----|:-----|
|Windows PowerShell 3.0  <br/> |Все серверы Skype для бизнеса Server нуждаются в установке Windows PowerShell 3,0.  <br/> • Если вы выполняете установку на Windows Server 2012 или Windows Server 2012 R2, вы можете настроить его, так как он уже есть.  <br/> • Если вы выполняете обновление на Windows Server 2008 R2, вы можете скачать [Windows Management Framework 3,0](https://www.microsoft.com/download/details.aspx?id=34595) , чтобы получить ее. <br/> **Совет:** Когда у вас есть правильный PowerShell, подтвердите, что это Буилдверсион 6.2.9200.0 или более поздней версии, перейдя к командной строке `$PSVersionTable`PowerShell и введя команду. При этом должны быть нужные сведения.  <br/> |
|Платформа Microsoft .NET Framework  <br/> |Службы WCF — это **компонент** , установленный в качестве компонента Windows, в разделе **Диспетчер серверов**не требуется загружать файлы для загрузки. <br/> • Необходимо убедиться, что при установке этого компонента, если он уже установлен и вы его проверяете, что параметр **активации HTTP** также установлен и установлен, как показано ниже: <br/> ![Снимок экрана, демонстрирующий вариант активации HTTP в компонентах .NET Framework 4,5. ](../../media/a4064fa0-fa49-4474-bd98-b9a79ff68f8b.png)Не беспокойтесь, если вы получаете дополнительное всплывающее окно с сообщением, что для установки активации HTTP необходимо установить другие компоненты. Это нормально, нажмите кнопку ОК, чтобы продолжить. Если вы не получите это всплывающее окно, предполагаю, что эти элементы уже установлены, и не стесняйтесь.  <br/> Microsoft .NET Framework обычно устанавливается при установке Windows Server 2012 R2 или Windows Server 2016. Skype для бизнеса Server работает со следующими версиями Microsoft .NET Framework:  <br/> • .NET 3,5  <br/> • .NET 4,5  <br/> • .NET 4.6. x  <br/> • .NET 4.7.1 (для Skype для бизнеса Server CU 5 или более поздних версий)  <br/> • .NET 4.7.2 (для Skype для бизнеса Server CU 6 или более поздних версий)  <br/>  По умолчанию платформа .NET Framework 3,5, скорее всего, будет установлена на компьютере с Windows Server 2008 R2 (безусловно убедитесь, что перед обновлением), но на серверах Windows Server 2012 и Windows Server 2012 R2 (для новых установок). Чтобы добавить его в, вам потребуется доступ к установочному диску или носителю (место, из которого была установлена система Windows Server, или там, где находятся файлы установки). После этого установите его как компонент в диспетчере серверов и наведите указатель на установочный носитель (в папку **\саурцес\сксс** ) и продолжайте устанавливать ее. <br/> |
|Media Foundation  <br/> |Для Windows Server 2016, Windows Server 2012 и Windows Server 2012 R2 среда выполнения формата Windows Media устанавливается вместе с Microsoft Media Foundation.  <br/> На всех серверах переднего плана и серверах Standard Edition, используемых для конференц-связи, требуется, чтобы среда выполнения формата Windows Media Audio (WMA) выполняла приложения Windows Media Audio (WMA), воспроизводимые для объявлений и музыкальных файлов.  <br/> |
|Windows Identity Foundation  <br/> |Для поддержки сценариев проверки подлинности "сервер-сервер" для Skype для бизнеса Server 2015 требуется Windows Identity Foundation 3,5.  <br/> • Для Windows Server 2012 и Windows Server 2012 R2 нет необходимости загружать ничего. Откройте **Диспетчер серверов**и перейдите к **мастеру добавления ролей и компонентов**. **Windows Identity Foundation 3,5** отображается в разделе " **компоненты** ". Если этот флажок установлен, то хорошо. В противном случае выберите его и нажмите кнопку Далее, чтобы открыть кнопку **установить** . <br/> |
|Средства администрирования удаленного сервера  <br/> |Средства администрирования ролей: средства AD DS и AD LDS  <br/> |
   
 **Для серверов переднего плана и сервера Standard Edition также необходимы:**
  
|**Программное обеспечение/роль**|**Details**|
|:-----|:-----|
|IIS  <br/> |Службы IIS необходимы на всех серверах переднего плана, а также на всех серверах Standard Edition со следующими выбранными модулями:  <br/> • Общие функции HTTP: документ по умолчанию, ошибки HTTP, статический контент  <br/> • Работоспособность и диагностика: ведение журнала HTTP, средства ведения журнала, трассировка  <br/> • Производительность: сжатие статического содержимого, сжатие динамического содержимого  <br/> • Безопасность: Фильтрация запросов, проверка подлинности сопоставления сертификатов клиентов, проверка подлинности Windows  <br/> • Разработка приложений: .NET Extensibility 3,5, расширяемость .NET 4,5, ASP.NET 3,5, ASP.NET 4,5, расширения ISAPI, фильтры ISAPI  <br/> • Средства управления: консоль управления IIS, сценарии и средства управления IIS  <br/> Кроме того, следует отметить анонимный доступ, но при установке IIS вы не можете выбрать нужное место в списке.  <br/> |
|Windows Media Format Runtime  <br/> | Для Windows Server 2016, Windows Server 2012 и Windows Server 2012 R2 необходимо установить компонент **Media Foundation** в **диспетчере серверов**. Теперь вы можете запустить установку Skype для бизнеса Server 2015 без этого пользователя, но вам будет предложено установить ее, а затем перезагрузить сервер, прежде чем установка Skype для бизнеса Server 2015 будет продолжена. Лучше сделать это заранее. <br/> |
|Silverlight  <br/> |Вы можете установить последнюю версию Silverlight по [этой ссылке](https://www.microsoft.com/silverlight/).  <br/> |
   
> [!NOTE] 
> Если вы используете подсистему балансировки нагрузки, вам также может потребоваться включить просмотр каталогов. В противном случае будет загружена пустая страница, из-за которой подсистема балансировки нагрузки может считать сбой. 

Чтобы помочь вам, Вот пример скрипта PowerShell, который можно использовать для автоматизации этой задачи:

```PowerShell
Add-WindowsFeature NET-Framework-Core, RSAT-ADDS, Windows-Identity-Foundation, Web-Server, Web-Static-Content, Web-Default-Doc, Web-Http-Errors, Web-Dir-Browsing, Web-Asp-Net, Web-Net-Ext, Web-ISAPI-Ext, Web-ISAPI-Filter, Web-Http-Logging, Web-Log-Libraries, Web-Request-Monitor, Web-Http-Tracing, Web-Basic-Auth, Web-Windows-Auth, Web-Client-Auth, Web-Filtering, Web-Stat-Compression, Web-Dyn-Compression, NET-WCF-HTTP-Activation45, Web-Asp-Net45, Web-Mgmt-Tools, Web-Scripting-Tools, Web-Mgmt-Compat, Server-Media-Foundation, BITS, Desktop-Experience, Telnet-Client
```

> [!NOTE] 
> Команда ищет исходные файлы в определенном порядке. Если вы подключены к сети, команда получает доступ к обновлению Windows. Однако при работе в автономном режиме необходимо убедиться, что исходные файлы доступны для команды. Дополнительные сведения об использовании PowerShell для установки ролей и компонентов можно найти в [статье Установка и удаление ролей, служб ролей или компонентов](https://technet.microsoft.com/library/hh831809.aspx) не забудьте снова запускать обновление Windows после установки необходимых компонентов, даже если вы используете команду PowerShell.

 **Директорам также необходимы:**
  
Службы IIS со следующими выбранными модулями:
  
- Основные возможности HTTP
    
  - Документ по умолчанию
    
  - ошибки HTTP;
    
  - Статическое содержимое
    
- Работоспособность и диагностика
    
  - Ведение журнала HTTP
    
  - средства ведения журнала;
    
  - Образца
    
- Производительность
    
  - Сжатие статического содержимого
    
- Безопасность
    
  - Фильтрация запросов
    
  - проверка подлинности с сопоставлением сертификата клиента.
    
  - Проверка подлинности Windows
    
- Разработка приложений
    
  - Расширяемость .NET 3,5
    
  - Расширяемость платформы .NET 4.5
    
  - ASP.NET 3,5
    
  - ASP.NET 4,5
    
  - Расширение ISAPI
    
  - Фильтры ISAPI
    
(Если вы захотите, это тот же набор модулей, что и серверы переднего плана и серверы Standard Edition, с помощью динамических средств сжатия и управления содержимым.)
  
Кроме того, у нас есть приведенный ниже код PowerShell:
  
```PowerShell
Add-WindowsFeature RSAT-ADDS, Web-Server, Web-Static-Content, Web-Default-Doc, Web-Http-Errors, Web-Asp-Net, Web-Net-Ext, Web-ISAPI-Ext, Web-ISAPI-Filter, Web-Http-Logging, Web-Log-Libraries, Web-Request-Monitor, Web-Http-Tracing, Web-Basic-Auth, Web-Windows-Auth, Web-Client-Auth, Web-Filtering, Web-Stat-Compression, NET-WCF-HTTP-Activation45, Web-Asp-Net45, Web-Scripting-Tools, Web-Mgmt-Compat, Desktop-Experience, Telnet-Client
```

 **Для серверов сохраняемого чата также необходимы:**
  
Очередь сообщений, которая также называется MSMQ. Это компонент Windows Server, который можно установить в разделе компоненты в диспетчере серверов. Если вы хотите узнать больше об этом, изучите [установку и управление службой очередей сообщений](https://technet.microsoft.com/library/cc771474.aspx).
  
 **Последние мысли:**
  
Не устанавливайте клиентское программное обеспечение сервера Microsoft Internet Security and Acceleration (ISA) или любое другое программное обеспечение (LSP) (сторонние брандмауэры или антивирусные программы для проверки сети). любые серверы переднего плана или отдельные серверы-посредники. При установке этого программного обеспечения было замечено снижение производительности мультимедийного трафика.
