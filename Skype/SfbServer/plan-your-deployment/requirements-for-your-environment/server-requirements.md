## <a name="hardware-for-skype-for-business-server-2015"></a>Оборудование для Skype для бизнеса Server 2015
<a name="Hardware"> </a>

Теперь, когда у вас есть топология (а если нет, вы можете проверить топологию основы для [Skype для бизнеса Server 2015),](../../plan-your-deployment/topology-basics/topology-basics.md) пришло время подумать о серверах. Серверы Skype для бизнеса Server 2015 потребуют 64-битного оборудования. Ниже приведены рекомендации по оборудованию. Это не требования, но они отражают требования, необходимые для оптимальной производительности. У нас есть документация по планированию емкости, которая поможет вам определить, нужно ли вам больше, чем это, в зависимости от обстоятельств.

|**Компонент оборудования**|**Рекомендуемая**|
|:-----|:-----|
|ЦП  <br/> |64-bit dual processor, hex-core, 2.26 gigahertz (GHz) or higher.  <br/> Процессоры Intel Itanium не поддерживаются для ролей Skype для бизнеса Server 2015.  <br/> |
|Память  <br/> |32 гигабайта (ГБ).  <br/> |

|Сеть  <br/> |1 сетевой адаптер с двойным портом, 1 Гбит/с и более (можно использовать 2 сетевых адаптера, но их необходимо использовать с одним mac-адресом и одним IP-адресом).  <br/> Двойные или многоуровневые  конфигурации не поддерживаются для серверов переднего конца, серверов back end, серверов Standard Edition и постоянных серверов чата. <br/> Пока они не подвергаются воздействию операционной системы и используются для мониторинга и управления серверным оборудованием, вы можете не использовать системы управления полосами, такие как DRAC или ILO. ### <a name="microsoft-exchange-storage"></a>Хранилище Microsoft Exchange
Файлы содержимого собраний, например презентации PowerPoint, архивируются как вложения. Если вы хотите хранить архивные данные Skype для бизнеса с данными о соответствии Exchange, необходимо использовать Exchange для развертывания Exchange и обеспечить максимальный размер хранилища для хранения файлов контента собраний. Перед развертыванием и включением архивации Exchange необходимо развернуть Exchange с помощью параметра интеграции Microsoft Exchange. 
    
    If you choose to use Exchange storage, you do not need to deploy separate SQL Server databases for archiving, unless you have Skype for Business users who are not homed on your Exchange servers. If you deploy archiving using the Microsoft Exchange integration option, Skype for Business archive data is stored with Exchange compliance data only for the users who are homed on your Exchange servers. 
  
## <a name="hardware-and-software-requirements-for-archiving-in-skype-for-business-server-2015"></a>Требования к оборудованию и программному обеспечению для архива в Skype для бизнеса Server 2015
  
Архивативка не является определенной ролью сервера, вам не нужно устанавливать отдельный сервер для архива. Агенты единого сбора данных устанавливаются и активируются автоматически на каждом переднем конце корпоративного выпуска и на каждом сервере Standard Edition. Вам потребуется включить и опубликовать топологию архива с помощью Topology Builder.
    
Архивативная система использует хранилище файлов Skype для бизнес-сервера для временного хранения файлов контента собраний, поэтому не нужно настроить отдельный хранилище файлов для архивирования.
    
Очередь сообщений Майкрософт не требуется.
    
Необходимо настроить инфраструктуру для хранения архива. Это включает выбор хранилища Exchange или архивирования с помощью SQL Server.   Требования к инфраструктуре архивации серверов Skype для бизнеса такие же, как и для развертывания Skype для бизнес-сервера. Подробные сведения см. [в материале Requirements for your Skype for Business environment.](../../plan-your-deployment/requirements-for-your-environment/requirements-for-your-environment.md) 
  
> [!NOTE]
> Чтобы поддерживать пользователей, не размещающихся на серверах Exchange, или если вы не хотите использовать вариант интеграции Microsoft Exchange, необходимо развернуть хранилище архивации с помощью 64-SQL Server базы данных. 
    
Необходимо настроить платформы SQL Server перед развертыванием и включением архива. Если у учетной записи, которая будет использоваться для публикации топологии, есть необходимые права и разрешения администратора, вы можете создать базу данных архивации (LcsLog) при публикации топологии. Вы также можете создать базу данных позже, включив ее в процедуру установки. Сведения о SQL Server см. в SQL Server [документации.](/sql/sql-server/)
    
Увеличение нагрузки для архива может быть значительным. Поэтому необходимо убедиться, что пространство диска является достаточным для серверов переднего конца, на которых включена архивативка.

```PowerShell
Add-WindowsFeature NET-Framework-Core, RSAT-ADDS, Windows-Identity-Foundation, Web-Server, Web-Static-Content, Web-Default-Doc, Web-Http-Errors, Web-Dir-Browsing, Web-Asp-Net, Web-Net-Ext, Web-ISAPI-Ext, Web-ISAPI-Filter, Web-Http-Logging, Web-Log-Libraries, Web-Request-Monitor, Web-Http-Tracing, Web-Basic-Auth, Web-Windows-Auth, Web-Client-Auth, Web-Filtering, Web-Stat-Compression, Web-Dyn-Compression, NET-WCF-HTTP-Activation45, Web-Asp-Net45, Web-Mgmt-Tools, Web-Scripting-Tools, Web-Mgmt-Compat, Server-Media-Foundation, BITS, Desktop-Experience, Telnet-Client
```

> [!NOTE] 
> Команда ищет исходные файлы в определенном порядке. Если вы находитесь в Сети, команда имеет доступ к Обновлению Windows. Однако, если вы находитесь в автономном режиме, необходимо убедиться, что исходные файлы доступны для команды. Дополнительные сведения об использовании PowerShell для установки ролей и функций см. в рублях [Install или Uninstall Roles, Role Services или Features](/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh831809(v=ws.11)) Don't forget to run Windows Update again after you install prerequisites, even if you use the PowerShell command.

 **Последние мысли:**
  
Пожалуйста, не устанавливайте программное обеспечение клиента Microsoft Internet Security and Acceleration (ISA) Server или любого другого программного обеспечения для поставщиков многоуровневой службы Winsock (LSP) (любые сторонние брандмауэры или программное обеспечение проверки антивирусной сети будут включены здесь) на любой из ваших серверов переднего или автономных серверов-посредников. При установке этого программного обеспечения была замечена низкая производительность трафика мультимедиа.
