## <a name="hardware-for-skype-for-business-server-2015"></a>Оборудование для Skype для бизнеса Server 2015
<a name="Hardware"> </a>

Now that you have your topology down (and if you don't, you can check out the [Topology Basics for Skype for Business Server 2015](../../plan-your-deployment/topology-basics/topology-basics.md) topic), it's time to think about servers. Для серверов Skype для бизнеса Server 2015 потребуется 64-битное оборудование. Ниже приведены наши рекомендации по оборудованию. Они не являются требованиями, но отражают требования, необходимые для оптимальной производительности. У нас есть документация по планированию мощности, которая поможет вам определить, требуется ли вам больше, в зависимости от обстоятельств.
  
Рекомендуемое оборудование для серверов переднего сервера, тыловые серверы, серверы Standard Edition и серверы сохраняемой беседы:
  
|**Компонент оборудования**|**Рекомендуемая**|
|:-----|:-----|
|ЦП  <br/> |64-битный двухъядерный шестиядерный процессор, 2,26 гигагерц (ГГц) или выше.  <br/> Процессоры Intel Itanium не поддерживаются для ролей Skype для бизнеса Server 2015.  <br/> |
|Память  <br/> |32 гигабайта (ГБ).  <br/> |
|Диск  <br/> |ЛИБО:  <br/> • 8 или более жестких дисков с 10000 об/мин и не менее 72 ГБ свободного места на диске (два диска используют RAID 1 и 6 с помощью RAID 10).  <br/> ИЛИ  <br/> • Твердомоуголи (SSD), способные обеспечить то же свободное пространство и производительность, что и 8 механических дисков с 10 000 об/мин.  <br/> |
|Сеть  <br/> |1 двухпортовый сетевой адаптер, 1 Гбит/с или выше (можно использовать 2 сетевых адаптера, но их необходимо совместь с одним MAC-адресом и одним IP-адресом).  <br/> Конфигурации с двумя или  несколькими домами не поддерживаются для серверов переднего сервера, внутренних серверов, серверов Standard Edition и серверов сохраняемого чата. <br/> Если они не действовать для операционной системы и используются для мониторинга серверного оборудования и управления им, можно использовать системы управления вне диапазона, такие как DRAC или ILO. Этот сценарий не является многоадамидным сервером и поддерживается.  <br/> |
   
Рекомендуемое оборудование для серверов-посредников, автономных серверов-посредников, серверов видеосвязи и директоров:
  
|**Компонент оборудования**|**Рекомендуемая**|
|:-----|:-----|
|ЦП  <br/> |64-битный двухъядерный четырехъядерный процессор, 2,26 гигагерц (ГГц) или выше.  <br/> Процессоры Intel Itanium не поддерживаются для ролей Skype для бизнеса Server 2015.  <br/> |
|Память  <br/> |16 гигабайт.  <br/> |
|Диск  <br/> |ЛИБО:  <br/> • 4 или более жестких дисков с 10000 об/мин и не менее 72 ГБ свободного места на диске (диски должны быть в конфигурации 2x RAID 1).  <br/> ИЛИ  <br/> • Твердомоуголи (SSD), способные обеспечить то же свободное пространство и производительность, что и 4 механических диска с 10 000 об/мин.  <br/> |
|Сеть  <br/> |1 двухпортовый сетевой адаптер, 1 Гбит/с или выше (можно использовать 2 сетевых адаптера, но их необходимо совместь с одним MAC-адресом и одним IP-адресом).  <br/> Для серверов и директоров  видеосвязи не поддерживаются конфигурации с двумя или несколькими домами. <br/> Для работы edge-серверов требуются два сетевых интерфейса, которые являются сетевыми адаптерами с двумя портами, 1 Гбит/с или выше (или два сопряженных сетевых адаптера в сумме четыре, каждая пара с одним MAC-адресом и одним IP-адресом состоит из двух пар).  <br/> На автономных серверах-посредниках поддерживается установка дополнительных сетевых карточек для настройки определенного IP-адреса PSTN.  <br/> |

### <a name="microsoft-exchange-storage"></a>Хранилище Microsoft Exchange
Файлы содержимого собраний, например презентации PowerPoint, архивируются как вложения. Если вы хотите хранить архивные данные Skype для бизнеса с данными соответствия требованиям Exchange, необходимо использовать Exchange для развертывания Exchange и убедиться, что максимальный размер хранилища поддерживает хранение файлов содержимого собраний. Перед развертыванием и включением архивации с помощью параметра интеграции Microsoft Exchange необходимо развернуть Exchange. 
    
    If you choose to use Exchange storage, you do not need to deploy separate SQL Server databases for archiving, unless you have Skype for Business users who are not homed on your Exchange servers. If you deploy archiving using the Microsoft Exchange integration option, Skype for Business archive data is stored with Exchange compliance data only for the users who are homed on your Exchange servers. 
  
## <a name="hardware-and-software-requirements-for-archiving-in-skype-for-business-server-2015"></a>Требования к оборудованию и программному обеспечению для архива в Skype для бизнеса Server 2015
  
Архивная архива не является определенной ролью сервера, устанавливать отдельный сервер для архива не требуется. Агенты единой системы сбора данных устанавливаются и активируются автоматически на каждом пуле переднего сервера Enterprise Edition и на каждом сервере Standard Edition. Вам потребуется включить и опубликовать топологию архива с помощью построитель топологий.
    
Для архивирования используется хранилище файлов Skype для бизнеса Server для временного хранения файлов содержимого собраний, поэтому не нужно устанавливать отдельное хранилище файлов для архивирования.
    
Очередь сообщений Майкрософт не требуется.
    
Вам потребуется настроить инфраструктуру для хранения архивов. Это включает выбор хранилища Exchange или архивирования с помощью SQL Server.   Требования к инфраструктуре архивации Skype для бизнеса Server такие же, как и для развертывания Skype для бизнеса Server. For details, see [Requirements for your Skype for Business environment](../../plan-your-deployment/requirements-for-your-environment/requirements-for-your-environment.md). 
  
> [!NOTE]
> Для поддержки пользователей, которые не размещены на серверах Exchange Или если вы не хотите использовать вариант интеграции Microsoft Exchange, необходимо развернуть хранилище архивации с помощью 64-SQL Server базы данных. 
    
Перед развертыванием и включением архива необходимо настроить SQL Server платформы. Если у учетной записи, которая будет использоваться для публикации топологии, есть необходимые права и разрешения администратора, вы можете создать базу данных архивации (LcsLog) при публикации топологии. Вы также можете создать базу данных позже, включив ее в процедуру установки. Подробные сведения о SQL Server см. в SQL Server [документации.](https://go.microsoft.com/fwlink/p/?linkID=129045)
    
Увеличение нагрузки для архива может быть значительным. Поэтому следует убедиться, что на диске достаточно места для серверов переднего сервера, на которых включена архивная архива.

### <a name="sql-mirroring-sql-clustering-and-sql-always-on"></a>SQL зеркальное отражение, SQL кластеризация и SQL Always On

You are able to use SQL Mirroring or SQL Clustering with Skype for Business Server 2015, it's supported. SQL зеркального отражания настроено с помощью построщика топологий Skype для бизнеса Server. Если вы намерены настраивать SQL кластеризация, это делается в SQL Server.
  
Убедитесь, что у вас есть активная/пассивная конфигурация для SQL кластеров, так как это то, что поддерживается. Не делитесь пассивным узлом с любым другим экземпляром SQL.
  
Для отбойной кластерной кластерии можно использовать следующие следующую следующую группу:

Шестнадцать узлов:

> [!NOTE] 
> Вам также может потребоваться включить просмотр каталогов, если вы используете под балансировщик нагрузки. В противном случае будет загружена пустая страница, которую балансировка нагрузки может считать сбоем. 

Чтобы помочь вам, запустите пример сценария PowerShell, чтобы автоматизировать эту возможность:

```PowerShell
Add-WindowsFeature NET-Framework-Core, RSAT-ADDS, Windows-Identity-Foundation, Web-Server, Web-Static-Content, Web-Default-Doc, Web-Http-Errors, Web-Dir-Browsing, Web-Asp-Net, Web-Net-Ext, Web-ISAPI-Ext, Web-ISAPI-Filter, Web-Http-Logging, Web-Log-Libraries, Web-Request-Monitor, Web-Http-Tracing, Web-Basic-Auth, Web-Windows-Auth, Web-Client-Auth, Web-Filtering, Web-Stat-Compression, Web-Dyn-Compression, NET-WCF-HTTP-Activation45, Web-Asp-Net45, Web-Mgmt-Tools, Web-Scripting-Tools, Web-Mgmt-Compat, Server-Media-Foundation, BITS, Desktop-Experience, Telnet-Client
```

> [!NOTE] 
> Команда ищет исходные файлы в определенном порядке. Если вы в сети, команда будет получать доступ к Обновлению Windows. Однако если вы находитесь в автономном режиме, необходимо убедиться, что исходные файлы доступны для команды. Дополнительные сведения об использовании PowerShell для установки ролей и компонентов см. в подстановке "Установка и удалить [роли",](https://technet.microsoft.com/library/hh831809.aspx) "Службы ролей" или " Функции" "Не забудьте снова запустить Обновление Windows после установки необходимых компонентов, даже если используется команда PowerShell".

 **Последние минуты:**
  
Не устанавливайте клиентские программы Microsoft Internet Security and Acceleration (ISA) Server или любое другое программное обеспечение многоуровневой службы Winsock (LSP) (здесь будут включены сторонние брандмауэры или антивирусные программы для проверки сети) на любые серверы переднего уровня или автономные серверы-посредники. При установке этого программного обеспечения была замечена низкая производительность трафика мультимедиа.
