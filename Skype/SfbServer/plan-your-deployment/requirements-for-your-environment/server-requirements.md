## <a name="hardware-for-skype-for-business-server-2015"></a>Оборудование для Skype для бизнеса Server 2015 г.
<a name="Hardware"> </a>

Теперь, когда у вас есть топология вниз (а если нет, вы можете проверить Основы топологии для [Skype для бизнеса Server 2015),](../../plan-your-deployment/topology-basics/topology-basics.md) пришло время подумать о серверах. Skype для бизнеса Server 2015 г. потребуется 64-битное оборудование. Ниже приведены рекомендации по оборудованию. Это не требования, но они отражают требования, необходимые для оптимальной производительности. У нас есть документация по планированию емкости, которая поможет вам определить, нужно ли вам больше, чем это, в зависимости от обстоятельств.
  
Рекомендуемое оборудование для серверов переднего конца, серверов back end, выпуск Standard серверов и постоянных серверов чата:
  
|Компонент оборудования|Рекомендации|
|:-----|:-----|
|ЦП   |64-bit dual processor, hex-core, 2.26 gigahertz (GHz) or higher.  <br/> Процессоры Intel Itanium не поддерживаются для ролей Skype для бизнеса Server 2015 г.   |
|Память   |32 гигабайта (ГБ).   |
|Диск   |ЛИБО:  <br/> • 8 или более 10000 жестких дисков RPM с свободным диском не менее 72 ГБ (два диска с использованием RAID 1 и 6 с помощью RAID 10).  <br/> ИЛИ  <br/> • Диски твердого состояния (SSD), способные обеспечить одинаковое свободное пространство и аналогичную производительность механическим дискам с 8 10000 об/мин.   |
|Сеть   |1 сетевой адаптер с двойным портом, 1 Гбит/с и более (можно использовать 2 сетевых адаптера, но их необходимо использовать с одним mac-адресом и одним IP-адресом).  <br/> Двойные или многоуровневые  конфигурации не поддерживаются для серверов переднего конца, серверов заднего выпуск Standard и постоянных серверов чата. <br/> Пока они не подвергаются воздействию операционной системы и используются для мониторинга и управления серверным оборудованием, вы можете не использовать системы управления полосами, такие как DRAC или ILO. Этот сценарий не представляет собой многоквартирный сервер, и он поддерживается.   |
   
Рекомендуемое оборудование для edge Servers, автономных серверов-посредников, серверов видеосвязи и директоров:
  
|Компонент оборудования|Рекомендации|
|:-----|:-----|
|ЦП   |64-bit dual processor, quad-core, 2.26 gigahertz (GHz) or higher.  <br/> Процессоры Intel Itanium не поддерживаются для ролей Skype для бизнеса Server 2015 г.   |
|Память   |16 гигабайт.   |
|Диск   |ЛИБО:  <br/> • 4 или более 10000 жестких дисков RPM с свободным диском не менее 72 ГБ (диски должны быть в конфигурации 2x RAID 1).  <br/> ИЛИ  <br/> • Диски твердого состояния (SSD), способные обеспечить одинаковое свободное пространство и аналогичную производительность с механическими дисками 4 10000 об/мин.   |
|Сеть   |1 сетевой адаптер с двойным портом, 1 Гбит/с и более (можно использовать 2 сетевых адаптера, но их необходимо использовать с одним mac-адресом и одним IP-адресом).  <br/> Двойные или многодомные конфигурации **не поддерживаются** для серверов и директоров video Interop Servers. <br/> Для edge-серверов потребуются два сетевых интерфейса с двойным портом сетевых адаптеров, 1 Гбит/с или более (или два парных сетевых адаптеров в общей сложности четыре, каждая пара которых состоит из одного MAC-адреса и одного IP-адреса в общей сложности двух пар).  <br/> На автономных серверах-посредниках поддерживается установка дополнительных карт сетевого интерфейса (NICs), позволяющих поддерживать конфигурацию определенного IP-адреса PSTN.   |

- Если у вас Windows Server 2012 R2, установите статью KB 2982006 перед обновлением. [Его можно найти здесь.](https://support.microsoft.com/kb/2982006/)

### <a name="microsoft-exchange-storage"></a>Хранилище Exchange Microsoft
Файлы содержимого собраний, например презентации PowerPoint, архивируются как вложения. Если вы хотите Skype для бизнеса архивные данные с Exchange соответствия требованиям, необходимо использовать Exchange для развертывания Exchange и убедиться, что максимальный размер хранилища поддерживает хранение файлов контента собраний. Необходимо развернуть Exchange перед развертыванием и включением архивации с помощью варианта Exchange Microsoft. 
    
Если вы решите использовать Exchange, вам не нужно развертывать отдельные SQL Server базы данных для архивирования, если у вас нет Skype для бизнеса пользователей, которые не размещены на Exchange серверах. При развертывании архивации с помощью Exchange microsoft Skype для бизнеса архивные данные хранятся с Exchange данных соответствия требованиям только для пользователей, размещающихся на Exchange серверах. 
  
## <a name="hardware-and-software-requirements-for-archiving-in-skype-for-business-server-2015"></a>Требования к оборудованию и программному обеспечению для архива в Skype для бизнеса Server 2015 г.
  
Архивативка не является определенной ролью сервера, вам не нужно устанавливать отдельный сервер для архива. Унифицированные агенты по сбору данных устанавливаются и активируются автоматически на каждом выпуск Enterprise переднем выпуск Standard Server. Вам потребуется включить и опубликовать топологию архива с помощью Topology Builder.
    
Архивативная система использует Skype для бизнеса Server для временного хранения файлов контента собраний, поэтому не нужно настроить отдельный хранилище файлов для архивирования.
    
Очередь сообщений Майкрософт не требуется.
    
Необходимо настроить инфраструктуру для хранения архива. Это включает выбор хранилища Exchange или архивирования с помощью SQL Server.   Skype для бизнеса Server Требования к инфраструктуре архивации такие же, как и для развертывания Skype для бизнеса Server. Подробные сведения см. [в материале Requirements for your Skype для бизнеса среды.](../../plan-your-deployment/requirements-for-your-environment/requirements-for-your-environment.md) 
  
> [!NOTE]
> Чтобы поддерживать пользователей, не размещающихся на Exchange серверах, или если вы не хотите использовать вариант интеграции Microsoft Exchange, необходимо развернуть хранилище архивации с помощью 64-SQL Server базы данных. 
    
Перед развертыванием и включением архива необходимо настроить SQL Server платформы. Если у учетной записи, которая будет использоваться для публикации топологии, есть необходимые права и разрешения администратора, вы можете создать базу данных архивации (LcsLog) при публикации топологии. Вы также можете создать базу данных позже, включив ее в процедуру установки. Сведения о SQL Server см. в SQL Server [документации.](/sql/sql-server/)
    
Увеличение нагрузки для архива может быть значительным. Поэтому необходимо убедиться, что пространство диска является достаточным для серверов переднего конца, на которых включена архивативка.

 **Последние мысли:**
  
Пожалуйста, не устанавливайте программное обеспечение клиента Microsoft Internet Security and Acceleration (ISA) Server или любого другого программного обеспечения для поставщиков многоуровневой службы Winsock (LSP) (любые сторонние брандмауэры или программное обеспечение проверки антивирусной сети будут включены здесь) на любой из ваших серверов переднего или автономных серверов-посредников. При установке этого программного обеспечения была замечена низкая производительность трафика мультимедиа.
