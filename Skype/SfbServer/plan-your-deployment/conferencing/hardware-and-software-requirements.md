# <a name="hardware-and-software-requirements-for-conferencing-in-skype-for-business-server"></a>Требования к оборудованию и программному обеспечению для Skype для бизнеса Server

**Сводка:** Ознакомьтесь с этой темой, чтобы узнать о требованиях к оборудованию и программному обеспечению для Skype для бизнеса Server.

В этом разделе описываются требования к оборудованию и программному обеспечению для веб-конференций, аудио- и видеоконференций, телефонных телефонных и мгновенных сообщений. Все возможности для конференциинга работают на серверах переднего входа; существуют дополнительные требования к различным типам конференций, как показано на следующей схеме.

Например, если вы хотите разрешить телефонные переговоры с телефонным подключением, вам потребуется развернуть сервер-посредник и шлюз для подключения к открытой телефонной сети с переключениями (PSTN). Если вы хотите разрешить веб-конференцию, необходимо убедиться, что Skype для бизнеса Server можно подключиться к серверу Office веб-приложений. Если вы хотите разрешить внешним пользователям участвовать в конференциях, необходимо развернуть edge Server.

## <a name="hardware-and-software-requirements-for-front-end-servers"></a>Требования к оборудованию и программному обеспечению для серверов переднего конца

Поскольку веб-conferencing, A/V conferencing, dial-in conferencing, and IM conferencing are all collocated with the Front End Server, the server hardware and software requirements are the same as for the Front End Servers. Сведения об этих требованиях см. в материале [Server requirements for Skype для бизнеса Server 2015](../../plan-your-deployment/requirements-for-your-environment/server-requirements.md) и [Environmental requirements for Skype для бизнеса Server 2015](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или [Server requirements for Skype для бизнеса Server 2019](../../../SfBServer2019/plan/system-requirements.md).

## <a name="requirements-for-web-conferencing"></a>Требования к веб-конференциям

Если принято решение включить функции веб-конференций, должны быть спланированы следующие компоненты:

- Доступ к хранилищу файлов, которое используется для хранения содержимого веб-конференций.

- Интеграция Office сервера веб-приложений, которая необходима для PowerPoint файлов во время конференции.

### <a name="file-store"></a>Хранилище файлов

Служба Skype для бизнеса Server веб-собраний хранит контент, общий во время собраний в хранилище файлов. В рамках развертывания необходимо указать файловый файл, который будет использоваться в качестве хранилище файлов для выпуск Standard сервера или выпуск Enterprise переднего конечного пула. Можно использовать существующую общую папку файлов для файлового хранилища или определить новую общую папку файлов, указав полное доменное имя файлового сервера, на котором будет располагаться общая папка файлов, и имя этой папки. Дополнительные сведения см. в раздел Создание доли файлов [в Skype для бизнеса Server](../../deploy/install/create-a-file-share.md). Перед сохранением содержимого в хранилище файлов содержимое шифруется службой веб-конференций.

Skype для бизнеса Server поддерживает использование файловых акций в прямом присоединенном хранилище (DAS) или сети области хранения (SAN), включая распределенную файловую систему (DFS), а также на избыточном массиве независимых дисков (RAID) для файловых магазинов. После того Skype для бизнеса Server как мастер развертывания определил расположение файла, Skype для бизнеса Server создает структуру папок в файле, аналогичную:

- 1-ApplicationServer-1

- 1-CentralMgmt-1

- 1-WebServices-1

  - CollabContent

  - CollabMetadata

  - DataConf

После этого содержимое, такое как слайды PowerPoint, доски, опросы и вложения, сохраняется службой веб-конференций в папках "CollabContent" и "CollabMetadata", расположенных в папке "WebServices".

### <a name="office-web-apps-server"></a>Сервер Office Web Apps

Чтобы использовать возможности веб-Office, необходимо установить сервер Skype для бизнеса Server и настроить Skype для бизнеса Server для связи с Office web Apps Server.

Office Сервер веб-приложений должен быть установлен на отдельном компьютере, который не работает Skype для бизнеса Server, SQL Server или любом другом серверном приложении. (На этом компьютере не должна Office установлена версия Office.) На любом компьютере, Office сервере веб-приложений, также должен быть установлен определенный набор программного обеспечения (в том числе платформа .NET Framework 4.5 и Windows PowerShell 3.0). Эти требования, а также сведения о настройке сертификатов и службы IIS (IIS), подробно обсуждаются на веб-сайте Microsoft Office развертывания [веб-приложений.](/webappsserver/deploy-the-infrastructure-office-web-apps-server)

Сведения о настройке Skype для бизнеса Server для работы с сервером Office приложений см. в странице Настройка интеграции с [сервером Office](../../deploy/deploy-conferencing/office-web-app-server.md)приложений в Skype для бизнеса Server .

## <a name="requirements-for-audio-and-video-conferencing"></a>Требования к аудио- и видеоконференциям

При планировании аудио- и видеоконференций следует учитывать пропускную способность сети, необходимую для типа медиаданных конференций, требуемых для организации. К этим медиаданным может относиться аудио, видео и панорамное видео. Без достаточной пропускной способности сети взаимодействие пользователей может значительно ухудшиться.

Сведения о планировании емкости аудио- и видеоконференции см. в таблице [Plan network requirements for Skype для бизнеса.](../../plan-your-deployment/network-requirements/network-requirements.md)

Вы можете использовать управление приемом вызовов (CAC) для управления пропускной способностью сети, используемой для A/V-конференций. Это особенно важно для сетей служебного пользования, таких как каналы с ограниченной пропускной способностью между центральным сайтом и сайтами филиалов. Подробные сведения см. [в материале Plan for call admission control in Skype для бизнеса Server.](../../plan-your-deployment/enterprise-voice-solution/call-admission-control.md)

Если в сети развертываются аудиоконференции, пользователям понадобятся аудиоустройства, например гарнитуры, для участия в аудиоконференциях. Если развертываются видеоконференции, пользователям понадобятся видеоустройства, например веб-камеры. Для аудио- и видео устройств развертывание устройств и обучение пользователей являются важными шагами для рассмотрения. Дополнительные сведения см. в [проекте Plan for clients and devices.](../../plan-your-deployment/clients-and-devices/clients-and-devices.md) Корпорация Майкрософт рекомендует использовать устройства единой связи(UC), сертифицированные Корпорацией Майкрософт для всех типов устройств, для обеспечения оптимального взаимодействия с пользователями. Сведения о устройствах, сертифицированных на UC, см. в материале Телефоны и [устройства для Skype для бизнеса.](../../../SfbPartnerCertification/certification/devices-ip-phones.md)

## <a name="requirements-for-dial-in-conferencing"></a>Требования для телефонных конференций

Диалоговое участие является необязательной функцией рабочей нагрузки Skype для бизнеса Server, которая включает в себя различные компоненты. Некоторые компоненты специфичнее для телефонных конференций, а некоторые Корпоративная голосовая связь. В этом разделе описываются требования к компонентам, необходимым для телефонных конференций. Дополнительные сведения о требованиях к серверу-посреднику и шлюзам для общедоступных телефонных сетей (PSTN) см. в Skype для бизнеса Server и развертывании сервера-посредника в [topology Builder](../../deploy/deploy-enterprise-voice/deploy-a-mediation-server.md)в Skype для бизнеса Server . [](../../plan-your-deployment/enterprise-voice-solution/mediation-server.md)

### <a name="required-components"></a>Необходимые компоненты

Чтобы настроить диалоговое телефонное Skype для бизнеса Server, необходимо установить следующие компоненты:

- Unified Communications Application Service (UCAS) (называемые службой приложения);

- приложение "Помощник по конференц-связи";

- приложение "Объявления для конференц-связи";

- веб-страница "Параметры конференц-связи с телефонным подключением";

- По крайней мере один сервер-посредник и по крайней мере один шлюз PSTN

Для телефонных конференций служба приложений, приложение и приложение имеют те же требования к операционной системе, что и к front End Servers. Подробные сведения [см. в материале Server requirements for Skype для бизнеса Server 2015](../../plan-your-deployment/requirements-for-your-environment/server-requirements.md).

приложение и приложение для Windows формата мультимедиа устанавливается на серверы переднего входа. Windows Время запуска формата мультимедиа требуется для воспроизведения Windows мультимедиа аудио (WMA), используемых для музыки в удержании, записанных имен и подсказок. Если вы устанавливаете на Windows Server 2012 или Windows Server 2012 R2 (что мы рекомендуем), вам потребуется установить Microsoft Media Foundation, чтобы получить Windows медиаформат. Если вы устанавливаете на любую версию Windows Server до Windows 2012 г., необходимо убедиться, что Windows Desktop Experience установлен для получения Windows формата мультимедиа.

### <a name="audio-file-requirements-for-dial-in-conferencing"></a>Требования к аудиофайлам для телефонных конференций

Skype для бизнеса Server не поддерживает настройку голосовых подсказок и музыки для телефонных конференций. Однако, если у вас есть сильная бизнес-потребность, требуемая для изменения аудиофайлов по умолчанию, см. статью Microsoft Knowledge Base 961177, How [to customize voice prompts or music files for dial-in audio conferencing](https://go.microsoft.com/fwlink/p/?linkid=3052&amp;kbid=961177).

Вы также можете использовать утилиту управления пользовательскими голосами Microsoft [Lync Server помощник по конференц-связи,](https://go.microsoft.com/fwlink/p/?LinkId=396880) которая позволяет администраторам заменять голосовые запросы по умолчанию, используемые, когда телефонный Skype для бизнеса присоединяется к собранию с пользовательскими подсказками для предоставления другого доступа к собранию. Настраиваемые голосовые запросы можно установить на Enterprise или выпуск Standard сервере.

приложение и приложение имеют следующие требования к музыке на удержание, записанное имя и аудиофайлы:

- Формат файлов WMA (Windows Media Audio)

- Моно 16 бит

- 48 Кбит/с, 2 прохода, CBR (постоянная скорость)

- Уровень речи на -24 дБ

### <a name="user-requirements-for-dial-in-conferencing"></a>Требования пользователей для телефонных конференций

Учетным записям пользователей конференц-связи с телефонным подключением должны быть уникальные номера телефонов или добавочные номера. Благодаря этому требованию становится возможным выполнять проверку подлинности во время телефонного подключения. Enterprise (то есть пользователи с учетными данными Active Directory Domain Services и учетными записями Skype для бизнеса Server в вашей организации) введите свой номер телефона (или расширение) и личный идентификационный номер (ПИН-код), чтобы войти на конференции в качестве пользователя с проверкой подлинности.
