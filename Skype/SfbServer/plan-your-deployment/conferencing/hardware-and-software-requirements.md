# <a name="hardware-and-software-requirements-for-conferencing-in-skype-for-business-server"></a>Требования к оборудованию и программному обеспечению для видеоконференций в Skype для бизнеса Server

**Сводка:** В этом разделе вы узнаете о требованиях к оборудованию и программному обеспечению для связи в Skype для бизнеса Server.

В этом разделе описываются требования к оборудованию и программному обеспечению для веб-аудио- и видеоконференций, аудио- и видеоконференций, аудио- и видеоконференций, а также для аудио- и голосовой связи. Все возможности, которые можно выполнить на серверах переднего сервера; существуют дополнительные требования для различных типов услуг, как показано на следующей схеме.

Например, если вы хотите разрешить телефонные подключения, вам потребуется развернуть сервер-посредник и шлюз для подключения к телефонной сети общего перейти на телефонную сеть (STN). Если вы хотите разрешить веб-трансляции, необходимо убедиться, что Skype для бизнеса Server может подключаться к серверу Office Web Apps. Если вы хотите разрешить внешним пользователям участвовать в конференциях, необходимо развернуть edge Server.

## <a name="hardware-and-software-requirements-for-front-end-servers"></a>Требования к оборудованию и программному обеспечению для серверов переднего сервера

## <a name="requirements-for-web-conferencing"></a>Требования к веб-трансляциям

Если принято решение включить функции веб-конференций, должны быть спланированы следующие компоненты:

- Доступ к хранилищу файлов, которое используется для хранения содержимого веб-конференций.

- Интеграция с сервером Office Web Apps, которая необходима для обмена файлами PowerPoint во время конференции.

### <a name="file-store"></a>Хранилище файлов

Служба веб-конференций Skype для бизнеса Server хранит общий контент во время собраний в хранилище файлов. В рамках развертывания необходимо указать обную папку файлов, которая будет использоваться в качестве файлового хранения для сервера Standard Edition или пула переднего сервера Enterprise Edition. Можно использовать существующую общую папку файлов для файлового хранилища или определить новую общую папку файлов, указав полное доменное имя файлового сервера, на котором будет располагаться общая папка файлов, и имя этой папки. Дополнительные сведения см. в [теме "Создание файловой папки в Skype для бизнеса Server".](../../deploy/install/create-a-file-share.md) Перед сохранением содержимого в хранилище файлов содержимое шифруется службой веб-конференций.

Skype для бизнеса Server поддерживает использование файловых файлов в непосредственно подключенном хранилище (DAS) или сети хранения данных (SAN), включая распределенную файловую систему (DFS), а также в избыточном массиве независимых дисков (RAID) для хранилищ файлов. После того как мастер развертывания Skype для бизнеса Server определит расположение файловой папки, Skype для бизнеса Server создаст в этой папке структуру папок, аналогичную:

- 1-ApplicationServer-1

- 1-CentralMgmt-1

- 1-WebServices-1

  - CollabContent

  - CollabMetadata

  - DataConf

После этого содержимое, такое как слайды PowerPoint, доски, опросы и вложения, сохраняется службой веб-конференций в папках "CollabContent" и "CollabMetadata", расположенных в папке "WebServices".

### <a name="office-web-apps-server"></a>Сервер Office Web Apps

Чтобы использовать возможности веб-конференций, необходимо установить Сервер Office Web Apps и настроить Skype для бизнеса Server для связи с сервером Office Web Apps.

Сервер Office Web Apps должен быть установлен на отдельном компьютере, на компьютере без Skype для бизнеса Server, SQL Server или любого другого серверного приложения. (На этом компьютере не должна быть установлена ни одна версия Office.) На любом компьютере, используемом для запуска сервера Office Web Apps, также должен быть установлен определенный набор программного обеспечения (включая .NET Framework 4.5 и Windows PowerShell 3.0). Эти требования, а также сведения о настройке сертификатов и служб IIS подробно обсуждаются на веб-сайте [Microsoft Office Web Apps.](https://go.microsoft.com/fwlink/p/?linkid=257525)

Сведения о настройке Skype для бизнеса Server для работы с сервером Office Web Apps см. в сведениях о настройке интеграции с сервером Office Web Apps в [Skype для бизнеса Server.](../../deploy/deploy-conferencing/office-web-app-server.md)

## <a name="requirements-for-audio-and-video-conferencing"></a>Требования к аудио- и видеоконференциям

При планировании аудио- и видеоконференций следует учитывать пропускную способность сети, необходимую для типа медиаданных конференций, требуемых для организации. К этим медиаданным может относиться аудио, видео и панорамное видео. Без достаточной пропускной способности сети взаимодействие пользователей может значительно ухудшиться.

Сведения о планировании мощности аудио- и видеоконференции см. в подсистеме планирования сетевых [требований для Skype для бизнеса.](../../plan-your-deployment/network-requirements/network-requirements.md)

Контроль допуска вызовов (CAC) можно использовать для управления пропускной способностью сети, используемой для видео- и видеоконференций. Это особенно важно для сетей служебного пользования, таких как каналы с ограниченной пропускной способностью между центральным сайтом и сайтами филиалов. For details, see [Plan for call admission control in Skype for Business Server.](../../plan-your-deployment/enterprise-voice-solution/call-admission-control.md)

Если в сети развертываются аудиоконференции, пользователям понадобятся аудиоустройства, например гарнитуры, для участия в аудиоконференциях. Если развертываются видеоконференции, пользователям понадобятся видеоустройства, например веб-камеры. Для аудио- и видео устройств развертывание устройств и обучение пользователей являются важными этапами, которые следует учитывать. Дополнительные сведения [см. в сведениях о планировании клиентов и устройств.](../../plan-your-deployment/clients-and-devices/clients-and-devices.md) Корпорация Майкрософт рекомендует использовать устройства объединенных коммуникаций( UC), сертифицированные корпорацией Майкрософт для всех типов устройств, чтобы обеспечить оптимальное взаимодействие с пользователем. For details about UC-certified devices, see [Phones and devices for Skype for Business](https://go.microsoft.com/fwlink/?LinkId=619916).

## <a name="requirements-for-dial-in-conferencing"></a>Требования для телефонной комконференции

Эта функция является необязательной функцией рабочей нагрузки, включаемой в себя различные компоненты. Некоторые компоненты являются специфическими для телефонной Корпоративная голосовая связь. В этом разделе описываются требования к компонентам, необходимым для телефонной связи. Дополнительные сведения о требованиях к серверу-посреднику и шлюзу телефонной сети общего звонков (PSTN) см. в компоненте сервера-посредника [в Skype](../../plan-your-deployment/enterprise-voice-solution/mediation-server.md) для бизнеса Server и развертывании сервера-посредника в построителе топологий в Skype для [бизнеса Server.](../../deploy/deploy-enterprise-voice/deploy-a-mediation-server.md)

### <a name="required-components"></a>Необходимые компоненты

Перед настройкой компоненты Skype для бизнеса Server необходимо установить следующие компоненты:

- Unified Communications Application Service (UCAS) (называемые службой приложения);

- приложение "Помощник по конференц-связи";

- приложение "Объявления для конференц-связи";

- веб-страница "Параметры конференц-связи с телефонным подключением";

- По крайней мере один сервер-посредник и хотя бы один шлюз STN

Требования к операционной системе для приложений, службы приложений, помощника по конференциам и приложения "Извещание для телефонной линии" такие же, как и у серверов переднего сервера. Подробные сведения см. в требованиях к [серверу для Skype для бизнеса Server 2015.](../../plan-your-deployment/requirements-for-your-environment/server-requirements.md)

Для работы приложения помощника по конференциам и приложения "Извещание для проведения веб-службы" требуется, чтобы на серверах переднего сервера была установлена windows Media Format Runtime. Windows Media Format Runtime требуется для воспроизведения звуковых файлов Windows Media (WMA), используемых для воспроизведения музыки на удержании, записанных имен и подсказок. При установке в Windows Server 2012 или Windows Server 2012 R2 (рекомендуется), необходимо установить Microsoft Media Foundation для получения windows Media Format Runtime. При установке на любую версию Windows Server до Windows 2012 необходимо убедиться, что для получения windows Media Format Runtime установлено приложение "Windows Desktop Experience".

### <a name="audio-file-requirements-for-dial-in-conferencing"></a>Требования к аудиофайлам для аудиоконференций

Skype для бизнеса Server не поддерживает настройку голосовых подсказок и музыки для телефонной сети. Тем не менее, если у вас есть потребность в сильной [бизнес-потребности,](https://go.microsoft.com/fwlink/p/?linkid=3052&amp;kbid=961177)требующий изменения звуковых файлов по умолчанию, см. статью 961177 базы знаний Майкрософт о настройке голосовых подсказок или музыкальных файлов для аудиоконференций с телефонным номером.

Вы также можете использовать с помощью специальной программы управления голосовых подсказок помощника по microsoft [Lync Server,](https://go.microsoft.com/fwlink/p/?LinkId=396880) которая позволяет администраторам заменять голосовые подсказки по умолчанию, используемые, когда звоняющий присоединяется к собранию Skype для бизнеса, настраиваемые подсказки для предоставления другого доступа к собранию. Настраиваемые голосовые подсказки можно установить на сервере Enterprise или Standard Edition.

К приложению помощника по аудиоконференциям и приложению "Извещание для аудиоконференций" предъявляются следующие требования:

- Формат файлов WMA (Windows Media Audio)

- Моно 16 бит

- 48 Кбит/с, 2 прохода, CBR (постоянная скорость)

- Уровень речи на -24 дБ

### <a name="user-requirements-for-dial-in-conferencing"></a>Требования пользователей для телефонной комконференции

Учетным записям пользователей конференц-связи с телефонным подключением должны быть уникальные номера телефонов или добавочные номера. Благодаря этому требованию становится возможным выполнять проверку подлинности во время телефонного подключения. Корпоративные пользователи (то есть пользователи, которые имеют учетные данные доменных служб Active Directory и учетные записи Skype для бизнеса Server в организации) введите свой телефонный номер (или расширение) и персональный идентификационный номер (ПИН-код) для телефонного звонка в конференции в качестве пользователя, подлинность которого прошла проверку подлинности.
