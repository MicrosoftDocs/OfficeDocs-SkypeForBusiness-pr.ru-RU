# <a name="hardware-and-software-requirements-for-conferencing-in-skype-for-business-server"></a>Требования к оборудованию и программному обеспечению для Конференции в Skype для бизнеса Server

**Сводка:** В этой статье приведены сведения о требованиях к оборудованию и программному обеспечению для проведения конференций в Skype для бизнеса Server.

В этом разделе рассматриваются требования к аппаратному и программному обеспечению веб-конференций, видео- и голосовой (A/V) конференц-связи, конференц-связи с телефонным подключением и конференц-связи с обменом мгновенными сообщениями (IM). Все функциональные возможности конференц-связи реализуются на серверах переднего плана; требования зависят от типа конференц-связи, как показано на следующей схеме.

Например, если вы хотите разрешить Конференц-связь с телефонным подключением, вам потребуется развернуть сервер-посредник и шлюз для подключения к телефонной сети общего пользования (КТСОП). Если вы хотите разрешить веб-конференции, необходимо убедиться, что Skype для бизнеса Server может подключаться к серверу Office Web Apps. Если требуется разрешить внешним пользователям участие в конференциях, необходимо развернуть пограничный сервер.

## <a name="hardware-and-software-requirements-for-front-end-servers"></a>Требования к оборудованию и программному обеспечению для серверов переднего плана

Поскольку веб-конференции, Конференции, конференции с телефонным подключением и конференции для обмена мгновенными сообщениями размещаются вместе с сервером переднего плана, требования к оборудованию и программному обеспечению сервера совпадают с серверами переднего плана. Подробнее об этих требованиях можно узнать в статьях [серверные требования для Skype для бизнеса server 2015](../../plan-your-deployment/requirements-for-your-environment/server-requirements.md) и [требования к рабочей среде для](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) skype для бизнеса Server 2015 или [серверных требований для Skype для бизнеса Server 2019](../../../SfBServer2019/plan/system-requirements.md).

## <a name="requirements-for-web-conferencing"></a>Требования для проведения веб-конференций

Если принято решение включить функции веб-конференций, должны быть спланированы следующие компоненты:

- Доступ к хранилищу файлов, которое используется для хранения содержимого веб-конференций.

- Взаимодействие с сервером с Office Web Apps, необходимое для предоставления доступа к файлам PowerPoint во время конференции.

### <a name="file-store"></a>файлов

Служба веб-конференций в Skype для бизнеса Server хранит содержимое, к которому предоставлен общий доступ во время собраний в хранилище файлов. В рамках развертывания необходимо указать общую файловую группу, которая будет использоваться в качестве хранилища файлов для пула стандартных интерфейсов выпуска Standard Edition Server или Enterprise Edition. Можно назначить хранилищем файлов существующий общий файловый ресурс или задать новый общий файловый ресурс, указав полное доменное имя файлового сервера, на котором требуется его расположить, и имя папки для нового файлового ресурса. Дополнительные сведения можно найти [в разделе Создание общего файлового файла в Skype для бизнеса Server](../../deploy/install/create-a-file-share.md). Перед сохранением материалов в хранилище файлов службы веб-конференций эти материалы шифруются.

Skype для бизнеса Server поддерживает использование общих файловых ресурсов на запоминающем устройстве (DAS) или сети хранения данных (SAN), включая распределенную файловую систему (DFS) и на избыточном массиве независимых дисков (RAID) для хранения файлов. После того как мастер развертывания в Skype для бизнеса Server заопределил расположение общего файлового ресурса, в Skype для бизнеса Server будет создана структура папок в файле общего доступа, как показано ниже.

- 1-ApplicationServer-1

- 1-CentralMgmt-1

- 1-WebServices-1

  - CollabContent

  - CollabMetadata

  - DataConf

После этого содержимое, такое как слайды PowerPoint, доски, опросы и вложения, сохраняется службой веб-конференций в папках "CollabContent" и "CollabMetadata", расположенных в папке "WebServices".

### <a name="office-web-apps-server"></a>Сервер Office Web Apps

Чтобы использовать возможности веб-конференций, необходимо установить Office Web Apps Server и настроить Skype для бизнеса Server для взаимодействия с сервером Office Web Apps.

Сервер Office Web Apps должен быть установлен на отдельном компьютере, на котором не запущены Skype для бизнеса Server, SQL Server и другие серверные приложения. (На компьютере не должна быть установлена какая-либо версия Office.) На любом компьютере, который использовался для запуска Office Web Apps, должен быть установлен определенный набор программного обеспечения (включая .NET Framework 4,5 и Windows PowerShell 3,0). Эти требования, а также сведения о настройке сертификатов и информационных служб Интернета (IIS) подробно описаны на [веб-сайте развертывания Microsoft Office Web Apps](https://go.microsoft.com/fwlink/p/?linkid=257525).

Сведения о том, как настроить Skype для бизнеса Server для работы с Office Web Apps Server, можно найти [в разделе Настройка интеграции с Office Web Apps Server в Skype для бизнеса Server](../../deploy/deploy-conferencing/office-web-app-server.md).

## <a name="requirements-for-audio-and-video-conferencing"></a>Требования в видео- и голосовой конференц-связи

При планировании видео- и голосовой конференц-связи следует понимать, какие мультимедийные данные требуется передавать при конференц-связи в организации и какая полоса пропускания сети необходима для этого типа мультимедийных данных. Возможные типы – звук, видеоизображение и панорамное видеоизображение. Недостаточная полоса пропускания сети может существенно ухудшить взаимодействие в пользователями.

О планировании пропускной способности видео- и голосовой связи для конференций см. в разделе [Plan network requirements for Skype for Business](../../plan-your-deployment/network-requirements/network-requirements.md).

Вы можете использовать управление допуском звонков (CAC), чтобы управлять пропускной способностью сети, используемой в конференц-связи/V. Это важно для сетей с ограниченным доступом, например, для ссылок с ограниченной пропускной способностью между центральным и филиалным сайтами. Подробнее смотрите в разделе [Планирование управления допуском звонков в Skype для бизнеса Server](../../plan-your-deployment/enterprise-voice-solution/call-admission-control.md).

При развертывании звуковых конференций в сети пользователям понадобятся звуковые устройства, например гарнитуры, которые будут участвовать в голосовой конференции. Если вы развертываете видеоконференции, вам нужно развернуть видеоустройства, например веб-камеру для пользователей. Для звуковых и видеоустройств необходимо учитывать важные действия по развертыванию устройств и обучение пользователей. Дополнительные сведения можно найти в разделе [Планирование клиентов и устройств](../../plan-your-deployment/clients-and-devices/clients-and-devices.md). Корпорация Майкрософт рекомендует использовать устройства UC, сертифицированные корпорацией Майкрософт для всех типов устройств, для обеспечения оптимального взаимодействия с пользователем. Дополнительные сведения об устройствах, сертифицированных в UC, можно найти в разделе [телефоны и устройства в Skype для бизнеса](https://go.microsoft.com/fwlink/?LinkId=619916).

## <a name="requirements-for-dial-in-conferencing"></a>Требования для конференц-связи с телефонным подключением

Конференц-связь с телефонным подключением — это необязательная функция рабочей нагрузки Skype для бизнеса Server, включающая различные компоненты. Некоторые компоненты относятся к конференц-связи с телефонным подключением, а некоторые – к корпоративным голосовым компонентам. В этом разделе описаны требования для компонентов, необходимых для конференц-связи с телефонным подключением. Подробные сведения о требованиях к шлюзам и требованиям сети PSTN можно найти в разделе сервер- [посредника в Skype для бизнеса Server](../../plan-your-deployment/enterprise-voice-solution/mediation-server.md) и [развернуть сервер-посредника в построителе топологии в Skype для бизнеса Server](../../deploy/deploy-enterprise-voice/deploy-a-mediation-server.md).

### <a name="required-components"></a>Обязательные компоненты

Для настройки конференц-связи с телефонным подключением вам потребуется установить следующие компоненты сервера Skype для бизнеса.

- Служба приложения объединенных коммуникаций (UCAS) (называемая просто службой приложения)

- помощника по конференц-связи

- приложение оповещения для конференц-связи;

- веб-страница "Параметры конференц-связи с телефонным подключением";

- хотя бы один сервер-посредник и хотя бы один шлюз ТСОП.

Для конференц-связи с телефонным подключением службы приложений, приложения для участников Конференции и объявления конференций имеют те же требования к операционной системе, что и серверы переднего плана. Подробности можно найти в разделе [требования к серверу для Skype для бизнеса Server 2015](../../plan-your-deployment/requirements-for-your-environment/server-requirements.md).

Приложение для рекламы в конференц-связи с приложениями и конференц-связь требуется, чтобы на серверах переднего плана была установлена среда выполнения формата Windows Media Format. Это необходимо для воспроизведения звуковых файлов Windows Media (WMA), содержащих музыку для режима удержания, записанные имена и голосовые приглашения. Если вы устанавливаете Windows Server 2012 или Windows Server 2012 R2 (что мы рекомендуем), вам потребуется установить Microsoft Media Foundation, чтобы получить доступ к формату среды выполнения Windows Media Format. При установке на любой версии Windows Server, более ранней по отношению к Windows 2012, для получения Windows Media Format Runtime потребуется установить программное обеспечение "Возможности рабочего стола Windows".

### <a name="audio-file-requirements-for-dial-in-conferencing"></a>Требования к звуковым файлам для конференц-связи с телефонным подключением

В Skype для бизнеса Server не поддерживается настройка голосовых запросов и музыки для конференц-связи с телефонным подключением. Тем не менее, если вам необходимо изменить аудиофайлы по умолчанию, ознакомьтесь со статьей Microsoft Knowledge Base 961177, [Настройка голосовых запросов и музыкальных файлов для голосовой конференции](https://go.microsoft.com/fwlink/p/?linkid=3052&amp;kbid=961177)с телефонным подключением.

Вы также можете использовать служебную программу помощника по [конференциям Microsoft Lync Server](https://go.microsoft.com/fwlink/p/?LinkId=396880) , которая позволяет администраторам заменить голосовые запросы по умолчанию, используемые при присоединении абонентом к собранию Skype для бизнеса с настраиваемыми запросами. для предоставления другого опыта ввода собрания. Настраиваемые голосовые приглашения можно установить на сервере корпоративного или стандартного выпуска.

Участие в программе для конференц-связи с приложениями и конференциями для музыки на удержании, записанных именах и файлах звуковых файлов не имеет приведенных ниже требований.

- Формат файлов WMA (Windows Media Audio)

- Моно 16 бит

- 48 Кбит/с, 2 прохода, CBR (постоянная скорость)

- Уровень речи на -24 дБ

### <a name="user-requirements-for-dial-in-conferencing"></a>Требования к пользователям для конференц-связи с телефонным подключением

Учетным записям пользователей конференц-связи с телефонным подключением должны быть назначены уникальные номера телефонов или добавочные номера. Это необходимо для проверки подлинности во время конференц-связи с телефонным подключением. Корпоративные пользователи (то есть пользователи, у которых есть учетные данные доменных служб Active Directory и учетные записи Skype для бизнеса сервера в вашей организации) вводят номер телефона (или расширение) и персональный идентификационный номер (ПИН-код) для подключения к конференциям как пользователь, прошедший проверку подлинности.
