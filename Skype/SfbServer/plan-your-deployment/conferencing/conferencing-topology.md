# <a name="plan-your-conferencing-topology-for-skype-for-business-server"></a>Планирование топологии конференц-связи в Skype для бизнеса Server
 
**Сводка:** В этом разделе рассказывается о планировании топологии конференций в Skype для бизнеса Server.
  
В этом разделе описываются основы топологии для конференц-связи в Skype для бизнеса Server:
  
- Поддерживаемые топологии
    
- Рекомендации по конференц-связи с телефонным подключением
    
- Рекомендации по использованию веб-конференций
    
- Требования к большим собраниям
    
Дополнительные сведения о требованиях к оборудованию и программному обеспечению приведены [в статье требования к оборудованию и программному обеспечению для конференций в Skype для бизнеса Server](hardware-and-software-requirements.md).
  
## <a name="supported-topologies"></a>Поддерживаемые топологии

В Skype для бизнеса Server сервер, на котором запущены службы конференций, всегда размещается вместе с серверами переднего плана или сервера Standard Edition. При развертывании Skype для бизнеса Server возможности конференций для обмена мгновенными сообщениями развертываются автоматически. Вы можете указать, следует ли развертывать веб-, аудио-и видеоконференции (A/V) и конференц-связь с телефонным подключением с помощью построителя топологий. Вы также можете использовать построитель топологий для добавления конференций к существующему развертыванию. Для получения дополнительных сведений об основах и сценариях совместного использования топологии обратитесь к разделу [основы топологии для Skype для бизнеса Server](../../plan-your-deployment/topology-basics/topology-basics.md).
  
Конференц-связь можно развернуть в следующих топологиях и конфигурациях:
  
- Skype для бизнеса Server Standard Edition
    
- Skype для бизнеса Server Enterprise Edition
    
- С корпоративной голосовой связью или без нее
    
## <a name="dial-in-conferencing-considerations"></a>Рекомендации по конференц-связи с телефонным подключением

При развертывании конференц-связи с телефонным подключением необходимо учитывать следующие возможности.
  
- Для конференц-связи с телефонным подключением требуется сервер-посредник для перевода сигналов (и носителей в некоторых конфигурациях) между Skype для бизнеса Server и шлюзом PSTN, а шлюз PSTN для перевода сигналов и мультимедиа между сервером-посредником и шлюзом PSTN .
    
   Перед настройкой конференц-связи с телефонным подключением необходимо развернуть либо корпоративную голосовую связь, либо сервер-посредник, а также хотя бы один из следующих компонентов:
    
  - шлюз ТСОП;
    
  - IP-УАТС
    
  - пограничный контроллер сеансов (SBC) для поставщика услуг Интернет-телефонии, подключение к которому выполняется путем настройки канала SIP.
    
- Вы можете развернуть службу приложения, приложение-помощник по конференц-связи и приложение извещения о конференции на центральном сайте, но не на сайте филиала.
    
- Конференц-связь с телефонным подключением необходимо развернуть в каждом пуле, где развертывается конференция Skype для бизнеса Server. Назначать номера доступа в каждом пуле не обязательно, но функция конференц-связи с телефонным подключением должна быть развернута в каждом пуле. Это требование поддерживает функцию записи имени, когда пользователь вызывает номер доступа из одного пула для присоединения к Конференции Skype для бизнеса Server в другом пуле. 
    
Для получения дополнительных сведений ознакомьтесь со статьей [планирование конференц-связи с телефонным подключением в Skype для бизнеса Server](dial-in-conferencing.md).
  
## <a name="web-conferencing-considerations"></a>Рекомендации по использованию веб-конференций

Для веб-конференций необходимы следующие компоненты: 
  
- Доступ к хранилищу файлов, которое используется для хранения содержимого веб-конференций.
    
- Интеграция с сервером Office Web Apps (Office Online Server), которая необходима для предоставления общего доступа к файлам PowerPoint во время конференции.
    
> [!NOTE]
> Последняя итерация сервер Office Web Apps называется Office Online Server, поддерживаемой Skype для бизнеса Server. Более подробную информацию можно найти в [документации по Office Online Server](https://technet.microsoft.com/library/jj219456%28v=office.16%29.aspx). 
  
Skype для бизнеса Server позволяет настроить сервер Office Web Apps или Office Online Server следующим образом. В зависимости от имеющихся потребностей вам доступны следующие варианты:
  
- **Установите как Skype для бизнеса Server, так и сервер Office Web Apps, а также локальный сервер Office Web Apps, за брандмауэром организации и в той же сетевой зоне.** В этой топологии внешний доступ к серверу Office Web Apps или Office Online Server будет предоставляться через обратный прокси-сервер. В идеале необходимо установить сервер Office Web Apps или Office Online Server в той же зоне сети, что и Skype для бизнеса Server.
    
    Внешние клиенты Skype для бизнеса могут подключаться к Skype для бизнеса Server и Office Web Apps Server/Office Online Server с помощью обратного прокси-сервера, который представляет собой сервер, который принимает запросы из Интернета и пересылает их во внутреннюю сеть. (Внутренним клиентам не требуется использовать обратный прокси-сервер, так как они могут подключаться к серверу Office Web Apps или Office Online Server напрямую.) Эта топология лучше всего подходит для использования выделенной серверной фермы Office Web Apps или фермы Office Online Server, которая используется только в Skype для бизнеса Server.
    
- **Используйте внешнее развернутый сервер Office Web Apps или Office Online Server.** В этой топологии Skype для бизнеса Server развернут локально и использует сервер Office Web Apps/Office Online Server, развернутый за пределами сетевой зоны Skype для бизнеса Server. Это может произойти, когда сервер Office Web Apps или Office Online Server совместно используется несколькими приложениями в корпорации и развернут в сети, требующей от Skype для бизнеса Server использования внешнего интерфейса сервера Office Web Apps/Office Online Server и наоборот.
    
    Нет необходимости устанавливать обратный прокси-сервер; Вместо этого все запросы от сервера Office Web Apps (Office Online Server) к Skype для бизнеса Server направляются через пограничный сервер. Как внутренний, так и внешний клиенты Skype для бизнеса подключаются к серверу Office Web Apps или Office Online Server с использованием внешнего URL-адреса.
    
    Если сервер Office Web Apps или Office Online Server развернут за пределами внутреннего брандмауэра, выберите параметр **сервер Office Web Apps развернут во внешней сети** (периметр/Интернет) в построителе топологий.
    
Дополнительную информацию можно узнать [в статье Настройка интеграции с сервером Office Web Apps в Skype для бизнеса Server](../../deploy/deploy-conferencing/office-web-app-server.md). 
  
Независимо от выбранной топологии крайне важно открыть правильные порты брандмауэра. Необходимо убедиться, что DNS-имена, IP-адреса и порты не заблокированы брандмауэрами на сервере Office Web Apps/Office Online Server, подсистеме балансировки нагрузки или Skype для бизнеса Server.
  
> [!NOTE]
> Другим вариантом предоставления внешнего доступа к серверу Office Web Apps или Office Online Server является развертывание сервера в сети периметра. При выборе этого варианта следует учитывать, что сервер Office Web Apps или Office Online Server должен быть членом домена Active Directory. Если политика сети не допускает, чтобы компьютеры в сети периметра были членами домена Active Directory, не рекомендуется устанавливать сервер Office Web Apps и Office Online Server в сети периметра. Вместо этого следует установить сервер Office Web Apps или Office Online Server во внутренней сети и предоставить доступ внешним пользователям через обратный прокси-сервер. 
  
## <a name="topology-requirements-for-large-meetings"></a>Требования к топологии для больших собраний

Для одного большого собрания требуется по крайней мере один сервер переднего плана и один фоновый сервер. Тем не менее, чтобы обеспечить высокий уровень доступности, рекомендуется использовать два пула серверов переднего плана с зеркальными зеркальными серверами, как показано на следующей схеме:
  
**Топология с крупным собранием**

![Топология с крупным собранием](../../media/06858900-a262-4a47-96d0-51abd6827064.png)
  
Учетная запись пользователя, на котором размещаются крупные собрания, должна быть размещена в интерфейсном пуле. Однако не рекомендуется размещать в этом пуле учетные записи других пользователей. Используйте его только для больших собраний. Рекомендуется создать в этом пуле специальную учетную запись пользователя, используемую только для размещения больших собраний. Так как большой параметр собраний оптимизирован для производительности, использование его в качестве обычного пользователя может привести к проблемам, например, невозможности продвижения сеанса P2P на собрание, когда задействована конечная точка PSTN.
  
При управлении пулом с двумя серверами переднего плана необходимо учитывать некоторые особенности. Для получения дополнительных сведений ознакомьтесь со статьей [основы топологии для Skype для бизнеса server 2015](../../plan-your-deployment/topology-basics/topology-basics.md) и [справочных топологий Skype для бизнеса Server 2015](../../plan-your-deployment/topology-basics/reference-topologies.md).
  
Кроме того, если вы хотите дополнительно создать резервную копию аварийного восстановления и отработку отказа для пула, используемого для больших собраний, вы можете связать его с аналогичной настройкой выделенного пула в другом центре обработки данных. Подробные сведения приведены [в статье Планирование обеспечения высокой доступности и аварийного восстановления в Skype для бизнеса Server](../../plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md).
  
Дополнительные замечания о топологии
  
- Файловый ресурс необходим для хранения контента собрания и, если сервер архивации развернут и включен, для хранения файлов архива. Общая папка может быть выделена для пула или использоваться совместно с другим пулом на сайте, на котором развернут пул. Подробнее о настройке общего файлового ресурса можно узнать [в статье Создание общего файлового ресурса в Skype для бизнеса Server 2015](../../deploy/install/create-a-file-share.md).
    
- Сервер Office Web Apps или Office Online Server необходим для поддержки функциональных возможностей презентации PowerPoint в больших собраниях. Сервер Office Web Apps или Office Online Server может быть выделен для пула больших собраний или может быть одним и тем же сервером Office Web Apps или Office Online Server, используемым другими пулами на сайте, в котором развернут выделенный пул. Дополнительную информацию можно узнать [в статье Настройка интеграции с сервером Office Web Apps в Skype для бизнеса Server](../../deploy/deploy-conferencing/office-web-app-server.md). 
    
- Балансировка нагрузки на серверах переднего плана требует аппаратной балансировки нагрузки для трафика HTTP (например, для скачивания контента собраний). Балансировка нагрузки DNS рекомендуется для трафика SIP. Подробнее: [требования к балансировке нагрузки для Skype для бизнеса](../../plan-your-deployment/network-requirements/load-balancing.md). 
    
- Если вы хотите использовать сервер мониторинга для выделенного пула больших собраний, рекомендуется использовать сервер мониторинга и базу данных, общие для всех пулов серверов переднего плана в развертывании Skype для бизнеса Server. Для получения дополнительных сведений ознакомьтесь со статьей [Планирование мониторинга в Skype для бизнеса Server](../../plan-your-deployment/monitoring.md).
