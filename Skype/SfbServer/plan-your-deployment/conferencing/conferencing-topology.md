# <a name="plan-your-conferencing-topology-for-skype-for-business-server"></a>Запланируйте топологию для Skype для бизнеса Server
 
**Сводка:** Ознакомьтесь с этой темой, чтобы узнать о планировании топологии для Skype для бизнеса Server.
  
В этом разделе описаны основы топологии для конференций в Skype для бизнеса Server:
  
- Поддерживаемые топологии
    
- Вопросы, которые необходимо учитывать при диалоговом телефонном диалоге
    
- Соображения веб-конференциинга
    
- Требования к большим собраниям
    
Дополнительные сведения о требованиях к оборудованию и программному обеспечению см. в [Skype для бизнеса Server.](hardware-and-software-requirements.md)
  
## <a name="supported-topologies"></a>Поддерживаемые топологии

В Skype для бизнеса Server, сервер, на который работают службы конференций, всегда находится в расположении с серверами переднего выпуск Standard серверов. При развертывании Skype для бизнеса Server автоматически развертываются возможности для конференций im. Вы можете указать, следует ли развертывать веб-, аудио- и видеоконференцию (A/V) и диалоговое видеоконференцию с помощью конструктора топологии. Можно также использовать Topology Builder для добавления конференций в существующее развертывание. Подробные сведения об основах топологии и сценариях коллокации см. в материале [Topology Basics for Skype для бизнеса Server.](../../plan-your-deployment/topology-basics/topology-basics.md)
  
Вы можете развернуть конференцию в следующих topologies и конфигурациях:
  
- Skype для бизнеса Server выпуск Standard
    
- Skype для бизнеса Server выпуск Enterprise
    
- С Корпоративная голосовая связь
    
## <a name="dial-in-conferencing-considerations"></a>Вопросы, которые необходимо учитывать при диалоговом телефонном диалоге

При развертывании телефонных конференций необходимо учитывать следующее:
  
- Для телефонной связи требуется сервер-посредник для перевода сигналов (и мультимедиа в некоторых конфигурациях) между Skype для бизнеса Server и шлюзом PSTN, а также шлюз PSTN для перевода сигналов и мультимедиа между сервером-посредником и шлюзом PSTN.
    
   Прежде чем настраивать диалоговые конференциации, необходимо развернуть Корпоративная голосовая связь или сервер-посредник и хотя бы одно из следующих ниже:
    
  - шлюз ТСОП;
    
  - IP-PBX
    
  - пограничный контроллер сеансов (SBC) для поставщика услуг Интернет-телефонии, подключение к которому выполняется путем настройки канала SIP.
    
- Вы можете развернуть службу приложения, приложение и приложение на центральном сайте, но не на сайте филиала.
    
- В каждом пуле, в котором Skype для бизнеса Server, необходимо развернуть диалоговое конференция. Назначать номера доступа в каждом пуле не обязательно, но функция конференц-связи с телефонным подключением должна быть развернута в каждом пуле. Это требование поддерживает записанную функцию имени, когда пользователь вызывает номер доступа из одного пула, чтобы присоединиться к Skype для бизнеса Server конференции в другом пуле. 
    
Дополнительные сведения см. в [проекте Plan for dial-in conferencing in Skype для бизнеса Server.](dial-in-conferencing.md)
  
## <a name="web-conferencing-considerations"></a>Соображения веб-конференциинга

Для веб-конференциинга требуется следующее: 
  
- Доступ к хранилищу файлов, которое используется для хранения содержимого веб-конференций.
    
- Интеграция Office web Apps Server/Office Online Server, которая необходима для обмена PowerPoint файлами во время конференции.
    
> [!NOTE]
> Последняя итерация Office web Apps Server называется Office Online Server, которая поддерживается Skype для бизнеса Server. Дополнительные подробности можно найти в Office Online Server [документации.](/officeonlineserver/office-online-server) 
  
Skype для бизнеса Server предоставляет следующие способы настройки Office web Apps Server/Office Online Server. В зависимости от имеющихся потребностей вам доступны следующие варианты:
  
- **Установите Skype для бизнеса Server и Office веб-приложения Server/Office Online Server локально за брандмауэром организации и в одной сетевой зоне.** С помощью этой топологии внешний доступ Office веб-Office Online Server серверу будет предоставлен через обратный прокси-сервер. В идеале необходимо установить Office веб-Office Online Server в той же сетевой зоне, что и Skype для бизнеса Server.
    
    Внешние Skype для бизнеса клиенты могут подключаться к Skype для бизнеса Server и Office web Apps Server/Office Online Server с помощью обратного прокси-сервера, который является сервером, который принимает запросы из Интернета и передает их во внутреннюю сеть. (Внутренним клиентам не нужно использовать обратный прокси-сервер, так как они могут подключаться к Office веб-Office Online Server серверу.) Эта топология работает лучше всего, если вы хотите использовать Office веб Office Online Server или ферму, которая используется только Skype для бизнеса Server.
    
- **Используйте внешне развернутый сервер Office веб-приложений Server/Office Online Server.** В этой топологии Skype для бизнеса Server развертывается локально и использует Office web Apps Server/Office Online Server, развернутый за пределами Skype для бизнеса Server сетевой зоны. Это может произойти, Office веб-приложения Server/Office Online Server используется для нескольких приложений в корпорации и развертывается в сети, требующей Skype для бизнеса Server использования внешнего интерфейса Office Web Apps Server/Office Online Server и наоборот.
    
    Вам не нужно устанавливать обратный прокси-сервер; вместо этого все запросы с сервера Office веб Office Online Server или Skype для бизнеса Server перенабраются через ваш edge Server. Как внутренние, так и внешние Skype для бизнеса подключаются к Office web Apps Server/Office Online Server с помощью внешнего URL-адреса.
    
    Если сервер Office веб-приложений Office Online Server развертывается за пределами внутреннего брандмауэра, выберите параметр Office сервер веб-приложений развернут во внешней сети **(то** есть периметре или Интернете) в Topology Builder.
    
Дополнительные сведения см. в [странице Настройка интеграции с Office web Apps Server в Skype для бизнеса Server.](../../deploy/deploy-conferencing/office-web-app-server.md) 
  
Независимо от выбранной топологии крайне важно открыть правильные порты брандмауэра. Необходимо убедиться, что имена DNS, IP-адреса и порты не блокируют брандмауэры на сервере Office web Apps Server/Office Online Server, балансире нагрузки или Skype для бизнеса Server.
  
> [!NOTE]
> Другим вариантом для предоставления внешнего доступа Office веб-Office Online Server или Office Online Server является развертывание сервера в сети периметра. Если вы хотите это сделать, помните, Office сервер веб Office Online Server или Office Online Server серверный компьютер должен быть членом домена Active Directory. Если ваша политика сети не позволяет компьютерам в сети периметра быть членами домена Active Directory, рекомендуется не устанавливать Office веб-приложения Server/Office Online Server в сети периметра. Вместо этого необходимо установить Office веб-Office Online Server во внутренней сети и предоставить внешний доступ к пользователю через обратный прокси-сервер. 
  
## <a name="topology-requirements-for-large-meetings"></a>Требования топологии для больших собраний

Для одного большого собрания требуется по крайней мере один сервер переднего и один сервер заднего конца. Однако для обеспечения высокой доступности рекомендуется использовать два пула front End Server с зеркальными серверами заднего вида, как показано на следующей схеме:
  
**Топология больших собраний**

![Топология больших собраний.](../../media/06858900-a262-4a47-96d0-51abd6827064.png)
  
Пользователь, на котором размещены крупные собрания, должен иметь учетную запись пользователя, размещенную в пуле интерфейса. Однако не рекомендуется размещать в этом пуле учетные записи других пользователей. Используйте его только для больших собраний. Рекомендуется создать в этом пуле специальную учетную запись пользователя, используемую только для размещения больших собраний. Так как большая настройка собрания оптимизирована для производительности, использование его в качестве обычного пользователя может иметь проблемы, такие как невозможность содействовать сеансу P2P на собрание при вовлечении конечной точки PSTN.
  
При управлении пулом с двумя серверами переднего плана необходимо учитывать некоторые особенности. Дополнительные сведения см. в Skype для бизнеса Server топологии [2015](../../plan-your-deployment/topology-basics/topology-basics.md) г. и топологии ссылок [Skype для бизнеса Server 2015 г.](../../plan-your-deployment/topology-basics/reference-topologies.md)
  
Кроме того, если вы хотите дополнительно предоставить резервное копирование и сбой аварийного восстановления для пула, используемого для больших собраний, вы можете объединить его с аналогичным набором выделенного пула в другом центре обработки данных. Подробные сведения см. [в материале Plan for high availability and disaster recovery in Skype для бизнеса Server.](../../plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md)
  
Дополнительные замечания о топологии
  
- Для хранения контента собраний требуется доля файлов и, если сервер архива развернут и включен, для хранения архивных файлов. Общая папка может быть выделена для пула или использоваться совместно с другим пулом на сайте, на котором развернут пул. Дополнительные сведения о настройке файла см. в материале [Create a file share in Skype для бизнеса Server 2015.](../../deploy/install/create-a-file-share.md)
    
- Для Office веб-Office Online Server для включения функции PowerPoint презентаций на больших собраниях требуется Office Online Server веб-приложения. Сервер Office-Office Online Server может быть посвящен большому пулу собраний или может быть таким же Office сервером веб Office Online Server или Office Online Server, используемым другими пулами на сайте, на котором развернут выделенный пул. Дополнительные сведения см. в [странице Настройка интеграции с Office web Apps Server в Skype для бизнеса Server.](../../deploy/deploy-conferencing/office-web-app-server.md) 
    
- Балансировка нагрузки для серверов переднего конечного сервера требует балансировки аппаратной нагрузки для трафика HTTP (например, загрузки контента собраний). Балансировка нагрузки DNS рекомендуется для трафика SIP. Подробные сведения [см. в материале Требования](../../plan-your-deployment/network-requirements/load-balancing.md)к балансировки нагрузки для Skype для бизнеса. 
    
- Если вы хотите использовать Службу мониторинга для выделенного пула больших собраний, рекомендуется использовать сервер мониторинга и его базу данных, общие для всех пулов переднего Skype для бизнеса Server развертывания. Дополнительные сведения см. в [Skype для бизнеса Server.](../../plan-your-deployment/monitoring.md)
