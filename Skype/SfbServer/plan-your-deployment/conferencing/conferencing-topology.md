# <a name="plan-your-conferencing-topology-for-skype-for-business-server"></a>Планирование топологии для skype для бизнеса Server
 
**Сводка:** В этом разделе вы узнаете о планировании топологии сконференций в Skype для бизнеса Server.
  
В этом разделе описаны основные принципы топологии для работы сконференцией в Skype для бизнеса Server.
  
- Поддерживаемые топологии
    
- Вопросы, которые следует учитывать при телефонной комконференции
    
- Вопросы, которые следует учитывать при веб-трансляциях
    
- Требования для больших собраний
    
Дополнительные сведения о требованиях к оборудованию и программному обеспечению см. в сведениях о требованиях к оборудованию и программному обеспечению для работы с видеоконференцией [в Skype для бизнеса Server.](hardware-and-software-requirements.md)
  
## <a name="supported-topologies"></a>Поддерживаемые топологии

В Skype для бизнеса Server сервер со службами conferencing всегда совмещается с серверами переднего уровня или серверами Standard Edition. При развертывании Skype для бизнеса Server функции im conferencing развертываются автоматически. Вы можете указать, следует ли развертывать веб-, аудио- и видеоконференцию и аудиоконференцию с помощью построитель топологий. Кроме того, с помощью построитель топологий можно добавить в существующее развертывание видеоконференцию. For details about topology basics and collocation scenarios, see [Topology Basics for Skype for Business Server.](../../plan-your-deployment/topology-basics/topology-basics.md)
  
Вы можете развернуть conferencing в следующих topologies и конфигурациях:
  
- Skype для бизнеса Server Standard Edition
    
- Skype для бизнеса Server Enterprise Edition
    
- С Корпоративная голосовая связь
    
## <a name="dial-in-conferencing-considerations"></a>Вопросы, которые следует учитывать при телефонной комконференции

При развертывании телефонной комконференции необходимо учесть следующее:
  
- Для телефонной связи требуется сервер-посредник для перевода сигналов (и мультимедиа в некоторых конфигурациях) между Skype для бизнеса Server и шлюзом STN, а также шлюз STN для перевода сигналов и мультимедиа между сервером-посредником и шлюзом STN.
    
   Прежде чем настраивать функцию телефонной видеоконференации, необходимо развернуть сервер-Корпоративная голосовая связь или сервер-посредник и хотя бы один из следующих серверов:
    
  - шлюз ТСОП;
    
  - IP-PBX
    
  - пограничный контроллер сеансов (SBC) для поставщика услуг Интернет-телефонии, подключение к которому выполняется путем настройки канала SIP.
    
- Вы можете развернуть службу приложений, приложение помощника по конференцию и приложение объявлений для conferencing на центральном сайте, но не на сайте филиала.
    
- You must deploy dial-in conferencing in every pool where you deploy Skype for Business Server conferencing. Назначать номера доступа в каждом пуле не обязательно, но функция конференц-связи с телефонным подключением должна быть развернута в каждом пуле. Это требование поддерживает функцию записанных имен, когда пользователь вызывает номер доступа из одного пула, чтобы присоединиться к конференции Skype для бизнеса Server в другом пуле. 
    
Дополнительные сведения см. в сведениях о планировании для телефонной комконференции [в Skype для бизнеса Server.](dial-in-conferencing.md)
  
## <a name="web-conferencing-considerations"></a>Вопросы, которые следует учитывать при веб-трансляциях

Для веб-conferencing требуется следующее: 
  
- Доступ к хранилищу файлов, которое используется для хранения содержимого веб-конференций.
    
- Интеграция с сервером Office Web Apps или Office Online Server, которая необходима для обмена файлами PowerPoint во время конференции.
    
> [!NOTE]
> Последняя итерация сервера Office Web Apps называется Office Online Server, поддерживаемой Skype для бизнеса Server. Дополнительные подробности можно найти в [документации по Office Online Server.](https://technet.microsoft.com/library/jj219456%28v=office.16%29.aspx) 
  
Skype для бизнеса Server предоставляет следующие способы настройки Сервера Office Web Apps или Office Online Server. В зависимости от имеющихся потребностей вам доступны следующие варианты:
  
- **Установите Skype для бизнеса Server и Сервер Office Web Apps или Office Online Server локально за брандмауэром организации и в одной сетевой зоне.** В этой топологии внешний доступ к серверу Office Web Apps или Office Online Server будет предоставляться через обратный прокси-сервер. В идеале следует установить Сервер Office Web Apps или Office Online Server в той же зоне сети, что и Skype для бизнеса Server.
    
    Внешние клиенты Skype для бизнеса могут подключаться к Skype для бизнеса Server и серверу Office Web Apps или Office Online Server с помощью обратного прокси-сервера, который принимает запросы из Интернета и перенаадружает их во внутреннюю сеть. (Внутренним клиентам не нужно использовать обратный прокси-сервер, так как они могут подключаться к серверу Office Web Apps или Office Online Server напрямую.) Эта топология лучше всего работает, если вы хотите использовать выделенную ферму Сервер Office Web Apps или Office Online Server, которая используется только Skype для бизнеса Server.
    
- **Используйте внешне развернутый сервер Office Web Apps или Office Online Server.** В этой топологии Skype для бизнеса Server развертывается локально и использует сервер Office Web Apps или Office Online Server, развернутый за пределами зоны сети Skype для бизнеса Server. Это может произойти, когда сервер Office Web Apps или Office Online Server совместно используется несколькими приложениями в корпорации и развертывается в сети, требующей от Skype для бизнеса Server использования внешнего интерфейса Сервера Office Web Apps или Office Online Server и наоборот.
    
    Вам не нужно устанавливать обратный прокси-сервер; Вместо этого все запросы от сервера Office Web Apps или Office Online Server к Skype для бизнеса Server маршрутются через ваш сервер. Внутренние и внешние клиенты Skype для бизнеса подключаются к серверу Office Web Apps или Office Online Server с помощью внешнего URL-адреса.
    
    Если сервер Office Web Apps или Office Online Server развернуты за пределами внутреннего брандмауэра, выберите вариант развертывания **сервера Office Web Apps** во внешней сети (то есть в периметре или Интернете) в построите топологий.
    
Дополнительные сведения см. в подстройке ["Настройка интеграции с сервером Office Web Apps" в Skype для бизнеса Server.](../../deploy/deploy-conferencing/office-web-app-server.md) 
  
Независимо от выбранной топологии крайне важно открыть правильные порты брандмауэра. Необходимо убедиться, что DNS-имена, IP-адреса и порты не блокируются брандмауэрами на сервере Office Web Apps, Office Online Server, под балансире нагрузки или Skype для бизнеса Server.
  
> [!NOTE]
> Другой вариант предоставления внешнего доступа к серверу Office Web Apps или Office Online Server — развертывание сервера в сети периметра. В этом случае следует помнить, что для установки Сервера Office Web Apps или Office Online Server необходимо, чтобы серверный компьютер был членом домена Active Directory. Если сетовая политика не разрешает компьютерам в сети периметра быть членами домена Active Directory, не рекомендуется устанавливать сервер Office Web Apps или Office Online Server в сети периметра. Вместо этого следует установить сервер Office Web Apps или Office Online Server во внутренней сети и предоставить внешним пользователям доступ через обратный прокси-сервер. 
  
## <a name="topology-requirements-for-large-meetings"></a>Требования к топологии для больших собраний

Для одного большого собрания требуется по крайней мере один сервер переднего сервера и один тыловой сервер. Однако для обеспечения высокой доступности рекомендуется использовать два пула серверов переднего сервера с зеркальными серверами, как показано на следующей схеме:
  
**Топология больших собраний**

![Топология больших собраний](../../media/06858900-a262-4a47-96d0-51abd6827064.png)
  
Учетная запись пользователя, на котором размещены большие собрания, должна быть размещена во интерфейсных пулах. Однако не рекомендуется размещать в этом пуле учетные записи других пользователей. Используйте его только для больших собраний. Рекомендуется создать в этом пуле специальную учетную запись пользователя, используемую только для размещения больших собраний. Так как параметр большого собрания оптимизирован для обеспечения производительности, его использование в качестве обычного пользователя может привести к проблемам, таким как невозможность продвижения сеанса P2P к собранию, когда используется конечная точка STN.
  
При управлении пулом с двумя серверами переднего плана необходимо учитывать некоторые особенности. For more information, see [Topology Basics for Skype for Business Server 2015](../../plan-your-deployment/topology-basics/topology-basics.md) and [Reference topologies for Skype for Business Server 2015](../../plan-your-deployment/topology-basics/reference-topologies.md).
  
Кроме того, если вы хотите при желании предоставить резервное копирование и отбой аварийного восстановления для пула, используемого для больших собраний, вы можете совметь его с аналогичным выделенным пулом в другом центре обработки данных. For details, see [Plan for high availability and disaster recovery in Skype for Business Server.](../../plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md)
  
Дополнительные замечания о топологии
  
- Для хранения содержимого собраний требуется файл, а если сервер архива развернут и включен, для хранения файлов архива. Общая папка может быть выделена для пула или использоваться совместно с другим пулом на сайте, на котором развернут пул. Дополнительные сведения о настройке файловой папки см. в сведениях о создании файловой папки [в Skype для бизнеса Server 2015.](../../deploy/install/create-a-file-share.md)
    
- Сервер Office Web Apps или Office Online Server необходим для включения функций презентации PowerPoint на больших собраниях. Сервер Office Web Apps или Office Online Server может быть выделен для большого пула собраний или может быть тем же сервером Office Web Apps или Office Online Server, который используется другими пулами на сайте, на котором развернут выделенный пул. Дополнительные сведения см. в подстройке ["Настройка интеграции с сервером Office Web Apps" в Skype для бизнеса Server.](../../deploy/deploy-conferencing/office-web-app-server.md) 
    
- Для балансировки нагрузки серверов переднего сервера требуется аппаратная балансировка нагрузки для ТРАФИКА HTTP (например, загрузка содержимого собрания). Балансировка нагрузки DNS рекомендуется для трафика SIP. For details see [Load balancing requirements for Skype for Business](../../plan-your-deployment/network-requirements/load-balancing.md). 
    
- Если вы хотите использовать сервер мониторинга для выделенного пула больших собраний, рекомендуем использовать сервер мониторинга и его базу данных, которые совместно используются всеми пулами серверов переднего сервера в развертывании Skype для бизнеса Server. Дополнительные сведения см. в [сведениях о планировании мониторинга в Skype для бизнеса Server.](../../plan-your-deployment/monitoring.md)
