# <a name="plan-for-large-meetings-in-skype-for-business-server"></a>Планирование больших собраний в Skype для бизнеса Server
 
**Сводка:** В этом разделе рассказывается о рекомендациях по внедрению больших собраний и управлению ими в Skype для бизнеса Server.
  
Размер собраний, поддерживаемых Skype для бизнеса Server, зависит от того, размещается ли Конференц-связь в общем или выделенном пуле: где-либо из 250 участников в общем пуле для 1000 участников в выделенном пуле. 
  
> [!NOTE]
> В этом разделе рассматриваются рекомендации по использованию больших собраний, поддерживаемых Skype для бизнеса Server. Если вашей организации требуются более крупные возможности для собраний, рекомендуется реализовать гибридную среду, которая использует трансляцию собраний Skype, новую веб-службу, которая входит в состав Office 365. 

> [!NOTE]
> Трансляция собраний Skype позволяет пользователям размещать и транслировать собрания для крупных интерактивных аудиторий до 10 000 участников. Для использования вещания собраний Skype необходимо, чтобы Skype для бизнеса Server уже был настроен в гибридной конфигурации с рабочей организацией Office 365. Все пользователи должны иметь интерактивный клиент, установленный в качестве необходимого компонента. Если вы заинтересованы в развертывании гибридного решения, в котором можно использовать трансляцию собраний Skype, посмотрите, [что такое трансляция собрания Skype?](https://go.microsoft.com/fwlink/?LinkId=617071) и [Настройте локальное развертывание для вещания собраний Skype](../../deploy/configure-skype-meeting-broadcast.md). 
  
Большие собрания, как правило, имеют следующие характеристики:
  
- формат собрания: презентация один-ко-многим;
    
- один или несколько пользователей являются докладчиками, а все остальные являются участниками;
    
- совместное использование презентации PowerPoint — это основное действие для совместной работы с данными;
    
- аудио является обязательным, видео также может использоваться;
    
- Выделенный человек, как правило, организатор собрания или помощник по организатору, заранее настраивает собрание.
    
- выделенные сотрудники (не докладчики) руководят собранием, в том числе контролируют подключение к собранию по сети, проверка работы аудио, видео и совместного использования слайдов, управляют залом ожидания и ролями пользователей, отключают и включают звук участников, принимают вопросов и управляют записями, по мере необходимости.
    
Когда пользователь планирует собрание, Skype для бизнеса Server создает запись в базе данных конференций, в которой хранятся данные конференц-связи, но не резервирует аппаратные ресурсы для запланированного собрания заранее. Вместо этого Skype для бизнеса Server имеет встроенную логику балансировки нагрузки для динамического выделения ресурсов конференц-связи на серверах переднего плана таким образом, что они распределяются одинаково на всех серверах переднего плана в пуле. Это эффективно предоставляет и использует аппаратные ресурсы, но важно правильно спланировать для поддержки очень больших собраний. 
  
Например, если пул Skype для бизнеса Server работает ближе к своей максимальной емкости, каждый сервер переднего плана может размещаться приблизительно в 125 среднего размера собраний. Добавление еще одного небольшого собрания не является проблемой, но при добавлении собрания для 1000 пользователей возникает проблема, так как сервер переднего плана может не поддерживать такое большое собрание одновременно с другими 125ми собрания.
  
Для поддержки больших собраний до 1000 участников требуется устранение проблем, связанных с моделью общего оборудования и моделью без резервирования. В общем случае необходимо запланировать выделенный пул и следовать рекомендациям, описанным в следующих разделах. 
  
## <a name="plan-for-a-dedicated-pool"></a>Планирование выделенного пула

Если для организации требуется проведение собраний с более чем 250 участниками, необходимо спланировать выделенный пул для поддержки нагрузки. 
  
Чтобы иметь достаточные ресурсы ЦП и памяти для собраний до 1000 пользователей, серверы переднего плана размещения не должны содержать другие рабочие нагрузки для обмена мгновенными сообщениями и присутствия и корпоративной голосовой связи. На серверах также должны размещаться не все другие собрания, независимо от размера других собраний. Для размещения собраний до 1000 пользователей необходимо настроить отдельный пул Skype для бизнеса Server, предназначенный для размещения больших собраний.
  
Пул Skype для бизнеса Server, предназначенный для размещения больших собраний, должен размещать одно и только одно собрание до 1000 пользователей одновременно, поэтому время проведения собрания должно быть зарезервировано заранее с помощью стандартного процесса планирования, чтобы обеспечить специальную поддержку на серверах переднего плана. Для одновременной поддержки нескольких больших собраний необходимо настроить несколько выделенных пулов больших собраний.
  
Дополнительные сведения о требованиях к оборудованию и программному обеспечению и планированию топологии, поддерживающей большое количество собраний, приведены [в статье требования к оборудованию и программному обеспечению для конференц-связи в Skype для бизнеса Server](hardware-and-software-requirements.md) и [Планирование топологии конференций Skype для бизнеса Server](conferencing-topology.md).
  
## <a name="implement-best-practices-for-large-meetings"></a>Применение рекомендаций для больших собраний

После настройки выделенного пула для больших собраний можно выполнить действия, чтобы убедиться, что крупные собрания, размещенные в пуле, обеспечивают оптимальное взаимодействие с пользователем. В следующих разделах приводятся рекомендации по управлению большими собраниями:
  
- Создание выделенных организаторов собраний
    
- Создание выделенных модераторов
    
- Ведение отдельного календаря для больших собраний
    
- Реализация процесса планирования больших собраний
    
- Указание подходящих сведений о планировании
    
- Создание политики конференц-связи для больших собраний
    
### <a name="create-dedicated-meeting-organizers"></a>Создание выделенных организаторов собраний

Чтобы свести к минимуму трафик для обмена данными в реальном времени в пуле больших собраний, корпорация Майкрософт не рекомендует размещать пользователей, которые регулярно входят в систему с помощью клиентов Skype для бизнеса, и участвовать в обмене мгновенными сообщениями, присутствии, конференц-связи и голосовых сеансах. Вместо этого выполните одно из следующих действий.
  
- Создайте одну или несколько выделенных учетных записей пользователей только для планирования больших собраний.
    
- Домены учетные записи сотрудников, ответственных за планирование больших собраний в пуле больших собраний
    
### <a name="create-dedicated-moderators"></a>Создание выделенных модераторов

С несколькими сотнями пользователей на собрании рекомендуется использовать выделенный человек, который умеренный сеанс работы в сети на большом собрании. Этот выделенный пользователь может быть представителем организатора собрания или сотрудником службы поддержки крупных собраний в Организации. При планировании собрания следует добавить этого специального модератора собрания в качестве выступающего, хотя повысить участника до роли выступающего можно прямо во время собрания.
  
Модератор собрания может использовать все функции докладчика для клиентов Skype для бизнеса, чтобы управлять большим собранием. К этим функциональным возможностям относятся:
  
- Мониторинг ожидания и приглашение или отклонение пользователей в зале ожидания;
    
- Удаление всех пользователей из собрания, которые не должны присутствовать в собрании
    
- Изменение типов доступа к собранию
    
- Изменение ролей участников
    
- Приглашение дополнительных участников на собрании с помощью функции перетаскивания, звонков по телефону или электронной почты
    
- Отключение звука и раззвука аудитории или отдельных пользователей
    
- Управление контентом собрания, включая отправку содержимого, удаление контента и переключение активного контента

    
### <a name="maintain-a-separate-calendar"></a>Ведение отдельного календаря

Для каждого пула больших собраний необходимо сохранять отдельный календарь для большого количества собраний, запланированных в этом пуле. Например, вы можете использовать единую учетную запись пользователя в пуле больших собраний и использовать надстройку Outlook с Exchange и "собрание по сети" для Skype для бизнеса, чтобы хранить отдельный календарь. Если вы используете несколько учетных записей пользователей, чтобы предоставить сотрудникам службы поддержки возможность создания больших собраний, можно настроить отдельный календарь, объединяющий все крупные собрания, созданные участниками сотрудника службы поддержки. 
  
Ведение отдельного календаря больших собраний помогает избежать конфликтов и убедиться, что в любой момент времени активно только одно большое собрание.
  
### <a name="implement-a-scheduling-process"></a>Реализация процесса планирования

Так как в выделенном большом пуле собраний поддерживается только одно большое собрание, необходимо реализовать большой процесс планирования собраний, чтобы упростить настройку больших собраний и предотвратить конфликты. Такая возможность не предусмотрена непосредственно для клиентов Skype для бизнеса Server или Skype для бизнеса. Один из способов реализации такого процесса заключается в использовании системы билетов группы поддержки вашей организации (если она доступна).
  
При планировании большого собрания включается выполнение следующих действий.
  
- Организатор собрания или делегат определяет  время, продолжительность и размер предстоящего собрания, а также список выступающих. Если ожидаемый размер собрания превышает 250 пользователей или если требуется улучшить взаимодействие с пользователями при числе участников меньше 250, организатор или делегат подает запрос на проведение большого собрания.
    
- Специалист по планированию проверяет свободны ли требуемые дата и время. Поскольку мы поддерживаем одновременное проведение только одного большого собрания в выделенном пуле, специалисту по планированию требуется свериться с календарем больших собраний, чтобы определить, не запланировано ли на этот день и это время другое собрание. Если запрошенное время доступно, специалист по планированию утверждает запрос на собрание.
    
- Если запрос утвержден, сотрудники планирования (с использованием учетных данных в выделенном пуле) используют надстройку "собрание по сети" для Skype для бизнеса с помощью Outlook, чтобы настроить собрание в выделенном пуле больших собраний. URL-адрес для присоединения к собранию предоставляется инициатору запроса в рамках процедуры утверждения.
    
- Организатор собрания или представитель использует Outlook для планирования предстоящего собрания, добавляя URL-адрес для присоединения к собранию к приглашению на собрание. После этого организатор собрания или делегат указывает приглашенных пользователей и рассылает приглашение на собрание.
    
### <a name="specify-appropriate-scheduling-details"></a>Указание подходящих сведений о планировании

После проверки отсутствия другого запланированного собрания на запрашиваемое время, персонал поддержки, который обрабатывает запрос, планирует собрание в пуле больших собраний. 
  
Чтобы обеспечить оптимальное взаимодействие с пользователем, важно запланировать большое собрание с нужными уровнями доступа и настройками собрания, соответствующими потребностям организатора собрания. Примите во внимание следующие параметры планирования, настроенные в параметрах собрания Skype для бизнеса:
  
- Используйте новое место собрания для каждого большого собрания вместо повторного использования выделенного места собрания. 
    
- Указывайте уровень доступа к собранию следующим образом.
    
  - Если по крайней мере один приглашенный является внешним по отношению к Организации, задайте тип доступа к собранию **всем (без ограничений)**. Это позволит исключить необходимость управления потенциально большим "залом ожидания" во время проведения собрания.
    
  - Если собрание только внутреннее, установите тип доступа к собранию **Все во организации**.
    
    > [!NOTE]
    > Избегайте устанавливать тип доступа к собранию **Приглашенные пользователи из организации**, поскольку при использовании этого параметра придется добавлять адреса электронной почты всех пользователей в список приглашенных. Кроме того, при использовании этого параметра не работает приглашение групп рассылки. Избегайте устанавливать тип доступа **Только организатор**, поскольку при использовании этого параметра во время проведения собрания в "зал ожидания" должны помещаться все участники собрания, включая выступающих. Лицо, отвечающее за проведение собрания, должно затем постоянно наблюдать за реестром "зала ожидания" и допускать новых пользователей из "зала ожидания".
  
- Разрешите пользователям автоматически присоединяться к собранию путем вызова с телефонного аппарата, включив параметр **Вызывающие абоненты присоединяются напрямую**.
    
- Явным образом приглашайте следующих пользователей:
    
  - организатор собрания и делегат (инициатор запроса);
    
  - список выступающих, предоставленный инициатором запроса на собрание.
    
    > [!NOTE]
    > Если установлен тип доступа к собранию **Выбранные пользователи**, то требуется явно добавлять каждого участника большого собрания как приглашенного на собрание. 
  
- Явно управляйте докладчиками вместо определения параметра докладчика в качестве одного из значений автоматического продвижения. Убедитесь, что следующие пользователи добавлены в качестве докладчиков:
    
  - организатор собрания и делегат (инициатор запроса);
    
  - список выступающих, предоставленный инициаторами запроса на большое собрание.
    
    Явно управляя докладчиками, вы можете ограничить докладчики достаточно большим числом, чтобы сделать возможным эффективное большое собрание. Если большинство участников имеет роль участника, это помогает уменьшить вероятность случайного получения контроля над выступлением, удаления презентации PowerPoint, включения или отключения микрофона выступающих и других нарушений в собрании. 
    
- Проверьте параметр **Отключить всех участников**, чтобы было слышно только докладчиков.
    
- Установите флажок **блокировать видео участников** , чтобы убедиться, что только докладчики могут транслировать видео на собрание.
    
### <a name="create-a-conferencing-policy"></a>Создание политики конференц-связи

Создайте новую политику конференц-связи для больших собраний, а затем назначьте политику конференц-связи пользователям, размещенным в выделенном пуле больших собраний. Настройте эту политику конференц-связи, используя следующие параметры.
  
- Задайте для параметра **максмитингсизе** значение 1000. (Значение по умолчанию — 250.)
    
- Для параметра **AllowLargeMeetings** установите значение **True**. 
    
- Для параметра **EnableAppDesktopSharing** установите значение **None**.
    
- Для параметра **AllowUserToScheduleMeetingsWithAppSharing** установите значение **False**.
    
- Для параметра **AllowSharedNotes** установите значение **False**.
    
- Для параметра **AllowAnnotations** установите значение **False**.
    
- Для параметра **DisablePowerPointAnnotations** установите значение **True**.
    
- Для параметра **AllowMultiview** установите значение **False**.
    
- Для параметра **EnableMultiviewJoin** установите значение **False**.
    
> [!NOTE]
> Для поддержки больших собраний в Skype для бизнеса Server необходимо, чтобы для параметра **алловларжемитингс** было задано значение true. Если для этого параметра задано значение true, взаимодействие с Skype для бизнеса будет оптимизировано для очень больших собраний, когда пользователи присоединяются к собранию. В частности, в большом собрании Skype для бизнеса не будет показывать первоначальное или обновление полного списка участников собрания, что является узким местом производительности для клиента и Skype для бизнеса Server. Вместо этого Skype для бизнеса будет показывать только сведения о пользователе и список докладчиков собрания. Skype для бизнеса по-прежнему будет показывать общее количество участников, доступных в больших собраниях.

Параметр **$true алловларжемитингс** приводит к следующим действиям:

- Скрывает список участников. 

- Отключает ошибки в окне мгновенных сообщений.

- Отключает видеосвязь с несколькими участниками.

- Отключает возможность продвижения участника на выступающий. Вы должны заранее спланировать и объявить всех докладчиков перед собранием.

- Отключает возможность включения звука отдельных участников.

- Отключает возможность применения функции "закрыть полезные данные" для участников.

- Пользователи с телефонным подключением PSTN не смогут включить микрофон с помощью 6, так как личный виртуальный помощник, отвечающий за команды DTMF в активных больших собраниях, отсутствует.

- Если докладчик или организатор планирует собрание, для которого необходимо отключить микрофон ("выключить все"), пользователи PSTN будут отключены во время звонка и не смогут включать микрофон.

За исключением параметра **Maximum meeting size (Максимальный размер собрания)**, все остальные представленные здесь параметры политики конференц-связи требуются для того, чтобы отключить возможности конференц-связи, которые не нужны в больших собраниях.
  
Кроме того, необходимо настроить выделенный пул больших собраний, чтобы каждый пользователь Skype для бизнеса Server, размещенный в пуле и отвечающий за управление расписанием собраний, получил соответствующие разрешения. Для этого выполните следующие действия.
  
- Для параметра **Designate as presenter (Назначить выступающим)** установите значение **None (Нет)**. Обычно только несколько человек (или даже один) среди всех участников больших собраний являются выступающими, поэтому участников не следует автоматически допускать к собраниям в качестве выступающих. Выступающие должны явно назначаться во время планирования собрания или явно выдвигаться во время самого собрания.
    
- Убедитесь, что флажок **Assigned conference type by default (Назначенный тип конференции по умолчанию)** снят. Этот параметр определяет, будет ли надстройка "собрание по сети" в Skype для бизнеса всегда планировать конференции с помощью назначенной конференции организатора, что означает, что для запланированных собраний используются одинаковые URL-адреса и звуковые данные. В сценариях совместной работы небольших групп такой назначенный тип конференции работает хорошо, поскольку у каждого есть своя собственная назначенная конференция, и постоянный URL-адрес присоединения и аудиоданные помогают упростить быстрое присоединение к собранию. Однако в сценариях больших собраний персонал поддержки планирует большие собрания с помощью одного набора пользовательских учетных данных, а затем предоставляет URL-адреса для присоединения и аудиоданные инициаторам собраний. В этом случае лучше работает использование разных URL-адресов для присоединения к разным собраниям.
    
- Убедитесь, что флажок **Admit anonymous users by default (Допускать анонимных пользователей по умолчанию)** снят, если только он не требуется в вашем случае. Этот параметр влияет на тип доступа к собранию по умолчанию, запланированный надстройкой "собрание по сети" для Skype для бизнеса, когда не используется назначенная конференция. Соответствующий параметр для этого параметра зависит от потребностей Организации. Если большинство больших собраний в организации составляют внутренние собрания, не следует устанавливать этот флажок. Если в большинстве больших собраний требуется присутствие внешних пользователей, установите этот флажок.
    
Дополнительные сведения о создании политики конференц-связи можно найти [в статье Управление политиками конференц-связи в Skype для бизнеса Server](../../manage/conferencing/conferencing-policies.md).
