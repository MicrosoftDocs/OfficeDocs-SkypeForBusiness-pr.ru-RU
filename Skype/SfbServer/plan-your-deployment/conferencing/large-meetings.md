# <a name="plan-for-large-meetings-in-skype-for-business-server"></a>Планирование больших собраний в Skype для бизнеса Server
 
**Сводка:** В этом разделе вы узнаете о практических советах по реализации больших собраний в Skype для бизнеса Server и управлению их.
  
Размер собраний, поддерживаемый Skype для бизнеса Server, зависит от того, будет ли конференции проводиться в общем или выделенном пуле: от 250 участников в общем пуле до 1000 участников в выделенном пуле. 
  
> [!NOTE]
> В этом разделе основное внимание уделяется практическим советам по большим собраниям, поддерживаемых Skype для бизнеса Server. Если вашей организации требуются более крупные возможности собраний, следует рассмотреть возможность реализации гибридной среды, которая будет использовать трансляцию собраний Skype, новую веб-службу, которая входит в состав Microsoft 365 и Office 365. 

> [!NOTE]
> Трансляция собраний Skype позволяет пользователям проводить и транслировать собрания для больших аудиторий в Интернете до 10 000 участников. Для использования трансляции собраний Skype необходимо, чтобы Skype для бизнеса Server уже был настроен в гибридной конфигурации с производственной организацией Microsoft 365 или Office 365. Все пользователи должны иметь сетевой клиент, установленный в качестве обязательного условия. If you are interested in deploying a hybrid solution that can take advantage of Skype Meeting Broadcast, see [What is a Skype Meeting Broadcast?](https://go.microsoft.com/fwlink/?LinkId=617071) and [Configure your on-premises deployment for Skype Meeting Broadcast](../../deploy/configure-skype-meeting-broadcast.md). 
  
Большие собрания обычно имеют следующие характеристики:
  
- формат собрания: презентация один-ко-многим;
    
- один или несколько пользователей являются докладчиками, а все остальные являются участниками;
    
- совместное использование презентации PowerPoint — это основное действие для совместной работы с данными;
    
- аудио является обязательным, видео также может использоваться;
    
- Выделенный человек, как правило, организатор собрания или помощник организатора, настраивает собрание заранее.
    
- выделенные сотрудники (не докладчики) руководят собранием, в том числе контролируют подключение к собранию по сети, проверка работы аудио, видео и совместного использования слайдов, управляют залом ожидания и ролями пользователей, отключают и включают звук участников, принимают вопросов и управляют записями, по мере необходимости.
    
Когда пользователь планирует собрание, Skype для бизнеса Server создает запись в базе данных проведения собраний, в которой хранится информация о собрании, но не резервировать аппаратные ресурсы для запланированного собрания заранее. Вместо этого в Skype для бизнеса Server есть встроенная логика балансировки нагрузки для динамического распределения ресурсов на серверах переднего сервера таким образом, чтобы равномерно распределять нагрузку по всем серверам переднего сервера в пуле. Это эффективно подготовка и использование аппаратных ресурсов, но важно правильно спланировать поддержку очень больших собраний. 
  
Например, когда пул Skype для бизнеса Server работает почти на верхней емкости, на каждом сервере переднего сервера может быть примерно 125 собраний среднего размера. Добавление еще одного небольшого собрания не является проблемой, но добавление собрания для 1000 пользователей может быть проблемой, так как серверы переднего сервера, вероятно, не смогут поддерживать такое большое собрание одновременно с другими 125 собраниями.
  
Для поддержки больших собраний с числом участников до 1000 необходимо решить проблемы, связанные с моделью общего оборудования и моделью без резервирования. В общем случае необходимо спланировать выделенный пул и следовать практическим методикам, описанным в следующих разделах. 
  
## <a name="plan-for-a-dedicated-pool"></a>Планирование выделенного пула

Если вашей организации требуются собрания с числом участников более 250, необходимо запланировать выделенный пул для поддержки нагрузки. 
  
Чтобы иметь достаточные ресурсы ЦП и памяти для собраний с количеством пользователей до 1000, на серверах переднего сервера не должны быть размещены другие рабочие нагрузки обмена мгновенными сообщениями и данными о присутствии или Корпоративная голосовая связь серверов. На серверах также не следует проводить другие собрания независимо от размера других собраний. Для размещения собраний с числом пользователей до 1000 необходимо настроить отдельный пул Skype для бизнеса Server, выделенный для размещения больших собраний.
  
В пуле Skype для бизнеса Server, выделенном для размещения больших собраний, должно быть одновременно размещено одно и только одно собрание из 1000 пользователей, поэтому необходимо заранее зарезервировать время собрания с помощью внеполюстного процесса планирования, чтобы обеспечить выделенную поддержку серверов переднего сервера. Чтобы одновременно поддерживать несколько больших собраний, необходимо настроить несколько выделенных пулов больших собраний.
  
For more information about hardware and software requirements, and planning a topology that supports large meetings, see [Hardware and software requirements for conferencing in Skype for Business Server](hardware-and-software-requirements.md) and Plan your [conferencing topology for Skype for Business Server](conferencing-topology.md).
  
## <a name="implement-best-practices-for-large-meetings"></a>Реализация практических методик для больших собраний

После настройки выделенного пула для больших собраний можно предпринять действия, чтобы гарантировать, что большие собрания, которые будут проводиться в пуле, обеспечивают наилучшее пользовательское интерфейс. В следующих разделах указаны рекомендации по управлению большими собраниями.
  
- Создание организаторов выделенных собраний
    
- Создание выделенных модераторов
    
- Ведение отдельного календаря больших собраний
    
- Реализация процесса планирования больших собраний
    
- Указание соответствующих сведений о планировании
    
- Создание политики проведения собраний для больших собраний
    
### <a name="create-dedicated-meeting-organizers"></a>Создание организаторов выделенных собраний

Чтобы свести к минимуму трафик связи в пуле больших собраний в режиме реального времени, корпорация Майкрософт не рекомендует проводить размещение пользователей, которые регулярно входим в систему с помощью клиентов Skype для бизнеса и участвуют в сеансах обмена мгновенными сообщениями, присутствия, проведения собраний и голосовой связи. Вместо этого сделайте одно из следующих:
  
- Создайте одну или несколько выделенных учетных записей пользователей только для планирования больших собраний
    
- Учетные записи сотрудников, ответственных за планирование больших собраний в пуле больших собраний
    
### <a name="create-dedicated-moderators"></a>Создание выделенных модераторов

При от нескольких сотнях до тысячи пользователей на собрании лучше, чтобы выделенный человек мог модерировать онлайн-сеанс большого собрания. Этот выделенный человек может быть делегатом организатора собрания или сотрудником службы поддержки больших собраний организации. При планировании собрания следует добавить этого специального модератора собрания в качестве выступающего, хотя повысить участника до роли выступающего можно прямо во время собрания.
  
Модератор собрания может использовать все функции ведущего в клиентах Skype для бизнеса для управления большим собранием. К этим функциональным возможностям относятся:
  
- Наблюдение за "lobby" и прием или отклонение пользователей в "
    
- Удаление из собрания всех пользователей, которых не должно быть на собрании
    
- Изменение типов доступа к собраниям
    
- Изменение ролей участников
    
- Приглашение дополнительных участников во время собрания с помощью функции перетаскивания, телефонного звонка или электронной почты
    
- Отключение и отключение темы для аудитории или отдельных пользователей
    
- Управление содержимым собраний, включая отправку содержимого, удаление контента и переключение активного содержимого

    
### <a name="maintain-a-separate-calendar"></a>Обслуживание отдельного календаря

Для каждого пула больших собраний следует вести отдельный календарь больших собраний, запланированных в этом пуле. Например, можно использовать одну учетную запись пользователя в пуле больших собраний и использовать Outlook с Надстройка собраний Exchange и собраний по сети для Skype для бизнеса для обслуживания отдельного календаря. Если вы используете несколько учетных записей пользователей, чтобы позволить сотрудникам службы поддержки создавать большие собрания, можно настроить отдельный календарь, в который будут сводные данные о всех больших собраниях, созданных сотрудниками службы поддержки. 
  
Ведение отдельного календаря больших собраний помогает избежать конфликтов и убедиться, что в любой момент времени активно только одно большое собрание.
  
### <a name="implement-a-scheduling-process"></a>Реализация процесса планирования

Поскольку в выделенном пуле больших собраний поддерживается только одно большое собрание за раз, следует реализовать процесс планирования больших собраний, чтобы упростить настройку больших собраний и предотвратить конфликты. Такая возможность не предоставляется напрямую клиентами Skype для бизнеса Server или Skype для бизнеса. Один из способов реализации такого процесса — использовать систему билетов группы поддержки вашей организации, если она доступна.
  
Планирование большого собрания включает выполнение следующих действий:
  
- Организатор собрания или делегат определяет  время, продолжительность и размер предстоящего собрания, а также список выступающих. Если ожидаемый размер собрания превышает 250 пользователей или если требуется улучшить взаимодействие с пользователями при числе участников меньше 250, организатор или делегат подает запрос на проведение большого собрания.
    
- Специалист по планированию проверяет свободны ли требуемые дата и время. Поскольку мы поддерживаем одновременное проведение только одного большого собрания в выделенном пуле, специалисту по планированию требуется свериться с календарем больших собраний, чтобы определить, не запланировано ли на этот день и это время другое собрание. Если запрошенное время доступно, специалист по планированию утверждает запрос на собрание.
    
- Если запрос утвержден, сотрудники по планированию (используя учетные данные в выделенном пуле) используют надстройка "Собрание по сети" для Skype для бизнеса с Outlook, чтобы настроить собрание в выделенном пуле больших собраний. URL-адрес для присоединения к собранию предоставляется инициатору запроса в рамках процедуры утверждения.
    
- Организатор или делегат собрания использует Outlook для расписания предстоящего собрания, добавляя URL-адрес для присоединения к собранию в приглашение на собрание. После этого организатор собрания или делегат указывает приглашенных пользователей и рассылает приглашение на собрание.
    
### <a name="specify-appropriate-scheduling-details"></a>Указание соответствующих сведений о планировании

После проверки отсутствия другого запланированного собрания на запрашиваемое время, персонал поддержки, который обрабатывает запрос, планирует собрание в пуле больших собраний. 
  
Для обеспечения наилучшего пользовательского интерфейса важно запланировать большое собрание с правильными уровнями доступа и настройками собрания, соответствующими потребностям организатора собрания. Рассмотрите следующие параметры планирования, настроенные в параметрах собраний Skype для бизнеса:
  
- Используйте новое место собрания для каждого большого собрания вместо повторного использования выделенного места собрания. 
    
- Указывайте уровень доступа к собранию следующим образом.
    
  - Если хотя бы один приглашенный является внешним по отношении к организации, установите тип доступа к собранию **"Все" (без ограничений).** Это позволит исключить необходимость управления потенциально большим "залом ожидания" во время проведения собрания.
    
  - Если собрание только внутреннее, установите тип доступа к собранию **Все во организации**.
    
    > [!NOTE]
    > Избегайте устанавливать тип доступа к собранию **Приглашенные пользователи из организации**, поскольку при использовании этого параметра придется добавлять адреса электронной почты всех пользователей в список приглашенных. Кроме того, при использовании этого параметра не работает приглашение групп рассылки. Избегайте устанавливать тип доступа **Только организатор**, поскольку при использовании этого параметра во время проведения собрания в "зал ожидания" должны помещаться все участники собрания, включая выступающих. Лицо, отвечающее за проведение собрания, должно затем постоянно наблюдать за реестром "зала ожидания" и допускать новых пользователей из "зала ожидания".
  
- Разрешите пользователям автоматически присоединяться к собранию путем вызова с телефонного аппарата, включив параметр **Вызывающие абоненты присоединяются напрямую**.
    
- Явным образом приглашайте следующих пользователей:
    
  - организатор собрания и делегат (инициатор запроса);
    
  - список выступающих, предоставленный инициатором запроса на собрание.
    
    > [!NOTE]
    > Если установлен тип доступа к собранию **Выбранные пользователи**, то требуется явно добавлять каждого участника большого собрания как приглашенного на собрание. 
  
- Явно управляйте докладчиками вместо определения параметра докладчика в качестве одного из значений автоматического продвижения. Убедитесь, что следующие пользователи добавлены в качестве докладчиков:
    
  - организатор собрания и делегат (инициатор запроса);
    
  - список выступающих, предоставленный инициаторами запроса на большое собрание.
    
    Явно управляя presenters, вы можете ограничить число, достаточное для проведения эффективного большого собрания. Если большинство участников имеет роль участника, это помогает уменьшить вероятность случайного получения контроля над выступлением, удаления презентации PowerPoint, включения или отключения микрофона выступающих и других нарушений в собрании. 
    
- Проверьте параметр **Отключить всех участников**, чтобы было слышно только докладчиков.
    
- Проверьте параметр **видео "Блокировать участников",** чтобы убедиться, что только участники могут транслировать видео в собрании.
    
### <a name="create-a-conferencing-policy"></a>Создание политики conferencing

Создайте новую политику проведения собраний специально для больших собраний, а затем назначьте ее пользователям, которые находятся в выделенном пуле больших собраний. Настройте эту политику конференц-связи, используя следующие параметры.
  
- Установите для **параметра MaxMeetingSize параметр** 1000. (Значение по умолчанию — 250.)
    
- Для параметра **AllowLargeMeetings** установите значение **True**. 
    
- Для параметра **EnableAppDesktopSharing** установите значение **None**.
    
- Для параметра **AllowUserToScheduleMeetingsWithAppSharing** установите значение **False**.
    
- Для параметра **AllowSharedNotes** установите значение **False**.
    
- Для параметра **AllowAnnotations** установите значение **False**.
    
- Для параметра **DisablePowerPointAnnotations** установите значение **True**.
    
- Для параметра **AllowMultiview** установите значение **False**.
    
- Для параметра **EnableMultiviewJoin** установите значение **False**.
    
> [!NOTE]
> Для поддержки больших собраний в Skype для бизнеса Server необходимо, чтобы параметр **AllowLargeMeetings** был установлен в true. Если для этого параметра установлено true, skype для бизнеса будет оптимизирован для очень больших собраний, когда пользователи присоединяются к собранию. В частности, при большом собрании Skype для бизнеса не будет показывать начальный или обновленный полный список участников собрания, что является узким местом производительности как для клиента, так и для Skype для бизнеса Server. Вместо этого в Skype для бизнеса будут показываться только сведения о пользователе и списке участников собрания. В Skype для бизнеса по-прежнему будет по-прежнему показываться общее количество участников, доступных для больших собраний.

Параметр **$true AllowLargeMeetings** вызывает следующее:

- Скрывает список участников. 

- Отключает ошибки в окне мгновенных данных.

- Отключает многостолбное видео.

- Отключает возможность продвижения участника в presenter. Необходимо заранее спланировать и объявить всех presenters перед собранием.

- Отключает возможность отключения звука отдельных участников.

- Отключает возможность применения функции "Блокировка видео в центре внимания" к участникам.

- Пользователи с номером STN не смогут отключить себя с помощью 6, так как отсутствует личная виртуальная помощь, которая отвечает за команды DTMF в активных больших собраниях.

- Если presenter/organizer запланирует собрание, где все должны быть сначала отключены ("Отключить все"), пользователи STN будут отключены во время вызова и не смогут отключить себя.

За исключением параметра **Maximum meeting size (Максимальный размер собрания)**, все остальные представленные здесь параметры политики конференц-связи требуются для того, чтобы отключить возможности конференц-связи, которые не нужны в больших собраниях.
  
Кроме того, необходимо настроить выделенный пул больших собраний, чтобы каждый пользователь Skype для бизнеса Server, который находится в пуле и отвечает за управление расписанием собраний, получил соответствующие разрешения. Для этого выполните следующие действия.
  
- Для параметра **Designate as presenter (Назначить выступающим)** установите значение **None (Нет)**. Обычно только несколько человек (или даже один) среди всех участников больших собраний являются выступающими, поэтому участников не следует автоматически допускать к собраниям в качестве выступающих. Выступающие должны явно назначаться во время планирования собрания или явно выдвигаться во время самого собрания.
    
- Убедитесь, что флажок **Assigned conference type by default (Назначенный тип конференции по умолчанию)** снят. Этот параметр управляет тем, всегда ли надстройка "Собрание по сети" в Skype для бизнеса планирует конференции с использованием назначенной организатором конференции, что означает, что запланированные собрания имеют одинаковый URL-адрес join и аудиоконференции. В сценариях совместной работы небольших групп такой назначенный тип конференции работает хорошо, поскольку у каждого есть своя собственная назначенная конференция, и постоянный URL-адрес присоединения и аудиоданные помогают упростить быстрое присоединение к собранию. Однако в сценариях больших собраний персонал поддержки планирует большие собрания с помощью одного набора пользовательских учетных данных, а затем предоставляет URL-адреса для присоединения и аудиоданные инициаторам собраний. В этом случае лучше работает использование разных URL-адресов для присоединения к разным собраниям.
    
- Убедитесь, что флажок **Admit anonymous users by default (Допускать анонимных пользователей по умолчанию)** снят, если только он не требуется в вашем случае. Этот параметр влияет на тип доступа к собранию по умолчанию, запланированный надстройой собрания по сети для Skype для бизнеса, если не используется назначенная конференция. Подходящий параметр для этого параметра зависит от потребностей организации. Если большинство больших собраний в организации составляют внутренние собрания, не следует устанавливать этот флажок. Если в большинстве больших собраний требуется присутствие внешних пользователей, установите этот флажок.
    
Дополнительные сведения о создании политикиконференций см. в под управлением политик в [Skype для бизнеса Server.](../../manage/conferencing/conferencing-policies.md)
