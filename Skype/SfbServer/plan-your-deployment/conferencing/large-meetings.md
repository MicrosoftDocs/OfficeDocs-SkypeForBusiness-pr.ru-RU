# <a name="plan-for-large-meetings-in-skype-for-business-server"></a>Планирование больших собраний в Skype для бизнеса Server
 
**Сводка:** Ознакомьтесь с этой темой, чтобы узнать о наилучших практиках реализации и управления крупными собраниями в Skype для бизнеса Server.
  
Размер собраний, которые Skype для бизнеса Server может поддерживаться, зависит от того, организованы ли конференции в общем или выделенном пуле: от 250 участников в общем пуле до 1000 участников в выделенном пуле. 
  
> [!NOTE]
> В этом разделе основное внимание уделяется лучшим практикам для крупных собраний, поддерживаемых Skype для бизнеса Server. Если организации требуются более крупные возможности для собраний, следует подумать о внедрении гибридной среды, которая будет использовать Skype Meeting Broadcast — новую онлайн-службу, которая является частью Microsoft 365 и Office 365. 

> [!NOTE]
> Skype Трансляция собраний позволяет пользователям проводить и транслировать собрания для большой аудитории в Интернете до 10 000 участников. Использование трансляции Skype собраний требует, чтобы Skype для бизнеса Server была настроена в гибридной установке с Microsoft 365 или Office 365 организации. Все пользователи должны иметь сетевого клиента, установленного в качестве обязательного условия. Если вы заинтересованы в развертывании гибридного решения, которое может использовать Skype трансляцию собраний, см. статью [What is a Skype Meeting Broadcast?](https://go.microsoft.com/fwlink/?LinkId=617071) и Настройка локального развертывания для Skype meeting [Broadcast](../../deploy/configure-skype-meeting-broadcast.md). 
  
Крупные собрания обычно имеют следующие характеристики:
  
- формат собрания: презентация один-ко-многим;
    
- один или несколько пользователей являются докладчиками, а все остальные являются участниками;
    
- совместное использование презентации PowerPoint — это основное действие для совместной работы с данными;
    
- аудио является обязательным, видео также может использоваться;
    
- Специальный человек, как правило, либо организатор собрания, либо помощник организатора, настраивает собрание заблаговременно.
    
- выделенные сотрудники (не докладчики) руководят собранием, в том числе контролируют подключение к собранию по сети, проверка работы аудио, видео и совместного использования слайдов, управляют залом ожидания и ролями пользователей, отключают и включают звук участников, принимают вопросов и управляют записями, по мере необходимости.
    
Когда пользователь запланировать собрание, Skype для бизнеса Server создает запись в базе данных собраний, которая хранит данные о конференциях, но не зарезервировать какие-либо аппаратные ресурсы для запланированного собрания заранее. Вместо этого Skype для бизнеса Server имеет встроенную логику балансировки нагрузки, чтобы динамически распределять ресурсы конференцировки на серверах переднего входа таким образом, чтобы распределять нагрузки одинаково между всеми серверами переднего входа в пул. Это эффективно предусматривает и использует аппаратные ресурсы, но важно правильно планировать поддержку очень больших собраний. 
  
Например, если пул Skype для бизнеса Server, близкий к его верхней емкости, каждый сервер переднего конечного сервера может принимать около 125 собраний среднего размера. Добавление еще одного небольшого собрания не будет проблемой, но добавление собрания для 1000 пользователей будет проблемой, так как серверы переднего входа, вероятно, не смогут поддерживать такое большое собрание одновременно с другими 125 собраниями.
  
Для поддержки больших собраний с числом участников до 1000 требуется решение проблем, связанных как с общей аппаратной моделью, так и с моделью без резервирования. Как правило, необходимо спланировать выделенный пул и следовать лучшим практикам, описанным в следующих разделах. 
  
## <a name="plan-for-a-dedicated-pool"></a>Планирование выделенного пула

Если вашей организации требуются собрания с более чем 250 участниками, необходимо спланировать специальный пул для поддержки нагрузки. 
  
Чтобы иметь достаточные ресурсы ЦП и памяти для собраний до 1000 пользователей, на серверах с передними серверами не должно быть других служб обмена мгновенными сообщениями, а также присутствия или Корпоративная голосовая связь рабочих нагрузок. Серверы также не должны принимать другие собрания, независимо от размера других собраний. Для размещения собраний до 1000 пользователей необходимо создать отдельный пул Skype для бизнеса Server, предназначенный для размещения больших собраний.
  
Пул Skype для бизнеса Server, предназначенный для размещения больших собраний, должен одновременно принимать одну и только одну встречу из 1000 пользователей, поэтому время собраний необходимо заранее зарезервировать с помощью внеплановой процедуры планирования для обеспечения выделенной поддержки от серверов переднего конца. Чтобы одновременно поддерживать несколько крупных собраний, необходимо настроить несколько выделенных пулов больших собраний.
  
Дополнительные сведения о требованиях к оборудованию и программному обеспечению и планировании топологии, которая поддерживает крупные собрания, см. в дополнительных сведениях о требованиях к оборудованию и программному обеспечению для проведения собраний в [Skype для бизнеса Server](hardware-and-software-requirements.md) и планировании топологии для [Skype для бизнеса Server](conferencing-topology.md).
  
## <a name="implement-best-practices-for-large-meetings"></a>Внедрение наилучших методов для крупных собраний

После настройки выделенного пула для больших собраний можно предпринять шаги, чтобы обеспечить наилучшее пользовательское впечатление от крупных собраний, которые будут проводиться в пуле. В следующих разделах указаны рекомендации по управлению крупными собраниями:
  
- Создание организаторов посвященных собраний
    
- Создание выделенных модераторов
    
- Ведение отдельного календаря больших собраний
    
- Реализация процесса планирования больших собраний
    
- Укажите соответствующие сведения о планировании
    
- Создание политики проведения собраний для больших собраний
    
### <a name="create-dedicated-meeting-organizers"></a>Создание организаторов посвященных собраний

Чтобы свести к минимуму трафик сообщений в режиме реального времени в пуле больших собраний, Корпорация Майкрософт не рекомендует принимать пользователей, которые регулярно участвуют в Skype для бизнеса клиентах и участвуют в сеансах обмена мгновенными сообщениями, присутствия, проведения собраний и голосовых сеансов. Вместо этого сделайте одно из следующих:
  
- Создание одной или более выделенных учетных записей пользователей только для планирования больших собраний
    
- Домашние учетные записи пользователей сотрудников, ответственных за планирование больших собраний в пуле больших собраний
    
### <a name="create-dedicated-moderators"></a>Создание выделенных модераторов

С несколькими сотнями до тысячи пользователей на собрании, это хорошая практика, чтобы специальный человек модерировать онлайн-сессии большого собрания. Этот специальный сотрудник может быть делегатом организатора собрания или членом службы поддержки больших собраний организации. При планировании собрания следует добавить этого специального модератора собрания в качестве выступающего, хотя повысить участника до роли выступающего можно прямо во время собрания.
  
Модератор собрания может использовать все функции Skype для бизнеса клиентов для управления большим собранием. Эти функции включают в себя:
  
- Мониторинг лобби и прием или отклонение пользователей в вестибюле
    
- Удаление пользователей из собрания, которые не должны быть на собрании
    
- Изменение типов доступа к собраниям
    
- Изменение ролей участников
    
- Приглашение дополнительных участников во время собрания с помощью функций перетаскивания и сброса, телефонного звонка или электронной почты
    
- Отключение и отключение аудитории или отдельных пользователей
    
- Управление контентом собраний, включая загрузку контента, удаление контента и переключение активного контента

    
### <a name="maintain-a-separate-calendar"></a>Ведение отдельного календаря

Для каждого пула больших собраний необходимо поддерживать отдельный календарь крупных собраний, запланированных в этом пуле. Например, вы можете использовать одну учетную запись пользователя в пуле больших собраний и использовать Outlook с надстройки Exchange и online meeting для Skype для бизнеса для поддержания отдельного календаря. Если вы используете несколько учетных записей пользователей, чтобы включить сотрудников службы поддержки для создания больших собраний, можно настроить отдельный календарь, который агрегируется все крупные собрания, созданные сотрудниками службы поддержки. 
  
Ведение отдельного календаря больших собраний помогает избежать конфликтов и убедиться, что в любой момент времени активно только одно большое собрание.
  
### <a name="implement-a-scheduling-process"></a>Реализация процесса планирования

Так как в выделенном большом пуле собраний поддерживается только одно большое собрание, необходимо реализовать большой процесс планирования собраний, чтобы облегчить настройку больших собраний и предотвратить конфликты. Такая возможность не предоставляется непосредственно Skype для бизнеса Server или Skype для бизнеса клиентами. Одним из способов реализации такого процесса является использование системы билетов группы поддержки вашей организации, если она доступна.
  
Планирование большого собрания предполагает выполнение следующих действий:
  
- Организатор собрания или делегат определяет  время, продолжительность и размер предстоящего собрания, а также список выступающих. Если ожидаемый размер собрания превышает 250 пользователей или если требуется улучшить взаимодействие с пользователями при числе участников меньше 250, организатор или делегат подает запрос на проведение большого собрания.
    
- Специалист по планированию проверяет свободны ли требуемые дата и время. Поскольку мы поддерживаем одновременное проведение только одного большого собрания в выделенном пуле, специалисту по планированию требуется свериться с календарем больших собраний, чтобы определить, не запланировано ли на этот день и это время другое собрание. Если запрошенное время доступно, специалист по планированию утверждает запрос на собрание.
    
- Если запрос утвержден, сотрудники планирования (с использованием учетных данных в выделенном пуле) используют надстройки online meeting для Skype для бизнеса с Outlook, чтобы настроить собрание в выделенном пуле больших собраний. URL-адрес для присоединения к собранию предоставляется инициатору запроса в рамках процедуры утверждения.
    
- Организатор или делегат собрания использует Outlook для расписания предстоящего собрания, добавляя URL-адрес для присоединения к приглашению собрания. После этого организатор собрания или делегат указывает приглашенных пользователей и рассылает приглашение на собрание.
    
### <a name="specify-appropriate-scheduling-details"></a>Укажите соответствующие сведения о планировании

После проверки отсутствия другого запланированного собрания на запрашиваемое время, персонал поддержки, который обрабатывает запрос, планирует собрание в пуле больших собраний. 
  
Для обеспечения наилучшего пользовательского интерфейса важно запланировать большое собрание с правильными уровнями доступа и настройками собраний, соответствующими потребностям организатора собрания. Рассмотрим следующие параметры планирования, настроенные в Skype для бизнеса собрания:
  
- Используйте новое место собрания для каждого большого собрания вместо повторного использования выделенного места собрания. 
    
- Указывайте уровень доступа к собранию следующим образом.
    
  - Если хотя бы один приглашенный является внешним для организации, установите тип доступа к собранию **любому (без ограничений).** Это позволит исключить необходимость управления потенциально большим "залом ожидания" во время проведения собрания.
    
  - Если собрание только внутреннее, установите тип доступа к собранию **Все во организации**.
    
    > [!NOTE]
    > Избегайте устанавливать тип доступа к собранию **Приглашенные пользователи из организации**, поскольку при использовании этого параметра придется добавлять адреса электронной почты всех пользователей в список приглашенных. Кроме того, при использовании этого параметра не работает приглашение групп рассылки. Избегайте устанавливать тип доступа **Только организатор**, поскольку при использовании этого параметра во время проведения собрания в "зал ожидания" должны помещаться все участники собрания, включая выступающих. Лицо, отвечающее за проведение собрания, должно затем постоянно наблюдать за реестром "зала ожидания" и допускать новых пользователей из "зала ожидания".
  
- Разрешите пользователям автоматически присоединяться к собранию путем вызова с телефонного аппарата, включив параметр **Вызывающие абоненты присоединяются напрямую**.
    
- Явным образом приглашайте следующих пользователей:
    
  - организатор собрания и делегат (инициатор запроса);
    
  - список выступающих, предоставленный инициатором запроса на собрание.
    
    > [!NOTE]
    > Если установлен тип доступа к собранию **Выбранные пользователи**, то требуется явно добавлять каждого участника большого собрания как приглашенного на собрание. 
  
- Явно управляйте докладчиками вместо определения параметра докладчика в качестве одного из значений автоматического продвижения. Убедитесь, что следующие пользователи добавлены в качестве докладчиков:
    
  - организатор собрания и делегат (инициатор запроса);
    
  - список выступающих, предоставленный инициаторами запроса на большое собрание.
    
    Явно управляя презентаторами, вы можете ограничить число, достаточное для эффективного большого собрания. Если большинство участников имеет роль участника, это помогает уменьшить вероятность случайного получения контроля над выступлением, удаления презентации PowerPoint, включения или отключения микрофона выступающих и других нарушений в собрании. 
    
- Проверьте параметр **Отключить всех участников**, чтобы было слышно только докладчиков.
    
- Проверьте параметр **видео для** участников блока, чтобы убедиться, что только участники могут транслировать видео на собрание.
    
### <a name="create-a-conferencing-policy"></a>Создание политики конференций

Создайте новую политику проведения собраний специально для больших собраний, а затем назначьте политику собраний пользователям, которые находятся в выделенном пуле больших собраний. Настройте эту политику конференц-связи, используя следующие параметры.
  
- Установите **параметр MaxMeetingSize** до 1000. (По умолчанию — 250.)
    
- Для параметра **AllowLargeMeetings** установите значение **True**. 
    
- Для параметра **EnableAppDesktopSharing** установите значение **None**.
    
- Для параметра **AllowUserToScheduleMeetingsWithAppSharing** установите значение **False**.
    
- Для параметра **AllowSharedNotes** установите значение **False**.
    
- Для параметра **AllowAnnotations** установите значение **False**.
    
- Для параметра **DisablePowerPointAnnotations** установите значение **True**.
    
- Для параметра **AllowMultiview** установите значение **False**.
    
- Для параметра **EnableMultiviewJoin** установите значение **False**.
    
> [!NOTE]
> Поддержка больших собраний в Skype для бизнеса Server требует, чтобы параметр **AllowLargeMeetings** был заметен как true. Если этот параметр заданной, Skype для бизнеса будет оптимизирован для вне больших собраний, когда пользователи присоединяются к собранию. В частности, на большом собрании Skype для бизнеса не будет показываться начальный или обновленный полный список участников собрания, что является узким местом производительности как для клиента, так и для Skype для бизнеса Server. Вместо этого Skype для бизнеса будет показывать только сведения о пользователе и списке участников собрания. Skype для бизнеса будет по-прежнему показывать общее число участников, доступных на больших собраниях.

Параметр **AllowLargeMeetings $true** вызывает следующее:

- Скрывает список участников. 

- Отключает ошибки в окне im.

- Отключает многопартийное видео.

- Отключает возможность продвижения участника в presenter. Необходимо заранее спланировать и объявить всех презентаторов перед собранием.

- Отключает возможность отключать отдельных участников.

- Отключает возможность применения функции Прожектор видео блокировки для участников.

- Набор PSTN у пользователей не сможет отключить себя с помощью 6, так как отсутствует персональная виртуальная помощь, отвечаемая за команды DTMF в активных больших собраниях.

- Если у организатора или устроиста будет запланировать собрание, на котором каждый должен быть приглушен первым ("Отключить все"), пользователи PSTN будут отключены во время вызова и не смогут отметь себя.

За исключением параметра **Maximum meeting size (Максимальный размер собрания)**, все остальные представленные здесь параметры политики конференц-связи требуются для того, чтобы отключить возможности конференц-связи, которые не нужны в больших собраниях.
  
Кроме того, необходимо настроить выделенный пул больших собраний, чтобы каждый Skype для бизнеса Server, который находится в пуле и отвечает за управление расписанием собраний, получил соответствующие разрешения. Для этого выполните следующие действия.
  
- Для параметра **Designate as presenter (Назначить выступающим)** установите значение **None (Нет)**. Обычно только несколько человек (или даже один) среди всех участников больших собраний являются выступающими, поэтому участников не следует автоматически допускать к собраниям в качестве выступающих. Выступающие должны явно назначаться во время планирования собрания или явно выдвигаться во время самого собрания.
    
- Убедитесь, что флажок **Assigned conference type by default (Назначенный тип конференции по умолчанию)** снят. Этот параметр контролирует, всегда ли надстройка online meeting for Skype для бизнеса запланировать конференции с помощью назначенной организатором конференции, что означает, что запланированные собрания имеют одинаковый URL-адрес и аудиозаписи. В сценариях совместной работы небольших групп такой назначенный тип конференции работает хорошо, поскольку у каждого есть своя собственная назначенная конференция, и постоянный URL-адрес присоединения и аудиоданные помогают упростить быстрое присоединение к собранию. Однако в сценариях больших собраний персонал поддержки планирует большие собрания с помощью одного набора пользовательских учетных данных, а затем предоставляет URL-адреса для присоединения и аудиоданные инициаторам собраний. В этом случае лучше работает использование разных URL-адресов для присоединения к разным собраниям.
    
- Убедитесь, что флажок **Admit anonymous users by default (Допускать анонимных пользователей по умолчанию)** снят, если только он не требуется в вашем случае. Этот параметр влияет на тип доступа к собраниям по умолчанию, запланированный надстройка online meeting для Skype для бизнеса при неисполнении назначенной конференции. Подходящий параметр для этого параметра зависит от потребностей организации. Если большинство больших собраний в организации составляют внутренние собрания, не следует устанавливать этот флажок. Если в большинстве больших собраний требуется присутствие внешних пользователей, установите этот флажок.
    
Дополнительные сведения о создании политики conferencing см. в [Skype для бизнеса Server.](../../manage/conferencing/conferencing-policies.md)
