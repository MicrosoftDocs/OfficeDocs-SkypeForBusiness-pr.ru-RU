# <a name="plan-for-conferencing-in-skype-for-business-server"></a>Планирование для conferencing в Skype для бизнеса Server
 
**Сводка:** Ознакомьтесь с этой темой, чтобы узнать о возможностях и возможностях Skype для бизнеса Server.
  
Конференц-конференции в Skype для бизнеса Server позволяет пользователям встречаться и проводить конференции в Интернете с помощью Skype для бизнеса клиента, а не всех участников в одной комнате. Участники собраний могут подключиться к встрече с клиентом Skype для бизнеса для полного аудио- и видеосвязи или набрать номер на конференцию с помощью телефона. Конференции также поддерживают обмен мгновенными сообщениями, обмен настольными компьютерами и приложениями, а также интерактивные белые доски.
  
В этом разделе содержатся следующие разделы:
  
- Функции и возможности конференциинга
    
- Компоненты conferencing
    
- Политики конференц-связи
    
- Поддержка больших собраний
    
- Определение потребностей организации
    
## <a name="conferencing-features-and-capabilities"></a>Функции и возможности конференциинга

В Skype для бизнеса Server доступны четыре типа телефонных Skype для бизнеса Server: веб-конференция, аудио- и видеоконференция (A/V), диалоговые и мгновенные сообщения .. 
  
Вы можете включить все типы конференций или использовать только один тип в зависимости от ваших потребностей. Например, можно включить все типы, в том числе конференц-связи с телефонным звоном, чтобы разрешить пользователям, которые не могут присоединиться к конференции с клиентом Skype для бизнеса для вызова и участия в аудиосъемке собрания с телефона. При развертывании Skype для бизнеса Server автоматически развертываются возможности для конференций im; вы указываете, следует ли развертывать веб-, A/V и диалоговое конференцию с помощью topology Builder. Дополнительные сведения см. в [Skype для бизнеса Server.](../../deploy/deploy-conferencing/deploy-conferencing.md) 
  
В следующих подсекциях описываются функции и возможности каждого типа конференций.
  
### <a name="web-conferencing"></a>Веб-конференции

Веб-конференции позволяют участникам собраний совместно работать над документами, общими во время собрания, а также для того, чтобы участник собрания делился приложениями через Skype для бизнеса клиента. Веб-конференцию предоставляют следующие функции: 
  
- **Доска и аннотации.** Доска — это пустой холст, который может использоваться для совместной работы с использованием текста, рукописного ввода, рисунков и изображений. Примечания, созданные на досках, отображаются для всех участников собрания. Использование доски упрощает совместную работу, позволяя участникам собрания генерировать и обсуждать идеи, делать заметки и выполнять многие другие действия.
    
- **Опрос.** Функция опроса повышает совместную работу, позволяя участникам быстро определять предпочтения участников. Во время сетевых собраний и обсуждений докладчики могут использовать опросы для сбора ответов участников при соблюдении анонимности. Все докладчики могут просматривать результаты, а также скрывать или демонстрировать эти результаты всем участникам.
    
- **Совместное использование приложений и совместное использование настольных компьютеров.** Во время конференции присутствующий на собрании может делиться всем своим настольным компьютером, отдельным приложением или отдельными мониторами в среде с несколькими мониторами. Помимо просмотра контента, другие участники конференции могут запрашивать контроль над экраном презентов и с разрешения взаимодействовать с контентом (включая прокрутку и редактирование). Участники собрания также могут взять на себя участие в качестве презентера и начать совместное использование контента во время собрания.
    
- **PowerPoint Общий доступ.** Позволяет пользователям обмениваться PowerPoint на собрании с помощью сервера Office приложений, что позволяет:
    
  - Отображается и поддерживается поддержка PowerPoint, таких как анимация, переходы на слайды и встроенное видео.
    
  - Мобильные устройства могут получать доступ к этим презентациям.
    
  - Пользователи с соответствующими разрешениями могут прокручивать PowerPoint презентацию независимо от самой презентации. Например, пока Кен представляет слайд-шоу, Хайди может посмотреть любой слайд, на который она хочет, не затрагивая презентацию Кена.
    
### <a name="audio-and-video-conferencing"></a>Аудио- и видеоконференция

Аудио- и видеоконференция позволяет использовать аудио- и видеоконференцию на собрании. Аудио позволяет участникам общаться друг с другом, как будто они находились в одной комнате. Видео позволяет отображать видео в клиенте Skype для бизнеса участников или презентаторов, присоединяющихся к собранию с веб-камерой или устройством конференции, которое поддерживает видео.
  
 Skype для бизнеса Server предоставляет несколько функций, которые пользователи могут использовать для настройки аудиоконференций для пользователя, в том числе следующие:
  
- **Аудитория немая.** Выступающий может использовать этот параметр для отключения микрофонов всех участников аудиоконференции и перевода конференции в состояние, при котором включать микрофоны могут только выступающие.
    
- **Объявления о входе и выходе на конференцию.** Если включена конференц-связь с телефонным подключением, выступающие могут использовать этот параметр для включения или отключения извещений о входе и выходе из конференции, чтобы свести к минимуму отвлекающие моменты во время проведения конференции.
    
- **Добавление пользователя путем набора номера.** Присутствующие и участники, которые получили разрешение, могут добавлять номера PSTN в конференции и иметь телефонный номер конференции к этим номерам.
    
  Skype для бизнеса Server предоставляет несколько функций, которые пользователи могут использовать для настройки интерфейса видеоконференций для пользователя, в том числе следующие:
  
- **Представление галереи.** В видеоконференциях, в которых участвует больше двух пользователей, пользователи автоматически видят всех участников конференции. Если в конференции участвует больше пяти пользователей, видео самых активных участников отображается в верхнем ряду, а для остальных участников отображается только фото. Многостороннее видео включено по умолчанию.
    
- **Панорамное видео.** Если в конференц-зале установлено устройство RoundTable, этот параметр обеспечивает полный круговой обзор конференц-зала. Полоса панорамного видео доступна только при использовании устройств RoundTable.
    
- **Только режим видео.** Выступающие могут настроить собрание так, чтобы отображалось только видео выступающего. Это ограничивает отвлекающие моменты на больших собраниях, когда доступно несколько видеопотоков, которые захватываются из разных источников. Этот режим также применяется к видео, которое захватывается и передается устройствами RoundTable.
    
- **Видео прожектор.** Выступающие могут настроить собрание так, чтобы пользователи видели только видео выбранного участника. Этот режим также применяется к видео, которое захватывается и передается устройствами панорамного видео RoundTable.
    
### <a name="dial-in-conferencing"></a>Конференц-связь с телефонным подключением

Телефонные конференции позволяют участникам собрания присоединяться к аудиозаписи собрания, звоня на собрание с телефона. Конференц-связь с телефонным подключением является поднабором звуковой конференц-связи, нуждающимся в дополнительной настройке. Дополнительные сведения о телефонных конференцах см. в Skype для бизнеса Server Plan [for dial-in conferencing in](dial-in-conferencing.md) [Skype для бизнеса Server.](../../deploy/deploy-conferencing/dial-in-conferencing.md) 
  
### <a name="instant-messaging-conferencing"></a>Conferencing обмена мгновенными сообщениями

Конференция обмена мгновенными сообщениями позволяет более чем двум сторонам общаться в одном сеансе обмена мгновенными сообщениями. Сведения о конференциях обмена мгновенными сообщениями см. в материале [Plan for instant messaging and presence in Skype для бизнеса Server.](../../plan-your-deployment/instant-messaging-and-presence.md)
  
## <a name="conferencing-components"></a>Компоненты conferencing

Компоненты, поддерживаюющие функции конференциинга, включают следующие:
  
- **Служба приложений.** Служба приложений предоставляет платформу для развертывания, размещения и управления приложениями единой связи (UC). Для телефонных конференций используются два приложения UC, которые требуют службы приложения: помощник по конференц-связи и оповещение для конференц-связи. Служба приложений устанавливается и активируется по умолчанию на каждом переднем сервере в пуле переднего конца. Он также устанавливается на каждый выпуск Standard сервер, чтобы включить и настроить диалоговое конференция.
    
- **приложение .** The приложение — это единое приложение для связи, которое принимает вызовы общедоступных переключеных телефонных сетей (PSTN), воспроизводит запросы и присоединяется к вызовам на конференцию A/V. Система приложение устанавливается и активируется по умолчанию при включении телефонных конференций.
    
- **приложение .** The приложение — это единое приложение для связи, которое воспроизводит сигналы и подсказки участникам PSTN по определенным действиям, например, когда участники присоединяются к конференции или покидают конференцию, участники отключены или отключены, кто-то входит в вестибюль конференции или конференция заблокирована или разблокирована. приложение также поддерживает команды двухтонной многочастотной (DTMF) с клавиатуры телефона. Система приложение автоматически устанавливается и активируется по умолчанию при включении телефонных конференций.
    
- **Веб-страница для телефонных Параметры.** На странице Dial-in Conferencing Параметры отображаются номера конференц-связи со своими доступными языками, назначены сведения о конференции (то есть для собраний, которые не требуются) и элементы управления DTMF на конференции, а также поддерживает управление личным идентификационным номером (PIN) и назначенной информацией о конференц-связи. Страница dial-in Conferencing Параметры автоматически устанавливается в составе веб-служб.
    
- **Сервер-посредник и шлюз PSTN.** Для телефонной связи требуется сервер-посредник для перевода сигналов (и мультимедиа в некоторых конфигурациях) между Skype для бизнеса Server и шлюзом PSTN, а также шлюз PSTN для перевода сигналов и мультимедиа между сервером-посредником и шлюзом PSTN. Для телефонных конференций необходимо развернуть по крайней мере один сервер-посредник и по крайней мере один из следующих:
    
  - шлюз ТСОП;
    
  - IP-PBX
    
  - пограничный контроллер сеансов (SBC) для поставщика услуг Интернет-телефонии, подключение к которому выполняется путем настройки канала SIP.
    
  > [!NOTE]
  > Если вы также развертываете Корпоративная голосовая связь, шлюзы-посредники и шлюзы PSTN являются частью Корпоративная голосовая связь развертывания. Если вы не развертываете Корпоративная голосовая связь, необходимо развернуть по крайней мере один сервер-посредник и по крайней мере один шлюз PSTN, IP-PBX или SBC для телефонных конференций. 
  
- **Хранилище файлов.** Хранилище файлов используется для хранения записанных звуковых файлов. Хранилище файлов является стандартным компонентом во всех выпуск Enterprise или выпуск Standard развертывания.
    
- **Пользовательский магазин.** Пользовательский магазин используется для хранения пользовательских Skype для бизнеса Server ПИН-данных. ПИН-коды хэшируются. Хранилище пользователей — стандартный компонент во всех выпуск Enterprise или выпуск Standard развертывания.
    
- **Сервер Office Web Apps.** Чтобы использовать возможности веб-конференций, администраторы должны установить Office сервер веб-приложений и настроить Skype для бизнеса Server для связи с Office web Apps Server.

Можно определить самые разнообразные политики конференций и назначить их отдельным пользователям и группам пользователей. Можно также задавать политики для управления одноранговыми беседами. Подробные сведения о настройке политик конференций см. в материале Управление политиками [конференций в Skype для бизнеса Server](../../manage/conferencing/conferencing-policies.md). Сведения об управлении пропускной способностью см. в [материале Plan for call admission control in Skype для бизнеса Server.](../../plan-your-deployment/enterprise-voice-solution/call-admission-control.md)
  
## <a name="support-for-large-meetings"></a>Поддержка больших собраний

Размер собраний, которые Skype для бизнеса Server может поддерживаться, зависит от того, организованы ли собрания в общем или выделенном пуле:
  
- В общем пуле Skype для бизнеса Server собрания с 250 пользователями. Общий пул — это пул, в котором Skype для бизнеса Server всех рабочих нагрузок, включая мгновенные сообщения и присутствие, конференцию и Корпоративная голосовая связь. 
    
- В выделенном пуле Skype для бизнеса Server могут поддерживать собрания с до 1000 участниками с помощью веб-конференций и аудио- и видеоконференций, в том числе PowerPoint презентаций. Такая поддержка требует выделенный пул, настроенный для поддержки больших собраний и управляемый таким способом, чтобы гарантировать размещение только одного большого собрания в каждый момент времени. 
    
Дополнительные сведения об управлении большими собраниями см. в рублях [Plan for large meetings in Skype для бизнеса Server.](large-meetings.md)
  
Если организации требуются более крупные возможности для собраний, следует рассмотреть вопрос о внедрении гибридной среды, которая будет использовать Skype Meeting Broadcast — онлайн-службу, которая является частью Microsoft 365 и Office 365. Skype Трансляция собраний позволяет пользователям проводить и транслировать собрания для большой аудитории в Интернете до 10 000 участников. Использование трансляции Skype собраний требует, чтобы Skype для бизнеса Server была настроена в гибридной установке с Microsoft 365 или Office 365 организации. Все пользователи должны иметь сетевого клиента, установленного в качестве обязательного условия. Если вы заинтересованы в развертывании гибридного решения, которое может использовать Skype трансляции собраний, см. в статью Настройка локального развертывания для Skype [трансляции собраний.](../../deploy/configure-skype-meeting-broadcast.md)
  
## <a name="determine-your-organizations-needs"></a>Определение потребностей организаций

При определении возможностей конференц-связи, которые следует развернуть, следует учитывать компоненты, доступные пользователям, а также пропускную способность сети. В следующем списке приводится процесс планирования конференций, чтобы определить, какие функции следует развернуть на основе требований организации.
  
> [!NOTE]
> При включив при развертывании конференцию, вы автоматически включаете веб-и A/V-конференцию. Однако можно отключить определенные функции, настроив политики конференций, как описано ранее в этой теме. 
  
- **Предполагается ли разрешить веб-конференции, которые включают совместную работу с документами и общий доступ к приложениям?**
    
    Если это так, необходимо включить конференцию для пула переднего плана с помощью средства планирования или с помощью Topology Builder. Дополнительные сведения см. в [Skype для бизнеса Server.](../../deploy/deploy-conferencing/deploy-conferencing.md)
    
    Общий доступ к приложениям требует и использует гораздо большую часть пропускной способности сети, чем совместная работа с документами. Skype для бизнеса Server механизм регулирования для управления каждым сеансом совместного использования приложений. По умолчанию устанавливается 1,5 КБ/с для каждого сеанса. Если вы не хотите включить совместное использование приложений, но хотите совместной работы с документами, вы можете включить конференцию и использовать политики conferencing для отключения общего доступа к приложениям. Сведения о настройке политик conferencing см. в материале Управление политиками конференций в [Skype для бизнеса Server.](../../manage/conferencing/conferencing-policies.md)
    
    Чтобы пользователи могли обмениваться PowerPoint презентациями, необходимо настроить Office сервер веб-приложений. Подробные сведения о настройке Office сервера веб-приложений см. в материале [Configure integration with Office Web Apps Server в Skype для бизнеса Server.](../../deploy/deploy-conferencing/office-web-app-server.md)
    
- **Вы хотите включить аудио- и видеоконференцию?**
    
    Если это так, необходимо включить конференцию для пула переднего плана с помощью средства планирования или с помощью Topology Builder. Дополнительные сведения см. в [Skype для бизнеса Server.](../../deploy/deploy-conferencing/deploy-conferencing.md)
    
    Аудио- и видеоконференциации требуют и используют больше пропускной способности сети, чем веб-конференциации (включая совместную работу с документами и совместное использование приложений). Если вы не хотите включить аудио- и видеоконференции, но хотите включить веб-конференции, вы можете включить конференц-конференции и использовать политики конференц-конференций для отключения конференций A/V.
    
    Если вы хотите включить аудиоконференции, но не видеоконференции, вы можете включить A/V-конференции и использовать политики конференц-конференций для предотвращения видеоконференции. Кроме того, можно разрешить аудио- и видеоконференции, а затем разрешить инициировать эти конференции или участвовать в них только определенным пользователям. 
    
    Дополнительные сведения о настройке политик conferencing см. в [Skype для бизнеса Server.](../../manage/conferencing/conferencing-policies.md)
    
    > [!NOTE]
    > Корпоративная голосовая связь не требуется использовать A/V-конференцию. Если вы включаете конференц-связи A/V, пользователи могут добавлять звук в свои конференции, если у них есть аудио-устройства, даже если вы используете PBX для вашего телефонного решения. 
  
- **Предполагается ли разрешить пользователям присоединяться к звуковой части конференций с помощью телефона ТСОП?**
    
    Если да, то разверните конференц-связь с телефонным подключением и разрешите ее. После этого приглашенные пользователи, как из организации, так и вне ее, смогут присоединяться к звуковой части конференции с помощью телефона ТСОП.
    
    Диалоговое конференция — это необязательная функция, которую можно настроить при развертывании Skype для бизнеса Server. Хотя для телефонных конференций используются одни и те же компоненты, которые Корпоративная голосовая связь, можно развернуть диалоговое телефонное Корпоративная голосовая связь. Конференц-связь с телефонным подключением поддерживает как корпоративных, так и анонимных пользователей. Дополнительные сведения о настройке телефонных конференций для корпоративных и анонимных пользователей см. в Skype для бизнеса Server [Deploy conferencing in Skype для бизнеса Server](../../deploy/deploy-conferencing/deploy-conferencing.md) и [Configure dial-in conferencing in Skype для бизнеса Server.](../../deploy/deploy-conferencing/dial-in-conferencing.md)
    
- **Вы хотите, чтобы внешние пользователи с Skype для бизнеса клиентами присоединялись к конференциям?**
    
    Позволяя внешнему участию в собраниях, вы увеличиваете инвестиции в Skype для бизнеса Server. К внешним пользователям относятся:
    
  - **Удаленные пользователи.** Собственные пользователи организации, когда они работают вне брандмауэров и используют свои ноутбуки или другие Skype для бизнеса Server устройства.
    
  - **Федераированные пользователи.** Пользователи из компаний, с которыми вы работаете, также Skype для бизнеса Server. Чтобы ваши пользователи могли легко общаться с такими пользователями, с этими компаниями создаются отношения федерации.
    
  - **Анонимные пользователи.** Любые другие внешние пользователи, которые приглашаются вашими пользователями в определенные конференции. Организатор собрания в вашей компании может отправить приглашение на конференцию по электронной почте внешнему пользователю. Сообщение содержит ссылку, по которой внешний пользователь может перейти, чтобы присоединиться к конференции.
    
    Если требуется разрешить внешним пользователям развертывание edge Servers. Кроме того, с развертыванием edge Servers вы можете создавать федератные отношения с другими организациями , такими как ваши клиенты или поставщики, и пользователи из этих организаций могут более легко сотрудничать с пользователями.
    
    Дополнительные сведения о развертывании edge Servers см. в материале Plan for Edge Servers and Deploy Edge Servers. Сведения о том, как включить внешний доступ Office сервера веб-приложений, см. в материале Настройка интеграции с Office web [Apps Server в Skype для бизнеса Server.](../../deploy/deploy-conferencing/office-web-app-server.md)
    
- **Вы хотите управлять клиентами, которые могут присоединиться к Skype для бизнеса Server собраниям?**
    
    Если да, то необходимо настроить страницу присоединения к собраниям, чтобы были доступны только те варианты клиентов, которые вы хотите поддерживать. Каждый раз, когда пользователь щелкает ссылку, чтобы присоединиться к запланированному собранию, Skype для бизнеса Server определяет, установлен ли клиент на компьютере. Затем он открывает клиент по умолчанию и открывает страницу присоединения к собраниям, на которой имеются ссылки на другие клиенты. Страница присоединиться к собранию всегда содержит возможность использования Skype для бизнеса Web App. В дополнение к этому параметру вы можете решить, включать ли ссылки для участника и предыдущие версии Communicator.
