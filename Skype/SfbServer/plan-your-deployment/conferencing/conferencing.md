# <a name="plan-for-conferencing-in-skype-for-business-server"></a>Планирование для работы сконференцией в Skype для бизнеса Server
 
**Сводка:** В этом разделе вы узнаете о возможностях и возможностях в Skype для бизнеса Server.
  
Конференц-конференция в Skype для бизнеса Server позволяет пользователям проводить конференции по сети с помощью клиента Skype для бизнеса, а не всех пользователей, которые вместе работают в одной комнате. Участники собрания могут подключаться к собранию с помощью своего клиента Skype для бизнеса для полноценного аудио- и видеосвязи или подключаться к конференции по телефону. Конференции также поддерживают обмен мгновенными сообщениями, общий доступ к рабочему столу и приложениям, а также интерактивные доски.
  
В этом разделе содержатся следующие разделы:
  
- Функции и возможности для conferencing
    
- Компоненты conferencing
    
- Политики конференц-связи
    
- Поддержка больших собраний
    
- Определение потребностей организации
    
## <a name="conferencing-features-and-capabilities"></a>Функции и возможности для conferencing

В Skype для бизнеса Server доступно четыре типа аудио- и видеоконференций, а также аудио- и видеоконференций. 
  
Вы можете включить все типы или использовать только один тип в зависимости от ваших потребностей. Например, можно включить все типы, включая конференц-связь с телефонным доступом, чтобы разрешить пользователям, которые не могут присоединиться к конференции с клиентом Skype для бизнеса, звонить и участвовать в собрании по телефону. При развертывании Skype для бизнеса Server автоматически развертываются возможности im conferencing; вы указываете, следует ли развертывать веб-, видео- и видеоконференцию и построитель топологий. Дополнительные сведения см. в теме ["Развертывание conferencing в Skype для бизнеса Server".](../../deploy/deploy-conferencing/deploy-conferencing.md) 
  
Ниже описаны функции и возможности каждого типа услуг.
  
### <a name="web-conferencing"></a>Веб-конференции

С помощью веб-конференций участники собрания могут совместно работать над документами, общими во время собрания, а также обмениваться приложениями с помощью клиента Skype для бизнеса. Веб-трансляции предоставляют следующие возможности: 
  
- **Доска и аннотации.** Доска — это пустой холст, который может использоваться для совместной работы с использованием текста, рукописного ввода, рисунков и изображений. Примечания, созданные на досках, отображаются для всех участников собрания. Использование доски упрощает совместную работу, позволяя участникам собрания генерировать и обсуждать идеи, делать заметки и выполнять многие другие действия.
    
- **Опрос.** Функция опроса улучшает совместную работу, позволяя участникам быстро определять предпочтения участников. Во время сетевых собраний и обсуждений докладчики могут использовать опросы для сбора ответов участников при соблюдении анонимности. Все докладчики могут просматривать результаты, а также скрывать или демонстрировать эти результаты всем участникам.
    
- **Общий доступ к приложениям и общий доступ к рабочему столу.** Во время конференции вы можете поделиться своим рабочим столом, отдельным приложением или отдельными мониторами в среде с несколькими мониторами. Помимо простого просмотра содержимого, другие участники конференции могут запрашивать управление экраном участника и, с разрешения, взаимодействовать с содержимым (включая прокрутку и редактирование). Участники собрания также могут взять на себя себя в качестве участника и начать общий доступ к содержимому во время собрания.
    
- **Общий доступ к PowerPoint.** Позволяет пользователям делиться презентациями PowerPoint на собрании через сервер Office Web Apps, что позволяет:
    
  - Отображение в высоком разрешении и поддержка возможностей PowerPoint, таких как анимация, переходы слайдов и внедренное видео.
    
  - Мобильные устройства могут получать доступ к этим презентациям.
    
  - Пользователи с соответствующими разрешениями могут прокручивать презентацию PowerPoint независимо от самой презентации. Например, пока Кен показывает слайд-шоу, Он может посмотреть на любой слайд, на который он хочет, не влияя на презентацию Кена.
    
### <a name="audio-and-video-conferencing"></a>Аудио- и видеоконференция

Аудио- и видеоконференция позволяет использовать аудио- и видеосвязь в собрании. Звук позволяет участникам общаться друг с другом, как будто они находились в одной комнате. Видео позволяет отображать видео в клиенте Skype для бизнеса всех участников или участников, которые присоединяются к собранию с помощью веб-камеры или устройства для проведения конференций, которое поддерживает видео.
  
 Skype для бизнеса Server предоставляет несколько функций, которые пользователи могут использовать для настройки аудиоконференций для пользователя, в том числе следующие:
  
- **Отключить звук аудитории.** Выступающий может использовать этот параметр для отключения микрофонов всех участников аудиоконференции и перевода конференции в состояние, при котором включать микрофоны могут только выступающие.
    
- **Объявления о входе и выходе из conferencing.** Если включена конференц-связь с телефонным подключением, выступающие могут использовать этот параметр для включения или отключения извещений о входе и выходе из конференции, чтобы свести к минимуму отвлекающие моменты во время проведения конференции.
    
- **Добавление пользователя путем набора номера.** Presenters and attendees that have been given permission, can add PSTN numbers to the conferences and have the conference dial-out to those numbers.
    
  Skype для бизнеса Server предоставляет несколько функций, которые пользователи могут использовать для настройки видеоконференций для пользователя, в том числе следующие:
  
- **Представление галереи.** В видеоконференциях, в которых участвует больше двух пользователей, пользователи автоматически видят всех участников конференции. Если в конференции участвует больше пяти пользователей, видео самых активных участников отображается в верхнем ряду, а для остальных участников отображается только фото. Многостороннее видео включено по умолчанию.
    
- **Панорамное видео.** Если в конференц-зале установлено устройство RoundTable, этот параметр обеспечивает полный круговой обзор конференц-зала. Полоса панорамного видео доступна только при использовании устройств RoundTable.
    
- **Режим видео только для presenter.** Выступающие могут настроить собрание так, чтобы отображалось только видео выступающего. Это ограничивает отвлекающие моменты на больших собраниях, когда доступно несколько видеопотоков, которые захватываются из разных источников. Этот режим также применяется к видео, которое захватывается и передается устройствами RoundTable.
    
- **Видео в центре внимания.** Выступающие могут настроить собрание так, чтобы пользователи видели только видео выбранного участника. Этот режим также применяется к видео, которое захватывается и передается устройствами панорамного видео RoundTable.
    
### <a name="dial-in-conferencing"></a>Конференц-связь с телефонным подключением

Аудиоконференция позволяет участникам собрания присоединяться к звуковой части собрания, звоня на собрание с телефона. Конференц-связь с телефонным подключением является поднабором звуковой конференц-связи, нуждающимся в дополнительной настройке. For more information about dial-in conferencing, see [Plan for dial-in conferencing in Skype for Business Server](dial-in-conferencing.md) and [Configure dial-in conferencing in Skype for Business Server](../../deploy/deploy-conferencing/dial-in-conferencing.md).

Обмен мгновенными сообщениями позволяет более чем двум сторонам общаться в одном сеансе обмена мгновенными сообщениями. For details about IM conferencing, see [Plan for instant messaging and presence in Skype for Business Server.](../../plan-your-deployment/instant-messaging-and-presence.md)
  
## <a name="conferencing-components"></a>Компоненты conferencing

К компонентам, поддерживаюным функции, относятся следующие:
  
- **Служба приложений.** Служба приложений предоставляет платформу для развертывания, размещения приложений объединенных коммуникаций и управления им. В этом случае используются два приложения объединенных объединенных служб, для работы с которые требуется служба приложений: помощник по конференциям и извещание о конференциях. Служба приложений устанавливается и активируется по умолчанию на каждом сервере переднего сервера в пуле переднего сервера. Кроме того, он устанавливается на каждом сервере Standard Edition, чтобы включить и настроить комконференцию с помощью телефонного номера.
    
- **Приложение помощника по conferencing.** Приложение помощника по конференц-связи — это приложение объединенных коммуникаций, которое принимает вызовы телефонной сети общего звонков (PSTN), воспроизводит подсказки и присоединяет вызовы к видео- и видеоконференции. Приложение помощника по предоставлению телефонных номеров устанавливается и активируется по умолчанию при в включенной телефонной комконференции.
    
- **Приложение объявлений для conferencing.** Приложение "Извещание конференц-связи" — это приложение объединенных коммуникаций, которое воспроизводит звукные сигналы и подсказки для участников PSTN при определенных действиях, например, когда участники присоединяются к конференции или покидают ее, участники отключаются или отключаются, кто-то входит в "зал" конференции, или конференция заблокирована или разблокирована. Приложение объявлений для видеоконференций также поддерживает команды двухтонального многочастотного (DTMF) с клавиатуры телефона. Приложение "Извещание для видеоконференций" автоматически устанавливается и активируется по умолчанию при включаемой телефонной комконференции.
    
- **Страница параметров телефонной сети.** На странице "Параметры конференц-связи с телефонным доступом" отображаются телефонные номера конференц-связи с доступными языками, назначенная информация о конференциях (т. е. для собраний, которые не требуется запланить) и элементы управления DTMF в конференции, а также поддерживается управление персональным идентификационным номером (PIN-кодом) и назначенной информацией о конференц-связи. Страница "Параметры телефонного номера" автоматически устанавливается как часть веб-служб.
    
- **Сервер-посредник и шлюз STN.** Для телефонной связи требуется сервер-посредник для перевода сигналов (и мультимедиа в некоторых конфигурациях) между Skype для бизнеса Server и шлюзом STN, а также шлюз STN для перевода сигналов и мультимедиа между сервером-посредником и шлюзом STN. Для телефонной службы необходимо развернуть хотя бы один сервер-посредник и хотя бы один из следующих серверов:
    
  - шлюз ТСОП;
    
  - IP-PBX
    
  - пограничный контроллер сеансов (SBC) для поставщика услуг Интернет-телефонии, подключение к которому выполняется путем настройки канала SIP.
    
  > [!NOTE]
  > Если вы также развертываете Корпоративная голосовая связь, сервер-посредник и шлюзы STN являются частью Корпоративная голосовая связь развертывания. Если развертывание Корпоративная голосовая связь не требуется, необходимо развернуть по крайней мере один сервер-посредник и хотя бы один шлюз PSTN, IP-PBX или SBC для телефонной комференации. 
  
- **Хранилище файлов.** Хранилище файлов используется для хранения записанных звуковых файлов. Хранилище файлов является стандартным компонентом в каждом развертывании Enterprise Edition или Standard Edition.
    
- **Хранилище пользователей.** Хранилище пользователей используется для хранения ПИН-данных пользователей Skype для бизнеса Server. ПИН-коды хэшируются. Хранилище пользователей является стандартным компонентом в каждом развертывании Enterprise Edition или Standard Edition.
    
- **Сервер Office Web Apps.** Чтобы использовать возможности веб-конференций, администраторы должны установить сервер Office Web Apps и настроить Skype для бизнеса Server для связи с сервером Office Web Apps.
    
## <a name="conferencing-policies"></a>Политики конференц-связи

Чтобы применить политики организации и контролировать использование пропускной способности, можно настроить политики для типов собраний, которые пользователи могут организовывать. Можно определить широкий спектр политик и назначить их отдельным пользователям и группам пользователей. Можно также задавать политики для управления одноранговыми беседами. For details about setting conferencing policies, see [Manage conferencing policies in Skype for Business Server.](../../manage/conferencing/conferencing-policies.md) For details about bandwidth management, see [Plan for call admission control in Skype for Business Server.](../../plan-your-deployment/enterprise-voice-solution/call-admission-control.md)
  
## <a name="support-for-large-meetings"></a>Поддержка больших собраний

Размер собраний, поддерживаемый Skype для бизнеса Server, зависит от того, находится ли она в общем или выделенном пуле:
  
- В общем пуле Skype для бизнеса Server может проводить собрания с до 250 пользователями. Общий пул — это пул, в котором размещены все рабочие нагрузки Skype для бизнеса Server, включая обмен мгновенными сообщениями и присутствие, а также Корпоративная голосовая связь. 
    
- В выделенном пуле Skype для бизнеса Server может поддерживать собрания с числом участников до 1000, используя веб-конференции и аудио- и видеоконференцию, включая общий доступ к презентациям PowerPoint. Такая поддержка требует выделенный пул, настроенный для поддержки больших собраний и управляемый таким способом, чтобы гарантировать размещение только одного большого собрания в каждый момент времени. 
    
Дополнительные сведения об управлении большими собраниями см. в сведениях о планировании больших собраний [в Skype для бизнеса Server.](large-meetings.md)
  
Если вашей организации требуются более крупные возможности собраний, следует рассмотреть возможность реализации гибридной среды, которая будет использовать трансляцию собраний Skype, веб-службу, которая входит в состав Microsoft 365 и Office 365. Трансляция собраний Skype позволяет пользователям проводить и транслировать собрания для больших аудиторий в Интернете до 10 000 участников. Для использования трансляции собраний Skype необходимо, чтобы Skype для бизнеса Server уже был настроен в гибридной конфигурации с производственной организацией Microsoft 365 или Office 365. Все пользователи должны иметь сетевой клиент, установленный в качестве обязательного условия. Если вы заинтересованы в развертывании гибридного решения, которое может использовать трансляцию собраний Skype, см. статью "Настройка локального развертывания для трансляции [собраний Skype".](../../deploy/configure-skype-meeting-broadcast.md)
  
## <a name="determine-your-organizations-needs"></a>Определение потребностей организации

При определении возможностей конференц-связи, которые следует развернуть, следует учитывать компоненты, доступные пользователям, а также пропускную способность сети. В следующем списке представлены руководства по планированию, чтобы определить, какие функции следует развернуть в соответствии с требованиями организации.
  
> [!NOTE]
> При включаемой в развертывании видео- и видеоконференциях автоматически включается как веб-, так и видеоконференция. Однако вы можете отключить определенные функции, настроив политики, описанные выше в этой теме. 
  
- **Предполагается ли разрешить веб-конференции, которые включают совместную работу с документами и общий доступ к приложениям?**
    
    В этом случае необходимо включить для пула переднего плана видеоконференцию с помощью средства планирования или построитель топологий. Дополнительные сведения см. в теме ["Развертывание conferencing в Skype для бизнеса Server".](../../deploy/deploy-conferencing/deploy-conferencing.md)
    
    Общий доступ к приложениям требует и использует гораздо большую часть пропускной способности сети, чем совместная работа с документами. Skype для бизнеса Server предоставляет механизм регулирования для управления каждым сеансом общего доступа к приложениям. По умолчанию устанавливается 1,5 КБ/с для каждого сеанса. Если вы не хотите включить общий доступ к приложениям, но хотите совместной работы с документами, вы можете включить совместную работу с документами и использовать политики для отключения общего доступа к приложениям. For details about configuring conferencing policies, see [Manage conferencing policies in Skype for Business Server.](../../manage/conferencing/conferencing-policies.md)
    
    Чтобы пользователи могли делиться презентациями PowerPoint, необходимо настроить сервер Office Web Apps. Подробные сведения о настройке сервера Office Web Apps см. в сведениях о настройке интеграции с [сервером Office Web Apps в Skype для бизнеса Server.](../../deploy/deploy-conferencing/office-web-app-server.md)
    
- **Вы хотите включить аудио- и видеоконференцию?**
    
    В этом случае необходимо включить для пула переднего плана видеоконференцию с помощью средства планирования или построитель топологий. Дополнительные сведения см. в теме ["Развертывание conferencing в Skype для бизнеса Server".](../../deploy/deploy-conferencing/deploy-conferencing.md)
    
    Для аудио- и видеоконференций требуется и используется более высокая пропускная способность сети, чем для веб-аудиоконференций (включая совместную работу с документами и общий доступ к приложениям). Если вы не хотите включить аудио- и видеоконференции, но хотите включить веб-конференции, вы можете включить конференц-конференцию и использовать политики конференц-залов для отключения аудио- и видеоконференции.
    
    Если вы хотите включить аудиоконференции, но не видеоконференции, вы можете включить аудио- и видеоконференции и использовать политики конференц-залов для предотвращения видеоконференции. Кроме того, можно разрешить аудио- и видеоконференции, а затем разрешить инициировать эти конференции или участвовать в них только определенным пользователям. 
    
    Дополнительные сведения о настройке политикконференций см. в под управлением политик в [Skype для бизнеса Server.](../../manage/conferencing/conferencing-policies.md)
    
    > [!NOTE]
    > Корпоративная голосовая связь не требуется использовать видео- и видеоконференцию. Если вы включаете аудио- и видеоконференции, пользователи могут добавлять звуковые файлы в свои конференции, если у них есть звуковые устройства, даже если для телефонного решения используется УАПС. 
  
- **Предполагается ли разрешить пользователям присоединяться к звуковой части конференций с помощью телефона ТСОП?**
    
    Если да, то разверните конференц-связь с телефонным подключением и разрешите ее. После этого приглашенные пользователи, как из организации, так и вне ее, смогут присоединяться к звуковой части конференции с помощью телефона ТСОП.
    
    Эта функция является дополнительной функцией, которую можно настроить при развертывании функции "Skype для бизнеса Server". Несмотря на то, что в этой теме используются некоторые из тех же компонентов, Корпоративная голосовая связь, можно развернуть ее даже в том случае, если она не развернута Корпоративная голосовая связь. Конференц-связь с телефонным подключением поддерживает как корпоративных, так и анонимных пользователей. Дополнительные сведения о настройке звонков с комконференцией для корпоративных и анонимных пользователей см. в подсетях ["Развертывание conferencing in Skype for Business Server"](../../deploy/deploy-conferencing/deploy-conferencing.md) и ["Configure dial-in conferencing in Skype for Business Server".](../../deploy/deploy-conferencing/dial-in-conferencing.md)
    
- **Хотите ли вы позволить внешним пользователям с клиентами Skype для бизнеса присоединяться к конференциям?**
    
    Разрешив внешнее участие в собраниях, вы можете увеличить инвестиции в Skype для бизнеса Server. К внешним пользователям относятся:
    
  - **Удаленные пользователи.** Пользователи вашей организации, работающие за пределами брандмауэра и использующие ноутбуки или другие устройства Skype для бизнеса Server.
    
  - **Федератерные пользователи.** Пользователи из компаний, с которыми вы работаете, также запускают Skype для бизнеса Server. Чтобы ваши пользователи могли легко общаться с такими пользователями, с этими компаниями создаются отношения федерации.
    
  - **Анонимные пользователи.** Любые другие внешние пользователи, которые приглашаются вашими пользователями в определенные конференции. Организатор собрания в вашей компании может отправить приглашение на конференцию по электронной почте внешнему пользователю. Сообщение содержит ссылку, по которой внешний пользователь может перейти, чтобы присоединиться к конференции.
    
    Если вы хотите разрешить внешним пользователям, необходимо развернуть edge Servers. Кроме того, с помощью развернутых серверов можно создавать федераированные отношения с другими организациями, такими как ваши клиенты или поставщики, и пользователи из этих организаций могут легко сотрудничать с вашими пользователями.
    
    Дополнительные сведения о развертывании edge servers см. в сведениях о планировании для edge servers и deploy Edge Servers. Подробные сведения о включаемом внешнем доступе для сервера Office Web Apps см. в сведениях о настройке интеграции с [сервером Office Web Apps в Skype для бизнеса Server.](../../deploy/deploy-conferencing/office-web-app-server.md)
    
- **Хотите управлять клиентами, которые могут присоединяться к собраниям Skype для бизнеса Server?**
    
    Если да, то необходимо настроить страницу присоединения к собраниям, чтобы были доступны только те варианты клиентов, которые вы хотите поддерживать. Каждый раз, когда пользователь щелкает ссылку, чтобы присоединиться к запланированному собранию, Skype для бизнеса Server определяет, установлен ли клиент на компьютере. Затем он открывает клиент по умолчанию и открывает страницу присоединения к собраниям, на которой имеются ссылки на другие клиенты. На странице присоединить к собранию всегда есть возможность использовать Skype для бизнеса Web App. В дополнение к этому параметру можно решить, следует ли включать ссылки для участника и предыдущих версий Communicator.
