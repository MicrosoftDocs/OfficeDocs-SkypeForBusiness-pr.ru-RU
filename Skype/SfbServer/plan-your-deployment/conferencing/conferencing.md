# <a name="plan-for-conferencing-in-skype-for-business-server"></a>Планирование конференц-связи в Skype для бизнеса Server
 
**Сводка:** В этом разделе рассказывается о функциях и возможностях конференций в Skype для бизнеса Server.
  
Конференц-связь в Skype для бизнеса Server позволяет пользователям выполнять и удерживать конференции через Интернет, используя клиент Skype для бизнеса, а не все вместе в одной комнате. Участники собрания могут подключаться к собранию с помощью клиента Skype для бизнеса для полноценного звука и видео, а также для подключения к Конференции по телефону. Конференции также поддерживают обмен мгновенными сообщениями, общий доступ к рабочим столам и приложениям, а также интерактивным белой плате.
  
В этом разделе содержатся следующие разделы:
  
- Функции и возможности конференц-связи
    
- Компоненты конференц-связи
    
- Политики конференц-связи
    
- Поддержка больших собраний
    
- Определение потребностей Организации
    
## <a name="conferencing-features-and-capabilities"></a>Функции и возможности конференц-связи

В Skype для бизнеса Server доступно четыре типа конференций: веб-конференции, конференции аудио-и видеоконференций (A/V), Конференц-связь с телефонным подключением и конференц-связь с помощью мгновенных сообщений. 
  
В зависимости от ваших потребностей можно включить все типы конференций или использовать только один тип. Например, вы можете включить все типы, в том числе Конференц-связь с телефонным подключением, чтобы разрешить пользователям, которые не могут присоединиться к Конференции с помощью клиента Skype для бизнеса, позвонить и участвовать в звуковом собрании по телефону. При развертывании Skype для бизнеса Server возможности конференций для обмена мгновенными сообщениями развертываются автоматически; Вы можете указать, следует ли развертывать веб-, аудио-и видеоконференции с помощью построителя топологий. Дополнительные сведения см в разделе [deploy конференций в Skype для бизнеса Server](../../deploy/deploy-conferencing/deploy-conferencing.md). 
  
В следующих подразделах описываются функции и возможности каждого типа конференц-связи.
  
### <a name="web-conferencing"></a>Веб-конференции

Веб-конференции позволяют участникам собрания совместно работать над документами на собрании, а для докладчика собраний доступ к приложениям через клиент Skype для бизнеса. Веб-конференции предоставляют следующие возможности: 
  
- **Доска и заметки.** Доска — это пустой холст, который может использоваться для совместной работы с использованием текста, рукописного ввода, рисунков и изображений. Примечания, созданные на досках, отображаются для всех участников собрания. Использование доски упрощает совместную работу, позволяя участникам собрания генерировать и обсуждать идеи, делать заметки и выполнять многие другие действия.
    
- **Опроса.** Функция опроса повышает совместную работу, позволяя докладчикам быстро определять предпочтения участников. Во время сетевых собраний и обсуждений докладчики могут использовать опросы для сбора ответов участников при соблюдении анонимности. Все докладчики могут просматривать результаты, а также скрывать или демонстрировать эти результаты всем участникам.
    
- **Общий доступ к приложениям и общий доступ к рабочему столу.** Во время проведения Конференции докладчик может совместно использовать весь Настольный компьютер, отдельное приложение или отдельные мониторы в среде с несколькими мониторами. Помимо просмотра контента, другие участники Конференции могут запрашивать управление экраном докладчика и, при наличии разрешения, взаимодействовать с содержимым (включая прокрутку и редактирование). Участники собрания также могут принять участие в выступающих и начать совместное использование контента во время собрания.
    
- **Общий доступ к PowerPoint.** Позволяет пользователям предоставлять общий доступ к презентациям PowerPoint на собрании через сервер Office Web Apps, что позволяет:
    
  - Отображение высокого разрешения и поддержка возможностей PowerPoint, таких как анимация, переходы слайдов и встроенное видео.
    
  - Мобильные устройства могут получать доступ к этим презентациям.
    
  - Пользователи с соответствующими разрешениями могут прокручивать презентацию PowerPoint независимо от самой презентации. Например, в то время как Ken просматривает слайд-шоу, Хеиди может просматривать любой слайд, который он хочет, не влияя на презентацию Ken.
    
### <a name="audio-and-video-conferencing"></a>Аудио-и видеоконференции

Аудио-и видеоконференции позволяют использовать звук и видео в собрании. Звук позволяет участникам общаться друг с другом, как будто они находятся в одной комнате. Видео позволяет отображать видео в клиенте Skype для бизнеса любому участнику или докладчикам, которые присоединяются к собранию с помощью устройства с веб-камеры или конференц-связи, поддерживающего видео.
  
 Skype для бизнеса Server предоставляет несколько функций, которые пользователи могут использовать для настройки взаимодействия голосовой связи для пользователя, включая следующие:
  
- **Выключение аудитории.** Выступающий может использовать этот параметр для отключения микрофонов всех участников аудиоконференции и перевода конференции в состояние, при котором включать микрофоны могут только выступающие.
    
- **Извещения входа/выхода конференц-связи.** Если включена конференц-связь с телефонным подключением, выступающие могут использовать этот параметр для включения или отключения извещений о входе и выходе из конференции, чтобы свести к минимуму отвлекающие моменты во время проведения конференции.
    
- **Добавление пользователя при исходящих звонках.** Выступающие и участники, которым предоставлено разрешение, могут добавлять в конференции номера PSTN и исключать Конференц-связь по этим номерам.
    
  Skype для бизнеса Server предоставляет несколько функций, которые пользователи могут использовать для настройки взаимодействия видеоконференций для пользователя, включая следующие:
  
- **Представление галереи.** В видеоконференциях, в которых участвует больше двух пользователей, пользователи автоматически видят всех участников конференции. Если в конференции участвует больше пяти пользователей, видео самых активных участников отображается в верхнем ряду, а для остальных участников отображается только фото. Многостороннее видео включено по умолчанию.
    
- **Панорамное видео.** Если в конференц-зале установлено устройство RoundTable, этот параметр обеспечивает полный круговой обзор конференц-зала. Полоса панорамного видео доступна только при использовании устройств RoundTable.
    
- **Видеорежим "только для докладчика".** Выступающие могут настроить собрание так, чтобы отображалось только видео выступающего. Это ограничивает отвлекающие моменты на больших собраниях, когда доступно несколько видеопотоков, которые захватываются из разных источников. Этот режим также применяется к видео, которое захватывается и передается устройствами RoundTable.
    
- **Видео в центре внимания.** Выступающие могут настроить собрание так, чтобы пользователи видели только видео выбранного участника. Этот режим также применяется к видео, которое захватывается и передается устройствами панорамного видео RoundTable.
    
### <a name="dial-in-conferencing"></a>Конференц-связь с телефонным подключением

Конференц-связь с телефонным подключением позволяет участникам собрания присоединяться к звуковой части собрания, вызывая абонента на собрание с телефона. Конференц-связь с телефонным подключением является поднабором звуковой конференц-связи, нуждающимся в дополнительной настройке. Для получения дополнительных сведений о конференц-связи с телефонным подключением [в разделе Планирование конференц-связи с телефонным подключением в Skype для бизнеса Server](dial-in-conferencing.md) и [Настройка конференц-связи с телефонным подключением в Skype для бизнеса Server](../../deploy/deploy-conferencing/dial-in-conferencing.md). 
  
### <a name="instant-messaging-conferencing"></a>Конференц-связь с мгновенными сообщениями

Конференц-связь с обменом мгновенными сообщениями позволяет более двух Сторон обмениваться сообщениями в едином сеансе обмена мгновенными сообщениями. Подробные сведения о конференц-связи можно найти [в статье Планирование обмена мгновенными сообщениями и сведений о присутствии в Skype для бизнеса Server](../../plan-your-deployment/instant-messaging-and-presence.md).
  
## <a name="conferencing-components"></a>Компоненты конференц-связи

Ниже перечислены компоненты, поддерживающие функции конференц-связи.
  
- **Служба приложения.** Служба приложения предоставляет платформу для развертывания приложений Объединенных коммуникаций и управления ими, а также для управления ими. Конференц-связь с телефонным подключением использует два приложения UC, для которых требуется служба приложений: помощник по конференц-связи и извещение конференц-связи. Служба приложения устанавливается и активируется по умолчанию на каждом сервере переднего плана в интерфейсном пуле. Он также устанавливается на каждом сервере Standard Edition для включения и настройки конференц-связи с телефонным подключением.
    
- **Приложение "помощник по конференц-связи".** Приложение "помощник по конференц-связи" — это единое приложение для обмена данными, которое принимает вызовы по телефонной сети общего пользования (PSTN), воспроизводит приглашения и присоединяет вызовы к Конференции аудио-и видеосвязи. Приложение-помощник по конференц-связи устанавливается и активируется по умолчанию при включении конференц-связи с телефонным подключением.
    
- **Приложение извещений конференц-связи.** Приложение извещений конференц-связи — это единое приложение для общения, которое воспроизводит звуки и запрашивает участников PSTN в определенных действиях, например, когда участники присоединяются к Конференции или отправляются из нее, если участники присоединяются к Конференции или не отключены или не заблокированы. Приложение для конференц-связи с конференц-связью поддерживает также команды с двумя тонами (DTMF) на клавиатуре телефона. Приложение извещений конференц-связи автоматически устанавливается и активируется по умолчанию при включении конференц-связи с телефонным подключением.
    
- **Страница параметров конференц-связи с телефонным подключением.** На странице параметров конференц-связи с телефонным подключением отображаются номера подключений к конференц-связи с доступными языками, назначены сведения о конференциях (то есть для собраний, которые не нуждаются в планировании), а также входящие в систему управления DTMF и поддерживают управление личным идентификационным номером (ПИН-кодом) и назначенной информацией о конференциях. Страница параметров конференц-связи с телефонным подключением устанавливается автоматически как часть веб-служб.
    
- **Сервер-посредник и шлюз PSTN.** Для конференц-связи с телефонным подключением требуется сервер-посредник для перевода сигналов (и носителей в некоторых конфигурациях) между Skype для бизнеса Server и шлюзом PSTN, а шлюз PSTN для перевода сигналов и мультимедиа между сервером-посредником и шлюзом PSTN. Для конференц-связи с телефонным подключением необходимо развернуть по крайней мере один сервер-посредник и хотя бы один из следующих компонентов:
    
  - шлюз ТСОП;
    
  - IP-УАТС
    
  - пограничный контроллер сеансов (SBC) для поставщика услуг Интернет-телефонии, подключение к которому выполняется путем настройки канала SIP.
    
  > [!NOTE]
  > Если вы также выполняете развертывание корпоративной голосовой связи, сервер-посредник и шлюзы PSTN являются частью развертывания корпоративной голосовой связи. Если вы не развертываете корпоративную голосовую связь, необходимо развернуть по крайней мере один сервер-посредник и по крайней мере один шлюз PSTN, IP-УАТС или SBC для конференц-связи с телефонным подключением. 
  
- **Хранилище файлов.** Хранилище файлов используется для хранения записанных звуковых файлов. Хранилище файлов является стандартным компонентом в каждом развертывании Enterprise Edition или Standard Edition.
    
- **Пользовательское хранилище.** Хранилище пользователей используется для хранения ПИН-кодов пользователя Skype для бизнеса Server. ПИН-коды хэшируются. Хранилище пользователей является стандартным компонентом в каждом развертывании Enterprise Edition или Standard Edition.
    
- **Сервер Office Web Apps.** Чтобы использовать возможности веб-конференций, администраторы должны установить сервер Office Web Apps и настроить Skype для бизнеса Server для связи с сервером Office Web Apps.
    
## <a name="conferencing-policies"></a>Политики конференц-связи

Для применения политик организации и контроля использования полосы пропускания можно задать политики для типов собраний, которые могут упорядочивать пользователи. Вы можете определить разнообразные политики конференц-связи и назначить их отдельным пользователям и группам пользователей. Можно также задавать политики для управления одноранговыми беседами. Подробнее о настройке политик конференц-связи можно узнать [в статье Управление политиками конференц-связи в Skype для бизнеса Server](../../manage/conferencing/conferencing-policies.md). Дополнительные сведения об управлении пропускной способностью приведены [в статье Plan for контроля допуска звонков в Skype для бизнеса Server](../../plan-your-deployment/enterprise-voice-solution/call-admission-control.md).
  
## <a name="support-for-large-meetings"></a>Поддержка больших собраний

Размер собраний, поддерживаемых Skype для бизнеса Server, зависит от того, размещается ли Конференц-связь в общем или выделенном пуле:
  
- В общем пуле Skype для бизнеса Server может размещать собрания до 250 пользователей. Общий пул — это пул, в котором размещаются все рабочие нагрузки Skype для бизнеса Server, в том числе обмен мгновенными сообщениями, сведения о присутствии, конференц-связи и корпоративной голосовой связи. 
    
- В выделенном пуле Skype для бизнеса Server может поддерживать собрания с 1000 участниками, использующими веб-и аудио-и видеоконференции (A/V), включая общий доступ к презентациям PowerPoint. Такая поддержка требует выделенный пул, настроенный для поддержки больших собраний и управляемый таким способом, чтобы гарантировать размещение только одного большого собрания в каждый момент времени. 
    
Для получения дополнительных сведений об управлении большим количеством собраний, ознакомьтесь со статьей [планирование больших собраний в Skype для бизнеса Server](large-meetings.md).
  
Если вашей организации требуются более крупные возможности для собраний, рекомендуется реализовать гибридную среду, которая использует трансляцию собраний Skype, веб-службу, которая входит в состав Microsoft 365 и Office 365. Трансляция собраний Skype позволяет пользователям размещать и транслировать собрания для крупных интерактивных аудиторий до 10 000 участников. Для использования вещания собраний Skype необходимо, чтобы Skype для бизнеса Server уже был настроен в гибридной настройке с рабочей организацией Microsoft 365 или Office 365. Все пользователи должны иметь интерактивный клиент, установленный в качестве необходимого компонента. Если вы заинтересованы в развертывании гибридного решения, в котором можно использовать трансляцию собраний Skype, ознакомьтесь со статьей [Настройка локального развертывания для трансляции собраний Skype](../../deploy/configure-skype-meeting-broadcast.md).
  
## <a name="determine-your-organizations-needs"></a>Определение потребностей Организации

При определении возможностей конференц-связи, которые следует развернуть, следует учитывать компоненты, доступные пользователям, а также пропускную способность сети. В приведенном ниже списке описано, какие функции конференц-связи следует развертывать в соответствии с требованиями Организации.
  
> [!NOTE]
> Когда вы включаете Конференц-связь при развертывании, вы автоматически включаете как веб-, так и аудио-и видеоконференции. Тем не менее, вы можете отключить определенные функции, настроив политики конференц-связи, как описано выше в этой статье. 
  
- **Предполагается ли разрешить веб-конференции, которые включают совместную работу с документами и общий доступ к приложениям?**
    
    В этом случае необходимо включить Конференц-связь для интерфейсного пула с помощью средства планирования или построителя топологий. Дополнительные сведения см в разделе [deploy конференций в Skype для бизнеса Server](../../deploy/deploy-conferencing/deploy-conferencing.md).
    
    Общий доступ к приложениям требует и использует гораздо большую часть пропускной способности сети, чем совместная работа с документами. Skype для бизнеса Server предоставляет механизм регулирования для управления каждым сеансом общего доступа к приложениям. По умолчанию устанавливается 1,5 КБ/с для каждого сеанса. Если вы не хотите включать общий доступ к приложениям, но при этом требуется совместная работа с документами, можно включить Конференц-связь и использовать политики конференц-связи, чтобы отключить общий доступ к приложениям. Дополнительные сведения о настройке политик конференц-связи можно найти [в статье Управление политиками конференц-связи в Skype для бизнеса Server](../../manage/conferencing/conferencing-policies.md).
    
    Чтобы предоставить пользователям доступ к презентациям PowerPoint, необходимо настроить сервер Office Web Apps. Дополнительные сведения о настройке сервера Office Web Apps приведены [в статье Настройка интеграции с сервером Office Web Apps в Skype для бизнеса Server](../../deploy/deploy-conferencing/office-web-app-server.md).
    
- **Вы хотите включить голосовые и видеоконференции?**
    
    В этом случае необходимо включить Конференц-связь для интерфейсного пула с помощью средства планирования или построителя топологий. Дополнительные сведения см в разделе [deploy конференций в Skype для бизнеса Server](../../deploy/deploy-conferencing/deploy-conferencing.md).
    
    Для голосовой и видеоконференций требуется больше полосы пропускания сети, чем для веб-конференций (включая совместную работу с документами и общий доступ к приложениям). Если вы не хотите включать голосовые и видеоконференции, но вы хотите включить веб-конференции, вы можете включить Конференц-связь и использовать политики конференц-связи для отключения аудио-и видеоконференций.
    
    Если вы хотите включить аудио-и видеоконференций, вы можете включить аудио-и видеоконференции и использовать политики конференц-связи для предотвращения видеоконференций. Кроме того, можно разрешить аудио- и видеоконференции, а затем разрешить инициировать эти конференции или участвовать в них только определенным пользователям. 
    
    Дополнительные сведения о настройке политик конференц-связи можно найти [в статье Управление политиками конференц-связи в Skype для бизнеса Server](../../manage/conferencing/conferencing-policies.md).
    
    > [!NOTE]
    > Корпоративная голосовая связь не требуется для использования аудио-и видеоконференций. Если вы включаете аудио-и видеоконференции, пользователи могут добавлять звук в свои конференции, если у них есть звуковые устройства, даже если вы используете УАТС для своего телефонного решения. 
  
- **Предполагается ли разрешить пользователям присоединяться к звуковой части конференций с помощью телефона ТСОП?**
    
    Если да, то разверните конференц-связь с телефонным подключением и разрешите ее. После этого приглашенные пользователи, как из организации, так и вне ее, смогут присоединяться к звуковой части конференции с помощью телефона ТСОП.
    
    Конференц-связь с телефонным подключением это дополнительный компонент, который можно настроить при развертывании конференций Skype для бизнеса Server. Несмотря на то, что в конференц-связи с телефонным подключением используются некоторые компоненты корпоративной голосовой связи, можно развернуть Конференц-связь с телефонным подключением, даже если вы не развертываете корпоративную голосовую связь. Конференц-связь с телефонным подключением поддерживает как корпоративных, так и анонимных пользователей. Для получения дополнительных сведений о настройке конференц-связи с телефонным подключением для корпоративных и анонимных пользователей, ознакомьтесь со статьей [развертывание конференций в Skype для бизнеса Server](../../deploy/deploy-conferencing/deploy-conferencing.md) и [Настройка конференц-связи с телефонным подключением в Skype для бизнеса Server](../../deploy/deploy-conferencing/dial-in-conferencing.md).
    
- **Вы хотите разрешить внешним пользователям с клиентами Skype для бизнеса присоединяться к конференциям?**
    
    Разрешив внешнее участие в собраниях, вы максимально максимизируем свои инвестиции в Skype для бизнеса Server. К внешним пользователям относятся:
    
  - **Удаленные пользователи.** Собственные пользователи вашей организации, работающие за пределами брандмауэров и использующие их ноутбуки или другие устройства Skype для бизнеса Server.
    
  - **Федеративные пользователи.** Пользователи из компаний, которые также работают с Skype для бизнеса Server. Чтобы ваши пользователи могли легко общаться с такими пользователями, с этими компаниями создаются отношения федерации.
    
  - **Анонимные пользователи.** Любые другие внешние пользователи, которые приглашаются вашими пользователями в определенные конференции. Организатор собрания в вашей компании может отправить приглашение на конференцию по электронной почте внешнему пользователю. Сообщение содержит ссылку, по которой внешний пользователь может перейти, чтобы присоединиться к конференции.
    
    Если вы хотите разрешить внешним пользователям, вам потребуется развернуть пограничные серверы. Кроме того, при развертывании пограничных серверов можно создавать федеративные отношения с другими организациями, такими как клиенты или поставщики, и пользователи из этих организаций могут более легко сотрудничать с пользователями.
    
    Дополнительные сведения о развертывании пограничных серверов приведены в статье Планирование пограничных серверов и развертывание пограничных серверов. Подробнее о том, как включить внешний доступ к серверу Office Web Apps, можно узнать [в статье Настройка интеграции с сервером Office Web Apps в Skype для бизнеса Server](../../deploy/deploy-conferencing/office-web-app-server.md).
    
- **Хотите контролировать клиенты, которые могут присоединиться к собраниям Skype для бизнеса Server?**
    
    Если да, то необходимо настроить страницу присоединения к собраниям, чтобы были доступны только те варианты клиентов, которые вы хотите поддерживать. Каждый раз, когда пользователь щелкает ссылку для присоединения к запланированному собранию, Skype для бизнеса Server обнаруживает, что на компьютере уже установлен клиент. Затем он открывает клиент по умолчанию и открывает страницу присоединения к собраниям, на которой имеются ссылки на другие клиенты. Страница присоединения к собранию всегда содержит вариант использования Skype для бизнеса Web App. Кроме того, вы можете указать, следует ли включать ссылки для участников и предыдущих версий Communicator.
