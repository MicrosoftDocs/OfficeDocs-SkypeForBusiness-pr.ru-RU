# <a name="plan-for-skype-for-business-in-vdi-environments"></a>Планирование Skype для бизнеса в средах VDI
 
В этом разделе обсуждаются вопросы планирования использования Skype для бизнеса при подключении к удаленному виртуальному рабочему столу. 
  
Среда инфраструктуры виртуальных рабочих столов (VDI) используется в некоторых организациях, где вопросы безопасности и соответствия особенно конфиденциальны. Их пользователи выполняют свою работу на виртуальном рабочем столе со всеми настольными приложениями и файлами, использующими службы удаленных рабочих столов, или аналогичным удаленному подключению. Использование Skype для бизнеса с полным звуком и видео для подключения, например, требует высокой загрузки аудио-и видеообработки на клиенте, размещенном на виртуальном рабочем столе. Доступно дополнительное программное обеспечение для подключаемого модуля VDI, которое размещает обработку на локальном компьютере конечного пользователя и сокращает нагрузку на виртуальный рабочий стол.
  
Существует три решения для компонента подключаемого модуля VDI, предлагаемые корпорацией Майкрософт, Citrix или VMWare. Для новых развертываний Корпорация Майкрософт рекомендует использовать решение Citrix HDX Optimization Pack или пакет виртуализации VMWare. Исходный подключаемый модуль Lync VDI по-прежнему поддерживается в оставшейся части жизненного цикла.
  
- **Подключаемый модуль VDI для Lync** был разработан для Lync 2013 и совместим с клиентом Lync 2013 или Skype для бизнеса 2015, запущенным на виртуальном рабочем столе. Это автономное приложение, устанавливаемое на локальном компьютере и позволяющее использовать локальные аудио и видеоустройства с клиентом на виртуальном рабочем столе. Для подключаемого модуля не требуется установка клиента Skype для бизнеса на локальном компьютере или тонком клиенте, на котором должны работать операционные системы Windows 7, Windows 8 или Windows Server 2008. (Тонкие клиентские устройства, использующие эти операционные системы и поддерживаемые корпорацией Майкрософт: Dell Wyse Z90D7, Dell Wyse R90L7, Dell Wyse X90m7, HP T610 и HP t5740e.) Этот подключаемый модуль по-прежнему поддерживается, но будущие обновления не планируется. Для виртуальных сред на основе Citrix рекомендуется использовать пакет оптимизации в режиме реального времени.
    
- **Пакет оптимизации в режиме реального времени** строится на подключаемом модуле Lync VDI и работает с клиентами Lync 2013 или Skype для бизнеса 2016 на виртуальном рабочем столе. Она была разработана компанией Citrix и Майкрософт для усовершенствования исходного подключаемого модуля VDI. Его можно установить на клиентах с операционными системами Windows и другими операционными системами (включая Windows 10, Mac и Linux). Он состоит из двух компонентов: разъема реального времени (установленный на виртуальном рабочем столе) и подсистемы реального времени (установленной на локальном компьютере конечного пользователя). Эти два компонента позволяют локальному компьютеру пользователя использовать клиент Skype для бизнеса, запущенный на виртуальном рабочем столе, с помощью обработки аудио-и видеоданных на локальном компьютере. Для сред виртуальных рабочих столов на основе Citrix рекомендуются пакеты оптимизации в режиме реального времени, а также запланирована дополнительная поддержка.
    
- **Пакет виртуализации Погоризонта VMware** для Skype для бизнеса, разработанный при совместной работе с VMware, позволяет доставлять Skype для бизнеса на виртуальном рабочем столе, в то же время обеспечивая отличное взаимодействие с пользователем. Решение использует модуль мультимедиа на клиенте для создания оптимизированного решения с конечной точкой клиента, обеспечивающей функции разгрузки мультимедиа для аудио и видеовызовов. Это решение, которое может доставлять аудио-и видеоданные непосредственно между конечными точками для совместной работы одной системы или разгрузить в центральном блоке управления MultiPoint (MCU) для многосторонних конференц-звонков или собраний.
    
> [!NOTE]
> Клиенты Skype для бизнеса Basic не поддерживаются с пакетом оптимизации HDX в режиме реального времени или пакетом виртуализации VMWare.

## <a name="microsofts-lync-vdi-plug-in"></a>Подключаемый модуль VDI для Microsoft Lync
<a name="Citrix_RT"> </a>

С помощью подключаемого модуля Microsoft Lync VDI пользователь должен находиться на компьютере с Windows или на тонком клиенте и иметь установленный подключаемый модуль VDI Microsoft Lync для обработки аудио-и видеопотоков из клиента на виртуальном рабочем столе. Пользователь может:
  
1. Подключите аудио-и видеоустройство (например, наушники или камера) к локальному компьютеру.
    
2. Подключитесь к удаленному виртуальному рабочему столу с клиентом Lync 2013 или Skype для бизнеса 2015.
    
3. Введите учетные данные для Skype для бизнеса на виртуальном рабочем столе.
    
4. Повторно введите учетные данные пользователя, чтобы установить соединение с подключаемым модулем Lync VDI на локальном компьютере Windows или тонком клиенте.
    
После установки подключения пользователь готов к выполнению и приему аудио-и видеовызовов. Трафик в сети и загрузка виртуального рабочего стола минимизированы, так как локальный компьютер обрабатывает обработку аудио-и видеосвязи.
  
Подключаемый модуль Microsoft Lync VDI поддерживается только в некоторых операционных системах Windows и поддерживает только клиенты Lync 2013 или Skype для бизнеса 2015. В этой статье приведены [Поддерживаемые технологии виртуализации и известные ограничения](vdi-environments.md#Supported_virt) для получения дополнительных сведений о поддерживаемых технологиях и ограничениях.
  
Для получения дополнительных сведений ознакомьтесь со следующими ссылками:

- [Предварительные требования для подключаемого модуля VDI Lync](vdi-environments.md#VDI_prereq)

- Статья [CTX138408](https://support.citrix.com/article/CTX138408) по центру знаний Citrix

### <a name="supported-virtualization-technologies-and-known-limitations"></a>Поддерживаемые технологии виртуализации и известные ограничения
<a name="Supported_virt"> </a>

Подключаемый модуль VDI для Lync позволяет осуществлять аудио и видео звонки для поддерживаемых технологий виртуализации. В соответствии со стандартными правилами для телефонов также включена поддержка E911. В следующих разделах описаны технологии виртуализации, поддерживаемые подключаемым модулем Lync VDI и известные ограничения функций.
  
#### <a name="support-for-virtualization-technologies"></a>Поддержка технологий виртуализации

Подключаемый модуль VDI для Lync поддерживает полный набор удаленных сеансов в сценарии личного виртуального рабочего стола, но не в сценарии подключения к удаленному рабочему столу. Эти сценарии можно описать следующим образом:
  
- **Поддерживается: настроенные виртуальные рабочие столы или инфраструктура виртуальных рабочих столов (VDI).** В этом сценарии каждый пользователь входит в настраиваемый виртуальный рабочий стол и может сохранять файлы на настольном компьютере, которые хранятся в разных сеансах. Примеры реализации служб удаленного рабочего стола (Майкрософт) и горизонта VMware, которые были проверены для использования с Skype для бизнеса 2015. К другим реализациям, которые проходят проверку, относится Citrix XenDesktop. Сведения о средах VDI и клиентских аппаратных средствах, протестированных корпорацией Майкрософт, представлены в разделе [инфраструктура, квалифицированная для Microsoft Lync](https://go.microsoft.com/fwlink/?LinkID=313435).
    
- **Не поддерживается: сеансы удаленного рабочего стола.** В этом сценарии каждый пользователь выполняет вход в общий сеанс виртуального рабочего стола, который не может быть изменен. Примеры включают сеансы удаленного рабочего стола Майкрософт (узлов сеансов удаленных рабочих столов) и Citrix XenApp в сочетании с получателем Citrix.
    
Подключаемый модуль VDI для Lync не поддерживает другие технологии виртуализации, такие как виртуализация приложений, что позволяет использовать приложение без необходимости установки полного приложения локально. Примеры реализации включают Citrix XenApp и Microsoft Application Virtualization (App-V). Потоки приложений, удаленное взаимодействие приложений и смешанные режимы виртуализации (например, удаленное взаимодействие приложений на полном удаленном рабочем месте) не поддерживаются.
  
Подключаемый модуль VDI для Lync предназначен для использования независимых от платформы API, называемых динамическими виртуальными каналами (DVC). Для сценариев, которые не поддерживаются явно, ознакомьтесь с инструкциями по поддержке от поставщика решений VDI.
  
#### <a name="lync-vdi-plug-in-prerequisites"></a>Предварительные требования для подключаемого модуля VDI Lync
<a name="VDI_prereq"> </a>

В среде VDI Виртуальные машины и локальные компьютеры пользователей должны соответствовать требованиям, описанным в этом разделе.
  
> [!NOTE]
>  Поставщик решения виртуализации может предоставить сведения об установке и развертывании среды. Общие сведения о развертывании виртуализованной среды на основе служб Hyper и удаленного рабочего стола представлены в следующих статьях библиотеки Майкрософт: [Hyper – V](https://go.microsoft.com/fwlink/p/?linkid=247514), [службы удаленных рабочих столов в Windows Server 2008 R2](https://go.microsoft.com/fwlink/p/?linkid=247513) 
  
Виртуальные машины должны быть настроены для Windows 8, Windows 7 или Windows Server 2008 R2 с последними пакетами обновления.
  
Локальный компьютер пользователя должен удовлетворять следующим требованиям:
  
- Пользователь должен быть размещен в Skype для бизнеса Server или Lync Server 2013.
    
- Локальный компьютер должен работать под управлением Windows Embedded Standard 7 с пакетом обновления 1 (SP1), Windows 7 с пакетом обновления 1 (SP1) или Windows 8.
    
- Если вы используете службы удаленных рабочих столов, выберите подключаемый модуль VDI для 32 бит или 64 – бит, соответствующий операционной системе локального компьютера. Для локального компьютера и виртуальной машины не требуется наличие 32 или 64 разрядов. Если вы используете другое решение или платформу виртуализации, ознакомьтесь с требованиями поставщика.
    
- На локальном компьютере должна быть установлена [Последняя версия клиента удаленного рабочего стола](https://go.microsoft.com/fwlink/p/?LinkId=268032). Установите последние обновления клиента служб удаленного рабочего стола корпорации Майкрософт или последнюю версию клиента удаленного рабочего стола поставщика решения виртуализации. 
    
- На локальном компьютере параметры клиента удаленного рабочего стола должны быть настроены таким образом, чтобы звук воспроизводился на локальном компьютере, а удаленная запись отключена. Чтобы настроить эти параметры для подключения к удаленному рабочему столу в Windows, ознакомьтесь со статьей "Настройка параметров подключения к удаленному рабочему столу" в следующем разделе. 
    
Подключаемый модуль Microsoft VDI доступен в подключаемом модуле Microsoft [LYNC vdi 2013 plugin (32 бит)](https://www.microsoft.com/download/details.aspx?id=35457) или [microsoft Lync vdi 2013 (64-бит)](https://www.microsoft.com/download/details.aspx?id=35454).
  
#### <a name="known-feature-limitations"></a>Ограничения известных компонентов
<a name="VDI_prereq"> </a>

Ниже приведены известные ограничения при использовании клиента Skype для бизнеса 2015 в среде VDI.
  
Существует ограниченная поддержка функций делегирования вызовов и анонимного агента группы ответа.
  
Следующие функции не поддерживаются:
  
- интегрированные страницы настройки аудио- и видеоустройств;
    
- Видео с несколькими представлениями.
    
- запись бесед;
    
- Анонимное присоединение к собраниям (то есть присоединяться к собраниям Skype для бизнеса, размещенным в Организации, которая не поддерживает эту организацию).
    
- Использование подключаемого модуля VDI для Lync и устройства Lync Phone Edition.
    
- непрерывность звонка при сбое сети;
    
- Пользовательские мелодии звонка и функции хранения музыки.
    
Подключаемый модуль VDI для Lync не поддерживается в средах Microsoft 365 или Office 365.
  
> [!NOTE]
> Пакет оптимизации в режиме реального времени поддерживает Microsoft 365 и Office 365. В случае виртуальных сред на основе Citrix ознакомьтесь с документацией по [техническим обзорам](https://docs.citrix.com/en-us/hdx-optimization/2-0/hdx-realtime-optimization-pack-about.mdl) Citrix, чтобы получить список поддерживаемых функций и версий.
