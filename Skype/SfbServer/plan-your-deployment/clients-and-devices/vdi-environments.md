# <a name="plan-for-skype-for-business-in-vdi-environments"></a>Планирование Skype для бизнеса в средах VDI
 
В этом разделе рассматриваются соображения планирования использования Skype для бизнеса при подключении к удаленному виртуальному рабочему столу. 
  
Среда виртуальной инфраструктуры настольных компьютеров (VDI) используется в некоторых организациях, где вопросы безопасности и соответствия требованиям являются особенно чувствительными. Их пользователи работают на виртуальном рабочем столе со всеми своими настольными приложениями и файлами с помощью служб удаленного рабочего стола или аналогичного удаленного подключения. Использование Skype для бизнеса с полным звуком и видео на подключении, которое требует тяжелых нагрузок обработки звука и видео на клиенте, размещенного на виртуальном рабочем столе. Доступно дополнительное программное обеспечение подключаемого программного обеспечения VDI, которое разгружает обработку локальной машины пользователя и снижает нагрузку на виртуальный рабочий стол.
  
Для компонента плагина VDI доступны три решения, предлагаемые Корпорацией Майкрософт, Citrix или VMWare. В новых развертываниях Корпорация Майкрософт рекомендует использовать решение пакета оптимизации реального времени Citrix HDX или пакет виртуализации VMWare Horizon. Исходный VDI-модуль Lync по-прежнему поддерживается до конца жизненного цикла.
  
- Плагин **Lync VDI** был разработан для Lync 2013 и совместим с клиентом Lync 2013 или Skype для бизнеса 2015, работающим на виртуальном рабочем столе. Это отдельное приложение, которое устанавливается на локальном компьютере и позволяет использовать локальные аудио- и видео-устройства с клиентом на виртуальном рабочем столе. Подключаемый модуль не требует установки клиента Skype для бизнеса на локальном компьютере или тонком клиенте, который должен запускать операционные системы Windows 7, Windows 8 или Windows Server 2008. (Тонкие клиентские устройства, использующие эти операционные системы и поддерживаемые Корпорацией Майкрософт, включают: Dell Wyse Z90D7, Dell Wyse R90L7, Dell Wyse X90m7, HP t610 и HP t5740e.) Этот подключаемый модуль по-прежнему поддерживается, но никаких будущих обновлений не планируется. Для виртуальных сред на основе Citrix рекомендуется использовать пакет оптимизации реального времени Citrix.
    
- Пакет **оптимизации citrix RealTime** строится на плагине Lync VDI и работает с клиентами Lync 2013 или Skype для бизнеса 2016 на виртуальном рабочем столе. Она была совместно разработана Citrix и Microsoft для улучшения исходного плагина VDI. Она может быть установлена на клиентов с операционными системами Windows и без Windows (включая Windows 10, Mac и Linux). Он состоит из двух компонентов: соединительная система RealTime (установленная на виртуальном рабочем столе) и Media Engine RealTime (установленная на локальной машине конечного пользователя). Эти два компонента позволяют локальному компьютеру пользователя использовать клиент Skype для бизнеса, работающий на виртуальном рабочем столе с обработкой A/V, перемещенной на локальный компьютер. Для виртуальных настольных сред на основе Citrix рекомендуется пакет оптимизации citrix RealTime, и планируется дальнейшая поддержка.
    
- Пакет **Виртуализации VMWare Horizon** для Skype для бизнеса, разработанный в сотрудничестве с VMWare, позволяет доставить Skype для бизнеса на виртуальном рабочем столе при одновременной доставке отличного пользовательского интерфейса. Решение работает с помощью средства массовой информации в клиенте для создания оптимизированного решения, а клиентская конечная точка предоставляет возможности разгрузки мультимедиа для аудио- и видеозвонков. Это решение, которое может доставить звук и видео непосредственно между конечными точками для совместной работы один на один или разгрузить его в центральный блок управления несколькими точками (MCU) для многопартийных конференц-звонков или собраний.
    
> [!NOTE]
> Клиенты Skype для бизнеса Basic не поддерживаются пакетом оптимизации citrix HDX в реальном времени или пакетом виртуализации VMWare Horizon.

## <a name="microsofts-lync-vdi-plug-in"></a>Подключаемый модуль Lync VDI от Microsoft
<a name="Citrix_RT"> </a>

С помощью решения плагина Microsoft Lync VDI пользователь должен быть на компьютере Windows или тонком клиенте и установить плагин Lync VDI от Microsoft для обработки аудио- и видеопотоков клиента на виртуальном рабочем столе. Пользователь будет:
  
1. Подключение аудио- и видео устройства (например, гарнитуры или камеры) к локальному компьютеру.
    
2. Подключение к удаленному виртуальному рабочему столу с клиентом Lync 2013 или Skype для бизнеса 2015.
    
3. Введите учетные данные для Skype для бизнеса на виртуальном рабочем столе.
    
4. Повторно введите учетные данные пользователей, чтобы установить подключение к плагину Lync VDI на локальном компьютере Windows или тонком клиенте.
    
После подключения пользователь готов к звукозаписывам и видеозвонам. Трафик в сети и нагрузка на виртуальный рабочий стол сведены к минимуму, так как локальный компьютер обрабатывает обработку звука и видео.
  
Плагин Microsoft Lync VDI поддерживается только в определенных операционных системах Windows и поддерживает только клиентов Lync 2013 или Skype для бизнеса 2015. Дополнительные сведения о поддерживаемых [технологиях](vdi-environments.md#Supported_virt) и ограничениях см. в дополнительных сведениях о поддерживаемых технологиях и ограничениях.

### <a name="supported-virtualization-technologies-and-known-limitations"></a>Поддерживаемые технологии виртуализации и известные ограничения
<a name="Supported_virt"> </a>

Плагин Lync VDI позволяет аудио- и видеозвонки для поддерживаемых технологий виртуализации. В соответствии со стандартными телефонными правилами также включена поддержка E911. В следующих разделах описываются технологии виртуализации, поддерживаемые плагином Lync VDI, и известные ограничения функций.
  
#### <a name="support-for-virtualization-technologies"></a>Поддержка технологий виртуализации

Подключаемый модуль Lync VDI поддерживает удаленные сеансы полного рабочего стола в сценарии личного виртуального рабочего стола, но не в сценарии сеанса удаленного рабочего стола. Эти сценарии можно описать следующим образом:
  
- **Поддерживается: персонализированные виртуальные настольные компьютеры или инфраструктура виртуальных настольных компьютеров (VDI).** В этом сценарии каждый пользователь входит в настраиваемый виртуальный рабочий стол и может сохранять файлы на рабочем столе, которые сохраняются во время сеансов. Microsoft Remote Desktop Services и VMware Horizon View — это примеры реализации, протестированная для использования в Skype для бизнеса 2015 г. Другие реализации, проходящие проверку, включают Citrix XenDesktop. Сведения о средах VDI, определенных для поставщиков, и клиентских аппаратных средствах, которые были протестированы Корпорацией Майкрософт, см. в рублях [Infrastructure Lync.](../../../SfbPartnerCertification/lync-cert/qualified-ip-pbx-gateway.md)
    
- **Не поддерживается: сеансы удаленного рабочего стола.** В этом сценарии каждый пользователь входит в общий сеанс виртуального рабочего стола, который невозможно настроить. В качестве примеров можно привести microsoft Remote Desktop Sessions (RDSH) и Citrix XenApp в сочетании с приемником Citrix.
    
Плагин Lync VDI не поддерживает другие технологии виртуализации, например виртуализацию приложений, что позволяет использовать приложение без необходимости локальной установки полного приложения. Примерами реализации являются Citrix XenApp и Microsoft Application Virtualization (App-V). Режимы потоковой передачи приложений, перенаправки приложений и смешанных режимов виртуализации (например, перезаправка приложений в полном объеме на рабочем столе) не поддерживаются.
  
Плагин VDI Lync был разработан для использования API, независимых от платформы, под названием Dynamic Virtual Channels (DVCs). Для сценариев, которые явно не поддерживаются, обратитесь к утверждениям поддержки от поставщика решений VDI.
  
#### <a name="lync-vdi-plug-in-prerequisites"></a>Предварительные условия подключаемого подключения Lync VDI
<a name="VDI_prereq"> </a>

В среде VDI виртуальные машины и локальный компьютер пользователя должны соответствовать требованиям, описанным в этом разделе.
  
> [!NOTE]
>  Поставщик решений для виртуализации может предоставить сведения об установке и развертывании среды. Общие сведения о развертывании виртуализированной среды на основе служб Hyper-V и удаленных настольных компьютеров см. в следующих статьях в Библиотеке Майкрософт: [Hyper-V](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc753637(v=ws.10)), Удаленные службы настольных компьютеров [в Windows Server 2008 R2](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/dd736539(v=ws.10)) 
  
Виртуальные машины должны быть настроены с Windows 8, Windows 7 или Windows Server 2008 R2 с последними пакетами служб.
  
Локальный компьютер пользователя должен соответствовать следующим требованиям:
  
- Пользователь должен быть домашним в Skype для бизнеса Server или Lync Server 2013.
    
- На локальном компьютере должна работать windows Embedded Standard 7 с SP1, Windows 7 с SP1 или Windows 8.
    
- Если вы используете службы удаленного рабочего стола, выберите 32-битный или 64-битный подключаемый модуль Lync VDI, чтобы соответствовать операционной системе локального компьютера. Для локального компьютера и виртуальной машины не требуется иметь 32-битные или 64-битные операционные системы. Если вы используете другое решение или платформу виртуализации, обратитесь к требованиям поставщика.
    
- На локальном компьютере должна быть запущена [последняя версия удаленного настольного клиента.](/windows-server/remote/remote-desktop-services/clients/remote-desktop-clients) Установите последние обновления клиента служб удаленного рабочего стола корпорации Майкрософт или последнюю версию клиента удаленного рабочего стола поставщика решения виртуализации. 
    
- На локальном компьютере необходимо настроить параметры удаленного клиента для настольных компьютеров, чтобы отключить воспроизведение звука на локальном компьютере и отключить удаленную запись. Чтобы настроить эти параметры для удаленного подключения к рабочему столу в Windows, см. в следующем разделе "Настройка параметров удаленного подключения к рабочему столу". 
    
Плагин Microsoft VDI доступен в [плагине Microsoft Lync VDI 2013 (32 бита)](https://www.microsoft.com/download/details.aspx?id=35457) или [плагине Microsoft Lync VDI 2013 (64 бита).](https://www.microsoft.com/download/details.aspx?id=35454)
  
#### <a name="known-feature-limitations"></a>Известные ограничения функций
<a name="VDI_prereq"> </a>

Известны следующие ограничения при использовании клиента Skype для бизнеса 2015 в среде VDI:
  
Существует ограниченная поддержка функций анонимизации агента группы вызовов и группы ответов.
  
Следующие функции не поддерживаются:
  
- интегрированные страницы настройки аудио- и видеоустройств;
    
- Видео с несколькими представлениями.
    
- запись бесед;
    
- Присоединение к собраниям анонимно (то есть присоединение к собраниям Skype для бизнеса, которые организованы организацией, не федеращей с вашей организацией).
    
- Использование плагина Lync VDI вместе с устройством Lync Phone Edition.
    
- непрерывность звонка при сбое сети;
    
- Настраиваемые мелодии и функции музыки на удержании.
    
Плагин VDI Lync не поддерживается в средах Microsoft 365 или Office 365.
  
> [!NOTE]
> Пакет оптимизации citrix RealTime поддерживает Microsoft 365 и Office 365. В виртуальных средах на основе Citrix [](https://docs.citrix.com/en-us/hdx-optimization/2-0/hdx-realtime-optimization-pack-about.mdl) просмотрите документацию По техническому обзору Citrix для списка поддерживаемых функций и версий.
