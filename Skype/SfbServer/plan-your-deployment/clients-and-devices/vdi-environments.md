# <a name="plan-for-skype-for-business-in-vdi-environments"></a>Планирование Skype для бизнеса в средах VDI
 
В этом разделе рассматриваются вопросы планирования использования Skype для бизнеса при подключении к удаленному виртуальному рабочему столу. 
  
Среда инфраструктуры виртуальных рабочих стола (VDI) используется в некоторых организациях, где вопросы безопасности и соответствия требованиям являются особенно конфиденциальными. Пользователи работают на виртуальном рабочем столе со всеми своими классическими приложениями и файлами с помощью служб удаленных рабочих стола или аналогичного удаленного подключения. Использование Skype для бизнеса с полным аудио- и видеосвязью для подключения, которое требует большой нагрузки обработки звука и видео на клиенте, размещенного на виртуальном рабочем столе. Доступно дополнительное подключаемое программное обеспечение VDI, которое разгружает обработку на локальный компьютер пользователя и снижает нагрузку на виртуальный рабочий стол.
  
Существует три решения для подключаемого компонента VDI, предлагаемых Корпорацией Майкрософт, Citrix или VMWare. В новых развертываниях Корпорация Майкрософт рекомендует использовать решение Citrix HDX RealTime Optimization Pack или VMWare Horizon Virtualization Pack. Исходный подключаемый модуль VDI Lync по-прежнему поддерживается в течение оставшегося срока его жизненного цикла.
  
- Подключаемый модуль **VDI Lync** разработан для Lync 2013 и совместим с клиентом Lync 2013 или Skype для бизнеса 2015, работающим на виртуальном рабочем столе. Это отдельное приложение, которое устанавливается на локальном компьютере и позволяет использовать локальные аудио- и видео устройства с клиентом на виртуальном рабочем столе. Подключаемый модуль не требует установки клиента Skype для бизнеса на локальном компьютере или тонкого клиента, на котором должны работать операционные системы Windows 7, Windows 8 или Windows Server 2008. (Тонкие клиентские устройства, использующие эти операционные системы и поддерживаемые корпорацией Майкрософт: Dell Wyse Z90D7, Dell Wyse R90L7, Dell Wyse X90m7, HP t610 и HP t5740e.) Этот подключаемый модуль по-прежнему поддерживается, но будущие обновления не планируются. Для виртуальных сред на основе Citrix рекомендуется использовать пакет оптимизации Citrix RealTime Optimization Pack.
    
- Пакет **оптимизации Citrix RealTime Optimization Pack** создается на основе подключаемого модуль Lync VDI и работает с клиентами Lync 2013 или Skype для бизнеса 2016 на виртуальном рабочем столе. Она была совместно разработана Citrix и Корпорацией Майкрософт для улучшения исходного подключаемого модуль VDI. Его можно установить на клиенты с операционными системами Windows и другими операционными системами (включая Windows 10, Mac и Linux). Он состоит из двух компонентов: соединительная система RealTime (установленная на виртуальном рабочем столе) и обдвижка мультимедиа RealTime (установленная на локальном компьютере конечного пользователя). Эти два компонента позволяют локальному компьютеру пользователя использовать клиент Skype для бизнеса, работающий на виртуальном рабочем столе, с обработкой A/V, перемещенной на локальный компьютер. Для сред виртуального рабочего стола на основе Citrix рекомендуется использовать пакет оптимизации Citrix RealTime Optimization Pack, и планируется дальнейшая поддержка.
    
- Пакет виртуализации **VMWare Horizon** для Skype для бизнеса, разработанный совместно с VMWare, позволяет предоставлять Skype для бизнеса на виртуальном рабочем столе, при этом обеспечивает отличное взаимодействие с пользователем. Решение работает с использованием мультимедиа-подсистемы на клиенте для создания оптимизированного решения, где конечная точка клиента предоставляет возможности разгрузки мультимедиа для аудио- и видеозвонков. Это решение, которое может доставлять звук и видео непосредственно между конечными точками для совместной работы по одному или разгружать его в центральное подразделение управления несколькими точками (MCU) для многостолбных конференций или собраний.
    
> [!NOTE]
> Клиенты Skype для бизнеса Basic не поддерживаются пакетом оптимизации Citrix HDX RealTime Optimization Pack или VMWare Horizon Virtualization Pack.

## <a name="microsofts-lync-vdi-plug-in"></a>Подключаемый модуль VDI для Lync корпорации Майкрософт
<a name="Citrix_RT"> </a>

С помощью подключаемого решения VDI для Microsoft Lync пользователь должен быть на компьютере с Windows или тонким клиентом и установить подключаемый модуль VDI microsoft Lync для обработки аудио- и видеопотоков от клиента на виртуальном рабочем столе. Пользователь:
  
1. Подключите аудио- и видео устройство (например, гарнитуру или камеру) к локальному компьютеру.
    
2. Подключение к удаленному виртуальному рабочему столу с помощью клиента Lync 2013 или Skype для бизнеса 2015.
    
3. Введите учетные данные skype для бизнеса на виртуальном рабочем столе.
    
4. Повторно введите учетные данные пользователя, чтобы установить подключение к подключаемому модулю Lync VDI на локальном компьютере с Windows или тонком клиенте.
    
После подключения пользователь готов делать и принимать аудио- и видеозвуки. Трафик в сети и нагрузка на виртуальный рабочий стол сведются к минимуму, так как обработка звука и видео обрабатывается на локальном компьютере.
  
Подключаемый модуль VDI microsoft Lync поддерживается только в определенных операционных системах Windows и поддерживает только клиенты Lync 2013 или Skype для бизнеса 2015. Дополнительные [сведения о](vdi-environments.md#Supported_virt) поддерживаемых технологиях и ограничениях см. в поддерживаемых технологиях виртуализации и известных ограничениях.

### <a name="supported-virtualization-technologies-and-known-limitations"></a>Поддерживаемые технологии виртуализации и известные ограничения
<a name="Supported_virt"> </a>

Подключаемый модуль VDI Lync позволяет аудио- и видеозвонки для поддерживаемых технологий виртуализации. В соответствии со стандартными телефонными нормами также включена поддержка E911. В следующих разделах описываются технологии виртуализации, поддерживаемые подключаемый модуль VDI Lync, и известные ограничения функций.
  
#### <a name="support-for-virtualization-technologies"></a>Поддержка технологий виртуализации

Подключаемый модуль VDI Lync поддерживает полные сеансы удаленного рабочего стола в сценарии личного виртуального рабочего стола, но не в сценарии сеанса удаленного рабочего стола. Эти сценарии можно описать следующим образом:
  
- **Поддерживается: персонализированные виртуальные рабочие столы или инфраструктура виртуальных рабочих стола (VDI).** В этом сценарии каждый пользователь входит в настраиваемый виртуальный рабочий стол и может сохранять файлы на рабочем столе, которые сохраняются во время сеансов. Службы удаленных рабочих стола Майкрософт и представление горизонта VMware — это примеры реализаций, которые были протестированы для использования со Skype для бизнеса 2015. Другие реализации, проходящие проверку, включают Citrix XenDesktop. Сведения о средах VDI для конкретных поставщиков и клиентского оборудования, которые были протестированы корпорацией Майкрософт, см. в сведениях об инфраструктуре, квалифицированной для [Microsoft Lync.](https://go.microsoft.com/fwlink/?LinkID=313435)
    
- **Не поддерживается: сеансы удаленного рабочего стола.** В этом сценарии каждый пользователь входит в общий сеанс виртуального рабочего стола, который невозможно настроить. Примеры: сеансы удаленного рабочего стола Майкрософт (RDSH) и Citrix XenApp в сочетании с приемником Citrix.
    
Подключаемый модуль Lync VDI не поддерживает другие технологии виртуализации, такие как виртуализация приложений, что позволяет использовать приложение без необходимости локальной установки полного приложения. Примеры реализации: Citrix XenApp и Microsoft Application Virtualization (App-V). Потоковая передача приложений, удаленное обращение к приложениям и смешанные режимы виртуализации (например, удаленное работа приложений при полном удаленном рабочем столе) не поддерживаются.
  
Подключаемый модуль VDI Lync был разработан для использования независимых от платформы API под названием Dynamic Virtual Channels (DVC). Для сценариев, которые не поддерживаются явным образом, обратитесь к заявлениям о поддержке от поставщика решения VDI.
  
#### <a name="lync-vdi-plug-in-prerequisites"></a>Необходимые условия для подключаемого модульного подключения Lync VDI
<a name="VDI_prereq"> </a>

В среде VDI виртуальные машины и локальный компьютер пользователя должны соответствовать требованиям, изложенным в этом разделе.
  
> [!NOTE]
>  Поставщик решения виртуализации может предоставить подробные сведения об установке и развертывании среды. Общие сведения о развертывании виртуализированной среды на основе Hyper-V и служб удаленных рабочих стола см. в следующих статьях в библиотеке Microsoft: [Hyper-V,](https://go.microsoft.com/fwlink/p/?linkid=247514)службы удаленных рабочих [Windows Server 2008 R2](https://go.microsoft.com/fwlink/p/?linkid=247513) 
  
Виртуальные машины должны быть настроены с Windows 8, Windows 7 или Windows Server 2008 R2 с последними пакетами обновления.
  
Локальный компьютер пользователя должен соответствовать следующим требованиям:
  
- Пользователь должен быть в Skype для бизнеса Server или Lync Server 2013.
    
- Локальный компьютер должен работать под управлением Windows Embedded Standard 7 с sp1, Windows 7 с sp1 или Windows 8.
    
- Если вы используете службы удаленных рабочих стола, выберите 32- или 64-битный подключаемый модуль Lync VDI в соответствие с операционной системой локального компьютера. Для локального компьютера и виртуальной машины не требуется иметь 32- или 64-битные операционные системы. Если вы используете другое решение или платформу виртуализации, обратитесь к требованиям поставщика.
    
- На локальном компьютере должна быть запущена [последняя версия клиента удаленного рабочего стола.](https://go.microsoft.com/fwlink/p/?LinkId=268032) Установите последние обновления клиента служб удаленного рабочего стола корпорации Майкрософт или последнюю версию клиента удаленного рабочего стола поставщика решения виртуализации. 
    
- На локальном компьютере необходимо настроить параметры клиента удаленного рабочего стола, чтобы звук воспроизводился на локальном компьютере, а удаленная запись была отключена. Чтобы настроить эти параметры для подключения к удаленному рабочему столу в Windows, см. следующий раздел: "Настройка параметров подключения к удаленному рабочему столу". 
    
Подключаемый модуль Microsoft VDI доступен в [подключаемом модулье Microsoft Lync VDI 2013 (32-битный)](https://www.microsoft.com/download/details.aspx?id=35457) или подключаемом [модулье VDI 2013 Microsoft Lync (64-битный).](https://www.microsoft.com/download/details.aspx?id=35454)
  
#### <a name="known-feature-limitations"></a>Известные ограничения функций
<a name="VDI_prereq"> </a>

При использовании клиента Skype для бизнеса 2015 в среде VDI используются следующие известные ограничения:
  
Существует ограниченная поддержка функций делегирования вызовов и анонимизации агента группы ответа.
  
Следующие функции не поддерживаются:
  
- интегрированные страницы настройки аудио- и видеоустройств;
    
- Видео с несколькими представлениями.
    
- запись бесед;
    
- Анонимное присоединение к собраниям (то есть присоединение к собраниям Skype для бизнеса, которые были организованы организацией, которая не имеет федерации с вашей организацией).
    
- Использование подключаемого модуль Lync VDI вместе с устройством Lync Phone Edition.
    
- непрерывность звонка при сбое сети;
    
- Настраиваемые мелодии звонка и функции музыки при удержании.
    
Подключаемый модуль VDI Lync не поддерживается в средах Microsoft 365 или Office 365.

> Пакет оптимизации Citrix RealTime Optimization Pack поддерживает Microsoft 365 и Office 365. Список поддерживаемых функций и версий для виртуальных сред на основе Citrix можно просмотреть в технической документации Citrix.
