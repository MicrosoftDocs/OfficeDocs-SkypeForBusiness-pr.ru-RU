# <a name="plan-for-skype-for-business-in-vdi-environments"></a>Планирование развертывания Skype для бизнеса в средах VDI
 
В этой статье обсуждаются вопросы планирования использования Skype для бизнеса во время подключения к удаленному виртуальному рабочему столу. 
  
Среда инфраструктуры удаленных рабочих столов (VDI) используется в некоторых организациях, имеющих особые требования к безопасности и соответствию. Пользователи в этих организациях работают в среде виртуального рабочего стола, используя все необходимые файлы и приложения с помощью служб удаленного рабочего стола или аналогичного удаленного подключения. Использование Skype для бизнеса с полным аудио-и видеоподключением, таким как, требует высокой нагрузки аудио-и видеообработки на клиентском компьютере, подключенном к виртуальному рабочему столу. Доступно дополнительное программное обеспечение для подключения к VDI, которое обрабатывается на локальном компьютере конечного пользователя и уменьшает нагрузку на виртуальный компьютер.
  
Существует три решения, которые можно найти в компоненте Plug-in VDI, предлагаемом корпорацией Microsoft, Citrix или VMWare. Для новых развертываний Корпорация Майкрософт рекомендует использовать решение Citrix HDX с пакетами оптимизации в режиме реального времени или пакет виртуализации для VMWare горизонта. Исходный плагин Lync VDI по-прежнему поддерживается в оставшейся части жизненного цикла.
  
- **Внешний элемент LYNC VDI** разработан для Lync 2013 и совместим с клиентом Lync 2013 или Skype для бизнеса 2015 на виртуальном рабочем столе. It's a stand-alone application that installs on the local computer and allows the use of local audio and video devices with a client on a virtual desktop. Для этого не требуется установка клиента Skype для бизнеса на локальном компьютере или в тонком клиенте, на котором должны выполняться операционные системы Windows 7, Windows 8 или Windows Server 2008. (Тонкие клиентские устройства, в которых используются следующие операционные системы и поддерживаемые корпорацией Майкрософт, включают: Dell Висе Z90D7, Dell Висе R90L7, Dell Висе X90m7, HP T610 и HP t5740e.) Этот плагин по-прежнему поддерживается, но последующие обновления не планируются. For Citrix-based virtual environments, the Citrix RealTime Optimization Pack is recommended.
    
- **Пакет оптимизации "в реальном времени** " строится на плагине Lync VDI и работает с клиентами Lync 2013 или Skype для бизнеса 2016 на виртуальном рабочем столе. Этот пакет был разработан совместно корпорациями Citrix и Майкрософт в целях расширения возможностей исходного подключаемого модуля VDI. Его можно устанавливать на клиенты с ОС Windows и других производителей (включая ОС Windows 10, Mac и Linux). Она состоит из двух компонентов: соединителя реального времени (установленного на виртуальном рабочем столе) и модуля мультимедиа реального времени (установленного на локальном компьютере конечного пользователя). Эти два компонента позволяют локальному компьютеру пользователя использовать клиент Skype для бизнеса, работающий на виртуальном рабочем столе, с помощью обработки A/V, которая перемещается на локальный компьютер. Для сред виртуальных рабочих столов на базе технологий Citrix пакет Citrix RealTime Optimization Pack рекомендуется к применению и будет поддерживаться в дальнейшем.
    
- **Пакет виртуализации для VMware** для бизнеса, разработанный для совместной работы с помощью VMware, позволяет использовать Skype для бизнеса на виртуальном рабочем столе, обеспечивая очень удобное взаимодействие с пользователями. Это решение работает путем использования мультимедийного модуля для создания оптимизированного решения с помощью клиентской конечной точки, предоставляющей возможности разгрузки мультимедиа для голосовой связи и видеозвонков. Это решение, которое может передавать аудио-и видеофайлы прямо между конечными точками для совместной работы или разгрузки на центральном управляющем блоке MultiPoint (MCU) для многофакторных конференций или собраний.
    
> [!NOTE]
> Клиенты Skype для бизнеса Basic не поддерживаются с пакетом оптимизации HDX в режиме реального времени или пакетом виртуализации для VMWare с горизонтом.

## <a name="microsofts-lync-vdi-plug-in"></a>Подключаемый модуль VDI для Lync (Майкрософт)
<a name="Citrix_RT"> </a>

С помощью подключаемого модуля Microsoft Lync VDI пользователь должен быть установлен на компьютере с Windows или на тонком клиенте, и для обработки звуковых и видеопотоков из клиента на виртуальном рабочем столе должен использоваться подключаемый модуль VDI Microsoft Lync. Пользователю потребуется:
  
1. подключить аудио- и видеоустройство (например гарнитуру или камеру) к локальному компьютеру;
    
2. Подключитесь к удаленному виртуальному рабочему столу с помощью клиента Lync 2013 или Skype для бизнеса 2015.
    
3. Введите учетные данные для Skype для бизнеса на виртуальном рабочем столе.
    
4. Повторно введите учетные данные пользователя, чтобы установить соединение с помощью плагина Lync VDI на локальном компьютере с ОС Windows или тонком клиенте.
    
После установки подключения пользователь будет готов выполнять и принимать аудио- и видеозвонки. Трафик в сети и нагрузка на виртуальный рабочий стол сводится к минимуму, так как обработка аудио- и видеоданных выполняется локальным компьютером.
  
Плагин Microsoft Lync VDI поддерживается только в некоторых операционных системах Windows и поддерживает только клиентов Lync 2013 и Skype для бизнеса 2015. Сведения о других ограничениях и поддерживаемых технологиях см. в разделе [Поддерживаемые технологии виртуализации и известные ограничения](vdi-environments.md#Supported_virt).

### <a name="supported-virtualization-technologies-and-known-limitations"></a>Поддерживаемые технологии виртуализации и известные ограничения
<a name="Supported_virt"> </a>

Подключаемый модулей Lync VDI позволяет осуществлять голосовую и видеосвязь в поддерживаемых технологиях виртуализации. В соответствии с требованиями обычной телефонной сети поддержка стандарта E911 также включена. В следующих разделах описаны технологии виртуализации, поддерживаемые внешним модулем Lync VDI и известными ограничениями функций.
  
#### <a name="support-for-virtualization-technologies"></a>Поддержка технологий виртуализации

Внешний доступ к рабочему столу в сценарии для персонального рабочего стола, но не в сценарии подключения к удаленному рабочему столу, поддерживается внешним сеансом Lync. Эти сценарии можно описать следующим образом.
  
- **Поддерживается: персонализированные виртуальные рабочие столы или инфраструктура виртуальных рабочих столов (VDI)**. В этом сценарии каждый пользователь входит в настраиваемый виртуальный рабочий стол и может сохранять файлы на рабочем столе на протяжении всего периода сеанса. Примеры реализации служб удаленных рабочих столов (Microsoft) и горизонта VMware, которые были проверены на использование в Skype для бизнеса 2015. Другие реализации,  включая Citrix XenDesktop, проходят проверку. Сведения о специфических средах VDI и клиентском оборудовании, проверенных корпорацией Майкрософт, описаны в разделе [инфраструктура, определенная для Microsoft Lync](https://go.microsoft.com/fwlink/?LinkID=313435).
    
- **Не поддерживается: сеансы удаленных рабочих столов**. В этом сценарии каждый пользователь выполняет вход в общий сеанс виртуального рабочего стола, настройка которого невозможна. Примеры включают сеансы удаленного рабочего стола Microsoft (ксенапп) и Citrix, Объединенные с помощью Citrix ресивер.
    
Плагин Lync VDI не поддерживает другие технологии виртуализации, такие как виртуализация приложений, что позволяет использовать приложение, не требуя установки полного приложения локально. Пример реализации: Citrix XenApp и Microsoft Application Virtualization (App-V). Потоковое использование приложений, удаленное использование приложений и смешанные режимы виртуализации (например, удаленное использование приложений в полном наборе функций удаленной работы рабочего стола) не поддерживаются.
  
Плагин Lync VDI разработан для использования независимых от платформы интерфейсов API, именуемых динамическими виртуальными каналами (Двкс). Для сценариев, которые не поддерживаются явным образом, см. заявления о поддержке, опубликованные поставщиком решения VDI.
  
#### <a name="lync-vdi-plug-in-prerequisites"></a>Необходимые условия для подключаемого модуля Lync VDI
<a name="VDI_prereq"> </a>

В среде VDI Виртуальные машины и локальный компьютер пользователя должны соответствовать требованиям, описанным в этом разделе.
  
> [!NOTE]
>  Обратитесь к поставщику решения виртуализации для получения сведений об установке и развертывании виртуализованной среды. Общие сведения о развертывании виртуализованной среды на основе служб Hyper-V и удаленных рабочих столов можно найти в следующих статьях библиотеки Майкрософт: [Hyper-v](https://go.microsoft.com/fwlink/p/?linkid=247514), [службы удаленных рабочих столов в Windows Server 2008 R2](https://go.microsoft.com/fwlink/p/?linkid=247513) 
  
Виртуальные машины должны быть настроены в Windows 8, Windows 7 или Windows Server 2008 R2 с самыми последними пакетами обновления.
  
Локальный компьютер пользователя должен соответствовать следующим требованиям:
  
- Пользователь должен быть расположен в Skype для бизнеса Server или Lync Server 2013.
    
- На локальном компьютере должна быть установлена операционная система Windows Embedded 7 с пакетом обновления 1 (SP1), Windows 7 с пакетом обновления 1 или Windows 8.
    
- Если вы используете службы удаленных рабочих столов, выберите один из 32-разрядных или 64-разрядных модулей Lync VDI в соответствии с операционной системой локального компьютера. Наличие 32- или 64-разрядных систем одновременно на локальном компьютере и виртуальной машине не обязательно. Если вы используете другие решение или платформу виртуализации, руководствуйтесь требованиями поставщика.
    
- На локальном компьютере должна быть установлена [самая свежая версия клиента удаленного доступа к удаленному рабочему столу](https://go.microsoft.com/fwlink/p/?LinkId=268032). Установите последние обновления клиента служб удаленного рабочего стола (Майкрософт) или последнюю версию клиента удаленного рабочего стола другого поставщика решения виртуализации. 
    
- Параметры клиента удаленного рабочего стола на локальном компьютере необходимо настроить так, чтобы звук воспроизводился на локальном компьютере, а удаленная запись была отключена. Чтобы настроить эти параметры подключения к удаленному рабочему столу в Windows, ознакомьтесь со следующей разделом "Настройка параметров подключения к удаленному рабочему столу". 
    
Подключаемый модуль Microsoft VDI доступен в подключаемом модуле [Microsoft LYNC vdi 2013 (с 32-bit)](https://www.microsoft.com/en-us/download/details.aspx?id=35457) или [microsoft Lync vdi 2013 (64 bit)](https://www.microsoft.com/en-us/download/details.aspx?id=35454).
  
#### <a name="known-feature-limitations"></a>Известные ограничения функций
<a name="VDI_prereq"> </a>

Ниже перечислены некоторые известные ограничения при использовании клиента Skype для бизнеса 2015 в среде VDI.
  
Ограниченная поддержка функций делегирования звонков и анонимного агента групп ответов.
  
Следующие функции не поддерживаются:
  
- интегрированные страницы настройки аудио- и видеоустройств;
    
- видео с несколькими представлениями;
    
- запись бесед;
    
- Анонимное присоединение к собраниям (то есть присоединение к собраниям Skype для бизнеса, размещенным в Организации, которая не входит в состав вашей организации).
    
- Использование плагина Lync VDI вместе с устройством Lync Phone Edition.
    
- непрерывность звонка при сбое сети;
    
- настраиваемые мелодии звонков и музыка при удержании звонка.
    
Плагин Lync VDI не поддерживается в среде Office 365.
  
> [!NOTE]
> Пакет оптимизации Citrix RealTime Optimization Pack поддерживает Office 365. Для виртуальных сред, использующих Citrix, ознакомьтесь с документацией по [техническим обзорам](https://docs.citrix.com/en-us/hdx-optimization/2-0/hdx-realtime-optimization-pack-about.mdl) Citrix, чтобы получить список поддерживаемых функций и версий.
