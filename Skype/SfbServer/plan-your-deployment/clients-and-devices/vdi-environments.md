# <a name="plan-for-skype-for-business-in-vdi-environments"></a>Планирование развертывания Skype для бизнеса в средах VDI
 
В этом разделе рассматриваются вопросы планирования использования Скайп для бизнеса при подключении к удаленной виртуальных рабочих столов. 
  
Среда инфраструктуры удаленных рабочих столов (VDI) используется в некоторых организациях, имеющих особые требования к безопасности и соответствию. Пользователи в этих организациях работают в среде виртуального рабочего стола, используя все необходимые файлы и приложения с помощью служб удаленного рабочего стола или аналогичного удаленного подключения. С помощью Скайп для бизнеса с полной аудио- и видеоконференций на подключение так, требует больших нагрузках аудио и видео, обработки на стороне клиента, размещенных на виртуальных рабочих столов. Дополнительное программное обеспечение инфраструктуры виртуальных рабочих СТОЛОВ для подключаемого модуля доступна разгружает обработки на компьютер конечного пользователя, а также снизить нагрузку на виртуальном рабочем столе.
  
Существует три решения для компонента подключаемого модуля инфраструктуры виртуальных рабочих СТОЛОВ, предлагаемых Microsoft, Citrix или VMWare. Для нового развертывания корпорация Майкрософт рекомендует использовать решение Citrix HDX в реальном времени Optimization Pack или пакет виртуализации VMWare горизонта. Исходный подключаемого модуля VDI Lync по-прежнему поддерживается для оставшейся части его жизненного цикла.
  
- **Подключаемый модуль VDI для Lync** был разработан для Lync 2013 и совместим с Lync 2013 или Скайп для клиента Business 2015, запущенные на виртуальном рабочем столе. Это автономное приложение, устанавливаемое на локальный компьютер, которое позволяет использовать локальные аудио- и видеоустройства для работы с клиентом в среде удаленного рабочего стола. Подключаемый модуль не требует Скайп для клиента Business необходимо установить на локальном компьютере или в тонком клиенте, который необходимо запустить операционные системы Windows 7, Windows 8 и Windows Server 2008. (Относятся следующие устройства тонком клиенте с помощью следующих операционных систем и поддерживаемых Майкрософт: Dell Wyse Z90D7, Dell Wyse R90L7, Dell Wyse X90m7, серии t610 HP и HP t5740e.) Этот подключаемый модуль по-прежнему поддерживается, но не планируется больше обновлений. Для виртуальных сред Citrix рекомендуется пакет оптимизации Citrix RealTime Optimization Pack.
    
- **Пакет оптимизации в реальном времени Citrix** развиваются подключаемого модуля VDI Lync и работает с Lync 2013 или Скайп для клиентов, 2016 бизнеса на виртуальном рабочем столе. Этот пакет был разработан совместно корпорациями Citrix и Майкрософт в целях расширения возможностей исходного подключаемого модуля VDI. Его можно устанавливать на клиенты с ОС Windows и других производителей (включая ОС Windows 10, Mac и Linux). Состоит из двух компонентов: соединитель в реальном времени (который устанавливается на виртуальном рабочем столе) и модуль мультимедиа в реальном времени (который устанавливается на компьютер конечного пользователя). Эти два компонента разрешить за пределами локального компьютера для использования Скайп для клиента Business, работающим на виртуальном рабочем столе с A / V обработки перемещены на локальном компьютере. Для сред виртуальных рабочих столов на базе технологий Citrix пакет Citrix RealTime Optimization Pack рекомендуется к применению и будет поддерживаться в дальнейшем.
    
- **Пакет виртуализации VMWare горизонта** для Скайп для бизнеса, разработана в сотрудничестве с VMWare, позволяет обеспечить Скайп для бизнеса в виртуальных рабочих столов доставки удобную работу пользователя. Возможности для аудио-и видеовызовы разгрузки works решения, используя модуль мультимедиа на стороне клиента для создания оптимизированного решения, с помощью конечная точка клиента, предоставляя мультимедиа. Это решение, которое можно проводить аудио и видео либо напрямую между конечными точками для индивидуального совместной работы и разгрузки его для центра узлом управления узла (MCU) для собраний или многосторонних конференциях.
    
> [!NOTE]
> Скайп для бизнеса базовой клиентов с в реальном времени оптимизации Citrix HDX пакета или VMWare горизонта виртуализации не поддерживаются.

## <a name="microsofts-lync-vdi-plug-in"></a>Подключаемый модуль VDI для Lync (Майкрософт)
<a name="Citrix_RT"> </a>

Возможности решения по подключаемого модуля инфраструктуры виртуальных рабочих СТОЛОВ Microsoft Lync пользователь имеет значение на компьютере Windows или тонком клиенте и подключаемого модуля Lync VDI корпорации Майкрософт, установленные для обработки потоков аудио и видео от клиента на виртуальном рабочем столе. Пользователю потребуется:
  
1. подключить аудио- и видеоустройство (например гарнитуру или камеру) к локальному компьютеру;
    
2. Подключение к удаленной виртуального рабочего стола с Lync 2013 или Скайп для клиента Business 2015.
    
3. Введите учетные данные для Скайп для бизнеса на виртуальном рабочем столе.
    
4. Введите учетные данные пользователя, чтобы установить соединение с подключаемого модуля на локальном компьютере Windows или тонком клиенте VDI Lync.
    
После установки подключения пользователь будет готов выполнять и принимать аудио- и видеозвонки. Трафик в сети и нагрузка на виртуальный рабочий стол сводится к минимуму, так как обработка аудио- и видеоданных выполняется локальным компьютером.
  
Подключаемый модуль VDI для Lync корпорации Майкрософт поддерживается только в определенных операционных системах Windows и поддерживает только Lync 2013 или Скайп для клиентов, 2015 бизнеса. Сведения о других ограничениях и поддерживаемых технологиях см. в разделе [Поддерживаемые технологии виртуализации и известные ограничения](vdi-environments.md#Supported_virt).

### <a name="supported-virtualization-technologies-and-known-limitations"></a>Поддерживаемые технологии виртуализации и известные ограничения
<a name="Supported_virt"> </a>

Подключаемый модуль VDI для Lync позволяет аудио- и видеоконференций, вызова для технологий виртуализации поддерживаются. В соответствии с требованиями обычной телефонной сети поддержка стандарта E911 также включена. В следующих разделах описываются технологий виртуализации, поддерживаемых приложением подключаемого модуля VDI Lync и известные ограничения.
  
#### <a name="support-for-virtualization-technologies"></a>Поддержка технологий виртуализации

Подключаемый модуль VDI для Lync поддерживает полноэкранный режим удаленных сеансов в личных сценариев виртуального рабочего стола, но не в сценарии сеанс удаленного рабочего стола. Эти сценарии можно описать следующим образом.
  
- **Поддерживается: персонализированные виртуальные рабочие столы или инфраструктура виртуальных рабочих столов (VDI)**. В этом сценарии каждый пользователь входит в настраиваемый виртуальный рабочий стол и может сохранять файлы на рабочем столе на протяжении всего периода сеанса. Службы удаленных рабочих столов Microsoft и VMware горизонта представление — пример реализации, которые будут проверены для использования с Скайп для бизнеса 2015. Другие реализации,  включая Citrix XenDesktop, проходят проверку. Дополнительные сведения о средах VDI конкретных поставщиков и оборудовании для клиентов, которые были проверены корпорацией Майкрософт, см. в разделе [Infrastructure qualified for Microsoft Lync](https://go.microsoft.com/fwlink/?LinkID=313435) (Инфраструктура, соответствующая требованиям Microsoft Lync).
    
- **Не поддерживается: сеансы удаленных рабочих столов**. В этом сценарии каждый пользователь выполняет вход в общий сеанс виртуального рабочего стола, настройка которого невозможна. Примерами являются сеансов удаленных рабочих столов Майкрософт (RDSH) и Citrix XenApp в сочетании с Citrix приемника.
    
Подключаемый модуль VDI для Lync не поддерживает другие технологии виртуализации, такие как виртуализации приложений, что позволяет использовать приложения без необходимости установки полное приложение локально. Пример реализации: Citrix XenApp и Microsoft Application Virtualization (App-V). Потоковое использование приложений, удаленное использование приложений и смешанные режимы виртуализации (например, удаленное использование приложений в полном наборе функций удаленной работы рабочего стола) не поддерживаются.
  
Подключаемый модуль VDI для Lync предназначено для использования API зависящего от платформы, называется динамической виртуальные каналы (DVCs). Для сценариев, которые не поддерживаются явным образом, см. заявления о поддержке, опубликованные поставщиком решения VDI.
  
#### <a name="lync-vdi-plug-in-prerequisites"></a>Необходимые условия для подключаемого модуля Lync VDI
<a name="VDI_prereq"> </a>

В среде VDI виртуальные машины и локальные компьютеры пользователей должны соответствовать требованиям, описанным в этом разделе.
  
> [!NOTE]
>  Обратитесь к поставщику решения виртуализации для получения сведений об установке и развертывании виртуализованной среды. Общие сведения о развертывании в виртуализованной среде на основе Hyper-V и служб удаленных рабочих столов, см в библиотеке Microsoft: [Hyper-V](https://go.microsoft.com/fwlink/p/?linkid=247514), [Служб удаленных рабочих столов в Windows Server 2008 R2](https://go.microsoft.com/fwlink/p/?linkid=247513) 
  
Виртуальные машины должны быть настроены Windows 8, Windows 7 или Windows Server 2008 R2 с последними пакетами обновления.
  
На локальном компьютере должен удовлетворять следующим требованиям:
  
- Пользователь должен быть размещен в Скайп для Business Server или Lync Server 2013.
    
- Локальный компьютер должен работать под управлением Windows Embedded Standard 7 с пакетом обновления 1, Windows 7 с пакетом обновления 1 или Windows 8.
    
- Если вы используете служб удаленных рабочих столов, выберите 32-разрядная или 64-разрядная версия подключаемого модуля VDI Lync в соответствии с локального компьютера операционной системы. Наличие 32- или 64-разрядных систем одновременно на локальном компьютере и виртуальной машине не обязательно. Если вы используете другие решение или платформу виртуализации, руководствуйтесь требованиями поставщика.
    
- На локальном компьютере должна использоваться [последняя версия клиента удаленного рабочего стола](https://go.microsoft.com/fwlink/p/?LinkId=268032). Установите последние обновления клиента служб удаленного рабочего стола (Майкрософт) или последнюю версию клиента удаленного рабочего стола другого поставщика решения виртуализации. 
    
- Параметры клиента удаленного рабочего стола на локальном компьютере необходимо настроить так, чтобы звук воспроизводился на локальном компьютере, а удаленная запись была отключена. Чтобы настроить эти параметры для подключения к удаленному рабочему столу в Windows, содержится в следующем разделе, «чтобы настроить параметры подключения к удаленному рабочему столу». 
    
Подключаемый модуль VDI (Майкрософт) можно скачать на странице [Microsoft Lync VDI 2013 Plug-In (32-разрядная)](https://www.microsoft.com/en-us/download/details.aspx?id=35457) или [Microsoft Lync VDI 2013 Plug-In (64-разрядная)](https://www.microsoft.com/en-us/download/details.aspx?id=35454).
  
#### <a name="known-feature-limitations"></a>Известные ограничения функций
<a name="VDI_prereq"> </a>

Ниже перечислены известные ограничения при использовании Скайп для клиента Business 2015 в среде VDI.
  
Существует ограниченная поддержка функций делегирования вызовов и анонимизации агентов группы группы ответа.
  
Следующие функции не поддерживаются:
  
- интегрированные страницы настройки аудио- и видеоустройств;
    
- видео с несколькими представлениями;
    
- запись бесед;
    
- Присоединение к собраниям анонимных пользователей (то есть, присоединение к Скайп для деловых встреч, размещенного в организации, не федерацию с вашей организации).
    
- С помощью подключаемого модуля вместе с устройством Lync Phone Edition VDI Lync.
    
- непрерывность звонка при сбое сети;
    
- настраиваемые мелодии звонков и музыка при удержании звонка.
    
Подключаемый модуль VDI для Lync не поддерживается в среде Office 365.
  
> [!NOTE]
> Пакет оптимизации Citrix RealTime Optimization Pack поддерживает Office 365. На основе Citrix виртуальных сред документации Citrix's [Технический обзор](https://docs.citrix.com/en-us/hdx-optimization/2-0/hdx-realtime-optimization-pack-about.mdl) список поддерживаемых функций и версий.
