# <a name="plan-the-skype-for-business-2015-client-experience-for-your-users"></a>Планирование взаимодействия с клиентом Skype для бизнеса 2015 для пользователей
 
**Сводка:** Узнайте о новом Skype для бизнеса и действиях, которые можно предпринять, чтобы подготовить среду и пользователей к обновлению независимо от того, используете ли вы Skype для бизнеса Online, Skype для бизнеса Server 2019, Skype для бизнеса Server 2015, Lync Server 2013 или Lync Server 2010.
  
Пакет обновления Office для 14th (апрель) 2015 для Lync 2013 включает новый пользовательский интерфейс Skype для бизнеса. Это обновление позволяет администраторам управлять внешним видом и поведением клиента и выбирать, следует ли сохранять взаимодействие с клиентом Lync 2013, или использовать улучшенное взаимодействие с клиентом Skype для бизнеса. Клиент Skype для бизнеса эффективно заменил клиент Lync 2013 и добавил возможность администраторов выбирать между существующими возможностями взаимодействия с клиентом Lync и новым интерфейсом клиента Skype для бизнеса. Сведения об этом обновлении вы найдете в [статье 14 апреля 2015 г. для Lync 2013 (Skype для бизнеса) (KB2889923)](https://support.microsoft.com/kb/2889923/).
  
В 12 мая 2015 существует ежемесячный пакет обновления для Office, который включает обновленный клиент Skype для бизнеса. Многие клиенты, которые не применяли обновление за Апрель, будут получать 12 мая: Обновление для Office 2013. Сведения, приведенные в этом разделе, помогут вам подготовить вашу организацию, среду и пользователей к обновлению клиента. Чтобы упростить переход для пользователей и поддержки Teams, используйте сведения, приведенные в этом разделе, чтобы определить, какой интерфейс требуется для пользователей, а затем внести изменения в среду, прежде чем развертывать клиентское обновление в Организации.

- [Подготовка среды для клиента Skype для бизнеса](user-experience.md#usinglync)

## <a name="what-client-experience-do-you-want-for-your-users"></a>Какие возможности клиента вы хотите использовать для ваших пользователей?
<a name="clientexperience"> </a>

С помощью нового клиента Skype для бизнеса вы можете управлять возможностями клиентов, которые получают пользователи: Lync или Skype для бизнеса. Взаимодействие с клиентом по умолчанию зависит от того, используется ли Lync или Skype для бизнеса в локальной среде или в сети. Если вы используете Skype для бизнеса Online (Lync Online) на сегодняшний день с приложениями Microsoft 365 для предприятий, Microsoft 365 бизнес Standard или Office 2013, то обновленный клиентский интерфейс Skype для бизнеса, который, в своюмся стиле и функции Skype, будет использоваться по умолчанию. Если используется локальная среда Lync Server, по умолчанию будет использоваться клиент Lync.
  
Вы можете настроить взаимодействие с клиентами, которые получают пользователи, с помощью политик клиентов. Политика клиента — это набор параметров конфигурации, которые применяются к пользователям при входе в Lync или Skype для бизнеса.
  
### <a name="skype-for-business-client-experience"></a>Взаимодействие с клиентом Skype для бизнеса

В дополнение ко всем функциям Lync, Skype для бизнеса предоставляет новые функции с упрощенными элементами управления и знакомыми значками Skype. Некоторые новые функции в Skype для бизнеса доступны только в новом интерфейсе клиента Skype для бизнеса. Дополнительные сведения о новых возможностях Skype для бизнеса можно найти в статье [Знакомство с Skype для бизнеса](https://go.microsoft.com/fwlink/p/?LinkId=528686).
  
### <a name="lync-client-experience"></a>Взаимодействие с клиентом Lync

Взаимодействие с клиентом Lync очень похоже на взаимодействие с клиентом Lync 2013, с которым пользователи уже знакомы, но существует несколько изменений, о которых вы хотели бы сообщить пользователям. Сведения о различиях между интерфейсом Lync и клиентом Lync 2013 приведены в статье [почему я вижу Skype для бизнеса при использовании Lync?](https://go.microsoft.com/fwlink/p/?LinkId=544712) и дополнительные ссылки, приведенные далее в этом разделе.
  
## <a name="prepare-your-environment-for-the-skype-for-business-client"></a>Подготовка среды для клиента Skype для бизнеса
<a name="usinglync"> </a>

Существует несколько действий, которые необходимо выполнить, чтобы подготовить среду к обновлению клиента. Прежде чем вносить какие-либо изменения в настройку взаимодействия с клиентом, необходимо убедиться, что используется версия Skype для бизнеса Server или Lync Server, поддерживающая параметры политики клиента.
  
После подтверждения того, что вы используете версию Skype для бизнеса Server или Lync Server, которая поддерживает параметры политики для управления интерфейсом клиента, необходимо настроить параметры политики в среде. Действия, которые необходимо выполнить, зависят от используемой версии Skype для бизнеса Server или Lync Server, а также от того, находятся ли ваши пользователи в локальной среде или в сети. 
  
Вы хотите внести эти изменения, прежде чем обновление клиента будет доставлено пользователям, чтобы вы могли управлять интерфейсом клиента при первом запуске клиента Skype для бизнеса. ## <a name="use-the-skype-client-experience-with-lync-server-2013-or-lync-server-2010-on-premises"></a>Использование взаимодействия с клиентом Skype в локальной среде Lync Server 2013 или Lync Server 2010
<a name="SkypewithLynconprem"> </a>

Если вы хотите настроить взаимодействие с клиентом Skype в локальном развертывании, выполните действия, описанные в этом разделе. По умолчанию для локальной среды
  
 **Шаг 1:** Для начала убедитесь, что вы используете версию Lync Server, поддерживающую параметры политики клиента.
  
- **Lync server 2013** — необходимо использовать накопительный пакет обновления за декабрь 2014 (5.0.8308.857) для lync Server 2013 или более поздней версии. Сведения о том, как найти [Обновление для Lync Server 2013](https://go.microsoft.com/fwlink/p/?LinkId=532772).
    
- **Lync server 2010** — необходимо использовать накопительный пакет обновления за Февраль 2015 (4.0.7577.710) для lync Server 2010 или более поздней версии. Сведения о том, как найти [Обновление для Lync Server 2010](https://go.microsoft.com/fwlink/p/?LinkId=532771).
    
  **Шаг 2:** Затем используйте политику клиента, чтобы настроить взаимодействие с клиентом Skype для клиента Skype для бизнеса. Существует **3 варианта** использования клиентской политики для настройки взаимодействия с клиентом.
  
  **Вариант 1:** Настройте взаимодействие с клиентом Skype с помощью глобальной политики. Обратите внимание, что глобальная политика применяется ко всем пользователям в развертывании, но политики на уровне пользователей и сайтов имеют приоритет над глобальной политикой:
  
```PowerShell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $True
```

 **Вариант 2:** Изменить существующую клиентскую политику, используемую в вашей среде, чтобы включить этот параметр, чтобы включить взаимодействие с клиентом Skype. Это позволяет назначить взаимодействие с клиентом Skype только тем пользователям, которым назначена существующая политика:
  
```PowerShell
Set-CsClientPolicy -Identity ExistingClientPolicyName -EnableSkypeUI $True
```

 **Вариант 3:** Создайте политику, которая будет назначена пользователям, включающим параметр для взаимодействия с клиентом Skype. Сначала создайте новую клиентскую политику и укажите имя политики в качестве значения параметра **Identity** :
  
```PowerShell
New-CsClientPolicy -Identity UseSkypeUI -EnableSkypeUI $True
```

Затем назначьте политику пользователям, используя имя политики (значение, используемое для параметра **Identity** ) в качестве значения параметра **PolicyName** :
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName UseSkypeUI
```

 **Шаг 3:** После настройки политик клиентов разверните клиент Skype для бизнеса, создайте 4711,1002 (Апрель, 2015) или более поздней версии.
  
## <a name="use-the-lync-client-experience-with-lync-server-2013-or-lync-server-2010-on-premises"></a>Использование взаимодействия с клиентом Lync в локальной среде Lync Server 2013 или Lync Server 2010
<a name="LyncwithLynconprem"> </a>

Это режим по умолчанию при развертывании клиента Skype для бизнеса в локальном развертывании Lync Server. Вам не нужно настраивать какие бы то ни было политики клиентов для использования взаимодействия с клиентом Lync, но вам может потребоваться управлять поведением первого запуска клиента. По умолчанию при первом запуске клиента Skype для бизнеса используется интерфейс Skype для клиента, и пользователям, запрашивающим перезапуск клиента для получения взаимодействия с клиентом Lync, будет выводиться уведомление. Можно настроить среду таким образом, чтобы при первом запуске клиента отображалось взаимодействие с клиентом Lync, а также отключить клиентский учебник, изменив системный реестр на клиентских компьютерах. Действия, которые необходимо выполнить перед развертыванием клиента Skype для бизнеса, представлены в следующих разделах:
  
- **Lync server 2013**, ознакомьтесь [со статьей Настройка взаимодействия с клиентом в Skype для бизнеса в Lync Server 2013](https://go.microsoft.com/fwlink/p/?LinkId=532732)
    
- **Lync server 2010** ознакомьтесь [со статьей Настройка взаимодействия с клиентом в Skype для бизнеса в Lync Server 2010](https://go.microsoft.com/fwlink/p/?LinkId=532733)
    
## <a name="use-the-lync-client-experience-with-skype-for-business-server-on-premises"></a>Использование взаимодействия с клиентом Lync в локальной среде Skype для бизнеса Server
<a name="LyncwithSfBServer"> </a>

Выполните действия, описанные в этом разделе, если необходимо настроить взаимодействие с клиентом Lync в локальном развертывании Skype для бизнеса Server.
  
Если вы хотите настроить взаимодействие с клиентом Skype в локальном развертывании, выполните действия, описанные в этом разделе. По умолчанию для локальной среды
  
 **Шаг 1:** Сначала разверните Skype для бизнеса Server.
  
 **Шаг 2:** Затем с помощью политики клиента настройте взаимодействие с клиентом Lync для клиента Skype для бизнеса. Существует **3 варианта** использования клиентской политики для настройки взаимодействия с клиентом.
  
 **Вариант 1:** Настройте взаимодействие с клиентом Lync с помощью глобальной политики. Обратите внимание, что глобальная политика применяется ко всем пользователям в развертывании, но политики на уровне пользователей и сайтов имеют приоритет над глобальной политикой:
  
```PowerShell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $False
```

 **Вариант 2:** Изменить существующую политику клиента, используемую в среде, чтобы включить этот параметр, чтобы включить взаимодействие с клиентом Lync. Это позволяет назначить взаимодействие с клиентом Lync только тем пользователям, которым назначена существующая политика:
  
```PowerShell
Set-CsClientPolicy -Identity ExistingClientPolicyName -EnableSkypeUI $False
```

 **Вариант 3:** Создайте политику, которая будет назначена пользователям, включающим параметр для взаимодействия с клиентом Lync. Сначала создайте новую клиентскую политику и укажите имя политики в качестве значения параметра **Identity** :
  
```PowerShell
New-CsClientPolicy -Identity UseLyncUI -EnableSkypeUI $False
```

Затем назначьте политику пользователям, используя имя политики (значение, используемое для параметра **Identity** ) в качестве значения параметра **PolicyName** :
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName UseLyncUI
```

 **Шаг 3: необязательный** параметр — по умолчанию при первом запуске клиента Skype для бизнеса используется взаимодействие с клиентом Skype, и пользователю отображается уведомление о необходимости перезапуска клиента для получения взаимодействия с клиентом Lync. Можно настроить среду таким образом, чтобы интерфейс пользователя Lync отображался при первом запуске клиента, а также для отключения руководства клиента путем изменения системного реестра на клиентских компьютерах. Действия, которые необходимо выполнить перед развертыванием клиента Skype для бизнеса, можно найти в статье [Настройка взаимодействия с клиентом в Skype для бизнеса](../../deploy/deploy-clients/configure-the-client-experience.md).
  
 **Шаг 4:** После настройки политик клиентов разверните клиент Skype для бизнеса, создайте 4711,1002 (Апрель, 2015) или более поздней версии.
  
## <a name="use-the-lync-client-experience-with-skype-for-business-online"></a>Использование взаимодействия с клиентом Lync в Skype для бизнеса Online
<a name="LyncwithSfBO"> </a>

Выполните действия, описанные в этом разделе, если вы хотите настроить взаимодействие с клиентом Lync и использовать Skype для бизнеса Online.
  
Если вы используете Skype для бизнеса Online, вы по-прежнему можете использовать взаимодействие с клиентом Lync для клиента Skype для бизнеса в вашей организации, используя удаленную оболочку PowerShell для настройки политик клиентов. Существует **3 варианта** использования клиентской политики для настройки взаимодействия с клиентом. Обратите внимание, что имена политики и параметров отличаются от параметров, используемых для настройки взаимодействия с клиентами при использовании локальной среды Skype для бизнеса или Lync Server.
  
 **Вариант 1:** Настройте взаимодействие с клиентом Lync с помощью глобальной политики. Обратите внимание, что политики клиентов и сайтов, применяемые к пользователям, имеют приоритет над глобальной политикой.
  
```PowerShell
Grant-CsClientPolicy -PolicyName ClientPolicyDisableSkypeUI
```

 **Вариант 2:** Изменить существующую политику клиента, используемую в среде, чтобы включить этот параметр, чтобы включить взаимодействие с клиентом Lync. Это позволяет назначить взаимодействие с клиентом Lync только тем пользователям, которым назначена существующая политика:
  
```PowerShell
Grant-CsClientPolicy -PolicyName ClientPolicyDisableSkypeUI
```

 **Вариант 3:** Используйте настраиваемый экземпляр политики, включающий параметр взаимодействия с клиентом Lync.
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName ClientPolicyNoIMURLDisableSkypeUI
```

После настройки политик клиентов разверните клиент Skype для бизнеса, создайте 4711,1002 (Апрель, 2015) или более поздней версии.
  
Подробные сведения о настройке взаимодействия с клиентом в Skype для бизнеса Online, в том числе инструкции по управлению процессом первого запуска и сценариями PowerShell, которые можно использовать для настройки среды, приведены в разделе [Переключение между интерфейсами пользователя Skype для бизнеса и клиента Lync](https://aka.ms/SfBOUI).
