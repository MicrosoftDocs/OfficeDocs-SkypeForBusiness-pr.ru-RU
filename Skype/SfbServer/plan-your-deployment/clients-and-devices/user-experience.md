# <a name="plan-the-skype-for-business-2015-client-experience-for-your-users"></a>Планирование клиентского опыта Skype для бизнеса 2015 для пользователей
 
**Сводка:** Узнайте о новом Skype для бизнеса и действиях, которые можно предпринять для подготовки среды и пользователей к обновлению, используете ли вы Skype для бизнеса Online, Skype для бизнеса Server 2019, Skype для бизнеса Server 2015, Lync Server 2013 или Lync Server 2010.
  
Обновление Office для Lync 2013 от 14 апреля 2015 г. включает новый пользовательский интерфейс Skype для бизнеса. Это обновление позволяет администраторам контролировать внешний вид клиента и выбирать, сохранять ли клиентский опыт Lync 2013 или использовать улучшенную клиентскую возможность Skype для бизнеса. Клиент Skype для бизнеса фактически заменил клиент Lync 2013 и добавил администраторам возможность выбора между существующим клиентом Lync и новым клиентом Skype для бизнеса. Сведения об этом обновлении см. в обновлениях от 14 апреля 2015 г. для [Lync 2013 (Skype для бизнеса) (KB2889923).](https://support.microsoft.com/kb/2889923/)
  
12 мая 2015 г. будет еще одно ежемесячное обновление от Office, которое включает обновленный клиент Skype для бизнеса. Многие клиенты, которые не применяли апрельские обновления, смогут получить обновление от 12 мая для Office 2013. Сведения в этом разделе помогут подготовить организацию, среду и пользователей к обновлению клиента. Чтобы сделать переход простым для пользователей и групп поддержки, используйте сведения в этом разделе, чтобы определить, какой клиентский опыт вы хотите для пользователей, а затем внести изменения в среду перед развертыванием обновления клиента в организации.
  
- [Какой клиентский опыт вы хотите для пользователей?](user-experience.md#clientexperience)
    
- [Подготовка среды для клиента Skype для бизнеса](user-experience.md#usinglync)
    
- [Ресурсы, которые помогут подготовить группы поддержки и конечных пользователей к обновлению](user-experience.md#support)
    
> [!NOTE]
> Клиентская возможность Lync 2013 не является вариантом для клиентских версий Skype для бизнеса 2016. Прежде чем пытаться настроить клиентскую среду для использования клиента Lync 2013, проверьте клиентскую версию, чтобы убедиться, что она не начинается с номера 16; например: 16.x.x.x. 
  
## <a name="what-client-experience-do-you-want-for-your-users"></a>Какой клиентский опыт вы хотите для пользователей?
<a name="clientexperience"> </a>

С помощью нового клиента Skype для бизнеса вы можете контролировать, какие клиенты получают пользователи, Lync или Skype для бизнеса. Клиент по умолчанию зависит от того, используете ли вы Lync или Skype для бизнеса на локальной основе или в Интернете. Если вы используете Skype для бизнеса Online (Lync Online) сегодня с Microsoft 365 Apps для предприятия, Microsoft 365 Business Standard или Office 2013, обновленная клиентская программа Skype для бизнеса, вдохновленная внешний вид Skype, будет работать по умолчанию. Если вы используете локальное использование Lync Server сегодня, по умолчанию будет работать клиент Lync.
  
Вы можете настроить клиентский опыт пользователей с помощью клиентских политик. Клиентская политика — это набор параметров конфигурации, которые применяются к пользователям при входе в Lync или Skype для бизнеса.
  
### <a name="skype-for-business-client-experience"></a>Клиентская программа Skype для бизнеса

Помимо всех функций Lync Skype для бизнеса предоставляет новые функции с упрощенным управлением и знакомыми значками из Skype. Некоторые новые функции в Skype для бизнеса доступны только с новым клиентом Skype для бизнеса. Дополнительные информацию о новых функциях в Skype для бизнеса см. в дополнительных приложениях [Discover Skype для бизнеса.](https://go.microsoft.com/fwlink/p/?LinkId=528686)
  
### <a name="lync-client-experience"></a>Клиентский опыт Lync

Клиентская возможность Lync очень похожа на клиентскую возможность Lync 2013, с чем пользователи уже знакомы, но есть несколько изменений, о которые необходимо дать пользователям знать. Чтобы узнать, чем отличается клиент Lync от клиента Lync 2013, см. в разделе Почему я вижу Skype для бизнеса при использовании [Lync?](https://go.microsoft.com/fwlink/p/?LinkId=544712) и дополнительные ссылки в этой теме.
  
## <a name="prepare-your-environment-for-the-skype-for-business-client"></a>Подготовка среды для клиента Skype для бизнеса
<a name="usinglync"> </a>

Необходимо сделать несколько вещей, чтобы ваша среда была готова к обновлению клиента. Прежде чем приступить к внесению каких-либо изменений для настройки клиентского опыта, сначала необходимо убедиться, что вы используете версию Skype для бизнеса Server или Lync Server, которая поддерживает параметры клиентской политики.
  
После подтверждения использования версии Skype для бизнеса Server или Lync Server, которая поддерживает параметры политики для управления клиентской деятельностью, необходимо настроить параметры политики в среде. Конкретные действия, которые необходимо выполнять, зависят от версии Skype для бизнеса Server или Lync Server, которые вы используете, а также от того, являются ли пользователи локальной или сетевой версией. 
  
Эти изменения необходимо внести до доставки клиентского обновления пользователям, чтобы вы могли управлять клиентом с первого запуска клиента Skype для бизнеса. В следующих таблицах указывается, какие действия необходимо предпринять для настройки среды для желаемого клиентского опыта для пользователей.

|Skype для бизнеса Online  <br/> |Нет дополнительных действий, кроме развертывания клиентской сборки 4711.1002 (апрель 2015 г.) или более поздней.  <br/> |[Использование клиентского опыта Lync в Skype для бизнеса Online](user-experience.md#LyncwithSfBO) <br/> |

## <a name="use-the-skype-client-experience-with-lync-server-2013-or-lync-server-2010-on-premises"></a>Использование клиентского опыта Skype с локальной помощью Lync Server 2013 или Lync Server 2010
<a name="SkypewithLynconprem"> </a>

Выполните действия в этом разделе, если вы хотите настроить клиентскую службу Skype в локальном развертывании. По умолчанию для локального
  
 **Шаг 1:** Во-первых, убедитесь, что вы работаете с версией Lync Server, которая поддерживает параметры клиентской политики.
  
- **Lync Server 2013** — необходимо запускать накопительное обновление за декабрь 2014 г. (5.0.8308.857) для Lync Server 2013 или более позднее обновление. Сведения см. [в обновлениях для Lync Server 2013](https://go.microsoft.com/fwlink/p/?LinkId=532772).
    
- **Lync Server 2010** — необходимо запускать накопительное обновление за февраль 2015 г. (4.0.7577.710) для Lync Server 2010 или более позднее обновление. Сведения см. [в обновлениях для Lync Server 2010.](https://go.microsoft.com/fwlink/p/?LinkId=532771)
    
  **Шаг 2:** Далее используйте клиентскую политику для набора клиентского опыта Skype с клиентом Skype для бизнеса. Существует **3 варианта использования** клиентской политики для настройки клиентского опыта.
  
  **Вариант 1:** Установите клиентскую базу Skype с помощью глобальной политики. Обратите внимание, что глобальная политика применяется ко всем пользователям в развертывании, но политики уровня пользователей и сайтов имеют приоритет над глобальной политикой:
  
```PowerShell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $True
```

 **Вариант 2:** Измените существующую клиентскую политику, используемую в среде, чтобы включить параметр, чтобы включить клиентскую возможность Skype. Это позволяет назначать клиентскую возможность Skype только тем пользователям, у кого назначена существующая политика:
  
```PowerShell
Set-CsClientPolicy -Identity ExistingClientPolicyName -EnableSkypeUI $True
```

 **Вариант 3:** Создайте новую политику, назначимую пользователям, которая включает параметры для клиентского опыта Skype. Сначала создайте новую клиентскую политику и укайте имя политики в качестве значения **параметра Identity:**
  
```PowerShell
New-CsClientPolicy -Identity UseSkypeUI -EnableSkypeUI $True
```

Затем назначьте политику пользователям, используя имя политики (значение, используемого для параметра **Identity)** в качестве значения **параметра PolicyName:**
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName UseSkypeUI
```

 **Шаг 3:** После настройки клиентских политик разверните клиент Skype для бизнеса, создайте 4711.1002 (апрель 2015 г.) или более позднее.
  
## <a name="use-the-lync-client-experience-with-lync-server-2013-or-lync-server-2010-on-premises"></a>Использование клиентской работы Lync с локальной помощью Lync Server 2013 или Lync Server 2010
<a name="LyncwithLynconprem"> </a>

По умолчанию клиент Skype для бизнеса развертывается в локальном развертывании Lync Server. Вам не нужно настраивать клиентские политики для использования клиентского опыта Lync, но может потребоваться управлять поведением первого запуска для клиента. По умолчанию при первом запуске клиентом Skype для бизнеса используется клиент Skype и отображается уведомление для пользователей, которые запрашивают перезапуск клиента для получения клиентского опыта Lync. Вы можете настроить среду так, чтобы клиентская среда Lync отображалась при первом запуске клиента пользователями, а также отключить клиентский учебник, изменяя реестр системы на клиентских компьютерах. Действия, которые необходимо выполнить перед развертывание клиента Skype для бизнеса, см. в одном из следующих разделов:
  
- **Lync Server 2013**, см. в руб. Настройка клиентского опыта в Skype для бизнеса [в Lync Server 2013](/previous-versions/office/lync-server-2013/configure-the-skype-for-business-client-in-lync-server-2013)
    
- **Lync Server 2010** см. в руб. Настройка клиентской работы с Skype для бизнеса [в Lync Server 2010](/previous-versions/office/skype-server-2010/dn955209(v=ocs.14))

## <a name="use-the-lync-client-experience-with-skype-for-business-online"></a>Использование клиентского опыта Lync в Skype для бизнеса Online
<a name="LyncwithSfBO"> </a>

Выполните действия в этом разделе, если вы хотите настроить клиентскую возможность Lync и использовать Skype для бизнеса Online.
  
Если вы используете Skype для бизнеса Online, вы по-прежнему можете использовать клиентскую возможность Lync с клиентом Skype для бизнеса в организации с помощью удаленной powerShell для настройки клиентских политик. Существует **3 варианта использования** клиентской политики для настройки клиентского опыта. Обратите внимание, что имена политик и параметров отличаются от параметров, которые используются для настройки клиентского опыта при использовании локального skype для бизнеса или Lync Server.
  
 **Вариант 1:** Установите клиентскую базу Lync с помощью глобальной политики. Обратите внимание, что политики клиента и сайта, применяемые к пользователям, будут иметь приоритет над глобальной политикой.
  
```PowerShell
Grant-CsClientPolicy -PolicyName ClientPolicyDisableSkypeUI
```

 **Вариант 2:** Измените существующую клиентскую политику, используемую в среде, чтобы включить параметр, чтобы включить клиентскую возможность Lync. Это позволяет назначать клиентскую возможность Lync только тем пользователям, у кого назначена существующая политика:
  
```PowerShell
Grant-CsClientPolicy -PolicyName ClientPolicyDisableSkypeUI
```

 **Вариант 3:** Используйте настраиваемый экземпляр политики, включаемый в параметр клиентского Lync.
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName ClientPolicyNoIMURLDisableSkypeUI
```

После настройки клиентских политик разверните клиент Skype для бизнеса, создайте 4711.1002 (апрель 2015 г.) или более позднее.
