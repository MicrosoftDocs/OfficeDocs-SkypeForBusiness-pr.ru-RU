# <a name="plan-the-skype-for-business-2015-client-experience-for-your-users"></a>Планирование работы с клиентом Skype для бизнеса 2015 для пользователей
 
**Сводка:** Learn about the new Skype for Business and the steps you can take to prepare your environment and your users for the update, whether you're using Skype for Business Online, Skype for Business Server 2019, Skype for Business Server 2015, Lync Server 2013, or Lync Server 2010.
  
Обновление Office для Lync 2013 за 14 апреля 2015 г. включает новый пользовательский интерфейс Skype для бизнеса. Это обновление позволяет администраторам контролировать внешний вид и внешний вид клиента, а также выбирать, следует ли сохранить клиентский опыт Lync 2013 или использовать улучшенный клиент Skype для бизнеса. Клиент Skype для бизнеса фактически заменил клиент Lync 2013 и добавил возможность администраторам выбирать между существующим и новым клиентом Skype для бизнеса. Сведения об этом обновлении см. в обновлении от [14 апреля 2015 г. для Lync 2013 (Skype для бизнеса) (KB2889923).](https://support.microsoft.com/kb/2889923/)
  
12 мая 2015 г. будет ежемесячная версия Office, включаемая обновленный клиент Skype для бизнеса. Многие клиенты, которые не применяли апрельские обновления, будут выбирать обновление для Office 2013 от 12 мая. Сведения в этом разделе помогут вам подготовить организацию, среду и пользователей к обновлению клиента. Чтобы у пользователей и групп поддержки было проще перейти на новую версию, воспользуйтесь сведениями в этом разделе, чтобы определить, какой клиентский процесс вы хотите использовать для пользователей, а затем внести изменения в среду перед развертыванием обновления клиента в организации.
  
- [Какой клиентский опыт вам нужен для пользователей?](user-experience.md#clientexperience)
    
- [Подготовка среды для клиента Skype для бизнеса](user-experience.md#usinglync)
    
- [Ресурсы по подготовке групп поддержки и конечных пользователей к обновлению](user-experience.md#support)
    
> [!NOTE]
> Клиентский опыт Lync 2013 не является вариантом для версий клиентов Skype для бизнеса 2016. Прежде чем пытаться настроить клиентскую среду для использования клиента Lync 2013, проверьте версию клиента, чтобы убедиться, что она не начинается с номера 16; например: 16.x.x.x. 
  
## <a name="what-client-experience-do-you-want-for-your-users"></a>Какой клиентский опыт вам нужен для пользователей?
<a name="clientexperience"> </a>

С помощью нового клиента Skype для бизнеса вы можете контролировать, какой клиент будет работать с вашими пользователями, Lync или Skype для бизнеса. Режим работы клиента по умолчанию зависит от того, используете ли вы Lync или Skype для бизнеса локально или через Интернет. Если вы используете Skype для бизнеса Online (Lync Online) в настоящее время с приложениями Microsoft 365 для предприятий, Microsoft 365 бизнес стандартный или Office 2013, обновленный интерфейс клиента Skype для бизнеса в стиле и стиле Skype будет стандартным пользовательским интерфейсом. Если вы используете Lync Server локально, по умолчанию используется клиент Lync.
  
Вы можете настроить, какой клиентский опыт получают пользователи, с помощью политик клиентов. Политика клиента — это набор параметров конфигурации, которые применяются к пользователям при входе в Lync или Skype для бизнеса.
  
### <a name="skype-for-business-client-experience"></a>Skype для бизнеса: клиентский опыт

Помимо всех функций Lync, Skype для бизнеса предоставляет новые функции с упрощенными средствами управления и знакомыми значками из Skype. Некоторые новые функции в Skype для бизнеса доступны только с новыми возможностями клиента Skype для бизнеса. Дополнительные информацию о новых функциях Skype для бизнеса см. в этой [теме.](https://go.microsoft.com/fwlink/p/?LinkId=528686)
  
### <a name="lync-client-experience"></a>Lync client experience

Клиентская Lync очень похожа на Lync 2013, с чем уже знакомы ваши пользователи, но есть несколько изменений, о чем вы хотите знать своим пользователям. To see what's different between the Lync client experience and the Lync 2013 client, see [Why do I see Skype for Business when I'm using Lync?](https://go.microsoft.com/fwlink/p/?LinkId=544712) and the additional links later in this topic.

После подтверждения использования версии Skype для бизнеса Server или Lync Server, которая поддерживает параметры политики для управления клиентской средой, необходимо настроить параметры политики в своей среде. ## <a name="use-the-skype-client-experience-with-lync-server-2013-or-lync-server-2010-on-premises"></a>Использование клиентского опыта Skype с локальной Lync Server 2013 или Lync Server 2010
<a name="SkypewithLynconprem"> </a>

Следуйте шагам в этом разделе, если вы хотите настроить клиентский опыт Skype в локальном развертывании. Стандартный режим для локальной работы
  
 **Шаг 1.** Сначала убедитесь, что вы работаете с версией Lync Server, которая поддерживает параметры политики клиента.
  
- **Lync Server 2013** — необходимо использовать накопительное обновление за декабрь 2014 г. (5.0.8308.857) для Lync Server 2013 или более позднее обновление. Сведения см. [в обновлениях для Lync Server 2013.](https://go.microsoft.com/fwlink/p/?LinkId=532772)
    
- **Lync Server 2010** — необходимо использовать накопительное обновление за февраль 2015 г. (4.0.7577.710) для Lync Server 2010 или более позднее обновление. Сведения см. [в обновлениях для Lync Server 2010.](https://go.microsoft.com/fwlink/p/?LinkId=532771)
    
  **Шаг 2.** Затем используйте клиентскую политику, чтобы настроить клиент Skype с клиентом Skype для бизнеса. Существует **3 варианта использования** клиентской политики для настройки работы с клиентом.
  
  **Вариант 1:** Настройка работы с клиентом Skype с помощью глобальной политики. Обратите внимание, что глобальная политика применяется ко всем пользователям в развертывании, но политики на уровне пользователей и сайтов имеют приоритет над глобальной политикой:
  
```PowerShell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $True
```

 **Вариант 2:** Измените существующую политику клиента, используемую в среде, чтобы включить параметр, позволяющий использовать клиент Skype. Это позволяет назначать клиентскую ставку Skype только тем пользователям, для них назначена существующая политика:
  
```PowerShell
Set-CsClientPolicy -Identity ExistingClientPolicyName -EnableSkypeUI $True
```

 **Вариант 3:** Создайте новую политику для назначения пользователям, которая включает параметр для работы с клиентом Skype. Сначала создайте новую клиентскую политику и укайте ее имя в качестве значения параметра **Identity:**
  
```PowerShell
New-CsClientPolicy -Identity UseSkypeUI -EnableSkypeUI $True
```

Затем назначьте политику пользователям, используя имя политики (значение, использованное для параметра **Identity)** в качестве значения параметра **PolicyName:**
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName UseSkypeUI
```

 **Шаг 3.** После настройки политик клиента разверните клиент Skype для бизнеса, сборку 4711.1002 (апрель 2015 г.) или более поздней.
  
## <a name="use-the-lync-client-experience-with-lync-server-2013-or-lync-server-2010-on-premises"></a>Использование клиентского опыта Lync с локальной Lync Server 2013 или Lync Server 2010
<a name="LyncwithLynconprem"> </a>

Это режим по умолчанию при развертывании клиента Skype для бизнеса в локальном развертывании Lync Server. Вам не нужно настраивать какие-либо клиентские политики для использования работы с клиентом Lync, но вы можете контролировать поведение при первом запуске для клиента. По умолчанию при первом запуске клиента Skype для бизнеса используется клиент Skype, а пользователям отображается уведомление о перезапуске клиента для получения клиентского опыта Lync. Вы можете настроить среду таким образом, чтобы при первом запуске клиента отображался клиент Lync, а также отключить руководство по клиенту, изменяя системный реестр на клиентских компьютерах. Действия, которые необходимо выполнить перед развертыванием клиента Skype для бизнеса, см. в одном из следующих разделов:
  
- **Lync Server 2013**, see [Configure the client experience with Skype for Business in Lync Server 2013](https://go.microsoft.com/fwlink/p/?LinkId=532732)
    
- **Lync Server 2010** see [Configure the client experience with Skype for Business in Lync Server 2010](https://go.microsoft.com/fwlink/p/?LinkId=532733)
    
## <a name="use-the-lync-client-experience-with-skype-for-business-server-on-premises"></a>Использование клиентского опыта Lync с локальной skype для бизнеса Server

Выполните действия из этого раздела, если вы хотите настроить клиентскую lync в локальном развертывании Skype для бизнеса Server.
  
Следуйте шагам в этом разделе, если вы хотите настроить клиентский опыт Skype в локальном развертывании. Стандартный режим для локальной работы
  
 **Шаг 1.** Сначала развернем Skype для бизнеса Server.
  
 **Шаг 2.** Затем используйте клиентскую политику, чтобы настроить клиентскую lync с клиентом Skype для бизнеса. Существует **3 варианта использования** клиентской политики для настройки работы с клиентом.
  
 **Вариант 1:** Настройка работы с клиентом Lync с помощью глобальной политики. Обратите внимание, что глобальная политика применяется ко всем пользователям в развертывании, но политики на уровне пользователей и сайтов имеют приоритет над глобальной политикой:
  
```PowerShell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $False
```

 **Вариант 2:** Измените существующую клиентскую политику, используемую в среде, чтобы включить параметр для обеспечения работы с клиентом Lync. Это позволяет назначать клиентскую lync только тем пользователям, для них назначена существующая политика:
  

 **Вариант 3:** Создайте новую политику для назначения пользователям, которая включает параметр для работы с клиентом Lync. Сначала создайте новую клиентскую политику и укайте ее имя в качестве значения параметра **Identity:**

Затем назначьте политику пользователям, используя имя политики (значение, использованное для параметра **Identity)** в качестве значения параметра **PolicyName:**
  
```PowerShell

```

 **Шаг 3.** Необязательный. По умолчанию при первом запуске клиента Skype для бизнеса используется клиент Skype, а пользователям отображается уведомление с просьбой перезапустить клиент, чтобы получить доступ к клиенту Lync. Вы можете настроить среду таким образом, чтобы при первом запуске клиента отображался клиент Lync, а также отключить руководство по клиенту, изменяя системный реестр на клиентских компьютерах. Действия, которые необходимо выполнить перед развертыванием клиента Skype для бизнеса, см. в подстройке "Настройка работы с [клиентом Skype для бизнеса".](../../deploy/deploy-clients/configure-the-client-experience.md)

## <a name="use-the-lync-client-experience-with-skype-for-business-online"></a>Использование клиентского опыта Lync со Skype для бизнеса Online
<a name="LyncwithSfBO"> </a>

Выполните действия из этого раздела, если вы хотите настроить клиентский режим Lync и использовать Skype для бизнеса Online.
  
Если вы используете Skype для бизнеса Online, вы по-прежнему можете использовать клиент Lync с клиентом Skype для бизнеса в организации с помощью удаленной powerShell для настройки политик клиентов. Существует **3 варианта использования** клиентской политики для настройки работы с клиентом. Обратите внимание, что имена политик и параметров отличаются от параметров, которые используются для настройки работы с клиентом при использовании Skype для бизнеса или локального сервера Lync Server.
  
 **Вариант 1:** Настройка работы с клиентом Lync с помощью глобальной политики. Обратите внимание, что политики клиентов и сайтов, применяемые к пользователям, будут иметь приоритет над глобальной политикой.
  
```PowerShell
Grant-CsClientPolicy -PolicyName ClientPolicyDisableSkypeUI
```

 **Вариант 2:** Измените существующую клиентскую политику, используемую в среде, чтобы включить параметр для обеспечения работы с клиентом Lync. Это позволяет назначать клиентскую lync только тем пользователям, для них назначена существующая политика:
  
```PowerShell
Grant-CsClientPolicy -PolicyName ClientPolicyDisableSkypeUI
```

 **Вариант 3:** Используйте экземпляр настраиваемой политики, включаемый параметр для работы с клиентом Lync.
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName ClientPolicyNoIMURLDisableSkypeUI
```

После настройки политик клиента разверните клиент Skype для бизнеса, сборку 4711.1002 (апрель 2015 г.) или более поздней.
  
Подробные сведения о настройке взаимодействия с клиентом в Skype для бизнеса Online, в том числе о том, как управлять интерфейсом первого запуска и сценарии PowerShell, которые можно использовать для настройки среды, см. в описании переключения между пользовательскими интерфейсами клиента Skype для бизнеса и [Lync.](https://aka.ms/SfBOUI)
