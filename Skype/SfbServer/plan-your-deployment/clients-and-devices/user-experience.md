# <a name="plan-the-skype-for-business-2015-client-experience-for-your-users"></a>Планирование взаимодействия с клиентами Skype для бизнеса 2015 для пользователей
 
**Сводка:** Узнайте о новых возможностях Skype для бизнеса и о том, как подготовить среду и пользователей к обновлению независимо от того, используете ли вы Skype для бизнеса Online, Skype для бизнеса Server 2019, Skype для бизнеса Server 2015, Lync Server 2013 или Lync Server 2010.
  
14 апреля 2015 Office для Lync 2013 включает новый пользовательский интерфейс Skype для бизнеса. Это обновление позволяет администраторам управлять внешним видом и поведением клиента и выбирать, следует ли сохранять интерфейс Lync 2013 или использовать улучшенный клиент для работы с клиентами Skype для бизнеса. Клиент Skype для бизнеса эффективно заменил клиент Lync 2013 и добавил возможность администраторам выбрать между существующими возможностями клиента Lync и новым клиентским интерфейсом Skype для бизнеса. Сведения об этом обновлении можно найти в статьях [14 апреля 2015 для Lync 2013 (Skype для бизнеса) (KB2889923)](https://support.microsoft.com/en-us/kb/2889923/).
  
12 мая 2015 г. в составе Office, который включает обновленный клиент Skype для бизнеса, будет ежемесячно обновляться. Многие пользователи, которые не применяли обновление за Апрель, получат обновление за 12 мая для Office 2013. Информация в этой теме поможет подготовить свою среду и своих пользователей к данному обновлению клиента. Чтобы облегчить переход для пользователей и групп поддержки, на основе представленной в этой теме информации выберите режим взаимодействия с клиентом для своих пользователей, а затем внесите в свою среду нужные изменения, прежде чем развертывать обновление клиента в своей организации.
  
- [What client experience do you want for your users?](user-experience.md#clientexperience)
    
- [Подготовка среды для клиента Skype для бизнеса](user-experience.md#usinglync)
    
- [Resources to help you prepare your support teams and your end users for the update](user-experience.md#support)
    
> [!NOTE]
> Взаимодействие с клиентом Lync 2013 не поддерживается в версиях клиентов Skype для бизнеса 2016. Перед настройкой клиентской среды на использование клиента Lync 2013 убедитесь, что версия клиента не начинается с цифр "16" (например: 16.x.x.x). 
  
## <a name="what-client-experience-do-you-want-for-your-users"></a>Какой режим взаимодействия с клиентом выбрать для своих пользователей?
<a name="clientexperience"> </a>

С помощью нового клиента Skype для бизнеса вы можете управлять тем, какие клиенты будут получать ваши пользователи: Lync или Skype для бизнеса. Взаимодействие с клиентом по умолчанию зависит от того, используете ли вы приложение Lync или Skype для бизнеса в локальной среде или в сети. Если вы используете Skype для бизнеса Online (Lync Online) в составе Office 365 профессиональный плюс, Office 365 Business Premium или Office 2013, обновленный клиентский интерфейс Skype для бизнеса — с учетом внешнего вида и функций Skype, будет использоваться по умолчанию. Если вы используете локальную версию Lync Server, в настоящее время интерфейс Lync будет использоваться по умолчанию.
  
Вариант взаимодействия пользователей с клиентом можно настроить с помощью политик клиента. Политика клиента — это набор параметров конфигурации, которые применяются к пользователям при входе в Lync или Skype для бизнеса.
  
### <a name="skype-for-business-client-experience"></a>Взаимодействие с клиентом Skype для бизнеса

Помимо всех функций Lync, в Skype для бизнеса предусмотрены новые функции, упрощенные элементы управления и знакомые значки из Skype. Некоторые новые возможности в Skype для бизнеса доступны только в новом интерфейсе клиента Skype для бизнеса. Дополнительные сведения о новых возможностях Skype для бизнеса можно найти в статье Знакомство со статьей [Skype для бизнеса](https://go.microsoft.com/fwlink/p/?LinkId=528686).
  
### <a name="lync-client-experience"></a>Взаимодействие с клиентом Lync

Взаимодействие с клиентским приложением Lync очень похоже на работу клиента Lync 2013, с которым пользователи уже знакомы, но есть несколько изменений, о которых вы хотите сообщить вашим пользователям. Чтобы узнать, какие отличия между интерфейсом Lync и клиентом Lync 2013, читайте в статье [почему я вижу Skype для бизнеса, когда я использую Lync?](https://go.microsoft.com/fwlink/p/?LinkId=544712) и дополнительные ссылки, приведенные далее в этом разделе.
  
## <a name="prepare-your-environment-for-the-skype-for-business-client"></a>Подготовка среды для клиента Skype для бизнеса

Чтобы подготовить свою среду к обновлению клиента, следует выполнить несколько действий. Прежде чем вносить изменения в настройку интерфейса клиента, необходимо убедиться в том, что вы используете версию Skype для бизнеса Server или Lync Server, которая поддерживает параметры клиентской политики.
  
После подтверждения того, что вы используете версию Skype для бизнеса Server или Lync Server, которая поддерживает параметры политики для управления интерфейсом клиента, вам потребуется настроить параметры политики в среде. Конкретные действия, которые необходимо выполнить, зависят от используемой версии Skype для бизнеса Server или Lync Server, а также от того, являются ли ваши пользователи локальными или подключенными к Интернету. 
  
Вы хотите внести эти изменения до того, как обновление клиента будет доставлено пользователям, чтобы вы могли управлять его интерфейсом при первом запуске клиента Skype для бизнеса. ## <a name="use-the-skype-client-experience-with-lync-server-2013-or-lync-server-2010-on-premises"></a>Использование клиента Skype для работы с Lync Server 2013 или Lync Server 2010 в локальной среде
<a name="SkypewithLynconprem"> </a>

В этом разделе описывается настройка взаимодействия с клиентом Skype в локальном развертывании. Это вариант взаимодействия по умолчанию для локальной установки.
  
 **Действие 1:** Прежде всего убедитесь, что у вас установлена версия Lync Server, поддерживающая параметры клиентской политики.
  
- **Lync server 2013** — для lync Server 2013 или более поздней версии вы должны использовать накопительное обновление за декабрь 2014 (5.0.8308.857). Дополнительные сведения можно найти в разделе [Обновление для Lync Server 2013](https://go.microsoft.com/fwlink/p/?LinkId=532772).
    
- **Lync server 2010** — для lync Server 2010 или более поздней версии вы должны использовать накопительное обновление февраля 2015 (4.0.7577.710). Дополнительные сведения можно найти в разделе [Обновление для Lync Server 2010](https://go.microsoft.com/fwlink/p/?LinkId=532771).
    
  **Шаг 2.** Затем используйте политику клиента, чтобы настроить интерфейс Skype для клиента Skype для бизнеса. Взаимодействие с клиентом можно задать с помощью политики клиента **3 способами**.
  
  **Способ 1.** Задать взаимодействие с клиентом Skype с помощью глобальной политики. Обратите внимание, что глобальная политика применяется ко всем пользователям развертывания, а у политик уровня пользователей и сайтов приоритет выше, чем у глобальной политики.
  
```PowerShell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $True
```

 **Способ 2.** Изменить используемую в среде политику клиента, включив настройку, разрешающую взаимодействие с клиентом Skype. Это позволяет назначить взаимодействие с клиентом Skype только пользователям, которым назначена существующая политика.
  
```PowerShell
Set-CsClientPolicy -Identity ExistingClientPolicyName -EnableSkypeUI $True
```

 **Способ 3.** Создать новую политику с параметром для взаимодействия с клиентом Skype, чтобы назначить пользователям. Сначала создайте новую политику клиента и предоставьте для нее имя как значение параметра **Идентификатор**.
  
```PowerShell
New-CsClientPolicy -Identity UseSkypeUI -EnableSkypeUI $True
```

Затем назначьте политику пользователям, используя имя политики (значение, использованное для параметра **Идентификатор**) в качестве значения параметра **PolicyName**.
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName UseSkypeUI
```

 **Шаг 3.** После того как вы настроили политики клиента, разверните клиент Skype для бизнеса, сборка 4711,1002 (Апрель, 2015) или более поздней версии.
  
## <a name="use-the-lync-client-experience-with-lync-server-2013-or-lync-server-2010-on-premises"></a>Работа с клиентом Lync с помощью Lync Server 2013 или Lync Server 2010 в локальной среде
<a name="LyncwithLynconprem"> </a>

Этот режим используется по умолчанию при развертывании клиента Skype для бизнеса в локальной среде Lync Server. Для использования взаимодействия с клиентом Lync не требуется настраивать политики клиента, но может возникнуть потребность в управлении поведением клиента при его первом выполнении. По умолчанию при первом запуске клиента Skype для бизнеса используется программа Skype для клиента, и для пользователей, которые запрашивают у них запрос на перезагрузку клиента Lync, выводится уведомление о том, что он перезапускает клиент. Можно настроить свою среду так, чтобы при запуске клиента начинающие пользователи переходили к взаимодействию с клиентом Lync, а также отключить руководство клиента, изменив реестр системы на клиентских компьютерах. Действия, которые необходимо выполнить перед развертыванием клиента Skype для бизнеса, можно найти в следующих статьях:
  
- **Lync server 2013**см. [в разделе Настройка взаимодействия с клиентами в Skype для бизнеса в Lync Server 2013](https://go.microsoft.com/fwlink/p/?LinkId=532732)
    
- **Lync server 2010** см. [в разделе Настройка взаимодействия с клиентами в Skype для бизнеса в Lync Server 2010](https://go.microsoft.com/fwlink/p/?LinkId=532733)
    
## <a name="use-the-lync-client-experience-with-skype-for-business-server-on-premises"></a>Использование взаимодействия с клиентом Lync в локальном развертывании Skype для бизнеса Server
<a name="LyncwithSfBServer"> </a>

Выполните действия, описанные в этом разделе, если вы хотите настроить взаимодействие с клиентом Lync в локальной среде развертывания Skype для бизнеса Server.
  
В этом разделе описывается настройка взаимодействия с клиентом Skype в локальном развертывании. Это вариант взаимодействия по умолчанию для локальной установки.
  
 **Действие 1:** Сначала выполните развертывание Skype для бизнеса Server.
  
 **Шаг 2.** Затем используйте политику клиента, чтобы настроить взаимодействие клиента Lync с клиентом Skype для бизнеса. Взаимодействие с клиентом можно задать с помощью политики клиента **3 способами**.
  
 **Способ 1.** Задать взаимодействие с клиентом Lync с помощью глобальной политики. Обратите внимание, что глобальная политика применяется ко всем пользователям развертывания, а у политик уровня пользователей и сайтов приоритет выше, чем у глобальной политики.
  
```PowerShell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $False
```

 **Способ 2.** Изменить используемую в среде политику клиента, включив настройку, разрешающую взаимодействие с клиентом Lync. Это позволяет назначить взаимодействие с клиентом Lync только пользователям, которым назначена существующая политика.
  
```PowerShell
Set-CsClientPolicy -Identity ExistingClientPolicyName -EnableSkypeUI $False
```

 **Способ 3.** Создать новую политику с параметром для взаимодействия с клиентом Lync, чтобы назначить пользователям. Сначала создайте новую политику клиента и предоставьте для нее имя как значение параметра **Идентификатор**.
  
```PowerShell
New-CsClientPolicy -Identity UseLyncUI -EnableSkypeUI $False
```

Затем назначьте политику пользователям, используя имя политики (значение, использованное для параметра **Идентификатор**) в качестве значения параметра **PolicyName**.
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName UseLyncUI
```

 **Шаг 3: необязательный** — по умолчанию при первом запуске клиента Skype для бизнеса используется интерфейс Skype для клиента, и пользователям выводится уведомление о необходимости перезапуска клиента, чтобы получить взаимодействие с клиентом Lync. Вы можете настроить среду таким образом, чтобы при первом запуске клиента выводиться приложение Lync, а также отключить учебник клиента, изменив системный реестр на клиентских компьютерах. Действия, которые необходимо выполнить перед развертыванием клиента Skype для бизнеса, можно найти в статье [Настройка взаимодействия с клиентами в Skype для бизнеса](../../deploy/deploy-clients/configure-the-client-experience.md).
  
 **Шаг 4:** После того как вы настроили политики клиента, разверните клиент Skype для бизнеса, сборка 4711,1002 (Апрель, 2015) или более поздней версии.
  
## <a name="use-the-lync-client-experience-with-skype-for-business-online"></a>Использование взаимодействия с клиентом Lync в локальном развертывании Skype для бизнеса Online
<a name="LyncwithSfBO"> </a>

Если вы хотите настроить взаимодействие с клиентом Lync и вы используете Skype для бизнеса Online, выполните действия, описанные в этом разделе.
  
Если вы используете Skype для бизнеса Online, вы по-прежнему можете использовать клиент Lync в клиенте Skype для бизнеса в своей организации, используя удаленную оболочку PowerShell для настройки политик клиентов. Взаимодействие с клиентом можно задать с помощью политики клиента **3 способами**. Обратите внимание, что имена политики и параметров отличаются от параметров, используемых для настройки взаимодействия с клиентами, если вы используете Skype для бизнеса или Lync Server в локальной среде.
  
 **Вариант 1:** Настройте взаимодействие с клиентом Lync с помощью глобальной политики. Обратите внимание, что политики клиентов и сайтов, примененные к пользователям, имеют приоритет над глобальной политикой.
  
```PowerShell
Grant-CsClientPolicy -PolicyName ClientPolicyDisableSkypeUI
```

 **Способ 2.** Изменить используемую в среде политику клиента, включив настройку, разрешающую взаимодействие с клиентом Lync. Это позволяет назначить взаимодействие с клиентом Lync только пользователям, которым назначена существующая политика.
  
```PowerShell
Grant-CsClientPolicy -PolicyName ClientPolicyDisableSkypeUI
```

 **Вариант 3:** Используйте настраиваемый экземпляр политики, который включает параметр для взаимодействия с клиентом Lync.
  
```PowerShell
Grant-CsClientPolicy username@contoso.com -PolicyName ClientPolicyNoIMURLDisableSkypeUI
```

После того как вы настроили политики клиента, разверните клиент Skype для бизнеса, сборка 4711,1002 (Апрель, 2015) или более поздней версии.
  
Подробные сведения о том, как настроить взаимодействие с клиентами в Skype для бизнеса Online, в том числе сведения о том, как управлять первым интерфейсом запуска и сценариями PowerShell, которые можно использовать для настройки среды, приведены в статье [Переключение между пользовательскими интерфейсами Skype для бизнеса и клиента Lync](https://aka.ms/SfBOUI).
