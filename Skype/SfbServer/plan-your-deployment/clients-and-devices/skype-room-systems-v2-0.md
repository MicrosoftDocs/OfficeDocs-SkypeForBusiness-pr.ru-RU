# <a name="plan-for-skype-room-systems-v2"></a>Планирование для Систем комнат Skype версии 2
 
В этой статье объясняется, соответствующие вопросы планирования развертывания систем комнаты Скайп v2 следующего поколения Скайп комнаты систем. 
  
 Систем комнаты Скайп версии 2 — корпорации Майкрософт предназначена для преобразования комнате для собраний в подробные, совместной работы Скайп для бизнеса программой последние решение конференц-связи. Обычные пользователи оценят хорошо знакомый интерфейс Skype для бизнеса, тогда как ИТ-администраторы смогут с легкостью развернуть приложение для собраний Skype для Windows 10 и управлять им. Версии 2 Скайп комнаты систем предназначен для использования существующего оборудования как Жидкокристаллических панелей для упрощения установки для переноса Скайп для бизнеса в комнате для собраний.
  
Скайп помещения систем v2 использует специальные UWP приложения, который выступает в качестве пользовательского интерфейса Скайп собрания. Она выполняется на 4 Surface Pro или Surface Pro в режиме консоли (после развертывания приложения UWP является единственным приложения, которое выполняется на устройстве) и требует собственную учетную запись устройства на вашей Скайп для реализации бизнеса. Это решение позволяет использовать существующее оборудование, например ЖК-панели и относительно недорогие периферийные камеры или микрофоны, для проведения эффективных собраний. Программное обеспечение обновляется как посредством Магазина Windows, так и посредством Центра обновления Windows.
  
Прежде чем начать подготовку среды, необходимо обеспечить наличие нужного оборудования и программного обеспечения. Для получения дополнительных сведений см [систем комнаты Скайп версии 2](requirements.md). 
  
> [!NOTE]
> Версии 2 Скайп комнаты систем предназначен для использования с Скайп для Business Server 2015 или Скайп для бизнеса в Интернет. Более ранних платформы, как Lync Server 2013 не требуются для работы с системами комнаты Скайп версии 2.

## <a name="preparing-your-skype-for-business-environment"></a>Подготовка среды Skype для бизнеса

Этот раздел содержит обзор действий, необходимых для подготовки среды, можно использовать все компоненты системы комнаты Скайп версии 2.
  
1. Подготовка учетной записи устройства для каждой версии 2 консоли Скайп комнаты систем. В разделе [Развертывание системы комнаты Скайп v2](../../deploy/deploy-clients/room-systems-v2.md) для получения дополнительных сведений.
    
2. Проверьте наличие рабочего подключения к сети или Интернету, которое будет использовать устройство.  
    
  - Необходимо иметь возможность получать IP-адреса с помощью DHCP (Примечание: Скайп комнаты систем v2 нельзя настроить статический IP-адрес при первом запуске единицы)
    
  - Необходимо открыть следующие порты (помимо обычных портов, открываемых для Skype для бизнеса).
    
  - HTTPS: 443
    
  - HTTP: 80
    
  - Если в вашей сети работает прокси-сервер, вам также потребуются его адрес и данные скрипта.
    
    > [!NOTE]
    > Версии 2 не поддерживает ввод HDCP, что наблюдалась вызвать проблемы с HDMI систем комнаты Скайп потребления функциональные возможности (видео). Будьте внимательны для обеспечения отключите параметры HDCP коммутаторы, подключенных к версии 2 Скайп комнаты систем. 
  
3. Корпорация Майкрософт осуществляет сбор данных с целью улучшения своих продуктов. Чтобы обеспечить сбор данных, необходимо добавить следующие сайты в белый список.
    
  - Конечная точка клиента телеметрии:https://vortex.data.microsoft.com/
    
  - Конечная точка параметры телеметрии:https://settings.data.microsoft.com/
    
### <a name="create-and-test-a-device-account"></a>Создание тестовой учетной записи устройства

*Устройства учетная запись* является учетной записью, клиентом версии 2 Скайп комнаты систем для доступа к функции из Exchange, например календаря и для включения Скайп для бизнеса. В разделе [Развертывание системы комнаты Скайп v2](../../deploy/deploy-clients/room-systems-v2.md) для получения дополнительных сведений.
  
### <a name="check-network-availability"></a>Проверка доступности сети

Для правильной, устройства v2 систем комнаты Скайп должна иметь доступ к проводной сети, которая соответствует следующим требованиям:
  
- Доступ к экземпляру Active Directory или Azure Active Directory (Azure AD), а также серверам Microsoft Exchange и Skype для бизнеса.
    
- Доступ к серверу, предоставляющему IP-адреса через DHCP. Версии 2 систем комнаты Скайп нельзя настроить статический IP-адрес.
    
- Доступ к HTTP-портам 80 и 443.
    
- Порты TCP и UDP, настроенных как описано в разделе [требования к портам для Lync Server 2013](https://technet.microsoft.com/en-us/library/gg398798%28v=ocs.15%29.aspx) для локально Скайп для реализации бизнеса или [диапазоны IP-адресов и URL-адреса Office 365](https://support.office.com/en-us/article/Office-365-URLs-and-IP-address-ranges-8548a211-3fe7-47cb-abb1-355ea5aa88a2?ui=en-US&amp;rs=en-US&amp;ad=US) для Скайп для реализации online Business.
    
> [!IMPORTANT]
> Для обеспечения достаточной пропускной способности используйте проводное сетевое подключение стандарта 1 Гбит/с.  
  
### <a name="certificates"></a>Сертификаты

Устройства системы комнаты Скайп версии 2 использует сертификаты для веб-служб Exchange, Скайп для бизнеса, использование сети и проверки подлинности. Если связанные серверы используют открытые сертификаты (как правило, в развертываниях Online и некоторых локальных реализациях), со стороны администратора не требуется никаких дополнительных действий по установке сертификатов. Если используется закрытый центр сертификации (как правило, в локальных развертываниях), на устройстве необходимо настроить доверие такому ЦС, то есть установить на нем сертификаты ЦС и цепочки ЦС. При добавлении устройства в домен эта задача может быть выполнена автоматически.
  
Сертификаты устанавливаются так же, как для любого другого клиента Windows.  
  
> [!NOTE]
> Чтобы использовать Скайп для Business Server версии 2 Скайп комнаты систем, может потребоваться сертификаты. 
  
### <a name="proxy"></a>Прокси-сервер

Версии 2 систем комнаты Скайп предоставляет наследование параметров прокси-сервера с ОС Windows. Для доступа к параметрам ОС Windows выполните следующие действия.
  
1. В пользовательском Интерфейсе версии 2 Скайп комнаты систем щелкните значок шестеренки параметров, где будет выведен запрос для пароля локального администратора на устройстве (пароль по умолчанию — «sfb»).
    
2. Нажмите кнопку «» на «Параметры», а затем кнопку «Перейти к Windows» и выбрав и на «есть Admin входа в» кнопки и затем нажать кнопку «Администратор» (если компьютер присоединен к домену выберите «Другого пользователя», а затем используйте. \admin имени пользователя).
    
3. В «поиска Windows» введите нижней левой в редакторе реестра (длинный press экрана либо справа щелкните и выберите «Запуск от имени администратора»).
    
4. Щелкните папку HKEY_USERS (вы увидите список идентификаторов SID пользователей компьютера) и убедитесь в том, что выбрана корневая папка HKEY_USERS.
    
    Будет выведен запрос имени ключа для вашей вновь загруженного куста; Введите в Скайп (должна появиться параметры реестра для пользователя Скайп).
    
5. В меню "Файл" выберите пункт "Загрузить куст".
    
6. Перейдите к папке C:\Users\Skype, в поле "Имя файла" введите NTUSER.dat и нажмите кнопку "Открыть".
    
7. Откройте раздел Скайп и перейдите к HKEY_USERS\Skype\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet параметры, затем убедитесь, что эти параметры будут введены: 
    
    [HKEY_USERS\Skype\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings]
    
    «MigrateProxy» = DWORD: 00000001
    
    «ProxyEnable» = DWORD: 00000001
    
    «ProxyServer"="xx.xx.xx.xx:8080»
    
    Если параметр ProxyServer не существует, его может потребоваться добавить в качестве строкового параметра. Замените xx.xx.xx.xx:8080 на IP-адрес или имя узла и порт вашего прокси-сервера.
    
8. После завершения внесения изменений выделения пользователя Скайп ключевые (корневой папки для Скайп) и выберите команду Выгрузить куст из меню файл реестра (будет выведен запрос на подтверждение - выберите **Да** ).
    
9. Теперь можно закрыть редактор реестра и ввести logoff в поле "Поиск в Windows".
    
10. Вернувшись на экран входа, выберите пользователя **Skype**. Если все предыдущие шаги выполнены успешно, устройство версии 2 Скайп комнаты систем будет вход успешно.

### <a name="admin-group-management"></a>Управление группой администраторов

После присоединения к домену вы можете настроить группу безопасности в качестве локального администратора с помощью групповой политики или оснастки управления локальным компьютером так же, как для компьютера с ОС Windows в своем домене. Любой участник этой группы безопасности сможет ввести свои учетные данные и разблокировать раздел "Параметры".
  
> [!NOTE]
> Если устройство v2 систем комнаты Скайп теряет отношение доверия с домена (например, если удалить v2 систем комнаты Скайп из домена после его присоединен к домену), нельзя будет для проверки подлинности в устройство и откройте копирование параметров. Обходным решением является вход с использованием учетной записи локального администратора. 
  
## <a name="local-accounts"></a>Локальные учетные записи

### <a name="skype-room-systems-local-user-account"></a>Локальная учетная запись пользователя системы комнат Skype

Учетная запись устройства, как правило, не использует пароль. При необходимости ей можно назначить пароль, однако это может иметь негативные последствия, включая вероятность блокировки консольного приложения при истечении срока действия пароля. Соответственно, администратор должен установить настройки таким образом, чтобы срок действия пароля не истекал.
  
### <a name="admin---local-administrator-account"></a>Admin — учетная запись локального администратора

Пароль по умолчанию версии 2 Скайп комнаты систем имеет значение «sfb». Пароль может быть изменен локально, перейдя к параметрам Windows \> перейти в Windows или в файле AutoUnattend.xml (использования диспетчера образа системы Windows с ADK на внесение изменений в XML-файл).
  
> [!CAUTION]
> Не забудьте изменить пароль v2 систем комнаты Скайп как можно скорее. 
  
Кроме того, для управления паролем локального администратора можно настроить групповую политику, в которой администраторы домена получают права локальных администраторов.
  
Пароль локального администратора нельзя выбрать во время настройки.
  
### <a name="machine-account"></a>Учетная запись компьютера

Гораздо как любого устройства Windows имя компьютера, имя переименовать можно щелкнуть правой кнопкой мыши в параметрах \> о \> переименование ПК.
  
 Если вы хотите переименовать компьютер после присоединения к домену, используйте команду Переименовать компьютер PowerShell с новое имя компьютера.
