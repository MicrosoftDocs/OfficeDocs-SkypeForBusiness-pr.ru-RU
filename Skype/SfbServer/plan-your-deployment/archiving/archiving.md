# <a name="plan-for-archiving-in-skype-for-business-server"></a>Планирование архива в Skype для бизнеса Server
 
**Сводка:** Ознакомьтесь с этой темой, чтобы узнать, как планировать архивировать в Skype для бизнеса Server.
  
К корпорациям и другим организациям применяется все больше отраслевых и государственных нормативов, требующих сохранения определенного типа коммуникаций. Если в вашей организации есть такие требования, архивировать в Skype для бизнеса Server можно для архива сообщений обмена мгновенными сообщениями (im) и conferencing (meeting) для поддержки некоторых требований соответствия требованиям.
  
## <a name="archiving-components"></a>Компоненты архива

Skype для бизнеса Server использует следующие компоненты архива:
  
- **Агенты архивации**. Агенты архива (также известные как агенты единого сбора данных) устанавливаются и активируются автоматически на каждом выпуск Enterprise и выпуск Standard Server. Хотя агенты архива активируются автоматически, сообщения фактически не захватываются до тех пор, пока архивативная система не будет включена и надлежащим образом настроена. По умолчанию архивка отключена.
    
- **Хранилище данных архивации**. Хранение данных для Skype для бизнеса Server может быть реализовано в Skype для бизнеса Server SQL Server баз данных или, если у вас есть развертывание Exchange, интегрированное с Exchange хранилищем. 
    
Архивативная система также требует хранения файлов, но для архивирования используется то же хранилище файлов, что и серверы переднего выпуск Standard Server.

  
## <a name="determine-your-organizations-requirements-for-archiving"></a>Определение требований к архивам для организаций

Для реализации архива необходимо решить, как выполнить требования организации по архивам, определив следующее:
  
- **Какой параметр хранения использовать**. Можно реализовать хранилище одним из двух способов или использовать сочетание обоих:
    
  - **Exchange хранения.** Если развертывание Exchange, можно интегрировать Skype для бизнеса Server и Exchange архивации, чтобы архивировать Skype для бизнеса Server и Exchange архивные данные в Exchange. Если включить вариант интеграции Microsoft Exchange, почтовые ящики пользователей, которые помещались в Exchange Server, используют Exchange хранения для архивных данных, но только в том случае, если почтовые ящики были In-Place Hold. По умолчанию интеграция microsoft Exchange не включена.
    
  - **Skype для бизнеса Server хранения.** Если у вас есть пользователи, которые не размещены в Exchange или у которых не были размещены почтовые ящики In-Place Hold, или если вы не хотите использовать интеграцию Microsoft Exchange для всех пользователей в развертывании, вы можете развернуть Skype для бизнеса Server базы данных архивации с помощью SQL Server.
    
- **При развертывании архива**. Архивации можно развернуть в рамках начального развертывания Skype для бизнеса Server или добавить в существующее развертывание. Чтобы использовать Skype для бизнеса Server архивирования (SQL Server баз данных), вы используете Topology Builder, чтобы добавить базы данных в топологию, а затем опубликовать топологию еще раз. Если все пользователи находятся в Exchange и их почтовые ящики наложены на In-Place Hold, вам не нужно обновлять топологию, а нужно только включить интеграцию Microsoft Exchange для хранения архивных данных в Exchange. 
    
- **Какие сайты и пользователи в организации требуют архива.** Можно настроить параметры архива для всей организации и, необязательно, для определенных сайтов, пулов, пользователей и групп пользователей.
    
- **Какой контент следует архивировать.** Если вы указываете архивативную информацию на глобальном уровне или для определенных сайтов и пользователей, на каждом из этих уровней необходимо указать, следует ли включить следующие типы контента: 
    
  - одноранговые мгновенные сообщения;
    
  - конференции (собрания), которые являются сеансами обмена мгновенными сообщениями между несколькими сторонами;
    
  - содержимое конференций, включая загруженное содержимого (например, раздаточные материалы), и связанное с событиями содержание (например, сведения о присоединении, выходе, загрузке, общем доступе и изменениях видимости);
    
  - доски и общие опросы конференции.
    
- **Какой контент нельзя архивировать.** Архивировать следующие типы контента невозможно: 
    
  - одноранговые передачи файлов;
    
  - аудио/видео для одноранговых мгновенных сообщений и конференций;
    
  - совместное использование рабочего стола и приложений для одноранговых мгновенных сообщений и конференций.
    
    Skype для бизнеса Server также не архивировать постоянные беседы чата. Чтобы архивировать постоянные беседы в чате, необходимо включить и настроить службу соответствия требованиям, которая является компонентом, который может быть развернут с помощью persistent Chat Server. Подробные сведения см. [в материале Plan for Persistent Chat Server в Skype для бизнеса Server 2015 г.](../../plan-your-deployment/persistent-chat-server/persistent-chat-server.md)

    > [!NOTE] 
    > Постоянный чат доступен в Skype для бизнеса Server 2015 г., но больше не поддерживается Skype для бизнеса Server 2019 г. Такая же функциональность доступна в Teams. Дополнительные сведения см. в ссылке Начало работы [с обновлением Microsoft Teams обновления.](/microsoftteams/upgrade-start-here) Если вам нужно использовать постоянный чат, вы можете либо перенести пользователей, требующих Teams, либо продолжить использование Skype для бизнеса Server 2015 г. 
    
- **Как долго должны храниться архивные материалы.** База данных архивирования не предназначена для долгосрочного хранения, и Skype для бизнеса Server не предоставляет решение электронного обнаружения (поиска) для архивных данных, поэтому данные должны быть перемещены в другое хранилище. Skype для бизнеса Server предоставляет средство экспорта сеансов, которое можно использовать для экспорта архивных данных и которое создает для поиска расшифровку архивных данных. 
    
     Для глобальной политики, а также для каждого создаемого сайта и политики пользователей можно указать время очистки архивных и экспортных данных. Дополнительные сведения о чистке данных см. в дополнительных сведениях об управлении чисткой архивных данных в [Skype для бизнеса Server.](../../manage/archiving/purging-of-archived-data.md) Дополнительные сведения об использовании средства экспорта сеансов см. в [Skype для бизнеса Server.](../../manage/archiving/export-archived-data.md)
    
- **Архивировать внутренние или внешние коммуникации.** Можно включить архивативную проверку внутренних коммуникаций (связи между внутренними пользователями), внешних коммуникаций (сообщений, включающих по крайней мере одного пользователя за пределами внутренней сети) или обоих. Вы можете указать эти параметры для всей организации или указать их для определенных сайтов и пулов. По умолчанию не включен ни один из вариантов.
    
    > [!NOTE]
    > Управление архивативом внутренних или внешних коммуникаций доступно только для Skype для бизнеса Policy. Для Exchange архива внутренних и внешних коммуникаций архивируются или не архивируются. 
  
- **Следует ли внедрять критический режим.** Если архивации является требованием для вашей организации, настройка критического режима блокирует сеансы чата и конференции в случае сбоя Skype для бизнеса Server, который предотвратит архивации. Например: 
    
  - Проблема с службой Skype для бизнеса Server хранения. В этом случае чат заблокирован для пользователей, которые включены для архива.
    
  - Недоступная доля файлов или проблема со службой хранения. В этом случае все активные конференции, размещенные в пуле на момент сбоя, переводятся в ограниченный режим, после чего можно активировать новые конференции.
    
    Сеансы обмена мгновенными сообщениями и конференции автоматически восстанавливаются после устранения ошибок.
    
## <a name="choose-archiving-deployment-and-configuration-options"></a>Выбор вариантов развертывания и конфигурации архивации

Архивировать автоматически устанавливается на каждом переднем сервере при развертывании сервера, но архивативная система не включена до его настройки. Настройка архива определяется тем, как его развертывать. Архивативную часть можно развернуть одним из следующих способов:
  
- Использование хранилища Exchange Microsoft
    
- Использование Skype для бизнеса Server хранения
    
> [!NOTE]
> Если вы реализуете Skype для бизнеса Server баз данных архивации и включаете интеграцию Microsoft Exchange, политики Exchange переопределяют политики Skype для бизнеса Server архивации, но только для пользователей, которые находятся в Exchange и имеют свои почтовые ящики. надев In-Place Удержание. Skype для бизнеса архива зависит от политики удержания Exchange In-Place Microsoft. 
  
При развертывании архивации для одного пула переднего выпуск Standard Server необходимо включить его для всех остальных пулов и выпуск Standard серверов в развертывании. Если архивативная обработка не включена в пуле, где находится беседа или собрание, все данные конференции не могут быть архивироваться. Архивативная работа по-прежнему будет работать для сообщений чата, но содержимое и события для конференций не могут быть архивироваться.
  
> [!NOTE]
> Чтобы включить делегирования административных задач при сохранении стандартов безопасности организации, Skype для бизнеса Server использует управление доступом на основе ролей (RBAC). С помощью RBAC административная привилегия предоставляется путем назначения пользователям заранее заданных административных ролей. Чтобы настроить Skype для бизнеса политик и конфигураций архивации, пользователю необходимо приурочено к роли CsArchivingAdministrator (если конфигурация не будет сделана непосредственно на сервере, на котором развертывается архивация, а не удаленно с другого компьютера). Список прав, разрешений и ролей пользователей, необходимых для развертывания архивации, см. в списке [Развертывание архивации для Skype для бизнеса Server.](../../deploy/deploy-archiving/deploy-archiving.md) 
  
> [!NOTE]
> Если вы используете microsoft Exchange интеграцию, конфигурация политик Exchange требует соответствующих прав и разрешений администратора. Подробные сведения см. в Exchange документации. 
  
### <a name="microsoft-exchange-storage"></a>Хранилище Exchange Microsoft

 Если вы выбираете Exchange Microsoft, Exchange политики и конфигурации для управления архиватива Skype для бизнеса Server. Вы можете настроить вариант интеграции microsoft Exchange на глобальном уровне, уровне сайта и уровне пула. Если развертывание включает несколько лесов, необходимо синхронизировать параметры между Skype для бизнеса Server и Exchange. Необходимо определить:

- Следует ли реализовать критический режим, который блокирует сеансы im и conferencing в случае Skype для бизнеса Server сбоя

Сведения о настройке политик Exchange In-Place и параметров для поддержки архивирования см. в Exchange документации по продуктам.
  
### <a name="skype-for-business-server-storage"></a>Skype для бизнеса Server хранения

Если вы Skype для бизнеса Server хранилище, Skype для бизнеса Server политики и конфигурации архивации для управления тем, как включена и реализована архивация. Skype для бизнеса Server хранилища SQL Server баз данных, поэтому необходимо добавить соответствующие SQL Server базы данных в топологию, а затем настроить политики архивирования. 
  
### <a name="add-storage-databases-to-your-topology"></a>Добавление баз данных хранения в топологию

При добавлении SQL Server баз данных хранения в топологию можно скомплексировать базы данных архивации с помощью любого из следующих ниже:
  
- База данных мониторинга
    
- Серверная база данных пула переднего плана Enterprise Edition
    
> [!NOTE]
> На сервере, на котором размещается база данных архивации, могут размещаться и другие базы данных. Однако, когда вы рассматриваете возможность выровненного размещения базы данных архивации совместно с другими базами данных, следует помнить, что при архивации сообщений большого количества пользователей для базы данных архивации может потребоваться очень большой объем дискового пространства. По этой причине не рекомендуется размещать базу данных архивации совместно с серверной базой данных. 
  
При коллокации базы данных архивации с базой данных мониторинга, задней базой данных или обеими этими базами данных можно использовать один экземпляр SQL для любой или всех баз данных, или можно использовать отдельный экземпляр SQL для каждой базы данных со следующим ограничением: каждый экземпляр SQL может содержать только одну базу данных,  единой базы данных мониторинга и одной базы данных архива.
  
Подробные сведения о коллабации всех ролей и баз данных серверов см. в материале [Topology Basics for Skype для бизнеса Server.](../../plan-your-deployment/topology-basics/topology-basics.md) Дополнительные сведения об обновлении топологии для хранения баз данных см. в материале [Create and publish new topology in Skype для бизнеса Server.](../../deploy/install/create-and-publish-new-topology.md)
  
### <a name="determine-archiving-options-and-user-policies"></a>Определение вариантов архива и политик пользователей

Необходимо определить:
  
- Включить или отключить архивировать внутренние и внешние коммуникации
    
- Архивировать im, conferencing или оба
    
- Следует ли реализовать критический режим, который блокирует сеансы im и conferencing в случае Skype для бизнеса Server сбоя 
    
- Включить ли политики для определенных пользователей и групп
    
Skype для бизнеса Server Параметры архива могут быть указаны на следующих уровнях. 
  
- **Параметр Глобального уровня**. Это конфигурация архивации по умолчанию и применяется ко всему развертыванию. Он создается при развертывании Skype для бизнеса Server и по умолчанию отключает архивную проверку внутренних и внешних коммуникаций. Этот параметр нельзя удалить. При выборе параметра удаления глобальный параметр сброшен в параметры по умолчанию.
    
- **Параметры уровня сайта**. Вы можете включить или отключить архивировать для одного или более определенных сайтов, создав и настроив параметр архива на уровне сайта. Вы можете удалить любой вариант архива на уровне сайта, который вы создаете. Параметр архива на уровне сайта переопределяет глобальный параметр, но только для сайта, указанного в параметре. 
    
    Например, если включить архивацию внутренних и внешних коммуникаций в глобальной конфигурации и создать конфигурацию сайта, в которой вы отключите архивацию внешних коммуникаций, для этого сайта будут архивироваться только внутренние коммуникации. Например, если включить архивацию только для im в глобальной конфигурации и создать конфигурацию сайта, в которой вы включаете архивацию для im и conferencing, то для сайта будут архивироваться только конференциации, а не для остальной части организации.
    
- **Параметры уровня пула**. Можно указать параметры архивации для одного или более определенных пулов, создав и настроив конфигурацию уровня пула для отдельного пула. Конфигурация архивации на уровне пула существует только при ее создании. Вы можете изменить и удалить любую конфигурацию архивации на уровне пула. Конфигурация архивации на уровне пула переопределяет глобальную конфигурацию и любую созданную конфигурацию архивации сайтов. 
    
    Например, предположим, что вы включаете архивацию для im только в глобальной конфигурации, затем создайте конфигурацию на уровне сайта, в которой вы включаете архивацию для im и conferencing, а затем создайте конфигурацию уровня пула, в которой вы включаете архивацию только для im. Сообщения будут архивироваться как для мгновенных сообщений, так и для всех пользователей сайта, за исключением пользователей, которые были в пуле, указанном в конфигурации уровня пула. Для всех остальных пользователей организации архивативная деятельность будет включена только для чата.
    
- **Политики архива пользователей**. Вы можете включить или отключить архивировать для одного или более определенных пользователей и групп пользователей путем создания, настройки и применения политики архива на уровне пользователей для указанных пользователей и групп пользователей. Вы можете удалить любую создаваемую политику архива на уровне пользователей и изменить, к какой группе пользователей применяется политика архива. Политика архива на уровне пользователей переопределяет глобальную политику и любые политики сайтов, но только для пользователей и групп пользователей, к которым применяется политика. 
    
    Например, предположим, что вы отключите архивацию внутренних и внешних коммуникаций в глобальной конфигурации, создайте политику на уровне сайта, в которой вы включаете архивацию внутренних и внешних коммуникаций, а затем создайте политику на уровне пользователей, в которой вы отключите архивацию для внешних коммуникаций. Сообщения будут архивироваться как для внешних, так и для внутренних коммуникаций для всех пользователей сайта, за исключением пользователей, к которым применяется политика на уровне пользователей, для этих пользователей будут архивироваться только внутренние коммуникации.
    
Дополнительные сведения о настройке начальных конфигураций архивации при развертывании архивации см. в материале [Deploy archiving for Skype для бизнеса Server.](../../deploy/deploy-archiving/deploy-archiving.md) Сведения об управлении архиванием после развертывания см. в материале [Manage archiving in Skype для бизнеса Server.](../../manage/archiving/archiving.md)

## <a name="access-archived-data"></a>Доступ к архивным данным

Доступ к архивированным данным зависит от места хранения данных: 
  
- **Microsoft Exchange хранения**. Если вы выбираете вариант Exchange интеграции, Skype для бизнеса Server хранение контента архивации в хранилище Exchange для всех пользователей, которые находятся в Exchange, и у которых были наложены почтовые ящики In-Place Hold. Архивные данные хранятся в папке извлекаемых элементов в почтовых ящиках пользователей, которая обычно незаметна для пользователей и может искаться только пользователями с ролью Exchange **discovery Management.** Exchange, если она развернута, включает федераированный поиск и обнаружение, а также SharePoint. Дополнительные сведения о хранении, хранении и обнаружении данных, хранимой в Exchange, см. в Exchange и SharePoint документации.
    
- **Skype для бизнеса Server хранения архивирования.** Если вы Skype для бизнеса Server базы данных архива, Skype для бизнеса Server вкладываю содержимое архива в базы данных архива Skype для бизнеса Server для всех пользователей, не помещающихся в Exchange и не наложенных на In-Place Hold. Эти данные недоступны для поиска, но их можно экспортировать в форматы, доступные для поиска с использованием других средств. Сведения об экспорте данных, хранимых в базах данных архива, см. в материале [Export archived data in Skype для бизнеса Server.](../../manage/archiving/export-archived-data.md)
