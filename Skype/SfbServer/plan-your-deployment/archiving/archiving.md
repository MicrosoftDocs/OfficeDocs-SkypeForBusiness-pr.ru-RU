# <a name="plan-for-archiving-in-skype-for-business-server"></a>Планирование архива в Skype для бизнеса Server
 
**Сводка:** В этом разделе вы узнаете, как планировать архивировать в Skype для бизнеса Server.
  
К корпорациям и другим организациям применяется все больше отраслевых и государственных нормативов, требующих сохранения определенного типа коммуникаций. Если в вашей организации есть такие требования, вы можете архивировать обмен мгновенными сообщениями и обмен мгновенными сообщениями (собрания) с помощью архива в Skype для бизнеса Server, чтобы обеспечить поддержку некоторых требований соответствия требованиям.
  
## <a name="archiving-components"></a>Компоненты архива

Skype для бизнеса Server использует следующие компоненты архива:
  
- **Агенты архивации**. Агенты архивизации (также известные как унифицированные агенты сбора данных) устанавливаются и активируются автоматически в каждом пуле переднего сервера Enterprise Edition и сервере Standard Edition. Хотя агенты архива активируются автоматически, никакие сообщения не захватываются до тех пор, пока архивироваться не будет включено и правильно настроено. По умолчанию архивная архива отключена.
    
- **Хранилище данных архивации**. Хранилище данных Для Skype для бизнеса Server можно реализовать как базы данных Skype для SQL Server или, если у вас развернуто развертывание Exchange, интегрированное с хранилищем Exchange. 
    
Для архивирования также требуется хранилище файлов, но для архивирования используется то же хранилище файлов, что и на серверах переднего сервера или сервере Standard Edition.

  
## <a name="determine-your-organizations-requirements-for-archiving"></a>Определение требований организации к архивам

Для реализации архивных реализации необходимо решить, как выполнить требования организации к архивам, определив следующее:
  
- **Какой вариант хранения использовать.** Вы можете реализовать хранилище одним из двух способов или использовать сочетание обоих:
    
  - **Хранилище Exchange.** Если у вас есть развертывание Exchange, вы можете интегрировать Skype для бизнеса Server и архивации Exchange, чтобы архивные данные Skype для бизнеса Server и Exchange хранились вместе в Exchange. Если включить параметр интеграции Microsoft Exchange, почтовые ящики пользователей, хранимые на сервере Exchange Server используют хранилище Exchange для архивных данных, но только в том случае, если почтовые ящики были поставлены на удержание In-Place хранения. По умолчанию интеграция Microsoft Exchange не включена.
    
  - **Хранилище Skype для бизнеса Server.** Если у вас есть пользователи, которые не размещены в Exchange или почтовые ящики которых не были поставлены на удержание In-Place, или если вы не хотите использовать интеграцию Microsoft Exchange для каких-либо или всех пользователей в развертывании, вы можете развернуть базы данных архивации Skype для бизнеса Server с помощью SQL Server.
    
- **Когда развертывать архив.** Архив можно развернуть в рамках первоначального развертывания Skype для бизнеса Server или добавить в существующее развертывание. To use Skype for Business Server archiving storage (SQL Server databases), you use Topology Builder to add the databases to your topology, and then publish the topology again. Если все пользователи находятся в Exchange и их почтовые ящики находятся на удержании In-Place, вам не нужно обновлять топологию, а только включить интеграцию Microsoft Exchange для хранения архивных данных в Exchange. 
    
- **Какие сайты и пользователи в организации требуют архива.** Можно настроить параметры архива для всей организации и( при желании) для определенных сайтов, пулов, пользователей и групп пользователей.
    
- **Какой контент следует архивировать.** Указываете ли вы архив на глобальном уровне или для определенных сайтов и пользователей на каждом из этих уровней, необходимо указать, следует ли включить следующие типы контента: 
    
  - одноранговые мгновенные сообщения;
    
  - конференции (собрания), которые являются сеансами обмена мгновенными сообщениями между несколькими сторонами;
    
  - содержимое конференций, включая загруженное содержимого (например, раздаточные материалы), и связанное с событиями содержание (например, сведения о присоединении, выходе, загрузке, общем доступе и изменениях видимости);
    
  - доски и общие опросы конференции.
    
- **Какой контент нельзя архивировать.** Архивировать контент следующих типов невозможно: 
    
  - одноранговые передачи файлов;
    
  - аудио/видео для одноранговых мгновенных сообщений и конференций;
    
  - совместное использование рабочего стола и приложений для одноранговых мгновенных сообщений и конференций.
    
    Skype для бизнеса Server также не архивировать беседы сохраняемой беседы. Чтобы архивировать беседы сохраняемой беседы, необходимо включить и настроить службу соответствия, которая является компонентом, который можно развернуть с помощью сервера сохраняемой беседы. For details, see [Plan for Persistent Chat Server in Skype for Business Server 2015](../../plan-your-deployment/persistent-chat-server/persistent-chat-server.md).

    > [!NOTE] 
    > Сохраняемая беседа доступна в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения [см. в сведениях о том,](/microsoftteams/upgrade-start-here)как начать обновление Microsoft Teams. Если необходимо использовать сохраняемого чата, вы можете либо перенести пользователей, которым требуются эти функции, в Teams, либо продолжить использование Skype для бизнеса Server 2015. 
    
- **Срок сохранения архивных материалов.** База данных архивирования не предназначена для долгосрочного хранения, и Skype для бизнеса Server не предоставляет решение обнаружения электронных данных (поиска) для архивных данных, поэтому данные необходимо перемещать в другое хранилище. Skype для бизнеса Server предоставляет средство экспорта сеансов, которое можно использовать для экспорта архивных данных и которое создает записи архивных данных, которые можно найти. 
    
     Для глобальной политики, а также для каждого создаемого сайта и пользовательской политики можно указать время очистки архивных и экспортных данных. Дополнительные сведения о защите данных см. в подсети ["Управление purging of archived data in Skype for Business Server".](../../manage/archiving/purging-of-archived-data.md) Дополнительные сведения об использовании средства экспорта сеансов см. в экспорте архивных данных [в Skype для бизнеса Server.](../../manage/archiving/export-archived-data.md)
    
- **Следует ли архивировать внутренние или внешние коммуникации.** Архив можно включить для внутренних коммуникаций (коммуникаций между внутренними пользователями), внешних коммуникаций (коммуникаций, включающих по крайней мере одного пользователя за пределами внутренней сети) или для обоих этих сообщений. Вы можете указать эти параметры для всей организации или для определенных сайтов и пулов. По умолчанию ни один из вариантов не включен.
    
    > [!NOTE]
    > Управление архивативами для внутренней или внешней связи доступно только для политики Skype для бизнеса. Для архивных данных, интегрированных с Exchange, как внутренние, так и внешние коммуникации архивируются или не архивируются. 
  
- **Следует ли реализовать критический режим.** Если для вашей организации существует требование архивации, настройка критического режима заблокировала сеансы im и conferencing в случае сбоя Skype для бизнеса Server, который предотвратит архивации. Например: 
    
  - Проблема со службой хранилища Skype для бизнеса Server. В этом случае для пользователей с включенной архивативаной поддержкой мгновенных данных блокируется im.
    
  - Недоступное файловое хранилище или проблема со службой хранения. В этом случае все активные конференции, размещенные в пуле на момент сбоя, переводятся в ограниченный режим, после чего можно активировать новые конференции.
    
    Сеансы обмена мгновенными сообщениями и конференции автоматически восстанавливаются после устранения ошибок.
    
## <a name="choose-archiving-deployment-and-configuration-options"></a>Выбор параметров развертывания и настройки архивации

Архивная система автоматически устанавливается на каждом сервере переднего сервера при развертывании сервера, но архивировать ее можно только после ее настройки. Настройка архива определяется тем, как она развертывается. Архив можно развернуть одним из следующих способов:
  
- Использование хранилища Microsoft Exchange
    
- Использование хранилища Skype для бизнеса Server
    
> [!NOTE]
> Если вы реализуете обе базы данных архивации Skype для бизнеса Server и включаете интеграцию с Microsoft Exchange, политики Exchange переопределяют политики архивации Skype для бизнеса Server, но только для пользователей, которые находятся в Exchange и для их почтовых ящиков было In-Place удержание. Архивная система Skype для бизнеса зависит от политики удержания In-Place Microsoft Exchange. 
  
При развертывании архивации для одного пула переднего сервера или сервера Standard Edition необходимо включить ее для всех остальных пулов переднего сервера и серверов Standard Edition в развертывании. Если в пуле, в котором находится беседа или собрание, не включена архивная архива, архивироваться могут не все данные конференции. Архивировать все равно можно для мгновенных сообщений, но содержимое и события для видеоконференций могут не архивироваться.
  
> [!NOTE]
> Чтобы включить делегирования административных задач при сохранении стандартов безопасности организации, Skype для бизнеса Server использует управление доступом на основе ролей (RBAC). С помощью RBAC административные привилегии лагаются путем назначения пользователям предварительно заданных административных ролей. Чтобы настроить политики и конфигурации архивации Skype для бизнеса, пользователю должна быть назначена роль CsArchivingAdministrator (если настройка не была сделана непосредственно на сервере, на котором развернута архивация, а не удаленно с другого компьютера). Список прав, разрешений и ролей пользователя, необходимых для развертывания архивации, см. в подтипе "Развертывание архивации для [Skype для бизнеса Server".](../../deploy/deploy-archiving/deploy-archiving.md) 
  
> [!NOTE]
> Если вы используете интеграцию Microsoft Exchange, для настройки политик Exchange требуются соответствующие права и разрешения администратора. Подробные сведения см. в документации Exchange. 
  
### <a name="microsoft-exchange-storage"></a>Хранилище Microsoft Exchange

 При выборе интеграции с Microsoft Exchange политики и конфигурации Exchange используются для управления архивативами Skype для бизнеса Server. Параметр интеграции Microsoft Exchange можно настроить на глобальном уровне, на уровне сайта и на уровне пула. Если развертывание включает несколько лесов, необходимо синхронизировать параметры между Skype для бизнеса Server и Exchange. Необходимо определить:
  
- Следует ли архивировать мгновенные документы, или и то, и другое, или и то, и другое
    
- Следует ли реализовать критический режим, блокющий сеансы im и conferencing в случае сбоя Skype для бизнеса Server 
    
- Выбор варианта интеграции с Microsoft Exchange для использования Exchange для хранения архивных данных
    
Сведения о настройке политик и параметров In-Place Exchange для поддержки архивирования см. в документации по продукту Exchange.
  
### <a name="skype-for-business-server-storage"></a>Хранилище Skype для бизнеса Server

При выборе хранилища Skype для бизнеса Server политики и конфигурации архивации Skype для бизнеса Server используются для управления тем, как архивация включена и реализована. Хранилище Skype для бизнеса Server использует SQL Server баз данных, поэтому вам потребуется добавить соответствующие базы данных SQL Server в топологию, а затем настроить политики архивирования. 
  
### <a name="add-storage-databases-to-your-topology"></a>Добавление баз данных хранилища в топологию

При добавлении SQL Server баз данных хранения в топологию можно выбрать вариант выполнив следующие условия:
  
- База данных мониторинга
    
- Серверная база данных пула переднего плана Enterprise Edition
    
> [!NOTE]
> На сервере, на котором размещается база данных архивации, могут размещаться и другие базы данных. Однако, когда вы рассматриваете возможность выровненного размещения базы данных архивации совместно с другими базами данных, следует помнить, что при архивации сообщений большого количества пользователей для базы данных архивации может потребоваться очень большой объем дискового пространства. По этой причине не рекомендуется размещать базу данных архивации совместно с серверной базой данных. 
  
При выполнив вымеав базу данных архивации с базой данных мониторинга, тыловой базой данных или обеими этими базами данных, можно использовать один экземпляр SQL для любой или всех баз данных либо отдельный экземпляр SQL для каждой базы данных со следующим ограничением: каждый экземпляр SQL может содержать только одну тыловую базу данных, одну базу данных мониторинга и одну базу данных архивации.
  
For details about collocation of all server roles and databases, see [Topology Basics for Skype for Business Server.](../../plan-your-deployment/topology-basics/topology-basics.md) Дополнительные сведения об обновлении топологии с учетом баз данных хранилища см. в сведениях о создании и публикации новой топологии [в Skype для бизнеса Server.](../../deploy/install/create-and-publish-new-topology.md)
  
### <a name="determine-archiving-options-and-user-policies"></a>Определение параметров архива и пользовательских политик

Необходимо определить:
  
- Следует ли включить или отключить архивировать для внутренних и внешних коммуникаций
    
- Следует ли архивировать мгновенные документы, или и то, и другое, или и то, и другое
    
- Следует ли реализовать критический режим, блокющий сеансы im и conferencing в случае сбоя Skype для бизнеса Server 
    
- Следует ли включить политики для определенных пользователей и групп
    
Параметры архива Skype для бизнеса Server можно у указаны на следующих уровнях. 
  
- **Параметр глобального уровня.** Это конфигурация архивации по умолчанию, которая применяется ко всему развертыванию. Он создается при развертывании Skype для бизнеса Server и по умолчанию отключает архив для внутренней и внешней связи. Удалить этот параметр невозможно. Если выбрать вариант удаления, для глобального параметра будут сброшены параметры по умолчанию.
    
- **Параметры уровня сайта.** Вы можете включить или отключить архивировать для одного или более определенных сайтов, создав и настроив для этого сайта параметр архива на уровне сайта. Можно удалить любой параметр архива на уровне сайта, который вы создаете. Параметр архива на уровне сайта переопределяет глобальный параметр, но только для сайта, указанного в параметре. 
    
    Например, если включить архивацию для внутренних и внешних коммуникаций в глобальной конфигурации и создать конфигурацию сайта, в которой вы отключите архивацию для внешних коммуникаций, для этого сайта будут архивироваться только внутренние коммуникации. Например, если включить архивацию только для мгновенных мгновенных данных в глобальной конфигурации и создать конфигурацию сайта, в которой архивация включается как для мгновенных, так и для видеоконференций, то архивация будет вестись только для сайта, а не для остальной части организации.
    
- **Параметры уровня пула.** Можно указать параметры архивации для одного или более определенных пулов, создав и настроив конфигурацию на уровне пула для отдельного пула. Конфигурация архивации на уровне пула существует только при ее создании. Можно изменить и удалить любую конфигурацию архивации на уровне пула. Конфигурация архивации на уровне пула переопределяет глобальную конфигурацию и любую созданную конфигурацию архивации сайта. 
    
    Например, предположим, что архивация для im включается только в глобальной конфигурации, затем создайте конфигурацию на уровне сайта, в которой включается архивация для мгновенных и для видеоконференций, а затем создайте конфигурацию уровня пула, в которой архивация включается только для мгновенных устройств. Коммуникации будут архивироваться как для мгновенных сообщений, так и для всех пользователей сайта, кроме пользователей, которые были в пуле, указанном в конфигурации уровня пула. Для всех остальных пользователей в организации архив можно включить только для мгновенных данных.
    
- **Политики архива пользователя.** Вы можете включить или отключить архивировать для одного или более определенных пользователей и групп пользователей, создав, настроив и применив политику архива на уровне пользователя для указанных пользователей и групп пользователей. Можно удалить любую политику архива на уровне пользователя, которую вы создаете, и изменить пользователей и группу пользователей, к которым применяется политика архива. Политика архива на уровне пользователя переопределяет глобальную политику и все политики сайта, но только для пользователей и групп пользователей, к которым применяется политика. 
    
    Например, предположим, что вы отключите архивацию для внутренней и внешней связи в глобальной конфигурации, создайте политику уровня сайта, в которой вы включаете архивацию для внутренней и внешней связи, а затем создайте политику на уровне пользователя, в которой вы отключите архивацию для внешних коммуникаций. Коммуникации будут архивироваться как для внешних, так и для внутренних коммуникаций для всех пользователей сайта, кроме тех пользователей, к которым применяется политика на уровне пользователя. Для этих пользователей будут архивироваться только внутренние коммуникации.
    
Дополнительные сведения о настройке начальных конфигураций архивации при развертывании архивации см. в подтипе "Развертывание архивации для [Skype для бизнеса Server".](../../deploy/deploy-archiving/deploy-archiving.md) For details about managing archiving after deployment, see [Manage archiving in Skype for Business Server.](../../manage/archiving/archiving.md)

## <a name="access-archived-data"></a>Доступ к архивным данным

Доступ к архивированным данным зависит от места хранения данных: 
  
- **Хранилище Microsoft Exchange.** Если выбрать вариант интеграции Exchange, Skype для бизнеса Server будет хранить содержимое архивации в хранилище Exchange для всех пользователей, которые находятся в Exchange и почтовые ящики которых были поставлены на удержание In-Place хранения. Архивные данные хранятся в папке элементов с возможностью восстановления в почтовых ящиках пользователей, которая обычно невидима для пользователей и может искаться только пользователями с ролью управления обнаружением **Exchange.** При развертывании Exchange включает федераированный поиск и обнаружение вместе с SharePoint. Дополнительные сведения о хранении, хранении и обнаружении данных, хранимой в Exchange, см. в документации по Exchange и SharePoint.
    
- **Хранилище архивирования Skype для** бизнеса Server. Если вы настроили базы данных архива Skype для бизнеса Server, Skype для бизнеса Server передает содержимое архива в базы данных архива Skype для бизнеса Server для всех пользователей, которые не были установлены в Exchange и почтовые ящики которых не были поставлены на удержание In-Place. Эти данные недоступны для поиска, но их можно экспортировать в форматы, доступные для поиска с использованием других средств. For details about exporting data stored in Archiving databases, see [Export archived data in Skype for Business Server.](../../manage/archiving/export-archived-data.md)
