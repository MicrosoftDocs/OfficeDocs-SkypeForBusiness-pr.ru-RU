# <a name="topology-basics-for-skype-for-business-server"></a>Основы топологии Skype для бизнеса Server

**Сводка.** Выберите топологию для Skype для бизнеса Server. Сведения о совместном размещении серверов для Skype для бизнеса Server.

Прежде чем готовиться к чему-либо другому, важно убедиться, что выбрана подходящая топология для развертывания Skype для бизнеса Server. В первую очередь необходимо решить, будете вы использовать локальное развертывание Skype для бизнеса Server или совместите его со службой Skype для бизнеса Server Online в гибридном развертывании. В любом случае рекомендуем читать дальше, так как здесь мы подробно рассмотрим локальные топологии, но сведения о гибридном развертывании представлены в отдельном разделе.

Примеры топологий представлены в статье [Эталонные топологии для Skype для бизнеса Server](reference-topologies.md).

## <a name="sites"></a>Сайты

В Skype для бизнеса Server вы указываете в своей сети сайты, на которых будут храниться компоненты Skype для бизнеса Server. Сайт – это набор компьютеров, соединенных с помощью сети с высокой пропускной способностью и низкой задержкой, например локальной сети, или две сети, соединенные с помощью высокоскоростного оптоволоконного канала. Обратите внимание, что сайты Skype для бизнеса Server никак не связаны с сайтами доменных служб Active Directory и сайтами Microsoft Exchange Server. Сайты Skype для бизнеса Server не должны соответствовать вашим сайтам Active Directory.

Skype для бизнеса Server поддерживает локальное развертывание одного или нескольких сайтов, которые можно масштабировать в соответствии с требованиями к высокой доступности и расположению.

В развертывании будет хотя бы один центральный сайт (он также является центром обработки данных для всех размещенных на нем серверов), а каждый центральный сайт в развертывании будет содержать один сервер Standard Edition или хотя бы один интерфейсный пул Enterprise Edition. Ниже перечислены различия между выпусками.

- Сервер Standard Edition включает базу данных SQL Server Express с совместным размещением.

- Интерфейсный пул Enterprise Edition включает:

  - один или несколько серверов переднего плана (в идеале хотя бы три для масштабируемости), но не более 12. Балансировка нагрузки потребуется для нескольких серверов;

  - отдельный тыловой сервер.

Дополнительные сведения о различных ролях серверов представлены далее в этом разделе.

Помимо центральных сайтов, также можно развернуть один или несколько сайтов филиалов, связанных с центральным сайтом. Почти все их функциональные возможности зависят от центрального сайта. Так из чего именно они состоят?

- Устройство для обеспечения связи в филиалах, в котором совмещаются шлюз телефонной сети общего пользования (ТСОП) и некоторые функции Skype для бизнеса Server.

- Сервер для обеспечения связи в филиалах — это сервер с Windows Server, на котором установлены регистратор Skype для бизнеса Server и программный сервер-посредник.

- Изолированный шлюз ТСОП (который не входит в состав устройства для обеспечения связи в филиалах).

- Автономный сервер-посредник или пул таких серверов (если вы не хотите сочетать эту роль с устройством для обеспечения связи в филиалах).

## <a name="whats-in-a-skype-for-business-server-site"></a>Состав сайта Skype для бизнеса Server

Рассмотрим этот вопрос подробнее. Центральный сайт также может включать перечисленные ниже компоненты.

- Несколько интерфейсных пулов, которые могут быть размещены в одном или в разных доменах (помните, что все серверы переднего плана в интерфейсном пуле, а также тыловой сервер для этого пула должны находиться на одном домене).

- Несколько серверов Standard Edition.

- Пограничный сервер или пограничный пул (в сети периметра). Они нужны, если развертывание должно поддерживать федеративных партнеров, общедоступную службу обмена мгновенными сообщениями, XMPP-шлюз и доступ удаленных пользователей. Дополнительные сведения см. в документации по планированию пограничного сервера.

Используется для поддержки сбора данных в целях обеспечения качества услуг (QoE) аудио- и видеосвязи (A/V) и регистрации вызовов (CDR) для службы "Корпоративная голосовая связь", а также аудио- и видеоконференций в развертывании. - Директор или пул директоров. Вы можете развернуть до 10 директоров на пул. Если вас интересует эта возможность, прочтите статью "Планирование для директоров".

- Обратный прокси-сервер. Он не является компонентом Skype для бизнеса Server, но его рекомендуется добавить в среду, если вам нужна поддержка общего доступа к веб-содержимому для федеративных пользователей или поддержка мобильного трафика либо вашим пользователям необходима адресная книга, возможность присоединяться к собраниям и другие функции. Когда вы будете готовы, ознакомьтесь с дополнительными сведениями в статье, посвященной настройке обратного прокси-сервера.

## <a name="server-roles"></a>Роли сервера

Каждый сервер, на котором работает Skype для бизнеса Server, выполняет одну или несколько ролей сервера. Роль сервера — это определенный набор функций Skype для бизнеса Server, предоставляемых этим сервером. Нет необходимости развертывать все доступные роли сервера в сети. Установите только те роли сервера, которые предоставляют нужные функции.

В целях улучшения масштабируемости и высокой доступности для большинства ролей серверов можно развертывать пулы, состоящие из нескольких серверов, на которых запущена одна и та же роль сервера. Каждый сервер в пуле должен выполнять одну и ту же роль (или роли). Для большинства типов пулов в Skype для бизнеса Server необходимо развернуть подсистему балансировки нагрузки для распределения трафика между различными серверами в пуле. Skype для бизнеса Server поддерживает как балансировку нагрузки на основе службы доменных имен (DNS), так и аппаратные подсистемы балансировки нагрузки.

### <a name="front-end-server-and-back-end-server"></a>Сервер переднего плана и тыловой сервер

В Skype для бизнеса Server Enterprise Edition основной ролью сервера является сервер переднего плана, на котором работают многие основные функции Skype для бизнеса Server. Сервер переднего плана и тыловые серверы — единственные роли сервера, которые обязательно должны присутствовать в любом развертывании Skype для бизнеса Server Enterprise Edition.

Пул переднего плана — это набор настроенных идентичным образом серверов переднего плана, которые работают вместе для предоставления служб общей группе пользователей. Пул предоставляет пользователям возможности масштабирования и отработки отказа.

Сервер переднего плана предоставляет следующие возможности:

- проверка подлинности и регистрация пользователей;

- сведения о присутствии и обмен карточками контактов;

- службы адресной книги и расширение списка рассылки;

- функции обмена мгновенными сообщениями, включая многопользовательские беседы;

- веб-конференции, конференц-связь с телефонным подключением через ТСОП, аудио- и видеоконференции (если они развернуты);

- размещение как приложений, входящих в состав Skype для бизнеса Server (например, помощника по конференц-связи и приложения "Группа ответа"), так и сторонних приложений;

- дополнительно: мониторинг для сбора сведений об использовании в форме записей регистрации вызовов и записей ошибок вызовов. Эти сведения содержат метрики о качестве мультимедиа-содержимого (аудио и видео), передаваемого по сети при вызове службы "Корпоративная голосовая связь", а также во время аудио- и видеоконференций;

- веб-компоненты для поддержки веб-задач, таких как веб-планировщик и компонент обеспечения процесса присоединения.

- дополнительно: архивация как отдельных мгновенных сообщений, так и собраний для соответствия нормативным требованиям. Более подробные сведения см. в разделе [Planning for Archiving](https://technet.microsoft.com/library/898b83c1-007b-43be-9484-08fe49c10951.aspx) документации по развертыванию.

    В Lync Server 2010 и более ранних версиях мониторинг и архивация были разными ролями сервера, которые не могли работать вместе на сервере переднего плана.

- При необходимости, если сохраняемый чат включен, веб-службы сохраняемого чата для управления комнатами чата и веб-службы сохраняемого чата для передачи и загрузки файлов.

Пулы переднего плана – это также основное хранилище для данных пользователей и конференций. Информация о каждом пользователе реплицируется среди трех серверов переднего плана в пуле, а ее резервная копия создается на фоновых серверах.

Кроме того, один сервер переднего плана в развертывании также содержит центральный сервер управления, который управляет данными базовой конфигурации и развертывает их на всех серверах Skype для бизнеса Server. Центральный сервер управления также предоставляет возможности командной консоли Lync Server и передачи файлов.

Тыловые серверы — это серверы баз данных, на которых работает Microsoft SQL Server. Они предоставляют службы баз данных для интерфейсного пула. Тыловые серверы служат в качестве хранилищ резервных копий для данных пользователей и конференций пула, а также основных хранилищ для других баз данных, таких как база данных группы ответа. У вас может быть один тыловой сервер, но для отработки отказа рекомендуем обеспечить [высокую доступность тылового сервера в Skype для бизнеса Server](../high-availability-and-disaster-recovery/back-end-server.md). На тыловых серверах не выполняется никакое программное обеспечение Skype для бизнеса Server.

> [!IMPORTANT]
> Не рекомендуем размещать базы данных Skype для бизнеса Server совместно с другими базами данными. В противном случае могут пострадать доступность и производительность.

> [!NOTE]
> Зеркальное отображение SQL доступно в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. В Skype для бизнеса Server 2019 рекомендуется использовать группы доступности AlwaysOn, экземпляры отказоустойчивых кластеров AlwaysOn (FCI), и методы отказоустойчивой кластеризации SQL.

Данные, хранящиеся в базах данных фонового сервера, включают сведения о присутствии, контактные списки пользователей, данные конференций, включая постоянные данные о состоянии всех текущих конференций, а также данные планирования конференций.

### <a name="edge-server"></a>Пограничный сервер

Пограничный сервер предоставляет пользователям возможность общаться и совместно работать с пользователями за пределами брандмауэров организации. В число этих внешних пользователей могут входить собственные пользователи организации, которые в настоящее время работают удаленно, пользователи из федеративных партнерских организаций, а также внешние пользователи, приглашенные к участию в конференциях, размещенных в развертывании Skype для бизнеса Server.

Развертывание пограничного сервера также дает возможность использовать службы мобильности, поддерживающие функции Lync на мобильных устройствах. Пользователи могут использовать поддерживаемые мобильные устройства на платформе Apple iOS, Android, Windows Phone или Nokia, чтобы выполнять такие действия, как отправка и прием мгновенных сообщений, просмотр контактов и просмотр сведений о присутствии. Кроме того, некоторые мобильные устройства поддерживают такие функции службы "Корпоративная голосовая связь", как присоединение к конференции одним щелчком, звонки с рабочего телефона, доступ по одному номеру, голосовая почта и пропущенные звонки. Функция мобильности также поддерживает push-уведомления для мобильных устройств, на которых приложения не могут работать в фоновом режиме. Push-уведомление — это передаваемое на мобильное устройство уведомление о событии, которое произошло в то время, когда мобильное приложение было неактивно.

Пограничные серверы также включают полностью интегрированный прокси-сервер XMPP, при этом шлюз XMPP встроен в серверы переднего плана. Вы можете настроить эти компоненты XMPP, чтобы пользователи Skype для бизнеса Server могли добавлять контакты партнеров на основе XMPP для обмена мгновенными сообщениями и состоянием присутствия.

> [!NOTE]
> Прокси-серверы и шлюзы XMPP доступны в Skype для бизнеса Server 2015, но больше не поддерживаются в Skype для бизнеса Server 2019. Дополнительные сведения см. в статье [Перенос федерации XMPP](../../../SfBServer2019/migration/migrating-xmpp-federation.md).

### <a name="mediation-server"></a>Сервер-посредник

Сервер-посредник является необходимым компонентом для реализации службы "Корпоративная голосовая связь", функции "Позвонить с рабочего" и конференц-связи с телефонным подключением. Сервер-посредник преобразует сигналы и, в некоторых конфигурациях, мультимедиа между внутренней инфраструктурой Skype для бизнеса Server и телефонной сетью общего пользователя (ТСОП), IP-УАТС и магистралями SIP. Сервер-посредник может работать на одном компьютере с сервером переднего плана или может быть выделен в отдельный пул серверов-посредников.

Дополнительные сведения см. в статье [Компонент сервера-посредника в Skype для бизнеса Server](../../plan-your-deployment/enterprise-voice-solution/mediation-server.md).

### <a name="video-interop-server"></a>Сервер видеовзаимодействия

Сервер видеовзаимодействия — это новая роль, которая появилась в Skype для бизнеса Server 2015. Она позволяет интегрировать развертывание Skype для бизнеса Server с определенными сторонними решениями VTC (системами видеоконференций). VIS выступает в качестве промежуточного звена между сторонней системой видеоконференций и развертыванием Skype для бизнеса Server. В этом выпуске VIS в основном используется для взаимодействия с видеосистемами Cisco и Tandberg.

Дополнительные сведения см. в статье [Планирование сервера видеосвязи в Skype для бизнеса Server](../../plan-your-deployment/video-interop-server.md).

### <a name="director"></a>Директор

Директоры могут выполнять проверку подлинности запросов пользователей Skype для бизнеса Server, но не содержат учетных записей пользователей и не предоставляют службы конференций или присутствия. Директоры оказываются особенно полезными при необходимости повышения уровня безопасности в развертываниях, где необходим доступ внешних пользователей. Директор может выполнять проверку подлинности запросов, прежде чем передавать эти запросы на внутренние серверы. При начале атак по принципу отказа в обслуживании атаки заканчиваются на директоре и не достигают серверов переднего плана.

### <a name="persistent-chat-server-roles"></a>Роли серверов сохраняемого чата

> [!NOTE]
> Сохраняемый чат доступен в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения см. в статье [Начало перехода на Microsoft Teams](/microsoftteams/upgrade-start-here). Если вам нужно использовать сохраняемый чат, то вы можете либо перенести пользователей, которым нужна эта функция, в Teams, либо продолжать использовать Skype для бизнеса Server 2015.

Возможности сохраняемого чата позволяют пользователям участвовать в многопользовательских тематических беседах, которые подлежат сохранению. На сервере переднего плана сохраняемого чата работает служба сохраняемого чата. Внутренний сервер сохраняемого чата хранит данные журнала бесед и сведения о категориях и комнатах чата. Дополнительный внутренний сервер сохраняемого чата для соответствия требованиям может хранить содержимое чата и события соответствия, которые будут использоваться для обеспечения соответствия требованиям.

На серверах под управлением Skype для бизнеса Server Standard Edition сохраняемый чат также может выполняться на одном сервере. Сервер переднего плана для сохраняемого чата невозможно размещать совместно с сервером переднего плана Enterprise Edition.

Дополнительные сведения см. в статье [Планирование для сервера сохраняемого чата в Skype для бизнеса Server 2015](../../plan-your-deployment/persistent-chat-server/persistent-chat-server.md).

## <a name="high-availability-and-disaster-recovery-support"></a>Поддержка высокой доступности и аварийного восстановления

Skype для бизнеса Server обеспечивает высокую доступность за счет избыточности серверов, объединенных в пул. При сбое сервера с определенными ролями остальные серверы в пуле с такими же ролями берут на себя нагрузку этого сервера. Это касается серверов переднего плана, пограничных серверов, серверов-посредников и Директоров.

Skype для бизнеса Server также поддерживает аварийное восстановление путем сопряжения пулов. Если развернуть эту топологию, будут назначены пары интерфейсных пулов, при этом каждый пул из этой пары размещается в отдельном центре обработке данных и в отдельном регионе. Если один пул или сайт завершает работу, можно перенаправить пользователей этого пула в другой пул пары, что приведет к минимальному перерыву в обслуживании.

Skype для бизнеса Server также поддерживает несколько вариантов обеспечения высокой доступности тылового сервера. К ним относятся:

- Зеркальное отображение баз данных

- Группы доступности AlwaysOn

- Экземпляры отказоустойчивых кластеров AlwaysOn (FCI)

- Отказоустойчивая кластеризация SQL

Сведения о сопряжении пулов и высокой доступности тыловых серверов см. в статье [Планирование высокой доступности и аварийного восстановления в Skype для бизнеса Server](../../plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md).

## <a name="server-collocation-in-skype-for-business-server"></a>Совместное размещение серверов в Skype для бизнеса Server

Мы уже упоминали термин "совместное размещение", но что он означает? В Skype для бизнеса Server можно разместить некоторые компоненты и роли серверов на одном сервере (совместное размещение) или на разных серверах, но поначалу в этом легко запутаться, какой бы выпуск сервера ни развертывался: Standard Edition или Enterprise Edition (в каждом из них действуют свои правила). Чтобы помочь вам с развертыванием, мы включаем совместное размещение серверов в развертывания серверов Standard Edition и интерфейсных пулов Enterprise Edition (в большинстве случаев эта информация идентична, а расхождениям уделяется особое внимание).

### <a name="collocation-of-server-roles"></a>Совместное размещение ролей серверов

Совместно с сервером Standard Edition размещена указанная ниже роль (однако требуется дополнительная настройка), а в интерфейсном пуле Enterprise Edition эту роль можно совместно разместить или развернуть на отдельном сервере:

- Посредник

Каждая из перечисленных ниже ролей сервера должна быть развернута на отдельном сервере.

- Директор

- Пограничный сервер

- Сервер видеовзаимодействия

- Office Web Apps

### <a name="databases"></a>Базы данных

Это область с действительными различиями между развертываниями сервера Standard Edition и пула серверов Enterprise Edition. Поэтому ниже предлагаются два раздела, за которыми следуют некоторые дополнительные правила для обоих случаев.

#### <a name="standard"></a>Standard

Так как SQL Server Express размещается совместно с сервером Standard Edition и не может быть перемещен, это довольно просто. Кроме того, развернув сервер сохраняемого чата на сервере Standard Edition, вы также сможете совместно разместить сохраняемый чат и базу данных соответствия для него на сервере Standard Edition (но это необязательно).

> [!NOTE]
> Сохраняемый чат доступен в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения см. в статье [Начало перехода на Microsoft Teams](/microsoftteams/upgrade-start-here). Если вам нужно использовать сохраняемый чат, то вы можете либо перенести пользователей, которым нужна эта функция, в Teams, либо продолжать использовать Skype для бизнеса Server 2015.

Указанные ниже компоненты невозможно совместно разместить на сервере Standard Edition, но они могут выполняться на отдельном сервере базы данных.

- База данных мониторинга

- База данных архивации

- Любая внутренняя база данных для интерфейсного пула Enterprise Edition

#### <a name="enterprise"></a>Enterprise

На одном тыловом сервере SQL Server можно совместно разместить указанные ниже базы данных.

- Внутренняя база данных

- База данных мониторинга

- База данных архивации

- База данных сохраняемого чата

- База данных соответствия сохраняемого чата

#### <a name="both"></a>Оба выпуска

Существуют некоторые дополнительные правила, которые следует соблюдать при совместном размещении баз данных Skype для бизнеса Server в одном экземпляре SQL или в нескольких экземплярах SQL из одной базы данных SQL Server.

- Каждый экземпляр SQL может содержать только одну внутреннюю базу данных (для интерфейсного пула Enterprise Edition), одну базу данных мониторинга, одну базу данных архивации, одну базу данных сохраняемого чата и одну базу данных соответствия сохраняемого чата.

- Сервер баз данных не может поддерживать более одного интерфейсного пула Enterprise Edition, одного сервера архивации, одного сервера мониторинга, одной базы данных сохраняемого чата и одной базы данных соответствия сохраняемого чата, но он может поддерживать по одному из указанных компонентов независимо от того, сколько экземпляров SQL Server используют базы данных.

    > [!NOTE]
    > Сохраняемый чат доступен в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения см. в статье [Начало перехода на Microsoft Teams](/microsoftteams/upgrade-start-here). Если вам нужно использовать сохраняемый чат, то вы можете либо перенести пользователей, которым нужна эта функция, в Teams, либо продолжать использовать Skype для бизнеса Server 2015.

### <a name="file-shares"></a>Общие файловые ресурсы

Общий файловый ресурс может быть отдельным сервером или может размещаться совместно на том же сервере, что следующие компоненты:

- сервер баз данных, в том числе тыловой сервер из интерфейсного пула Enterprise Edition;

- база данных мониторинга;

- база данных архивации;

- база данных сохраняемого чата;

- база данных соответствия сохраняемого чата.

> [!CAUTION]
> Обратите внимание, что хотя вы можете разместить на этих серверах общие папки, это не рекомендуется. При размещении общей папки с любой другой ролью сервера требуется регулярно отслеживать дисковое пространство и проблемы с производительностью.

### <a name="keep-in-mind"></a>Замечания

- Совместное размещение не поддерживается для обратного прокси-сервера, который не является компонентом Skype для бизнеса Server и может даже не входить в вашу топологию. Обратный прокси-сервер потребуется, если вам, среди всего прочего, нужна поддержка совместного использования веб-содержимого для федеративных пользователей. Если эта возможность необходима, продолжите процесс и внедрите поддержку обратного прокси-сервера для Skype для бизнеса Server, настроив существующий в вашей организации обратный прокси-сервер, который используется другими приложениями.

- Никакие компоненты Exchange UM или SharePoint Server невозможно совместно размещать с ролью сервера Skype для бизнеса Server.
