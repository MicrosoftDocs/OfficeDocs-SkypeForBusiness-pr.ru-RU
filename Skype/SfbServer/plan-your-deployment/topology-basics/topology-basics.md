# <a name="topology-basics-for-skype-for-business-server"></a>Основные сведения о топологии для Скайп для Business Server

Первое, что вам необходимо решить — Если вы собираетесь имеют локальное развертывание Скайп для Business Server, или если вы собираетесь сопоставляя Скайп Business Server Online развертывания в гибридном развертывании. В любом случае вы собираетесь, стоит прочесть более того, как подробно локальные топологии здесь, но сведения о гибридных приведен собственные раздела.

## <a name="sites"></a>Сайты

В Скайп Business Server можно определить сайтов локальной сети, содержащие Скайп для компонентов Business Server. Сайт — это набор компьютеров, соединенных высокоскоростной, небольшой задержкой сети, например одной локальной сети (LAN) или две сети, подключенных к сети оптический высокоскоростную оптоволоконной. Обратите внимание, что Скайп для сайтов Business Server отдельные концепции из доменных служб Active Directory — сайты и сайты Microsoft Exchange Server. Ваше Скайп для сайтов Business Server, соответствуют на сайтах Active Directory не требуется.

Скайп для Business Server поддерживает локальное развертывание одного или нескольких сайтов, которые может масштабироваться в соответствии с высокой доступности и требования к расположению.

Развертывание будет иметь по крайней мере один центральный сайт (также называемая в центре обработки данных, это центра обработки данных для всех серверов, расположенных в ней), и каждый центральный узел в вашем развертывании будут иметь один сервер Standard Edition или Enterprise Edition Front End pool. Отличия для каждого варианта указаны ниже.

- Сервер Standard Edition включает в себя выровненных баз данных SQL Server Express.

- Пул переднего плана Enterprise Edition включает в себя:

  - Один или несколько серверов переднего плана (в идеале по крайней мере три, масштабируемость), более двенадцать. Для нескольких серверов потребуется балансировка нагрузки;

  - Отдельные обратно конечных сервер.

Подробнее о различных ролях серверов можно узнать далее в этом разделе.

В дополнение к сайтам центра может также получается один или несколько узлов филиалов, связанных с центрального сайта. Они зависят от центрального сайта почти все функциональные возможности, поэтому какой являются они включает в себя, точно?

- Для обеспечения связи в филиалах, который объединяет шлюза телефонной сети (общего пользования PSTN), с помощью некоторых Скайп для функции Business Server.

- Для обеспечения связи в филиалах, это сервере под управлением Windows Server, которое имеет Скайп для регистратора Business Server и установленным программным обеспечением сервера-посредника.

- Автономный шлюз ТСОП (который не является частью устройства для обеспечения связи в филиалах).

- Автономный сервер-посредник или изолированный сервер-посредник пула (Если вы не хотите совместно с этой ролью с устройства для обеспечения связи в филиалах).

## <a name="server-roles"></a>Роли сервера

Каждый сервер под управлением Скайп для Business Server запускает один или несколько ролей сервера. Роль сервера — это определенный набор Скайп для Business Server функциональных возможностей, предоставляемых этим сервером. Нет необходимости развертывать все доступные роли сервера в сети. Установите только те роли сервера, которые предоставляют нужные функции.

В целях улучшения масштабируемости и высокой доступности для большинства ролей серверов можно развертывать пулы, состоящие из нескольких серверов, на которых запущена одна и та же роль сервера. Каждый сервер в пуле должен содержать идентичные роли сервера. Для большинства пулов в Скайп для Business Server необходимо развернуть балансировки нагрузки для распространения трафика между различными серверами в пуле. Скайп для Business Server поддерживает балансировки нагрузки доменных имен (DNS) и аппаратных балансировщиков нагрузки.

### <a name="front-end-server-and-back-end-server"></a>Сервер переднего плана и внутренний сервер

В Скайп Business Server Enterprise Edition сервер переднего плана — это роль сервера основных и запускает множество базового Скайп для функций Business Server. Сервер переднего плана, а также внутренними серверами, являются только роли сервера, должны выполняться в любой Скайп для развертывания Business Server Enterprise Edition.

Пул переднего плана — это набор серверов переднего плана, одинаково, настроенных, используемых совместно предоставляют услуги одной группе пользователей. Пул обеспечивает для пользователей возможности масштабируемости и отработки отказа.

Сервер переднего плана включает в себя следующее:

- проверка подлинности и регистрация пользователей;

- сведения о присутствии и обмен карточками контактов;

- службы адресной книги и расширение списка рассылки;

- функции обмена мгновенными сообщениями, включая многопользовательские беседы;

- веб-конференции, конференц-связь с телефонным подключением через ТСОП, аудио- и видеоконференции (если они развернуты);

- Размещение приложений как включенных в состав Скайп Business Server (например, помощник по конференц-связи и группы ответа приложение), так и приложениями сторонних производителей.

- дополнительно: мониторинг для сбора сведений об использовании в форме записей регистрации вызовов и записей ошибок вызовов. Эти сведения представлены метрики о качестве мультимедиа (звук и видео) обход сети для вызовов корпоративной голосовой связи и A / V конференций.

- веб-компоненты для поддержки веб-задач, таких как веб-планировщик и компонент обеспечения процесса присоединения.

- дополнительно: архивация как отдельных мгновенных сообщений, так и собраний для соответствия нормативным требованиям. Дополнительные сведения см [Planning for Archiving](https://technet.microsoft.com/library/898b83c1-007b-43be-9484-08fe49c10951.aspx) в документации по планированию.

    В Lync Server 2010 и более ранних версий мониторинг и архивация были разными ролями сервера, не располагается на сервере переднего плана.

- При необходимости, если сохраняемый чат включен, веб-службы сохраняемого чата для управления комнатами чата и веб-службы сохраняемого чата для передачи и загрузки файлов.

Пулы переднего плана – это также основное хранилище для данных пользователей и конференций. Информация о каждом пользователе реплицируется среди трех серверов переднего плана в пуле, а ее резервная копия создается на фоновых серверах.

Кроме того один сервер переднего плана в развертывании также работает центральный сервер управления, который управляет и развертывает базовую конфигурацию данных для всех серверов с Скайп для Business Server. Центральный сервер управления также предоставляет возможности передачи файлов и Командная консоль Lync Server.

Внутренними серверами, серверов баз данных под управлением Microsoft SQL Server, предоставляющие службы базы данных для пула переднего плана. Внутренними серверами служат в качестве резервного хранилища пользователя и конференции данных пула и являются основного хранилища для других баз данных, таких как база данных группы ответа. У вас есть один Тыловой сервер, но [Тыловой сервер высокой доступности в Скайп для Business Server](../high-availability-and-disaster-recovery/back-end-server.md) рекомендуется для отработки отказа. Обратная серверы не запускайте любой Скайп для программного обеспечения Business Server.

> [!IMPORTANT]
> Мы не рекомендуем collocating Скайп для сервера баз данных с другими базами данных. В противном случае может пострадать доступность и производительность.

> [!NOTE]
> Зеркальное отображение SQL доступна в Скайп для Business Server 2015, но в Скайп Business Server 2019 больше не поддерживается. Методы кластеризации отработки отказа группы обеспечения доступности AlwaysOn, экземпляры кластера AlwaysOn отработки отказа (FCI) и SQL, являются предпочтительными с Скайп для Business Server 2019.

Данные, хранящиеся в базах данных фонового сервера, включают сведения о присутствии, контактные списки пользователей, данные конференций, включая постоянные данные о состоянии всех текущих конференций, а также данные планирования конференций.

### <a name="edge-server"></a>Пограничный сервер

Пограничный сервер позволяет пользователям общаться и совместно работать с пользователями за пределами брандмауэров организации. Эти внешние пользователи могут включать пользователей организации, которые в настоящее время работы вне офиса, пользователи из организаций федеративного партнера и внешних пользователей, которые были приглашены участвовать в конференциях, размещенного на вашей Скайп для развертывания Business Server.

Также развертывание пограничного сервера включает службы мобильности, которая поддерживает функциональные возможности Lync на мобильных устройствах. Пользователи могут использовать поддерживаемые Apple iOS, Android (en), Windows Phone и Nokia мобильных устройств для выполнения действий, таких как отправка и получение мгновенных сообщений, Просмотр контактов и просмотр сведений о присутствии. Кроме того мобильные устройства поддерживают некоторые функции корпоративной голосовой связи, например, щелкните, чтобы присоединиться к конференции, позвонить с работой, один номер reach, голосовой почты и пропущенных звонков. Функции мобильной связи также поддерживает push-уведомления для мобильных устройств, которые не поддерживают приложений, работающих в фоновом режиме. Push-уведомлений — это уведомление, которое отправляется на мобильное устройство о событие, происходящее при неактивных мобильных приложений.

Пограничные серверы также включают полностью интегрированный прокси-сервер XMPP, при этом шлюз XMPP встроен в серверы переднего плана. Можно настроить эти компоненты XMPP для включения Скайп для пользователей Business Server добавить контакты партнеров XMPP для обмена мгновенными сообщениями и сведениями о присутствии.

> [!NOTE]
> XMPP шлюзов и прокси-серверы, доступные в Скайп для Business Server 2015, но больше не поддерживается в Скайп для Business Server 2019. Для получения дополнительных сведений в разделе [федерации XMPP миграции](../../../SfBServer2019/migration/migrating-xmpp-federation.md) .

### <a name="mediation-server"></a>Сервер-посредник

Сервер-посредник является компонентом необходимых для реализации корпоративной голосовой связи, звонок с помощью рабочих и конференц-связи. Сервер-посредник преобразует сигналы и некоторые конфигурации мультимедиа между вашей внутренней Скайп для инфраструктуры Business Server и общего пользования шлюза телефонной сети (общего пользования PSTN), IP-УАТС или магистрали Session Initiation Protocol (SIP). Можно запустить сервер-посредник с совместным размещением на том же сервере, что и сервер переднего плана, или в отдельном пуле сервера-посредника.

Дополнительные сведения см [компонент сервера-посредника в Скайп для Business Server](../../plan-your-deployment/enterprise-voice-solution/mediation-server.md).

### <a name="video-interop-server"></a>сервер видеовзаимодействия

Видео взаимодействия сервер — это создать роль по состоянию на Скайп для Business Server 2015. Она позволяет интегрировать вашей Скайп для развертывания Business Server с определенных решений сторонних производителей VTC (система телеконференций видео). VIS выступает в качестве посредника между сторонних производителей телеконференцию и Скайп для развертывания Business Server. В этом выпуске VIS ориентирован на обеспечение взаимодействия с видеосистемами Cisco/Tandberg.

Дополнительные сведения см [Планирование Server взаимодействия видео в Скайп для Business Server](../../plan-your-deployment/video-interop-server.md).

### <a name="director"></a>Директор

Директоров может выполнять проверку подлинности Скайп для запросов пользователей Business Server, но они не номер домашнего учетные записи и предоставления сведений о присутствии или служб конференц-связи. Директоры оказываются особенно полезными при необходимости повышения уровня безопасности в развертываниях, где необходим доступ внешних пользователей. Директор может выполнять проверку подлинности запросов, прежде чем передавать эти запросы внутренним серверам. В случае атак типа "отказ в обслуживании" они заканчиваются на директоре и не достигают серверов переднего плана.

### <a name="persistent-chat-server-roles"></a>Роли серверов сохраняемого чата

> [!NOTE]
> Сохраняемый чат доступна в Скайп для Business Server 2015, но больше не поддерживается в Скайп для Business Server 2019. Те же функциональные возможности доступны в группах. Для получения дополнительных сведений см [Реализация из Скайп для бизнеса для групп Майкрософт](/microsoftteams/journey-skypeforbusiness-teams). Если необходимо использовать сохраняемого чата, возможны либо перенос пользователей, которым требуется эта функция групп, или для дальнейшего использования Скайп для Business Server 2015.

Возможности сохраняемого чата позволяют пользователям участвовать в многопользовательских тематических беседах, которые подлежат сохранению. На сервере переднего плана сохраняемого чата работает служба сохраняемого чата. Внутренний сервер сохраняемого чата хранит данные журнала бесед и сведения о категориях и комнатах чата. Дополнительный внутренний сервер сохраняемого чата для соответствия требованиям может хранить содержимое чата и события соответствия, которые будут использоваться для обеспечения соответствия требованиям.

Серверы Скайп for Business Server Standard Edition можно также выполнить Persistent chat с совместным размещением на том же сервере. Нельзя совместно размещать сохраняемый чат сервер переднего плана с сервером переднего плана Enterprise Edition.

Дополнительные сведения см [Планирование сервера сохраняемого чата в Скайп для Business Server 2015](../../plan-your-deployment/persistent-chat-server/persistent-chat-server.md).

## <a name="high-availability-and-disaster-recovery-support"></a>Поддержка высокой доступности и аварийного восстановления

Скайп для Business Server, достигается высокая доступность, избыточности серверов с помощью пула. Если сервер, на котором работает определенная роль, завершает работу со сбоем, другие серверы в пуле, выполняющие ту же самую роль, берут на себя нагрузку первого сервера. Это применимо к интерфейсным серверам, пограничным серверам, серверам-посредникам и директорам.

Скайп для Business Server также предоставляет аварийного восстановления меры, включив сопряжения пула. Если развернуть эту топологию, будут назначены пары интерфейсных пулов, при этом каждый пул из этой пары размещается в отдельном центре обработке данных и в отдельном регионе. Если один пул или сайт завершает работу, можно перенаправить пользователей этого пула в другой пул пары, что приведет к минимальному перерыву в обслуживании.

Скайп для Business Server также поддерживает ряд параметров для обеспечения высокой доступности Тыловой сервер. Среди них:

- Зеркальное отображение баз данных

- Группы доступности AlwaysOn

- Экземпляры отказоустойчивого кластера AlwaysOn (FCI)

- Отказоустойчивая кластеризация SQL

Для получения дополнительных сведений о сопряжения пула и высокой доступности Тыловой сервер видеть [Планирование высокой доступности и аварийного восстановления в Скайп для Business Server](../../plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md).

## <a name="server-collocation-in-skype-for-business-server"></a>Выровненное размещение серверов в Скайп для Business Server

Мы использовали уже совместно термин, но что это означает? Скайп для Business Server позволяет находить некоторые роли и компоненты сервера на том же сервере, который является выровненное размещение, или на разных серверах, но может быть некоторая путаница перед началом работы в работе и ли вы выполняете сервера Standard Edition или Enterprise Edition развертывание (каждый из них поступают с помощью собственных правил). Чтобы помочь в планировании, мы в случае включая выровненное размещение серверов в развертывании сервера Standard Edition и развертываний пула переднего плана Enterprise Edition (во многих случаях эти сведения не совпадают, и там, где оно отличается, он является специально).

### <a name="collocation-of-server-roles"></a>Совместное размещение ролей серверов

Сервер Standard Edition имеет следующие роли с совместным размещением (дополнительная настройка не требуется не менее), находясь в пуле переднего плана Enterprise Edition с этой ролью могут быть с совместным размещением или развернут на отдельный сервер:

- Посредник

Каждая из перечисленных ниже ролей сервера должна быть развернута на отдельном сервере.

- Директор

- Пограничный

- сервер видеовзаимодействия

- Веб-приложения Office Web App

### <a name="databases"></a>Базы данных

Это область с реально различия между развертывания серверов Standard Edition и развертывания пула Enterprise Edition server, поэтому мы должны двух следующих разделах, а затем некоторые дополнительные правила для обоих.

#### <a name="standard"></a>Standard

Поскольку SQL Server Express совмещен сервер Standard Edition, а не могут быть перемещены, это довольно просто. Кроме того Если выполняется развертывание сервера сохраняемого чата на сервере Standard Edition, вы также можете слишком совместно размещать сохраняемый чат и Persistent Chat база данных соответствия на сервере Standard Edition, но не нужно.

    > [!NOTE]
    > Persistent chat is available in Skype for Business Server 2015 but is no longer supported in Skype for Business Server 2019. The same functionality is available in Teams. For more information, see [Journey from Skype for Business to Microsoft Teams](/microsoftteams/journey-skypeforbusiness-teams). If you need to use Persistent chat, your choices are to either migrate users requiring this functionality to Teams, or to continue using Skype for Business Server 2015.

Они не могут размещаться совместно на сервере Standard Edition, но можно перейдите на одном сервере от своего имени:

- данных мониторинга

- данных архивации

- Любой серверной базы данных для пула переднего плана Enterprise Edition

#### <a name="enterprise"></a>Enterprise

Следующие базы данных могут размещаться совместно на одном серверном компоненте SQL Server:

- Внутренняя база данных

- данных мониторинга

- данных архивации

- База данных persistent Chat

- База данных соответствия persistent Chat

#### <a name="both"></a>Оба

Теперь существуют некоторые дополнительные правила следует придерживаться при collocating Скайп для сервера баз данных в одном экземпляре SQL или в нескольких экземпляров SQL в той же базе данных SQL Server:

- Каждый экземпляр SQL может содержать только один серверной базы данных для пула переднего плана Enterprise Edition, одну базу данных мониторинга, одну базу данных архивации, одну базу данных Persistent Chat и одного Persistent Chat базы данных соответствия.

- Сервер базы данных не поддерживает несколько пулов переднего плана Enterprise Edition, один сервер с запущенным архивации, один сервер мониторинга, одну базу данных Persistent Chat и одной сохраняемого чата соответствия базы данных, но он может поддерживать один из них, вне зависимости от ли баз данных использовать тот же экземпляр SQL Server или отдельные экземпляры SQL Server.

    > [!NOTE]
    > Сохраняемый чат доступна в Скайп для Business Server 2015, но больше не поддерживается в Скайп для Business Server 2019. Те же функциональные возможности доступны в группах. Для получения дополнительных сведений см [Реализация из Скайп для бизнеса для групп Майкрософт](/microsoftteams/journey-skypeforbusiness-teams). Если необходимо использовать сохраняемого чата, возможны либо перенос пользователей, которым требуется эта функция групп, или для дальнейшего использования Скайп для Business Server 2015.

### <a name="file-shares"></a>Общие файловые ресурсы

Общий файловый ресурс может быть отдельным сервером или может размещаться совместно на том же сервере, что следующие компоненты:

- Сервер базы данных, в том числе фоновый сервер пула переднего плана Enterprise Edition

- данных мониторинга

- данных архивации

- База данных persistent Chat

- База данных соответствия persistent Chat

> [!CAUTION]
> Обратите внимание, что вы можете разместить файловый ресурс на этих серверах, но это не рекомендуется. При размещении файлового ресурса с любой другой ролью сервера требуется регулярно отслеживать дисковое пространство и проблемы с производительностью.

### <a name="keep-in-mind"></a>Помните

- Нельзя совместно размещать обратного прокси-сервера, который не будет Скайп для компонента Business Server и может даже не используемых в топологии. Если вы хотите обеспечить поддержку совместно работать с веб-содержимым для федеративных пользователей среди многих других параметров вам понадобится обратного прокси-сервера. Если необходимо, пойти дальше и реализации поддержки обратного прокси-сервера для Скайп Business Server путем настройки существующего обратного прокси-сервера, который уже присутствует в вашей организации, который используется с другими приложениями.

- Нельзя совместно размещать любой Exchange единой системы обмена СООБЩЕНИЯМИ SharePoint Server компонент или с любого Скайп для роли сервера.
