# <a name="topology-basics-for-skype-for-business-server"></a>Основы топологии для Skype для бизнеса Server

**Сводка:** Выберите топологию для Skype для бизнеса Server. Узнайте о выполнимом выполнимом сервере для Skype для бизнеса Server.

Перед подготовкой других данных необходимо знать, что планируется использовать подготовимую топологию для развертывания Skype для бизнеса Server. В первую очередь необходимо решить, будет ли развертываться Skype для бизнеса Server локально или вы собираетесь объединить его с развертыванием Skype для бизнеса Server Online в гибридном развертывании. В любом случае вы захотите прочитать дополнительные сведения, так как мы подробно разберем здесь о локальной topologies, но сведения о гибридной модели задокументированы в собственном разделе.

## <a name="sites"></a>Сайты

В Skype для бизнеса Server определяются сайты в сети, содержащие компоненты Skype для бизнеса Server. Сайт — это набор компьютеров, соединенных с помощью сети с высокой пропускной способностью и низкой задержкой, например локальной сети, или две сети, соединенные с помощью высокоскоростного оптоволоконного канала. Обратите внимание, что сайты Skype для бизнеса Server — это отдельная концепция от сайтов доменных служб Active Directory и Microsoft Exchange Server сайтов. Сайты Skype для бизнеса Server не должны соответствовать сайтам Active Directory.

Skype для бизнеса Server поддерживает локальное развертывание одного или более сайтов, которые можно масштабировать в соответствии с высокими требованиями к доступности и расположению.

В развертывании будет по крайней мере один центральный сайт (также называемый центром обработки данных, это центр обработки данных для всех серверов, расположенных в нем), а каждый центральный сайт в развертывании будет иметь один сервер Standard Edition или по крайней мере один пул переднего сервера Enterprise Edition. Отличия в каждом из вариантов приведены ниже.

- Сервер Standard Edition включает в себя базу данных Express с SQL Server с SQL Server.

- Пул переднего входа Enterprise Edition включает в себя:

  - Один или несколько серверов переднего сервера (в идеале по крайней мере три сервера для масштабируемости) и не более 12 серверов. Балансировка нагрузки потребуется для более чем одного сервера.

  - Отдельный тыл.сервер.

Подробнее о различных ролях сервера можно узнать немного позже в этом разделе.

Помимо центральных сайтов, с центральным сайтом также может быть связан один или несколько сайтов филиалов. Почти все функции зависят от центрального сайта, поэтому из чего они в точности состоит?

- Устройство для связи в филиалах, объединяющее шлюз телефонной сети общего звонков (PSTN) с некоторыми функциями Skype для бизнеса Server.

- Survivable Branch Server это сервер под управлением Windows Server, на который установлен регистратор Skype для бизнеса Server и программное обеспечение сервера-посредника.

- Автономный шлюз STN (который не входит в состав устройства для связи в филиалах).

- Автономный сервер-посредник или автономный пул серверов-посредников (если вы не хотите выполнять эту роль с устройством для связи в филиалах).

## <a name="server-roles"></a>Роли серверов

На каждом сервере со Skype для бизнеса Server выполняется одна или несколько ролей сервера. Роль сервера — это определенный набор функций Skype для бизнеса Server, предоставляемых этим сервером. Вам не нужно развертывать все доступные роли сервера в сети. Установите только те роли сервера, которые предоставляют нужные функции.

В целях улучшения масштабируемости и высокой доступности для большинства ролей серверов можно развертывать пулы, состоящие из нескольких серверов, на которых запущена одна и та же роль сервера. Каждый сервер в пуле должен содержать идентичные роли сервера. Для большинства типов пулов в Skype для бизнеса Server необходимо развернуть балансировку нагрузки для распространения трафика между различными серверами в пуле. Skype для бизнеса Server поддерживает как балансировку нагрузки службы доменных имен (DNS), так и аппаратные балансировщиков нагрузки.

### <a name="front-end-server-and-back-end-server"></a>Сервер переднего плана и внутренний сервер

В Skype для бизнеса Server Enterprise Edition сервер переднего плана является основной ролью сервера и выполняет множество базовых функций Skype для бизнеса Server. Сервер переднего плана и серверы переднего плана — это единственные роли сервера, необходимые для развертывания Skype для бизнеса Server Enterprise Edition.

Пул переднего плана — это набор настроенных идентичным образом серверов переднего плана, которые работают вместе для предоставления служб общей группе пользователей. Пул обеспечивает для пользователей возможности масштабируемости и отработки отказа.

Сервер переднего плана предоставляет следующие возможности:

- Проверка подлинности и регистрация пользователей.

- Сведения о присутствии и обмен карточками контактов.

- Службы адресной книги и расширение списка рассылки.

- Функции im, включая многостолбные конференции с мгновенными данными.

- Веб-conferencing, PSTN Dial-in conferencing and A/V conferencing (if deployed).

- Размещение приложений как для приложений, включенных в Skype для бизнеса Server (например, помощника по конференцию и приложения группы ответа), так и для сторонних приложений.

- дополнительно: мониторинг для сбора сведений об использовании в форме записей регистрации вызовов и записей ошибок вызовов. Эти сведения предоставляют метрики о качестве мультимедиа (аудио и видео), которое проходит по сети как для Корпоративная голосовая связь вызовов, так и для аудио- и видеоконференции.

- веб-компоненты для поддержки веб-задач, таких как веб-планировщик и компонент обеспечения процесса присоединения.

- дополнительно: архивация как отдельных мгновенных сообщений, так и собраний для соответствия нормативным требованиям. Более подробные сведения см. в разделе [Planning for Archiving](https://technet.microsoft.com/library/898b83c1-007b-43be-9484-08fe49c10951.aspx) документации по развертыванию.

    В Lync Server 2010 и предыдущих версиях мониторинг и архивы были отдельными ролями сервера, а не были размещены на сервере переднего плана.

- При необходимости, если сохраняемый чат включен, веб-службы сохраняемого чата для управления комнатами чата и веб-службы сохраняемого чата для передачи и загрузки файлов.

Пулы переднего плана — это также основное хранилище для данных пользователей и конференций. Информация о каждом пользователе реплицируется среди трех серверов переднего плана в пуле, а ее резервная копия создается на фоновых серверах.

Кроме того, один сервер переднего сервера в развертывании также запускает центральный сервер управления, который управляет и развертывает базовые данные конфигурации на всех серверах Skype для бизнеса Server. Центральный сервер управления также предоставляет возможности для управления Lync Server и передачи файлов.

Тыловые серверы — это серверы баз данных с Microsoft SQL Server, которые предоставляют службы баз данных для пула переднего сервера. Тыловые серверы служат резервными хранилищами для данных пользователей и конференций пула, а также являются основными хранилищами для других баз данных, таких как база данных группы ответа. У вас может быть один сервер, но высокая доступность тыл-сервера в Skype для бизнеса [Server](../high-availability-and-disaster-recovery/back-end-server.md) рекомендуется для отоверки. На тыловом сервере не работает программное обеспечение Skype для бизнеса Server.

> [!IMPORTANT]
> Не рекомендуется использовать базы данных Skype для бизнеса Server вместе с другими базами данных. В противном случае может пострадать доступность и производительность.

> [!NOTE]
> SQL зеркальное отражение доступно в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. В Skype для бизнеса Server 2019 предпочтительнее использовать группы доступности AlwaysOn, экземпляры кластера для отстройки AlwaysOn и SQL кластеров.

Данные, хранящиеся в базах данных фонового сервера, включают сведения о присутствии, контактные списки пользователей, данные конференций, включая постоянные данные о состоянии всех текущих конференций, а также данные планирования конференций.

### <a name="edge-server"></a>Пограничный сервер

Edge Server позволяет пользователям взаимодействовать и совместно работать с пользователями за пределами брандмауэров организации. Эти внешние пользователи могут включать собственных пользователей организации, которые в настоящее время работают за пределами организации, пользователей из федераированных партнерских организаций и внешних пользователей, приглашенных присоединиться к конференциям, которые размещены в развертывании Skype для бизнеса Server.

Развертывание edge Server также позволяет использовать мобильные службы, которые поддерживают функции Lync на мобильных устройствах. Пользователи могут использовать поддерживаемые мобильные устройства Apple iOS, Android, Windows Phone или Nokia для выполнения таких действий, как отправка и получение мгновенных сообщений, просмотр контактов и просмотр присутствия. Кроме того, мобильными устройствами поддерживаются некоторые возможности корпоративной голосовой связи (Enterprise Voice), такие как присоединение к конференции по щелчку, звонок с работы (Call via Work), многосторонняя связь по одному номеру, голосовая почта и пропущенные вызовы. Функция мобильности также поддерживает push-уведомления для мобильных устройств, которые не поддерживают приложения, работающие в фоновом режиме. Push-уведомление — это уведомление, которое отправляется на мобильное устройство о событии, которое происходит, когда мобильное приложение неактивно.

Пограничные серверы также включают полностью интегрированный прокси-сервер XMPP, при этом шлюз XMPP встроен в серверы переднего плана. Вы можете настроить эти компоненты XMPP, чтобы пользователи Skype для бизнеса Server могли добавлять контакты от партнеров на основе XMPP для обмена мгновенными сообщениями и присутствия.

> [!NOTE]
> Шлюзы и прокси-серверы XMPP доступны в Skype для бизнеса Server 2015, но больше не поддерживаются в Skype для бизнеса Server 2019. Дополнительные сведения см. в переносе [федерации XMPP.](../../../SfBServer2019/migration/migrating-xmpp-federation.md)

### <a name="mediation-server"></a>Сервер-посредник

Сервер-посредник является необходимым компонентом для реализации Корпоративная голосовая связь, телефонной работы и телефонной службы. Сервер-посредник преобразует сигналы и, в некоторых конфигурациях, мультимедиа между внутренней инфраструктурой Skype для бизнеса Server и шлюзом телефонной сети общего присоединения (STN), IP-УАПС или магистралью SIP. Сервер-посредник может работать на одном компьютере с сервером переднего плана или может быть выделен в отдельный пул серверов-посредников.

Подробные сведения см. в [компоненте сервера-посредника в Skype для бизнеса Server.](../../plan-your-deployment/enterprise-voice-solution/mediation-server.md)

### <a name="video-interop-server"></a>Сервер видеовзаимодействия

Сервер видеосвязи — это новая роль skype для бизнеса Server 2015. Это позволяет интегрировать развертывание Skype для бизнеса Server с определенными сторонними решениями VTC (видеоконференции). ViS действует как посредник между стороной системой телеконференции и развертыванием Skype для бизнеса Server. В этом выпуске VIS ориентирован на возможности обеспечения связи с видеосистемами Cisco/Tandberg.

For details, see [Plan for Video Interop Server in Skype for Business Server.](../../plan-your-deployment/video-interop-server.md)

### <a name="director"></a>Директор

Директоры могут проверить подлинность запросов пользователей Skype для бизнеса Server, но они не предоставляют учетные записи пользователей и не предоставляют службы присутствия или услуг. Директоры оказываются особенно полезными при необходимости повышения уровня безопасности в развертываниях, где необходим доступ внешних пользователей. Директор может выполнять проверку подлинности запросов, прежде чем передавать эти запросы на внутренние серверы. При начале атак по принципу отказа в обслуживании атаки заканчиваются на директоре и не достигают серверов переднего плана.

### <a name="persistent-chat-server-roles"></a>Роли сервера сохраняемой беседы

Дополнительные сведения [см. в сведениях о том,](/microsoftteams/upgrade-start-here)как начать обновление Microsoft Teams. Если необходимо использовать сохраняемого чата, вы можете либо перенести пользователей, которым требуются эти функции, в Teams, либо продолжить использование Skype для бизнеса Server 2015.

Возможности сохраняемого чата позволяют пользователям участвовать в многопользовательских тематических беседах, которые подлежат сохранению. На сервере переднего плана сохраняемого чата работает служба сохраняемого чата. Внутренний сервер сохраняемого чата хранит данные журнала бесед и сведения о категориях и комнатах чата. Дополнительный внутренний сервер сохраняемого чата для соответствия требованиям может хранить содержимое чата и события соответствия, которые будут использоваться для обеспечения соответствия требованиям.

На серверах Под управлением Skype для бизнеса Server Standard Edition также можно запускать сохраняемого чата, размещенного на одном сервере. Сервер переднего сервера сохраняемой беседы нельзя выполнить с сервером переднего сервера Enterprise Edition.

For details, see [Plan for Persistent Chat Server in Skype for Business Server 2015](../../plan-your-deployment/persistent-chat-server/persistent-chat-server.md).

## <a name="high-availability-and-disaster-recovery-support"></a>Поддержка высокой доступности и аварийного восстановления

Skype для бизнеса Server обеспечивает высокую доступность за счет избыточности серверов с помощью пулов. При сбое сервера с определенными ролями остальные серверы в пуле с такими же ролями берут на себя нагрузку этого сервера. Это касается серверов переднего плана, пограничных серверов, серверов-посредников и Директоров.

Skype для бизнеса Server также предоставляет меры аварийного восстановления, включив сопряжение пулов. Если развернуть эту топологию, будут назначены пары интерфейсных пулов, при этом каждый пул из этой пары размещается в отдельном центре обработке данных и в отдельном регионе. Если один пул или сайт завершает работу, можно перенаправить пользователей этого пула в другой пул пары, что приведет к минимальному перерыву в обслуживании.

Skype для бизнеса Server также поддерживает несколько вариантов обеспечения высокой доступности тылов. К ним относятся:

- Зеркальное отображение базы данных

- группы доступности AlwaysOn;

- Экземпляры кластера для отокрутки AlwaysOn (FCI)

- SQL отбойной кластерии

For details about pool pairing and Back End Server high availability, see [Plan for high availability and disaster recovery in Skype for Business Server.](../../plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md)

## <a name="server-collocation-in-skype-for-business-server"></a>Выполнимые серверы в Skype для бизнеса Server

Мы уже использовали термин "collocate", но что это значит? Skype для бизнеса Server позволяет находить некоторые роли и функции сервера на одном сервере , который является выстоянием, или на разных серверах, но это может запутать при запуске и развертывании сервера Standard Edition или Enterprise Edition (каждый из них содержит собственные правила). Для помощи в планировании мы включаем выполнимое размещение серверов в развертываниях серверов Standard Edition и развертывание пула переднего плана Enterprise Edition (в большинстве случаев эти сведения идентичны и где они отличаются, они называются в частности).

### <a name="collocation-of-server-roles"></a>Выполнимые роли сервера

Сервер Standard Edition имеет следующую роль с размещением (однако требуется дополнительная настройка), а в пуле переднего плана Enterprise Edition эту роль можно совмеделить или развернуть на отдельном сервере:

- Посредник

Каждая из этих ролей сервера должна быть развернута на отдельном сервере:

- Директор

- Microsoft Edge

- Сервер видеовзаимодействия

- Office Web Apps

### <a name="databases"></a>Databases

Это область с реальными различиями между развертываниями серверов Standard Edition и пулом серверов Enterprise Edition, поэтому мы будем иметь два раздела ниже, за которыми следуют некоторые дополнительные правила для обоих вариантов.

#### <a name="standard"></a>Стандартный

Так SQL Server Express размещена на сервере Standard Edition и не может быть перемещена, это довольно просто. Кроме того, при развертывании сервера сохраняемой беседы на сервере Standard Edition вы также можете разместить сохраняемую беседу и базу данных соответствия сохраняемой беседы на сервере Standard Edition, но это не нужно.

> [!NOTE]
> Сохраняемая беседа доступна в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения [см. в сведениях о том,](/microsoftteams/upgrade-start-here)как начать обновление Microsoft Teams. Если необходимо использовать сохраняемого чата, вы можете либо перенести пользователей, которым требуются эти функции, в Teams, либо продолжить использование Skype для бизнеса Server 2015.

Они не могут быть размещены на сервере Standard Edition, но могут быть размещены на отдельном сервере баз данных:

- база данных мониторинга;

- база данных архивации;

- Любая тыловая база данных для пула переднего входа Enterprise Edition

#### <a name="enterprise"></a>Предприятие

Следующие базы данных могут быть размещены на одном и том же тыловом SQL Server:

- Тыловая база данных

- база данных мониторинга;

- база данных архивации;

- База данных сохраняемой беседы

- База данных соответствия сохраняемой беседы

#### <a name="both"></a>Оба

Теперь существуют некоторые дополнительные правила, которые необходимо соблюдать при выполномодеринге баз данных Skype для бизнеса Server в одном экземпляре SQL или в нескольких экземплярах SQL в одной SQL Server базе данных:

- Каждый SQL может содержать только одну тыловую базу данных для пула переднего конца Enterprise Edition, одну базу данных мониторинга, одну базу данных архива, одну базу данных сохраняемой беседы и одну базу данных соответствия Persistent Chat.

- Сервер базы данных не может поддерживать более одного пула переднего сервера Enterprise Edition, один сервер с архиватива, один сервер мониторинга, одну базу данных сохраняемого чата и одну базу данных соответствия Persistent Chat, но он может поддерживать одну из них независимо от того, используют ли базы данных один экземпляр SQL Server или отдельные экземпляры SQL Server.

    > [!NOTE]
    > Сохраняемая беседа доступна в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения [см. в сведениях о том,](/microsoftteams/upgrade-start-here)как начать обновление Microsoft Teams. Если необходимо использовать сохраняемого чата, вы можете либо перенести пользователей, которым требуются эти функции, в Teams, либо продолжить использование Skype для бизнеса Server 2015.

### <a name="file-shares"></a>Общие файловые ресурсы

Файловая папка может быть на отдельном сервере или вы можете совместно ее использовать на одном сервере или на любом из следующих серверов:

- сервер баз данных, в том числе фоновый сервер из интерфейсного пула Enterprise Edition;

- база данных мониторинга;

- база данных архивации;

- База данных сохраняемой беседы

- База данных соответствия сохраняемой беседы

> [!CAUTION]
> Обратите внимание, что хотя вы можете совместно использовать обойму файловых файлов на этих серверах, важно отметить, что это не рекомендуется. При совместном месте файлового пространства с любой другой ролью сервера необходимо регулярно следить за проблемой дискового пространства и производительности.

### <a name="keep-in-mind"></a>Помните

- Вы не можете выполнив Обратный прокси-сервер потребуется, помимо всего прочего, для поддержки общего доступа к веб-содержимому для федераированных пользователей. Если необходимо, реализуйте поддержку обратного прокси-сервера для Skype для бизнеса Server, настроив существующий обратный прокси-сервер, который уже используется в вашей организации, который используется другими приложениями.

- Вы не можете совместно использовать любой компонент exchange UM или компонент SharePoint Server с любой ролью Skype для бизнеса Server.
