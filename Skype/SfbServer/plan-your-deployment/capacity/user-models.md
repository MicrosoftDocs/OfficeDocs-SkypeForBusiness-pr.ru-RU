# <a name="user-models-in-skype-for-business-server"></a>Пользовательские модели в Skype для бизнеса Server
 
Описанные здесь пользовательские модели обеспечивают основу для показателей планирования емкости и рекомендаций, описанных в модели планирования емкости для [Skype для бизнеса Server.](user-model.md)
  
## <a name="skype-for-business-server-user-models"></a>Пользовательские модели Skype для бизнеса Server

В следующей таблице описана пользовательская модель регистрации, контактов, обмена мгновенными сообщениями и присутствия в Skype для бизнеса Server.
  
**Пользовательская модель среды и регистрации**

|**Категория**|**Описание**|
|:-----|:-----|
|Размер и распределение развертывания  <br/> |Мы моделируем большое развертывание с тремя центральными сайтами и с одним интерфейсным пулом на каждом сайте.  <br/> |
|Процент пользователей Active Directory  <br/> |Мы предполагаем, что 70% всех пользователей Active Directory в организации включены для Skype для бизнеса Server. 80 % включенных пользователей ежедневно вошел в Skype для бизнеса Server (80 % совмества). На основе количества параллельных пользователей рассчитываются числа в оставшейся части этого раздела.  <br/> |
|Изменения Active Directory  <br/> |Мы предполагаем, что 0,5% от общего числа пользователей каждую неделю создаются и активна для Skype для бизнеса в Active Directory, а 0,5 % пользователей отключаются из Active Directory и Skype для бизнеса каждую неделю. Для 5% пользователей каждую неделю изменяется по крайней мере один атрибут Active Directory.  <br/> |
|Группы рассылки Active Directory  <br/> |Мы предполагаем, что число групп рассылки Active Directory в организации втрое больше количества всех пользователей в Active Directory. Эти группы рассылки имеют следующие размеры:  <br/> • 64 % имеют от 2 до 30 пользователей  <br/> • 13 % имеют от 31 до 50 пользователей  <br/> • 10 % имеют от 51 до 100 пользователей  <br/> • 13 % имеют от 101 до 500 пользователей  <br/> |
|Пользователи протокола VoIP  <br/> |60 % пользователей Skype для бизнеса Server могут использовать единую связь (UC), то есть их номера телефонов принадлежат Skype для бизнеса Server.  <br/> |
|Распределение зарегистрированных клиентов  <br/> |65 % клиентов работают с программным обеспечением Skype для бизнеса, включая Skype для бизнеса и Lync Phone Edition.  <br/> 30 % клиентов, работающих под управлением клиентского программного обеспечения из предыдущей версии Lync.  <br/> 5 % клиентов, использующих Skype для бизнеса Web App.  <br/> Если мобильность включена, мы предполагаем, что 40% пользователей используют мобильность одновременно с другими ранее зарегистрированными вариантами клиентов. В этом случае соотношение количества точек присутствия клиента (MPOP) составляет 1:1,9. Если мобильность отключена, то коэффициент MPOP составляет 1:1,5.  <br/> |
|Распределение удаленных пользователей  <br/> |70% пользователей подключается внутренним образом.  <br/> 30 % пользователей, подключающихся через edge Server (вы также можете иметь здесь директора, но это необязательно).  <br/> |
|Распределение контактов  <br/> |Пользователь имеет не более 1000 контактов. Менее 1% пользователей имеет 1000 контактов. Менее 25% пользователей имеет как минимум 100 контактов.  <br/> Пользователи с подключением к общедоступному облаку в среднем имеют по 80 контактов. Для этих пользователей:  <br/> • 50 % контактов находятся внутри организации. 10 % этих пользователей являются удаленными пользователями, подключаясь за пределами брандмауэра.  <br/> • 40 % контактов являются пользователями Skype.  <br/> • 10 % контактов являются федератными партнерами.  <br/> Пользователи без возможности подключения к общедоступному облаку в среднем имеют по 50 контактов. Для этих пользователей:  <br/> • 80 % контактов находятся внутри организации. 10 % этих пользователей являются удаленными пользователями, подключаясь за пределами брандмауэра.  <br/> • 20 % контактов являются федератными партнерами.  <br/> Каждый пользователь имеет одну группу рассылки в своем списке контактов. Для тестирования производительности мы предполагаем, что группы рассылки всегда расширенные.  <br/> |
|Время сеанса  <br/> |Среднее время сеанса входа пользователя в систему составляет 12 часов. Все пользователи выполняют вход в течение 120 минут после начала сеанса.  <br/> |
   
**Пользовательская модель обмена мгновенными сообщениями и присутствия**

|**Категория**|**Описание**|
|:-----|:-----|
|Одноранговые сеансы обмена мгновенными сообщениями  <br/> |Каждый пользователь в среднем имеет по шесть одноранговых сеансов обмена мгновенными сообщениями в день.  <br/> 10 мгновенных сообщений на сеанс.  <br/> Каждому сообщению соответствуют два сообщения SIP INFO и 2 сообщения SIP 200 OK (для индикаторов состояния, таких как \<Name\> "ввод текста")  <br/> |
|Групповые сеансы im  <br/> |Среднее количество сообщений, отправленных в сеансе только для мгновенных сообщений группы, составляет 5 на пользователя.  <br/> Среднее количество сообщений, отправленных в im-части конференции av, составляет 2 на пользователя.  <br/> |
|Опрос присутствия  <br/> |В основном для опроса присутствия мы предполагаем по 60 опросов на каждого пользователя в час. Для каждого пользователя предполагаются следующие средние значения.  <br/> • Один опрос в день присутствия пользователей на вкладке "Организация" пользователя (но не в списке контактов). Среднее количество пользователей, не относяхся к контактам, на вкладке "Организация" пользователя составляет 15 пользователей. В день выполняется две операции просмотра карточки контакта.  <br/> • Один опрос присутствия каждый раз, когда пользователь щелкает другого пользователя, чтобы начать беседу, примерно раз в час.  <br/> • Шесть поисковых запросов пользователей в час. При каждом выполнении поиска пакетный опрос отправляется каждому в списке результатов поиска. Мы предполагаем, что средний размер результатов поиска — 20. Если результаты поиска остаются на экране, то пакетный опрос обновляется каждые 5 минут; мы предполагаем, что будет два таких обновления в час.  <br/> • Когда пользователь открывает или просматривает сообщение электронной почты в Outlook, опрос присутствия пользователей в полях "To" и "CC: " электронной почты" (по пять сообщений в час и четыре пользователя на сообщение).  <br/> |
|Подписки на сведения о присутствии  <br/> |Когда пользователь добавляет другого пользователя в качестве контакта, он подписывается на пять категорий сведений об этом пользователе. Обновления этих категорий сведений автоматически отправляются первому пользователю. <br/> Для каждого клиента отправляется один пакетный запрос подписки, чтобы получить сведения о присутствии в среднем 40 контактов, с дополнительными 40 диалогами, чтобы получить сведения о присутствии для федеративных контактов.  <br/> Сведения о присутствии для членов расширенной группы рассылки извлекаются с помощью постоянных подписок на сведения о присутствии, а не опросов, и моделируются как 1 расширение на пользователя каждые 2 часа.  <br/> Короткие подписки происходят, когда пользователь входит в систему, для всех контактов пользователя есть пакетная подписка, после чего пользователь вскоре выключается. Мы предполагаем 6 кратких подписок на пользователя в час, при этом каждая подписка продолжается 10 минут. <br/> |
|Публикация сведений о присутствии  <br/> |Состояние присутствия публикуется в среднем 4 раза на пользователя в час, максимально 6 раз на пользователя в час.  <br/> |
|Размер документа сведений о присутствии  <br/> |Средний размер полного документа сведений о присутствии оценивается в 4 КБ, максимальный размер — 25 КБ.  <br/> |

В обычных пулах переднего сервера в Skype для бизнеса Server поддерживается не более 250 пользователей. В каждом пуле в каждый момент времени может быть размещено одно собрание в 250 пользователей. Во время проведения такого большого собрания в пуле также могут размещаться другие небольшие конференции. Кроме того, можно поддерживать собрания до 1000 пользователей, настраивая выделенный пул для размещения таких собраний. For details, see [Plan for large meetings in Skype for Business Server.](../../plan-your-deployment/conferencing/large-meetings.md)

- 5% конференций имели одиннадцать участников;
    
- одна большая конференция с 250 участниками.

## <a name="busy-hour"></a>Период максимальной нагрузки

Для одноранговых сеансов пиковая нагрузка вычисляется с помощью попыток вызовов в час наибольшей нагрузки (BHCA). Этот термин из сферы голосовой связи предполагает, что 50% всех вызовов в день будут выполняться в 20% времени. Это значение вычисляется по следующей формуле:
  
 `BHCA=(total calls * 0.5) / 1.6`
  
При тестировании производительности период максимальной нагрузки моделируется путем запуска сеансов VoIP и других одноранговых сеансов с уровнем максимальной нагрузки по крайней мере 1,6 часов в день.
  
Пиковая нагрузка конференц-связи предполагает, что 75% всех конференций для восьмичасового дня происходит в 4 часа максимальной нагрузки. В эти часы нагрузка в полтора раза выше средней нагрузки конференц-связи.
  
## <a name="enterprise-voice-to-pstn-calls"></a>Корпоративная голосовая связь вызовов STN

Следующие предположения применимы к Корпоративная голосовая связь вызовам:
  
- Для 60% пользователей включена Корпоративная голосовая связь, а для 60% этих пользователей — вызовы по STN.
    
- Каждый из пользователей, которым разрешены звонки в ТСОП, выполняет 4 звонка в ТСОП в течение часов максимальной нагрузки. Каждый звонок длится 3 минуты.
    
- В 65% этих голосовых звонков в ТСОП используется обход сервера-посредника.
    
## <a name="mobility"></a>Мобильность

Предполагается. что для 40% зарегистрированных пользователей включена поддержка мобильности. Для каждого из пользователей с поддержкой мобильности предполагается, что использование мобильного клиента является дополнительным к использованию других экземпляров MPOP для этого пользователя, за исключением взаимодействий в конференциях, в которых мобильный клиент является просто еще одним типом клиента, используемым для участия в конференциях.
  
## <a name="persistent-chat"></a>Сохраняемый чат

Мы предполагаем, что 25% зарегистрированных пользователей будут принимать участие в сеансах сохраняемого чата со следующими характеристиками:
  
- в среднем по 1,5 комнат чата на пользователя;
    
- каждая комната чата приводит к выполнению 12 запросов опроса в час, и каждый из опросов в среднем выполняется для 10 пользователей.
    
## <a name="response-group-and-call-park"></a>Группа ответа и парковка вызовов

Мы предполагаем, что 0,15% зарегистрированных пользователей входят в группы ответа. Мы также предполагаем, что в каждый момент времени 0,02% зарегистрированных пользователей имеют запаркованные вызовы.
