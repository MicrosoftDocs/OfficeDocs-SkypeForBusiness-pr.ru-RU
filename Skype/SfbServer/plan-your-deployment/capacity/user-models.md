# <a name="user-models-in-skype-for-business-server"></a>Пользовательские модели в Skype для бизнеса Server
 
Описанные здесь модели пользователей предоставляют основу для измерения и рекомендации по планированию производительности, описанные в статье [Использование модели пользователей "Планирование мощности" для Skype для бизнеса Server](user-model.md).
  
## <a name="skype-for-business-server-user-models"></a>Модели пользователя в Skype для бизнеса Server

В приведенной ниже таблице описаны пользовательские модели для регистрации, контактов, обмена мгновенными сообщениями и присутствия в Skype для бизнеса Server.
  
**Пользовательская модель среды и регистрации**

|**Категория**|**Описание**|
|:-----|:-----|
|Размер и распределение развертывания  <br/> |Мы моделируем большое развертывание с тремя центральными сайтами и с одним интерфейсным пулом на каждом сайте.  <br/> |
|Процент пользователей Active Directory  <br/> |Предполагается, что 70% от всех пользователей Active Directory в организации включены для Skype для бизнеса Server. 80% от пользователей, которые поддерживаются пользователями, подключены к Skype для бизнеса Server каждый день (80% параллелизма). На основе количества параллельных пользователей рассчитываются числа в оставшейся части этого раздела.  <br/> |
|Изменения в службе каталогов Active Directory  <br/> |Предполагается, что 0,5% от общего числа пользователей созданы и включены для Skype для бизнеса в Active Directory, а это 0,5% от общего числа пользователей отключается из Active Directory и из Skype для бизнеса на каждую неделю. 5% пользователей по крайней мере один атрибут Active Directory изменялся каждую неделю.  <br/> |
|Группы рассылки Active Directory  <br/> |Предполагается, что количество групп рассылки Active Directory в Организации равно трем большему числу пользователей в службе каталогов Active Directory. Эти группы рассылки имеют следующие размеры:  <br/> • у 64% есть 2-30 пользователей  <br/> • 13% имеют 31-50 пользователей  <br/> • 10% пользователей 51-100  <br/> • 13% имеют 101-500 пользователей  <br/> |
|Пользователи протокола VoIP  <br/> |60% пользователей Skype для бизнеса Server поддерживает единую связь (то есть их номера телефонов принадлежат Skype для бизнеса Server).  <br/> |
|Распределение зарегистрированных клиентов  <br/> |65% от клиентов, работающих по Skype для бизнеса, включая Skype для бизнеса и Lync Phone Edition.  <br/> 30% клиентов с запущенным клиентским программным обеспечением из предыдущей версии Lync.  <br/> 5% клиентов, использующих Skype для бизнеса Web App.  <br/> Мы предполагаем, что если мобильность включена, то 40 % пользователей используют эту функцию параллельно с другими вышеуказанными зарегистрированными клиентами. В этом случае коэффициент нескольких точек подключения клиента (MPOP) составляет 1:1,9. Если мобильность отключена, то коэффициент MPOP составляет 1:1,5.  <br/> |
|Распределение удаленных пользователей  <br/> |70 % пользователей подключается внутренним образом.  <br/> 30% пользователей, подключающихся через пограничный сервер (возможно, у вас также есть директор, но это не обязательно).  <br/> |
|Распределение контактов  <br/> |Пользователь имеет не более 1000 контактов. Менее 1 % пользователей имеет 1000 контактов. Менее 25 % пользователей имеет как минимум 100 контактов.  <br/> Пользователи с подключением к общедоступному облаку в среднем имеют по 80 контактов. Среди них:  <br/> • 50% контактов в Организации. 10% от пользователей — это удаленные пользователи, подключающиеся извне межсетевого экрана.  <br/> • 40% контактов являются пользователями Skype.  <br/> • 10% контактов из федеративных партнеров.  <br/> Пользователи без возможности подключения к общедоступному облаку в среднем имеют по 50 контактов. Среди них:  <br/> • 80% контактов в Организации. 10% от пользователей — это удаленные пользователи, подключающиеся извне межсетевого экрана.  <br/> • 20% контактов из федеративных партнеров.  <br/> Каждый пользователь имеет одну группу рассылки в своем списке контактов. Для тестирования производительности мы предполагаем, что группы рассылки всегда расширенные.  <br/> |
|Время сеанса  <br/> |Среднее время сеанса входа пользователя в систему составляет 12 часов. Все пользователи выполняют вход в течение 120 минут после начала сеанса.  <br/> |
   
**Пользовательская модель обмена мгновенными сообщениями и присутствия**

|**Категория**|**Описание**|
|:-----|:-----|
|Одноранговые сеансы обмена мгновенными сообщениями  <br/> |Каждый пользователь в среднем имеет по шесть одноранговых сеансов обмена мгновенными сообщениями в день.  <br/> 10 мгновенных сообщений на сеанс.  <br/> Каждое сообщение соответствует двум ИНФОРМАЦИОНным сообщениям SIP и 2 сообщениям SIP 200 ОК (для индикаторов состояния, таких как\<"\> имя является вводом")  <br/> |
|Сеансы группового чата  <br/> |Среднее число сообщений, отправленных в сеансе группового чата: 5 на пользователя.  <br/> Среднее число сообщений, отправленных при обмене мгновенными сообщениями в процессе аудио- или видеоконференции: 2 на пользователя.  <br/> |
|Опрос присутствия  <br/> |В основном для опроса присутствия мы предполагаем по 60 опросов на каждого пользователя в час. Для каждого пользователя предполагаются следующие средние значения.  <br/> • Один опрос за день присутствия пользователей на вкладке организации пользователя (но не в списке контактов). Среднее количество неконтактных данных на вкладке организации пользователя составляет 15 пользователей. В день выполняется две операции просмотра карточки контакта.  <br/> • Один опрос присутствия каждый раз, когда пользователь щелкает другого пользователя, чтобы начать беседу, в расчете на час.  <br/> • Шесть поисков, выполненных пользователем, в течение часа. При каждом выполнении поиска пакетный опрос отправляется каждому пользователю в списке результатов поиска. Мы предполагаем, что средний размер результатов поиска — 20. Если результаты поиска остаются на экране, то пакетный опрос обновляется каждые 5 минут; мы предполагаем, что будет два таких обновления в час.  <br/> • Когда пользователь открывает или предварительно просматривают электронную почту в Outlook, в полях "Кому" и "копия" в сообщении электронной почты запрашивается по пять сообщений в часах и четырех пользователей на электронную почту.  <br/> |
|Подписки на сведения о присутствии  <br/> |Когда пользователь добавляет другого пользователя в качестве контакта, он подписывается на пять категорий сведений об этом пользователе. Обновления этих категорий сведений автоматически отправляются первому пользователю. <br/> Для каждого клиента отправляется один пакетный запрос подписки, чтобы получить сведения о присутствии в среднем 40 контактов, с дополнительными 40 диалогами, чтобы получить сведения о присутствии для федеративных контактов.  <br/> Сведения о присутствии для членов расширенной группы рассылки извлекаются с помощью постоянных подписок на сведения о присутствии, а не опросов, и моделируются как 1 расширение на пользователя каждые 2 часа.  <br/> Краткосрочные подписки происходят, когда пользователь входит в систему, для всех контактов пользователя имеется Пакетная подписка, а затем пользователь вскоре выходит из нее. Корпорация Майкрософт предполагала 6 коротких подписок на пользователей в час, где каждая подписка длится 10 минут. <br/> |
|Публикация сведений о присутствии  <br/> |Состояние присутствия публикуется в среднем 4 раза на пользователя в час, максимально 6 раз на пользователя в час.  <br/> |
|Размер документа сведений о присутствии  <br/> |Средний размер полного документа сведений о присутствии оценивается в 4 КБ, максимальный размер — 25 КБ.  <br/> |

В обычных интерфейсных пулах Skype для бизнеса Server имеет максимально поддерживаемый размер собрания для пользователей 250. В каждом пуле в каждый момент времени может быть размещено одно собрание в 250 пользователей. Во время проведения такого большого собрания в пуле также могут размещаться другие небольшие конференции. Кроме того, можно поддерживать собрания до 1000 пользователей, настраивая выделенный пул для размещения таких собраний. Подробные сведения можно найти [в разделе Планирование собраний на больших собраниях в Skype для бизнеса Server](../../plan-your-deployment/conferencing/large-meetings.md).
  
Конференции моделировались следующим образом:
  
- 85 % конференций имели четырех участников;
    
- 10 % конференций имели шесть участников;
    
- 5 % конференций имели одиннадцать участников;
    
- одна большая конференция с 250 участниками.

|**Категория**|**Описание**|

## <a name="busy-hour"></a>Период максимальной нагрузки

Для одноранговых сеансов пиковая нагрузка вычисляется с помощью попыток вызовов в час наибольшей нагрузки (BHCA). Этот термин из сферы голосовой связи предполагает, что 50 % всех вызовов в день будут выполняться в 20 % времени. Это значение вычисляется по следующей формуле:
  
 `BHCA=(total calls * 0.5) / 1.6`
  
При тестировании производительности период максимальной нагрузки моделируется путем запуска сеансов VoIP и других одноранговых сеансов с уровнем максимальной нагрузки по крайней мере 1,6 часов в день.
  
Пиковая нагрузка конференц-связи предполагает, что 75 % всех конференций для восьмичасового дня происходит в 4 часа максимальной нагрузки. В эти часы нагрузка в полтора раза выше средней нагрузки конференц-связи.
  
## <a name="enterprise-voice-to-pstn-calls"></a>Корпоративная голосовая связь по КОММУТИРУЕМым звонкам

Следующие предположения применимы к голосовым звонкам для предприятий:
  
- 60% пользователей разрешено для поддержки корпоративной голосовой связи, а 60% от этих пользователей разрешено звонить через PSTN.
    
- Каждый из пользователей, которым разрешены звонки в ТСОП, выполняет 4 звонка в ТСОП в течение часов максимальной нагрузки. Каждый звонок длится 3 минуты.
    
- В 65 % этих голосовых звонков в ТСОП используется обход сервера-посредника.
    
## <a name="mobility"></a>Мобильность

Предполагается, что для 40 % зарегистрированных пользователей включена поддержка мобильности. Для каждого из пользователей с поддержкой мобильности предполагается, что использование мобильного клиента является дополнительным к использованию других экземпляров MPOP для этого пользователя, за исключением взаимодействий в конференциях, в которых мобильный клиент является просто еще одним типом клиента, используемым для участия в конференциях.
  
## <a name="persistent-chat"></a>Сохраняемый сеанс беседы

Мы предполагаем, что 25 % зарегистрированных пользователей будут принимать участие в сеансах сохраняемого чата со следующими характеристиками:
  
- В среднем по 1,5 комнаты чата на пользователя
    
- Каждая комната чата приводит к выполнению 12 запросов опроса в час, и каждый из опросов в среднем выполняется для 10 пользователей.
    
## <a name="response-group-and-call-park"></a>Группа ответа и парковка вызовов

Мы предполагаем, что 0,15 % зарегистрированных пользователей входят в группы ответа. Мы также предполагаем, что в каждый момент времени 0,02 % зарегистрированных пользователей имеют запаркованные вызовы.
