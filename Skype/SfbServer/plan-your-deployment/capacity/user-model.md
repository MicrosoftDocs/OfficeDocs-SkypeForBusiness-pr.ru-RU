## <a name="front-end-server"></a>переднего плана

> [!NOTE]
> Расширенные пулы для этой роли сервера не поддерживаются.

В пуле переднего плана вы должны иметь один сервер переднего плана для всех пользователей 6 660, размещенных в пуле, предполагая, что технология Hyper-Threading включена на всех серверах пула, и что оборудование сервера соответствует рекомендациям в [требованиях к серверу для Skype для Business Server 2015](../requirements-for-your-environment/server-requirements.md) или [требования к системе для Skype для бизнеса Server 2019](../../../SfBServer2019/plan/system-requirements.md). Максимальное количество пользователей в одном пуле переднего плана — 80 000, но при этом в предположении, что технология Hyper-Threading включена на всех серверах в пуле. Если у вас более 80 000 пользователей на сайте, вы можете развернуть более одного пула переднего плана.

Если вы учитываете количество пользователей в пуле переднего плана, включите пользователей, которые размещаются на устройствах с бесперебойной подразделением и бесперебойно работающих серверах филиалов в филиалах, связанных с этим пулом переднего плана.

When an active server is unavailable, its connections are transferred automatically to the other servers in the pool. В сценарии, в котором есть пользователи 30 000 и пять серверов переднего плана, если один сервер недоступен, подключение 6000 ваших пользователей должно быть передано на другие четыре оставшихся сервера. These four remaining servers will then each have 7500 users, which is a larger number than recommended.

Если вместо этого вы начали пользоваться шестью серверами переднего плана для пользователей 30 000 и они становятся недоступными, общее количество пользователей 5000 нужно перейти на оставшиеся пять серверов. These five remaining servers will then each host 6000 users, which is in the recommended range.

Максимальное количество пользователей в пуле переднего плана — 80 000. Максимальное число серверов переднего плана в пуле составляет 12.

Для пула переднего плана с пользователями 80 000 более двенадцати серверных серверов будут хорошо подходящими для повышения производительности, в типичных развертываниях, которые следуют за [пользовательскими моделями в Skype для бизнеса Server](user-models.md). В развертываниях, разработанных для поддержки перехода на аварийный сбой, предполагается, что в каждом из двух подключенных интерфейсных пулов может быть размещено не более 40 000 пользователей, в котором каждый из них должен иметь достаточное количество серверов переднего плана o другой.

Количество пользователей, которые поддерживают хорошую производительность с помощью определенного пула переднего плана, может отличаться от приведенных ниже.

## <a name="conferencing-maximums"></a>Максимумы конференц-связи

Given the user model that 5% of users in a pool may be in a conference at any one time, a pool of 80,000 users could have about 4,000 users in conferences simultaneously. These conferences are expected to be a mix of media (some IM-only, some IM with audio, some audio/video, for example) and number of participants. There isn't a hard limit for the actual number of conferences allowed, and actual usage determines the actual performance. Например, если ваша организация имеет множество конференций смешанного режима, чем предполагается для пользовательской модели, возможно, потребуется развернуть другие серверы переднего плана или серверы конференц-связи по сравнению с рекомендациями, приведенными в этой статье. Подробные сведения о допущениях в пользовательской модели можно найти в разделе [модели пользователей в Skype для бизнеса Server](user-models.md).

Максимально допустимый размер Конференции, размещенный на обычном пуле переднего плана в Skype для бизнеса Server, где также размещены пользователи, — 250 участников. While a 250-user conference is happening, the pool still supports other conferences as well, such that a total of 5% of pool users are in concurrent conferences. Например, в пуле двенадцати серверов переднего плана и пользователей 80 000, а также при Конференции 250-User, Skype для бизнеса Server поддерживает 3 750 других пользователей, участвующих в небольших конференциях.

Независимо от количества пользователей, подключенного к внешнему интерфейсу или серверу Standard Edition, в Skype для бизнеса Server поддерживается не менее 125 других пользователей, участвующих в небольших конференциях в том же пуле или на сервере, где размещается конференция 250-пользователи.

Чтобы включить Конференц-связь между пользователями 250 и 1000, вы можете настроить отдельный пул переднего плана только для того, чтобы разместить эти конференции. Этот пул переднего плана не будет содержать пользователей. Подробные сведения можно найти [в разделе Планирование собраний на больших собраниях в Skype для бизнеса Server](../../plan-your-deployment/conferencing/large-meetings.md).

Если в вашей организации есть гораздо больше конференций смешанного режима, чем предполагается для пользовательской модели, вам может потребоваться развернуть другие серверы переднего плана, чем рекомендовано в этом документе (до 12 внешних серверов). Подробные сведения о допущениях в пользовательской модели можно найти в разделе [модели пользователей в Skype для бизнеса Server](user-models.md).

## <a name="edge-server"></a>пограничный сервер

> [!NOTE]
> Расширенные пулы для этой роли сервера не поддерживаются.

Вы должны развернуть один пограничный сервер для каждого удаленного пользователя 12 000, который будет одновременно получать доступ к сайту. Как минимум, мы рекомендуем использовать два пограничных сервера для обеспечения высокой доступности. Эти рекомендации предполагают, что оборудование пограничных серверов соответствует рекомендациям, приведенным в разделе [аппаратные платформы сервера](https://technet.microsoft.com/library/c964c1c0-0153-472b-88ad-a38866e0df0c.aspx).

При указании числа пользователей для пограничных серверов включите пользователей, относящихся к бесперебойным устройствам филиалов и к бесперебойным серверам филиалов в филиалах, которые связаны с пулом переднего плана на этом сайте.

> [!NOTE]
> Чтобы повысить производительность службы пограничного конференц-связи A/V на пограничных серверах, необходимо включить масштабирование на стороне приема (RSS) на сетевых адаптерах пограничного сервера. RSS поддерживает параллельную обработку входящих пакетов несколькими процессорами сервера. Для получения дополнительных сведений установите флажок "масштабирование на[стороне приема (RSS)" в Windows Server 2012](https://go.microsoft.com/fwlink/p/?linkId=268731)". For details about how to enable RSS, you'll need to refer to your network adapter documentation.

Подробные сведения можно найти[в разделе масштабирование на стороне приема в Windows Server 2012](https://go.microsoft.com/fwlink/p/?linkId=268731). For details about how to enable RSS, you'll need to refer to your network adapter documentation.

## <a name="back-end-server"></a>внутренний сервер

Несмотря на то, что многие сведения из базы данных хранятся на серверах переднего плана, необходимо убедиться в том, что серверные серверы соответствуют рекомендациям по оборудованию, перечисленным ранее в этом разделе и на [серверных аппаратных платформах](https://technet.microsoft.com/library/c964c1c0-0153-472b-88ad-a38866e0df0c.aspx).

Для обеспечения высокой доступности серверного сервера мы рекомендуем развернуть группы доступности AlwaysOn или зеркальное отображение сервера. For more information, see [Back End Server high availability in Skype for Business Server](../../plan-your-deployment/high-availability-and-disaster-recovery/back-end-server.md).

## <a name="video-interop-server-capacity"></a>Емкость сервера видеовзаимодействия

Если вы разработали сервер видеосвязи и вам нужно определить емкость, вы можете узнать о максимальном количестве систем для проведения видеоконференций (Вткс), которые будут находиться в одновременных звонках. For example, if you have 250 VTCs in your organization and your user model estimates that at most, 20% of them might be in concurrent calls, you base your capacity planning on 50 concurrent VTCs.
