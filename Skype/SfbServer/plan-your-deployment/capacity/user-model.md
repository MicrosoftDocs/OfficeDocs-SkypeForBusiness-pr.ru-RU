# <a name="capacity-planning-user-model-usage-for-skype-for-business-server"></a>Использование пользовательской модели планирования емкости для Skype для бизнеса Server

В этой статье содержится руководство по количеству серверов, необходимых на сайте для количества пользователей на этом сайте, в соответствии с использованием, описанным в пользовательских моделях [в Skype для бизнеса Server.](user-models.md)

> Во всех рекомендациях в этой статье предполагается, что на серверах установлен накопительный обновления Skype для бизнеса за ноябрь 2015 г. или более поздней версии.

## <a name="front-end-server"></a>Сервер переднего плана

> [!NOTE]
> Растянутые пулы не поддерживаются для этой роли сервера.

В пуле переднего сервера должен быть один сервер переднего сервера для каждого 6660 пользователей, которые будут иметься в пуле при условии, что hyper-threading включен на всех серверах в пуле и что оборудование сервера соответствует рекомендациям в требованиях к серверу для Skype для бизнеса [Server 2015](../requirements-for-your-environment/server-requirements.md) или системных требований для Skype для бизнеса [Server 2019.](../../../SfBServer2019/plan/system-requirements.md) Максимальное число пользователей в одном пуле переднего сервера составляет 80 000, опять же при условии, что hyper-threading включен на всех серверах в пуле. Если на сайте более 80 000 пользователей, можно развернуть несколько пулов переднего входа.

При учете количества пользователей в пуле переднего сервера включайте всех пользователей, которые находятся на устройствах для связи в филиалах и серверах филиалов в филиалах, связанных с этим пулом переднего сервера.

Если активный сервер недоступен, его подключения автоматически перенаправляются на другие серверы в пуле. В сценарии с 30 000 пользователей и пятью серверами переднего сервера, если один сервер недоступен, подключения 6000 пользователей необходимо перенести на остальные четыре оставшихся сервера. Эти четыре оставшихся сервера будут иметь 7500 пользователей, что превышает рекомендуемое число.

Если вместо этого вы начали работу с шестью серверами переднего сервера для 30 000 пользователей и один из них становится недоступным, 5000 пользователей должны перейти на оставшиеся пять серверов. Эти пять оставшихся серверов будут принимать на каждом из них 6000 пользователей, что находится в рекомендуемом диапазоне.

Максимальное количество пользователей в пуле переднего ряда составляет 80 000. Максимальное число серверов переднего сервера в пуле — 12.

Для интерфейсного пула с 80 000 пользователей 12 серверов переднего интерфейса будут хорошо работать в типичных развертываниях, которые следуют пользовательским моделям [в Skype для бизнеса Server.](user-models.md) В развертываниях, предназначенных для поддержки аварийного восстановления, предполагается, что в каждом из двух сопряженных пулов переднего сервера может быть размещено не более 40 000 пользователей, в которых каждый пул имеет достаточно серверов переднего сервера для размещения пользователей в обоих пулах, если требуется перенаправление одного пула в другой.

Количество пользователей, поддерживаемых определенным пулом переднего ряда с хорошей производительностью, может отличаться от числа пользователей по следующим причинам:

- Оборудование для серверов переднего сервера не соответствует рекомендациям.

- Использование в организации сильно отличается от пользовательских моделей, например, при гораздо большем трафике для работы сконференцией.

|**Средняя пропускная способность на пользователя**|**Требования к пропускной способности для каждого сервера переднего сервера с 6660 пользователями**|

## <a name="conferencing-maximums"></a>Максимумы конференц-связи

С учетом модели пользователей, когда 5 % пользователей в пуле могут одновременно быть в конференции, пул из 80 000 пользователей может одновременно иметь около 4000 пользователей. Эти конференции могут использовать разные типы мультимедиа (например, некоторые могут использовать только обмен сообщениями, а другие — обмен сообщениями и аудио или аудио и видео) и охватывать разное количество участников. Не существует жесткого ограничения на фактическое количество разрешенных конференций, а фактическое использование определяет фактическую производительность. Например, если в организации имеется намного больше конференций в смешанном режиме, чем предполагается в пользовательской модели, может потребоваться развернуть больше серверов переднего интерфейса или серверов видео- и видеоконференции, чем по рекомендациям из этой статьи. For details about the assumptions in the user model, see [User models in Skype for Business Server.](user-models.md)

The maximum supported conference size hosted by a regular Skype for Business Server Front End pool which also hosts users is 250 participants. Во время проведения конференции с 250 пользователями пул также поддерживает другие конференции, но количество пользователей, одновременно участвующих в конференциях, не должно превышать 5% от общего числа пользователей пула. Например, в пуле из двенадцати серверов переднего интерфейса и 80 000 пользователей во время конференции с 250 пользователями Skype для бизнеса Server поддерживает 3750 других пользователей, участвующих в конференциях меньшего количества.

Независимо от количества пользователей, которые размещены в интерфейсном пуле или на сервере Standard Edition, Skype для бизнеса Server поддерживает не менее 125 других пользователей, участвующих в конференциях меньшего размера в том же пуле или сервере, на котором размещена конференция с 250 пользователями.

Чтобы включить конференции с от 250 до 1000 пользователей, можно настроить отдельный пул переднего входа только для их проведения. В этом пуле переднего входа пользователи не будут вмещаяся. For details, please see [Plan for large meetings in Skype for Business Server.](../../plan-your-deployment/conferencing/large-meetings.md)

Если в организации имеется намного больше конференций в смешанном режиме, чем предполагает пользовательская модель, может потребоваться развернуть больше серверов переднего интерфейса, чем мы решили в этом документе (до 12 интерфейсных серверов). For details about the assumptions in the user model, see [User models in Skype for Business Server.](user-models.md)

## <a name="edge-server"></a>Пограничный сервер

> [!NOTE]
> Растянутые пулы не поддерживаются для этой роли сервера.

Необходимо развернуть один сервер для каждого 12 000 удаленных пользователей, которые будут получать доступ к сайту одновременно. Как минимум рекомендуется использовать два сервера для высокой доступности. В этих рекомендациях предполагается, что оборудование для ваших серверов соответствует рекомендациям в серверных [аппаратных платформах.](https://technet.microsoft.com/library/c964c1c0-0153-472b-88ad-a38866e0df0c.aspx)

При учете количества пользователей для этих серверов включайте пользователей, которые находятся на устройствах для связи в филиалах и серверах филиалов в филиалах, связанных с пулом переднего сервера на этом сайте.

> [!NOTE]
> Чтобы повысить производительность службы A/V Conferencing Edge на ваших серверах, необходимо включить масштабирование на стороне получения (RSS) на сетевых адаптерах на ваших серверах. RSS позволяет обрабатывать входящие пакеты параллельно несколькими процессорами на сервере. Дополнительные сведения см. в "Масштабирование на стороне получения[(RSS) в Windows Server 2012".](https://go.microsoft.com/fwlink/p/?linkId=268731) Подробные сведения о в том, как включить RSS, необходимо обратиться к документации по сетевому адаптеру.

## <a name="video-interop-server-capacity"></a>Емкость сервера видеосвязи

Если развернут сервер видеосвязи и необходимо определить емкость, необходимо посмотреть максимальное количество систем видеоконференции (VTC), которые будут одновременно вызовами. Например, если в организации имеется 250 VTC и пользовательская модель оценивает, что не более 20 % из них могут быть в вызовах одновременно, планирование емкости зависит от 50 одновременного планирования.
