# <a name="capacity-planning-for-persistent-chat-server-in-skype-for-business-server-2015"></a>Планирование емкости для стойких чат-серверов в Skype для бизнеса Server 2015 г.
 
**Сводка:** Ознакомьтесь с этой темой, чтобы узнать о планировании емкости для сервера сохраняемой беседы в Skype для бизнеса Server 2015 г.
  
Постоянный чат-сервер может выполнять много пользовательский чат в режиме реального времени, который может сохраняться для поиска и поиска в будущем. В отличие от группового обмена мгновенными сообщениями (IM), которые сохраняются в почтовом ящике пользователя, если настроена история беседы, сеанс сохраняемого сервера чата остается открытым дольше, а содержимое сохраняется на сервере вместе с сообщениями, файлами, URL-адресами и другими данными, которые являются частью текущей беседы.
  
Планирование емкости является важной частью подготовки к развертыванию сервера сохраняемого чата. Этот раздел содержит таблицы планирования емкости, которые можно использовать для определения наилучшей конфигурации для развертывания. В нем также описывается, как наилучшим образом управлять развертываниями сохраняемой системы чат-сервера, которые требуют большей емкости в часы пик.
  
Перед чтением этого раздела необходимо ознакомиться с топологиями сохраняемого чата. Дополнительные сведения см. в [топологии Plan Persistent Chat Server.](topology.md)

> [!NOTE] 
> Постоянный чат доступен в Skype для бизнеса Server 2015 г., но больше не поддерживается Skype для бизнеса Server 2019 г. Такая же функциональность доступна в Teams. Дополнительные сведения см. в ссылке Начало работы [с обновлением Microsoft Teams обновления.](/microsoftteams/upgrade-start-here) Если вам нужно использовать постоянный чат, вы можете либо перенести пользователей, требующих Teams, либо продолжить использование Skype для бизнеса Server 2015 г. 
  
## <a name="persistent-chat-server-capacity-planning"></a>Постоянное планирование емкости чат-сервера

Следующие таблицы помогут вам с планированием емкости для стойких чат-серверов, смоделируя влияние различных настроек сервера чата на емкость.
  
- Планирование емкости для числа пользователей
    
- Планирование емкости для доступа к чату
    
- Планирование емкости для доступа к чату по приглашению
    
- Планирование производительности
    
### <a name="plan-capacity-for-number-of-users-for-persistent-chat-server"></a>Планирование емкости для числа пользователей для сохраняемого сервера чата

Используйте следующую таблицу для определения, какое количество пользователей вы сможете поддерживать.
  
**Пример максимальной емкости пула чат-сервера**

- Экземпляры службы службы активного сохраняемой чата: 4  <br/> 
- Постоянные экземпляры службы чата: 8 (максимум 4 могут быть активными; 4 должны быть неактивными)  <br/>
- Подключение активных пользователей: 80 000  <br/>
- Общее количество пользователей с предварительной провизией: 150 000  <br/>
- Количество конечных точек: 120 000  <br/>
   
В предыдущем примере планируется поддерживать максимальное число пользователей, которое позволяет стойкий сервер чата: четыре сервера или экземпляры службы сохраняемого чата (может иметь еще четыре пассивных сервера, работающих на стойком сервере чата для высокой доступности и аварийного восстановления) и 20 000 пользователей на сервер, в общей сложности 80 000 активных пользователей.

В данном примере приведены комнаты чата, классифицируемые как малые (30 активных пользователей в любое время), средние (150 активных пользователей) и большие (16 000 активных пользователей). Поддерживаемый номер комнат чата определенного размера в таблице выше вычисляется на основе общего числа:

- активных пользователей в комнатах чата определенного размера;
    
- комнат чата определенного размера, к которым присоединяется один пользователь.
    
В предыдущей таблице планирования мощности для каждой комнаты чата указывается число элементов управления доступом, связанных с этой комнатой чата, включая элементы, назначенные непосредственно этой комнате чата. Управлять доступом к отдельным комнатам чата можно с помощью списков управления доступом (ACL). Кроме того, можно управлять доступом на уровне категорий. В ACL отдельный элемент управления доступом может быть либо группой пользователей (такой как группа безопасности или список рассылки), либо одним пользователем. Можно определять элементы управления доступом для управляющих, выступающих и членов комнаты чата.
  
> [!IMPORTANT]
> При планировании стратегии управления комнатами чата следует иметь в виду, что разрешено 2 миллиона элементов управления доступом. Если вычисленное количество элементов управления доступом превышает 2 миллиона, это может значительно ухудшить серверную производительность. Чтобы такая проблема не возникала, везде, где это возможно, старайтесь использовать в элементах управления доступом группы пользователей, а не отдельных пользователей. 
  
### <a name="plan-capacity-for-managing-chat-room-access-by-invitation"></a>Планирование емкости для управления доступом к чату по приглашению

Вы можете использовать следующую таблицу планирования емкости, чтобы понять количество приглашений, которые создает и хранится в базе данных "Постоянный чат" при настройке отправки приглашений. Вы управляете приглашениями в категории, используя страницу параметров категории чата в панели управления Skype для бизнеса Server или с помощью Windows PowerShell, **set-csPersistentChatCategory**.  Вы можете управлять приглашениями в чате (в соответствии с тем,  что позволяет категория) с помощью страницы управления комнатами, запущенной из клиента Skype для бизнеса, или с помощью **Windows PowerShell,set-csPersistentChatRoom**.
  
В демонстрационных данных следующей таблицы предполагается, что на странице **Chat room settings (Параметры комнаты чата)** для 50% всех комнат чата параметр **Invitations (Приглашения)** установлен в значение **Yes (Да)**.
  
> [!IMPORTANT]
> Если рассчитанное количество приглашений, создаваемых сервером, превышает 1 миллион, то производительность сервера может значительно снизиться. Чтобы избежать этой проблемы, необходимо свести к минимуму количество комнат чатов, настроенных для отправки приглашений, или ограничить число пользователей, которые могут присоединиться к чатам, настроенным для отправки приглашений.
