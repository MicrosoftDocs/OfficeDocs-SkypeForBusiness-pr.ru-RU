# <a name="plan-for-persistent-chat-server-in-skype-for-business-server-2015"></a>Планирование на стойкий сервер чата в Skype для бизнеса Server 2015 г.
 
**Сводка:** Ознакомьтесь с этой темой, чтобы узнать, как планировать постоянный сервер чата в Skype для бизнеса Server 2015 г.
  
Постоянный сервер чата — это необязательная роль, которая позволяет нескольким пользователям в организации участвовать в беседах в чате, которые сохраняются с течением времени. Несмотря на то, что пользователи могут общаться в режиме реального времени во время сеанса чата, содержимое каждого сеанса, включая текст, ссылки и файлы, является настойчивым, что означает, что пользователи могут просматривать и искать все содержимое сеанса в любое время.
  
Постоянный сервер чата может помочь улучшить связь в организации с помощью:
  
- Расширение осведомленности об информации и участие в организации
    
- Включение эффективного обмена информацией 
    
- Улучшение связи между группами, в том числе географически рассредоточенными и межфункциональными группами.
    
- Уменьшение перегрузки информации
    
- Следуя правилам соответствия требованиям, необязательный развертывание службы сохраняемой системы соответствия требованиям чата

> [!NOTE] 
> Постоянный чат доступен в Skype для бизнеса Server 2015 г., но больше не поддерживается Skype для бизнеса Server 2019 г. Такая же функциональность доступна в Teams. Дополнительные сведения см. в ссылке Начало работы [с обновлением Microsoft Teams обновления.](/microsoftteams/upgrade-start-here) Если вам нужно использовать постоянный чат, вы можете либо перенести пользователей, требующих Teams, либо продолжить использование Skype для бизнеса Server 2015 г. 
    
## <a name="persistent-chat-server-high-level-architecture"></a>Архитектура сохраняемого сервера чата высокого уровня

На следующей схеме показано представление архитектуры сохраняемого сервера чата на высоком уровне. 
  
![Сохраняемая архитектура High-Level chat Server.](../../media/0344f6e2-0c6d-4391-b4b3-ec31062b1576.png)
  
Persistent Chat состоит из роли переднего сервера, который предоставляет службы сохраняемого чата, а также компонент базы SQL базы данных. Как передние, так и задние компоненты включены в выделенный пул постоянных чатов. Каждый компьютер, на котором размещен постоянный сервер чата, должен иметь доступ к существующей топологии Skype для бизнеса Server 2015 года. На этой схеме имеется один пул сохраняемого сервера чата (A), который зависит от Skype для бизнеса Server пула A для маршрутивки сообщений в него.
  
Вы можете развернуть один или несколько пулов серверов сохраняемого чата, каждый из которых поддерживает до четырех активных постоянных серверов чата, поддерживающих до 80K одновременно пользователей.
  
Skype для бизнеса Server 2015 г. взаимодействует со службой сохраняемого чата с помощью протокола инициации сеанса (SIP) для регистрации, а также с протоколом SIP (XCCOS) для чата. 
  
## <a name="persistent-chat-services"></a>Постоянные службы чата

На следующей схеме показаны службы сохраняемого сервера чата, а также то, как эти службы взаимодействуют с компонентами базы данных. В числе основных компонентов — службы сохраняемого чата и служба соответствия требованиям. Компоненты задней части включают хранилище "Стойкий чат" и хранилище соответствия требованиям для сохраняемого чата.
  
![Постоянные службы High-Level chat Server.](../../media/bcdbadbe-e868-4a46-8a73-36562648fdf7.png)
  
### <a name="chat-service"></a>Служба чата

Служба чата, также называемая службой Channel, является основной службой, ответственной за постоянный сервер чата. Служба чата предоставляет следующие функции:
  
- Принимает входящие сообщения
    
- Регистрация и перечисление участников в режиме онлайн в комнате сохраняемой беседы
    
- Передает сообщения другим подписчикам канала
    
- Реализует логику управления каналами, приглашений в чат, поиска и новых уведомлений о контенте
    
Служба persistent Chat сохраняет и получает доступ к содержимому комнаты чата и другим метаданным системы (правила авторизации и так далее) с помощью хранилища сохраняемого чата. В службе хранятся файлы, которые загружаются в комнаты чатов в хранилище файлов "Стойкий чат".
  
### <a name="compliance-service"></a>Служба соответствия требованиям

Если в организации есть правила, которые требуют архивировать действия сохраняемой беседы, вы можете развернуть необязательный службу сохраняемой системы соответствия требованиям. Служба соответствия требованиям отвечает за архивавку контента чата и событий, таких как присоединение и выход из комнат, в хранилище файлов "Сохраняемая переписка". Служба соответствия требованиям устанавливается на каждый постоянный сервер чата в пуле постоянных чатов. 
  
### <a name="web-services"></a>Веб-службы

Веб-службы сохраняемой беседы работают на Skype для бизнеса серверов переднего конца. Веб-службы зависят от службы IIS iiS и реализуются в качестве веб-компонентов:
  
- Постоянные веб-службы чата для отправки и скачивания файлов отвечают за размещение и поиск файлов из комнат чатов.
    
- Постоянные веб-службы чата для управления комнатами чата отвечают за предоставление пользователям возможности управлять своими комнатами чатов и создание новых комнат чатов.
    
## <a name="defining-requirements-for-your-organization"></a>Определение требований к организации

Если вы решите развернуть постоянный сервер чата, необходимо определить бизнес-требования организации, а затем определить топологию, инфраструктуру и технические требования для поддержки бизнес-потребностей. Чтобы оптимизировать развертывание, необходимо ответить на следующие вопросы:
  
- Вы мигрируете из предыдущей версии группового чат-сервера или предыдущей версии сервера сохраняемого чата или впервые развертываете стойкий сервер чата?
    
- Кто может использовать стойкий сервер чата? Для определения доступа пользователей на глобальном уровне, сайте или пользователе необходимо указать политики сохраняемой беседы.
    
- Сколько пользователей потребуется доступ к серверу сохраняемой чат-сервера? Persistent Chat Server поддерживает 150 000 поддерживаемых пользователей (включено политикой) и не более 80 000 одновременно пользователей. Один постоянный сервер чата может поддерживать 20 000 подключенных пользователей, а один пул сохраняемого сервера чата может иметь до 4 активных серверов для в общей сложности 80 000 одновременно подключенных пользователей.
    
- Каким образом можно контролировать сферу применения, этические нормы и права доступа? Можно определить категории для разделения этих границ и выбрать, кому разрешено быть в комнатах, созданных в каждой из этих категорий.
    
- Как можно ограничить права на создание комнат? Можно определить создателей, которые могут создавать комнаты. Создатели могут назначать других членов в качестве менеджеров комнат чата для текущего управления комнатами.
    
- Как создать комнаты? Persistent Chat Server предоставляет веб-функцию для создания и управления комнатами. Это можно запускать из Skype для бизнеса клиента. Вы можете выбрать решение клиента, которое реализует бизнес-требования и рабочий процесс, и настраивает persistent Chat Server, чтобы направлять пользователей к настраиваемому решению.
    
- Какие типы надстроек следует предоставлять? Надстройки улучшают возможность работы в комнате, используя области расширения в клиенте Skype для бизнеса для обеспечения контекста, который имеет отношение к комнате. Можно указать, какие общие надстройки будут наиболее полезными (например, веб-сайт компании, внутренние документы для совместной работы и т. д.). Менеджеры комнат чата могут выбирать зарегистрированные надстройки и сопоставлять их с комнатами. 
    
- Какие требования к высокой доступности и аварийному восстановлению у вас есть? Persistent Chat Server поддерживает SQL Server и SQL Server кластеризация для высокой доступности. Для аварийного восстановления сохраняемая поддержка сервера чата поддерживает до 8 серверов (4 активных и 4 в режиме ожидания) в растянутом пуле с SQL Server доставкой журналов. 
    
- Существуют ли нормативные требования? Если ваша компания находится в стране или регионе, где данные должны храниться внутри страны, может потребоваться развернуть несколько пулов сохраняемой системы чата, каждый из которых локализуется в определенной географической области. Комната, категория или надстройка не охватывает пулы— она принадлежит только одному пулу сохраняемого сервера чата. 
    
    > [!NOTE]
    > Наличие нескольких пулов сохраняемого сервера чата не дает вам больше масштабирования (вы все еще можете иметь только 80 000 одновечерных пользователей во всех пулах сохраняемого сервера чата). Основная причина поддержки нескольких пулов сохраняемой службы чат-сервера заключается в поддержке нормативных проблем.
