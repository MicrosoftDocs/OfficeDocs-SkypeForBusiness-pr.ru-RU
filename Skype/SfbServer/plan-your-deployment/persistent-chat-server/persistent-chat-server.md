# <a name="plan-for-persistent-chat-server-in-skype-for-business-server-2015"></a>Планирование для сервера сохраняемого чата в Skype для бизнеса Server 2015
 
**Сводка:** В этой статье рассказывается, как спланировать сохраняемый сервер чата в Skype для бизнеса Server 2015.
  
Сервер сохраняемого чата — это необязательная роль, которая позволяет нескольким пользователям в Организации принимать участие в беседах в чате, которые сохраняются с течением времени. Несмотря на то, что пользователи могут общаться в реальном времени во время сеанса чата, содержимое каждого сеанса, включая текст, ссылки и файлы, является сохраняемым, а это значит, что пользователи могут в любое время просматривать и искать весь контент сеанса.
  
С помощью сохраняемого сервера чата вы можете повысить эффективность связи в вашей организации, выполнив указанные ниже действия.
  
- Усовершенствованное информирование и участие в работе на уровне организации
    
- Обеспечение эффективного обмена информацией 
    
- Совершенствование обмена данными между участниками географически распределенных многофункциональных рабочих групп
    
- Сокращение перегруженности информацией
    
- Выполнение нормативных требований благодаря возможности развертывания, если необходимо, службы соответствия сохраняемого чата

> [!NOTE] 
> Сохраняемый чат доступен в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения см. в статье [Начало перехода на Microsoft Teams](/microsoftteams/upgrade-start-here). Если вам нужно использовать сохраняемый чат, то вы можете либо перенести пользователей, которым нужна эта функция, в Teams, либо продолжать использовать Skype для бизнеса Server 2015. 
    
## <a name="persistent-chat-server-high-level-architecture"></a>Высокоуровневая архитектура сервера сохраняемого чата

На приведенной ниже схеме показано общее представление архитектуры сервера сохраняемого чата. 
  
![Высокоуровневая архитектура сервера сохраняемого чата](../../media/0344f6e2-0c6d-4391-b4b3-ec31062b1576.png)
  
Функция сохраняемого чата включает в себя роль сервера переднего плана, в рамках которой предоставляются службы сохраняемого чата, а также внутренний компонент базы данных SQL. Как интерфейсные, так и внутренние компоненты включены в пул сохраняемого чата. Каждый компьютер, на котором размещен сервер для работы с постоянным чат, должен иметь доступ к существующей топологии 2015 в Skype для бизнеса Server. На этой схеме имеется один серверный пул (A), который зависит от пула сервера Skype для бизнеса для маршрутизации сообщений.
  
Вы можете развернуть один или несколько пулов серверов для постоянного чата, каждый из которых имеет до четырех активных серверов сохраняемого чата, поддерживающих одновременные пользователи 80K.
  
Skype для бизнеса Server 2015 взаимодействует со службой постоянного чата с помощью протокола SIP для регистрации и расширенного общения в чате по протоколу SIP (КСККОС) для чата. 
  
## <a name="persistent-chat-services"></a>Службы сохраняемого чата

На приведенной ниже схеме показаны серверные службы сохраняемого чата, а также взаимодействие этих служб с серверными компонентами базы данных. К интерфейсным компонентам относятся службы сохраняемого чата и служба соответствия. К внутренним компонентам относятся хранилище сохраняемого чата и хранилище соответствия сохраняемого чата.
  
![Высокоуровневые службы сервера сохраняемого чата](../../media/bcdbadbe-e868-4a46-8a73-36562648fdf7.png)
  
### <a name="chat-service"></a>Служба чата

Служба чата, называемая также службой канала, является основной службой, ответственной за сервер сохраняемого чата. Служба чата предоставляет следующие функции:
  
- Принимает входящие сообщения
    
- Регистрирует участников, подключенных к комнате сохраняемого чата, и выводит их списки
    
- Передает сообщения другим подписчикам канала
    
- Реализует логику управления каналом, приглашений в комнату чата, поиска и рассылки уведомлений о появлении нового контента
    
Служба «сохраняемый чат» хранит и обращается к содержимому комнаты чата и другим системным метаданным (правилам авторизации и т. д.) с помощью хранилища сохраняемого чата. Служба хранит файлы, которые отправляются в комнаты чата в хранилище файлов сохраняемого чата.
  
### <a name="compliance-service"></a>Служба соответствия

Если организация придерживается нормативных требований, в соответствии с которыми активность на сервере сохраняемого чата должна архивироваться, можно развернуть дополнительную службу соответствия сохраняемого чата. Служба соответствия отвечает за архивацию содержимого чата и событий (таких как вход в комнаты и выход из них) в хранилище файлов соответствия сохраняемого чата. Служба соответствия устанавливается на каждый сервер сохраняемого чата в пуле сохраняемого чата. 
  
### <a name="web-services"></a>Веб-службы

Веб-службы сохраняемого чата выполняются на серверах переднего плана Skype для бизнеса. Веб-службы зависят от служб IIS, и реализованы в качестве веб-компонентов:
  
- Веб-службы сохраняемого чата для выгрузки и загрузки файлов отвечают за публикацию и извлечение файлов из комнат чата.
    
- Веб-службы сохраняемого чата для управления комнатами чата отвечают за предоставление пользователям возможности управления комнатами чата и их создания.
    
## <a name="defining-requirements-for-your-organization"></a>Определение требований организации

Если вы решили развернуть сервер сохраняемого чата, вам нужно будет определить бизнес-требования Организации, а затем определить топологию, инфраструктуру и технические требования, необходимые для поддержки вашей компании. Чтобы оптимизировать развертывание, вам нужно ответить на следующие вопросы:
  
- Вы выполняете миграцию из предыдущей версии сервера группового чата или предыдущей версии сохраняемого сервера чата или вы развертываете постоянный сервер чата в первый раз?
    
- Кто может использовать сервер сохраняемого чата? Политики для доступа пользователей к сохраняемому чату могут быть установлены на глобальном уровне, на уровне сайта или на уровне пользователя.
    
- Какому количеству пользователей потребуется доступ к серверу сохраняемого чата? Сервер сохраняемого чата поддерживает 150 000 подготовленных пользователей (включаются политикой) и не более 80 000 одновременно работающих пользователей. Отдельный сервер сохраняемого чата может поддерживать до 20 000 подключенных пользователей, а отдельный пул серверов сохраняемого чата может иметь до 4 активных серверов, что в совокупности составляет 80 000 одновременно подключенных пользователей.
    
- Каким образом можно контролировать сферу применения, этические нормы и права доступа? Вы можете определить категории для разграничения прав и выбора пользователей, которые могут присутствовать в тех или иных комнатах.
    
- Как можно управлять правами на создание комнат? Вы можете определить пользователей, которые могут создавать комнаты. Авторы могут назначать других участников в качестве диспетчеров комнат чата для текущего управления комнатами.
    
- Как вы хотите создавать помещения? Сервер сохраняемого чата предоставляет веб-функцию для создания комнат и управления ими. Это можно запустить в клиенте Skype для бизнеса. Вы можете определить решение клиента, в котором реализованы бизнес-требования и рабочие процессы, и настроить сохраняемый сервер чата, чтобы направить пользователей в свое собственное решение.
    
- Какие типы надстроек вы хотите подготовить? Надстройки улучшают работу пользователей в комнате, используя панель расширений в клиенте Skype для бизнеса, на которой предоставляются возможности, имеющие отношение к данной комнате. Можно указать, какие общие надстройки будут наиболее полезными (например, веб-сайт компании, внутренние документы для совместной работы и т. д.). Диспетчеры комнат чата могут выбирать зарегистрированные надстройки и сопоставлять их с комнатами, если требуется. 
    
- Какие предъявляются требования к высокой доступности и аварийному восстановлению? Сервер сохраняемого чата поддерживает зеркальное отображение SQL Server и кластеризацию SQL Server для обеспечения высокой доступности. Для аварийного восстановления сервер сохраняемого чата поддерживает до 8 серверов (4 активных и 4 режимов) в растянутом пуле с доставкой журналов SQL Server. 
    
- Существуют ли нормативные требования? Если ваша компания находится в стране или регионе, где данные должны храниться в стране, вам может потребоваться развернуть несколько пулов серверов сохраняемого чата, каждый из которых должен быть указан в определенном географическом элементе. Комната, категория или надстройка не перемещается в пулы — она входит только в один пул серверов сохраняемого чата. 
    
    > [!NOTE]
    > Если несколько пулов серверов для работы с несколькими пользователями не дают вам больше масштаба (вы можете по-прежнему иметь только 80 000 пользователей одновременно для всех пулов серверов для вашего постоянного чата). Основная причина поддержки множества пулов серверов для чатов — обеспечение соответствия нормативным требованиям.
