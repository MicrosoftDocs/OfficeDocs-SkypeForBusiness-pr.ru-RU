# <a name="plan-for-persistent-chat-server-in-skype-for-business-server-2015"></a>Планирование сервера сохраняемой беседы в Skype для бизнеса Server 2015
 
**Сводка:** В этом разделе вы узнаете, как планировать сервер сохраняемой беседы в Skype для бизнеса Server 2015.
  
Сервер сохраняемой беседы — это необязательная роль, которая позволяет нескольким пользователям в организации участвовать в беседах комнат чата, которые сохраняются с течением времени. Хотя пользователи могут общаться в режиме реального времени во время сеанса чата, содержимое каждого сеанса (включая текст, ссылки и файлы) постоянно, что означает, что пользователи могут просматривать и искать все содержимое сеанса в любое время.
  
Сервер сохраняемой беседы помогает улучшить взаимодействие в организации с помощью:
  
- Расширение осведомленности и участия в работе организации
    
- Обеспечение эффективного общего доступа к информации 
    
- Улучшение взаимодействия между командами, включая географически распределенные и кроссфункциональные команды
    
- Уменьшение перегрузки информации
    
- Соблюдение нормативных требований путем необязательного развертывания службы соответствия сохраняемой беседы

> [!NOTE] 
> Сохраняемая беседа доступна в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения [см. в сведениях о том,](/microsoftteams/upgrade-start-here)как начать обновление Microsoft Teams. Если необходимо использовать сохраняемого чата, вы можете либо перенести пользователей, которым требуются эти функции, в Teams, либо продолжить использование Skype для бизнеса Server 2015. 
    
## <a name="persistent-chat-server-high-level-architecture"></a>Высокоуровневая архитектура сервера сохраняемого чата

На следующей схеме показано высокоуровневое представление архитектуры сервера сохраняемого чата. 
  
![Высокоуровневая архитектура сервера сохраняемого чата](../../media/0344f6e2-0c6d-4391-b4b3-ec31062b1576.png)
  
Сохраняемая беседа состоит из роли сервера переднего плана, которая предоставляет службы сохраняемого чата, а также серверной SQL базы данных. Как компоненты переднего, так и тыловой части включены в выделенный пул сохраняемого чата. Каждый компьютер, на котором размещен сервер сохраняемой беседы, должен иметь доступ к существующей топологии Skype для бизнеса Server 2015. На этой схеме имеется один пул серверов сохраняемого чата (A), который зависит от пула skype для бизнеса Server A для маршрутки сообщений в него.
  
Можно развернуть один или несколько пулов серверов сохраняемого чата, каждый из которых поддерживает до четырех активных серверов сохраняемого чата, поддерживающих до 80 000 одновременно действующих пользователей.
  
Skype для бизнеса Server 2015 взаимодействует со службой сохраняемого чата с помощью протокола SIP для регистрации и протокола XCCOS для чата. 
  
## <a name="persistent-chat-services"></a>Службы сохраняемой беседы

На следующей схеме показаны службы переднего сервера сохраняемого чата и то, как эти службы взаимодействуют с серверными компонентами базы данных. К компонентам переднего входа относятся службы сохраняемого чата и служба соответствия требованиям. К ним относятся хранилище сохраняемого чата и хранилище соответствия сохраняемого чата.
  
![Высокоуровневые службы сервера сохраняемого чата](../../media/bcdbadbe-e868-4a46-8a73-36562648fdf7.png)
  
### <a name="chat-service"></a>Служба чата

Служба чата, также называемая службой канала, является основной службой, ответственной за сервер сохраняемого чата. Служба чата предоставляет следующие функции:
  
- Принимает входящие сообщения
    
- Регистрация и перечисление онлайн-участников в комнате сохраняемой беседы
    
- Передает сообщения другим подписчикам канала
    
- Реализует логику управления каналами, приглашений в комнату чата, поиска и уведомлений о новом контенте
    
Служба сохраняемого чата сохраняет и получает доступ к содержимому комнаты чата и другим системным метаданным (правилам авторизации и так далее) с помощью хранилища сохраняемого чата. Служба сохраняет файлы, загруженные в комнаты чата, в хранилище файлов сохраняемой беседы.
  
### <a name="compliance-service"></a>Служба соответствия требованиям

Если в организации есть нормативы, которые требуют архивировать действия сохраняемой беседы, можно развернуть дополнительную службу соответствия сохраняемой беседы. Служба соответствия отвечает за архивное содержимое чата и события, такие как присоединение к комнатам и выход из них, в хранилище файлов соответствия сохраняемой беседы. Служба соответствия устанавливается на каждом сервере сохраняемой беседы в пуле сохраняемой беседы. 
  
### <a name="web-services"></a>Веб-службы

Веб-службы сохраняемой беседы работают на серверах переднего сервера Skype для бизнеса. Веб-службы зависят от служб IIS и реализованы в качестве веб-компонентов:
  
- Веб-службы сохраняемой беседы для отправки и загрузки файлов отвечают за публикацию и и загрузку файлов из комнат чата.
    
- Веб-службы сохраняемой беседы для управления комнатами чата отвечают за предоставление пользователям возможности управлять комнатами чата и создавать новые комнаты чата.
    
## <a name="defining-requirements-for-your-organization"></a>Определение требований для организации

Если вы решите развернуть сервер сохраняемой беседы, вам потребуется определить бизнес-требования организации, а затем определить топологию, инфраструктуру и технические требования для поддержки бизнес-потребностей. Чтобы оптимизировать развертывание, необходимо ответить на следующие вопросы:
  
- Перенос с предыдущей версии сервера группового чата или предыдущей версии сервера сохраняемого чата или развертывание сервера сохраняемого чата впервые?
    
- Кто может использовать сервер сохраняемой беседы? Политики сохраняемой беседы указываются для определения доступа пользователей на глобальном уровне, на уровне сайта или пользователя.
    
- Сколько пользователей потребуется доступ к серверу сохраняемой беседы? Сервер сохраняемого чата поддерживает 150 000 пользователей с поддержкой политики и не более 80 000 одновременно. Один сервер сохраняемого чата может поддерживать 20 000 подключенных пользователей, а один пул серверов сохраняемого чата может иметь до 4 активных серверов для 80 000 одновременно подключенных пользователей.
    
- Каким образом можно контролировать сферу применения, этические нормы и права доступа? Можно определить категории для разделения этих границ и выбрать, кто может быть в комнатах, созданных в каждой из этих категорий.
    
- Как можно ограничить права на создание комнат? Можно определить создателей, которые могут создавать комнаты. Авторы могут назначать других участников в качестве менеджеров комнат чата для непрерывного управления комнатами.
    
- Как вы хотите создавать комнаты? Сервер сохраняемого чата предоставляет веб-функцию для создания комнат и управления ними. Его можно запускать из клиента Skype для бизнеса. Вы можете выбрать решение клиента, которое реализует бизнес-требования и бизнес-процессы, а также настроить сервер сохраняемой беседы, чтобы направлять пользователей к вашему пользовательскому решению.
    
- Какие типы надстроек следует предоставлять? Надстройки улучшают качество работы в комнате, используя области расширения в клиенте Skype для бизнеса, чтобы предоставить контекст, релевантный для комнаты. Можно указать, какие общие надстройки будут наиболее полезными (например, веб-сайт компании, внутренние документы для совместной работы и т. д.). Менеджеры комнат чата могут выбирать зарегистрированные надстройки и сопоставлять их с комнатами. 
    
- Какие у вас есть требования к высокой доступности и аварийному восстановлению? Сервер сохраняемой беседы SQL Server зеркальное SQL Server кластеров для обеспечения высокой доступности. Для аварийного восстановления сервер сохраняемой беседы поддерживает до 8 серверов (4 активных и 4 в режиме ожидания) в растянутом пуле с SQL Server доставки журналов. 
    
- Существуют ли нормативные требования? Если ваша компания находится в стране или регионе, где данные должны храниться внутри страны, может потребоваться развернуть несколько пулов серверов сохраняемой беседы, каждый из которых находится в определенном географическом регионе. Комната, категория или надстройка не охватывают пулы— она принадлежит только одному пулу сервера сохраняемого чата. 
    
    > [!NOTE]
    > Наличие нескольких пулов серверов сохраняемого чата не дает дополнительных масштабов (во всех пулах серверов сохраняемого чата можно одновременно использовать только 80 000 пользователей). Основной причиной поддержки нескольких пулов серверов сохраняемой беседы является поддержка нормативных проблем.
