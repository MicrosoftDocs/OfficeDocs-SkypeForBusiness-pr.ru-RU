# <a name="active-directory-domain-services-for-skype-for-business-server"></a>Службы домена Active Directory для Skype для бизнеса Server
 
Служба доменов Active Directory выполняет функции службы каталогов для сетей Windows Server 2003, Windows Server 2008, Windows Server 2012 и Windows Server 2012 R2. Службы домена Active Directory также служат основой для Skype для бизнеса Server инфраструктуры безопасности. В этом разделе описывается, как Skype для бизнеса Server службы домена Active Directory для создания надежной среды для чата, веб-конференций, мультимедиа и голосовой связи. Дополнительные сведения о подготовке среды для служб домена Active Directory [см.](../../deploy/install/install.md) в Skype для бизнеса Server в документации по развертыванию. Сведения о роли служб домена Active Directory в сетях Windows Server см. в документации по используемой версии операционной системы.
  
Skype для бизнеса Server службы домена Active Directory для хранения:
  
- Требуются глобальные параметры, Skype для бизнеса Server серверы, работающие в лесу.
    
- Сведения о службе, определяя роли всех серверов, работающих Skype для бизнеса Server в лесу.
    
- Некоторые параметры пользователя.
    
## <a name="active-directory-infrastructure"></a>Инфраструктура Active Directory

Требования к инфраструктуре active Directory включают следующие:
  
- Требования к операционной системе для контроллеров домена
    
- Требования к функциональному уровню домена и леса
    
- Требования к домену глобального каталога
    
Подробные сведения см. в материале Экологические требования [Skype для бизнеса Server 2015](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) г. или требования [к серверу Skype для бизнеса Server 2019 г.](../../../SfBServer2019/plan/system-requirements.md)
  
## <a name="universal-groups"></a>Универсальные группы

Во время подготовки леса Skype для бизнеса Server универсальные группы в службах домена Active Directory, которые имеют разрешение на доступ к глобальным настройкам и службам и управлять ими. Эти универсальные группы включают:
  
- **Административные группы**. Эти группы определяют основные роли администратора для Skype для бизнеса Server сети. Во время подготовки леса эти группы администраторов добавляются в Skype для бизнеса Server группы инфраструктуры.
    
- **Группы служб.** Эти группы — это учетные записи служб, необходимые для доступа к различным службам, предоставляемым Skype для бизнеса Server.
    
- **Группы инфраструктуры.** Эти группы предоставляют разрешения на доступ к определенным областям Skype для бизнеса Server инфраструктуры. Они функционируют как компоненты административных групп, и не следует изменять их или добавлять пользователей в них напрямую. Во время подготовки леса в соответствующие группы инфраструктуры добавляются определенные группы служб и администрирования.
    
Дополнительные сведения о конкретных универсальных группах, созданных при подготовке AD для Skype для бизнеса Server, а также [](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-forest-preparation.md) группах служб и администрирования, добавленных в группы инфраструктуры, см. в документации по развертыванию изменения, внесенные подготовкой леса Skype для бизнеса Server в документации по развертыванию.
  
> [!NOTE]
> Skype для бизнеса Server поддерживает универсальные группы в Windows Server 2012, а также операционные системы Windows Server 2003 для контроллеров домена. Участниками универсальных групп могут быть другие группы и учетные записи из любого домена в дереве доменов или лесу и могут получать разрешения в любом домене в дереве доменов или лесу. Универсальная поддержка групп в сочетании с делегирования администратора упрощает управление развертыванием Skype для бизнеса Server. Например, не нужно добавлять один домен к другому, чтобы разрешить администратору управлять обоими. 
  
## <a name="role-based-access-control"></a>Управление доступом на основе ролей

Помимо создания универсальных групп обслуживания и администрирования и добавления групп служб и администрирования в соответствующие универсальные группы, подготовка леса также создает Role-Based управления доступом (RBAC). Подробные сведения о конкретных группах RBAC, созданных в ходе подготовки леса, см. в документации по развертыванию в Skype для бизнеса Server изменения, внесенные при подготовке леса. [](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-forest-preparation.md) Дополнительные сведения о группах RBAC см. в группе [Role-based access control (RBAC) для Skype для бизнеса Server.](role-based-access-control-rbac.md)
  
## <a name="access-control-entries-aces-and-inheritance"></a>Записи управления доступом (ACEs) и наследование

Подготовка леса создает как частные, так и общедоступные acEs и, добавляя acEs для универсальных групп, которые он создает. В контейнере глобальных параметров, используемом Skype для бизнеса Server, создаются специальные частные Skype для бизнеса Server. Этот контейнер используется только Skype для бизнеса Server и находится либо в контейнере Конфигурация, либо в контейнере System в корневом домене в зависимости от того, где хранятся глобальные параметры.
  
На этапе подготовки домена необходимые элементы управления доступом (ACE) добавляются в универсальные группы, которые предоставляют разрешения для размещения пользователей в домене и управления ими. При подготовке домена создаются элементы управления доступом в корне домена и три встроенных контейнера: пользователей, компьютеров и контроллеров домена.
  
Дополнительные сведения о общедоступных acEs, созданных и добавленных в процессе подготовки [](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-domain-preparation.md) леса и подготовки домена, см. в материале Изменения, внесенные подготовкой леса в [Skype для бизнеса Server](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-forest-preparation.md) и изменения, внесенные подготовкой домена в Skype для бизнеса Server документации по развертыванию.
  
Организации часто блокировать службы домена Active Directory (AD DS), чтобы снизить риски безопасности. Однако заблокированная среда Active Directory может ограничить разрешения, Skype для бизнеса Server требуется. Это может включать удаление acEs из контейнеров и OUs и отключение наследования разрешений на объектах User, Contact, InetOrgPerson или Computer. В заблокированной среде Active Directory разрешения должны устанавливаться вручную на контейнерах и OUs, которые требуют их.
  
## <a name="server-information"></a>Сведения о сервере

Во время активации Skype для бизнеса Server сведения о сервере в трех следующих расположениях служб домена Active Directory:
  
- Точка подключения к службе (SCP) на каждом объекте компьютера Active Directory, соответствующем физическому компьютеру, на котором Skype для бизнеса Server установлена система.
    
- Серверные объекты, созданные в контейнере **класса msRTCSIP-Pools.**
    
- Доверенные серверы, указанные в Topology Builder.
    
## <a name="service-connection-points"></a>Точки подключения к службе

Каждый Skype для бизнеса Server в службах домена Active Directory имеет SCP под названием RTC Services, который, в свою очередь, содержит ряд атрибутов, которые идентифицируют каждый компьютер и указывают службы, которые он предоставляет. К числу наиболее важных атрибутов SCP относится *службаDNSName,* *serviceDNSNameType,* *serviceClassname* и *serviceBindingInformation.* Сторонние приложения управления активами могут получать сведения о сервере в развертывании путем запроса на эти и другие атрибуты SCP.
  
## <a name="active-directory-server-objects"></a>Объекты Active Directory Server

Каждая Skype для бизнеса Server сервера имеет соответствующий объект Active Directory, атрибуты которого определяют службы, предоставляемые этой ролью. Кроме того, при активации выпуск Standard сервера или выпуск Enterprise пуле Skype для бизнеса Server создается новый объект **msRTCSIP-Pool** в контейнере **msRTCSIP-Pools.** Класс **msRTCSIP-Pool** указывает полное доменное имя (FQDN) пула, а также связь между компонентами переднего и заднего уровня пула. (Сервер выпуск Standard считается логическим пулом, передние и задние концы которого расположены на одном компьютере.)
  
## <a name="trusted-servers"></a>Доверенные серверы

В Skype для бизнеса Server доверенные серверы указаны при запуске Topology Builder и публикации топологии. Опубликованная топология, включая всю информацию о сервере, хранится в магазине Central Management. Доверяются только серверам, определенным в магазине Central Management. В Skype для бизнеса Server доверенный сервер отвечает следующим критериям:
  
- FQDN сервера происходит в топологии, хранимой в магазине Central Management.
    
- Сервер представляет действительный сертификат из доверенного ЦС. Подробные сведения см. в Skype для бизнеса Server [2015](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) г. или требования [к системе Skype для бизнеса Server 2019 г.](../../../SfBServer2019/plan/system-requirements.md)
    
Если один из этих критериев отсутствует, серверу не доверяют, и подключение к ним отказано. Это двойное требование предотвращает возможную, если маловероятно, атаку, в которой сервер-изгой пытается взять на себя FQDN допустимый сервер.
  
Кроме того, чтобы Microsoft Office серверов связи 2007 R2 и Microsoft Office Communications Server 2007 для связи с Skype для бизнеса Server серверами, Skype для бизнеса Server создает контейнеры в лесу подготовка к проведению списков доверенных серверов для предыдущих выпусков. В следующей таблице описываются контейнеры, созданные для обеспечения совместимости с предыдущими развертываниями.
  
**Списки доверенных серверов и их контейнеры Active Directory для совместимости с предыдущими выпусками**

|**Список доверенных серверов**|**Контейнер Active Directory**|
|:-----|:-----|
|выпуск Standard серверов и Enterprise пула передних конечных серверов  <br/> |RTC Service/Global Параметры  <br/> |
|Серверы conferencing  <br/> |RTC Service/Trusted MCUs  <br/> |
|Серверы веб-компонентов  <br/> |Служба RTC/TrustedWebComponentsServers  <br/> |
|Серверы-посредники и Communicator веб-серверы, сервер приложений, регистратор с QoE, A/V Conferencing Service (также серверы SIP 3rd-party)  <br/> |Служба RTC/Доверенные службы  <br/> |
|Прокси-серверы  <br/> |Skype для бизнеса Server не поддерживает обратную совместимость для прокси-серверов  <br/> |
