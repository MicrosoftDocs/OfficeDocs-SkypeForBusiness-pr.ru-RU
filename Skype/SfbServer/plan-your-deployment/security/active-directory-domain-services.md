# <a name="active-directory-domain-services-for-skype-for-business-server"></a>Доменные службы Active Directory для Скайп для Business Server
 
Доменные службы Active Directory работает как службы каталогов для Windows Server 2003, Windows Server 2008, Windows Server 2012 и Windows Server 2012 R2 сетей. Доменные службы Active Directory также выступает в качестве foundation, на котором создается Скайп для инфраструктуры безопасности Business Server. Этот раздел предназначен для описания как Скайп для Business Server использует доменных служб Active Directory для создания надежного среды обмена мгновенными Сообщениями, веб-конференций, мультимедиа и голосовой связи. Для получения дополнительных сведений о подготовке среды для доменных служб Active Directory см в документации по развертыванию [Установка Скайп для Business Server](../../deploy/install/install.md) . Для получения дополнительных сведений о роли доменных служб Active Directory в сетях Windows Server документации для версии операционной системы, которую вы используете.
  
Скайп для Business Server с помощью доменных служб Active Directory для хранения:
  
- Глобальные параметры, которые требуют всех серверов с Скайп для Business Server в лесу.
    
- Сведения о службе, идентифицирующее роли всех серверов с Скайп для Business Server в лесу.
    
- некоторые пользовательские параметры.
    
## <a name="active-directory-infrastructure"></a>Инфраструктуре Active Directory

Существуют следующие требования к инфраструктуре для Active Directory.
  
- требования к ОС для доменных контроллеров;
    
- требования для функциональных уровней домена и леса;
    
- требования для домена глобального каталога.
    
Дополнительные сведения см [окружающей среды требования к Скайп для Business Server 2015](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или [требования к серверу для Скайп для Business Server 2019](../../../SfBServer2019/plan/system-requirements.md).
  
## <a name="universal-groups"></a>Универсальные группы

Во время подготовки леса Скайп для Business Server создает различные универсальных групп в доменных службах Active Directory, которая имеет разрешение на доступ и управление глобальные параметры и службы. Универсальные группы делятся на следующие группы:
  
- **административные группы**. Эти группы определения ролей фундаментальные администратора для Скайп для сети Business Server. Во время подготовки леса этих групп администраторов добавляются Скайп для группы инфраструктуры Business Server.
    
- **группы обслуживания**. Эти группы — это учетные записи служб, которые необходимы для доступа к различных служб, предоставляемых Скайп для Business Server.
    
- **группы инфраструктур**. Эти группы предоставить разрешения на доступ к конкретной части Скайп для инфраструктуры Business Server. Они работают как компоненты административных групп; не следует изменять их или добавлять в них пользователей непосредственно. При подготовке леса определенные группы обслуживания и администрирования добавляются в соответствующие группы инфраструктур.
    
Для получения дополнительных сведений о конкретных универсальные группы, созданные при подготовке AD для Скайп Business Server, а также служб и административные группы, которые добавляются в группы инфраструктуры увидеть [изменений, внесенных путем подготовки леса в Скайп для бизнеса Сервер](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-forest-preparation.md) в документации по развертыванию.
  
> [!NOTE]
> Скайп для Business Server поддерживает универсальные группы в Windows Server 2012, а также ОС Windows Server 2003 для контроллеров домена. Members of universal groups can include other groups and accounts from any domain in the domain tree or forest and can be assigned permissions in any domain in the domain tree or forest. Поддержка универсальной группы, в сочетании с делегированием администратора упрощает управление Скайп для развертывания Business Server. For example, it is not necessary to add one domain to another to enable an administrator to manage both. 
  
## <a name="role-based-access-control"></a>Управление доступом на основе ролей

Кроме создания универсальных групп обслуживания и администрирования и добавления этих групп в соответствующие универсальные группы, подготовка леса также создает группы управления доступом на основе ролей (RBAC). Подробные сведения об определенных группах RBAC, созданных при подготовке леса, см. в разделе [Changes made by forest preparation in Skype for Business Server](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-forest-preparation.md) документации по развертыванию. Дополнительные сведения о группах RBAC [управления доступом на основе ролей (RBAC) для Скайп для Business Server](role-based-access-control-rbac.md)см.
  
## <a name="access-control-entries-aces-and-inheritance"></a>Записи управления доступом (ACE) и наследование

В процессе подготовки леса создаются частные и общедоступные записи ACE, а также создаются записи ACE универсальных групп при их добавлении. Он создает определенные частных элементов ACE для контейнера глобальные параметры, используемые Скайп для Business Server. Этот контейнер используется только Скайп для Business Server и расположен в контейнере конфигурации или контейнера System корневого домена, в зависимости от того, где хранятся глобальные параметры.
  
На этапе подготовки домена в универсальные группы добавляются необходимые записи управления доступом (ACE), которые предоставляют разрешения для размещения и управления пользователями в домене. При подготовке домена создаются записи ACE в корне домена и три встроенных контейнера: пользователи, компьютеры и контроллеры домена.
  
Подробные сведения о Общие элементы управления доступом создан и добавлен, подготовка леса и подготовка домена см [изменения, внесенные с подготовки леса в Скайп для Business Server](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-forest-preparation.md) и [изменений, внесенных подготовки домена в Скайп для Business Server](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-domain-preparation.md) Документация по развертыванию.
  
Организациях часто заблокируйте доменных служб Active Directory (AD DS) для снижения риска безопасности. Тем не менее заблокированной среде Active Directory можно ограничить разрешения, которые требуется Скайп для Business Server. Это может включать в себя записи ACE из контейнеров и подразделений, а также отключение разрешений наследования для объектов пользователя, контакта, InetOrgPerson или компьютеров. В заблокированной работу среды Active Directory, необходимо задать разрешения вручную контейнеры и подразделения, которые требуют их.
  
## <a name="server-information"></a>Информация о сервере

Во время активации Скайп для Business Server публикует сведения о сервере в трех следующих мест в доменных службах Active Directory:
  
- Подключение к службе точки подключения службы для каждого объекта Active Directory компьютера, соответствующий физического компьютера, на котором установлен Скайп для Business Server.
    
- объекты сервера, созданные в контейнере класса **msRTCSIP-Pools**;
    
- Доверенные серверы, указанных в построителе топологий.
    
## <a name="service-connection-points"></a>Точки подключения служб

Каждый Скайп для объекта Business Server в доменных службах Active Directory имеет точки подключения службы RTC-службами, который в свою очередь содержит несколько атрибутов, определяющих каждый компьютер и укажите службы, которые он содержит. Между более важных SCP атрибуты, *serviceDNSName* , *serviceDNSNameType* , *serviceClassname* и *serviceBindingInformation* . Сторонние приложения по управлению активами могут извлекать информацию о сервере в развертывании путем выдачи запросов по отношению этих и других атрибутов SCP.
  
## <a name="active-directory-server-objects"></a>Объекты сервера Active Directory

Каждый Скайп для роли сервера Business Server имеет соответствующие Active Directory объекта атрибуты которого определение службы, предоставляемые этой роли. Кроме того при активации сервера Standard Edition или при создании пула Enterprise Edition, Скайп для Business Server создает новый объект **msRTCSIP-Pool** в контейнере **MsRTCSIP пулов** . Класс **msRTCSIP-Pool ** указывает полное доменное имя пула и сопоставление между компонентами переднего плана и встроенными компонентами пула. (Сервер Standard Edition рассматривается как логическая пула которого заканчивается переднего и заднего должны размещаться совместно на одном компьютере).
  
## <a name="trusted-servers"></a>Доверенные серверы

В Скайп для Business Server доверенных серверов, указанных при выполнении Topology Builder и публикации топологии. Опубликованная топология, включая все сведения сервера, сохраняется в центральном хранилище управления. Только серверы, определенные в центральном хранилище управления, являются доверенными. В Скайп для Business Server доверенного сервера — это свойство, соответствуют следующим критериям:
  
- Полное доменное имя сервера встречается в топологии, сохраненной в центральном хранилище управления.
    
- Сервер предоставляет действительный сертификат от доверенного ЦС. Дополнительные сведения см [окружающей среды требования к Скайп для Business Server 2015](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или [требования к системе для Скайп для Business Server 2019](../../../SfBServer2019/plan/system-requirements.md).
    
При отсутствии любого из этих критериев сервер не является доверенным, а при подключении приходит отказ. Это требование двойная не позволяет это возможно, если вероятнее всего, не, атаки, в котором неавторизованный сервер пытается имеют перед полное доменное имя допустимого сервера.
  
Кроме того чтобы включить развертывание Microsoft Office Communications Server 2007 R2 и Microsoft Office Communications Server 2007 для связи с Скайп для серверов Business Server, Скайп для Business Server создает контейнеры во время подготовки леса для хранения списка доверенных серверов для предыдущих выпусков. В таблице ниже описаны контейнеры, созданные для обеспечения совместимости с предыдущими развертываниями.
  
**Надежные списки сервера и их контейнеры Active Directory для обеспечения совместимости с предыдущими версиями**

|**Список доверенных серверов**|**Контейнер Active Directory**|
|:-----|:-----|
|Серверы стандартного выпуска и серверы переднего плана пула Enterprise  <br/> |Служба RTC Service/глобальные параметры  <br/> |
|Серверы для конференций  <br/> |Служба RTC Service/доверенные MCU  <br/> |
|Серверы веб-компонентов  <br/> |Служба RTC Service/доверенные серверы веб-компонентов  <br/> |
|Серверы-посредники и серверы веб-клиента Communicator, сервер приложений, регистратор с качеством взаимодействия, служба аудио- и видеоконференции (а также сторонние серверы SIP)  <br/> |Служба RTC Service/доверенные службы  <br/> |
|Прокси-серверы  <br/> |Скайп для Business Server не поддерживает обратной совместимости для прокси-серверов  <br/> |
