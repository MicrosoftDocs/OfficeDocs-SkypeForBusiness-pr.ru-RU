# <a name="active-directory-domain-services-for-skype-for-business-server"></a>Доменные службы Active Directory для Skype для бизнеса Server
 
Доменные службы Active Directory функционируют как служба каталогов для сетей Windows Server 2003, Windows Server 2008, Windows Server 2012 и Windows Server 2012 R2. Доменные службы Active Directory также служат основой для построения инфраструктуры безопасности Skype для бизнеса Server. Цель этого раздела — описать, как в Skype для бизнеса Server используются доменные службы Active Directory для создания надежной среды для обмена сообщениями, веб-конференций, мультимедиа и голосовой связи. Подробнее о том, как подготовить среду для доменных служб Active Directory, можно найти в разделе [Установка сервера Skype для бизнеса](../../deploy/install/install.md) в документации по развертыванию. Подробные сведения о роли доменных служб Active Directory в сетях Windows Server можно найти в документации по используемой версии операционной системы.
  
Skype для бизнеса Server использует доменные службы Active Directory для хранения следующих параметров:
  
- Глобальные параметры, необходимые для всех серверов, работающих в Skype для бизнеса Server в лесу.
    
- Сведения о службе, определяющие роли всех серверов, работающих в Skype для бизнеса Server, в лесу.
    
- некоторые пользовательские параметры.
    
## <a name="active-directory-infrastructure"></a>Инфраструктура службы каталогов Active Directory

Требования к инфраструктуре для Active Directory включают следующее:
  
- требования к ОС для доменных контроллеров;
    
- требования для функциональных уровней домена и леса;
    
- требования для домена глобального каталога.
    
Дополнительные сведения можно найти в разделе [требования к окружающей среде для Skype для бизнеса server 2015](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или [серверных требований для Skype для бизнеса Server 2019](../../../SfBServer2019/plan/system-requirements.md).
  
## <a name="universal-groups"></a>Универсальные группы

Во время подготовки леса Skype для бизнеса Server создает в доменных службах Active Directory различные универсальные группы, у которых есть разрешение на доступ к глобальным параметрам и службам и управление ими. Универсальные группы делятся на следующие группы:
  
- **административные группы**. Эти группы определяют роли основных администраторов для сети Skype для бизнеса Server. Во время подготовки леса эти группы администраторов добавляются в группы инфраструктуры Skype для бизнеса Server.
    
- **группы обслуживания**. Эти группы представляют собой учетные записи служб, необходимые для доступа к различным службам, предоставляемым приложением Skype для бизнеса Server.
    
- **группы инфраструктур**. Эти группы предоставляют разрешения на доступ к определенным областям инфраструктуры Skype для бизнеса Server. Они работают как компоненты административных групп; не следует изменять их или добавлять в них пользователей непосредственно. При подготовке леса определенные группы обслуживания и администрирования добавляются в соответствующие группы инфраструктур.
    
Сведения о конкретных универсальных группах, созданных при подготовке рекламы для Skype для бизнеса Server, а также о группах обслуживания и администрирования, добавленных в группы инфраструктуры, приведены в разделе [изменения, внесенные с помощью подготовки леса в Skype для бизнеса. Сервер](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-forest-preparation.md) в документации по развертыванию.
  
> [!NOTE]
> Skype для бизнеса Server поддерживает универсальные группы в Windows Server 2012, а также операционные системы Windows Server 2003 для контроллеров домена. Members of universal groups can include other groups and accounts from any domain in the domain tree or forest and can be assigned permissions in any domain in the domain tree or forest. Поддержка универсальной группы в сочетании с делегированием администратора упрощает управление развертыванием сервера в Skype для бизнеса Server. For example, it is not necessary to add one domain to another to enable an administrator to manage both. 
  
## <a name="role-based-access-control"></a>Управление доступом на основе ролей

Кроме создания универсальных групп обслуживания и администрирования и добавления этих групп в соответствующие универсальные группы, подготовка леса также создает группы управления доступом на основе ролей (RBAC). Подробные сведения об определенных группах RBAC, созданных при подготовке леса, см. в разделе [Changes made by forest preparation in Skype for Business Server](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-forest-preparation.md) документации по развертыванию. Дополнительные сведения о группах RBAC можно найти в разделе [Управление доступом на основе ролей (RBAC) для Skype для бизнеса Server](role-based-access-control-rbac.md).
  
## <a name="access-control-entries-aces-and-inheritance"></a>Записи управления доступом (ACE) и наследование

В процессе подготовки леса создаются частные и общедоступные записи ACE, а также создаются записи ACE универсальных групп при их добавлении. Она создает определенные закрытые ACE в контейнере глобальных параметров, используемом в Skype для бизнеса Server. Этот контейнер используется только в Skype для бизнеса Server и находится в контейнере конфигурации или контейнере System в корневом домене (в зависимости от того, где хранятся глобальные параметры).
  
На этапе подготовки домена в универсальные группы добавляются необходимые записи управления доступом (ACE), которые предоставляют разрешения для размещения и управления пользователями в домене. При подготовке домена создаются записи ACE в корне домена и три встроенных контейнера: пользователи, компьютеры и контроллеры домена.
  
Подробные сведения об общедоступных ACE, созданных и добавленных с помощью подготовки леса и подготовки домена, приведены в разделе [изменения подготовки домена в Skype для бизнеса Server](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-forest-preparation.md) и [изменения, внесенные с помощью подготовки доменов в Skype для бизнеса Server](../../schema-reference/active-directory-schema-extensions-classes-and-attributes/changes-made-by-domain-preparation.md) Документация по развертыванию.
  
Организации часто блокируют доменные службы Active Directory (AD DS), чтобы снизить риски безопасности. Однако заблокированная служба Active Directory может ограничивать разрешения, необходимые для работы Skype для бизнеса Server. Это может включать в себя записи ACE из контейнеров и подразделений, а также отключение разрешений наследования для объектов пользователя, контакта, InetOrgPerson или компьютеров. В заблокированной среде Active Directory разрешения должны быть установлены вручную в контейнерах и подразделениях, в которых они необходимы.
  
## <a name="server-information"></a>Информация о сервере

Во время активации в Skype для бизнеса Server сведения о сервере публикуются в трех указанных ниже расположениях доменных служб Active Directory.
  
- Точка подключения службы (SCP) для каждого объекта компьютера Active Directory, соответствующего физическому компьютеру, на котором установлен Skype для бизнеса Server.
    
- объекты сервера, созданные в контейнере класса **msRTCSIP-Pools**;
    
- Доверенные серверы, указанные в построителе топологии.
    
## <a name="service-connection-points"></a>Точки подключения служб

Каждый серверный объект Skype для бизнеса в доменных службах Active Directory имеет точку обслуживания служб RTC, которая, в свою очередь, содержит несколько атрибутов, которые определяют каждый компьютер и указывают предоставляемые им службы. Среди наиболее важных атрибутов SCP — *сервицеднснаме* , *сервицеднснаметипе* , *сервицекласснаме* и *сервицебиндингинформатион* . Сторонние приложения по управлению активами могут извлекать информацию о сервере в развертывании путем выдачи запросов по отношению этих и других атрибутов SCP.
  
## <a name="active-directory-server-objects"></a>Объекты сервера службы каталогов Active Directory

Каждая роль сервера Skype для бизнеса Server имеет соответствующий объект Active Directory, атрибуты которого определяют службы, предоставленные этой ролью. Кроме того, при активации стандартного сервера выпуска или создании пула Enterprise Edition приложение Skype для бизнеса Server создаст новый объект **msRTCSIP** в контейнере **msRTCSIP-** Pools. Класс **msRTCSIP-Pool ** указывает полное доменное имя пула и сопоставление между компонентами переднего плана и встроенными компонентами пула. (Сервер Standard Edition рассматривается как логический пул, передний и задний концов которого выровнены на одном компьютере.)
  
## <a name="trusted-servers"></a>Доверенные серверы

В Skype для бизнеса Server при запуске Topology Builder и публикации топологии будут использоваться доверенные серверы. Опубликованная топология, включая все сведения сервера, сохраняется в центральном хранилище управления. Только серверы, определенные в центральном хранилище управления, являются доверенными. В Skype для бизнеса Server доверенный сервер — это один из указанных ниже критериев.
  
- Полное доменное имя сервера встречается в топологии, сохраненной в центральном хранилище управления.
    
- Сервер предоставляет действительный сертификат от доверенного ЦС. Дополнительные сведения можно найти в разделе [требования к окружающей среде для Skype для бизнеса server 2015](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или [требования к системе для Skype для бизнеса Server 2019](../../../SfBServer2019/plan/system-requirements.md).
    
При отсутствии любого из этих критериев сервер не является доверенным, а при подключении приходит отказ. Это двойное требование предотвращает возможную атаку, при которой фальшивый сервер попытается получить более полное доменное имя сервера.
  
Кроме того, чтобы включить Microsoft Office Communications Server 2007 R2 и Microsoft Office Communications Server 2007 для связи с серверными серверами Skype для бизнеса, в процессе подготовки леса в Skype для бизнеса Server создаются контейнеры. для хранения списков доверенных серверов для предыдущих выпусков. В таблице ниже описаны контейнеры, созданные для обеспечения совместимости с предыдущими развертываниями.
  
**Списки надежных серверов и их контейнеры Active Directory для обеспечения совместимости с предыдущими выпусками**

|**Список доверенных серверов**|**Контейнер Active Directory**|
|:-----|:-----|
|Серверы стандартного выпуска и серверы переднего плана пула Enterprise  <br/> |Служба RTC Service/глобальные параметры  <br/> |
|Серверы для конференций  <br/> |Служба RTC Service/доверенные MCU  <br/> |
|Серверы веб-компонентов  <br/> |Служба RTC Service/доверенные серверы веб-компонентов  <br/> |
|Серверы-посредники и серверы веб-клиента Communicator, сервер приложений, регистратор с качеством взаимодействия, служба аудио- и видеоконференции (а также сторонние серверы SIP)  <br/> |Служба RTC Service/доверенные службы  <br/> |
|Прокси-серверы  <br/> |Skype для бизнеса Server не поддерживает обратную совместимость с прокси-серверами  <br/> |
