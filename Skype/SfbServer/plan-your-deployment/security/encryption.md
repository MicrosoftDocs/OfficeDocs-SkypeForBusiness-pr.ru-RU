# <a name="encryption-for-skype-for-business-server"></a>Шифрование для Skype для бизнеса Server
 
Skype для бизнеса Server использует TLS и MTLS для шифрования мгновенных сообщений. Для всего трафика между серверами требуется MTLS независимо от того, ограничен ли трафик внутренней сетью или пересекает внутренний периметр сети. При подключении Skype для бизнеса Server к сторонним системам IPPBX или SIP-магистрали TLS является необязательным, но настоятельно рекомендуется использовать между сервером-посредником и медиа-шлюзом. Если для этой ссылки настроен TLS, требуется MTLS. Поэтому шлюз должен быть настроен с помощью сертификата из ЦС, которому доверяет сервер-посредник.
  
> [!NOTE]
> Рекомендации по безопасности, касающиеся SSL 3.0, были опубликованы в 2014 г. Отключение SSL 3.0 в Skype для бизнеса Server 2015 поддерживается. Дополнительные рекомендации по безопасности см. в подзаголовок "Отключение [SSL 3.0 в Lync Server 2013 и Skype для бизнеса Server 2015".](https://blogs.technet.microsoft.com/uclobby/2014/10/22/disabling-ssl-3-0-in-lync-server-2013/)<br/>
**Примечание о безопасности:** Чтобы обеспечить наиболее высокий уровень шифрования, Skype для бизнеса Server 2015 предлагает клиентам протоколы шифрования TLS в следующем порядке: **TLS 1.2, TLS 1.1, TLS 1.0.** TLS является критически важным аспектом Skype для бизнеса Server 2015, поэтому он необходим для поддержания поддерживаемой среды.<br/>
**Примечание о безопасности:** Чтобы обеспечить наиболее высокий уровень криптографического протокола, Skype для бизнеса Server 2019 будет предлагать клиентам протоколы шифрования TLS в следующем порядке: **TLS 1.3, TLS 1.2.** TLS является критически важным аспектом Skype для бизнеса Server 2019, поэтому он необходим для поддержания поддерживаемой среды.

## <a name="media-encryption"></a>Шифрование мультимедиа

Трафик мультимедиа шифруется с помощью протокола SRTP — профиля протокола Real-Time Transport Protocol (RTP), который обеспечивает конфиденциальность, проверку подлинности и защиту от атак с повтором для трафика RTP. Кроме того, носитель, проходящий в обоих направлениях между сервером-посредником и его внутренним следующим участком, также шифруется с использованием SRTP. Мультимедиа, которые потоком в обоих направлениях между сервером-посредником и медиа-шлюзом, при желании шифруются и рекомендуются. Сервер-посредник может поддерживать шифрование медиа-шлюза, но шлюз должен поддерживать MTLS и хранение сертификата.

## <a name="fips"></a>FIPS

Skype для бизнеса Server и Microsoft Exchange Server 2016 поддерживают алгоритмы FIPS 140-2, если операционные системы Windows Server настроены на использование алгоритмов FIPS 140-2 для шифрования системы. Чтобы реализовать поддержку FIPS, необходимо настроить поддержку каждого сервера Skype для бизнеса Server.
