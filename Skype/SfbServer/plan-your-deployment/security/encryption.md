# <a name="encryption-for-skype-for-business-server"></a>Шифрование для Skype для бизнеса Server
 
Skype для бизнеса Server использует TLS и MTLS для шифрования мгновенных сообщений. Для трафика от сервера к серверу требуется MTLS, независимо от того, ограничивается ли трафик внутренней сетью или пересекает внутренний периметр сети. При подключении сервера Skype для бизнеса к сторонним системам ИППБКС или магистральным магистрали SIP протокол TLS не является обязательным, но его настоятельно рекомендуется использовать между сервером-посредником и шлюзом мультимедиа. If TLS is configured on this link, MTLS is required. Таким образом, шлюз должен быть настроен на сертификат от центра сертификации, которому доверяет сервер-посредник.
  
> [!NOTE]
> Советы по безопасности в отношении SSL 3.0 были опубликованы в 2014 г. Отключение SSL 3,0 в Skype для бизнеса Server 2015 является поддерживаемым вариантом. Дополнительные сведения о рекомендациях по безопасности приведены [в статье Отключение SSL 3,0 в Lync server 2013 и Skype для бизнеса Server 2015](https://blogs.technet.microsoft.com/uclobby/2014/10/22/disabling-ssl-3-0-in-lync-server-2013/).<br/>
**Примечание о безопасности:** Для обеспечения использования наиболее надежного криптографических протоколов в Skype для бизнеса Server 2015 будут предлагаться протоколы TLS-шифрования в следующем порядке: **tls 1,2, tls 1,1, tls 1,0**. Протокол TLS является важнейшим аспектом использования Skype для бизнеса Server 2015, поэтому он необходим для поддержки поддерживаемой среды.<br/>
**Примечание о безопасности:** Для обеспечения использования наиболее надежного криптографических протоколов в Skype для бизнеса Server 2019 будут предлагаться протоколы TLS-шифрования в следующем порядке: **tls 1,3, tls 1,2**. Протокол TLS является важнейшим аспектом использования Skype для бизнеса Server 2019, поэтому он необходим для поддержки поддерживаемой среды.

## <a name="media-encryption"></a>Шифрование мультимедиа

Трафик мультимедиа шифруется по протоколу SRTP, профилю протокола RTP, который обеспечивает конфиденциальность, проверку подлинности и защиту от атак с повторением пакетов для трафика RTP. Кроме того, потоки мультимедиа в обоих направлениях между сервером-посредником и его внутренним узлом следующего прыжка также шифруются с помощью SRTP. Дополнительно выполняется (и рекомендуется) шифрование трафика мультимедиа в обоих направлениях между сервером-посредником и медиашлюзом. Сервер-посредник может обеспечивать шифрование на медиашлюзе, однако шлюз должен поддерживать MTLS и хранилище сертификатов.

## <a name="fips"></a>FIPS

Skype для бизнеса Server и Microsoft Exchange Server 2016 работают в алгоритмах, поддерживающих стандарт FIPS 140-2, если серверные операционные системы Windows настроены на использование алгоритмов FIPS 140-2 для системы криптографии . Для реализации поддержки FIPS необходимо настроить каждый сервер, на котором работает Skype для бизнеса Server, для его поддержки.
