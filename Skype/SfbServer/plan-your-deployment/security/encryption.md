# <a name="encryption-for-skype-for-business-server"></a>Шифрование для Skype для бизнес-сервера
 
Skype для бизнеса Server использует TLS и MTLS для шифрования мгновенных сообщений. Весь трафик от сервера к серверу требует MTLS, независимо от того, ограничивается ли трафик внутренней сетью или пересекает внутренний периметр сети. Подключение Skype для бизнеса Server к сторонним системам IPPBX или SIP-магистральм TLS необязательно, но настоятельно рекомендуется между сервером-посредником и шлюзом мультимедиа. Если TLS настроен по этой ссылке, требуется MTLS. Поэтому шлюз должен быть настроен с помощью сертификата от ЦС, которому доверяет сервер-посредник.
  
> [!NOTE]
> В 2014 г. был опубликован совет по безопасности в отношении SSL 3.0. Отключение SSL 3.0 в Skype для бизнеса Server 2015 является поддерживаемой опцией. Дополнительные новости о рекомендации по безопасности см. в рублях [Отключение SSL 3.0 в Lync Server 2013 и Skype for Business Server 2015.](/archive/blogs/uclobby/disabling-ssl-3-0-in-lync-server-2013)<br/>
**Примечание безопасности:** Чтобы обеспечить наиболее сильный криптографический протокол, Skype для бизнеса Server 2015 предложит клиентам протоколы шифрования TLS в следующем порядке: **TLS 1.2, TLS 1.1, TLS 1.0**. TLS является критически важным аспектом Skype для бизнеса Server 2015, поэтому он необходим для поддержания поддерживаемой среды.<br/>
**Примечание безопасности:** Чтобы обеспечить наиболее сильный криптографический протокол, Skype для бизнеса Server 2019 предложит клиентам протоколы шифрования TLS в следующем порядке: **TLS 1.3, TLS 1.2**. TLS является критически важным аспектом Skype для бизнеса Server 2019, поэтому он необходим для поддержания поддерживаемой среды.

## <a name="media-encryption"></a>Шифрование мультимедиа

Трафик мультимедиа шифруется с помощью secure RTP (SRTP), профиля протокола транспорта Real-Time (RTP), который обеспечивает конфиденциальность, проверку подлинности и защиту от атак повторно для трафика RTP. Кроме того, носитель, проходящий в обоих направлениях между сервером-посредником и его внутренним следующим участком, также шифруется с использованием SRTP. Носителю, течет в обоих направлениях между сервером-посредником и шлюзом мультимедиа, необязательно шифруется и рекомендуется. Сервер-посредник может поддерживать шифрование шлюза мультимедиа, но шлюз должен поддерживать MTLS и хранение сертификата.

## <a name="fips"></a>FIPS

Skype для бизнеса Server и Microsoft Exchange Server 2016 г. работают с поддержкой алгоритмов Федерального стандарта обработки информации (FIPS) 140-2, если операционные системы Windows Server настроены на использование алгоритмов FIPS 140-2 для системной криптографии. Чтобы реализовать поддержку FIPS, необходимо настроить каждый сервер под управлением Skype для бизнеса Server для его поддержки.
