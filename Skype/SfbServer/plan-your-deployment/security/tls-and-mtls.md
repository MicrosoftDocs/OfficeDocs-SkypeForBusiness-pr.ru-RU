# <a name="tls-and-mtls-for-skype-for-business-server"></a>TLS и MTLS для Skype для бизнеса Server
 
Протоколы TLS и MTLS обеспечивают шифрованную связь и проверку подлинности конечных точек в Интернете. Skype для бизнеса Server использует эти два протокола для создания сети доверенных серверов и обеспечения шифрования всех коммуникаций по этой сети. Все SIP-связи между серверами происходят через MTLS. SIP-связь между клиентом и сервером происходит по TLS.
  
TLS позволяет пользователям с помощью клиентского программного обеспечения проходить проверку подлинности серверов Skype для бизнеса Server, к которым они подключаются. В соединении TLS клиент запрашивает с сервера действительный сертификат. Чтобы сертификат был действительным, сертификат должен быть выдан ЦС, которому также доверяет клиент, а DNS-имя сервера должно соответствовать DNS-имени в сертификате. Если сертификат действителен, клиент использует открытый ключ в сертификате для шифрования симметричных ключей шифрования, используемых для связи, поэтому только исходный владелец сертификата может использовать его закрытый ключ для расшифровки содержимого связи. Итоговая связь является надежной и с этой точки не является проблемой для других доверенных серверов или клиентов. В этом контексте SSL, используемый с веб-службами, может быть связан как TLS.
  
Соединения между серверами используют MTLS для взаимной проверки подлинности. В подключении MTLS сервер, и исходящие сообщения, и сервер, принимающий его, обмениваются сертификатами из взаимно доверенного ЦС. Сертификаты подтверждают удостоверение каждого сервера на другом. В развертываниях Skype для бизнеса Server сертификаты, выдамые корпоративным ЦС в течение срока действия и не отозванные выдательным ЦС, автоматически считаются действительными всеми внутренними клиентами и серверами, так как все члены домена Active Directory доверяют корпоративному ЦС в этом домене. В федераированных сценариях оба федераированных партнера должны доверять выдаваемого ЦС. При желании каждый партнер может использовать другой ЦС, если этот ЦС также является доверенным для другого партнера. Это доверие проще всего сделать, если у edge-серверов есть сертификат корневого ЦС партнера в их доверенных корневых ЦС или сторонний ЦС, которому доверяют обе стороны.
  
TLS и MTLS помогают предотвратить перехват и атаки "человек в середине". В атаке "злоумышленник в середине" злоумышленник перенауляет связь между двумя сетевыми объектами через компьютер злоумышленника без ведома какой-либо из сторон. Спецификации TLS и Skype для бизнеса Server для доверенных серверов (только те, которые указаны в построитель топологий) частично снижает риск атаки "злоумышленник в середине" на уровне приложения, используя шифрование, координирующее шифрование открытого ключа между двумя конечными точками, и злоумышленнику необходимо иметь действительный и доверенный сертификат с соответствующим закрытым ключом и выданный имени службы, с которой клиент сообщает для расшифровки связи. Однако в конечном итоге необходимо следовать лучшим методикам безопасности с сетевой инфраструктурой (в данном случае корпоративной DNS). В Skype для бизнеса Server предполагается, что DNS-сервер является доверенным так же, как контроллеры домена и глобальные каталоги, но DNS обеспечивает уровень защиты от атак с перехватом DNS, не мешая серверу злоумышленника успешно отвечать на запрос подмененного имени.
