# <a name="common-security-threats-in-modern-day-computing"></a>Распространенные угрозы безопасности в современных вычислениях
 
Поскольку Skype для бизнеса Server является системой связи корпоративного класса, следует помнить о распространенных атаках безопасности, которые могут повлиять на ее инфраструктуру и коммуникации.
  
## <a name="compromised-key-attack"></a>Compromised-Key атака

Ключ — это секретный код или номер, используемый для шифрования, расшифровки или проверки секретных сведений. В инфраструктуре общедоступных ключей (PKI) используются два конфиденциальных ключа, которые необходимо учитывать: 
  
- Закрытый ключ, который есть у каждого владельца сертификата
    
- Ключ сеанса, используемый после успешного обмена ключами идентификации и сеанса партнерами по коммуникации
    
Атака с скомпрометированным ключом возникает, когда злоумышленник определяет закрытый ключ или ключ сеанса. Если злоумышленнику удалось определить ключ, злоумышленник может использовать ключ для расшифровки зашифрованных данных без ведома отправившего.
  
Skype для бизнеса Server использует функции PKI в операционной системе Windows Server для защиты ключевых данных, используемых для шифрования подключений безопасности транспортных слоев .TLS. Ключи, используемые для шифрования мультимедиа, обмениваются через TLS-соединения.
  
## <a name="network-denial-of-service-attack"></a>Атака на отказ в обслуживании сети

Атака отказа в обслуживании возникает, когда злоумышленник препятствует нормальному использованию и функционированию сети действительными пользователями. Это делается, когда злоумышленник заполоняет службу законными запросами, которые переполняют использование службы законными пользователями. С помощью атаки отказа в обслуживании злоумышленник может сделать следующее:
  
- Отправка недействительных данных приложениям и службам, работающим в атакуемой сети, чтобы нарушить их нормальную функцию.
    
- Отправка большого объема трафика, перегрузка системы до тех пор, пока она не прекратит отвечать или медленно реагирует на законные запросы.
    
- Скрыть данные о нападениях.
    
- Запретить пользователям доступ к сетевым ресурсам.
    
## <a name="eavesdropping-sniffing-snooping"></a>Подслушивание (sniffing, Snooping)

Подслушивание может происходить, когда злоумышленник получает доступ к пути данных в сети и имеет возможность отслеживать и читать трафик. Это также называется нюханием или снюханием. Если трафик находится в простом тексте, злоумышленник может читать трафик, когда злоумышленник получает доступ к пути. Пример — это атака, выполняемая путем управления маршрутизатором на пути данных. 
  
По умолчанию для трафика в Skype для бизнеса Server используется взаимный TLS (MTLS) между доверенными серверами и TLS от клиента к серверу. Эта защитная мера сделает атаку очень сложной или невозможной для достижения в течение периода времени, в течение которого происходит заданный разговор. Протокол TLS проверяет подлинность всех участников и шифрует весь трафик. Это не предотвращает подслушивание, но злоумышленник не может прочитать трафик, если шифрование не будет нарушено.
  
Протокол Traversal Using Relay NAT (TURN) не предписывает шифровать трафик, а сведения, которые он отправляет, защищены целостностью сообщений. Несмотря на то, что она открыта для прослушивания, отправляемая информация (то есть IP-адреса и порт) может быть извлечена напрямую, просто глядя на исходные и назначения пакетов. Служба A/V Edge гарантирует, что данные действительны, проверяя целостность сообщения с помощью ключа, полученного из нескольких элементов, включая пароль TURN, который никогда не отправляется четким текстом. Если используется безопасный протокол реального времени (SRTP), трафик мультимедиа также шифруется.
  
## <a name="identity-spoofing-ip-address-and-caller-id-spoofing"></a>Спуфинг удостоверений (IP-адрес и идентификатор вызывающего пользователя)

Спуфинг удостоверений возникает, когда злоумышленник определяет и использует номер телефона допустимого пользователя (идентификатор вызывающего пользователя) или IP-адрес сетевого, компьютерного или сетевого компонента без разрешения. Успешная атака позволяет злоумышленнику работать так, будто злоумышленник — это объект, который обычно идентифицирован по номеру телефона (номер вызывающего лица) или IP-адресу.

В контексте Skype для бизнеса Server, спуфинг IP-адресов вступает в игру только в том случае, если администратор сделал следующее:
  
- Настроенные подключения, поддерживаюные только протокол управления передачей (TCP) (который не рекомендуется, так как коммуникации TCP не расшифрованы).
    
- Отмечены IP-адреса этих подключений в качестве надежных хостов.
    
Это меньше проблем для подключений к безопасности транспортных слоев (TLS), так как TLS-кодирует все стороны и шифрует весь трафик. Использование TLS не позволяет злоумышленнику выполнять спуфинг IP-адресов на определенном подключении (например, взаимные подключения TLS). Но злоумышленник по-прежнему может подменить адрес DNS-сервера, который Skype для бизнеса Server использует. Однако, поскольку проверка подлинности в Skype для бизнеса выполняется с помощью сертификатов, злоумышленник не будет иметь действительный сертификат, необходимый для подмены одной из сторон в сообщении.

С другой стороны, спуфинг caller Id вступает в игру, когда вы установили магистраль SIP между поставщиком, шлюзом PSTN или другой системой PBX и Skype для бизнеса Server. В этих случаях Skype для бизнеса Server не предлагает никакой защиты для предотвращения спуфинга id вызываемого. Это означает, что Skype для бизнеса может получать вызов из магистрали SIP с поддельным именем звониста, отображающий номер телефона или отображаемого имени (если применяется обратный просмотр номера) другого Skype для бизнеса пользователя. Защита к этому должна применяться со стороны поставщика, шлюза PSTN или PBX.
  
## <a name="man-in-the-middle-attack"></a>Атака "Человек в центре"

Атака между злоумышленниками происходит, когда злоумышленник перенабивается связь между двумя пользователями через компьютер злоумышленника без ведома двух общих пользователей. Злоумышленник может отслеживать и читать трафик, прежде чем отправлять его получателю. Каждый пользователь в сообщении незнайно отправляет трафик злоумышленнику и получает его, думая, что он общается только с предполагаемым пользователем. Это может произойти, если злоумышленник может изменить службы домена Active Directory, чтобы добавить его или ее сервер в качестве доверенного сервера или изменить систему доменных имен (DNS), чтобы клиенты подключались через злоумышленника по пути к серверу. Атака "человек в середине" также может происходить с трафиком мультимедиа между двумя клиентами. Однако в Skype для бизнеса Server аудио, видео и совместном использовании приложений потоки шифруются с помощью SRTP с помощью криптографических ключей, которые согласовываются между однорангами, использующими протокол инициации сеансов (SIP) над TLS. Серверы, такие как групповой чат, используют HTTPS для повышения безопасности веб-трафика.
  
## <a name="rtp-replay-attack"></a>Атака воспроизведения RTP

Атака повтора возникает, когда допустимая передача мультимедиа между двумя сторонами перехвачена и перетранслирована в вредоносных целях. SRTP, используемая в связи с протоколом безопасной сигнализации, защищает передачи от атак повтора, позволяя приемнику поддерживать индекс уже полученных пакетов RTP и сравнивать каждый новый пакет с теми, которые уже указаны в индексе.
  
## <a name="spim"></a>Spim

Spim — это нежелательные коммерческие мгновенные сообщения или запросы на подписку на присутствие. Хотя сама по себе не является компромиссом сети, она, по крайней мере, раздражает, может снизить доступность ресурсов и производство и, возможно, приведет к компромиссу в сети. Пример этого — пользователи, которые шпионили друг за другом, отправляя запросы. Пользователи могут блокировать друг друга, чтобы предотвратить это, но с помощью федерации, если установлена скоординированная атака spim, это может быть трудно преодолеть, если вы не отключаете федерацию для партнера.
  
## <a name="viruses-and-worms"></a>Вирусы и черви

Вирус — это единица кода, целью которой является воспроизведение дополнительных аналогичных единиц кода. Для работы вирусу необходим хост, например файл, электронная почта или программа. Aworm — это единица кода, целью которой является воспроизведение дополнительных аналогичных единиц кода, но для этого не требуется хост. Вирусы и черви в основном показываются во время передачи файлов между клиентами или при отправлении URL-адресов от других пользователей. Если вирус находится на компьютере, он может, например, использовать удостоверение и отправлять мгновенные сообщения от вашего имени.
  
## <a name="personally-identifiable-information"></a>Личные сведения;

Skype для бизнеса Server может раскрывать сведения в общедоступных сетях, которые могут быть связаны с отдельными данными. Типы сведений можно разбить на две определенные категории:
  
- **Расширенные данные о присутствии** Расширенные данные присутствия — это сведения, которые пользователь может делиться или не делиться ссылкой на федератного партнера или контактов в организации. Эти данные не делятся с пользователями в общей сети мгновенных данных. Политики клиента и другая конфигурация клиентов могут поставить определенный контроль с системным администратором. В Skype для бизнеса Server режим конфиденциальности может быть настроен для отдельного пользователя, чтобы Skype для бизнеса пользователи, не в списке контактов пользователя, не видели сведения о присутствии пользователя. Расширенный режим конфиденциальности присутствия не мешает пользователям Microsoft Office Communicator 2007 и Microsoft Office Communicator 2007 R2 видеть сведения о присутствии пользователя. Дополнительные сведения о развертывании клиента и присутствия см. в Skype для бизнеса Server Deploy [clients for Skype для бизнеса Server](../../deploy/deploy-clients/deploy-clients.md) и Plan for instant [messaging and presence in Skype для бизнеса Server.](../../plan-your-deployment/instant-messaging-and-presence.md)
    
- **Обязательные данные** Обязательные данные необходимы для правильной работы сервера или клиента и не находятся под контролем клиента или системного администрирования. Это сведения, необходимые на уровне сервера или сети для маршрутизаживания, обслуживания состояния и сигнализации.
