# <a name="file-sharing-high-availability-in-skype-for-business-server"></a>Общий доступ к файлам с высокой доступностью в Skype для бизнеса Server
 
Узнайте о том, как обеспечить высокую доступность общих файловых ресурсов в Skype для бизнеса Server с помощью DFS.
  
Для обеспечения высокого уровня доступности общего доступа к файлам в среде развертывания Skype для бизнеса Server вы можете использовать распределенную файловую систему (DFS). DFS поддерживает отработку отказов с переключением на другой файловый сервер в пределах того же центра обработки данных. В случае крупномасштабного развертывания рекомендуется пользоваться выделенными файловыми серверами, объединенными в пары с помощью DFS. Дополнительные сведения о DFS в Windows Server 2012 можно найти в [https://go.microsoft.com/fwlink/?LinkId=524384](https://go.microsoft.com/fwlink/?LinkId=524384)разделе. Сведения о DFS в Windows Server 2008 можно найти в [https://go.microsoft.com/fwlink/p/?LinkId=524385](https://go.microsoft.com/fwlink/p/?LinkId=524385)разделе.
  
В зависимости от размера сети и объема устойчивости вы можете использовать одну пару серверов для размещения всех общих файловых ресурсов на сайте или использовать одну пару для каждого из них на передней стороне пула.
  
DFS предоставляет собой наилучший механизм репликации файлов без опубликованных обязательств по целевому времени восстановления (RTO) и целевой точке восстановления (RPO). Отработка отказа между серверами DFS должна выполняться быстро, но задержка репликации данных может препятствовать тому, что пользователи смогут продолжить работу, когда происходит переход на другой ресурс.
  
Если при работе с DFS на общем файловом ресурсе хранятся критически важные данные, следует создавать резервные копии общего файлового ресурса с высокой периодичностью, например каждые 4–8 часов. Если на момент отказа одного общего файлового ресурса данные репликации устарели, можно восстановить содержимое сбойного сервера на другом сервере, объединенном с ним в пару.
