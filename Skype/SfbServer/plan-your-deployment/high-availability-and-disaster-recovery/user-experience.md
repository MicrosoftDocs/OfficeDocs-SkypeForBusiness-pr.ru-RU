# <a name="user-experience-during-pool-failure-in-skype-for-business-server"></a>Пользовательский опыт во время сбоя пула в Skype для бизнеса Server
 
Узнайте о том, что пользователи испытывают при сбойе или сбойе пула front end во время аварийного восстановления в Skype для бизнеса Server.
  
В случае сбой пула все пользователи в затрагиваемом пуле вынуждены выйти и войти в пул резервного копирования. В течение непродолжительного периода времени пользователи, выполнившие вход в резервный пул, могут находиться в режиме устойчивости. В режиме отказоустойчивости пользователи не могут выполнять задачи, которые вызывают постоянные изменения в Skype для бизнеса Server, например добавление контакта. После завершения отработки отказа все пользователи могут использовать любые службы резервного пула.
  
Любые вызовы, собрания или беседы, которые пользователь имеет при сбойе пула, нарушаются, и пользователь должен восстановить эти сеансы после срыва, чтобы продолжиться.
  
В ходе отработки отказа или восстановления размещения пользователи не перемещаются. Пользователи, размещенные в сбойном пуле, временно обслуживаются резервным пулом. После восстановления домашнего пула администратор может не вернуть этих пользователей, которые будут обслужить их исходный пул, где они по-прежнему находятся дома.
  
Обратите внимание, что база данных сервера сведений о расположении не реплицируется в пул резервного копирования. Администратору рекомендуется регулярно создавать резервные копии базы данных сервера информирования о местонахождении и использовать последнюю резервную копию для восстановления базы данных сервера в резервном пуле после отработки отказа.
  
## <a name="user-experience-during-failover"></a>Пользовательский опыт при сбойе

Если пользователь находится в пуле, который выходит из строя, он выходит из строя. Любая одноранговая сессия, в которую принимал участие пользователь, завершена, равно как и конференции, организованные этим пользователем. Пользователь не сможет выполнить вход до тех пор, пока не истечет время ожидания таймера устойчивости Регистратора или администратор не начнет процедуры аварийного восстановления, независимо от того, что произойдет раньше. Следующий вход пользователя будет осуществляться на резервном пуле. Если вход выполнен до завершения отработки отказа, то пользователи будут находиться в режиме устойчивости до завершения отработки отказа. Только после этого пользователь может установить новые сеансы или восстановить предыдущие сеансы.
  
## <a name="user-experience-during-failback"></a>Пользовательский опыт при сбойе

Сбой пула может произойти, когда пострадавший пользователь вошел в пул резервного копирования, и пользователь остается в журнале и работает во время срабаты. Обратите внимание, что процесс сбойной работы занимает несколько минут. В качестве справки: отработка отказа для пула в 20000 пользователей занимает до 60 минут.

Термин затронутый пользователь относится к любому пользователю, который был переведен из основного пула и обслуживается резервным пулом. Пользователь, изначально означаемый в пуле резервного копирования, не является затронутым пользователем.
