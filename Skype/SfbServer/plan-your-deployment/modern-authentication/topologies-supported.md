# <a name="skype-for-business-topologies-supported-with-modern-authentication"></a>Топологии Skype для бизнеса, поддерживаемые современной проверкой подлинности

В этой статье перечислены локальные и локальные топологии, поддерживаемые с помощью современной проверки подлинности в Skype для бизнеса, а также функции безопасности, применимые к каждой топологии.

## <a name="modern-authentication-in-skype-for-business"></a>Современная проверка подлинности в Skype для бизнеса

Skype для бизнеса может использовать преимущества современной проверки подлинности. Так как Skype для бизнеса работает в тесном взаимодействии с Exchange, поведение входа для пользователей клиента Skype для бизнеса будет также зависеть от статуса MA Exchange. Это также применимо, если используется гибридная среда Skype для бизнеса с разделенным доменом. Это большое количество движущихся частей, но цель здесь — простой визуализация списка поддерживаемых топологий.

Какие топологии поддерживаются с помощью MA в Skype для бизнеса, Skype для бизнеса Online, Exchange Server и Exchange Online?

### <a name="supported-ma-topologies-in-skype-for-business"></a>Поддерживаемые топологии MA в Skype для бизнеса

Существует потенциально два серверных приложения и две рабочие нагрузки Office 365, которые относятся к топологии Skype для бизнеса, используемым в MA.

- Локальная среда Skype для бизнеса Server (CU 5)

- Skype для бизнеса Online (SFBO)

- Локальная среда Exchange Server

- Exchange Server Online (EXO)

Еще одна важная часть MA — знать, где будет выполняться проверка подлинности (определения) и авторизация (authZ) пользователей. Существует два варианта:

- Azure AD, в сети в Microsoft Cloud

- Локальный сервер федерации Active Directory (ADFS)

Поэтому он выглядит так же, как EXO и SFBO в облаке с Azure AD, а Exchange Server (курс) и Skype для бизнеса Server (SFB) в локальной среде.
