# <a name="skype-for-business-topologies-supported-with-modern-authentication"></a>Топологии Skype для бизнеса, поддерживаемые современной проверкой подлинности

В этой статье перечислены сетевые и локально-топологии, поддерживаемые современной проверкой подлинности в Skype для бизнеса, а также функции безопасности, применимые к каждой топологии.

## <a name="modern-authentication-in-skype-for-business"></a>Современная проверка подлинности в Skype для бизнеса

Skype для бизнеса может использовать преимущества безопасности современной проверки подлинности. Поскольку Skype для бизнеса тесно работает с Exchange, поведение входа Skype для бизнеса, которое увидят клиенты, также будет затронуто состоянием ma Exchange. Это также применимо, если у Skype для бизнеса гибрид с раздельным доменом. Это много движущихся частей, но цель здесь проста в визуализации списка поддерживаемых топологий.

Учитывая Skype для бизнеса, Skype для бизнеса, Exchange Server и Exchange в Интернете, какие топологии поддерживаются с помощью ma?

### <a name="supported-ma-topologies-in-skype-for-business"></a>Поддерживаемые топологии MA в Skype для бизнеса

Потенциально существует два серверных приложения и две Microsoft 365 или Office 365, связанные с Skype для бизнеса топологиями, используемыми ma.

- Skype для бизнеса (CU 5) на локальном сервере

- Skype для бизнеса (SFBO)

- Exchange локальном сервере

- Exchange сервер в Интернете (EXO)

Другой важной частью ma является знание, где будет проходить проверка подлинности (authN) и авторизация (authZ) пользователей. Два варианта:

- Azure AD, онлайн в облаке Microsoft

- Сервер Федерации Active Directory (ADFS) на локальной основе

Так выглядит немного так: EXO и SFBO в облаке с Azure AD, а также Exchange Server (EXCH) и Skype для бизнеса сервера (SFB) на преме.

- Если значок затененный или серый, он не используется в сценарии.

- EXO Exchange Online.

- SFBO является Skype для бизнеса Online.

- EXCH Exchange локально.

- SFB Skype для бизнеса локально.

- Авторизованные серверы представлены треугольниками, например, Azure AD — это треугольник с облаком за ним.

- Стрелки указывают на авторизованный сервер, который будет использоваться при попытке клиентов достичь указанного ресурса сервера.

Сначала рассмотрим ma с помощью Skype для бизнеса в локальной или облачной топологиях.
