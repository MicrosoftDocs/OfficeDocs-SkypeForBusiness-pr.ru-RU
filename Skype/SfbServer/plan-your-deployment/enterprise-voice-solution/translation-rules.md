# <a name="translation-rules-in-skype-for-business-server"></a>Правила трансляции в Skype для бизнеса Server

Узнайте о правилах трансляции и нормализации строк набора номера в Skype для бизнеса Server Корпоративная голосовая связь.

 Корпоративная голосовая связь необходимо, чтобы все строки набора были нормализованы в формат E.164 для выполнения обратного подбирания номеров (RNL). Правила трансляции поддерживаются как для вызываемого, так и для вызываемого номера. Для одноранговой магистрали (т. е. связанного шлюза, УАЧС или магистрали SIP) может потребоваться, чтобы номера были в локальном формате набора номера. Чтобы преобразовать номера из формата E.164 в формат местного набора, можно определить правила преобразования для работы с URI запроса перед его направлением в магистральный узел. Например, можно написать правило преобразования, чтобы заменить +44 в начале строки набора на 0144.

Выполняя преобразование исходящего маршрута на сервере, можно уменьшить требования к конфигурации в каждом отдельном магистральном узле для трансляции телефонных номеров в формат местного набора. При планировании того, какие шлюзы и сколько шлюзов следует связать с конкретным кластером сервера-посредника, может быть полезно сгруппировать одноранговые магистрали с аналогичными локальными требованиями набора номера. Это позволяет уменьшить необходимое количество правил преобразования и сэкономить время, затрачиваемое на их создание.

> [!IMPORTANT]
> Связывание одного или более правил трансляции с конфигурацией магистрали Корпоративная голосовая связь в качестве альтернативы настройке правил трансляции на магистральной магистрали. Не связывать правила трансляции с Корпоративная голосовая связь магистрали, если вы настроили правила трансляции на магистрали, так как эти два правила могут конфликтовировать.
