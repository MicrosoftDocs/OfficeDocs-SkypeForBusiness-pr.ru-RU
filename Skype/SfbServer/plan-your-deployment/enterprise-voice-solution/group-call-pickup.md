# <a name="plan-for-group-call-pickup-in-skype-for-business"></a>Планирование группового вызова в Skype для бизнеса
 
Планирование группового вызова в Skype для бизнеса Server Корпоративная голосовая связь, что позволяет пользователям отвечать на вызовы, изначально предназначенные для других.
  
Групповые вызовы позволяют пользователям отвечать на входящие вызовы своим коллегам с собственных телефонов. Это повышает доступность пользовательской строки, позволяя другим пользователям отвечать на входящий вызов, набирая номер группы вызова. При развертывании группового вызова количество входящих вызовов, которые направлены на голосовую почту, может быть значительно уменьшено, что особенно полезно для звонков клиентов, внешних для вашей организации.
  
Функция группового вызова предназначена, в частности, для бизнес-подразделений в открытых средах офиса. Входящие вызовы не являются разрушительными, так как звонят только по назначению. Однако другие пользователи, которые слышат звонок, по-прежнему могут получить вызов, просто набрав номер группы. 
  
В средах, где пользователи не находятся в макете открытого офиса или где географически распределены пользователи, которые разделяют общую ответственность, наиболее подходящим решением является вызов группы. Основное различие между групповым вызовом и групповым вызовом заключается в том, что при наборе групповых вызовов входящие звонки звонят только по назначению, но любой может ответить на него, набрав номер группы. При вызове группы звонки звонят на всех телефонах членов группы, и любой пользователь в команде может взять трубку, чтобы ответить на вызов. Дополнительным отличием между групповым вызовом и вызовом группы является то, что сбор групповых вызовов управляется администратором через Skype для бизнеса Server. При вызове группы конечные пользователи управляют этой функцией с помощью Skype для бизнеса клиента. Таким образом, при групповом вызове этот аспект управления вызовами можно централизовать.
  
Прием групповых вызовов построен на приложении Call Park. При развертывании группового вызова настраивается таблица орбиты парка вызовов с отдельными диапазонами номеров расширений, назначенных в качестве номеров групп вызова. Как и номера орбиты парка вызовов, номера групп вызова должны быть виртуальными расширениями, которые не имеют пользователя или телефона, назначенного им. Каждый пул переднего конца, в котором развертывается групповой вызов, может иметь один или несколько диапазонов номеров групп вызова. Диапазоны номеров групп должны быть глобально уникальными для Skype для бизнеса Server развертывания. 
  
> [!NOTE]
> Диапазоны номеров, назначенные в качестве номеров группового вызова в таблице орбиты парка вызовов, не могут управляться или просматриваться с помощью панели управления Skype для бизнеса Server. Единственный способ увидеть все диапазоны номеров в таблице орбиты парка вызовов — это использовать Skype для бизнеса Server управленческой оболочки. Кроме того, единственным способом добавления, изменения или удаления номеров группового вызова является использование Skype для бизнеса Server управленческой оболочки. 
  
После настройки групповых номеров группы вызова вы назначаете пользователям группу вызова. Любой пользователь, которому назначена группа вызова, может отвечать на их звонки другими пользователями. При вызове пользователя, назначенного группе вызова, любой другой пользователь, который заметит вызов, может ответить на него вручную, набрав номер группы вызова. Пользователь, который выбирает вызов, не должен быть членом группы. При вызове другого пользователя уведомление отправляется на первоначально вызываемый номер.
  
> [!NOTE]
> Пользователь может быть членом только одной группы вызова. 
  
> [!NOTE]
> Хотя любой пользователь в развертывании Skype для бизнеса Server может ответить на вызов участнику группы вызова, отвечающий на вызов должен знать правильный номер группы вызова. 
  
Если пользователь набирает номер группы вызова, чтобы ответить на вызов, когда звонят несколько телефонов в группе, пользователь отвечает на звонок, который звонил наиболее долго.
  
Параметры одновременного звонка будут работать для пользователей, у которых есть групповой вызов. То есть звонок, сделанный пользователю с групповым вызовом, будет звонить для всех настроенных направлений, и другой пользователь может ответить на вызов. Исключением из этого правила является то, что пользователь настраивает одновременный звон, чтобы вызвать всех участников группы.
  
Для ответа на следующие типы вызовов нельзя использовать групповой вызов:
  
- Вызовы на личную линию
    
- Вызовы от контакта, которому назначены отношения конфиденциальности "Друзья и семья"
    
    > [!TIP]
    > Пользователь, который входит в группу вызова, может предотвратить извлечение определенных вызовов с помощью группового вызова, пометив контакт как личный контакт в клиенте Skype для бизнеса клиента. Чтобы отметить контакт в качестве личного контакта, установите связь конфиденциальности для контакта с друзьями и семьей. Любой входящий вызов из контактов с набором конфиденциальности "Друзья и семья" невозможно получить с помощью группового вызова. 
  
- Видео-часть аудио- и видеозвонков 
    
    > [!NOTE]
    > Если пользователь отвечает на аудио- и видеозвоки, он получает только звук. Вызов, вызывающий или отвечающий на вызов, может привести к эскалации вызова для добавления видео. 
  
- Одновременные вызовы вызова, которые будут маршрутизовы для участников командных звонков
    
- Вызовы, переданные делегату
    
- Вызовы, перена маршрутизовав группу отклика
    
Следующие типы пользователей не могут участвовать в групповом вызове. То есть они не должны включаться в группу группового вызова и не могут вызывать пользователей, у которых включена возможность группового вызова.
  
- Пользователи, которые размещены в Интернете в гибридном развертывании
    
- Пользователи, которые не размещены в пуле Skype для бизнеса Server 2015 г. или пуле Lync Server 2013 с накопительными обновлениями для Lync Server 2013: февраль 2013 г. в локальном развертывании
    
Если никто не отвечает на звонок члену группы вызова, вызов передается по указанному в параметрах клиента. То есть вызов отправляется на голосовую почту или отправляется в другой пункт назначения, как указано в параметрах клиента.
  
## <a name="deployment-and-requirements"></a>Развертывание и требования

Групповые вызовы автоматически развертываются при развертывании Корпоративная голосовая связь и приложения Call Park. Вы включаете групповую группу вызовов, настроив таблицу орбиты Call Park с отдельными диапазонами номеров, назначенных в качестве номеров групп вызова, а затем назначив пользователям вызывать группы и включить пользователей для группового вызова.
  
## <a name="clients-supported-for-group-call-pickup"></a>Клиенты, поддерживаемые для группового вызова

Любой из следующих клиентов можно использовать для ответа на вызовы для участников группового вызова:
  
- Skype для бизнеса
    
- Lync 2013
    
- Lync 2010
    
- Lync Phone Edition
    
> [!NOTE]
> Пользователи могут использовать любой из этих клиентов для ответа на вызовы для участников группового вызова, но они должны быть окочены в пуле Skype для бизнеса Server или пуле Lync Server 2013 с накопительными обновлениями для Lync Server 2013: февраль 2013 г. 
  
Следующие клиенты и устройства не поддерживаются для подбирать вызовы для участников группового вызова:
  
- Lync Mobile
    
- Приложение Lync для Windows 8 и Windows RT
    
- Lync для iPad
    
- Аналоговые телефоны
    
- Телефоны с номерами общедоступных переключеных телефонов (PSTN)
    
## <a name="capacity-planning"></a>Планирование мощности

В следующей таблице описывается пользовательская модель группового вызова, которую можно использовать в качестве основы для требований к планированию емкости.
  
> [!IMPORTANT]
> Выбор группового вызова основан на приложении Call Park. Помните, что для планирования аварийного восстановления емкости каждый пул парного пула должен иметь возможность обрабатывать рабочие нагрузки для служб Call Park, в том числе группового вызова, в обоих пулах. 
  
**Модель пользователя группового вызова**

|**Метрика**|**Для каждого пула переднего конца  <br/>  (с 8 серверами переднего конца)**|**Для каждого сервера Standard Edition**|
|:-----|:-----|:-----|
|Рекомендуемое число пользователей в группе  <br/> |50  <br/> |50  <br/> |
|Рекомендуемое количество групп  <br/> |500  <br/> |60  <br/> |
|Максимальное число пользователей в пуле, включенного для группового вызова  <br/> |25 000   <br/> |3,000  <br/> |
|Максимальная скорость входящих вызовов для общего числа пользователей, включенных для группового вызова в пул в минуту  <br/> |500  <br/> |60  <br/> |
|Максимальная скорость вызовов, полученных пользователями с помощью группового вызова в пул в минуту  <br/> |200  <br/> |25  <br/> |
   
> [!NOTE]
> Для пулов переднего конечного конца, которые имеют менее восьми серверов переднего конечного сервера, вычислять метрик линейно. Например, если в переднем пуле имеется один передний конечный сервер, высчитайте максимальную нагрузку в 1/8 значений, показанных в таблице. 
  
> [!NOTE]
> Вы можете увеличить или уменьшить рекомендуемое число пользователей на одну группу и количество групп до тех пор, пока не превысите максимальное число пользователей в пуле. Например, на сервере выпуск Standard может быть 120 групп с 25 пользователями на одну группу, так как число пользователей, включенных для группового вызова, по-прежнему находится в пределах максимальной модели пользователя (то есть 120 групп раз 25 пользователей — это 3 000 пользователей, включенных для группового вызова).
