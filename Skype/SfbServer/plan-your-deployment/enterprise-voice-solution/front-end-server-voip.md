# <a name="front-end-server-voip-components-for-skype-for-business-server"></a>Компоненты VoIP сервера переднего сервера для Skype для бизнеса Server

Узнайте о компонентах Корпоративная голосовая связь, расположенных на серверах переднего сервера в Skype для бизнеса Server, включая службу перевода и различные компоненты маршрутки.

Компоненты VoIP, расположенные на серверах переднего сервера:

- служба перевода;

- компонент маршрутизации входящих вызовов;

- компонент маршрутизации исходящих вызовов;

- компонент маршрутизации единой системы обмена сообщениями Exchange;

- компонент маршрутизации между кластерами.

- [Компонент сервера-посредника в Skype для бизнеса Server](mediation-server.md)

## <a name="translation-service"></a>Служба перевода

Служба перевода — это серверный компонент, который отвечает за преобразование набираемого номера в формат E.164 или другой формат согласно правилам нормализации, определенным администратором. Служба перевода может преобразовывать номера и в другие форматы, не только E.164, если ваша организация использует частную систему нумерации или шлюз или УАТС, которые не поддерживают формат E.164.

## <a name="inbound-routing-component"></a>Компонент маршрутизации входящих пакетов

Компонент маршрутизов входящих вызовов обрабатывает входящие вызовы в основном в соответствии с предпочтениями, заданными пользователями в своих Корпоративная голосовая связь клиентах. Он также осуществляет делегированные и одновременные звонки, если они настроены пользователем. Например, пользователи указывают, переадресуются ли неотвеченные вызовы или просто записываются в журнал. Если переадстройка звонков включена, пользователи могут указать, следует ли перенаададировать неотвеченные вызовы на другой номер или на сервер exchange UM, настроенный для предоставления ответа на вызовы. Компонент маршрутизации входящий почты устанавливается по умолчанию на всех серверах Standard Edition и серверах переднего уровня.

## <a name="outbound-routing-component"></a>Компонент маршрутизации исходящих вызовов

Компонент маршрутизации исходящих вызовов направляет вызовы на УАТС или ТСОП. Он применяет правила авторизации вызовов, определенные политикой голосовой почты пользователя, к вызывателям и определяет оптимальный шлюз STN для маршрутизации каждого вызова. Компонент маршрутизации исходящие компоненты устанавливаются по умолчанию на всех серверах Standard Edition и серверах переднего сервера.

Логика маршрутизации, используемая компонентом маршрутизации исходящих вызовов, в значительной степени настраивается сетью или администраторами телефонной связи в соответствии с требованиями организации.

## <a name="exchange-um-routing-component"></a>Компонент маршрутизации единой системы обмена сообщениями Exchange

Компонент маршрутки единой системы обмена сообщениями Exchange обрабатывает маршрутику между Skype для бизнеса Server и серверами единой системы обмена сообщениями Exchange для интеграции Skype для бизнеса Server с функциями единой системы обмена сообщениями.

Компонент маршрутирования exchange UM также обрабатывает перенаправляемую маршрутику голосовой почты через PSTN, если серверы этой системы Exchange недоступны. Если у Корпоративная голосовая связь есть пользователи на сайтах филиалов, которые не имеют устойчивого соединения WAN с центральным сайтом, устройство для обеспечения связи в филиалах, развернутое на сайте филиала, обеспечивает поддержку голосовой почты для пользователей филиалов во время отключения сети WAN. При нарушении WAN-соединения для обеспечения связи в филиалах выполняется следующее:

- переадресует неотвеченные вызовы по ТСОП на сервер единой системы обмена сообщениями Exchange в центральном узле;

- обеспечивает пользователю возможность получения сообщений голосовой почты через ТСОП;

- ставит в очередь уведомления о пропущенных звонках, а затем загружает их на сервер единой системы обмена сообщениями Exchange при восстановлении канала глобальной сети.

Чтобы включить перенаправление голосовой почты, администратору Exchange рекомендуется настроить автосекретарь Exchange um (AA) только на прием сообщений.

Дополнительные сведения об этих возможностях см. в разделах [On-Premises Exchange Unified Messaging Integration](https://technet.microsoft.com/library/e7c63a71-2d99-4aa9-b649-36c1a431bdf1.aspx) и [Planning for Enterprise Voice Resiliency](https://technet.microsoft.com/library/ca116700-1055-4ca5-9b87-4c7f380c3655.aspx) соответственно.

## <a name="intercluster-routing-component"></a>Компонент маршрутизации между кластерами

Компонент маршрутной маршрутки между компонентами отвечает за маршрутику вызовов в основной пул регистратора вызываемой системы. Если этот компонент недоступен, компонент перенананошет вызов в резервный пул регистратора вызываемого вызываемого. Если основной и резервный пулы регистратора вызываемого вызываемого пользователя недоступны по IP-сети, компонент маршрутации между компонентами перенастройки перенаносит вызов по ТСТС на номер телефона пользователя.

## <a name="other-front-end-server-components-required-for-voip"></a>Другие компоненты интерфейсных серверов, необходимые для VoIP

К другим компонентам, которые находятся на сервере переднего сервера или директоре, которые обеспечивают необходимую поддержку VoIP, но сами не являются компонентами VoIP, относятся следующие:

- **Пользовательские службы.** Выполняют поиск обратного номера для целевого телефонного номера каждого входящего вызова и сопоставляют этот номер с SIP URI конечного пользователя. Используя эти сведения, компонент маршрутизов входящие вызовы распространяется на зарегистрированные конечные точки SIP этого пользователя. Службы пользователей — это основной компонент на всех серверах переднего интерфейса и директорах.

- **Репликатор пользователей.** Извлекает номера телефонов пользователей из доменных служб Active Directory и записывает их в таблицы базы данных RTC, где они доступны для служб пользователей и сервера адресной книги. Репликатор пользователей — это основной компонент на всех серверах переднего интерфейса.

- **Сервер адресной книги.** Предоставляет сведения о глобальном списке адресов из доменных служб Active Directory клиентам Skype для бизнеса Server. Кроме того, он извлекает сведения о пользователях и контактах из базы данных RTC, записывает их в файлы адресной книги, а затем сохраняет файлы в общей папке, в которую их скачивает клиент Skype для бизнеса. Сервер адресной книги записывает сведения в базу данных RTCAb, которая используется службой веб-запросов адресной книги для ответа на поисковые запросы пользователей из Skype для бизнеса mobile. При желании он нормализует корпоративные номера телефонов пользователей, которые записаны в базу данных RTC для предоставления контактов пользователей в Skype для бизнеса. Служба адресной книги устанавливается по умолчанию на всех серверах переднего сервера. По умолчанию служба веб-запросов адресной книги устанавливается вместе с веб-службами на каждом сервере переднего плана.
