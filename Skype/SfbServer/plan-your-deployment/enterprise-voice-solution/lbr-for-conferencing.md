# <a name="location-based-routing-for-conferencing-in-skype-for-business-server"></a>Location-Based маршруты для conferencing в Skype для бизнеса Server

Планирование маршрутации на основе расположения для проведения в Skype для бизнеса Server Корпоративная голосовая связь, включая переключение вызовов с консультацией.

Location-Based маршрутизов позволяет ограничить маршрутику вызовов между конечными точками VoIP и конечными точками STN в зависимости от расположения сторон в вызове. Location-Based маршрутизирование для конференц-конференций позволяет применять правила Location-Based маршрутов для собраний (то есть конференций), чтобы запретить обход платных платы PSTN. Приложение отслеживает активную конференцию и применяет Location-Based маршрутов в зависимости от расположения участвующих пользователей. Маршрут Location-Based для приложения для работы с conferencing также позволяет обеспечить применение ограничений Location-Based маршрутов для передачи консультаций с использованием конечных точек STN.

Приложение Location-Based конференц-залов маршрутов предоставляет конференциям Skype для бизнеса механизм предотвращения обхода платных звонков ЗВОНКОВ. Приложение отслеживает активные конференции и применяет Location-Based маршрутов в зависимости от расположения пользователей Skype для бизнеса.

Приложение Location-Based для проведения Location-Based маршрутов определяет, будет ли применяться Location-Based к собранию Skype для бизнеса при следующих критериях:

- Организатор собрания включен для Location-Based маршрутов. Location-Based маршрутов будут применяться только к конференциям, организованным пользователями с включенной Location-Based маршрутизов.

- По крайней мере один участник собрания является конечной точкой STN. Location-Based маршрутов применяются только к конференциям, включаемым конечные точки STN.

- Сетевой сайт, на котором шлюз STN, используемый для подключения конференции к STN, расположен, а также сетевые сайты, из которых подключаются организаторы и участники.

The Location-Based Routing for Conferencing application prevents the participation of Skype for Business users and PSTN endpoints from different network sites to the same conference. Если организатор собрания включен для Location-Based, приложение для конференций применяет следующие ограничения:

- Конечные точки, которые могут присоединиться к собранию Skype для бизнеса, зависят от конечных точек, которые уже присоединились к конференции, и это ограничение регулируется по мере того, как присоединились конечные точки и новые конечные точки присоединяются к конференции. Если организаторы и участники присоединяются к собранию Skype для бизнеса с того же сетевого сайта, конечная точка STN, другой участник из того же сетевого сайта, другой участник из другого сетевого сайта или участник из неизвестного сетевого сайта могут присоединиться.

- Если организаторы и участники присоединяются к собранию с разных или неизвестных сетевых сайтов, конечная точка STN не может присоединиться к собранию, если вызов STN впадает из магистрали SIP с включенной Location-Based маршрутизацией.

- Если организаторы и участники присоединяются к собранию с одного сетевого сайта и есть участники, присоединяясь к одному собранию из STN, конечная точка Skype для бизнеса из другого сетевого сайта не может присоединиться к собранию.

Эти ограничения Location-Based маршрутов, указанные в следующей таблице.

|Пользователи в конференции в любой момент|Пользователи, которые могут присоединиться к конференции|Пользователям не разрешено присоединяться к конференции|
|:-----|:-----|:-----|
|Пользователи клиента VoIP Skype для бизнеса с одного сетевого сайта  <br/> |Пользователь клиента VoIP Skype для бизнеса с того же сетевого сайта  <br/> Пользователь клиента VoIP Skype для бизнеса с другого сетевого сайта  <br/> Пользователь клиента VoIP Skype для бизнеса с неизвестного сетевого сайта  <br/> Федераированный пользователь клиента VoIP Skype для бизнеса  <br/> Присоединение пользователей из конечной точки STN  <br/> |Нет  <br/> |
|Пользователи клиента VoIP Skype для бизнеса с неизвестного сетевого сайта  <br/> |Пользователь клиента VoIP Skype для бизнеса с любого сайта  <br/> Пользователь клиента VoIP Skype для бизнеса с неизвестного сайта  <br/> Федераированный пользователь клиента VoIP Skype для бизнеса  <br/> |Присоединение пользователя через конечную точку STN  <br/> |

|Пользователи клиента VoIP Skype для бизнеса с одного сетевого сайта и пользователи, присоединяясь с конечной точки ЗВОНКОВ  <br/> |Пользователь клиента VoIP Skype для бизнеса с того же сетевого сайта  <br/> |Пользователь клиента VoIP Skype для бизнеса с другого сетевого сайта  <br/> Пользователь клиента VoIP Skype для бизнеса с неизвестного сетевого сайта  <br/> Федераированный пользователь клиента VoIP Skype для бизнеса  <br/> |

Ниже слабются дополнительные характеристики Location-Based для приложения для работы с conferencing.

- Если пользователю не разрешено присоединяться к конференции с учетом ограничений Location-Based маршрутизов, вызов на конференцию будет отклонен, а клиент Skype для бизнеса будет сообщать о том, что звонок не был завершен или завершен.

- Конечная точка STN, присоединяемая к конференции с помощью Location-Based-принудительных служб маршрутов, не будет ограничена присоединением к конференции независимо от ее состояния, если конечная точка присоединяется через магистраль, для Location-Based маршрутизации не включена.

- Система УАПС, подключенная к серверу-посреднику через магистраль SIP, которая не передает вызовы в PSTN, будет иметь те же принудители, что и пользователи Skype для бизнеса, расположенные на том же сетевом сайте, где определена магистраль SIP. Например, конечная точка STN сможет присоединиться к конференции с пользователем УАПС и пользователем Skype для бизнеса, если они находятся на одном сетевом сайте; в противном случае конечная точка PSTN не сможет присоединиться к конференции, если пользователь УАПС находится на сетевом сайте, отличаемом от пользователя Skype для бизнеса.

## <a name="consultative-call-transfers"></a>Передача вызовов с консультацией

Помимо принудительной Location-Based маршрутов в собрания Skype для бизнеса, приложение Location-Based Routing for Conferencing применяет ограничения Location-Based на маршрутизацию при переназначениях звонков с консультациями, которые переходят на конечные точки STN. Переналадка звонка с консультацией — это вызов, установленный между двумя сторонами, где одна из сторон передает звонок новому пользователю. Например, конечная точка STN вызывает пользователя A (вызываемая программа Skype для бизнеса). Пользователь А определяет, что пользователь PSTN должен быть переадпределен пользователю Б (пользователю Skype для бизнеса). Пользователь А помещает звонок с пользователем PSTN на удержание и вызывает пользователя Б. Пользователь Б соглашается поговорить с пользователем STN. Пользователь А передает вызов на удержании пользователю Б.

Когда пользователь с включенной маршрутиали Location-Based инициирует переключение конечной точки STN с консультацией (как показано на предыдущем рисунке), создается два активных вызова: один звонок между пользователем PSTN и пользователем Skype для бизнеса A, а другой между пользователем Skype для бизнеса A и пользователем Skype для бизнеса Б. В приложении Location-Based Routing for Conferencing выполняется следующее поведение:

- Если магистраль SIP, маршрутизированная вызову STN, уполномочена перенаправка вызова STN на сетевой сайт, где находится пользователь Skype для бизнеса Б (то есть целевой объект передачи), то перенастройка звонка будет разрешена; в противном случае перена перевести вызов на консультацию будет заблокирован. Эта авторизация выполняется в зависимости от расположения перенаправленной стороны на том же сетевом сайте, что и магистраль SIP, которая выполняет маршрутизацию активного вызова в конечную точку STN.

- Если магистраль SIP, маршрутизщая входящий вызов STN, не имеет разрешения на маршрутизации вызовов на сетевой сайт, где находится перенаправленная сторона (пользователь Skype для бизнеса Б) или перенаправленная сторона расположена на неизвестном сетевом сайте, то перенаправление на конечную точку ЗВОНКОВ (то есть на конечный объект перенаправления вызовов) будет заблокировано.

## <a name="configuration-of-location-based-routing-for-conferencing"></a>Конфигурация Location-Based маршрутов для conferencing

Маршрут Location-Based для приложения для conferencing зависит от конфигурации Location-Based маршрутов. Основные конфигурации:

- Расположение участников, присоединявшихся к собранию, определяется на основе их сетевого сайта. Сетевой сайт и связанные с ним сетевые подсети должны быть определены в Skype для бизнеса Server, чтобы обеспечить Location-Based маршрутизации.

- Чтобы обеспечить Location-Based маршрутику собраний, участникам Skype для бизнеса необходимо включить Location-Based маршрутику.

- Чтобы обеспечить Location-Based маршрутику конечных точек STN, присоединяющихся к собраниям, магистраль SIP, используемая для подключения конечных точек STN, должна быть настроена для Location-Based маршрутов.

## <a name="enabling-the-location-based-routing-for-conferencing"></a>Включение Location-Based для conferencing

Маршрут Location-Based для приложения для conferencing отключен по умолчанию. Перед включением этого приложения необходимо определить правильный приоритет, который необходимо назначить приложению. Чтобы определить этот приоритет, запустите следующий cmdlet в оболочке управления Skype для бизнеса Server:

```powershell
Get-CsServerApplication -Identity Service:Registrar:<Pool FQDN>
```

В этом cmdlet это пул, в котором Location-Based для приложения для \<Pool FQDN\> conferencing должна быть включена.

Этот cmdlet возвращает список приложений, которые были в Skype для бизнеса Server, и значение приоритета для каждого из них. Маршрут Location-Based для приложения для conferencing должно быть назначено значение приоритета больше, чем приложение UdcAgent, и меньше, чем приложения "DefaultRouting", "ExumRouting" и "OutboundRouting". Рекомендуется назначить приложению Location-Based маршрутизацию для приложения conferencing значение приоритета, которое на одну точку выше значения приоритета приложения UdcAgent.

Например, если приложение UdcAgent имеет значение приоритета "2", "DefaultRouting" имеет значение приоритета "8", приложение "ExumRouting" имеет значение приоритета "9", а приложение "OutboundRouting" имеет значение приоритета "10", то необходимо назначить для приложения маршрутирования Location-Based приоритет "3". Это позволит разместить приоритет приложений в следующем порядке: другие приложения (приоритеты: от 0 до 1), "UdcAgent" (приоритет: 2), Location-Based приложение для работы с маршрутизацией (приоритет: 3), другие приложения (приоритеты: от 4 до 8), DefaultRouting (приоритет: 9), ExumRouting (приоритет: 10) и OutboundRouting (приоритет: 11).

После того как вы найдете правильное значение приоритета для приложения Location-Based Routing for Conferencing, введите следующий cmdlet для каждого пула Front-End или сервера Standard Edition, на который включены пользователи, для которых включена Location-Based Routing:

```powershell
New-CsServerApplication -Identity Service:Registrar:<Pool FQDN>/LBRouting -Priority <Application Priority> -Enabled $true -Critical $true -Uri <http://www.microsoft.com/LCS/LBRouting>
```

Например:

```powershell
New-CsServerApplication -Identity Service:Registrar:LS2013CU2LBRPool.contoso.com/LBRouting -Priority 3 -Enabled $true -Critical $true -Uri http://www.microsoft.com/LCS/LBRouting
```

После использования этого Location-Based перезапустите все серверы переднего сервера в пуле или на серверах Standard Edition, на которых включена Location-Based маршрутизации для приложения conferencing.

> [!IMPORTANT]
> Location-Based принудительное применение маршрутизации к конференциям или консультационной передаче не будет применяться до тех пор, пока не будут перезапущены все серверы переднего сервера в соответствующих пулах или серверах Standard Edition. Если в предыдущих $true задается значение **-Critical,** службы Skype для бизнеса Server будут немедленно перезапущены.  Если вы не хотите, чтобы эти службы перезагружались немедленно, установите для этого значение **-Critical** **$false,** а затем с помощью **set-CsServerApplication** измените **значение -Critical** на **$true** позже, после перезапуска служб.

После успешного включения Location-Based маршрутизации для конференц-конференций и перезапуска всех применимых серверов все конференции, организованные пользователями Skype для бизнеса с включенной поддержкой Location-Based, будут отслеживаться для предотвращения обхода платных звонков ЗВОНКОВ.
