# <a name="location-based-routing-for-conferencing-in-skype-for-business-server"></a>Маршрутизация конференц-связи в Скайп для Business Server на основе расположения

Планирование для расположения маршрутизация на основе конференц-связи в Скайп Business Server корпоративной голосовой связи, включая консультативное вызов передает.

Маршрутизация на основе местоположения позволяет ограничить маршрутизации вызовов между конечными точками VoIP и на основе местоположения стороны в вызове конечных точек PSTN. Маршрутизация конференц-связи на основе местоположения позволяет добавить правила маршрутизации на основе местоположения на собрания (то есть конференций), чтобы предотвратить ТСОП номер телефона сервера-посредника. Приложение отслеживает active конференции и применяет ограничения на основе расположения маршрутизации на основе местоположения пользователи, участвующие. Кроме того, маршрутизации на основе расположения для конференц-связи приложения позволяет наложения ограничений на основе расположения маршрутизации для консультативное операций передачи конечных точек PSTN.

Приложение конференц-связи маршрутизации на основе расположения предоставляет к Скайп для конференций Business пропускать механизм для предотвращения международную PSTN. Приложение отслеживает active конференций и применяет ограничения на основе расположения маршрутизации на основе местоположения Скайп для бизнес-пользователи участвуют.

Приложение конференц-связи маршрутизации на основе расположения определяет, является ли маршрутизации на основе расположения в силу на Скайп для собраний при выполнении следующих условий:

- Организатор собрания включается для маршрутизации на основе местоположения. Ограничения на основе расположения маршрутизации будет применяться только к конференции, организованные пользователей, которым разрешена для маршрутизации на основе местоположения.

- Хотя бы один участник собрания является конечной точкой ТСОП. Ограничения на основе расположения маршрутизации применимы только для конференций, которые включают конечные точки PSTN.

- Сетевой сайт, где находится шлюз ТСОП, используемый для подключения конференции к ТСОП, а также сетевые сайты, из которых подключаются организаторы и участники.

Маршрутизация на основе расположения для конференц-связи приложения не позволяет участия Скайп для бизнес-пользователи и конечные точки ТСОП из различных сетевых узлов для одной конференции. Если этот параметр включен организатора собрания для маршрутизации на основе местоположения, приложение конференц-связи обеспечивает следующие ограничения:

- Конечные точки, которые могут присоединиться к Скайп для собраний зависят от конечных точек, которые уже присоединился к конференции, и это ограничение изменяется оставлять соединенных конечные точки и новые конечные точки присоединиться к конференции. Если организаторов и участники участии Скайп для собраний в том же сайте сети, а затем конечной точки PSTN, другому участнику из того же сети узла, другому участнику из разных сетевого узла или участника от неизвестного сетевого узла могут присоединиться к.

- Если организаторы и участники присоединились к собранию из разных или неизвестных сетевых сайтов, конечной точке ТСОП запрещено присоединяться к собранию в случае, если звонок ТСОП поступает из магистрали SIP, для которой включена маршрутизация на основе расположения.

- Если организаторов и участники все присоединиться к собранию с одного узла сети и участники, присоединяющиеся к одной из ТСОП, Скайп для бизнеса конечную точку из разных сетевого узла не может присоединиться к собранию.

## <a name="consultative-call-transfers"></a>Переключения консультативных звонков

В дополнение к применение зависимостью от расположения маршрутизации для Скайп для деловых встреч, маршрутизации на основе расположения для конференц-связи приложения требуют соблюдения маршрутизации на основе местоположения ограничения на доступ к консультативное перевод звонков, которые выходной PSTN конечных точек. Переключение консультативного вызова — это вызов, установленный между двумя абонентами, в котором один из абонентов передает вызов новому абоненту. Например, для конечной точки ТСОП вызывает пользователя (Скайп для бизнеса вызываемого абонента). Пользователь определяет, что пользователь ТСОП должна пересылаться для пользователя B (Скайп для корпоративных пользователей). Абонент A переводит вызов пользователя ТСОП в режим ожидания и вызывает абонента B. Абонент B подтверждает согласие на разговор с пользователем ТСОП. Абонент A переключает вызов, находящийся в режиме ожидания, на абонента B.

## <a name="configuration-of-location-based-routing-for-conferencing"></a>Конфигурация маршрутизации на основе расположения для конференц-связи

Маршрутизация на основе расположения для конференц-связи приложения зависит от конфигурации маршрутизации на основе местоположения. Основные конфигурации следующие:

- Местоположение участников, присоединяющихся к собранию, определяется на основе их сетевого сайта. С сетевым узлом и его связанных подсетей должны быть определены в Скайп для Business Server для применения параметров маршрутизации на основе местоположения.

- Для применения параметров маршрутизации на основе местоположения собраний, Скайп для участников Business должен быть включен для маршрутизации на основе местоположения.

- Для применения параметров маршрутизации на основе расположения конечных точек PSTN присоединения к собраниям, магистраль SIP, используемый для подключения конечных точек ТСОП должен быть настроен для маршрутизации на основе местоположения.

## <a name="enabling-the-location-based-routing-for-conferencing"></a>Включение зависимостью от расположения маршрутизации для конференц-связи

По умолчанию отключен маршрутизации на основе расположения для приложения конференц-связи. Перед включением этого приложения необходимо определить и назначить для него правильное значение приоритета. Чтобы определить этот приоритет, выполните следующий командлет в Скайп оболочки управления Business Server:

Командлет Get-CsServerApplication-удостоверения службы: Registrar:<Pool FQDN>в этот командлет \<полное доменное имя пула\> — это пула, в котором необходимо включить маршрутизации на основе расположения для приложения конференц-связи.

Этот командлет возвращает список приложений, размещенных Скайп Business Server и значение приоритета для каждого из них. Маршрутизация на основе расположения для приложения конференц-связи необходимо назначить значение приоритета, размер которых превышает приложения «UdcAgent» и меньше, чем приложения «DefaultRouting», «ExumRouting» и «OutboundRouting». Рекомендуется назначить маршрутизации на основе расположения для конференц-связи приложения значение приоритета, который является одной точки выше, чем значение приоритета приложения «UdcAgent».

К примеру Если приложения «UdcAgent» имеет значение приоритета «2», приложение «DefaultRouting» имеет значение приоритета «8», приложение «ExumRouting» имеет значение приоритета «9» и приложение «OutboundRouting» имеет значение приоритета «10» нажмите Маршрутизация на основе расположения для конференц-связи приложения должны назначить значение приоритета «3». Таким образом поместит приоритет приложений в следующем порядке: других приложений (приоритетов: 0 до 1), «UdcAgent» (приоритет: 2), приложение маршрутизации конференц-связи на основе расположения (приоритет: 3), других приложений (приоритетов: 4-8), " DefaultRouting» (приоритет: 9), «ExumRouting» (приоритет: 10) и «OutboundRouting» (приоритет: 11).

После найти значение приоритета правильный для маршрутизации на основе расположения для приложения конференц-связи, введите следующий командлет для каждого пула переднего плана или сервера Standard Edition, домашние пользователи включены для маршрутизации на основе местоположения:

New-CsServerApplication-удостоверения службы: Registrar:`<Pool FQDN`> / LBRouting-приоритет \<приоритет приложений\> -включено $true-критические $true - Uri<http://www.microsoft.com/LCS/LBRouting> 

Например:

New-CsServerApplication-Service:Registrar:LS2013CU2LBRPool.contoso.com/LBRouting удостоверение-приоритет 3 - Enabled $true-критические $true - Urihttp://www.microsoft.com/LCS/LBRouting 

После использования этого командлета, перезапустите все серверы переднего плана в пуле или серверы Standard Edition, где был включен маршрутизации на основе расположения для приложения конференц-связи.

> [!IMPORTANT]
> Зависимостью от расположения принудительным ограничением маршрутизации для конференции и передачи консультативное не будет применяться до всех серверов переднего плана в применимое пулов или после перезапуска серверы Standard Edition. Если задать **-критические** значение **$true** в предыдущем командлеты вашей Скайп для служб Business Server сразу же перезапускается. Если не хотите, чтобы эти службы Немедленный перезапуск, задайте **-критические** значение **$false** для сейчас и затем с помощью **Set-CsServerApplication** изменение **-критические** значение **$true** более поздних версий, после перезапуска служб.
