# <a name="location-based-routing-for-conferencing-in-skype-for-business-server"></a>Маршрутизация на основе местоположения для конференций в Skype для бизнеса Server

Планирование маршрутизации на основе местоположения для конференц-связи в Skype для бизнеса Server Enterprise, включая передачу звонков консултативе.

Маршрутизация на основе местоположения позволяет ограничить маршрутизацию звонков между конечными точками VoIP и КОММУТИРУЕМыми конечными точками, основываясь на местоположении сторон в звонке. Маршрутизация на основе местоположения позволяет применять правила маршрутизации на основе местоположения для собраний (например, конференций), чтобы предотвратить бесплатный звонок. Приложение отслеживает активную конференцию и применяет ограничения для маршрутизации на основе местоположения на основе местоположения пользователей, участвующих в программе. Приложение для маршрутизации на основе местоположения дополнительно позволяет применять ограничения маршрутизации на основе местоположения для консултативе передач, использующих конечные точки PSTN.

Приложение для конференц-связи с маршрутизацией на основе местоположения предоставляет механизм для конференций Skype для бизнеса для предотвращения бесплатных бесплатных обоснований. Приложение отслеживает активные Конференции и применяет ограничения для маршрутизации на основе местоположения на основе местоположения пользователей Skype для бизнеса.

Приложение для конференц-связи с маршрутами на основе местоположения определяет, следует ли применять маршрутизацию на основе местоположения на собрании Skype для бизнеса, если выполняются указанные ниже условия.

- Организатор собрания включен для маршрутизации на основе местоположения. Ограничения маршрутизации, основанные на расположении, будут применяться только к конференц-атакам, упорядоченным по пользователям, для которых включена маршрутизация на основе местоположения.

- Хотя бы один участник собрания является конечной точкой ТСОП. Ограничения маршрутизации, основанные на расположении, применимы только для конференций, использующих конечные точки PSTN.

- Сетевой сайт, где находится шлюз ТСОП, используемый для подключения конференции к ТСОП, а также сетевые сайты, из которых подключаются организаторы и участники.

Приложение для маршрутизации на основе местоположения для конференций предотвращает участие пользователей Skype для бизнеса и конечных точек PSTN с разных сетевых сайтов на одну и ту же конференцию. Если для организатора собрания включена маршрутизация на основе местоположения, приложение для конференц-связи накладывает указанные ниже ограничения.

- Конечные точки, которые могут присоединиться к собранию в Skype для бизнеса, зависят от конечных точек, которые уже присоединились к Конференции, и это ограничение настраивается так, чтобы присоединиться к Конференции и добавить конечные точки Если организаторов и участники присоединяются к собранию Skype для бизнеса с одного и того же сетевого сайта, конечная точка PSTN, другой участник того же сетевого сайта, другой участник из другого сетевого сайта или участник с неизвестного сетевого сайта разрешено присоединиться.

- Если организаторы и участники присоединились к собранию из разных или неизвестных сетевых сайтов, конечной точке ТСОП запрещено присоединяться к собранию в случае, если звонок ТСОП поступает из магистрали SIP, для которой включена маршрутизация на основе расположения.

- Если организаторов и участники присоединяются к собранию из того же сайта сети, а участники, присоединяющиеся с одним и тем же собранием из PSTN, не могут присоединиться к собранию с помощью конечной точки Skype для бизнеса с другого сайта сети.

## <a name="consultative-call-transfers"></a>Переключения консультативных звонков

В дополнение к принудительному применению маршрутизации на основе местоположения к собраниям Skype для бизнеса, маршрутизация на основе местоположения обеспечивает ограничения на маршрутизацию на основе местоположения при передаче звонков консултативе в конечные точки PSTN. Переключение консультативного вызова — это вызов, установленный между двумя абонентами, в котором один из абонентов передает вызов новому абоненту. Например, конечная точка PSTN вызывает пользователя A (вызываемый Skype для бизнеса). Пользователь а определяет, нужно ли перенаправлять пользователя PSTN на пользователя B (пользователя Skype для бизнеса). Абонент A переводит вызов пользователя ТСОП в режим ожидания и вызывает абонента B. Абонент B подтверждает согласие на разговор с пользователем ТСОП. Абонент A переключает вызов, находящийся в режиме ожидания, на абонента B.

- Если магистральная маршрутизация SIP, на которую помещается Звонок по протоколу PSTN, может перенаправить звонок на сетевой сайт, на котором находится пользователь B для бизнеса (например, для передачи), будет разрешено передача вызова. в противном случае передача вызова консултативе будет заблокирована. Эта авторизация выполняется в зависимости от расположения, которое передается на тот же сайт сети, что и магистраль SIP, который пересылает активный Звонок в конечную точку PSTN.

- Если маршрутизация по магистральной КОММУТИРУЕМой линии SIP не авторизована для маршрутизации звонков на сетевой сайт, на котором находится переданный абонент (Skype для бизнеса B) или если он находится на неизвестном сетевом сайте, перенесите вызов консултативе Конечная точка КТСОП (например, цель передачи звонка) будет заблокирована.

## <a name="configuration-of-location-based-routing-for-conferencing"></a>Настройка маршрутизации на основе местоположения для конференций

Приложение для маршрутизации на основе местоположения для конференций использует конфигурацию маршрутизации на основе местоположения. Основные конфигурации следующие:

- Местоположение участников, присоединяющихся к собранию, определяется на основе их сетевого сайта. Чтобы обеспечить маршрутизацию на основе местоположения, необходимо определить сайт сети и связанные с ним подсети в Skype для бизнеса Server.

- Для принудительной маршрутизации собраний участники Skype для бизнеса должны быть доступны для маршрутизации на основе местоположения.

- Для принудительной маршрутизации конечных точек PSTN, присоединяющихся к собраниям, необходимо настроить для маршрутизации на основе местоположения магистраль SIP.

## <a name="enabling-the-location-based-routing-for-conferencing"></a>Включение маршрутизации на основе местоположения для конференц-связи

Приложение по умолчанию для конференций с учетом расположения отключено. Перед включением этого приложения необходимо определить и назначить для него правильное значение приоритета. Чтобы определить приоритет, выполните в командной консоли Skype для Business Server следующий командлет:

Get-Кссервераппликатион-Identity Service: регистратор:<Pool FQDN>в этом командлете \<доменное\> имя пула — это пул, в котором должна быть включена маршрутизация на основе местоположения для конференций.

Этот командлет вернет список приложений, размещенных в Skype для бизнеса Server, и значение приоритета для каждого из них. Службе маршрутизации на основе местоположения для конференций необходимо назначить значение приоритета, большее, чем приложение "Удкажент", и меньше, чем приложения "Дефаултраутинг", "Ексумраутинг" и "Аутбаундраутинг". Мы рекомендуем назначить для приложения для конференц-связи с маршрутизацией значение приоритета, равное одной точке выше значения приоритета для приложения "Удкажент".

Например, если для приложения "Удкажент" установлено значение приоритета "2", в приложении "Дефаултраутинг" значение priority равно "8", а для приложения "Ексумраутинг" установлено значение Priority "9", а для приложения "Аутбаундраутинг" установлено значение приоритета "10". Вы должны назначить маршрут на основе местоположения для приложения для конференций со значением приоритета 3. Это приведет к порядку приоритетов приложений в следующем порядке: другие приложения (приоритеты: 0 – 1), "Удкажент" (приоритет: 2), приложение для конференц-связи с маршрутами (приоритет: 3) и другие приложения (приоритеты: 4 – 8) Дефаултраутинг "(приоритет: 9)," Ексумраутинг "(приоритет: 10) и" Аутбаундраутинг "(приоритет: 11).

После того как вы найдете правильные значения приоритета для приложения для маршрутизации с учетом расположения для конференций, введите следующий командлет для всех интерфейсов переднего плана или сервера Standard Edition, для которых включены пользователи для маршрутизации на основе местоположения:

New-Кссервераппликатион-Identity Service: регистратор:`<Pool FQDN`_Гт_/Лбраутинг-Priority \<(приоритет\> приложения) с поддержкой $true, критическое $true-URI<http://www.microsoft.com/LCS/LBRouting> 

Например:

Служба New-Кссервераппликатион-Identity:регистрар:LS2013CU2LBRPool. contoso. com/Лбраутинг-Priority $true — критический $true-URIhttp://www.microsoft.com/LCS/LBRouting 

После использования этого командлета перезапустите все серверы переднего плана в пуле или стандартные серверы выпусков, для которых включена маршрутизация на основе местоположения для конференций.

> [!IMPORTANT]
> Принудительная маршрутизация на основе местоположения для конференций или консултативе передач не будет применена, пока не будут перезапускаются все серверы переднего плана в соответствующих пулах или на серверах стандартных выпусков. Если вы установили **критически важный** для **$true** в указанных выше командлетах, службы Skype для бизнеса Server будут сразу же перезапускаться. Если вы не хотите, чтобы эти службы немедленно перезапускались **** , установите для них критические **$false** , а затем используйте команду **Set-кссервераппликатион** для изменения **критических** на **$true** позже после перезапуска служб.

После успешного включения и повторного запуска всех применимых серверов все конференции пользователей Skype для бизнеса, для которых включена маршрутизация на основе местоположения, будут отслеживаться, чтобы предотвратить Бесплатные звонки по КТСОП
