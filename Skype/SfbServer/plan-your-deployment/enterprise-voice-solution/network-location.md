# <a name="define-the-network-elements-used-to-determine-location-in-skype-for-business-server"></a>Определение сетевых элементов, используемых для определения расположения в Skype для бизнеса Server
 
Решения, необходимые для планирования сетевых компонентов, которые вы будете использовать для картографии вызывателей к расположениям для развертывания E9-1-1 в Skype для бизнеса Server Корпоративная голосовая связь.
  
Если вы настраиваете инфраструктуру Skype для бизнеса Server для поддержки автоматического обнаружения местоположения клиента, сначала необходимо решить, какие сетевые элементы вы собираетесь использовать для картографии звониющих в расположения. В Skype для бизнеса Server можно связать следующие сетевые элементы Layer 2 и Layer 3 с расположениями:
  
- адреса BSSID точек беспроводного доступа (WAP) (уровень 2);
    
- порт коммутатора LLDP (уровень 2);
    
- идентификаторы шасси коммутаторов LLDP (уровень 2);
    
- IP-подсети (уровень 3);
    
- MAC-адреса клиентов (уровень 2).
    
Элементы сети перечислены в порядке приоритета. Если клиент может быть расположен с помощью более чем одного сетевого элемента, Skype для бизнеса Server используется порядок приоритета, чтобы определить, какой механизм использовать. 
  
В следующих подразделах приводятся более подробные сведения об использовании каждого элемента сети.
  
> [!IMPORTANT]
> При использовании сетевых элементов для картографии вызывателей к расположениям крайне важно сохранить базу данных службы сведений о местоположении в соответствии с данными о местоположении. Так, если вы добавляете или изменяете элемент сети, например WAP, вам нужно удалить старую запись и добавить новую в базу данных местоположений. 
  
## <a name="wireless-access-point"></a>Точка беспроводного доступа

Когда клиент подключается к сети по беспроводной связи, запрос расположения использует BSSID-адрес WAP для определения его расположения. Если клиент находится в роуминге, указанный WAP может быть не самым близким, и можно даже выбрать WAP, расположенную на другом этаже здания. Чтобы указать, что расположение приблизительное, можно предуказать значение расположения с помощью дескриптора **[Near]** или **[Closeto].**
  
Этот метод определения местоположения предполагает, что адреса BSSID всех точек WAP являются статическими. Однако, если поставщик WAP использует динамически назначенные BSSID, BSSID, полученный из WAP, может измениться (это может произойти после изменения конфигурации WAP), и беспроводные клиенты могут быть оставлены в ситуации, когда они не получают расположение. Чтобы предотвратить эту возможность, необходимо заполнить базу данных службы сведений о местоположении ERLs для всех возможных адресов BSSID, используемых каждым WAP. 
  
## <a name="lldp-ports-and-switches"></a>Порты и коммутаторы LLDP

Управляемые коммутаторы Ethernet, поддерживающие протокол LLDP-MED, могут сообщить свои идентификационные данные и сведения о портах клиентам, совместимым с LLDP-MED. Это позволяет затем выполнять запросы к базе данных местоположений для определения местоположения устройства. Вы можете связать сведения о местоположении для экстренных служб с идентификатором шасси коммутатора или сопоставить их также на более низком уровне — уровне портов.
  
> [!NOTE]
> Skype для бизнеса Server поддерживает использование LLDP-MED для определения расположения только устройств Lync Телефон Edition и Skype для бизнеса клиентов, работающих Windows 8. Если вам необходимо использовать данные уровня коммутатора 2 для определения расположения других подключенных клиентов Skype для бизнеса Server пк, необходимо использовать метод клиентского MAC-адреса. 
  
## <a name="subnet"></a>Подсеть

Подсети IP-адресов уровня 3 предоставляют механизм, поддерживаемый всеми Skype для бизнеса Server клиентами, которые могут быть использованы для автоматического обнаружения расположения клиента. Использование IP-подсетей является самым простым способом определения местоположения в плане настройки клиентов проводного доступа и управления ими. Однако прежде чем выбрать его, ответьте на следующие вопросы, чтобы определить, будет ли он достаточно точным в вашей ситуации.
  
- Охватывают ли какие-либо клиентские подсети несколько этажей?
    
- Охватывают ли какие-либо подсети несколько зданий?
    
- Какую площадь покрывает каждая клиентская подсеть?
    
Если подсеть покрывает слишком большую площадь, может потребоваться использовать другой способ для определения местоположения клиентов. Однако, если это целесообразно, мы рекомендуем клиентам реорганизовать структуру IP-подсетей в соответствии с требованиями к точности определения местоположений ERL вместо того, чтобы реализовывать сторонние решения на основе SNMP, связанные с дополнительными затратами и повышением сложности.
  
## <a name="client-mac-address"></a>MAC-адрес клиента

Чтобы использовать MAC-адрес клиентского компьютера для поиска вызываемого пользователя, необходимо использовать управляемые переключатели Ethernet, и необходимо развернуть стороннее решение SNMP, которое может обнаруживать mac-адреса Skype для бизнеса клиентов, подключенных к (или через) эти переключатели. Решение SNMP постоянно оповешает управляемые переключатели для получения текущих сопоставлений конечных mac-адресов, подключенных к каждому порту, и получает соответствующие портовые ID. Во время Skype для бизнеса запроса клиента в службу сведений о расположении служба сведений о расположении запрашивает стороне приложение с помощью MAC-адреса клиента, а затем возвращает все совпадающие IP-адреса коммутатора и портовые ID. Служба сведений о расположении использует эти сведения для запроса опубликованной прослушки слоя 2 для записи совпадения и возвращает расположение клиенту. При использовании этого параметра убедитесь, что идентификаторы портов коммутатора совпадают между приложением SNMP и опубликованными записями баз данных данных о расположении.
  
> [!NOTE]
> Некоторые сторонние решения SNMP могут поддерживать неугомонные переключатели доступа; если переключатель, который Skype для бизнеса клиент неуменим, но имеет связь с управляемым переключателем рассылки, управляемый переключатель может сообщить в приложение SNMP mac-адреса клиентов, подключенных к коммутатору доступа. Эта информация позволяет службе сведений о местоположении определять местоположение пользователя. Однако можно назначить только один ERL всем портам на неуголовном переключателье, поэтому специфика расположения доступна только на уровне шасси коммутатора доступа, а не на уровне порта.
