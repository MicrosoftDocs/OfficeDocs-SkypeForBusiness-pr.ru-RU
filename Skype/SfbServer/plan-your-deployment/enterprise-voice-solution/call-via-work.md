# <a name="plan-for-call-via-work-in-skype-for-business-server"></a>Планирование позвонить с рабочего в Скайп для Business Server
 
Планирование для звонков с помощью Скайп Business Server, что позволяет интеграцию Скайп для бизнеса и вашей телефонной системы УАТС, чтобы пользователи могли использовать Скайп для бизнеса для управления телефонах УАТС.
  
 **Звонок с помощью рабочих** — это новая возможность в Скайп для Business Server, который позволяет интегрировать вашей Скайп для решения бизнес-с вашей существующий телефон УАТС. Включен для звонков с помощью рабочих щелчке в Скайп для бизнеса для звонка другому пользователю, в развертывании или внешних пользователей. Вызов выполняется с помощью телефона УАТС пользователя. Это позволяет пользователю с телефон УАТС включает аудио в их расширенными возможностями Скайп для бизнеса бесед. В предыдущих версиях Lync Server удаленного вызова элемент управления был функции, которая позволяет пользователям управлять телефонах УАТС с Lync Server. В Скайп Business Server этот компонент был заменен звонок с помощью рабочих.
  
Звонок с помощью рабочих включает следующие параметры для пользователей телефон УАТС
  
- возможности режима «звонок щелчком» с использованием аудиосвязи, предоставляемой телефоном УАТС;
    
- Сведения о присутствии, поиск пользователей и обмена мгновенными Сообщениями интеграции--, например двух пользователей звонок с помощью рабочего сеанса обмена мгновенными Сообщениями позволяет проводить аудиоконференции их сеанса аудио, предоставленные с помощью на телефонах УАТС.
    
- Возможность добавления обмена мгновенными Сообщениями, общий доступ к приложениям и передачи файлов для звонка звонок с помощью рабочих.
    
- возможность присоединения к собранию одним щелчком.
    
## <a name="how-it-works"></a>Принципы работы

Звонок с помощью рабочих использует Unified Communications Web API (UCWA) как агент с двумя межсетевыми экранами пользователя (B2BUA) между системой УАТС и вашей Скайп для развертывания Business Server, чтобы шлюз компьютерного телекоммуникационного приложения (CSTA) необходима для подключения Скайп Business Server с системой УАТС. UCWA — это служба, введенные в предыдущих версиях Lync Server для поддержки связи с мобильных устройств и веб-клиентов и автоматически устанавливается на каждом сервере переднего плана.
  
### <a name="call-workflow-for-a-call-via-work-call"></a>Вызов рабочего процесса для звонка звонок с помощью рабочих

Ниже показано, как пользователь включен для звонков с помощью рабочих можно использовать Скайп для Business Server звонок:
  
![Показывает шаги во время вызова "Позвонить с рабочего". Сначала абонент нажимает вызов пользователя в клиенте Skype для бизнеса, затем UCWA выполняет вызов телефона абонента. После принятия вызова абонентом выполняется вызов получателя.](../../media/050e88ed-e18e-40c0-84d5-b17fe40c305a.jpg)
  
1. Пользователь выбирает пользователь в их Скайп для клиента Business и щелкает значок телефона позвонить ему. Аналогично, во время текстовой беседы пользователь может щелкнуть пользователя, с которым установлен сеанс беседы.
    
2. Телефон УАТС пользователя, который разместил вызов начинает звонить. Идентификатор звонящего для этого телефона показывает глобальные телефонный номер, который настроен для отображения в Звонящего всех пользователей, голосовая связь звонок с помощью рабочих. Данный глобальный телефонный номер не является фактическим телефонными номером, который соответствует телефону любого пользователя. Это визуальный сигнал, который дает пользователю понять, что это его собственный исходящий звонок, а не входящий вызов. При развертывании звонок с помощью рабочих Сообщите пользователям о этот глобальный номер и описание.
    
3. Принимает свой телефон УАТС пользователя, выполнившего звонок. Затем Скайп для бизнеса инициирует голосовых вызовов для вызываемого абонента. 
    
4. При ответе вызываемого абонента начинается голосовой вызов. Если два пользователя уже находятся в сеансе обмена мгновенными сообщениями, сеанс можно продолжить.
    
### <a name="joining-a-conference-with-call-via-work"></a>Присоединение к конференции с помощью функции «Позвонить с рабочего»

Звонок с помощью рабочих пользователь может присоединиться к запланированным собранием, щелкнув URL-адрес собрания. Скайп для бизнеса отображает сообщение **для исходящих звонков** до собрания службы набравший телефон УАТС пользователя. Затем пользователь звонок с помощью рабочих принимает телефон УАТС и присоединился к собранию.
  
Для создания провести собрание собраний пользователя звонок с помощью рабочих можно использовать в в Скайп для бизнеса **Команде провести** собрание. После этого пользователь видит сообщение **Выполняется вызов**, а телефон УАТС начинает звонить.
  
Звонок с помощью рабочих пользователь может также набрать в собрании, вызвав номер конференции Bridge из внутри Скайп для бизнеса. При необходимости использования для конференции ПИН-кода пользователь должен использовать свой телефон УАТС для ввода кода.
  
### <a name="incoming-calls"></a>Входящие звонки

Когда пользователь включен для звонков с помощью рабочих получает Скайп для бизнеса звонок, телефон УАТС и Скайп пользователя пользователей всех звонков одновременно (если пользователь Настройка одновременных звонков) которых. Пользователь может принять звонок путем телефонного УАТС или нажав кнопку **принять** на Скайп для бизнес-уведомления. Если пользователь принимает звонок с помощью Скайп для бизнеса, Скайп для бизнеса окна для вызова остается открытым. Но если пользователь принимает вызова с телефонного УАТС, закрывает Скайп для бизнеса окно уведомления и нет не Скайп для бизнеса сеанса, только голосовой звонок через телефон УАТС.
  
Когда пользователь включен для звонков с помощью рабочих получает вызов УАТС, телефонного звонка УАТС.
  
## <a name="limitations-of-call-via-work"></a>Ограничения позвонить с рабочего

Звонок с помощью рабочих — это решение голосовой связи, который требуется маленьким Настройка оборудования, но имеет ограничения по сравнению с функции, доступные в полный корпоративной голосовой связи или удаленного управления звонками. Звонок с помощью рабочих имеет следующие ограничения:
  
- Если пользователь звонок с помощью рабочих настроил переадресацию звонков для звонков с помощью рабочий номер обратного вызова, а кто-то пытается пригласить этого пользователя к собранию по номеру телефона пользователя, приглашение не будет связаться с пользователем. Следует сообщить пользователям, что для приглашения участников необходимо щелкать их имена, а не номера телефонов. 
    
- Усовершенствованные возможности 911 и отслеживание нежелательных звонков недоступны во время вызова с помощью рабочих вызовов.
    
- Пользователям, включенным для звонков с помощью рабочих нельзя использовать делегирование, групповой звонок или функции работы с группами ответа.
    
- Звонок с помощью работы нельзя использовать Скайп для бизнеса запись собрания, отключение или включение звука звонок, хранения или перевести звонок или использования парковки вызовов.
    
- Звонок с помощью рабочих нельзя использовать для доступа к своим сообщениям голосовой почты УАТС.
    
- Пользователи из вызывать с помощью рабочих невозможно перевести сеанс, работы в качестве голосовой звонок в собрание для совместной работы, которая включает в себя коммуникации, такие как видео, Powerpoint, доски или одной заметки.
    
- Пользователи звонок с помощью рабочих нельзя добавить большее количество пользователей на вызов 2 человека.
    
- Отсутствует поддержка сопряжения стационарного телефона или подключаемого модуля VDI.
    
- Если пользователь делает или отвечает на звонок с помощью телефона УАТС (и не Скайп для окна бизнеса), будет не журнала вызовов.
    
- Если система УАТС не поддерживает метод **REFER с заменами**, будет наблюдаться следующее поведение. Во время на звонок с помощью рабочих звонка, если пользователь передает разговора по телефону УАТС, окно вызова не исчезнет из их Скайп для бизнеса окна. Если пользователь затем закроет окно вызова, вызов между целевым объектом и передающим звонок пользователем будет завершен.
