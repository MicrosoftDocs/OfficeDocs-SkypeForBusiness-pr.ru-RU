# <a name="plan-for-location-based-routing-in-skype-for-business"></a>Планирование зависимостью от расположения маршрутизации Скайп для бизнеса
 
Планирование расположения маршрутизация на основе в Скайп Business Server корпоративной голосовой связи, включая взаимодействие с одновременных звонков и делегирование и поддерживаемых сценариях для маршрутизации на основе местоположения.
  
Маршрутизация на основе местоположения позволяет ограничить маршрутизации вызовов между конечными точками VoIP и на основе местоположения стороны в вызове конечных точек PSTN. Маршрутизация на основе расположения — это функция управления вызовами, который определяет, как вызовы направляются с Скайп Business Server. Он обеспечивает правил авторизации вызовов на звонки могут направляться для УАТС или ТСОП конечные точки, основанные на Скайп для бизнеса вызывающего абонента географического расположения.
  
Маршрутизация на основе расположения вводит новый набор правил, который изменяет маршрутизацию местных и международных звонков на номер ТСОП для предотвращения обхода платных звонков. Маршрутизация на основе расположения предоставляет гибкость для анализа этих правил в пределах определенных областей, шлюзов или только наборов пользователей.
  
Следующие сценарии иллюстрируют основные типы ограничения, которые может задать маршрутизации на основе местоположения:
  
- Исходящих вызовов - маршрутизация на основе расположения может задать исходящие вызовы для исходящего трафика на шлюз ТСОП, расположенного в той же области, как где вызывающего — для предотвращения PSTN международную сервера-посредника, которого не позволяет вызовы для исходящего трафика на шлюз ТСОП, расположенный в другой регион как вызывающего абонента.
    
- Входящих вызовов - маршрутизация на основе расположения можно запретить входящих звонков ТСОП звонить Скайп для бизнеса конечных точек, если шлюз ТСОП, маршрутизация входящего звонка не находится в той же области, как называемое Скайп для бизнес-пользователя.
    
- Неизвестный областей - маршрутизации на основе местоположения ограничивает входящие и исходящие вызовы ТСОП в и из него пользователей, которые расположены в неопределенные расположения (то есть удаленных пользователей подключения из Интернета или в неизвестный регионы).
    
- Международные областей - маршрутизации на основе местоположения обеспечивает маршрутизацию исходящие вызовы через международные шлюзы ТСОП, если не удается найти шлюз локального расположения пользователя.
    
## <a name="guidance-for-where-to-apply-location-based-routing"></a>Руководство для применения маршрутизации на основе расположения

Маршрутизация на основе расположения в зависимости от ситуации можно применить на расположение узла сети конечной точки пользователя или на расположение узла сети шлюза ТСОП. В этом разделе представлены применен инструкции по маршрутизации на основе местоположения.
  
### <a name="applying-location-based-routing-at-the-users-location"></a>Применение маршрутизации на основе расположения в расположении пользователя

Расположение на основе маршрутизации использует то же области сети, сайтов и подсетей, как определено в Скайп Business сервер, используемый E9-1-1, контроль допуска звонков и сервера-посредника для применения ограничений маршрутизации вызовов для предотвращения международную PSTN обхода. Расположение пользователя определяет, какие IP-подсеть пользователя Скайп for Business конечные подключаются из. Каждая IP-подсеть связана с сетевым сайтом, которые объединяются в сетевых областях, определяемых администратором. Маршрутизация на основе местоположения при принудительном применении на основании пользователя сетевого узла.
  
Правила маршрутизации на основе местоположения, применяются на каждой узла сети, то есть применения данного набор правил для всех конечных точках включен для маршрутизации на основе расположения, которые расположены в одном сайте сети. Администраторы могут применять зависимостью от расположения маршрутизации для сетевых узлов, которые требуют его.
  
Политики маршрутизации голосовой связи могут задаваться для каждого отдельного сетевого сайта для выявления конкретного шлюза ТСОП, через который все пользователи, расположенные на этом сайте, должны звонить на телефонные номера ТСОП. Такие политики маршрутизации голосовой связи будет иметь приоритет над определенные в политику голосовой связи, когда пользователь находится в области сети включен для маршрутизации на основе местоположения, и он будет препятствовать маршрутизации вызовов с помощью других шлюзы ТСОП, которые разрешены для маршрутизации Маршрутизация на основе местоположения. При Скайп для бизнес-пользователь совершает звонок PSTN, политику голосовой связи определяет, можно ли право пользователя для выполнения вызова. Если политика голосовой связи пользователя пользователь может выполнить вызов, зависимостью от расположения маршрутизации определяет, какие следует egress звонок от шлюза ТСОП. Маршрутизация на основе местоположения осуществляет это определение на основе местоположения пользователя.
  
Местоположение пользователя может относиться к одному из перечисленных ниже типов.
  
- Пользователь находится в известных сетевого узла включен для маршрутизации на основе расположения и завершает свой номер DID (прямого входящего набора) на шлюз ТСОП, расположенные в том же сайте сети (то есть office). Маршрутизация исходящих звонков будет осуществляться на основе политики маршрутизации голосовой связи сетевого сайта, на котором расположен пользователь. Входящие звонки ТСОП данному пользователю маршрутизируются на конечные точки, которые расположены на том же сетевом сайте, что и шлюз ТСОП.
    
- Пользователь расположен на известном сайте сети, отличном от того, в котором расположен шлюз ТСОП (например, пользователь перешел в другой офис того же предприятия). Маршрутизация исходящих звонков будет выполняться на основе политики маршрутизации голосовой связи сайта сети, в котором расположен пользователь. Входящие звонки ТСОП данному пользователю не будут направляться на конечные точки, расположенные в отличных от сайта ТСОП сайтах, чтобы избежать обхода платных звонков ТСОП.
    
- Если пользователь находится в области сети, которая не известен Скайп для развертывания Business Server, маршрутизации исходящих вызовов зависит политика голосовой связи, назначенные пользователю, чтобы шлюзы ТСОП, не привязанных к ограничений на основе местоположения маршрутизации. Входящие звонки ТСОП не будут направляться на конечные точки, расположенные на неизвестных сетевых сайтах для предотвращения обхода платных звонков ТСОП. 
    
### <a name="applying-location-based-routing-at-the-pstn-gateways-location"></a>Применение маршрутизации на основе расположения в расположении шлюза ТСОП

Звонки перенаправляться через шлюзы ТСОП и УАТС может потребоваться ограничения на основе расположения маршрутизации в зависимости от расположения такие системы. Маршрутизация на основе расположения можно включить на уровне детализации на основе каждой магистрали.
  
Маршрутизация на основе местоположения представляется следующий набор правил, при включении этого параметра в канале:
  
- При включении маршрутизации на основе местоположения на основе каждой магистрали правила определяют, на которые магистрали будет применяться только вызовы, направляемые по этой линии связи.
    
- Чтобы запретить обход расходов на международную PSTN, где звонков берутся из различных с сетевым узлом, который сетевого узла, где находится шлюз ТСОП, маршрутизации на основе местоположения представляется связь с сетевым узлом для данной магистрали. Таким образом, определяется сетевой сайт, который позволяет осуществлять маршрутизацию звонков на данную магистраль.
    
Магистральных линий связи можно включить для маршрутизации на основе местоположения двумя способами:
  
- Магистраль определяется для шлюза ТСОП, который переадресовывает звонки на номер ТСОП. Входящие звонки, перенаправленные на магистраль данного типа, будут перенаправлены только на конечные точки, расположенные на том же сайте сети, что и эта магистраль.
    
- Магистрали определяется для сервера-посредника однорангового узла не исходящих звонков ТСОП и службы пользователям с помощью устаревшего телефоны в статических расположения (то есть телефонах УАТС). При данной конкретной конфигурации все входящие звонки, перенаправленные на магистраль данного типа, будут считаться исходящими из того же сетевого сайта, что и эта магистраль. Вызовы от пользователей УАТС будут иметь же принудительные маршрутизации на основе местоположения как Скайп для пользователей, расположенных в том же сайте сетевой магистрали. Если две системы УАТС, расположенный в отдельные сетевые узлы подключенные через Скайп для сервера, на основе расположения маршрутизации позволит маршрутизации из одной конечной точки УАТС в одном узле сети к другой конечной точке УАТС в узле сети. Этот сценарий с зависимостью от расположения маршрутизации не блокируются. Помимо этого сценария и аналогичным образом как Скайп для корпоративных пользователей в одном месте конечные точки, подключенные к peer сервер-посредник с этой конфигурацией смогут выполнять либо принимать звонки в и из других одноранговых сервера-посредника, которые не маршрутизировать звонки t o PSTN (то есть конечную точку, подключенных к различных УАТС) независимо от сетевого узла, с которым связана однорангового узла сервера-посредника. Все входящие звонки, исходящие вызовы, вызовите передачи и вызвать пересылки, связанные с конечными точками PSTN будут может быть расположение на основе маршрутизации для использования только шлюзы ТСОП, определенные как локальный таких одноранговому сервера-посредника.

### <a name="incoming-calls"></a>Входящие звонки

Маршрутизация входящих вызовов пользователям с поддержкой маршрутизации на основе местоположения зависит от местоположения пользователя конечной точки. Маршрутизация входящих звонков подвергается влиянию следующим образом. Если пользователь имеет входящего звонка на конечную точку, расположенный в маршрутизации на основе расположения включены сетевого узла и конечной точке находится в том же сайте сети шлюза ТСОП, будут направляться вызова. Если пользователь имеет входящего звонка на конечную точку, расположенный в маршрутизации на основе расположения поддержкой сетевого узла и конечной точке расположен в разных сетевого узла, чем шлюз ТСОП, не будут направляться вызова. Если пользователь не находится в том же сайте сети как шлюз ТСОП где исходящим входящего звонка из конечных точек, о входящем звонке будут направляться непосредственно к голосовой почте пользователя и уведомление о пропущенном звонке будет отправляться вызываемый абонент.
  
Параметры пользователя с поддержкой маршрутизации на основе расположения для переадресации звонков будет применяться, тем не менее, будут переадресовываться звонки может быть ограничения маршрутизации на основе местоположения пользователя.

Маршрутизация на основе расположения определяет, перенесены или пересылку в зависимости от расположения обе конечные точки вызова.

В зависимости от сети сайта лицо, получающее результате в конечной точки маршрутизации на основе расположения влияет на маршрутизации передачи звонок или перенаправлять.

### <a name="delegation"></a>Делегирование

Возможности делегирования в Скайп для бизнеса, влияют на основе расположения маршрутизации следующим образом:
  
- Когда делегат включен для маршрутизации на основе местоположения местах звонок от имени руководителя, политика голосовой связи делегата используется для авторизации вызова и голосовой связи сайтов делегата политику маршрутизации будет использоваться для маршрутизации вызова
    
- Что касается исходящих звонков ТСОП диспетчеру, действуют те же правила, регламентирующие переадресацию звонков и порядок обработки одновременных звонков, что и в соответствующих разделах.
    
- При делегата устанавливает в качестве целевого одновременных звонков, для входящего звонка на диспетчер конечной точки PSTN политику маршрутизации голосовой связи сайта, который связан входящей линии связи будет использоваться для перенаправить вызов к конечной точке PSTN делегата.
    
- Для делегирования рекомендуется, руководитель и его связанных делегаты, должна находиться на том же сайте сети обычно.
    
## <a name="other-planning-considerations"></a>Другие факторы, которые следует учесть при планировании

При планировании маршрутизации на основе местоположения, необходимо учитывать последствия для следующих сценариев.
  
### <a name="disaster-recovery"></a>Аварийное восстановление

Во время отработки отказа из основного пула в резервный пул Если при восстановлении нормальная работа в основной пул, маршрутизации на основе местоположения остается Принудительная на всех элементов во время процедуры аварийного и восстановления.
  
### <a name="survivable-branch-appliance"></a>для обеспечения связи в филиалах

Настройка маршрутизации на основе расположения влияет на планирование где развертывание шлюзов, связанный со вашей для обеспечения связи в филиалах. Шлюз, связанный со ваше устройство должен находиться в том же сайте сети вашей для обеспечения связи в филиалах; в противном случае пользователей, размещенных на Survivable Branch Appliance будет запрещено для помещения исходящие вызовы, если настроена маршрутизация на основе местоположения. При глобальной сети соединение между вашей для обеспечения связи в филиалах и центрального сайта не работает, остается Принудительная ограничений на основе местоположения маршрутизации.

## <a name="capabilities-not-supported-by-location-based-routing"></a>Функции, не поддерживаемые маршрутизацией на основе расположения

Маршрутизация на основе расположения не применяется для следующих типов взаимодействия. Маршрутизация на основе расположения не применяется при взаимодействии Скайп для бизнеса конечных точек с конечными точками PSTN, с помощью этих возможностей.
  
- Конференции с телефонным подключением
    
- Входящие и исходящие вызовы ТСОП через группу ответов
    
- Парковка вызовов или преобразование вызовов ТСОП с помощью функции парковки вызовов
    
- Входящие вызовы ТСОП в службу оповещений
    
- Входящие вызовы ТСОП, преобразованные через групповой ответ на вызовы
    
Для применения правил маршрутизации на основе расположения для типов взаимодействия в приведенном ниже списке, необходимо включить маршрутизации на основе расположения для конференц-связи:
  
- Исходящие вызовы по ТСОП из конференций
    
- Эскалации из сеансов одноранговой голосовой связи к сеансам конференций с привлечением конечных точек ТСОП 
    
- Передачи после консультации с привлечением конечных точек ТСОП
    
Включение маршрутизации на основе расположения для конференц-связи, см [зависимостью от расположения для конференц-связи](http://technet.microsoft.com/library/e1acb1ba-0ed2-4abf-8a7b-1ca3049e95e3.aspx).
