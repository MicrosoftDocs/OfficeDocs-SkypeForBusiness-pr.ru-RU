# <a name="plan-for-location-based-routing-in-skype-for-business"></a>Планирование Location-Based маршрутов в Skype для бизнеса

Планирование маршрутов на основе расположения в Skype для бизнеса Server Корпоративная голосовая связь, включая взаимодействие с одновременными звонками и делегирования, а также поддерживаемые сценарии маршрутации на основе расположения.

Location-Based маршрутизов позволяет ограничить маршрутику вызовов между конечными точками VoIP и конечными точками STN в зависимости от расположения сторон в вызове. Location-Based маршрутизов — это функция управления звонками, управляющая маршрутизовами звонков в Skype для бизнеса Server. Он применяет правила авторизации вызовов, чтобы определить, могут ли вызовы перенаставляться на конечные точки УАПС или STN в зависимости от географического расположения вызываемой службы Skype для бизнеса.

Location-Based маршрутизов вводится новый набор правил, который изменяет маршрутику национальных и международных вызовов STN для предотвращения обхода платных номеров. Location-Based маршрутов обеспечивает гибкость для применения этих правил к определенным областям, шлюзам или только определенному набору пользователей.

Следующие сценарии иллюстрируют основные типы ограничений, Location-Based маршруты могут применяться:

- Исходяющие вызовы Location-Based маршруты могут применять исходяющие вызовы для исходяния на шлюз STN, расположенный в том же регионе, в котором находится вызывающий, чтобы предотвратить обход платных звонков по STN, что предотвращает исходяние вызовов на шлюз STN, расположенный в другом регионе, в котором находится вызывающий.

- Ingress calls - Location-Based Routing can prevent incoming PSTN calls to ring Skype for Business endpoints if the PSTN gateway routing the incoming call is not located in the same region as the called Skype for Business user.

- Неизвестные регионы Location-Based маршрутизации ограничивает входящие и исходяющие вызовы STN пользователями, расположенными в неизвестных расположениях (т. е. удаленных пользователей, подключающихся из Интернета или расположенных в неизвестных регионах).

- Международные регионы Location-Based маршрутиза между регионами и маршрутизализой обеспечивает маршрутику исходяющих вызовов через международные шлюзы STN, если не удается найти локальный шлюз в расположение пользователя.

## <a name="guidance-for-where-to-apply-location-based-routing"></a>Руководство по месту применения Location-Based маршрутов

Location-Based маршрут в зависимости от ситуации может применяться в расположении сетевого сайта конечной точки пользователя или на сетевом сайте шлюза STN. В этом разделе содержится руководство по Location-Based маршрутов.

### <a name="applying-location-based-routing-at-the-users-location"></a>Применение Location-Based маршрутов в расположении пользователя

Location-Based маршрутизация использует те же области сети, сайты и подсети, что и в Skype для бизнеса Server, используемом E9-1-1, CAC и обходом сервера-посредника, чтобы применить ограничения маршрутизации вызовов для предотвращения обхода платных звонков ЗВОНКОВ. Местоположение пользователя определяется IP-подсетей конечных точек Skype для бизнеса, из которые подключаются пользователи. Каждая IP-подсеть связана с сетевым сайтом, которые объединяются в области сети, определенные администратором. Location-Based маршруты применяются на основе сетевого сайта пользователя.

Location-Based маршрутов применяются для каждого сетевого узла, то есть данный набор правил будет применяться ко всем конечным точкам, включенным для маршрутной Location-Based, расположенным на одном сетевом сайте. Администраторы могут применять Location-Based к сетевым сайтам, которые этого требуют.

Политики маршрутизов голосовой связи можно определить на уровне узла сети, чтобы определить определенный шлюз STN, который должен использоваться всеми пользователями, расположенными на сетевом сайте, для вызова номеров телефонов STN. Такие политики маршрутинга голосовых вызовов будут иметь приоритет над маршрутией, определенной политикой голосовой связи пользователя, если пользователь расположен на сетевом сайте с включенной маршрутицией Location-Based, и предотвратит маршрутику вызовов через другие шлюзы STN, для них включена Location-Based Routing. Когда пользователь Skype для бизнеса звонит по STN, политика голосовой почты пользователя определяет, может ли пользователь иметь право на его звонок. Если политика голосовой почты пользователя позволяет пользователю звонить, Location-Based маршруты определяют шлюз STN, из которого должен перенаступать вызов. Location-Based маршрутивка делает это определение в зависимости от расположения пользователя.

Расположение пользователя можно классифицовать следующими способами:

- Пользователь находится на известном сетевом сайте, на который включена Location-Based Routing, а его номер DID (Direct Inward Dial) завершается на шлюзе STN, размещенном на том же сетевом сайте (то есть офисе). Маршрутизов исходящие вызовы будут проходить через политику маршрутизов голосовой связи на сетевом сайте, в котором находится пользователь. Входящие вызовы STN, входящие для пользователя, перенауются на конечные точки, расположенные на том же сетевом сайте, что и шлюз STN.

- Пользователь находится на известном сетевом сайте, который отличается от сетевого сайта, на котором расположен шлюз STN. (то есть пользователь переехав в другой корпоративный офис). Маршруты исходящие вызовы будут использовать политику маршрутизов голосовой связи на сетевом сайте, в котором находится пользователь. Входящие вызовы STN для пользователя не будут перенаходить на конечные точки, расположенные на разных сайтах, чем шлюз STN, чтобы предотвратить обход платных звонков по STN.

- Если пользователь находится на сетевом сайте, который неизвестен развертыванию Skype для бизнеса Server, маршрутизируются исходящие вызовы на основе политики голосовой связи, назначенной пользователю шлюзам STN, не привязанным к Location-Based маршруту. Входящие вызовы STN не будут перенаходить на конечные точки, расположенные на неизвестных сетевых сайтах, чтобы запретить обход платных звонков ЗВОНКОВ.

### <a name="applying-location-based-routing-at-the-pstn-gateways-location"></a>Применение Location-Based маршрутов в расположении шлюза STN

Для звонков, маршрутизовающихся через шлюзы STN и УАПС, может потребоваться Location-Based маршрутов в зависимости от расположения таких систем. Location-Based маршрутную маршрутику можно включить с разной детализацией на уровне магистрали.

Location-Based маршрутике в магистрали вводится следующий набор правил:

- Если Location-Based маршрутизов включена для каждого магистрали, правила, определенные для этой магистрали, будут применяться только к вызовам, маршрутным через эту магистраль.

- Чтобы избежать обхода платных звонков ЗВОНКОВ, при котором вызовы происходят с сетевого сайта, который отличается от сетевого сайта, на котором расположен шлюз STN, в маршруте Location-Based вводится связь сетевого сайта с заданной магистралью. Таким образом определяется сетевой сайт, который позволяет перенаупределять вызовы в заданную магистраль.

Магистрали можно включить для Location-Based маршрутов двумя способами:

- Магистраль определяется для шлюза STN, который переадресовает вызовы в STN. Входящие вызовы, маршрутизовамые магистралью этого типа, будут маршрутизовы в конечные точки, расположенные на том же сетевом сайте, что и магистраль.

- Магистраль определяется для одноранговой сети сервера-посредника, которая не передает вызовы в PSTN и служит для обслуживания пользователей устаревшими телефонами в статических расположениях (например, телефонах УАЦ). Для этой конкретной конфигурации все входящие вызовы, маршрутизовались магистралью этого типа, будут считаться исходящая из того же сетевого узла, что и магистраль. Вызовы от пользователей УАПС будут иметь то же Location-Based маршрутизов, что и для пользователей Skype для бизнеса, расположенных на том же сетевом сайте, что и магистраль. Если две системы УАКС, расположенные на отдельных сетевых узлах, подключены через Skype для бизнеса Server, Location-Based Routing will allow routing from one PBX endpoint in one network site to another PBX endpoint in the other network site. Этот сценарий не будет заблокирован Location-Based маршрутов. В дополнение к этому сценарию и аналогично пользователю Skype для бизнеса в одном расположении конечные точки, подключенные к одноранговому серверу-посреднику с такой конфигурацией, смогут принимать или принимать вызовы на другой узел сервера-посредника, который не маршрутит вызовы в PSTN (то есть конечную точку, подключенную к другой УАО), независимо от сетевого сайта, с которым связан одноранговой сервер-посредник. Все входящие вызовы, исходящие вызовы, переключение вызовов и переададации вызовов с использованием конечных точек STN будут подчиняться маршруту на основе расположения, чтобы использовать только шлюзы STN, которые определены как локальные для такого одноранговых серверов-посредников.

### <a name="incoming-calls"></a>Входящие вызовы

Маршрутка входящих вызовов для пользователей, для которых включена Location-Based маршрутизов, зависит от расположения конечной точки пользователя. На маршрутку входящих вызовов влияет следующий способ. Если у пользователя есть входящий вызов в конечную точку, расположенную на сетевом сайте с включенной маршрутией Location-Based и конечная точка расположена на том же сетевом сайте, что и шлюз STN, вызов будет переназначен. Если у пользователя есть входящий вызов в конечную точку, расположенную на сетевом сайте с включенной маршрутией Location-Based, а конечная точка расположена на сетевом сайте, который отличается от шлюза STN, вызов не будет маршрутизован. Если у пользователя нет конечных точек, расположенных на том же сетевом сайте, что и шлюз STN, из которых происходит входящий вызов, входящий вызов будет перенаправл непосредственно в голосовую почту пользователя, а вызываемой стороне будет отправлено уведомление о пропущенном звонке.

Параметры переадантки звонков пользователя, для пользователя с включенной поддержкой Location-Based маршруты, будут по-прежнему применяться, однако переададантные вызовы будут налагаться на Location-Based маршрутизов пользователя.

### <a name="delegation"></a>Делегирование

На возможности делегирования в Skype для бизнеса влияет Location-Based маршруты следующим образом:

- Когда делегат с включенной поддержкой Location-Based маршрутизации помещает вызов от имени руководителя, для авторизации вызова используется политика голосовой почты делегата, а для маршрутизации вызова будет использоваться политика маршрутизации голосовых вызовов на сайте делегата.

- Для входящих вызовов STN руководителю применяются те же правила, которые применяются для переадправления вызовов или одновременных звонков, как описано в темах о переадправлении вызовов, переадправлении и одновременных звонках.

- Когда делегат задает конечную точку STN в качестве одновременного звонка, для входящих вызовов диспетчеру будет использоваться политика маршрутирования голосовой связи сайта, связанного с входящий магистралью, для маршрутирования вызова в конечную точку STN делегата.

- Для делегирования рекомендуется, чтобы руководитель и связанные с ним делегаты обычно находились на одном сетевом сайте.

## <a name="other-planning-considerations"></a>Другие вопросы планирования

При планировании Location-Based маршрутов следует учитывать влияние на следующие сценарии.

### <a name="disaster-recovery"></a>Аварийное восстановление

Во время отсеки основного пула в резервный пул, а также при восстановлении нормальной работы в основном пуле маршрут Location-Based всегда будет применяться во время аварийного восстановления.

### <a name="survivable-branch-appliance"></a>Устройство для обеспечения связи в филиалах

Настройка Location-Based влияет на планирование развертывания шлюзов, связанных с устройствами для связи в филиалах. Шлюз, связанный с SBA, должен быть расположен на том же сетевом сайте, что и устройство для связи в филиалах; в противном случае пользователи, которые были на устройстве для survivable Branch Appliance, не смогут делать исходящие вызовы, Location-Based настроена маршрутка. Если подключение WAN между устройством для связи в филиалах и центральным сайтом не подключено, Location-Based маршрутизируемые ограничения по-прежнему применяются.

## <a name="capabilities-not-supported-by-location-based-routing"></a>Возможности, не поддерживаемые Location-Based маршрутии

Location-Based маршруты не применяются к следующим типам взаимодействия. Location-Based маршруты не применяются, когда конечные точки Skype для бизнеса взаимодействуют с конечными точками STN с помощью этих возможностей.

- Конференц-телефонное подмостки STN для конференций

- Входящие и исходяющие вызовы STN через группу ответа

- Парковка вызовов или искомый вызовов PSTN через парковку вызовов

- Входящие вызовы STN в службу объявлений

- Входящие вызовы STN, полученные с помощью группового вызова

Чтобы применить Location-Based маршрутов к типам взаимодействий в следующем списке, необходимо включить Location-Based маршрутику для проведения Location-Based.

- Dial-out STN from conferences

- Эскалации из одноранговых аудиоконферентов в аудиоконференцию с использованием конечных точек STN

- Переключение на консультации с использованием конечных точек STN

Чтобы включить Location-Based для проведения conferencing, см. "Маршруты на основе расположения" для проведения [conferencing.](https://technet.microsoft.com/library/e1acb1ba-0ed2-4abf-8a7b-1ca3049e95e3.aspx)
