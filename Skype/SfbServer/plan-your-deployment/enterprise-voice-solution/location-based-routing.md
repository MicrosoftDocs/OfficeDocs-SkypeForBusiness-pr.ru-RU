# <a name="plan-for-location-based-routing-in-skype-for-business"></a>Планирование Location-Based маршрутов в Skype для бизнеса

Планирование маршрутов на основе расположения в Skype для бизнеса Server Корпоративная голосовая связь, включая взаимодействие с одновременным звоном и делегирования, а также поддерживаемые сценарии маршрутирования на основе расположения.

Location-Based маршрутизов позволяет ограничить маршрутику вызовов между конечными точками VoIP и конечными точками PSTN в зависимости от расположения сторон в вызове. Location-Based маршрутизов — это функция управления вызовами, которая управляет маршрутизоваемой Skype для бизнеса Server. Он применяет правила авторизации вызовов, чтобы определить, можно ли переназначить вызовы в конечные точки PBX или PSTN в зависимости Skype для бизнеса географического расположения вызываемой точки.

Location-Based маршрутивка представляет новый набор правил, который изменяет маршрутику национальных и международных вызовов PSTN для предотвращения обхода платных звонков. Location-Based маршрутивка обеспечивает гибкость для охвата этих правил определенными регионами, определенными шлюзами или только определенным набором пользователей.

В следующих сценариях иллюстрируются основные типы ограничений, Location-Based маршрутивка может применяться:

- Egress вызовов — Location-Based маршрутинг может обеспечить выполнение исходяющих вызовов, чтобы отойти на шлюз PSTN, расположенный в том же регионе, где вызывается, чтобы предотвратить обход платных вызовов PSTN, что не позволяет вызовам отходить к шлюзу PSTN, расположенному в другом регионе, как вызывающий.

- Ingress calls - Location-Based Routing can prevent incoming PSTN calls to ring Skype для бизнеса endpoints if the PSTN gateway routing the incoming call is not located in the same region as the called Skype для бизнеса user.

- Неизвестные регионы Location-Based маршрутизации ограничивают входящие и исходяющие вызовы PSTN пользователям, расположенным в неустановленных местах (например, удаленных пользователей, подключающихся из Интернета или расположенных в неизвестных регионах).

- Международные регионы — Location-Based маршрутизака обеспечивает маршрутику исходяющих вызовов через международные шлюзы PSTN, если не удается найти шлюз, локальный для расположения пользователя.

## <a name="guidance-for-where-to-apply-location-based-routing"></a>Руководство по применении Location-Based маршрутов

Location-Based маршрутику в зависимости от ситуации можно применять в расположении конечной точки сети пользователя или в расположении сетевого сайта шлюза PSTN. В этом разделе приводится руководство по Location-Based методов маршрутной маршрутики.

### <a name="applying-location-based-routing-at-the-users-location"></a>Применение Location-Based маршрутов в расположении пользователя

Location-Based маршрутизация использует те же сетевые регионы, сайты и подсети, что и в Skype для бизнеса Server, используемых E9-1-1, CAC и Media Bypass для применения ограничений маршрутизации вызовов для предотвращения обхода платных платы PSTN. Местоположение пользователя определяется подсетей IP конечной точки Skype для бизнеса пользователя. Каждая подсеть IP связана с сетевым сайтом, который агрегируется в сетевые регионы, определенные администратором. Location-Based маршрутивка выполняется на основе сетевого сайта пользователя.

Location-Based маршрутных маршрутов применяются на основе каждого сетевого сайта, что означает, что данный набор правил будет применяться ко всем конечным точкам, включенным для Location-Based маршрутов, расположенных в пределах одного сетевого сайта. Администраторы могут применять Location-Based маршрутику к сетевым сайтам, которые этого требуют.

Политики маршрутизов голосовой связи можно определить на основе каждого сетевого сайта для определения определенного шлюза PSTN, который должен использоваться всеми пользователями, расположенными на сетевом сайте, для вызова номеров телефонов PSTN. Такие политики маршрутизов голосовой маршрутики будут иметь приоритет над маршрутией, определенной голосовой политикой пользователя, когда пользователь находится в сетевом сайте, включенном для Location-Based маршрутинга, и это предотвратит маршрутику вызовов через другие шлюзы PSTN, которые включены для Location-Based маршрутинга. Когда Skype для бизнеса помещает вызов PSTN, голосовая политика пользователя определяет, можно ли разрешить пользователю разместить вызов. Если голосовая политика пользователя позволяет пользователю разместить вызов, Location-Based маршрутная маршрутия определяет, из каких шлюзов PSTN следует отойти. Location-Based маршрутивка делает это определение в зависимости от расположения пользователя.

Расположение пользователя можно классифицизировать следующим образом:

- Пользователь расположен в известном сетевом сайте, включенном для Location-Based маршрутизбирки, а его номер DID (Direct Inward Dial) прекращается на шлюзе PSTN, размещенном на том же сетевом сайте (например, в офисе). Маршрутивка исходящие вызовы будет проходить через политику маршрутизов голосовой связи сетевого сайта, на котором находится пользователь. Входящие вызовы PSTN для пользователя переназначаются в конечные точки, расположенные в том же сетевом сайте, что и шлюз PSTN.

- Пользователь расположен на известном сетевом сайте, который отличается от сетевого сайта, на котором расположен шлюз PSTN. (например, пользователь отправился в другой корпоративный офис). Маршрутизов исходящие вызовы будут использовать политику маршрутизов голосовой связи сетевого сайта, на котором находится пользователь. Входящие вызовы PSTN пользователю не будут переназначены в конечные точки, расположенные на разных сайтах, чем шлюз PSTN, чтобы предотвратить обход платных звонков PSTN.

- Если пользователь находится на сетевом сайте, неизвестном развертыванию Skype для бизнеса Server, маршрутиздание исходящие вызовы будет основываться на голосовой политике, назначенной пользователю шлюзам PSTN, не связанным с Location-Based маршрутизов. Входящие вызовы PSTN не будут переназначены в конечные точки, расположенные на неизвестных сетевых сайтах, чтобы предотвратить обход платных звонков PSTN.

### <a name="applying-location-based-routing-at-the-pstn-gateways-location"></a>Применение Location-Based маршрутов в расположении шлюза PSTN

Вызовы, маршрутизовавшись через шлюзы PSTN и PBX, могут потребовать Location-Based ограничений маршрутизов в зависимости от расположения таких систем. Location-Based маршрутику можно включить на уровне детализации на основе магистрали.

Location-Based маршрутике вводится следующий набор правил при включении в магистрали:

- Если Location-Based маршрутная маршрутия включена на основе магистрали, правила, определяющие этот магистраль, будут применяться только к вызовам, проложенным через этот магистраль.

- Чтобы предотвратить обход платных звонков ПСП, где вызовы происходят с сетевого сайта, отличаемого от сетевого сайта, на котором расположен шлюз PSTN, Location-Based маршрутирование вводит связь сетевого сайта с заданным магистральом. Это определяет сетевой сайт, который позволяет перенапределять вызовы в определенный магистраль.

Магистрали можно включить для Location-Based маршрутов двумя способами:

- Магистраль определяется для шлюза PSTN, который отгрешает вызовы в PSTN. Входящие вызовы, маршрутизовамые магистралью этого типа, будут переназначатся только в конечные точки, расположенные в том же сетевом сайте, что и магистраль.

- Магистраль определяется для одноранговых серверов-посредников, которые не отгребает вызовы в ПСПС и службы пользователей с устаревшими телефонами в статичных расположениях (например, PBX-телефонах). Для этой конкретной конфигурации все входящие вызовы, маршрутизовавшись магистралью этого типа, будут считаться исходя из того же сетевого сайта, что и магистраль. Вызовы пользователей PBX будут иметь те же Location-Based, что и Skype для бизнеса пользователей, расположенных в том же сетевом сайте, что и магистраль. Если две системы PBX, расположенные на отдельных сетевых сайтах, подключены через Skype для бизнеса Server, Location-Based маршрутивка позволит маршрутить маршрутику из одной конечной точки PBX в одном сетевом сайте в другую конечную точку PBX на другом сетевом сайте. Этот сценарий не будет заблокирован Location-Based маршрутивка. Помимо этого сценария и аналогично пользователю Skype для бизнеса в том же расположении конечные точки, подключенные к одноранговой точке сервера-посредника с этой конфигурацией, смогут принимать вызовы в другой одноранговой сервер-посредник, который не передает вызовы в PSTN (то есть конечную точку, подключенную к другому PBX) независимо от сетевого сайта, к которому связан одноранговой сервер-посредник. Все входящие вызовы, исходящие вызовы, переводы вызовов и переадновки вызовов с конечными точками PSTN будут подлежать маршрутной маршрутике на основе расположения, чтобы использовать только шлюзы PSTN, которые определяются как локальные для таких одноранговых серверов-посредников.

### <a name="incoming-calls"></a>Входящие вызовы

Маршрутка входящих звонков пользователям, включенным для Location-Based маршрутов, зависит от расположения конечной точки пользователя. Маршрутия входящих вызовов влияет следующим образом. Если у пользователя есть входящий вызов в конечную точку, расположенную в сетевом сайте с включенной Location-Based маршрутизов, а конечная точка расположена в том же сетевом сайте, что и шлюз PSTN, вызов будет маршрутизован. Если у пользователя есть входящий вызов в конечную точку, расположенную в сетевом сайте с включенной Location-Based маршрутизов, а конечная точка расположена на другом сетевом сайте, чем шлюз PSTN, вызов не будет маршрутизован. Если у пользователя нет конечных точек, расположенных в том же сетевом сайте, что и шлюз PSTN, откуда исходит входящий вызов, входящий вызов будет перенаправлен непосредственно на голосовую почту пользователя, а вызываемой стороне будет отправлено уведомление о пропущенном вызове.

Параметры переададки звонков пользователя, включенного для Location-Based маршрутинга, будут по-прежнему применяться, однако переададные вызовы будут Location-Based маршрутизов пользователя.

### <a name="delegation"></a>Делегирование

Возможности делегирования в Skype для бизнеса влияют на Location-Based маршрутику следующим образом:

- Если делегат включен для Location-Based маршрутизации помещает вызов от имени руководителя, политика голосовой политики делегата используется для авторизации вызова, а политика маршрутизации голосовой маршрутизации сайта делегата будет использоваться для маршрутизации вызова.

- Для входящих вызовов PSTN менеджеру применяются те же правила, которые применяются для переададки вызовов или одновременного звонка, как описано в темах Передачи вызовов, переадправления и одновременного звонка.

- Когда делегат задает конечную точку PSTN как одновременную цель кольца, для входящих вызовов диспетчеру будет использоваться политика маршрутирования голосовой связи сайта, связанного с входящий магистралью, для маршрутирования вызова в конечную точку PSTN делегата.

- Для делегирования рекомендуется, чтобы диспетчер и связанные с ним делегаты обычно располагались на одном сетевом сайте.

## <a name="other-planning-considerations"></a>Другие соображения планирования

При планировании Location-Based маршрутов следует учитывать влияние на следующие сценарии.

### <a name="disaster-recovery"></a>Аварийное восстановление

При сбойе из основного пула в резервный пул, а также при восстановлении обычных операций в основном пуле Location-Based маршрутизация всегда выполняется во время аварийной и аварийной процедуры восстановления.

### <a name="survivable-branch-appliance"></a>Устройство для обеспечения связи в филиалах

Настройка Location-Based маршрутов влияет на планирование развертывания шлюзов, связанных с устройствами для ветвей. Шлюз, связанный с вашей SBA, должен располагаться в том же сетевом сайте, что и устройство для выживших филиалов; в противном случае пользователям, устроившимся на вашем устройстве для выживших филиалов, не будет разрешено Location-Based настройки маршрутной маршрутки. Когда связь WAN между устройством для выживших филиалов и центральным сайтом не выполняется, Location-Based маршрутивка сохраняется.

## <a name="capabilities-not-supported-by-location-based-routing"></a>Возможности, не поддерживаемые Location-Based маршрутивка

Location-Based маршрутивка не применяется к следующим типам взаимодействий. Location-Based маршрутивка не выполняется, Skype для бизнеса конечные точки взаимодействуют с конечными точками PSTN с помощью этих возможностей.

- PSTN dial-in to conferences

- Входящие и исходяющие вызовы PSTN через группу реагирования

- Парк вызовов или истреблять вызовы PSTN через Call Park

- Входящие вызовы PSTN в службу объявлений

- Входящие вызовы PSTN, полученные с помощью группового вызова

Чтобы применить Location-Based маршрутику к типам взаимодействий в следующем списке, необходимо включить Location-Based маршрутику для конференций:

- PSTN dial-out from conferences

- Эскалации от одноранговых аудиоконферентов до конференций с участием конечных точек PSTN

- Консультативные переводы с конечными точками PSTN

Чтобы включить Location-Based маршрутику для conferencing, см. в этой ленте маршрутивка на основе расположения для [конференцинга.](/previous-versions/office/lync-server-2013/lync-server-2013-location-based-routing-for-conferencing)
