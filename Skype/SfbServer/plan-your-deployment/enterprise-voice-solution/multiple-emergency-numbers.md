# <a name="plan-for-multiple-emergency-numbers-in-skype-for-business-server"></a>Планирование нескольких номеров экстренной помощи в Skype для бизнеса Server
 
В этой статье рассказывается о том, как планировать несколько номеров для экстренного реагирования в Skype для бизнеса Server.
  
Skype для бизнеса Server теперь поддерживает конфигурацию нескольких номеров экстренного реагирования для клиента. Несколько номеров для экстренного реагирования — это новая функция, представленная в накопительном обновлении за июнь 2016. Тогда как в США существует единый номер вызова экстренных служб, 911, во многих странах поддерживается несколько экстренных номеров. Например, Великобритания — это номер телефона, который можно использовать как 999, так и для Великобритании и 112 — номер экстренного предложения Европейского союза. 
  
Эта функция также удобна для поставщиков услуг здравоохранения в США, которым требуется поддержка роуминга для нескольких экстренных номеров с сигналом о необходимости реанимации пациента.
  
## <a name="multiple-emergency-numbers-and-location-policies"></a>Несколько номеров экстренных служб и политики расположения

Настройка номеров экстренных служб выполняется путем создания политик расположения, которые определяют реализацию экстренных вызовов. Для определения номера, который является вызовом экстренной помощи, например 911 в США, используется политика местоположения. 999 и 112 в Великобритании. Политика расположения определяет, включена ли для пользователя поддержка вызовов экстренных служб, а также режим таких вызовов. Кроме того, можно определить, как осуществлять маршрутизацию вызовов и следует ли отправлять автоматическое уведомление в службу безопасности организации.
  
Дополнительные сведения об определении и изменении политики расположения можно найти в разделе [политики расположения планов для Skype для бизнеса Server](location-policies.md) и [Создание политик местоположений в Skype для бизнеса Server](../../deploy/deploy-enterprise-voice/create-location-policies.md). В этих разделах описаны основные понятия политики местоположения; Однако вы должны выполнить инструкции по [настройке нескольких номеров для экстренной помощи в Skype для бизнеса](../../deploy/deploy-enterprise-voice/configure-multiple-emergency-numbers.md) , чтобы настроить несколько номеров для экстренного реагирования.
  
При планировании нескольких номеров экстренных служб необходимо учитывать следующее.
  
- С помощью накопительного обновления от июня 2016 г. Вы можете указать до 5 номеров Экстренн для заданной политики местоположения. Благодаря накопительному обновлению 2016 за Ноябрь, этот номер возрастает до 100.
    
    > [!NOTE]
    > Если вы еще не обновили накопительный пакет обновления за ноябрь 2016 ноября, ознакомьтесь с [обновлением для Skype для бизнеса Server 2015](https://support.microsoft.com/en-us/help/3061064/updates-for-skype-for-business-server-2015). 
  
- Для каждого номера для экстренного реагирования вы можете указать несколько масок экстренных номеров, которые уникальны для определенной политики местоположения.
    
    Маска набора — это число, которое вы хотите переводить в значение номера экстренного набора номера при наборе номера. Например, предположим, что в этом поле введено значение 212, а в поле «номер экстренного набора» — значение 911. При набираемом пользователем 212 номер будет преобразован в 911. Это позволяет использовать дополнительные номера для экстренного реагирования (например, если кто-то из страны или региона с другим номером для экстренного реагирования пытается набрать номер страны или региона, а не номер для страна или регион, в которых они находятся в настоящее время). Вы можете определить несколько масок для экстренного набора номера, разделяя их точками с запятой. Например, 212; 414. Максимальная длина строки для маски набора составляет 100 символов. Каждый символ должен быть цифрой от 0 до 9.
    
- Для каждой политики расположения задается отдельный режим работы с ТСОП, который определяет, какой маршрут голосовых вызовов используется для перенаправления экстренных вызовов от клиентов с помощью этой политики. Можно задать уникальный маршрут на каждый экстренный номер.
    
- Если для политики определены оба параметра, EmergencyNumbers и DialString и клиент поддерживает несколько номеров экстренных служб, то приоритет имеет номер экстренной службы. Если клиент не поддерживает несколько номеров экстренных служб, используется строка набора номера.
    
- Сведения о том, какие клиенты Skype для бизнеса и Lync поддерживают получение нескольких номеров для экстренных случаев, масок звонков и использование коммутируемых телефонных сетей (PSTN), можно найти в разделе [Поддержка клиентов](multiple-emergency-numbers.md#BKMK_Clients).
    
> [!NOTE]
> Вы не можете настроить несколько номеров для экстренного реагирования с помощью панели управления "Skype для бизнеса". Вы должны использовать PowerShell для настройки нескольких номеров для экстренного реагирования. 
  
Перед настройкой нескольких номеров экстренных служб необходимо учитывать следующее.
  
- Чтобы настроить несколько номеров для экстренного реагирования, необходимо использовать командлет New-Ксемерженцинумбер, и вы должны определить политики расположения, которые поддерживают несколько номеров для экстренных случаев, указав параметр Емерженцинумберс с помощью командлетов [New-кслокатионполици](https://docs.microsoft.com/powershell/module/skype/new-cslocationpolicy?view=skype-ps) и [Set-кслокатионполици](https://docs.microsoft.com/powershell/module/skype/set-cslocationpolicy?view=skype-ps) .
    
- 	При наличии существующих номеров, заданных с помощью командлета Set-CsLocationPolicy или New-CsLocationPolicy с параметрами EmergencyDialString и EmergencyDialMask, значения, указанные с помощью параметра EmergencyNumbers, будут иметь преимущество над старыми значениями. Таким образом, значения параметров EmergencyDialString и EmergencyDialMask будут игнорироваться.
    
- Если у вас есть номера, определенные с помощью командлета Set-Кслокатионполици или New-Кслокатионполици с параметрами Емерженцидиалстринг и Емерженцидиалмаск, *и вы не настраиваете новые номера для экстренного* реагирования, существующие номера будут использоваться по-прежнему.
    
- Для работы нескольких номеров экстренных служб используемые версии клиентов должны поддерживать эту новую функцию. Клиенты прежних версий будут использовать старые значения, указанные с помощью командлета Set-CsLocationPolicy или New-CsLocationPolicy с параметрами EmergencyDialString и EmergencyDialMask. 
    
- Если пользователь набирает номер, который соответствует строке набора, маска набора не требуется. Например, если пользователь набирает 911, строка набора — 911 и маска не требуется. 
    
Дополнительные сведения о настройке нескольких номеров для экстренных случаев можно найти [в разделе Настройка нескольких номеров экстренного реагирования в Skype для бизнеса](../../deploy/deploy-enterprise-voice/configure-multiple-emergency-numbers.md).

 **США** — не требуется наличие нескольких номеров для экстренного реагирования. В США вы используете старую строку набора экстренных номеров и конфигурацию масок набора.
  
 **Американская для больницы** — требуется не маска "450". Для клиентов, которые еще не поддерживают несколько номеров для экстренного реагирования, вы можете использовать старую строку набора номера для экстренного звонка и конфигурацию масок звонков. Для клиентов, которые поддерживают несколько номеров для экстренного реагирования, вы можете указать номер для экстренного реагирования как "911", так и "450" вместо масок 450.
  
 **Лондон** — для клиентов, которые еще не поддерживают несколько номеров для экстренного реагирования, вы можете использовать старую строку набора номера для экстренного звонка и конфигурацию масок звонков. Для клиентов, которые поддерживают несколько номеров для экстренного реагирования, вы можете указать номер для экстренного реагирования как "999", так и "112" с масками для каждого из них.
  
 **Индия** — все развернутые клиенты поддерживают несколько номеров для экстренного реагирования. В Индии вам нужно настроить только несколько номеров для экстренной помощи.
