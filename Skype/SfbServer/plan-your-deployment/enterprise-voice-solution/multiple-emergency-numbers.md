# <a name="plan-for-multiple-emergency-numbers-in-skype-for-business-server"></a>Планирование для нескольких аварийного номера в Скайп Business Server
 
В данном разделе приведены в этой статье описывается планирование нескольких аварийного номера в Скайп для Business Server.
  
Скайп для Business Server теперь поддерживает конфигурации нескольких аварийного номера для клиента. Несколько аварийного номера — это новая возможность, введенные в 2016 июня накопительного обновления. Тогда как в США существует единый номер вызова экстренных служб, 911, во многих странах поддерживается несколько экстренных номеров. Соединенное Королевство, например поддерживает 999, относящиеся к Соединенное Королевство номер экстренного вызова и 112 номер экстренного вызова для Европейского союза. 
  
Эта функция также удобна для поставщиков услуг здравоохранения в США, которым требуется поддержка роуминга для нескольких экстренных номеров с сигналом о необходимости реанимации пациента.
  
## <a name="multiple-emergency-numbers-and-location-policies"></a>Несколько номеров экстренных служб и политики расположения

Настройка номеров экстренных служб выполняется путем создания политик расположения, которые определяют реализацию экстренных вызовов. Использовать политику расположения для определения, какой номер составляет экстренных вызовов — например, 911 в США. 999 и 112 в Великобритании. Политика расположения определяет, включена ли для пользователя поддержка вызовов экстренных служб, а также режим таких вызовов. Кроме того, можно определить, как осуществлять маршрутизацию вызовов и следует ли отправлять автоматическое уведомление в службу безопасности организации.
  
Дополнительные сведения об определении и изменение политики расположения можно [планирование политики расположения для Скайп для Business Server](location-policies.md) и [Создание политик расположения в Скайп для Business Server](../../deploy/deploy-enterprise-voice/create-location-policies.md). Этих разделах описываются основные понятия о политик расположения; Тем не менее необходимо следуйте инструкциям в [настроить несколько аварийного номера в Скайп для бизнеса](../../deploy/deploy-enterprise-voice/configure-multiple-emergency-numbers.md) , чтобы настроить несколько аварийного номера.
  
При планировании нескольких номеров экстренных служб необходимо учитывать следующее.
  
- С помощью 2016 июня накопительные пакеты обновления, можно определить до 5 аварийного номера для политики данного расположения. С помощью 2016 ноября накопительные пакеты обновления, этот номер увеличивается до 100.
    
    > [!NOTE]
    > Если вы еще не был обновлен для 2016 ноября накопительные пакеты обновления, посетите страницу [обновления для Скайп для Business Server 2015](https://support.microsoft.com/en-us/help/3061064/updates-for-skype-for-business-server-2015). 
  
- Для каждого аварийного номера можно указать ноль или больше экстренного вызова масок, которые являются уникальными для политики данного расположения.
    
    Маска номера — номер, который требуется перевести в значение номер экстренного вызова при его набираемым. Например предположим, введите значение 212 в этом поле и поле номер экстренного вызова имеет значение 911. При вызове 212 номер будут переведены 911. Это позволяет альтернативного аварийного номера набираемых и по-прежнему иметь связи с аварийными службами вызова (например, в том случае, если кто-либо из страны или региона с другой номер экстренного вызова пытается наберите номер, а не на основе номер для страны или региона Страна или регион, в которых они находятся в настоящее время). Вы можете определить несколько масок экстренного вызова значения, разделив их точкой с запятой. Например 212; 414. Ограничение строки по телефонным маска — 100 символов. Каждый символ должен быть цифры от 0 до 9.
    
- Для каждой политики расположения задается отдельный режим работы с ТСОП, который определяет, какой маршрут голосовых вызовов используется для перенаправления экстренных вызовов от клиентов с помощью этой политики. Можно задать уникальный маршрут на каждый экстренный номер.
    
- Если для политики определены оба параметра, EmergencyNumbers и DialString и клиент поддерживает несколько номеров экстренных служб, то приоритет имеет номер экстренной службы. Если клиент не поддерживает несколько номеров экстренных служб, используется строка набора номера.
    
- Сведения о которых Скайп для бизнес-деятельности и Lync клиенты поддерживать получение нескольких аварийного номера, наберите маски и режимы работы с телефонной сети (общего пользования PSTN) содержатся в разделе [поддержки клиентов](multiple-emergency-numbers.md#BKMK_Clients).
    
> [!NOTE]
> Несколько аварийного номера нельзя настроить с помощью Скайп для панели управления бизнеса. Необходимо использовать PowerShell для настройки нескольких аварийного номера. 
  
Перед настройкой нескольких номеров экстренных служб необходимо учитывать следующее.
  
- Чтобы настроить несколько аварийного номера, необходимо использовать командлет New-CsEmergencyNumber и необходимо определить политик расположения, которые поддерживают более одного номер экстренного вызова путем указания параметра EmergencyNumbers с [Командлет New-CsLocationPolicy](https://docs.microsoft.com/powershell/module/skype/new-cslocationpolicy?view=skype-ps) и Командлеты [Set-CsLocationPolicy](https://docs.microsoft.com/powershell/module/skype/set-cslocationpolicy?view=skype-ps) .
    
- 	При наличии существующих номеров, заданных с помощью командлета Set-CsLocationPolicy или New-CsLocationPolicy с параметрами EmergencyDialString и EmergencyDialMask, значения, указанные с помощью параметра EmergencyNumbers, будут иметь преимущество над старыми значениями. Таким образом, значения параметров EmergencyDialString и EmergencyDialMask будут игнорироваться.
    
- Если у вас есть существующих телефонов, определенные с помощью командлет New-CsLocationPolicy или Set-CsLocationPolicy с параметрами EmergencyDialString и EmergencyDialMask, *и не следует настраивать новые аварийного номера* , существующих номеров будет продолжать использовать.
    
- Для работы нескольких номеров экстренных служб используемые версии клиентов должны поддерживать эту новую функцию. Клиенты прежних версий будут использовать старые значения, указанные с помощью командлета Set-CsLocationPolicy или New-CsLocationPolicy с параметрами EmergencyDialString и EmergencyDialMask. 
    
- Если пользователь набирает номер, который соответствует строке набора, маска набора не требуется. Например, если пользователь набирает 911, строка набора — 911 и маска не требуется. 
    
Дополнительные сведения о настройке нескольких аварийного номера в статье [Configure нескольких аварийного номера в Скайп для бизнеса](../../deploy/deploy-enterprise-voice/configure-multiple-emergency-numbers.md).

 **Соединенные Штаты** — не является обязательным для нескольких аварийного номера. В Соединенных Штатах Америки используйте старой конфигурации аварийного номера строки и маска номера.
  
 **Больница США** — требуется не для скрытие «450». Для клиентов, которые еще не поддерживает несколько аварийного номера можно использовать старой конфигурации аварийного номера строки и маска номера. Для клиентов, которые поддерживают несколько аварийного номера вы можете определить аварийного номера для «911» и «450» вместо маскирование 450.
  
 **Лондон** — для клиентов, которые еще не поддерживает несколько аварийного номера, можно использовать старой конфигурации аварийного номера строки и маска номера. Для клиентов, которые поддерживают несколько аварийного номера можно определить аварийного номера для «999» и «112» с помощью масок для каждого из них.
  
 **Индия** — все развернутые клиенты поддерживают несколько аварийного номера. В Индии вам необходимо настроить только несколько аварийного номера.
