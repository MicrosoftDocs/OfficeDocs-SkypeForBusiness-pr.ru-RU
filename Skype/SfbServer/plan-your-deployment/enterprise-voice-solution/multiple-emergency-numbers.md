# <a name="plan-for-multiple-emergency-numbers-in-skype-for-business-server"></a>Планирование нескольких номеров экстренных служб в Skype для бизнеса Server
 
В этом разделе рассказывается, как планировать несколько номеров экстренных служб в Skype для бизнеса Server.
  
Skype для бизнеса Server теперь поддерживает настройку нескольких номеров экстренных служб для клиента. Несколько номеров для экстренных служб — это новая функция, появившаяся в накопительном пакете обновления за июнь 2016 года. Хотя у США есть один номер для экстренного реагирования, 911, многие страны поддерживают несколько номеров экстренных служб. Соединенное Королевство, например, поддерживает как 999, так и номер экстренного реагирования, характерный для Великобритании, и 112, номер экстренного союза для Европейского союза. 
  
Эта функция также полезна для поставщиков услуг здравоохранения в США, которым требуется поддержка роуминга для нескольких синих номеров экстренного кода.
  
## <a name="multiple-emergency-numbers-and-location-policies"></a>Несколько номеров экстренных служб и политик расположения

Вы настраиваете экстренные вызовы, создавая политики расположения, которые определяют, как будут реализовываться экстренные вызовы. Используйте политику расположения, чтобы определить, какое число составляют экстренные вызовы, например 911 в США; 999 и 112 в Соединенном Королевстве. Политика расположения определяет, активирован ли пользователь для экстренных вызовов, и в каких случаях происходит аварийное обращение. Кроме того, можно определить, следует ли автоматически уведомлять корпоративную систему безопасности и как перенаправлять вызов.
  
Для получения дополнительных сведений об определении и изменении политики расположения, ознакомьтесь со статьей [Планирование политик расположения для Skype для бизнеса Server](location-policies.md) и [Создание политик расположения в Skype для бизнеса Server](../../deploy/deploy-enterprise-voice/create-location-policies.md). В этих разделах описываются основные понятия политик расположения; Тем не менее, вы должны следовать инструкциям, приведенным в разделе [Настройка нескольких номеров экстренных служб в Skype для бизнеса](../../deploy/deploy-enterprise-voice/configure-multiple-emergency-numbers.md) , для настройки нескольких номеров экстренных служб.
  
При планировании нескольких номеров экстренных служб учитывайте следующее.
  
- Накопительный пакет обновления за июнь 2016 июня позволяет указать до 5 номеров экстренных служб для определенной политики расположения. С накопительным пакетом обновления за ноябрь 2016 ноября это значение возрастает до 100.
    
    > [!NOTE]
    > Если вы еще не обновили накопительный пакет обновления 2016 для системы, ознакомьтесь со статьей [Updates to Skype для бизнеса Server 2015](https://support.microsoft.com/help/3061064/updates-for-skype-for-business-server-2015). 
  
- Для каждого номера для экстренного реагирования можно указать ноль или несколько масок телефонных номеров, которые уникальны для определенной политики расположения.
    
    Маска набора — это номер, который необходимо перевести в значение номера экстренного набора при его наборе. Например, предположим, что в этом поле введено значение 212, а в поле номер экстренного набора задано значение 911. Когда пользователь набирает номер 212, он будет преобразован в 911. Это позволяет использовать альтернативные номера для экстренного реагирования, которые по-прежнему могут вызывать экстренные службы (например, если кто-то из страны или региона с другим номером экстренной службы пытается набрать номер страны или региона, а не номер для страна или регион, в текущий момент находящиеся в данный момент). Можно определить несколько масок набора номера экстренных служб, разделяя значения точками с запятыми. Например, 212;414. Количество строк для маски набора составляет 100 символов. Все символы должны быть цифрами от 0 до 9.
    
- Каждая политика расположения имеет единое использование телефонной сети общего пользования (PSTN), которое используется для определения маршрута голосовой связи, используемого для маршрутизации экстренных вызовов от клиентов с помощью этой политики. Использование может иметь уникальный маршрут для каждого номера экстренного реагирования.
    
- Если в политике расположения определены параметры EmergencyNumbers и DialString, а клиент поддерживает несколько номеров экстренных служб, то приоритет будет приоритетным. Если клиент не поддерживает несколько номеров экстренных служб, используется строка набора номера для экстренного набора.
    
- Сведения о том, какие клиенты Skype для бизнеса и Lync поддерживают получение нескольких номеров экстренных служб, масок звонков и использования телефонной сети общего пользования (PSTN), можно узнать в разделе [Поддержка клиентов](multiple-emergency-numbers.md#BKMK_Clients).
    
> [!NOTE]
> Вы не можете настроить несколько номеров экстренных служб с помощью панели управления Skype для бизнеса. Для настройки нескольких номеров экстренных служб необходимо использовать PowerShell. 
  
Перед настройкой нескольких номеров экстренных служб следует учитывать следующее.
  
- Чтобы настроить несколько номеров экстренных служб, необходимо использовать командлет New – Ксемерженцинумбер, и необходимо определить политики расположения, поддерживающие несколько номеров экстренных служб, указав параметр EmergencyNumbers с командлетами [New + CsLocationPolicy](https://docs.microsoft.com/powershell/module/skype/new-cslocationpolicy?view=skype-ps) и [Set + CsLocationPolicy](https://docs.microsoft.com/powershell/module/skype/set-cslocationpolicy?view=skype-ps) .
    
- Если у вас есть существующие номера, заданные с помощью командлета Set – CsLocationPolicy или New CsLocationPolicy с параметрами Емерженцидиалстринг и EmergencyDialMask, значения, указанные с помощью параметра EmergencyNumbers, будут иметь приоритет над старым значений. То есть значения параметров Емерженцидиалстринг и EmergencyDialMask будут игнорироваться.
    
- Если у вас есть номера, заданные с помощью командлета Set – CsLocationPolicy или New CsLocationPolicy с параметрами Емерженцидиалстринг и EmergencyDialMask, *и вы не настраиваете новые номера экстренных* служб, существующие номера продолжат использоваться.
    
- Для работы нескольких номеров экстренных служб клиентские версии должны поддерживать новый компонент. Старые клиенты продолжат использовать старые значения, указанные командлетами Set — CsLocationPolicy или New CsLocationPolicy, с параметрами Емерженцидиалстринг и EmergencyDialMask. 
    
- Если пользователи набирает номер, который соответствует строке набора номера, маска набора не требуется. Например, если набираемый пользователем номер 911, то строка набора — 911, а маска не является обязательной. 
    
Дополнительные сведения о настройке нескольких номеров экстренных служб можно найти [в статье Настройка нескольких номеров экстренных служб в Skype для бизнеса](../../deploy/deploy-enterprise-voice/configure-multiple-emergency-numbers.md).

 **США** : не требуется несколько номеров экстренных служб. В США вы используете старую строку набора экстренных номеров и конфигурацию масок звонков.
  
 **US-больницы** — нет необходимости в маскировке "450". Для клиентов, которые пока не поддерживают несколько номеров экстренных служб, можно использовать старую строку набора экстренных номеров и конфигурацию масок звонков. Для клиентов, поддерживающих несколько номеров экстренных служб, можно определить номер для экстренного реагирования: "911" и "450" вместо маскировки 450.
  
 **Лондон** — для клиентов, которые пока не поддерживают несколько номеров экстренных служб, можно использовать старую строку набора экстренных номеров и конфигурацию масок звонков. Для клиентов, поддерживающих несколько номеров экстренных служб, можно определить номер для экстренного реагирования: "999" и "112" с использованием масок для каждого из них.
  
 **Индия** — все развернутые клиенты поддерживают несколько номеров экстренных служб. В Индии необходимо настроить несколько номеров экстренных служб.
