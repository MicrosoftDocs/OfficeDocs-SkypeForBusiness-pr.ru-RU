# <a name="plan-for-multiple-emergency-numbers-in-skype-for-business-server"></a>Планирование нескольких номеров экстренных служб в Skype для бизнеса Server
 
В этом разделе вы узнаете, как спланировать несколько номеров экстренных служб в Skype для бизнеса Server.
  
Skype для бизнеса Server теперь поддерживает конфигурацию нескольких номеров экстренных служб для клиента. Несколько номеров экстренных служб — это новая функция, представленная в накопительном обновлении за июнь 2016 г. Хотя в США имеется один номер экстренной службы ( 911), во многих странах поддерживается несколько номеров экстренных служб. Например, Соединенное Королевство поддерживает номер экстренной службы 999 для Соединенного Королевства и номер 112 для Ес. 
  
Эта функция также полезна для поставщиков услуг здравоохранения в США, которым нужна поддержка в роуминге для нескольких кодовых синих номеров экстренных служб.
  
## <a name="multiple-emergency-numbers-and-location-policies"></a>Несколько номеров экстренных служб и политик расположения

Чтобы настроить экстренные вызовы, создадим политики расположения, которые определяют реализации экстренных вызовов. Политика расположения используется для определения номера экстренного вызова, например 911 в США; 999 и 112 в Соединенном Королевстве. Политика расположения определяет, включен ли для пользователя экстренный вызов, и, если да, то каков его поведение. Вы также можете определить, следует ли автоматически уведомить о безопасности предприятия и как следует перенаададовка вызова.
  
Дополнительные сведения об определении и изменении политики расположения см. в дополнительных сведениях о планировании политик расположения [для Skype](location-policies.md) для бизнеса Server и создании политик расположения в Skype для [бизнеса Server.](../../deploy/deploy-enterprise-voice/create-location-policies.md) В этих темах описываются понятия, связанные с политиками расположения; However, you must follow the instructions in [Configure multiple emergency numbers in Skype for Business](../../deploy/deploy-enterprise-voice/configure-multiple-emergency-numbers.md) to configure multiple emergency numbers.
  
При планировании нескольких номеров экстренных служб следует иметь в виду следующее:
  
- С помощью накопительного обновления за июнь 2016 г. можно определить до 5 номеров экстренных служб для заданной политики расположения. С накопительным обновлением за ноябрь 2016 г. это число увеличивается до 100.
    
    > [!NOTE]
    > If you have not yet upgraded to the November 2016 Cumulative Update, see [Updates to Skype for Business Server 2015](https://support.microsoft.com/help/3061064/updates-for-skype-for-business-server-2015). 
  
- Для каждого номера экстренных служб можно указать ноль или больше масок набора номера экстренных служб, которые уникальны для заданной политики расположения.
    
    Маска набора номера — это номер, который необходимо преобразовыть в значение номера экстренного номера при наборе номера. Например, предположим, что в этом поле введите значение 212, а поле экстренного номера имеет значение 911. Когда пользователь набирает 212, номер будет переведен на 911. Это позволяет набирать альтернативные номера экстренных служб и по-прежнему звонить в экстренные службы (например, если кто-то из страны или региона с другим номером экстренной службы пытается набрать номер этой страны или региона, а не номер страны или региона, в который он сейчас находится). Можно определить несколько масок набора номера экстренных служб, разделяя значения точками с запятыми. Например, 212;414. Ограничение строк для маски набора составляет 100 символов. Все символы должны быть цифрами от 0 до 9.
    
- Каждая политика расположения использует единую телефонную сеть общего пользования (PSTN), которая используется для определения маршрута голосовой связи, используемого для маршрутации экстренных вызовов от клиентов, использующих эту политику. Использование может иметь уникальный маршрут на номер экстренного экстренного номера.
    
- Если в политике определения местоположения определены параметры EmergencyNumbers и DialString, а клиент поддерживает несколько номеров экстренных служб, приоритет имеет номер экстренной службы. Если клиент не поддерживает несколько номеров экстренных служб, используется строка набора номера экстренных служб.
    
- Сведения о том, какие клиенты Skype для бизнеса и Lync поддерживают получение нескольких номеров экстренных служб, масок набора номера и режимов работы с телефонной сетью общего пользования (PSTN), см. в службе поддержки [клиентов.](multiple-emergency-numbers.md#BKMK_Clients)
    
> [!NOTE]
> С помощью панели управления Skype для бизнеса нельзя настроить несколько номеров экстренных служб. Для настройки нескольких номеров экстренных служб необходимо использовать PowerShell. 
  
Перед настройкой нескольких номеров экстренных служб необходимо иметь в виду следующее:
  
- Чтобы настроить несколько номеров экстренных служб, необходимо использовать New-CsEmergencyNumber, а также определить политики расположения, которые поддерживают несколько номеров экстренных служб, указав параметр EmergencyNumbers с помощью cmdlets [New-CsLocationPolicy](https://docs.microsoft.com/powershell/module/skype/new-cslocationpolicy?view=skype-ps) и [Set-CsLocationPolicy.](https://docs.microsoft.com/powershell/module/skype/set-cslocationpolicy?view=skype-ps)
    
- Если имеются номера, определенные с помощью Set-CsLocationPolicy или New-CsLocationPolicy с параметрами EmergencyDialString и EmergencyDialMask, значения, указанные с помощью параметра EmergencyNumbers, будут иметь приоритет над старыми значениями. То есть значения параметров EmergencyDialString и EmergencyDialMask будут игнорироваться.
    
- Если у вас есть существующие номера, определенные с помощью Set-CsLocationPolicy или New-CsLocationPolicy с параметрами EmergencyDialString  и EmergencyDialMask, а новые номера экстренных служб не настроены, существующие номера будут по-прежнему использоваться.
    
- Для работы нескольких номеров экстренных служб запущенные клиентские версии должны поддерживать новую функцию. Старые клиенты продолжат использовать старые значения, указанные в Set-CsLocationPolicy или New-CsLocationPolicy с параметрами EmergencyDialString и EmergencyDialMask. 
    
- Если пользователи будут набирать номер, который соответствует строке набора номера, маска набора номера не требуется. Например, если номер, набираемого пользователем, — 911, строка набора — 911, маска не требуется. 
    
Дополнительные сведения о настройке нескольких номеров экстренных служб см. в подстройке "Настройка нескольких номеров экстренных [служб в Skype для бизнеса".](../../deploy/deploy-enterprise-voice/configure-multiple-emergency-numbers.md)

 **США —** не существует требований для нескольких номеров экстренных служб. В США используются старые конфигурации строки набора номера экстренного номера и маски набора номера.
  
 **US-Hospital** — необходимо не маскировать "450". Для клиентов, которые еще не поддерживают несколько номеров экстренных служб, можно использовать старые конфигурации строки набора номера и маски набора номера. Для клиентов, которые поддерживают несколько номеров экстренных служб, можно определить номер экстренной службы для "911" и "450", а не маскировать 450.
  
 **Лондон** — для клиентов, которые еще не поддерживают несколько номеров экстренных служб, можно использовать старую конфигурацию строки набора номера и маски набора номера. Для клиентов, которые поддерживают несколько номеров экстренных служб, можно определить номер экстренной службы для "999" и "112" с масками для каждого из них.
  
 **Индия** — все развернутые клиенты поддерживают несколько номеров экстренных служб. В Индии необходимо настроить только несколько номеров экстренных служб.
