# <a name="branch-site-sip-trunking-in-skype-for-business-server"></a>Магистраль SIP-сайта филиала в Skype для бизнеса Server
 
Узнайте о магистрали SIP на сайтах филиалов в Skype для бизнеса Server Корпоративная голосовая связь.
  
В некоторых случаях может потребоваться реализовать распределенную магистраль SIP на выбранных сайтах филиалов. Чтобы определить, нужен ли магистраль SIP для сайта филиала, а также сведения о поддерживаемых вариантах топологии для развертывания магистрали SIP на сайтах филиалов см. в материале [SIP trunking in Skype для бизнеса Server.](sip-trunking.md)
  
## <a name="example-branch-site-sip-trunk-requirements-analysis"></a>Пример анализа требований к магистрали SIP для офиса филиала

При развертывании магистрали SIP-сайта филиала необходимо выполнить анализ затрат, определенных для конкретного сайта. Например, предприятию с центральным сайтом в Редмонде( штат Вашингтон) и филиалом в Нью-Йорке следует провести анализ, чтобы определить, следует ли реализовать магистраль SIP с сайта в Нью-Йорке местному поставщику услуг.
  
Чтобы определить, является ли экономически выгодной распределенная магистраль SIP в Нью-Йорке, определите, какие номера для прямого соединения с внутренними абонентами будет использовать магистраль SIP, и проанализируйте число звонков, совершаемых из офиса в Нью-Йорке в области, не включающие Рэдмонд (425). Вы можете прекратить DID для сайта филиала на центральном сайте. Например, центральный сайт Redmond может принимать номера DID для сайта филиала в Нью-Йорке. Если затраты на реализацию магистрали распределенных SIP меньше, чем стоимость этих вызовов, рассмотрите возможность реализации магистрали SIP на сайте филиала в Нью-Йорке. 
  
## <a name="other-branch-site-sip-trunk-requirements"></a>Другие требования к магистрали SIP для офиса филиала

Выбор между развертыванием магистрали SIP вместо шлюза основан на разнице тарифов за междугородные звонки в телефонной сети общего пользования в каждом из вариантов. При развертывании магистрали SIP-сайта филиала также необходимо определить требования к устойчивости и пропускной способности. Если связь между сайтом филиала и центральным сайтом устойчива и имеет достаточную пропускную способность, можно развернуть магистраль SIP или шлюз. Вам не нужно развертывать устройство для выживших филиалов на сайте филиала. Если связь между веб-сайтом филиала и центральным сайтом не устойчива, разверните уцелевшие устройства филиала или разверните оставшийся в живых сервер филиала с шлюзом или магистралью SIP на сайте филиала.
