# <a name="plan-for-the-response-group-application-in-skype-for-business-server"></a>Планирование приложения группы ответа в Skype для бизнеса Server

Планирование групп ответа в Skype для бизнеса Server Корпоративная голосовая связь, что позволяет настроить маршрутику звонков для групп пользователей. Включает требования к аудиофайлам.

Если в организации есть группы людей, которые отвечают на определенные типы вызовов, например службу поддержки клиентов, внутреннюю службу технической поддержки или общую телефонную поддержку отдела, вы можете развернуть приложение группы ответа для управления этими типами звонков. Приложение группы ответа маршрутит и ставит в очередь входящие звонки назначенным лицам, которые называются агентами. С помощью групп ответа можно увеличить использование телефонной службы поддержки и сократить расходы на работу этих служб.

Когда вызывающий абонент звонит в группу ответа, этот вызов направляется агенту исходя из сервисной группы или ответов вызывающего абонента на вопросы интерактивного автоответчика (IVR). Приложение группы ответа использует стандартные методы маршрутизации группы ответа для маршрутизации вызова следующему доступному агенту. Поддерживаемые методы маршрутики вызовов включают последовательную маршрутику с самым длительным бездействием, параллельную, круговую маршрутику и маршрутику помощника (то есть все агенты вызываются одновременно для каждого входящих вызовов независимо от их текущего присутствия).

Если ни один агент не доступен, то вызов удерживается в очереди, пока какой-нибудь агент не освободится. Находясь в очереди, вызывающий абонент слушает музыку, пока доступный агент не примет вызов. Если очередь заполнена или время ожидания вызова в очереди, то звонячий может прослушать сообщение, а затем отключиться или перенаправить его в другое место назначения, например другой номер телефона или голосовую почту. Когда агент принимает вызов, то в зависимости от того, как администратор настроил группу ответа, вызывающий абонент может видеть или не видеть удостоверение этого агента. Агенты могут быть либо формальными, что означает, что они должны входить в группу, прежде чем смогут принимать направленные в эту группу вызовы, либо неформальными, что означает, что им не нужно входить в группу или выходить из нее, чтобы принять вызов.

> [!NOTE]
> Агентами могут быть только локальные пользователи. Если агент переходит из локального режима в интерактивный, ему не будут направляться звонки группы ответа.

> [!NOTE]
> Приложение группы ответа использует внутреннюю службу Match Making для очереди вызовов и поиска доступных агентов. На каждом компьютере, на который выполняется приложение группы ответа, запускается служба Match Making, но одновременно активна только одна служба Match Making для каждого пула; остальные являются пассивными. Если активная служба Match Making становится недоступной в результате внепланового отключения, то становится активной одна из пассивных служб Match Making. Приложение группы ответа делает все возможное, чтобы маршрутия и очередь вызовов не прерывались. Однако когда происходит переход службы Match Making, все передаваемые в это время вызовы теряются. Например, если переход вызван с ошибкой сервера переднего сервера, все вызовы, обрабатываемые активной службой match Making на этом сервере, также будут потеряны.

## <a name="response-group-workflows"></a>Рабочий процесс группы ответа

Рабочий процесс определяет порядок обработки вызова с момента телефонного звонка до момента ответа на вызов. Этот рабочий процесс задает очередь удержания вызова и определяет метод маршрутизации, используемый для сервисных групп, а также вопросы и ответы, используемые для интерактивных групп ответа. Рабочий процесс также задает такие параметры, как приветственное сообщение, музыка при удержании, часы работы и выходные.

> [!NOTE]
> Вы должны создать группы агентов и очереди перед созданием рабочего процесса, в котором они используются.

## <a name="management-of-response-groups"></a>Управление группами ответа

В Skype для бизнеса Server для управления группами ответа доступны две роли управления: руководитель группы ответа и администратор группы ответа. Администраторы группы ответа могут управлять любыми аспектами любой группы ответа. Менеджеры группы ответа могут управлять только определенными аспектами и только для тех групп ответа, которые им принадлежат. Роль диспетчера помогает сократить затраты на администрирование, так как вы можете делегировать ограниченные обязанности за определенные группы ответа любому пользователю, у которого включена Корпоративная голосовая связь. Обратите внимание, что пользователь может быть руководителем группы ответа и администратором группы ответа.

Для поддержки роли диспетчера приложение  группы ответа использует тип рабочего процесса "Управляемый" или "Неуправленный". В следующей таблице описываются управляемые и неуправляемые группы ответа.

## <a name="deployment-and-requirements"></a>Развертывание и требования

Приложение группы ответа автоматически включено при развертывании Корпоративная голосовая связь.

### <a name="hardware-and-software-requirements"></a>Требования к оборудованию и программному обеспечению

Приложение группы ответа предъявляет те же требования к оборудованию, требованиям к операционной системе и программному обеспечению, что и серверы переднего сервера.

Если вы используете файлы Windows Media Audio (WMA) для музыки и объявлений группы ответа, на всех серверах переднего уровня или серверах Standard Editions, на которые запущено приложение группы ответа, должна быть установлена система Windows Media Format Runtime для серверов с Windows Server 2008 R2 или Microsoft Media Foundation для серверов под управлением Windows Server 2012 или Windows Server 2012 R2. Для Windows Server 2008 R2 windows Media Format Runtime устанавливается как часть рабочего стола Windows.

Группа ответа использует **языковые пакеты** для поддержки распознавания текста в речь и речи. Эти технологии речи используются при настройке сообщений, например приветственного сообщения и других приглашений, а также вопросов и ответов для система предварительно записанных голосовых сообщений (interactive voice response — IVR). По умолчанию при развертывании Skype для бизнеса Server устанавливаются 26 поддерживаемых языковых пакетов.

### <a name="port-requirements"></a>Требования к портам

Приложение группы ответа использует следующие порты:

- **Порт 5071 для**   запросов прослушивания SIP

- **Порт 8404 для**   межсерверных коммуникаций

    > [!NOTE]
    > Этот порт используется для службы match Making и требуется, когда приложение группы ответа развертывается в пуле с более чем одним сервером переднего сервера.

   > [!NOTE]
   > Эти порты отражают параметры по умолчанию, которые вы можете изменить с помощью командлета **Set-CsApplicationServer**. Подробные сведения об этом cmdlet см. в документации по skype для бизнеса Server Management Shell.

### <a name="audio-file-requirements"></a>Требования к аудиофайлам

Приложение группы ответа поддерживает формат файлов WAV и формат звуковых файлов Windows Media (WMA) для сообщений группы ответа, музыки на удержании или вопросов интерактивного голосового ответа (IVR).

Для формата аудиофайла Windows Media требуется, чтобы на серверах переднего Windows Server 2008 R2 и Windows Server 2008 была установлена windows Media Format Runtime. Дополнительные сведения см. выше в подразделе "Требования к программному обеспечению".

#### <a name="supported-wave-file-formats"></a>Поддерживаемые форматы файлов WAV

Все файлы WAV должны удовлетворять следующим требованиям:

- 8-разрядный или 16-разрядный файл

- Формат линейной импульсно-кодовой модуляции, A-Law или mu-Law

- Моно или стерео

- 4 МБ или меньше

Для обеспечения наилучшей производительности файлов WAV рекомендуется использовать монофонический 16-разрядный файл WAV с частотой 16 кГц.

#### <a name="supported-windows-media-audio-file-formats"></a>Поддерживаемые форматы файлов WMA

Если вы используете файл WMA рекомендуется применять низкие скорости и проверить производительность системы под нагрузкой.

Преобразовать файл в формат WMA вы можете с помощью Microsoft Expression Encoder 4. Чтобы скачать кодер expression 4, [https://go.microsoft.com/fwlink/p/?linkId=202843](https://go.microsoft.com/fwlink/p/?linkId=202843) см.

## <a name="client-support"></a>Поддержка клиента

Приложение группы ответа поддерживает следующие клиенты:

- Настольный клиент Skype для бизнеса

- Настольный клиент Lync 2013

- Настольный клиент Lync 2010

- Lync 2010 Attendant

- Помощник Office Communications Server 2007 R2

- Lync Phone Edition

> [!NOTE]
> Приложение группы ответа не поддерживается в мобильных клиентах Lync.

Конкретный клиент, который можно использовать, зависит от типа пользователя группы ответа, который вы:

- **Вызывающие** могут вызывать группу ответа с помощью любого вышеуказанного клиента или обычного телефона ТСОП.

- **Неофициальные** агенты (агенты, которые не входят в свои группы и не выходят из них для звонков) могут принимать звонки с помощью attendant, Lync или Lync Phone Edition. Неофициальные агенты автоматически вписываются в свои группы при входе в Skype для бизнеса Server с помощью одного из этих клиентов.

-  Формальные агенты (агенты, которым необходимо впуск в группы и выход из них для звонков) могут принимать звонки с помощью Skype для бизнеса и доступа к консоли агента из пункта меню или с помощью помощника и доступа к консоли агента непосредственно из Internet Explorer.
