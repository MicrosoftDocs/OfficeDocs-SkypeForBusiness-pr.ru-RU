# <a name="plan-for-the-response-group-application-in-skype-for-business-server"></a>Планирование приложения группы реагирования в Skype для бизнеса Server

Планирование групп ответа в Skype для бизнеса Server Корпоративная голосовая связь, что позволяет настроить маршрутику вызовов для групп пользователей. Включает требования к аудиофайлам.

Если в вашей организации есть группы людей, которые отвечают и управляют определенными типами вызовов, например службой поддержки клиентов, внутренней службой поддержки или общей телефонной поддержкой отдела, вы можете развернуть приложение Response Group для управления этими типами вызовов. Маршруты и очереди входящих вызовов для назначенных лиц, известных как агенты, маршруты и очереди. С помощью групп ответа можно увеличить использование телефонной службы поддержки и сократить расходы на работу этих служб.

Когда вызывающий абонент звонит в группу ответа, этот вызов направляется агенту исходя из сервисной группы или ответов вызывающего абонента на вопросы интерактивного автоответчика (IVR). Приложение Группы реагирования использует стандартные методы маршрутизации группы ответов для маршрутизации вызова к следующему доступному агенту. Поддерживаемые методы маршрутивки вызовов включают последовательный, самый длинный простой, параллельный, круглую маршрутику и маршрутную маршрутику (то есть все агенты вызывались одновременно для каждого входящих вызовов, независимо от их текущего присутствия).

Если ни один агент не доступен, то вызов удерживается в очереди, пока какой-нибудь агент не освободится. Находясь в очереди, вызывающий абонент слушает музыку, пока доступный агент не примет вызов. Если очередь заполнена или время ожидания вызова в очереди, вызываемая может услышать сообщение, а затем отключается или переносится в другой пункт назначения, например другой номер телефона или голосовая почта. Когда агент принимает вызов, то в зависимости от того, как администратор настроил группу ответа, вызывающий абонент может видеть или не видеть удостоверение этого агента. Агенты могут быть либо формальными, что означает, что они должны входить в группу, прежде чем смогут принимать направленные в эту группу вызовы, либо неформальными, что означает, что им не нужно входить в группу или выходить из нее, чтобы принять вызов.

> [!NOTE]
> Агентами могут быть только локальные пользователи. Если агент переходит из локального режима в интерактивный, ему не будут направляться звонки группы ответа.

> [!NOTE]
> Приложение Группы ответа использует внутреннюю службу под названием Match Making для очереди звонков и поиска доступных агентов. На каждом компьютере, на который работает приложение Группы ответов, выполняется служба Match Making, но одновременно действует только одна служба match Making для каждого пула, остальные — пассивные. Если активная служба Match Making становится недоступной в результате внепланового отключения, то становится активной одна из пассивных служб Match Making. Приложение Группы реагирования делает все возможное, чтобы маршрутизационные маршруты и очереди вызовов не прерывались. Однако когда происходит переход службы Match Making, все передаваемые в это время вызовы теряются. Например, если переход происходит из-за сбития переднего сервера, все вызовы, которые в настоящее время обрабатываются активной службой match Making на этом переднем сервере, также теряются.

## <a name="response-group-workflows"></a>Рабочий процесс группы ответов

Рабочий процесс определяет порядок обработки вызова с момента телефонного звонка до момента ответа на вызов. Этот рабочий процесс задает очередь удержания вызова и определяет метод маршрутизации, используемый для сервисных групп, а также вопросы и ответы, используемые для интерактивных групп ответа. Рабочий процесс также задает такие параметры, как приветственное сообщение, музыка при удержании, часы работы и выходные.

> [!NOTE]
> Вы должны создать группы агентов и очереди перед созданием рабочего процесса, в котором они используются.

## <a name="management-of-response-groups"></a>Управление группами реагирования

В Skype для бизнеса Server для управления группами реагирования доступны две роли управления: диспетчер группы реагирования и администратор группы реагирования. Администраторы групп реагирования могут управлять любым аспектом любой группы отклика. Руководители групп реагирования могут управлять только определенными аспектами и только для тех групп реагирования, которые им принадлежат. Роль Manager может помочь вам сократить расходы на администрирование, так как вы можете делегировать ограниченные обязанности для определенных групп отклика любому пользователю, который включен для Корпоративная голосовая связь. Обратите внимание, что пользователь может быть как менеджером группы реагирования, так и администратором группы реагирования.

## <a name="deployment-and-requirements"></a>Развертывание и требования

Приложение Группы реагирования автоматически включено при развертывании Корпоративная голосовая связь.

### <a name="hardware-and-software-requirements"></a>Требования к оборудованию и программному обеспечению

Приложение Группы реагирования имеет те же требования к оборудованию, требованиям операционной системы и требованиям к программному обеспечению, что и серверы переднего конца.

Если для музыки и анонсов группы откликов используются файлы Windows Media Audio (WMA), все серверы front End Server или Standard Editions, на которые запущено приложение Группы отклика, должны иметь время запуска Windows формата мультимедиа для серверов, работающих Windows Server 2008 R2, или Microsoft Media Foundation для серверов, работающих Windows Server 2012 или Windows Server 2012 R2. Для Windows Server 2008 R2 Windows формата мультимедиа устанавливается в Windows Desktop Experience.

Группа реагирования использует **языковые пакеты** для поддержки распознавания речи и речи. Эти технологии речи используются при настройке сообщений, например приветственного сообщения и других приглашений, а также вопросов и ответов для система предварительно записанных голосовых сообщений (interactive voice response — IVR). По умолчанию 26 поддерживаемых языковых пакетов устанавливаются при развертывании Skype для бизнеса Server.

### <a name="port-requirements"></a>Требования к портам

Приложение Группы реагирования использует следующие порты:

- **Порт 5071**   для запросов на прослушивание SIP

- **Порт 8404 для**   межсерверных коммуникаций

    > [!NOTE]
    > Этот порт используется для службы Match Making и требуется, когда приложение Группы реагирования развернуто в пуле с более чем одним передним конечным сервером.

   > [!NOTE]
   > Эти порты отражают параметры по умолчанию, которые вы можете изменить с помощью командлета **Set-CsApplicationServer**. Сведения об этом комлете см. в документации Skype для бизнеса Server Management Shell.

### <a name="audio-file-requirements"></a>Требования к аудиофайлам

Приложение Группы реагирования поддерживает формат файлов волны (wav) и формат файлов Windows Мультимедиа аудио (wma) для сообщений группы ответов, музыки на удержание или интерактивных голосовых ответов (IVR).

Формат аудиофайла Windows мультимедиа требует установки Windows media Format Runtime на серверах переднего Windows Server 2008 R2 и Windows Server 2008. Дополнительные сведения см. выше в подразделе "Требования к программному обеспечению".

#### <a name="supported-wave-file-formats"></a>Поддерживаемые форматы файлов WAV

Все файлы WAV должны удовлетворять следующим требованиям:

- 8-разрядный или 16-разрядный файл

- Формат линейной импульсно-кодовой модуляции, A-Law или mu-Law

- Моно или стерео

- 4 МБ или меньше

Для обеспечения наилучшей производительности файлов WAV рекомендуется использовать монофонический 16-разрядный файл WAV с частотой 16 кГц.

#### <a name="supported-windows-media-audio-file-formats"></a>Поддерживаемые форматы файлов WMA

Если вы используете файл WMA рекомендуется применять низкие скорости и проверить производительность системы под нагрузкой.

Преобразовать файл в формат WMA вы можете с помощью Microsoft Expression Encoder 4. Чтобы скачать экспрессию Encoder 4, [https://go.microsoft.com/fwlink/p/?linkId=202843](https://go.microsoft.com/fwlink/p/?linkId=202843) см. в .

## <a name="client-support"></a>Поддержка клиента

Приложение Группы реагирования поддерживает следующих клиентов:

- Skype для бизнеса настольного клиента

- Клиент настольного компьютера Lync 2013

- Клиент настольного компьютера Lync 2010

- Lync 2010 Attendant

- Помощник Office Communications Server 2007 R2

- Lync Phone Edition

> [!NOTE]
> Приложение Группы реагирования не поддерживается в мобильных клиентах Lync.

Конкретный клиент, который можно использовать, зависит от типа пользователя группы реагирования, который вы:

- **Вызывающие** могут вызывать группу ответа с помощью любого вышеуказанного клиента или обычного телефона ТСОП.

- **Неформальные** агенты (агенты, которые не въехают в группы и не принимают вызовы) могут принимать вызовы с помощью Attendant, Lync или Lync Телефон Edition. Неформальные агенты автоматически подписываются в свои группы при входе Skype для бизнеса Server с помощью одного из этих клиентов.

-  Официальные агенты (агенты, которые должны войти в свои группы и выйти из них для звонков) могут принимать вызовы с помощью Skype для бизнеса и доступа к консоли агента из элемента меню или с помощью сопровождающего и доступа к консоли агента непосредственно из Internet Explorer.
