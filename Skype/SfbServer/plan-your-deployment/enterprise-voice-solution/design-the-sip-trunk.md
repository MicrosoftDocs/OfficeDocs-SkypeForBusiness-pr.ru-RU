# <a name="design-the-sip-trunk-for-e9-1-1-in-skype-for-business-server-2015"></a>Проектирование магистральной сети SIP для службы E9-1-1 в Skype для бизнеса Server 2015
 
Планирование вашей топологии распределения каналов SIP для развертывания E9-1-1, использующего поставщиками распределения каналов SIP, в Скайп Business Server корпоративной голосовой связи.
  
Скайп для Business Server используется магистралях SIP экстренных вызовов поставщику услуг E9-1-1. Вы можете установить магистрали SIP экстренной службы для E9-1-1 на одном центральном сайте, на нескольких центральных сайтах или на каждом сайте филиалов. Тем не менее, если связи глобальной сети между вызывающего абонента и узел, на котором размещаются магистраль SIP экстренные службы недоступен, то звонок с помощью пользователем на отключенные сайта потребуется записи об использовании телефона в политике голосовой связи пользователя, который будет перенаправлять звонок в ECRC через шлюз локальную телефонную сеть (общего пользования PSTN). Это также относится к ситуации, когда для вызовов действуют совместные ограничения функции контроля допуска звонков.
  
Существует два способа внедрить магистраль SIP в Скайп для среды Business Server:
  
- Использование серверов-посредников многосетевого, используют свои общедоступные интерфейсы наружу адрес для взаимодействия с поставщиком услуг магистралей SIP.
    
- Используйте локальный пограничный контроллер сеансов (SBC) для предоставления безопасной точки разграничения между серверов-посредников и канала SIP поставщика услуг.
    
Если вы выбираете последний способ, убедитесь, что образец SBS и выбранная вами модель были сертифицирована и поддерживали передачу данных расположения в виде PIDF-LO (объекта расположения, использующий формат данных о присутствии), в составе SIP INVITE. В противном случае вызовы будут приходить поставщику услуг экстренных служб без информации о расположении. Для получения дополнительных сведений о сертифицированного SBC видеть [«инфраструктуры рекомендованные для Microsoft Lync»](https://go.microsoft.com/fwlink/p/?LinkId=248425) и [«Телефония инфраструктуры для Скайп для бизнес-»](https://technet.microsoft.com/en-us/office/dn947483).
  
Для подстраховки поставщики услуг E9-1-1 предоставляют доступ к паре пограничных контроллеров сеансов. Необходимо принять несколько решений относительно сервера-посредника топологии и конфигурации маршрутизации вызовов. Будете ли вы рассматривать оба пограничных контроллера SBC в качестве равноправных одноранговых узлов сети и применять метод маршрутизации с циклическим перебором для звонков между ними или же назначите один пограничный контроллер SBC в качестве основного, а другой — в качестве дополнительного?
  
Для получения дополнительных сведений о развертывании магистраль SIP в Скайп для Business Server см [SIP-магистрали в Скайп для Business Server 2015](sip-trunking.md). Следующие вопросы помогут вам принять решение о способе развертывания магистралей SIP для E9-1-1.
  
 **Следует ли развернуть магистраль SIP на базе выделенного или общего подключения к Интернету?**
  
> Для экстренных вызовов важно обеспечить постоянное подключение. Выделенная линия предоставляет соединение, на которое не будет влиять остальной трафик в сети, и позволяет реализовать качество обслуживания. Помните о том, что если при подключении к поставщикам услуг экстренных служб через общедоступное соединение с Интернетом требуется гарантировать конфиденциальность экстренных звонков, то требуется шифрование IPSec. 
    
 **Развертывание E9-1-1, предназначенной для авариям?**
  
> Поскольку это решение для экстренных ситуаций, устойчивость имеет важное значение. Развертывание вашей основных и дополнительных серверов-посредников и SIP магистралей в расположениях отказоустойчивые аварийного. Рекомендуется развернуть основной сервер-посредник наиболее близкого к пользователям, которые он поддерживает и отработки отказа маршрут вызывает через дополнительный сервер-посредник (находится в разных географических расположениях). 
    
 **Следует ли вам развернуть отдельную магистраль SIP для каждого филиала?**
  
> Скайп для Business Server предоставляет несколько стратегий для обработки устойчивости голосовой связи в филиалах, в том числе: необходимости устойчивое данных сетей, развертывание магистраль SIP в каждой ветки или внедряет вызовов на локальный шлюз во время простоев. Дополнительные сведения см [SIP-магистрали в Скайп для Business Server 2015](sip-trunking.md).
    
 **Включается ли контроль допуска звонков?**
  
> Скайп для Business Server не дескриптор экстренных вызовов какие-либо отличается от обычных вызова. Поэтому управление пропускной способностью или контроль допуска звонков могут оказывать негативное влияние на конфигурацию E9-1-1. При включенном контроле допуска звонков экстренные вызовы будут блокироваться или перенаправляться на локальный шлюз ТСОП, а на канале, куда перенаправляются экстренные звонки, будет превышен заданный предел. Как уже было указано выше в этом подразделе, такие звонки не будут содержать данных о расположении, и их следует вручную перенаправлять в ECRC.
