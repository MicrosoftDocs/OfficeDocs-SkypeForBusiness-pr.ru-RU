## <a name="server-hardware"></a>Серверное оборудование
<a name="S_hard"> </a>

Сетевой адаптер каждого сервера в топологии сервера Skype для бизнеса должен поддерживать по крайней мере 1 гигабит в секунду (Гбит/с). Как правило, вы должны подключить все роли сервера в топологии сервера Skype для бизнеса, используя слабую задержку и локальную сеть с высокой пропускной способностью (ЛВС). Размер локальной сети зависит от размера топологии.

- В стандартных топологиях выпуска серверы должны находиться в сети, поддерживающей 1 Гбит/с Ethernet или аналогичный.

- В топологиях Enterprise Edition большинство серверов должны находиться в сети, поддерживающей более 1 Гбит/с, особенно при поддержке конференции аудио-или видеосвязи и совместного использования приложений.

Для интеграции с телефонной сетью общего пользования (ТСОП) можно использовать линии T1/E1 или распределение каналов SIP.

## <a name="audiovideo-network-requirements"></a>Требования к сети для передачи звука и видео
<a name="AV_req"> </a>

Требования к сети для аудио-и видеосвязи (A/V) в развертывании сервера Skype для бизнеса Server включают следующее:

- Если вы разворачиваете один сервер пограничного сервера или Граничный пул с помощью балансировки нагрузки DNS, вы можете настроить _внешний_ брандмауэр для выполнения трансляции сетевых адресов (NAT). Невозможно настроить _internal_ брандмауэр для выполнения преобразования сетевых адресов (NAT). Подробности можно найти в разделе [планирование порта и брандмауэра](../edge-server-deployments/edge-environmental-requirements.md#port-and-firewall-planning).

    > [!IMPORTANT]
    > Если у вас есть пул EDGE и вы используете аппаратную подсистему балансировки нагрузки, вы должны использовать общедоступные IP-адреса на пограничных серверах и не можете использовать NAT для серверов или пула на устройстве с поддержкой NAT (например, на устройстве межсетевого экрана или коммутаторе локальной сети). Подробности можно найти [в разделе сценарии пограничного сервера в Skype для бизнеса Server](../edge-server-deployments/scenarios.md).

- Если в вашей организации используется инфраструктура качества обслуживания (QoS), подсистема передачи мультимедиа настроена на работу в этой существующей инфраструктуре.

- Если вы используете протоколы IPsec, мы рекомендуем отключить их для диапазона портов, который используется для аудио- и видеотрафика. Подробнее об этом можно узнать в разделе [исключения IPSec](#ipsec-exceptions).

Чтобы обеспечить оптимальное качество мультимедиа, сделайте следующее.

- Обеспечьте пропускную способность сетевых каналов в периоды пикового использования в 65 килобит в секунду (Кбит/с) для аудиопотока и 500 Кбит/с для видеопотока, если они включены. Двунаправленный аудио- или видеосеанс использует два потока, поэтому для простой голосовой/телефонной связи будет требоваться 130 Кбит/с для охвата каждого потока. Видео также будет использовать общую сумму 1000 Кбит/с для переноса восходящих и нижестоящих соединений.

- Для решения непредвиденных скачков трафика и более высокой степени использования в течение времени конечные точки мультимедиа в Skype для бизнеса Server могут адаптироваться к различным условиям сети и поддерживать в три раза больше пропускной способности для аудио-и видеосвязи, а также для поддержания приемлемого качества. Не стоит рассчитывать на то, что благодаря такой адаптации сеть с недостаточными ресурсами сможет функционировать нормально. В неподготовленной сети способность конечных точек мультимедиа в Skype для бизнеса Server динамически справляться с различными сетевыми условиями (например, временной потерей данных) уменьшается.

- Если обеспечение достаточных ресурсов сети является очень дорогостоящим и трудоемким мероприятием, можно попытаться снизить объем трафика. В этом сценарии для эластичности серверных конечных точек мультимедиа в Skype для бизнеса Server уменьшается разница между объемом трафика и уровнем пикового трафика за счет снижения качества голоса. Кроме того, в этом случае будет снижаться запас по уровню, который позволяет справляться с резкими увеличениями объема трафика.

- Если вам не удается подготовить соответствующие каналы в краткосрочной перспективе (например, если узел имеет каналы глобальной сети с очень низкой пропускной способностью), вы можете отключить видео для некоторых пользователей.

- При подготовке сети обеспечьте максимальную задержку сквозной передачи на уровне не более 150 миллисекунд (мс) при максимальной нагрузке. Задержка — это нарушение в сети того, что компоненты мультимедиа в Skype для бизнеса Server не могут быть сокращены, и важно найти и устранить слабые точки.

- Для серверов с антивирусной программой включите все серверы, на которых запущено приложение Skype для бизнеса Server, в список исключений, чтобы обеспечить оптимальную производительность и качество звука.

Требуемая полоса пропускания зависит от разрешения, качества, частоты кадров и количества движений или изменений изображения. Для каждого разрешения существует две подходящие скорости передачи данных:

- **Максимальная частота битов полезных данных** Это битовая скорость, которую конечная точка будет использовать для разрешения при максимальной частоте кадров. Это значение важно, так как оно обеспечивает наилучшее качество для видео и звука.

- **Минимальная частота битов полезных данных** Это скорость, под которой конечная точка сервера Skype для бизнеса Server будет переходить к следующему более низким разрешению. Чтобы гарантировать определенное разрешение, доступная скорость передачи полезной нагрузки видео не должна быть меньше этой минимальной скорости для данного разрешения. Это значение помогает понять минимальное допустимое значение, когда максимальная скорость передачи недоступна или не требуется. Для некоторых пользователей видео с такой низкой скоростью передачи может быть неприемлемым, поэтому будьте осторожны при рассмотрении таких низких скоростей. Обратите внимание, что для видеосцен без движения фактическая скорость передачи данных может временно становиться меньше минимальной скорости.

В Skype для бизнеса Server поддерживается множество разрешений. Это позволяет настроить в Skype для бизнеса Server различные пропускную способность сети и получать клиентские возможности. Пропорции по умолчанию для Skype для бизнеса Server — 16:9. Устаревшая пропорция 4:3 все еще поддерживается для веб-камер, которые не позволяют захватить пропорции 16:9.

FEC для видео включается в скорость передачи полезной нагрузки видео, при использовании этого метода, поэтому нет отдельных значений с FEC для видео и без FEC для видео.

Конечные точки не передают потоковые пакеты аудио или видео непрерывно. В зависимости от сценария существуют различные уровни активности потока, указывающие, как часто пакеты передаются в потоке. Активность потока зависит от мультимедиа и сценария и не зависит от используемого кодека. В одноранговом сценарии:

- Аудиопотоки отправляются из конечных точек, только когда пользователи говорят.

- Оба участника получают аудиопотоки.

- Если используется видео, обе конечные точки передают и получают видеопотоки в течение звонка.

- Для видеосцен без движения фактическая скорость передачи данных может временно становиться совсем небольшой, так как видеокодек пропускает области без изменения видео с момента предыдущего образца.

В сценарии конференций:

- Аудиопотоки отправляются из конечных точек, только когда пользователи говорят.

- Все участники получают аудиопотоки.

- Если используется видео, все участники могут получать до пяти видеопотоков и один панорамный (например, с отношением сторон 20:3) видеопоток. По умолчанию пять видеопотоков основаны на журнале активного докладчика, но пользователи могут вручную выбрать участников, от которых они хотят получать видеопоток. Если активирован режим с несколькими видео, требование к разрешению и полосе пропускания каждого видеопотока будет ниже.

- Каждый участник, который включает видеосвязь пользователя, отправляет один или несколько видеопотоков. В Skype для бизнеса Server есть возможность отправки до пяти видеопотоков, чтобы оптимизировать качество видео для всех получающих клиентов. Фактическое число передаваемых видеопотоков определяется отправителем на основе мощности ЦП, доступной полосы пропускания для передачи и числа принимающих клиентов, запросивших определенных видеопоток. Чаще всего передается один поток H.264 и один поток RTVideo, если устаревший клиент подключается к конференции. Другим распространенным сценарием является передача нескольких видеопотоков H.264 (например, с разным разрешением видео) для удовлетворения запросов различных принимающих клиентов.

## <a name="managing-quality-of-service"></a>Управление качеством обслуживания
<a name="man_QOS"> </a>

Качество обслуживания (QoS) – это сетевая технология, которая используется в некоторых организациях для обеспечения оптимального качества аудио- и видеосвязи. Качество обслуживания наиболее часто используется в сетях с ограниченной пропускной способностью, в которых большое число сетевых пакетов конкурирует за относительно небольшую доступную пропускную способность. Качество обслуживания (QoS) позволяет назначать более высокие приоритеты пакетам, используемым для передачи аудио- и видеоданных. Назначение пакетам аудио- и видеоданных высокого приоритета позволяет повысить скорость аудио- и видеосвязи и уменьшить задержки по сравнению с сеансами передачи файлов, просмотра веб-страниц или резервного копирования баз данных. По этой причине сетевым пакетам, используемым при передаче файлов или резервном копировании баз данных, назначается приоритет "best effort" (наилучший вариант).

> [!NOTE]
> Обычно качество обслуживания применяется только к сеансам связи во внутренней сети. Реализация качества обслуживания подразумевает настройку серверов и маршрутизаторов для отметки пакетов с использованием определенного способа, который может не поддерживаться в Интернете или других сетях. Даже если качество обслуживания в других сетях поддерживается, нет гарантии, что качество обслуживания будет настроено точно так же, как вы настроили службу. Если вы используете MPLS, вам необходимо работать с поставщиком MPLS.

В Skype для бизнеса Server не требуется QoS, но настоятельно рекомендуется. Если у вас возникли проблемы с потерей пакетов в сети, вы можете добавить дополнительную пропускную способность или реализовать качество обслуживания. Если увеличить пропускную способность невозможно, то единственным решением может быть внедрение службы качества обслуживания.

Skype для бизнеса Server обеспечивает полную поддержку QoS: это означает, что Организации, уже использующие QoS, могут легко интегрировать Skype для бизнеса Server с существующей сетевой инфраструктурой. Для этого необходимо выполнить указанные ниже действия.

- [Включение качества обслуживания в Skype для бизнеса Server для устройств, которые не основаны на Windows](../../manage/network-management/qos/enabling-qos-for-devices-that-are-not-based-on-windows.md). По умолчанию качество обслуживания отключено для компьютеров и других устройств (например, iPhones), работающих под управлением ОС, отличных от Windows. Несмотря на то что вы можете использовать Skype для бизнеса Server для включения и отключения качества обслуживания для устройств, обычно нельзя использовать продукт для изменения кодов DSCP, используемых этими устройствами.

- [Настройка диапазонов портов и политики качества обслуживания для конференций, приложений и серверов-исправлений](../../manage/network-management/qos/configuring-port-ranges-for-your-conferencing-application-and-mediation-servers.md). Вам необходимо зарезервировать уникальный набор портов для разных типов пакетов, например аудио и видео. С помощью Skype для бизнеса Server вы не включаете и не отключайте QoS, задавая значение свойства true или false. Вместо этого для включения качества обслуживания необходимо настроить диапазоны портов и затем создать и применить групповую политику. Если в дальнейшем вы решите не использовать QoS, вы можете отключить QoS, удалив соответствующие объекты групповой политики.

- [Настройка диапазонов портов и политики качества обслуживания для пограничных серверов](../../manage/network-management/qos/configuring-port-ranges-for-your-edge-servers.md). Вы можете использовать на пограничных серверах и остальных серверах одни и те же диапазоны портов, хотя это необязательно. Настройка политики качества обслуживания выполняется только для внутренней стороны пограничного сервера. Это связано с тем, что Служба QoS разработана для внутренней сети, а не через Интернет.

- [Настройка диапазонов портов и политики качества обслуживания для ваших клиентов в Skype для бизнеса Server](../../manage/network-management/qos/configuring-port-ranges-for-your-skype-clients.md). Эти диапазоны портов применяются только к клиентским компьютерам и обычно не совпадают с диапазонами портов, заданными на серверах. Обратите внимание, что Skype для бизнеса Server не поддерживает QoS для операционных систем Windows, отличных от Windows 10.


> [!NOTE]
> Если вы используете Windows Server 2012 или Windows Server 2012 R2, вам может потребоваться новый набор командлетов Windows PowerShell, которые можно использовать для управления QoS на этой платформе. Дополнительные сведения можно найти [в разделе командлеты QoS сети в Windows PowerShell](https://go.microsoft.com/fwlink/p/?LinkId=285379).

Качество обслуживания также обсуждается в разделе [планирование, мониторинг и устранение неполадок в работе с Lync Server](https://www.microsoft.com/en-us/download/details.aspx?id=39084) с дополнительными сведениями и глубиной. Несмотря на то, что содержимое явным образом ссылается на Lync 2010 и Lync 2013, вопросы, касающиеся Skype для бизнеса Server, не изменяются.
