## <a name="server-hardware"></a>Серверное оборудование
<a name="S_hard"> </a>

Сетевой адаптер каждого сервера в топологии Skype для бизнеса Server должен поддерживать не менее 1 гигабит в секунду (Гбит/с). Как правило, все роли серверов в топологии Skype для бизнеса Server следует подключать с помощью локальной сети с низкой задержкой и высокой пропускной способностью. Размер локальной сети зависит от размера топологии:

- Вpologies Standard Edition серверы должны быть в сети с поддержкой Ethernet 1 Гбит/с или эквивалентной сети.

- В техологиях Enterprise Edition большинство серверов должны быть в сети, поддерживающей более 1 Гбит/с, особенно при поддержке аудио- и видеоконференций и общего доступа к приложениям.

Для интеграции с телефонной сетью общего пользования (ТСОП) можно использовать линии T1/E1 или распределение каналов SIP.

## <a name="audiovideo-network-requirements"></a>Требования к сети для аудио- и видеосвязи
<a name="AV_req"> </a>

Требования к сети для аудио- и видеосвязи в развертывании Skype для бизнеса Server включают следующие:

- При развертывании одного сервера или пула с помощью балансировки нагрузки  на DNS можно настроить внешний брандмауэр для выполнения перевода сетевых адресов (NAT). Нельзя настроить внутренний брандмауэр _для_ выполнения NAT. Подробные сведения см. в [планировании портов и брандмауэра.](../edge-server-deployments/edge-environmental-requirements.md#port-and-firewall-planning)

    > [!IMPORTANT]
    > Если у вас есть пул edge и используется аппаратный балансировка нагрузки, необходимо использовать общедоступные IP-адреса на этих серверах, и нельзя использовать NAT для серверов или пула на устройстве с возможностью NAT (например, брандмауэра или локального коммутатора). For details, see [Edge Server scenarios in Skype for Business Server](../edge-server-deployments/scenarios.md).

- Если в вашей организации используется инфраструктура качества обслуживания (QoS), подсистема передачи мультимедиа настроена на работу в этой существующей инфраструктуре.

- Если вы используете протоколы IPsec, мы рекомендуем отключить их для диапазона портов, который используется для аудио- и видеотрафика. Подробные сведения см. в [исключениях IPsec.](#ipsec-exceptions)

Чтобы обеспечить оптимальное качество мультимедиа, сделайте следующее:

- Подготовка сетевых ссылок для поддержки пропускной способности 65 килобит в секунду (кбит/с) для аудиопотока и 500 Кбит/с на видеопоток, если они включены, в периоды пиковой нагрузки. В сеансе двухпутного аудио- или видеосвязи используются два потока, поэтому для простого аудио- и телефонного подключения потребуется 130 Кб/с, чтобы охватить каждый поток. Видео также будет использовать всего 1000 Кбит/с для подключения вверх и вниз.

- To with unexpected spikes in traffic and increased usage over time, Skype for Business Server media endpoints can adapt to varying network conditions and support three times the throughput for audio and video while still maintaining acceptable quality. Не думайте, что такая адаптируемость будет маскировать проблему, когда сеть не вймяется в резерв. В сети с ненадегой способность конечных точек мультимедиа Skype для бизнеса Server динамически работать с различными условиями сети (например, временной потерей пакетов) снижается.

- Для сетевых связей, где подготовка очень затратна и сложна, может потребоваться подготовка для меньшего объема трафика. В этом сценарии гибкость конечных точек мультимедиа Skype для бизнеса Server позволит совмещать разницу между объемом трафика и пиковым уровнем трафика за счет некоторого снижения качества голосовой связи. Кроме того, в противном случае будет уменьшаться запас, доступный для сглавия резких пиков трафика.

- Для ссылок, которые не могут быть правильно размещены в краткосрочной перспективе (например, сайт, использующий очень неудовлетворительные связи WAN), рассмотрите возможность отключения видео для определенных пользователей.

- Подготовка сети для обеспечения максимальной задержки от конца до конца (150 миллисекунд (мс) при пиковой нагрузке. Задержка — это одно из недостатков сети, которое не могут уменьшить компоненты мультимедиа Skype для бизнеса Server, и важно найти и устранить слабые точки.

- Для серверов, на которые запущено антивирусное ПРОГРАММНОЕ ОБЕСПЕЧЕНИЕ, включаем все серверы со Skype для бизнеса Server в список исключений, чтобы обеспечить оптимальную производительность и качество звука.

Кодеком по умолчанию для видео является стандарт H.264/MPEG-4 part 10 Advanced Video Coding, а также его масштабируемые расширения для временного масштабирования. Для обеспечения взаимодействия с устаревшими клиентами кодек RTVideo по-прежнему используется для одноранговых звонков между Skype для бизнеса Server и устаревшими клиентами. В сеансах конференц-связи со Skype для бизнеса Server и устаревшими клиентами конечная точка Skype для бизнеса Server может закодировать видео с помощью видеокодеков и отправить поток H.264 клиентам Skype для бизнеса Server и bitstream RTVideo на устаревшие клиенты.

Требуемая полоса пропускания зависит от разрешения, качества, частоты кадров и объема движения или изменения изображения. Для каждого разрешения существует два уровня битов:

- **Максимальная скорость полезной нагрузки** Это скорость, которую конечная точка будет использовать для разрешения при максимальной частоте кадров. Это значение обеспечивает наивысшее качество видео и звука.

- **Минимальная скорость полезной нагрузки** Это скорость, ниже которой конечная точка Skype для бизнеса Server переключается на следующее более низкое разрешение. Чтобы гарантировать определенное разрешение, доступная скорость полезной нагрузки видео не должна быть ниже этой минимальной скорости для этого разрешения. Это значение помогает понять минимальное возможное значение, если максимальная скорость бита недоступна или недоступна. Для некоторых пользователей такое видео с низкой скоростью может привести к неприемлемым видеосвязи, поэтому следует соблюдать осторожность при минимальной скорости полезной нагрузки видео. Обратите внимание, что для статических, неизменяющихся видеосъемок фактическая скорость бита может временно опадать ниже минимальной скорости.

Skype для бизнеса Server поддерживает множество решений. Это позволяет Skype для бизнеса Server адаптироваться к различным возможностям пропускной способности сети и получения клиентов. Соотношение сторон по умолчанию для Skype для бизнеса Server составляет 16:9. Устаревшее соотношение сторон 4:3 по-прежнему поддерживается для веб-камер, которые не позволяют захват в пропорциях 16:9.

FeC видео всегда включается в скорость полезной нагрузки видео при его использования, чтобы не было отдельных значений для видео FEC и без FEC видео.

Конечные точки не передают потоковые пакеты аудио или видео непрерывно. В зависимости от сценария существуют различные уровни активности потока, указывающие, как часто пакеты передаются в потоке. Активность потока зависит от мультимедиа и сценария и не зависит от используемого кодека. В одноранговом сценарии:

- Конечные точки отправляют звуковые потоки, только когда пользователи говорят.

- Оба участника получают аудиопотоки.

- Если используется видео, обе конечные точки отправляют и получают видеопотоки во время вызова.

- Для статических видеосъемок фактическая скорость бита может временно быть очень низкой, так как видеокодец пропускает области кода видео без изменений, начиная с предыдущего примера.

В сценарии конференций:

- Аудиопотоки отправляются из конечных точек, только когда пользователи говорят.

- Все участники получают аудиопотоки.

- Если используется видео, все участники могут получать до пяти видеопотоков и один панорамный (например, с отношением сторон 20:3) видеопоток. По умолчанию пять видеопотоков основаны на журнале активного докладчика, но пользователи могут вручную выбрать участников, от которых они хотят получать видеопоток. Если включено несколько видео, требования к разрешению и пропускной способности для каждого видеопотока будут ниже.

- Каждый участник, который включает видеопоток отправки пользователя, будет отправлять один или несколько видеопотоков. Skype для бизнеса Server может отправлять до пяти видеопотоков для оптимизации качества видео для всех принимающих клиентов. Фактическое число передаваемых видеопотоков определяется отправителем на основе мощности ЦП, доступной полосы пропускания для передачи и числа принимающих клиентов, запросивших определенных видеопоток. Чаще всего передается один поток H.264 и один поток RTVideo, если устаревший клиент подключается к конференции. Другим распространенным сценарием является передача нескольких видеопотоков H.264 (например, с разным разрешением видео) для удовлетворения запросов различных принимающих клиентов.

## <a name="managing-quality-of-service"></a>Управление качеством обслуживания
<a name="man_QOS"> </a>

Качество обслуживания (QoS) — это технология сети, используемая в некоторых организациях для обеспечения оптимального взаимодействия пользователей с аудио- и видеосвязью. QoS чаще всего используется в сетях с ограниченной пропускной способностью: из-за большого количества сетевых пакетов, конкурирующих за довольно небольшой объем доступной пропускной способности, QoS позволяет администраторам назначать более высокие приоритеты пакетам с аудио- или видеосъемными данными. Если приоритет этих пакетов выше, аудио- и видеосвязь, скорее всего, завершится быстрее и с меньшим прерываниям, чем сетевые сеансы, включающие передачу файлов, просмотр веб-страниц или резервное копирование баз данных. Это потому, что сетевые пакеты, используемые для передачи файлов или резервного копирования баз данных, имеют приоритет "наилучших усилий".

> [!NOTE]
> Как правило, QoS применяется только к сеансам связи во внутренней сети. При реализации QoS серверы и маршрутизаторы настраиваются для поддержки маркировки пакетов определенным образом, который может не поддерживаться в Интернете или других сетях. Даже если качество обслуживания поддерживается в других сетях, нет гарантии, что качество обслуживания будет настроено точно так же, как вы настроили службу. Если вы используете MPLS, вам потребуется работать с поставщиком MPLS.

Skype для бизнеса Server не требует QoS, но настоятельно рекомендуется. Если в сети имеются проблемы с потерей пакетов, доступные решения должны добавить полосу пропускания или реализовать QoS. Если добавление большей пропускной способности невозможно, реализация QoS может быть единственным платным решением проблемы.

Skype для бизнеса Server обеспечивает полную поддержку QoS: это означает, что организации, которые уже используют QoS, могут легко интегрировать Skype для бизнеса Server в существующую сетевую инфраструктуру. Для этого выполните следующие действия:

- [Включение QoS в Skype для бизнеса Server](../../manage/network-management/qos/enabling-qos-for-devices-that-are-not-based-on-windows.md)для устройств, не основанных на Windows. По умолчанию качество обслуживания отключено для компьютеров и других устройств (например, iPhones), работающих под управлением ОС, отличных от Windows. Несмотря на то что skype для бизнеса Server можно использовать для обеспечения и отключения качества обслуживания для устройств, этот продукт обычно нельзя использовать для изменения кодов DSCP, используемых этими устройствами.

- Настройка диапазонов портов и политики качества обслуживания для серверов [conferencing, Application и Mediation.](../../manage/network-management/qos/configuring-port-ranges-for-your-conferencing-application-and-mediation-servers.md) Вам необходимо зарезервировать уникальный набор портов для разных типов пакетов, например аудио и видео. С помощью Skype для бизнеса Server вы не включаете и не отключает QoS, устанавливая для свойства значение True или False. Вместо этого можно включить QoS, настроив диапазоны портов, а затем создав и применив групповую политику. Если позднее вы решите не использовать QoS, вы можете "отключить" QoS, удалив соответствующие объекты групповой политики.

- [Настройка диапазонов портов и политики качества обслуживания для ваших серверов.](../../manage/network-management/qos/configuring-port-ranges-for-your-edge-servers.md) Вы можете использовать на пограничных серверах и остальных серверах одни и те же диапазоны портов, хотя это необязательно. Настройка политики QoS может быть сделана только на внутренней стороне ваших серверов. Это происходит потому, что QoS предназначен для использования во внутренней сети, а не в Интернете.

- [Настройка диапазонов портов](../../manage/network-management/qos/configuring-port-ranges-for-your-skype-clients.md)и политики качества обслуживания для клиентов в Skype для бизнеса Server. Эти диапазоны портов применяются только к клиентских компьютерам и обычно отличаются от диапазонов портов, настроенных на серверах. Обратите внимание, что Skype для бизнеса Server не поддерживает QoS для операционных систем Windows, кроме Windows 10.


> [!NOTE]
> Если вы используете Windows Server 2012 или Windows Server 2012 R2, вас может заинтересовать новый набор Windows PowerShell для управления QoS на этой платформе. Дополнительные сведения [см.](https://go.microsoft.com/fwlink/p/?LinkId=285379)в подмосковных Windows PowerShell.

QoS также обсуждается в официальном проекте по планированию, мониторингу и устранению неполадок в [Lync Server](https://www.microsoft.com/download/details.aspx?id=39084) с дополнительными сведениями и глубиной. Хотя содержимое явно относится к Lync 2010 и Lync 2013, соображения для Skype для бизнеса Server не изменились.
