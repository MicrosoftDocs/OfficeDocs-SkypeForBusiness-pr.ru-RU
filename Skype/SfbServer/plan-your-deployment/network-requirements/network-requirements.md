## <a name="server-hardware"></a>Серверное оборудование
<a name="S_hard"> </a>

Сетевой адаптер каждого сервера в топологии Skype для бизнеса Server должен поддерживать по крайней мере 1 гигабит в секунду (Гбит/с). В общем случае необходимо подключить все роли сервера в топологии Skype для бизнеса Server, используя низкую задержку и локальную сеть с высокой пропускной способностью (ЛВС). Размер локальной сети зависит от размера топологии:

- В топологиях Standard Edition серверы должны быть в сети, поддерживающей 1 Гбит/с, или аналогичной.

- В топологиях Enterprise Edition большинство серверов должны находиться в сети, поддерживающей более 1 Гбит/с, особенно при поддержке конференц-связи аудио-и видеоданных (аудио-и видеоконференций) и общего доступа к приложениям.

Для интеграции с телефонной сетью общего пользования (ТСОП) можно использовать линии T1/E1 или распределение каналов SIP.

## <a name="audiovideo-network-requirements"></a>Требования к аудио-и видеосети
<a name="AV_req"> </a>

Требования к сети для аудио/видео (A/V) в развертывании Skype для бизнеса Server включают следующее:

- Если вы развертываете один пограничный сервер или пограничный пул с помощью балансировки нагрузки на DNS, вы можете настроить _внешний_ брандмауэр для выполнения преобразования сетевых адресов (NAT). Невозможно настроить _внутренний_ брандмауэр для выполнения преобразования сетевых адресов (NAT). Подробнее: [планирование портов и брандмауэров](../edge-server-deployments/edge-environmental-requirements.md#port-and-firewall-planning).

    > [!IMPORTANT]
    > Если у вас есть пограничный пул, использующий аппаратную подсистему балансировки нагрузки, необходимо использовать общедоступные IP-адреса на пограничных серверах, и вы не можете использовать NAT для серверов или пула на устройстве с поддержкой NAT (например, через брандмауэр или коммутатор локальной сети). Дополнительные сведения см [в разделе сценарии пограничного сервера в Skype для бизнеса Server](../edge-server-deployments/scenarios.md).

- Если в вашей организации используется инфраструктура качества обслуживания (QoS), подсистема передачи мультимедиа настроена на работу в этой существующей инфраструктуре.

- Если вы используете протоколы IPsec, мы рекомендуем отключить их для диапазона портов, который используется для аудио- и видеотрафика. [Дополнительные сведения см.](#ipsec-exceptions)

Чтобы обеспечить оптимальное качество мультимедиа, выполните следующие действия:

- Подготовьте сетевые ссылки для поддержки пропускной способности 65 килобит в секунду (кбит/с) для каждого звукового потока и 500 Кбит/с для каждого видеопотока, если они включены, в периоды пиковой нагрузки. В двустороннем аудио-или видеосеансе используются два потока, поэтому для простого подключения к аудио-и телефонной линии потребуется 130Kbps для охвата каждого потока. В видеоролике также будет использоваться общее количество 1000 Кбит/с для переноса вышестоящего и подчиненного подключения.

- Чтобы справиться с неожиданными пиковиями трафика и при увеличении использования с течением времени, конечные точки сервера мультимедиа Skype для бизнеса Server могут адаптироваться к различным условиям сети и поддерживать в три раза больше пропускную способность для аудио-и видеосвязи, сохраняя приемлемое качество. Не следует предполагать, что эта адаптируемость будет маскировать проблемы при подготовке сети. В подготовной сети способность конечных точек мультимедиа Skype для бизнеса Server динамически работать с переменными состояния сети (например, временная потеря высокой потери пакетов).

- Для сетевых ссылок, в которых подготовка является очень дорогостоящей и сложной, может потребоваться рассмотреть возможность подготовки к снижению объема трафика. В этом сценарии для эластичности конечных точек мультимедиа Skype для бизнеса Server уменьшается разница между объемом трафика и уровнем пикового трафика за счет снижения качества голосовой связи. Кроме того, будет уменьшено уменьшение запаса, которое можно использовать в других случаях для снижения интенсивности трафика.

- Для ссылок, которые не могут быть правильно подготовлены в краткосрочном периоде (например, сайт, использующий очень плохую связь WAN), рассмотрите возможность отключения видео для определенных пользователей.

- Подготовьте сеть, чтобы обеспечить максимальную сквозную задержку (задержку) 150 миллисекунд (МС) при пиковой нагрузке. Задержка — это сетевое нарушение, в результате которого компоненты мультимедиа Skype для бизнеса Server не могут уменьшиться, и важно найти и устранить слабые точки.

- Для серверов, на которых работает антивирусное программное обеспечение, включите все серверы со Skype для бизнеса Server в список исключений, чтобы обеспечить оптимальную производительность и качество звука.

Чтобы обеспечить взаимодействие с устаревшими клиентами, кодек RTVideo по-прежнему используется для одноранговых вызовов между Skype для бизнеса Server и устаревшими клиентами. В сеансах конференц-связи с клиентами Skype для бизнеса Server и устаревшими клиентами конечная точка Skype для бизнеса Server может кодировать видео с помощью видеокодеков и отправлять битстреам. 264 на клиенты Skype для бизнеса Server и RTVideo битстреам в устаревший заказчика.

Требуемая пропускная способность зависит от разрешения, качества, частоты кадров и объема движений или изменения изображения. Для каждого решения существует две соответствующие скорости:

- **Максимальная скорость передачи полезных данных** Это скорость, с которой конечная точка будет использовать для разрешения при максимальной частоте кадров. Это значение позволяет максимально высокое качество видео и звука.

- **Скорость передачи данных по минимальной нагрузке** Это скорость, под которой конечная точка Skype для бизнеса Server будет переключаться на следующее более низкое разрешение. Чтобы гарантировать определенное разрешение, скорость передачи полезных данных видео не должна превышать эту минимальную скорость потока для этого решения. Это значение помогает определить наименьшее допустимое значение, если максимальная скорость недоступна или непрактична. Для некоторых пользователей такой недопустимый видеоролик может обеспечить неприемлемое взаимодействие с видео, поэтому используйте эти минимальные скорости полезных данных видео. Обратите внимание на то, что для статических, неменяющихся видеороликов фактическая скорость потока может временно попадать ниже минимальной скорости.

Skype для бизнеса Server поддерживает множество разрешений. Это позволяет Skype для бизнеса Server настраивать пропускную способность сети и получать клиентские возможности. Пропорциями по умолчанию для Skype для бизнеса Server является 16:9. Пропорции 4:3 прежних версий по-прежнему поддерживаются для камер, которые не поддерживают захват в пропорции 16:9.

Video FEC всегда включается в битовую ставку полезных данных видео, если она используется, поэтому для FEC с видео и без видео FEC не существует отдельных значений.

Конечные точки не передают потоковые пакеты аудио или видео непрерывно. В зависимости от сценария существуют различные уровни активности потока, указывающие, как часто пакеты передаются в потоке. Активность потока зависит от мультимедиа и сценария и не зависит от используемого кодека. В одноранговом сценарии:

- Конечные точки отправляют звуковые потоки только тогда, когда говорят пользователи.

- Оба участника получают аудиопотоки.

- Если используется видео, обе конечные точки отправляют и получают видеопотоки во время вызова.

- Для статических видеороликов фактическая скорость потока может временно быть очень малой, так как видеокодек будет пропускать области кодирования видео без изменений с момента предыдущего примера.

В сценарии конференций:

- Аудиопотоки отправляются из конечных точек, только когда пользователи говорят.

- Все участники получают аудиопотоки.

- Если используется видео, все участники могут получать до пяти видеопотоков и один панорамный (например, с отношением сторон 20:3) видеопоток. По умолчанию пять видеопотоков основаны на журнале активного докладчика, но пользователи могут вручную выбрать участников, от которых они хотят получать видеопоток. Если включено несколько видеороликов, требование к разрешению и пропускной способности для каждого видеопотока будет ниже.

- Каждый участник, который включает видеопоток отправки пользователя, отправляет один или несколько видеопотоков. Skype для бизнеса Server обеспечивает возможность отправки до пяти видеопотоков для оптимизации качества видео для всех принимающих клиентов. Фактическое число передаваемых видеопотоков определяется отправителем на основе мощности ЦП, доступной полосы пропускания для передачи и числа принимающих клиентов, запросивших определенных видеопоток. Чаще всего передается один поток H.264 и один поток RTVideo, если устаревший клиент подключается к конференции. Другим распространенным сценарием является передача нескольких видеопотоков H.264 (например, с разным разрешением видео) для удовлетворения запросов различных принимающих клиентов.

## <a name="managing-quality-of-service"></a>Управление качеством обслуживания
<a name="man_QOS"> </a>

Качество обслуживания (QoS) — это сетевая технология, которая используется в некоторых организациях для обеспечения оптимального взаимодействия конечных пользователей с аудио-и видеосообщениями. Служба QoS чаще всего используется в сетях, в которых пропускная способность ограничена: с большим количеством сетевых пакетов, конкурирующих за довольно малый объем доступной полосы пропускания, качество обслуживания позволяет администраторам назначать более высокие приоритеты для пакетов с аудио-и видеоданными. Предоставляя этим пакетам более высокий приоритет, воспроизведение аудио-и видеосвязи, скорее всего, будет выполняться быстрее и с меньшим перерывом, чем сетевые сеансы, связанные с переносом файлов, просмотром веб-страниц или резервным копированием баз данных. Это связано с тем, что для сетевых пакетов, используемых для переноса файлов или резервных копий базы данных, назначается приоритет "Лучший объем работы".

> [!NOTE]
> Как правило, качество обслуживания применяется только к сеансам связи во внутренней сети. При реализации QoS вы настраиваете серверы и маршрутизаторы так, чтобы они поддерживали маркировку пакетов определенным образом, которые могут не поддерживаться в Интернете или в других сетях. Даже если качество службы поддерживается в других сетях, нет гарантии того, что качество обслуживания будет настроено точно так же, как и служба. Если вы используете MPLS, вам потребуется работать с поставщиком MPLS.

В Skype для бизнеса Server не требуется качество обслуживания, но настоятельно рекомендуется. Если у вас возникли проблемы с потерей пакетов в сети, вы можете увеличить пропускную способность или повысить пропускную способность или реализовать качество обслуживания. Если не удается увеличить пропускную способность, то для решения этой проблемы можно использовать только платный доступ к качеству.

Skype для бизнеса Server обеспечивает полную поддержку QoS: это означает, что Организации, которые уже используют QoS, могут легко интегрировать Skype для бизнеса Server с существующей сетевой инфраструктурой. Для этого необходимо выполнить указанные ниже действия.

- [Включение качества обслуживания в Skype для бизнеса Server для устройств, не основанных на Windows](../../manage/network-management/qos/enabling-qos-for-devices-that-are-not-based-on-windows.md). По умолчанию качество обслуживания отключено для компьютеров и других устройств (например, iPhones), работающих под управлением ОС, отличных от Windows. Несмотря на то, что вы можете использовать Skype для бизнеса Server для включения и отключения качества обслуживания для устройств, обычно невозможно использовать продукт для изменения кодов DSCP, используемых этими устройствами.

- [Настройка диапазонов портов и политики качества обслуживания для серверов конференц-связи, приложений и серверов-посредников](../../manage/network-management/qos/configuring-port-ranges-for-your-conferencing-application-and-mediation-servers.md). Вам необходимо зарезервировать уникальный набор портов для разных типов пакетов, например аудио и видео. С помощью Skype для бизнеса Server вы не включаете или отключаете качество обслуживания, присвоив свойству значение true или false. Вместо этого вы включаете службу QoS, настраивая диапазоны портов, а затем создавая и применяя групповую политику. Если позднее вы решите не использовать службу QoS, вы можете отключить QoS, удалив соответствующие объекты групповой политики.

- [Настройка диапазонов портов и политики качества обслуживания для пограничных серверов](../../manage/network-management/qos/configuring-port-ranges-for-your-edge-servers.md). Вы можете использовать на пограничных серверах и остальных серверах одни и те же диапазоны портов, хотя это необязательно. Настройка политики качества обслуживания выполняется только для внутренней стороны пограничных серверов. Это связано с тем, что качество обслуживания предназначено для внутренней сети, а не для Интернета.

- [Настройка диапазонов портов и политики качества обслуживания для клиентов в Skype для бизнеса Server](../../manage/network-management/qos/configuring-port-ranges-for-your-skype-clients.md). Эти диапазоны портов применяются только к клиентским компьютерам и обычно отличаются от диапазонов портов, настроенных на серверах. Обратите внимание, что Skype для бизнеса Server не поддерживает QoS для операционных систем Windows, отличных от Windows 10.


> [!NOTE]
> Если вы используете Windows Server 2012 или Windows Server 2012 R2, возможно, вас интересует новый набор командлетов Windows PowerShell, доступных для управления QoS на этой платформе. Дополнительные сведения см. [в разделе командлеты QoS сети в Windows PowerShell](https://go.microsoft.com/fwlink/p/?LinkId=285379).

Качество обслуживания также обсуждается в [планировании сети, мониторинге и устранении неполадок в Lync Server](https://www.microsoft.com/download/details.aspx?id=39084) с дополнительными сведениями и более подробными сведениями. Несмотря на то, что содержимое явно относится к Lync 2010 и Lync 2013, вопросы для Skype для бизнеса Server не меняются.
