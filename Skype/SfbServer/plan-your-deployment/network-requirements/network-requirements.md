## <a name="server-hardware"></a>Серверное оборудование
<a name="S_hard"> </a>

Сетевой адаптер каждого сервера в топологии Skype для бизнеса Server должен поддерживать не менее 1 гигабит в секунду (Gbps). В общем, необходимо подключить все роли сервера в топологии Skype для бизнеса Server с помощью низкой задержки и высокой пропускной способности локальной сети области (LAN). Размер локальной сети зависит от размера топологии:

- В выпуск Standard топологии серверы должны быть в сети, которая поддерживает 1 Гбит/с Ethernet или эквивалент.

- В выпуск Enterprise топологии большинство серверов должно быть в сети, которая поддерживает более 1 Гбит/с, особенно при поддержке аудио- и видеоконференций и общего доступа к приложениям.

Для интеграции с телефонной сетью общего пользования (ТСОП) можно использовать линии T1/E1 или распределение каналов SIP.

## <a name="audiovideo-network-requirements"></a>Требования к сети аудио- и видеосвязи
<a name="AV_req"> </a>

Требования к сети для аудио/видео (A/V) в развертывании Skype для бизнеса Server включают следующие:

- При развертывании одного edge Server или пула Edge с помощью балансировки нагрузки DNS можно настроить внешний брандмауэр для выполнения перевода сетевых адресов (NAT). Нельзя настроить внутренний _брандмауэр_ для выполнения NAT. Подробные сведения см. в [материале Планирование порта и брандмауэра](../edge-server-deployments/edge-environmental-requirements.md#port-and-firewall-planning).

    > [!IMPORTANT]
    > Если у вас есть пул Edge и используется балансировка нагрузки оборудования, необходимо использовать общедоступные IP-адреса на edge Servers и нельзя использовать NAT для серверов или пула на устройстве, способном nat (например, брандмауэре или локальном коммутаторе). Подробные сведения см. в сценариях [Edge Server в Skype для бизнеса Server](../edge-server-deployments/scenarios.md).

- Если в вашей организации используется инфраструктура качества обслуживания (QoS), подсистема передачи мультимедиа настроена на работу в этой существующей инфраструктуре.

- Если вы используете протоколы IPsec, мы рекомендуем отключить их для диапазона портов, который используется для аудио- и видеотрафика. Подробные сведения см. в [сведениях об исключениях IPsec](#ipsec-exceptions).

Чтобы обеспечить оптимальное качество мультимедиа, сделайте следующее:

- Предоставление сетевых ссылок для поддержки пропускной способности 65 килобит в секунду (Kbps) для аудиопотока и 500 Кбит/с на видеопоток, если они включены, в периоды максимального использования. Двухпутная аудио- или видео-сессия использует два потока, поэтому для простого подключения к аудио/телефону потребуется 130Kbps для каждого потока. Видео также будет использовать 1000 Кбит/с для переноса подключения вверх по течению и вниз по течению.

- Чтобы справиться с непредвиденными всплесками трафика и увеличением использования с течением времени, Skype для бизнеса Server конечные точки мультимедиа могут адаптироваться к различным условиям сети и поддерживать в три раза пропускную способность для аудио- и видео, сохраняя при этом приемлемое качество. Не думайте, что эта адаптивность замаскировала проблему при недооптовке сети. В недоозначаемой сети снижается способность конечных точек мультимедиа Skype для бизнеса Server динамически работать с различными сетевыми условиями (например, временная высокая потеря пакета).

- Для сетевых ссылок, где подготовка очень затратна и трудна, может потребоваться подготовка к более низкому объему трафика. В этом сценарии пусть эластичность конечных точек Skype для бизнеса Server средств массовой информации поглощает разницу между объемом трафика и пиковым уровнем трафика за счет некоторого снижения качества голосовой связи. Кроме того, будет уменьшаться в загонах в противном случае доступны для поглощения внезапных пиков трафика.

- Для ссылок, которые не могут быть правильно размещены в краткосрочной перспективе (например, на сайте с очень плохими ссылками WAN), рассмотрите возможность отключения видео для определенных пользователей.

- Предоставление сети для обеспечения максимальной задержки (задержки) в 150 миллисекунд (мс) при пиковой нагрузке. Задержка — это одно сетевое ухудшение, которое Skype для бизнеса Server компонентов мультимедиа не может уменьшиться, и важно найти и устранить слабые места.

- Для серверов, на которые работает антивирусное программное обеспечение, включаем все серверы, Skype для бизнеса Server в списке исключений, чтобы обеспечить оптимальную производительность и качество звука.

## <a name="ipsec-exceptions"></a>Исключения IPsec

Кодеком для видео по умолчанию является стандарт предварительного кодирования видео H.264/MPEG-4 части 10, а также расширения масштабируемого видеокодирования для временной масштабируемости. Для обеспечения взаимодействия с устаревшими клиентами кодек RTVideo по-прежнему используется для одноранговых звонков между Skype для бизнеса Server и устаревшими клиентами. В сеансах конференций с Skype для бизнеса Server и устаревшими клиентами конечная точка Skype для бизнеса Server может кодировать видео с помощью как видеокодок, так и отправлять битстрим H.264 клиентам Skype для бизнеса Server и битстрим RTVideo устаревшим клиентам.

Пропускная способность зависит от разрешения, качества, частоты кадров и объема движения или изменения изображения. Для каждого разрешения существуют две узаконивные битовые ставки:

- **Максимальная скорость бита полезной нагрузки** Это скорость бита, которую конечная точка будет использовать для разрешения с максимальной частотой кадров. Это значение позволяет обеспечить самое высокое качество видео и звука.

- **Минимальная скорость бита полезной нагрузки** Это битовая скорость, ниже которой конечная Skype для бизнеса Server переключается на следующее нижнее разрешение. Чтобы гарантировать определенное разрешение, доступная скорость загрузки видео не должна быть ниже этого минимального бита для этого разрешения. Это значение поможет вам понять самое низкое значение, если максимальная скорость бита недоступна или практичная. Для некоторых пользователей такое видео с низкой скоростью может обеспечить неприемлемое видео, поэтому используйте осторожность при использовании этих минимальных битратов нагрузки видео. Обратите внимание, что для статичных, неизменяющихся видеосюстановок фактическая скорость бита может временно опуститься ниже минимальной скорости бита.

Skype для бизнеса Server поддерживает множество решений. Это позволяет Skype для бизнеса Server для настройки пропускной способности сети и получения клиентских возможностей. Соотношение аспектов по умолчанию для Skype для бизнеса Server 16:9. Устаревшее соотношение аспектов 4:3 по-прежнему поддерживается веб-камерами, которые не позволяют захватывать в соотношении аспектов 16:9.

Видео FEC всегда включено в скорость бита полезной нагрузки видео, когда она используется, чтобы не было отдельных значений для с видео FEC и без видео FEC.

Конечные точки не передают потоковые пакеты аудио или видео непрерывно. В зависимости от сценария существуют различные уровни активности потока, указывающие, как часто пакеты передаются в потоке. Активность потока зависит от мультимедиа и сценария и не зависит от используемого кодека. В одноранговом сценарии:

- Конечные точки отправляют звуковые потоки только при разговоре пользователей.

- Оба участника получают аудиопотоки.

- Если видео используется, обе конечные точки отправляют и получают видеопотоки во время вызова.

- Для статичных видео-сцен фактическая скорость бита может временно быть очень низкой, так как кодек видео пропустит кодить области видео без изменений с момента предыдущего примера.

В сценарии конференций:

- Аудиопотоки отправляются из конечных точек, только когда пользователи говорят.

- Все участники получают аудиопотоки.

- Если используется видео, все участники могут получать до пяти видеопотоков и один панорамный (например, с отношением сторон 20:3) видеопоток. По умолчанию пять видеопотоков основаны на журнале активного докладчика, но пользователи могут вручную выбрать участников, от которых они хотят получать видеопоток. Если включено несколько видео, требования к разрешению и пропускной способности для каждого из видеопотоков будут ниже.

- Каждый участник, включив поток видео отправки пользователя, отправит один или несколько видеопотоков. Skype для бизнеса Server возможность отправки до пяти видеопотоков для оптимизации качества видео для всех принимающих клиентов. Фактическое число передаваемых видеопотоков определяется отправителем на основе мощности ЦП, доступной полосы пропускания для передачи и числа принимающих клиентов, запросивших определенных видеопоток. Чаще всего передается один поток H.264 и один поток RTVideo, если устаревший клиент подключается к конференции. Другим распространенным сценарием является передача нескольких видеопотоков H.264 (например, с разным разрешением видео) для удовлетворения запросов различных принимающих клиентов.

## <a name="managing-quality-of-service"></a>Управление качеством обслуживания
<a name="man_QOS"> </a>

Качество службы (QoS) — это технология сетей, которая используется в некоторых организациях для обеспечения оптимального взаимодействия с конечными пользователями для аудио- и видеосвязи. QoS чаще всего используется в сетях с ограниченной пропускной способностью: при большом количестве сетевых пакетов, конкурирующих за довольно небольшой объем доступной пропускной способности, QoS позволяет администраторам назначать более высокие приоритеты пакетам с аудио- или видео данными. Предоставляя этим пакетам более высокий приоритет, аудио- и видеосвязь, скорее всего, завершится быстрее и с меньшим перерывом, чем сетевые сеансы, связанные с такими вещами, как передача файлов, просмотр веб-страниц или резервное копирование баз данных. Это потому, что сетевые пакеты, используемые для передачи файлов или резервного копирования баз данных, имеют приоритет "наилучших усилий".

> [!NOTE]
> Как правило, QoS применяется только к сеансам связи во внутренней сети. При реализации QoS настраиваются серверы и маршрутизаторы для поддержки маркировки пакетов определенным образом, которые могут не поддерживаться в Интернете или в других сетях. Даже если качество службы поддерживается в других сетях, нет никакой гарантии, что QoS будет настроен точно так же, как вы настраивали службу. Если вы используете MPLS, вам потребуется работать с поставщиком MPLS.

Skype для бизнеса Server не требует QoS, но настоятельно рекомендуется. Если у вас проблемы с потерей пакетов в сети, доступные решения должны добавить больше пропускной способности или реализовать QoS. Если добавление большей пропускной способности невозможно, реализация QoS может быть единственным платным решением проблемы.

Skype для бизнеса Server полностью поддерживает QoS: это означает, что организации, которые уже используют QoS, могут легко интегрировать Skype для бизнеса Server в существующую сетевую инфраструктуру. Для этого необходимо следовать следующим шагам:

- [Включение QoS в Skype для бизнеса Server для устройств,](../../manage/network-management/qos/enabling-qos-for-devices-that-are-not-based-on-windows.md) не основанных на Windows. По умолчанию качество обслуживания отключено для компьютеров и других устройств (например, iPhones), работающих под управлением ОС, отличных от Windows. Хотя вы можете Skype для бизнеса Server для обеспечения и отключения качества службы для устройств, вы обычно не можете использовать продукт для изменения кодов DSCP, используемых этими устройствами.

- [Настройка диапазонов портов и политика качества обслуживания для серверов conferencing, Application и Mediation](../../manage/network-management/qos/configuring-port-ranges-for-your-conferencing-application-and-mediation-servers.md). Вам необходимо зарезервировать уникальный набор портов для разных типов пакетов, например аудио и видео. С помощью Skype для бизнеса Server не включить или отключить QoS, установив значение свойства True или False. Вместо этого вы включаете QoS, настроив диапазоны портов, а затем создав и применяя групповую политику. Если позже вы решите не использовать QoS, можно отключить QoS, удалив соответствующие объекты групповой политики.

- [Настройка диапазонов портов и политика качества обслуживания для ваших edge Servers](../../manage/network-management/qos/configuring-port-ranges-for-your-edge-servers.md). Вы можете использовать на пограничных серверах и остальных серверах одни и те же диапазоны портов, хотя это необязательно. Настройка политики QoS может быть сделана только для внутренней стороны ваших серверов Edge. Это потому, что QoS предназначен для использования во внутренней сети, а не в Интернете.

- [Настройка диапазонов портов и политика](../../manage/network-management/qos/configuring-port-ranges-for-your-skype-clients.md) качества обслуживания для клиентов в Skype для бизнеса Server. Эти диапазоны портов применяются только к клиентских компьютерам и обычно отличаются от диапазонов портов, настроенных на серверах. Обратите внимание, Skype для бизнеса Server не поддерживает QoS для Windows операционных систем, кроме Windows 10.


> [!NOTE]
> Если вы используете Windows Server 2012 или Windows Server 2012 R2, вам может быть интересен новый набор Windows PowerShell, доступных для управления QoS на этой платформе. Дополнительные сведения см. [в Windows PowerShell cmdlets for Networking](/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/jj717268(v=ws.11)).

QoS также обсуждается в whitepaper [Network Planning, Monitoring и Troubleshooting with Lync Server](https://www.microsoft.com/download/details.aspx?id=39084) с дополнительными сведениями и глубиной. Хотя содержимое явно ссылается на Lync 2010 и Lync 2013, соображения по Skype для бизнеса Server остаются неизменными.
