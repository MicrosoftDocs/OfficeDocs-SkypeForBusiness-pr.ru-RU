# <a name="dns-requirements-for-skype-for-business-server"></a>Требования DNS для Skype для бизнеса Server

**Сводка:** Просмотрите соображения DNS в этом разделе перед реализацией Skype для бизнеса Server.

В этой статье решается только планирование DNS для Skype для бизнеса Server развертывания в локальной сети организации. Для Skype для бизнеса Online со ссылкой на "Office 365 URL-адреса и диапазоны IP-адресов" по адресу [https://aka.ms/o365ips](/microsoft-365/enterprise/urls-and-ip-address-ranges).

Сервер службы доменных имен (DNS) картирует имена хост-имен (например, www.<span></span> contoso.com<span></span>, предположительно веб-сервер) на IP-адреса (например, 10.10.10.10). Это помогает клиентам и взаимозависимым серверам взаимодействовать друг с другом в сети. При организации реализации Skype для бизнеса Server 2015 г. необходимо убедиться, что сопоставление новых имен серверов (обычно отражающих роль, которую они будут играть) совпадает с IP-адресами, на которые они назначены.

Хотя это может показаться немного сложной на первый взгляд, тяжелая работа по планированию этого можно сделать с помощью Skype для бизнеса Server [2015 года.](https://www.microsoft.com/download/details.aspx?id=50357) После того как вы проявите вопросы мастера о том, какие функции вы планируете использовать, для каждого определенного сайта можно просмотреть отчет DNS в отчете об администраторе edge и использовать сведения, перечисленные в нем, для создания записей DNS. Вы также можете внести изменения во многие из используемых имен и IP-адресов, подробнее см. в [обзоре отчета DNS](../../management-tools/planning-tool/review-the-administrator-reports.md#DNS_Report). Имейте в виду, что вы можете экспортировать отчет об администраторе edge в Excel таблицу, и отчет DNS будет одним из таблиц в файле. Несмотря на то, что этот инструмент включает функции, которые были Skype для бизнеса Server [2019](../../../SfBServer2019/deprecated.md) г., его можно использовать для создания первоначального плана, если эти функции не выбраны.

При установке новой реализации, как описано в записях создания [DNS для Skype для бизнеса Server](../../deploy/install/create-dns-records.md) и создании топологии для Skype для бизнеса Server, мы признаем, что вы можете использовать возможности DNS, встроенные в Windows Server 2016  или сторонний пакет DNS, поэтому мы будем держать обсуждения в этой статье общими, а не конкретными. Мы подробно проявим, что необходимо, и как вы можете удовлетворить эту потребность, это ваше решение.

## <a name="simple-urls"></a>Простые URL-адреса
<a name="BK_Simple"> </a>

Единый локатор ресурсов (URL-адрес) — это ссылка на веб-ресурс, который указывает его расположение в компьютерной сети и протокол, используемый для его получения.

Skype для бизнеса Server поддерживает использование трех "простых" URL-адресов для доступа к службам:

- **Meet** используется в качестве базового URL-адреса для всех конференций на сайте. Пример простого URL-адреса Meet — https:<span></span>//<span></span>meet.<span></span> contoso.com<span></span>. URL-адрес для определенного собрания может быть https:<span></span>//<span></span>meet.<span></span> contoso.com <span></span>/_username_/7322994.

    С помощью простого URL-адреса Meet ссылки на собрания для соединения легко понять и легко общаться.

- **Диалоговое окно** позволяет получить доступ к веб-странице dial-in Conferencing Параметры. На этой странице отображаются номера конференц-связи с доступными языками, заданной информацией о конференции (то есть для собраний, которые не требуются намечены), а также элементы управления DTMF на конференции, а также поддерживается управление личным идентификационным номером (ПИН-кодом) и назначенной информацией о конференц-связи. Простой URL-адрес dial-in включен во все приглашения на собрания, чтобы пользователи, желающие набрать номер собрания, могли получить доступ к необходимому номеру телефона и пин-коду. Пример простого URL-адреса dial-in — https://<span></span> диалин.<span></span> contoso.com<span></span>.

- **Администратор** обеспечивает быстрый доступ к панели Skype для бизнеса Server управления. На любом компьютере в брандмауэрах организации администратор может открыть панель управления Skype для бизнеса Server, введя простой URL-адрес администратора в браузер. Этот URL-адрес используется внутри организации. Пример простого URL-адреса администратора — https://<span></span> admin.<span></span> contoso.com<span></span>.

Простые URL-адреса подробно обсуждаются в [DNS-требованиях к простым URL-адресам в Skype для бизнеса Server](simple-urls.md).
