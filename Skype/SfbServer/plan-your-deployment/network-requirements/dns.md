# <a name="dns-requirements-for-skype-for-business-server"></a>Требования DNS для Скайп для Business Server

**Сводка:** Обзор рекомендаций DNS в этом разделе прежде чем внедрять Скайп для Business Server.

В этой статье только решает DNS планирование Скайп для развертываний Business Server потребности организации в локальной сети. Скайп для бизнеса в Интернет можно найти «Office 365 URL-адреса и IP-адреса диапазонов» в [https://aka.ms/o365ips](https://aka.ms/o365ips).

Сопоставляет имена узлов DNS службы DNS-сервера (например www.<span> </span> Contoso<span></span>.com, предположительно на веб-сервере) в IP-адреса (например, 10.10.10.10). Это облегчает взаимодействие клиентов и взаимозависимых серверов между собой в сети. При настройке реализация Скайп для Business Server 2015 вам потребуется убедитесь в том, что сопоставление новые имена серверов (как правило, что отражает роли, которые они будут передаваться на) соответствует IP-адресов, назначенных им.

Хотя это может показаться несколькими бит в первую очередь, сложных задач для планирования, это можно сделать с помощью [Скайп для средство планирования 2015 Business Server](https://www.microsoft.com/en-us/download/details.aspx?id=50357). После ответов на все вопросы мастера о том, какие функции вы планируете использовать, администратор пограничного сервера отобразит отчет по DNS для всех определенных вами сайтов, который вы можете использовать для создания записей DNS. Вы можете также внести корректировки в используемые имена и IP-адреса. Подробнее см. [Просмотр отчета по DNS](../../management-tools/planning-tool/review-the-administrator-reports.md#DNS_Report). Отчет администратора пограничного сервера можно экспортировать в электронную таблицу Excel, и отчет по DNS будет находиться на одном из листов файла. Хотя это средство включает компоненты [, удаленные из Скайп для Business Server 2019](../../../SfBServer2019/deprecated.md), его можно использовать по-прежнему создать начальный план, если эти компоненты не выбраны

При установке новой реализации как описано в [статье Создание записей DNS для Скайп для Business Server](../../deploy/install/create-dns-records.md) и построение топологии для Скайп для Business Server, мы понимаем, можно использовать возможности DNS, встроенный в Windows Server 2016 или пакет DNS сторонних производителей, поэтому мы будем держать обсуждения в этой статье, общие, а не определенным. Мы только опишем требования, а вы сами решите, как их реализовать.

Опытный Скайп для бизнеса, Lync и набор приложений Office Communications администраторов, вероятно, полезные следующих таблицах. Если понимание информации в таблице вызывает у вас затруднения, то ознакомьтесь со сведениями следующих разделов.

**Разделение мозг DNS** 
 <a name="BK_split"> </a>

Разделенная DNS — конфигурация DNS, в которой две зоны DNS имеют одно пространство имен. При этом первая зона DNS обслуживает только внутренние запросы, а вторая — только внешние запросы, как представлено в таблицах выше. Дополнительные сведения: [Разделенная DNS](../../plan-your-deployment/edge-server-deployments/advanced-edge-server-dns.md#SplitBrainDNS).

## <a name="simple-urls"></a>Простые URL-адреса 
<a name="BK_Simple"> </a>

URL-адрес — это ссылка на интернет-ресурс, содержащая его расположение в компьютерной сети и протокол, по которому необходимо к нему обращаться. 

Скайп для Business Server поддерживает использование три «простой» URL-адреса для доступа к службам.

- URL-адрес **собраний** (meet) используется в качестве основного URL-адреса для всех конференций на сайте. Пример собраний простой URL-адрес является https:<span></span>//<span></span>провести. <span> </span>contoso<span></span>. com. URL-адрес для определенного собрания может быть https:<span></span>//<span></span>провести. <span> </span>contoso<span></span>.com /_имя пользователя_/7322994.

    Простой URL-адрес собраний делает ссылки на присоединение к собраниям более понятными и удобными для обмена.

- 
                URL-адрес **телефонных подключений** (dial-in) обеспечивает доступ к веб-странице "Параметры конференц-связи с телефонным подключением". Эта страница отображает номера для телефонного подключения к конференции с доступными языками, сведениями о назначенных конференциях (т. е. о собраниях, не требующих планирования) и элементами управления DTMF в ходе конференции; поддерживается также управление личными идентификационными номерами (PIN-кодами) и сведениями о назначенных конференциях. Простой URL-адрес телефонного подключения включается во все приглашения на собрания. Таким образом пользователи, которые хотят подключиться к собранию по телефону, могут найти в нем необходимый номер телефона и ПИН-код. Пример-связи простых URL-адрес: https://<span></span>dialin. <span> </span>contoso<span></span>. com.

- **Admin** позволяет быстро получить доступ к Скайп для панели управления Business Server. С любого компьютера в рамках вашей организации брандмауэры администратор можно открыть Скайп для панели управления сервера Business простых URL-адрес администратора, введя в браузере. Этот URL-адрес используется внутри организации. Пример простой URL-адрес администрирования — https://<span></span>администратор. <span> </span>contoso<span></span>. com.

Простые URL-адреса рассматриваются более подробно на [требования к DNS для простых URL-адресов в Скайп для Business Server](simple-urls.md).
