# <a name="dns-requirements-for-skype-for-business-server"></a>Требования к DNS для Skype для бизнеса Server

**Сводка:** Прежде чем приступить к внедрению Skype для бизнеса Server, ознакомьтесь с разделом, приведенным в этой статье

Эта статья содержит сведения о планировании DNS для развертывания сервера Skype для бизнеса в локальной сети Организации. В Skype для бизнеса Online можно найти по адресу [https://aka.ms/o365ips](https://aka.ms/o365ips)"URL-адреса и диапазоны IP-адресов Office 365".

Сервер службы доменных имен (DNS) сопоставляет имена узлов (например www)<span> </span> . Contoso<span></span>. com, предположительно веб-сервер) в IP-адреса (например, 10.10.10.10). Это облегчает взаимодействие клиентов и взаимозависимых серверов между собой в сети. При настройке реализации Skype для бизнеса Server 2015 необходимо убедиться в том, что сопоставление новых имен серверов (обычно отражает роль, которую они будут занимать) соответствует IP-адресам, которым они назначены.

Хотя это может показаться немного трудным для планирования, это можно сделать с помощью [средства планирования Skype для бизнеса Server 2015](https://www.microsoft.com/en-us/download/details.aspx?id=50357). После того, как вы просматриваете вопросы мастера о функциях, которые вы планируете использовать, для каждого сайта вы можете просмотреть отчет DNS в отчете администратора EDGE и использовать указанные там сведения для создания DNS-записей. Вы также можете вносить изменения во многие из используемых имен и IP-адресов, чтобы узнать подробности [в разделе Просмотр отчета DNS](../../management-tools/planning-tool/review-the-administrator-reports.md#DNS_Report). Имейте в виду, что вы можете экспортировать отчет "Администратор Edge" в электронную таблицу Excel, а отчет DNS будет одним из листов файла. Несмотря на то, что эта программа включает возможности, [устаревшие в Skype для бизнеса Server 2019](../../../SfBServer2019/deprecated.md), ее можно использовать для создания начального плана, если не выбраны эти функции.

При установке новой реализации, описанной в разделе [Создание DNS-записей для сервера Skype для бизнеса Server](../../deploy/install/create-dns-records.md) и создании топологии для Skype для бизнеса Server, мы понимаем, что вы можете выбрать использование возможностей DNS, встроенных в Windows Server. 2016 или сторонние сторонние пакеты DNS, поэтому мы будем хранить обсуждения, описанные в этой статье, а не конкретные. Мы подробно рассмотрим, что вам нужно, и как вы отвечаете за то, что ваше решение должно быть принято.

Опытные администраторы Skype для бизнеса, Lync и Office Communications Suite, скорее всего, будут полезны для поиска следующих таблиц. Если понимание информации в таблице вызывает у вас затруднения, то ознакомьтесь со сведениями следующих разделов.

**Разделение мозгового DNS-сервера** 
 <a name="BK_split"> </a>

Разделенная DNS — конфигурация DNS, в которой две зоны DNS имеют одно пространство имен. При этом первая зона DNS обслуживает только внутренние запросы, а вторая — только внешние запросы, как представлено в таблицах выше. Дополнительные сведения: [Разделенная DNS](../../plan-your-deployment/edge-server-deployments/advanced-edge-server-dns.md#SplitBrainDNS).

## <a name="simple-urls"></a>Простые URL-адреса 
<a name="BK_Simple"> </a>

URL-адрес — это ссылка на интернет-ресурс, содержащая его расположение в компьютерной сети и протокол, по которому необходимо к нему обращаться. 

Skype для бизнеса Server поддерживает три простого URL-адреса для доступа к службам.

- URL-адрес **собраний** (meet) используется в качестве основного URL-адреса для всех конференций на сайте. В качестве примера простого URL-адреса можно встретиться HTTPS<span></span>//<span></span>: встреча. <span> </span>Contoso<span></span>. com. URL-адрес для определенного собрания может быть HTTPS:<span></span>//<span></span>встреча. <span> </span>Contoso<span></span>. com/_имя_пользователя_/7322994.

    Простой URL-адрес собраний делает ссылки на присоединение к собраниям более понятными и удобными для обмена.

- **При подключении** к веб-странице параметров конференц-связи с телефонным подключением доступ разрешен. На этой странице отображаются номера конференц-связи со своими доступными языками, назначенные сведения о Конференции (то есть для собраний, которые не нуждаются в расписании), а также встроенные элементы управления DTMF и поддерживают управление личным идентификационным номером ( ПИН-код) и назначенные сведения о конференциях. Простой URL-адрес с телефонным подключением включается во все приглашения на собрания, чтобы пользователи, которые хотят звонить на него, могли получить доступ к требуемому номеру телефона и КОНТАКТным данным. Примером простого URL-адреса с телефонным подключением является<span></span>HTTPS://Dial. <span> </span>Contoso<span></span>. com.

- **Администратор** предоставляет быстрый доступ к панели управления "Skype для бизнеса Server". С любого компьютера в брандмауэре организации администратор может открыть панель управления Skype для бизнеса Server, введя простой URL-адрес администратора в браузере. Этот URL-адрес используется внутри организации. Примером простого URL-адреса администратора является https://<span></span>администратор. <span> </span>Contoso<span></span>. com.

Простые URL-адреса подробно обсуждаются в [DNS-требованиях для простых URL-адресов в Skype для бизнеса Server](simple-urls.md).
