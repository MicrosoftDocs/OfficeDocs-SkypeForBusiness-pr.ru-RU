# <a name="dns-requirements-for-skype-for-business-server"></a>Требования к DNS для Skype для бизнеса Server

**Сводка:** Прежде чем внедрять Skype для бизнеса Server, ознакомьтесь со статьями, приведенными в этом разделе.

В этой статье рассматриваются планирование DNS только для развертываний Skype для бизнеса Server в локальной сети Организации. Для Skype для бизнеса Online обратитесь к разделу "URL-адреса и диапазоны IP [https://aka.ms/o365ips](https://aka.ms/o365ips)-адресов Office 365" по адресу.

Сервер службы доменных имен (DNS) сопоставляет имена узлов (например, www<span> </span> ). Contoso<span></span>. com, предположительно веб-сервер) в IP-адреса (например, 10.10.10.10). Это помогает клиентам и взаимозависимым серверам связываться друг с другом в сети. При настройке реализации Skype для бизнеса Server 2015 необходимо убедиться, что сопоставление новых имен серверов (обычно отражает ту роль, в которой они будут выполняться) соответствует IP-адресам, которым они назначены.

Хотя это может показаться немного довольно сложным, тяжелую работу по планированию можно выполнить с помощью [средства планирования Skype для бизнеса Server 2015](https://www.microsoft.com/download/details.aspx?id=50357). После того, как вы проверили вопросы мастера о функциях, которые планируется использовать, для каждого сайта, который вы планируете использовать, можно просмотреть отчет DNS в отчете пограничного администратора и использовать приведенные в нем сведения для создания DNS-записей. Кроме того, вы можете внести изменения во многие имена и IP-адреса, чтобы получить сведения в статье [Просмотр отчета DNS](../../management-tools/planning-tool/review-the-administrator-reports.md#DNS_Report). Помните, что вы можете экспортировать отчет "администратор пограничного сервера" в электронную таблицу Excel, а отчет DNS будет одним из листов в файле. Несмотря на то что эта программа включает функции, [устаревшие в Skype для бизнеса Server 2019](../../../SfBServer2019/deprecated.md), она по-прежнему может использоваться для создания начального плана, если эти компоненты не выбраны.

При установке новой реализации, описанной в статье [CREATE DNS Records for Skype for Business Server](../../deploy/install/create-dns-records.md) и построении топологии для Skype для бизнеса Server, мы понимаем, что вы можете использовать возможности DNS, встроенные в Windows Server 2016 или СТОРОННИЙ пакет DNS, поэтому мы будем хранить обсуждения, описанные в этой статье, как общие, а не специальные. Мы подробно опишем, что нужно, и как вы отвечаете, что ваше решение должно быть принято.

Опытные администраторы в Skype для бизнеса, Lync и Office Communications Suite, скорее всего, будут использовать следующие таблицы. Если таблица непонятна, в последующих разделах или статьях будут проливается некоторые световые элементы для следующих концепций:

## <a name="simple-urls"></a>Простые URL-адреса
<a name="BK_Simple"> </a>

URL-адрес — это ссылка на веб-ресурс, указывающий его расположение в компьютерной сети и протокол, используемый для его извлечения.

Skype для бизнеса Server поддерживает использование трех "простых" URL-адресов для доступа к службам:

- " **Встреча** " используется в качестве базового URL-адреса для всех конференций на сайте. Пример простого URL-адреса для соответствия: "HTTPS<span></span>//<span></span>: встреча". <span> </span>Contoso<span></span>. com. URL-адрес для определенного собрания может быть HTTPS:<span></span>//<span></span>встреча. <span> </span>Contoso<span></span>. com/_username_/7322994.

    Благодаря простому URL-адресу ссылки на присоединение к собраниям легко воспринимать и легко общаться.

- С помощью **телефонного подключения** вы можете получить доступ к веб-странице параметров конференц-связи с телефонным подключением. На этой странице отображаются номера конференц-связи с доступными языками, назначены сведения о конференциях (то есть для собраний, которые не нуждаются в планировании), а также входящие в систему управления DTMF и поддерживают управление персональным идентификационным номером ( ПИН-код) и назначенные сведения о конференциях. Простой URL-адрес для телефонного подключения включается во все приглашения на собрания, чтобы пользователи, которые хотят звонить на собрание, могли получить доступ к требуемому номеру телефона и ПИН-код. Примером простого URL-адреса для телефонного подключения является https://<span></span>Dialing. <span> </span>Contoso<span></span>. com.

- **Администратор** обеспечивает быстрый доступ к панели управления Skype для бизнеса Server. С любого компьютера, входящего в брандмауэры Организации, администратор может открыть панель управления Skype для бизнеса Server, введя простой URL-адрес администратора в браузере. Этот URL-адрес используется внутри организации. Примером простого URL-адреса администратора может быть https://<span></span>администратор. <span> </span>Contoso<span></span>. com.

Простые URL-адреса подробно обсуждаются в [DNS-требованиях для простых URL-адресов в Skype для бизнеса Server](simple-urls.md).
