# <a name="dns-basics"></a>Основы DNS
 
В Windows Server 2016 имеется встроенное программное обеспечение, которое может предоставлять службы DNS, поэтому вам может потребоваться ознакомиться с этой документацией, например с помощью [руководства по политике DNS](https://docs.microsoft.com/windows-server/networking/dns/deploy/dns-policy-scenario-guide). При желании вы можете выбрать стороннее решение.
  
Рекомендуем использовать для создания DNS определенный сервер в вашей реализации. Вы можете настроить его на одном из серверов, выделенных для одной из ролей сервера Skype для бизнеса, но если этот сервер был также недоступен в рамках пула и был списан случайно, пока не будут установлены службы DNS.
  
## <a name="dns-records"></a>DNS-записи

Каждое сопоставление имени с IP-адресом (и адресом IPv4 или IPv6) хранится в записи DNS на DNS-сервере. Имя описано в DNS-отчете в виде полного доменного имени (FQDN). Несмотря на то, что *contoso.com* является допустимым именем домена, это краткая форма для * \*contoso.com* , поэтому она неоднозначна и может ссылаться на любой сервер в домене. В качестве примера полного доменного имени, которое может ссылаться на один сервер в вашем домене, можно использовать **meeting01.contoso.com**.
  
> [!IMPORTANT]
> По умолчанию имя компьютера, не подключенного к домену, — это имя узла, а не полное доменное имя (FQDN). В построителе топологии используются полные доменные имена, а не названия узлов. Поэтому для компьютера, развертываемого в качестве пограничного сервера, не присоединенного к домену, потребуется указать DNS-суффикс. **Использовать только стандартные символы** (включая A-Z, a-z, 0-9 и дефисы) при назначении полных доменных имен серверам, использующим Skype для бизнеса Server. Не используйте символы Unicode или подчеркивания. Нестандартные символы в полном доменном имени часто не поддерживаются внешними DNS-серверами и общими центрами сертификации (например, когда полное доменное имя необходимо назначить имени субъекта в сертификате).
  
В дополнение к IP-адресу полное доменное имя может сопоставляться с виртуальным IP-адресом ( **VIP** ). VIP — это IP-адрес, который не соответствует фактическому физическому сетевому интерфейсу. Виртуальный IP-адрес часто указывает на пул серверов, выполняющих роль сервера, или на пару серверов, настроенных на избыточность и отказоустойчивость.
  
Существует несколько типов DNS-записей, которые наиболее важны для этого обсуждения: 
  
- **A** — запись адреса или запись узла, возвращающая 32-разрядный адрес IPv4. Чаще всего используется для сопоставления имен узлов с IP-адресом узла.
    
- **AAAA** — запись IPv6-адреса. Возвращает 128-разрядный адрес IPv6. Чаще всего используется для сопоставления имен узлов с IP-адресом узла.
    
- **CNAME** — запись канонического имени. Это решение обрабатывает одно имя на другое: поиск DNS будет повторять подстановку с новым именем.
    
- **SRV** — запись службы (запись SRV) определяет службу (процесс на сервере), доступ к которому осуществляется по определенному порту и в комбинации IP-адреса. Имена служб, которым требуется запись службы, являются фиксированными, и их нельзя настроить в рамках домена SIP. Некоторые службы пользователей используют простые URL-адреса. SRV-запись должна указывать на расположение в том же домене SIP, поэтому, если у вас несколько доменов, для каждой службы потребуется несколько записей SRV.
    
## <a name="how-to-choose-a-sip-domain-name"></a>Как выбрать доменное имя SIP
<a name="BK_NameSIP"> </a>

Доменное имя SIP организации обычно выравнивается по адресам электронной почты, предоставленным им. Если пользователь вашей организации имеет адрес электронной почты, например Brown@contoso.com, предпочтительный \<SIP-домен\> для организации с именем домена contoso.com — это просто contoso.com.
  
### <a name="multiple-sip-domains"></a>Несколько доменов SIP

 В некоторых случаях для организации может потребоваться несколько доменов SIP. Например, если Fabrikam.com получил contoso.com, возможно, потребуется создать новый домен SIP, который приложение Skype для бизнеса Server распознает и будет поддерживать подключение. Для этого вам потребуется создать дополнительный набор всех DNS-записей, использующих contoso.com, с новыми полными доменными именами, показывающими, куда отправлять запросы для Fabrikam.
  
## <a name="dns-load-balancing"></a>DNS Load Balancing
<a name="BK_NameSIP"> </a>

Вы можете использовать DNS для обмена трафиками между несколькими серверами, которые настроены в качестве пула серверов. Для этого нужно создать несколько записей для полного доменного имени пула, каждый из которых указывает на IP-адрес узла в пуле.
  
Дополнительные обсуждения по распределению нагрузки содержатся в разделе [Балансировка нагрузки DNS](../../plan-your-deployment/edge-server-deployments/advanced-edge-server-dns.md#DNSLB) .
