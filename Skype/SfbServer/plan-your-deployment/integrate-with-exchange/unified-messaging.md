# <a name="plan-for-exchange-unified-messaging-integration-in-skype-for-business"></a>Планирование интеграции Exchange единой системы обмена сообщениями в Skype для бизнеса

**Сводка:** Просмотрите эту тему при планировании интеграции Skype для бизнеса Server с Exchange 2013 или 2016 г.

Skype для бизнеса Server поддерживает интеграцию с Exchange единой системы обмена сообщениями (единой системы обмена сообщениями) для объединения голосовых сообщений и сообщений электронной почты в единую инфраструктуру обмена сообщениями. В Exchange, Exchange единой системы обмена сообщениями (единой системы обмена сообщениями) является одной из нескольких ролей Exchange серверов, которые можно установить и настроить.

В Microsoft Exchange Server 2013 и 2016 Exchange 2016 г. служба Exchange сервера почтовых ящиков. Для Skype для бизнеса Server Корпоративная голосовая связь развертывания единая система обмена сообщениями объединяет голосовую и электронную почту в один магазин, к который пользователи могут получить доступ с телефона (Outlook голосового доступа) или компьютера. Единая система обмена сообщениями Skype для бизнеса Server совместной работы по предоставлению пользователям Outlook голосового доступа и автосервисов для Корпоративная голосовая связь.

> [!NOTE]
> Exchange В 2019 Skype для бизнеса Server 2019 г. при интеграции Skype для бизнеса 2019 г. с Exchange 2013 г. или Exchange 2016 г. Из-за изменений в поддержке Exchange 2019 г. Exchange интеграция um отостановки в пользу облачная голосовая почта и облачных автосекретарь функций.  Дополнительные [сведения см. в](../../../sfbhybrid/hybrid/plan-cloud-voicemail.md) облачная голосовая почта службе Plan Skype для бизнеса Server [и Exchange Server миграции.](../../../sfbhybrid/hybrid/plan-um-migration.md)


Чтобы эти функции поддерживалися в локальном развертывании Exchange, необходимо использовать один из следующих компонентов:

- Microsoft Exchange Server 2010 или последний пакет услуг (только Skype для бизнеса Server 2015 г.)
- Microsoft Exchange Server 2013
- Microsoft Exchange Server 2016 г.

> [!NOTE]
> Exchange В Skype для бизнеса Server 2019 г. единая система обмена сообщениями больше недоступна, телефонная система используется для записи сообщений голосовой почты, а затем оставляет запись в Exchange почтовом ящике пользователя. Дополнительные [сведения см. в облачная голосовая почта службе](../../../sfbhybrid/hybrid/plan-cloud-voicemail.md) Plan облачная голосовая почта.

## <a name="features-of-integrated-unified-messaging-and-skype-for-business-server"></a>Функции интегрированной единой системы обмена сообщениями и Skype для бизнеса Server

Skype для бизнеса Server Корпоративная голосовая связь использует инфраструктуру Exchange единой системы обмена сообщениями для предоставления ответов на вызовы, уведомления о вызове, голосового доступа (в том числе голосовой почты) и автопослужб.

- **Ответы на вызовы** Ответ на вызов — это получение голосовых сообщений от имени пользователей, на звонки которых не отвечают или заняты. При этом воспроизводится персональное приветствие и записывается сообщение, которое затем помещается в очередь на доставку в почтовый ящик пользователя, хранящийся на сервере почтовых ящиков Exchange.

    Если звонящий оставляет сообщение, оно помещается в ящик входящих сообщений пользователя. Если звонящий решает не оставлять сообщение, в почтовом ящике сохраняется уведомление о пропущенном звонке. Пользователь затем может получить доступ к своему ящику входящих сообщений с помощью клиента обмена сообщениями и совместной работы Microsoft Outlook, Outlook Web Access, технологии Exchange ActiveSync или голосового доступа к Outlook. Темы и приоритет звонков могут отображаться так же, как для электронной почты.

- **Outlook** голосовой доступ Outlook голосовой доступ позволяет пользователю Корпоративная голосовая связь получать доступ не только к голосовой почте, но и к почтовому ящику Exchange, включая электронную почту, календарь и контакты из интерфейса телефонии. Номер доступа абонента назначен администратором Exchange um.

- **Автобюсер** Автоспутник — это Exchange, которая может быть использована для настройки номера телефона, который внешние пользователи могут набрать, чтобы связаться с представителями компании. В частности, она предоставляет ряд голосовых подсказок, облегчающих звонящему навигацию по системе меню. Список доступных параметров настраивается администратором Exchange на сервере Exchange.

- **Службы факсимиле** Exchange службы электронной почты включают функции факса, которые позволяют пользователям получать входящие факсы в Exchange почтовых ящиках. Подробные сведения см. [в Microsoft Exchange Server](/previous-versions/office/exchange-server-2007/bb123911(v=exchg.80)) единой документации.

    > [!NOTE]
    > Факсимиловые службы, предоставляемые сервером единой системы Exchange, недоступны в Skype для бизнеса Server развертываниях, интегрированных с Microsoft Exchange Server 2010 г. Exchange 2010 г. с последним пакетом Exchange 2013 г. или Exchange 2016 г.

## <a name="components-and-topologies-for-on-premises-unified-messaging-in-skype-for-business-server"></a>Компоненты и топологии для локальной единой системы обмена сообщениями в Skype для бизнеса Server

### <a name="exchange-server-components"></a>Компоненты Exchange Server

Чтобы предоставить Exchange функции и службы единой системы обмена сообщениями, описанные в статьях Features [of integrated Unified Messaging and Skype для бизнеса Server](#features-of-integrated-unified-messaging-and-skype-for-business-server) Корпоративная голосовая связь пользователям в вашей организации, необходимо развернуть сервер почтовых ящиков Microsoft Exchange и клиентский доступ, на котором размещаются почтовые ящики пользователей и предоставляется единое хранилище для электронной почты и голосовой почты. Exchange Um выполняется в качестве службы на Exchange почтовых ящиках и серверах клиентского доступа.

Дополнительные сведения о Exchange компонентах системы электронной почты в Microsoft Exchange Server 2010 г. см. в материале [Deploying On-Premises Exchange UM to Provide Lync Server 2013 Preview Voice Mail](/previous-versions/office/lync-server-2013/lync-server-2013-deploying-on-premises-exchange-um-to-provide-lync-server-2013-voice-mail) .

### <a name="supported-topologies"></a>Поддерживаемые топологии

Вы можете развернуть Skype для бизнеса Server и Exchange единой системы обмена сообщениями (ЕДИНОЙ системы обмена сообщениями) в одном лесу или нескольких лесах. Если развертывание охватывает несколько лесов, необходимо выполнить Exchange интеграции для каждого Exchange леса. Кроме того, необходимо настроить каждый лес microsoft Exchange, чтобы доверять лесу Skype для бизнеса Server и лесу Skype для бизнеса Server, чтобы доверять Exchange лесам. Помимо этого доверия к лесу, параметры Exchange для всех пользователей должны быть установлены на объектах пользователей в Skype для бизнеса Server лесу.

Skype для бизнеса Server поддерживает следующие топологии для Exchange интеграции.

- один лес;

- Единый домен (то есть один лес с одним доменом). Skype для бизнеса Server, microsoft Exchange и все пользователи находятся в одном домене.

- Несколько доменов (то есть корневой домен с одним или несколькими детскими доменами). Skype для бизнеса Server и серверы Microsoft Exchange развертываются в различных доменах из домена, в котором создаются пользователи. Exchange Серверы um можно развернуть в различных доменах из поддерживаемого Skype для бизнеса Server пула.

- Несколько лесов (то есть лес ресурсов). Skype для бизнеса Server развертывается в одном лесу, а затем пользователи распределяются по нескольким лесам. Атрибуты Exchange um должны быть реплицированы в лес Skype для бизнеса Server.

    > [!NOTE]
    > Exchange можно развернуть в нескольких лесах. Каждая Exchange может предоставлять Exchange пользователям, или Exchange может быть развернута в том же лесу, что и Skype для бизнеса Server.

## <a name="guidelines-for-integrating-on-premises-unified-messaging-and-skype-for-business-server"></a>Рекомендации по интеграции локальной единой системы обмена сообщениями и Skype для бизнеса Server

Ниже приведены рекомендации по развертыванию Enterprise Voice:

> [!IMPORTANT]
> Exchange Единая система обмена сообщениями поддерживает IPv6 только в том случае, если вы также используете UCMA 4.

- Развертывание Skype для бизнеса Server выпуск Standard сервера или пула переднего конца.

- Работайте вместе с администраторами Exchange, чтобы подтвердить, какие задачи каждый из вас будет выполнять для обеспечения успешной интеграции.

- Развертывание ролей Exchange почтовых ящиков в каждом лесу Exchange единой системы обмена сообщениями, где необходимо включить пользователей для Exchange единой системы обмена сообщениями. Дополнительные сведения об установке ролей Exchange сервера см. в Microsoft Exchange Server документации.

    > [!IMPORTANT]
    > При Exchange единой системы обмена сообщениями он настраивается на использование самозаверяемого сертификата. Самозаверяется сертификат, который не позволяет Skype для бизнеса Server и Exchange um доверять друг другу, поэтому необходимо запросить отдельный сертификат от органа сертификации, которому доверяют оба сервера.

- Если Skype для бизнеса Server и Exchange в разных лесах, настройте каждый лес Exchange, чтобы доверять лесу Skype для бизнеса Server и лесу Skype для бизнеса Server доверять каждому лесу Exchange. Кроме того, установите параметры Exchange для объектов пользовательского интерфейса в лесу Skype для бизнеса Server, как правило, с помощью скрипта или межлесного средства, например Диспетчер жизненного цикла удостоверений (ILM).

- При необходимости установите консоль управления Exchange для управления серверами единой системы обмена сообщениями.

- Получите допустимые телефонные номера для голосового доступа к Outlook и автосекретарю.

- Если вы используете версию Exchange um раньше Microsoft Exchange Server 2010 Пакет обновления 1 (SP1), координаты имен для Exchange SIP URI um и Корпоративная голосовая связь номера.

### <a name="deploying-redundant-exchange-um-servers"></a>Развертывание избыточных серверов единой системы обмена сообщениями Exchange

> [!IMPORTANT]
> Рекомендуется развернуть не менее двух серверов, на которых Exchange службы um для каждого номера Exchange SIP URI, настроенного для вашей организации. Помимо дополнительной емкости развертывание избыточных серверов обеспечивает высокую доступность. В случае сбоя сервера можно настроить Skype для бизнеса Server на другой сервер.

В следующем примере конфигурации единой системы обмена сообщениями Exchange обеспечивают устойчивость.

**Пример 1. Устойчивость единой системы обмена сообщениями Exchange**

![Exchange Схема устойчивости к um](../../media/d8381ecc-0e4e-47ea-9bf7-e54fec9414e7.png)

В примере 1 серверы единой системы обмена сообщениями Exchange 1 и 2 включены в центре обработки данных в Таквила, серверы единой системы обмена сообщениями Exchange 3 и 4 включены в центре данных в Дублине. В случае сбоя в Таквила записи DNS A для серверов 1 и 2 следует настроить так, чтобы они указывали на серверы 3 и 4 соответственно. В случае сбоя сервера единой системы обмена сообщениями Exchange в Дублине записи DNS A для серверов 3 и 4 необходимо настроить так, чтобы они указывал на серверы 1 и 2 соответственно.

> [!NOTE]
> Например 1, вы также должны назначить один из следующих сертификатов на каждом сервере Exchange um: либо использовать сертификат с подмыкальным знаком в альтернативном имени субъекта (SAN) или поместить полностью квалифицированное доменное имя (FQDN) каждого из четырех Exchange серверов um в SAN.

**Пример 2. Устойчивость единой системы обмена сообщениями Exchange**

![Exchange Схема устойчивости к um](../../media/4ad101c3-f318-4fc0-b4da-c05f2e92a943.png)

В примере 2 при обычных условиях работы серверов единой системы обмена сообщениями Exchange 1 и 2 включены в центре обработки данных в Таквила, серверы единой системы обмена сообщениями Exchange 3 и 4 включены в центре обработки данных Дублина. Все четыре сервера включены в абонентскую группу SIP URI Таквила. Но серверы 3 и 4 отключены. В случае сбоя единой системы обмена сообщениями Exchange в Таквила, например, серверы единой системы обмена сообщениями Exchange 1 и 2 должны быть отключены, а серверы единой системы обмена сообщениями Exchange 4 и 3 должны быть включены, чтобы трафик единой системы обмена сообщениями Exchange из Таквила направлялся на серверы в Дублине.
