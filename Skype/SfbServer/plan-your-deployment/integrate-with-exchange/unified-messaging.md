# <a name="plan-for-exchange-unified-messaging-integration-in-skype-for-business"></a>Планирование интеграции единой системы обмена сообщениями Exchange в Skype для бизнеса

**Сводка:** Просмотрите этот раздел при планировании интеграции Skype для бизнеса Server с Exchange 2013 или 2016.

Skype для бизнеса Server поддерживает интеграцию с единой системы обмена сообщениями Exchange для объединения голосовых сообщений и сообщений электронной почты в единую инфраструктуру обмена сообщениями. В Exchange единая система обмена сообщениями Exchange — это одна из нескольких ролей сервера Exchange, которые можно установить и настроить.

В Microsoft Exchange Server 2013 и 2016 г. служба exchange выполняется как служба на сервере почтовых ящиков Exchange. Для развертывания Skype для бизнеса Server Корпоративная голосовая связь единой системы обмена сообщениями единая система обмена сообщениями объединяет голосовые сообщения и сообщения электронной почты в одном хранилище, к котором пользователи могут получить доступ с телефона (голосовой доступ к Outlook) или компьютера. Единая система обмена сообщениями и Skype для бизнеса Server совместно предоставляют службы автоототвода, голосовой доступ к Outlook и автослужб пользователям Корпоративная голосовая связь.

> [!NOTE]
> При интеграции Skype для бизнеса 2019 с Exchange 2013 или Exchange 2016 в Skype для бизнеса Server 2019 остается доступной um exchange. Из-за изменений в поддержке в Exchange 2019 интеграция с exchange UM будет отостановка в пользу функций облачной голосовой почты и облачных автосекретарь.  Дополнительные [сведения см.](../../../sfbhybrid/hybrid/plan-cloud-voicemail.md) в сведениях о планировании службы облачной голосовой почты и планировании для Skype для бизнеса [Server Exchange Server миграции.](../../../sfbhybrid/hybrid/plan-um-migration.md)


Для поддержки этих функций в локальном развертывании exchange um необходимо использовать одно из следующих компонентов:

- Microsoft Exchange Server 2010 или более поздней версии (только в Skype для бизнеса Server 2015)
- Microsoft Exchange Server 2013
- Microsoft Exchange Server 2016 г.

> [!NOTE]
> Как уже было известно, единая система обмена сообщениями Exchange больше недоступна в Skype для бизнеса Server 2019, который использует телефонную систему для записи сообщений голосовой почты, а затем оставляет запись в почтовом ящике Exchange пользователя. Дополнительные [сведения см. в](../../../sfbhybrid/hybrid/plan-cloud-voicemail.md) сведениях о планировании облачной службы голосовой почты.

## <a name="features-of-integrated-unified-messaging-and-skype-for-business-server"></a>Функции интегрированной единой системы обмена сообщениями и Skype для бизнеса Server

Skype для бизнеса Server Корпоративная голосовая связь использует инфраструктуру единой системы обмена сообщениями Exchange для предоставления автоотвещений, уведомлений о вызовах, голосового доступа (включая голосовую почту) и автослужб.

- **Ответ на вызовы** Ответ на вызовы — это получение голосовых сообщений от имени пользователей, чьи вызовы не отвечают или заняты. При этом воспроизводится персональное приветствие и записывается сообщение, которое затем помещается в очередь на доставку в почтовый ящик пользователя, хранящийся на сервере почтовых ящиков Exchange.

    Если звонящий оставляет сообщение, оно помещается в ящик входящих сообщений пользователя. Если звонящий решает не оставлять сообщение, в почтовом ящике сохраняется уведомление о пропущенном звонке. Пользователь затем может получить доступ к своему ящику входящих сообщений с помощью клиента обмена сообщениями и совместной работы Microsoft Outlook, Outlook Web Access, технологии Exchange ActiveSync или голосового доступа к Outlook. Темы и приоритет звонков могут отображаться так же, как для электронной почты.

- **голосовой доступ к Outlook** голосовой доступ к Outlook позволяет пользователю Корпоративная голосовая связь получать доступ не только к голосовой почте, но и к почтовому ящику Exchange, включая электронную почту, календарь и контакты из телефонного интерфейса. Номер доступа подписчика назначен администратором exchange UM.

- **Автоотвессник** Автостат — это функция exchange UM, которую можно использовать для настройки номера телефона, который внешние пользователи могут набирать для связи с представителями компании. В частности, она предоставляет ряд голосовых подсказок, облегчающих звонящему навигацию по системе меню. Список доступных параметров настраивается на сервере exchange um администратором exchange UM.

- **Службы факсов** В exchange UM есть функции факсов, которые позволяют пользователям получать входящие факсы в своих почтовых ящиках Exchange. Подробные сведения см. в [документации](https://go.microsoft.com/fwlink/p/?linkId=135652) по Microsoft Exchange Server единой системы обмена сообщениями.

    > [!NOTE]
    > Службы факсов, предоставляемые сервером единой системы обмена данными Exchange, недоступны в развертываниях Skype для бизнеса Server, интегрированных с Microsoft Exchange Server 2010, Exchange 2010 с последним пакетом обновления, Exchange 2013 или Exchange 2016.

## <a name="components-and-topologies-for-on-premises-unified-messaging-in-skype-for-business-server"></a>Компоненты и topologies для локальной единой системы обмена сообщениями в Skype для бизнеса Server

### <a name="exchange-server-components"></a>Компоненты Exchange Server

Чтобы предоставить функции и службы единой системы обмена сообщениями Exchange, описанные в функции интегрированной единой системы обмена сообщениями и Skype для бизнеса [Server](#features-of-integrated-unified-messaging-and-skype-for-business-server) для пользователей Корпоративная голосовая связь в организации, необходимо развернуть сервер почтовых ящиков Microsoft Exchange и сервер клиентского доступа, на котором размещаются почтовые ящики пользователей и предоставляется единое хранилище для электронной и голосовой почты. Exchange UM работает как служба на серверах почтовых ящиков и клиентского доступа Exchange.

Дополнительные сведения о компонентах exchange UM в Microsoft Exchange Server 2010 см. в сведениях о развертывании локальной системы обмена данными Exchange для предоставления голосовой почты предварительного просмотра [Lync Server 2013.](https://technet.microsoft.com/library/9673bd73-a3a3-425d-870f-04d801c6d0d5.aspx)

### <a name="supported-topologies"></a>Поддерживаемые топологии

Skype для бизнеса Server и единая система обмена сообщениями Exchange можно развернуть в одном или нескольких лесах. Если развертывание охватывает несколько лесов, необходимо выполнить действия по интеграции Exchange для каждого леса exchange UM. Кроме того, необходимо настроить каждый лес Microsoft Exchange для доверия лесу Skype для бизнеса Server и лесу Skype для бизнеса Server для доверия каждому лесу exchange UM. Помимо этого доверия леса, параметры exchange UM для всех пользователей должны быть установлены для объектов пользователей в лесу Skype для бизнеса Server.

Skype для бизнеса Server поддерживает следующие topologies для интеграции с exchange UM:

- один лес;

- Один домен (то есть один лес с одним доменом). Skype для бизнеса Server, Microsoft Exchange и пользователи находятся в одном домене.

- Несколько доменов (то есть корневой домен с одним или несколькими дополнительными доменами). Skype для бизнеса Server и серверы Microsoft Exchange развертываются в доменах, не в которых создаются пользователи. Серверы exchange UM могут быть развернуты в разных доменах от поддерживаемого ими пула Skype для бизнеса Server.

- Несколько лесов (то есть лес ресурсов). Skype для бизнеса Server развертывается в одном лесу, а затем пользователи распределяются между несколькими лесами. Атрибуты пользователей exchange UM должны быть реплицированы в лес Skype для бизнеса Server.

    > [!NOTE]
    > Exchange можно развернуть в нескольких лесах. Каждая организация Exchange может предоставить пользователям exchange um или развернуть ее в том же лесу, что и Skype для бизнеса Server.

## <a name="guidelines-for-integrating-on-premises-unified-messaging-and-skype-for-business-server"></a>Руководство по интеграции локальной единой системы обмена сообщениями и Skype для бизнеса Server

Ниже приведены рекомендации по развертыванию Enterprise Voice:

> [!IMPORTANT]
> Единая система обмена сообщениями Exchange поддерживает IPv6, только если вы также используете UCMA 4.

- Развертывание сервера Skype для бизнеса Server Standard Edition или пула переднего сервера.

- Работайте вместе с администраторами Exchange, чтобы подтвердить, какие задачи каждый из вас будет выполнять для обеспечения успешной интеграции.

- Развертывание ролей сервера почтовых ящиков Exchange в каждом лесу единой системы обмена сообщениями Exchange, в котором необходимо включить для пользователей единую систему обмена сообщениями Exchange. Дополнительные сведения об установке ролей сервера Exchange см. в Microsoft Exchange Server документации.

    > [!IMPORTANT]
    > При установке единой системы обмена сообщениями Exchange она настраивается на использование самозаверяющих сертификатов. Самозаверяячий сертификат не позволяет skype для бизнеса Server и exchange UM доверять друг другу, поэтому необходимо запросить отдельный сертификат от сертификационного органа, которому доверяют оба сервера.

- Если Skype для бизнеса Server и система обмена данными Exchange установлены в разных лесах, настройте каждый лес Exchange, чтобы доверять лесу Skype для бизнеса Server и лесу Skype для бизнеса Server доверие каждому лесу Exchange. Кроме того, установите пользовательские параметры exchange UM для объектов пользователей в лесу Skype для бизнеса Server, как правило, с помощью скрипта или межлесного средства, например диспетчера жизненного цикла удостоверений (ILM).

- При необходимости установите консоль управления Exchange для управления серверами единой системы обмена сообщениями.

- Получите допустимые телефонные номера для голосового доступа к Outlook и автосекретарю.

- Если вы используете версию exchange UM более ранней, чем Microsoft Exchange Server 2010 Пакет обновления 1 (SP1), имена координат для телефонных планов URI SIP и Корпоративная голосовая связь URI.

### <a name="deploying-redundant-exchange-um-servers"></a>Развертывание избыточных серверов единой системы обмена сообщениями Exchange

> [!IMPORTANT]
> Рекомендуется развернуть не менее двух серверов, на которых запущены службы exchange UM, для каждой настраиваемой для организации телефонной программы SIP URI для exchange UM. Помимо дополнительной емкости развертывание избыточных серверов обеспечивает высокую доступность. В случае сбоя сервера Skype для бизнеса Server можно настроить для перенастройки на другой сервер.

В следующем примере конфигурации единой системы обмена сообщениями Exchange обеспечивают устойчивость.

**Пример 1. Устойчивость единой системы обмена сообщениями Exchange**

![Схема устойчивости для exchange UM](../../media/d8381ecc-0e4e-47ea-9bf7-e54fec9414e7.png)

В примере 1 серверы единой системы обмена сообщениями Exchange 1 и 2 включены в центре обработки данных в Таквила, серверы единой системы обмена сообщениями Exchange 3 и 4 включены в центре данных в Дублине. В случае сбоя в Таквила записи DNS A для серверов 1 и 2 следует настроить так, чтобы они указывали на серверы 3 и 4 соответственно. В случае сбоя сервера единой системы обмена сообщениями Exchange в Дублине записи DNS A для серверов 3 и 4 необходимо настроить так, чтобы они указывал на серверы 1 и 2 соответственно.

> [!NOTE]
> Например, в примере 1 необходимо назначить один из следующих сертификатов на каждом сервере exchange UM: использовать сертификат с поддиаптером в альтернативном имени субъекта (SAN) или поместить полное доменное имя (FQDN) каждого из четырех серверов exchange UM в SAN.

**Пример 2. Устойчивость единой системы обмена сообщениями Exchange**

![Схема устойчивости для exchange UM](../../media/4ad101c3-f318-4fc0-b4da-c05f2e92a943.png)

В примере 2 при обычных условиях работы серверов единой системы обмена сообщениями Exchange 1 и 2 включены в центре обработки данных в Таквила, серверы единой системы обмена сообщениями Exchange 3 и 4 включены в центре обработки данных Дублина. Все четыре сервера включены в абонентскую группу SIP URI Таквила. Но серверы 3 и 4 отключены. В случае сбоя единой системы обмена сообщениями Exchange в Таквила, например, серверы единой системы обмена сообщениями Exchange 1 и 2 должны быть отключены, а серверы единой системы обмена сообщениями Exchange 4 и 3 должны быть включены, чтобы трафик единой системы обмена сообщениями Exchange из Таквила направлялся на серверы в Дублине.
