# <a name="plan-for-exchange-unified-messaging-integration-in-skype-for-business"></a>Планирование интеграции единой системы обмена сообщениями Exchange в Skype для бизнеса

**Сводка:** Просмотрите эту тему при планировании интеграции Skype для бизнеса Server с Exchange 2013 или 2016.

Skype для бизнеса Server поддерживает интеграцию с Exchange Unified Messaging (UM) для объединения голосовых сообщений и сообщений электронной почты в единую инфраструктуру обмена сообщениями. В Exchange единая система обмена сообщениями Exchange является одной из нескольких ролей сервера Exchange, которые можно установить и настроить.

В Microsoft Exchange Server 2013 и 2016 годах служба обмена данными Exchange выполняется в качестве службы на сервере почтовых ящиков Exchange. Для развертывания Skype для бизнеса Server Корпоративная голосовая связь единой системы обмена сообщениями объединяет голосовую связь и обмен сообщениями электронной почты в один магазин, к который пользователи могут получить доступ с телефона (голосовой доступ к Outlook) или компьютера. Единая система обмена сообщениями и Skype для бизнеса Server работают вместе, чтобы предоставлять пользователям голосовой доступ к Outlook, голосовой доступ к Outlook и автосервисные службы пользователям Корпоративная голосовая связь.

> [!NOTE]
> При интеграции Skype for Business Server 2019 с Exchange 2013 или Exchange 2016 skype for Business 2019 остается доступной в Skype для бизнеса Server 2019. Из-за изменений в поддержке в Exchange 2019 интеграция exchange um отозвучит в пользу функций облачной голосовой почты и облачных автосекретарь.  Дополнительные сведения см. в приложениях Plan [Cloud Voicemail service](../../../sfbhybrid/hybrid/plan-cloud-voicemail.md) and Plan for Skype for Business Server и Exchange Server [миграции.](../../../sfbhybrid/hybrid/plan-um-migration.md)


Чтобы эти функции поддерживалися в локальном развертывании Exchange UM, необходимо использовать один из следующих компонентов:

- Microsoft Exchange Server 2010 или последний пакет услуг (только Skype для бизнеса Server 2015)
- Microsoft Exchange Server 2013
- Microsoft Exchange Server 2016 г.

> [!NOTE]
> Exchange Unified Messaging, как уже было известно, больше недоступна в Skype для бизнеса Server 2019, который использует телефонную систему для записи сообщений голосовой почты, а затем оставить запись в почтовом ящике exchange пользователя. Дополнительные сведения см. в службе голосовой почты Plan [Cloud.](../../../sfbhybrid/hybrid/plan-cloud-voicemail.md)

## <a name="features-of-integrated-unified-messaging-and-skype-for-business-server"></a>Функции интегрированной единой системы обмена сообщениями и Skype для бизнеса Server

Skype для бизнеса Server использует Корпоративная голосовая связь exchange Unified Messaging (UM) для предоставления ответов на вызовы, уведомления о вызове, голосового доступа (в том числе голосовой почты) и автопослужб.

- **Ответы на вызовы** Ответ на вызов — это получение голосовых сообщений от имени пользователей, на звонки которых не отвечают или заняты. При этом воспроизводится персональное приветствие и записывается сообщение, которое затем помещается в очередь на доставку в почтовый ящик пользователя, хранящийся на сервере почтовых ящиков Exchange.

    Если звонящий оставляет сообщение, оно помещается в ящик входящих сообщений пользователя. Если звонящий решает не оставлять сообщение, в почтовом ящике сохраняется уведомление о пропущенном звонке. Пользователь затем может получить доступ к своему ящику входящих сообщений с помощью клиента обмена сообщениями и совместной работы Microsoft Outlook, Outlook Web Access, технологии Exchange ActiveSync или голосового доступа к Outlook. Темы и приоритет звонков могут отображаться так же, как для электронной почты.

- **голосовой доступ к Outlook** голосовой доступ к Outlook позволяет пользователю Корпоративная голосовая связь получать доступ не только к голосовой почте, но и к почтовому ящику Exchange, включая электронную почту, календарь и контакты из интерфейса телефонии. Номер доступа абонента назначен администратором обмена обмена данными.

- **Автобюсер** Автопоследник — это функция обмена данными Exchange, которая может быть использована для настройки номера телефона, который внешние пользователи могут набрать, чтобы связаться с представителями компании. В частности, она предоставляет ряд голосовых подсказок, облегчающих звонящему навигацию по системе меню. Список доступных параметров настраивается администратором exchange UM на сервере Exchange.

- **Службы факса** Система обмена данными Exchange включает функции факса, которые позволяют пользователям получать входящие факсы в почтовых ящиках Exchange. Подробные сведения см. [в Microsoft Exchange Server](/previous-versions/office/exchange-server-2007/bb123911(v=exchg.80)) единой документации.

    > [!NOTE]
    > Факсовые службы, предоставляемые сервером единой системы обмена данными Exchange, недоступны в развертываниях Skype для бизнеса Server, интегрированных с Microsoft Exchange Server 2010 г., Exchange 2010 с последним пакетом служб Exchange 2013 или Exchange 2016.

## <a name="components-and-topologies-for-on-premises-unified-messaging-in-skype-for-business-server"></a>Компоненты и топологии для локальной единой системы обмена сообщениями в Skype для бизнеса Server

### <a name="exchange-server-components"></a>Компоненты Exchange Server

Чтобы предоставить функции и службы единой системы обмена сообщениями Exchange, описанные в Характеристиках интегрированной единой системы обмена сообщениями и [Skype для](#features-of-integrated-unified-messaging-and-skype-for-business-server) бизнеса Server для Корпоративная голосовая связь пользователей в вашей организации, необходимо развернуть сервер почтовых ящиков Microsoft Exchange и сервер клиентского доступа, на котором размещаются почтовые ящики пользователей и предоставляется единое хранилище для электронной почты и голосовой почты. Служба обмена данными Exchange выполняется в качестве службы на серверах почтовых ящиков Exchange и клиентского доступа.

Дополнительные сведения о компонентах exchange UM Microsoft Exchange Server 2010 г. см. в материале [Deploying On-Premises Exchange UM to Provide Lync Server 2013 Preview Voice Mail.](/previous-versions/office/lync-server-2013/lync-server-2013-deploying-on-premises-exchange-um-to-provide-lync-server-2013-voice-mail)

### <a name="supported-topologies"></a>Поддерживаемые топологии

Вы можете развернуть Skype для бизнеса Server и Exchange Unified Messaging (ЕДИНОЙ Системы обмена сообщениями) в одном лесу или нескольких лесах. Если развертывание охватывает несколько лесов, необходимо выполнить этапы интеграции Exchange для каждого леса обмена данными Exchange. Кроме того, необходимо настроить каждый лес Microsoft Exchange, чтобы доверять лесу Skype для бизнеса Server и лесу Skype для бизнеса Server, чтобы доверять каждому лесу Exchange. В дополнение к этому лесу параметры обмена данными для всех пользователей должны быть установлены на объектах пользователей в лесу Skype для бизнеса Server.

Skype для бизнеса Server поддерживает следующие topologies для интеграции exchange UM:

- один лес;

- Единый домен (то есть один лес с одним доменом). Skype для бизнеса Server, Microsoft Exchange и пользователи находятся в одном домене.

- Несколько доменов (то есть корневой домен с одним или несколькими детскими доменами). Серверы Skype для бизнеса и Microsoft Exchange развернуты в разных доменах от домена, в котором создаются пользователи. Серверы обмена обмена данными могут быть развернуты в различных доменах из поддерживаемого ими пула Skype для бизнес-серверов.

- Несколько лесов (то есть лес ресурсов). Skype для бизнеса Server развертывается в одном лесу, а затем пользователи распределяются по нескольким лесам. Атрибуты Exchange UM пользователей должны быть реплицированы в лес Skype для бизнеса Server.

    > [!NOTE]
    > Exchange можно развернуть в нескольких лесах. Каждая организация Exchange может предоставлять пользователям ОМ Exchange, а exchange UM может быть развернута в том же лесу, что и Skype для бизнеса Server.

## <a name="guidelines-for-integrating-on-premises-unified-messaging-and-skype-for-business-server"></a>Рекомендации по интеграции локальной единой системы обмена сообщениями и Skype для бизнеса Server

Ниже приведены рекомендации по развертыванию Enterprise Voice:

> [!IMPORTANT]
> Единая система обмена сообщениями Exchange поддерживает IPv6 только в том случае, если вы также используете UCMA 4.

- Развертывание сервера стандартного выпуска Skype для бизнеса Server или пула переднего конца.

- Работайте вместе с администраторами Exchange, чтобы подтвердить, какие задачи каждый из вас будет выполнять для обеспечения успешной интеграции.

- Развертывание ролей сервера почтовых ящиков Exchange в каждом лесу единой системы обмена сообщениями Exchange, где необходимо включить пользователей для единой системы обмена сообщениями Exchange. Дополнительные сведения об установке ролей сервера Exchange см. в Microsoft Exchange Server документации.

    > [!IMPORTANT]
    > При установке Единой системы обмена сообщениями Exchange она настраивается на использование самозаверяемого сертификата. Самозаверяется сертификат, который не позволяет Skype для бизнеса Server и Exchange um доверять друг другу, поэтому необходимо запросить отдельный сертификат от органа сертификации, которому доверяют оба сервера.

- Если skype for Business Server и Exchange um установлены в различных лесах, настройте каждый лес Exchange, чтобы доверять лесу Skype для бизнеса Server и лесу Skype для бизнеса Server, чтобы доверять каждому лесу Exchange. Кроме того, установите параметры обмена обмена данными пользователей на объекты пользователей в лесу Skype для бизнес-сервера, как правило, с помощью скрипта или межлесного средства, например Диспетчер жизненного цикла удостоверений (ILM).

- При необходимости установите консоль управления Exchange для управления серверами единой системы обмена сообщениями.

- Получите допустимые телефонные номера для голосового доступа к Outlook и автосекретарю.

- Если вы используете версию exchange UM раньше Microsoft Exchange Server 2010 Пакет обновления 1 (SP1), координаты имен для телефонных планов URI SIP Exchange и Корпоративная голосовая связь набора.

### <a name="deploying-redundant-exchange-um-servers"></a>Развертывание избыточных серверов единой системы обмена сообщениями Exchange

> [!IMPORTANT]
> Рекомендуется развернуть не менее двух серверов, на которых работают службы ОБМЕНА Exchange для каждого набора URI SIP SIP Exchange, настроенного для вашей организации. Помимо дополнительной емкости развертывание избыточных серверов обеспечивает высокую доступность. В случае сбоя сервера Skype для бизнеса Server можно настроить для отказа на другом сервере.

В следующем примере конфигурации единой системы обмена сообщениями Exchange обеспечивают устойчивость.

**Пример 1. Устойчивость единой системы обмена сообщениями Exchange**

![Схема устойчивости к обмена обмена данными](../../media/d8381ecc-0e4e-47ea-9bf7-e54fec9414e7.png)

В примере 1 серверы единой системы обмена сообщениями Exchange 1 и 2 включены в центре обработки данных в Таквила, серверы единой системы обмена сообщениями Exchange 3 и 4 включены в центре данных в Дублине. В случае сбоя в Таквила записи DNS A для серверов 1 и 2 следует настроить так, чтобы они указывали на серверы 3 и 4 соответственно. В случае сбоя сервера единой системы обмена сообщениями Exchange в Дублине записи DNS A для серверов 3 и 4 необходимо настроить так, чтобы они указывал на серверы 1 и 2 соответственно.

> [!NOTE]
> Например 1, вы также должны назначить один из следующих сертификатов на каждом сервере обмена обмена данными: либо использовать сертификат с подыскной карточкой в альтернативном имени субъекта (SAN) или поместить полностью квалифицированное доменное имя (FQDN) каждого из четырех серверов ОБМЕНА Exchange в SAN.

**Пример 2. Устойчивость единой системы обмена сообщениями Exchange**

![Схема устойчивости к обмена обмена данными](../../media/4ad101c3-f318-4fc0-b4da-c05f2e92a943.png)

В примере 2 при обычных условиях работы серверов единой системы обмена сообщениями Exchange 1 и 2 включены в центре обработки данных в Таквила, серверы единой системы обмена сообщениями Exchange 3 и 4 включены в центре обработки данных Дублина. Все четыре сервера включены в абонентскую группу SIP URI Таквила. Но серверы 3 и 4 отключены. В случае сбоя единой системы обмена сообщениями Exchange в Таквила, например, серверы единой системы обмена сообщениями Exchange 1 и 2 должны быть отключены, а серверы единой системы обмена сообщениями Exchange 4 и 3 должны быть включены, чтобы трафик единой системы обмена сообщениями Exchange из Таквила направлялся на серверы в Дублине.
