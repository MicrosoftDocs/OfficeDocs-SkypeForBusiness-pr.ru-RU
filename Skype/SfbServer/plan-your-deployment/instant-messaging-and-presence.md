## <a name="plan-for-instant-messaging-and-presence-in-skype-for-business-server"></a>Планирование обмена мгновенными сообщениями и присутствия в Skype для бизнеса Server

Серверы переднего Skype для бизнеса Server предоставляют основные функции, такие как обмен мгновенными сообщениями и присутствие, и включаются во все Skype для бизнеса Server развертывания. Доступно два выпуска: Skype для бизнеса Server выпуск Enterprise, предназначенная в основном для крупных организаций, и Skype для бизнеса Server выпуск Standard, которая предназначена в основном для небольших организаций, которые хотят меньших инвестиций в оборудование и не требуют параметров полной доступности. Оба выпуска поддерживают все Skype для бизнеса Server, включая чат, присутствие, конференцию и Корпоративная голосовая связь.
  
Обмен мгновенными сообщениями дает пользователям возможность общаться друг с другом в режиме реального времени со своих компьютеров, используя текстовые сообщения. Поддерживаются как двухсторонние, так и многосторонние сеансы обмена мгновенными сообщениями. Участник двухсторонней текстовой беседы может в любое время пригласить третьего участника. Когда это происходит, окно беседы изменяется, чтобы поддерживать функции конференции.
  
Presence предоставляет пользователям сведения о состоянии других пользователей в сети. Состояние присутствия пользователя предоставляет сведения, которые помогут другим решить, следует ли им пытаться связаться с пользователем и использовать ли мгновенные сообщения, телефон или электронную почту. Функция присутствия предлагает обмен мгновенными сообщениями всегда, когда это возможно, но также сообщает, когда пользователь находится на собрании или вне офиса, указывая, что обмен мгновенными сообщениями невозможен. Этот статус присутствия отображается как значок присутствия в Skype для бизнеса и других приложениях, в том числе клиенте microsoft Outlook обмена сообщениями и совместной работы, технологиях Microsoft SharePoint и Microsoft Office. Значок присутствия представляет текущую доступность и готовность пользователя к общению. 
  
### <a name="technical-requirements"></a>Технические требования

Мгновенные сообщения и присутствие всегда работают на выпуск Enterprise пулах и выпуск Standard серверах. Сведения о поддерживаемом оборудовании, операционных системах и программном обеспечении баз данных см. в приложениях [Certified Gateways, Requirements](../../SfbPartnerCertification/certification/infra-gateways.md) [for your Skype для бизнеса 2015](requirements-for-your-environment/requirements-for-your-environment.md)и [Infrastructure requirements for Skype для бизнеса Server 2019](../../SfBServer2019/plan/system-requirements.md).
  
### <a name="enabling-communication-with-external-users"></a>Включение связи с внешними пользователями

Вы можете значительно увеличить преимущества инвестиций в Skype для бизнеса Server, позволяя пользователям общаться с внешними пользователями. К внешним пользователям относятся:
  
- Удаленные пользователи: собственные пользователи вашей организации, когда они работают вне брандмауэров и используют свои ноутбуки или другие Skype для бизнеса Server устройства.
    
- Федераированные пользователи. Пользователи из компаний, с которыми вы работаете, также Skype для бизнеса Server. Чтобы ваши пользователи могли легко общаться с такими пользователями, с этими компаниями создаются отношения федерации. 
    
- Skype: Skype для бизнеса пользователи могут достигать сотен миллионов пользователей в Skype с помощью чата, голосовой связи и видео.
    
> [!NOTE]
> AOL, Yahoo и Google Talk больше не поддерживаются. 
  
> [!NOTE]
> Чтобы включить любой из этих сценариев или все эти сценарии, необходимо развернуть edge Server, чтобы обеспечить безопасную связь между Skype для бизнеса Server развертывания и внешними пользователями. Удаленные пользователи и пользователи вашей организации в федерадных организациях смогут видеть присутствие друг друга и общаться с помощью чата. 
  
> [!NOTE]
> Extensible Messaging and Presence Protocol (XMPP) поддерживается только для сценариев сертификации объединенной платформы совместной проверки интероперабельности (UCCP). 
  
### <a name="archiving-im-content"></a>Архивавка контента im

Skype для бизнеса Server предоставляет функции, которые можно использовать, если ваша организация должна соблюдать правила соответствия требованиям. С помощью архивации можно архивировать контент мгновенных сообщений для всех пользователей в организации или только для некоторых указанных пользователей. Подробные сведения см. [в материале Plan for archiving in Skype для бизнеса Server.](archiving/archiving.md) 
  
Если вы также Microsoft Exchange Server 2013 г., вы можете интегрировать архивировать Exchange с архиванию данных Skype для бизнеса Server и использовать единый инструмент для поиска обоих типов архивных данных. Дополнительные сведения см. в [Skype для бизнеса Server настройка Exchange Server архива.](../deploy/integrate-with-exchange-server/use-exchange-archiving.md)
  
### <a name="topologies-and-components"></a>Topologies and components

Для обмена мгновенными сообщениями и сведениями о присутствии требуются только следующие компоненты:
  
- Серверы переднего конца организации (известные как пул) или выпуск Standard сервер. Функции обмена мгновенными сообщениями и сведениями о присутствии всегда включены на этих серверах. Дополнительные сведения о topologies и управлении пулом переднего конца см. в этой части. [](high-availability-and-disaster-recovery/high-availability.md)
    
- Балансировка нагрузки, если у вас выпуск Enterprise пул переднего конца.
