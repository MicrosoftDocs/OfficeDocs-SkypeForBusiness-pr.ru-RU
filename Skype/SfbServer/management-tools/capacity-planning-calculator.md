# <a name="skype-for-business-server-capacity-planning-calculator"></a>Калькулятор планирования мощности Skype для бизнеса Server
 
**Сводка:** Использование средства калькулятора емкости.

> [!NOTE]
> В этой статье Skype для бизнеса Server загрузки 2015 г., но она применяется к:
> - Skype для бизнеса Server 2019 г.
> - Skype для бизнеса Server 2015 г.
  
Калькулятор [емкостей Skype для бизнеса Server 2015](https://www.microsoft.com/download/details.aspx?id=51196) г. и калькулятор емкости [Skype для бизнеса Server 2019](https://www.microsoft.com/download/details.aspx?id=57509) г. дополнены инструментом [планирования Skype для бизнеса](https://www.microsoft.com/download/details.aspx?id=50357) и документацией по развертыванию (Планирование развертывания Skype для бизнеса Server[2015](../plan-your-deployment/plan-your-deployment.md) г. и планирование развертывания Skype для бизнеса Server [2019](../../SfBServer2019/plan/plan-your-deployment-2019.md) г. соответственно). Используйте калькулятор после того, как вы рассмотрели руководство и создали рекомендуемую топологию с помощью средства планирования.
  
Калькулятор Skype для бизнеса Server емкости позволяет определить требования к серверу в зависимости от количества пользователей и средств связи, которые использует ваша организация. После определения профиля пользователя и функций, которые необходимо включить для пользователей, используйте калькулятор для определения количества серверов, памяти и пропускной способности. В этой версии калькулятора не указаны требования к диску I/O.
  
Вы можете извлечь большую пользу из калькулятора, если у вас есть точные подробные сведения о вашем профиле пользователя. Например, процент пользователей с поддержкой голосовой почты, среднее количество вызовов на одного пользователя в час, продолжительность звонков и процент одновоментных пользователей в конференциях могут внести огромную разницу в требованиях к серверу. Точность рекомендаций, созданных калькулятором, зависит от точности предоставляемой вами информации.
  
После использования средства планирования и калькулятора планирования емкости следует смоделировать предлагаемую и запланированную нагрузку, чтобы убедиться, что Skype для бизнеса Server будет надлежащим образом обеспечена. Чтобы выполнить стресс-тестирование под смоделированной нагрузкой, используйте средство Skype для бизнеса Server [и](https://www.microsoft.com/download/details.aspx?id=50367) производительности, задокументированное в [Skype для бизнеса Server Stress and Performance Tool.](./stress-and-performance-tool/stress-and-performance-tool.md)
  
## <a name="using-the-capacity-calculator"></a>Использование калькулятора емкости

Калькулятор — это Microsoft Excel таблица. Ячейки ввода окрашены в оранжевый цвет. Значения по умолчанию вписались в ячейки (за Skype для бизнеса Server 2015 г. 80 000 пользователей в одном пуле с двенадцатью серверами переднего конца, в то время как за Skype для бизнеса Server 2019 г. 106 000 пользователей в одном пуле с шестнадцатью серверами переднего конца), но эти значения следует изменить в соответствие с потребностями организации.
  
Модель использования содержит следующие разделы. Чтобы рассчитать требования к емкости, введите данные, описанные в верхней части листа и работая вниз по строке: 
  
 **Обмен мгновенными сообщениями и сведениями о присутствии**
  
- В **статье Число пользователей** введите число пользователей, которые будут подписаны одновременно. Это число, как правило, составляет 80% от общего числа предварительно заданной пользователей. В большинстве случаев 100% ваших одновременно включенных пользователей будут включены для чата и присутствия. По умолчанию для 2015 Skype для бизнеса Server 80 000 пользователей и 106 000 пользователей для Skype для бизнеса Server 2019 г.
    
- **Среднее число контактов в списке Контактов** указывает количество контактов, которые мы используем для проверки требований системы. Это число фиксированное, а не то, что нужно изменить.
    
  **Корпоративная голосовая связь**
  
- В **"Пользователи** Корпоративная голосовая связь" введите процент пользователей, включенных для Корпоративная голосовая связь. По умолчанию — 60%. 
    
- В среднем количестве вызовов на одного пользователя в час **(пик)** введите количество вызовов в час, в которые, как ожидается, будет участвовать средний пользователь во время пиковой нагрузки. По умолчанию используется значение 4. 
    
- В **процентном соотношении** вызовов, которые используют обход мультимедиа, введите процент звонков, размещенных пользователями, которые будут обходить сервер-посредник. По умолчанию этот показатель составляет 65%, но может быть ниже, если вы распространяете географически или имеете большой процент пользователей, которые работают из дома.
    
- В процентном соотношении голосовых пользователей, участвующих в **вызовах UC-PSTN,** введите процент звонков организации, которые являются телефонными вызовами UC-PSTN. По умолчанию — 60%.
    
- Процент голосовых пользователей, участвующих в **вызовах UC-UC,** показывает процент пользователей, которые включены для Корпоративная голосовая связь, которые будут включены только для вызовов UC-UC. Этот номер рассчитывается на основе ввода для процента голосовых пользователей, включенных **для вызовов UC-PSTN.** 
    
  **Конференции**
  
- В **процентах пользователей,** участвующих в одноваторных конференциях, введите процент пользователей, которые будут одновременно участвовать в конференциях. По умолчанию — 5%. 
    
- В процентном соотношении конференций только с групповым **чатом (без голоса)** введите процент конференций, которые будут включать только мгновенные сообщения и не включают аудио. По умолчанию — 10%
    
- В процентном соотношении пользователей, использующих конференц-конференции с использованием **dial-in,** введите процент участников конференций, которые будут одновременно использовать конференц-конференции с помощью dial-in. По умолчанию — 15%.
    
- В **процентном соотношении конференций с использованием голосовой** почты введите процент конференций, которые будут включать звук. 
    
  - Если в 20% голосовых конференций также будет регулярное видео, выберите поле "Включить видео" **(без** нескольких просмотров).
    
  - Если в 20% конференций будет также включено видео с несколькими представлениями, выберите поле **"Включите** много просмотр".
    
  - Если в 50% голосовых конференций также будет включен общий доступ к приложениям, выберите контрольный ящик **"В том числе совместное** использование приложений".
    
  - Если 20% голосовых конференций включают отправку данных, например PowerPoint презентаций, выберите контрольный ящик **Including web conferencing.**
    
  **Мобильность**
  
- В **процентном соотношении** пользователей, включенных для mobility, введите процент пользователей, которым будет включено подключение к Skype для бизнеса Server с помощью мобильных устройств. По умолчанию — 40%. 
    
Когда вы ввели всю необходимую информацию, калькулятор емкости оценивает ваши требования. Желтые ячейки показывают рассчитанные значения для ЦП, памяти и пропускной способности на основе тестов, Skype для бизнеса Server лабораториях производительности. Номера предоставляются в качестве руководства, не каждый вариант проверяется и проверяется. Вычисляются следующие значения: 
  
- **Front End CPU**: Процент использования ЦП, если весь груз обрабатывается одним передним конечным сервером тех же спецификаций, что и сервер, используемый при тестировании (см. описание в конце этой статьи).
    
- **Сеть в Mbps:** требования к пропускной способности в мегабитах в секунду (Mbps) для соответствующей рабочей нагрузки.
    
- **Память в ГБ:** память, требуемая в гигабайтах (ГБ) для соответствующей рабочей нагрузки.
    
Зеленые ячейки показывают рекомендации для введенной модели использования. 
  
- Общее число серверов переднего конечного **сервера:** количество необходимых физических серверов основано на выделенных серверах, работающих Skype для бизнеса Server 2015 г. с двойным процессором, шестиъядерным, с 2260 мегациклами или Skype для бизнеса Server 2019 г. с intel Xeon E5-2673 v3, двойным процессором, шестиъядерным.
    
    Обратите внимание, что рекомендуется включить гипертеку и было доказано, что она повышает производительность для серверов, поддерживаюющих аудио- и видеосъемку.
    
- **Edge Servers**. Количество необходимых edge Servers, основанное на 30% всех одновременно связывающихся пользователей через edge Servers. Этот процент не может быть изменен в калькуляторе. 
    
- Архивации и записи данных о **вызове/качестве службы experience Store:** количество магазинов, необходимых для функций архивации или мониторинга, если они включены в организации.
    
- **Back End Database Server Required (Pools Required)**— количество серверов базы данных, необходимых для поддержки выбранной рабочей нагрузки.
    
Кроме того, в строке рядом с Total Front End Servers предоставляются дополнительные сведения о нагрузке на серверы и сеть для всех запланированных рабочих нагрузок вместе взятых.
  
- **Средняя загрузка ЦП:** среднее использование ЦП на сервере переднего конца.
    
- **Сеть в Mbps:** необходимое распределение пропускной способности для поддержки введенной модели использования.
    
- **Память в ГБ:** память в гигабайтах, необходимая для каждого переднего сервера.
    
### <a name="adjusting-for-your-processors"></a>Настройка для процессоров

Все цифры использования ЦП в электронной таблице предполагают, что каждый сервер Skype для бизнеса Server 2015 года имеет двойной процессор, шестиъядерный с 2,26 ГГц, не менее 32 ГБ памяти и 8 или более 10 000-RPM жестких дисков с свободным диском не менее 72 ГБ. Для каждого сервера Skype для бизнеса Server 2019 г. все цифры использования ЦП в электронной таблице предполагают, что каждый сервер имеет двойной процессор, шестиъядерный с intel Xeon E5-2673 v3, не менее 64 ГБ памяти и 8 или более жестких дисков с дисками с дисками с объемом диски 72 ГБ.
  
Если на серверах разные процессоры, можно настроить цифры в соответствие с оборудованием.
