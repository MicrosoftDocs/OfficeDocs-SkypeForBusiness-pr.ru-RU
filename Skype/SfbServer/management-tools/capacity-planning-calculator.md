# <a name="skype-for-business-server-capacity-planning-calculator"></a>Калькулятор планирования мощности Skype для бизнеса Server
 
**Сводка:** Использование средства "Калькулятор емкости".

> [!NOTE]
> Эта статья содержит ссылку на загрузку Skype для бизнеса Server 2015, но она применима к:
> - Skype для бизнеса Server 2019.
> - Skype для бизнеса Server 2015.
  
Калькулятор емкости [Skype для бизнеса server 2015 Capacity](https://www.microsoft.com/download/details.aspx?id=51196) и [Skype для бизнеса Server 2019](https://www.microsoft.com/download/details.aspx?id=57509) , чтобы расширить [средство планирования Skype для бизнеса](https://www.microsoft.com/download/details.aspx?id=50357) и документацию по развертыванию ([Планирование развертывания Skype для бизнеса Server 2015](../plan-your-deployment/plan-your-deployment.md) и [Планирование развертывания Skype для бизнеса Server 2019](../../SfBServer2019/plan/plan-your-deployment-2019.md) соответственно). Используйте калькулятор после просмотра руководства и создания рекомендуемой топологии с помощью средства планирования.
  
Калькулятор емкости Skype для бизнеса Server помогает определить требования к серверу на основе количества пользователей и средств связи, которые использует Организация. Определив профиль пользователя и функции, которые вы хотите включить для пользователей, с помощью калькулятора Определите количество серверов, памяти и полосу пропускания. Эта версия калькулятора не предоставляет рекомендаций по требованиям к диску ввода-вывода.
  
Вы можете воспользоваться преимуществами калькулятора, если у вас есть точные сведения об определенном профиле пользователя. Например, процент пользователей с включенной поддержкой голосовой связи, среднее количество вызовов на одного пользователя в час, продолжительность вызова и процент одновременных пользователей в конференциях могут стать огромным различием в требованиях сервера. Точность рекомендаций, создаваемых калькулятором, зависит от точности предоставляемых сведений.
  
После использования средства планирования и калькулятора планирования мощности необходимо имитировать предлагаемую и запланированную нагрузку, чтобы убедиться, что в Skype для бизнеса Server будет достаточно готов к работе. Для выполнения нагрузочного тестирования при имитации нагрузки используйте средство работы с [нагрузочными испытаниями и производительности Skype для бизнеса Server](https://www.microsoft.com/download/details.aspx?id=50367) , задокументированное в [средстве нагрузки и производительности Skype для бизнеса Server](https://technet.microsoft.com/library/mt631400.aspx).
  
## <a name="using-the-capacity-calculator"></a>Использование калькулятора мощности

Калькулятор представляет собой электронную таблицу Microsoft Excel. Ваши входные ячейки окрашены в оранжевый цвет. Значения по умолчанию вводятся в ячейки (для Skype для бизнеса Server 2015, 80 000 пользователей в одном пуле с двенадцать интерфейсных серверов, в то время как Skype для бизнеса Server 2019, 106 000 пользователей в одном пуле с шестнадцатем интерфейсных серверов), но эти значения следует изменить на в соответствии с потребностями Организации.
  
Модель использования содержит следующие разделы. Чтобы рассчитать требования к емкости, введите данные в соответствии с описанием, начиная с верхнего края листа и по строкам: 
  
 **Обмен мгновенными сообщениями и сведениями о присутствии**
  
- В разделе **число пользователей**введите число пользователей, которые будут одновременно входить в систему. Это значение обычно составляет 80% от общего числа подготовленных пользователей. В большинстве случаев 100% от параллельных пользователей будут включены для обмена мгновенными сообщениями и присутствия. Значение по умолчанию: 80 000 для Skype для бизнеса Server 2015 и 106 000 пользователей Skype для бизнеса Server 2019.
    
- **Среднее число контактов в списке** контактов указывает количество контактов, которые мы используем для проверки требований к системе. Это значение является фиксированным, а не каким-либо изменением.
    
  **Корпоративная голосовая связь**
  
- В разделе **Пользователи, для которых включена поддержка корпоративной голосовой связи**, введите процент пользователей, для которых включена поддержка корпоративной голосовой связи. Значение по умолчанию — 60%. 
    
- В **среднем число вызовов на пользователя в час (пик)** введите число звонков в час, в течение которых пользователь должен принять участие в период пиковой нагрузки. По умолчанию используется значение 4. 
    
- В **процентах от вызовов, использующих обход сервера-посредника**, введите процент звонков, включенных пользователями, которые будут обходить сервер-посредник. Значение по умолчанию составляет 65%, но может быть ниже, если вы географически разбросаны географически или используете большой процент пользователей, работающих из дома.
    
- В **процентах от голосовых пользователей, участвующих в вызовах UC для PSTN**, введите процент вызовов Организации, которые являются телефонными звонками UC/PSTN. Значение по умолчанию — 60%.
    
- **Процент пользователей голосовой связи, участвующих в вызовах UC с** UC, показывает процент пользователей, для которых включена поддержка корпоративной голосовой связи, которые будут включены только для вызовов UC UC. Это значение рассчитывается на основе введенного **процента пользователей голосовой связи, для которых включены вызовы UC-PSTN**. 
    
  **Конференции**
  
- В **процентах от числа пользователей на одновременных конференциях**введите процент пользователей, которые будут одновременно участвовать в конференциях. Значение по умолчанию — 5%. 
    
- В **процентах конференций только для групповых мгновенных сообщений (без голосовой связи)** введите процент конференций, в которых будет использоваться только обмен мгновенными сообщениями и не включайте звук. Значение по умолчанию — 10%
    
- В **процентах от пользователей, использующих Конференц-связь с телефонным подключением**, введите процент участников в конференциях, которые будут использовать конференц-связь с телефонным подключением одновременно. Значение по умолчанию составляет 15%.
    
- В **процентах от конференций, использующих Voice**, введите процент конференций, в которых будет использоваться звук. 
    
  - Если в 20% ваших голосовых конференций также будет использоваться обычный видеоролик, установите флажок **включая видео (без просмотра)** .
    
  - Если в 20% конференций также будет использоваться видео с несколькими представлениями, установите флажок **включая множественное представление** .
    
  - Если 50% ваших голосовых конференций также включает общий доступ к приложениям, установите флажок **включить общий доступ к приложениям** .
    
  - Если 20% ваших голосовых конференций включают отправку данных, например презентаций PowerPoint, установите флажок **Включение веб-конференций** .
    
  **Мобильность**
  
- В **процентах пользователей, для которых включена мобильность**, введите процент пользователей, которым будет разрешено подключаться к Skype для бизнеса Server с помощью мобильных устройств. Значение по умолчанию — 40%. 
    
После ввода всех необходимых сведений калькулятор емкости оценивает требования. В желтых ячейках отображаются вычисляемые значения для ЦП, памяти и требования к пропускной способности, основанные на тестах, выполняемых в лабораториях по производительности Skype для бизнеса Server Эти номера предоставляются в качестве рекомендации, а не каждый одиночный вариант тестируется и проверяется. Рассчитываются следующие значения: 
  
- **Интерфейсный ЦП**: процент использования ЦП, если нагрузка обрабатывается одним сервером переднего плана тех же спецификаций, что и сервер, который использовался при тестировании (описание в конце этой статьи).
    
- **Сеть в Мбит/с**: требования к пропускной способности в мегабит в секунду (Мбит/с) для соответствующей рабочей нагрузки.
    
- **Память в ГБ**: объем памяти, необходимый для соответствующей рабочей нагрузки, в гигабайтах (ГБ).
    
В зеленых ячейках отображаются рекомендации для модели использования, которую вы ввели. 
  
- **Общее количество серверов переднего плана**: требуемое количество физических серверов основано на выделенных серверах Skype для бизнеса Server 2015 с двумя процессорами, Hex-ядрами с 2 260 мегациклов или Skype для бизнеса Server 2019 с Intel Xeon 3 – 2673 v3, с двумя процессорами, Hex и Core.
    
    Обратите внимание, что включение функции Hyper-Threading рекомендуется и было проверено на повышение производительности для серверов, поддерживающих аудио и видео.
    
- **Пограничные серверы**: необходимое количество пограничных серверов на основе 30% всех параллельных пользователей, взаимодействующих через пограничные серверы. Этот процент не может быть изменен в калькуляторе. 
    
- **Архивация/запись сведений о вызовах/хранилище служб качества взаимодействия**: количество хранилищ, необходимых для функций архивации или мониторинга, если они включены в Организации.
    
- **Требуется сервер серверной базы данных (требуются пулы)**: количество внутренних серверов баз данных, необходимых для поддержки выбранной рабочей нагрузки.
    
Кроме того, в строке рядом с пунктом общее число серверов переднего плана предоставляются дополнительные сведения о загрузке серверов и сети для всех запланированных рабочих нагрузок.
  
- **Средняя загрузка ЦП**: средняя загрузка ЦП на сервер переднего плана.
    
- **Сеть в Мбит/** с: необходимое распределение пропускной способности для поддержки указанной модели использования.
    
- **Память в ГБ**: объем памяти (в гигабайтах), необходимый для каждого сервера переднего плана.
    
### <a name="adjusting-for-your-processors"></a>Настройка процессоров

Все цифры использования ЦП в электронной таблице предполагают, что на каждом сервере Skype для бизнеса 2015 установлен двойной процессор, Hex-ядра с частотой 2,26 ГГц, по крайней мере до 32 ГБ памяти, а также 8 или более жестких дисков с 10 000 об/мин и не менее 72 ГБ свободного места на диске. Для каждого сервера Skype для бизнеса Server 2019 на всех рисунках, используемых в электронной таблице, предполагается, что на каждом сервере установлен двойной процессор в шестнадцатеричном режиме с Intel Xeon 3 – 2673 v3, не менее 64 ГБ памяти, а также 8 или более 10 000 жестких дисков емкостью не менее 72 ГБ свободного места Сегодня.
  
Если у ваших серверов разные процессоры, вы можете настроить их для согласования с оборудованием.
