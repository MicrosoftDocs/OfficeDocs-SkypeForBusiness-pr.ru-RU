# <a name="cls-logger-for-skype-for-business-server-2015"></a>ClS Logger для Skype для бизнеса Server 2015
 
**Сводка:** Learn how to use the Centralized Logging Service (CLS) Logger in Skype for Business Server 2015.
  
ClS Logger — это средство, которое помогает управлять журналами, созданными службой централизованного ведения журналов.
  
## <a name="prerequisites"></a>Предварительные условия

Чтобы успешно использовать CLS Logger, необходимо убедиться, что истинны следующие:
  
- Средство используется на компьютере, который является членом домена, в котором запущена служба централизованного ведения журналов (CLS). В настоящее время это средство не поддерживается в удаленных сеансах PowerShell.
    
- Файл Default.tmx из папки трасситуры (папки, в которой данные трассии фиксируются для CLS) и Snooper необходимо скопировать в ту же папку, в которой установлено средство CLS Logger.
    
## <a name="check-the-logging-status-of-a-set-of-poolscomputers"></a>Проверка состояния ведения журнала для набора пулов или компьютеров

Используйте следующие команды для проверки состояния ведения журнала:
  
1. На вкладке "Сценарии запуска и остановки" выберите группу пулов и/или компьютеров в представлении дерева топологии.
    
2. Нажмите кнопку "Состояние ведения журнала".
    
3. Просмотреть выходные данные команды в области вывода команд PowerShell, чтобы получить конкретные данные о состоянии ведения журнала выбранных пулов и/или компьютеров.
    
## <a name="start-an-existing-scenario"></a>Запуск существующего сценария

Чтобы запустить существующий сценарий:
  
1. На вкладке "Сценарии запуска/остановки" выберите существующий сценарий в выпадаемом меню "Сценарии".
    
2. Выберите группу пулов и/или компьютеров в представлении дерева топологии.
    
3. Нажмите кнопку "Начать сценарий". Пользовательский интерфейс будет отключен до завершения операции. В крупных развертываниях это может происходить медленно.
    
4. Пользовательский интерфейс снова будет включен после успешного начала сценария, подробные сведения о действии также будут показаны в области вывода команд PowerShell.
    
5. Задача может занять некоторое время до того, как clS заберет журнал перед новыми данными из этого сценария.
    
## <a name="stop-an-existing-scenario"></a>Остановка существующего сценария

Чтобы остановить существующий сценарий:
  
1. На вкладке "Сценарии запуска/остановки" выберите существующий сценарий в выпадаемом меню "Сценарии".
    
2. Выберите группу пулов и/или компьютеров в представлении дерева топологии.
    
3. Нажмите кнопку "Остановить сценарий". Пользовательский интерфейс будет отключен до завершения операции. В крупных развертываниях это может происходить медленно.
    
4. Пользовательский интерфейс снова будет включен после того, как сценарий будет остановлен, подробные сведения о действии также будут показаны в области вывода команд PowerShell.

## <a name="search-for-logs"></a>Поиск журналов

Чтобы найти журналы, перейдите на вкладку "Поиск журналов CLS" и нажмите кнопку "Журналы поиска" после заполнения отображаемой области, как описано ниже:
  
> **Папка файла журнала** Папка для сохранения результатов поиска в журнале. (Обязательно)
> 
> **Уровень журнала** Это определяет самый низкий уровень, который будет показываться в результатах. Например, если выбрано предупреждение, будут показаны только warning, Error и Fatal. Значение по умолчанию : Debug.
> 
> **Пулы** Пулы компьютеров для выполнения поиска в журнале. Это родительские узлы в представлении дерева. (Обязательно)
> 
> **Компьютеры** Отдельные компьютеры, на которые выполняется поиск в журнале. Это все узлы в древовидном представлении. (Обязательно)
> 
> **Время начала** Период времени, в течение которых CLS запрашивает журналы. (Обязательно)
> 
> **Время окончания** Период времени, в течение которых CLS перестанет запрашивать журналы. (Обязательно)
> 
> **Компоненты** Используется для выбора компонентов, которые необходимо добавить в запрос. (Необязательно)
> 
> **ИД вызова** ИД вызова любых диалогов SIP, по которые необходимо отфильтровать. Обратите внимание, что в этом поле используется точное соответствие. (Необязательно)
> 
> **ИД конференции** ИД любой конференции, по которая будет фильтроваться. Обратите внимание, что в этом поле используется точное соответствие. (Необязательно)
> 
> **IP-адрес** IP-адрес, по который необходимо отфильтровать. Обратите внимание, что в этом поле используется точное соответствие. (Необязательно)
> 
> **ИД корреляции** Отчеты трассировки, логически связанные друг с другом по этому ИД. (Необязательно)
> 
> **Номер телефона** Фильтрация по номеру телефона. (Необязательно)
> 
> **URI SIP** Фильтрация по SIP URI. (Необязательно)
> 
> **Содержимое сообщения SIP содержит** Фильтрация по содержимому сообщений SIP, это подстрока поиска в этом поле. (Необязательно)
> 
> **Match Any** Выполняет поиск с помощью логического запроса OR, если он был проверен. Значение по умолчанию : Точное совпадение всех параметров.
> 
> **Пропуск сетевых журналов** Пропускает поиск по любым сетевым журналам, если он был проверен.

## <a name="create-a-scenario"></a>Создание сценария

1. На **вкладке "Изменение сценариев"** нажмите кнопку **"Создать** сценарий".
    
    > [!NOTE]
    > При создании нового сценария будет клонирована конфигурация выбранного сценария. Если **щелкнуть "Очистить параметры"** перед созданием нового сценария, он начнется без выбранных компонентов и флажков.
  
2. Введите имя создаемого сценария и нажмите клавишу ВВОД или нажмите кнопку "ОК".
    
3. Теперь будет создан новый сценарий. После успешного создания будет выбран выпадание "Сценарии" с только что созданным сценарием.
    
## <a name="modify-a-scenario"></a>Изменение сценария

![Снимок экрана CLS Logger, сценарии редактирования](../../media/abbbcac0-8a2e-48af-a22f-4fee0283a29f.png)
  
1. На **вкладке "Изменение сценариев"** найдите нужный сценарий для изменения.
    
2. Внести нужные изменения в компоненты, уровни и флаги.
    
3. Нажмите **кнопку "Сохранить сценарий".**
    
4. После успешного сохранения сценария в области сведений о сценарии будет обновлена конфигурация.
    
## <a name="delete-a-scenario"></a>Удаление сценария

1. На **вкладке "Изменение сценариев"** выберите существующий сценарий в выпадаемом меню "Сценарии".
    
2. Щелкните **"Удалить** сценарий", чтобы удалить сценарий.
    
3. После подтверждения действия сценарий будет удален.
