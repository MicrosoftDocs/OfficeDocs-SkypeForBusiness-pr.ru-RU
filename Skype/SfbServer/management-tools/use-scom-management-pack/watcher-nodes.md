# <a name="learn-how-to-install-and-configure-watcher-nodes"></a>Узнайте, как установить и настроить узлы для просмотра
 
**Сводка:** Установка и настройка узлов для просмотра для Skype для бизнеса Server синтетических транзакций.
  
Узлы watcher — это компьютеры, которые периодически Skype для бизнеса Server синтетической транзакции. Синтетические транзакции Windows PowerShell, которые проверяют, работают ли ключевые сценарии пользователей, такие как возможность войти или обмениваться мгновенными сообщениями, как и ожидалось. В Skype для бизнеса Server 2015 System Center operations Manager может запускать синтетические транзакции, показанные в следующей таблице, которая включает три типа синтетических транзакций:
  
- **По умолчанию** Синтетические транзакции, которые узел просмотра выполняет по умолчанию. При создании нового узла просмотра можно указать, какие синтетические транзакции будут запускаться этим узлом. (Это цель параметра Tests, используемого в New-CsWatcherNodeConfiguration.) Если при запуске узла просмотра не используется параметр Tests, он автоматически запустит все синтетические транзакции по умолчанию и не будет запускать ни одной из синтетических транзакций по умолчанию. Это означает, например, что узел просмотра будет настроен для запуска Test-CsAddressBookService, но не будет настроен для запуска Test-CsExumConnectivity теста.
    
- **Не по умолчанию** Тесты, которые узлы для просмотра не запускались по умолчанию. (Подробные сведения см. в описании типа Default.) Однако узел просмотра может быть включен для запуска любых синтетических транзакций по умолчанию. Это можно сделать при создании узла просмотра (с помощью New-CsWatcherNodeConfiguration или в любое время после создания узла просмотра. Обратите внимание, что многие синтетические транзакции по умолчанию требуют дополнительных действий по настройке. Дополнительные сведения об этих шагах см. в материале [Special Setup Instructions for Synthetic Transactions.](test-users-and-settings.md#special_synthetictrans)
    
- **Расширенный** Особый тип синтетической транзакции по умолчанию. В отличие от других искусственных транзакций расширенные тесты могут выполняться несколько раз в течение каждого прохода. Это полезно при проверке поведения, например нескольких маршрутов голосовой связи между общедоступными телефонными сетями (PSTN) для пула. Это можно настроить, просто добавив несколько экземпляров расширенного теста в узел просмотра.
    
Дополнительные сведения о процессе добавления других синтетических транзакций в узел просмотра см. в материале [Configure a Watcher Node to Run Synthetic Transactions.](watcher-nodes.md#enable_synthetic_trans) Вы также можете использовать Skype для бизнеса Server для удаления синтетических транзакций из узла просмотра.

Вам не нужно устанавливать узлы для просмотра, чтобы использовать System Center Operations Manager. Если эти узлы не устанавливаются, можно получать оповещения в режиме реального времени из компонентов Skype для бизнеса Server 2015 г. при любой проблеме. (Пакет управления компонентами и пользователями не использует узлы просмотра.) Однако для мониторинга сценариев с помощью пакета Active Monitoring Management необходимы узлы наблюдателей.
  
> [!NOTE]
> Администраторы также могут запускать синтетические транзакции вручную, без использования или установки диспетчера операций. В зависимости от размера развертывания Skype для бизнеса Server, синтетические транзакции могут использовать большое количество памяти компьютера и времени процессора. В связи с этим рекомендуется использовать выделенный компьютер в качестве узла просмотра. Например, не следует настраивать сервер Skype для бизнеса Server для действий в качестве узла просмотра. Узлы watcher должны соответствовать тем же основным требованиям к оборудованию, что и любой другой компьютер Skype для бизнеса Server топологии. 
  
> [!NOTE]
> Устаревший узел просмотра Lync Server 2013 не может быть collocated на том же компьютере, что и узел просмотра Skype для бизнеса Server 2015 г., так как основные системные файлы для Lync Server 2013 и Skype для бизнеса Server 2015 не могут быть установлены на одном компьютере. Однако узлы Skype для бизнеса Server 2015 года могут одновременно отслеживать Skype для бизнеса Server 2015 и Lync Server 2013. Синтетические транзакции по умолчанию поддерживаются для обеих версий продукта. 
  
Узлы для просмотра Lync Server 2013 могут быть развернуты внутри или за пределами предприятия для проверки:
  
- Подключение пользователей внутри предприятия к пулам
    
- Подключение через сети периметра для удаленных пользователей, работающих за пределами предприятия.
    
- Подключение к устройствам обеспечения связи в филиалах.
    
- Подключение к Lync Server 2013 внутри предприятия и через сети периметра.
    
Чтобы упростить администрирование, для внутри и за пределами предприятия доступны различные параметры проверки подлинности. Подробные сведения см. в материале Настройка узла [watcher для запуска синтетических транзакций.](watcher-nodes.md#enable_synthetic_trans)
  
Чтобы настроить компьютер в качестве узла просмотра, сначала необходимо выполнить следующие необходимые условия: 
  
- Установите System Center диспетчера операций и импортировать Skype для бизнеса Server 2015 г. Кроме того, сначала необходимо убедиться, что компьютер узла просмотра отвечает всем требованиям для установки Skype для бизнеса Server 2015 г.
    
- Установка следующих элементов на компьютер узла просмотра:
    
  - Полная версия платформа .NET Framework 4.5
    
  - Windows Identity Foundation
    
  - Windows PowerShell 3.0
    
После того как необходимые условия будут выполнены, можно настроить узел просмотра, следуя следующим шагам:
  
1. Установите Skype для бизнеса Server 2015 основных файлов на компьютере узла просмотра.
    
2. Установите System Center диспетчера операций на компьютер узла просмотра.
    
3. Запустите Watchernode.msi исполняемый файл.
    
4. Чтобы настроить тестовые учетные записи пользователей, которые будут использовать узел просмотра, используйте комлет **New-CsWatcherNodeConfiguration.**
    
## <a name="install-the-skype-for-business-server-2015-core-files-and-the-rtclocal-database"></a>Установка Skype для бизнеса Server 2015 г. и базы данных RTCLocal

Чтобы установить Skype для бизнеса Server 2015 основных файлов на компьютере, выполните следующую процедуру. База данных RTCLocal будет автоматически установлена при установке основных файлов. Обратите внимание, что вам не нужно SQL Server на узлы просмотра. SQL Server Express автоматически устанавливается.
  
Для установки Skype для бизнеса Server 2015 г. и базы данных RTCLocal:
  
1. На компьютере узла-наблюдателя щелкните кнопку Пуск, последовательно щелкните пункты Все программы, Стандартные, щелкните правой кнопкой мыши значок Командная строка и в контекстном меню выберите пункт Запуск от имени администратора.
    
2. В окне консоли введите следующую команду и нажмите кнопку ENTER. Убедитесь, что введите соответствующий путь к файлам установки Skype для бизнеса Server: D:\Setup.exe/BootstrapLocalMgmtTo убедитесь, что основные компоненты Skype для бизнеса Server успешно установлены, нажмите кнопку Начните **,** нажмите кнопку Все программы **,** нажмите **Skype для бизнеса Server 2015,** а затем нажмите **кнопку Skype для бизнеса Server Management Shell**. В командной Skype для бизнеса Server введите следующую команду Windows PowerShell и нажмите кнопку ENTER:
  
```PowerShell
Get-CsWatcherNodeConfiguration
```

> [!NOTE]
> При первом запуске этой команды данные не будут возвращены, так как вы еще не настроили компьютеры узлов просмотра. Если команда выполняется без возврата ошибки, можно предположить, что Skype для бизнеса Server успешно выполнена. 
  
Если ваш узел просмотра находится в сети периметра, можно выполнить следующую команду для проверки установки Skype для бизнеса Server 2015 г.:
  
Get-CsPinPolicyYou будет получать аналогичные сведения в зависимости от количества политик ПИН-кода, настроенных для использования в организации:
  
Identity : Global
  
Описание:
  
MinPasswordLength : 5
  
PINHistoryCount : 0
  
AllowCommonPatterns : False
  
PINLifetime : 0
  
MaximumLogonAttempts:
  
Если вы видите сведения о пин-политиках, основные компоненты успешно установлены.
  
## <a name="install-the-operation-manager-agent-files-on-a-watcher-node"></a>Установка файлов агентов диспетчера операций на узел watcher

Как и Skype для бизнеса Server для оповещения компонентов отчетности, узел Skype для бизнеса Server 2015 года требует установки System Center агентов operations Manager. Это позволяет запускать синтетические транзакции и оповещать об этом на сервере корневого управления System Center operations Manager.
  
Чтобы установить файлы агентов, выполните процедуры, перечисленные в Настройка Skype для бизнеса Server компьютеров, которые [будут отслеживаться](configure-computers-to-monitor.md).
  
## <a name="configure-a-watcher-node-to-run-synthetic-transactions"></a>Настройка узла watcher для запуска синтетических транзакций
<a name="enable_synthetic_trans"> </a>

После установки System Center диспетчера операций необходимо настроить сам узел просмотра. Действия, которые необходимо предпринять для этого, зависят от того, находится ли ваш узел просмотра в сети периметра или за пределами сети периметра. 
  
При настройке узла-наблюдателя необходимо также выбрать тип метода проверки подлинности для развертывания этим узлом. Skype для бизнеса Server 2015 г. позволяет выбрать один из двух методов проверки подлинности: доверенный сервер или проверка подлинности учетных данных. В следующей таблице показаны различия между этими двумя методами:
  
|&nbsp;|**Описание**|**Поддерживаемые расположения**|
|:-----|:-----|:-----|
|TrustedServer  <br/> |Использует сертификат для олицетворения внутреннего сервера и обхода проблем, связанных с проверкой подлинности.  <br/> Полезно для администраторов, предпочитающих управлять одним сертификатом, вместо множества паролей пользователей на каждом узле пользователя.  <br/> |На предприятии.  <br/> С помощью этого метода узел наблюдатель должен быть в том же домене, что и отслеживаемые пулы. Если узел просмотра и пулы находятся в разных доменах, используйте проверку подлинности учетных данных.  <br/> |
|Согласование  <br/> |Надежно сохраняет имена пользователей и пароли в диспетчере учетных данных Windows на каждом узле-наблюдателе.  <br/> Этот режим требует больше управления паролями, но является единственным вариантом для узлов просмотра за пределами предприятия. Эти узлы-наблюдатели не могут рассматриваться как доверенные конечные точки при прохождении проверки подлинности.  <br/> |Вне предприятия.  <br/> На предприятии.  <br/> |
   
## <a name="configure-a-watcher-node-to-use-trusted-server-authentication"></a>Настройка узла watcher для использования проверки подлинности доверенных серверов
<a name="enable_synthetic_trans"> </a>

Если ваш узел просмотра находится в сети периметра, использование проверки подлинности Trusted Server может значительно сократить задачи администрирования, сохранив один сертификат, а не используя множество паролей учетных записей пользователей.
  
Чтобы настроить проверку подлинности доверенных серверов, сначала необходимо создать доверенный пул приложений для узла просмотра компьютера. После создания доверенного пула приложений необходимо настроить синтетические транзакции на этом узле для просмотра в качестве доверенных приложений.
  
> [!NOTE]
> Доверенным приложением является приложение, которому предоставляется доверенный статус для работы в Skype для бизнеса Server 2015 г., но не является встроенной частью продукта. Состояние доверенного приложения означает, что приложение не должно будет проходить проверку подлинности при каждом запуске.
  
Чтобы создать доверенный пул приложений, откройте командную Skype для бизнеса Server и запустите команду, аналогичную этой:
  
```PowerShell
New-CsTrustedApplicationPool -Identity atl-watcher-001.litwareinc.com -Registrar atl-cs-001.litwareinc.com -ThrottleAsServer $True -TreatAsAuthenticated $True -OutboundOnly $False -RequiresReplication $True -ComputerFqdn atl-watcher-001.litwareinc.com -Site Redmond
```

> [!NOTE]
> Подробные сведения о параметрах предыдущей команды введите следующее из Skype для бизнеса Server командной оболочки: 
  
```PowerShell
Get-Help New-CsTrustedApplicationPool -Full | more
```

После создания доверенного пула приложений можно настроить узел пользователя для запуска синтетических транзакций в качестве доверенного приложения с помощью **командлета New-CsTrustedApplication** и команды, аналогичной этому:
  
```PowerShell
New-CsTrustedApplication -ApplicationId STWatcherNode -TrustedApplicationPoolFqdn atl-watcher-001.litwareinc.com -Port 5061
```

После завершения этой команды и создания доверенного приложения необходимо запустить командлет **Enable-CsTopology,** чтобы убедиться, что изменения вступает в силу:
  
```PowerShell
Enable-CsTopology
```

Учетная запись узла узла просмотра требует возможности запрашивать CMS для некоторых синтетических транзакций. Чтобы разрешить эту возможность, добавьте учетную запись компьютера узла просмотра в группу безопасности RTCUniversalReadOnlyAdmins. После репликации AD перезапустите компьютер.
  
Чтобы убедиться в том, что новое доверяемого приложения создано, введите следующее в запросе Skype для бизнеса Server Management Shell:
  
```PowerShell
Get-CsTrustedApplication -Identity "atl-watcher-001.litwareinc.com/urn:application:STWatcherNode"
```

## <a name="configure-a-default-certificate-on-the-watcher-node"></a>Настройка сертификата по умолчанию в узле watcher
<a name="enable_synthetic_trans"> </a>

Каждый узел просмотра, использующий проверку подлинности TrustedServer, должен иметь сертификат по умолчанию, заданный с помощью мастера Skype для бизнеса Server развертывания. 
  
Назначение сертификата по умолчанию:
  
1. Нажмите кнопку Начните, щелкните Все программы, нажмите кнопку Skype для бизнеса Server 2015, а затем нажмите кнопку Skype для бизнеса Server мастер развертывания. 
    
2. В мастере Skype для бизнеса Server развертывания щелкните Установить или обновить Skype для бизнеса Server систему, а затем нажмите кнопку Выполнить под заголовком Запрос, Установка или Назначение сертификата. 
    
> [!NOTE]
> Если кнопка Выполнить не активна, может понадобиться щелкнуть пункт Выполнить в разделе Установка локального хранилища конфигурации. 
  
Выполните одно из указанных ниже действий.
  
- Если у вас уже есть сертификат, который можно использовать в качестве сертификата по умолчанию, нажмите кнопку По умолчанию в мастере сертификата, а затем нажмите Назначить. Выполните действия, приведенные в мастере назначения сертификатов, чтобы назначить сертификат.
    
- Если вам нужно запросить сертификат для использования сертификата по умолчанию, щелкните Запрос, а затем выполните действия мастера запроса сертификата, чтобы запросить этот сертификат. Если использовать значения по умолчанию для сертификата веб-сервера, будет получен сертификат, который можно назначить в качестве сертификата по умолчанию.
    
## <a name="install-and-configure-a-watcher-node"></a>Установка и настройка узла watcher
<a name="enable_synthetic_trans"> </a>

После перезапуска узла просмотра и настройки сертификата необходимо запустить файл Watchernode.msi. (Необходимо выполнить Watchernode.msi на любом компьютере, на котором установлены файлы агентов Operations Manager и Skype для бизнеса Server основных компонентов 2015 г.). 
  
Установка и настройка узла просмотра:
  
1. Откройте оболочку управления Skype для бизнеса Server, нажав кнопку Начните, щелкнув все программы, нажав Skype для бизнеса Server 2015 г., а затем нажав Skype для бизнеса Server управленческой оболочки. 
    
2. В командной оболочке введите следующую команду и нажмите кнопку ENTER (убедитесь, что указан фактический путь к копии Watchernode.msi):
    
```PowerShell
C:\Tools\Watchernode.msi Authentication=TrustedServer
```

> [!NOTE]
> Вы также можете выполнить Watchernode.msi n из окна команды. Чтобы открыть командное окно, нажмите кнопку Пуск, щелкните правой кнопкой мыши пункт Командная строка и выберите Запуск от имени администратора. Когда откроется окно команды, введите ту же команду, показанную на шаге 2 выше. 
  
> [!IMPORTANT]
> В предыдущей команде проверка подлинности между именем и значением=TrustedServer является конфиденциальной. Его необходимо ввести точно так, как показано. Например, эта команда не работает, так как не использует правильную оболочку буквы: 
  
```PowerShell
C:\Tools\Watchernode.msi authentication=trustedserver
```

Режим TrustedServer можно использовать только с компьютерами, которые находятся в сети периметра. Если узел просмотра работает в режиме TrustedServer, администраторам не нужно поддерживать тестовые пароли пользователей на компьютере.
  
## <a name="configure-a-watcher-node-to-use-negotiate"></a>Настройка узла watcher для использования переговоров
<a name="enable_synthetic_trans"> </a>

Если ваш узел просмотра находится за пределами сети периметра, необходимо выполнить несколько другую процедуру, чтобы настроить узел просмотра для запуска синтетических транзакций: в частности, не следует создавать надежный пул приложений или надежное приложение. Это означает, что вам потребуется выполнить следующие две задачи.
  
### <a name="update-membership-in-the-rtc-local-read-only-administrators-group"></a>Обновление членства в группе администраторов Read-Only RTC

Если узел просмотра находится за пределами сети периметра, необходимо добавить учетную запись Сетевой службы в группу администраторов только для чтения RTC на компьютере узла просмотра, завершив следующую процедуру на узле просмотра:
  
1. В меню Пуск щелкните правой кнопкой мыши пункт Компьютер и выберите пункт Управление.
    
2. В диспетчере серверов разверните узел Конфигурация, узел Локальные пользователи и группы и щелкните элемент Группы.
    
3. В области "Группы" дважды щелкните элемент Локальные администраторы RTC с правом только на чтение.
    
4. В диалоговом окне Локальные администраторы с правами только на чтение RTC нажмите кнопку Добавить.
    
5. В диалоговом окне Выбор пользователей, компьютеров, учетных записей служб или групп щелкните Расположения.
    
6. В диалоговом окне Расположения выберите имя компьютера узла-наблюдателя и нажмите кнопку ОК.
    
7. В поле Введите имена выбираемых объектов введите Сетевая служба и нажмите кнопку ОК.
    
8. В диалоговом окне Свойства локальных администраторов RTC с правом только на чтение нажмите кнопку ОК и закройте Диспетчер серверов.
    
9. Перезапустите компьютер узла-наблюдателя.
    
### <a name="install-the-watcher-node-configuration-files"></a>Установка файлов конфигурации узлов watcher

Следующим шагом является запуск файла Watchernode.msi: 
  
1. Откройте microsoft Skype для бизнеса Server 2015. Нажмите кнопку Начните, нажмите кнопку Все программы, щелкните Microsoft Skype для бизнеса Server 2015, а затем нажмите кнопку Skype для бизнеса Server управленческой оболочки. 
    
2. В Skype для бизнеса Server Командная оболочка введите следующую команду и нажмите кнопку ENTER (не забудьте указать фактический путь к копии Watchernode.msi):
    
   ```PowerShell
   c:\Tools\Watchernode.msi Authentication=Negotiate
   ```

> [!NOTE]
> Как упоминалось ранее, Watchernode.msi также можно запускать из окна команды. Чтобы открыть командное окно, нажмите кнопку **Пуск**, щелкните правой кнопкой мыши пункт **Командная строка** и выберите **Запуск от имени администратора**. Когда откроется окно команды, введите ту же команду, показанную на шаге 2 выше. 
  
Режим согласования используется в том случае, если узел-наблюдатель невозможно настроить как пул доверенных приложений. В этом режиме администраторы должны осуществлять управление паролями тестовых пользователей на узле-наблюдателе.
