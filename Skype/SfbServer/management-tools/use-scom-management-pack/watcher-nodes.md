# <a name="install-and-configure-watcher-nodes"></a>Установка и настройка узлов-наблюдателей
 
**Сводка:** Установите и настройте узлы-наблюдатели для искусственных транзакций Skype для бизнеса Server.
  
Узлы-наблюдатели — это компьютеры, периодически выполняющие синтетические транзакции Skype для бизнеса Server. Искусственные транзакции — это командлеты Windows PowerShell, которые проверяют, что ключевые пользовательские сценарии, например возможность входа в систему или обмена мгновенными сообщениями, работают должным образом. Для Skype для бизнеса Server 2015 System Center Operations Manager может выполнять синтетические транзакции, показанные в следующей таблице, в том числе три типа искусственных транзакций:
  
- **По умолчанию** Искусственные транзакции, выполняемые узлом-наблюдателем по умолчанию. При создании нового узла-наблюдателя можно указать, какие искусственные транзакции будет выполняться в узле. (Это назначение параметра Tests, используемого командлетом New-CsWatcherNodeConfiguration). Если вы не используете параметр Tests при создании узла-наблюдателя, он будет автоматически выполнять все синтетические транзакции по умолчанию и не будет выполнять искусственные транзакции, не связанные с по умолчанию. Это означает, например, что узел-наблюдатель будет настроен для запуска теста Test-CsAddressBookService, но не будет настроен для запуска теста Test-CsExumConnectivity.
    
- **Не по умолчанию** Проверка того, что узлы-наблюдатели не запускаются по умолчанию. (Дополнительные сведения см. в описании типа, используемого по умолчанию.) Однако узел-наблюдатель можно включить для запуска любой из искусственных транзакций, не являющихся по умолчанию. Это можно сделать при создании узла-наблюдателя (с помощью командлета New-CsWatcherNodeConfiguration) или в любое время после создания узла-наблюдателя. Обратите внимание на то, что многие из искусственных транзакций по умолчанию требуют дополнительных действий по настройке. Дополнительные сведения об этих действиях приведены в [статье специальные инструкции по настройке для искусственных транзакций](test-users-and-settings.md#special_synthetictrans).
    
- **Расширенные** функции Специальный тип искусственной транзакции, не являющейся по умолчанию. В отличие от других искусственных транзакций расширенные тесты могут выполняться несколько раз в течение каждого прохода. Это полезно при проверке поведения, например при использовании для пула нескольких маршрутов голосовой связи по протоколу PSTN. Это можно сделать, добавив несколько экземпляров расширенного теста на узел-наблюдатель.
    
Сведения о процессе добавления других искусственных транзакций на узел-наблюдатель можно найти [в разделе Настройка узла-наблюдателя для запуска искусственных транзакций](watcher-nodes.md#enable_synthetic_trans). Вы также можете использовать командную консоль Skype для бизнеса Server, чтобы удалить искусственные транзакции из узла-наблюдателя.
  
Узлам-наблюдателям доступны следующие искусственные транзакции.

Нет необходимости устанавливать узлы-наблюдатели для использования System Center Operations Manager. Если вы не установите эти узлы, вы по-прежнему можете получать оповещения в режиме реального времени от компонентов Skype для бизнеса Server 2015 при возникновении проблемы. (Пакет управления компонентом и пользовательским пакетом не использует узлы-наблюдатели.) Однако узлы-наблюдатели необходимы для мониторинга сквозных сценариев с помощью пакета управления Active Monitoring Management Pack.
  
> [!NOTE]
> Кроме того, администраторы могут выполнять искусственные транзакции вручную, не используя или не устанавливая Operations Manager. В зависимости от размера развертывания Skype для бизнеса Server искусственные транзакции могут использовать большой объем памяти компьютера и процессорного времени. Поэтому мы рекомендуем использовать выделенный компьютер в качестве узла-наблюдателя. Например, не следует настраивать сервер переднего плана Skype для бизнеса Server, чтобы он действовал в качестве узла-наблюдателя. Узлы-наблюдатели должны соответствовать тем же базовым требованиям к оборудованию, что и любой другой компьютер в топологии Skype для бизнеса Server. 
  
> [!NOTE]
> Узел-наблюдатель сервера Lync Server 2013 не может быть размещен на том же компьютере, что и узел-наблюдатель Skype для бизнеса Server 2015, так как основные системные файлы для Lync Server 2013 и Skype для бизнеса Server 2015 невозможно установить на один и тот же компьютер. Однако узлы-наблюдатели Skype для бизнеса Server 2015 могут одновременно отслеживать Skype для бизнеса Server 2015 и Lync Server 2013. Искусственные транзакции по умолчанию поддерживаются для обеих версий продукта. 
  
Узлы-наблюдатели Lync Server 2013 могут быть развернуты внутри или за пределами предприятия, чтобы проверить следующее:
  
- Подключение пользователей внутри предприятия к пулам
    
- Подключение через сети периметра для удаленных пользователей, работающих за преработкой на предприятии.
    
- Подключение к устройствам обеспечения связи в филиалах.
    
- Подключение к Lync Server 2013 внутри предприятия и через сети периметра.
    
Чтобы упростить администрирование, различные варианты проверки подлинности доступны для внутреннего и внешнего предприятия. Дополнительные сведения см. [в разделе Настройка узла-наблюдателя для запуска искусственных транзакций](watcher-nodes.md#enable_synthetic_trans).
  
Чтобы настроить компьютер в качестве узла-наблюдателя, необходимо сначала выполнить следующие предварительные требования: 
  
- Установите System Center Operations Manager и импортируйте пакеты управления Skype для бизнеса Server 2015. Сначала необходимо убедиться, что компьютер узла-наблюдателя соответствует всем требованиям для установки Skype для бизнеса Server 2015.
    
- На компьютере узла-наблюдателя установите следующие компоненты:
    
  - Полная версия .NET Framework 4,5
    
  - Windows Identity Foundation
    
  - Windows PowerShell 3.0
    
После выполнения необходимых условий можно настроить узел-наблюдатель, выполнив указанные ниже действия.
  
1. Установите основные файлы Skype для бизнеса Server 2015 на компьютере узла-наблюдателя.
    
2. Установите агент System Center Operations Manager на компьютере узла-наблюдателя.
    
3. Запустите исполняемый файл Watchernode.msi.
    
4. Используйте командлет **New-CsWatcherNodeConfiguration** для настройки тестовых учетных записей пользователей, которые будут использоваться узлом-наблюдателем.
    
## <a name="install-the-skype-for-business-server-2015-core-files-and-the-rtclocal-database"></a>Установка основных файлов Skype для бизнеса Server 2015 и базы данных RTCLocal

Чтобы установить на компьютер основные файлы Skype для бизнеса Server 2015, выполните следующую процедуру. При установке основных файлов будет автоматически установлена база данных RTCLocal. Обратите внимание, что не требуется устанавливать SQL Server на узлах-наблюдателях. SQL Server Express будет автоматически установлен.
  
Установка основных файлов Skype для бизнеса Server 2015 и базы данных RTCLocal:
  
1. На компьютере узла-наблюдателя щелкните кнопку Пуск, последовательно щелкните пункты Все программы, Стандартные, щелкните правой кнопкой мыши значок Командная строка и в контекстном меню выберите пункт Запуск от имени администратора.
    
2. В окне консоли введите следующую команду и нажмите клавишу ВВОД. Обязательно введите соответствующий путь к файлам установки Skype для бизнеса Server: D:\Setup.exe/Бутстраплокалмгмтто убедитесь, что основные компоненты Skype для бизнеса Server успешно установлены, нажмите кнопку **Пуск**, выберите **все программы**, **Skype для бизнеса Server 2015**и щелкните **Командная консоль Skype для бизнеса Server**. В командной консоли Skype для бизнеса Server введите следующую команду Windows PowerShell и нажмите клавишу ВВОД:
  
```PowerShell
Get-CsWatcherNodeConfiguration
```

> При первом запуске этой команды данные не будут возвращены, так как вы еще не настроили узлы узла-наблюдателя. Если команда выполняется без возврата ошибки, можно предположить, что установка Skype для бизнеса Server успешно завершена.

## <a name="install-the-operation-manager-agent-files-on-a-watcher-node"></a>Установка файлов агента Operations Manager на узле-наблюдателе

Аналогично программе установки Skype для бизнеса Server для создания отчетов о компонентах, для узла-наблюдателя Skype для бизнеса Server 2015 требуется установить файлы агента System Center Operations Manager. Это позволяет выполнять искусственные транзакции и сообщать о них оповещения на корневом сервере управления System Center Operations Manager.
  
Чтобы установить файлы агента, выполните действия, описанные в разделе [Настройка компьютеров Skype для бизнеса Server, которые будут отслеживаться](configure-computers-to-monitor.md).
  
## <a name="configure-a-watcher-node-to-run-synthetic-transactions"></a>Настройка узла-наблюдателя для запуска искусственных транзакций
<a name="enable_synthetic_trans"> </a>

После установки файлов агента System Center Operations Manager необходимо настроить сам узел-наблюдатель. Действия, которые необходимо выполнить, зависят от того, находится ли компьютер узла-наблюдателя внутри сети периметра или за пределами сети периметра. 
  
При настройке узла-наблюдателя необходимо также выбрать тип метода проверки подлинности для развертывания этим узлом. Skype для бизнеса Server 2015 позволяет выбрать один из двух способов проверки подлинности: доверенный сервер или проверка подлинности с помощью учетных данных. В следующей таблице показаны различия между этими двумя методами:
  
||**Описание**|**Поддерживаемые расположения**|
|:-----|:-----|:-----|
|TrustedServer  <br/> |Использует сертификат для олицетворения внутреннего сервера и обхода проблем, связанных с проверкой подлинности.  <br/> Полезен администраторам, которые предпочитают управлять одним сертификатом вместо множества паролей пользователей на каждом узле-наблюдателе.  <br/> |На предприятии.  <br/> При использовании этого метода узел-наблюдатель должен находиться в том же домене, что и отслеживаемые пулы. Если узел-наблюдатель и пулы находятся в разных доменах, используйте проверку подлинности учетных данных.  <br/> |
|Подключение  <br/> |Надежно сохраняет имена пользователей и пароли в диспетчере учетных данных Windows на каждом узле-наблюдателе.  <br/> Этот режим требует больше управления паролями, но является единственным вариантом для узлов-наблюдателей за прев Организации. Эти узлы-наблюдатели не могут рассматриваться как доверенные конечные точки при прохождении проверки подлинности.  <br/> |Вне предприятия.  <br/> На предприятии.  <br/> |
   
## <a name="configure-a-watcher-node-to-use-trusted-server-authentication"></a>Настройка узла-наблюдателя для использования проверки подлинности на доверенных серверах
<a name="enable_synthetic_trans"> </a>

Если узел-наблюдатель находится внутри сети периметра, использование проверки подлинности с использованием доверенных серверов может значительно снизить административные задачи, сохраняя один сертификат, а не с помощью многочисленных паролей учетных записей пользователей.
  
Чтобы настроить проверку подлинности на доверенных серверах, необходимо сначала создать доверенный пул приложений для размещения компьютера узла-наблюдателя. После создания пула доверенных приложений необходимо настроить синтетические транзакции на этом узле-наблюдателе для запуска в качестве доверенных приложений.
  
> [!NOTE]
> Доверенное приложение — это приложение, которое получает состояние "доверенное" для работы в рамках Skype для бизнеса Server 2015, но не является встроенной частью продукта. Состояние доверенного приложения означает, что приложение не должно будет проходить проверку подлинности при каждом запуске.
  
Чтобы создать пул доверенных приложений, откройте командную консоль Skype для бизнеса Server и выполните команду, подобную следующей:
  
```PowerShell
New-CsTrustedApplicationPool -Identity atl-watcher-001.litwareinc.com -Registrar atl-cs-001.litwareinc.com -ThrottleAsServer $True -TreatAsAuthenticated $True -OutboundOnly $False -RequiresReplication $True -ComputerFqdn atl-watcher-001.litwareinc.com -Site Redmond
```

> [!NOTE]
> Для получения сведений о параметрах, описанных в предыдущей команде, введите в командной консоли Skype для бизнеса Server следующую команду: 
  
```PowerShell
Get-Help New-CsTrustedApplicationPool -Full | more
```

После создания пула доверенных приложений можно настроить компьютер узла-наблюдателя для запуска искусственных транзакций в качестве надежного приложения с помощью командлета **New-кструстедаппликатион** и команды, аналогичной следующей:
  
```PowerShell
New-CsTrustedApplication -ApplicationId STWatcherNode -TrustedApplicationPoolFqdn atl-watcher-001.litwareinc.com -Port 5061
```

После выполнения этой команды и создания доверенного приложения необходимо выполнить командлет **Enable-CsTopology** , чтобы убедиться в том, что изменения вступают в силу.
  
```PowerShell
Enable-CsTopology
```

Для учетной записи компьютера узла-наблюдателя требуется возможность запрашивать CMS для некоторых искусственных транзакций. Чтобы разрешить эту возможность, добавьте учетную запись компьютера узла-наблюдателя в группу безопасности RTCUniversalReadOnlyAdmins. После завершения репликации AD перезагрузите компьютер.
  
Чтобы убедиться в том, что создано новое доверенное приложение, введите в командной консоли Skype для бизнеса Server следующую команду:
  
```PowerShell
Get-CsTrustedApplication -Identity "atl-watcher-001.litwareinc.com/urn:application:STWatcherNode"
```

## <a name="configure-a-default-certificate-on-the-watcher-node"></a>Настройка сертификата по умолчанию для узла-наблюдателя
<a name="enable_synthetic_trans"> </a>

Для каждого узла-наблюдателя, использующего проверку подлинности TrustedServer, должен быть назначен сертификат по умолчанию с помощью мастера развертывания Skype для бизнеса Server. 
  
Чтобы назначить сертификат по умолчанию:
  
1. В меню Пуск выберите пункт Все программы, щелкните Skype для бизнеса Server 2015 и выберите Мастер развертывания Skype для бизнеса Server. 
    
2. В мастере развертывания Skype для бизнеса Server щелкните установить или обновить систему Skype для бизнеса Server, а затем щелкните выполнить под заголовком запрос, установка или назначение сертификата. 
    
> [!NOTE]
> Если кнопка Выполнить не активна, может понадобиться щелкнуть пункт Выполнить в разделе Установка локального хранилища конфигурации. 
  
Выполните одно из указанных ниже действий.
  
- Если у вас уже есть сертификат, который можно использовать в качестве сертификата по умолчанию, щелкните по умолчанию в мастере сертификатов, а затем нажмите кнопку назначить. Выполните действия, приведенные в мастере назначения сертификатов, чтобы назначить сертификат.
    
- Если вам необходимо запросить сертификат для использования сертификата по умолчанию, щелкните запрос, а затем следуйте указаниям мастера запроса сертификата, чтобы запросить этот сертификат. Если использовать значения по умолчанию для сертификата веб-сервера, будет получен сертификат, который можно назначить в качестве сертификата по умолчанию.
    
## <a name="install-and-configure-a-watcher-node"></a>Установка и настройка узла-наблюдателя
<a name="enable_synthetic_trans"> </a>

После перезапуска компьютера узла-наблюдателя и настройки сертификата необходимо запустить файл Watchernode.msi. (Watchernode.msi необходимо запускать на любом компьютере, на котором установлены файлы агента Operations Manager и основные компоненты Skype для бизнеса Server 2015.) 
  
Установка и настройка узла-наблюдателя:
  
1. Откройте командную консоль Skype для бизнеса Server, нажав кнопку Пуск, выбрав все программы, затем Skype для бизнеса Server 2015 и выбрав пункт Командная консоль Skype для бизнеса Server. 
    
2. В командной консоли введите следующую команду и нажмите клавишу ВВОД (укажите фактический путь к копии Watchernode.msi):
    
```PowerShell
C:\Tools\Watchernode.msi Authentication=TrustedServer
```

> [!NOTE]
> Вы также можете запускать Watchernode.msi n в командном окне. Чтобы открыть командное окно, нажмите кнопку Пуск, щелкните правой кнопкой мыши пункт Командная строка и выберите Запуск от имени администратора. Когда откроется командное окно, введите ту же команду, что и на шаге 2. 
  
> [!IMPORTANT]
> В предыдущей команде для параметра Authentication/value Authentication = TrustedServer учитывается регистр. Его необходимо вводить в точности так, как показано ниже. Например, эта команда завершится с ошибками, так как она не использует правильный регистр букв: 
  
```PowerShell
C:\Tools\Watchernode.msi authentication=trustedserver
```

Режим TrustedServer можно использовать только с компьютерами, находящимися внутри сети периметра. Если узел-наблюдатель работает в режиме TrustedServer, администраторам не нужно поддерживать тестовые пароли пользователей на компьютере.
  
## <a name="configure-a-watcher-node-to-use-negotiate"></a>Настройка узла-наблюдателя для использования согласования
<a name="enable_synthetic_trans"> </a>

Если компьютер узла-наблюдателя находится вне сети периметра, то для настройки этого узла-наблюдателя для запуска искусственных транзакций необходимо выполнить несколько другие действия: в частности, не следует создавать доверенный пул приложений или доверенное приложение. Это означает, что вам потребуется выполнить следующие две задачи.
  
### <a name="update-membership-in-the-rtc-local-read-only-administrators-group"></a>Обновление членства в группе администраторов локальной Read-Only RTC

Если узел-наблюдатель находится за превнешней сетью периметра, необходимо добавить учетную запись Network Service в группу локальных администраторов RTC, доступную только для чтения, на компьютере узла-наблюдателя, выполнив следующую процедуру на узле-наблюдатель:
  
1. В меню Пуск щелкните правой кнопкой мыши пункт Компьютер и выберите пункт Управление.
    
2. В диспетчере серверов разверните узел Конфигурация, узел Локальные пользователи и группы и щелкните элемент Группы.
    
3. В области "Группы" дважды щелкните элемент Локальные администраторы RTC с правом только на чтение.
    
4. В диалоговом окне Локальные администраторы с правами только на чтение RTC нажмите кнопку Добавить.
    
5. В диалоговом окне Выбор пользователей, компьютеров, учетных записей служб или групп щелкните Расположения.
    
6. В диалоговом окне Расположения выберите имя компьютера узла-наблюдателя и нажмите кнопку ОК.
    
7. В поле Введите имена выбираемых объектов введите Сетевая служба и нажмите кнопку ОК.
    
8. В диалоговом окне Свойства локальных администраторов RTC с правом только на чтение нажмите кнопку ОК и закройте Диспетчер серверов.
    
9. Перезапустите компьютер узла-наблюдателя.
    
### <a name="install-the-watcher-node-configuration-files"></a>Установка файлов конфигурации узла-наблюдателя

Следующий шаг — запуск Watchernode.msi файлов: 
  
1. Откройте консоль управления Microsoft Skype для бизнеса Server 2015. Нажмите кнопку Пуск, выберите все программы, Microsoft Skype для бизнеса Server 2015 и щелкните Командная консоль Skype для бизнеса Server. 
    
2. В командной консоли Skype для бизнеса Server введите следующую команду, а затем нажмите клавишу ВВОД (не забудьте указать фактический путь к копии Watchernode.msi):
    
   ```PowerShell
   c:\Tools\Watchernode.msi Authentication=Negotiate
   ```

> [!NOTE]
> Как упоминалось ранее, Watchernode.msi также можно запускать из командного окна. Чтобы открыть командное окно, нажмите кнопку **Пуск**, щелкните правой кнопкой мыши пункт **Командная строка** и выберите **Запуск от имени администратора**. Когда откроется командное окно, введите ту же команду, что и на шаге 2. 
  
Режим согласования используется в том случае, если узел-наблюдатель невозможно настроить как пул доверенных приложений. В этом режиме администраторы должны осуществлять управление паролями тестовых пользователей на узле-наблюдателе.
