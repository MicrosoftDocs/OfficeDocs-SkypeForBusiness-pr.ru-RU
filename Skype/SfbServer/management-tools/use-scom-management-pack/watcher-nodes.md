# <a name="install-and-configure-watcher-nodes"></a>Установка и настройка узлов-watcher
 
**Сводка:** Установка и настройка узлов-watcher для искусственных транзакций Skype для бизнеса Server.
  
Узлы-watcher — это компьютеры, на которых периодически запускаются искусственные транзакции Skype для бизнеса Server. Искусственные транзакции Windows PowerShell, которые проверяют работу ключевых пользовательских сценариев, таких как возможность выполнять вход или обмен мгновенными сообщениями. Для Skype для бизнеса Server 2015 System Center Operations Manager может запускать искусственные транзакции, показанные в следующей таблице, которая включает три типа искусственных транзакций:
  
- **По умолчанию** Искусственные транзакции, которые узел-watcher выполняет по умолчанию. При создании нового узла-watcher можно указать, какие искусственные транзакции этот узел будет запускать. (Это цель параметра Tests, используемого New-CsWatcherNodeConfiguration.) Если при этом не используется параметр Tests, он автоматически будет запускать все искусственные транзакции по умолчанию и не будет запускать какие-либо искусственные транзакции, не заданные по умолчанию. Это означает, например, что узел-Test-CsAddressBookService будет настроен на запуск тестового Test-CsAddressBookService, но не будет настроен для запуска Test-CsExumConnectivity проверки.
    
- **Не по умолчанию** Проверяет, не запускают ли узлы-watcher по умолчанию. (Подробные сведения см. в описании типа Default.) Однако узел-watcher можно включить для запуска любых искусственных транзакций, не включаемых по умолчанию. Это можно сделать при создании узла-watcher (с помощью New-CsWatcherNodeConfiguration-New-CsWatcherNodeConfiguration) или в любое время после создания узла-watcher. Обратите внимание, что для многих искусственных транзакций, не вносяых по умолчанию, требуются дополнительные действия по настройке. Дополнительные сведения об этих действиях см. в специальных [инструкциях по настройке искусственных транзакций.](test-users-and-settings.md#special_synthetictrans)
    
- **Extended** Особый тип искусственных транзакций, не заданная по умолчанию. В отличие от других искусственных транзакций расширенные тесты могут выполняться несколько раз в течение каждого прохода. Это полезно при проверке поведения, например при нескольких маршрутах голосовой связи по телефонной сети общего следования (STN) для пула. Это можно настроить, просто добавив несколько экземпляров расширенного теста в узел-watcher.
    
Дополнительные сведения о добавлении других искусственных транзакций в узел-watcher см. в подстроке "Настройка узла-watcher для запуска [искусственных транзакций".](watcher-nodes.md#enable_synthetic_trans) Вы также можете использовать Skype для бизнеса Server Management Shell для удаления искусственных транзакций из узла-watcher.

Вам не нужно устанавливать узлы-watcher для использования System Center Operations Manager. Если эти узлы не устанавливаются, вы по-прежнему можете получать оповещения в режиме реального времени от компонентов Skype для бизнеса Server 2015 при любой проблеме. (Пакет управления компонентами и пользователями не использует узлы-узлы-watcher.) Однако узлы-наблюдатели необходимы, если вы хотите отслеживать все сценарии с помощью пакета управления активным мониторингом.
  
> [!NOTE]
> Администраторы также могут запускать искусственные транзакции вручную без использования или установки Operations Manager. В зависимости от размера развертывания Skype для бизнеса Server искусственные транзакции могут использовать большой объем памяти компьютера и процессорного времени. Поэтому рекомендуется использовать выделенный компьютер в качестве узла-watcher. Например, не следует настраивать сервер переднего сервера Skype для бизнеса Server в качестве узла-watcher. Узлы-watcher должны соответствовать тем же базовым требованиям к оборудованию, что и любой другой компьютер в топологии Skype для бизнеса Server. 
  
> [!NOTE]
> Устаревший узел-watcher Lync Server 2013 не может быть размещен на том же компьютере, что и узел-watcher Skype для бизнеса Server 2015, так как основные системные файлы для Lync Server 2013 и Skype для бизнеса Server 2015 невозможно установить на одном компьютере. Однако узлы-наблюдатели Skype для бизнеса Server 2015 могут одновременно отслеживать Skype для бизнеса Server 2015 и Lync Server 2013. Искусственные транзакции по умолчанию поддерживаются для обеих версий продуктов. 
  
Узлы-узлы-watcher Lync Server 2013 могут быть развернуты внутри или за пределами предприятия для проверки:
  
- Подключение пользователей внутри предприятия к пулам
    
- Подключение через сети периметра для удаленных пользователей, работающих за пределами предприятия.
    
- Подключение к устройствам обеспечения связи в филиалах.
    
- Подключение к Lync Server 2013 внутри предприятия и через сети периметра.
    
Чтобы упростить администрирование, для предприятий и за ее пределами доступны различные варианты проверки подлинности. Подробные сведения см. в подстроке "Настройка [узла-watcher для запуска искусственных транзакций".](watcher-nodes.md#enable_synthetic_trans)
  
Чтобы настроить компьютер в качестве узла-watcher, необходимо сначала выполнить следующие предварительные условия: 
  
- Установите System Center Operations Manager и импортировать пакеты управления Skype для бизнеса Server 2015. Кроме того, сначала необходимо убедиться, что компьютер узла-watcher соответствует всем предварительным требованиям для установки Skype для бизнеса Server 2015.
    
- Установите следующие элементы на компьютере узла-watcher:
    
  - Полная версия .NET Framework 4.5
    
  - Windows Identity Foundation
    
  - Windows PowerShell 3.0
    
После того как все необходимые условия будут выполнены, можно настроить узел-watcher, вы выполнили следующие действия:
  
1. Установите основные файлы Skype для бизнеса Server 2015 на компьютер узла-watcher.
    
2. Установите агент System Center Operations Manager на компьютер узла-watcher.
    
3. Запустите Watchernode.msi исполняемый файл.
    
4. С помощью **cmdlet New-CsWatcherNodeConfiguration** настройте тестовые учетные записи пользователей, которые будут применяться узлом-watcher.
    
## <a name="install-the-skype-for-business-server-2015-core-files-and-the-rtclocal-database"></a>Установка основных файлов Skype для бизнеса Server 2015 и базы данных RTCLocal

Чтобы установить основные файлы Skype для бизнеса Server 2015 на компьютер, выполните следующую процедуру. База данных RTCLocal будет автоматически установлена при установке основных файлов. Обратите внимание, что не требуется устанавливать SQL Server узлах-watcher. SQL Server Express будет установлен автоматически.
  
Установка основных файлов Skype для бизнеса Server 2015 и базы данных RTCLocal:
  
1. На компьютере узла-наблюдателя щелкните кнопку Пуск, последовательно щелкните пункты Все программы, Стандартные, щелкните правой кнопкой мыши значок Командная строка и в контекстном меню выберите пункт Запуск от имени администратора.
    
2. В окне консоли введите следующую команду и нажмите ввод. Обязательно введите соответствующий путь к файлам установки Skype для бизнеса Server: D:\Setup.exe /BootstrapLocalMgmtTo убедитесь, что основные компоненты Skype для бизнеса Server успешно установлены, нажмите кнопку "Начните", выберите "Все программы", выберите "Skype для бизнеса Server **2015"** и щелкните "Skype для бизнеса Server Management **Shell".** In the Skype for Business Server Management Shell, type the following Windows PowerShell command and press ENTER:
  
```PowerShell
Get-CsWatcherNodeConfiguration
```

> [!NOTE]
> При первом запуске этой команды данные не возвращаются, так как компьютеры узлов-watcher еще не настроены. Если команда выполняется без ошибки, можно предположить, что настройка Skype для бизнеса Server успешно завершена. 
  
Если компьютер узла-watcher расположен в сети периметра, можно выполнить следующую команду, чтобы проверить установку Skype для бизнеса Server 2015:
  
Get-CsPinPolicyYou будет получать сведения, аналогичные этому, в зависимости от количества политик ПИН-кодов, настроенных для использования в организации:

Описание:

Если вы увидите сведения о политиках ПИН-кодов, основные компоненты успешно установлены.
  
## <a name="install-the-operation-manager-agent-files-on-a-watcher-node"></a>Установка файлов агента Operation Manager на узле-watcher

Аналогично настройке Skype для бизнеса Server для оповещения компонентов отчетов узел-watcher Skype для бизнеса Server 2015 требует установки файлов агента System Center Operations Manager. Это позволяет запускать искусственные транзакции и донаправление оповещений корневому серверу управления System Center Operations Manager.
  
Чтобы установить файлы агентов, выполните процедуры, перечисленные в списке "Настройка компьютеров Skype для бизнеса Server", которые [будут отслеживаться.](configure-computers-to-monitor.md)
  
## <a name="configure-a-watcher-node-to-run-synthetic-transactions"></a>Настройка узла-watcher для запуска искусственных транзакций
<a name="enable_synthetic_trans"> </a>

После установки файлов агента System Center Operations Manager необходимо настроить сам узел-watcher. Действия, которые необходимо предпринять для этого, зависят от того, находится ли компьютер узла-watcher внутри сети периметра или вне сети периметра. 
  
При настройке узла-наблюдателя необходимо также выбрать тип метода проверки подлинности для развертывания этим узлом. Skype для бизнеса Server 2015 позволяет выбрать один из двух способов проверки подлинности: доверенный сервер или проверка подлинности учетных данных. В следующей таблице показаны различия между этими двумя методами:
  
||**Описание**|**Поддерживаемые расположения**|
|:-----|:-----|:-----|
|TrustedServer  <br/> |Использует сертификат для олицетворения внутреннего сервера и обхода проблем, связанных с проверкой подлинности.  <br/> Полезно для администраторов, которые предпочитают управлять одним сертификатом, а не множеством паролей пользователей на каждом узле-просмотре.  <br/> |На предприятии.  <br/> При этом методе узел-наблюдатель должен быть в том же домене, что и отслеживаемые пулы. Если узел-watcher и пулы находятся в разных доменах, используйте проверку подлинности учетных данных.  <br/> |
|Согласование  <br/> |Надежно сохраняет имена пользователей и пароли в диспетчере учетных данных Windows на каждом узле-наблюдателе.  <br/> Этот режим требует больше управления паролями, но является единственным вариантом для узлов-watcher за пределами предприятия. Эти узлы-наблюдатели не могут рассматриваться как доверенные конечные точки при прохождении проверки подлинности.  <br/> |Вне предприятия.  <br/> На предприятии.  <br/> |
   
## <a name="configure-a-watcher-node-to-use-trusted-server-authentication"></a>Настройка узла-watcher для использования проверки подлинности доверенного сервера
<a name="enable_synthetic_trans"> </a>

Если компьютер узла-watcher находится внутри сети периметра, использование проверки подлинности доверенного сервера может значительно сократить административные задачи, сохранив один сертификат, вместо использования большого количество паролей учетных записей пользователей.
  
Чтобы настроить проверку подлинности доверенного сервера, необходимо сначала создать пул доверенных приложений для узла-watcher. После создания пула доверенных приложений необходимо настроить искусственные транзакции на этом узле-watcher для запуска в качестве доверенных приложений.
  
> [!NOTE]
> Доверенного приложения это приложение, которое имеет состояние доверия для запуска в skype для бизнеса Server 2015, но не является встроенной частью продукта. Состояние доверенного приложения означает, что приложение не должно будет проходить проверку подлинности при каждом запуске.
  
Чтобы создать пул доверенных приложений, откройте командную оболочку Skype для бизнеса Server и запустите команду, аналогичную этой:
  
```PowerShell
New-CsTrustedApplicationPool -Identity atl-watcher-001.litwareinc.com -Registrar atl-cs-001.litwareinc.com -ThrottleAsServer $True -TreatAsAuthenticated $True -OutboundOnly $False -RequiresReplication $True -ComputerFqdn atl-watcher-001.litwareinc.com -Site Redmond
```

> [!NOTE]
> For details about the parameters in the preceding command, type the following from the Skype for Business Server Management Shell prompt: 
  
```PowerShell
Get-Help New-CsTrustedApplicationPool -Full | more
```

После создания пула доверенных приложений можно настроить компьютер узла-watcher для запуска искусственных транзакций в качестве доверенного приложения с помощью командлета **New-CsTrustedApplication** и команды, похожей на данной:
  
```PowerShell
New-CsTrustedApplication -ApplicationId STWatcherNode -TrustedApplicationPoolFqdn atl-watcher-001.litwareinc.com -Port 5061
```

После выполнения этой команды и создания доверенного приложения необходимо выполнить командлет **Enable-CsTopology,** чтобы убедиться, что изменения вступили в силу:
  
```PowerShell
Enable-CsTopology
```

Учетной записи компьютера узла-watcher требуется возможность запрашивать у CMS некоторые искусственные транзакции. Чтобы разрешить эту возможность, добавьте учетную запись компьютера узла-watcher в группу безопасности RTCUniversalReadOnlyAdmins. После репликации AD перезапустите компьютер.
  
Чтобы убедиться, что новое доверенного приложения создано, введите следующую информацию в запросе в оболочке управления Skype для бизнеса Server:
  
```PowerShell
Get-CsTrustedApplication -Identity "atl-watcher-001.litwareinc.com/urn:application:STWatcherNode"
```

## <a name="configure-a-default-certificate-on-the-watcher-node"></a>Настройка сертификата по умолчанию на узле-watcher
<a name="enable_synthetic_trans"> </a>

Каждому узлу-watcher, использующему проверку подлинности TrustedServer, должен быть назначен сертификат по умолчанию с помощью мастера развертывания Skype для бизнеса Server. 
  
Чтобы назначить сертификат по умолчанию:
  
1. Click Start, click All Programs, click Skype for Business Server 2015, and then click Skype for Business Server Deployment Wizard. 
    
2. В мастере развертывания Skype для бизнеса Server выберите "Установить или обновить систему Skype для бизнеса Server", а затем нажмите кнопку "Выполнить" под заголовком "Запрос", "Установить" или "Назначить сертификат". 
    
> [!NOTE]
> Если кнопка Выполнить не активна, может понадобиться щелкнуть пункт Выполнить в разделе Установка локального хранилища конфигурации. 
  
Выполните одно из указанных ниже действий.
  
- Если у вас уже есть сертификат, который можно использовать в качестве сертификата по умолчанию, выберите "По умолчанию" в мастере сертификатов и нажмите кнопку "Назначить". Выполните действия, приведенные в мастере назначения сертификатов, чтобы назначить сертификат.
    
- Если необходимо запросить сертификат для использования сертификата по умолчанию, щелкните "Запрос", а затем выполните действия в мастере запроса сертификата, чтобы запросить этот сертификат. Если использовать значения по умолчанию для сертификата веб-сервера, будет получен сертификат, который можно назначить в качестве сертификата по умолчанию.
    
## <a name="install-and-configure-a-watcher-node"></a>Установка и настройка узла-watcher
<a name="enable_synthetic_trans"> </a>

После перезапуска компьютера узла-watcher и настройки сертификата необходимо запустить файл Watchernode.msi. (Необходимо запустить Watchernode.msi на любом компьютере, на котором установлены файлы агента Operations Manager и основные компоненты Skype для бизнеса Server 2015.) 
  
Установка и настройка узла-watcher:
  
1. Откройте оболочку управления Skype для бизнеса Server, щелкнув "Начните", щелкните "Все программы", выберите Skype для бизнеса Server 2015, а затем щелкните "Skype для бизнеса Server Management Shell". 
    
2. В командной Watchernode.msi введите следующую команду и нажмите ввод (убедитесь, что указан фактический путь к копии Watchernode.msi):
    
```PowerShell
C:\Tools\Watchernode.msi Authentication=TrustedServer
```

> [!NOTE]
> Вы также можете запустить Watchernode.msi n из командного окна. Чтобы открыть командное окно, нажмите кнопку Пуск, щелкните правой кнопкой мыши пункт Командная строка и выберите Запуск от имени администратора. Когда откроется командное окно, введите ту же команду, которая показана на шаге 2 выше. 
  
> [!IMPORTANT]
> В предыдущей команде для пары "имя-значение" authentication=TrustedServer задается значение с чувствительностью к делу. Его необходимо ввести точно так же, как показано ниже. Например, эта команда не будет работать из-за неправильного использования букв: 
  
```PowerShell
C:\Tools\Watchernode.msi authentication=trustedserver
```

Режим TrustedServer можно использовать только с компьютерами, которые находятся в сети периметра. Когда узел-watcher работает в режиме TrustedServer, администраторам не нужно поддерживать тестовые пароли пользователей на компьютере.
  
## <a name="configure-a-watcher-node-to-use-negotiate"></a>Настройка узла-watcher для использования согласование
<a name="enable_synthetic_trans"> </a>

Если компьютер узла-watcher находится за пределами сети периметра, необходимо выполнить несколько другую процедуру, чтобы настроить этот узел-watcher для запуска искусственных транзакций: в частности, не следует создавать пул доверенных приложений или доверенного приложения. Это означает, что вам потребуется выполнить следующие две задачи.
  
### <a name="update-membership-in-the-rtc-local-read-only-administrators-group"></a>Обновление членства в локальной группе администраторов Read-Only RTC

Если узел-watcher находится за пределами сети периметра, необходимо добавить учетную запись сетевой службы в локализованную группу администраторов RTC только для чтения на компьютере узла-watcher, выполив следующую процедуру на узле-просмотре:
  
1. В меню Пуск щелкните правой кнопкой мыши пункт Компьютер и выберите пункт Управление.
    
2. В диспетчере серверов разверните узел Конфигурация, узел Локальные пользователи и группы и щелкните элемент Группы.
    
3. В области "Группы" дважды щелкните элемент Локальные администраторы RTC с правом только на чтение.
    
4. В диалоговом окне Локальные администраторы с правами только на чтение RTC нажмите кнопку Добавить.
    
5. В диалоговом окне Выбор пользователей, компьютеров, учетных записей служб или групп щелкните Расположения.
    
6. В диалоговом окне Расположения выберите имя компьютера узла-наблюдателя и нажмите кнопку ОК.
    
7. В поле Введите имена выбираемых объектов введите Сетевая служба и нажмите кнопку ОК.
    
8. В диалоговом окне Свойства локальных администраторов RTC с правом только на чтение нажмите кнопку ОК и закройте Диспетчер серверов.
    
9. Перезапустите компьютер узла-наблюдателя.
    
### <a name="install-the-watcher-node-configuration-files"></a>Установка файлов конфигурации узла-watcher

Следующий шаг — запуск файла Watchernode.msi: 
  
1. Откройте microsoft Skype для бизнеса Server 2015 Management Shell. Click Start, click All Programs, click Microsoft Skype for Business Server 2015, and then click Skype for Business Server Management Shell. 
    
2. In Skype for Business Server Management Shell, type the following command, and then press ENTER (be sure to specify the actual path to your copy of Watchernode.msi):
    
   ```PowerShell
   c:\Tools\Watchernode.msi Authentication=Negotiate
   ```

> [!NOTE]
> Как упоминалось ранее, Watchernode.msi также можно запустить из командного окна. Чтобы открыть командное окно, нажмите кнопку **Пуск**, щелкните правой кнопкой мыши пункт **Командная строка** и выберите **Запуск от имени администратора**. Когда откроется командное окно, введите ту же команду, которая показана на шаге 2 выше. 
  
Режим согласования используется в том случае, если узел-наблюдатель невозможно настроить как пул доверенных приложений. В этом режиме администраторы должны осуществлять управление паролями тестовых пользователей на узле-наблюдателе.
