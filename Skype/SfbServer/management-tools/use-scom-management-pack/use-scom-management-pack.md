### <a name="alert-prioritization"></a><span data-ttu-id="526b1-203">Приоритет оповещений</span><span class="sxs-lookup"><span data-stu-id="526b1-203">Alert Prioritization</span></span>

 <span data-ttu-id="526b1-205">**Оповещения с высоким приоритетом:** Эти оповещения указывают на условия, которые приводят к недостаточности обслуживания для больших групп пользователей и требуют немедленного вмешательства.</span><span class="sxs-lookup"><span data-stu-id="526b1-205">**High Priority alerts:** These alerts indicate conditions that cause service outages for large groups of users and require immediate action.</span></span> <span data-ttu-id="526b1-206">Сообщения о нарушениях, обнаруженных искусственными транзакциями и автономными службами (например, в Skype для бизнеса Server аудио-и видеоконференции), определяются как высокоприоритетные уведомления.</span><span class="sxs-lookup"><span data-stu-id="526b1-206">Outages detected by synthetic transactions and offline services (such as Skype for Business Server Audio/Video Conferencing) qualify as High Priority alerts.</span></span> <span data-ttu-id="526b1-207">Однако оповещениям о сбое компонента на отдельном компьютере не назначается высокий приоритет.</span><span class="sxs-lookup"><span data-stu-id="526b1-207">In contrast, a component failure on a single machine is not a High Priority alert.</span></span> <span data-ttu-id="526b1-208">Skype для бизнеса Server 2015 имеет встроенные возможности высокой доступности для таких ситуаций, как, например, несколько серверов переднего плана за пределами подсистем балансировки нагрузки.</span><span class="sxs-lookup"><span data-stu-id="526b1-208">Skype for Business Server 2015 has built-in high-availability features for these situations—for example, multiple Front End Servers behind load balancers.</span></span>

 <span data-ttu-id="526b1-p119">**Прочие оповещения.** Эти оповещения формируются в компонентах, которые могут повлиять на работу конкретного пользователя или нескольких пользователей.
