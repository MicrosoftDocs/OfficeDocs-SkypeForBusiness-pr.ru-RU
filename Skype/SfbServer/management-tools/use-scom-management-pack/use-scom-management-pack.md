# <a name="manage-skype-for-business-server-2015-using-scom-management-pack"></a>Управление Skype для бизнеса Server 2015 с помощью пакета управления SCOM
 
**Сводка:** Сведения о настройке Скайп для инфраструктуры Business Server 2015 для работы с System Center Operations Manager.
  
В идеальном мире будет никогда не возникают проблемы с Скайп для Business Server 2015. Тем не менее, внешние факторы могут повлиять Скайп для Business Server — сбои и сбои оборудования, например сети. С помощью Скайп для пакетов управления 2015 Business Server, можно определить и проактивное решить потенциальных проблем. Таким образом Скайп для пакетов управления 2015 Business Server расширения возможностей System Center Operations Manager.
  
Эти сведения написан на основании 9319.0 версию пакета мониторинга для Скайп для программного обеспечения Business Server 2015.
  
## <a name="configuration-overview"></a>Общие сведения о конфигурации

 Чтобы настроить Скайп для инфраструктуры Business Server 2015 для работы с System Center Operations Manager, необходимо выполнить три действия:
  
Определение и [Настройка основного сервера управления](configure-the-primary.md). Для этого необходимо установить System Center Operations Manager 2012 с пакетом обновления 1 или R2. 
  
 Определение и [Настройка Скайп для компьютеров Business Server, которые будут отслеживаться](configure-computers-to-monitor.md). Для наблюдения за Скайп для сервера с помощью System Center Operations Manager, необходимо установить файлы агента System Center Operations Manager и настроить каждый сервер, который будет выполнять роль прокси-сервер. 
  
 Определение и [Установка и настройка узлов-наблюдателей](watcher-nodes.md). Узлы-наблюдатели являются компьютеры, которые периодически выполнять Скайп для искусственных транзакций Business Server — командлеты Windows PowerShell, убедитесь, что ключ Скайп для компонентов Business Server, например возможность подключения к системе или возможность обмена мгновенными сообщения, работают, как ожидалось.

## <a name="whats-new"></a>Новые возможности

Следующие возможности появились в Скайп для пакетов управления 2015 Business Server.
  
- **Автоматическое обнаружение для вход клиента** Клиентские приложения, которые войти в Скайп для Business Server 2015 часто автоматически обнаружить сервер для входа в систему для. Теперь искусственные транзакции поддерживают проверку правильности настройки автоматического обнаружения.
    
- **Искусственная транзакция настроенный запустите интервалы времени** Чтобы упростить процесс узлах установки, искусственные транзакции могут совместно использовать учетные записи пользователей. Это снижает периодичность проверок, так как во избежание конфликтов проверки выполняются последовательно. По умолчанию искусственные транзакции выполняются каждые 15 минут: этого обычно достаточно для выполнения всех проверок. Администраторы, увеличивающие количество пользователей или уменьшающие количество проверок на каждого пользователя, теперь могут также сократить интервал между проверками.
    
- **Искусственная транзакция видео взаимодействия служб** Пользователи, которые будет перенесен в Скайп для Business Server 2015 от других поставщиков решений часто стремятся для дальнейшего использования устройств видеоконференций (VTCs) этих других поставщиков. Видео взаимодействия сервер — это новый Скайп для роли сервера Business Server 2015, который позволяет пользователям продолжить использовать Cisco VTCs в конференц-залов, подключение к Cisco CUCM через видео канала SIP. Эта функция предусматривает также новую искусственную транзакцию, позволяющую убедиться в том, что сервер видеовзаимодействия работает и в состоянии обрабатывать входящие подключения по магистральной линии видеосвязи SIP.
    
- **Искусственная транзакция конференц-связи с общим доступом к приложениям** Теперь поддерживается проверка конференций с общим доступом к приложениям на всех этапах.

При выполнении этих запросов формируются оповещения, указывающие на такие ситуации во время вызовов и конференций, в которых наиболее вероятно ухудшение качества мультимедийных ресурсов для пользователей. В основе этих данных лежат важнейшие показатели, непосредственно влияющие на качество взаимодействия с пользователем, такие как задержка и потеря пакетов.  <br/> |

Эти оповещения могут указывать на разнообразные ситуации, включая незапущенные службы, высокий уровень отказов, длительную задержку сообщений и неполадки подключения.  <br/> |

Пакет управления обеспечивает отслеживание и сбор данных о важнейших внешних зависимостях, которые могут указывать на серьезные неполадки. ### <a name="alert-prioritization"></a>Приоритет оповещений

Оповещения подразделяются на следующие категории. 
  
 **Оповещений наивысший приоритет:** Эти оповещения указания условий, приводить к сбоям службы для больших групп пользователей и требуют немедленных действий. Перебоев в выполнении обнаруживается искусственные транзакции и автономные службы (например, Скайп Business сервера аудио-и видеоконференций) считаться наивысший приоритет оповещения. Однако оповещениям о сбое компонента на отдельном компьютере не назначается высокий приоритет. Скайп для Business Server 2015 имеет встроенные функции высокой доступности для этих ситуациях восстановление — например, для нескольких серверов переднего плана за подсистемы балансировки нагрузки.
  
 **Оповещений средний приоритет:** Эти оповещения указания условий, которые влияют на части пользователей или указать проблемы в качестве вызовов, например, сбой компонентов, задержка в звонка при установке или снижение качества звука в вызовах. Оповещения в этой категории с ведением базы данных (то есть, характер оповещения изменения на основе состояния подключения к сети.) Например если звонка при установке времени указать задержку, но затем вернуться на обычных пороговое значение, это предупреждение средний приоритет будет автоматическое сопоставление в System Center Operations Manager и администраторы не нужно выполнять действия. Оповещения, причина которых не может быть устранена автоматически, как правило, рассматриваются администраторами в течение рабоего дня.
  
 **Прочие оповещения.** Эти оповещения формируются в компонентах, которые могут повлиять на работу конкретного пользователя или нескольких пользователей. Типовым примером служит оповещение о невозможности синтаксического анализа записи доменных служб Active Directory для пользователя testuser@contoso.com с помощью службы адресной книги. Администраторы работают с такими оповещениями при наличии свободного времени.

|Скайп для пула наблюдателя Business Server  <br/> |Экземпляр искусственных транзакций, выполняемых в отношении одного пула.  <br/> |
|Скайп для регистратора наблюдателя Business Server  <br/> |Экземпляр искусственных транзакций, выполняемых в отношении одного пула регистраторов.  <br/> |
|Скайп для Business Server пользователя службы пула наблюдателя  <br/> |Экземпляр искусственных транзакций, выполняемых в отношении одного пула служб пользователей.  <br/> |
|Скайп для наблюдателя пула Business Server голосовой связи  <br/> |Экземпляр искусственных транзакций, выполняемых в отношении одного пула голосовой связи.  <br/> |

|Наблюдатель за простым URL-адресом  <br/> |Испытание по протоколу HTTPS простых URL-адресов, заданных в развертывании.  <br/> |

Скайп для пула Business Server может содержать несколько отдельных Скайп для систем Business Server (с более одного Скайп для роли, Скайп для службы Business Server и Скайп для компонента Business Server Business Server). Таким образом в случае отказа отдельного сервера или компонент важно менее общее состояние Скайп для пула Business Server, так как других серверов в тот же самый пул можно предоставлять клиенту службы приложения. Работоспособность будет выполнять сведение на уровне процент Скайп для пула Business Server.

## <a name="best-practice-create-a-management-pack-for-customizations"></a>Рекомендация. Создавайте пакеты управления для пользовательских конфигураций.

По умолчанию в Operations Manager сохраняются все пользовательские конфигурации, в том числе переопределения пакета управления по умолчанию. Рекомендуется для каждого запечатанного пакета управления, который требуется настроить, создать отдельный пакет управления. 
  
Когда для запечатанного пакета управления создается пакет управления, глде будут храниться пользовательские параметры, рекомендуется присвоить новому пакету подходящее имя, например, "Наборы пользовательских параметров Skype для бизнеса Server 2015". 
  
Создание нового пакета управления для хранения наборов пользовательских параметров каждого запечатанного пакета управления упрощает экспорт наборов пользовательских параметров из ситуации испытаний в производственную среду. Упрощается также удаление пакета управления, поскольку перед этим необходимо удалить все зависимости. Если наборы пользовательских параметров для всех пакетов управления сохранены в пакете управления по умолчанию, для удаления отдельного пакета управления потребуется сначала удалить пакет управления по умолчанию; при этом будут потеряны изменения, внесенные в ходе пользовательской настройки других пакетов управления.
