# <a name="manage-skype-for-business-server-2015-using-scom-management-pack"></a>Управление Skype для бизнеса Server 2015 г. с помощью пакета управления SCOM
 
**Сводка:** Узнайте, как настроить инфраструктуру Skype для бизнеса Server 2015 г. для работы с System Center Operations Manager.
  
В идеальном мире проблемы с Skype для бизнеса Server 2015 г. никогда не возникнут. Однако на Skype для бизнеса Server могут повлиять внешние факторы, например сбои в сети и сбои оборудования. С помощью Skype для бизнеса Server пакетов управления 2015 года можно упреждающие действия по выявлению и устранению потенциальных проблем. Таким образом, пакеты Skype для бизнеса Server 2015 расширяют возможности System Center operations Manager.
  
Эта информация была написана на основе версии 9319.0 пакета мониторинга для Skype для бизнеса Server 2015 года.
  
## <a name="configuration-overview"></a>Общие сведения о конфигурации

 Чтобы настроить инфраструктуру Skype для бизнеса Server 2015 г. для работы с System Center Operations Manager, необходимо сделать три действия:
  
Определение и [настройка основного сервера управления.](configure-the-primary.md) Для этого необходимо установить System Center manager 2012 SP1 или R2. 
  
 Определение [и настройка Skype для бизнеса Server компьютеров, которые будут отслеживаться.](configure-computers-to-monitor.md) Чтобы отслеживать Skype для бизнеса Server с помощью System Center Operations Manager, необходимо установить файлы агентов System Center Operations Manager и настроить каждый сервер для действий в качестве прокси. 
  
 Определение и [установка и настройка узлов для просмотра.](watcher-nodes.md) Узлы watcher — это компьютеры, которые периодически запускают Skype для бизнеса Server синтетические транзакции Windows PowerShell, которые проверяют, работают ли ключевые компоненты Skype для бизнеса Server, такие как возможность входа в систему или возможность обмена мгновенными сообщениями, как и ожидалось.

## <a name="whats-new"></a>Новые возможности

Следующие функции являются новыми для Skype для бизнеса Server пакетов управления 2015.

- **Изменения в [обновлении за сентябрь 2019 г.](https://www.microsoft.com/en-in/download/details.aspx?id=47364)** В некоторых оповещениях были удалены специальные символы. В некоторых случаях специальные символы мешают функции уведомления командного канала SCOM.

- **Автоматическое открытие для клиентского входного знака** Клиентские приложения, которые во время Skype для бизнеса Server 2015 года часто автоматически обнаруживают сервер для входов. Синтетические транзакции теперь поддерживают проверку правильности автоматического обнаружения.
    
- **Настраиваемые интервалы запуска синтетических транзакций** Чтобы упростить процесс создания узлов watcher, синтетические транзакции могут совместно использовать учетные записи пользователей. Это замедляет частоту, с которой проводятся тесты, так как тесты сериализируются, чтобы избежать конфликтов. По умолчанию синтетические транзакции запускают каждые 15 минут, чтобы убедиться, что все тесты имеют время для запуска. Администраторы, которые предпочитают использовать больше пользователей или меньше тестов на одного пользователя, теперь также могут сократить интервал запуска.
    
- **Синтетическая транзакция Video Interop Services** Клиенты, которые мигрируют Skype для бизнеса Server 2015 г. из других решений поставщиков, часто желают продолжить использование устройств телеконференции видео (VTCs) от этих других поставщиков. Video Interop Server — это новая роль сервера Skype для бизнеса Server 2015 г., которая позволяет клиентам продолжать использовать VTCs Cisco в конференц-залах, подключившись к Cisco CUCM с помощью магистрали SIP видео. Эта функция также добавляет синтетическую транзакцию, чтобы убедиться, что сервер межопераций видео подключен и может обрабатывать входящие подключения через магистраль SIP видео.
    
- **Синтетическая транзакция совместного доступа к приложениям** Теперь поддерживается проверка сценариев для конференций общего доступа к приложениям.

### <a name="alert-prioritization"></a>Оповещение о приоритетах

Оповещения классифицируются по следующим категориям: 
  
 **Оповещений с высоким приоритетом:** Эти оповещения указывают условия, которые вызывают отключения служб для больших групп пользователей и требуют незамедлительных действий. Отключения, обнаруженные синтетическими транзакциями и автономными службами (например, Skype для бизнеса Server аудио- и видеоконференций), квалифицируются как оповещений о высоком приоритете. В отличие от этого, отказ компонентов на одной машине не является оповещением о высоком приоритете. Skype для бизнеса Server 2015 г. для этих ситуаций имеются встроенные функции высокой доступности, например несколько серверов переднего конца за балансителями нагрузки.
  
 **Оповещений о среднем приоритете:** Эти оповещения указывают условия, которые влияют на подмножество пользователей или указывают на проблемы с качеством вызовов, например, сбои компонентов, задержки в создании вызовов или более низкое качество звука в звонках. Оповещений в этой категории является состояние (то есть характер изменений оповещения в зависимости от состояния сетевого подключения.) Например, если время создания вызовов указывает задержку, а затем возвращается к нормальному пороговом значению, это оповещение о среднем приоритете будет автоматически разрешено в System Center и администраторам не потребуется принимать меры. Предупреждения, которые не могут быть разрешены автоматически, обычно решаются администраторами в один и тот же день работы.
  
 **Другие оповещений:** Эти оповещения создаются из компонентов, которые могут повлиять на конкретного пользователя или подмножество пользователей. Например, типичным предупреждением является то, что служба адресной книги не может развести запись Active Directory® Domain Services (AD DS) для пользователя: testuser@contoso.com. Администраторы могут обращаться к этим оповещениям всякий раз, когда у них есть время.

Пул Skype для бизнеса Server может содержать несколько отдельных Skype для бизнеса Server (с несколькими Skype для бизнеса Server ролью, Skype для бизнеса Server службой и Skype для бизнеса Server компонентом). Поэтому отказ отдельного сервера или компонента менее важен для общего Skype для бизнеса Server пула, так как другие серверы в том же пуле могут предоставить клиенту службу приложения. Состояние здоровья будет свернуто на процентном уровне в Skype для бизнеса Server пула. 
  
Наблюдатель Skype для бизнеса Server пула выполняет синтетические транзакции Skype для бизнеса Server пула. Последовательный сбой одной или более синтетических транзакций (процесс, известный как последовательный интервал опроса) будет свернуть критическое состояние здоровья до уровня пула (хуже всех синтетических транзакций), как показано на следующей схеме.

## <a name="best-practice-create-a-management-pack-for-customizations"></a>Лучшая практика: создание пакета управления для настроек

По умолчанию диспетчер операций сохраняет все настройки, например переопределения в пакет управления по умолчанию. В качестве наилучшей практики необходимо создать отдельный пакет управления для каждого закрытого пакета управления, который необходимо настроить. 
  
При создании пакета управления для хранения настраиваемых параметров для закрытого пакета управления рекомендуется правильно назвать новый пакет управления, например "Skype для бизнеса Server 2015". 
  
Создание нового пакета управления для хранения настроек каждого закрытого пакета управления упрощает экспорт настроек из тестовой среды в производственную среду. Это также упрощает удаление пакета управления, так как необходимо удалить все зависимости, прежде чем удалить пакет управления. Если настройки для всех пакетов управления сохраняются в пакете управления по умолчанию и необходимо удалить один пакет управления, сначала необходимо удалить пакет управления по умолчанию, который также удаляет настройки для других пакетов управления.
