# <a name="director-planning-tool"></a>Средство планирования директора
 
Director — это сервер, на Skype для бизнеса Server 2015 г., который может проверить подлинность запросов пользователей, но не имеет учетных записей пользователей. 
  
Эта роль необязательна, вы можете развернуть директора в следующих двух сценариях:
  
- Если вы включаете доступ гостевых пользователей с помощью развертывания edge Servers, необходимо также развернуть director. В этом сценарии Директор сдает проверку подлинности гостям, а затем передает трафик на внутренние серверы. Когда директор используется для проверки подлинности гостей, он освобождает серверы пула передней части от накладных расходов на проверку подлинности этих пользователей. Он также помогает изолировать внутренние пулы переднего конечного входа от вредоносных трафика, таких как атаки с отказом в обслуживании. Если во время такой атаки сеть заполняется неподходящим внешним трафиком, этот трафик останавливается на директоре.
    
- Если развернуть несколько пулов переднего конца на центральном сайте, добавив на этот сайт директора, можно оптимизировать запросы на проверку подлинности и повысить производительность. В этом сценарии все запросы сначала идут к Директору, который затем передает их в правильный пул переднего конца.
