# <a name="edit-the-topology-in-skype-for-business-server-2015"></a>Изменение топологии в Skype для бизнеса Server 2015

После завершения вопросов о начальном собеседовании можно изменить полное доменное имя и IP-адреса сайта. Для этого дважды  щелкните сайт, который нужно изменить, на странице "Глобальная топология".

В средстве планирования отображается топология выбранного сайта. В нижней части страницы сайта находятся четыре вкладки:

![Топология сайта средства планирования](../../media/Planning_Tool_Site_Topology.png)

- Топология сайта — отображаемая в настоящее время страница с визуальным обзором топологии, как рекомендуется.

- Схема сети edge — страница схемы сети по краям, на которой конструктор делает большую часть работы в средстве планирования. На схеме показана конфигурация сети для рекомендуемой топологии Skype для бизнеса Server 2015 с редактируемыми записями для IP-адресов и FQDNs для серверов, пулов, а также аппаратных и DNS-балансиеров нагрузки.

- Отчет об администрировании edge : отчет об администрировании по краям содержит четыре отчета:

     ![Страница отчета об администрировании edge](../../media/Planning_Tool_Summary_Report.png)

  - Сводный отчет — общий отчет о параметрах конфигурации сети edge. Если изменить значения на  странице схемы сети в топологию TCP/IP и значения FQDN, которые будут использоваться в фактическом развертывании, эти адреса и имена будут представлены здесь. В противном случае появится текст по умолчанию.

  - Отчет по сертификатам — в отчете по сертификатам перечисляются имена субъектов и альтернативные имена субъектов для сертификатов, необходимых для топологии.

  - Отчет по брандмауэру — в отчете о брандмауэре перечислены сведения, необходимые для настройки брандмауэров периметра в инфраструктуре. К ним относятся IP-адреса (значения по умолчанию или измененные), роль сервера, исходный IP-адрес и порт, ip-адрес назначения и порт, транспортный протокол, протокол приложения и соответствующие примечания.

  - Отчет DNS . В отчете DNS перечислены важные сведения о записях DNS, которые необходимо создать. Включаются тип записи, FQDN, IP-адрес и комментарии, необходимые для правильной работы.

- Сводка по сайту — в сводке сайта представлен обзор выбора, который вы сделали, ответив на вопросы начального собеседования или заполнив значения в **сайте разработки.** Также представлены сведения о емкости.

    > [!NOTE]
    > Сведения на странице "Сводка сайта" настраиваются для каждого проекта и могут содержать не все разделы или подробные сведения.

## <a name="edit-the-network-configuration-diagram"></a>Изменение схемы конфигурации сети
<a name="Edit_Network_diagram"> </a>

Большая часть работы, которую разработчик делает в средстве планирования Skype для бизнеса Server 2015, состоит в определении записей для IP-адресов и полного доменного имени для записей на сетевой схеме. Информация, введенная на этой странице, передается в отчеты и другие сведения, содержащиеся в средстве планирования.

![Схема сети средства планирования](../../media/Planning_Tool_Network_Diagram.png)

Средство планирования создает сетевую схему с текстом по умолчанию для IP-адресов и FQDNs.

Изменение сетевой схемы и входных значений:

1. Выберите раздел сети для начала работы. Например, дважды щелкните **текст,** access1.contoso.com. В открываемом диалоговом окне введите фактическое FQDN сервера access1.contoso.com и фактический IP-адрес, заменив 131.107.155.3.

2. Нажмите **кнопку** "ОК", чтобы сохранить записи.

3. Продолжайте изменять IP-адреса и FQDNs, предоставляя виртуальные IP-адреса для аппаратных балансировщиков нагрузки или записи серверов для балансировки нагрузки DNS для серверов в пулах.

Полезная функция средства планирования состоит в том, что оно может постепенно назначать диапазон IP-адресов и имен серверов, а не требовать от разработчика изменять каждый отдельный сервер в пуле. Например:

1. Дважды щелкните пульные серверы переднего сервера. Когда откроется диалоговое окно, выберите "Использовать ли вы IPS и FQDN в качестве отправной точки для всех эквивалентных серверов **в этом кластере?".**

2. Например, начальным значением для первого сервера является fe0101.contoso.com IP-адрес 192.168.21.122.

3. Введите fe0.contoso.com в **FQDN** сервера переднего сервера, введите 192.168.21.131 в **IP-адресе** сервера переднего сервера, а затем нажмите кнопку **"ОК".**

4. Функция автоматического приращения обновляет все серверы в пуле до версии fe01–fe06 и все IP-адреса с 192.168.21.131 до 136.

После завершения всех изменений сохраните топологию, выполив следующие действия:

Чтобы сохранить проект средства планирования, щелкните **"Файл"** и выберите "Сохранить топологию" или **"Сохранить топологию как".**  Если **появится диалоговое** окно "Сохранить средство планирования как", введите имя файла в имени файла **и** нажмите кнопку **"Сохранить".**
