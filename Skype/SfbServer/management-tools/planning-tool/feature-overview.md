# <a name="feature-overview-planning-tool"></a>Средство планирования обзор функций
 
Skype для бизнеса Server 2015 г.
  
Вы можете использовать **страницу Центральные** сайты средства планирования для разработки Skype для бизнеса Server развертывания. Можно создать два централизованного или распределенного развертывания. Централизованное развертывание имеет только один центральный сайт, на котором размещены все Skype для бизнеса пользователей в организации. Распределенная развертывание имеет несколько центральных сайтов. Если развертывание Skype для бизнеса Server на нескольких центральных сайтах, вы введите число пользователей на каждом центральном сайте в средстве планирования.
  
Чтобы завершить определение центрального сайта, сначала необходимо предоставить следующую информацию:
  
- **Имя сайта** Введите имя центрального сайта.
    
- **Количество пользователей** Введите число пользователей, в том числе пользователей на сайтах филиалов, которые находятся на центральном сайте.
    
- **Пользователи облачных домашних данных** Введите число пользователей, которые находятся на центральном сайте из Skype для бизнеса Online.
    
> [!NOTE]
> Этот инструмент не будет обновляться Skype для бизнеса Server 2019 г.

## <a name="ui-elements"></a>Элементы пользовательского интерфейса

Остальные элементы заполнены либо ответами на вопросы, представленные в  мастере Начало работы, либо, если вы пропустили мастера, автоматически заполняются средством планирования.
  
### <a name="online-collaboration"></a>Совместная работа в Интернете

 **Online Collaboration** содержит следующие параметры:
  
- **Чат и присутствие**
    
    Мгновенные сообщения (IM) позволяют пользователям общаться друг с другом в режиме реального времени на своих компьютерах с помощью текстовых сообщений. Поддерживаются как двухсторонние, так и многосторонние сеансы обмена мгновенными сообщениями. Presence предоставляет пользователям сведения о состоянии других пользователей в сети. Состояние присутствия пользователя предоставляет сведения, которые помогут другим определить, находится ли он в Сети и как лучше связаться с пользователем. Например, с пользователем, который находится на собрании, лучше всего связаться по электронной почте.
    
- **Аудио- и видеоконференции**
    
    Аудио- и видеоконференции в режиме реального времени позволяют вести аудио- и видеоконференции.
    
- **Конференц-связь с телефонным подключением**
    
    Телефонные переговоры позволяют пользователям присоединяться к A/V с телефона на PSTN. Для телефонных конференций требуется развернуть помощник по конференц-связи и оповещение для конференц-связи службы.
    
- **Веб-конференции**
    
    Веб-конференции позволяют корпоративным пользователям внутри брандмауэра и за его пределами создавать и присоединяться к конференциям в режиме реального времени, которые находятся на внутренних серверах.
    
- **Сохраняемый чат**
    
    Постоянный чат позволяет нескольким пользователям участвовать в беседах, в которых они вывешив и получить доступ к контенту по определенным темам, включая текст, ссылки и файлы. Хотя пользователи в течение сеанса могут взаимодействовать в режиме реального времени, контент каждого сеанса является постоянным, и доступен после окончания сеанса.
    
### <a name="users"></a>Пользователи

 **Пользователи** содержат следующие параметры:
  
- **Внутренняя организация**
    
- **Федерация с другими организациями**
    
- **Федерация с предыдущими версиями**
    
- **Федерация с поставщиками общедоступных услуг im** Позволяет пользователям в организации устанавливать связь с общедоступными поставщиками услуг обмена мгновенными сообщениями, такими как MSN, Yahoo!, aOL. Для создания федерации с общедоступными сетями обмена мгновенными сообщениями требуется отдельная лицензия.
    
- **Федерация с поставщиком услуг на основе XMPP**
    
    Skype для бизнеса Server 2015 г. представляет полностью интегрированный прокси-сервер XMPP (развернутый на edge Servers) и шлюз XMPP, развернутый на передних конечных серверах. Можно развернуть добавление и настройку прокси-сервера XMPP и шлюза XMPP, что позволит пользователям Skype для бизнеса Server 2015 года добавлять контакты от партнеров на основе XMPP для обмена мгновенными сообщениями и присутствия.

> [!NOTE]
> Шлюзы и прокси XMPP доступны в Skype для бизнеса Server 2015 г., но больше не поддерживаются Skype для бизнеса Server 2019 г. Дополнительные сведения см. в дополнительных сведениях о переносе федерации [XMPP.](../../../SfBServer2019/migration/migrating-xmpp-federation.md)
    
- **Мобильность**
    
    При развертывании службы мобильности Skype для бизнеса Server 2015 пользователи могут использовать поддерживаемые мобильные устройства Apple iOS, Android, Windows Phone или Nokia для выполнения таких действий, как отправка и получение мгновенных сообщений, просмотр контактов и просмотр присутствия.
    
- **Почтовый ящик W15 Exchange**
    
    Skype для бизнеса Server 2015 г. позволяет хранить сообщения голосовой почты в Exchange единой системы обмена сообщениями; эти сообщения голосовой почты будут отображаться в качестве сообщений электронной почты в почтовых ящиках пользователей.
    
### <a name="voice"></a>Голос

 **Голос** содержит следующие параметры:
  
- **Корпоративная голосовая связь**
    
    Enterprise — это решение VoIP на программном обеспечении Microsft. Enterprise позволяет пользователям использовать Skype для бизнеса, чтобы разместить телефонный звонок с компьютера.
    
- **Единая система обмена сообщениями Exchange**
    
    Exchange Единая система обмена сообщениями (ЕДИНОЙ СИСТЕМЫ обмена сообщениями) объединяет голосовую почту и электронную почту в единую инфраструктуру обмена сообщениями. Skype для бизнеса Server 2015 г. Exchange для предоставления ответов на вызовы, доступа к абонентам, уведомлений о вызове и автопослужб. Если вы используете эти службы, вам потребуется интегрировать Exchange и Skype для бизнеса Server в топологию active Directory.
    
### <a name="additional-deployment-options"></a>Дополнительные параметры развертывания

 **Дополнительные параметры** развертывания содержат следующие параметры:
  
- **Высокая доступность**
    
    Высокая доступность позволяет автономным серверам поддерживать сбой.
    
- **Аварийное восстановление**
    
    Меры аварийного восстановления позволяют спарить пулы переднего конца, расположенные в двух центрах обработки данных.
    
- **Мониторинг**
    
    Мониторинг записей детализации вызовов, связанных с сеансами связи. Он также собирает метрики из аудио- и видео сеансов в конечных точках участника. Monitoring Server предоставляет статистику использования, тенденции и статистику качества мультимедиа.
    
- **Архивация**
    
    Архивативная система хранит беседы и конференции обмена мгновенными сообщениями.
    
- **Exchange Интеграция архивации**
    
    Если у вас есть пользователи, которые находятся в Exchange 2013 г. и их почтовые ящики были поставлены на In-Place Hold, вы можете выбрать вариант интеграции хранилища Skype для бизнеса Server 2015 г. с Exchange хранилищем.
    
- **IPv4**
    
    Адреса IPv4 — 32-разрядные адреса, которые позволяют компьютеру осуществлять взаимодействие через Интернет. Из-за увеличения числа устройств по всему миру доступные адреса IPv4 вышли из жизни. Из-за этого многие новые устройства переходят на использование адресов IPv6.
    
- **IPv6**
    
    Адреса IPv6 выполняют ту же задачу, что и адреса IPv4 (с некоторыми дополнительными возможностями), но вместо использования 32 бит в адресах IPv6 используется 128 бит. Это не только предоставляет новый набор адресов, но существенно увеличивает их количество.
    
- **Веб-служба обновления устройств**
    
    Веб-служба обновления устройств предоставляет автоматизированный способ обновления всех устройств, например Skype для бизнеса на Windows Phone, развернутых за пределами организации.
    
### <a name="server-applications"></a>Приложения сервера

 **Серверные приложения** содержат следующие параметры:
  
- **Группа ответа**
    
    Приложение Группы реагирования автоматически отвечает и распространяет вызовы доступному агенту helpdesk.
    
- **Объявление**
    
    Если планируется развертывание Корпоративная голосовая связь, может потребоваться настроить обработку телефонных звонков, если набраный номер действителен, но не назначен в общую область пользователя. Администраторы могут настроить службу объявлений таким образом, чтобы эти вызовы перенаправлены в заранее задаваемый пункт назначения (номер телефона или SIP URI) или воспроизведения звукового объявления или обоих. Использование службы объявлений позволяет избежать ситуации, в которой вызываемая ошибка и слышит занятую мелодию, или клиент SIP получает сообщение об ошибке. Функциональность службы объявлений — это типичная функция PBX. 
    
- **Приостановка звонков**
    
    Приложение Call Park позволяет пользователю Корпоративная голосовая связь на удержание с одного телефона, а затем получать вызов с другого телефона, не завяв при этом ресурсы на телефоне, который получил вызов. Приложение Call Park полезно, когда пользователю требуется перенести вызов, но конкретный получатель неизвестен. 
    
- **Сопровождающий конференции**
    
    приложение предоставляет возможности аудиоконференций для телефонных пользователей без обслуживания стороннее поставщика аудиоконференций.
    
- **оповещение для конференц-связи**
    
    приложение создает сигналы, сигнализировав при входе или выходе из конференции, а также уведомления пользователям телефонов при отключении или отключении.
    
- **Контроль допуска звонков**
    
    Управление приемом вызовов (CAC), также известное как управление пропускной способностью WAN, помогает предотвратить плохое качество работы пользователей в перенаправленных сетях, определяя, исходя из доступной пропускной способности, разрешить и установить новые сеансы связи в режиме реального времени. 
    
    > [!NOTE]
    > CAC только управляет трафиком в режиме реального времени и не влияет на трафик данных. 
  
    Если новый сеанс голосовой или видеосвязи превышает ограничения пропускной способности, выделенные на WAN, сеанс либо блокируется, либо (только для телефонных звонков) перенастрояется в PSTN.
