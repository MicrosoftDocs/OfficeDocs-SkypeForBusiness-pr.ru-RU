# <a name="plan-for-statistics-manager-for-skype-for-business-server"></a>Планирование диспетчера статистики в Skype для бизнеса Server

**Сводка:** В этом разделе вы узнаете о диспетчере статистики в Skype для бизнеса Server.

 Диспетчер статистики для Skype для бизнеса Server — это мощное средство, которое позволяет просматривать данные о работоспособности и производительности Skype для бизнеса Server в режиме реального времени. Вы можете опросить данные о производительности на нескольких сотнях серверов каждые несколько секунд и мгновенно просмотреть результаты на веб-сайте диспетчера статистики.

Диспетчер статистики можно использовать для выявления текущих проблем с производительностью, просмотра результатов запланированного изменения среды, отслеживания разрешения с отключений и многое другое. Диспетчер статистики настроен с порогами ключевого индикатора состояния (KHI) и может быть настроен в соответствии с уникальными потребностями развертывания.

Диспетчер статистики можно развернуть в локальном развертывании, где на одном сервере размещаются все серверные компоненты диспетчера статистики. Дополнительные сведения о развертывании диспетчера статистики см. в сведениях о развертывании диспетчера статистики [в Skype для бизнеса Server.](deploy.md) If you already have an existing deployment of Statistics Manager, but you have not yet upgraded to Release 2.0, see [What's new in Release 2.0](plan.md#BKMK_WhatsNew) and [Upgrade Statistics Manager for Skype for Business Server](upgrade.md).

## <a name="features-and-capabilities"></a>Функции и возможности
<a name="BKMK_Features"> </a>

Диспетчер статистики позволяет:

- Просмотр необработанных данных для всех серверов в режиме реального времени. (Данные высмеяются с очень высокой скоростью и отправляются на веб-сайт менее чем за одну секунду.)

- Просмотр данных, агрегированных для определенной роли; например, сервер переднего сервера, сервер-посредник, сервер-посредник и так далее.

- Прокрутка для просмотра данных для определенных сайтов, определенных пулов на сайте, а затем для определенных серверов в пуле.

- Создайте пользовательские диаграммы, чтобы выбранные счетчики по умолчанию были показаны.

- Масштабирование и панорамирование осей x и y или только на оси X.

- Используйте диапазоны дат или точки времени для фильтрации данных.

- Просмотр производительности сервера на основе установленных ключевых индикаторов работоспособности (KHIS). KhIs представляют коллекцию счетчиков производительности с определенным диапазоном работоспособности.

- Просмотр подробных метрик для каждого счетчика.

- Сравните данные для нескольких пользователей или серверов.

- Просмотр отчетов счетчиков с зависанием, чтобы определить агентов, которые не сообщают службе панели мониторинга текущие данные.

- Сохраните определенный экземпляр данных диаграммы в файл.

- Просмотр KHIS в режиме реального времени, включая обновления. Если представление истории включено, будут показаны только новые ошибки.

  - Просмотр всех КХИ за один раз

  - Просмотр КХИ по серверам (альбомная точка зрения)

  - Просмотр определений KHI

## <a name="components"></a>Компоненты
<a name="BKMK_Components"> </a>

Диспетчер статистики состоит из следующих компонентов:

- **Агент.** Облегченный агент, который выполняется на каждом отслеживаемом сервере. Агент позволяет настраивать высокоскоростной опрос счетчиков производительности с помощью локального агрегирования.

- **Прослушиватель.** Серверный API, который получает данные от всех агентов и собирает данные между совокупностями.

- **Hub.** Выступает в качестве клиентского API для системы, работает на веб-серверах и предоставляет обновления данных в режиме реального времени клиентам, подключенным через веб-сайт. (Концентратор автоматически устанавливается как часть MSI веб-сайта.)

- **Веб-сайт.** Пользовательский интерфейс, который объединяет все функции, доступные в системе.

Кроме того, диспетчеру статистики требуется **Redis**, сервер структуры данных с открытым исходным кодом для кэша в памяти. Дополнительные сведения о загрузке Redis см. в диспетчере [развертывания статистики.](deploy.md#BKMK_Deploy)

## <a name="on-premises-deployment"></a>Локальное развертывание
<a name="BKMK_DeploymentOptions"> </a>

В локальном развертывании на одном сервере размещены все серверные компоненты диспетчера статистики.

На следующей схеме показано локальное развертывание, в котором веб-сайт диспетчера статистики, концентратор, прослушиватель и система кэшации Redis размещены на одном компьютере. Диспетчер статистики отслеживает три сервера Skype для бизнеса, каждый из которых имеет один агент, передающий данные прослушивательу. Пользователи подключаются к одному веб-сайту для просмотра всех данных, агрегированных диспетчером статистики:

## <a name="requirements"></a>Требования
<a name="BKMK_Requirements"> </a>

Перед развертыванием диспетчера статистики необходимо учесть следующие требования к программному обеспечению, сети и оборудованию.

### <a name="software-requirements"></a>Требования к программному обеспечению

- Windows Server 2016 и 2019

- IIS (автоматически установлено)

- Redis

- Службы диспетчера статистики (автоматически установленные)

- PSExec — требуется для развертывания удаленного агента

- .NET 4.5 (входит в состав 2012 R2) — требуется для агентов и серверных компонентов
- Скачайте [Skype для бизнеса Server Real-Time statistics Manager (64-битная)](https://www.microsoft.com/en-in/download/details.aspx?id=57518)

|**Сервер размещения**|**Agents**|**Прослушиватель**|

### <a name="hardware-requirements"></a>Требования к оборудованию

В локальном развертывании, где на одном сервере размещены все серверные компоненты диспетчера статистики, сервер с 16 ГБ ОЗУ и 4 ЦП должен поддерживать в среднем около 150 образцов в секунду. Чтобы определить, сколько счетчиков или агентов можно поддерживать, используйте следующее вычисление:

100 серверов 80 счетчиков по 1 выборке в минуту от каждого агента \* \* / 60 секунд = ~ 133 образцов в секунду.

## <a name="security-considerations"></a>Рекомендации по безопасности
<a name="BKMK_Security"> </a>

Весь трафик между серверами шифруется.

- Зашифрованный HTTPS-трафик будет отправлен через порт 8443 (по умолчанию) от агента на сервер прослушивателей.

- Агент проверит отпечаток SSL на сервере, чтобы убедиться, что сервер прослушиватель является ожидаемым получателем. Обратите внимание, что агент использует проверку отпечатка сертификата (вместо проверки цепочки). Он не будет делать полную проверку сертификатов, так как можно использовать самозаверяяние сертификатов.

- После того как агент будет удовлетворены проверкой подлинности прослушиватель, агент получит пароль, который затем проверяется прослушивательом.

- Агент начинает передавать данные о производительности через подключение к прослушивательу.
