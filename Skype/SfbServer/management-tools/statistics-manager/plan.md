# <a name="plan-for-statistics-manager-for-skype-for-business-server"></a>Планирование диспетчера статистики в Skype для бизнеса Server

**Сводка:** Прочтите этот раздел, чтобы узнать о диспетчере статистики для Skype для бизнеса Server.

 Диспетчер статистики в Skype для бизнеса Server — это эффективный инструмент, который позволяет просматривать данные о состоянии и производительности Skype для бизнеса Server в режиме реального времени. Вы можете вести опрос данных о производительности нескольких сотен серверов каждые несколько секунд и мгновенно просматривать результаты на веб-сайте диспетчера статистики.

С помощью диспетчера статистики можно выявлять текущие проблемы с производительностью, просматривать результаты запланированных изменений в среде, устранять разрешения и многое другое. Из этого поля диспетчер статистики настраивается с помощью пороговых значений индикатор работоспособности (КХИ), и его можно настроить в соответствии с уникальными потребностями развертывания.

Вы можете развернуть диспетчер статистики в локальной среде, в которой на одном сервере размещаются все компоненты диспетчера статистики на стороне сервера. Дополнительные сведения о развертывании диспетчера статистики можно найти в разделе [Развертывание диспетчера статистики для Skype для бизнеса Server](deploy.md). Если у вас уже есть развертывание диспетчера статистики, но вы еще не выполнили обновление до выпуска 2,0, ознакомьтесь с [новыми возможностями, выпущенными в Выпуске 2,0](plan.md#BKMK_WhatsNew) и [обновлением диспетчера статистики для Skype для бизнеса Server](upgrade.md).

## <a name="features-and-capabilities"></a>Возможности и возможности
<a name="BKMK_Features"> </a>

Диспетчер статистики позволяет выполнять указанные ниже действия.

- Просмотр необработанных данных для всех серверов в режиме реального времени. (Данные выдаются с очень высокой частотой и отправляются на веб-сайт менее чем за одну секунду).

- Просмотр данных, агрегированных для определенной роли; Например, сервер переднего плана, сервер обновлений, пограничный сервер и т. д.

- Детализация для просмотра данных конкретных сайтов, определенных пулов на сайте и отдельных серверов в пуле.

- Создание настраиваемых диаграмм для отображения выбранных счетчиков по умолчанию.

- Масштабирование и сдвиг на осях x и y либо только на оси x.

- Используйте диапазоны дат или точки времени для фильтрации данных.

- Просмотр производительности сервера на основе установленных индикаторов работоспособности ключа (Кхис). Кхис представляет коллекцию счетчиков производительности с определенным работоспособным диапазоном.

- Просмотр подробных метрик для каждого счетчика.

- Сравнение данных на нескольких Генеральной совокупности или серверах.

- Просматривайте отчеты о скрытых счетчиках, чтобы определить агенты, которые не сообщают текущие данные службе панели мониторинга.

- Сохранение определенного экземпляра данных диаграммы в файле.

- Просматривайте Кхис в режиме реального времени, включая обновления. Если включено представление журнала, отображаются только новые ошибки.

  - Просмотр всех Кхисов за один раз

  - Просмотр Кхис по серверам (альбомный вид)

  - Просмотр определений КХИ

## <a name="components"></a>Компоненты
<a name="BKMK_Components"> </a>

Диспетчер статистики состоит из следующих компонентов:

- **Без.** Упрощенный агент, который выполняется на каждом отслеживаемом сервере. Агент позволяет настроить высокоскоростные опросы счетчиков производительности с помощью локального агрегата.

- **Средство.** Серверный API-интерфейс, который получает данные от всех агентов и объединяет данные за пределы Генеральной совокупности.

- **Ступиц.** Служит клиентским API для системы, выполняется на веб-серверах и предоставляет обновления данных в режиме реального времени для клиентов, подключенных через веб-сайт. (Центр автоматически устанавливается как часть файла MSI веб-сайта.)

- **Веб-сайта.** Пользовательский интерфейс, который собирает вместе все возможности, доступные в системе.

Кроме того, диспетчеру статистики требуется **Redis**— сервер структуры данных с открытым исходным кодом для кэширования в памяти. Дополнительные сведения о том, как скачать Redis, можно найти в разделе [Развертывание диспетчера статистики](deploy.md#BKMK_Deploy) .

## <a name="security-considerations"></a>Вопросы безопасности
<a name="BKMK_Security"> </a>

Весь трафик между серверами шифруется.

- Зашифрованный HTTPS-трафик будет отправляться через порт 8443 (по умолчанию) от агента на сервер прослушивателя.

- Агент проверит отпечаток SSL на сервере, чтобы убедиться, что сервер прослушивателя является ожидаемым получателем. Обратите внимание, что агент использует проверку по отпечатку сертификата (вместо проверки цепочки сертификатов). Он не будет выполнять полную проверку сертификатов, так как возможно использование самозаверяющих сертификатов.

- После того как агент доверит, что прослушиватель является достоверным, пароль будет представлен агентом, который затем проверяется прослушивателем.

- Агент начинает передачу данных о производительности через соединение с прослушивателем.
