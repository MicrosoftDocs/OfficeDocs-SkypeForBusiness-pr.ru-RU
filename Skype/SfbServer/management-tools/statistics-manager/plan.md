# <a name="plan-for-statistics-manager-for-skype-for-business-server"></a>Планирование диспетчера статистики в Skype для бизнеса Server

**Сводка:** Ознакомьтесь с этой темой, чтобы узнать о диспетчере статистики для Skype для бизнеса Server.

 Диспетчер статистики для Skype для бизнеса Server — это мощный инструмент, позволяющий просматривать данные о Skype для бизнеса Server работоспособности и производительности в режиме реального времени. Вы можете просматривать данные о производительности на сотнях серверов каждые несколько секунд и мгновенно просматривать результаты на веб-сайте диспетчера статистики.

Вы можете использовать диспетчер статистики для выявления текущих проблем с производительностью, просмотра результатов запланированного изменения среды, отслеживания отключений и многое другое. Вне окна диспетчер статистики настроен с порогами key Health Indicator (KHI) и может быть настроен в соответствии с уникальными потребностями развертывания.

Диспетчер статистики можно развернуть в локальном развертывании, в котором на одном сервере размещены все компоненты диспетчера статистики на стороне сервера. Дополнительные сведения о развертывании диспетчера статистики см. в этой [Skype для бизнеса Server.](deploy.md) Если у вас уже есть существующее развертывание диспетчера статистики, но вы еще не обновлены до выпуска 2.0, см. новые возможности в выпуске [2.0](plan.md#BKMK_WhatsNew) и Диспетчер статистики обновления для [Skype для бизнеса Server](upgrade.md).

## <a name="features-and-capabilities"></a>Функции и возможности
<a name="BKMK_Features"> </a>

Диспетчер статистики позволяет:

- Просмотр необработанных данных для всех серверов в режиме реального времени. (Данные взяты с очень высокой скоростью и отправляются на веб-сайт менее чем за одну секунду.)

- Просмотр данных, агрегированных для определенной роли; например, front End Server, Mediation Server, Edge Server и так далее.

- Сверлить, чтобы просмотреть данные для определенных сайтов, определенных пулов на сайте, а затем определенных серверов в пуле.

- Создайте настраиваемые диаграммы, чтобы выбранные счетчики показывали по умолчанию.

- Масштабирование и панорамирование как x- и y-axes, так и только на оси x.

- Вовремя используйте диапазоны дат или точки для фильтрации данных.

- Просмотр производительности сервера на основе установленных ключевых показателей работоспособности (KHIS). КХИ представляют коллекцию счетчиков производительности с определенным здоровым диапазоном.

- Просмотр подробных метрик для каждого счетчика.

- Сравните данные между несколькими популяциями или серверами.

- Просмотр ненастояных встречных отчетов, чтобы определить агентов, которые не сообщают текущие данные в службу мониторинга.

- Сохраните определенный экземпляр данных диаграммы в файл.

- Просмотр КХИ в режиме реального времени, включая обновления. Если включено представление истории, показано только новое сбои.

  - Просмотр всех КХИ одновременно

  - Просмотр КХИ по серверу (представление ландшафта)

  - Просмотр определений KHI

## <a name="components"></a>Компоненты
<a name="BKMK_Components"> </a>

Диспетчер статистики состоит из следующих компонентов:

- **Агент.** Легкий агент, который работает на каждом отслеживаемом сервере. Агент позволяет настраивать высокоскоростной опрос счетчиков производительности с помощью локальной агрегации.

- **Слушатель.** API стороне сервера, который получает данные от всех агентов и агрегируется данные между популяциями.

- **Концентратор.** Выполняет функции клиентского API для системы, выполняется на веб-сервере и предоставляет обновления данных в режиме реального времени клиентам, подключенным через веб-сайт. (Концентратор автоматически устанавливается в составе веб-сайта msi.)

- **Веб-сайт.** Пользовательский интерфейс, который объединяет все функции, доступные в системе.

Кроме того, диспетчеру статистики требуется **Redis**— сервер структуры данных с открытым исходным кодом для кэшинга в памяти. Дополнительные сведения о загрузке Redis см. в [ссылке Deploy Statistics Manager.](deploy.md#BKMK_Deploy)

## <a name="on-premises-deployment"></a>Локальное развертывание
<a name="BKMK_DeploymentOptions"> </a>

В локальном развертывании на одном сервере размещены все компоненты диспетчера статистики на стороне сервера.

На следующей схеме показано локальное развертывание, в котором веб-сайт диспетчера статистики, концентратор, прослушиватель и система кэшинга Redis размещены на одной машине. Диспетчер статистики отслеживает три Skype для бизнеса, каждый из которых имеет один агент, передающий данные слушателю. Пользователи подключаются к одному веб-сайту, чтобы просмотреть все данные, агрегированные диспетчером статистики:

## <a name="requirements"></a>Требования
<a name="BKMK_Requirements"> </a>

Перед развертыванием диспетчера статистики необходимо рассмотреть следующие требования к программному обеспечению, сетевым и аппаратным средствам.

### <a name="software-requirements"></a>Требования к программному обеспечению

- Windows Server 2016 2019 г.

- IIS (автоматически установленный)

- Redis

- Службы диспетчера статистики (автоматически установленные)

- PSExec — обязательное развертывание удаленного агента

- .NET 4.5 (включен в 2012 R2) — требуется для агентов и компонентов на стороне сервера
- Скачайте [Skype для бизнеса Server, Real-Time диспетчер статистики (64-bit)](https://www.microsoft.com/en-in/download/details.aspx?id=57518)

|**Сервер хостинга**|**Agents**|**Слушатель**|

|Минимальный гигабитный полный дуплекс сети.  <br/> |Исходящие TCP-порт 8443 (настраиваемый номер порта) для связи с слушателем.  <br/> |Порт listener должен быть одинаковым на всех серверах.  <br/> |

При установке автоматически создаются порты брандмауэра для слушателя и веб-сайта. Для агентов установка предполагает, что исходящие подключения TCP разрешены по умолчанию.

### <a name="hardware-requirements"></a>Требования к оборудованию

В локальном развертывании, в котором на одном сервере размещены все компоненты диспетчера статистики на стороне сервера, сервер с 16 ГБ оперативной памяти и 4 ЦП должен иметь возможность поддерживать в среднем около 150 образцов в секунду. Чтобы определить количество счетчиков/агентов, которые можно поддерживать, используйте следующий расчет:

100 серверов 80 счетчиков 1 образец в минуту от каждого агента \* \* / 60 секунд = ~ 133 образцов в секунду.

## <a name="security-considerations"></a>Рекомендации по безопасности
<a name="BKMK_Security"> </a>

Весь трафик между серверами зашифрован.

- Зашифрованный трафик HTTPS будет отправлен через порт 8443 (по умолчанию) из агента на сервер listener.

- Агент проверит отпечатки SSL на сервере, чтобы убедиться, что сервер прослушиватель является ожидаемым получателем. Обратите внимание, что агент использует проверку отпечатков пальцев сертификата (вместо проверки цепочки). Он не будет делать полную проверку сертификата, так как можно использовать самозаверяемые сертификаты.

- После того как агент будет удовлетворен, слушатель будет подлинным, пароль будет представлен агентом, который затем проверяется слушателем.

- Агент начинает передачу данных о производительности по подключению к прослушивательу.
