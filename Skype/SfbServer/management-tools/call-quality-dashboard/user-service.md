# <a name="user-service-for-cqd"></a>Служба пользователей для CQD
 
**Сводка:** Сведения о службе пользователя, которая является частью API репозитория для панели мониторинга качества звонков. Панель мониторинга качества звонков — это средство для Скайп для Business Server.
  
Служба пользователей является частью API репозитория для панели мониторинга качества звонков.
  
## <a name="user-service"></a>Пользовательская служба

API репозитория предоставляет упрощенный пользовательская модель управления, где подготовки (создание новых учетных записей пользователей) выполняется автоматически и неявные. Когда пользователь для выполнения запроса с API репозитория в первый раз, репозиторий создает новую запись пользователя. 
  
Вызов, панель мониторинга качества автоматически создает пользователя выделенных элементов для нового пользователя. Новые элементы выделенных пользователей, завершения копирует элементы системы пользователя. Таким образом, пользователи начать с их копиями собственные отчеты и запросы, что они могут немедленно начать настройку. 
  
> [!NOTE]
> С помощью панели мониторинга качества звонков, пользователи могут сбросить свои выделенные элементы в любое время. 
  
 **Идентификаторы специального пользователя**
  
API репозитория включает в себя коды URI API REST, требующей целое значение для указания конкретного пользователя. Пример: `https://<portal>/QoERepositoryService/repository/user/{userId}`. Здесь {userId} следует заменить целое значение 0, 1, и т.д.
  
Кроме того API репозитория будет принимать два специальных пользовательских идентификаторы в {userId} в URI.
  
-  *по умолчанию* — представляет пользователя, который в данный момент взаимодействует с использованием интерфейса API. Это позволяет приложениям для доступа к контенту текущего пользователя не позволяет отслеживать их значение идентификатора пользователя. Пример: ` https://<portal>/QoERepositoryService/repository/user/default`.
    
-  *Система* — представляет пользователя в системе. Это позволяет приложениям для доступа к контенту пользователя системы неизвестны значение идентификатора пользователя. Пример: `https://<portal>/QoERepositoryService/repository/user/system`.
    
Если не указано иное, специальные идентификаторы пользователей можно использовать в {userId} в URI.
