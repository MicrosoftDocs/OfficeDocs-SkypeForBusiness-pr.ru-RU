# <a name="user-service-for-cqd"></a>Служба пользователей для CQD
 
**Сводка:** Узнайте о службе пользователей, которая является частью API репозитория для панели мониторинга качества вызовов. Панель мониторинга качества звонков — это инструмент для Skype для бизнеса Server.
  
Служба пользователей является частью API репозитория для панели мониторинга качества вызовов.
  
## <a name="user-service"></a>Пользовательская служба

API репозитория предоставляет упрощенную модель управления пользователями, в которой подготовка пользователей (создание новых учетных записей пользователей) является автоматической и неявной. Когда пользователь впервые запрашивает API репозитория, репозиторий создает новую запись пользователя. 
  
Панель мониторинга качества вызовов также автоматически создает выделенные пользователем элементы для нового пользователя. Новые выделенные пользователем элементы являются полными клонами элементов системного пользователя. Таким образом, пользователи начинают со своих собственных копий отчетов и запросов, которые они могут сразу же приступить к настройке. 
  
> [!NOTE]
> С помощью панели мониторинга качества вызовов пользователи могут в любое время сбросить выделенные элементы. 
  
 **Специальные ИД пользователей**
  
API репозитория включает YPI REST API, которые ожидают, что для указания конкретного пользователя будет указано значение, определенное в качестве значения. Пример:  `https://<portal>/QoERepositoryService/repository/user/{userId}` . Здесь {userId} следует заменить на значение, например 0, 1 и т. д.
  
Кроме того, API репозитория будет принимать два специальных пользовательских ИД в {userId} в URIS.
  
-  *по*  умолчанию представляет пользователя, который в настоящее время взаимодействует с API. Это позволяет приложениям получать доступ к содержимому текущего пользователя, не отслеживая фактическое значение ИД пользователя. Пример: `https://<portal>/QoERepositoryService/repository/user/default`.
    
-  *system*  — представляет системного пользователя. Это позволяет приложениям получать доступ к содержимому системного пользователя, не зная фактического значения пользовательского ИД. Пример: `https://<portal>/QoERepositoryService/repository/user/system`.
    
Если не указано иное, специальные ИД пользователей можно использовать в {userId} в URIS.
