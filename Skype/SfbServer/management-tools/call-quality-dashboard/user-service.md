# <a name="user-service-for-cqd"></a>Служба пользователей для CQD
 
**Сводка:** Узнайте о службе пользователей, которая входит в API репозиториев для панели мониторинга качества вызовов. Панель мониторинга качества вызовов — это средство для Skype для бизнеса Server.
  
Служба пользователей является частью API репозиториев для панели мониторинга качества вызовов.
  
## <a name="user-service"></a>Пользовательская служба

API репозитория предоставляет упрощенную модель управления пользователями, в которой подготовка пользователей (создание новых учетных записей пользователей) является автоматической и неявной. Когда пользователь впервые делает запрос на API репозиториев, репозиторий создает новую запись пользователя. 
  
Панель мониторинга качества вызовов также автоматически создает выделенные пользователем элементы для нового пользователя. Новые элементы, посвященные пользователю, являются полными клонами элементов системного пользователя. Таким образом, пользователи начинают с собственных копий отчетов и запросов, которые они могут немедленно приступить к настройке. 
  
> [!NOTE]
> С помощью панели мониторинга качества вызовов пользователи могут сбросить выделенные элементы в любое время. 
  
 **Специальные пользовательские ID**
  
API репозиториев включает URL-адресы REST API, которые ожидают, что для определенного пользователя будет указано значение в несколько часов. Пример:  `https://<portal>/QoERepositoryService/repository/user/{userId}` . Здесь {userId} следует заменить значением 0, 1 и т. д.
  
Кроме того, API репозитория принимает два специальных пользовательских ID в {userId} в URL-адресе.
  
-  *по*  умолчанию — представляет пользователя, который в настоящее время взаимодействует с API. Это позволяет приложениям получать доступ к содержимому текущего пользователя, не отслеживая фактическое значение пользовательского ID. Пример: `https://<portal>/QoERepositoryService/repository/user/default`.
    
-  *система*  — представляет пользователя системы. Это позволяет приложениям получать доступ к содержимому пользователя системы, не зная фактического значения пользовательского ID. Пример: `https://<portal>/QoERepositoryService/repository/user/system`.
    
Если не указано иное, специальные пользовательские ИД можно использовать в {userId} в URL-адресах.
