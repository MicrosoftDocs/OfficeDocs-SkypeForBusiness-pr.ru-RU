# <a name="user-service-for-cqd"></a>Пользовательская служба для CQD
 
**Сводка:** Узнайте о службе пользователей, которая входит в состав API репозитория для панели мониторинга качества звонков. Панель мониторинга качества звонков — это средство для Skype для бизнеса Server.
  
Служба User входит в API репозитория для панели мониторинга качества звонков.
  
## <a name="user-service"></a>Пользовательская служба

API репозитория предоставляет упрощенную модель управления пользователями, в которой подготовка пользователей (создание новых учетных записей пользователей) выполняется автоматически и неявным образом. Когда пользователь впервые создает запрос для API репозитория, в репозитории создается новая запись пользователя. 
  
Кроме того, панель мониторинга качества звонка автоматически создает выделенные пользователем элементы для нового пользователя. Новые пользовательские элементы — это полные клоны элементов системного пользователя. Таким образом, пользователи запускаются с помощью собственных копий отчетов и запросов, которые можно сразу же приступить к настройке. 
  
> [!NOTE]
> С помощью панели мониторинга качества звонков пользователи могут в любое время сбросить их выделенные элементы. 
  
 **Особые идентификаторы пользователей**
  
API репозитория содержат API-интерфейсы оставшейся части, которые ожидают целочисленное значение для указания определенного пользователя. Пример: `https://<portal>/QoERepositoryService/repository/user/{userId}`. Здесь {userId} следует заменить целым числом, таким как 0, 1 и т. д.
  
Кроме того, API репозитория будут допускать два специальных идентификатора пользователя в {userId} в URI.
  
-  *по умолчанию* — пользователь, который в данный момент взаимодействует с API. Это позволяет приложениям получать доступ к содержимому текущего пользователя без учета значения фактического идентификатора пользователя. Пример: `https://<portal>/QoERepositoryService/repository/user/default`.
    
-  *система* — представляет пользователя системы. Это позволяет приложениям получать доступ к содержимому системного пользователя, не зная фактического идентификатора пользователя. Пример: `https://<portal>/QoERepositoryService/repository/user/system`.
    
Если не указано иное, коды специальных пользователей можно использовать в идентификаторах URI в {userId}.
