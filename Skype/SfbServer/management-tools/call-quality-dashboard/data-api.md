# <a name="data-api-for-call-quality-dashboard-cqd-in-skype-for-business-server"></a>API данных для панели мониторинга качества звонков (CQD) в Skype для бизнеса Server
 
**Сводка:** Сведения об API данных для панели мониторинга качества звонков. Панель мониторинга качества звонков — это средство для Skype для бизнеса Server.
  
API данных обеспечивает программный доступ к панели мониторинга качества звонков для Skype для бизнеса Server.
  
## <a name="data-api-for-call-quality-dashboard"></a>API данных для панели мониторинга качества звонков

API данных предоставляют интерфейс запроса для куба QoE. API данных — это интерфейс API для работы с многомерной базой данных, который обеспечивает агрегированные метрики QoE на основе указанных размеров и фильтров.
  
Операции RESTFUL включены в таблицу ниже.
  

|**Операция**|**Описание**|
|:-----|:-----|
|[Получение куба](get-cube.md) <br/> |Получение списка доступных размеров и размеров.  <br/> |
|[Получение элементов измерений](get-dimension-members.md) <br/> |Функция "получить элементы измерения" возвращает список элементов определенного измерения. Кроме того, она дает возможность отфильтровать список участников и получить подмножество, чтобы снизить стоимость передачи данных в сети.  <br/> |
|[Выполнение запроса](run-query.md) <br/> |Операция выполнения запроса предоставляет возможность выполнения запроса для куба на основе указанных размеров, измерений и фильтров, а также возврата данных.  <br/> |
|[Очистка кэша](clear-cache.md) <br/> |Операция очистки кэша удаляет кэш на сервере для запросов и данных. Это приведет к сбросу кэша, и мы постараемся получать новые данные из куба QoE для новых запросов.  <br/> |
|[Получение журнала интеграции](get-integration-log.md) <br/> |Операция Get Integration log возвращает список записей журнала, описывающих действия в обработке куба QoE.  <br/> |
|[Получение последних данных интеграции](get-last-integration-data.md) <br/> |Получение данных о последней интеграции из куба.  <br/> |
   
 **Поддержка средства общего разделения ресурсов (CORS) для API данных**
  
API данных поддерживает межисточниковый доступ к ресурсам (CORS). CORS — это функция HTTP, которая позволяет веб-приложению, работающему в рамках одного домена, получать доступ к ресурсам другого домена. Веб-браузеры реализуют ограничение безопасности, известное как политика на основе [одной и](https://www.w3.org/Security/wiki/Same_Origin_Policy) той же исходной политики, которая предотвращает вызов API на веб-странице другого домена. CORS обеспечивает безопасный способ вызова API для одного домена (исходного домена) для другого домена. Подробнее об CORS см. в [спецификации CORS](https://www.w3.org/TR/cors/) .
  
 **Включение CORS для API данных**
  
 Ниже приведено фрагмент интерфейса API данных Web. config, в котором показаны два домена, перечисленных в разделе Параметры приложения Корструстедоригин. Все запросы, отправленные сценариями, загруженными с этих серверов, являются надежными через API данных.
  
Не забудьте включить точный протокол, имя узла и порт (если есть). Не следует размещать в конце символа прямой косой черты (/). Несколько записей можно задать, разделяя их запятыми.
  
```
<configuration>
  <appSettings>
    <add key="corsTrustedOrigin" value="https://<trusted-server>,http://<another-trusted-domain>:8080" /> <!-- Domains which are trusted to get the data -->
    <add key="QoEDataLib.DebugMode" value="True" /> <!-- Setting this to True, allows seeing of the detail logs in status page -->
…  </appSettings>
</configuration>
```
