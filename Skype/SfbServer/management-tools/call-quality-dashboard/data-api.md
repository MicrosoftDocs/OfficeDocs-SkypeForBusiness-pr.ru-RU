# <a name="data-api-for-call-quality-dashboard-cqd-in-skype-for-business-server"></a>API данных для панели мониторинга качества звонков (CQD) в Skype для бизнеса Server
 
**Сводка:** Сведения об API данных для панели мониторинга качества звонков. Панель мониторинга качества звонков — это инструмент для Skype для бизнеса Server.
  
API данных предоставляет программный доступ к панели мониторинга качества звонков для Skype для бизнеса Server.
  
## <a name="data-api-for-call-quality-dashboard"></a>API данных для панели мониторинга качества звонков

API данных предлагает интерфейс запроса кубу QoE. API данных — это REST API для работы с многомерной базой данных, которая предоставляет агрегированные метрики качества взаимодействия на основе указанных измерений и фильтров.
  
Операции REST включены в следующую таблицу.
  

|**Операция**|**Описание**|
|:-----|:-----|
|[Получение куба](get-cube.md) <br/> |Получение списка доступных измерений и измерений.  <br/> |
|[Получение элементов измерений](get-dimension-members.md) <br/> |Операция получения элементов измерения возвращает список элементов определенного измерения. Он также дает возможность фильтровать список элементов и получать подмножество, чтобы снизить расходы на передачу по сети.  <br/> |
|[Выполнение запроса](run-query.md) <br/> |Операция выполнения запроса позволяет выполнять запрос к кубу на основе указанных измерений, измерений и фильтров, а также возвращать данные.  <br/> |
|[Очистка кэша](clear-cache.md) <br/> |Операция очистки кэша удаляет кэш на сервере для запросов и данных. Это приведет к сбросу кэша, а затем мы будем получать новые данные из QoE Cube для новых запросов.  <br/> |
|[Получение журнала интеграции](get-integration-log.md) <br/> |Операция получения журнала интеграции возвращает список записей журнала, описывающий действия в обработке кубов качества взаимодействия.  <br/> |
|[Получение последних данных интеграции](get-last-integration-data.md) <br/> |Получение последних данных интеграции из куба.  <br/> |
   
 **Поддержка общего доступа к ресурсам независимо от источника (CORS) для API данных**
  
API данных поддерживает общий доступ к ресурсам независимо от источника (CORS). CORS — это функция HTTP, которая позволяет веб-приложению, работающему в одном домене, получать доступ к ресурсам в другом домене. Веб-браузеры реализуют ограничение безопасности, известное как политика одинакового источника, которая запрещает [веб-странице](https://www.w3.org/Security/wiki/Same_Origin_Policy) вызывать API в другом домене. CORS предоставляет безопасный способ разрешить одному домену (исходному домену) вызывать API в другом домене. Дополнительные [сведения о CORS](https://www.w3.org/TR/cors/) см. в спецификации CORS.
  
 **Включение CORS для API данных**
  
 Ниже приведен фрагмент API данных, web.config двух доменов, перечисленных в параметрах приложения corsTrustedOrigin. Все запросы, выполняемые скриптами, загруженные с этих серверов, являются доверенными через API данных.
  
Не забудьте указать точный протокол, имя узла и порт (при наличии). Не размещайте символ косой черты (/) в конце. Можно указать несколько записей, разделив их запятыми.
  
```xml
<configuration>
  <appSettings>
    <add key="corsTrustedOrigin" value="https://<trusted-server>,http://<another-trusted-domain>:8080" /> <!-- Domains which are trusted to get the data -->
    <add key="QoEDataLib.DebugMode" value="True" /> <!-- Setting this to True, allows seeing of the detail logs in status page -->
...  </appSettings>
</configuration>
```
