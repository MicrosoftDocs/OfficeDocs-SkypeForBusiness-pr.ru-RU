# <a name="data-api-for-call-quality-dashboard-cqd-in-skype-for-business-server"></a>API данных для панели мониторинга качества вызовов (CQD) в Skype для бизнеса Server
 
**Сводка:** Сведения об API данных для панели мониторинга качества вызовов. Панель мониторинга качества вызовов — это средство для Skype для бизнеса Server.
  
API данных предоставляет программный доступ для панели мониторинга качества вызовов для Skype для бизнеса Server.
  
## <a name="data-api-for-call-quality-dashboard"></a>API данных для панели мониторинга качества вызовов

API данных предлагает интерфейс запроса кубу QoE. API данных — это API REST для работы с многомерной базой данных, которая предоставляет агрегированные показатели QoE на основе указанных размеров и фильтров.
  
Операции REST включены в следующую таблицу.
  

|**Операция**|**Описание**|
|:-----|:-----|
|[Получение куба](get-cube.md) <br/> |Получите список доступных измерений и измерений.  <br/> |
|[Получение элементов измерений](get-dimension-members.md) <br/> |Операция "Участники измерения" возвращает список участников определенного измерения. Это также дает возможность фильтровать список участников и получать подмножество, чтобы снизить стоимость переноса провода.  <br/> |
|[Выполнение запроса](run-query.md) <br/> |Операция Run Query предоставляет возможность запуска запроса на кубе на основе заданных размеров, измерений и фильтров и возврата данных.  <br/> |
|[Очистка кэша](clear-cache.md) <br/> |Операция Clear Cache удаляет кэш на сервере для запросов и данных. Это сбросит кэш, после чего мы будем получать свежие данные из QoE Cube для новых запросов.  <br/> |
|[Получение журнала интеграции](get-integration-log.md) <br/> |Операция Журнала интеграции возвращает список записей журналов, описывающих действия в обработке кубов QoE.  <br/> |
|[Получение последних данных интеграции](get-last-integration-data.md) <br/> |Получите последние данные интеграции из куба.  <br/> |
   
 **Поддержка API данных для общего доступа к ресурсам (CORS)**
  
API данных поддерживает совместное использование ресурсов поперек происхождения (CORS). CORS — это функция HTTP, которая позволяет веб-приложению, работающим под одним доменом, получать доступ к ресурсам в другом домене. Веб-браузеры реализуют ограничение безопасности, известное как политика одинакового происхождения, которая не позволяет [веб-странице](https://www.w3.org/Security/wiki/Same_Origin_Policy) вызывать API в другом домене. CORS предоставляет безопасный способ разрешить одному домену (домену происхождения) вызывать API в другом домене. Подробные [сведения о CORS](https://www.w3.org/TR/cors/) см. в спецификации CORS.
  
 **Включение CORS для API данных**
  
 Ниже приводится отрывок из web.config API данных, в котором показаны два домена, перечисленных в параметрах приложений corsTrustedOrigin. Все запросы, сделанные скриптами, загруженными с этих серверов, доверяются API данных.
  
Не забудьте включить точный протокол, имя хозяина и порт (если таковые есть). Не помещай в конце какой-либо перенападающего символа slash (/). Несколько записей можно указать, разделив их запятой.
  
```xml
<configuration>
  <appSettings>
    <add key="corsTrustedOrigin" value="https://<trusted-server>,http://<another-trusted-domain>:8080" /> <!-- Domains which are trusted to get the data -->
    <add key="QoEDataLib.DebugMode" value="True" /> <!-- Setting this to True, allows seeing of the detail logs in status page -->
…  </appSettings>
</configuration>
```
