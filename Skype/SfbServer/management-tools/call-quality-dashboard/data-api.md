# <a name="data-api-for-call-quality-dashboard-cqd-in-skype-for-business-server"></a>API данных для панели мониторинга качества звонков (CQD) в Skype для бизнеса Server
 
**Сводка:** Сведения об API данных для панели мониторинга качества вызовов. Панель мониторинга качества звонков — это инструмент для Skype для бизнеса Server.
  
API данных предоставляет программный доступ к панели мониторинга качества звонков для Skype для бизнеса Server.
  
## <a name="data-api-for-call-quality-dashboard"></a>API данных для панели мониторинга качества вызовов

API данных предоставляет интерфейс запроса кубу QoE. API данных — это REST API для работы с многомерной базой данных, которая предоставляет сводные показатели качества качества связи на основе указанных измерений и фильтров.
  
Операции REST включены в следующую таблицу.
  

|**Операция**|**Описание**|
|:-----|:-----|
|[Получение куба](get-cube.md) <br/> |Получите список доступных измерений и измерений.  <br/> |
|[Получение элементов измерений](get-dimension-members.md) <br/> |Операция "Получить члены измерения" возвращает список членов определенного измерения. Кроме того, это позволяет фильтровать список участников и получать подмножество, чтобы сократить затраты на передачу данных по проводной сети.  <br/> |
|[Выполнение запроса](run-query.md) <br/> |Операция "Выполнить запрос" позволяет выполнять запрос к кубу на основе указанных измерений, измерений и фильтров, а также возвращать данные.  <br/> |
|[Очистка кэша](clear-cache.md) <br/> |Операция очистки кэша удаляет кэш на сервере для запросов и данных. После этого будет сброшен кэш, и мы будем получать новые данные из куба QoE для новых запросов.  <br/> |
|[Получение журнала интеграции](get-integration-log.md) <br/> |Операция "Получить журнал интеграции" возвращает список записей журнала, описывающих действия в обработке куба QoE.  <br/> |
|[Получение последних данных интеграции](get-last-integration-data.md) <br/> |Получите последние данные интеграции из куба.  <br/> |
   
 **Поддержка общего доступа к ресурсам между источниками (CORS) для API данных**
  
API данных поддерживает общий доступ к ресурсам между источниками (CORS). CORS — это функция HTTP, которая позволяет веб-приложению под одним доменом получать доступ к ресурсам в другом домене. Веб-браузеры реализуют ограничение безопасности, известное как политика одинакового источника, которая не позволяет [веб-странице](https://www.w3.org/Security/wiki/Same_Origin_Policy) вызывать API в другом домене. CORS предоставляет безопасный способ разрешить одному домену (домену источника) вызывать API в другом домене. Подробные [сведения о CORS](https://www.w3.org/TR/cors/) см. в спецификации CORS.
  
 **Включение CORS для API данных**
  
 Далее приводится фрагмент списка API данных, web.config двух доменов, перечисленных в параметрах приложения corsTrustedOrigin. API данных доверяет всем запросам, сделанным скриптами, загруженными с этих серверов.
  
Не забудьте включить точный протокол, имя хоста и порт (если таковые есть). Не помещай косую черту (/) в конце. Несколько записей можно уделить запятой.
  
```xml
<configuration>
  <appSettings>
    <add key="corsTrustedOrigin" value="https://<trusted-server>,http://<another-trusted-domain>:8080" /> <!-- Domains which are trusted to get the data -->
    <add key="QoEDataLib.DebugMode" value="True" /> <!-- Setting this to True, allows seeing of the detail logs in status page -->
…  </appSettings>
</configuration>
```
