# <a name="update-item"></a>Обновление элемента
 
**Сводка:** Узнайте об операции обновления элемента, которая входит в состав службы элементов. Служба элементов является частью API репозитория для панели мониторинга качества вызовов. Панель мониторинга качества звонков — это инструмент для Skype для бизнеса Server.
  
Операция "Обновление элемента" является частью службы элементов в API репозитория для панели мониторинга качества вызовов.
  
## <a name="update-item"></a>Обновление элемента

Обновление элемента обновляет определенный элемент в репозитории.
  

|**Способ**|**URI запроса**|**Версия HTTP**|
|:-----|:-----|:-----|
|PUT  <br/> |https:// \<portal\> /QoERepositoryService/repository/item/{itemId}  <br/> |HTTP/1.1  <br/> |
   
 **Параметры URI** — нет.
  
 **Request Headers** -Content-Type: application/json.
  
 **Тело запроса** — JSON.
  
Пример полезной нагрузки запроса:
  
```json
{
  content : "{ 'Product' : 'New Product Name'",
  type: "application/json"
}
```

 *content*  Данные в формате JSON, которые будут храниться в качестве нового содержимого существующего в подчиненного элемента. С технической точки000 г. репозиторий может хранить любое содержимое любой схемы, но при этом он должен быть либо отчетом, либо запросом. *type*  Всегда укажите "application/json" для панели мониторинга качества вызовов.
  
 **Ответ.** Ответ включает код состояния HTTP и набор загодеров отклика.
  
 **Код состояния** : успешная операция возвращает код состояния 204 (без содержимого). Если указанный код элемента не найден, возвращается код состояния 404 (не найден).
  
> [!IMPORTANT]
> Состояние "Нет содержимого" не является состоянием ошибки. Это означает, что ответ не вернул ничего в теле (с другой стороны, 200 OK возвращает содержимое в теле). Это означает, что элемент был успешно обновлен. 
  
 **Response Headers** — None.
  
 **Тело ответа** — нет.
