# <a name="run-query"></a>Запуск запроса

**Сводка:** Сведения об операции Run Query, которая входит в API данных для панели мониторинга качества вызовов. Панель мониторинга качества вызовов — это средство для Skype для бизнеса Server.

Операция Run Query является частью API данных для панели мониторинга качества вызовов.

## <a name="run-query"></a>Запуск запроса

Операция Run Query предоставляет возможность запуска запроса на кубе на основе заданных размеров, измерений и фильтров и возврата данных.


|**Способ**|**Запрос URI**|**ВЕРСИЯ HTTP**|
|:-----|:-----|:-----|
|POST  <br/> |https:// \<portal\> /QoEDataService/RunQuery  <br/> |HTTP/1.1  <br/> |

 **Параметры URI** . Нет.

 **Заглавные заглавные** запросы — дополнительных загонах не будет.

 **Тело запроса** . Вот пример полезной нагрузки запроса в JSON. Он содержит размеры, фильтры и измерения, необходимые для запроса.

```json
{
"Filters": [{
"DataModelName": "[StartDate].[Month]",
"Caption": "July 2013",
"Value": "[2015-03-01T00:00:00]",
"Operand": 0,
"UnionGroup": ""
}],
"Dimensions": [{
"DataModelName": "[StartDate].[Month]"
}],
"Measurements": [{
"DataModelName": "[Measures].[Audio Good Streams Count]"
},
{
"DataModelName": "[Measures].[Audio UnClassified Streams Count]"
},
{
"DataModelName": "[Measures].[Audio Poor Streams Count]"
},
{
"DataModelName": "[Measures].[AudioPoorPercentage]"
}],
"Trend": {
"EnableTrend": true,
"SpanCount": 7,
"TrendDate": "2015-3",
"Type": 0
}
}
```

 *Filters*  — список выражений фильтра, которые необходимо применить таким образом, чтобы полученный набор данных отражал только подмножество интересующих данных.

 *Dimensions*  — список измерений, которые будут использоваться для агрегации данных. Требуется по крайней мере одно измерение, но для получения дополнительного уровня суб-агрегаций могут быть указаны несколько измерений.

 *Измерения*  — список измерений, также известных как факты, которые являются нужными метриками, которые будут агрегироваться в зависимости от указанных измерений.

 *Trend*  — дополнительные инструкции по контролю для настройки данных результатов.

 **Ответ.** Ответ включает код состояния HTTP и набор заглавных заглавных ответов.

 **Код состояния** — успешная операция возвращает код состояния 200 (OK).

 **Заготчики** ответа . Дополнительные заготчики не имеются.

 **Тело ответа** . Ниже приведен пример полезной нагрузки ответа в JSON. Она содержит таблицу данных, которая содержит данные, а также метаданные, которые показывают время выполнения запроса и данные из кэша или нет.

        4.171],
        ["October 2014",
        37017,

        3305,
        8.197],
        ["November 2014",

        6.557],

 *Время выполнения*  — общее время, необходимое серверу для возврата данных. Это может быть кэш, а может и не быть.

 *Результат данных*  — результат запроса. Это двухмерный массив, содержащий все перемывления участников измерений, и каждый элемент, содержащий имена участников измерений, а также агрегированные значения указанных измерений.

 *Результат — От кэша*  — для диагностики. Указывает, пришел ли результат из кэша или из куба QoE.
