# <a name="run-query"></a>Выполнение запроса

**Сводка:** Узнайте о операцию выполнить запрос, который является частью API данных для панели мониторинга качества звонков. Панель мониторинга качества звонков — это средство для Скайп для Business Server.

Операция выполнить запрос является частью API данных для панели мониторинга качества звонков.

## <a name="run-query"></a>Выполнение запроса

Выполнить запрос, операция предоставляет возможность выполнения запроса в кубе, на основе указанного измерения, измерения и фильтры и возвращать данные обратно.


|**Метод**|**URI запроса**|**Версия HTTP**|
|:-----|:-----|:-----|
|Поместить  <br/> |https://\<портала\>/QoEDataService/RunQuery  <br/> |HTTP/1.1  <br/> |

 **Параметры URI** — нет.

 **Заголовки запроса** — без дополнительных заголовков.

 **Текст запроса** — здесь — это полезных данных запроса примера в JSON. Он содержит измерений, фильтры и измерения, необходимые для запроса.

```
{
"Filters": [{
"DataModelName": "[StartDate].[Month]",
"Caption": "July 2013",
"Value": "[2015-03-01T00:00:00]",
"Operand": 0,
"UnionGroup": ""
}],
"Dimensions": [{
"DataModelName": "[StartDate].[Month]"
}],
"Measurements": [{
"DataModelName": "[Measures].[Audio Good Streams Count]"
},
{
"DataModelName": "[Measures].[Audio UnClassified Streams Count]"
},
{
"DataModelName": "[Measures].[Audio Poor Streams Count]"
},
{
"DataModelName": "[Measures].[AudioPoorPercentage]"
}],
"Trend": {
"EnableTrend": true,
"SpanCount": 7,
"TrendDate": "2015-3",
"Type": 0
}
}
```

 *Фильтры* - список выражений фильтра для применения таким образом, результирующий набор данных, будет содержать только подмножества данных, которые могут представлять интерес.

 *Измерения* - список измерений, которые будут использоваться для группирования данных. Требуется по крайней мере одно измерение, но несколько измерений может быть указано для получения дополнительных уровней вложенных статистические выражения.

 *Измерения* — список единиц измерения, также известной как фактов, что желаемую метрики суммировать на основе измерений, указанному.

 *ТЕНДЕНЦИЯ* - дополнительный элемент управления инструкции для настройки данных результатов.

 **Ответа** - ответ включает код состояния HTTP и набор заголовков ответов.

 **Код состояния** - успешные операции возвращает код состояния 200 (ОК).

 **Заголовки ответа** — без дополнительных заголовков.

 **Тело ответа** - ниже приведен пример полезные данные ответа в формате JSON в. Он содержит таблицу данных, который содержит данные, также будет содержать метаданные, который показывает время выполнения запроса и ли данные из кэша.

 *Время выполнения* — общее время, которое потребовалось для возвращения данных с сервера. Это может и не может включать в себя кэша.

 *Данные результатов* - результатов запроса. Это двухмерный массив, содержащий все перестановок элементов измерений и каждый элемент, содержащий имена элементов измерений, а также статистические значения указанного измерения.

 *Результатом является из кэша* - для диагностики. Указывает, будет ли результат поступил из кэша или из куба качества взаимодействия.
