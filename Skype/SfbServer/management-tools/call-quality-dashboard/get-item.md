# <a name="get-item"></a>Получение элемента
 
**Сводка:** Сведения об операции получения элемента, которая входит в состав службы номенклатур. Служба item входит в API репозитория для панели мониторинга качества звонков. Панель мониторинга качества звонков — это средство для Skype для бизнеса Server.
  
Операция Get Item является частью службы элементов в API репозитория для панели мониторинга качества вызова.
  
## <a name="get-item"></a>Получение элемента

Функция Get Item возвращает определенный элемент в репозитории.
  
|**Способов**|**URI запроса**|**Версия HTTP**|
|:-----|:-----|:-----|
|Получить  <br/> |/Коерепоситорисервице/репоситори/итем/{итемид}\<портала\>HTTPS://  <br/> |HTTP/1.1  <br/> |
   
 **Параметры универсального кода ресурса** (None).
  
 **Заголовки запроса** — без дополнительных заголовков.
  
 **Запросить текст** -None.
  
 **Response (ответ** ) — ответ включает код состояния HTTP и набор заголовков ответа.
  
 **Код состояния** — успешная операция возвращает код состояния 200 (ОК). Если указанный идентификатор элемента не найден, возвращается код состояния 404 (не найден).
  
 **Заголовки ответа** — без дополнительных заголовков.
  
 **Тело ответа** — ниже показан пример полезных данных ответа в JSON.
  
```json
{
"itemId": 1652,
"userId": 0,
"content": "{\"Title\":\"All Audio Streams\",...}",
"type": "application/json",
"subItemIds": [1653, 1710]
}
```

 *ItemId* — идентификатор элемента.
  
 *UserID* -идентификатор пользователя, которому принадлежит этот элемент.
  
 *Content* (содержимое) — содержимое, относящееся к приложению.
  
 *Type (тип* ) — тип содержимого. Это поле задается приложениями.
  
 *субитемидс* — идентификаторы дочерних элементов, если таковые имеются. Это короткая схема операции "получить подэлементы" для сохранения звонка. Приложения могут также получать те же данные с помощью операции Get Sub-Items.
