# <a name="plan-for-call-quality-dashboard-for-skype-for-business-server-2015"></a>Планирование для Скайп Business Server 2015 для панели мониторинга качества звонка
 
**Сводка:** Сведения о том, что необходимо учитывать при планировании для панели мониторинга качества звонков.
  
## <a name="overview-of-the-skype-for-business-server-2015-call-quality-dashboard"></a>Общие сведения о Скайп для панели мониторинга качества Business Server 2015 звонка

Скайп для Business Server 2015 вызов качества панели мониторинга (CQD) — это отчетов уровня качества из базы данных взаимодействия на сервере мониторинга в Скайп для Business Server 2015 и Lync Server 2013. CQD использует Microsoft SQL Server Analysis Services для предоставления сбору статистических данных об использовании и качества сведений о вызовах также как для фильтрации и сведение набора данных. Функции CQD:
  
- **Архивация хранилища данных о качестве взаимодействия с помощью компонента качества взаимодействия архив CQD.** Архив качества взаимодействия компонентов можно хранить данные о качестве взаимодействия в течение намного более чем can сервера мониторинга. Это позволяет прибора и предоставления отчетов до 7 месяцев данных за раз, с возможностью слайда окна отчетов дальше back, данные.
    
- **Отчеты и анализ с помощью питания и скорости работы служб аналитики Microsoft SQL Server.** CQD использует службы аналитики Microsoft SQL для обеспечения быстрой Сводка, фильтра и сведение возможности для питания панели мониторинга с помощью куба анализа. Скорость выполнения и возможность детализировать данные отчетов могут сократить время анализа кардинально.
    
- **Оптимизировано для отчетов качества звонков новую схему данных.** Куб имеет схемы, предназначенные для создания отчетов качества голосовой связи и расследований. Пользователи портала можно сосредоточиться на отчетами вместо чтобы определить, как показатели качества взаимодействия базы данных соответствует схеме представления, которые им предстоит. Сочетание архив качества взаимодействия и куб содержит абстракцию, уменьшает сложность отчетов и анализа с помощью CQD. Схема базы данных качества взаимодействия архив также содержит таблицы, которые заполняются с данными, относящиеся к развертыванию для улучшения общей значение данных.
    
- **Конструктор встроенных отчетов и редактирования на месте отчетов.** Компонент портала содержит несколько встроенных отчетов, образцу методика качества звонков. Пользователи портала можно изменить отчеты и создание новых отчетов с помощью функции редактирования портала.
    
- **Веб-API доступа к структуры отчетов и анализа данных куба.** Платформа отчетов мониторинга не является единственным способом для отображения данных из куба. CQD предоставляет несколько примеров использования HTML и JavaScript для извлечения данных из API-интерфейсов Web CQD и отображения данных в пользовательском формате. Редактор отчетов и API-интерфейсов Web CQD комбинацию быстрого создания прототипов отчетов и макет настраиваемого отчета.
    
## <a name="cqd-design-goals"></a>Цели разработки CQD

Панель мониторинга качества звонков (CQD) позволяет ИТ-специалистам использовать сводные данные, чтобы выяснить, на что необходимо обратить внимание в рабочей среде, где есть проблемы с качеством медиаданных. Эта панель позволяет ИТ-специалистам сравнивать статистические данные различных групп пользователей и выявлять тенденции и закономерности. Это средство призвано устранять не отдельные проблемы со звонками, а выявлять проблемы и находить решения, которые будут применяться для большого числа пользователей в этой рабочей среде. 
  
## <a name="call-quality-dashboard-components"></a>Вызов компонентов панели мониторинга качества

Панель мониторинга качества звонков состоит из нескольких баз данных, агента SQL Microsoft заданий, процессов и веб-приложений. Задания агента SQL Microsoft периодически копировать данные из базы данных показатели качества взаимодействия в качества взаимодействия архивной базы данных и процессы куба с данными в базе данных архива качества взаимодействия. База данных репозитория Сохранение определения отчетов, power портала. Портал предоставляет браузера доступ к данным куба. 
  
Компоненты CQD, включая базы данных архива качества взаимодействия, куб и репозиторий можно установлен на сервере мониторинга, установлена на отдельном сервере или установлены на нескольких серверах. Метод установки определенного зависит от требований к его производительности CQD, а также воздействия на другие процессы на тех же серверах. Для получения дополнительных сведений обратитесь к разделу «Компоненты и топологии для CQD» далее в этой статье.

## <a name="information-available-through-cqd"></a>Информация, доступная через CQD

CQD можно отобразить Скайп для Business Server аудио-, видео и совместного использования счетчики потока и количество хороший сравнение неправильное звонки, а также соотношения вызовов, недопустимый для хорошей приложений. Представления можно срез и фильтруется по множество различных измерений. CQD выводит данные из базы данных показатели качества взаимодействия на сервере мониторинга. Данные затем объединяются с любой обеспечивается данных, таких как сети подсети для построения сопоставления делают возможным отчеты, такие как «Вызов качества каждого построения». 
  
CQD также выделяет многие из внутреннего особенности данных о качестве взаимодействия «вызывающего» и «вызываемого» таким образом, что пользователь может быть направлены на создание представления отчета «сервер» и «клиент». После качества методика вызова CQD упрощены для определения условий, группами вызовов плохого общие — один из принципов повышения качества звонка.
  
## <a name="viewing-data-in-cqd"></a>Просмотр данных в CQD

Данные CQD можно просматривать через портал CQD и получить доступ через вызовы API-Интерфейс REST.
  
### <a name="cqd-portal"></a>Портал CQD

Портал является быстрым способом для просмотра данных в кубе. Портал содержит несколько встроенных отчетов, которые могут использоваться сразу. Встроенные отчеты связаны структурированных так, как и руководство пользователя по последовательно меньшего размера и небольших фрагментов данных вызовов. Встроенные отчеты также выделить различные способы данные могут отображаться демонстрируется сочетание диаграмм и таблиц с помощью различных сводные таблицы, фильтры и меры. Каждый пользователь, который осуществляет доступ к порталу может иметь свой собственный набор отчетов, которые он/она может изменить и общий доступ. Дополнительные сведения об использовании веб-портала CQD видеть [Использования вызова качества панель мониторинга для Скайп для Business Server 2015](use.md).
  
Поддерживаемые операционные системы для CQD портала: Windows 8.1, Windows 8, Windows Server 2012 R2 и Windows Server 2012.
  
Поддерживаемые браузеры для портала CQD: Internet Explorer 11, Internet Explorer 10 и Internet Explorer 9.
  
### <a name="rest-apis"></a>API-интерфейсы REST

Куб данных доступен через вызовы API-Интерфейс REST. Данные, извлеченные через API-Интерфейс REST вызовы можно отобразить с помощью HTML-страниц. Пользователи могут воспользоваться преимуществами скорость обработки запросов и высокого уровня схемы CQD во время создания настраиваемых отчетов по-прежнему подходят для их потребностям. Дополнительные сведения по API и примеры, можно [Разрабатывать вызов качества панель мониторинга для Скайп для Business Server 2015](develop.md). 
  
## <a name="defining-your-organizations-requirements-for-cqd"></a>Определение требований организации к CQD

CQD предоставляет качества взаимодействия данных архивации и быстрого и глубокого анализа данные о качестве вызовов. Следующее руководство поможет вам решить, когда и почему бы развертывание CQD.
  
### <a name="when-to-deploy-cqd"></a>Когда следует развернуть CQD

 **CQD могут быть развернуты для установления измерения качества звонков базового, даже в том случае, если организация не возникают проблемы качества звонков.** Установление измерения качества звонков базового важно, поскольку каждый организации есть разным Wi-Fi и проводные и удаленные и сотрудников office. При возникновении проблем вызов качества, последних показателей качества звонков можно по сравнению с предыдущей интервалы времени. CQD's тенденции возможности позволяют легко обнаружения изменений в качестве вызовов по времени.
  
 **CQD могут быть развернуты проактивное поиск проблем, которые могут влиять на качество звонка.** Даже в том случае, если вызов метода среднего качества для организации может провести заданных целевых значений на уровне организации, может быть группами проблемы качества звонков, которые являются скрытыми за средние показатели. По большое число измерений в базе данных QoEMetrics CQD позволяет использовать сводной таблице аналогичные декомпозиции метрик качества звонков. Распознавание выбросов в одноранговые группы — это возможность быстрого проактивное найдите проблемы качества звонков.
  
 **Следует развернуть CQD, если в организации, чтобы сократить время, требуемое для устранения неполадок при есть проблемы качества звонков.** CQD позволяет упростить существующих расследований качество звонка, предложив fast отчетов производительности и динамических детализированным возможности. CQD предназначен для различных типов рабочих процессов в проверка расследований качества звонка для восстановления в среде.
  
### <a name="why-deploy-cqd"></a>Почему целесообразно развертывать CQD

 **Если отчеты о качестве взаимодействия должны быть выполнены для более чем 3 месяца данных следует развернуть CQD.** В базе данных QoEMetrics и отчетов сервера мониторинга предназначены для удержания и сообщить о небольшой набор данных. Оптимизированный базы данных показатели качества взаимодействия для быстрой вставки и поэтому отчетов производительности можно задержек с большой объем звонков или конкурирующие отчетов доступ к базе данных. База данных архива качества взаимодействия CQD предоставляет второй копии данных показатели качества взаимодействия с много больше возможностей хранения. Портал также оптимизирован для отображения до 7 месяцев данных за раз и можно сообщить о всех данных архива качества взаимодействия, при необходимости.
  
 **CQD должны развертываться в случае необходимости настраиваемых отчетов о качестве взаимодействия.** Портал имеет возможность редактор отчетов для отчетов о создании и создания прототипов быстро и легко. Также становится доступные API REST для программного доступа к данным куба, позволяя настраиваемых презентации с помощью HTML/JavaScript или других платформ. Больше не требуется для создания новых запросов SQL при создании настраиваемых представлений данных для создания отчетов.
  
 **Следует развернуть CQD, если существующие качества взаимодействия, функции отчетности не соответствует скорость или глубины, необходимые для организации.** CQD содержит множество встроенных отчетов. Отчеты сразу же полезны и демонстрации как постепенно углубление в данных можно предоставить пользователям дополнительные полезные сведения о каждом уровне. Иерархия отчетов также помогает с управлением множество отчетов логическая структура и стимуляция создания многие другие отчеты, которые можно легко доступными и понятными. CQD не поддерживает только что скорости и гибкость, но также оптимизирован для рабочих процессов, разработанных службой методика качества звонков.

### <a name="single-server-configuration"></a>Конфигурации с одним сервером

Все компоненты CQD и зависимые компоненты могут быть установлены на одном компьютере. Конфигурация отдельном блоке — Это простейшая конфигурация и позволяет CQD быть изолированная. CQD только что потребуется доступ к базе данных показатели качества взаимодействия на сервере мониторинга. Сервер CQD может быть изолированном компьютере виртуальной машины или даже может быть сервер мониторинга, в зависимости от того, доступные ресурсы компьютера узла и требований к производительности. 
  
Во время установки, пользователя для выполнения установки необходимо предоставить Microsoft SQL Server и экземпляры служб аналитики Microsoft SQL Server, которые были ранее настроены на компьютере, где CQD должна быть установлена. Для получения дополнительных сведений обратитесь к [Развертывание вызов качества панель мониторинга для Скайп для Business Server 2015](deploy-0.md) .
  
### <a name="multiserver-configuration"></a>Многосерверное конфигурации

В многосерверной конфигурации, архив качества взаимодействия, куб и портала все можно на разных компьютерах. Существует два основных применения для многосерверной конфигурации.
  
- Размещение веб-портала CQD и CQD куба на разных серверах.
    
- Размещение «разработка» портал отдельно от портала «производственной». 
    
 **Размещение веб-портала CQD и CQD куба на разных компьютерах.** Организациям, который может иметь требования для разделения портала CQD от установки SQL Server или, которая может возникнуть необходимость объединения и соответствие выпусков SQL Server для экземпляра SQL Server и экземпляра служб аналитики SQL Server можно установить на портале CQD и Куб CQD на разных компьютерах. Компонент архив качества взаимодействия также может быть единственным CQD компонента, который устанавливается, если организации просто хочет устойчивой метод Архивация данных о качестве взаимодействия не превысить ограничения производительности на сервере мониторинга.
  
![Один сервер CQD](../../media/f65be6f3-6bba-4c3d-b3ae-c05e03551b5b.png)
  
 **Размещение «разработка» портал отдельно от портала «производственной».** Организации, создающие собственные настраиваемые отчеты (с помощью API-интерфейсы REST) могут предпочитают развернуть дополнительные экземпляры портала (CQD) вместе с рабочей портала, обычных пользователей доступ для мониторинга качества звонков и расследований. Разработка портала можно изолировать каких-либо изменений в портал в рабочей среде. Дополнительные веб-порталов можно развернуть на разных компьютерах (как показано ниже) или развернута в различных веб-каталогов на том же компьютере (не показано). Чтобы выполнить последний, дополнительные веб-портала CQD нужно скопировать рабочем компьютере вручную так как процесс установки CQD всегда развертывает веб-портала CQD веб-сайт по умолчанию с помощью имена предопределенных веб-приложений.

### <a name="supported-topologies"></a>Поддерживаемые топологии

Ниже перечислены минимальные требования к оборудованию и программному обеспечению. Доступность данных и запросов производительности может меняться в зависимости от минут часов, в зависимости от числа активных Скайп для пользователей Business Server и оборудования и конфигурации, поэтому некоторые измерение производительности, приведенные ниже.

CQD использует Microsoft SQL Server, Microsoft SQL Server Analysis Services и служб Microsoft IIS, поэтому его CQD минимальные аппаратные и требования к программному обеспечению по сути то же самое, что эти зависимые компоненты. Тем не менее основанных на требованиях организации вокруг актуальность данных (к которым будет зависеть от частично корпоративных данных о качестве взаимодействия, которые создает организации) и затрат на развертывание, рекомендации по развертыванию дополнительные делаться.
  
Обработка данных в CQD делится на два основных этапа: 
  
- Процесс архив качества взаимодействия
    
- Обработка CQD куба
    
 **Обработка архив качества взаимодействия.** Задача обработки архива качества взаимодействия копирует данные из базы данных показатели качества взаимодействия на сервере мониторинга качества взаимодействия архивной базы данных. Существует две ситуации, где время обработки задачи бы характеристики производительности существенно отличается. Во-первых, после первоначальной установки CQD. После выполнения задания в первый раз после новой установки задачи обработки архива качества взаимодействия копироваться все данные, которые в базе данных показатели качества взаимодействия в базу данных о качестве взаимодействия архива. Второй — периодической обработки после этой первоначальной round. Архив качества взаимодействия, обработки задач можно было запускать каждые 15 минут и обработки новых записей качества взаимодействия, которые находятся в базе данных показатели качества взаимодействия. Как правило начальное время обработки не так, как он выполняется только в первый раз, когда устанавливается CQD. Тем не менее если сервер CQD важности в разделе предоставляемые, этой задачи может занять несколько часов. Обратитесь к таблице ниже например начальной QoE архив обработки раз.
  
 **Обработка CQD куба.** Задача обработки куба собираются из базы данных качества взаимодействия архива в куб. Время обработки исходного куба и время обработки последующих куба определяются выпуск служб аналитики SQL Server, используемое для куба CQD. При использовании выпуска Standard edition не отличается от времени обработки исходного куба и последующих куба, время обработки, так как каждый раз при обновлении данных куба, всегда будет полной обработки всех доступных данных. (То есть, время обработки куба с увеличением объема данных в архив качества взаимодействия, увеличивается базы данных.) Поскольку Business Intelligence версии и выпуска SQL Server Enterprise раздел поддержки, если используется любой выпуск, только начальной выполнения будет обрабатывать все данные в базе данных архива качества взаимодействия. В последующих запусках при запуске задачу каждые 15 минут задачи только обрабатывать новые записи, добавляемые к базе данных архива качества взаимодействия со времени последнего запуска задания. Один раз в день, также будет полной обработки в разделе, который содержит данные текущего месяца.
  
Характеристики физического компьютера может повлиять на производительность CQD, а также функции программного обеспечения, которые доступны из компонентов SQL Server. Компонент качества взаимодействия архив будет отнять большое количество более дисков по сравнению с другими компонентами в то время как компонент куба будет дополнительные ЦП и памяти связано со значительными затратами. Все эти факторы вносить свой вклад в CQD общее обработки данных время непосредственно влияет на доступность и актуальность данных. Организаций должны принимать решения на оборудование и программное обеспечение, основанное на отдельных потребностям организации.

- Один, которые уже имеют участника безопасности для входа для базы данных показатели качества взаимодействия (с правами db_datareader) и участника безопасности для входа в качества взаимодействия архив экземпляр сервера SQL Server (необходимо для создания объекта связанного сервера во время установки). Эта учетная запись будет использоваться для запуска «Данных о качестве взаимодействия архива» шаг задания агента SQL Server.
    
- Один, который будет использоваться для запуска «Процесс куба» шаг задания агента SQL Server. Создание имени входа участника безопасности к базе данных архива качества взаимодействия (с чтения и записи прав) и также создать элемент в роль качества взаимодействия (с правами полного доступа) для выбранного куба.
    
- Один, который будет использоваться для запуска рабочего процесса IIS для веб-порталов и веб-интерфейсы API. Создание имени входа участника безопасности к базе данных архива качества взаимодействия (с привилегиями на чтение), участника безопасности для входа в базу данных репозитория (с чтения и записи прав) и членство в QoERole (с правами полного доступа) для выбранного куба. 
    
    > [!NOTE]
    > При QoE архивной базы данных и базы данных репозитория размещаются в одном SQL Server, создается только один участник с двумя сопоставления пользователей безопасности входа в систему. 
  
Первые две учетные записи логически можно рассматривать как «учетные записи служб серверного компонента» и последнего учетная запись является «переднего плана учетной записи службы». Не рекомендуется, его можно использовать одну учетную запись во всех случаях.

## <a name="capacity-planning"></a>Планирование мощности
<a name="Infrastructure_Req"> </a>

CQD предназначен для минимальное влияние на QoEMetrics: код был оптимизирован для не блокируют данные и задания импорта настраиваемых.
  
Тип оборудования для использования зависит от требований для скорость должна запускаться синхронизации. Изменения размера диска выглядит следующим образом:
  
- QoEArchive — это больше, чем QoEMetrics DB ~1.5x изначально
    
- Куб SSIS сжатие данных почти 10 x по сравнению с DB
    
- Данные секционируются ежемесячно. можно ли удалить разделы
