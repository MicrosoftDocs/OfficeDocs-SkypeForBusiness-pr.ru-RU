# <a name="repository-api-for-call-quality-dashboard-cqd-in-skype-for-business-server"></a>API репозиториев для панели мониторинга качества вызовов (CQD) в Skype для бизнеса Server
 
**Сводка:** Узнайте об API репозитория для панели мониторинга качества вызовов. Панель мониторинга качества вызовов — это средство для Skype для бизнеса Server.
  
API репозитория предоставляет программный доступ для панели мониторинга качества вызовов для Skype для бизнеса Server.
  
## <a name="repository-api-for-call-quality-dashboard"></a>API репозиториев для панели мониторинга качества вызовов

API репозитория предлагает интерфейс доступа к данным в базе данных репозиториев. Репозиторий позволяет организовать содержимое в структуре дерева или графа таким образом, чтобы пользователи могли группировать их так, как это имеет смысл для пользователей. Репозиторий поддерживает два общих типа пользователей: пользователя системы, который является встроенным пользователем, представляющим репозиторий, и обычных пользователей, представляющих авторизованного пользователя репозитория.
  
API репозитория состоит из трех общих служб: 
  
- [Служба пользователей для CQD](user-service.md) — для доступа к пользователям.
    
- [Служба элементов для панели мониторинга качества вызовов (CQD)](item-service.md) — для доступа к элементам и контенту, хранямуся в Items.
    
- [Служба Параметры для мониторинга качества вызовов (CQD)](user-settings-service.md) для доступа к пользовательским Параметры.
    
Панель мониторинга качества вызовов использует API репозиториев для управления следующей информацией: 
  
- **Пользователь** — представление пользователей, которые имеют доступ к репозиторию.
    
- **Отчет** — содержит список запросов, хранимый в качестве контента в пунктах репозиториев.
    
- **Запрос** — используется для получения данных из API данных, хранимые в качестве контента в репозиториях.
    
- **Параметр пользователя** — описывает необязательное поведение приложения для пользователя.
    
  **Поддержка API репозитория для общего доступа к ресурсам (CORS)**
  
API репозиториев поддерживает совместное использование ресурсов поперек происхождения (CORS). CORS — это функция HTTP, которая позволяет веб-приложению, работающим под одним доменом, получать доступ к ресурсам в другом домене. Веб-браузеры реализуют ограничение безопасности, известное как политика одинакового происхождения, которая не позволяет [веб-странице](https://www.w3.org/Security/wiki/Same_Origin_Policy) вызывать API в другом домене. CORS предоставляет безопасный способ разрешить одному домену (домену происхождения) вызывать API в другом домене. Подробные [сведения о CORS](https://www.w3.org/TR/cors/) см. в спецификации CORS.
  
 **Включение API CORS для репозиториев**
  
 Ниже приводится выдержка из API репозиториев web.config двух доменов, перечисленных в параметрах приложений corsTrustedOrigin. Все запросы, сделанные скриптами, загруженными с этих серверов, доверяются API репозиториев.
  
Не забудьте включить точный протокол, имя хозяина и порт (если таковые есть). Не помещай в конце какой-либо перенападающего символа slash (/). Несколько записей можно указать, разделив их запятой.
  
```xml
<repositoryConfiguration>
    <service corsTrustedOrigin="https://<trusted-server>,http://<another-trusted-domain>:8080"" />
    <diagnostics eventLevel="Verbose" systemLoggedEventLevel="Error">
      <traceLog enabled="true" fileName="repository_trace.log" />
    </diagnostics>
 </repositoryConfiguration>
```
