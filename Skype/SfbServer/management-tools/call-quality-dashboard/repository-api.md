# <a name="repository-api-for-call-quality-dashboard-cqd-in-skype-for-business-server"></a>API репозитория для панели мониторинга качества звонков (CQD) в Skype для бизнеса Server
 
**Сводка:** Узнайте об API репозитория для панели мониторинга качества вызовов. Панель мониторинга качества звонков — это инструмент для Skype для бизнеса Server.
  
API репозитория предоставляет программный доступ к панели мониторинга качества звонков для Skype для бизнеса Server.
  
## <a name="repository-api-for-call-quality-dashboard"></a>API репозитория для панели мониторинга качества вызовов

API репозитория предоставляет интерфейс доступа к данным для базы данных репозитория. Репозиторий позволяет организовать содержимое в виде дерева или структуры графа, чтобы пользователи могли группировать их так, как это имеет смысл для пользователей. Репозиторий поддерживает два общих типа пользователей: системный пользователь, который является встроенным пользователем, представляющим репозиторий, и обычных пользователей, представляющих авторизованного пользователя репозитория.
  
API репозитория состоит из трех общих служб: 
  
- [Служба пользователей для CQD](user-service.md) — для доступа к пользователям.
    
- [Item Service for Call Quality Dashboard (CQD)](item-service.md) - for accessing Items and the contents stored in Items.
    
- Служба параметров пользователя для панели мониторинга качества вызовов [(CQD)](user-settings-service.md) для доступа к пользовательским настройкам.
    
Панель мониторинга качества вызовов использует API репозитория для управления следующими сведениями: 
  
- **Пользователь** — представление пользователей, у которых есть доступ к репозиторию.
    
- **Отчет** содержит список запросов, хранимый в качестве контента в элементов репозитория.
    
- **Запрос** — используется для извлечения данных из API данных, хранимого в качестве контента в элементов репозитория.
    
- **Параметр пользователя** — описывает необязательное поведение приложения для пользователя.
    
  **Поддержка общего доступа к ресурсам между источниками (CORS) для API репозитория**
  
API репозитория поддерживает общий доступ к ресурсам между источниками (CORS). CORS — это функция HTTP, которая позволяет веб-приложению под одним доменом получать доступ к ресурсам в другом домене. Веб-браузеры реализуют ограничение безопасности, известное как политика одинакового источника, которая не позволяет [веб-странице](https://www.w3.org/Security/wiki/Same_Origin_Policy) вызывать API в другом домене. CORS предоставляет безопасный способ разрешить одному домену (домену источника) вызывать API в другом домене. Подробные [сведения о CORS](https://www.w3.org/TR/cors/) см. в спецификации CORS.
  
 **Включение CORS для API репозитория**
  
 Далее приводится фрагмент API репозитория web.config с двумя доменами, перечисленными в параметрах приложения corsTrustedOrigin. API репозитория доверяет всем запросам, сделанным скриптами, загруженными с этих серверов.
  
Не забудьте включить точный протокол, имя хоста и порт (если таковые есть). Не помещай косую черту (/) в конце. Несколько записей можно уделить запятой.
  
```xml
<repositoryConfiguration>
    <service corsTrustedOrigin="https://<trusted-server>,http://<another-trusted-domain>:8080"" />
    <diagnostics eventLevel="Verbose" systemLoggedEventLevel="Error">
      <traceLog enabled="true" fileName="repository_trace.log" />
    </diagnostics>
 </repositoryConfiguration>
```
