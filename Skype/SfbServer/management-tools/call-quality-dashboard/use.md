# <a name="use-call-quality-dashboard-for-skype-for-business-server"></a>Использование панели мониторинга качества звонков для Skype для бизнеса Server

**Сводка:** Узнайте, как использовать панель мониторинга качества вызовов. Панель мониторинга качества звонков — это инструмент для Skype для бизнеса Server.

Панель мониторинга качества вызовов (CQD) позволяет ИТ-специалисты использовать сводные данные для выявления проблем, связанных с качеством мультимедиа, путем сравнения статистики для групп пользователей с целью определения тенденций и шаблонов. CQD ориентирован не на решение отдельных проблем с вызовами, а на выявление проблем и решений, которые применяются ко многим пользователям.

## <a name="call-quality-dashboard-user-guide"></a>Руководство пользователя панели мониторинга качества вызовов

CQD — это веб-портал для быстрого создания и организации отчетов на основе данных о качестве работы (QoE). CQD развертывает куб SSAS для сбора данных в базе данных метрик качества и позволяет администраторам создавать и изменять отчеты или вести расследования в режиме реального времени. Хотя можно использовать Excel для прямого подключения к кубу, портал оптимизирован для нескольких процессов, включающих данные о QoE. Эти данные включают:

- Кэшные данные отчетов для быстрого доступа
- Глубокие ссылки на страницы отчетов для обмена информацией и публикации
- Упрощенное редактирование и создание отчетов, а также редактируемые метаданные для описаний отчетов.

Кроме того, CQD предоставляет веб-API, которые дают пользователям программный доступ к данным куба для использования в пользовательских панелях мониторинга.

### <a name="feature-overview"></a>Обзор возможностей

При посещении панели мониторинга качества вызовов вы увидите следующий экран:

![Использование CQD](../../media/1e061858-db6f-452b-9ae4-eab507220371.png)

1. В области сводки можно найти контекст для набора отчетов (справа).
2. Нажмите кнопку "Изменить" в области сводки, чтобы установить свойства уровня (включая высоту оси Y).
3. Навигация помогает определить текущее расположение в иерархии набора отчетов.
4. Отчеты с в подрепортами показаны с синей ссылкой. Щелкните ссылку, чтобы развернуть отчеты.

Чтобы показать подробные значения, нанося указатель мыши на лиовые диаграммы и линии тренда. В отчете с фокусом показано меню действий: "Изменить", "Клонировать", "Удалить" и "Скачать".

### <a name="default-reports"></a>Отчеты по умолчанию

При первом доступе к порталу панели мониторинга качества вызовов автоматически создается набор отчетов по умолчанию. Эти отчеты иногда называют системные отчеты. Вы можете свободно изменять или удалять эти отчеты или расширять их, создавая новые отчеты того же и потомка.

На верхнем уровне в отчете "Ежемесячный тренд аудиопотоков" показана помесячная тенденция для всех аудиопотоков. Чтобы показать более подробное представление данных, представленных лиловой диаграммой, на панели на панели. Щелкните заголовок отчета monthly Trend для аудиопотоков, чтобы перейти к отчету "Управляемые и неугодные аудиопотоки", где отчеты разделены между управляемыми и неугодными вызовами. Управляемые вызовы — это вызовы внутри корпоративного брандмауэра через проводные подключения. Неугодные вызовы включают вызовы, сделанные за пределами корпоративного брандмауэра, и все вызовы, сделанные через Wi-Fi.

Другой отчет верхнего уровня называется "Гистограмма оценки качества вызовов, сообщаемая пользователем". Оценки качества звонков — это номера, которые пользователи Skype для бизнеса предоставили в конце звонка, чтобы указать качество звонка. Номера оценки в диапазоне от 1 до 5, 1 — это наихудшее, а 5 — наилучшее. Гистограмма показывает количество аудиозвонков с задаваемой оценкой за месяц.

Щелкните заголовок любого отчета, чтобы перейти к отчетам с дополнительными фильтрами данных. В системных отчетах каждый из них отображает подмножество данных, доступных в родительском отчете. Модель устранения проблем проста: изучите, к какову вложу данные или тенденции, предлагающие проблему, и постепенно сужаем пространство проблемы. Возможность создавать подрепорты позволяет вам самостоятельно гадать о причинах определенных тенденций данных.

Нажмите кнопку "плюс" рядом с заголовком, чтобы открыть диалоговое окно, в котором можно добавить новое измерение, меру или фильтр.

4. В нижней части окна редактирования можно создать подробное описание отчета.

Меню параметров содержит ссылки на полезные страницы, такие как "Состояние системы" и "Сведения" и расположено в правом верхнем углу панели мониторинга.

![Использование CQD](../../media/0e9ae123-e231-4fea-94e1-5788e8f3e1d3.png)

Будут ли показываться описания и отметки времени, могут ли отдельные пользователи, и эти параметры влияют только на версию панели мониторинга отдельного пользователя и не изменяют набор отчетов или то, что видят другие пользователи. ## <a name="cqd-streams"></a>Потоки CQD

Поток CQD считается хорошим, неудовлетворительнным или неклассифицированным. В CQM 1.5 теперь используется следующее определение CQD:

- Неудовлетворительный поток — это любое сочетание метрик вызовов низкого качества, пре превышенных пороговых значений.
- Если один поток в вызове неудовлетворительн, оба потока вызова помечаются как неудовлетворительные. В конференциях каждый участник считается уникальным вызовом и сообщается независимо от всех остальных.
- Неклассифицированные потоки — это потоки без метрик качества (искусственных транзакций или коротких вызовов).
- Допустимые потоки = немобильные клиенты
- Классификатор нельзя изменить

**Определение или классификатор вызовов низкого качества**

|Метрика|Порог|
|:-----|:-----|
|DegradationAvg |Больше 1.0 (-1 MOS сети) |
|RoundTrip |Больше 500 |
|PacketLossRate |Больше 0,1 (10%) |
|JitterInterArrival |Больше 30 |
|RatioConcealedSamplesAvg |Больше 0,07 |
|||

Определение JPDR = определение звонка низкого качества минус RatioConcealedSamplesAvg

## <a name="where-is-callercallee"></a>Где находится вызываемая или вызываемая?

В CQD не используются поля "Вызывающий", "Вызывающий", а "Первый" и "Второй", так как между вызываемой и вызываемой находятся этапы.

 **First** Всегда будет конечной точкой сервера (например, AV MCU или сервер-посредник), если сервер участвует в потоке.

 **Second** Всегда будет конечной точкой клиента, если это не поток Server-Server клиента.

**Пример классификации "Первая" и "Вторая"**

|Конечная точка 1 UAType|Конечная точка 2 UUAType|Первый|Секунды|
|:-----|:-----|:-----|:-----|
|2 (AVMCU) |4 (Skype для бизнеса) |Конечная точка 1 |Конечная точка 2 |
|2 (AVMCU) |1 (mMediationServer) |Конечная точка 2 |Конечная точка 1 |
|4 (Skype для бизнеса) |4 (Skype для бизнеса) |Вызываемая в MediaLine |Вызываемая в MMediaLine |
|||||

Если обе конечные точки имеют одинаковый тип, CQD делает запись "Вызываемая" первой и вызываемой второй. Дополнительные сведения об именах конечных точек см. [в этом блоге.](https://blogs.technet.com/b/jenstr/archive/2015/05/22/call-quality-dashboard-tips-and-tricks.aspx)

## <a name="accounting-for-vpn"></a>Учет VPN

Если известно, что VPN-решение правильно установило флаг VPN, все настроено. В противном случае используйте один из следующих методов:

- Создайте тип сети VPN (предпочтительный), а затем привяжите vpn-подсети к этому новому типу СЕТИ VPN.
- Создайте здание VPN, а затем привяжите vpn-подсети к этому окне.

## <a name="query-fundamentals"></a>Основы запросов

Хорошо сформированный запрос содержит все три этих параметра:

- Измерение
- Dimension
- Фильтр

Примером хорошо сформированного запроса может быть "Show me Poor Streams [Measurement] by Subnet [Dimension] for Building 6 [Filter]".

## <a name="what-does-union-do"></a>Что делает UNION?

Union позволяет фильтровать условия с помощью оператора AND. Существуют сценарии, в которых можно объединить несколько условий фильтра, чтобы получить результат, аналогичный операции OR.

Пример. Чтобы получить все потоки из здания, UNION предоставляет отдельное представление объединенного наборов данных. Чтобы использовать UNION, вставьте общий текст в поле UNION при двух условиях фильтрации, которые вы хотите использовать в UNION.

## <a name="operational-processes"></a>Операционные процессы

Сначала просмотрите и управлите управляемые потоки. Качество в этой области должно быть на 100 % в вашем средстве контроля, поэтому его проще всего устранять.

### <a name="managed-streams"></a>Управляемые потоки

Проверяйте и устраняйте управляемые потоки в следующем порядке:

1. Server-Server
2. Server-Wired-Inside
3. Wired-Wired-Inside

### <a name="unmanaged-streams"></a>Неугодные потоки

Проверяйте и устраняйте неупорядоченные потоки в следующем порядке:

1. Server-Wifi-Inside
2. Проводная проводная проводная прокси-сервер
3. Server-Wifi-Outside
4. Wired-Outside-Direct
5. Wired-Outside-Relay
6. Другие неугодные
