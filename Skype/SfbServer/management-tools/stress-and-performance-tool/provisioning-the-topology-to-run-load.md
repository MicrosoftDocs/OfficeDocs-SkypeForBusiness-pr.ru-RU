# <a name="provisioning-the-topology-to-run-load-in-stress-and-performance-scenarios"></a>Подготовка топологии для запуска загрузки в сценариях нагрузки и производительности
 
В Skype для бизнеса Server 2015 изменения или подготовка топологии, позволяющие пользователям успешно запускать средство "стресс и производительность".
  
В зависимости от существующих настроек и конфигурации для развертывания Skype для бизнеса Server 2015 может потребоваться внести некоторые изменения в свою среду. Ниже перечислены эти изменения.
  
1. Установите для политики выполнения Windows PowerShell неограниченный доступ. Если вы не уверены в том, что в данный момент установлено приложение, вы можете открыть командную консоль управления Skype для бизнеса Server и выполнить следующую команду:
    
   ```PowerShell
   Get-ExecutionPolicy
   ```

   Если значение "неограниченный" не возвращено, необходимо выполнить следующее:
    
   ```PowerShell
   Set-ExecutionPolicy -Unrestricted
   ```

2. Чтобы эффективно настроить Skype для бизнеса Server, необходимо выполнить указанные ниже действия.
    
    - Вы должны быть знакомы с топологией Skype для бизнеса Server 2015 (именами компьютеров, экземплярами служб, именами сайтов и политиками).
    
    - Назначьте некоторых пользователей, которые созданы для групп, например групп слежения группы ответа (например, URI SIP).
    
3. Чтобы запустить сценарий из командной строки, можно использовать:
    
   ```PowerShell
   PowerShell.exe -file <path to the file>
   ```

4. Как правило, после того как вы запустите сценарий из этого пакета, полученные результаты будут храниться в файле в том же пути, из которого был выполнен сценарий. Также есть формат именования, \<имя_сценария\>$h $ m $ s. txt. Таким образом, если вы запустили Арчивингполици. ps1 на 12:15 PM, вы получите файл журнала с именем ArchivingPolicy121500. txt.
    
5. Несмотря на то, что мы предоставили эти примеры для настройки сервера, вы можете как изменить конфигурацию, так и восстановить ее, после чего вы закончите выполнение нагрузочного тестирования.
