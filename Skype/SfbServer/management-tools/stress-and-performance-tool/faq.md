# <a name="faq-for-the-skype-for-business-server-2015-stress-and-performance-tool"></a>Вопросы и ответы о средстве работы и производительности Skype для бизнеса Server 2015
 
Вопросы и ответы о поддержке средств нагрузки и производительности в Skype для бизнеса 2015 (часто задаваемые вопросы), которые полезны для выяснения того, какие конфигурации средств поддерживаются, проблемы с поиском и устранения неполадок, а также более четкое поведение, которое может отображаться при запуске средства .
  
 В этой статье описаны некоторые наиболее часто задаваемые вопросы о средстве работы и производительности Skype для бизнеса Server 2015, а также об устранении неполадок и выборе средств настройки.
  
## <a name="can-i-run-lyncperftoolexe-in-production"></a>Могу ли я запускать Линкперфтул. exe в рабочей среде?

Это **не** рекомендуется. Это средство влияет на производительность вашего рабочего сервера, безопасность и работу конечных пользователей.
  
## <a name="im-logging-my-users-on-for-the-first-time-why-are-my-servers-running-a-high-load"></a>Я занимаюсь регистрацией моих пользователей в первый раз. Почему мои серверы работают с высокой загрузкой?

При первом входе пользователя в систему дополнительные операции выполняются в фоновом режиме. В результате производительность серверного сервера Microsoft SQL Server может быть снижена. Рекомендуется запустить короткий тест для входа на всех пользователей, а затем перезапустить его перед тем, как начать оценивать результаты с помощью этого средства. Skype для бизнеса Server не поддерживает более 12 одновременных сеансов входа пользователей в секунду, но имейте в виду, что реальный номер, который может обрабатывать ваши серверы, зависит от конфигурации оборудования и может быть ниже поддерживаемого значения.
  
## <a name="my-clients-are-running-out-of-memory-what-should-i-do"></a>У моих клиентов недостаточно памяти. Что мне делать?

Если у клиентов недостаточно памяти, вы должны уменьшить количество пользователей, выполнивших вход на сервер, на котором установлен интерфейс Skype для Business Server. Вы также можете выбрать масштабирование пулов переднего плана, если проблема сохраняется.
  
## <a name="can-i-run-this-tool-on-a-skype-for-business-server-itself"></a>Можно ли запустить эту программу на сервере Skype для бизнеса?

Это не следует делать. Этот сценарий не поддерживается, так как он может завершиться сбоем из-за несовпадения двоичных файлов, а также в том случае, если цель состоит в измерении потребления ресурсов на сервере. Фактическое выполнение этого средства может повлиять на производительность сервера и сделать недействительными ваши данные и измерения.
  
## <a name="can-i-run-lyncperftoolexe-on-a-virtual-server-or-on-microsoft-hyper-v-server-20082012"></a>Можно ли запускать Линкперфтул. exe на виртуальном сервере или на сервере Microsoft Hyper-V 2008/2012?

Да, Вы можете.
  
## <a name="what-does-mpop-mean"></a>Что означает МПОП?

МПОП — это сокращенный способ сказать "несколько точек присутствия". МПОП предназначен для имитации сценариев, в которых пользователи входят в состав клиента Skype для бизнеса 2015 с нескольких компьютеров или устройств. Имейте в виду, что в Линкперфтул. exe каждая конечная точка использует профиль по умолчанию. Другими словами, профиль не делится между двумя точками присутствия.
  
## <a name="i-started-lyncperftoolexe-but-nothing-is-happening-whats-going-on"></a>Я начал Линкперфтул. exe, но ничего не происходит. Что происходит?

Установите флажок Общий счетчик активных конечных точек на серверах, чтобы убедиться, что пользователи подключаются. Если пользователи не подключаются к сети, проверьте конфигурацию 2015 в Skype для бизнеса Server. Проблема, которую вы видите, обычно возникает из-за неправильного имени сервера, префикса пользователя или пароля. Обратите внимание, что внешние клиенты должны указать прокси и значения Таржетсервер доступа. Проверьте номер порта в файле конфигурации.
  
## <a name="how-can-i-be-sure-that-something-is-being-measured"></a>Как убедиться в том, что что-то измеряется?

Существуют счетчики производительности Линкперфтул, указывающие, являются ли пользователи подключаются и выполняются действия, но самый простой способ убедиться в том, что меры измеряются — вход в одну из учетных записей с помощью клиента Skype для бизнеса 2015 и выполните указанные выше действия. действия. Проверьте результаты, чтобы подтвердить произведенные измерения.
  
## <a name="i-have-lync-server-2010-capacity-planning-tools-andor-lync-server-2013-capacity-planning-tools-installed-is-that-okay"></a>У меня установлены инструменты планирования для Lync Server 2010 и/или инструменты по планированию производительности Lync Server 2013. Это так?

 У этих средств есть проблемы взаимодействия! Чтобы получить действительные данные с помощью средства нагрузки и производительности Skype для бизнеса Server 2015, необходимо удалить все предыдущие версии этих средств.
  
## <a name="will-the-stress-and-performance-tools-set-up-the-caa-call-information-server-topology"></a>Настроили ли средства для работы с загрузкой и производительностью топология сервера информации о звонке Каа?

Нет, средства не будут выполняться. Средства создают только пользователи, контакты и списки рассылки, предназначенные для моделирования нагрузки пользователей.
  
## <a name="what-is-the-maximum-number-of-users-that-the-tools-support"></a>Каково максимальное число пользователей, которые поддерживаются средствами?

При тестировании мы создали общее количество пользователей 80 000 и проработали тесты для групповых пользователей 30 000, которые работают с этими инструментами. Мы рекомендуем использовать максимум пользователей 120 000, но при этом в технических ограничениях допускайте более высокие значения. Помните, что эти значения зависят от сервера и оборудования в среде.
