# <a name="faq-for-the-skype-for-business-server-2015-stress-and-performance-tool"></a>Вопросы и ответы по Скайп для Business Server 2015 Stress and Performance Tool
 
Скайп для бизнеса 2015 Stress and Performance Tool часто задаваемые вопросы (FAQ), полезен для узнать, какие средства конфигурации поддерживаются, устранение неполадок средства и разъясняющее настройки, которые могут отображаться при запуске средства нагрузка и производительность .
  
 Часто задаваемые вопросы рассматриваются некоторые наиболее часто задаваемые вопросы о Скайп для Business Server 2015 Stress and Performance tool и может помочь с помощью средства устранения неполадок и параметры настройки.
  
## <a name="can-i-run-lyncperftoolexe-in-production"></a>Можно ли запускать LyncPerfTool.exe в рабочей среде?

Это **не** рекомендуется. Средство может влиять на производительность сервера, безопасности и конечных пользователей.
  
## <a name="im-logging-my-users-on-for-the-first-time-why-are-my-servers-running-a-high-load"></a>Я вход Мои пользователи в первый раз. Почему мои серверы работы высокая загрузка?

При первом входе в систему, дополнительные операции выполняются в фоновом режиме. В результате может снижаться производительность на Microsoft SQL Server фоновый сервер. Рекомендуется выполнить проверку короткий, который заносит на всех пользователей и перезапустите клиентов перед началом работы для измерения результатов с помощью средства. Скайп для Business Server не поддерживает более 12 сеансов входа числа пользователей в секунду, но имейте в виду, фактический номер, могут быть обработаны серверов зависит от конфигурации аппаратного обеспечения и может быть меньше, чем значение, поддерживаемое.
  
## <a name="my-clients-are-running-out-of-memory-what-should-i-do"></a>Мои клиенты хватает памяти. Что мне делать?

Если клиенты хватает памяти, следует уменьшить количество пользователей, войти в систему на Скайп для пула переднего плана Business Server. Можно также масштабирование выход интерфейсных пулов плана, если проблемы является постоянным.
  
## <a name="can-i-run-this-tool-on-a-skype-for-business-server-itself"></a>Можно ли запускать это средство на Скайп для Business server самого?

Не следует этого. В этом сценарии не поддерживается, так как он может завершиться ошибкой из-за несоответствие двоичных, а также из-за цели для измерения потребления ресурсов на сервере. Фактически запуском средства существует может влиять на производительность сервера и недействительными данные и измерения.
  
## <a name="can-i-run-lyncperftoolexe-on-a-virtual-server-or-on-microsoft-hyper-v-server-20082012"></a>Можно ли запускать LyncPerfTool.exe на виртуальном сервере или на Microsoft Hyper-V Server 2008/2012 г.

Да, Вы можете.
  
## <a name="what-does-mpop-mean"></a>Что означает MPOP?

MPOP сокращенный способ говорится о том, «несколько точек присутствия». MPOP предназначен для моделирования сценариях, где пользователь вошел в Скайп для клиента Business 2015 из нескольких компьютерах или устройствах. Обратите внимание, что в LyncPerfTool.exe, каждая конечная точка использует профиль по умолчанию. Другими словами профиль не будет разделение между двумя точками присутствия.
  
## <a name="i-started-lyncperftoolexe-but-nothing-is-happening-whats-going-on"></a>Я работы LyncPerfTool.exe, но ничего не происходит. Что происходит?

Проверьте счетчик общее активных конечных точек на серверах, чтобы увидеть, если он выполняется подключение. Если пользователи не подключаетесь, проверьте вашей Скайп для конфигурации Business Server 2015. Проблемы, которые вы видите обычно возникающая из-за одно имя сервера, префикс пользователя или пароль. Обратите внимание, что внешних клиентов следует указать значения прокси-сервера доступа и TargetServer. Проверьте номер порта в файле конфигурации.
  
## <a name="how-can-i-be-sure-that-something-is-being-measured"></a>Как убедиться в том, что что-то измеряется?

Существует LyncPerfTool счетчики производительности, которые указывают ли подключение пользователей и выполнения действий, но это самый простой способ убедитесь, что измерение действия является выполните вход на один из учетных записей с Скайп для клиента Business 2015 и выполните те действия самостоятельно. Проверка результатов для подтверждения измерения были выполнены.
  
## <a name="i-have-lync-server-2010-capacity-planning-tools-andor-lync-server-2013-capacity-planning-tools-installed-is-that-okay"></a>Я установлены инструменты средства планирования Lync Server 2010 емкости и/или планирования мощности Lync Server 2013. Это допустимо?

 Эти средства имеют проблем взаимодействия! Необходимо удалить все предыдущие версии этих средств для получения допустимых данных из Скайп для нагрузочного 2015 Business Server и проверки производительности.
  
## <a name="will-the-stress-and-performance-tools-set-up-the-caa-call-information-server-topology"></a>Нагрузка и производительность средства настраивает топологии серверов CAA вызова сведения?

Нет, не сделать Сервис. Средства создавать только пользователи, контакты и списки рассылки для имитации пользовательской нагрузки.
  
## <a name="what-is-the-maximum-number-of-users-that-the-tools-support"></a>Что такое максимальное количество пользователей, которые поддерживают средства?

Во время тестирования, мы создали не более 80000 пользователей и выполнении тестов, группировка 30 000 пользователей, работающих в этих средств. Рекомендуется не более 120 000 пользователей, несмотря на то, что технические ограничения дать более высокие значения. Имейте в виду, что эти значения зависят от сервера и оборудования в вашей среде.
