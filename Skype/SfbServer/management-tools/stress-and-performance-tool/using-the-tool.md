# <a name="using-the-skype-for-business-server-2015-stress-and-performance-tool"></a>С помощью Скайп for Business Server 2015 Stress and Performance Tool
 
Для запуска Скайп для Business Server 2015 Stress and Performance Tool, вы будете должны должны иметь возможность управлять пользователи, контакты и профили пользователей, Настройка средства для выполнения и просмотрите выходные данные или результаты, созданные с помощью данного средства.
  
Существует четыре области, связанные с управлением Скайп для Business Server 2015 Stress and Performance Tool (исполняемый файл — LyncPerfTool.exe):
  
- [Создание пользователей и контактов](using-the-tool.md#BKMK_CreateUsersAndContacts)
    
- [Настройка профилей пользователей](using-the-tool.md#BKMK_UserProfile)
    
- [Запустите LyncPerfTool](using-the-tool.md#BKMK_RunTool)
    
- [Толкование результатов](using-the-tool.md#BKMK_Interpret)
    
## <a name="create-users-and-contacts"></a>Создание пользователей и контактов
<a name="BKMK_CreateUsersAndContacts"> </a>

Необходимо использовать Скайп средства подготовки 2015 Business Server (SB 2015) пользователя (UserProvisioningTool.exe) для создания пользователей и контактов для нагрузочного тестирования и тестирования производительности.
  
Это список полезные термины, которые могут оказаться полезными при чтении разделов:
  
- **Подразделение** - доменных служб Active Directory (AD DS) подразделение (OU).
    
- **Федеративные аудио- и кросс-пула** - пользователей, которые могут общаться с пользователями из других служб мгновенного обмена Мгновенными сообщениями.
    
- **Списки рассылки** - или списки рассылки. Это объекты в Доменных службах Active Directory, которая содержит список пользователей, Доменные службы Active Directory. Они используются для упрощения связи между разными группами пользователей.
    
- **Сведения о расположении службы** — Скайп для службы Business Server 2015, если он включен и настроен на телефон, обеспечивающий для извлечения физическое расположение для служб улучшены 911 (службы E911).
    
- **Номера телефонов США** - номера телефонов, назначенных пользователю в дополнение к SIP URI, используемый для маршрутизации входящих и исходящих вызовов, в обратном номер подстановки (RNL).
    
### <a name="create-users-and-contacts-by-using-userprovisioningtoolexe"></a>Создание пользователей и контактов с помощью UserProvisioningTool.exe

> [!NOTE]
> Прежде чем даже начать, убедитесь, что абсолютно необходимо войти систему как член группы безопасности "Администраторы домена" для запуска этой программы. Необходимо сделать это, так как вы будете создавать пользователей Active Directory. 
  
Необходимо использовать Скайп для средство подготовки пользователей Business Server для создания пользователей и контактов для моделирования нагрузки.
  
Пакет **Скайп для Business Server Stress and Performance Tool** установка **Скайп для средство подготовки пользователей Business Server** . Убедитесь, что пакет установщика (CapacityPlanningTool.msi) уже была запущена на сервере переднего плана или сервера Standard Edition, которые планируется тестировать.
  
На сервере переднего плана или сервера Standard Edition, можно начать Скайп средства подготовки пользователей Business Server, запустив файл UserProvisioningTool.exe (находится в папке % InstalledDirectory%LyncStressAndPerfTool\LyncStress).
  
> [!IMPORTANT]
> При создании большого числа пользователей (например, 10 000 или более), запустите UserProvisioningTool.exe. Вам потребуется сделать это, так как средство будет создание и настройка *нового* AD пользователей.
  
Откроется программа подготовки пользователей, нажмите кнопку настройки и выберите Конфигурация нагрузки. 
  
Чтобы приступить к настройке пользователей и контактов, загрузите файл по умолчанию, включенные в состав пакета, называется «SampleData.xml». Это подготовит полей с образцами данных, который необходимо изменить чтобы сделать соответствующий для вашего развертывания.
  
Если имеется предварительно настроенный XML-файл, который уже содержит индивидуальные настройки, можно загрузить файл вместо этого. Заполните поля в средстве подготовки пользователей, как описано в следующих разделах.
  
### <a name="to-configure-server-options"></a>Чтобы настроить параметры сервера:

1. В поле **Полное доменное имя пула переднего плана** введите полное доменное имя (FQDN) сервера Standard Edition или пула переднего плана, где будут размещены пользователей.
    
2. В поле **Префикс имени пользователя** введите префикс, который будет использоваться для bust вашей имена пользователей для тестирования (например, «TestUser»).
    
3. В поле **пароль** введите пароль, который будет использоваться для всех тестовых учетных записей пользователей.
    
4. В поле **Домен учетной записи** введите имя домена из текущего домена AD (один, в котором вы хотите создать тестовых пользователей).
    
5. В поле **Подразделение** , введите имя домена AD, где вы хотите создать эти тестовых пользователей. (Если Подразделение еще не существует, он будет создан автоматически).
    
6. В поле **код области телефона** введите код города трехзначный, чтобы использовать во всех тестовых учетных записей пользователей. Убедитесь, что код города выбранное вами не конфликтовала с кодами области других пользователей в AD.
    
7. Щелкните, чтобы установить флажок **Включено голосовой связи** , если вы хотите включить тестовых пользователей корпоративной голосовой связи.
    
8. В поле **Количество пользователей** предоставьте общее число тестовых пользователей, которые необходимо создать.
    
9. В поле **Начать индекса** , предоставьте начальный номер, который будет использоваться в качестве суффикса с префиксом имя пользователя (, например «TestUser» — это префикс и имя приведет к сбросу в «0» в приведенном ниже примере.)
    
     ![Средство подготовки пользователей с открытой вкладкой User Creation (Создание пользователя).](../../media/591d8280-8979-4a8c-83bc-af126e87bf29.png)
  
#### <a name="create-users-button"></a>Кнопка "Создать" пользователей

При нажатии кнопки **Создания пользователей** проверяются входные параметры, которые вы ввели. В случае ошибки проверки, вам будет предложено исправить их. Или, если все значения верны, пользователей будет запущен процесс появления в AD (в Подразделении независимо от выбранного указанному). Вы увидите индикатор хода выполнения в нижней части Средство во время его выполнения. Не закрывайте приложение во время активности индикатор хода выполнения.
  
При создании пользователей времени, так что произведите план соответствующим образом. Этот процесс может занять от нескольких минут для нескольких пользователей до нескольких часов для большого числа пользователей.
  
Если у вас нет доступа к контроллеру домена AD в тестовой среде, по-прежнему можно проверить при создании пользователей войдите в систему как один из пользователей в диапазоне от пользователей, которого требуется создать. Не забудьте префикс и суффикс, а также @sipDomain как имя пользователя. Ниже приведен пример: <em>TestUser20@contoso.net</em> .
  
> [!NOTE]
> Если пользователи уже существует, нажав кнопку Создать пользователей будет обновлена их изменений конфигурации. 
  
#### <a name="delete-users-button"></a>Пользователи кнопка "Удалить"

При нажатии кнопки **Удалить пользователей** проверяются на вкладке входные параметры. В случае ошибки проверки, вам будет предложено исправить их, и если входного значения верны, указанного тестовых пользователей будет отключены и удалены из службы каталогов Active Directory. Опять же индикатор хода выполнения будет отображаться в нижней части на этой вкладке и не следует закрыть приложение во время активности индикатор хода выполнения.
  
> [!NOTE]
> Поддерживаются только США телефонные номера в формате. Номера телефонов всегда назначить пользователям и всех пользователей, созданные с UserProvisioningTool.exe по умолчанию включены для корпоративной голосовой связи. Сценарии, используйте номер телефона, например автосекретарь конференц-связи или звонки по ТСОП объединенных Коммуникаций, используйте этот номер телефона для правильной маршрутизации вызовов. По этой причине *каждый пользователь* должен иметь *Уникальный номер телефона* .
  
> [!NOTE]
> **При наличии для создания пользователей дважды команда завершится ошибкой, если не использовать другой код области или предыдущей пользователи были отключены с помощью командлета Disable-CsUser.**
  
> [!IMPORTANT]
> Перед созданием контактов, необходимо сначала выполнить репликацию пользователя (который выполняется на вкладке Пользователи). 
  
> [!IMPORTANT]
> Если вы только что создали пользователей, то необходимо ожидать завершения Скайп репликации Business Server и заполняет учетных записей пользователей в базе данных. **Если он еще не завершено репликации, вы узнаете об ошибке.** Вы узнаете, по окончании пользователей репликации, если запущен Скайп для службы Business Server 2015 переднего плана или запустив командлет Get-CsUser на последний пользователя общему числу указанному успешно.
  
#### <a name="contacts-creation-tab"></a>Вкладка Создание контактов

Эта вкладка позволяет предоставить сведения о контактах пользователей для тестирования.
  
![Средство подготовки пользователей с открытой вкладкой Contacts Creation (Создание контактов).](../../media/dfb7fdf1-fb97-4e8e-8608-c4995f95dd5b.png)
  
### <a name="to-configure-users-contacts-do-the-following"></a>Чтобы настроить контактов, выполните следующие действия.

1. В поле **Средняя контактов на пользователя** введите среднее число контактов, чтобы заполнить в список контактов для каждого пользователя.
    
2. Выберите флажок **фиксированный** , если вы хотите создать равно числу контактов для каждого пользователя. Если вы хотите варьировать число контактов, созданной для пользователей, снимите этот флажок.
    
3. В поле **Средняя группы контактов на пользователя** введите число групп контактов на пользователя. Этот номер должен быть меньше, чем **Средняя контактов на пользователя**.
    
4. В поле **федеративных аудио- и кросс-процентное отношение контакты пула** задайте число в диапазоне от 0 до 100. Этот процент контакты будут создаваться с федеративными пользователями.
    
5. В поле **федеративных аудио- и кросс-префикс пула пользователя** ввести имя пользователя для федеративных пользователей, которые будут добавлены в список контактов локальных пользователей.
    
6. В поле **федеративных аудио- и кросс-домен SIP пользователя пула** присвойте имя домена SIP федеративных пользователей.
    
7. **При создании пользователей** на вкладке убедитесь, что указаны правильные данные. Контакты будут создаваться из значений на вкладке Создать пользователя.
    
8. Нажмите кнопку **Создать контакты** для начала создания контакта. Этот процесс может занять несколько минут. После завершения, будет отображаться диалоговое окно с сообщением «операция успешно завершена.» Можно проверить контакты, которые были созданы с вход в систему под учетной записью пользователя, который был создан на вкладке Создать пользователя.
    
> [!NOTE]
> После создания контактов это средство перезапустится сервера переднего плана в пуле целевой. Может потребоваться больше времени (вверх на 2 часа) для серверов переднего плана начать, в зависимости от того, сколько контактов были созданы в данной операции. 
  
#### <a name="distribution-list"></a>Список рассылки

Скайп для Business Server 2015 Stress and Performance Tool можно смоделировать компонент расширение списка рассылки (DL) в Скайп для клиента 2015 бизнеса. Этот шаг можно пропустить, если не планируется включить расширение списка Рассылки в средство подготовки пользователей.
  
![Средство подготовки пользователей с открытой вкладкой Distribution List Creation (Создание списка рассылки).](../../media/4b689306-70c4-4569-9842-15c73f038eb6.png)
  
Вкладка списка рассылки позволяет создавать списки рассылки, которые будут использовать Stress and Performance Tool для функции раскрытие списков рассылки. Прежде чем создавать списки рассылки, Скайп для Business Server 2015 должен быть развернут, включая запуска только. Если это не будет сделано, атрибутов списка Рассылки отсутствует в схеме AD, поэтому средство не будут иметь возможность создавать списки рассылки.
  
### <a name="to-configure-distribution-lists"></a>Чтобы настроить списки рассылки:

1. В поле **Номер из списков рассылки** , предоставьте общее число библиотек документов, необходимо создать (здесь рекомендуется начать с значение типа double, количество пользователей, у вас есть.).
    
2. В поле **Префикс списка рассылки** введите префикс, который будет иметь все списки рассылки, вы создаете, например *testDL* . Это означает, в 100 списки рассылки имена списков Рассылки будет иметь вид: testDL0, testDL1 до testDL99.
    
3. В поле **Минимальный участников в список Dist.** введите минимальное число пользователей для помещения в каждом списка Рассылки.
    
4. В поле **Максимальное число элементов в списке Dist.** введите максимальное количество пользователей, чтобы добавить в каждый список Рассылки.
    
#### <a name="create-distribution-lists-button"></a>Кнопка "Создать списки рассылки"

При нажатии кнопки Создать списки рассылки, средство запрашивает см Если сопоставление префикса и номера уже существует в Active Directory. Средство создает списки, которые не существуют. При добавлении членов в эти новые списки рассылки, он будет выберите пользователей из диапазона, указанного на вкладке при создании пользователей.
  
#### <a name="location-info-service-config-tab"></a>Перейдите на вкладку расположение файла конфигурации службы Info

Скайп для Business Server 2015 Stress and Performance Tool также можно создать файлы фиктивного конфигурации для информирования о местонахождении. Обратите внимание на то, что служба информирования о местонахождении обычно не имеет значительно влияет на производительность на серверах. 
  
![Средство подготовки пользователей с открытой вкладкой Location Info Service Config (Конфигурация сервиса данных о местоположении).](../../media/227662a2-e0c3-4e34-ab54-5f1459344f30.png)
  
Если нужно проверить эту возможность, введите значения в форме и нажмите кнопку Создать файлы Config локальный источник установки, которая будет создана. Вызывать CSV-файлов:
  
- LIS_Subnet.csv
    
- LIS_Switches.csv

- LIS_WAP.csv
    
Для импорта этих файлов в базу данных локальный источник установки используйте эти командлеты PowerShell:

- Set-CsLisSwitch
    
- Set-CsLisPort
    
- SET-CsWirelessAccessPoint
    
## <a name="configure-user-profile"></a>Настройка профилей пользователей
<a name="BKMK_UserProfile"> </a>

После создания пользователей (с помощью средства создания пользователя) можно настроить профили пользователей с Скайп для Business Server 2015 нагрузки средства настройки (UserProfileGenerator.exe).
  
### <a name="running-the-skype-for-business-server-2015-load-configuration-tool"></a>Под управлением Скайп для средства настройки нагрузки 2015 Business Server

Запустите программу настройки нагрузки (UserProfileGenerator.exe) и заполните поля на вкладках. Это средство создает каталог для каждого клиента компьютеры, которые необходимо выполнить на моделирования. Каждого клиента каталога, поступающие с помощью скрипта запуска Скайп Business Server 2015 Stress and Performance tool (LyncPerfTool.exe). В следующих разделах предоставит примеры того, как для заполнения полей на всех листах Скайп для средства настройки нагрузки 2015 Business Server.
  
> [!IMPORTANT]
> Пользовательские значения, используемые в средстве настройки нагрузки (UserProfileGenerator.exe) должен совпадать значения, заданные в Скайп для создания пользователя средство (UserProvisioningTool.exe) Business Server 2015 для пула. 
  
#### <a name="common-configuration-tab"></a>Вкладка Общие настройки

Ниже показаны на вкладке **Общие конфигурация** средства настройки нагрузки. Заполните поля на вкладке Общие конфигурация, как описано в следующих шагах.
  
![Вкладка User Provisioning (Подготовка пользователей) с открытой вкладкой Common Configuration (Общая конфигурация).](../../media/c25df343-3550-47fb-88e0-29194338fee2.png)
  
1. В поле **Номер доступные компьютеры** введите количество компьютеров, которые необходимо использовать для запуска Stress and Performance tool (LyncPerfTool.exe). Мы рекомендуем, что у вас есть один компьютер для каждых 4500 пользователей, которые вы будете моделирования, но этот номер может различаться, если снизить уровень нагрузки или использовать только подмножество доступных возможностей этого средства (уровни загрузки устанавливаются на вкладке Общие сценарии).
    
2. В поле **префикс для имен пользователей** введите префикс в поле имя пользователя для всех пользователей. Для записи в журнал в универсальный код ресурса (URI) будет: *UserPrefix [пользователя запустите индекса... (Число пользователей-1)] @User домена* , например, myUser009@Contoso.com.
    
3. В поле **пароль для всех пользователей,** введите пароль, используемый во время создания пользователей. Если оставить это поле пустым имя пользователя будет установлен в качестве пароля.
    
4. В поле **Начать индекса пользователя** введите индекс первого пользователя, который требуется настроить. Можно настроить несколько диапазонов для различных типов или уровня нагрузки, но необходимо запустить программу настройки загрузки (UserProfileGenerator.exe) после каждого диапазона, который необходимо настроить.
    
5. В поле **Количество пользователей** введите общее количество пользователей, которые нужно настроить.
    
6. В поле **Домен** введите домен, используемый для SIP URI. Используется для создания URI SIP каждого пользователя, войдите в систему Скайп для сервера переднего плана Business Server 2015 или сервере Standard Edition и может отличаться от учетной записи домена.
    
7. В поле **Домен учетной записи** введите имя домена AD DS.
    
8. В поле **Процент MPOP** (нескольких точка присутствия в процентах) задайте значение для процент пользователей, вошедших из нескольких компьютеров или устройств, например 10 процентов.
    
9. Введите максимальное число параллельных конечных точек в поле **Вход в секунду, (на каждый экземпляр)** . Это максимальное количество входов пользователей и рекомендуется частота меньше / равно 2 в секунду (< = 2).
    
10. В поле **прокси-сервера доступа или полное доменное имя пула** введите полное доменное имя (FQDN) сервера, будут клиентам подключаться к. При выполнении входа в извне, необходимо ввести прокси-сервера доступа. Если пользователи являются внутренними, задайте полное доменное имя их корпоративного пула или сервере Standard Edition.
    
11. В поле **порт** укажите порт, который пользователи должны использовать для SIP (по умолчанию — 5061).
    
12. В поле **Внешний параметры сети сервера** укажите прокси-сервера доступа или полное доменное имя пула и, еще раз, **порт**. Эти параметры используются только для моделирования нагрузки внешних конечных точек.

## <a name="run-lyncperftool"></a>Запустите LyncPerfTool
<a name="BKMK_RunTool"> </a>

Вам потребуется создать пользователей, контактов и сценарии перед запуском Скайп для Business Server 2015 Stress and Performance Tool (LyncPerfTool.exe). Для получения дополнительных сведений об использовании средств для выполнения этих действий просмотрите [Создание пользователей и контактов](using-the-tool.md#BKMK_CreateUsersAndContacts) и [Настройка профилей пользователей](using-the-tool.md#BKMK_UserProfile) ранее в этой статье. Выполнение этих средств также создание файла, которое выполняется с помощью средства нагрузка и производительность как часть пакетного файла с требуемые параметры включены.
  
### <a name="running-the-skype-for-business-server-2015-stress-and-performance-tool"></a>Под управлением Скайп для Business Server 2015 Stress and Performance tool

Средство настройки нагрузки (UserProfileGenerator.exe) создает пакетного файла, который позволяет выполнять Stress and Performance tool (LyncPerfTool.exe), регистрация счетчики производительности и загрузке XML-файл конфигурации. Пакетный файл выполняет один экземпляр LyncPerfTool.exe каждого файла конфигурации. Для запуска пакетного файла выполните следующие действия.
  
### <a name="run-the-stress-and-performance-test"></a>Выполнение теста нагрузки и производительности

1. Скопируйте папку с конфигурации папки и файлы в каталог, в котором LyncPerfTool.exe на каждом клиентском компьютере. (К примеру, если создана файлы конфигурации в папку с именем 1.28_13.16.16 скопируйте этой папки в папку с LyncPerfTool.exe в нем. Это сделать на каждом клиенте).
    
2. Перейдите в папку клиента и запуск пакетного сценария **RunClient** . Дважды щелкните значок пакетный файл в проводнике Windows и его можно было запускать все файлы конфигурации клиента. Можно также запустить скрипт из папки клиента, используя следующий синтаксис:
    
   ```
   RunClient0.bat "C:\Program Files\Skype for Business Server 2015\LyncStressAndPerfTool\LyncStress" 
   ```

Чтобы запустить средство нагрузка и производительность напрямую, откройте командную строку и введите следующую команду в командной строке (и когда это делается в первый раз, убедитесь, что для регистрации счетчики производительности `regsvr32 /i /n /s LyncPerfToolPerf.dll`, как показано в Примечание далее в этом разделе):
  
```
LyncPerfTool.exe /file:IM_client0.xml
```

Чтобы средство отображения значения в файле конфигурации, включают `/displayfile` параметр в предыдущей команде, которая выглядит следующим образом:
  
```
LyncPerfTool.exe /file:IM_client0.xml /displayfile
```

*Окончания* процесса нажмите клавиши Ctrl + C.
  
> [!NOTE]
> Перед запуском средства нагрузка и производительность напрямую, необходимо зарегистрировать счетчики производительности с помощью следующей команды:`regsvr32 /i /n /s LyncPerfToolPerf.dll`
  
> [!NOTE]
> Каждый экземпляр средства нагрузка и производительность при запуске начнется сразу же вход пользователей обычно со скоростью одного пользователя в секунду. 
  
Пиковое пользователя входа процент для пула составляет около 12 в секунду. Это означает, что не следует начинать с более 12 LyncPerfTool.exe экземпляры в то же время во время пользователи по-прежнему вход. Одной тысячи пользователей займет примерно 20 минут полностью войти на одно в секунду.
  
## <a name="interpreting-the-results"></a>Толкование результатов
<a name="BKMK_Interpret"> </a>

Скайп для Business Server 2015 Stress and Performance Tool имеет много счетчиков, которые помогут вам понять, что делает клиента и ли он возникли проблемы.
  
### <a name="client-counters"></a>Счетчики клиента

Каждый экземпляр LyncPerfTool.exe под управлением имеет отдельный экземпляр счетчики. Каждый экземпляр называется по идентификатору процесса Если клиенты перегруженные могут возникнуть другие проблемы. Чтобы избежать этих проблем:
  
- Мониторинг использования ЦП и памяти на клиентских компьютерах. Если Процессора постоянно превышает 90 процентов, сократить число пользователей.
    
- При высоком объем памяти может вызывать проблемы при начнется избежать переполнения файла страницы. Убедитесь, что выделение памяти не является попадание ограничение на компьютере. При запуске в ограничений памяти необходимо учитывайте размер файла подкачки увеличение или уменьшение числа пользователей.
