# <a name="install-and-open-administrative-tools"></a>Установка и открытие средств администрирования

В этом разделе описывается установка средств администрирования, необходимых для развертывания Skype для бизнеса Server и управления им. Средства администрирования устанавливаются по умолчанию на каждом сервере со Skype для бизнеса Server. Кроме того, вы можете установить административные средства на другие компьютеры, например на выделенные административные консоли. Настоятельно рекомендуется установить средства администрирования на компьютере, который находится в том же домене или лесу, что и развертывание Skype для бизнеса Server, которое вы создаете, чтобы убедиться, что действия по подготовке доменных служб Active Directory уже завершены, что позволит использовать средства администрирования на этом компьютере позже для публикации топологии. Кроме того, обязательно просмотрите необходимые требования перед установкой или использованием средств администрирования Skype для бизнеса Server. См. документацию по требованиям [в Skype для бизнеса Server 2019](../../SfBServer2019/plan/system-requirements.md) или Skype для бизнеса Server [2015.](../plan-your-deployment/requirements-for-your-environment/requirements-for-your-environment.md)
 
> [!Important]
> Если в организации требуется разместить службы IIS и все веб-службы на диске, кроме системного, можно изменить путь установки файлов Skype для бизнеса Server в диалоговом окне установки. Если установить файлы установки по этому пути, в том числе OCSCore.msi, остальные файлы Skype для бизнеса Server также будут развернуты на этом диске. 

## <a name="to-install-the-administrative-tools"></a>Установка средств администрирования

1. Войдите в качестве локального администратора (минимальное требование) на компьютер, на котором требуется установить средства администрирования. Если вы вошли как обычный пользователь в Windows и включен контроль учетных записей (UAC), вам будет предложено вводить имя пользователя и пароль, эквивалентные домену.
2. Найдите установочный носитель на компьютере, а затем дважды щелкните \Setup\amd64\setup.exe.
3. Если вам будет предложено установить распространяемый microsoft Visual C++, нажмите кнопку **"Да".**
4. На странице Папка установки нажмите кнопку **ОК**. Измените этот путь на другую папку или диск, если вы хотите установить файлы в другом местоположении.

    > [!Important]
    > Если в организации требуется разместить службы IIS и все веб-службы на диске, кроме системного, можно изменить путь установки файлов Skype для бизнеса Server в диалоговом окне установки. Если установить файлы установки по этому пути, в том числе OCSCore.msi, остальные файлы Skype для бизнеса Server также будут развернуты на этом диске. 

5. На странице Лицензионное соглашение просмотрите условия лицензионного соглашения, нажмите кнопку **Принимаю**, а затем нажмите **ОК**. Этот шаг обязателен для продолжения.
6. На странице "Мастер развертывания" щелкните **"Установить средства администратора".** 
7. После успешного завершения установки нажмите кнопку **Выход**.

Используйте следующие процедуры, чтобы открыть средства администрирования для развертывания, настройки или устранения неполадок топологии Skype для бизнеса Server.

- [Мастер развертывания](#deployment-wizard)
- [Построитель топологий](#topology-builder) 
- [Панель управления Skype для бизнеса Server](#skype-for-business-server-control-panel)
- [Командная консоль Skype для бизнеса Server](#skype-for-business-server-management-shell)

## <a name="deployment-wizard"></a>Мастер развертывания

Используйте следующую процедуру для локального запуска мастера развертывания для добавления или удаления файлов компонентов.

**Запуск мастера развертывания Skype для бизнеса Server**

1. Войдите на компьютер, на котором установлен мастер развертывания Skype для бизнеса Server, в качестве члена группы администраторов домена и группы RTCUniversalServerAdmins.
2. Click **Start**, click **All Programs,** click Skype for Business **Server**, and then click Skype for Business Server **Deployment Wizard.**


## <a name="topology-builder"></a>Построитель топологий 

Используйте следующую процедуру, чтобы открыть построитель топологий, чтобы определить серверы, которые необходимо развернуть в топологии Skype для бизнеса Server.

**Открытие построитель топологий Skype для бизнеса Server для разработки топологии**

1. Войдите в систему компьютера, на котором построитель топологий установлен как член группы администраторов доменов и группы RTCUniversalServerAdmins.
    > [!NOTE]
    > Определить топологию можно с помощью учетной записи, которая является членом локальной группы пользователей, но для чтения, публикации или разрешения топологии, необходимой для установки Skype для бизнеса Server на сервере, необходимо использовать учетную запись, которая является членом группы администраторов домена и группы RTCUniversalServerAdmins и имеет разрешения на полный доступ (т. е. , чтение, запись и изменение) в файловом хранилище, которое вы собираетесь использовать для хранения файлов архива, чтобы построитель топологий можно было настроить необходимый список управления доступом на основе дискреционного доступа (DACLs) или учетную запись с эквивалентными правами пользователя.
 
2. Start Topology Builder: Click **Start,** click **All Programs,** click **Skype for Business Server**, and then click Skype for Business Server **Topology Builder**.

## <a name="skype-for-business-server-control-panel"></a>Панель управления Skype для бизнеса Server 

Используйте одну из следующих процедур, чтобы открыть панель управления Skype для бизнеса Server для управления конфигурацией серверов, пользователей, клиентов и устройств в вашей среде.

> [!NOTE]
> Вы можете использовать учетную запись пользователя, назначенную роли CsAdministrator, для выполнения любой задачи на панели управления Skype для бизнеса Server. Вы можете использовать другие роли для входа в панель управления Skype для бизнеса Server для выполнения определенных задач администрирования в зависимости от задачи, которую необходимо выполнить. Например, можно использовать CSArchivingAdministrator для администрирования архива в панели управления Skype для бизнеса Server. Подробные сведения о ролях см. в планировании [управления доступом на основе ролей.](https://technet.microsoft.com/library/gg425917(v=ocs.15).aspx) Подробные сведения о ролях, которые можно использовать для выполнения особых задач, см. в документации по этим задачам. 

**Открытие панели управления Skype для бизнеса Server с любого компьютера в брандмауэре организации**

1. С помощью учетной записи, которой назначена роль CsAdministrator или другая роль с соответствующими правами и разрешениями для выполняемой задачи, войдите в любой компьютер во внутреннем развертывании, разрешение экрана которого не ниже 1024 x 768.

    > [!IMPORTANT]
    > Если вы настроили простой URL-адрес администрирования, вы можете получить доступ к панели управления Skype для бизнеса Server из браузера, запущенного на любом компьютере в пределах брандмауэра организации. Подробные сведения о настройке простого URL-адреса администрирования см. в планировании простых [URL-адресов](https://technet.microsoft.com/library/gg398287(v=ocs.15).aspx) и редактировании или [настройке простых URL-адресов.](https://technet.microsoft.com/library/gg398063(v=ocs.15).aspx) 

2. Откройте окно браузера и введите URL-адрес администрирования, настроенный для вашей организации.

**Открытие панели управления Skype для бизнеса Server на компьютере со Skype для бизнеса Server**

1. Из учетной записи пользователя, которая является членом роли CsAdministrator или другой роли с соответствующими правами и разрешениями пользователя для выполнения задачи, войдите на компьютер, на котором установлен Skype для бизнеса Server, или как минимум средства администрирования Skype для бизнеса Server. Для настройки параметров разрешение экрана компьютера должно быть не ниже 1024 x 768.
2. Запустите панель управления Skype для бизнеса Server: нажмите кнопку "Начните", выберите пункты "Все программы", "Администрирование", "Skype для бизнеса **Server"** и "Панель управления Skype для бизнеса **Server".**

## <a name="skype-for-business-server-management-shell"></a>Командная консоль Skype для бизнеса Server 

Используйте следующую процедуру для открытия командной оболочки Skype для бизнеса Server для администрирования серверов, пользователей, клиентов и устройств в вашей среде с помощью командной строки.

> [!NOTE]
> Вы можете использовать учетную запись пользователя, назначенную роли CsAdministrator, для выполнения любой задачи в оболочке управления Skype для бизнеса Server. Для выполнения особых административных задач можно использовать для входа другие роли, в зависимости от конкретной задачи. Например, роль CSArchivingAdministrator можно использовать для запуска командлетов, имеющих отношение к администрированию архивации. Подробные сведения о ролях см. в планировании [управления доступом на основе ролей.](https://technet.microsoft.com/library/gg425917(v=ocs.15).aspx) Подробные сведения о ролях, которые можно использовать для выполнения конкретных командлетов, см. в документации по этим командлетам.<br/><br/>Определенные командлеты можно также выполнять с помощью пользовательской учетной записи, входящей в группу RTCUniversalServerAdmins, RTCUniversalUserAdmins или RTCUniversalReadOnlyAdmins, в зависимости от командлета. 

**Открытие оболочки управления Skype для бизнеса Server**

If you open a Windows PowerShell window rather than the Skype for Business Server Management Shell, by default you cannot run the Skype for Business Server cmdlets. Чтобы запустить командлеты Skype для бизнеса Server в Windows PowerShell, введите в командной Windows PowerShell следующую команду:

`Import-Module Lync`

Запустите оболочку управления Skype для бизнеса Server: нажмите кнопку "Начните", выберите "Все программы", "Skype для бизнеса **Server",** а затем щелкните "Skype для бизнеса Server Management **Shell".**
