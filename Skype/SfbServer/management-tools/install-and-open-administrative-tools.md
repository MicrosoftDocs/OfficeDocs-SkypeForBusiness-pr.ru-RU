# <a name="install-and-open-administrative-tools"></a>Установка и открытие средств администрирования

В этом разделе описывается установка административных средств, необходимых для развертывания и управления Skype для бизнеса Server. Административные средства устанавливаются по умолчанию на каждом сервере, Skype для бизнеса Server. Кроме того, вы можете установить административные средства на другие компьютеры, например на выделенные административные консоли. Настоятельно рекомендуем установить административные средства на компьютере, который находится в том же домене или лесу, что и развертывание Skype для бизнеса Server, которое вы создаете, чтобы убедиться, что этапы подготовки Active Directory Domain Services уже завершены, что позволяет использовать административные средства на этом компьютере позже для публикации топологии. Кроме того, убедитесь, что перед установкой или использованием средств администрирования Skype для бизнеса Server необходимо просмотреть необходимые требования. См. документацию по требованиям [Skype для бизнеса Server 2019](../../SfBServer2019/plan/system-requirements.md) или [Skype для бизнеса Server 2015 г](../plan-your-deployment/requirements-for-your-environment/requirements-for-your-environment.md).
 
> [!Important]
> Если вашей организации требуется найти службы IIS (IIS) и все веб-службы на диске, помимо системного диска, можно изменить путь расположения установки для Skype для бизнеса Server файлов в диалоговом окне Установки. При установке файлов установки на этот путь, включая OCSCore.msi, остальные Skype для бизнеса Server будут развернуты и на этом диске. 

## <a name="to-install-the-administrative-tools"></a>Установка административных средств

1. Войдите в качестве локального администратора (минимальное требование) на компьютер, на котором требуется установить средства администрирования. Если вы вошли в систему в качестве стандартного пользователя в Windows включено управление учетной записью пользователя (UAC), вам будет предложено имя и пароль, эквивалентный домену.
2. Найдите установочный носитель на компьютере, а затем дважды щелкните \Setup\amd64\setup.exe.
3. Если вам предложено установить Microsoft Visual C++, нажмите **кнопку Да**.
4. На странице Папка установки нажмите кнопку **ОК**. Измените этот путь на другую папку или диск, если вы хотите установить файлы в другом местоположении.

    > [!Important]
    > Если вашей организации требуется найти службы IIS (IIS) и все веб-службы на диске, помимо системного диска, можно изменить путь расположения установки для Skype для бизнеса Server файлов в диалоговом окне Установки. Если установить файлы установки на этот путь, включая OCSCore.msi, остальные Skype для бизнеса Server будут развернуты и на этом диске. 

5. На странице Лицензионное соглашение просмотрите условия лицензионного соглашения, нажмите кнопку **Принимаю**, а затем нажмите **ОК**. Этот шаг обязателен для продолжения.
6. На странице Мастер развертывания нажмите **кнопку Установите средства администратора**. 
7. После успешного завершения установки нажмите кнопку **Выход**.

Используйте следующие процедуры, чтобы открыть средства администрирования для развертывания, настройки или устранения неполадок Skype для бизнеса Server топологии.

- [Мастер развертывания](#deployment-wizard)
- [Построитель топологий](#topology-builder) 
- [Панель управления Skype для бизнеса Server](#skype-for-business-server-control-panel)
- [Командная консоль Skype для бизнеса Server](#skype-for-business-server-management-shell)

## <a name="deployment-wizard"></a>Мастер развертывания

Используйте следующую процедуру, чтобы запустить мастер развертывания локально, чтобы добавить или удалить файлы компонентов.

**Запуск мастера Skype для бизнеса Server развертывания**

1. Войдите на компьютер, на котором мастер развертывания Skype для бизнеса Server как член группы администраторов домена и группы RTCUniversalServerAdmins.
2. Нажмите **кнопку** Начните, нажмите кнопку Все **программы,** **нажмите Skype для бизнеса Server** и нажмите кнопку **Skype для бизнеса Server мастер развертывания**.


## <a name="topology-builder"></a>Построитель топологий 

Используйте следующую процедуру для открытия топологического конструктора для определения серверов, которые необходимо развернуть в топологии Skype для бизнеса Server области.

**Чтобы открыть Skype для бизнеса Server топологии для разработки топологии**

1. Войдите в систему компьютера, на котором построитель топологий установлен как член группы администраторов доменов и группы RTCUniversalServerAdmins.
    > [!NOTE]
    > Топологию можно определить с помощью учетной записи, которая входит в местную группу пользователей, но для чтения, публикации или установки топологии, необходимой для установки Skype для бизнеса Server на сервере, необходимо использовать учетную запись, которая входит в группу администраторов домена и группу RTCUniversalServerAdmins, и которая имеет полные разрешения управления (т. е.  чтение, запись и изменение в файле, который вы собираетесь использовать для архивного файла, чтобы topology Builder настроил необходимый список управления дискреционным доступом (DACLs) или учетную запись с эквивалентными правами пользователя.
 
2. Начните строитель топологии: нажмите кнопку **Начните, нажмите** кнопку Все **программы, нажмите** кнопку **Skype для бизнеса Server, а** затем нажмите кнопку Skype для бизнеса Server **Топология Builder**.

## <a name="skype-for-business-server-control-panel"></a>Панель управления Skype для бизнеса Server 

Используйте одну из следующих процедур, чтобы открыть панель управления Skype для бизнеса Server для управления конфигурацией серверов, пользователей, клиентов и устройств в вашей среде.

> [!NOTE]
> Вы можете использовать учетную запись пользователя, назначенную роли CsAdministrator для выполнения любой задачи Skype для бизнеса Server панели управления. Вы можете использовать другие роли для входа в панель управления Skype для бизнеса Server для выполнения определенных задач администрирования в зависимости от задачи, которую необходимо выполнить. Например, для администрирования архива в панели управления Skype для бизнеса Server CSArchivingAdministrator можно использовать CSArchivingAdministrator. Подробные сведения о ролях см. в [материале Planning for role-based access control](/previous-versions/office/lync-server-2013/lync-server-2013-planning-for-role-based-access-control). Подробные сведения о ролях, которые можно использовать для выполнения особых задач, см. в документации по этим задачам. 

**Открытие панели управления Skype для бизнеса Server с любого компьютера во брандмауэре организации**

1. С помощью учетной записи, которой назначена роль CsAdministrator или другая роль с соответствующими правами и разрешениями для выполняемой задачи, войдите в любой компьютер во внутреннем развертывании, разрешение экрана которого не ниже 1024 x 768.

    > [!IMPORTANT]
    > Если вы настроили простой единый локатор ресурсов администрирования (URL-адрес), вы можете получить доступ к панели управления Skype для бизнеса Server из интернет-браузера, который работает на любом компьютере в брандмауэре организации. Сведения о настройке простого URL-адреса администрирования см. в материале [Планирование](/previous-versions/office/lync-server-2013/lync-server-2013-planning-for-simple-urls) простых URL-адресов и [изменение или настройка простых URL-адресов](/previous-versions/office/lync-server-2013/lync-server-2013-edit-or-configure-simple-urls). 

2. Откройте окно браузера и введите URL-адрес администрирования, настроенный для вашей организации.

**Открытие панели Skype для бизнеса Server на компьютере с Skype для бизнеса Server**

1. С учетной записи пользователя, которая является членом роли CsAdministrator или другой роли, которая имеет соответствующие права и разрешения для выполнения задачи, войдите на компьютер, на котором Skype для бизнеса Server или, как минимум, Skype для бизнеса Server средства администрирования. Для настройки параметров разрешение экрана компьютера должно быть не ниже 1024 x 768.
2. Начните Skype для бизнеса Server панели управления: нажмите кнопку **Начните, нажмите** кнопку Все **программы,** указать на инструменты администрирования **,** указать на **Skype для бизнеса Server, а** затем **нажмите кнопку Skype для бизнеса Server панели управления**.

## <a name="skype-for-business-server-management-shell"></a>Командная консоль Skype для бизнеса Server 

Используйте следующую процедуру, чтобы открыть Skype для бизнеса Server для администрирования серверов, пользователей, клиентов и устройств в вашей среде с помощью командной строки.

> [!NOTE]
> Вы можете использовать учетную запись пользователя, назначенную роли CsAdministrator, для выполнения любой задачи в Skype для бизнеса Server management Shell. Для выполнения особых административных задач можно использовать для входа другие роли, в зависимости от конкретной задачи. Например, роль CSArchivingAdministrator можно использовать для запуска командлетов, имеющих отношение к администрированию архивации. Подробные сведения о ролях см. в [материале Planning for role-based access control](/previous-versions/office/lync-server-2013/lync-server-2013-planning-for-role-based-access-control). Подробные сведения о ролях, которые можно использовать для выполнения конкретных командлетов, см. в документации по этим командлетам.<br/><br/>Определенные командлеты можно также выполнять с помощью пользовательской учетной записи, входящей в группу RTCUniversalServerAdmins, RTCUniversalUserAdmins или RTCUniversalReadOnlyAdmins, в зависимости от командлета. 

**Открытие Skype для бизнеса Server shell**

Если вы открываете Windows PowerShell, а не Skype для бизнеса Server, по умолчанию нельзя запускать Skype для бизнеса Server. Чтобы запустить Skype для бизнеса Server командлеты из Windows PowerShell, введите следующее в командной Windows PowerShell:

`Import-Module Lync`

Запустите Skype для бизнеса Server: нажмите кнопку **Начните, нажмите** кнопку Все **программы, нажмите** кнопку **Skype для бизнеса Server и** нажмите кнопку **Skype для бизнеса Server shell**.
