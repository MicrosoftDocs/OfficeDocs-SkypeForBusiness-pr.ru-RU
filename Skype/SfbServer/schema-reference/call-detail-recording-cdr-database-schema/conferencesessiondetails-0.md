# <a name="conferencesessiondetails-table-in-skype-for-business-server-2015"></a>Таблица ConferenceSessionDetails в Скайп для Business Server 2015
 
Каждая запись представляет один сеанс конференции, который может быть либо сеансом с центром конференций, либо сеансом с конкретным сервером конференций.
  
|**Столбец**|**Тип данных**|**Ключ/индекс**|**Сведения**|
|:-----|:-----|:-----|:-----|
|**SessionIdTime** <br/> |Даты и времени  <br/> |Основной, внешний  <br/> |Время запроса сеанса; используется совместно с **SessionIdSeq** для уникальной идентификации сеанса конференции. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**SessionIdSeq** <br/> |целое  <br/> |Основной, внешний  <br/> |Номер идентификатора для идентификации сеанса. Используется в сочетании с **SessionIdTime** для уникальной идентификации сеанса конференции. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. * <br/> |
|**ConferenceUriId** <br/> |целое  <br/> |Внешний  <br/> |URI конференции фокус связанные с данного сеанса. [Таблица ConferenceUris в Скайп для Business Server 2015](conferenceuris.md) для получения дополнительных сведений см. URI является конференц-связи с центром URI. <br/> |
|**ConfInstance** <br/> |uniqueIdentifier  <br/> ||Идентификатор, между экземплярами повторяющихся конференций, отделяя его. Каждый экземпляр повторяющейся конференции имеет одинаковые параметр ConferenceURI, но разные значения ConfInstance.  <br/> В этом поле было представлено в Microsoft Lync Server 2013.  <br/> |
|**McuConferenceUriId** <br/> |целое  <br/> |Внешний  <br/> |Связанные с URI конференции сервера конференц-связи для данного сеанса. [Таблица ConferenceUris в Скайп для Business Server 2015](conferenceuris.md) для получения дополнительных сведений см. URI является конференц-связи на стороне сервера URI конференции. Для сеансов Focus конференции этот столбец будет null. <br/> |
|**Идентификатор пользователя** <br/> |целое  <br/> |Внешний  <br/> |Идентификатор одного пользователя в сеансе конференц-связи. В [таблице пользователей](users.md) для получения дополнительных сведений см. <br/> |
|**UserEndpointId** <br/> |uniqueidentifier  <br/> ||Идентификатор GUID для определения экземпляра конечной точки. Например если один пользователь вошел в систему на разных компьютерах с ту же учетную запись, затем каждого машина будет иметь идентификатор другой конечной точки.  <br/> |
|**OnBehalfOfId** <br/> |целое  <br/> |Внешний  <br/> |Указывает идентификатор пользователя, что вызывающий абонент находится от имени. В [таблице пользователей](users.md) для получения дополнительных сведений см. <br/> |
|**ReferredById** <br/> |целое  <br/> |Внешний  <br/> |Идентификатор пользователя, который называется звонок. В [таблице пользователей](users.md) для получения дополнительных сведений см. <br/> |
|**UserClientVersionId** <br/> |целое  <br/> |Внешний  <br/> |Версия клиента, используемого пользователем конференции. [Таблица ClientVersions в Скайп для Business Server 2015](clientversions.md) для получения дополнительных сведений см. <br/> |
|**ConfClientVersionId** <br/> |целое  <br/> |Внешний  <br/> |Версия клиента, используемых сервером конференций. [Таблица ClientVersions в Скайп для Business Server 2015](clientversions.md) для получения дополнительных сведений см. <br/> |
|**ReplaceDialogIdTime** <br/> |datetime  <br/> |Внешний  <br/> |Номер идентификатора для идентификации диалоговое окно, в котором был заменен текущего сеанса. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**ReplaceDialogIdSeq** <br/> |целое  <br/> |Внешний  <br/> |Номер идентификатора для идентификации сеанса. Используется в сочетании с **ReplacesDialogIdTime** для уникальной идентификации сеанса, который будет заменен в данном сеансе. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**IsStartedByConfServer** <br/> |бит  <br/> ||Указывает ли сеанс начат сервером конференций.  <br/> |
|**IsEndedByConfServer** <br/> |бит  <br/> ||Указывает, если сеанс завершен сервером конференц-связи.  <br/> |
|**IsUserInternal** <br/> |бит  <br/> ||Ли пользователь вошел в систему из внутренней или нет.  <br/> |
|**ResponseCode** <br/> |целое  <br/> ||Код ответа сеанса Initiation Protocol (SIP) на приглашение на сеанс. Это поле обычно находятся файлы данных, созданный из начального сообщения ПРИГЛАСИТЬ в сеанс. Если сообщения не ПРИГЛАСИТЬ поле заполняется с датой и временем первого соответствующего сообщения SIP (BYE, Отмена, сообщения или сведения).  <br/> |
|**DiagnosticId** <br/> |целое  <br/> ||КОД диагностики из заголовка SIP.  <br/> |
|**Который** <br/> |целое  <br/> |Внешний  <br/> |Идентификатор сервера переднего плана, используемый для данного сеанса. В [таблице серверы](servers.md) для получения дополнительных сведений см. <br/> |
|**PoolId** <br/> |целое  <br/> |Внешний  <br/> |Идентификатор пула, в котором были записаны сеанса. В [таблице пулы](pools.md) для получения дополнительных сведений см. <br/> |
|**MediationServerId** <br/> |целое  <br/> |Внешний  <br/> |Использование сервера-посредника вызова. [Таблица mediationservers](mediationservers.md) для получения дополнительных сведений см. <br/> |
|**GatewayId** <br/> |целое  <br/> |Внешний  <br/> |Использование шлюза вызова. В разделе [Таблица шлюзы в Скайп для Business Server 2015](gateways.md) для получения дополнительных сведений. <br/> |
|**EdgeServerId** <br/> |целое  <br/> |Внешний  <br/> |Пограничного сервера, с помощью вызова. [Таблица edgeservers в Скайп для Business Server 2015](edgeservers.md) для получения дополнительных сведений см. <br/> |
|**ContentTypeId** <br/> |целое  <br/> |Внешний  <br/> |Тип содержимого, используемый в сеансе. В разделе [типы содержимого таблицы в Скайп для Business Server 2015](contenttypes.md) для получения дополнительных сведений. <br/> |
|**InviteTime** <br/> |datetime  <br/> ||Время первого запроса INVITE. Это поле обычно находятся файлы данных, созданный из начального сообщения ПРИГЛАСИТЬ в сеанс. Если сообщения не ПРИГЛАСИТЬ поле заполняется с датой и временем первого соответствующего сообщения SIP (BYE, Отмена, сообщения или сведения).  <br/> |
|**ResponseTime** <br/> |datetime  <br/> ||Время первого ОТВЕТА SIP. Это поле обычно находятся файлы данных, созданный из начального сообщения ПРИГЛАСИТЬ в сеанс. Если сообщения не ПРИГЛАСИТЬ поле заполняется с датой и временем первого соответствующего сообщения SIP (BYE, Отмена, сообщения или сведения).  <br/> |
|**SessionEndTime** <br/> |datetime  <br/> ||Время завершения этого сеанса.  <br/> |
|**UriTypeId** <br/> |tinyint  <br/> |Внешний  <br/> |Содержит значение типа MCU URI в [таблице UriTypes](uritypes.md). Это поле используется для повышения производительности запросов.  <br/> В этом поле было представлено в Microsoft Lync Server 2013.  <br/> |
|**UserFlag** <br/> |smallint  <br/> || Немного значение, которое указывает атрибуты пользователя. Список определений следующих атрибутов: <br/>  Интегрированное со стационарным телефоном — 1 <br/> |
|**CallFlag** <br/> |smallint  <br/> || Немного задано, который указывает атрибуты вызова. Список определений следующих атрибутов: <br/>  Повторный сеанс — 1 <br/> |
|**LastModifiedTime** <br/> |Даты и времени  <br/> ||Для внутреннего использования службой мониторинга.  <br/> В этом поле было представлено в Скайп для Business Server 2015.  <br/> |
   
\*Для большинства сеансов SessionIdSeq будут иметь значение 1. Если нескольких сеансов запускается в точности то же время, SessionIdSeq для одного равно 1, а для другой будет иметь значение 2 и т. д.
