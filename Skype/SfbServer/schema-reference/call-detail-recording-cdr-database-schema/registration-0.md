# <a name="registration-view"></a>Представление регистрации
 
В представлении регистрация хранятся сведения о регистрации пользователей. Это представление было представлено в Lync Server 2013.
  
|**Столбец**|**Тип данных**|**Сведения**|
|:-----|:-----|:-----|
|**сессионидтиме** <br/> |datetime  <br/> |Время запроса сеанса. Используется в сочетании с Сессионидсек для уникальной идентификации сеанса. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**сессионидсек** <br/> |целое  <br/> |ИДЕНТИФИКАЦИОНный номер для идентификации сеанса. Используется в сочетании с Сессионидтиме для уникальной идентификации сеанса. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**регистертиме** <br/> |datetime  <br/> |Время, когда была выполнена регистрация.  <br/> |
|**усерури** <br/> |nvarchar (450)  <br/> |URI пользователя, который зарегистрировался.  <br/> |
|**усеруритипе** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, который зарегистрировался. Для получения дополнительных сведений ознакомьтесь с [таблицей уритипес](uritypes.md) . <br/> |
|**усертенант** <br/> |nvarchar(256)  <br/> |Клиент пользователя, который зарегистрировался. Дополнительные сведения см. в [таблице "клиенты](tenants.md) ". <br/> |
|**ендпоинтид** <br/> |идентификатора  <br/> |Уникальный идентификатор конечной точки пользователя, зарегистрированного в.  <br/> |
|**ендпоинтера** <br/> |идентификатора  <br/> |Уникальный идентификатор, который используется для различения регистраций, включающих одного и того же пользователя и ту же конечную точку.  <br/> |
|**DeRegisterType** <br/> |datetime  <br/> |Время, когда была произведена отмена регистрации.  <br/> |
|**дерегистерреасон** <br/> |nvarchar(256)  <br/> |Причина отмены регистрации.  <br/> |
|**клиентверсион** <br/> |nvarchar(256)  <br/> |Версия клиента, используемая пользователем, который зарегистрировался.  <br/> |
|**клиенттипе** <br/> |целое  <br/> |Клиент, используемый зарегистрированным пользователем. Для получения дополнительных сведений ознакомьтесь с [таблицей усеражентдеф](useragentdef.md) . <br/> |
|**клиенткатегори** <br/> |nvarchar (64)  <br/> |Категория клиента, используемая пользователем, который зарегистрировался.  <br/> |
|**Следующий** <br/> |nvarchar(256)  <br/> |IP-адрес, с которым пользователь зарегистрировался. Это может быть адрес IPv4 или IPv6.  <br/> |
|**диалогид** <br/> |варстринг (775)  <br/> |ИДЕНТИФИКАТОР диалогового окна SIP. Формат:  <br/> диалоговое окно; тег "from-Tag"  <br/> |
|**респонсекоде** <br/> |целое  <br/> |Код ответа SIP на приглашение на сеанс. Это поле обычно заполняется данными, созданными на основе исходного сообщения INVITE в сеансе. Если сообщение приглашения отсутствует, поле заполняется датой и временем первого соответствующего сообщения SIP (пока, CANCEL, MESSAGE или INFO).  <br/> |
|**диагностиЦид** <br/> |целое  <br/> |Идентификатор диагностики, полученный в заголовке SIP.  <br/> |
|**регистратор** <br/> |nvarchar(256)  <br/> |Полное доменное имя регистратора.  <br/> |
|**Pool** <br/> |nvarchar(256)  <br/> |Полное доменное имя пула, который захватывает данные для сеанса.  <br/> |
|**Пограничный сервер** <br/> |nvarchar(256)  <br/> |Полное доменное имя пограничного сервера, используемое пользователем, который зарегистрировался.  <br/> |
|**Internal (внутренний)** <br/> |бит  <br/> |Указывает, вошел ли пользователь из внутренней сети.  <br/> |
|**исусерсервицеаваилабле** <br/> |бит  <br/> |Указывает, был ли Усерсервице доступен на момент регистрации.  <br/> |
|**испримарирегистрар** <br/> |бит  <br/> |Указывает, была ли регистрация основным регистратором.  <br/> |
|**девицемакаддресс** <br/> |bigint  <br/> |MAC-адрес устройства зарегистрирован.  <br/> |
|**девицемануфактурер** <br/> |nvarchar(256)  <br/> |Производитель зарегистрированного устройства. Более подробную информацию вы увидите [в таблице изготовителей в Skype для бизнеса Server 2015](manufacturers.md) . <br/> |
|**девицехардвареверсион** <br/> |nvarchar(256)  <br/> |Аппаратная версия зарегистрированного устройства. Для получения дополнительных сведений ознакомьтесь с [таблицей хардвареверсионс в Skype для бизнеса Server 2015](hardwareversions.md) . <br/> |
