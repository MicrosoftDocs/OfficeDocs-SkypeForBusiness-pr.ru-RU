# <a name="sessiondetails-table"></a>Таблица SessionDetails
 
Каждая запись представляет одноранговый сеанс, которым может быть телефонный звонок VoIP-VoIP, двухсторонний сеанс обмена мгновенными сообщениями или друг тип сеанса. Вы можете объединить таблицу с таблицей [Media,](media.md) чтобы найти сведения о каждом мультимедиа, задействованном в этом сеансе.
  
Обратите внимание, что поля IsUser1IntegratedWithDeskPhone и IsUser2IntegratedWithDeskPhone были исключены из таблицы SessionDetails, используемой в Skype для бизнеса Server 2015.
  
|**Столбец**|**Тип данных**|**Ключ/индекс**|**Details**|
|:-----|:-----|:-----|:-----|
|**SessionIdTime** <br/> |datetime  <br/> |Основной, внешний  <br/> |Время запроса сеанса. В сочетании с параметром **SessionIdSeq** определяет сеанс уникальным образом. Дополнительные сведения см. в таблице [Dialogs в Skype для бизнеса Server 2015.](dialogs.md) <br/> |
|**SessionIdSeq** <br/> |int  <br/> |Основной, внешний  <br/> |Идентификатор для идентификации сеанса. Используется в сочетании с **SessionIdTime** для уникальной идентификации сеанса.* Дополнительные сведения см. в таблице Dialogs в Skype для бизнеса [Server 2015.](dialogs.md) <br/> |
|**CorrelationId** <br/> |uniqueidentifier  <br/> ||GUID для корреляции нескольких сеансов.  <br/> |
|**ReplaceDialogIdTime** <br/> |datetime  <br/> |Внешняя  <br/> |Идентификационный номер для определения диалога, замененного текущим сеансом. Дополнительные сведения см. в таблице [Dialogs в Skype для бизнеса Server 2015.](dialogs.md) <br/> |
|**ReplaceDialogIdSeq** <br/> |int  <br/> |Внешняя  <br/> |Идентификатор для идентификации сеанса. Используется в сочетании с **ReplacesDialogIdTime** для уникальной идентификации сеанса. Дополнительные сведения см. в таблице [Dialogs в Skype для бизнеса Server 2015.](dialogs.md) <br/> |
|**User1Id** <br/> |int  <br/> |Внешняя  <br/> |Идентификатор одного пользователя в сеансе. Дополнительные [сведения см.](users.md) в таблице "Пользователи". <br/> |
|**User2Id** <br/> |int  <br/> |Внешняя  <br/> |Идентификатор другого пользователя в сеансе. Дополнительные [сведения см.](users.md) в таблице "Пользователи". <br/> |
|**User1EndpointId** <br/> |uniqueIdentifier  <br/> ||GUID, идентифицирующий конечную точку, применяемую первым пользователем в сеансе.  <br/> Это поле было впервые введено в Microsoft Lync Server 2013.  <br/> |
|**User2EndpointId** <br/> |uniqueIdentifier  <br/> ||GUID, идентифицирующий конечную точку, применяемую вторым пользователем в сеансе.  <br/> Это поле было впервые введено в Microsoft Lync Server 2013.  <br/> |
|**TargetUserId** <br/> |int  <br/> |Внешняя  <br/> |URI-адрес исходного вызываемого пользователя в SIP-запросе. дополнительные [сведения см.](users.md) в таблице "Пользователи". <br/> |
|**SessionStartedById** <br/> |int  <br/> |Внешняя  <br/> |Идентификатор пользователя, инициировавшего сеанс. Дополнительные [сведения см.](users.md) в таблице "Пользователи". <br/> |
|**OnBehalfOfId** <br/> |int  <br/> |Внешняя  <br/> |Определяет идентификатор пользователя, которого представляет вызывающий абонент. Дополнительные [сведения см.](users.md) в таблице "Пользователи". <br/> |
|**ReferredById** <br/> |int  <br/> |Внешняя  <br/> |Идентификатор пользователя, который ссылается на вызов. Дополнительные [сведения см.](users.md) в таблице "Пользователи". <br/> |
|**ServerId** <br/> |int  <br/> |Внешняя  <br/> |Идентификатор сервера переднего плана, используемого для этого сеанса. Дополнительные [сведения см. в](servers.md) таблице "Серверы". <br/> |
|**PoolId** <br/> |int  <br/> |Внешняя  <br/> |Идентификатор пула, в котором был записан сеанс. Дополнительные [сведения см. в](pools.md) таблице "Пулы". <br/> |
|**ContentTypeID** <br/> |int  <br/> |Внешняя  <br/> |Тип контента, используемый в сеансе. Дополнительные сведения см. в таблице [ContentTypes в Skype для бизнеса Server 2015.](contenttypes.md) <br/> |
|**User1ClientVerId** <br/> |int  <br/> |Внешняя  <br/> |Версия клиента, применяемая пользователем User1. Дополнительные сведения см. в таблице [ClientVersions в Skype для бизнеса Server 2015.](clientversions.md) <br/> |
|**User2ClientVerId** <br/> |int  <br/> |Внешняя  <br/> |Версия клиента, применяемая пользователем User2. Дополнительные сведения см. в таблице [ClientVersions в Skype для бизнеса Server 2015.](clientversions.md) <br/> |
|**User1EdgeServerid** <br/> |int  <br/> |Внешняя  <br/> |Пограничный сервер, применяемый пользователем User1. Дополнительные сведения см. в таблице [EdgeServers в Skype для бизнеса Server 2015.](edgeservers.md) <br/> |
|**User2EdgeServerid** <br/> |int  <br/> |Внешняя  <br/> |Пограничный сервер, применяемый пользователем User2. Дополнительные сведения см. в таблице [EdgeServers в Skype для бизнеса Server 2015.](edgeservers.md) <br/> |
|**IsUser1Internal** <br/> |bit  <br/> ||Вошел ли пользователь User1 из внутренней сети или нет.  <br/> |
|**IsUser2Internal** <br/> |bit  <br/> ||Вошел ли пользователь User2 из внутренней сети или нет.  <br/> |
|**InviteTime** <br/> |datetime  <br/> ||Время первого запроса INVITE. Это поле обычно заполняется данными, генерируемыми из исходного сообщения INVITE в сеансе. Если сообщения INVITE нет, поле заполняется датой и временем первого релевантного SIP-сообщения (BYE, CANCEL, MESSAGE или INFO). Это поле обычно заполняется данными, генерируемыми из исходного сообщения INVITE в сеансе. Если сообщения INVITE нет, поле заполняется датой и временем первого релевантного SIP-сообщения (BYE, CANCEL, MESSAGE или INFO).  <br/> |
|**ResponseTime** <br/> |datetime  <br/> ||Время ответа на первое сообщение INVITE. Обычно это поле содержит данные из первоначального сообщения INVITE в сеансе. Если сообщение INVITE отсутствует, в поле записывается дата и время первого соответствующего SIP-сообщения (BYE, CANCEL, MESSAGE или INFO).  <br/> |
|**ResponseCode** <br/> |int  <br/> ||Код SIP-ответа на приглашение в сеанс. Это поле обычно заполняется данными, генерируемыми из исходного сообщения INVITE в сеансе. Если сообщения INVITE нет, поле заполняется датой и временем первого релевантного SIP-сообщения (BYE, CANCEL, MESSAGE или INFO).  <br/> |
|**DiagnosticId** <br/> |int  <br/> ||Идентификатор диагностики из заголовка SIP.  <br/> |
|**CallPriority** <br/> |int  <br/> |Внешняя  <br/> |Приоритет звонка. Дополнительные сведения см. в таблице [CallPriorities в Skype для бизнеса Server 2015.](callpriorities.md) <br/> |
|**User1MessageCount** <br/> |int  <br/> ||Число сообщений, отправленных пользователем User1 в сеансе.  <br/> |
|**User2MessageCount** <br/> |int  <br/> ||Число сообщений, отправленных пользователем User2 в сеансе.  <br/> |
|**SessionEndTime** <br/> |datetime  <br/> ||Время окончания сеанса.  <br/> |
|**MediaTypes** <br/> |int  <br/> ||Бит, указывающий тип носителя этого сеанса. Дале перечислены определения типов:  <br/> 1- мгновенные изве-ва  <br/> 2- FILE_TRANSFER  <br/> 4- REMOTE_ASSISTANCE  <br/> 8- APP_SHARING  <br/> 16- АУДИО  <br/> 32- ВИДЕО  <br/> 64- APP_INVITE  <br/> |
|**User1Flag** <br/> |smallint  <br/> ||Бит, обозначающий атрибуты пользователя User1. Далее перечислены определения атрибутов:  <br/> 0x01 — интегрировано со стационарным телефоном  <br/> |
|**User2Flag** <br/> |smallint  <br/> ||Бит, обозначающий атрибуты пользователя User2. Далее перечислены определения атрибутов:  <br/> 0x01 — интегрировано со стационарным телефоном  <br/> |
|**CallFlag** <br/> |smallint  <br/> ||Бит, обозначающий атрибуты пользователя User2. Далее перечислены определения атрибутов:  <br/> 0x01 — повторенный сеанс  <br/> 0x02 — вызов, выполненный агентом от имени группы ответа  <br/> |
|**Обработано** <br/> |bit  <br/> ||Для внутреннего использования службой мониторинга.  <br/> Это поле было впервые введено в Microsoft Lync Server 2013.  <br/> |
|**LastModifiedTime** <br/> |Datetime  <br/> ||Для внутреннего использования службой мониторинга.  <br/> Это поле было впервые введено в Skype для бизнеса Server 2015.  <br/> |
   
\* Для большинства сеансов значение SessionIdSeq будет иметь значение 1. Если несколько сеансов начинаются в одно и то же время, для одного из них SessionIdSeq будет иметь значение 1, для другого — 2 и т. д.
