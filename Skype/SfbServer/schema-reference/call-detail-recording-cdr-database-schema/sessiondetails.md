# <a name="sessiondetails-table"></a>Таблица SessionDetails
 
Каждая запись представляет одноранговый сеанс, которым может быть телефонный звонок VoIP-VoIP, двухсторонний сеанс обмена мгновенными сообщениями или друг тип сеанса. Вы можете выполнить соединяемую таблицу со таблицей [Мультимедиа,](media.md) чтобы найти сведения о каждом мультимедиа, участвующих в этом сеансе.
  
Обратите внимание, что поля IsUser1IntegratedWithDeskPhone и isUser2IntegratedWithDeskPhone были исключены из таблицы SessionDetails, используемой в Skype для бизнеса Server 2015 г.
  
|**Столбец**|**Тип данных**|**Key/Index**|**Details**|
|:-----|:-----|:-----|:-----|
|**SessionIdTime** <br/> |datetime  <br/> |Основной, внешний  <br/> |Время запроса сеанса. В сочетании с параметром **SessionIdSeq** определяет сеанс уникальным образом. Дополнительные сведения см. в таблице [Диалоги Skype для бизнеса Server 2015](dialogs.md) г. <br/> |
|**SessionIdSeq** <br/> |int  <br/> |Основной, внешний  <br/> |Идентификатор для идентификации сеанса. Дополнительные сведения см. в таблице диалогов в Skype для бизнеса Server [2015](dialogs.md) г., используемой совместно с **SessionIdTime.** <br/> |
|**CorrelationId** <br/> |uniqueidentifier  <br/> ||GUID для корреляции нескольких сеансов.  <br/> |
|**ReplaceDialogIdTime** <br/> |datetime  <br/> |Foreign  <br/> |Идентификационный номер для определения диалога, замененного текущим сеансом. Дополнительные сведения см. в таблице [Диалоги Skype для бизнеса Server 2015](dialogs.md) г. <br/> |
|**ReplaceDialogIdSeq** <br/> |int  <br/> |Foreign  <br/> |Идентификатор для идентификации сеанса. Используется в сочетании с **ReplacesDialogIdTime** для уникальной идентификации сеанса. Дополнительные сведения см. в таблице [Диалоги Skype для бизнеса Server 2015](dialogs.md) г. <br/> |
|**User1Id** <br/> |int  <br/> |Foreign  <br/> |Идентификатор одного пользователя в сеансе. Дополнительные [сведения см.](users.md) в таблице Пользователи. <br/> |
|**User2Id** <br/> |int  <br/> |Foreign  <br/> |Идентификатор другого пользователя в сеансе. Дополнительные [сведения см.](users.md) в таблице Пользователи. <br/> |
|**User1EndpointId** <br/> |uniqueIdentifier  <br/> ||GUID, идентифицирующий конечную точку, применяемую первым пользователем в сеансе.  <br/> Это поле было внедрено в Microsoft Lync Server 2013.  <br/> |
|**User2EndpointId** <br/> |uniqueIdentifier  <br/> ||GUID, идентифицирующий конечную точку, применяемую вторым пользователем в сеансе.  <br/> Это поле было внедрено в Microsoft Lync Server 2013.  <br/> |
|**TargetUserId** <br/> |int  <br/> |Foreign  <br/> |URI-адрес исходного вызываемого пользователя в SIP-запросе. дополнительные [сведения см.](users.md) в таблице Пользователи. <br/> |
|**SessionStartedById** <br/> |int  <br/> |Foreign  <br/> |Идентификатор пользователя, инициировавшего сеанс. Дополнительные [сведения см.](users.md) в таблице Пользователи. <br/> |
|**OnBehalfOfId** <br/> |int  <br/> |Foreign  <br/> |Определяет идентификатор пользователя, которого представляет вызывающий абонент. Дополнительные [сведения см.](users.md) в таблице Пользователи. <br/> |
|**ReferredById** <br/> |int  <br/> |Foreign  <br/> |Идентификатор пользователя, который ссылается на вызов. Дополнительные [сведения см.](users.md) в таблице Пользователи. <br/> |
|**ServerId** <br/> |int  <br/> |Foreign  <br/> |Идентификатор сервера переднего плана, используемого для этого сеанса. Дополнительные сведения см. в таблице [Servers.](servers.md) <br/> |
|**PoolId** <br/> |int  <br/> |Foreign  <br/> |Идентификатор пула, в котором был записан сеанс. Дополнительные [сведения см. в](pools.md) таблице Пулы. <br/> |
|**ContentTypeID** <br/> |int  <br/> |Foreign  <br/> |Тип контента, используемый в сеансе. Дополнительные сведения см. в таблице [ContentTypes Skype для бизнеса Server 2015](contenttypes.md) г. <br/> |
|**User1ClientVerId** <br/> |int  <br/> |Foreign  <br/> |Версия клиента, применяемая пользователем User1. Дополнительные сведения см. в таблице [ClientVersions Skype для бизнеса Server 2015](clientversions.md) г. <br/> |
|**User2ClientVerId** <br/> |int  <br/> |Foreign  <br/> |Версия клиента, применяемая пользователем User2. Дополнительные сведения см. в таблице [ClientVersions Skype для бизнеса Server 2015](clientversions.md) г. <br/> |
|**User1EdgeServerid** <br/> |int  <br/> |Foreign  <br/> |Пограничный сервер, применяемый пользователем User1. Дополнительные сведения см. в таблице [EdgeServers Skype для бизнеса Server 2015](edgeservers.md) г. <br/> |
|**User2EdgeServerid** <br/> |int  <br/> |Foreign  <br/> |Пограничный сервер, применяемый пользователем User2. Дополнительные сведения см. в таблице [EdgeServers Skype для бизнеса Server 2015](edgeservers.md) г. <br/> |
|**IsUser1Internal** <br/> |bit  <br/> ||Вошел ли пользователь User1 из внутренней сети или нет.  <br/> |
|**IsUser2Internal** <br/> |bit  <br/> ||Вошел ли пользователь User2 из внутренней сети или нет.  <br/> |
|**InviteTime** <br/> |datetime  <br/> ||Время первого запроса INVITE. Это поле обычно заполняется данными, генерируемыми из исходного сообщения INVITE в сеансе. Если сообщения INVITE нет, поле заполняется датой и временем первого релевантного SIP-сообщения (BYE, CANCEL, MESSAGE или INFO). Это поле обычно заполняется данными, генерируемыми из исходного сообщения INVITE в сеансе. Если сообщения INVITE нет, поле заполняется датой и временем первого релевантного SIP-сообщения (BYE, CANCEL, MESSAGE или INFO).  <br/> |
|**ResponseTime** <br/> |datetime  <br/> ||Время ответа на первое сообщение INVITE. Обычно это поле содержит данные из первоначального сообщения INVITE в сеансе. Если сообщение INVITE отсутствует, в поле записывается дата и время первого соответствующего SIP-сообщения (BYE, CANCEL, MESSAGE или INFO).  <br/> |
|**ResponseCode** <br/> |int  <br/> ||Код SIP-ответа на приглашение в сеанс. Это поле обычно заполняется данными, генерируемыми из исходного сообщения INVITE в сеансе. Если сообщения INVITE нет, поле заполняется датой и временем первого релевантного SIP-сообщения (BYE, CANCEL, MESSAGE или INFO).  <br/> |
|**DiagnosticId** <br/> |int  <br/> ||Идентификатор диагностики из заголовка SIP.  <br/> |
|**CallPriority** <br/> |int  <br/> |Foreign  <br/> |Приоритет звонка. Дополнительные сведения см. в таблице [CallPriorities Skype для бизнеса Server 2015](callpriorities.md) г. <br/> |
|**User1MessageCount** <br/> |int  <br/> ||Число сообщений, отправленных пользователем User1 в сеансе.  <br/> |
|**User2MessageCount** <br/> |int  <br/> ||Число сообщений, отправленных пользователем User2 в сеансе.  <br/> |
|**SessionEndTime** <br/> |datetime  <br/> ||Время окончания сеанса.  <br/> |
|**MediaTypes** <br/> |int  <br/> ||Бит, указывающий тип носителя этого сеанса. Дале перечислены определения типов:  <br/> 1- IM  <br/> 2- FILE_TRANSFER  <br/> 4- REMOTE_ASSISTANCE  <br/> 8- APP_SHARING  <br/> 16- АУДИО  <br/> 32- ВИДЕО  <br/> 64- APP_INVITE  <br/> |
|**User1Flag** <br/> |smallint  <br/> ||Бит, обозначающий атрибуты пользователя User1. Далее перечислены определения атрибутов:  <br/> 0x01 — интегрировано со стационарным телефоном  <br/> |
|**User2Flag** <br/> |smallint  <br/> ||Бит, обозначающий атрибуты пользователя User2. Далее перечислены определения атрибутов:  <br/> 0x01 — интегрировано со стационарным телефоном  <br/> |
|**CallFlag** <br/> |smallint  <br/> ||Бит, обозначающий атрибуты пользователя User2. Далее перечислены определения атрибутов:  <br/> 0x01 — повторенный сеанс  <br/> 0x02 — вызов, выполненный агентом от имени группы ответа  <br/> |
|**Обработано** <br/> |bit  <br/> ||Для внутреннего использования службой мониторинга.  <br/> Это поле было внедрено в Microsoft Lync Server 2013.  <br/> |
|**LastModifiedTime** <br/> |Datetime  <br/> ||Для внутреннего использования службой мониторинга.  <br/> Это поле было введено Skype для бизнеса Server 2015 г.  <br/> |
   
\* Для большинства сеансов значение SessionIdSeq будет 1. Если несколько сеансов начинаются в одно и то же время, для одного из них SessionIdSeq будет иметь значение 1, для другого — 2 и т. д.
