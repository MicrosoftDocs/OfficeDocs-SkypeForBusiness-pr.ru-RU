# <a name="conferencesessiondetails-view"></a>Представление ConferenceSessionDetails
 
Представление ConferenceSessionDetails сохранение информации о многопользовательских сеансов. Каждая запись представляет один сеанс конференции, который может быть либо сеансом с центром конференций, либо сеансом с конкретным сервером конференций. В этом представлении была введена в Microsoft Lync Server 2013.
  
|**Столбец**|**Тип данных**|**Сведения**|
|:-----|:-----|:-----|
|**SessionIdTime** <br/> |datetime  <br/> |Время запроса сеанса. Используется совместно с SessionIdSeq для уникальной идентификации сеанса. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**SessionIdSeq** <br/> |целое  <br/> |Номер идентификатора для идентификации сеанса. Используется в сочетании с SessionIdTime для уникальной идентификации сеанса. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**InviteTime** <br/> |datetime  <br/> |Время первого запроса INVITE. Это поле обычно находятся файлы данных, созданный из начального сообщения ПРИГЛАСИТЬ в сеанс. Если сообщения не ПРИГЛАСИТЬ поле заполняется с датой и временем первого соответствующего сообщения SIP (BYE, Отмена, сообщения или сведения).  <br/> |
|**ConferenceUri** <br/> |nvarchar(450)  <br/> |URI конференции.  <br/> |
|**ConferenceUriType** <br/> |nvarchar(256)  <br/> |Тип URI конференции. В [таблице UriTypes](uritypes.md) для получения дополнительных сведений см. <br/> |
|**ConfInstance** <br/> |uniqueidentifier  <br/> |Идентификатор, между экземплярами повторяющихся конференций, отделяя его. Каждый экземпляр повторяющейся конференции имеет одинаковые параметр ConferenceURI, но разные значения ConfInstance.  <br/> |
|**McuConferenceUri** <br/> |nvarchar(450)  <br/> |URI сервера конференц-связи.  <br/> |
|**McuConferenceUriType** <br/> |nvarchar(256)  <br/> |Тип URI сервера конференц-связи. В [таблице UriTypes](uritypes.md) для получения дополнительных сведений см. <br/> |
|**UserUri** <br/> |nvarchar(450)  <br/> |URI пользователя, участвующего в сеансе.  <br/> |
|**UserUriType** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, который являлся участником сеанса. В [таблице UriTypes](uritypes.md) для получения дополнительных сведений см. <br/> |
|**UserTenant** <br/> |nvarchar(256)  <br/> |Клиент пользователя, который являлся участником сеанса. В [таблице клиентов](tenants.md) для получения дополнительных сведений см. <br/> |
|**UserEndpointId** <br/> |uniqueidentifier  <br/> |Уникальный идентификатор пользователя, который являлся участником сеанса.  <br/> |
|**EndTime** <br/> |datetime  <br/> |Время окончания сеанса.  <br/> |
|**ConferenceClientVersion** <br/> |nvarchar(256)  <br/> |Версия сервера конференц-связи.  <br/> |
|**ConferenceClientType** <br/> |целое  <br/> |Тип сервера конференц-связи. [Таблица useragentdef](useragentdef.md) для получения дополнительных сведений см. <br/> |
|**ConferenceCategory** <br/> |nvarchar(64)  <br/> |Категория сервера конференций.  <br/> |
|**UserClientVersion** <br/> |nvarchar(256)  <br/> |Версия клиента, используемого пользователем, который являлся участником сеанса.  <br/> |
|**UserClientType** <br/> |целое  <br/> |Клиент, используемый пользователем, который являлся участником сеанса. [Таблица useragentdef](useragentdef.md) для получения дополнительных сведений см. <br/> |
|**UserClientCategory** <br/> |nvarchar(64)  <br/> |Имя категории клиента, используемого пользователем, который являлся частью сеанса.  <br/> |
|**OnBehalfOfUri** <br/> |nvarchar(450)  <br/> |URI пользователя, от чьего имени был запущен сеанс.  <br/> |
|**OnBehalfOfUriType** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, от чьего имени был запущен сеанс. В [таблице UriTypes](uritypes.md) для получения дополнительных сведений см. <br/> |
|**OnBehalfOfTenant** <br/> |nvarchar(256)  <br/> |Клиент пользователя, от чьего имени был запущен сеанс. В [таблице клиентов](tenants.md) для получения дополнительных сведений см. <br/> |
|**ReferredByUri** <br/> |nvarchar(450)  <br/> |URI пользователя, который ссылался на сеанс.  <br/> |
|**ReferredByUriType** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, который ссылался на сеанс. В [таблице UriTypes](uritypes.md) для получения дополнительных сведений см. <br/> |
|**ReferredByUriTenant** <br/> |nvarchar(256)  <br/> |Клиент пользователя, который ссылался на сеанс. В [таблице клиентов](tenants.md) для получения дополнительных сведений см. <br/> |
|**DialogId** <br/> |varstring(775)  <br/> |SIP-идентификатор диалогового окна. Имеет формат  <br/> : диалоговое окно, из тега; для тега  <br/> |
|**ReplaceDialogIdTime** <br/> |datetime  <br/> |Номер идентификатора для идентификации диалоговое окно, в котором был заменен текущего сеанса. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**ReplaceDialogIdSeq** <br/> |целое  <br/> |Номер идентификатора для идентификации сеанса. Используется в сочетании с ReplaceDialogIdTime для уникальной идентификации сеанса, который будет заменен в данном сеансе. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**ReplacesDialogId** <br/> |varchar(775)  <br/> |Идентификатор диалогового окна сеанса заменяет по протоколу SIP. Формат — это:  <br/> диалоговое окно, из тега; для тега  <br/> |
|**IsStartedByConfServer** <br/> |бит  <br/> |Указывает, будет ли сервером конференц-связи был запущен сеанс.  <br/> |
|**IsEndedByConfServer** <br/> |бит  <br/> |Указывает, был ли сеанс завершен сервером конференц-связи.  <br/> |
|**IsUserInternal** <br/> |бит  <br/> |Указывает, является ли пользователь вошел в систему из внутренней сети.  <br/> |
|**ResponseTime** <br/> |datetime  <br/> |Время отклика для первого сообщения INVITE. Это поле обычно находятся файлы данных, созданный из начального сообщения ПРИГЛАСИТЬ в сеанс. Если сообщения не ПРИГЛАСИТЬ поле заполняется с датой и временем первого соответствующего сообщения SIP (BYE, Отмена, сообщения или сведения).  <br/> |
|**ResponseCode** <br/> |целое  <br/> |Код ответа SIP на приглашение на сеанс. Это поле обычно находятся файлы данных, созданный из начального сообщения ПРИГЛАСИТЬ в сеанс. Если сообщения не ПРИГЛАСИТЬ поле заполняется с датой и временем первого соответствующего сообщения SIP (BYE, Отмена, сообщения или сведения).  <br/> |
|**DiagnosticId** <br/> |целое  <br/> |КОД диагностики из заголовков SIP сеанса.  <br/> |
|**ContentType** <br/> |nvarchar(256)  <br/> |Тип контента для сеанса.  <br/> |
|**FrontEnd** <br/> |nvarchar(256)  <br/> |Полное доменное имя сервера переднего плана, который получил данные этого сеанса.  <br/> |
|**Pool** <br/> |nvarchar(256)  <br/> |Полное доменное имя пула, который получил данные этого сеанса.  <br/> |
|**MediationServer** <br/> |nvarchar(256)  <br/> |Сервер-посредник пользователя, который являлся участником сеанса.  <br/> |
|**Шлюз** <br/> |nvarchar(256)  <br/> |Шлюз, используемый пользователем, который являлся сеанса.  <br/> |
|**Пограничный сервер** <br/> |nvarchar(256)  <br/> |Полное доменное имя пограничного сервера, используемого пользователем, который являлся участником сеанса.  <br/> |
|**UserFlag** <br/> |smallint  <br/> |Указывает атрибуты пользователя, который являлся участником сеанса. Разрешены следующие определения атрибутов:  <br/> 0x01 — интегрировано со стационарным телефоном  <br/> |
|**CallFlag** <br/> |smallint  <br/> |Указывает атрибуты вызова. Допускаются следующие определения атрибутов:  <br/> 0x01 — Списанный сеанс 0  <br/> x02 — вызов, выполненный агентом от имени группы ответа  <br/> |
