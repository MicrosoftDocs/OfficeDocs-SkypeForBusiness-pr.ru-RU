# <a name="conferencesessiondetails-view"></a>Представление Конференцесессиондетаилс
 
В представлении Конференцесессиондетаилс хранятся сведения о многокомпонентных сеансах. Каждая запись представляет один сеанс конференции, который может быть либо сеансом с фокусом, либо сеансом с определенным сервером конференц-связи. Это представление было представлено в Microsoft Lync Server 2013.
  
|**Столбец**|**Тип данных**|**Сведения**|
|:-----|:-----|:-----|
|**Сессионидтиме** <br/> |datetime  <br/> |Время запроса сеанса. Используется в сочетании с Сессионидсек для уникальной идентификации сеанса. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**Сессионидсек** <br/> |целое  <br/> |ИДЕНТИФИКАЦИОНный номер для идентификации сеанса. Используется в сочетании с Сессионидтиме для уникальной идентификации сеанса. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**Инвитетиме** <br/> |datetime  <br/> |Время первого запроса приглашения. Это поле обычно заполняется данными, созданными на основе исходного сообщения INVITE в сеансе. Если сообщение приглашения отсутствует, поле заполняется датой и временем первого соответствующего сообщения SIP (пока, CANCEL, MESSAGE или INFO).  <br/> |
|**ConferenceUri** <br/> |nvarchar (450)  <br/> |Универсальный код ресурса (URI) Конференции.  <br/> |
|**Конференцеуритипе** <br/> |nvarchar(256)  <br/> |Тип URI конференции. Для получения дополнительных сведений ознакомьтесь с [таблицей уритипес](uritypes.md) . <br/> |
|**Конфинстанце** <br/> |идентификатора  <br/> |Идентификатор, отличающийся между экземплярами повторяющихся конференций. Каждый экземпляр повторяющейся Конференции имеет один и тот же Конференцеури, но другое значение Конфинстанце.  <br/> |
|**Мкуконференцеури** <br/> |nvarchar (450)  <br/> |Универсальный код ресурса (URI) сервера конференц-связи.  <br/> |
|**Мкуконференцеуритипе** <br/> |nvarchar(256)  <br/> |Тип URI сервера конференц-связи. Для получения дополнительных сведений ознакомьтесь с [таблицей уритипес](uritypes.md) . <br/> |
|**Усерури** <br/> |nvarchar (450)  <br/> |Универсальный код ресурса (URI) пользователя, участвующего в сеансе.  <br/> |
|**Усеруритипе** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, который является частью сеанса. Для получения дополнительных сведений ознакомьтесь с [таблицей уритипес](uritypes.md) . <br/> |
|**Усертенант** <br/> |nvarchar(256)  <br/> |Клиент пользователя, который является частью сеанса. Дополнительные сведения см. в [таблице "клиенты](tenants.md) ". <br/> |
|**Усерендпоинтид** <br/> |идентификатора  <br/> |Уникальный идентификатор пользователя, который является частью сеанса.  <br/> |
|**EndTime** <br/> |datetime  <br/> |Время окончания сеанса.  <br/> |
|**Конференцеклиентверсион** <br/> |nvarchar(256)  <br/> |Версия сервера конференций.  <br/> |
|**Конференцеклиенттипе** <br/> |целое  <br/> |Тип сервера конференций. Для получения дополнительных сведений ознакомьтесь с [таблицей усеражентдеф](useragentdef.md) . <br/> |
|**Конференцекатегори** <br/> |nvarchar (64)  <br/> |Категория "сервер конференции".  <br/> |
|**Усерклиентверсион** <br/> |nvarchar(256)  <br/> |Версия клиента, используемая пользователем, который принимал участие в сеансе.  <br/> |
|**Усерклиенттипе** <br/> |целое  <br/> |Клиент, используемый пользователем, который участвовал в сеансе. Для получения дополнительных сведений ознакомьтесь с [таблицей усеражентдеф](useragentdef.md) . <br/> |
|**Усерклиенткатегори** <br/> |nvarchar (64)  <br/> |Имя категории клиента, используемой пользователем, который является частью сеанса.  <br/> |
|**Онбехалфофури** <br/> |nvarchar (450)  <br/> |Универсальный код ресурса (URI) пользователя, от имени которого был запущен сеанс.  <br/> |
|**Онбехалфофуритипе** <br/> |nvarchar(256)  <br/> |Тип универсального кода ресурса (URI) пользователя, от имени которого был запущен сеанс. Для получения дополнительных сведений ознакомьтесь с [таблицей уритипес](uritypes.md) . <br/> |
|**Онбехалфофтенант** <br/> |nvarchar(256)  <br/> |Клиент пользователя, от имени которого был запущен сеанс. Дополнительные сведения см. в [таблице "клиенты](tenants.md) ". <br/> |
|**Реферредбюри** <br/> |nvarchar (450)  <br/> |URI пользователя, который ссылался на сеанс.  <br/> |
|**Реферредбюритипе** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, который ссылался на сеанс. Для получения дополнительных сведений ознакомьтесь с [таблицей уритипес](uritypes.md) . <br/> |
|**Реферредбюритенант** <br/> |nvarchar(256)  <br/> |Клиент пользователя, который ссылался на сеанс. Дополнительные сведения см. в [таблице "клиенты](tenants.md) ". <br/> |
|**Диалогид** <br/> |варстринг (775)  <br/> |ИДЕНТИФИКАТОР диалогового окна SIP. Формат  <br/> :d иалог; to-Tag  <br/> |
|**Реплацедиалогидтиме** <br/> |datetime  <br/> |ИДЕНТИФИКАЦИОНный номер, определяющий диалоговое окно, которое было заменено текущим сеансом. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**Реплацедиалогидсек** <br/> |целое  <br/> |ИДЕНТИФИКАЦИОНный номер для идентификации сеанса. Используется в сочетании с Реплацедиалогидтиме для уникальной идентификации сеанса, который заменяется этим сеансом. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**Реплацесдиалогид** <br/> |varchar (775)  <br/> |ИДЕНТИФИКАТОР диалогового окна SIP, в котором будет заменяться сеанс. Формат:  <br/> диалоговое окно; тег "from-Tag"  <br/> |
|**Исстартедбиконфсервер** <br/> |бит  <br/> |Указывает, был ли запущен сервер конференц-связи.  <br/> |
|**Исендедбиконфсервер** <br/> |бит  <br/> |Указывает, завершен ли сеанс на сервере конференций.  <br/> |
|**Исусеринтернал** <br/> |бит  <br/> |Указывает, вошел ли пользователь из внутренней сети.  <br/> |
|**Респонсетиме** <br/> |datetime  <br/> |Время ответа на первое сообщение приглашения. Это поле обычно заполняется данными, созданными на основе исходного сообщения INVITE в сеансе. Если сообщение приглашения отсутствует, поле заполняется датой и временем первого соответствующего сообщения SIP (пока, CANCEL, MESSAGE или INFO).  <br/> |
|**Респонсекоде** <br/> |целое  <br/> |Код ответа SIP на приглашение на сеанс. Это поле обычно заполняется данными, созданными на основе исходного сообщения INVITE в сеансе. Если сообщение приглашения отсутствует, поле заполняется датой и временем первого соответствующего сообщения SIP (пока, CANCEL, MESSAGE или INFO).  <br/> |
|**ДиагностиЦид** <br/> |целое  <br/> |Идентификатор диагностики, полученный от заголовков SIP сеанса.  <br/> |
|**Контента** <br/> |nvarchar(256)  <br/> |Тип контента для сеанса.  <br/> |
|**FrontEnd** <br/> |nvarchar(256)  <br/> |Полное доменное имя сервера переднего плана, который захватил данные для сеанса.  <br/> |
|**Pool** <br/> |nvarchar(256)  <br/> |Полное доменное имя пула, который захватывает данные для сеанса.  <br/> |
|**Медиатионсервер** <br/> |nvarchar(256)  <br/> |Сервер исправлений, используемый пользователем, который принимал участие в сеансе.  <br/> |
|**Шлюз** <br/> |nvarchar(256)  <br/> |Шлюз, используемый пользователем, который участвовал в сеансе.  <br/> |
|**Пограничный сервер** <br/> |nvarchar(256)  <br/> |Полное доменное имя пограничного сервера, используемое пользователем, который принимал участие в сеансе.  <br/> |
|**Усерфлаг** <br/> |smallint  <br/> |Указывает атрибуты пользователя, который принимал участие в сеансе. Допустимы следующие определения атрибутов:  <br/> 0x01 — интеграция с настольным телефоном  <br/> |
|**Каллфлаг** <br/> |smallint  <br/> |Указывает атрибуты вызова. Допустимы следующие определения атрибутов:  <br/> 0x01 — повторное попыток Session0  <br/> x02 — вызов, сделанный агентом от имени группы ответа  <br/> |
