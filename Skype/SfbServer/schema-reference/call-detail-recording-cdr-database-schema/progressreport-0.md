# <a name="progressreport-view"></a>Представление Прогрессрепорт
 
В представлении Прогрессрепорт хранятся сведения о завершенных сеансах. Отчеты о ходе выполнения будут записываться только для звонков и сеансов, которые определяет Lync Server 2013, может быть полезен в целях диагностики. Это представление было представлено в Microsoft Lync Server 2013.
  
> [!NOTE]
> Поля Еррортиме, Ерроррепортсек и Прогрессрепортсек не обязательно ссылаются на ошибки, а также на сообщения, указывающие на состояние вызовов или сообщений. 
  
|**Столбец**|**Тип данных**|**Сведения**|
|:-----|:-----|:-----|
|**еррортиме** <br/> |datetime  <br/> |Время возникновения ошибки. Используется в сочетании с Ерроррепортсек для уникальной идентификации ошибки.  <br/> |
|**ерроррепортсек** <br/> |целое  <br/> |ИДЕНТИФИКАЦИОНный номер для идентификации ошибки. Используется в сочетании с Еррортиме для уникальной идентификации ошибки.  <br/> |
|**прогрессрепортсек** <br/> |целое  <br/> |Идентификатор для идентификации отчета о ходе выполнения. Используется, чтобы отличать отчеты о ходе выполнения одного отчета об ошибках.  <br/> |
|**мсдиагид** <br/> |целое  <br/> |Идентификатор диагностики для отчета об ошибке.  <br/> |
|**Источник** <br/> |nvarchar(256)  <br/> |Имя сервера, отправившего ошибку (при отправке отчета из серверного компонента).  <br/> |
|**Приложение** <br/> |nvarchar(256)  <br/> |Имя приложения, которое поступило об ошибке (при отправке отчета из серверного компонента).  <br/> |
|**телеметрид** <br/> |идентификатора  <br/> |Уникальный идентификатор, соответствующий сведениям о времени соединения для различных компонентов, участвующих в Конференции.  <br/> |
|**сессионсетуптиме** <br/> |целое  <br/> |Время (в миллисекундах), требуемое конкретным компонентом для присоединения к Конференции.  <br/> |
|**мсдиагхеадер** <br/> |varchar (max)  <br/> |Дополнительные сведения об ошибке.  <br/> |
