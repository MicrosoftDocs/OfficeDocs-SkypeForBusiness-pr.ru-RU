# <a name="progressreport-table"></a>Таблица ProgressReport
 
Отчеты о состоянии основываются на данных, отправленных клиентом в базу данных после завершения вызова или сеанса. Отчеты о ходе выполнения будут записываться только для звонков и сеансов, которые может использовать Skype для бизнеса Server 2015, в целях диагностики.
  
Поля Еррортиме, Ерроррепортсек и Прогрессрепортсек не обязательно ссылаются на ошибки, а также на сообщения, указывающие на состояние вызовов или сообщений.
  
|**Столбец**|**Тип данных**|**Ключ/индекс**|**Сведения**|
|:-----|:-----|:-----|:-----|
|**еррортиме** <br/> |datetime  <br/> |Основной, внешний  <br/> |Дата и время отчета об ошибках хода выполнения, который включает этот отчет о ходе выполнения. Для получения дополнительных сведений ознакомьтесь с [таблицей ерроррепорт в Skype для бизнеса Server 2015](errorreport.md) . <br/> |
|**еррорид** <br/> |целое  <br/> |Основной, внешний  <br/> |ИДЕНТИФИКАЦИОНный номер, используемый в сочетании с Еррортиме, Прогрессрепортсек для уникальной идентификации отчета о ходе выполнения. Для получения дополнительных сведений ознакомьтесь с [таблицей ерроррепорт в Skype для бизнеса Server 2015](errorreport.md) . <br/> |
|**ерроррепортсек** <br/> |целое  <br/> |Основной, внешний  <br/> |ИДЕНТИФИКАЦИОНный номер, идентифицирующий отчет об ошибке. Ерроррепорсек используется в сочетании с Еррортиме для уникальной идентификации отчета об ошибке. Дополнительные сведения приведены [в таблице ерроррепорт в Skype для бизнеса Server 2015](errorreport.md) <br/> Это поле было введено в Microsoft Lync Server 2013.  <br/> |
|**прогрессрепортсек** <br/> |целое  <br/> |Primary  <br/> |ИДЕНТИФИКАЦИОНный номер для идентификации отчета о ходе выполнения. Используется совместно с Еррортиме и Ерроррепортсек для уникальной идентификации отчета о ходе выполнения.  <br/> |
|**мсдиагид** <br/> |целое  <br/> ||Идентификатор диагностики отчета о ходе выполнения.  <br/> Это поле было введено в Microsoft Lync Server 2013.  <br/> |
|**Источника** <br/> |целое  <br/> |Другом  <br/> |Сервер, отправивший отчет об ошибке (при отправке отчета из серверного компонента). Для получения дополнительных сведений ознакомьтесь с [таблицей Servers](servers.md) . Это поле было введено в Microsoft Lync Server 2013. <br/> |
|**ApplicationId** <br/> |целое  <br/> ||Процесс Lync Server, к которому относится отчет. Для получения дополнительных сведений ознакомьтесь с таблицей Application.  <br/> |
|**Подробности** <br/> |изображение  <br/> ||Сведения о отчете о состоянии, сохраняемые в двоичном формате для экономии места. Эти данные можно преобразовать в текстовый формат, используя следующий синтаксис:  <br/> Cast (CAST (данные в формате varbinary (max)) AS varchar (max))  <br/> |
|**телеметрид** <br/> |Идентификатора  <br/> ||Уникальный идентификатор, который соответствует сведениям о времени соединения для различных компонентов, участвующих в Конференции.  <br/> Это поле было введено в Microsoft Lync Server 2013.  <br/> |
|**сессионсетуптиме** <br/> |целое  <br/> ||Время (в миллисекундах), в течение которого конкретный компонент присоединяется к Конференции.  <br/> Это поле было введено в Microsoft Lync Server 2013.  <br/> |
