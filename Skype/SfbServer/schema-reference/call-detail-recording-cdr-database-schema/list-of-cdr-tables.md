# <a name="list-of-cdr-tables-in-skype-for-business-server-2015"></a>Список таблиц CDR в Skype для бизнеса Server 2015
 
Схема базы данных регистрации вызовов (call detail recording — CDR) состоит из следующих таблиц. 
  
## <a name="static-tables"></a>Статические таблицы

|**Table**|**Описание**|
|:-----|:-----|
|[Таблица CallPriorities в Skype для бизнеса Server 2015](callpriorities.md) <br/> |Сохранение списка возможных приоритетов вызовов (включая аварийные, срочные, обычные, несрочные и т. д.).  <br/> |
|[Таблица ConferenceJoinTimeThresholds в Skype для бизнеса Server 2015](conferencejointimethresholds.md) <br/> |Сохранение границ классификации, используемых в отчете со сводными данными по времени подсоединения к конференции.  <br/> |
|[Таблица DeRegisterType в Skype для бизнеса Server 2015](deregistertype.md) <br/> |Сохранение списка возможных причин отмены регистрации пользователя, например «Инициируется клиентом», «Срок действия регистрации истек», «Сбой клиента» и т. д.  <br/> |
|[Таблица MediaList](medialist.md) <br/> |Сохранение списка типов мультимедиа, которые могут создавать записи в базе данных (например, чат, аудио, видео и передача файлов).  <br/> |
|[Таблица PurgeSettings](purgesettings.md) <br/> |Сохранение информации, определяющей, следует ли (и когда) автоматически удалять устаревшие записи о вызовах из базы данных CDR.  <br/> |
|[Таблица Roles](roles.md) <br/> |Сохранение списка возможных ролей участников конференции (например, участник и докладчик).  <br/> |
|[Таблица SIPResponseMetaData](sipresponsemetadata.md) <br/> |Сохранение списка кодов ответов SIP, классификации и определения каждого из этих кодов.  <br/> |

## <a name="table-for-voip-call-details"></a>Таблица для сведений о вызовах VoIP

|**Table**|**Описание**|
|:-----|:-----|
|[Таблица VoipDetails](voipdetails-0.md) <br/> |Сохранение информации о вызове, включая идентификатор телефона VoIP, используемый шлюз и сторону, прервавшую вызов, для каждого двустороннего вызова VoIP/ТСОП. Ссылается на [таблицу SessionDetails](sessiondetails.md) для времени начала и окончания вызова и код ответа. <br/> |
   
> [!NOTE]
> Если один участник вызова является пользователем VoIP или при выполнении вызова задействован сервер-посредник, в этой таблице создается соответствующая запись. Сведения о вызовах VoIP/VoIP, не связанных с телефоном телефонной сети общего звонков (PSTN), фиксироваться в таблице [SessionDetails.](sessiondetails.md)
