# <a name="list-of-cdr-tables-in-skype-for-business-server-2015"></a>Список таблиц регистрации звонков в Скайп для Business Server 2015
 
Регистрации вызовов (CDR) схему базы данных состоит из следующих таблиц. 
  
## <a name="static-tables"></a>Статические таблицы

|**Таблица**|**Описание**|
|:-----|:-----|
|[Таблица CallPriorities в Скайп для Business Server 2015](callpriorities.md) <br/> |Сохранение списка возможных приоритетов вызовов, например аварийного срочным, обычный, не срочный и многое другое.  <br/> |
|[Таблица conferencejointimethresholds в Скайп для Business Server 2015](conferencejointimethresholds.md) <br/> |Сохранение границ классификации, используемых отчету о Сводка времени присоединения к конференции.  <br/> |
|[Таблица DeRegisterType в Скайп для Business Server 2015](deregistertype.md) <br/> |Сохранение списка возможных пользователя отменять регистрацию причин, например «клиент инициировал,» «срок действия регистрации истек,» «после сбоя клиента» и многое другое.  <br/> |
|[Таблица MediaList](medialist.md) <br/> |Сохранение списка типов мультимедиа, которые могут создавать записи в базу данных (например, обмен мгновенными Сообщениями, аудио, видео и передача файлов).  <br/> |
|[Таблица PurgeSettings](purgesettings.md) <br/> |Сохранение информации, указывает ли (и когда) устаревшую вызова информации о сеансах будут автоматически удалены из базы данных регистрации Вызовов.  <br/> |
|[Таблица Roles](roles.md) <br/> |Сохранение списка возможных ролей участников конференции (например, участников и выступающих).  <br/> |
|[Таблица sipresponsemetadata](sipresponsemetadata.md) <br/> |Сохранение списка коды ответа SIP, классификации и определения каждого из этих кодов.  <br/> |

## <a name="table-for-voip-call-details"></a>Таблица для сведений о вызовах VoIP

|**Таблица**|**Описание**|
|:-----|:-----|
|[Таблица VoipDetails](voipdetails-0.md) <br/> |Для каждого вызова VoIP/ТСОП двусторонних хранятся сведения о вызове, такие как идентификатор VoIP телефон, шлюз, используемый и какие стороны соединены по сети. Относится к [таблице SessionDetails](sessiondetails.md) для времени начала и окончания вызова и код ответа. <br/> |
   
> [!NOTE]
> Если одна из сторон на вызов является пользователем VoIP или участвующей в вызов на сервер-посредник, записи будут создаваться в этой таблице. Сведения о вызовах VoIP и VoIP, не связанных с телефона телефонной сети (общего пользования PSTN) регистрируются в [таблице SessionDetails](sessiondetails.md).
