# <a name="errorreport-view"></a>Представление ErrorReport
 
Представление ErrorReport сохранение информации о ошибки. Каждая запись — один ошибку. Ошибки, которые отслеживались либо агентом CDR, работающим на сервере переднего плана или передаче от клиента. В этом представлении была введена в Microsoft Lync Server 2013.
  
|**Столбец**|**Тип данных**|**Сведения**|
|:-----|:-----|:-----|
|**ErrorTime** <br/> |datetime  <br/> |Ошибки времени. Используется в сочетании с ErrorReportSeq для уникальной идентификации ошибку.  <br/> |
|**ErrorReportSeq** <br/> |целое  <br/> |Номер идентификатора для определения ошибки. Используется в сочетании с ErrorTime для уникальной идентификации ошибку.  <br/> |
|**MsDiagId** <br/> |целое  <br/> |КОД диагностики для отчета об ошибках.  <br/> |
|**FromUri** <br/> |nvarchar(450)  <br/> |URI пользователя, инициировавшего ошибку.  <br/> |
|**FromUriType** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, инициировавшего ошибку. В [таблице UriTypes](uritypes.md) для получения дополнительных сведений см. <br/> |
|**FromTenant** <br/> |nvarchar(256)  <br/> |Клиент пользователя, инициировавшим ошибку. В [таблице клиентов](tenants.md) для получения дополнительных сведений см. <br/> |
|**ToUri** <br/> |nvarchar(450)  <br/> |URI-идентификатор пользователя, который был целью сообщения об ошибке.  <br/> |
|**ToUriType** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, предназначенных для сообщения об ошибке. В таблице UriTypes для получения дополнительных сведений см.  <br/> |
|**ToTenant** <br/> |nvarchar(256)  <br/> |Клиент пользователя, предназначенных для сообщения об ошибке. В [таблице клиентов](tenants.md) для получения дополнительных сведений см. <br/> |
|**URI конференции** <br/> |nvarchar(450)  <br/> |URI конференции, которая была целью сообщения об ошибке.  <br/> |
|**ConferenceUriType** <br/> |nvarchar(256)  <br/> |Тип URI конференции, которая была целью сообщения об ошибке. В [таблице UriTypes](uritypes.md) для получения дополнительных сведений см. <br/> |
|**SessionIdTime** <br/> |datetime  <br/> |Время запроса сеанса, в котором произошла отчет об ошибках. Используется совместно с SessionIdSeq для уникальной идентификации сеанса. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**SessionIdSeq** <br/> |целое  <br/> |Номер идентификатора для идентификации запроса сеанса, в котором произошла отчет об ошибках. Используется в сочетании с SessionIdTime для уникальной идентификации сеанса. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**DialogId** <br/> |varstring(775)  <br/> |SIP-идентификатор диалогового окна в сеансе, в котором произошла ошибка. Имеет следующий формат:  <br/> диалоговое окно, из тега; для тега  <br/> Эти данные можно преобразовать в текстовый формат с помощью следующего синтаксиса:  <br/> Приведение (приведения (ExternalId как varbinary(max)) как varchar(max))  <br/> |
|**ClientVersion** <br/> |nvarchar(256)  <br/> |Версия клиента, используемого пользователем, инициировавшим ошибку.  <br/> |
|**Типа клиента** <br/> |целое  <br/> |Клиент используемого пользователем, инициировавшим ошибку. [Таблица useragentdef](useragentdef.md) для получения дополнительных сведений см. <br/> |
|**ClientCategory** <br/> |nvarchar(64)  <br/> |Имя категории клиента, используемого пользователем, инициировавшим ошибку.  <br/> |
|**Источник** <br/> |nvarchar(256)  <br/> |Имя сервера, на котором произошла ошибка (Если отчет был отправлен серверным компонентом).  <br/> |
|**Приложение** <br/> |nvarchar(256)  <br/> |Имя приложения, в котором произошла ошибка (Если отчет был отправлен серверным компонентом).  <br/> |
|**ResponseCode** <br/> |целое  <br/> |Код ответа сеанса SIP-сообщения, содержащего отчет об ошибках SIP.  <br/> |
|**RequestType** <br/> |varchar(max)  <br/> |Тип запроса с отказом.  <br/> |
|**ContentType** <br/> |varchar(max)  <br/> |Тип содержимого запроса с отказом.  <br/> |
|**CallType** <br/> |nvarchar(256)  <br/> |Тип сеанса. [Таблица CallType в Скайп для Business Server 2015](calltype.md) для получения дополнительных сведений см. <br/> |
|**TelemetryId** <br/> |uniqueidentifier  <br/> |Уникальный идентификатор времени присоединения для разных компонентов, задействованных в конференции.  <br/> |
|**SetupTime** <br/> |целое  <br/> |Время (в миллисекундах), требуемое определенному компоненту для присоединения к конференции.  <br/> |
|**IsCapturedByServer** <br/> |бит  <br/> |Указывает, отчет об ошибках была отслеживались агентом CDR, запущенные на сервере переднего плана, или отправлен клиентом.  <br/> |
|**Флаг** <br/> |smallint  <br/> |Зарезервировано для будущего использования.  <br/> |
|**MsDiagHeader** <br/> |varchar(max)  <br/> |Дополнительные сведения об ошибке.  <br/> |
|**FrontEnd** <br/> |nvarchar  <br/> |Полное доменное имя сервера переднего плана, предоставленные отчета.  <br/> |
|**Пул** <br/> |nvarchar  <br/> |Полное доменное имя пула, содержащего сервера переднего плана, для отправки отчета.  <br/> |
