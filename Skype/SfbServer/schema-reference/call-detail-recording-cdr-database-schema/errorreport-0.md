# <a name="errorreport-view"></a>Представление ErrorReport
 
В представлении ErrorReport сохраняется информация о возникших ошибках. Каждая запись соответствует одному случаю возникновения ошибки. Ошибки фиксируются агентом CDR, работающим на сервере переднего плана, или передаются клиентом. Это представление впервые было введено в Microsoft Lync Server 2013.
  
|**Столбец**|**Тип данных**|**Details**|
|:-----|:-----|:-----|
|**ErrorTime** <br/> |datetime  <br/> |Время возникновения ошибки. Используется в сочетании с параметром ErrorReportSeq для уникальной идентификации ошибки.  <br/> |
|**ErrorReportSeq** <br/> |int  <br/> |Идентификатор ошибки. Используется в сочетании с параметром ErrorTime для уникальной идентификации ошибки.  <br/> |
|**MsDiagId** <br/> |int  <br/> |Диагностический идентификатор для сообщения об ошибке.  <br/> |
|**FromUri** <br/> |nvarchar(450)  <br/> |URI пользователя, инициировавшего ошибку.  <br/> |
|**FromUriType** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, ившего ошибку. Дополнительные сведения см. в таблице [UriTypes.](uritypes.md) <br/> |
|**FromTenant** <br/> |nvarchar(256)  <br/> |Клиент пользователя, задавшего ошибку. Дополнительные [сведения см.](tenants.md) в таблице Tenants. <br/> |
|**ToUri** <br/> |nvarchar(450)  <br/> |URI пользователя, который был целью сообщения об ошибке.  <br/> |
|**ToUriType** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, который был целью сообщения об ошибке. См. таблицу UriTypes для получения дополнительной информации.  <br/> |
|**ToTenant** <br/> |nvarchar(256)  <br/> |Клиент пользователя, нацелившегося на отчет об ошибках. Дополнительные [сведения см.](tenants.md) в таблице Tenants. <br/> |
|**ConferenceUri** <br/> |nvarchar(450)  <br/> |URI конференции, которая была целью сообщения об ошибке.  <br/> |
|**ConferenceUriType** <br/> |nvarchar(256)  <br/> |Тип URI конференции, которая была целью отчета об ошибках. Дополнительные сведения см. в таблице [UriTypes.](uritypes.md) <br/> |
|**SessionIdTime** <br/> |datetime  <br/> |Время запроса сеанса, который ил отчета об ошибках. Используется вместе с параметром SessionIdSeq для уникального определения сеанса. Дополнительные сведения см. в таблице [Dialogs в Skype для бизнеса Server 2015.](dialogs.md) <br/> |
|**SessionIdSeq** <br/> |int  <br/> |ID number to identify the session request that originated the error report. Используется совместно с параметром SessionIdTime для уникальной идентификации сеанса. Дополнительные сведения см. в таблице [Dialogs в Skype для бизнеса Server 2015.](dialogs.md) <br/> |
|**DialogId** <br/> |varstring(775)  <br/> |Диалоговый идентификатор SIP сеанса, инициировавшего ошибку. Используется следующий формат:  <br/> dialog;from-tag;to-tag  <br/> Эти данные можно преобразовать в текстовый формат, используя следующий синтаксис:  <br/> cast(cast(ExternalId as varbinary(max)) as varchar(max))  <br/> |
|**ClientVersion** <br/> |nvarchar(256)  <br/> |Версии клиента, используемого пользователем, инициировавшим ошибку.  <br/> |
|**ClientType** <br/> |int  <br/> |Клиент, используемый пользователем, за которого возникла ошибка. Дополнительные сведения см. в таблице [UserAgentDef.](useragentdef.md) <br/> |
|**ClientCategory** <br/> |nvarchar(64)  <br/> |Имя категории клиента, используемого пользователем, инициировавшим ошибку.  <br/> |
|**Source** <br/> |nvarchar(256)  <br/> |Имя сервера, с которого поступило сообщение об ошибке (если отчет был отправлен серверным компонентом).  <br/> |
|**Приложение** <br/> |nvarchar(256)  <br/> |Имя приложения, инициировавшего ошибку (если отчет был отправлен серверным компонентом).  <br/> |
|**ResponseCode** <br/> |int  <br/> |Код ответа SIP на сеанс SIP-сообщения, содержащего отчет об ошибках.  <br/> |
|**RequestType** <br/> |varchar(max)  <br/> |Тип запроса с отказом.  <br/> |
|**ContentType** <br/> |varchar(max)  <br/> |Тип содержимого запроса с отказом.  <br/> |
|**CallType** <br/> |nvarchar(256)  <br/> |Тип сеанса. Дополнительные сведения см. в таблице [CallType в Skype для бизнеса Server 2015.](calltype.md) <br/> |
|**TelemetryId** <br/> |uniqueidentifier  <br/> |Уникальный идентификатор времени присоединения для различных компонентов, участвующих в конференции.  <br/> |
|**SetupTime** <br/> |int  <br/> |Время (в миллисекундах), необходимое для присоединения к конференции определенного компонента.  <br/> |
|**IsCapturedByServer** <br/> |bit  <br/> |Указывает, был ли данный отчет об ошибках получен агентом CDR, работающим на сервере переднего плана, или отправлен клиентом.  <br/> |
|**Flag** <br/> |smallint  <br/> |Зарезервировано для будущего использования.  <br/> |
|**MsDiagHeader** <br/> |varchar(max)  <br/> |Дополнительные сведения об ошибке.  <br/> |
|**FrontEnd** <br/> |nvarchar  <br/> |Полное доменное имя сервера переднего сервера, который отправил отчет.  <br/> |
|**Pool** <br/> |nvarchar  <br/> |Полное доменное имя пула, содержащего сервер переднего сервера, который отправил отчет.  <br/> |
