# <a name="errorreport-view"></a>Представление Ерроррепорт
 
В представлении Ерроррепорт хранятся сведения об ошибках, о которых сообщается. Каждая запись содержит один экземпляр ошибки. Эти ошибки регистрируются агентом CDR, который запущен на сервере переднего плана или отправлен клиентом. Это представление было представлено в Microsoft Lync Server 2013.
  
|**Столбец**|**Тип данных**|**Сведения**|
|:-----|:-----|:-----|
|**еррортиме** <br/> |datetime  <br/> |Время возникновения ошибки. Используется в сочетании с Ерроррепортсек для уникальной идентификации ошибки.  <br/> |
|**ерроррепортсек** <br/> |целое  <br/> |ИДЕНТИФИКАЦИОНный номер для идентификации ошибки. Используется в сочетании с Еррортиме для уникальной идентификации ошибки.  <br/> |
|**мсдиагид** <br/> |целое  <br/> |Идентификатор диагностики для отчета об ошибке.  <br/> |
|**фромури** <br/> |nvarchar (450)  <br/> |URI пользователя, который поступил с ошибкой.  <br/> |
|**фромуритипе** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, который поступил на ошибку. Для получения дополнительных сведений ознакомьтесь с [таблицей уритипес](uritypes.md) . <br/> |
|**фромтенант** <br/> |nvarchar(256)  <br/> |Клиент пользователя, который поступил с ошибкой. Дополнительные сведения см. в [таблице "клиенты](tenants.md) ". <br/> |
|**таури** <br/> |nvarchar (450)  <br/> |Универсальный код ресурса (URI) пользователя, который является целью отчета об ошибке.  <br/> |
|**тауритипе** <br/> |nvarchar(256)  <br/> |Тип URI пользователя, который является конечным лицом отчета об ошибке. Для получения дополнительных сведений ознакомьтесь с таблицей Уритипес.  <br/> |
|**тотенант** <br/> |nvarchar(256)  <br/> |Клиент пользователя, который является конечным лицом отчета об ошибке. Дополнительные сведения см. в [таблице "клиенты](tenants.md) ". <br/> |
|**ConferenceUri** <br/> |nvarchar (450)  <br/> |Универсальный код ресурса (URI) Конференции, которая была целью отчета об ошибке.  <br/> |
|**конференцеуритипе** <br/> |nvarchar(256)  <br/> |Тип URI конференции, которая была целью отчета об ошибке. Для получения дополнительных сведений ознакомьтесь с [таблицей уритипес](uritypes.md) . <br/> |
|**сессионидтиме** <br/> |datetime  <br/> |Время запроса сеанса, в котором был создан отчет об ошибке. Используется в сочетании с Сессионидсек для уникальной идентификации сеанса. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**сессионидсек** <br/> |целое  <br/> |ИДЕНТИФИКАЦИОНный номер для идентификации запроса на сеанс, который является источником отчета об ошибке. Используется в сочетании с Сессионидтиме для уникальной идентификации сеанса. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**диалогид** <br/> |варстринг (775)  <br/> |ИДЕНТИФИКАТОР диалогового окна SIP сеанса, на который поступила ошибка. Формат:  <br/> диалоговое окно; тег "from-Tag"  <br/> Эти данные можно преобразовать в текстовый формат, используя следующий синтаксис:  <br/> Cast (CAST (Екстерналид AS varbinary (max)) AS varchar (max))  <br/> |
|**клиентверсион** <br/> |nvarchar(256)  <br/> |Версия клиента, используемая пользователем, который поступил с ошибкой.  <br/> |
|**клиенттипе** <br/> |целое  <br/> |Клиент, использованный пользователем, который поступил с ошибкой. Для получения дополнительных сведений ознакомьтесь с [таблицей усеражентдеф](useragentdef.md) . <br/> |
|**клиенткатегори** <br/> |nvarchar (64)  <br/> |Имя категории клиента, используемой пользователем, который поступил с ошибкой.  <br/> |
|**Источник** <br/> |nvarchar(256)  <br/> |Имя сервера, отправившего ошибку (при отправке отчета из серверного компонента).  <br/> |
|**Приложение** <br/> |nvarchar(256)  <br/> |Имя приложения, которое поступило об ошибке (при отправке отчета из серверного компонента).  <br/> |
|**респонсекоде** <br/> |целое  <br/> |Код ответа SIP в сеанс сообщения SIP с отчетом об ошибке.  <br/> |
|**RequestType** <br/> |varchar (max)  <br/> |Тип запроса, который завершился сбоем.  <br/> |
|**Контента** <br/> |varchar (max)  <br/> |Тип контента запроса, который завершился сбоем.  <br/> |
|**CallType** <br/> |nvarchar(256)  <br/> |Тип сеанса. Для получения дополнительных сведений ознакомьтесь с [таблицей каллтипе в Skype для бизнеса Server 2015](calltype.md) . <br/> |
|**телеметрид** <br/> |идентификатора  <br/> |Уникальный идентификатор, соответствующий сведениям о времени соединения для различных компонентов, участвующих в Конференции.  <br/> |
|**сетуптиме** <br/> |целое  <br/> |Время (в миллисекундах), требуемое конкретным компонентом для присоединения к Конференции.  <br/> |
|**искаптуредбисервер** <br/> |бит  <br/> |Указывает, был ли отчет об ошибке записан агентом CDR, запущенным на сервере переднего плана или отправлен клиентом.  <br/> |
|**Пометка** <br/> |smallint  <br/> |Зарезервировано для будущего использования.  <br/> |
|**мсдиагхеадер** <br/> |varchar (max)  <br/> |Дополнительные сведения об ошибке.  <br/> |
|**FrontEnd** <br/> |nvarchar  <br/> |Полное доменное имя сервера переднего плана, отправившего отчет.  <br/> |
|**Pool** <br/> |nvarchar  <br/> |Полное доменное имя пула, содержащего сервер переднего плана, отправивший отчет.  <br/> |
