# <a name="errorreport-view"></a>Представление ErrorReport
 
В представлении ErrorReport сохраняется информация о возникших ошибках. Каждая запись соответствует одному случаю возникновения ошибки. Ошибки фиксируются агентом CDR, работающим на сервере переднего плана, или передаются клиентом. Это представление было представлено в Microsoft Lync Server 2013.
  
|**Столбец**|**Тип данных**|**Сведения**|
|:-----|:-----|:-----|
|**ErrorTime** <br/> |datetime  <br/> |Время возникновения ошибки. Используется в сочетании с параметром ErrorReportSeq для уникальной идентификации ошибки.  <br/> |
|**ErrorReportSeq** <br/> |int  <br/> |Идентификатор ошибки. Используется в сочетании с параметром ErrorTime для уникальной идентификации ошибки.  <br/> |
|**MsDiagId** <br/> |int  <br/> |Диагностический идентификатор для сообщения об ошибке.  <br/> |
|**FromUri** <br/> |nvarchar (450)  <br/> |URI пользователя, инициировавшего ошибку.  <br/> |
|**FromUriType** <br/> |nvarchar (256)  <br/> |Тип URI пользователя, зародившего ошибку. Дополнительные сведения см. в таблице [UriTypes.](uritypes.md) <br/> |
|**FromTenant** <br/> |nvarchar (256)  <br/> |Клиент пользователя, зародившего ошибку. Дополнительные сведения см. в таблице [Tenants.](tenants.md) <br/> |
|**ToUri** <br/> |nvarchar (450)  <br/> |URI пользователя, который был целью сообщения об ошибке.  <br/> |
|**ToUriType** <br/> |nvarchar (256)  <br/> |Тип URI пользователя, который был целью сообщения об ошибке. См. таблицу UriTypes для получения дополнительной информации.  <br/> |
|**ToTenant** <br/> |nvarchar (256)  <br/> |Клиент пользователя, на которого направлен отчет об ошибке. Дополнительные сведения см. в таблице [Tenants.](tenants.md) <br/> |
|**ConferenceUri** <br/> |nvarchar (450)  <br/> |URI конференции, которая была целью сообщения об ошибке.  <br/> |
|**ConferenceUriType** <br/> |nvarchar (256)  <br/> |Тип URI конференции, которая была объектом отчета об ошибке. Дополнительные сведения см. в таблице [UriTypes.](uritypes.md) <br/> |
|**SessionIdTime** <br/> |datetime  <br/> |Время запроса сеанса, зародив отчет об ошибке. Используется вместе с параметром SessionIdSeq для уникального определения сеанса. Дополнительные сведения см. в таблице [Диалоги Skype для бизнеса Server 2015](dialogs.md) г. <br/> |
|**SessionIdSeq** <br/> |int  <br/> |ID-номер для определения запроса сеанса, зародив отчет об ошибке. Используется совместно с параметром SessionIdTime для уникальной идентификации сеанса. Дополнительные сведения см. в таблице [Диалоги Skype для бизнеса Server 2015](dialogs.md) г. <br/> |
|**Диалоговое окно** <br/> |varstring (775)  <br/> |Диалоговый идентификатор SIP сеанса, инициировавшего ошибку. Используется следующий формат:  <br/> диалоговое окно; от тега;до тега  <br/> Эти данные можно преобразовать в текстовый формат, используя следующий синтаксис:  <br/> cast(cast(ExternalId as varbinary(max)) as varchar(max))  <br/> |
|**ClientVersion** <br/> |nvarchar (256)  <br/> |Версии клиента, используемого пользователем, инициировавшим ошибку.  <br/> |
|**ClientType** <br/> |int  <br/> |Клиент, используемый пользователем, у которого возникла ошибка. Дополнительные сведения см. в таблице [UserAgentDef.](useragentdef.md) <br/> |
|**ClientCategory** <br/> |nvarchar (64)  <br/> |Имя категории клиента, используемого пользователем, инициировавшим ошибку.  <br/> |
|**Source** <br/> |nvarchar (256)  <br/> |Имя сервера, с которого поступило сообщение об ошибке (если отчет был отправлен серверным компонентом).  <br/> |
|**Приложение** <br/> |nvarchar (256)  <br/> |Имя приложения, инициировавшего ошибку (если отчет был отправлен серверным компонентом).  <br/> |
|**ResponseCode** <br/> |int  <br/> |Код ответа SIP на сеанс SIP-сообщения, содержащего отчет об ошибках.  <br/> |
|**RequestType** <br/> |varchar (max)  <br/> |Тип запроса с отказом.  <br/> |
|**ContentType** <br/> |varchar (max)  <br/> |Тип содержимого запроса с отказом.  <br/> |
|**CallType** <br/> |nvarchar (256)  <br/> |Тип сеанса. Дополнительные сведения см. в [таблице CallType Skype для бизнеса Server 2015](calltype.md) г. <br/> |
|**TelemetryId** <br/> |uniqueidentifier  <br/> |Уникальный идентификатор времени присоединения для различных компонентов, участвующих в конференции.  <br/> |
|**SetupTime** <br/> |int  <br/> |Время (в миллисекундах), необходимое для присоединения к конференции определенного компонента.  <br/> |
|**IsCapturedByServer** <br/> |bit  <br/> |Указывает, был ли данный отчет об ошибках получен агентом CDR, работающим на сервере переднего плана, или отправлен клиентом.  <br/> |
|**Флаг** <br/> |smallint  <br/> |Зарезервировано для будущего использования.  <br/> |
|**MsDiagHeader** <br/> |varchar (max)  <br/> |Дополнительные сведения об ошибке.  <br/> |
|**FrontEnd** <br/> |nvarchar  <br/> |Полное доменное имя переднего конечного сервера, который представил отчет.  <br/> |
|**Pool** <br/> |nvarchar  <br/> |Полное доменное имя пула, содержащего сервер переднего конца, который представил отчет.  <br/> |
