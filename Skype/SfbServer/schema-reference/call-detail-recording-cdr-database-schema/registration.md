# <a name="registration-table"></a>Таблица Registration
 
Каждая запись представляет одно событие регистрации пользователя.
  
|**Столбец**|**Тип данных**|**Ключ/индекс**|**Сведения**|
|:-----|:-----|:-----|:-----|
|**сессионидтиме** <br/> |datetime  <br/> |Основной, внешний  <br/> |Время запроса сеанса. Используется в сочетании с **сессионидсек** для уникальной идентификации сеанса. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**сессионидсек** <br/> |целое  <br/> |Основной, внешний  <br/> |ИДЕНТИФИКАЦИОНный номер для идентификации сеанса. Используется в сочетании с **сессионидтиме** для уникальной идентификации сеанса. Дополнительные сведения приведены [в таблице диалоговые окна в Skype для бизнеса Server 2015](dialogs.md) . <br/> |
|**Идентификатора пользователя** <br/> |целое  <br/> |Другом  <br/> |Идентификатор пользователя. Дополнительные сведения [можно найти в таблице Users](users.md) . <br/> |
|**ендпоинтид** <br/> |идентификатора  <br/> ||GUID для идентификации конечной точки регистрации. Обычно событие Register на том же компьютере того же пользователя будет иметь тот же идентификатор конечной точки. У разных компьютеров другой идентификатор конечной точки.  <br/> |
|**ендпоинтера** <br/> |Идентификатора  <br/> ||Идентификатор, который используется для различения регистраций, включающих одного и того же пользователя и ту же конечную точку.  <br/> Это поле было введено в Microsoft Lync Server 2013.  <br/> |
|**клиентверсионид** <br/> |целое  <br/> |Другом  <br/> |Клиентская версия текущего пользователя. Для получения дополнительных сведений ознакомьтесь с [таблицей клиентверсионс в Skype для бизнеса Server 2015](clientversions.md) . <br/> |
|**регистрарид** <br/> |целое  <br/> |Другом  <br/> |Идентификатор сервера регистратора, используемого для регистрации. Для получения дополнительных сведений ознакомьтесь с [таблицей Servers](servers.md) . <br/> |
|**пулид** <br/> |целое  <br/> |Другом  <br/> |Идентификатор пула, в котором был собран сеанс. Для получения дополнительных сведений ознакомьтесь с [таблицей Pools](pools.md) . <br/> |
|**еджесерверид** <br/> |целое  <br/> |Другом  <br/> |Пограничный сервер, на котором проходит регистрация. Для получения дополнительных сведений ознакомьтесь с [таблицей еджесерверс в Skype для бизнеса Server 2015](edgeservers.md) . <br/> |
|**Internal (внутренний)** <br/> |Разрядная версия  <br/> ||Вход пользователя из внутреннего режима или нет.  <br/> |
|**исусерсервицеаваилабле** <br/> |бит  <br/> ||Доступен ли Усерсервице или нет.  <br/> |
|**испримарирегистрар** <br/> |бит  <br/> ||Регистрация для основного регистратора или нет.  <br/> |
|**испримарирегистрарцентрал** <br/> |бит  <br/> ||Указывает, зарегистрирован ли пользователь в работающем устройстве филиала.  <br/> Это поле было введено в Microsoft Lync Server 2013.  <br/> |
|**регистертиме** <br/> |datetime  <br/> ||Время регистрации.  <br/> |
|**дерегистертиме** <br/> |datetime  <br/> ||Время отмены регистрации.  <br/> |
|**респонсекоде** <br/> |целое  <br/> ||Код ответа на запрос на регистрацию.  <br/> |
|**диагностиЦид** <br/> |целое  <br/> ||Идентификатор диагностики запроса на регистрацию. Это означает, что тип данных диагностики.  <br/> |
|**Устройства** <br/> |целое  <br/> |Другом  <br/> |Устройство, от которого поступил запрос на регистрацию. Для получения дополнительных сведений ознакомьтесь с [таблицей "устройства" в Skype для бизнеса Server 2015](devices.md) . <br/> |
|**дерегистертипеид** <br/> |tinyint  <br/> |Другом  <br/> |Причина отмены регистрации, например "пользователь инициировал", "регистрация просрочена", "клиент не проходит" и многое другое. Для получения дополнительных сведений ознакомьтесь с [таблицей дерегистертипе в Skype для бизнеса Server 2015](deregistertype.md) . <br/> |
|**IPAddress** <br/> |nvarchar(256)  <br/> ||IP-адрес конечной точки, зарегистрированной пользователем. Это может быть адрес IPv4 или IPv6.  <br/> Это поле было введено в Microsoft Lync Server 2013.  <br/> |
|**LastModifiedTime** <br/> |Датой  <br/> ||Для внутреннего использования службой мониторинга.  <br/> Это поле было представлено в Skype для бизнеса Server 2015.  <br/> |
