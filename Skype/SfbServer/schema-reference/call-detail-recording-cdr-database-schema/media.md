# <a name="media-table"></a>Таблица Media
 
Каждая запись представляет один тип носителя, используемый в сеансе peer-to-peer. Один сеанс может быть представлена нескольких записей в таблице, если используется более одного типа мультимедиа.
  
> [!NOTE]
> Таблица Media не должен использоваться для расчета длительности мультимедиа для сеанса. В этой таблице содержат сигналы exchange мультимедиа в сеансе. Мультимедиа exchange, это делается путем запроса INVITE и StartTime указывает время отправки ПРИГЛАШЕНИЯ в работе. Время пригласить не означает, что время начала мультимедиа из-за мультимедиа начинается только после sessionee принимает сеанса. Время окончания обычно означает, что время окончания сеанса. 
  
|**Столбец**|**Тип данных**|**Ключ/индекс**|**Сведения**|
|:-----|:-----|:-----|:-----|
|**SessionIdTime** <br/> |datetime  <br/> |Основной, внешний  <br/> |Время запроса сеанса. Используется совместно с **SessionIdSeq** для уникальной идентификации сеанса. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**SessionIdSeq** <br/> |целое  <br/> |Основной, внешний  <br/> |Номер идентификатора для идентификации сеанса. Используется в сочетании с **SessionIdTime** для уникальной идентификации сеанса. В разделе [диалоговых окон в таблице в Скайп для Business Server 2015](dialogs.md) для получения дополнительных сведений. <br/> |
|**MediaId** <br/> |tinyint  <br/> |Основной, внешний  <br/> |Уникальный номер, идентифицирующий этот тип носителя. [Таблица MediaList](medialist.md) для получения дополнительных сведений см. <br/> |
|**StartTime** <br/> |datetime  <br/> |Primary  <br/> |Это время отправки запроса мультимедиа в работе, время начала не реальных мультимедиа. **Время начала** включает в себя время настройки сеанса. <br/> |
|**EndTime** <br/> |datetime  <br/> ||Это время окончания сеанса.  <br/> |
