# <a name="qoe-view-details"></a>Данные представлений о качестве взаимодействия
 
Представления охватывают наиболее распространенные сценарии возврата данных из базы данных QoE SQL данных. Рекомендуется использовать представления для создания настраиваемого отчета, а не для прямого доступа к таблицам баз данных; это потому, что представления с большей вероятностью будут поддерживать обратную совместимость с будущими выпусками.
  
|**Имя представления**|**Описание**|
|:-----|:-----|
|[Представление AudioStreamDetail](audiostreamdetail.md) <br/> |Хранение информации о каждом аудиопотоке в базе данных.  <br/> |
|[Представление MediaLine](medialine.md) <br/> |Хранит сведения о каждой строке мультимедиа в базе данных. Один сеанс аудиосвязи обычно содержит одну строку мультимедиа. Один сеанс аудио- и видеосвязи обычно содержит одну аудио- и видеомагимичную линию; Однако сеанс может содержать две видеомайки, если используется устройство для видеоконференций или используется представление галереи.  <br/> |
|[Представление NetworkConfigurationSettings](networkconfigurationsettings.md) <br/> |Хранит сведения о конфигурации сети.  <br/> |
|[Представление сеанса](session-0.md) <br/> |Хранит информацию о сеансах с записями в базе данных.  <br/> |
|[Представление UserAgent](useragent-0.md) <br/> |Хранит информацию об агентах пользователя, участвовавших в сеансах с записями в базе данных.  <br/> |
|[Представление VideoStreamDetail](videostreamdetail.md) <br/> |Хранит информацию о каждом видеопотоке в базе данных.  <br/> |
