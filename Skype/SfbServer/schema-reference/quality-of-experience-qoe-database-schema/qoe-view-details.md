# <a name="qoe-view-details"></a>Данные представлений о качестве взаимодействия
 
Представления охватывают наиболее распространенные сценарии возврата данных из базы данных QoE SQL. Рекомендуется использовать представления для создания пользовательских отчетов, а не непосредственного доступа к таблицам баз данных; это потому, что представления с большей вероятностью поддерживают обратную совместимость с будущими выпусками.
  
|**Просмотр имени**|**Описание**|
|:-----|:-----|
|[Представление AudioStreamDetail](audiostreamdetail.md) <br/> |Хранение информации о каждом аудиопотоке в базе данных.  <br/> |
|[Представление MediaLine](medialine.md) <br/> |Сохраняет сведения о каждой строке мультимедиа в базе данных. Один сеанс аудио обычно содержит одну строку мультимедиа. Один сеанс аудио- и видео (A/V) обычно содержит одну аудио-медиа-линию и одну линию мультимедиа; однако сеанс может содержать две линии мультимедиа видео, если используется устройство для конференций или используется Представление галереи.  <br/> |
|[Представление NetworkConfigurationSettings](networkconfigurationsettings.md) <br/> |Хранит сведения о конфигурации сети.  <br/> |
|[Представление сеанса](session-0.md) <br/> |Хранит информацию о сеансах с записями в базе данных.  <br/> |
|[Представление UserAgent](useragent-0.md) <br/> |Хранит информацию об агентах пользователя, участвовавших в сеансах с записями в базе данных.  <br/> |
|[Представление VideoStreamDetail](videostreamdetail.md) <br/> |Хранит информацию о каждом видеопотоке в базе данных.  <br/> |
