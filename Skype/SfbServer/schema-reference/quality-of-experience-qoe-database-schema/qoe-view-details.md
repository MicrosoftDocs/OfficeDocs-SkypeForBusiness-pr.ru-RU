# <a name="qoe-view-details"></a>Данные представлений о качестве взаимодействия
 
В представлениях рассматриваются наиболее распространенные сценарии возврата данных из базы данных SQL QoE. Рекомендуется использовать представления для создания настраиваемых отчетов вместо прямого доступа к таблицам базы данных; Это связано с тем, что представления более удобны для обеспечения обратной совместимости с будущими выпусками.
  
|**Имя представления**|**Описание**|
|:-----|:-----|
|[Представление Аудиостреамдетаил](audiostreamdetail.md) <br/> |Хранит сведения о каждом звуковом потоке в базе данных.  <br/> |
|[Представление Медиалине](medialine.md) <br/> |Хранит сведения о каждой строке мультимедиа в базе данных. Один звуковой сеанс обычно состоит из одной линии звукового файла. Один из звуковых и видеофайлов (A/V) обычно состоит из одной линии звукового файла и одной линии видеоклипа; Тем не менее, в этом сеансе могут содержаться две строки видеофайла, если используется устройство конференц-связи или если используется представление коллекции.  <br/> |
|[Представление Нетворкконфигуратионсеттингс](networkconfigurationsettings.md) <br/> |Хранит сведения о конфигурации сети.  <br/> |
|[Представление сеанса](session-0.md) <br/> |Хранит сведения о сеансах с записями в базе данных.  <br/> |
|[Представление "UserAgent"](useragent-0.md) <br/> |Хранит сведения об агентах пользователей, вовлеченных в сеансы с записями в базе данных.  <br/> |
|[Представление Видеостреамдетаил](videostreamdetail.md) <br/> |Хранит сведения о каждом видеопотоке в базе данных.  <br/> |
