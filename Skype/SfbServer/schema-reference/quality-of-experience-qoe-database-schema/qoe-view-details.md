# <a name="qoe-view-details"></a>Просмотр сведений о качестве взаимодействия
 
Представления охватывают наиболее распространенные сценарии для возвращения данных из базы данных SQL качества взаимодействия. Рекомендуется представления, используемые для создания настраиваемых отчетов вместо прямой доступ к таблиц базы данных; Вот так как представления, скорее всего, для обеспечения обратной совместимости с будущими версиями.
  
|**Имя представления**|**Описание**|
|:-----|:-----|
|[Представление audiostreamdetail](audiostreamdetail.md) <br/> |Сохранение информации о каждом аудиопотоке в базе данных.  <br/> |
|[Представление MediaLine](medialine.md) <br/> |Сохранение информации о каждом канале передачи мультимедиа в базе данных. Один аудиосеанса обычно содержит одну строку аудио. Один аудио- и видеоконференций (A / V) сеанса обычно содержит одну строку аудио и одну строку видео; Тем не менее сеанс может содержать две строки видео при использовании устройства конференц-связи или при использовании представления галереи.  <br/> |
|[Представление networkconfigurationsettings](networkconfigurationsettings.md) <br/> |Сохранение информации о конфигурации сети.  <br/> |
|[Просмотр сеанса](session-0.md) <br/> |Сохранение информации о сеансах с записями в базе данных.  <br/> |
|[Представление UserAgent](useragent-0.md) <br/> |Хранит информацию об агентах пользователя, участвовавших в сеансах с записями в базе данных.  <br/> |
|[Представление videostreamdetail](videostreamdetail.md) <br/> |Сохранение информации о каждом видеопотоке в базе данных.  <br/> |
