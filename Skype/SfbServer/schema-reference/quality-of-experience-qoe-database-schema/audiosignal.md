# <a name="audiosignal-table"></a>Таблица AudioSignal
 
Каждая запись представляет метрики звукового сигнала для одной конечной точки. Как правило, каждый вызов имеет две записи, одна для вызываемой, а одна для вызываемой. 
  
|**Столбец**|**Тип данных**|**Key/Index**|**Сведения**|
|:-----|:-----|:-----|:-----|
|**ConferenceDateTime** <br/> |datetime  <br/> |Primary  <br/> |Ссылки из [таблицы MediaLine](medialine-0.md).  <br/> |
|**SessionSeq** <br/> |int  <br/> |Primary  <br/> |Ссылки из [таблицы MediaLine](medialine-0.md).  <br/> |
|**MediaLineLabel** <br/> |tinyint  <br/> |Primary  <br/> |Ссылки из [таблицы MediaLine](medialine-0.md).  <br/> |
|**FromCaller** <br/> |bit  <br/> |Primary  <br/> |0. Данные вызываемой  <br/> 1. Данные вызываемой  <br/> |
|**SendSignalLevel** <br/> |int  <br/> | <br/> |Представляет уровень звукового сигнала Post-Analog Gain Control. Единица измерения для этого показателя — дБмо. Для приемлемого качества значение должно быть не менее 30 дБмо. Этот показатель не сообщается сервером аудио- и видеоконференций и IP-телефонами.  <br/> |
|**RecvSignalLevel** <br/> |int  <br/> | <br/> |См. в рублях SendSignalLevel.  <br/> |
|**SendNoiseLevel** <br/> |int  <br/> | <br/> |Представляет уровень звукового шума post-Analog Gain Control. Единица измерения для этого показателя — дБмо. Для приемлемого качества значение должно быть менее 35 дБмо. Этот показатель не сообщается сервером аудио- и видеоконференций и IP-телефонами.  <br/> |
|**RecvNoiseLevel** <br/> |int  <br/> | <br/> |См. в этой ленте SendNoiseLevel.  <br/> |
|**EchoReturn** <br/> |int  <br/> | <br/> |Метрика повышения потери echo return. Единица измерения для этого показателя — дБ. Чем ниже значение, тем меньше эхосигнал. Этот показатель не сообщается сервером аудио- и видеоконференций и IP-телефонами.  <br/> |
|**AudioSpeakerGlitchRate** <br/> |int  <br/> | <br/> |Среднее количество сбоев за пять минут при отрисовке громкоговоритель. Для хорошего качества должно быть не более одного сбоя в пять минут. Этот показатель не сообщается серверами аудио- и видеоконференций, серверами-посредниками и IP-телефонами.  <br/> |
|**AudioMicGlitchRate** <br/> |int  <br/> | <br/> |Средние сбои за пять минут для захвата микрофона. Для хорошего качества должно быть не более одного сбоя в пять минут. Этот показатель не сообщается серверами аудио- и видеоконференций, серверами-посредниками и IP-телефонами.  <br/> |
|**AudioTimestampDriftRateMic** <br/> |десятичной (9,2)  <br/> | <br/> |Скорость дрейфа часов устройства микрофона относительно часов ЦП.  <br/> |
|**AudioTimestampDriftRateSpk** <br/> |десятичной (9,2)  <br/> | <br/> |Скорость дрейфа часов устройства динамика по отношению к часам ЦП.  <br/> |
|**AudioTimestampErrorMicMs** <br/> |десятичной (9,2)  <br/> | <br/> |Скорость дрейфа часов устройства динамика по отношению к часам ЦП.  <br/> Средняя ошибка метки времени для потока захвата микрофоном, в миллисекундах, в последние 20 секунд вызова.  <br/> |
|**AudioTimestampErrorSpkMs** <br/> |десятичной (9,2)  <br/> | <br/> |Средняя ошибка штампа времени отображения потока в миллисекунде за последние 20 секунд звонка.  <br/> |
|**VsEntryCauses** <br/> |smallint  <br/> | <br/> |Речевой коммутатор — это полудуплексный режим с уменьшенной возможностью прерывания. Причины ввода голосового переключателя:  <br/> ENTER_VS_BADTS 0x01  <br/> ENTER_VS_ECHO 0x02  <br/> ENTER_VS_FORCEORCONVERGENCE 0x04  <br/> ENTER_VS_DNLP 0x08  <br/> Причиной может быть сочетание этих отдельных причин. ENTER_VS_FORCEORCONVERGENCE можно включить только с помощью regkey для целей тестирования.  <br/> Тип данных для этого столбца был изменен в Microsoft Lync Server 2013.  <br/> |
|**EchoEventCauses** <br/> |tinyint  <br/> | <br/> |Причины события эхо-  <br/> ECHO_EVENT_BAD_TIMESTAMP 0x01  <br/> ECHO_EVENT_POSTAEC_ECHO 0x02  <br/> ECHO_EVENT_ANLP 0x04  <br/> ECHO_EVENT_DNLP 0x08  <br/> ECHO_EVENT_MIC_CLIPPING 0x10  <br/> ECHO_EVENT_BAD_STATE 0x20  <br/> Причиной может быть сочетание этих отдельных причин.  <br/> |
|**EchoPercentMicIn** <br/> |десятичной (5,2)  <br/> | <br/> |Процент времени обнаружения эхо в потоке захвата микрофона. Как правило, значения для гарнитур или телефонов являются низкими, а для телефонов динамиков или автономных динамиков — более высокими. Для устройств, поддерживаюных отмену акустического эхо на борту, высокие значения указывают на утечку эхо. Для других устройств эта метрика не должна использоваться для оценки качества устройства.  <br/> |
|**EchoPercentSend** <br/> |десятичной (5,2)  <br/> ||Процент времени обнаружения эхо в отправленном потоке. Высокое значение в отправленных потоках указывает на утечку эхосигнала.  <br/> |
|**RxAGCSignalLevel** <br/> |int  <br/> | <br/> |Уровень полученного сигнала на сервере-посреднике от шлюза; это относится только к серверу-посреднику. Единица измерения — dBoV. Приемлемый диапазон для хорошего качества — от -30 до -18 dBoV.  <br/> |
|**RxAGCNoiseLevel** <br/> |int  <br/> | <br/> |Полученный уровень сигнала на сервере-посреднике из шлюза. Этот показатель относится только к серверу-посреднику. Единица измерения — dBoV. Приемлемый диапазон для хорошего качества — ниже -50 dBoV.  <br/> |
|**RxAvgAGCGain** <br/> |int  <br/> | <br/> |Автоматическое управление получением (AGC) на стороне сервера-посредника.  <br/> |
|**InitialSignalLevelRMS** <br/> |float  <br/> | <br/> |Корневой квадрат (RMS) входящих сигналов до первых 30 секунд вызова.  <br/> |
|**RecvSignalLevelCh1** <br/> |int  <br/> ||Уровень сигнала, полученный на канале 1.  <br/> Этот столбец был представлен в Microsoft Lync Server 2013.  <br/> |
|**RecvSignalLevelCh2** <br/> |int  <br/> ||Уровень сигнала, полученный на канале 2.  <br/> Этот столбец был представлен в Microsoft Lync Server 2013.  <br/> |
|**RecvNoiseLevelCh1** <br/> |int  <br/> ||Уровень шума, полученный на канале 1.  <br/> Этот столбец был представлен в Microsoft Lync Server 2013.  <br/> |
|**RecvNoiseLevelCh2** <br/> |int  <br/> ||Уровень шума, полученный на канале 2.  <br/> Этот столбец был представлен в Microsoft Lync Server 2013.  <br/> |
|**SendSignalLevelCh1** <br/> |int  <br/> ||Уровень сигнала, отправленный на канале 1.  <br/> Этот столбец был представлен в Microsoft Lync Server 2013.  <br/> |
|**SendSignalLevelCh2** <br/> |int  <br/> ||Уровень сигнала, отправленный на канале 2.  <br/> Этот столбец был представлен в Microsoft Lync Server 2013.  <br/> |
|**SendNoiseLevelCh1** <br/> |int  <br/> ||Уровень шума при отправлении на канале 1.  <br/> Этот столбец был представлен в Microsoft Lync Server 2013.  <br/> |
|**SendNoiseLevelCh2** <br/> |int  <br/> ||Уровень шума при отправлении на канале 2.  <br/> Этот столбец был представлен в Microsoft Lync Server 2013.  <br/> |
|**RenderLoopbackSignalLevel** <br/> |int  <br/> ||Уровень в dBFS сигнала, отправленного в громкоговоритель для воспроизведения. Учет любых корректировок для получения полученного сигнала. <br/> Этот столбец был представлен в Microsoft Lync Server 2013.  <br/> |   
|**RenderNoiseLevel** <br/> |int  <br/> ||Уровень в dBFS шумового контента в сигнале, отправленном в громкоговоритель для воспроизведения <br/> |
