# <a name="audiosignal-table"></a>Таблица AudioSignal
 
Каждая запись представляет метрики звукового сигнала для одной конечной точки. Обычно каждый вызов имеет две записи: одна для вызываемой стороны, а одна для вызываемой. 
  
|**Столбец**|**Тип данных**|**Ключ/индекс**|**Details**|
|:-----|:-----|:-----|:-----|
|**ConferenceDateTime** <br/> |datetime  <br/> |Primary  <br/> |Ссылка из таблицы [MediaLine.](medialine-0.md)  <br/> |
|**SessionSeq** <br/> |int  <br/> |Primary  <br/> |Ссылка из таблицы [MediaLine.](medialine-0.md)  <br/> |
|**MediaLineLabel** <br/> |tinyint  <br/> |Primary  <br/> |Ссылка из таблицы [MediaLine.](medialine-0.md)  <br/> |
|**FromCaller** <br/> |bit  <br/> |Primary  <br/> |0: данные вызываемой  <br/> 1: данные вызываемой  <br/> |
|**SendSignalLevel** <br/> |int  <br/> | <br/> |Представляет уровень звукового сигнала после аналогового контроля усиления. Единица измерения для этого показателя — дБмо. Для приемлемого качества значение должно быть не менее 30 дБмо. Этот показатель не сообщается сервером аудио- и видеоконференций и IP-телефонами.  <br/> |
|**RecvSignalLevel** <br/> |int  <br/> | <br/> |См. sendSignalLevel.  <br/> |
|**SendNoiseLevel** <br/> |int  <br/> | <br/> |Представляет уровень звукового шума после аналогового контроля усиления. Единица измерения для этого показателя — дБмо. Для приемлемого качества значение должно быть менее 35 дБмо. Этот показатель не сообщается сервером аудио- и видеоконференций и IP-телефонами.  <br/> |
|**RecvNoiseLevel** <br/> |int  <br/> | <br/> |См. sendNoiseLevel.  <br/> |
|**EchoReturn** <br/> |int  <br/> | <br/> |Метрика "Повышение потери эхо-возвращаемой потери". Единица измерения для этого показателя — дБ. Чем ниже значение, тем меньше эхосигнал. Этот показатель не сообщается сервером аудио- и видеоконференций и IP-телефонами.  <br/> |
|**AudioSpeakerGlitchRate** <br/> |int  <br/> | <br/> |Среднее количество временных сбоев за пять минут при отрисовке хламов. Для хорошего качества должно быть не более одного сбоя в пять минут. Этот показатель не сообщается серверами аудио- и видеоконференций, серверами-посредниками и IP-телефонами.  <br/> |
|**AudioMicGlitchRate** <br/> |int  <br/> | <br/> |Среднее количество временных сбоев за пять минут при захвате микрофона. Для хорошего качества должно быть не более одного сбоя в пять минут. Этот показатель не сообщается серверами аудио- и видеоконференций, серверами-посредниками и IP-телефонами.  <br/> |
|**AudioTimestampDriftRateMic** <br/> |decimal(9,2)  <br/> | <br/> |Скорость смещения часов микрофона относительно часов ЦП.  <br/> |
|**AudioTimestampDriftRateSpk** <br/> |decimal(9,2)  <br/> | <br/> |Скорость смещения часов динамика относительно часов ЦП.  <br/> |
|**AudioTimestampErrorMicMs** <br/> |decimal(9,2)  <br/> | <br/> |Скорость смещения часов динамика относительно часов ЦП.  <br/> Средняя ошибка метки времени для потока захвата микрофоном, в миллисекундах, в последние 20 секунд вызова.  <br/> |
|**AudioTimestampErrorSpkMs** <br/> |decimal(9,2)  <br/> | <br/> |Средняя ошибка отметки времени потока отрисовки динамика (в миллисекунах) за последние 20 секунд вызова.  <br/> |
|**VsEntryCauses** <br/> |smallint  <br/> | <br/> |Речевой коммутатор — это полудуплексный режим с уменьшенной возможностью прерывания. Причины ввода голосовой коммутатора:  <br/> ENTER_VS_BADTS 0x01  <br/> ENTER_VS_ECHO 0x02  <br/> ENTER_VS_FORCEORCONVERGENCE 0x04  <br/> ENTER_VS_DNLP 0x08  <br/> Причиной может быть сочетание этих отдельных причин. ENTER_VS_FORCEORCONVERGENCE для тестирования можно включить только путем регистрации.  <br/> Тип данных для этого столбца был изменен в Microsoft Lync Server 2013.  <br/> |
|**EchoEventCauses** <br/> |tinyint  <br/> | <br/> |Причины возникновения эхо-события:  <br/> ECHO_EVENT_BAD_TIMESTAMP 0x01  <br/> ECHO_EVENT_POSTAEC_ECHO 0x02  <br/> ECHO_EVENT_ANLP 0x04  <br/> ECHO_EVENT_DNLP 0x08  <br/> ECHO_EVENT_MIC_CLIPPING 0x10  <br/> ECHO_EVENT_BAD_STATE 0x20  <br/> Причиной может быть сочетание этих отдельных причин.  <br/> |
|**EchoPercentMicIn** <br/> |decimal(5,2)  <br/> | <br/> |Процент времени, когда в потоке захвата микрофона было обнаружено эхо. Как правило, значения для гарнитур или телефонов являются низкими, а для телефонов динамиков или автономных динамиков — более высокими. Для устройств, которые поддерживают подавления акустического эхо на уровне, высокие значения указывают на утечку эха. Для других устройств этот показатель не следует использовать для оценки качества устройства.  <br/> |
|**EchoPercentSend** <br/> |decimal(5,2)  <br/> ||Процент времени, когда в отправленном потоке обнаруживается эхо. Высокое значение в отправленных потоках указывает на утечку эхосигнала.  <br/> |
|**RxAGCSignalLevel** <br/> |int  <br/> | <br/> |Уровень сигнала, полученный на сервере-посреднике от шлюза; Это относится только к серверу-посреднику. Единица измерения — dBoV. Приемлемый диапазон для хорошего качества — от -30 до -18 dBoV.  <br/> |
|**RxAGCNoiseLevel** <br/> |int  <br/> | <br/> |Уровень сигнала, полученный на сервере-посреднике от шлюза. Этот показатель относится только к серверу-посреднику. Единица измерения — dBoV. Приемлемый диапазон для хорошего качества — ниже -50 dBoV.  <br/> |
|**RxAvgAGCGain** <br/> |int  <br/> | <br/> |Автоматическое управление усилением (AGC) на стороне сервера-посредника.  <br/> |
|**InitialSignalLevelRMS** <br/> |float  <br/> | <br/> |Корневой квадрат (RMS) входящих сигналов до первых 30 секунд вызова.  <br/> |
|**RecvSignalLevelCh1** <br/> |int  <br/> ||Уровень сигнала, полученный на канале 1.  <br/> Этот столбец был впервые представлен в Microsoft Lync Server 2013.  <br/> |
|**RecvSignalLevelCh2** <br/> |int  <br/> ||Уровень сигнала, полученный на канале 2.  <br/> Этот столбец был впервые представлен в Microsoft Lync Server 2013.  <br/> |
|**RecvNoiseLevelCh1** <br/> |int  <br/> ||Уровень шума, полученный на канале 1.  <br/> Этот столбец был впервые представлен в Microsoft Lync Server 2013.  <br/> |
|**RecvNoiseLevelCh2** <br/> |int  <br/> ||Уровень шума, полученный на канале 2.  <br/> Этот столбец был впервые представлен в Microsoft Lync Server 2013.  <br/> |
|**SendSignalLevelCh1** <br/> |int  <br/> ||Уровень сигнала, отправленный на канале 1.  <br/> Этот столбец был впервые представлен в Microsoft Lync Server 2013.  <br/> |
|**SendSignalLevelCh2** <br/> |int  <br/> ||Уровень сигнала, отправленный на канале 2.  <br/> Этот столбец был впервые представлен в Microsoft Lync Server 2013.  <br/> |
|**SendNoiseLevelCh1** <br/> |int  <br/> ||Уровень шума, отправленный на канале 1.  <br/> Этот столбец был впервые представлен в Microsoft Lync Server 2013.  <br/> |
|**SendNoiseLevelCh2** <br/> |int  <br/> ||Уровень шума, отправленный на канале 2.  <br/> Этот столбец был впервые представлен в Microsoft Lync Server 2013.  <br/> |
|**RenderLoopbackSignalLevel** <br/> |int  <br/> ||Уровень в dBFS сигнала, от отправленного в динамик для воспроизведения. Учитывает все корректировки усиления, внося в полученный сигнал. <br/> Этот столбец был впервые представлен в Microsoft Lync Server 2013.  <br/> |   
|**RenderNoiseLevel** <br/> |int  <br/> ||Уровень в dBFS содержимого шума в сигнале, от отправленном в динамик для воспроизведения <br/> |
