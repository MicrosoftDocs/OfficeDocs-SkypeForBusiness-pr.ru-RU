# <a name="audiosignal-table"></a>Таблица AudioSignal
 
Каждая запись представляет метрики звукового сигнала для одной конечной точки. Как правило каждого вызова имеет две записи, также является вызывающего абонента и является вызываемого абонента. 
  
|**Столбец**|**Тип данных**|**Ключ/индекс**|**Сведения**|
|:-----|:-----|:-----|:-----|
|**ConferenceDateTime** <br/> |datetime  <br/> |Primary  <br/> |Ссылка из [MediaLine table](medialine-0.md).  <br/> |
|**SessionSeq** <br/> |целое  <br/> |Primary  <br/> |Ссылка из [MediaLine table](medialine-0.md).  <br/> |
|**MediaLineLabel** <br/> |tinyint  <br/> |Primary  <br/> |Ссылка из [MediaLine table](medialine-0.md).  <br/> |
|**FromCaller** <br/> |бит  <br/> |Primary  <br/> |0: данные вызываемой стороны  <br/> 1: данные вызывающего абонента  <br/> |
|**SendSignalLevel** <br/> |целое  <br/> | <br/> |Представляет уровнем звукового сигнала после аналогового получить доступ. Единицы измерения для этой метрики — dBmo. Для обеспечения приемлемой качества должно быть по крайней мере 30 dBmo. Этот показатель не отображается сервером аудио- и видеоконференций или IP-телефоны.  <br/> |
|**RecvSignalLevel** <br/> |целое  <br/> | <br/> |В разделе SendSignalLevel.  <br/> |
|**SendNoiseLevel** <br/> |целое  <br/> | <br/> |Представляет уровнем звука пропускаемых после аналогового получить доступ. Единицы измерения для этой метрики — dBmo. Для обеспечения приемлемой качества должно быть меньше, чем 35 dBmo. Этот показатель не отображается сервером аудио- и видеоконференций или IP-телефоны.  <br/> |
|**RecvNoiseLevel** <br/> |целое  <br/> | <br/> |В разделе SendNoiseLevel.  <br/> |
|**EchoReturn** <br/> |целое  <br/> | <br/> |Показатель улучшения потери возврата эхо. Единицы измерения для этой метрики — база данных. Меньшие значения представляют меньше эхо. Этот показатель не отображается сервером аудио- и видеоконференций или IP-телефоны.  <br/> |
|**AudioSpeakerGlitchRate** <br/> |целое  <br/> | <br/> |Среднее значение ошибки на пять минут для отрисовки динамика. Для высокого качества это должен быть меньше, чем по одному в пять минут. Не сообщил A / V серверов, серверов-посредников или IP-телефоны.  <br/> |
|**AudioMicGlitchRate** <br/> |целое  <br/> | <br/> |Среднее значение ошибки на пять минут для захвата микрофоном. Для высокого качества это должен быть меньше, чем по одному в пять минут. Не сообщил A / V серверов, серверов-посредников или IP-телефоны.  <br/> |
|**AudioTimestampDriftRateMic** <br/> |Decimal(9,2)  <br/> | <br/> |Скорость часов микрофона ухода, относительно часов Процессора.  <br/> |
|**AudioTimestampDriftRateSpk** <br/> |Decimal(9,2)  <br/> | <br/> |Скорость часов динамика ухода, относительно часов Процессора.  <br/> |
|**AudioTimestampErrorMicMs** <br/> |Decimal(9,2)  <br/> | <br/> |Скорость часов динамика ухода, относительно часов Процессора.  <br/> Средняя микрофон захвата потока ошибка метки времени, в миллисекундах, в последние 20 секунд вызова.  <br/> |
|**AudioTimestampErrorSpkMs** <br/> |Decimal(9,2)  <br/> | <br/> |Среднее значение динамик воспроизведения ошибка метки времени потока в миллисекундах, в последние 20 секунд вызова.  <br/> |
|**VsEntryCauses** <br/> |smallint  <br/> | <br/> |Переключатель голосовой связи — это режим полудуплексную с возможностью сокращение прерывания. Причины запись переключатель голосовой связи:  <br/> ENTER_VS_BADTS 0X01  <br/> ENTER_VS_ECHO 0X02  <br/> ENTER_VS_FORCEORCONVERGENCE 0X04  <br/> ENTER_VS_DNLP 0X08  <br/> Причиной может быть комбинацией этих отдельных причин. ENTER_VS_FORCEORCONVERGENCE можно включить только с regkey для целей тестирования.  <br/> Тип данных для этого столбца был изменен в Microsoft Lync Server 2013.  <br/> |
|**EchoEventCauses** <br/> |tinyint  <br/> | <br/> |Причины события эхосигнала:  <br/> ECHO_EVENT_BAD_TIMESTAMP 0X01  <br/> ECHO_EVENT_POSTAEC_ECHO 0X02  <br/> ECHO_EVENT_ANLP 0X04  <br/> ECHO_EVENT_DNLP 0X08  <br/> ECHO_EVENT_MIC_CLIPPING 0X10  <br/> ECHO_EVENT_BAD_STATE 0X20  <br/> Причиной может быть комбинацией этих отдельных причин.  <br/> |
|**EchoPercentMicIn** <br/> |Decimal(5,2)  <br/> | <br/> |Процент времени, когда в потоке захвата микрофоном обнаруживался эхосигнал. Обычно низкие значения отображаются для гарнитур или телефонных трубок, а высокие значения – для телефонных или автономных динамиков. Для устройств, которые поддерживают встроенную возможность подавления акустического эхосигнала, высокие значения указывают утечку эхосигнала. Для других устройств эти единицы измерения не должны использоваться для оценки качества устройства.  <br/> |
|**EchoPercentSend** <br/> |Decimal(5,2)  <br/> ||Процент времени, при обнаружении эхо в отправленной потока. Процент высокой эхо в отправки потоков указателем утечек эхо.  <br/> |
|**RxAGCSignalLevel** <br/> |целое  <br/> | <br/> |Уровень сигнала на сервер-посредник принятых шлюза; Это относится только к серверу-посреднику. Единица этот показатель является dBoV. Для высокого качества допустимый диапазон должен быть [-к -18 30] dBoV.  <br/> |
|**RxAGCNoiseLevel** <br/> |целое  <br/> | <br/> |Уровень сигнала, полученного на сервер-посредник от шлюза. Это относится только к серверу-посреднику. Единица этот показатель является dBoV. Для высокого качества допустимый диапазон должен быть меньше, чем-50 dBoV.  <br/> |
|**RxAvgAGCGain** <br/> |целое  <br/> | <br/> |Автоматическая регулировка (Усиления) на стороне сервера-посредника.  <br/> |
|**InitialSignalLevelRMS** <br/> |число с плавающей точкой  <br/> | <br/> |Среднеквадратическое (RMS) входящего сигнала до 30 первых секунд вызова.  <br/> |
|**RecvSignalLevelCh1** <br/> |целое  <br/> ||Уровень сигнала, полученного по каналу 1.  <br/> Этот столбец появился в Microsoft Lync Server 2013.  <br/> |
|**RecvSignalLevelCh2** <br/> |целое  <br/> ||Уровень сигнала, полученного по каналу 2.  <br/> Этот столбец появился в Microsoft Lync Server 2013.  <br/> |
|**RecvNoiseLevelCh1** <br/> |целое  <br/> ||Уровень шума, полученного по каналу 1.  <br/> Этот столбец появился в Microsoft Lync Server 2013.  <br/> |
|**RecvNoiseLevelCh2** <br/> |целое  <br/> ||Уровень шума, полученного по каналу 2.  <br/> Этот столбец появился в Microsoft Lync Server 2013.  <br/> |
|**SendSignalLevelCh1** <br/> |целое  <br/> ||Уровень сигнала, переданного по каналу 1.  <br/> Этот столбец появился в Microsoft Lync Server 2013.  <br/> |
|**SendSignalLevelCh2** <br/> |целое  <br/> ||Уровень сигнала, переданного по каналу 2.  <br/> Этот столбец появился в Microsoft Lync Server 2013.  <br/> |
|**SendNoiseLevelCh1** <br/> |целое  <br/> ||Уровень шума, переданного по каналу 1.  <br/> Этот столбец появился в Microsoft Lync Server 2013.  <br/> |
|**SendNoiseLevelCh2** <br/> |целое  <br/> ||Уровень шума, переданного по каналу 2.  <br/> Этот столбец появился в Microsoft Lync Server 2013.  <br/> |
|**RenderLoopbackSignalLevel** <br/> |целое  <br/> ||Уровень в dBFS сигнала, отправляемого динамик для воспроизведения. Учетные записи для корректировок полученного сигнала любой рост. <br/> Этот столбец появился в Microsoft Lync Server 2013.  <br/> |   
|**RenderNoiseLevel** <br/> |целое  <br/> ||Уровень в dBFS содержимого пропускаемых сигнала, отправляемого динамик для воспроизведения <br/> |
