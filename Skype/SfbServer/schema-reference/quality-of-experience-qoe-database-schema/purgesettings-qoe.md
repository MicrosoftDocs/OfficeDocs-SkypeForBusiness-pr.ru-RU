# <a name="purgesettings-table-qoe"></a>Таблица Пуржесеттингс (QoE)
 
В таблице Пуржесеттингс содержатся сведения о том, что (и когда) устаревшие записи качества обслуживания будут автоматически удалены из базы данных QoE. Обратите внимание, что информация, связанная с удалением, также может быть получена из командной консоли Skype для бизнеса Server, выполнив следующую команду:
  
```
Get-CsQoEConfiguration
```

Эта таблица введена в Microsoft Lync Server 2013.
  
|**Столбец**|**Тип данных**|**Ключ/индекс**|**Сведения**|
|:-----|:-----|:-----|:-----|
|**ID** <br/> |целое  <br/> |Primary  <br/> |Уникальный идентификатор коллекции параметров очистки QoE.  <br/> |
|**Енаблепурже** <br/> |бит  <br/> ||Если установлено значение true (1), Microsoft Lync Server 2013 будет периодически очищать устаревшие записи из базы данных QoE. Очистка будет выполняться каждый день в томе, указанном в параметре Пуржехаур. Если для этого свойства задано значение false (0), записи не будут автоматически удалены из базы данных. По умолчанию используется значение True.  <br/> |
|**KeepQoEDataForDays** <br/> |целое  <br/> ||Указывает возраст QoE записей (в днях), которые будут очищены из базы данных: Если включена очистка, QoE записи, возраст которых превышает это значение, будут удалены из базы данных. Значение по умолчанию — 60 дней.  <br/> |
|**Пуржехаур** <br/> |целое  <br/> ||Указывает местное время суток, когда будет выполняться очистка базы данных. Время суток задается в 24-часовом формате (0 соответствует полуночи (00:00), а 23 — 23:00 часам вечера). Обратите внимание, что вы можете указать только час дня: значение 10 (указывает на 10:00 AM) разрешено, но значение 10:30 от 10,5 (это означает 10:30 AM) не разрешено. Значение по умолчанию — 1 (1:00 AM). Указывает местное время суток, когда будет выполняться очистка базы данных. Время суток задается в 24-часовом формате (0 соответствует полуночи (00:00), а 23 — 23:00 часам вечера). Обратите внимание, что вы можете указать только час дня: значение 10 (указывает на 10:00 AM) разрешено, но значение 10:30 от 10,5 (это означает 10:30 AM) не разрешено. Значение по умолчанию — 1 (1:00 AM).  <br/> |
