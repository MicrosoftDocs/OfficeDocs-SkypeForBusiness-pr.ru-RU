# <a name="tblroletype"></a>tblRoleType
 
tblRoleType — это статическая таблица подстановки с типами ролей и сопоставленными с ними наборами разрешений.
  
**Columns**

|**Столбец**|**Тип**|**Описание**|
|:-----|:-----|:-----|
|rtypeID  <br/> |int, не NULL  <br/> |Идентификатор типа роли.  <br/> |
|rtypeDesc  <br/> |nvarchar (256), не NULL  <br/> | Описание типа роли. Доступно четыре роли: <br/>  Member: член комнаты чата <br/>  Manager: руководитель комнаты чата <br/>  Voiced: выступающий для аудиторной комнаты чата <br/>  Creator: человек, который может создавать комнаты чата <br/> |
|rtypeAllowedPermSet  <br/> |bigint, не NULL  <br/> | Набор разрешений для роли. Используются следующие биты: <br/>  2: True if the role can manage nodes. <br/>  4: имеет значение true, если роль может создавать дочерние узлы. <br/>  7: имеет значение true, если роль может присоединяться к комнате чата (или дочерним комнатам чата категории). <br/>  8: имеет значение true, если роль может общаться с помощью чата в комнате чата (или в дочерних комнатах чата категории). <br/>  10: имеет значение true, если роль может считывать журнал чата даже без присоединения к комнате чата. <br/>  11: имеет значение true, если роль может просматривать комнату чата. (Данная возможность дополнительно уточняется различными факторами, такими как область и видимость.) <br/>  12: имеет значение true, если роль может общаться с помощью чата в аудиторной комнате чата. <br/>  13: имеет значение true, если роль может обходить правила видимости при просмотре узлов. <br/> |
   
**Ключ**

|**Столбец**|**Описание**|
|:-----|:-----|
|rtypeID  <br/> |Первичный ключ.  <br/> |
