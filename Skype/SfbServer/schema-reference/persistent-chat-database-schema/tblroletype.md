# <a name="tblroletype"></a>tblRoleType
 
tblRoleType — это статическая таблица подстановки с типами ролей и их наборы разрешений.
  
**Столбцы**

|**Столбец**|**Тип**|**Описание**|
|:-----|:-----|:-----|
|rtypeID  <br/> |int, не null  <br/> |Идентификатор типа роли.  <br/> |
|rtypeDesc  <br/> |nvarchar (256), отлично от null  <br/> | Описание типа роли. Существует четыре доступных ролей: <br/>  Member: член комнаты чата <br/>  Manager: руководитель комнаты чата <br/>  Voiced: Выступающий для аудиторной комнаты чата <br/>  Creator: Может создавать комнаты чата <br/> |
|rtypeAllowedPermSet  <br/> |bigint, не может быть null  <br/> | Набор разрешений для роли. Используемые биты: <br/>  2: имеет значение true, если роль может управлять узлов. <br/>  4: имеет значение true, если роль может создавать дочерние узлы. <br/>  7: имеет значение true, если роль может присоединяться к комнате чата (или дочерним комнатам чата категории). <br/>  8: имеет значение true, если роль может общаться в комнате чата (или в дочерних комнатах чата категории). <br/>  10: имеет значение true, если роль может считывать журнал чата даже в том случае, если не подключен к комнате чата. <br/>  11: имеет значение true, если роль может просматривать комнаты чата. (Это является дальнейшей уточнение по факторов, например, области и видимости.) <br/>  12: имеет значение true, если роль может общаться в чате аудитории. <br/>  13: имеет значение true, если роль может обходить правила видимости при просмотре узлов. <br/> |
   
**Ключ**

|**Столбец**|**Описание**|
|:-----|:-----|
|rtypeID  <br/> |Первичный ключ.  <br/> |
