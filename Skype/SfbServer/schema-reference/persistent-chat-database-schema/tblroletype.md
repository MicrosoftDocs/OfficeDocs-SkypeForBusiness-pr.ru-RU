# <a name="tblroletype"></a>tblRoleType
 
Тблролетипе — это статическая таблица подстановки с типами ролей и связанными с ними наборами разрешений.
  
**Столбцов**

|**Столбец**|**Тип**|**Описание**|
|:-----|:-----|:-----|
|ртипеид  <br/> |int, NOT NULL  <br/> |Идентификатор типа роли.  <br/> |
|ртипедеск  <br/> |nvarchar (256), NOT NULL  <br/> | Описание типа роли. Доступно четыре роли: <br/>  Член: участник комнаты чата <br/>  Руководитель: руководитель комнаты чата <br/>  Выставлено сообщение: выступающий для комнаты чата Аудиториум <br/>  Создатель: может создавать комнаты чата <br/> |
|ртипеалловедпермсет  <br/> |bigint, NOT NULL  <br/> | Наборы разрешений для роли. Используются следующие биты: <br/>  2: значение true, если роль может управлять узлами. <br/>  4: значение true, если роль может создавать дочерние узлы. <br/>  7: значение true, если роль может присоединиться к комнате чата (или дочерним комнатам чатов для категории). <br/>  8: true, если роль может общаться в комнате чата (или в дочерних комнатах чата). <br/>  10: true, если роль может читать историю чата, даже если она не присоединяется к комнате чата. <br/>  11: true, если роль может видеть комнату чата. (Это улучшено с помощью таких факторов, как область видимости и видимость.) <br/>  12: true, если роль может общаться в комнате чата Аудиториум. <br/>  13: true, если роль может обойти правила видимости при просмотре узлов. <br/> |
   
**Ключ**

|**Столбец**|**Описание**|
|:-----|:-----|
|ртипеид  <br/> |Первичный ключ.  <br/> |
