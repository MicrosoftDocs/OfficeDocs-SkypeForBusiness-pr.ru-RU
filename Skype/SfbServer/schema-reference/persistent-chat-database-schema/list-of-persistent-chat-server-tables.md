# <a name="list-of-persistent-chat-server-tables"></a>Список таблиц сервера сохраняемого чата
 
Схема базы данных сохраняемого чата состоит из следующих таблиц.
  
## <a name="active-directory-sync"></a>Синхронизация службы каталогов Active Directory

|**Таблица**|**Описание**|
|:-----|:-----|
|[tblADCookie](tbladcookie.md) <br/> |В этой папке содержатся файлы cookie для синхронизации протокола Lightweight Directory Access. Каждая строка соответствует домену доменных служб Active Directory, в котором активный сервер чата отслеживает изменения. (В этой таблице представлены только те домены службы каталогов Active Directory, которые относятся к серверу сохраняемого чата.)  <br/> |
|[tblPrincipalMemberDifference](tblprincipalmemberdifference.md) <br/> |Содержит изменения членства в группах (добавленные и удаленные участники), которые еще не были обработаны с помощью последующих шагов синхронизации Active Directory и являются одной из временных таблиц (вместе с таблицей Тбладупдатес), которая используется на первом этапе синхронизации Active Directory.  <br/> Изменения в членстве сохраняются, обрабатываются или отображаются только для групп, указанных в таблице ТблпринЦипал или уже содержащихся в них участников.  <br/> |
|[tblADUpdates](tbladupdates.md) <br/> |Содержит изменения, внесенные в доменные службы Active Directory, которые еще не обрабатывались с помощью последующих шагов синхронизации Active Directory, и являются одной из временных таблиц (вместе с таблицей ТблпринЦипалмембердифференце), которая используется на первом этапе службы каталогов Active Directory. Идет.  <br/> Изменения, вносимые в Active Directory, сохраняются, обрабатываются или обе будут только для участников, которые уже указаны в таблице ТблпринЦипал.  <br/> |
|[tblPrincipalMembers](tblprincipalmembers.md) <br/> |Сведения о членстве участников.  <br/> |
|[tblPrincipalMeta](tblprincipalmeta.md) <br/> |Содержит участников, которые необходимо обновлять из службы каталогов Active Directory.  <br/> |
|[tblSkippedAffiliations](tblskippedaffiliations.md) <br/> |Содержат назначения, которые не удалось обновить по некоторым причинам, обычно из-за ошибок доступа в службе каталогов Active Directory.  <br/> Эта таблица предназначена исключительно для информационных целей. Его содержимое не используется.  <br/> Участники с назначением, которые не удалось обновить надлежащим образом, хранятся в таблице ТблпринЦипалмета и получают другой шанс на обновление.  <br/> |
   
## <a name="principals-affiliations-nodes-scopes-and-roles"></a>Участники, принадлежности, узлы, области и роли

|**Таблица**|**Описание**|
|:-----|:-----|
|[tblPrincipalType](tblprincipaltype.md) <br/> |Содержит типы принципалов для классификации содержимого в таблице ТблпринЦипал. Эта таблица является статичной. Он настроен во время создания базы данных и не меняется.  <br/> |
|[tblPrincipal](tblprincipal.md) <br/> |Содержат все участники (пользователи, папки, группы и т. д.). Сервер сохраняемого чата обрабатывает его как плоский разнородный список. Различные столбцы основываются на типе каждого участника.  <br/> Большинство из этих участников кэшируют копии объектов, которые хранятся в Active Directory. Создание кэшированной копии в основной таблице этих объектов Active Directory называется подготовка.  <br/> Некоторые принципалы создаются более агрессивно, чем другие, а некоторые объекты Active Directory пропускаются полностью.  <br/> |
|[tblPrincipalAffiliations](tblprincipalaffiliations.md) <br/> |Содержат основные принадлежности, описывающие участие в группах безопасности Active Directory, контейнерах Active Directory и т. д.  <br/> |
|[tblNode](tblnode.md) <br/> |Элемент управления Category (категория), управляемый на панели инструментов.  <br/> |
|[tblRoleType](tblroletype.md) <br/> |Содержат типы ролей и связанные с ними наборы разрешений. Эта таблица подстановки является статичной.  <br/> |
|[tblScopePrincipal](tblscopeprincipal.md) <br/> |Содержат области, назначенные узлам.  <br/> |
|[tblPrincipalRole](tblprincipalrole.md) <br/> |Содержат роли, назначенные узлам.  <br/> |
|[tblSiopWhiteList](tblsiopwhitelist.md) <br/> |Содержат зарегистрированные надстройки, которые можно связать с узлами.  <br/> |
|[tblEnumAttribute](tblenumattribute.md) <br/> |В таблице Тблноде содержатся только закодированные атрибуты "Visibility" и "Behavior".  <br/> |
|[tblEnumValue](tblenumvalue.md) <br/> |Имеет значения встроенных атрибутов Visibility и Behavior, которые используются в таблице Тблноде.  <br/> |
   
## <a name="invites-chats-and-other-client-support"></a>Приглашения, разговоры и другие службы поддержки клиентов

|**Таблица**|**Описание**|
|:-----|:-----|
|[tblPrincipalInvites](tblprincipalinvites.md) <br/> |Содержат приглашения для всех подготовленных пользователей в системе для всех узлов с включенным параметром "автоматическое приглашение".  <br/> |
|[tblChat](tblchat.md) <br/> |Содержат все сообщения чата.  <br/> |
|[tblLastInviteId](tbllastinviteid.md) <br/> |Содержащий идентификатор последнего приглашения, который был создан (и использован в таблице ТблпринЦипалинвитес) для каждого пользователя.  <br/> |
|[tblLastChatId](tbllastchatid.md) <br/> |Содержащий последний идентификатор чата, который был создан (и использован в таблице Тблчат) для каждого пользователя.  <br/> |
|[tblPreference](tblpreference.md) <br/> |Содержат пользовательские настройки клиента (используется только устаревшими клиентами).  <br/> |
|[tblFileToken](tblfiletoken.md) <br/> |Содержат временные маркеры для целей передачи файлов. Каждый раз, когда файл отправляется или загружается, служба сохраняемого чата создает маркер, который используется клиентом для доступа к хранилищу файлов веб-служб.  <br/> |
   
## <a name="server-support"></a>Поддержка сервера

|**Таблица**|**Описание**|
|:-----|:-----|
|[tblServerIdentity](tblserveridentity.md) <br/> |Включает активные серверы в пуле сервера сохраняемого чата.  <br/> |
|[tblAdminLock](tbladminlock.md) <br/> |Включает блокировку администратора для выполнения некоторых команд администратора. При каждом выпуске блокировки в таблице Тблсистемревисион увеличивается запись о редакции системы.  <br/> |
|[tblSystemRevision](tblsystemrevision.md) <br/> |Включает номер редакции (вместе с таблицей Тбладминлокк) для достижения согласованности на нескольких серверах.  <br/> |
|[tblActivePeers](tblactivepeers.md) <br/> |В этом поле содержатся текущие одноранговые соединения между службами сохраняемого чата.  <br/> |
|[tblConfig](tblconfig.md) <br/> |Неподдерживаемая конфигурация сервера сохраняемого чата.  <br/> |
