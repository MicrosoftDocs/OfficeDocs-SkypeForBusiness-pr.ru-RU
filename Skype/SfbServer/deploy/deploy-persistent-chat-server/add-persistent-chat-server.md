# <a name="add-persistent-chat-server-to-your-skype-for-business-server-2015-topology"></a>Добавление сервера сохраняемой беседы в топологию Skype для бизнеса Server 2015
 
**Сводка:** В этом разделе вы узнаете, как добавить сервер сохраняемой беседы в топологию Skype для бизнеса Server 2015.
  
После установки необходимого программного обеспечения на каждом сервере, на котором планируется развертывание сервера сохраняемой беседы, построитель топологий используется для: 
  
- Обновите топологию, включив сервер сохраняемой беседы
    
- Публикация обновленной топологии
    
> [!NOTE] 
> Сохраняемая беседа доступна в Skype для бизнеса Server 2015, но больше не поддерживается в Skype для бизнеса Server 2019. Такие же функции доступны в Teams. Дополнительные сведения [см. в сведениях о том,](/microsoftteams/upgrade-start-here)как начать обновление Microsoft Teams. Если необходимо использовать сохраняемого чата, вы можете либо перенести пользователей, которым требуются эти функции, в Teams, либо продолжить использование Skype для бизнеса Server 2015. 

## <a name="update-your-topology-to-include-persistent-chat-server"></a>Обновите топологию, включив сервер сохраняемой беседы

Выполните следующие действия для установки одного пула серверов сохраняемой беседы без настройки аварийного восстановления. For configuring a stretched Persistent Chat Server pool for high availability and disaster recovery, see [Configure high availability and disaster recovery for Persistent Chat Server in Skype for Business Server 2015](../../deploy/deploy-high-availability-and-disaster-recovery/configure-hadr-for-persistent-chat.md).
  
Чтобы развернуть несколько пулов серверов сохраняемой беседы, повторите один и тот же процесс для каждого пула.
  
1. На компьютере со Skype для бизнеса Server или на котором установлены средства администрирования Skype для бизнеса Server, войдите в систему с помощью учетной записи, которая является членом локальной группы пользователей (или учетной записи с эквивалентными правами пользователя).
    
    > [!NOTE]
    > Топологию можно определить с помощью учетной записи, которая является членом локальной группы пользователей. но для публикации топологии, необходимой для  установки Skype для бизнеса Server, необходимо использовать учетную запись, которая является членом группы администраторов домена и группы **RTCUniversalServerAdmins** и имеет разрешения на полный доступ (чтение, запись и изменение) в хранилище файлов, которое вы собираетесь использовать для хранения файлов сервера сохраняемого чата (чтобы построитель топологий можно было настроить необходимые DACLs) или учетную запись с эквивалентными правами.
  
2. Запустите построитель топологий.
    
3. In the console tree, navigate to the **Persistent Chat Pools** node and expand it to select a Skype for Business Server pool, or right-click the node and select **New Persistent Chat Pool**. Необходимо определить полное доменное имя пула и указать, будет ли пул развертываться с одним или несколькими серверами.
    
    Можно выбрать **Пул из нескольких компьютеров** или **Пул из одного компьютера**. Выберите первый вариант, если вы планируете использовать несколько серверов переднего плана в пуле серверов сохраняемой беседы. Сделайте этот выбор сейчас или позднее, потому что после создания пула с одним компьютером вы не сможете добавить в него дополнительные серверы. При выборе пула из нескольких компьютеров введите имена отдельных серверов переднего сервера, составляющих пул.
    
    > [!IMPORTANT]
    > Если роль сервера сохраняемого чата устанавливается на сервере Standard Edition, то FQDN должно соответствовать FQDN сервера Standard Edition. 
  
4. Определите простое **отображаемое имя** для пула серверов сохраняемой беседы. Отображаемые имена могут использоваться настраиваемые клиенты, особенно если существует несколько пулов серверов сохраняемой беседы для различения комнат.
    
5. Определите порт, используемый сервером сохраняемой беседы для связи с серверами переднего сервера Skype для бизнеса Server. Порт по умолчанию — 5041.
    
6. Если организация должна соблюдать требования, установите флажок **Включить соблюдение требований**. Если этот выбор выбран, служба соответствия сервера сохраняемой беседы устанавливается на том же компьютере, что и сервер переднего сервера сохраняемой беседы. Вам будет предложено выбрать сервер SQL Server для обеспечения соответствия серверу сохраняемой беседы позже.
    
7. Назначьте сходство сайтов для пула серверов сохраняемой беседы. Выберите **этот \<SiteName\>** пул по умолчанию для  проверки сайта или используйте этот пул по умолчанию для всех сайтов, чтобы назначить этот пул серверов сохраняемой беседы в качестве пула по умолчанию для текущего сайта или всех сайтов. Когда клиент Skype для бизнеса используется для создания комнат и управления ими, пул по умолчанию, связанный с сайтом пользователя, используется интерфейсом создания комнат и управления ими, чтобы он перенаправл операции создания комнат и управления в этот пул. Это применимо только в том случае, если развернуто несколько пулов серверов сохраняемого чата и необходимо использовать функции создания комнат и управления сервером сохраняемого чата.
    
    > [!IMPORTANT]
    > Вы можете настроить функции создания комнат и управления ими с помощью SDK для сервера сохраняемого чата. 
  
8. Определите SQL для серверного сервера сохраняемого чата **(где** хранится содержимое комнаты чата), выдав одно из следующих содержимого:
    
   - Чтобы использовать существующее SQL Server, в выпадаемом списке щелкните имя SQL Server, которое вы хотите использовать.
    
   - Чтобы указать новую базу SQL Server, нажмите кнопку **"Новый"** и в магазине "Определение нового **SQL"** выполните следующее:
    
   - В **SQL Server FQDN** укажите FQDN SQL Server, на котором необходимо создать новую SQL Server данных.
    
   - Либо выберите **Экземпляр по умолчанию** либо, чтобы указать другой экземпляр, выберите **Именованный экземпляр** и укажите нужный.
    
     > [!NOTE]
     > Подробные сведения о настройке резервных SQL Server для аварийного восстановления см. в подстроке "Настройка высокой доступности и аварийного восстановления для сервера сохраняемой беседы" в Skype для бизнеса [Server 2015.](../../deploy/deploy-high-availability-and-disaster-recovery/configure-hadr-for-persistent-chat.md) 
  
9. Определите хранилище SQL Server соответствия требованиям, если вы включили соответствие требованиям.
    
    > [!IMPORTANT]
    > Подробные сведения о настройке зеркал SQL Server для обеспечения высокой доступности для базы данных сервера сохраняемой беседы и базы данных соответствия для сервера сохраняемой беседы см. в подстроке "Настройка высокой доступности и аварийного восстановления для сервера сохраняемой беседы" в Skype для бизнеса [Server 2015.](../../deploy/deploy-high-availability-and-disaster-recovery/configure-hadr-for-persistent-chat.md) 
  
10. Определите хранилище файлов. Хранилище файлов — это папка, в которой хранится копия любого файла, загруженного в репозиторий файлов, например файлового вложения, отправленного в комнату чата. В случае топологии сервера сохраняемого чата с несколькими серверами это должен быть UNC-путь; а для топологии сервера сохраняемого чата с одним сервером это может быть локальный путь к файлу.
    
    Чтобы использовать существующее хранилище файлов, выполните следующие действия.
    
    - В **файловом сервере укажите FQDN** компьютера, на котором требуется создать новое хранилище файлов.
    
    - В поле **Файловый ресурс** укажите хранилище файлов, которое следует использовать.
    
      > [!IMPORTANT]
      > Хранилище файлов можно определить в построителье топологий перед созданием файлов, но необходимо создать хранилище файлов в определенном расположении перед публикацией топологии. Если хранилище файлов еще не существует, попытки опубликовать топологию не удастся. 
  
11. Выберите пул серверов переднего сервера, который будет использоваться в качестве следующего прыжка для этого пула серверов сохраняемой беседы. Это пул серверов переднего сервера, который сможет маршрутить запросы сервера сохраняемой беседы в этот пул.
    
12. Чтобы сохранить конфигурацию, нажмите кнопку **Готово**. Пул серверов сохраняемой беседы отображается в построитете топологий вместе с определенными настройками пула.
    
    Чтобы опубликовать обновленную топологию, в которую вы добавили сервер сохраняемой беседы, см. "Публикация обновленной топологии".
    
    > [!NOTE]
    > Если построитель топологий уже открыт, можно перейти к шагу 3 публикации обновленной топологии, чтобы начать публикацию обновленной топологии. 
  
## <a name="publish-the-updated-topology"></a>Публикация обновленной топологии
<a name="BKMK_PublishTopology"> </a>

После обновления топологии в построите топологий необходимо опубликовать топологию в центральном хранилище управления, прежде чем можно будет настроить и использовать Skype для бизнеса Server. Доступные только для чтения копии данных реплицируются на все серверы в топологии, чтобы обеспечивать синхронизацию всех серверов для внесения изменений топологии и других изменений конфигурации.
  
Перед публикацией топологии установите базы данных для сервера сохраняемой беседы. Использование построитель топологий для установки баз данных путем выбора действий **и** установки **базы данных.**
  
1. На компьютере со Skype для бизнеса Server или на котором установлены средства администрирования Skype для бизнеса Server, войдите в систему с помощью учетной записи, которая является членом группы администраторов домена и группы **RTCUniversalServerAdmins** и имеет разрешения на полный доступ (чтение, запись и изменение) в хранилище файлов, которое будет использоваться для хранения файлов сервера сохраняемого чата (чтобы построитель топологий может настроить необходимые списки управления доступом на основе дискреционного доступа (DACLs) или учетную запись с эквивалентными правами пользователя. 
    
2. Запустите построитель топологий. Выберите **"Открыть топологию" из локального файла,** если вы сохранили ее локально.
    
3. В дереве консоли щелкните правой кнопкой **мыши Skype для бизнеса Server 2015** и выберите "Опубликовать **топологию".**
    
4. На странице **Публикация топологии** щелкните **Далее**.
    
5. На странице **Завершение мастера публикации** убедитесь, что топология была успешно опубликована, а затем щелкните **Готово**.
