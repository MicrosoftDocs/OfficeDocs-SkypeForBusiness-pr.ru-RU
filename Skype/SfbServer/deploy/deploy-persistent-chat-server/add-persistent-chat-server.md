# <a name="add-persistent-chat-server-to-your-skype-for-business-server-2015-topology"></a>Добавление постоянного сервера чата в топологию Skype для бизнеса Server 2015 г.
 
**Сводка:** Ознакомьтесь с этой темой, чтобы узнать, как добавить постоянный сервер чата в топологию Skype для бизнеса Server 2015 г.
  
После установки необходимого программного обеспечения на каждом сервере, на котором планируется развертывание стойких серверов чата, вы используете Topology Builder для: 
  
- Обновите топологию, чтобы включить сервер сохраняемой системы чата
    
- Публикация обновленной топологии
    
> [!NOTE] 
> Постоянный чат доступен в Skype для бизнеса Server 2015 г., но больше не поддерживается Skype для бизнеса Server 2019 г. Такая же функциональность доступна в Teams. Дополнительные сведения см. в [ссылке Начало работы с Microsoft Teams обновления](/microsoftteams/upgrade-start-here). Если вам нужно использовать постоянный чат, вы можете либо перенести пользователей, требующих Teams, либо продолжить использование Skype для бизнеса Server 2015 г. 

## <a name="update-your-topology-to-include-persistent-chat-server"></a>Обновите топологию, чтобы включить сервер сохраняемой системы чата

Выполните следующие действия для установки единого пула стойких чат-серверов без конфигурации аварийного восстановления. Настройка растянутой пула стойких серверов чата для высокой доступности и аварийного восстановления см. в этой рубке Настройка высокой доступности и аварийного восстановления для persistent [Chat Server в Skype для бизнеса Server 2015](../../deploy/deploy-high-availability-and-disaster-recovery/configure-hadr-for-persistent-chat.md) г.
  
Чтобы развернуть несколько пулов сохраняемой системы чата, повторите один и тот же процесс для каждого пула.
  
1. На компьютере, на котором Skype для бизнеса Server или на котором установлены Skype для бизнеса Server администратора, войдите в систему с помощью учетной записи, которая является членом локальной группы пользователей (или учетной записи с эквивалентными правами пользователя).
    
    > [!NOTE]
    > Топологию можно определить с помощью учетной записи, в которую входит локальная группа пользователей, но для публикации топологии, необходимой для установки Skype для бизнеса Server, необходимо использовать учетную запись, которая входит в группу администраторов домена  и **RTCUniversalServerAdmins**.  группа, которая имеет полные разрешения управления (чтение, запись и изменение) в хранилище файлов, который вы собираетесь использовать для хранилище файлов сохраняемой системы чата (чтобы topology Builder можно настроить необходимые DACLs) или учетную запись с эквивалентными правами.
  
2. Запустите построитель топологий.
    
3. В дереве консоли перейдите к узлу  Постоянные пулы чата и расширйте его, чтобы выбрать пул Skype для бизнеса Server или щелкнуть правой кнопкой мыши узел и выбрать новый пул постоянных **чатов**. Необходимо определить полное доменное имя пула (FQDN) и указать, будет ли пул одним сервером или развертыванием пула с несколькими серверами.
    
    Можно выбрать **Пул из нескольких компьютеров** или **Пул из одного компьютера**. Выберите первый, если вы планируете иметь несколько серверов переднего плана в пуле стойких чат-серверов. Сделайте этот выбор сейчас или позднее, потому что после создания пула с одним компьютером вы не сможете добавить в него дополнительные серверы. Если вы выбираете несколько пулов компьютеров, введите имена отдельных серверов переднего конца, которые входят в пул.
    
    > [!IMPORTANT]
    > Если роль сохраняемого сервера чата устанавливается на выпуск Standard сервере, FQDN должен соответствовать FQDN выпуск Standard сервера. 
  
4. Определите **простое имя отображения для** пула сохраняемой сервера чата. Имя дисплея может использоваться настраиваемые клиенты, особенно если существует несколько пулов сохраняемой сервера чата для дифференцировать комнаты.
    
5. Определите порт, используемый стойким сервером чата для связи с Skype для бизнеса Server серверами переднего конца. Порт по умолчанию — 5041.
    
6. Если организация должна соблюдать требования, установите флажок **Включить соблюдение требований**. Если выбрано, служба соответствия требованиям серверов чата установлена на том же компьютере, что и сервер сохраняемой серверной части чата. Вам будет предложено выбрать SQL Server сервер для постоянного соответствия требованиям сервера чата позже.
    
7. Назначение сродства сайта для пула серверов сохраняемой переписки. Выберите этот пул по умолчанию для проверки сайта или используйте этот  пул в качестве по умолчанию для всех сайтов, чтобы назначить этот пул сохраняемой сервера чата пулом по умолчанию для текущего сайта или всех сайтов.**\<SiteName\>** Если клиент Skype для бизнеса для создания и управления комнатами, пул по умолчанию, связанный с сайтом пользователя, используется опытом создания и управления комнатами, чтобы он может маршрутить операции по созданию и управлению комнатами в этот пул. Это применяется только в том случае, если развернуто несколько пулов сохраняемого сервера чата, и вы хотите использовать функции создания комнаты и управления сохраняемого сервера чата.
    
    > [!IMPORTANT]
    > Вы можете настроить функции создания комнаты и управления с помощью комплекта разработки программного обеспечения для серверов чата (SDK). 
  
8. Определите **SQL для** постоянного заднего конца чат-сервера (где хранится содержимое комнаты чата) с помощью одного из следующих ниже.
    
   - Чтобы использовать существующий SQL Server, в списке выпаданий нажмите имя SQL Server магазина, который вы хотите использовать.
    
   - Чтобы указать новую базу данных SQL Server, щелкните **New** и в **магазине Define New SQL** выполните следующее:
    
   - В **SQL Server FQDN** укажите FQDN SQL Server, на котором необходимо создать новую базу данных SQL Server.
    
   - Либо выберите **Экземпляр по умолчанию** либо, чтобы указать другой экземпляр, выберите **Именованный экземпляр** и укажите нужный.
    
     > [!NOTE]
     > Подробные сведения о настройке SQL Server резервных баз данных для аварийного восстановления см. в материале [Configure high availability and disaster recovery for Persistent Chat Server в Skype для бизнеса Server 2015 г](../../deploy/deploy-high-availability-and-disaster-recovery/configure-hadr-for-persistent-chat.md). 
  
9. Определите SQL Server, если вы включили соответствие требованиям.
    
    > [!IMPORTANT]
    > Подробные сведения о настройке зеркал SQL Server для высокой доступности для базы данных persistent Chat Server и базы данных соответствия серверу сохраняемой системы чата см. в материале [Configure high availability and disaster recovery for Persistent Chat Server в Skype для бизнеса Server 2015](../../deploy/deploy-high-availability-and-disaster-recovery/configure-hadr-for-persistent-chat.md) г. 
  
10. Определите хранилище файлов. Хранилище файлов — это папка, в которой хранится копия любого файла, загруженного в репозиторий файлов, например файлового вложения, отправленного в комнату чата. В случае топологии с несколькими серверами сохраняемого сервера чата это должен быть путь универсальной конвенции имен (UNC). а для топологии сохраняемого сервера чата с одним сервером это может быть локальный путь к файлам.
    
    Чтобы использовать существующее хранилище файлов, выполните следующие действия.
    
    - В **файловом сервере FQDN** укажите FQDN компьютера, на котором необходимо создать новый файловый магазин.
    
    - В поле **Файловый ресурс** укажите хранилище файлов, которое следует использовать.
    
      > [!IMPORTANT]
      > Перед созданием магазина файлов можно определить хранилище файлов в Topology Builder, но перед публикацией топологии необходимо создать хранилище файлов в определенном расположении. Если хранилище файлов еще не существует, попытки опубликовать топологию сбой. 
  
11. Выберите пул переднего сервера, который будет использоваться в качестве следующего перехода для этого пула сохраняемой сервера чата. Это пул переднего сервера, который сможет маршрутить запросы на стойкий сервер чата в этот пул.
    
12. Чтобы сохранить конфигурацию, нажмите кнопку **Готово**. Пул стойких чат-серверов отображается в Topology Builder в сопровождении определенных параметров пула.
    
    Чтобы опубликовать обновленную топологию, к которой добавлен постоянный сервер чата, см. в публикации обновленной топологии.
    
    > [!NOTE]
    > С помощью topology Builder вы можете приступить к шагу 3 в Публикации обновленной топологии, чтобы начать публикацию обновленной топологии. 
  
## <a name="publish-the-updated-topology"></a>Публикация обновленной топологии
<a name="BKMK_PublishTopology"> </a>

После обновления топологии в Topology Builder необходимо опубликовать топологию в центральном хранилище управления, прежде чем можно настроить и использовать Skype для бизнеса Server. Доступные только для чтения копии данных реплицируются на все серверы в топологии, чтобы обеспечивать синхронизацию всех серверов для внесения изменений топологии и других изменений конфигурации.
  
Перед публикацией топологии установите базы данных для сохраняемой системы чат-сервера. Используйте Topology Builder для установки баз данных, выбрав **Action** и **Install Database**.
  
1. На компьютере, на котором Skype для бизнеса Server или на котором установлены Skype для бизнеса Server административные средства, войдите в систему с помощью учетной записи, которая входит в группу администраторов домена и **RTCUniversalServerAdmins**.  группа, которая имеет разрешения на полный контроль (чтение, запись и изменение) в хранилище файлов, который будет использоваться для хранилище файлов сохраняемой системы чат-сервера (чтобы строитель топологии может настроить необходимые списки управления дискреционным доступом (DACLs)) или учетную запись с равными правами пользователя.
    
2. Запустите построитель топологий. Выберите **Open Topology из локального файла** , если вы сохранили его локально.
    
3. В дереве консоли щелкните правой кнопкой мыши **Skype для бизнеса Server 2015 г.**, а затем нажмите **кнопку Опубликовать топологию**.
    
4. На странице **Публикация топологии** щелкните **Далее**.
    
5. На странице **Завершение мастера публикации** убедитесь, что топология была успешно опубликована, а затем щелкните **Готово**.
