# <a name="add-persistent-chat-server-to-your-skype-for-business-server-2015-topology"></a>Добавление сервера сохраняемого чата для вашей Скайп для топологии Business Server 2015
 
**Сводка:** Прочтите этот раздел, чтобы узнать, как добавление сервера сохраняемого чата для вашей Скайп для топологии Business Server 2015.
  
После установки необходимого программного обеспечения на каждом сервере, на котором планируется развернуть сервера сохраняемого чата, необходимо использовать построитель топологий для: 
  
- Обновление топологии для включения сервера сохраняемого чата
    
- Публикация обновленной топологии
    
> [!NOTE] 
> Сохраняемый чат доступна в Скайп для Business Server 2015, но больше не поддерживается в Скайп для Business Server 2019. Те же функциональные возможности доступны в группах. Для получения дополнительных сведений см [Реализация из Скайп для бизнеса для групп Майкрософт](/microsoftteams/journey-skypeforbusiness-teams). Если необходимо использовать сохраняемого чата, возможны либо перенос пользователей, которым требуется эта функция групп, или для дальнейшего использования Скайп для Business Server 2015. 

## <a name="update-your-topology-to-include-persistent-chat-server"></a>Обновление топологии для включения сервера сохраняемого чата

Выполните следующие действия для установки один пул серверов сохраняемого чата без конфигурацию аварийного восстановления. Настройка вытянутый пул серверов сохраняемого чата для обеспечения высокой доступности и аварийного восстановления, в разделе [Настройка высокой доступности и аварийного восстановления для сервера сохраняемого чата в Скайп для Business Server 2015](../../deploy/deploy-high-availability-and-disaster-recovery/configure-hadr-for-persistent-chat.md).
  
Чтобы развернуть несколько пулов сервера сохраняемого чата, повторите тот же самый процесс для каждого пула.
  
1. На компьютере, на котором работает Скайп для Business Server или на котором Скайп для установлены средства администрирования Business Server, выполните вход с помощью учетной записи, которая является членом локальной группы пользователей (или имеет эквивалентные права пользователя).
    
    > [!NOTE]
    > Вы можете определить топологию с помощью учетной записи, которая является членом локальной группы пользователей, но для публикации топологии, которая требуется для установки Скайп для Business Server, необходимо использовать учетную запись, которая является членом группы **Администраторов домена** и ** RTCUniversalServerAdmins** группы, который имеет разрешения на полный доступ (чтение, запись и изменение) на хранилище файлов, который будет использоваться для сервера сохраняемого чата хранилище файлов (то есть, Topology Builder можно настроить требуемые доступом), или учетной записи с эквивалентные права.
  
2. Запустите построитель топологий.
    
3. В дереве консоли перейдите к узлу **Пулы Persistent Chat** и развернуть его для выбора Скайп для пула Business Server, или щелкните правой кнопкой мыши узел и выберите **Нового пула сохраняемого чата**. Определение пула полное доменное имя (FQDN) и указать, будет ли пула пула отдельного сервера или развертывания нескольких серверов пула.
    
    Можно выбрать **Пул из нескольких компьютеров** или **Пул из одного компьютера**. Выберите первое, если вы планируете имеют более одного сервера переднего плана в пуле серверов сохраняемого чата. Сделайте этот выбор сейчас или позднее, потому что после создания пула с одним компьютером вы не сможете добавить в него дополнительные серверы. Если вы решаете пул на нескольких компьютерах, введите имена отдельных серверов переднего плана, образующих пула.
    
    > [!IMPORTANT]
    > Если роль сервера сохраняемого чата установлена на сервере Standard Edition, полное доменное имя должно соответствовать полное доменное имя сервера Standard Edition. 
  
4. Определение простое **Отображаемое имя** для пула сервера сохраняемого чата. Отображаемое имя можно использовать настраиваемые клиентами, особенно в том случае, если существует несколько пулов сервера сохраняемого чата для различения комнат.
    
5. Определите порт, используемый с сервера сохраняемого чата для взаимодействия с Скайп для серверов переднего плана Business Server. Порт по умолчанию: 5041.
    
6. Если организация должна соблюдать требования, установите флажок **Включить соответствие**. Если выбран, служба соответствия Persistent Chat Server устанавливается на том же компьютере Persistent Chat Server сервер переднего плана. Запрос на выбор SQL Server Тыловой сервер для соответствия Persistent Chat Server более поздней версии.
    
7. Назначьте сходства сайта для пула сервера сохраняемого чата. Выберите **использовать этот пул по умолчанию для сайта \<SiteName\> ** флажок или **использовать этот пул по умолчанию для всех сайтов** , чтобы определить этот пул серверов сохраняемого чата как пул по умолчанию для текущего сайта или все сайты. При Скайп для клиента Business используется для создания комнат и управления ими, по умолчанию пула, связанного с сайтом пользователя используется качества комнаты создание и управление, чтобы его маршрутизировать операций создания и управления комнаты с этим пулом. Применяется только после несколько пулов сервера сохраняемого чата развернут и требуется использовать возможности создания и управления комнаты сервера сохраняемого чата.
    
    > [!IMPORTANT]
    > Можно настроить с помощью Persistent Chat Server программного обеспечения разработки SDK для функции создания и управления комнаты. 
  
8. Определите **хранилище SQL для сохраняемого чата внутреннего сервера (где хранится содержимое комнат чата)** , выполнив одно из следующих действий.
    
   - Чтобы использовать существующее хранилище SQL Server, в раскрывающемся списке щелкните имя хранилища SQL Server, который требуется использовать.
    
   - Чтобы указать новую базу данных SQL Server, нажмите кнопку **Создать**и в **Окне Определение нового хранилища SQL**, сделайте следующее:
    
   - В **Поле полное имя SQL Server**укажите полное доменное имя SQL Server, на котором необходимо создать новую базу данных SQL Server.
    
   - Выберите **Экземпляр по умолчанию**, чтобы использовать экземпляр по умолчанию, или **Именованный экземпляр**, чтобы указать другой экземпляр, и укажите экземпляр, который требуется использовать.
    
    > [!NOTE]
    > Для получения дополнительных сведений о настройке резервного копирования базы данных SQL Server для аварийного восстановления видеть [Настройка высокой доступности и аварийного восстановления для сервера сохраняемого чата в Скайп для Business Server 2015](../../deploy/deploy-high-availability-and-disaster-recovery/configure-hadr-for-persistent-chat.md). 
  
9. Определение хранилища соответствия SQL Server, если включено соответствие требованиям.
    
    > [!IMPORTANT]
    > Для получения дополнительных сведений о настройке зеркала SQL Server для обеспечения высокой доступности для базы данных сервера сохраняемого чата и база данных соответствия Persistent Chat Server см [Настройка высокой доступности и аварийного восстановления для сервера сохраняемого чата в Скайп для Business Server 2015](../../deploy/deploy-high-availability-and-disaster-recovery/configure-hadr-for-persistent-chat.md). 
  
10. Определите хранилище файлов. Хранилище файлов — это папка, в которой хранится копия любого файла, загруженного в репозиторий файлов, например файлового вложения, отправленного в комнату чата. В случае топологии серверов сохраняемого чата нескольких серверов это должно быть путь соглашения об универсальных именах (UNC); и для топологии серверов сохраняемого чата одного сервера, может быть путь к локальному файлу.
    
    Чтобы использовать существующее хранилище файлов, выполните следующие действия.
    
    - В поле **Полное доменное имя файлового сервера** укажите полное доменное имя компьютера, на котором следует создать новое хранилище файлов.
    
    - В поле **Файловый ресурс** укажите хранилище файлов, которое следует использовать.
    
     > [!IMPORTANT]
     > Можно определить хранилище файлов в построителе топологий, прежде чем создать хранилище файлов, но необходимо создать хранилище файлов в определенном месте, определить до публикации топологии. Если хранилища файлов еще нет, опубликовать топологию не удастся. 
  
11. Выберите пул сервера переднего плана для использования в качестве следующего прыжка для этого пула серверов сохраняемого чата. Это пула серверов переднего плана, который будет иметь возможность маршрутизации запросов сервера сохраняемого чата в этот пул.
    
12. Чтобы сохранить конфигурацию, нажмите кнопку **Готово**. Пул серверов сохраняемого чата появится в построителе топологий сопровождаться параметры конкретного пула.
    
    Публикация обновленной топологии, к которому был добавлен сервера сохраняемого чата, ознакомиться с разделом Publish обновленной топологии.
    
    > [!NOTE]
    > С помощью Topology Builder открыто можно перейти к шагу 3 в публикации обновленной топологии для публикации обновленной топологии. 
  
## <a name="publish-the-updated-topology"></a>Публикация обновленной топологии
<a name="BKMK_PublishTopology"> </a>

После обновления топологии в построителе топологий, необходимо опубликовать топологию центрального хранилища управления перед настройкой и использовать Скайп для Business Server. Копии данных, доступные только для чтения, реплицируются на все серверы в топологии, чтобы они были синхронизированы с топологией и другими изменениями конфигурации.
  
Перед публикацией топологии Установка баз данных для сервера сохраняемого чата. Для установки баз данных, выбрав **Действие** и **Установка базы данных**с помощью построителя топологии.
  
1. На компьютере, на котором работает Скайп для Business Server или на котором Скайп для установлены средства администрирования Business Server, выполните вход с помощью учетной записи, которая является членом группы **Администраторов домена** и группу **RTCUniversalServerAdmins** и имеет разрешения на полный доступ (чтение, запись и изменение) в хранилище файлов, которое будет использоваться для сервера сохраняемого чата хранилище файлов (то есть, Topology Builder можно настроить списки управления доступом на уровне пользователей требуемые (доступом)), или учетной записи с эквивалентный пользователя права.
    
2. Запустите построитель топологий. Выберите **Открыть топологию из локального файла**, если она сохранена локально.
    
3. В дереве консоли щелкните правой кнопкой мыши **Скайп для Business Server 2015**и нажмите кнопку **Опубликовать топологию**.
    
4. На странице **Публикация топологии** нажмите кнопку **Далее**.
    
5. На странице **Работа мастера публикации завершена** убедитесь, что топология успешно опубликована, а затем щелкните **Готово**.
