# <a name="deploy-archiving-for-skype-for-business-server"></a>Развертывание архивации в Skype для бизнеса Server
 
**Сводка:** В этой статье рассказывается, как развернуть архивирование для Skype для бизнеса Server.
  
Архивация автоматически устанавливается на каждый сервер переднего плана в развертывании Skype для бизнеса Server, но для его использования вам по-прежнему нужно выполнить начальную настройку и этапы настройки. Прежде чем начать, убедитесь, что вы знакомы с концепциями, изложенными в разделе [Планирование архивации в Skype для бизнеса Server](../../plan-your-deployment/archiving/archiving.md).
  
## <a name="deployment-checklist"></a>Контрольный список развертывания

Настройка архивации зависит от выбранного варианта хранилища. 
  
- Если вы используете Microsoft Exchange Integration для всех пользователей в развертывании, вам не нужно настраивать политики архивации в Skype для бизнеса Server для пользователей. Вместо этого вы настраиваете политики хранения на месте Exchange для поддержки архивирования для пользователей, размещенных на Exchange, с почтовыми ящиками, которые помещаются на хранение на месте. Подробные сведения о настройке этих политик можно найти в документации по продукту Exchange.
    
- Если вы не используете интеграцию Microsoft Exchange для всех пользователей в развертывании, необходимо добавить в топологию базы данных архивации в Skype для бизнеса Server (базы данных SQL Server), опубликовать обновленную топологию, а затем настроить политики архивации и параметры пользователей. Базы данных архивации можно разворачивать одновременно с первоначальной топологией или после развертывания хотя бы одного пула переднего плана или сервера Standard Edition. В этом документе описывается порядок развертывания баз данных архивации путем добавления их в существующее развертывание.
    
При включении архивации в одном пуле переднего плана или сервере Standard Edition следует включить ее для всех других пулов переднего плана и серверов Standard Edition вашего развертывания. Это следует сделать, поскольку пользователи, чьи коммуникации требуется архивировать, могут быть приглашены в текстовую беседу или на собрания, размещаемые в другом пуле. Если там архивация не включена, то сеанс может быть архивирован не полностью. В этих случаях мгновенные сообщения пользователей с включенной архивацией могут быть архивированы, но не для файлов содержимого конференций, а также событий присоединения и выхода из конференций.
  
> [!IMPORTANT]
> Если в вашей организации важна архивация в целях обеспечения соответствия требованиям, не забудьте развернуть архивацию, настроить политики и другие параметры на нужном уровне, а затем включите архивацию для всех нужных пользователей, прежде чем включать эти пользователи в Skype для Business Server.
