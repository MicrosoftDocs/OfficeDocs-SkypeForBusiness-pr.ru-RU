# <a name="deploy-monitoring-in-skype-for-business-server"></a>Развертывание мониторинга в Skype для бизнеса Server

**Сводка:** Узнайте, как развернуть мониторинг в Skype для бизнеса Server.

Перед выполнением этих задач просмотрите [План мониторинга в Skype для бизнеса Server.](../../plan-your-deployment/monitoring.md)

Обычно в топологии будут реализованы службы мониторинга, завершив следующие два шага:

1. Включение мониторинга одновременно с созданием нового пула Skype для бизнеса Server. (В Skype для бизнеса Server мониторинг включен или отключен на основе пула по пулу.) Обратите внимание, что вы можете включить мониторинг для пула, не собирая фактически данные мониторинга, процесс, который объясняется в разделе Настройка записи параметров вызова и качества этой документации.

2. Свяжите хранилище данных мониторинга (то есть базы данных мониторинга) с новым пулом. Обратите внимание, что одно и то же хранилище данных мониторинга можно связать с несколькими пулами. В зависимости от количества пользователей, размещенных в пулах регистраторов, бассейны, это означает, что не требуется настраивать отдельную базу данных мониторинга для каждого пула. Вместо этого несколько пулов могут использовать одно и то же хранилище данных мониторинга.

Хотя зачастую проще включить мониторинг одновременно с созданием нового пула, также можно создать пул с отключенным мониторингом. В этом случае вы сможете использовать построитель топологии для включения службы в дальнейшем: построитель топологии позволяет включить или отключить мониторинг для пула или связать пул с другим хранилищем данных мониторинга. Имейте в виду, что несмотря на то, что роль Сервера мониторинга больше не используется, вам все равно потребуется создать один или несколько магазинов мониторинга: базы данных, используемые для хранения данных, собранных службой мониторинга. Эти базы данных можно создать с Microsoft SQL Server R2, Microsoft SQL Server 2012, Microsoft SQL Server 2014 или Microsoft SQL Server 2019.

> [!NOTE]
> Если мониторинг включен для пула, вы можете отключить процесс сбора данных мониторинга, не изменяя топологию: Skype для бизнеса Server позволяет отключить (а затем повторно включить) запись данных о звонках (CDR) или сбор данных Качества работы (QoE). Дополнительные сведения см. в разделе "Настройка регистрации звонков и параметров качества взаимодействия" данной документации.

Еще одним важным улучшением мониторинга в Skype для бизнеса Server является тот факт, что отчеты по мониторингу Skype для бизнес-серверов теперь поддерживают IPv6: отчеты, в которых используется поле IP-адресов, будут отображаться либо адреса IPv4, либо IPv6 в зависимости от : 1) используемого SQL запроса; и 2) где или не хранится адрес IPv6 в базе данных мониторинга.

> [!NOTE]
> Убедитесь, что тип запуска службы SQL Server агентов является автоматическим, а служба агентов SQL Server работает для экземпляра SQL, в котором находятся базы данных мониторинга, с тем чтобы задания мониторинга SQL Server по умолчанию могли запускаться по умолчанию под контролем службы агентов SQL Server.

В этой документации вы проходите процесс установки и настройки отчетов по мониторингу и мониторингу для Skype для бизнеса Server. В документации представлены пошаговые инструкции, позволяющие выполнить следующие действия.

- Включить мониторинг в вашей топологии и связать хранилище данных мониторинга с интерфейсным пулом.

- Установите SQL Server службы отчетности и отчеты по мониторингу Skype для бизнес-серверов. Отчеты мониторинга — это предварительно настроенные отчеты, предоставляющие различные представления сведений в базе данных мониторинга.

- Настройка сбора данных для записи деталей вызовов (CDR) и качества работы (QoE). Запись детализации вызовов позволяет отслеживать использование возможностей Skype для бизнес-сервера, таких как телефонные вызовы Voice over IP (VoIP). обмен мгновенными сообщениями (IM); перенос файлов; аудио- и видеоконференция; и сеансы совместного использования приложений. Показатели QoE отслеживают качество аудио- и видеозвонков в вашей организации, в том числе такие данные, как число потерянных сетевых пакетов, фоновый шум и объем "дрожания" (разницы задержек пакетов).

- Вручную удалить записи CDR и QoE из базы данных мониторинга.

## <a name="enable-monitoring"></a>Включить мониторинг

Хотя унифицированные агенты по сбору данных автоматически устанавливаются и активируются на каждом переднем сервере, это не означает, что вы автоматически начнете собирать данные мониторинга сразу после установки Skype для бизнеса Server. Вместо этого необходимо сделать две вещи: необходимо связать свои пулы переднего и переднего конечного серверов с базой данных мониторинга, а также включить запись детализации вызовов (CDR) и(или) мониторинг качества работы (QoE) в глобальной области и/или области сайта.

Инструкции по пошаговой связи серверов переднего или переднего конца с базой данных мониторинга см. в разделе Связать хранилище мониторинга с пулом переднего конца в [Skype для](associate-a-monitoring-store.md) бизнеса Server в руководстве по развертыванию. После этих ассоциаций и публикации новой топологии Skype для бизнес-сервера вы все равно не сможете собирать данные мониторинга. Это происходит потому, что по умолчанию при установке Skype для бизнес-сервера отключается как коллекция данных CDR, так и QoE.

Для начала сбора данных необходимо включить мониторинг CDR и/или QoE. (Обратите внимание, что не нужно включить мониторинг CDR и QoE; если вы хотите, вы можете включить один тип мониторинга, оставив другой тип отключен.) Чтобы включить мониторинг CDR в глобальной области, запустите следующую команду из командной оболочки Skype для бизнес-серверов:

```powershell
Set-CsCdrConfiguration -Identity "global" -EnableCDR $True
```

Кроме того, можно включить мониторинг CDR из панели управления Skype для бизнес-серверов. В панели управления Skype для бизнес-серверов выполните следующую процедуру:

1. Щелкните **Мониторинг**.

2. На вкладке **Регистрация вызовов** дважды щелкните параметр **Глобальные**.

3. В области **Изменить настройку регистрации вызовов (CDR)** выберите **Разрешить мониторинг регистрации вызовов** и нажмите **Зафиксировать**.

Чтобы включить мониторинг QoE в глобальном масштабе, запустите эту команду из командной оболочки Skype для бизнес-серверов:

```powershell
Set-CsQoEConfiguration -Identity "global" -EnableQoE $True
```

Если вы хотите, вы также можете включить мониторинг QoE из панели управления Skype для бизнес-серверов. На панели управления выполните следующую процедуру:

1. Щелкните **Мониторинг**.

2. На вкладке **Данные о качестве взаимодействия** дважды щелкните параметр **Глобальные**.

3. В области **Изменить настройку качества взаимодействия (CDR)** выберите **Разрешить мониторинг качества взаимодействия** и нажмите **Зафиксировать**.

Как уже отмечено ранее, предыдущие примеры демонстрируют включение мониторинга в глобальной области, то есть, включение мониторинга регистрации вызовов и качества взаимодействия в пределах организации. Как вариант, можно также создать отдельные параметры конфигурации регистрации вызовов и качества взаимодействия в области сайта, после чего выборочно включить или отключить мониторинг для каждого из сайтов. Например, можно включить мониторинг регистрации вызовов для сайта Redmond и отключить при этом мониторинг регистрации вызовов для сайта Dublin. Дополнительные сведения об управлении настройками мониторинга см. в разделе Руководство по развертыванию Настройка записи сведений о вызове и параметры качества работы в [Skype для бизнеса Server.](call-detail-recording-and-qoe.md)
