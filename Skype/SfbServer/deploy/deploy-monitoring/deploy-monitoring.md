# <a name="deploy-monitoring-in-skype-for-business-server"></a>Развертывание мониторинга в Skype для бизнеса Server

**Сводка:** Узнайте, как развернуть мониторинг в Skype для бизнеса Server.

Перед выполнением этих задач просмотрите план [мониторинга в Skype для бизнеса Server.](../../plan-your-deployment/monitoring.md)

Как правило, службы мониторинга в топологии реализуются путем выполнения следующих двух действий:

1. Включение мониторинга одновременно с настройками нового пула Skype для бизнеса Server. (В Skype для бизнеса Server мониторинг включен или отключен для пула по пулу.) Обратите внимание, что вы можете включить мониторинг для пула, не собирая данные мониторинга, процесс, который объясняется в разделе "Настройка регистрации вызовов и параметров качества работы" данной документации.

2. Свяжите хранилище данных мониторинга (то есть базы данных мониторинга) с новым пулом. Обратите внимание, что одно и то же хранилище данных мониторинга можно связать с несколькими пулами. В зависимости от количества пользователей, размещенных в пулах регистраторов, бассейны, это означает, что не требуется настраивать отдельную базу данных мониторинга для каждого пула. Вместо этого несколько пулов могут использовать одно и то же хранилище данных мониторинга.

Хотя зачастую проще включить мониторинг одновременно с созданием нового пула, также можно создать пул с отключенным мониторингом. В этом случае вы сможете использовать построитель топологии для включения службы в дальнейшем: построитель топологии позволяет включить или отключить мониторинг для пула или связать пул с другим хранилищем данных мониторинга. Помните, что хотя роль сервера мониторинга больше не существует, вам все равно потребуется создать одно или несколько хранилищ мониторинга: серверные базы данных, используемые для хранения данных, собранных службой мониторинга. Эти базы данных можно создать с помощью Microsoft SQL Server 2008 R2, Microsoft SQL Server 2012, Microsoft SQL Server 2014 или Microsoft SQL Server 2019.

> [!NOTE]
> Если мониторинг включен для пула, можно отключить процесс сбора данных мониторинга, не изменяя топологию: Skype для бизнеса Server позволяет отключить (а затем повторно включить) регистрацию вызовов (CDR) или сбор данных качества обслуживания (QoE). Дополнительные сведения см. в разделе "Настройка регистрации звонков и параметров качества взаимодействия" данной документации.

Еще одно важное улучшение мониторинга в Skype для бизнеса Server состоит в том, что отчеты мониторинга Skype для бизнеса Server теперь поддерживают IPv6: отчеты, в которых используется поле IP-адреса, будут отображать IPv4- или IPv6-адреса в зависимости от : 1) используемого SQL запроса; и 2) где или не хранится IPv6-адрес в базе данных мониторинга.

> [!NOTE]
> Убедитесь, что тип запуска службы агента SQL Server автоматический, а служба агентов SQL Server запущена для экземпляра SQL, в котором находятся базы данных мониторинга, чтобы задания по умолчанию для мониторинга SQL Server обслуживания могли запускаться по расписанию под управлением службы агентов SQL Server.

В этой документации вы можете установить и настроить отчеты мониторинга и мониторинга для Skype для бизнеса Server. В документации представлены пошаговые инструкции, позволяющие выполнить следующие действия.

- Включить мониторинг в вашей топологии и связать хранилище данных мониторинга с интерфейсным пулом.

- Установите SQL Server reporting Services и отчеты мониторинга Skype для бизнеса Server. Отчеты мониторинга — это предварительно настроенные отчеты, предоставляющие различные представления сведений в базе данных мониторинга.

- Настройка сбора данных регистрации вызовов (CDR) и качества обслуживания (QoE). Регистрация вызовов позволяет отслеживать использование возможностей Skype для бизнеса Server, таких как телефонные звонки по voIP; обмен мгновенными сообщениями; передача файлов; аудио- и видеоконференцию; и сеансы общего доступа к приложениям. Показатели QoE отслеживают качество аудио- и видеозвонков в вашей организации, в том числе такие данные, как число потерянных сетевых пакетов, фоновый шум и объем "дрожания" (разницы задержек пакетов).

- Вручную удалить записи CDR и QoE из базы данных мониторинга.

## <a name="enable-monitoring"></a>Включить мониторинг

Хотя агенты единого сбора данных автоматически устанавливаются и активируются на каждом сервере переднего сервера, это не означает, что сбор данных мониторинга начнется автоматически сразу после завершения установки Skype для бизнеса Server. Вместо этого необходимо сделать два действия: необходимо связать серверы переднего и переднего серверов с базой данных мониторинга и включить мониторинг регистрации вызовов (CDR) и /или качества обслуживания (QoE) на глобальном уровне и/или уровне сайта.

Пошаговая инструкция по связыванию серверов переднего и переднего серверов с базой данных мониторинга см. в разделе "Связывание хранения данных мониторинга с пулом переднего сервера в Skype для бизнеса [Server"](associate-a-monitoring-store.md) руководства по развертыванию. After these associations have been made, and after your new Skype for Business Server topology has been published, you will still not be able to collect monitoring data. Это происходит потому, что по умолчанию сбор данных CDR и QoE отключен при установке Skype для бизнеса Server.

Чтобы начать сбор данных, необходимо включить мониторинг CDR и/или QoE. (Обратите внимание, что не нужно включить как мониторинг CDR, так и QoE. При этом можно включить один тип мониторинга, оставив другой отключенным.) Чтобы включить мониторинг CDR на глобальном уровне, запустите следующую команду в командной оболочке Skype для бизнеса Server:

```powershell
Set-CsCdrConfiguration -Identity "global" -EnableCDR $True
```

Кроме того, можно включить мониторинг CDR в панели управления Skype для бизнеса Server. В панели управления Skype для бизнеса Server выполните следующую процедуру:

1. Щелкните **Мониторинг**.

2. На вкладке **Регистрация вызовов** дважды щелкните параметр **Глобальные**.

3. В области **Изменить настройку регистрации вызовов (CDR)** выберите **Разрешить мониторинг регистрации вызовов** и нажмите **Зафиксировать**.

Чтобы включить мониторинг качества связи на глобальном уровне, в командной оболочке Skype для бизнеса Server запустите команду:

```powershell
Set-CsQoEConfiguration -Identity "global" -EnableQoE $True
```

При этом вы также можете включить мониторинг качества работы в панели управления Skype для бизнеса Server. На панели управления выполните следующую процедуру:

1. Щелкните **Мониторинг**.

2. На вкладке **Данные о качестве взаимодействия** дважды щелкните параметр **Глобальные**.

3. В области **Изменить настройку качества взаимодействия (CDR)** выберите **Разрешить мониторинг качества взаимодействия** и нажмите **Зафиксировать**.

Как уже отмечено ранее, предыдущие примеры демонстрируют включение мониторинга в глобальной области, то есть, включение мониторинга регистрации вызовов и качества взаимодействия в пределах организации. Как вариант, можно также создать отдельные параметры конфигурации регистрации вызовов и качества взаимодействия в области сайта, после чего выборочно включить или отключить мониторинг для каждого из сайтов. Например, можно включить мониторинг регистрации вызовов для сайта Redmond и отключить при этом мониторинг регистрации вызовов для сайта Dublin. Дополнительные сведения об управлении настройками конфигурации мониторинга см. в разделе "Руководство по развертыванию" "Настройка регистрации вызовов и параметров качества работы [в Skype для бизнеса Server".](call-detail-recording-and-qoe.md)
