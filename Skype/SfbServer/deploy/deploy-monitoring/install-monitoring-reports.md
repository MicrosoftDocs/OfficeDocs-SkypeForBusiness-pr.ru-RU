# <a name="install-monitoring-reports-in-skype-for-business-server"></a>Установка отчетов мониторинга в Skype для бизнеса Server
 
**Сводка:** Узнайте, как установить службу, которая будет создавать отчеты мониторинга в Skype для бизнеса Server.
  
Skype для бизнеса Server Отчеты мониторинга предоставляют вам множество сведений о качестве и количестве сеансов связи, которые проходят в вашей организации. 
  
## <a name="install-monitoring-reports"></a>Установка отчетов мониторинга

Отчеты мониторинга не устанавливаются автоматически при установке Skype для бизнеса Server; Вместо этого необходимо установить отчеты мониторинга отдельно и только после Skype для бизнеса Server установлен на компьютере.
  
> [!NOTE]
> Рекомендуется устанавливать отчеты мониторинга на тот компьютер, на котором установлена база данных мониторинга. Это упрощает процесс назначения разрешений на доступ к отчетам: установка отчетов наблюдения на компьютер, на котором расположено хранилище данных наблюдения, означает, что не придется настраивать разрешения, которые позволяют базе данных на одном компьютере взаимодействовать со службами отчетов на другом компьютере. 
  
Skype для бизнеса Server Отчеты мониторинга включают более 30 отчетов, предназначенных для предоставления подробных сведений о конференциях, одноранговых сеансах чата, регистрации пользователей, приложении группы реагирования и многое другое. В версии 2013 Skype для бизнеса Server отчеты по мониторингу включают ряд улучшений:
  
- **Новые отчеты о качестве голосовой связи**. Эти новые отчеты включают отчет о сравнении качества мультимедиа в [Skype для бизнеса Server](../../manage/health-and-monitoring/comparison.md), который сравнивает качество между различными типами вызовов (например, между проводными вызовами и беспроводными вызовами); и [отчет о времени](../../manage/health-and-monitoring/join-time-report.md)Skype для бизнеса Server конференции, в котором содержится информация о времени, требуемом для того, чтобы пользователи присоединились к конференции. 
    
- **Улучшенные отчеты для анализа и диагностики видео связи и сеансов совместного использования приложений.** [Сводный](../../manage/health-and-monitoring/summary.md) отчет о качестве мультимедиа в Skype для бизнеса Server позволяет анализировать вызовы для [](../../manage/health-and-monitoring/server-performance.md) обмена видео и приложений, в то время как отчет о производительности сервера в Skype для бизнеса Server содержит сведения о производительности серверов, генерирующих эти вызовы. Метрики совместного доступа к видео [](../../manage/health-and-monitoring/peer-to-peer-session-detail-report.md) и приложениям теперь также сообщаются в отчете о подробностях одноранговых сеансов в Skype для бизнеса Server и отчете о подробностях конференции [в Skype для бизнеса Server](../../manage/health-and-monitoring/detail-report.md).
    
- **Улучшенная производительность отчетов**. В том числе более быстрое время ответа и получения данных, а также более быстрая и простая навигация по отчетам.
    
Дополнительные сведения об отдельных отчетах см. в документации по отчетам мониторинга.
  
> [!NOTE]
> Существует еще один отчет - QoE Call Detail Subreport , включенный в Skype для бизнеса Server. Однако этот отчет существует главным образом для внутреннего использования и не предназначен для прямого доступа. 
  
Существует два способа установки отчетов Skype для бизнеса Server мониторинга: вы можете использовать мастер развертывания Skype для бизнеса Server или сценарий Windows PowerShell, включенный в Skype для бизнеса Server установку файлов. Независимо от выбранного способа установки этих отчетов сначала необходимо убедиться в следующем.
  
- Имеется право добавления роди базы данных в учетную запись пользователя в базе данных мониторинга.
    
- Имеется роль диспетчера содержимого в службах SQL Server Reporting Services. Эта роль дает право на развертывание отчетов в службах SQL Server Reporting Services.
    
Чтобы установить отчеты мониторинга с помощью мастера развертывания, выполните следующие действия.
  
1. Нажмите **кнопку Начните,** нажмите кнопку Все **программы,** **нажмите Skype для бизнеса Server,** а затем нажмите кнопку **Skype для бизнеса Server мастер развертывания**.
    
2. В мастере развертывания нажмите команду **Deploy Monitoring Reports (Развернуть отчеты мониторинга)**, чтобы запустить мастер развертывания отчетов мониторинга.
    
3. В мастере развертывания отчетов мониторинга на странице **указания базы данных мониторинга** убедитесь, что в раскрывающемся списке **Monitoring database (База данных мониторинга)** отображается полное доменное имя компьютера, на котором размещается хранилище данных наблюдения. (Если имеется несколько хранилищ данных наблюдения, то потребуется выбрать соответствующий сервер в этом раскрывающемся списке.) Убедитесь, что в поле **SQL Server Reporting Services (SSRS) instance (Экземпляр служб SQL Server Reporting Services)** отображается правильный экземпляр SQL Server (например, **atl-sql-001.litwareinc.com/archinst**), а затем нажмите кнопку **Далее**.
    
4. На странице **указания учетных данных** в поле **Имя пользователя** введите имя домена и имя пользователя учетной записи, которая будет использоваться при доступе к отчетам мониторинга (например, **litwareinc\kenmyer**). Если вы не используете этот формат (domain\user name), произойдет ошибка.
    
    Введите пароль для этой учетной записи в поле **Пароль**, а затем нажмите кнопку **Далее**. Обратите внимание, что для этой учетной записи не требуется никаких специальных прав. После завершения настройки учетной записи автоматически будут предоставлены необходимые разрешения на использование логотипа и базы данных.
    
5. На странице **указания группы только для чтения** в поле "Группа пользователя" введите имя группы безопасности, которой будет предоставлен доступ только для чтения в службах SQL Server Reporting Services. Например, чтобы предоставить администраторам доступ к отчетам только для чтения, введите **RTCUniversalReadOnlyAdmins**. Нажмите кнопку **Далее**.
    
6. На странице **выполнения команд** нажмите кнопку **Готово**.
    
Отчеты мониторинга также можно установить из Skype для бизнеса Server с помощью скрипта DeployReports.ps1; этот Windows PowerShell можно найти в \<install location\> папке \Skype для бизнеса Server 2015\Deployment\Setup. Чтобы установить отчеты мониторинга с помощью скрипта DeployReports.ps1, в командной консоли введите команду, аналогичную следующей:
  
```powershell
C:\Program Files\Skype for Business Server 2015\Deployment\Setup\DeployReports.ps1 -storedUserName "litwareinc\kenmyer" -storedPassword "p@ssw0rd" -readOnlyGroupName "RTCUniversalReadOnlyAdmins" -reportServerSqlInstance "atl-sql-001.litwareinc.com" -monitoringDatabaseId "MonitoringDatabase:atl-sql-001.litwareinc.com"
```

Параметры, используемые в этой команде, описаны в следующей таблице.
  
|**Имя параметра**|**Обязательный**|**Описание**|
|:-----|:-----|:-----|
|storedUserName  <br/> |Да  <br/> |Учетная запись пользователя (в формате домен\имя_пользователя) для доступа к хранилищу данных наблюдения; например:  <br/> ```-storedUserName "litwareinc\kenmyer"```Эта учетная запись должна иметь ранее указанные SQL Server и SQL Server Reporting Services разрешения или сценарий не удастся.  <br/> |
|storedPassword  <br/> |Да  <br/> |Пароль учетной записи пользователя для доступа к хранилищу данных наблюдения.  <br/> |
|readOnlyGroupName  <br/> |Нет  <br/> |Домен или локальная группа безопасности, чьи члены получат доступ только для чтения отчетов мониторинга. Обратите внимание, что если указанная группа не существует, то скрипт завершится неудачно. Если позднее вы решите отозвать эти разрешения или предоставить разрешения на доступ другим пользователям или группам, то это можно будет сделать с помощью диспетчера отчетов служб SQL Server Reporting Services.  <br/> |
|reportSqlServerInstance  <br/> |Нет  <br/> |Экземпляр SQL Server, на котором размещаются службы отчетов. Экземпляр для служб отчетов следует указывать в виде полного доменного имени сервера отчетов, например:<br/> ```-reportServerSqlInstance atl-sql-001.litwareinc.com```Если этот параметр не включен, сценарий предполагает, что службы отчетности размещены в том же экземпляре SQL Server, в котором размещена база данных мониторинга.  <br/> |
|monitoringDatabaseId  <br/> |Нет  <br/> |Удостоверение службы для базы данных мониторинга. Чтобы получить удостоверения для баз данных мониторинга, можно выполнить следующую команду:<br/> ```Get-CsService -MonitoringDatabase```|
   
После установки отчетов мониторинга необходимо использовать New-CsReportingConfiguration для настройки URL-адреса, используемого для доступа к этим отчетам. Эта задача может быть выполнена из командной Skype для бизнеса Server с помощью следующей Windows PowerShell команды. Обратите внимание, что при настройке URL-адреса отчетов рекомендуется, но не требуется использовать протокол HTTPS:
  
```powershell
New-CsReportingConfiguration -Identity 'service:MonitoringDatabase:atl-sql-001.litwareinc.com' -ReportingURL 'https://atl-sql-001.litwareinc.com:443/Reports_ARCHINST'
```

в предыдущей команде в свойстве ReportingUrl должен быть установлен URL-адрес диспетчера отчетов, используемый службами SQL Server 2008 R2 Reporting Services. Чтобы определить URL-адрес диспетчера отчетов, на компьютере с установленными службами отчетов SQL Server выполните следующие действия.
  
1. Нажмите кнопку "Пуск" и последовательно выберите пункты "Программы", "Microsoft SQL Server 2008 R2", "Средства настройки" и "Диспетчер конфигурации служб Reporting Services".
    
2. В диалоговом окне "Соединение конфигурации служб Reporting Services" убедитесь, что в поле "Имя сервера" отображается имя компьютера со службами отчетов. Выберите экземпляр SQL Server в раскрывающемся списке "Экземпляр сервера отчетов" и нажмите кнопку "Подключить".
    
3. В диспетчере конфигурации служб Reporting Services выберите "URL-адрес диспетчера отчетов". В области "URL-адрес диспетчера отчетов" должен появиться хотя бы один URL-адрес. Любой из появившихся URL-адресов можно использовать в качестве значения ReportingURL, хотя рекомендуется, чтобы этот ReportingUrl использовал протокол HTTPS.
    
Если для базы данных мониторинга создана зеркальная база данных, необходимо также связать отчеты мониторинга с зеркальной базой данных. Подробные сведения см. в статье [Associate Monitoring Reports with a mirror database in Skype для бизнеса Server.](monitoring-reports-with-a-mirror-database.md)
