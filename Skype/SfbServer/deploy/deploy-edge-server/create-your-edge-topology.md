# <a name="create-your-edge-topology-for-skype-for-business-server"></a>Создание топологии edge для Skype для бизнеса Server
 
**Сводка:** Learn how to build, publish, and export your Edge Server topology in Skype for Business Server.
  
Построитель топологий — это средство, необходимое для построения топологии edge Server, так же, как оно используется для любого компонента топологии Skype для бизнеса Server. Прежде чем следовать следующим шагам, необходимо настроить по крайней мере один пул переднего сервера или сервер Standard Edition.
  
В этой статье мы освещаем следующие темы:
  
- Создание топологии edge server
    
- Публикация топологии с edge-сервером
    
- Экспорт топологии с edge-сервером
    
  > [!NOTE]
  > Чтобы следовать указанным ниже шагам, вам потребуется войти на серверы домена, указанные ниже, как пользователь, который является членом следующих доменных групп: 
  
- RTCUniversalServerAdmins
    
- Администраторы домена
    
## <a name="build-your-edge-server-topology"></a>Создание топологии edge server

Первым этапом развертывания является создание топологии skype для бизнеса Server Edge Server, состоящей из одного из трех вариантов:
  
- Один сервер
    
- Пул DNS с балансой нагрузки (один или несколько серверов)
    
- Аппаратный пул с балансой нагрузки (один или несколько серверов)
    
Если вы не уверены в том, что вам нужно, прежде чем приступить к следующим шагам, следует перейти к документации по планированию. В противном случае начнем.
  
### <a name="defining-the-topology-for-a-single-edge-server"></a>Определение топологии для одного сервера

1. Войдите на сервер Skype для бизнеса Server Standard Edition или в пул переднего сервера Skype для бизнеса Server.
    
2. После этого откройте **построитель топологий Skype для бизнеса Server.**
    
3. В дереве консоли разместите сайт, на который будет развернут edge Server.
    
4. Щелкните правой **кнопкой мыши пулы** edge и выберите новый **пул.**
    
5. На экране **"Определение** нового пула по краям" нажмите **кнопку "Далее".**
    
6. На  экране "Определение полного доменного имени пула" введите внутреннее полное доменное имя (FQDN), которое будет использовать  этот сервер, и выберите пул с одним компьютером, нажав кнопку "Далее".
    
7. На экране **"Выбор компонентов"** можно выбрать один из вариантов:
    
   - Вы можете использовать одно и то же FQDN и IP-адрес для службы доступа SIP, службы веб-конференций Skype для бизнеса Server и службы A/V Edge. В этом случае необходимо выбрать флажки "Использовать одно **FQDN"** и "IP-адрес" (и помните об этом в шаге 9 ниже).
    
   - Если вы планируете включить федерацию, выберите "Включить федерацию" для этого пула **(порт 5061).**
    
8. Нажав кнопку **"Далее",** вы увидите себя на экране **"Параметры IP".** Возможные варианты:
    
   - Включить IPv4 во внутреннем интерфейсе
    
   - Включить IPv6 во внутреннем интерфейсе
    
   - Включить IPv4 во внешнем интерфейсе
    
   - Включить IPv6 во внешнем интерфейсе
    
   Это довольно понятные, как IPv4- или IPv6-адреса, так и внутренние или внешние адреса (это необходимо помнить на шаге 10). Кроме того, можно настроить для внешнего IP-адреса адрес трансляции сетевых адресов (NAT). Это можно сделать, выбрав внешний **IP-адрес** этого пула, который будет переведен с помощью nat. По **готовности нажмите кнопку** "Далее".
    
9. На экране "Внешние FQDNs" варианты зависят от выбора, выполненного на шаге 7 выше.
    
   - Если вы выбрали один флажок "Использовать одно **FQDN и IP-адрес",** необходимо ввести одно внешнее FQDN в поле доступа **SIP.** Затем необходимо ввести разные номера портов для каждой службы, чтобы разрешить всем им независимое подключение. Мы рекомендуем 5061 для службы **SIP Access** Edge, 444 для службы по **краям** веб-служб и 443 для службы **A/V** Edge. По завершении нажмите кнопку **Далее**.
    
   - Если флажок "Использовать одно **FQDN** и IP-адрес" не был выбран, необходимо ввести три внешних FQDN для службы доступа **SIP,** службы веб-службы и службы **A/V** Edge.  По завершении нажмите кнопку **Далее**.
    
10. Теперь вы на экране "Определение **внутреннего IP-адреса".** Здесь вы введите IP-адрес вашего сервера в текстовые поля "Внутренний **IPv4-адрес"** и **"Внутренний IPv6-адрес"** в зависимости от вариантов, сделанных на шаге 8. По **готовности нажмите кнопку** "Далее".
    
11. На экране **"Определение внешнего IP-адреса"** имеется несколько вариантов в зависимости от предыдущих вариантов.
    
    - Возможно, вы используете одно FQDN для всех служб. В этом случае введите внешний IPv4-адрес или IPv6-адрес (какой бы вы ни использовали) в текстовое поле **доступа SIP,** а затем нажмите кнопку **"Далее".**
    
    - Возможно, вы решили использовать три отдельных FQDN и IP-адреса для служб. В этом случае введите внешние IPv4- или IPv6-адреса для службы **доступа SIP,** службы веб-служб и службы **A/V** Edge, а затем нажмите кнопку **"Далее".** 
    
    > [!NOTE]
    > Если вы ранее не решили включить и не назначить IPv6-адрес, это диалоговое окно не будет видно. Это нормально, просто перейдите к следующему шагу. 
  
12. Если вы решили использовать NAT еще на шаге 8, вы получите экран с текстовым адресом **"Общедоступный IP-адрес".** Вам потребуется ввести общедоступный IPv4-адрес и/или IPv6-адрес, который вы настроили для службы A/V Edge, для перевода с помощью NAT. Нажмите кнопку **Далее**.
    
13. Следующий экран вверх — **определение следующего прыжка.** В поле **пула** следующего прыжка выберите имя внутреннего пула, который может быть пулом переднего или автономным. Если в вашей среде есть директор, следует выбрать его. Нажмите кнопку **Далее**.
    
14. На экране **"Связывать** пулы переднего сервера" необходимо указать один или несколько внутренних пулов, включая пулы переднего сервера и серверы Standard Edition, для связи с этим сервером. Просто выберите имена внутренних пулов, которые будут использовать этот сервер для связи с поддерживаемами внешними пользователями. Нажмите **Далее**.
    
    > [!NOTE]
    > Вот что нужно помнить, если внутренние пулы или автономные серверы уже используют другой сервер Skype для бизнеса Server, они не могут иметь несколько ассоциаций. Если выбрать внутренний пул или автономный сервер в такой ситуации, появится предупреждение с предупреждением о другом сервере, и вы можете решить, продолжить или нет. Если вы продолжите с этой новой связью, подключение к другому серверу будет остановиться. 
  
15. Нажмите **кнопку "Готово"** на следующем экране.
    
16. Теперь вы сможете опубликовать эту обновленную технологию и следовать инструкциям в данной теме развертывания edge Server в Skype для бизнеса [Server,](deploy-edge-servers.md) чтобы развернуть ее на своем сервере.

11. Теперь вы на экране "Определение **внутреннего IP-адреса".** Здесь вы введите IP-адрес вашего сервера в текстовые поля "Внутренний **IPv4-адрес"** и **"Внутренний IPv6-адрес"** в зависимости от вариантов, сделанных на шаге 8. По **готовности нажмите кнопку** "Далее".
    
12. На экране **"Определение внешнего IP-адреса"** имеется несколько вариантов в зависимости от предыдущих вариантов.
    
    - Возможно, вы используете одно FQDN для всех служб. В этом случае введите внешний IPv4-адрес или IPv6-адрес (какой бы вы ни использовали) в текстовое поле **доступа SIP,** а затем нажмите кнопку **"Далее".**
    
    - Возможно, вы решили использовать три отдельных FQDN и IP-адреса для служб. В этом случае введите внешние IPv4- или IPv6-адреса для службы **доступа SIP,** службы веб-служб и службы **A/V** Edge, а затем нажмите кнопку **"Далее".** 
    
    > [!NOTE]
    > Если вы ранее не решили включить и не назначить IPv6-адрес, это диалоговое окно не будет видно. Это нормально, просто перейдите к следующему шагу. 
  
13. Нажмите кнопку **Готово**. Только что созданный сервер должен быть указан в диалоговом окне "Определение компьютеров" в **этом диалоговом** окне пула.
    
14. Еще на  экране "Определение компьютеров" на  экране пула нажмите кнопку "Добавить" и повторите шаги с 11 по 13, пока не добавят все серверы, которые будут добавлены в этот пул. После этого нажмите кнопку **"Далее".**
    
15. Следующий экран вверх — **определение следующего прыжка.** В поле **пула** следующего прыжка выберите имя внутреннего пула, который может быть пулом переднего или автономным. Если в вашей среде есть директор, следует выбрать его. Нажмите кнопку **Далее**.
    
16. На экране **"Связывать** пулы переднего сервера" необходимо указать один или несколько внутренних пулов, включая пулы переднего сервера и пулы Standard Edition, для связи с этим сервером. Просто выберите имена внутренних пулов, которые будут использовать этот сервер для связи с поддерживаемами внешними пользователями. Нажмите **Далее**.
    
    > [!NOTE]
    > Вот что нужно помнить, если внутренние пулы или автономные серверы уже используют другой сервер Skype для бизнеса Server, они не могут иметь несколько ассоциаций. Если выбрать внутренний пул или автономный сервер в такой ситуации, появится предупреждение с предупреждением о другом сервере, и вы можете решить, продолжить или нет. Если вы продолжите с этой новой связью, подключение к другому серверу будет остановиться. 
  
17. Нажмите **кнопку "Готово"** на следующем экране.
    
18. Теперь вы сможете опубликовать эту обновленную технологию и следовать инструкциям в данной теме развертывания edge Server в Skype для бизнеса [Server,](deploy-edge-servers.md) чтобы развернуть ее на своем сервере.
    
## <a name="publish-your-edge-server-topology"></a>Публикация топологии с edge-сервером

После завершения вышеперечисленной топологии необходимо опубликовать новую топологию, которая также позволит экспортировать ее в ваш сервер Skype для бизнеса Server или в пул. Выполните приведенные ниже действия.
  
1. Запустите **построитель** топологий (если он еще не запущен из предыдущей процедуры).
    
2. In **Topology Builder**, in the console tree, right-click **Skype for Business Server** and then click Skype for Business **Server Topology Builder**.
    
3. На странице **Приветствия** мастера нажмите кнопку **Далее**.
    
4. На странице **Создание других баз данных** нажмите кнопку **Далее**.
    
5. Если состояние указывает на успешное создание базы данных, сделайте следующее:
    
    - Чтобы просмотреть журнал, нажмите кнопку **Просмотреть журнал**.
    
    - Чтобы закрыть мастер, нажмите кнопку **Готово**.
    
## <a name="export-your-edge-server-topology"></a>Экспорт топологии с edge-сервером

Для успешного развертывания мастеру развертывания Skype для бизнеса Server требуется доступ к данным центрального банка управления. Для внутренних серверов в домене или лесу это обычно просто. Edge Servers are outside of the domain, and so it's necessary to manually export the topology file to the Edge Server location, usually on a physical media. Этот экспорт осуществляется с помощью PowerShell:
  
1. Запустите **оболочку управления Skype для бизнеса Server.**
    
2. В **оболочке управления Skype для бизнеса Server** запустите следующие команды:
    
   ```powershell
   Export-CsConfiguration -FileName <ConfigurationFilePath.zip>
   ```

3. Скопируйте экспортируемую копию файла на внешний носители (например, USB-накопитель или сетевую папку, до которую можно получить доступ из расположения edge Server).
