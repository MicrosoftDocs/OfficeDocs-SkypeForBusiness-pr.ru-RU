# <a name="create-your-edge-topology-for-skype-for-business-server"></a>Создание топологии пограничных Скайп для Business Server
 
**Сводка:** Узнайте, как создание, публикация и экспорт топологии пограничного сервера в Скайп для Business Server.
  
Topology Builder — это средство, которое необходимо использовать для построения топологии пограничного сервера, так же, как его используется для любого компонента топологии для Скайп для Business Server. Прежде чем выполнять следующие действия, необходимо будет настроено хотя бы один пул переднего плана или сервера Standard Edition.
  
В этой статье рассматриваются следующие вопросы.
  
- Построение топологии пограничного сервера
    
- Публикация топологии пограничного сервера
    
- Экспорт топологии пограничного сервера
    
  > [!NOTE]
  > Для выполнения следующих действий потребуется войти на серверы домена в качестве пользователя, принадлежащего к следующим группам домена. 
  
- RTCUniversalServerAdmins
    
- Администраторы домена
    
## <a name="build-your-edge-server-topology"></a>Построение топологии пограничного сервера

Первый шаг развертывания — это построение вашей Скайп для пограничного сервера Business Server топология, состоящая из один из трех вариантов:
  
- Отдельного пограничного сервера
    
- Балансировка нагрузки пограничного пула DNS (один или несколько серверов)
    
- Аппаратный балансировщик нагрузки, пограничный пул (один или несколько серверов)
    
В случае затруднений при выборе подходящего варианта рекомендуется перед выполнением следующих действий изучить документацию по планированию.
  
### <a name="defining-the-topology-for-a-single-edge-server"></a>Определение топологии отдельного пограничного сервера

1. Войдите на вашей Скайп для сервера Business Server Standard Edition или Скайп для пула переднего плана Business Server.
    
2. Один раз, откройте **Скайп для Business Server Topology Builder**.
    
3. В дереве консоли разверните сайт, который вы собираетесь развертывание пограничного сервера для.
    
4. Щелкните правой кнопкой мыши **пограничные пулы**и нажмите кнопку **новый пограничный пул**.
    
5. Щелкните **Далее** в окне **Определение нового пограничного пула** .
    
6. На экране **определить полное доменное имя пограничного пула** введите внутреннее полное доменное имя (FQDN), которые будут использовать пограничного сервера и выберите **пул из одного компьютера**, нажав кнопку **Далее** после завершения.
    
7. На экране **Выбор компонентов** выберите один из следующих вариантов.
    
   - Планируется использовать тот же полное доменное имя и IP-адрес для службы доступ к SIP, вашей Скайп для службы Business Server веб-конференций и вашей A / V Edge службы. Если Да, необходимо выбрать, **используйте один флажок полное доменное имя и IP-адрес** (и имейте в виду, шаг 9 ниже)
    
   - Если планируется разрешить федерацию, установите флажок **Включить поддержку федерации для этого пограничного пула (порт 5061)**.
    
8. После нажатия кнопки **Next** отображается экран **Параметры IP**. Можно выбрать следующие варианты:
    
   - включение протокола IPv4 во внутреннем интерфейсе;
    
   - включение протокола IPv6 во внутреннем интерфейсе;
    
   - включение протокола IPv4 во внешнем интерфейсе;
    
   - включение протокола IPv6 во внешнем интерфейсе.
    
   Далее представлены сами за себя, является ли вы используете адресов IPv4 или IPv6, а в случае применения этих адресов на пограничном сервере внутреннем или внешнем ресурсе (необходимо иметь в виду шаг 10). Также имеется возможность настройки пограничного сервера или пограничный пул для использования адрес Преобразования сетевых адресов сети для внешнего IP-адреса. You can do this by choosing **The external IP address of this Edge pool is translated by NAT** check box . Click **Next** when ready.
    
9. На экране "Внешние полные доменные имена" следует выбрать параметры в зависимости от варианта, выбранного ранее на шаге 7.
    
   - Если установлен флажок **использовать один полное доменное имя и IP-адрес** , необходимо ввести один внешний поле полное доменное имя в поле **SIP-доступ** . Затем необходимо ввести разные порты для каждой службы пограничного сервера для них все подключения независимо друг от друга. Рекомендуется назначить номер порта 5061 пограничной службе **доступа по протоколу SIP**, 444 – пограничной службе **веб-конференций** и 443 – пограничной службе **видео-/голосовой связи**. По завершении нажмите кнопку **Далее**.
    
   - Если вы не установите флажок **использовать один полное доменное имя и IP-адрес** , теперь необходимо ввести три внешние полные доменные имена для SIP-пограничной службы **Доступа** , служба пограничного сервера **Веб-конференций** и **A / V** пограничную службу. По завершении нажмите кнопку **Далее**.
    
10. Теперь вы в окне **Определение внутреннего IP-адрес** . Здесь будет введите IP-адрес пограничного сервера в текстовых полях **Внутренний адрес IPv4** и **внутреннего IPv6-адрес** , в зависимости от значений, выбранных в шаге 8. Click **Next** when ready.
    
11. Действия на экране **Определение внешнего IP-адреса** зависят от выбранного ранее варианта.
    
    - Вы можете использовать один полного доменного ИМЕНИ для всех служб. Если это так, введите в текстовом поле **Доступ к SIP** вашей внешний адрес IPv4 или IPv6 (что вы используете) и нажмите кнопку **Далее**.
    
    - Выбрано для использования трех отдельных полные доменные имена и IP-адресов для службы. Если это так, введите на внешние адреса IPv4 или IPv6 для SIP-пограничной службы **Доступа** , служба пограничного сервера **Веб-конференций** и **A / V** пограничную службу, а затем нажмите кнопку **Далее**.
    
    > [!NOTE]
    > Если включение и назначение адресов IPv6 ранее не выбрано, это диалоговое окно не отображается. В этом случае перейдите к следующему шагу. 
  
12. Если вы решили использовать преобразование сетевых адресов в шаге 8, вы получите экрана с элемента управления textbox **общедоступный IP-адрес** . Вам потребуется ввести общедоступный IPv4 и IPv6-адрес, заданные для A / V Edge службы, для преобразования, преобразование сетевых адресов. Затем нажмите кнопку **Далее**.
    
13. Next screen up is **Define the next hop**. В поле **пул следующих прыжков** выберите имя внутреннего пула, который может быть пула переднего плана или автономного пула. Если в вашей среде имеется директор, следует выбрать директора. Then click **Next**.
    
14. На экране **связать интерфейсных пулов** необходимо указать один или несколько внутренних пулов, включая пулов переднего плана и серверов Standard Edition, чтобы связать с этой пограничного сервера. Только что выберите имена внутренних пулов, с помощью этой пограничного сервера для взаимодействия с внешними пользователями поддерживаемые. Нажмите кнопку **Далее**.
    
    > [!NOTE]
    > Что-то следует помнить о здесь — это, если внутренний пулов или отдельных серверов уже используется другой Скайп для пограничного сервера Business Server, они не могут иметь несколько сопоставлений. Если вы решаете внутреннего пула или изолированного сервера, который находится в этой ситуации, вы увидите предупреждение появляется, содержащие сведения о других пограничного сервера и решить, следует ли продолжить или нет. Если продолжить процедуру с новой ассоциацией, соединение с другим пограничным сервером разрывается. 
  
15. На следующем экране нажмите кнопку **Готово**.
    
16. Теперь вы сможете опубликовать этот обновленный технологии и следуйте инструкциям в [Развертывании пограничных серверов в Скайп для Business Server](deploy-edge-servers.md) для развертывания на пограничный сервер здесь.

     > [!NOTE]
     > В отличие от других двух варианты топологии, при использовании аппаратного балансировщика нагрузки **Не должен** выбрать параметр **внешний IP-адрес пограничного пула преобразовываются NAT**. Этот режим **не поддерживается**.
  
9. На экране "Внешние полные доменные имена" следует выбрать параметры в зависимости от варианта, выбранного ранее на шаге 7.
    
   - Если установлен флажок **использовать один полное доменное имя и IP-адрес** , необходимо ввести один внешний поле полное доменное имя в поле **SIP-доступ** . Затем необходимо ввести разные порты для каждой службы пограничного сервера для них все подключения независимо друг от друга. Рекомендуется назначить номер порта 5061 пограничной службе **доступа по протоколу SIP**, 444 – пограничной службе **веб-конференций** и 443 – пограничной службе **видео-/голосовой связи**. По завершении нажмите кнопку **Далее**.
    
   - Если вы не установите флажок **использовать один полное доменное имя и IP-адрес** , теперь необходимо ввести три внешние полные доменные имена для SIP-пограничной службы **Доступа** , служба пограничного сервера **Веб-конференций** и **A / V** пограничную службу. По завершении нажмите кнопку **Далее**.
    
10. Теперь достигается экрана **Определение компьютеров в этом пуле** . Нажмите кнопку **Добавить**.
    
11. Теперь вы в окне **Определение внутреннего IP-адрес** . Здесь будет введите IP-адрес пограничного сервера в текстовых полях **Внутренний адрес IPv4** и **внутреннего IPv6-адрес** , в зависимости от значений, выбранных в шаге 8. Click **Next** when ready.
    
12. Действия на экране **Определение внешнего IP-адреса** зависят от выбранного ранее варианта.
    
    - Вы можете использовать один полного доменного ИМЕНИ для всех служб. Если это так, введите в текстовом поле **Доступ к SIP** вашей внешний адрес IPv4 или IPv6 (что вы используете) и нажмите кнопку **Далее**.
    
    - Выбрано для использования трех отдельных полные доменные имена и IP-адресов для службы. Если это так, введите на внешние адреса IPv4 или IPv6 для SIP-пограничной службы **Доступа** , служба пограничного сервера **Веб-конференций** и **A / V** пограничную службу, а затем нажмите кнопку **Далее**.
    
    > [!NOTE]
    > Если включение и назначение адресов IPv6 ранее не выбрано, это диалоговое окно не отображается. В этом случае перейдите к следующему шагу. 
  
13. Нажмите **Готово**. Пограничный сервер, которую вы только что создали теперь должно быть указано в диалоговом окне **Определение компьютеров в этом пуле** .
    
14. Во время по-прежнему в окне **Определение компьютеров в этом пуле** еще раз нажмите кнопку **Добавить** и повторите шаги 11-13, пока не будут добавлены все пограничные серверы, должны в этом пуле. По завершении нажмите кнопку **Далее**.
    
15. Next screen up is **Define the next hop**. В поле **пул следующих прыжков** выберите имя внутреннего пула, который может быть пула переднего плана или автономного пула. Если в вашей среде имеется директор, следует выбрать директора. Then click **Next**.
    
16. На экране **связать интерфейсных пулов** необходимо указать один или несколько внутренних пулов, включая интерфейсных пулов и пулов Standard Edition, чтобы связать с этой пограничного сервера. Только что выберите имена внутренних пулов, с помощью этой пограничного сервера для взаимодействия с внешними пользователями поддерживаемые. Нажмите кнопку **Далее**.
    
    > [!NOTE]
    > Что-то следует помнить о здесь — это, если внутренний пулов или отдельных серверов уже используется другой Скайп для пограничного сервера Business Server, они не могут иметь несколько сопоставлений. Если вы решаете внутреннего пула или изолированного сервера, который находится в этой ситуации, вы увидите предупреждение появляется, содержащие сведения о других пограничного сервера и решить, следует ли продолжить или нет. Если продолжить процедуру с новой ассоциацией, соединение с другим пограничным сервером разрывается. 
  
17. На следующем экране нажмите кнопку **Готово**.
    
18. Теперь вы сможете опубликовать этот обновленный технологии и следуйте инструкциям в [Развертывании пограничных серверов в Скайп для Business Server](deploy-edge-servers.md) для развертывания на пограничный сервер здесь.
    
## <a name="publish-your-edge-server-topology"></a>Публикация топологии пограничного сервера

Закончив указанные выше шаги, настало время публикации этой новой топологии также позволяют экспортировать ее в вашей Скайп для пограничного сервера Business Server или пограничный пул. Выполните следующие действия
  
1. Запустите **построитель топологий** (если он не был запущен в ходе выполнения предыдущей процедуры).
    
2. В **Построителе топологий**, в дереве консоли щелкните правой кнопкой мыши **Скайп для Business Server** и нажмите кнопку **Скайп для Business Server Topology Builder**.
    
3. На странице **приветствия** в мастере нажмите кнопку **Далее**.
    
4. На странице **создания других баз данных** нажмите кнопку **Далее**.
    
5. Дождитесь изменения состояния, указывающего на успешное создание базы данных, и выполните следующие действия.
    
    - Для просмотра журнала нажмите кнопку **Просмотреть журнал**.
    
    - Для завершения работы мастера нажмите кнопку **Готово**.
    
## <a name="export-your-edge-server-topology"></a>Экспорт топологии пограничного сервера

Для успешного развертывания, Скайп для мастер развертывания Business Server требуется доступ к данным хранилища централизованного управления. For internal servers in your domain or forest, this typically is straightforward. Пограничных серверов, за пределами домена и поэтому не требуется вручную Экспорт файла топология местоположение пограничного сервера, обычно на физический носитель. This export is done via PowerShell:
  
1. Запустите **Скайп для консоли Business Server**.
    
2. В **Скайп для консоли Business Server**выполните следующую команду:
    
   ```
   Export-CsConfiguration -FileName <ConfigurationFilePath.zip>
   ```

3. Скопируйте экспортированный файл на внешний носитель (например, USB диске или в сетевой папке, можно сделать доступными из расположения пограничного сервера).
