# <a name="create-your-edge-topology-for-skype-for-business-server"></a>Создайте топологию edge для Skype для бизнеса Server
 
**Сводка:** Узнайте, как создать, опубликовать и экспортировать топологию edge Server в Skype для бизнеса Server.
  
Топология Builder — это средство, необходимое для создания топологии edge Server, точно так же, как оно используется для любого компонента топологии для Skype для бизнеса Server. Прежде чем следовать ниже, необходимо настроить по крайней мере один пул переднего выпуск Standard сервера.
  
В этой статье мы освещаем следующие темы:
  
- Создание топологии edge Server
    
- Публикация топологии edge Server
    
- Экспорт топологии edge Server
    
  > [!NOTE]
  > Чтобы следовать ниже, необходимо войти на серверы домена, указанные ниже, как пользователь, который является членом следующих групп домена: 
  
- RTCUniversalServerAdmins
    
- Администраторы домена
    
## <a name="build-your-edge-server-topology"></a>Создание топологии edge Server

Первым этапом развертывания является построение топологии Skype для бизнеса Server Edge Server, которая состоит из одного из трех вариантов:
  
- Единый edge Server
    
- Сбалансированный пул нагрузки DNS Edge (один или несколько серверов)
    
- Сбалансированный пул edge для аппаратной нагрузки (один или несколько серверов)
    
Если вы не уверены в том, что вам нужно, прежде чем приступить к следующим шагам, следует переходить к документации по планированию. В противном случае давайте начнем.
  
### <a name="defining-the-topology-for-a-single-edge-server"></a>Определение топологии для одного edge Server

1. Войдите на Skype для бизнеса Server выпуск Standard сервер или Skype для бизнеса Server пул переднего конца.
    
2. После этого откройте **Skype для бизнеса Server topology Builder**.
    
3. В дереве консоли разместим сайт, на который будет развернут edge Server.
    
4. Правой кнопкой мыши **по краям пулы,** а затем нажмите **кнопку New Edge pool**.
    
5. Нажмите кнопку **Далее** на экране **пула Define New Edge.**
    
6. На экране **Определить FQDN** пула Edge введите внутреннее полностью квалифицированное доменное имя (FQDN), которое будет  использовать Edge Server, и выберите пул одного **компьютера,** нажав кнопку Далее, когда это будет сделано.
    
7. На экране **Select features** у вас есть выбор:
    
   - Вы можете использовать один и тот же FQDN и IP-адрес для службы SIP Access, службы веб-Skype для бизнеса Server веб-Skype для бизнеса Server и службы A/V Edge. Если это так, вам необходимо выбрать использование единого флажка **FQDN** и IP-адресов (и имейте это в виду для шага 9 ниже)
    
   - Если вы планируете включить федерацию, выберите федеративную возможность для этого контрольного окна **Edge pool (Port 5061).**
    
8. Как только вы нажмете **кнопку Далее,** вы окажетесь на **экране ПАРАМЕТРЫ IP.** Параметры:
    
   - Включить IPv4 во внутреннем интерфейсе
    
   - Включить IPv6 во внутреннем интерфейсе
    
   - Включить IPv4 во внешнем интерфейсе
    
   - Включить IPv6 во внешнем интерфейсе
    
   Это довольно понятные, независимо от того, используете ли вы адреса IPv4 или IPv6, и вы применяете эти адреса на вашем edge Server внутренне или внешне (это необходимо иметь в виду для шага 10). Кроме того, у вас есть возможность настройки edge Server или пула Edge для использования адреса сетевого перевода (NAT) для внешнего IP-адреса. Это можно сделать, выбрав внешний IP-адрес этого пула **Edge, переведенный с помощью контрольного** окна NAT. Нажмите **кнопку Далее,** когда готовы.
    
9. На экране внешних FQDNs выбор зависит от выбора, который был сделан на шаге 7 выше.
    
   - Если вы проверили флажок Использование единого **флажка FQDN** и IP-адреса, необходимо ввести единый внешний FQDN в поле **SIP Access.** Затем вам потребуется ввести разные номера портов для каждой службы края, чтобы позволить всем им подключаться самостоятельно. Рекомендуется 5061 для службы **SIP Access** Edge,  444 для службы edge веб-конференций и 443 для **службы A/V** Edge. По завершении нажмите кнопку **Далее**.
    
   - Если вы не проверили флажок Использование единого флажка **FQDN** и IP-адреса, теперь необходимо ввести три  внешних FQDN для службы **SIP Access** Edge, службы веб-конференций Edge и **службы A/V** Edge. По завершении нажмите кнопку **Далее**.
    
10. Теперь вы на экране **Определите внутренний IP-адрес.** Здесь вы введите IP-адрес вашего edge Server во внутреннем **адресе IPv4** и текстовых ящиках внутренних **адресов IPv6** в зависимости от решений, сделанных в шаге 8. Нажмите **кнопку Далее,** когда готовы.
    
11. На экране **Определите внешний IP-адрес,** у вас есть несколько вариантов в зависимости от предыдущих вариантов:
    
    - Возможно, для всех служб используется единый FQDN. Если это так, введите внешний адрес IPv4 или IPv6 (независимо от того, что вы используете) в текстовое поле **SIP Access,** а затем нажмите **кнопку Далее**.
    
    - Возможно, вы решили использовать три отдельных FQDN и IP-адреса для служб. Если это так, введите внешние адреса IPv4 или IPv6 для **службы SIP Access** Edge, службы веб-конференций Edge и службы  **A/V** Edge, а затем нажмите **кнопку Далее**.
    
    > [!NOTE]
    > Если вы ранее не решили включить и назначить адрес IPv6, диалоговое окно не будет видеться. Это нормально, просто перейдите к следующему шагу. 
  
12. Если вы решили использовать NAT еще в шаге 8, теперь вы получите экран с текстовым ящиком **общедоступных IP-адресов.** Необходимо ввести общедоступный адрес IPv4 и/или IPv6, который вы задали для службы A/V Edge, которая будет переведена службой NAT. Затем нажмите **Далее**.
    
13. Следующий экран — **определение следующего прыжка.** В поле **Следующий пул хопа** выберите имя внутреннего пула, который может быть пулом переднего конца или автономным пулом. Если у вас есть директор в вашей среде, вы должны выбрать директора. Затем нажмите **Далее**.
    
14. На экране **пулов** ассоциа-ного переднего конца необходимо указать один или несколько внутренних пулов, включая пулы переднего выпуск Standard серверов, чтобы связаться с этим edge Server. Просто выберите имена внутренних пулов, которые необходимо использовать с помощью этого edge Server для связи с поддерживаемых внешних пользователей. Нажмите **Далее**.
    
    > [!NOTE]
    > Здесь необходимо помнить, что если внутренние пулы или автономные серверы уже используют другой Skype для бизнеса Server Edge Server, у них не может быть нескольких ассоциаций. Если вы выберете внутренний пул или автономный сервер, который находится в этой ситуации, появится предупреждение о другом edge Server, и вы можете решить, хотите ли вы продолжить или нет. Если вы продолжите эту новую связь, подключение к другому edge Server прекратится. 
  
15. Нажмите **кнопку Готово** на следующем экране.
    
16. Теперь вы сможете опубликовать эту обновленную технологию и следовать инструкциям в Deploy [Edge Servers в](deploy-edge-servers.md) Skype для бизнеса Server развертывание на вашем edge Server отсюда.

11. Теперь вы на экране **Определите внутренний IP-адрес.** Здесь вы введите IP-адрес вашего edge Server во внутреннем **адресе IPv4** и текстовых ящиках внутренних **адресов IPv6** в зависимости от решений, сделанных в шаге 8. Нажмите **кнопку Далее,** когда готовы.
    
12. На экране **Определите внешний IP-адрес,** у вас есть несколько вариантов в зависимости от предыдущих вариантов:
    
    - Возможно, для всех служб используется единый FQDN. Если это так, введите внешний адрес IPv4 или IPv6 (независимо от того, что вы используете) в текстовое поле **SIP Access,** а затем нажмите **кнопку Далее**.
    
    - Возможно, вы решили использовать три отдельных FQDN и IP-адреса для служб. Если это так, введите внешние адреса IPv4 или IPv6 для **службы SIP Access** Edge, службы веб-конференций Edge и службы  **A/V** Edge, а затем нажмите **кнопку Далее**.
    
    > [!NOTE]
    > Если вы ранее не решили включить и назначить адрес IPv6, диалоговое окно не будет видеться. Это нормально, просто перейдите к следующему шагу. 
  
13. Нажмите кнопку **Готово**. Созданный только что edge Server должен быть указан в диалоговом окне **Определить компьютеры** в диалоговом окне пула.
    
14. Еще на экране **"Определите** компьютеры" на этом экране пула нажмите кнопку **Добавить** и повторите действия с 11 по 13, пока не добавят все серверы edge Servers, которые будут иметься в этом пуле. Когда это будет завершено, нажмите **кнопку Далее**.
    
15. Следующий экран — **определение следующего прыжка.** В поле **Следующий пул хопа** выберите имя внутреннего пула, который может быть пулом переднего конца или автономным пулом. Если у вас есть директор в вашей среде, вы должны выбрать директора. Затем нажмите **Далее**.
    
16. На экране **пулов** ассоциа-ного переднего конца необходимо указать один или несколько внутренних пулов, включая пулы переднего выпуск Standard пулов, чтобы связаться с этим edge Server. Просто выберите имена внутренних пулов, которые необходимо использовать с помощью этого edge Server для связи с поддерживаемых внешних пользователей. Нажмите **Далее**.
    
    > [!NOTE]
    > Здесь необходимо помнить, что если внутренние пулы или автономные серверы уже используют другой Skype для бизнеса Server Edge Server, у них не может быть нескольких ассоциаций. Если вы выберете внутренний пул или автономный сервер, который находится в этой ситуации, появится предупреждение о другом edge Server, и вы можете решить, хотите ли вы продолжить или нет. Если вы продолжите эту новую связь, подключение к другому edge Server прекратится. 
  
17. Нажмите **кнопку Готово** на следующем экране.
    
18. Теперь вы сможете опубликовать эту обновленную технологию и следовать инструкциям в Deploy [Edge Servers в](deploy-edge-servers.md) Skype для бизнеса Server развертывание на вашем edge Server отсюда.
    
## <a name="publish-your-edge-server-topology"></a>Публикация топологии edge Server

После завершения вышеперечисленных действий пришло время опубликовать эту новую топологию, которая также позволит экспортировать ее в Skype для бизнеса Server Edge Server или Edge Pool. Выполните приведенные ниже действия.
  
1. Запустите **топологию Builder** (если она еще не запущена с предыдущей процедуры).
    
2. В **Topology Builder** в дереве консоли щелкните правой кнопкой мыши Skype для бизнеса Server **нажмите** кнопку Skype для бизнеса Server **Топология Builder**.
    
3. На странице **Приветствия** мастера нажмите кнопку **Далее**.
    
4. На странице **Создание других баз данных** нажмите кнопку **Далее**.
    
5. Если состояние указывает на успешное создание базы данных, сделайте следующее:
    
    - Чтобы просмотреть журнал, нажмите кнопку **Просмотреть журнал**.
    
    - Чтобы закрыть мастер, нажмите кнопку **Готово**.
    
## <a name="export-your-edge-server-topology"></a>Экспорт топологии edge Server

Для успешного развертывания мастеру Skype для бизнеса Server развертывания необходим доступ к данным центра управления. Для внутренних серверов в домене или лесу это обычно просто. Edge Servers находятся за пределами домена, поэтому необходимо вручную экспортировать топологию файла в расположение Edge Server, как правило, на физическом носителю. Этот экспорт осуществляется с помощью PowerShell:
  
1. Запуск **оболочки Skype для бизнеса Server управления**.
    
2. В Skype для бизнеса Server **управленческой оболочки** запустите следующее:
    
   ```powershell
   Export-CsConfiguration -FileName <ConfigurationFilePath.zip>
   ```

3. Скопируйте экспортируемую папку на внешние носитли (например, USB-накопитель или сеть, которую можно получить из расположения edge Server).
