# <a name="create-your-edge-topology-for-skype-for-business-server"></a>Создание топологии Edge для Skype для бизнеса Server
 
**Сводка:** Сведения о том, как создавать, публиковать и экспортировать топологию пограничного сервера в Skype для бизнеса Server.
  
Topology Builder — это инструмент, который необходимо использовать для создания топологии пограничного сервера, так же как и для любого компонента топологии в Skype для бизнеса Server. Перед выполнением приведенных ниже действий вам потребуется настроить по крайней мере один пул переднего плана или сервер Standard Edition.
  
В этой статье рассматриваются следующие вопросы.
  
- Создание топологии пограничного сервера
    
- Публикация топологии пограничного сервера
    
- Экспорт топологии пограничного сервера
    
  > [!NOTE]
  > Для выполнения следующих действий потребуется войти на серверы домена в качестве пользователя, принадлежащего к следующим группам домена. 
  
- RTCUniversalServerAdmins
    
- Администраторы домена
    
## <a name="build-your-edge-server-topology"></a>Создание топологии пограничного сервера

Первый шаг развертывания — это создание топологии пограничного сервера в Skype для бизнеса Server, которая включает один из трех вариантов:
  
- Один пограничный сервер
    
- Пул граничного пула балансировки нагрузки DNS (один или несколько серверов).
    
- Пул Edge с балансировкой нагрузки для оборудования (один или несколько серверов)
    
В случае затруднений при выборе подходящего варианта рекомендуется перед выполнением следующих действий изучить документацию по планированию.
  
### <a name="defining-the-topology-for-a-single-edge-server"></a>Определение топологии для одного пограничного сервера

1. Войдите на сервер Skype для Business Server Standard Edition или пул переднего плана в Skype для бизнеса Server.
    
2. Запустите **Построитель топологии Skype для бизнеса Server**.
    
3. В дереве консоли разверните сайт, на который вы хотите развернуть сервер пограничного сервера.
    
4. Щелкните правой кнопкой мыши группу **пограничных групп**и выберите команду **создать пул пограничных серверов**.
    
5. На экране **Определение нового пограничного пула** нажмите кнопку **Далее** .
    
6. На экране **Определение полного** доменного имени для пула пограничного сервера введите внутреннее полное доменное имя (FQDN), которое будет использоваться пограничным сервером, и выберите **один пул компьютеров**, после чего нажмите кнопку **Далее** .
    
7. На экране **Выбор компонентов** выберите один из следующих вариантов.
    
   - Возможно, вы хотите использовать один и тот же FQDN и IP-адрес для службы доступа SIP, службы веб-конференций Skype для бизнеса Server и службы Edge. Если да, необходимо выбрать **один из полных доменных имен и IP-адресов** (и, не забывайте, что делать это для шага 9).
    
   - Если планируется разрешить федерацию, установите флажок **Включить поддержку федерации для этого пограничного пула (порт 5061)**.
    
8. После нажатия кнопки **Next** отображается экран **Параметры IP**. Можно выбрать следующие варианты:
    
   - включение протокола IPv4 во внутреннем интерфейсе;
    
   - включение протокола IPv6 во внутреннем интерфейсе;
    
   - включение протокола IPv4 во внешнем интерфейсе;
    
   - включение протокола IPv6 во внешнем интерфейсе.
    
   Это совсем понятное, независимо от того, используете ли вы адрес IPv4 или IPv6, и вы используете эти адреса на пограничном и внешнем сервере (для этого необходимо помнить шаг 10). Кроме того, у вас есть возможность настроить пограничный сервер или пул Edge так, чтобы он использовал адрес преобразования сетевых адресов (NAT) для внешнего IP-адреса. You can do this by choosing **The external IP address of this Edge pool is translated by NAT** check box . Click **Next** when ready.
    
9. На экране "Внешние полные доменные имена" следует выбрать параметры в зависимости от варианта, выбранного ранее на шаге 7.
    
   - Если установлен флажок **использовать одно полное доменное имя и IP-адрес** , необходимо ввести одно внешнее полное доменное имя в поле **SIP Access** . После этого вам потребуется указать разные номера портов для каждой службы пограничного сервера, чтобы они могли подключаться независимо друг от друга. Рекомендуется назначить номер порта 5061 пограничной службе **доступа по протоколу SIP**, 444 – пограничной службе **веб-конференций** и 443 – пограничной службе **видео-/голосовой связи**. По завершении нажмите кнопку **Далее**.
    
   - Если вы не установили флажок **использовать одно полное доменное имя и IP-адрес** , вам потребуется ввести три внешних полных доменных имен для службы пограничного **доступа SIP** , службы Edge для **веб-конференций** и службы Edge. **** По завершении нажмите кнопку **Далее**.
    
10. Теперь вы видите экран **определение внутреннего IP-адреса** . Здесь вы вводите IP-адрес пограничного сервера в текстовых полях **внутренний адрес IPv4** и **внутренний адрес IPv6** , в зависимости от параметров, выбранных на этапе 8. Click **Next** when ready.
    
11. Действия на экране **Определение внешнего IP-адреса** зависят от выбранного ранее варианта.
    
    - Возможно, вы используете одно полное доменное имя для всех служб. Если это так, введите свой внешний адрес IPv4 или IPv6 (в зависимости от используемого вами) в текстовом поле **доступ SIP** и нажмите кнопку **Далее**.
    
    - Возможно, вы выбрали три отдельных доменных имен и IP-адреса для служб. Если это так, введите свои внешние адреса IPv4 или IPv6 для службы Edge **Access** , служба Edge для **веб-конференций** и служба EDGE ( **A/V** ), а затем нажмите кнопку **Далее**.
    
    > [!NOTE]
    > Если включение и назначение адресов IPv6 ранее не выбрано, это диалоговое окно не отображается. В этом случае перейдите к следующему шагу. 
  
12. Если вы решили использовать NAT на этапе 8, вы получите экран с общедоступным полем **IP-адреса** . Вы должны ввести общедоступный адрес IPv4 и (или) IPv6, установленный для службы Edge/V, для перевода с помощью NAT. Затем нажмите кнопку **Далее**.
    
13. Next screen up is **Define the next hop**. В поле **пул следующего прыжка** выберите имя своего внутреннего пула, которое может быть пулом переднего плана или автономным пулом. Если у вас есть режиссер в своей среде, выберите режиссер. Then click **Next**.
    
14. На экране " **сопоставление пулов переднего плана** " необходимо указать один или несколько внутренних пулов, в том числе интерфейсных пулов и серверов стандартных выпусков, которые нужно связать с этим пограничным сервером. Просто выберите имена внутренних пулов, для которых вы хотите использовать этот пограничный сервер для связи с поддерживаемыми внешними пользователями. Нажмите кнопку **Далее**.
    
    > [!NOTE]
    > Что делать, если у ваших внутренних пулов или изолированных серверов уже используется другой сервер пограничного сервера Skype для бизнеса, у него не может быть нескольких ассоциаций. Если вы выбрали внутренний или изолированный сервер, который находится в этой ситуации, появится предупреждение о том, что у вас есть пограничный сервер, и вы можете решить, следует ли продолжать или нет. Если продолжить процедуру с новой ассоциацией, соединение с другим пограничным сервером разрывается. 
  
15. На следующем экране нажмите кнопку **Готово**.
    
16. Теперь вы сможете опубликовать эту обновленную технологию и следуйте инструкциям в статье [Развертывание пограничных серверов в Skype для бизнеса Server](deploy-edge-servers.md) для развертывания на пограничном сервере.

### <a name="defining-the-topology-for-a-hardware-load-balanced-edge-server-pool"></a>Определение топологии для пула пограничного сервера с балансировкой нагрузки для оборудования

     > В отличие от других двух вариантов топологии при использовании аппаратной подсистемы балансировки нагрузки вы **не должны** выбирать параметр **внешний IP-адрес пограничного пула преобразуется NAT**. Этот режим **не поддерживается**.
  
9. На экране "Внешние полные доменные имена" следует выбрать параметры в зависимости от варианта, выбранного ранее на шаге 7.
    
   - Если установлен флажок **использовать одно полное доменное имя и IP-адрес** , необходимо ввести одно внешнее полное доменное имя в поле **SIP Access** . После этого вам потребуется указать разные номера портов для каждой службы пограничного сервера, чтобы они могли подключаться независимо друг от друга. Рекомендуется назначить номер порта 5061 пограничной службе **доступа по протоколу SIP**, 444 – пограничной службе **веб-конференций** и 443 – пограничной службе **видео-/голосовой связи**. По завершении нажмите кнопку **Далее**.
    
   - Если вы не установили флажок **использовать одно полное доменное имя и IP-адрес** , вам потребуется ввести три внешних полных доменных имен для службы пограничного **доступа SIP** , службы Edge для **веб-конференций** и службы Edge. **** По завершении нажмите кнопку **Далее**.
    
10. Теперь вы достигли экрана " **Определение компьютеров в этом пуле** ". Нажмите кнопку **Добавить**.
    
11. Теперь вы видите экран **определение внутреннего IP-адреса** . Здесь вы вводите IP-адрес пограничного сервера в текстовых полях **внутренний адрес IPv4** и **внутренний адрес IPv6** , в зависимости от параметров, выбранных на этапе 8. Click **Next** when ready.
    
12. Действия на экране **Определение внешнего IP-адреса** зависят от выбранного ранее варианта.
    
    - Возможно, вы используете одно полное доменное имя для всех служб. Если это так, введите свой внешний адрес IPv4 или IPv6 (в зависимости от используемого вами) в текстовом поле **доступ SIP** и нажмите кнопку **Далее**.
    
    - Возможно, вы выбрали три отдельных доменных имен и IP-адреса для служб. Если это так, введите свои внешние адреса IPv4 или IPv6 для службы Edge **Access** , служба Edge для **веб-конференций** и служба EDGE ( **A/V** ), а затем нажмите кнопку **Далее**.
    
    > [!NOTE]
    > Если включение и назначение адресов IPv6 ранее не выбрано, это диалоговое окно не отображается. В этом случае перейдите к следующему шагу. 
  
13. Нажмите **Готово**. Только что созданный пограничный сервер должен быть указан в диалоговом окне **Определение компьютеров в этом пуле** .
    
14. Находясь в окне **Определение компьютеров в этом пуле** , снова нажмите кнопку **Добавить** , а затем повторите действия 11 – 13, пока не добавите все пограничные серверы, которые планируется использовать в этом пуле. По завершении нажмите кнопку **Далее**.
    
15. Next screen up is **Define the next hop**. В поле **пул следующего прыжка** выберите имя своего внутреннего пула, которое может быть пулом переднего плана или автономным пулом. Если у вас есть режиссер в своей среде, выберите режиссер. Then click **Next**.
    
16. На экране " **сопоставление пулов переднего плана** " необходимо указать один или несколько внутренних пулов, включая пулы интерфейсных служб и стандартные пулы выпусков, которые нужно связать с этим пограничным сервером. Просто выберите имена внутренних пулов, для которых вы хотите использовать этот пограничный сервер для связи с поддерживаемыми внешними пользователями. Нажмите кнопку **Далее**.
    
    > [!NOTE]
    > Что делать, если у ваших внутренних пулов или изолированных серверов уже используется другой сервер пограничного сервера Skype для бизнеса, у него не может быть нескольких ассоциаций. Если вы выбрали внутренний или изолированный сервер, который находится в этой ситуации, появится предупреждение о том, что у вас есть пограничный сервер, и вы можете решить, следует ли продолжать или нет. Если продолжить процедуру с новой ассоциацией, соединение с другим пограничным сервером разрывается. 
  
17. На следующем экране нажмите кнопку **Готово**.
    
18. Теперь вы сможете опубликовать эту обновленную технологию и следуйте инструкциям в статье [Развертывание пограничных серверов в Skype для бизнеса Server](deploy-edge-servers.md) для развертывания на пограничном сервере.
    
## <a name="publish-your-edge-server-topology"></a>Публикация топологии пограничного сервера

После выполнения описанных выше действий можно опубликовать эту новую топологию, которая также позволит вам экспортировать ее на граничный сервер Skype для бизнеса Server или пул Edge. Выполните следующие действия
  
1. Запустите **построитель топологий** (если он не был запущен в ходе выполнения предыдущей процедуры).
    
2. В **построителе топологии**в дереве консоли щелкните правой кнопкой мыши **Skype для бизнеса Server** и выберите **Построитель топологии Skype для бизнеса Server**.
    
3. На странице **приветствия** в мастере нажмите кнопку **Далее**.
    
4. На странице **создания других баз данных** нажмите кнопку **Далее**.
    
5. Дождитесь изменения состояния, указывающего на успешное создание базы данных, и выполните следующие действия.
    
    - Для просмотра журнала нажмите кнопку **Просмотреть журнал**.
    
    - Для завершения работы мастера нажмите кнопку **Готово**.
    
## <a name="export-your-edge-server-topology"></a>Экспорт топологии пограничного сервера

Для успешного развертывания мастеру развертывания Skype для бизнеса Server требуется доступ к данным центрального хранилища. For internal servers in your domain or forest, this typically is straightforward. Пограничные серверы находятся за пределами домена, поэтому необходимо вручную экспортировать файл топологии в расположение пограничного сервера (обычно на физическом носителе). This export is done via PowerShell:
  
1. Запустите **консоль управления Skype для бизнеса Server**.
    
2. В **командной консоли управления Skype для бизнеса Server**выполните следующие действия:
    
   ```
   Export-CsConfiguration -FileName <ConfigurationFilePath.zip>
   ```

3. Скопируйте экспортированный файл на внешний носитель (например, USB-накопитель или сетевую папку, которую можно получить из расположения на пограничном сервере).
