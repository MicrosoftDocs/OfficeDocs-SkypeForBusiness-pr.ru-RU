# <a name="deploy-edge-servers-in-skype-for-business-server"></a>Развертывание edge Servers в Skype для бизнеса Server
 
**Сводка:** Узнайте, как развернуть edge Servers в Skype для бизнеса Server среде.
  
В следующих разделах содержатся действия, которые должны следовать после Skype для бизнеса Server развертывания edge [Server](../../plan-your-deployment/edge-server-deployments/edge-server-deployments.md) в Skype для бизнеса Server документации. Этапы развертывания следующие:
  
- Сетевые интерфейсы
    
- Установка
    
- Сертификаты
    
- Запуск edge Servers
    
## <a name="network-interfaces"></a>Сетевые интерфейсы

Как отмечено в планировании, вы либо будете настраивать сетевой интерфейс с DNS в сети периметра, где размещены ваши edge Servers, либо без DNS в сети периметра.
  
### <a name="interface-configuration-with-dns-servers-in-the-perimeter-network"></a>Конфигурация интерфейса с DNS-серверами в сети периметра

1. Установите два сетевых адаптера для каждого edge Server, один для интерфейса внутреннего интерфейса и один для внешнего интерфейса.
    
    > [!NOTE]
    > Внутренние и внешние подсети должны поддерживать маршрутизацию между собой. 
  
2. На внешнем интерфейсе вы настроите **один** из следующих:
    
   а. Три статических IP-адреса в подсети внешней сети периметра и указывая шлюз по умолчанию на внутренний интерфейс внешнего брандмауэра. Настройте параметры DNS адаптеров, чтобы указать на пару DNS-серверов периметра.
    
   Б. Один статичный IP-адрес в подсети внешней сети периметра и указывав шлюз по умолчанию на внутренний интерфейс внешнего брандмауэра. Настройте параметры DNS адаптеров, чтобы указать на пару DNS-серверов периметра. Эта конфигурация является приемлемой только в том случае, если вы предварительно настроили топологию на нестандартные значения в назначениях порта, которые охватываются в статье [Create your Edge topology для Skype для бизнеса Server](create-your-edge-topology.md) статьи.
    
3. Во внутреннем интерфейсе настройте один статичный IP-адрес во внутренней сетевой сети периметра и не устанавливайте шлюз по умолчанию. Настройте параметры DNS адаптора, чтобы указать хотя бы на один DNS-сервер, но желательно на пару DNS-серверов периметра.
    
4. Создание постоянных статических маршрутов во внутреннем интерфейсе для всех внутренних сетей, в которых Skype для бизнеса Server и Exchange единой системы обмена сообщениями.
    
### <a name="interface-configuration-without-dns-servers-in-the-perimeter-network"></a>Конфигурация интерфейса без DNS-серверов в сети периметра

1. Установите два сетевых адаптера для каждого edge Server, один для интерфейса внутреннего интерфейса и один для внешнего интерфейса.
    
    > [!NOTE]
    > Внутренние и внешние подсети должны поддерживать маршрутизацию между собой. 
  
2. На внешнем интерфейсе вы настроите **один** из следующих:
    
   а. Три статических IP-адреса в подсети внешней сети периметра. Кроме того, необходимо настроить шлюз по умолчанию на внешнем интерфейсе, например, определяя маршрутизатор с выходом в Интернет или внешний брандмауэр в качестве шлюза по умолчанию. Настройте параметры DNS адаптеров, чтобы указать на внешний DNS-сервер, в идеале это пара внешних DNS-серверов.
    
   Б. Один статичный IP-адрес в подсети внешней сети периметра. Кроме того, необходимо настроить шлюз по умолчанию на внешнем интерфейсе, например, определяя маршрутизатор с выходом в Интернет или внешний брандмауэр в качестве шлюза по умолчанию. Настройте параметры DNS адаптеров, чтобы указать на внешний DNS-сервер или в идеале пару внешних DNS-серверов. Эта конфигурация является приемлемой только в том случае, если вы предварительно настроили топологию на нестандартные значения в назначениях порта, которые охватываются в статье [Create your Edge topology для Skype для бизнеса Server](create-your-edge-topology.md) статьи.
    
3. Во внутреннем интерфейсе настройте один статичный IP-адрес во внутренней сетевой сети периметра и не устанавливайте шлюз по умолчанию. Также оставьте параметры DNS адаптеров пустыми.
    
4. Создание постоянных статических маршрутов во внутреннем интерфейсе для всех внутренних сетей, в которых Skype для бизнеса Server и Exchange единой системы обмена сообщениями.
    
5. Изменить файл HOST на каждом краевом сервере, чтобы содержать запись для следующего хоп-сервера или виртуального IP(VIP). Эта запись будет директором, выпуск Standard сервером или пулом переднего конца, настроенным в качестве следующего хоп-адреса edge Server в Topology Builder. Если используется балансировка нагрузки DNS, включай строку для каждого члена следующего пула хмеля.
    
## <a name="installation"></a>Установка

Для успешного выполнения этих действий необходимо выполнить действия в топологии [Create your Edge для Skype для бизнеса Server](create-your-edge-topology.md) статьи.
  
1. Войдите на сервер, который вы настраивали для роли Edge Server, с учетной записью, которая входит в группу местного администратора.
    
2. Вам понадобится файл конфигурации топологии, который вы скопировали в конце документации по топологии edge Server на этом компьютере. Доступ к внешнему мультимедиа, в который вы поместили этот файл конфигурации (например, USB-накопитель или файл обмена).
    
3. Запустите **мастер развертывания**.
    
4. Как только мастер откроется, нажмите **кнопку Установить или обновить Skype для бизнеса Server system**.
    
5. Мастер запустит проверки, чтобы узнать, установлено ли что-либо. Так как мастер работает в первый раз, вы захотите начать с **шага 1. Установка локального магазина конфигурации.**
    
6. Появится **диалоговое окно** Настройка локальной реплики центрального магазина управления. Необходимо нажать **кнопку Импорт из файла (рекомендуется для edge Servers).**
    
7. Отсюда просмотрите расположение топологии, экспортируемой ранее, выберите файл .zip, нажмите кнопку **Открыть,** а затем нажмите **кнопку Далее**.
    
8. Мастер развертывания будет читать файл конфигурации и записывать XML-файл конфигурации на локальный компьютер.
    
9. После завершения процесса **выполнения команд** нажмите кнопку **Готово**.
    
10. В мастере развертывания щелкните **шаг 2. Настройка или удаление Skype для бизнеса Server компонентов**. Затем мастер установит Skype для бизнеса Server Edge, указанные в XML-файле конфигурации, который хранится на локальном компьютере.
    
11. После завершения установки можно перейти к шагам в разделе **Сертификаты** ниже.
    
## <a name="certificates"></a>Сертификаты

Требования к сертификату для edge Server можно найти в документации по планированию edge certificate. Ниже приведены действия по настройке сертификатов.
  
> [!NOTE]
> При запуске мастера сертификатов необходимо войти в качестве учетной записи с правильными разрешениями для типа шаблона сертификата, который вы собираетесь использовать. По умолчанию запрос Skype для бизнеса Server сертификата будет использовать шаблон сертификата Веб-сервера. Если вы вошли в систему с учетной записью, которая входит в группу RTCUniversalServerAdmins, чтобы запросить сертификат с помощью этого шаблона, перепроверяйте, чтобы убедиться, что группе были назначены разрешения на регистрацию для использования этого шаблона. 
  
### <a name="internal-edge-interface-certificates"></a>Сертификаты интерфейса Internal Edge

 
### <a name="1-download-or-export-the-ca-certification-chain"></a>1. Скачайте или экспортируете цепочку сертификации ca

 
#### <a name="nbspnbspnbsp-a-download-using-certsrv-web-site"></a>&nbsp;&nbsp;&nbsp; a. Скачивание с помощью веб-сайта certsrv

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;i. Войдите Skype для бизнеса Server во внутреннюю сеть в качестве члена локальной группы администраторов.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;II. Откройте **Начните** и **запустите** (или **поиск** и **запуск),** а затем введите следующее:
    
  ```console
  https://<NAME OF YOUR ISSUING CA SERVER>/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Например:
    
  ```console
  https://ca01/contoso.com/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iii. На веб-странице сертификации ЦС в статье **Выберите** задачу нажмите кнопку Скачайте сертификат CA, цепочку сертификатов **или CRL.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iv. В **статье Загрузка сертификата CA, цепочки сертификатов или CRL** нажмите **кнопку Скачать цепочку сертификатов CA.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;v. В поле **Загрузка файлов** нажмите кнопку **Сохранить**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. Сохраните файл .p7b на жестком диске на сервере, а затем скопируйте его в папку на каждом из ваших edge Servers.
    
### <a name="nbspnbspnbspb-export-using-mmc"></a>&nbsp;&nbsp;&nbsp;b. Экспорт с помощью MMC

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;i. С помощью MMC можно экспортировать корневой сертификат CA из любого компьютера, присоединив к домену. Либо перейдите к **запуску** **и запуску,** либо откройте **поиск** и введите **MMC,** чтобы открыть.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;II. В консоли MMC щелкните **Файл,** а затем нажмите **кнопку Добавить или Удалить snap-In**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iii. В **диалоговом списке Добавить или Удалить snap-ins** выберите **Сертификаты** и нажмите **кнопку Добавить**. При запросе выберите **учетную запись компьютера,** а затем **далее**. В диалоговом окне **Выбор компьютера** выберите **Локальный компьютер**. Нажмите **кнопку Готово,** а затем **ОК**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iv. Расширение **сертификатов (локальный компьютер).** Расширение **доверенных корневых органов сертификации**. Выбор **сертификатов**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;v. Щелкните корневой сертификат, выпущенный вашим ЦС. Щелкните правой кнопкой мыши сертификат, выберите **все задачи** в меню и выберите **Экспорт**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. Откроется **мастер экспорта сертификатов.** Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vii. В **диалоговом окте "Формат экспортных** файлов" выберите формат, в который необходимо экспортировать. Наша рекомендация — стандарт синтаксиса криптографических сообщений **— сертификаты PKCS #7 (P7b).** Если это ваш выбор, не забудьте также  выбрать включить все сертификаты в пути сертификации, если это возможно, так как это также экспортирует цепочку сертификатов, включая корневой сертификат ЦС и любые промежуточные сертификаты. Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;viii. В **диалоговом** окте Файл для экспорта в записи имени файла введите путь и имя файла (расширение по умолчанию будет .p7b) для экспортируемого сертификата. Если вам проще, выберите кнопку **Обзор,** чтобы перейти к расположению, в которое необходимо сохранить экспортируемую справку, и назовите экспортируемую справку здесь. Нажмите **кнопку Сохранить,** а **затем,** когда вы будете готовы.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ix. Просмотрите сводку действий и нажмите **кнопку Готово,** чтобы завершить экспорт сертификата. Нажмите кнопку **ОК**, чтобы подтвердить успешное выполнение экспорта.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;x. Скопируйте файл .p7b для каждого из ваших edge Servers.
    
### <a name="2-import-the-ca-certification-chain"></a>2. Импорт цепочки сертификации ca

&nbsp;&nbsp;&nbsp;a. На каждом краевом сервере откройте MMC (выберите **запуск** и **запуск** или поиск **и** введите **MMC** для открытия).
    
&nbsp;&nbsp;&nbsp;b. В меню **File** нажмите **кнопку Добавить или Удалить привязку,** а затем выберите **Добавить**.
    
&nbsp;&nbsp;&nbsp;c. В поле **Добавить или Удалить snap-ins** щелкните **Сертификаты** и нажмите **кнопку Добавить**.
    
&nbsp;&nbsp;&nbsp;d. В диалоговом окне **Оснастка диспетчера сертификатов** выберите пункт **Учетная запись компьютера** и нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;e. В **диалоговом** окне Выберите компьютер убедитесь, что для локального компьютера **(на** компьютере эта консоль запущена) будет выбрано поле", а затем нажмите кнопку **Готово**.
    
&nbsp;&nbsp;&nbsp;f. Нажмите **кнопку Закрыть,** а затем **ОК**.
    
&nbsp;&nbsp;&nbsp;g. В дереве консоли разойдитесь по сертификатам **(локальный компьютер),** щелкните правой кнопкой мыши **Доверенные** корневые органы сертификации, перейдите во все **задачи,** а затем нажмите **кнопку Импорт**.
    
&nbsp;&nbsp;&nbsp;h. В мастере, который отображается в текстовом ящике **File to Import,** укажите имя файла сертификата (имя, которое вы дали файлу p7b в предыдущем разделе). Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;i. Оставьте кнопку радио на **месте всех сертификатов** в следующем магазине, так как должны быть выбраны доверенные корневые органы сертификации. Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;j. Просмотрите сводку и нажмите **кнопку Готово,** чтобы завершить импорт.
    
&nbsp;&nbsp;&nbsp;k. Это необходимо сделать для всех развернутых серверов Edge Server.
    
### <a name="3-create-the-certificate-request"></a>3. Создание запроса на сертификат

&nbsp;&nbsp;&nbsp;a. Войдите в один из ваших edge Servers, запустите мастер развертывания и на **шаге 3: Запрос,** установка или назначение сертификатов , нажмите **кнопку Выполнить** (или **запустить** снова, если вы уже запустите этот мастер).
    
&nbsp;&nbsp;&nbsp;b. На странице **Запрос сертификата** убедитесь, что **выбран внутренний** краевой сертификат, и нажмите кнопку **Запрос**.
    
&nbsp;&nbsp;&nbsp;c. На странице **"Задержки** или немедленные запросы" выберите Отправить запрос немедленно в веб-орган сертификации, если у вас есть доступ к нему из среды Edge, или подготовьте запрос  **сейчас,** но отправьте его позже в противном случае.
    
&nbsp;&nbsp;&nbsp;d. На странице **Файл запроса** сертификата введите полную часть и имя файла, где будет сохранен файл (например, c:\SkypeInternalEdgeCert.cer). Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;e. На странице **"Укажите** альтернативный шаблон сертификата", чтобы использовать шаблон по умолчанию, кроме шаблона WebServer, проверьте альтернативный шаблон использования сертификата для выбранного контрольного окна **Certificate Authority.** В противном случае ничего не делать.
    
&nbsp;&nbsp;&nbsp;f. На странице "Name and Security Settings" ("Имя и параметры безопасности") выполните следующие действия.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   i. В **friendly name** введите имя отображения сертификата (например, Internal Edge).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  II. В **длине Bit** выберите длину бита (по умолчанию 2048, можно перейти выше и быть более безопасным, но это будет замедлять производительность).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  iii. Если вам нужен экспортируемый сертификат, необходимо проверить закрытый ключ сертификата Mark в качестве **экспортируемого** контрольного окна.
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  iv. Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;g. На странице **Информация организации** введите имя организации и организационного подразделения (OU). Вы можете ввести свой отдел или отдел (например, ИТ).
    
&nbsp;&nbsp;&nbsp;h. На странице **Географической информации** введите сведения о расположении.
    
&nbsp;&nbsp;&nbsp;i. На странице **Имя субъекта или альтернативные** имена субъекта этот объект должен быть автоматически заполнен мастером.
    
&nbsp;&nbsp;&nbsp;j. На странице **Настройка дополнительных альтернативных** имен субъекта необходимо добавить дополнительные альтернативные имена субъекта, которые вам нужны.
    
&nbsp;&nbsp;&nbsp;k. На странице **Сводка запросов** посмотрите сведения о сертификате, которые будут использоваться для создания запроса. Если вам нужно внести изменения, возвращайся и делайте это сейчас.
    
&nbsp;&nbsp;&nbsp;l. Затем нажмите **кнопку Далее,** чтобы создать файл CSR, который необходимо предоставить центру сертификации (вы также можете нажать кнопку **Просмотр** журнала, чтобы просмотреть журнал для запроса сертификата).
    
&nbsp;&nbsp;&nbsp;m. После получения запроса можно щелкнуть **Представление,** чтобы  просмотреть сертификат, и закрыть окно. Содержимое CSR-файла должно быть передано вашему центру сертификации, чтобы они могли создать сертификат для импорта на этот компьютер в следующем разделе.
    

### <a name="4-import-the-certificate"></a>4. Импорт сертификата

&nbsp;&nbsp;&nbsp;a. Войдите в группу локальных администраторов в Edge Server, из-за последней процедуры вы сделали запрос сертификата.
    
&nbsp;&nbsp;&nbsp;b. В мастере развертывания рядом с **шагом 3. Запрос, установка или назначение сертификатов**, нажмите **кнопку Запустить снова**.
    
&nbsp;&nbsp;&nbsp;c. На странице **Доступные задачи сертификатов** щелкните **Импорт сертификата из . P7b, .pfx или .cer file**.
    
&nbsp;&nbsp;&nbsp;d. На странице **Сертификат** импорта введите полный путь и имя файла сертификата, который вы получили в предыдущем разделе (или вы можете нажать кнопку **Обзор,** чтобы найти и выбрать файл таким образом).
    
&nbsp;&nbsp;&nbsp;e. Если вы импортируете сертификаты для других членов вашего пула Edge, а ваш  сертификат содержит закрытый ключ, выберите файл Сертификата, содержащий личный контрольный ящик ключа сертификата, и укажите пароль. Для продолжения нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;f. На странице **Сводка** нажмите **кнопку Далее,** как  только вы подтвердит эту информацию, и завершите после успешного импорта сертификата.
    
 
### <a name="5-export-the-certificate"></a>5. Экспорт сертификата

&nbsp;&nbsp;&nbsp;a. Убедитесь, что вы вошли на edge Server, в который вы импортировали сертификат ранее, как член локальной группы администраторов.
    
&nbsp;&nbsp;&nbsp;b. Нажмите **кнопку Начните,** **запустите** (или откройте **поиск)** и введите **MMC**.
    
&nbsp;&nbsp;&nbsp;c. На консоли MMC щелкните **Файл** и нажмите **кнопку Добавить/Удалить привязку.**
    
&nbsp;&nbsp;&nbsp;d. Из окна **Добавить или Удалить snap-ins** щелкните **Сертификаты** и нажмите **кнопку Добавить**.
    
&nbsp;&nbsp;&nbsp;e. В **диалоговом окне** "Сертификаты" выберите учетную запись **Компьютера.** Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;f. В **диалоговом окте Выберите** компьютер выберите **локальный компьютер: (на** компьютере, на который запущена консоль) . Нажмите кнопку **Готово**. Нажмите **кнопку ОК,** и конфигурация консоли MMC завершена.
    
&nbsp;&nbsp;&nbsp;g. Чтобы развернуть хранилища сертификатов, дважды щелкните **Сертификаты (локальный компьютер)**. Дважды щелкните **Личный,** а затем щелкните **Сертификаты**.
    
  > [!NOTE]
  > Вы можете быть здесь и не видите никаких сертификатов в личном магазине Сертификаты для локального компьютера. Вам не нужно охотиться, если ключа нет, импортируемый сертификат не имеет частного ключа, связанного с ним. Попробуйте сделать шаг запроса и импортировать больше, чем один раз, и если вы уверены, что все правильно, поговорите с администратором или поставщиком ca. 
  
&nbsp;&nbsp;&nbsp;h. В **личном магазине Сертификаты** для локального компьютера щелкните правой кнопкой мыши сертификат, который вы экспортируете. Выберите **все задачи** из итоговом меню и нажмите кнопку **Экспорт**.
    
&nbsp;&nbsp;&nbsp;i. В **мастере экспорта сертификатов** нажмите кнопку **Далее**. Установите переключатель **Да, экспортировать закрытый ключ**. Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;j. В **диалоговом окне Форматы** экспортных файлов выберите личные **Exchange - PKCS#12 (. PFX)** и выберите следующее:
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   i. Если это возможно, включаем все сертификаты в путь сертификации.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   II. Экспорт всех расширенных свойств.
    
   > [!NOTE]
   > **НИКОГДА** не **выберите Удаление закрытого ключа, если экспорт успешно.** Это будет означать, что сертификат и закрытый ключ должны быть перезахороняться на этот edge Server.
  
&nbsp;&nbsp;&nbsp;k. Если вы хотите назначить пароль для защиты закрытого ключа, вы можете ввести пароль для закрытого ключа. Повторно вводить пароль, чтобы подтвердить, а затем нажмите **кнопку Далее**.
    
&nbsp;&nbsp;&nbsp;l. Введите путь и имя файла для экспортируемого сертификата с помощью расширения **файла .pfx.** Путь должен быть доступен другим edge Servers в пуле, или вам нужно переместить файл с помощью внешних средств массовой информации (например, USB-накопителя). Нажмите **кнопку Далее,** когда вы сделали свой выбор.
    
&nbsp;&nbsp;&nbsp;m. Просмотрите сводку в **диалоговом окте "Завершение** мастера экспорта сертификатов", а затем нажмите кнопку **Готово**.
    
&nbsp;&nbsp;&nbsp;n. В диалоговом окне успешного завершения экспорта нажмите кнопку **ОК**.
    
 
### <a name="6-assign-the-certificate"></a>6. Назначение сертификата

&nbsp;&nbsp;&nbsp;a. На каждом краевом сервере в мастере развертывания рядом с **шагом 3. Запрос, установка или назначение сертификатов**, нажмите **кнопку Запустить снова**.
    
&nbsp;&nbsp;&nbsp;b. На странице **Доступные задачи сертификатов** щелкните **Назначение существующего сертификата**.
    
&nbsp;&nbsp;&nbsp;c. На странице **Назначение сертификата** выберите **Внутренний интерфейс пограничного сервера**.
    
&nbsp;&nbsp;&nbsp;d. На странице **Хранилище сертификатов** выберите импортируемый сертификат для внутреннего края (из предыдущего раздела).
    
&nbsp;&nbsp;&nbsp;e. На странице **Сводка назначения сертификатов** посмотрите параметры и нажмите **кнопку Далее,** чтобы назначить сертификат.
    
&nbsp;&nbsp;&nbsp;f. На странице завершения мастера нажмите кнопку **Готово**.
    
&nbsp;&nbsp;&nbsp;g. После завершения этой процедуры будет очень хорошо открыть оснастку сертификатов MMC на каждом краевом сервере, расширить сертификаты **(локальный компьютер),** расширить персональные, нажмите сертификаты и подтвердить, что внутренний сертификат Edge указан в области сведений.
    
### <a name="external-edge-interface-certificates"></a>Сертификаты интерфейса external Edge

 
### <a name="1-create-the-certificate-request"></a>1. Создание запроса на сертификат

&nbsp;&nbsp;&nbsp;a. Войдите в один из ваших edge Servers, запустите мастер развертывания и на **шаге 3: Запрос,** установка или назначение сертификатов, нажмите кнопку Выполнить (или **Выполнить** снова, если вы уже запустите этот мастер).
    
&nbsp;&nbsp;&nbsp;b. На странице **Available Certificate Tasks** (Доступные задачи сертификатов) щелкните команду **Create a new certificate request** (Создать новый запрос сертификата).
    
&nbsp;&nbsp;&nbsp;c. На странице **Запрос сертификата** убедитесь, что **выбран внешний edge Certificate,** и нажмите **кнопку Далее**.
    
&nbsp;&nbsp;&nbsp;d. На странице **Отложенные или немедленные запросы** установите переключатель **Подготовить запрос сейчас, чтобы отправить его позже**.
    
&nbsp;&nbsp;&nbsp;e. На странице **Файл запроса** сертификата введите полную часть и имя файла, где будет сохранен файл (например, c:\SkypeInternalEdgeCert.cer). Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;f. На странице **"Укажите** альтернативный шаблон сертификата", чтобы использовать шаблон по умолчанию, кроме шаблона WebServer, проверьте альтернативный шаблон использования сертификата для выбранного контрольного окна **Certificate Authority.**
    
&nbsp;&nbsp;&nbsp;g. На странице "Name and Security Settings" ("Имя и параметры безопасности") выполните следующие действия.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   i. В **friendly name** введите имя отображения сертификата (например, External Edge).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  II. В **длине Bit** выберите длину бита (по умолчанию 2048, можно перейти выше и быть более безопасным, но это будет замедлять производительность).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  iii. Если вам нужен экспортируемый сертификат, необходимо проверить закрытый ключ сертификата Mark в качестве **экспортируемого** контрольного окна.
    
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; iv. Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;h. На странице **Информация организации** введите имя организации и организационного подразделения (OU). Вы можете ввести свой отдел или отдел (например, ИТ).
    
&nbsp;&nbsp;&nbsp;i. На странице **Географической информации** введите сведения о расположении.
    
&nbsp;&nbsp;&nbsp;j. На странице **Имя субъекта и альтернативные** имена субъекта необходимые сведения должны быть автоматически заполнены мастером.
    
&nbsp;&nbsp;&nbsp;k. На странице **SIP Domain Setting on Subject Alternate Names (SANs)** проверьте почтовый ящик домена, чтобы добавить глоток.\<sipdomain> запись в список альтернативных имен субъекта.
    
&nbsp;&nbsp;&nbsp;l. На странице **Настройка дополнительных альтернативных** имен субъекта необходимо добавить дополнительные альтернативные имена субъекта, которые вам нужны.
    
&nbsp;&nbsp;&nbsp;m. На странице **Сводка запросов** посмотрите сведения о сертификате, которые будут использоваться для создания запроса. Если вам нужно внести изменения, возвращайся и делайте это сейчас.
    
&nbsp;&nbsp;&nbsp;n. Когда вы будете готовы, нажмите кнопку Далее, чтобы создать файл CSR, который необходимо предоставить центру сертификации (вы также можете щелкнуть журнал **Просмотра,** чтобы просмотреть журнал для запроса сертификата). 
    
&nbsp;&nbsp;&nbsp;o. После получения запроса можно щелкнуть **Представление,** чтобы  просмотреть сертификат, и закрыть окно. Содержимое CSR-файла должно быть передано вашему центру сертификации, чтобы они могли создать сертификат для импорта на этот компьютер в следующем разделе.
    
&nbsp;&nbsp;&nbsp;p. (НЕОБЯЗАТЕЛЬНЫЙ) При отправке содержимого CSR может потребоваться определенная информация, как следует (CAs сильно различаются, поэтому это может не потребоваться):
    
  - **Microsoft** как серверная платформа
    
  - **IIS** в качестве версии
    
  - **Веб-сервер** как тип использования
    
  - **PKCS7 как** формат ответа
    
 
### <a name="2-import-the-certificate"></a>2. Импорт сертификата

&nbsp;&nbsp;&nbsp;a. Войдите в группу локальных администраторов в Edge Server, из-за последней процедуры вы сделали запрос сертификата.
    
&nbsp;&nbsp;&nbsp;b. В мастере развертывания рядом с **шагом 3. Запрос, установка или назначение сертификатов**, нажмите **кнопку Запустить снова**.
    
&nbsp;&nbsp;&nbsp;c. На странице **Доступные задачи сертификатов** щелкните **Импорт сертификата из . P7b, .pfx или .cer file**.
    
&nbsp;&nbsp;&nbsp;d. На странице **Сертификат** импорта введите полный путь и имя файла сертификата, который вы получили в предыдущем разделе (или вы можете нажать кнопку **Обзор,** чтобы найти и выбрать файл таким образом). Если сертификат содержит закрытый ключ, выберите файл **Сертификата,** содержащий закрытый ключ сертификата, и введите пароль для закрытого ключа. Нажмите **кнопку Далее,** когда готовы.

&nbsp;&nbsp;&nbsp;f. На странице **Выполнение команд** можно просмотреть результат импорта по его завершению, нажав **журнал Представления.** Нажмите **кнопку Готово,** чтобы завершить импорт сертификата.
    
&nbsp;&nbsp;&nbsp;g. Если в пуле есть другие edge Servers, вам также потребуется выполнять следующие две процедуры. Если это автономный edge Server, вы можете получить внешние сертификаты.
    
 
### <a name="3-export-the-certificate"></a>3. Экспорт сертификата

&nbsp;&nbsp;&nbsp;e. В **диалоговом окне** "Сертификаты" выберите учетную запись **Компьютера.** Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;f. В **диалоговом окте Выберите** компьютер выберите **локальный компьютер: (на** компьютере, на который запущена консоль) . Нажмите кнопку **Готово**. Нажмите **кнопку ОК,** и конфигурация консоли MMC завершена.
    
&nbsp;&nbsp;&nbsp;g. Чтобы развернуть хранилища сертификатов, дважды щелкните **Сертификаты (локальный компьютер)**. **Дважды щелкните Личный,** а затем щелкните **Сертификаты**.
    
   > [!NOTE]
   > Вы можете быть здесь и не видите никаких сертификатов в личном магазине Сертификаты для локального компьютера. Вам не нужно охотиться, если ключа нет, импортируемый сертификат не имеет частного ключа, связанного с ним. Попробуйте сделать шаг запроса и импортировать больше, чем один раз, и если вы уверены, что все правильно, поговорите с администратором или поставщиком ca. 
  
&nbsp;&nbsp;&nbsp;h. В **личном магазине Сертификаты** для локального компьютера щелкните правой кнопкой мыши сертификат, который вы экспортируете. **Выберите все задачи** из итоговом меню и нажмите кнопку **Экспорт**.

   > [!NOTE]
   > Если **да, экспорт частного** ключа не доступен, то закрытый ключ для этого сертификата не был помечен для экспорта до его его. Прежде чем успешно это сделать, необходимо снова запросить сертификат у поставщика с закрытым набором ключей для экспорта.
  
&nbsp;&nbsp;&nbsp;j. В диалоговом окне Форматы экспортных файлов выберите личные Exchange - PKCS#12 (. PFX) и выберите следующее:
    
 &nbsp;&nbsp;&nbsp;  i. Если это возможно, включаем все сертификаты в путь сертификации.
    
 &nbsp;&nbsp;&nbsp;  II. Экспорт всех расширенных свойств.

Введите путь и имя файла для экспортируемого сертификата с помощью расширения **файла .pfx.** Путь должен быть доступен другим edge Servers в пуле, или вам нужно переместить файл с помощью внешних средств массовой информации (например, USB-накопителя). Нажмите **кнопку Далее,** когда вы сделали свой выбор.

&nbsp;&nbsp;&nbsp;n. В диалоговом окне успешного завершения экспорта нажмите кнопку **ОК**.
    
&nbsp;&nbsp;&nbsp;o. Теперь необходимо вернуться к разделу Импорт сертификата до этого и импортировать сертификат на все оставшиеся edge Servers, а затем приступить к назначению ниже.

   > [!NOTE]

## <a name="starting-the-edge-servers"></a>Запуск edge Servers

После завершения установки необходимо запустить службы на каждом сервере Edge в развертывании:
  
1. На каждом краевом сервере в **мастере развертывания** рядом с шагом **4: Начните** службы, нажмите **кнопку Выполнить**.
