# <a name="deploy-edge-servers-in-skype-for-business-server"></a>Развертывание серверов в Skype для бизнеса Server
 
**Сводка:** Узнайте, как развернуть edge Servers в среде Skype для бизнеса Server.
  
В следующих разделах содержатся действия, которые необходимо следовать после проверки документации skype для бизнеса Server [Plan for Edge Server deployments in Skype for Business Server.](../../plan-your-deployment/edge-server-deployments/edge-server-deployments.md) Действия по развертыванию следующие:
  
- Сетевые интерфейсы
    
- Установка
    
- Сертификаты
    
- Запуск серверов
    
## <a name="network-interfaces"></a>Сетевые интерфейсы

Как было отмечено в планировании, вы будете настраивать сетевой интерфейс с помощью DNS в сети периметра, где размещены ваши edge-серверы, или без DNS в сети периметра.
  
### <a name="interface-configuration-with-dns-servers-in-the-perimeter-network"></a>Конфигурация интерфейса с DNS-серверами в сети периметра

1. Установите два сетевых адаптера для каждого сервера, один для внутреннего интерфейса и один для интерфейса внешнего интерфейса.
    
    > [!NOTE]
    > Внутренние и внешние подсети должны поддерживать маршрутизацию между собой. 
  
2. На внешнем интерфейсе вы настроите один **из** следующих ок.
    
   а. Три статических IP-адреса во внешней подсети сети периметра и указатель шлюза по умолчанию на внутренний интерфейс внешнего брандмауэра. Настройте параметры DNS адаптеров так, чтобы они указывают на пару DNS-серверов периметра.
    
   б. Один статический IP-адрес во внешней подсети сети периметра и указатель шлюза по умолчанию на внутренний интерфейс внешнего брандмауэра. Настройте параметры DNS адаптеров так, чтобы они указывают на пару DNS-серверов периметра. Эта конфигурация приемлема, только если вы ранее настроили топологию на нестандартные значения в назначениях портов, что освещается в статье "Создание топологии edge для Skype для бизнеса [Server".](create-your-edge-topology.md)
    
3. На внутреннем интерфейсе настройте один статический IP-адрес во внутренней подсети сети периметра и не устанавливайте шлюз по умолчанию. Настройте параметры DNS для адаптеров так, чтобы они указывают хотя бы на один DNS-сервер, но желательно на пару DNS-серверов периметра.
    
4. Создание постоянных статических маршрутов во внутреннем интерфейсе для всех внутренних сетей, в которых находятся клиенты, Skype для бизнеса Server и серверы единой системы обмена сообщениями Exchange.
    
### <a name="interface-configuration-without-dns-servers-in-the-perimeter-network"></a>Конфигурация интерфейса без DNS-серверов в сети периметра

1. Установите два сетевых адаптера для каждого сервера, один для внутреннего интерфейса и один для интерфейса внешнего интерфейса.
    
    > [!NOTE]
    > Внутренние и внешние подсети должны поддерживать маршрутизацию между собой. 
  
2. На внешнем интерфейсе вы настроите один **из** следующих ок.
    
   а. Три статических IP-адреса во внешней подсети сети периметра. Вам также потребуется настроить шлюз по умолчанию во внешнем интерфейсе, например определить маршрутизатор с выходом в Интернет или внешний брандмауэр в качестве шлюза по умолчанию. Настройте параметры DNS адаптеров так, чтобы они указывают на внешний DNS-сервер, в идеале это пара внешних DNS-серверов.
    
   б. Один статический IP-адрес во внешней подсети сети периметра. Вам также потребуется настроить шлюз по умолчанию во внешнем интерфейсе, например определить маршрутизатор с выходом в Интернет или внешний брандмауэр в качестве шлюза по умолчанию. Настройте параметры DNS адаптеров так, чтобы они указывают на внешний DNS-сервер или, в идеале, на пару внешних DNS-серверов. Эта конфигурация приемлема, только если вы ранее настроили топологию на нестандартные значения в назначениях портов, что освещается в статье "Создание топологии edge для Skype для бизнеса [Server".](create-your-edge-topology.md)
    
3. На внутреннем интерфейсе настройте один статический IP-адрес во внутренней подсети сети периметра и не устанавливайте шлюз по умолчанию. Также оставьте параметры DNS адаптеров пустыми.
    
4. Создание постоянных статических маршрутов во внутреннем интерфейсе для всех внутренних сетей, в которых находятся клиенты, Skype для бизнеса Server и серверы единой системы обмена сообщениями Exchange.
    
5. Отредактировать файл HOST на каждом сервере, чтобы он содержал запись для сервера следующего прыжка или виртуального IP-адреса (VIP). Эта запись будет директором, сервером Standard Edition или пулом переднего сервера, настроенным в качестве адреса следующего перехода на edge Server в построителье топологий. Если вы используете балансировку нагрузки на DNS, включаем строку для каждого члена пула следующего прыжка.
    
## <a name="installation"></a>Установка

To complete these steps successfully, you will need to have followed the steps in the [Create your Edge topology for Skype for Business Server](create-your-edge-topology.md) article.
  
1. Войдите на сервер, настроенный для роли сервера, с учетной записью, входя в группу локального администратора.
    
2. Вам потребуется файл конфигурации топологии, скопированный в конце документации по топологии сервера на этом компьютере. Доступ к внешнему носитему, на который вы поместили этот файл конфигурации (например, USB-накопителю или к совместному доступу).
    
3. Запустите **мастер развертывания.**
    
4. После открытия мастера нажмите кнопку **"Установить" или "Обновить систему Skype для бизнеса Server".**
    
5. Мастер будет запускать проверки, чтобы узнать, установлены ли уже что-либо. Так как это первый запуск мастера, вам нужно начать с **шага 1. Установите локальное хранилище конфигурации.**
    
6. Появится **диалоговое** окно "Настройка локальной реплики центрального банка управления". Необходимо нажать **кнопку "Импорт" из файла (рекомендуется для edge Servers).**
    
7. Здесь перейдите к расположению ранее экспортируемой топологии, выберите ZIP-файл, нажмите кнопку **"Открыть"** и нажмите кнопку **"Далее".**
    
8. Мастер развертывания считыет файл конфигурации и записи XML-файла конфигурации на локальный компьютер.
    
9. После завершения процесса **выполнения команд** нажмите кнопку **Готово**.
    
10. В мастере развертывания щелкните **шаг 2. Установка или удаление компонентов Skype для** бизнеса Server. Мастер установит компоненты Skype для бизнеса Server Edge, указанные в XML-файле конфигурации, который хранится на локальном компьютере.
    
11. После завершения установки можно перейти к шагам в разделе **"Сертификаты"** ниже.
    
## <a name="certificates"></a>Сертификаты

Требования к сертификатам для edge Server можно найти в документации по планированию сертификатов. Ниже приведены действия по настройке сертификатов.
  
> [!NOTE]
> При запуске мастера сертификатов необходимо войти как учетная запись с правильными разрешениями для типа шаблона сертификата, который вы собираетесь использовать. По умолчанию запрос сертификата Skype для бизнеса Server будет использовать шаблон сертификата веб-сервера. Если вы вошли с помощью учетной записи, которая входит в группу RTCUniversalServerAdmins, чтобы запросить сертификат с помощью этого шаблона, дважды проверьте, назначены ли группе разрешения на регистрацию для использования этого шаблона. 
  
### <a name="internal-edge-interface-certificates"></a>Сертификаты внутреннего интерфейса edge

 
### <a name="1-download-or-export-the-ca-certification-chain"></a>1. Загрузка или экспорт цепочки сертификации ЦС

 
#### <a name="nbspnbspnbsp-a-download-using-certsrv-web-site"></a>&nbsp;&nbsp;&nbsp; а. Загрузка с помощью веб-сайта certsrv

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;i. Войдите в Skype для бизнеса Server во внутренней сети как участник локальной группы администраторов.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ii. Откройте **"Начните"** и **запустите** (или **"Поиск** **и** запуск"), а затем введите следующее:
    
  ```console
  https://<NAME OF YOUR ISSUING CA SERVER>/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Например:
    
  ```console
  https://ca01/contoso.com/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iii. На веб-странице certsrv выдаваемой службы сертификации в области "Выбор задачи" щелкните "Скачать **сертификат ЦС",**"Цепочку сертификатов" или CRL. 
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iv. В **области загрузки сертификата ЦС, цепочки сертификатов или CRL** щелкните **"Скачать цепочку сертификатов ЦС".**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;v. В поле **"Загрузка** файла" нажмите кнопку **"Сохранить".**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. Сохраните P7B-файл на жестком диске на сервере и скопируйте его в папку на каждом из ваших серверов.
    
### <a name="nbspnbspnbspb-export-using-mmc"></a>&nbsp;&nbsp;&nbsp;б) Экспорт с помощью MMC

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;i. Корневой сертификат ЦС можно экспортировать с любого компьютера, который присоединился к домену, с помощью MMC. Перейдите в **"Начните** и **запустите"** или откройте **"Поиск"** и введите **MMC,** чтобы открыть.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ii. В консоли MMC щелкните **"Файл"** и выберите "Добавить или удалить **оснастку".**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iii. В **списке диалогов "Добавление или удаление оснастки"** выберите "Сертификаты" **и** нажмите кнопку **"Добавить".** При запросе выберите **"Учетная запись компьютера"** и "Далее".  В диалоговом окне **Выбор компьютера** выберите **Локальный компьютер**. Нажмите **кнопку "Готово"** и **"ОК".**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;iv. Развернуть **сертификаты (локальный компьютер).** Развернуть **доверенные корневые органы сертификации.** Выберите **сертификаты.**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;v. Щелкните корневой сертификат, выпущенный вашим ЦС. Щелкните сертификат правой кнопкой мыши, выберите пункт **"Все** задачи" в меню и выберите пункт **"Экспорт".**
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. Откроется **мастер экспорта сертификатов.** Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vii. В **диалоговом окте** "Формат экспортируемого файла" выберите формат, в который нужно экспортировать файлы. Мы читаем стандарт синтаксиса криптографических сообщений — **сертификаты PKCS #7 (P7b).** Если вы также выбрали этот вариант, по  возможности не забудьте включить все сертификаты в путь сертификации, так как при этом также будет экспортироваться цепочка сертификатов, включая сертификат корневого ЦС и все промежуточные сертификаты. Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;viii. В **диалоговом** окте "Файл для экспорта" введите путь и имя файла (расширение по умолчанию — P7B) для экспортируемого сертификата. Если вам проще, нажать кнопку "Обзор", чтобы перейти в расположение, в которое вы хотите сохранить экспортный сертификат, и назовите его здесь.  Нажмите **кнопку**"Сохранить", а затем нажмите кнопку "Далее", когда будете готовы. 
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;ix. Просмотрите сводку действий и нажмите кнопку **"Готово",** чтобы завершить экспорт сертификата. Нажмите кнопку **ОК**, чтобы подтвердить успешное выполнение экспорта.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;x. Скопируйте P7B-файл на каждый из ваших серверов.
    
### <a name="2-import-the-ca-certification-chain"></a>2. Импорт цепочки сертификации ЦС

&nbsp;&nbsp;&nbsp;а. На каждом сервере откройте MMC (выберите "Начните" и **"Выполнить"** или "Поиск" и введите  **MMC,** чтобы открыть). 
    
&nbsp;&nbsp;&nbsp;б) В меню **"Файл"** щелкните "Добавить или удалить **оснастку"** и выберите **"Добавить".**
    
&nbsp;&nbsp;&nbsp;в. В поле "Добавление или удаление **оснастки"** щелкните "Сертификаты" **и** нажмите кнопку **"Добавить".**
    
&nbsp;&nbsp;&nbsp;г. В диалоговом окне **Оснастка диспетчера сертификатов** выберите пункт **Учетная запись компьютера** и нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;д. В **диалоговом** окне "Выбор компьютера" убедитесь, что на локальном компьютере (на компьютере, на который запущена эта **консоль),** и нажмите кнопку **"Готово".**
    
&nbsp;&nbsp;&nbsp;е. Нажмите **кнопку "Закрыть"** и **"ОК"**.
    
&nbsp;&nbsp;&nbsp;g. В дереве консоли разйдите "Сертификаты **(локальный** компьютер"), щелкните правой кнопкой мыши доверенные корневые органы сертификации, перейдите в "Все задачи" и выберите **"Импорт"**.
    
&nbsp;&nbsp;&nbsp;h. В мастере, который  отображается в текстовом ящике "Файл для импорта", укажите имя файла сертификата (имя, которое вы предоставили P7B-файлу в предыдущем разделе). Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;i. Оставьте эту кнопку на месте **всех сертификатов** в следующем хранилище, так как следует выбрать доверенные корневые органы сертификации. Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;j. Просмотрите сводку и нажмите кнопку **"Готово",** чтобы завершить импорт.
    
&nbsp;&nbsp;&nbsp;k. Это необходимо сделать для каждого развертываемго сервера.
    
### <a name="3-create-the-certificate-request"></a>3. Создание запроса на сертификат

&nbsp;&nbsp;&nbsp;а. Войдите на один из своих edge servers, запустите мастер развертывания и на **шаге 3: запрос, установка** или назначение сертификатов , нажмите кнопку **"Выполнить"** **(или**"Запустить повторно", если вы уже запускали этот мастер).
    
&nbsp;&nbsp;&nbsp;б) На странице **"Запрос сертификата"** **убедитесь,** что выбран сертификат внутренней границы, и нажмите кнопку **"Запрос".**
    
&nbsp;&nbsp;&nbsp;в. На странице **"Отложенные** или  немедленные запросы" выберите "Отправить запрос немедленно в интернет-сертификационный орган, если у вас есть доступ к нему из среды edge", или подготовьте запрос **сейчас,** но отправьте его позже в противном случае.
    
&nbsp;&nbsp;&nbsp;г. На странице **"Файл** запроса сертификата" введите полную часть и имя файла, в котором будет сохранен файл (например, c:\SkypeInternalEdgeCert.cer). Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;д. На странице **"Указание** альтернативного шаблона сертификата" для использования шаблона, который не является шаблоном WebServer по умолчанию, проверьте использование альтернативного шаблона сертификата для выбранного **сертификата.** В противном случае ничего не делать.
    
&nbsp;&nbsp;&nbsp;е. На странице "Name and Security Settings" ("Имя и параметры безопасности") выполните следующие действия.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   i. В **имени friendly** введите отображаемую имя сертификата (например, Internal Edge).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  ii. В **битовом** окне выберите длину бита (значение по умолчанию — 2048, вы можете перейти выше и быть более безопасным, но это замедлит производительность).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  iii. Если вам нужен экспортируемый сертификат, необходимо проверить закрытый ключ сертификата **как экспортируемый.**
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  iv. Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;g. На странице **"Сведения об** организации" введите имя организации и подразделения. Вы можете ввести свое подразделение или отдел (например, ИТ).
    
&nbsp;&nbsp;&nbsp;h. На странице **"Сведения о географическом** регионе" введите сведения о расположении.
    
&nbsp;&nbsp;&nbsp;i. На странице **"Имя субъекта/альтернативные** имена субъектов" этот список должен автоматически заполняться мастером.
    
&nbsp;&nbsp;&nbsp;j. На странице **"Настройка дополнительных** альтернативных имен субъектов" необходимо добавить дополнительные альтернативные имена субъектов.
    
&nbsp;&nbsp;&nbsp;k. На странице **"Сводка** запросов" посмотрите на сведения о сертификате, которые будут использоваться для создания запроса. Если вам нужно внести изменения, вернемся и сделайте это сейчас.
    
&nbsp;&nbsp;&nbsp;l. Затем нажмите **кнопку "Далее",** чтобы создать CSR-файл,  который необходимо предоставить ЦС (вы также можете щелкнуть "Просмотреть журнал", чтобы просмотреть журнал для запроса сертификата).
    
&nbsp;&nbsp;&nbsp;м. После того как запрос создан,  можно щелкнуть "Вид", чтобы просмотреть сертификат, и закрыть окно.  Содержимое CSR-файла должно быть передано вашему ЦС, чтобы они могли создать сертификат для импорта на этот компьютер в следующем разделе.
    

### <a name="4-import-the-certificate"></a>4. Импорт сертификата

&nbsp;&nbsp;&nbsp;а. Войдите в качестве члена локальной группы администраторов на edge-сервер, на который был сделан запрос сертификата в последней процедуре.
    
&nbsp;&nbsp;&nbsp;б) В мастере развертывания рядом с **этапом 3. Запрос, установка или назначение сертификатов,** нажмите **кнопку "Выполнить еще раз".**
    
&nbsp;&nbsp;&nbsp;в. На странице **"Доступные задачи сертификатов"** щелкните **"Импорт сертификата" из . P7B-, PFX- или CER-файл.**
    
&nbsp;&nbsp;&nbsp;г. На странице **"Импорт** сертификата" введите полный путь и имя файла сертификата,  который вы получили в предыдущем разделе (или нажмите кнопку "Обзор", чтобы найти и выбрать файл таким образом).
    
&nbsp;&nbsp;&nbsp;д. Если вы импортируете сертификаты для других членов пула edge и ваш сертификат содержит  закрытый ключ, обязательно выберите файл сертификата, содержащий частного ключа сертификата, и укажите пароль. Для продолжения нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;е. На странице **"Сводка"** нажмите кнопку **"Далее"** после подтверждения информации и завершите работу после успешного импорта сертификата. 
    
 
### <a name="5-export-the-certificate"></a>5. Экспорт сертификата

&nbsp;&nbsp;&nbsp;а. Убедитесь, что вы вошли на сервер, на который вы импортировали сертификат ранее, в качестве члена локальной группы администраторов.
    
&nbsp;&nbsp;&nbsp;б) Нажмите **кнопку** **"Начните", "Выполнить"**  (или откройте "Поиск") и введите **MMC.**
    
&nbsp;&nbsp;&nbsp;в. В консоли MMC щелкните **"Файл"** и выберите **"Добавить/удалить оснастку".**
    
&nbsp;&nbsp;&nbsp;г. В **окне "Добавление или удаление оснастки"** щелкните "Сертификаты" и нажмите кнопку **"Добавить".** 
    
&nbsp;&nbsp;&nbsp;д. В **диалоговом окне** оснастки "Сертификаты" выберите **учетную запись компьютера.** Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;е. В **диалоговом окке** "Выбор компьютера" выберите **"Локальный компьютер" (компьютер,** на который запущена эта консоль). Нажмите кнопку **Готово**. Нажмите **кнопку**"ОК", и настройка консоли MMC будет завершена.
    
&nbsp;&nbsp;&nbsp;g. Чтобы развернуть хранилища сертификатов, дважды щелкните **Сертификаты (локальный компьютер)**. Дважды **щелкните "Личные"** и выберите **"Сертификаты".**
    
  > [!NOTE]
  > Возможно, вы здесь и не видите никаких сертификатов в личном хранилище сертификатов для локального компьютера. Вам не нужно искать, если ключ там не существует, импортируемый сертификат не имеет закрытого ключа, связанного с ним. Попробуйте еще раз опробовать запрос и импортировать действия, и если вы уверены, что все в порядке, поговорите с администратором или поставщиком ЦС. 
  
&nbsp;&nbsp;&nbsp;h. В **личном хранилище сертификатов** для локального компьютера щелкните правой кнопкой мыши сертификат, который вы экспортируете. Выберите **все задачи** в итоговом меню и нажмите кнопку **"Экспорт".**
    
&nbsp;&nbsp;&nbsp;i. В **мастере экспорта сертификатов** нажмите кнопку **Далее**. Установите переключатель **Да, экспортировать закрытый ключ**. Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;j. В **диалоговом окне "Форматы** экспортируемого файла" выберите "Обмен личной информацией **— PKCS#12 ( PFX)** и выберите следующее:
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   i. По возможности включит все сертификаты в путь сертификации.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   ii. Экспорт всех расширенных свойств.
    
   > [!NOTE]
   > **НИКОГДА** не **выбирайте "Удалить закрытый ключ" в случае успешного экспорта.** Это означает, что сертификат и закрытый ключ необходимо повторно вернуть на этот сервер.
  
&nbsp;&nbsp;&nbsp;k. Чтобы назначить пароль для защиты закрытого ключа, можно ввести пароль для закрытого ключа. Повторно вводить пароль для подтверждения и нажмите кнопку **"Далее".**
    
&nbsp;&nbsp;&nbsp;l. Введите путь и имя файла экспортируемого сертификата, используя расширение **PFX.** Путь либо должен быть доступен другими серверами в пуле, либо вам потребуется переместить файл с помощью внешнего носители (например, USB-накопителя). После **выбора** нажмите кнопку "Далее".
    
&nbsp;&nbsp;&nbsp;м. Просмотрите сводку  в диалоговом окке завершения мастера экспорта сертификатов и нажмите кнопку **"Готово".**
    
&nbsp;&nbsp;&nbsp;n. В диалоговом окне успешного завершения экспорта нажмите кнопку **ОК**.
    
 
### <a name="6-assign-the-certificate"></a>6. Назначение сертификата

&nbsp;&nbsp;&nbsp;а. На каждом из этих серверов в мастере развертывания рядом с **шагом 3. Запрос, установка или назначение сертификатов,** нажмите **кнопку "Запустить еще раз".**
    
&nbsp;&nbsp;&nbsp;б) На странице **"Доступные задачи сертификатов"** щелкните **"Назначить существующий сертификат".**
    
&nbsp;&nbsp;&nbsp;в. На странице **Назначение сертификата** выберите **Внутренний интерфейс пограничного сервера**.
    
&nbsp;&nbsp;&nbsp;г. На странице **"Хранилище** сертификатов" выберите сертификат, импортируемый для внутреннего края (из предыдущего раздела).
    
&nbsp;&nbsp;&nbsp;д. На странице **"Сводка по** назначениям сертификатов" найдите параметры и нажмите кнопку "Далее", чтобы назначить сертификат. 
    
&nbsp;&nbsp;&nbsp;е. На странице завершения мастера нажмите кнопку **Готово**.
    
&nbsp;&nbsp;&nbsp;g. После завершения этой процедуры очень хорошо открыть оснастку MMC "Сертификаты" на каждом сервере, развернуть "Сертификаты  **(** локальный компьютер)", "Личные" **и**"Сертификаты" и подтвердить, что внутренний сертификат edge указан в области сведений.
    
### <a name="external-edge-interface-certificates"></a>Сертификаты интерфейса внешнего интерфейса

 
### <a name="1-create-the-certificate-request"></a>1. Создание запроса на сертификат

&nbsp;&nbsp;&nbsp;а. Войдите на один из своих серверов, запустите мастер развертывания и на шаге **3. Запрос, установка** или назначение сертификатов, нажмите кнопку "Выполнить" (или "Запустить повторно", если вы уже запускали этот мастер).
    
&nbsp;&nbsp;&nbsp;б) На странице **Available Certificate Tasks** (Доступные задачи сертификатов) щелкните команду **Create a new certificate request** (Создать новый запрос сертификата).
    
&nbsp;&nbsp;&nbsp;в. **Убедитесь, что** на странице "Запрос сертификата" выбран **внешний** сертификат и нажмите кнопку **"Далее".**
    
&nbsp;&nbsp;&nbsp;г. На странице **Отложенные или немедленные запросы** установите переключатель **Подготовить запрос сейчас, чтобы отправить его позже**.
    
&nbsp;&nbsp;&nbsp;д. На странице **"Файл** запроса сертификата" введите полную часть и имя файла, в котором будет сохранен файл (например, c:\SkypeInternalEdgeCert.cer). Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;е. На странице **"Указание** альтернативного шаблона сертификата" для использования шаблона, который не является шаблоном WebServer по умолчанию, проверьте использование альтернативного шаблона сертификата для выбранного **сертификата.**
    
&nbsp;&nbsp;&nbsp;g. На странице "Name and Security Settings" ("Имя и параметры безопасности") выполните следующие действия.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;   i. Введите **отображаемую** имя сертификата (например, External Edge).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  ii. В **битовом** окне выберите длину бита (значение по умолчанию — 2048, вы можете перейти выше и быть более безопасным, но это замедлит производительность).
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  iii. Если вам нужен экспортируемый сертификат, необходимо проверить закрытый ключ сертификата **как экспортируемый.**
    
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; iv. Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;h. На странице **"Сведения об** организации" введите имя организации и подразделения. Вы можете ввести свое подразделение или отдел (например, ИТ).
    
&nbsp;&nbsp;&nbsp;i. На странице **"Сведения о географическом** регионе" введите сведения о расположении.
    
&nbsp;&nbsp;&nbsp;j. На странице **"Имя субъекта/альтернативные** имена субъектов" необходимые сведения должны автоматически заполняться мастером.
    
&nbsp;&nbsp;&nbsp;k. На странице **"Параметры домена SIP** для альтернативных имен субъектов" (SANs) проверьте этот параметр, чтобы добавить SIP-имя.<sipdomain> запись в список альтернативных имен субъектов.
    
&nbsp;&nbsp;&nbsp;l. На странице **"Настройка дополнительных** альтернативных имен субъектов" необходимо добавить дополнительные альтернативные имена субъектов.
    
&nbsp;&nbsp;&nbsp;м. На странице **"Сводка** запросов" посмотрите на сведения о сертификате, которые будут использоваться для создания запроса. Если вам нужно внести изменения, вернемся и сделайте это сейчас.
    
&nbsp;&nbsp;&nbsp;n. Когда все будет готово, нажмите кнопку **"Далее",** чтобы создать CSR-файл, который необходимо предоставить ЦС (также можно щелкнуть "Просмотреть журнал", чтобы просмотреть журнал для запроса сертификата). 
    
&nbsp;&nbsp;&nbsp;o. После того как запрос создан,  можно щелкнуть "Вид", чтобы просмотреть сертификат, и закрыть окно.  Содержимое CSR-файла должно быть передано вашему ЦС, чтобы они могли создать сертификат для импорта на этот компьютер в следующем разделе.
    
&nbsp;&nbsp;&nbsp;p. (НЕОБЯЗАТЕЛЬНО) При отправке содержимого CSR можно запрошены определенные сведения следующим образом (ЦС сильно отличаются, поэтому это может не потребоваться):
    
  - **Корпорация Майкрософт** в качестве серверной платформы
    
  - **IIS** в качестве версии
    
  - **Веб-сервер** в качестве типа использования
    
  - **PKCS7** в качестве формата ответа

   > [!NOTE]
   > Вам также потребуется настроить сертификаты для обратного прокси-сервера. 
  
## <a name="starting-the-edge-servers"></a>Запуск серверов

После завершения настройки необходимо запустить службы на каждом из серверов в развертывании:
  
1. На каждом сервере в мастере развертывания **рядом** с **этапом 4. Запустите службы,** нажмите кнопку **"Выполнить".**
    
2. На странице **"Запуск служб Skype** для бизнеса Server" просмотрите список служб и нажмите кнопку "Далее", чтобы запустить службы. 
    
3. После начала работы служб можно нажать кнопку **"Готово",** чтобы закрыть мастер.
    
4. (Необязательно) Еще в **шаге 4. Запустите службы,** щелкните **"Состояние служб".**
    
5.  В **MMC служб** на каждом сервере убедитесь, что все службы Skype для бизнеса Server запущены.
