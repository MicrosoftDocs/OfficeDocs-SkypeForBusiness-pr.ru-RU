# <a name="deploy-edge-servers-in-skype-for-business-server"></a>Развертывание пограничных серверов в Скайп для Business Server
 
**Сводка:** Сведения о развертывании пограничных серверов в вашей Скайп среды Business Server.
  
В следующих разделах содержатся действия, предназначенные для следовать после рассмотрено Скайп документация Business Server [спланировать развертывание пограничного сервера в Скайп для Business Server](../../plan-your-deployment/edge-server-deployments/edge-server-deployments.md) . Развертывание состоит из следующих этапов:
  
- Сетевые интерфейсы
    
- Установка
    
- Сертификаты
    
- Запуск пограничных серверов
    
## <a name="network-interfaces"></a>Сетевые интерфейсы

Как было указано в статье Planning, будут либо настраиваться сетевой интерфейс с DNS в сети периметра, размещение пограничных серверов или без DNS в сети периметра.
  
### <a name="interface-configuration-with-dns-servers-in-the-perimeter-network"></a>Конфигурация интерфейсов с DNS-серверами в сети периметра

1. Установите два сетевых адаптера для каждого пограничного сервера, другая — для интерфейса общении и один внешнем интерфейсе.
    
    > [!NOTE]
    > Внутренние и внешние подсети должны поддерживать маршрутизацию между собой. 
  
2. На вашей внешнего интерфейса вы настроите **один** из следующих:
    
   а. Укажите три статических IP-адреса в подсети внешней сети периметра и настройте шлюз по умолчанию на внутренний интерфейс внешнего брандмауэра. С помощью параметров DNS укажите пару DNS-серверов сети периметра.
    
   б. Один статический IP-адрес адрес подсети внешнего периметра сети и выберите пункт шлюз по умолчанию во внутренний интерфейс внешнего брандмауэра. С помощью параметров DNS укажите пару DNS-серверов сети периметра. Эта конфигурация только в том случае приемлемым, если вы ранее настроили топологии, чтобы иметь нестандартных значений в поле порт назначения рассматривается в статье [Create пограничной топологии для Скайп для Business Server](create-your-edge-topology.md) .
    
3. На внутренний интерфейс настройте один статический IP-адрес подсети сети внутреннего периметра и не задано шлюза по умолчанию. С помощью параметров DNS укажите как минимум один DNS-сервер, но предпочтительнее пару DNS-серверов сети периметра.
    
4. Создайте постоянные статические маршруты на внутреннем интерфейсе для всех внутренних сетей, в которых расположены клиенты, Скайп для Business Server и серверов Exchange единой системы обмена сообщениями (единой системы обмена СООБЩЕНИЯМИ).
    
### <a name="interface-configuration-without-dns-servers-in-the-perimeter-network"></a>Конфигурация интерфейсов без DNS-серверов в сети периметра

1. Установите два сетевых адаптера для каждого пограничного сервера, другая — для интерфейса общении и один внешнем интерфейсе.
    
    > [!NOTE]
    > Внутренние и внешние подсети должны поддерживать маршрутизацию между собой. 
  
2. На вашей внешнего интерфейса вы настроите **один** из следующих:
    
   а. Три статических IP-адресов на внешнего периметра сети подсети. Вам также потребуется настроить шлюз по умолчанию на внешнем интерфейсе, например определение маршрутизатора в Интернете или внешний брандмауэр как шлюз по умолчанию. Настройка параметров DNS адаптер для указания на внешние DNS-сервер, оптимальным является размещение пары внешних DNS-серверов.
    
   б. Один статический IP-адрес подсети внешнего периметра сети. Вам также потребуется настроить шлюз по умолчанию на внешнем интерфейсе, например определение маршрутизатора в Интернете или внешний брандмауэр как шлюз по умолчанию. Настройка параметров DNS адаптер для указания на внешние DNS-сервере или в идеальном варианте пары внешних DNS-серверов. Эта конфигурация только в том случае приемлемым, если вы ранее настроили топологии, чтобы иметь нестандартных значений в поле порт назначения рассматривается в статье [Create пограничной топологии для Скайп для Business Server](create-your-edge-topology.md) .
    
3. На внутренний интерфейс настройте один статический IP-адрес подсети сети внутреннего периметра и не задано шлюза по умолчанию. Также оставьте параметры DNS адаптера пустыми.
    
4. Создайте постоянные статические маршруты на внутреннем интерфейсе для всех внутренних сетей, в которых расположены клиенты, Скайп для Business Server и серверов Exchange единой системы обмена сообщениями (единой системы обмена СООБЩЕНИЯМИ).
    
5. Редактирование файла узла на каждом пограничном сервере для хранения записей для сервера следующего прыжка или виртуальный IP-адрес (VIP). Эта запись будет директора, сервера Standard Edition или интерфейсного пула, которые вы настроили в качестве адрес следующего перехода пограничного сервера в построителе топологий. Если вы используете Балансировка нагрузки на DNS, содержать строку для каждого участника пул следующих прыжков.
    
## <a name="installation"></a>Установка

Для успешного выполнения этих действий необходимо были выполнены действия, описанные в статье [Create пограничной топологии для Скайп для Business Server](create-your-edge-topology.md) .
  
1. Войдите на сервер, который вы Настройка для роли пограничного сервера с использованием учетной записи, которая является членом группы локальных администраторов.
    
2. Вам понадобится файл конфигурации топологии, которые были скопированы в конце документации топологии пограничных серверов на данном компьютере. Доступ к внешний носитель, обмена, файл конфигурации (например, USB-диска или общий доступ).
    
3. Запустите **Мастер развертывания**.
    
4. Когда мастер откроется, щелкните **установить или обновление Скайп для бизнес-системе сервера**.
    
5. Мастер проверит Если что-либо еще установлен. Как в первый раз, запустив мастер, необходимо начать с **Шаг 1. Установка локального хранилища конфигурации.**
    
6. Откроется диалоговое окно **Настройка локальной реплики из центра управления хранения** . Вам потребуется щелкните **Импорт из файла (рекомендуемый вариант для пограничных серверов)**.
    
7. Здесь следует перейти к файлу ранее экспортированной топологии, выбрать ZIP-файл, нажать кнопку **Открыть**, а затем — кнопку **Далее**.
    
8. Мастер развертывания будет чтение файла конфигурации и запись XML-файл конфигурации на локальный компьютер.
    
9. После завершения процесса **выполнения команд** нажмите кнопку **Готово**.
    
10. В мастере развертывания выберите **Шаг 2. Установка или удаление компонентов сервера Business Скайп**. Мастер будет установить Скайп для компонентов Business пограничного сервера, указанного в XML-файл конфигурации, которые хранятся на локальном компьютере.
    
11. После завершения установки, вы можете переместить фигуру на действия, описанные в представленном ниже разделе **сертификаты** .
    
## <a name="certificates"></a>Сертификаты

Требования к сертификатам для пограничного сервера можно найти в документации по планированию сертификат пограничного сервера. Пошаговое руководство по настройке сертификатов приведены ниже.
  
> [!NOTE]
> При запуске мастера сертификатов, необходимо войти в систему с учетной записью с правильные разрешения для типа шаблон сертификата, который будет использоваться. По умолчанию Скайп для запроса на сертификат Business Server будет использовать шаблон сертификата веб-сервера. Если вы вошли в систему с учетной записью, которая является членом группы RTCUniversalServerAdmins, чтобы запросить сертификат с помощью этого шаблона, убедитесь, что в группу был назначен заявка разрешений на использование данного шаблона. 
  
### <a name="internal-edge-interface-certificates"></a>Сертификаты внутреннего интерфейса пограничного сервера

 
### <a name="1-download-or-export-the-ca-certification-chain"></a>1. Загрузите или экспорт цепочки сертификатов ЦС

 
#### <a name="nbspnbspnbsp-a-download-using-certsrv-web-site"></a>&nbsp;&nbsp;&nbsp;. Загрузка с помощью веб-сайта certsrv

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;я. Войдите в Скайп для Business Server в вашей внутренней сети как член группы локальных администраторов.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;II. Откройте **Пуск**и **запустите** (или **поиска** и **запустите** ) и затем введите следующую команду:
    
  ```
  https://<NAME OF YOUR ISSUING CA SERVER>/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Например:
    
  ```
  https://ca01/contoso.com/certsrv
  ```

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;III. На веб-странице certsrv выдающего ЦС в списке **выберите задачу**, нажмите кнопку **Загрузка сертификата ЦС, цепочки сертификатов или CRL**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;IV. В окне **Загрузка сертификата ЦС, цепочки сертификатов или CRL** щелкните **Загрузка цепочки сертификатов ЦС**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;v. В диалоговом окне **Загрузка файла** щелкните **Сохранить**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. Сохраните файл с расширением p7b на жестком диске на сервере и скопируйте его в папку на каждый из пограничных серверов.
    
### <a name="nbspnbspnbspb-export-using-mmc"></a>&nbsp;&nbsp;&nbsp;б. Экспорт с помощью MMC

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;я. Корневой сертификат ЦС можно экспортировать с любого компьютера, присоединенного к домену, с помощью MMC. Щелкните **Пуск**, выберите **Выполнить** или откройте окно **Поиск**, затем введите **MMC**, чтобы открыть консоль.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;II. В консоли управления (MMC) последовательно выберите пункты **Файл** и **Добавить или удалить оснастку**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;III. В списке диалогового окна **Добавление или удаление оснасток** выберите **сертификаты**и нажмите кнопку **Добавить**. После появления запроса выберите **Учетную запись компьютера**, а затем кнопку **Далее**. В диалоговом окне **Выбор компьютера** выберите **Локальный компьютер**. Нажмите кнопку **Готово**, а затем **кнопку ОК**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;IV. Разверните узел **Сертификаты (локальный компьютер)**. Разверните узел **Доверенные корневые центры сертификации**. Установите **сертификаты**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;v. Щелкните корневой сертификат, выпущенный вашим ЦС. Правой кнопкой мыши щелкните нужный сертификат, выберите **Все задачи**, а затем — **Экспорт**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vi. Откроется **мастер экспорта сертификатов**. Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;vii. В диалоговом окне **Формат экспортируемого файла** выберите формат файла, в который будет выполняться экспорт. Мы рекомендуем использовать формат **Стандарт Cryptographic Message Syntax - сертификаты PKCS #7 (.p7b)**. Если это также почтой, не забудьте также установите флажок **Включить по возможности все сертификаты в путь сертификации** , как это будет также экспорт цепочки сертификатов, включая корневого центра сертификации и промежуточные сертификаты. Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;viii. В диалоговом окне **Имя экспортируемого файла** в записи имени файла введите путь и имя файла (расширение имени по умолчанию: .p7b) для экспортированного сертификата. Если проще на то, нажмите кнопку **Обзор** , чтобы перейти к расположению, необходимо сохранить экспортированный сертификат, который и назовите здесь экспортированный сертификат. Когда вы будете готовы, щелкните **Сохранить**и нажмите **Далее** .
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;IX. Просмотрите сводку своих действий и нажмите кнопку **Готово**, чтобы завершить экспорт сертификата. Нажмите кнопку **ОК**, чтобы подтвердить успешное выполнение экспорта.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;x. Скопируйте файл с расширением p7b каждый из пограничных серверов.
    
### <a name="2-import-the-ca-certification-chain"></a>2. Импорт цепочки сертификатов ЦС

&nbsp;&nbsp;&nbsp;. На каждом пограничном сервере откройте консоль MMC (нажмите кнопку **Пуск** и **запустите**или **поиска**и введите **MMC** для открытия).
    
&nbsp;&nbsp;&nbsp;б. В меню **Файл** щелкните **Добавить или удалить оснастку** и затем выберите **Добавить**.
    
&nbsp;&nbsp;&nbsp;c. В окне **Добавление и удаление оснасток** щелкните **Сертификаты** и затем нажмите кнопку **Добавить**.
    
&nbsp;&nbsp;&nbsp;d. В диалоговом окне **Оснастка диспетчера сертификатов** выберите **Учетная запись компьютера** и затем нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;e. Убедитесь, что в диалоговом окне **Выбор компьютера** установлен флажок **Локальный компьютер: (компьютер, на котором запущена эта консоль)**, и затем нажмите кнопку **Готово**.
    
&nbsp;&nbsp;&nbsp;f. Нажмите кнопку **Закрыть**, а затем — кнопку **ОК**.
    
&nbsp;&nbsp;&nbsp;g. В дереве консоли разверните узел **Сертификаты (локальный компьютер)**, правой кнопкой мыши щелкните пункт **Доверенные корневые центры сертификации**, перейдите к списку **Все задачи**, а затем щелкните **Импорт**.
    
&nbsp;&nbsp;&nbsp;h. В появившемся мастере в текстовом поле **Файл для импорта** укажите имя файла сертификата (имя, назначенное P7B-файлу в предыдущем разделе). Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;я. Оставьте переключатель в положении **Place all certificates in the following store, as Trusted Root Certification Authorities** (Разместить все сертификаты в следующем хранилище как доверенные корневые центры сертификации). Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;j. Просмотрите сводку и нажмите кнопку **Готово**, чтобы завершить импорт.
    
&nbsp;&nbsp;&nbsp;k. Это необходимо выполнить для каждого пограничного сервера при развертывании.
    
### <a name="3-create-the-certificate-request"></a>3. Создание запроса на сертификат

&nbsp;&nbsp;&nbsp;. Выполните вход на один из пограничных серверов, запустите мастер развертывания и на **Шаг 3: запрос, установка или назначение сертификатов**, нажмите кнопку **запуска** (или **Выполнить еще раз**, если уже выполнения этого мастера).
    
&nbsp;&nbsp;&nbsp;б. На странице **Запроса сертификата** убедитесь, что выбран вариант **Сертификат внутренней границы** и нажмите кнопку **Запрос**.
    
&nbsp;&nbsp;&nbsp;c. На странице **отложенные или немедленные запросы** выберите **Отправить запрос в локальную службу сертификации немедленно** , если у вас есть доступ к одному из среды пограничного сервера или **подготовить запрос сейчас, чтобы отправить его позже** в противном случае.
    
&nbsp;&nbsp;&nbsp;d. На странице **Файл запроса сертификата** введите полный путь и имя файла, в который должен сохраняться запрос (например, c:\SkypeInternalEdgeCert.cer). Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;e. На странице **Указание альтернативного шаблона сертификата** установите флажок **Использовать альтернативный шаблон сертификата для выбранного ЦС**, чтобы использовать шаблон, отличный от шаблона WebServer по умолчанию. В противном случае ничего не предпринимать.
    
&nbsp;&nbsp;&nbsp;f. На странице  настроек имени и безопасности  выполните следующие действия.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;я. В поле **Понятное имя** введите отображаемое имя для сертификата (например, "Внутренняя граница").
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;II. В списке **Длина в битах** выберите длину в битах (по умолчанию 2048). Можно выбрать более высокое значение и увеличить безопасность, но при этом снижается производительность.
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;III. Если нужен экспортируемый сертификат, необходимо установить флажок **Пометить закрытый ключ сертификата как экспортируемый**.
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;IV. Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;g. На странице **Сведения об организации** введите название организации и подразделения. Можно указать отдел или подразделения (например, ИТ).
    
&nbsp;&nbsp;&nbsp;h. На странице **Сведения о местоположении** укажите сведения о своем местоположении.
    
&nbsp;&nbsp;&nbsp;я. На странице **Имени субъекта/альтернативное имя субъекта** эти сведения должны быть автоматически заполнены мастером.
    
&nbsp;&nbsp;&nbsp;j. На странице **Настройка дополнительных альтернативных имен субъекта** необходимо добавить все нужные дополнительные альтернативные имена субъекта.
    
&nbsp;&nbsp;&nbsp;k. На странице " **Сводка по запросу** " просмотрите различные сведения о сертификате, которые будут использоваться для создания запроса. При необходимости вернитесь обратно и внесите изменения.
    
&nbsp;&nbsp;&nbsp;l. Нажмите кнопку **Далее** для создания файла CSR, необходимые для обеспечения ЦС (, можно нажать кнопку **Просмотреть журнал** для просмотра журнала для запроса на сертификат).
    
&nbsp;&nbsp;&nbsp;m. После создания запроса можно нажать кнопку **Просмотр**, чтобы просмотреть сертификат, и **Готово**, чтобы закрыть окно. Содержимое CSR-файла необходимо передать вашему ЦС, чтобы они могли создать сертификат для вас, и вы могли импортировать его на этот компьютер в следующем разделе.
    

### <a name="4-import-the-certificate"></a>4. Импорт сертификата

&nbsp;&nbsp;&nbsp;. Войдите, должна быть членом локальной группы администраторов на пограничный сервер, ваш запрос на сертификат из произведенной последнего действия.
    
&nbsp;&nbsp;&nbsp;б. В мастере развертывания рядом с **Шаг 3. Запрос, установка или назначение сертификатов**, нажмите кнопку **Запустить еще раз**.
    
&nbsp;&nbsp;&nbsp;c. На странице **Доступные задачи для сертификатов** щелкните **Импортировать сертификат из P7B-, PFX- или CER-файла**.
    
&nbsp;&nbsp;&nbsp;d. На странице **Импорт сертификата** введите полный путь и имя файла сертификата, который был получен в предыдущем разделе (или щелкните **Обзор**, чтобы найти и выбрать файл).
    
&nbsp;&nbsp;&nbsp;e. При импорте сертификатов для других участников в пограничный пул и содержит закрытый ключ сертификата, убедитесь, что установлен флажок **файл сертификата, который содержит закрытый ключ сертификата** и укажите пароль. Для продолжения нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;f. На странице "**Сводка** " нажмите кнопку **Далее** , после подтверждения сведения и **Готово** после успешного импорта сертификата.
    
 
### <a name="5-export-the-certificate"></a>5. Экспорт сертификата

&nbsp;&nbsp;&nbsp;. Убедитесь, что вы вошли в пограничного сервера, импортировать сертификат, который ранее, являясь членом локальной группы администраторов.
    
&nbsp;&nbsp;&nbsp;б. Нажмите кнопку **Пуск**, **запустите** (или откройте **поиска** ) и введите **MMC**.
    
&nbsp;&nbsp;&nbsp;c. В консоли MMC щелкните **Файл** и выберите **Добавить или удалить оснастку**.
    
&nbsp;&nbsp;&nbsp;d. В окне **Добавление и удаление оснасток** щелкните **Сертификаты** и затем нажмите кнопку **Добавить**.
    
&nbsp;&nbsp;&nbsp;e. В диалоговом окне **Оснастка сертификата** выберите **Учетная запись компьютера**. Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;f. В диалоговом окне **Выбор компьютера** установите **локальный компьютер: (котором запущена эта консоль)**. Нажмите кнопку **Готово **. Нажмите кнопку **ОК**и завершить настройку консоли MMC.
    
&nbsp;&nbsp;&nbsp;g. Дважды щелкните пункт **Сертификаты (локальный компьютер)**, чтобы развернуть хранилища сертификатов. Дважды щелкните пункт **Личные**, а затем щелкните пункт **Сертификаты**.
    
  > [!NOTE]
  > Когда вы находитесь здесь и не отображается все сертификаты в личных сертификатов хранения для локального компьютера. Вам не нужно слежения, если ключ не существует, импортированный сертификат не пришлось закрытого ключа, связанного с ним. Запрос и импорт действия над еще раз, а если вы уверены, что есть все, что вправо, обратитесь к администратору центра сертификации или поставщика. 
  
&nbsp;&nbsp;&nbsp;h. В поле **хранить личные сертификаты** для локального компьютера щелкните правой кнопкой мыши сертификат, который выполняется импорт. Выберите пункт **Все задачи** и нажмите **Экспорт**.
    
&nbsp;&nbsp;&nbsp;я. В окне **Мастера экспорта сертификатов**нажмите кнопку **Далее**. Выберите **Да, экспортировать закрытый ключ**. Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;j. В диалоговом окне **форматов экспортируемых файлов** установите переключатель **Файл обмена личной информацией - PKCS #12 (.PFX)**, а затем выберите следующее:
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;я. Включить по возможности все сертификаты в путь сертификации.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;II. Экспортировать все расширенные свойства.
    
   > [!NOTE]
   > **NEVER** выберите **Удалить закрытый ключ после успешного экспорта**. Оно будет означает, что вам нужно повторно импортировать сертификат и закрытый ключ обратно в этот пограничный сервер.
  
&nbsp;&nbsp;&nbsp;k. Если требуется назначить пароль для защиты закрытого ключа, введите пароль для закрытого ключа. Введите пароль еще раз для подтверждения и нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;l. Введите путь и имя файла для экспортируемого сертификата, используя расширение файла **PFX-файл**. Либо путь должен быть доступен с других пограничных серверов в пуле или вам потребуется переместить файл с помощью внешний носитель (например, USB-диска). После внесения свой выбор, нажмите кнопку **Далее** .
    
&nbsp;&nbsp;&nbsp;m. Проверьте сводные данные в диалоговом окне **Завершение мастера экспорта сертификатов**, а затем нажмите кнопку **Готово**.
    
&nbsp;&nbsp;&nbsp;n. В диалоговом окне с сообщением об успешном экспорте нажмите кнопку **ОК**.
    
 
### <a name="6-assign-the-certificate"></a>6. назначение сертификата

&nbsp;&nbsp;&nbsp;. На EACH пограничный сервер, в мастере развертывания рядом с **Шаг 3. Запрос, установка или назначение сертификатов**, выберите команду **выполнить еще раз**.
    
&nbsp;&nbsp;&nbsp;б. На странице **Доступные задачи для сертификатов** выберите **Назначить имеющийся сертификат**.
    
&nbsp;&nbsp;&nbsp;c. На странице **Назначение сертификата** выберите в списке **Внутренний пограничный сервер**.
    
&nbsp;&nbsp;&nbsp;d. На странице **Хранилища сертификатов** выберите сертификат, который импортирован для внутреннего интерфейса пограничного сервера (выше).
    
&nbsp;&nbsp;&nbsp;e. На странице **Сводка по назначению сертификата** проверьте параметры и затем нажмите кнопку **Далее**, чтобы назначить сертификат.
    
&nbsp;&nbsp;&nbsp;f. На странице завершения работы мастера нажмите кнопку **Готово**.
    
&nbsp;&nbsp;&nbsp;g. После выполнения этой процедуры, это очень полезно определить откройте оснастку консоли Управления сертификатами на каждом пограничном сервере, разверните узел **Сертификаты (локальный компьютер)**, разверните узел **Личные**, щелкните **сертификаты**и убедитесь, что внутренний пограничный сервер сертификат отображается в области сведений.
    
### <a name="external-edge-interface-certificates"></a>Сертификаты интерфейса внешней границы

 
### <a name="1-create-the-certificate-request"></a>1. Создание запроса на сертификат

&nbsp;&nbsp;&nbsp;. Выполните вход на один из пограничных серверов, запустите мастер развертывания и на **Шаг 3: запрос, установка или назначение сертификатов выберите пункт выполнить** (или **Выполнить еще раз**, если уже выполнения этого мастера).
    
&nbsp;&nbsp;&nbsp;б. На странице **Доступные задачи сертификатов** щелкните команду **Создать новый запрос сертификата**.
    
&nbsp;&nbsp;&nbsp;c. На странице **Запроса сертификата** убедитесь, что выбран вариант **Сертификат внешней границы** и нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;d. На странице **Отложенные и мгновенные запросы** выберите **Подготовить запрос сейчас, чтобы отправить его позже**.
    
&nbsp;&nbsp;&nbsp;e. На странице **Файл запроса сертификата** введите полный путь и имя файла, в который должен сохраняться запрос (например, c:\SkypeInternalEdgeCert.cer). Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;f. На странице **Указание альтернативного шаблона сертификата** установите флажок **Использовать альтернативный шаблон сертификата для выбранного ЦС**, чтобы использовать шаблон, отличный от шаблона WebServer по умолчанию.
    
&nbsp;&nbsp;&nbsp;g. На странице  настроек имени и безопасности  выполните следующие действия.
    
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;я. В поле **Понятное имя** введите отображаемое имя для сертификата (например, "Внешняя граница").
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;II. В списке **Длина в битах** выберите длину в битах (по умолчанию 2048). Можно выбрать более высокое значение и увеличить безопасность, но при этом снижается производительность.
    
 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;III. Если нужен экспортируемый сертификат, необходимо установить флажок **Пометить закрытый ключ сертификата как экспортируемый**.
    
  &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;IV. Нажмите **Далее**.
    
&nbsp;&nbsp;&nbsp;h. На странице **Сведения об организации** введите название организации и подразделения. Можно указать отдел или подразделения (например, ИТ).
    
&nbsp;&nbsp;&nbsp;я. На странице **Сведения о местоположении** укажите сведения о своем местоположении.
    
&nbsp;&nbsp;&nbsp;j. На странице **Имени субъекта/альтернативное имя субъекта** нужные сведения должны быть автоматически заполнены мастером.
    
&nbsp;&nbsp;&nbsp;k. На странице **Параметров домена SIP в альтернативных имен субъекта (SAN)** установите флажок домена, чтобы добавить sip.<sipdomain> запись в список альтернативных имен субъектов.
    
&nbsp;&nbsp;&nbsp;l. На странице **Настройка дополнительных альтернативных имен субъекта** необходимо добавить все нужные дополнительные альтернативные имена субъекта.
    
&nbsp;&nbsp;&nbsp;m. На странице " **Сводка по запросу** " просмотрите различные сведения о сертификате, которые будут использоваться для создания запроса. При необходимости вернитесь обратно и внесите изменения.
    
&nbsp;&nbsp;&nbsp;n. Когда вы будете готовы, нажмите кнопку **Далее** для создания файла CSR, необходимые для обеспечения ЦС (, можно нажать кнопку **Просмотреть журнал** для просмотра журнала для запроса на сертификат).
    
&nbsp;&nbsp;&nbsp;o. После создания запроса можно нажать кнопку **Просмотр**, чтобы просмотреть сертификат, и **Готово**, чтобы закрыть окно. Содержимое CSR-файла необходимо передать вашему ЦС, чтобы они могли создать сертификат для вас, и вы могли импортировать его на этот компьютер в следующем разделе.
    
&nbsp;&nbsp;&nbsp;p. (ДОПОЛНИТЕЛЬНО) При отправке содержимого CSR-файла может быть запрошена определенная информация, например, следующая (ЦС сильно отличаются, поэтому это может не потребоваться):
    
  - **Microsoft** в качестве серверной платформы;
    
  - **IIS** в качестве версии;
    
  - **Web Server** в качестве типа использования;
    
  - **PKCS7** в качестве формата ответа.
    
 
### <a name="2-import-the-certificate"></a>2. Импорт сертификата

&nbsp;&nbsp;&nbsp;. Войдите, должна быть членом локальной группы администраторов на пограничный сервер, ваш запрос на сертификат из произведенной последнего действия.
    
&nbsp;&nbsp;&nbsp;б. В мастере развертывания рядом с **Шаг 3. Запрос, установка или назначение сертификатов**, нажмите кнопку **Запустить еще раз**.
    
&nbsp;&nbsp;&nbsp;c. На странице **Доступные задачи для сертификатов** щелкните **Импортировать сертификат из P7B-, PFX- или CER-файла**.
    
&nbsp;&nbsp;&nbsp;d. На странице **Импорт сертификата** введите полный путь и имя файла сертификата, который был получен в предыдущем разделе (или щелкните **Обзор**, чтобы найти и выбрать файл). Если ваш сертификат содержит закрытый ключ, убедитесь в том выбрать **файл сертификата содержит закрытый ключ сертификата**и введите пароль для закрытого ключа. По готовности нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;e. На странице **Сводка импорта сертификата** просмотрите сводную информацию и нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;f. На странице **Выполнения команд** можно просмотреть в результате импорта по ее завершении, нажав кнопку **Просмотреть журнал**. Нажмите кнопку **Готово**, чтобы завершить импорт сертификата.
    
&nbsp;&nbsp;&nbsp;g. При наличии других пограничных серверов в пуле необходимо выполнить следующие две процедуры, приведенные также. Если это изолированная пограничного сервера, работа с внешние сертификаты.
    
 
### <a name="3-export-the-certificate"></a>3. Экспорт сертификата

&nbsp;&nbsp;&nbsp;. Убедитесь, что вы вошли в импортировать сертификат, который как локальный администратор пограничного сервера.
    
&nbsp;&nbsp;&nbsp;б. Нажмите кнопку **Пуск**, **запустите** (или откройте **поиска** ) и введите **MMC**.
    
&nbsp;&nbsp;&nbsp;c. В консоли MMC щелкните **Файл** и выберите **Добавить или удалить оснастку**.
    
&nbsp;&nbsp;&nbsp;d. В окне **Добавление и удаление оснасток** щелкните **Сертификаты** и затем нажмите кнопку **Добавить**.
    
&nbsp;&nbsp;&nbsp;e. В диалоговом окне **Оснастка сертификата** выберите **Учетная запись компьютера**. Нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;f. В диалоговом окне **Выбор компьютера** установите **локальный компьютер: (котором запущена эта консоль)**. Нажмите кнопку **Готово **. Нажмите кнопку **ОК**и завершить настройку консоли MMC.
    
&nbsp;&nbsp;&nbsp;g. Дважды щелкните пункт **Сертификаты (локальный компьютер)**, чтобы развернуть хранилища сертификатов. Дважды щелкните пункт **Личные**, а затем щелкните пункт **Сертификаты**.
    
   > [!NOTE]
   > Когда вы находитесь здесь и не отображается все сертификаты в личных сертификатов хранения для локального компьютера. Вам не нужно слежения, если ключ не существует, импортированный сертификат не пришлось закрытого ключа, связанного с ним. Запрос и импорт действия над еще раз, а если вы уверены, что есть все, что вправо, обратитесь к администратору центра сертификации или поставщика. 
  
&nbsp;&nbsp;&nbsp;h. В поле **хранить личные сертификаты** для локального компьютера щелкните правой кнопкой мыши сертификат, который выполняется импорт. Выберите пункт **Все задачи** и нажмите **Экспорт**.
    
&nbsp;&nbsp;&nbsp;я. В окне **Мастера экспорта сертификатов**нажмите кнопку **Далее**. Выберите **Да, экспортировать закрытый ключ**. Нажмите **Далее**.
    
   > [!NOTE]
   > Если **Да, экспортировать закрытый ключ** не поддерживается, затем закрытый ключ для этого сертификата не был помеченные для экспорта перед верно. Для успешного экспорта необходимо снова запросить сертификат у поставщика с закрытым ключом, помеченным для экспорта.
  
&nbsp;&nbsp;&nbsp;j. В диалоговом окне форматов экспортируемых файлов выберите "Файл обмена личной информацией - PKCS #12 (.PFX)", а затем выберите следующее:
    
 &nbsp;&nbsp;&nbsp;я. Включить по возможности все сертификаты в путь сертификации.
    
 &nbsp;&nbsp;&nbsp;II. Экспортировать все расширенные свойства.
    
   > [!NOTE]
   > **NEVER** выберите **Удалить закрытый ключ после успешного экспорта**. Оно будет означает, что вам нужно повторно импортировать сертификат и закрытый ключ обратно в этот пограничный сервер.
  
&nbsp;&nbsp;&nbsp;k. Если требуется назначить пароль для защиты закрытого ключа, введите пароль для закрытого ключа. Введите пароль еще раз для подтверждения и нажмите кнопку **Далее**.
    
&nbsp;&nbsp;&nbsp;l. Введите путь и имя файла для экспортируемого сертификата, используя расширение файла **PFX-файл**. Либо путь должен быть доступен с других пограничных серверов в пуле или вам потребуется переместить файл с помощью внешний носитель (например, USB-диска). После внесения свой выбор, нажмите кнопку **Далее** .
    
&nbsp;&nbsp;&nbsp;m. Проверьте сводные данные в диалоговом окне **Завершение мастера экспорта сертификатов**, а затем нажмите кнопку **Готово**.
    
&nbsp;&nbsp;&nbsp;n. В диалоговом окне с сообщением об успешном экспорте нажмите кнопку **ОК**.
    
&nbsp;&nbsp;&nbsp;o. Теперь необходимо вернуться импорта сертификата раздел до этого и Импорт сертификата для всех остальных пограничных серверов, а затем перейти к назначение, ниже.
    
 
### <a name="4-assign-the-certificate"></a>4. назначение сертификата

&nbsp;&nbsp;&nbsp;. На **EACH** пограничный сервер, в мастере развертывания рядом с **Шаг 3. Запрос, установка или назначение сертификатов**, выберите команду **выполнить еще раз**.
    
&nbsp;&nbsp;&nbsp;б. На странице **Доступные задачи для сертификатов** выберите **Назначить имеющийся сертификат**.
    
&nbsp;&nbsp;&nbsp;c. На странице **Назначение сертификата** выберите **Внешнего пограничного сервера** в списке.
    
&nbsp;&nbsp;&nbsp;d. На странице **Хранилища сертификатов** выберите сертификат, который импортирован для внешнего интерфейса пограничного сервера (выше).
    
&nbsp;&nbsp;&nbsp;e. На странице **Сводка по назначению сертификата** проверьте параметры и затем нажмите кнопку **Далее**, чтобы назначить сертификат.
    
&nbsp;&nbsp;&nbsp;f. На странице завершения работы мастера нажмите кнопку **Готово**.
    
&nbsp;&nbsp;&nbsp;g. После выполнения этой процедуры, это очень полезно определить откройте оснастку консоли Управления сертификатами на каждом сервере, разверните узел **Сертификаты (локальный компьютер)**, разверните узел **Личные**, щелкните **сертификаты**и убедитесь, что внутренний пограничный сервер сертификат отображается в области сведений.
    
   > [!NOTE]
   > Также необходимо настроить сертификаты для своего обратного прокси-сервера. 
  
## <a name="starting-the-edge-servers"></a>Запуск пограничных серверов

После завершения установки вам потребуется запустить службы на каждом пограничном сервере в развертывании:
  
1. На каждом пограничном сервере в **Мастере развертывания**рядом с элементом **Шаг 4: запуск служб**, выберите команду **выполнить**.
    
2. На странице **Запуск Скайп для служб Business Server** просмотрите список служб и нажмите кнопку **Далее** для запуска служб.
    
3. После запуска служб нажмите кнопку **Готово**, чтобы закрыть мастер.
    
4. Также в разделе **Шаг 4. Запуск служб** можно щелкнуть **Состояние службы (дополнительно)**.
    
5.  В консоли **MMC службы** на каждом сервере убедитесь, что запущены все Скайп для служб Business Server.
