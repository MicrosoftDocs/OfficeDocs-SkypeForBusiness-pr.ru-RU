# <a name="configure-the-client-experience-with-skype-for-business-2015"></a>Настройка работы с клиентом в Skype для бизнеса 2015
 
**Сводка:** В этом разделе вы узнаете, как настроить клиентский опыт для пользователей Skype для бизнеса 2015.
  
Skype для бизнеса 2015 предоставляет новый пользовательский интерфейс, основанный на пользовательском интерфейсе Skype. Помимо всех функций Lync, Skype для бизнеса предоставляет новые функции с упрощенными средствами управления и знакомыми значками. Подробные сведения о новом клиенте см. в описании [Skype для бизнеса.](https://go.microsoft.com/fwlink/?LinkId=529022)
  
Skype для бизнеса Server поддерживает новые возможности клиента Skype для бизнеса, а также Lync. Как администратор вы можете выбрать предпочтительный вариант работы с клиентом для пользователей. Например, вам может потребоваться развернуть клиентский опыт Lync до тех пор, пока пользователи в вашей организации не будут полностью обучены новому опыту работы со Skype для бизнеса. Или, если вы еще не обновили всех пользователей до Skype для бизнеса Server, может потребоваться, чтобы все пользователи могли использовать одинаковые клиентские возможности до тех пор, пока все не будут обновлены до нового сервера.
  
> [!IMPORTANT]
> Если в организации развернуты Skype для бизнеса Server и Lync Server, интерфейс клиента по умолчанию будет отличаться в зависимости от версий сервера и параметров пользовательского интерфейса. При первом запуске Skype для бизнеса пользователи всегда будут видеть пользовательский интерфейс Skype для бизнеса, даже если вы выбрали интерфейс клиента Lync. Через несколько минут пользователям будет предложено перейти в режим Lync. Дополнительные сведения см. в **разделе "Поведение клиента первого запуска"** далее в этом разделе.
  
> [!NOTE]
> Клиентский опыт Lync 2013 не является вариантом для версий клиентов Skype для бизнеса 2016 или более поздних версий. Прежде чем пытаться настроить клиентскую среду для использования клиента Lync 2013, проверьте версию клиента, чтобы убедиться, что она не начинается с номера 16; например: 16.x.x.x. 
  
## <a name="configure-the-client-experience"></a>Настройка взаимодействия с клиентом

Вы можете указать интерфейс клиента, который будут видеть пользователи в организации, с помощью cmdlet **Set-CSClientPolicy** с параметром EnableSkypeUI:
  
```powershell
Set-CsClientPolicy  [-Identity <XdsIdentity] [-EnableSkypeUI <$true | $false>]
```

где XdsIdentity ссылается на глобальную политику или именоваемую политику сайта.
  
Следующая команда выбирает интерфейс клиента Skype для бизнеса для всех пользователей в организации, на которых влияет глобальная политика (помните, что политики сайта или пользователя переопрепреют глобальную политику): 
  
```powershell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $true
```

Следующая команда выбирает клиент Lync для всех пользователей в организации, на которых влияет глобальная политика:
  
```powershell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $false
```

Следующая команда выбирает клиент skype для бизнеса для всех пользователей на сайте Redmond:
  
```powershell
Set-CsClientPolicy -Identity site:Redmond -EnableSkypeUI $true
```

Если вы хотите настроить клиентский интерфейс для определенных пользователей в организации, можно создать новую политику пользователей с помощью cmdlet **New-CsClientPolicy,** а затем назначить ее определенным пользователям с помощью этого желета **Grant-CsClientPolicy.**
  
Например, следующая команда создает новую клиентскую политику SalesClientUI, которая выбирает интерфейс клиента Skype для бизнеса:
  
```powershell
New-CsClientPolicy -Identity SalesClientUI -EnableSkypeUI $true
```

Следующая команда назначает политику SalesClientUI всем сотрудникам отдела продаж:
  
```powershell
Get-CsUser -LDAPFilter "Department=Sales" | Grant-CsClientPolicy -PolicyName SalesClientUI
```

## <a name="first-launch-client-behaviors"></a>Поведение клиента при первом запуске

По умолчанию при первом запуске Skype для бизнеса 2015 пользователи всегда будут видеть пользовательский интерфейс Skype для бизнеса, даже если вы выбрали интерфейс клиента Lync, установив для параметра EnableSkypeUI значение $False как описано выше. Через несколько минут пользователям будет предложено перейти в режим Lync.
  
Если вы хотите отобразить пользовательский интерфейс Lync при первом запуске клиента Skype для бизнеса, выполните следующие действия перед первым запуском клиента после обновления:
  
1. Подтвердим, что значение этого $False в используемой вами  `EnableSkypeUI` политике, как описано выше.
    
2. Обновим системный реестр на компьютере пользователя. Это следует сделать до того, как пользователи впервые запускают клиент Skype для бизнеса, и это следует сделать только один раз. Сведения о том, как создать объект групповой политики для обновления реестра на компьютере, который присоединился к домену, см. в разделе далее в этом разделе.
    
    В **ключе [HKEY_CURRENT_USER\Software\Microsoft\Office\Lync]** создайте новое **двоичное** значение.
    
    Имя **значения должно** быть **EnableSkypeUI,** а для данных **значения** должно быть установлено **значение 00 00 00 00**.
    
    Ключ должен выглядеть следующим образом:
    
   <pre>
   [HKEY_CURRENT_USER\Software\Microsoft\Office\Lync]
   "CanSharePptInCollab"=dword:00000001
   "CanShareOneNoteInCollab"=dword:00000001
   "CanAppShareInCollab"=dword:00000001
   "EnableSkypeUI"=hex:00,00,00,00
   </pre>

Теперь пользовательский интерфейс Lync будет отображаться при первом запуске клиентом Skype для бизнеса.
  
### <a name="control-the-display-of-the-welcome-screen-tutorial"></a>Управление отображением учебника по экрану приветствия

Когда пользователи открывают клиент Skype для бизнеса, по умолчанию отображается экран приветствия, включающее 7 быстрых советов, которые большинство *людей запросить.* Вы можете отключить отображение экрана приветствия, но при этом разрешить пользователям доступ к учебнику, добавив следующее значение реестра на клиентский компьютер:
  
В **ключе [HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Lync]** создайте новое значение **DWORD (32-битное).** Имя **значения должно** быть **IsBasicTutorialSeenByUser,** а для данных **Значения** должно быть установлено **значение 1**.
  
Ключ должен выглядеть следующим образом:
  
`"IsBasicTutorialSeenByUser"=dword:00000001`

### <a name="turn-off-the-client-tutorial"></a>Отключение руководства по клиенту

Если вы не хотите, чтобы пользователи могли получать доступ к учебнику, вы можете отключить клиентский учебник со следующим значением реестра:
  
В **ключе [HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Lync]** создайте новое значение **DWORD (32-битное).** Имя **значения должно** быть **TutorialFeatureEnabled,** а для данных **Значения** должно быть установлено **значение 0**.
  
Lync
  
```console
"TutorialFeatureEnabled"=dword:00000000
```

Вы можете снова включить учебник, установив для значения **значение** **1**.

## <a name="create-a-group-policy-object-to-modify-the-registry-on-a-domain-joined-computer"></a>Создание объекта групповой политики для изменения реестра на компьютере, который присоединился к домену

Обновление реестра для отображения интерфейса клиента Lync при первом запуске клиента Skype для бизнеса 2015 должно быть сделано только один раз. При использовании объекта групповой политики (GPO) для обновления реестра необходимо определить объект для создания нового значения, а не обновлять данные значения. При применении GPO, если новое значение не существует, то GPO создаст его и примеет значение 0. 
  
В следующей процедуре описывается изменение реестра таким образом, чтобы интерфейс клиента Lync отображался при первом запуске клиента Skype для бизнеса 2015. Вы также можете использовать эту процедуру для обновления реестра, чтобы отключить учебник по экрану приветствия, как описано выше.
  
### <a name="to-create-the-gpo"></a>Создание GPO

1. Запустите **консоль управления групповыми политиками.**
    
    Сведения об использовании консоли управления групповыми политиками см. в консоли [управления групповыми политиками.](https://go.microsoft.com/fwlink/?LinkId=532759)
    
2. Щелкните правой кнопкой мыши узел **"Объекты групповой** политики" и выберите **пункт "Новый"** в меню.
    
3. В **диалоговом** окну "Новый GPO" введите имя для GPO, например MakeLyncDefaultUI, и нажмите кнопку **"ОК".**
    
4. Щелкните правой кнопкой мыши только что  созданный новый GPO и выберите "Изменить" в меню.
    
5. В **редакторе управления групповыми** политиками разоберем конфигурацию **пользователя,** расширьте параметры, разоберем **параметры Windows,** а затем выберите узел **реестра.**
    
6. Щелкните правой кнопкой мыши узел **реестра** и выберите **новый**  >  **элемент реестра.**
    
7. В **диалоговом окте "Новые свойства** реестра" обновите следующее:
    
   |**Field**|**Значение для выбора или ввода**|
   |:-----|:-----|
   |**Действие** <br/> |**Создание** <br/> |
   |**Куст** <br/> | HKEY_CURRENT_USER <br/> |
   |**Путь к ключу** <br/> |Software\Microsoft\Office\Lync  <br/> |
   |**Имя значения** <br/> |EnableSkypeUI  <br/> |
   |**Тип значения** <br/> |REG_BINARY  <br/> |
   |**Value data** <br/> |00000000  <br/> |
   
8. Нажмите **кнопку** "ОК", чтобы сохранить изменения, а затем закройте GPO.
    
Затем необходимо связать созданный групповой политики с группой пользователей, для которую вы хотите назначить политику, например с OU.
  
### <a name="to-use-the-gpo-to-assign-the-policy"></a>Назначение политики с помощью GPO

1. В консоли управления групповыми политиками щелкните правой кнопкой мыши нужное назначение политики, а затем выберите ссылку на **существующий GPO.**
    
2. В **диалоговом окте** "Выбор GPO" выберите созданный вами GPO, а затем выберите "ОК". 
    
3. На компьютере целевого пользователя откройте командную подсказку и введите следующую команду:
       
```console
gpupdate /target:user
```

    
    The message "Updating policy..." is displayed while the GPO is applied. When it is completed, the message "User Policy update has completed successfully" is displayed.
    
4. Введите следующую команду:
    
    **gpresult /r**
    
    Ниже должно отобразиться имя созданного объекта групповой политики "Assigned Group Policy Objects".
    
Вы также можете убедиться, что GPO успешно обновил реестр на компьютере пользователя, проверив реестр. Откройте редактор реестра и перейдите к **ключу [HKEY_CURRENT_USER\Software\Microsoft\Office\Lync].** Если GPO успешно обновил реестр, вы увидите значение EnableSkypeUI со значением 0.
