# <a name="configure-the-client-experience-with-skype-for-business-2015"></a>Настройка клиентской работы с Skype для бизнеса 2015 г.
 
**Сводка:** Ознакомьтесь с этой темой, чтобы узнать, как настроить клиентскую Skype для бизнеса пользователей 2015 года.
  
Skype для бизнеса 2015 г. предоставляет новый пользовательский интерфейс, основанный на опыте Skype продукта. Помимо всех функций Lync, Skype для бизнеса новые функции с упрощенным управлением и знакомыми значками. Подробные сведения о новом клиенте см. в [Skype для бизнеса.](https://go.microsoft.com/fwlink/?LinkId=529022)
  
Skype для бизнеса Server поддерживает новые возможности Skype для бизнеса, а также клиентскую службу Lync. В качестве администратора вы можете выбрать предпочтительный клиентский опыт для пользователей. Например, может потребоваться развернуть клиентский опыт Lync до тех пор, пока пользователи в организации не будут полностью обучены новому Skype для бизнеса. Или, если вы еще не обновили все Skype для бизнеса Server, вы можете захотеть, чтобы у всех пользователей был одинаковый клиентский опыт до тех пор, пока все не будут обновлены до нового сервера.
  
> [!IMPORTANT]
> Если в вашей организации развернуты Skype для бизнеса Server и Lync Server, по умолчанию клиентский опыт будет отличаться в зависимости от версий сервера и параметров пользовательского интерфейса. При первом запуске Skype для бизнеса пользователи всегда будут видеть интерфейс Skype для бизнеса, даже если вы выбрали клиентскую возможность Lync. Через несколько минут пользователям будет предложено перейти в режим Lync. Дополнительные сведения см. в **разделе First launch client behaviour** later in this topic.
  
> [!NOTE]
> Клиентская возможность Lync 2013 не является вариантом для Skype для бизнеса 2016 или более поздней версии. Прежде чем пытаться настроить клиентскую среду для использования клиента Lync 2013, проверьте клиентскую версию, чтобы убедиться, что она не начинается с номера 16; например: 16.x.x.x. 
  
## <a name="configure-the-client-experience"></a>Настройка взаимодействия с клиентом

Вы можете указать клиентский опыт, который увидят пользователи в организации с помощью комлета **Set-CSClientPolicy** с параметром EnableSkypeUI:
  
```powershell
Set-CsClientPolicy  [-Identity <XdsIdentity] [-EnableSkypeUI <$true | $false>]
```

где XdsIdentity ссылается на глобальную политику или именоваемую политику сайта.
  
Следующая команда выбирает Skype для бизнеса для всех пользователей в организации, затронутых глобальной политикой (помните, политики сайта или пользователя переопределяют глобальную политику): 
  
```powershell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $true
```

Следующая команда выбирает клиентскую возможность Lync для всех пользователей в организации, затронутых глобальной политикой:
  
```powershell
Set-CsClientPolicy -Identity Global -EnableSkypeUI $false
```

Следующая команда выбирает Skype для бизнеса для всех пользователей сайта Redmond:
  
```powershell
Set-CsClientPolicy -Identity site:Redmond -EnableSkypeUI $true
```

Если вы хотите настроить клиентский интерфейс для определенных пользователей в организации, можно создать новую политику пользователей с помощью комлета **New-CsClientPolicy,** а затем назначить политику конкретным пользователям с помощью cmdlet **Grant-CsClientPolicy.**
  
Например, следующая команда создает новую клиентскую политику SalesClientUI, которая выбирает Skype для бизнеса клиента:
  
```powershell
New-CsClientPolicy -Identity SalesClientUI -EnableSkypeUI $true
```

Следующая команда назначает политику SalesClientUI всем сотрудникам отдела продаж:
  
```powershell
Get-CsUser -LDAPFilter "Department=Sales" | Grant-CsClientPolicy -PolicyName SalesClientUI
```

## <a name="first-launch-client-behaviors"></a>Поведение клиентов первого запуска

По умолчанию при первом запуске Skype для бизнеса 2015 пользователи всегда будут видеть пользовательский интерфейс Skype для бизнеса , даже если вы выбрали клиентский интерфейс Lync, установив значение параметра EnableSkypeUI до $False, как описано выше. Через несколько минут пользователям будет предложено перейти в режим Lync.
  
Если вы хотите отобразить пользовательский интерфейс Lync при первом запуске клиентом Skype для бизнеса, выполните следующие действия перед первым запуском клиента после обновления:
  
1. Подтвердим, что значение $False в используемой политике,  `EnableSkypeUI` как описано ранее.
    
2. Обновление системного реестра на компьютере пользователя. Это необходимо сделать перед первым запуском клиентом Skype для бизнеса, и это следует сделать только один раз. Сведения о том, как создать объект групповой политики для обновления реестра на компьютере, присоединимом к домену, см. в разделе далее в этой статье.
    
    В **ключе [HKEY_CURRENT_USER\Software\Microsoft\Office\Lync]** создайте новое **двоичное** значение.
    
    Имя **Value должно** быть **EnableSkypeUI,** а данные **значения** должны быть за установлены до **00 00 00 00**.
    
    Ключ должен выглядеть следующим образом:
    
   <pre>
   [HKEY_CURRENT_USER\Software\Microsoft\Office\Lync]
   "CanSharePptInCollab"=dword:00000001
   "CanShareOneNoteInCollab"=dword:00000001
   "CanAppShareInCollab"=dword:00000001
   "EnableSkypeUI"=hex:00,00,00,00
   </pre>

Пользовательский интерфейс Lync будет отображаться при первом запуске Skype для бизнеса клиента.
  
### <a name="control-the-display-of-the-welcome-screen-tutorial"></a>Управление отображением учебника по экрану Welcome

Когда пользователи открывают клиент Skype для бизнеса, по умолчанию отображается экран Welcome, который включает *7* быстрых советов, которые большинство людей просят . Вы можете отключить отображение экрана Welcome, но все же разрешить пользователям доступ к учебнику, добавив следующее значение реестра на клиентский компьютер:
  
В **ключе [HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Lync]** создайте новое **значение DWORD (32-bit).** Имя **Значения должно** быть **IsBasicTutorialSeenByUser,** а данные значения должны быть за установлены **до 1**. 
  
Ключ должен выглядеть следующим образом:
  
`"IsBasicTutorialSeenByUser"=dword:00000001`

### <a name="turn-off-the-client-tutorial"></a>Отключение клиентского учебника

Если вы не хотите, чтобы пользователи могли получить доступ к учебнику, вы можете отключить клиентский учебник со следующим значением реестра:
  
В **ключе [HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Lync]** создайте новое **значение DWORD (32-bit).** Имя **Value должно** быть **TutorialFeatureEnabled,** а данные **значения** должны быть заданы **0**.
  
Lync
  
```console
"TutorialFeatureEnabled"=dword:00000000
```

Вы можете включить учебник, установив значение **данных** до **1**.

## <a name="create-a-group-policy-object-to-modify-the-registry-on-a-domain-joined-computer"></a>Создание объекта групповой политики для изменения реестра на компьютере, присоединимом к домену.

Обновление реестра для отображения клиентского интерфейса Lync при первом запуске клиентского Skype для бизнеса 2015 года должно быть сделано только один раз. Если для обновления реестра используется объект групповой политики (GPO), необходимо определить объект, чтобы создать новое значение, а не обновить данные value. При применении GPO, если нового значения не существует, GPO создаст его и закадрит данные Значения до 0. 
  
В следующей процедуре описывается изменение реестра таким образом, чтобы клиент Lync отображался при первом запуске клиентом Skype для бизнеса 2015 года. Вы также можете использовать эту процедуру для обновления реестра, чтобы отключить учебник по экрану Welcome, как описано ранее.
  
### <a name="to-create-the-gpo"></a>Создание GPO

1. Запустите **консоль управления групповой политикой.**
    
    Сведения об использовании консоли управления групповой политикой см. в консоли [управления групповой политикой.](/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/dn265969(v=ws.11))
    
2. Щелкните правой кнопкой мыши узел **объекты групповой политики** и **выберите New** в меню.
    
3. В **диалоговом окну New GPO** введите имя GPO, например MakeLyncDefaultUI, а затем нажмите **кнопку ОК**.
    
4. Щелкните правой кнопкой мыши по только что созданному новому GPO, а затем выберите **Изменить** из меню.
    
5. В **редакторе управления групповой** политикой развягте конфигурацию **пользователя,** расширите **настройки,** **Windows Параметры** и выберите узел **реестра.**
    
6. Щелкните правой кнопкой мыши узел **реестра,** а затем выберите **элемент New**  >  **Registry Item**.
    
7. В **диалоговом октаге New Registry Properties** обновите следующее:
    
   |**Field**|**Значение для выбора или ввода**|
   |:-----|:-----|
   |**Действие** <br/> |**Создание** <br/> |
   |**Куст** <br/> | HKEY_CURRENT_USER <br/> |
   |**Путь ключа** <br/> |Software\Microsoft\Office\Lync  <br/> |
   |**Имя значения** <br/> |EnableSkypeUI  <br/> |
   |**Тип значения** <br/> |REG_BINARY  <br/> |
   |**Value data** <br/> |00000000  <br/> |
   
8. Нажмите **кнопку ОК,** чтобы сохранить изменения, а затем закрыть GPO.
    
Далее необходимо связать созданный GPO с группой пользователей, на которую нужно назначить политику, например с OU.
  
### <a name="to-use-the-gpo-to-assign-the-policy"></a>Использование GPO для назначения политики

1. В консоли управления групповой политикой щелкните правой кнопкой мыши по OU, на который необходимо назначить политику, а затем выберите ссылку на **существующую GPO.**
    
2. В **диалоговом окте Выберите GPO** выберите созданную GPO и выберите **ОК.**
    
3. На компьютере целевого пользователя откройте командную подсказку и введите следующую команду:
       
   ```console
   pupdate /target:user
   ```
     Сообщение "Обновление политики...". отображается при применении GPO. По его завершению отображается сообщение "Обновление политики пользователя успешно завершено".
    
4. Введите следующую команду:
    
    **gpresult /r**
    
    Вы должны увидеть "Присвоенные объекты групповой политики" с именем созданного вами GPO, отображаемого ниже.
    
Вы также можете убедиться, что GPO успешно обновила реестр на компьютере пользователя, изучив реестр. Откройте редактор реестра и перейдите к **клавише [HKEY_CURRENT_USER\Software\Microsoft\Office\Lync].** Если GPO успешно обновила реестр, вы увидите значение EnableSkypeUI со значением 0.
