# <a name="skype-room-system-manageability-and-tools"></a>Возможности управления и инструменты для системы комнат Skype
 
В этом разделе описываются возможности инструментов управления для Системы комнат Skype.
  
## <a name="administrative-portal"></a>Портал администрирования

Для локальных развертываний Skype для бизнеса Server вы можете управлять разрешениями на управление комнатой Skype в Организации с помощью административного портала для помещения в Skype.
  
Для получения дополнительных сведений ознакомьтесь со следующей статьей:
  
- [Развертывание веб-портала администрирования SRS v1 в Skype для бизнеса Server](../deploy-conferencing/room-system-v1-administrative-web-portal.md)
    
## <a name="system-center-operations-manager"></a>System Center Operations Manager

С помощью System Center Operations Manager (SCOM) можно наблюдать за системой комнаты Skype, загружая [пакет управления комнатой Skype](https://www.microsoft.com/download/details.aspx?id=42320) и устанавливая его на сервер SCOM. Вы можете использовать SCOM для настройки оповещений, контроля за работоспособностью системы помещения в Skype, создания отчетов о времени работы и многого другого. Система комнат Skype поставляется с предварительно установленным агентом мониторинга, который можно настроить так, чтобы он указывал на сервер SCOM. Ознакомьтесь с руководством по установке, предоставленным производителем системы для помещения в Skype, чтобы узнать, как настроить агент мониторинга в системе комнат Skype.
  
## <a name="exchange-checklist"></a>Контрольный список Exchange

- Убедитесь, что автообнаружение настроено и внутренняя запись DNS A/CNAME RECORD доступна для домена autodiscover.domain.com.
    
- Проверьте связь автообнаружения (например, проверьте связь с доменом Autodiscover.contoso.com).
    
- Протестируйте службу автообнаружения с помощью анализатора подключений корпорации Microsoft. Выберите первый тест, "я не могу войти в Office Outlook".
    
- Если в комнате собрания уже есть почтовый ящик ресурсов, расширьте эту учетную запись для системы помещения Skype (пример сценария в нижней части страницы).
    
## <a name="skype-for-business-checklist"></a>Контрольный список Skype для бизнеса

- Запустите следующие инструменты:
    
  - Анализатор соответствия рекомендациям для Skype для бизнеса     
  - Средство анализа работоспособности Skype для бизнеса (Excel)    
  - Анализатор подключений к Skype для бизнеса (32 – бит или 64-разр.)
    
- Ознакомьтесь [с новыми полезными инструментами для устранения неполадок и анализа для Office 365](https://blogs.technet.microsoft.com/educloud/2013/08/13/useful-new-troubleshooting-and-analysis-tools-for-office-365/). Убедитесь в том, что у вас есть пул Skype для бизнеса и сервер Office Web Apps, и можете поделиться колодой PowerPoint с помощью клиента Skype для бизнеса.
    
- Если в комнате собрания уже есть почтовый ящик ресурсов, включите его для Skype для бизнеса.
    
- При необходимости запросите DID (номер телефона) для системы комнат переговоров и обновите поле "Общий телефон" в инструменте Active Directory.
    
## <a name="network"></a>Сеть

- Убедитесь, что у вас есть проводное сетевое подключение для системы комнат Skype.
    
- Ознакомьтесь с руководством по планированию сети для Skype для бизнеса.
    
- Запросить оценку сети Skype для бизнеса у партнера Skype для бизнеса.
    
- Ознакомьтесь с данными о производительности, сохраненными в средстве анализа работоспособности Skype для бизнеса (Excel).
    
- Запустите инструмент анализа сети.
    
- Запустите инструмент PreCall Diagnostics.
    
## <a name="skype-room-system-security"></a>Безопасность системы для помещения в Skype

Система комнат Skype — это внедренная система, которая может быть полностью интегрирована в развертывание Windows с помощью модели безопасности Skype для бизнеса, управления правами и инструментов управления, например SCOM. Доступны следующие функции.
  
- Фильтр записи для предоставления записи на диск в пользовательском режиме 
    
- Блокировщик приложений для предотвращения запуска несанкционированных приложений. В пользовательском режиме все порты USB отключены.
    
  - Никакие стандартные приложения и зрители не находились на оборудовании системы комнат Skype. All content is rendered via HTTP or RDP protocols.
    
  - Бытовой компьютер работает под управлением операционной системы Windows Embedded Standard 7. Все оборудование, включая устройства, предоставляются партнерами изготовителя оборудования.
    
  - Присоединение необязательного домена к службе Active Directory Domain Services (AD DS) обеспечивает управление и контроль локальных учетных записей безопасности.
    
- Вы также можете управлять учетной записью локального администратора с помощью центра администрирования Skype для бизнеса.
    
- Система комнат Skype обновляется с помощью стандартных процессов центра обновления Майкрософт.
    
- Система комнат Skype подключается к Skype для бизнеса.
    
  - Skype для бизнеса использует полное шифрование и авторизацию для всех режимов общения.
    
  - Система комнат Skype поддерживает стандарты безопасности и соответствия требованиям Skype для бизнеса. Дополнительные сведения приведены [в разделе Планирование безопасности в Skype для бизнеса Server](../../plan-your-deployment/security/security.md) .
    
## <a name="license"></a>Лицензия

Убедитесь, что используете KMS для активации программного обеспечения. Если да, возможно, потребуется проверить или добавить в него ключ KMS для клиента Skype для бизнеса. Если вы не используете службу KMS, запросите ключ корпоративного лицензирования для ключа MAK клиента Skype для бизнеса.
  
## <a name="license-keys"></a>Ключи лицензии

Система помещения Skype для настольного компьютера запускает Настольный клиент Skype для бизнеса в фоновом режиме. Если система комнаты Skype входит в состав домена, она обнаружит ваш сервер управления ключами. (и если у него есть ключ KMS для корпоративного лицензирования, он будет активирован автоматически). Корпоративное лицензирование также предоставляет ключ MAK, который вводится в формате XXXXX-XXXXX-XXXXX-XXXXX. (Доступ к Интернету требуется для активации с использованием ключа MAK, но не для KMS). Дополнительные сведения можно найти в разделе многопользовательская активация Office 2013.
  
- Для ввода ключа MAK перейдите к средству лицензирования SRS \> для OEM-параметров. Выберите "Проверка состояния". Когда отображается сообщение о том, что продукт не активирован, введите ключ.
    
- Если во время активации появляется сообщение об ошибке "" Служба лицензирования программного обеспечения сообщила, что этот ключ продукта недействителен ", убедитесь в том, что:
    
  - Ключ введен правильно.
    
  - Вы ввели ключ MAK Skype для бизнеса, а не другой ключ.
    
  - У системы есть доступ к Интернету.
    
- Вы можете выполнить активацию с помощью телефона, но сначала потребуется выполнить попытку активации с помощью средства лицензирования SRS. Для активации через телефон начните тестовое собрание (не тестовый звонок, так как он слишком короткий). В мастере активации Office выберите Активация по телефону, звоните в корпорацию Майкрософт, введите длинный номер и введите ответ.
    
## <a name="certificate-authority"></a>Центр сертификации

Убедитесь, что центр сертификации, выдавший сертификат Office Web Apps Server 2013, включил путь HTTP в свойство списка отзыва сертификатов (CRL).
  
Импорт файла сертификата (CRT) в систему комнаты Skype при использовании Skype для бизнеса Server. It's easily obtained from the CertEnroll share of the CA server, or in the Trusted Root folder of any domain joined PC.
  
## <a name="certificates"></a>Сертификаты

Убедитесь, что у Центра сертификации есть путь http для списка отзыва сертификатов (CRL). Если нет, обновите Центр сертификации, чтобы добавить путь.
  
Установите сертификаты в настройках администратора системы комнат Skype в диспетчере сертификатов параметры \> системы. You need the Enterprise Root CA for your internal certificate.
  
Единственный способ получить необходимые сертификаты — обнаружить центр сертификации, выдавший сертификаты. В Skype для бизнеса Server на компьютере с Windows в Skype для бизнеса выберите параметры \> инструменты \> параметры конференц-связи с телефонным подключением. Откроется веб-страница, защищенная центром сертификации, который выдал внутренние сертификаты. Нажмите на значок замка в адресной строке браузера для отображения отчета по безопасности. Нажмите "Просмотр сертификатов" и проверьте свойство "Точка распространения списка отзыва сертификатов". Второй параметр CN должен быть именем сервера центра сертификации. Теперь откройте проводник для этого имени \\ \< \>сервера ЦС для этого адреса \цертенролл. Скопируйте два файла .crl и файл .crt на устройство флэш-памяти и поместите его с левой части доски SMART.
  
Импортируйте файл. CRT в систему помещения Skype в разделе центр сертификации доверенной комнаты.
  
Импортируйте файлы. CRL из системы помещения Skype в папке "промежуточные центры сертификации". (You need to change the file extension filter in Certificate Manager to .crl to see the files).
  
Примечание: Office Web Apps 2013 может использовать тот же центр сертификации, что и Skype для бизнеса. Если вы не можете поделиться PowerPoint на собрании. Узнайте, как получить доступ к файлам CRT и CRL за пределами сетевого центра сертификации Цертенролл, как описано выше. 
  
Членство в домене может упростить некоторые действия, так как вы можете рассматривать систему комнат Skype как систему Windows, и для некоторых аспектов сертификата она может полагаться на службу каталогов Active Directory. However, it's best to manually manage this.
