# <a name="conferencing-policy-for-skype-room-system-accounts"></a>Политика конференц-связи для учетных записей системы комнат Skype
 
В этом разделе описывается назначение политик конференц-связи для учетных записей Системы комнат Skype.
  
## <a name="conferencing-policy-features"></a>Функции политики конференц-связи

Политика конференц-связи, назначенные Скайп комнаты системную учетную запись необходимо иметь определенные характеристики. В большинстве случаев, Скайп комнаты системы клиентского соединения запланированные собрания и поэтому политики конференц-связи организатора собрания влияет на конференции. Тем не менее в Скайп Business Server, определенные компоненты зависят от конфигурации участника. Например если политика участников позволяет максимальное разрешение видео из 1080p, участники смогут работать эту выше возможность видео решение в конференции даже в том случае, если политика организатора не поддерживать его. ## <a name="meeting-authentication"></a>Проверка подлинности в собрании

Система комнаты Скайп выдает запрос для проверки подлинности пользователей при использовании собрания ссылку присоединиться к присоединение к собранию ограниченных; Например, для каких зал собраний в Outlook были настроены параметры собрания. Этот параметр всегда включен для настраиваемых собраний, и пользователям всегда отправляется запрос. Однако к собраниях с неограниченным доступом пользователи могут присоединиться, не проходя проверку подлинности. 
  
Следующая команда позволяет администраторам запрашивать проверку подлинности для всех собраний, включая собрания с неограниченным доступом: 
  
```
Set-CsMeetingConfiguration -RequireRoomSystemsAuthorization $TRUE
```

По умолчанию параметр RequireRoomSystemsAuthorization имеет значение FALSE.
