# <a name="configure-client-bootstrapping-policies"></a>Настройка политик начальной загрузки клиентов
 
**Сводка:** Управление групповыми политиками.
  
Консоль управления групповыми политиками и редактор объектов групповой политики — это инструменты, которые используются для управления групповой политикой. К административному шаблону групповой политики Office прилагаются административные шаблоны lync16. ADMX (ADMX) и. ADML (ADML), которые содержат параметры политики на основе реестра для Skype для бизнеса, которые вы настроили для объектов групповой политики в домене. ADML-файлы дополняют ADMX-файлы для определенных языков. Каждый ADMX- или ADML-файл содержит параметры политики для одного приложения Office. Вы можете [скачать файлы административных шаблонов Office 2016 (ADMX/ADML)](https://www.microsoft.com/en-us/download/details.aspx?id=49030) бесплатно в центре загрузки Майкрософт.
  
Для клиентов Skype для бизнеса существует несколько политик начальной настройки клиентов, которые необходимо настроить, прежде чем пользователи будут впервые входить на сервер. Например, можно настроить серверы по умолчанию и режим безопасности, который должен использоваться клиентом до завершения процесса входа. Вы можете использовать групповую политику, чтобы установить эти параметры в реестре компьютера пользователя, прежде чем они будут входить в систему и начать получать параметры подготовки на стандартах с сервера. В таблице ниже перечислены параметры групповой политики, доступные в Skype для бизнеса.
  
**Параметры групповой политики для Skype для бизнеса**

|Параметр групповой политики|Описание|
|:-----|:-----|
|Укажите сервер (Конфигуратионмоде)  <br/> | Определяет, как Skype для бизнеса определяет транспорт и сервер для использования при входе. В этом параметре задаются следующие значения. <br/>  ServerAddressExternal: указывает имя или IP-адрес сервера, используемого клиентами и федеративными контактами при подключении за пределами внешнего брандмауэра. <br/>  Сервераддрессинтернал: указывает имя сервера или IP-адрес, который используется при подключении клиентов из межсетевого экрана Организации. <br/>  Transport: указывает протокол TCP (Transmission Control Protocol) или протокол TLS (Transport Layer Security). <br/> |
|Поддерживаемые дополнительные серверные версии (Конфигуредсерверчекквалуес)  <br/> |Задает список имен серверных версий, разделенных точками с запятой, на которые приложение Skype для бизнеса Server будет входить в дополнение к серверным версиям, которые поддерживаются по умолчанию.  <br/> |
|Отключает автоматическое обновление журналов сбоев при входе (DisableAutomaticSendTracing)  <br/> |Автоматическая отправка журналов отказов при входе в Skype для бизнеса Server для анализа. Журналы не отправляются автоматически, если вход выполнен успешно. Если эта политика не настроена, происходит следующее.  <br/> Для пользователей Skype для бизнеса Online: журналы отказов при входе автоматически отправляются. Для локальных пользователей Skype для бизнеса: перед отправкой появится диалоговое окно подтверждения. Если этот параметр отключен, журналы входа автоматически отправляются на сервер Skype для бизнеса для локальных пользователей Skype для бизнеса и Skype для бизнеса Online. Если этот параметр включен, журналы входа никогда не отправляются автоматически.  <br/> |
|Отключение возврата HTTP для SIP-соединения (Дисаблехттпконнект)  <br/> |Предотвращает попытки сервера Skype для бизнеса Server подключиться к серверу по протоколу HTTP, если протокол TLS или TCP недоступен. По умолчанию Skype для бизнеса сначала пытается подключиться к серверу по протоколу TLS или TCP и, если ни один из этих способов не прошел успешно, Skype для бизнеса пытается выполнить подключение с помощью HTTP. С помощью этой политики можно отключить попытки резервного использования HTTP.  <br/> |
|Требовать учетные данные для входа (Дисабленткредентиалс)  <br/> |Пользователю необходимо предоставить учетные данные для входа в Skype для бизнеса, а не автоматически использовать учетные данные Windows при входе на сервер SIP.  <br/> |
|Отключение проверки версии сервера (Дисаблесерверчекк)  <br/> |Если для этой политики установлено значение 1, приложение Skype для бизнеса не будет проверять имя и версию сервера перед входом. По умолчанию в Skype для бизнеса перед входом выполняются эти проверки.  <br/> |
|Включите использование BITS для скачивания файлов службы адресной книги (Енаблебитсфоргалдовнлоад).  <br/> |Позволяет Skype для бизнеса использовать фоновую интеллектуальную службу передачи (BITS) для загрузки файлов служб адресной книги.  <br/> |
|Настройка режима безопасности SIP (Енаблесифигхсекуритимоде)  <br/> |В Skype для бизнеса можно безопасно отправлять и получать мгновенные сообщения. Эта политика не оказывает влияние на службы Windows .NET или Microsoft Exchange Server.  <br/> Если вы не настраиваете этот параметр политики, Skype для бизнеса может использовать любой транспорт. Но если она не использует TLS и если сервер выполняет проверку подлинности пользователей, Skype для бизнеса должен использовать проверку подлинности NTLM или Kerberos.  <br/> |
|Начальная задержка скачивания глобальной адресной книги (Галдовнлоадинитиалделай)  <br/> |Задает период времени перед загрузкой глобального списка адресов (GAL). Значение по умолчанию — 60 минут, что означает, что сервер задерживает загрузку файла глобального списка адресов на произвольное время от 0 до 60 минут.  <br/> |
|Запретить пользователям запускать Skype для бизнеса (Превентрун)  <br/> |Запрещает пользователям запускать Skype для бизнеса. Этот параметр политики можно настраивать как в конфигурации компьютера, так и в конфигурации пользователя, однако приоритет имеет параметр политики, заданный в конфигурации компьютера.  <br/> |
|Разрешить хранение паролей пользователей (Савепассворд)  <br/> |Позволяет Skype для бизнеса хранить пароли.  <br/> |
|Настройка режима сжатия SIP (Сипкомпрессион)  <br/> |Указывает, когда следует включать сжатие SIP. По умолчанию сжатие SIP включается в зависимости от скорости адаптера. Следует учитывать, что установка этого параметра может привести к увеличению времени, затрачиваемого на вход.  <br/> |
|Список доверенных доменов (TrustModelData)  <br/> |Отображает список доверенных доменов, не соответствующих префиксу домена SIP клиента.  <br/> |
   
Политики, настроенные на сервере, имеют преимущество перед параметрами групповой политики и параметрами клиентов, настроенными пользователем. В следующей таблице приводится порядок приоритетов в случае возникновения конфликта.
  
**Приоритет групповой политики**

|**Приоритет**|**Место или способ установки**|
|:-----|:-----|
|1  <br/> |Подготовка к работе в Skype для бизнеса Server по стандартным каналам  <br/> |
|2  <br/> |HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Office\16.0\Lync  <br/> |
|3  <br/> |HKEY_CURRENT_USER\SOFTWARE\Policies\Microsoft\Office\16.0\Lync  <br/> |
|4  <br/> |Диалоговое окно "Параметры" в Skype для бизнеса  <br/> |
   
### <a name="to-define-group-policy-settings-by-using-the-skype-for-business-administrative-template-files"></a>Определение параметров групповой политики с помощью файлов административных шаблонов Skype для бизнеса

1. Создайте на корневом уровне папку для хранения всех не зависящих от языка ADMX-файлов. Например, создайте корневую папку для центрального хранилища в контроллере домена в следующем расположении:
    
     `%systemroot%\sysvol\domain\policies\PolicyDefinitions`
    
    > [!NOTE]
    > В данной процедуре предполагается, что требуется управление несколькими компьютерами в домене. В таком случае шаблоны сохраняются в центральном хранилище в папке Sysvol основного контроллера домена. Это обеспечивает реплицируемое расположение центрального хранилища для административных шаблонов домена. 
  
2. Создайте вложенную папку для каждого языка, который вы хотите использовать. Эти вложенные папки будут содержать зависящие от языка ADML-файлы ресурсов. Например, создайте вложенную папку для американского английского (EN-US) в следующем расположении:
    
     `%systemroot%\sysvol\domain\policies\PolicyDefinitions\EN-US`
