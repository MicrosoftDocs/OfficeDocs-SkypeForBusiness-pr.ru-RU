# <a name="configure-an-e9-1-1-voice-route-in-skype-for-business-server"></a>Настройка голосового маршрута E9-1-1 в Skype для бизнеса Server
 
Настройка голосовых маршрутов E9-1-1 в Skype для бизнеса Server Корпоративная голосовая связь. 
  
Чтобы развернуть E9-1-1, сначала необходимо настроить маршрут голосовой связи для выполнения экстренных вызовов. Дополнительные сведения о создании голосовых маршрутов см. в материале [Create or modify a voice route in Skype для бизнеса](create-or-modify-a-voice-route.md). Можно определить более одного маршрута, если, например, развертывание включает основной магистральный канал SIP и дополнительный магистральный канал SIP. 
  
> [!NOTE]
> Чтобы включить информацию о местоположении в приглашение E9-1-1 INVITE, необходимо настроить магистральный канал SIP, который служит для подключения к поставщику службы E9-1-1 для маршрутизации экстренных вызовов через шлюз. Для этого установите для флага EnablePIDFLOSupport в командлете **Set-CsTrunkConfiguration** значение True. Значение по умолчанию для EnablePIDFLOSupport — False. Например: `Set-CsTrunkConfiguration Service:PstnGateway:192.168.0.241 -EnablePIDFLOSupport $true.` не требуется включить прием местоположений для шлюзов отключенной телефонной сети общего нахождения (PSTN) и шлюзов идентификации аварийного местоположения (ELIN).
  
### <a name="to-configure-an-e9-1-1-voice-route"></a>Настройка маршрута голосовой связи E9-1-1

1. Выполните вход в систему компьютера с использованием учетной записи, которая является членом группы RTCUniversalServerAdmins или членом роли администратора CsVoiceAdministrator.
    
2.  Запустите Skype для бизнеса Server: нажмите кнопку **Начните, нажмите** кнопку Все **программы, нажмите** кнопку Skype для бизнеса **2015**, а затем нажмите кнопку **Skype для бизнеса Server управленческой оболочки**.
    
3. Выполните следующий командлет, чтобы создать новую запись об использовании ТСОП. 
    
    Это должно быть то же самое имя, которое будет использоваться для параметра **ТСОП** в политике определения местоположения. Хотя развертывание будет включать несколько записей об использовании телефона, в следующем примере показано, как добавить «Использование экстренных вызовов» в текущий список доступных использований ТСОП. Подробные сведения см. в [материале Configure voice policies, PSTN use records and voice routes in Skype для бизнеса](voice-and-pstn.md).
    
   ```powershell
   Set-CsPstnUsage -Usage @{add='EmergencyUsage'}
   ```

4. Выполните следующий командлет, чтобы создать новый маршрут голосовой связи с помощью записи об использовании ТСОП, созданной на предыдущем этапе.
    
    Шаблон номера должен соответствовать шаблону номера, который используется в параметре **Строка набора номера для экстренной связи** политики определения местоположения. Знак "+" необходим, так как Skype для бизнеса добавляет "+" к вызовам экстренных служб. «Co1-pstngateway-1» — это идентификатор службы магистрального канала SIP для поставщика службы E9-1-1 или для идентификатора службы шлюза ELIN. В следующем примере в качестве имени маршрута голосовой связи используется EmergencyRoute.
    
   ```powershell
   New-CsVoiceRoute -Name "EmergencyRoute" -NumberPattern "^\+911$" -PstnUsages @{add="EmergencyUsage"} -PstnGatewayList @{add="co1-pstngateway-1"}
   ```

5. Дополнительно для подключений магистрали SIP рекомендуется выполнить следующий кодлет, чтобы создать локальный маршрут для вызовов, которые не обрабатываются магистралью SIP поставщика услуг E9-1-1. Этот маршрут будет использоваться, если подключение к поставщику службы E9-1-1 недоступно. 
    
    В следующем примере предполагается, что в политику голосовой связи пользователя включено использование "Локально".
    
   ```powershell
   New-CsVoiceRoute -Name "LocalEmergencyRoute" -NumberPattern "^\+911$" -PstnUsages @{add="Local"} -PstnGatewayList @{add="co1-pstngateway-2"}
   ```
