# <a name="deployment-process-for-call-park-in-skype-for-business"></a>Процесс развертывания парковки вызовов в Skype для бизнеса
 
Процесс развертывания и действия для парковки вызовов в Skype для бизнеса Server Корпоративная голосовая связь.
  
Парковка вызовов позволяет пользователю Корпоративная голосовая связь поставить вызов на удержание с одного телефона, а затем получить вызов позже, набрав внутренний номер (известный как орбита парковки вызовов) с любого телефона.
  
Компоненты, которые использует парковка вызовов, автоматически устанавливаются и включены на сервере переднего Корпоративная голосовая связь. Однако для настройки парковки вызовов, прежде чем она будет доступна пользователям, необходимо использовать следующие действия. 
  
**Процесс развертывания парковки вызовов**

|**Этап**|**Действия**|**Необходимые группы и роли**|**Документация по развертыванию**|
|:-----|:-----|:-----|:-----|
|Настройка диапазонов орбит парковки вызовов в таблице орбит  <br/> |Используйте панель управления Skype для бизнеса Server или с помощью cmdlet **New-CSCallParkOrbit,** чтобы создать диапазоны орбит в таблице орбит парковки вызовов и связать их со службой приложений, в котором размещено приложение парковки вызовов. <br/> **Примечание.** Для бесперебойной интеграции с существующими телефонными планами диапазоны орбит обычно настраиваются как блок виртуальных расширений. Назначение номеров DID как номеров орбит в таблице орбит парковки вызовов не поддерживается. <br/> |RTCUniversalServerAdmins  <br/> CsVoiceAdministrator  <br/> CsServerAdministrator  <br/> CsAdministrator  <br/> |[Создание или изменение диапазона орбит парковки вызовов в Skype для бизнеса](create-or-modify-a-call-park-orbit-range.md) <br/> |
|Настройка параметров парковки вызовов  <br/> | Используйте для настройки параметров парковки вызовов с помощью cmdlet **Set-CsCpsConfiguration.** Как минимум, рекомендуется настроить параметр **OnTimeoutURI,** чтобы настроить пункт назначения для отката, который будет применяться, когда время работы припаркованного звонка испадет. Вы также можете настроить следующие параметры: <br/>  **EnableMusicOnHold** для включения и выключения музыки при удержании вызова (необязательный параметр); <br/>  **MaxCallPickupAttempts** для определения количества ответных звонков припаркованным вызовом в адрес отвечающего телефона до переадресации звонка на резервный URI (необязательный параметр); <br/>  **CallPickupTimeoutThreshold** для определения времени между парковкой вызова и вызовом телефонного номера, по которому на вызов был получен ответ (необязательный параметр); <br/> |RTCUniversalServerAdmins  <br/> CsVoiceAdministrator  <br/> CsServerAdministrator  <br/> CsAdministrator  <br/> |[Настройка параметров парковки вызовов в Skype для бизнеса](configure-call-park-settings.md) <br/> |
|Можно настроить музыку при удержании (необязательно)  <br/> |Используйте командлет **Set-CsCallParkServiceMusicOnHoldFile** для настройки и загрузки аудиофайла, если вы не хотите использовать стандартную музыку при удержании. <br/> |RTCUniversalServerAdmins  <br/> CsVoiceAdministrator  <br/> CsServerAdministrator  <br/> CsAdministrator  <br/> |[Настройка музыки для парковки вызовов на удержании вSkype для бизнеса](customize-call-park-music-on-hold.md) <br/> |
|Настройка политики голосовой почты для пользователей, чтобы включить парковку вызовов  <br/> |Чтобы включить парковку вызовов для пользователей в политике голосовой связи, используйте панель управления Skype для бизнеса Server или с помощью параметра **Set-CSVoicePolicy** с параметром **EnableCallPark.** <br/> По умолчанию парковка вызовов отключена для всех пользователей.  <br/> При наличии нескольких политик голосовой связи убедитесь в том, что свойство EnableCallPark задано для каждой политики голосовой связи, а не только для политики по умолчанию.  <br/> |RTCUniversalServerAdmins  <br/> CsVoiceAdministrator  <br/> CsUserAdministrator  <br/> CsAdministrator  <br/> |[Включить парковку вызовов для пользователей в Skype для бизнеса](enable-call-park-for-users.md) <br/> |
|Проверка правил нормализации для парковки вызовов  <br/> |Орбиты парковки вызовов не требуют нормализации. Убедитесь, что правила нормализации не включают ваши диапазоны орбиты. При необходимости создайте дополнительные правила нормализации для блокирования нормализации орбит.  <br/> |RTCUniversalServerAdmins  <br/> CsVoiceAdministrator  <br/> CsServerAdministrator  <br/> CsAdministrator  <br/> |[Проверка правил нормализации для парковки вызовов в Skype для бизнеса](verify-normalization-rules-for-call-park.md) <br/> |
|Проверка развертывания парковки вызовов  <br/> |Протестировать парковку вызовов и получить их, чтобы убедиться, что конфигурация работает ожидаемым образом.  <br/> |-  <br/> |[(Необязательно) Проверка развертывания парковки вызовов в Skype для бизнеса](optional-verify-call-park-deployment.md) <br/> |
