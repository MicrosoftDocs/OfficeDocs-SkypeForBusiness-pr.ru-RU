# <a name="create-or-modify-a-normalization-rule-in-skype-for-business"></a>Создание или изменение правила нормализации в Скайп для бизнеса

**Сводка:** Узнайте, как определить, создание и изменение правила нормализации в Скайп для Business Server.

Определите, создание и изменение правил нормализации в Скайп для Business Server.

### <a name="to-define-a-normalization-rule-by-using-build-a-normalization-rule"></a>Задание правила нормализации с помощью инструмента "Построение правила нормализации"

1. Откройте Скайп для панели управления Business Server

2. (Необязательно) Выполните действия, описанные в [Создание и изменение абонентской группы в Скайп для Business Server](dial-plans.md) через шаг 11 или [Изменить телефонной](https://technet.microsoft.com/library/a91f02df-cf60-40cf-82fe-e0342c118b91.aspx) по действие 10.

3. **Создание правила нормализации** или **Изменение правила нормализации**введите имя, описывающее шаблон номера для нормализации в поле **имя** (например, 5DigitExtension).

4. В поле **Описание** введите описание правила нормализации (например, "Преобразование пятизначных добавочных номеров") (не обязательно).

5. В разделе **Построение правила нормализации** введите значения в следующих полях.

   - **Начальных цифр** (Необязательно) Укажите начальные цифры должны сопоставляться с шаблоном для сопоставления набираемых номеров. Например type425 Если вы хотите, чтобы шаблон для поиска соответствия набора номера, начинающиеся с 425.

   - **Длина** Укажите количество цифр в шаблоне соответствия и выберите должны сопоставляться с шаблоном на точное соответствие длину, совпадение набираемым номерам, которые по крайней мере длину или совпадение набираемым номерам, любой длины.

   - **Цифры для удаления** (Необязательно) Укажите количество начальных цифр для удаления из набираемых номеров, которые должны сопоставляться с шаблоном в соответствии с.

   - **Цифры для добавления** (Необязательно) Укажите количество цифр для добавления к набираемых номеров, требуется шаблон для поиска соответствия.

     Значения, введенные в этих полях, отражаются в полях **Шаблон для поиска соответствия** и **Правило трансляции**. Например если оставить пустым, type7 **начала цифр** в поле **Длина** и выберите **точно**и укажите 0 в **цифры для удаления**результате регулярное выражение в **шаблон для поиска соответствия** — это:

     ^(\d{7})$

6. В поле **Правило трансляции** укажите шаблон для формата преобразованных телефонных номеров E.164 следующим образом.

   - Значение количества цифр, указанных в шаблоне соответствия. Например, если шаблон поиска соответствия ^(\d{7})$, а затем $1 в перевода правила представляет 7-значных набираемых номеров.

   - (Необязательно) Введите значение в поле **цифры для добавления** , указывающее цифры, которые должны добавляться в начало преобразуемого номер (например, + 1425).

     Например, если **шаблон для поиска соответствия** содержит ^(\d{7})$ как шаблон для набора номера и содержит **правила преобразования** + 1425$ 1 как шаблон для формата E.164 номера телефонов, правило будет выполнять нормализацию номера 5550100 до + 14255550100.

7. Если в результате применения правила нормализации получается внутренний телефонный номер организации, выберите **Внутреннее расширение** (не обязательно).

8. Введите номер для проверки правила нормализации, затем нажмите кнопку **Перейти** (необязательно). Результаты проверки отображаются под полем **Введите номер телефона для проверки**.

    > [!NOTE]
    > Можно сохранить правило нормализации, которое еще не прошло проверку, и изменить его настройки позднее. См. раздел [Test Voice Routing](https://technet.microsoft.com/library/d3aae909-fef6-440f-b144-0b62dc82bf5d.aspx).

9. Нажмите кнопку **ОК** для сохранения правила нормализации.

10. Нажмите кнопку **ОК** для сохранения абонентской группы.

11. На странице **Абонентская группа** нажмите кнопку **Сохранить**, затем **Сохранить все**.

    > [!NOTE]
    > При создании или изменение правила нормализации, необходимо выполнить команду **Сохранить все** публикация изменений конфигурации. Дополнительные сведения см в документации по операциям [Публикация ожидающих изменений конфигурации маршрутизации голосовой связи в Скайп для бизнеса](voice-route-config-changes.md) .

### <a name="to-define-a-normalization-rule-manually"></a>Определение правила нормализации вручную

1. Откройте Скайп для панели управления Business Server

2. (Необязательно) Выполните действия, описанные в [Создание и изменение абонентской группы в Скайп для Business Server](dial-plans.md).

3. **Создание правила нормализации** или **Изменение правила нормализации**введите имя, описывающее шаблон номера для нормализации (например, имя rule5DigitExtension нормализации) в поле **имя** .

4. В поле **Описание** введите описание правила нормализации (например, "Преобразование пятизначных добавочных номеров") (не обязательно).

5. В разделе **Построение правила нормализации** нажмите кнопку **Изменить**.

6. В разделе **Введите регулярное выражение** введите следующие данные.

   - В поле **Сопоставить этот шаблон** укажите шаблон, которому должен соответствовать набираемый телефонный номер.

   - В поле **Правило трансляции** укажите шаблон для формата преобразуемых телефонных номеров E.164.

     Например, если вводится ^(\d{7})$ в **соответствие этому шаблону** и + 1425$ 1 в поле **правило преобразования**, правило будет выполнять нормализацию номера 5550100 до + 14255550100.

7. Если в результате применения правила нормализации получается внутренний телефонный номер организации, выберите **Внутреннее расширение** (не обязательно).

8. Введите номер для проверки правила нормализации, затем нажмите кнопку **Перейти** (необязательно). Результаты проверки отображаются под полем **Введите номер телефона для проверки**.

9. Нажмите кнопку **ОК** для сохранения правила нормализации.

10. Нажмите кнопку **ОК** для сохранения абонентской группы.

11. На странице **Абонентская группа** нажмите кнопку **Сохранить**, затем **Сохранить все**.

    > [!NOTE]
    > При создании или изменение правила нормализации, необходимо выполнить команду **Сохранить все** публикация изменений конфигурации. Дополнительные сведения см в документации по операциям [Публикация ожидающих изменений конфигурации маршрутизации голосовой связи в Скайп для бизнеса](voice-route-config-changes.md) .
