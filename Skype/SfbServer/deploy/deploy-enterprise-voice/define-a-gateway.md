# <a name="define-a-gateway-in-topology-builder-in-skype-for-business-server"></a>Определение шлюза в Topology Builder в Skype для бизнеса Server
 
**Сводка:** Узнайте, как определить шлюз PSTN в Topology Builder в Skype для бизнеса Server.
  
Следуйте этим шагам, чтобы с помощью Topology Builder определить одноранговой сервер, с которым можно связать сервер-посредник, чтобы обеспечить подключение к открытой телефонной сети (PSTN) для пользователей, включенных для Корпоративная голосовая связь. Одноранговой сервер-посредник может быть шлюзом PSTN, IP-PBX или контроллером пограничной связи сеанса (SBC) для поставщика услуг интернет-телефонии (ITSP), к которому можно подключиться, настроив магистраль SIP.
  
### <a name="to-define-a-peer-for-the-mediation-server"></a>Определение одноранговой области для сервера-посредника

1. Начните строитель топологии: нажмите кнопку Начните, щелкните Все **программы,** нажмите кнопку **Skype для бизнеса Server 2015,** а затем нажмите кнопку Skype для бизнеса Server **2015Topology Builder**.
    
2. В Skype для бизнеса Server имя сайта — Общие компоненты — щелкните правой кнопкой мыши узел **шлюзов PSTN** и нажмите кнопку **New PSTN Gateway**.
3. В диалоговом окне **Определение нового шлюза IP/ТСОП** введите полное доменное имя или IP-адрес узла и нажмите кнопку **Далее**.
    
    > [!NOTE]
    > Если вы указываете безопасность транспортного слоя (TLS) в качестве транспортного типа, необходимо указать FQDN вместо IP-адреса однорангового сервера-посредника. 
  
4. Определите режим прослушивания (IPv6 или IPv4) IP-адреса нового шлюза ТСОП и нажмите **Далее**.

5. Определите корневую магистраль для шлюза ТСОП. Магистраль — это логическое подключение между сервером-посредником и шлюзом, уникальным образом идентифицированным по пучку.
    
    {Mediation Server FQDN, порт прослушивания сервера-посредника (TLS или TCP) : IP-адрес шлюза и FQDN, порт прослушивания шлюза}
    
     - При определении шлюза PSTN в Topology Builder необходимо определить корневой магистраль, чтобы успешно добавить шлюз PSTN в топологию.
    
     - Корневая магистраль не может быть удалена, пока не будет удален шлюз ТСОП.
    
6. В статье **Listening Port for IP/PSTN Gateway** введите порт прослушивания, который шлюз, PBX или SBC будут использовать для SIP-сообщений с сервера-посредника, которые будут связаны с корневым стволом шлюза PSTN. (По умолчанию порты 5066 для протокола управления передачей (TCP) и 5067 для безопасности транспортного слоя (TLS) на шлюзе PSTN, PBX или SBC. На устройстве для сохранившихся филиалов на сайте филиала порты по умолчанию 5081 для TCP и 5082 для TLS.)
    
7. В разделе **Транспортный протокол SIP** выберите тип транспорта, используемый одноранговым узлом, и нажмите кнопку **ОК**.
    
    > [!NOTE]
    > В целях безопасности настоятельно рекомендуется развернуть одноранговой сервер- посредник, который может использовать TLS. 
  
8. В **associated Mediation Server** выберите пул сервера-посредника для связи с корневым стволом этого шлюза PSTN.
    
9. В **порту Associated Mediation Server** введите порт прослушивания, который сервер-посредник будет использовать для сообщений SIP из шлюза.
    
    > [!NOTE]
    > С несколькими службами поддержки магистральных Skype для бизнеса Server вы можете определить несколько портов сигналов SIP на сервере-посреднике для связи с несколькими шлюзами PSTN. При определении магистрали порт **Связанного** сервера-посредника должен быть в диапазоне портов прослушивания для соответствующего протокола, разрешенного сервером-посредником. Этот диапазон портов определяется в пулах Skype для бизнеса Server и посреднических пулах. Щелкните правой кнопкой мыши интересуемого пула сервера-посредника и выберите **Изменить свойства**. Укажите диапазон портов в поле **Прослушивающие порты**.
  
10. Убедитесь, что определенный вами одноранговой адрес работает и использует указанный вами IP-адрес или FQDN. Нажмите кнопку **Готово**.
    
11. Щелкните правой кнопкой **мыши узел Skype для бизнеса Server** и нажмите **кнопку Опубликовать топологию**.
