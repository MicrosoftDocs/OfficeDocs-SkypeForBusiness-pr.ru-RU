# <a name="create-or-modify-a-dial-plan-in-skype-for-business-server"></a>Создание или изменение набора номера в Skype для бизнеса Server

**Сводка:** Узнайте, как создать или изменить наборную группу с помощью панели Skype для бизнеса Server управления.

### <a name="to-create-a-dial-plan"></a>Создание абонентской группы

1. Откройте панель Skype для бизнеса Server управления.

2. В левой панели навигации щелкните **Маршрутизация голосовых вызовов** и затем щелкните **Абонентская группа**.

3. На странице **Абонентская группа** нажмите кнопку **Создать**, а затем выберите область для абонентской группы.

   - **Абонентская группа сайта** применяется к целому сайту, кроме пользователей и групп, которым назначена пользовательская абонентская группа. Если вы **выбираете сайт** для области телефонной программы, необходимо выбрать сайт из диалогового окна **Выберите** сайт. Если абонентская группа уже создана для сайта, он не отображается в диалоговом окне **Выбор сайта**.

   - **Абонентская группа пула** может применяться к шлюзу телефонной сети общего пользования (ТСОП) или регистратору. Если вы **выбираете Пул** для области набора, выберите шлюз PSTN или регистратор из диалогового окна **Выберите службу.** Если абонентская группа уже создана для службы (шлюза ТСОП или регистратора), служба не отображается в списке.

   - **Абонентская группа пользователя** может применяться к определенным пользователям или группам.

     > [!NOTE]
     > После выбора области действия абонентской группы изменить ее нельзя.

4. Если вы создаете абонентскую группу пользователя, введите ее имя в поле **Имя** в диалоговом окне **Создание абонентской группы**. После сохранения имени изменить его нельзя.

    > [!NOTE]
    > Для планов набора веб-сайтов поле **Имя** предварительно заселяется и не может быть изменено.> Для планов набора пула поле Имя предварительно заселяется шлюзом PSTN или именем регистратора и не может быть изменено. 

5. В поле  **Простое имя**, которое содержит имя, отображаемое в поле  **Имя**, укажите описательное имя, которое указывает на сайт, службу или пользователя, к которым применяется абонентская группа (необязательно).

   > [!IMPORTANT]
   > Имя **Simple должно** быть уникальным среди всех планов набора в развертывании. Он не может превышать 256 символов Юникод, каждый из которых может быть алфавитным или числимым, дефис  (-), период (.) или подчеркивать (_).> Символы, не поддерживаемые, включают пробелы и зарезервированные символы, как определено в RFC 3966 ( <http://www.ietf.org/rfc/rfc3966.txt> ). Зарезервированные **символы, не** поддерживаемые в **простом имени,** включают следующее: > ";" "/" "?" ":" "@" "&amp;" "=" "+" "$" ","

6. В поле **Описание** можно ввести дополнительные сведения об абонентской группе.

7. Если эта абонентская группа будет использоваться в качестве региона для номеров доступа к конференц-связи, то укажите **Регион конференц-связи с телефонным подключением** (необязательно). В противном случае оставьте это поле пустым.

    > [!NOTE]
    > Регионы конференц-связи с телефонным подключением требуются для связи номеров доступа к конференц-связи с телефонным подключением с одной абонентской группой или несколькими.

8. Поле **Префикс выхода на внешнюю линию** должно содержать значение только в том случае, если для выхода на внешнюю линию пользователям требуется набирать дополнительные цифры, например 9 (необязательно). Длина префикса не может превышать 4 символа ( #, * и цифры 0–9).

    > [!NOTE]
    > При указании префикса выхода на внешнюю линию не требуется создавать новое правило нормализации для преобразования префикса.

9. Настройте и свяжите правила нормализации для абонентской группы:

    - Чтобы выбрать одно или несколько правил из списка всех правил нормализации, доступных в Корпоративная голосовая связь развертывания, щелкните **Выберите**. В диалоговом окне **Выбор правил нормализации** выделите правила, которые необходимо связать с абонентской группой, и затем нажмите кнопку **ОК**.

   - Чтобы создать новое правило нормализации и связать его с абонентской группой, нажмите кнопку **Создать**. Дополнительные сведения о определении нового правила см. в материале [Create or modify a normalization rule in Skype для бизнеса.](normalization-rules.md)

   - Чтобы изменить правило нормализации, которое уже связано с абонентской группой, выделите имя правила и щелкните **Show details** (Подробности).

   - Чтобы скопировать существующее правило нормализации и использовать его в качестве шаблона для нового правила, выделите имя правила и затем щелкните **Copy** (Копировать) и **Paste** (Вставить).

   - Чтобы удалить правило нормализации из абонентской группы, выделите имя правила и щелкните **Remove** (Удалить).

     > [!NOTE]
     > Каждая абонентская группа должна иметь как минимум одно правило нормализации, связанное с ней. Сведения о том, как определить все правила нормализации, необходимые для набора голосовых номеров, см. в документе Plan [for outbound voice routing in Skype для бизнеса Server](../../plan-your-deployment/enterprise-voice-solution/outbound-voice-routing.md) документации по планированию.

10. Убедитесь, что правила нормализации телефонной программы расположены в правильном порядке. Чтобы изменить положение правила в списке, выберите имя правила и нажмите стрелку вверх или вниз.

    > [!IMPORTANT]
    > Skype для бизнеса Server перебирает список правил нормализации сверху вниз и использует первое правило, которое соответствует набраному номеру. Если вы настроили абонентскую группу таким образом, что набираемый номер может соответствовать нескольким правилам нормализации, убедитесь, что после сортировки более строгие правила располагаются выше менее строгих правил. > Правило Keep **All** normalization ^(\d )$ соответствует любому {11} 11-значной цифре. Например, если вы добавляете правило нормализации, которое совпадает с 11-значными числами, которые начинаются с 1425, убедитесь, что keep **All** сортироваться ниже более строгого правила ^(1425\d {7} )$.

11. Введите номер для проверки абонентской группы и щелкните **Go** (Проверить) (необязательно). Результаты проверки отображаются в **Enter a number to test** (Введите номер для проверки).

12. Нажмите кнопку **ОК**.

13. На странице **Абонентская группа** нажмите кнопку **Зафиксировать**, а затем нажмите кнопку **Зафиксировать все**.

    > [!NOTE]
    > Для публикации изменений конфигурации необходимо выполнять команду **Фиксировать все** при каждом создании абонентской группы. Подробные сведения см. в [публикации "Публикация в](voice-route-config-changes.md) ожидании изменений конфигурации маршрутирования голосовой почты в Skype для бизнеса документации по операциям".

### <a name="to-modify-a-dial-plan"></a>Изменение абонентской группы

1. Войдите на компьютер как член группы RTCUniversalServerAdmins или роли CsVoiceAdministrator, CsServerAdministrator или CsAdministrator. Дополнительные сведения см. в разделе **Delegate Setup Permissions**.

2. Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления.

3. На панели навигации слева выберите **Маршрутизация телефонных звонков** и нажмите **Абонентская группа**.

4. На странице **Dial Plan** (Абонентская группа) дважды щелкните имя абонентской группы.

    > [!NOTE]
    > Область абонентской группы и ее имя задаются при создании абонентской группы. Они не могут быть изменены.

5. В поле **Simple name** (Простое имя) (содержит имя, отображаемое в поле **Name** (Имя)) окна **Edit Dial Plan** (Изменение абонентской группы) укажите описательное имя, которое указывает на сайт, службу или пользователя, к которым применяется абонентская группа (необязательно).

    > [!IMPORTANT]
    > Имя **Simple должно быть** уникальным среди всех планов набора в развертывании Lync Server 2013. Он не может превышать 256 символов Юникод, каждый из которых может быть алфавитным или числимым, дефис (-), период (.), знак плюс (+) или подчеркивать (_).> Пробелы не допускаются в поле **Простое имя.**

6. В поле **Description** (Описание) укажите дополнительные сведения об абонентской группе (необязательно).

7. Если эта абонентская группа будет использоваться в качестве региона для номеров доступа к конференц-связи, то укажите **Dial-in conferencing region** (Регион конференц-связи с телефонным подключением) (необязательно). В противном случае оставьте это поле пустым.

    > [!NOTE]
    > Регионы конференц-связи с телефонным подключением требуются для связи номеров доступа к конференц-связи с телефонным подключением с одной абонентской группой или несколькими.

8. Поле **External access prefix** (Префикс выхода на внешнюю линию) должно содержать значение только в том случае, если для выхода на внешнюю линию пользователям требуется набирать дополнительные цифры, например 9 (необязательно). Длина префикса не может превышать 4 символа ( #, * и цифры 0–9).

    > [!NOTE]
    > При указании префикса выхода на внешнюю линию не требуется создавать новое правило нормализации для преобразования префикса.

9. Настройте и свяжите правила нормализации для абонентской группы:

   - Чтобы выбрать одно или несколько правил из списка всех правил нормализации, доступных в Корпоративная голосовая связь развертывания, щелкните **Выберите**. В диалоговом окне **Select Normalization Rules** (Выбор правил нормализации) выделите правила, которые необходимо связать с абонентской группой, и затем нажмите кнопку **ОК**.

   - Чтобы создать новое правило нормализации и связать его с абонентской группой, щелкните **New** (Создать). Дополнительные сведения о определении нового правила см. в материале [Create or modify a normalization rule in Skype для бизнеса.](normalization-rules.md)

   - Чтобы изменить правило нормализации, которое уже связано с абонентской группой, выделите имя правила и щелкните **Show details** (Подробности).

   - Чтобы скопировать существующее правило нормализации и использовать его в качестве шаблона для нового правила, выделите имя правила и затем щелкните **Copy** (Копировать) и **Paste** (Вставить).

   - Чтобы удалить правило нормализации из абонентской группы, выделите имя правила и щелкните **Remove** (Удалить).

     > [!NOTE]
     > Каждая абонентская группа должна иметь как минимум одно правило нормализации, связанное с ней. Дополнительные сведения о том, как определить все правила нормализации, необходимые для набора голосовых номеров, см. в материале [Plan for outbound voice routing in Skype для бизнеса Server](../../plan-your-deployment/enterprise-voice-solution/outbound-voice-routing.md) документации по планированию.

10. Убедитесь, что правила нормализации телефонной программы расположены в правильном порядке. Чтобы изменить положение правила в списке, выберите имя правила и нажмите стрелку вверх или вниз.

    > [!IMPORTANT]
    > Skype для бизнеса Server перебирает список правил нормализации сверху вниз и использует первое правило, которое соответствует набраному номеру. Если вы настроили абонентскую группу таким образом, что набираемый номер может соответствовать нескольким правилам нормализации, убедитесь, что после сортировки более строгие правила располагаются выше менее строгих правил. > Правило Keep **All** normalization ^(\d )$ соответствует любому {11} 11-значной цифре. Если, например, вы добавляете правило нормализации, которое совпадает с 11-значными числами, которые начинаются с 1425, убедитесь, что keep **All** сортироваться ниже более строгого правила ^(1425\d {7} )$ .

11. Введите номер для проверки абонентской группы и щелкните **Go** (Проверить) (необязательно). Результаты проверки отображаются в **Enter a number to test** (Введите номер для проверки).

    > [!NOTE]
    > Если абонентской группе не удается пройти проверку, то вы можете сохранить ее и настроить позже. Дополнительные сведения см. в разделе [Testing Voice Routing](/previous-versions/office/lync-server-2013/lync-server-2013-test-voice-routing).

12. Нажмите кнопку **ОК**.

13. На странице **Абонентская группа** нажмите кнопку **Зафиксировать**, а затем нажмите кнопку **Зафиксировать все**.

    > [!NOTE]
    > Для публикации изменений конфигурации необходимо выполнять команду **Commit all** (Фиксировать все) при каждом изменении или создании абонентской группы. Подробные сведения см. в [публикации "Публикация в](voice-route-config-changes.md) ожидании изменений конфигурации маршрутирования голосовой почты в Skype для бизнеса документации по операциям".
