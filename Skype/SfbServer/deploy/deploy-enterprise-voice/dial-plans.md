# <a name="create-or-modify-a-dial-plan-in-skype-for-business-server"></a>Создание и изменение абонентской группы в Skype для бизнеса Server

**Сводка:** Сведения о том, как создать или изменить абонентскую группу с помощью панели управления "Skype для бизнеса Server".

### <a name="to-create-a-dial-plan"></a>Создание абонентской группы

1. Откройте панель управления Skype для бизнеса Server.

2. В левой панели навигации щелкните **Маршрутизация голосовой связи** и затем щелкните **Абонентская группа**.

3. На странице **Абонентская группа** нажмите кнопку **Создать**, а затем выберите область для абонентской группы.

   - **Абонентская группа сайта** применяется к целому сайту, кроме пользователей и групп, которым назначена пользовательская абонентская группа. Если выбрать **сайт** для области абонентской группы, необходимо выбрать его в диалоговом окне " **Выбор сайта** ". Если абонентская группа уже создана для сайта, он не отображается в диалоговом окне **Выбор сайта**.

   - **Абонентская группа пула** может применяться к шлюзу телефонной сети общего пользования (ТСОП) или регистратору. Если вы выбрали для области действия абонентской **группы вариант Группа** , выберите шлюз или регистратор PSTN в диалоговом окне **Выбор службы** . Если абонентская группа уже создана для службы (шлюза ТСОП или регистратора), служба не отображается в списке.

   - **Абонентская группа пользователя** может применяться к определенным пользователям или группам.

     > [!NOTE]
     > После выбора области действия абонентской группы изменить ее нельзя.

4. Если вы создаете абонентскую группу пользователя, введите ее имя в поле **Имя** в диалоговом окне **Создание абонентской группы**. После сохранения имени изменить его нельзя.

    > [!NOTE]
    > Для абонентских групп это поле " **имя** " предварительно заполнено именем сайта и не может быть изменено. > для абонентской группы, поле " **имя** " предзаполнено с помощью шлюза КТСОП или имени регистратора и не может быть изменено.

5. В поле **Простое имя**, которое содержит имя, отображаемое в поле **Имя**, укажите описательное имя, которое указывает на сайт, службу или пользователя, к которым применяется абонентская группа (необязательно).

   > [!IMPORTANT]
   > **Простое имя** должно быть уникальным среди всех абонентских планов в развертывании. Она не может содержать 256 символов Юникода, каждый из которых может представлять собой алфавитную или цифровую букву, дефис (-), точку (.) или символ подчеркивания (_). > символы **не поддерживаются** : пробелы и зарезервированные символы, определенные<http://www.ietf.org/rfc/rfc3966.txt>в RFC 3966 (). Зарезервированные символы, которые **не поддерживаются** в **простом имени** , включают в себя следующее: > ";" "/" "?" ":" "@" "&amp;" "=" "+" "$" ","

6. (Необязательно) В поле **Описание** можно ввести дополнительные сведения об абонентской группе.

7. Если эта абонентская группа будет использоваться в качестве региона для номеров доступа к конференц-связи, то укажите **Регион конференц-связи с телефонным подключением** (необязательно). В противном случае оставьте это поле пустым.

    > [!NOTE]
    > Регионы конференц-связи с телефонным подключением требуются для связи номеров доступа к конференц-связи с телефонным подключением с одной абонентской группой или несколькими.

8. Поле **Префикс внешнего доступа** должно содержать значение только в том случае, если для выхода на внешнюю линию пользователям требуется набирать дополнительные цифры, например 9 (необязательно). Длина префикса не может превышать 4 символа ( #, * и цифры 0–9).

    > [!NOTE]
    > При указании префикса выхода на внешнюю линию не требуется создавать новое правило нормализации для преобразования префикса.

9. Настройте и свяжите правила нормализации для абонентской группы:

    - Чтобы выбрать одно или несколько правил из списка всех правил нормализации, доступных в вашем корпоративном развертывании, нажмите кнопку **выбрать**. In **Select Normalization Rules**, highlight the rules you want to associate with the dial plan and then click **OK**.

   - Чтобы создать новое правило нормализации и связать его с абонентской группой, нажмите кнопку **Создать**. Подробнее о том, как определить новое правило, можно найти [в разделе Создание или изменение правила нормализации в Skype для бизнеса](normalization-rules.md).

   - Чтобы изменить правило нормализации, которое уже связано с абонентской группой, выделите имя правила и нажмите кнопку **Подробности**.

   - Чтобы скопировать существующее правило нормализации и использовать его в качестве шаблона для нового правила, выделите имя правила и затем нажмите кнопку **Копировать** и **Вставить**.

   - Чтобы удалить правило нормализации из абонентской группы, выделите имя правила и нажмите кнопку **Удалить**.

     > [!NOTE]
     > Каждая абонентская группа должна иметь как минимум одно правило нормализации, связанное с ней. Сведения о том, как определить все правила нормализации, требуемые для абонентской группы, приведены в разделе [Планирование исходящей голосовой маршрутизации в Skype для бизнеса Server](../../plan-your-deployment/enterprise-voice-solution/outbound-voice-routing.md) в документации по планированию.

10. Убедитесь в том, что правила нормализации для абонентской группы расположены в правильном порядке. Чтобы изменить расположение правила в списке, выделите его имя и нажмите кнопку со стрелкой вверх или вниз.

    > [!IMPORTANT]
    > Skype для бизнеса Server обходит список правил нормализации сверху вниз и использует первое правило, которое соответствует номеру набора. Если вы настроили абонентскую группу таким образом, что набираемый номер может соответствовать нескольким правилам нормализации, убедитесь, что после сортировки более строгие правила располагаются выше, чем менее строгие правила. > значение по умолчанию " **сохранять все** правила нормализации{11}^ (\d) $" соответствует любому 11-значному числу. Например, если вы добавите правило нормализации **, которое соответствует** 11-значным числам, начинающимся с 1425, убедитесь, что они отсортированы ниже более ограниченного правила ^ (1425 \{7}d) $.

11. Введите номер для проверки абонентской группы и щелкните **Проверить** (необязательно). Результаты проверки отображаются в **Введите номер для проверки**.

12. Нажмите **ОК**.

13. На странице **Абонентская группа** нажмите кнопку **Сохранить**, а затем нажмите кнопку **Сохранить все**.

    > [!NOTE]
    > Для публикации изменений конфигурации необходимо выполнять команду **Сохранить все** при каждом создании абонентской группы. Дополнительные сведения можно найти в разделе [Публикация ожидающих изменений в конфигурации голосовой маршрутизации в Skype для бизнеса](voice-route-config-changes.md) в документации по эксплуатации.

### <a name="to-modify-a-dial-plan"></a>Изменение абонентской группы

1. Войдите на компьютер как член группы RTCUniversalServerAdmins или роли CsVoiceAdministrator, CsServerAdministrator или CsAdministrator. Дополнительные сведения см. в разделе **Delegate Setup Permissions**.

2. Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server.

3. В левой панели навигации щелкните **Маршрутизация голосовой связи** и затем щелкните **Абонентская группа**.

4. На странице **Абонентская группа** дважды щелкните имя абонентской группы.

    > [!NOTE]
    > Область абонентской группы и ее имя задаются при создании абонентской группы. Они не могут быть изменены.

5. В поле **Простое имя** (содержит имя, отображаемое в поле **Имя**) окна **Изменение абонентской группы** укажите описательное имя, которое указывает на сайт, службу или пользователя, к которым применяется абонентская группа (необязательно).

    > [!IMPORTANT]
    > **Простое имя** должно быть уникальным среди всех абонентских группа в рамках развертывания Lync Server 2013. Она не может содержать 256 символов Юникода, каждый из которых может представлять собой алфавитную или цифровую цифру, дефис (-), точку (.), знак "плюс" (+) или символ подчеркивания (_). в поле **простого имени** > пробелы недопустимы.

6. В поле **Описание** укажите дополнительные сведения об абонентской группе (необязательно).

7. Если эта абонентская группа будет использоваться в качестве региона для номеров доступа к конференц-связи, то укажите **Регион конференц-связи с телефонным подключением** (необязательно). В противном случае оставьте это поле пустым.

    > [!NOTE]
    > Регионы конференц-связи с телефонным подключением требуются для связи номеров доступа к конференц-связи с телефонным подключением с одной абонентской группой или несколькими.

8. Поле **Префикс внешнего доступа** должно содержать значение только в том случае, если для выхода на внешнюю линию пользователям требуется набирать дополнительные цифры, например 9 (необязательно). Длина префикса не может превышать 4 символа ( #, * и цифры 0–9).

    > [!NOTE]
    > При указании префикса выхода на внешнюю линию не требуется создавать новое правило нормализации для преобразования префикса.

9. Настройте и свяжите правила нормализации для абонентской группы:

   - Чтобы выбрать одно или несколько правил из списка всех правил нормализации, доступных в вашем корпоративном развертывании, нажмите кнопку **выбрать**. In the **Select Normalization Rules** dialog box, highlight the rules that you want to associate with the dial plan and then click **OK**.

   - Чтобы создать новое правило нормализации и связать его с абонентской группой, нажмите кнопку **Создать**. Подробнее о том, как определить новое правило, можно найти [в разделе Создание или изменение правила нормализации в Skype для бизнеса](normalization-rules.md).

   - Чтобы изменить правило нормализации, которое уже связано с абонентской группой, выделите имя правила и нажмите кнопку **Подробности**.

   - Чтобы скопировать существующее правило нормализации и использовать его в качестве шаблона для нового правила, выделите имя правила и затем нажмите кнопку **Копировать** и **Вставить**.

   - Чтобы удалить правило нормализации из абонентской группы, выделите имя правила и нажмите кнопку **Удалить**.

     > [!NOTE]
     > Каждая абонентская группа должна иметь как минимум одно правило нормализации, связанное с ней. Подробнее о том, как определить все правила нормализации, требуемые абонентской группе, приведены в разделе Планирование [исходящих голосовых маршрутов в Skype для бизнеса Server](../../plan-your-deployment/enterprise-voice-solution/outbound-voice-routing.md) в документации по планированию.

10. Убедитесь в том, что правила нормализации для абонентской группы расположены в правильном порядке. Чтобы изменить расположение правила в списке, выделите его имя и нажмите кнопку со стрелкой вверх или вниз.

    > [!IMPORTANT]
    > Skype для бизнеса Server обходит список правил нормализации сверху вниз и использует первое правило, которое соответствует номеру набора. Если вы настроили абонентскую группу таким образом, что набираемый номер может соответствовать нескольким правилам нормализации, убедитесь, что после сортировки более строгие правила располагаются выше, чем менее строгие правила. > значение по умолчанию " **сохранять все** правила нормализации{11}^ (\d) $" соответствует любому 11-значному числу. Если, например, вы добавите правило нормализации **, которое соответствует** 11-значным числам, начинающимся с 1425, убедитесь, что они отсортированы ниже более ограниченного правила ^ (1425 \{7}d) $.

11. Введите номер для проверки абонентской группы и щелкните **Проверить** (необязательно). Результаты проверки отображаются в **Введите номер для проверки**.

    > [!NOTE]
    > Если абонентской группе не удается пройти проверку, то вы можете сохранить ее и настроить позже. Дополнительные сведения см. в разделе [Testing Voice Routing](https://technet.microsoft.com/library/d3aae909-fef6-440f-b144-0b62dc82bf5d.aspx).

12. Нажмите **ОК**.

13. На странице **Абонентская группа** нажмите кнопку **Сохранить**, а затем нажмите кнопку **Сохранить все**.

    > [!NOTE]
    > Для публикации изменений конфигурации необходимо выполнять команду **Сохранить все** при каждом изменении или создании абонентской группы. Дополнительные сведения можно найти в разделе [Публикация ожидающих изменений в конфигурации голосовой маршрутизации в Skype для бизнеса](voice-route-config-changes.md) в документации по эксплуатации.
