# <a name="create-or-modify-a-voice-route-in-skype-for-business"></a>Создание или изменение голосового маршрута в Skype для бизнеса
 
**Сводка:** Узнайте, как создать или изменить голосовой маршрут в Skype для бизнеса Server с помощью панели Skype для бизнеса Server управления.
  
### <a name="to-create-a-voice-route-by-using-the-skype-for-business-server-control-panel"></a>Создание голосового маршрута с помощью панели Skype для бизнеса Server управления

1. Войдите на компьютер в качестве члена группы RTCUniversalServerAdmins или роли администратора **CsVoiceAdministrator**, **CsServerAdministrator** или **CsAdministrator**.
    
2. Откройте панель Skype для бизнеса Server управления.
    
3. В левой области навигации щелкните элемент **Маршрутизация голосовых вызовов**.
    
4. Перейдите на вкладку **Route** (Маршрут).
    
5. Нажмите кнопку **Создать**, чтобы вывести на экран диалоговое окно **Создание маршрута голосовых вызовов**.
    
6. В поле **Name** (Имя) введите описательное имя маршрута голосовых вызовов.
    
7. (Дополнительно). В поле **Description** (Описание) введите дополнительные сведения, описывающие маршрут голосовых вызовов.
    
8. Чтобы указать шаблоны, которые будет включать этот маршрут, можно использовать средство **создания шаблона для сопоставления**, чтобы создать регулярное выражение или записать это выражение вручную.
    
   - Чтобы использовать средство **создания шаблона для сопоставления** для создания регулярного выражения, введите следующие значения. Можно указать два типа сопоставления шаблонов.
    
   - **Starting digits for numbers that you want to allow** (Начальные цифры номеров): введите значения префикса, которые должен содержать этот маршрут (включая начальные цифры номера и остальные цифры при необходимости). Например, введите +425 и нажмите кнопку **Добавить**. Повторите этот шаг для каждого значения префикса, которое необходимо включить в маршрут.
    
   - **Исключения**: если требуется указать одно или несколько исключений для значения префикса, выделите префикс и затем щелкните **Исключения**. Введите одно или несколько значений для совпадающих шаблонов, которые вы  *не*  хотите, чтобы этот маршрут вмещал. Например, чтобы исключить номера, начиная с +425237 из маршрута, введите значение  +425237 в поле Исключения, а затем нажмите **кнопку ОК**.
    
   - Чтобы вручную определить соответствующие шаблоны, в средстве **создания шаблонов для сопоставления** нажмите кнопку **Edit** (Изменить), а затем введите регулярное выражение .NET Framework, чтобы указать соответствующий шаблон для целевых телефонных номеров, к которым применяется маршрут. Подробные сведения о записи регулярных выражений см. в материале ["платформа .NET Framework регулярные выражения".](/dotnet/standard/base-types/regular-expressions) 
    
9. Выберите **Блокировать АнтиАОН**, если требуется запретить отображение ИД телефона, с которого поступает исходящий вызов, для получателя звонка. При выборе этого параметра необходимо  указать альтернативный ИД вызываемого звонка, который появится на экране ИД получателя.
    
10. Чтобы связать одну или несколько магистралей с маршрутом голосовых вызовов, нажмите кнопку **Добавить**, а затем выберите магистраль из списка.
    
11. Чтобы связать одно или несколько общедоступных скоммутанных телефонных сетей (PSTN) с голосовым маршрутом, щелкните Выберите и выберите запись из списка записей использования PSTN, которые были определены для Корпоративная голосовая связь развертывания. 
    
    > [!NOTE]
    > Чтобы просмотреть свойства каждой из доступных записей использования [PSTN,](view-pstn-usage-records.md)см. в Skype для бизнеса. > для создания или редактирования записей использования PSTN см. в статью Создание или изменение политики голосовой политики и настройку записей использования [PSTN](voice-policy-and-pstn-usage-records.md) в Skype для бизнеса
  
12. Чтобы оптимизировать производительность, рекомендуется упорядочить записи использования ТСОП. Чтобы изменить положение записи в списке, выберите имя записи и нажмите стрелку вверх или вниз.
    
    > [!NOTE]
    > В отличие от голосовой политики, в которой важен порядок, в котором перечислены записи использования PSTN, порядок, в котором записи использования PSTN перечислены в голосовом маршруте, является незначительным. Однако рекомендуется организовать список по частоте использования. Например: RedmondLocal, RedmondLongDist, RedmondInternational, RedmondBackup. (Skype для бизнеса Server пересекает список сверху вниз.) 
  
13. (Дополнительно). Введите значение в поле **Введите преобразованное число для проверки** и нажмите кнопку **Перейти**. Результаты проверки отобразятся в области под полем.
    
14. Чтобы сохранить маршрут голосовых вызовов, нажмите кнопку **ОК**.
    
    > [!IMPORTANT]
    > Каждый раз при создании маршрута голосовых вызовов необходимо запускать команду **Commit All** (Фиксировать все), чтобы опубликовать изменение в конфигурации. Подробные сведения см. в [материале Публикация ожидающих](voice-route-config-changes.md)изменений конфигурации маршрутивки голосовой маршрутики в Skype для бизнеса. 
  
### <a name="to-modify-a-voice-route"></a>Изменение маршрута голосовых вызовов

1. Откройте панель Skype для бизнеса Server управления.
    
2. В левой панели навигации выберите пункт **Маршрутизация голосовой связи**, а затем — **Route**.
    
3. На странице **Route** (Маршрут) измените маршрут голосовых вызовов с помощью одного из следующих методов:
    
   - Последовательно щелкните имя маршрута голосовых вызовов, **Edit** (Изменить) и **Show details** (Подробности).
    
   - Последовательно щелкните имя маршрута голосовых вызовов, **Edit** (Изменить), **Copy** (Копировать) и **Paste** (Вставить). Последовательно щелкните созданную копию маршрута голосовых вызовов, **Edit** (Изменить) и **Show details** (Подробности).
    
4. На странице **Edit Voice Route** (Изменение маршрута голосовых вызовов) введите описательное имя маршрута голосовых вызовов в поле **Name** (Имя).
    
5. В поле **Description** (Описание) введите дополнительные сведения о маршруте голосовых вызовов (необязательно).
    
6. Чтобы указать шаблоны для маршрута, создайте регулярное выражение с помощью средства **Build a pattern to match** (Создание шаблона для поиска соответствия) или вручную.
    
   - Для создания регулярного выражения с помощью средства **Build a pattern to match** (Создание шаблона для поиска соответствия) введите значения, показанные ниже. Вы можете указать два типа поиска соответствия по шаблону:
    
   - **Starting digits for numbers that you want to allow** (Начальные цифры номеров): введите значения префикса, которые должен содержать этот маршрут (включая начальные цифры номера и остальные цифры при необходимости). Например, введите +425 и затем щелкните **Add** (Добавить). Повторите этот шаг для каждого значения префикса, которое необходимо включить в маршрут.
    
   - **Исключения**: если требуется указать одно или несколько исключений для значения префикса, выделите префикс и затем щелкните **Исключения**. Введите одно или несколько значений для совпадающих шаблонов, которые вы  *не*  хотите, чтобы этот маршрут вмещал. Например, чтобы исключить номера, начиная с +425237 из маршрута, введите значение  +425237 в поле Исключения, а затем нажмите **кнопку ОК**.
    
   - Чтобы определить шаблон совпадения вручную, нажмите кнопку Изменить в шаблоне **Сборка,** чтобы соответствовать инструменту, а затем введите платформа .NET Framework регулярное выражение, чтобы указать соответствие шаблону для номеров телефонов назначения, к которым применяется маршрут.  Подробные сведения о записи регулярных выражений см. в материале ["платформа .NET Framework регулярные выражения".](/dotnet/standard/base-types/regular-expressions) 
    
7. Если вы не хотите, чтобы получатель звонка видел идентификатор телефона, используемого для исходящего вызова, выберите параметр **Скрыть идентификатор звонящего**. При выборе этого параметра необходимо  указать альтернативный ИД вызываемого звонка, который появится на экране ИД получателя.
    
8. Чтобы связать один или несколько магистральных каналов ТСОП с маршрутом голосовых вызовов, нажмите кнопку **Добавить** и затем выберите магистральный канал в списке.
    
9. Чтобы связать одно или несколько использования PSTN  с голосовым маршрутом, щелкните Выберите и выберите запись из списка записей использования PSTN, которые были определены для Корпоративная голосовая связь развертывания.
    
    > [!NOTE]
    > Чтобы просмотреть свойства каждой из доступных записей использования [PSTN,](view-pstn-usage-records.md)см. в Skype для бизнеса. > для создания или редактирования записей использования PSTN см. в статью Создание или изменение голосовой политики и настройка записей использования [PSTN](voice-policy-and-pstn-usage-records.md)в Skype для бизнеса . 
  
10. Чтобы оптимизировать производительность, рекомендуется упорядочить записи использования ТСОП. Чтобы изменить положение записи в списке, выберите имя записи и нажмите стрелку вверх или вниз.
    
    > [!NOTE]
    > Для политики голосовой связи важен порядок, в котором перечислены режимы работы с ТСОП, а для маршрута голосовых вызовов — нет. Однако мы рекомендуем упорядочить список по частоте использования, например: RedmondLocal, RedmondLongDist, RedmondInternational, RedmondBackup. (Skype для бизнеса Server пересекает список сверху вниз.) 
  
11. (Дополнительно). Введите значение в поле **Введите преобразованное число для проверки** и нажмите кнопку **Перейти**. Результаты проверки отобразятся в области под полем.
    
12. Нажмите кнопку **ОК**.
    
13. На странице **Route** (Маршрут) щелкните **Commit** (Фиксировать) и затем щелкните **Commit all** (Фиксировать все). 
    
    > [!NOTE]
    > Для публикации изменений конфигурации необходимо использовать команду **Сохранить все** при каждом изменении или создании маршрута голосовых вызовов. Подробные сведения см. в [публикации "Публикация в](voice-route-config-changes.md) ожидании изменений конфигурации маршрутирования голосовой почты в Skype для бизнеса документации по операциям".
