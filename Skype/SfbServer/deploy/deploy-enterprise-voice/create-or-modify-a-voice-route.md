# <a name="create-or-modify-a-voice-route-in-skype-for-business-2015"></a>Создание или изменение голосового маршрута в Skype для бизнеса 2015
 
**Сводка:** Узнайте, как создать или изменить маршрут голосовой связи в Скайп для Business Server 2015 с помощью Скайп для панели управления Business Server.
  
### <a name="to-create-a-voice-route-by-using-the-skype-for-business-server-control-panel"></a>Создание маршрута голосовых вызовов с помощью Скайп для панели управления Business Server

1. Войдите на компьютер в качестве члена группы RTCUniversalServerAdmins или роли администратора **CsVoiceAdministrator**, **CsServerAdministrator** или **CsAdministrator**.
    
2. Откройте Скайп для панели управления Business Server.
    
3. В левой области навигации щелкните **Маршрутизация голосовой связи**.
    
4. Щелкните **Маршрут**.
    
5. Нажмите кнопку **Создать** для отображения диалогового окна **создания маршрута голосовой связи**.
    
6. В поле **Имя** введите информативное имя маршрута голосовой связи.
    
7. В поле **Описание** введите дополнительные сведения, характеризующие маршрут голосовой связи (не обязательно).
    
8. Регулярное выражение для задания шаблонов, с которыми требуется согласовать этот маршрут, можно сформировать с помощью инструмента **Построение шаблона для поиска соответствия** или написать вручную.
    
   - Для формирования регулярного выражения с помощью инструмента **Построение шаблона для поиска соответствия** для создания регулярного выражения, введите следующие значения. Можно задать два типа соответствия шаблону.
    
   - **Начальных цифр для номера, который вы хотите разрешить**: введите префикс значения, которые необходимо учесть этот маршрут (включая открывающие + при необходимости). Например, введите +425 и нажмите кнопку **Добавить**. Повторите эти шаги для каждого значения префикс, который вы хотите включить в маршрут.
    
   - **Исключения**: Если вы хотите указать одно или несколько исключений для префикса значения, выделите префикс и нажмите кнопку **исключения**. Введите одно или несколько значений в сопоставлении шаблонов, которые вы действие *не* хотите это маршрут для учета. Например чтобы исключить номера, начиная с +425237 с маршрутом, введите значение + 425237 в поле **исключения** и нажмите кнопку **ОК**.
    
   - Для задания шаблона соответствия вручную нажмите кнопку **Изменить** в инструменте **Построение шаблона для поиска соответствия**, затем введите регулярное выражение .NET Framework, определяющее шаблон соответствия для телефонных номеров адресатов, к которым применяется маршрут. Для получения дополнительных сведений о том, как создавать регулярные выражения см [«.NET Framework Regular Expressions»](https://go.microsoft.com/fwlink/p/?linkId=140927). 
    
9. Если требуется, чтобы идентификатор телефона, с которого выполняется посылка вызова, не отображался на устройстве вызываемого абонента, выберите **Скрыть идентификатор звонящего**. Если выбран этот параметр, необходимо указать **альтернативного Звонящего** , которое будет отображаться на получателя вызывающего идентификатор отображения.
    
10. Для связывания одной или нескольких линий связи с маршрутом голосовой связи нажмите кнопку **Добавить**, затем выберите в списке линию связи.
    
11. Для связывания с маршрутом голосовой связи одного или нескольких режимов работы с ТСОП нажмите кнопку **Выбрать** и выберите запись в списке записей о режимах работы с ТСОП, определенных для развертывания корпоративной голосовой связи.
    
    > [!NOTE]
    > Для просмотра свойств каждого из доступных записей использования ТСОП, видеть [представление работы с ТСОП в Скайп для 2015 бизнеса](view-pstn-usage-records.md). > Для создания и изменения записей использования ТСОП, видеть [Создание или изменение политики голосовой связи и Настройка записей использования ТСОП в Скайп для бизнеса 2015](voice-policy-and-pstn-usage-records.md)
  
12. Упорядочите записи использования ТСОП для достижений оптимальной производительности. Чтобы изменить положение записи в списке, выделите имя записи и нажмите кнопку вверх или стрелку вниз.
    
    > [!NOTE]
    > В отличие от политики голосовой связи, где важен порядок, в котором указаны работы с ТСОП, теряет порядке, в котором PSTN записей использования указаны в маршрута голосовых вызовов. Тем не менее мы рекомендуем организовать список с частоты их использования. Например: RedmondLocal, RedmondLongDist, RedmondInternational, RedmondBackup. (Скайп для Business Server проходит по списку сверху вниз.) 
  
13. Введите значение в поле **Преобразованный номер для проверки** и нажмите кнопку **Перейти** (Не обязательно). Результаты проверки отображаются под полем.
    
14. Нажмите кнопку **ОК** для сохранения маршрута голосовой связи.
    
    > [!IMPORTANT]
    > При каждом создании маршрут голосовой связи, необходимо выполнить команду **Сохранить все** публикация изменений конфигурации. Для получения дополнительных сведений ознакомиться с разделом [Publish ожидающих изменений конфигурации маршрутизации голосовой связи в Скайп для 2015 бизнеса](voice-route-config-changes.md). 
  
### <a name="to-modify-a-voice-route"></a>Изменение маршрута голосовых вызовов

1. Откройте Скайп для панели управления Business Server.
    
2. В левой панели навигации щелкните **Маршрутизация голосовой связи**, затем **Маршрут**.
    
3. На странице **Маршрут** (Маршрут) измените маршрут голосовой связи одним из следующих способов.
    
   - Щелкните имя маршрута голосовой связи, затем **Изменить** и **Показать подробности**.
    
   - Щелкните имя маршрута голосовой связи, затем **Изменить**, **Копировать** и **Вставить** (Вставить). Щелкните только что созданную копию маршрута голосовой связи, затем **Изменить** и **Показать подробности**.
    
4. В поле **Имя** на странице **изменения маршрута голосовой связи** введите информативное имя маршрута голосовой связи.
    
5. В поле **Описание** введите дополнительные сведения о маршруте голосовой связи (не обязательно).
    
6. Регулярное выражение для задания шаблонов, с которыми требуется согласовать этот маршрут, можно сформировать с помощью инструмента **Построение шаблона для поиска соответствия** или написать вручную.
    
   - Для формирования регулярного выражения с помощью инструмента **Построение шаблона для поиска соответствия** для создания регулярного выражения, введите следующие значения. Можно задать два типа соответствия шаблону.
    
   - **Начальных цифр для номера, который вы хотите разрешить**: введите префикс значения, которые необходимо учесть этот маршрут (включая открывающие + при необходимости). Например введите +425 и нажмите кнопку **Добавить**. Повторите эти шаги для каждого значения префикс, который вы хотите включить в маршрут.
    
   - **Исключения**: Если вы хотите указать одно или несколько исключений для префикса значения, выделите префикс и нажмите кнопку **исключения**. Введите одно или несколько значений в сопоставлении шаблонов, которые вы действие *не* хотите это маршрут для учета. Например чтобы исключить номера, начиная с +425237 с маршрутом, введите значение + 425237 в поле **исключения** и нажмите кнопку **ОК**.
    
   - Чтобы определить шаблон поиска соответствия вручную, нажмите кнопку **Изменить** в средстве **построения шаблон для поиска** и введите в регулярное выражение .NET Framework, чтобы задать шаблон соответствия для конечных телефонных номеров, к которым применяется маршрут. Для получения дополнительных сведений о том, как создавать регулярные выражения см [«.NET Framework Regular Expressions»](https://go.microsoft.com/fwlink/p/?linkId=140927). 
    
7. Если требуется, чтобы идентификатор телефона, с которого выполняется посылка вызова, не отображался на устройстве вызываемого абонента, выберите **Скрыть идентификатор звонящего**. Если выбран этот параметр, необходимо указать **альтернативного Звонящего** , которое будет отображаться на получателя вызывающего идентификатор отображения.
    
8. Для связывания одной или нескольких линий связи ТСОП с маршрутом голосовой связи нажмите кнопку **Добавить**, затем выберите в списке линию связи.
    
9. Чтобы связать один или несколько случаев использования PSTN с маршрутом голосовых вызовов, нажмите кнопку **выбрать** и выберите запись в списке записей использования ТСОП, которые были определены для развертывания корпоративной голосовой связи.
    
    > [!NOTE]
    > Для просмотра свойств каждого из доступных записей использования ТСОП, видеть [представление работы с ТСОП в Скайп для 2015 бизнеса](view-pstn-usage-records.md). > Для создания и изменения записей использования ТСОП, видеть [Создание или изменение политики голосовой связи и Настройка записей использования ТСОП в Скайп для бизнеса 2015](voice-policy-and-pstn-usage-records.md). 
  
10. Упорядочите записи использования ТСОП для достижений оптимальной производительности. Чтобы изменить положение записи в списке, выделите имя записи и нажмите кнопку вверх или стрелку вниз.
    
    > [!NOTE]
    > В отличие от политики голосовой связи, где важна порядок, в котором PSTN списке записей использования теряет порядок работы с ТСОП в маршрут голосовой связи. Тем не менее, мы рекомендуем организовать списка по частоте использования, например: RedmondLocal, RedmondLongDist, RedmondInternational, RedmondBackup. (Скайп для Business Server проходит по списку сверху вниз.) 
  
11. Введите значение в поле **Преобразованный номер для проверки** и нажмите кнопку **Перейти** (Не обязательно). Результаты проверки отображаются под полем.
    
12. Нажмите **ОК**.
    
13. На странице **Маршрут** (Маршрут) щелкните **Сохранить**, затем **Сохранить все**. 
    
    > [!NOTE]
    > При создании или изменении маршрута голосовых необходимо выполнить команду **Сохранить все** публикация изменений конфигурации. Дополнительные сведения см в документации по операциям [Публикация ожидающих изменений конфигурации маршрутизации голосовой связи в Скайп для 2015 бизнеса](voice-route-config-changes.md) .
