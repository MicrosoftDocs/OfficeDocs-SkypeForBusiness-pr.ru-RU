# <a name="deployment-process-for-response-group-in-skype-for-business"></a>Процесс развертывания для группы ответа в Скайп для бизнеса
 
Процесс развертывания и действия для группы ответа в Скайп Business Server корпоративной голосовой связи.
  
Группа ответа — это функция корпоративной голосовой связи, которая маршрутизирует и ставит в очередь входящие звонки для группы людей, называемых агентами, например для организации справочной службы или службы поддержки клиентов.
  
Компоненты, необходимые для группы ответа установлены и включены автоматически на сервере переднего плана или сервере Standard Edition, при развертывании корпоративной голосовой связи. Чтобы сделать доступными группы ответа для пользователей, необходимо настроить группы агентов, очереди и рабочие процессы. Кроме того администратор группы ответа можно передать конфигурации существующим рабочим процессом для менеджера группы ответа, который можно изменять и повторно настройте рабочий процесс и его группы связанных агентов и очереди.

## <a name="response-group-configuration-prerequisites"></a>Необходимые требования для настройки группы ответа

Группы ответа требуются следующие компоненты:
  
- приложения
    
- "Группа ответа"
    
- Языковые пакеты
    
- Хранилище файлов (для хранения аудиофайлов)
    
- Веб-службы (включая средства настройки группы ответа и консоль входа и выхода агентов)
    
Все эти компоненты устанавливаются по умолчанию при развертывании корпоративной голосовой связи.
  
Необходимо выполнить следующие задачи перед настройкой группы ответа:
  
- Включите пользователей для Lync Server 2013 и корпоративной голосовой связи.
    
- Изменить файл конфигурации для соответствия стандартам Federal Information Processing (FIPS).
    
- Изменить параметры сортировки базы данных для поддержки символов транскрипций ий, мэн, цзан для названий очередей и групп агентов.
    
### <a name="enabling-users"></a>Включение пользователей

Первым шагом для настройки группы ответа — это для создания групп агентов. Перед созданием группу агентов, необходимо включить пользователей, которые будут агентами для группы ответа для Скайп для бизнеса и корпоративной голосовой связи. Предоставление пользователям Скайп для бизнеса обычно имеет этапом развертывания сервера Standard Edition или сервер Enterprise Edition. Для получения дополнительных сведений о включении пользователей на Скайп для бизнеса см. [Включение и отключение пользователей для Lync Server 2013 Preview](http://technet.microsoft.com/library/12497d00-f665-4a97-be68-854c5a8be4fc.aspx). Предоставление пользователям корпоративной голосовой связи обычно является шагом в развертывании корпоративной голосовой связи. Дополнительные сведения см [Включить пользователей для корпоративной голосовой связи в Скайп для Business Server](enable-users-for-enterprise-voice.md). 
  
### <a name="complying-with-fips-requirements"></a>Соответствие требованиям стандарта FIPS

Данный подраздел касается вас, только если вашей организации необходимо соответствовать федеральному стандарту по (FIPS, США).
  
Чтобы обеспечить совместимость с FIPS, необходимо изменить файл Web.config уровня приложения для использования различных криптографический алгоритм после установки веб-служб. Необходимо указать, в которых используется ASP.NET трижды стандарта шифрования данных (3DES) алгоритм для обработки данных о состоянии представления. Для приложения группы ответа это требование применяется к консоли входа и выхода агентов и средство настройки группы ответа. Дополнительные сведения по этому вопросу содержатся в статье базы знаний Майкрософт 911722, «может появиться сообщение об ошибке при получении доступа к веб-страницам ASP.NET, для которого включено после обновления из ASP.NET 1.1 в ASP.NET 2.0, состояние просмотра» в [https://go.microsoft.com/fwlink/p/?linkId=196183](https://go.microsoft.com/fwlink/p/?linkId=196183).
  
Чтобы изменить файл Web.config, сделайте следующее:
  
1. В текстовом редакторе, например в Блокноте, файл Web.config уровня приложения.
    
2. В файле Web.config найдите `<system.web>` раздел.
    
3. Добавьте следующие `<machineKey>` раздел, чтобы в `<system.web>` раздел:
    
   ```
   <machineKey validationKey="AutoGenerate,IsolateApps" decryptionKey="AutoGenerate,IsolateApps" validation="3DES" decryption="3DES"/>
   ```

4. Сохраните файл Web.config.
    
5. Перезапуск службы Internet Information Services (IIS), выполнив следующую команду в командной строке: 
    
   ```
   iisreset
   ```

### <a name="supporting-yi-meng-and-zang-characters"></a>Поддержка символов транскрипций ий, мэн и цзан

Данный раздел относится к вам, только если вашей организации нужна поддержка подобных символов
  
> [!NOTE]
> Подробную информацию об ий, Мэн и Цзан знаки являются и почему они могут быть важными для развертывания, просмотреть сведения в наборы символов GB18030 [https://go.microsoft.com/fwlink/p/?linkId=240223](https://go.microsoft.com/fwlink/p/?linkId=240223). 
  
Для поддержки символов ий, мэн и цзан необходимо изменить параметры сортировки базы данных Rgsconfig. Измените параметры сортировки столбца **Имя** в следующих таблицах каждой базы данных Rgsconfig:
  
- dbo. AgentGroups
    
- dbo. BusinessHours
    
- dbo. HolidaySets
    
- dbo. Очереди
    
- dbo. Рабочие процессы
    
Для SQL Server 2008 R2 и SQL Server 2012, параметры сортировки использования Latin_General_100 (с учетом диакритических знаков). Если применяются эти параметры сортировки во всех именах объектов не учитывается регистр.
  
Изменить параметры сортировки можно с помощью Microsoft SQL Server Management Studio. Для получения дополнительных сведений об использовании этого средства просмотра [«с помощью SQL Server Management Studio»](https://go.microsoft.com/fwlink/p/?linkId=196184). Для изменения параметров сортировки выполните следующие действия.
  
1. Убедитесь, что SQL Server Management Studio настроена для разрешить изменения, которые требуют таблицы следует создать заново. Дополнительные сведения см [«Сохранить (не допускается) диалоговое окно»](https://go.microsoft.com/fwlink/p/?linkId=196186). Для получения дополнительных сведений о настройке параметры сортировки столбца видеть [«как: набор параметров сортировки столбца (визуальные инструменты для баз данных)»](https://go.microsoft.com/fwlink/p/?linkId=196185).
    
2. Используя Microsoft SQL Server Management Studio подключитесь к базе данных Rgsconfig.
    
3. Найдите в базе данных Rgsconfig таблицу, которую требуется изменить, щелкните ее правой кнопкой мыши и выберите **Конструктор**. 
    
4. Измените параметры сортировки столбца **Имя** и сохраните таблицу.

## <a name="overview-of-workflow-creation-scenarios"></a>Обзор сценариев создания рабочих процессов

Существует два возможных сценария создания рабочих процессов.
  
- **Администратор создает и настраивает рабочий процесс**. Участник роли CsResponseGroupAdministrator (или равнозначной роли) создает и активирует рабочий процесс и все его элементы, включая группы агентов, очереди, выходные дни, рабочие часы, музыку в режиме удержания и т. п.
    
- **Администратор создает рабочий процесс, а руководитель настраивает параметры**. Участник роли CsResponseGroupAdministrator (или равнозначной роли) задает основной универсальный код ресурса SIP, отображаемое имя, назначает хотя бы одного участника роли CsResponseGroupManager, выбирает очередь и активирует рабочий процесс. После этого участник роли CsResponseGroupManager может войти и изменить конфигурацию рабочего процесса: создать группы агентов и назначать их очереди, задать телефонный номер, выходные дни, рабочие часы, музыку в режиме удержания и т. п.
    
    > [!NOTE]
    > Если нужно создать управляемый рабочий процесс, рабочий процесс необходимо создать как активный. После сохранения активный управляемый процесс можно изменить и отключить.
