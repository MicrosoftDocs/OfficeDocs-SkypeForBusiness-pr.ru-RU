# <a name="deployment-process-for-group-call-pickup-in-skype-for-business"></a>Процесс развертывания для группового приема звонков в Skype для бизнеса
 
Процесс развертывания и шаги для отправки группового звонка в Skype для бизнеса Server Enterprise.
  
Отправка группового звонка позволяет пользователям отвечать на входящие звонки с коллегами на свои телефоны. 
  
 Компоненты, используемые для отправки группового звонка, автоматически устанавливаются и включаются на сервере переднего плана или стандартном сервере Standard Edition при развертывании корпоративной голосовой связи. Однако для настройки отправки группового звонка необходимо выполнить описанные ниже действия, прежде чем она будет доступна для пользователей.
  
**Процесс развертывания компонента группового ответа на звонки**

|**Этап**|**Шаги**|**Необходимые группы и роли**|**Документация по развертыванию**|
|:-----|:-----|:-----|:-----|
|Включение средства Сефаутил в топологии|Используйте командлет New-Кструстедаппликатионпул для создания нового доверенного пула приложений. С помощью командлета New-Кструстедаппликатион можно указать средство Сефаутил как надежное приложение. Запустите командлет Enable-Кстопологи, чтобы включить топологию. Если вы еще не сделали этого, Скачайте версию средства Сефаутил в Skype для бизнеса Server из этого расположения и установите ее в доверенной группе приложений, созданной на этапе 1. Убедитесь, что SEFAUtil выполняется правильно, воспользовавшись им для отображения параметров переадресации звонков какого-либо пользователя в развертывании. |RTCUniversalServerAdmins  <br/> |[Развертывание средства Сефаутил в Skype для бизнеса](deploy-the-sefautil-tool.md) <br/> [New-CsTrustedApplicationPool](https://docs.microsoft.com/powershell/module/skype/new-cstrustedapplicationpool?view=skype-ps) </br>[New-CsTrustedApplication](https://docs.microsoft.com/powershell/module/skype/new-cstrustedapplication?view=skype-ps)</br>[Enable-CsTopology](https://docs.microsoft.com/powershell/module/skype/enable-cstopology?view=skype-ps) <br/> [Документация по средствам пакета ресурсов в Skype для бизнеса Server 2015](../../management-tools/resource-kit-tools.md). |Настройте диапазоны номеров для ответа на звонки в таблице орбит парковки вызовов  <br/> |С помощью командлета **New-CSCallParkOrbit** создайте диапазоны номеров для ответа на вызовы в таблице орбит приостановки вызовов и назначьте этим диапазонам тип **GroupPickup**.  <br/> Для простой интеграции с существующими абонентскими группами диапазоны номеров обычно настраиваются как блок виртуальных расширений. |Назначение абоненту номера для отправки и включения группового приема для пользователей  <br/> |Используйте параметр/енаблеграуппиккуп в средстве Сефаутил Resource Kit для включения отправки группового звонка и назначения номера для отправки звонков пользователям.  <br/> |-  <br/> |[Включение отправки группового звонка для пользователей и назначение номера группы в Skype для бизнеса](enable-group-call-pickup-for-users-and-assign-a-group-number.md) <br/> |

Поскольку пользователи могут принимать звонки для других пользователей, даже если они находятся в разных группах, им может понадобиться номер группы ответа на звонок для нескольких групп.  <br/> |-  <br/> ||
|Проверка развертывания для группового приема звонков  <br/> | Для того чтобы убедиться в правильности работы конфигурации, проверьте размещение и извлечение вызовов. <br/>  Позвоните пользователю с включенной поддержкой группового ответа на звонки, после чего другой пользователь должен ответить на этот звонок. Второй пользователь может быть членом той же или иной группы либо вообще не иметь поддержки группового ответа на звонки. <br/>  Позвоните пользователю с включенной поддержкой группового ответа на звонки и не отвечайте на этот звонок. <br/> |-  <br/> ||
