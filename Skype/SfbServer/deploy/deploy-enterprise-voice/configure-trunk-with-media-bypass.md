# <a name="skype-for-business-server-configure-a-trunk-with-media-bypass"></a>Skype для бизнеса Server: Настройка магистрали с помощью обхода мультимедиа

**Сводка:** Настройка магистрали с включенным обходом мультимедиа для Skype для бизнеса Server. Это позволит свести к минимуму количество серверов-посредников, предумножив поддержку поставщика магистральных серверов SIP.

Выполните следующие действия, чтобы настроить магистраль с включенным режимом обхода сервера-посредника. Чтобы настроить магистраль с отключенным обходом мультимедиа, см. в этой ссылке Настройка магистрали без обхода мультимедиа [в Skype для бизнеса Server.](configure-trunk-without-media-bypass.md)

Обход сервера-посредника удобно использовать, когда требуется максимально сократить число развернутых серверов-посредников. Дополнительные сведения см. в [ссылке Plan for media bypass in Skype для бизнеса](../../plan-your-deployment/enterprise-voice-solution/media-bypass.md)

Настоятельно рекомендуется включить обход мультимедиа. Однако перед включением обхода мультимедиа в магистрали SIP подтвердите, что ваш квалифицированный поставщик магистральных каналов SIP поддерживает обход мультимедиа и может удовлетворить требования для успешного включения сценария. В частности, поставщик должен иметь IP-адреса серверов во внутренней сети организации.

> [!NOTE]
> Обход средств массовой информации не будет синхронно работать со всеми шлюзами общедоступных переключеных телефонных сетей (PSTN), IP-PBX и Пограничным контролером сеансов (SBC). Корпорация Майкрософт проверила набор шлюзов и SBCs PSTN с сертифицированными партнерами. Обход мультимедиа поддерживается только продуктами и версиями, перечисленными на странице Инфраструктура телефонии [для Skype для бизнеса Server.](../../../SfbPartnerCertification/certification/infra-gateways.md)

Описанная ниже конфигурация магистрали содержит набор параметров, применяемых к магистралям, которым она назначается. Отдельная конфигурация магистрали может действовать в глобальной области (для всех магистралей, которым не назначена более конкретная конфигурация сайта или пула), в области сайта или в области пула. Конфигурация магистрали уровня пула используется для назначения отдельной магистрали.

### <a name="to-configure-a-trunk-with-media-bypass"></a>Настройка магистрали с обходом сервера-посредника

1. Панель Skype для бизнеса Server управления

2. В левой панели навигации щелкните **Маршрутизация голосовой связи**, а затем выберите **Настройка линии связи**.

3. На странице **Конфигурация магистрали** используйте один из следующих методов для настройки магистрали:

   - Дважды щелкните существующую магистраль (например, в **глобальную** магистраль), чтобы открыть диалоговое окно **Изменение конфигурации магистрали**.

   - Нажмите **Создать** и выберите область для новой конфигурации магистрали:

   - **Магистраль сайта**: выберите сайт для этой конфигурации магистрали в разделе **Выбор сайта** и нажмите кнопку **ОК**. Обратите внимание, что сайты, для которых уже создана конфигурация магистрали, не отображаются в разделе **Выбор сайта**. Эта конфигурация будет применяться ко всем магистралям на сайте.

   - **Магистраль пула**: выберите имя магистрали, к которой применяется эта конфигурация. Этот магистраль может быть корневым стволом или любыми дополнительными магистральями, определенными в Topology Builder. В разделе **Выбор службы** нажмите кнопку **ОК**. Обратите внимание, что магистрали, для которых уже создана конфигурация, не отображаются в разделе **Выбор службы**.

      > [!NOTE]
      > Выбранную область действия конфигурации магистрали уже нельзя изменить. > **Поле Имя** предварительно заселяется и не может быть изменено с именем связанного сайта или службы конфигурации магистрали.

4. Укажите значение в поле **Максимальное количество поддерживаемых предварительных диалогов**. Это максимальное число разветвленных ответов, которые может получить ТСОП-шлюз, IP-УАТС или пограничный контроллер сеансов ITSP (SBC) на запрос INVITE, отправленный серверу-посреднику. Значение по умолчанию — 20.

    > [!NOTE]
    > Перед сменой данного значения, проконсультируйтесь со своим поставщиком или производителем оборудования насчет информации о возможностях своей системы.

5. Выберите один из следующих вариантов для **Уровня поддержки шифрования**:

   - **Требуется**. Необходимо использовать шифрование SRTP для защиты трафика между сервером-посредником и шлюзом или УАТС.

   - **Необязательный**: шифрование SRTP будет использоваться, если поставщик услуг или производитель оборудования поддерживает его.

   - **Не поддерживается**: SRTP-шифрование не поддерживается поставщиком или оборудованием, и потому использоваться не будет.

6. Установите флажок **Включить режим обхода сервера-посредника**, если нужно, чтобы одноранговые узлы магистрали обходили сервер-посредник при передаче медиаданных.

    > [!IMPORTANT]
    > Для успешной работы обхода мультимедиа шлюз PSTN, ip-PBX или пограничный контроллер сеанса ITSP должны поддерживать определенные возможности. Подробные сведения см. [в материале Plan for media bypass in Skype для бизнеса.](../../plan-your-deployment/enterprise-voice-solution/media-bypass.md)

7. Установите флажок **Централизованная обработка медиаданных**, если имеется хорошо известная конечная точка для медиаданных (например, ТСОП-шлюз, где завершение медиаданных имеет тот же IP что и завершение сигнала). Сбросьте данный флажок, если у магистрали нет такой точки.

8. Если одноранговая магистраль поддерживает получение запросов SIP REFER от сервера-посредника, выберите ссылку Включить отправку на **шлюзовую** шкатулку.

    > [!NOTE]
    > Если вы отключили этот параметр во время выбора параметра **Обхода** мультимедиа, необходимы дополнительные параметры. Если одноранговая магистраль не поддерживает получение запросов SIP REFER с сервера-посредника и включен обход мультимедиа, необходимо также запустить комлет **Set-CsTrunkConfiguration,** чтобы отключить RTCP для активных и удерживаемых вызовов, чтобы поддерживать надлежащие условия для обхода мультимедиа. Если необходимо, чтобы переключенные звонки обходили медиаданные и шлюз не поддерживал запросы SIP REFER, выберите **Разрешить ссылку с использованием стороннего контроля вызовов**.

9. (Необязательно) Чтобы включить маршрутизацию между магистральными линиями связи, сопоставьте и настройте записи режима работы с ТСОП для этой конфигурации магистральной линии связи. Использование PSTN, связанное с этой конфигурацией магистрали, будет применяться для всех входящих вызовов через магистраль, которая не Skype для бизнеса Server конечной точке. Чтобы управлять записями режима работы с ТСОП, связанными с конфигурацией магистральной линии связи, используйте один из приведенных ниже методов.

   - Чтобы выбрать одну или несколько записей из списка всех записей использования PSTN, доступных в Корпоративная голосовая связь развертывания, щелкните **Выберите**. Выделите записи, которые следует связать с этой конфигурацией магистральной линии связи, и нажмите кнопку **ОК**.

   - Чтобы удалить запись режима работы с ТСОП из этой конфигурации магистральной линии связи, выберите эту запись и щелкните **Удалить**.

   - Чтобы определить новую запись режима работы с ТСОП и сопоставить ее с этой конфигурацией магистральной линии связи, выполните следующие действия.

     А. Нажмите кнопку **Создать**.

     Б. В поле **Имя** укажите уникальное описательное имя для записи.

     > [!NOTE]
     > Имя записи режима работы с ТСОП должно быть уникальным в рамках развертывания корпоративной голосовой связи. После сохранения записи поле **Имя** изменить невозможно.

     c. Используйте один из следующих методов для сопоставления и настройки маршрутов для этой записи режима работы с ТСОП:

     - Чтобы выбрать один или несколько маршрутов из списка всех доступных маршрутов в развертывании Корпоративная голосовая связь, щелкните **Выберите**. Выделите маршруты, которые хотите сопоставить с этой записью режима использования ТСОП, и нажмите кнопку **ОК**.

     - Чтобы удалить маршрут из записи режима работы с ТСОП, выберите этот маршрут и щелкните **Удалить**.

   - Чтобы определить новый маршрут и сопоставить его с этой записью режима работы с ТСОП, щелкните **Создать**. Дополнительные сведения см. [в материале Create or modify a voice route in Skype для бизнеса.](create-or-modify-a-voice-route.md)

     - Чтобы изменить маршрут, сопоставленный с этой записью режима работы с ТСОП, выберите маршрут и щелкните **Подробнее**.

     d. Нажмите кнопку **ОК**.

     - Чтобы изменить запись режима работы с ТСОП, уже сопоставленную с этой конфигурацией магистральной линии связи, выполните следующие действия:

       А. Выберите запись режима работы с ТСОП, которую требуется изменить, и щелкните **Подробнее**.

       Б. Используйте один из следующих методов для сопоставления и настройки маршрутов для этой записи режима работы с ТСОП:

   - Чтобы выбрать один или несколько маршрутов из списка всех доступных маршрутов в развертывании Корпоративная голосовая связь, щелкните **Выберите**. Выделите маршруты, которые хотите сопоставить с этой записью режима использования ТСОП, и нажмите кнопку **ОК**.

   - Чтобы удалить маршрут из записи режима работы с ТСОП, выберите этот маршрут и щелкните **Удалить**.

   - Чтобы определить новый маршрут и сопоставить его с этой записью режима работы с ТСОП, щелкните **Создать**. Дополнительные сведения см. [в материале Create or modify a voice route in Skype для бизнеса.](create-or-modify-a-voice-route.md)

   - Чтобы изменить маршрут, сопоставленный с этой записью режима работы с ТСОП, выберите маршрут и щелкните **Подробнее**.

     c. Нажмите кнопку **ОК**.

     > [!IMPORTANT]
     > Важно связать записи использования PSTN в соответствии с одноранговой записью сервера-посредника, связанной с настраиваемой магистралью. Если одноранговой сервер-посредник является шлюзом PSTN или пограничным контроллером сеанса (SBC), настоятельно рекомендуется, чтобы конфигурация магистрали не была связана с записью использования PSTN, которая передается в пункт назначения PSTN или в другие нижестояльные системы, подключенные через Skype для бизнеса Server.

10. Чтобы оптимизировать производительность, рекомендуется упорядочить записи использования ТСОП. Чтобы изменить положение записи в списке, выберите запись использования PSTN и нажмите стрелки вверх или вниз.

    > [!IMPORTANT]
    > Порядок расположения записей режима работы с ТСОП в списке является важным. Skype для бизнеса Server пересекает список сверху вниз.

11. Чтобы включить обход медиаданных для клиентов, защищенных системой преобразования сетевых адресов (NAT) или брандмауэром и SBC с поддержкой блокировки, выберите **Разрешить блокирование RTP**.

12. **Необходимо выбрать историю вызовов,** чтобы включить отправку сведений об истории вызовов в одноранговой шлюз сервера-посредника.

13. Необходимо выбрать предварительные данные **P-Asserted-Identity,** чтобы в настоящее время информация об источнике вызовов P-Asserted-Identity (PAI) была переадреждена между стороной сервера-посредника и стороной шлюза (и наоборот).

14. Установите флажок **Включить таймер отработки отказа внешней маршрутизации**, чтобы включить быструю отработку отказа. Шлюз, сопоставленный с этой магистралью, может в течение 10 секунд выдать уведомление о том, что он обрабатывает исходящий вызов. Если это уведомление не получено сервером-посредником, будет происходить перенанаружение в другой магистраль. В сетях, где задержка может увеличить время ответа или где на ответ шлюзу требуется более 10 секунд, быструю отработку отказа следует отключить.

15. (Необязательно) Сопоставьте и настройте **правила трансляции вызывающего номера** для магистральной линии связи. Эти правила применяются к вызывающим номерам для исходящих вызовов

    - Чтобы выбрать одно или несколько правил из списка всех правил перевода, доступных в Корпоративная голосовая связь развертывания, нажмите **кнопку Выберите**. В окне **Выбор правил трансляции** выберите правила, которые требуется сопоставить с магистральной линией связи, а затем нажмите кнопку **ОК**.

    - Чтобы определить новое правило преобразования и сопоставить его с магистралью, нажмите кнопку **Создать**. Подробные сведения о правилах перевода [см.](../../plan-your-deployment/enterprise-voice-solution/translation-rules.md)в Skype для бизнеса Server.

    - Чтобы изменить правило преобразования, которое уже сопоставлено с магистралью, щелкните имя правила, затем нажмите кнопку **Подробнее**.

    - Чтобы скопировать существующее правило преобразования и использовать его в качестве отправной точки для определения нового правила, щелкните имя правила и нажмите кнопку **Копировать**, а затем кнопку **Вставить**.

    - Чтобы удалить правило преобразования из магистральной линии связи, выделите имя этого правила, а затем нажмите кнопку **Удалить**.

    > [!CAUTION]
    > Не сопоставляйте правила преобразования с магистралью, если вы настроили правила преобразования на сопоставленном одноранговом узле, поскольку два правила могут конфликтовать.

16. (Необязательно) Сопоставьте и настройте **правила трансляции набранного номера** для магистральной линии связи. Эти правила применяются к вызываемым номерам для исходящих вызовов.

    - Чтобы выбрать одно или несколько правил из списка всех правил перевода, доступных в Корпоративная голосовая связь развертывания, нажмите **кнопку Выберите**. В окне **Выбор правил трансляции** выберите правила, которые требуется сопоставить с магистральной линией связи, а затем нажмите кнопку **ОК**.

    - Чтобы определить новое правило преобразования и сопоставить его с магистралью, нажмите кнопку **Создать**. Подробные сведения о правилах перевода [см.](../../plan-your-deployment/enterprise-voice-solution/translation-rules.md)в Skype для бизнеса Server.

    - Чтобы изменить правило преобразования, которое уже сопоставлено с магистралью, щелкните имя правила, затем нажмите кнопку **Подробнее**.

    - Чтобы скопировать существующее правило преобразования и использовать его в качестве отправной точки для определения нового правила, щелкните имя правила и нажмите кнопку **Копировать**, а затем кнопку **Вставить**.

    - Чтобы удалить правило преобразования из магистральной линии связи, выделите имя этого правила, а затем нажмите кнопку **Удалить**.

    > [!CAUTION]
    > Не сопоставляйте правила преобразования с магистральной линией связи, если вы уже настроили их для связанной одноранговой магистральной линии связи, поскольку эти два правила могут конфликтовать друг с другом.

17. Убедитесь, что правила перевода магистрали расположены в правильном порядке. Чтобы изменить положение правила в списке, выберите имя правила и нажмите стрелку вверх или вниз.

    > [!IMPORTANT]
    > Skype для бизнеса Server перебирает список правил перевода сверху вниз и использует первое правило, которое соответствует набраному номеру. Если вы настраиваете магистраль таким образом, чтобы набираемый номер мог соответствовать нескольким правилам преобразования, убедитесь, что более строгие правила расположены над менее строгими. Например, если вы включили правило перевода, которое совпадает с любым 11-значным номером и правилом перевода, которое соответствует только 11-значным числам, которые начинаются  с +1425, убедитесь, что правило, которое совпадает с любым 11-значным номером, сортироваться ниже более строгого правила.

18. Закончив настройку магистрали, нажмите кнопку **ОК**.

19. На странице **Конфигурация магистрали** нажмите кнопку **Передать**, а затем щелкните **Передать все**.

    > [!NOTE]
    > При каждом создании или изменении конфигурации магистрали необходимо выполнить команду **Сохранить все**, чтобы опубликовать изменение конфигурации. Подробные сведения см. в [публикации "Публикация в](voice-route-config-changes.md) ожидании изменений конфигурации маршрутирования голосовой почты в Skype для бизнеса документации по операциям".

После настройки магистрали [продолжайте настраивать](deploy-media-bypass.md) обход мультимедиа, выбирая между глобальными вариантами обхода мультимедиа, как описано в обходе мультимедиа в Skype для бизнеса Server в документации развертывания.
