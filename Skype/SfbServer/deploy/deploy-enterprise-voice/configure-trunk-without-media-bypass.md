# <a name="skype-for-business-server-configure-a-trunk-without-media-bypass"></a>Skype для бизнеса Server: Настройка магистрали без обхода мультимедиа

**Сводка:** Настройка магистрали без обхода мультимедиа, включенной для Skype для бизнеса Server.

Если вы хотите настроить магистраль с обходом сервера-посредника, выполните эти действия. Если вы хотите настроить магистраль с включенным обходом мультимедиа, см. в этой ссылке Настройка магистрали с обходом мультимедиа [в Skype для бизнеса Server](configure-trunk-with-media-bypass.md).

Конфигурация магистрали представляет собой набор параметров, применяемых к магистрали, которой назначается эта конфигурация. Определенная конфигурация магистрали может применяться глобально (ко всем магистралям, у которых нет особых настроек сайта или пула), к сайту или к пулу. Конфигурация магистрали уровня пула позволяет применить особую конфигурацию к отдельной магистрали.

### <a name="to-configure-a-trunk-without-media-bypass"></a>Настройка магистрали без обхода сервера-посредника

1. Откройте панель Skype для бизнеса Server управления.

2. В левой панели навигации щелкните **Маршрутизация голосовой связи**, а затем выберите **Настройка линии связи**.

3. На странице **Конфигурация магистрали** используйте один из следующих методов для настройки магистрали:

   - Дважды щелкните существующую магистраль (например, в **глобальную** магистраль), чтобы открыть диалоговое окно **Изменение конфигурации магистрали**.

   - Нажмите кнопку **Создать**, а затем выберите область для конфигурации новой магистрали:

   - **Магистраль сайта:** Выберите сайт для этой конфигурации магистрали в **Выберите сайт,** а затем нажмите **кнопку ОК**. Обратите внимание, что сайты, для которых уже создана конфигурация магистрали, не отображаются в разделе **Выбор сайта**. Эта конфигурация будет применяться ко всем магистралям на сайте.

   - **Магистраль** пула. Выберите имя магистрали, к которое применяется эта конфигурация магистрали в **Выберите** службу и нажмите **кнопку ОК.** Этот магистраль может быть корневым стволом или любыми дополнительными магистральми, определенными в Topology Builder. Обратите внимание, что магистрали, для которых уже создана конфигурация, не отображаются в разделе **Выбор службы**.

     > [!NOTE]
     > Выбранную область действия конфигурации магистрали уже нельзя изменить. > Поле **Имя** предварительно заполнено и не может быть изменено с именем связанного сайта или службы конфигурации магистрали.

4. Выберите один из следующих параметров в поле **Уровень поддержки шифрования**:

   - **Обязательно.** Шифрование безопасного транспортного протокола в реальном времени (SRTP) должно использоваться для защиты трафика между сервером-посредником и шлюзом или частным филиалом eXchange (PBX).

   - **Необязательный**: шифрование SRTP будет использоваться, если поставщик услуг или производитель оборудования поддерживает его.

   - **Не поддерживается**: шифрование SRTP не поддерживается поставщиком услуг или производителем оборудования, поэтому оно не будет использоваться.

5. Убедитесь, что флажок **Разрешить обход мультимедиа** снят.

6. Выберите  поле централизованной обработки мультимедиа, если имеется хорошо известная точка прекращения мультимедиа (например, шлюз для общедоступных переключеных телефонных сетей (PSTN), в котором прекращение мультимедиа имеет тот же IP, что и завершение сигнала). Снимите этот флажок, если в магистрали нет известной точки терминирования медиаданных.

7. Если одноранговая магистраль поддерживает получение запросов SIP REFER от сервера-посредника, выберите ссылку Включить отправку на **шлюзовую** шкатулку.

8. (Необязательно) Чтобы включить маршрутизацию между магистральными линиями связи, сопоставьте и настройте записи режима работы с ТСОП для этой конфигурации магистральной линии связи. Использование PSTN, связанное с этой конфигурацией магистрали, будет применяться для всех входящих вызовов через магистраль, которая не Skype для бизнеса Server конечной точке. Чтобы управлять записями режима работы с ТСОП, связанными с конфигурацией магистральной линии связи, используйте один из приведенных ниже методов.

   - Чтобы выбрать одну или несколько записей из списка всех записей использования PSTN, доступных в Корпоративная голосовая связь развертывания, щелкните **Выберите**. Выделите записи, которые следует связать с этой конфигурацией магистральной линии связи, и нажмите кнопку **ОК**.

   - Чтобы удалить запись режима работы с ТСОП из этой конфигурации магистральной линии связи, выберите эту запись и щелкните **Удалить**.

   - Чтобы определить новую запись режима работы с ТСОП и сопоставить ее с этой конфигурацией магистральной линии связи, выполните следующие действия.

     А. Нажмите кнопку **Создать**.

     Б. В поле **Имя** укажите уникальное описательное имя для записи.

     > [!NOTE]
     > Имя записи режима работы с ТСОП должно быть уникальным в рамках развертывания корпоративной голосовой связи. После сохранения записи поле **Имя** изменить невозможно.

     c. Используйте один из следующих методов для сопоставления и настройки маршрутов для этой записи режима работы с ТСОП:

     - Чтобы выбрать один или несколько маршрутов из списка всех доступных маршрутов в развертывании Корпоративная голосовая связь, щелкните **Выберите**. Выделите маршруты, которые хотите сопоставить с этой записью режима использования ТСОП, и нажмите кнопку **ОК**.

     - Чтобы удалить маршрут из записи режима работы с ТСОП, выберите этот маршрут и щелкните **Удалить**.

     - Чтобы определить новый маршрут и сопоставить его с этой записью режима работы с ТСОП, щелкните **Создать**. Дополнительные сведения см. [в материале Create or modify a voice route in Skype для бизнеса.](create-or-modify-a-voice-route.md)

     - Чтобы изменить маршрут, сопоставленный с этой записью режима работы с ТСОП, выберите маршрут и щелкните **Подробнее**.

     d. Нажмите кнопку **ОК**.

   - Чтобы изменить запись режима работы с ТСОП, уже сопоставленную с этой конфигурацией магистральной линии связи, выполните следующие действия:

     А. Выберите запись режима работы с ТСОП, которую требуется изменить, и щелкните **Подробнее**.

     Б. Используйте один из следующих методов для сопоставления и настройки маршрутов для этой записи режима работы с ТСОП:

     - Чтобы выбрать один или несколько маршрутов из списка всех доступных маршрутов в развертывании Корпоративная голосовая связь, щелкните **Выберите**. Выделите маршруты, которые хотите сопоставить с этой записью режима использования ТСОП, и нажмите кнопку **ОК**.

     - Чтобы удалить маршрут из записи режима работы с ТСОП, выберите этот маршрут и щелкните **Удалить**.

     - Чтобы определить новый маршрут и сопоставить его с этой записью режима работы с ТСОП, щелкните **Создать**. Дополнительные сведения см. [в материале Create or modify a voice route in Skype для бизнеса.](create-or-modify-a-voice-route.md)

     - Чтобы изменить маршрут, сопоставленный с этой записью режима работы с ТСОП, выберите маршрут и щелкните **Подробнее**.

     c. Нажмите кнопку **ОК**.

     > [!IMPORTANT]
     > Важно связать записи использования PSTN в соответствии с одноранговой записью сервера-посредника, связанной с настраиваемой магистралью. Если одноранговой сервер-посредник является шлюзом PSTN или пограничным контроллером сеанса (SBC), настоятельно рекомендуется, чтобы конфигурация магистрали не была связана с записью использования PSTN, которая передается в пункт назначения PSTN или в другие нижестояльные системы, подключенные через Skype для бизнеса Server.

9. Чтобы оптимизировать производительность, рекомендуется упорядочить записи использования ТСОП. Чтобы изменить положение записи в списке, выберите запись использования PSTN и нажмите стрелки вверх или вниз.

    > [!IMPORTANT]
    > Порядок расположения записей режима работы с ТСОП в списке является важным. Skype для бизнеса Server пересекает список сверху вниз.

10. Установите флажок **Разрешить блокирование RTP**, чтобы разрешить обход сервера-посредника для клиентов, находящихся за устройством NAT (преобразование сетевых адресов) или брандмауэром, и пограничного контроллера сеансов, поддерживающего блокирование.

11. **Необходимо выбрать историю вызовов,** чтобы включить отправку сведений об истории вызовов в одноранговой шлюз сервера-посредника.

12. Необходимо выбрать предварительные данные **P-Asserted-Identity,** чтобы в настоящее время информация об источнике вызовов PAI была переадреждена между стороной сервера-посредника и стороной шлюза (и наоборот).

13. Установите флажок **Включить таймер отработки отказа внешней маршрутизации**, чтобы включить быструю отработку отказа. Шлюз, сопоставленный с этой магистралью, может в течение 10 секунд выдать уведомление о том, что он обрабатывает исходящий вызов. Если это уведомление не получено сервером-посредником, будет происходить перенанаружение в другой магистраль. В сетях, где задержка может увеличить время ответа или где на ответ шлюзу требуется более 10 секунд, быструю отработку отказа следует отключить.

14. (Необязательно) Сопоставьте и настройте **правила трансляции вызывающего номера** для магистральной линии связи. Эти правила применяются к вызывающим номерам для исходящих вызовов

    - Чтобы выбрать одно или несколько правил из списка всех правил перевода, доступных в Корпоративная голосовая связь развертывания, нажмите **кнопку Выберите**. В окне **Выбор правил трансляции** выберите правила, которые требуется сопоставить с магистральной линией связи, а затем нажмите кнопку **ОК**.

    - Чтобы определить новое правило преобразования и сопоставить его с магистралью, нажмите кнопку **Создать**. Подробные сведения о правилах перевода [см.](../../plan-your-deployment/enterprise-voice-solution/translation-rules.md)в Skype для бизнеса Server.

    - Чтобы изменить правило преобразования, которое уже сопоставлено с магистралью, щелкните имя правила, затем нажмите кнопку **Подробнее**.

    - Чтобы скопировать существующее правило преобразования и использовать его в качестве отправной точки для определения нового правила, щелкните имя правила и нажмите кнопку **Копировать**, а затем кнопку **Вставить**.

    - Чтобы удалить правило преобразования из магистральной линии связи, выделите имя этого правила, а затем нажмите кнопку **Удалить**.

      > [!CAUTION]
      > Не сопоставляйте правила преобразования с магистралью, если вы настроили правила преобразования на сопоставленном одноранговом узле, поскольку два правила могут конфликтовать.

15. (Необязательно) Сопоставьте и настройте **правила трансляции набранного номера** для магистральной линии связи. Эти правила применяются к вызываемым номерам для исходящих вызовов.

    - Чтобы выбрать одно или несколько правил из списка всех правил перевода, доступных в Корпоративная голосовая связь развертывания, нажмите **кнопку Выберите**. В окне **Выбор правил трансляции** выберите правила, которые требуется сопоставить с магистральной линией связи, а затем нажмите кнопку **ОК**.

    - Чтобы определить новое правило преобразования и сопоставить его с магистралью, нажмите кнопку **Создать**. Подробные сведения о правилах перевода [см.](../../plan-your-deployment/enterprise-voice-solution/translation-rules.md)в Skype для бизнеса Server.

    - Чтобы изменить правило преобразования, которое уже сопоставлено с магистралью, щелкните имя правила, затем нажмите кнопку **Подробнее**.

    - Чтобы скопировать существующее правило преобразования и использовать его в качестве отправной точки для определения нового правила, щелкните имя правила и нажмите кнопку **Копировать**, а затем кнопку **Вставить**.

    - Чтобы удалить правило преобразования из магистральной линии связи, выделите имя этого правила, а затем нажмите кнопку **Удалить**.

      > [!CAUTION]
      > Не сопоставляйте правила преобразования с магистральной линией связи, если вы уже настроили их для связанной одноранговой магистральной линии связи, поскольку эти два правила могут конфликтовать друг с другом.

16. Убедитесь, что правила перевода магистрали расположены в правильном порядке. Чтобы изменить положение правила в списке, выберите имя правила и нажмите стрелку вверх или вниз.

    > [!IMPORTANT]
    > Skype для бизнеса Server перебирает список правил перевода сверху вниз и использует первое правило, которое соответствует набраному номеру. Если вы настраиваете магистраль таким образом, чтобы набираемый номер мог соответствовать нескольким правилам преобразования, убедитесь, что более строгие правила расположены над менее строгими. Например, если вы включили правило перевода, которое совпадает с любым 11-значным номером и правилом перевода, которое соответствует только 11-значным числам, которые начинаются  с +1425, убедитесь, что правило, которое совпадает с любым 11-значным номером, сортироваться ниже более строгого правила.

17. Закончив настройку магистрали, нажмите кнопку **ОК**.

18. На странице **Конфигурация магистрали** нажмите кнопку **Передать**, а затем щелкните **Передать все**.

    > [!NOTE]
    > При каждом создании или изменении конфигурации магистрали необходимо выполнить команду **Сохранить все**, чтобы опубликовать изменение конфигурации. Подробные сведения см. в [публикации "Публикация в](voice-route-config-changes.md) ожидании изменений конфигурации маршрутирования голосовой почты в Skype для бизнеса документации по операциям".
