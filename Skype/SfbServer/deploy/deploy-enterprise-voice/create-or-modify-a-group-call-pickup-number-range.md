# <a name="create-or-modify-a-group-call-pickup-number-range-in-skype-for-business"></a>Создание или изменение диапазона номеров для группового звонка в Skype для бизнеса

Create or modify a Group Call Pickup number range in Skype for Business Server Корпоративная голосовая связь.

Групповой прием вызовов основан на приложении парковки вызовов. При развертывании группового вызова необходимо настроить таблицу орбит парковки вызовов с диапазонами телефонных номеров, назначенных в качестве номеров группы. Эти номера групп — это номера, которые пользователи набирают для звонков, которые звонят другому пользователю.

Как и номера орбит парковки вызовов, номера группы звонков должны быть виртуальными расширениями, для которых не назначен ни пользователь, ни телефон. Каждый пул переднего ряда, в котором развертывается групповой звонок, может иметь один или несколько диапазонов номеров групп. Диапазоны номеров групп должны быть глобально уникальными в развертывании и должны быть назначены в качестве типа **GroupPickup.**

Используйте следующую процедуру для создания или изменения диапазона номеров группы разных вызовов в таблице орбит парковки вызовов.

> [!NOTE]
> Для создания, изменения, удаления и просмотра диапазонов номеров группового звонка в таблице орбит парковки вызовов необходимо использовать skype for Business Server Management Shell. Диапазоны номеров группового звонка недоступны в панели управления Skype для бизнеса Server.

Диапазоны номеров группы разозвать вызовов должны соответствовать следующим правилам:

- Начальный номер диапазона должен быть меньше или равен конечному номеру диапазона.

- Значение начального номера диапазона должно иметь такую же длину, что и конечный номер диапазона.

- Диапазон номеров должен быть уникальным. Он не может накладываться на любой другой диапазон.

- Если диапазон номеров начинается с символа #, он должен быть больше \* 100.

- Допустимые значения: должны соответствовать строке регулярного выражения ([ \\ *|#]?[ 1–9]\d {0,7} )| ([1–9]\d {0,8} ). Это означает, что значение должно быть строкой, начинаемой с символа или #, или числом от 1 до 9 (первый символ не может \* быть нулем). Если первый символ является или #, следующий символ должен быть \* числом от 1 до 9 (не может быть нулем). Последующие символы могут быть любым числом от 0 до 9 до семи дополнительных символов (например, "#6000", \* "92000", \* "95551212" и "915551212"). Если первый символ не является или #, первый символ должен быть числом от 1 до 9 (не может быть нулем), за которым следует до восьми символов, каждый из которых имеет значение от 0 до \* 9 (например, "915551212", "41212", "300").

### <a name="to-create-or-modify-a-call-pickup-group-range"></a>Создание или изменение диапазона группы разных вызовов

1. Войдите на компьютер, на котором установлена оболочка управления Skype для бизнеса Server, в качестве члена группы RTCUniversalServerAdmins или с необходимыми правами пользователя, как описано в делегирования разрешений на **установку.**

2. Запустите оболочку управления Skype для бизнеса Server: нажмите кнопку "Начните", выберите "Все программы", "Skype для бизнеса **2015",** а затем щелкните "Skype для бизнеса Server Management **Shell".**

3. Используйте **New-CsCallParkOrbit** для создания нового диапазона номеров группы разбора вызовов. Используйте **Set-CsCallParkOrbit** для изменения существующего диапазона номеров для разбора вызовов.

    В командной строке выполните следующую команду:

   ```powershell
   New-CsCallParkOrbit -Identity <name of call pickup group range> -NumberRangeStart <first number in range> -NumberRangeEnd <last number in range> -CallParkService <FQDN or service ID of the Application service that hosts the Call Park application> -Type GroupPickup
   ```

    Пример:

   ```powershell
   New-CsCallParkOrbit -Identity "Redmond call pickup" -NumberRangeStart 100 -NumberRangeEnd 199 -CallParkService redmond-applicationserver-1 -Type GroupPickup
   ```

    В следующем примере показано, как изменить диапазон номеров с орбит парковки вызовов на группы разных групп.

   ```powershell
   Set-CsCallParkOrbit -Identity "Redmond call pickup" -Type GroupPickup
   ```

    > [!IMPORTANT]
    > Используйте этот cmdlet для изменения типа, назначенного диапазонам номеров, только если вы изначально указали неправильный тип и диапазон группы еще не используется. Если изменить диапазон номеров с CallPark на GroupPickup или наоборот, а диапазон номеров уже используется, парковка вызовов или групповой звонок перестанут работать для этого диапазона номеров. Например, если изменить диапазон номеров с CallPark на GroupPick, приложение парковки вызовов больше не сможет использовать этот диапазон орбит для парковки вызовов.
