# <a name="upgrade-to-skype-for-business-server-2015"></a>Обновление до Skype для бизнеса Server 2015
 
**Сводка:** Узнайте, как обновить Lync Server 2013 до Skype для бизнеса Server 2015. 
  
Используйте процедуры, описанные в этом документе, чтобы обновить Lync Server 2013 до Skype для бизнеса Server 2015 с помощью Skype для бизнеса Server построитель топологий и новой функции In-Place обновления. Если вы хотите выполнить обновление с Lync Server 2010 или Office Communications Server 2007 R2, см. план обновления до [Skype для бизнеса Server 2015](../plan-your-deployment/upgrade.md).

> [!NOTE]
> Обновления на месте были доступны в Skype для бизнеса Server 2015, но больше не поддерживаются Skype для бизнеса Server 2019. Параллельное сосуществование поддерживается. Дополнительные сведения см. в Skype для бизнеса Server [2019](../../SfBServer2019/migration/migration-to-skype-for-business-server-2019.md).
  
## <a name="upgrade-from-lync-server-2013"></a>Обновление с Lync Server 2013

Обновление Lync Server 2013 до Skype для бизнеса Server 2015 включает установку необходимого программного обеспечения, использование Skype для бизнеса Server построитель топологий для обновления баз данных в пуле и использование Skype для бизнеса Server  In-Place на каждом сервере, связанном с пулом. Чтобы завершить обновление, выполните восемь действий, описанных в этом разделе.
  
### <a name="before-you-begin"></a>Прежде чем начать

- [Просмотрите план обновления до Skype для бизнеса Server 2015](../plan-your-deployment/upgrade.md).
    
- [Ознакомьтесь с требованиями к серверу для Skype для бизнеса Server 2015.](../plan-your-deployment/requirements-for-your-environment/server-requirements.md)
    
- [Установите необходимые компоненты для Skype для бизнеса Server 2015](install/install-prerequisites.md).
    
- [Установите Skype для бизнеса Server 2015](install/install.md).
    
### <a name="step-1-install-administrator-tools-and-download-topology"></a>Шаг 1. Установка средств администратора и скачивание топологии

1. Подключение на компьютер в топологии, на котором не установлено Lync OCSCore или другие компоненты Lync.
    
2. На Skype для бизнеса Server 2015 запустите **Setup.exeиз** **OCS_Volume\Setup\AMD64**. 
    
3. Нажать кнопку **Установить**. 
    
4. Принятие условий лицензионного соглашения.
    
5. В мастере развертывания щелкните **"Установить средства** администратора" и выполните действия по установке.
    
     ![Снимок экрана мастера развертывания со ссылкой на выделенные средства установки администратора.](../media/5bbac2d6-a5b3-42b4-a243-7bcf2b04477a.png)
  
6. В Windows начальный экран откройте Skype для бизнеса Server построитель топологий.
    
7. **Щелкните "Скачать топологию" из существующего развертывания** и нажмите кнопку **"Далее"**.
    
8. Введите имя топологии и нажмите кнопку " **Сохранить"**.
    
9. Перейдите к расположению, в котором сохранена топология, и сделайте копию топологии.
    
### <a name="step-2-upgrade-and-publish-topology-using-topology-builder"></a>Шаг 2. Обновление и публикация топологии с помощью построитель топологий

Перед началом процесса обновления все службы должны быть запущены для пулов, которые планируется обновить. Это позволяет реплицировать изменения топологии в локальную базу данных серверов в пуле.
  
> [!IMPORTANT]
>  Перед обновлением сохраните копию файла топологии. После обновления вы не сможете понизить уровень топологии.> Если службы находятся на тех же серверах, что и базы данных, например служба сохраняемого чата, находится на том же сервере, что и база данных сохраняемого чата, пропустите этот шаг и перейдите к шагу 4. После остановки служб запустите программу In-Place обновления на каждом сервере, чтобы обновить локальные базы данных.
  
> [!NOTE]
> Если в топологии есть зеркальная серверная база данных, то при публикации топологии с помощью построитель топологий будут отображаться как основной, так и  зеркальный базы данных. Убедитесь, что все базы данных запущены в субъекте, и при публикации топологии выберите только субъект, а не зеркальное отображение. В противном случае после публикации топологии появится предупреждение.
  
Выберите один из следующих вариантов обновления и публикации новой топологии с помощью Skype для бизнеса Server 2015 построитель топологий. После выполнения действий и публикации обновленной топологии перейдите к шагу 3 в этом разделе.
  
#### <a name="option-1-upgrade-an-isolated-front-end-pool-and-associated-archiving-and-monitoring-stores"></a>Вариант 1. Обновление изолированного пула переднего плана и связанных хранилищ архивации и мониторинга

Если обновляемый пул имеет зависимость хранилища архивации и мониторинга, при выполнении следующих действий также будет обновлено хранилище архивации и мониторинга.
  
1. В построитель топологий щелкните правой кнопкой мыши пул Lync Server 2013, выберите "**Обновить до Skype для бизнеса Server 2015**" и выполните действия. 
    
     ![Снимок экрана: контекстное меню с параметром обновления для Lync Server 2013.](../media/7d5b25b1-e5c0-474c-a024-a5ba33f1b3a1.png)
  
2. В построитель топологий выберите топологию  >  публикации **действия или** **публикацию** > **топологии действия** > . 
    
     ![Снимок экрана: меню "Действие" с параметром "Опубликовать топологию" в построитель топологий.](../media/d6712634-9205-401f-a0b0-3ea096ca51bf.png)
  
3. Во время публикации выберите установку базы данных в хранилище архивации и мониторинга.
    
#### <a name="option-2-upgrade-front-end-pool-without-upgrading-archiving-and-monitoring-stores"></a>Вариант 2. Обновление пула переднего плана без обновления хранилищ архивации и мониторинга

Если вы выполните следующие действия, архивация и мониторинг для выбранного пула будут отключены. После обновления в пуле не будут храниться хранилища архивации и мониторинга.
  
1. В построитель топологий выберите пул Lync Server 2013, который требуется обновить.
    
2. Удалите зависимость от хранилищ архивации и мониторинга Lync Server 2013. 
    
   - Перейдите **к свойству "** > **Изменение действия"**.
    
   - Снимите **флажок архивации** .
    
     ![Снимок экрана: флажок "Архивация" в диалоговом окне "Изменение свойств".](../media/9a88427e-80ee-49d0-a767-809fa9a5faf1.png)
  
   - Снимите **флажок "** Мониторинг".
    
     ![Снимок экрана: диалоговое окно "Изменение свойств", в котором отображается флажок "Мониторинг".](../media/880acf33-57bb-4521-8717-cf5b67261ed4.png)
  
3. Щелкните правой кнопкой мыши пул Lync Server 2013, выберите "Обновить до **Skype для бизнеса Server 2015**" и выполните указанные ниже действия. 
    
     ![Снимок экрана: контекстное меню с параметром обновления для Lync Server 2013.](../media/7d5b25b1-e5c0-474c-a024-a5ba33f1b3a1.png)
  
4. В построитель топологий выберите топологию  >  публикации **действия или** **публикацию** > **топологии действия** > . 
    
#### <a name="option-3-upgrade-front-end-pool-and-associated-it-to-new-skype-for-business-server-2015-archiving-and-monitoring-stores"></a>Вариант 3. Обновление пула переднего плана и его сопоставление с новыми хранилищами архивации и мониторинга Skype для бизнеса Server 2015 г.

Если вы выполните следующие действия, архивация и мониторинг остановится в предыдущем хранилище и запустится в новом хранилище, которое вы создали. 
  
1. В построитель топологий выберите пул Lync Server 2013, который требуется обновить. 
    
2. Удалите зависимость от хранилищ архивации и мониторинга Lync Server 2013. 
    
   - Перейдите **к свойству "** > **Изменение действия"**.
    
   - Снимите **флажок архивации** .
    
     ![Снимок экрана: флажок "Архивация" в диалоговом окне "Изменение свойств".](../media/9a88427e-80ee-49d0-a767-809fa9a5faf1.png)
  
   - Снимите **флажок "** Мониторинг".
    
     ![Снимок экрана: диалоговое окно "Изменение свойств", в котором отображается флажок "Мониторинг".](../media/880acf33-57bb-4521-8717-cf5b67261ed4.png)
  
3. Щелкните правой кнопкой мыши пул Lync Server 2013, выберите "Обновить до **Skype для бизнеса Server 2015**" и выполните указанные ниже действия.

     ![Снимок экрана: диалоговое окно "Изменение свойств" с кнопкой "Создать" в разделе "Архивация".](../media/3a4a18e7-8251-4736-837c-2b486f64f896.png)

### <a name="step-3-wait-for-replication"></a>Шаг 3. Ожидание репликации

Дайте репликации некоторое время для публикации обновленной топологии на всех серверах в среде.
  
### <a name="step-4-stop-all-services-in-pool-to-be-upgraded"></a>Шаг 4. Остановка всех служб в пуле для обновления

На каждом сервере, обслуживаемом пулом, который вы собираетесь обновить, выполните следующий командлет в PowerShell:
  
```powershell
Disable-CsComputer -Scorch
```

Рекомендуется использовать Disable-CsComputer, так как может потребоваться перезагрузка сервера во время In-Place обновления. Если вы используете Stop-CsWindowsService, некоторые службы могут автоматически перезапуститься после перезагрузки. Это может привести к In-Place обновления.
  
### <a name="step-5-upgrade-front-end-pools-and-non-front-end-pool-servers"></a>Шаг 5. Обновление пулов переднего плана и серверов, отличных от серверов пула переднего плана

> [!NOTE]
>  Перед обновлением установите все необходимые компоненты, необходимые для Skype для бизнеса Server 2015 г., включая:> не менее 32 ГБ свободного пространства перед попыткой обновления. Кроме того, убедитесь, что диск является фиксированным локальным диском, не подключен usb или Firewire. форматируется с помощью файловой системы NTFS, не сжимается и не содержит файла страницы.> PowerShell версии 6.2.9200.0 или более поздней.> Последнее установленное накопительное обновление Lync Server 2013.> SQL Server 2012 с пакетом обновления 1 (SP1) установлено.> Установленная ниже статья базы знаний (устанавливается автоматически при использовании Центра обновления Майкрософт): > Windows Server 2008 R2 -[KB2533623](https://support.microsoft.com/kb/2533623)> Windows Server 2012 -[KB2858668](https://support.microsoft.com/kb/2858668)> Windows Server 2012 R2 -[KB2982006](https://support.microsoft.com/kb/2982006)
  
Используйте In-Place обновления на каждом сервере, чтобы обновить пул переднего плана, пограничный пул, сервер-посредник и пул сохраняемого чата.
  
1. На каждом сервере запустите **Setup.exe** **из OCS_Volume\Setup\amd64** на Skype для бизнеса Server 2015.
    
2. Примите лицензионное соглашение и следуйте инструкциям по обновлению In-Place обновления.
    
3. Повторите эти действия для каждого сервера в пуле переднего плана и на каждом сервере, отличном от сервера пула переднего плана.
    
> [!NOTE]
> Возможно, вам будет предложено перезагрузить сервер во время In-Place обновления. Порядок. После перезагрузки обновление In-Place продолжится с того места, где оно было отключено. 
  
После In-Place обновления вы увидите следующее сообщение.
  
![Снимок экрана, на котором показано успешное обновление на месте.](../media/2f52cb50-9bb4-4714-a982-9c7a0865f78a.png)
  
### <a name="step-6-restart-services-on-all-upgraded-servers"></a>Шаг 6. Перезапуск служб на всех обновленных серверах

> [!NOTE]
> Перед перезапуском служб убедитесь, что %ProgramData%\WindowsFabric не существует на всех интерфейсных серверах. Если он существует, удалите его перед запуском служб. 
  
- После обновления всех серверов в пуле переднего плана перезапустите службы с помощью следующей команды PowerShell: 
    
  ```powershell
  Start-CsPool
  ```

    > [!NOTE]
    > Если перед запуском обновления In-Place требуется ожидающая перезагрузка системы, In-Place обновление не будет запрашивать перезагрузку в конце установки. Это приведет к возникновению некоторых исключений сборки на первом сервере переднего плана при попытке запустить службы с помощью Start-CSPool командлета. Чтобы устранить эти ошибки, перезагрузите все серверы в пуле и снова выполните командлет. 
  
- Перезапустите службы на серверах, отличных от серверов пула переднего плана, с помощью следующей команды:
    
  ```powershell
  Start-CsWindowsService
  ```

После нажатия **кнопки "** ОК" на странице In-Place обновления вы увидите следующее напоминание, чтобы завершить этот шаг.
  
![Снимок экрана, на котором показаны дальнейшие действия после успешного завершения обновления на месте.](../media/6a7236b6-9ef9-4df3-8682-b0e4021810f9.png)
  
### <a name="step-7-verify-skype-for-business-functionality-works"></a>Шаг 7. Проверка Skype для бизнеса функциональных возможностей

Чтобы убедиться, что обновление прошло успешно, протестируйте пул, Skype для бизнеса, чтобы убедиться, что функциональность работает должным образом.
