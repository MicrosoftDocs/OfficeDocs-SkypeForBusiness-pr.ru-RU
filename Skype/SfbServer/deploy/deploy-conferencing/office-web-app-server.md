# <a name="configure-integration-with-office-web-apps-server-in-skype-for-business-server"></a>Настройка интеграции с office Web Apps Server в Skype для бизнеса Server
 
**Сводка:** Ознакомьтесь с этой темой, чтобы узнать, как настроить интеграцию между Office Web Apps Server и Skype для бизнес-сервера, чтобы включить презентации PowerPoint для веб-конференциации.
  
Skype для бизнеса Server использует Office Web Apps Server для обработки презентаций PowerPoint для веб-конференций. Сведения о преимуществах этого подхода см. в веб-сведениях [Plan for conferencing in Skype for Business Server.](../../plan-your-deployment/conferencing/conferencing.md)
  
Прежде чем настроить Skype для бизнес-сервера для использования Office Web Apps Server, необходимо убедиться, что Office Web Apps Server уже развернут и настроен. Сведения о office Web Apps Server см. в статье [Развертывание инфраструктуры Office Online Server.](/webappsserver/deploy-the-infrastructure-office-web-apps-server) 
  
После успешной установки Office Web Apps Server и правильной настройки веб-фермы необходимо настроить Skype для бизнеса Server для связи с новым сервером, добавив URL-адрес обнаружения Office Web Apps Server в топологию Skype для бизнеса Server. 
  
> [!NOTE]
> Последняя итерация Office Web Apps Server называется Office Online Server, которая поддерживается Skype для бизнеса Server. Дополнительные подробности можно найти в [документации Office Online Server.](/officeonlineserver/office-online-server) 
  
## <a name="configure-skype-for-business-server-to-communicate-with-office-web-apps-server"></a>Настройка Skype для бизнес-сервера для связи с сервером Office Web Apps

Чтобы добавить сервер Office Web Apps в топологию, выполните следующие действия:
  
1. Откройте Skype для строителя топологии бизнес-серверов.
    
2. В диалоговом окне **Построитель топологии** выберите **Загрузить топологию из существующего развертывания**, а затем щелкните **ОК**.
    
3. В диалоговом окне **Сохранить топологию как** введите имя документа топологии (например, **PreWebAppsServerTopology**) в поле **Имя файла**, а затем щелкните **Сохранить**. Затем можно будет восстановить и повторно опубликовать эту топологию в случае возникновения проблем с новой топологией.
    
4. В Topology Builder разойдите **Skype для** бизнес-сервера, развяжите имя своего сайта, расширите пулы переднего конца enterprise **Edition,** щелкните правой кнопкой мыши имя одного из пулов, а затем нажмите кнопку **Изменить свойства**.
    
5. В диалоговом окне **Изменить свойства** на вкладке **Общие** найдите заголовок **Связать сервер Office Web Apps**, а затем щелкните **Создать** (или выберите существующий сервер Office Web Apps в раскрывающемся списке).
    
6. В диалоговом окне **Указать новый сервер Office Web Apps** введите полное доменное имя (FQDN) сервера Office Web Apps в поле **Полное доменное имя сервера Office Web Apps**; при этом URL-адрес обнаружения сервера Office Web Apps должен автоматически отобразиться в поле **URL-адрес обнаружения сервера Office Web Apps**.
    
   - Если сервер Office Web Apps установлен локально и находится в той же сетевой зоне, что и Skype для бизнеса Server, то параметр Office Web Apps Server развертывается во внешней сети (то есть периметре **или Интернете)** не следует выбирать.
    
   - Если сервер Office Web Apps развернут за пределами внутреннего брандмауэра, выберите параметр **Сервер Office Web Apps развернут во внешней сети (то есть в периметре/Интернете)**.
    
7. В диалоговом окне **Указать новый сервер Office Web Apps** щелкните **ОК**, а затем щелкните **ОК** в диалоговом окне **Изменить свойства**. Url-адрес обнаружения будет указан в качестве одной из ассоциаций пула.
    
Затем необходимо повторить этот процесс для каждого пула, который необходимо связать с сервером Office Web Apps.
  
После добавленного URL-адреса обнаружения в топологию необходимо опубликовать обновленную топологию. Для этого выполните следующие действия в построителе топологии:
  
1. Щелкните **Действие**, а затем щелкните **Опубликовать топологию**.
    
2. В мастере публикации топологии на странице **Опубликовать топологию** щелкните **Далее**.
    
3. На странице **Завершение мастера публикации** щелкните **Готово**.
    
4. Закройте построитель топологии.
    
## <a name="configure-access-for-external-users"></a>Настройка доступа для внешних пользователей

Если вы хотите, чтобы внешние пользователи (то есть пользователи, входив из-за брандмауэра организации) получили доступ к презентациям Office Web Apps Server PowerPoint, вам потребуется использовать Office Web Apps Server и обратный прокси-сервер. Вам также потребуется создать и настроить правило публикации веб-сайта, которое поможет убедиться, что пользователи могут подключаться к серверу. 
  
## <a name="validate-the-configuration"></a>Проверка конфигурации

После того как Office Web Apps Server был добавлен в топологию и после публикации топологии, в журнале событий Skype для бизнес-сервера должны быть добавлены два новых события журнала событий. Сначала необходимо добавить событие LS Data MCU (event ID 41034). в этом событии будет отчет об обнаружении сервера веб-приложений Office:
  
 **Обнаружен сервер веб-конференциинга Server Office Web Apps Server, включен контент PowerPoint.**
  
Кроме того, должно отображаться другое событие LS Data MCU (идентификатор события — 41032), которое отправляют URL-адреса сервера Office Web Apps. Это событие может иметь, например, следующий вид:
  
 **Веб-conferencing Server Office Web Apps Server discovery has succeeded.**
  
 **Страница внутреннего презента Office Web Apps Server: https://atl-officewebapps-001.litwareinc.com/m/Presenter.aspx?a=0&amp ;embed=**
  
 **Внутренняя страница участника Office Web Apps Server: https://atl-officewebapps-001.litwareinc.com/m/ParticipantFrame.aspx?a=0&amp ;embed=true&amp;=**
  
Если у вас есть настроенный доступ для внешних пользователей, вы также увидите что-то похожее на:
  
 **Страница внешнего презентера Office Web Apps Server: https://atl-officewebapps-001.litwareinc.com/m/Presenter.aspx?a=0&amp ;embed**
  
 **Внутренняя страница участника Office Web Apps Server: <https://atl-officewebapps-001.litwareinc.com/m/ParticipantFrame.aspx?a=0&amp;embed=true&amp> ;**
  
Если событие LS Data MCU отображается с идентификатором события 41033, это означает, что произошел сбой обнаружения сервера Office Web Apps Server. В этом случае Skype для бизнеса Server будет пытаться столько раз, сколько необходимо, чтобы обнаружить недавно настроенный сервер веб-приложений Office. Если происходит многократный сбой процесса обнаружения, следует удалить сервер Office Web Apps из документа топологии, опубликовать обновленную топологию, а затем снова попробовать добавить сервер Office Web Apps в топологию после устранения неполадок, связанных с подключением.
  
Если сервер Office Web Apps Server, как представляется, настроен правильно и распознается в процессе обнаружения, можно убедиться, что Office Web Apps Server работает так, как ожидалось, разделив презентацию PowerPoint между парой клиентов Skype для бизнеса. Если пользователь А может загружать и отображать презентацию PowerPoint, а пользователь Б затем может присоединяться к собранию и просматривать презентацию, сервер Office Web Apps функционирует правильно.
  
Даже если сервер Office Web Apps Server настроен правильно, вы можете получить сообщение об ошибке "Некоторые функции общего доступа недоступны из-за проблем с подключением к серверу", когда вы пытаетесь поделиться презентацией PowerPoint. В случае отображения этого сообщения об ошибке следует перезапустить серверы переднего плана, сопоставленные с новым сервером Office Web Apps.
