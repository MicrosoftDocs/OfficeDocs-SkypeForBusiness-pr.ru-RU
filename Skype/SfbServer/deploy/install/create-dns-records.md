# <a name="create-dns-records-for-skype-for-business-server"></a>Создание записей DNS для Skype для бизнеса Server
 
**Сводка:** Узнайте, как настроить DNS и создать записи DNS для установки Skype для бизнеса Server.
  
Чтобы Skype для бизнеса Server правильно работать, необходимо настроить ряд параметров системы доменных имен (DNS). Это необходимо для того, чтобы клиенты знали, как получить доступ к службам и что серверы знают друг о друге. Эти параметры необходимо заполнять только один раз для каждого развертывания, так как после назначения записи DNS она будет доступна в домене. Вы можете выполнить шаги с 1 по 5 в любом порядке. Однако необходимо выполнить шаги 6, 7 и 8 по порядку и после шагов 1–5, как показано на схеме. Создание записей DNS состоит из шага 5 из 8. Дополнительные сведения о планировании DNS см. в разделе "Требования к среде [Skype для бизнеса Server](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или требования к серверу для Skype для бизнеса Server [2019](../../../SfBServer2019/plan/system-requirements.md) г.
  
> [!IMPORTANT]
> Важно отметить, что это лишь пример создания записей DNS в среде DNS Windows server. Существует множество других записей DNS, необходимых для Skype для бизнеса Server, и процедура создания записей DNS зависит от системы, которую вы используете для управления DNS в организации. Полный список требований к DNS см. в статье о [требованиях DNS для Skype для бизнеса Server](../../plan-your-deployment/network-requirements/dns.md). 
  
![Обзорная схема.](../../media/d2fc733c-6a80-4d17-a02f-93b8c4bfb999.png)

### <a name="create-dns-records"></a>Создание записей DNS

1. Войдите на DNS-сервер и откройте **диспетчер сервера**.
    
2. Щелкните **раскрывающееся** меню "Сервис" и выберите **DNS**.
    
3. В дереве консоли для домена SIP разверните зоны прямого **просмотра, а** затем разверните домен SIP, Skype для бизнеса Server будет установлен.
    
4. Щелкните правой кнопкой мыши домен SIP и выберите новый хост **(A или AAAA),** как показано на рисунке.
    
     ![выбор новой записи A.](../../media/f89c5c1f-b5b7-428c-a6e3-2bcd12e878c3.png)
  
5. В поле **"Имя** " введите имя записи узла (доменное имя будет автоматически добавлено).
    
6. В поле **IP-адреса** введите IP-адрес отдельного интерфейсного сервера, а затем выберите "Создать связанную запись указателя **(PTR)** или разрешить любому прошедшему проверку подлинности пользователю обновлять записи **DNS** с тем же именем владельца, если это применимо. Обратите внимание, что предполагается, что DNS используется для балансировки нагрузки всего трафика, за исключением веб-служб. В этом примере у нас есть три интерфейсных сервера, как показано в таблице.
    
   |**Имя сервера**|**Type (Тип)**|**Data (Данные)**|
   |:-----|:-----|:-----|
   |SFB01  <br/> |Узел (A)  <br/> |10.0.0.5  <br/> |
   |SFB02  <br/> |Узел (A)  <br/> |10.0.0.6  <br/> |
   |SFB03  <br/> |Узел (A)  <br/> |10.0.0.7  <br/> |
   
7. Затем создайте записи балансировки нагрузки DNS для пула. Балансировка нагрузки DNS позволяет DNS отправлять запросы на отдельные серверы в пуле, используя одно и то же имя пула DNS. Дополнительные сведения о DNS и балансировке нагрузки см. в [требованиях DNS для Skype для бизнеса Server](../../plan-your-deployment/network-requirements/dns.md). 
    
    > [!NOTE]
    > Объединить несколько серверов в пул можно только в выпуск Enterprise развертываниях. При развертывании одного сервера Enterprise или выпуск Standard необходимо создать только запись A для одного сервера. 
  
    Например, если у вас есть пул с именем pool.contoso.local и три интерфейсных сервера, необходимо создать следующие записи DNS:
    
   |**FQDN**|**Type (Тип)**|**Data (Данные)**|
   |:-----|:-----|:-----|
   |pool.contoso.local  <br/> |Узел (A)  <br/> |10.0.0.5  <br/> |
   |pool.contoso.local  <br/> |Узел (A)  <br/> |10.0.0.6  <br/> |
   |pool.contoso.local  <br/> |Узел (A)  <br/> |10.0.0.7  <br/> |
   
8. Продолжайте создавать записи A для всех серверов в запланированном развертывании. 
    
9. Чтобы создать запись службы (SRV) для устаревшего обнаружения, щелкните правой кнопкой мыши домен SIP и выберите **другие новые записи**.
    
10. В разделе **Выбор типа записи ресурса** выберите **Service Location (SRV) (Расположение службы (запись SRV))**, а затем нажмите кнопку **Создать запись**.
    
11. Щелкните элемент **Служба** и введите **_sipinternaltls**.
    
12. Нажмите пункт **Протокол** и введите **_tcp**.
    
13. Нажмите **Номер порта** и введите **5061**.
    
14. **Щелкните host, предлагая** эту службу, а затем введите полное доменное имя пула или выпуск Standard сервера.
    
     ![Снимок экрана: диалоговое окно создания записи ресурса.](../../media/54b1aac5-a2ec-41fe-90c0-02eaeaa9d1b4.png)
  
15. Нажмите кнопку **ОК**, а затем нажмите кнопку **Готово**.
    
### <a name="verify-dns-records"></a>Проверка записей DNS

1. Выполните вход на клиентский компьютер с использованием учетной записи, которая является членом группы прошедших проверку пользователей или имеет эквивалентные разрешения.
    
2. Нажмите **кнопку "** Пуск", а затем **введите командную строку** и нажмите клавишу ВВОД.
    
3. **Введите nslookup \<FQDN of the Front End pool\>** или **\<FQDN of the Standard Edition server or single Enterprise Edition server\>** нажмите клавишу ВВОД.
    
4. Продолжайте проверять остальные записи A для развертывания.
    
5. Если вы поддерживаете устаревшие клиенты и создали запись SRV, проверьте ее, введя set **type=srv** в командной строке **nslookup** , а затем нажмите клавишу ВВОД.
    
6. **Введите _sipinternaltls._tcp. *Домена*** (например, _sipinternaltls._tcp.contoso.local), а затем нажмите клавишу ВВОД.
    
7. Ожидаемые выходные данные должны быть похожи на выходные данные, показанные на рисунке. Обратите внимание, что не все записи DNS отображаются в примере выходных данных, но все записи должны быть проверены. 
    
     ![Проверьте параметры DNS.](../../media/4fcaa70f-7717-4939-9652-d2048d6293cc.png)
