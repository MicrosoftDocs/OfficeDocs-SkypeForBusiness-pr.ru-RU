# <a name="create-dns-records-for-skype-for-business-server"></a>Создание записей DNS для Skype для бизнеса Server
 
**Сводка:** Узнайте, как настроить DNS и создать записи DNS для установки Skype для бизнеса Server. Скачайте бесплатную пробную версия Skype для бизнеса Server в Центре оценки Майкрософт по [https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server](https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server) ссылке:
  
Для правильной работы Skype для бизнеса Server необходимо установить ряд параметров системы доменных имен (DNS). Это необходимо для того, чтобы клиенты знали, как получить доступ к службам, и что серверы знают друг о друге. Эти параметры должны быть выполнены только один раз для каждого развертывания, так как после назначения записи DNS она будет доступна во всем домене. Шаги 1-5 можно делать в любом порядке. Однако шаги 6, 7 и 8 необходимо делать по порядку и после шагов с 1 по 5, как описано на схеме. Создание записей DNS состоит из шага 5 из 8. Дополнительные сведения о планировании DNS см. в требованиях к среде [для Skype для](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) бизнеса Server или Сервера для Skype для бизнеса Server [2019.](../../../SfBServer2019/plan/system-requirements.md)
  
> [!IMPORTANT]
> Важно отметить, что это лишь пример создания записей DNS в среде DNS Windows Server. Существует множество других записей DNS, необходимых для Skype для бизнеса Server, и процедура создания записей DNS зависит от системы, используемой для управления DNS в организации. Полный список требований для DNS см. в [DNS-требованиях для Skype для бизнеса Server.](../../plan-your-deployment/network-requirements/dns.md) 
  
![Обзорная схема](../../media/d2fc733c-6a80-4d17-a02f-93b8c4bfb999.png)
  
## <a name="configure-dns"></a>Настройка DNS

Сервер переднего сервера Standard Edition может содержать только один сервер. При использовании Standard Edition полное доменное имя (FQDN) отдельного сервера Standard Edition используется только при ссылке на роль переднего плана вместо создания пула серверов с балансой нагрузки на DNS, как показано в этом примере. В этом простом примере, использующем только роль переднего плана, содержатся записи DNS в следующей таблице. ### <a name="create-dns-records"></a>Создание записей DNS

1. Войдите на DNS-сервер и откройте **диспетчер серверов.**
    
2. Выберите в **меню "Инструменты"** пункт **"DNS".**
    
3. В дереве консоли для домена SIP развяжете зоны forward Lookup (Зоны forward **Lookup)** и затем развяжете домен SIP, в котором будет установлен Skype для бизнеса Server.
    
4. Щелкните правой кнопкой мыши домен SIP и выберите новый хост **(A или AAAA),** как показано на рисунке.
    
     ![выбор новой записи A](../../media/f89c5c1f-b5b7-428c-a6e3-2bcd12e878c3.png)
  
5. В поле **"Имя"** введите имя записи хоста (доменное имя будет автоматически примеся).
    
6. В поле **IP-адреса** введите IP-адрес отдельного сервера переднего интерфейса, а затем выберите "Создать связанную запись указателя **(PTR)** или разрешить любому пользователю, подлинность которого была выбрана, обновить **записи DNS** с тем же именем владельца, если применимо. Обратите внимание, что предполагается, что DNS используется для балансивной нагрузки всего трафика за исключением веб-служб. В этом примере у нас есть три сервера переднего сервера, как показано в таблице.
    
   |**Имя сервера**|**Type (Тип)**|**Data (Данные)**|
   |:-----|:-----|:-----|
   |SFB01  <br/> |Узел (A)  <br/> |10.0.0.5  <br/> |
   |SFB02  <br/> |Узел (A)  <br/> |10.0.0.6  <br/> |
   |SFB03  <br/> |Узел (A)  <br/> |10.0.0.7  <br/> |
   
7. Затем создайте записи балансировки нагрузки на DNS для пула. Балансировка нагрузки на DNS позволяет DNS отправлять запросы на отдельные серверы в пуле, используя одно и то же имя пула DNS. Дополнительные сведения о DNS и балансировку нагрузки см. в требованиях [К DNS для Skype для бизнеса Server.](../../plan-your-deployment/network-requirements/dns.md) 
    
    > [!NOTE]
    > Объединение нескольких серверов в пул доступно только в развертываниях Enterprise Edition. При развертывании одного сервера Enterprise Server или Standard Edition необходимо создать только запись A для одного сервера. 
  
    Например, если у вас есть пул с именем pool.contoso.local и три сервера переднего сервера, необходимо создать следующие записи DNS:
    
   |**FQDN**|**Type (Тип)**|**Data (Данные)**|
   |:-----|:-----|:-----|
   |pool.contoso.local  <br/> |Узел (A)  <br/> |10.0.0.5  <br/> |
   |pool.contoso.local  <br/> |Узел (A)  <br/> |10.0.0.6  <br/> |
   |pool.contoso.local  <br/> |Узел (A)  <br/> |10.0.0.7  <br/> |
   
8. Продолжите создание записей A для всех серверов в запланированном развертывании. 
    
9. Чтобы создать запись SRV для устаревшего обнаружения, щелкните правой кнопкой мыши домен SIP и выберите другие новые **записи.**
    
10. В разделе **Выбор типа записи ресурса** выберите **Service Location (SRV) (Расположение службы (запись SRV))**, а затем нажмите кнопку **Создать запись**.
    
11. Щелкните элемент **Служба** и введите **_sipinternaltls**.
    
12. Нажмите пункт **Протокол** и введите **_tcp**.
    
13. Нажмите **Номер порта** и введите **5061**.
    
14. Щелкните **host, предлагающий эту** службу, а затем введите FQDN пула или сервера Standard Edition.
    
     ![Снимок экрана: диалоговое окно с новой записью ресурса.](../../media/54b1aac5-a2ec-41fe-90c0-02eaeaa9d1b4.png)
  
15. Нажмите кнопку **ОК**, а затем нажмите кнопку **Готово**.
    
### <a name="verify-dns-records"></a>Проверка записей DNS

1. Выполните вход на клиентский компьютер с использованием учетной записи, которая является членом группы прошедших проверку пользователей или имеет эквивалентные разрешения.
    
2. Нажмите **кнопку**"Начните", а **затем введите cmd** и нажмите ввод.
    
3. Введите **nslookup или \<FQDN of the Front End pool\>** нажмите **\<FQDN of the Standard Edition server or single Enterprise Edition server\>** ввод.
    
4. Продолжите проверку остальных записей A для развертывания.
    
5. Если вы поддерживаете устаревшие клиенты и создали запись SRV, проверьте ее, введя set **type=srv** в **запросе nslookup,** а затем нажмите ввод.
    
6. Введите **_sipinternaltls._tcp. *domain*** (например, _sipinternaltls._tcp.contoso.local) и нажмите ввод.
    
7. Ожидаемый результат должен быть аналогиен показанному на рисунке. Обратите внимание, что не все записи DNS показаны в примере выходных данных, но все записи должны быть проверены. 
    
     ![Проверьте параметры DNS.](../../media/4fcaa70f-7717-4939-9652-d2048d6293cc.png)
