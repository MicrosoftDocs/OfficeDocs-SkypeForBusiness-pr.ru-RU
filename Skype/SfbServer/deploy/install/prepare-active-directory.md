# <a name="skype-for-business-server-prepare-active-directory"></a>Skype для бизнеса Server: Подготовка Active Directory
 
**Сводка:** Узнайте, как подготовить домен Active Directory к установке Skype для бизнеса Server. Скачайте бесплатную пробную Skype для бизнеса Server из [Центра оценки Майкрософт.](https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server)
  
Skype для бизнеса Server тесно сотрудничает с Active Directory. Необходимо подготовить домен Active Directory для работы с Skype для бизнеса Server. Этот процесс выполняется в мастере развертывания и выполняется только один раз для домена. Это потому, что процесс создает группы и изменяет домен, и это необходимо сделать только один раз. Вы можете сделать шаги от 1 до 5 в любом порядке. Однако необходимо сделать шаги 6, 7 и 8 в порядке и после шагов 1-5, как описано на схеме. Подготовка Active Directory — шаг 4 из 8. Дополнительные сведения о планировании Active [](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) Directory см. в Skype для бизнеса Server требования к Skype для бизнеса Server или Server для Skype для бизнеса Server [2019 г.](../../../SfBServer2019/plan/system-requirements.md)
  
![схема обзора.](../../media/2c52d307-7859-4009-9489-024b2e130bb3.png)
  
## <a name="prepare-active-directory"></a>Подготовка Active Directory

Skype для бизнеса Server тесно интегрирована с службами домена Active Directory (AD DS). Прежде Skype для бизнеса Server можно установить впервые, необходимо подготовить Active Directory. Раздел Мастер развертывания под названием **Prepare Active Directory** готовит среду Active Directory для использования с Skype для бизнеса Server.
  
> [!NOTE]
> Skype для бизнеса Server (AD DS) для отслеживания и связи со всеми серверами топологии. Большинство этих серверов должны быть соединены с доменом, чтобы Skype для бизнеса Server работать должным образом. Помните, что к серверам, таким как Edge и Reverse Proxy, не следует присоединяться к домену.
  
> [!IMPORTANT]
> Процедура Prepare Active Directory должна запускаться только один раз для каждого домена в развертывании. 
  
Просмотрите видео действия по **подготовке Active Directory:**
  
> [!video https://www.microsoft.com/videoplayer/embed/RE1Ybuk]
  
### <a name="prepare-active-directory-from-the-deployment-wizard"></a>Подготовка Active Directory из мастера развертывания

1. Войдите в систему как пользователь с учетными данными администраторов схемы для домена Active Directory.
    
2. Откройте Skype для бизнеса Server мастер развертывания.
    
    > [!TIP]
    > Если вы хотите просмотреть файлы журнала, созданные мастером развертывания Skype для бизнеса Server, вы можете найти их на компьютере, на котором запускался мастер развертывания, в каталоге пользователей пользователя AD DS, который запустил этот шаг. Например, если пользователь вошел в качестве администратора домена в домене contoso.local, файлы журнала находятся в: C:\Users\Administrator.Contoso\AppData\Local\Temp. 
  
3. Щелкните **ссылку Prepare Active Directory.**
    
4. **Шаг 1. Подготовка схемы**
    
    а. Просмотрите необходимые сведения для шага 1, к которым можно получить доступ, щелкнув выпадение под заголовком Шаг 1.
    
    Б. Нажмите **кнопку Выполнить** в шаге 1, чтобы запустить мастер подготовки схемы.
    
    c. Обратите внимание, что процедуру следует выполнить только один раз для каждого развертывания, а затем нажмите **кнопку Далее**.
    
    d. После подготовки схемы можно просмотреть журнал, щелкнув **журнал View Log**. 
    
    e. Нажмите **кнопку Готово,** чтобы закрыть мастер подготовки схемы и вернуться к шагам Prepare Active Directory.
    
5. **Шаг 2. Проверка репликации раздела схемы**
    
    а. Войдите в контроллер домена для домена.
    
    Б. Откройте **ADSI Редактирование** из выпадаемого меню **Tools** в **Server Manager.**
    
    c. В меню **Действие** выберите команду **Подключиться к другому компьютеру**.
    
    d. В диалоговом окне **Параметры подключения** выберите **Схема** в раскрывающемся списке **Выберите известный контекст именования** и затем нажмите кнопку **ОК**.
    
    e. В контейнере схемы поиск **CN=ms-RTC-SIP-SchemaVersion**. Если этот объект существует, а значение атрибута **rangeUpper** — 1150, а атрибут **rangeLower** — 3, схема была успешно обновлена и реплицирована. Если этого объекта не существует или значения атрибутов **rangeUpper** и **rangeLower** не указаны, схема не была изменена или не реплицирована.
    
6. **Шаг 3. Подготовка текущего леса**
    
    а. Просмотрите необходимые сведения для шага 3, к которым можно получить доступ, щелкнув выпадение под заголовком Шаг 3.
    
    Б. Нажмите **кнопку Выполнить** в шаге 3, чтобы запустить мастер подготовки текущего леса.
    
    c. Обратите внимание, что процедура должна запускаться только один раз за развертывание, а затем нажмите **кнопку Далее**.
    
    d. Укажите домен, в котором будут созданы универсальные группы. Если сервер является частью домена, вы можете выбрать **локальный домен** и нажмите **кнопку Далее**.
    
    e. После подготовки леса можно просмотреть журнал, щелкнув **журнал View Log**. 
    
    е) Нажмите **кнопку Готово,** чтобы закрыть мастер подготовки текущего леса и вернуться к шагам Подготовка active Directory.
    
    ж. Нажмите **Skype для бизнеса Server на** странице **Приложения,** чтобы запустить PowerShell.
    
    з. Введите команду Get-CsAdForest и нажмите **кнопку Ввод**.
    
    i. Если результат **LC_FORESTSETTINGS_STATE_READY,** лес успешно подготовлен, как показано на рисунке.
    
     ![Проверка репликации леса.](../../media/20835669-b8ff-445b-aa8d-51cce46a8b0e.png)
  
7. **Шаг 4. Проверка репликации глобального каталога**
    
    а. На контроллере домена (желательно в месте, удаленном от других контроллеров домена) в лесу, для которого выполнялась подготовка, откройте окно **Active Directory — пользователи и компьютеры**.
    
    Б. В окне **Active Directory — пользователи и компьютеры** разверните имя домена своего леса или дочернего домена.
    
    c. Щелкните **контейнер Users** на левой стороне области и найдите универсальный центр **CsAdministrator** в правой стороне. Если CsAdministrator (среди других новых универсальных групп, которые начинаются с CS) присутствует, репликация Active Directory была успешной.
    
    d. Если группы еще не представлены, можно заставить репликацию или подождать 15 минут и обновить правую боковую часть. Если группы появились, репликация завершена.
    
8. **Шаг 5. Подготовка текущего домена**
    
    а. Просмотрите необходимые сведения для шага 5.
    
    Б. Нажмите **кнопку Выполнить** в шаге 5, чтобы запустить мастер подготовки текущего домена.
    
    c. Обратите внимание, что процедуру следует выполнить только один раз для каждого домена в развертывании, а затем нажмите **кнопку Далее**.
    
    d. После подготовки домена можно просмотреть журнал, щелкнув **журнал View Log**. 
    
    e. Щелкните **Кнопку Готово,** чтобы закрыть мастер подготовки текущего домена и вернуться к шагам Prepare Active Directory.
    
    Эти действия должны быть завершены в каждом домене, Skype для бизнеса Server находятся объекты, в противном случае службы могут не запускаться. Это включает любой тип объекта Active Directory, например пользователей, контактных объектов, административных групп или любого другого типа объекта. Вы можете использовать Set-CsUserReplicatorConfiguration-ADDomainNamingContextList, чтобы при необходимости добавлять только домены с Skype для бизнеса Server объектами.
    
9. **Шаг 6. Проверка репликации в домене**
    
    а. Щелкните **Skype для бизнеса Server на** странице **Приложения,** чтобы запустить PowerShell.
    
    Б. Используйте командную команду Get-CsAdDomain для проверки репликации в домене.
    
   ```powershell
   Get-CsAdDomain [-Domain <Fqdn>] [-DomainController <Fqdn>] [-GlobalCatalog <Fqdn>] [-GlobalSettingsDomainController <Fqdn>]
   ```

    > [!NOTE]
    > Если параметр Domain не указан, в качестве значения используется локальный домен. 
  
    Пример запуска команды для домена contoso.local:
    
   ```powershell
   Get-CsAdDomain -Domain contoso.local -GlobalSettingsDomainController dc.contoso.local
   ```

    > [!NOTE]
    > С помощью параметра GlobalSettingsDomainController можно указать, где хранятся глобальные параметры. Если параметры хранятся в контейнере System (что характерно для развертывания обновления, в котором глобальный параметр не был перенесен в контейнер Configuration), вы определяете контроллер домена в корне леса AD DS. Если глобальные параметры размещены в контейнере конфигурации (что типично для новых или обновленных развертываний, в которых глобальный параметр был перенесен в контейнер конфигурации), определите любой контроллер домена в лесу. Если этот параметр не указан, то в этом документе предполагается, что параметры хранятся в контейнере Configuration и ссылаются на любой контроллер домена в Active Directory. 
  
    c. Если результат **LC_DOMAINSETTINGS_STATE_READY,** домен успешно реплицируется.
    
10. **Шаг 7. Добавление пользователей для предоставления административного доступа к панели Skype для бизнеса Server управления**
    
    а. Войдите как участник группы администраторов домена или группы RTCUniversalServerAdmins.
    
    Б. Откройте пользователи и компьютеры Active **Directory,**  расширите домен, щелкните контейнер "Пользователи", щелкните правой кнопкой мыши CSAdministrator и выберите **свойства**.
    
    c. В окне **Свойства CSAdministrator** перейдите на вкладку **Участники**.
    
    d. На вкладке **Участники** нажмите кнопку **Добавить**. В окне **Выбрать пользователей, контакты, компьютеры, учетные записи служб или группы** найдите элемент **Введите имена объектов для выбора**. Введите имена пользователей или групп для добавления в группу CSAdministrators. Нажмите кнопку **ОК**.
    
    e. На **вкладке Участники** подтвердим, что выбранные вами пользователи или группы присутствуют. Нажмите кнопку **ОК**.
    
    > [!CAUTION]
    > Панель Skype для бизнеса Server управления — это средство управления доступом на основе ролей. Членство в группе CsAdministrator предоставляет пользователю, использующему панель управления Skype для бизнеса Server для всех доступных функций конфигурации. Имеются также другие роли, предназначенные для выполнения определенных функций. Подробные сведения о доступных ролях см. в Skype для бизнеса Server требования к Skype для бизнеса Server [2019](../../../SfBServer2019/plan/system-requirements.md)г. [](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) Обратите внимание, что пользователям не нужно включить Skype для бизнеса Server для того, чтобы быть членами групп управления. 
  
    > [!CAUTION]
    > Чтобы сохранить целостность управления доступом на основе безопасности и ролей, добавьте пользователей в группы, которые определяют роль, которую выполняет пользователь в управлении Skype для бизнеса Server развертывания. 
  
11. Войдите в систему, а затем войдите в Windows, чтобы маркер безопасности обновился с помощью новой группы Skype для бизнеса Server безопасности, а затем снова открыть мастер развертывания.
    
12. Убедитесь, что рядом с **подготовкой Active Directory** отображается зеленый знак, чтобы подтвердить успех, как показано на рисунке.
    
     ![Подготовка Active Directory Completed.](../../media/1fbb655a-25c3-4652-96f9-af0427def17d.png)
