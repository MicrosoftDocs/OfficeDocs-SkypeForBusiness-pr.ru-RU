# <a name="prepare-active-directory-for-skype-for-business-server"></a>Подготовка Active Directory для Skype для бизнеса Server
 
**Сводка:** Узнайте, как подготовить домен Active Directory к установке Skype для бизнеса Server. Скачайте бесплатную пробную версия Skype для бизнеса Server из [Центра оценки Майкрософт.](https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server)
  
Skype для бизнеса Server тесно работает с Active Directory. Необходимо подготовить домен Active Directory для работы со Skype для бизнеса Server. Этот процесс выполняется в мастере развертывания и выполняется только один раз для домена. Это необходимо потому, что процесс создает группы и изменяет домен, и это необходимо сделать только один раз. Шаги 1-5 можно делать в любом порядке. Однако шаги 6, 7 и 8 необходимо делать по порядку и после шагов с 1 по 5, как описано на схеме. Подготовка Active Directory — шаг 4 из 8. Дополнительные сведения о планировании active Directory см. в требованиях к среде [для Skype для](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) бизнеса Server или Сервера для Skype для бизнеса Server [2019.](../../../SfBServer2019/plan/system-requirements.md)
  
![обзорная схема](../../media/2c52d307-7859-4009-9489-024b2e130bb3.png)
  
## <a name="prepare-active-directory"></a>Подготовка Active Directory

Skype для бизнеса Server тесно интегрирован с доменными службами Active Directory (AD DS). Перед первой установкой Skype для бизнеса Server необходимо подготовить Active Directory. В разделе мастера развертывания **"Подготовка Active Directory"** подготавливается среда Active Directory для использования со Skype для бизнеса Server.
  
> [!NOTE]
> Skype для бизнеса Server использует AD DS для отслеживания и связи со всеми серверами в топологии. Большинство этих серверов необходимо присоединить к домену, чтобы Skype для бизнеса Server работал правильно. Помните, что такие серверы, как Edge и обратный прокси-сервер, не должны присоединяться к домену.
  
> [!IMPORTANT]
> Процедуру подготовки Active Directory следует запускать только один раз для каждого домена в развертывании. 
  
Посмотрите видео по подготовке **Active Directory:**
  
> [!video https://www.microsoft.com/videoplayer/embed/RE1Ybuk]
  
### <a name="prepare-active-directory-from-the-deployment-wizard"></a>Подготовка Active Directory из мастера развертывания

1. Войдите как пользователь с учетными данными администраторов схемы для домена Active Directory.
    
2. Откройте мастер развертывания Skype для бизнеса Server.
    
    > [!TIP]
    > Если вы хотите просмотреть файлы журналов, созданные мастером развертывания Skype для бизнеса Server, их можно найти на компьютере, где был запускался мастер развертывания, в каталоге Users пользователя AD DS, выдавшего этот шаг. Например, если пользователь вошел как администратор домена в домене contoso.local, файлы журнала будут расположены в папке C:\Users\Administrator.Contoso\AppData\Local\Temp. 
  
3. Щелкните **ссылку "Подготовка Active Directory".**
    
4. **Шаг 1. Подготовка схемы**
    
    а. Просмотрите сведения о предварительных условиях для шага 1, к которым можно получить доступ, щелкнув выпадающий вниз под заголовком шага 1.
    
    б. Нажмите **кнопку** "Выполнить" в шаге 1, чтобы запустить мастер подготовки схемы.
    
    в. Обратите внимание, что процедуру следует запускать только один раз для каждого развертывания, а затем нажмите кнопку **"Далее".**
    
    г. После подготовки схемы можно просмотреть журнал, щелкнув **"Просмотреть журнал".** 
    
    д. Нажмите **кнопку** "Готово", чтобы закрыть мастер подготовки схемы, и вернитесь к этапу подготовки Active Directory.
    
5. **Шаг 2. Проверка репликации раздела схемы**
    
    а. Войдите в контроллер домена для домена.
    
    б. Откройте **редактор ADSI из** выпадаемого меню "Инструменты" в **диспетчере серверов.** 
    
    в. В меню **Действие** выберите команду **Подключиться к другому компьютеру**.
    
    г. В диалоговом окне **Параметры подключения** выберите **Схема** в раскрывающемся списке **Выберите известный контекст именования** и затем нажмите кнопку **ОК**.
    
    д. В контейнере схемы найди **CN=ms-RTC-SIP-SchemaVersion.** Если этот объект существует, а значение атрибута **rangeUpper** — 1150, а значение атрибута **rangeLower** — 3, схема успешно обновлена и реплицирована. Если этот объект не существует или значения атрибутов **rangeUpper** и **rangeLower** не указаны, схема не была изменена или не реплицирована.
    
6. **Шаг 3. Подготовка текущего леса**
    
    а. Просмотрите необходимые сведения для шага 3, к которым можно получить доступ, щелкнув выпадающий вниз под заголовком шага 3.
    
    б. Нажмите **кнопку** "Выполнить" в шаге 3, чтобы запустить мастер подготовки текущего леса.
    
    в. Обратите внимание, что процедуру следует запускать только один раз для каждого развертывания, а затем нажмите кнопку **"Далее".**
    
    г. Укажите домен, в котором будут созданы универсальные группы. Если сервер является частью домена, можно выбрать локальный домен **и** нажать кнопку **"Далее".**
    
    д. После подготовки леса можно просмотреть журнал, щелкнув **"Просмотреть журнал".** 
    
    е. Нажмите **кнопку** "Готово", чтобы закрыть мастер подготовки текущего леса, и вернитесь к этапу подготовки Active Directory.
    
    ж. Щелкните **На странице "Приложения" щелкните** оболочку управления Skype для бизнеса Server, чтобы запустить PowerShell. 
    
    h. Введите команду Get-CsAdForest и **нажмите** ввод.
    
    i. Если результат **LC_FORESTSETTINGS_STATE_READY,** лес успешно подготовлен, как показано на рисунке.
    
     ![Проверьте репликацию леса.](../../media/20835669-b8ff-445b-aa8d-51cce46a8b0e.png)
  
7. **Шаг 4. Проверка репликации глобального каталога**
    
    а. На контроллере домена (желательно в месте, удаленном от других контроллеров домена) в лесу, для которого выполнялась подготовка, откройте окно **Active Directory — пользователи и компьютеры**.
    
    б. В окне **Active Directory — пользователи и компьютеры** разверните имя домена своего леса или дочернего домена.
    
    в. Щелкните **контейнер Users** в левой области и найдите универсальную группу **CsAdministrator** в правой области. Если присутствует CsAdministrator (среди других новых универсальных групп, которые начинаются с CS), репликация Active Directory прошла успешно.
    
    г. Если группы еще не представлены, можно принудительно использовать репликацию или подождать 15 минут и обновить правую часть области. Если группы появились, репликация завершена.
    
8. **Шаг 5. Подготовка текущего домена**
    
    а. Просмотрите необходимые сведения для шага 5.
    
    б. Нажмите **кнопку** "Выполнить" в шаге 5, чтобы запустить мастер подготовки текущего домена.
    
    в. Обратите внимание, что процедуру следует выполнить только один раз для каждого домена в развертывании, а затем нажмите кнопку **"Далее".**
    
    г. После подготовки домена можно просмотреть журнал, щелкнув **"Просмотреть журнал".** 
    
    д. Нажмите **кнопку** "Готово", чтобы закрыть мастер подготовки текущего домена, и вернитесь к этапу подготовки Active Directory.
    
    Эти действия должны быть выполнены в каждом домене, где находятся объекты Skype для бизнеса Server, в противном случае службы могут не запускаться. Это относится к любому типу объекта Active Directory, например пользователям, контактным объектам, административным группам или любому другому типу объекта. При необходимости Set-CsUserReplicatorConfiguration -ADDomainNamingContextList можно использовать для добавления только доменов с объектами Skype для бизнеса Server.
    
9. **Шаг 6. Проверка репликации в домене**
    
    а. Щелкните на странице "Приложения"  оболочку управления Skype для бизнеса **Server,** чтобы запустить PowerShell.
    
    б. Используйте командную Get-CsAdDomain для проверки репликации в домене.
    
   ```powershell
   Get-CsAdDomain [-Domain <Fqdn>] [-DomainController <Fqdn>] [-GlobalCatalog <Fqdn>] [-GlobalSettingsDomainController <Fqdn>]
   ```

    > [!NOTE]
    > Если параметр Domain не указан, в качестве значения используется локальный домен. 
  
    Пример запуска команды для домена contoso.local:
    
   ```powershell
   Get-CsAdDomain -Domain contoso.local -GlobalSettingsDomainController dc.contoso.local
   ```

    > [!NOTE]
    > С помощью параметра GlobalSettingsDomainController можно указать, где хранятся глобальные параметры. Если параметры хранятся в контейнере System (что обычно используется в развертываниях обновления, в которых глобальный параметр не был перенесен в контейнер Configuration), необходимо определить контроллер домена в корне леса AD DS. Если глобальные параметры размещены в контейнере конфигурации (что типично для новых или обновленных развертываний, в которых глобальный параметр был перенесен в контейнер конфигурации), определите любой контроллер домена в лесу. Если этот параметр не указан, он предполагает, что параметры хранятся в контейнере Configuration и ссылаются на любой контроллер домена в Active Directory. 
  
    в. Если результат **LC_DOMAINSETTINGS_STATE_READY,** домен успешно реплицирован.
    
10. **Шаг 7. Добавление пользователей для предоставления административного доступа к панели управления Skype для бизнеса Server**
    
    а. Войдите как участник группы администраторов домена или группы RTCUniversalServerAdmins.
    
    б. Откройте "Пользователи и компьютеры **Active Directory",** разкройте свой домен, щелкните контейнер **"Пользователи",** щелкните правой кнопкой мыши CSAdministrator и выберите **"Свойства".**
    
    в. В окне **Свойства CSAdministrator** перейдите на вкладку **Участники**.
    
    г. На вкладке **Участники** нажмите кнопку **Добавить**. В окне **Выбрать пользователей, контакты, компьютеры, учетные записи служб или группы** найдите элемент **Введите имена объектов для выбора**. Введите имена пользователей или групп для добавления в группу CSAdministrators. Нажмите кнопку **ОК**.
    
    д. На **вкладке "Участники"** подтвердим, что присутствуют выбранные пользователи или группы. Нажмите **ОК**.
    
    > [!CAUTION]
    > Панель управления Skype для бизнеса Server — это средство управления доступом на основе ролей. Членство в группе CsAdministrator предоставляет пользователю, использующему панель управления Skype для бизнеса Server, полный контроль над всеми доступными функциями конфигурации. Имеются также другие роли, предназначенные для выполнения определенных функций. For details on the roles available, see [Environmental requirements for Skype for Business Server](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) or Server [requirements for Skype for Business Server 2019](../../../SfBServer2019/plan/system-requirements.md). Обратите внимание, что пользователям не нужно включить Skype для бизнеса Server, чтобы они были членами групп управления. 
  
    > [!CAUTION]
    > Чтобы обеспечить безопасность и целостность управления доступом на основе ролей, добавьте пользователей в группы, которые определяют, какую роль выполняет пользователь при управлении развертыванием Skype для бизнеса Server. 
  
11. Выйдите из системы и снова войдите в Windows, чтобы обновить маркер безопасности с помощью новой группы безопасности Skype для бизнеса Server, а затем снова откроете мастер развертывания.
    
12. Убедитесь, что рядом с подготовкой **Active Directory** для подтверждения успешности отображается зеленый контрольный знак, как показано на рисунке.
    
     ![Подготовка Active Directory Завершена.](../../media/1fbb655a-25c3-4652-96f9-af0427def17d.png)
