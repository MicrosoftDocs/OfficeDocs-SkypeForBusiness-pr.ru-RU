# <a name="prepare-active-directory-for-skype-for-business-server"></a>Подготовка Active Directory для Skype для бизнеса Server
 
**Сводка:** Узнайте, как подготовить домен Active Directory для установки Skype для бизнеса Server. Скачайте бесплатную пробную версию Skype для бизнеса Server в [центре оценки Майкрософт](https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server).
  
Skype для бизнеса Server работает в тесном сотрудничестве с Active Directory. Для работы с Skype для бизнеса Server необходимо подготовить домен Active Directory. Этот процесс выполняется в мастере развертывания и выполняется только один раз для домена. Это связано с тем, что в процессе создаются группы и изменяются домены, и их необходимо сделать только один раз. Шаги с 1 по 5 можно выполнять в любом порядке. Однако шаги 6, 7 и 8 необходимо выполнять по порядку, а после шагов 1 – 5, как показано на схеме. Подготовка Active Directory — шаг 4 из 8. Дополнительные сведения о планировании Active Directory приведены в статье [требования к окружающей среде для Skype для бизнеса Server](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или [сервера для Skype для бизнеса Server 2019](../../../SfBServer2019/plan/system-requirements.md).
  
![Обзорная схема](../../media/2c52d307-7859-4009-9489-024b2e130bb3.png)
  
## <a name="prepare-active-directory"></a>Подготовка Active Directory

Skype для бизнеса Server тесно интегрирован с доменными службами Active Directory (AD DS). Прежде чем Skype для бизнеса Server можно будет установить в первый раз, необходимо подготовить Active Directory. Раздел мастера развертывания с названием **Подготовка Active Directory** подготавливает среду Active Directory для использования с Skype для бизнеса Server.
  
> [!NOTE]
> Skype для бизнеса Server использует (AD DS) для отслеживания и общения со всеми серверами в топологии. Большинство этих серверов необходимо присоединить к домену, чтобы Skype для бизнеса Server мог работать правильно. Имейте в виду, что серверы, такие как пограничный и обратный прокси, не должны быть присоединены к домену.
  
> [!IMPORTANT]
> Процедуру Prepare Active Directory следует выполнять только один раз для каждого домена в развертывании.

### <a name="prepare-active-directory-from-the-deployment-wizard"></a>Подготовка Active Directory из мастера развертывания

1. Выполните вход в качестве пользователя с учетными данными администраторов схемы для домена Active Directory.
    
2. Откройте мастер развертывания Skype для бизнеса Server.
    
    > [!TIP]
    > Если вы хотите просмотреть файлы журналов, созданные мастером развертывания Skype для бизнеса Server, вы можете найти их на компьютере, на котором запущен мастер развертывания, в каталоге Users пользователя AD DS, запустившего шаг. Например, если пользователь вошел в систему в качестве администратора домена в домене contoso. local, файлы журнала находятся в: К:\усерс\администратор.контосо\аппдата\локал\темп. 
  
3. Щелкните ссылку **подготовить Active Directory** .
    
4. **Шаг 1: подготовка схемы**
    
    а. Просмотрите сведения о необходимых условиях для шага 1, к которым можно получить доступ, щелкнув раскрывающийся список под заголовком Step 1 (шаг 1).
    
    б. Нажмите **выполнить** на шаге 1, чтобы запустить мастер подготовки схемы.
    
    в. Обратите внимание, что для каждого развертывания процедура должна быть выполнена только один раз, а затем нажмите кнопку **Далее**.
    
    г. После подготовки схемы можно просмотреть журнал, нажав кнопку **Просмотреть журнал**. 
    
    д. Нажмите кнопку **Готово** , чтобы закрыть мастер подготовки схемы и вернуться к процедуре подготовки Active Directory.
    
5. **Шаг 2: проверка репликации раздела схемы**
    
    а. Войдите в систему на контроллере домена.
    
    б. Откройте редактор **ADSI** в раскрывающемся меню **Сервис** **диспетчера серверов**.
    
    в. В меню **Действие** выберите команду **Подключиться к другому компьютеру**.
    
    г. В диалоговом окне **Параметры подключения** выберите **Схема** в раскрывающемся списке **Выберите известный контекст именования** и затем нажмите кнопку **ОК**.
    
    д. В контейнере схемы выполните поиск по параметру **CN = MS – RTC/SIP – SchemaVersion**. Если этот объект существует, а значение атрибута **rangeUpper** — 1150, а значение атрибута **rangeLower** — 3, схема успешно обновлена и реплицирована. Если этот объект не существует или значения атрибутов **rangeUpper** и **rangeLower** не указаны, схема не была изменена или не была реплицирована.
    
6. **Шаг 3: подготовка текущего леса**
    
    а. Просмотрите сведения о предварительных требованиях для шага 3, к которым можно получить доступ, щелкнув раскрывающийся список под заголовком Step 3.
    
    б. Нажмите кнопку **выполнить** на шаге 3, чтобы запустить мастер подготовки текущего леса.
    
    в. Обратите внимание на то, что процедуру следует выполнять только один раз для каждого развертывания, а затем нажмите кнопку **Далее**.
    
    г. Укажите домен, в котором будут создаваться универсальные группы. Если сервер входит в домен, вы можете выбрать **локальный домен**и нажать кнопку **Далее**.
    
    д. После подготовки леса можно просмотреть журнал, нажав кнопку **Просмотреть журнал**. 
    
    е. Нажмите кнопку **Готово** , чтобы закрыть мастер подготовки текущего леса и вернуться к процедуре подготовки Active Directory.
    
    ж. Откройте **консоль управления Skype для бизнеса Server** на странице **приложения** , чтобы запустить PowerShell.
    
    высоты. Введите команду Get – CsAdForest и нажмите клавишу **Ввод**.
    
    узнать. Если результат **LC_FORESTSETTINGS_STATE_READY**, лес успешно подготовлен, как показано на рисунке.
    
     ![Проверьте репликацию леса.](../../media/20835669-b8ff-445b-aa8d-51cce46a8b0e.png)
  
7. **Шаг 4: проверка репликации глобального каталога**
    
    а. На контроллере домена (желательно в месте, удаленном от других контроллеров домена) в лесу, для которого выполнялась подготовка, откройте окно **Active Directory — пользователи и компьютеры**.
    
    б. В окне **Active Directory — пользователи и компьютеры** разверните имя домена своего леса или дочернего домена.
    
    в. Щелкните контейнер **Пользователи** в левой области и найдите универсальную группу **CsAdministrator** в правой области. Если присутствует CsAdministrator (среди других новых универсальных групп, начинающихся с CS), репликация Active Directory успешно выполнена.
    
    г. Если эти группы пока отсутствуют, можно принудительно выполнить репликацию или подождать 15 минут и обновить правую область. Если группы появились, репликация завершена.
    
8. **Шаг 5: подготовка текущего домена**
    
    а. Просмотрите сведения о предварительных требованиях для шага 5.
    
    б. Нажмите кнопку **выполнить** на шаге 5, чтобы запустить мастер подготовки текущего домена.
    
    в. Обратите внимание на то, что процедуру следует выполнять только один раз для каждого домена в развертывании, а затем нажмите кнопку **Далее**.
    
    г. После подготовки домена можно просмотреть журнал, нажав кнопку **Просмотреть журнал**. 
    
    д. Нажмите кнопку **Готово** , чтобы закрыть мастер подготовки текущего домена и вернуться к процедуре подготовки Active Directory.
    
    Эти действия необходимо выполнить в каждом домене, где найдены объекты Skype для бизнеса Server, иначе службы могут не запуститься. Сюда входят все типы объектов Active Directory, такие как пользователи, контактные объекты, административные группы или другие типы объектов. При необходимости вы можете использовать Set-CsUserReplicatorConfiguration-ADDomainNamingContextList, чтобы добавлять только домены с объектами Skype для бизнеса Server.
    
9. **Шаг 6: проверка репликации в домене**
    
    а. Откройте **консоль управления Skype для бизнеса Server** на странице **приложения** , чтобы запустить PowerShell.
    
    б. С помощью команды Get – CsAdDomain можно проверить репликацию в домене.
    
   ```powershell
   Get-CsAdDomain [-Domain <Fqdn>] [-DomainController <Fqdn>] [-GlobalCatalog <Fqdn>] [-GlobalSettingsDomainController <Fqdn>]
   ```

    > [!NOTE]
    > Если параметр Domain не указан, в качестве значения используется локальный домен. 
  
    Пример выполнения команды для домена contoso. local:
    
   ```powershell
   Get-CsAdDomain -Domain contoso.local -GlobalSettingsDomainController dc.contoso.local
   ```

    > [!NOTE]
    > С помощью параметра Глобалсеттингсдомаинконтроллер можно указать, где хранятся глобальные параметры. Если параметры хранятся в системном контейнере (типичный при развертывании обновлений, для которых не был создан глобальный параметр в контейнере конфигурации), необходимо определить контроллер домена в корне леса AD DS. Если глобальные параметры размещены в контейнере конфигурации (что типично для новых или обновленных развертываний, в которых глобальный параметр был перенесен в контейнер конфигурации), определите любой контроллер домена в лесу. Если не указать этот параметр, командлет предполагает, что параметры хранятся в контейнере конфигурации и ссылаются на любой контроллер домена в Active Directory. 
  
    в. Если результат **LC_DOMAINSETTINGS_STATE_READY**, значит, репликация домена выполнена успешно.
    
10. **Шаг 7: Добавление пользователей для предоставления административного доступа к панели управления Skype для бизнеса Server**
    
    а. Войдите как участник группы администраторов домена или группы RTCUniversalServerAdmins.
    
    б. Откройте оснастку " **Active Directory — пользователи и компьютеры**", разверните домен, щелкните контейнер **Пользователи** , щелкните правой кнопкой мыши CSAdministrator и выберите пункт **Свойства**.
    
    в. В окне **Свойства CSAdministrator** перейдите на вкладку **Участники**.
    
    г. На вкладке **Участники** нажмите кнопку **Добавить**. В окне **Выбрать пользователей, контакты, компьютеры, учетные записи служб или группы** найдите элемент **Введите имена объектов для выбора**. Введите имена пользователей или групп для добавления в группу CSAdministrators. Нажмите кнопку **ОК**.
    
    д. На вкладке **Участники** проверьте наличие выбранных пользователей или групп. Нажмите кнопку **ОК**.
    
    > [!CAUTION]
    > Панель управления "Skype для бизнеса Server" — это средство управления доступом на основе ролей. Членство в группе CsAdministrator дает пользователю, который использует элемент панели управления Skype для бизнеса Server, полный доступ ко всем функциям конфигурации. Имеются также другие роли, предназначенные для выполнения определенных функций. Подробнее о доступных ролях можно узнать в статье [требования к окружающей среде для Skype для бизнеса Server](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или [серверных требований Skype для бизнеса Server 2019](../../../SfBServer2019/plan/system-requirements.md). Обратите внимание, что пользователям не нужно включать Skype для бизнеса Server, чтобы сделать их членами групп управления. 
  
    > [!CAUTION]
    > Чтобы обеспечить целостность системы безопасности и управления доступом на основе ролей, добавьте в группы пользователей, определяющие роль, которую пользователь выполняет при управлении развертыванием Skype для бизнеса Server. 
  
11. Выйдите из системы, а затем снова войдите в Windows, чтобы обновить маркер безопасности с помощью новой группы безопасности Skype для бизнеса Server, а затем снова откройте мастер развертывания.
    
12. Убедитесь, что отображается зеленая галочка рядом с полем **Подготовка Active Directory** к подтверждению успеха, как показано на рисунке.
    
     ![Подготовка Active Directory завершена.](../../media/1fbb655a-25c3-4652-96f9-af0427def17d.png)
