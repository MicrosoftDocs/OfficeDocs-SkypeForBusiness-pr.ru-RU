# <a name="skype-for-business-server-prepare-active-directory"></a>Skype для бизнеса Server: подготовка Active Directory
 
**Сводка:** Узнайте, как подготовить домен Active Directory к установке Skype для бизнеса Server.
  
Skype для бизнеса Server тесно работает с Active Directory. Для работы с Skype для бизнеса Server необходимо подготовить домен Active Directory. Этот процесс выполняется в мастере развертывания и выполняется только один раз для домена. Это связано с тем, что процесс создает группы и изменяет домен, и это необходимо сделать только один раз. Вы можете выполнить шаги с 1 по 5 в любом порядке. Однако необходимо выполнить шаги 6, 7 и 8 по порядку и после шагов 1–5, как показано на схеме. Подготовка Active Directory — шаг 4 из 8. Дополнительные сведения о планировании Active Directory см. [](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) в разделе "Требования к среде для Skype для бизнеса Server или требования к серверу для Skype для бизнеса Server [2019 г."](../../../SfBServer2019/plan/system-requirements.md).
  
![обзорная схема.](../../media/2c52d307-7859-4009-9489-024b2e130bb3.png)
  
## <a name="prepare-active-directory"></a>Подготовка Active Directory

Skype для бизнеса Server тесно интегрирована с доменные службы Active Directory (AD DS). Прежде Skype для бизнеса Server можно будет установить в первый раз, необходимо подготовить Active Directory. В разделе мастера развертывания под названием **Prepare Active Directory (Подготовка Active Directory**) выполняется подготовка среды Active Directory к использованию с Skype для бизнеса Server.
  
> [!NOTE]
> Skype для бизнеса Server использует (AD DS) для отслеживания и взаимодействия со всеми серверами в топологии. Большинство этих серверов должны быть присоединены к домену, чтобы Skype для бизнеса Server работать правильно. Помните, что такие серверы, как Edge и Обратный прокси-сервер, не должны быть присоединены к домену.
  
> [!IMPORTANT]
> Процедура подготовки Active Directory должна выполняться только один раз для каждого домена в развертывании. 
  
Просмотрите видео по **подготовке Active Directory**:

### <a name="prepare-active-directory-from-the-deployment-wizard"></a>Подготовка Active Directory из мастера развертывания

1. Войдите как пользователь с учетными данными администраторов схемы для домена Active Directory.
    
2. Откройте Skype для бизнеса Server развертывания.
    
    > [!TIP]
    > Если вы хотите просмотреть файлы журналов, созданные мастером развертывания Skype для бизнеса Server, их можно найти на компьютере, где был запущен мастер развертывания, в каталоге "Пользователи" пользователя AD DS, выполнившего этот шаг. Например, если пользователь выполнил вход с правами администратора домена в домене contoso.local, файлы журнала находятся в папке C:\Users\Administrator.Contoso\AppData\Local\Temp. 
  
3. Щелкните **ссылку "Подготовка Active Directory** ".
    
4. **Шаг 1. Подготовка схемы**
    
    а. Просмотрите необходимые сведения для шага 1, к которым можно получить доступ, щелкнув раскрывающийся список под заголовком шага 1.
    
    б. Нажмите **кнопку "** Выполнить" на шаге 1, чтобы запустить мастер подготовки схемы.
    
    в. Обратите внимание, что процедура должна выполняться только один раз для каждого развертывания, а затем нажмите кнопку **"Далее"**.
    
    г. После подготовки схемы можно просмотреть журнал, щелкнув **"Просмотреть журнал"**. 
    
    д) Нажмите **кнопку "** Готово", чтобы закрыть мастер подготовки схемы, и вернитесь к шагам подготовки Active Directory.
    
5. **Шаг 2. Проверка репликации раздела схемы**
    
    а. Войдите на контроллер домена для домена.
    
    б. Откройте **adsI Edit** **из раскрывающегося** меню "Сервис **" в диспетчер сервера**.
    
    в. В меню **Действие** выберите команду **Подключиться к другому компьютеру**.
    
    г. В диалоговом окне **Параметры подключения** выберите **Схема** в раскрывающемся списке **Выберите известный контекст именования** и затем нажмите кнопку **ОК**.
    
    д) В контейнере схемы найдите **CN=ms-RTC-SIP-SchemaVersion**. Если этот объект существует, а значение атрибута **rangeUpper** равно 1150, а значение атрибута **rangeLower** равно 3, схема была успешно обновлена и реплицирована. Если этот объект не существует или значения атрибутов **rangeUpper** и **rangeLower** не указаны, схема не была изменена или не реплицирована.
    
6. **Шаг 3. Подготовка текущего леса**
    
    а. Просмотрите необходимые сведения для шага 3, к которым можно получить доступ, щелкнув раскрывающийся список под заголовком шага 3.
    
    б. Нажмите **кнопку "** Выполнить" на шаге 3, чтобы запустить мастер подготовки текущего леса.
    
    в. Обратите внимание, что процедура должна выполняться только один раз для каждого развертывания, а затем нажмите кнопку **"Далее"**.
    
    г. Укажите домен, в котором будут созданы универсальные группы. Если сервер является частью домена, можно **выбрать локальный** домен и нажать кнопку **"Далее"**.
    
    д) После подготовки леса можно просмотреть журнал, щелкнув **"Просмотреть журнал"**. 
    
    f. Нажмите **кнопку "** Готово", чтобы закрыть мастер подготовки текущего леса, и вернитесь к шагам подготовки Active Directory.
    
    ж. **Щелкните Skype для бизнеса Server командной консоли** на странице **"Приложения**", чтобы запустить PowerShell.
    
    з. Введите команду Get-CsAdForest и нажмите **клавишу ВВОД**.
    
    и. Если **результат LC_FORESTSETTINGS_STATE_READY,** лес успешно подготовлен, как показано на рисунке.
    
     ![Проверьте репликацию леса.](../../media/20835669-b8ff-445b-aa8d-51cce46a8b0e.png)
  
7. **Шаг 4. Проверка репликации глобального каталога**
    
    а. На контроллере домена (желательно в месте, удаленном от других контроллеров домена) в лесу, для которого выполнялась подготовка, откройте окно **Active Directory — пользователи и компьютеры**.
    
    б. В окне **Active Directory — пользователи и компьютеры** разверните имя домена своего леса или дочернего домена.
    
    в. Щелкните **контейнер "** Пользователи" на левой боковой панели и найдите **CsAdministrator** универсальной группы в области справа. Если присутствует CsAdministrator (среди других новых универсальных групп, которые начинаются с CS), репликация Active Directory выполнена успешно.
    
    г. Если группы еще нет, можно принудительно выполнить репликацию или подождать 15 минут и обновить правую боковую панель. Если группы появились, репликация завершена.
    
8. **Шаг 5. Подготовка текущего домена**
    
    а. Ознакомьтесь с предварительными требованиями для шага 5.
    
    б. Нажмите **кнопку "** Выполнить" на шаге 5, чтобы запустить мастер подготовки текущего домена.
    
    в. Обратите внимание, что процедура должна выполняться только один раз для каждого домена в развертывании, а затем нажмите кнопку **"Далее"**.
    
    г. После подготовки домена можно просмотреть журнал, щелкнув **"Просмотреть журнал"**. 
    
    д) Нажмите **кнопку "** Готово", чтобы закрыть мастер подготовки текущего домена, и вернитесь к шагам подготовки Active Directory.
    
    Эти действия должны выполняться в каждом домене, где Skype для бизнеса Server объекты, в противном случае службы могут не запускаться. К ним относится любой тип объекта Active Directory, например пользователи, объекты контактов, административные группы или любой другой тип объекта. Вы можете использовать Set-CsUserReplicatorConfiguration -ADDomainNamingContextList, чтобы при необходимости добавлять только домены с Skype для бизнеса Server объектами.
    
9. **Шаг 6. Проверка репликации в домене**
    
    а. Щелкните **Skype для бизнеса Server Management Shell** на странице **"Приложения**", чтобы запустить PowerShell.
    
    б. Используйте команду Get-CsAdDomain для проверки репликации в домене.
    
   ```powershell
   Get-CsAdDomain [-Domain <Fqdn>] [-DomainController <Fqdn>] [-GlobalCatalog <Fqdn>] [-GlobalSettingsDomainController <Fqdn>]
   ```

    > [!NOTE]
    > Если параметр Domain не указан, в качестве значения используется локальный домен. 
  
    Пример выполнения команды для домена contoso.local:
    
   ```powershell
   Get-CsAdDomain -Domain contoso.local -GlobalSettingsDomainController dc.contoso.local
   ```

    > [!NOTE]
    > С помощью параметра GlobalSettingsDomainController можно указать, где хранятся глобальные параметры. Если параметры хранятся в системном контейнере (что типично для развертываний обновления, для которых не был перенесен глобальный параметр в контейнер конфигурации), необходимо определить контроллер домена в корне леса AD DS. Если глобальные параметры размещены в контейнере конфигурации (что типично для новых или обновленных развертываний, в которых глобальный параметр был перенесен в контейнер конфигурации), определите любой контроллер домена в лесу. Если этот параметр не указан, командлет предполагает, что параметры хранятся в контейнере конфигурации и ссылается на любой контроллер домена в Active Directory. 
  
    в. Если **результат LC_DOMAINSETTINGS_STATE_READY,** домен успешно реплицирован.
    
10. **Шаг 7. Добавление пользователей для предоставления административного доступа к Skype для бизнеса Server панель управления**
    
    а. Войдите как участник группы администраторов домена или группы RTCUniversalServerAdmins.
    
    б. Откройте **Пользователи и компьютеры Active Directory**, разверните домен, щелкните контейнер **"** Пользователи", щелкните правой кнопкой мыши CSAdministrator и выберите **пункт "Свойства"**.
    
    в. В окне **Свойства CSAdministrator** перейдите на вкладку **Участники**.
    
    г. На вкладке **Участники** нажмите кнопку **Добавить**. В окне **Выбрать пользователей, контакты, компьютеры, учетные записи служб или группы** найдите элемент **Введите имена объектов для выбора**. Введите имена пользователей или групп для добавления в группу CSAdministrators. Нажмите кнопку **ОК**.
    
    д) Убедитесь **,** что на вкладке "Участники" присутствуют выбранные пользователи или группы. Нажмите кнопку **ОК**.
    
    > [!CAUTION]
    > Это Skype для бизнеса Server панель управления управления доступом на основе ролей. Членство в группе CsAdministrator предоставляет пользователю, который использует Skype для бизнеса Server панель управления полный доступ ко всем доступным функциям конфигурации. Имеются также другие роли, предназначенные для выполнения определенных функций. Дополнительные сведения о доступных ролях см. в разделе "Требования к среде [Skype для бизнеса Server](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или требования к серверу для Skype для бизнеса Server [2019](../../../SfBServer2019/plan/system-requirements.md). Обратите внимание, что пользователям не нужно Skype для бизнеса Server, чтобы быть членами групп управления. 
  
    > [!CAUTION]
    > Чтобы обеспечить безопасность и целостность управления доступом на основе ролей, добавьте пользователей в группы, определяющие, какую роль выполняет пользователь при управлении Skype для бизнеса Server развертывания. 
  
11. Выйдите из системы, а затем снова войдите в Windows, чтобы обновить маркер безопасности с помощью новой Skype для бизнеса Server безопасности, а затем снова откройте мастер развертывания.
    
12. Убедитесь, что рядом с элементом **Prepare Active Directory (Подготовка Active Directory** ) отображается зеленый флажок, как показано на рисунке.
    
     ![Подготовка Active Directory завершена.](../../media/1fbb655a-25c3-4652-96f9-af0427def17d.png)
