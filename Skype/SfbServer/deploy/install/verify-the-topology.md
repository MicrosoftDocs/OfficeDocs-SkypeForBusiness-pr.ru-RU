# <a name="verify-the-topology-in-skype-for-business-server"></a>Проверьте топологию в Skype для бизнеса Server
 
**Сводка:** Узнайте, как проверить, Skype для бизнеса Server и серверы Active Directory работают должным образом.
  
После публикации топологии и установки Skype для бизнеса Server компонентов системы на каждом из серверов в топологии можно убедиться, что топология работает должным образом. Это включает в себя проверку распространения конфигурации на все серверы Active Directory, чтобы весь домен знал, Skype для бизнеса доступен в домене. Вы можете выполнить шаги с 1 по 5 в любом порядке. Однако необходимо выполнить шаги 6, 7 и 8 по порядку и после шагов 1–5, как показано на схеме. Проверка топологии выполняется на шаге 8 из 8.
  
![Обзорная схема.](../../media/c8698b53-1282-4978-a9a6-ca3f7a778f60.png)
  
## <a name="test-the-front-end-pool-deployment"></a>Тестирование развертывания пула переднего плана

Последним шагом является проверка пула переднего плана и подтверждение того, Skype для бизнеса клиенты могут взаимодействовать друг с другом. 
  
### <a name="add-users-and-verify-client-connectivity"></a>Добавление пользователей и проверка подключения клиента

1. Используйте компьютеры и пользователи Active Directory, чтобы добавить объект пользователя Active Directory роли администратора для развертывания Skype для бизнеса Server (на котором Skype для бизнеса Server панель управления установлен) в группу **CSAdministrator**.
    
    > [!IMPORTANT]
    > Если вы не добавите соответствующих пользователей и группы в группу CsAdministors, при открытии Skype для бизнеса Server панель управления который считывает сообщение "Unauthorized: Access is denied due to a role-based access control (RBAC) authorization failure". 
  
2. Если объект-пользователь уже выполнил вход в систему, выйдите из системы, а затем повторите вход, чтобы зарегистрировать назначение новой группы.
    
    > [!NOTE]
    > Учетная запись пользователя не может быть локальным администратором любого сервера, на котором Skype для бизнеса Server. 
  
3. Используйте учетную запись администратора для входа на компьютер, на котором Skype для бизнеса Server панель управления установлен.
    
4. Запустите Skype для бизнеса Server панель управления и при появлении запроса укажите учетные данные. Skype для бизнеса Server панель управления отображает сведения о развертывании.
    
5. На панели навигации слева щелкните "Топология **", а** затем убедитесь Skype для бизнеса Server, что в состоянии службы отображается компьютер с зеленой стрелкой, а рядом с каждой развернутой и подключенной к сети ролью репликации установлена зеленая галочка. 
    
6. В левой области навигации щелкните элемент **Пользователи**, а затем **Включить пользователей**. 
    
7. На странице **"Skype для бизнеса Server пользователя**" нажмите кнопку "**Добавить"**.
    
8. Чтобы определить параметры поиска для объектов, на странице **Select from Active Directory** (Выбор в Active Directory) можно щелкнуть меню **Поиск** и выбрать пункт **Добавить фильтр** (необязательно). Можно также выбрать команду **Поиск LDAP** и ввести выражение LDAP для фильтрации или ограничения числа возвращаемых объектов. Выбрав параметры поиска, нажмите кнопку " **Найти"**.
    
9. В области результатов поиска выберите пользователей, которых вы хотите добавить, и нажмите кнопку " **ОК"**.
    
10. На странице **"Skype для бизнеса Server пользователя**" выбранные пользователи отображаются на экране **"Пользователи**". В **списке "Назначение пользователей пулу** " выберите сервер, на котором должны находиться пользователи.
    
    Ниже приведен список параметров, которые можно использовать для настройки объектов.
    
    - **Создание URI SIP пользователя**
    
    - **Телефонии**
    
    - **Line URI** (URI линии),
    
    - **Политика конференц-связи**,
    
    - **Политика версий клиентов**,
    
    - **PIN policy** (Политика ПИН-кода),
    
    - **Политика внешнего доступа**,
    
    - **Archiving policy** (Политика архивации),
    
    - **Location policy** (Политика расположения),
    
    - **Client policy** (Политика клиента).
    
    Чтобы протестировать базовую функциональность, выберите параметр, который вы предпочитаете, для параметра "Создать **URI SIP** пользователя" (другие параметры в конфигурации используют параметры по умолчанию), а затем нажмите кнопку "Включить **", как** показано на рисунке.
    
     ![Включите пользователей в панель управления.](../../media/7ee8717d-9a1f-4864-8f45-71071c88878f.png)
  
11. Отобразится страница сводки, на которой в столбце "Включено" отображается флажок, указывающий, что пользователи настроены. В столбце **SIP-адрес** будет отображаться адрес, который требуется для настройки входа пользователя.
    
     ![Пользователи, добавленные в Skype для бизнеса Server панель управления.](../../media/8960548a-8d6d-44c5-bc01-6f9fb11b7588.png)
  
12. Войдите в систему одного пользователя на компьютере, присоединенном к домену, а другого — на другом компьютере в домене.
    
13. Установите Skype для бизнеса клиента на каждом из двух клиентских компьютеров, а затем убедитесь, что оба пользователя могут войти в Skype для бизнеса Server и могут отправлять мгновенные сообщения друг другу.
