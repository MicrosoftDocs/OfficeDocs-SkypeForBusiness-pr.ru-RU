# <a name="install-skype-for-business-server"></a>Установка Skype для бизнеса Server
 
**Сводка:** Узнайте, как подготовить среду для установки Skype для бизнеса Server. Скачайте бесплатную пробную версию Skype для бизнеса Server в центре оценки Майкрософт по адресу[https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server](https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server):.
  
В этой статье описывается пример установки Skype для бизнеса Server. В этой статье не рассматриваются все процедуры, необходимые для выполнения полной установки Skype для бизнеса Server. Цель состоит в том, чтобы предоставить примеры процедур в узкой топологии, включающей в себя основные функции соответствия и общего доступа.
  
## <a name="overview-of-the-install-process-for-skype-for-business-server"></a>Обзор процесса установки Skype для бизнеса Server

Установка Skype для бизнеса Server включает множество различных процедур. Процедуры, необходимые для получения Skype для бизнеса Server, работающего в вашей среде, зависят от конкретных настроек среды. Например, если вы используете Windows Server для DNS, вы получите преимущества из примера процедуры для добавления записи DNS. Если вы используете другую систему для DNS, вам необходимо выполнить процедуры для конкретной DNS-системы. Это справедливо для многих процедур, описанных в этом разделе.
  
Skype для бизнеса Server доступен в версии Standard Edition и Enterprise Edition. Основное отличие заключается в том, что стандартный выпуск не поддерживает функции высокого уровня доступности, включенные в выпуск Enterprise Edition. 
  
Skype для бизнеса Server это расширенный продукт, и точный процесс установки зависит от конкретных обстоятельств. В этом разделе описываются общие действия по установке продукта. Однако каждая процедура может различаться в зависимости от среды и принятия решений по планированию. Например, для небольших организаций один сервер может быть подходящим, в то время как в крупной организации могут присутствовать серверы 50, расположенные по всему миру, выделенному для продукта.
  
> [!NOTE]
> Чтобы узнать о последних накопительных обновлениях, ознакомьтесь с разработкой [обновлений для Skype для бизнеса Server](https://support.microsoft.com/kb/3061064). После установки исправления CU1 администратору необходимо выполнить `Update-CsAdminRole` командлет. Этот командлет необходим для доступа к новым командлетам ГКП по сравнению с удаленной оболочкой PowerShell.
  
> [!IMPORTANT]
> Процедуры, описанные в этом разделе, служат в качестве примера с помощью узкого набора требований и предполагают, что определенные решения уже выполнены. Фактические процедуры, необходимые для установки Skype для бизнеса Server, скорее всего, будут сильно отличаться. Используйте процедуры, описанные в этом разделе, в качестве примера, а не пошаговое руководство по установке Skype для бизнеса Server в каждой среде. 
  
При первом запуске Skype для бизнеса Server включается восемь основных этапов. Вы должны знать, что примеры процедур, описанных в этом разделе, не являются единственными процедурами, необходимыми для установки Skype для бизнеса Server. Следующие восемь шагов являются простыми примерами, которые помогут вам лучше понять общий процесс и получить базовую рабочую среду. Шаги с 1 по 5 можно выполнять в любом порядке. Однако шаги 6, 7 и 8 необходимо выполнять по порядку, а после шагов 1 – 5, как показано на схеме. Восемь шагов:
  
![Обзор процесса установки.](../../media/b1a59b39-a7f0-4781-ac4d-2dfef7ca3700.png)
  
- [Установите необходимые компоненты для Skype для бизнеса Server](install-prerequisites.md) : установите необходимые компоненты на все серверы, которые составляют топологию Skype для бизнеса Server. Обратите внимание, что предварительные требования не одинаковы для всех ролей. Например, на серверах, предоставляющих роль сервера переднего плана, есть набор необходимых компонентов, а серверы, которые предоставляют роль директора, имеют разный набор необходимых компонентов. Для получения дополнительных сведений ознакомьтесь с документацией по планированию.
    
- [Создайте файловый ресурс в Skype для бизнеса Server](create-a-file-share.md) : Создайте файловый ресурс, который будет использоваться серверами в топологии Skype для бизнеса Server.
    
- [Установка средств администрирования в Skype для бизнеса Server](install-administrative-tools.md) : средства администрирования включают построитель топологий и панель управления. Необходимо установить средства администрирования по крайней мере на одном сервере в топологии или в 64-разрядной рабочей станции управления под управлением ОС Windows версии, поддерживаемой Skype для бизнеса Server.
    
- [Подготовка Active Directory для Skype для бизнеса Server](prepare-active-directory.md) : Skype для бизнеса Server работает в тесном сотрудничестве с Active Directory. Для работы с Skype для бизнеса Server необходимо подготовить домен Active Directory. Это можно сделать с помощью мастера развертывания, и оно выполняется только один раз для домена. Это связано с тем, что в процессе создаются группы и изменяются домены, и их необходимо сделать только один раз.
    
- [Создание записей DNS для Skype для бизнеса Server](create-dns-records.md) : для правильной работы Skype для бизнеса Server необходимо установить ряд параметров DNS. Это значит, что клиенты знают, как получить доступ к службам, а серверы знают друг о друга. Эти параметры необходимо выполнить только один раз для каждого развертывания, так как при назначении записи DNS она будет доступна в домене.
    
- [Создание и публикация новой топологии в Skype для бизнеса Server](create-and-publish-new-topology.md) : прежде чем устанавливать систему Skype для бизнеса на каждом сервере в топологии, необходимо создать топологию и опубликовать ее. При публикации топологии вы загружаете сведения о топологии в базу данных центрального хранилища управления. Если это пул Enterprise Edition, база данных центрального хранилища управления создается при первой публикации новой топологии. Если это выпуск Standard Edition, перед публикацией топологии необходимо запустить процесс подготовки первого сервера Standard Edition в мастере развертывания. Это подготавливается для выпуска Standard Edition путем установки экземпляра SQL Server Express Edition и создания центрального хранилища управления.
    
- [Установите Skype для бизнеса Server на серверах в топологии](install-skype-for-business-server.md) : после того как топология будет загружена в хранилище центрального хранилища управления, а Active Directory будет знать, какие роли будут выполнять все роли, необходимо установить серверную систему Skype для бизнеса на каждом сервере в топологии.
    
- [Проверьте топологию в Skype для бизнеса Server](verify-the-topology.md) : после публикации топологии и установки компонентов системы Skype для бизнеса Server на каждом из серверов в топологии можно убедиться, что топология работает должным образом. Это включает проверку того, что конфигурация распространена на все серверы Active Directory, чтобы весь домен знал, что Skype для бизнеса доступен в домене.
