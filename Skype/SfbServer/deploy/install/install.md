# <a name="install-skype-for-business-server"></a>Установка Skype для бизнеса Server
 
**Сводка:** Сведения о том, как подготовить среду для установки Skype для бизнеса Server. Загрузите бесплатную пробную версию Skype для бизнеса на сервере Microsoft Evaluation Center по адресу[https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server](https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server):.
  
В этой статье описывается пример установки Skype для бизнеса Server. В этой статье не рассматриваются все процедуры, необходимые для выполнения полной установки Skype для бизнеса Server. Приведенные примеры процедур в специализированной топологии охватывают основные функциональные возможности обмена данными и проведения собраний.
  
## <a name="overview-of-the-install-process-for-skype-for-business-server"></a>Обзор процесса установки Skype для бизнеса Server

Установка Skype для бизнеса Server включает множество различных процедур. Процедуры, необходимые для работы сервера Skype для бизнеса Server в вашей среде, зависят от конкретных особенностей вашей среды. Например, процедура добавления записи службы доменных имен в приведенном здесь примере потребуется в том случае, если эта служба работает в Windows Server. Если служба доменных имен работает в другой системе, необходимо выполнить соответствующую процедуру для этой системы. Это же относится ко многим другим процедурам из этого раздела.
  
Skype для бизнеса Server доступен в стандартном выпуске и выпуске Enterprise Edition. Основное различие заключается в том, что Standard Edition не поддерживает возможностей Enterprise Edition, обеспечивающих функции высокой доступности. 
  
Skype для бизнеса Server — это усовершенствованный продукт, и точный процесс установки зависит от конкретных обстоятельств. В этом разделе описаны общие действия по установке продукта. Однако каждая процедура может различаться в зависимости от вашей среды и решений по планированию. Например, в небольших организациях один сервер может подойти под управлением Skype для Business Server Standard Edition, в то время как в крупных организациях могут быть серверы 50 в разных странах мира, предназначенные для продукта.
  
> [!NOTE]
> Сведения о последних накопительных обновлениях можно найти в статье [Обновление для сервера Skype для бизнеса Server](https://support.microsoft.com/en-us/kb/3061064). После установки исправления CU1 администратор должен выполнить `Update-CsAdminRole` командлет. Этот командлет необходим для доступа к новым командлетам GCP в удаленном режиме с помощью PowerShell.
  
> [!IMPORTANT]
> Примеры процедур в этом разделе относятся к специализированным требованиям в ситуации, когда уже приняты определенные решения. Фактические процедуры, необходимые для установки сервера Skype для бизнеса, скорее этого, будут очень разными. В этом разделе приведены инструкции, а не пошаговые инструкции по установке сервера Skype для бизнеса Server в каждой среде. 
  
В первый раз при первом запуске Skype для бизнеса Server требуется восемь основных шагов. Вы должны понимать, что в примере процедуры, описанные в этом разделе, не являются единственными процедурами, необходимыми для установки Skype для бизнеса Server. Эти восемь шагов позволяют понять процесс в целом и подготовить основные условия для работы. Шаги с 1 по 5 могут выполняться в произвольном порядке. Но шаги 6, 7 и 8 должны выполняться в указанном порядке после шагов с 1 по 5, как показано на схеме. Далее следует описание восьми шагов.
  
![Обзор процесса установки.](../../media/b1a59b39-a7f0-4781-ac4d-2dfef7ca3700.png)
  
- [Установка предварительных требований для Skype для бизнеса Server](install-prerequisites.md) : Установка предварительных требований для всех серверов, которые составляют топологию сервера Skype для бизнеса. Обратите внимание, что предварительные условия для всех ролей не совпадают. Например, серверы, предоставляющие клиентскую роль, имеют набор предварительных требований, а серверы, предоставляющие роль режиссера, имеют другой набор необходимых компонентов. Для получения дополнительных сведений ознакомьтесь с документацией по планированию необходимых компонентов.
    
- [Создание общего файлового файла в Skype для бизнеса Server](create-a-file-share.md) : создание общего файлового файла, который будет использоваться серверами во всей топологии сервера Skype для бизнеса.
    
- [Установка средств администрирования в Skype для бизнеса Server](install-administrative-tools.md) : средства администрирования включают Topology Builder и панель управления. Вы должны установить средства администрирования по крайней мере на одном сервере в топологии или на рабочей станции управления 64-bit для Windows, которая поддерживается в Skype для бизнеса Server.
    
- [Подготовка службы каталогов Active Directory для Skype для бизнеса Server](prepare-active-directory.md) : Skype для бизнеса Server работает почти так же, как и в службе каталогов Active Directory. Для работы с приложением Skype для бизнеса Server необходимо подготовить домен Active Directory. Это можно сделать с помощью мастера развертывания, и оно будет выполняться только один раз для домена. Так как при этом создаются группы и в домен вносятся необходимые изменения, повторение данного процесса не требуется.
    
- [Создание записей DNS для Skype для бизнеса Server](create-dns-records.md) : для правильной работы Skype для бизнеса Server необходимо установить некоторые параметры DNS. Это значит, что клиенты знают, как получить доступ к службам, а серверы знают друг друга. Эти параметры должны быть выполнены только один раз для каждого развертывания, так как после назначения DNS-записи она будет доступна для всего домена.
    
- [Создание и публикация новой топологии в Skype для бизнеса Server](create-and-publish-new-topology.md) : прежде чем устанавливать серверную систему Skype для бизнеса на каждый сервер в топологии, необходимо создать топологию и опубликовать ее. При создании топологии информация о ней загружается в базу данных центрального хранилища управления. В случае пула Enterprise Edition база данных центрального хранилища управления создается при первой публикации новой топологии. Если это выпуск Standard Edition, прежде чем публиковать топологию, необходимо запустить процесс подготовки первого выпуска сервера из мастера развертывания. Этот процесс предусматривает установку экземпляра экспресс-выпуска SQL Server и создание центрального хранилища управления.
    
- [Установка сервера Skype для бизнеса на серверах в топологии](install-skype-for-business-server.md) : после того как топология будет загружена в хранилище Центрального управления и Служба каталогов Active Directory сможет определить, какие серверы будут выполнять роли, необходимо установить серверную систему Skype для бизнеса на каждом из серверов в топологии.
    
- [Проверка топологии в Skype для бизнеса Server](verify-the-topology.md) : после публикации топологии и компонентов системы Skype для бизнеса Server на каждом из серверов в топологии вы можете убедиться, что топология работает должным образом. Сюда входит проверка того, что конфигурация распространялась на все серверы службы каталогов Active Directory, чтобы весь домен знал, что в этом домене есть Skype для бизнеса.
