# <a name="install-skype-for-business-server"></a>Установка Skype для бизнеса Server
 
**Сводка:** сведения о подготовке среды к установке Skype для бизнеса Server. Скачайте бесплатную пробную версию Skype для бизнеса Server из центра оценки Майкрософт по адресу: [https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server](https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server).
  
В этой статье описано на примере, как установить Skype для бизнеса Server. В этой статье не содержится описание всех процедур, необходимых для полной установки Skype для бизнеса Server. Цель состоит в том, чтобы указать примеры процедур в очень специфической топологии, в которой используются основные функциональные возможности обмена данными и проведения собраний.
  
## <a name="overview-of-the-install-process-for-skype-for-business-server"></a>Обзор процесса установки Skype для бизнеса Server

Установка Skype для бизнеса Server включает множество различных процедур. Процедуры, необходимые для обеспечения работы Skype для бизнеса Server в конкретной среде, зависят от особенностей этой среды. Например, процедура добавления записи службы доменных имен в приведенном здесь примере потребуется в том случае, если эта служба работает в Windows Server. Если служба доменных имен работает в другой системе, необходимо выполнить соответствующую процедуру для этой системы. Это же относится ко многим другим процедурам из этого раздела.
  
Skype для бизнеса Server доступен в выпусках Standard Edition и Enterprise Edition. Основное различие заключается в том, что Standard Edition не поддерживает возможностей Enterprise Edition, обеспечивающих функции высокой доступности. 
  
Skype для бизнеса Server — это усовершенствованный продукт, точный процесс установки которого зависит от многих конкретных обстоятельств. В этом разделе приводятся инструкции по основным шагам установки продукта. Тем не менее, каждая процедура может отличаться в зависимости от вашей среды и решений по планированию. Например, для небольших организаций может быть достаточно одного сервера с работающим Skype для бизнеса Server Standard Edition, тогда как в большой многонациональной организации для этого продукта может быть выделено 50 серверов в разных точках мира.
  
> [!NOTE]
> Сведения о последних накопительных пакетах обновления см. в разделе [Обновления для Skype для бизнеса Server](https://support.microsoft.com/en-us/kb/3061064). После установки исправления CU1 администратор должен запустить командлет `Update-CsAdminRole`. Этот командлет необходим для доступа к новым командлетам GCP в удаленном режиме с помощью PowerShell.
  
> [!IMPORTANT]
> Примеры процедур в этом разделе относятся к специализированным требованиям в ситуации, когда уже приняты определенные решения. Реальные процедуры установки Skype для бизнеса Server могут значительно отличаться. Этот раздел служит иллюстрацией, а не пошаговым руководством по установке Skype для бизнеса Server в любой среде. 
  
Начальная подготовка Skype для бизнеса Server к работе включает в себя восемь основных шагов. Следует понимать, что процесс установки Skype для бизнеса Server не ограничивается процедурами, рассматриваемыми в этом примере. Эти восемь шагов позволяют понять процесс в целом и подготовить основные условия для работы. Шаги с 1 по 5 могут выполняться в произвольном порядке. Но шаги 6, 7 и 8 должны выполняться в указанном порядке после шагов с 1 по 5, как показано на схеме. Далее следует описание восьми шагов.
  
![Обзор процесса установки.](../../media/b1a59b39-a7f0-4781-ac4d-2dfef7ca3700.png)
  
- Установка необходимых компонентов для Skype для бизнеса Server](install-prerequisites.md). Установите необходимые компоненты на всех серверах, которые составляют топологию Skype для бизнеса Server. Обратите внимание на то, что необходимые компоненты не являются одинаковыми для всех ролей. Например, серверы, играющие роль переднего плана, имеют набор необходимых компонентов, а серверы, которые обеспечивают роль директора, имеют отличный набор необходимых компонентов. Дополнительные сведения см. в документации по планированию необходимых компонентов.
    
- [Создание файлового ресурса в Skype для бизнеса Server](create-a-file-share.md). Создайте файловый ресурс, который будет использоваться серверами на всех топологиях Skype для бизнеса Server.
    
- [Установка средств администрирования в Skype для бизнеса Server](install-administrative-tools.md). Средства администрирования включают  построитель топологий и панель управления. Средства администрирования необходимо установить хотя бы на одном сервере в топологии или на 64‑разрядной рабочей станции управления на платформе ОС Windows, поддерживаемой Skype для бизнеса Server.
    
- Подготовка службы каталогов Active Directory для Skype для бизнеса Server](prepare-active-directory.md). Приложение Skype для бизнеса Server тесно взаимодействует с Active Directory. Для работы с Skype для бизнеса Server необходимо подготовить домен Active Directory. Это можно сделать с помощью мастера развертывания (выполняется только один раз для домена). Это вызвано тем, что ходе этого процесса создаются группы и изменяется домен, поэтому эту операцию нужно выполнить только один раз.
    
- [Создание записей DNS для Skype для бизнеса Server](create-dns-records.md). Чтобы Skype для бизнеса Server работал правильно, необходимо установить ряд параметров DNS. Эти параметры предоставляют клиентам информацию о способе доступа к службам, а серверам — о других серверах. Для каждого развертывания достаточно задать параметры один раз, так как назначенная вами запись службы доменных имен доступна во всем домене.
    
- Создание и публикация новой топологии в Skype для бизнеса Server](create-and-publish-new-topology.md). Перед установкой системы Skype для бизнеса Server на каждый сервер в топологии, необходимо создать топологию и опубликовать ее. При создании топологии информация о ней загружается в базу данных центрального хранилища управления. В случае пула Enterprise Edition база данных центрального хранилища управления создается при первой публикации новой топологии. В случае Standard Edition перед публикацией топологии необходимо с помощью мастера развертывания выполнить процесс подготовки первого сервера Standard Edition. С его помощью осуществляется подготовка Standard Edition путем установки экземпляра SQL Server Express Edition и создания центрального хранилища управления.
    
- [Установка Skype для бизнеса Server на серверах в топологии](install-skype-for-business-server.md). После загрузки топологии в центральное хранилище управления, обеспечивающей передачу информации о назначенных серверам ролях в службу Active Directory, необходимо установить систему Skype для бизнеса Server на каждом сервере в топологии.
    
- [Проверка топологии в Skype для бизнеса Server](verify-the-topology.md). После публикации топологии и установки компонентов системы Skype для бизнеса Server на каждом из серверов в этой топологии можно приступить к проверке ее функционирования. При этом проверяется распространение конфигурации на все серверы Active Directory, что необходимо для передачи в рамках домена информации о доступности Skype для бизнеса Server в пределах домена.
