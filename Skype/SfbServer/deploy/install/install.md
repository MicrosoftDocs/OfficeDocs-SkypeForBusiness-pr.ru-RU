# <a name="install-skype-for-business-server"></a>Установка Skype для бизнеса Server
 
**Сводка:** Узнайте, как подготовить среду для установки Скайп для Business Server. Загрузить бесплатную пробную версию программы Скайп для Business Server в центре Microsoft оценки по:[https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server](https://www.microsoft.com/evalcenter/evaluate-skype-for-business-server).
  
В этой статье рассматриваются основные пример установки Скайп для Business Server. В этой статье не пытается охватывают все эти процедуры для выполнения полного Скайп для установки Business Server. Приведенные примеры процедур в специализированной топологии охватывают основные функциональные возможности обмена данными и проведения собраний.
  
## <a name="overview-of-the-install-process-for-skype-for-business-server"></a>Общие сведения о процессе установки для Скайп для Business Server

Установка Скайп для Business Server включает в себя множество различных процедур. Процедуры, необходимые для получения Скайп Business Server, в вашей среде зависят от особенностей вашей среде. Например, процедура добавления записи службы доменных имен в приведенном здесь примере потребуется в том случае, если эта служба работает в Windows Server. Если служба доменных имен работает в другой системе, необходимо выполнить соответствующую процедуру для этой системы. Это же относится ко многим другим процедурам из этого раздела.
  
Скайп для Business Server доступна в Standard Edition и Enterprise Edition. Основное различие заключается в том, что Standard Edition не поддерживает возможностей Enterprise Edition, обеспечивающих функции высокой доступности. 
  
Скайп для Business Server является расширенной продукта и процесс установки точное гораздо зависит от определенных обстоятельствах. В этом разделе описывается общие действия, чтобы установить продукт. Тем не менее каждой процедуры может отличаться в зависимости от среды и решения по планированию. Например для небольших организаций одного сервера под управлением Скайп для Business Server Standard Edition может применяться, тогда как большой международной организации может иметь 50 серверов в расположениях по всему миру, выделенным для продукта.
  
> [!NOTE]
> Чтобы узнать о последние накопительные пакеты обновления, посетите страницу [обновления для Скайп для Business Server](https://support.microsoft.com/en-us/kb/3061064). После установки исправлений накопительным пакетом обновления 1 должен выполнять администратор `Update-CsAdminRole` командлета. Этот командлет необходим для доступа к новым командлетам GCP в удаленном режиме с помощью PowerShell.
  
> [!IMPORTANT]
> Примеры процедур в этом разделе относятся к специализированным требованиям в ситуации, когда уже приняты определенные решения. Фактические процедуры, необходимо установить Скайп для Business Server будет очень разные. Используйте процедуры этого раздела только в качестве примера, а не как пошаговое руководство по установке Скайп для Business Server в любой среде. 
  
Начало Скайп Business Server для и выполнение в первый раз состоит из восьми основных этапов. Следует ознакомиться, что примеры процедур в этом разделе не только процедуры, необходимые для установки Скайп для Business Server. Эти восемь шагов позволяют понять процесс в целом и подготовить основные условия для работы. Шаги с 1 по 5 могут выполняться в произвольном порядке. Но шаги 6, 7 и 8 должны выполняться в указанном порядке после шагов с 1 по 5, как показано на схеме. Далее следует описание восьми шагов.
  
![Обзор процесса установки.](../../media/b1a59b39-a7f0-4781-ac4d-2dfef7ca3700.png)
  
- [Установка необходимых компонентов для Скайп для Business Server](install-prerequisites.md) : Установка необходимых компонентов на всех серверах, которые составляют Скайп для топологии Business Server. Обратите внимание на то, что необходимые компоненты не совпадают для всех ролей. К примеру серверы, которые обеспечивают интерфейсных роли имеют набор необходимых компонентов и серверы, которые обеспечивают роль директора использовать другой набор необходимых компонентов. Предварительные документация по планированию для получения дополнительных сведений см.
    
- [Создание общей папки в Скайп для Business Server](create-a-file-share.md) : Создание общей папки, который будет использоваться серверами во всем Скайп для топологии Business Server.
    
- [Установка средств администрирования в Скайп для Business Server](install-administrative-tools.md) : средства администрирования включают построитель топологий и панель управления. По крайней мере на одном сервере в топологии или рабочей станции управления 64-разрядная версии ОС Windows, который поддерживается для Скайп для Business Server необходимо установить средства администрирования.
    
- [Подготовка Active Directory для Скайп для Business Server](prepare-active-directory.md) : Скайп для Business Server работает в тесном сотрудничестве с Active Directory. Необходимо подготовить домен Active Directory для работы с Скайп для Business Server. Это можно сделать с помощью мастера развертывания, и он выполняется только один раз для домена. Так как при этом создаются группы и в домен вносятся необходимые изменения, повторение данного процесса не требуется.
    
- [Создание записей DNS для Скайп для Business Server](create-dns-records.md) : чтобы Скайп для Business Server для правильной работы, должен быть число параметры DNS на месте. Это, чтобы клиенты знать, как получить доступ к службам и серверы знать о друг с другом. Эти параметры только необходимо выполнить один раз для развертывания, так как после того как записи DNS, оно доступно в домене.
    
- [Создания и публикации новой топологии в Скайп для Business Server](create-and-publish-new-topology.md) : перед установкой Скайп для системы Microsoft Business Server на каждом из серверов в топологии необходимо создать топологию и опубликовать его. При создании топологии информация о ней загружается в базу данных центрального хранилища управления. В случае пула Enterprise Edition база данных центрального хранилища управления создается при первой публикации новой топологии. Если это Standard Edition, необходимо запустить процесс подготовить первый сервер Standard Edition с помощью мастера развертывания перед публикацией топологии. Этот процесс предусматривает установку экземпляра экспресс-выпуска SQL Server и создание центрального хранилища управления.
    
- [Установка Скайп для Business Server на серверах в топологии](install-skype-for-business-server.md) : после топологии загружается в центрального хранилища управления и серверов, которые будут выполнять роли, которые известно службе Active Directory, необходимо установить Скайп для системы Microsoft Business Server на каждом из серверов в топологии.
    
- [Проверка топологии в Скайп для Business Server](verify-the-topology.md) : после публикации топологии и Скайп для компонентов системы Business Server, установленных на каждом из серверов в топологии, вы готовы к убедитесь, что топология работает как надо. Включают в себя проверку, что конфигурации распространенных из на все серверы Active Directory, чтобы весь домен известно, что Скайп для предприятий доступно в домене.
