# <a name="create-a-file-share-in-skype-for-business-server"></a>Создание общей папки в Skype для бизнеса Server
 
**Сводка:** Узнайте, как создать общую папку Windows Server при установке Skype для бизнеса Server.
  
Skype для бизнеса Server требуется общий файл, чтобы компьютеры в топологии могли обмениваться файлами. Создание общей папки — это шаг 2 из 8 в процессе установки Skype для бизнеса Server. Вы можете выполнить шаги с 1 по 5 в любом порядке. Однако необходимо выполнить шаги 6, 7 и 8 по порядку и после шагов 1–5, как показано на схеме. Дополнительные сведения о планировании общей папки см. в разделе "Требования к среде [Skype для бизнеса Server](../../plan-your-deployment/requirements-for-your-environment/environmental-requirements.md) или требования к серверу [для Skype для бизнеса Server 2019](../../../SfBServer2019/plan/system-requirements.md).
  
![Обзорная схема.](../../media/e69de059-3040-45ab-9379-1932f9fbb37f.png)
  
## <a name="create-a-basic-file-share"></a>Создание базовой общей папки

В этом разделе описано, как создать базовую общую папку Windows Server. Базовая Windows server поддерживается в Skype для бизнеса Server. Однако она явно не обеспечивает высокий уровень доступности. Для среды с высоким уровнем доступности рекомендуется использовать общую папку распределенной файловой системы (DFS). Дополнительные сведения о общей папке с высоким уровнем доступности и DFS см. в разделе "Планирование высокой доступности и [аварийного восстановления](../../plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md) в Skype для бизнеса Server".
  
> [!NOTE]
> Windows Server 2012 R2 внесли значительные изменения в предоставление служба хранилища решений для общей папки, аналогичной сети SAN, с помощью платформы Windows Server. По сравнению с традиционным устройством на основе SAN решение хранилища Windows Server 2012 R2 может сократить затраты в два раза с минимальным влиянием на производительность. Дополнительные сведения о параметрах общей папки в Windows Server 2012 R2 см. в техническом документе, который можно Windows Server 2012 [R2 служба хранилища](https://download.microsoft.com/download/9/4/A/94A15682-02D6-47AD-B209-79D6E2758A24/Windows_Server_2012_R2_Storage_White_Paper.pdf).

### <a name="create-a-basic-file-share"></a>Создание базовой общей папки

1. Войдите на компьютер, на котором будет размещен общий файл.
    
2. Щелкните правой кнопкой мыши папку, к которую вы планируете поделиться, и выберите **пункт "Свойства"**.
    
3. Выберите вкладку **"Общий** доступ" и нажмите кнопку **"Расширенный общий доступ"**.
    
4. **Щелкните "Поделиться этой папкой"**.
    
5. Выберите пункт **Разрешения**.
    
6. Добавьте **группу** локальных администраторов на сервере, на котором размещен общий файловый ресурс, предоставьте разрешение: полный доступ, изменение и **чтение** прав, а затем нажмите кнопку **"ОК"**.
    
7. Снова **нажмите кнопку** "ОК" и запишите сетевой путь.
    
8. Нажмите **кнопку "** Готово", чтобы закрыть мастер.
    
     ![Вкладка "Общий доступ" для общего доступа к папке.](../../media/78fe8441-dead-43ed-9a04-3c7c8c657c15.png)
  
> [!NOTE]
>Если хранилище файлов размещено в общей папке DFS, будет получено следующее предупреждение:

`Warning: Unable to access share permissions for "\\<domain>\<share>".`

>Это ожидается, если вы не администратор на файловом сервере или если это общий ресурс распределенной файловой системы (DFS). Если разрешения общего ресурса уже настроены, это предупреждение можно игнорировать. Если это новый общий ресурс, дополнительные сведения о настройке разрешений на общий ресурс вручную см. в документации.

>Из-за недоступности доступа к разрешениям общего доступа к общей папке DFS Skype для бизнеса Server не сможет явно задать группы в общей папке. Чтобы Skype для бизнеса Server компоненты могут получить доступ к общей папке с соответствующими разрешениями, убедитесь, что следующие группы RTC добавлены с разрешениями на чтение и изменение уровня общего доступа в дополнение к локальным администраторам с разрешениями на общий доступ с полным доступом.
* RTCHSUniversalServices
* RTCComponentUniversalServices
* RTCUniversalServerAdmins
