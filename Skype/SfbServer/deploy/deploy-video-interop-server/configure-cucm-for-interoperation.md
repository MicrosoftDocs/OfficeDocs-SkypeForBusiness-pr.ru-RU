# <a name="configure-cucm-for-interoperation-with-skype-for-business-server"></a>Настройка КУКМ для взаимодействия с Skype для бизнеса Server
 
**Сводка:** Настройка КУКМ для работы с приложением Skype для бизнеса Server.
  
> [!CAUTION]
> Эта возможность протестирована с помощью диспетчера единой системы обмена сообщениями Cisco (Каллманажер или КУКМ) версии 10,5, использующей настройку каналов только по протоколу TCP. Прежде чем продолжить, проверьте, что среда CUCM соответствует этим критериям. 
  
Описанные здесь параметры предназначены только для примера того, как КУКМ можно настроить для работы с ВИС. Такого же результата также можно достичь с помощью других настроек и/или использования альтернативных функциональных возможностей CUCM. Рекомендаций в отношении оптимальной конфигурации для конкретного сценария нет.
  
Для взаимодействия с VIS необходимо подтвердить или настроить несколько параметров CUCM. Выполните описанные ниже процедуры, чтобы не пропустить обязательные настройки.
  
### <a name="configure-the-cucm"></a>Настройка CUCM

1. Войдите в КУКМ и перейдите в раздел Администрирование единого диспетчера подключений Cisco\>— маршрутизация\>звонков-класс элементов\>управления-секций.
    
2. На экране Partition Configuration (Конфигурация раздела) введите имя и описание раздела и нажмите кнопку **Добавить новый**.
    
3. Перейдите в раздел Администрирование единого диспетчера подключений\>Cisco — маршрутизация звонков —\>класс\>модуля поиска, вызывающего элементы управления.
    
4. На экране Calling Search Space Configuration (Конфигурация пространства поиска вызовов) введите имя пространства поиска вызовов и в поле Selected Partitions (Выбранные разделы) введите имя только что созданного раздела. После завершения нажмите кнопку **Сохранить**.
    
5. Перейдите на странице Администрирование единого диспетчера подключений\>Cisco —\>система-\>безопасность магистральной безопасности SIP.
    
6. На экране SIP Trunk Security Profile Configuration (Конфигурация профилей безопасности магистрали SIP) задайте сведения о профиле безопасности магистрали SIP, как показано на рисунке, а затем нажмите кнопку **Добавить новый**.
    
   |**Параметр**|**Рекомендованная настройка**|
   |:-----|:-----|
   |Имя  <br/> |SfBVideoInterop_SecurityProfile  <br/> |
   |Режим безопасности устройства  <br/> |Небезопасно  <br/> |
   |Тип входящего транспорта  <br/> |TCP + UDP  <br/> |
   |Тип исходящего транспорта  <br/> |TCP  <br/> |
   |Входящий порт  <br/> |5060  <br/> |
   
7. Перейдите в раздел Администрирование единого диспетчера подключений\>Cisco —\>параметры устройства —\>профиль SIP.
    
8. На экране SIP Profile Configuration (Конфигурация профиля SIP), задайте параметры в разделе SIP Profile Information (Сведения о профиле SIP), как показано на рисунке. 
    
   |**Параметр**|**Рекомендованная настройка**|
   |:-----|:-----|
   |Имя  <br/> |SfBVideoInterop_SIPProfile  <br/> |
   |Описание  <br/> |SfBVideoInterop_SIPProfile  <br/> |
   
9. Прокрутите страницу вниз до раздела сведения о профиле SDP. Параметру **SDP Session-level Bandwidth Modifier for Early Offer and Re-invites** (Модификатор пропускной способности уровня сеансов SDP для ранних предложений и повторных приглашений) по умолчанию задано значение "TIAS and AS" (TIAS и AS). Измените значение этого параметра на "TIAS only" (Только TIAS). Если вы оставите этот параметр по умолчанию, в Skype для бизнеса Server не будет распознаваться информация об модификаторе пропускной способности в сообщении SIP. TIAS означает Transport Independent Application Specific (для приложений, не зависящих от транспорта), а AS — Application Specific (для приложений). Эти параметры SIP описаны в RFC3890.
    
10. На том же экране прокрутите содержимое дальше. В соответствии с конфигурацией канала профиля SIP выберите **Поддержка раннего предложения для голосовых и видеозвонков** , а затем установите флажок **обязательная (при необходимости вставить MTP)** . Это позволит CUCM настроить исходящий вызов SIP с ранним предложением. В CUCM 8.5 и более поздних версиях есть новая возможность — поддержка настройки исходящих вызовов с ранним предложением без необходимости в оконечной точке мультимедиа (MTP).
    
11. Проверьте, что в разделе SIP Options ping (Проверка связи для параметров SIP) установлен флажок "Enable OPTIONS Ping to monitor destination status for Trunks with Service Type 'None (Default)'" (Разрешить команде проверки связи для параметров отслеживать состояние назначения для магистралей с типом службы "Нет (по умолчанию)").
    
12. По завершении нажмите кнопку **Добавить новый**.
    
13. Перейдите в раздел Администрирование единого диспетчера подключений\>Cisco —\>магистральное устройство. 
    
14. Задайте протокол устройства для SIP и нажмите кнопку **Далее**.
    
15. В разделе Device Information (Информация об устройстве) задайте имя и описание устройства (вероятно, что-то наподобие SfBVideoInterop_SIPTrunk) и в списке групп мультимедийных ресурсов выберите список с нужными метаданными ресурсами. 
    
16. Прокрутите содержимое дальше. Точка завершения мультимедиа (MTP) не требуется для видеозвонков, если он еще не установлен, снимите его. Установите флажок для **запуска на всех активных узлах единого диспетчера подключений**. Обратите внимание на то, что вы должны добавить все узлы КУКМ в конфигурацию Skype для бизнеса Server.
    
17. Прокрутите содержимое дальше. Задайте настройки входящих вызовов и подключенных сторон, как показано.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    |Пространство поиска вызовов  <br/> |CSS_SfBVideoInterop  <br/> |
    |Пространство поиска вызовов AAR  <br/> |CSS_SfBVideoInterop  <br/> |
    |CSS преобразования подключенных сторон  <br/> |CSS_SfBVideoInterop  <br/> |
   
18. Продолжайте прокрутку. В разделе Назначение информации SIP в конфигурации магистральной магистрали SIP укажите полное доменное имя ВИС пула или IP-адрес отдельного сервера ВИС в пуле (добавляя несколько записей). В разделе "Порт назначения" укажите порт, через который на сервере видеовзаимодействия прослушиваются подключения из CUCM (по умолчанию: 6001). Укажите также профиль безопасности для линии связи SIP и ранее созданный профиль SIP, как показано в таблице.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    |Профиль безопасности магистрали SIP  <br/> |SfBVideoInterop_SecurityProfile  <br/> |
    |Переадресация пространства поиска вызовов AAR  <br/> |CSS_SfBVideoInterop  <br/> |
    |Ссылка на пространство поиска вызовов извне диалогового окна  <br/> |CSS_SfBVideoInterop  <br/> |
    |Пространство поиска вызовов подписи  <br/> |CSS_SfBVideoInterop  <br/> |
    |Профиль SIP  <br/> |SfBVideoInterop_SIPProfile  <br/> |
    |Метод подачи сигналов DTMF  <br/> |RFC 2833  <br/> |
   
19.  Прокрутите содержимое дальше. Задайте информацию о записи в соответствии со своей системой. Вы можете оставить значение **None**. 
    
20. По завершении нажмите кнопку **Добавить новый**.
    
21. Перейдите в раздел Администрирование единого диспетчера подключений\>Cisco — маршрутизация звонков —\>шаблон\>маршрута/слежения-Route.
    
22. На экране Route Pattern Configuration (Конфигурация шаблона маршрута) введите параметры определения шаблона, показанные ниже. Прокрутите содержимое до раздела Called Party Transformations (Преобразования вызываемой стороны) и задайте маску, как показано. После завершения нажмите кнопку **Добавить новый**.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    |Шаблон маршрута  <br/> |7779999  <br/> |
    |Раздел маршрута  <br/> |SfBVideoInterop_RoutePartition  <br/> |
    |Описание  <br/> |Раздел SfBVideoInterop  <br/> |
    |Список шлюзов/маршрутов  <br/> |SfBVideoInterop_SIPTrunk  <br/> |
    |Маска преобразования вызывающей стороны  <br/> |+14257779999  <br/> |
   
23. Перейдите к разделу Администрирование единого диспетчера\>подключений Cisco —\>маршрутизация звонков-схема маршрута SIP.
    
24. На экране SIP Route Pattern Configuration (Конфигурация шаблона маршрута SIP) задайте параметры определения шаблона и нажмите кнопку **Добавить новый**.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    | Использование шаблона <br/> |Маршрутизация маршрута  <br/> |
    |Шаблон IPv4  <br/> |contoso.com (оставьте пустым, если используется IPv6)  <br/> |
    |Шаблон IPv6  <br/> |contoso.com (оставьте пустым, если используется IPv4)  <br/> |
    |Описание  <br/> |Шаблон SIPRoute для mediarv  <br/> |
    |Раздел маршрута  <br/> |SfBVideoInterop_RoutePartition  <br/> |
    |Список магистралей/маршрутов SIP  <br/> |SfBVideoInterop_SIPTrunk  <br/> |
    |Флажок Block Pattern (Блокировать шаблон)  <br/> |оставьте невыбранным  <br/> |
   
25. Если для скорости передачи аудио- и видеоданных в битах были изменены значения по умолчанию, их потребуется восстановить. Чтобы установить частоту дискретизации для голосовой и видеосвязи, перейдите на вкладку Администрирование единого диспетчера подключений\>Cisco —\>\>регион, в котором находятся данные системного региона. Для справки ниже приведены значения по умолчанию.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    |Регионы  <br/> |По умолчанию  <br/> |
    |Список предпочитаемых аудиокодеков  <br/> |Системные настройки по умолчанию  <br/> |
    |Максимальная скорость звукового потока  <br/> |64 кбит/с (G.722, G.711)  <br/> |
    |Максимальная скорость передачи для видеовызовов во время сеанса  <br/> |200000 кбит/с  <br/> |
    |Максимальная скорость передачи во время сеанса  <br/> |2000000000 кбит/с  <br/> |
   
В данный момент видеошлюз CUCM настроен на работу с VIS. Соответствующую настройку потребуется выполнить в каждой системе VTC, которую требуется интегрировать.
> [!NOTE]
> Чтобы улучшить устойчивость, вы можете настроить этот шлюз КУКМ для работы с вторым сервером видеовзаимодействия или ВИС пулом. Дополнительные сведения см. в разделе [Resiliency mechanisms](../../plan-your-deployment/video-interop-server.md#resiliency).
