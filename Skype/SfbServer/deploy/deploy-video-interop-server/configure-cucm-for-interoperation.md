# <a name="configure-cucm-for-interoperation-with-skype-for-business-server"></a>Настройка CUCM для связи со Skype для бизнеса Server
 
**Сводка:** Настройка CUCM для работы со Skype для бизнеса Server.
  
> [!CAUTION]
> Эта возможность тестирована с помощью Cisco Unified Communications Manager (CallManager или CUCM) версии 10.5 с помощью настройки магистралей только по TCP. Перед тем как ировать, убедитесь, что среда CUCM соответствует этим критериям. 
  
Описанные здесь параметры предназначены только в качестве примеров настройки CUCM для работы с VIS. Другие параметры и/или использование альтернативных функций CUCM также можно использовать для достижения такого же результата. Рекомендации по оптимальной конфигурации для конкретного сценария не подразумеваются.
  
Для работы с VIS необходимо подтвердить или изменить ряд параметров CUCM. Выполните процедуры ниже, чтобы избежать отсутствующих необходимых параметров.
  
### <a name="configure-the-cucm"></a>Настройка CUCM

1. Войдите в CUCM и перейдите к разделу Cisco Unified CM Administration- \> Call Routing- \> Class of \> Control-Partition.
    
2. На экране "Конфигурация раздела" введите имя и описание раздела и нажмите кнопку **"Добавить новый".**
    
3. Перейдите к cisco Unified CM Administration- \> Call Routing- \> Class of \> Control-Calling Search Space.
    
4. На экране "Конфигурация пространства поиска вызовов" введите имя пространства поиска вызовов, а в области "Выбранные разделы" введите имя только что созданного раздела. Нажмите **кнопку** "Сохранить", когда все будет готово.
    
5. Перейдите в веб-сайт Cisco Unified CM Administration- \> \> System- \> Security-SIP Trunk Security Profile.
    
6. На экране "Настройка профиля безопасности магистрали SIP" установите параметры сведений о профиле безопасности магистрали SIP, как показано ниже, и нажмите кнопку **"Добавить новый".**
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   |Name  <br/> |SfBVideoInterop_SecurityProfile  <br/> |
   |Режим безопасности устройства  <br/> |Небезопасная  <br/> |
   |Тип входящих транспортных средств  <br/> |TCP + UDP  <br/> |
   |Тип исходятого транспорта  <br/> |TCP  <br/> |
   |Входящий порт  <br/> |5060  <br/> |
   
7. Перейдите в веб-сайт Cisco Unified CM Administration- \> Device- \> Device Settings- \> SIP Profile.
    
8. На экране "Конфигурация профиля SIP" установите параметры сведений о профиле SIP, как показано ниже. 
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   |Name  <br/> |SfBVideoInterop_SIPProfile  <br/> |
   |Описание  <br/> |SfBVideoInterop_SIPProfile  <br/> |
   
9. На том же экране прокрутите вниз до раздела "Сведения профиля SDP". **Модификатор** пропускной способности уровня сеанса SDP для раннего предложения и повторного приглашения по умолчанию имеет значение TIAS и AS. Измените этот параметр только на TIAS. Если оставить этот параметр в параметре по умолчанию, Skype для бизнеса Server не будет понимать сведения о модификаторе пропускной способности в сообщении SIP. TIAS означает, что независимое от транспорта приложение зависит от приложения, а AS — конкретное приложение. Это параметры SIP, указанные в RFC3890.
    
10. На том же экране прокрутите вниз. В конфигурации магистрали профиля SIP  выберите поддержку раннего предложения для голосовых и видеозвонков и установите для него параметр "Обязательный (при необходимости вставить **MTP)".** Это позволит CUCM настроить исходяющий вызов SIP с ранним предложением. Одна из новых функций в CUCM 8.5 и более новых функциях — поддержка настройки исходяющих вызовов с ранним предложением без необходимости использования точки терминации мультимедиа (MTP).
    
11. Убедитесь, что в разделе "Проверка связи параметров SIP" рядом с полем "Включить проверку параметров связи для отслеживания состояния назначения для магистральных магистрали с типом службы "Нет (по умолчанию)".
    
12. По завершению нажмите кнопку **"Добавить новый".**
    
13. Перейдите к cisco Unified CM Administration- \> \> Device-Trunk. 
    
14. Установите для протокола устройства SIP и нажмите **кнопку "Далее".**
    
15. В области "Сведения об устройстве" установите имя и описание устройства (возможно, на SfBVideoInterop_SIPTrunk) и установите для списка групп ресурсов мультимедиа MRGL, который содержит нужные ресурсы мультимедиа. 
    
16. Прокрутите вниз. Точка терминации мультимедиа (MTP) не требуется для видеовызовов, если она еще не снята, ее необходимо ото снятие. Проверьте возможность запуска **на всех активных узлах объединенных cm.** Обратите внимание, что в конфигурацию Skype для бизнеса Server необходимо добавить все узлы CUCM.
    
17. Прокрутите вниз. Установите параметры параметров входящего вызова и подключенной стороны, как показано ниже.
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    |Пространство поиска вызовов  <br/> |CSS_SfBVideoInterop  <br/> |
    |Пространство поиска вызовов AAR  <br/> |CSS_SfBVideoInterop  <br/> |
    |CSS преобразования подключенных сторон  <br/> |CSS_SfBVideoInterop  <br/> |
   
18. Прокрутите вниз. В разделе "Назначение сведений SIP" конфигурации магистрали SIP укажите FQDN пула VIS или IP-адрес отдельных серверов VIS в пуле (добавление нескольких записей). В пункте "Порт назначения" укажите порт, который viS прослушивает для подключений из CUCM (значение по умолчанию — 6001). Также укажите профиль безопасности магистрали SIP и созданный ранее профиль SIP, как показано ниже.
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    |Профиль безопасности магистрали SIP  <br/> |SfBVideoInterop_SecurityProfile  <br/> |
    |Rerouting Calling Search Space  <br/> |CSS_SfBVideoInterop  <br/> |
    |Пространство поиска вызовов при вызове из диалогового окна  <br/> |CSS_SfBVideoInterop  <br/> |
    |Пространство поиска вызовов подписки  <br/> |CSS_SfBVideoInterop  <br/> |
    |Профиль SIP  <br/> |SfBVideoInterop_SIPProfile  <br/> |
    |Метод сигналов DTMF  <br/> |RFC 2833  <br/> |
   
19.  Прокрутите вниз. Установите запись сведений в соответствующем для вашей системы образом. Можно оставить для него **"Нет".** 
    
20. По завершению нажмите кнопку **"Добавить новый".**
    
21. Перейдите к шаблону Cisco Unified CM Administration- \> Call Routing- \> Route/Hunt-Route \> pattern.
    
22. На экране "Конфигурация шаблона маршрута" введите параметры определения шаблона, показанные ниже. Прокрутите страницу вниз до раздела "Преобразования на основе вызванной стороны" и установите маску, как показано ниже, а затем нажмите кнопку **"Добавить** новый" после завершения.
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    |Шаблон маршрута  <br/> |7779999  <br/> |
    |Раздел маршрута  <br/> |SfBVideoInterop_RoutePartition  <br/> |
    |Описание  <br/> |Раздел для SfBVideoInterop  <br/> |
    |Список шлюзов и маршрутов  <br/> |SfBVideoInterop_SIPTrunk  <br/> |
    |Маска преобразования вызванной стороны  <br/> |+14257779999  <br/> |
   
23. Перейдите к cisco Unified CM Administration- \> Call Routing- \> SIP Route Pattern.
    
24. На экране "Настройка шаблона маршрута SIP" установите параметры определения шаблона, как показано ниже, и нажмите кнопку **"Добавить новый".**
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    | Использование шаблона <br/> |Маршрутка домена  <br/> |
    |Шаблон IPv4  <br/> |contoso.com (оставьте пустым при использовании IPv6)  <br/> |
    |Шаблон IPv6  <br/> |contoso.com (оставьте пустым при использовании IPv4)  <br/> |
    |Описание  <br/> |SIPRoute Pattern to mediarv  <br/> |
    |Раздел маршрута  <br/> |SfBVideoInterop_RoutePartition  <br/> |
    |SIP Trunk/Route List  <br/> |SfBVideoInterop_SIPTrunk  <br/> |
    |Блокировка шаблона  <br/> |оставьте невызванным  <br/> |
   
25. Если вы изменили значения битов звука или видео с параметров по умолчанию, вам потребуется вернуть их к значениям по умолчанию. Чтобы настроить скорость аудио- и видеозвонков, перейдите к веб-сайту Cisco Unified CM Administration- \> System- \> Region \> Information- Region. Значения по умолчанию приведены ниже для справки:
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    |Region  <br/> |По умолчанию  <br/> |
    |Список предпочтений аудио кодека  <br/> |Системный по умолчанию  <br/> |
    |Максимальная скорость звукового сигнала  <br/> |64 кбайт/с (G.722, G.711)  <br/> |
    |Максимальная скорость сеанса для видеозвонков  <br/> |200 000 кб/с  <br/> |
    |Максимальная скорость сеанса  <br/> |20000000000 к/с  <br/> |
   
На этом этапе видео шлюз CUCM настроен для работы с VIS. Соответствующую настройку необходимо будет сделать на каждом VTC, который вы хотите интегрировать.
> [!NOTE]
> Для повышения устойчивости может потребоваться настроить этот шлюз CUCM для работы со вторым сервером видеосвязи или пулом VIS. Дополнительные [сведения см. в](../../plan-your-deployment/video-interop-server.md#resiliency) механизмах устойчивости.
