# <a name="configure-cucm-for-interoperation-with-skype-for-business-server"></a>Настройка CUCM для интеропераций с помощью Skype для бизнеса Server
 
**Сводка:** Настройка CUCM для работы с Skype для бизнеса Server.
  
> [!CAUTION]
> Эта возможность тестирована с помощью Cisco Unified Communications Manager (CallManager или CUCM) версии 10.5 с помощью установки магистралей только через TCP. Убедитесь, что среда CUCM соответствует этим критериям перед началом работы. 
  
Описанные здесь параметры предназначены только в качестве примеров настройки CUCM для работы с VIS. Для достижения такого же результата также можно использовать другие параметры и/или использование альтернативных функций CUCM. Рекомендации относительно оптимальной конфигурации для определенного сценария не подразумеваются.
  
Для связи с VIS необходимо подтвердить или изменить ряд параметров CUCM. Выполните ниже процедуры, чтобы не пропустить необходимые параметры.
  
### <a name="configure-the-cucm"></a>Настройка CUCM

1. Войдите в систему CUCM и перейдите в Cisco Unified CM Administration-Call\> Routing-Class\> control-Partition\>.
    
2. На экране конфигурации раздела введите имя и описание раздела и нажмите кнопку **Добавить новое**.
    
3. Перейдите к единому пространству\>\>\> поиска по маршруту администрирования и вызовов администрирования и вызовов Cisco.
    
4. На экране Конфигурация пространства поиска вызовов введите имя пространства поиска вызова, а в выбранных разделах введите имя только что созданного раздела. Нажмите **кнопку Сохранить** , когда это сделано.
    
5. Перейдите в профиль безопасности магистральных магистрали Cisco Unified CM Administration-System-Security-SIP\>\>\>.
    
6. На экране конфигурации профилей безопасности магистрали SIP установите параметры сведений о профиле безопасности магистрали SIP, как показано, и нажмите **кнопку Добавить новое**.
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   |Имя  <br/> |SfBVideoInterop_SecurityProfile  <br/> |
   |Режим безопасности устройств  <br/> |Небезопасная  <br/> |
   |Тип входящих транспортных средств  <br/> |TCP + UDP  <br/> |
   |Исходяющий тип транспорта  <br/> |TCP  <br/> |
   |Входящий порт  <br/> |5060  <br/> |
   
7. Перейдите в профиль Cisco Unified CM Administration-Device-Device\>\> Параметры-SIP\>.
    
8. На экране конфигурации профилей SIP установите параметры сведений о профиле SIP, как показано. 
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   |Имя  <br/> |SfBVideoInterop_SIPProfile  <br/> |
   |Описание  <br/> |SfBVideoInterop_SIPProfile  <br/> |
   
9. На этом же экране прокрутите вниз раздел Сведения о профиле SDP. **Модификатор** пропускной способности сеансов SDP для раннего предложения и повторного приглашения по умолчанию устанавливается для TIAS и AS. Измените этот параметр только на TIAS. Если вы оставите этот параметр в его параметре по умолчанию, Skype для бизнеса Server не будет понимать сведения о модификаторе пропускной способности в сообщении SIP. TIAS означает Transport Independent Application Specific, в то время как AS означает Application Specific. Это параметры SIP, указанные в RFC3890.
    
10. На том же экране прокрутите вниз далее. В рамках специальной конфигурации магистрали профиля SIP выберите  поддержку раннего предложения для голосовых и видеозвонков и установите ее в обязательный (при необходимости **) параметр MTP**. Это позволит CUCM настроить исходяющий вызов SIP с ранним предложением. Одна из новых функций в CUCM 8.5 и далее в том, что она поддерживает установку исходяющих вызовов с ранним предложением без необходимости точки прерывания мультимедиа (MTP).
    
11. Убедитесь, что в разделе PING Параметры SIP поле проверяется рядом с разделом "Включить параметры Ping для мониторинга состояния назначения для магистральных магистрали с помощью типа службы "Нет (по умолчанию)".
    
12. По завершению нажмите кнопку **Добавить новое**.
    
13. Перейдите к Единой службе администрирования и магистрали\>\> CM Cisco. 
    
14. Установите протокол устройства для SIP и нажмите **кнопку Далее**.
    
15. В статье Сведения об устройстве установите имя и описание устройства (возможно, что-то SfBVideoInterop_SIPTrunk) и установите список групп медиаресумов в MRGL, содержащий нужные медиаресумы. 
    
16. Прокрутите вниз. Точка прекращения мультимедиа (MTP) не требуется для видеозвонков, если она еще не снята, ото всех. Проверьте возможность запуска **на всех активных узлах единой системы CM**. Обратите внимание, что необходимо добавить все узлы CUCM в Skype для бизнеса Server конфигурацию.
    
17. Прокрутите вниз. Установите параметры входящие вызовы и подключенные Параметры, как показано.
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    |Вызываемая пространство поиска  <br/> |CSS_SfBVideoInterop  <br/> |
    |Пространство поиска вызова AAR  <br/> |CSS_SfBVideoInterop  <br/> |
    |CSS преобразования подключенных сторон  <br/> |CSS_SfBVideoInterop  <br/> |
   
18. Прокрутите вниз. В разделе SIP Information Destination конфигурации магистрали SIP укажите FQDN пула VIS или IP-адрес отдельных VIS-серверов в пуле (добавление нескольких записей). В порту назначения укажите порт, который VIS прослушивает для подключений из CUCM (по умолчанию — 6001). Также укажите профиль безопасности магистрали SIP и SIP-профиль, созданный ранее, как показано.
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    |Профиль безопасности магистрали SIP  <br/> |SfBVideoInterop_SecurityProfile  <br/> |
    |Rerouting Calling Search Space  <br/> |CSS_SfBVideoInterop  <br/> |
    |Не в диалоговом октаге ссылаться на пространство поиска вызова  <br/> |CSS_SfBVideoInterop  <br/> |
    |Подписка на поиск по вызову  <br/> |CSS_SfBVideoInterop  <br/> |
    |Профиль SIP  <br/> |SfBVideoInterop_SIPProfile  <br/> |
    |Метод сигнализации DTMF  <br/> |RFC 2833  <br/> |
   
19.  Прокрутите вниз. Установите сведения о записи, соответствующие вашей системе. Это нормально, чтобы оставить его за набором **Нет**. 
    
20. По завершению нажмите кнопку **Добавить новое**.
    
21. Перейдите к шаблону Маршрут маршрутов маршрутов и маршрутов хант-маршрутов\> cisco Unified CM Administration-Call.\>\>
    
22. На экране Конфигурация шаблона маршрутов введите параметры определения шаблона, показанные ниже. Прокрутите вниз в раздел "Преобразование партии" и установите маску, как показано, а затем нажмите кнопку **Добавить новое** по завершению.
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    |Шаблон маршрута  <br/> |7779999  <br/> |
    |Раздел маршрутов  <br/> |SfBVideoInterop_RoutePartition  <br/> |
    |Описание  <br/> |Раздел для SfBVideoInterop  <br/> |
    |Список шлюзов и маршрутов  <br/> |SfBVideoInterop_SIPTrunk  <br/> |
    |Вызванная маска преобразования партии  <br/> |+14257779999  <br/> |
   
23. Перейдите к шаблону маршрутного маршрута маршрутов администрирования и\> вызова унифицированных cm-вызовов\> Cisco.
    
24. На экране конфигурации шаблона маршрутов SIP установите параметры определения шаблона, как показано, и нажмите кнопку **Добавить новое**.
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    | Использование шаблонов <br/> |Маршрутивка домена  <br/> |
    |Шаблон IPv4  <br/> |contoso.com (оставьте пустым при использовании IPv6)  <br/> |
    |Шаблон IPv6  <br/> |contoso.com (оставьте пустым при использовании IPv4)  <br/> |
    |Описание  <br/> |Шаблон SIPRoute в mediarv  <br/> |
    |Раздел маршрутов  <br/> |SfBVideoInterop_RoutePartition  <br/> |
    |Список магистральных и маршрутных маршрутов SIP  <br/> |SfBVideoInterop_SIPTrunk  <br/> |
    |Блок-почтовый ящик Pattern  <br/> |оставьте без перепроверки  <br/> |
   
25. Если вы изменили тарифы звукового или видео бита из параметров по умолчанию, вам потребуется вернуть их по умолчанию. Чтобы установить битовую скорость для аудио- и видеозвонков, перейдите в Cisco Unified CM Administration-System-Region\>\> Information-Region\>. По умолчанию показано ниже для справки:
    
    |**Параметр**|**Рекомендуемый параметр**|
    |:-----|:-----|
    |Регион  <br/> |По умолчанию  <br/> |
    |Список предпочтений аудио-кодека  <br/> |По умолчанию системы  <br/> |
    |Максимальная скорость звукового бита  <br/> |64 kbps (G.722, G.711)  <br/> |
    |Максимальная скорость битовых сеансов для видеозвонков  <br/> |200000 kbps  <br/> |
    |Максимальная скорость бита сеанса  <br/> |2000000000 kbps  <br/> |
   
На этом этапе видео шлюз CUCM настроен для работы с VIS. Соответствующая конфигурация должна быть сделана на каждом VTC, который вы хотите интегрировать.
> [!NOTE]
> Чтобы повысить устойчивость, может потребоваться настроить этот шлюз CUCM для работы со вторым пулом Video Interop Server или VIS. [Дополнительные сведения см. в](../../plan-your-deployment/video-interop-server.md#resiliency) дополнительных сведениях о механизмах устойчивости.
