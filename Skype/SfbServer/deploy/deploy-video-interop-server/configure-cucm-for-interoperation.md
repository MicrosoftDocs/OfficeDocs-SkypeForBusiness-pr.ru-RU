# <a name="configure-cucm-for-interoperation-with-skype-for-business-server"></a>Настройка CUCM для взаимодействия с Скайп для Business Server
 
**Сводка:** Настройка CUCM для работы с Скайп для Business Server.
  
> [!CAUTION]
> Эта возможность проверяется с помощью Cisco объединенных коммуникаций диспетчера (CallManager или CUCM) версии 10.5 магистральных линий связи с помощью программы установки по протоколу TCP только. Прежде чем продолжить, проверьте, что среда CUCM соответствует этим критериям. 
  
Параметры, описанные здесь приведены только в качестве примеров того, как можно настроить CUCM для работы с VIS. Такого же результата также можно достичь с помощью других настроек и/или использования альтернативных функциональных возможностей CUCM. Рекомендаций в отношении оптимальной конфигурации для конкретного сценария нет.
  
Для взаимодействия с VIS необходимо подтвердить или настроить несколько параметров CUCM. Выполните описанные ниже процедуры, чтобы не пропустить обязательные настройки.
  
### <a name="configure-the-cucm"></a>Настройка CUCM

1. Выполните вход в систему CUCM и перейдите к Cisco единой CM администрирования -\>маршрутизации вызовов -\>класс из элемента управления -\>раздела.
    
2. На экране Partition Configuration (Конфигурация раздела) введите имя и описание раздела и нажмите кнопку **Добавить новый**.
    
3. Перейдите к единой Cisco CM администрирования -\>звонок маршрутизации -\>класс элемента управления-\>вызов пространство поиска.
    
4. На экране Calling Search Space Configuration (Конфигурация пространства поиска вызовов) введите имя пространства поиска вызовов и в поле Selected Partitions (Выбранные разделы) введите имя только что созданного раздела. После завершения нажмите кнопку **Сохранить**.
    
5. Перейдите к единой Cisco CM администрирования -\>системы -\>безопасности -\>профиль безопасности магистрали SIP.
    
6. На экране SIP Trunk Security Profile Configuration (Конфигурация профилей безопасности магистрали SIP) задайте сведения о профиле безопасности магистрали SIP, как показано на рисунке, а затем нажмите кнопку **Добавить новый**.
    
   |**Параметр**|**Рекомендованная настройка**|
   |:-----|:-----|
   |Имя  <br/> |SfBVideoInterop_SecurityProfile  <br/> |
   |Режим безопасности устройства  <br/> |Небезопасно  <br/> |
   |Тип входящего транспорта  <br/> |TCP + UDP  <br/> |
   |Тип исходящего транспорта  <br/> |TCP  <br/> |
   |Входящий порт  <br/> |5060  <br/> |
   
7. Перейдите к единой Cisco CM администрирования -\>устройства -\>устройства параметры -\>профилей SIP.
    
8. На экране SIP Profile Configuration (Конфигурация профиля SIP), задайте параметры в разделе SIP Profile Information (Сведения о профиле SIP), как показано на рисунке. 
    
   |**Параметр**|**Рекомендованная настройка**|
   |:-----|:-----|
   |Имя  <br/> |SfBVideoInterop_SIPProfile  <br/> |
   |Описание  <br/> |SfBVideoInterop_SIPProfile  <br/> |
   
9. На экране перейдите к разделу сведений профиля SDP. Параметру **SDP Session-level Bandwidth Modifier for Early Offer and Re-invites** (Модификатор пропускной способности уровня сеансов SDP для ранних предложений и повторных приглашений) по умолчанию задано значение "TIAS and AS" (TIAS и AS). Измените значение этого параметра на "TIAS only" (Только TIAS). Если этот параметр, оставьте параметры по умолчанию, Скайп для Business Server не понимают сведения модификатор пропускной способности в SIP-сообщения. TIAS означает Transport Independent Application Specific (для приложений, не зависящих от транспорта), а AS — Application Specific (для приложений). Эти параметры SIP описаны в RFC3890.
    
10. На том же экране прокрутите содержимое дальше. В разделе определенные конфигурации магистрали SIP профилей выберите **Первых обеспечивают поддержку голосовые звонки и видеозвонки** и задайте для него значение параметра **обязательность (insert MTP при необходимости)** . Это позволит CUCM настроить исходящий вызов SIP с ранним предложением. В CUCM 8.5 и более поздних версиях есть новая возможность — поддержка настройки исходящих вызовов с ранним предложением без необходимости в оконечной точке мультимедиа (MTP).
    
11. Проверьте, что в разделе SIP Options ping (Проверка связи для параметров SIP) установлен флажок "Enable OPTIONS Ping to monitor destination status for Trunks with Service Type 'None (Default)'" (Разрешить команде проверки связи для параметров отслеживать состояние назначения для магистралей с типом службы "Нет (по умолчанию)").
    
12. По завершении нажмите кнопку **Добавить новый**.
    
13. Перейдите к единой Cisco CM администрирования -\>устройства -\>линии связи. 
    
14. Задайте протокол устройства для SIP и нажмите кнопку **Далее**.
    
15. В разделе Device Information (Информация об устройстве) задайте имя и описание устройства (вероятно, что-то наподобие SfBVideoInterop_SIPTrunk) и в списке групп мультимедийных ресурсов выберите список с нужными метаданными ресурсами. 
    
16. Прокрутите содержимое дальше. Точка завершения мультимедиа (MTP) не является обязательным для вызовов, видео, если она еще не снимаются, снимите соответствующий флажок. Установите флажок для **запуска на всех активных узлах CM с единой**. Обратите внимание, что следует добавить все узлы CUCM Скайп для конфигурации Business Server.
    
17. Прокрутите содержимое дальше. Задайте настройки входящих вызовов и подключенных сторон, как показано.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    |Пространство поиска вызовов  <br/> |CSS_SfBVideoInterop  <br/> |
    |Пространство поиска вызовов AAR  <br/> |CSS_SfBVideoInterop  <br/> |
    |CSS преобразования подключенных сторон  <br/> |CSS_SfBVideoInterop  <br/> |
   
18. Продолжайте прокрутку. В разделе SIP назначения сведения о конфигурации магистрали SIP укажите полное доменное имя пула VIS или IP-адрес отдельных серверов VIS в пуле (Добавление несколько записей). В разделе "Порт назначения" укажите порт, через который на сервере видеовзаимодействия прослушиваются подключения из CUCM (по умолчанию: 6001). Укажите также профиль безопасности для линии связи SIP и ранее созданный профиль SIP, как показано в таблице.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    |Профиль безопасности магистрали SIP  <br/> |SfBVideoInterop_SecurityProfile  <br/> |
    |Переадресация пространства поиска вызовов AAR  <br/> |CSS_SfBVideoInterop  <br/> |
    |Ссылка на пространство поиска вызовов извне диалогового окна  <br/> |CSS_SfBVideoInterop  <br/> |
    |Пространство поиска вызовов подписи  <br/> |CSS_SfBVideoInterop  <br/> |
    |Профиль SIP  <br/> |SfBVideoInterop_SIPProfile  <br/> |
    |Метод подачи сигналов DTMF  <br/> |RFC 2833  <br/> |
   
19.  Прокрутите содержимое дальше. Задайте информацию о записи в соответствии со своей системой. Дает возможность оставить значение **None**. 
    
20. По завершении нажмите кнопку **Добавить новый**.
    
21. Перейдите к Cisco единой CM администрирования -\>маршрутизации вызовов -\>маршрут/сервисной-\>шаблону маршрута.
    
22. На экране Route Pattern Configuration (Конфигурация шаблона маршрута) введите параметры определения шаблона, показанные ниже. Прокрутите содержимое до раздела Called Party Transformations (Преобразования вызываемой стороны) и задайте маску, как показано. После завершения нажмите кнопку **Добавить новый**.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    |Шаблон маршрута  <br/> |7779999  <br/> |
    |Раздел маршрута  <br/> |SfBVideoInterop_RoutePartition  <br/> |
    |Описание  <br/> |Раздел SfBVideoInterop  <br/> |
    |Список шлюзов/маршрутов  <br/> |SfBVideoInterop_SIPTrunk  <br/> |
    |Маска преобразования вызывающей стороны  <br/> |+14257779999  <br/> |
   
23. Перейдите к единой Cisco CM администрирования -\>звонок маршрутизации -\>шаблону маршрута SIP.
    
24. На экране SIP Route Pattern Configuration (Конфигурация шаблона маршрута SIP) задайте параметры определения шаблона и нажмите кнопку **Добавить новый**.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    | Использование шаблона <br/> |Маршрутизация маршрута  <br/> |
    |Шаблон IPv4  <br/> |contoso.com (оставьте пустым, если используется IPv6)  <br/> |
    |Шаблон IPv6  <br/> |contoso.com (оставьте пустым, если используется IPv4)  <br/> |
    |Описание  <br/> |Шаблон SIPRoute для mediarv  <br/> |
    |Раздел маршрута  <br/> |SfBVideoInterop_RoutePartition  <br/> |
    |Список магистралей/маршрутов SIP  <br/> |SfBVideoInterop_SIPTrunk  <br/> |
    |Флажок Block Pattern (Блокировать шаблон)  <br/> |оставьте невыбранным  <br/> |
   
25. Если для скорости передачи аудио- и видеоданных в битах были изменены значения по умолчанию, их потребуется восстановить. Чтобы задать скорость потока для вызовов, аудио и видео, перейдите к Cisco единой CM администрирования -\>системы -\>область сведений -\>области. Для справки ниже приведены значения по умолчанию.
    
    |**Параметр**|**Рекомендованная настройка**|
    |:-----|:-----|
    |Регионы  <br/> |По умолчанию  <br/> |
    |Список предпочитаемых аудиокодеков  <br/> |Системные настройки по умолчанию  <br/> |
    |Максимальная скорость звукового потока  <br/> |64 кбит/с (G.722, G.711)  <br/> |
    |Максимальная скорость передачи для видеовызовов во время сеанса  <br/> |200000 кбит/с  <br/> |
    |Максимальная скорость передачи во время сеанса  <br/> |2000000000 кбит/с  <br/> |
   
В данный момент видеошлюз CUCM настроен на работу с VIS. Соответствующую настройку потребуется выполнить в каждой системе VTC, которую требуется интегрировать.
> [!NOTE]
> Чтобы повысить устойчивость, может потребоваться настроить этот шлюз CUCM для работы со второй сервер взаимодействия видео или VIS пула. Дополнительные сведения см. в разделе [Resiliency mechanisms](../../plan-your-deployment/video-interop-server.md#resiliency).
