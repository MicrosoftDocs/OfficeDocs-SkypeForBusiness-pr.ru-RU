# <a name="configure-a-vtc-for-interoperation-with-skype-for-business-server"></a>Настройка VTC для связи со Skype для бизнеса Server
 
**Сводка:** Настройте устройства VTC для работы со Skype для бизнеса Server.
  
Необходимо выполнить следующие процедуры настройки конфигурации для каждого VTC, который будет подключаться к серверу VIS Skype для бизнеса через магистраль SIP и видео шлюз Cisco Unified Communications Manager (CallManager или CUCM).
  
Описанные здесь параметры предназначены только в качестве примеров настройки CUCM для работы с VIS. Другие параметры и/или использование альтернативных функций CUCM также можно использовать для достижения такого же результата. Рекомендации по оптимальной конфигурации для конкретного сценария не подразумеваются.
  
### <a name="configure-a-vtc-registered-with-cucm"></a>Настройка VTC, зарегистрированного с помощью CUCM

1. Войдите на устройство Cisco VTC и перейдите к пункту \> Configuration-System Configuration- \> Provisioning.
    
2. Проверьте следующие параметры, исправив их по мере необходимости: 
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   |Режим предоставления  <br/> | CUCM <br/> |
   |Адрес ExternalManager  <br/> | FQDN CUCM <br/> |
   | Домен ExternalManager <br/> |Домен CUCM  <br/> |
   
3. Перейдите к configuration- \> System Configuration- \> Network.
    
4. Проверьте следующие параметры, исправив их по мере необходимости: 
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   |DNS-имя домена  <br/> | Имя домена CUCM <br/> |
   |Адрес DNS-сервера 1  <br/> | нужный адрес DNS-сервера <br/> |
   
5. Перейдите к сайту Configuration- \> System Configuration— \> Network Services. Убедитесь, что режим H.323 отключен и режим SIP включен. 
    
6. Эти параметры устанавливаются автоматически при регистрации конечной точки с помощью CUCM. Проверьте следующие параметры, исправив их по мере необходимости: 
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   |Режим H.323  <br/> | Выкл. <br/> |
   |Режим HTTP  <br/> | Вкл. <br/> |
   | Режим SIP <br/> | Вкл. <br/> |
   |Режим Telnet  <br/> | Вкл. <br/> |
   |WelcomeText  <br/> | Вкл. <br/> |
   |Режим XMLAPI  <br/> | Вкл. <br/> |
   
7. Перейдите к \> configuration-System Configuration- \> SIP.
    
8. Проверьте следующие параметры, исправив их по мере необходимости: 
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   |Профиль 1 — DefaultTransport  <br/> | TCP <br/> |
   |Профиль 1 — исходящие данные  <br/> | Выкл. <br/> |
   |Профиль 1 — TlsVerify  <br/> | Вкл. <br/> |
   |Профиль 1 — тип  <br/> | Cisco <br/> |
   |Профиль 1 — URI  <br/> | Автоматическое присвоение при регистрации CUCM <br/> |
   |Прокси-сервер 1 — адрес  <br/> |Имя хоста CUCM  <br/> |
   
Теперь VTC настроена для межсервейного межсерваторного перенастройки. Перед началом работы службы необходимо выполнить последние действия на стороне CUCM.
### <a name="configure-vtc-devices-on-cucm"></a>Настройка устройств VTC на CUCM

1. Войдите в CUCM и перейдите к cisco Unified CM Administration- \> Device- \> Phone- \> Find. 
    
2. Выберите устройство VTC, которое необходимо настроить. Проверьте следующие параметры на экране "Конфигурация телефона", исправив их по мере необходимости. После изменения или проверки этих параметров нажмите кнопку **"Сохранить".**
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   |Сведения об устройстве — шаблон кнопки телефона  <br/> | Standard Cisco Telepresence Codec C40 <br/> |
   |Сведения об устройстве — общий профиль телефона  <br/> | Стандартный общий профиль телефона <br/> |
   |Сведения об устройстве — пространство поиска вызовов  <br/> | CSS_SfBVideoInterop <br/> |
   |Сведения об устройстве — пространство поиска вызовов AAR  <br/> | CSS_SfBVideoInterop <br/> |
   |Сведения об устройстве — список групп ресурсов мультимедиа  <br/> | MRGL_SfBVideoInterop <br/> |
   |Сведения о протоколе — профиль безопасности устройства  <br/> | Cisco Telepresence Codec C40 <br/> |
   |Сведения о протоколе — перенамеряние пространства поиска вызовов  <br/> | CSS_SfBVideoInterop <br/> |
   |Сведения о протоколе — пространство поиска вызовов SUBSCRIBE  <br/> | CSS_SfBVideoInterop <br/> |
   |Сведения о протоколе - профиль SIP  <br/> | Стандартный профиль SIP для конечной точки телеприсутства <br/> |
   
3. После с сохраненной конфигурации VTC перейдите на экран конфигурации телефона для устройства. В верхней части экрана в группе "Связь" щелкните связь для видеосвязи. При этом будет отсвеяна экран настройки номера каталога. 
    
4. Проверьте следующие параметры, исправив их по мере необходимости: 
    
    Внести соответствующие изменения, как показано в параметрах "Сведения о номере каталога" и "Параметры номера каталога".
    
   |**Параметр**|**Рекомендуемый параметр**|
   |:-----|:-----|
   | Сведения о номере каталога — раздел маршрута <br/> | SfBVideoInterop_RoutePartition <br/> |
   |Параметры номера каталога — пространство поиска вызовов  <br/> | CSS_SfBVideoInterop <br/> |
   |Параметры альтернативной стороны и уровня конфиденциального доступа MLPP — пространство поиска вызовов MLPP  <br/> | CSS_SfBVideoInterop <br/> |
   |Строка 1 на устройстве — отображение (ИД вызываемого)  <br/> | При желании <br/> |
   |Строка 1 на устройстве — asCII Display (Caller ID)  <br/> | При желании <br/> |
   
5. По завершению прокрутите экран до верхней части экрана и нажмите кнопку **"Сохранить".** 
    
Настройка для этого устройства VTC завершена. Вам потребуется повторить этот процесс для других устройств VTC на предприятии.
