# <a name="deploy-skype-connectivity-in-skype-for-business-server"></a>Развертывание функции подключения Skype в Skype для бизнеса Server

**Сводка.** Сведения о том, как подключить Skype для бизнеса Server к Skype для потребителей (функции подключения Skype).
  
В данной статье описывается развертывание функции подключения Skype.
  
## <a name="skype-connectivity-overview-for-it-professionals"></a>Обзор подключения Skype для ИТ-специалистов

Подключение Skype предоставляет пользователям Skype для бизнеса возможность поиска и добавления пользователей Skype. Подключение Skype — это функция Skype для бизнеса, поддерживающая федерацию и поиск в каталогах с пользователями Skype. После включения подключения Skype пользователи Skype для бизнеса смогут выполнять поиск и добавлять пользователей Skype.
  
## <a name="skype-directory-search"></a>Поиск в каталогах Skype

Функциональность поиска в каталогах Skype предоставляет пользователям Skype для бизнеса возможность поиска контактов Skype. Функциональность поиска позволяет пользователям выполнять поиск с помощью следующего:
  
- **поиск по отображаемому имени, например "Иван Воронков",** — этот запрос вернет множество результатов, поэтому можно не найти нужного человека;
    
- **поиск по отображаемому имени и расположению, например "Иван Воронков в Барселоне",** — этот запрос значительно сократит область результатов;
    
- **поиск по адресу электронной почты, например ivanvoronkov@outlook.com,** — этот запрос в большинстве случаев должен вернуть один результат, который в точности соответствует указанному электронному адресу. Но если этот адрес электронной почты связан с несколькими учетными записями, могут быть возвращены несколько результатов;
    
- **поиск по номеру телефона, например "123-123-1234",** — этот запрос в большинстве случаев должен вернуть один результат, который в точности соответствует указанному номеру телефона. Номер телефона должен содержать код страны (т.е. 1-xxx-yyy-zzzz). Если номер телефона связан с несколькими учетными записями, могут быть возвращены несколько результатов;
    
- **поиск по логину в Skype, например "IvanVoronkov1456",** — если найдено точное совпадение, оно будет отображаться первым в списке. Также могут быть возвращены другие возможные совпадения логинов.
    
    > [!NOTE]
    > Поиск в каталоге Skype должен поддерживать связь со следующими IP-адресами через порт 443: 104.40.75.246, 23.101.135.34 и 40.113.86.19.

## <a name="deploying-skype-connectivity-for-skype-for-business-server"></a>Развертывание функции подключения Skype для Skype для бизнеса Server

В Skype для бизнеса Server используется архитектура федеративного доступа для поддержки подключения к Skype. Данное подключение позволяет пользователям Skype для бизнеса Server добавить Skype. Клиенты Skype также могут добавлять пользователей Skype для бизнеса в свой список контактов. На основе административно установленных политик в Skype для бизнеса пользователи смогут взаимодействовать с помощью мгновенных сообщений, просматривать состояние присутствия и начинать голосовые и видеозвонки. Подключение Skype также является функцией Skype для бизнеса Online, которая может быть включена для клиентов Skype для бизнеса Online из центра администрирования Skype для бизнеса на портале Office 365.
  
> [!NOTE]
> Если Skype для бизнеса Server уже настроен на подключение к Windows Messenger с помощью возможности подключения к общедоступным службам обмена мгновенными сообщениями (PIC), ваше развертывание уже настроено на подключение к Skype. Единственное изменение, которое вы, возможно, захотите сделать, — переименование существующей записи Messenger PIC в запись Skype. 
  
### <a name="the-skype-for-business-server-public-im-connectivity-provisioning-site-is-no-longer-available"></a>Сайт предоставления общедоступной службы обмена мгновенными сообщениями Skype для бизнеса Server больше недоступен

Сайт, ранее использовавшийся для ручной подготовки федерации между локальными развертываниями Skype для бизнеса и Skype, больше не требуется, и его работа будет завершена 15.08.2019. Федерация со Skype теперь использует обнаружение федеративных партнеров. Это тот же механизм, который требуется для федерации со Skype для бизнеса Online.

Для взаимодействия между локальным развертыванием Skype для бизнеса и пользователями Skype с помощью существующей инфраструктуры общедоступной службы обмена мгновенными сообщениями теперь требуется, чтобы конфигурация локального пограничного сервера была совместимой со Skype для бизнеса Online.

> [!NOTE]
> Большинству пользователей не требуется выполнять никаких действий, включая все развертывания, создающие федерацию со Skype для бизнеса Online.
  
Для каждого домена, размещаемого в локальных развертываниях, требуется публикация записи SRV DNS федерации. Инструкции см. в разделе [Планирование DNS](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#dns-planning). Каждый домен должен разрешить запрос SRV DNS для полного доменного имени пограничного сервера, соответствующего суффиксу верхнего уровня домена. Например, рассмотрим домен contoso.com:

|**Допустимые полные доменные имена**|**Примечание**|
|:-----|:-----|
|sip.contoso.com   ||
|sipfed.contoso.com   |В каждом случае точное полное доменное имя должно присутствовать в SN или SAN внешнего сертификата, установленного на пограничном сервере.   |
|access.contoso.com   ||
|**Недопустимые полные доменные имена**|**Причина**|
|sip.contoso-edge.com   |Не совпадает с суффиксом.  |
|sip.it.contoso.com   |Не совпадает с суффиксом верхнего уровня.   |

Дополнительные инструкции о внешних сертификатах см. в разделе [Планирование сертификатов](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#certificate-planning).

#### <a name="faqs"></a>Вопросы и ответы

**Почему завершается работа веб-сайта подготовки?**
Механизм подготовки общедоступной службы обмена мгновенными сообщениями (PIC), развернутый в 2006 г., больше не обслуживается и будет остановлен 15.08.2019. Вместо этого в федерации общедоступной службы обмена мгновенными сообщениями используется такая же модель федерации, как в Skype для бизнеса Online, называемая "обнаружением партнеров", при которой локальное развертывание доступно для обнаружения федерацией записей SRV DNS.

**Это изменение означает, что федерация общедоступной службы обмена мгновенными сообщениями устарела?**
Нет. Федерация общедоступной службы обмена мгновенными сообщениями будет поддерживаться в течение многих лет, вероятно, до конца жизненного цикла локального продукта Skype для бизнеса.

**У нашей организации есть гибридная связь (общее адресное пространство) со Skype для бизнеса Online. Это повлияет на нас?**
Нет, так как вы уже используете федерацию со Skype для бизнеса Online, это изменение не повлияет на вас.
 
**Это изменение означает, что нашей организации нужно включить федерацию со Skype для бизнеса Online?**
Нет. Если параметры пограничного прокси-сервера не поддерживают федерацию с поставщиком услуг размещения Skype для бизнеса Online (sipfed.online.lync.com), это изменение не повлияет на это. Однако требования к DNS и сертификату, применяемые к федерации со Skype для бизнеса Online, теперь также применяются к федерации с пользователями Skype.
 
**В нашей большой организации невозможно изменить конфигурацию пограничного сервера из-за норм, обеспечения соответствия требованиям или прочих причин... Что мы можем сделать?**
Все локальные организации, которые не могут изменить конфигурацию пограничного сервера в соответствии с указаниями, должны обратиться в службу поддержки продукта при первой возможности.

### <a name="enabling-federation-and-public-im-connectivity-pic"></a>Включение федерации и возможности подключения к общедоступным службам обмена мгновенными сообщениями (PIC)

Теперь сосредоточимся на подготовке среды Skype для бизнеса Server и выполнении административных задач, необходимых для настройки подключения Skype. В этом разделе предполагается, что администратор развернул Skype для бизнеса Server и настроил внешний доступ, также известный как пограничные серверы. 
  
Три основных этапа, необходимые для включения федерации и PIC:
  
1. настройка федерации и PIC;
    
2. настройка не менее одной политики для поддержки доступа федеративных пользователей;
    
3. настройка параметра поставщика общедоступной службы обмена мгновенными сообщениями Skype.
    
#### <a name="1-configure-federation-and-pic"></a>1. Настройка федерации и PIC

Федерация нужна для того, чтобы разрешить пользователям Skype взаимодействовать с пользователями Skype для бизнеса в своей организации. Общедоступная служба обмена мгновенными сообщениями (PIC) — это класс федерации, в котором для пользователей Skype для бизнеса должно быть настроено разрешение на взаимодействие с пользователями Skype. Федерация и PIC настраиваются с помощью панели управления Skype для бизнеса Server.
  
> [!NOTE]
> Федерация PIC больше не поддерживается в выпусках продуктов до Lync Server 2010 (Live Communication Server, Office Communications Server). Платформы с поддержкой федерации PIC включают Skype для бизнеса Server, Lync Server 2013 и Lync Server 2010. 
  
Федерация нужна для того, чтобы разрешить пользователям Skype взаимодействовать с пользователями Skype для бизнеса в своей организации. Общедоступная служба обмена мгновенными сообщениями (PIC) — это класс федерации, в котором для пользователей Skype для бизнеса Server должно быть настроено разрешение на взаимодействие с пользователями Skype. Федерация и PIC настраиваются с помощью диалогового окна настройки пограничных серверов панели управления Skype для бизнеса Server, как показано на рисунке ниже.
  
![Определение нового пограничного пула](../media/32d7f255-c6ad-426d-96c2-2ef4d81f3b51.png)
  
> [!NOTE]
> Атрибутам EnableSkypeIdRouting и EnableSkypeDirectorySearch необходимо установить значение True в настройках общедоступного поставщика (инструкции см. ниже), чтобы поиск работал должным образом. 
  
Это все административные задачи, которые нужно выполнить на сервере. Подключение Skype настроено.
  
#### <a name="2-configure-at-least-one-policy-to-support-federated-user-access"></a>2. Настройка не менее одной политики для поддержки доступа федеративных пользователей

С помощью панели управления Skype для бизнеса Server администратору необходимо настроить одну или несколько политик доступа внешних пользователей, чтобы управлять совместной работой пользователей Skype и пользователей Skype для бизнеса Server в пределах организации.
  
#### <a name="3-configure-the-skype-pic-provider-setting"></a>3. Настройка параметра поставщика общедоступной службы обмена мгновенными сообщениями Skype

С помощью командной консоли Skype для бизнеса Server администратору необходимо настроить политику клиента Skype для бизнеса Server, чтобы отображать Skype как дополнительного поставщика PIC. 
  
> [!NOTE]
> Пользователи поставщиков общедоступной службы обмена мгновенными сообщениями (PIC) не смогут участвовать в обмене мгновенными сообщениями и конференциях организации, пока не будет настроена хотя бы одна политика поддержки этой службы (действие 2, описанное выше). 
  
Для новых установок подключение Skype можно настроить, включив общедоступного поставщика Skype на панели управления Skype для бизнеса Server, как показано на рисунке ниже.
  
![Федеративные поставщики SIP](../media/8fc7b566-72b5-4c43-961c-9249fdf7e575.png)
  
> [!NOTE]
> Чтобы настроить подключение Skype при обновлении Skype для бизнеса Server, необходимо удалить и повторно добавить существующего общедоступного поставщика Skype. 
  
Настройку подключения Skype также можно выполнить с помощью использования исключительно PowerShell. Настройка подключения Skype с помощью PowerShell:
  
1. На сервере переднего плана Skype для бизнеса Server откройте командную консоль Skype для бизнеса Server.
    
2. Выполните две следующие команды:
    
   ```
    Remove-CsPublicProvider -Identity <identity-name>
   ```

    > [!NOTE]
    > Если в среде еще не установлен поставщик службы PIC и выполняется создание нового поставщика службы PIC, выполнять командлет Remove-CsPublicProvider не требуется. 
  
   ```
   New-CsPublicProvider -Identity Skype -ProxyFqdn federation.messenger.msn.com -IconUrl https://images.edge.messenger.live.com/Messenger_16x16.png -NameDecorationRoutingDomain msn.com -NameDecorationExcludedDomainList "msn.com,outlook.com,live.com,hotmail.com" -Enabled $true -EnableSkypeIdRouting $true -EnableSkypeDirectorySearch $true
   ```

    Функции менее очевидных параметров
    
   - ProxyFqdn — местоположение пограничного сервера федерации со Skype (принадлежит корпорации Майкрософт и управляется ей)
    
   - IconURL — значок используется клиентом Lync и Skype для бизнеса для визуального определения контактов Skype
    
   - NameDecorationRoutingDomain и NameDecorationExcludedDomainList — установка данных параметров позволяет пользователям вводить MSA пользователей Skype без необходимости знать об "оформлении" доменов, не принадлежащих Майкрософт, с помощью msn.com. Это устраняет необходимость ввода "user(contoso.com)@msn.com" для всех доменов, которые НЕ указаны в списке исключенных доменов (ExcludedDomainList). Клиент Skype для бизнеса автоматически форматирует MSA, если домен НЕ указан в списке исключенных доменов. В список исключения добавлены самые распространенные домены учетной записи Майкрософт.
    
     > [!NOTE]
     > При внесении изменений общедоступный поставщик должен быть удален и добавлен заново. Изменения на месте запрещены. 
  
     > [!NOTE]
     > Добавленные в Lync Server 2013 CU5 и клиент рабочего стола Lync в Office 2013 с пакетом обновления 1 (SP1) параметры NameDecorationRoutingDomain и NameDecorationExcludedDomainList исправляют ситуацию, при которой пользователи Lync, добавляющие контакты Skype, должны были "оформить" домены, не принадлежащие Майкрософт, для определения и направления их в Skype (формат: user(contoso.com)@msn.com). Эти новые параметры позволят автоматически выполнять форматирование адреса пользователя, указанного в диалоговом окне "Добавление контактов Skype" с параметром NameDecorationRoutingDomain (для которого должно быть указано значение "msn.com"), если отсутствуют домены из NameDecorationExcludedDomainList (в настоящее время возможна поддержка доменов msn.com, live.com, Hotmail.com, outlook.com). 
  
3. В клиенте Skype для бизнеса пользователям теперь доступен поиск и возможность добавления пользователей Skype.
