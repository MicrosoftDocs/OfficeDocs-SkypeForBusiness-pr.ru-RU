# <a name="deploy-skype-connectivity-in-skype-for-business-server"></a>Развертывание подключения к Skype в Skype для бизнеса Server

**Сводка:** Сведения о том, как подключить сервер Skype для бизнеса к потребителю Skype. Также называется подключением Skype.
  
В данной статье описывается развертывание функции подключения Skype.
  
## <a name="skype-connectivity-overview-for-it-professionals"></a>Обзор подключения Skype для ИТ-специалистов

Подключение к Skype предоставляет пользователям Skype для бизнеса возможность поиска и добавления пользователей Skype. Skype для бизнеса – это функция, которая позволяет вам включить поиск Федерации и каталогов с пользователями Skype. После включения возможности подключения к Skype пользователи Skype для бизнеса смогут находить и добавлять пользователей Skype.
  
## <a name="skype-directory-search"></a>Поиск в каталогах Skype

Функциональность поиска в каталогах Skype предоставляет пользователям Skype для бизнеса возможность поиска контактов Skype. Функциональность поиска позволяет пользователям выполнять поиск с помощью следующего:
  
- **Поиск по отображаемому имени, например "Иван Петров"** — это может привести к получению большого количества результатов, поэтому вы, возможно, не нашли то, что ищете.
    
- **Поиск по отображаемому имени и расположению, например "Иван Петров в Барцелона"** — это позволит значительно сузить результаты поиска.
    
- **Поиск по электронной почте, например "JohnDoe@outlook.com",** в большинстве случаев возвращает один результат; тот, который точно соответствует указанному сообщению электронной почты. Но в случае связи этого адреса эл. почты с несколькими учетными записями, могут быть возвращены несколько результатов;
    
- **Поиск по номеру телефона, например "123-123-1234"** — это должен возвращать один результат в большинстве случаев; именно тот, который соответствует указанному номеру телефона. Номер телефона должен содержать код страны (т.е. 1-xxx-yyy-zzzz). Если номер телефона связан с несколькими учетными записями, могут быть возвращены несколько результатов;
    
- **Поиск по логину Skype, например "JohnDoe1456"** , если найдено точное совпадение, оно будет возвращено в качестве первого результата. Могут возвращаться другие возможные совпадения Name.
    
    > [!NOTE]
    > Поиск в каталоге Skype должен поддерживать связь со следующими IP-адресами через порт 443: 104.40.75.246, 23.101.135.34 и 40.113.86.19.

## <a name="deploying-skype-connectivity-for-skype-for-business-server"></a>Развертывание подключения к Skype для Skype для бизнеса Server

Skype для бизнеса Server использует архитектуру доступа Федерации для поддержки подключения к Skype. Данное подключение позволяет пользователям Skype для бизнеса Server добавить Skype. Клиенты Skype также могут добавлять пользователей Skype для бизнеса в свой список контактов. На основе политик, настроенных администратором в Skype для бизнеса Server, можно обмениваться мгновенными сообщениями, просматривать сведения о присутствии других пользователей, а также запускать голосовые и видеозвонки. Подключение Skype также является функцией Skype для бизнеса Online, которая может быть включена для клиентов Skype для бизнеса Online из центра администрирования Skype для бизнеса на портале Office 365.
  
> [!NOTE]
> Если Skype для бизнеса Server уже настроен на подключение к Windows Messenger с помощью возможности подключения к общедоступным службам обмена мгновенными сообщениями (PIC), ваше развертывание уже настроено на подключение к Skype. Единственное изменение, которое вы, возможно, захотите сделать, — переименование существующей записи Messenger PIC в запись Skype.  
  
### <a name="the-skype-for-business-server-public-im-connectivity-provisioning-site-is-no-longer-available"></a>Сайт подготовки службы подключения к общедоступным службам обмена мгновенными сообщениями в Skype для бизнеса Server больше не доступен

Сайт, который ранее использовался для автоматической подготовки Федерации между локальными развертываниями Skype для бизнеса, и Skype больше не нужен и будет остановлен в 8/15/2019. Интеграция с Skype теперь использует функцию обнаружения федеративного партнера, которая является одним из механизмов, необходимых для интеграции с Skype для бизнеса Online.

Связь между всеми локальными приложениями Skype для бизнеса и пользователями Skype через существующую общедоступную инфраструктуру обмена мгновенными сообщениями теперь требует, чтобы конфигурация локального пограничного сервера была совместима с Skype для бизнеса Online.

> [!NOTE]
> Большинству пользователей, включая все развертывания, которые применяют Skype для бизнеса Online, никаких действий не требуется.
  
Локальное развертывание требуется для публикации SRV-записи Федерации для каждого домена, на котором они размещены. Руководство можно найти в статье [Планирование DNS](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#dns-planning). Каждый домен должен разрешаться DNS SRV-запросом к доменному имени пограничного сервера, удовлетворяющему суффиксу верхнего уровня, соответствующему домену. Например, предположим, что домен "contoso.com":

|**Допустимые полные доменные имена**|**Комментарий**|
|:-----|:-----|
|sip.contoso.com   ||
|sipfed.contoso.com   |В каждом случае точное полное доменное имя должно быть представлено либо в SN, либо в SAN внешнего сертификата, установленного на пограничном сервере.   |
|access.contoso.com   ||
|**Недопустимые полные доменные имена**|**Причина**|
|sip.contoso-edge.com   |Не совпадает суффикс.  |
|sip.it.contoso.com   |Поиск не является суффиксом верхнего уровня.   |

Дополнительные рекомендации по использованию внешних сертификатов можно найти в статье [Планирование сертификатов](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#certificate-planning).

#### <a name="faqs"></a>Вопросы и ответы

**Зачем закрываться веб-сайт для подготовки к работе?**
Механизм подготовки общедоступного обмена мгновенными сообщениями (PIC), развернутый в 2006, больше не поддается обслуживанию и будет остановлен в 8/15/2019. Вместо этого общедоступная служба обмена мгновенными сообщениями предполагает такую же модель Федерации, которая используется в Skype для бизнеса Online, известном как "обнаружение партнеров", так что локальное развертывание является общедоступным по ее DNS SRV-записям Федерации.

**Означает ли это изменение, что служба интеграции с общедоступными СООБЩЕНИЯми устарела?**
№ Общедоступная служба обмена мгновенными сообщениями по-прежнему будет поддерживаться в течение многих лет, возможно, до окончания жизненного цикла продукта Skype для бизнеса.

**Наша компания имеет гибридное отношение (общую адресное пространство) в Skype для бизнеса Online?**
Нет, так как вы уже используете Skype для бизнеса Online, это изменение не повлияет на вас.
 
**Это означает, что наша компания должна включить федерацию в Skype для бизнеса Online?**
№ Если параметры прокси-сервера пограничных серверов не включают Федерацию с поставщиком услуг размещения Skype для бизнеса Online (sipfed.online.lync.com), это изменение не повлияет на этот процесс. Однако те же требования к DNS и сертификатам, которые распространяются на Федерацию с помощью Skype для бизнеса Online, теперь также распространяются на Федерацию с пользователями Skype.
 
**Наша компания является крупной и не может изменить ее конфигурацию в соответствии с нормативными и нормативными соображениями... что можно сделать?**
Любая локальная организация, которая не может изменить конфигурацию пограничного сервера, должна быть указана в разделе Поддержка продукта при первой возможности.

### <a name="enabling-federation-and-public-im-connectivity-pic"></a>Включение федерации и возможности подключения к общедоступным службам обмена мгновенными сообщениями (PIC)

Теперь вы можете сосредоточиться на серверной среде Skype для бизнеса и административных задачах, необходимых для настройки подключения к Skype. В этом разделе предполагается, что администратор развернул Skype для бизнеса Server и настроил внешний доступ, также известный как пограничные серверы. 
  
Три основных этапа, необходимые для включения федерации и PIC:
  
1. настройка федерации и PIC;
    
2. настройка не менее одной политики для поддержки доступа федеративных пользователей;
    
3. настройка параметра поставщика общедоступной службы обмена мгновенными сообщениями Skype.
    
#### <a name="1-configure-federation-and-pic"></a>1. Настройка федерации и PIC

Федерация нужна для того, чтобы разрешить пользователям Skype взаимодействовать с пользователями Skype для бизнеса в своей организации. Общедоступная служба обмена мгновенными сообщениями (PIC) — это класс федерации, в котором для пользователей Skype для бизнеса должно быть настроено разрешение на взаимодействие с пользователями Skype. Федерация и PIC настраиваются с помощью панели управления Skype для бизнеса Server.
  
> [!NOTE]
> Интеграция PIC больше не поддерживается в версиях продуктов, предшествующих Lync Server 2010 (сервер Live Communication Server, Office Communications Server). Поддерживаются следующие платформы для интеграции PIC: Skype для бизнеса Server, Lync Server 2013 и Lync Server 2010. 
  
Федерация нужна для того, чтобы разрешить пользователям Skype взаимодействовать с пользователями Skype для бизнеса в своей организации. Общедоступная служба обмена мгновенными сообщениями (PIC) — это класс федерации, в котором для пользователей Skype для бизнеса Server должно быть настроено разрешение на взаимодействие с пользователями Skype. Федерация и PIC настраиваются с помощью диалогового окна настройки пограничных серверов панели управления Skype для бизнеса Server, как показано на рисунке ниже.
  
![Определение нового пограничного пула](../media/32d7f255-c6ad-426d-96c2-2ef4d81f3b51.png)
  
> [!NOTE]
> Атрибутам EnableSkypeIdRouting и EnableSkypeDirectorySearch необходимо установить значение True в настройках общедоступного поставщика (инструкции см. ниже), чтобы поиск работал должным образом. 
  
Это все административные задачи, которые нужно выполнить на сервере. Подключение Skype настроено.
  
#### <a name="2-configure-at-least-one-policy-to-support-federated-user-access"></a>2. Настройте хотя бы одну политику для поддержки федеративного доступа пользователей.

С помощью панели управления Skype для бизнеса Server администратору необходимо настроить одну или несколько политик доступа внешних пользователей, чтобы управлять совместной работой пользователей Skype и пользователей Skype для бизнеса Server в пределах организации.
  
#### <a name="3-configure-the-skype-pic-provider-setting"></a>3. Настройка параметров поставщика в Skype PIC

С помощью командной консоли Skype для бизнеса Server администратору необходимо настроить политику клиента Skype для бизнеса Server, чтобы отображать Skype как дополнительного поставщика PIC.  
  
> [!NOTE]
> Пользователи поставщиков услуг общедоступной службы обмена мгновенными сообщениями (PIC) не могут принимать участие в СООБЩЕНИЯх и конференциях в вашей организации, пока не настроите хотя бы одну политику (шаг 2, ранее в этой процедуре), чтобы обеспечить поддержку общедоступной службы обмена мгновенными сообщениями. 
  
Для новых установок подключение Skype можно настроить, включив общедоступного поставщика Skype на панели управления Skype для бизнеса Server, как показано на рисунке ниже.
  
![Федеративные поставщики SIP](../media/8fc7b566-72b5-4c43-961c-9249fdf7e575.png)
  
> [!NOTE]
> Чтобы настроить подключение Skype при обновлении Skype для бизнеса Server, необходимо удалить и повторно добавить существующего общедоступного поставщика Skype. 
  
Настройку подключения Skype также можно выполнить с помощью использования исключительно PowerShell. Настройка подключения Skype с помощью PowerShell:
  
1. На сервере переднего плана Skype для бизнеса Server откройте командную консоль Skype для бизнеса Server.
    
2. Выполните две следующие команды:
    
   ```
    Remove-CsPublicProvider -Identity <identity-name>
   ```

    > [!NOTE]
    > Если в среде еще не установлен поставщик службы PIC и выполняется создание нового поставщика службы PIC, выполнять командлет Remove-CsPublicProvider не требуется.  
  
   ```
   New-CsPublicProvider -Identity Skype -ProxyFqdn federation.messenger.msn.com -IconUrl https://images.edge.messenger.live.com/Messenger_16x16.png -NameDecorationRoutingDomain msn.com -NameDecorationExcludedDomainList "msn.com,outlook.com,live.com,hotmail.com" -Enabled $true -EnableSkypeIdRouting $true -EnableSkypeDirectorySearch $true
   ```

    Функции менее очевидных параметров
    
   - ProxyFqdn — местоположение пограничного сервера федерации со Skype (принадлежит корпорации Майкрософт и управляется ей)
    
   - Иконурл: значок, используемый клиентским приложением Lync &amp; Skype для бизнеса для наглядной идентификации контактов Skype
    
   - Намедекоратионраутингдомаин и Намедекоратионексклудеддомаинлист: в этом случае пользователи смогут вводить МСАС пользователей Skype, не зная о "оформления" доменов, не являющихся Майкрософт, с помощью "msn.com". Это избавляет от необходимости вводить "User (contoso. com) @msn. com" для всех доменов, которых нет в Ексклудеддомаинлист. Клиент Skype для бизнеса автоматически форматирует MSA, если домен НЕ указан в списке исключенных доменов. Мы добавили наиболее распространенные домены учетных записей Майкрософт в список исключенных.
    
     > [!NOTE]
     > При внесении изменений общедоступный поставщик должен быть удален и добавлен заново. Изменения на месте запрещены. 
  
     > [!NOTE]
     > Добавлена в Lync Server 2013 CU5 &amp; Lync для настольных компьютеров в Office 2013 с пакетом обновления 1 (SP1), Намедекоратионраутингдомаин и намедекоратионексклудеддомаинлист улучшают ситуацию, когда пользователи Lync добавляют контакты Skype, необходимые для «украшения» доменов, не являющихся Microsoft Определите и отправьте их в Skype (формат: User (contoso. com) @msn. com). Эти новые параметры позволят автоматически форматировать вводимые пользователем адреса в диалоговом окне "Добавление контакта Skype" с помощью Намедекоратионраутингдомаин (который должен иметь значение msn.com), если он не содержит домены в Намедекоратионексклудеддомаинлист ( в настоящее время мы можем поддерживать msn.com, live.com, Hotmail.com, outlook.com). 
  
3. В клиенте Skype для бизнеса пользователям теперь доступен поиск и возможность добавления пользователей Skype.
