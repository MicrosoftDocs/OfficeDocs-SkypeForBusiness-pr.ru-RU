# <a name="deploy-skype-connectivity-in-skype-for-business-server"></a>Развертывание Skype подключения в Skype для бизнеса Server

**Сводка:** Узнайте, как подключить Skype для бизнеса Server с Skype потребителем. Также называется Skype подключения.
  
В этой статье рассматривается развертывание Skype подключения.
  
## <a name="skype-connectivity-overview-for-it-professionals"></a>Skype по подключению для ИТ-специалистов

Skype подключения предоставляет Skype для бизнеса пользователям возможность поиска и добавления Skype пользователей. Skype подключения — это функция Skype для бизнеса, которая позволяет включить поиск федерации и каталогов с Skype пользователями. После включения Skype подключения Skype для бизнеса пользователи смогут искать и добавлять Skype пользователей.
  
## <a name="skype-directory-search"></a>Skype поиска в каталоге

Skype поиска по каталогам Skype для бизнеса пользователям возможность поиска Skype контактов. Функция поиска позволяет пользователям выполнять поиск с помощью следующих средств:
  
- **Поиск по отображаемого имени, например "John Doe". Это** может возвращать много результатов, поэтому вы можете не найти то, что ищете.
    
- **Поиск по отображаемого имени и расположению, например "John Doe in Веб"** . Это значительно сузит результаты поиска.
    
- **Поиск по электронной почте, пример "johndoe@outlook.com" —** в большинстве случаев должен возвращаться один результат; значение, точно соответствующее указанному сообщению электронной почты. Но если одно и то же сообщение связано с несколькими учетными записями, может быть возвращено несколько результатов.
    
- **Поиск по номеру телефона, например "123-123-1234"** . В большинстве случаев он должен возвращать один результат. Тот, который точно соответствует указанному телефону. Телефон номер должен содержать код страны (например, 1-xxx-yyyy-zzzz). Если один и тот же номер телефона связан с несколькими учетными записями, может быть возвращено несколько результатов.
    
- **Поиск по Skype, например "JohnDoe1456**". Если найдено точное совпадение, оно будет возвращено в качестве первого результата. Могут быть возвращены другие возможные совпадения "name".
    
    > [!NOTE]
    > Skype службы поиска каталогов должны иметь возможность взаимодействовать со следующими IP-адресами через порт 443: 104.40.75.246, 23.101.135.34 и 40.113.86.19.

## <a name="deploying-skype-connectivity-for-skype-for-business-server"></a>Развертывание Skype подключения для Skype для бизнеса Server

Skype для бизнеса Server использует архитектуру доступа к федерации для поддержки подключения к Skype. Это подключение позволяет вашим Skype для бизнеса Server пользователям добавлять Skype. Skype клиенты также могут добавлять Skype для бизнеса пользователей в список контактов. В зависимости от политик, заданных в Skype для бизнеса Server пользователи смогут обмениваться мгновенными сообщениями, просматривать сведения о присутствии друг друга и инициировать аудио- и видеозвонки. Skype подключения также является функцией Skype для бизнеса Online и может быть включена для Skype для бизнеса Online из центра администрирования Skype для бизнеса в Центр администрирования Microsoft 365.
  
> [!NOTE]
> Если Skype для бизнеса Server уже настроено для подключения к Windows Messenger с помощью общедоступного подключения к обмену мгновенными сообщениями (PIC), развертывание уже настроено для подключения Skype. Единственным изменением, которое вы можете учесть, является переименование существующей записи PIC Messenger как Skype. 
  
### <a name="the-skype-for-business-server-public-im-connectivity-provisioning-site-is-no-longer-available"></a>Сайт Skype для бизнеса Server подключения к общедоступному обмену мгновенными сообщениями больше не доступен

Сайт, который ранее использовался для подготовки федерации между локальными развертываниями Skype для бизнеса и Skype, больше не требуется и будет закрыт 15.08.2019. Федерация с Skype теперь использует федеративное обнаружение партнеров, которое является тем же механизмом, который требуется для федерации с Skype для бизнеса Online.

Для связи между любым локальным развертыванием Skype для бизнеса и Skype через существующую общедоступную инфраструктуру обмена мгновенными сообщениями теперь требуется, чтобы конфигурация локального пограничного сервера была совместима с Skype для бизнеса Online.

> [!NOTE]
> Большинство клиентов не требуют никаких действий, включая все развертывания, которые федерации с Skype для бизнеса Online.
  
Локальные развертывания необходимы для публикации записи SRV DNS федерации для каждого домена, в котором они размещены. Рекомендации доступны при [планировании DNS](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#dns-planning). Каждый домен должен разрешаться запросом DNS SRV к FQDN пограничного сервера, удовлетворяющее суффиксу верхнего уровня домена. Например, рассмотрим домен "contoso.com":

|**Допустимые полные доменные имена**|**Comment**|
|:-----|:-----|
|sip.contoso.com   ||
|sipfed.contoso.com   |В каждом случае точное полное доменное имя должно присутствовать в SN или SAN внешнего сертификата, установленного на пограничном сервере.   |
|access.contoso.com   ||
|**Недопустимые полные доменные имена**|**Причина**|
|sip.contoso-edge.com   |Не соответствует суффиксу.  |
|sip.it.contoso.com   |Не соответствует суффиксу верхнего уровня.   |

Дополнительные рекомендации по внешним сертификатам можно найти в [планировании сертификатов](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#certificate-planning).

#### <a name="faqs"></a>Вопросы и ответы

**Почему завершается работа веб-сайта подготовки?**
Механизм подготовки общедоступных мгновенных сообщений pic.lync.com (PIC), развернутый в 2006 г., больше не поддерживается и будет закрыт 15.08.2019. Вместо этого общедоступная федерация обмена мгновенными сообщениями будет предполагать ту же модель федерации, которая используется Skype для бизнеса Online, которая называется обнаружением партнеров, при этом локальное развертывание является общедоступным для обнаружения его записями DNS SRV федерации.

**Означает ли это изменение, что общедоступная федерация мгновенных сообщений устарела?**
Нет. Общедоступная федерация мгновенных сообщений будет по-прежнему поддерживаться в течение многих лет, Skype для бизнеса до окончания срока действия локального продукта.

**У нашей компании есть гибридная связь (общее адресное пространство) с Skype для бизнеса Online, затронуты ли мы?**
Нет, так как вы уже выполняете федерацию с Skype для бизнеса Online, это изменение не повлияет на вас.
 
**Означает ли это изменение, что наша компания должна включить федерацию с Skype для бизнеса Online?**
Нет. Если параметры прокси-сервера пограничного сервера не обеспечивают федерацию с поставщиком услуг Skype для бизнеса Online (sipfed.online.lync.com), это изменение не повлияет на это. Однако те же требования к DNS и сертификатам, которые применяются к федерации с Skype для бизнеса Online, теперь также применяются к федерации с Skype пользователями.
 
**Наша компания имеет большой размер и не может изменить конфигурацию пограничного сервера из-за нормативных требований, соответствия требованиям и т. д. Что можно сделать?**
Любая локальная организация, которая не может изменить конфигурацию пограничного сервера, как указано выше, должна обратиться в службу поддержки продуктов при возможности.

### <a name="enabling-federation-and-public-im-connectivity-pic"></a>Включение федерации и общедоступного обмена мгновенными сообщениями (PIC)

Теперь сосредоточьтесь на Skype для бизнеса Server среде и административных задачах, необходимых для настройки Skype подключения. В этом разделе предполагается, что администратор развернул Skype для бизнеса Server внешний доступ, также известный как пограничные серверы. 
  
Чтобы включить федерацию и PIC, необходимо выполнить три основных шага. Они перечислены ниже.
  
1. Настройка федерации и PIC
    
2. Настройка по крайней мере одной политики для поддержки федеративного доступа пользователей
    
3. Настройка параметра Skype PIC
    
#### <a name="1-configure-federation-and-pic"></a>1. Настройка федерации и PIC

Федерация необходима, чтобы пользователи Skype взаимодействовать с Skype для бизнеса пользователями в вашей организации. Подключение к общедоступной службе обмена мгновенными сообщениями (PIC) — это класс федерации, который необходимо настроить, чтобы пользователи Skype для бизнеса могли взаимодействовать с Skype пользователями. Федерация и PIC настраиваются с помощью Skype для бизнеса Server панель управления.
  
> [!NOTE]
> Федерация PIC больше не поддерживается выпусками продуктов до Lync Server 2010 (Live Communication Server, Office Communications Server). Поддерживаемые платформы для федерации PIC включают Skype для бизнеса Server, Lync Server 2013 и Lync Server 2010. 
  
Федерация необходима, чтобы пользователи Skype взаимодействовать с Skype для бизнеса пользователями в вашей организации. Подключение к общедоступной службе обмена мгновенными сообщениями (PIC) — это класс федерации, который необходимо настроить, чтобы пользователи Skype для бизнеса Server могли взаимодействовать с Skype пользователями. Федерация и PIC настраиваются с помощью диалогового окна конфигурации Edge Skype для бизнеса Server панель управления, как показано на рисунке.

При этом выполняются административные задачи, которые должны выполняться на сервере. Теперь вы настроены для Skype подключения.
  
#### <a name="2-configure-at-least-one-policy-to-support-federated-user-access"></a>2. Настройка по крайней мере одной политики для поддержки федеративного доступа пользователей

С помощью Skype для бизнеса Server панель управления администратор должен настроить одну или несколько политик доступа внешних пользователей, чтобы контролировать, могут ли пользователи Skype работать совместно с внутренними Skype для бизнеса Server пользователями.
  
#### <a name="3-configure-the-skype-pic-provider-setting"></a>3. Настройка параметра Skype PIC

С помощью Skype для бизнеса Server management Shell администратор должен настроить политику клиента Skype для бизнеса для отображения Skype в качестве дополнительного поставщика PIC. 
  
> [!NOTE]
> Пользователи поставщиков услуг подключения к общедоступному обмену мгновенными сообщениями (PIC) не могут участвовать в мгновенных сообщениях или конференциях в организации, пока вы не настроите хотя бы одну политику (шаг 2, ранее в этой процедуре) для поддержки подключения к общедоступному обмену мгновенными сообщениями. 
  
Для новых установок можно настроить подключение Skype, включив общедоступный поставщик Skype с помощью Skype для бизнеса Server панель управления, как показано на рисунке.
  
![Федеративные поставщики SIP.](../media/8fc7b566-72b5-4c43-961c-9249fdf7e575.png)
  
> [!NOTE]
> Чтобы настроить Skype подключения при обновлении до Skype для бизнеса Server необходимо удалить и повторно добавить существующий Skype общедоступный поставщик. 
  
Настройка Skype подключения также может выполняться только с помощью PowerShell. Чтобы настроить подключение Skype с помощью PowerShell, выполните следующие действия.
  
1. На сервере Skype для бизнеса Server интерфейса откройте Skype для бизнеса Server Management Shell.
    
2. Выполните следующие две команды:
    
   ```powershell
    Remove-CsPublicProvider -Identity <identity-name>
   ```

    > [!NOTE]
    > Если у вас еще нет поставщика PIC в вашей среде и вы создаете новый поставщик PIC, вам не Remove-CsPublicProvider командлет. 
  
   ```powershell
   New-CsPublicProvider -Identity Skype -ProxyFqdn federation.messenger.msn.com -IconUrl https://images.edge.messenger.live.com/Messenger_16x16.png -NameDecorationRoutingDomain msn.com -NameDecorationExcludedDomainList "msn.com,outlook.com,live.com,hotmail.com" -Enabled $true -EnableSkypeIdRouting $true -EnableSkypeDirectorySearch $true
   ```

    Что делать с менее очевидными параметрами?
    
   - ProxyFqdn: расположение Skype федерации (принадлежит или обслуживается корпорацией Майкрософт)
    
   - IconURL: значок, используемый клиентом Lync &amp; Skype для бизнеса для визуальной идентификации Skype контактов
    
   - NameDecorationRoutingDomain и NameDecorationExcludedDomainList: это позволяет пользователям вводить msAs Skype пользователей без необходимости знать о "декорировании" доменов сторонних поставщиков с помощью msn.com. Это устраняет необходимость в вводе "user(contoso.com)@msn.com" для всех доменов, которые не указаны в ExcludedDomainList. Клиент SfB будет автоматически форматировать MSA, если домен отсутствует в списке исключенных. Мы добавили в исключенный список наиболее распространенные домены учетной записи Майкрософт.
    
     > [!NOTE]
     > При внесении изменений общедоступный поставщик должен быть удален и добавлен. Изменения на месте не допускаются. 
  
     > [!NOTE]
     > Добавленные в клиент Lync Server 2013 CU5 &amp; Lync в Office 2013 с пакетом обновления 1 (SP1), nameDecorationRoutingDomain и NameDecorationExcludedDomainList улучшают ситуацию, когда пользователи Lync, добавляя контакты Skype, необходимые для "оформления" доменов сторонних поставщиков для идентификации и маршрутизации их в Skype (формат: user(contoso.com)@msn.com). Эти новые параметры позволяют автоматически форматировать ввод адреса пользователя в диалоговом окне "Добавление контакта Skype" с доменом NameDecorationRoutingDomain (который должен иметь значение msn.com), если он не содержит домены в NameDecorationExcludedDomainList (в настоящее время поддерживается msn.com, live.com, Hotmail.com, outlook.com). 
  
3. Теперь пользователи Skype для бизнеса клиента могут искать и добавлять Skype пользователя.
