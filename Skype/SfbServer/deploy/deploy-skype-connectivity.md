# <a name="deploy-skype-connectivity-in-skype-for-business-server"></a>Развертывание подключения Skype в Skype для бизнеса Server

**Сводка:** Сведения о том, как подключить Skype для бизнеса Server с потребителем Skype. Также называется подключением Skype.
  
В этой статье рассматривается развертывание подключения к Skype.
  
## <a name="skype-connectivity-overview-for-it-professionals"></a>Обзор подключения Skype для ИТ-специалистов

Подключения Skype предоставляют пользователям Skype для бизнеса возможность поиска и добавления пользователей Skype. Подключение Skype — это функция Skype для бизнеса, позволяющая включить функцию Федерации и поиска в каталогах для пользователей Skype. После включения подключения Skype пользователи Skype для бизнеса смогут искать и добавлять пользователей Skype.
  
## <a name="skype-directory-search"></a>Поиск в справочнике Skype

Функции поиска в каталогах Skype предоставляют пользователям Skype для бизнеса возможность поиска контактов Skype. Функция поиска позволяет пользователям выполнять поиск с помощью следующих средств:
  
- **Поиск по отображаемому имени, например "Иван Петров"** , может возвращать множество результатов, поэтому вы можете не найти то, что искали.
    
- **Поиск по отображаемому имени и расположению, например "Иван Петров в Барселоне"** — это позволит значительно сузить результаты поиска.
    
- **Поиск по электронной почте, например "JohnDoe@outlook.com",** в большинстве случаев возвращает один результат; тот, который соответствует указанному адресу электронной почты в точности. Но если один и тот же адрес электронной почты связан с несколькими учетными записями, может быть возвращено несколько результатов.
    
- **Поиск по номеру телефона, например "123-123-1234",** в большинстве случаев возвращается один результат; тот, который точно соответствует указанному телефону. Номер телефона должен содержать код страны (1-XXX-YYY-Зззз). Если один и тот же номер телефона связан с несколькими учетными записями, может быть возвращено несколько результатов.
    
- **Поиск по имени Skype, например "JohnDoe1456"** , если найдено точное совпадение, оно будет возвращено в качестве первого результата. Могут возвращаться другие возможные совпадения "имя".
    
    > [!NOTE]
    > Поиск в справочнике Skype должен иметь возможность связываться со следующими IP-адресами на порте 443:104.40.75.246, 23.101.135.34 и 40.113.86.19.

> [!NOTE]
> Skype для бизнеса Server будет регулировать входящие запросы, если в очереди слишком много ожидающих запросов.

## <a name="deploying-skype-connectivity-for-skype-for-business-server"></a>Развертывание подключения Skype для Skype для бизнеса Server

Skype для бизнеса Server использует архитектуру доступа Федерации для поддержки подключения к Skype. Эта возможность подключения позволяет пользователям Skype для бизнеса Server добавлять Skype. Клиенты Skype также могут добавлять пользователей Skype для бизнеса в свои списки контактов. В соответствии с политиками, настроенными администратором в Skype для бизнеса Server, можно обмениваться мгновенными сообщениями, просматривать сведения о присутствии друг друга и инициировать аудио-и видеовызовы. Подключение Skype также является компонентом Skype для бизнеса Online, и его можно включить для клиентов Skype для бизнеса Online в центре администрирования Skype для бизнеса в центре администрирования Microsoft 365.
  
> [!NOTE]
> Если в Skype для бизнеса Server уже настроено подключение к Windows Messenger с помощью подключения к общедоступным службам обмена мгновенными сообщениями (PIC), развертывание уже настроено для подключения Skype. Единственное изменение, которое вы можете подумать — переименовать существующую запись Messenger PIC как Skype. 
  
### <a name="the-skype-for-business-server-public-im-connectivity-provisioning-site-is-no-longer-available"></a>Сайт подготовки общедоступной службы обмена мгновенными сообщениями Skype для бизнеса Server больше недоступен

Сайт, который ранее использовался для автоматической подготовки Федерации между локальными развертываниями Skype для бизнеса и Skype, больше не нужен и будет выключен по 8/15/2019. Федерация с Skype теперь использует обнаружение федеративного партнера, который является одним и тем же механизмом для Федерации со Skype для бизнеса Online.

Для связи между локальным развертыванием Skype для бизнеса и пользователями Skype через существующую общедоступную инфраструктуру обмена мгновенными сообщениями теперь необходимо, чтобы конфигурация локального пограничного сервера была совместима со Skype для бизнеса Online.

> [!NOTE]
> Большинству клиентов не требуется никаких действий, включая все развертывания, которые Федерацию со Skype для бизнеса Online.
  
Локальные развертывания необходимы для публикации SRV-записи Федерации для каждого домена, который они размещаются. Руководство доступно при [планировании DNS](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#dns-planning). Каждый домен должен разрешаться DNS SRV-запросом к полному доменному имени пограничного сервера, удовлетворяющему суффиксу верхнего уровня, соответствующему домену. Например, рассмотрим домен "contoso.com":

|**Допустимые полные доменные имена**|**Comment**|
|:-----|:-----|
|sip.contoso.com   ||
|sipfed.contoso.com   |В каждом случае полное доменное имя должно присутствовать в SN или в SAN внешнего сертификата, установленного на пограничном сервере.   |
|access.contoso.com   ||
|**Недопустимые полные доменные имена**|**Причина**|
|sip.contoso-edge.com   |Не является соответствующим суффиксом.  |
|sip.it.contoso.com   |Не является соответствующим суффиксом верхнего уровня.   |

Дополнительные рекомендации относительно внешних сертификатов можно найти в статье [Планирование сертификатов](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#certificate-planning).

#### <a name="faqs"></a>Вопросы и ответы

**Почему закрывается веб-сайт подготовки?**
Механизм подготовки общедоступного обмена мгновенными сообщениями (PIC), развернутый в 2006, больше не обслуживается и будет выключен по 8/15/2019. Вместо этого в общедоступной Федерации обмена мгновенными сообщениями подразумевается та же модель Федерации, которая используется в Skype для бизнеса Online, которая называется "обнаружение партнеров", поэтому локальное развертывание общедоступно для своих DNS SRV-записей Федерации.

**Означает ли это изменение, что общедоступная Федерация обмена мгновенными сообщениями является устаревшей?**
Неверно Поддержка Федерации общедоступной службы обмена мгновенными сообщениями будет поддерживаться в течение многих лет, возможно, до завершения работы локального продукта Skype для бизнеса.

**У нашей компании есть гибридное отношение (общее адресное пространство) со Skype для бизнеса Online?**
Нет, так как вы уже вскроете Skype для бизнеса Online, это изменение не повлияет на вас.
 
**Означает ли это изменение, что наша компания должна активировать Федерацию с Skype для бизнеса Online?**
Неверно Если параметры прокси-сервера пограничного сервера не включают Федерацию с поставщиком услуг хостинга Skype для бизнеса Online (sipfed.online.lync.com), это изменение не повлияет на это. Тем не менее, те же требования к DNS и сертификатам, которые применимы к Федерации с Skype для бизнеса Online, теперь также применяются и для Федерации с пользователями Skype.
 
**Наша компания является крупной и не может изменить ее настройку в соответствии с нормативными и нормативными соображениями... что можно делать?**
Любая локальная организация, которая не может изменить конфигурацию пограничного сервера, не должна быть указана в разделе Поддержка продукта при первой возможности.

### <a name="enabling-federation-and-public-im-connectivity-pic"></a>Включение Федерации и общедоступной службы обмена мгновенными сообщениями (PIC)

Теперь необходимо сосредоточиться на среде Skype для бизнеса Server и задачах администрирования, необходимых для настройки подключения к Skype. В этом разделе предполагается, что администратор развернул Skype для бизнеса Server и настроил внешний доступ, также называемый пограничными серверами. 
  
Для включения Федерации и PIC необходимо выполнить три основных действия. Они перечислены ниже.
  
1. Настройка Федерации и PIC
    
2. Настройка по крайней мере одной политики для поддержки федеративного доступа пользователей
    
3. Настройка параметра поставщика Skype PIC
    
#### <a name="1-configure-federation-and-pic"></a>1. Настройка Федерации и PIC

Для предоставления пользователям Skype возможности общаться с пользователями Skype для бизнеса в Организации требуется Федерация. Общедоступная служба обмена мгновенными сообщениями (PIC) — это класс Федерации, который необходимо настроить, чтобы позволить пользователям Skype для бизнеса общаться с пользователями Skype. Федерация и PIC настраиваются с помощью панели управления Skype для бизнеса Server.
  
> [!NOTE]
> Федерация PIC больше не поддерживается в выпусках продуктов, предшествующих Lync Server 2010 (сервер Live Communications, Office Communications Server). Поддерживаемые платформы для Федерации PIC включают Skype для бизнеса Server, Lync Server 2013 и Lync Server 2010. 
  
Для предоставления пользователям Skype возможности общаться с пользователями Skype для бизнеса в Организации требуется Федерация. Общедоступная служба обмена мгновенными сообщениями (PIC) — это класс Федерации, который необходимо настроить, чтобы позволить пользователям Skype для бизнеса Server общаться с пользователями Skype. Федерация и PIC настраиваются с помощью диалогового окна Конфигурация пограничной панели управления Skype для бизнеса Server, как показано на рисунке.
  
![Определение нового пограничного пула](../media/32d7f255-c6ad-426d-96c2-2ef4d81f3b51.png)
  
> [!NOTE]
> Для атрибутов Енаблескипеидраутинг и Енаблескипедиректорисеарч необходимо задать значение true в параметрах общедоступных поставщиков (дополнительные инструкции) для работы поиска. 
  
Это завершает задачи администрирования, которые необходимо выполнить на сервере. Теперь вы можете настроить подключение к Skype.
  
#### <a name="2-configure-at-least-one-policy-to-support-federated-user-access"></a>2. Настройте по крайней мере одну политику для поддержки федеративного доступа пользователей.

С помощью панели управления Skype для бизнеса Server администратор должен настроить одну или несколько политик доступа внешних пользователей, чтобы определить, могут ли пользователи Skype совместно работать с внутренними пользователями Skype для бизнеса Server.
  
#### <a name="3-configure-the-skype-pic-provider-setting"></a>3. Настройка параметра поставщика Skype PIC

С помощью командной консоли Skype для бизнеса Server администратор должен настроить политику клиента Skype для бизнеса для отображения Skype в качестве дополнительного поставщика PIC. 
  
> [!NOTE]
> Пользователи служб подключения к общедоступным службам обмена мгновенными сообщениями не могут участвовать в обмене мгновенными сообщениями или конференциях в Организации, пока не будет настроена хотя бы одна политика (шаг 2, ранее в этой процедуре) для поддержки подключения к общедоступным системам обмена мгновенными сообщения 
  
Для новых установок можно настроить подключение Skype, указав общедоступный поставщик Skype с помощью панели управления Skype для бизнеса Server, как показано на рисунке.
  
![Федеративные поставщики SIP](../media/8fc7b566-72b5-4c43-961c-9249fdf7e575.png)
  
> [!NOTE]
> Чтобы настроить подключение Skype при обновлении до Skype для бизнеса Server, необходимо удалить и повторно добавить существующего общедоступного поставщика Skype. 
  
Настройка подключения Skype также может быть выполнена только с помощью PowerShell. Чтобы настроить подключение Skype с помощью PowerShell:
  
1. На сервере переднего плана Skype для бизнеса Server откройте консоль управления Skype для бизнеса Server.
    
2. Выполните две следующие команды:
    
   ```powershell
    Remove-CsPublicProvider -Identity <identity-name>
   ```

    > [!NOTE]
    > Если у вас еще нет поставщика PIC в среде и создается новый поставщик PIC, выполнять командлет Remove – CsPublicProvider не требуется. 
  
   ```powershell
   New-CsPublicProvider -Identity Skype -ProxyFqdn federation.messenger.msn.com -IconUrl https://images.edge.messenger.live.com/Messenger_16x16.png -NameDecorationRoutingDomain msn.com -NameDecorationExcludedDomainList "msn.com,outlook.com,live.com,hotmail.com" -Enabled $true -EnableSkypeIdRouting $true -EnableSkypeDirectorySearch $true
   ```

    Для чего нужны менее очевидные параметры?
    
   - ProxyFqdn: расположение для Федерации Skype (владельцем/сопровождением Майкрософт)
    
   - IconURL: значок, используемый клиентом &amp; Lync Skype для бизнеса для визуального определения контактов Skype
    
   - Намедекоратионраутингдомаин и Намедекоратионексклудеддомаинлист: параметр позволяет пользователям вводить Skype пользователей МСАС, не требуя знать о том, что домены не из Майкрософт и "msn.com". Это избавляет от необходимости вводить "User (contoso. com) @msn. com" для всех доменов, не включенных в Ексклудеддомаинлист. Клиент SfB будет автоматически форматировать MSA, если домен не включен в список исключенных. Мы добавили наиболее распространенные домены учетных записей Майкрософт в список исключенных.
    
     > [!NOTE]
     > Общедоступный поставщик должен быть удален и добавлен в случае внесения изменений. Изменения на месте не разрешены. 
  
     > [!NOTE]
     > Добавлено в Lync Server 2013 CU5 &amp; для настольных ПК Lync в Office 2013 SP1, Намедекоратионраутингдомаин и намедекоратионексклудеддомаинлист совершенствуют ситуацию, в которой пользователи Lync добавляют контакты Skype, необходимые для "украшения" доменов, отличных от Майкрософт, для идентификации и направления их в Skype (формат: User (contoso. com) @msn. com). Эти новые параметры позволят автоматически форматировать введенные пользователем адреса в диалоговом окне "Добавление контакта Skype" с параметром Намедекоратионраутингдомаин (который должен иметь значение msn.com), если он не содержит домены в Намедекоратионексклудеддомаинлист (в настоящее время поддерживается поддержка msn.com, live.com, Hotmail.com, outlook.com). 
  
3. Теперь пользователи Skype для бизнеса могут найти и добавить пользователя Skype.
