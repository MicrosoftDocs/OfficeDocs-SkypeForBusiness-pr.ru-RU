# <a name="deploy-skype-connectivity-in-skype-for-business-server-2015"></a>Развертывание подключений к Skype в Skype для бизнеса Server 2015
 
**Краткое содержание.** Сведения о том, как подключить Skype для бизнеса Server 2015 к Skype для потребителей (функции подключения Skype).
  
В данной статье описывается развертывание функции подключения Skype.
  
## <a name="skype-connectivity-overview-for-it-professionals"></a>Обзор подключения Skype для ИТ-специалистов

Подключения к Скайп предоставляет Скайп для бизнес-пользователей с возможностью поиска и добавление пользователей Скайп. Подключения к Скайп функция Скайп для бизнеса, которое позволяет включить федерацию и каталог поиска с пользователями Скайп. После включения Скайп подключения к вашей Скайп для бизнес-пользователи смогут поиск и добавление пользователей Скайп.
  
## <a name="skype-directory-search"></a>Поиск в каталогах Skype

Функциональность поиска в каталогах Skype предоставляет пользователям Skype для бизнеса возможность поиска контактов Skype. Функциональность поиска позволяет пользователям выполнять поиск с помощью следующего:
  
- **Поиск по отображаемому имени, пример «Иван Петров»** — это может вернуть много результатов, поэтому вам не удается найти требуется найти.
    
- **Поиск по отображаемое имя, а также расположение, пример «Иван Петров в Барселоне»** — это будет сузить результаты поиска значительно.
    
- **Поиск по адресу электронной почты, пример «johndoe@outlook.com»** — это должна вернуть один результат в большинстве случаев; то, которое точно соответствует указанным электронной почты. Но в случае связи этого адреса эл. почты с несколькими учетными записями, могут быть возвращены несколько результатов;
    
- **Поиск по номеру телефона, пример «123-123-1234»** — это должна вернуть один результат в большинстве случаев; то, которое точно соответствует указанным телефона. Номер телефона должен содержать код страны (т.е. 1-xxx-yyy-zzzz). Если номер телефона связан с несколькими учетными записями, могут быть возвращены несколько результатов;
    
- **Поиск по имени Скайп, пример «JohnDoe1456»** — если точного совпадения, он будет возвращен в качестве первого результата. Другие возможные варианты «имя» может быть возвращено.
    
    > [!NOTE]
    > Поиск в каталоге Skype должен поддерживать связь со следующими IP-адресами через порт 443: 104.40.75.246, 23.101.135.34 и 40.113.86.19.

## <a name="deploying-skype-connectivity-for-skype-for-business-server-2015"></a>Развертывание функции подключения Skype для Skype для бизнеса Server 2015

В Skype для бизнеса Server 2015 используется архитектура федеративного доступа для поддержки подключения к Skype. Данное подключение позволяет пользователям Skype для бизнеса Server добавить Skype. Клиенты Skype также могут добавлять пользователей Skype для бизнеса в свой список контактов. На основе политик административно задание в Скайп для Business Server, пользователи смогут взаимодействовать с помощью мгновенных сообщений, видеть присутствие друг друга и начать аудио-и видео. Подключение Skype также является функцией Skype для бизнеса Online, которая может быть включена для клиентов Skype для бизнеса Online из центра администрирования Skype для бизнеса на портале Office 365.
  
> [!NOTE]
> Если Skype для бизнеса Server уже настроен на подключение к Windows Messenger с помощью возможности подключения к общедоступным службам обмена мгновенными сообщениями (PIC), ваше развертывание уже настроено на подключение к Skype. Единственное изменение, которое вы, возможно, захотите сделать, — переименование существующей записи Messenger PIC в запись Skype.  
  
### <a name="accessing-the-skype-for-business-server-public-im-connectivity-provisioning-site-from-skype-for-business-server-2015"></a>Доступ к сайту предоставления общедоступной службы обмена мгновенными сообщениями Skype для бизнеса Server из Skype для бизнеса Server 2015

Процесс подготовки может занять до 30 дней, но в зависимости от количества запросов это может занять всего несколько дней. Мы рекомендуем начать этот процесс, прежде чем приступать к выполнению следующих действий, описанных в этом документе. По завершении процесса подключения Skype для учетной записи она активируется, и пользователи с соответствующими правами смогут использовать общедоступную службу обмена мгновенными сообщениями.  
  
Для подготовки подключения Skype вам потребуются указанные ниже сведения.
  
- Номер соглашения Майкрософт.
    
- Полное доменное имя службы пограничного доступа.
    
- Домены протокола SIP.
    
- Дополнительные полные доменные имена службы пограничного доступа.
    
- Контактные данные.
    
Запуск процесса подготовки подключения Skype
  
1. Вход на веб-сайт https://pic.lync.com, с помощью своего Microsoft Windows Live ID.
    
2. Выберите тип лицензионного соглашения Майкрософт.
    
3. Установите флажок, подтверждая, что вы прочитали и принимаете права на использование продукта Skype для бизнеса Server.
    
4. На странице "Начало процесса подготовки" щелкните соответствующую ссылку, чтобы начать процесс подготовки.
    
5. На странице "сведения о подготовке" введите полное доменное имя службы пограничного сервера доступа. Например sip.contoso.com.
    
    > [!IMPORTANT]
    > С 1 июля 2017 года для подключения к общедоступным службам обмена мгновенными сообщениями у клиентов также должна быть развернута запись SRV для служб федерации DNS. 
  
6. Введите одно или несколько доменных имен SIP и нажмите кнопку "Добавить".
    
    > [!NOTE]
    > Для выполнения процесса подготовки необходим хотя бы один пограничный сервер доступа. Полное доменное имя пограничного сервера доступа может относиться к нескольким доменам SIP, однако один домен SIP не может быть представлен несколькими полными доменными именами пограничных серверов доступа. Домен SIP и пограничный сервер доступа должны быть активными, функционирующими и достижимыми по сети.              
  
7. В списке "Поставщики общедоступных служб обмена мгновенными сообщениями" выберите пункт "Skype", а затем нажмите "Далее", чтобы добавить контактные данные и отправить запрос на подготовку.
    
После отправки запроса на подготовку может пройти 30 дней (но это может занять и всего несколько дней в зависимости от очереди), прежде чем учетная запись активируется и пользователи смогут использовать подключение к Skype.
  
### <a name="enabling-federation-and-public-im-connectivity-pic"></a>Включение федерации и возможности подключения к общедоступным службам обмена мгновенными сообщениями (PIC)

После отправки запроса на подготовку можно сосредоточиться на подготовке среды Skype для бизнеса Server и выполнении административных задач, необходимых для настройки подключения Skype. В этом разделе предполагается, что администратор развернул Skype для бизнеса Server и настроил внешний доступ, также известный как пограничные серверы.  
  
Три основных этапа, необходимые для включения федерации и PIC:
  
1. настройка федерации и PIC;
    
2. настройка не менее одной политики для поддержки доступа федеративных пользователей;
    
3. настройка параметра поставщика общедоступной службы обмена мгновенными сообщениями Skype.
    
#### <a name="1-configure-federation-and-pic"></a>1. Настройка федерации и PIC

Федерация нужна для того, чтобы разрешить пользователям Skype взаимодействовать с пользователями Skype для бизнеса в своей организации. Общедоступная служба обмена мгновенными сообщениями (PIC) — это класс федерации, в котором для пользователей Skype для бизнеса должно быть настроено разрешение на взаимодействие с пользователями Skype. Федерация и PIC настраиваются с помощью панели управления Skype для бизнеса Server.
  
> [!NOTE]
> Федерация PIC больше не поддерживается в Live Communication Server 2005 с пакетом обновления 1 (SP1) и Office Communications Server 2007. Среди платформ с поддержкой федерации PIC: Skype для бизнеса Server 2015, Lync Server 2013, Lync Server 2010 и Office Communications Server 2007 R2. 	 
  
Федерация нужна для того, чтобы разрешить пользователям Skype взаимодействовать с пользователями Skype для бизнеса в своей организации. Общедоступная служба обмена мгновенными сообщениями (PIC) — это класс федерации, в котором для пользователей Skype для бизнеса Server должно быть настроено разрешение на взаимодействие с пользователями Skype. Федерация и PIC настраиваются с помощью диалогового окна настройки пограничных серверов панели управления Skype для бизнеса Server, как показано на рисунке ниже.

> [!NOTE]

Это все административные задачи, которые нужно выполнить на сервере. Подключение Skype настроено.
  
#### <a name="2-configure-at-least-one-policy-to-support-federated-user-access"></a>2. Настройте хотя бы одну политику для поддержки доступа федеративных пользователей

С помощью панели управления Skype для бизнеса Server администратору необходимо настроить одну или несколько политик доступа внешних пользователей, чтобы управлять совместной работой пользователей Skype и пользователей Skype для бизнеса Server в пределах организации.
  
#### <a name="3-configure-the-skype-pic-provider-setting"></a>3. Настройка параметра поставщика Скайп PIC

С помощью командной консоли Skype для бизнеса Server администратору необходимо настроить политику клиента Skype для бизнеса Server, чтобы отображать Skype как дополнительного поставщика PIC.  
  
> [!NOTE]
> Пользователи поставщиков услуг общедоступных мгновенного обмена сообщениями подключения к (PIC) не может участвовать в обмена мгновенными Сообщениями и конференции в вашей организации только после настройки по крайней мере одна политика (шаг 2, ранее в этой процедуре) для поддержки общедоступной службы обмена Мгновенными сообщениями. 
  
Для новых установок подключение Skype можно настроить, включив общедоступного поставщика Skype на панели управления Skype для бизнеса Server, как показано на рисунке ниже.
  
![Федеративные поставщики SIP](../media/8fc7b566-72b5-4c43-961c-9249fdf7e575.png)
  
> [!NOTE]
> Чтобы настроить подключение Skype при обновлении Skype для бизнеса Server, необходимо удалить и повторно добавить существующего общедоступного поставщика Skype. 
  
Настройку подключения Skype также можно выполнить с помощью использования исключительно PowerShell. Настройка подключения Skype с помощью PowerShell:
  
1. На сервере переднего плана Skype для бизнеса Server откройте командную консоль Skype для бизнеса Server.
    
2. Выполните две следующие команды:
    
   ```
    Remove-CsPublicProvider -Identity <identity-name>
   ```

    > [!NOTE]
    > Если в среде еще не установлен поставщик службы PIC и выполняется создание нового поставщика службы PIC, выполнять командлет Remove-CsPublicProvider не требуется.  
  
   ```
   New-CsPublicProvider -Identity Skype -ProxyFqdn federation.messenger.msn.com -IconUrl https://images.edge.messenger.live.com/Messenger_16x16.png -NameDecorationRoutingDomain msn.com -NameDecorationExcludedDomainList "msn.com,outlook.com,live.com,hotmail.com" -Enabled $true -EnableSkypeIdRouting $true -EnableSkypeDirectorySearch $true
   ```

    Функции менее очевидных параметров
    
  - ProxyFqdn — местоположение пограничного сервера федерации со Skype (принадлежит корпорации Майкрософт и управляется ей)
    
  - IconURL: значок, используемый с Lync &amp; Скайп для клиента Business визуально определить Скайп контактов
    
  - NameDecorationRoutingDomain и NameDecorationExcludedDomainList: задания этих позволяет пользователям вводить MSAs Скайп пользователей без необходимости знать о «оформления» домены не Майкрософт с «msn.com». Это позволяет отказаться от ввода «пользователя (contoso.com) @msn.com» для всех доменов, не входящих в ExcludedDomainList. Клиент Skype для бизнеса автоматически форматирует MSA, если домен НЕ указан в списке исключенных доменов. Мы добавили наиболее распространенных доменов учетную запись Майкрософт в список исключенных.
    
    > [!NOTE]
    > При внесении изменений общедоступный поставщик должен быть удален и добавлен заново. Изменения на месте запрещены. 
  
    > [!NOTE]
    > Добавлена в Lync Server 2013 CU5 &amp; рабочего стола клиента Lync в Office 2013 с пакетом обновления 1, NameDecorationRoutingDomain и NameDecorationExcludedDomainList улучшения ситуации, где контактов для включения Скайп необходимые для пользователей Lync «оформление» домены, не корпорацией Майкрософт Определение и направлять их Скайп (формат: user(contoso.com)@msn.com). Эти новые параметры будут разрешить автоматическое форматирование пользователя адрес enter в диалоговом окне «Добавить контакт Скайп» с NameDecorationRoutingDomain (который должен иметь значение msn.com) Если не содержит домены в NameDecorationExcludedDomainList ( в настоящее время поддерживаемые msn.com live.com Hotmail.com, outlook.com). 
  
3. В клиенте Skype для бизнеса пользователям теперь доступен поиск и возможность добавления пользователей Skype.
