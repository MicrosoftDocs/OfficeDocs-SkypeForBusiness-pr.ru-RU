# <a name="deploy-skype-connectivity-in-skype-for-business-server"></a>Развертывание подключения Skype в Skype для бизнеса Server

**Сводка:** Узнайте, как подключить Skype для бизнеса Server к потребителю Skype. Также называется подключением Skype.
  
Эта статья посвящена развертыванию службы подключения Skype.
  
## <a name="skype-connectivity-overview-for-it-professionals"></a>Обзор подключения Skype для ИТ-специалистов

Подключение Skype предоставляет пользователям Skype для бизнеса возможность поиска и добавления пользователей Skype. Подключение Skype — это функция Skype для бизнеса, которая позволяет включить федерацию и поиск в каталоге с пользователями Skype. После того как вы встроите подключение Skype, пользователи Skype для бизнеса смогут искать и добавлять пользователей Skype.
  
## <a name="skype-directory-search"></a>Поиск в каталоге Skype

Функция поиска в каталогах Skype предоставляет пользователям Skype для бизнеса возможность поиска контактов Skype. Функция поиска позволяет пользователям искать с помощью следующих функций:
  
- **Поиск по отображаемого имени, например "John Doe".** Это может привести к возвращению множества результатов, поэтому вы можете не найти то, что вы ищете.
    
- **Поиск по отображаемого имени и расположению, например "John Doe in Вывод"** — это значительно сузит результаты поиска.
    
- **Поиск по электронной почте, например "johndoe@outlook.com".** В большинстве случаев возвращается один результат; тот, который в точности соответствует указанному сообщению электронной почты. Но если одно и то же сообщение связано с несколькими учетными записьми, может быть возвращено несколько результатов.
    
- **Поиск по номеру телефона, например "123-123-1234".** В большинстве случаев возвращается один результат; тот, который в точности соответствует указанному телефону. Номер телефона должен включать код страны (например, 1-xxx-y-zzzz). Если один и тот же номер телефона связан с несколькими учетными записьми, может быть возвращено несколько результатов.
    
- **Search by Skype Name, example "JohnDoe1456"** - If exact match is found, it will be returned as the first result. Могут быть возвращены другие возможные совпадения с именем.
    
    > [!NOTE]
    > Поиск в каталоге Skype должен взаимодействовать со следующими IP-адресами через порт 443: 104.40.75.246, 23.101.135.34 и 40.113.86.19.

> [!NOTE]
> Если у клиента есть несколько сайтов в локальном развертывании и он развернул только один сервер/пул Skype для бизнеса Server, трафик поиска со всех сайтов будет проходить через один доступный сервер. Администратор должен убедиться, что пулы со всех сайтов могут получать доступ к развернутый сервер/пул Skype для бизнеса Server. 
  
> [!NOTE]
> Служба График Skype будет отлаготировать поисковые запросы от любого локального клиента, клиента Microsoft 365 или Office 365, если скорость запросов превышает 15 запросов в секунду. 
  
> [!NOTE]
> Для крупных корпоративных пользователей в локальной организации домены должны быть исключены из списка в службе поиска Skype, чтобы обеспечить более высокий уровень запросов. 
  
> [!NOTE]
> Skype для бизнеса Server будет отлаговлять входящие запросы, если в очереди слишком много ожидающих запросов.

## <a name="deploying-skype-connectivity-for-skype-for-business-server"></a>Развертывание подключения Skype для Skype для бизнеса Server

Skype для бизнеса Server использует архитектуру доступа федерации для поддержки подключения к Skype. Это подключение позволяет пользователям Skype для бизнеса Server добавлять Skype. Клиенты Skype также могут добавлять пользователей Skype для бизнеса в список контактов. На основе политик, административно установленных в Skype для бизнеса Server, пользователи смогут общаться с помощью обмена мгновенными сообщениями, видеть присутствие друг друга и инициировать аудио- и видеозвонки. Подключение Skype также является функцией Skype для бизнеса Online и может быть включено для клиентов Skype для бизнеса Online из Центра администрирования Skype для бизнеса в Центре администрирования Microsoft 365.
  
> [!NOTE]
> Если Skype для бизнеса Server уже настроен на подключение к Windows Messenger с помощью подключения к общедоступным службам обмена мгновенными сообщениями (PIC), развертывание уже настроено для подключения Skype. Единственное изменение, которые вы можете учесть, — переименовать существующую запись MESSENGER PIC в Skype. 
  
### <a name="the-skype-for-business-server-public-im-connectivity-provisioning-site-is-no-longer-available"></a>Сайт для предоставления общедоступных услуг для подключения к im Skype для бизнеса Server больше не доступен

Сайт, который ранее использовался для ручного предоставления федерации между локального развертывания Skype для бизнеса и Skype, больше не требуется и будет закрыт 15.08.2019. Федерация со Skype теперь использует обнаружение федераированных партнеров, которое является тем же механизмом, что и для федерации со Skype для бизнеса Online.

Для связи между любым локальном развертыванием Skype для бизнеса и пользователями Skype через существующую инфраструктуру общедоступных служб мгновенных сообщений теперь требуется совместимая конфигурация локального сервера Edge Server со Skype для бизнеса Online.

> [!NOTE]
> Большинству клиентов не требуется никаких действий, включая все развертывания, которые федерации со Skype для бизнеса Online.
  
Для публикации SRV-записи DNS федерации для каждого домена, в которых они размещены, требуются локальное развертывание. Руководство доступно при планировании [DNS.](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#dns-planning) Каждый домен должен разрешаться запросом DNS SRV к имени FQDN сервера, которое соответствует суффиксу верхнего уровня домена. Например, рассмотрим домен "contoso.com":

|**Допустимые FQDNs**|**Примечание**|
|:-----|:-----|
|sip.contoso.com   ||
|sipfed.contoso.com   |В каждом случае точное FQDN должно присутствовать в SN или SAN внешнего сертификата, установленного на сервере.   |
|access.contoso.com   ||
|**Недопустимые FQDNs**|**Причина**|
|sip.contoso-edge.com   |Не совпадает с суффиксом.  |
|sip.it.contoso.com   |Не совпадает с суффиксом верхнего уровня.   |

Дополнительные рекомендации по внешним сертификатам можно найти в [планировании сертификатов.](../plan-your-deployment/edge-server-deployments/edge-environmental-requirements.md#certificate-planning)

#### <a name="faqs"></a>Вопросы и ответы

**Почему веб-сайт предоставления закрывается?**
Механизм предоставления общедоступных мгновенных pic.lync.com, развернутый в 2006 г., больше не работает и будет отключен 15.08.2019. Вместо этого для федерации общедоступных служб мгновенных мгновенных данных будет использоваться та же модель федерации, что и в Skype для бизнеса Online, известной как "обнаружение партнеров", при которой локальное развертывание может быть открыто обнаруживаемым с помощью SRV-записей федерации DNS.

**Означает ли это изменение, что федерация общедоступных служб мгновенных мгновенных служб является неподдершенной?**
Нет. Федерация общедоступных служб мгновенных мгновенных услуг будет поддерживаться в течение многих лет, возможно, до тех пор, пока не завершится срок службы локального продукта Skype для бизнеса.

**У нашей компании есть гибридные отношения (общее адресное пространство) со Skype для бизнеса Online, это влияет на нас?**
Нет, так как вы уже федерации со Skype для бизнеса Online, это изменение не повлияет на вас.
 
**Означает ли это изменение, что нашей компании необходимо включить федерацию со Skype для бизнеса Online?**
Нет. Если параметры прокси-сервера не обеспечивают федерацию с поставщиком услуг размещения Skype для бизнеса Online (sipfed.online.lync.com), это изменение не повлияет на это. Однако те же требования К DNS и сертификатам, которые применяются к федерации со Skype для бизнеса Online, теперь также применяются к федерации с пользователями Skype.
 
**Наша компания является крупной и не может изменить свою конфигурацию границы из-за нормативных требований, соответствия требованиям и т. д. ... что можно сделать?**
Любая локальной организации, которая не может изменить конфигурацию своего сервера, как указано, должна при первой возможности связаться с службой поддержки продуктов.

### <a name="enabling-federation-and-public-im-connectivity-pic"></a>Включение федерации и подключения к общедоступным службе im (PIC)

Теперь сосредоточьте внимание на среде Skype для бизнеса Server и административных задачах, необходимых для настройки подключения Skype. В этом разделе предполагается, что администратор развернул Skype для бизнеса Server и настроил внешний доступ, также известный как edge-серверы. 
  
Чтобы включить федерацию и PIC, необходимо три основных шага. Они перечислены ниже.
  
1. Настройка федерации и PIC
    
2. Настройка по крайней мере одной политики для поддержки доступа федератных пользователей
    
3. Настройка параметра поставщика SKYPE PIC
    
#### <a name="1-configure-federation-and-pic"></a>1. Настройка федерации и PIC

Федерация необходима, чтобы позволить пользователям Skype взаимодействовать с пользователями Skype для бизнеса в вашей организации. Подключение к общедоступным службам обмена мгновенными сообщениями (PIC) — это класс федерации, который необходимо настроить, чтобы пользователи Skype для бизнеса могли общаться с пользователями Skype. Федерация и PIC настраиваются с помощью панели управления Skype для бизнеса Server.
  
> [!NOTE]
> Федерация PIC больше не поддерживается выпусками продуктов до Lync Server 2010 (Live Communication Server, Office Communications Server). Поддерживаемые платформы для федерации PIC включают Skype для бизнеса Server, Lync Server 2013 и Lync Server 2010. 
  
Федерация необходима для того, чтобы пользователи Skype могли общаться с пользователями Skype для бизнеса в организации. Подключение к общедоступным службам обмена мгновенными сообщениями (PIC) — это класс федерации, который необходимо настроить, чтобы позволить пользователям Skype для бизнеса Server взаимодействовать с пользователями Skype. Для настройки федерации и PIC используется диалоговое окно конфигурации Edge панели управления Skype для бизнеса Server, как показано на рисунке.
  
![Определение нового пула](../media/32d7f255-c6ad-426d-96c2-2ef4d81f3b51.png)
  
> [!NOTE]
> Для работы поиска атрибуты EnableSkypeIdRouting и EnableSkypeDirectorySearch должны иметь true в параметрах общедоступных поставщиков (см. инструкции далее). 
  
Это выполняет административные задачи, которые должны выполняться на сервере. You are now set up for Skype Connectivity.
  
#### <a name="2-configure-at-least-one-policy-to-support-federated-user-access"></a>2. Настройте по крайней мере одну политику для поддержки доступа федераированных пользователей

С помощью панели управления Skype для бизнеса Server администратору необходимо настроить одну или несколько политик доступа внешних пользователей, чтобы контролировать возможность совместной работы пользователей Skype с внутренними пользователями Skype для бизнеса Server.
  
#### <a name="3-configure-the-skype-pic-provider-setting"></a>3. Настройка параметра поставщика PIC Skype

С помощью оболочки управления Skype для бизнеса Server администратор должен настроить клиентскую политику Skype для бизнеса для отображения Skype в качестве дополнительного поставщика PIC. 
  
> [!NOTE]
> Пользователи поставщиков услуг подключения к общедоступным службам обмена мгновенными сообщениями (PIC) не смогут участвовать в обмене мгновенными сообщениями или конференциях в организации, пока вы не настроите хотя бы одну политику (шаг 2 ранее в этой процедуре) для поддержки подключения к общедоступным службам обмена мгновенными сообщениями. 
  
Для новых установок можно настроить подключение Skype, включив общедоступный поставщик Skype с помощью панели управления Skype для бизнеса Server, как показано на рисунке.
  
![Федеративные поставщики SIP](../media/8fc7b566-72b5-4c43-961c-9249fdf7e575.png)
  
> [!NOTE]
> Чтобы настроить подключение Skype при обновлении до Skype для бизнеса Server, необходимо удалить и повторно добавить существующего поставщика общедоступных служб Skype. 
  
Настройка подключения Skype также может быть сделана с помощью только PowerShell. Настройка подключения Skype с помощью PowerShell:
  
1. На сервере переднего сервера Skype для бизнеса Server откройте оболочку управления Skype для бизнеса Server.
    
2. Запустите следующие две команды:
    
   ```powershell
    Remove-CsPublicProvider -Identity <identity-name>
   ```

    > [!NOTE]
    > Если у вас еще нет поставщика PIC в вашей среде и вы создаете нового поставщика PIC, вам не нужно запускать Remove-CsPublicProvider. 
  
   ```powershell
   New-CsPublicProvider -Identity Skype -ProxyFqdn federation.messenger.msn.com -IconUrl https://images.edge.messenger.live.com/Messenger_16x16.png -NameDecorationRoutingDomain msn.com -NameDecorationExcludedDomainList "msn.com,outlook.com,live.com,hotmail.com" -Enabled $true -EnableSkypeIdRouting $true -EnableSkypeDirectorySearch $true
   ```

    Что делают менее очевидные параметры?
    
   - ProxyFqdn: расположение границы федерации Skype (принадлежит или поддерживается корпорацией Майкрософт)
    
   - IconURL: значок, используемый клиентом Lync Skype для бизнеса для визуального &amp; определения контактов Skype
    
   - NameDecorationRoutingDomain и NameDecorationExcludedDomainList: установка этих параметров позволяет пользователям вводить MSAs пользователей Skype без необходимости знать о "оформлении" доменов, не вложенных в Майкрософт, с помощью "msn.com". Это устраняет необходимость ввести "user(contoso.com)@msn.com" для всех доменов, которые НЕ находятся в ExcludedDomainList. Клиент SfB автоматически форматируется MSA, если домен НЕ находится в списке исключенных. Мы добавили в список исключений наиболее распространенные домены учетных записей Майкрософт.
    
     > [!NOTE]
     > При внесении изменений общедоступный поставщик должен быть удален и добавлен. Изменения на месте не допускаются. 
  
     > [!NOTE]
     > Добавленные в настольный клиент Lync Server 2013 CU5 &amp; Lync в Office 2013 с sp1, nameDecorationRoutingDomain и NameDecorationExcludedDomainList улучшают ситуацию, когда пользователи Lync, добавляя контакты Skype, необходимые для "оформления" доменов, не вложенных в Майкрософт, для идентификации и маршрутизации их в Skype (формат: user(contoso.com)@msn.com). Эти новые параметры позволяют автоматически форматировать ввод адреса пользователя в диалоговом окне "Добавление контакта Skype" с параметром NameDecorationRoutingDomain (для которого следует установить msn.com), если он не содержит домены в списке NameDecorationExcludedDomainList (в настоящее время мы можем поддерживать msn.com, live.com, Hotmail.com, outlook.com). 
  
3. Теперь пользователи клиента Skype для бизнеса могут искать и добавлять пользователей Skype.
