# <a name="configure-skype-for-business-server-to-use-exchange-server-archiving"></a>Настройка Skype для бизнеса Server для использования Exchange Server архива

**Сводка:** Настройте расшифровки мгновенных Exchange Server 2016 или Exchange Server 2013 и Skype для бизнеса Server.

Skype для бизнеса Server позволяет администраторам архивировать расшифровки мгновенных сообщений и веб-конференций в почтовом ящике пользователя Exchange Server 2016 или Exchange Server 2013, а не в SQL Server базе данных. Если выбран этот параметр, запись расшифровок выполняется в папку "Удаленные" в почтовом ящике пользователя. Папка "Удаленные" представляет собой скрытую папку, размещенную в папке корзины. Хотя эта папка не видна конечным пользователям, она индексется поисковой системы Exchange и может быть обнаружена с помощью поиска в почтовых ящиках Exchange и(или) Microsoft SharePoint Server 2013. Поскольку сведения хранятся в той же папке, которая используется функцией хранения In-Place Exchange (отвечая за архивную электронную почту и другие коммуникации Exchange), администраторы могут использовать одно средство для поиска всех электронных коммуникаций, архивных для пользователя.

> [!IMPORTANT]
> Чтобы полностью отключить архивировать беседы, необходимо также отключить историю бесед. Дополнительные сведения см. в следующих темах: управление архивативами внутренних и внешних коммуникаций в Skype для бизнеса [Server,](https://technet.microsoft.com/library/6c2cf941-3204-4f1a-a7e0-416c828056d9.aspx) [New-CsClientPolicy](https://docs.microsoft.com/powershell/module/skype/new-csclientpolicy?view=skype-ps)и [Set-CsClientPolicy.](https://docs.microsoft.com/powershell/module/skype/set-csclientpolicy?view=skype-ps)

Для архивации расшифровок в Exchange Server необходимо начать с настройки проверки подлинности "сервер-сервер" между Skype для бизнеса Server и Exchange Server. После проверки подлинности "сервер-сервер" можно выполнить следующие задачи в Skype для бизнеса Server (обратите внимание, что в зависимости от настройки и конфигурации может не потребоваться выполнение всех этих задач):

1. Включить архивацию Exchange, изменяя параметры конфигурации архивации Skype для бизнеса Server. Это действие является обязательным для всех развертываний.

2. Включите архивацию для внутреннего и/или внешнего обмена данными пользователей. Это действие является обязательным для всех развертываний.

3. Настройте свойство ExchangeArchivingPolicy для каждого из пользователей. Это необходимо, только если Skype для бизнеса Server и Exchange Server находятся в разных лесах.

## <a name="step-1-enabling-exchange-archiving"></a>Шаг 1. Включение архива exchange

Архивация в Skype для бизнеса Server в основном управляется с помощью параметров конфигурации архивации. При установке Skype для бизнеса Server вам автоматически предоставляется одна глобальная коллекция этих параметров. (При желании администраторы могут создавать новые коллекции параметров архива на уровне сайта.) По умолчанию архивная архива не включена в глобальных параметрах и в этих параметрах не включена архивировка Exchange. Чтобы использовать архивацию Exchange, администраторы должны настроить свойства EnableArchiving и EnableExchangeArchiving в этих параметрах конфигурации. Для свойства EnableArchiving можно установить одно из трех возможных значений:

- **Нет.** Архивация отключена. Это значение используется по умолчанию. Если для EnableArchiving установлено "Нет", архивироваться не будет ни в базе данных архива Skype для бизнеса Server, ни в Exchange Server.

- **ImOnly**. Архивируются только расшифровки мгновенных сообщений. Если архивировать Exchange включено, эти записи будут архивироваться в Exchange Server. Если архивировать Exchange отключено, эти записи будут архивироваться в Skype для бизнеса Server.

- **ImAndWebConf**. Архивируются расшифровки мгновенных сообщений и веб-конференций. Если архивировать Exchange включена, эти записи будут архивироваться в Exchange Server. Если архивировать Exchange отключено, эти записи будут архивироваться в Skype для бизнеса Server.

Свойство EnableExchangeArchiving — это boolean value: set EnableExchangeArchiving to True ($True) to enable Exchange archiving or set EnableExchangeArchiving to False ($False) to disable Exchange archiving. Например, эта команда включает архивы расшифровок мгновенных сообщений, а также архивацию Exchange:

```powershell
Set-CsArchivingConfiguration -Identity "global" -EnableArchiving ImOnly -EnableExchangeArchiving $True
```

Чтобы отключить архивировать Exchange, используйте команду, аналогичную следующей, которая позволяет архивировать мгновенные сообщения, но отключает архивировать в Exchange (другими словами, записи будут архивироваться в Skype для бизнеса Server):

```powershell
Set-CsArchivingConfiguration -Identity "global" -EnableArchiving ImOnly -EnableExchangeArchiving $False
```

> [!NOTE]
> Если для свойства EnableArchiving установлено свойство None, Skype для бизнеса Server не будет архивировать расшифровки мгновенных сообщений и веб-конференций. В этом случае сервер просто игнорирует значение, настроенное для EnableExchangeArchiving.

Архивную запись Exchange также можно включить (или отключить) с помощью Skype для бизнеса Server. Для этого выполните следующую процедуру:

1. На панели управления нажмите **Мониторинг и архивация**, затем выберите **Конфигурация архивации**.

2. На вкладке **Конфигурация архивации** дважды щелкните мышью коллекцию настроек архивации, которую необходимо изменить (например, коллекция **Глобальные**).

3. В области **Редактирование настроек архивации** щелкните раскрывающийся список **Настройка архивации** и выберите **Архивировать сеансы обмена мгновенными сообщениями** (архивация только сеансов мгновенных сообщений) или **Архивировать сеансы обмена мгновенными сообщениями и веб-конференции** (архивация сеансов обмена мгновенными сообщениями и веб-конференций).

4. Выбрав элементы для архивации, выберите Exchange Server **интеграции,** чтобы включить архивацию Exchange. Чтобы отключить архивировать Exchange, сключим этот контрольный ящик.

> [!NOTE]
> Флажок **Интеграция с Exchange Server** не отображается, если для параметра **Настройка архивации** установлено значение **Отключить архивацию**. Сначала необходимо включить архив, а затем включить архивы Exchange.

Если Skype для бизнеса Server и Exchange Server расположены в одном лесу, архивативом для отдельных пользователей (или по крайней мере для пользователей с учетными записями электронной почты в Exchange Server) управляется с помощью политик удержания In-Place Exchange. Если у вас есть пользователи, которые находятся в предыдущей версии Exchange, архивом для этих пользователей будет управляться с помощью политик архива Skype для бизнеса Server. Обратите внимание, что архивировать записи Skype для бизнеса в Exchange могут только пользователи с учетными Exchange Server 2016 или Exchange Server 2013.

Если Skype для бизнеса Server и Exchange Server находятся в разных лесах, архивацией для отдельных пользователей управляет настройка свойства ExchangeArchivingPolicy для каждой отдельной учетной записи пользователя. Дополнительные сведения см. в описании шага 3.

## <a name="step-2-enabling-the-archiving-of-internal-andor-external-communications"></a>Шаг 2. Включение архивации внутреннего и/или внешнего обмена данными

После включения архива (и архива exchange) необходимо изменить соответствующие политики архива, чтобы обеспечить фактическую архивировать сеансы пользователей. Обратите внимание, что простое включение архивных данных (шаг 1) не приведет к тому, что Skype для бизнеса Server не начнет архивную запись записей мгновенных сообщений и веб-конференций. Вместо этого необходимо использовать политики архивации, чтобы включить архивацию внутреннего и/или внешнего обмена данными. При установке Skype для бизнеса Server также устанавливается одна глобальная политика архива, которая содержит два свойства:

- **ArchiveInternal**. Если для этого свойства задано значение True ($True), выполняется архивация сеансов внутреннего обмена данными, в которых участвуют только те пользователи, у которых есть учетные записи Active Directory в текущей организации.

- **ArchiveЕхternal**. Если для этого свойства задано значение True ($True), выполняется архивация сеансов внешнего обмена данными, в которых участвует хотя бы один пользователь, у которого нет учетной записи Active Directory в текущей организации.

По умолчанию для обоих свойств установлено значение False, что означает, что ни внутренний, ни внешний обмен данными не архивируется. To modify the global policy, you can use the Skype for Business Server Management Shell and the Set-CsArchivingPolicy cmdlet. Эта команда включает архивацию сеансов как внутреннего, так и внешнего обмена данными.

```powershell
Set-CsArchivingPolicy -Identity "global" -ArchiveInternal $True -ArchiveExternal $True
```

Также можно использовать New-CsArchivingPolicy для создания новой политики в области сайта или в области пользователя. Например, эта команда создает новую индивидуальную политику архивации с именем RedmondArchivingPolicy:

```powershell
New-CsArchivingPolicy -Identity "RedmondArchivingPolicy" -ArchiveInternal $True -ArchiveExternal $True
```

Если создана индивидуальная политика, потребуется назначить ее для соответствующих пользователей. Например:

```powershell
Grant-CsArchivingPolicy -Identity "Ken Myer" -PolicyName  "RedmondArchivingPolicy"
```

Управление политиками архива также можно с помощью панели управления Skype для бизнеса Server. На панели управления нажмите **Мониторинг и архивация**, затем выберите **Политика архивации**. Чтобы изменить существующую политику, дважды щелкните политику (например, "Глобальные"), а затем в области **Редактирование политики архивации** установите или снимите флажки **Архивировать внутренний обмен данным** и **Архивировать внешний обмен данными**. Чтобы создать новую политику архива, нажмите кнопку **"Создать",** а затем выберите политику **"Сайт"** или **"Политика пользователя".** Если создана новая политика пользователя, необходимо перейти к соответствующим пользовательским учетным записям (вкладка **Пользователи**) и назначить для этих пользователей новую политику.

## <a name="step-3-configuring-the-exchangearchivingpolicy-property"></a>Шаг 3. Настройка свойства ExchangeArchivingPolicy

Если Skype для бизнеса Server и Exchange Server находятся в разных лесах, то недостаточно просто включить архивацию Exchange в параметрах конфигурации архивации; это не приведет к архивам расшифровки мгновенных сообщений и веб-конференций в Exchange. Вместо этого необходимо также настроить свойство ExchangeArchivingPolicy для каждой соответствующей учетной записи пользователя Skype для бизнеса Server. Этому свойству можно установить одно из четырех возможных значений:

1. **Неинициализированный**. Указывает, что архивировать будет на основе параметров In-Place удержания, настроенных для почтового ящика Exchange пользователя; Если In-Place в почтовом ящике пользователя не включено удержание, то расшифровки сообщений и веб-конференций будут архивироваться в Skype для бизнеса Server.

2. **UseLyncArchivingPolicy**. Указывает, что расшифровки мгновенных сообщений и веб-конференций пользователя должны архивироваться в Skype для бизнеса Server, а не в Exchange.

3. **NoArchiving**. Указывает, что архивация расшифровок обмена мгновенными сообщениями и веб-конференций для пользователя не выполняется. Обратите внимание, что этот параметр переопределяет все политики архива Skype для бизнеса Server, которые назначены пользователю.

4. **ArchivingToExchange**. Указывает, что расшифровки мгновенных сообщений и веб-конференций пользователя должны архивироваться в Exchange независимо от параметров In-Place удержания, которые были назначены (или не были) назначены почтовому ящику пользователя.

Например, чтобы настроить учетную запись пользователя так, чтобы расшифровки мгновенных сообщений и веб-конференций всегда архивались в Exchange, можно использовать команду, аналогичную этой, из командной оболочки Skype для бизнеса Server:

```powershell
Set-CsUser -Identity "Ken Myer" -ExchangeArchivingPolicy ArchivingToExchange
```

Чтобы настроить ту же политику архивации для группы пользователей (например, для всех пользователей, размещенных в указанном пуле регистратора), можно использовать команду следующего вида:

```powershell
Get-CsUser -Filter {RegistrarPool -eq "atl-cs-001.litwareinc.com"} | Set-CsUser -ExchangeArchivingPolicy ArchivingToExchange
```

Обратите внимание, что для настройки значения свойства ExchangeArchivingPolicy необходимо использовать skype for Business Server Management Shell (и Windows PowerShell). Это свойство не является доступом для администраторов в Skype для бизнеса Server.

Чтобы просмотреть список всех пользователей, для которых назначена конкретная политика архивации, можно использовать команду следующего вида. Эта команда возвращает отображаемые имена Active Directory всех пользователей, для которых свойству ExchangeArchivingPolicy присвоено значение Uninitialized:

```powershell
Get-CsUser | Where-Object {$_.ExchangeArchivingPolicy -eq "Uninitialized"} | Select-Object DisplayName
```

Эта команда также возвращает отображаемые имена пользователей, для которых свойству ExchangeArchivingPolicy не назначено значение UseLyncArchivingPolicy:

```powershell
Get-CsUser | Where-Object {$_.ExchangeArchivingPolicy -ne "UseLyncArchivingPolicy"} | Select-Object DisplayName
```
