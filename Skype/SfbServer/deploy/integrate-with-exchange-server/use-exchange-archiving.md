# <a name="configure-skype-for-business-server-to-use-exchange-server-archiving"></a>Настройка Skype для бизнеса Server для Exchange Server архива

**Сводка:** Настройка стенограмм чата для Exchange Server 2016 или Exchange Server 2013 и Skype для бизнеса Server.

Skype для бизнеса Server предоставляет администраторам возможность архивировать записи мгновенных сообщений и веб-конференций в почтовый ящик пользователя Exchange Server 2016 или Exchange Server 2013 г., а не SQL Server базы данных. Если выбран этот параметр, запись расшифровок выполняется в папку "Удаленные" в почтовом ящике пользователя. Папка "Удаленные" представляет собой скрытую папку, размещенную в папке корзины. Хотя эта папка не видна конечным пользователям, она индексироваться поисковой Exchange и может быть обнаружена с помощью поиска Exchange почтовых ящиков и/или Microsoft SharePoint Server 2013 г. Так как сведения хранятся в одной папке, используемой функцией Exchange In-Place Hold (отвечают за архивировать электронную почту и другие Exchange связи), администраторы могут использовать единый инструмент для поиска всех электронных сообщений, архивных для пользователя.

> [!IMPORTANT]
> Чтобы полностью отключить архивировать беседы, необходимо также отключить историю беседы. Дополнительные сведения см. в следующих темах: Управление архивативией внутренних и внешних коммуникаций в [Skype для бизнеса Server,](/previous-versions/office/lync-server-2013/lync-server-2013-managing-the-archiving-of-internal-and-external-communications) [New-CsClientPolicy](/powershell/module/skype/new-csclientpolicy?view=skype-ps)и [Set-CsClientPolicy](/powershell/module/skype/set-csclientpolicy?view=skype-ps).

Чтобы архивировать записи Exchange Server, необходимо настроить проверку подлинности между сервером и сервером между Skype для бизнеса Server и Exchange Server. После проверки подлинности от сервера к серверу можно выполнить следующие задачи в Skype для бизнеса Server (обратите внимание, что в зависимости от настройки и конфигурации выполнение всех этих задач может не потребоваться):

1. Включить Exchange архивацию, изменяя параметры конфигурации Skype для бизнеса Server архивации. Это действие является обязательным для всех развертываний.

2. Включите архивацию для внутреннего и/или внешнего обмена данными пользователей. Это действие является обязательным для всех развертываний.

3. Настройте свойство ExchangeArchivingPolicy для каждого из пользователей. Этот шаг необходим только в том случае, Skype для бизнеса Server и Exchange Server находятся в разных лесах.

## <a name="step-1-enabling-exchange-archiving"></a>Шаг 1. Включение Exchange архива

Архивация в Skype для бизнеса Server в основном управляется с помощью параметров конфигурации архивации. При установке Skype для бизнеса Server автоматически предоставляется единая глобальная коллекция этих параметров. (Администраторы могут необязательно создавать новые коллекции параметров архива в области сайта.) По умолчанию архивировка не включена в глобальных параметрах и не включена Exchange архива в этих параметрах. Чтобы использовать Exchange архивацию, администраторы должны настроить свойства EnableArchiving и EnableExchangeArchiving в этих параметрах конфигурации. Свойство EnableArchiving можно установить одно из трех возможных значений:

- **Нет**. Архивация отключена. Это значение используется по умолчанию. Если enableArchiving задавалось как None, ничего не будет архивироваться ни в Skype для бизнеса Server, ни в Exchange Server.

- **ImOnly**. Архивируются только расшифровки мгновенных сообщений. Если Exchange архивация включена, эти записи будут архивироваться в Exchange Server. Если Exchange архивация отключена, эти записи будут архивироваться в Skype для бизнеса Server.

- **ImAndWebConf**. Архивируются расшифровки мгновенных сообщений и веб-конференций. Если Exchange архивация включена, эти записи будут архивироваться в Exchange Server. Если Exchange архивация отключена, эти записи будут архивироваться в Skype для бизнеса Server.

Свойство EnableExchangeArchiving — это значение Boolean: установите EnableExchangeArchiving для True ($True), чтобы включить Exchange архивировать или установить EnableExchangeArchiving для false ($False) для отключения Exchange архива. Например, эта команда позволяет архивовать стенограммы обмена мгновенными сообщениями, а также Exchange архивационные работы:

```powershell
Set-CsArchivingConfiguration -Identity "global" -EnableArchiving ImOnly -EnableExchangeArchiving $True
```

Чтобы отключить Exchange архива, используйте команду, аналогичную следующей, которая позволяет архивировать мгновенные сообщения, но отключает архив Exchange (другими словами, записи будут архивироваться в Skype для бизнеса Server):

```powershell
Set-CsArchivingConfiguration -Identity "global" -EnableArchiving ImOnly -EnableExchangeArchiving $False
```

> [!NOTE]
> Если свойство EnableArchiving настроено на None, Skype для бизнеса Server не будет архивировать стенограммы обмена мгновенными сообщениями и веб-конференций. В этом случае сервер просто игнорирует значение, настроенное для EnableExchangeArchiving.

Exchange архива можно также включить (или отключить) с помощью Skype для бизнеса Server. Для этого выполните следующую процедуру:

1. На панели управления нажмите **Мониторинг и архивация**, затем выберите **Конфигурация архивации**.

2. На вкладке **Конфигурация архивации** дважды щелкните мышью коллекцию настроек архивации, которую необходимо изменить (например, коллекция **Глобальные**).

3. В области **Редактирование настроек архивации** щелкните раскрывающийся список **Настройка архивации** и выберите **Архивировать сеансы обмена мгновенными сообщениями** (архивация только сеансов мгновенных сообщений) или **Архивировать сеансы обмена мгновенными сообщениями и веб-конференции** (архивация сеансов обмена мгновенными сообщениями и веб-конференций).

4. После выбора элементов для архивации  выберите Exchange Server интеграции, чтобы включить Exchange архивацию. Чтобы отключить Exchange архива, очистить этот почтовый ящик.

> [!NOTE]
> Флажок **Интеграция с Exchange Server** не отображается, если для параметра **Настройка архивации** установлено значение **Отключить архивацию**. Сначала необходимо включить архивативную и Exchange архива.

Если Skype для бизнеса Server и Exchange Server находятся в одном лесу, архивативка для отдельных пользователей (или по крайней мере для пользователей с учетными записями электронной почты в Exchange Server) управляется с помощью политик Exchange In-Place Hold. Если у вас есть пользователи, которые находятся в предыдущей версии Exchange, архивативка для этих пользователей будет управляться с помощью Skype для бизнеса Server политик архива. Обратите внимание, что только пользователи с учетными записями Exchange Server 2016 или Exchange Server 2013 года могут архивировать Skype для бизнеса для Exchange.

Если Skype для бизнеса Server и Exchange Server находятся в различных лесах, архивативизация для отдельных пользователей управляется настройкой свойства ExchangeArchivingPolicy для каждой отдельной учетной записи пользователя. Дополнительные сведения см. в описании шага 3.

## <a name="step-2-enabling-the-archiving-of-internal-andor-external-communications"></a>Шаг 2. Включение архивации внутреннего и/или внешнего обмена данными

После включения архива (и Exchange архива) необходимо изменить соответствующие политики архива, чтобы убедиться, что сеансы пользователей действительно архивироваться. Обратите внимание, что простое включение архива (шаг 1) не Skype для бизнеса Server приступить к архивам обмена мгновенными сообщениями и стенограммами веб-конференций. Вместо этого необходимо использовать политики архивации, чтобы включить архивацию внутреннего и/или внешнего обмена данными. При установке Skype для бизнеса Server вы также установите одну глобальную политику архива, которая содержит два свойства:

- **ArchiveInternal**. Если для этого свойства задано значение True ($True), выполняется архивация сеансов внутреннего обмена данными, в которых участвуют только те пользователи, у которых есть учетные записи Active Directory в текущей организации.

- **ArchiveЕхternal**. Если для этого свойства задано значение True ($True), выполняется архивация сеансов внешнего обмена данными, в которых участвует хотя бы один пользователь, у которого нет учетной записи Active Directory в текущей организации.

По умолчанию для обоих свойств установлено значение False, что означает, что ни внутренний, ни внешний обмен данными не архивируется. Чтобы изменить глобальную политику, можно использовать Skype для бизнеса Server и Set-CsArchivingPolicy. Эта команда включает архивацию сеансов как внутреннего, так и внешнего обмена данными.

```powershell
Set-CsArchivingPolicy -Identity "global" -ArchiveInternal $True -ArchiveExternal $True
```

Также можно использовать New-CsArchivingPolicy для создания новой политики в области сайта или в области пользователя. Например, эта команда создает новую индивидуальную политику архивации с именем RedmondArchivingPolicy:

```powershell
New-CsArchivingPolicy -Identity "RedmondArchivingPolicy" -ArchiveInternal $True -ArchiveExternal $True
```

Если создана индивидуальная политика, потребуется назначить ее для соответствующих пользователей. Например:

```powershell
Grant-CsArchivingPolicy -Identity "Ken Myer" -PolicyName  "RedmondArchivingPolicy"
```

Политики архива также можно управлять с помощью панели Skype для бизнеса Server управления. На панели управления нажмите **Мониторинг и архивация**, затем выберите **Политика архивации**. Чтобы изменить существующую политику, дважды щелкните политику (например, "Глобальные"), а затем в области **Редактирование политики архивации** установите или снимите флажки **Архивировать внутренний обмен данным** и **Архивировать внешний обмен данными**. Чтобы создать новую политику архива, щелкните **New,** а затем выберите политику **сайта или** **политику пользователя.** Если создана новая политика пользователя, необходимо перейти к соответствующим пользовательским учетным записям (вкладка **Пользователи**) и назначить для этих пользователей новую политику.

## <a name="step-3-configuring-the-exchangearchivingpolicy-property"></a>Шаг 3. Настройка свойства ExchangeArchivingPolicy

Если Skype для бизнеса Server и Exchange Server находятся в разных лесах, недостаточно просто включить Exchange архивацию в параметрах конфигурации архивации; это не приведет к архиву стенограмм обмена мгновенными сообщениями и веб-конференций в Exchange. Вместо этого необходимо также настроить свойство ExchangeArchivingPolicy для каждой из соответствующих учетных записей Skype для бизнеса Server пользователей. Это свойство может быть заданной для одного из четырех возможных значений:

1. **Uninitialized**. Указывает, что архивировать будет на основе параметров In-Place удержания, настроенных для почтового ящика Exchange пользователя; если In-Place удержание не включено в почтовом ящике пользователя, то в Skype для бизнеса Server будут архивироваться записи обмена сообщениями и веб-конференций.

2. **UseLyncArchivingPolicy**. Указывает, что протоколы обмена мгновенными сообщениями и веб-конференций пользователя должны архивироваться в Skype для бизнеса Server, а не в Exchange.

3. **NoArchiving**. Указывает, что архивация расшифровок обмена мгновенными сообщениями и веб-конференций для пользователя не выполняется. Обратите внимание, что этот параметр переопределяет Skype для бизнеса Server политик архива, заданных пользователю.

4. **ArchivingToExchange**. Указывает, что протоколы обмена мгновенными сообщениями и веб-конференций пользователя должны быть архивироваться в Exchange независимо от параметров удержания In-Place, которые были назначены (или не назначены) почтовому ящику пользователя.

Например, чтобы настроить учетную запись пользователя таким образом, чтобы протоколы обмена мгновенными сообщениями и веб-конференций всегда архивались до Exchange можно использовать команду, аналогичную этой, из Skype для бизнеса Server Management Shell:

```powershell
Set-CsUser -Identity "Ken Myer" -ExchangeArchivingPolicy ArchivingToExchange
```

Чтобы настроить ту же политику архивации для группы пользователей (например, для всех пользователей, размещенных в указанном пуле регистратора), можно использовать команду следующего вида:

```powershell
Get-CsUser -Filter {RegistrarPool -eq "atl-cs-001.litwareinc.com"} | Set-CsUser -ExchangeArchivingPolicy ArchivingToExchange
```

Обратите внимание, что для настройки значения свойства ExchangeArchivingPolicy необходимо использовать Skype для бизнеса Server(Windows PowerShell и Windows PowerShell). Это свойство не подвергается воздействию администраторов в Skype для бизнеса Server.

Чтобы просмотреть список всех пользователей, для которых назначена конкретная политика архивации, можно использовать команду следующего вида. Эта команда возвращает отображаемые имена Active Directory всех пользователей, для которых свойству ExchangeArchivingPolicy присвоено значение Uninitialized:

```powershell
Get-CsUser | Where-Object {$_.ExchangeArchivingPolicy -eq "Uninitialized"} | Select-Object DisplayName
```

Эта команда также возвращает отображаемые имена пользователей, для которых свойству ExchangeArchivingPolicy не назначено значение UseLyncArchivingPolicy:

```powershell
Get-CsUser | Where-Object {$_.ExchangeArchivingPolicy -ne "UseLyncArchivingPolicy"} | Select-Object DisplayName
```
