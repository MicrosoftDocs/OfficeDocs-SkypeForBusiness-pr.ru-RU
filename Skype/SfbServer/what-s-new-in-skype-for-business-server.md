# <a name="whats-new-in-skype-for-business-server-2015"></a>Новые возможности в Skype для бизнеса Server 2015

**Сводка:** Прочтите этот раздел, чтобы узнать о новых функций в Скайп для Business Server 2015. Подробные сведения о новых возможностей клиента [Lync является теперь Скайп для бизнеса – новые](https://go.microsoft.com/fwlink/p/?LinkId=529022)см.
  
Lync является теперь Скайп для бизнеса, коммуникаций и совместной работы платформы, который объединяет обеспечения взаимодействия по Скайп безопасности уровня предприятия, соответствие требованиям и управления Lync. Скайп для бизнеса предлагает компонентов, включая сведения о присутствии, обмен мгновенными Сообщениями, голосовой связи и видеовызовы и собрания по сети. Скайп для бизнеса предоставляет новый клиент интерфейс для новой версии сервера и обновлений в службу Office 365. Если пользователи в вашей организации уже знакомы с Скайп, они оценят power и простоты Скайп для бизнеса, где упрощает поиск и общаться с коллегами так. Если пользователи в вашей организации приходят Скайп для бизнеса из Lync, они будут распознает все компоненты они уже используется, но с новым нового интерфейса с упрощенное письмо, элементы управления и новые функции. Помимо нового взаимодействия с пользователем Скайп для Business Server 2015 предоставляет ряд новых возможностей для улучшения управляемости локальных серверов и гибридные решения.
  
Новые функции в Скайп для Business Server 2015 улучшения:
  
- Взаимодействие с пользователем  
- Поддержка голосовой и видеосвязи
- Поддержка мобильности
- Управление локальными серверами
- Развертывание гибридных решений и управление ими
- Поддержка многофакторной проверки подлинности
    
## <a name="user-experience"></a>Взаимодействие с пользователем

Скайп для клиента Business очень похож на пользовательской версии системы Скайп и использует ту же кнопки и значки. Меньшее число меню и более плоская иерархия задач позволяет пользователям быстрее находить нужные элементы управления и команды. 
  
Скайп для бизнеса включает в себя нового пользовательского интерфейса, описанных выше и пользовательский интерфейс Lync 2013, выпущенные ранее. Включение обоих каждый раз позволяет предприятиям управлять изменений для своих пользователей, управление процесс и синхронизации нового клиента roll масштабированием. Взаимодействие с пользователем по умолчанию зависит от того, какая версия сервера при использовании. Для выбора предпочтительного варианта взаимодействия с пользователем администраторы могут выполнить командлет **Set-CsClientPolicy** с параметром EnableSkypeUI. Дополнительные сведения о настройке с клиентом содержатся в разделе [Настройка взаимодействия с пользователем с Скайп для бизнеса](deploy/deploy-clients/configure-the-client-experience.md) и [Сравнение функций для настольных компьютеров для Скайп для бизнеса](plan-your-deployment/clients-and-devices/desktop-feature-comparison.md).
  
> [!NOTE]
> С клиентом Lync 2013 недоступно для Скайп для версий клиента 2016 бизнеса. Перед настройкой клиентской среды на использование клиента Lync 2013 убедитесь, что версия клиента не начинается с цифр "16" (например: 16.x.x.x).

После завершения вызова пользователям предлагается принять участие в опросе. Для получения дополнительных сведений см [скорость Мой вызов Скайп для Business Server 2015](manage/health-and-monitoring/rate-my-call.md).

Сервер видео взаимодействия (VIS) выступает в качестве посредника между Скайп для Business Server и Cisco систем видеоконференций (VTC). Теперь при присоединении к собранию пользователи могут выбрать систему Cisco VTC. Сервер видеовзаимодействия (VIS) реализован как автономная роль сервера для локальных развертываний. Для получения дополнительных сведений см. [Планирование Server взаимодействия видео в Скайп для Business Server 2015](plan-your-deployment/video-interop-server.md).
  
### <a name="call-via-work"></a>Вызов с рабочего телефона

Звонок с помощью компонента рабочих позволяет корпоративным пользователям звонить со Скайп для клиента Business. Когда пользователь помещает голосовой звонок, маршрутизируется из Скайп для бизнеса на телефон УАТС или ТСОП отправителя сообщения. Когда инициатор вызова отвечает на вызов данного телефона, вызов перенаправляется номеру назначения. Ответы получателя вызова и вызова устанавливается с Скайп для бизнеса, служат в качестве панели управления. Инициатор можно управлять их сведения о присутствии и управление вызовами с Скайп для бизнеса. Для предприятия функцию "Позвонить с рабочего" включают и настраивают администраторы серверов. Для получения дополнительных сведений см. [Планирование для звонков с помощью Скайп для Business Server 2015](plan-your-deployment/enterprise-voice-solution/call-via-work.md).

## <a name="management-of-on-premises-servers"></a>Управление локальными серверами

Скайп для Business Server 2015 предоставляет ряд новых возможностей для улучшения управляемости локальных серверов, включая обновление на месте, смарт-программы установки, улучшенные исправления и обновления обрабатывает, улучшенные возможности холодного Пуск пула переднего плана, SQL Server AlwaysOn Поддержка и централизованного ведения журналов и устранения неполадок.
  
### <a name="in-place-upgrade-for-on-premises-servers"></a>Обновление на месте для локальных серверов

Теперь можно обновление системы Lync Server 2013 до Скайп для Business Server 2015 с помощью новых на месте обновления компонента, которое использует существующие инвестиции оборудования и сервера Lync Server 2013, уменьшая общую стоимость развертывания Скайп для Business Server 2015.
  
Предусмотрено два сценария обновления на месте: метод "Перемещение пользователей", для которого не требуются простои, и автономный метод, для которого требуются простои. Дополнительные сведения о том, какие процедуры обновления для вашего бизнеса содержатся в разделе [Планирование обновления до Скайп для Business Server 2015](plan-your-deployment/upgrade.md). 
  
> [!NOTE]
> Параметр на месте недоступен, при обновлении с Lync Server 2010. Дополнительные сведения об обновлении с Lync Server 2010 можно [Планирование обновления до Скайп для Business Server 2015](plan-your-deployment/upgrade.md). 
  
### <a name="smart-setup"></a>Интеллектуальная установка

Теперь в программе установки есть функция интеллектуальной установки, которая автоматически обнаруживает и загружает обновления. Во время установки пользователю выдается запрос, если процесс установки проверяет наличие обновлений. Дополнительные сведения можно [Установить Скайп для Business Server 2015](deploy/install/install.md).
  
### <a name="improved-front-end-server-patching-and-upgrade-process"></a>Улучшенный процесс исправления и обновления сервера переднего плана

Скайп для Business Server представлены два новых командлетов, которые помогут обновления или исправлений серверов переднего плана, намного проще, чем в предыдущих версиях Lync Server.
  
Когда следует применять исправления, или выполните другие обслуживания, чтобы сервер переднего плана, просто введите **Invoke-CsComputerFailOver** и укажите имя этого сервера. Скайп для Business Server перемещает рабочей нагрузки, сервер временно других серверов в пуле. Затем обслуживания и затем используйте командлет **Invoke-CsComputerFailback** для вернуть этого сервера в службе. Если требуется применить исправление каждый сервер в пуле, просто выполните следующую процедуру для каждого сервера, по одному. Эти новые командлеты позволяют исправление серверы намного быстрее, чем, в предыдущих версиях, а также с надежности и упрощения рабочего процесса.
  
### <a name="improved-front-end-pool-cold-start-capability"></a>Улучшенная возможность холодного запуска интерфейсного пула

Скайп для Business Server представлены новые командлет, который упрощает и улучшает процесс холодного запуска весь пул переднего плана. При использовании новый командлет **Start-CsPool** проверяет необходимые условия для всех серверов переднего плана в пуле, а затем пытается запустить каждый сервер. В случае обнаружения неполадок он выполняет их диагностику и предоставляет оповещения с подробными сведениями и временными решениями. В некоторых случаях данный командлет позволяет запустить пул, даже если некоторые отдельные серверы не удается запустить.
  
### <a name="sql-server-alwayson-support-for-on-premises-servers"></a>Поддержка SQL Server AlwaysOn для локальных серверов

Скайп для Business Server 2015 добавляет поддержку для группы обеспечения доступности AlwaysOn SQL Server и экземплярами отказоустойчивого кластера SQL Server AlwaysOn. Помимо этих функций Скайп для Business Server продолжает поддержка зеркального отображения базы данных и кластеризации SQL Server, как и в предыдущих версиях Lync Server.
  
Группы обеспечения доступности AlwaysOn SQL Server — это высокой доступности и решение аварийного восстановления в SQL Server 2012 и SQL Server 2014 г., которая является альтернативой для зеркального отображения базы данных. Группы доступности поддерживают отказоустойчивую среду для дискретного набора баз данных (известного как базы данных доступности), которые совместно обрабатывают отказ. Группы доступности поддерживают набор баз данных-источников для записи и 1–4 набора соответствующих баз данных-получателей. Дополнительно, базы данных-получатели можно сделать доступными только для чтения и для некоторых операций резервного копирования.
  
Экземплярами отказоустойчивого кластера SQL Server использующих функциональные возможности Windows кластер отработки отказа сервера (WSFC) для обеспечения локального высокой доступности через избыточности на уровне экземпляра сервера — экземпляра отказоустойчивого кластера (FCI). FCI используется один экземпляр SQL Server, установленной на узлах Windows кластер отработки отказа сервера (WSFC) и, возможно, на несколько подсетей.
  
Для получения дополнительных сведений см. [Планирование высокой доступности и аварийного восстановления в Скайп для Business Server 2015](plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md).
  
### <a name="centralized-logging-and-troubleshooting-improvements-for-on-premises-servers"></a>Улучшения централизованного ведения журнала и устранения неполадок для локальных серверов

Службы централизованного ведения журналов — это среда предпочитаемый ведения журнала для Скайп для Business Server 2015. В этом выпуске нет новых возможностей, но надежность и производительность были повышены как для централизованной службы ведения журналов, так и для агента этой службы (ClsAgent.exe) — исполняемого файла службы, который обменивается данными с контроллером и принимает команды, подаваемые администратором.
  
Скайп для Business Server 2015 использует командлетов Windows PowerShell для управления агентами службы ведения журнала, инициировать трассировки и создания отчетов. (Lync Server 2013 используется ClsController.exe для выполнения этих задач).
  
Службы централизованного ведения журналов можно запустить на любой Скайп для Business Server 2015. Встроенные сценарии (предварительно заданные трассировки) остаться такой же, как и возможность создания пользовательских сценариев. Нет специальных сценарий вызывается AlwaysOn, всегда выполняется и позволяет администраторам найдите распространенных проблем в почти в режиме реального времени.
  
Средства отладки Snooper также была обновлена для отладки журналов мобильности и будет работать с устройств, подключенных к Lync 2013 или Скайп для Business Server 2015. Средство доступно для загрузки Web в [меню Отладка](https://go.microsoft.com/fwlink/?LinkId=285257).
  
## <a name="hybrid-deployment-and-management"></a>Гибридное развертывание и управление

Скайп для Business Server 2015 включает возможности управления и администрирования развертывания гибридного, введя следующие функции:
  
- Рекомендации для гибридных развертываний, в зависимости от состояния активов локального клиента, согласно OnRamp для O365 Автоматизированное средство.
- Усовершенствования в Скайп для панели управления Business Server и Скайп по центру администрирования Business Server, чтобы администраторы могут использовать эти средства для управления гибридного развертывания.
- Контроль панель усовершенствования, которые позволяют администраторам вход на портал клиента O365 и настроить гибридное развертывание с Скайп для бизнеса в Интернет с помощью мастера гибридной конфигурации.
- Поддержка панель для перемещение локальных пользователей к Скайп для бизнеса в Интернет или перемещение Скайп для пользователей Business Online обратно в локальную элемента управления.
- Управление функции панели для идентификации и фильтрации локальных объектов-пользователей, которые были перемещены в Скайп для бизнеса в Интернет (то есть пользователи гибридного) из локальных пользователей.
- Функции центра администрирования для идентификации и фильтрации облачных пользователей, изначально созданного в Скайп для бизнеса в Интернет из пользователей гибридной миграции из локальной сети.
- Возможность администрировать гибридного пользователей с помощью панели управления для свойств контролируемый из локальной и Центр администрирования для свойств контролируемый из Скайп для бизнеса в Интернет.
- использование синхронизации паролей с помощью DirSync, возможность синхронизации паролей локального каталога Active Directory с сетевым клиентом. Если эта возможность настроена, устраняется необходимость в развертывании служб федерации Active Directory для федеративной проверки подлинности, однако эти службы все еще необходимы для многофакторной проверки подлинности. 
- Продолжение поддержка совместную работу Скайп для бизнеса в Интернет и локальную систему Exchange.
    
> [!NOTE]
> Нет без изменений по сравнению с Lync Online 2013 и Exchange локальных совместной работы и поддержки взаимодействия. 
  
## <a name="multi-factor-authentication"></a>Многофакторная проверка подлинности

Многофакторная проверка подлинности предусматривает применение по меньшей мере двух способов проверки подлинности и обеспечивает второй уровень безопасности, имеющий важное значение при входе в систему и других выполняемых пользователем операциях. Примером может служить требование указать сертификат наряду с именем пользователя и паролем. Скайп для Business Server 2015 по-прежнему производится формирование для функции многофакторной проверки подлинности, доступные в Lync Server 2013 накопительный пакет обновлений. В многофакторную проверку подлинности внесены следующие существенные изменения:
  
- использование библиотеки проверки подлинности Active Directory в Office 2013 SP1 для интеграции с Exchange и SharePoint;
- Поддержка функции многофакторной проверки подлинности в Скайп для клиента Business Web App
    
С помощью Скайп для многофакторной проверки подлинности Business стало возможным для предоставления параметры проверки подлинности на основе География. Например, заказчики могут настроить среду таким образом, чтобы встроенная проверка подлинности Windows применялась внутри организации, а сотрудники, находящиеся за ее пределами, подвергались многофакторной проверке подлинности. 
  
Скайп для бизнеса программой многофакторная проверка подлинности не вызывает затруднений вне зависимости от:
  
- Географическое расположение - ли пользователь является вход из внутри или за пределами организации 
- Тип клиента и устройства - какие Скайп для клиента Business используется и какие устройства запущена (ПК, мобильный, iPad, и т.д.)
- Учетная запись Location — ли пользователь размещается в Active Directory в локальной или Azure Active Directory Online (O365)
