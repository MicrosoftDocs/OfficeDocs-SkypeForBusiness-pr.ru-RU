# <a name="whats-new-in-skype-for-business-server-2015"></a>Новые возможности Skype для бизнеса Server 2015 г.

**Сводка:** Ознакомьтесь с этой темой, чтобы узнать о новых Skype для бизнеса Server 2015 г. Подробные сведения о новом клиенте см. в Skype для бизнеса [Lync.](https://go.microsoft.com/fwlink/p/?LinkId=529022)
  
Теперь Lync Skype для бизнеса, платформа связи и совместной работы, которая объединяет опыт, Skype с безопасностью, соответствием требованиям и управлением Lync на корпоративном уровне. Skype для бизнеса предлагает функции, включая присутствие, чат, голосовые и видеозвонков, а также собрания в Интернете. Skype для бизнеса предоставляет новый клиентский опыт, новый выпуск сервера и обновления для службы в Microsoft 365 или Office 365. Если пользователи в вашей организации уже знакомы с Skype, они оценят силу и простоту Skype для бизнеса, где легко найти и связаться с сотрудниками. Если пользователи в организации приходят Skype для бизнеса Lync, они узнают все функции, которые они уже используют, но в новом интерфейсе с упрощенным управлением и новыми дополнениями. Помимо нового клиентского опыта, Skype для бизнеса Server 2015 г. предоставляет ряд новых функций для улучшения управляемости локального сервера и гибридных решений.
  
Новые функции Skype для бизнеса Server 2015 г. включают улучшения:
  
- Взаимодействие с пользователем  
- Поддержка голосовой и видеосвязи
- Поддержка мобильных устройств
- Управление локальной серверной частью
- Развертывание и управление гибридными решениями
- Поддержка многофакторной проверки подлинности
    
## <a name="user-experience"></a>Взаимодействие с пользователем

Клиент Skype для бизнеса очень похож на потребительскую версию Skype и использует те же кнопки и значки. Меньшее количество меню и более лестная иерархия задач позволяет пользователям быстро находить необходимые элементы управления и команды. 
  
Skype для бизнеса включает описанный выше новый пользовательский интерфейс и ранее выпущенный пользовательский интерфейс Lync 2013. Включение обоих опытом позволяет предприятиям управлять изменениями для своих пользователей, контролируя процесс и сроки нового клиентского выката. Пользовательский интерфейс по умолчанию зависит от версии используемого сервера. Администраторы выбирают предпочтительный опыт с помощью **комлета Set-CsClientPolicy** с параметром EnableSkypeUI. Дополнительные сведения о настройке клиентского опыта см. в Skype для бизнеса клиентских функций [Skype для бизнеса.](plan-your-deployment/clients-and-devices/desktop-feature-comparison.md) [](deploy/deploy-clients/configure-the-client-experience.md)
  
> [!NOTE]
> Клиентская возможность Lync 2013 не является вариантом для Skype для бизнеса 2016 года. Прежде чем пытаться настроить клиентскую среду для использования клиента Lync 2013, проверьте клиентскую версию, чтобы убедиться, что она не начинается с номера 16; например: 16.x.x.x. 
  
## <a name="voice-and-video-improvements"></a>Улучшения голосовой и видеосвязи

Skype для бизнеса Server 2015 г. включает улучшения функций голосового и видеосвязи, включая сбор и анализ данных вызовов, а также улучшенную работоспособность с сторонними системами телеконференции видео.
  
### <a name="call-data-collection-and-analysis"></a>Сбор и анализ данных вызовов

Функция Rate My Call позволяет администраторам Skype для бизнеса Server 2015 года собирать данные о вызове. Эта функция доступна только для локального развертывания. После завершения вызова пользователям будет предложено сделать опрос. Дополнительные сведения см. в рублях [Rate my Call in Skype для бизнеса Server 2015](manage/health-and-monitoring/rate-my-call.md).
  
### <a name="improved-interoperability-with-third-party-video-teleconferencing-systems"></a>Улучшенная интероперабельность с сторонними системами телеконференции видео

Video Interop Server (VIS) выступает в качестве посредника между системами видеоконференции Skype для бизнеса Server Cisco и видеоконференции (VTC). При присоединении к собранию пользователи теперь могут выбрать систему VTC Cisco. Video Interop Server (VIS) реализуется в качестве роли автономных серверов для локального развертывания. Дополнительные сведения см. в [записи Plan for Video Interop Server в Skype для бизнеса Server 2015 г.](plan-your-deployment/video-interop-server.md)
  
### <a name="call-via-work"></a>Вызов с рабочего телефона

Функция Call via Work позволяет корпоративным пользователям делать голосовые вызовы из Skype для бизнеса клиента. Когда пользователь помещает голосовой вызов, он передается из Skype для бизнеса на PBX или PSTN-телефон пользователя. Как только источник отвечает на телефонный звонок, звонок направляется на номер назначения. Получатель вызовов отвечает, и вызов устанавливается с Skype для бизнеса в качестве панели управления. Источник может управлять своим присутствием и управлением вызовами из Skype для бизнеса. Администраторы серверов позволяют и настраивают вызов с помощью Work для предприятия. Дополнительные сведения см. в [веб-сайте Plan for Call Via Work in Skype для бизнеса Server 2015](plan-your-deployment/enterprise-voice-solution/call-via-work.md)г.

## <a name="management-of-on-premises-servers"></a>Управление локальной серверной частью

Skype для бизнеса Server 2015 г. предоставляет ряд новых функций для улучшения управляемости локального сервера, включая локальное обновление, интеллектуальную настройку, усовершенствование процессов исправления и обновления, улучшенную возможность холодного запуска пула передней части, поддержку SQL Server AlwaysOn, а также централизованную регистрацию и устранение неполадок.
  
### <a name="in-place-upgrade-for-on-premises-servers"></a>Локальное обновление для локального сервера

Теперь вы можете обновить системы Lync Server 2013 до Skype для бизнеса Server 2015 г. с помощью новой функции обновления на месте, в которой используются существующие инвестиции в оборудование и серверы Lync Server 2013, что снижает общие затраты на развертывание Skype для бизнеса Server 2015 г.
  
Существует два сценария обновления на месте: метод Move User, который не требует простоя, и автономный метод, который требует простоя. Дополнительные сведения о том, какая процедура обновления является правильной для вашего бизнеса, см. в Skype для бизнеса Server [2015](plan-your-deployment/upgrade.md)г. 
  
> [!NOTE]
> При обновлении с Lync Server 2010 опция на месте недоступна. Дополнительные сведения об обновлении с Lync Server 2010 см. в Skype для бизнеса Server [2015](plan-your-deployment/upgrade.md)г. 
  
### <a name="smart-setup"></a>Smart Setup

Функция smart setup, которая автоматически обнаруживает и загружает обновления, теперь является частью программы установки. Во время процесса установки пользователю будет предложено проверить, следует ли проверять процесс установки на обновления. Дополнительные сведения см. в [Skype для бизнеса Server Install Skype для бизнеса Server 2015](deploy/install/install.md)г.
  
### <a name="improved-front-end-server-patching-and-upgrade-process"></a>Улучшенный процесс исправления и обновления переднего сервера

Skype для бизнеса Server представляет два новых комлета, которые помогают значительно упростить обновление или исправление серверов переднего конца, чем в предыдущих версиях Lync Server.
  
Если требуется применить исправление или выполнить любое другое обслуживание на переднем сервере, просто введите **Invoke-CsComputerFailOver** и укажите имя этого сервера. Skype для бизнеса Server временно перемещает нагрузку этого сервера на другие серверы пула. Затем можно выполнить техническое обслуживание, а затем использовать кодлет **Invoke-CsComputerFailback,** чтобы вернуть сервер в эксплуатацию. Если требуется исправление каждого сервера в пуле, просто выполните эту процедуру для каждого сервера по одному. Эти новые cmdlets позволяют значительно быстрее латать серверы, чем в предыдущих версиях, и с большей надежностью и более простым рабочий процесс.
  
### <a name="improved-front-end-pool-cold-start-capability"></a>Улучшена возможность холодного запуска пула переднего конца

Skype для бизнеса Server представляет новый cmdlet, который упрощает и улучшает процесс холодного запуска всего пула переднего конца. При использовании нового комлета **Start-CsPool** он проверяет предварительные требования для всех серверов переднего конца в пуле, а затем пытается запустить каждый сервер. Если возникают какие-либо проблемы, он диагностирует их и оповещает вас с подробными сведениями и обходными решениями. В некоторых случаях это позволяет запустить пул, даже если некоторые отдельные серверы не могут запуститься.
  
### <a name="sql-server-alwayson-support-for-on-premises-servers"></a>SQL Server Поддержка AlwaysOn для локального сервера

Skype для бизнеса Server 2015 г. добавляет поддержку как для групп доступности SQL Server AlwaysOn, так и для SQL Server экземпляров кластера неудачной остановки AlwaysOn. В дополнение к этим функциям Skype для бизнеса Server поддержку зеркального и SQL Server баз данных, как и в прошлых версиях Lync Server.
  
SQL Server AlwaysOn Availability Groups — это решение для восстановления доступности и аварийного восстановления в 2012 SQL Server и SQL Server 2014 г., которое предоставляет альтернативу зеркальным данным баз данных. Группа доступности поддерживает среду сбоя для дискретного набора баз данных (известных как базы данных доступности), которые не работают вместе. Группа доступности поддерживает набор первичных баз данных для чтения и от одного до четырех наборов соответствующих вторичных баз данных. Дополнительно можно сделать дополнительные базы данных доступными для доступа только для чтения и для некоторых операций резервного копирования.
  
SQL Server Экземпляры кластеров неудачной работы используют Windows кластеризация неудачных серверов (WSFC) для обеспечения локальной высокой доступности за счет избыточности на уровне экземпляра сервера — экземпляра кластера неудачной работы (FCI). FCI — это один экземпляр SQL Server, установленный в Windows кластеризации серверов (WSFC) и, возможно, в нескольких подсетях.
  
Дополнительные сведения см. в [проекте Plan for high availability and disaster recovery in Skype для бизнеса Server 2015.](plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md)
  
### <a name="centralized-logging-and-troubleshooting-improvements-for-on-premises-servers"></a>Централизованное ведение журнала и устранение неполадок для локального сервера

Централизованная служба ведения журнала является предпочтительной средой ведения журнала для Skype для бизнеса Server 2015 г. В этом выпуске нет новых функций, но надежность и производительность улучшены как для службы, так и для агента централизованной службы ведения журнала (ClsAgent.exe) — службы, которая взаимодействует с контроллером и получает команды, выдаемые администратором.
  
Skype для бизнеса Server 2015 Windows PowerShell для управления агентами службы ведения журнала, инициировать отслеживание и создавать отчеты. (Lync Server 2013 ClsController.exe для выполнения этих задач.)
  
Централизованная служба ведения журнала может работать на любом Skype для бизнеса Server 2015 г. Встроенные сценарии (заранее определенные трассировки) остаются одинаковыми, равно как и возможность создания настраиваемого сценария. Существует специальный сценарий под названием AlwaysOn, который всегда запущен, и позволяет администраторам находить распространенные проблемы в режиме реального времени.
  
Средство отладки Snooper также обновлено, чтобы разрешить отладку журналов мобильности и будет работать с устройствами, подключенными к Lync 2013 или Skype для бизнеса Server 2015. Средство доступно в качестве веб-загрузки из средств [отладки.](https://go.microsoft.com/fwlink/?LinkId=285257)
  
## <a name="hybrid-deployment-and-management"></a>Гибридное развертывание и управление

Skype для бизнеса Server 2015 г. позволяет использовать возможности администрирования и управления гибридными развертываниями, введя следующие функции:
  
- Рекомендации гибридных развертываниях на основе состояния локального актива клиента, определяемого средством автоматической помощи OnRamp для Office 365 службы.
- Усовершенствования панели Skype для бизнеса Server и центра администрирования Skype для бизнеса Server, чтобы администраторы могли использовать эти средства для управления гибридным развертыванием.
- Улучшения панели управления, которые могут позволить администраторам войти в Microsoft 365 или Office 365 клиента и настроить гибрид с Skype для бизнеса Online с помощью мастера гибридной конфигурации.
- Панель управления поддерживает перемещение локального пользователя в Skype для бизнеса Или перемещение Skype для бизнеса в локальное помещение.
- Функции Панели управления для идентификации и фильтрации локального пользовательского объекта, перемещенного в Skype для бизнеса Online (то есть гибридных пользователей) от локального пользователя.
- Функции Центра администрирования для идентификации и фильтрации облачных пользователей, изначально созданных в Skype для бизнеса Online из гибридных пользователей, перенесенных из локального в Online.
- Возможность администрирования гибридных пользователей с помощью панели управления для свойств, управляемых из локального центра, и Центра администрирования свойств, управляемых Skype для бизнеса Online.
- С помощью синхронизации паролей с DirSync можно синхронизировать локально пароли Active Directory с клиентом в Интернете. Если эта функция настроена, она удаляет необходимость развертывания AD FS для федерадной проверки подлинности, но AD FS по-прежнему требуется для многофакторной проверки подлинности. 
- Поддержка сосуществования между Skype для бизнеса Online и Exchange локальной службой.
    
> [!NOTE]
> Существует никаких изменений от Lync Online 2013 и Exchange локального сосуществования и поддержки. 
  
## <a name="multi-factor-authentication"></a>Многофакторная проверка подлинности

Многофакторная проверка подлинности — это метод проверки подлинности, который требует использования нескольких методов проверки и добавляет критически важный второй уровень безопасности для входов и транзакций пользователей. Например, требуется имя пользователя и пароль, а также сертификат. Skype для бизнеса Server 2015 г. продолжает развиваться на основе функций многофакторной проверки подлинности, доступных в накопительных обновлениях Lync Server 2013. Существенные изменения в многофакторной проверке подлинности:
  
- Использование библиотеки проверки подлинности Office sp1 Active Directory 2013 для интеграции с Exchange и SharePoint
- Поддержка многофакторной проверки подлинности в Skype для бизнеса Web App клиенте
    
Благодаря Skype для бизнеса многофакторной проверке подлинности теперь можно предоставлять различные параметры проверки подлинности в зависимости от географии. Например, клиенты могут настроить свою среду так, чтобы внутренняя проверка подлинности опиралась на интегрированную Windows проверку подлинности, в то время как сотрудники, выехав проверку подлинности из-за пределов организации, используют многофакторную проверку подлинности. 
  
Многофакторная проверка подлинности Skype для бизнеса без проблем независимо от:
  
- Географическое расположение — независимо от того, входит ли пользователь изнутри или за ее пределами в организацию. 
- Тип клиента или устройства — Skype для бизнеса используется и на какое устройство работает клиент (ПК, мобильный, iPad и т.д.)
- Расположение учетной записи — размещение пользователя в локальном каталоге Active Directory или в Azure Active Directory Online.
