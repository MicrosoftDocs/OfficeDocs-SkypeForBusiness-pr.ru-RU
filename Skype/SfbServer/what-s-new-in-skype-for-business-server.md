# <a name="whats-new-in-skype-for-business-server-2015"></a>Новые возможности Skype для бизнеса Server 2015

**Сводка:** В этом разделе рассказывается о новых возможностях Skype для бизнеса Server 2015. Более подробную информацию о новом интерфейсе клиента можно узнать в статье [Lync to Skype for Business-просмотреть новые](https://go.microsoft.com/fwlink/p/?LinkId=529022)возможности.
  
Lync теперь — это платформа для общения и совместной работы, которая объединяет возможности Skype с обеспечением безопасности, соответствием и управлением Lync в корпоративной среде. Skype для бизнеса включает в себя функции присутствия, обмена мгновенными сообщениями, голосовых и видеоконференций, а также собраний по сети. Skype для бизнеса обеспечивает новую среду взаимодействия с клиентом, новый серверный выпуск и обновления службы в Microsoft 365 или Office 365. Если пользователи в вашей организации уже знакомы с Skype, они оценят мощь и простоту Skype для бизнеса, где легко находить и подключаться к коллегам. Если пользователи в вашей организации поступают в Skype для бизнеса из Lync, они будут распознавать все уже используемые функции, но в новом интерфейсе с упрощенными элементами управления и новыми дополнениями. В дополнение к новому интерфейсу клиента Skype для бизнеса Server 2015 предоставляет несколько новых функций для усовершенствования управляемости локальных серверов и гибридных решений.
  
Новые возможности Skype для бизнеса Server 2015 включают улучшения:
  
- Взаимодействие с пользователем  
- Поддержка голосовой связи и видео
- Поддержка мобильных устройств
- Управление локальными серверами
- Развертывание гибридных решений и управление ими
- Поддержка многофакторной проверки подлинности
    
## <a name="user-experience"></a>Взаимодействие с пользователем

Клиент Skype для бизнеса очень похож на версию Skype для потребителей и использует те же кнопки и значки. Меньшее количество меню и иерархия задач плоская позволяют пользователям быстро находить необходимые элементы управления и команды. 
  
Skype для бизнеса включает новый пользовательский интерфейс, описанный выше, и сведения о пользовательском интерфейсе Lync 2013, выпущенные ранее. Включение обоих интерфейсов позволяет предприятиям управлять изменениями для пользователей, контролируя процесс и время развертывания нового клиента. Используемый по умолчанию интерфейс пользователя зависит от используемой версии сервера. Администраторы выбирают предпочтительный интерфейс с помощью командлета **Set – CsClientPolicy** с параметром EnableSkypeUI. Дополнительные сведения о настройке взаимодействия с клиентом можно узнать в статье [Настройка взаимодействия с клиентом в Skype для бизнеса](deploy/deploy-clients/configure-the-client-experience.md) и [функции клиента для настольных ПК в Skype для бизнеса](plan-your-deployment/clients-and-devices/desktop-feature-comparison.md).
  
> [!NOTE]
> Взаимодействие с клиентом Lync 2013 не является возможностью для версий клиентов Skype для бизнеса 2016. Прежде чем попытаться настроить клиентскую среду для использования клиента Lync 2013, проверьте версию клиента, чтобы убедиться, что она не начинается с номера 16; Пример: 16. КС.КС.КС. 
  
## <a name="voice-and-video-improvements"></a>Улучшения голосовой и видеосвязи

Skype для бизнеса Server 2015 включает улучшения функций голосовой и видеосвязи, в том числе сбор и анализ данных вызовов, а также улучшенное взаимодействие с системами видеоконференций сторонних производителей.
  
### <a name="call-data-collection-and-analysis"></a>Сбор и анализ данных вызовов

Функция оценки моего звонка позволяет администраторам в Skype для бизнеса Server 2015 собирать данные о вызовах. Эта функция доступна только для локальных развертываний. После выполнения звонка пользователям предлагается принять участие в опросе. Дополнительную информацию можно узнать [в статье Rate звонки в Skype для бизнеса Server 2015](manage/health-and-monitoring/rate-my-call.md).
  
### <a name="improved-interoperability-with-third-party-video-teleconferencing-systems"></a>Улучшенное взаимодействие с системами видеоконференций сторонних производителей

Сервер видеовзаимодействия (VIS) выступает в качестве посредника между Skype для бизнеса Server и системами видеоконференций Cisco (VTC). Когда вы присоединяетесь к собранию, пользователи могут выбрать систему Cisco VTC. Сервер видеовзаимодействия (VIS) реализован в виде автономной роли сервера для локальных развертываний. Дополнительные сведения приведены в статье [Plan for Video Interop Server в Skype для бизнеса server 2015](plan-your-deployment/video-interop-server.md).
  
### <a name="call-via-work"></a>Вызов с рабочего телефона

Функция "позвонить с рабочего" позволяет корпоративным пользователям совершать голосовые вызовы от клиента Skype для бизнеса. Когда пользователь помещает голосовой вызов, он перенаправляется из Skype для бизнеса на телефон УАТС или телефон PSTN. После того как инициатор ответит на телефон, Звонок направляется на целевой номер. Ответ получателя вызываемого абонента, а вызов устанавливается с помощью Skype для бизнеса, обслуживающего в качестве панели управления. Инициатор может управлять сведениями о присутствии и средства контроля звонков из Skype для бизнеса. Администраторы серверов включают и настраивают звонки через работу на предприятии. Для получения дополнительных сведений ознакомьтесь со статьей [планирование вызовов с помощью Works в Skype для бизнеса Server 2015](plan-your-deployment/enterprise-voice-solution/call-via-work.md). 
  
## <a name="mobile-device-support-improvements"></a>Улучшения поддержки мобильных устройств

Улучшения поддержки мобильных устройств включают функции, позволяющие улучшить взаимодействие с мобильными пользователями для пользователей Enterprise Edition, таких как доступ к журналу бесед и данным журналов, расширенные возможности для мобильных собраний и поддержка единого входа в Office. Кроме того, существуют улучшения поддержки Android, улучшения производительности и функции для упрощения интеграции с помощью общей платформы приложений. 
  
> [!NOTE]
> Для поддержки мобильных клиентов необходимо обновить серверы Lync 2013 UCWA до Skype для бизнеса Server 2015. 
  
### <a name="server-side-conversation-history-is-now-available-on-mobile-devices"></a>Журнал бесед на стороне сервера теперь доступен на мобильных устройствах

Чтобы разрешить мобильному доступу к журналу бесед, пропущенному мгновенным сообщениям и данным журнала вызовов, архивация этих сведений теперь обработана на сервере для всех мобильных клиентов. Функция автоматического принятия МГНОВЕНных сообщений повышает надежность, предотвращая таймауты сервера, когда мобильный пользователь не реагирует на МГНОВЕНные сообщения немедленно. Чтобы воспользоваться преимуществами интеграции серверных папок Exchange/Outlook, требуется сервер Exchange Server 2013 или более поздней версии, а также обновленные мобильные клиенты. 
  
### <a name="enhanced-meeting-experiences"></a>Расширенные возможности для собраний

Функции собраний, доступные на рабочем столе, теперь доступны и пользователям мобильных устройств. Новые функции включают:
  
- Асинхронная Навигация по общему содержимому PowerPoint
- Возможность докладчика для управления общим содержимым PowerPoint
- Центр управления для докладчиков, позволяющий отклонять или отказывать участников
    
### <a name="skype-for-business-on-android-improvements"></a>Улучшения в Skype для бизнеса на Android

Skype для бизнеса на Android теперь предлагает функции, аналогичные доступным в Skype для бизнеса в iOS и Skype для бизнеса в Windows.
  
- Продолжение или повторное присоединение бесед
- Включение сертификата и пассивной проверки подлинности
- Приглашение других пользователей в беседу
- Простой запуск групповых бесед
- Присоединение к собранию без участия пользователя Skype для бизнеса
- Включение пользовательского интерфейса смартфона на планшетах с Android
- Управление собраниями путем добавления или удаления участников
    
> [!NOTE]
> Для этих компонентов требуется ОС Android версии 4,0 или более поздней. 
  
## <a name="management-of-on-premises-servers"></a>Управление локальными серверами

Skype для бизнеса Server 2015 предоставляет несколько новых функций для улучшения управляемости локальных серверов, в том числе обновление на месте, интеллектуальная установка, усовершенствованные процессы исправления и обновления, улучшенная возможность холодного запуска пула, поддержка SQL Server AlwaysOn и централизованное ведение журнала и устранение неполадок.
  
### <a name="in-place-upgrade-for-on-premises-servers"></a>Обновление на месте для локальных серверов

Теперь вы можете обновить системы Lync Server 2013 на Skype для бизнеса Server 2015, используя новую функцию обновления на месте, которая использует существующие серверные инвестиции Lync Server 2013, что позволяет сократить общую стоимость развертывания Skype для бизнеса Server 2015.
  
Существует два сценария обновления на месте: метод Move User, для которого не требуются простои, и метод Offline, требующий простоя. Для получения дополнительных сведений о том, какая процедура обновления подходит для вашего бизнеса, ознакомьтесь со статьей [Планирование обновления до Skype для бизнеса Server 2015](plan-your-deployment/upgrade.md). 
  
> [!NOTE]
> При обновлении с Lync Server 2010 параметр "на месте" недоступен. Для получения дополнительных сведений об обновлении с Lync Server 2010, ознакомьтесь со статьей [Планирование обновления до Skype для бизнеса server 2015](plan-your-deployment/upgrade.md). 
  
### <a name="smart-setup"></a>Интеллектуальная настройка

Компонент Smart Setup, который автоматически обнаруживает и загружает обновления, теперь является частью программы установки. Во время процесса установки пользователю предлагается указать, следует ли проверять наличие обновлений. Дополнительные сведения можно найти в [статье Установка Skype для бизнеса Server 2015](deploy/install/install.md).
  
### <a name="improved-front-end-server-patching-and-upgrade-process"></a>Усовершенствованный процесс исправления и обновления сервера переднего плана

В Skype для бизнеса Server представлено два новых командлета, которые помогают выполнять обновление или исправление серверов переднего плана значительно проще, чем в предыдущих версиях Lync Server.
  
Чтобы применить исправление или выполнить любое другое обслуживание на сервере переднего плана, просто введите **Invoke – кскомпутерфаиловер** и укажите имя сервера. Skype для бизнеса Server временно перемещает рабочую нагрузку сервера на другие серверы в пуле. Затем можно выполнить обслуживание, а затем с помощью командлета **Invoke – кскомпутерфаилбакк** вернуть этот сервер в службу. Если вам нужно исправить каждый сервер в пуле, просто выполните эту процедуру для каждого сервера по одному за раз. Эти новые командлеты позволяют изменять серверы гораздо быстрее, чем в предыдущих версиях, с большей надежностью и более простым рабочим процессом.
  
### <a name="improved-front-end-pool-cold-start-capability"></a>Улучшенная возможность холодного запуска пула переднего плана

В Skype для бизнеса Server представлен новый командлет, упрощающий и улучшающий процесс холодного запуска всего интерфейсного пула. При использовании нового командлета **Start – CsPool** он проверяет предварительные требования для всех серверов переднего плана в пуле, а затем пытается запустить каждый сервер. Если обнаружены какие – либо проблемы, он выводит их и оповещает о них. В некоторых случаях он позволяет запустить пул даже в том случае, если не удается запустить некоторые из них.
  
### <a name="sql-server-alwayson-support-for-on-premises-servers"></a>Поддержка SQL Server AlwaysOn для локальных серверов

Skype для бизнеса Server 2015 добавляет поддержку для групп доступности AlwaysOn SQL Server и экземпляров отказоустойчивого кластера SQL Server AlwaysOn. В дополнение к этим функциям Skype для бизнеса Server продолжает поддерживать зеркальное отображение баз данных и кластеризацию SQL Server, как в предыдущих версиях Lync Server.
  
Группы доступности AlwaysOn SQL Server — это решение для обеспечения высокой доступности и аварийного восстановления в SQL Server 2012 и SQL Server 2014, которое предоставляет альтернативу зеркальному отображению баз данных. Группа доступности поддерживает отказоустойчивую среду для дискретного набора баз данных (называемых базами данных доступности), которые отменяют совместный отказ. Группа доступности поддерживает набор основных баз данных для чтения и записи, а от одного до четырех наборов соответствующих баз данных-получателей. Кроме того, дополнительные базы данных можно сделать доступными только для чтения и для некоторых операций резервного копирования.
  
Экземпляры отказоустойчивого кластера SQL Server используют функции отказоустойчивой кластеризации Windows Server (WSFC), чтобы обеспечить локальную высокую степень доступности с помощью избыточности на уровне экземпляра сервера — экземпляра отказоустойчивого кластера (FCI). FCI — это один экземпляр SQL Server, устанавливаемый на узлах отказоустойчивой кластеризации Windows Server (WSFC) и, возможно, между несколькими подсетями.
  
Для получения дополнительных сведений ознакомьтесь [со статьей Планирование высокой доступности и аварийного восстановления в Skype для бизнеса Server 2015](plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md).
  
### <a name="centralized-logging-and-troubleshooting-improvements-for-on-premises-servers"></a>Улучшения централизованного ведения журналов и устранения неполадок для локальных серверов

Централизованная служба ведения журналов — это предпочтительная среда ведения журнала для Skype для бизнеса Server 2015. В этом выпуске нет новых функций, но в службе и агенте централизованной службы ведения журналов (ClsAgent. exe) были улучшены надежность и производительность, а также исполняемый файл службы, который взаимодействует с контроллером и получает команды, выданные администратором.
  
Skype для бизнеса Server 2015 использует командлеты Windows PowerShell для управления агентами службы ведения журналов, запуска трассировки и создания отчетов. (Для выполнения этих задач в Lync Server 2013 использовался ClsController. exe.)
  
Централизованная служба ведения журналов может выполняться на любом Skype для бизнеса Server 2015. Встроенные сценарии (предварительно определенные трассировки) остались без изменений, что позволяет создавать пользовательские сценарии. Существует специальный сценарий с именем AlwaysOn, который всегда выполняется, и позволяет администраторам искать распространенные проблемы почти в режиме реального времени.
  
Средство отладки Snooper также было обновлено для поддержки отладки журналов мобильности и будет работать с устройствами, которые подключаются к Lync 2013 или Skype для бизнеса Server 2015. Средство можно загрузить из [средств отладки](https://go.microsoft.com/fwlink/?LinkId=285257)с помощью веб-сайта.
  
## <a name="hybrid-deployment-and-management"></a>Гибридное развертывание и управление

Skype для бизнеса Server 2015 обеспечивает возможности администрирования и управления гибридным развертыванием, предвводя следующие возможности:
  
- Рекомендации по развертыванию гибридных развертываний в зависимости от состояния локальных ресурсов клиента, определенных средством автоматической поддержки Office 365.
- Расширения для панели управления Skype для бизнеса Server и центра администрирования Skype для бизнеса Server, чтобы администраторы могли использовать эти средства для управления гибридным развертыванием.
- Расширения панели управления, позволяющие администраторам входить в клиент Microsoft 365 или Office 365 и настраивать гибридную среду со Skype для бизнеса Online с помощью мастера гибридной конфигурации.
- Панель управления поддерживает перемещение локальных пользователей в Skype для бизнеса Online или перемещение пользователей Skype для бизнеса Online обратно в локальную среду.
- Функции панели управления для определения и фильтрации локальных объектов пользователей, которые были перемещены в Skype для бизнеса Online (то есть гибридные пользователи) от локальных пользователей.
- Функции центра администрирования для идентификации и фильтрации облачных пользователей, изначально созданных в Skype для бизнеса Online, от гибридных пользователей, перенесенных из локальной среды в сетевую.
- Возможность администрирования гибридных пользователей с помощью панели управления для свойств, которые можно управлять из локальной среды, и центра администрирования для свойств, которые можно управлять из Skype для бизнеса Online.
- С помощью синхронизации паролей с DirSync можно синхронизировать локальные пароли Active Directory с сетевым клиентом. При настройке эта функция удаляет необходимость развертывания AD FS для федеративной проверки подлинности, но AD FS все еще требуется для многофакторной проверки подлинности. 
- Поддержка сосуществования между Skype для бизнеса Online и локальной системой Exchange.
    
> [!NOTE]
> Поддержка совместной работы в Lync Online 2013 и Exchange Online и поддержка в локальной среде не изменяется. 
  
## <a name="multi-factor-authentication"></a>Многофакторная проверка подлинности

Многофакторная проверка подлинности — это способ проверки подлинности, который требует использования нескольких методов проверки и добавляет важный второй уровень безопасности для пользовательских входов и транзакций. Например, если требуется имя пользователя и пароль, а затем — сертификат. Skype для бизнеса Server 2015 продолжает разрабатывать функции многофакторной проверки подлинности, доступные в накопительных пакетах обновления Lync Server 2013. Ниже приведены существенные изменения многофакторной проверки подлинности.
  
- Использование библиотеки проверки подлинности Active Directory для Office 2013 с пакетом обновления 1 (SP1) для интеграции с Exchange и SharePoint
- Поддержка функции многофакторной проверки подлинности в клиенте веб-приложения Skype для бизнеса
    
С помощью многофакторной проверки подлинности Skype для бизнеса теперь можно использовать различные варианты проверки подлинности на основе географического уровня. Например, пользователи могут настраивать среду, чтобы внутренняя проверка подлинности полагается на встроенную проверку подлинности Windows, а сотрудники, прошедшие проверку подлинности извне организации, используют многофакторную проверку подлинности. 
  
Многофакторная проверка подлинности в Skype для бизнеса работает без проблем независимо от:
  
- Географическое расположение — является ли пользователь подписанным из внутренней организации или за ее пределами. 
- Тип клиента/устройства, используемый клиент Skype для бизнеса и устройство, на котором работает клиент (PC, Mobile, iPad и т. д.).
- Расположение учетной записи — указывает, размещается ли пользователь в локальной службе каталогов Active Directory или в службе Azure Active Directory Online.
