# <a name="whats-new-in-skype-for-business-server-2015"></a>Новые возможности Skype для бизнеса Server 2015

**Сводка:** В этом разделе представлены новые функции Skype для бизнеса Server 2015. Подробные сведения о новом клиенте см. в описании новых функций [Lync](https://go.microsoft.com/fwlink/p/?LinkId=529022)в Skype для бизнеса.
  
Теперь Lync — это Skype для бизнеса, платформа для общения и совместной работы, которая объединяет в себя взаимодействие, на основе которой Skype обеспечивает безопасность, соответствие требованиям и контроль над Lync корпоративного уровня. Skype для бизнеса предлагает функции, в том числе функции присутствия, мгновенных услуг, голосовых и видеозвонков и собраний по сети. Skype для бизнеса предоставляет новый клиентский доступ, новый выпуск сервера и обновления для службы в Microsoft 365 или Office 365. Если пользователи в вашей организации уже знакомы со Skype, они оценят всю мощь и простоту Skype для бизнеса, где легко найти и связаться с сотрудниками. Если пользователи в вашей организации приходят в Skype для бизнеса из Lync, они распознают все функции, которые они уже используют, но в новом интерфейсе с упрощенными средствами управления и новыми дополнениями. Помимо новых возможностей клиента, Skype для бизнеса Server 2015 предоставляет ряд новых функций для улучшения управляемости локального сервера и гибридных решений.
  
Новые возможности Skype для бизнеса Server 2015 включают следующие улучшения:
  
- Взаимодействие с пользователем  
- Поддержка голосовой и видеосвязи
- Поддержка мобильных устройств
- Управление локальной серверной частью
- Развертывание гибридных решений и управление ими
- Поддержка многофакторной проверки подлинности
    
## <a name="user-experience"></a>Взаимодействие с пользователем

Клиент Skype для бизнеса очень похож на клиентскую версию Skype и использует те же кнопки и значки. Меньшее количество меню и ровная иерархия задач позволяет пользователям быстро находить нужные элементы управления и команды. 
  
В Skype для бизнеса есть новые возможности, описанные выше, и Lync 2013, выпущенные ранее. Включение обоих функций позволяет предприятиям управлять изменениями для своих пользователей, контролируя процесс и сроки нового клиентского пакета. Пользовательский интерфейс по умолчанию зависит от используемой версии сервера. Администраторы выбирают предпочтительный интерфейс с помощью cmdlet **Set-CsClientPolicy** с параметром EnableSkypeUI. Дополнительные сведения о настройке клиентского опыта см. в этой [теме.](plan-your-deployment/clients-and-devices/desktop-feature-comparison.md) [](deploy/deploy-clients/configure-the-client-experience.md)
  
> [!NOTE]
> Клиентский опыт Lync 2013 не является вариантом для версий клиентов Skype для бизнеса 2016. Прежде чем пытаться настроить клиентскую среду для использования клиента Lync 2013, проверьте версию клиента, чтобы убедиться, что она не начинается с номера 16; например: 16.x.x.x. 
  
## <a name="voice-and-video-improvements"></a>Улучшения голосовой и видеосвязи

Skype для бизнеса Server 2015 включает усовершенствования функций голосовой и видеосвязи, включая сбор и анализ данных о вызовах, а также улучшенное общение со сторонними системами видеоконференции.
  
### <a name="call-data-collection-and-analysis"></a>Сбор и анализ данных вызовов

Функция "Оценить мой звонок" позволяет администраторам Skype для бизнеса Server 2015 собирать данные о вызовах. Эта функция доступна только для локального развертывания. После завершения вызова пользователям будет предложено принять участие в опросе. For more information, see [Rate my Call in Skype for Business Server 2015](manage/health-and-monitoring/rate-my-call.md).
  
### <a name="improved-interoperability-with-third-party-video-teleconferencing-systems"></a>Улучшенное качество работы со сторонними системами видеоконференции

Сервер видеосвязи (VIS) действует как посредник между Skype для бизнеса Server и системами видеоконференции Cisco (VTC). При присоединении к собранию пользователи могут выбрать систему Cisco VTC. Сервер видеосвязи (VIS) реализован в качестве роли автономных серверов для локального развертывания. Дополнительные сведения см. в сведениях [о планировании сервера видеосвязи в Skype для бизнеса Server 2015.](plan-your-deployment/video-interop-server.md)
  
### <a name="call-via-work"></a>Вызов с рабочего телефона

Функция "Позвонить с работы" позволяет корпоративным пользователям делать голосовые вызовы из клиента Skype для бизнеса. Когда пользователь звонит по голосовой связи, он перенанается из Skype для бизнеса на телефон УАПС или PSTN источника. После того как источник отвечает на звонок, вызов перенаправскуется на номер назначения. The call recipient answers, and the call is established with Skype for Business serving as the control panel. The originator can manage their presence and call controls from Skype for Business. Администраторы сервера могут включить и настроить "Позвонить с работы" для предприятия. For more information, see [Plan for Call Via Work in Skype for Business Server 2015](plan-your-deployment/enterprise-voice-solution/call-via-work.md).

Чтобы воспользоваться преимуществами интеграции серверных папок Exchange и Outlook, Exchange Server 2013 или более новых версий, а также обновленные мобильные клиенты.

Skype для бизнеса на Android теперь предоставляет функции, аналогичные функциям, доступным в Skype для бизнеса для iOS и Skype для бизнеса в Windows:

- Присоединяйтесь к собранию, не являясь пользователем Skype для бизнеса
- Включить пользовательский интерфейс смартфона на планшетах с Android

## <a name="management-of-on-premises-servers"></a>Управление локальной серверной частью

Skype для бизнеса Server 2015 предоставляет ряд новых функций для улучшения управляемости локального сервера, включая обновление на месте, интеллектуальную установку, улучшенные процессы исправления и обновления, улучшенные возможности холодного запуска пула переднего сервера, поддержку SQL Server AlwaysOn, централизованное ведение журнала и устранение неполадок.
  
### <a name="in-place-upgrade-for-on-premises-servers"></a>Обновление на месте для локального сервера

Теперь системы Lync Server 2013 можно обновить до Skype для бизнеса Server 2015 с помощью новой функции обновления на месте, которая использует существующие инвестиции в оборудование и сервер Lync Server 2013, тем самым сокращая общие затраты на развертывание Skype для бизнеса Server 2015.
  
Существует два сценария обновления на месте: метод Move User, который не требует простоя, и автономный метод, который требует простоя. For more information about which upgrade procedure is right for your business, see [Plan to upgrade to Skype for Business Server 2015.](plan-your-deployment/upgrade.md) 
  
> [!NOTE]
> При обновлении с Lync Server 2010 параметр на месте недо доступен. For more information about upgrading from Lync Server 2010, see [Plan to upgrade to Skype for Business Server 2015.](plan-your-deployment/upgrade.md)

Функция интеллектуальной установки, которая автоматически обнаруживает и скачивает обновления, теперь входит в программу установки. ### <a name="improved-front-end-server-patching-and-upgrade-process"></a>Улучшенный процесс исправления и обновления сервера переднего сервера

Skype для бизнеса Server представляет два новых cmdlets, которые упрощают обновление или исправление серверов переднего сервера, чем в предыдущих версиях Lync Server.
  
Если необходимо применить исправление или выполнить любое другое обслуживание на сервере переднего сервера, просто введите **Invoke-CsComputerFailOver** и укажите имя этого сервера. Skype для бизнеса Server временно перемещает нагрузку этого сервера на другие серверы в пуле. Затем можно выполнить обслуживание, а затем с помощью cmdlet **Invoke-CsComputerFailback** вернуть этот сервер в эксплуатацию. Если необходимо в обновлении каждого сервера в пуле, просто выполните эту процедуру для каждого сервера по одному. Эти новые cmdlets позволяют значительно быстрее исправления серверов, чем в предыдущих версиях, а также с большей надежностью и простой рабочий процесс.
  
### <a name="improved-front-end-pool-cold-start-capability"></a>Улучшена возможность "холодного запуска" пула переднего входа

В Skype для бизнеса Server вводится новый cmdlet, который упрощает и улучшает процесс холодного запуска всего пула переднего сервера. При использовании нового cmdlet **Start-CsPool** он проверяет предварительные требования для всех серверов переднего сервера в пуле, а затем пытается запустить каждый сервер. Если возникают какие-либо проблемы, он диагностирует их и оповещает вас с подробными сведениями и обходными решениями. В некоторых случаях это позволяет запустить пул, даже если некоторые отдельные серверы не могут запуститься.
  
### <a name="sql-server-alwayson-support-for-on-premises-servers"></a>SQL Server AlwaysOn для локального сервера

Skype для бизнеса Server 2015 добавляет поддержку как для SQL Server групп доступности AlwaysOn, так и для SQL Server экземпляров кластера AlwaysOn. Помимо этих функций, Skype для бизнеса Server продолжает поддержку зеркального SQL Server баз данных и кластеров, как в прошлых версиях Lync Server.
  
SQL Server группы доступности AlwaysOn — это решение для высокой доступности и аварийного восстановления в SQL Server 2012 и SQL Server 2014 года, которое предоставляет альтернативу зеркальным отражаниям баз данных. Группа доступности поддерживает отказоупозивную среду для дискретного набора баз данных (которые называются базами данных доступности), которые отлаготились вместе. Группа доступности поддерживает набор основных баз данных для чтения и записи и от одного до четырех наборов соответствующих дополнительных баз данных. При желании дополнительные базы данных можно сделать доступными только для чтения и для некоторых операций резервного копирования.
  
SQL Server экземпляры резервного кластера используют функции windows Server Failover Clustering (WSFC) для обеспечения высокой доступности на локальном уровне за счет избыточности на уровне экземпляра сервера — экземпляра резервного кластера (FCI). FCI — это один экземпляр SQL Server, который устанавливается на узлах windows Server Failover Clustering (WSFC) и, возможно, в нескольких подсетях.
  
Дополнительные сведения см. в плане высокой доступности и аварийного восстановления [в Skype для бизнеса Server 2015.](plan-your-deployment/high-availability-and-disaster-recovery/high-availability-and-disaster-recovery.md)
  
### <a name="centralized-logging-and-troubleshooting-improvements-for-on-premises-servers"></a>Улучшения централизованного ведения журнала и устранения неполадок для локального сервера

Централизованная служба ведения журналов является предпочтительной средой ведения журнала для Skype для бизнеса Server 2015. В этом выпуске нет новых функций, но надежность и производительность были улучшены как для службы, так и для агента службы централизованного ведения журналов (ClsAgent.exe) — исполняемого средства службы, который взаимодействует с контроллером и получает команды, выдаемые администратором.
  
Skype для бизнеса Server 2015 использует Windows PowerShell для управления агентами службы ведения журналов, инициировать трассировку и создавать отчеты. (Для выполнения этих задач используется Lync Server 2013 ClsController.exe Lync Server 2013.)
  
Централизованная служба ведения журналов может работать в любом Skype для бизнеса Server 2015. Встроенные сценарии (предварительно определенные трассировки) остаются одинаковыми, как и возможность создания пользовательских сценариев. Всегда работает специальный сценарий AlwaysOn, который позволяет администраторам находить распространенные проблемы практически в режиме реального времени.
  
Средство отладки Snooper также было обновлено для отладки журналов мобильности и будет работать с устройствами, подключенными к Lync 2013 или Skype для бизнеса Server 2015. Средство доступно в качестве веб-загрузки из средств [отладки.](https://go.microsoft.com/fwlink/?LinkId=285257)
  
## <a name="hybrid-deployment-and-management"></a>Гибридное развертывание и управление

Skype для бизнеса Server 2015 позволяет использовать возможности администрирования гибридного развертывания и управления ими, внедряя следующие функции:
  
- Рекомендации по гибридным развертываниям на основе состояния локального актива клиента, определяемого средством автоматической помощи OnRamp для Office 365.
- Усовершенствования панели управления Skype для бизнеса Server и Центра администрирования Skype для бизнеса Server, с помощью которых администраторы могут управлять гибридным развертыванием.
- Улучшения панели управления, которые администраторы могут войти в клиент Microsoft 365 или Office 365 и настроить гибридную сеть Skype для бизнеса Online с помощью мастера гибридной конфигурации.
- Поддержка панели управления для перемещения пользователей локальной сети в Skype для бизнеса Online или перемещения пользователей Skype для бизнеса Online обратно в локальное.
- Функции панели управления для идентификации и фильтрации объектов локального пользователя, которые были перемещены в Skype для бизнеса Online (то есть гибридных пользователей) от пользователей локальной системы.
- Функции Центра администрирования для идентификации и фильтрации облачных пользователей, изначально созданных в Skype для бизнеса Online, от гибридных пользователей, перенесенных из локальной сети в Online.
- Возможность администрирования гибридных пользователей с помощью панели управления для управляемых свойств из локальной сети и Центра администрирования для управляемых свойств из Skype для бизнеса Online.
- С помощью синхронизации паролей с DirSync можно синхронизировать пароли локальной службы Active Directory с сетевым клиентом. Если эта функция настроена, она устраняет необходимость развертывания AD FS для федерательной проверки подлинности, но она по-прежнему требуется для многофакторной проверки подлинности. 
- Поддержка сосуществования Skype для бизнеса Online и локальной службы Exchange.
    
> [!NOTE]
> Сосуществование и поддержка локальной службы Lync Online 2013 и Exchange не изменились. 
  
## <a name="multi-factor-authentication"></a>Многофакторная проверка подлинности

Многофакторная проверка подлинности — это метод проверки подлинности, который требует использования нескольких методов проверки и добавляет критический второй уровень безопасности для входов и транзакций пользователей. Например, требуется имя пользователя и пароль, а также сертификат. Skype для бизнеса Server 2015 продолжает использовать функции многофакторной проверки подлинности, доступные в накопительных обновлениях Lync Server 2013. Существенные изменения многофакторной проверки подлинности:
  
- Использование библиотеки проверки подлинности Active Directory Office 2013 с sp1 для интеграции с Exchange и SharePoint
- Поддержка многофакторной проверки подлинности в клиенте Skype для бизнеса Web App
    
С помощью многофакторной проверки подлинности Skype для бизнеса теперь можно предоставлять различные параметры проверки подлинности в зависимости от географического региона. Например, клиенты могут настроить среду таким образом, чтобы внутренняя проверка подлинности опиралась на встроенную проверку подлинности Windows, а сотрудники, работающие с проверкой подлинности извне организации, использовали многофакторную проверку подлинности. 
  
Многофакторная проверка подлинности Skype для бизнеса без проблем не зависит от:
  
- Географическое расположение — вход пользователя из организации или за ее пределами 
- Тип клиента или устройства — используемый клиент Skype для бизнеса и устройство, на котором работает клиент (ПК, мобильные устройства, iPad и т. д.)
- Расположение учетной записи — находится ли пользователь в локальной службе Active Directory или в Azure Active Directory Online.
