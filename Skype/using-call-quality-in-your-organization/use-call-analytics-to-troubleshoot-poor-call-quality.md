# Устранение неполадок с качеством звонков в Skype для бизнеса с помощью средства аналитики звонков

> [!IMPORTANT]
> Данная статья переведена с помощью машинного перевода, см. [Отказ от ответственности](66945036-ae87-4c08-a0bb-984e50d6b009.md#MT_Footer). Используйте английский вариант этой статьи, который находится [здесь](https://support.office.com/en-us/article/66945036-ae87-4c08-a0bb-984e50d6b009), в качестве справочного материала. 
  
Средство аналитики звонков позволяет устранять неполадки со звонками и проблемы с соединением в Skype для бизнеса. В нем представлены подробные сведения об устройствах, сетях и соединениях, связанных с конкретными звонками, для каждого пользователя в учетной записи Skype для бизнеса. Если в это средство загружены сведения о здании, участке и клиенте, также будет отображаться доступная информация о каждом звонке и сеансе, что позволяет выявить причины проблем с качеством звонков или собраний для конкретного пользователя. 
  
> [!NOTE]
> На данный момент доступна лишь предварительная версия средства аналитики звонков Описание и изображения в этой статье могут не соответствовать фактическому интерфейсу. 
  
## Устранение неполадок с качеством звонков с помощью средства аналитики звонков

Тип информации, доступной в средстве аналитики звонков, определяется назначенным вам уровнем разрешений:
  
- **Администратор Skype для бизнеса**. Вы получаете доступ ко всей информации в средстве аналитики звонков и Центре администрирования Skype для бизнеса.
    
- **Агент службы технической поддержки с разрешениями уровня 1**. Вы можете просматривать ограниченный набор личных сведений в средстве аналитики звонков и устранять неполадки со звонками, но проблемы с собраниями следует переадресовывать агентам уровня 2. Вы не получаете прав на доступ к другим компонентам Центра администрирования Skype для бизнеса.
    
- **Агент службы технической поддержки с разрешениями уровня 2**. Вы можете просматривать все доступные в средстве аналитики звонков данные, а также устранять неполадки со звонками и собраниями. Вы не получаете прав на доступ к другим компонентам Центра администрирования Skype для бизнеса.
    
Дополнительные сведения о разрешениях вы можете получить у своего администратора Skype для бизнеса.
  
 **Откройте средство аналитики звонков с правами агента службы технической поддержки уровня 1 или 2**
  
1. Перейдите по ссылке [https://adminportal.services.skypeforbusiness.com](https://adminportal.services.skypeforbusiness.com) и введите имя пользователя и пароль для входа в систему.
    
2. В поле **Поиск пользователей** начните вводить имя или SIP-адрес пользователя, для которого требуется устранить неполадки с качеством звонков. Выберите пользователя в списке.
    
    ![Screenshot of the User Search box of Call Analytics in the Skype for Business Admin Center.](../images/db52efc5-dac1-4623-ba72-41e42f0a0fb4.png)
  
3. В разделе **Журнал звонков** выберите звонок или собрание, для которого требуется устранить неполадки.
    
    ![Screenshot shows the call history page for a user with information such as the user's contact details, a summary of the 7-day quality and activity for meetings and calls, and an overview of dates and times, recipients, and audio quality,](../images/aef80e09-3b37-46db-8e7b-8cf71712349b.png)
  
4. Перейдите на вкладку **Дополнительно** и просмотрите элементы, выделенные желтым или красным цветом, которые относятся к проблемам с качеством звонков или соединения.
    
    В сведениях о сеансе для каждого звонка или собрания желтым цветом выделяются незначительные проблемы. Например, на следующем снимке экрана желтым цветом выделены значения "Среднее дрожание", "Максимальное дрожание" и "Средняя скорость потери пакетов. Значения, выделенные желтым цветом, выходят за пределы допустимого диапазона и могут в определенной степени влиять на возникновение проблемы, но в большинстве случаев не являются ее основной причиной. Красным цветом выделяются серьезные проблемы, которые могут быть основной причиной плохого качества звонка в ходе соответствующего сеанса. 
    
    ![Screenshot shows the Advanced tab of a user's Call history with the Inbound network section expanded to reveal that the data for average jitter, max jitter, and average packet loss rate are shown in a yellow color, meaning they are minor issues.](../images/13f314ce-97cf-4bd0-a147-14b177d07040.png)
  
В некоторых случаях для звуковые сеансы не получил качество качества данных. Часто причиной Удаление вызова и подключения с помощью клиента завершает работу. В этом случае сеанс оценки «недоступна».
  
Для звуковые сеансы, которые содержат качество данных взаимодействия в таблице ниже описаны основные проблемы, которые имеете право на сеанс как «низкое».
  
|**Проблема**|**Область**|**Описание**|
|:-----|:-----|:-----|
|Настройка звонка  <br/> |Сеанса  <br/> |Код ошибки Ms-diag 20 29 указывает сбой установки вызова. Пользователь не удалось присоединиться звонка или собрания.  <br/> |
|Сети звука классифицироваться низкого звонка  <br/> |Сеанса  <br/> |Проблемы с качеством сети обнаружены в областях, например потери пакетов, колебания, снижение NMOS, RTT, или скрытого отношение. Дополнительные сведения об условиях используется для классификации низкого вызовы увидеть это [в блоге Microsoft](https://go.microsoft.com/fwlink/p/?linkid=852133).  <br/> |
|Устройство не работает  <br/> |устройство;  <br/> | Устройство не работает правильно. Устройство не работает сжатия являются: <br/>  DeviceRenderNotFunctioningEventRatio > = 0,005 <br/>  DeviceCaptureNotFunctioningEventRatio > = 0,005 <br/> |
