# <a name="media-quality-and-network-connectivity-performance-in-microsoft-teams"></a>Качество мультимедиа и производительность сетевого подключения в Microsoft Teams

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

В этом разделе определен набор требований к производительности сети для служб Microsoft Teams и способ использования Интернета или ExpressRoute для подключения между сетью и сетью Microsoft Teams на основе вашей оценки сетевого подключения. Если вы решили развернуть Azure ExpressRoute для выделенного подключения к Microsoft 365 или Office 365, этот документ также содержит рекомендации по планированию подключений ExpressRoute в различных сценариях Microsoft Teams развертывания.
  
Качество мультимедиа Real-Time (аудио, видео и общий доступ к приложениям) по IP-адресу в значительной части влияет на качество подключения между конечными сетями. Для оптимального Microsoft Teams качества мультимедиа важно обеспечить высокое качество подключения между сетью организации и сетью Microsoft Teams. Для этого лучше всего настроить внутреннее сетевое и облачное подключение на основе пропускной способности сети для максимального объема трафика Microsoft Teams по всем подключениям.
  
Azure ExpressRoute не является требованием для Microsoft 365 и Office 365, включая Microsoft Teams. Однако Azure ExpressRoute — это один из доступных вариантов развертывания, который гарантирует, что подключение к Microsoft 365 или Office 365 соответствует требованиям к производительности сети Microsoft Teams и обеспечивает оптимальное качество Microsoft Teams мультимедиа.
  
> [!TIP]
> Хотя в этой теме приводится общее руководство по производительности сети, полное руководство по оценке сети не является частью этого документа. Чтобы найти список партнеров Microsoft Teams которые могут помочь вам оценить производительность сети в ходе тщательной и полной оценки сети, посетите веб-сайт [Skype для бизнеса Partner Solutions.](http://partnersolutions.skypeforbusiness.com/) 
  
## <a name="network-connectivity-requirements-to-microsoft-teams"></a>Требования к сетевому подключению для Microsoft Teams

### <a name="factors-that-impact-microsoft-teams-media-quality"></a>Факторы, влияющие на Microsoft Teams качества мультимедиа

Существует множество различных факторов, влияющих на качество Microsoft Teams Real-Time мультимедиа (аудио, видео и общий доступ к приложениям), включая используемые устройства, среду и сетевое подключение. 
  
#### <a name="devices"></a>Устройства

Во время Real-Time мультимедиа огромное влияние на общее качество звука и видео оказывают устройства захвата и отрисовки мультимедиа, используемые всеми участниками, такие как гарнитуры и веб-камеры. Устройства более низкого качества или устройства с неправильным драйверами будут выдавать и принимать менее качественный звук и видео. Сертифицированные устройства и устройства высокого качества помогают с подавления эха, фильтрацией шума, разрешением видео и снижением задержки.
  
Хотя сертифицированные звуковые и видеофайл-устройства не требуются, настоятельно рекомендуется использовать устройства, сертифицированные для Microsoft Teams для оптимальной работы с мультимедиа. Список всех устройств, сертифицированных Microsoft Teams, см. в этой [Skype для бизнеса.](../../SfbPartnerCertification/certification/devices-ip-phones.md) Вы можете воспользоваться панелью Microsoft Teams качества звонка [,](/microsoftteams/turning-on-and-using-call-quality-dashboard)которая находится в Центре администрирования **Skype для бизнеса**, чтобы убедиться, что устройства работают правильно и отслеживают качество звука и видео.
  
> [!TIP]
> **Сертифицированное устройство требуется для наиболее оптимального** Skype для бизнеса качества мультимедиа.
  
Важно помнить, что все устройства мультимедиа, Microsoft Teams клиенты и серверы Skype для бизнеса, через которые Real-Time потоки мультимедиа, являются частью задержки. Устройства и программное обеспечение, обрабатывающие задержки, а также сетевые задержки, оказывают значительное влияние на общую задержку и производительность конечных пользователей.
  
#### <a name="environment"></a>Среда

Еще одним важным фактором, влияющим на качество звука и видео, является среда и окружающее пространство, где пользователи могут встречать и использовать звуковые и видео устройства. При звонках пользователей из шумной среды будет эхо, приглушен и нечеткий звук. А пользователи, находящиеся в условиях низкой освещенности, не смогут передавать яркое и четкое видеоизображение. При подготовке конференц-зала непосредственное влияние на качество получаемого звука и изображения оказывает расположение микрофона и видеоустройства.
  
Чтобы получить более четкое представление о звуке и видео для пользователя, используйте Skype для бизнеса в приложении Инструментынастройки Звуковое устройство или Видео устройство, чтобы внести изменения в используемом устройстве и настроить его  >    >   параметры. 

#### <a name="network"></a>Сеть

Качество мультимедиа Real-Time по IP-сети значительно влияет на качество сетевого подключения, но в особенности на объем:
  
- **Задержка** Это время, необходимое для получения IP-пакета из точки A в точку B в сети. Эта задержка распространения сети связана с физическим расстоянием между двумя точками и скоростью света, включая дополнительную нагрузку, наложенную различными маршрутизаторами между ними. Задержка измеряется как однонамерная или время кругового пути (RTT).
    
- **Packet Loss (Потеря пакетов)** Это часто определяется как процент пакетов, которые теряются в заданном периоде времени. Потеря пакетов напрямую влияет на качество звука — отдельные потерянные пакеты почти не оказывают влияния, а сплошные последовательные потери приводят к полному исчезновению звука.
    
- **Дрожать между пакетами или просто дрожать** Это среднее значение изменения задержки между последовательными пакетами. Большинство современных программ VoIP, включая Microsoft Teams, могут адаптироваться к некоторым уровням дрожания путем буферизации. Участник заметит эффект дрожания только в том случае, если дрожание превышает буферику.
    
> [!NOTE]
>  Буферы для дрожания увеличивают задержки между концами.
  
При одновременном использовании многих сеансов Microsoft Teams Real-Time мультимедиа и другого сетевого трафика, генерируемого другими службами Microsoft 365 или Office 365 и другими бизнес-приложениями, важно убедиться, что пропускная способность сети достаточно высока для всей сети, которая соединяет сеть со службой Microsoft Teams, чтобы избежать перегрузки сети и обеспечить отличное качество мультимедиа Real-Time (аудио, видео и общего доступа к приложениям). 
  
#### <a name="implementing-quality-of-service-qos-across-congested-networks"></a>Реализация качества обслуживания (QoS) в сети с загроможаемой сетью

Кроме того, нагрузка на трафик в сети существенно влияет на качество мультимедиа. Чтобы обеспечить более быструю передачу аудио- и видеокаетов по сети и приоритет над другим сетевым трафиком в убываемой сети, можно использовать QoS для обеспечения оптимальной работы пользователей с аудио- и видеосознакомиями.
  
QoS позволяет назначать более высокий приоритет сетевым пакетам, которые имеют звуковые и видеофайлы. При назначении этим пакетам более высокого приоритета аудио- и видеосвязь, скорее всего, будет работать по сети быстрее и с меньшими прерываниями, чем сетевые сеансы, связанные с передачей файлов, просмотром веб-страниц или резервными копиями баз данных. Это потому, что сетевым пакетам, используемым для передачи файлов или резервного копирования баз данных, по умолчанию назначены "наилучшие усилия" в качестве приоритета, а перегрузка сети не будет иметь большого влияния. Если не назначить пакеты мультимедиа (аудио, видео и общий доступ к приложениям) более высоким приоритетом и оставить их на должном уровне, они будут обрабатываться вместе с другим сетевым трафиком. В зависимости от объема сетевой нагрузки это может понизить общее качество звука и видео для пользователей.
  
Настоятельно рекомендуется внедрить QoS в сети, чтобы убедиться, что перегрузка сети в сети не повлияет на сеть. Однако для максимального влияния на это все конечные точки сети должны поддерживать QoS, то есть все конечные точки должны соблюдать маркировку QoS и расценки пакетов. Microsoft Teams учетом маркировки QoS и расценки приоритетов в сети Майкрософт. Однако в трафике, маршрутном через такое открытое подключение, как Интернет из сети организации в сеть Майкрософт, не сохраняются маркировки QoS и расценки пакетов. Частные подключения из сети к Microsoft 365 или Office 365 с помощью [Azure ExpressRoute](https://azure.microsoft.com/services/expressroute/) предлагают решение для развертывания, которое сохраняет маркировку QoS и расценки пакетов, что, в свою очередь, повышает общее качество звука и видео для конечных пользователей.
  
## <a name="network-performance-requirements-to-connect-to-microsoft-teams"></a>Требования к производительности сети для подключения к Microsoft Teams
<a name="bkNetworkPerf"> </a>

Skype для бизнеса Real-Time мультимедиа проходят через различные устройства, клиентские приложения, серверное программное обеспечение и разные сети. End-to-end latency of Real-Time media — это общая задержка, которая вводится во всех компонентах и сегментах сети. Качество конечного сетевого подключения определяется сегментом сети с наихудшим качеством. Этот сегмент является узким местом для сетевого трафика.
  
На следующей схеме показано, как в конференции один участник может Microsoft Teams к другому.
  
![Вызовы expressRoute Flow.](../images/c026e8e5-ba09-42c0-9e03-60fbfda1cb02.png)
  
В этом случае путь к мультимедиа состоит из следующих сетевых сегментов:
  
1. **Подключение от пользователя 1 к границе сети Майкрософт** Обычно оно включает сетевое подключение, например Wi-Fi или Ethernet, подключение по сети WAN от пользователя 1 к точке выходов в Интернет (ваше устройство границы сети), а также подключение к Интернету от границы сети вашей сети к границе сети Майкрософт.
    
2. **Подключение в сети Майкрософт** Находится между Microsoft Edge Microsoft Teams, где используются серверы A/V Conferencing.
    
3. **Подключение в сети Майкрософт** Находится между центром обработки Microsoft Teams и границей сети Майкрософт.
    
4. **Подключение от границы сети Майкрософт к пользователю 2** К таким данным относятся подключение к Интернету от границы сети вашей сети к границе сети Майкрософт, подключение по сети WAN от пользователя 2 к точке выходов в Интернет (границе сети) и сетевое подключение, например Wi-Fi или Ethernet.
    
На следующей схеме показана разбивка компонентов и сетевых сегментов Microsoft Teams ОКП:
  
![Вызов оператора связи через ПСС ExpressRoute Flow.](../images/768a88df-c8a9-4171-a158-565a698f0193.png)
  
В сценарии вызова через ПСС путь мультимедиа пересекает следующие сетевые сегменты:
  
1. **Подключение от Skype для бизнеса клиента к границе сети Майкрософт** Обычно оно включает сетевое подключение, например Wi-Fi или Ethernet, подключение по сети WAN от звоняшего клиента Skype для бизнеса к точке выходов в Интернет (сетевое устройство) и подключение к Интернету от границы сети вашей сети к границе сети Майкрософт.
    
2. **Подключение в сети Майкрософт** Это находится между Microsoft Edge данными Microsoft Teams, где используется сервер-посредник.
    
3. **Подключение в сети Майкрософт** Находится между центром обработки Microsoft Teams и границей сети Майкрософт.
    
4. **Подключение между сетью Майкрософт и партнерами- поставщиками услуг STN** Это подключение, которое существует для вызова через Skype для бизнеса клиента, который находится за пределами сети Майкрософт.
    
### <a name="network-performance-requirements-from-a-skype-for-business-client-to-microsoft-network-edge"></a>Требования к производительности сети от клиента Skype для бизнеса до границы сети Майкрософт
<a name="bkSfBClienttoEdge"></a>

Для оптимального Skype для бизнеса качества мультимедиа для подключения от сети организации к границе сети Майкрософт необходимы следующие целевые показатели производительности сети или пороговые значения. Этот сегмент сети включает внутреннюю сеть, включая все подключения Wi-Fi и Ethernet, трафик компании между сайтом и сайтом по подключению к сети WAN, например MPLS, а также партнерские подключения Через Интернет или ExpressRoute к границе сети Майкрософт.
  
> [!CAUTION]
> **Подключение между клиентом Skype для бизнеса в сети организации к Microsoft 365 или Office 365 должны соответствовать следующим требованиям к производительности сети и пороговых значениях.**

|**Показатель** <br/> |**Target (Назначение)** <br/> |
|:-----|:-----|
|Задержка (один из способов)  <br/> |< 50 мс  <br/> |
|Задержка (RTT или время кругового пути)  <br/> |< 100 мс  <br/> |
|Вспышка потери пакетов  <br/> |<10 % в течение интервала 200 мс  <br/> |
|Потеря пакетов  <br/> |<1 % в течение интервала 15 с  <br/> |
|Дрожать между поступлениями пакетов  <br/> |<30 мс в течение интервала 15 с  <br/> |
|Переупонка пакетов  <br/> |<пакеты с пакетами, не в порядке порядка 0,05 %.  <br/> |
   
 **Другие целевые требования к производительности:**
  
- Сеть Майкрософт имеет более 160 местоположений Edge по всему миру. Мы работаем с основными поставщиками услуг Интернета (ISP) по всему миру через эти edge-сайты. Целевой показатель задержки предполагает, что ваш сайт или сайты компании и границы Майкрософт находятся на одном континенте.
    
- К сайту организации или сайтам к границе сети Майкрософт относится сетевой доступ первого перехода, который может быть Wi-Fi или другой беспроводной технологией. 
    
- Целевой показатель производительности сети предполагает надлежащее планирование пропускной способности и (или) качества обслуживания. Другими словами, это относится непосредственно Skype для бизнеса Real-Time трафику мультимедиа, когда сетевое подключение находится под пиковой нагрузкой.
    
### <a name="network-performance-requirements-from-your-network-edge-to-microsoft-network-edge"></a>Требования к производительности сети от границы сети до границы сети Майкрософт

Ниже ются целевые или пороговые значения производительности сети, необходимые для подключения между границей сети вашей сети и границей сети Майкрософт. Этот сегмент сети исключает внутреннюю сеть или сетевую сеть клиента и предназначен в качестве руководства при тестировании сетевого трафика, отосланного через Интернет или через партнерскую сеть ExpressRoute, а также может использоваться при подписании соглашения об уровне обслуживания (SLA) с поставщиком ExpressRoute.
  
> [!CAUTION]
> **Подключение между границей сети организации и границей сети Майкрософт должно соответствовать следующим требованиям и порогам производительности сети.**

 **Другие целевые требования к производительности:**

- Целевой показатель производительности сети предполагает надлежащее планирование пропускной способности и (или) качества обслуживания. Это также относится Skype для бизнеса Real-Time трафику мультимедиа, когда сетевое подключение находится под пиковой нагрузкой. Для правильной пропускной способности и планирования QoS обратитесь к [ExpressRoute и QoS в Microsoft Teams.](https://support.office.com/article/ExpressRoute-and-QoS-in-Skype-for-Business-Online-20c654da-30ee-4e4f-a764-8b7d8844431d)
    
## <a name="measuring-network-performance"></a>Измерение производительности сети
<a name="bkNetworkPerf"> </a>

 **Ниже следуют рекомендации высокого уровня по оценке производительности сети.**
  
- Необходимо оценить внутреннюю сеть и подключения к Microsoft 365 или Office 365.
    
- Следует оценивать и собирать данные по всем сетям за длительный период времени. Рекомендуем вам выполнить тестирование производительности сети не менее недели, чтобы увидеть шаблоны использования для всех бизнес-дней и часов. Это позволит вам показать время пиковой нагрузки.

Использование сети меняется с течением времени из-за изменения шаблона использования, новых корпоративных приложений, которые используют большой объем пропускной способности, и изменений в расположении организации или организации. Важно постоянно отслеживать производительность сети с учетом этих требований к производительности сети, целевых и пороговых значений и вносить в нее вовремя корректировки, чтобы обеспечить наиболее оптимальное Real-Time качеством мультимедиа.

## <a name="media-quality-and-expressroute"></a>Качество мультимедиа и ExpressRoute
<a name="bkNetworkPerf"> </a>

Azure ExpressRoute для Microsoft 365 или Office 365 — это выделенное сетевое подключение для подключения к Microsoft 365 или Office 365. Она позволяет клиентам управлять маршрутом сетевого трафика. Им больше не нужно беспокоиться о непредсказуемой маршрутике, которая происходит в Интернете, где данные переносят неизвестные операторы, поставщики услуг Интернета и поставщики услуг Интернета. Сетевой трафик, отсланный через ExpressRoute, отправляется непосредственно через сеть партнера ExpressRoute в сеть Майкрософт. Это позволяет клиентам обрабатывать Microsoft 365 или Office 365 как если бы они находились в их собственных центрах обработки данных вне сайта со выделенным подключением.
  
Azure ExpressRoute доступен для всех Microsoft 365 и Office 365 лицензирования. Однако надстройка Azure ExpressRoute Premium для Microsoft 365 и Office 365, чтобы включить глобальную маршрутику. Клиенты с не менее чем 500 мест, внедряющие ExpressRoute, могут получить необходимую надстройку *ExpressRoute Premium* без дополнительных расходов.
  
### <a name="is-expressroute-required-for-good-media-quality"></a>Требуется ли ExpressRoute для высокого качества мультимедиа?

Azure ExpressRoute не является требованием для получения наиболее оптимального Microsoft Teams качества мультимедиа. Однако это один из вариантов развертывания, который помогает убедиться, что облачное подключение соответствует целевым Skype для бизнеса производительности сети.
  
Microsoft 365 и Office 365 высокой производительности и безопасности служб, которые используют Интернет. Мы продолжаем улучшать новые возможности безопасности и региональные узлы edge для непрерывного повышения безопасности и производительности. Azure ExpressRoute не является требованием для Microsoft 365 или Office 365, включая Microsoft Teams. Azure ExpressRoute — один из доступных вариантов развертывания, который гарантирует, что подключение к Microsoft 365 или Office 365 соответствует требованиям к производительности сети Skype для бизнеса и обеспечивает наиболее оптимальное качество Microsoft Teams мультимедиа.
  
Для Microsoft Teams качества мультимедиа важно, чтобы подключение между сайтами организации и границей сети Майкрософт соответствовало целевым показателям производительности в требованиях к производительности сети от клиента [Skype для бизнеса](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge) к границе сети Майкрософт и что подключение между границей сети организации и границей сети Майкрософт соответствует целевым показателям производительности в требованиях к производительности сети от границы сети организации до границы сети [Майкрософт.](media-quality-and-network-connectivity-performance.md#bkYourNetworkEdge)  
  
Кроме того, важно, чтобы физическое сетевое подключение вашей компании, включая внутреннее сетевое и облачное подключение, было рассчитано на пиковый объем медиапотока. Azure ExpressRoute — это один из многих способов, которые помогут клиентам обеспечить Microsoft Teams облачным подключением, отвечающим всем этим требованиям к производительности.
  
### <a name="is-expressroute-required-for-voice-quality-sla"></a>Требуется ли ExpressRoute для SLA качества голосовой почты?

Нет, ExpressRoute не требуется для SLA Microsoft Teams качества голосовой почты. Стандарт [SLA Microsoft Teams](http://www.microsoftvolumelicensing.com/DocumentSearch.aspx?Mode=3&amp;DocumentTypeId=37) качества голосовой почты применяется ко любым подходящим звонкам, Microsoft Teams пользователям голосовой службы в рамках нужной лицензии и подписки, которые позволяют этому пользователю звонить по voIP или ПСПУ любого типа. SLA о качестве голосовой почты должен включать в себя следующие условия:
  
- Звонки с сертифицированных IP-телефонов Майкрософт.
    
- Проводные ethernet-подключения.
    
- Проблемы с качеством голосовой связи из-за проблем с сетью Майкрософт.
    
> [!NOTE]
> SLA о качестве голосовой связи исключает звонки, для которых низкое качество звонков вызвано проблемами в сетях вне Майкрософт, включая партнер ExpressRoute и других сетях. 
  
### <a name="internet-or-azure-expressroute"></a>Интернет или Azure ExpressRoute?

Прежде чем принимать решение о параметрах сетевого подключения Microsoft Teams, клиенты должны оценить свое сетевое и текущее подключение к Интернету на основе требований к производительности сети, описанных в требованиях к производительности сети для подключения к [Microsoft Teams.](media-quality-and-network-connectivity-performance.md#bkNetworkPerf)
  
Если производительность сети по текущему подключению к Интернету настроена на достаточный период пиковой нагрузки и соответствует требованиям к производительности сети от сайтов до краев сети Майкрософт и от краев сети до краев сети Майкрософт, вы можете продолжать использовать существующее подключение к Интернету для подключения к Microsoft Teams.
  
Для сайтов компании, где требования к производительности сети не выполнены, настоятельно рекомендуем сначала поработать с существующими поставщиками сетевых услуг, чтобы повысить общую производительность сети. Однако если они по-прежнему не выполнены, с помощью Azure ExpressRoute можно обеспечить Microsoft Teams облачное подключение поможет вам выполнить требования к производительности сети.
  
Azure ExpressRoute предоставляет следующие дополнительные преимущества:
  
- Соглашение об уровне обслуживания (SLA) о доступности подключения между вашей сетью и сетью Майкрософт. В ExpressRoute есть SLA с гарантированной доступностью, которая составляет 99,9 %.
    
- Запланированная и гарантированная пропускная способность, необходимая для Microsoft 365 и Office 365 служб. Это можно сделать, отправив только трафик Microsoft 365, Office 365 или Skype для бизнеса через ExpressRoute, а затем весь интернет-трафик будет проходить через другие точки точки доступа к Интернету.
    
- ExpressRoute предназначен для сохранения маркировки QoS DSCP между вашей сетью и сетью Майкрософт.
    
Дополнительные сведения о QoS и планировании емкости ExpressRoute можно найти в теме [ExpressRoute и QoS в Microsoft Teams.](https://support.office.com/article/ExpressRoute-and-QoS-in-Skype-for-Business-Online-20c654da-30ee-4e4f-a764-8b7d8844431d)
  
### <a name="can-i-set-up-azure-expressroute-for-microsoft-teams-only"></a>Можно ли настроить Azure ExpressRoute только для Microsoft Teams?

Да, вы можете настроить Azure ExpressRoute, чтобы обеспечить отличное сетевое подключение из сети организации только Microsoft Teams. Это обеспечит оптимальное качество Real-Time мультимедиа для пользователей, но вы сможете и дальше подключаться к другим службам Microsoft 365 или Office 365 через Интернет.
  
Протокол border Gateway Protocol (BGP) — это протокол маршрутизинга в Интернете, который используется для маршрутизки сетевого трафика через Интернет. Он предназначен для обмена данными маршрутов между автономными системами (AS), которые находятся в Интернете. Значения сообществ BGP — это теги атрибутов, которые можно применять к входящие и исходячие маршруты. Сообщества BGP часто используются для сигнала принимающим as, исходящие ссылки на которые будут использоваться для достижения данной цели на основе географического положения, типа службы или других критериев.
  
При поддержке сообществ BGP Корпорация Майкрософт помечает префиксы и маршруты соответствующими значениями сообщества BGP на основе службы, в службе, в которые они входят. Майкрософт помечает префиксы, объявленные через общедоступный пиринг и пиринг Майкрософт, соответствующими значениями сообщества BGP, указывающими регион, в котором размещаются префиксы. Вы можете использовать значения сообщества для принятия решений по маршрутике, чтобы обеспечить оптимальную маршрутику. С помощью значения Microsoft Teams BGP можно настроить подключение ExpressRoute только для Microsoft Teams. Дополнительные узнать об этом можно в [требованиях к маршрутике ExpressRoute.](/azure/expressroute/expressroute-routing)

### <a name="online-only-deployment---multiple-sites-on-different-continents"></a>Развертывание только в Интернете — несколько сайтов на разных континентах

Если все пользователи используют службу Microsoft Teams и офисы находятся в разных физических расположениях на нескольких континентах, при развертывании Azure ExpressRoute необходимо настроить по крайней мере одно подключение ExpressRoute для каждого континента между основным сайтом на континенте и ближайшим расположением пиринга [ExpressRoute.](/azure/expressroute/expressroute-locations) В зависимости от затрат и преимуществ вы можете развернуть дополнительные подключения ExpressRoute с сайтов, где целевые показатели производительности не выполнены.
  
В следующем примере Contoso — это крупная юридическая компания с офисами в крупных городах Северной Америки и Европы. С учетом подключения к Интернету и оценки производительности внутренней сети компания Contoso решила развернуть два подключения ExpressRoute в Северной Америке и один канал ExpressRoute для всех своих европейских офисов.

### <a name="hybrid-deployment"></a>Гибридное развертывание

Если у вас есть локальное развертывание Lync или Microsoft Teams и вы решили внедрить гибридную интеграцию Microsoft Teams, мы рекомендуем, чтобы при развертывании Azure ExpressRoute было необходимо по крайней мере одно подключение ExpressRoute для каждого локального сайта Lync или Microsoft Teams Edge и хотя бы одно подключение ExpressRoute для каждого континента с офисами. В зависимости от затрат и преимуществ на каждом континенте можно развернуть дополнительные подключения ExpressRoute из офисов, где целевые показатели производительности не выполнены.
  
Если у вас есть локальное развертывание Microsoft Teams, следуйте руководству по планированию и [развертыванию Edge Server.](../../SfbServer/plan-your-deployment/edge-server-deployments/edge-server-deployments.md) В частности, внешние серверы должны быть недоступны. Обычно для этого необходимо назначить маршрутируемый общедоступный IP-адрес на edge-сервере или с помощью перевода сетевых адресов (NAT).
  
В следующем примере у Contoso уже есть локальное развертывание Microsoft Teams Корпоративная голосовая связь развертывания. Они хотят перенести пользователей из локальной службы в Microsoft 365 или Office 365 службы. Кроме того, было принято решение о гибридном развертывании, которое позволяет продолжать использовать существующую инфраструктуру STN для всех пользователей локальной и сетевой сети. Локальное центр обработки данных Contoso и Skype для бизнеса Edge Servers находятся в Чикаго. Для развертывания компания Contoso решила настроить одно подключение ExpressRoute между центром обработки данных в Чикаго и Чикаго ExpressRoute. Кроме того, они добавили подключение ExpressRoute на западное побережье, чтобы лучше обслуживать офис в Гонолулу.

### <a name="online-deployment-with-cloud-connector-edition"></a>Развертывание Online с Cloud Connector Edition

Skype для бизнеса Cloud Connector Edition — это гибридное предложение, которое состоит из набора упакованных виртуальных машин (ВМ), реализующих локальное подключение по STN. Развертывая минимальную топологию Skype для бизнеса Server виртуализированной среде, вы сможете отправлять и принимать звонки с стационарных и мобильных телефонов с помощью существующей локальной инфраструктуры голосовой связи ПСС.
  
Если вы решите развернуть Azure ExpressRoute и Cloud Connector Edition, рекомендуем настроить по крайней мере одно подключение Express Route для каждого континента между основным сайтом на каждом континенте и ближайшим расположением пиринга [ExpressRoute.](/azure/expressroute/expressroute-locations) В зависимости от затрат и преимуществ на каждом континенте можно развернуть дополнительные подключения ExpressRoute с сайтов, где целевые показатели производительности не выполнены.
  
Если у вас есть локальное развертывание Microsoft Teams, необходимо следовать руководству по планированию [для Skype для бизнеса Cloud Connector Edition.](../../SfbServer/skype-for-business-hybrid-solutions/plan-your-phone-system-cloud-pbx-solution/plan-skype-for-business-cloud-connector-edition.md) В частности, службам Edge Access и A/V Edge должны быть назначены общедоступные IP-адреса и доступны Microsoft 365 или Office 365 центрам обработки данных.
  
В следующем примере Contoso является европейской бухгалтерской фирмой, которая находится в нескольких крупных европейских странах и городах. При регистрации в Microsoft Teams для совместной работы было принято решение создать облачный соединитель для каждой страны и региона, в которое они могут продолжать использовать инфраструктуру ННР и уже существовав контракты с операторами связи. На основании тестирования на всех сайтах и в границе сети Майкрософт они определили, что одно подключение ExpressRoute в Лондоне поможет совместить с целевыми показателями производительности сети Microsoft Teams, описанными в Skype для бизнеса требованиях к производительности сети от клиента Skype для бизнеса до границы сети [Майкрософт.](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge)
