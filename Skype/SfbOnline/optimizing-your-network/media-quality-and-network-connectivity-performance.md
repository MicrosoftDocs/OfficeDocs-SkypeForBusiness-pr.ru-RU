# <a name="media-quality-and-network-connectivity-performance-in-skype-for-business-online"></a>Качество медиаданных и производительность сетевого подключения в Skype для бизнеса Online

В этой статье описаны требования к производительности сети для служб Skype для бизнеса Online и способы их использования в Интернете и ExpressRoute для связи между вашей сетью и Skype для бизнеса Online в соответствии с оценкой сетевого подключения. Если вы решили развернуть Azure ExpressRoute для выделенного подключения к Office 365, в этом документе также содержатся инструкции по планированию подключений ExpressRoute в разных сценариях развертывания Skype для бизнеса Online.
  
Качество мультимедиа в режиме реального времени (передача звука, видео и общего доступа к приложениям) на уровне IP существенно влияет на качество подключения к сети. Для достижения оптимального качества передачи данных при использовании Skype для бизнеса Online очень важно обеспечить максимально эффективное подключение между сетью организации и Skype для бизнеса Online. Наилучшим способом является настройка внутреннего сетевого и облачного подключения на основе производительности сети с целью поддержки Skype для бизнеса Online в периоды наибольшей нагрузки.
  
Azure ExpressRoute не является обязательным требованием для служб Office 365, в том числе Skype для бизнеса Online. Однако Azure ExpressRoute — это один из доступных вариантов развертывания, которые помогут удостовериться в том, что подключение к Office 365 соответствует требованиям к производительности сети Skype для бизнеса и обеспечивает оптимальное качество мультимедиа в Skype для бизнеса Online.
  
> [!TIP]
> Несмотря на то, что этот раздел содержит общее руководство по производительности сети, полные инструкции по оценке сети находятся за пределами области этого документа. Чтобы найти список партнеров Skype для бизнеса Online, с помощью которых можно получить измерения производительности сети в рамках полной и полной оценки сети, посетите страницу [партнерских решений Skype для бизнеса](http://partnersolutions.skypeforbusiness.com/). 
  
## <a name="network-connectivity-requirements-to-skype-for-business-online"></a>Требования к сетевому подключению к Skype для бизнеса Online

### <a name="factors-that-impact-skype-for-business-online-media-quality"></a>Факторы, влияющие на качество мультимедиа в Skype для бизнеса Online

В Skype для бизнеса Online доступно множество различных факторов (звук, видео и общий доступ к приложениям), включая используемые устройства, среду и сетевое подключение. 
  
#### <a name="devices"></a>Устройства

В сеансе мультимедиа в реальном времени устройства для захвата и отрисовки мультимедиа, используемые всеми участниками, такими как гарнитуры и веб-камеры, имеют отличное влияние на качество звука и видео. Устройства более низкого качества или устройства с неправильным драйверами будут выдавать и принимать менее качественный звук и видео. С другой стороны, сертифицированные или высококачественные устройства позволяют подавлять эхо, фильтровать шумы, поддерживать оптимальное разрешение видеоизображения и уменьшать задержки.
  
Несмотря на то, что сертифицированные звуковые и видеоустройства не являются обязательными, настоятельно рекомендуется использовать для Skype для бизнеса наиболее оптимальные устройства для работы с мультимедиа. Список всех сертифицированных устройств Skype для бизнеса можно найти в разделе [телефоны и устройства в Skype для бизнеса](https://technet.microsoft.com/office/dn947482). Вы можете использовать [панель мониторинга качества звонков Skype для бизнеса Online](/microsoftteams/turning-on-and-using-call-quality-dashboard), которая находится в **центре администрирования Skype для бизнеса**, чтобы проверить, правильно ли работают используемые устройства, а также отслеживать качество аудио-и видеофайлов.
  
> [!TIP]
> **Сертифицированное устройство требуется для оптимального качества мультимедиа в Skype для бизнеса**.
  
Важно помнить, что все устройства мультимедиа, клиенты Skype для бизнеса и серверы Skype для бизнеса, с помощью которых ведется поток мультимедиа в реальном времени, вводят некоторую задержку. Задержка обработки устройств и программного обеспечения вместе с задержкой сети, имеет значительное влияние на общую задержку и работу конечных пользователей.
  
#### <a name="environment"></a>Среда

Другим важным фактором, от которого зависит качество звука и видео, является среда и окружающее пространство, где пользователи проводят свои встречи и используют аудио- и видеоустройства. Звук, принимаемый во время звонка от пользователей из шумной обстановки, будет приглушенным, нечетким или будет отдаваться эхом. А пользователи, находящиеся в условиях низкой освещенности, не смогут передавать яркое и четкое видеоизображение. При подготовке конференц-зала непосредственное влияние на качество получаемого звука и изображения оказывает расположение микрофона и видеоустройства.
  
Чтобы получить более четкий снимок звука и видео, используйте **средства** > **Options** > приложения Skype для бизнеса "**звуковые устройства** " или " **видеоустройство** ", чтобы изменить параметры устройства на работе и настроить его настройки.

#### <a name="network"></a>Сеть

Качество сети мультимедиа в режиме реального времени значительно влияет на качество сетевого подключения, но особенно в указанных ниже случаях.
  
- **Задержка** Это время, необходимое для получения IP-пакета от а до точки B в сети. Эта задержка распространения сети по сути связана с физическим расстоянием между двумя точками и скоростью света, включая дополнительные накладные расходы, взятые различными маршрутизаторами между ними. Задержка измеряется как одно-или время кругового пути (RTT).
    
- **Потеря пакетов** Это часто определяется в процентах от количества пакетов, которые теряются в течение заданного окна. Потери пакетов прямо влияют на качество звука — от небольшого до отдельных потерянных пакетов, которые практически не повлияли, для обратной связи между пределами, которые приводят к вырезке полного звука.
    
- **Нарушение колебаний межпакетной связи или просто перерыв** Это среднее значение задержки между последовательными пакетами. Большинство современных программ для VoIP, в том числе Skype для бизнеса, могут адаптироваться к определенным уровням колебаний с помощью буферизации. Это может быть только в том случае, если нарушение превышают размер буфера, который участник пометит на эффект колебаний.
    
> [!NOTE]
>  Буферизация для колебаний увеличивает задержку на конец.
  
Большое количество одновременных сеансов видеосвязи в Skype для бизнеса Online в реальном времени и другой сетевой трафик, созданный другими службами Office 365 и другими бизнес-приложениями, гарантирует достаточную пропускную способность по всему сетевому пути, который подключает сеть к службе Skype для бизнеса Online, чтобы избежать перегрузки сети и обеспечить высокое удобство мультимедиа в реальном времени (звук). , видео и общий доступ к приложениям). 
  
#### <a name="implementing-quality-of-service-qos-across-congested-networks"></a>Реализация качества обслуживания (QoS) в перегруженных сетях

Кроме того, перегрузка трафика в сети значительно повлияет на качество мультимедиа. Чтобы обеспечить оптимальное взаимодействие для голосовой и видеосвязи, можно использовать для звуковых и видеофайлов ускоренную работу сети, а также назначать приоритеты по сравнению с другим сетевым трафиком в перегруженной сети, качества обслуживания (QoS).
  
Качество обслуживания позволяет назначать более высокие приоритеты сетевым пакетам, которые поддерживают звуковые и видеоданные. Назначая более высокий приоритет этим пакетам, голосовая и видеосвязь-связь, скорее чем, перемещается по сети и с меньшим перерывом, чем сетевые сеансы, такие как передача файлов, просмотр веб-страниц или резервные копии базы данных. Это связано с тем, что сетевые пакеты, которые используются для передачи файлов или резервных копий базы данных по умолчанию, назначены "Оптимальная работа" в качестве приоритета, а перегрузка сети не повлияют на большое значение. Если вы не назначаете пакеты с более высоким приоритетом (звук, видео и общий доступ к приложениям) и оставите их также как "лучшие усилия", они тоже будут обрабатываться вместе с любым другим сетевым трафиком. В зависимости от количества перегрузки сети это может привести к уменьшению общего качества звука и видео для пользователей.
  
Настоятельно рекомендуется реализовать качество обслуживания в сети, чтобы убедиться в том, что перегрузка сети в сети не повлияет. Тем не менее, в целях достижения максимально возможного воздействия все конечные точки сети должны поддерживать QoS, что означает, что все конечные точки должны учитывать маркировку QoS и определение приоритетов пакетов. Службы Skype для бизнеса Online учитывают маркировку QoS и определение приоритетов в сети Microsoft. Тем не менее трафик, который передается через открытое подключение, например Интернет из корпоративной сети, не сохраняет метки QoS и определение приоритетов пакетов. Личные подключения из сети в Office 365 с помощью [Azure ExpressRoute](https://azure.microsoft.com/services/expressroute/) предлагают решение для развертывания, которое сохраняет маркировку QoS и определение приоритетов пакетов, которые в свою очередь увеличивают общее качество звука и видео для конечных пользователей.
  
## <a name="network-performance-requirements-to-connect-to-skype-for-business-online"></a>Требования к производительности сети для подключения к Skype для бизнеса Online
<a name="bkNetworkPerf"> </a>

Носители в реальном времени Skype для бизнеса продаются с помощью множества различных устройств, клиентских приложений, серверного программного обеспечения и разных сетей. Сквозная задержка для мультимедиа в реальном времени — это общая величина задержки, которая появилась на всех компонентах и сегментах сети. Качество однорангового сетевого соединения определяется сетевым сегментом, в котором используется Наихудшее качество. Этот сегмент действует в качестве узкого места для этого сетевого трафика.
  
На приведенной ниже схеме показан односторонняя поток звука на Конференции из одного участника Skype для бизнеса в другой.
  
![Поток звонков ExpressRoute.](../images/c026e8e5-ba09-42c0-9e03-60fbfda1cb02.png)
  
В этом сценарии для Конференции путь к носителю состоит из следующих сегментов сети:
  
1. **Соединение между пользователем 1 и краем Microsoft Network** Обычно это включает сетевое подключение, например WiFi или Ethernet, подключение к глобальной сети от пользователя 1 к точке выхода в Интернет (на устройстве с сетевым интерфейсом) и подключение к Интернету с сетевого края до Microsoft Network Edge.
    
2. **Подключение в сети Microsoft** Это может быть между Microsoft EDGE и Интернет-центром обработки данных Skype для бизнеса Online, в котором используются серверы конференц-связи с/V.
    
3. **Подключение в сети Microsoft** Это между центром обработки данных Skype для бизнеса Online и Microsoft Network Edge.
    
4. **Подключение из Microsoft Network Edge к пользователю 2** Сюда входит подключение к Интернету из сетевого края в сеть Microsoft Network EDGE, подключение к глобальной сети от пользователя 2 к точке выхода в Интернет (к сетевому краю), а также сетевое подключение, например WiFi или Ethernet.
    
На приведенной ниже диаграмме показано, как выделена подразделение компонентов и сегментов сети для звонков в Skype для бизнеса Online PSTN:
  
![Поток звонков для коммутируемой телефонной сети ExpressRoute.](../images/768a88df-c8a9-4171-a158-565a698f0193.png)
  
В случае КОММУТИРУЕМого звонка путь к носителю пересекает следующие сегменты сети:
  
1. **Подключение клиента Skype для бизнеса к краю сети Microsoft** Обычно это включает в себя сетевое подключение, например WiFi или Ethernet, подключение к глобальной сети из клиента Skype для бизнеса, вызывающее клиентскую точку выхода в Интернет (сетевое устройство с сетевым подключением) и Интернет-соединение с краем сети до Microsoft Network Edge.
    
2. **Подключение в сети Microsoft** Это между Microsoft EDGE и центром обработки данных Skype для бизнеса Online, в котором используется сервер-посредник.
    
3. **Подключение в сети Microsoft** Это между центром обработки данных Skype для бизнеса Online и Microsoft Network Edge.
    
4. **Связь между сетью Microsoft и партнерами поставщика услуг PSTN** Это соединение, которое используется для помещения звонка из клиента Skype для бизнеса, находящегося за пределами сети Microsoft.

- Конечный объект производительности сети подразумевает надлежащую пропускную способность и/или планирование обслуживания. Другими словами, это применимо к мультимедийному трафику в реальном времени в Skype для бизнеса, когда сетевое подключение находится в максимальной нагрузке.

- Конечный объект производительности сети подразумевает надлежащую пропускную способность и/или планирование обслуживания. Это также относится к мультимедийному трафику в реальном времени в Skype для бизнеса, когда сетевое подключение находится на пиковой загрузке. ## <a name="media-quality-and-expressroute"></a>Качество мультимедиа и ExpressRoute
<a name="bkNetworkPerf"> </a>

Azure ExpressRoute для Office 365 — это выделенное сетевое подключение для подключения к Office 365. Она дает клиентам возможность контролировать путь к сетевому трафику Office 365. Теперь им больше не придется беспокоиться о непредсказуемой маршрутизации, которая происходит в Интернете, где данные переносятся неизвестными поставщиками, провайдерами и поставщиками услуг Интернета. Сетевой трафик, отправляемый через ExpressRoute, отправляется непосредственно через сеть партнера ExpressRoute в сеть Microsoft. Это позволит клиентам рассматривать Office 365 так, как если бы они находились в отдельном центре обработки данных с выделенным подключением.
  
Azure ExpressRoute доступен для всех предложений лицензирования Office 365. Тем не менее, надстройка Azure ExpressRoute Premium требуется для поддержки глобальной маршрутизации в Office 365. Пользователи, имеющие по крайней мере 500 мест, которые реализуют ExpressRoute, могут бесплатно получить необходимую *надстройку Expressroute Premium* .
  
### <a name="is-expressroute-required-for-good-media-quality"></a>Требуется ли для работы с мультимедиа в ExpressRoute качество для воскачественной видеосвязи?

Azure ExpressRoute не является обязательным требованием для получения оптимального качества мультимедиа в Skype для бизнеса Online. Тем не менее, это может быть один из вариантов развертывания, которые помогут вам убедиться в том, что ваше подключение к облаку соответствует конечным объектам производительности в сети Skype для бизнеса или пороговым значениям.
  
Office 365 – это высокопроизводительная и безопасная служба, использующая Интернет. Мы продолжаем вкладывать новые возможности безопасности и узлы региональных краев, чтобы повысить безопасность и производительность. Azure ExpressRoute не является обязательным требованием для служб Office 365, в том числе Skype для бизнеса Online. Azure ExpressRoute — это один из доступных вариантов развертывания, которые помогут удостовериться в том, что подключение к Office 365 соответствует требованиям к производительности сети Skype для бизнеса, и обеспечивает оптимальное качество работы мультимедиа в Skype для бизнеса Online.
  
Для качества мультимедийных файлов в Skype для бизнеса Online важно, чтобы соединение между сайтами Организации и краями сети соответствовало [требованиям к производительности сети от клиента Skype для бизнеса до Microsoft Network Edge](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge) , и что связь между краями сети и краями Microsoft Network Edge отвечает требованиям к производительности в соответствии [с требованиями к производительности сети, предъявляемыми сетевым краем к Microsoft Network](media-quality-and-network-connectivity-performance.md#bkYourNetworkEdge)Edge.  
  
Кроме того, очень важно, чтобы физическая сеть вашей компании была подключена к сети, в том числе внутренняя производительность сети и подключение к облаку. Azure ExpressRoute – это один из многочисленных способов, которые помогут клиентам гарантировать, что их подключение к облачному облаку Skype для бизнеса Online будет соответствовать всем этим требованиям к производительности.
  
### <a name="is-expressroute-required-for-voice-quality-sla"></a>Требуется ли ExpressRoute для качества обслуживания голосовой связи?

Нет, ExpressRoute не требуется для соглашения об уровне обслуживания голосовой связи в Skype для бизнеса Online. [Соглашение об уровне обслуживания голосовой связи в Skype для бизнеса Online](http://www.microsoftvolumelicensing.com/DocumentSearch.aspx?Mode=3&amp;DocumentTypeId=37) распространяется на все подходящие звонки, размещенные любым пользователем голосовой почты Skype для бизнеса Online в правильной лицензии и подписке, которая позволяет этому пользователю совершать звонки по протоколу VoIP или КТСОП. Соглашение об уровне обслуживания голосовой связи должно включать в себя все указанные ниже условия.
  
- Звонки с сертифицированных IP-телефонов Майкрософт.
    
- Подключения к сети Ethernet.
    
- Проблемы с качеством голосовой связи из-за проблем в сети Майкрософт.
    
> [!NOTE]
> Соглашение об уровне обслуживания голосовой связи исключает такие звонки, в которых качество сигнала может быть вызвано проблемами в сетях, отличных от Microsoft, включая партнера ExpressRoute и других сетей. 
  
### <a name="internet-or-azure-expressroute"></a>Интернет или Azure ExpressRoute?

Прежде чем принимать решение о вариантах сетевого подключения к Skype для бизнеса Online, пользователи должны оценить их сетевое и текущее подключение к Интернету согласно требованиям к производительности сети, описанным в разделе Требования к производительности сети, [чтобы подключиться к Skype для бизнеса Online](media-quality-and-network-connectivity-performance.md#bkNetworkPerf).
  
Если в течение пикового подключения к Интернету настраивается производительность сети и она отвечает требованиям к производительности сети от сайтов до краев Microsoft Network, а также от краев сети до Microsoft Network, вы можете продолжать использовать существующее подключение к Интернету, чтобы подключиться к Skype для бизнеса Online.
  
Для сайтов компании, в которых не удовлетворены требования к производительности сети, настоятельно рекомендуется сначала ознакомиться с существующими поставщиками сетевых услуг для повышения общей производительности сети. Тем не менее, если они еще не соблюдаются, с помощью Azure ExpressRoute вы можете убедиться, что ваше облачное подключение к Skype для бизнеса Online поможет вам удовлетворить требования к производительности сети.
  
Azure ExpressRoute предоставляет следующие дополнительные преимущества:
  
- Соглашение об уровне обслуживания (SLA) по доступности соединения между вашей сетью и сетью Microsoft. ExpressRoute имеет гарантированное соглашение о доступности 99,9%.
    
- Запланированная и гарантированная пропускная способность, необходимая для служб Office 365. Это можно сделать, отправив трафик по протоколу Office 365 или Skype для бизнеса с помощью ExpressRoute и попросите все прочие Интернет-трафик пройти через другую точку выхода и входные точки для вашей сети.
    
- ExpressRoute разработан для сохранения пометок QoS с помощью DSCP между вашей сетью и сетью Microsoft.
    
Дополнительные сведения о QoS и планировании ресурсов ExpressRoute можно найти в статье [ExpressRoute и QoS в Skype для бизнеса Online](https://support.office.com/article/ExpressRoute-and-QoS-in-Skype-for-Business-Online-20c654da-30ee-4e4f-a764-8b7d8844431d).
  
### <a name="can-i-setup-azure-expressroute-for-skype-for-business-online-only"></a>Можно ли настроить Azure ExpressRoute только для Skype для бизнеса Online?

Да, вы можете настроить Azure ExpressRoute для обеспечения доступа к сети из сети компании только в Skype для бизнеса Online. Это обеспечит оптимальное качество мультимедиа в режиме реального времени для пользователей, но после этого вы сможете продолжить подключение к другим службам Office 365 через Интернет.
  
Протокол BGP — это протокол маршрутизации в Интернете, который используется для маршрутизации сетевого трафика через Интернет. Оно предназначено для обмена сведениями о маршрутизации между автономными системами (как), которые находятся в Интернете. Значения сообществ BGP — это теги атрибутов, которые можно применять к входящим и исходящим маршрутам. Сообщества BGP часто используются для сигнализации приема, так как исходящие ссылки, которые используются для достижения определенного места назначения, зависят от географии, типа службы или других условий.
  
С помощью поддержки сообществ BGP Microsoft размечет префиксы и маршруты, используя соответствующие значения сообщества BGP на основе службы, к которой они принадлежат. Корпорация Майкрософт размечет префиксы, объявленные с помощью общедоступного пиринга и пиринга Майкрософт, с соответствующими значениями сообщества BGP, указывающими на область, в которой размещены префиксы. Вы можете полагаться на значения сообщества для принятия необходимых решений по маршрутизации, чтобы обеспечить оптимальную маршрутизацию. Вы можете использовать значение сообщества BGP в Skype для бизнеса Online для настройки подключения ExpressRoute только в Skype для бизнеса Online. Вы можете найти дополнительные сведения о [требованиях к маршрутизации в ExpressRoute](https://azure.microsoft.com/documentation/articles/expressroute-routing/).

Работа с офисными организациями осуществляется через глобальную сеть, которая использует MPLS для подключения к Office 365. Изначально они настроили подключение ExpressRoute из Интернет-маршрутизатора в Hoboken, новый Джерси и MeetMe. 
  
Благодаря этой настройке сетевой трафик от большинства своих сайтов до сети Microsoft (на Нью-Йорке) можно использовать для подключения клиентов Skype для бизнеса, описанных в разделе [требования к производительности сети от клиента Skype для бизнеса и до Microsoft Network Edge](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge). Однако задержка между равнозначными офисами Contoso в Нью-Йорке — это 50ms одностороннее. Кроме того, Гонолулу является вторым крупнейшим Office для Contoso, а задержка из Гонолулу в Нью-Йорк выходит за пределы 80ms одностороннего. Чтобы обеспечить высокое качество мультимедиа для пользователей в этих офисах, компания Contoso решила добавить соединение на основе ExpressRoute для Западного побережье с сайта San для Сан-Хосе и MeetMe на сайт Silicon впадин ExpressRoute.
  
![Один и тот же континент на Экспресс-сайте маршрутизатора.](../images/bf57a473-01e1-4271-9675-385767bc58e1.png)
  
### <a name="online-only-deployment---multiple-sites-on-different-continents"></a>Развертывание только в Интернете-несколько сайтов на разных континентах

Если все пользователи работают со службой Skype для бизнеса Online, и если вы хотите развернуть Azure ExpressRoute, вы должны настроить по крайней мере одно подключение ExpressRoute для каждого из них между главным сайтом в Expressroute и ближе к ближайшему [расположению для пиринга](https://azure.microsoft.com/documentation/articles/expressroute-locations/)3G. В зависимости от стоимости и льготы вы можете выбрать развертывание дополнительных подключений ExpressRoute с сайтов, на которых не выполняются целевые показатели производительности сети.
  
В приведенном ниже примере компания Contoso — это крупная корпоративная фирма компании с офисами в крупных странах Северной Америки и Европы. В зависимости от того, какое подключение к Интернету и внутренняя оценка производительности сети компания Contoso решила развернуть два соединения ExpressRoute в Северной Америке и одну цепь ExpressRoute для всех европейских офисов.

### <a name="hybrid-deployment"></a>Гибридное развертывание

Если у вас есть локальное развертывание Lync или Skype для бизнеса и вы решили внедрить гибридную интеграцию Skype для бизнеса Online, рекомендуется, чтобы при развертывании Azure ExpressRoute вы могли использовать хотя бы одно подключение ExpressRoute для каждого локального сайта Lync или Skype для бизнеса Edge и по крайней мере одно подключение ExpressRoute для каждого континента с офисами. В зависимости от цены и льготы для каждого континента вы можете развернуть дополнительные подключения ExpressRoute из офисов, в которых не соблюдаются целевые показатели производительности сети.
  
Если вы планируете использовать локальное развертывание Skype для бизнеса, необходимо выполнить руководство по [планированию и развертыванию пограничного сервера](https://technet.microsoft.com/library/mt346417.aspx). В частности, пограничные серверы должны быть доступны за пределами вашей сети. Обычно это можно достичь, назначив маршрутизируемый общедоступный IP-адрес пограничного сервера или с помощью преобразования сетевых адресов (NAT).
  
В приведенном ниже примере компания Contoso имеет существующее локальное развертывание Skype для бизнеса Enterprise. Они хотят перенести локальных пользователей в службы Office 365 Online Services. Кроме того, они решили использовать гибридное развертывание, чтобы они могли продолжать использовать существующую инфраструктуру PSTN для всех локальных и сетевых пользователей. Локальный центр обработки данных Contoso и пограничные серверы Skype для бизнеса в Чикаго. Для развертывания компания Contoso решила настроить одно подключение ExpressRoute между центром данных в Чикаго и Чикаго ExpressRoute. Кроме того, они добавляют подключение к западу ExpressRoute для более удобного обслуживания своих Гонолулуных офисов.
  
![Гибридная среда ExpressRoute.](../images/a7467c56-642f-44e5-adfb-ecca91ba2dd3.png)
  
### <a name="online-deployment-with-cloud-connector-edition"></a>Развертывание через Интернет с помощью Cloud Connector Edition

Skype для бизнеса Online Connector Edition — это гибридное предложение, состоящее из набора виртуальных машин (ВМ), которые реализуют локальную сеть PSTN. Развернув в виртуализованной среде топологию сервера Skype для бизнеса, вы сможете отправлять и принимать звонки с помощью стационарные телефоны и мобильных телефонов с помощью существующей локальной инфраструктуры PSTN голосовой связи.
  
Если вы решили развернуть Azure ExpressRoute и Cloud Connector Edition, мы рекомендуем вам настроить по крайней мере одно подключение к каждому из двух маршрутов между главным узлом континента в ближайшем [расположении для пиринга ExpressRoute](https://azure.microsoft.com/documentation/articles/expressroute-locations/). В зависимости от цены и льготы для каждого континента вы можете развернуть дополнительные подключения ExpressRoute с сайтов, для которых не удовлетворены целевые показатели производительности сети.
  
Если у вас есть локальное развертывание Skype для бизнеса, вы должны выполнить руководство по [планированию для Skype для бизнеса Cloud Connector Edition](https://technet.microsoft.com/library/mt605227.aspx). В частности, EDGE Access и службы Edge/V должны быть назначены общедоступные IP-адреса и доступны в центрах обработки данных Office 365.
  
В приведенном ниже примере компания Contoso — Европейская бухгалтерская фирма с присутствием в нескольких крупных европейских странах и городах. При регистрации в Skype для бизнеса Online в целях совместной работы они решили добавить облачный соединитель для каждой страны, в которой у них есть физическое расположение, чтобы продолжать использовать свою инфраструктуру PSTN и контракты перевозчиков, которые уже существуют. В соответствии с их тестированием на всех сайтах и в Microsoft Network EDGE, вы определили, что одно подключение ExpressRoute в Лондоне поможет соблюсти целевые показатели производительности сети для подключения клиента Skype для бизнеса, описанные в разделе [требования к производительности сети от клиента Skype для бизнеса до Microsoft Network Edge](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge).
