# <a name="media-quality-and-network-connectivity-performance-in-skype-for-business-online"></a>Качество медиаданных и производительность сетевого подключения в Skype для бизнеса Online

В этом разделе определен набор требований к производительности сети для служб Skype для бизнеса Online, а также способ использования Интернета или ExpressRoute для подключения между сетью и Skype для бизнеса Online в зависимости от вашей оценки сетевого подключения. Если вы решили развернуть Azure ExpressRoute для выделенного подключения к Microsoft 365 или Office 365, в этом документе также вы сможете найти рекомендации по планированию подключений ExpressRoute в различных сценариях развертывания Skype для бизнеса Online.
  
На качество Real-Time (аудио, видео и общий доступ к приложениям) по IP-адресу в значительной части оказывает влияние качество подключения между конечными сетями. Для достижения оптимального качества передачи данных при использовании Skype для бизнеса Online очень важно обеспечить максимально эффективное подключение между сетью организации и Skype для бизнеса Online. Наилучшим способом является настройка внутреннего сетевого и облачного подключения на основе производительности сети с целью поддержки Skype для бизнеса Online в периоды наибольшей нагрузки.
  
Azure ExpressRoute не является требованием для служб Microsoft 365 и Office 365, включая Skype для бизнеса Online. Тем не менее, Azure ExpressRoute — это один из доступных вариантов развертывания, который гарантирует, что подключение к Microsoft 365 или Office 365 соответствует требованиям к производительности сети Skype для бизнеса и обеспечивает оптимальное качество мультимедиа в Skype для бизнеса Online.
  
> [!TIP]
> Хотя в этом разделе приводится общее руководство по производительности сети, полные инструкции по оценке сети не предусмотрены в этом документе. Список партнеров Skype для бизнеса Online, которые могут помочь вам в оценке производительности сети в рамках тщательной и полной оценки сети, можно найти на веб-сайте решений партнеров Skype для [бизнеса.](http://partnersolutions.skypeforbusiness.com/) 
  
## <a name="network-connectivity-requirements-to-skype-for-business-online"></a>Требования к сетевому подключению к Skype для бизнеса Online

### <a name="factors-that-impact-skype-for-business-online-media-quality"></a>Факторы, влияющие на качество мультимедиа в Skype для бизнеса Online

Существует множество различных факторов, влияющих на качество Real-Time Skype для бизнеса Online (аудио, видео и общий доступ к приложениям), включая используемые устройства, среду и сетевое подключение. 
  
#### <a name="devices"></a>Устройства

Во время Real-Time мультимедиа огромное воздействие на общее качество звука и видео оказывают устройства захвата и отображения мультимедиа, используемые всеми участниками, такие как гарнитуры и веб-камеры. Устройства более низкого качества или устройства с неправильным драйверами будут выдавать и принимать менее качественный звук и видео. С другой стороны, сертифицированные или высококачественные устройства позволяют подавлять эхо, фильтровать шумы, поддерживать оптимальное разрешение видеоизображения и уменьшать задержки.
  
Хотя сертифицированные устройства аудио- и видеосвязи не требуются, настоятельно рекомендуется использовать устройства, сертифицированные для Skype для бизнеса, для обеспечения оптимальной работы с мультимедиа. Список всех устройств, сертифицированных для Skype для бизнеса, см. в списке "Телефоны и устройства [для Skype для бизнеса".](https://technet.microsoft.com/office/dn947482) С помощью панели мониторинга качества звонков Skype для бизнеса [Online,](/microsoftteams/turning-on-and-using-call-quality-dashboard)которая находится в Центре администрирования **Skype** для бизнеса, можно проверить, правильно ли работают работающие устройства и отслеживать качество звука и видео.
  
> [!TIP]
> **Для обеспечения оптимального качества** мультимедиа в Skype для бизнеса требуется сертифицированное устройство.
  
Важно помнить, что все устройства мультимедиа, клиенты Skype для бизнеса и серверы Skype для бизнеса, через которые Real-Time потоки мультимедиа, являются частью задержки. Устройства и программное обеспечение, обрабатывающие задержки, а также сетевые задержки, оказывают значительное влияние на общую задержку и производительность конечных пользователей.
  
#### <a name="environment"></a>Среда

Другим важным фактором, от которого зависит качество звука и видео, является среда и окружающее пространство, где пользователи проводят свои встречи и используют аудио- и видеоустройства. Звук, принимаемый во время звонка от пользователей из шумной обстановки, будет приглушенным, нечетким или будет отдаваться эхом. А пользователи, находящиеся в условиях низкой освещенности, не смогут передавать яркое и четкое видеоизображение. При подготовке конференц-зала непосредственное влияние на качество получаемого звука и изображения оказывает расположение микрофона и видеоустройства.
  
Чтобы получить более четкое представление о звуке и видео пользователя, используйте инструменты приложения Skype для бизнеса "Параметры звукового устройства" или "Видео устройство" для внесения изменений в используемом устройстве и настройки его   >    >   параметров. 

#### <a name="network"></a>Сеть

На качество Real-Time мультимедиа по IP-сети в значительной части оказывает влияние качество сетевого подключения, но в особенности количество:
  
- **Задержка** Это время, необходимое для получения IP-пакета из точки A в точку B сети. Эта задержка распространения сети, по существу, связана с физическим расстоянием между двумя точками и скоростью света, включая дополнительные накладные расходы, которые связаны с различными маршрутизаторами между ними. Задержка измеряется как однонамерная или времени кругового пути (RTT).
    
- **Packet Loss (Потеря пакетов)** Это часто определяется как процент пакетов, потерянных за определенный период времени. Потеря пакетов напрямую влияет на качество звука — небольшие потерянные пакеты практически не оказывают влияния, а потери пакетов в очереди приводят к полному из-за разрыва звука.
    
- **Дрожать между прибытием пакетов или просто дрожать** Это среднее значение изменения задержки между последовательными пакетами. Большинство современных программ VoIP, включая Skype для бизнеса, могут адаптироваться к некоторым уровням дрожания с помощью буферизации. Участник заметит эффект дрожания только тогда, когда уровень дрожания превышает буферику.
    
> [!NOTE]
>  Буфериз при дрожание увеличивает задержку между конечными.
  
Благодаря одновременному сеансу мультимедиа в Skype для бизнеса Online Real-Time, а также другому сетевому трафику, генерируемму другими службами Microsoft 365 или Office 365 и другими бизнес-приложениями, убедитесь, что для подключения сети к службе Skype для бизнеса Online достаточно пропускной способности, чтобы избежать перегрузки сети и обеспечить отличное качество мультимедиа Real-Time (аудио- и видеосвязь и общий доступ к приложениям). 
  
#### <a name="implementing-quality-of-service-qos-across-congested-networks"></a>Реализация качества обслуживания (QoS) в сети с высоким качеством обслуживания

Кроме того, качество мультимедиа значительно влияет на перегрузку сети трафиком. Чтобы обеспечить более быструю передачу аудио- и видео пакетов по сети и расставить приоритеты над другим сетевым трафиком в сети в смежной сети, можно использовать QoS для обеспечения оптимального взаимодействия пользователей с аудио- и видеосознативами.
  
QoS позволяет назначать более высокий приоритет сетевым пакетам с аудио- и видеосъемами. При назначении этим пакетам более высокого приоритета аудио- и видеосвязь, скорее всего, будет работать по сети быстрее и с меньшими прерываниями, чем сетевые сеансы, связанные с передачей файлов, просмотром веб-страниц или резервной копией базы данных. Это потому, что сетевым пакетам, используемым для передач файлов или резервного копирования баз данных, по умолчанию назначены приоритет "наилучшие усилия", а перегрузка сети не будет иметь большого влияния. Если пакеты мультимедиа (аудио, видео и общий доступ к приложениям) не назначены более высоким приоритетам, а они также назначены как наилучшие, они обрабатываются вместе со всем другим сетевым трафиком. В зависимости от объема сетевой нагрузки это может понизить общее качество звука и видео для пользователей.
  
Настоятельно рекомендуется внедрить QoS в сети, чтобы не повлиять на загруженность сети. Но для максимального влияния этого все сетевые конечные точки должны поддерживать QoS, то есть все конечные точки должны поддерживать маркировку QoS и рас приоритет пакетов. Службы Skype для бизнеса Online с учетом маркировки qoS и расценки приоритетов в сети Майкрософт. Однако в трафике, маршрутном через открытое подключение, например в Интернет из сети организации к сети Майкрософт, не сохраняются маркировки QoS и рас приоритеты пакетов. Частные подключения от сети к Microsoft 365 или Office 365 с помощью [Azure ExpressRoute](https://azure.microsoft.com/services/expressroute/) предлагают решение для развертывания, которое сохраняет маркировки QoS и рас приоритеты пакетов, что, в свою очередь, увеличит общее качество звука и видео для конечных пользователей.
  
## <a name="network-performance-requirements-to-connect-to-skype-for-business-online"></a>Требования к производительности сети для подключения к Skype для бизнеса Online
<a name="bkNetworkPerf"> </a>

Skype для Real-Time мультимедиа проходит через множество разных устройств, клиентских приложений, серверного программного обеспечения и разных сетей. End-to-end latency of Real-Time media is the total of latency that is introduced across all components and network segments. Качество конечного сетевого подключения определяется сегментом сети с наихудшим качеством. Этот сегмент является узким местом для сетевого трафика.
  
На следующей схеме показано, как проходит аудиопоток в одну сторону во время конференции от одного участника Skype для бизнеса к другому.
  
![Поток вызовов ExpressRoute.](../images/c026e8e5-ba09-42c0-9e03-60fbfda1cb02.png)
  
В этом сценарии медиаконференция путь к мультимедиа состоит из следующих сетевых сегментов:
  
1. **Подключение от пользователя 1 к границе сети Майкрософт** Обычно оно включает сетевое подключение, такое как Wi-Fi или Ethernet, подключение по сети WAN от пользователя 1 к точке выход в Интернет (вашему границе сети) и подключение к Интернету от границы сети вашей сети до границы сети Майкрософт.
    
2. **Подключение в сети Майкрософт** Находится между границей сети Майкрософт и ЦЦ Skype для бизнеса Online, где используются серверы A/V conferencing.
    
3. **Подключение в рамках сети Майкрософт** Находится между центром обработки данных Skype для бизнеса Online и границей сети Майкрософт.
    
4. **Подключение от границы сети Майкрософт к пользователю 2** К ним относятся подключение к Интернету от границы сети вашей сети к границе сети Майкрософт, подключение по сети WAN от пользователя 2 к точке выходов в Интернет (границе сети сети) и сетевое подключение, такое как Wi-Fi или Ethernet.
    
На следующей схеме показана разбивка компонентов и сегментов сети при звонке по ПС Skype для бизнеса Online:
  
![Поток зовов оператора ННР ExpressRoute.](../images/768a88df-c8a9-4171-a158-565a698f0193.png)
  
При звонке по ДНР медиана пути пересекают следующие сетевые сегменты:
  
1. **Подключение от вызываемого клиента Skype** для бизнеса к границе сети Майкрософт Обычно оно включает сетевое подключение, такое как Wi-Fi или Ethernet, подключение по сети WAN от звоняшего клиента Skype для бизнеса к точке выход в Интернет (вашему границе сети) и подключение к Интернету от границы сети вашей сети к границе сети Майкрософт.
    
2. **Подключение в сети Майкрософт** Находится между границей сети Майкрософт и центром обработки данных Skype для бизнеса Online, где используется сервер-посредник.
    
3. **Подключение в рамках сети Майкрософт** Находится между центром обработки данных Skype для бизнеса Online и границей сети Майкрософт.
    
4. **Подключение между сетью Майкрософт и партнерами- поставщиками услуг STN** Это подключение для звонков по ПС из клиента Skype для бизнеса, который находится за пределами сети Майкрософт.
    
### <a name="network-performance-requirements-from-a-skype-for-business-client-to-microsoft-network-edge"></a>Требования к производительности сети от клиента Skype для бизнеса до границы сети Майкрософт

Этот сегмент сети включает внутреннюю сеть организации, включая все подключения Wi-Fi и Ethernet, трафик организации между сайтами и сайтами по подключению по сети WAN, например MPLS, а также партнерские подключения через Интернет или ExpressRoute к границе сети Майкрософт.
  
> [!CAUTION]
> **Подключение между клиентом Skype для бизнеса в сети организации и службами Microsoft 365 или Office 365 должно соответствовать следующим требованиям и порогам производительности сети.**

 **Другие требования к целевым показателям производительности:**
  
- Сеть Майкрософт имеет более 160 edge-местоположений по всему миру. Мы работаем с основными поставщиками услуг Интернета (ISP) по всему миру через эти граниковые сайты. При целевом показателе задержки предполагается, что узел или узлы организации и границы сети Майкрософт находятся на одном континенте.
    
- К сайту организации или сайтам к границе сети Майкрософт относится сетевой доступ первого перехода, который может использовать Wi-Fi или другую беспроводную технологию. 
    
- При целевом показателе производительности сети предполагается надлежащее планирование пропускной способности и (или) качества обслуживания. Другими словами, это относится непосредственно к Skype для Real-Time трафику мультимедиа, когда сетевое подключение находится под пиковой нагрузкой.

## <a name="media-quality-and-expressroute"></a>Качество мультимедиа и ExpressRoute
<a name="bkNetworkPerf"> </a>

Azure ExpressRoute для Microsoft 365 или Office 365 — это выделенное сетевое подключение для подключения к Microsoft 365 или Office 365. Она позволяет клиентам управлять маршрутом сетевого трафика. Им больше не нужно беспокоиться о непредсказуемой маршрутике в Интернете, где данные переносят неизвестные операторы, поставщики и поставщики услуг Интернета. Сетевой трафик, от который отправляется через ExpressRoute, напрямую через сеть партнера ExpressRoute в сеть Майкрософт. Это позволяет клиентам обрабатывать Microsoft 365 или Office 365 так, как если бы они находились в их собственных центрах обработки данных вне сайта со выделенным подключением.
  
Azure ExpressRoute доступен для всех предложений лицензирования Microsoft 365 и Office 365. Однако для глобальной маршрутки требуется надстройка Azure ExpressRoute premium для Microsoft 365 и Office 365. Клиенты с минимум 500 пользователями, внедряющие ExpressRoute, могут получить необходимую надстройку *ExpressRoute Premium* без дополнительных затрат.
  
### <a name="is-expressroute-required-for-good-media-quality"></a>Требуется ли ExpressRoute для высокого качества мультимедиа?

Azure ExpressRoute не является требованием для получения наиболее оптимального качества мультимедиа в Skype для бизнеса Online. Однако это один из вариантов развертывания, который помогает убедиться, что облачное подключение соответствует целевым показателям или порогам производительности сети Skype для бизнеса.
  
Microsoft 365 и Office 365 — это высокопроивратные и безопасные службы, которые используют Интернет. Мы продолжаем улучшать новые возможности безопасности и региональные узлы на границе, чтобы постоянно повышать безопасность и производительность. Azure ExpressRoute не является требованием для служб Microsoft 365 или Office 365, включая Skype для бизнеса Online. Azure ExpressRoute — это один из доступных вариантов развертывания, который гарантирует, что подключение к Microsoft 365 или Office 365 соответствует требованиям к производительности сети Skype для бизнеса и обеспечивает оптимальное качество мультимедиа в Skype для бизнеса Online.
  
Для качества мультимедиа в Skype для бизнеса Online важно, чтобы подключение между сайтами организации и границей сети Майкрософт соответствовало целевым показателям производительности в требованиях к производительности сети от клиента [Skype](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge) для бизнеса к границе сети Майкрософт и что подключение между границей сети организации и границей сети Майкрософт соответствует целевым показателям производительности в требованиях к производительности сети от границы сети организации до границы сети [Майкрософт.](media-quality-and-network-connectivity-performance.md#bkYourNetworkEdge)  
  
Кроме того, важно, чтобы физическое сетевое подключение организации, включая внутреннее сетевое и облачное подключение, было подключено к пиковым объемам мультимедиа. Azure ExpressRoute — это один из многих способов, который гарантирует, что облачное подключение клиентов Skype для бизнеса Online соответствует всем этим требованиям к производительности.
  
### <a name="is-expressroute-required-for-voice-quality-sla"></a>Требуется ли ExpressRoute для SLA о качестве голосовой почты?

Нет, ExpressRoute не требуется для SLA о качестве голосовой почты Skype для бизнеса Online. SLA о качестве голосовой почты Skype для бизнеса Online применяется ко любым подходящим звонкам, размещенным любым пользователем голосовой службы [Skype](http://www.microsoftvolumelicensing.com/DocumentSearch.aspx?Mode=3&amp;DocumentTypeId=37) для бизнеса Online в рамках нужной лицензии и подписки, которая позволяет этому пользователю делать любые звонки по VoIP или ПСN. SLA о качестве голосовой почты должно включать в себя все следующие условия:
  
- Звонки с сертифицированных IP-телефонов Майкрософт.
    
- Проводные ethernet-подключения.
    
- Проблемы с качеством голосовой связи из-за проблем с сетью Майкрософт.
    
> [!NOTE]
> SLA о качестве голосовой связи исключает звонки, где низкое качество звонков вызвано проблемами в сетях, не связанных с Майкрософт, включая партнеров ExpressRoute и другие сети. 
  
### <a name="internet-or-azure-expressroute"></a>Интернет или Azure ExpressRoute?

Прежде чем принимать решение о вариантах сетевого подключения к Skype для бизнеса Online, клиенты должны оценить свое сетевое и текущее подключение к Интернету на основе требований к производительности сети, описанных в требованиях к производительности сети для подключения к Skype для [бизнеса Online.](media-quality-and-network-connectivity-performance.md#bkNetworkPerf)
  
Если производительность сети по текущему подключению к Интернету настроена для достаточной производительности в пиковые часы и соответствует требованиям к производительности сети между сайтами и границей сети Майкрософт и границей сети Майкрософт, вы можете продолжать использовать существующее подключение к Интернету для подключения к Skype для бизнеса Online.
  
Для сайтов компании, на которых не выполнены требования к производительности сети, настоятельно рекомендуем сначала поработать с существующими поставщиками сетевых услуг, чтобы улучшить общую производительность сети. Однако если им по-прежнему не соответствуют, используя Azure ExpressRoute, вы можете обеспечить облачное подключение к Skype для бизнеса Online и обеспечить выполнение требований к производительности сети.
  
Azure ExpressRoute предоставляет следующие дополнительные преимущества:
  
- Соглашение SLA о доступности подключения между вашей сетью и сетью Майкрософт. Для ExpressRoute есть SLA о гарантированной доступности, которая составляет 99,9 %.
    
- Запланированная и гарантированная пропускная способность, необходимая для служб Microsoft 365 и Office 365. Это можно сделать, отправив только трафик Microsoft 365, Office 365 или Skype для бизнеса с помощью ExpressRoute, а затем направить весь весь интернет-трафик через другие точки (точки доступа к Интернету) для вашей сети.
    
- ExpressRoute предназначен для сохранения маркировки QoS DSCP между вашей сетью и сетью Майкрософт.
    
Дополнительные сведения о QoS и планировании емкости ExpressRoute можно найти в [ExpressRoute и QoS в Skype для бизнеса Online.](https://support.office.com/article/ExpressRoute-and-QoS-in-Skype-for-Business-Online-20c654da-30ee-4e4f-a764-8b7d8844431d)
  
### <a name="can-i-setup-azure-expressroute-for-skype-for-business-online-only"></a>Можно ли настроить Azure ExpressRoute только для Skype для бизнеса Online?

Да, вы можете настроить Azure ExpressRoute, чтобы обеспечить отличное сетевое подключение от сети организации только к Skype для бизнеса Online. Это обеспечит наиболее оптимальное Real-Time мультимедиа для пользователей, но вы сможете по-прежнему подключаться к другим службам Microsoft 365 или Office 365 через Интернет.
  
Протокол Border Gateway Protocol (BGP) — это протокол маршрутинга в Интернете, который используется для направления сетевого трафика через Интернет. Он предназначен для обмена данными маршрутов между автономными системами (AS) в Интернете. Значения сообществ BGP — это теги атрибутов, которые можно применять к входящих и исходяющим маршрутам. Сообщества BGP часто используются для того, чтобы укажете принимающим AS, какую исходящие ссылки использовать для достижения данного места назначения на основе географического местоположения, типа службы или других критериев.
  
При поддержке сообществ BGP Майкрософт помечает префиксы и маршруты соответствующими значениями сообщества BGP на основе службы, в службе, в которая они входят. Майкрософт помечает префиксы, объявленные через общедоступный пиринг и пиринг Майкрософт, соответствующими значениями сообщества BGP, указывающими регион, в котором размещаются префиксы. Вы можете использовать значения сообщества для принятия решений об оптимальной маршрутике. Значение сообщества BGP в Skype для бизнеса можно использовать для настройки подключения ExpressRoute только для Skype для бизнеса Online. Дополнительные узнать о требованиях [ExpressRoute к маршрутингу](https://azure.microsoft.com/documentation/articles/expressroute-routing/)можно здесь.

Офисы компании подключены по сети WAN, использующей MPLS для подключения к Microsoft 365 или Office 365. Изначально компания настроил подключение ExpressRoute от интернет-маршрутизатора в Хобокене (Нью-Джерси) к сайту MeetMe в Нью-Йорке. 
  
В этом случае сетевой трафик с большинства сайтов компании до границы сети Майкрософт (границе сети в Нью-Йорке) может соответствовать целевым показателям производительности сети клиентского подключения Skype для бизнеса, описанным в требованиях к производительности сети от клиента [Skype](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge)для бизнеса до границы сети Майкрософт. Однако задержка между офисами Contoso на западном побережье и Нью-Йорком составляет более 50 мс в одну сторону. Кроме того, Гонолулу — это второй по величине офис Contoso, задержка из Гонолулу в Нью-Йорк превышает 80 мс в одну сторону. Чтобы обеспечить высокое качество мультимедиа для пользователей этих офисов, компания Contoso решила добавить подключение ExpressRoute западного побережье между своим сайтом в Сан-Франциско и сайтом MeetMe ExpressRoute в Кремниевой долине.
  
![Несколько сайтов Express Router на одном континенте.](../images/bf57a473-01e1-4271-9675-385767bc58e1.png)
  
### <a name="online-only-deployment---multiple-sites-on-different-continents"></a>Только развертывание Online — несколько сайтов на разных континентах

Если все ваши пользователи используют службу Skype для бизнеса Online и ваши офисы находятся в разных физических расположениях на нескольких континентах, при развертывании Azure ExpressRoute необходимо настроить по крайней мере одно подключение ExpressRoute на каждом континенте между основным сайтом каждого континента и ближайшим расположением пиринга [ExpressRoute.](https://azure.microsoft.com/documentation/articles/expressroute-locations/) В зависимости от затрат и преимуществ вы можете развернуть дополнительные подключения ExpressRoute с сайтов, где целевые показатели производительности не выполнены.
  
В следующем примере Contoso — это большая юридическая фирма с офисами в крупных городах Северной Америки. В зависимости от подключения к Интернету и оценки производительности внутренней сети компания Contoso решила развернуть два подключения ExpressRoute в Северной Америке и один канал ExpressRoute для всех своих европейских офисов.
  
![ExpressRoute с несколькими сайтами и континентами.](../images/c967550b-dc85-4b37-a7bc-cd825ec86854.png)
  
### <a name="hybrid-deployment"></a>Гибридное развертывание

Если у вас развернуто локальное развертывание Lync или Skype для бизнеса, и вы решили внедрить гибридную интеграцию Skype для бизнеса Online, мы рекомендуем вам, чтобы при развертывании Azure ExpressRoute было необходимо по крайней мере одно подключение ExpressRoute для каждого локального узла Lync или границы Skype для бизнеса, а также хотя бы одно подключение ExpressRoute на каждом континенте с офисами. В зависимости от затрат и преимуществ на каждом континенте можно развернуть дополнительные подключения ExpressRoute из офисов, в которых целевые показатели производительности не выполнены.
  
Если у вас есть локальное развертывание Skype для бизнеса, вам необходимо следовать руководству по планированию и развертыванию [edge Server.](https://technet.microsoft.com/library/mt346417.aspx) В частности, edge servers must be reachable from outside your network. Обычно для этого необходимо назначить маршрутивируемый общедоступный IP-адрес границей сервера или использовать перевод сетевых адресов (NAT).
  
В следующем примере в компании Contoso уже есть локальное развертывание skype для Корпоративная голосовая связь бизнеса. Они хотят перенести пользователей локальной службы в веб-службы Microsoft 365 или Office 365. Кроме того, было принято решение о гибридном развертывании, чтобы продолжить использовать существующую инфраструктуру STN для всех пользователей локальной и сетевой сети. Локальное центр обработки данных Contoso и edge-серверы Skype для бизнеса находятся в Чикаго. Для развертывания компания Contoso решила настроить одно подключение ExpressRoute между центрами обработки данных в Чикаго и Чикаго ExpressRoute. Они также добавили подключение ExpressRoute на западном побережье, чтобы лучше обслуживать офис в Гонолулу.
  
![Гибридная система ExpressRoute.](../images/a7467c56-642f-44e5-adfb-ecca91ba2dd3.png)
  
### <a name="online-deployment-with-cloud-connector-edition"></a>Развертывание Online с Cloud Connector Edition

Skype для бизнеса Online Cloud Connector Edition — это гибридное предложение, которое состоит из набора упакованных виртуальных машин (ВМ), реализующих локальное подключение через ДНР. Если в виртуализированной среде развернуть минимальную топологию Skype для бизнеса Server, вы сможете отправлять и принимать звонки на стационарные и мобильные телефоны с помощью существующей локальной инфраструктуры голосовой связи ПС.
  
Если вы решите развернуть Azure ExpressRoute и Cloud Connector Edition, рекомендуем настроить по крайней мере одно подключение Express Route для каждого континента между основным сайтом на каждом континенте с ближайшим расположением пиринга [ExpressRoute.](https://azure.microsoft.com/documentation/articles/expressroute-locations/) В зависимости от затрат и преимуществ на каждом континенте можно развернуть дополнительные подключения ExpressRoute с сайтов, где целевые показатели производительности не выполнены.
  
Если у вас есть локальное развертывание Skype для бизнеса, вам необходимо следовать руководству по планированию Skype для бизнеса [Cloud Connector Edition.](https://technet.microsoft.com/library/mt605227.aspx) В частности, службам Access Edge и A/V Edge должны быть назначены общедоступные IP-адреса и доступны центры обработки данных Microsoft 365 или Office 365.
  
В следующем примере Contoso — это европейской бухгалтерской фирмой с присутствием в нескольких крупных европейских странах и городах. О регистрации в Skype для бизнеса Online для совместной работы было принято решение создать Cloud Connector для каждой страны и региона, где они будут продолжать использовать свою инфраструктуру ЗВОНКОВ и уже договоры с оператором. На основе тестирования на всех сайтах компании и границе сети Майкрософт они определили, что одно подключение ExpressRoute в Лондоне поможет установить целевые показатели производительности сети клиентского подключения Skype для бизнеса, описанные в требованиях к производительности сети от клиента Skype для бизнеса до границы сети [Майкрософт.](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge)
