# <a name="media-quality-and-network-connectivity-performance-in-skype-for-business-online"></a>Качество медиаданных и производительность сетевого подключения в Skype для бизнеса Online

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

В этом разделе определен набор требований к производительности сети для веб-служб Skype для бизнеса Online и способ использования Интернета или ExpressRoute для подключения между сетью и Skype для бизнеса Online на основе вашей оценки сетевого подключения. Если вы решили развернуть Azure ExpressRoute для выделенного подключения к Microsoft 365 или Office 365, этот документ также содержит рекомендации по планированию подключений ExpressRoute в различных сценариях развертывания Skype для бизнеса Online.
  
Качество мультимедиа Real-Time (аудио, видео и общий доступ к приложениям) по IP-адресу в значительной части влияет на качество связи между конечными сетями. Для достижения оптимального качества передачи данных при использовании Skype для бизнеса Online очень важно обеспечить максимально эффективное подключение между сетью организации и Skype для бизнеса Online. Наилучшим способом является настройка внутреннего сетевого и облачного подключения на основе производительности сети с целью поддержки Skype для бизнеса Online в периоды наибольшей нагрузки.
  
Azure ExpressRoute не является требованием для служб Microsoft 365 и Office 365, включая Skype для бизнеса Online. Однако Azure ExpressRoute — это один из доступных вариантов развертывания, который гарантирует, что подключение к Microsoft 365 или Office 365 соответствует требованиям к производительности сети Skype для бизнеса и обеспечивает наиболее оптимальное качество мультимедиа Skype для бизнеса Online.

## <a name="network-connectivity-requirements-to-skype-for-business-online"></a>Требования к сетевому подключению для Skype для бизнеса Online

### <a name="factors-that-impact-skype-for-business-online-media-quality"></a>Факторы, влияющие на Skype для бизнеса качества мультимедиа в Интернете

Существует множество различных факторов, влияющих на качество Skype для бизнеса Online Real-Time мультимедиа (аудио, видео и общий доступ к приложениям), включая используемые устройства, среду и сетевое подключение. 
  
#### <a name="devices"></a>Устройства

Во время Real-Time мультимедиа огромное влияние на общее качество звука и видео оказывают устройства захвата и отрисовки мультимедиа, используемые всеми участниками, например гарнитуры и веб-камеры. Устройства более низкого качества или устройства с неправильным драйверами будут выдавать и принимать менее качественный звук и видео. С другой стороны, сертифицированные или высококачественные устройства позволяют подавлять эхо, фильтровать шумы, поддерживать оптимальное разрешение видеоизображения и уменьшать задержки.
  
Хотя сертифицированные звуковые и видеофайл-устройства не требуются, настоятельно рекомендуется использовать устройства, сертифицированные для Skype для бизнеса для оптимальной работы с мультимедиа. Список всех устройств, сертифицированных Skype для бизнеса, см. в этой [Skype для бизнеса.](../../SfbPartnerCertification/certification/devices-ip-phones.md) Вы можете использовать панель [Skype для бизнеса Качества](/microsoftteams/turning-on-and-using-call-quality-dashboard)звонка в Интернете , которая находится в Центре администрирования **Skype для бизнеса**, чтобы убедиться, что устройства работают правильно и отслеживают качество звука и видео.
  
> [!TIP]
> **Для наиболее оптимального качества** Skype для бизнеса требуется сертифицированное устройство.
  
Важно помнить, что любые устройства мультимедиа, Skype для бизнеса клиенты и Skype для бизнеса Серверы, через которые Real-Time потоков мультимедиа, являются частью задержки. Устройства и программное обеспечение, обрабатывающие задержки, а также сетевые задержки, оказывают значительное влияние на общую задержку и производительность конечных пользователей.
  
#### <a name="environment"></a>Среда

Другим важным фактором, от которого зависит качество звука и видео, является среда и окружающее пространство, где пользователи проводят свои встречи и используют аудио- и видеоустройства. Звук, принимаемый во время звонка от пользователей из шумной обстановки, будет приглушенным, нечетким или будет отдаваться эхом. А пользователи, находящиеся в условиях низкой освещенности, не смогут передавать яркое и четкое видеоизображение. При подготовке конференц-зала непосредственное влияние на качество получаемого звука и изображения оказывает расположение микрофона и видеоустройства.
  
Чтобы получить более четкое представление о звуке и видео для пользователя, внести изменения в используемом устройстве и настроить его параметры можно с помощью Skype для бизнеса ИнструментынастройкиНастройки Звуковое устройство или Видео  >    >   устройство. 

#### <a name="network"></a>Сеть

Качество мультимедиа Real-Time по IP-сети значительно влияет на качество сетевого подключения, но в особенности на объем:
  
- **Задержка** Это время, необходимое для получения IP-пакета из точки A в точку B в сети. Эта задержка распространения по сети в основном связана с физическим расстоянием между двумя точками и скоростью света, а также дополнительными затратами времени на различные маршрутизаторы между ними. Задержка измеряется как однонамерная или время кругового пути (RTT).
    
- **Packet Loss (Потеря пакетов)** Это часто определяется как процент пакетов, которые теряются в заданном окне времени. Потеря пакетов напрямую влияет на качество звука — отдельные потерянные пакеты почти не оказывают влияния, а сплошные последовательные потери приводят к полному исчезновению звука.
    
- **Дрожать между пакетами или просто дрожать** Это среднее значение изменения задержки между последовательными пакетами. Большинство современных программ VoIP, включая Skype для бизнеса, могут адаптироваться к некоторым уровням дрожания путем буферизации. Участник заметит эффект дрожания только в том случае, если дрожание превышает буферику.
    
> [!NOTE]
>  Буферы для дрожания увеличивают задержки между концами.
  
При одновременном использовании многих сеансов Skype для бизнеса Online Real-Time мультимедиа, а также другого сетевого трафика, генерируемого другими службами Microsoft 365 или Office 365 и другими бизнес-приложениями, важно обеспечить достаточный объем пропускной способности по всему сетевому пути, который подключает сеть к службе Skype для бизнеса Online, чтобы избежать перегрузки сети и обеспечить отличное качество мультимедиа Real-Time мультимедиа (аудио, видео и общий доступ к приложениям). 
  
#### <a name="implementing-quality-of-service-qos-across-congested-networks"></a>Реализация качества обслуживания (QoS) в сети с загроможаемой сетью

Кроме того, нагрузка на трафик в сети существенно влияет на качество мультимедиа. Чтобы обеспечить более быструю передачу аудио- и видеокаетов по сети и приоритет над другим сетевым трафиком в убываемой сети, можно использовать QoS для обеспечения оптимальной работы пользователей с аудио- и видеосознакомиями.
  
QoS позволяет назначать более высокий приоритет сетевым пакетам, которые имеют звуковые и видеофайлы. При назначении этим пакетам более высокого приоритета аудио- и видеосвязь, скорее всего, будет работать по сети быстрее и с меньшими прерываниями, чем сетевые сеансы, связанные с передачей файлов, просмотром веб-страниц или резервными копиями баз данных. Это потому, что сетевым пакетам, используемым для передачи файлов или резервного копирования баз данных, по умолчанию назначены "наилучшие усилия" в качестве приоритета, а перегрузка сети не будет иметь большого влияния. Если не назначить более высокий приоритет пакетам мультимедиа (аудио, видео и общего доступа к приложениям) и оставить их также на должном уровне, они будут обрабатываться вместе с другим сетевым трафиком. В зависимости от объема сетевой нагрузки это может понизить общее качество звука и видео для пользователей.
  
Настоятельно рекомендуется внедрить QoS в сети, чтобы убедиться, что перегрузка сети в сети не повлияет на сеть. Однако для максимального влияния на это все конечные точки сети должны поддерживать QoS, то есть все конечные точки должны соблюдать маркировку QoS и расценки пакетов. Skype для бизнеса Веб-службы с учетом маркировки QoS и расценки приоритетов в сети Майкрософт. Однако в трафике, маршрутном через такое открытое подключение, как Интернет из сети организации в сеть Майкрософт, не сохраняются маркировка QoS и приоритеты пакетов. Частные подключения из сети к Microsoft 365 или Office 365 с помощью [Azure ExpressRoute](https://azure.microsoft.com/services/expressroute/) предлагают решение для развертывания, которое сохраняет маркировку QoS и расценки пакетов, что, в свою очередь, повышает общее качество звука и видео для конечных пользователей.
  
## <a name="network-performance-requirements-to-connect-to-skype-for-business-online"></a>Требования к производительности сети для подключения к Skype для бизнеса Online
<a name="bkNetworkPerf"> </a>

Skype для бизнеса Real-Time мультимедиа проходят через различные устройства, клиентские приложения, серверное программное обеспечение и разные сети. End-to-end latency of Real-Time media — это общая задержка, которая вводится во всех компонентах и сегментах сети. Качество конечного сетевого подключения определяется сегментом сети с наихудшим качеством. Этот сегмент является узким местом для сетевого трафика.
  
На следующей схеме показано, как один участник конференции может Skype для бизнеса аудиопотоком.
  
![Вызовы expressRoute Flow.](../images/c026e8e5-ba09-42c0-9e03-60fbfda1cb02.png)
  
В этом случае путь к мультимедиа будет включаться в следующие сетевые сегменты:
  
1. **Подключение от пользователя 1 к границе сети Майкрософт** Обычно оно включает сетевое подключение, например Wi-Fi или Ethernet, подключение по сети WAN от пользователя 1 к точке выходов в Интернет (ваше устройство границы сети), а также подключение к Интернету от границы сети вашей сети к границе сети Майкрософт.
    
2. **Подключение в сети Майкрософт** Это находится между Microsoft Edge в Skype для бизнеса Online, где используются серверы A/V Conferencing.
    
3. **Подключение в сети Майкрософт** Находится между центром обработки Skype для бизнеса Online и границей сети Майкрософт.
    
4. **Подключение от границы сети Майкрософт к пользователю 2** К таким данным относятся подключение к Интернету от границы сети вашей сети к границе сети Майкрософт, подключение по сети WAN от пользователя 2 к точке выходов в Интернет (границе сети) и сетевое подключение, например Wi-Fi или Ethernet.
    
На следующей схеме показана разбивка компонентов и сетевых сегментов звонка по Skype для бизнеса ДНР:
  
![Вызов оператора связи через ПСС ExpressRoute Flow.](../images/768a88df-c8a9-4171-a158-565a698f0193.png)
  
При вызове через ПСС путь мультимедиа пересекает следующие сетевые сегменты:
  
1. **Подключение от Skype для бизнеса клиента к границе сети Майкрософт** Обычно оно включает сетевое подключение, например Wi-Fi или Ethernet, подключение по сети WAN от звоняшего клиента Skype для бизнеса к точке выходов в Интернет (сетевое устройство) и подключение к Интернету от границы сети вашей сети к границе сети Майкрософт.
    
2. **Подключение в сети Майкрософт** Это находится между Microsoft Edge в Skype для бизнеса Online, где используется сервер-посредник.
    
3. **Подключение в сети Майкрософт** Находится между центром обработки Skype для бизнеса Online и границей сети Майкрософт.
    
4. **Подключение между сетью Майкрософт и партнерами- поставщиками услуг STN** Это подключение, которое существует для вызова через Skype для бизнеса клиента, который находится за пределами сети Майкрософт.

## <a name="media-quality-and-expressroute"></a>Качество мультимедиа и ExpressRoute
<a name="bkNetworkPerf"> </a>

Azure ExpressRoute для Microsoft 365 или Office 365 — это выделенное сетевое подключение для подключения к Microsoft 365 или Office 365. Она позволяет клиентам управлять маршрутом сетевого трафика. Им больше не нужно беспокоиться о непредсказуемой маршрутике, которая происходит в Интернете, где данные переносят неизвестные операторы, поставщики услуг Интернета и поставщики услуг Интернета. Сетевой трафик, отсланный через ExpressRoute, отправляется непосредственно через сеть партнера ExpressRoute в сеть Майкрософт. Это позволяет клиентам обрабатывать Microsoft 365 или Office 365 как если бы они находились в их собственных центрах обработки данных вне сайта со выделенным подключением.
  
Azure ExpressRoute доступен для всех Microsoft 365 и Office 365 лицензирования. Однако надстройка Azure ExpressRoute Premium для Microsoft 365 и Office 365, чтобы включить глобальную маршрутику. Клиенты с не менее чем 500 мест, внедряющие ExpressRoute, могут получить необходимую надстройку *ExpressRoute Premium* без дополнительных расходов.
  
### <a name="is-expressroute-required-for-good-media-quality"></a>Требуется ли ExpressRoute для высокого качества мультимедиа?

Azure ExpressRoute не является требованием для получения наиболее оптимального качества Skype для бизнеса в Интернете. Однако это один из вариантов развертывания, который помогает убедиться, что облачное подключение соответствует целевым Skype для бизнеса производительности сети.
  
Microsoft 365 и Office 365 высокой производительности и безопасности служб, которые используют Интернет. Мы продолжаем улучшать новые возможности безопасности и региональные узлы edge для непрерывного повышения безопасности и производительности. Azure ExpressRoute не является требованием для Microsoft 365 или Office 365, включая Skype для бизнеса Online. Azure ExpressRoute — один из доступных вариантов развертывания, который гарантирует, что подключение к Microsoft 365 или Office 365 соответствует требованиям к производительности сети Skype для бизнеса и обеспечивает наиболее оптимальное качество мультимедиа Skype для бизнеса Online.
  
Для Skype для бизнеса Качества мультимедиа в Интернете важно, чтобы подключение между сайтами организации и границей сети Майкрософт соответствовало целевым показателям производительности в требованиях к производительности сети от клиента [Skype для бизнеса](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge) до границы сети Майкрософт и что подключение между границей сети организации и границей сети Майкрософт соответствует целевым показателям производительности в требованиях к производительности сети от границы сети организации до границы сети [Майкрософт.](media-quality-and-network-connectivity-performance.md#bkYourNetworkEdge)  
  
Кроме того, важно, чтобы физическое сетевое подключение организации, включая внутреннее сетевое и облачное подключение, было рассчитано на пиковый объем медиапотока. Azure ExpressRoute — это один из многих способов, которые помогут клиентам обеспечить Skype для бизнеса подключение через Интернет соответствует всем этим требованиям к производительности.
  
### <a name="is-expressroute-required-for-voice-quality-sla"></a>Требуется ли ExpressRoute для SLA качества голосовой почты?

Нет, ExpressRoute не требуется для SLA качества Skype для бизнеса Online. SLA [Skype для бизнеса качества](http://www.microsoftvolumelicensing.com/DocumentSearch.aspx?Mode=3&amp;DocumentTypeId=37) голосовой почты Online применяется ко любым подходящим звонкам, размещенным любым пользователем голосовой службы Skype для бизнеса Online в рамках нужной лицензии и подписки, которые позволяют этому пользователю звонить по voIP или ОКП. SLA о качестве голосовой почты должен включать в себя следующие условия:
  
- Звонки с сертифицированных IP-телефонов Майкрософт.
    
- Проводные ethernet-подключения.
    
- Проблемы с качеством голосовой связи из-за проблем с сетью Майкрософт.
    
> [!NOTE]
> SLA о качестве голосовой связи исключает звонки, для которых низкое качество звонков вызвано проблемами в сетях вне Майкрософт, включая партнер ExpressRoute и других сетях. 
  
### <a name="internet-or-azure-expressroute"></a>Интернет или Azure ExpressRoute?

Прежде чем принимать решение о параметрах сетевого подключения для Skype для бизнеса Online, клиенты должны оценить свое сетевое и текущее подключение к Интернету на основе требований к производительности сети, описанных в требованиях к производительности сети для подключения к [Skype для бизнеса Online.](media-quality-and-network-connectivity-performance.md#bkNetworkPerf)
  
Если производительность сети по текущему подключению к Интернету настроена на достаточный период пиковой нагрузки и соответствует требованиям к производительности сети от сайтов до краев сети Майкрософт и от краев сети до краев сети Майкрософт, вы можете продолжать использовать существующее подключение к Интернету для подключения к Skype для бизнеса Online.
  
Для сайтов компании, где требования к производительности сети не выполнены, настоятельно рекомендуем сначала поработать с существующими поставщиками сетевых услуг, чтобы повысить общую производительность сети. Однако если они по-прежнему не выполнены, использование Azure ExpressRoute поможет обеспечить подключение Skype для бизнеса Online может помочь вам в обеспечении производительности сети.
  
Azure ExpressRoute предоставляет следующие дополнительные преимущества:
  
- Соглашение об уровне обслуживания (SLA) о доступности подключения между вашей сетью и сетью Майкрософт. У ExpressRoute есть обязательное SLA доступности, которая составляет 99,9 %.
    
- Запланированная и гарантированная пропускная способность, необходимая для Microsoft 365 и Office 365 служб. Это можно сделать, отправив только трафик Microsoft 365, Office 365 или Skype для бизнеса через ExpressRoute, а затем весь интернет-трафик будет проходить через другие точки точки доступа к Интернету.
    
- ExpressRoute предназначен для сохранения маркировки QoS DSCP между вашей сетью и сетью Майкрософт.
    
Дополнительные сведения о QoS и планировании емкости ExpressRoute можно найти в [expressRoute и QoS](https://support.office.com/article/ExpressRoute-and-QoS-in-Skype-for-Business-Online-20c654da-30ee-4e4f-a764-8b7d8844431d)в Skype для бизнеса Online.
  
### <a name="can-i-setup-azure-expressroute-for-skype-for-business-online-only"></a>Можно ли настроить Azure ExpressRoute только для Skype для бизнеса Online?

Да, вы можете настроить Azure ExpressRoute, чтобы обеспечить отличное сетевое подключение из сети организации только Skype для бизнеса Online. Это обеспечит оптимальное качество Real-Time мультимедиа для пользователей, но вы сможете и дальше подключаться к другим Microsoft 365 или Office 365 через Интернет.
  
Протокол border Gateway Protocol (BGP) — это протокол маршрутизинга в Интернете, который используется для маршрутизки сетевого трафика через Интернет. Он предназначен для обмена данными маршрутов между автономными системами (AS), которые находятся в Интернете. Значения сообществ BGP — это теги атрибутов, которые можно применять к входящие и исходячие маршруты. Сообщества BGP часто используются для сигнала принимающим as, какую исходящие ссылки будут использоваться для достижения данной цели на основе географического положения, типа службы или других критериев.
  
При поддержке сообществ BGP майкрософт помечает префиксы и маршруты соответствующими значениями сообщества BGP в зависимости от службы, в службе, в которые они входят. Майкрософт помечает префиксы, объявленные через общедоступный пиринг и пиринг Майкрософт, соответствующими значениями сообщества BGP, указывающими регион, в котором размещаются префиксы. Вы можете использовать значения сообщества для принятия решений по маршрутистике, чтобы обеспечить оптимальную маршрутику. Значение сообщества BGP Skype для бизнеса Online можно использовать для настройки подключения ExpressRoute только для Skype для бизнеса Online. Дополнительные узнать об этом можно в [требованиях к маршрутике ExpressRoute.](/azure/expressroute/expressroute-routing)

Офисы компании соединены по сети WAN, которая использует MPLS для подключения к Microsoft 365 или Office 365. Изначально они настроили подключение ExpressRoute от интернет-маршрутизатора в Хобокене (Нью-Джерси) к сайту MeetMe в Нью-Йорке. 
  
При такой настройке сетевой трафик с большинства сайтов на сайт Microsoft Network (в Нью-Йорке) может соответствовать целевым показателям производительности сети клиента Skype для бизнеса, описанным в статьи Требования к производительности сети от клиента Skype для бизнеса до границы сети [Майкрософт.](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge) Однако задержка между офисами Contoso на западном побережье в Нью-Йорк составляет более 50 мс в одну сторону. Кроме того, Гонолулу — это второй по величине офис Contoso, задержка в пути из Гонолулу в Нью-Йорк превышает 80 мс в одну сторону. Чтобы обеспечить высокое качество мультимедиа для пользователей в этих офисах, компания Contoso решила добавить подключение ExpressRoute западного побережье между сайтом San Веб и сайтом MeetMe в Кремниевой долине ExpressRoute.
  
![Несколько сайтов Express Router на одном континенте.](../images/bf57a473-01e1-4271-9675-385767bc58e1.png)
  
### <a name="online-only-deployment---multiple-sites-on-different-continents"></a>Развертывание только в Интернете — несколько сайтов на разных континентах

Если все пользователи используют службу Skype для бизнеса Online и офисы находятся в разных физических расположениях на нескольких континентах, при развертывании Azure ExpressRoute необходимо настроить по крайней мере одно подключение ExpressRoute для каждого континента между основным сайтом на континенте и ближайшим расположением пиринга [ExpressRoute.](/azure/expressroute/expressroute-locations) В зависимости от затрат и преимуществ вы можете развернуть дополнительные подключения ExpressRoute с сайтов, где целевые показатели производительности не выполнены.
  
В следующем примере Contoso — это крупная юридическая компания с офисами в крупных городах Северной Америки и Европы. С учетом подключения к Интернету и оценки производительности внутренней сети компания Contoso решила развернуть два подключения ExpressRoute в Северной Америке и один канал ExpressRoute для всех своих европейских офисов.
  
![ExpressRoute с несколькими сайтами и континентами.](../images/c967550b-dc85-4b37-a7bc-cd825ec86854.png)
  
### <a name="hybrid-deployment"></a>Гибридное развертывание

Если у вас есть локальное развертывание Lync или Skype для бизнеса и вы решили внедрить гибридную интеграцию Skype для бизнеса Online, мы рекомендуем вам, если вы решите развернуть Azure ExpressRoute, необходимо по крайней мере одно подключение ExpressRoute для каждого локального сайта Lync или Skype для бизнеса Edge и хотя бы одно подключение ExpressRoute для каждого континента с офисами. В зависимости от затрат и преимуществ на каждом континенте можно развернуть дополнительные подключения ExpressRoute из офисов, где целевые показатели производительности не выполнены.
  
Если у вас есть локальное развертывание Skype для бизнеса, следуйте руководству по планированию и [развертыванию edge Server.](../../SfbServer/plan-your-deployment/edge-server-deployments/edge-server-deployments.md) В частности, внешние серверы должны быть недоступны. Обычно для этого необходимо назначить маршрутируемый общедоступный IP-адрес на edge-сервере или с помощью перевода сетевых адресов (NAT).
  
В следующем примере у Contoso уже есть локальное развертывание Skype для бизнеса Корпоративная голосовая связь развертывания. Они хотят перенести пользователей локально в Microsoft 365 или Office 365 службы. Кроме того, было принято решение о гибридном развертывании, которое позволяет продолжать использовать существующую инфраструктуру STN для всех пользователей локальной и сетевой сети. Локальное центр обработки данных Contoso и Skype для бизнеса Edge Servers находятся в Чикаго. Для развертывания компания Contoso решила настроить одно подключение ExpressRoute между центром обработки данных в Чикаго и Чикаго ExpressRoute. Кроме того, они добавили подключение ExpressRoute на западное побережье, чтобы лучше обслуживать офис в Гонолулу.
  
![Гибридная система ExpressRoute.](../images/a7467c56-642f-44e5-adfb-ecca91ba2dd3.png)
  
### <a name="online-deployment-with-cloud-connector-edition"></a>Развертывание Online с Cloud Connector Edition

Skype для бизнеса Online Cloud Connector Edition — это гибридное предложение, которое состоит из набора упакованных виртуальных машин (VMs), реализующих локальное подключение по STN. Развертывая минимальную топологию Skype для бизнеса Server в виртуализированной среде, вы сможете отправлять и принимать звонки с стационарных и мобильных телефонов через существующую локализованную инфраструктуру голосовой связи ЗВОНКОВ по ОКП.
  
Если вы решите развернуть Azure ExpressRoute и Cloud Connector Edition, рекомендуем настроить по крайней мере одно подключение Express Route для каждого континента между основным сайтом на каждом континенте и ближайшим расположением пиринга [ExpressRoute.](/azure/expressroute/expressroute-locations) В зависимости от затрат и преимуществ на каждом континенте можно развернуть дополнительные подключения ExpressRoute с сайтов, где целевые показатели производительности не выполнены.
  
Если у вас есть локальное развертывание Skype для бизнеса, следуйте руководству по планированию [для Skype для бизнеса Cloud Connector Edition.](../../SfbServer/skype-for-business-hybrid-solutions/plan-your-phone-system-cloud-pbx-solution/plan-skype-for-business-cloud-connector-edition.md) В частности, службам Edge Access и A/V Edge должны быть назначены общедоступные IP-адреса и доступны Microsoft 365 или Office 365 центрам обработки данных.
  
В следующем примере Contoso является европейской бухгалтерской фирмой, которая находится в нескольких крупных европейских странах и городах. При регистрации в Skype для бизнеса Online для совместной работы было принято решение создать облачный соединитель для каждой страны и региона, в которое они могут продолжать использовать инфраструктуру ННР и уже существовав контракты с операторами связи. На основе результатов тестирования на всех сайтах и в границе сети Майкрософт они определили, что одно подключение ExpressRoute в Лондоне поможет совместить с целевыми показателями производительности сети Skype для бизнеса, описанными в Skype для бизнеса требованиях к производительности сети от клиента Skype для бизнеса до границы сети [Майкрософт.](media-quality-and-network-connectivity-performance.md#bkSfBClienttoEdge)
