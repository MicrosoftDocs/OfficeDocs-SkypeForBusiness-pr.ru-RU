# <a name="set-up-and-troubleshoot-skype-for-business-online-delegation"></a>Настройка делегирования в Skype для бизнеса Online и устранение неполадок

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

В этой статье объясняется, как настроить и устранить неполадки Skype для бизнеса делегирования в Интернете. В этой статье вы можете получить рекомендации по настройке, рекомендациям и шагам по устранению неполадок.
  
## <a name="guidelines-and-requirements"></a>Рекомендации и требования

### <a name="guidelines-for-delegation"></a>Рекомендации по делегирования

Настройка и получение правильной работы делегирования зависят от следующих рекомендаций:
  
- Вы должны использовать клиент Skype для бизнеса 2015 с последними обновлениями или клиент Skype для бизнеса 2016.
    
- Необходимо использовать клиент Outlook 2013 с последними обновлениями или клиентом Outlook 2016 клиента.
    
- Убедитесь, что у компьютеров представителя и делегатов есть один профиль Outlook, который является основным или профилем по умолчанию. Этот профиль должен содержать только одну учетную запись.
    
- Skype для бизнеса представителем должны быть пользователи Online. Кроме того, Exchange Server почтовые ящики для каждой учетной записи должны быть либо в сети, либо оба они должны быть локально.
    
- Как представитель, так и представитель должны использовать один и тот же основной Outlook.
    
- Значение **атрибута EnableExchangeDelegateSync** должно иметь значение **true** в клиентской политике. Вы можете проверить этот параметр, задав **командлет Get-CSClientPolicy** в модуле Skype для бизнеса Online PowerShell.
    
- Как представитель, так и представитель должны быть одновременно Skype для бизнеса и Outlook на разных рабочих станциях.
    
- Общие почтовые ящики не поддерживаются для делегирования Skype для бизнеса Online. Это потому, что у общего почтового ящика нет списка управления доступом **к** отправителю.

## <a name="set-up-and-verify-delegation"></a>Настройка и проверка делегирования

Чтобы настроить делег Skype для бизнеса Online, выполните следующие действия:
  
### <a name="for-windows-clients"></a>Для Windows клиентов

 **Вкладка "Переад вызовы"**
  
1. Выберите   >  **Инструменты**  >  **Параметры Переад** Параметры .
    
2. Выберите **Изменить участников-делегатов**.
    
3. Выберите **Добавить**, выберите делегата, который вы хотите добавить, а затем выберите **ОК**.
    
 **Вкладка "Не переадваровка зовите"**
  
1. В Outlook выберите **Учетная** запись  >  **Параметры**  >  **делегировать**  >  **доступ Добавить**.
    
2. Найдите и добавьте имя человека, который будет делегатом.
    
3. В меню **Календарь** выберите пункт Редактор **(может читать, создавать и изменять элементы).**
    
### <a name="for-mac-clients---lync"></a>Для клиентов Mac — Lync

 **Вкладка "Переад вызовы"**
  
- Если у клиента нет  вкладки "Переадстройка зовов" со ссылкой Изменить участников-делегатов, а представитель находится на компьютере Mac, для того чтобы настроить делегирование, представитель должен войти на Windows на компьютере на базе Windows.  Это потому, что клиенты Mac не могут устанавливать подключения MAPI, и это необходимо для Skype для бизнеса делегирования Outlook.
    
### <a name="verify-success"></a>Проверка успешности

Если настройка прошла успешно, делегат должен увидеть, что вы были добавлены в качестве делегата  для < Name **>,** а также о том, что создана группа Люди, которые управляют звонками для. Делегат должен увидеть, что **создана** группа Делегаты.
  
> [!NOTE]
> Разрешения делегирования обычно появляются в течение 30 минут после настройки. Однако этот процесс может занять до 24 часов. 
  
## <a name="troubleshooting"></a>Устранение неполадок

### <a name="common-issues"></a>Распространенные проблемы

- > **Проблема 1** Запись делегата по-прежнему  будет отображаться в группе Люди, которые управляют звонками, после того как представитель удалил делегата из Outlook клиента.
    
  - > **Разрешение 1** В клиенте Skype для бизнеса щелкните правой кнопкой  мыши делегата в группе Представители и выберите **удалить из группы**.
    
- > **Проблема 2** После предоставления делегатского доступа через клиента Outlook ни подтверждение, ни  группа "Люди, которые управляют звонками для" не отображаются для делегата.
    
  - > **Разрешение 2** Удалите делегирование из клиента Outlook, подождите около 15 минут для репликации, а затем снова добавьте представителя.
    
### <a name="other-common-issues"></a>Другие распространенные проблемы

- Делегирования не работают, если превышен пороговое значение в 25 представителей и 25 делегатов.
    
- Клиент Skype для бизнеса Basic не поддерживается.
    
    > [!NOTE]
    > Если вы установите клиент Skype для бизнеса Basic, делегирования будут удаляться и разрываться. 
  
- Если значение **СОСТОЯНИЕ MAPI** не **ОК,** убедитесь, что значения **SIP** и **SMTP** совпадают.
    
    > [!NOTE]
    > Состояние MAPI отображается как ОК после  первого Skype для бизнеса и Outlook.
  
- Создание группы безопасности и добавление для нее разрешений делегирования не поддерживается.
    
- MapI недоступен. См. [сообщение об ошибке MAPI в Skype для бизнеса 2016.](https://support.microsoft.com/help/3147130)
    
- Почтовый Exchange Online будет недоступн через Skype для бизнеса клиента. В этом случае запустите проверку Outlook [подключения,](https://testconnectivity.microsoft.com/) чтобы убедиться, что он прошел.
