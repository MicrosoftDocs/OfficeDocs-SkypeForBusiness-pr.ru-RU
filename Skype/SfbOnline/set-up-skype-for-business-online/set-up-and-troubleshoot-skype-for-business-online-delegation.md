# <a name="set-up-and-troubleshoot-skype-for-business-online-delegation"></a>Настройка делегирования в Skype для бизнеса Online и устранение неполадок

В этой статье объясняется, как настроить и устранить неполадки, связанные с делегированием Skype для бизнеса Online. В этой статье приводятся инструкции по настройке, рекомендации и инструкции по устранению неполадок.
  
## <a name="guidelines-and-requirements"></a>Рекомендации и требования

### <a name="guidelines-for-delegation"></a>Рекомендации для делегирования

Настройка и обеспечение правильной работы делегирования зависят от указанных ниже правил.
  
- Вы должны использовать полную версию клиента Skype для бизнеса 2015 с последними обновлениями или полноценным клиентом Skype для бизнеса 2016.
    
- Необходимо использовать клиент Outlook 2013 с последними обновлениями или клиентом Outlook 2016.
    
- Убедитесь в том, что представитель и делегаты компьютеров имеют один профиль электронной почты Outlook, который является основным или профилем по умолчанию. Этот почтовый профиль должен содержать только одну учетную запись.
    
- Skype для бизнеса для представителя и делегат должны быть пользователями сети. Кроме того, почтовые ящики Exchange Server для каждой учетной записи должны быть либо в сети, либо локально.
    
- И представитель, и делегат должны использовать одну и ту же основную версию Outlook.
    
- В политике клиента для значения атрибута **енабликсчанжеделегатесинк** должно быть задано значение **true** . Вы можете проверить этот параметр, запустив командлет **Get-CSClientPolicy** в модуле PowerShell Skype для бизнеса Online.
    
- И представитель, и делегат должны входить в Skype для бизнеса и Outlook одновременно на разных рабочих станциях.
    
- Общие почтовые ящики не поддерживаются для делегирования Skype для бизнеса Online. Это связано с тем, что в общем почтовом ящике нет списка управления доступом (ACL) **сендонбехалф** .

## <a name="set-up-and-verify-delegation"></a>Настройка и проверка делегирования

Чтобы настроить делегирование Skype для бизнеса Online, выполните указанные ниже действия.
  
### <a name="for-windows-clients"></a>Для клиентов Windows

 **Вкладка "переадресация звонков"**
  
1. Выберите **Сервис** > **Параметры** > **переадресации звонков**.
    
2. Выберите команду **изменить участников делегата**.
    
3. Нажмите кнопку **Добавить**, выберите делегат, который вы хотите добавить, и нажмите кнопку **ОК**.
    
 **На вкладке "переадресация звонков"**
  
1. В **Outlook выберите** >  > **Параметры учетной записи**для**делегирования доступа** > **Добавить**.
    
2. Найдите и добавьте имя человека, который будет представителем.
    
3. Откройте меню **Календарь** и выберите **редактор (может просматривать, создавать и изменять элементы)**.
    
### <a name="for-mac-clients---lync"></a>Для клиентов Mac — Lync

 **Вкладка "переадресация звонков"**
  
- Если у клиента нет вкладки **переадресации звонков** , в которой есть ссылка **изменить участников-делегатов** , а он расположен на компьютере Mac, он должен войти на компьютер под управлением Windows, чтобы настроить делегирование. Это связано с тем, что клиенты Macintosh не могут устанавливать подключения по протоколу MAPI, и это требование для делегирования Skype для бизнеса из Outlook.
    
### <a name="verify-success"></a>Проверка успешности

Если установка прошла успешно, представителю следует видеть, что **вы добавили в качестве делегата для < именем>** сообщения, а также о том, что люди, которые **управляют звонками для** группы, созданы. Представитель должен увидеть, что группа **делегатов** создана.
  
> [!NOTE]
> Разрешения на делегирование обычно выводятся в течение 30 минут после установки. Тем не менее, этот процесс может занять до 24 часов. 
  
## <a name="troubleshooting"></a>Поиск и устранение неполадок

### <a name="common-issues"></a>Распространенные проблемы

- > **Вопрос 1** Запись делегата продолжает появляться в том случае, если **Пользователи управляют звонками для** групп после того, как он удалил делегат из клиента Outlook.
    
  - > **Разрешение 1** В клиенте Skype для бизнеса щелкните делегат в группе **представители** правой кнопкой мыши и выберите команду **удалить из группы**.
    
- > **Вопрос 2** После того как представителю предоставлен доступ через клиент Outlook, ни сообщение с подтверждением, ни **Пользователи, которые не управляют вызовами для** группы, появятся для этого представителя.
    
  - > **Разрешение 2** Удалите делегирование из клиента Outlook, подождите около 15 минут для репликации, а затем снова добавьте делегат.
    
### <a name="other-common-issues"></a>Другие распространенные проблемы

- Делегирование не работает в том случае, если превышено пороговое значение для 25 делегирований и 25 делегатов.
    
- Клиент Skype для бизнеса базовый не поддерживается.
    
    > [!NOTE]
    > При установке клиента Skype для бизнеса Basic будет удалено и разорвано делегирование. 
  
- Если значение **состояния MAPI** **не подходит, убедитесь, что**значения **SIP** и **SMTP** совпадают.
    
    > [!NOTE]
    > После первого запуска Skype для бизнеса и Outlook может потребоваться несколько минут, чтобы состояние MAPI отображалось как " **ОК** ".
  
- Создание группы безопасности и Добавление разрешений на делегирование для этой группы безопасности не поддерживается.
    
- MAPI недоступен. [В клиенте Skype для бизнеса 2016 появляется сообщение об ошибке "MAPI недоступен"](https://support.microsoft.com/help/3147130).
    
- Почтовый ящик Exchange Online недоступен в клиенте Skype для бизнеса. В этом случае выполните [проверку подключения Outlook](https://testconnectivity.microsoft.com/) , чтобы убедиться, что она пройдет.
