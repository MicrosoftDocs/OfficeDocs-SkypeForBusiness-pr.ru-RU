# <a name="set-up-and-troubleshoot-skype-for-business-online-delegation"></a>Настройка делегирования в Skype для бизнеса Online и устранение неполадок

В этой статье объясняется, как настроить делегирования в Skype для бизнеса Online и устранить неполадки с этим приложением. В этой статье указаны рекомендации по настройке, рекомендации и инструкции по устранению неполадок.
  
## <a name="guidelines-and-requirements"></a>Инструкции и требования

### <a name="guidelines-for-delegation"></a>Рекомендации по делегирования

Настройка и правильная настройка делегирования зависят от того, что вы совмести с этими правилами:
  
- Необходимо использовать полный клиент Skype для бизнеса 2015 с последними обновлениями или полный клиент Skype для бизнеса 2016.
    
- Необходимо использовать клиент Outlook 2013 с последними обновлениями или клиент Outlook 2016.
    
- Убедитесь, что у делегатов и компьютеров с делегированием есть один профиль почты Outlook, который является основным или профилем по умолчанию. Профиль почты должен содержать только одну учетную запись.
    
- Представитель и представитель Skype для бизнеса должны быть пользователями Online. Кроме того, Exchange Server почтовые ящики для каждой учетной записи должны быть либо сетевыми, либо оба они должны быть локально.
    
- У представителя и представителя должна быть одна и та же основная версия Outlook.
    
- В политике клиента значение атрибута **EnableExchangeDelegateSync** должно быть установлено как истинное.  Вы можете проверить этот параметр, задав командлет **Get-CSClientPolicy** в модуле Skype для бизнеса Online PowerShell.
    
- Как представитель, так и делегат должны одновременно ввести учетные данными в Skype для бизнеса и Outlook на разных рабочих станциях.
    
- Делегирования в Skype для бизнеса Online не поддерживаются для общих почтовых ящиков. Это значит, что у общего почтового ящика нет списка управления доступом **от** имени отправляемой почты.

## <a name="set-up-and-verify-delegation"></a>Настройка и проверка делегирования

Чтобы настроить делегирования в Skype для бизнеса Online, выполните следующие действия:
  
### <a name="for-windows-clients"></a>Для клиентов Windows

 **Вкладка "Переадные вызовы"**
  
1. Выберите **"Параметры**  >  **переададной**  >  **связи" в меню "Инструменты".**
    
2. Выберите **"Изменить участников-делегатов".**
    
3. Выберите **"Добавить",** выберите делегата, который вы хотите добавить, а затем выберите **"ОК".**
    
 **Вкладка "Не переад вызовов"**
  
1. В Outlook выберите **"Добавить"**  >  **в параметрах учетной** записи для  >  **учетной записи с делегированием**  >  **доступа.**
    
2. Найдите и добавьте имя человека, который будет делегатом.
    
3. В меню **"Календарь"** выберите "Редактор" **(может читать, создавать и изменять элементы).**
    
### <a name="for-mac-clients---lync"></a>Для клиентов Mac — Lync

 **Вкладка "Переадные вызовы"**
  
- Если у клиента нет  вкладки "Переадстройка вызовов", которая имеет ссылку "Изменить участников-делегатов", а представитель находится на компьютере Mac, то для того чтобы настроить делегирование, необходимо войти на компьютер с Windows.  Это необходимо для того, чтобы в клиентах Mac не было подключений MAPI, а это необходимо для делегирования skype для бизнеса из Outlook.
    
### <a name="verify-success"></a>Проверка успешности

В случае успешной настройки представитель  увидит, что вас добавили в качестве делегата < Имя >, а также о том, что создана группа "Люди, управление звонками для которых я". Делегат должен увидеть, что **группа** "Делегаты" создана.
  
> [!NOTE]
> Разрешения делегирования обычно появляются в течение 30 минут после настройки. Однако этот процесс может занять до 24 часов. 
  
## <a name="troubleshooting"></a>Устранение неполадок

### <a name="common-issues"></a>Распространенные проблемы

- > **Проблема 1** Запись делегата по-прежнему будет отображаться в группе **"Люди,** которые управляют звонками в группе", после того как представитель удал делегата из клиента Outlook.
    
  - > **Разрешение 1** В клиенте Skype для бизнеса щелкните  правой кнопкой мыши делегата в группе "Делегаты" и выберите **"Удалить из группы".**
    
- > **Проблема 2** После предоставления делегата в клиенте Outlook ни подтверждение,  ни группа "Люди, управление звонками для которых я" не отображаются для делегата.
    
  - > **Разрешение 2** Удалите делегирование из клиента Outlook, подождите около 15 минут репликации и снова добавьте делегата.
    
### <a name="other-common-issues"></a>Другие распространенные проблемы

- Делегирования не работают, если превышено пороговое значение в 25 представителей и 25 делегатов.
    
- Клиент Skype для бизнеса Basic не поддерживается.
    
    > [!NOTE]
    > При установке клиента Skype для бизнеса Basic он удаляет и разрывает делегирования. 
  
- Если значение **состояния MAPI** не **соответствует,** убедитесь, что значения **SIP** и **SMTP** совпадают.
    
    > [!NOTE]
    > Состояние MAPI отображается в состоянии "ОК" в течение нескольких минут после первого запуска Skype для бизнеса и Outlook. 
  
- Создание группы безопасности и добавление для нее разрешений делегирования не поддерживается.
    
- MapI недоступна. См. [сообщение об ошибке MAPI в клиенте Skype для бизнеса 2016.](https://support.microsoft.com/help/3147130)
    
- Почтовый ящик Exchange Online не доступен через клиент Skype для бизнеса. В этом случае запустите тест [подключения к Outlook,](https://testconnectivity.microsoft.com/) чтобы убедиться, что он прошел.
