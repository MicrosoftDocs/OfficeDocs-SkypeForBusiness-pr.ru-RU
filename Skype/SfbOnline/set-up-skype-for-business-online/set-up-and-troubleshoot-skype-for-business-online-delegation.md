# <a name="set-up-and-troubleshoot-skype-for-business-online-delegation"></a>Настройка и устранение неполадок Скайп для бизнеса в Интернет делегирования

В этой статье объясняется, как настроить и устранении неполадок Скайп для бизнеса в Интернет делегирования. В этой статье описаны рекомендации по рекомендации по настройке, рекомендации и действия по устранению неполадок.
  
## <a name="guidelines-and-requirements"></a>Рекомендации и требования

### <a name="guidelines-for-delegation"></a>Рекомендации для делегирования

Настройка и делегирования для правильной работы зависит от следующие рекомендации:
  
- Необходимо использовать Скайп для бизнеса 2015 полный клиент с последними обновлениями или Скайп для полного клиента 2016 бизнеса.
    
- Необходимо использовать клиент Outlook 2013 с последними обновлениями или в клиенте Outlook 2016.
    
- Убедитесь в том, что представитель и делегат компьютеры имеют один профиль почты Outlook, который является основной или профиля по умолчанию. Что профиль электронной почты должен содержать только одну учетную запись.
    
- Скайп для бизнеса для сотрудника и делегат должны быть активные пользователи. Кроме того почтовые ящики Exchange Server для каждой учетной записи, необходимо быть Online или быть локальной.
    
- Представитель и делегат должен с помощью той же основной версии Outlook.
    
- Значение атрибута **EnableExchangeDelegateSync** должен иметь значение **true** в политике клиента. Этот параметр можно проверить с помощью командлета **Get-CSClientPolicy** в Скайп для модуля Online PowerShell бизнеса.
    
- Представитель и делегат должен входить в систему Скайп для бизнес-деятельности и Outlook в то же время на разных рабочих станциях.
    
- Общие почтовые ящики не поддерживаются для Скайп для бизнеса в Интернет делегирования. Это так, как общего почтового ящика не имеет **sendonbehalf** управления доступом (ACL).

## <a name="set-up-and-verify-delegation"></a>Установка и проверка делегирования

Чтобы настроить Скайп для бизнеса в Интернет делегирования, выполните следующие действия:
  
### <a name="for-windows-clients"></a>Для клиентов Windows

 **Вкладка переадресации звонков**
  
1. Выбор **инструментов** > **Параметры** > **параметры переадресации звонков**.
    
2. Выберите команду **изменить участников-делегатов**.
    
3. Выберите **Добавить**, выберите делегата, который вы хотите добавить и нажмите кнопку « **ОК»**.
    
 **Отсутствует вкладка переадресация звонков**
  
1. В программе Outlook, выберите **файл** > **Параметры учетной записи** > **Делегированный доступ** > **Add**.
    
2. Найдите, а затем добавьте имя человека, который будет делегата.
    
3. Выберите меню « **Календарь** », а затем выберите **редактора (можно читать, создавать и изменять элементы)**.
    
### <a name="for-mac-clients---lync"></a>Для клиентов, Mac - Lync

 **Вкладка переадресации звонков**
  
- Если клиент не настроено на вкладке **Переадресация звонков** , имеющей ссылку **изменить участников-делегатов** и представитель расположен на компьютере Mac, представитель необходимо выполнить вход на компьютере под управлением Windows для настройки делегирования. Это так, как клиентов не может подключаться MAPI, и это требование для установления Скайп для делегирования Business из Outlook.
    
### <a name="verify-success"></a>Проверка успешности

Если настройка выполнена успешно, делегат должны отображаться **были добавлены в качестве представителя для < имя >** сообщения и Кроме того, создается группа **людей я управление вызовы для** . Представитель должны видеть, что создана группа **делегаты** .
  
> [!NOTE]
> Делегирование разрешений обычно отобразиться в течение 30 минут процесс установки. Тем не менее этот процесс может занять более 24 часов. 
  
## <a name="troubleshooting"></a>Поиск и устранение неполадок

### <a name="common-issues"></a>Распространенные проблемы

- > **Проблема 1** Запись делегат остается в группе **людей я управление вызовы для** после представитель делегата из клиента Outlook.
    
  - > **Решение 1** На Скайп для клиента Business щелкните правой кнопкой мыши пользователя в группу **делегатов** и затем выберите команду **Удалить из группы**.
    
- > **Проблема 2** После делегированный доступ предоставляется с помощью клиента Outlook, ни сообщении о подтверждении, ни **людей я управление звонки для** группы отображаются для делегата.
    
  - > **Решение 2** Удаление делегирования из клиента Outlook, подождите в течение 15 минут для репликации и затем снова добавьте делегат.
    
### <a name="other-common-issues"></a>Другие распространенные проблемы

- Делегирование не работает при превышении порогового значения 25 выполнившему и 25 делегатов.
    
- Скайп для бизнеса базовый клиент не поддерживается.
    
    > [!NOTE]
    > При установке Скайп для бизнеса базовой клиента, его можно было удалить и нарушение делегирования. 
  
- Если значение **Состояния MAPI** не **ОК**, убедитесь в том, что соответствующие значения **SIP** и **SMTP** .
    
    > [!NOTE]
    > Может потребоваться несколько минут состояние MAPI для отображения как **ОК** после сначала запустите Скайп для бизнес-деятельности и Outlook.
  
- Создание группы безопасности и добавления разрешений делегирования для этой группы безопасности не поддерживается.
    
- MAPI недоступна. В разделе [«недоступен» ошибка в Скайп для клиента 2016 бизнеса](https://support.microsoft.com/en-us/help/3147130).
    
- Почтовый ящик Exchange Online не будет доступна через Скайп для клиента Business. При возникновении этой ошибки запустите [Outlook Проверка подключения](https://testconnectivity.microsoft.com/) к убедитесь в том, что они передают.
