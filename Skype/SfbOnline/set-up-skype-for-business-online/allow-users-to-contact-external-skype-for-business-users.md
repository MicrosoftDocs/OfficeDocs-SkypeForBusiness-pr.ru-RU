# <a name="allow-users-to-contact-external-skype-for-business-users"></a>Разрешите пользователям связываться с внешними пользователями Skype для бизнеса

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]
  
Выполните действия, описанные в этой статье, в следующих случаях.
  
- У вас есть пользователи в разных доменах в вашей компании. Например, Rob@ContosoEast.com и Ann@ContosoWest.com.

- Вы хотите, чтобы люди в вашей организации использовали Skype для бизнеса для связи с людьми в определенных организациях за пределами организации.

- Вы хотите, чтобы все, кто использует Skype для бизнеса, могли найти вас и связаться с вами, используя ваш адрес электронной почты. Если вы и они используете параметры Skype для бизнеса по умолчанию, это будет работать автоматически. В этом случае необходимо убедиться, что конфигурация не блокирует ваш домен.

## <a name="enable-business-to-business-communications-for-your-users"></a>Включить взаимодействие между пользователями

<a name="bk_preview"> </a>

Для этого у вас [должны](https://support.office.com/article/da585eea-f576-4f55-a1e0-87090b6aaa9d) быть разрешения администратора Microsoft 365 или Office 365 в обеих организациях.

![Значок с логотипом Microsoft Teams.](../images/teams-logo-30x30.png) **Использование центра Teams администрирования**
  
1. Во sign in with your Microsoft 365 or Office 365 admin account.

2. В Центре администрирования перейдите **в** центр администрирования  >  Teams.

    ![Выберите администратор Teams администратора.](../images/MS-Teams-Admin.png)
  
3. В центре **Teams** выберите **Skype** > **устаревший портал** Выберите 
  ![ устаревший портал SfB.](../images/SFBlegacy-size65.png)

4. В **Центре администрирования Skype для бизнеса** выберите пункт **Организация** > **Внешний обмен данными**.
5. Чтобы настроить обмен данными с конкретной компанией или с пользователями из другого домена, в раскрывающемся списке выберите **Включить только для разрешенных доменов**.

    Или, если вы хотите включить связь со всеми пользователями в мире, которые Skype для бизнеса политики, выберите В сети, кроме **заблокированных доменов.** Это параметр по умолчанию.

6. В **области Заблокированные или разрешенные домены** выберите и добавьте имя домена, **+** который вы хотите разрешить.

7. Убедитесь, что администратор другой организации делает те же действия в Skype для бизнеса **администрирования**. Например, в списке **разрешенных доменов** его администратор должен ввести домен для вашей компании.

8. Если вы используете брандмауэр Windows, Skype для бизнеса автоматически открывает необходимые порты.

    Если в вашей организации используется другое решение брандмауэра для ограничения подключения компьютеров в сети к Интернету, убедитесь, что клиентские компьютеры имеют доступ к следующим [URL Office 365 URL-адресам](/microsoftteams/office-365-urls-ip-address-ranges)и диапазонам IP-адресов. Для этого может потребоваться добавить FQDNs в список исходящие разрешит в конфигурации брандмауэра или прокси-инфраструктуры: **\* .api.skype.com,** \* *_.users.storage.live.com_* и **graph.skype.com.** Инструкции по открытием этих портов в брандмауэре можно найти в документации к ним.

    Список всех портов, которые нужно открыть, см. в Office 365 [URL-адреса и диапазоны IP-адресов.](/microsoftteams/office-365-urls-ip-address-ranges)

9. Убедитесь, что администратор организации также следовал этим шагам.

10. **ПОДОЖДИТЕ ДО 24 ЧАСОВ.** Изменение параметров внешней связи может занять до 24 часов, чтобы изменения были внесены во все центры обработки данных.

![Skype.](../images/58550720-2a68-42d1-a926-1884e6aeb55c.png) Вы можете разрешить пользователям искать и мгновенные мгновенные данные для всех, кто пользуется бесплатным Skype приложением для потребителей! Дополнительные информация см. [в Skype для бизнеса пользователям добавлять Skype контактов.](let-skype-for-business-users-add-skype-contacts.md)
  
## <a name="test-and-troubleshoot"></a>Проверка и устранение неполадок

<a name="bk_preview"> </a>

 **Самая распространенная проблема, с которой сталкиваются люди при настройке бизнес-связи, — это правильное Office 365 [URL-адреса и диапазоны IP-адресов.](/microsoftteams/office-365-urls-ip-address-ranges)**
  
Чтобы проверить настройку, вам нужен контакт Skype для бизнеса, который не работает в брандмауэре вашей компании.
  
1. После изменения параметров внешней связи ПОдождите **до 24 ЧАСОВ.**

2. В Skype для бизнеса найщите контакт в Skype для бизнеса и отправьте запрос на чат.

    Если вы получаете сообщение о том, что не удалось отправить его из-за политики компании, необходимо перепроверим URL-Office 365 и [диапазоны IP-адресов.](/microsoftteams/office-365-urls-ip-address-ranges)

3. Попросите контакт Skype для бизнеса отправить вам запрос на чат. Если вы не получили его запрос, проблема в настройках вашего брандмауэра (предполагается, что он подтвердил корректность настроек своего брандмауэра).

4. Еще один способ проверить, есть ли проблема в брандмауэре, — перейти в расположение Wi-Fi, не расположенную за брандмауэром, например в кафе. Используйте Skype для бизнеса, чтобы отправить контакту запрос на чат. Если сообщение там проходит, но не на работе, то вы знаете, что проблема в брандмауэре.

## <a name="how-to-find-others-and-be-found-when-connecting-with-another-business"></a>Как найти других людей и найти их при подключении к другой компании

<a name="bk_preview"> </a>

После того как вы ввяжете внешнюю связь с другими Skype для бизнеса пользователями, пользователи смогут находить федеративную Skype для бизнеса пользователей, ища их имена для входов. Например, Rob@contoso.com. Затем нужно будет добавить этого человека в свой список контактов.

## <a name="tips-on-setting-up-communications-with-federated-businesses"></a>Советы о настройке связи с федеративными организациями

<a name="bk_preview"> </a>

- Чтобы настроить федерацию между Skype для бизнеса 2015 и Skype для бизнеса Online, см. статью Настройка федерации с [Skype для бизнеса Online.](../../SfbHybrid/hybrid/configure-federation-with-skype-for-business-online.md?bc=%2fSkypeForBusiness%2fbreadcrumb%2ftoc.json&toc=%2fSkypeForBusiness%2ftoc.json)

- Чтобы настроить федерацию между Lync и Skype для бизнеса Online, см. статью Настройка поддержки федерации для [клиента Lync Online.](/previous-versions/office/lync-server-2013/lync-server-2013-configuring-federation-support-for-a-lync-online-customer)

- Когда два Skype для бизнеса в Microsoft 365 или Office 365 общаются друг с другом в разных доменах, они могут использовать только функции Skype для бизнеса (например, видеособрания или совместный доступ к рабочему столу), которые включены в обеих организациях.

- Если Skype для бизнеса пользователя в вашей организации поставлен на удержание In-Place или для судебного разбирательства, все мгновенные сообщения между ним и  другими пользователями Skype для бизнеса или Skype сохраняются в почтовом ящике в элементе с возможностью восстановления. Эти беседы не сохраняются в **папке** "История бесед" их почтового ящика.

## <a name="turn-off-external-communication-for-specific-individuals"></a>Отключение внешней связи для определенных пользователей

<a name="bk_preview"> </a>

После того как вы включите внешнюю связь для всей компании, вы можете отключить ее только для определенных пользователей.
  
1. Во sign in with your Microsoft 365 or Office 365 admin account.

2. В Центре администрирования перейдите в группу **Пользователи**  >  **Активные пользователи**.

3. В списке пользователей выберите пользователя, а затем в списке Дополнительные Параметры **щелкните** **Изменить** свойства Skype для бизнеса .

    ![Выберите Skype для бизнеса.](../images/2b0f9a7b-3fee-4f4b-968a-68c429eeb395.png)
  
4. В Центре **администрирования Skype для бизнеса выберите** **Внешняя связь**.

    На странице **Параметры** будут выбраны все варианты. Очистка сообщений, которые вы хотите отключить. На рисунке ниже показано, что пользователь Jakob сможет общаться с людьми из других доверенных компаний, но не с другими Skype пользователями.

    ![Выберите Внешние контакты.](../images/4e546321-a065-48ed-8ac7-1e112a780eab.png)
  
5. Нажмите кнопку **Сохранить**.

> [!NOTE]
> Чтобы изменения вступили в силу, может потребоваться до 24 часов.
