# <a name="end-of-life-program-for-the-integration-of-skype-for-business-with-third-party-audio-conferencing-providers"></a>Конец жизненного программы для интеграции Скайп для бизнеса с аудиоконференций сторонних поставщиков 

Microsoft объявили об изменении Пуск конец жизненного программы для интеграции Скайп для бизнеса с аудиоконференций сторонних поставщиков (ACPs). 

В заключение конец жизненного программы будет на 1 апреля 2019. При завершается программа интеграция Скайп для бизнеса с аудиоконференций сторонних поставщиков перестанет работать и следующие изменения будут настраиваться на этой даты (1 апреля 2019): 

- Участники, у которых попытались присоединиться к любой Скайп для собраний с помощью телефонных номеров, предоставляемые с ACP сторонних производителей будут подключены службы Скайп для собраний.
 
- Приглашает пользователей включены для сторонних служб ACP больше не будут иметь свои данные для подключения автоматически включается в любой новый Скайп для собраний. 

Как часть объявлений Пуск конец жизненного программы указанные ниже изменения вступили в силу и останется на месте до заключение конец жизненного программы: 

- Клиенты с не Скайп для бизнес-пользователи, которые настроены для использования службы ACP сторонних производителей не сможет настроить все пользователи использовать службу ACP сторонних производителей. 

- Существующие клиенты с Скайп для бизнес-пользователи, которые настроены для использования службы ACP сторонних производителей будет иметь возможность добавления новых пользователей во время окончания срока хранения. Обратите внимание, что не рекомендуется настроить дополнительные Скайп для бизнес-пользователям использовать службу ACP сторонних производителей, как изменения, которые будут вступают в силу в 1 апреля 2019, также применимы к ним. 

## <a name="preparing-for-this-change"></a>Подготовка к этому изменению

Чтобы подготовиться к этому изменению, мы рекомендуем затронутых организациями для уведомления включенных пользователей плановых обновления до 1 апреля 2019. 

После 1 апреля 2019 пользователи могут продолжить использовать Скайп для бизнеса с отсутствуют не изменяя их собрания по сети; Тем не менее организации потребуется включить пользователей для аудиоконференций, предоставляемым корпорацией Майкрософт, если они требуют звуковых конференций с Скайп для бизнеса или группами Майкрософт. Чтобы узнать больше о Microsoft звук конференц-связи, обратитесь к разделу [Аудиоконференции](https://products.office.com/en-us/skype-for-business/audio-conferencing). 

В зависимости от состояния нужную организации существует три путей, которые может подписаться.

- Перенос в конференц-связи Microsoft звука. 
- Продолжайте отдельно использовать поставщика аудиоконференций сторонних производителей. 
- Прекратите использование телефонных конференций полностью.

### <a name="path-1-migrate-to-microsoft-audio-conferencing"></a>Путь к #1: Перенос в конференц-связи Microsoft звука   

Организации, которые решить, перенос в конференц-связи Microsoft аудио и выполнить их миграции до 1 апреля 2019, не смогут работать любой влияние службы во время или после этой даты. Миграция на Microsoft аудиоконференции появятся следующие изменения в организации: 

- Служба будет международные с другими службами Office 365. 

- Если приобретен стандартной подписки, международную телефонных расходов будут включены в индивидуальной месячная подписка расходы. 

- Новый набор телефонные номера телефонов будут предоставлены каждой организации и пользователей. Географического покрытия служба конференц-связи Microsoft звук, см [страны и региона доступности для конференц-связи аудио и вызов планов](/microsoftteams/country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans).
 
- Собрания, которые уже были помощнику по доступны с помощью сторонних производителей ACP пользователей будет автоматически запланировано на конференц-связи Microsoft аудио-связи с телефонным сведения.
 
- Идентификаторы для каждого собрания конференции будет динамическим, что означает, что каждому собранию будет иметь собственный идентификатор выделенного конференции. Идентификаторы динамических конференции предоставляют расширенные возможности безопасности и улучшают для собраний с двумя межсетевыми экранами.

- Все использования службы — это может быть звук конференц-связи, службы используют условия. 

Миграция на Microsoft аудиоконференции простой и его можно выполнить в несколько действий после получения лицензии для службы. Сведения о способах миграции для конференц-связи Microsoft звук, см.

- [Пробная и платная аудиоконференции в Office 365](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md)
 
**Краткое содержание**.

- Организации, которые перенос в конференц-связи Microsoft аудио и выполнение их миграции до 1 апреля 2019, не будут отображаться любые влияния их службе во время или после этой даты.

- Дополнительные сведения о переходе на конференц-связи Microsoft звук видеть [Попробуйте или Покупка Audio конференция в Office 365](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md). 

### <a name="path-2-continue-to-separately-use-a-third-party-audio-conferencing-provider"></a>Путь к #2: По-прежнему отдельно использование поставщика аудиоконференций сторонних производителей

Организации, которые решить для дальнейшего использования ACP сторонних производителей, после 1 апреля 2019, и смогут работать влияние службы, так как сторонних производителей ACP данные для подключения больше не будет использоваться для присоединения к звуковой части Скайп для собраний. 

Чтобы запретить фрагментации аудио в Скайп для собраний бизнеса благодаря использованию некоторых участников, подключающихся по протоколу VoIP и других через ACP сторонних производителей, рекомендуется для этих организаций для отключения с помощью VoIP на собраниях своим пользователям. Таким образом, все участники должны будут подключаться к аудиоканалу собрания с помощью ACP сторонних производителей и других рабочих нагрузок собрания (например, беседы или совместное использование рабочего стола) по-прежнему имели поддерживается по Скайп для бизнеса. 

- Чтобы отключить VoIP из всех собраний данного организатора, установите для параметра AllowIPAudio свой политики конференц-связи значение false с помощью командлета Set-CsConferencingPolicy. Для получения дополнительных сведений см [Set-CsConferencingPolicy](https://docs.microsoft.com/en-us/powershell/module/skype/set-csconferencingpolicy?view=skype-ps).
 
С точки зрения планирования, а также по состоянию на 1 апреля 2019 данные для подключения от сторонних производителей ACP больше не включается автоматически в Скайп для приглашения собрание бизнеса. Пользователи должны вручную добавить данные для подключения на их Скайп для собраний приглашает возможным для продолжения, включая эти сведения в составе свои собрания. 

Обратите внимание, что на 1 апреля 2019, существующих собраний пользователей будет не выполнено автоматически для удаления любого сторонних производителей ACP данные для подключения. Организации, которые сохранять VoIP, включенные для собраний своим пользователям следует подумать об отключении интеграции ACP сторонних производителей для своих пользователей и настройки расписания свои собрания с помощью собрания службы миграции для удаления аудио сторонних производителей конференц-связи данные для подключения из их существующих собраний и предотвратить степени фрагментации звука на уже запланированные собрания. 

- Чтобы отключить интеграцию аудиоконференций сторонних производителей для данного организатора, используйте командлет Remove-CsUserAcp. Для получения дополнительных сведений см [Remove-CsUserAcp](https://docs.microsoft.com/en-us/powershell/module/skype/remove-csuseracp?view=skype-ps). 

- Чтобы автоматически изменить расписание собраний пользователей после отключения интеграции с помощью поставщика аудиоконференций сторонних производителей, см «Как выполнить перенос собрания вручную для пользователя?» на странице [параметр копирование служба миграции собрания (MMS)](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md). 

**Краткое содержание**.

- Организации, которые решить для дальнейшего использования ACP сторонних производителей, после 1 апреля 2019, и затрагиваются так как ACP сторонних производителей не смогут использовать для присоединения к Скайп для собраний и новые собрания не будет включать сторонних производителей ACP данные для подключения. 

- Рекомендуется отключить VoIP для всех собраний всех заинтересованных пользователей до 1 апреля 2019 для предотвращения звук фрагментированные через участников, подключающихся по протоколу VoIP и через ACP сторонних производителей. 

    - Для отключения VoIP из всех собраний данного организатора, значение false с помощью командлета Set-CsConferencingPolicy параметр AllowIPAudio политику конференц-связи. Для получения дополнительных сведений см [Set-CsConferencingPolicy](https://docs.microsoft.com/en-us/powershell/module/skype/set-csconferencingpolicy?view=skype-ps).
 
- Если организация не отключить VoIP для всех собраний, рекомендуется для пользователей отключить с помощью Скайп для бизнеса в Интернет интеграции с ACP сторонних производителей и настройки расписания свои собрания для удаления сторонних производителей ACP данные для подключения к Предотвращение фрагментации звука.

    - Чтобы отключить интеграцию аудиоконференций сторонних производителей для данного организатора, используйте командлет [Remove-CsUserAcp](https://docs.microsoft.com/en-us/powershell/module/skype/remove-csuseracp?view=skype-ps) . 

    - Чтобы автоматически изменить расписание собраний, см «Как выполнить перенос собрания вручную для пользователя?» на странице [параметр копирование служба миграции собрания (MMS)](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md).

### <a name="path-3-stop-using-dial-in-conferencing-altogether"></a>Путь к #3: Остановите полностью с помощью конференц-связи

Организациям, которые решено прекратить использование телефонных конференций полностью (не предоставляется корпорацией Майкрософт, ни с ACP сторонних производителей) можно полностью полагаться на VoIP для поддержки звуковой части Скайп для собраний. 

Эти организации необходимо отключить пользователей с помощью поставщика аудиоконференций сторонних производителей и иметь свои собрания автоматически перенести с помощью миграции конференц-службы для удаления данных их телефонных конференций. 

- Чтобы отключить интеграцию аудиоконференций сторонних производителей для данного организатора, используйте командлет Remove-CsUserAcp. Для получения дополнительных сведений см [Remove-CsUserAcp](https://docs.microsoft.com/en-us/powershell/module/skype/remove-csuseracp?view=skype-ps). 

- Чтобы автоматически изменить расписание собраний пользователей после отключения интеграции с помощью поставщика аудиоконференций сторонних производителей, см «Как выполнить перенос собрания вручную для пользователя?» на странице [параметр копирование служба миграции собрания (MMS)](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md). 

**Краткое содержание**. 

- Организации, которые решить, чтобы прекратить использование аудиоконференций полностью до 1 апреля 2019, не повлияет.

- Чтобы отключить интеграцию аудиоконференций сторонних производителей для данного организатора, используйте командлет Remove-CsUserAcp. Для получения дополнительных сведений см [Remove-CsUserAcp](https://docs.microsoft.com/en-us/powershell/module/skype/remove-csuseracp?view=skype-ps). 

- Чтобы автоматически изменить расписание собраний пользователей после отключения интеграция со службами аудиоконференций сторонних производителей, см «Как выполнить перенос собрания вручную для пользователя?» на странице [параметр копирование служба миграции собрания (MMS)](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md).
