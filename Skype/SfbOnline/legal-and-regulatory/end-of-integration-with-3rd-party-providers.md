# <a name="end-of-life-program-for-the-integration-of-skype-for-business-with-third-party-audio-conferencing-providers"></a>Программа окончания жизненного процесса интеграции Skype для бизнеса со сторонними поставщиками аудиоконференций 

Корпорация Майкрософт объявила о начале жизненного процесса интеграции Skype для бизнеса со сторонними поставщиками аудиоконференций (ACP). 

Жизненный срок программы заканчивается 31 июля 2021 г. После завершения программы интеграция Skype для бизнеса со сторонними поставщиками аудиоконференций перестанет работать, и в эту дату будут внесены следующие изменения (31 июля 2021 г.):

- Участники, которые пытаются присоединиться к любому собранию Skype для бизнеса с помощью номеров телефонного подключения, предоставленных стороной службой ACP, больше не будут подключаться к собранию Skype для бизнеса.
 
- Для пользователей, включенных в сторонние службы ACP, больше не будут автоматически включаться данные телефонного дозвона во все новые приглашения на собрания Skype для бизнеса.

После объявления о начале окончания жизненного года вступает в силу следующее изменение, которое будет продолжаться до заключения программы: 

- Клиенты, у пользователей Skype для бизнеса не настроенные для использования сторонних служб ACP, не смогут настроить их для работы со сторонними службами ACP.

- Существующие клиенты с пользователями Skype для бизнеса, настроенными на использование сторонних служб ACP, по-прежнему смогут добавлять новых пользователей в течение периода окончания жизненного периода. Обратите внимание, что мы не рекомендуем настраивать дополнительных пользователей Skype для бизнеса для работы со сторонними службами ACP, поскольку изменения, внося в них 31 июля 2021 г., также будут применяться.

## <a name="preparing-for-this-change"></a>Подготовка к этому изменению

Для подготовки к этому изменению рекомендуется уведомить пользователей затронутых организаций об этом запланированном обновлении до 31 июля 2021 г. 

После 31 июля 2021 г. пользователи могут продолжать использовать Skype для бизнеса без каких-либо перерывов в работе с онлайн-собраниями. тем не менее организациям необходимо включить для пользователей аудиоконференцию, предоставляемую Корпорацией Майкрософт, если требуется аудиоконференция с телефонным доступом в Skype для бизнеса или Microsoft Teams. Дополнительные информацию об аудиоконференциях (Майкрософт) см. в [записи аудиоконференции.](https://products.office.com/skype-for-business/audio-conferencing) 

В зависимости от желаемого конечного состояния организации существует три пути, по которым можно следовать:

- Переход на аудиоконференцию Майкрософт. 
- Продолжайте использовать отдельно стороного поставщика аудиоконференций. 
- Полностью отменить использование телефонной и телефонной телефонной телефонии.

### <a name="path-1-migrate-to-microsoft-audio-conferencing"></a>Путь #1: переход на аудиоконференцию Майкрософт   

Организации, которые решили перейти на аудиоконференцию Майкрософт и завершить миграцию до 31 июля 2021 г., не будут влиять на работу службы в этот период или после нее. При переходе на аудиоконференцию (Майкрософт) в организации введутся следующие изменения: 

- Счета за службу будут выставлены вместе со всеми другими службами Microsoft 365 или Office 365. 

- Если вы приобрели стандартную подписку, платные затраты на телефонное звонок включаются в месячную стоимость подписки для каждого пользователя. 

- Каждой организации и ее пользователям будет предоставлен новый набор телефонных номеров для телефонного звонка. Узнать о географическом покрытия службы аудиоконференц Майкрософт можно в области "Страна и регион" для аудиоконференций и планов [звонков.](/microsoftteams/country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans)
 
- Собрания, которые уже были запланированы пользователями, включенными сторонним аудиоконференцией, будут автоматически перепланируются таким образом, чтобы включить сведения о телефонном подмногу Microsoft Audio Conferencing.
 
- Все собрания будут иметь динамические и динамические iD, то есть у каждого собрания будет свой выделенный конференц-зал. Динамические ИД конференц-залов обеспечивают улучшенную безопасность и улучшенные возможности для проведения собраний сзади.

- Все использование службы облагается условиями использования служб аудиоконференций. 

Перейти на аудиоконференцию Майкрософт очень просто, и это можно сделать всего за пару шагов после приобретения лицензий для службы. Чтобы узнать, как перейти на аудиоконференцию Майкрософт, см. 365.

- [Попробуйте или приобретйте аудиоконференцию в Microsoft 365 или Office 365](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md)
 
**Сводка.**

- Организации, которые переход на аудиоконференцию Майкрософт и завершают миграцию до 31 июля 2021 г., не будут видеть влияния на службу в течение или после этой даты.

- Дополнительные информацию о переходе на аудиоконференцию (Майкрософт) см. в записи "Попробуйте или приобретйте аудиоконференцию в [Microsoft 365 или Office 365".](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md) 

### <a name="path-2-continue-to-separately-use-a-third-party-audio-conferencing-provider"></a>Путь #2: использование стороннее поставщика аудиоконференций отдельно

Организации, которые решили продолжить использовать стороннюю аудиосистему аудиосвязи после 31 июля 2021 г., будут иметь влияние на службу, так как сторонние данные для телефонного дозвона больше не будут использоваться для звонков в Skype для бизнеса. 

Чтобы предотвратить фрагментацию звука в собраниях Skype для бизнеса путем присоединения некоторых участников к собранию по VoIP и другому через сторонний ACP, организациям рекомендуется отключить использование VoIP на собраниях пользователей. Таким образом, всем участникам потребуется присоединиться к звуковому фрагменту собрания с помощью сторонних аудиосистем, и все другие рабочие нагрузки собрания (например, чат или совместный доступ к экрану) могут по-прежнему поддерживаться через Skype для бизнеса. 

- Чтобы отключить VoIP для всех собраний организатора, задайте для параметра AllowIPAudio его политики конференций Set-CsConferencingPolicy false. Дополнительные сведения [см. в поле Set-CsConferencingPolicy.](https://docs.microsoft.com/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
 
С точки зрения планирования, а с 31 июля 2021 г. данные для телефонного подбираемого номера сторонних ACP больше не будут автоматически включаться в приглашения на собрания Skype для бизнеса. Пользователям потребуется вручную добавить данные телефонного дозвона в приглашения на собрания Skype для бизнеса, если они хотят и дальше добавлять эти сведения в свои собрания. 

Обратите внимание, что 31 июля 2021 г. существующие собрания пользователей не будут автоматически запланированы повторно для удаления сторонних данных для телефонного дозвона через ACP. Организациям, решивших поддерживать функцию VoIP для собраний пользователей, следует отключить интеграцию сторонних аудиоконференций для пользователей и повторно запланирования собраний с помощью службы переноса собраний, чтобы удалить сторонние данные для аудиоконференций для телефонного дозвона из существующих собраний и предотвратить фрагментацию звука в уже запланированных собраниях. 

- Чтобы отключить интеграцию аудиоконференции сторонних Remove-CsUserAcp для данного организатора. Дополнительные сведения [см. в remove-CsUserAcp.](https://docs.microsoft.com/powershell/module/skype/remove-csuseracp?view=skype-ps) 

- Чтобы автоматически перенести собрания пользователей после отключения интеграции со сторонним поставщиком аудиоконференций, см. "Как вручную выполнить перенос собраний для пользователя?". в [настройках службы переноса собраний (MMS).](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md) 

**Сводка.**

- Организации, которые решили продолжить использовать сторонний ACP до 31 июля 2021 г., будут затронуты тем, что сторонний ACP не сможет присоединиться к собранию Skype для бизнеса, а новые собрания не будут включать сторонние данные для телефонного дозвона. 

- Мы рекомендуем отключить VoIP для всех собраний всех затронутых пользователей до 31 июля 2021 г., чтобы предотвратить фрагментацию звука среди участников, присоединявшихся по VoIP или через сторонних аудиоконплентов. 

    - Чтобы отключить VoIP для всех собраний с определенным организатором, задайте для параметра AllowIPAudio политики конференций пользователя ложное с помощью Set-CsConferencingPolicy-Set-CsConferencingPolicy. Дополнительные сведения [см. в поле Set-CsConferencingPolicy.](https://docs.microsoft.com/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
 
- Если организация не отключает VoIP для всех собраний, пользователям рекомендуется отключить интеграцию Skype для бизнеса Online со сторонним ACP и повторно захотеть перенести собрания, чтобы удалить сторонние данные для телефонного набора ACP, чтобы предотвратить фрагментацию звука.

    - Чтобы отключить интеграцию аудиоконференции сторонних сторон с заданным организатором, используйте cmdlet [Remove-CsUserAcp.](https://docs.microsoft.com/powershell/module/skype/remove-csuseracp?view=skype-ps) 

    - Инструкции по автоматическому перенаписи собраний см. в руководстве "Как вручную выполнить перенос собраний для пользователя?". в [настройках службы переноса собраний (MMS).](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)

### <a name="path-3-stop-using-dial-in-conferencing-altogether"></a>Путь #3: полностью прекратить использование телефонной и телефонной телефонной телефонии

Организации, которые решили полностью прекратить использование услуг аудиоконференций (ни от корпорации Майкрософт, ни от сторонного аудиоконферента), не могут полностью полагаться на VoIP для поддержки аудиоканла собрания Skype для бизнеса. 

Таким организациям потребуется отключить для пользователей стороннее аудиоконференцию и автоматически перепланировать собрания с помощью службы переноса собраний, чтобы удалить данные для этих услуг. 

- Чтобы отключить интеграцию аудиоконференции сторонних Remove-CsUserAcp для данного организатора. Дополнительные сведения [см. в remove-CsUserAcp.](https://docs.microsoft.com/powershell/module/skype/remove-csuseracp?view=skype-ps) 

- Чтобы автоматически перенести собрания пользователей после отключения интеграции со сторонним поставщиком аудиоконференций, см. "Как вручную выполнить перенос собраний для пользователя?". в [настройках службы переноса собраний (MMS).](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md) 

**Сводка.** 

- Организации, которые решили полностью прекратить использование аудиоконференций до 31 июля 2021 г., не будут влиять.

- Чтобы отключить интеграцию аудиоконференции сторонних Remove-CsUserAcp для данного организатора. Дополнительные сведения [см. в remove-CsUserAcp.](https://docs.microsoft.com/powershell/module/skype/remove-csuseracp?view=skype-ps) 

- Чтобы автоматически перенести собрания пользователей после отключения интеграции со сторонними поставщиками аудиоконференций, см. "Как вручную выполнить перенос собраний для пользователя?". в [настройках службы переноса собраний (MMS).](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)
