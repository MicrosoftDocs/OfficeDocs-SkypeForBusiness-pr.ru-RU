# <a name="end-of-life-program-for-the-integration-of-skype-for-business-with-third-party-audio-conferencing-providers"></a>Программа окончания жизненного процесса для интеграции Skype для бизнеса со сторонними поставщиками услуг аудиоконференации 

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

Корпорация Майкрософт объявила о начале жизненного процесса интеграции Skype для бизнеса со сторонними поставщиками аудиоконференций. 

Окончание жизненного года завершится 31 июля 2021 г. После завершения работы программы интеграция Skype для бизнеса сторонними поставщиками аудиоконференций перестанет работать, и в эту дату (31 июля 2021 г.) будут наблюдаться следующие изменения:

- Участники, которые пытаются присоединиться к Skype для бизнеса собрания с помощью номеров телефонного подключения, предоставленных стороной службой ACP, больше не будут подключаться к Skype для бизнеса собранию.
 
- Пользователи, включенные в сторонную службу ACP, больше не будут автоматически включать свои данные для телефонного входа в Skype для бизнеса приглашения на собрание.

В рамках объявления о начале окончания жизненного года вступает в силу следующее изменение, которое будет действовать до завершения жизненного года: 

- Клиенты без Skype для бизнеса, настроенных для использования сторонной службы ACP, не смогут настроить их использование сторонними службами ACP.

- Существующие клиенты с Skype для бизнеса, настроенными для использования сторонной службы ACP, по-прежнему смогут добавлять новых пользователей в течение срока службы. Обратите внимание, что мы не рекомендуем настраивать дополнительных Skype для бизнеса пользователей для использования сторонной службы ACP, так как изменения, внося в нее 31 июля 2021 г., также будут применяться к ним.

## <a name="preparing-for-this-change"></a>Подготовка к этому изменению

Чтобы подготовиться к этому изменению, мы рекомендуем затронутым организациям уведомлять пользователей об этом запланированном обновлении до 31 июля 2021 г. 

После 31 июля 2021 г. пользователи могут продолжать использовать Skype для бизнеса собрания по сети; однако организациям потребуется включить для своих пользователей аудиоконференцию, предоставленную корпорацией Майкрософт, если для них требуется аудиоконференцию с Skype для бизнеса или Microsoft Teams. Дополнительные информацию о аудиоконференциях (Майкрософт) см. в этой [записи.](https://www.microsoft.com/microsoft-teams/audio-conferencing) 

В зависимости от желаемого конечного состояния организации существует три пути, по которым можно следовать:

- Перейти на аудиоконференцию Майкрософт. 
- Продолжайте использовать стороннее аудиоконференцию отдельно. 
- Полностью отменить использованиеконференции с использованием телефонной телефонии.

### <a name="path-1-migrate-to-microsoft-audio-conferencing"></a>Путь #1: переход на аудиоконференцию Майкрософт   

Организации, которые решили перейти на аудиоконференцию Майкрософт и завершить миграцию до 31 июля 2021 г., не будут влиять на работу службы в течение этой даты или после нее. При переходе на аудиоконференцию Майкрософт в организацию будут внесены следующие изменения: 

- Счета за службу будут выставлены всем Microsoft 365 или Office 365 службам. 

- Если приобретена стандартная подписка, платные затраты на телефонноеном подключении включаются в ежемесячную стоимость подписки "на пользователя". 

- Каждой организации и ее пользователям будет предоставлен новый набор номеров телефонов для телефонного звонка. Чтобы узнать о географическом охвате службы аудиоконференц Майкрософт, см. доступность страны и региона для аудиоконференций и планов [звонков.](/microsoftteams/country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans)
 
- Собрания, которые уже были запланированы пользователями, включенными с помощью стороннее аудиоконференцию, будут автоматически перепланируются таким образом, чтобы включить в них сведения для телефонного входа в аудиоконференцию Майкрософт.
 
- У каждого собрания будут динамические ИД конференции, то есть у каждого собрания будет свой выделенный ИД конференции. Динамические ИД конференц-залов обеспечивают улучшенную безопасность и улучшенные возможности для собраний с задней таблицей.

- Все использование службы подется условиям использования служб аудиоконференций. 

Перейти на аудиоконференцию Майкрософт очень просто, и это можно сделать всего за несколько шагов после приобретения лицензий для службы. Чтобы узнать, как перейти на аудиоконференцию Майкрософт, см.:

- [Попробуйте или приобретйте аудиоконференцию в Microsoft 365 или Office 365](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md)
 
**Сводка.**

- Организации, которые переходят на аудиоконференцию Майкрософт и завершают миграцию до 31 июля 2021 г., не увидят влияния на службу в течение этой даты или после нее.

- Дополнительные информацию о переходе на аудиоконференцию Майкрософт см. в этой Microsoft 365 или [Office 365.](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md) 

### <a name="path-2-continue-to-separately-use-a-third-party-audio-conferencing-provider"></a>Путь #2: продолжайте использовать стороннее аудиоконференцию отдельно

Организации, которые решили продолжать использовать сторонний аудиофайл в течение и после 31 июля 2021 г., будут оказывать влияние на службу, так как сторонние сведения для телефонного вступлкого собрания больше не будут использоваться для Skype для бизнеса аудиосвязи. 

Чтобы предотвратить фрагментацию звука в собраниях Skype для бизнеса с помощью VoIP, а других — через сторонний ACP, организациям рекомендуется отключить использование VoIP на собраниях пользователей. Таким образом, всем участникам потребуется присоединиться к звуковой части собрания с помощью сторонного аудиосистемы, и все остальные рабочие нагрузки собрания (например, чат или совместный доступ к экрану) могут по-прежнему поддерживаться через Skype для бизнеса. 

- Чтобы отключить VoIP для всех собраний с определенным организатором, задайте для параметра AllowIPAudio его политики конференций Set-CsConferencingPolicy ложь. Дополнительные сведения [см. в поле Set-CsConferencingPolicy.](/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
 
В плане и с 31 июля 2021 г. данные для телефонного номера сторонних ACP больше не будут автоматически включаться в Skype для бизнеса приглашения на собрания. Пользователям потребуется вручную добавить данные для телефонного Skype для бизнеса приглашений на собрания, если они хотят и дальше добавлять эти сведения в свои собрания. 

Обратите внимание, что на 31 июля 2021 г. существующие собрания пользователей не будут автоматически запланированы на удаление сторонних данных для телефонного номера ACP. Организациям, которые решили включить VoIP для собраний своих пользователей, следует отключить интеграцию сторонних аудиоконференций для своих пользователей и повторно запланирования собраний с помощью службы переноса собраний, чтобы удалить сторонние данные для телефонного входа в аудиоконференцию из существующих собраний и предотвратить фрагментацию звука в уже запланированных собраниях. 

- Чтобы отключить интеграцию стороннее аудиоконференцию с заданным организатором, используйте Remove-CsUserAcp аудиоконференцию. Дополнительные сведения см. [в этой теме.](/powershell/module/skype/remove-csuseracp?view=skype-ps) 

- Чтобы автоматически перенести собрания пользователей после отключения интеграции со сторонним поставщиком аудиоконференций, см. "Как выполнить перенос собраний вручную для пользователя?". в [настройках службы переноса собраний (MMS).](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md) 

**Сводка.**

- Организации, которые решили продолжать использовать сторонний ACP до и после 31 июля 2021 г., будут затронуты тем, что сторонний ACP не сможет присоединиться к собранию Skype для бизнеса, а новые собрания не будут включать сторонние данные для телефонного связи. 

- Мы рекомендуем отключить VoIP для всех собраний всех затронутых пользователей до 31 июля 2021 г., чтобы предотвратить разблокацию звука между участниками, которые присоединяются к собранию через VoIP и через сторонний ACP. 

    - Чтобы отключить VoIP для всех собраний с определенным организатором, задайте для параметра AllowIPAudio политики конференций пользователя ложный с помощью Set-CsConferencingPolicy видео. Дополнительные сведения [см. в поле Set-CsConferencingPolicy.](/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)
 
- Если организация не отключает VoIP для всех собраний, пользователям рекомендуется отключить интеграцию Skype для бизнеса Online со сторонним ACP и повторно захотеть собрания, чтобы удалить сторонние сведения для телефонного набора ACP, чтобы предотвратить фрагментацию звука.

    - Чтобы отключить интеграцию стороннее аудиоконференцию с заданным организатором, воспользуйтесь cmdlet [Remove-CsUserAcp.](/powershell/module/skype/remove-csuseracp?view=skype-ps) 

    - Инструкции по автоматическому перенаписи собраний см. в вопросе "Как выполнить перенос собраний вручную для пользователя?". в [настройках службы переноса собраний (MMS).](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)

### <a name="path-3-stop-using-dial-in-conferencing-altogether"></a>Путь #3: полностью прекратить использование телефонной телефонной телефонии

Организации, которые решили полностью прекратить использование функции аудиоконференции с телефонным Skype для бизнеса майкрософт и сторонних ACP, могут полностью полагаться на VoIP. 

Таким организациям потребуется отключить для своих пользователей использование сторонних поставщиков аудиоконференций и автоматически перепланировать собрания с помощью службы переноса собраний, чтобы удалить данные для этих услуг. 

- Чтобы отключить интеграцию стороннее аудиоконференцию с заданным организатором, используйте Remove-CsUserAcp аудиоконференцию. Дополнительные сведения см. [в этой теме.](/powershell/module/skype/remove-csuseracp?view=skype-ps) 

- Чтобы автоматически перенести собрания пользователей после отключения интеграции со сторонним поставщиком аудиоконференций, см. "Как выполнить перенос собраний вручную для пользователя?". в [настройках службы переноса собраний (MMS).](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md) 

**Сводка.** 

- На организации, решающие полностью прекратить использование аудиоконференций до 31 июля 2021 г., это не повлияет.

- Чтобы отключить интеграцию стороннее аудиоконференцию с заданным организатором, используйте Remove-CsUserAcp аудиоконференцию. Дополнительные сведения см. [в этой теме.](/powershell/module/skype/remove-csuseracp?view=skype-ps) 

- Чтобы автоматически перенести собрания пользователей после отключения интеграции со сторонними поставщиками аудиоконференций, см. "Как выполнить перенос собраний вручную для пользователя?". в [настройках службы переноса собраний (MMS).](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)
