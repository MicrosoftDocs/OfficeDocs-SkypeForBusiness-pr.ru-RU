# <a name="using-the-meeting-migration-service-mms"></a>С помощью службы миграции собрания (MMS)

Служба миграции собрания (MMS) — служба, которая обновления существующих собраний пользователя в следующих случаях:

- При миграции пользователя из локального в облако (между Скайп для бизнеса в Интернет и TeamsOnly).
- Когда администратор изменяет параметры аудиоконференций пользователя 
- При обновлении активных пользователей группам только или пользователя в TeamsUpgradePolicy режим SfBwithTeamsCollabAndMeetings
- При использовании PowerShell 


По умолчанию MMS автоматически запускается в каждом из этих случаев, несмотря на то, что администраторы могут отключить его на уровне клиента. Кроме того администраторы могут использовать командлет PowerShell, чтобы вручную запустить миграции собрания для данного пользователя.


**Ограничения**: собрания нельзя использовать службы миграции, если выполняется одно из следующих условий:

- Почтовый ящик пользователя размещается в локальную систему Exchange.
- Пользователь является переносе из облака в Скайп для Business Server локальных.

В этих случаях конечных пользователей можно использовать [Средство миграции собрания](https://www.microsoft.com/en-us/download/details.aspx?id=51659) для миграции свои собственные собрания вместо этого.

## <a name="how-mms-works"></a>Как работает MMS

При MMS запускается для определенного пользователя, запроса на миграцию для этого пользователя помещаются в очередь. Чтобы избежать любой гонки, помещенных в очередь запрос намеренно не обрабатывается пока не менее 90 минут решаются. После MMS обрабатывает запрос, выполняет следующие задачи:

1. Будет выполняться поиск этого пользователя почтового ящика для всех существующих собраний упорядоченные по этому пользователю и по расписанию в будущем.
2. Основываясь на данные, находящиеся в почтовом ящике пользователя, он обновляет или планирует нового собрания в группы или Скайп для бизнеса в Интернет для этого пользователя, в зависимости от конкретного сценария.
3. В сообщении электронной почты он заменяет блока собрание по сети в сведения о собрании.
4. Он отправляет обновленные версии этого собрания всем получателям собрания от имени Организатор собрания. Приглашенные участники собрания будет получать обновления собрания с помощью координат обновленное собрание в своей электронной почте. 

    ![Блокировка собрания, обновляемого MMS](../images/210a03ee-30c1-46f3-808f-4c2ebdaa3ea1.png)

С момента запуска MMS обычно занимает около 2 часа до миграции пользователя собрания. Тем не менее если у пользователя есть большое число собраний, он может занимать значительно больше времени. Если MMS возникает ошибка, перенос одного или нескольких собраний для пользователя, он будет периодически пытаться до 9 раз по длительности 24 часа.

**Заметки**.

- При переносе собрания MMS заменяет все сведения в блоке информации о собрании по сети. Если же пользователь изменил этот блок, внесенные им изменения будут перезаписаны. Все остальные данные в сведениях о собрании, за исключением этого блока, останутся нетронутыми.
- Переносятся только те собрания Skype для бизнеса или Microsoft Teams, которые были запланированы с помощью кнопки **Добавить собрание Skype** в Outlook в Интернете или с помощью надстройки "Собрание Skype" для Outlook. Если пользователь копирует и вставляет данные Скайп собрания по сети из одного собрания на новое собрание, эти новые собрания не будут обновляться, так как нет собрания в исходной службы.
- После запуска MMS не сохраняются собрания контент, который был создан или подключенного к собранию (доски, опросы и т. д.). Если вашей организации присоединен содержимого собраний заранее, содержимое потребуется заново после запуска MMS.
- Ссылки на общие заметки собрания в элементе календаря и в собрании Skype также будут перезаписаны. Обратите внимание, что фактический собрания, хранящиеся в OneNote будет по-прежнему быть существует; Это только ссылки на общие заметки, который будет перезаписан.
- Собрания с числом участников больше 250 (включая организатора) не будут перенесены.
- Некоторые знаки ЮНИКОДА в тексте приглашения могут быть обновлены неправильно к одному из следующих специальных символов: ï, ¿, ½,.

## <a name="triggering-mms-for-a-user"></a>Запуск MMS для пользователя

В этом разделе описывается, что происходит при запуске MMS в каждом из следующих случаев:

- При миграции пользователя из локального в облако
- Когда администратор изменяет параметры аудиоконференций пользователя 
- Когда пользователя в TeamsUpgradePolicy режим TeamsOnly или SfBWithTeamsCollabAndMeetings (с помощью Powershell или портала администрирования группы)
- При использовании командлета PowerShell Start-CsExMeetingMigration

### <a name="updating-meetings-when-you-move-an-on-premises-user-to-the-cloud"></a>Обновление собрания при перемещении локального пользователя в облако

Это наиболее распространенные сценарии, где MMS позволяет создать плавный переход для пользователей. Без собрания миграции существующих собраний, организованных пользователем в Скайп для Business Server локальных больше не будет работать после пользователь перейдет Интернет-версия. Таким образом, при использовании средств администрирования в локальной (либо `Move-CsUser` или панель управления администратора) перемещение пользователя с облаком, существующих собраний автоматически перемещаются в облако следующим образом:

- Если `MoveToTeams` переключитесь в `Move-CsUser` указано, собрания перенесены непосредственно для групп и пользователь будет находиться в режиме TeamsOnly. Этот параметр требуется Скайп для Business Server CU8 или более поздней версии. Эти пользователи по-прежнему могут присоединиться к любой Скайп для собраний, они могут принимать участие в, с помощью Скайп для клиента Business или приложение Скайп собрания.
- В противном случае собрания перенесены Скайп для бизнеса в Интернет.

В любом случае если пользователь находится в лицензию на аудиоконференции до перемещаются в облако собраний будут создаваться с координаты-связи. Если вы намереваетесь для этого пользователя для использования аудиоконференций перемещение пользователя из локального в облако, рекомендуется сначала назначение участие в конференции, перед перемещением пользователя, чтобы происходит только 1 собрания миграции.


### <a name="updating-meetings-when-a-users-audio-conferencing-settings-change"></a>Обновление собраний при изменении параметров аудиоконференции для пользователя

В следующих случаях MMS приведут к обновлению существующих Скайп для бизнеса и группами Майкрософт собраний для добавления, удаления или изменения координаты-связи:

- Если назначение и удаление лицензии службы Microsoft аудиоконференций к пользователю и пользователь не включен для поставщика аудиоконференций сторонних производителей.
- При изменении поставщика аудиоконференций пользователя из любого другого поставщика в корпорацию Майкрософт, предоставляемые пользователя назначена лицензия Microsoft аудиоконференций. Дополнительные сведения можно [Назначить Microsoft в качестве поставщика аудиоконференций](https://docs.microsoft.com/en-us/skypeforbusiness/audio-conferencing-in-office-365/assign-microsoft-as-the-audio-conferencing-provider). Также Обратите внимание на то, что поддержка сторонних поставщиков аудиоконференций [ACP] запланировано окончание срока действия на 1 апреля 2019, как [ранее объявили об изменении](https://docs.microsoft.com/skypeforbusiness/legal-and-regulatory/end-of-integration-with-3rd-party-providers).
- При включении или отключении аудиоконференций для пользователя.
- Если изменить или сбросить идентификатор конференции для пользователя, настроены на использование общедоступных собраний.
- При переключении пользователя на новый мост аудиоконференции.
- Когда номер телефона, из bridge аудиоконференций не присвоено значение. Это сложные сценарии, необходимо выполнить дополнительные действия. Дополнительные сведения можно [Изменить номера телефона в звукового конференц-канала](https://docs.microsoft.com/en-us/MicrosoftTeams/change-the-phone-numbers-on-your-audio-conferencing-bridge).

Не все изменения в параметрах аудиоконференции для пользователя приводят к запуску MMS. В частности, MMS не будет обновлять собрания в двух следующих случаях:

- При изменении SIP-адреса организатора собрания (его имени пользователя SIP или домена SIP)
- При изменении вашей организации в собрание по URL-адрес `Update-CsTenantMeetingUrl` команды.


### <a name="updating-meetings-when-assigning-teamsupgradepolicy"></a>Обновление собрания при назначении TeamsUpgradePolicy

По умолчанию миграции на собрания автоматически запускается, когда пользователь получает экземпляр `TeamsUpgradePolicy` с `mode=TeamsOnly` или `mode= SfBWithTeamsCollabAndMeetings`. Если вы не хотите перенос собрания при предоставлении любой из этих режимов, а затем укажите `MigrateMeetingsToTeams $false` в `Grant-CsTeamsUpgradePolicy` (при использовании с PowerShell) или снимите флажок, чтобы перенос собрания при задании режима совместной работы пользователя (если используется портала администрирования группы).

Также Обратите внимание на следующее:

- Миграция на собрания вызывается только при предоставлении `TeamsUpgradePolicy` для определенного пользователя. При предоставлении `TeamsUpgradePolicy` с `mode=TeamsOnly` или `mode=SfBWithTeamsCollabAndMeetings` на *всем клиента* , не вызываются миграции на собрания.
- Пользователь может предоставляться только TeamsOnly режиме, если пользователь, размещенный через Интернет. Необходимо переместить пользователей, которые являются, размещенные локально с помощью `Move-CsUser` как описано выше.
- Предоставление режим, отличный от TeamsOnly или SfBWithTeamsCollabAndMeetings не преобразует существующих собраний группы Скайп для деловых встреч.

### <a name="trigger-meeting-migration-manually-via-powershell-cmdlet"></a>Триггер миграции собрания вручную с помощью командлета PowerShell

В дополнение к миграции автоматического собрания "Администраторы" может вручную активировать собрания миграции для пользователя с помощью командлета `Start-CsExMeetingMigration`. Этот командлет ставит в очередь запроса на миграцию для указанного пользователя.  В дополнение к обязательным `Identity` для параметра принимает два дополнительных параметра, `SourceMeetingType` и `TargetMeetingType`, позволяют указать переносу собраний:

**TargetMeetingType:**

- С помощью `TargetMeetingType Current` указывает, что Скайп для собраний Business остаются Скайп для деловых встреч и собраний группы остаются собраний группы. Тем не менее аудиоконференций координирует могут быть изменены и любой локальной Скайп для собраний Business будут перенесены в Скайп для бизнеса в Интернет. Это значение по умолчанию для TargetMeetingType.
- С помощью `TargetMeetingType Teams` указывает, что любые существующие собрания необходимо перенести в группы, требуются независимо от того, собрание размещается ли в Скайп для работы через Интернет или локальной и независимо от того, требуется ли какие-либо обновления аудиоконференций. 

**SourceMeetingType:**
- С помощью `SourceMeetingType SfB` указывает, только Скайп для собраний Business (ли в локальной или online) следует обновить.
- С помощью `SourceMeetingType Teams` указывает, что обновление только собраний группы должно.
- С помощью `SourceMeetingType All` указывает, что оба Skyep для деловых встреч и собраний группы должны обновляться. Это значение по умолчанию для SourceMeetingType.
    

В приведенном ниже примере показано, как начать собрание миграции для пользователя ashaw@contoso.com, чтобы все собрания перенесены на группы:

```
Start-CsExMeetingMigration -Identity ashaw@contoso.com -TargetMeetingType Teams
```



## <a name="managing-mms"></a>Управление MMS

С помощью Windows PowerShell, можно проверить состояние текущих миграций, вручную запуск переноса собрания и полностью отключить миграции. 

### <a name="check-the-status-of-meeting-migrations"></a>Проверка состояния приглашений на собрания миграции

Вы используете `Get-CsMeetingMigrationStatus` командлет, чтобы проверить состояние миграции приглашений на собрания. Ниже приведено несколько примеров.

- Для получения сводного состояние всех миграций MMS, выполните следующую команду, содержащей табличные представления всех состояний миграции:

    ```
    Get-CsMeetingMigrationStatus -SummaryOnly

    State      UserCount
    ------     ---------
    Pending      21
    InProgress    6
    Failed            2 
    Succeeded   131
    ```
- Чтобы получить полные сведения обо всех миграций в пределах определенного периода времени, используйте `StartTime` и `EndTime` параметры. Например следующую команду вернется полные сведения на всех миграций, которые произошли с 1 октября 2018 8 октября 2018.

    ```
    Get-CsMeetingMigrationStatus -StartTime "10/1/2018" -EndTime "10/8/2018"
    ```
- Чтобы проверить состояние миграции для определенного пользователя, используйте `Identity` параметр. Например, следующая команда получит состояние переноса для пользователя ashaw@contoso.com:

    ```
    Get-CsMeetingMigrationStatus -Identity ashaw@contoso.com
    ```
Если отображаются все миграций, которые не были выполнены действия для устранения этих проблем как можно скорее, так как пользователи не сможет получить-связи на собрания, упорядоченные по этих пользователей до их устранения. Если `Get-CsMeetingMigrationStatus` отображает все миграции в состоянии сбоя, выполните следующие действия:
 
1. Определите пользователей, которых затронул сбой. Для получения списка пользователей, которых затронул сбой, и возникших ошибок выполните следующую команду:

    ```
    Get-CsMeetingMigrationStatus| Where {$_.State -eq "Failed"}| Format-Table Identity, LastMessage
    ```
2. Для каждого пользователя затронутых запустите средство миграции собрания вручную перенести свои собрания.

3. Если с помощью средства переноса собраний перенос выполнить все равно не удается, есть два варианта действий.

    - Попросить пользователей создать собрания Skype заново.
    - [Связаться со службой поддержки](https://go.microsoft.com/fwlink/p/?LinkID=518322).


### <a name="enabling-and-disabling-mms"></a>Включение и отключение MMS

MMS включен по умолчанию для всей организации, но его можно отключить следующим образом:

- Отключите полностью для клиента. 
- Отключение только изменения, связанные с аудиоконференций. В этом случае MMS будет выполняться при миграции пользователя из локального в облако или при предоставлении в режиме TeamsOnly или SfBWithTeamsCollabAndMeetings `TeamsUpgradePolicy`.

Например необходимо вручную перенести все собрания или временно отключить MMS время внесения существенные изменения в параметры аудиоконференций для вашей организации

Чтобы просмотреть, если MMS включается для вашей организации, выполните следующую команду. Если включена MMS `MeetingMigrationEnabled` — это параметр `$true`.
```
Get-CsTenantMigrationConfiguration
```
Чтобы включить или отключить MMS полностью, используйте `Set-CsTenantMigrationConfiguration` команды. Например чтобы отключить MMS, выполните следующую команду:

```
Set-CsTenantMigrationConfiguration -MeetingMigrationEnabled $false
```
В случае MMS включено в организации, необходимо проверить, если он включен для обновления звукового конференц-связи, проверьте значение `AutomaticallyMigrateUserMeetings` параметр в выходные данные `Get-CsOnlineDialInConferencingTenantSettings`. Чтобы включить или отключить MMS звукового конференц-связи, используйте `Set-CsOnlineDialInConferencingTenantSettings`. Например чтобы отключить MMS звукового конференц-связи, выполните следующую команду:

```
Set-CsOnlineDialInConferencingTenantSettings  -AutomaticallyMigrateUserMeetings $false
```
