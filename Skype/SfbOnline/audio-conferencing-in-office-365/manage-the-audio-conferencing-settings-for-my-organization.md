# <a name="manage-the-audio-conferencing-settings-for-my-organization-in-skype-for-business-online"></a>Управление параметрами аудиоконференций организации в Skype для бизнеса Online

> [!NOTE]
> Для получения информации о том, как управлять этими параметрами с помощью Microsoft Teams, см. [Управление параметрами аудиоконференций для организаций в Microsoft Teams](/MicrosoftTeams/manage-the-audio-conferencing-settings-for-my-organization-in-teams).

Вероятно, вам будет удобнее просматривать все параметры аудиоконференций Skype для бизнеса в одном месте.


## <a name="assign-an-audio-conferencing-license"></a>Назначение лицензии на аудиоконференцию

> [!NOTE]
> Назначать лицензии с помощью **центра администрирования Skype для бизнеса** нельзя. Для этой цели необходимо использовать центр администрирования Office 365. См. [Назначение лицензий Skype для бизнеса](../skype-for-business-and-microsoft-teams-add-on-licensing/assign-skype-for-business-and-microsoft-teams-licenses.md).

 **Чтобы назначить пользователю лицензию, выполните следующие действия**

1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. На расположенной слева панели навигации **центра администрирования Office 365** перейдите к элементу **Пользователи** > ** Активные пользователи**, а затем выберите одного или нескольких пользователей из списка доступных пользователей.

    > [!NOTE]
    > При одновременном назначении лицензий нескольким пользователям, количество которых не превышает 20, можно воспользоваться раскрывающимся меню **Выберите представление**, а затем выбрать один из имеющихся вариантов или создать свое собственное представление. Нажмите на кнопку **Редактировать**, после чего, дважды нажмите на кнопку **Далее**, выделите лицензию и нажмите **Отправить**. Для назначения лицензии нескольким пользователям можно также воспользоваться Windows Powershell. Для ознакомления с инструкцией и примерами скриптов PowerShell см. [Назначение лицензий в Skype для бизнеса](../skype-for-business-and-microsoft-teams-add-on-licensing/assign-skype-for-business-and-microsoft-teams-licenses.md).

3. На панели «Действия» в секции **Лицензии на продукты** нажмите на кнопку **Изменить**.

4. На странице **Лицензии на продукты** включите функцию **Аудиконференции**, а затем нажмите **Сохранить**. Дополнительную информацию о лицензировании можно найти в [Лицензирование надстроек Skype для бизнеса](../skype-for-business-and-microsoft-teams-add-on-licensing/skype-for-business-and-microsoft-teams-add-on-licensing.md).

> [!NOTE]
> После назначения лицензии Майкрософт может не отображаться в списке, как поставщик аудиоконференций. В этом случае, выйдите из центра администрирования Office 365 или нажмите на клавиши CTRL+F5, чтобы обновить окно браузера.

## <a name="enable-or-disable-emails-sent-to-audio-conferencing-users"></a>Включение или отключение функции отправки электронной почты участникам аудиоконференции

![sfb-logo-30x30.png](../images/sfb-logo-30x30.png) **Использование центра администрирования Skype для бизнеса**

1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. Перейдите к **центру администрирования Office 365** > **Skype для бизнеса** и на панели навигации слева нажмите на кнопку **Аудиоконференция**.

3. На странице **Параметры моста Microsoft** поставьте или снимите флажок **Автоматически отправлять сообщения пользователям при изменении их параметров аудиоконференций**.

4. Нажмите на кнопку **Сохранить**.

    Также пользователю можно отправить письмо с параметрами аудиоконференции; для этого перейдите к пользовательским свойствам аудиоконференции и нажмите  ** Отправить сведения о конференции по электронной почте**. В приглашении на собрание содержится идентификатор конференции и телефонный номер, используемый по умолчанию для проведения аудиоконференции, но ПИН-код при этом не указывается.

    См. [Отправка пользователям сообщений с информацией об их параметрах аудиоконференций](send-an-email-to-a-user-with-their-dial-in-information.md).

> [!NOTE]
> [!INCLUDE [updating-admin-interfaces](../includes/updating-admin-interfaces.md)]

**Использование Windows PowerShell**

- С помощью Windows PowerShell можно запустить:

  ```
  Set-CsOnlineDialInConferencingTenantSettings -AutomaticallySendEmailsToUsers $true|$false
  ```

    Для управления другими параметрами организации, включая параметры электронной почты, можно использовать [Set-CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285).

## <a name="change-the-senders-contact-information-in-email-messages-sent-to-users"></a>Изменение контактной информации отправителя в сообщениях, отправляемых пользователям

В сообщение, которое отправляется пользователям автоматически, можно внести изменения, включив в него фактический адрес электронной почты и отобразив имя отправителя в составе контактной информации. По умолчанию отправителем сообщений является Office 365, но адрес электронной почты и отображаемое имя можно изменить с помощью Windows PowerShell и командлета [Set-CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285). Чтобы изменить адрес электронной почты, с которого сообщения отправляются пользователям, необходимо выполнить следующие действия.

- Введите адрес электронной почты в параметр _SendEmailFromAddress_.

- Введите отображаемое имя в параметр _SendEmailFromDisplayName_.

- Присвойте параметру _SendEmailOverride_ значение _True_.

Если вы хотите изменить адрес электронной почты, вам нужно убедиться, что политика входящих сообщений организации позволяет получать письма, отправленные с настроенного адреса электронной почты.

```
Set-CsOnlineDialInConferencingTenantSetting -SendEmailOverride $true -SendEmailFromAddress amos.marble@contoso.com -SendEmailFromDisplayName "Amos Marble"
```

Чтобы изменить адрес электронной почты, нужно убедиться в том, что применяемая организацией политика входящих сообщений позволяет принимать сообщения, отправленные с настраиваемого адреса электронной почты.

Для управления другими параметрами организации, включая электронную почту, можно использовать командлет [Set-CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285).

См. [Сообщения, автоматически отправляемые пользователям при изменении их параметров аудиоконференций](emails-sent-to-users-when-their-settings-change.md).

## <a name="reset-the-meeting-conference-id"></a>Сброс идентификатора конференции

1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. Перейдите к **центру администрирования Office 365** > **Skype для бизнеса**.

3. На панели навигации слева в **центре администрирования Skype для бизнеса** перейдите к секции **Аудиоконференции** и на панели действий в области**Идентификатор конференции** нажмите на кнопку **Сброс**.

4. В отобразившемся окне**Сбросить идентификатор конференции?** нажмите на кнопку **Да**. При этом будет автоматически создан идентификатор конференции и, если функция отправки сообщений включена, пользователям будут отправлены сообщения с новым идентификатором конференции. По умолчанию эта настройка остается включенной.

    > [!IMPORTANT]
    >  После создания нового идентификатора конференции старый идентификатор не может использоваться для выполнения вызовов. Чтобы убедиться в том, что новый идентификатор конференции добавлен в приглашения, пользователей следует известить о необходимости изменения имеющихся приглашений на собрание. Для обновления информации о запланированных собраниях пользователи могут воспользоваться средством переноса данных о собраниях в Skype для бизнеса. Чтобы узнать, как загрузить, установить и запустить средством переноса данных о собраниях в Skype для бизнеса, см.: [Средство переноса данных о собраниях для Skype для бизнеса и Lync](https://support.office.com/article/2b525fe6-ed0f-4331-b533-c31546fcf4d4), [Skype для бизнеса Online, средство переноса данных о собраниях (64-разрядная версия)](https://go.microsoft.com/fwlink/?LinkID=626047)и [Skype для бизнеса Online, средство переноса данных о собраниях (32-разрядная версия)](https://www.microsoft.com/en-us/download/details.aspx?id=54079).

См. [Сброс идентификатора конференции для пользователя](reset-a-conference-id-for-a-user.md).

## <a name="reset-a-conference-organizers-pin"></a>Сброс ПИН-кода организатора конференции

Каждому собранию, запланированному пользователем, назначается уникальный идентификатор конференции. Идентификатор конференции создается и назначается пользователю автоматически, тем не менее, иногда возникают ситуации, когда пользователь не желает использовать его, когда в качестве идентификатора требуется использовать определенное число, либо кода пользователь теряет или не может вспомнить идентификатор конференции. В этом случае можно войти в центр администрирования Skype для бизнеса и Windows PowerShell, чтобы просмотреть, изменить или сбросить идентификатор конференции пользователя.


1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. Перейдите к **центру администрирования Office 365** > **Skype для бизнеса** и на панели навигации слева нажмите на кнопку **Аудиоконференция**.

3. Нажмите на кнопку **Пользователи** и выберите пользователя, для которого требуется сбросить ПИН-код.

4. На панели «Действия», в секции **ПИН-код**, нажмите на кнопку **Сброс**.

Пользователи получают сообщение с ПИН-кодом при подключении к аудиоконференции или при сбросе ПИН-кода. Однако, если автоматическая рассылка сообщений выключена, отправка письма о сбросе ПИН-кода не производится, и ПИН-код следует отправить пользователю вручную. ПИН-код показывается только один раз, после его сброса. После этого ПИН-код перестает отображаться в свойствах пользователя, и вместо него появляются символы *****.

См.[Сброс ПИН-кода для аудиоконференции](reset-the-audio-conferencing-pin.md).

## <a name="choosing-audio-conferencing-bridge-settings"></a>Выбор параметров моста аудиоконференций

**Настройка порядка проведения собрания при присоединении к нему пользователей, совершающих вызов**


1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. Перейдите к **центру администрирования Office 365** > **Skype для бизнеса**.

3. В **центре администрирования Skype для бизнеса**, на панели навигации слева, перейдите к секции **Аудиоконференции** > **Параметры моста Microsoft**.

4. В секции ** Порядок присоединения к собранию**  выполните следующие действия.

  - **Включить уведомления о входе на собрание и выходе из него** Этот параметр задается по умолчанию. Если снять этот флажок, то при использовании настроек по умолчанию пользователи, которые уже присоединились к собранию, не получат уведомления о присоединении или выходе из него других участников.

    Этот параметр можно настраивать отдельно для каждого собрания, к которому пользователь присоединяется с помощью приложения Skype для бизнеса, изменяя при этом параметр **Объявлять о входе или выходе пользователей** в меню **Параметры** собраний Skype.

  - **Попросить абонентов записывать свое имя перед присоединением к собранию.** Этот параметр задается по умолчанию. Если снять этот флажок, то предложение записать свое имя перед присоединением к собранию пользователям, совершающим вызов, не поступает.

5. После внесения требуемых изменений нажмите на кнопку **Сохранить**.
    
См. [Изменение параметров моста аудиоконференций](/MicrosoftTeams/change-the-settings-for-an-audio-conferencing-bridge).
  
 **Настройка длины ПИН-кода для собраний**

1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. Перейдите к **центру администрирования Office 365** > **Skype для бизнеса**.

3. В **центре администрирования Skype для бизнеса**, на панели навигации слева, перейдите к секции **Аудиоконференции** > **Параметры моста Microsoft**.

4. В секции **Безопасность** введите количество цифр, которое следует использовать для ПИН-кода, в списке **Длина ПИН-код**, поле чего нажмите **Сохранить**.

    ПИН-код должен включать в себя от 4 до 12 цифр. Длина ПИН-кода, заданная по умолчанию, составляет 5 символов.
    
См. [Изменение параметров моста аудиоконференций](/MicrosoftTeams/change-the-settings-for-an-audio-conferencing-bridge).
  
 **Включение или отключение функции отправки сообщений электронной почты участникам аудиоконференции**

1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. Перейдите к **центру администрирования Office 365** > **Skype для бизнеса** и на панели навигации слева нажмите на кнопку **Аудиоконференция**.

3. На странице **Параметры моста Microsoft** поставьте или снимите флажок **Автоматически отправлять сообщения пользователям при изменении их параметров аудиоконференций**.

4. Нажмите на кнопку **Сохранить**.

    Также пользователю можно отправить сообщение с параметрами аудиоконференции; для этого необходимо перейти к пользовательским свойствам аудиоконференции и нажать **Отправить сведения о конференции по электронной почте**.

    В этом случае, пользователям отправляются сообщения, содержащие идентификатор конференции и номер телефона, но не включающие в себя ПИН-код.

    См. [Отправка пользователям сообщений с информацией об их параметрах аудиоконференций](send-an-email-to-a-user-with-their-dial-in-information.md).

## <a name="see-and-set-the-primary-default-and-secondary-alternate-languages-on-an-audio-conferencing-bridge"></a>Просмотр и выбор основного (используемого по умолчанию) и дополнительного (альтернативного) языка для моста аудиоконференции


1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. Перейдите к **центру администрирования Office 365** > **Skype для бизнеса**.

3. В **центре администрирования Skype для бизнеса**, на расположенной слева панели навигации, перейдите к секции **Аудиоконференции**, а затем нажмите на кнопку **Мост Microsoft**.

4. На панели «Действия» Выберите из списка номер телефона и нажмите на кнопку **Настроить языки**, а затем, на странице **Настройка языков**, выберите список **Основной язык** для просмотра всех поддерживаемых языков.

     Также основной и дополнительный языки можно выбрать из списка поддерживаемых языков при выборе Microsoft в качестве поставщика аудиоконференций. Языки отображаются для участников конференции в том порядке, в котором они выбираются из списков.

См. [Настройка языков автосекретаря для аудиоконференций](set-auto-attendant-languages-for-audio-conferencing.md).

## <a name="see-audio-conferencing-dial-in-numbers"></a>Просмотр номеров для конференц-связи с телефонным подключением

1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. Перейдите к **центру администрирования Office 365** > **Skype для бизнеса**.

3. В **центре администрирования Skype для бизнеса**, на панели навигации слева, перейдите к секции **Аудиоконференции** > **Мост Microsoft**. Здесь можно выполнить следующие действия:

  - просмотреть телефонные номера, установленные с помощью Office 365 для использования при проведении аудиоконференции;

  - просмотреть местоположение, а также основные и дополнительные языки, которые будут использоваться автосекретарем аудиоконференции;

  - выбрать используемые по умолчанию номера телефонов, которые будут предоставлены пользователям, участвующим в аудиоконференции. Тем не менее, при изменении номера телефона, используемого по умолчанию для моста аудиоконференции, телефонные номера по умолчанию, предоставленные существующим пользователям, не изменяются.

Чтобы изменить телефонный номер по умолчанию, указанный для пользователя, можно перейти к секции **Аудиоконференции** > **Пользователи** и, выбрав свойства пользователя, задать новый номер из списка доступных номеров организации.

См. [Просмотр списка номеров для аудиоконференций](see-a-list-of-audio-conferencing-numbers.md).

## <a name="see-a-list-of-users-that-are-enabled"></a>Просмотр списка пользователей, участвующих в аудиоконференции.

1. Войдите в Office 365 со своей учебной или рабочей учетной записью.

2. Перейдите к **центру администрирования Office 365** > **Skype для бизнеса**.

3. В **центре администрирования Skype для бизнеса**, на расположенной слева панели навигации, перейдите к секции **Аудиоконференции**>, а затем — к секции **Пользователи**.

См. [Просмотр списка пользователей, подключенных к аудиоконференции](see-a-list-of-users-that-are-enabled-for-audio-conferencing.md).

## <a name="want-to-know-how-to-manage-with-windows-powershell"></a>Хотите знать, как управлять параметрами с помощью Windows PowerShell?

Некоторыми параметрами можно управлять на уровне организации с помощью Windows PowerShell. Это упрощает применение заданных параметров ко всем пользователям.

Для получения дополнительной информации о каждом из командлетов см. [Командлеты Skype для бизнеса Online](https://go.microsoft.com/fwlink/?LinkId=627324).

Здесь приводятся параметры, настраиваемые на уровне организации.

- **Настройка уведомлений о входе и выходе** Значением по умолчанию является _$true_.
  ```
  Set-CsOnlineDialInConferencingTenantSettings -EnableEntryExitNotifications $true|$false
  ```

- **Настройка записи имени** Значением по умолчанию является _$true_.
  ```
  Set-CsOnlineDialInConferencingTenantSettings -EnableNameRecording $true|false
  ```

- **Настройка длины ПИН-кода** Значение по умолчанию равно 5.
  ```
  Set-CsOnlineDialInConferencingTenantSettings -PinLength 7
  ```

- **Настройка с телефона только для собраний с телефонным подключением** Значением по умолчанию является _$false_.
  ```
  Set-CsOnlineDialInConferencingTenantSettings -AllowPSTNOnlyMeetingsByDefault $true|$false
  ```

- **Настройки отправки сообщений пользователям** Значением по умолчанию является _$true_.
  ```
  Set-CsOnlineDialInConferencingTenantSettings -AutomaticallySendEmailsToUsers $true|$false
  ```

- **Настройка отправки сообщений с другой учетной записи** Значением по умолчанию является _$false_.
  ```
  Set-CsOnlineDialInConferencingTenantSettings -SendEmailFromOverride $true|$false
  ```

- **Настройка адреса отправителя в сообщениях, отправляемых пользователям** Значением по умолчанию является _$null_.
  ```
  Set-CsOnlineDialInConferencingTenantSettings -SendEmailFromAddress
  ```

- **Настройка имени, отображаемого в сообщениях, отправляемых пользователям** Значением по умолчанию является _$null_.
  ```
  Set-CsOnlineDialInConferencingTenantSettings -SendEmailFromDisplayName
  ```
