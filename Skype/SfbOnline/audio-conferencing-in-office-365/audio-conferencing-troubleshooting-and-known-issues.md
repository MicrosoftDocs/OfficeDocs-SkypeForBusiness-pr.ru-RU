# <a name="audio-conferencing-troubleshooting-and-known-issues"></a>Известные проблемы аудиоконференций и устранение неполадок

 **[] **Эта статья предназначена для пользователей Skype для бизнеса и Microsoft Teams, использующих Майкрософт в качестве поставщика аудиоконференций. Приведенные здесь сведения не применимы для клиентов, использующих сторонних поставщиков аудиоконференций.****
  
## <a name="troubleshooting-and-known-issues"></a>Известные проблемы и устранение неполадок

Аудиоконференция с использованием Майкрософт как поставщика услуг аудиоконференций имеет ряд текущих проблем, которые отслеживаются, исследуются и будут устранены при обновлении этой функции в будущих сборках Office 365.
  
Сейчас вы можете использовать предоставленные в этом разделе сведения для справки, когда производите поиск и устранение проблем, которые могут возникнуть при настройке и работе пользователей вашей организации с функцией аудиоконференции для приложений Skype для бизнеса или Microsoft Teams.
  
### <a name="microsoft-teams-app"></a>Приложение Microsoft Teams

|**Проблема**|**Поведение или признаки**|**Известный обходной путь**|**Дата обнаружения**|
|:-----|:-----|:-----|:-----|
|Абоненты ТСОП с одинаковым исходящим номером отображаются в списке участников собрания как один и тот же пользователь.  <br/> |Если к собранию присоединяются несколько абонентов ТСОП, идентификаторы которых маскируются как одно число, они будут отображаться в списке участников собрания как один и тот же пользователь.  <br/> |Обходной путь отсутствует.  <br/> |25.09.2017  <br/> |
|Панель сведений о собрании открывается не сразу.  <br/> |Панель сведений о собрании может не отображаться в клиенте Microsoft Teams, когда пользователи пытаются найти телефонные номера моста конференц-связи или идентификатор конференц связи.  <br/> |Просмотрите сведения о собрании или календарь Outlook, чтобы найти телефонные номера моста конференц-связи или идентификатор конференц связи.  <br/> |25.09.2017  <br/> |
|В приглашениях на собрания из надстройки Outlook в координатах ТСОП отображаются лишние символы для языков, отличных от en-US.  <br/> |При планировании личных собраний с помощью надстройки Outlook для Microsoft Teams на компьютере, где установлены языки, отличные от en-US, координаты ТСОП могут содержать лишние символы.  <br/> |Обходной путь отсутствует.  <br/> |25.09.2017  <br/> |
|Для исходящего звонка требуется набрать 5 или больше цифр.  <br/> |Пользователи, пытающиеся совершить исходящий звонок, должны вводить 5 или больше цифр, даже если доступно правило нормализации абонентской группы для нормализации короткого набора согласно рекомендации E.164.  <br/> |Для обратных звонков следует использовать формат полного номера DID или формат локального номера, а не внутренний добавочный номер.  <br/> |25.09.2017  <br/> |
|Элемент управления исходящего набора открывается не сразу.  <br/> |Элемент управления исходящего набора может не отображаться на панели сведений о собрании.  <br/> |Обходной путь отсутствует.  <br/> |25.09.2017  <br/> |
|Статический идентификатор конференции не поддерживается для собраний Microsoft Teams.  <br/> |Если администратор переопределяет заданный по умолчанию динамический идентификатор конференции на статический идентификатор, этот параметр не действует для собраний Microsoft Teams. См. статью [Использование динамических идентификаторов конференц-связи с телефонным подключением в вашей организации](using-audio-conferencing-dynamic-ids-in-your-organization.md).  <br/> |Обходной путь отсутствует.  <br/> |25.09.2017  <br/> |
|Координаты собрания, проводимого по телефонной сети общего пользования (ТСОП), недоступны локальным пользователям в Skype для бизнеса  <br/> |Если пользователь является локальным пользователем Skype для бизнеса, которому назначены лицензии на Skype для бизнеса Online, аудиоконференции и Microsoft Teams, все собрания, запланированные с использованием Microsoft Teams, не будут включать координаты собрания ТСОП. <br/> |Обходной путь отсутствует.  <br/> |01.02.2018  <br/> |
   
### <a name="skype-for-business-app"></a>Приложение Skype для бизнеса

|**Ошибка**|**Поведение или признаки**|**Известный обходной путь**|**Дата обнаружения**|

|Уведомления о входе и выходе включаются при начале собрания, но затем вскоре выключаются.  <br/> |По умолчанию уведомления о входе и выходе отключаются для собраний, когда участники присоединяются из приложений Skype для бизнеса и через телефонный вызов. Можно включить оповещения в **Параметрах собраний Skype** в приложении Skype для бизнеса. Для собраний, где все участники присоединяются к собранию через телефонное подключение, уведомления о входе и выходе включены по умолчанию, так как список участников недоступен любому участнику. Когда собрание началось только для участников с телефонным подключением, включаются уведомления о входе и выходе, но когда какой-либо участник присоединится через приложение Skype для бизнеса, уведомления отключаются. Затем их можно снова включить в разделе **Параметры собраний Skype** в приложении Skype для бизнеса. <br/> |Обходной путь отсутствует.  <br/> |30.08.2017  <br/> |

См. [Включение и отключение отправки писем при смене настроек аудиоконференций](enable-or-disable-sending-emails-when-their-settings-change.md).  <br/> **Примечание.** Для повторной отправки пользователю ПИН-кода аудиоконференции необходимо сбросить ПИН-код. Это также можно выполнить в разделе **Аудиоконференция** центра администрирования Skype для бизнеса или с помощью PowerShell.          |Обходной путь отсутствует.  <br/> |30.08.2017  <br/> |
|Отображение телефонных вызовов для подключения к аудиоконференции в отчетах по использованию может занять до 24 часов.  <br/> |Мы работаем над улучшением этого в следующих обновлениях службы.  <br/> |Обходной путь отсутствует.  <br/> |30.08.2017  <br/> |
|Когда абонент звонит на мост конференций после того, как собрание заблокировал пользователь Skype для бизнеса, в приложении Skype для бизнеса не появляется уведомление о том, что пользователь находится в зале ожидания.  <br/> |В настоящее время это запланированное поведение, но мы учитываем ваше мнение и намерены поддерживать эту возможность в следующих обновлениях службы.  <br/> |Обходной путь отсутствует.  <br/> |30.08.2017  <br/> |
