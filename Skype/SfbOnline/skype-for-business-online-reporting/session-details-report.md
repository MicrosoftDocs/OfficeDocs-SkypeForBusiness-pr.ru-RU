# <a name="session-details-report"></a>Отчет со сведениями о сеансах

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

На **панели мониторинга** Отчеты представлен обзор действий Microsoft 365 или Office 365 продуктов в организации. С ее помощью можно детализированно придать отдельным продуктам отчеты, чтобы получить более подробную информацию о действиях в каждом продукте. Например, с помощью отчета Skype для бизнеса **"Сведения** о сеансе" можно получить подробные сведения о звонках отдельных пользователей.
  
Дополнительные [сведения о доступных отчетах](https://support.office.com/article/0d6dfb17-8582-4172-a9a9-aed798150263) можно найти в обзоре отчетов.
  
В этом и других отчетах Skype для бизнеса подробные сведения о действиях, включая сведения о сеансе в вашей организации. Эти сведения полезны при планировании, планировании и принятии других деловых решений в организации, а также при настройке кредитов на [связь.](/microsoftteams/what-are-communications-credits)
  
> [!NOTE]
> Вы можете видеть все отчеты Skype для бизнеса при входе в Центр администрирования Microsoft 365 с учетной записью администратора. 
  
## <a name="how-to-get-to-the-skype-for-business-session-details-report"></a>Как получить отчет о Skype для бизнеса сеансе

1. Перейдите в центр администрирования > **отчетов**
    
2. В **меню слева** выберите пункт Отчеты и нажмите кнопку **Использование.**
    
3. В списке Выберите **отчет** выберите Skype для бизнеса **сведения о сеансе**.
    
    > [!TIP]
    > Если этот отчет не указан в списке, перейдите в Skype для бизнеса **Отчеты**  >  **о** сеансе.  >   
  
    > [!IMPORTANT]
    > В зависимости от Microsoft 365 или Office 365 подписки могут быть показаны не все продукты и отчеты, показанные здесь. 
  
## <a name="interpret-the-skype-for-business-session-details-report"></a>Интерпретация Skype для бизнеса сеанса

Чтобы просмотреть сведения о сеансе Skype для бизнеса, можно просмотреть все отображаемые столбцы.
  
Отчет имеет следующий вид.
  
![Skype для бизнеса Панель мониторинга "Сведения о сеансе".](../images/3d87ab39-6eaa-46b5-b5f9-7f54dc987ae0.png)
  
***
![Номер 1](../images/sfbcallout1.png)<br/>**Поиск пользователя по псевдониму** позволяет найти одного пользователя и отображает все сведения о сеансе пользователя в таблице ниже. 
***
![Номер 2](../images/sfbcallout2.png)<br/>**Ввод с даты** позволяет уложить дату начала. В календаре можно выбрать дату или ввести ее вручную. Это поле должно быть заполнено.
***
![Номер 3](../images/sfbcallout3.png)<br/>**Ввести в дату** можно уложить дату окончания. В календаре можно выбрать дату или ввести ее вручную. Если дата окончания не за установлена, по умолчанию она находится в 30 днях от даты начала.
***
![Номер 4](../images/sfbcallout4.png)<br/>В таблице показаны сведения по всем сеансам для каждого пользователя. Здесь показаны все пользователи, Skype для бизнеса им назначены данные о сеансе. При необходимости можно добавлять и удалять столбцы таблицы. <br/><br/>Таблица состоит из следующих столбцов для каждого сеанса:
*    **Идентификатор диалогов —** это идентификатор уникального идентификатора сеанса SIP.
*    **В описании** типов мультимедиа описывается, является ли сеанс конференцией или сеансом P2P, а также тип используемых мультимедиа (Аудио/Видео/Общий доступ к приложениям).
*    **Время начала** — это время начала сеанса.
*    **Время окончания** — это время окончания сеанса.
*    **From URI** — это URI пользователя или службы, которые инициировали сеанс. Может быть пустым, если пользователь инициировал сеанс с телефона ЗВОНКОВ по ННР.
*    **URI** — это URI пользователя или службы, которые были целевыми при инициации сеанса. В случае конференции это URI организатора. Может быть пустым, если целью сеанса был номер телефона ЗВОНКОВ.
*    **В клиентской** версии указывается агент пользователя и версия клиента, использованная пользователем или службой, которая инициировала сеанс.
*    **В клиентскую** версию указывается агент пользователя и версия клиента, используемая пользователем или службой, которая была целью инициации сеанса.
*    **URL-адрес конференции** — это URL-адрес SIP конференции, если сеанс был конференцией. У всех пользователей, которые будут проходить один и тот же конференц-вызов, будет один и тот же URL-адрес конференции. 
*    **From Tel number** — это телефонный номер, который, если применимо, был целевым объектом сеанса. Чтобы защитить конфиденциальность пользователей, последние цифры телефонного номера могут быть заменены на "x".
*    **To Tel number** — это телефонный номер, который, если применимо, был целевым объектом сеанса. Чтобы защитить конфиденциальность пользователей, последние цифры телефонного номера могут быть заменены на "x".
*    **Из ИД конечной точки** — это уникальный GUID конечной точки, используемой пользователем From. Используется для определения того, передает ли пользователь несколько сеансов с одной конечной точки. Может быть пустым, если пользователь использует телефон ЗВОНКОВ или сеанс инициировался из службы.
*    **To Endpoint Id** — это уникальный GUID конечной точки, используемой пользователем To. Используется для определения того, передает ли пользователь несколько сеансов с одной конечной точки. Может быть пустым, если пользователь использует телефон ЗВОНКОВ, если сеанс инициировался из службы или не удалось установить сеанс.
*    **Экземпляр conf** — это уникальный GUID для экземпляра конференции, использующий URL-адрес конференции. У повторяющихся собраний будет один и тот же URL-адрес конференции, но у каждого экземпляра собрания будет один и тот же экземпляр конференции.
*    **От имени URI** — это URI делегатора, от имени которого устанавливается сеанс. <br/> **URI — это URI** пользователя, который ссылался на установку сеанса.
*    **Код ответа** — это код ответа SIP для сеанса, который показывает, успешно ли он установлен.

Для каждого сеанса существует под таблица с различными данными в зависимости от сценария. Ниже перечислены вкладки, доступные в под таблице для пользователей и служб "От" и "К".
*    На вкладке СЕАНС показаны данные об компьютерах и операционных системах.
*    На вкладке MEDIALINES показаны сведения о сетевом под подключениях и сведения об устройстве.
*    На вкладке AUDIOSTREAMS показаны данные о производительности сети для аудиопотоков, участвующих в сеансе.
*    На вкладке AUDIOCLIENTEVENTS показаны данные об обнаруженных клиентах проблемах, которые влияют на качество звука.
*    На вкладке AUDIOSIGNALS показаны данные об обработке звуковых сигналов для сеанса.
*    Вкладка APPSHARINGSTREAMS отображает данные о производительности сети, связанные с совместным использованием приложений или потоков общего доступа к рабочему столу, которые участвуют в сеансе.
*    На вкладке ВИДЕОКЛИЕНТА показаны данные об обнаруженных клиентах проблемах, которые влияют на видео.
*    Вкладка ВИДЕОПОТОКИ отображает данные о производительности сети о видеопотоках, участвующих в сеансе.
*    На вкладке TRACEROUTES показаны сетевые переходы, собранные с помощью traceroute во время сеанса. Фактический путь к мультимедиа, используемый для сеанса, может отличаться, и эти данные доступны только при наличии звука в сеансе.
*    На вкладке FEEDBACKREPORTS показаны все данные опроса, предоставленные пользователями в ходе сеанса.
***
![Номер 5](../images/sfbcallout5.png)<br/>Чтобы создать представление, объединяющее все данные из одного или нескольких столбцов, перетащите столбец в раздел **Чтобы сгруппировать по определенному столбцу, перетащите сюда заголовок столбца**. 
***
![Номер 6](../images/sfbcallout6.png)<br/>Вы также можете экспортировать данные отчета в CSV-файл Excel, нажав кнопку **Экспорт в Excel**. <br/><br/> При этом данные всех пользователей будут экспортированы в формат, позволяющий сортировать и фильтровать их для дальнейшего анализа. Если у вас менее 2000 пользователей, вы можете сортировать и фильтровать значения в самой таблице отчета. Если пользователей больше 2000, для фильтрации и сортировки потребуется экспортировать данные.
