# <a name="session-details-report"></a>Отчет со сведениями о сеансах

Office 365 **отчеты** панели мониторинга показывает обзор активности для продуктов Office 365 в вашей организации. Он позволяет работать отдельных продуктов уровня отчетами, чтобы предоставить более детальные сведения о действия в рамках каждого продукта. К примеру **Скайп для сведений о сеансе Business** отчет используется для ознакомиться с подробными сведениями о работе вызова отдельных пользователей.
  
Извлечь [Обзор отчеты](https://support.office.com/article/0d6dfb17-8582-4172-a9a9-aed798150263) для нескольких отчетов, которые доступны.
  
Этот отчет, а также другие Скайп для бизнес-отчеты предоставляют сведения о работой, включая сведения о сеансе внутри организации. Эти сведения используются при вы исследование, планирование и выполнение других business принятии решений для вашей организации, а также для настройки [связи кредитов](/microsoftteams/what-are-communications-credits).
  
> [!NOTE]
> Вы можете просмотреть все отчеты Skype для бизнеса, если войдете в Центр администрирования Office 365 в качестве администратора. 
  
## <a name="how-to-get-to-the-skype-for-business-session-details-report"></a>Как получить Скайп для отчета о сеансе бизнеса

1. Перейдите в **Центр администрирования Office 365** > **отчетов**
    
2. Выберите **отчеты** в левом меню и нажмите кнопку **об использовании**.
    
3. В списке **Выберите отчет**нажмите кнопку **Скайп для сведений о сеансе бизнеса**.
    
    > [!TIP]
    > Если не отображается в этом отчете из списка, перейдите к **Скайп по центру администрирования Business** > **отчетов** > **сведения о сеансе**. 
  
    > [!IMPORTANT]
    > Для некоторых подписок на Office 365 могут быть доступны не все продукты и отчеты об активности, показанные в этой статье. 
  
## <a name="interpret-the-skype-for-business-session-details-report"></a>Интерпретация Скайп для отчета о сеансе бизнеса

Представления можно получить в Скайп пользователя для сведений о сеансе бизнеса, посмотрев каждого из столбцов, которые отображаются.
  
Отчет имеет следующий вид.
  
![Скайп для панели мониторинга отчета о сеансе бизнеса.](../images/3d87ab39-6eaa-46b5-b5f9-7f54dc987ae0.png)
  
***
![Номер 1](../images/sfbcallout1.png)<br/>**Поиск пользователя с псевдонимом** позволяет выполнять поиск для одного пользователя и отображает все сведения о сеансе пользователя в следующей таблице. 
***
![Номер 2](../images/sfbcallout2.png)<br/>Позволяет **Ввод из Дата и время** , помещенные в начальную дату. Можно использовать календарь для выбора даты или вручную введите дату. В этом поле должен быть указан.
***
![Номер 3](../images/sfbcallout3.png)<br/>Позволяет **ВВОД, чтобы дата и время** , помещенные в конечную дату. Можно использовать календарь для выбора даты или вручную введите дату. Если выбрано значение Нет даты окончания, по умолчанию — 30 дней от даты начала.
***
![Номер 4](../images/sfbcallout4.png)<br/>В таблице показаны разбивкой все сведения о сеансе на одного пользователя. Это показывает все пользователи, имеющие Скайп для бизнеса, назначенные им и сведений о сеансе. При необходимости можно добавлять и удалять столбцы таблицы. <br/><br/>Для каждого сеанса в таблице есть следующие столбцы:
*    **Идентификатор диалогового окна** — это идентификатор для уникальный идентификатор сеанса SIP.
*    **Описание типов мультимедиа** описание того, является ли сеанс конференц-связи или сеанса P2P и используемого носителя (совместное использование аудио или видео или приложение).
*    **Время начала** — это время начала сеанса.
*    **Время окончания** — это время завершения этого сеанса.
*    **URI из** — это URI пользователя или службы, который инициировал сеанс. Может быть пустым, если пользователь инициировал сеанс с телефона ТСОП.
*    **URI для** — это URI пользователя или служба, которая была целью инициации сеанса. В случае конференции это организатора URI. Может быть пустым, если целевой сеанс был номер телефона ТСОП.
*    **Версии клиента** вы найдете агента пользователя и версия клиента пользователя или службой, начавшего сеанс.
*    **Для версии клиента** вы найдете агента пользователя и версии клиента, используемая пользователем или служба, которая была целью инициации сеанса.
*    **URL-адрес конференции** — это URL-адрес SIP для конференции, если сеанс конференц-связи. Все пользователи в этой же конференции будут иметь же URL-адрес конференции. 
*    **Номер из Tel** является телефонный номер, который был целью сеанса, если это возможно. Последний цифры номера телефона может быть заменено на «x», чтобы обеспечить конфиденциальность пользователей.
*    **Чтобы Tel** равен телефонный номер, который был целью сеанса, если это возможно. Последний цифры номера телефона может быть заменено на «x», чтобы обеспечить конфиденциальность пользователей.
*    **Из идентификатор конечной точки** — уникальный идентификатор GUID пользователя от конечной точки. Используется для определения, является ли пользователь соединен нескольких сеансов от одной конечной точкой. Если пользователь является с помощью телефона ТСОП или сеанс из службы, может быть пустым.
*    **Чтобы идентификатор конечной точки** — уникальный идентификатор GUID пользователя в конечной точки. Используется для определения, является ли пользователь соединен нескольких сеансов от одной конечной точкой. Может быть пустым, если пользователь использует телефона ТСОП, если сеанс из службы или сеанса не удалось установить.
*    **Экземпляр conf** — уникальный идентификатор GUID для экземпляра конференции, используя URL-адрес конференции. Повторяющиеся собрания будут иметь же URL-адрес конференции, но каждый экземпляр собрания будут иметь различие Conf экземпляра.
*    **URI для имени д** — это URI сотрудника, от имени которого установлен сеанс. <br/> **Ссылка с URI** — это URI пользователя, который ссылался при установке сеанса.
*    **Код ответа** — это код ответа SIP для установления сеанса, которое указывает, если сеанс успешно установлено.

Для каждого сеанса остается sub таблицы с различными данными в зависимости от сценария. В следующем списке перечислены вкладки, доступные в таблице sub для отправителя и для пользователей или служб.
*    Вкладка СЕАНСА отображаются данные о компьютерах и операционные системы.
*    MEDIALINES вкладке отображаются сведения о подключения к сети и сведения об устройстве.
*    Вкладка AUDIOSTREAMS показывает данные о производительности сети о звуковых потоков сеанса.
*    Вкладка AUDIOCLIENTEVENTS демонстрирует данные вопросам обнаруженных клиента, влияющее на качество звука.
*    Вкладка AUDIOSIGNALS отображаются данные о обработки для сеанса звукового сигнала.
*    Вкладка APPSHARINGSTREAMS показывает данные о производительности сети о общий доступ к приложениям и рабочему столу общего доступа потоков сеанса.
*    Вкладка VIDEOCLIENTEVENTS демонстрирует данные вопросам обнаруженных клиента, влияющее качества видео.
*    Вкладка "VIDEOSTREAMS" отображаются данные о производительности сети об потоковое видео участвующего в сеансе.
*    Вкладка ОПРЕДЕЛЯТЬ демонстрирует переходов между сети, собранных с помощью traceroute во время сеанса. Действительные пути, используемом для сеанса могут отличаться от указанных и эти данные доступна только при наличии звука в сеанс.
*    Вкладка FEEDBACKREPORTS демонстрирует любой конец данных опроса вызовов, предоставляемые пользователями в сеансе.
***
![Номер 5](../images/sfbcallout5.png)<br/>Чтобы создать представление, объединяющее все данные из одного или нескольких столбцов, перетащите столбец в раздел **Чтобы сгруппировать по определенному столбцу, перетащите сюда заголовок столбца**. 
***
![Номер 6](../images/sfbcallout6.png)<br/>Вы также можете экспортировать данные отчета в CSV-файл Excel, нажав кнопку **Экспорт в Excel**. <br/><br/> При этом данные всех пользователей будут экспортированы в формат, позволяющий сортировать и фильтровать их для дальнейшего анализа. Если у вас менее 2000 пользователей, вы можете сортировать и фильтровать значения в самой таблице отчета. Если пользователей больше 2000, для фильтрации и сортировки потребуется экспортировать данные.
