# <a name="retaining-large-files-attached-to-a-skype-for-business-meeting"></a>Сохранение больших файлов, вложенных в Skype для бизнеса собрания

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

Вы можете вложить файлы в Skype для бизнеса собрания, которое участники могут открывать и скачивать. Файлы, присоединенные к собраниям Skype для бизнеса, хранятся в почтовых ящиках любого участника, почтовый ящик которого помещен на хранение для судебного разбирательства, к ним применена политика хранения Microsoft 365 или Office 365 или удержание, связанное с делом обнаружения электронных данных на портале соответствия требованиям Microsoft Purview. Это содержимое сохраняется в папках **"** Элементы с возможностью восстановления" участников в почтовых ящиках.
  
Файлы, которые &amp; хранятся в почтовых ящиках на удержании, индексируются и поэтому могут выполнять поиск при выполнении поиска контента в Центре соответствия требованиям безопасности при поиске почтового ящика участника. Однако вложенные файлы размером более 30 МБ разбиваются на два или более файлов меньшего размера и сохраняются в виде сжатых (.zip) файлов. Содержимое  *этих*  небольших файлов не индексируются для поиска и могут не возвращаться в поиске контента. Однако метаданные *этих*  файлов (например, имя файла и автор) индексируются для поиска и могут быть возвращены в поиске контента.
  
> [!IMPORTANT]
> Параметры MaxReceiveSize и MaxSendSize для почтового ящика Exchange Online могут повлиять на возможность хранения больших файлов Skype для бизнеса собраний. Параметры по умолчанию для MaxReceiveSize и MaxSendSize — 36 МБ и 35 МБ соответственно. Однако эти параметры по умолчанию слишком малы для хранения любого файла из собрания Skype для бизнеса размером более 30 МБ. Это связано с Exchange Online кодировка Base64 вложений сообщений и других двоичных данных. При кодировании сообщения его размер увеличивается примерно на 33 %. Таким образом, чтобы обеспечить хранение больших файлов из Skype для бизнеса собраний, рекомендуется увеличить значение maxReceiveSize и MaxSendSize до 39 МБ (что примерно на 33 % больше ранее описанного ограничения в 30 МБ) для пользователей, помещенных на удержание. В противном случае большой файл, присоединенный к собранию Skype для бизнеса, может не сохраниться. Дополнительные сведения об использовании команд **Set-Mailbox -MaxReceiveSize** и **Set-Mailbox -MaxSendSize** в Exchange Online PowerShell см. в разделе [Set-Mailbox](/powershell/module/exchange/mailboxes/Set-Mailbox).
  
В почтовых ящиках, которые не находятся на удержании, не будут сохранены данные собрания. Например, на собрании с тремя пользователями, где почтовые ящики двух участников помечены для хранения, данные собрания сохраняются в почтовых ящиках этих двух участников, но не в почтовый ящик третьего участника, почтовый ящик которого не находится на удержании.
