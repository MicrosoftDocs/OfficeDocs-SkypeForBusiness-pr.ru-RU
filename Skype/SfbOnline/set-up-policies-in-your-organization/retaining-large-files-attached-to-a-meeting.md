# <a name="retaining-large-files-attached-to-a-skype-for-business-meeting"></a>Сохранение больших файлов, вложенных в собрание Skype для бизнеса

Вы можете вложите файлы в собрание Skype для бизнеса, чтобы участники могли открыть и скачать их. Файлы, вложенные в собрания Skype для бизнеса, сохраняются в почтовых ящиках всех участников, чьи почтовые ящики помещены на хранение для судебного разбирательства, к которым применена политика хранения Microsoft 365 или Office 365 или помещены на удержание, связанное с делом eDiscovery в Центре соответствия требованиям Microsoft 365. Это содержимое будет сохранено в папках **"Элементы** с возможностью восстановления" участников в их почтовых ящиках.
  
Файлы, которые сохраняются в почтовых ящиках на удержании, индексироваться, поэтому при выполнении поиска контента в Центре безопасности и соответствия требованиям можно искать при поиске в почтовом ящике &amp; участника. Однако вложенные файлы размером более 30 МБ разделены на два или более файлов меньшего размера и сохраняются в сжатом ZIP-файле. Содержимое  *таких*  небольших файлов не индексется для поиска и может не возвращаться при поиске контента. Однако метаданные *этих*  файлов (например, имя файла и автор) индексироваться для поиска и могут возвращаться при поиске контента.
  
> [!IMPORTANT]
> Параметры MaxReceiveSize и MaxSendSize для почтового ящика Exchange Online могут повлиять на возможность сохранения больших файлов из собраний Skype для бизнеса. По умолчанию параметры MaxReceiveSize и MaxSendSize имеют соответственно 36 и 35 МБ. Однако эти параметры по умолчанию слишком маленькие, чтобы сохранить файл собрания Skype для бизнеса размером более 30 МБ. Дело в том, что в Exchange Online используется кодирная кодивка для вложений в сообщения и других двоичных данных. При кодировании сообщения его размер увеличивается примерно на 33 %. Поэтому для сохранения больших файлов из собраний Skype для бизнеса рекомендуем увеличить значение максимального размера MaxReceiveSize и MaxSendSize до 39 МБ (что приблизительно на 33 % больше ограничения в 30 МБ, о котором говорилось ранее) для пользователей, помещенных на удержание. В противном случае большой файл, вложенный в собрание Skype для бизнеса, может не сохраниться. Дополнительные сведения об использовании команд **Set-Mailbox -MaxReceiveSize** и **Set-Mailbox -MaxSendSize** в Exchange Online PowerShell см. в [set-mailbox.](https://docs.microsoft.com/powershell/module/exchange/mailboxes/Set-Mailbox)
  
В почтовых ящиках, которые не находятся на удержании, не будут сохранены данные собрания. Например, при собрании с тремя участниками, когда почтовые ящики двух участников помечены для хранения, данные собрания сохраняются в почтовых ящиках этих двух участников, а не в почтовых ящиках третьего участника, почтовый ящик которого не находится на удержании.
