# Управление настройками аудиоконференции для организации

Вам будет проще видеть все настройки аудиоконференции для Skype для бизнеса и Microsoft Teams в одном месте. 
  
## Назначение лицензии на аудиоконференцию

> [!NOTE]
> Невозможно назначать лицензии с помощью **Центр администрирования Skype для бизнеса**. Следует использовать центр администрирования Office 365. См. статью [Назначение лицензий Skype для бизнеса и Microsoft Teams](../skype-for-business-and-microsoft-teams-add-on-licensing/assign-skype-for-business-and-microsoft-teams-licenses.md). 
  
 **Порядок назначения лицензии для пользователя**
  
1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. В **Центре администрирования Office 365** на панели навигации слева перейдите в раздел **Пользователи** > **Активные пользователи**, затем выберите пользователя или пользователей из списка доступных.
    
    > [!NOTE]
    > Если число пользователей, которым вы назначаете лицензии, не превышает 20, вы можете использовать раскрывающийся список **Выбор представления**, а затем выбрать один из вариантов или создать собственное представление. Затем выберите пункт **Правка**, дважды нажмите кнопку **Далее**, выберите лицензию и нажмите кнопку **Отправить**. Лицензии нескольким пользователям также можно назначить с помощью Windows Powershell. Инструкции и примеры сценариев PowerShell см. [Назначение лицензий Skype для бизнеса и Microsoft Teams](../skype-for-business-and-microsoft-teams-add-on-licensing/assign-skype-for-business-and-microsoft-teams-licenses.md). 
  
3. На панели действий в разделе **Лицензии на продукты** нажмите **Изменить**. 
    
4. На странице **Лицензии на продукты** установите флажок **Аудиоконференция** и нажмите кнопку **Сохранить**. Дополнительные сведения о лицензировании см. в статье [Лицензирование надстроек Skype для бизнеса и Microsoft Teams](../skype-for-business-and-microsoft-teams-add-on-licensing/skype-for-business-and-microsoft-teams-add-on-licensing.md).
    
> [!NOTE]
> После назначения лицензии Майкрософт может сначала не отображаться в списке как поставщик услуг аудиоконференций. В таком случае выйдите из центра администрирования Office 365 или нажмите сочетание клавиш CTRL+F5 для обновления окна браузера. 
  
## Назначение пользователю идентификатора конференции

Идентификатор конференции автоматически назначается пользователю, если для него настроена аудиоконференция через Майкрософт как поставщика услуг аудиоконференций. Идентификатор конференции может быть статическим или динамическим. Он отправляется в приглашении на собрание при планировании собрания. 
  
Статические идентификаторы используются, если пользователи в организации не хотят запоминать случайные числа. В этом случае они могут выбрать определенное число, которое проще запомнить. Если используются динамические идентификаторы конференции, каждому собранию, планируемому пользователем, будет присвоен уникальный идентификатор конференции. Если требуется назначить динамические, а не статические идентификаторы конференции, [Аудиоконференции с помощью динамических идентификаторы в вашей организации](using-audio-conferencing-dynamic-ids-in-your-organization.md).
  
Центр администрирования Skype для бизнеса нельзя использовать для назначения пользователю идентификатора конференции, однако для этого можно использовать командлет Windows PowerShell.
  
Чтобы задать идентификатор конференции для пользователя, выполните следующее:
  
```
Set-CsOnlineDialInConferencingUser -Identity "Amos Marble"  -ConferenceId 8271964 
```

> [!IMPORTANT]
> Идентификатор конференции должен содержать 7 цифр, и его нельзя изменить в центре администрирования Skype для бизнеса или с помощью Windows PowerShell. 
  
Для получения дополнительных сведений о командлете см. статью [Set-CsOnlineDialInConferencingUser](https://go.microsoft.com/fwlink/?LinkId=617688 ).
  
> [!IMPORTANT]
>  После создания нового идентификатора конференции абоненты не смогут использовать старый идентификатор конференции. Следует уведомить пользователей о том, что они должны перепланировать существующие приглашения на собрания, указав в них новый идентификатор конференции. Пользователи могут использовать инструмент переноса собраний Skype для бизнеса, чтобы обновить существующие собрания. Инструкции по загрузке, установке и запуску средства обновления для собраний Skype для бизнеса см. в следующей статье:> [Средство обновления для собраний Skype для бизнеса и Lync](https://support.office.com/article/2b525fe6-ed0f-4331-b533-c31546fcf4d4)> [Skype для бизнеса Online, инструмент переноса собраний (64-разрядная версия)](http://go.microsoft.com/fwlink/?LinkID=626047)> [Skype для бизнеса Online, инструмент переноса собраний (32-разрядная версия).](https://go.microsoft.com/fwlink/?LinkID=626046)
  
См. статью [Просмотр, изменение и сброс идентификатора конференц-связи, назначенного пользователю](see-change-and-reset-a-conference-id-assigned-to-a-user.md).
  
## Изменение поставщика аудиоконференций с Microsoft на стороннего поставщика

1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в раздел **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. В **Центр администрирования Skype для бизнеса**на панели навигации слева выберите **Аудиоконференция** > **Пользователи** и укажите пользователя, для которого следует изменить поставщика услуг аудиоконференций.
    
4. На панели действий нажмите **Изменить**. 
    
5. На странице **Свойства** в разделе **Имя поставщика** выберите поставщика услуг аудиоконференций для пользователя.
    
    > [!NOTE]
    > Если выбрано несколько пользователей, можно выбрать только поставщика услуг аудиоконференций Microsoft или **Нет**. 
  
6. Нажмите кнопку **Сохранить**. 
    
См. статью [Изменение поставщика конференц-связи с телефонным подключением для пользователей](https://support.office.com/article/9b74f053-4d23-485f-9232-3d30370a8c6e).
  
## Включение или отключение отправки электронных писем пользователям аудиоконференции

Чтобы включить или отключить отправку электронных писем пользователям, можно использовать центр администрирования Skype для бизнеса или Windows PowerShell.
  
 **Использование Центра администрирования Skype для бизнеса**
  
1. Войдите в Office 365 с помощью учебной или рабочей учетной записи.
    
2. Перейдите в **Центр администрирования Office 365** > **Skype для бизнеса** и на панели навигации слева выберите **Аудиоконференция**.
    
3. На странице **Настройки моста Microsoft** установите или снимите флажок **Automatically send emails to users if their audio conferencing settings change (Автоматически отправлять электронные письма пользователям в случае изменения параметров аудиоконференции)**.
    
4. Нажмите кнопку **Сохранить**.
    
    Также можно отправлять электронные письма пользователям с помощью параметров аудиоконференции, выбрав в свойствах аудиоконференции пользователя параметр **Отправить информацию о конференции по электронной почте**. Идентификатор конференции и номер телефона для подключения к аудиоконференции по умолчанию (но не ПИН-код) содержатся в приглашении на собрание.
    
    См. статью [Отправка пользователям электронных писем с информацией о конференции с телефонным подключением](send-an-email-to-a-user-with-their-audio-conferencing-information.md).
    
 **Использование Windows PowerShell**
  
- Также можно использовать Windows PowerShell и выполнить 
    
  ```
  Set-CsOnlineDialInConferencingTenantSetting -AutomaticallySendEmailsToUsers $true|$false
  ```

    Для управления другими настройками для организации, включая электронную почту, можно использовать командлет [Set-CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285).
    
## Изменение контактных данных отправителей электронных писем, отправляемых пользователям

Можно вносить изменения в электронные письма, которые автоматически отправляются пользователям, включая актуальный адрес электронной почты и отображаемое имя в контактных данных отправителя. По умолчанию отправитель электронных писем  Office 365, но можно изменить адрес электронной почты и отображаемое имя с помощью Windows PowerShell и командлета [Set-CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285). Чтобы изменить адрес электронной почты, с которого выполняется отправка сообщений пользователям, выполните следующие действия.
  
- Введите адрес электронной почты в параметр  _SendEmailFromAddress_.
    
- Введите отображаемое имя в параметр  _SendEmailFromDisplayName_.
    
- Установите параметру  _SendEmailOverride_ значение _True_.
    
Чтобы внести изменения в электронные письма, отправляемые пользователям (например, адрес электронной почты, с которого выполняется отправка сообщений, или отображаемое имя), выполните следующие действия:
  
```
Set-CsOnlineDialInConferencingTenantSetting -SendEmailOverride $true -SendEmailFromAddress amos.marble@contoso.com -SendEmailFromDisplayName "Amos Marble"
```

Если вы хотите изменить адрес электронной почты, вам нужно убедиться, что политика входящих сообщений организации позволяет получать письма, отправленные с настроенного адреса электронной почты.
  
Для управления другими настройками для организации, включая электронную почту, можно использовать командлет [Set-CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285).
  
См. статью [Электронные письма, которые автоматически отправляются пользователям при смене настроек конференц-связи с телефонным подключением](emails-that-are-automatically-sent-to-users-when-their-audio-conferencing-settin.md).

    > [!IMPORTANT]

## Сброс ПИН-кода организатора конференции

Статические идентификаторы используются, если пользователи в организации не хотят запоминать случайные числа. В этом случае они могут использовать определенное число, которое проще запомнить. Если используются динамические идентификаторы конференции, каждому собранию, планируемому пользователем, будет присвоен уникальный идентификатор конференции. Если требуется назначить динамические, а не статические идентификаторы конференции, [Аудиоконференции с помощью динамических идентификаторы в вашей организации](using-audio-conferencing-dynamic-ids-in-your-organization.md).
  
Хотя статический идентификатор конференции будет автоматически создан и назначен пользователю, могут возникнуть случаи, когда пользователь не захочет использовать этот идентификатор, а предпочтет определенный номер, либо пользователь не сможет вспомнить или даже потеряет свой идентификатор конференции. В этом случае используйте центр администрирования Skype для бизнеса и Windows PowerShell для просмотра, изменения и сброса соответствующего идентификатора конференции.
  
1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в **Центр администрирования Office 365** > **Skype для бизнеса** и на панели навигации слева выберите **Аудиоконференция**.
    
3. Нажмите **Пользователи** и выберите пользователя, для которого необходимо сбросить ПИН-код.
    
4. На панели действий нажмите **ПИН-код**, затем **Сбросить**.
    
Пользователи получат электронное письмо с ПИН-кодом, если у них разрешена аудиоконференция или сброшен ПИН-код. Однако если автоматическая отправка уведомлений по электронной почте отключена, пользователь не получит электронное письмо при сбросе ПИН-кода и эти данные придется пересылать вручную. ПИН-код показывается только после сброса. Он отображается сразу после сброса и больше не показывается в свойствах пользователя; вместо него отображается *****. 
  
См. статью [Сброс ПИН-кода для конференц-связи с телефонным подключением для пользователя](reset-the-audio-conferencing-pin-for-a-user.md).
  
## Отправка пользователям электронных писем с информацией об аудиоконференции

1. Войдите в Office 365 с помощью учебной или рабочей учетной записи.
    
2. Перейдите в **Центр администрирования Office 365** > **Skype для бизнеса** и на панели навигации слева выберите **Аудиоконференция**.
    
3. Нажмите **Пользователи** и выберите пользователя, для которого необходимо сбросить ПИН-код.
    
4. На панели действий щелкните **Отправить информацию о конференции по электронной почте**.
    
    > [!NOTE]
    > В этом случае ПИН-код аудиоконференции не отправляется пользователю. 
  
См. статью [Отправка пользователям электронных писем с информацией о конференции с телефонным подключением](send-an-email-to-a-user-with-their-audio-conferencing-information.md).
  
## Настройка номера для аудиоконференции по умолчанию для организаторов собраний

 **Процедура настройки номера для аудиоконференции по умолчанию для организаторов собраний при активации пользователя для аудиоконференции**
  
1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в раздел **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. На панели навигации слева перейдите в раздел **Аудиоконференция** > **Пользователи с телефонным подключением**. Выберите пользователя, для которого нужно включить аудиоконференцию.
    
4. На панели действий в разделе свойств для этого пользователя нажмите кнопку **Изменить**.
    
5. На странице **Свойства** в разделе **Имя поставщика** воспользуйтесь раскрывающимся списком для выбора поставщика услуг аудиоконференций.
    
  - Если в качестве поставщика аудиоконференции вы выберете Microsoft, вы также сможете указать номер телефона аудиоконференции по умолчанию из списка.
    
  - При выборе стороннего поставщика услуг ACP в качестве поставщика аудиоконференции вам будет необходимо вручную ввести платный (и если требуется, бесплатный) номер телефона. Эти номера телефонов будут номерами телефонов по умолчанию.
    
    Номер аудиоконференции по умолчанию для пользователя  это номер из приглашения на собрание при планировании собрания.
    
6. Нажмите кнопку **Сохранить**.
    
См. статью [Определение телефонных номеров для аудиоконференций, которые организаторы собраний могут включить в приглашения](set-the-audio-conferencing-phone-numbers-for-meeting-organizers-that-are-include.md).
  
 **Процедура настройки номера для аудиоконференции по умолчанию для организаторов собраний после активации пользователя для аудиоконференции**

2. Перейдите в раздел **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. В **Центр администрирования Skype для бизнеса**на панели навигации слева выберите **Аудиоконференция** > **Пользователи**, укажите нужного пользователя и на панели действий щелкните **Изменить**.
    
4. На странице **Свойства** в разделе **Имя поставщика** воспользуйтесь раскрывающимся списком для выбора поставщика услуг аудиоконференций.
    
  - Если в качестве поставщика аудиоконференции вы выберете Microsoft, вы также сможете указать номер телефона аудиоконференции по умолчанию из списка.
    
  - При выборе пользователем стороннего поставщика услуг ACP в качестве поставщика аудиоконференции вам будет необходимо вручную ввести платный (и если требуется, бесплатный) номер телефона.
    
    Номер аудиоконференции по умолчанию для пользователя  это номер из приглашения на собрание при планировании собрания.
    
5. Нажмите кнопку **Сохранить**.
    
См. статью [Определение телефонных номеров для аудиоконференций, которые организаторы собраний могут включить в приглашения](set-the-audio-conferencing-phone-numbers-for-meeting-organizers-that-are-include.md).
  
## Настройка параметров моста аудиоконференции

 **Установка параметров проведения собрания, когда вызывающие абоненты присоединяются к собранию**
  
1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в раздел **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. В **Центр администрирования Skype для бизнеса** на панели навигации слева выберите **Аудиоконференция** > **Настройки моста Microsoft**.
    
4. В разделе **Присоединение к собранию** выберите указанные далее действия.
    
  - **Включить уведомления о входе и выходе из собрания**. По умолчанию этот флажок установлен.Если снять флажок, то пользователи, уже присоединившиеся к собранию, по умолчанию не будут получать уведомления о том, что кто-то вошел или покинул собрание.
    
    Данный параметр можно установить для конкретного собрания, когда пользователь присоединяется к собранию с помощью приложения Skype для бизнеса или Microsoft Teams, и затем изменить значение **Объявлять о входе или выходе пользователей** в меню **Параметры** собраний Skype или Microsoft Teams.
    
  - **Просить абонентов произносить свое имя перед присоединением к собранию** По умолчанию этот флажок установлен. Если снять флажок, система не будет просить абонентов произносить свое имя перед присоединением к собранию
    
5. Внесите изменения и нажмите кнопку **Сохранить**.
    
См. статью [Изменение параметров моста аудиоконференций](change-the-settings-for-an-audio-conferencing-bridge.md).
  
 **Установка длины ПИН-кода для собраний**
  
1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в раздел **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. В **Центр администрирования Skype для бизнеса** на панели навигации слева выберите **Аудиоконференция** > **Настройки моста Microsoft**.
    
4. В разделе **Безопасность** в списке **Длина ПИН-кода** укажите количество цифр в ПИН-коде, затем нажмите кнопку **Сохранить**.
    
    ПИН-код должен содержать от 4 до 12 цифр. Значение по умолчанию  5.
    
См. статью [Изменение параметров моста аудиоконференций](change-the-settings-for-an-audio-conferencing-bridge.md).
  
 **Включение или отключение отправки электронных писем пользователям аудиоконференции**
  
1. Войдите в Office 365 с помощью учебной или рабочей учетной записи.
    
2. Перейдите в **Центр администрирования Office 365** > **Skype для бизнеса** и на панели навигации слева выберите **Аудиоконференция**.
    
3. На странице **Настройки моста Microsoft** установите или снимите флажок **Automatically send emails to users if their audio conferencing settings change (Автоматически отправлять электронные письма пользователям в случае изменения пааметров аудиоконференции)**.
    
4. Нажмите кнопку **Сохранить**.
    
    Также можно отправить электронное письмо пользователю с помощью параметров аудиоконференции, выбрав на странице свойств аудиоконференции для пользователя параметр **Отправить информацию о конференции по электронной почте**.
    
    В этом случае в отправленном письме будет указан только идентификатор конференции и телефонный номер для подключения. ПИН-код в этом случае не указывается.
    
    См. раздел [Отправка пользователям электронных писем с информацией о конференции с телефонным подключением](send-an-email-to-a-user-with-their-audio-conferencing-information.md).
    
## Просмотр и установка основного и дополнительных языков для моста аудиоконференций

1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в раздел **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. В **Центр администрирования Skype для бизнеса** на панели навигации слева выберите **Аудиоконференция** и щелкните **Мост Microsoft**.
    
4. Выберите номер телефона из списка, на панели действий нажмите **Установить языки**, затем на странице **Установка языков** нажмите список в разделе **Основной язык** для просмотра полного списка поддерживаемых языков.
    
    Вы также можете задать основной и дополнительные языки, которые поддерживаются, когда в качестве поставщика услуг аудиоконференций выбирается Майкрософт. Порядок выбора языков в списках определяет порядок представления этих языков абонентам.

## См. номера для аудиоконференции с телефонным подключением.

1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в раздел **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. В **Центр администрирования Skype для бизнеса** на панели навигации слева выберите **Аудиоконференция** > **Настройки моста Microsoft**. Вам будут предоставлены следующие возможности:
    
  - просматривать телефонные номера, установленные системой Office 365, которые следует использовать для аудиоконференции; 
    
  - просматривать местоположение, основной и дополнительный языки, которые будет использовать автосекретарь аудиоконференции;
    
  - выбрать номер телефона по умолчанию для аудиоконференции, который будет предоставляться пользователям в случае разрешения аудиоконференции. Однако в случае изменения номера телефона по умолчанию для моста аудиоконференций номер телефона для имеющихся пользователей останется прежним.
    
Можно перейти в раздел **Аудиоконференция** > **Пользователи** и выбрать свойства пользователя для изменения номера пользователя по умолчанию, указав новый номер в списке номеров, доступных для организации.
  
См. статью [Просмотр списка номеров для конференц-связи с телефонным подключением](see-a-list-of-audio-conferencing-numbers.md).

## Сведения по управлению с помощью Windows PowerShell

- Можно управлять определенными параметрами на уровне организации с использованием Windows PowerShell. Таким образом, упрощается применение параметров для всех пользователей. Далее представлены параметры на уровне организации:
    
    Для получения дополнительной информации о каждом командлете см. раздел [Командлеты Skype для бизнеса Online](https://go.microsoft.com/fwlink/?LinkId=627324).
    
> 
  ```
  Set-CsOnlineDialInConferencingTenantSettings -EnableEntryExitNotifications $true|$false
  ```

    Значение по умолчанию   _$true_.
    
> 
  ```
  Set-CsOnlineDialInConferencingTenantSettings -EnableNameRecording $true|false
  ```

    Значение по умолчанию   _$true_.
    
> 
  ```
  Set-CsOnlineDialInConferencingTenantSettings -PinLength 7
  ```

    Значение по умолчанию  5.
    
> 
  ```
  Set-CsOnlineDialInConferencingTenantSettings -AllowPSTNOnlyMeetingsByDefault $true|$false
  ```

    Значение по умолчанию   _$false_.
    
> 
  ```
  Set-CsOnlineDialInConferencingTenantSettings -AutomaticallySendEmailsToUsers $true|$false
  ```

    Значение по умолчанию   _$true_.
    
> 
  ```
  Set-CsOnlineDialInConferencingTenantSettings -SendEmailFromOverride $true|$false
  ```

    Значение по умолчанию   _$false_.
    
> 
  ```
  Set-CsOnlineDialInConferencingTenantSettings -SendEmailFromAddress
  ```

    Значение по умолчанию   _$null_.
    
> 
  ```
  Set-CsOnlineDialInConferencingTenantSettings -SendEmailFromDisplayName
  ```

    Значение по умолчанию   _$null_.
    
- Windows PowerShell дает возможность управлять пользователями, предоставляя им права на определенные действия. С его помощью вы можете управлять Office 365, используя единый центр администрирования, который упростит выполнение ваших повседневных задач. Для начала работы с Windows PowerShell ознакомьтесь с приведенными ниже разделами.

Подробнее об этих преимуществах можно узнать в следующих разделах:

    Модуль Windows PowerShell для Skype для бизнеса online позволяет запускать удаленные сеансы Windows PowerShell с подключением к Skype для бизнеса online. Этот модуль для 64-разрядных систем можно загрузить из Центра загрузки Microsoft здесь: [Модуль Windows PowerShell для Skype для бизнеса Online](https://go.microsoft.com/fwlink/?LinkId=294688).
