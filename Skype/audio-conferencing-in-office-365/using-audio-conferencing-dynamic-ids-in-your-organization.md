# Аудиоконференции с помощью динамических идентификаторы в вашей организации

> [!IMPORTANT]
> Данная статья переведена с помощью машинного перевода, см. [Отказ от ответственности](e55e4bff-fb67-4389-8695-f03024baa9b6.md#MT_Footer). Используйте английский вариант этой статьи, который находится [здесь](https://support.office.com/en-us/article/e55e4bff-fb67-4389-8695-f03024baa9b6), в качестве справочного материала. 
  
Служба аудиоконференций обновляется для предоставления каждого Скайп для бизнеса и группами Майкрософт собраний различных конференции идентификаторы. Динамические конференции идентификаторы превышают существенные улучшения статические конференции идентификаторы как они предоставляют:
  
- **Усиленная безопасность** Идентификаторы конференций являются уникальными для каждого Скайп для бизнеса или Microsoft группам собрания и создан при планировании собрания.
    
- **Улучшенная работа для последовательных и параллельных собраний**. Собрания с одним организатором получают определенные сведения для подключения, которые не дают перепутать использующих телефонные подключения участников одного собрания с участниками другого собрания, если эти собрания запланированы примерно на одинаковое время.
    
- **Беспроблемный переход**. Если ваша организация поддерживает динамические идентификаторы конференц-связи, все собрания, которые уже были запланированы в вашей организации с использованием статических идентификаторов конференц-связи, останутся активными.
    
> [!TIP]
> Динамические идентификаторы будут доступны только для пользователей, активированных для ** Аудиоконференции** и у корпорации Майкрософт сделайте их поставщику услуг аудиоконференций. Вы можете[Назначение Майкрософт в качестве поставщика услуг аудиоконференций](assign-microsoft-as-the-audio-conferencing-provider.md) для пользователей.
  
## Какие изменения увидят пользователи в моей организации?

После динамических конференции, которые были включены идентификаторы для вашей организации, любые новые Скайп для бизнеса или Microsoft группам собрания, запланированное для пользователей в вашей организации, поддерживающими аудиоконференций, будут иметь идентификаторы конференций, которые будут отличаться от Идентификатор конференции статические, они были перед. Для организаторов, которые были идентификаторы статические конференции перед они должны напоминания о присоединении свои собрания, которые они необходимо использовать новый идентификатор конференции в приглашении на собрание, прежде чем они могут присоединиться к ней.
  
> [!NOTE]
> Собрания, которые были запланированы пользователем с использованием статических идентификаторов конференц-связи еще до того, как ваша организация перешла на динамические идентификаторы, будут работать с применением статических идентификаторов, поэтому можно продолжать планировать собрания без каких-либо негативных эффектов.
