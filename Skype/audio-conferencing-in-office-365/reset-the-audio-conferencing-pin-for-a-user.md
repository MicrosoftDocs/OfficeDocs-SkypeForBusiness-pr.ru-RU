# Сброс ПИН-кода для конференц-связи с телефонным подключением для пользователя

ПИН-код представляет собой последовательность цифр. Этот код создается для каждого пользователя, которому необходимо подключиться к конференции. ПИН-код для конференц-связи с телефонным подключением позволяет организатору собрания контролировать подключение участников и начинать собрания по телефону. Если собрание открыто в клиенте Skype для бизнеса, ПИН-код не требуется. Если пользователь забыл ПИН-код и не может найти электронное письмо с кодом для подключения к конференции по телефону, администратору необходимо сбросить ПИН-код. Пользователь не может сбросить ПИН-код самостоятельно.
  
Собрание начинается, когда аутентифицированный пользователь подключается с помощью клиента Skype для бизнеса или когда организатор подключается с использованием собственного ПИН-кода по телефону. Если для участия в телефонном собрании требуется ПИН-код, то все пользователи, подключившиеся к этому собранию по телефону, перенаправляются в зал ожидания и могут прослушивать музыку до начала собрания. Если организатор собрания не запрашивает ПИН-код при открытии собрания по телефону, то при подключении к собранию участникам не потребуется вводить ПИН-код.
  
## 

### Сброс ПИН-кода организатора конференции

1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в **Центр администрирования Office 365** > **Skype для бизнеса** и на левой панели навигации выберите **Конференц-связь с телефонным подключением**.
    
3. Нажмите **Подключенные пользователи<**, и выберите пользователя, для которого необходимо сбросить ПИН-код.
    
4. На панели действий щелкните **Назначить ПИН-код**.
    
## Что еще необходимо знать о ПИН-кодах?

- В целях безопасности ПИН-код отображается однократно и только для администратора, выполняющего сброс. После сброса ПИН-кода он отображается в виде символов *********** в Центре администрирования Skype для бизнеса и в области результатов при использовании **Get-CsCsOnlineDialInConfencingUser** в Windows PowerShell.
    
- Автоматическая отправка уведомлений по электронной почте включена по умолчанию. Пользователи получают электронные письма с приглашениями на телефонную конференцию и с информацией о сбросе ПИН-кода. Однако если автоматическая отправка уведомлений по электронной почте отключена, пользователь не получит электронное письмо при сбросе ПИН-кода, и эти данные придется пересылать вручную.
    
- Когда начинается собрание, к нему автоматически присоединяются все пользователи, находящиеся в зале ожидания. Пример: если два участника попытаются присоединиться к собранию, когда оно еще не началось, эти участники будут перенаправлены в зал ожидания. Когда с телефона присоединяется организатор собрания под собственным ПИН-кодом, собрание начинается, и к нему подключаются все участники, находящиеся в зале ожидания.
    
- По умолчанию анонимные абоненты не могут начинать собрание.
    
- Когда пользователя приглашают на конференцию с телефонным подключением, он по умолчанию получает сообщение электронной почты с информацией о конференции и ПИН-кодом. Пользователю требуется почтовый ящик Office 365, так как при сбросе ПИН-кода новый ПИН-код отправляется на основной электронный адрес SMTP (псевдоним), назначенный этому пользователю.
    
- При настройке конференции с телефонным подключением организатор устанавливает сложность ПИН-кода, которой должны соответствовать все ПИН-коды в организации. ПИН-коды могут быть от 4 до 12 цифр. По умолчанию это 5. Если изменить длину ПИН-кода, эта настройка будет применяться только к новым ПИН-кодам. Длина ПИН-кодов для существующих пользователей, которым разрешено участвовать в конференциях с телефонным подключением, останется без изменения. См. [Установка длины ПИН-кода для собраний с телефонным подключением](set-the-length-of-the-pin-for-audio-conferencing-meetings.md).
    
- Отправка уведомлений по электронной почте по умолчанию настроена на основной SMTP-адрес Office 365. Однако сообщение электронной почты можно отправить и на другие адреса, например на адрес Hotmail или MSN. Переопределить используемый по умолчанию адрес электронной почты можно с помощью Windows PowerShell. Это удобно, если у пользователей нет почтового ящика Exchange в Office 365.
    
- Переопределить адрес электронной почты, используемый для отправки уведомлений, можно с помощью следующего командлета:  `Set-CsOnlineDialInConferencingUser -amos.marble -ResetLeaderPIN -SendEmail -SendEmailToAddress "u@hotmail.com"`. Для переопределения адреса электронной почты пользователя требуется параметр  _SendEmail_.
    
## Сведения по управлению с помощью Windows PowerShell

- Для экономии времени или автоматизации процесса можно воспользоваться командлетом [Set-CsOnlineDialInConferencingUser](http://go.microsoft.com/fwlink/?LinkId=617688 ).
    
- Чтобы задать ПИН-код для Amos Marble, выполните следующую команду:
    
  ```
  Set-CsOnlineDialInConferencingUser -id amos.marble@contoso.com -ResetLeaderPIN
  ```
