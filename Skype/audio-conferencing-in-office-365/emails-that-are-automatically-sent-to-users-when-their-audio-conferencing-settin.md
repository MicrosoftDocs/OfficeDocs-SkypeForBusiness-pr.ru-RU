# Электронные письма, которые автоматически отправляются пользователям при смене настроек конференц-связи с телефонным подключением

> [!IMPORTANT]
> Данная статья переведена с помощью машинного перевода, см. [Отказ от ответственности](1b46da6d-f93a-4cc0-9ae8-af765935b007.md#MT_Footer). Используйте английский вариант этой статьи, который находится [здесь](https://support.office.com/en-us/article/1b46da6d-f93a-4cc0-9ae8-af765935b007), в качестве справочного материала. 
  
Сообщения электронной почты автоматически отправляются пользователям, для которых [Настройка аудиоконференций в Skype для бизнеса и Microsoft Teams](set-up-audio-conferencing-for-skype-for-business-and-microsoft-teams.md) с указанием Майкрософт в качестве поставщика услуги телефонного подключения.
  
По умолчанию существует четыре типа сообщений электронной почты, которые отправляются пользователям, для которых включена услуга конференц-связи с телефонным подключением. Однако можно отключить некоторые из них, чтобы уменьшить количество сообщений, отправляемых пользователям. Конференц-связь с телефонным подключением в Skype для бизнеса Online отправляет пользователям электронные сообщения в следующих случаях.
  
- **При назначении пользователям лицензии на **Конференц-связь через ТСОП в Skype для бизнеса** или выборе Microsoft в качестве поставщика конференц-связи с телефонным подключением.**
    
     Это электронное сообщение содержит идентификатор конференции, номер телефона для подключения к собраниям по умолчанию, ПИН-код для пользователей, инструкции и ссылку на средство обновления собраний Skype для бизнеса Online. См. разделы[Назначение лицензий Skype для бизнеса и Microsoft Teams](../skype-for-business-and-microsoft-teams-add-on-licensing/assign-skype-for-business-and-microsoft-teams-licenses.md) и[Назначение Майкрософт в качестве поставщика услуг аудиоконференций](assign-microsoft-as-the-audio-conferencing-provider.md).
    
    > [!NOTE]
    > Если в вашей организации включены динамические идентификаторы конференц-связи, всем планируемым пользователями собраниям будут назначаться уникальные идентификаторы конференц-связи. При необходимости вы можете настроить [Аудиоконференции с помощью динамических идентификаторы в вашей организации](using-audio-conferencing-dynamic-ids-in-your-organization.md).

- **Идентификатор конференции или телефонный номер конференции по умолчанию изменен.**
    
    Это электронное сообщение содержит идентификатор конференции, номер телефонного подключения по умолчанию, инструкции и ссылку на инструмент для обновления собраний Skype для бизнеса Online. При необходимости вы можете настроить [Аудиоконференции с помощью динамических идентификаторы в вашей организации](using-audio-conferencing-dynamic-ids-in-your-organization.md).

- **ПИН-код пользователя для конференц-связи с телефонным подключением сброшен.**
    
    Это электронное сообщение содержит ПИН-код организатора конференц-связи с телефонным подключением, текущий идентификатор конференции и номер телефонного подключения по умолчанию для пользователя. ## Внесение изменений в электронные письма, отправляемые пользователям

Сообщение, которое автоматически отправляется пользователями, включая адрес электронной почты и отображаемое имя, которое входит в состав контактные данные  *из*  можно вносить изменения. По умолчанию будет отправителя сообщения электронной почты из Office 365, но вы можете изменить адрес электронной почты и отображаемое имя с помощью Windows PowerShell и командлет[Set-CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285) . Для внесения изменений на адрес электронной почты, отправки электронной почты для пользователей необходимо выполнить следующее:
  
- Введите адрес электронной почты в параметр  _SendEmailFromAddress_.
    
- Введите отображаемое имя в параметр  _SendEmailFromDisplayName_.
    
- Установите параметру  _SendEmailOverride_ значение _True_.
    
Чтобы изменить текст электронных сообщений, отправляемых пользователям (например, адрес электронной почты, с которого выполняется отправка сообщений, и отображаемое имя), выполните следующие действия:
  
```
Set-CsOnlineDialInConferencingTenantSetting -SendEmailOverride $true -SendEmailFromAddress amos.marble -SendEmailFromDisplayName "Amos Marble"
```

> [!NOTE]
> Если требуется изменить данные адреса электронной почты, убедитесь, что действующие в вашей организации политики входящей электронной почты позволяют получать сообщения с дополнительных адресов. > Чтобы заменить контактные данные  *отправителя*  , убедитесь, что электронные сообщения правильно отправляются пользователям. Для этого можно проверить отправку на одном из пользователей в организации.
  
Управление другими параметрами для вашей организации, включая электронной почты, можно воспользоваться командлетом [Set-CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285) .
  
## Что если вы не хотите, чтобы выполнялась отправка сообщений пользователям?

Если отключить отправку электронных сообщений пользователям, сообщения не будут отправляться, даже когда пользователю назначается лицензия. В этом случае идентификатор конференции, телефонный номер по умолчанию, а также (что важно) ПИН-код для подключения к конференции не будут отправляться пользователю. В этом случае вы должны сообщить пользователю эти данные по телефону или в отдельном электронном сообщении.
  
По умолчанию пользователям отправляются электронные сообщения, но если вы не хотите, чтобы они получали сообщения для конференц-связи с телефонным подключением, используйте Центр администрирования Skype для бизнеса или Windows PowerShell. 
  
### Использование Центра администрирования Skype для бизнеса

1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в раздел **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. В окне **Центр администрирования Skype для бизнеса**на панели навигации слева выберите **Конференц-связь с телефонным подключением** > **Параметры моста Microsoft**.
    
4. На странице **Параметры моста Microsoft** установите или снимите флажок **Автоматически отправлять электронные письма пользователям в случае изменения конфигурации телефонного подключения по ТСОП**
    
5. Нажмите кнопку **Сохранить**.
    
### Использование Windows PowerShell

- Чтобы отключить отправку электронных сообщений всем пользователям, выполните следующую команду:
    
  ```
  Set-CsOnlineDialInConferencingTenantSetting -AutomaticallySendEmailsToUsers $false
  ```

[Набор CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285) можно использовать для управления другими параметрами для вашей организации, включая электронной почты.

## Сведения по управлению с помощью Windows PowerShell

- По умолчанию будет отправителя сообщения электронной почты из Office 365, но вы можете изменить адрес электронной почты и отображаемое имя с помощью Windows PowerShell и командлет [Set-CsOnlineDialInConferencingTenantSettings](https://go.microsoft.com/fwlink/?LinkId=627285) .
    
- Windows PowerShell дает возможность управлять пользователями, предоставляя им права на определенные действия. С его помощью вы можете управлять Office 365, используя единый центр администрирования, который упростит выполнение ваших повседневных задач. Для начала работы с Windows PowerShell ознакомьтесь с приведенными ниже разделами.

Подробнее об этих преимуществах можно узнать в следующих разделах:
