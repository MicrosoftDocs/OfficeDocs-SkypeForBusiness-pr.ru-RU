# Просмотр, изменение и сброс идентификатора конференц-связи, назначенного пользователю

Идентификатор конференции автоматически назначается пользователю, если для него настроена конференц-связь с телефонным подключением через Майкрософт как поставщика услуг конференц-связи с телефонным подключением. Идентификатор конференции может быть статическим или динамическим. Он отправляется в приглашении на собрание при планировании собрания. 
  
Статические идентификаторы используются, если люди в вашей организации не хотят запоминать случайные числа. Тогда они могут выбрать определенное число, которое проще запомнить. Если используются динамические идентификаторы конференции, каждое собрание, планируемое пользователем, получит уникальный идентификатор конференции. Если вы хотите назначить динамические, а не статические идентификаторы конференции, [Аудиоконференции с помощью динамических идентификаторы в вашей организации](using-audio-conferencing-dynamic-ids-in-your-organization.md).
  
Хотя статический идентификатор конференции будет автоматически создан и назначен пользователю, могут возникнуть случаи, когда пользователь не захочет использовать этот идентификатор, а пожелает определенный номер, или пользователь не сможет вспомнить или даже потеряет свой идентификатор конференции. В этом случае используйте центр администрирования Skype для бизнеса и Windows PowerShell для просмотра, изменения и сброса соответствующего идентификатора конференции.
  
Пользователю будет отправлено сообщение электронной почты с идентификатором конференции и телефонными номерами конференц-связи с телефонным подключением. В случае сброса идентификатора конференции пользователю будет отправлено другое сообщение электронной почты, содержащее идентификатор конференции, но не содержащее ПИН-код. 
  
## 

### Просмотр идентификатора конференц-связи

Вы можете просмотреть идентификатор конференции пользователя и отправить ему этот идентификатор.
  
1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. Перейдите в Центр администрирования Skype для бизнеса Online > **Конференц-связь с телефонным подключением**. В списке пользователей выберите пользователя, которому требуется идентификатор.
    
4. На странице действий просмотрите раздел **Идентификатор конференции**.
    
    > [!TIP]
    > Всю информацию о конференц-связи, включая идентификатор конференции и номера телефонов для телефонного подключения, можно отправить пользователю по электронной почте, щелкнув ссылку **Отправить сведения о конференции по эл. почте** после выбора пользователя на странице **Подключенные пользователи**. 
  
    Для просмотра идентификатора конференции пользователя можно использовать Windows PowerShell, выполните следующее:
    
  ```
  Get-CsOnlineDialInConferencingUser -Identity "Amos Marble"  
  ```

    Для получения дополнительных сведений о командлете см. [Get-CsOnlineDialInConferencingUser](http://go.microsoft.com/fwlink/?LinkId=617693 ).
    
### Назначение или изменение идентификатора конференции

Вы можете назначить или изменить идентификатор конференции для пользователя, если, к примеру, пользователь желает использовать легко запоминаемый идентификатор.
  
> [!NOTE]
> Центр администрирования Skype для бизнеса нельзя использовать для изменения идентификатора конференции, который был создан автоматически, однако можно использовать Windows PowerShell для редактирования или изменения заданного идентификатора конференции. 
  
- Чтобы отредактировать или изменить идентификатор конференции для пользователя, выполните следующее:
    
    > [!TIP]
    > Идентификатор конференции должен содержать 7 цифр, и его нельзя изменить в Центре администрирования Skype для бизнеса или с помощью Windows PowerShell. 
  
  ```
  Set-CsOnlineDialInConferencingUser -Identity "Amos Marble"  -ConferenceId 8271964
  ```

### Сброс идентификатора конференции

Вы можете сбросить идентификатор конференции, если, к примеру, пользователь забыл его.
  
1. Войдите в Office 365 под своей учебной или рабочей учетной записью.
    
2. Перейдите в раздел **Центр администрирования Office 365** > **Skype для бизнеса**.
    
3. В **Центр администрирования Skype для бизнеса**> **Конференц-связь с телефонным подключением** на странице действий в разделе **Идентификатор конференции** щелкните **Сброс**.
    
4. В окне **Сбросить идентификатор конференции?** выберите **Да**. Будет автоматически создан идентификатор конференции, и на электронную почту пользователя будет отправлено сообщение с новым идентификатором конференции.
    
    Идентификатор конференции пользователя можно сбросить с помощью Windows PowerShell. Для этого выполните следующее:
    
  ```
  Set-CsOnlineDialInConferencingUser -Identity "Amos Marble"  -ResetLeaderPIN 8271964
  ```

## Дополнительные сведения

-     > [!IMPORTANT]
    >  После создания нового идентификатора или сброса текущего старый идентификатор конференции не может использоваться абонентами. Необходимо уведомить пользователей о необходимости изменения планирования существующих приглашений на собрания, чтобы в приглашения был добавлен новый идентификатор конференции. Пользователи могут использовать средство миграции собраний Skype для бизнеса для обновления существующих собраний. Сведения о загрузке, установке и запуске этого средства см. в разделе> [Средство обновления для собраний Skype для бизнеса и Lync](https://support.office.com/article/2b525fe6-ed0f-4331-b533-c31546fcf4d4)> [Skype для бизнеса Online, инструмент переноса собраний (64-разрядная версия)](http://go.microsoft.com/fwlink/?LinkID=626047)> [Skype для бизнеса Online, инструмент переноса собраний (32-разрядная версия)](http://go.microsoft.com/fwlink/?LinkID=626046)
  
- Для получения дополнительных сведений о командлете см. [Set-CsOnlineDialInConferencingUser](http://go.microsoft.com/fwlink/?LinkId=617688 ).
    
- Идентификатор конференции должен соответствовать длине, заданной на мосте конференц-связи с телефонным подключением. В этих идентификаторах невозможно использовать буквенные и специальные символы, а можно использовать только цифры.
    
- По умолчанию идентификатор конференции любого пользователя конференц-связи с телефонным подключением будет состоять из 7 цифр. Количество цифр является фиксированным.
    
- В случае перехода пользователя из Microsoft в качестве поставщика конференц-связи с телефонным подключением к стороннему поставщику аудиоконференций или установки значения **Нет** для поставщика конференц-связи с телефонным подключением идентификатор конференции перестанет работать. См. раздел[Назначение стороннего поставщика услуг аудиоконференций](assign-a-third-party-as-the-audio-conferencing-provider.md) или[Изменение поставщика конференц-связи с телефонным подключением для пользователей](https://support.office.com/article/9b74f053-4d23-485f-9232-3d30370a8c6e).

Подробнее об этих преимуществах можно узнать в следующих разделах:
