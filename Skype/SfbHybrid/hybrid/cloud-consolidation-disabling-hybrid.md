# <a name="disable-hybrid-to-complete-migration-to-the-cloud"></a>Отключение гибридной среды для завершения миграции в облако

После перемещения всех пользователей из локальной среды в облако вы можете удалить локальное развертывание Skype для бизнеса. Помимо удаления любого оборудования важно логически отделить локальное развертывание от Microsoft 365 или Office 365 путем отключения гибридной системы. Отключение гибридной системы состоит из 3 этапов:

1. Обновив записи DNS, указав на Microsoft 365 или Office 365.

2. Отключите разделенный домен в организации Microsoft 365 или Office 365.

3. Отключать возможность локального взаимодействия с Microsoft 365 или Office 365.

Эти действия следует делать вместе в качестве единицы. Подробные сведения приведены ниже. Кроме того, предоставляются рекомендации по управлению номерами телефонов для перенесенных пользователей после отключения локального развертывания.

После выполнения этих действий локальное серверы Skype для бизнеса больше не используются, и эти серверы можно переопоимять.

> [!Important] 
>Следует продолжать синхронизацию атрибутов msRTCSIP в Active Directory с помощью Azure AD Connect в Azure AD.  Не очищать какие-либо из этих атрибутов, если они не были направлены службой поддержки.  Не запускать Disable-CsUser в локальной среде. Если вам нужно изменить SIP-адрес пользователя, сделайте это в локальной службе Active Directory и позвольте этому изменению синхронизироваться с Azure AD через Azure AD Connect, как описано ниже. Аналогичным образом, если необходимо изменить номер телефона, а lineURI пользователя уже определен в локальной службе Active Directory, измените его.
>Очистка атрибутов локальной службы msRTCSIP после миграции из локальной сети может привести к потере службы для пользователей!


1.  *Обновив DNS, указав на Microsoft 365 или Office 365.*
Необходимо обновить внешнюю службу DNS организации для локальной организации, чтобы записи Skype для бизнеса указывают на Microsoft 365 или Office 365, а не на локальное развертывание. Это означает следующее:

    |Тип записи|Name|TTL|Value (Значение)|
    |---|---|---|---|
    |SRV|_sipfederationtls._tcp|3600|100 1 5061 sipfed.online.lync. <span> com|
    |SRV|_sip._tls|3600|100 1 443 sipdir.online.lync. <span> com|
    |CNAME| lyncdiscover|   3600|   webdir.online.lync. <span> com|
    |CNAME| sip|    3600|   sipdir.online.lync. <span> com|
    |CNAME| meet|   3600|   webdir.online.lync. <span> com|
    |CNAME| dialin  |3600|  webdir.online.lync. <span> com|

    Кроме того, записи CNAME для meet или dialin (если они есть) можно удалить. Наконец, все записи DNS для Skype для бизнеса во внутренней сети необходимо удалить.

    > [!Note] 
    > В редких случаях изменение DNS с локального указателя на Office 365 для вашей организации может привести к тому, что федерация с некоторыми другими организациями перестает работать, пока другая организация не обнорит конфигурацию федерации:
    >
    > - Все федераированные организации, использующие старую модель прямой федерации (также известные как разрешенный сервер партнеров), должны будут обновить разрешенные записи домена для своей организации, чтобы удалить имя FQDN прокси-сервера. Эта устаревшая модель федерации не основана на записях DNS SRV, поэтому такая конфигурация станет устаревшей после перемещения организации в облако.
    > 
    > - Любая федератная организация, в которую не включен поставщик услуг размещения для sipfed.online.lync. <span> чтобы включить эту возможность, com необходимо обновить конфигурацию. Такая ситуация возможна только в том случае, если федератная организация является чисто локальной и никогда не была федераированной с гибридным или сетевым клиентом. В этом случае федерация с этими организациями не будет работать до тех пор, пока они не вятят своего поставщика услуг размещения.
    >
    > Если вы подозреваете, что какие-либо из федеративных партнеров могут использовать прямую федерацию или не были федеративны с какой-либо сетевой или гибридной организацией, мы рекомендуем отправить им сообщение об этом при подготовке к завершению миграции в облако.


2.  *Отключение общего адресного пространства SIP в организации Microsoft 365 или Office 365.*
Приведенную ниже команду необходимо сделать в окне PowerShell Skype для бизнеса Online.

    ```PowerShell
    Set-CsTenantFederationConfiguration -SharedSipAddressSpace $false
    ```
 
3.  *Отключать возможность локального взаимодействия с Microsoft 365 или Office 365.*  
Приведенную ниже команду необходимо сделать из локального окна PowerShell:

    ```PowerShell
    Get-CsHostingProvider|Set-CsHostingProvider -Enabled $false
    ```

### <a name="manage-sip-addresses-and-phone-numbers-for-users-who-were-migrated-from-on-premises"></a>Управление SIP-адресами и номерами телефонов для пользователей, перенесенных из локальной сети

Администраторы могут управлять пользователями, которые ранее были перемещены из локального сервера Skype для бизнеса Server в облако, даже после списки локального развертывания. Если вы хотите внести изменения в SIP-адрес пользователя или URI линии пользователя (а SIP-адрес или URI строки уже определены в локальной службе Active Directory), это необходимо сделать в локальной службе Active Directory, чтобы значения перетекали в Azure AD. Для этого НЕ требуется локальное приложение Skype для бизнеса Server. Вместо этого вы можете изменить эти атрибуты непосредственно в локальной службе Active Directory с помощью оснастки "Пользователи и компьютеры Active Directory" MMC или с помощью PowerShell. Если вы используете оснастку MMC, откройте страницу свойств пользователя, щелкните вкладку "Редактор атрибутов" и найдите соответствующие атрибуты для изменения:

- Чтобы изменить SIP-адрес пользователя, измените параметр `msRTCSIP-PrimaryUserAddress` . Кроме того, если атрибут содержит значение SIP, обновим его в соответствие с `ProxyAddresses` новым значением `msRTCSIP-PrimaryUserAddress` . Если значение SIP не содержится, его можно оставить пустым.

- Чтобы изменить номер телефона пользователя, измените, если он `msRTCSIP-Line` *уже имеет значение.*

  ![Средство для пользователей и компьютеров Active Directory](../media/disable-hybrid-1.png)
  
Если изначально у пользователя не было значения для локальной точки перед перемещением, вы можете изменить LineURI с помощью параметра - в `LineURI` командлете `onpremLineUri` [Set-CsUser](https://docs.microsoft.com/powershell/module/skype/set-csuser?view=skype-ps) в модуле PowerShell Skype для бизнеса Online.
