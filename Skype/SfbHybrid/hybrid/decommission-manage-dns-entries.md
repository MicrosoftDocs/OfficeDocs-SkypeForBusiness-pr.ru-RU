# <a name="update-dns-entries-to-enable-your-organization-to-be-all-teams-only"></a>Обновите записи DNS, чтобы сделать организацию доступной только Teams

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

Организации, в которых ранее были локальные развертывания Skype для бизнеса Server или Lync Server, могут по-прежнему иметь записи DNS, указывающие на локальное Skype для бизнеса развертывания. Эти записи необходимы, если в вашей организации есть локальные Skype для бизнеса пользователей. Однако после того как в вашей организации больше нет локальных пользователей Skype для бизнеса или Lync Server, эти исходные записи больше не требуются в локальном развертывании, и эти записи DNS необходимо обновить так, чтобы они указывали на **Microsoft 365 (** или в некоторых случаях были удалены) в ходе миграции из локальной среды в Teams Только. *Корпорация Майкрософт не может обновить эти записи DNS от вашего имени.*

При попытке предоставить TeamsOnly для всего клиента Teams будет проверять DNS, чтобы определить, существуют ли какие-либо из перечисленных ниже записей DNS в каждом Microsoft 365 проверенных доменах в вашей организации. Если какие-либо записи найдены и указывают на что-либо, отличное от Microsoft 365, попытка изменить режим сосуществования клиента на TeamsOnly завершится ошибкой. Это не позволяет гибридным организациям с локальными пользователями по ошибке применять режим TeamsOnly к клиенту, так как это приведет к разрыву федерации для всех локальных пользователей Skype для бизнеса в организации (как с помощью Teams, так и Skype для бизнеса).

## <a name="how-to-identify-stale-dns-records"></a>Как определить устаревшие записи DNS

Чтобы определить все записи DNS, которые не помешает вашей организации стать Teams Только, можно использовать центр администрирования Teams, чтобы изменить режим сосуществования на TeamsOnly. Перейдите **Teams** >  **Teams параметров обновления**. В сообщение об ошибке будут включены все записи DNS, которые препятствуют Teams организации.  Если записи DNS не найдены, режим сосуществования для вашей организации будет изменен на TeamsOnly.

Кроме того, вы можете использовать Teams PowerShell для выполнения тех же задач, как показано ниже:

   ```PowerShell
   Grant-CsTeamsUpgradePolicy -PolicyName UpgradeToTeams -Global
   ```

## <a name="dns-records-to-be-updated"></a>Обновляемые записи DNS

Если в вашей организации больше нет пользователей, размещенных в локальной среде Skype для бизнеса Server или Lync Server, необходимо сделать следующее:

- Обновите Skype для бизнеса записей DNS ниже, чтобы указать Microsoft 365 (вместо локального развертывания). Если у вас несколько доменов SIP, это необходимо сделать для каждого домена SIP, который Microsoft 365 проверенным доменом.

- Удалите Skype для бизнеса DNS, если домен SIP больше не используется.

В каждом домене, где находятся следующие записи, обновите их следующим образом:

|Тип записи|Имя|TTL|Приоритет|Насыщенность|Порт|Value (Значение)|
|---|---|---|---|---|---|---|
|SRV|_sipfederationtls._tcp|3600|100|1|5061|sipfed.online.lync.com|
|SRV|_sip._tls|3600|100|1|443|sipdir.online.lync.com|
|CNAME|lyncdiscover|3600|Н/Д|Н/Д|Недоступно|webdir.online.lync.com|
|CNAME|sip|3600|Н/Д|Н/Д|Недоступно|sipdir.online.lync.com|

Кроме того, записи CNAME для собрания или диалога (при наличии) можно удалить. Наконец, все записи DNS для Skype для бизнеса во внутренней сети должны быть удалены.

> [!NOTE]
> В редких случаях изменение DNS с локального на Microsoft 365 для вашей организации может привести к тому, что федерация с другими организациями перестанет работать до тех пор, пока другая организация не обнотит конфигурацию федерации:
>
> - Все федеративные организации, использующие старую модель прямой федерации (также известная как Разрешенный сервер партнеров), должны обновить разрешенные записи домена для своей организации, чтобы удалить полное доменное имя прокси-сервера. Эта устаревшая модель федерации не основана на записях SRV DNS, поэтому такая конфигурация станет устаревшей после перемещения организации в облако.
>
> - Любая федеративная организация, у которой нет включенного поставщика услуг размещения для sipfed.online.lync.<span> Чтобы включить эту функцию, com необходимо обновить конфигурацию. Такая ситуация возможна только в том случае, если федеративная организация является исключительно локальной и никогда не была федеративной с гибридным или сетевым клиентом. В этом случае федерация с этими организациями не будет работать, пока они не включит своего поставщика услуг размещения.
>
> Если вы подозреваете, что любой из ваших федеративных партнеров может использовать прямую федерацию или не был федеративным с какой-либо сетевой или гибридной организацией, мы рекомендуем отправить им сообщение об этом при подготовке к завершению миграции в облако.
