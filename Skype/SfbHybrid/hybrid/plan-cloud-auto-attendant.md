# <a name="plan-cloud-auto-attendants"></a>Планирование автосекретарей в облаке

Автосекретарь, используемый с единой системой обмена сообщениями Exchange (Exchange Server 2013 или Exchange Server 2016), больше недоступен в Exchange Server 2019 или Exchange Online. Если ваша реализация Skype для бизнеса Server 2019 интегрируется с любой из этих версий Exchange, вам потребуется использовать функции голосовой связи в Интернете, связанные с телефонной системой. Сведения о перемещении служб единой системы обмена сообщениями Exchange, размещенных на сервере Exchange Server 2013 и 2016, приведены в статье [Plan for Skype for Business Server and Exchange Server Migration](plan-um-migration.md) .

Это означает, что у вас будет гибридная реализация Skype для бизнеса Server 2019, если вы хотите использовать функции единой системы обмена сообщениями, такие как автосекретарей. Для получения дополнительных сведений см. [Настройка гибридного подключения между Skype для бизнеса Server и Office 365](configure-hybrid-connectivity.md) .

Автосекретарь — это облачная служба, принимающая звонки пользователей и воспроизводимые приветствием, предоставляющие им варианты меню и взаимодействующие с абонентами, используя голосовые функции или диалпад, для направления вызовов в нужное место. Каждому автосекретарьу назначена **учетная запись ресурса** (см.[Настройка учетных записей ресурсов](configure-onprem-ra.md)) в системе Skype для бизнеса Server 2019, которая будет связана непосредственно с автосекретарем в центре администрирования Microsoft Teams. Узнайте [, что такое автосекретарей для облачных приложений?](/SkypeForBusiness/what-is-phone-system-in-office-365/what-are-phone-system-auto-attendants.md) дополнительные сведения о том, какие автосекретаря и какие функции доступны для автосекретарей.

> [!NOTE]
> Автосекретарь может назначать несколько номеров служб Майкрософт или гибридных номеров.

Входящий вызов облачного автосекретаря может принимать один из нескольких путей, как показано ниже:

![Схема автосекретарей](../../SfBServer2019/media/AA-plan-concept.png)

1. С помощью Skype для бизнеса Server 2019
2. Через [пограничный контроллер сеансов](/MicrosoftTeams/direct-routing-border-controllers.md) и [прямую маршрутизацию](/MicrosoftTeams/direct-routing-plan.md)
3. Через номер, размещенный в Интернете в Office 365.

См. также:

- [Настройка облачного автосекретаря](/microsoftteams/create-a-phone-system-auto-attendant)
- [Автоматический ответ и маршрутизация для входящих вызовов](https://docs.microsoft.com/exchange/voice-mail-unified-messaging/automatically-answer-and-route-calls/automatically-answer-and-route-calls)

## <a name="requirements"></a>Требования

В приведенных ниже требованиях предполагается, что у вас уже развернут Skype для бизнеса Server 2019 в поддерживаемой топологии.  Требования зависят от вашего сценария:

- Если вы уже используете единую систему обмена сообщениями Exchange Online или локально и хотите выполнить обновление до Skype для бизнеса 2019, необходимо записать структуру автосекретарей и повторно создать их в облаке с помощью облачных автосекретарей. Дополнительную информацию можно узнать [в статье перемещение автосекретаря единой системы обмена сообщениями Exchange или очереди звонков в телефонную систему](configure-onprem-ra.md#moving-an-exchange-um-auto-attendant-or-call-queue-to-phone-system).

- Для создания новой конфигурации автоматических автосекретарей выполните действия, описанные в разделе [Настройка учетных записей ресурсов](configure-onprem-ra.md).

В дополнение к приведенным выше требованиям необходимо настроить следующие требования для подключения к службе автоматического секретаря Microsoft Cloud:

- Гибридное подключение. Если вы уже развернули Skype для бизнеса Server и хотите включить облачный автосекретарь для локальных пользователей, необходимо убедиться в том, что гибридное подключение настроено между локальной средой и интернет-средой. Иногда это называется разделенной конфигурацией домена.

   Для получения дополнительных сведений ознакомьтесь со статьей [планирование гибридного подключения между Skype для бизнеса Server и office 365](plan-hybrid-connectivity.md) и [Настройка гибридного подключения между Skype для бизнеса server и Office 365](configure-hybrid-connectivity.md).

- Если вы назначаете номер телефона автосекретаря, вам потребуется лицензия на [Office 365 Корпоративная](/skypeforbusiness/skype-for-business-and-microsoft-teams-add-on-licensing/license-options-based-on-your-plan/office-365-enterprise-e5-with-audio-conferencing) .
- Создайте локальную учетную [запись ресурса](/MicrosoftTeams/manage-resource-accounts.md) для каждого автосекретаря и назначьте номера телефонов и лицензии. 

## <a name="migration-and-interoperability"></a>Миграция и взаимодействие

При планировании развертывания Skype для бизнеса Server 2019 и/или Exchange Server 2019 необходимо тщательно спланировать миграцию, чтобы обеспечить поддержку автосекретарей. Учитывайте следующее:

- Exchange Server 2019 больше не поддерживает функции единой системы обмена сообщениями Exchange
- Единая система обмена сообщениями Exchange Online находится в режиме выбытия
- Skype для бизнеса Server 2019 больше не интегрируется с единой системой обмена сообщениями Exchange Online

Для настройки облачных автосекретарей можно использовать Skype для бизнеса Server 2019, 2015 и 2013.

Корпорация Майкрософт рекомендует следующие пути миграции:

- При обновлении до Skype для бизнеса Server 2019 вы можете использовать единую систему обмена сообщениями Exchange в Exchange Server 2013 или 2016, но если вы используете Exchange Server 2019, вам необходимо выполнить обновление до облачного автоматического секретаря.

- Если вы выполняете обновление до Exchange Server 2019 и используете предыдущие версии единой системы обмена сообщениями Exchange Server для голосовой почты Skype для бизнеса Server, корпорация Майкрософт рекомендует перед обновлением почтовых ящиков обновить приложение до Skype для бизнеса Server 2019.  В противном случае возможности обмена голосовыми сообщениями будут потеряны.

Для получения дополнительных сведений о планировании миграции, ознакомьтесь со статьей [Plan for Skype для бизнеса Server и Exchange Server Migration](plan-um-migration.md).

### <a name="migrating-a-previously-implemented-exchange-um-auto-attendant-system"></a>Перенос ранее реализованной системы автосекретаря единой системы обмена сообщениями Exchange

В настоящее время мы не поддерживаем автоматическую миграцию в облако системы автосекретаря единой системы обмена сообщениями, созданной в Exchange 2013 или 2016. Для повторного создания системы автосекретаря вручную необходимо выполнить следующие действия:

1. Используйте команды PowerShell для администрирования Exchange, чтобы просмотреть структуру старой системы автосекретаря, включая всех вложенных автосекретарей и очередей звонков.  
2. Создайте копии сценариев преобразования текста в речь или записанных сообщений, связанных с каждым узлом автосекретаря единой системы обмена сообщениями.
3. Создайте локальные конечные точки для каждого узла автосекретаря, включая назначение тестовых номеров телефонов и лицензий объектам. Обратите внимание, что теперь у вас есть возможность назначать лицензии на локальные номера телефонов, используемые интернет-службами, такими как телефонная система.
4. Реализация новой облачной службы автоматического секретаря с помощью Skype для бизнеса Online и телефонной системы. В разделе [Настройка учетных записей ресурсов](configure-onprem-ra.md) для получения сведений о реализации. После этого отправьте сценарии преобразования текста в речь или записанные сообщения, связанные с каждым узлом автосекретаря единой системы обмена сообщениями.
5. Протестируйте функции автосекретаря облачных приложений.
6. Переназначение номера телефона, назначенного старому автосекретарь единой системы обмена сообщениями Exchange, новому автоматически созданному главному облачному подучастнику.

Сведения об этих действиях приведены в статье [Перемещение автосекретаря единой системы обмена сообщениями Exchange или очереди звонков в телефонную систему](configure-onprem-ra.md#moving-an-exchange-um-auto-attendant-or-call-queue-to-phone-system) .
