# <a name="plan-cloud-voicemail-service-for-on-premises-users"></a>Планирование облачной службы голосовой почты для локального пользователя

## <a name="overview"></a>Общие сведения

В этой статье описываются преимущества, соображения планирования и требования к реализации службы голосовой почты Microsoft Cloud для локального пользователя. Сведения о настройке облачной голосовой почты см. в [сайте Configure Cloud Voicemail service.](configure-cloud-voicemail.md)

Облачная голосовая почта занимает место Exchange Unified Messaging (UM) для предоставления функций голосовых сообщений для пользователей голосовой почты Skype для бизнеса 2019 г., у которых есть почтовые ящики на Exchange Server 2019 г. или Exchange Online. Облачная голосовая почта предоставляет следующие преимущества как для локального, так и для интернет-пользователей:

- Функции голосовой почты и депонировки с расширенной транскрипцией речи

- Доступ к голосовой почте в почтовом ящике Exchange пользователя с помощью клиентов Skype для бизнеса Online или Outlook

- Возможность использования центра администрирования Microsoft 365 для управления вариантами голосовой почты

- Поддержка почтовых ящиков Exchange в помещениях или в облаке

- Использование существующих приветствий пользователей из Единой системы обмена сообщениями Exchange Online

> [!Important]
> Skype для бизнеса Online будет отменен 31 июля 2021 г., после чего пользователи больше не смогут получать доступ к голосовой почте в почтовом ящике Exchange через клиент Skype для бизнеса Online.

Дополнительные сведения о сопоставлении функций см. в [вехе Plan for Skype for Business Server и Exchange Server миграции.](plan-um-migration.md)

Skype для бизнеса Server 2019 продолжает использовать Exchange UM для пользователей, почтовые ящики которых находятся в предыдущих версиях Exchange Server (2013, 2016).  Понимание того, какое решение голосовой почты будет использоваться на основе версии Exchange Server и Skype для бизнес-сервера, является важной частью планирования переноса в Skype для бизнеса Server 2019 или Exchange Server 2019 г. Дополнительные сведения о миграции и интероперабельности см. в веб-сведениях [Plan for Skype for Business Server и Exchange Server миграции.](plan-um-migration.md)

С помощью облачной голосовой почты задачи администрирования значительно упрощаются, так как:

- Нет необходимости настраивать роль Обмена обмена данными.
- Задачи настройки облачной голосовой почты проще.
- Обновления функций голосовой почты доставляются непосредственно в облаке, поэтому пользователи всегда имеют доступ к последним функциям и обновлениям с меньшей зависимостью от накопительных обновлений (CUs).
- У вас одинаковый набор элементов управления как для локального, так и для почтовых ящиков Exchange в Интернете. Дополнительные сведения об этих средствах управления см. в [сайте Настройка голосовой почты системы телефона.](https://support.office.com/article/Set-up-Phone-System-voicemail-Admin-help-9c590873-b014-4df3-9e27-1bb97322a79d)

На следующей схеме показана облачная голосовая почта в гибридном развертывании:

![Голосовая почта облака SfB](../../sfbserver2019/media/plan-cloud-voice-mail-server1.png)

Вызовы без ответа обрабатываются следующим образом:  

1. Для пользователей, которые находятся в Skype для бизнеса 2019 на локальной основе, вызовы без ответа отправляются локальной службой Skype для бизнеса Server в службу голосовой почты облака в Интернете.
2. Служба обрабатывает голосовую почту, включая транскрипцию.
3. Затем служба передает голосовую почту в почтовый ящик Exchange пользователя, независимо от того, находится ли почтовый ящик в локальном или онлайновом режиме.  
4. Пользователи могут получать доступ к своей голосовой почте либо из своего клиента Skype для бизнеса или Outlook.

## <a name="requirements"></a>Требования

Следующие требования предполагают, что Skype для бизнеса Server уже развернут в поддерживаемой топологии.  Ваши требования зависят от сценария:

- Если вы уже используете Exchange UM online и перенаправите его в Skype для бизнеса 2019, вам потребуется изменить размещенную политику голосовой почты и убедиться, что поставщики хостинга настроены правильно. Дополнительные сведения см. в [дополнительных сведениях в службе Настройка облачной голосовой почты.](configure-cloud-voicemail.md)

- Если вы используете службу обмена данными Exchange на локальной основе или у вас есть набор пользователей, использующих exchange UM online и на локальной основе, вам потребуется изменить как размещенную политику голосовой почты, так и поставщика хостинга.  Дополнительные сведения см. в [дополнительных сведениях в службе Настройка облачной голосовой почты.](configure-cloud-voicemail.md)

- Для новой конфигурации облачной голосовой почты выполните действия, описанные в [службе Настройка облачной голосовой почты.](configure-cloud-voicemail.md)

В дополнение к вышеуказанным требованиям необходимо настроить ниже требования для подключения к службе голосовой почты Microsoft Cloud:

- Гибридное подключение. Если у вас уже развернут Skype для бизнеса Server, и вы хотите включить облачную голосовую почту для локального пользователя, необходимо убедиться, что между локальной и сетевой средами установлена гибридная связь. Иногда это называется разделенной конфигурацией домена.

   Дополнительные сведения см. в веб-сайте Plan [hybrid connectivity between Skype for Business Server и Microsoft 365 или Office 365,](plan-hybrid-connectivity.md) а также настройка гибридного подключения между Skype для бизнеса Server и [Office 365.](configure-hybrid-connectivity.md)

- Локальное приложение для пользователей должно быть включено для Корпоративная голосовая связь голосовой почты в Skype для бизнеса Server.

- Необходимо настроить URL-адрес и автообнаружить внешние веб-службы Exchange (EWS) или ограничить некоторые функции облачной голосовой почты.

- Если у вас есть локальное сервер Exchange, настройка облачной голосовой почты с помощью действий в Настройка облачной голосовой почты для [пользователей Exchange Server почтовых ящиков](/microsoftteams/set-up-phone-system-voicemail#set-up-cloud-voicemail-for-exchange-server-mailbox-users).

## <a name="migration-and-interoperability"></a>Миграция и интероперабельность

Если планируется развертывание Skype для бизнеса Server 2019 и/или Exchange Server 2019 г., необходимо тщательно планировать миграцию, чтобы обеспечить продолжение службы голосовых сообщений. Учитывайте следующее:

- Exchange Server 2019 г. больше не предоставляет функции Обмена обмена данными
- Skype для бизнеса Server 2019 больше не интегрируется с Exchange Online UM

В следующей таблице перечислены варианты интероперабельности и поддерживаемые топологии облачной голосовой почты, которая сравнивает версии Skype для бизнес-сервера, в которых пользователь может быть дома, с возможной версией, обеспечиваемой почтовым ящиком Exchange. Если вы хотите использовать Skype для бизнеса 2019 с Exchange Online или Exchange Server 2019, необходимо использовать облачную голосовую почту.

| | Exchange Server 2013 | Exchange Server 2016 | Exchange Server 2019 | Exchange Online   |
|:---    |:--- |:--- |:--- |:---  |
| Skype for Business Server 2019 | Exchange Server UM | Exchange Server UM | Облачная голосовая почта | Облачная голосовая почта |
| Skype для бизнеса Server 2015 | Exchange Server UM | Exchange Server UM | Не поддерживается | Облачная голосовая почта |
| Lync Server 2013 <br>  | Exchange Server UM | Exchange Server UM | Не поддерживается | Облачная голосовая почта |

Корпорация Майкрософт рекомендует следующие пути миграции:

- При обновлении до Skype для бизнеса Server 2019 вы можете использовать exchange UM в Exchange Server 2013 или 2016 гг., но при использовании Exchange Server 2019 г. необходимо перейти на облачную голосовую почту.
- Если вы обновляете Exchange Server 2019 г. и используете предыдущие версии системы обмена сообщениями Exchange Server для голосовых сообщений Skype для бизнеса Server, Корпорация Майкрософт рекомендует перейти на Skype для бизнеса Server 2019 до обновления почтовых ящиков.  В противном случае возможности голосовой передачи сообщений будут потеряны.
- При обновлении до Skype для бизнеса Server 2019 и настройке Skype для бизнеса Server 2015 для голосовой почты с помощью системы обмена голосовой почтой Exchange Online голосовая почта пользователей автоматически перемещается из exchange Online UM в облачную голосовую почту при перемещении учетной записи в Skype для бизнеса Server 2019. 

Дополнительные сведения о планировании миграции см. в веб-сведениях [Plan for Skype for Business Server и Exchange Server миграции.](plan-um-migration.md)
