# <a name="plan-cloud-voicemail-service-for-on-premises-users"></a>Планирование облачной службы голосовой почты для локальных пользователей

## <a name="overview"></a>Общие сведения

В этой статье описываются преимущества, рекомендации по планированию и требования для реализации облачной службы голосовой почты Майкрософт для локальных пользователей. Сведения о настройке облачной голосовой почты вы найдете в статье [Настройка облачной службы голосовой почты](configure-cloud-voicemail.md).

Облачная Голосовая почта использует единую систему обмена сообщениями Exchange, обеспечивая функции обмена голосовыми сообщениями для пользователей голосовой связи Skype для бизнеса 2019 с почтовыми ящиками в Exchange Server 2019 или Exchange Online. Облачная Голосовая почта предоставляет следующие преимущества для локальных пользователей и пользователей в сети:

- Функции ответа на голосовую почту и функции депозита с помощью расширенной речевой расшифровки

- Доступ к голосовой почте в почтовом ящике Exchange пользователя с помощью Skype для бизнеса Online или клиентов Outlook

- Возможность использования веб-портала Office 365 для управления параметрами голосовой почты

- Поддержка почтовых ящиков Exchange в локальной среде или в облаке

- Использование существующих приветствий пользователей единой системы обмена сообщениями Exchange Online

Для получения дополнительных сведений о сравнении функций, ознакомьтесь со статьей [Планирование миграции в Skype для бизнеса Server и Exchange Server](plan-um-migration.md).

Skype для бизнеса Server 2019 продолжает использовать единую систему обмена сообщениями Exchange для тех пользователей, чьи почтовые ящики находятся в предыдущих версиях Exchange Server (2013, 2016).  Сведения о том, какое решение голосовой почты будет использоваться на основе версии Exchange Server и Skype для бизнеса Server — важная часть планирования миграции в Skype для бизнеса Server 2019 или Exchange Server 2019. Дополнительные сведения о переносе и взаимодействии приведены в статье [Планирование переноса Skype для бизнеса Server и Exchange Server](plan-um-migration.md).

Благодаря облачной голосовой почте задачи администрирования значительно упрощаются по следующим причинам:

- Настраивать роль единой системы обмена сообщениями Exchange не требуется.
- Задачи настройки для облачной голосовой почты проще.
- Обновления функций голосовой почты доставляются непосредственно в облаке, поэтому пользователи всегда имеют доступ к последним функциям и обновлениям с меньшим зависимостью от накопительных обновлений (накопительные пакеты обновления).
- Вы используете один и тот же набор элементов управления для локальных и сетевых почтовых ящиков Exchange. Дополнительные сведения об этих элементах управления приведены в разделе [Настройка голосовой почты телефонной системы](https://support.office.com/en-us/article/Set-up-Phone-System-voicemail-Admin-help-9c590873-b014-4df3-9e27-1bb97322a79d?ui=en-US&rs=en-US&ad=US).

На следующей схеме показана облачная Голосовая почта в гибридном развертывании:

![Облачная Голосовая почта SfB](../../sfbserver2019/media/plan-cloud-voice-mail-server1.png)

Неотвеченные звонки обрабатываются следующим образом:  

1. Для пользователей, размещенных в Skype для бизнеса 2019 локально, неотвеченные звонки отправляются локальным сервером Skype для бизнеса в службу голосовой почты в Интернете.
2. Служба обрабатывает голосовую почту, в том числе транскрипцию.
3. Затем служба применяет голосовую почту в почтовом ящике Exchange пользователя независимо от того, находится ли почтовый ящик в локальной среде или в сети.  
4. Пользователи могут получать доступ к голосовой почте из своего клиента Skype для бизнеса или Outlook.

## <a name="requirements"></a>Requirements

В следующих требованиях предполагается, что у вас уже развернут Skype для бизнеса Server в поддерживаемой топологии.  Требования зависят от вашего сценария:

- Если вы уже используете единую систему обмена сообщениями Exchange Online и хотите выполнить обновление до Skype для бизнеса 2019, необходимо изменить политику размещенной голосовой почты и убедиться, что поставщики услуг размещения настроены правильно. Дополнительную информацию можно узнать в статье [Настройка облачной службы голосовой почты](configure-cloud-voicemail.md).

- При использовании единой системы обмена сообщениями Exchange на локальном сервере или при использовании единой системы обмена сообщениями Exchange Online и локальных пользователей необходимо изменить политику размещенной голосовой почты и поставщика услуг размещения.  Дополнительную информацию можно узнать в статье [Настройка облачной службы голосовой почты](configure-cloud-voicemail.md).

- Чтобы создать новую конфигурацию облачной голосовой почты, выполните действия, описанные в разделе [Настройка облачной службы голосовой почты](configure-cloud-voicemail.md).

В дополнение к приведенным выше требованиям необходимо настроить следующие требования для подключения к облачной службе голосовой почты Майкрософт:

- Гибридное подключение. Если вы уже развернули Skype для бизнеса Server и хотите включить облачную голосовую почту для локальных пользователей, необходимо убедиться в том, что гибридное подключение настроено между локальной средой и интернет-средой. Иногда это называется разделенной конфигурацией домена.

   Для получения дополнительных сведений ознакомьтесь со статьей [планирование гибридного подключения между Skype для бизнеса Server и office 365](plan-hybrid-connectivity.md) и [Настройка гибридного подключения между Skype для бизнеса server и Office 365](configure-hybrid-connectivity.md).

- Локальные пользователи должны быть включены для корпоративной голосовой связи и размещенной голосовой почты в Skype для бизнеса Server.

- Необходимо настроить внешний URL-адрес веб-службы Exchange (EWS) и службу автообнаружения, или некоторые функции голосовой голосовой почты будут ограничены.

- Если у вас есть только&#x2014;развертывания, то есть только локальные серверы Exchange и Skype для бизнеса&#x2014;но вы хотите использовать преимущества облачной голосовой почты, дополнительные лицензии не требуются.

## <a name="migration-and-interoperability"></a>Миграция и взаимодействие

При планировании развертывания Skype для бизнеса Server 2019 и/или сервера Exchange Server 2019 необходимо тщательно спланировать миграцию, чтобы обеспечить продолжение обслуживания голосовой почты. Учитывайте следующее:

- Exchange Server 2019 больше не поддерживает функции единой системы обмена сообщениями Exchange
- Skype для бизнеса Server 2019 больше не интегрируется с единой системой обмена сообщениями Exchange Online

Взаимодействие версий и поддерживаемые топологии для облачной голосовой почты приведены в следующей таблице, в которой сравниваются версии Skype для бизнеса Server, на которые может быть размещен пользователь, с помощью возможной версии, предоставляющей свои почтовые ящики Exchange. Облачная Голосовая почта работает только со Skype для бизнеса Server и Exchange Server 2019 или Exchange Online.

| Skype для бизнеса Server 2019 | Единая система обмена сообщениями Exchange Server | Единая система обмена сообщениями Exchange Server | Облачная Голосовая почта | Облачная Голосовая почта |
| Skype для бизнеса Server 2015 | Единая система обмена сообщениями Exchange Server | Единая система обмена сообщениями Exchange Server | Облачная Голосовая Почта<sup>1</sup> | Облачная Голосовая почта <br> Служба единой системы обмена сообщениями Exchange Online<sup>2</sup> |

Корпорация Майкрософт рекомендует следующие пути миграции:

- При обновлении до Skype для бизнеса Server 2019 вы можете использовать единую систему обмена сообщениями Exchange в Exchange Server 2013 или 2016, но при использовании Exchange Server 2019 необходимо выполнить обновление до облачной голосовой почты.
- Если вы выполняете обновление до Exchange Server 2019 и используете предыдущие версии единой системы обмена сообщениями Exchange Server для голосовой почты Skype для бизнеса Server, корпорация Майкрософт рекомендует перед обновлением почтовых ящиков обновить приложение до Skype для бизнеса Server 2019.  В противном случае возможности обмена голосовыми сообщениями будут потеряны.
- При обновлении до Skype для бизнеса Server 2019 и использовании Skype для бизнеса Server 2015, настроенного для голосовой почты с Exchange Online единой системы обмена сообщениями, голосовая почта пользователя автоматически переносится из единой системы обмена сообщениями Exchange Online в облачную голосовую почту при перемещении учетной записи в Skype для бизнеса Server 2019. 

Для получения дополнительных сведений о планировании миграции, ознакомьтесь со статьей [Plan for Skype для бизнеса Server и Exchange Server Migration](plan-um-migration.md).
