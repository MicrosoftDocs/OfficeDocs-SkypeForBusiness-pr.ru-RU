# <a name="configure-resource-accounts"></a>Настройка учетных записей ресурсов

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

Skype для бизнеса Server 2019 г. гибридные реализации используют только облачные службы, предоставляемые телефонная система для единой системы обмена сообщениями, и не интегрируются с Exchange Online. В Skype для бизнеса Server 2019 г. вы можете использовать очереди вызовов и автовласти Облака, описанные в ниже, что вы получаете с телефонная система в Microsoft 365 или [Office 365](/MicrosoftTeams/here-s-what-you-get-with-phone-system).

Чтобы использовать телефонная система или очередь вызова Skype для бизнеса Server 2019, необходимо создать учетные записи ресурсов, которые выступают в качестве конечных точек приложения и могут быть назначены номера телефонов, а затем использовать центр администрирования Teams для настройки очереди вызова или автоспутника. Эта учетная запись ресурса может быть домашней в Интернете (см. в статье Управление учетными записями ресурсов в [Microsoft Teams](/MicrosoftTeams/manage-resource-accounts) для создания учетных записей ресурсов в Интернете) или в помещениях, описанных в этой статье. Как правило, у телефонная система автопроизводицы или узлы очереди вызова, каждый из которых соотносят с учетными записями ресурсов, которые можно использовать в Интернете или Skype для бизнеса Server 2019 г.

Если у вас есть существующая Exchange и система очередей на вызовы, перед переходом на Exchange Server 2019 или Exchange в Интернете необходимо вручную записывать сведения, описанные ниже, а затем внедрять совершенно новую систему с помощью центра администрирования Teams.

## <a name="overview"></a>Обзор

Если вашему телефонная система или очереди вызовов потребуется номер службы, различные зависимости могут быть выполнены в следующей последовательности:

1. Получение номера службы.
2. Получение бесплатной телефонная система [лицензии](/MicrosoftTeams/teams-add-on-licensing/virtual-user) виртуального пользователя или телефонная система лицензии для использования с учетной записью ресурса.
3. Создание учетной записи ресурса. Автоспутник или очередь вызова должны иметь связанную учетную запись ресурса.
4. Подождите, пока не будет синхронизирована активная синхронизация каталогов между онлайн и локально.
5. Назначить телефонная система лицензию на учетную запись ресурса.
6. Назначить номер службы учетной записи ресурса.
7. Создание очереди телефонная система вызова или автобомки.
8. Связать учетную запись ресурса с автопутессой или очередью вызовов: (New-CsApplicationInstanceAssociation).

Если автопутессница или очередь вызова вложена под автоспутник верхнего уровня, для связанной учетной записи ресурса требуется только номер телефона, если требуется несколько точек входа в структуру автоспутников и очередей вызовов.

Чтобы перенаправить вызовы для людей в вашей организации, которые находятся в Интернете, они должны иметь лицензию телефонная система и быть включена для Корпоративная голосовая связь или иметь Microsoft 365 или Office 365 телефонные планы.  См. [Microsoft Teams лицензии.](/MicrosoftTeams/assign-teams-licenses) Чтобы включить их для Корпоративная голосовая связь, вы можете использовать Windows PowerShell. Например, выполнить:  `Set-CsUser -identity "Amos Marble" -EnterpriseVoiceEnabled $true`

Если Телефон или создаваемая очередь вызовов будет вложена и не потребуется номер телефона, процесс:

1. Создание учетной записи ресурса  
2. Ожидание синхронизации активного каталога между интернетом и локальной сетью
3. Создание телефонная система или очереди вызова
4. Связать учетную запись ресурса с телефонная система или очередью вызовов

## <a name="create-a-resource-account-with-a-phone-number"></a>Создание учетной записи ресурса с номером телефона

Создание учетной записи ресурса, использующей номер телефона, потребует выполнения следующих задач в следующем порядке:

1. Порт или получить платный или бесплатный номер службы. Номер не может быть назначен другим голосовым службам или учетным записям ресурсов.

   Прежде чем назначить номер телефона учетной записи ресурса, необходимо получить или перепортить существующие номера платных или бесплатных служб. После получения платных или бесплатных номеров телефонов службы они будут Microsoft Teams номера центра администрирования Voice Телефон, а указанный тип номера будет указан как  >    >  Service **- Toll-Free**.  Чтобы получить номера службы, см. в статью [Получение](/MicrosoftTeams/getting-service-phone-numbers) номеров телефонов службы или передачу существующего номера службы см. в [Teams.](/MicrosoftTeams/phone-number-calling-plans/transfer-phone-numbers-to-teams)

   Если вы находитесь за пределами США, вы не можете использовать центр администрирования Microsoft Teams для получения номеров служб. Перейдите [к управлению телефонными](/MicrosoftTeams/manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization) номерами для организации, чтобы узнать, как это сделать за пределами США.

2. Купить лицензию телефонная система. См. следующие статьи:  
   - [телефонная система-виртуальная лицензия пользователя](/MicrosoftTeams/teams-add-on-licensing/virtual-user)
   - [Office 365 корпоративный E1 и E3](/MicrosoftTeams/teams-add-on-licensing/office-365-enterprise-e1-e3)
   - [Office 365 корпоративный E5](/MicrosoftTeams/teams-add-on-licensing/office-365-enterprise-e5-with-audio-conferencing)
   - [Office 365 корпоративный Программное обеспечение для бизнеса E5](https://products.office.com/business/office-365-enterprise-e5-business-software)

3. Создайте учетную запись локального ресурса, создав комлет для телефонная система или очереди вызова, и дайте каждому из них имя, SIP-адрес и так `New-CsHybridApplicationEndpoint` далее.

    ``` Powershell
    New-CsHybridApplicationEndpoint -ApplicationID <GUID> -DisplayName appinstance01 -SipAddress sip:appinstance01@contoso.com -OU "ou=Redmond,dc=litwareinc,dc=com"
    ```

    Дополнительные сведения об этой команде см. в материале [New-CsHybridApplicationEndpoint.](/powershell/module/skype/new-cshybridapplicationendpoint?view=skype-ps)

4. (Необязательный) После создания учетных записей ресурсов можно либо дождаться синхронизации AD между онлайн и на месте, либо принудить к синхронизации и перейти к он-лайн конфигурации телефонная система или очередей вызовов. Чтобы выполнить синхронизацию, на компьютере под управлением AAD Подключение (если вы этого еще не сделали, потребуется загрузить для запуска `import-module adsync` команды):

    ``` Powershell
    Start-ADSyncSyncCycle -PolicyType Delta
    ```

    Дополнительные сведения об этой команде см. в материале [Start-ADSyncSyncCycle.](/azure/active-directory/connect/active-directory-aadconnectsync-feature-scheduler)
    
    Примечание. На данном этапе учетная запись может синхронизироваться, но подготовка может быть не завершена.  Проверьте выход [Get-CsOnlineApplicationEndpoint](/powershell/module/skype/get-csonlineapplicationendpoint).  Если синхронизированная конечная точка еще не завершила подготовка, она не появится здесь.  Вы можете проверить состояние запросов на подготовка на портале M365 в соответствии с Teams [состояние установки](https://admin.microsoft.com/AdminPortal/Home#/teamsprovisioning).  Этот этап подготовка может занять до 24 часов.

5. Назначить телефонная система - виртуальный пользователь или телефонная система лицензию на учетную запись ресурса. См. [Microsoft Teams надстройки](/MicrosoftTeams/teams-add-on-licensing/assign-teams-add-on-licenses) и [присвоение лицензий пользователям.](/microsoft-365/admin/manage/assign-licenses-to-users)

   Если вы назначаете номер телефона учетной записи ресурса, вы можете использовать бесплатную телефонная система - лицензию виртуального пользователя. Это предоставляет телефонная система телефонным номерам на организационном уровне и позволяет создавать возможности автоотвода и очереди на вызовы.


6. Назначить номер службы учетной записи ресурса. Используйте команду для назначения номера телефона (с параметром `Set-CsHybridApplicationEndpoint` -LineURI) в учетную запись ресурса.

    ``` Powershell
    Set-CsHybridApplicationEndpoint -Identity appinstance01@contoso.com -LineURI tel:+14255550100
    ```

    Дополнительные сведения об этой команде см. в материале [Set-CsHybridApplicationEndpoint.](/powershell/module/skype/set-cshybridapplicationendpoint?view=skype-ps)

    Чтобы назначить учетной записи ресурса прямой маршрут или гибридный номер, используйте следующий кодлет:

   ``` Powershell
   Set-CsOnlineApplicationInstance -Identity appinstance01@contoso.com -OnpremPhoneNumber +14250000000
   ```

   Учетной записи ресурса потребуется присвоение номера телефона, если он будет назначен автопроводнику верхнего уровня или очереди вызова. Номера телефонов пользователей (абонентов) не могут быть назначены учетной записи ресурса, можно использовать только платные или бесплатные номера телефонов.

     Для учетной записи ресурса можно назначить прямой маршрут или гибридный номер. Подробные сведения см. [в материале Plan Direct Routing](/MicrosoftTeams/direct-routing-plan) and [Plan Cloud auto attendants.](plan-cloud-auto-attendant.md)

     > [!NOTE]
     > Номера служб прямой маршрутной маршрутики, присвоенные учетным записям ресурсов для автопроизводителей и очереди на вызовы, поддерживаются только для Microsoft Teams пользователей и агентов.

7. Создайте телефонная система или очередь вызова. Выберите один из указанных ниже вариантов.

   - [Настройка облачного автосекретаря](/MicrosoftTeams/create-a-phone-system-auto-attendant)
   - [Создание облачной очереди вызовов](/MicrosoftTeams/create-a-phone-system-call-queue)  

8. Связать учетную запись ресурса с телефонная система или очередью вызовов, выбранной ранее.

## <a name="create-a-resource-account-without-a-phone-number"></a>Создание учетной записи ресурса без номера телефона

Эти действия необходимы независимо от того, создаете ли вы новую телефонная система или структуру очереди вызовов или структуру восстановления, изначально созданную в Exchange единой системы единой системы Exchange единой системы управления.

Войдите на Skype для бизнеса переднего сервера и запустите следующие cmdlets PowerShell:

1. Создайте учетную запись локального ресурса, создав комлет для телефонная система или очереди вызова, и дайте каждому из них имя, SIP-адрес и так `New-CsHybridApplicationEndpoint` далее.

2. (Необязательный) После создания учетных записей ресурсов можно либо дождаться синхронизации AD между онлайн и на месте, либо принудить к синхронизации и перейти к он-лайн конфигурации телефонная система или очередей вызовов. Чтобы выполнить синхронизацию, на компьютере под управлением AAD Подключение (если вы этого еще не сделали, потребуется загрузить для запуска `import-module adsync` команды):

3. Создайте телефонная система или очередь вызова. Выберите один из указанных ниже вариантов.
   - [Настройка облачного автосекретаря](/MicrosoftTeams/create-a-phone-system-auto-attendant)

## <a name="test-the-implementation"></a>Тестирование реализации

Лучший способ проверить реализацию — вызвать номер, настроенный для телефонная система или очереди вызова, и подключиться к одному из агентов или меню. Вы также можете быстро разместить тестовый вызов с помощью кнопки **Test** в области действий центра администрирования. Если вы хотите внести изменения в телефонная система или очередь вызова, выберите его и нажмите кнопку Изменить в области **действий.** 

> [!TIP]
> Если у вашей учетной записи ресурса возникли проблемы с присвоением очереди вызова [](https://techcommunity.microsoft.com/t5/Microsoft-Teams-Blog/Auto-Attendant-and-Call-Queues-Service-Update/ba-p/564521) или автоспутнику, см. в разделе Известные проблемы для [Microsoft Teams](/MicrosoftTeams/Known-issues#phone-system) и раздел "Исправление экземпляров гибридных приложений" в разделе Microsoft Teams Blog.

## <a name="moving-an-exchange-um-auto-attendant-or-call-queue-to-phone-system"></a>Перемещение Exchange или очереди вызова в телефонная система

Для переноса Exchange в телефонная система потребуется воссоздание очереди вызовов и структуры автоспутника, непосредственно перенос из одного в другой не поддерживается. Чтобы повторно реализовать набор очередей вызовов и автоспутников:

1. Получите список всех Exchange и очередей вызовов, выстроив следующую команду в системе Exchange 2013 или 2016 г., во время входа в систему в качестве администратора:

    ``` Powershell
    Get-UMAutoAttendant | Format-List
    ```

2. Для каждой из перечисленных Exchange очередей вызовов или автоспутника, обратите внимание на его место в структуре, параметрах и получите копии связанных звуковых или текстовых файлов (guid в выходе будет именем папки, в которой хранятся файлы). Эти сведения можно получить, запуская команду:

    ``` Powershell
    Get-UMAutoAttendant -Identity MyUMAutoAttendant
    ```

    Дополнительные сведения об этой команде см. в материале [Get-UMAutoAttendant.](/powershell/module/exchange/unified-messaging/get-umautoattendant?view=exchange-ps) Полный список параметров, которые необходимо зафиксировать, находится в [членах UMAutoAttendant,](/previous-versions/office/exchange-server-api/ff340649(v=exchg.150)) но наиболее важные параметры, которые следует отметить, это:

    - Рабочие часы
    - Нестандартные часы
    - Язык
    - Расписание праздников.

3. Создайте новые конечные точки локального назначения, как описано ранее.
   Назначьте автопроводнику верхнего уровня временный номер для целей тестирования.

4. Настройка автобомки телефонная система или очереди вызовов, которая использует конечные точки, как описано ранее.

5. Проверьте телефонная система или очередь вызова.

6. Перенастройка номера телефона, связанного с Exchange очереди вызова или автопроводника к соответствующему телефонная система или очереди вызова.  

   На данный момент, если вы уже мигрировали голосовую почту um, вы должны быть в состоянии перейти на Exchange Server 2019.
