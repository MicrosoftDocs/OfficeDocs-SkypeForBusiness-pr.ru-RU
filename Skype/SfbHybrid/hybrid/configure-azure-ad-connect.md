# <a name="configure-azure-ad-connect-for-teams-and-skype-for-business"></a>Настройка Azure AD Connect для Teams и Skype для бизнеса

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

 
Чтобы использовать Teams, организации с локальной развертыванием Skype для бизнеса Server или Lync Server 2013 должны настроить Azure AD Подключение для синхронизации локального каталога с Microsoft 365. Организации с Skype для бизнеса Server должны обеспечить синхронизацию соответствующих атрибутов msRTCSIP в Azure AD. В этой статье любая ссылка на "Skype для бизнеса Server" также применяется к Lync Server 2013.

> [!NOTE]
> - Чтобы получить полную функциональность, Teams пользователям с Skype для бизнеса локальной учетной записью необходимо Skype для бизнеса локальной учетной записи. Например, для получения таких функций, как возможность взаимодействия с Skype для бизнеса пользователями, а также для общения с пользователями в федерадных организациях. Если локального пользователя будет использовать только Teams, необходимо переместить пользователя в облако, чтобы предоставить полный Teams функций в качестве пользователя TeamsOnly. Чтобы эта миграция состоялась, необходимо настроить Azure AD Подключение, чтобы можно было включить гибрид.
> - Если вы не планируете в ближайшее время перемещать пользователей из локального облака в облако, необходимо настроить Azure AD Подключение, чтобы Teams и Skype для бизнеса Server учетные записи.
 

## <a name="background-information"></a>Базовые сведения

Azure Active Directory Подключение поддерживает непрерывное синхронизацию локального Active Directory с Microsoft 365. Локальное каталог остается авторитетным источником удостоверений, а изменения локальной среды синхронизируются с Azure AD. Дополнительные сведения см. в [Подключение Azure AD.](/azure/active-directory/hybrid/how-to-connect-sync-whatis)  Пользователи в вашей организации будут представлены как в локальном, так и *в онлайн-каталоге.*  Чтобы обеспечить сосуществование этих учетных записей, все пользователи, Teams или Skype для бизнеса локально, должны быть синхронизированы из локального в Azure AD. Кроме того, вы можете упростить связь между локальной и интернет-пользователями с помощью Skype для бизнеса гибридного подключения, что также требует конфигурации Azure AD Подключение.


## <a name="configuring-azure-ad-when-you-have-skype-for-business-server"></a>Настройка Azure AD для работы со Skype для бизнеса Server 

### <a name="general-requirements"></a>Общие требования 

Чтобы локальное развертывание Skype для бизнеса Server сосуществовать с Teams, некоторые атрибуты Active Directory из локального развертывания должны быть синхронизированы в Azure AD с помощью Azure AD Подключение. Настройка azure AD Подключение автоматически настраивает необходимые атрибуты для синхронизации по умолчанию при обнаружении Skype для бизнеса Server в локальной среде. Эти атрибуты включают общие атрибуты удостоверений, такие как основное имя пользователя, а также атрибуты, предисловие к "msRTCSIP", которые относятся к Skype для бизнеса Server. Полный набор атрибутов перечислены в [Azure AD Подключение синхронизации:](/azure/active-directory/hybrid/reference-connect-sync-attributes-synchronized#teams-and-skype-for-business-online)атрибуты синхронизируются с Azure Active Directory .

Если вы решите настроить параметры синхронизации в Azure AD Подключение, необходимо обеспечить синхронизацию следующих атрибутов для объектов пользователя:
</br>

|Атрибут|Описание|
|---|---|
|msRTCSIP-PrimaryUserAddress| SIP-адрес пользователя в локальной среде|
|msRTCSIP-DeploymentLocator| Указывает, находится ли пользователь в локальном помещении или в облаке.|
|msRTCSIP-UserEnabled| Включен ли пользователь для функции SIP|
|||

</br>
</br>
Кроме того, если вы управляете атрибутами телефонной системы с помощью локального развертывания, необходимо также синхронизировать следующие атрибуты:

|Атрибут|Описание|
|:---|:---|
|msRTCSIP-Line| Номер телефона пользователя|
|msRTCSIP-OptionFlags| Указывает, включен ли пользователь для голосовой функции|
|msRTCSIP-OwnerUrn| Используется для определения конечных точек гибридного приложения|
|||

</br>
Корпорация Майкрософт рекомендует синхронизировать все леса, содержащие удостоверения пользователей, а также все леса, содержащие Skype для бизнеса Server. Если удостоверения пользователей существуют в нескольких лесах, необходимо выполнить объединение с помощью Azure AD Connect. При соблюдении этих инструкций Azure AD Connect автоматически синхронизирует правильные атрибуты, при условии что вы не меняете соединители или правила синхронизации в Azure AD Connect. 
  
Если вы не синхронизируете все леса, содержащие удостоверения пользователей и развертывание Skype для бизнеса Server, необходимо убедиться, что соответствующие удостоверения и атрибуты Skype для бизнеса правильно заполняются в Azure AD для любого пользователя, используюющего Teams или Skype для бизнеса (будь то локально или локально) он-лайн). Для этого, скорее всего, потребуется дополнительная синхронизация локального каталога. Дополнительные сведения см. в [Подключение Azure AD: атрибуты, синхронизированные с Azure Active Directory.](/azure/active-directory/hybrid/reference-connect-sync-attributes-synchronized)

Клиент несет ответственность за обеспечение правильной конфигурации для заполнения атрибутов в Azure AD. Учитывайте следующее: 

- Использование нестандартной конфигурации для синхронизации с Azure AD является рискованным. Нестандартные конфигурации могут привести к повреждениям данных в каталоге в Интернете.

- По мере развития продуктов корпорация Майкрософт продолжит проверять стандартные конфигурации синхронизации, в которых синхронизируются все соответствующие леса. Клиенты с пользовательскими конфигурациями синхронизации обязаны гарантировать, что их конфигурации доставляют правильные атрибуты и значения в Azure AD. 

Независимо от того, имеется ли у вас один локальной лес Active Directory или несколько лесов, Azure AD Подключение можно использовать в различных поддерживаемых topologies, как описано в [topologies для Azure AD Подключение](/azure/active-directory/hybrid/plan-connect-topologies). С точки зрения Skype для бизнеса Server существует три варианта: 

1. Один лес, содержащий полномочные удостоверения пользователей и Skype для бизнеса Server. 

2. Несколько лесов, в одном из которых размещается Skype для бизнеса Server, а также один или несколько других лесов, содержащих полномочные удостоверения пользователей (лесов учетных записей). 

3. Несколько развертываний Skype для бизнеса Server в нескольких лесах. При условии, что определенные требования будут выполнены, организации смогут консолидировать эти несколько развертывание в одну Microsoft 365 организацию.

### <a name="single-forest"></a>Один лес 

Если все учетные записи пользователей и Skype для бизнеса размещены в одном лесу, необходимо убедиться, что настроено средство Azure AD Connect, чтобы синхронизировать пользователей из этого леса с Azure AD.  По умолчанию соответствующие атрибуты будут автоматически синхронизированы с Azure Active Directory. Клиентам рекомендуется не изменять встроенные правила синхронизации и/или соединители, которые управляют конфигурацией (что невозможно от мастера установки).  

### <a name="multiple-forests-with-one-skype-for-business-deployment"></a>Несколько лесов с одним развертыванием Skype для бизнеса 

Этот сценарий часто называют топологией леса ресурсов. Полномочные удостоверения пользователей размещаются в одном или нескольких лесах учетных записей, а Skype для бизнеса развертывается в отдельном лесу ресурсов (в котором также могут размещаться полномочные удостоверения пользователей). В общем случае полномочные удостоверения пользователей Skype для бизнеса могут находиться в том же лесу, что и Skype для бизнеса Server, и/или в другом лесу при соблюдении указанных ниже условий. 

- Пользователи с авторитетными удостоверениями из леса учетных записей представлены в лесу ресурсов (Skype для бизнеса Server развернуты) в качестве объектов отключенных пользователей. Атрибут msRTCSIP-OriginatorSID в лесу ресурсов должен соответствовать SID в лесу учетных записей. Дополнительные сведения [см. в материале Configure a multi-forest environment for hybrid Skype для бизнеса.](configure-a-multi-forest-environment-for-hybrid.md)

- Лес ресурсов, в котором размещается Skype для бизнеса Server, доверяет лесам учетных записей.  

- Все соответствующие пользовательские объекты и атрибуты для удостоверений (из лесов учетных записей) и Skype для бизнеса (из леса ресурсов) синхронизируются в Azure AD с правильными значениями через Azure AD Подключение.  

  Чтобы добиться правильной синхронизации объектов и [](configure-a-multi-forest-environment-for-hybrid.md)атрибутов в Azure AD в локальном сценарии с несколькими лесами, Корпорация Майкрософт настоятельно рекомендует использовать Azure AD Подключение для синхронизации всех лесов, включив учетные записи пользователей и лес, содержащий Skype для бизнеса. Если выполняется синхронизация из всех лесов, далее необходимо настроить Azure AD Connect для объединения этих удостоверений и их синхронизации в Azure AD. Средство Azure AD Connect предназначено для работы в этом сценарии, а в мастере установки предусмотрен встроенный параметр для его настройки, включая настройку привязок для объединения удостоверений. Выберите **следующее:** идентификаторы пользователей существуют в нескольких каталогах и совпадают с атрибутами **--> ObjectSID и msExchangeMasterAccountSID.**


### <a name="multiple-skype-for-business-server-deployments-in-multiple-forests"></a>Несколько развертываний Skype для бизнеса Server в нескольких лесах 

В этом сценарии существует несколько лесов, каждый из которых содержит Skype для бизнеса Server и одну Microsoft 365 организацию. Каждый лес, содержащий Skype для бизнеса Server, можно синхронизировать в Azure AD для этой организации с помощью Azure AD Подключение. В любой момент для гибридной среды Skype для бизнеса может быть настроен только один лес. Перед включением гибрида в лесу все домены SIP из всех других лесов должны быть отключены с помощью [отключения-csonlineSipDomain.](/powershell/module/skype/disable-csonlinesipdomain) Дополнительные сведения см. в см. в Teams [и Skype для бизнеса.](cloud-consolidation.md)
