# <a name="configure-cloud-voicemail-service-for-on-premises-users"></a>Настройка облачная голосовая почта для локального пользователя

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]


## <a name="overview"></a>Обзор 
В этой статье описывается настройка голосовая облачная почта (Майкрософт) службы для Skype для бизнеса локального пользователя.  

В этой статье предполагается, что Skype для бизнеса Server развернуты в поддерживаемой топологии, и что вы выполнили необходимые условия для настройки гибридного подключения.

Дополнительные сведения о преимуществах, соображениях планирования и требованиях к реализации облачная голосовая почта см. в облачная голосовая почта [Plan облачная голосовая почта.](plan-cloud-voicemail.md)




Настройка облачная голосовая почта включает в себя следующие задачи:

1.  Убедитесь, что вы выполнили необходимые условия, описанные в службе [Plan облачная голосовая почта.](plan-cloud-voicemail.md)

2.  Убедитесь, что вы настроили гибридное подключение, как описано в [plan hybrid connectivity](plan-hybrid-connectivity.md) и [Configure hybrid connectivity.](configure-hybrid-connectivity.md) 

3.  [Настройка облачная голосовая почта как поставщика хостинга](#configure-cloud-voicemail-as-the-hosting-provider) на переднем сервере, как описано в этой статье.

4.  [Настройка политики голосовой почты, как](#configure-a-hosted-voicemail-policy) описано в этой статье.

5.  [Назначение политики голосовой почты,](#assign-a-hosted-voicemail-policy) как описано в этой статье.

6.  [Включить пользователя для облачная голосовая почта,](#enable-a-user-for-cloud-voicemail) как описано в этой статье.


## <a name="configure-cloud-voicemail-as-the-hosting-provider"></a>Настройка облачная голосовая почта как поставщика хостинга 

Вы настраиваете облачная голосовая почта как поставщика хостинга на переднем сервере с помощью New-CsHostingProvider с помощью следующих параметров:

- **Identity** указывает уникальный идентификатор строковых значений для создающегося поставщика хостинга; например, облачная голосовая почта. 

- Параметр **Enabled** указывает, разрешено ли сетевое подключение между вашим доменом и поставщиком услуг размещения. Этот параметр должен быть задан true.

- **EnabledSharedAddressSpace** определяет, используется ли поставщик услуг размещения в общем адресном пространстве SIP. Этот параметр должен быть задан true.

- **HostsOCSUsers** указывает, используется ли поставщик хостинга для размещения Skype для бизнеса Server учетных записей. Этот параметр должен быть задан false.

- **ProxyFQDN** указывает полное доменное имя (FQDN) для прокси-сервера, используемого поставщиком хостинга; например, proxyserver.contoso.com. Чтобы получить эту информацию, обратитесь к своему поставщику услуг размещения. Это значение нельзя изменить. Если поставщик хостинга меняет прокси-сервер, необходимо удалить и повторно создать запись для этого поставщика.

- **IsLocal** указывает, содержится ли прокси-сервер, используемый поставщиком хостинга, в топологии Skype для бизнеса Server. Этот параметр должен быть задан false.

Например, в оболочке Skype для бизнеса управления следующий облачная голосовая почта в качестве поставщика хостинга:


```PowerShell
New-CsHostingProvider -Identity "Exchange Online" -Enabled $True -EnabledSharedAddressSpace $True -HostsOCSUsers $False -ProxyFqdn "exap.um.outlook.com" -IsLocal $False -VerificationLevel UseSourceVerification
```

## <a name="configure-a-hosted-voicemail-policy"></a>Настройка политики голосовой почты с хост-почтой

Чтобы обеспечить маршрутизации голосовой почты для организации в службу облачная голосовая почта, необходимо настроить политику голосовой почты для организации. Во многих случаях требуется только одна политика голосовой почты, и вы можете изменить глобальную политику для удовлетворения всех ваших потребностей. Если в организации требуется несколько политик голосовой почты, можно добавить политики с помощью нового cmdlet cshostedvoicemailpolicy.

Чтобы изменить глобальную политику, запустите следующую команду в оболочке управления Skype для бизнеса Server после обновления вашей Организации и TenantID:

```PowerShell
Set-CsHostedVoicemailPolicy -Identity Global -Description "Global Cloud Voicemail Policy" -Destination exap.um.outlook.com -Organization YourDefaultDomain.onmicrosoft.com
```

- **Назначение** указывает полное доменное имя (FQDN) принимающей облачная голосовая почта службы. Это значение должно быть заданной **для exap.um.outlook.com**.

- **Организация** — это домен по умолчанию, назначенный вашему клиенту. Эту информацию можно получить, office.com администратора клиента, щелкните приложение Центр администрирования,  перейдите к установке слева и щелкните **Домены**. Например: mytenant.onmicrosoft.com.

    Имя Организации также является доменным именем по умолчанию в Microsoft 365 или Office 365.

Чтобы убедиться, что политика голосовой почты с хост-почтой создана успешно, запустите следующую команду:

```PowerShell
Get-CsHostedVoicemailPolicy
```

## <a name="assign-a-hosted-voicemail-policy"></a>Назначение политики голосовой почты с хост-почтой

По умолчанию политика голосовой почты глобального хостинга назначена всем пользователям. Если используется другая политика, прежде чем включить пользователей для принимающей голосовой почты, сначала необходимо предоставить пользователям желаемую политику голосовой почты с помощью cmdlet [Grant-CSHostedVoicemailPolicy.](/powershell/module/skype/grant-cshostedvoicemailpolicy?view=skype-ps)

Например, следующая команда назначает пользователю политику голосовой почты, не относящаяся к глобальной сети:


```PowerShell
Get-CsUser -Identity "User1" | Grant-CsHostedVoicemailPolicy -PolicyName "Tag:CloudVoiceMailUsers" 
```

## <a name="enable-a-user-for-cloud-voicemail"></a>Включить пользователя для облачная голосовая почта

Чтобы включить маршрутные вызовы голосовой почты пользователя в облачная голосовая почта, используйте комлет [Set-CsUser](/powershell/module/skype/set-csuser?view=skype-ps) с параметром HostedVoiceMail. 

Например, следующая команда включает учетную запись пользователя для облачная голосовая почта: 

```powershell
Set-CsUser -Identity "User1" -HostedVoiceMail $True
```

В этом кодлете проверяется, что облачная голосовая почта на глобальном, веб-сайте или уровне пользователя применяется к этому пользователю. Если политика не применяется, то командлет завершается неудачно.  

В следующем примере отключает учетную запись пользователя для облачная голосовая почта:

```powershell
Set-CsUser -Identity "User1" -HostedVoiceMail $False
```

В этом кодлете проверяется, что к этому пользователю не применяется политика голосовой почты на глобальном уровне, на сайте или на уровне пользователя. Если политика применяется, то командлет завершается неудачно.

> [!NOTE]
>  Пользователи должны иметь корпоративный голос, чтобы использовать службу голосовая облачная почта (Майкрософт).
