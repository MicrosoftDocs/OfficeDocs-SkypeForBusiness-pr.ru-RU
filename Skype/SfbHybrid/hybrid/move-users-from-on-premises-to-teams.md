# <a name="move-users-from-on-premises-to-teams"></a>Перемещение пользователей из локальной среды в Teams

Когда пользователь перемещается только из локальной организации в Teams, Домашняя страница пользователя Skype для бизнеса перемещается из локальной сети в сеть, а пользователь назначается Теамсупградеполици с Mode = Теамсонли.  После того как пользователь перемещается из локальной в режим Теамсонли, выполните указанные ниже действия.

- Все входящие звонки и разговоры других пользователей (от Skype для бизнеса или Microsoft Teams) будут направлены на клиента Teams пользователя.
- Пользователь сможет взаимодействовать с другими пользователями Skype для бизнеса (как в Интернете, так и в локальной среде).
- Пользователь сможет общаться с пользователями в федеративных организациях.
- Новые собрания, запланированные этим пользователем, являются собраниями Teams.
- Пользователь все еще может присоединиться к собраниям Skype для бизнеса.
- Уже существующие собрания, запланированные для будущего пользователя, будут перенесены из локальной версии в Teams.
- Контакты, которые существовали локально, доступны в Teams вскоре после первого входа пользователя в систему.
- Пользователи не могут инициировать звонки и разговоры из Skype для бизнеса, а также не могут планировать новые собрания в Skype для бизнеса. При попытке открыть клиент Skype для бизнеса они будут перенаправлены для использования Teams, как показано ниже. Если клиент Teams не установлен, он будет перенаправлен на веб-версию Teams в Teams с помощью браузера.<br><br>
    ![Сообщение перенаправление пользователя в Teams](../media/go-to-teams-page.png)

Прежде чем перемещать пользователей, обязательно изучите [необходимые условия](move-users-between-on-premises-and-cloud.md#prerequisites) для перемещения пользователей в облако. Кроме того, ознакомьтесь с [рекомендациями по миграции и взаимодействию для организаций, использующих Teams вместе со Skype для бизнеса](/microsoftteams/migration-interop-guidance-for-teams-with-skype).


> [!NOTE]
> Единое хранилище контактов необходимо отключить в локальной учетной записи SfB для перемещения контакта в Teams.


Существует два способа перемещения пользователя из локальной в Teams.

- Если вы используете более раннюю версию, чем Skype для бизнеса Server 2015 8, для перемещения необходимо выполнить два действия (при необходимости можно выполнить скрипты):
  - [Переместите пользователя из Skype для бизнеса Server (локально) в Skype для бизнеса Online](move-users-from-on-premises-to-skype-for-business-online.md).
  - Когда пользователь размещен в Skype для бизнеса Online, назначьте ему пользователя Теамсупградеполици с режимом = Теамсонли. Чтобы предоставить режим Теамсонли, выполните следующий командлет в окне PowerShell для Skype для бизнеса Online:`Grant-CsTeamsUpgradePolicy -Identity $user -PolicyName UpgradeToTeams`
- Если у вас есть средства администрирования из Skype для бизнеса Server 2015 8 или более поздней версии, вы можете воспользоваться приведенным выше способом или выполнить это перемещение, как описано ниже. Кроме того, вы можете указать уведомление в клиенте Skype для бизнеса, прежде чем перемещать их только в Teams, а также при необходимости автоматически скачать клиент Microsoft Teams клиентом Skype для бизнеса.

## <a name="move-a-user-directly-from-skype-for-business-on-premises-to-teams-only"></a>Перемещение пользователя непосредственно из Skype для бизнеса локально в Teams

Локальные средства администрирования в Skype для бизнеса Server 2015 с 8, а также в Skype для бизнеса Server 2019 позволяют перемещать пользователей из локальной среды в режим "только для Teams" за один шаг с помощью командлета Move-CsUser в PowerShell или панели управления Skype для бизнеса Server, как описано ниже.

### <a name="move-to-teams-using-move-csuser"></a>Перемещение в Teams с помощью Move — CsUser

Move-CsUser доступен из локального окна PowerShell для командной консоли Skype для бизнеса. Действия, необходимые ниже и разрешения, совпадают с перемещением пользователя в Skype для бизнеса Online, за исключением того, что необходимо также указать параметр Моветотеамс, и необходимо убедиться, что пользователю также предоставлена лицензия для Teams (в дополнение к Skype для бизнеса Online).

У вас должны быть достаточные права как в локальной среде, так и в организации Office 365, как описано в разделе [обязательные административные учетные данные](move-users-between-on-premises-and-cloud.md#required-administrative-credentials). Можно использовать одну учетную запись с правами в обеих средах или можно запустить локальное окно командной консоли Skype для бизнеса Server с локальными учетными данными и использовать `-Credential` параметр для указания учетных данных для учетной записи Office 365 с необходимой административной ролью Office 365.

Перемещение пользователя в режим "только Teams" с помощью команды Move — CsUser:

- Укажите пользователя для перемещения с помощью `Identity` параметра.
- Укажите параметр-target со значением "sipfed. Online. Lync. <span>com ".
- Укажите `MoveToTeams` параметр.
- Если у вас нет одной учетной записи с достаточными разрешениями в локальной среде и Office 365 `-credential` , используйте параметр для предоставления учетной записи с достаточными разрешениями в Office 365.
- Если учетная запись с разрешениями в Office 365 не заканчивается на "onmicrosoft. <span>com ", необходимо указать `-HostedMigrationOverrideUrl` параметр с правильным значением, как описано в разделе [обязательные административные учетные данные](move-users-between-on-premises-and-cloud.md#required-administrative-credentials).

Для перемещения пользователя в Теамсонли можно использовать следующую последовательность командлетов, и предполагается, что учетные данные Office 365 являются отдельной учетной записью и предоставляются в качестве входных данных для приглашения Get-Credential.

  ```powershell
  $cred=Get-Credential
  $url="https://admin1a.online.lync.com/HostedMigration/hostedmigrationService.svc"
  Move-CsUser -Identity username@contoso.com -Target sipfed.online.lync.com -MoveToTeams -Credential $cred -HostedMigrationOverrideUrl $url
  ```

### <a name="move-to-teams-using-skype-for-business-server-control-panel"></a>Перемещение в Teams с помощью панели управления Skype для бизнеса Server

1. Откройте приложение панели управления Skype для бизнеса Server.
2. В области навигации слева выберите пункт **Пользователи**.
3. Используйте **Find** , чтобы найти пользователей, которых вы хотите переместить в Teams.
4. Выберите пользователей, а затем в раскрывающемся списке **действий** над списком выберите пункт **Перемещение выбранных пользователей в Teams**.
5. В мастере нажмите кнопку **Далее**.
6. При появлении соответствующего запроса войдите в Office 365, используя учетную запись, которая оканчивается на. onmicrosoft.com и обладает достаточными разрешениями.
7. Нажмите кнопку **Далее**, а **затем еще раз,** чтобы переместить пользователя.
8. Обратите внимание на то, что сообщения о состоянии Success или Failure представлены в верхней части главного приложения панели управления, а не в мастере.

## <a name="notify-your-skype-for-business-on-premises-users-of-the-upcoming-move-to-teams"></a>Уведомление локальных пользователей Skype для бизнеса о предстоящем перемещении в Teams

Локальные средства администрирования в Skype для бизнеса Server 2015 с 8, а также в Skype для бизнеса Server 2019 позволяют уведомить локальных пользователей Skype для бизнеса о предстоящем перемещении в Teams. Когда вы включаете эти уведомления, пользователи увидят уведомление в своем клиенте Skype для бизнеса (Win32, Mac, веб-сайт и мобильное устройство), как показано ниже. Если пользователи нажимайте кнопку **try** , клиент Teams будет запущен, если он установлен; в противном случае пользователи будут переходить на веб-версию Teams в браузере. По умолчанию при включении уведомлений клиенты Skype для бизнеса Win32 автоматически скачивают клиент Teams, чтобы обеспечить доступность расширенного клиента до перехода в режим только Teams. Однако вы также можете отключить это поведение.  Уведомления настраиваются с использованием локальной версии `TeamsUpgradePolicy`, а автоматическая загрузка для клиентов Win32 контролируется с помощью локального `TeamsUpgradeConfiguration` командлета.

> [!TIP]
> Для работы в Skype для бизнеса 2015 с 8 может потребоваться перезагрузка некоторых серверов.

![Уведомление о предстоящем перемещении в Teams](../media/teams-upgrade-notification.png)

Чтобы уведомить локальных пользователей о том, что они скоро будут обновлены до Teams, создайте новый экземпляр Теамсупградеполици с Нотифисфбусерс = true. Затем назначьте эту политику пользователям, которых вы хотите уведомить, либо назначая политику непосредственно пользователю, либо задав политику на уровне сайта, пула или глобального уровня. Следующие командлеты создают и предоставляют политику на уровне пользователя:

```powershell
New-CsTeamsUpgradePolicy -Identity EnableNotifications -NotifySfbUser $true
Grant-CsTeamsUpgradePolicy -Identity username@contoso.com -PolicyName EnableNotifications
```

Автоматическая загрузка Teams через клиент Win32 в Skype для бизнеса контролируется с помощью локального командлета Теамсупградеконфигуратион с параметром Довнлоадтеамс. Эту конфигурацию можно создать на глобальном уровне, уровне сайта и на уровне пула. Например, следующая команда создает конфигурацию для сайта Redmond1:

```powershell
New-CsTeamsUpgradeConfiguration -Identity "site:redmond1"
```

По умолчанию значение параметра Довнлоадтеамс равно true; Однако оно обрабатывается, *только* если Нотифисфбусер = true для определенного пользователя.
