# <a name="plan-hybrid-connectivity-between-skype-for-business-server-and-teams"></a>Планирование гибридного подключения между Skype для бизнеса Server и Teams

> [!Important]
> Несмотря на Skype для бизнеса Online с 2021 года, локальные продукты Skype для бизнеса Server 2019, Skype для бизнеса Server 2015 и Lync Server 2013 по-прежнему поддерживаются. Кроме того, корпорация Майкрософт поддерживает гибридные среды между этими локальными продуктами и Microsoft Teams. Это позволяет организациям с этими локальными развертываниями переносить пользователей в TeamsOnly.  Наконец, cloud Connector Edition Skype для бизнеса Server больше не поддерживается. Клиенты, которым требуется локальное подключение к ТСОП, должны использовать [прямую маршрутизацию](/MicrosoftTeams/direct-routing-landing-page).


В этом разделе описано, как спланировать гибридное подключение между Skype для бизнеса Server или Lync Server 2013 и Teams. Настройка гибридного подключения является первым шагом в переходе от локальной среды к среде Microsoft Teams только в облаке.

В локальном развертывании Skype для бизнеса Server пользователи Skype для бизнеса могут также использовать Teams, но не все функции Teams доступны таким пользователям, если они настроены для использования локального Skype для бизнеса Server развертывания. Считается, что эти пользователи размещены в локальной среде, и некоторые функции Teams недоступны, когда эти пользователи размещены локально, например:

- Федеративные звонки и чаты через клиент Teams пользователя с пользователями в других организациях недоступны
- Взаимодействие через клиент Teams пользователя с другими пользователями в организации, которые используют Skype для бизнеса клиента.
- Функции звонков по ТСОП (если пользователю назначена лицензия телефонной системы).

Чтобы получить полную функциональность Teams, этих пользователей необходимо переместить из локальной Skype для бизнеса в облако, после которого пользователь становится TeamsOnly. При перемещении пользователя из локальной среды в облако для режима сосуществования пользователя будет установлено значение TeamsOnly. После перемещения пользователей в облако и TeamsOnly все входящие чаты и вызовы попадают в клиент Teams (в отличие от параллельного использования Teams).  Дополнительные сведения см. в статье о сосуществовании [Teams](/microsoftteams/coexistence-chat-calls-presence) с руководством Skype для бизнеса миграции и взаимодействия для организаций, использующих [Teams вместе с](/microsoftteams/migration-interop-guidance-for-teams-with-skype) Skype для бизнеса.

Для перемещения пользователей между локальной средой и TeamsOnly в облаке требуется настроить Skype для бизнеса гибридном режиме. Кроме того, перед списанием локального развертывания Skype для бизнеса переместить всех пользователей из локальной среды в облако.   После настройки гибридного подключения вы можете выбрать перенос пользователей в облако на основе своего расписания и бизнес-потребностей.  В случае прямой маршрутизации вы можете использовать локальную инфраструктуру голосовой связи при переходе в облако и после завершения миграции.

В этом разделе описываются требования к инфраструктуре и системе, необходимые для настройки гибридного подключения между существующим локальным развертыванием Skype для бизнеса Server и Teams.

Прочитав этот раздел и подготовив его к настройке гибридного подключения, см. статью "Настройка гибридного подключения между Skype для бизнеса Server [и Teams"](configure-hybrid-connectivity.md). Разделы конфигурации содержат пошаговые инструкции по настройке гибридного подключения между локальным развертыванием и Teams.


## <a name="implications-of-the-retirement-of-skype-for-business-online"></a>Последствия прекращения использования Skype для бизнеса Online
Важно помнить, что как до, так и после прекращения использования Skype для бизнеса Online пользователи, размещенные в локальной Skype для бизнеса Server, могут использовать Teams, но они не могут быть TeamsOnly. (Локальные пользователи по умолчанию находятся в режиме "Острова" и назначаются в любом режиме, кроме TeamsOnly). Пользователи могут получить все преимущества Teams, в частности федерацию, поддержку ТСОП и гарантии того, что все входящие чаты и вызовы попадать в Teams, когда они будут в режиме TeamsOnly. 

Прекращение поддержки Skype для бизнеса Online не влияет на существующий жизненный цикл поддержки Skype для бизнеса Server или Lync Server 2013.  Однако прекращение поддержки Skype для бизнеса Online повлияло на некоторые аспекты перехода пользователей в облако и  перехода на *TeamsOnly* в организациях с локальными Skype для бизнеса Server или Lync Server 2013, включая существующие гибридные организации. Использование гибридной среды в качестве необходимой конфигурации для перехода с локальной среды на облако (например, TeamsOnly) остается неизменным.

До прекращения использования Skype для бизнеса Online гибридные организации могли состоять из трех основных типов пользователей: 
- Локальные пользователи (которые могут или не могут использовать Teams, но не в режиме только Teams) 
- Интернет-пользователи с любым режимом сосуществования, кроме TeamsOnly
- Пользователи TeamsOnly.

Однако после прекращения использования Skype для бизнеса Online гибридные организации могут состоять только из двух основных типов пользователей: 
- Локальные пользователи (кто может или не может использовать Teams, но не в режиме TeamsOnly)
- Только пользователи Teams. 

Чтобы организации могли перейти с Skype для бизнеса Server или Lync Server 2013 на Teams, они по-прежнему должны настраивать гибридную среду с помощью того же набора *инструментов, что* и до прекращения поддержки. При перемещении пользователя из локальной среды в TeamsOnly `-MoveToTeams` больше не требуется указывать переключатель `Move-CsUser`. Ранее, если этот параметр не был указан, пользователи переходят с локального Skype для бизнеса Server на Skype для бизнеса Online, а их режим не изменился. Тем не менее, так как Skype Business Online больше не используется, `Move-CsUser` при перемещении пользователя из локальной среды в облако  автоматически назначается режим TeamsOnly и инициируется преобразование собраний из локальной среды в собрания Teams независимо от того, `-MoveToTeams` указан ли переключатель. Это также означает, что организации с Lync Server 2013, `MoveToTeams` у которых никогда не было переключателя, могут перемещать пользователей непосредственно в TeamsOnly из локальной среды. 

Аналогичным образом, если новый пользователь создается непосредственно в Microsoft 365, а не в локальной среде, он автоматически будет иметь режим "Только Teams" независимо от режима клиента. Помните, что в гибридной организации с по крайней мере одним локальным пользователем в локальная служба Active Directory (а затем синхронизируются с Microsoft 365) следует создавать новых пользователей, а не напрямую создавать пользователей в Microsoft 365, чтобы локальные пользователи могли перенаправляться к новому пользователю, даже если вы планируете, чтобы новый пользователь был облачным *пользователем*. После создания в локальном каталоге эти новые пользователи должны быть включены sip в локальном развертывании Skype для бизнеса, а затем, при необходимости, перемещены в облако, чтобы стать TeamsOnly. 

Режимы сосуществования по-прежнему существуют после прекращения использования Skype для бизнеса Online. Как и раньше, пользователям с учетными записями, Skype для бизнеса Server локальной среде, можно назначить любой режим сосуществования, кроме TeamsOnly. Однако после прекращения поддержки пользователи, которые размещены в Интернете, могут быть только TeamsOnly. Пользователю, размещенного в Сети, больше нельзя назначить режим, отличный от TeamsOnly.


> [!Important]
> Существующие гибридные организации с пользователями, размещенными в Skype для бизнеса Online, которые не являются TeamsOnly, должны как можно скорее обновить этих пользователей до режима "Только Teams". Если в вашей организации по-прежнему есть пользователи, размещенные в Skype для бизнеса *Online*, которые не являются TeamsOnly, вам будет назначено обновление с поддержкой Майкрософт, чтобы перевести этих пользователей в TeamsOnly. **Обновления с поддержкой Майкрософт не влияют на пользователей, которые размещены в Skype для бизнеса Server локальной среде.** Уведомления о планировании будут заранее отправляться гибридным клиентам с пользователями, которые размещены в Skype для бизнеса Online, прежде чем эти пользователи, не связанные с Teams, будут обновлены до Teams.

## <a name="about-shared-sip-address-space-functionality"></a>Общие функции адресного пространства SIP

<a name="BKMK_Overview"> </a>

Благодаря настройке гибридного подключения между локальным развертыванием Skype для бизнеса Server и Teams вы можете включить в сеть некоторых пользователей, которые размещены в локальной среде, а некоторые — в Интернете.

Этот тип конфигурации зависит от общих функций адресного пространства SIP и иногда называется "разделенным доменом". Это означает, что пользователи домена, например contoso.com, разделены между использованием локального Skype для бизнеса Server и Teams, как показано на следующей схеме:

![гибридный Skype для бизнеса подключения — разделенный домен.](../../sfbserver2019/media/plan-hybrid-connectivity-2019-1.png)

При настройке общего адресного пространства SIP:

- Azure Active Directory Connect используется для синхронизации локального каталога с Microsoft 365.
- Пользователи, которые размещены локально, взаимодействуют с локальными Skype для бизнеса серверами.
- Пользователи, которые размещены в Интернете, взаимодействуют с Teams.
- Пользователи из обеих сред могут взаимодействовать друг с другом.
- Этот локальная служба Active Directory является полномочным. Сначала все пользователи должны быть созданы в локальная служба Active Directory, а затем синхронизированы с Azure AD. Даже если вы планируете, чтобы пользователь был размещен в Интернете, необходимо сначала создать пользователя в локальной среде, а затем переместить пользователя в режим "в сети", чтобы обеспечить его обнаружение локальными пользователями.

Перед перемещением пользователя в режим «в сети» пользователю необходимо назначить лицензию Teams, а также Skype для бизнеса Online (план 2). **Назначение лицензии Skype для бизнеса Online требуется даже после прекращения использования Skype для бизнеса Online.** Если пользователи хотят использовать дополнительные сетевые функции, такие как аудиоконференции или телефонная система, необходимо назначить им соответствующую лицензию в Microsoft 365.

## <a name="hybrid-connectivity-infrastructure-requirements"></a>Требования к инфраструктуре гибридных подключений

<a name="BKMK_Infrastructure"> </a>

Чтобы реализовать гибридное подключение между локальной средой и службами связи Microsoft 365, необходимо выполнить следующие требования к инфраструктуре:

- Одно локальное развертывание Skype для бизнеса Server или Lync Server, развернутое в поддерживаемой топологии. См [. требования к топологии](plan-hybrid-connectivity.md#BKMK_Topology) в этом разделе.

- Организация Microsoft 365 с Teams.
    > [!NOTE]
    > Для гибридной конфигурации с локальным развертыванием можно использовать только один клиент.
    
- Azure Active Directory Connect для синхронизации локального каталога с Microsoft 365. Дополнительные сведения см. в статье [Azure AD Connect: учетные записи и разрешения](/azure/active-directory/connect/active-directory-aadconnect-accounts-permissions).

- Skype для бизнеса Server администрирования. Они необходимы для перемещения пользователей из локальной среды в облако. Эти инструменты должны быть установлены на сервере с доступом как к локальному развертыванию, так и к Интернету.
- Сетевые средства администрирования. Вы можете использовать центр администрирования Teams или Windows PowerShell для управления Teams. Чтобы использовать PowerShell для управления Teams, скачайте и установите модуль Teams PowerShell. (Skype для бизнеса online Connector больше не поддерживается).
- Необходимо включить общее адресное пространство SIP, а локальное развертывание должно быть настроено для использования Microsoft 365 в качестве поставщика услуг размещения. Дополнительные сведения о шагах, необходимых для настройки гибридного подключения, см. в разделе ["Настройка гибридного подключения"](configure-hybrid-connectivity.md).

После настройки гибридного подключения можно переместить пользователей в Teams. Дополнительные сведения см. в статье "Перемещение пользователей из [локальной среды в Teams"](move-users-from-on-premises-to-teams.md).

## <a name="multi-forest-support"></a>Поддержка нескольких лесов

<a name="BKMK_MultiForest"> </a>

Корпорация Майкрософт поддерживает следующие типы гибридных сценариев с несколькими лесами:

- **Топология леса ресурсов.** В таком типе топологии существует один лес, в котором Skype для бизнеса Server (лес ресурсов), и один или несколько дополнительных лесов, в которых размещаются удостоверения учетных записей, которые имеют доступ к Skype для бизнеса Server в лесу ресурсов. Как правило, пользователи могут получить доступ к функциям Skype для бизнеса в другом лесу, если выполняются следующие требования.
  - Пользователи должным образом синхронизируются с лесом, в котором размещен Skype для бизнеса. В гибридных конфигурациях это означает, что пользователи должны быть синхронизированы как отключенные объекты пользователей.
  - Лес, в котором размещается Skype для бизнеса, должен доверять лесу, содержащему пользователей.
    Дополнительные сведения о гибридных сценариях с лесом ресурсов см. в статье "Развертывание топологии леса [ресурсов для гибридных Skype для бизнеса"](configure-a-multi-forest-environment-for-hybrid.md).

- **Несколько развертываний Skype для бизнеса Server в нескольких лесах.** Такая конфигурация может возникнуть в результате сценариев слияния и приобретения, а также на более сложных предприятиях. Консолидация всех пользователей из локальной среды в облако в одной организации Microsoft 365 может быть достигнута для любой организации с несколькими развертываниями Skype для бизнеса при условии выполнения следующих ключевых требований:
  - Должна быть задействована не более одной организации Microsoft 365. Консолидация в сценариях с несколькими организацией не поддерживается.
  - В любой момент времени только один локальный лес Skype для бизнеса может находиться в гибридном режиме (общее адресное пространство SIP). Все остальные локальные Skype для бизнеса должны оставаться полностью локальными (и, предположительно, федеративными друг с другом). Обратите внимание, что эти другие локальные организации могут синхронизироваться с AAD при необходимости с новыми функциями для отключения сетевых [доменов SIP](/powershell/module/skype/disable-csonlinesipdomain) , доступных в декабре 2018 г.

    Клиенты с развертываниями Skype для бизнеса в нескольких лесах должны полностью перенести каждый Skype для бизнеса лес по отдельности в организацию Microsoft 365 с помощью функции разделенного домена (общего адресного пространства SIP). После завершения миграции леса клиенты должны отключить гибридное развертывание с локальным развертыванием, прежде чем перейти к миграции следующего локального развертывания Skype для бизнеса развертывания. Кроме того, перед миграцией в облако локальные пользователи остаются в федеративном состоянии с любыми пользователями, которые не представлены в локальном каталоге одного пользователя. Дополнительные сведения см. в статье [о консолидации облачных решений для Teams и Skype для бизнеса](cloud-consolidation.md).

## <a name="federation-requirements"></a>Требования к федерации

<a name="BKMK_Federation"> </a>

При настройке Skype для бизнеса гибридном режиме необходимо убедиться, что локальные и сетевые среды могут федерации друг с другом.  В сетевой среде по умолчанию открытая федерация. В локальной среде часто по умолчанию закрыта федерация.  

Для успешного настройки гибридного развертывания необходимо соблюдение следующих требований:

- Сопоставление доменов должно быть настроено одинаково для локального развертывания и организации Microsoft 365. Если обнаружение партнеров включено в локальном развертывании, необходимо настроить открытую федерацию для вашей сетевой организации. Если обнаружение партнеров не включено, необходимо настроить закрытую федерацию для вашей интернет-организации.
- Список заблокированных доменов в локальном развертывании должен полностью совпадать со списком заблокированных доменов для интерактивного клиента.
- Список разрешенных доменов в локальном развертывании должен полностью совпадать со списком разрешенных доменов для интерактивного клиента.
- Федерация должна быть включена для внешних подключений для сетевого клиента.

## <a name="network-considerations"></a>Сеть

В следующих разделах описываются рекомендации по следующим:

- Параметры DNS
- Рекомендации по брандмауэру

### <a name="dns-settings-for-hybrid-deployments"></a>Параметры DNS для гибридных развертываний

<a name="BKMK_DNS"> </a>

При создании записей DNS для гибридных развертываний все внешние записи DNS Skype для бизнеса должны указывать на локальную инфраструктуру. Дополнительные сведения о необходимых записях DNS см. в требованиях [DNS для Skype для бизнеса Server](../../sfbserver/plan-your-deployment/network-requirements/dns.md).

Кроме того, необходимо убедиться, что разрешение DNS, описанное в таблице ниже, работает в локальном развертывании. (Если вы уже настроили федерацию для локальной среды, у вас, скорее всего, уже есть эти элементы.)

|Запись DNS  <br/> |Разрешение с помощью  <br/> |Требование DNS  <br/> |
|:-----|:-----|:-----|
|Запись DNS SRV для _sipfederationtls._tcp.\<sipdomain.com\> для всех поддерживаемых доменов SIP, разрешаемого во внешние IP-адреса Access Edge  <br/> |Пограничные серверы  <br/> |Включите федеративную связь в гибридной конфигурации. Пограничному серверу необходимо знать, куда направлять федеративный трафик для домена SIP, разделенного между локальным и сетевым трафиком.  <br/> Необходимо использовать строгое сопоставление DNS-имени между доменом в имени пользователя и записи SRV.  <br/> |
|Записи DNS A для полного доменного имени пограничной службы веб-конференций, например webcon.contoso.com разрешения на внешние IP-адреса пограничных веб-конференций.  <br/> |Компьютеры пользователей, подключенные к внутренней корпоративной сети  <br/> |Разрешите пользователям в сети представлять или просматривать содержимое на собраниях, размещенных в локальной среде. Содержимое включает файлы PowerPoint, доски, опросы и общие заметки.  <br/> |

В зависимости от настройки DNS в организации может потребоваться добавить эти записи во внутреннюю размещенную зону DNS для соответствующих доменов SIP, чтобы обеспечить разрешение внутренних DNS-имен в эти записи.

### <a name="firewall-considerations-for-hybrid-deployments"></a>Рекомендации по брандмауэру для гибридных развертываний

<a name="BKMK_Firewall"> </a>

Компьютеры в сети должны поддерживать выполнение стандартных уточняющих интернет-запросов DNS. Если эти компьютеры могут осуществлять доступ к стандартным интернет-сайтам, сеть отвечает этому требованию.

В зависимости от расположения центра обработки данных Microsoft Online Services необходимо также настроить сетевые брандмауэры для принятия подключений на основе доменных имен с подстановочными знаками (например, \*весь трафик из .outlook.com). Если брандмауэры вашей организации не поддерживают конфигурации имен с подстановочными знаками, необходимо вручную определить диапазоны IP-адресов, которые вы хотите разрешить, и указанные порты.

Дополнительные сведения, включая сведения о портах и требованиях к протоколу, см. в статье о [URL-адресах и диапазонах IP-адресов Microsoft 365](/microsoft-365/enterprise/urls-and-ip-address-ranges).
