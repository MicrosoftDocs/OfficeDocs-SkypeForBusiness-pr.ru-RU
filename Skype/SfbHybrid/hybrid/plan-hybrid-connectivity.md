# <a name="plan-hybrid-connectivity-between-skype-for-business-server-and-microsoft-365-or-office-365"></a>Планирование гибридного подключения между Skype для бизнеса Server и Microsoft 365 или Office 365

Ознакомьтесь с этой темой, чтобы узнать, как планировать гибридное подключение между Skype для бизнеса Server и Teams или Skype для бизнеса Online. Настройка гибридного подключения — это первый этап переноса локальной среды в облако.

Если у вас есть пользователи локальной среды Skype для бизнеса, которые (параллельно) используют Teams, то эти пользователи не смогут взаимодействовать с пользователями Skype для бизнеса или общаться с пользователями федеративных организаций из своего клиента Teams. Чтобы воспользоваться этими возможностями в Teams, этих пользователей нужно перенести из локальной среды Skype для бизнеса в облако, что требует настройки гибридного режима Skype для бизнеса. Кроме того, для лучшего опыта эти пользователи должны быть в режиме Teams Only, который обеспечивает все входящие вызовы и чаты с любой пользовательской земли в клиенте Teams пользователя.

Также требуется настройка гибридного подключения и перенос всех пользователей в облако перед удалением локального развертывания Skype для бизнеса.  После настройки гибридного подключения вы можете выбрать перенос пользователей в облако на основе своего расписания и бизнес-потребностей. В случае прямой маршрутизации вы можете использовать локальную инфраструктуру голосовой связи при переходе в облако и после завершения миграции.

В этом разделе описываются требования к инфраструктуре и системе, необходимые для настройки гибридного подключения между существующими локальной развертыванием Skype для бизнеса Server и Teams или Skype для бизнеса Online.

После прочтя эту тему и готовы настроить гибридное подключение, см. в разделе Настройка гибридного подключения между Skype для бизнеса Server и [Microsoft 365 или Office 365](configure-hybrid-connectivity.md). Разделы конфигурации предоставляют пошаговую инструкцию по настройке гибридного подключения между локальной развертыванием и teams или Skype для бизнеса Online.

> [!Important]
> Skype для бизнеса Online будет отменен 31 июля 2021 г., после чего служба будет больше недоступна.  Кроме того, подключение PSTN между локальной средой, будь то Skype для бизнеса Server или Cloud Connector Edition и Skype для бизнеса Online, больше не будет поддерживаться.  Узнайте, как подключить сеть локальной телефонии к Teams с помощью [прямого маршрутного маршрутинга.](/MicrosoftTeams/direct-routing-landing-page)

## <a name="about-shared-sip-address-space-functionality"></a>Об общих функциях пространства адресов SIP

<a name="BKMK_Overview"> </a>

 Благодаря гибридному подключению между локальной развертыванием Skype для бизнеса Server и Teams или Skype для бизнеса Online вы можете использовать для некоторых пользователей размещение в локальной сети, а некоторых пользователей — в Интернете.

Этот тип конфигурации зависит от общих функций пространства адресов SIP и иногда именуется "разделенным доменом", т. е. пользователи домена, например contoso.com, делятся между использованием Skype для бизнеса Server в помещениях и Teams или Skype для бизнеса Online, как показано на следующей схеме:

![Гибридное подключение Skype для бизнеса — разделенный домен](../../sfbserver2019/media/plan-hybrid-connectivity-2019-1.png)

При настройке общего пространства адресов SIP:

- Azure Active Directory Connect используется для синхронизации локального каталога с Microsoft 365 или Office 365.
- Пользователи, которые находятся в помещении, взаимодействуют с локальной skype для бизнес-серверов.
- Пользователи, которые находятся в сети, могут взаимодействовать со службами Skype для бизнеса Online или Teams.
- Пользователи из обеих сред могут общаться друг с другом.
- Локальное active Directory является авторитетным. Сначала все пользователи должны создаваться в локальном Active Directory, а затем синхронизироваться с Azure AD. Даже если предполагается, что пользователь будет дома в Интернете, сначала необходимо создать пользователя в локальной среде, а затем переместить пользователя в интернет, чтобы убедиться, что он обнаружен локально пользователями.

Перед тем как переселить пользователя в Интернет, пользователю должна быть назначена лицензия Skype для бизнеса Online (Plan 2). Если пользователь будет использовать Teams, ему также должна быть назначена лицензия Teams (а лицензия Skype для бизнеса должна оставаться включенной). Если пользователи хотят воспользоваться дополнительными функциями в Интернете, такими как аудиоконференция или телефонная система, необходимо назначить им соответствующую лицензию в Microsoft 365 или Office 365.

## <a name="hybrid-connectivity-infrastructure-requirements"></a>Требования к инфраструктуре гибридной инфраструктуры подключения

<a name="BKMK_Infrastructure"> </a>

Чтобы реализовать гибридную связь между локальной средой и службами связи Microsoft 365 или Office 365, необходимо выполнить следующие требования к инфраструктуре:

- Одно локальное развертывание Skype для бизнеса Server или Lync Server, развернутых в поддерживаемой топологии. В [этом разделе см. требования](plan-hybrid-connectivity.md#BKMK_Topology) топологии.

- Организация Microsoft 365 или Office 365 с поддержкой Skype для бизнеса Online.
    > [!NOTE]
    > Для гибридной конфигурации с локальной развертыванием можно использовать только одного клиента.
    
- Azure Active Directory Connect для синхронизации локального каталога с Microsoft 365 или Office 365. Дополнительные сведения см. [в видеоролике Azure AD Connect: Accounts and permissions.](/azure/active-directory/connect/active-directory-aadconnect-accounts-permissions)

- Инструменты администрирования Skype для бизнеса Server. Они необходимы для перемещения пользователей из локального в облако. Эти средства должны быть установлены на сервере с доступом как к локальному развертыванию, так и к Интернету.
- Средства администрирования в Интернете. Вы можете использовать центр администрирования Teams или Windows PowerShell для управления Teams и Skype для бизнеса Online. Чтобы использовать PowerShell для управления командой или Skype для бизнеса Online, скачайте и установите соединитель Skype для бизнеса Online.
- Необходимо включить общее пространство адресов SIP и настроить локальное развертывание для использования Microsoft 365 или Office 365 в качестве поставщика хостинга. Дополнительные сведения о действиях, необходимых для настройки гибридного подключения, см. в дополнительных сведениях [о настройке гибридного подключения.](configure-hybrid-connectivity.md)

После настройки гибридного подключения можно переместить пользователей в Teams или Skype для бизнеса Online. Дополнительные сведения см. в дополнительных [сведениях:](move-users-from-on-premises-to-teams.md) Перемещение пользователей из локального в Teams и перемещение пользователей из локального в [Skype для бизнеса Online.](move-users-from-on-premises-to-skype-for-business-online.md)

## <a name="server-version-requirements"></a>Требования к версии сервера

<a name="BKMK_Topology"> </a>

Чтобы настроить развертывание для гибридного с **Teams или Skype для бизнеса Online,** необходимо иметь одну из следующих поддерживаемых топологий:

- Развертывание Skype для бизнеса Server 2019, в котором на всех серверах работает Skype для бизнеса Server 2019.
- Развертывание Skype для бизнеса Server 2015, в котором на всех серверах работает Skype для бизнеса Server 2015.
- Развертывание Lync Server 2013 со всеми серверами под управлением Lync Server 2013.  Однако, если требуется гибридная голосовая связь, необходимо использовать смешанную топологию версий, как отмечено ниже.
- Развертывание с максимальным количеством 2 различных версий сервера, как указано ниже:
  - Skype для бизнеса Server 2015 и Skype для бизнеса Server 2019
  - Lync Server 2013 и Skype для бизнеса Server 2019
  - Lync Server 2013 и Skype для бизнеса Server 2015

*Если гибридный* голос требуется в любой топологии, то в Skype для бизнеса 2015 или более поздней части должен работать как край федерации, так и пул, связанный с федерацией SIP. Пользователи могут оставаться в пуле Lync 2013, если он существует. Дополнительные сведения см. в [материале Plan Phone System with PSTN Connectivity in Skype for Business Server.](../../SfbServer/skype-for-business-hybrid-solutions/plan-your-phone-system-cloud-pbx-solution/plan-phone-system-with-on-premises-pstn-connectivity.md)

Следующие топологии, включающие **Lync Server 2010,** поддерживаются Skype для бизнеса Online для обмена мгновенными сообщениями и собраний. Topologies, которые включают **Lync Server 2010,** не поддерживаются для гибридного голоса или Teams.

- Смешанное развертывание Lync Server 2010 и Skype для бизнеса Server 2015
- Смешанное развертывание Lync Server 2010 и Lync Server 2013
- Развертывание Lync Server 2010 со всеми серверами под управлением Lync Server 2010 с последними накопительными обновлениями.

В Lync Server 2010 с последними накопительными обновлениями должен работать сервер федерации Edge Server и следующий сервер хмеля федерации Edge Server. Административные средства Skype для бизнеса Server 2015 или Lync Server 2013 должны быть установлены по крайней мере на одном сервере или рабочей станции управления.

## <a name="multi-forest-support"></a>Поддержка нескольких лесов

<a name="BKMK_MultiForest"> </a>

Корпорация Майкрософт поддерживает следующие типы гибридных сценариев с несколькими лесами:

- **Топология лесов ресурсов.** В этом типе топологии существует один лес, в котором размещен Skype для бизнеса Server (лес ресурсов), а в лесу ресурсов есть один или несколько дополнительных лесов, в которых размещены удостоверения учетных записей, которые имеют доступ к Skype для бизнеса Server в лесу ресурсов. Как правило, пользователи могут получить доступ к функциям Skype для бизнеса в другом лесу, если будут выполнены следующие требования:
  - Пользователи правильно синхронизируются в лесу, в котором размещен Skype для бизнеса. В гибридных конфигурациях это означает, что пользователи должны синхронизироваться с отключенными объектами пользователей.
  - Лес, в котором размещен Skype для бизнеса, должен доверять лесу, содержащим пользователей.
    Дополнительные сведения о гибридных сценариях лесов ресурсов см. в материале Развертывание топологии леса ресурсов [для гибридного Skype для бизнеса.](configure-a-multi-forest-environment-for-hybrid.md)

- **Несколько развертываний Skype для бизнеса Server в нескольких лесах.** Такая конфигурация может возникать в результате сценариев слияния и приобретения, а также в более сложных предприятиях. Консолидация всех пользователей из локального облака в одну организацию Microsoft 365 или Office 365 может быть достигнута для любой организации с несколькими развертываниями Skype для бизнеса при условии, что будут выполнены следующие ключевые требования:
  - Должна быть как минимум одна организация Microsoft 365 или Office 365. Консолидация в сценариях с более чем одной организацией не поддерживается.
  - В любой момент времени только один локальной лес Skype для бизнеса может быть в гибридном режиме (общее пространство адресов SIP). Все остальные локальное лесопарки Skype для бизнеса должны оставаться полностью на месте (и, предположительно, федератированы друг с другом). Обратите внимание, что эти другие локальной организации могут синхронизироваться с AAD при желании с новыми функциями для отключения доменов [SIP](/powershell/module/skype/disable-csonlinesipdomain) в Интернете, доступных с декабря 2018 г.

    Клиенты с развертыванием Skype для бизнеса в нескольких лесах должны полностью перенести каждый лес Skype для бизнеса по отдельности в организацию Microsoft 365 или Office 365 с помощью функций split-domain (Shared SIP Address Space), а затем отключить гибрид с локальной развертыванием, прежде чем перейти к миграции следующего локального развертывания Skype для бизнеса. Кроме того, перед переносом в облако, локально пользователи остаются в федерарном состоянии с любыми пользователями, которые не представлены в локальном каталоге одного пользователя. Дополнительные сведения см. в [материале Cloud consolidation for Teams и Skype for Business.](cloud-consolidation.md)

## <a name="federation-requirements"></a>Требования федерации

<a name="BKMK_Federation"> </a>

При настройке гибридного режима Skype для бизнеса необходимо обеспечить, чтобы локальное и сетевое окружение могли федератерироваться друг с другом.  Интернет-среда по умолчанию имеет открытую федерацию; в локальной среде часто по умолчанию закрыта федерация.  

Для успешного настройки гибридного развертывания необходимо соблюдение следующих требований:

- Соответствие домена должно быть настроено одинаково для локального развертывания и организации Microsoft 365 или Office 365. Если обнаружение партнера включено в локальном развертывании, необходимо настроить открытую федерацию для вашей организации в Интернете. Если обнаружение партнера не включено, необходимо настроить закрытую федерацию для вашей организации в Интернете.
- Список заблокированных доменов в локальном развертывании должен полностью совпадать со списком заблокированных доменов для интерактивного клиента.
- Список разрешенных доменов в локальном развертывании должен полностью совпадать со списком разрешенных доменов для интерактивного клиента.
- Федерация должна быть включена для внешних коммуникаций для клиента в Интернете.

### <a name="dns-settings-for-hybrid-deployments"></a>Параметры DNS для гибридного развертывания

При создании записей DNS для гибридных развертывания все внешние DNS-записи Skype для бизнеса должны указать на локальной инфраструктуры. Дополнительные сведения о необходимых записях DNS, пожалуйста, обратитесь к [требованиям DNS для Skype для бизнеса Server.](../../sfbserver/plan-your-deployment/network-requirements/dns.md)

Кроме того, необходимо убедиться, что решение DNS, описанное в следующей таблице, работает в локальном развертывании. (Если вы уже настроены федерации для локального, то, скорее всего, у вас уже есть эти.)

|Запись DNS  <br/> |Разрешимый путем  <br/> |Требование DNS  <br/> |
|:-----|:-----|:-----|
|Запись SRV DNS для _sipfederationtls._tcp.\<sipdomain.com\> для всех поддерживаемых доменов SIP, разрешающих доступ к внешним IP-адресам Access Edge (s)  <br/> |Edge server (s)  <br/> |Включить федеративную связь в гибридной конфигурации. |DNS A record (s) для службы веб-конференций Edge FQDN, например webcon.contoso.com разрешения для внешних IP-адресов веб-конференций Edge (s)  <br/> |Компьютеры пользователей, подключенных к внутренней корпоративной сети  <br/> |Включить пользователям в Интернете представление или просмотр контента на локальном собрании. В зависимости от настройки DNS в организации может потребоваться добавить эти записи во внутреннюю зону DNS для соответствующего домена SIP(ы), чтобы обеспечить внутреннее разрешение DNS для этих записей.

### <a name="firewall-considerations-for-hybrid-deployments"></a>Соображения брандмауэра для гибридных развертывание

<a name="BKMK_Firewall"> </a>

Компьютеры в сети должны поддерживать выполнение стандартных уточняющих интернет-запросов DNS. Если эти компьютеры могут осуществлять доступ к стандартным интернет-сайтам, сеть отвечает этому требованию.

В зависимости от расположения Microsoft Online Services центра обработки данных необходимо также настроить сетевые брандмауэры для нахождения подключений на основе имен доменов под диктовки (например, весь трафик с \* .outlook.com). Если брандмауэры организации не поддерживают конфигурации имен подмастерьев, необходимо вручную определить диапазоны IP-адресов, которые вы хотите разрешить, и указанные порты.

Дополнительные сведения, включая сведения о портах и требованиях протокола, см. в url-адресах [Microsoft 365 и Office 365 и диапазонах IP-адресов.](/microsoft-365/enterprise/urls-and-ip-address-ranges)
