# <a name="plan-hybrid-connectivity-between-skype-for-business-server-and-microsoft-365-or-office-365"></a>Планирование гибридного подключения между Skype для бизнеса Server и Microsoft 365 или Office 365

## <a name="overview"></a>Общие сведения

В этом разделе рассказывается, как спланировать гибридное подключение между Skype для бизнеса Server и Teams и Skype для бизнеса Online. Настройка гибридного подключения — это первый этап переноса локальной среды в облако.

Если у вас есть пользователи локальной среды Skype для бизнеса, которые (параллельно) используют Teams, то эти пользователи не смогут взаимодействовать с пользователями Skype для бизнеса или общаться с пользователями федеративных организаций из своего клиента Teams. Чтобы воспользоваться этими возможностями в Teams, этих пользователей нужно перенести из локальной среды Skype для бизнеса в облако, что требует настройки гибридного режима Skype для бизнеса. Кроме того, для оптимального использования эти пользователи должны находиться в режиме "только Teams", который обеспечивает все входящие звонки и разговоры от пользователя в клиенте Teams пользователя.

Также требуется настройка гибридного подключения и перенос всех пользователей в облако перед удалением локального развертывания Skype для бизнеса.  После настройки гибридного подключения вы можете выбрать перенос пользователей в облако на основе своего расписания и бизнес-потребностей. В случае прямой маршрутизации вы можете использовать локальную инфраструктуру голосовой связи при переходе в облако и после завершения миграции.

В этом разделе описываются требования к инфраструктуре и системе, которые необходимы для настройки гибридного подключения между существующим локальным развертыванием Skype для бизнеса Server и Teams или Skype для бизнеса Online.

После ознакомления с этой статьей и готовности к настройке гибридного подключения, ознакомьтесь с разделом [Настройка гибридного подключения между Skype для бизнеса Server и microsoft 365 или Office 365](configure-hybrid-connectivity.md). В разделах, посвященных настройке, представлены пошаговые инструкции по настройке гибридного подключения между локальным развертыванием и Teams и Skype для бизнеса Online.

> [!Important]
> Skype для бизнеса Online будет прекращен до 31 июля, 2021 после чего служба станет недоступна.  Кроме того, связь по протоколу PSTN между локальной средой и в Skype для бизнеса Server или Cloud Connector Edition и Skype для бизнеса Online больше не будет поддерживаться.  Узнайте, как подключить локальную телефонную сеть к Teams с помощью [прямой маршрутизации](https://docs.microsoft.com/MicrosoftTeams/direct-routing-landing-page).

## <a name="about-shared-sip-address-space-functionality"></a>Общие сведения о функциональных возможностях адресного пространства SIP

<a name="BKMK_Overview"> </a>

 Если гибридное подключение настроено между локальным развертыванием Skype для бизнеса Server и Microsoft Teams или Skype для бизнеса Online, некоторые пользователи могут быть размещены в локальной среде, а некоторые — в сети.

Этот тип конфигурации опирается на общую функциональность адресного пространства SIP и иногда называется "разделенным доменом", то есть пользователи домена (например, contoso.com) делятся между локальным сервером Skype для бизнеса Server и Microsoft Teams или Skype для бизнеса Online, как показано на следующей схеме:

![Гибридное подключение SfB — разделенный домен](../../sfbserver2019/media/plan-hybrid-connectivity-2019-1.png)

При настройке общего адресного пространства SIP:

- Azure Active Directory Connect используется для синхронизации локального каталога с Microsoft 365 или Office 365.
- Пользователи, размещенные в локальной среде, взаимодействуют с локальными серверами Skype для бизнеса.
- Пользователи, размещенные в сети, могут взаимодействовать со службами Skype для бизнеса Online или Teams.
- Пользователи из обеих сред могут общаться друг с другом.
- Локальная служба Active Directory является удостоверяющей. Сначала необходимо создать все пользователи в локальной службе Active Directory, а затем синхронизировать их с Azure AD. Даже если вы хотите, чтобы пользователь был подключен к сети, необходимо сначала создать пользователя в локальной среде, а затем переместить пользователя в оперативный режим, чтобы пользователь был обнаруживаемым для локальных пользователей.

Прежде чем пользователь сможет перемещаться по сети, пользователю должна быть назначена лицензия на Skype для бизнеса Online (план 2). Если пользователь будет использовать Teams, ему также должна быть назначена лицензия Teams (а лицензия Skype для бизнеса должна быть включена). Если пользователям требуется использовать дополнительные сетевые функции, такие как аудио-и видеоконференции, необходимо назначить им соответствующую лицензию в Microsoft 365 или Office 365.

## <a name="infrastructure-requirements"></a>Требования к инфраструктуре

<a name="BKMK_Infrastructure"> </a>

Для реализации гибридного подключения между локальной средой и службами связи Microsoft 365 или Office 365 необходимо соблюдение следующих требований к инфраструктуре:

- Одно локальное развертывание Skype для бизнеса Server или Lync Server, развернутое в поддерживаемой топологии. В этом разделе описаны [требования к топологии](plan-hybrid-connectivity.md#BKMK_Topology) .

- Организация Microsoft 365 или Office 365 со Skype для бизнеса Online включена.
    > [!NOTE]
    > Для гибридной конфигурации с локальным развертыванием можно использовать только один клиент.
    
- Azure Active Directory Connect для синхронизации локального каталога с Microsoft 365 или Office 365. Дополнительные сведения см. в статье [Azure AD Connect: accounts and Permissions](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-accounts-permissions).

- Средства администрирования Skype для бизнеса Server.  Они необходимы для перемещения пользователей из локальной организации в облако. Эти средства должны быть установлены на сервере с доступом как к локальному развертыванию, так и к Интернету.
- Администрирование через Интернет.  Для управления Teams и Skype для бизнеса Online можно использовать центр администрирования Teams или Windows PowerShell. Чтобы использовать PowerShell для управления Teams или Skype для бизнеса Online, скачайте и установите соединитель Skype для бизнеса Online.
- Общее адресное пространство SIP должно быть включено, а локальное развертывание должно быть настроено на использование Microsoft 365 или Office 365 в качестве поставщика услуг хостинга. Дополнительные сведения о действиях, необходимых для настройки гибридного подключения, приведены в разделе [Настройка гибридного подключения](configure-hybrid-connectivity.md).

После настройки гибридного подключения вы можете переместить пользователей в Teams или Skype для бизнеса Online. Дополнительную информацию можно узнать в статье [Перемещение пользователей из локальной сети в Teams](move-users-from-on-premises-to-teams.md) и [Перемещение пользователей из локальной сети в Skype для бизнеса Online](move-users-from-on-premises-to-skype-for-business-online.md).

## <a name="server-version-requirements"></a>Требования к версии сервера

<a name="BKMK_Topology"> </a>

Чтобы настроить развертывание для гибридной среды с **Teams или Skype для бизнеса Online**, необходима одна из следующих поддерживаемых топологий:

- Развертывание Skype для бизнеса Server 2019, в котором на всех серверах работает Skype для бизнеса Server 2019.
- Развертывание Skype для бизнеса Server 2015, в котором на всех серверах работает Skype для бизнеса Server 2015.
- Развертывание Lync Server 2013 со всеми серверами, на которых работает Lync Server 2013.  Однако если необходимо использовать гибридную голосовую связь, необходимо использовать топологию смешанной версии, как указано ниже.
- Развертывание с максимальным количеством двух разных версий сервера, как указано ниже:
  - Skype для бизнеса Server 2015 и Skype для бизнеса Server 2019
  - Lync Server 2013 и Skype для бизнеса Server 2019
  - Lync Server 2013 и Skype для бизнеса Server 2015

*Если в какой-либо топологии требуется Гибридная голосовая связь*, то пограничный сервер, назначенный как край Федерации, а также пул, связанный с Федерации SIP, должен работать под управлением Skype для бизнеса 2015 или более поздней версии. Пользователи могут оставить пул Lync 2013, если он существует. Более подробную информацию можно узнать [в статье Планирование телефонной системы с подключением PSTN в Skype для бизнеса Server](https://docs.microsoft.com/skypeforbusiness/skype-for-business-hybrid-solutions/plan-your-phone-system-cloud-pbx-solution/plan-phone-system-with-on-premises-pstn-connectivity).

Следующие топологии, включающие **Lync Server 2010, поддерживаются в Skype для бизнеса Online** для обмена мгновенными сообщениями и собраний.  Топологии, включающие **Lync Server 2010, не поддерживаются для гибридной голосовой связи и Teams**.

- Смешанное развертывание Lync Server 2010 и Skype для бизнеса Server 2015
- Смешанное развертывание Lync Server 2010 и Lync Server 2013
- Развертывание Lync Server 2010 со всеми серверами Lync Server 2010 с последними накопительными обновлениями.

Пограничный сервер федерации и сервер следующего прыжка на пограничном сервере федерации должны работать под управлением Lync Server 2010 с последними накопительными обновлениями. Средства администрирования Skype для бизнеса Server 2015 или Lync Server 2013 должны быть установлены по крайней мере на одном сервере или рабочей станции управления.

## <a name="multi-forest-support"></a>Поддержка нескольких лесов

<a name="BKMK_MultiForest"> </a>

Корпорация Майкрософт поддерживает следующие типы гибридных сценариев с несколькими лесами:

- **Топология леса ресурсов.** В такой топологии существует один лес, на котором размещается Skype для бизнеса Server (лес ресурсов), а также один или несколько дополнительных лесов, в которых размещаются удостоверения учетных записей, которые обращаются к серверу Skype для бизнеса в лесу ресурсов. Как правило, пользователи могут получать доступ к функциям Skype для бизнеса в другом лесу, если соблюдены следующие требования:
  - Пользователи должны правильно синхронизироваться с лесом, на котором размещается Skype для бизнеса. В гибридных конфигурациях пользователи должны синхронизироваться с отключенными объектами пользователей.
  - Лес, в котором размещается Skype для бизнеса, должен доверять лесу, содержащему пользователей.
    Подробнее о гибридных сценариях леса ресурсов можно узнать в статье [Deploy a Topology for Resource Forest for гибридный Skype для бизнеса](configure-a-multi-forest-environment-for-hybrid.md).

- **Несколько развертываний Skype для бизнеса Server в нескольких лесах.** Такая конфигурация может возникнуть в результате слияния и получения сценариев, а также на более сложных предприятиях. Консолидация всех пользователей из локальной организации в облако в одной организации Microsoft 365 или Office 365 может быть достигнута для любой организации с несколькими развертываниями Skype для бизнеса при условии соблюдения следующих ключевых требований:
  - Должна быть хотя бы одна организация Microsoft 365 или Office 365. Консолидация в сценариях, в которых используется более одной организации, не поддерживается.
  - В любой момент времени только один локальный лес Skype для бизнеса может находиться в гибридном режиме (общее адресное пространство SIP). Все остальные локальные леса Skype для бизнеса должны быть полностью включены в локальную (и предположительно Федеративные друг с другом). Обратите внимание, что эти другие локальные организации могут синхронизироваться с AAD, если вы хотите [Добавить новые функции, чтобы отключить домены SIP в Интернете](https://docs.microsoft.com/powershell/module/skype/disable-csonlinesipdomain) , доступные на 2018 декабря.

    Клиенты с разрешениями Skype для бизнеса в нескольких лесах должны полностью переносить каждый лес Skype для бизнеса по отдельности в организацию Microsoft 365 или Office 365 с помощью функции Split-Domain (общее адресное пространство SIP), а затем отключить гибридную среду с локальным развертыванием, прежде чем переходить к следующему локальному развертыванию Skype для бизнеса. Кроме того, прежде чем приступать к миграции в облако, локальные пользователи не будут работать в федеративного состояния с пользователями, которые не представлены в локальном каталоге пользователя. Дополнительные сведения см в разделе [объединение в облако для Teams и Skype для бизнеса](cloud-consolidation.md).

## <a name="federation-requirements"></a>Требования Федерации

<a name="BKMK_Federation"> </a>

При настройке гибридной среды необходимо убедиться, что локальные и Интернет-среды могут осуществлять Федерацию друг друга.  По умолчанию в интерактивной среде открыта Федерация; по умолчанию для локальной среды часто используется замкнутая Федерация.  

Для успешного настройки гибридного развертывания необходимо соблюдение следующих требований:

- Для локального развертывания и вашей организации Microsoft 365 или Office 365 необходимо настроить совпадение доменов. Если для локального развертывания включено обнаружение партнеров, то для своей организации необходимо настроить открытую Федерацию. Если обнаружение партнеров не включено, необходимо настроить закрытую Федерацию для вашей организации в Интернете.
- Список заблокированных доменов в локальном развертывании должен полностью совпадать со списком заблокированных доменов для интерактивного клиента.
- Список разрешенных доменов в локальном развертывании должен полностью совпадать со списком разрешенных доменов для интерактивного клиента.
- Для подключения к сети Интернет-клиента должна быть включена Федерация.

### <a name="dns-settings"></a>Параметры DNS

<a name="BKMK_DNS"> </a>

При создании записей DNS для гибридных развертываний все внешние записи DNS Skype для бизнеса должны указать на локальную инфраструктуру. Для получения дополнительных сведений о необходимых записях DNS обратитесь к разделу [требования к DNS для Skype для бизнеса Server](../../sfbserver/plan-your-deployment/network-requirements/dns.md).

Кроме того, необходимо убедиться в том, что разрешение DNS, описанное в следующей таблице, работает в локальном развертывании. (Если вы уже настроили Федерацию для локальной организации, то, скорее всего, у вас уже есть.)

|Запись DNS  <br/> |Разрешимо  <br/> |Требование DNS  <br/> |
|:-----|:-----|:-----|
|Запись DNS SRV для _sipfederationtls. _tcp.\<sipdomain.com\> для всех поддерживаемых доменов SIP, разрешающих внешние IP-адреса пограничного сервера доступа  <br/> |Пограничные серверы  <br/> |Включение Федеративной связи в гибридной конфигурации. Пограничный сервер должен знать, где следует маршрутизировать федеративный трафик для домена SIP, который разделяется между локальным и подключенным.  <br/> Необходимо использовать только DNS-сопоставление имен домена в имени пользователя и записи SRV.  <br/> |
|Записи A DNS для службы пограничного веб-конференц-связи, например webcon.contoso.com, разрешающие внешние IP-адреса веб-конференций.  <br/> |Компьютеры пользователей, подключенных к внутренней корпоративной сети  <br/> |Позволяет интерактивным пользователям отображать или просматривать содержимое на локальных размещенных собраниях. ### <a name="firewall-considerations"></a>Рекомендации по брандмауэру

<a name="BKMK_Firewall"> </a>

Компьютеры в сети должны поддерживать выполнение стандартных уточняющих интернет-запросов DNS. Если эти компьютеры могут осуществлять доступ к стандартным интернет-сайтам, сеть отвечает этому требованию.

В зависимости от расположения центра обработки данных Microsoft Online Services необходимо также настроить сетевые брандмауэры для приема подключений на основе доменных имен с подстановочными знаками (например, весь трафик от \* . Outlook.com). Если брандмауэры вашей организации не поддерживают конфигурации с использованием подстановочных имен, необходимо вручную определить диапазоны IP-адресов, которые нужно разрешить, и указанные порты.

Дополнительные сведения, включая сведения о портах и протоколах, приведены в [статье Microsoft 365 и URL-адреса и диапазоны IP-адресов для Office 365](https://go.microsoft.com/fwlink/p/?LinkId=252942).
