# <a name="plan-hybrid-connectivity-between-skype-for-business-server-and-teams"></a>Планирование гибридного подключения между Skype для бизнеса Server и Teams

> [!Important]
> Несмотря на то Skype для бизнеса что с 2021 г. Skype для бизнеса, в 2019 г., Skype для бизнеса Server 2015 г. и Lync Server 2013 по Skype для бизнеса Server-прежнему поддерживаются продукты локального производства. Кроме того, Корпорация Майкрософт поддерживает гибридные среды между этими локальной продукцией и Microsoft Teams. Это позволяет организациям с этими локальной развертываниями перенести своих пользователей в TeamsOnly.  Наконец, cloud Connector Edition Skype для бизнеса Server больше не поддерживается. Клиенты, требующие локального подключения к PSTN, должны использовать [прямые маршруты](/MicrosoftTeams/direct-routing-landing-page).


Ознакомьтесь с этой темой, чтобы узнать, как планировать гибридное подключение между Skype для бизнеса Server или Lync Server 2013 и Teams. Настройка гибридного подключения — это первый шаг к перемещению локальной среды в Microsoft Teams.

Если у вас есть локальное Skype для бизнеса пользователей, которые уже используют Teams (бок о бок), эти пользователи не имеют возможности взаимодействовать с Skype для бизнеса пользователями из Teams клиента, а также общаться с пользователями в федерадных организациях, из Teams клиента. Чтобы получить эту функциональность в Teams, эти пользователи должны быть перемещены из локального Skype для бизнеса в облако, чтобы они стали пользователями TeamsOnly, что требует настройки Skype для бизнеса гибридного режима. В отличие от одностороннего использования Teams, когда пользователь находится в режиме TeamsOnly, все входящие вызовы и чаты из любой страны пользователя в клиенте Teams пользователя.

Также требуется настройка гибридного подключения и перенос всех пользователей в облако перед удалением локального развертывания Skype для бизнеса.  После настройки гибридного подключения вы можете выбрать перенос пользователей в облако на основе своего расписания и бизнес-потребностей. В случае прямой маршрутизации вы можете использовать локальную инфраструктуру голосовой связи при переходе в облако и после завершения миграции.

В этом разделе описываются требования к инфраструктуре и системе, необходимые для настройки гибридного подключения между существующими локальной Skype для бизнеса Server и Teams.

После прочтия этой темы и готовности к настройке гибридного подключения см. в разделе Настройка гибридного подключения между Skype для бизнеса Server [и Teams](configure-hybrid-connectivity.md). Разделы конфигурации предоставляют пошаговую инструкцию по настройке гибридного подключения между локальной развертыванием и Teams.


## <a name="implications-of-the-retirement-of-skype-for-business-online"></a>Последствия выхода на пенсию Skype для бизнеса Online
Важно помнить, что как до, так и после выхода на пенсию Skype для бизнеса Online пользователи, Skype для бизнеса Server локальной сети, могли использовать Teams, но они не могли быть TeamsOnly. (Локально пользователи по умолчанию находятся в режиме Islands). Пользователи могут испытывать все преимущества Teams, в частности федерации, поддержки PSTN и уверенности в том, что все входящие чаты и вызовы приземлились в Teams, когда они находятся в режиме TeamsOnly. 

Выход на Skype для бизнеса Online не влияет на существующий жизненный цикл поддержки Skype для бизнеса Server или Lync Server 2013.  Однако выход из Skype для бизнеса Online повлиял на некоторые аспекты перехода в облако клиентов с локальной Skype для бизнеса Server или Lync Server 2013, включая существующие гибридные организации. Использование гибрида в качестве средства перехода из локального в облако (например, TeamsOnly) не влияет на выход Skype для бизнеса Online.

До выхода на пенсию Skype для бизнеса Online гибридные организации могли состоять из трех основных типов пользователей: 
- Локальное использование пользователей (которые могут или не могут использовать Teams, но не в режиме Teams только) 
- Интернет-пользователи с любым режимом сосуществования, кроме TeamsOnly
- Пользователи TeamsOnly.

Однако после выхода Skype для бизнеса Online гибридные организации могут состоять только из двух основных типов пользователей: 
- Пользователи на месте (Кто могут использовать Teams, но не в режиме TeamsOnly)
- Teams только пользователи. 

Чтобы организации перемещались с Skype для бизнеса Server или Lync Server 2013 на Teams, они по-прежнему должны настраивать и настраивать гибрид с помощью того же набора *инструментов, как* и до выхода на пенсию. При перемещении пользователя из локального в TeamsOnly `-MoveToTeams` больше не требуется указывать переключатель `Move-CsUser`. Ранее, если этот переключатель не был указан, пользователи переходили из локального Skype для бизнеса Server в Skype для бизнеса Online, и их режим оставался неизменным. Однако, поскольку Skype Business Online был снят, `Move-CsUser` перемещение пользователя из локального облака в облако автоматически назначает режим TeamsOnly и инициирует преобразование собраний из локального в Teams собрания, независимо от того,  `-MoveToTeams` задан ли переключатель. Это также означает, что организации с Lync Server 2013, `MoveToTeams` у которых никогда не было переключателя, могут перемещать пользователей напрямую в TeamsOnly из локального помещения. 

Аналогичным образом, если новый пользователь создается непосредственно в Microsoft 365, а не локально, этот пользователь будет автоматически Teams только режим независимо от режима клиента. Имейте в виду, что в гибридной организации новые пользователи должны создаваться в локальном Active Directory (а затем синхронизироваться с Microsoft 365), а не непосредственно создавать пользователя в Microsoft 365, чтобы убедиться, что локально пользователи могут маршрутизировать нового пользователя.

Режимы сосуществования по-прежнему существуют после выхода Skype для бизнеса Online. Как и раньше, пользователям с учетными записями, Skype для бизнеса Server в локальном режиме, может быть назначен любой режим сосуществования, кроме TeamsOnly. После выхода на пенсию, однако, пользователи, в которых находится интернет, могут быть только TeamsOnly (в отличие от настоящего времени, в котором Skype для бизнеса Online пользователи могут быть в любом режиме). Невозможно назначить режим, кроме TeamsOnly, пользователю, который находится в интернете.


> [!Important]
> Существующие гибридные организации с пользователями, Skype для бизнеса в Интернете, которые не являются TeamsOnly должны сосредоточиться на обновлении этих пользователей, чтобы Teams только режим как можно скорее. Если в вашей организации по-прежнему находятся пользователи Skype для бизнеса *Online*, которые не являются TeamsOnly, может быть назначено обновление с помощью Майкрософт для перехода этих пользователей в TeamsOnly. **Обновления microsoft Assisted не влияют на пользователей, которые Skype для бизнеса Server локально.** Уведомления о планировании будут заранее отправлены гибридным клиентам с пользователями, которые находятся в Skype для бизнеса Online, прежде чем они будут обновлены до Teams.

## <a name="about-shared-sip-address-space-functionality"></a>Об общих функциях пространства адресов SIP

<a name="BKMK_Overview"> </a>

Благодаря гибридному подключению между локальной развертыванием Skype для бизнеса Server и Teams, некоторые пользователи могут быть размещены в локальном помещении, а некоторые пользователи размещены в Интернете.

Этот тип конфигурации зависит от общих функций пространства адресов SIP и иногда именуется "разделенным доменом", т. е. пользователи домена, например contoso.com, делятся между использованием Skype для бизнеса Server в помещениях и Teams, как показано на следующей схеме:

![гибридный Skype для бизнеса подключения — разделенный домен.](../../sfbserver2019/media/plan-hybrid-connectivity-2019-1.png)

При настройке общего пространства адресов SIP:

- Azure Active Directory Подключение используется для синхронизации локального каталога с Microsoft 365.
- Пользователи, которые находятся в помещении, взаимодействуют с локальной Skype для бизнеса серверами.
- Пользователи, которые находятся в сети, взаимодействуют с Teams.
- Пользователи из обеих сред могут общаться друг с другом.
- Локальное active Directory является авторитетным. Сначала все пользователи должны создаваться в локальном Active Directory, а затем синхронизироваться с Azure AD. Даже если предполагается, что пользователь будет дома в Интернете, сначала необходимо создать пользователя в локальной среде, а затем переместить пользователя в интернет, чтобы убедиться, что он обнаружен локально пользователями.

Перед тем как переселить пользователя в Интернет, пользователю необходимо Teams лицензию, а также Skype для бизнеса Online (план 2). **Назначение лицензии Skype для бизнеса Online требуется даже после выхода на пенсию Skype для бизнеса Online.** Если пользователи хотят использовать дополнительные функции в Интернете, например аудиоконференцию или телефонная система, необходимо назначить им соответствующую лицензию в Microsoft 365.

## <a name="hybrid-connectivity-infrastructure-requirements"></a>Требования к инфраструктуре гибридной инфраструктуры подключения

<a name="BKMK_Infrastructure"> </a>

Чтобы реализовать гибридную связь между локальной средой и Microsoft 365 службами связи, необходимо выполнить следующие требования к инфраструктуре:

- Отдельное локальное развертывание Skype для бизнеса Server или Lync Server, развернутых в поддерживаемой топологии. В [этом разделе см. требования](plan-hybrid-connectivity.md#BKMK_Topology) топологии.

- Организация Microsoft 365 с Teams.
    > [!NOTE]
    > Для гибридной конфигурации с локальной развертыванием можно использовать только одного клиента.
    
- Azure Active Directory Подключение синхронизировать локальное каталог с Microsoft 365. Дополнительные сведения см. в [Подключение Azure AD: Учетные записи и разрешения](/azure/active-directory/connect/active-directory-aadconnect-accounts-permissions).

- Skype для бизнеса Server средства администрирования. Они необходимы для перемещения пользователей из локального в облако. Эти средства должны быть установлены на сервере с доступом как к локальному развертыванию, так и к Интернету.
- Средства администрирования в Интернете. Вы можете использовать центр администрирования Teams или Windows PowerShell для управления Teams. Чтобы использовать PowerShell для управления Teams, скачайте и установите модуль Teams PowerShell. (Подключенный Skype для бизнеса подключений к Сети был отменен).
- Необходимо включить общее пространство адресов SIP и настроить локальное развертывание для использования Microsoft 365 в качестве поставщика хостинга. Дополнительные сведения о действиях, необходимых для настройки гибридного подключения, см. в дополнительных сведениях [о настройке гибридного подключения](configure-hybrid-connectivity.md).

После настройки гибридного подключения можно переместить пользователей в Teams. Дополнительные сведения см. в [дополнительных сведениях о](move-users-from-on-premises-to-teams.md) том, как переместить пользователей из локального помещения в Teams.

## <a name="server-version-requirements"></a>Требования к версии сервера

<a name="BKMK_Topology"> </a>

Чтобы настроить развертывание для гибридного Teams с помощью **Teams**, необходимо иметь одну из следующих поддерживаемых топологий:

- Развертывание Skype для бизнеса Server 2019, в котором на всех серверах работает Skype для бизнеса Server 2019.
- Развертывание Skype для бизнеса Server 2015, в котором на всех серверах работает Skype для бизнеса Server 2015.
- Развертывание Lync Server 2013 со всеми серверами под управлением Lync Server 2013.  Однако, если требуется гибридная голосовая связь, необходимо использовать смешанную топологию версий, как отмечено ниже.
- Развертывание с максимальным количеством 2 различных версий сервера, как указано ниже:
  - Skype для бизнеса Server 2015 и Skype для бизнеса Server 2019 г.
  - Lync Server 2013 и Skype для бизнеса Server 2019
  - Lync Server 2013 и Skype для бизнеса Server 2015

*Если гибридный* голос требуется в любой топологии, то в 2015 или более позднем Skype для бизнеса должен быть запущен как край федерации, так и пул, связанный с федерацией SIP. Пользователи могут оставаться в пуле Lync 2013, если он существует. Дополнительные сведения см. в [материале Plan your voice solution](/MicrosoftTeams/cloud-voice-landing-page).

> [!NOTE]
> - Перемещение пользователей между локальной развертыванием и Teams теперь требует протокола проверки подлинности OAuth. Ранее OAuth было рекомендовано, но не требовалось. Skype для бизнеса Server 2019 и Skype для бизнеса Server 2015 cu12 (KB 3061064) уже требуют OAuth. Если вы используете Skype для бизнеса Server 2015 с cu8 до CU11, необходимо передать переключатель -UseOAuth, который обеспечивает проверку подлинности локального кода с помощью OAuth или желательно обновление до CU12. Если вы используете версию Skype для бизнеса Server 2015 до CU8, необходимо обновить до CU12 или более поздней версии. Если вы используете Lync Server 2013, сначала необходимо обновить до накопительного обновления Lync Server 2013 10 Hotfix 5 (KB 2809243) или позже.
> - Lync Server 2010 не поддерживается Teams.


## <a name="multi-forest-support"></a>Поддержка нескольких лесов

<a name="BKMK_MultiForest"> </a>

Корпорация Майкрософт поддерживает следующие типы гибридных сценариев с несколькими лесами:

- **Топология лесов ресурсов.** В этом типе топологии существует один лес, Skype для бизнеса Server (лес ресурсов), и есть один или несколько дополнительных лесов, в которых размещены удостоверения учетных записей, которые Skype для бизнеса Server в лесу ресурсов. Как правило, пользователи могут Skype для бизнеса в другом лесу, если будут выполнены следующие требования:
  - Пользователи должным образом синхронизируются с лесом, в котором Skype для бизнеса. В гибридных конфигурациях это означает, что пользователи должны синхронизироваться с отключенными объектами пользователей.
  - Лес, Skype для бизнеса должен доверять лесу, содержащим пользователей.
    Дополнительные сведения о гибридных сценариях лесов ресурсов см. в материале [Развертывание топологии лесов ресурсов для гибридных Skype для бизнеса](configure-a-multi-forest-environment-for-hybrid.md).

- **Несколько развертывание Skype для бизнеса Server в нескольких лесах.** Такая конфигурация может возникать в результате сценариев слияния и приобретения, а также в более сложных предприятиях. Консолидация всех пользователей из локального в облако в одной Microsoft 365 организации может быть достигнута для любой организации с несколькими Skype для бизнеса развертываниями при условии, что будут выполнены следующие ключевые требования:
  - Должна быть задействована как минимум одна Microsoft 365 организация. Консолидация в сценариях с более чем одной организацией не поддерживается.
  - В любой момент времени только один лес Skype для бизнеса может быть в гибридном режиме (общее пространство адресов SIP). Все остальные Skype для бизнеса леса должны полностью оставаться на территории (и, предположительно, федератированы друг с другом). Обратите внимание, что эти другие локальной организации могут синхронизироваться с AAD при желании с новыми функциями для отключения доменов [SIP](/powershell/module/skype/disable-csonlinesipdomain) в Интернете, доступных по данным на декабрь 2018 г.

    Клиенты с развертыванием Skype для бизнеса в нескольких лесах должны полностью перенести каждый Skype для бизнеса лес по отдельности в организацию Microsoft 365 с помощью функции split-domain (Shared SIP Address Space). После завершения миграции леса клиенты должны отключить гибрид с локальной развертыванием, прежде чем перейти к миграции следующего локального Skype для бизнеса развертывания. Кроме того, перед переносом в облако, локально пользователи остаются в федерарном состоянии с любыми пользователями, которые не представлены в локальном каталоге одного пользователя. Дополнительные сведения см. в [материале Cloud consolidation for Teams и Skype для бизнеса](cloud-consolidation.md).

## <a name="federation-requirements"></a>Требования федерации

<a name="BKMK_Federation"> </a>

При настройке Skype для бизнеса гибридного режима необходимо обеспечить, чтобы локальное и сетевое окружение могли федератировать друг с другом.  Интернет-среда по умолчанию имеет открытую федерацию; в локальной среде часто по умолчанию закрыта федерация.  

Для успешного настройки гибридного развертывания необходимо соблюдение следующих требований:

- Соответствие домена должно быть настроено одинаково для локального развертывания и Microsoft 365 организации. Если обнаружение партнера включено в локальном развертывании, необходимо настроить открытую федерацию для вашей организации в Интернете. Если обнаружение партнера не включено, необходимо настроить закрытую федерацию для вашей организации в Интернете.
- Список заблокированных доменов в локальном развертывании должен полностью совпадать со списком заблокированных доменов для интерактивного клиента.
- Список разрешенных доменов в локальном развертывании должен полностью совпадать со списком разрешенных доменов для интерактивного клиента.
- Федерация должна быть включена для внешних коммуникаций для клиента в Интернете.

## <a name="network-considerations"></a>Сеть

В следующих разделах описываются соображения для:

- Параметры DNS
- Соображения брандмауэра

### <a name="dns-settings-for-hybrid-deployments"></a>Параметры DNS для гибридного развертывания

<a name="BKMK_DNS"> </a>

При создании записей DNS для гибридных развертывания все Skype для бизнеса DNS-записи должны указать на локальной инфраструктуры. ### <a name="firewall-considerations-for-hybrid-deployments"></a>Соображения брандмауэра для гибридных развертывание

<a name="BKMK_Firewall"> </a>

Компьютеры в сети должны поддерживать выполнение стандартных уточняющих интернет-запросов DNS. Если эти компьютеры могут осуществлять доступ к стандартным интернет-сайтам, сеть отвечает этому требованию.

В зависимости от расположения Microsoft Online Services центра обработки данных необходимо также настроить сетевые брандмауэрные устройства для нахождения подключений на основе имен доменных имен под диктовки (например, \*весь трафик с .outlook.com). Если брандмауэры организации не поддерживают конфигурации имен подмастерьев, необходимо вручную определить диапазоны IP-адресов, которые вы хотите разрешить, и указанные порты.

Дополнительные сведения, включая сведения о портах и требованиях протокола, см. в Microsoft 365 [URL-адресов и диапазонов IP-адресов](/microsoft-365/enterprise/urls-and-ip-address-ranges).
