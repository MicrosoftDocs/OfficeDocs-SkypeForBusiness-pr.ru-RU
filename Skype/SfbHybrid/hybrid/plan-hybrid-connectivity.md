# <a name="plan-hybrid-connectivity-between-skype-for-business-server-and-microsoft-365-or-office-365"></a>Планирование гибридного подключения между Skype для бизнеса Server и Microsoft 365 или Office 365

В этом разделе вы узнаете, как планировать гибридное подключение между Skype для бизнеса Server и Teams или Skype для бизнеса Online. Настройка гибридного подключения — это первый этап переноса локальной среды в облако.

Если у вас есть пользователи локальной среды Skype для бизнеса, которые (параллельно) используют Teams, то эти пользователи не смогут взаимодействовать с пользователями Skype для бизнеса или общаться с пользователями федеративных организаций из своего клиента Teams. Чтобы воспользоваться этими возможностями в Teams, этих пользователей нужно перенести из локальной среды Skype для бизнеса в облако, что требует настройки гибридного режима Skype для бизнеса. Кроме того, для лучшего интерфейса эти пользователи должны быть в режиме "Только Teams", что гарантирует, что все входящие звонки и чаты любого пользователя посетят клиент Teams пользователя.

Также требуется настройка гибридного подключения и перенос всех пользователей в облако перед удалением локального развертывания Skype для бизнеса.  После настройки гибридного подключения вы можете выбрать перенос пользователей в облако на основе своего расписания и бизнес-потребностей. В случае прямой маршрутизации вы можете использовать локальную инфраструктуру голосовой связи при переходе в облако и после завершения миграции.

В этом разделе описываются требования к инфраструктуре и системе, необходимые для настройки гибридного подключения между существующим развертыванием Skype для бизнеса Server и Teams или Skype для бизнеса Online.

Прочитав этот раздел и подготовив его к настройке гибридного подключения, см. статью "Настройка гибридного подключения между Skype для бизнеса Server и [Microsoft 365 или Office 365".](configure-hybrid-connectivity.md) Разделы конфигурации предоставляют пошаговую инструкцию по настройке гибридного подключения между вашим локальном развертыванием и Teams или Skype для бизнеса Online.

> [!Important]
> Skype для бизнеса Online будет отменен 31 июля 2021 г., после чего служба станет недоступна.  Кроме того, подключение по STN между локальной средой через Skype для бизнеса Server или Cloud Connector Edition и Skype для бизнеса Online больше не будет поддерживаться.  Узнайте, как подключить свою локальной телефонной сети к Teams с помощью [прямой маршрутки.](https://docs.microsoft.com/MicrosoftTeams/direct-routing-landing-page)

## <a name="about-shared-sip-address-space-functionality"></a>Общие функции адресного пространства SIP

<a name="BKMK_Overview"> </a>

 Благодаря гибридному подключению между локальной развертыванием Skype для бизнеса Server и Teams или Skype для бизнеса Online некоторые пользователи могут быть размещены локально, а некоторые — в Интернете.

Этот тип конфигурации зависит от общих функций адресного пространства SIP и иногда называется "разделенным доменом". Это означает, что пользователи домена, например contoso.com, разделены между использованием локального Skype для бизнеса Server и Teams или Skype для бизнеса Online, как показано на следующей схеме:

![Гибридное подключение Skype для бизнеса — разделенный домен](../../sfbserver2019/media/plan-hybrid-connectivity-2019-1.png)

При настройке общего адресного пространства SIP:

- Azure Active Directory Connect используется для синхронизации локального каталога с Microsoft 365 или Office 365.
- Пользователи, которые находятся локально, взаимодействуют с локальной skype для бизнеса servers.
- Пользователи, которые находятся в Сети, могут взаимодействовать со службами Skype для бизнеса Online или Teams.
- Пользователи из обеих сред могут общаться друг с другом.
- Локальное управление Active Directory является до полномочного. Сначала необходимо создать всех пользователей в локальной службе Active Directory, а затем синхронизировать их с Azure AD. Даже если вы собираетесь, чтобы пользователь был в сети, необходимо сначала создать пользователя в локальной среде, а затем переместить пользователя в интернет, чтобы обеспечить его обнаружение пользователями локальной среды.

Перед тем как пользователь может быть перемещен в сеть, пользователю должна быть назначена лицензия Skype для бизнеса Online (план 2). Если пользователь будет использовать Teams, ему также должна быть назначена лицензия Teams (и лицензия Skype для бизнеса должна оставаться включенной). Если пользователи хотят использовать дополнительные сетевые функции, например аудиоконференцию или телефонную систему, необходимо назначить им соответствующую лицензию в Microsoft 365 или Office 365.

## <a name="hybrid-connectivity-infrastructure-requirements"></a>Требования к инфраструктуре гибридных подключений

<a name="BKMK_Infrastructure"> </a>

Чтобы реализовать гибридное подключение между локальной средой и службами связи Microsoft 365 или Office 365, необходимо выполнить следующие требования к инфраструктуре:

- Одно локальное развертывание Skype для бизнеса Server или Lync Server, развернутого в поддерживаемой топологии. См. [требования к топологии](plan-hybrid-connectivity.md#BKMK_Topology) в этом разделе.

- Организация Microsoft 365 или Office 365 с включенной поддержкой Skype для бизнеса Online.
    > [!NOTE]
    > Для гибридной конфигурации в локальном развертывании можно использовать только один клиент.
    
- Azure Active Directory Connect для синхронизации локального каталога с Microsoft 365 или Office 365. Дополнительные сведения [см. в подключении Azure AD Connect: учетные записи и разрешения.](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-accounts-permissions)

- Средства администрирования Skype для бизнеса Server. Они необходимы для перемещения пользователей из локальной сети в облако. Эти средства должны быть установлены на сервере с доступом как к локальному развертыванию, так и к Интернету.
- Сетевые средства администрирования. Вы можете использовать Центр администрирования Teams или Windows PowerShell для управления Teams и Skype для бизнеса Online. Чтобы использовать PowerShell для управления Teams или Skype для бизнеса Online, скачайте и установите соединитель Skype для бизнеса Online.
- Необходимо включить общее адресное пространство SIP, а локальное развертывание должно быть настроено на использование Microsoft 365 или Office 365 в качестве поставщика услуг размещения. Дополнительные сведения о действиях, необходимых для настройки гибридного подключения, см. в [подстройке "Настройка гибридного подключения".](configure-hybrid-connectivity.md)

После настройки гибридного подключения можно переместить пользователей в Teams или Skype для бизнеса Online. Дополнительные сведения см. в подсети "Перемещение пользователей из локальной сети в [Teams"](move-users-from-on-premises-to-teams.md) и "Перемещение пользователей из локальной сети [в Skype для бизнеса Online".](move-users-from-on-premises-to-skype-for-business-online.md)

## <a name="server-version-requirements"></a>Требования к версии сервера

<a name="BKMK_Topology"> </a>

Чтобы настроить гибридное развертывание с Teams или Skype для бизнеса **Online,** необходимо иметь одну из следующих поддерживаемых технологий:

- Развертывание Skype для бизнеса Server 2019, в котором на всех серверах работает Skype для бизнеса Server 2019.
- Развертывание Skype для бизнеса Server 2015, в котором на всех серверах работает Skype для бизнеса Server 2015.
- Развертывание Lync Server 2013 со всеми серверами под управлением Lync Server 2013.  Однако если требуется гибридное голосовое подключение, необходимо использовать смешанную топологию версий, как было отмечено ниже.
- Развертывание с не более чем 2 различными версиями серверов, как указано ниже:
  - Skype для бизнеса Server 2015 и Skype для бизнеса Server 2019
  - Lync Server 2013 и Skype для бизнеса Server 2019
  - Lync Server 2013 и Skype для бизнеса Server 2015

Если в какой-либо топологии требуется гибридная голосовая *система,* то как на edge-сервере, назначенном в качестве сервера федерации, так и в пуле, связанном с федерацией SIP, должен работать Skype для бизнеса 2015 или более поздней. Пользователи могут оставаться в пуле Lync 2013, если он существует. Дополнительные сведения см. в подсистеме ["Планирование телефонной системы с подключением к STN" в Skype для бизнеса Server.](https://docs.microsoft.com/skypeforbusiness/skype-for-business-hybrid-solutions/plan-your-phone-system-cloud-pbx-solution/plan-phone-system-with-on-premises-pstn-connectivity)

Для обмена мгновенными сообщениями и собраний в Skype для бизнеса Online поддерживаются следующие topologies, включающие **Lync Server 2010.** Topologies that include **Lync Server 2010 are not supported for hybrid voice nor Teams.**

- Смешанное развертывание Lync Server 2010 и Skype для бизнеса Server 2015
- Смешанное развертывание Lync Server 2010 и Lync Server 2013
- Развертывание Lync Server 2010 со всеми серверами под управлением Lync Server 2010 с последними накопительными обновлениями.

На федеративном сервере и сервере следующего прыжка с федеративного сервера должны быть запущены Lync Server 2010 с последними накопительными обновлениями. Средства администрирования Skype для бизнеса Server 2015 или Lync Server 2013 должны быть установлены по крайней мере на одном сервере или рабочей станции управления.

## <a name="multi-forest-support"></a>Поддержка нескольких лесов

<a name="BKMK_MultiForest"> </a>

Корпорация Майкрософт поддерживает следующие типы гибридных сценариев с несколькими лесами:

- **Топология леса ресурсов.** В такой топологии существует один лес, в котором размещен Skype для бизнеса Server (лес ресурсов), а также один или несколько дополнительных лесов, в которых размещены удостоверения учетных записей, которые имеют доступ к Skype для бизнеса Server в лесу ресурсов. Как правило, пользователи могут получить доступ к функциям Skype для бизнеса в другом лесу, если выполнены следующие требования:
  - Пользователи надлежащим образом синхронизируются с лесом, в котором размещен Skype для бизнеса. В гибридных конфигурациях это означает, что пользователи должны быть синхронизированы как отключенные объекты пользователей.
  - Лес, в котором размещен Skype для бизнеса, должен доверять лесу, содержащим пользователей.
    Дополнительные сведения о гибридных сценариях с лесом ресурсов см. в сведениях о развертывании топологии леса [ресурсов для гибридного развертывания Skype для бизнеса.](configure-a-multi-forest-environment-for-hybrid.md)

- **Несколько развертываний Skype для бизнеса Server в нескольких лесах.** Такая конфигурация может возникнуть в результате сценариев слияния и приобретения, а также в более сложных предприятиях. Консолидацию всех пользователей из локальной сети в облако в одной организации Microsoft 365 или Office 365 можно достичь для любой организации с несколькими развертываниями Skype для бизнеса при условии, что выполнены следующие ключевые требования:
  - Должна быть задействована не более одной организации Microsoft 365 или Office 365. Консолидация в сценариях с более чем одной организацией не поддерживается.
  - В любой момент времени только один локального леса Skype для бизнеса может быть в гибридном режиме (общее адресное пространство SIP). Все остальные локального леса Skype для бизнеса должны оставаться полностью в локальной (и, предположительно, федераированной друг с другом). Обратите внимание, что эти другие локальной организации могут синхронизироваться с AAD при наличии новых функций для отключения сетевых [доменов SIP,](https://docs.microsoft.com/powershell/module/skype/disable-csonlinesipdomain) доступных в декабре 2018 г.

    Клиенты с развертываниями Skype для бизнеса в нескольких лесах должны полностью перенести каждый лес Skype для бизнеса по отдельности в организацию Microsoft 365 или Office 365 с помощью функций разделенного домена (общее адресное пространство SIP), а затем отключить гибридную функциональность с локальной развертыванием, прежде чем перейти к миграции следующего локального развертывания Skype для бизнеса. Кроме того, до миграции в облако пользователи локальной сети остаются в федераированном состоянии с пользователями, которые не представлены в локальном каталоге одного пользователя. Дополнительные сведения см. в консолидации [облака для Teams и Skype для бизнеса.](cloud-consolidation.md)

## <a name="federation-requirements"></a>Требования к федерации

<a name="BKMK_Federation"> </a>

При настройке гибридного режима Skype для бизнеса необходимо обеспечить возможность федерации между локальной и сетевой средами.  В сетевой среде по умолчанию открыта федерация; В локальной среде часто по умолчанию закрыта федерация.  

Для успешного настройки гибридного развертывания необходимо соблюдение следующих требований:

- Для локального развертывания и организации Microsoft 365 или Office 365 необходимо настроить одинаковое соответствие домена. Если в локальном развертывании включено обнаружение партнеров, для вашей сетевой организации должна быть настроена открытая федерация. Если обнаружение партнеров не включено, для вашей сетевой организации должна быть настроена закрытая федерация.
- Список заблокированных доменов в локальном развертывании должен полностью совпадать со списком заблокированных доменов для интерактивного клиента.
- Список разрешенных доменов в локальном развертывании должен полностью совпадать со списком разрешенных доменов для интерактивного клиента.
- Федерация должна быть включена для внешних коммуникаций для интерактивного клиента.

### <a name="dns-settings-for-hybrid-deployments"></a>Параметры DNS для гибридных развертывание

<a name="BKMK_DNS"> </a>

При создании записей DNS для гибридных развертывание все внешние записи DNS Skype для бизнеса должны указать на локальной инфраструктуре. Дополнительные сведения о необходимых записях DNS можно найти в требованиях [К DNS для Skype для бизнеса Server.](../../sfbserver/plan-your-deployment/network-requirements/dns.md)

Кроме того, необходимо убедиться, что разрешение DNS, описанное в следующей таблице, работает в локальном развертывании. (Если вы уже настроили федерацию для локальной конфигурации, скорее всего, они уже есть.)

|Запись DNS  <br/> |Разрешаемая с помощью  <br/> |Требование DNS  <br/> |
|:-----|:-----|:-----|
|DNS-запись SRV для _sipfederationtls._tcp.\<sipdomain.com\> для всех поддерживаемых доменов SIP, разрешающих доступ к внешним IP-адресам по границе доступа  <br/> |Edge server(s)  <br/> |Включить федеративную связь в гибридной конфигурации. Edge Server needs to know where to route federated traffic for the SIP domain that is split between on premises and online.  <br/> Должен использовать строгое DNS-имя, совпадающие между доменом в имени пользователя и записью SRV.  <br/> |
|DNS-записи A для FQDN службы веб-конференций edge, например, webcon.contoso.com разрешения на внешние IP-адреса edge веб-службы.  <br/> |Компьютеры пользователей, подключенных к внутренней корпоративной сети  <br/> |Позволяет пользователям из Интернета представлять или просматривать контент на локальном собрании. Содержимое включает файлы PowerPoint, доски, опросы и общие заметки.  <br/> |

В зависимости от конфигурации DNS в организации может потребоваться добавить эти записи во внутреннюю зону DNS для соответствующих доменов SIP, чтобы обеспечить внутреннее разрешение DNS для этих записей.

### <a name="firewall-considerations-for-hybrid-deployments"></a>Аспекты брандмауэра для гибридных развертывание

<a name="BKMK_Firewall"> </a>

Компьютеры в сети должны поддерживать выполнение стандартных уточняющих интернет-запросов DNS. Если эти компьютеры могут осуществлять доступ к стандартным интернет-сайтам, сеть отвечает этому требованию.

В зависимости от расположения центра обработки Microsoft Online Services необходимо также настроить сетевые брандмауэры на прием подключений на основе поддиапных доменных имен (например, весь трафик с \* .outlook.com). Если брандмауэры организации не поддерживают конфигурации с подданными именами, необходимо вручную определить диапазоны IP-адресов, которые необходимо разрешить, и указанные порты.

Дополнительные сведения, включая сведения о портах и требованиях протокола, см. в url-адресах и диапазонах IP-адресов [Microsoft 365 и Office 365.](https://go.microsoft.com/fwlink/p/?LinkId=252942)
