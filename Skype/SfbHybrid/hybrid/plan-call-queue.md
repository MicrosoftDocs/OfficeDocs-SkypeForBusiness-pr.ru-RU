# <a name="plan-cloud-call-queues"></a>Планирование очередей звонков в облаке

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

Очередь облачных вызовов — это служба, которая принимает звонки клиентов, воспроизводит приветствие, а затем помещает эти вызовы в очередь ожидания при поиске предварительно настроенного списка агентов для ответа на эти вызовы. Набор агентов можно определить в списках рассылки или группах безопасности с поддержкой почты. В вашей организации может быть одна или несколько очередей звонков. Очереди звонков обычно используются в сочетании с автосекретарями.

Кроме того, облачные очереди вызовов могут предоставлять:

- Музыка, пока вызывающие объекты ожидают удержания
- Настроенные параметры для максимального размера очереди вызовов, времени ожидания и параметров обработки вызовов

Каждой очереди вызовов назначается учетная запись **ресурса (см**. раздел "Настройка учетных записей [ресурсов") в](configure-onprem-ra.md) системе Skype для бизнеса Server 2019, которая будет связана непосредственно с очередью звонков в Центре администрирования Microsoft Teams. [Дополнительные сведения о](/MicrosoftTeams/create-a-phone-system-call-queue) том, что такое очереди вызовов и какие параметры и функции существуют для очередей вызовов, см. в статье "Создание облачной очереди вызовов".

> [!NOTE]
> В очередь звонков можно назначить несколько номеров телефонов, но это должны быть номера служб Майкрософт, номера прямой маршрутизации или гибридные номера.

## <a name="requirements"></a>Требования

В следующих требованиях предполагается, что вы уже развернули Skype для бизнеса Server 2019 в поддерживаемой топологии.  Ваши требования зависят от вашего сценария:

- Для новой конфигурации очередей вызовов Облака выполните действия, описанные в разделе "Настройка [учетных записей ресурсов"](configure-onprem-ra.md). Вам потребуется создать учетные записи ресурсов в Интернете или Skype для бизнеса Server 2019 г., а также связать номер телефона с очередью звонков.

В дополнение к приведенным выше требованиям необходимо настроить следующие требования для подключения к службе очереди вызовов Microsoft Cloud:

- Гибридное подключение. Если вы уже развернули Skype для бизнеса Server и хотите включить очереди облачных вызовов для локальных пользователей, необходимо убедиться, что у вас настроено гибридное подключение между локальной и сетевой средами. Иногда это называется конфигурацией разделенного домена.

   Дополнительные сведения см. в статье "Планирование гибридного подключения между [Skype для бизнеса Server и Microsoft 365 или Office 365](plan-hybrid-connectivity.md) и настройка гибридного подключения между [Skype для бизнеса Server и Microsoft 365 или Office 365](configure-hybrid-connectivity.md).

- Если вы назначаете номер телефона учетной записи ресурса, вы можете использовать бесплатную Телефонная система Microsoft Teams **учетной записи** ресурса. Это предоставляет возможности телефонной системы для телефонных номеров на уровне организации и позволяет создавать функции автосекретаря и очереди звонков.

- Создайте локальную учетную запись [ресурса](configure-onprem-ra.md) для каждой очереди звонков и при необходимости назначьте лицензию и номер телефона.  

При наличии надежной структуры, соответствующей вашим потребностям, и скрипта, который эффективно помогает клиентам, перейдите к  [настройке учетных записей ресурсов](configure-onprem-ra.md).
