# <a name="plan-call-data-connector"></a>Планирование соединителя данных звонка

## <a name="overview"></a>Общие сведения

В этом разделе описываются преимущества, вопросы планирования и требования для реализации соединители данных звонков Skype для бизнеса Server. Дополнительные сведения о настройке соединители данных о вызовах см. в подключении ["Настройка соединители данных вызовов".](configure-call-data-connector.md)


Соединителю обработки данных вызовов значительно упрощается мониторинг вызовов в гибридной среде, так как вам больше не нужно использовать различные наборы средств локальной и сетевой среды для отслеживания качества вызовов всех пользователей. Независимо от того, где находятся пользователи : локально или через Интернет, вы можете просмотреть качество вызовов для всей организации в Интернете.

С помощью соединители данных о вызовах можно выполнять следующие задачи с помощью одного набор инструментов:

- Отслеживайте пользовательский интерфейс в Microsoft Teams, Skype для бизнеса Online и Skype для бизнеса Server.

- Просмотр и устранение неполадок в сети.

- Назначьте роли службы справки и администратора для службы аналитики вызовов, чтобы сотрудники службы могли просматривать и устранять неполадки в их областях ответственности.

С помощью соединители данных звонков Skype для бизнеса Server перенагваширует данные звонков в облачную службу, чтобы вы могли использовать средства аналитики звонков Skype для бизнеса Online и панели мониторинга качества звонков (CQD), как показано на следующей схеме:

![Облачная голосовая почта SfB](../../sfbserver2019/media/call-data-connector-plan-1.png)

Сервер и данные качества обслуживания (QoE), и данные регистрации вызовов (CDR) в веб-службу.

Средства аналитики звонков и CQD позволяют отслеживать качество звонков и устранять неполадки подключения к Службам Microsoft Teams и Skype для бизнеса следующим образом:

- Аналитика звонков фокусируется на проблемах с качеством при определенных звонках. Здесь показаны подробные сведения о звонках и собраниях для каждого пользователя в учетной записи Skype для бизнеса.  С помощью аналитики звонков можно назначить разрешения оператору справки, который сможет отслеживать звонки, не имея доступа к остальной части Центра администрирования Skype для бизнеса.

- Панель мониторинга качества вызовов фокусируется на производительности сети и проблемах в организации. Администраторы Skype для бизнеса и сетевые инженеры используют это средство для устранения неполадок и оптимизации производительности сети.

Дополнительные сведения см. в [аналитике вызовов и панели мониторинга качества вызовов.](https://docs.microsoft.com/SkypeForBusiness/using-call-quality-in-your-organization/difference-between-call-analytics-and-call-quality-dashboard)

Конечно, вам может потребоваться хранить некоторые данные о качестве вызовов в локальной компании. Это может быть так, например, если вы используете стороне решение с настраиваемыми отчетами и процессами.  Соединители данных вызовов позволяют настроить отправку данных в веб-службу, а также сохранить копию данных на локальном сервере, как показано на следующей схеме:

## <a name="requirements"></a>Requirements

В следующих требованиях предполагается, что skype для бизнеса Server уже развернут в поддерживаемой топологии.  Дополнительные сведения о развертывании Skype для бизнеса Server и поддерживаемых топологиях см. в основных [сведениях о топологии.](https://docs.microsoft.com/SkypeForBusiness/plan-your-deployment/topology-basics/topology-basics) Чтобы настроить соединители данных о вызовах, необходимо:

- Включить гибридное подключение. Если у вас уже развернут Skype для бизнеса Server и вы хотите включить соединительную связь с данными звонков, необходимо убедиться, что между локальной и сетевой средами настроено гибридное подключение. Такая конфигурация иногда называется конфигурацией разделенного домена.

   Дополнительные сведения см. в сведениях о планировании гибридного подключения между Skype для бизнеса Server и [Microsoft 365 или Office 365](plan-hybrid-connectivity.md) и настройке гибридного подключения между Skype для бизнеса Server и [Microsoft 365 или Office 365.](configure-hybrid-connectivity.md)

- Проверка подлинности в организации Microsoft 365 или Office 365 и убедитесь, что включены следующие роли:

  - Администратор Skype для бизнеса Server
  - Глобальный администратор Microsoft 365 или Office 365

- Если вы еще этого не сделали, включите панель мониторинга качества звонков, как описано в описании "Включение и использование панели мониторинга качества звонков для Microsoft Teams и [Skype для бизнеса Online".](/microsoftteams/turning-on-and-using-call-quality-dashboard)

- Включить для мониторинга пул переднего входа с локальными базами данных LCSCdr и QoEMetrics. Без этого соединители данных вызова не будут иметь данных метрик для работы.

> [!IMPORTANT]
> Соединители данных вызовов не будут работать, если мониторинг не включен в пуле переднего входа.

- Правильно настроенная [проверка подлинности "сервер-сервер".](https://docs.microsoft.com/skypeforbusiness/manage/authentication/server-to-server-and-partner-applications)
