# <a name="move-users-from-the-cloud-to-on-premises"></a>Перемещение пользователей из облака в локальное 

При необходимости можно переместить пользователя, ранее перенесенного из локального в облако (независимо от того, используется ли Skype для бизнеса в Интернете или только для команд) обратно в локальное помещение. Чтобы переместить пользователей из режима Skype для бизнеса Online или TeamsOnly в локальное развертывание Skype для бизнеса Server, используйте командлет Move-CsUser или панель управления Skype для бизнес-серверов, которые являются локальной программой. При переходе пользователя в локальное развертывание необходимо решить, в какой пул переместить пользователя.

> [!Important]
> Если пользователь ранее был в режиме TeamsOnly, и вы используете более раную версию, чем Skype для бизнеса Server 2015 с CU8, необходимо также удалить назначение режима TeamsOnly TeamsUpgradePolicy для этого пользователя. Локальное пользователи не должны иметь mode= TeamsOnly.  Последующие версии Skype для бизнеса Server автоматически удаляют это назначение. Дополнительные сведения см. [в материале Grant-CsTeamsUpgradePolicy.](/powershell/module/skype/grant-csteamsupgradepolicy)

## <a name="prerequisites"></a>Необходимые компоненты

- Организация должна правильно настроить Azure AD Connect и синхронизировать все соответствующие атрибуты для пользователя, как описано в [Configure Azure AD Connect.](configure-azure-ad-connect.md)
- Пользователь, перемещаемый из сети в локальное, должен уже существовать в локальном Active Directory.
- Гибрид Skype для бизнеса необходимо настроить, как описано в [настройках гибрида Skype для бизнеса.](configure-federation-with-skype-for-business-online.md)

## <a name="moving-users-back-to-on-premises"></a>Перемещение пользователей в локальное помещение

После перемещения пользователя из облака обратно в локальное:

- Пользователь взаимодействует с развертыванием Skype для бизнес-сервера для своих функций. 
- Все контакты, существовавшие в Skype для бизнеса Online или Teams, переносят в Skype для бизнеса Server. Два набора контактов объединяются, а затем переносят их в локальное помещение.  Кроме того, в Teams остаются контакты, уже существующие в Teams.
- Если пользователь также использует Teams, он не сможет взаимодействовать с пользователями Skype для бизнеса и не сможет общаться с пользователями в федерадных организациях.
- Собрания в Skype для бизнеса Online не *переносят* автоматически в локальное помещение. Пользователям следует либо перенести собрания, либо при желании использовать средство миграции [собраний.](https://support.office.com/article/2b525fe6-ed0f-4331-b533-c31546fcf4d4)

### <a name="move-users-with-move-csuser"></a>Перемещение пользователей с помощью Move-CsUser

Move-CsUser доступно в локальном окне Skype для управления бизнесом PowerShell. Вы должны иметь достаточные привилегии как в локальной среде, так и в организации облачных служб (Microsoft 365 или Office 365), как описано в необходимых административных учетных [данных.](move-users-between-on-premises-and-cloud.md#required-administrative-credentials) Можно использовать одну учетную запись, которая имеет привилегии в обеих средах, или можно запустить локальное окно Skype для управления бизнес-серверами с учетными данными локального сервера и использовать параметр для указания учетных данных для учетной записи `-Credential` Microsoft 365 или Office 365 с необходимой административной ролью.

Перемещение пользователя в локальное помещение с помощью Move-CsUser:

- Укажите, что пользователь перемещается с помощью параметра Identity.
- Укажите параметр -Target с полностью квалифицированным доменным именем нужного локального пула, который будет принимать пользователя.
- Если у вас нет одной учетной записи с достаточными разрешениями как в локальной, так и в облачной службе (Microsoft 365 или Office 365), используйте параметр -credential для обеспечения учетной записи достаточными разрешениями в Microsoft 365 или Office 365.
- Если учетная запись с разрешениями в Microsoft 365 или Office 365 не заканчивается в "on.microsoft.com", необходимо указать параметр -HostedMigrationOverrideUrl с правильным значением, как описано в необходимых административных учетных данных [.](move-users-between-on-premises-and-cloud.md#required-administrative-credentials)

Следующая последовательность смдлетов может использоваться для перемещения пользователя в Skype для бизнеса Server и предполагает, что учетная запись Microsoft 365 или Office 365 является отдельной учетной записью и предоставляется в качестве ввода для Get-Credential запроса.

```PowerShell
$cred=Get-Credential
$url="https://admin1a.online.lync.com/HostedMigration/hostedmigrationService.svc"
Move-CsUser -Identity username@contoso.com -Target pool.corp.contoso.com -Credential $cred -HostedMigrationOverrideUrl $url
```

### <a name="move-users-with-the-skype-for-business-server-control-panel"></a>Перемещение пользователей с панели управления Skype для бизнес-серверов

1. Откройте панель управления Skype для бизнес-серверов.
2. В левой навигации выберите **Пользователи**.
3. Используйте **Поиск,** чтобы найти пользователя(ы), который вы хотите переместить обратно в локальное помещение.
4. Выберите пользователя(ы), а  затем из выпадаемого действия над списком выберите Перемещение выбранных пользователей в **локальное место.**
5. В мастере выберите пул пользователей, который будет принимать пользователя, и нажмите **кнопку Далее**.
6. При запросе вопишитесь в Microsoft 365 или Office 365 с учетной записью, которая заканчивается в .onmicrosoft.com и имеет достаточно разрешений.
7. Нажмите **кнопку Далее,** а **затем** еще раз, чтобы переместить пользователя.
8. Обратите внимание, что сообщения о состоянии, касающиеся успеха или сбоя, предоставляются в верхней части основного приложения панели управления, а не в мастере.

### <a name="removing-teamsonly-mode"></a>Удаление режима TeamsOnly

Если вы используете версию раньше Skype для бизнеса 2015 с cu8 и пользователь возвращается в локальном режиме TeamsOnly, необходимо удалить экземпляр UpgradeToTeams перед перемещением пользователя на локальном `TeamsUpgradePolicy` сайте. Вы можете либо явно предоставить политику в другом режиме, либо просто удалить существующее назначение политики для этого пользователя для использования глобальной политики (если глобальная политика клиента не является UpgradeToTeams).

Чтобы удалить назначение пользователя teamsUpgradePolicy, запустите следующий командлет из окна Skype для бизнеса Online PowerShell:

`Grant-CsTeamsUpgradePolicy -Identity $user -PolicyName $null`

Кроме того, чтобы назначить другой экземпляр TeamsUpgradePolicy, у которого нет mode=TeamsOnly, можно указать имя нужного экземпляра в качестве значения параметра PolicyName в командлете. Чтобы увидеть список доступных экземпляров TeamsUpgradePolicy, запустите Get-CsTeamsUpgradePolicy.
