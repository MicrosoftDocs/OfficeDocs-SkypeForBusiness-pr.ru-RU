# <a name="move-users-from-the-cloud-to-on-premises"></a>Перемещение пользователей из облака в локальное 

При необходимости можно переместить пользователя, который ранее был перенесен из локальной сети в облако (с помощью Skype для бизнеса Online или Только Teams) обратно в локальное. Чтобы переместить пользователей из Skype для бизнеса Online или режима TeamsOnly обратно в локальное развертывание Skype для бизнеса Server, используйте командлет Move-CsUser или панель управления Skype для бизнеса Server, оба из которых являются локальной программой. При обратном переходе пользователя в локальное развертывание необходимо решить, в какой пул следует переместить пользователя.

> [!Important]
> Если пользователь ранее работал в режиме TeamsOnly и вы используете более ранную версию, чем Skype для бизнеса Server 2015 с cu8, необходимо также удалить для этого пользователя назначение teamsUpgradePolicy в режиме TeamsOnly. У пользователей локальной сети не должен быть режим = TeamsOnly.  Последующие версии Skype для бизнеса Server автоматически удаляют это назначение. Дополнительные сведения [см. в поле Grant-CsTeamsUpgradePolicy.](https://docs.microsoft.com/powershell/module/skype/grant-csteamsupgradepolicy)

## <a name="prerequisites"></a>Предварительные условия

- Организация должна правильно настроить Azure AD Connect и синхронизировать все соответствующие атрибуты для пользователя, как описано в описании настройки [Azure AD Connect.](configure-azure-ad-connect.md)
- Пользователь, перемещаемый из Сети обратно в локальное, должен уже существовать в локальной службе Active Directory.
- Гибридная конфигурация Skype для бизнеса должна быть настроена, как описано в описании настройки гибридной конфигурации [Skype для бизнеса.](configure-federation-with-skype-for-business-online.md)

## <a name="moving-users-back-to-on-premises"></a>Перемещение пользователей обратно в локальное

После перемещения пользователя из облака обратно в локальное решение:

- Пользователь взаимодействует с развертыванием Skype для бизнеса Server для своих функций. 
- Все контакты, существовавшие в Skype для бизнеса Online или Teams, переносят в Skype для бизнеса Server. Два набора контактов объединяются, а затем переносят их обратно в локальное решение.  Кроме того, контакты, которые уже есть в Teams, остаются в Teams.
- Если пользователь также использует Teams, он не сможет взаимодействовать с пользователями Skype для бизнеса и не сможет взаимодействовать с пользователями в федера организациях.
- Собрания в Skype для бизнеса Online *не* переносят автоматически обратно в локальное приложение. Пользователям следует либо повторно захещать свои собрания, либо при желании использовать средство [переноса собраний.](https://support.office.com/article/2b525fe6-ed0f-4331-b533-c31546fcf4d4)

### <a name="move-users-with-move-csuser"></a>Перемещение пользователей с Move-CsUser

Move-CsUser доступен в окне PowerShell локальной оболочки управления Skype для бизнеса. Необходимо иметь достаточные привилегии как в локальной среде, так и в облачной службе (Microsoft 365 или Office 365), как описано в необходимых административных учетных [данных.](move-users-between-on-premises-and-cloud.md#required-administrative-credentials) Можно использовать одну учетную запись с привилегиями в обеих средах или запустить окно локальной оболочки управления Skype для бизнеса Server с учетными данными локальной среды и использовать параметр для указания учетных данных для учетной записи `-Credential` Microsoft 365 или Office 365 с необходимой административной ролью.

Чтобы переместить пользователя в локальное помещение с помощью Move-CsUser:

- Укажите пользователя, который будет перемещаться, с помощью параметра Identity.
- Укажите параметр -Target с полное доменное имя нужного локального пула, в который будет работать пользователь.
- Если у вас нет одной учетной записи с достаточными разрешениями как в локальной, так и в облачной службе (Microsoft 365 или Office 365), используйте параметр -credential, чтобы предоставить учетную запись с достаточными разрешениями в Microsoft 365 или Office 365.
- Если учетная запись с разрешениями в Microsoft 365 или Office 365 не заканчивается на "on.microsoft.com", необходимо указать параметр -HostedMigrationOverrideUrl с правильным значением, как описано в необходимых административных учетных [данных.](move-users-between-on-premises-and-cloud.md#required-administrative-credentials)

Для перемещения пользователя в Skype для бизнеса Server можно использовать следующую последовательность, предполагая, что учетные данные Microsoft 365 или Office 365 — это отдельная учетная запись, которая предоставляется в качестве входных данных для Get-Credential запроса.

```PowerShell
$cred=Get-Credential
$url="https://admin1a.online.lync.com/HostedMigration/hostedmigrationService.svc"
Move-CsUser -Identity username@contoso.com -Target pool.corp.contoso.com -Credential $cred -HostedMigrationOverrideUrl $url
```

### <a name="move-users-with-the-skype-for-business-server-control-panel"></a>Перемещение пользователей с помощью панели управления Skype для бизнеса Server

1. Откройте приложение панели управления Skype для бизнеса Server.
2. В области навигации слева выберите **"Пользователи".**
3. Используйте **поиск,** чтобы найти пользователей, которые вы хотите переместить обратно в локализовать.
4. Выберите пользователей, а затем в  выпадаемом списке "Действие" выберите "Переместить выбранных пользователей в **локальное место".**
5. В мастере выберите пул пользователей, в который будет работать пользователь, и нажмите кнопку **"Далее".**
6. При запросе во sign in to Microsoft 365 or Office 365 with an account that ends in .onmicrosoft.com and has sufficient permissions.
7. Нажмите **кнопку**"Далее", а затем еще раз, чтобы переместить пользователя. 
8. Обратите внимание, что сообщения о состоянии, касающиеся успешного или неудачного сбоя, предоставляются в верхней части основного приложения панели управления, а не в мастере.

### <a name="removing-teamsonly-mode"></a>Удаление режима TeamsOnly

Если вы используете версию более ранней, чем Skype для бизнеса 2015 с cu8, и пользователь перемещается обратно в локальное приложение в режиме TeamsOnly, необходимо удалить экземпляр UpgradeToTeams, прежде чем перемещать пользователя в локальное `TeamsUpgradePolicy` приложение. Вы можете явно предоставить политику в другом режиме или просто удалить существующее назначение политики для этого пользователя, чтобы использовать глобальную политику (если глобальная политика клиента не upgradeToTeams).

Чтобы удалить назначение пользователя TeamsUpgradePolicy, запустите следующий командлет в окне PowerShell Skype для бизнеса Online:

`Grant-CsTeamsUpgradePolicy -Identity $user -PolicyName $null`

Кроме того, чтобы назначить другой экземпляр TeamsUpgradePolicy без mode=TeamsOnly, можно указать имя нужного экземпляра в качестве значения параметра PolicyName в командлете. Чтобы увидеть список доступных экземпляров TeamsUpgradePolicy, запустите командную команду Get-CsTeamsUpgradePolicy.
