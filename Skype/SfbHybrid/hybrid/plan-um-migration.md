# <a name="plan-for-skype-for-business-server-and-exchange-server-migration"></a>Планирование миграции Skype для бизнеса Server и Exchange Server

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

В этом разделе описывается, что необходимо учитывать при переносе существующих развертываний Skype для бизнеса Server или Exchange Server в Exchange Online. То, что можно перенести и когда, во многом зависит от того, что вы уже настроили в организации.

## <a name="feature-changes-in-exchange-2019-and-skype-for-business-server-2019"></a>Изменения компонентов в Exchange 2019 и Skype для бизнеса Server 2019

В Exchange 2019 и Skype для бизнеса Server 2019 мы внося некоторые изменения в поддерживаемые функции.

### <a name="unified-messaging-support-in-exchange-2019"></a>Поддержка единой системы обмена сообщениями в Exchange 2019

Единая система обмена сообщениями (UM) объявлена нерекомендуемой в Exchange 2019. Это означает, что Exchange 2019 больше не предлагает следующие функции:

- Голосовая почта
- Автосекретарь

Если вы развернули роль единой системы обмена сообщениями в Exchange 2013 или службу единой системы обмена сообщениями в Exchange 2016 и хотите перейти на Exchange 2019, вам потребуется перенести голосовую почту в службу голосовая облачная почта (Майкрософт) в Microsoft 365 или Office 365. Если вы хотите перенести голосовую почту в облачная голосовая почта, ознакомьтесь с разделами [Exchange 2013/Exchange 2016 и Skype для бизнеса 2015 на Exchange 2019 и Skype для бизнеса 2019](#exchange-2013exchange-2016-and-skype-for-business-2015-to-exchange-2019-and-skype-for-business-2019) ниже.
> [!IMPORTANT]
> Если у пользователей на серверах Exchange 2013 или Exchange 2016 есть почтовые ящики с поддержкой единой системы обмена сообщениями, не перемещайте их в Exchange 2019 перед обновлением серверов Skype для бизнеса до Skype для бизнеса Server 2019 и перемещайте пользователей на них, чтобы избежать сбоя голосовой связи.

### <a name="pbx-support-in-exchange-2019-and-skype-for-business-server-2019"></a>Поддержка УАТС в Exchange 2019 и Skype для бизнеса Server 2019

облачная голосовая почта не предоставляет функции голосового обмена сообщениями для обмена сообщениями частных ветвей (УАТС). Если вы используете Exchange Server Unified Messaging for PBXs и хотите выполнить обновление до Exchange Server 2019, необходимо использовать один из трех вариантов, перечисленных в записи блога "Новая дата" для прекращения поддержки пограничных контроллеров сеансов в [Exchange Online Unified Messaging](https://blogs.technet.microsoft.com/exchange/2018/04/24/new-date-for-discontinuation-of-support-for-session-border-controllers-in-exchange-online-unified-messaging/) в блоге [группы Exchange](https://blogs.technet.microsoft.com/exchange/).

### <a name="exchange-online-um-support-in-skype-for-business-server-2019"></a>Exchange Online единой системы обмена сообщениями в Skype для бизнеса Server 2019 г.

В Skype для бизнеса Server 2019 мы перейдите с Exchange Online единой системы обмена сообщениями на облачная голосовая почта. При перемещении пользователя на сервер Skype для бизнеса 2019 он автоматически начнет использовать облачная голосовая почта при настройке для размещенной голосовой почты. Если вы используете Exchange Online единой системы обмена сообщениями, вам не нужно ничего делать, кроме перемещения пользователя в Skype для бизнеса Server 2019, чтобы начать использовать облачная голосовая почта. Однако существуют некоторые изменения в функциональных возможностях, которые необходимо учитывать:

- Автосекретарь организации — это замена автосекретаря в Exchange Online единой системы обмена сообщениями.
- Параметры голосовой почты пользователя в Outlook в Интернете не применяются к облачная голосовая почта.

## <a name="on-premises-um-migration-scenarios"></a>Сценарии миграции локальной единой системы обмена сообщениями

Мы поддерживаем следующие сценарии, которые позволяют переносить пользователей как в Exchange 2019, так и в облачная голосовая почта.

- Exchange 2013/Exchange 2016 и Skype для бизнеса Server с 2015 по Exchange 2019 и Skype для бизнеса Server 2019
- Skype для бизнеса Server 2015 Skype для бизнеса Server 2019 с Exchange 2013 или Exchange 2016

В следующих сценариях требуется, чтобы конфигурации УАТС или SBC не существовали в рамках текущего развертывания, и предполагается, что на локальных серверах Exchange настроена единой системы обмена сообщениями. В каждом из этих решений также предполагается, что вы решили настроить гибридное развертывание между локальными серверами Skype для бизнеса и Microsoft 365 или Office 365. Дополнительные сведения о Skype для бизнеса гибридных развертываниях см. в разделе ["Планирование гибридного подключения"](plan-hybrid-connectivity.md).

### <a name="exchange-2013exchange-2016-and-skype-for-business-2015-to-exchange-2019-and-skype-for-business-2019"></a>Exchange 2013/Exchange 2016 и Skype для бизнеса 2015 до Exchange 2019 и Skype для бизнеса 2019

В этом сценарии необходимо перенести существующие серверы Exchange 2013, Exchange 2016 и Skype для бизнеса 2015 в Exchange 2019 и Skype для бизнеса 2019.

Как упоминалось ранее в этом разделе, Exchange 2019 больше не включает службу единой системы обмена сообщениями. Это означает, что для всех почтовых ящиков, которые вы хотите перенести в Exchange 2019, необходимо использовать облачная голосовая почта для замены функций, предоставляемых службой единой системы обмена сообщениями. Если вы настроили Skype для бизнеса Server 2019 и гибридное развертывание между ним и Microsoft 365 или Office 365, облачная голосовая почта заменит эти службы голосовой почты единой системы обмена сообщениями Exchange.

Порядок перемещения пользователей в Exchange 2019 и Skype для бизнеса Server 2019 крайне важен для обеспечения доступности функций голосовой почты для всех пользователей. Место обработки голосовой почты также определяется расположением почтовых ящиков и Skype для бизнеса Exchange и пользователей. Ознакомьтесь со следующей таблицей, чтобы узнать, какие сочетания Exchange и Skype для бизнеса Server поддерживаются и где обрабатывается голосовая почта.

| Почтовый ящик, расположенный в:            | Пользователь, расположенный Skype для бизнеса Server 2015 | Пользователь, расположенный в Skype для бизнеса Server 2019 г.  |
|--------------------------------|-----------------------------------------|------------------------------------------|
| Exchange 2013 или Exchange 2016    | единой системы обмена сообщениями Exchange                             | единой системы обмена сообщениями Exchange                              |
| Exchange 2019                  | Не поддерживается                           | Облачная голосовая почта                          |

Перед началом миграции на Skype для бизнеса Server 2019 и Exchange 2019 учитывайте следующее:

- Перед перемещением почтовых ящиков в Exchange 2019 необходимо настроить хотя бы один  сервер Skype для бизнеса 2019 и переместить пользователей на этот сервер. Если этого не сделать, эти почтовые ящики не смогут получать сообщения голосовой почты.
- Звонки, отправляемые в голосовую почту, будут облачная голосовая почта, где они будут записаны. После завершения звонка сообщение голосовой почты будет отправлено в почтовый ящик получателя на локальном сервере Exchange 2019. При определении того, достаточно ли подключения к Интернету для поддержки облачная голосовая почта.

Ниже приведены общие инструкции по завершению этой миграции.

1. Установите и настройте Skype для бизнеса Server 2019 на новом сервере.
2. Обновите конфигурацию гибридного развертывания, чтобы включить новый Skype для бизнеса 2019.
3. Установите и настройте Exchange Server 2019 на новом сервере.
4. Переместите пользователей с Skype для бизнеса 2015 на сервер Skype для бизнеса 2019.
5. Задайте политику размещенной голосовой почты для каждого пользователя, перемещенного в Skype для бизнеса Server 2019, чтобы использовать облачная голосовая почта.
6. Переместите почтовые ящики с сервера Exchange 2013 или Exchange 2016 на сервер Exchange 2019.
7. Выведите серверы Skype для бизнеса 2015 после того, как с них был удален последний пользователь.
8. Выведите из эксплуатации серверы Exchange 2013 или Exchange 2016 после перемещения последнего почтового ящика.


### <a name="skype-for-business-server-2015-to-skype-for-business-server-2019-with-exchange-2013exchange-2016"></a>Skype для бизнеса Server 2015 Skype для бизнеса Server 2019 с Exchange 2013 или Exchange 2016

В этом сценарии необходимо перенести существующий сервер Skype для бизнеса 2015 на Skype для бизнеса Server 2019, но оставить его в Exchange 2013 или Exchange 2016.

Когда Skype для бизнеса Server 2015 и Skype для бизнеса Server 2019 сосуществуют в одной организации, они легко работают с единой системой обмена сообщениями Exchange и облачная голосовая почта, чтобы обеспечить правильную доставку голосовой почты в почтовые ящики Exchange. Выбор единой системы обмена сообщениями Exchange или облачная голосовая почта обработка голосовой почты зависит от того, находится ли пользователь в Skype для бизнеса Server 2015 или Skype для бизнеса Server 2019:

- Если пользователь находится в Skype для бизнеса Server 2015 г., единой системой обмена сообщениями Exchange будет обрабатываться сообщение голосовой почты.
- Если пользователь находится в Skype для бизнеса Server 2019, облачная голосовая почта будет обрабатывать сообщение голосовой почты.

Независимо от того, облачная голосовая почта или единой системы обмена сообщениями Exchange обрабатывает сообщение голосовой почты, сообщение будет храниться в почтовом ящике Exchange пользователя.

Перед началом миграции Skype для бизнеса Server 2019 следует учитывать следующее:

- Звонки, отправляемые в голосовую почту, будут облачная голосовая почта, где они будут записаны. После завершения звонка сообщение голосовой почты будет отправлено в почтовый ящик получателя на локальном сервере Exchange. При определении того, достаточно ли подключения к Интернету для поддержки облачная голосовая почта.

Ниже приведены общие инструкции по завершению этой миграции.

1. Установите и настройте Skype для бизнеса Server 2019 на новом сервере.
2. Обновите конфигурацию гибридного развертывания, чтобы включить новый Skype для бизнеса 2019.
3. Переместите пользователей с Skype для бизнеса 2015 на сервер Skype для бизнеса 2019.
4. Задайте политику размещенной голосовой почты для каждого пользователя, перемещенного в Skype для бизнеса Server 2019, чтобы использовать облачная голосовая почта.
5. Выведите серверы Skype для бизнеса 2015 после того, как с них был удален последний пользователь.
