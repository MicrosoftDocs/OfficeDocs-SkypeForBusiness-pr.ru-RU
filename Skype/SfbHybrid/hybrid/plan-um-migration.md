# <a name="plan-for-skype-for-business-server-and-exchange-server-migration"></a>Планирование миграции Skype для бизнеса Server и Exchange Server

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

В этом разделе рассматриваются все, что необходимо учитывать при переносе существующих Skype для бизнеса Server или Exchange Server в Exchange Online. То, что можно перенести и когда, во многом зависит от того, что вы уже создали в организации.

## <a name="feature-changes-in-exchange-2019-and-skype-for-business-server-2019"></a>Изменения функций Exchange 2019 и Skype для бизнеса Server 2019

В Exchange 2019 и Skype для бизнеса Server 2019 года мы внося некоторые изменения в поддерживаемые функции.

### <a name="unified-messaging-support-in-exchange-2019"></a>Поддержка единой системы обмена сообщениями в Exchange 2019 г.

В 2019 г. система единой системы обмена сообщениями (единой системы обмена сообщениями) Exchange 2019 г. Это означает, что Exchange 2019 больше не предлагает следующие функции:

- Голосовая почта
- Автосекретарь

Если вы развернули роль um в Exchange 2013 г. или службу электронной почты в Exchange 2016 г. и хотите перейти на Exchange 2019 г., вам потребуется перенести голосовую почту в службу голосовая облачная почта (Майкрософт) в Microsoft 365 или Office 365. Если вы хотите перенести голосовую почту на облачная голосовая почта, взгляните на раздел [Exchange 2013/Exchange 2016 и Skype для бизнеса 2015](#exchange-2013exchange-2016-and-skype-for-business-2015-to-exchange-2019-and-skype-for-business-2019) г. на Exchange 2019 г. и Skype для бизнеса 2019 г. ниже.
> [!IMPORTANT]
> Если у пользователей на серверах Exchange или Exchange 2016 г. есть почтовые ящики с поддержкой системы обмена сообщениями, не перемещайте их в Exchange 2019 г. перед обновлением серверов Skype для бизнеса до Skype для бизнеса Server 2019 г. и перемещайте пользователей к ним, чтобы избежать перебоя с голосовой передачей сообщений.

### <a name="pbx-support-in-exchange-2019-and-skype-for-business-server-2019"></a>Поддержка PBX в Exchange 2019 и Skype для бизнеса Server 2019 г.

облачная голосовая почта не предоставляет функции голосовых сообщений частным филиалам Exchanges (PBXs). Если вы используете Exchange Server единой системы обмена сообщениями для PBXs и хотите обновиться до Exchange Server 2019 г., вам потребуется принять один из трех вариантов, перечисленных в записи блога Новая дата прекращения поддержки контроллеров пограничной сессии в [Exchange Online](https://blogs.technet.microsoft.com/exchange/2018/04/24/new-date-for-discontinuation-of-support-for-session-border-controllers-in-exchange-online-unified-messaging/) Единой системы обмена сообщениями в [блоге Exchange Team](https://blogs.technet.microsoft.com/exchange/)Blog .

### <a name="exchange-online-um-support-in-skype-for-business-server-2019"></a>Exchange Online Поддержка um в Skype для бизнеса Server 2019 г.

С Skype для бизнеса Server 2019 г. мы переходим от Exchange Online к облачная голосовая почта. Когда пользователь перемещается на сервер 2019 Skype для бизнеса 2019 года, он автоматически начнет использовать облачная голосовая почта при настройке для принимающей голосовой почты. Если в настоящее время используется Exchange Online, вам не нужно ничего делать, кроме перемещения пользователя в 2019 Skype для бизнеса Server, чтобы начать использовать облачная голосовая почта. Однако необходимо помнить о некоторых изменениях в функциональных возможностях:

- Организационные автосекретарь является заменой автобомбюсеру в Exchange Online um.
- Параметры голосовой почты пользователей в Outlook в Интернете не применяются к облачная голосовая почта.

## <a name="on-premises-um-migration-scenarios"></a>Сценарии миграции um на локальной основе

Мы поддерживаем следующие сценарии, которые позволят перенести пользователей Exchange 2019 г. и облачная голосовая почта.

- Exchange 2013/Exchange 2016 и Skype для бизнеса Server 2015 Exchange 2019 и Skype для бизнеса Server 2019
- Skype для бизнеса Server 2015 Skype для бизнеса Server 2019 Exchange 2013/Exchange 2016

В следующих сценариях необходимо, чтобы конфигурации PBX или SBC не существовали в рамках текущего развертывания и предполагалось, что на локальном Exchange настраивается um. Каждое из этих решений также предполагает, что вы решили настроить гибридное развертывание между локальной Skype для бизнеса серверами и Microsoft 365 или Office 365. Дополнительные сведения о Skype для бизнеса гибридных развертываниях см. в [руб. Plan hybrid connectivity.](plan-hybrid-connectivity.md)

### <a name="exchange-2013exchange-2016-and-skype-for-business-2015-to-exchange-2019-and-skype-for-business-2019"></a>Exchange 2013/Exchange 2016 и Skype для бизнеса 2015 Exchange 2019 и Skype для бизнеса 2019

В этом сценарии необходимо перенести существующие серверы Exchange 2013, Exchange 2016 и Skype для бизнеса 2015 на Exchange 2019 и Skype для бизнеса 2019.

Как упоминалось ранее в этом разделе, Exchange 2019 году больше не включает службу um. Это означает, что для всех почтовых ящиков, которые необходимо переместить в Exchange 2019 г., необходимо использовать облачная голосовая почта для замены функций, предоставляемых службой um. При установке Skype для бизнеса Server 2019 года и гибридного развертывания между ней и Microsoft 365 или Office 365, облачная голосовая почта заменяет Exchange службы голосовой почты Exchange системы электронной почты.

Порядок перемещения пользователей в 2019 Exchange 2019 и Skype для бизнеса Server 2019 имеет решающее значение для обеспечения того, чтобы функциональность голосовой почты оставалась доступной для всех пользователей. Где обрабатывается голосовая почта, определяется также местом Exchange и Skype для бизнеса почтовых ящиков и пользователей. Посмотрите на следующую таблицу, чтобы узнать, какие Exchange и Skype для бизнеса Server поддерживаются и где обрабатывается голосовая почта.

| Почтовый ящик, расположенный на:            | Пользователь, расположенный Skype для бизнеса Server 2015 г. | Пользователь, расположенный Skype для бизнеса Server 2019 г.  |
|--------------------------------|-----------------------------------------|------------------------------------------|
| Exchange 2013/Exchange 2016    | единой системы обмена сообщениями Exchange                             | единой системы обмена сообщениями Exchange                              |
| Exchange 2019                  | Не поддерживается                           | Облачная голосовая почта                          |

Прежде чем приступить к миграции Skype для бизнеса Server 2019 и Exchange 2019 г., помните следующее:

- Облачная голосовая почта не поддерживает организационную автосекретарь в ga. Если вы хотите, чтобы почтовые ящики перемещались в облачная голосовая почта, чтобы они по-прежнему были доступны через автослужбу, необходимо сохранить хотя бы один сервер Exchange 2013 или Exchange 2016 г., на который будет доступна роль или служба um.
- Необходимо настроить хотя бы один сервер Skype для бизнеса 2019 г. и переместить пользователей на этот сервер, прежде чем перенести их почтовые ящики на Exchange 2019 г.  В противном случае эти почтовые ящики не смогут получать сообщения голосовой почты.
- Вызовы, отправленные на голосовую почту, будут облачная голосовая почта, где они будут записаны. После окончания вызова сообщение голосовой почты будет отправлено в почтовый ящик получателя на локальном сервере Exchange 2019 года. Этот голосовой трафик необходимо учитывать при определении достаточности подключения к Интернету для поддержки облачная голосовая почта.

Вот шаги высокого уровня для завершения этой миграции.

1. Установка и настройка Skype для бизнеса Server 2019 г. на новом сервере.
2. Обновите конфигурацию гибридного развертывания, чтобы включить новый Skype для бизнеса 2019.
3. Установка и настройка Exchange Server 2019 г. на новом сервере.
4. Перемещение пользователей с сервера Skype для бизнеса 2015 г. на Skype для бизнеса 2019 г.
5. Установите политику голосовой почты для каждого пользователя, перемещенного Skype для бизнеса Server 2019 г., чтобы использовать облачная голосовая почта.
6. Перемещение почтовых ящиков с сервера Exchange 2013 или Exchange 2016 г. на Exchange 2019 г.
7. Вывод из эксплуатации Skype для бизнеса серверов 2015 г. после отключения последнего пользователя.
8. Вывод из эксплуатации Exchange серверов 2013 или Exchange 2016 г. после отключения последнего почтового ящика.

    > [!IMPORTANT]
    > Если вы полагаетесь на автоспутника, держите в Exchange 2013 или Exchange 2016 год по крайней мере один Exchange 2016 г.

### <a name="skype-for-business-server-2015-to-skype-for-business-server-2019-with-exchange-2013exchange-2016"></a>Skype для бизнеса Server 2015 Skype для бизнеса Server 2019 Exchange 2013/Exchange 2016

В этом сценарии необходимо перенести существующий сервер Skype для бизнеса 2015 г. на Skype для бизнеса Server 2019 г., но оставаться на Exchange 2013 или Exchange 2016 г.

Если Skype для бизнеса Server 2015 и Skype для бизнеса Server 2019 г. будут сосуществовать в одной организации, они будут без проблем работать с Exchange и облачная голосовая почта, чтобы обеспечить правильную доставку голосовой почты в Exchange почтовые ящики. Независимо от Exchange или облачная голосовая почта голосовой почты зависит от того, находится ли пользователь Skype для бизнеса Server 2015 или Skype для бизнеса Server 2019 г.:

- Если пользователь находится на Skype для бизнеса Server 2015 г., Exchange будет обрабатывать сообщение голосовой почты.
- Если пользователь находится на Skype для бизнеса Server 2019 г., облачная голосовая почта будет обрабатывать сообщение голосовой почты.

Независимо от Exchange или облачная голосовая почта обработки сообщения голосовой почты, сообщение будет храниться в почтовом Exchange пользователя.

Если вы хотите, чтобы почтовые ящики перемещались в облачная голосовая почта, чтобы они по-прежнему были доступны через автослужбу, необходимо сохранить хотя бы один сервер Exchange 2013 или Exchange 2016 г., на который будет доступна роль или служба um.

Этот голосовой трафик необходимо учитывать при определении достаточности подключения к Интернету для поддержки облачная голосовая почта.

Вот шаги высокого уровня для завершения этой миграции.
