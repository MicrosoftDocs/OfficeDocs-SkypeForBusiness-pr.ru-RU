# <a name="plan-for-skype-for-business-server-and-exchange-server-migration"></a>Планирование миграции Skype для бизнеса Server и Exchange Server

В этом разделе описывается, что следует учитывать при переносе существующего развертывания Skype для бизнеса Server Exchange Server в Exchange Online. То, что можно перенести и когда, в значительной степени зависит от того, что вы уже настроили в организации.

## <a name="feature-changes-in-exchange-2019-and-skype-for-business-server-2019"></a>Изменения функций в Exchange 2019 и Skype для бизнеса Server 2019

В Exchange 2019 и Skype для бизнеса Server 2019 мы внося некоторые изменения в поддерживаемые функции.

### <a name="unified-messaging-support-in-exchange-2019"></a>Поддержка единой системы обмена сообщениями в Exchange 2019

Единая система обмена сообщениями (UM) была неподготовлена в Exchange 2019. Это означает, что в Exchange 2019 больше нет следующих функций:

- Голосовая почта
- Автосекретарь

Если вы развернули роль этой системы в Exchange 2013 или службу этой системы в Exchange 2016 и хотите перейти на Exchange 2019, вам потребуется перенести голосовую почту в службу голосовой почты Microsoft Cloud в Microsoft 365 или Office 365. Если вы хотите перенести голосовую почту в облачную голосовую почту, см. раздел Exchange [2013/Exchange 2016 и Skype для бизнеса 2015 на Exchange 2019](#exchange-2013exchange-2016-and-skype-for-business-2015-to-exchange-2019-and-skype-for-business-2019) и Skype для бизнеса 2019 ниже.
> [!IMPORTANT]
> Если у пользователей на серверах Exchange 2013 или Exchange 2016 есть почтовые ящики с включенной поддержкой этой системы, не перемещайте их в Exchange 2019 перед обновлением серверов Skype для бизнеса до Skype для бизнеса Server 2019 и переместите пользователей на них, чтобы избежать сбоя голосовых сообщений.

### <a name="pbx-support-in-exchange-2019-and-skype-for-business-server-2019"></a>Поддержка УАПС в Exchange 2019 и Skype для бизнеса Server 2019

Облачная голосовая почта не предоставляет функции голосовой почты для УАПС. Если вы используете единую систему обмена сообщениями Exchange Server для УАПС и хотите обновить ее до Exchange Server 2019, необходимо использовать один из трех вариантов, перечисленных в записи блога "Новая дата прекращения поддержки пограничных контроллеров сеансов" в единой системы обмена сообщениями [Exchange Online](https://blogs.technet.microsoft.com/exchange/2018/04/24/new-date-for-discontinuation-of-support-for-session-border-controllers-in-exchange-online-unified-messaging/) в блоге группы [Exchange.](https://blogs.technet.microsoft.com/exchange/)

### <a name="exchange-online-um-support-in-skype-for-business-server-2019"></a>Поддержка exchange Online в Skype для бизнеса Server 2019

В Skype для бизнеса Server 2019 мы переходим с exchange Online um на облачную голосовую почту. Когда пользователь перемещается на сервер Skype для бизнеса 2019, он автоматически начинает использовать облачную голосовую почту при настройке для этой службы. If you're currently using Exchange Online UM, you don't need to do anything other than move a user to Skype for Business Server 2019 to start using Cloud Voicemail. Однако существуют некоторые изменения в функциональных возможностях, которые необходимо знать:

- Организационные автосекретарь заменяют автоотправления в exchange Online UM.
- Параметры голосовой почты пользователя в Outlook в Интернете не применяются к облачной голосовой почте.

## <a name="on-premises-um-migration-scenarios"></a>Сценарии миграции локальной um

Поддерживаются следующие сценарии, позволяющие перенести пользователей в Exchange 2019 и облачную голосовую почту.

- Exchange 2013/Exchange 2016 и Skype для бизнеса Server 2015 до Exchange 2019 и Skype для бизнеса Server 2019
- От Skype для бизнеса Server 2015 к Skype для бизнеса Server 2019 с Exchange 2013 или Exchange 2016

В следующих сценариях требуется, чтобы в рамках текущего развертывания не существовало конфигураций PBX или SBC, и предполагается, что на локальном сервере Exchange настроена um. В каждом из этих решений также предполагается, что вы решили настроить гибридное развертывание между вашими серверами Skype для бизнеса и Microsoft 365 или Office 365. Дополнительные сведения о гибридных развертываниях Skype для бизнеса см. в сведениях [о планировании гибридного подключения.](plan-hybrid-connectivity.md)

### <a name="exchange-2013exchange-2016-and-skype-for-business-2015-to-exchange-2019-and-skype-for-business-2019"></a>Exchange 2013/Exchange 2016 и Skype для бизнеса 2015 до Exchange 2019 и Skype для бизнеса 2019

В этом сценарии необходимо перенести существующие серверы Exchange 2013, Exchange 2016 и Skype для бизнеса 2015 в Exchange 2019 и Skype для бизнеса 2019.

Как упоминалось ранее в этом разделе, Exchange 2019 больше не включает службу этой службы. Это означает, что для всех почтовых ящиков, которые вы хотите переместить в Exchange 2019, необходимо использовать облачную голосовую почту для замены функций, предоставленных службой этой службы. При установке Skype для бизнеса Server 2019 и гибридном развертывании между ней и Microsoft 365 или Office 365 облачная голосовая почта заменяет эти службы голосовой почты exchange.

Порядок перемещения пользователей в Exchange 2019 и Skype для бизнеса Server 2019 критически важен для обеспечения того, чтобы функции голосовой почты оставались доступными для всех пользователей. Место обработки голосовой почты также определяется местом, где расположены почтовые ящики и пользователи Exchange и Skype для бизнеса. Посмотрите на следующую таблицу, чтобы узнать, какие сочетания Exchange и Skype для бизнеса Server поддерживаются и где обрабатывается голосовая почта.

| Почтовый ящик, расположенный в:            | Пользователь, расположенный в Skype для бизнеса Server 2015 | Пользователь, расположенный в Skype для бизнеса Server 2019  |
|--------------------------------|-----------------------------------------|------------------------------------------|
| Exchange 2013 и Exchange 2016    | единой системы обмена сообщениями Exchange                             | единой системы обмена сообщениями Exchange                              |
| Exchange 2019                  | Не поддерживается                           | Облачная голосовая почта                          |

Перед началом миграции в Skype для бизнеса Server 2019 и Exchange 2019 необходимо иметь в виду следующее:

- Облачная голосовая почта не поддерживает организационные автосекретарь в ga. Если вы хотите, чтобы почтовые ящики, перенесенные в облачную голосовую почту, по-прежнему были доступны через автоответчик, необходимо оставить доступным по крайней мере один сервер Exchange 2013 или Exchange 2016 с ролью или службой этой службы.
- Перед перемещением почтовых ящиков в Exchange 2019 необходимо настроить хотя бы один сервер Skype для бизнеса 2019 и переместить пользователей на этот сервер.  В противном случае эти почтовые ящики не смогут получать сообщения голосовой почты.
- Вызовы, отправленные на голосовую почту, будут перенаправлены в облачную голосовую почту, где они будут записаны. После окончания вызова сообщение голосовой почты будет отправлено в почтовый ящик получателя на локальном сервере Exchange 2019. Этот голосовой трафик необходимо учитывать при определении того, достаточно ли подключения к Интернету для поддержки облачной голосовой почты.

Вот высокоуровневые действия для завершения этой миграции.

1. Установите и настройте Skype для бизнеса Server 2019 на новом сервере.
2. Обновите конфигурацию гибридного развертывания, включив новый сервер Skype для бизнеса 2019.
3. Установите и настройте Exchange Server 2019 на новом сервере.
4. Переместите пользователей с сервера Skype для бизнеса 2015 на сервер Skype для бизнеса 2019.
5. Настройка политики голосовой почты для каждого пользователя, перемещенного в Skype для бизнеса Server 2019, для использования облачной голосовой почты.
6. Переместите почтовые ящики с сервера Exchange 2013 или Exchange 2016 на сервер Exchange 2019.
7. Списание серверов Skype для бизнеса 2015 после того, как с них был перемещен последний пользователь.
8. Списание серверов Exchange 2013 или Exchange 2016 после сбоя последнего почтового ящика.

    > [!IMPORTANT]
    > Если вы полагаетесь на автозамену, оохраняем по крайней мере один exchange 2013 или Exchange 2016, работающий и доступный.

### <a name="skype-for-business-server-2015-to-skype-for-business-server-2019-with-exchange-2013exchange-2016"></a>От Skype для бизнеса Server 2015 к Skype для бизнеса Server 2019 с Exchange 2013 или Exchange 2016

В этом сценарии необходимо перенести существующий сервер Skype для бизнеса 2015 в Skype для бизнеса Server 2019, но остаться в Exchange 2013 или Exchange 2016.

Когда Skype для бизнеса Server 2015 и Skype для бизнеса Server 2019 совместно работают в одной организации, они без проблем работают с exchange UM и облачной голосовой почтой, чтобы обеспечить правильную доставку голосовой почты в почтовые ящики Exchange. Обработка голосовой почты в exchange UM или облачной голосовой почте зависит от того, находится ли пользователь в Skype для бизнеса Server 2015 или Skype для бизнеса Server 2019:

- Если пользователь находится в Skype для бизнеса Server 2015, то почта голосовой почты будет обрабатываться в exchange UM.
- Если пользователь находится в Skype для бизнеса Server 2019, облачная голосовая почта будет обрабатывать сообщение голосовой почты.

Независимо от того, обрабатывает ли почта голосовая почта exchange или облачная голосовая почта, сообщение будет храниться в почтовом ящике Exchange пользователя.
