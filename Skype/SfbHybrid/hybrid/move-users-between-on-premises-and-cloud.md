# <a name="move-users-between-on-premises-and-cloud"></a>Перемещение пользователей между локальной средой и облаком

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

В локальном развертывании Skype для бизнеса Server, которое включено для гибридного развертывания, можно перемещать пользователей между локальной средой и Teams. Размещается ли пользователь локально или в облаке, это местоположение называется пользовательским домом Skype для бизнеса.

- Пользователи, которые размещены локально, взаимодействуют с локальными Skype для бизнеса серверами.
- Пользователи, которые размещены в Интернете, могут взаимодействовать с Teams службой.

*Teams пользователи по сути имеют Skype для бизнеса, независимо от того, Skype для бизнеса или нет.* Если у вас есть локальные Skype для бизнеса, которые также используют Teams (параллельно), эти пользователи размещены локально. Teams пользователи с Skype для бизнеса локально не могут взаимодействовать с Skype для бизнеса пользователями из своего Teams-клиента и не могут взаимодействовать с Teams пользователями в федеративной организации. Такая функция полностью доступна только после того, как пользователь будет перемещен из локальной Skype для бизнеса в интернет и в teamsOnly. Рекомендуется переместить пользователей в режим TeamsOnly, что гарантирует, что маршрутизация всех входящих чатов и звонков будет выполняться в Teams клиенте. Дополнительные сведения см. [в](/microsoftteams/coexistence-chat-calls-presence) Teams руководствах по Skype для бизнеса миграции и взаимодействию для организаций, использующих Teams совместно с [Skype для бизнеса.](/microsoftteams/migration-interop-guidance-for-teams-with-skype)

## <a name="prerequisites"></a>Предварительные требования

Необходимые условия для перемещения пользователя в режим TeamsOnly:

- Организация должна правильно Azure AD Подключение и синхронизировать все соответствующие атрибуты для [пользователя](configure-azure-ad-connect.md), как описано в Azure AD Подключение.
- Skype для бизнеса гибридная среда должна быть настроена, как описано в [Skype для бизнеса гибридной среде](configure-federation-with-skype-for-business-online.md).
- Пользователю должна быть назначена лицензия на Teams и Skype для бизнеса Online (план 2). Даже после прекращения использования Skype для бизнеса Online лицензия Skype для бизнеса Online по-прежнему требуется.  Кроме того:
    - Если пользователь включен для конференц-связи с телефонным подключением в локальной среде, перед перемещением пользователя в сеть также должна быть назначена Аудиоконференции лицензия Teams. После миграции в облако пользователь будет готов к проведению звуковой конференц-связи в облаке. 
    - Если пользователь включен для Корпоративная голосовая связь в локальной среде, пользователю должна быть назначена телефонная система лицензия, назначенная Teams перед перемещением пользователя в интернет. После миграции в облако пользователь будет подготовлен для телефонная система в облаке. 


## <a name="moving-users"></a>Перемещение пользователей

При перемещении пользователя из локальной среды в облако:

- Teams пользователей становятся доступными для взаимодействия с Skype для бизнеса пользователями, и если они являются TeamsOnly, они также могут федерации с другими организациями.

- Контакты из локальной среды перемещаются в Teams.

- Существующие организованные ими собрания, запланированные в будущем, преобразуются в Teams собрания. Миграция собраний происходит асинхронно и запускается примерно через 90 минут после переноса пользователя.  Чтобы определить состояние миграции собрания, можно использовать командлет [Get-csMeetingMigrationStatus](../../SfbOnline/audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md#managing-mms). Содержимое, отправленное до собрания, не перемещается.

Чтобы переместить пользователей в Teams, используйте командлет Move-CsUser или Skype для бизнеса администратора панель управления, которые являются локальными средствами. Эти средства поддерживают следующие пути перемещения:

- [Из Skype для бизнеса Server (локально) непосредственно в Teams только.](move-users-from-on-premises-to-teams.md)  Поведение при перемещении непосредственно из локальной среды в Teams теперь выполняется автоматически, независимо от используемой версии Skype для бизнеса Server или Lync Server. Для этого больше не нужно `-MoveToTeams` указывать переключатель.  
- [Из Интернета (независимо Teams или нет) в локальную среду](move-users-from-the-cloud-to-on-premises.md).

> [!NOTE] 
> Больше не требуется указывать параметр -MoveToTeams в Move-CsUser для перемещения пользователей непосредственно из локальной среды в TeamsOnly. Ранее, если этот параметр не был указан, пользователи переходят с локального Skype для бизнеса Server на Skype для бизнеса Online, а их режим не изменился. Теперь при перемещении пользователя из локальной среды в облако с помощью Move-CsUser пользователям автоматически назначается режим TeamsOnly, а их собрания из локальной среды автоматически преобразуются в собрания Teams, `-MoveToTeams` как если бы был указан переключатель, независимо от того, был ли указан переключатель. 
> 

## <a name="required-administrative-credentials"></a>Необходимые учетные данные администратора

Для перемещения пользователей между локальной средой и облаком необходимо использовать учетную запись с достаточными привилегиями как в локальной среде Skype для бизнеса Server, так и в Teams организации. Вы можете использовать одну учетную запись со всеми необходимыми привилегиями или две учетные записи. Если вы используете две учетные записи, вы получите доступ к локальным средствам с помощью локальных учетных данных, а затем в этих средствах укажите дополнительные учетные данные для Teams учетной записи администратора.  

- В локальной среде пользователь, выполняющий перемещение, должен иметь роли CSServerAdministrator, CsUserAdministrator и RTCUniversalUserAdmins в Skype для бизнеса Server.
- В Teams пользователь, выполняю следующий переход, должен быть членом по крайней мере одной из следующих ролей:
  - Роль глобального администратора
  - Teams роли администратора
  - Skype для бизнеса роли администратора.  

    > [!Important]
    > - Если вы используете Skype для бизнеса администратора панель управления, вам будет предложено ввести учетные данные для учетной записи Microsoft 365 с соответствующими ролями, как указано выше. Необходимо указать учетную запись, которая заканчивается на .onmicrosoft.com. Если это невозможно, используйте Move-CsUser командлета.
    >- Если вы используете Move-CsUser в PowerShell, вы можете использовать учетную запись, которая заканчивается на .onmicrosoft.com, или любую локальную учетную запись, синхронизированную с Azure AD при условии, что в командлете также указан параметр HostedMigrationOverrideUrl. Значение URL-адреса переопределения размещенной миграции является вариантом следующего URL-адреса: https://adminXX.online.lync.com/HostedMigration/hostedmigrationService.svc<br>В приведенном выше URL-адресе замените XX двумя или тремя символами, которые определяются следующим образом:
    >   - В сеансе Teams PowerShell выполните следующий командлет:<br>`Get-CsTenant | ft ServiceInstance`
    >   - Результирующее значение будет иметь следующий формат:<br>`MicrosoftCommunicationsOnline/YYYY-XX-ZZ`
    >   - Двух- или трех символьным кодом является XX, содержащийся в разделе YYYY-XX-ZZ. Если это двухзначный код, это будет цифра, за которой следует число (например, 4A). Если это трехзначный код, он будет содержать две буквы, за которыми следует цифра (например, JP1). Например, NOAM-4A-S7.


## <a name="voice-configuration-requirements"></a>Требования к конфигурации голосовой связи

Если пользователи настроены для корпоративной голосовой связи в локальной среде, необходимо координировать обновление конфигурации голосовой связи при их перемещении в режим "в сети". Кроме того, их можно перенести без возможности телефонии. Доступные варианты зависят от того, будет ли пользователь использовать клиент Teams или Skype для бизнеса после подключения к сети:

- Вы можете обновить поставщик телефонии пользователя, чтобы использовать [план звонков Майкрософт](/microsoftteams/calling-plans-for-office-365). Это параметр, который определяет, будут ли пользователи использовать Teams или Skype для бизнеса клиентах.
- Вы можете продолжать использовать локальный поставщик ТСОП:
  - Пользователи голосовой связи, которые будут Teams, должны быть настроены для [прямой маршрутизации](/microsoftteams/direct-routing-plan). Прямая маршрутизация доступна только после перемещения пользователя из локальной среды в сеть.
  - Пользователи голосовой связи, которые будут использовать Skype для бизнеса клиента после их перемещения в режим "в сети", должны быть настроены для гибридный Skype для бизнеса голосовой связи.

Дополнительные сведения о параметрах телефонии в гибридных средах, включая матрицу поддержки, см. в разделе "Учетные записи пользователей в гибридной среде с подключением по [ТСОП"](/microsoftteams/direct-routing-user-accounts-in-a-hybrid-environment).

## <a name="other-considerations"></a>Другие особенности

Политики (например, для управления обменом сообщениями, собраниями и звонками) в локальных и сетевых средах независимы друг от друга. Вы можете настроить любые политики в среде и назначить их пользователю перед перемещением этого пользователя из локальной среды в облако, чтобы у них была правильная конфигурация сразу после переноса в режим "в сети".
