# <a name="move-users-between-on-premises-and-cloud"></a>Перемещение пользователей между локальной средой и облаком

[!INCLUDE [sfbo-retirement](../../Hub/includes/sfbo-retirement.md)]

В локальном развертывании Skype для бизнеса Server пользователи Skype для бизнеса могут также использовать Teams, но не все функции Teams доступны таким пользователям, если они настроены для использования локального Skype для бизнеса Server развертывания. Считается, что эти пользователи размещены в локальной среде, и некоторые функции Teams недоступны, когда эти пользователи размещены локально, например:
- Федеративные звонки и чаты через клиент Teams пользователя с пользователями в других организациях недоступны
- Взаимодействие через клиент Teams пользователя с другими пользователями в организации, которые используют Skype для бизнеса клиента.
- Функции звонков по ТСОП (если пользователю назначена лицензия телефонной системы).

Чтобы получить полную функциональность Teams, этих пользователей необходимо переместить из локальной Skype для бизнеса в облако, после которого пользователь становится TeamsOnly. При перемещении пользователя из локальной среды в облако для режима сосуществования пользователя будет установлено значение TeamsOnly. После перемещения пользователей в облако и TeamsOnly все входящие чаты и вызовы попадают в клиент Teams. Дополнительные сведения см. в статье о сосуществовании [Teams](/microsoftteams/coexistence-chat-calls-presence) с Skype для бизнеса и миграции, а также рекомендации по взаимодействию для организаций, использующих [Teams вместе с](/microsoftteams/migration-interop-guidance-for-teams-with-skype) Skype для бизнеса.

Для перемещения пользователей из локальной среды Skype для бизнеса Server в облако требуется конфигурация Skype для бизнеса [гибридной среды](/skypeforbusiness/hybrid/plan-hybrid-connectivity).  После включения гибридного развертывания можно переместить пользователей из локальной среды в облако, чтобы сделать их TeamsOnly, как описано ниже. При необходимости можно также переместить пользователей TeamsOnly обратно в локальное развертывание Skype для Bsuiness. 



## <a name="prerequisites"></a>Предварительные требования

Необходимые условия для перемещения пользователя в режим TeamsOnly:

- Организация должна правильно Azure AD Connect и синхронизировать все соответствующие атрибуты для пользователя, как описано в Azure AD [Connect](configure-azure-ad-connect.md).
- Skype для бизнеса гибридная среда должна быть настроена, как описано в [Skype для бизнеса гибридной среде](configure-federation-with-skype-for-business-online.md).
- Пользователю должна быть назначена лицензия на Teams и Skype для бизнеса Online (план 2). Даже после прекращения использования Skype для бизнеса Online лицензия Skype для бизнеса Online по-прежнему требуется.  Кроме того:
    - Если пользователь включен для конференц-связи с телефонным подключением в локальной среде, то перед перемещением пользователя в сеть также должна быть назначена лицензия на аудиоконференции в Teams. После миграции в облако пользователь будет готов к проведению звуковой конференц-связи в облаке. 
    - Если пользователь включен для Корпоративная голосовая связь в локальной среде, пользователю должна быть назначена лицензия телефонной системы в Teams, прежде чем вы переместите пользователя в Интернет. После миграции в облако пользователь будет подготовлен для телефонной системы в облаке. 
  
С 31 июля 2022 г. для перемещения пользователей между локальным развертыванием и облаком необходимо использовать следующую минимальную версию Skype для бизнеса Server или Lync Server:

## <a name="moving-users"></a>Перемещение пользователей

При перемещении пользователя из локальной среды в облако:

- Пользователи Teams становятся доступными для взаимодействия с Skype для бизнеса пользователями, и если они являются TeamsOnly, они также могут взаимодействовать с другими организациями.
- Контакты из локальной среды перемещаются в Teams.
- Существующие организованные ими собрания, запланированные в будущем, преобразуются в собрания Teams. Миграция собраний происходит асинхронно и запускается примерно через 90 минут после переноса пользователя.  Чтобы определить состояние миграции собрания, можно использовать командлет [Get-csMeetingMigrationStatus](../../SfbOnline/audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md#managing-mms). Содержимое, отправленное до собрания, не перемещается.
- Пользователи, которым назначена телефонная система, могут получить доступ к функциям ТСОП после правильной настройки.
 
Чтобы переместить пользователей в Teams, используйте командлет Move-CsUser или Skype для бизнеса Администратор панель управления, которые являются локальными средствами. Эти средства поддерживают следующие пути перемещения:

- [От Skype для бизнеса Server (локально) до Только Teams](move-users-from-on-premises-to-teams.md).
- [Из Интернета (независимо от того, только Teams или нет) в локальную среду](move-users-from-the-cloud-to-on-premises.md).


> [!NOTE] 
>  Перемещение непосредственно из локальной среды в Teams теперь выполняется автоматически, независимо от используемой версии Skype для бизнеса Server или Lync Server. Больше не требуется указывать параметр -MoveToTeams в Move-CsUser для перемещения пользователей непосредственно из локальной среды в режим "Только Teams". Ранее, если этот параметр не был указан, пользователи переходят с локального Skype для бизнеса Server на Skype для бизнеса Online, а их режим не изменился. Теперь при перемещении пользователя из локальной среды в облако с помощью Move-CsUser пользователям автоматически назначается режим TeamsOnly, а их собрания из локальной среды автоматически преобразуются в собрания Teams, `-MoveToTeams` как если бы параметр был указан, независимо от того, был ли указан переключатель. 


## <a name="required-administrative-credentials"></a>Необходимые учетные данные администратора

Чтобы переместить пользователей между локальной средой и облаком, необходимо использовать учетную запись с достаточными привилегиями как в локальной среде Skype для бизнеса Server, так и в организации Teams. Вы можете использовать одну учетную запись со всеми необходимыми привилегиями или две учетные записи. Если вы используете две учетные записи, вы получите доступ к локальным средствам с помощью локальных учетных данных, а затем в этих средствах укажите дополнительные учетные данные для учетной записи администратора Teams.  

- В локальной среде пользователь, выполняющий перемещение, должен иметь роли CSServerAdministrator, CsUserAdministrator и RTCUniversalUserAdmins в Skype для бизнеса Server.
- В Teams пользователь, выполняю следующий переход, должен быть членом по крайней мере одной из следующих ролей:
  - Роль глобального администратора
  - Роль администратора Teams
  - Skype для бизнеса роли администратора.  

    > [!Important]
    > - Если вы используете Skype для бизнеса Администратор панель управления, вам будет предложено ввести учетные данные для учетной записи Microsoft 365 с соответствующими ролями, как указано выше. Необходимо указать учетную запись, которая заканчивается на .onmicrosoft.com. Если это невозможно, используйте Move-CsUser командлета.
    >- Если вы используете Move-CsUser в PowerShell, вы можете использовать учетную запись, которая заканчивается на .onmicrosoft.com, или любую локальную учетную запись, синхронизированную с Azure AD при условии, что в командлете также указан параметр HostedMigrationOverrideUrl. Значение URL-адреса переопределения размещенной миграции является вариантом следующего URL-адреса: https://adminXX.online.lync.com/HostedMigration/hostedmigrationService.svc<br>В приведенном выше URL-адресе замените XX двумя или тремя символами, которые определяются следующим образом:
    >   - В сеансе Teams PowerShell выполните следующий командлет:<br>`Get-CsTenant | ft ServiceInstance`
    >   - Результирующее значение будет иметь следующий формат:<br>`MicrosoftCommunicationsOnline/YYYY-XX-ZZ`
    >   - Двух- или трех символьным кодом является XX, содержащийся в разделе YYYY-XX-ZZ. Если это двухзначный код, это будет цифра, за которой следует число (например, 4A). Если это трехзначный код, он будет содержать две буквы, за которыми следует цифра (например, JP1). Например, NOAM-4A-S7.


## <a name="voice-configuration-requirements"></a>Требования к конфигурации голосовой связи

Если пользователи настроены для корпоративной голосовой связи в локальной среде, необходимо координировать обновление конфигурации голосовой связи при их перемещении в режим "в сети". Кроме того, их можно перенести без возможности телефонии. 
- Вы можете обновить поставщик телефонии пользователя, чтобы использовать [план звонков Майкрософт](/microsoftteams/calling-plans-for-office-365). Это параметр, который определяет, будут ли пользователи использовать Teams или Skype для бизнеса клиентами.
- Вы можете продолжать использовать локальный поставщик ТСОП:
  - Пользователи голосовой связи, которые будут использовать Teams, должны быть настроены для [прямой маршрутизации](/microsoftteams/direct-routing-plan). Прямая маршрутизация доступна только после перемещения пользователя из локальной среды в сеть.

Дополнительные сведения о параметрах телефонии в гибридных средах, включая матрицу поддержки, см. в разделе "Учетные записи пользователей в гибридной среде с подключением по [ТСОП"](/microsoftteams/direct-routing-user-accounts-in-a-hybrid-environment).

## <a name="other-considerations"></a>Другие особенности

Политики (например, для управления обменом сообщениями, собраниями и звонками) в локальных и сетевых средах независимы друг от друга. Вы можете настроить любые политики в среде и назначить их пользователю перед перемещением этого пользователя из локальной среды в облако, чтобы у них была правильная конфигурация сразу после переноса в режим "в сети".
