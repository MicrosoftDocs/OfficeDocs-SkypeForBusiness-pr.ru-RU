# <a name="move-users-between-on-premises-and-cloud"></a>Перемещение пользователей между локальной и облачной средами

В локальном развертывании Skype для бизнеса Server, для которого включена гибридная среда, вы можете перемещать пользователей между локальной средой и облаком (в Microsoft Teams или в Skype для бизнеса Online). Если пользователь находится в локальной среде или в облаке, он называется доменом пользователя Skype для бизнеса.

- Пользователи, размещенные в локальной среде, взаимодействуют с локальными серверами Skype для бизнеса.
- Пользователи, размещенные в Интернете, могут взаимодействовать со службой Skype для бизнеса Online.

*Пользователи Teams имеют домашнюю страницу Skype для бизнеса, независимо от того, используют ли они Skype для бизнеса или нет.* Если у вас есть локальные пользователи Skype для бизнеса, которые также используют Teams (параллельно), эти пользователи размещены локально. Пользователи Teams, работающие в локальной среде Skype для бизнеса, не могут взаимодействовать с пользователями Skype для бизнеса из своих клиентов Teams и не могут взаимодействовать от Teams с пользователями в федеративной организации. Такие функции доступны только после того, как пользователь перемещается из Skype для бизнеса локально в Интернет. Когда пользователь перемещается в интерактивный режим, вы можете разрешить им использовать Skype для бизнеса Online (и, при необходимости, Teams) или делать их только для Teams. Если в вашей организации уже используются Teams, настоятельно рекомендуется переместить их в режим "только Teams", который обеспечит маршрутизацию всех входящих чатов и звонков в клиенте Teams. Более подробную информацию можно узнать в статье поддержка совместной работы [в Microsoft Teams со Skype для бизнеса](/microsoftteams/coexistence-chat-calls-presence) и [миграцией и совместимостью для организаций, использующих Teams вместе со Skype для бизнеса](/microsoftteams/migration-interop-guidance-for-teams-with-skype).

## <a name="prerequisites"></a>Необходимые условия

Необходимые условия для перемещения пользователя в облако (только в режиме Skype для бизнеса или только в Teams):

- Для Организации необходимо правильно настроить подключение Azure AD и синхронизировать все нужные атрибуты для пользователя, как описано в разделе [Configure Azure AD Connect](configure-azure-ad-connect.md).
- Необходимо настроить гибридную среду Skype для бизнеса, как описано в разделе [Настройка гибридной среды Skype для бизнеса](configure-federation-with-skype-for-business-online.md).
- Пользователю должна быть назначена лицензия на Skype для бизнеса Online (план 2), и если они будут использовать Teams, они также должны иметь лицензию Teams.  Дополнительно:
    - Если пользователю разрешено использовать конференц-связь с телефонным подключением в локальной среде, то по умолчанию у пользователя также должна быть назначена лицензия на аудио-конференцию в Office 365, прежде чем вы перемещаете пользователя в сети. После переноса в облако пользователь будет подготовлен для передачи аудио-и видеоконференций в облако. Если по какой-либо причине вы хотите переместить пользователя в облако, но не использовать функцию конференц-связи, эту проверку можно переопределить, указав `BypassAudioConferencingCheck` параметр в. `Move-CsUser`
    - Если пользователю разрешено использовать корпоративную голосовую связь в локальной среде, по умолчанию у пользователя должна быть назначена лицензия на телефонную систему в Office 365, прежде чем вы перемещаете пользователя в Интернете. После переноса в облако пользователь будет подготовлен для телефонной системы в облаке. Если по той или иной причине вы хотите переместить пользователя в облако, но не использовать функции телефонной системы, эту проверку можно переопределить, указав `BypassEnterpriseVoiceCheck`параметр в. `Move-CsUser`


## <a name="moving-users"></a>Перемещение пользователей

Когда пользователь перемещается из локальной организации в облако:

- Пользователь начинает использовать службы Skype для бизнеса Online в облаке для любой функции Skype для бизнеса.
- Пользователи Teams станут доступны для взаимодействия с пользователями Skype для бизнеса и могут также использовать федерацию с другими организациями.
- Контакты из локальной компании перемещаются в облако (Skype для бизнеса или Microsoft Teams).
- Существующие собрания, запланированные в будущем, переносятся в Online: Если пользователи перемещаются непосредственно в Теамсонли (см. ниже), собрания преобразуются в собрания Teams, иначе собрания останутся в Skype для бизнеса, но они будут перенесены размещено в Интернете, а не в локальной среде.  Миграция собраний происходит асинхронно и начинается приблизительно через 90 минут после перемещения пользователя.  Для определения состояния миграции собраний можно использовать [Get – ксмитингмигратионстатус](../../SfbOnline/audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md#managing-mms). Обратите внимание, что все содержимое, которое было отправлено заранее, не перемещается.

Для перемещения пользователей между локальной средой и облаком (как в Teams, так и в Skype для бизнеса Online) используйте командлет Move-CsUser или панель управления администрирования Skype для бизнеса, которые являются локальными средствами. Эти средства поддерживают три разных пути перемещения:

- [Из Skype для бизнеса Server (локально) в Skype для бизнеса Online](move-users-from-on-premises-to-skype-for-business-online.md).
- [Из Skype для бизнеса Server (локально) только непосредственно в Teams](move-users-from-on-premises-to-teams.md) (которая также перемещается в Skype для бизнеса Online).  Возможность перемещения непосредственно из локальной группы в Microsoft Teams доступна в Skype для бизнеса Server 2019, а также накопительном обновлении 8 для Skype для бизнеса Server 2015. Организации, использующие более ранние версии Skype для бизнеса Server, могут перемещать пользователей в Teams только с помощью их первого перемещения в Skype для бизнеса Online, а затем применить режим Теамсонли к этим пользователям после подключения к сети.
- [Из сети (только в том случае, если у Teams нет), локальная](move-users-from-the-cloud-to-on-premises.md).

## <a name="required-administrative-credentials"></a>Необходимые административные учетные данные

Для перемещения пользователей между локальной средой и облаком необходимо использовать учетную запись с достаточными привилегиями как в локальной среде Skype для бизнеса Server, так и в клиенте Office 365. Вы можете использовать одну учетную запись со всеми необходимыми привилегиями или использовать две учетные записи, в этом случае вы можете получить доступ к локальным средствам с помощью локальных учетных данных, а затем в этих средствах вы передаете дополнительные учетные данные для Office 365 Учетная запись администратора.  

- В локальной среде пользователь, выполняющий перемещение, должен иметь роль Кссерверадминстратор в Skype для бизнеса Server.
- В Office 365 пользователь, выполняющий перемещение, должен быть либо глобальным администратором, либо иметь роль администратора Skype для бизнеса и администратора пользователей.  

    > [!Important]
    > - Если вы используете панель управления для администрирования Skype для бизнеса, вам будет предложено ввести учетные данные для учетной записи Office 365 с соответствующими ролями, как отмечалось выше. Необходимо указать учетную запись, заканчивающуюся на. onmicrosoft.com. Если это невозможно, используйте командлет Move-CsUser.
    >- При использовании командлета Move-CsUser в PowerShell можно использовать учетную запись, которая заканчивается на. onmicrosoft.com, или можно использовать любую локальную учетную запись, которая синхронизируется в Azure AD, при условии, что вы также указываете параметр Хостедмигратионоверридеурл в командлете . Значение URL-адреса переопределения размещенной миграции является вариантом следующего URL-адреса:https://adminXX.online.lync.com/HostedMigration/hostedmigrationService.svc<br>В приведенном выше URL-адресе замените XX на два или три символа, который определяется следующим образом:
    >   - В сеансе PowerShell Skype для бизнеса Online выполните следующий командлет:<br>`Get-CsTenant|ft identity`
    >    - Полученное значение будет иметь следующий формат:<br>`OU=<guid>,OU=OCS Tenants,DC=lyncXX001,DC=local`
    >    - Код из двух или трех цифр — XX, содержащийся в разделе, DC = lyncXX001. Если это код из двух символов, он будет содержать цифру, за которой следует число (например, 0A). Если это код из трех символов, он будет содержать две буквы, за которыми следует цифра (например, JP1). Во всех случаях вы увидите 001 сразу после кода XX.


## <a name="voice-configuration-requirements"></a>Требования к настройке голосовой связи

Если для пользователей настроена Корпоративная голосовая связь в локальной среде, вам потребуется координировать обновление своей настройки голосовой связи при переносе в Online, или же их можно перенести без возможностей телефонной связи. Доступные параметры зависят от того, будет ли пользователь использовать клиент Teams или Skype для бизнеса в сети:

- Вы можете обновить поставщик телефонии пользователя, чтобы использовать [план звонков корпорации Майкрософт](/microsoftteams/calling-plans-for-office-365). Этот вариант позволяет пользователям работать с клиентами Teams или Skype для бизнеса.
- Вы можете продолжать использовать локальную службу PSTN:
  - Голосовые пользователи, которые будут использовать Teams, должны быть настроены для [прямой маршрутизации](/microsoftteams/direct-routing-plan). Прямая маршрутизация доступна только после того, как пользователь перемещается из локальной сети в сеть.
  - Голосовые пользователи, которые будут использовать клиент Skype для бизнеса после перемещения в Интернете, должны быть настроены для функций гибридной голосовой связи Skype для бизнеса.

Дополнительные сведения о параметрах телефонии в гибридных средах, а также о матрице поддержки можно узнать [в статье учетные записи пользователей в гибридной среде с подключением PSTN](/microsoftteams/direct-routing-user-accounts-in-a-hybrid-environment).

## <a name="other-considerations"></a>Прочие рекомендации

Политики (например, для управления обменом сообщениями, собраниями и вызовами) в локальных и онлайн-средах независимы. Рекомендуется настроить все политики в среде и назначить их пользователю перед перемещением этого пользователя из локальной среды в облако, чтобы у них была правильная конфигурация, как только они будут перенесены в оперативный режим.
