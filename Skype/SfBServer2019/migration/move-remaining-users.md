# <a name="move-remaining-users-to-skype-for-business-server-2019"></a>Перемещение остальных пользователей в Skype для бизнеса Server 2019 г.

Вы можете переместить пользователей в новое развертывание Skype для бизнеса Server 2019 г., используя панель управления Skype для бизнеса Server или Skype для бизнеса Server управленческой оболочки. Чтобы обеспечить плавный переход к Skype для бизнеса Server 2019 г., необходимо выполнить некоторые требования. Сведения о необходимых предпосылках для завершения процедур в этом разделе см. в разделе [Настройка клиентов для миграции.](configure-clients-for-migration.md) Подробные действия по перемещению пользователей см. в [этапе 4. Перемещение](phase-4-move-test-users-to-the-pilot-pool.md)тестовых пользователей в пилотный пул.
  
> [!IMPORTANT]
> Нельзя использовать оснастку Пользователей и компьютеров Active Directory или устаревшие средства администрирования для перемещения пользователей из устаревшей среды в Skype для бизнеса Server 2019 г. 
  
При перемещении пользователя в пул Skype для бизнеса Server 2019 г. данные для пользователя перемещаются в базу данных, связанную с новым пулом. 
  
> [!IMPORTANT]
> Это относится к активным собраниям, созданным пользователем старой системы. Например, если устаревший пользователь настроил мою конференцию собраний, эта конференция по-прежнему будет доступна в пуле Skype для бизнеса Server 2019 г. после перемещений пользователя.  Сведения для доступа к этому собранию — те же **URL-адрес и код конференции**. Единственное отличие состоит в том, что конференция теперь находится в пуле Skype для бизнеса Server 2019, а не в пуле устаревших. 
  
> [!NOTE]
> Для размещения пользователей в Skype для бизнеса Server 2019 г. не требуется одновременное развертывание обновленных клиентов. Новые функциональные возможности станут доступными пользователям только после обновления до новой версии клиентского программного обеспечения. 
  
### <a name="post-migration-task"></a>Задача после миграции

1. После перемещения пользователей убедитесь, что для них назначена политика конференц-связи. 
    
2. Для обеспечения бесперебойной работы собраний, организованных пользователями Skype для бизнеса Server 2019 г. с федеративными пользователями, которые находятся на устаревшей установке, политика конференций, назначенная перенаселенным пользователям, должна разрешать анонимным участникам.
    
3. Политики конференций, позволяющие анонимным  участникам приглашать анонимных пользователей, выбранных в панели управления Skype для бизнеса Server 2019 г., и позволяют **РазрешитьAnonymousParticipantsInMeetings** в выводах из команды **Get-CsConferencingPolicy** в Skype для бизнеса Server Management Shell.
