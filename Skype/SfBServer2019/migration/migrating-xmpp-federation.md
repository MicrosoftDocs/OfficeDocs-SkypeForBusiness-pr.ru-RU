# <a name="migrating-xmpp-federation"></a>Перенос федерации XMPP

В предыдущих версиях был предусмотрен шлюз протокола обмена сообщениями и присутствия (XMPP), который можно было развернуть в качестве отдельной роли сервера, чтобы разрешить федерирование с помощью развертывания XMPP. Функциональность XMPP больше недоступна и не будет Skype для бизнеса Server 2019 г. Если вы хотите продолжить работу с функциями XMPP, вы можете сделать это в среде сосуществования с устаревшей версией (Skype для бизнеса Server 2015 или Lync Server 2013). Функции XMPP установлены в двух частях: в качестве прокси-сервера XMPP, который работает на устаревшем edge Server, и шлюза XMPP, который работает на устаревшем переднем сервере. 
  
С точки зрения миграции пользователи, которые хотят воспользоваться функцией XMPP, должны оставаться на устаревшем сервере и не должны быть перемещены в пул Skype для бизнеса Server 2019 г., но продолжают использовать устаревший шлюз XMPP. Это возможно только при настройке федерательного партнера XMPP в Skype для бизнеса Server 2015 или Lync Server 2013. Не следует перенести устаревший edge Server на Skype для бизнеса Server 2019 г., если вы хотите продолжить работу с функциями XMPP. Однако вы можете иметь сосуществование устаревшего edge Server (с XMPP Proxy) и Skype для бизнеса 2019 Edge Server.
