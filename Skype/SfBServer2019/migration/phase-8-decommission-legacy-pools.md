# <a name="phase-8-decommission-legacy-pools"></a>Этап 8: ликвидация старых пулов

В следующем разделе содержится руководство по обновлению записей DNS, перемещению сервера управления контентом, выводу пулов из эксплуатации, а также отключению и удалянию серверов и пулов из устаревшего развертывания. Не все процедуры, рассматриваемые в этом разделе, являются обязательными. Ознакомьтесь в документацией и определите, какую из процедур списания следует использовать. 
  
Чтобы получить датированную, но всеядную статью об удалении серверов и ролей серверов, а также пошаговую руководство по списанию развертывания, скачайте удаление [Microsoft Lync Server](https://go.microsoft.com/fwlink/p/?linkId=246227)и удаление ролей сервера. 
  
> [!IMPORTANT]
> Сведения о переносе и обновлении приложений Microsoft Unified Communications Managed API (UCMA) перед списанием устаревшей среды см. в сценариях сосуществования, миграции и обновления [UCMA.](https://go.microsoft.com/fwlink/p/?LinkId=269555)
