# <a name="verify-user-replication-has-completed"></a>Проверка выполнения пользовательской репликации

При запуске **комлета Move-CsUser** может возникнуть сбой, если сведения пользователей между службами домена Active Directory (AD DS) и базами данных Skype для бизнеса Server 2019 г. не синхронизируются, так как первоначальная репликация не завершена. Время, необходимое для успешного завершения начальной синхронизации службы репликатора пользователей Skype для бизнеса Server 2019 г., зависит от количества контроллеров домена, которые размещены в лесу Active Directory, в котором размещен пул Skype для бизнеса Server 2019 г. Начальный Skype для бизнеса Server службы репликатора пользователей в 2019 г. происходит при первом Skype для бизнеса Server интерфейсного сервера 2019 г. После этого синхронизация основана на интервале репликатора пользователя. Выполните следующие действия, чтобы убедиться в том, что репликация пользователей завершена перед запуском **cmdlet Move-CsUser.** 
  
### <a name="to-verify-that-user-replication-has-completed"></a>Проверка завершения репликации пользователей

1. Войдите в систему компьютера, на котором построитель топологий установлен как член группы администраторов доменов и группы RTCUniversalServerAdmins.
    
2. Нажмите кнопку **Пуск** и выберите пункт **Выполнить**. 
    
3. Введите **eventvwr.exe,** а затем нажмите **кнопку ОК**.
    
4. В поле Просмотра событий щелкните **журналы Приложения** и Службы, чтобы расширить его, а затем выберите Skype для бизнеса Server. 
    
5. В области **Действия** щелкните **Фильтр текущего журнала**.
    
6. В списке **Источники событий** щелкните пункт **LS User Replicator**.
    
7. **\<All Event IDs\>** Введите **30024** и нажмите **кнопку ОК**. 
    
8. В списке отфильтрованных событий на вкладке **General** найди запись, в которую будет задано, что репликация пользователей успешно завершена.
