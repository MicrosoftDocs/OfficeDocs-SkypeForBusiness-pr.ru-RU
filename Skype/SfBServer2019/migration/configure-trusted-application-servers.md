# <a name="configure-trusted-application-servers"></a>Настройка доверенных серверов приложений

В смешанной среде при создании нового доверенного сервера приложений необходимо установить пул следующего прыжка в пул Skype для бизнеса Server 2019. В смешанной среде устаревший пул и пул Skype для бизнеса Server 2019 отображаются в выпадаемом списке. Предыдущую версию пула выбрать нельзя.
  
> [!IMPORTANT]
> При переносе доверенного сервера приложений также следует обновить используемую версию UCMA. При создании нового пула доверенных приложений для Skype для бизнеса Server 2019 необходимо обновить UCMA до версии, включенной в Skype для бизнеса Server 2019 или последней доступной версии. 
  
### <a name="select-skype-for-business-server-2019-as-next-hop-when-creating-a-trusted-application-server"></a>Выбор Skype для бизнеса Server 2019 в качестве следующего перехода при создании сервера доверенных приложений

1. Откройте построитель топологий.
    
2. В левой области щелкните  правой кнопкой мыши серверы доверенных приложений и выберите **"Новый пул доверенных приложений".**
    
3. Введите **FQDN** пула доверенных приложений и выберите, будет ли это один или несколько серверов. 
    
4. Нажмите **Далее**.
    
5. On the **Select the next hop** page, from the list, select the Skype for Business Server 2019 Front End pool. 
    
6. Нажмите кнопку **Готово**.
    
7. Выберите верхний узел **Skype для бизнеса Server** и в меню действий выберите **"Опубликовать".** 
    
    Убедитесь, **что пул** доверенных приложений успешно создан и связан с правильным пулом переднего входа.
