# <a name="configure-xmpp-gateway-access-policies-and-certificates"></a>Настройка политик доступа и сертификатов шлюза XMPP

Федерация XMPP определяет внешнее развертывание на основе протокола XMPP. Конфигурация XMPP позволяет пользователям получать доступ к пользователям домена XMPP по:
  
- Im и Presence — только от человека к человеку
    
- Создание федерационных контактов XMPP в Skype для бизнеса клиенте
    
При настройке политик для поддержки федерадных партнеров XMPP политики применяются к пользователям федераированных доменов XMPP, но не к пользователям протоколов инициации сеанса (SIP) поставщиков услуг мгновенных сообщений (IM) или федератальных доменов SIP. Для каждого федератного домена XMPP настраивается федераированный партнер XMPP, который позволяет пользователям добавлять контакты и общаться с ними. После задания политик необходимо настроить сертификаты шлюза XMPP. 
  
> [!NOTE]
> В 2019 г. функции XMPP не будут Skype для бизнеса Server, но могут быть продолжены на устаревшем сервере в сосуществовании с Skype для бизнеса Server 2019 г. Убедитесь, что вы уже развернули устаревший сервер (Skype для бизнеса Server 2015 / Lync Server 2013) XMPP Gateway и настроили политики доступа, чтобы пользователи могли использовать устаревшие шлюзы XMPP. Подробные сведения см. в [материале Migrating XMPP Federation](migrating-xmpp-federation.md). 
  
### <a name="configure-an-external-access-policy-to-enable-users-for-legacy-xmpp-gateway"></a>Настройка политики внешнего доступа, чтобы включить пользователей для устаревших шлюзов XMPP

1. Откройте устаревшую панель Skype для бизнеса Server управления.
    
2. На левой панели навигации щелкните **Federation and External Access** (Федерация и внешний доступ), а затем щелкните **External Access Policy** (Политика внешнего доступа).
    
3. Нажмите кнопку **Создать** и щелкните **Политика пользователя**.
    
4. Введите имя для политики внешнего доступа пользователей.
    
5. Добавьте описание политики внешнего доступа пользователей.
    
6. Выберите **Enable communications with federated users** (Разрешить взаимодействие с федеративными пользователями).
    
7. Выберите **Enable communications with XMPP federated users** (Разрешить взаимодействие с федеративными пользователями XMPP).
    
8. Щелкните **Commit** (Сохранить), чтобы сохранить изменение политики сайта или пользователей.
