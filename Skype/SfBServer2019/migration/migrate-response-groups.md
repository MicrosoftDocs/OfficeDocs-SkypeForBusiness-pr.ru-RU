# <a name="migrate-response-groups"></a>Перенос групп ответа

После переноса пользователей в пулы Skype для бизнеса Server 2019 года можно перенести группы ответов. Миграционные группы ответов включают копирование групп агентов, очередей, рабочего процесса, аудиофайлов и перемещение контактных объектов группы реагирования из устаревшего развертывания в пул Skype для бизнеса Server 2019 г. После переноса устаревших групп ответов вызовы в группы ответов обрабатываются приложением Группы ответов в пуле Skype для бизнеса Server 2019 г. Вызовы групп ответа больше не будут обрабатываться устаревшим пулом.
  
> [!NOTE]
> Хотя перед перемещением всех пользователей в пул Skype для бизнеса Server 2019 года можно перенести группы откликов, рекомендуется сначала переместить всех пользователей. В частности, пользователи, которые являются агентами группы отклика, не будут иметь полную функциональность новых функций до тех пор, пока они не будут перемещены в пул Skype для бизнеса Server 2019 г. 
  
Перед переносом групп ответов необходимо развернуть пул Skype для бизнеса Server 2019 г., включив приложение Группы реагирования. Приложение Группы реагирования устанавливается и активируется по умолчанию при развертывании Корпоративная голосовая связь. Вы можете убедиться, что приложение Группы реагирования установлено с помощью комлета **Get-CsService-ApplicationServer.** 
  
> [!NOTE]
> Вы можете создать новые группы Skype для бизнеса Server 2019 в пуле Skype для бизнеса Server 2019, прежде чем перенести устаревшие группы откликов. 
  
Чтобы перенести группы откликов из устаревшего пула в Skype для бизнеса Server 2019 г., вы запустите группу **Move-CsRgsConfiguration.** 
  
> [!IMPORTANT]
> Комлет группы реагирования перемещает конфигурацию группы реагирования для всего пула. Нельзя выбрать для миграции конкретные группы, очереди или рабочие процессы. 
  
После переноса групп реагирования необходимо использовать Skype для бизнеса Server панели управления или Skype для бизнеса Server команды Management Shell, чтобы убедиться, что все группы агентов, очереди и рабочий процесс успешно перемещаются. 
  
При переносе групп ответов устаревшие группы ответов не удаляются. При управлении группами ответа после миграции с помощью панели управления Skype для бизнеса Server или Skype для бизнеса Server Management Shell можно увидеть как устаревшие группы ответов, так и группы Skype для бизнеса Server 2019 года. Обновления следует применять только к группам Skype для бизнеса Server 2019 года. Устаревшие группы ответов сохраняются только для целей отката. 
  
> [!CAUTION]
> После завершения миграции и создания новых групп реагирования панель управления Skype для бизнеса Server и панель управления Skype для бизнеса Server управления будут отображать устаревшие и Skype для бизнеса Server версии каждой группы ответа за 2019 год. Не используйте панель Skype для бизнеса Server или Skype для бизнеса Server для удаления устаревших групп ответа. Если удалить один из них, соответствующая группа откликов, созданная во время миграции, прекратит работу. Устаревшие группы ответов будут удалены при выводе из эксплуатации устаревшего пула. 
  
> [!IMPORTANT]
> Рекомендуется не удалять данные предыдущего развертывания до ликвидации пула. Кроме того, настоятельно рекомендуется экспортировать группы ответов сразу же после миграции. Если устаревшую группу ответов следует удалить, можно восстановить группы ответов из резервного копирования, чтобы Skype для бизнеса Server 2019 групп ответа снова. 
  
Skype для бизнеса Server 2019 г. представляет новую функцию Группы ответа под названием **Тип рабочего процесса.** **Тип рабочего процесса** может быть **управляемым** или **неуправляемым**. Все группы ответа переносятся с помощью **неуправляемого****типа рабочего процесса** и с пустым списком управляющих. 
  
При выполнении командлета **Move-CsRgsConfiguration** группы агентов, очереди, рабочие процессы и звуковые файлы остаются в устаревшем пуле на случай отката. Однако если потребуется откат в устаревший пул, придется выполнить командлет **Move-CsApplicationEndpoint**, чтобы переместить контактные объекты обратно в устаревший пул. 
  
Следующая процедура переноса конфигураций групп реагирования предполагает, что между устаревшими пулами и пулами Skype для бизнеса Server 2019 года у вас есть отношение один к одному. Если вы планируете консолидировать или разделить пулы во время миграции и развертывания, необходимо спланировать, какие устаревшие карты пула Skype для бизнеса Server 2019 г.
  
## <a name="to-migrate-response-group-configurations"></a>Перенос конфигураций группы реагирования

1. Войдите в компьютер с учетной записью, которая является членом группы RTCUniversalServerAdmins или имеет эквивалентные права и разрешения администратора.
    
2. Запустите Skype для бизнеса Server: нажмите кнопку Начните, нажмите кнопку Все **программы,** нажмите **Microsoft Skype для бизнеса Server 2019,** а затем нажмите кнопку **Skype для бизнеса Server управленческой оболочки**.
    
3. Запуск:
    
   ```PowerShell
   Move-CsRgsConfiguration -Source <source pool FQDN> -Destination <destination pool FQDN>
   ```

    Пример:
    
   ```PowerShell
   Move-CsRgsConfiguration -Source skype-old.contoso.net -Destination skype-new.contoso.net
   ```

4. После переноса групп откликов и агентов в пул Skype для бизнеса Server 2019 г. URL-адрес, который агенты используют для регистрации и регистрации, является URL-адресом Skype для бизнеса Server 2019 г. и доступен в меню **Tools.** Напомните агентам обновить все ссылки, например закладки, с учетом этого нового URL-адреса. 
    
## <a name="to-verify-response-group-migration-by-using-skype-for-business-server-control-panel"></a>Проверка миграции группы реагирования с помощью панели Skype для бизнеса Server управления

1. Войдите в компьютер с учетной записью, которая является членом группы RTCUniversalReadOnlyAdmins или как минимум членом роли CsViewOnlyAdministrator.
    
2. Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель Skype для бизнеса Server управления. Сведения о различных методах, которые можно использовать для Skype для бизнеса Server панели управления, см. в материале [Open Skype для бизнеса Server 2019.](/previous-versions/office/lync-server-2013/lync-server-2013-open-lync-server-administrative-tools) 
    <!-- The above link points to un-rebranded 2013 content we will need to discuss rebrand or bring forward -->
3. В левой панели навигации выберите пункт **Группы ответа**.
    
4. На **вкладке Workflow** убедитесь, что все процессы в устаревшей среде включены в список. 
    
5. Щелкните **вкладку Queue** и убедитесь, что все очереди в устаревшей среде включены в список. 
    
6. Щелкните **вкладку Group** и убедитесь, что все группы агентов в устаревшей среде включены в список. 
    
## <a name="to-verify-response-group-migration-by-using-skype-for-business-server-management-shell"></a>Проверка миграции группы реагирования с помощью Skype для бизнеса Server shell

1. Войдите в компьютер с учетной записью, которая является членом группы RTCUniversalReadOnlyAdmins или как минимум членом роли CsViewOnlyAdministrator.
    
2. Запустите Skype для бизнеса Server: нажмите кнопку Начните, нажмите кнопку Все **программы,** нажмите **Microsoft Skype для бизнеса Server 2019,** а затем нажмите кнопку **Skype для бизнеса Server управленческой оболочки**.
    
    Чтобы получить сведения о следующих командлетах, выполните следующую команду:
    
   ```PowerShell
   Get-Help <cmdlet name> -Detailed
   ```

3. Запуск:
    
   ```PowerShell
   Get-CsRgsAgentGroup
   ```

4. Убедитесь, что все группы агентов в устаревшей среде включены в список.
    
5. Запуск:
    
   ```PowerShell
   Get-CsRgsQueue
   ```

6. Убедитесь, что все очереди в устаревшей среде включены в список.
    
7. Запуск:
    
   ```PowerShell
   Get-CsRgsWorkflow
   ```

8. Убедитесь, что все процессы в устаревшей среде включены в список.
