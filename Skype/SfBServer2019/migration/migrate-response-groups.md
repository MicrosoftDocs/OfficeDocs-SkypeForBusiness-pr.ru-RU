# <a name="migrate-response-groups"></a>Перенос групп ответа

После переноса пользователей в пулы Skype для бизнеса Server 2019 можно перенести группы ответа. Миграция групп ответа включает копирование групп агентов, очередей, бизнес-процессов, звуковых файлов и перемещение контактных объектов группы ответа из устаревшего развертывания в пул Skype для бизнеса Server 2019. После переноса устаревших групп ответа вызовы в группы ответа обрабатываются приложением группы ответа в пуле Skype для бизнеса Server 2019. Вызовы групп ответа больше не будут обрабатываться устаревшим пулом.
  
> [!NOTE]
> Хотя вы можете перенести группы ответа перед перемещением всех пользователей в пул Skype для бизнеса Server 2019, рекомендуется сначала переместить всех пользователей. В частности, пользователи, которые являются агентами группы ответа, не будут иметь полной функциональности новых функций, пока они не будут перемещены в пул Skype для бизнеса Server 2019. 
  
Перед миграцией групп ответа необходимо развернуть пул Skype для бизнеса Server 2019, который включает приложение группы ответа. Приложение группы ответа устанавливается и активируется по умолчанию при развертывании Корпоративная голосовая связь. Чтобы убедиться в том, что приложение группы ответа установлено, вы можете с помощью **get-CsService -ApplicationServer.** 
  
> [!NOTE]
> Вы можете создать новые группы ответа Skype для бизнеса Server 2019 в пуле Skype для бизнеса Server 2019 перед переносом устаревших групп ответа. 
  
Чтобы перенести группы ответа из устаревшего пула в Skype для бизнеса Server 2019, запустите **cmdlet Move-CsRgsConfiguration.** 
  
> [!IMPORTANT]
> При миграции группы ответа конфигурация группы ответа перемещается для всего пула. Нельзя выбрать для миграции конкретные группы, очереди или рабочие процессы. 
  
After you migrate the response groups, you need to use Skype for Business Server Control Panel or Skype for Business Server Management Shell cmdlets to verify that all agent groups, queues, and workflows moved successfully. 
  
При переносе групп ответа устаревшие группы ответа не удаляются. When you manage response groups after migration by using either Skype for Business Server Control Panel or Skype for Business Server Management Shell, you can see both the legacy response groups and the Skype for Business Server 2019 response groups. Обновления следует применять только к группам ответа Skype для бизнеса Server 2019. Устаревшие группы ответа сохраняются только для отката. 
  
> [!CAUTION]
> После завершения миграции и создания новых групп ответа на панели управления Skype для бизнеса Server и в оболочке управления Skype для бизнеса Server будут отображаться устаревшие версии и версии каждой группы ответа в Skype для бизнеса Server 2019. Не используйте панель управления Skype для бизнеса Server или skype для бизнеса Server Management Shell для удаления устаревших групп ответа. Если удалить его, соответствующая группа ответа, созданная во время миграции, перестанет работать. Устаревшие группы ответа будут удалены при выводе устаревшего пула из эксплуатации. 
  
> [!IMPORTANT]
> Рекомендуется не удалять данные предыдущего развертывания до ликвидации пула. Кроме того, настоятельно рекомендуется экспортировать группы ответов сразу же после миграции. Если устаревшая группа ответа должна быть удалена, вы можете восстановить группы ответа из резервной копии, чтобы снова запускать группы ответа Skype для бизнеса Server 2019. 
  
Skype для бизнеса Server 2019 представляет новую функцию группы ответа под названием **"Тип рабочего процесса".** **Тип рабочего процесса** может быть **управляемым** или **неуправляемым**. Все группы ответа переносятся с помощью **неуправляемого****типа рабочего процесса** и с пустым списком управляющих. 
  
При выполнении командлета **Move-CsRgsConfiguration** группы агентов, очереди, рабочие процессы и звуковые файлы остаются в устаревшем пуле на случай отката. Однако если потребуется откат в устаревший пул, придется выполнить командлет **Move-CsApplicationEndpoint**, чтобы переместить контактные объекты обратно в устаревший пул. 
  
Следующая процедура переноса конфигураций группы ответа предполагает, что между устаревшими пулами и пулами Skype для бизнеса Server 2019 имеется отношение "один к одному". Если вы планируете консолидировать или разделить пулы во время миграции и развертывания, необходимо спланировать, какой устаревший пул сопобится с пулом Skype для бизнеса Server 2019.
  
## <a name="to-migrate-response-group-configurations"></a>Перенос конфигураций группы ответа

1. Войдите в компьютер с учетной записью, которая является членом группы RTCUniversalServerAdmins или имеет эквивалентные права и разрешения администратора.
    
2. Запустите оболочку управления Skype для бизнеса Server: нажмите кнопку "Начните", выберите "Все программы", "Microsoft Skype для бизнеса Server **2019"** и щелкните "Skype для бизнеса Server Management **Shell".**
    
3. Запустите:
    
   ```PowerShell
   Move-CsRgsConfiguration -Source <source pool FQDN> -Destination <destination pool FQDN>
   ```

    Пример:
    
   ```PowerShell
   Move-CsRgsConfiguration -Source skype-old.contoso.net -Destination skype-new.contoso.net
   ```

4. После переноса групп ответа и агентов в пул Skype для бизнеса Server 2019 url-адрес, который агенты используют для входов  и выходов, является URL-адресом Skype для бизнеса Server 2019 и доступен в меню "Сервис". Напомните агентам обновить все ссылки, например закладки, с учетом этого нового URL-адреса. 
    
## <a name="to-verify-response-group-migration-by-using-skype-for-business-server-control-panel"></a>Проверка миграции группы ответа с помощью панели управления Skype для бизнеса Server

1. Войдите в компьютер с учетной записью, которая является членом группы RTCUniversalReadOnlyAdmins или как минимум членом роли CsViewOnlyAdministrator.
    
2. Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Skype для бизнеса Server. Подробные сведения о различных методах запуска панели управления Skype для бизнеса Server см. в средствах администрирования Open Skype для бизнеса [Server 2019.](https://technet.microsoft.com/library/gg195741(v=ocs.15).aspx) 
    <!-- The above link points to un-rebranded 2013 content we will need to discuss rebrand or bring forward -->
3. В левой панели навигации выберите пункт **Группы ответа**.
    
4. На **вкладке "Рабочий** процесс" убедитесь, что все процессы в устаревшей среде включены в список. 
    
5. Перейдите **на вкладку** "Очередь" и убедитесь, что все очереди в устаревшей среде включены в список. 
    
6. Перейдите **на вкладку "Группа"** и убедитесь, что все группы агентов в устаревшей среде включены в список. 
    
## <a name="to-verify-response-group-migration-by-using-skype-for-business-server-management-shell"></a>Проверка миграции группы ответа с помощью оболочки управления Skype для бизнеса Server

1. Войдите в компьютер с учетной записью, которая является членом группы RTCUniversalReadOnlyAdmins или как минимум членом роли CsViewOnlyAdministrator.
    
2. Запустите оболочку управления Skype для бизнеса Server: нажмите кнопку "Начните", выберите "Все программы", "Microsoft Skype для бизнеса Server **2019"** и щелкните "Skype для бизнеса Server Management **Shell".**
    
    Чтобы получить сведения о следующих командлетах, выполните следующую команду:
    
   ```PowerShell
   Get-Help <cmdlet name> -Detailed
   ```

3. Запустите:
    
   ```PowerShell
   Get-CsRgsAgentGroup
   ```

4. Убедитесь, что все группы агентов в устаревшей среде включены в список.
    
5. Запустите:
    
   ```PowerShell
   Get-CsRgsQueue
   ```

6. Убедитесь, что все очереди в устаревшей среде включены в список.
    
7. Запустите:
    
   ```PowerShell
   Get-CsRgsWorkflow
   ```

8. Убедитесь, что все процессы в устаревшей среде включены в список.
