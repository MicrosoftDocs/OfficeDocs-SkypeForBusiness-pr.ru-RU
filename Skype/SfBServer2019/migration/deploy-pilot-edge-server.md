# <a name="deploy-pilot-edge-server"></a>Развертывание пилотного пограничного сервера

В этом разделе освещаются параметры конфигурации, которые необходимо знать перед развертывание Skype для бизнеса Server 2019 edge Server. Процессы развертывания и конфигурации Skype для бизнеса Server 2019 г. очень похожи на Skype для бизнеса Server 2015 г. В этом разделе рассматриваются только ключевые моменты, которые следует учитывать при развертывании пилотной версии пула. <!-- For detailed steps, see 
 [Deploying external user access in Skype for Business Server 2019](../deployment/deploying-external-user-access/deploying-external-user-access.md) in the Deployment documentation, which describes the deployment process and also gives configuration information for external user access.  -->
  
По мере перехода по страницам мастера **Определение нового пограничного пула** анализируйте основные параметры конфигурации, показанные на следующих шагах. Обратите внимание, что показано лишь несколько страниц мастера **Определение нового пограничного пула**. 
  
### <a name="to-define-an-edge-pool"></a>Определение пула edge

1. Войдите в систему компьютера, на котором построитель топологий установлен как член группы администраторов доменов и группы RTCUniversalServerAdmins.
    
2. Перейдите к узлу Skype для бизнеса Server 2019. Щелкните правой кнопкой мыши пункт **Пограничные пулы** и выберите в контекстном меню команду **Создать пограничный пул**.
    
     ![Определите диалоговое окно New Edge Pool.](../media/migration_ocs_topo_edgepool_page1.JPG)
  
3. Пограничный пул может представлять собой **Пул нескольких компьютеров** или **Пул одиночного компьютера**.
    
     ![Определите диалоговое окно Edge Pool FQDN.](../media/migration_ocs_topo_edgepool_page2.JPG)
  
4. На странице **Выбор компонентов** не включайте ни федерацию, ни федерацию XMPP. Федерация и федерация XMPP в настоящее время проходят маршрут через устаревший edge Server. Эти компоненты настраиваются на более позднем этапе миграции. 

  
5. Продолжайте заполнять следующие страницы мастера: **внешние FQDNs,** Определение внутреннего **IP-адреса** и определение **внешнего IP-адреса**.
    
6. На странице **Определите следующую страницу сервера хопа** выберите Director для следующего перехода устаревшего пула Edge. 
    
     ![Определите диалоговое окно Next Hop.](../media/migration_ocs_topo_edgepool_page7.JPG)
  
7. На странице **Пулы-посредники** или пулы-посредники не связывать пул с этим пулом Edge в это время. Внешний трафик мультимедиа в настоящее время проходит через устаревший edge Server. Этот параметр настраивается на более позднем этапе миграции. 
    
     ![Диалоговое окно Для диалоговых пулов переднего конца.](../media/migration_ocs_topo_edgepool_page8.JPG)
  
8. Нажмите **кнопку Готово** и **опубликуй** топологию. 
    
9. Выполните действия в документации по развертыванию, чтобы установить файлы на новом edge Server, настроить сертификаты и запустить службы. 
<!-- [Install Edge Servers for Skype for Business Server 2019](../deployment/deploying-external-user-access/install-edge-servers.md) in -->
    
Очень важно следовать рекомендациям в темах документации по развертыванию. В этом разделе приводятся только некоторые указания по параметрам конфигурации при установке данных серверных ролей. 
<!-- [Deploying external user access in Skype for Business Server 2019](../deployment/deploying-external-user-access/deploying-external-user-access.md) -->
  
Теперь необходимо развернуть устаревший edge Server параллельно с развертыванием Skype для бизнеса Server 2019 edge server. Перед переходом на следующий этап убедитесь, что оба развертывания функционируют должным образом, службы запущены и вы можете осуществлять управление каждым развертыванием.
