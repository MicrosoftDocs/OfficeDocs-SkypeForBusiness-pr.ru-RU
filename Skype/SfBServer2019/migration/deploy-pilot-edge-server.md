# <a name="deploy-pilot-edge-server"></a>Развертывание пилотного пограничного сервера

В этом разделе выделены параметры конфигурации, которые необходимо знать перед развертыванием skype для бизнеса Server 2019 Edge Server. Процессы развертывания и настройки Skype для бизнеса Server 2019 очень похожи на skype для бизнеса Server 2015. В этом разделе рассматриваются только ключевые моменты, которые следует учитывать при развертывании пилотной версии пула. <!-- For detailed steps, see 
 [Deploying external user access in Skype for Business Server 2019](../deployment/deploying-external-user-access/deploying-external-user-access.md) in the Deployment documentation, which describes the deployment process and also gives configuration information for external user access.  -->
  
По мере перехода по страницам мастера **Определение нового пограничного пула** анализируйте основные параметры конфигурации, показанные на следующих шагах. Обратите внимание, что показано лишь несколько страниц мастера **Определение нового пограничного пула**. 
  
### <a name="to-define-an-edge-pool"></a>Определение пула

1. Войдите в систему компьютера, на котором построитель топологий установлен как член группы администраторов доменов и группы RTCUniversalServerAdmins.
    
2. Перейдите к узлу Skype для бизнеса Server 2019. Щелкните правой кнопкой мыши пункт **Пограничные пулы** и выберите в контекстном меню команду **Создать пограничный пул**.
    
     ![Диалоговое окно "Определение нового пула по краям"](../media/migration_ocs_topo_edgepool_page1.JPG)
  
3. Пограничный пул может представлять собой **Пул нескольких компьютеров** или **Пул одиночного компьютера**.
    
     ![Диалоговое окно определения FQDN пула](../media/migration_ocs_topo_edgepool_page2.JPG)
  
4. На странице **Выбор компонентов** не включайте ни федерацию, ни федерацию XMPP. Федерация и федерация XMPP в настоящее время маршруты через устаревший сервер. Эти компоненты настраиваются на более позднем этапе миграции. 

  
5. Продолжите заполнение следующих страниц мастера: внешние **FQDNs,** определение внутреннего **IP-адреса** и **определение внешнего IP-адреса.**
    
6. На странице **"Определение сервера следующего прыжка"** выберите директор для следующего прыжка устаревшего пула. 
    
     ![Диалоговое окно "Определение следующего перехода"](../media/migration_ocs_topo_edgepool_page7.JPG)
  
7. На странице **"Связать пулы переднего края"** или "Пулы-посредники" не связывать пул с этим пулом в данный момент. В настоящее время внешний трафик мультимедиа проходит через устаревший сервер. Этот параметр настраивается на более позднем этапе миграции. 
    
     ![Диалоговое окно "Связывать пулы переднего ящика"](../media/migration_ocs_topo_edgepool_page8.JPG)
  
8. Нажмите **кнопку "Готово",** а **затем опубликуем** топологию. 
    
9. Следуйте шагам в документации по развертыванию, чтобы установить файлы на новом сервере, настроить сертификаты и запустить службы.

Очень важно следовать рекомендациям в темах документации по развертыванию. В этом разделе приводятся только некоторые указания по параметрам конфигурации при установке данных серверных ролей. 
<!-- [Deploying external user access in Skype for Business Server 2019](../deployment/deploying-external-user-access/deploying-external-user-access.md) -->
  
Теперь у вас должен быть устаревший сервер, развернутый параллельно с развертыванием сервера Skype для бизнеса Server 2019. Перед переходом на следующий этап убедитесь, что оба развертывания функционируют должным образом, службы запущены и вы можете осуществлять управление каждым развертыванием.
