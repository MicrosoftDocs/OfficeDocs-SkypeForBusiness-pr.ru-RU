# <a name="deploy-skype-for-business-server-2019-pilot-pool"></a>Развертывание Скайп for Business Server 2019 пилотного пула

Один из первого действия, необходимые для перехода на Скайп для Business Server 2019 — развертывание пилотного пула. Пилотного пула —, где выполняется тестирование сосуществования Скайп для Business Server 2019 с устаревшее развертывание. Совместная работа — это временное состояние, занимающего, пока не будут перемещены все пользователи и группы в Скайп для Business Server 2019. 
  
Во время развертывания пилотного пула, используйте мастер определение нового пула переднего плана. Следует развернуть же функции и рабочие нагрузки в вашей Скайп для пилотного пула Business Server 2019, на которых установлена в устаревшем пуле. Если развернут сервер архивации, сервер мониторинга или System Center Operations Manager для архивации и мониторинга старой среды и продолжить архивации и мониторинга во всем переноса, необходимо также развернуть эти функции в вашей Пилотная среда. Версия, развернут в архив и отслеживать ваше прежних версий среды не вводятся данные в вашей Скайп среды Business Server 2019. 
  
> [!NOTE]
> Следующая процедура рассматриваются возможности и параметры, которые следует рассматривать как часть процесса развертывания в целом пилотного пула. В данном разделе рассматриваются только ключевые моменты, которые следует рассматривать как часть развертывания пилотного пула. <!-- For detailed steps, refer to the 
 [Deploying Skype for Business Server 2019](../deployment/deploying-lync-server-2013/deploying-lync-server-2013.md) deployment guide.  -->
  
### <a name="to-deploy-a-skype-for-business-server-2019-pilot-pool"></a>Чтобы развернуть Скайп для пилотного пула Business Server 2019

1. Войдите на компьютер, где установлен построитель топологий, с использованием учетной записи, входящей в группу администраторов домена и группу RTCUniversalServerAdmins.
    
2. Разверните дерево, пока не достигните **Скайп для Business Server 2019** > **Пулы переднего плана Enterprise Edition**.
    
3. Щелкните правой кнопкой мыши **Пулы переднего плана Enterprise Edition** и выберите **Создать пул переднего плана**.
  
4. Введите полное доменное имя пула (FQDN). При определении пилотного пула можно развернуть для пула переднего плана Enterprise Edition или сервера Standard Edition. Скайп для Business Server 2019 не требуется, что функций пилотного пула равны, что было развернуто в устаревшем пуле.
    
    > [!CAUTION]
    > Пул или сервер полное доменное имя, определенное для пилотного пула должно быть уникальным. Он не может совпадать с именем уже развернутого пула прежних версий или других в настоящее время развернуты серверы. 
  
5. На странице " **Выбор компонентов** " установите флажки для компонентов, которые будут на этот пул переднего плана. Например при развертывании только мгновенные сообщения (IM) и сведений о присутствии, следует выбрать флажок конференц-связи, чтобы разрешить многосторонний обмен мгновенными Сообщениями, но (общего пользования PSTN) конференций, корпоративной голосовой связи, не следует выбрать или проверьте контроля допуска звонков диалоговые окна, так как они представляют голосовых, видео-и совместной работы функций. <!-- For additional information on selecting features, see 
 [Define and configure a Front End pool or Standard Edition server in Skype for Business Server 2019](../deployment/deploying-lync-server-2013/define-and-configure-a-front-end-pool-or-standard-edition-server.md) in the Deployment documentation.  -->
  
6. На странице **Выбор серверов с совмещенным расположением ролей** рекомендуется выбирать совместно размещать в Скайп сервера-посредника для Business Server 2019. При объединении топологии прежних версий с Скайп для Business Server 2019, требуется сначала совместно размещать устаревшего сервера-посредника. Слияние топологий и настройка Скайп для сервера-посредника 2019 Business Server, могут принять решение о оставьте выровненных сервера-посредника, или измените его на изолированном сервере при перемещении роль сервера-посредника Скайп для Business Server 2019 далее в процессе развертывания. 
   
7. На странице **связывания ролей сервера с помощью этого пула переднего плана** во время развертывания пилотного пула *не* выберите параметр **Включить для пограничного пула, который следует использовать компонент мультимедиа для этого пула переднего плана** . Эта функция будет включен и включить в более позднем этапе миграции. Оставьте этот параметр снят в данный момент. 
  
8. На странице " **Выбор сервера Office Web Apps** " щелкните **Создать**и укажите полное доменное имя сервера приложений.
  
9. На странице **Определение хранилища архивации SQL Server** , при определении хранилища SQL Server для обоих Скайп для бизнеса сервера архивирование и мониторинг выберите экземпляр SQL Server, созданный ранее для Скайп для Business Server 2019. 
  
10. Чтобы опубликовать топологию, щелкните правой кнопкой мыши узел **Скайп для Business Server** и нажмите кнопку **Опубликовать топологию**.
  
11. По завершении процесса публикации нажмите кнопку **Готово**.
