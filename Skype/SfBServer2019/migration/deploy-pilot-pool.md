# <a name="deploy-skype-for-business-server-2019-pilot-pool"></a>Развертывание Skype для бизнеса Server пилотного пула 2019 г.

Одним из первых шагов, необходимых для переноса Skype для бизнеса Server 2019 г., является развертывание пилотного пула. Пилотный пул — это проверка сосуществования Skype для бизнеса Server 2019 года с устаревшим развертыванием. Сосуществование — это временное состояние, которое длится до тех пор, пока вы не переместили все пользователи и пулы в Skype для бизнеса Server 2019 г. 
  
При развертывании пилотной версии пула используется мастер определения нового интерфейсного пула. Необходимо развернуть те же функции и рабочие нагрузки в пилотном пуле Skype для бизнеса Server 2019 г., что и в устаревшем пуле. Если вы развернули архивацию сервера, сервера мониторинга или System Center диспетчера операций для архивации или мониторинга устаревшей среды, и вы хотите продолжить архивацию или мониторинг во время миграции, необходимо также развернуть эти функции в пилотной среде. Версия, развернутая для архива или мониторинга устаревшей среды, не будет захватывать данные в среде Skype для бизнеса Server 2019 г. 
  
> [!NOTE]
> В описании следующей процедуры рассматриваются компоненты и настройки, которые следует использовать в рамках общего процесса развертывания пилотной версии пула. В этом разделе рассматриваются только ключевые моменты, которые следует учитывать при развертывании пилотной версии пула. <!-- For detailed steps, refer to the 
 [Deploying Skype for Business Server 2019](../deployment/deploying-lync-server-2013/deploying-lync-server-2013.md) deployment guide.  -->
  
### <a name="to-deploy-a-skype-for-business-server-2019-pilot-pool"></a>Развертывание пула Skype для бизнеса Server 2019 г.

1. Войдите в систему компьютера, на котором построитель топологий установлен как член группы администраторов доменов и группы RTCUniversalServerAdmins.
    
2. Расширь дерево, пока **не Skype для бизнеса Server 2019** выпуск Enterprise  >  **пулы переднего конца.**
    
3. Щелкните **правой кнопкой мыши выпуск Enterprise пулы** переднего конца и выберите **Новый пул переднего конца**.
  
4. Введите полное доменное имя пула (FQDN). При выборе пилотного пула можно развернуть пул выпуск Enterprise или выпуск Standard сервер. Skype для бизнеса Server 2019 г. не требуется, чтобы функции пилотного пула совпадали с тем, что было развернуто в вашем устаревшем пуле.
    
    > [!CAUTION]
    > Пул или сервер FQDN, который вы определяете для пилотного пула, должен быть уникальным. Он не может совпадать с именем развернутого в настоящее время устаревшего пула или других серверов, развернутых в настоящее время. 
  
5. На странице **Выбор компонентов** установите флажки рядом с теми компонентами, которые вы хотите добавить в этот интерфейсный пул. Например, при развертывании только функций обмена мгновенными сообщениями (IM) и присутствия необходимо выбрать флажок Conferencing для многопартийного обмена мгновенными сообщениями, но вы не будете выбирать флажки dial-in (PSTN), Корпоративная голосовая связь или call Admission Control, так как они представляют функции голосового, видео- и совместного взаимодействия. <!-- For additional information on selecting features, see 
 [Define and configure a Front End pool or Standard Edition server in Skype for Business Server 2019](../deployment/deploying-lync-server-2013/define-and-configure-a-front-end-pool-or-standard-edition-server.md) in the Deployment documentation.  -->
  
6. На странице **Выберите роли** серверов, расположенных в коллайтах, рекомендуем вам сделать коллокацию сервера-посредника в Skype для бизнеса Server 2019 г. При объединении устаревшей топологии с Skype для бизнеса Server 2019 г. необходимо сначала обнять устаревший сервер-посредник. После объединения топологий и настройки сервера-посредника Skype для бизнеса Server 2019 года можно решить, следует ли сохранить сервер-посредник или изменить его на автономный сервер при переходе роли сервера-посредника на Skype для бизнеса Server 2019 г. в процессе развертывания. 
   
7. На **партнерской** роли сервера на этой странице пула  переднего плана во время развертывания пилотного пула не следует выбирать пул Enable **an Edge,** который будет использоваться медиакомпонентом этого параметра пула переднего плана. Эта функция включается и активируется на более позднем этапе миграции. Пока не устанавливайте этот флажок. 
  
8. На странице **Выбор сервера Office Web Apps** выберите **Создать** и укажите полное доменное имя сервера приложений.
  
9. На странице **Определите** SQL Server архива, определяя хранилище SQL Server для архива Skype для бизнеса Server и мониторинга, выберите экземпляр SQL Server, созданный ранее для Skype для бизнеса Server 2019 года. 
  
10. Чтобы опубликовать топологию, щелкните правой кнопкой **мыши узел Skype для бизнеса Server** и нажмите **кнопку Опубликовать топологию**.
  
11. По завершении процесса публикации нажмите **Готово**.

12. Перед переходом в следующий раздел "Проверка сосуществования пилотного пула с устаревшим пулом" необходимо установить новый пул пилотов Skype для бизнеса Server переднего плана, который мы только что определили в опубликованной топологии, следуйте процедурам, описанным здесь [Install Skype для бизнеса Server](../../SfbServer/deploy/install/install-skype-for-business-server.md) на серверах топологии

13. После завершения предыдущего шага переходить в следующий раздел, чтобы проверить сосуществование пилотного пула с устаревшим пулом.
