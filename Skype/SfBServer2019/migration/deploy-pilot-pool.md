# <a name="deploy-skype-for-business-server-2019-pilot-pool"></a>Развертывание пилотного пула Skype для бизнеса Server 2019

Один из первых действий, необходимых для перехода на Skype для бизнеса Server 2019, заключается в развертывании пилотного пула. Пилотный пул позволяет протестировать сосуществование Skype для бизнеса Server 2019 с предыдущим развертыванием. Сосуществование — это временное состояние, которое продолжается до тех пор, пока все пользователи и пулы не будут перемещены в Skype для бизнеса Server 2019. 
  
При развертывании пилотной версии пула используется мастер определения нового интерфейсного пула. Необходимо развернуть те же функции и рабочие нагрузки в пилотном пуле Skype для бизнеса Server 2019, который находится в устаревшем пуле. Если вы развернули сервер архивации, сервер мониторинга или System Center Operations Manager для архивации или мониторинга устаревшей среды и хотите продолжить архивацию или мониторинг во время миграции, необходимо также развернуть эти функции в пилотной среде. Версия, развернутая для архивации или мониторинга устаревшей среды, не захватывает данные в среде Skype для бизнеса Server 2019. 
  
> [!NOTE]
> В описании следующей процедуры рассматриваются компоненты и настройки, которые следует использовать в рамках общего процесса развертывания пилотной версии пула. В этом разделе рассматриваются только ключевые моменты, которые следует учитывать при развертывании пилотной версии пула. <!-- For detailed steps, refer to the 
 [Deploying Skype for Business Server 2019](../deployment/deploying-lync-server-2013/deploying-lync-server-2013.md) deployment guide.  -->
  
### <a name="to-deploy-a-skype-for-business-server-2019-pilot-pool"></a>Развертывание пилотного пула Skype для бизнеса Server 2019

1. Войдите в систему компьютера, на котором построитель топологий установлен как член группы администраторов доменов и группы RTCUniversalServerAdmins.
    
2. Разверните дерево до тех пор, пока вы не пойдете за интерфейсные пулы **Skype для бизнеса Server 2019**  >  **Enterprise Edition**.
    
3. Щелкните правой кнопкой мыши **интерфейсные пулы Enterprise Edition** и выберите **создать пул переднего плана**.
  
4. Введите полное доменное имя пула. При определении пилотного пула можно развернуть пул переднего плана Enterprise Edition или сервер Standard Edition. В Skype для бизнеса Server 2019 не требуется, чтобы функции пилотного пула точно были развернуты в устаревшем пуле.
    
    > [!CAUTION]
    > Пул или полное доменное имя сервера, определенное для пилотного пула, должны быть уникальными. Он не может быть сопоставлен с именем развернутого устаревшего пула или других серверов, развернутых в данный момент. 
  
5. На странице **Выбор компонентов** установите флажки рядом с теми компонентами, которые вы хотите добавить в этот интерфейсный пул. Например, если развертываются только функции обмена мгновенными сообщениями и сведениями о присутствии, установите флажок конференц-связи, чтобы разрешить многокомпонентную передачу мгновенных сообщений, но не устанавливайте флажки конференц-связи, корпоративной голосовой связи или управления допуском звонков, так как они представляют возможности для конференц-связи, видео и общения. <!-- For additional information on selecting features, see 
 [Define and configure a Front End pool or Standard Edition server in Skype for Business Server 2019](../deployment/deploying-lync-server-2013/define-and-configure-a-front-end-pool-or-standard-edition-server.md) in the Deployment documentation.  -->
  
6. На странице **Выбор размещенных ролей сервера** рекомендуется выбрать расположение сервера-посредника в Skype для бизнеса Server 2019. При объединении устаревшей топологии со Skype для бизнеса Server 2019 необходимо сначала разместить устаревший сервер-посредник. После объединения топологий и настройки сервера-посредника Skype для бизнеса Server 2019 вы можете решить, следует ли оставить совмещенный сервер-посредник или изменить его на изолированный сервер при перемещении роли сервера-посредника в Skype для бизнеса Server 2019 позже в процессе развертывания. 
   
7. На странице " **Сопоставление ролей сервера со страницей пула переднего плана** " во время развертывания пилотного пула *не выбирайте режим* **включения пограничного пула, который будет использоваться компонентом мультимедиа этого пула переднего плана** . Эта функция включается и активируется на более позднем этапе миграции. Пока не устанавливайте этот флажок. 
  
8. На странице **Выбор сервера Office Web Apps** выберите **Создать** и укажите полное доменное имя сервера приложений.
  
9. На странице **Определение хранилища SQL Server для архивации** при определении хранилища SQL Server для архивации и мониторинга Skype для бизнеса Server выберите экземпляр SQL Server, созданный ранее для Skype для бизнеса Server 2019. 
  
10. Чтобы опубликовать топологию, щелкните правой кнопкой мыши узел **сервера Skype для бизнеса Server** и выберите команду **опубликовать топологию**.
  
11. По завершении процесса публикации нажмите **Готово**.

12. Перед переходом к следующему разделу под названием "Проверка сосуществования пилотного пула с устаревшим пулом" необходимо установить новый пул пилотного сервера в Skype для бизнеса Server, который был определен в опубликованной топологии, выполнив описанные здесь процедуры [Установка Skype для бизнеса Server на серверах в топологии](https://docs.microsoft.com/skypeforbusiness/deploy/install/install-skype-for-business-server)

13. После выполнения предыдущего действия перейдите к следующему разделу, чтобы проверить сосуществование пилотного пула с устаревшим пулом.
