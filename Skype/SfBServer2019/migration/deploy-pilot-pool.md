# <a name="deploy-skype-for-business-server-2019-pilot-pool"></a>Развертывание пула Skype для бизнеса Server 2019 Pilot

Один из первых шагов, необходимых для перехода на Skype для бизнеса Server 2019, — это развертывание пилотного пула. В пилотном пуле вы тестируете сосуществование Skype для бизнеса Server 2019 с помощью устаревшего развертывания. Сосуществование — это временное состояние, которое продолжается до тех пор, пока все пользователи и пулы не будут перенесены в Skype для бизнеса Server 2019. 
  
При развертывании пилотного пула вы используете мастер определения нового пула переднего плана. Вы должны развернуть те же функции и рабочие нагрузки в пуле пилотного пула в Skype для бизнеса Server 2019, который вы хотите использовать в вашем пуле старого. Если вы развернули сервер архивации, сервер мониторинга или System Center Operations Manager для архивации или наблюдения за вашей устаревшей средой и хотите продолжить архивацию и мониторинг во время миграции, необходимо также развернуть эти функции в Пилотная среда. Версия, развернутая для архивации или наблюдения за старой средой, не захватывает данные в среде Skype для бизнеса Server 2019. 
  
> [!NOTE]
> В приведенной ниже процедуре описаны возможности и параметры, которые следует рассмотреть в рамках всего процесса развертывания пилотного пула. В этом разделе выделены только ключевые моменты, которые следует принимать во время развертывания пилотного пула. <!-- For detailed steps, refer to the 
 [Deploying Skype for Business Server 2019](../deployment/deploying-lync-server-2013/deploying-lync-server-2013.md) deployment guide.  -->
  
### <a name="to-deploy-a-skype-for-business-server-2019-pilot-pool"></a>Развертывание пула пилотных проектов в Skype для бизнеса Server 2019

1. Войдите на компьютер, где установлен построитель топологий, с использованием учетной записи, входящей в группу администраторов домена и группу RTCUniversalServerAdmins.
    
2. Разверните дерево, пока не дойдете до группы "интерфейс" **в Skype для бизнеса Server 2019** > **Enterprise Edition**.
    
3. Щелкните правой кнопкой мыши **Пулы переднего плана Enterprise Edition** и выберите пункт **создать пул интерфейсов переднего плана**.
  
4. Введите полное доменное имя пула (FQDN). При определении пилотного пула вы можете выбрать развертывание пула переднего плана Enterprise Edition или сервера Standard Edition. В Skype для бизнеса Server 2019 не требуется, чтобы функции пилотного пула соответствовали развернутым в пуле старого.
    
    > [!CAUTION]
    > Полные доменные имена сервера или пула, определенные для пилотного пула, должны быть уникальными. Оно не может совпадать с именем развернутого устаревшего пула или других серверов, развернутых в данный момент. 
  
5. На странице " **Выбор компонентов** " установите флажки для нужных функций в пуле переднего плана. Например, если вы разворачиваете только возможности обмена мгновенными сообщениями и сведения о присутствии, установите флажок Конференц-связь, чтобы разрешить многофакторный обмен мгновенными сообщениями, но вы не выбираете контроль конференций с телефонным подключением (КТСОП), корпоративной голосовой связью или управления допуском звонков. поля, которые представляют собой голосовую, видеосвязь и возможности совместной конференции. <!-- For additional information on selecting features, see 
 [Define and configure a Front End pool or Standard Edition server in Skype for Business Server 2019](../deployment/deploying-lync-server-2013/define-and-configure-a-front-end-pool-or-standard-edition-server.md) in the Deployment documentation.  -->
  
6. На странице **Выбор размещенных ролей сервера** мы рекомендуем вам выбрать вариант выделять сервер-посредник в Skype для бизнеса Server 2019. При слиянии устаревших топологий с помощью Skype для бизнеса Server 2019 мы постараемся, чтобы вы сначала выработали сервер устаревшего исправления. После слияния топологий и настройки сервера исправлений Skype для Business Server 2019 вы можете решить, нужно ли сохранять Объединенный сервер или изменить его на отдельный сервер при перемещении роли сервера-посредника на сервер Skype для бизнеса Server. 2019 позже в процессе развертывания. 
   
7. **В процессе** развертывания пилотного пула *не выбирайте* **пул краев, который будет использоваться компонентом мультимедиа для этого параметра пула переднего плана** . Это функция, которую вы сможете включить и подключить к Интернету на более поздней стадии миграции. Не Запусти этот параметр, пока не будет снят флажок. 
  
8. На странице **выберите сервер Office Web Apps** нажмите кнопку **создать**и укажите полное доменное имя сервера приложений.
  
9. На странице **Определение хранилища сервера SQL** Server при определении магазина SQL Server для архивации и наблюдения в Skype для бизнеса Server выберите экземпляр SQL Server, созданный ранее для Skype для бизнеса Server 2019. 
  
10. Чтобы опубликовать топологию, щелкните правой кнопкой мыши узел **сервера Skype для бизнеса Server** и выберите пункт **топология публикации**.
  
11. После завершения процесса публикации нажмите кнопку **Готово**.

12. Перед переходом к следующему разделу "Проверка сосуществования пилотного пула с помощью устаревшего пула" вам нужно установить новый пул пилотного сервера в Skype для бизнеса, который мы только что определили в опубликованной топологии, выполните указанные ниже действия. [Установка Skype для бизнеса Server на серверах в топологии](https://docs.microsoft.com/skypeforbusiness/deploy/install/install-skype-for-business-server)

13. После того как вы завершите предыдущий шаг, перейдите к следующему разделу, чтобы проверить сосуществование пилотной группы с устаревшим пулом.
