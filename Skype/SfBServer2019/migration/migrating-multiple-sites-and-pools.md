# <a name="migrating-multiple-sites-and-pools"></a>Перенос нескольких сайтов и пулов

Skype для бизнеса Server 2019 г. поддерживает развертывание с несколькими сайтами и несколькими пулами. Процесс переноса нескольких пулов на Skype для бизнеса Server 2019 г. требует следующих соображений: 
  
1. После развертывания пилотного пула Skype для бизнеса Server 2019 г. необходимо определить подмножество пилотных пользователей, которые будут перемещены в пул Skype для бизнеса Server 2019 г., а также методику проверки функциональных возможностей пользователей. Например, после перемещения пользователя в пул пилотов убедитесь, что политика конференций пользователя перенесена на Skype для бизнеса Server 2019 г. 
    
2. После развертывания edge Server в пилотном пуле необходимо проверить, могут ли внешние пользователи взаимодействовать с пулом Skype для бизнеса Server 2019 г.

3. В Skype для бизнеса Server 2019 г. функции persistent Chat, SQL Mirroring и XMPP перестают быть доступны в качестве функций Skype для бизнеса Server 2019 г., но они могут быть продолжены в среде сосуществования, если они ранее были развернуты в устаревшей среде. Если вы хотите продолжать использовать эти функции, следует планировать продолжение среды сосуществования, в которой некоторые пользователи останутся в устаревших пулах.
    
4. После перехода федератированных серверов маршрутов на пилотные Skype для бизнеса Server 2019 edge Servers необходимо проверить, могут ли федераированные пользователи общаться с пулом Skype для бизнеса Server 2019 года.
    
5. После перемещения всех пользователей и контактных объектов, не влиянных на пользователя, необходимо проверить пустость устаревшего пула.
    
6. После проверки пустости устаревшего пула можно отключить пул. 
    
    Сведения о деактивации устаревшего пула и серверов см. в материале [Phase 8: Decommission legacy pools.](phase-8-decommission-legacy-pools.md)
