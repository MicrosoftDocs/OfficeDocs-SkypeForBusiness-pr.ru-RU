# <a name="configure-federation-routes-and-media-traffic"></a>Настройка маршрутов федерации и трафика мультимедиа

Федерация — это отношение доверия между двумя или более доменами SIP, которое разрешает пользователям в разных организациях поддерживать связь по границам сети. После миграции в пилотном пуле необходимо перехода от федеративный маршрут с предыдущей версии пограничных серверов для федеративного маршрута к Скайп для Business Server 2019 пограничных серверов.
  
Используйте следующие процедуры для переноса федеративного маршрута и маршрута мультимедиа-трафика с пограничного сервера и директора предыдущей версии для вашей Скайп Business Server 2019 пограничного сервера для развертывания одним сайтом.
  
> [!IMPORTANT]
> Изменение федеративного маршрута и маршрута мультимедиа-трафика необходимо запланировать время простоя обслуживания Скайп Business Server 2019 и предыдущей версии пограничных серверов. Этот процесс перехода всей также означает, что федеративного доступа будут недоступны во время выполнения во время отказа. Следует планировать время простоя в течение времени, когда ожидать действие минимальной пользователя. Необходимо также предоставить достаточно уведомления конечных пользователей. Составьте соответствующий план для этого сбоя и набор подходят ожиданиям вашей организации. 
  
> [!IMPORTANT]
> Если устаревшего пограничного сервера настроены на использование того же полное доменное имя для пограничной службы доступа, служба пограничного сервера конференц-связи Web и A аудио- и видеоконференций, процедур, описанных в этом разделе, не поддерживаются. Если устаревших служб пограничного сервера настроены на использование того же полное доменное имя, необходимо сначала перенос всех пользователей, а затем ликвидация предыдущих версий пограничного сервера перед включением федерации на Скайп Business Server 2019 пограничного сервера. 
  
> [!IMPORTANT]
> Если ваше федерации XMPP маршрутизированы через Скайп Business Server 2019 пограничного сервера, пользователи из предыдущей версии не смогут для взаимодействия с федеративного партнера XMPP, пока все пользователи были перемещены в Скайп для 2019 Business Server, политики XMPP и сертификаты были настроены, федеративного партнера XMPP настроен на Скайп для Business Server 2019 и, наконец, были обновлены DNS-записи. 
  
## <a name="to-remove-the-legacy-federation-association-from-skype-for-business-server-2019-sites"></a>Удаление устаревшей связи федерации из Скайп для сайтов Business Server 2019

1. На Скайп для сервера переднего плана Business Server 2019 откройте существующую топологию в построителе топологий. 
    
2. В левой области перейдите к узлу сайта, расположенному непосредственно под **Скайп для Business Server**.
    
3. Щелкните сайт правой кнопкой мыши и выберите команду **Изменить свойства**.
    
4. В левой области выберите **Федеративный маршрут**. 
    
5. В разделе **Назначение федеративного маршрута сайта**снимите флажок **Включить SIP-федерацию** , чтобы отключить федеративный маршрут с помощью старой среды. 
  
6. Нажмите **кнопку ОК** , чтобы закрыть страницу изменения свойств. 
    
7. В **Построителе топологии**выберите верхний узел **Скайп для Business Server**.
    
8. В меню **Действие** выберите пункт **Опубликовать топологию**.
    
9. Нажмите кнопку **Далее** для завершения процесса публикации и нажмите кнопку **Готово** после завершения процесса публикации. 
    
## <a name="to-configure-the-legacy-edge-server-as-a-non-federating-edge-server"></a>Настройка устаревшего пограничного сервера без федерации пограничного сервера

1. В левой области перейдите к узлу установки прежних версий, а затем к узлу **пограничные пулы** . 
    
2. Щелкните правой кнопкой мыши пограничный сервер и выберите команду **Изменить свойства**.
    
3. В левой области выберите пункт **Общие** . 
    
4. Снимите флажок **Включение федерации для этого пограничного пула (порт 5061)** и выберите **кнопку ОК** , чтобы закрыть диалоговое окно. 
  
5. В меню **Действие** выберите команду **Опубликовать топологию**и нажмите кнопку **Далее**.
    
6. После завершения работы **мастера публикации** , нажмите кнопку **Готово** , чтобы закрыть мастер. 
    
7. Убедитесь в том, что федерация для устаревшего пограничного сервера отключена в построителе топологий.
  
## <a name="to-configure-certificates-on-the-legacy-edge-server"></a>Настройка сертификатов на устаревшего пограничного сервера

1. Экспортируйте внешний сертификат прокси-сервера доступа с закрытым ключом из устаревшего пограничного сервера. 
    
2. На Скайп для пограничного сервера Business Server 2019 и Импорт внешних прокси-сервера доступа сертификата из предыдущего шага.
    
3. Назначьте внешний сертификат прокси-сервера доступа Скайп для Business Server 2019 внешний интерфейс пограничного сервера.
    
4. Сертификат внутреннего интерфейса Скайп для пограничного сервера Business Server 2019 следует запросить у доверенного центра сертификации и назначен. 
    
## <a name="to-change-the-previous-versions-federation-route-to-use-skype-for-business-server-2019-edge-server"></a>Изменение федеративного маршрута предыдущей версии в целях Скайп Business Server 2019 пограничного сервера

1. В построителе топологий в левой области перейдите к узлу **Скайп для Business Server 2019** , а затем к узлу **пограничные пулы** . 
    
2. Щелкните правой кнопкой мыши пограничный сервер и выберите команду **Изменить свойства**.
    
3. В левой области выберите пункт **Общие** . 
    
4. Установите флажок для **включения федерации для этого пограничного пула (порт 5061)** и нажмите кнопку **ОК** , чтобы закрыть диалоговое окно. 
  
5. В меню **Действие** выберите команду **Опубликовать топологию**и нажмите кнопку **Далее**.
    
6. После завершения работы **мастера публикации** , нажмите кнопку **Готово** , чтобы закрыть мастер. 
    
7. Убедитесь, что **Федерация (порт 5061)** задано значение **включено** в построителе топологий.
    
  
## <a name="to-update-skype-for-business-server-2019-edge-server-federation-next-hop"></a>Для обновления Скайп для следующего перехода федерации пограничного сервера Business Server 2019

1. В построителе топологий в левой области перейдите к узлу **Скайп для Business Server 2019** , а затем к узлу **пограничные пулы** . 
    
2. Разверните узел, щелкните правой кнопкой указанный пограничный сервер и выберите команду **Изменить свойства**. 
    
3. На странице " **Общие** " в разделе **Выбор узла следующего перехода**выберите в раскрывающемся списке Скайп для пула Business Server 2019.
  
4. Нажмите **кнопку ОК** , чтобы закрыть страницу изменения свойств. 
    
5. В **Построителе топологии**выберите верхний узел **Скайп для Business Server**. 
    
6. В меню **Действие** щелкните **Опубликовать топологию** и завершите работу мастера. 
    
## <a name="to-configure-skype-for-business-server-2019-edge-server-outbound-media-path"></a>Настройка Скайп для пограничного сервера Business Server 2019 исходящего пути к мультимедиа

1. В построителе топологий в левой области перейдите к узлу **Скайп для Business Server 2019** , а затем к пулу под **Standard Edition серверы переднего плана** или **Пулы переднего плана Enterprise Edition**.
    
2. Щелкните пул правой кнопкой мыши и выберите команду **Изменить свойства**.
    
3. В разделе **связи** установите флажок **сопоставить пограничный пул (для компонентов мультимедиа)** . 
  
4. Из раскрывающегося списка выберите Скайп Business Server 2019 пограничного сервера.
    
5. Нажмите **кнопку ОК** , чтобы закрыть страницу **Изменения свойств** . 
    
## <a name="to-turn-on-skype-for-business-server-2019-edge-server-federation"></a>Чтобы включить Скайп Business Server 2019 пограничного сервера федерации

1. В построителе топологий в левой области перейдите к узлу **Скайп для Business Server 2019** , а затем к узлу **пограничные пулы** . 
    
2. Разверните узел, щелкните правой кнопкой указанный пограничный сервер и выберите команду **Изменить свойства**. 
    
    > [!NOTE]
    > Федерации можно включить только для одного пограничного пула. Если у вас есть несколько пулов пограничного сервера, выберите один для использования в качестве federating пограничного пула. 
  
3. На странице " **Общие** " Убедитесь, что установлен флажок **Включение федерации для этого пограничного пула (порт 5061)** . 
  
4. Нажмите **кнопку ОК** , чтобы закрыть страницу изменения свойств. 
    
5. Перейдите к узлу сайта. 
    
6. Щелкните сайт правой кнопкой мыши и выберите команду **Изменить свойства**.
    
7. В левой области выберите **Федеративный маршрут**.
    
8. В разделе **Назначение федеративного маршрута сайта**выберите **Включить федерацию SIP**и выберите в списке Скайп Business Server 2019 пограничного сервера из списка. 
  
9. Нажмите **кнопку ОК** , чтобы закрыть страницу **Изменения свойств** . 
    
     Для развертывания нескольких сайтах выполните эту процедуру на каждом сайте. 
    
## <a name="to-publish-edge-server-configuration-changes"></a>Публикация изменений конфигурации пограничного сервера

1. В **Построителе топологии**выберите верхний узел **Скайп для Business Server**. 
    
2. В меню **Действие** выберите команду **Опубликовать топологию** и завершите работу мастера. 
    
3. Дождитесь репликации Active Directory во всех пулах в развертывании.
    
    > [!NOTE]
    > Может появиться следующее сообщение: **Предупреждение: топология содержит более одного федеративного пограничного сервера. Это может произойти при переходе к более поздней версии продукта. В этом случае только один пограничного сервера активно используется для федерации. Убедитесь, что внешние DNS-запись SRV указывает на правильные пограничный сервер. Если вы хотите развернуть несколько федерации пограничного сервера, чтобы быть одновременно активный (то есть не сценарий миграции), убедитесь, что все федеративных партнеров Скайп для используются Business Server. Убедитесь, что внешние DNS-запись SRV перечислены все пограничные серверы включена функция федерации.** Появление этого предупреждения ожидаемое и его можно спокойно проигнорировать. 
  
## <a name="to-configure-skype-for-business-server-2019-edge-server"></a>Чтобы настроить Скайп Business Server 2019 пограничного сервера

1. Переведите все Скайп Интернет-версия для Business Server 2019 пограничных серверов. 
    
2. Обновление правил маршрутизации внешний брандмауэр или параметры балансировки нагрузки оборудования для отправки трафик SIP для внешнего доступа (обычно это порт 443) и федерации (обычно это порт 5061) для Скайп Business Server 2019 пограничного сервера, вместо устаревшего пограничного сервера.
    
    > [!NOTE]
    > Если у вас аппаратный балансировщик нагрузки, вам потребуется обновить запись DNS A для федерации для решения новый Скайп для сервера Business Server Access Edge. Для выполнения этой задачи мешая уменьшите значение TLL для внешних Скайп для полного доменного ИМЕНИ пограничного сервера доступа Business Server, чтобы при обновлении DNS для указания на новый Скайп для пограничного сервера доступа Business сервера федерации и удаленного доступа будут обновлены быстро. 
  
3. Остановите **Скайп для Business Server Access Edge** на каждом компьютере пограничного сервера. 
    
4. На каждом компьютере устаревшего пограничного сервера откройте приложение **службы** в меню **Администрирование**.
    
5. В списке служб найдите **Скайп для пограничного сервера доступа Business Server**.
    
6. Щелкните правой кнопкой мыши имя службы и выберите пункт **Остановить** , чтобы остановить службу. 
    
7. Задайте тип запуска **отключено**. 
    
8. Нажмите **кнопку ОК** , чтобы закрыть окно **свойств** .
