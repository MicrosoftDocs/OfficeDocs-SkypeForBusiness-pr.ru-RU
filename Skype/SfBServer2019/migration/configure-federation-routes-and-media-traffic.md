# <a name="configure-federation-routes-and-media-traffic"></a>Настройка маршрутов федерации и трафика мультимедиа

Федерация — это доверительные отношения между двумя или более SIP-доменами, которые позволяют пользователям в различных организациях обмениваться информацией через сетевые границы. После перехода в пилотный пул необходимо перейти с маршрута федерации edge Servers предыдущей версии на маршрут федерации Skype для бизнеса Server 2019 edge Servers.
  
Используйте следующие процедуры для перехода маршрута федерации и маршрута трафика мультимедиа с edge Server и Director предыдущей версии на Skype для бизнеса Server 2019 Edge Server для развертывания на одном сайте.
  
> [!IMPORTANT]
> Чтобы изменить маршрут федерации и маршрут трафика мультимедиа, необходимо запланировать время простоя обслуживания для Skype для бизнеса Server 2019 года и предыдущей версии Edge Servers. Весь процесс перехода также означает, что федеративный доступ будет отключен во время простоя. Следует запланировать простой на период, когда активность пользователей будет минимальной. Также следует реализовать своевременное уведомление конечных пользователей. Тщательно запланируйте простой и задайте соответствующие ожидания в организации. 
  
> [!IMPORTANT]
> Если устаревший edge Server настроен на использование того же FQDN для службы Access Edge, службы Веб-конференциации Edge и службы A/V Edge, процедуры в этом разделе не поддерживаются. Если устаревшие службы Edge настроены на использование одного и того же FQDN, необходимо сначала перенести всех пользователей, а затем выключить предыдущие версии Edge Server, прежде чем включить федерацию на Skype для бизнеса Server Edge Server 2019. 
  
> [!IMPORTANT]
> Если федерация XMPP проходит через Skype для бизнеса Server 2019, Пользователи в предыдущей версии не смогут общаться с федерадным партнером XMPP до тех пор, пока все пользователи не будут перемещены в Skype для бизнеса Server 2019 г., будут настроены политики и сертификаты XMPP, федератированный партнер XMPP настроен на Skype для бизнеса Server 2019 г., и, наконец, записи DNS были обновлены. 
  
## <a name="to-remove-the-legacy-federation-association-from-skype-for-business-server-2019-sites"></a>Удаление устаревшей ассоциации федерации с Skype для бизнеса Server 2019 г.

1. На переднем Skype для бизнеса Server 2019 откройте существующую топологию в Topology Builder. 
    
2. В левой области перейдите к узлу сайта, который расположен непосредственно ниже **Skype для бизнеса Server.**
    
3. Щелкните сайт правой кнопкой мыши и выберите команду **Изменить свойства**.
    
4. В левой области выберите **Федеративный маршрут**. 
    
5. В **соответствии с назначением** маршрута федерации сайта , очистить поле **Включить SIP федерации,** чтобы отключить маршрут федерации через устаревшую среду. 
  
6. Нажмите кнопку **ОК**, чтобы закрыть страницу "Изменение свойств". 
    
7. Из **Topology Builder** выберите верхний узел **Skype для бизнеса Server**.
    
8. В меню **Действие** выберите пункт **Опубликовать топологию**.
    
9. Нажмите **кнопку Далее,** чтобы завершить процесс публикации, а затем нажмите **кнопку Готово,** когда процесс публикации завершен. 
    
## <a name="to-configure-the-legacy-edge-server-as-a-non-federating-edge-server"></a>Настройка устаревшего пограничного сервера в качестве пограничного сервера без федерации

1. В левой области перейдите к устаревшему узлу установки, а затем к узлу **пулов Edge.** 
    
2. Щелкните правой кнопкой мыши пограничный сервер и выберите в контекстном меню пункт **Изменить свойства**.
    
3. Выберите **Общие** в левой панели. 
    
4. Очистка федеративы Включить для этого края **пула (порт 5061)** и выберите **ОК,** чтобы закрыть страницу. 
  
5. В меню **Действие** выберите пункт **Публикация топологии**, а затем щелкните **Далее**.
    
6. После завершения работы **мастера публикации**, нажмите кнопку **Готово**, чтобы закрыть мастер. 
    
7. Убедитесь, что федерация для устаревшего сервера Edge отключена в Topology Builder.
  
## <a name="to-configure-certificates-on-the-legacy-edge-server"></a>Настройка сертификатов на устаревшем edge Server

1. Экспорт внешнего сертификата прокси-доступа с закрытым ключом из устаревшего Edge Server. 
    
2. На Skype для бизнеса Server 2019 edge Server импортировать внешний сертификат Access Proxy с предыдущего шага.
    
3. Назначьте внешний сертификат Access Proxy внешнему Skype для бизнеса Server интерфейсу edge Server 2019.
    
4. Внутренний сертификат интерфейса Skype для бизнеса Server 2019 edge Server следует запрашивать у доверенного ca и назначенного. 
    
## <a name="to-change-the-previous-versions-federation-route-to-use-skype-for-business-server-2019-edge-server"></a>Изменение маршрута федерации предыдущей версии на использование Skype для бизнеса Server 2019 Edge Server

1. Из Topology Builder в левой области перейдите к узлу **Skype для бизнеса Server 2019,** а затем к узлу **пулов Edge.** 
    
2. Щелкните правой кнопкой мыши пограничный сервер и выберите в контекстном меню пункт **Изменить свойства**.
    
3. Выберите **Общие** в левой панели. 
    
4. Выберите поле "Включить федерацию" для этого пула **Edge (порт 5061)** и нажмите кнопку **ОК,** чтобы закрыть страницу. 
  
5. В меню **Действие** выберите пункт **Публикация топологии**, а затем щелкните **Далее**.
    
6. После завершения работы **мастера публикации**, нажмите кнопку **Готово**, чтобы закрыть мастер. 
    
7. **Убедитесь, что Federation (порт 5061)** установлен для **включения** в Topology Builder.
    
  
## <a name="to-update-skype-for-business-server-2019-edge-server-federation-next-hop"></a>Обновление федерации Skype для бизнеса Server 2019 edge Server в следующем переходе

1. Из Topology Builder в левой области перейдите к узлу **Skype для бизнеса Server 2019,** а затем к узлу **пулов Edge.** 
    
2. Разверните узел, щелкните правой кнопкой указанный пограничный сервер и выберите команду **Изменить свойства**. 
    
3. На странице **Общие,** в следующем **выборе хопа,** выберите из списка drop-down Skype для бизнеса Server 2019.
  
4. Нажмите кнопку **ОК**, чтобы закрыть страницу "Изменение свойств". 
    
5. Из **Topology Builder** выберите верхний узел **Skype для бизнеса Server**. 
    
6. В меню **Действие** выберите команду **Опубликовать топологию** и завершите работу мастера. 
    
## <a name="to-configure-skype-for-business-server-2019-edge-server-outbound-media-path"></a>Настройка пути Skype для бизнеса Server edge Server 2019

1. Из Topology Builder в левой области перейдите к узлу **Skype для бизнеса Server 2019** г., а затем к пулу ниже **выпуск Standard front End Servers** **или выпуск Enterprise** пулам переднего выпуск Enterprise.
    
2. Щелкните пул правой кнопкой мыши и выберите команду **Изменить свойства**.
    
3. В разделе **Связи** установите флажок **Связать пограничный пул (для компонентов мультимедиа)**. 
  
4. В выпадаемом окне выберите Skype для бизнеса Server 2019 Edge Server.
    
5. Нажмите кнопку **ОК**, чтобы закрыть страницу **изменения свойств**. 
    
## <a name="to-turn-on-skype-for-business-server-2019-edge-server-federation"></a>Включить федерацию Skype для бизнеса Server Edge Server 2019

1. Из Topology Builder в левой области перейдите к узлу **Skype для бизнеса Server 2019,** а затем к узлу **пулов Edge.** 
    
2. Разверните узел, щелкните правой кнопкой указанный пограничный сервер и выберите команду **Изменить свойства**. 
    
    > [!NOTE]
    > Федерация может быть включена только для одного пула Edge. Если у вас несколько пулов Edge, выберите один из них, который будет использовать в качестве федерательных пулов Edge. 
  
3. На общей **странице** убедитесь, что выбрано поле Enable federation для этого пула **Edge (Port 5061).** 
  
4. Нажмите кнопку **ОК**, чтобы закрыть страницу "Изменение свойств". 
    
5. Перейдите к узлу сайта. 
    
6. Щелкните сайт правой кнопкой мыши и выберите команду **Изменить свойства**.
    
7. В левой области, выберите **Федеративный маршрут**.
    
8. В **соответствии с назначением** маршрута федерации сайта выберите **включить федерацию SIP,** а затем из списка выберите Skype для бизнеса Server 2019 Edge Server. 
  
9. Нажмите кнопку **ОК**, чтобы закрыть страницу **Изменение свойств**. 
    
     В случае развертывании с несколькими сайтами выполните эту процедуру на каждом сайте. 
    
## <a name="to-publish-edge-server-configuration-changes"></a>Публикация изменений конфигурации пограничного сервера

1. Из **Topology Builder** выберите верхний узел **Skype для бизнеса Server**. 
    
2. В меню **Действие** выберите команду **Опубликовать топологию** и завершите работу мастера. 
    
3. Дождитесь выполнения репликации Active Directory во всех пулах в развертывании.
    
    > [!NOTE]
    > Вы можете увидеть следующее сообщение: **Предупреждение: топология содержит несколько федерательных edge Server. Это может произойти во время миграции в более недавнюю версию продукта. В этом случае для федерации будет активно использоваться только один edge Server. Убедитесь, что внешняя запись SRV DNS указывает на правильный edge Server. Если вы хотите развернуть несколько серверов федерации edge Server для одновременной активности (т. е. не сценарий миграции), убедитесь, что все федерадные партнеры используют Skype для бизнеса Server. Убедитесь, что во внешней записи DNS SRV перечислены все включенные edge Servers федерации федерации.** Появление этого предупреждения ожидаемое и его можно спокойно проигнорировать. 
  
## <a name="to-configure-skype-for-business-server-2019-edge-server"></a>Настройка Skype для бизнеса Server 2019 edge Server

1. Принесите все Skype для бизнеса Server 2019 edge Servers в Интернете. 
    
2. Обнови правила маршрутинга внешнего брандмауэра или параметры балансиры нагрузки оборудования, чтобы отправить трафик SIP для внешнего доступа (обычно порт 443) и федерации (обычно порт 5061) в Skype для бизнеса Server 2019 Edge Server, а не на устаревший edge Server.
    
    > [!NOTE]
    > Если у вас нет балансиры нагрузки оборудования, необходимо обновить запись DNS A для федерации, чтобы решить проблему с новым сервером Skype для бизнеса Server Access Edge. Чтобы выполнить это с минимальными нарушениями, уменьшите значение TLL для внешнего Skype для бизнеса Server Access Edge FQDN, чтобы при обновлении DNS указать на новый Skype для бизнеса Server Access Edge, федерация и удаленный доступ будут быстро обновляться. 
  
3. Остановите **Skype для бизнеса Server доступа с** каждого компьютера Edge Server. 
    
4. С каждого устаревшего компьютера Edge Server откройте applet **Services** из **администрирования.**
    
5. В списке служб найдите Skype для бизнеса Server **access Edge**.
    
6. Щелкните правой кнопкой имя служб, а затем выберите команду **Остановить**, чтобы остановить службу. 
    
7. Выберите тип запуска **Отключено**. 
    
8. Нажмите кнопку **ОК**, чтобы закрыть окно **Свойства**.
