# <a name="configure-federation-routes-and-media-traffic"></a>Настройка маршрутов федерации и трафика мультимедиа

Федерация — это отношение доверия между двумя или более доменами SIP, позволяющее пользователям в разных организациях взаимодействовать через границы сети. После того как вы перейдете на пилотный проект, вам нужно будет перейти с маршрута Федерации на пограничных серверах предыдущей версии на маршрут Федерации на сервере пограничного сервера Skype для бизнеса Server 2019.
  
Выполните указанные ниже действия, чтобы перенести маршрут Федерации и трафик мультимедиа с пограничного сервера и режиссера на сервер в Skype для бизнеса Server 2019 для развертывания с одним сайтом.
  
> [!IMPORTANT]
> Для изменения маршрута Федерации и маршрутизации трафика мультимедиа требуется запланировать время бездействия на обслуживание для серверов Skype для бизнеса Server 2019 и более ранних версий. Весь процесс перехода также означает, что федеративное Access будет недоступен в течение сбоя. Вы должны планировать время простоя, когда ожидается минимальная активность пользователей. Кроме того, вы должны предоставить вам достаточное уведомление для конечных пользователей. Спланируйте этот сбой и настройте соответствующие ожидания в Организации. 
  
> [!IMPORTANT]
> Если сервер старой версии пограничного сервера настроен на использование того же полного доменного имени для службы пограничного доступа, службы Edge для веб-конференций и службы EDGE, то процедуры, описанные в этом разделе, не поддерживаются. Если для устаревших служб пограничного приложения задано то же полное доменное имя, необходимо сначала перенести всех пользователей, а затем списать более ранние версии пограничного сервера, прежде чем включить федерацию на пограничный сервер Skype для бизнеса Server 2019. 
  
> [!IMPORTANT]
> Если КСМПП Федерация передается через пограничный сервер Skype для бизнеса Server 2019, пользователи из предыдущей версии не смогут общаться с федеративными партнерами КСМПП, пока все пользователи не будут перемещены в Skype для бизнеса Server 2019, КСМПП политики и сертификаты настроены: КСМПП федеративный партнер настроен на странице Skype для бизнеса Server 2019, а наконец, записи DNS были обновлены. 
  
## <a name="to-remove-the-legacy-federation-association-from-skype-for-business-server-2019-sites"></a>Удаление устаревшего сопоставления Федерации из сайта Skype для бизнеса Server 2019

1. На сервере переднего плана Skype для бизнеса Server 2019 откройте существующую топологию в построителе топологии. 
    
2. На левой панели перейдите к узлу сайта, расположенному непосредственно под **Skype для Business Server**.
    
3. Щелкните сайт правой кнопкой мыши и выберите команду **изменить свойства**.
    
4. На левой панели выберите **маршрут Федерации**. 
    
5. В разделе **назначение маршрута Федерации сайтов**снимите флажок **включить федерацию SIP** , чтобы отключить маршрут Федерации в устаревшей среде. 
  
6. Нажмите кнопку **ОК** , чтобы закрыть страницу Изменение свойств. 
    
7. В **построителе топологии**выберите верхний узел в **Skype для бизнеса Server**.
    
8. В меню **действия** выберите пункт **топология публикации**.
    
9. Нажмите кнопку **Далее** , чтобы завершить процесс публикации, а затем нажмите кнопку **Готово** после завершения процесса публикации. 
    
## <a name="to-configure-the-legacy-edge-server-as-a-non-federating-edge-server"></a>Настройка устаревшего пограничного сервера в качестве пограничного сервера без поддержки Федерации

1. На левой панели перейдите к старому узлу установки, а затем — узлу **группы Edges** . 
    
2. Щелкните правой кнопкой мыши пограничный сервер и выберите команду **изменить свойства**.
    
3. На левой панели выберите **Общие** . 
    
4. Снимите флажок **включить федерацию для этого пограничного пула (порт 5061)** и нажмите кнопку **ОК** , чтобы закрыть страницу. 
  
5. В меню **действие** выберите пункт **топология публикации**и нажмите кнопку **Далее**.
    
6. После завершения работы **мастера публикации** нажмите кнопку **Готово** , чтобы закрыть окно мастера. 
    
7. Убедитесь в том, что в построителе топологии отключена Федерация для устаревшего пограничного сервера.
  
## <a name="to-configure-certificates-on-the-legacy-edge-server"></a>Настройка сертификатов для устаревшего пограничного сервера

1. Экспортируйте сертификат прокси внешнего доступа с закрытым ключом с прежнего пограничного сервера. 
    
2. На пограничном сервере Skype для бизнеса Server 2019 и импортируйте внешний сертификат прокси-сервера доступа на предыдущем этапе.
    
3. Назначьте внешний сертификат прокси-сервера доступа внешнему интерфейсу Skype для бизнеса Server 2019 на пограничном сервере.
    
4. Сертификат внутренней сети Skype для Business Server 2019, пограничного сервера должен быть указан в центре сертификации и назначен. 
    
## <a name="to-change-the-previous-versions-federation-route-to-use-skype-for-business-server-2019-edge-server"></a>Изменение маршрута Федерации для предыдущей версии на использование Skype для Business Server 2019 Edge Server

1. Из построителя топологии в левой области перейдите на узел **Skype для бизнеса Server 2019** , а затем в узел **пулов Edge** . 
    
2. Щелкните правой кнопкой мыши пограничный сервер и выберите команду **изменить свойства**.
    
3. На левой панели выберите **Общие** . 
    
4. Установите флажок **включить федерацию для этого пограничного пула (порт 5061)**, а затем нажмите кнопку **ОК** , чтобы закрыть страницу. 
  
5. В меню **действие** выберите пункт **топология публикации**и нажмите кнопку **Далее**.
    
6. После завершения работы **мастера публикации** нажмите кнопку **Готово** , чтобы закрыть окно мастера. 
    
7. Убедитесь в том, что для **Федерации (порт 5061)** задано значение **Enabled** в построителе топологии.
    
  
## <a name="to-update-skype-for-business-server-2019-edge-server-federation-next-hop"></a>Обновление следующего прыжка для Федерации Skype для бизнеса Server 2019 (пограничного сервера)

1. Из построителя топологии в левой области перейдите на узел **Skype для бизнеса Server 2019** , а затем в узел **пулов Edge** . 
    
2. Разверните узел, щелкните правой кнопкой мыши пограничный сервер в списке и выберите команду **изменить свойства**. 
    
3. На странице **Общие** в разделе **следующий**Переход выберите пункт из раскрывающегося списка в группе Skype для бизнеса Server 2019.
  
4. Нажмите кнопку **ОК** , чтобы закрыть страницу Изменение свойств. 
    
5. В **построителе топологии**выберите верхний узел в **Skype для бизнеса Server**. 
    
6. В меню **действия** выберите команду **топология публикации** и завершите работу мастера. 
    
## <a name="to-configure-skype-for-business-server-2019-edge-server-outbound-media-path"></a>Настройка пути к исходящему носителю в Skype для бизнеса Server 2019 для пограничного сервера

1. Из построителя топологии в левой области перейдите на узел **Skype для бизнеса Server 2019** , а затем в пул ниже **стандартных внешних серверов выпуска** и пулов предварительной **версии Enterprise Edition**.
    
2. Щелкните пул правой кнопкой мыши и выберите команду **изменить свойства**.
    
3. В разделе " **связи** " установите флажок " **связать пул EDGE (для компонентов мультимедиа)** ". 
  
4. В раскрывающемся списке выберите пункт Пограничный сервер Skype для бизнеса Server 2019.
    
5. Нажмите кнопку **ОК** , чтобы закрыть страницу **изменение свойств** . 
    
## <a name="to-turn-on-skype-for-business-server-2019-edge-server-federation"></a>Включение поддержки интеграции Skype для бизнеса Server 2019 Server Edge

1. Из построителя топологии в левой области перейдите на узел **Skype для бизнеса Server 2019** , а затем в узел **пулов Edge** . 
    
2. Разверните узел, щелкните правой кнопкой мыши пограничный сервер в списке и выберите команду **изменить свойства**. 
    
    > [!NOTE]
    > Интеграция может быть включена только для одного пула пограничного сервера. Если у вас несколько пулов EDGE, выберите один из них, чтобы использовать его в качестве пула пограничного сервера. 
  
3. Убедитесь, что на странице " **Общие** " установлен флажок **включить федерацию для этого пула Edge (порт 5061)** . 
  
4. Нажмите кнопку **ОК** , чтобы закрыть страницу Изменение свойств. 
    
5. Перейдите к узлу сайта. 
    
6. Щелкните сайт правой кнопкой мыши и выберите команду **изменить свойства**.
    
7. В левой области выберите пункт **маршрут Федерации**.
    
8. В разделе **назначение маршрута Федерации сайтов**выберите **включить федерацию SIP**, а затем в списке выберите пункт Пограничный сервер Skype для бизнеса Server 2019. 
  
9. Нажмите кнопку **ОК** , чтобы закрыть страницу **изменение свойств** . 
    
     Для развертываний с несколькими сайтами выполните эту процедуру на каждом сайте. 
    
## <a name="to-publish-edge-server-configuration-changes"></a>Публикация изменений в конфигурации пограничного сервера

1. В **построителе топологии**выберите верхний узел в **Skype для бизнеса Server**. 
    
2. В меню **действия** выберите пункт **топология публикации** и завершите работу мастера. 
    
3. Дождитесь, когда репликация службы каталогов Active Directory будет выполняться для всех пулов в развертывании.
    
    > [!NOTE]
    > Может появиться следующее сообщение: **Предупреждение: топология состоит из нескольких федеративных пограничных серверов. Это может происходить при миграции в более позднюю версию продукта. В этом случае только один сервер пограничного сервера используется только для Федерации. Убедитесь, что внешняя SRV-запись DNS указывает на правильный пограничный сервер. Если вы хотите одновременно развернуть несколько пограничного сервера федерации (то есть не сценарий миграции), убедитесь, что все Федеративные партнеры используют Skype для бизнеса Server. Убедитесь, что внешняя SRV-запись содержит все пограничные серверы, поддерживающие Федерацию.** Это предупреждение ожидается, и его можно игнорировать. 
  
## <a name="to-configure-skype-for-business-server-2019-edge-server"></a>Настройка пограничного сервера Skype для бизнеса Server 2019

1. Переведите все серверы в Skype для бизнеса Server 2019 в Интернете. 
    
2. Обновите правила маршрутизации внешних брандмауэров или подсистемы балансировки нагрузки оборудования, чтобы отправлять трафик SIP для внешнего доступа (обычно это порт 443) и Федерацию (обычно это порт 5061) на пограничный сервер Skype для бизнеса Server 2019 вместо устаревшего пограничного сервера.
    
    > [!NOTE]
    > Если у вас нет аппаратной подсистемы балансировки нагрузки, вам нужно обновить запись DNS A для Федерации, чтобы устранить проблему с новым сервером пограничного доступа к Skype для бизнеса Server. Для выполнения этой задачи с минимальным пределами, уменьшите значение ТЛЛ для внешнего доменного имени для доступа в Skype для бизнеса Server, чтобы при обновлении DNS для указания на новый сервер доступа к Skype для бизнеса Server были быстро обновлены службы федерации и удаленного доступа. 
  
3. Остановите **доступ к серверу Skype для бизнеса Server** с каждого компьютера пограничного сервера. 
    
4. На каждом компьютере пограничного серверного сервера откройте приложение " **службы** " из **меню**"Администрирование".
    
5. В списке услуги найдите элемент **доступ к серверу Skype для бизнеса Server**.
    
6. Щелкните правой кнопкой мыши имя службы, а затем выберите команду **остановить** , чтобы остановить службу. 
    
7. Установите для типа запуска значение **отключено**. 
    
8. Нажмите кнопку **ОК** , чтобы закрыть окно " **свойства** ".
