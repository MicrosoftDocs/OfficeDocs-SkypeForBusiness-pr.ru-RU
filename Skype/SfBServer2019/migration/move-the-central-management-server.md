# <a name="move-the-legacy-central-management-server-to-skype-for-business-server-2019"></a>Перемещение устаревшего центрального сервера управления в Skype для бизнеса Server 2019 г.

После переноса в Skype для бизнеса Server 2019 г. и перед удалением устаревшего сервера необходимо переместить центральный сервер управления на Skype для бизнеса Server 2019 front End Server или пул. 
  
Центральный сервер управления — это единая система репликации, в которой копия базы данных для чтения и записи хранится на переднем конечном сервере, который содержит центральный сервер управления. Каждый компьютер в топологии, включая сервер переднего конечного сервера, содержащий центральный сервер управления, имеет только для чтения копию данных центра управления в базе данных SQL Server (по умолчанию именуемой RTCLOCAL), установленной на компьютере во время установки и развертывания. Локализованная база данных получает обновления реплики с помощью агента репликатора Skype для бизнеса Server репликатора, который выполняется в качестве службы на всех компьютерах. Имя фактической базы данных на центральном сервере управления и локальной реплики — XDS, которая состоит из файлов xds.mdf и xds.ldf. Расположение базы данных ссылается на пункт управления службами (SCP) в службах домена Active Directory. Все средства, которые используют Центральный сервер управления для управления и настройки Skype для бизнеса Server SCP, чтобы найти центральный магазин управления.
  
После успешного перемещений центрального сервера управления необходимо удалить базы данных Центрального сервера управления с исходного переднего сервера. Сведения об удалении баз данных центрального сервера управления см. в SQL Server базы данных для пула [переднего конца.](remove-the-sql-server-database-for-a-front-end-pool.md)
  
Вы используете Windows PowerShell cmdlet **Move-CsManagementServer** в оболочке управления Skype для бизнеса Server для перемещения базы данных из устаревшей базы данных SQL Server установки в базу данных Skype для бизнеса Server 2019 SQL Server, а затем обновить SCP, чтобы указать на расположение центрального сервера управления Skype для бизнеса Server 2019. 
  
Используйте процедуры в этом разделе для подготовки серверов Skype для бизнеса Server 2019 перед перемещением центрального сервера управления.
  
## <a name="to-prepare-an-enterprise-edition-front-end-pool"></a>Подготовка пула выпуск Enterprise переднего конца

1. В Skype для бизнеса Server 2019 выпуск Enterprise, где необходимо переместить центральный сервер управления, войдите на компьютер, на котором установлена Skype для бизнеса Server Management Shell в качестве члена группы **RTCUniversalServerAdmins.** Вы также должны SQL Server права и разрешения пользователей базы данных sysadmin в базе данных, где необходимо установить центральный магазин управления. 
    
2. Откройте оболочку Skype для бизнеса Server управления.
    
3. Чтобы создать новый центр управления в Skype для бизнеса Server 2019 SQL Server, в Skype для бизнеса Server management Shell введите:
    
   ```PowerShell
   Install-CsDatabase -CentralManagementDatabase -SQLServerFQDN <FQDN of your SQL Server> -SQLInstanceName <name of instance>
   ```

4. Подтвердим, что **состояние Skype для бизнеса Server-end-службы** **запущено.**
    
## <a name="to-prepare-a-standard-edition-front-end-server"></a>Подготовка выпуск Standard переднего end Server

1. На Skype для бизнеса Server 2019 выпуск Standard переднего конечного сервера, на котором необходимо переместить центральный сервер управления, войдите на компьютер, на котором установлена Skype для бизнеса Server Management Shell в качестве члена группы **RTCUniversalServerAdmins.** 
    
2. Откройте мастер Skype для бизнеса Server развертывания.
    
3. В мастере Skype для бизнеса Server развертывания нажмите **кнопку Подготовка первого выпуск Standard сервера**.
    
4. На странице **Выполнение команд** SQL Server Express как центральный сервер управления. Создаются необходимые правила брандмауэра. После завершения установки базы данных и предварительно устанавливаемого ПО нажмите кнопку **Готово**.
    
    > [!NOTE]
    > При первоначальной установке может пройти некоторое время, прежде чем на экране вывода результатов команд появятся какие-либо изменения. Это связано с установкой SQL Server Express. Если необходимо отслеживать установку базы данных, используйте для этого диспетчер задач. 
  
5. Чтобы создать новый центр управления на Skype для бизнеса Server 2019 выпуск Standard front End Server, в Skype для бизнеса Server управленческой оболочки введите: 
    
   ```PowerShell
   Install-CsDatabase -CentralManagementDatabase -SQLServerFQDN <FQDN of your Standard Edition Server> -SQLInstanceName <name of instance - RTC by default>
   ```

6. Подтвердим, что **состояние Skype для бизнеса Server-end-службы** **запущено.**
    
## <a name="to-move-the-legacy-installs-central-management-server-to-skype-for-business-server-2019"></a>Для перемещения устаревшего сервера устанавливается центральный сервер управления Skype для бизнеса Server 2019 г.

1. На Skype для бизнеса Server 2019 года, который будет центральным сервером управления, войдите на компьютер, на котором Skype для бизнеса Server Management Shell установлена в качестве члена группы **RTCUniversalServerAdmins.** Необходимо также иметь права и разрешения администратора базы данных SQL Server. 
    
2. Откройте Skype для бизнеса Server (запустите в качестве администратора).
    
3. В Skype для бизнеса Server управленческой оболочки введите: 
    
   ```PowerShell
   Enable-CsTopology
   ```

    > [!CAUTION]
    > Если `Enable-CsTopology` это не удалось, уладить проблему, не разрешаемую выполнению команды перед продолжением. Если **Enable-CsTopology** не будет успешной, перемещение будет неудачным и может оставить топологию в состоянии, в котором нет центра управления. 
  
4. В Skype для бизнеса Server 2019 front End Server или front End pool в Skype для бизнеса Server management Shell введите: 
    
   ```PowerShell
   Move-CsManagementServer
   ```

5. Skype для бизнеса Server В таблице управления отображаются серверы, хранилища файлов, хранилища баз данных и точки подключения к службе текущего состояния и предлагаемого состояния. Внимательно просмотрите сведения об исходном и целевом состоянии и убедитесь, что они правильны. Введите **Y**, чтобы продолжить, или **N**, чтобы остановить перемещение. 
    
6. Проверьте все предупреждения и ошибки, сгенерированные командой **Move-CsManagementServer**, и устраните их. 
    
7. На сервере Skype для бизнеса Server 2019 откройте мастер Skype для бизнеса Server развертывания. 
    
8. В Skype для бизнеса Server мастер развертывания щелкните Установите или обновите систему **Skype для бизнеса Server,** щелкните шаг **2:** Установка или удаление компонентов Skype для бизнеса Server, нажмите кнопку Далее, просмотрите сводку, а затем нажмите кнопку **Готово**. 
    
9. На устаревшем сервере установки откройте мастер развертывания. 
    
10. В Skype для бизнеса Server мастер развертывания щелкните Установите или обновите систему **Skype для бизнеса Server,** щелкните шаг **2:** Установка или удаление компонентов Skype для бизнеса Server, нажмите кнопку Далее, просмотрите сводку, а затем нажмите кнопку **Готово**. 
    
11. Перезагрузка сервера Skype для бизнеса Server 2019 г. Это необходимо из-за изменения членства в группе для доступа к базе данных Центрального сервера управления.
    
12. Чтобы подтвердить, что репликация с новым хранилищем централизованного управления происходит, в Skype для бизнеса Server управленческой оболочки введите: 
    
    ```PowerShell
    Get-CsManagementStoreReplicationStatus
    ```

    > [!NOTE]
    > Обновление всех текущих реплик может занять некоторое время. 
  
## <a name="to-remove-legacy-install-central-management-store-files-after-a-move"></a>Удаление устаревших файлов центрального магазина управления после перемещения

1. На устаревшем сервере установки войдите на компьютер, на котором Skype для бизнеса Server в качестве члена группы **RTCUniversalServerAdmins.** Необходимо также иметь права и разрешения администратора базы данных SQL Server. 
    
2. Open Skype для бизнеса Server Management Shell
    
    > [!CAUTION]
    > Не удаляйте файлы старой базы данных, пока репликация не завершится и не станет стабильной. Если вы удалите файлы до завершения репликации, вы нарушите процесс репликации и оставите недавно перенесенный Центральный сервер управления в неизвестном состоянии. Используйте командлет **Get-CsManagementStoreReplicationStatus** для подтверждения состояния репликации. 
  
3. Чтобы удалить файлы базы данных центрального магазина управления из устаревшей установки Центрального сервера управления, введите:
    
   ```PowerShell
   Uninstall-CsDatabase -CentralManagementDatabase -SqlServerFqdn <FQDN of SQL Server> -SqlInstanceName <Name of source server>
   ```

    Например,
    
   ```PowerShell
   Uninstall-CsDatabase -CentralManagementDatabase -SqlServerFqdn sql.contoso.net -SqlInstanceName rtc
   ```

    Где это либо устаревшая установка back End Server в развертывании выпуск Enterprise или _\<FQDN of SQL Server\>_ FQDN выпуск Standard сервера.
