# <a name="move-users-from-the-cloud-to-on-premises"></a>Перемещение пользователей из облака к локально 

При необходимости можно переместить пользователя, который был ранее перенести из локально в облако (ли для бизнеса в Интернет или только группы с помощью Скайп) обратно в локально. Для перемещения пользователей из любого из Скайп для бизнеса в Интернет или TeamsOnly режима обратно в развертывание локального Скайп для сервера, используйте командлет Move-CsUser или Скайп для панели управления Business Server, которые являются локальные средства. При перемещении пользователя обратно в локальном развертывании, необходимо решить, какой пул для перемещения пользователя.

> [!Important]
> Если пользователь был ранее в режиме TeamsOnly и с помощью более ранней версии, чем Скайп для 2015 Business Server с CU8, необходимо также удалить назначения TeamsOnly режиме TeamsUpgradePolicy для этого пользователя. Локальные пользователи не должны иметь mode = TeamsOnly.  Последующие версии Скайп для сервера автоматического удаления назначения. Для получения дополнительных сведений см [Grant-CsTeamsUpgradePolicy](https://docs.microsoft.com/en-us/powershell/module/skype/grant-csteamsupgradepolicy).

## <a name="prerequisites"></a>Необходимые компоненты

- Организации должны иметь Azure AD подключить правильность настройки и синхронизации все соответствующие атрибуты пользователя, как описано в статье [Настройка Azure AD подключение](configure-azure-ad-connect.md).
- Пользователь перемещается из сети к значениям при локальном должны уже существовать в локальной Active Directory.
- Скайп для гибридной среды Business должны быть настроены, как описано в [Настройка Скайп для гибридных бизнеса](configure-federation-with-skype-for-business-online.md).

## <a name="moving-users-back-to-on-premises"></a>Перемещение пользователей обратно на собственных

Один раз перемещении пользователя из облака обратно в локальную:

- Пользователь взаимодействует с вашей Скайп для развертывания Business Server для его функциональные возможности. 
- Все контакты, существовавшие в Скайп для бизнеса в Интернет переносятся обратно в Скайп для Business Server. На данный момент контакты, участвующие в группах не переносятся вернуться к локально.
- Если пользователь использует также группами, они не будет иметь возможность взаимодействовать с Скайп для бизнес-пользователей, а также они могут общаться с пользователями в федеративных организаций.
- Собрания в Скайп для бизнеса в Интернет, *не* автоматически переносятся вернуться к локально. Пользователи либо перенести свои собрания, или, если это необходимо, используйте [Средство миграции собрания](https://support.office.com/en-us/article/2b525fe6-ed0f-4331-b533-c31546fcf4d4).

### <a name="move-users-with-move-csuser"></a>Перемещение пользователей с Move-CsUser

Move-CsUser доступен из локальной Скайп для окна командной консоли управления Business PowerShell. Должно иметь полномочия в локальной среде как, так и клиента Office 365, как описано в [необходимые учетные данные администратора](move-users-between-on-premises-and-cloud.md#required-administrative-credentials). Можно либо использовать одной учетной записи, имеющей права в обеих сред, или можно запустить Скайп на месте для окна консоли Business Server с использованием учетных данных на месте и использовать `-Credential` параметр, чтобы указать учетные данные для Office 365 Учетная запись с необходимые административные роли Office 365.

Перемещение пользователя к локально с помощью Move-CsUser:

- Укажите пользователей для перемещения с помощью параметра Identity.
- Укажите параметр - целевой с полное доменное имя пула желаемую локальных, используемом для пользователя.
- Если у вас одну учетную запись с разрешениями, достаточными в обоих на локальную организацию и Office 365, используйте параметр - учетных данных для предоставления учетной записи с достаточные разрешения в Office 365.
- Если учетной записи с разрешениями в Office 365 не заканчивается «on.microsoft.com», необходимо указать параметр - HostedMigrationOverrideUrl с правильное значение, как описано в [необходимые административные учетные данные](move-users-between-on-premises-and-cloud.md#required-administrative-credentials).

Следующую последовательность командлет можно использовать для перемещения пользователя Скайп для Business Server и предполагается учетные данные Office 365 — это отдельную учетную запись и указано в качестве входных данных для строки Get-Credential.

```
$cred=Get-Credential
$url="https://admin1a.online.lync.com/HostedMigration/hostedmigrationService.svc"
Move-CsUser -Identity username@contoso.com -Target pool.corp.contoso.com -Credential $cred -HostedMigrationOverrideUrl $url
```

### <a name="move-users-with-the-skype-for-business-server-control-panel"></a>Перемещение пользователей с Скайп для панели управления Business Server

1. Откройте Скайп для элемента управления сервера Business панель приложения.
2. В левой панели навигации выберите **пользователей**.
3. Используйте **Поиск** для поиска пользователя, которого вы хотите вернуться к локально.
4. Выберите одного или нескольких и в раскрывающемся списке **Действие** над списком, выберите команду **переместить выбранных пользователей в локальную**.
5. В мастере выберите пула пользователя, в котором будет размещаться пользователя и нажмите кнопку **Далее**.
6. При появлении соответствующего запроса, вход в Office 365 с использованием учетной записи, заканчивающиеся на. onmicrosoft.com и имеет достаточные разрешения.
7. Нажмите кнопку **Далее**, а затем **Далее** еще раз для перемещения пользователя.
8. Обратите внимание на то, что сообщения о состоянии относительно успешное или неудачное предоставляются в верхней части главного приложения панель управления, не в мастере.

### <a name="removing-teamsonly-mode"></a>Удаление TeamsOnly режим

Если вы используете версию более ранних чем Скайп для бизнеса 2015 с CU8 и пользователь перемещается обратно на собственных в режиме TeamsOnly, необходимо удалить экземпляр UpgradeToTeams `TeamsUpgradePolicy` перед перемещением пользователя локально. Можно явно предоставить политику с помощью другого режима или просто удалите существующие назначения политики для этого пользователя использовать глобальную политику (поскольку глобальную политику вашего клиента не UpgradeToTeams).

Чтобы удалить назначение TeamsUpgradePolicy, выполните следующий командлет из Скайп для окна Online PowerShell бизнеса:

`Grant-CsTeamsUpgradePolicy -Identity $user -PolicyName $null`

Кроме того, чтобы назначить другого экземпляра TeamsUpgradePolicy, который не имеет режим = TeamsOnly, можно указать имя нужного экземпляра в качестве значения параметра PolicyName в командлет. Чтобы просмотреть список доступных экземпляров TeamsUpgradePolicy, запустите Get-CsTeamsUpgradePolicy.
