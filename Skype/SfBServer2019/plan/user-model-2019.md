## <a name="front-end-server"></a>переднего плана

> [!NOTE]
> Расширенные пулы для этой роли сервера не поддерживаются.

В пуле переднего плана должен иметь один сервер переднего плана для каждого 6660 пользователями, размещенными в пуле, при условии, что технологии hyper-threading включена на всех серверах в пуле, используются для SQL Server Express Edition, который серверное оборудование соответствует рекомендациям в разделе [требования к серверу для Скайп для Business Server 2019](system-requirements.md). Максимальное количество пользователей в одном пуле переднего плана еще раз — 106,000, если предполагается, что технологии hyper-threading включено, используется SQL Server Express Edition на всех серверах в пуле. При наличии более чем 106,000 пользователей на сайте, можно развернуть несколько пулов переднего плана.

Когда счета для числа пользователей в пул переднего плана, включить всех пользователей, размещенных на устройств и серверов для обеспечения связи в филиалах, связанных с этой пула переднего плана.

When an active server is unavailable, its connections are transferred automatically to the other servers in the pool. В сценарии, когда у вас есть 30 000 пользователей и пять серверов переднего плана Если один сервер недоступен, подключений 6000 пользователей должны быть перенесены на другие четыре остальные серверы. These four remaining servers will then each have 7500 users, which is a larger number than recommended.

Если вместо этого разработку шести серверов переднего плана для 30 000 пользователей и один становится недоступным, всего 5000 пользователей при необходимости переместить оставшиеся пяти серверов. These five remaining servers will then each host 6000 users, which is in the recommended range.

Максимальное количество пользователей в пул переднего плана — 106,000. Максимальное число серверов переднего плана в пуле, равно 16.

Для пула переднего плана с 80 000 пользователей 16 серверов переднего плана будет хорошей производительности в типичного развертываниях, следуйте [моделей пользователя в Скайп для Business Server](../../SfbServer/plan-your-deployment/capacity/user-models.md). Развертывания, предназначенные для поддержки аварийного восстановления резервной предполагается, что не более 53,000 пользователи могут размещаться в каждой из двух связанных интерфейсных пулах, в которых каждый пул имеет достаточного количества серверов переднего плана для пользователей в обоих пулах один пул необходимости перехода на другой ресурс t o другое.

Число пользователей, поддерживаемых с хорошей производительности с конкретного пула переднего плана может отличаться от этих значений по следующим причинам:

- Оборудование для вашего сервера переднего плана не соответствует рекомендациям.
- Вместо использования SQL Server Express Edition, использование другого выпуска SQL Server, может иметь возможность размещения дополнительных пользователей в каждом пуле переднего плана.
- Использования в организации отличается от пользовательских моделей, например, если у вас больше трафика конференц-связи.

## <a name="conferencing-maximums"></a>Максимумы конференц-связи

Учитывая пользовательская модель 5% пользователей в пуле может использоваться в конференции в любой момент времени, пул 106,000 пользователи могут иметь около 5300 пользователей во время конференций одновременно. These conferences are expected to be a mix of media (some IM-only, some IM with audio, some audio/video, for example) and number of participants. There isn't a hard limit for the actual number of conferences allowed, and actual usage determines the actual performance. Например, если ваша организация многие дополнительные смешанным режимом конференции не предполагается, что в пользовательской модели, может потребоваться развернуть несколько серверов переднего плана или A аудио- и видео конференций, чем рекомендации, используемых в этой статье. Для получения дополнительных сведений о допущения в пользовательской модели видеть [моделей пользователя в Скайп для Business Server](../../SfbServer/plan-your-deployment/capacity/user-models.md).

Размер максимальный поддерживаемый конференции, размещенного в регулярных Скайп для пула переднего плана Business Server, который также содержит данные пользователей составляет 250 участников. While a 250-user conference is happening, the pool still supports other conferences as well, such that a total of 5% of pool users are in concurrent conferences. К примеру в пуле 16 серверов переднего плана и 106,000 пользователей, во время конференции 250 пользователей сведений о событиях, Скайп для Business Server поддерживает 5,050 других пользователей, участвующих в конференции меньшего размера.

Вне зависимости от числа пользователей, размещенных на пул переднего плана или сервера Standard Edition, Скайп для Business Server поддерживает как минимум 125 пользователи, участвующие в небольших конференции на тот же пул или сервер, который размещается конференции 250 пользователей.

Чтобы включить конференции, в которых от 1000 до 250 пользователей, можно настроить отдельный пул переднего плана только для размещения эти конференции. Этот пул переднего плана не будут размещаться все пользователи. Дополнительные сведения можно найти, [Планирование больших собраний в Скайп для Business Server](../../SfbServer/plan-your-deployment/conferencing/large-meetings.md).

Если вашей организации с большим смешанный режим конференции не предполагается, что в пользовательской модели, может потребоваться развернуть несколько серверов переднего плана, чем мы рекомендации в этом документе (максимум до 16 серверов переднего плана). Для получения дополнительных сведений о допущения в пользовательской модели видеть [моделей пользователя в Скайп для Business Server](../../SfbServer/plan-your-deployment/capacity/user-models.md).

## <a name="edge-server"></a>пограничный сервер

> [!NOTE]
> Расширенные пулы для этой роли сервера не поддерживаются.

Следует развернуть один пограничного сервера для каждого 18 000 удаленных пользователей, которые получат доступ к сайту одновременно. По крайней мере мы рекомендуем два пограничных серверов для обеспечения высокой доступности. Эти рекомендациях предполагается, что оборудование для пограничных серверов соответствует рекомендациям, приведенным в [Аппаратные серверные платформы](https://technet.microsoft.com/library/c964c1c0-0153-472b-88ad-a38866e0df0c.aspx).

> [!NOTE]
> Для повышения эффективности A / V Edge конференц-связи службы на пограничных серверах, следует включить стороне приема масштабирования (RSS) на сетевых адаптеров на пограничных серверах. RSS enables incoming packets to be handled in parallel by multiple processors on the server. Для получения дополнительных сведений проверьте [Получать со стороны масштабирование (RSS) в Windows Server 2012](https://go.microsoft.com/fwlink/p/?linkId=268731). For details about how to enable RSS, you'll need to refer to your network adapter documentation.

> [!NOTE]
> В целях повышения производительности сервера-посредника можно включить стороне приема масштабирования (RSS) на сетевых адаптеров на серверы-посредники. RSS enables incoming packets to be handled in parallel by multiple processors on the server. Дополнительные сведения см «[Масштабирование на стороне приема в Windows Server 2012](https://go.microsoft.com/fwlink/p/?linkId=268731)». For details about how to enable RSS, you'll need to refer to your network adapter documentation.

## <a name="video-interop-server-capacity"></a>Видео мощность сервера взаимодействия

При развертывании сервера взаимодействия видео, необходимо определить емкость, вы откроете максимальное число систем телеконференций видео (VTCs), который будет в одновременных звонков. For example, if you have 250 VTCs in your organization and your user model estimates that at most, 20% of them might be in concurrent calls, you base your capacity planning on 50 concurrent VTCs.
