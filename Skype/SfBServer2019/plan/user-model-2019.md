## <a name="front-end-server"></a>переднего плана

> [!NOTE]
> Расширенные пулы для этой роли сервера не поддерживаются.

В пуле переднего плана вы должны иметь один сервер переднего плана для всех пользователей 6 660, размещенных в пуле, предполагая, что технология Hyper-Threading включена на всех серверах в пуле, для которых используется SQL Server, Экспресс-выпуск и что оборудование сервера соответствует рекомендациям в [серверных требованиях для Skype для бизнеса Server 2019](system-requirements.md). Максимальное количество пользователей в одном пуле переднего плана — 106 000, при этом предполагается, что технология Hyper-Threading включена и SQL Server Express Edition используется на всех серверах в пуле. Если у вас более 106 000 пользователей на сайте, вы можете развернуть более одного пула переднего плана.

Если вы учитываете количество пользователей в пуле переднего плана, включите пользователей, которые размещаются на устройствах с бесперебойной подразделением и бесперебойно работающих серверах филиалов в филиалах, связанных с этим пулом переднего плана.

When an active server is unavailable, its connections are transferred automatically to the other servers in the pool. В сценарии, в котором есть пользователи 30 000 и пять серверов переднего плана, если один сервер недоступен, подключение 6000 ваших пользователей должно быть передано на другие четыре оставшихся сервера. These four remaining servers will then each have 7500 users, which is a larger number than recommended.

Если вместо этого вы начали пользоваться шестью серверами переднего плана для пользователей 30 000 и они становятся недоступными, общее количество пользователей 5000 нужно перейти на оставшиеся пять серверов. These five remaining servers will then each host 6000 users, which is in the recommended range.

Максимальное количество пользователей в пуле переднего плана — 106 000. Максимальное число серверов переднего плана в пуле составляет 16.

Для пула переднего плана с пользователями 80 000 пользователи из 16 интерфейсов будут хорошо подходящими для повышения производительности, в типичных развертываниях, которые следуют за [пользовательскими моделями в Skype для бизнеса Server](../../SfbServer/plan-your-deployment/capacity/user-models.md). В развертываниях, разработанных для поддержки перехода на аварийный сбой, предполагается, что в каждом из двух подключенных интерфейсных пулов может быть размещено не более 53 000 пользователей, в котором каждый из них должен иметь достаточное количество серверов переднего плана o другой.

Количество пользователей, которые поддерживают хорошую производительность с помощью определенного пула переднего плана, может отличаться от приведенных ниже.

- Оборудование для серверов переднего плана не соответствует рекомендациям.
- Вместо использования SQL Server, Экспресс Edition вы используете другой выпуск SQL Server, вы можете размещать дополнительных пользователей в каждом пуле интерфейсов.
- Использование вашей организации сильно отличается от пользовательских моделей, например, если у вас больше трафика для проведения конференций.

## <a name="conferencing-maximums"></a>Максимумы конференц-связи

Если пользовательская модель, в которой 5% пользователей в пуле, может находиться в одной конференции, группа пользователей 106 000 может иметь доступ одновременно к 5 300 пользователям на конференциях. These conferences are expected to be a mix of media (some IM-only, some IM with audio, some audio/video, for example) and number of participants. There isn't a hard limit for the actual number of conferences allowed, and actual usage determines the actual performance. Например, если ваша организация имеет множество конференций смешанного режима, чем предполагается для пользовательской модели, возможно, потребуется развернуть другие серверы переднего плана или серверы конференц-связи по сравнению с рекомендациями, приведенными в этой статье. Подробные сведения о допущениях в пользовательской модели можно найти в разделе [модели пользователей в Skype для бизнеса Server](../../SfbServer/plan-your-deployment/capacity/user-models.md).

Максимально допустимый размер Конференции, размещенный на обычном пуле переднего плана в Skype для бизнеса Server, где также размещены пользователи, — 250 участников. While a 250-user conference is happening, the pool still supports other conferences as well, such that a total of 5% of pool users are in concurrent conferences. Например, в пуле 16 серверов переднего плана и пользователей 106 000, в то время как выполняется конференция 250-User, Skype для бизнеса Server поддерживает 5 050 других пользователей, участвующих в небольших конференциях.

Независимо от количества пользователей, подключенного к внешнему интерфейсу или серверу Standard Edition, в Skype для бизнеса Server поддерживается не менее 125 других пользователей, участвующих в небольших конференциях в том же пуле или на сервере, где размещается конференция 250-пользователи.

Чтобы включить Конференц-связь между пользователями 250 и 1000, вы можете настроить отдельный пул переднего плана только для того, чтобы разместить эти конференции. Этот пул переднего плана не будет содержать пользователей. Подробные сведения можно найти [в разделе Планирование собраний на больших собраниях в Skype для бизнеса Server](../../SfbServer/plan-your-deployment/conferencing/large-meetings.md).

Если в вашей организации есть гораздо больше конференций смешанного режима, чем предполагается для пользовательской модели, вам может потребоваться развернуть другие серверы переднего плана, чем рекомендовано в этом документе (до 16 внешних серверов). Подробные сведения о допущениях в пользовательской модели можно найти в разделе [модели пользователей в Skype для бизнеса Server](../../SfbServer/plan-your-deployment/capacity/user-models.md).

## <a name="edge-server"></a>пограничный сервер

> [!NOTE]
> Расширенные пулы для этой роли сервера не поддерживаются.

Вы должны развернуть один пограничный сервер для каждого удаленного пользователя 18 000, который будет одновременно получать доступ к сайту. Как минимум, мы рекомендуем использовать два пограничных сервера для обеспечения высокой доступности. Эти рекомендации предполагают, что оборудование пограничных серверов соответствует рекомендациям, приведенным в разделе [аппаратные платформы сервера](https://technet.microsoft.com/library/c964c1c0-0153-472b-88ad-a38866e0df0c.aspx).

При указании числа пользователей для пограничных серверов включите пользователей, относящихся к бесперебойным устройствам филиалов и к бесперебойным серверам филиалов в филиалах, которые связаны с пулом переднего плана на этом сайте.

> [!NOTE]
> Чтобы повысить производительность службы пограничного конференц-связи A/V на пограничных серверах, необходимо включить масштабирование на стороне приема (RSS) на сетевых адаптерах пограничного сервера. RSS поддерживает параллельную обработку входящих пакетов несколькими процессорами сервера. Подробнее об этом можно узнать [в Windows Server 2012 с масштабированием на стороне получения (RSS)](https://go.microsoft.com/fwlink/p/?linkId=268731). For details about how to enable RSS, you'll need to refer to your network adapter documentation.

## <a name="video-interop-server-capacity"></a>Емкость сервера видеовзаимодействия

Если вы разработали сервер видеосвязи и вам нужно определить емкость, вы можете узнать о максимальном количестве систем для проведения видеоконференций (Вткс), которые будут находиться в одновременных звонках. For example, if you have 250 VTCs in your organization and your user model estimates that at most, 20% of them might be in concurrent calls, you base your capacity planning on 50 concurrent VTCs.
