# <a name="exchange-unified-messaging-online-migration-support"></a>Поддержка сетевой единой системы обмена сообщениями Exchange

> [!IMPORTANT]
> **По состоянию на 28 февраля 2020 г. в 17:00 по тихоокеанскому времени служба единой системы обмена сообщениями в Exchange Online не поддерживается. Все учетные записи голосовой почты были перенесены облачная голосовая почта службу корпорацией Майкрософт. Весь оставшийся трафик автосекретаря не отслеживается и может быть нарушен в любое время.**

В связи с объявлением от 8 февраля 2019 г. корпорация Майкрософт прекращает работу службы Exchange Unified Messaging Online (ExchUMO) до 28 февраля 2020 г.[](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) В этой статье содержится сводка по затронутым клиентам, которые должны знать и делать для планирования непрерывности бизнес-процессов.

ExchUMO развертывается клиентами для голосовой почты, автосекретаря, очереди вызовов и служб интеграции факсов. Корпорация Майкрософт планирует помочь клиентам перейти на телефонная система службы, которые уже поддерживают тысячи клиентов Skype для бизнеса Online и Microsoft Teams.

Голосовая почта — это в основном миграция, управляемая корпорацией Майкрософт. Для подмножества клиентов может потребоваться участие администратора и (или) инвестиции. Автосекретарь — это миграция, управляемая администратором. Вам потребуется повторно создать существующие деревья автосекретаря ExchUMO в облачной облачной службе автосекретаря. Клиенты, использующие какие-либо функции ExchUMO со сторонней УАТС, не будут перенесены в облачные службы Skype, так как они не поддерживают сторонние системы УАТС. В этом блоге был объявлен план прекращения поддержки сторонних поставщиков [, и](https://techcommunity.microsoft.com/t5/Exchange-Team-Blog/New-date-for-discontinuation-of-support-for-Session-Border/ba-p/607853) клиенты в этой модели развертывания могут перенести своих пользователей на одну из платформ или служб Единой коммуникации Майкрософт или получить стороннее решение голосовой почты и (или) автосекретаря для этих пользователей. Интеграция факсов не поддерживается в облачных службах. клиентам потребуется перейти на стороннее решение.

## <a name="who-is-affected"></a>Кто затронуты?

Клиенты, использующие любые из следующих функций Exchange единой службы обмена сообщениями Online, затронуты:

- Служба голосовой почты
- Служба автосекретаря
- Служба очереди вызовов
- Интеграция факсов

> [!Note]
> Клиенты, использующие какой-либо Exchange Server локально с единой системой обмена сообщениями, не затрагиваются.

Узнайте больше о влиянии взаимодействия пользователя и администратора на [взаимодействие с пользователем](#user-experience-impact).

## <a name="voicemail-migration-guidelines"></a>Рекомендации по миграции голосовой почты

### <a name="get-informed"></a>Получение сведений

Ознакомьтесь с [объявлением блога](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) и этой статьей, чтобы спланировать плавную миграцию для пользователей. [Дополнительные сведения о возможностях](https://support.office.com/article/check-skype-for-business-voicemail-and-options-2deea7f8-831f-4e85-a0d4-b34da55945a8) Skype для бизнеса голосовой почты см. в разделе телефонная система Skype для бизнеса и параметрах голосовой почты.  

### <a name="establish-a-skype-for-business-hybrid-topology"></a>Создание Skype для бизнеса гибридной топологии

Если у вас нет Skype для бизнеса гибридной топологии, необходимо сделать это, чтобы обеспечить бесперебойную миграцию пользователей голосовой почты. [Дополнительные сведения см. в Skype для бизнеса configure Skype для бизнеса](../../SfbHybrid/hybrid/configure-federation-with-skype-for-business-online.md) hybrid.

> [!Note]
> Для миграции службы голосовой почты не нужно переносить пользователей в Сети. Однако, чтобы локальные пользователи могли использовать телефонная система голосовой почты, необходимо установить гибридную топологию.

### <a name="plan-your-auto-attendant-migration"></a>Планирование миграции автосекретаря

Администраторы могут начать миграцию автосекретарей из ExchUMO в облачный автосекретарь в любое время. Дополнительные сведения см. в разделе "Настройка [облачного автосекретаря](/microsoftteams/create-a-phone-system-auto-attendant) ".

Корпорация Майкрософт продолжает предоставлять дополнительные возможности автосекретаря, которые клиенты могут считать необходимыми для миграции. Администраторы должны оценить набор функций и соответствующим образом перенести свои экземпляры автосекретаря. Сравнение списков функций см. в таблице функций [ExchUMO](#exchumo-and-azure-cloud-based-services-feature-matrix) и облачных служб Azure.

### <a name="plan-for-your-voicemail-post-migration-validation-and-testing"></a>Планирование проверки и тестирования голосовой почты после миграции

Миграция голосовой почты выполняется корпорацией Майкрософт. Администраторам не требуется выполнять какие-либо действия, учитывая, что установлена необходимая гибридная топология. Корпорация Майкрософт выполняет необходимую проверку и тестирование, чтобы убедиться, что миграция голосовой почты пользователей не нарушена. Администраторам рекомендуется выполнять тестирование и проверку на их стороне. [Рекомендуемый план тестирования и](#suggested-test-plan-and-post-migration-validation-for-admins) проверка после миграции для администраторов см. в разделе "Рекомендуемый план тестирования" и "Проверка после миграции".

> [!Note]
> Lync Server 2010 не поддерживается. При развертывании сервера 2010 следует спланировать обновление сервера или перенести пользователей в Microsoft Teams.  

### <a name="monitor-the-admin-notification-center"></a>Мониторинг центра Администратор уведомлений

Обратите внимание на уведомление в Администратор уведомлений с дополнительными сведениями и временной шкалой, касающейся миграции пользователей. Уведомления отправляются не менее чем за 30 дней до периода миграции.

> [!Note]
> Если вы получили уведомление с временной шкалой миграции пользователей и хотите отложить миграцию по критически важной для бизнеса причине, это можно сделать, обратитесь к служба поддержки Майкрософт. Вы не можете отложить миграцию после даты прекращения использования 28 февраля 2020 г. Для клиентов, у которых могут возникнуть дополнительные вопросы, обратитесь в службу поддержки вашей учетной записи или служба поддержки Майкрософт. Клиенты, Microsoft 365 или Office 365 могут отправить запрос в службу поддержки через Центр администрирования Microsoft 365.

### <a name="consider-opting-in-for-a-planned-migration"></a>Рассмотрите возможность использования плановой миграции

Вы можете согласиться на плановую миграцию службы голосовой почты в CVM. Прежде чем принять участие, ознакомьтесь с подробными сведениями, приведенными в этой статье, особенно в следующих разделах:

- Этапы миграции (в этом разделе)
- Таблица функций exchUMO и облачных служб Azure
- Влияние на взаимодействие с пользователем

При выборе управляемой миграции вы не получите уведомление за 30 дней до миграции в центре сообщений Microsoft 365 администрирования.

Чтобы согласиться на плановую миграцию, отправьте по электронной почте запрос с адреса электронной почты администратора на адрес [cvm@microsoft.com со](mailto:cvm@microsoft.com) следующими сведениями:

- Предпочтительные даты (вторники): миграционные волны выполняются каждый вторник. Выберите дату во вторник, которая не выходит за пределы 3.03.2019.
 
- Идентификатор клиента: 32 символа в этом формате 0046728c-688a-4472-a38f-098fec60ac6x. Идентификатор клиента можно найти на портале администрирования Microsoft 365 в Azure AD или с помощью следующего командлета PowerShell:`Get-CsTenant | Select ObjectId`

После успешной миграции клиента вы получите подтверждение по электронной почте.

## <a name="auto-attendant-migration-guidelines"></a>Рекомендации по миграции автосекретаря

Microsoft 365 и Office 365 организации должны повторно создать автосекретарей Exchange UM Online в службе облачного автосекретаря Майкрософт и переключить на них локальные номера телефонов до даты прекращения использования службы Exchange UMO 28 февраля 2020 г. Это рекомендуемая рекомендация для успешной миграции и тестирования новых облачных автосекретарей. При наличии большого количества автосекретарей можно использовать сценарии миграции Exchange единой [системы](https://github.com/NathanJBennett/ExUMAAMigrationToCloudAA) обмена сообщениями в облачный автосекретарь, чтобы упростить массовую миграцию автосекретарей.

### <a name="auto-attendant-setup"></a>Настройка автосекретаря

Мы настоятельно рекомендуем начать настройку новых автосекретарей заранее, чтобы избежать проблем с последней минутой и ознакомиться с функциями и возможностями облачной службы автосекретаря. Для автосекретарей, для которых требуется одна или несколько функций зазора, можно создать и протестировать автосекретарей, когда функции зазора доступны для подготовки к развертыванию. Дополнительные сведения о функциях зазора см. в [приложении](#appendix).

1. Используйте командлеты Exchange UMO для экспорта конфигурации существующих автосекретарей с помощью [Get-UMAutoAttendant](/powershell/module/exchange/unified-messaging/get-umautoattendant).  

2. Используйте [командлет Export-UMprompt](/powershell/module/exchange/unified-messaging/export-umprompt) в Exchange Online PowerShell, чтобы экспортировать файлы мультимедиа приветствия (если они используются) и преобразовать их в .mp3 формате.

3. Следуйте инструкциям в разделе [](../../SfbHybrid/hybrid/plan-cloud-auto-attendant.md) "Планирование автосекретарей облака" и "Настройка облачного автосекретаря" для создания автосекретарей с помощью Microsoft Teams центра администрирования или PowerShell.[](/microsoftteams/create-a-phone-system-auto-attendant)

4. Просмотрите приветствия, если параметры меню изменились.

5. Настройте передачу данных в группы ответов с помощью обходного решения "Автосекретарь перенаправления вызовов в ТСОП" в разделе "Известные проблемы" этой статьи.[](#known-issues)  

6. Протестируйте новых автосекретарей, вызвав их внутри организации или назначив тестовый номер телефона.  

### <a name="cutover"></a>Прямая

1. Переключение телефонных номеров Exchange автосекретарей UMO на новых автосекретарей.
2. Переместите URI SIP из объекта контакта в учетную запись ресурса.
3. Проверьте и проверьте автосекретарей, используя только что назначенные номера телефонов.

### <a name="suggested-test-plan-and-post-migration-validation-for-admins"></a>Предлагаемый план тестирования и проверка после миграции для администраторов

Чтобы убедиться, что пользователи были перенесены в облачная голосовая почта, оставьте голосовую почту пользователю и проверьте текст сообщения в Outlook. облачная голосовая почта сообщения имеют нижний колонтитул, который считывает:

"Спасибо, что используете транскрибирование! Если вы не видите расшифровку выше, это связано с тем, что качество звука было недостаточно четким для транскрибирования".

При тестировании функций голосовой почты после миграции пользователей обязательно рассмотрите следующие сценарии:

- Проверьте доступ голосовой почты ко всем типам конечных точек в организации, например приложениям и IP-телефонам.
- Проверьте у образцов пользователей, что настроенные персонализированные приветствия воспроизводится для вызывающих абонентов.
- Если в вашей организации есть юридические требования или требования к соответствию требованиям для отключения транскрибирования для пользователей, убедитесь, что она отключена после миграции. Дополнительные сведения см. в [разделе "Настройка облачная голосовая почта](/microsoftteams/set-up-phone-system-voicemail)".
- Если вы ранее настроите Exchange виртуальных машин, убедитесь, что они эффективны.
- Ознакомьтесь с командлетами PowerShell облачная голосовая почта службы для изменения параметров пользователя.

### <a name="known-issues"></a>Известные проблемы

#### <a name="greeting-inconsistencies"></a>Несоответствия приветствия

Доступ подписчика может продолжать работать для вашего клиента до полного прекращения поддержки службы, даже если все пользователи были перенесены в облачная голосовая почта. Чтобы избежать путаницы и несогласованности взаимодействия с пользователем, отключите доступ подписчика после изменения приветствия после миграции. Для этого удалите контакт EXUM для каждой строки доступа подписчика с помощью `Get-CsExUmContact | ?{$_.IsSubscriberAccess -eq $true} | Remove-CsExUmContact`.

#### <a name="auto-attendant-call-transfer-to-pstn"></a>Передача вызовов автосекретаря в ТСОП

Чтобы передать вызов автосекретаря на внешний номер телефона ТСОП через службу Skype для бизнеса Server или службу группы ответа (RGS) в Skype для бизнеса Server, создайте локального пользователя с переадресаемой звонками на номер телефона ТСОП или номер телефона RGS. Пользователь должен быть включен и правильно настроен для Корпоративная голосовая связь и иметь назначенную политику голосовой связи.

#### <a name="shared-mailbox-is-still-accessible"></a>Общий почтовый ящик по-прежнему доступен

Общий почтовый ящик, настроенный с помощью Exchange UM Online, продолжает получать сообщения после миграции в CVM и быть доступен пользователям через Outlook. Однако доступ к изменению приветствия этих почтовых ящиков будет недоступен после переноса в CVM. Клиенты с общими почтовыми ящиками, которые используются для записи вызывающих автосекретарей, должны использовать возможности автосекретарей и общих почтовых ящиков очередей звонков после выпуска (ETA, октябрь 2019 г.).
  
#### <a name="username-is-not-using-skype-for-business-banner-displays"></a>Отображается баннер "Username is not using Skype для бизнеса" (Имя пользователя не использует Skype для бизнеса)

Служба CVM основана на инфраструктуре Microsoft Teams, и вызовы клиента Skype для бизнеса могут привести к отображению информационного баннера на клиенте, который считывает: "Имя пользователя не использует Skype для бизнеса. Чтобы получить более широкие возможности, перейдите Teams или начните собрание Skype собрания".
Обязательно обновите клиент Skype для бизнеса пользователя до последнего обновления клиента C2R, чтобы предотвратить отображение этого баннера.
  
#### <a name="set-up-voice-mail-takes-you-to-owa"></a>"Настройка голосовой почты" откроет OWA

Если нажать кнопку "Настроить голосовую почту" от клиента, Skype для бизнеса Server 2015/2013 будут перенаться на страницу портала Office Web Access (OWA) после миграции в CVM. Все параметры были удалены с вкладки "Голосовая почта" в OWA, и отобразится баннер со ссылкой на перенаправление для перенаправления пользователей на портал параметров пользователя CVM.

#### <a name="changing-greeting-remotely"></a>Удаленное изменение приветствия

Доступ подписчика ТСОП не поддерживается в CVM. Для пользователей, которым необходимо удаленно изменить приветствие, в службу голосовой почты IVR для мобильных клиентов добавлен пункт меню "Изменить приветствие". Пользователи могут вызвать эту службу, нажав и удерживая клавишу "1" на панели набора номера мобильного клиента.
