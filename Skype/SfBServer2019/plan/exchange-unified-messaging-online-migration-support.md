# <a name="exchange-unified-messaging-online-migration-support"></a>Поддержка сетевой единой системы обмена сообщениями Exchange

В ссылке на [объявление](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) 8 февраля 2019 служба единой системы обмена сообщениями Exchange в Интернете (ексчумо) не входит в состав 2020 февраля. В этой статье рассказывается о том, какие уязвимые пользователи должны знать и как планировать бесперебойную деятельность своей компании. 
 
Ексчумо разворачивается пользователями для голосовой почты, автосекретаря и (или) служб интеграции факса. Корпорация Майкрософт планирует помочь этим клиентам перейти на облачные службы, поддерживающие тысячи пользователей, которые уже есть в Skype для бизнеса Online и Microsoft Teams. 

Голосовая почта — это преимущественно процесс миграции с помощью Microsoft. для подмножества пользователей может потребоваться участие администратора и/или инвестиция. Автоматический секретарь — это миграция, управляемая администратором; Вам потребуется повторно создать существующие деревья автосекретаря Ексчумо в облачной службе облачного секретаря. Пользователи, которые используют любую из функций Ексчумо с сторонней АТС, не будут перенесены в облачные службы Skype, так как они не поддерживают сторонние системы УАТС. План выбытия для сторонней поддержки был объявлен в прошлом году в [этом блоге](https://blogs.technet.microsoft.com/exchange/2017/07/18/discontinuation-of-support-for-session-border-controllers-in-exchange-online-unified-messaging
), и пользователи в этой модели развертывания могут переносить пользователей на одну из платформ и служб единой системы обмена сообщениями Microsoft, а также получать голосовую почту и (или) третьи лица. для этих пользователей — решение для автоматического ассистента. Интеграция с факсами не поддерживается в облачных службах; Пользователи должны выполнить миграцию на стороннее решение. 

### <a name="who-is-affected"></a>Кто повлиял на это?

Это повлияет на пользователей, использующих любые из перечисленных ниже функций из веб-службы единой системы обмена сообщениями Exchange.

- Служба голосовой почты
- Служба автосекретаря
- Интеграция с факсами

> [!Note]
> Пользователи, работающие на одном из локальных серверов Exchange Server с единой системой обмена сообщениями, не затрагиваются. 

Узнайте больше о том, как пользователи и администраторы влияют [](#user-experience-impact)на работу пользователей.

## <a name="migration-steps"></a>Инструкции по миграции

1.  **Получать уведомления**
 
    Ознакомьтесь с объявлением [блога](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) и этой статьей, чтобы спланировать плавную миграцию для пользователей. Подробные сведения о возможностях голосовой почты в облаке вы увидите в разделе [Проверка голосовой почты в Skype для бизнеса и параметры](https://support.office.com/en-us/article/check-skype-for-business-voicemail-and-options-2deea7f8-831f-4e85-a0d4-b34da55945a8) .  
 
2.  **Создание гибридной топологии Skype для бизнеса**

    Если вы не установили гибридную топологию Skype для бизнеса, вам нужно сделать это, чтобы обеспечить плавную миграцию пользователей голосовой почты. Более подробную информацию вы увидите в разделе [Настройка гибридного развертывания Skype для бизнеса](../../SfbHybrid/hybrid/configure-federation-with-skype-for-business-online.md) . 

    > [!Note]
    > Для миграции службы голосовой почты не нужно переносить пользователей в Интернет. Тем не менее, для локальных пользователей, использующих службу голосовой почты на основе облака, необходимо установить гибридную топологию.

3. **Планирование миграции автоматического ассистента**
    
    Администраторы могут в любое время начать переводить свои автосекретарей из Ексчумо в облачный автоматический секретарь. Дополнительные сведения вы видите [в разделе Настройка автоматического ассистента облачного облака](https://docs.microsoft.com/microsoftteams/create-a-phone-system-auto-attendant) . Корпорация Майкрософт планирует вам добиваться дополнительных возможностей автоматического ассистента на то, что пользователи имеют важное значение для миграции на 2019 марта. Администраторы должны оценивать наборы функций и переносить их экземпляры автосекретаря соответствующим образом. Сравнение списка функций можно найти в разделе [Матрица компонентов служб ексчумо и Azure на основе облака](#exchumo-and-azure-cloud-based-services-feature-matrix).

4. **Планирование проверки голосовой почты после миграции и тестирования**

    Миграция голосовой почты ориентирована на Microsoft; Администраторам не требуется выполнение каких-либо действий с учетом того, что вы установили гибридную топологию с предварительным требованием. Корпорация Майкрософт пройдет необходимую проверку и тестирование, чтобы убедиться в том, что миграция голосовой почты пользователей не была прервана. Однако администраторам рекомендуется выполнять тестирование и проверку на своих сторонах.  Чтобы ознакомиться с рекомендуемым планом тестирования, ознакомьтесь с разкрывающимся [планом тестирования и проверкой после миграции для администраторов](#suggested-test-plan-and-post-migration-validation-for-admins) . 

    > [!Note]
    > Lync Server 2010 не поддерживается. Если вы используете развертывание сервера 2010, вы должны планировать обновление сервера или переходить пользователей в Microsoft Teams или Skype для бизнеса Online.  

5. **Наблюдение за центром уведомлений администратора**

    Ознакомьтесь с уведомлением в центре уведомлений администратора с дополнительными сведениями и временной шкалой, связанной с миграцией голосовой почты ваших пользователей. Уведомления будут отправлены по крайней мере за 30 дней до начала миграции. 

    > [!Note]
    > Если вы получили уведомление с временной шкалой миграции пользователей и хотите отложить миграцию на задачу, которая является критической для бизнеса, обратитесь в службу поддержки Майкрософт. Обратите внимание, что вы не можете отложить переход за пределы даты ухода с 2020 февраля. Для пользователей, у которых могут быть дополнительные вопросы, обратитесь в отдел своей учетной записи или в службу поддержки Майкрософт. Пользователи Office 365 могут отправить обращение в службу поддержки с помощью портала администрирования Office 365. 

6. **Рассматривайте план для запланированной миграции**

    Вы можете присоединиться к запланированной миграции службы голосовой почты в КВМ. Перед тем как приступить к отрецензированию, ознакомьтесь с подробными сведениями в этой статье, особенно в следующих разделах:

    - Шаги миграции (этот раздел)
    - Ексчумо и служба Azure на основе облака
    - Влияние на взаимодействие с пользователем

    Обратите внимание на то, что при управляемом переносе вы не будете получать уведомления о предмиграции в течение 30 дней в центре сообщений портала администрирования Microsoft 365.
 
    Чтобы отказаться от запланированной миграции, отправьте запрос по электронной почте в [CVM@microsoft.com](mailto:cvm@microsoft.com)с помощью адреса электронной почты администратора, выполнив указанные ниже сведения.

    - Предпочтительная дата (Туесдайс): волны миграции выполняются каждый вторник. Пожалуйста, Выбери дату, не превышающую 12/3/2019.
 
    - Идентификатор клиента: 32 символов в этом формате 0046728c-688A-4472-a38f-098fec60ac6x. Идентификатор клиента можно найти на портале администрирования Microsoft 365 в службе Azure Active Directory или выполнив следующий командлет PowerShell:`Get-CsTenant | Select ObjectId`
 
    После успешной миграции клиента вы получите подтверждение по электронной почте.

### <a name="suggested-test-plan-and-post-migration-validation-for-admins"></a>Предлагаемый план тестирования и проверка после миграции для администраторов

Чтобы убедиться в том, что ваши пользователи были перенесены в облачную голосовую почту, просто оставьте голосовую почту для пользователя и проверьте текст сообщения в Outlook, в облачных голосовых сообщениях есть дополнительный нижний колонтитул, который выглядит так:

**Благодарим вас за использование транскрипции! Если вы не видите запись, описанную выше, это связано с тем, что качество звука не было достаточно ясно, чтобы затем переносите.**

При тестировании функций голосовой почты после миграции пользователей убедитесь в том, что вы просматриваете указанные ниже сценарии.

- Проверка доступа голосовой почты между всеми типами конечных точек в Организации: приложения и IP-телефоны. 
- Проверяйте с помощью примеров пользователей, которые продаются с помощью настроенных персональных приветствий.   
- Если в вашей организации есть требования, предъявляемые к законным или требованиям для отключения транскрипции для пользователей, убедитесь, что она отключена после миграции. Дополнительные сведения можно найти в разделе [Настройка голосовой почты](/microsoftteams/set-up-phone-system-voicemail)в облаке.
- Если вы уже настроили политики и правила виртуальной машины Exchange, убедитесь, что они вступают в силу.
- Ознакомьтесь с командлетами PowerShell облачной службы голосовой почты, чтобы изменить параметры пользователя.
