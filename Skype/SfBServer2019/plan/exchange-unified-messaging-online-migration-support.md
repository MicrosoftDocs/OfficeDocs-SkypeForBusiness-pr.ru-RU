# <a name="exchange-unified-messaging-online-migration-support"></a>Поддержка сетевой единой системы обмена сообщениями Exchange

> [!IMPORTANT]
> **Поддержка службы единой системы обмена сообщениями в Exchange Online не поддерживается 28 февраля 2020 г. в 17:00 по тихоокеанскому времени. Все учетные записи голосовой почты были перенесены в облачную службу голосовой почты корпорацией Майкрософт. Любой оставшийся трафик автоотвода не отслеживается и может быть прерван в любое время.**

В связи [](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) с сообщением от 8 февраля 2019 г., корпорация Майкрософт отзовет службу единой системы обмена сообщениями Exchange Online (ExchUMO) до 28 февраля 2020 г. В этой статье вы узнаете, что должны знать затронутые клиенты и как планировать непрерывность их бизнеса.

Клиенты развертывают ExchUMO для голосовой почты, автоответчика, очереди вызовов и служб интеграции факсов. Корпорация Майкрософт планирует помочь клиентам перейти на службы телефонной системы, которые уже поддерживают тысячи клиентов в Skype для бизнеса Online и Microsoft Teams.

Голосовая почта в основном управляется корпорацией Майкрософт; для подмножество клиентов может потребоваться участие администратора и/или инвестиции. Автоспутник — это миграция, которая управляется администратором; Вам потребуется повторно создать существующие деревья автослужб ExchUMO в облачной автосекретарь облачной службе. Клиенты, которые будут использовать какие-либо функции ExchUMO со стороной УАТК, не будут перенесены в облачные службы Skype, так как они не поддерживают сторонние системы УАКСИ. В этом блоге был объявлен план для поддержки сторонних [пользователей,](https://techcommunity.microsoft.com/t5/Exchange-Team-Blog/New-date-for-discontinuation-of-support-for-Session-Border/ba-p/607853)и клиенты в этой модели развертывания могут перенести своих пользователей на одну из платформ или служб объединенных коммуникаций Майкрософт или приобрести для них стороне решение голосовой почты и/или автоответчик. Интеграция факсов не поддерживается в облачных службах; клиентам потребуется перейти на стороне решение.

## <a name="who-is-affected"></a>Кто затрагивается?

Клиенты, использующие любые из следующих функций из службы единой системы обмена сообщениями Exchange Online, затронуты:

- Служба голосовой почты
- автосекретарь службы
- Служба очереди вызовов
- Интеграция факсов

> [!Note]
> Клиенты, использующие какие-либо Exchange Server локальной системы с единой системы обмена сообщениями, не затронуты.

Узнайте больше о влиянии на пользователей и администраторов в [влиянии на пользовательский интерфейс.](#user-experience-impact)

## <a name="voicemail-migration-guidelines"></a>Рекомендации по миграции голосовой почты

### <a name="get-informed"></a>Получить информацию

Ознакомьтесь с [объявлением](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) блога и этой статьей, чтобы спланировать плавную миграцию для пользователей. Подробные [сведения о возможностях](https://support.office.com/article/check-skype-for-business-voicemail-and-options-2deea7f8-831f-4e85-a0d4-b34da55945a8) голосовой почты телефонной системы см. в сведениях о голосовой почте Skype для бизнеса.  

### <a name="establish-a-skype-for-business-hybrid-topology"></a>Создание гибридной топологии Skype для бизнеса

If you do not have a Skype for Business hybrid topology established, you need to do that to enable a smooth migration of your voicemail users. Дополнительные сведения см. в подстройке ["Настройка гибридной](../../SfbHybrid/hybrid/configure-federation-with-skype-for-business-online.md) конфигурации Skype для бизнеса".

> [!Note]
> Для миграции службы голосовой почты не требуется перенос пользователей через Интернет. Однако для использования службы голосовой почты телефонной системы для локального пользователя необходимо установить гибридную топологию.

### <a name="plan-your-auto-attendant-migration"></a>Планирование миграции автоспутника

Администраторы могут в любое время приступить к переносу своих автоотвечающих из ExchUMO в облачный. Дополнительные [сведения см.](https://docs.microsoft.com/microsoftteams/create-a-phone-system-auto-attendant) в подстройке "Настройка облачного автозаметаря".

Корпорация Майкрософт продолжает предоставлять дополнительные возможности автозаполнеия, которые клиенты могут считать требуемой для миграции. Администраторам следует оценить набор функций и соответствующим образом перенести экземпляры автоотвода. Сравнение списков функций см. в матрице функций облачных служб [ExchUMO](#exchumo-and-azure-cloud-based-services-feature-matrix)и Azure.

### <a name="plan-for-your-voicemail-post-migration-validation-and-testing"></a>Планирование проверки и тестирования голосовой почты после миграции

Миграция голосовой почты управляется корпорацией Майкрософт. Администраторы не обязаны ничего делать, учитывая, что установлена необходимая гибридная топология. Корпорация Майкрософт выполняет необходимую проверку и тестирование, чтобы убедиться, что миграция голосовой почты пользователей не прерывается. Администраторам рекомендуется выполнять тестирование и проверку на стороне администратора. Рекомендуемый план тестирования и проверка после [миграции](#suggested-test-plan-and-post-migration-validation-for-admins) для администраторов см. в рекомендуемом плане тестирования.

> [!Note]
> Lync Server 2010 не поддерживается. При развертывании сервера 2010 следует спланировать обновление сервера или перенести пользователей в Microsoft Teams.  

### <a name="monitor-the-admin-notification-center"></a>Мониторинг Центра уведомлений администратора

Просмотрите уведомление в Центре уведомлений администратора с дополнительными сведениями и временной шкалой, касающейся миграции пользователей. Уведомления отправляются по крайней мере за 30 дней до периода миграции.

> [!Note]
> Если вы получили уведомление с временной шкалой миграции пользователей и хотите отложить миграцию по критически важной для бизнеса причине, это можно сделать, обратившись в службу поддержки Майкрософт. Вы не можете отложить миграцию после даты выхода из нее 28 февраля 2020 г. Для пользователей, у которых могут возникнуть дополнительные вопросы, обратитесь в вашу учетную запись или службу поддержки Майкрософт. Клиенты, уже использующие Microsoft 365 или Office 365, могут подать заявку в службу поддержки через Центр администрирования Microsoft 365.

### <a name="consider-opting-in-for-a-planned-migration"></a>Рассмотрите возможность выбора запланированной миграции

Вы можете выбрать плановый перенос службы голосовой почты в CVM. Перед тем как при этом отказаться, просмотрите подробные сведения этой статьи, особенно в следующих разделах:

- Этапы миграции (этот раздел)
- Матрица функций облачных служб ExchUMO и Azure
- Влияние на пользовательский интерфейс

При выборе управляемой миграции вы не получите уведомление за 30 дней до миграции в Центре сообщений портала администрирования Microsoft 365.

Чтобы отказаться от запланированной миграции, отправьте на адрес электронной почты администратора запрос электронной почты cvm@microsoft.com [со](mailto:cvm@microsoft.com) следующими сведениями:

- Предпочитаемая дата (по вторникам): волны миграции выполняются каждый вторник. Выберите дату во вторник, которая не выходит за 3.03.2019.
 
- ИД клиента: номер 32 символов в этом формате 0046728c-688a-4472-a38f-098fec60ac6x. ИД клиента можно найти на портале администрирования Microsoft 365 в Azure AD или с помощью следующего cmdlet PowerShell: `Get-CsTenant | Select ObjectId`

После успешной миграции клиента вы получите подтверждение по электронной почте.

## <a name="auto-attendant-migration-guidelines"></a>Рекомендации по миграции автоотвода

Администраторам организаций Microsoft 365 и Office 365 необходимо повторно создать автослужб exchange UM Online в службе Microsoft Cloud автосекретарь и переключить на них номера телефонов до даты отключения службы UMO Exchange 28 февраля 2020 г. Это рекомендуемое руководство для успешной миграции и тестирования новых облачных автоспутников. Если у вас большое количество автоотвечающего персонала, вы можете использовать сценарии миграции автосекретарь exchange [um to Cloud автосекретарь,](https://github.com/NathanJBennett/ExUMAAMigrationToCloudAA) чтобы упростить массовую миграцию автоотборов.

### <a name="auto-attendant-setup"></a>Настройка автосекретаря

Мы настоятельно рекомендуем начать установку новых автослужб на ранних стадиях, чтобы избежать проблем с последней минутой и ознакомиться с функциями и возможностями облачной автосекретарь службы. Для автоотводов, которые требуют одной или более функций разрывов, можно создавать и тестировать их, когда функции разрывов доступны для подготовки к развертыванию. Дополнительные сведения о функции разрывов см. в [приложении.](#appendix)

1. Используйте cmdlets Exchange UMO для экспорта конфигурации существующих автозавершений с помощью [Get-UMAutoAttendant.](https://docs.microsoft.com/powershell/module/exchange/unified-messaging/get-umautoattendant)  
2. Чтобы экспортировать файлы мультимедиа приветствия (если они используются) и преобразовать их в формат MP3, используйте в Exchange Online PowerShell cmdlet [Export-UMprompt.](https://docs.microsoft.com/powershell/module/exchange/unified-messaging/export-umprompt)
3. Следуйте инструкциям [](../../SfbHybrid/hybrid/plan-cloud-auto-attendant.md) в области "Планирование [](https://docs.microsoft.com/microsoftteams/create-a-phone-system-auto-attendant) облачных автозаетарей" и "Настройка облачного автозаметаря" для создания автозахватарей с помощью Центра администрирования Microsoft Teams или Powershell.
4. Просмотрите приветствия, если параметры меню изменились.
5. Настройте передачу данных в группы ответа с помощью обходного пути "автосекретарь [](#known-issues) Call Transfer to PSTN" в разделе "Известные проблемы" этой статьи.  
6. Протестировать новые автоответцы, назначив им внутренний вызов или назначив тестовый номер телефона.  

### <a name="cutover"></a>Прямая

1. Переключение номеров телефонов с автоотвода Exchange UMO на новые.
2. Переместит URI SIP из объекта контакта в учетную запись ресурса.
3. Протестировать и проверить автозаверяющие номера, используя только что назначенные номера телефонов.

### <a name="suggested-test-plan-and-post-migration-validation-for-admins"></a>Рекомендуемый план тестирования и проверка после миграции для администраторов

Чтобы проверить, перенесены ли пользователи в облачную голосовую почту, оставьте голосовую почту пользователю и проверьте текст сообщения в Outlook. В сообщениях облачной голосовой почты есть footer, который считывет:

"Благодарим вас за использование транскрибации! Если вы не видите запись выше, это потому, что качество звука было недостаточно понятным для транскрибировки".

При тестировании функций голосовой почты после миграции пользователей необходимо учесть следующие сценарии:

- Проверка доступа к голосовой почте во всех типах конечных точек в организации, таких как приложения и IP-телефоны.
- Проверьте у образцов пользователей, что настроенные персонализированные приветствия будут играть для вызывающих.
- Если в вашей организации имеется юридическое или соответствие требованиям для отключения транскрибации для пользователей, убедитесь, что она отключена после миграции. Дополнительные сведения см. в [настройках облачной голосовой почты.](/microsoftteams/set-up-phone-system-voicemail)
- Если вы ранее настроили политики и правила ВМ Exchange, убедитесь, что они эффективны.
- Ознакомьтесь с помощью cmdlets облачной службы голосовой почты PowerShell для изменения параметров пользователя.

### <a name="known-issues"></a>Известные проблемы

#### <a name="greeting-inconsistencies"></a>Несоответствия приветствия

Абонентский доступ может работать в клиенте до полного выхода из службы даже после переноса всех пользователей в облачную голосовую почту. Чтобы избежать путаницы и несогласованного интерфейса, отключать доступ подписчика, так как приветствия изменяются после миграции. Для этого удалите контакт EXUM для каждой линии доступа подписчика, используя `Get-CsExUmContact | ?{$_.IsSubscriberAccess -eq $true} | Remove-CsExUmContact` .

#### <a name="auto-attendant-call-transfer-to-pstn"></a>автосекретарь передачи вызовов в PSTN

To transfer an auto attendant call to an external PSTN phone number via Skype for Business Server or a Response Group service (RGS) on Skype for Business Server, create a new on-premises user with call forwarding set to the PSTN phone number or RGS phone number. Пользователь должен быть включен и правильно настроен для Корпоративная голосовая связь и иметь назначенную политику голосовой почты.

#### <a name="shared-mailbox-is-still-accessible"></a>Общий почтовый ящик по-прежнему доступен

Общий почтовый ящик, настроенный с помощью Exchange UM Online, продолжает получать сообщения после миграции в CVM и доступен пользователям через Outlook. Однако после переноса в CVM доступ к приветствию этих почтовых ящиков будет недоступн. Клиенты с общими почтовыми ящиками, которые используются для захвата абонентов автоотвода, должны использовать функции общего почтового ящика автозахвателей и очередей вызовов после выпуска (ETA за октябрь 2019 г.).
  
#### <a name="username-is-not-using-skype-for-business-banner-displays"></a>Отображается баннер "Имя пользователя не использует Skype для бизнеса"

Служба CVM основана на инфраструктуре Microsoft Teams, и вызовы клиента Skype для бизнеса могут привести к отображке на клиенте информационного баннера: "Имя пользователя не использует Skype для бизнеса. Для более простого опыта переключиться на Teams или начать собрание Skype".
Обязательно обновите клиент Skype для бизнеса пользователей до последнего обновления клиента C2R, чтобы этот баннер не появлялось.
  
#### <a name="set-up-voice-mail-takes-you-to-owa"></a>"Настройка голосовой почты" приводит к перенастройки OWA

При  нажатии кнопки "Настройка голосовой почты" клиенты Skype для бизнеса Server 2015/2013 по-прежнему переходят на страницу портала веб-клиента Office (OWA) после миграции в CVM. Все параметры были удалены со вкладки "Голосовая почта" в OWA, и будет отображаться баннер со ссылкой на перенаправление для перенаправления пользователей на портал параметров пользователя CVM.

#### <a name="changing-greeting-remotely"></a>Удаленное изменение приветствия

Абонентский доступ к STN не поддерживается в CVM. Для пользователей, которые должны удаленно изменить приветствие, в службу голосовой почты IVR для мобильных клиентов добавлен параметр меню "Изменить приветствие". Пользователи могут вызывать эту службу, нажимая и удерживая клавишу "1" на панели мобильного клиента.
