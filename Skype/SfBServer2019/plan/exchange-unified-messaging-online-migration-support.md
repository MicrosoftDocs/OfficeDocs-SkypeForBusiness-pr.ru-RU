# <a name="exchange-unified-messaging-online-migration-support"></a>Поддержка сетевой единой системы обмена сообщениями Exchange

В ссылке на [объявление](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) 8 февраля 2019 служба единой системы обмена сообщениями Exchange в Интернете (ексчумо) не входит в состав 2020 февраля. В этой статье рассказывается о том, какие уязвимые пользователи должны знать и как планировать бесперебойную деятельность своей компании. 
 
Ексчумо разворачивается пользователями для голосовой почты, автосекретаря, очереди звонков и служб интеграции факса. Корпорация Майкрософт планирует, чтобы пользователи могли перейти на службы телефонной системы, которые уже поддерживают тысячи пользователей в Skype для бизнеса Online и Microsoft Teams. 

Голосовая почта — это преимущественно процесс миграции с помощью Microsoft. для подмножества пользователей может потребоваться участие администратора и/или инвестиция. Автоматический секретарь — это миграция, управляемая администратором; Вам потребуется повторно создать существующие деревья автосекретаря Ексчумо в облачной службе облачного секретаря. Пользователи, которые используют любую из функций Ексчумо с сторонней АТС, не будут перенесены в облачные службы Skype, так как они не поддерживают сторонние системы УАТС. В [этом блоге](https://techcommunity.microsoft.com/t5/Exchange-Team-Blog/New-date-for-discontinuation-of-support-for-Session-Border/ba-p/607853)был объявлен план выбытия для сторонней поддержки, и пользователи в этой модели развертывания могут перенести пользователей на одну из платформ и служб единой системы обмена сообщениями Microsoft или получить собственное решение голосовой почты и/или автосекретаря для этих пользователей. Интеграция с факсами не поддерживается в облачных службах; Пользователи должны выполнить миграцию на стороннее решение.

### <a name="who-is-affected"></a>Кто повлиял на это?

Это повлияет на пользователей, использующих любые из перечисленных ниже функций из веб-службы единой системы обмена сообщениями Exchange.

- Служба голосовой почты
- Служба автосекретаря
- Служба очереди звонков
- Интеграция с факсами

> [!Note]
> Пользователи, работающие на одном из локальных серверов Exchange Server с единой системой обмена сообщениями, не затрагиваются. 

Узнайте больше о том, как пользователи и администраторы влияют [на работу пользователей.](#user-experience-impact)

## <a name="migration-steps"></a>Инструкции по миграции

1.  **Получать уведомления**
 
    Ознакомьтесь с [объявлением блога](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) и этой статьей, чтобы спланировать плавную миграцию для пользователей. Подробные сведения о возможностях голосовой почты в телефонной системе вы видите в разделе [Проверка голосовой почты в Skype для бизнеса и параметры](https://support.office.com/en-us/article/check-skype-for-business-voicemail-and-options-2deea7f8-831f-4e85-a0d4-b34da55945a8) .  
 
2.  **Создание гибридной топологии Skype для бизнеса**

    Если вы не установили гибридную топологию Skype для бизнеса, вам нужно сделать это, чтобы обеспечить плавную миграцию пользователей голосовой почты. Более подробную информацию вы увидите в разделе [Настройка гибридного развертывания Skype для бизнеса](../../SfbHybrid/hybrid/configure-federation-with-skype-for-business-online.md) . 

    > [!Note]
    > Для миграции службы голосовой почты не нужно переносить пользователей в Интернет. Тем не менее, для локальных пользователей, использующих службу голосовой почты телефонной системы, необходимо установить гибридную топологию.

3. **Планирование миграции автоматического ассистента**
    
    Администраторы могут в любое время начать переводить свои автосекретарей из Ексчумо в облачный автоматический секретарь. Дополнительные сведения вы видите [в разделе Настройка автоматического ассистента облачного облака](https://docs.microsoft.com/microsoftteams/create-a-phone-system-auto-attendant) . Корпорация Майкрософт продолжает поддерживать дополнительные возможности автосекретаря, которые могут принимать пользователи, необходимые для миграции, администраторы должны оценить наборы функций и соответствующим образом переносить их экземпляры автоматического секретаря. Сравнение списка функций можно найти в разделе [Матрица компонентов служб ексчумо и Azure на основе облака](#exchumo-and-azure-cloud-based-services-feature-matrix).

4. **Планирование проверки голосовой почты после миграции и тестирования**

    Миграция голосовой почты осуществляется на основе Microsoft. Администраторам не требуется выполнение каких-либо действий с учетом того, что вы установили гибридную топологию с предварительным требованием. Корпорация Microsoft выполняет проверку подлинности и тестирование, чтобы убедиться в том, что миграция голосовой почты пользователя не была прервана. Администраторам рекомендуется выполнять тестирование и проверку на своих сторонах. Чтобы ознакомиться с рекомендуемым планом тестирования, ознакомьтесь с [разкрывающимся планом тестирования и проверкой после миграции для администраторов](#suggested-test-plan-and-post-migration-validation-for-admins) . 

    > [!Note]
    > Lync Server 2010 не поддерживается. Если вы используете развертывание сервера 2010, вы должны планировать обновление сервера или переходить пользователей в Microsoft Teams или Skype для бизнеса Online.  

5. **Наблюдение за центром уведомлений администратора**

    Загляните в центр уведомлений администратора с дополнительными сведениями и временной шкалой, касающейся миграции пользователей. Уведомления отправляются по крайней мере за 30 дней до начала миграции. 

    > [!Note]
    > Если вы получили уведомление с временной шкалой миграции пользователей и хотите отложить миграцию на задачу, которая является критической для бизнеса, обратитесь в службу поддержки Майкрософт. Обратите внимание, что вы не можете отложить переход за пределы даты ухода с 2020 февраля. Для пользователей, у которых могут быть дополнительные вопросы, обратитесь в отдел своей учетной записи или в службу поддержки Майкрософт. Пользователи Office 365 могут отправить обращение в службу поддержки с помощью портала администрирования Office 365. 

6. **Рассматривайте план для запланированной миграции**

    Вы можете присоединиться к запланированной миграции службы голосовой почты в КВМ. Перед тем как приступить к отрецензированию, ознакомьтесь с подробными сведениями в этой статье, особенно в следующих разделах:

    - Шаги миграции (этот раздел)
    - Ексчумо и служба Azure на основе облака
    - Влияние на взаимодействие с пользователем

    При выборе управляемой миграции вы не будете получать уведомления о том, что вы предваряем миграции в течение 30 дней в центре сообщений на портале администрирования Microsoft 365.
 
    Чтобы принять участие в запланированной миграции, отправьте запрос по электронной почте, предоставленный администратором электронной почты, в [CVM@microsoft.com](mailto:cvm@microsoft.com) со следующими сведениями:

    - Предпочтительная дата (Туесдайс): волны миграции выполняются каждый вторник. Пожалуйста, Выбери дату, не превышающую 12/3/2019.
 
    - Идентификатор клиента: 32 символов в этом формате 0046728c-688A-4472-a38f-098fec60ac6x. Идентификатор клиента можно найти на портале администрирования Microsoft 365 в Azure AD или с помощью следующего командлета PowerShell:`Get-CsTenant | Select ObjectId`
 
    После успешной миграции клиента вы получаете подтверждение по электронной почте.

### <a name="suggested-test-plan-and-post-migration-validation-for-admins"></a>Предлагаемый план тестирования и проверка после миграции для администраторов

Чтобы убедиться в том, что ваши пользователи были перенесены в облачную голосовую почту, оставьте голосовую почту для пользователя и проверьте текст сообщения в Outlook.  В облачных сообщениях голосовой почты есть нижний колонтитул, который выглядит так:

**Благодарим вас за использование транскрипции! Если вы не видите запись, описанную выше, это связано с тем, что качество звука не было достаточно ясно, чтобы затем переносите.**

При тестировании функций голосовой почты после миграции пользователей убедитесь в том, что вы просматриваете указанные ниже сценарии.

- Проверка доступа голосовой почты между всеми типами конечных точек в Организации: приложения и IP-телефоны. 
- Проверяйте с помощью примеров пользователей, которые продаются с помощью настроенных персональных приветствий.   
- Если в вашей организации есть требования, предъявляемые к законным или требованиям для отключения транскрипции для пользователей, убедитесь, что она отключена после миграции. Дополнительные сведения можно найти в разделе [Настройка голосовой почты в облаке](/microsoftteams/set-up-phone-system-voicemail).
- Если вы уже настроили политики и правила виртуальной машины Exchange, убедитесь, что они вступают в силу.
- Ознакомьтесь с командлетами PowerShell облачной службы голосовой почты, чтобы изменить параметры пользователя.

#### <a name="known-issues"></a>Известные проблемы

Переадресация **звонков по автосекретарем в КТСОП** Пользователям рекомендуется настроить временное решение, чтобы удовлетворить требованиям, предъявляемым к передаче автосекретаря на внешний телефонный номер или в экземпляр RGS. 
 
Обнаружена проблема, связанная с обеспечением качества передачи данных по сети PSTN, которая не будет решена, когда клиенты начнут переходить на Exchange УМО Service до запланированной даты выбытия февраля 1 – 2020. В качестве обходного пути администраторы могут переносить вызывающие абонента на локальный виртуальный пользователь с активным параметром переадресации звонков на требуемый телефонный номер КТСОП или в RGS-номер телефона. 
 
Ожидаемый опыт
- Администраторам не требуется лицензировать виртуального пользователя, так как это решение для решения проблемы 
- Администраторы могут управлять ИДЕНТИФИКАТОРом вызывающего абонента, который будет видеть получатель PSTN, назначив требуемый номер виртуальному пользователю или воспользовавшись функциями SBC для однозначных цифр. 
- Вызывающие абоненты PSTN не сталкиваются с задержкой во время передачи звонка, и они продолжают видеть идентификатор вызывающего абонента для автосекретаря после того, как передача будет выполнена успешно.  

**Общий почтовый ящик:** Общий почтовый ящик, настроенный с использованием единой системы обмена сообщениями Exchange Online, продолжает получать сообщения после миграции в КВМ и будет по-прежнему доступен для пользователей через Outlook. Тем не менее, если вы перейдете на КВМ, Access не будет доступен для изменения сообщений приветствия этих почтовых ящиков. Пользователи с общими почтовыми ящиками, которые используются для захвата автоматических абонентов, должны использовать возможности автосекретарей и очередей звонков для общих почтовых ящиков после выпуска (за Октябрь 2019).
  
**Переход на баннер Teams в клиенте SFB:** Служба КВМ основана на инфраструктуре Microsoft Teams; звонки из клиента Skype для бизнеса могут привести к отображению баннера данных на клиенте, который читает: "ИмяПользователя не использует Skype для бизнеса". Для более богатого опыта переключитесь на Teams или начните собрание Skype. "
Убедитесь, что вы обновили клиента Skype для бизнеса в последнюю версию клиента C2R, чтобы не появлялись эти объявления. 
  
**Настройка голосовой почты позволит вам перейти в OWA:** При нажатии кнопки "Настройка голосовой почты" с клиента все пользователи Skype для бизнеса Server 2015/2013 станут доступны на странице портала Office Web Access (OWA) после миграции в КВМ. Все параметры удалены с вкладки "Голосовая почта" в Outlook Web App, а баннер появится в виде ссылки для перенаправления, чтобы перевести пользователей на портал параметров пользователя КВМ. 
 
**Изменение мобильного доступа к приветствию:** Доступ к абонентам PSTN не поддерживается в КВМ. Для пользователей, которые должны удаленно изменить свое приветствие, к службе голосовой почты интерактивный автоответчик для мобильных клиентов добавляется параметр меню "изменить приветствие". Пользователи могут звонить на эту службу, нажав и удерживая клавишу "1" на наборе номера для мобильных клиентов.
