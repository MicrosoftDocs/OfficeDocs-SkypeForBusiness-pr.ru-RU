# <a name="exchange-unified-messaging-online-migration-support"></a>Поддержка сетевой единой системы обмена сообщениями Exchange  

Возможно использование [объявлений](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) на 8 февраля 2019 Microsoft — это закончится единой системы обмена сообщениями службу Exchange Online (ExchUMO) с февраля 2020. В этой статье предлагает Сводка какие пользователей, столкнувшихся должны знать и сделать, чтобы спланировать их непрерывность ведения бизнеса. 
 
ExchUMO развертывается для голосовой почты, автосекретаря и/или службы факсов интеграции клиентов. Microsoft планов для этих клиентов перенос в его облачные службы, поддерживающие тысячи пользователей на Скайп для бизнеса в Интернет и рабочих групп Microsoft. 

Голосовая почта — это в основном миграции с под управлением Microsoft; участие администратора и/или инвестиций может потребоваться для подмножества клиентов. Автосекретарь является миграции с под управлением администратора; необходимо будет повторно создать существующего автосекретаря ExchUMO деревьями в облаке автосекретаря облачная служба. Пользователи, которые потребляют ресурсы какие-либо функции ExchUMO с помощью сторонних производителей УАТС не будут перенесены Скайп облачных служб, так как они не поддерживают сторонними системами УАТС. Плана пенсионного обеспечения для поддержки сторонних производителей появилась в [этом блоге](https://blogs.technet.microsoft.com/exchange/2017/07/18/discontinuation-of-support-for-session-border-controllers-in-exchange-online-unified-messaging
)предыдущего года, и пользователи в этой модели развертывания можно перенос пользователей к одной платформы объединенных коммуникаций корпорации Майкрософт и служб или получить голосовой почты сторонних производителей и (или) Auto attendant решение для этих пользователей. Интеграция факсов не поддерживается в облачные службы; клиентам будет необходимо перенести решения сторонних производителей. 

### <a name="who-is-affected"></a>Кто зависит?

Пользователи, которые потребляют ресурсы какие-либо из перечисленных ниже компонентов из единой системы обмена сообщениями службу Exchange Online могут измениться.

1. Служба голосовой почты 
2. Службы Auto Attendant 
3. Интеграция факсов 

> [!Note]
> Не влияет на клиентов, использующих любой из Exchange Server на локальном с единой системой обмена сообщениями. 

Дополнительные сведения о пользователя и влияние интерфейс администрирования в [воздействия взаимодействие с пользователем](#user-experience-impact).

## <a name="migration-steps"></a>Этапы миграции

1.  **Получение уведомляться**
 
    Ознакомьтесь с [блог объявлений](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) и планирование легко перенести для пользователей. В разделе [Проверка Скайп Business голосовой почты и параметров](https://support.office.com/en-us/article/check-skype-for-business-voicemail-and-options-2deea7f8-831f-4e85-a0d4-b34da55945a8) для получения дополнительных сведений о возможностях голосовой почты в облаке.  
 

2.  **Установить Скайп для бизнеса гибридной топологии**

    Если у вас Скайп для установления Business гибридной топологии, необходимо выполнить, которая позволяет легко перенести пользователей голосовой почты. Для получения дополнительных сведений в разделе [Настройка Скайп для гибридных бизнеса](../../SfbHybrid/hybrid/configure-federation-with-skype-for-business-online.md) . 

    > [!Note]
    > Перенос пользователей в сети для переноса служба голосовой почты не требуется. Тем не менее, для локальных пользователей по использованию службы голосовой почты на основе облака гибридной топологии — это должно быть установлено.

3. **Планирование миграции автосекретаря**
    
    «Администраторы» можно начать перенос их автосекретари ExchUMO автосекретаря облака в любое время. [Автосекретарь телефонной системой](https://docs.microsoft.com/microsoftteams/create-a-phone-system-auto-attendant) для получения дополнительных сведений см. Планы Microsoft для предоставления дополнительных возможностей автосекретаря, пользователям сделать это критически важные для их миграции с 2019 марта. «Администраторы» следует оценить набор компонентов и перенос их экземпляры автосекретаря соответствующим образом. Сравнение список компонентов в разделе [сведения о функций облачные службы ExchUMO и Azure](#exchumo-and-azure-cloud-based-services-feature-matrix).

4. **Планирование проверки после миграции голосовой почты и тестирование**

    Миграция голосовой почты является Microsoft событиями; Администраторы не нужно выполнять любые действия, что необходимые предварительные гибридной топологии определен. Корпорация Майкрософт будет выполнять проверки и тестирования, чтобы убедиться в том, что не прерывается, голосовая почта миграции пользователей; Тем не менее рекомендуется "Администраторы" для выполнения тестирования и проверки их части.  План тестирования рекомендуемые в разделе [предлагаемые тестирования план и проверки после завершения переноса для администраторов](#suggested-test-plan-and-post-migration-validation-for-admins) . 

    > [!Note]
    > Lync Server 2010 не поддерживается. Если в развертывании server 2010, следует Планирование обновления сервера или рассмотрите возможность миграции пользователей для групп Майкрософт или Скайп для бизнеса в Интернет.  

5. **Монитор уведомлений центра администрирования**

    Найдите в работе уведомлений в центре администрирования уведомлений дальнейший содержатся дополнительные сведения и временной шкалы о миграции пользователей голосовой почты. По крайней мере 30 дней до срок миграции будут отправляться уведомления. 

    > [!Note]
    > Если получено уведомление с временная шкала миграции пользователей и хотите отложить перехода по важным причине вы можно обратиться в службу технической поддержки Майкрософт. Обратите внимание, что не может отложить за дату пенсионного обеспечения февраля 2020 миграции. Для пользователей, которые могут иметь дополнительные вопросы обратитесь к учетной записи группы или службы поддержки Майкрософт. Клиенты, которые уже используют Office 365 можно отправлять дела поддержка через портал администрирования Office 365. 

6. **Необходимо учитывать добиться начиная с мая 2019**

    Можно выбрать в первых миграции службы голосовой почты с мая 2019 (Если вы не получите уведомление о миграции), чтобы выровнять перехода с лицензии выплат или администратор персонала отпуска или во избежание критические для предприятия периодов. В этой статье будут обновлены сведения о процессе явного согласия пользователя перед мая 2019.

### <a name="suggested-test-plan-and-post-migration-validation-for-admins"></a>План предложенного тестирования и проверки после завершения переноса для администраторов

При тестировании функциональные возможности голосовой почты после завершения переноса пользователей, убедитесь, что необходимо учитывать следующие сценарии:

- Проверка доступа к голосовой почты для всех типов конечных точек в вашей организации: приложений и IP-телефоны. 
- Проверьте с помощью образца пользователей, которые будут воспроизводиться настроенного Настраиваемое приветствие для вызывающих объектов.   
- Если в вашей организации есть требования юридическим и соответствие требованиям для отключения транскрибирования для пользователей, убедитесь в том, что это отключено записи миграции. Для получения дополнительных сведений см [голосовой почты в облаке](/microsoftteams/set-up-phone-system-voicemail).
- Если вы ранее настроили виртуальной Машины Exchange политик и правил, убедитесь, что они эффективны.
- Ознакомиться с помощью командлетов PowerShell голосовой почты облачных служб для изменения параметров пользователя.
