# <a name="exchange-unified-messaging-online-migration-support"></a>Поддержка сетевой единой системы обмена сообщениями Exchange

> [!IMPORTANT]
> **Служба единой системы обмена сообщениями в Exchange Online не поддерживается по 28 февраля 2020 г. в 17:00 по тихоокеанскому времени. Все учетные записи голосовой почты были перенесены в облачная голосовая почта службу Майкрософт. Любой оставшийся трафик автоспутника не отслеживается и может быть нарушен в любое время.**

Ссылаясь [](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) на сообщение от 8 февраля 2019 г., Корпорация Майкрософт до 28 февраля 2020 г. отзовет службу Exchange Единой системы обмена сообщениями в Интернете (ExchUMO). В этой статье предлагается сводка о том, что должны знать и делать пострадавшие клиенты для планирования непрерывности бизнеса.

ExchUMO развертывается клиентами для служб голосовой почты, автопоезда, очереди вызовов и интеграции факсов. Корпорация Майкрософт планирует помочь клиентам перейти телефонная система службы, которые уже поддерживают тысячи клиентов Skype для бизнеса и Microsoft Teams.

Голосовая почта — это в первую очередь миграция, ориентированная на Microsoft; Для подмножество клиентов может потребоваться участие администратора и/или инвестиции. Автоспутник — это перенос, управляемый администратором; Вам потребуется повторно создать существующие деревья автоконцерна ExchUMO в облачной службе автосекретарь облака. Клиенты, потреблявшие какие-либо функции ExchUMO с помощью сторонних PBX, не будут Skype облачные службы, так как не поддерживают сторонние системы PBX. В этом блоге был объявлен план выхода на пенсию для сторонних служб [поддержки,](https://techcommunity.microsoft.com/t5/Exchange-Team-Blog/New-date-for-discontinuation-of-support-for-Session-Border/ba-p/607853)и клиенты этой модели развертывания могут перенести своих пользователей на одну из платформ и служб Единой коммуникации Майкрософт или приобрести сторонние решения голосовой почты и/или автопослужающего для этих пользователей. Интеграция факсов не поддерживается в облачных службах; клиенты должны будут перейти на сторонное решение.

## <a name="who-is-affected"></a>Кто влияет?

Пользователи, использующие все следующие функции из Exchange единой службы обмена сообщениями в Интернете, затронуты:

- Служба голосовой почты
- автосекретарь службы
- Служба очереди вызовов
- Интеграция факсов

> [!Note]
> Клиенты, использующие какой-либо Exchange Server локальной системы единой системы обмена сообщениями, не затронуты.

Дополнительные данные о влиянии на пользовательский и администраторский опыт при воздействии [на пользовательский опыт.](#user-experience-impact)

## <a name="voicemail-migration-guidelines"></a>Рекомендации по миграции голосовой почты

### <a name="get-informed"></a>Получить информацию

Ознакомьтесь с объявлением [в блоге](https://blogs.technet.microsoft.com/exchange/2019/02/08/retiring-unified-messaging-in-exchange-online/) и этой статьей, чтобы спланировать плавную миграцию для пользователей. Сведения [о возможностях телефонная система](https://support.office.com/article/check-skype-for-business-voicemail-and-options-2deea7f8-831f-4e85-a0d4-b34da55945a8) голосовой почты см. в Skype для бизнеса в Skype для бизнеса голосовой почты.  

### <a name="establish-a-skype-for-business-hybrid-topology"></a>Создание Skype для бизнеса гибридной топологии

Если у вас нет Skype для бизнеса гибридной топологии, это необходимо сделать, чтобы обеспечить плавную миграцию пользователей голосовой почты. Дополнительные [сведения см. в Skype для бизнеса hybrid.](../../SfbHybrid/hybrid/configure-federation-with-skype-for-business-online.md)

> [!Note]
> Для миграции службы голосовой почты не требуется перенос пользователей в Интернете. Однако для использования локальной службы голосовой почты телефонная система пользователей необходимо установить гибридную топологию.

### <a name="plan-your-auto-attendant-migration"></a>Планирование миграции автоспутников

Администраторы могут в любое время приступить к миграции автоспутников из ExchUMO в автоспутник Cloud. Дополнительные [сведения см. в материале Настройка автопутепроводника](/microsoftteams/create-a-phone-system-auto-attendant) Cloud.

Корпорация Майкрософт продолжает предоставлять дополнительные возможности автоспутника, которые клиенты могут считать нужными для их миграции. Администраторы должны оценить набор функций и соответствующим образом перенести экземпляры автоотвода. Для сравнения списка функций см. матрицу функций облачных служб [ExchUMO](#exchumo-and-azure-cloud-based-services-feature-matrix)и Azure.

### <a name="plan-for-your-voicemail-post-migration-validation-and-testing"></a>Планирование проверки и тестирования голосовой почты после миграции

Миграция голосовой почты определяется Корпорацией Майкрософт. Администраторы не обязаны ничего делать, учитывая, что установлена предварительная гибридная топология. Корпорация Майкрософт выполняет необходимую проверку и тестирование, чтобы убедиться, что миграция голосовой почты пользователей не нарушается. Администраторам рекомендуется выполнять тестирование и проверку на своей стороне. См. [рекомендуемый план](#suggested-test-plan-and-post-migration-validation-for-admins) тестирования и проверку после миграции для администраторов для рекомендуемого плана тестирования.

> [!Note]
> Lync Server 2010 не поддерживается. Если вы находитесь в развертывании сервера в 2010 году, следует спланировать обновление сервера или рассмотреть возможность переноса пользователей на Microsoft Teams.  

### <a name="monitor-the-admin-notification-center"></a>Мониторинг Центра уведомлений администратора

Обратите внимание на уведомление в Центре уведомлений администратора с дополнительными сведениями и временной шкалой, касающейся миграции пользователей. Уведомления отправляются не менее чем за 30 дней до периода миграции.

> [!Note]
> Если вы получили уведомление со временем миграции пользователей и хотите отложить миграцию по бизнес-критическим причинам, вы можете сделать это, обратившись в Службу поддержки Майкрософт. Вы не можете отложить миграцию после даты выхода на пенсию 28 февраля 2020 г. Для клиентов, у которых может возникнуть больше вопросов, свяжитесь с командой учетной записи или службой поддержки Майкрософт. Клиенты, уже использующие Microsoft 365 или Office 365, могут отправить заявку на поддержку через Центр администрирования Microsoft 365.

### <a name="consider-opting-in-for-a-planned-migration"></a>Рассмотрите возможность выбора запланированной миграции

Вы можете выбрать плановый перенос службы голосовой почты в CVM. Перед тем как войти, просмотрите сведения из этой статьи, особенно в следующих разделах:

- Этапы миграции (в этом разделе)
- Матрица функций облачных служб ExchUMO и Azure
- Влияние на пользовательский опыт

При выборе управляемой миграции в центре сообщений Microsoft 365 портала администратора не будет получать уведомление о переносе за 30 дней.

Чтобы выбрать запланированную миграцию, отправьте запрос электронной почты с адреса электронной почты администратора в cvm@microsoft.com [с](mailto:cvm@microsoft.com) следующей информацией:

- Предпочитаемая дата (по вторникам): миграционные волны выполняются каждый вторник. Выберите дату во вторник, которая не выходит за пределы 12/3/2019.
 
- Номер клиента: номер 32 символов в этом формате 0046728c-688a-4472-a38f-098fec60ac6x. Код клиента можно найти на портале администрирования Microsoft 365 Azure AD или с помощью следующего cmdlet PowerShell:`Get-CsTenant | Select ObjectId`

После успешной миграции клиента вы получите подтверждение электронной почты.

## <a name="auto-attendant-migration-guidelines"></a>Инструкции по миграции, сопровождающие автопроизводие

Microsoft 365 и Office 365 администраторы организаций должны повторно создать свои автослужатели Exchange um Online в службе Microsoft Cloud автосекретарь и переключить свои собственные номера телефонов на них до даты выхода на пенсию Exchange службы UMO 28 февраля 2020 г. Это рекомендуемое руководство для успешной миграции и тестирования новых автоспутников Cloud. Если у вас большое количество автоспутников, вы можете использовать скрипты миграции Exchange [um автосекретарь](https://github.com/NathanJBennett/ExUMAAMigrationToCloudAA) Cloud автосекретарь для упрощения массовой миграции автоспутников.

### <a name="auto-attendant-setup"></a>Настройка автосекретаря

Мы настоятельно рекомендуем начать установку новых автовласти с раннего возраста, чтобы избежать проблем в последнюю минуту и ознакомиться с функциональными возможностями и опытом службы cloud автосекретарь. Для автовласти, которые требуют одного или более функций пробела, вы можете создать и проверить автовласти, когда функции пробела доступны для подготовки к развертыванию. Дополнительные сведения об особенностях пробела см. в [приложении .](#appendix)

1. Используйте Exchange UMO для экспорта конфигурации существующих автопроводников с помощью [Get-UMAutoAttendant.](/powershell/module/exchange/unified-messaging/get-umautoattendant)  

2. Чтобы экспортировать файлы приветствия (если используется .mp3) и преобразовать их в формат Exchange Online PowerShell, используйте в [Exchange Online-uMprompt.](/powershell/module/exchange/unified-messaging/export-umprompt)

3. Следуйте инструкциям в автоспутниках [Plan Cloud](../../SfbHybrid/hybrid/plan-cloud-auto-attendant.md) и установите автоспутник Cloud для создания автоспутников с помощью центра администрирования Microsoft Teams Или PowerShell. [](/microsoftteams/create-a-phone-system-auto-attendant)

4. Просмотрите приветствия, если изменились параметры меню.

5. Настройка передачи в группы ответов с помощью обходного решения "автосекретарь вызовов в PSTN" в разделе [Известные](#known-issues) проблемы этой статьи.  

6. Проверьте новых автоспутников, назначив их внутренними или назначив тестовый номер телефона.  

### <a name="cutover"></a>Прямая

1. Переключение номеров телефонов с Exchange автоспутников UMO на новые автоспутники.
2. Перемещение SIP URI из контактного объекта в учетную запись ресурса.
3. Проверка и проверка автовластов с помощью недавно назначенных номеров телефонов.

### <a name="suggested-test-plan-and-post-migration-validation-for-admins"></a>Рекомендуемый план тестирования и проверка после миграции для администраторов

Чтобы проверить, что пользователи были перенесены в облачная голосовая почта, оставьте голосовую почту пользователю и проверьте тело сообщения в Outlook. облачная голосовая почта сообщения имеют подножку, которая читает:

"Спасибо за использование транскрипции! Если вы не видите стенограмму выше, это потому, что качество звука не было достаточно ясно, чтобы транскрипция ".

При тестировании функций голосовой почты после переноса пользователей обязательно рассмотрите следующие сценарии:

- Проверка доступа голосовой почты ко всем типам конечных точек в организации, например приложениям и IP-телефонам.
- Проверьте у примерных пользователей, что настроенные персонализированные приветствия играются для вызываемых пользователей.
- Если в вашей организации имеется юридическое или соответствие требованиям для отключения транскрипции для пользователей, убедитесь, что она отключена после переноса. Дополнительные сведения см. в [облачная голосовая почта.](/microsoftteams/set-up-phone-system-voicemail)
- Если вы ранее настраивали Exchange политики и правила VM, убедитесь, что они эффективны.
- Ознакомьтесь с облачная голосовая почта службы PowerShell для изменения параметров пользователя.

### <a name="known-issues"></a>Известные проблемы

#### <a name="greeting-inconsistencies"></a>Несоответствия приветствия

Доступ к абонентам может продолжать работать для клиента, пока служба не будет полностью отменена, даже после того, как все пользователи будут перенесены в облачная голосовая почта. Чтобы избежать путаницы пользователей и несогласованного опыта, отключать доступ к абоненту после изменения приветствия после миграции. Для этого удалите контакт EXUM для каждой линии доступа абонента с помощью `Get-CsExUmContact | ?{$_.IsSubscriberAccess -eq $true} | Remove-CsExUmContact` .

#### <a name="auto-attendant-call-transfer-to-pstn"></a>автосекретарь передачи вызовов в PSTN

Чтобы перенести вызов автопослужающего на внешний номер телефона PSTN через Skype для бизнеса Server или службу группы ответа (RGS) в Skype для бизнеса Server, создайте нового локального пользователя с набором переадружения вызовов на номер телефона PSTN или номер телефона RGS. Пользователь должен быть включен и правильно настроен для Корпоративная голосовая связь и ему назначена голосовая политика.

#### <a name="shared-mailbox-is-still-accessible"></a>Общий почтовый ящик по-прежнему доступен

Общий почтовый ящик, настроенный с Exchange um Online, продолжает получать сообщения после переноса в CVM и быть доступным для пользователей через Outlook. Однако доступ к изменению приветствия этих почтовых ящиков не будет доступен после переноса в CVM. Клиенты с общими почтовыми ящиками, которые используются для захвата абонентов автосервера, должны использовать возможности автоматических помощников и очередей вызовов после выпуска (ETA октябрь 2019 г.).
  
#### <a name="username-is-not-using-skype-for-business-banner-displays"></a>"Имя пользователя не использует Skype для бизнеса"

Служба CVM основана на инфраструктуре Microsoft Teams, а вызовы клиента Skype для бизнеса могут привести к отобралке на клиенте информационного баннера с надписью: "Имя пользователя не использует Skype для бизнеса. Для более богатого опыта переключение Teams или начало собрания Skype".
Не забудьте обновить клиент Skype для бизнеса клиента до последнего обновления клиента C2R, чтобы не допустить появления этого баннера.
  
#### <a name="set-up-voice-mail-takes-you-to-owa"></a>"Настройка голосовой почты" принимает вас в OWA

Щелкнув настройка голосовой почты от клиента, клиенты Skype для бизнеса Server 2015/2013 на страницу портала Office веб-доступа (OWA) после миграции в CVM.  Все параметры удалены со вкладки Голосовой почты в OWA, и баннер будет отображаться со ссылкой на перенаправление, чтобы демонстрировать пользователей на портал параметров пользователя CVM.

#### <a name="changing-greeting-remotely"></a>Удаленное изменение приветствия

Доступ абонентов PSTN не поддерживается в CVM. Для пользователей, которые должны изменить приветствие удаленно, в службу IVR голосовой почты для мобильных клиентов добавлена опция меню "Измените приветствие". Пользователи могут вызвать эту службу, нажав и удерживая клавишу "1" на мобильной клиентской панели.
