## <a name="hardware-for-skype-for-business-server-2019"></a>Оборудование для Skype для бизнеса Server 2019

Серверы Skype для бизнеса Server 2019 требуют 64-битного оборудования. Ниже приведены рекомендации по оборудованию. Это не требования, но они отражают требования, необходимые для оптимальной производительности. ## <a name="operating-systems-for-skype-for-business-server-2019"></a>Операционные системы для Skype для бизнеса Server 2019
<a name="OS"> </a>

После установки оборудования необходимо установить операционную систему (ОС), которая позволит установить и успешно использовать Skype для бизнеса Server 2019.
  
|||
|:-----|:-----|
|Windows Server 2019 <br/> |
|Windows Server 2016 <br/> ||
||
   
Все, кроме перечисленных здесь операционных систем, не будет работать должным образом; Не пытайтесь установить Skype для бизнеса Server 2019. Например, параметр Server Core не указан и поэтому не поддерживается.

> [!NOTE]
> Обновление ОС на месте не поддерживается с помощью Lync Server 2013. Необходимо развернуть отдельный пул и перенести пользователей в новый пул с другой ОС. Все серверы пула должны иметь ту же версию ОС.

> [!NOTE]
> 
> Если вы устанавливаете Центр администрирования Windows 2019 на компьютере Windows Server 2019, это подсказит вам, что порт будет прослушиваться. Существует liklihood, который можно выбрать порт 443, но если на этой машине установлен Skype для бизнеса Server 2019 или на ней будет установлен Skype for Business Server 2019, необходимо выбрать другой номер порта.
> 
>Почему это так? Если Центр администрирования Windows 2019 работает в порту 443, вы не сможете подключиться к серверу с помощью панели управления Skype для бизнеса и не сможете подключиться к любой внутренней веб-службе, запущенной на сервере (веб-служба адресной книги, служба автообнаружия, служба webTicket и т.д.).  Фактически вы не сможете подключиться к любому URL-адресу внутренней веб-службы. Выберите другой порт в случае необходимости или необходимости поместить Центр администрирования Windows 2019 на сервер с Помощью Skype для бизнеса Server 2019.
>

### <a name="sql-clustering-and-sql-always-on"></a>SQL кластеризация и SQL Всегда на

SQL кластеризация с Помощью Skype для бизнеса Server 2019 поддерживается. Если вы хотите настроить SQL, это делается в SQL Server.
  
Убедитесь, что у вас есть активная/пассивная конфигурация SQL кластеризация, которая поддерживается. Не делитесь пассивным узлом с другим экземпляром SQL.
  
Для кластера неудачных сбойов можно использовать следующее:
  
Два узла:
  
- Microsoft SQL Server 2019 Standard (64-битное издание), и мы рекомендуем работать с последним пакетом служб.
- Microsoft SQL Server стандарт 2017 (64-битная версия), и мы рекомендуем работать с последним пакетом служб.
- Microsoft SQL Server 2016 Standard (64-битное издание), и мы рекомендуем работать с последним пакетом служб.

Шестнадцать узлов:
  
- Microsoft SQL Server 2019 Корпоративный (64-битный выпуск), и мы рекомендуем работать с последним пакетом служб.
- Microsoft SQL Server 2017 Корпоративный (64-битный выпуск), и мы рекомендуем работать с последним пакетом службы.
- Microsoft SQL Server 2016 Enterprise (64-битное издание), и мы рекомендуем работать с последним пакетом служб.

SQL Всегда поддерживается, и вы можете узнать больше об этом в сервере Back End Server с высокой доступностью в [Skype для бизнеса Server 2019](../../SfbServer/plan-your-deployment/high-availability-and-disaster-recovery/back-end-server.md).
  

###  <a name="additional-server-installation-recommendations"></a>Дополнительные рекомендации по установке сервера:
  
Пожалуйста, не устанавливайте программное обеспечение клиента Microsoft Internet Security and Acceleration (ISA) Server или любого другого программного обеспечения для поставщиков многоуровневой службы Winsock (LSP) (любые сторонние брандмауэры или программное обеспечение проверки антивирусной сети будут включены здесь) на любой из ваших серверов переднего или автономных серверов-посредников. При установке этого программного обеспечения была замечена низкая производительность трафика мультимедиа.

Важно знать, что Skype для бизнеса Server 2019 не поддерживает домены с одними метами. Что это такое? Если у вас есть корневой домен с меткой contoso.local, это будет нормально. Наконец, вы можете иметь дело с доменом с заблокированной средой AD DS, и это нормально. Дополнительные сведения о развертывании Skype для бизнеса Server 2019 в заблокированной среде AD DS в документации по развертыванию.

#### <a name="single-forest-with-single-domain"></a>один лес с одним доменом;

![Схема единого леса Active Directory с одним доменом](../../SfbServer/media/24921a0b-3a3e-4bad-8427-49300e2e3f7a.png)
  
Это не становится проще, чем это; это один лес домена, общая топология.
  
#### <a name="single-forest-with-a-single-tree-and-multiple-domains"></a>один лес с одним деревом и несколькими доменами;

![Диаграмма единого леса, единого дерева и mutiple доменов](../../SfbServer/media/63b9f0dd-6bac-4ba9-ae68-8be032d09dcb.png)
  
На этой схеме снова показан один лес, но в нем также есть один или несколько детских доменов (в этом конкретном примере три). Таким образом, созданный пользователями домен может быть отличается от домена, в который развернут Skype для бизнеса Server 2019. Зачем беспокоиться об этом? Важно помнить, что при развертывании пула переднего конца Skype для бизнес-сервера все серверы в этом пуле должны быть в одном домене. Администрирование с помощью Skype для бизнеса Server можно использовать для поддержки универсальных групп администраторов Windows.
  
На приведенной выше схеме видно, что пользователи из одного домена могут получать доступ к пулам Skype для бизнеса Server из одного домена или из разных доменов, даже если эти пользователи находятся в детском домене.
  
#### <a name="single-forest-with-multiple-trees-and-disjoint-namespaces"></a>один лес с несколькими деревьями и несвязанными пространствами имен;

Возможно, у вас есть топология, похожая на эту схему, где у вас есть один лес, но в этом лесу находятся несколько доменов с отдельными пространствами имен AD. В этом случае эта схема является хорошей иллюстрацией, так как включает пользователей в трех различных доменах, которые имеют доступ к Skype для бизнеса Server 2019. Сплошные строки указывают, что они имеют доступ к пулу Skype для бизнеса Server в собственном домене, в то время как пунктирная строка указывает, что они собираются в пул в другом дереве вообще.
  
Как видите, пользователи одного домена, одного дерева или даже другого дерева могут успешно получать доступ к пулам.
  
#### <a name="multiple-forests-in-a-central-forest-topology"></a>несколько лесов в топологии с центральным лесом;

![Несколько лесов на схеме топологии центрального леса](../../SfbServer/media/fec40746-4254-4c84-86b9-aad4a616ea2f.png)
  
Skype for Business Server 2019 поддерживает несколько лесов, настроенных в центральной топологии леса. Если вы не уверены, что это то, что у вас есть, центральный лес в топологии использует объекты в нем, чтобы представлять пользователей в других лесах, и размещены учетные записи пользователей для всех пользователей в лесу.
  
Как это работает? Продукт синхронизации каталогов (например, Forefront Identity Manager или FIM) управляет учетной записью пользователей организации на протяжении всего их существования. Когда учетная запись создается или удаляется из леса, это изменение синхронизируется до соответствующего контакта в центральном лесу.
  
Очевидно, что если инфраструктура AD на месте, переход на эту топологию может оказаться непростой задачей, но если вы уже там или все еще планируете свою лесную инфраструктуру, это может быть хорошим выбором. Вы можете централизовать развертывание Skype для бизнеса Server 2019 в одном лесу, а пользователи могут искать, общаться и просматривать присутствие других пользователей в любом лесу. Все обновления контактов пользователей обрабатываются автоматически с помощью программного обеспечения синхронизации.
  
#### <a name="multiple-forests-in-a-skype-for-business-resource-forest-topology"></a>Несколько лесов в топологии лесных ресурсов Skype для бизнеса
<a name="BKMK_multipleforestopology"> </a>

![Несколько лесов на схеме топологии леса ресурсов](../../SfbServer/media/41efa3b6-d9e6-47df-992b-fefcfc39a80d.png)
  
Также поддерживается топология лесных ресурсов; здесь лес предназначен для запуска серверных приложений, таких как Microsoft Exchange Server Skype для бизнеса Server 2019. В этом лесу ресурсов также размещено синхронизированное представление объектов активных пользователей, но нет учетных записей пользователей с поддержкой логотипа. Таким образом, лес ресурсов — это среда общих служб для других лесов, в которых находятся объекты пользователей, и они имеют отношение доверия уровня леса к лесу ресурсов.
  
Обратите внимание, Exchange Server можно развернуть в том же лесу ресурсов, что и Skype для бизнеса Server или в другом лесу.
  
Чтобы развернуть Skype для бизнеса Server 2019 в этом типе топологии, в лесу ресурсов для каждой учетной записи пользователя в лесах пользователей будет создаваться один отключенный объект пользователя (если Microsoft Exchange Server уже находится в среде, это может быть сделано для вас). Затем вам потребуется средство синхронизации каталогов (например, Forefront Identity Manager или FIM), чтобы управлять учетной записью пользователей в течение жизненного цикла.

## <a name="domain-name-system-dns"></a>Система доменных имен (DNS)
<a name="DNS"> </a>

Skype для бизнеса Server 2019 требует DNS по следующим причинам:
  
- DNS позволяет Skype для бизнеса Server 2019 открывать внутренние серверы или пулы, что позволяет использовать связь между серверами и серверами.
    
- DNS позволяет клиентские машины обнаружить пул переднего конца или сервер Standard Edition, используемый для SIP-транзакций.
    
- Он связывает простые URL-адреса для конференций с серверами, на которые размещены эти конференции.
    
- DNS позволяет внешним пользователям и клиентских машинам подключаться к вашим edge Servers или обратному прокси-серверу HTTP для обмена мгновенными сообщениями (IM) или конференциинга.
    
- Он позволяет устройствам единой связи (UC), которые не вошли в систему, обнаружить пул передней части или сервер Standard Edition, на которых работает веб-служба Обновления устройств, для получения обновлений и отправки журналов.
    
- Использование DNS позволяет мобильным клиентам автоматически открывать ресурсы веб-служб без необходимости вручную вводить URL-адреса в настройках устройств.
    
- Он используется для балансировки нагрузки DNS.
    
Важно отметить, что Skype для бизнеса Server 2019 не поддерживает интернационализированные доменные имена (IDNs).
  
И очень важно помнить, что любое имя в DNS идентично имени компьютера, настроенного на любом сервере, используемом Skype для бизнеса Server 2019. В частности, мы не можем иметь короткие имена в среде и должны иметь FQDNs для Topology Builder.
  
Это кажется логичным для любого компьютера, уже присоединились к домену, но если у вас есть edge Server, который не присоединился к вашему домену, он может иметь по умолчанию короткое имя, без суффикса домена. Убедитесь, что это не так, как в DNS или на краевом сервере, или любой сервер или пул Skype для бизнеса Server 2019, если на то пошло.
  
Определенно не используйте символы Юникод и не подчеркивайте. Стандартные символы (такие как A-Z, a-z, 0-9 и hyphens) поддерживаются внешними службами DNS и общедоступными сертификатами (необходимо назначить FQDN SN в сертификате, важно помнить), поэтому вы избавите себя от проблем, если назовете это с самого начала.
  
Дополнительные статьи о требованиях DNS к сетевой сети можно узнать в разделе [Networking](../../SfbServer/plan-your-deployment/network-requirements/network-requirements.md) документации по планированию.
  
## <a name="certificates"></a>Сертификаты
<a name="Certs"> </a>

Одна из самых важных вещей, которые вы можете сделать перед развертывание, это убедиться, что у вас есть сертификаты в порядке. Skype for Business Server 2019 нуждается в инфраструктуре общедоступных ключей (PKI) для обеспечения безопасности транспортного слоя (TLS) и обеспечения безопасности на взаимном транспортном уровне (MTLS). В принципе, для безопасного обмена данными стандартизированным способом Skype для бизнеса Server использует сертификаты, выдавлимые органами сертификации (CAs).
  
Вот некоторые из вещей, которые Skype для бизнеса Server 2019 использует сертификаты для:
  
- TLS-подключения между клиентами и серверами
    
- для подключений MTLS между серверами;
    
- для федерации с использованием автоматического обнаружения DNS партнеров;
    
- для доступа удаленных пользователей к обмену мгновенными сообщениями;
    
- Внешний доступ пользователей к сеансам аудио- и видеосвязи, совместному доступу к приложениям и конференциям
    
- Беседа с веб-приложениями и Веб-доступ Outlook (OWA)
    
Поэтому планирование сертификатов является обязательным. Теперь давайте рассмотрим список некоторых вещей, которые необходимо иметь в виду при запросе сертификатов:
  
- Все сертификаты серверов должны поддерживать авторизацию сервера (EKU сервера).
    
- Все сертификаты серверов должны содержать точку распространения списка отзыва сертификатов (CDP).
    
- Все сертификаты должны подписываться с помощью алгоритма подписи, поддерживаемых операционной системой. Skype for Business Server 2019 поддерживает наборы пакетов дайджестов SHA-1 и SHA-2 (224, 256, 384 и 512-бит) и отвечает требованиям операционной системы или превышает их.
    
- Автозарегистра поддерживается для внутренних серверов под управлением Skype для бизнеса Server 2019.
    
- Автоматическая регистрация не поддерживается для серверов Skype для бизнеса Server 2019 Edge Server.
    
> [!NOTE]
> Использование алгоритма подписи RSASSA-PSS неподтверждется и может привести к ошибкам при входе в систему и проблемам с переадпортом вызовов. 
  
- Поддерживаются длины ключей шифрования 1024, 2048 и 4096. Рекомендуется использовать длины ключей 2048 и более.
    
- Дайджест по умолчанию или подписание хаша — алгоритм RSA. Поддерживаются ECDH_P256, ECDH_P384 и ECDH_P521 алгоритмы.
    
Это много обдумать, и существует множество уровней комфорта с запросом сертификатов из ЦС. Ниже приведены дополнительные рекомендации, которые сделают планирование максимально безболезненно.
  
### <a name="certificates-for-your-internal-servers"></a>Сертификаты для внутренних серверов

Вам потребуется сертификаты для большинства внутренних серверов, и, скорее всего, вы получите их из внутреннего ЦС (это ЦС, расположенный в вашем домене). Если вы хотите, вы можете запросить эти сертификаты из внешнего ЦС (один из них расположен в Интернете). Если вам интересно, в какой общедоступный ЦС следует перейти, вы можете проверить список партнеров сертификата [единой](../../SfbPartnerCertification/certification/services-ssl.md) связи.
  
Вам также потребуется сертификат, когда Skype для бизнеса Server 2019 взаимодействует с другими приложениями и серверами, например с Microsoft Exchange Server. Очевидно, что это должен быть сертификат, который другие приложения и серверы могут использовать с поддержкой. Skype для бизнеса Server 2019 и другие продукты Майкрософт поддерживают протокол Open Authorization (OAuth) для проверки подлинности и авторизации от сервера к серверу. Если вы заинтересованы в этом, у нас есть дополнительная статья планирования для OAuth и Skype для бизнеса Server 2019.
  
Skype for Business Server 2019 также включает поддержку (без необходимости) сертификатов, подписанных с помощью функции криптографического хаша SHA-256. Для поддержки внешнего доступа с помощью SHA-256 внешний сертификат должен быть выдан общедоступным ЦС с помощью SHA-256.

### <a name="certificates-for-external-user-access-edge"></a>Сертификаты для внешнего доступа к пользователю (Edge)

Skype для бизнеса Server 2019 поддерживает  использование единого открытого сертификата для внешних интерфейсов доступа и веб-конференций Edge, а также службы проверки подлинности A/V, которая предоставляется через edge Server (s). Внутренний интерфейс Edge обычно использует частный сертификат, выданный внутренним ЦС, но если вы хотите, для этого можно использовать и общедоступный сертификат, если он из доверенного ЦС.
  
Обратный прокси-сервер (RP) также будет использовать общедоступный сертификат, который шифрует сообщение от RP до клиентов и RP на внутренние серверы с помощью HTTP (точнее, TLS над HTTP).
  
### <a name="certificates-for-mobility"></a>Сертификаты для мобильности

Если вы развертываете мобильность и поддерживаете автоматическое обнаружение для мобильных клиентов, вам потребуется включить дополнительные записи альтернативных имен субъекта в сертификаты для поддержки безопасных подключений от мобильных клиентов.
  
Для автоматического обнаружения в следующих сертификатах вам потребуется имя SAN:
  
- Пул директоров
    
- Интерфейсный пул
    
- Обратный прокси-сервер
    
Эти особенности перечислены в таблицах ниже.
  
Здесь хорошо немного предварительного планирования, но иногда вы развернули Skype для бизнеса Server 2019, не намереваясь развернуть мобильность, и это возникает позже, когда у вас уже есть сертификаты в среде. Переиздание их через внутренний ЦС, как правило, довольно просто, но с общедоступными сертификатами из общедоступных ЦС это может быть немного более pricy.
  
Если это то, что вы ищете, и если у вас много доменов SIP (что сделает добавление SANS более дорогим), вы можете настроить обратный прокси для использования HTTP для первоначального запроса службы автообнаружания, а не с помощью HTTPS (это конфигурация по умолчанию). В [статье Plan for Mobility](../../SfbServer/plan-your-deployment/mobility.md) дополнительные сведения об этом.

## <a name="file-share"></a>Файловый ресурс
<a name="Fileshare"> </a>

Skype для бизнеса Server 2019 может использовать один и тот же файл для всего хранилища файлов. Необходимо помнить следующее:
  
- Доля файлов должна быть на непосредственном присоединенном хранилище (DAS) или сети области хранения (SAN), и это включает распределенную файловую систему (DFS), а также избыточный массив независимых дисков (RAID) для файловых магазинов. Дополнительные книги по DFS для Windows Server 2012 можно узнать на этой странице [DFS.](/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/jj127250(v=ws.11))
    
- Рекомендуется общий кластер для общего файла. Если вы уже используете один, необходимо кластеризация Windows Server 2012 или более высоких версий

> [!Note]
> **Почему последняя Windows?** Более старые версии могут не иметь нужных разрешений, чтобы включить все функции. Для создания файловой доли можно использовать администратора кластера. Дополнительные сведения см. в этой статье Поддержка создания [файловой доли в кластере.](https://support.microsoft.com/help/224967)
    
> [!CAUTION]
> Вы должны знать, что использование сетевого подключенного хранилища (NAS) в качестве файла не поддерживается, поэтому используйте один из перечисленных выше вариантов. Это ограничение поддержки обусловлено переменной конструкцией устройств NAS, которые должны обеспечить адаптируемость файловой системы к компьютеру на базе Windows Server, который имеет доступ к общей файловой системе устройств.
