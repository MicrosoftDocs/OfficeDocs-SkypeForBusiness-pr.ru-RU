## <a name="hardware-for-skype-for-business-server-2019"></a>Оборудование для Skype для бизнеса Server 2019
<a name="Hardware"> </a>

После понижения топологии (а если вы не можете ознакомиться со статьей [основы топологии для Skype для бизнеса Server 2019](../../SfbServer/plan-your-deployment/topology-basics/topology-basics.md) ), следует подумать о серверах. Серверам Skype для бизнеса Server 2019 требуется 64-разрядное оборудование. Рекомендации по оборудованию приведены ниже. Они не являются обязательными, но они отражают требования, необходимые для достижения оптимальной производительности. Документация по планированию емкости поможет вам определить, нужно ли вам больше, в зависимости от обстоятельств.

## <a name="operating-systems-for-skype-for-business-server-2019"></a>Операционные системы для Skype для бизнеса Server 2019
<a name="OS"> </a>

После установки оборудования вам потребуется установить операционную систему (ОС), которая позволит вам установить и успешно использовать Skype для бизнеса Server 2019.
  
|||
|:-----|:-----|
|Windows Server 2019 <br/> |
|Windows Server 2016 <br/> ||
||
   
Все остальные операционные системы, перечисленные ниже, не будут работать должным образом; не используйте его для установки Skype для бизнеса Server 2019. Например, параметр Server Cores отсутствует в списке и поэтому не поддерживается.

> [!NOTE]
> Обновление на месте для ОС не поддерживается в Lync Server 2013. Необходимо развернуть отдельный пул и перенести пользователей в новый пул с другой операционной системой. Все серверы в пуле должны иметь одну и ту же версию ОС.

> [!NOTE]
> 
> Если вы устанавливаете центр администрирования Windows 2019 на компьютере с Windows Server 2019, он выводит запрос на прослушивание порта. Существует ликлихуд вы можете выбрать порт 443, но если на компьютере установлен Skype для бизнеса Server 2019 или на нем установлен Skype для бизнеса Server 2019, необходимо выбрать другой номер порта.
> 
>Почему это так? Если Windows Admin Center 2019 работает на порте 443, вы не сможете подключиться к серверу с помощью панели управления Skype для бизнеса, а также не сможете подключаться к внутренней веб-службе, запущенной на сервере (веб-служба адресной книги, служба автообнаружения, служба веб-билетов и т. д.).  На самом деле вы не сможете подключаться к внутреннему URL-адресу веб-службы. Выберите другой порт в нужном событии или необходимо разместить центр администрирования Windows 2019 на сервере с помощью Skype для бизнеса Server 2019.
>

### <a name="sql-clustering-and-sql-always-on"></a>Кластеризация SQL и SQL Always on

Поддерживается кластеризация SQL с Skype для бизнеса Server 2019. Если вы хотите настроить кластеризацию SQL, то это выполняется в SQL Server.
  
Убедитесь, что у вас есть активная/пассивная конфигурация для кластеризации SQL, которая поддерживается. Не предоставляйте пассивный узел к любому другому экземпляру SQL.
  
Для отказоустойчивой кластеризации можно выполнить указанные ниже действия.
  
Два узла:
  
- Microsoft SQL Server 2019 Standard (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.
- Microsoft SQL Server 2017 Standard (64-разрядный выпуск), и мы рекомендуем работать с последним пакетом обновления.

Очевидно, если ваша инфраструктура AD существует, перемещение в эту топологию может быть непростой задачей, но если у вас уже есть такая же учетная возможность или при планировании инфраструктуры леса, это может быть хорошим выбором. Вы можете централизовать развертывание Skype для бизнеса Server 2019 в пределах одного леса, в то время как пользователи могут выполнять поиск, Обмен информацией и Просмотр сведений о присутствии других пользователей в любом лесу. Все обновления контактов пользователей автоматически обрабатываются с помощью программного обеспечения для синхронизации.

## <a name="domain-name-system-dns"></a>Система доменных имен (DNS)
<a name="DNS"> </a>

Для работы с Skype для бизнеса Server 2019 требуется DNS по следующим причинам:
  
- DNS позволяет Skype для бизнеса Server 2019 обнаруживать внутренние серверы или пулы, обеспечивая межсерверную связь.
    
- DNS позволяет клиентским компьютерам обнаруживать интерфейсный пул или сервер Standard Edition, используемый для транзакций SIP.
    
- Он связывает простые URL-адреса для конференций с серверами, на которых размещаются эти конференции.
    
- DNS позволяет внешним пользователям и клиентским компьютерам подключаться к пограничным серверам или обратному прокси-серверу HTTP для обмена мгновенными сообщениями или конференц-связи.
    
- Он позволяет устройствам Объединенных коммуникаций (UC), которые не вошли в систему, обнаруживать интерфейсный пул или сервер Standard Edition, на котором работает веб-служба обновления устройств, чтобы получать обновления и отправлять журналы.
    
- Использование DNS позволяет мобильным клиентам автоматически обнаруживать ресурсы веб-служб, не требуя, чтобы пользователи вручную вводили URL-адреса в своих параметрах устройства.
    
- Он используется в балансировке нагрузки на DNS.
    
Важно отметить, что Skype для бизнеса Server 2019 не поддерживает международные доменные имена (IDN).
  
Очень важно помнить, что любое имя в DNS должно совпадать с именем компьютера, настроенным на любом сервере, используемом Skype для бизнеса Server 2019. В частности, у нас не может быть коротких имен в среде и у них должно быть FQDN для построителя топологий.
  
Кажется, что он будет логическим для любого компьютера, уже присоединенного к домену, но если у вас есть пограничный сервер, который не присоединен к домену, для него может быть задано короткое имя без суффикса домена. Убедитесь, что это не так, как в DNS или на пограничном сервере или на любом сервере или в любом сервере Skype для бизнеса Server 2019.
  
В определенном случае не следует использовать символы Юникода или символы подчеркивания. Стандартные символы (A-Z, a-z, 0-9 и дефисы) поддерживаются внешними DNS и общедоступными центрами сертификации (вам потребуется назначить полное доменное имя для SN в сертификате, поэтому вы можете запоминать о большом объеме проблем, если вы придаете это значение с самого начала.
  
Для дальнейшего ознакомления с требованиями к DNS для работы в сети ознакомьтесь с разделом " [сети](../../SfbServer/plan-your-deployment/network-requirements/network-requirements.md) " документации по планированию.
  
## <a name="certificates"></a>Сертификаты
<a name="Certs"> </a>

Один из важнейших моментов, которые можно выполнить перед развертыванием, заключается в том, что у вас есть сертификаты по порядку. В Skype для бизнеса Server 2019 требуется инфраструктура открытых ключей (PKI) для подключений по протоколу TLS и взаимного подключения безопасности на уровне транспорта (MTLS). Для безопасного обмена сообщениями в стандартном случае Skype для бизнеса Server использует сертификаты, выданные центрами сертификации (CA).
  
Ниже приведены некоторые из возможностей, которые Skype для бизнеса Server 2019 использует сертификаты для:
  
- Подключения TLS между клиентами и серверами
    
- для подключений MTLS между серверами;
    
- для федерации с использованием автоматического обнаружения DNS партнеров;
    
- для доступа удаленных пользователей к обмену мгновенными сообщениями;
    
- Доступ внешних пользователей к сеансам аудио-и видеосвязи (AV), совместному использованию приложений и конференц-связи
    
- Беседы с веб-приложениями и Outlook Web Access (OWA)
    
Для этого требуется планирование сертификатов. Теперь рассмотрим список некоторых вещей, которые необходимо иметь в виду при запросе сертификатов.
  
- Все сертификаты серверов должны поддерживать авторизацию сервера (EKU сервера).
    
- Все сертификаты серверов должны содержать точку распространения списка отзыва сертификатов (CDP).
    
- Все сертификаты должны быть подписаны с помощью алгоритма подписи, поддерживаемого операционной системой. Skype для бизнеса Server 2019 поддерживает набор размеров дайджеста SHA-1 и SHA-2 (224, 256, 384 и 512), а также соответствует требованиям к операционной системе или превышает их.
    
- Автоматическая регистрация поддерживается для внутренних серверов, на которых работает Skype для бизнеса Server 2019.
    
- Автоматическая регистрация не поддерживается для пограничных серверов Skype для бизнеса Server 2019.
    
> [!NOTE]
> Использование алгоритма подписи RSASSA-PSS не поддерживается и может привести к ошибкам при входе в систему и переадресации вызовов, среди прочих проблем. 
  
- Поддерживаются длины ключей шифрования 1024, 2048 и 4096. Рекомендуется использовать ключи длиной 2048 и выше.
    
- Для дайджеста по умолчанию или хэш-подписи используется алгоритм RSA. Также поддерживаются алгоритмы ECDH_P256, ECDH_P384 и ECDH_P521.
    
Это очень важно, и существует множество наиболее комфортных уровней, запрашивающих сертификаты из центра сертификации. Ниже приведено несколько дальнейших рекомендаций по планированию как можно более безболезненных.
  
### <a name="certificates-for-your-internal-servers"></a>Сертификаты для внутренних серверов

Вам потребуются сертификаты для большинства внутренних серверов, и, скорее всего, они будут получены из внутреннего центра сертификации (ЦС, расположенного в вашем домене). При желании вы можете запросить эти сертификаты из внешнего центра сертификации (который расположен в Интернете). Если вы не хотите, чтобы получить доступ к общедоступному центру сертификации, вы можете ознакомиться со списком [партнеров единой системы обмена сообщениями](/SkypeForBusiness/certification/services-ssl) .
  
Кроме того, вам потребуется использовать сертификаты, когда Skype для бизнеса Server 2019 обменивается данными с другими приложениями и серверами, такими как Microsoft Exchange Server. Это, конечно же, должен быть сертификатом, который могут использовать другие приложения и серверы. Skype для бизнеса Server 2019 и другие продукты Майкрософт поддерживают протокол Open Authorization (OAuth) для проверки подлинности и авторизации "сервер-сервер". Если вы заинтересованы в этом, у нас есть дополнительная статья по планированию для OAuth и Skype для бизнеса Server 2019.
  
Skype для бизнеса Server 2019 также включает поддержку сертификатов (без необходимости), подписанных с помощью криптографического хэш-функции SHA-256. Для поддержки внешнего доступа с помощью SHA – 256 внешний сертификат должен быть выдан общедоступным центром сертификации с помощью SHA – 256.

### <a name="certificates-for-mobility"></a>Сертификаты для мобильной работы

При развертывании мобильности и поддержке автоматического обнаружения для мобильных клиентов необходимо включить в сертификаты дополнительные записи альтернативного имени субъекта для поддержки безопасных подключений мобильных клиентов.
  
Для автоматического обнаружения в следующих сертификатах необходимы имена SAN:
  
- Пул директоров
    
- Интерфейсный пул
    
- Обратный прокси-сервер
    
Конкретные особенности перечислены в приведенных ниже таблицах.
  
В этом состоит небольшое предварительное планирование, но иногда вы развернули Skype для бизнеса Server 2019, не выполняя развертывание мобильности, и оно поступает позже, если у вас уже есть сертификаты в вашей среде. Их повторное выдача с помощью внутреннего ЦС, как правило, довольно прост, но с общедоступными сертификатами из общедоступного центра сертификации, которые могут быть немного более затраты.

В статье [Plan for Mobility Service](../../SfbServer/plan-your-deployment/mobility.md) содержатся дополнительные сведения.

## <a name="file-share"></a>Файловый ресурс
<a name="Fileshare"> </a>

Skype для бизнеса Server 2019 может использовать один общий файловый ресурс для всех хранилищ файлов. Необходимо учитывать следующее:
  
- Файловый ресурс должен находиться в либо непосредственно подключенном хранилище (DAS), либо в сети хранения данных (SAN), и это включает в себя распределенную файловую систему (DFS), а также избыточный массив независимых дисков (RAID) для хранения файлов. Чтобы получить дополнительные сведения о DFS для Windows Server 2012, изучите [эту страницу DFS](https://technet.microsoft.com/library/jj127250.aspx).
    
- Рекомендуется использовать общий кластер для общего файлового ресурса. Если вы уже используете один из них, следует использовать Cluster для Windows Server 2012 или более поздней версии.

> [!Note]
> **Зачем нужна последняя версия Windows?** В более ранних версиях могут отсутствовать права на включение всех компонентов. Для создания общих файловых ресурсов можно использовать администратор кластера. В этой статье содержатся [инструкции по созданию общих файловых ресурсов на кластере](https://support.microsoft.com/help/224967) для получения дополнительных сведений.
    
> [!CAUTION]
> Вы должны знать, что использование подключенного к сети хранилища (NAS) в качестве общего файлового ресурса не поддерживается, поэтому используйте один из перечисленных выше вариантов. Это ограничение поддержки вызвано тем, что переменная схема устройств NAS, которые должны обеспечивать адаптируемость файловой системы, на компьютер с Windows Server, который получает доступ к общей файловой системе устройств.
