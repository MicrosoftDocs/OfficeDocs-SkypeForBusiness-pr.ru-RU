## <a name="hardware-for-skype-for-business-server-2019"></a>Оборудование для Skype для бизнеса Server 2019 г.

После окончания топологии (а если нет, вы можете проверить основные принципы топологии для [Skype для бизнеса Server 2019](../../SfbServer/plan-your-deployment/topology-basics/topology-basics.md) г.), пришло время подумать о серверах. Skype для бизнеса Server 2019 г. для серверов требуется 64-битное оборудование. Ниже приведены рекомендации по оборудованию. Это не требования, но они отражают требования, необходимые для оптимальной производительности. ## <a name="operating-systems-for-skype-for-business-server-2019"></a>Операционные системы Skype для бизнеса Server 2019 г.
<a name="OS"> </a>

После установки оборудования необходимо установить операционную систему (ОС), которая позволит установить и успешно использовать Skype для бизнеса Server 2019 г.
  
- Windows Server 2019 
- Windows Server 2016
   
Все, кроме перечисленных здесь операционных систем, не будет работать должным образом; Пожалуйста, не пытайтесь установить Skype для бизнеса Server 2019. Например, параметр Server Core не указан и поэтому не поддерживается.

> [!NOTE]
> Обновление ОС на месте не поддерживается с помощью Lync Server 2013. Необходимо развернуть отдельный пул и перенести пользователей в новый пул с другой ОС. Все серверы пула должны иметь ту же версию ОС.

> [!NOTE]
> 
> Если вы устанавливаете Windows Admin Center 2019 на компьютере Windows Server 2019, это подсказит вам, чтобы порт прослушивал. Существует liklihood можно выбрать порт 443, но если эта машина Skype для бизнеса Server 2019 установлена на нем, или будет установлена Skype для бизнеса Server 2019, то необходимо выбрать другой номер порта.
> 
>Почему это так? Если Windows Центр администрирования 2019 работает в порту 443, вы не сможете подключиться к серверу с помощью панели управления Skype для бизнеса, а также не сможете подключиться к какой-либо внутренней веб-службе, запущенной на сервере (веб-служба адресной книги, служба автообнаружия, webTicket Service и т.д.).  Фактически вы не сможете подключиться к любому URL-адресу внутренней веб-службы. Выберите другой порт в случае необходимости или необходимости Windows Центра администрирования 2019 на сервере с Skype для бизнеса Server 2019.
>

## <a name="back-end-databases-that-will-work-with-skype-for-business-server-2019"></a>Базы данных заднего Skype для бизнеса Server 2019 г.
<a name="DBs"> </a>

При установке Skype для бизнеса Server 2019 выпуск Standard 2019 г. SQL Server 2016 express (64-bit edition).

Skype для бизнеса Server 2019 выпуск Enterprise потребуются полные SQL Server, как указано ниже (только 64-битное издание; не используйте 32-битные выпуски):
  
- Microsoft SQL Server 2019 (64-битное издание), и вы должны запустить с последними обновлениями.
- Microsoft SQL Server 2017 (64-битное издание), и вы должны запустить с последними обновлениями.
- Microsoft SQL Server 2016 (64-битное издание), и вы должны запустить с последними обновлениями.

Если вы не видите SQL Server, который вы хотите использовать, перечисленные здесь, вы не можете использовать его.
  
> [!NOTE]
> Также необходимо установить SQL Server Reporting Services для роли Monitoring Server. 
  
### <a name="sql-clustering-and-sql-always-on"></a>SQL Кластеризация и SQL всегда

SQL Кластеризация с Skype для бизнеса Server 2019 г. поддерживается. Если вы хотите настроить SQL кластеризация, это делается в SQL Server.
  
Убедитесь, что у вас есть активная/пассивная конфигурация SQL кластеризация, которая поддерживается. Не делитесь пассивным узлом с другим экземпляром SQL.
  
Для кластера неудачных сбойов можно использовать следующее:
  
Два узла:
  
- Microsoft SQL Server 2019 Standard (64-битное издание), и мы рекомендуем работать с последним пакетом служб.
- Microsoft SQL Server 2017 standard (64-bit edition), и мы рекомендуем работать с последним пакетом служб.
- Microsoft SQL Server 2016 standard (64-bit edition), и мы рекомендуем работать с последним пакетом служб.

Шестнадцать узлов:
  
- Microsoft SQL Server 2019 Enterprise (64-битное издание), и мы рекомендуем работать с последним пакетом служб.
- Microsoft SQL Server 2017 Enterprise (64-битное издание), и мы рекомендуем работать с последним пакетом служб.
- Microsoft SQL Server 2016 Enterprise (64-битное издание), и мы рекомендуем работать с последним пакетом служб.

SQL Всегда поддерживается, и вы можете прочитать больше об этом в back End Server высокой доступности в [Skype для бизнеса Server 2019](../../SfbServer/plan-your-deployment/high-availability-and-disaster-recovery/back-end-server.md).

При установке этого программного обеспечения была замечена низкая производительность трафика мультимедиа.

Наконец, вы можете иметь дело с доменом с заблокированной средой AD DS, и это нормально. Дополнительные сведения о развертывании Skype для бизнеса Server 2019 в заблокированной среде AD DS в документации по развертыванию.

Это не становится проще, чем это; это один лес домена, общая топология.
  
#### <a name="single-forest-with-a-single-tree-and-multiple-domains"></a>один лес с одним деревом и несколькими доменами;

![Один лес, одно дерево и схема доменов mutiple.](../../SfbServer/media/63b9f0dd-6bac-4ba9-ae68-8be032d09dcb.png)
  
На этой схеме снова показан один лес, но в нем также есть один или несколько детских доменов (в этом конкретном примере три). Таким образом, созданный пользователями домен может быть отличается от домена, в который развернут Skype для бизнеса Server 2019. Зачем беспокоиться об этом? Возможно, у вас есть топология, похожая на эту схему, где у вас есть один лес, но в этом лесу находятся несколько доменов с отдельными пространствами имен AD. В этом случае эта схема является хорошей иллюстрацией, так как включает пользователей в трех различных доменах, которые имеют доступ к Skype для бизнеса Server 2019. Сплошные строки указывают, что они имеют доступ к пулу Skype для бизнеса Server в собственном домене, в то время как пунктирная строка указывает, что они собираются в пул в другом дереве вообще.
  
Как видите, пользователи одного домена, одного дерева или даже другого дерева могут успешно получать доступ к пулам.
  
#### <a name="multiple-forests-in-a-central-forest-topology"></a>несколько лесов в топологии с центральным лесом;

![Несколько лесов на схеме топологии центрального леса.](../../SfbServer/media/fec40746-4254-4c84-86b9-aad4a616ea2f.png)
  
Skype for Business Server 2019 поддерживает несколько лесов, настроенных в центральной топологии леса. Если вы не уверены, что это то, что у вас есть, центральный лес в топологии использует объекты в нем, чтобы представлять пользователей в других лесах, и размещены учетные записи пользователей для всех пользователей в лесу.
  
Как это работает? Продукт синхронизации каталогов (например, Forefront Identity Manager или FIM) управляет учетной записью пользователей организации на протяжении всего их существования. Когда учетная запись создается или удаляется из леса, это изменение синхронизируется до соответствующего контакта в центральном лесу.
  
Очевидно, что если инфраструктура AD на месте, переход на эту топологию может оказаться непростой задачей, но если вы уже там или все еще планируете свою лесную инфраструктуру, это может быть хорошим выбором. Вы можете централизовать развертывание Skype для бизнеса Server 2019 в одном лесу, а пользователи могут искать, общаться и просматривать присутствие других пользователей в любом лесу. Все обновления контактов пользователей обрабатываются автоматически с помощью программного обеспечения синхронизации.
  
#### <a name="multiple-forests-in-a-skype-for-business-resource-forest-topology"></a>Несколько лесов в топологии лесных ресурсов Skype для бизнеса
<a name="BKMK_multipleforestopology"> </a>

![Несколько лесов на схеме топологии леса ресурсов.](../../SfbServer/media/41efa3b6-d9e6-47df-992b-fefcfc39a80d.png)
  
Также поддерживается топология лесных ресурсов; здесь лес предназначен для запуска серверных приложений, таких как Microsoft Exchange Server Skype для бизнеса Server 2019. В этом лесу ресурсов также размещено синхронизированное представление объектов активных пользователей, но нет учетных записей пользователей с поддержкой логотипа. Таким образом, лес ресурсов — это среда общих служб для других лесов, в которых находятся объекты пользователей, и они имеют отношение доверия уровня леса к лесу ресурсов.

## <a name="domain-name-system-dns"></a>Служба доменных имен (DNS)
<a name="DNS"> </a>

Skype для бизнеса Server 2019 г. требуется DNS по следующим причинам:
  
- DNS позволяет Skype для бизнеса Server 2019 г. для обнаружения внутренних серверов или пулов, что позволяет использовать связь между серверами и серверами.
    
- DNS позволяет клиентских машинам открывать пул переднего выпуск Standard сервер, используемый для транзакций SIP.
    
- Он связывает простые URL-адреса для конференций с серверами, на которые размещены эти конференции.
    
- DNS позволяет внешним пользователям и клиентских машинам подключаться к вашим edge Servers или обратному прокси-серверу HTTP для обмена мгновенными сообщениями (IM) или конференциинга.
    
- Он позволяет устройствам единой связи(UC), которые не вошли в систему, обнаружить пул переднего выпуск Standard сервера, на который работает веб-служба Обновления устройств, для получения обновлений и отправки журналов.
    
- Использование DNS позволяет мобильным клиентам автоматически открывать ресурсы веб-служб без необходимости вручную вводить URL-адреса в настройках устройств.
    
- Он используется для балансировки нагрузки DNS.
    
Важно отметить, что Skype для бизнеса Server 2019 г. не поддерживает интернационализированные доменные имена (IDNs).
  
И очень важно помнить, что любое имя в DNS идентично имени компьютера, настроенного на любом сервере, используемом Skype для бизнеса Server 2019. В частности, мы не можем иметь короткие имена в среде и должны иметь FQDNs для Topology Builder.
  
Это кажется логичным для любого компьютера, уже присоединились к домену, но если у вас есть edge Server, который не присоединился к вашему домену, он может иметь по умолчанию короткое имя, без суффикса домена. Убедитесь, что это не так, как в DNS или на edge Server, или любой Skype для бизнеса Server 2019 или пула, если на то пошло.
  
Определенно не используйте символы Юникод и не подчеркивайте. Стандартные символы (такие как A-Z, a-z, 0-9 и hyphens) поддерживаются внешними службами DNS и общедоступными сертификатами (необходимо назначить FQDN SN в сертификате, важно помнить), поэтому вы избавите себя от проблем, если назовете это с самого начала.
  
Дополнительные статьи о требованиях DNS к сетевой сети можно узнать в разделе [Networking](../../SfbServer/plan-your-deployment/network-requirements/network-requirements.md) документации по планированию.
  
## <a name="certificates"></a>Сертификаты
<a name="Certs"> </a>

Одна из самых важных вещей, которые вы можете сделать перед развертывание, это убедиться, что у вас есть сертификаты в порядке. Skype для бизнеса Server 2019 г. для обеспечения безопасности транспортных слоев (TLS) и взаимной безопасности транспортного слоя (MTLS) необходимо создание инфраструктуры общедоступных ключей (PKI). В основном для безопасной связи стандартизированным способом Skype для бизнеса Server сертификаты, выдавлимые органами сертификации (CAs).
  
Вот некоторые из вещей, которые Skype для бизнеса Server 2019 использует сертификаты для:
  
- TLS-подключения между клиентами и серверами
    
- для подключений MTLS между серверами;
    
- для федерации с использованием автоматического обнаружения DNS партнеров;
    
- для доступа удаленных пользователей к обмену мгновенными сообщениями;
    
- Внешний доступ пользователей к сеансам аудио- и видеосвязи, совместному доступу к приложениям и конференциям
    
- Беседа с веб-приложениями и Outlook веб-доступом (OWA)
    
Поэтому планирование сертификатов является обязательным. Теперь давайте рассмотрим список некоторых вещей, которые необходимо иметь в виду при запросе сертификатов:
  
- Все сертификаты серверов должны поддерживать авторизацию сервера (EKU сервера).
    
- Все сертификаты серверов должны содержать точку распространения списка отзыва сертификатов (CDP).
    
- Все сертификаты должны подписываться с помощью алгоритма подписи, поддерживаемых операционной системой. Skype для бизнеса Server 2019 г. поддерживает набор пакетов дайджестов SHA-1 и SHA-2 (224, 256, 384 и 512-битных) и соответствует требованиям операционной системы или превышает их.
    
- Автоматическая регистрация поддерживается для внутренних серверов, работающих Skype для бизнеса Server 2019 г.
    
- Автоматическая регистрация не поддерживается для Skype для бизнеса Server 2019 edge Servers.
    
> [!NOTE]
> Использование алгоритма подписи RSASSA-PSS неподтверждется и может привести к ошибкам при входе в систему и проблемам с переадпортом вызовов. 
  
- Поддерживаются длины ключей шифрования 1024, 2048 и 4096. Рекомендуется использовать длины ключей 2048 и более.
    
- Дайджест по умолчанию или подписание хаша — алгоритм RSA. Поддерживаются ECDH_P256, ECDH_P384 и ECDH_P521 алгоритмы.
    
Это много обдумать, и существует множество уровней комфорта с запросом сертификатов из ЦС. Ниже приведены дополнительные рекомендации, которые сделают планирование максимально безболезненно.
  
### <a name="certificates-for-your-internal-servers"></a>Сертификаты для внутренних серверов

Вам потребуется сертификаты для большинства внутренних серверов, и, скорее всего, вы получите их из внутреннего ЦС (это ЦС, расположенный в вашем домене). Если вы хотите, вы можете запросить эти сертификаты из внешнего ЦС (один из них расположен в Интернете). Если вам интересно, в какой общедоступный ЦС следует перейти, вы можете проверить список партнеров сертификата [единой](../../SfbPartnerCertification/certification/services-ssl.md) связи.
  
Кроме того, сертификаты будут нужны при Skype для бизнеса Server 2019 г. с другими приложениями и серверами, такими как Microsoft Exchange Server. Очевидно, что это должен быть сертификат, который другие приложения и серверы могут использовать с поддержкой. Skype для бизнеса Server 2019 г. и другие продукты Майкрософт поддерживают протокол Open Authorization (OAuth) для проверки подлинности и авторизации от сервера к серверу. Если вы заинтересованы в этом, у нас есть дополнительная статья планирования для OAuth и Skype для бизнеса Server 2019.
  
Skype для бизнеса Server 2019 г. также включает поддержку (без необходимости) сертификатов, подписанных с помощью функции криптографического хаша SHA-256. Для поддержки внешнего доступа с помощью SHA-256 внешний сертификат должен быть выдан общедоступным ЦС с помощью SHA-256.
  
Чтобы все было просто, мы поместили требования к сертификатам для выпуск Standard серверов, пулов переднего плана и других ролей в следующие таблицы, в примере используются вымышленные contoso.com (возможно, вы будете использовать что-то другое для вашей среды). Это все стандартные сертификаты веб-сервера с закрытыми ключами, которые не экспортируются. Некоторые дополнительные вещи, которые следует отметить:
  
- Использование расширенного ключа сервера (EKU) автоматически настраивается при использовании мастера сертификатов для запроса сертификатов.
    
- Каждое удобное имя сертификата должно быть уникальным в магазине компьютера.

### <a name="certificates-for-external-user-access-edge"></a>Сертификаты для внешнего доступа к пользователю (Edge)

Skype для бизнеса Server 2019 г. поддерживает использование  единого открытого сертификата для внешних интерфейсов edge доступа и веб-конференций, а также службы проверки подлинности A/V, которая предоставляется через edge Server(s). Внутренний интерфейс Edge обычно использует частный сертификат, выданный внутренним ЦС, но если вы хотите, для этого можно использовать и общедоступный сертификат, если он из доверенного ЦС.
  
Обратный прокси-сервер (RP) также будет использовать общедоступный сертификат, который шифрует сообщение от RP до клиентов и RP на внутренние серверы с помощью HTTP (точнее, TLS над HTTP).
  
### <a name="certificates-for-mobility"></a>Сертификаты для мобильности

Если вы развертываете мобильность и поддерживаете автоматическое обнаружение для мобильных клиентов, вам потребуется включить дополнительные записи альтернативных имен субъекта в сертификаты для поддержки безопасных подключений от мобильных клиентов.
  
Для автоматического обнаружения в следующих сертификатах вам потребуется имя SAN:
  
- Пул директоров
    
- Интерфейсный пул
    
- Обратный прокси-сервер
    
Эти особенности перечислены в таблицах ниже.
  
Здесь хорошо немного предварительного планирования, но иногда вы развернули Skype для бизнеса Server 2019, не намереваясь развернуть мобильность, и это возникает позже, когда у вас уже есть сертификаты в среде. Переиздание их через внутренний ЦС, как правило, довольно просто, но с общедоступными сертификатами из общедоступных ЦС это может быть немного более pricy.
  
Если это то, что вы ищете, и если у вас много доменов SIP (что сделает добавление SANS более дорогим), вы можете настроить обратный прокси для использования HTTP для первоначального запроса службы автообнаружания, а не с помощью HTTPS (это конфигурация по умолчанию). В [статье Plan for Mobility](../../SfbServer/plan-your-deployment/mobility.md) дополнительные сведения об этом.
  
Требования к пулу директоров и сертификату переднего конечных пула:
  
|Описание|Запись SAN|
|:-----|:-----|
|URL-адрес внутренней службы автооткрытия   |SAN=lyncdiscoverinternal.\<sipdomain\>   |
|URL-адрес службы внешней автооткрытия   |SAN=lyncdiscover.\<sipdomain\>   |
   
Можно также использовать SAN= \* .\<sipdomain\>
  
Требования к сертификату обратного прокси-сервера (public CA):
  
|Описание|Запись SAN|
|:-----|:-----|
|URL-адрес службы внешней автооткрытия   |SAN=lyncdiscover.\<sipdomain\>   |
   
Этот SAN должен быть назначен сертификату, назначенного прослушивательу SSL на обратном прокси-сервере.
  
> [!NOTE]
> Обратный прокси-прослушиватель будет иметь SANs для внешнего URL-адреса веб-служб. В качестве некоторых примеров можно привести SAN=skypewebextpool01.contoso.com и dirwebexternal.contoso.com, если вы развернули director (это необязательно). 
  
## <a name="file-share"></a>Файловый ресурс
<a name="Fileshare"> </a>

Skype для бизнеса Server 2019 г. можно использовать один и тот же файл для всего хранилища файлов. Необходимо помнить следующее:
  
- Доля файлов должна быть на непосредственном присоединенном хранилище (DAS) или сети области хранения (SAN), и это включает распределенную файловую систему (DFS), а также избыточный массив независимых дисков (RAID) для файловых магазинов. Дополнительные чтения на DFS для Windows Server 2012, ознакомьтесь [с этой страницей DFS](/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/jj127250(v=ws.11)).
    
- Рекомендуется общий кластер для общего файла. Если вы уже используете один из них, следует Windows Server 2012 или более высокие версии

> [!Note]
> **Почему последний Windows?** Более старые версии могут не иметь нужных разрешений, чтобы включить все функции. Для создания файловой доли можно использовать администратора кластера. Дополнительные сведения см. в этой статье Поддержка создания [файловой доли в кластере.](https://support.microsoft.com/help/224967)
    
> [!CAUTION]
> Вы должны знать, что использование сетевого подключенного хранилища (NAS) в качестве файла не поддерживается, поэтому используйте один из перечисленных выше вариантов. Это ограничение поддержки обусловлено переменной конструкцией устройств NAS, которые должны обеспечить адаптируемость файловой системы к компьютеру на Windows серверу, который имеет доступ к общей файловой системе устройств.
