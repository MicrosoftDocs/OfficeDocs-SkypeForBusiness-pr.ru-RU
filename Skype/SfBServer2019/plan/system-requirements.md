## <a name="hardware-for-skype-for-business-server-2019"></a>Оборудование для Skype для бизнеса Server 2019

Для серверов Skype для бизнеса Server 2019 требуется 64-битное оборудование. Ниже приведены наши рекомендации по оборудованию. Они не являются требованиями, но отражают требования, необходимые для оптимальной производительности. > [!NOTE]
> Обновление ос на месте не поддерживается в Lync Server 2013. Необходимо развернуть отдельный пул и перенести пользователей в новый пул с другой ОС. Все серверы в пуле должны иметь одинаковые версии ОС.

> [!NOTE]
> 
> Если вы устанавливаете Windows Admin Center 2019 на компьютере с Windows Server 2019, вам будет предложено прослушать порт. Вы можете выбрать порт 443, но если на этом компьютере установлен Skype для бизнеса Server 2019 или на нем будет установлен Skype для бизнеса Server 2019, необходимо выбрать другой номер порта.
> 
>Почему это так? Если Windows Admin Center 2019 работает на порту 443, вы не сможете подключиться к серверу с помощью панели управления Skype для бизнеса и не сможете подключиться к внутренней веб-службе, запущенной на сервере (веб-служба адресной книги, служба автообнаружия, служба WebTicket и т. д.).  Фактически, подключение к URL-адресу внутренней веб-службы будет не удастся. Выберите другой порт в случае необходимости или необходимости поместить Windows Admin Center 2019 на сервер со Skype для бизнеса Server 2019.
>

### <a name="sql-clustering-and-sql-always-on"></a>SQL кластеризация и SQL Always On

SQL кластеризация с помощью Skype для бизнеса Server 2019 поддерживается. Если вы хотите настроить SQL кластеров, это делается в SQL Server.
  
Убедитесь, что у вас есть активная/пассивная конфигурация для SQL кластеров, которая поддерживается. Не делитесь пассивным узлом с любым другим экземпляром SQL.
  
Для отбойной кластерной кластерии можно использовать следующие следующую следующую группу:
  
Два узла:
  
- Microsoft SQL Server 2019 Standard (64-битная версия), рекомендуется использовать последний пакет обновления.
- Microsoft SQL Server 2017 Standard (64-битная версия), рекомендуется использовать последний пакет обновления.
- Microsoft SQL Server 2016 Standard (64-битная версия), и мы рекомендуем использовать последний пакет обновления.

Шестнадцать узлов:
  
- Microsoft SQL Server 2019 Enterprise (64-битная версия), рекомендуется использовать последний пакет обновления.
- Microsoft SQL Server 2017 Enterprise (64-битная версия), рекомендуется использовать последний пакет обновления.
- Microsoft SQL Server 2016 Enterprise (64-битная версия), рекомендуется использовать последний пакет обновления.

SQL поддерживается Always On, и вы можете узнать больше об этом в skype для бизнеса [Server 2019.](../../SfbServer/plan-your-deployment/high-availability-and-disaster-recovery/back-end-server.md)
  

###  <a name="additional-server-installation-recommendations"></a>Дополнительные рекомендации по установке сервера:
  
Не устанавливайте клиентские программы Microsoft Internet Security and Acceleration (ISA) Server или любое другое программное обеспечение многоуровневой службы Winsock (LSP) (здесь будут включены какие-либо сторонние брандмауэры или антивирусные программы для проверки сети) на любые серверы переднего уровня или автономные серверы-посредники. При установке этого программного обеспечения была замечена низкая производительность трафика мультимедиа.

- Windows Server 2016

Функциональный уровень домена любого домена, в который развертывается Skype для бизнеса Server 2019, и функциональный уровень леса любого леса, в который развертывается Skype для бизнеса Server 2019, должен быть одним из следующих:
  
- Windows Server 2016

- Windows Server 2012
    
Можно ли использовать в этих средах контроллеры домена только для чтения? Конечно, при наличии доступных также доступных для написи контроллеров домена.
  
Важно знать, что Skype для бизнеса Server 2019 не поддерживает домены с одним меткой. Что это такое? Если у вас есть корневой домен с меткой contoso.local, это будет нормально. Если у вас есть корневой домен с именем local, он не будет работать и в результате не будет поддерживаться. Наконец, вы можете работать с доменом в заблокированной среде AD DS, и это нормально. Дополнительные сведения о развертывании Skype для бизнеса Server 2019 в заблокированной среде AD DS можно найти в документации по развертыванию.

Skype для бизнеса Server 2019 поддерживает несколько лесов, настроенных в топологии центрального леса. Если вы не уверены в том, что это так, в центральном лесу в топологии используются объекты для представления пользователей в других лесах и размещены учетные записи пользователей для всех пользователей в лесу.
  
Как это работает? Продукт синхронизации каталогов (например, Forefront Identity Manager или FIM) управляет учетной записью пользователя организации на протяжении всего их существования. Когда учетная запись создается или удаляется из леса, это изменение синхронизируется с соответствующим контактом в центральном лесу.
  
Очевидно, что если инфраструктура AD уже существует, переход к этой топологии может оказаться непростой задачей, но если вы уже там или планируете использовать инфраструктуру леса, это может оказаться хорошим выбором. Развертывание Skype для бизнеса Server 2019 можно централизовать в одном лесу, а пользователи могут искать, общаться и просматривать присутствие других пользователей в любом лесу. Все обновления контактов пользователей автоматически обрабатываются с помощью программного обеспечения синхронизации.

<a name="BKMK_multipleforestopology"> </a>

![Схема топологии с несколькими лесами в лесу ресурсов](../../SfbServer/media/41efa3b6-d9e6-47df-992b-fefcfc39a80d.png)
  
Также поддерживается топология леса ресурсов; Именно здесь лес выделен для запуска серверных приложений, таких как Microsoft Exchange Server и Skype для бизнеса Server 2019. ## <a name="domain-name-system-dns"></a>Система доменных имен (DNS)
<a name="DNS"> </a>

Skype для бизнеса Server 2019 требует DNS по следующим причинам:
  
- Служба DNS позволяет Skype для бизнеса Server 2019 обнаруживая внутренние серверы или пулы, что позволяет использовать связь между серверами.
    
- DNS позволяет клиентские компьютеры обнаруживать пул переднего сервера или сервер Standard Edition, используемый для транзакций SIP.
    
- Он связывает простые URL-адреса для конференций с серверами, на которые размещены эти конференции.
    
- Служба DNS позволяет внешним пользователям и клиентские компьютеры подключаться к вашим серверам или обратному прокси-серверу HTTP для обмена мгновенными сообщениями или для обмена мгновенными сообщениями.
    
- Это позволяет устройствам объединенных коммуникаций, которые не вошли в систему, обнаруживая пул переднего сервера или сервер Standard Edition, на которых запущена веб-служба обновления устройств, получать обновления и отправлять журналы.
    
- Использование DNS позволяет мобильным клиентам автоматически обнаруживать ресурсы веб-служб без необходимости ввода URL-адресов вручную в параметрах устройства.
    
- Он используется для балансировки нагрузки на DNS.
    
Важно отметить, что Skype для бизнеса Server 2019 не поддерживает международные доменные имена (IDNs).
  
Кроме того, очень важно помнить, что любое имя в DNS совпадает с именем компьютера, настроенным на любом сервере, используемом Skype для бизнеса Server 2019. В частности, у нас не может быть коротких имен в среде и должны быть FQDNs для построщика топологий.
  
Это кажется логическим для любого компьютера, уже присоединив его к домену, но если у вас есть сервер, который не присоединялся к вашему домену, он может иметь короткое имя по умолчанию без суффикса домена. Убедитесь, что это не так, в DNS или на edge Server, или на любом сервере или пуле Skype для бизнеса Server 2019.
  
Не используйте символы Юникода и подчеркиваия. Стандартные символы (A-Z, a-z, 0-9 и дефис) поддерживаются внешними службами DNS и общедоступными службами сертификации (вам потребуется назначить FQDN имени SN в сертификате, важно помнить), поэтому вы пощадите себя от многих проблем, если у вас будет это имя с самого начала.
  
Дополнительные статьи о требованиях к DNS для сети можно узнать в разделе ["Сеть"](../../SfbServer/plan-your-deployment/network-requirements/network-requirements.md) документации по планированию.
  
## <a name="certificates"></a>Сертификаты
<a name="Certs"> </a>

Одно из самых важных вещей, которые можно сделать перед развертыванием, — убедиться, что у вас есть сертификаты в порядке. Skype для бизнеса Server 2019 нуждается в инфраструктуре открытых ключей (PKI) для подключений TLS и MTLS. По сути, для безопасного обмена данными стандартизированным способом Skype для бизнеса Server использует сертификаты, выданные центром сертификации (ЦС).
  
Вот некоторые из того, для чего Skype для бизнеса Server 2019 использует сертификаты:
  
- Подключения TLS между клиентами и серверами
    
- для подключений MTLS между серверами;
    
- для федерации с использованием автоматического обнаружения DNS партнеров;
    
- для доступа удаленных пользователей к обмену мгновенными сообщениями;
    
- Доступ внешних пользователей к сеансам аудио- и видеосвязи, общий доступ к приложениям и аудиоконференциям
    
- Беседа с веб-приложениями и Outlook Web Access (OWA)
    
Поэтому планирование сертификатов является обязательным. Теперь рассмотрим список некоторых моментов, которые необходимо помнить при запросе сертификатов:
  
- Все сертификаты серверов должны поддерживать авторизацию сервера (EKU сервера).
    
- Все сертификаты серверов должны содержать точку распространения списка отзыва сертификатов (CDP).
    
- Все сертификаты должны быть подписаны с помощью алгоритма подписи, поддерживаемой операционной системой. Skype для бизнеса Server 2019 поддерживает наборы дайджест-размеров SHA-1 и SHA-2 (224, 256, 384 и 512 бит) и соответствует требованиям операционной системы или превышает их.
    
- Автоматическая регистрация поддерживается для внутренних серверов со Skype для бизнеса Server 2019.
    
- Автоматическая регистрация не поддерживается для серверов Skype для бизнеса Server 2019.
    
> [!NOTE]
> Использование алгоритма подписи RSASSA-PSS неподтвершено и, среди прочего, может привести к ошибкам при входе в систему и переададации вызовов. 
  
- Поддерживаются ключи шифрования длиной 1024, 2048 и 4096. Рекомендуется использовать ключи 2048 и большей длины.
    
- По умолчанию дайджест-алгоритмом или подписью с hash является RSA. Также ECDH_P256, ECDH_P384 и ECDH_P521 алгоритмы.
    
Это очень много обдумайте, и существует множество уровней удобства при запросе сертификатов из ЦС. Ниже приведены некоторые дополнительные рекомендации по максимальной безболезненной подготовке к планированию.
  
### <a name="certificates-for-your-internal-servers"></a>Сертификаты для внутренних серверов

Вам потребуется сертификаты для большинства внутренних серверов, и, скорее всего, вы получите их из внутреннего ЦС (который является ЦС, расположенный в вашем домене). При желании вы можете запросить эти сертификаты у внешнего ЦС (расположенного в Интернете). Если вы хотите узнать, в какой общедоступный ЦС следует перейти, ознакомьтесь со списком партнеров по сертификатам [unified Communications.](/SkypeForBusiness/certification/services-ssl)
  
Кроме того, вам потребуется сертификат, когда Skype для бизнеса Server 2019 взаимодействует с другими приложениями и серверами, например Microsoft Exchange Server. Очевидно, что это должен быть сертификат, поддерживаемый другими приложениями и серверами. Skype для бизнеса Server 2019 и другие продукты Майкрософт поддерживают протокол Open Authorization (OAuth) для проверки подлинности и авторизации между серверами. Если вы заинтересованы в этом, у нас есть дополнительная статья по планированию OAuth и Skype для бизнеса Server 2019.
  
Skype для бизнеса Server 2019 также включает поддержку (без необходимости) сертификатов, подписанных с помощью функции шифрования SHA-256. Для поддержки внешнего доступа с помощью SHA-256 внешний сертификат должен быть выдан общедоступным ЦС с помощью SHA-256.
  
Для простоты мы поместили требования к сертификатам для серверов Standard Edition, пулов переднего плана и других ролей в следующие таблицы, где в качестве примеров используется вымышленный contoso.com (скорее всего, вы будете использовать что-то другое для своей среды). - Расширенное использование ключа сервера (EKU) настраивается автоматически при использовании мастера сертификатов для запроса сертификатов.

### <a name="certificates-for-external-user-access-edge"></a>Сертификаты для доступа внешних пользователей (Edge)

Skype для бизнеса Server 2019 поддерживает  использование единого открытого сертификата для внешних интерфейсов доступа и веб-аудиоконференций, а также службу проверки подлинности аудио- и видеоконференций, которая предоставляется через серверы. Внутренний интерфейс Edge обычно использует частный сертификат, выданный внутренним ЦС, но при этом вы также можете использовать общедоступный сертификат, если он выдан доверенным ЦС.
  
Обратный прокси-сервер (RP) также будет использовать общедоступный сертификат и шифрует связь от RP к клиентам и RP на внутренние серверы по протоколу HTTP (или, точнее, по протоколу TLS через HTTP).
  
### <a name="certificates-for-mobility"></a>Сертификаты для мобильности

Если вы развертываете мобильность и поддерживаете автоматическое обнаружение для мобильных клиентов, вам потребуется включить в сертификаты дополнительные записи альтернативных имен субъектов для поддержки безопасных подключений от мобильных клиентов.
  
Имена SAN необходимы для автоматического обнаружения в следующих сертификатах:
  
- Пул директоров
    
- Интерфейсный пул
    
- Обратный прокси-сервер
    
Конкретные особенности перечислены в таблицах ниже.
  
В этом случае хорошо провести небольшое предварительное планирование, но иногда вы развернули Skype для бизнеса Server 2019, не планируя развертывание мобильности, и это происходит позже, когда у вас уже есть сертификаты в вашей среде. Повторное выдача их через внутренний ЦС обычно довольно проста, но с общедоступными сертификатами из общего ЦС это может быть немного дороже.
  
Если это то, что вы ищете, и если у вас много доменов SIP (что сделает добавление SANS более дорогим), можно настроить обратный прокси-сервер для использования HTTP для исходного запроса службы автообнаружния, а не https (конфигурация по умолчанию). Дополнительные [сведения об](../../SfbServer/plan-your-deployment/mobility.md) этом можно получить в статье "Планирование мобильности".
  
Требования к сертификатам пула директоров и пула переднего входа:
  
|**Описание**|**Запись SAN**|
|:-----|:-----|
|URL-адрес внутренней службы автонаружия  <br/> |SAN=lyncdiscoverinternal. \< sipdomain\>  <br/> |
|URL-адрес внешней службы автооружия  <br/> |SAN=lyncdiscover. \< sipdomain\>  <br/> |
   
Можно также использовать SAN= \* . \< sipdomain\>
  
Требования к сертификатам обратного прокси-сервера (public CA):
  
|**Описание**|**Запись SAN**|
|:-----|:-----|
|URL-адрес внешней службы автооружия  <br/> |SAN=lyncdiscover. \< sipdomain\>  <br/> |
   
Это SAN должно быть назначено сертификату, назначенного прослушивательУ SSL на обратном прокси-сервере.
  
> [!NOTE]
> Прослушиватель обратного прокси-сервера будет иметь SANs для URL-адресов внешних веб-служб. Some examples would be SAN=skypewebextpool01.contoso.com and dirwebexternal.contoso.com, if you've deployed the Director, (which is optional). 
  
## <a name="file-share"></a>Файловый ресурс
<a name="Fileshare"> </a>

Skype для бизнеса Server 2019 может использовать один и тот же файл для всех хранилищ файлов. Необходимо иметь в виду следующее:
  
- Файловый файл должен быть либо в непосредственно подключенном хранилище (DAS), либо в сети хранения данных (SAN), в том числе в распределенной файловой системе (DFS), а также в избыточном массиве независимых дисков (RAID) для хранилищ файлов. Для дальнейшего чтения на DFS для Windows Server 2012, ознакомьтесь с [этой страницей DFS](https://technet.microsoft.com/library/jj127250.aspx).
    
- Мы рекомендуем общий кластер для общей папки. Если вы уже используете его, следует кластерить Windows Server 2012 или более высоких версий

> [!Note]
> **Почему последняя версия Windows?** Более старые версии могут не иметь нужных разрешений для всех функций. Для создания файловой папки можно использовать администратора кластера. Дополнительные сведения см. в этой статье о создании файловой [папки в кластере.](https://support.microsoft.com/help/224967)
    
> [!CAUTION]
> Следует знать, что использование хранилища NAS в качестве файлового хранилища не поддерживается, поэтому используйте один из указанных выше вариантов. Это ограничение поддержки вызвано переменной конструкцией устройств NAS, которые должны обеспечить адаптируемость файловой системы к компьютеру с Windows Server, который имеет доступ к общей файловой системе устройств.
