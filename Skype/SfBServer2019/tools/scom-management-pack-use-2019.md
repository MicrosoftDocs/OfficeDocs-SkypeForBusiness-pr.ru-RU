# <a name="manage-skype-for-business-server-2019-using-scom-management-pack"></a>Управление Skype для бизнеса Server 2019 с помощью пакета управления SCOM
 
**Сводка:** В этой статье объясняется, как настроить инфраструктуру 2019 в Skype для бизнеса Server для работы с System Center Operations Manager.
  
В идеале вы никогда не столкнулись с проблемами в Skype для бизнеса Server 2019. Тем не менее, в Skype для бизнеса Server могут влиять внешние факторы, например сбои в работе сети и сбои оборудования. Используя пакеты управления для Skype для бизнеса Server 2019, вы можете легко определять и устранять потенциальные проблемы. Таким образом, пакеты управления для Skype для бизнеса Server 2019 расширяют возможности System Center Operations Manager.
  
Эти данные были написаны на основе версии 9319,0 пакета мониторинга для коммуникационного программного обеспечения Skype для Business Server 2019.
  
## <a name="configuration-overview"></a>Общие сведения о конфигурации

 Чтобы настроить инфраструктуру 2019 в Skype для бизнеса Server для работы с System Center Operations Manager, необходимо выполнить три действия:
  
Определение и [Configure the Primary Management Server](../../SfbServer/management-tools/use-scom-management-pack/configure-the-primary.md). Для этого необходимо установить System Center Operations Manager 2012 с пакетом обновления 1 (SP1) или R2. 
  
 Определите и [Настройте серверные компьютеры Skype для бизнеса, которые будут отслеживаться](../../SfbServer/management-tools/use-scom-management-pack/configure-computers-to-monitor.md). Для наблюдения за компьютером сервера Skype для бизнеса с помощью System Center Operations Manager необходимо установить файлы агента System Center Operations Manager и настроить каждый сервер для работы в качестве прокси-сервера. 
  
 Определите и [установите и настройте узлы наблюдателей](../../SfbServer/management-tools/use-scom-management-pack/watcher-nodes.md). Узлы-наблюдатели — это компьютеры, периодически запускающие синтетические транзакции в Skype для бизнеса Server — командлеты Windows PowerShell, которые проверяют, что ключевые компоненты сервера Skype для бизнеса (например, возможность входа в систему или возможности обмена мгновенными сообщениями) сообщения работают должным образом.

## <a name="whats-new"></a>Новые возможности

Ниже описаны новые возможности пакетов управления для Skype для бизнеса Server 2019.

- **Изменения в [обновлении за Сентябрь 2019](https://www.microsoft.com/download/details.aspx?id=57511) ** Некоторые оповещения содержат удаленные специальные символы. В некоторых случаях специальные символы конфликтуют с функцией уведомлений канала команд SCOM.

- **Автоматическое обнаружение для входа клиента** Клиентские приложения, которые входят в Skype для бизнеса Server 2019, часто автоматически обнаруживают сервер для входа. Теперь искусственные транзакции поддерживают проверку правильности настройки автоматического обнаружения.
    
- **Настроенные интервалы выполнения искусственной транзакции** Для упрощения процесса настройки узлов наблюдения синтетические транзакции могут предоставлять общий доступ к учетным записям пользователей. Это снижает периодичность проверок, так как во избежание конфликтов проверки выполняются последовательно. По умолчанию искусственные транзакции выполняются каждые 15 минут: этого обычно достаточно для выполнения всех проверок. Администраторы, увеличивающие количество пользователей или уменьшающие количество проверок на каждого пользователя, теперь могут также сократить интервал между проверками.
    
- **Синтетическая транзакция служб видеозвонка** Пользователи, которые переходят на Skype для бизнеса Server 2019 из других решений компании, часто хотят продолжать использовать устройства для видеоконференций (Вткс) из других поставщиков. Сервер видеосвязи — это новая серверная роль Skype для бизнеса Server 2019, позволяющая клиентам продолжать использовать Cisco Вткс в конференц-залах, подключивсь к КУКМу Cisco через канал видеомагистрали по видеосвязи. Эта функция предусматривает также новую искусственную транзакцию, позволяющую убедиться в том, что сервер видеовзаимодействия работает и в состоянии обрабатывать входящие подключения по магистральной линии видеосвязи SIP.
    
- **Искусственная транзакция конференц-связи с общим доступом к приложениям** Теперь поддерживается проверка конференций с общим доступом к приложениям на всех этапах.

При выполнении этих запросов формируются оповещения, указывающие на такие ситуации во время вызовов и конференций, в которых наиболее вероятно ухудшение качества мультимедийных ресурсов для пользователей. В основе этих данных лежат важнейшие показатели, непосредственно влияющие на качество взаимодействия с пользователем, такие как задержка и потеря пакетов.  <br/> |

Эти оповещения могут указывать на разнообразные ситуации, включая незапущенные службы, высокий уровень отказов, длительную задержку сообщений и неполадки подключения.  <br/> |

Пакет управления обеспечивает отслеживание и сбор данных о важнейших внешних зависимостях, которые могут указывать на серьезные неполадки. Эти зависимости включают сведения о доступности служб IIS и ЦП серверов, используемых для Skype для бизнеса Server.  <br/> |
   
### <a name="alert-prioritization"></a>Приоритет оповещений

Оповещения подразделяются на следующие категории. 
  
 **Оповещения с высоким приоритетом:** Эти оповещения указывают на условия, которые приводят к недостаточности обслуживания для больших групп пользователей и требуют немедленного вмешательства. Сообщения о нарушениях, обнаруженных искусственными транзакциями и автономными службами (например, в Skype для бизнеса Server аудио-и видеоконференции), определяются как высокоприоритетные уведомления. Однако оповещениям о сбое компонента на отдельном компьютере не назначается высокий приоритет. Skype для бизнеса Server 2019 имеет встроенные возможности высокой доступности для таких ситуаций, как, например, несколько серверов переднего плана за пределами подсистем балансировки нагрузки.
  
 **Оповещения со средним приоритетом:** Эти оповещения указывают на условия, влияющие на подмножество пользователей, и указывают на проблемы, связанные с качеством звонков (например, сбои компонентов, задержка во время установки звонка или низкое качество звука при звонках). Оповещения в этой категории представляют собой состояние (то есть изменения, вносимые оповещением, в зависимости от состояния сетевого подключения). Например, если во время установки нужно указать задержку, а затем вернуться к обычному пороговому значению, это оповещение со средним приоритетом будет автоматически разрешено в System Center Operations Manager, и администраторам не нужно предпринимать никаких действий. Оповещения, причина которых не может быть устранена автоматически, как правило, рассматриваются администраторами в течение рабоего дня.
  
 **Прочие оповещения.** Эти оповещения формируются в компонентах, которые могут повлиять на работу конкретного пользователя или нескольких пользователей. Типовым примером служит оповещение о невозможности синтаксического анализа записи доменных служб Active Directory для пользователя testuser@contoso.com с помощью службы адресной книги. Администраторы работают с такими оповещениями при наличии свободного времени.

Пул серверов в Skype для бизнеса Server может содержать несколько отдельных серверных систем Skype для бизнеса (с более чем одной ролью сервера Skype для бизнеса, службой Skype для бизнеса Server и компонентом Skype для бизнеса Server). Таким образом, сбой отдельного сервера или компонента менее важен для общей работоспособности пула сервера Skype для бизнеса, так как другие серверы в том же пуле могут предоставить клиенту услуги приложения. Последовательный сбой одной или нескольких искусственных транзакций (процесс, называемый последовательным интервалом опроса) выполняет сведение критического состояния работоспособности на уровне пула (наихудшее значение любой искусственной транзакции), как показано на следующей схеме.

## <a name="best-practice-create-a-management-pack-for-customizations"></a>Рекомендация. Создавайте пакеты управления для пользовательских конфигураций.

По умолчанию в Operations Manager сохраняются все пользовательские конфигурации, в том числе переопределения пакета управления по умолчанию. Рекомендуется для каждого запечатанного пакета управления, который требуется настроить, создать отдельный пакет управления. 
  
При создании пакета управления для хранения настроенных параметров для запечатанного пакета управления мы рекомендуем правильно называть новый пакет управления, например "Настройка Skype для бизнеса Server 2019".
  
Создание нового пакета управления для хранения наборов пользовательских параметров каждого запечатанного пакета управления упрощает экспорт наборов пользовательских параметров из ситуации испытаний в производственную среду. Упрощается также удаление пакета управления, поскольку перед этим необходимо удалить все зависимости. Если наборы пользовательских параметров для всех пакетов управления сохранены в пакете управления по умолчанию, для удаления отдельного пакета управления потребуется сначала удалить пакет управления по умолчанию; при этом будут потеряны изменения, внесенные в ходе пользовательской настройки других пакетов управления.
