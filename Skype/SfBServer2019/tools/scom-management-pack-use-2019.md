# <a name="manage-skype-for-business-server-2019-using-scom-management-pack"></a>Управление Skype для бизнеса Server 2019 с помощью пакета управления SCOM
 
**Сводка:** Узнайте, как настроить инфраструктуру Skype для бизнеса Server 2019 для работы с System Center Operations Manager.
  
В идеальном мире вы никогда не столкнулись с проблемами Skype для бизнеса Server 2019. Тем не менее, на Skype для бизнеса Server могут влиять внешние факторы, например сбои сети и сбои оборудования. С помощью пакетов управления Skype для бизнеса Server 2019 вы можете в активном состоянии определять и устранять потенциальные проблемы. Таким образом, пакеты управления Skype для бизнеса Server 2019 расширяют возможности System Center Operations Manager.
  
Эта информация была написана на основе версии 9319,0 пакета мониторинга для программного обеспечения для связи Skype для бизнеса Server 2019.
  
## <a name="configuration-overview"></a>Общие сведения о конфигурации

 Чтобы настроить инфраструктуру Skype для бизнеса Server 2019 для работы с System Center Operations Manager, необходимо выполнить три действия:
  
Определите и [Настройте основной сервер управления](../../SfbServer/management-tools/use-scom-management-pack/configure-the-primary.md). Для этого необходимо установить System Center Operations Manager 2012 с пакетом обновления 1 (SP1) или R2. 
  
 Определите и [настройте компьютеры Skype для бизнеса Server, которые будут отслеживаться](../../SfbServer/management-tools/use-scom-management-pack/configure-computers-to-monitor.md). Для наблюдения за компьютером Skype для бизнеса Server с помощью System Center Operations Manager необходимо установить файлы агента System Center Operations Manager и настроить каждый сервер для работы в качестве прокси-сервера. 
  
 Определение и [Установка и настройка узлов-наблюдателей](../../SfbServer/management-tools/use-scom-management-pack/watcher-nodes.md). Узлы-наблюдатели — это компьютеры, которые периодически выполняют искусственные транзакции Skype для бизнеса Server — командлеты Windows PowerShell, которые проверяют наличие ключевых компонентов Skype для бизнеса Server, таких как возможность входа в систему или возможность Exchange Instant сообщения работают должным образом.

## <a name="whats-new"></a>Новые возможности

Ниже приведены новые возможности пакетов управления Skype для бизнеса Server 2019.

- **Изменения в [обновлении за Сентябрь 2019](https://www.microsoft.com/download/details.aspx?id=57511) ** Некоторые оповещения удалили специальные символы. В некоторых случаях специальные символы мешают работе с функцией уведомления канала команд SCOM.

- **Автоматическое обнаружение для входа клиента** Клиентские приложения, которые входят в Skype для бизнеса Server 2019, часто автоматически обнаруживают сервер для входа в систему. Искусственные транзакции теперь поддерживают проверку правильности настройки автоматического обнаружения.
    
- **Интервалы выполнения настраиваемых искусственных транзакций** Для упрощения процесса настройки узлов-наблюдателей синтетические транзакции могут обмениваться учетными записями пользователей. Это замедляет частоту выполнения тестов, так как тесты сериализуются во избежание конфликтов. По умолчанию искусственные транзакции выполняются каждые 15 минут, чтобы обеспечить время выполнения всех тестов. Администраторы, которые могут использовать больше пользователей или меньше тестов для пользователя, теперь могут сократить интервалы выполнения.
    
- **Искусственная транзакция служб Видеовзаимодействия** Клиенты, которые переходят на Skype для бизнеса Server 2019 из других решений, часто захотите продолжить использовать устройства видеоконференций (VTC) от этих других поставщиков. Сервер видеовзаимодействия — это новая роль сервера Skype для бизнеса Server 2019, которая позволяет клиентам продолжать использовать Cisco VTC в конференц-залах, подключаясь к Cisco CUCM с помощью видеомагистрали SIP. Эта функция также добавляет искусственную транзакцию, чтобы убедиться в том, что сервер видеовзаимодействия работает и может обрабатывать входящие подключения через видеоканал SIP.
    
- **Искусственная транзакция конференц-связи с общим доступом к приложениям** Проверка сквозного сценария для конференций совместного использования приложений теперь поддерживается.

### <a name="alert-prioritization"></a>Приоритизация оповещений

Оповещения классифицируются по следующим категориям: 
  
 **Оповещения с высоким приоритетом:** Эти оповещения указывают на условия, вызывающие перебои в обслуживании для больших групп пользователей и требующие немедленные действия. Непростои, обнаруженные искусственными транзакциями и автономными службами (например, Skype для бизнеса Server аудио-и видеоконференцией), соответствуют высокоприоритетным оповещениям. В отличие от того, что сбой компонента на отдельном компьютере не является оповещением с высоким приоритетом. В Skype для бизнеса Server 2019 имеются встроенные функции высокого уровня доступности для таких ситуаций, например, несколько серверов переднего плана за пределами подсистем балансировки нагрузки.
  
 **Оповещения среднего приоритета:** Эти оповещения указывают на условия, влияющие на подмножество пользователей, или указывают на проблемы, связанные с качеством звонков, например сбой компонентов, задержка во время установки звонка или низкое качество звука в вызовах. Оповещения в этой категории являются состояниями (то есть характером изменений оповещений в зависимости от состояния сетевого подключения). Например, если во время установки необходимо указать задержку, а затем вернуться к обычному пороговому значению, это предупреждение среднего приоритета будет автоматически разрешено в System Center Operations Manager, а администраторам не нужно предпринимать действия. Оповещения, которые не могут быть автоматически разрешены, обычно решаются администраторами в рамках одного рабочего дня.
  
 **Другие оповещения:** Эти оповещения создаются на основе компонентов, которые могут повлиять на определенного пользователя или подмножества пользователей. Например, типичным предупреждением является то, что службе адресной книги не удалось выполнить синтаксический анализ записи Active Directory® доменных служб (AD DS) для пользователя: testuser@contoso.com. Администраторы могут обращаться к этим оповещениям при наличии достаточного времени.

Пул Skype для бизнеса Server может содержать несколько отдельных систем Skype для бизнеса Server (с более чем одной ролью Skype для бизнеса Server, службой Skype для бизнеса Server и компонентом Skype для бизнеса Server). Таким образом, сбой отдельного сервера или компонента менее критичен для общей работоспособности пула Skype для бизнеса Server, так как другие серверы в том же пуле могут предоставить клиенту службу приложения. Работоспособность будет сведена на процентный уровень в пул Skype для бизнеса Server. 
  
Наблюдатель пула Skype для бизнеса Server выполняет искусственные транзакции для пула Skype для бизнеса Server. Последовательный сбой одной или нескольких искусственных транзакций (процесс, называемый интервалом опроса) будет выполнять накат критического состояния работоспособности на уровне пула (наихудшее значение любой искусственной транзакции), как показано на следующей схеме.

## <a name="best-practice-create-a-management-pack-for-customizations"></a>Рекомендация: создание пакета управления для настроек

По умолчанию Operations Manager сохраняет все настройки, такие как переопределения пакета управления по умолчанию. Рекомендуется создать отдельный пакет управления для каждого запечатанного пакета управления, который необходимо настроить. 
  
При создании пакета управления для хранения настраиваемых параметров запечатанного пакета управления рекомендуется соответствующим образом называть новый пакет управления, например "Skype для бизнеса Server 2019 настроек".
  
Создание нового пакета управления для хранения настроек каждого запечатанного пакета управления упрощает экспорт настроек из тестовой среды в производственную среду. Это также упрощает удаление пакета управления, так как перед удалением пакета управления необходимо удалить все зависимости. Если настройки для всех пакетов управления сохраняются в пакете управления по умолчанию и необходимо удалить один пакет управления, необходимо сначала удалить пакет управления по умолчанию, который также удаляет настройки для других пакетов управления.
