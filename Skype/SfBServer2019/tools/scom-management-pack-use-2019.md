# <a name="manage-skype-for-business-server-2019-using-scom-management-pack"></a>Управление Skype для бизнеса Server 2019 с помощью пакета управления SCOM
 
**Сводка:** Узнайте, как настроить инфраструктуру Skype для бизнеса Server 2019 для работы с System Center Operations Manager.
  
В идеальном мире проблемы со Skype для бизнеса Server 2019 никогда не возникнут. Однако на Skype для бизнеса Server могут повлиять внешние факторы, например сбои сети и сбои оборудования. С помощью пакетов управления Skype для бизнеса Server 2019 можно выявлять потенциальные проблемы и решать их заранее. Таким образом пакеты управления Skype для бизнеса Server 2019 расширяют возможности System Center Operations Manager.
  
Эти сведения были написаны на основе версии 9319.0 пакета мониторинга для программного обеспечения связи Skype для бизнеса Server 2019.
  
## <a name="configuration-overview"></a>Общие сведения о конфигурации

 Чтобы настроить инфраструктуру Skype для бизнеса Server 2019 для работы с System Center Operations Manager, необходимо сделать три действия:
  
Определите [и настройте основной сервер управления.](../../SfbServer/management-tools/use-scom-management-pack/configure-the-primary.md) Для этого необходимо установить System Center Operations Manager 2012 с sp1 или R2. 
  
 Определите [и настройте компьютеры Skype для бизнеса Server, которые будут отслеживаться.](../../SfbServer/management-tools/use-scom-management-pack/configure-computers-to-monitor.md) Чтобы отслеживать компьютер Skype для бизнеса Server с помощью System Center Operations Manager, необходимо установить файлы агента System Center Operations Manager и настроить каждый сервер в качестве прокси-сервера. 
  
 [Определите, установите и настройте узлы-watcher.](../../SfbServer/management-tools/use-scom-management-pack/watcher-nodes.md) Узлы-watcher — это компьютеры, которые периодически запускают искусственные транзакции Skype для бизнеса Server— Windows PowerShell, которые проверяют, работают ли ключевые компоненты Skype для бизнеса Server, такие как возможность входа в систему или обмен мгновенными сообщениями.

## <a name="whats-new"></a>Новые возможности

Следующие функции являются новыми в пакетах управления Skype для бизнеса Server 2019.

- **Изменения в [обновлении за сентябрь 2019 г.](https://www.microsoft.com/download/details.aspx?id=57511)** В некоторых оповещениях были удалены специальные символы. В некоторых случаях специальные символы мешают функции уведомлений канала команд SCOM.

- **Автоматическое обнаружение для входов клиентов** Клиентские приложения, которые во время входов в Skype для бизнеса Server 2019 часто автоматически обнаруживают сервер для входов. Искусственные транзакции теперь поддерживают проверку правильности настройки автоматического обнаружения.
    
- **Настраиваемые интервалы запуска искусственных транзакций** Чтобы упростить процесс создания узлов-watcher, искусственные транзакции могут совместно использовать учетные записи пользователей. Это замедляет частоту запуска тестов по мере сериализации тестов во избежание конфликтов. По умолчанию искусственные транзакции запускают каждые 15 минут, чтобы убедиться, что все тесты имеют время для запуска. Администраторы, которые решили использовать больше пользователей или меньшее количество тестов для каждого пользователя, также могут сократить интервалы запуска.
    
- **Искусственная транзакция служб видеосвязи** Клиенты, которые переходят на Skype для бизнеса Server 2019 из решений других поставщиков, часто хотят продолжать использовать устройства видеоконференции (VTCs) от этих других поставщиков. Сервер видеосвязи — это новая роль сервера Skype для бизнеса Server 2019, которая позволяет клиентам продолжать использовать cisco VTCs в конференц-залах, подключаясь к Cisco CUCM через видеомаголь SIP. Эта функция также добавляет искусственную транзакцию, чтобы убедиться, что сервер видеосвязи подключен к сети и может обрабатывать входящие подключения через магистраль SIP видео.
    
- **Искусственная транзакция с совместным использованием приложений** Теперь поддерживается проверка всех сценариев для конференций с совместным использованием приложений.
    
## <a name="monitoring-scenarios"></a>Сценарии мониторинга

Пакет управления Skype для бизнеса Server 2019 management Pack использует различные функции для обнаружения и диагностики проблем. Эти функции обеспечивают видимость состояния среды Skype для бизнеса Server 2019 в режиме реального времени.
  
|**Сценарий мониторинга**|**Описание**|
|:-----|:-----|
|Искусственные транзакции  <br/> | Windows PowerShell для тестирования и обеспечения высокой доступности таких сценариев, как вход, присутствие, мгновенные извещающие и видеоконференция для пользователей. <br/> Искусственные транзакции можно запускать из любого географического расположения, в том числе внутри предприятия, за пределами предприятия и в филиалах.  <br/> При сбое искусственной транзакции создаются журналы HTML, чтобы определить точный характер сбоя. Это включает в себя понимание того, какое действие не удалось выполнить, задержку каждого действия, командную строку, используемую для запуска теста, и определенную возникную ошибку.  <br/> |
|Оповещения о надежности вызовов  <br/> |Записи cdrs, написанные серверами Skype для бизнеса Server 2019, отражают, могут ли пользователи подключаться к звонкам или почему звонок прерывается. Оповещения о надежности звонков запрашивают базу данных CDR для получения оповещений, которые указывают, когда большое количество пользователей испытывают проблемы с подключением для одноранговых вызовов или основных функций конференций.  <br/> Охват сценария включает в себя аудиозвуки, одноранговые мгновенные сообщения и другие функции аудиоконференций.  <br/> |
|Оповещения о качестве мультимедиа  <br/> |Запросы к базам данных, которые смотрят на отчеты о качестве обслуживания (QoE), опубликованные клиентами Skype для бизнеса Server 2019 в конце каждого вызова. Эти запросы создают оповещения, которые выявляют сценарии, в которых пользователи с наибольшей вероятностью будут скомпрометировать качество мультимедиа во время звонков и конференций. Данные построены на ключевых метриках, таких как задержка и потеря пакетов, которые непосредственно способствуют качеству работы пользователя.  <br/> |
|Оповещения о состоянии компонентов  <br/> |Отдельные серверные компоненты вызывают оповещения с помощью журналов событий и счетчиков производительности, чтобы указать условия сбоя, которые могут значительно повлиять на сценарии пользователей. Эти оповещения указывают на различные условия, такие как не запущенные службы, высокий уровень сбоев, высокая задержка сообщений или проблемы с подключением.  <br/> |
|Мониторинг состояния зависимостей  <br/> |Skype для бизнеса Server может не сбой по ряду внешних причин. Пакет управления отслеживает и собирает данные для критически важных внешних зависимостей, которые могут указывать на серьезные проблемы. Эти зависимости включают доступность служб IIS и ЦП серверов, используемых для Skype для бизнеса Server.  <br/> |
   
### <a name="alert-prioritization"></a>Приоритет оповещений

Оповещения классифицируются по следующим категориям: 
  
 **Оповещения с высоким приоритетом:** Эти оповещения указывают условия, которые приводят к отключению служб для больших групп пользователей и требуют немедленного действия. Отключения, обнаруженные искусственные транзакции и автономные службы (например, аудио- и видеоконференции Skype для бизнеса Server), квалифицировать как оповещения с высоким приоритетом. С другой стороны, сбой компонента на одном компьютере не является оповещением с высоким приоритетом. В Skype для бизнеса Server 2019 есть встроенные функции высокой доступности для этих ситуаций, например несколько серверов переднего сервера за подстройки нагрузки.
  
 **Оповещения среднего приоритета:** Эти оповещения указывают условия, которые влияют на подмножество пользователей или указывают на проблемы с качеством звонков, например сбои компонентов, задержку при установке вызова или снижение качества звука в вызовах. Оповещения в этой категории с учетом состояния (то есть характер изменений оповещений в зависимости от состояния сетевого подключения).) Например, если время установки вызова указывает на задержку, но затем возвращается к обычному пороговом значению, это оповещение с средним приоритетом будет автоматически разрешено в System Center Operations Manager, и администраторам не потребуется действовать. Оповещения, которые не могут быть разрешены автоматически, обычно решаются администраторами в один и тот же день работы.
  
 **Другие оповещения:** Эти оповещения создаются из компонентов, которые могут повлиять на конкретного пользователя или подмножество пользователей. Например, типичным оповещением может быть то, что службе адресной книги не удалось разлить запись доменных служб Active Directory® (AD DS) для пользователя: testuser@contoso.com. Администраторы могут обращаться к этим оповещениям, когда у них есть время.

|Skype for Business Server Voice Pool Watcher  <br/> |Экземпляр искусственных транзакций, которые запускают в одном пуле голосовой почты.  <br/> |

Пул Skype для бизнеса Server может содержать несколько отдельных систем Skype для бизнеса Server (с несколькими ролами Skype для бизнеса Server, службой Skype для бизнеса Server и компонентом Skype для бизнеса Server). Таким образом, сбой отдельного сервера или компонента менее важен для общего состояния пула Skype для бизнеса Server, так как другие серверы в том же пуле могут предоставлять службу приложений клиенту. The health will roll up on a percentage level to the Skype for Business Server pool. 
  
Watcher Skype for Business Server Pool Watcher performs synthetic transactions against a Skype for Business Server pool. Последовательный сбой одной или более искусственных транзакций (процесс, известный как последовательный интервал опроса) приводит к сбою критического состояния (наихудшего состояния искусственной транзакции), как показано на следующей схеме.

## <a name="best-practice-create-a-management-pack-for-customizations"></a>Best Practice: Create a Management Pack for Customizations

По умолчанию Operations Manager сохраняет все настройки, например переопределения пакета управления по умолчанию. Лучше всего создать отдельный пакет управления для каждого запечатанном пакете управления, который необходимо настроить. 
  
При создании пакета управления для хранения настроенных параметров для запечатанном пакета управления рекомендуется присвоить новому пакету управления соответствующее имя, например "Настройки Skype для бизнеса Server 2019".
  
Создание нового пакета управления для хранения настроек каждого запечатанном пакете управления упрощает экспорт настроек из тестовой среды в производственную среду. Это также упрощает удаление пакета управления, так как перед удалением пакета управления необходимо удалить все зависимости. Если настройки для всех пакетов управления сохранены в пакете управления по умолчанию и необходимо удалить один пакет управления, сначала необходимо удалить пакет управления по умолчанию, который также удаляет настройки для других пакетов управления.
