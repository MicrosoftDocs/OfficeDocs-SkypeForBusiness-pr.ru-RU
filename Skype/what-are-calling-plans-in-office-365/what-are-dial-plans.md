# Каковы планы телефонного подключения?

> [!IMPORTANT]
> Данная статья переведена с помощью машинного перевода, см. [Отказ от ответственности](2f0cfb59-1ca1-4e31-84ce-09d0b1a7ce1b.md#MT_Footer). Используйте английский вариант этой статьи, который находится [здесь](https://support.office.com/en-us/article/2f0cfb59-1ca1-4e31-84ce-09d0b1a7ce1b), в качестве справочного материала. 
  
Абонентская группа  это именованный набор правил нормализации, который преобразует номера телефонов отдельных пользователей в альтернативный формат (обычно E.164) для авторизации и маршрутизации звонков.
  
В абонентской состоит из одного или нескольких нормализация правила, которые определяют, как перевести альтернативный формат номера телефонов, представленные в различных форматах. Одну и ту же строку набора может распознаваться и перевод по-разному в различных абонентских, поэтому в зависимости от того, какие абонентской назначен данному пользователю, то же набрать номер может быть преобразованы и перенаправлено по-разному.
  
Просмотреть [Создание и управление ими абонентские группы](create-and-manage-dial-plans.md) создавать и управлять ими абонентских клиента.
  
## Область действия абонентской группы для клиента

Область абонентской группы определяет уровень иерархии можно применять абонентской группы. Области отличается от в Скайп для бизнеса Server 2015 локального развертывания. Клиенты получить соответствующие абонентской группой через параметры, которые предоставляются автоматически, когда пользователи входа Скайп для бизнеса Online подготовки. Как администратор можно управлять и назначение уровней области плана набора с помощью Remote PowerShell.
  
В Скайп для бизнеса Online есть два типа абонентских  служба областью и клиента (используется для вашей организации) области. Служба областью абонентской определяется для каждой страны или региона, где будет доступно телефонной системой Office 365. Каждому пользователю автоматически назначается абонентской группы страны службы, который соответствует место использования Office 365, назначенные пользователю. Невозможно изменить абонентской группы страны службы, но вы можете создать абонентских областью клиента, которых дополнить абонентской группы страны службы. Проведена клиентов, они получают «эффективные абонентской,» что представляет собой сочетание абонентской службы страны и абонентской группы с соответствующим образом областью клиента. Таким образом не требуется определить все правила нормализация в абонентских клиента, как они могут уже содержатся в абонентской группе страны службы.
  
Абонентские группы на основе клиентов можно дополнительно разбить на две области действия: область действия клиента и область действия пользователя. Если клиент определяет и назначает абонентскую группу на уровне пользователя, для этого пользователя подготавливается эффективная абонентская группа с учетом абонентской группы страны обслуживания и назначенной абонентской группы пользователя. Если клиент определяет абонентскую группу с областью действия клиента, но не назначает абонентскую группу на уровне пользователя, для этого пользователя подготавливается эффективная абонентская группа с учетом абонентской группы страны обслуживания и абонентской группы клиента.

Ниже перечислены возможные эффективные абонентские группы.
  
 **Служба страны** Если определенные без области клиента единой системы обмена сообщениями и не клиента областью пользователя единой системы обмена сообщениями не назначен предоставляемую пользователя, пользователь получит эффективные единой системы обмена сообщениями сопоставлением службы страны, связанной с их расположение использование Office 365.
  
 **Глобальные  клиента службы страны** Если определен абонентской пользователя клиента, но не назначен пользователю, предоставляемую пользователь получит эффективные единой системы обмена сообщениями состоящую из абонентской группы объединенных клиента и службы страны абонентская группа связанных с их расположение использование Office 365.
  
 **Пользователь клиента  служба страны** Если в клиенте пользователя абонентской определенные и назначен пользователю, предоставляемую пользователь получит эффективные единой системы обмена сообщениями состоящую из абонентской группы пользователей объединенные клиента и службы страны абонентская группа связанных с их расположение использование Office 365.

## Планирование абонентских групп клиента

Для планирования настраиваемых абонентских групп выполните следующие шаги.
  
- **Шаг 1** Решите, требуется ли настраиваемого набора план для улучшения набора интерфейс пользователя. Как правило потребность в один бы для поддержки набора не E.164, например расширения или сокращение национальный набора.
    
- **Шаг 2** Определите ли требуются глобального клиента или абонентских областью пользователя клиента и / или. Пользователь областью абонентских нужны, если пользователи имеют различные локального набора требований.
    
- **Шаг 3.** Определите допустимые шаблоны номеров для каждой из необходимых абонентских групп. Требуются только те шаблоны, которые не заданы в абонентских группах страны обслуживания.
    
- **Шаг 4.** Разработайте схему именования абонентских групп на уровне организации. Использование стандартной схемы именования обеспечивает согласованность работы, а также упрощает обслуживание и обновление системы.
    
[Развертывания](https://fasttrack.microsoft.com/microsoft365/capabilities?view=voice) имеет дополнительные ресурсы и абонентских партнеров, которые могут помочь реализации клиента.
  
## Создание абонентской группы клиента

При создании абонентской группы следует указать необходимые сведения.
  
### Имя и простое имя

Абонентские группы пользователей следует указать, что будут назначаться описательное имя, указывающее пользователям абонентской группы. Простое имя абонентской группы предустановленную со строкой, полученному из набора имя плана. Простое имя поля можно редактировать, позволяет создавать более понятное соглашение о наименовании для вашего абонентских. Простое имя значение не может быть пустым и должно быть уникальным. Рекомендуется разрабатывать соглашение о наименовании для всей организации и затем согласованное использование данное соглашение на все сайты и пользователи.
  
### Описание

Рекомендуется указать общеупотребительное и узнаваемое название географического расположения или группы пользователей, к которому будет применена соответствующая абонентская группа.
  
### Префикс для внешнего доступа

> [!CAUTION]
> Префикс внешнего доступа на данный момент не поддерживается. 
  
Если пользователям нужно набирать одну или несколько начальных цифр (например, цифру 9) для доступа к внешней линии, можно указать префикс для внешнего доступа длиной до четырех символов (допускаются символы #, * и 0-9).
  
> [!NOTE]
> В случае указания префикса создавать дополнительное правило нормализации для него не требуется. 
  
Просмотреть [Создание и управление ими абонентские группы](create-and-manage-dial-plans.md) для создания абонентских к клиенту.
  
## Правила нормализации

Правила нормализации определяют, как преобразуются номера телефонов, указанные в разных форматах. Одна и та же строка номера может быть интерпретирована и преобразована по-разному, в зависимости от языкового стандарта, который применяется к набравшему ее пользователю. Правила нормализации могут потребоваться, если пользователям нужно набирать сокращенные внутренние или внешние номера.
  
Одно или несколько правил нормализация должно быть назначено для абонентской группы. Нормализация правила сравниваются из сверху вниз, поэтому важно порядок показа в абонентской группы клиента. Например если абонентской группы клиента 10 нормализация правила, набираемого номера совпадающие логики будет использована, начиная с первого правила нормализация, если нет соответствия выберите второй и т.д. Если внесены совпадения, используется это правило и нет никаких усилий в соответствии с заданных правил. В абонентской группы заданного клиента могут содержать не более 25 нормализация правил.
  
### Определение необходимых правил нормализации

Так как любая абонентская группа клиента фактически объединяется с абонентской группой страны обслуживания для заданного пользователя, то для определения необходимых правил нормализации абонентской группы клиента, скорее всего, потребуется оценка правил нормализации абонентской группы для страны обслуживания. Для этого можно воспользоваться командлетом **Get-CsEffectiveTenantDialPlan**. Командлет использует удостоверение пользователя в качестве входного параметра и возвращает все правила нормализации, которые применяются к этому пользователю.
  
### Создание правил нормализации

Правила нормализации используют регулярные выражения платформы .NET Framework для указания шаблонов числовых соответствий, которые используются сервером для преобразования набираемых строк в формат E.164 для обратного преобразования номеров. Для создания правил нормализации можно указать регулярное выражение для проверки соответствия, и при обнаружении соответствия будет выполнено преобразование. После окончания настройки можно ввести тестовый номер, чтобы проверить работу правил нормализации.
  
Дополнительные сведения об использовании регулярных выражений .NET Framework читайте в статье [Регулярные выражения .NET Framework](https://go.microsoft.com/fwlink/p/?linkId=140927).
  
Просмотреть [Создание и управление ими абонентские группы](create-and-manage-dial-plans.md) создавать и управлять ими нормализация абонентских правила для клиента.
