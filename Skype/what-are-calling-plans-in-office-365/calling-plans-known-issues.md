Вызов планами в Office 365  новая функция найти в Skype для бизнеса online. Существует текущей проблемы, которые отслеживаются, активно исследовать и потенциально разрешается при обновлении в будущих сборки в Office 365 и Skype для бизнеса online и перечисленные ниже компонент.
  
## Известные проблемы звонков через ТСОП

|**Известная проблема**|**Примечания**|
|:-----|:-----|
|Лицензия автоматически не обновляется при переходе с лицензий на техническую версию на лицензии для звонков через ТСОП в определенных продуктах.  <br/> |Покупка новых лицензий сначала так, чтобы они готовы назначенных пользователям. Удаление пользователя лицензию рекламного (Technical Preview) и **сразу ЖЕ** назначить новые **Планирование Местные звонки** и/или **Международные звонки планирование** лицензии пользователю. <br/> При удалении и назначении лицензий сразу нескольким пользователям крайне важно удалить лицензии у всех пользователей с помощью Windows PowerShell и затем также с помощью Windows PowerShell **СРАЗУ** назначить им лицензии. Так можно избежать простоя в работе службы при обработке пользовательских лицензий в больших количествах. Примеры сценариев PowerShell см. в разделе[Назначение лицензий Skype для бизнеса и Microsoft Teams](../skype-for-business-and-microsoft-teams-add-on-licensing/assign-skype-for-business-and-microsoft-teams-licenses.md).  <br/> > [!CAUTION]> При использовании подключения локальных через ТСОП для пользователей гибридной можно  *только*  необходимо назначить лицензию **Телефонной системой**. **Не** должен также назначать плана вызывающего голосовой связи.> При включении вызова планы в Office 365 для пользователей, которые находятся в Office 365, необходимо по-прежнему назначить лицензию **Планирование Местные звонки** и/или **Международные звонки план** для этих пользователей. Просмотреть[Назначение лицензий Skype для бизнеса и Microsoft Teams](../skype-for-business-and-microsoft-teams-add-on-licensing/assign-skype-for-business-and-microsoft-teams-licenses.md).           |
