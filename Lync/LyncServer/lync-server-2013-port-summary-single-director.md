# Сводка по портам — единственный директор в Lync Server 2013

 

_**Дата изменения раздела:** 2015-03-09_

Порты, которые должны быть разрешены в брандмауэре для отдельного Директор, включают порты, используемые для установления соединения с Директором со стороны внутреннего интерфейса или внутренней сети обратного прокси-сервера. По умолчанию для работы сервера Microsoft Lync Server 2013 необходимо, чтобы порты HTTP/TCP 8080 и HTTPS/TCP 4443 были открыты для связи обратного прокси-сервера с Директор, а также переднего плана и переднего плана. Кроме того, должна быть возможность связи по протоколу SIP между внутренним интерфейсом сервер и Директор, а также переднего плана и переднего плана. Для связи сервер по протоколу SIP с переднего плана и переднего плана используется порт SIP/MTLS/TCP 5061. Также необходимо создать правило, разрешающее связь Директор, переднего плана и переднего плана с внутренним интерфейсом сервер через порт SIP/MTLS/TCP 5061.
