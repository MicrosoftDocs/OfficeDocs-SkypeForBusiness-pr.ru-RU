<div>

## <a name="archive-event-logs"></a>Архивные журналы событий

Если журналы событий не настроены на соответствующую перезапись событий, их необходимо регулярно архивировать и удалять. Выполнение этой задачи является особенно важным для журналов безопасности, которые могут потребоваться при расследовании преднамеренных нарушений безопасности.

Организация должна определять политики и процедуры для архивации журналов.

</div>

<div>

## <a name="create-reports"></a>Создание отчетов

Создайте отчеты о состоянии, которые помогут при планировании мощности, анализе соглашений об уровне обслуживания и анализе производительности. Используйте ежедневные данные из журнала событий и системного монитора для создания отчетов на диске, памяти и загрузка ЦП. Используйте System Center Operations Manager для создания отчетов о времени работы и доступности.

Организация должна определять политики и процедуры для отчетов о состоянии.

</div>

<div>

## <a name="incident-reports"></a>Отчеты об инцидентах

Выполнение еженедельного аудита отчетов об инцидентах Организации, относящихся к Lync Server. Этот аудит должен включать следующие компоненты:

  - Топ созданных, разрешенных и ожидающих инцидентов.

  - Решения для неразрешенных происшествий.

  - Обновление отчетов с добавлением новых билетов на проблемы.

  - Обновление репозитория документов для руководства по устранению неполадок и устранение неисправностей в отношении простоев.

Так как система отслеживания инцидентов вашей организации не зависит от Lync Server, конкретные инструкции или указатели недоступны. Обратитесь к документации по системе, выбранной вашей организацией.

</div>

## <a name="check-iis-logs-and-performance"></a>Проверка журналов и производительности IIS

Еженедельный обзор журналов и производительности служб IIS. Дополнительные сведения о мониторинге журналов и производительности IIS можно найти в статье [описание ведения журнала событий служб IIS для Windows Server 2003](https://go.microsoft.com/fwlink/?linkid=36077). Проверка должна включать следующее:

  - Счетчики кэша веб-службы для мониторинга кэша службы WWW.

  - Счетчики страниц Active Server (АСПС) для мониторинга приложений, запускаемых как АСПС.

## <a name="generate-database-reports"></a>Создание отчетов базы данных

**Создание отчетов для базы данных SQL**

1.  Откройте Lync Server 2013.

2.  В дереве консоли разверните узел лес, разверните узел **Пулы предприятий**и выберите пул, для которого необходимо создать отчет базы данных.

3.  В области сведений перейдите на вкладку **база данных** .

4.  На вкладке **база данных** выполните следующие действия:
    
    1.  Чтобы просмотреть имя базы данных, разверните раздел **Общие параметры**и просмотрите имя базы данных.
    
    2.  Чтобы получить сводную статистику по текущему пользователю для пула, разверните раздел **Сводные отчеты пользователя**, щелкните **Перейти**и просмотрите результаты.
    
    3.  Для получения текущих данных для отдельного пользователя пула разверните узел **отчеты для каждого пользователя**, введите URI SIP, нажмите кнопку **Перейти**и просмотрите результаты.

Чтобы получить текущую сводную статистику для пула, разверните раздел **Сводные отчеты по конференциям**, нажмите кнопку **Перейти**и просмотрите результаты.

## <a name="check-for-security-and-lync-server-updates"></a>Проверка на наличие обновлений для системы безопасности и Lync Server

Проверьте наличие новых пакетов обновления, исправлений или обновлений. При необходимости протестируйте их в тестовой лаборатории и используйте процедуры управления изменениями для упорядочения развертывания на рабочих серверах. Кроме того, обновления компонентов Lync Server теперь доступны в составе центра обновления Windows. Все обновления компонентов Lync Server необходимо одновременно обновить на всех серверах, на которых работает Lync Server, к которым применяются обновления.

## <a name="run-the-lync-server-2013-best-practice-analyzer"></a>Запуск анализатора соответствия рекомендациям для Lync Server 2013

Анализатор соответствия рекомендациям Lync Server 2013 — средство диагностики, которое собирает сведения о конфигурации и определяет, настроена ли конфигурация в соответствии с рекомендациями Майкрософт. Документация для этого средства находится в [анализаторе соответствия рекомендациям Lync Server 2013](lync-server-2013-lync-server-best-practices-analyzer.md).

Средство сравнивает данные конфигурации развертывания с набором предопределенных правил для Lync Server и сообщает о возможных проблемах. Для всех обнаруженных ошибок средство предоставляет текущую конфигурацию в среде Lync Server и рекомендуемую конфигурацию.

При правильном доступе к сети средство может проверить службы AD DS и серверы, на которых работает Lync Server 2013, для выполнения следующих действий:

  - Упреждающее выполнение проверок работоспособности, проверка того, что конфигурация настроена в соответствии с рекомендуемыми рекомендациями

  - Создание списка проблем, таких как Неоптимальные параметры конфигурации или неподдерживаемые или не рекомендуемые варианты

  - Оценить общую работоспособность системы

  - Помощь в устранении определенных проблем

  - Запрос на скачивание обновлений, если они доступны

  - Предоставление интерактивной и локальной документации по отчетным вопросам и включение советов по устранению неполадок

  - Создание сведений о конфигурации, которые можно записать для последующего просмотра

Убедитесь, что RTCBPA.msi установлен на всех серверах Lync Server 2013, и создайте отчет о работоспособности еженедельной проверки работоспособности. Запишите результаты и при необходимости измените их.

## <a name="running-bandwidth-utilization-analyzer"></a>Запуск анализатора использования полосы пропускания

Анализатор использования полосы пропускания — это средство, которое создает отчеты о различных режимах использования полосы пропускания в конечных точках UC по каналам глобальной сети в корпоративной сети. Эти отчеты можно использовать для ознакомления с текущей моделью потребления полосы пропускания и для планирования пропускной способности. Кроме того, он выполняет итерацию по пропускной способности, назначенной различным каналам.

Это средство выполняет следующие действия:

  - Создает конкретные отчеты по использованию звука в сети

  - Способствует более эффективному планированию емкости и итерации по емкости полосы пропускания, назначенной различным связям

Анализатор использования полосы пропускания может создавать графические графики для отчетов о емкости и использовании полосы пропускания. ��� �������� ��������� �������:

  - Все связи WAN в корпоративной сети

  - Фильтрация по выбранным выбранным каналам глобальной сети

  - Фильтрация по каналам WAN с превышением установленной емкости канала

  - Фильтрация по каналам WAN с использованием подготовленной полосы пропускания

  - Фильтрация по каналам глобальной сети, которые достигли критического уровня (использование пропускной способности превышает 90% от пропускной способности канала WAN)

  - Фильтрация по типу канала WAN — связи сайтов сети, взаимосвязи между регионами и ссылки на сайте

  - Фильтрация по области сети

Документация для этого средства доступна в [документации по средствам набора ресурсов Lync Server 2013](https://go.microsoft.com/fwlink/?linkid=623245).
