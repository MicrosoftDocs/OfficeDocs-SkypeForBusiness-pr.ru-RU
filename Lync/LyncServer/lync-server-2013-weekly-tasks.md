# <a name="weekly-tasks-in-lync-server-2013"></a>Еженедельные задачи в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Тема последнего изменения:** 2015-08-17_

Еженедельные задачи обычно связаны с сбором и анализом журналов и отчетов.

<div>

## <a name="archive-event-logs"></a>Архивация журналов событий

Если журналы событий не настроены на перезапись событий, они должны быть регулярно заархивированы и удалены. Это действие особенно важно для журналов безопасности, которые могут потребоваться при исследовании попыток нарушения безопасности.

Организация должна определять политики и процедуры для архивации журнала.

</div>

<div>

## <a name="create-reports"></a>Создание отчетов

Создание отчетов о состоянии, помогающих при планировании ресурсов, проверках SLA и анализе производительности. Используйте ежедневные данные из журнала событий и системного монитора для создания отчетов об использовании диска, памяти и ЦП. С помощью System Center Operations Manager Создавайте отчеты о доступности и работоспособности.

Организация должна определять политики и процедуры для отчетов о состоянии.

</div>

<div>

## <a name="incident-reports"></a>Отчеты об инцидентах

Еженедельный аудит отчетов об инцидентах Организации, связанных с Lync Server. Этот аудит должен включать в себя следующее:

  - Самые популярные, разрешенные и незавершенные инциденты.

  - Решения для неразрешенных происшествий.

  - Обновление отчетов для включения новых билетов на проблемы.

  - Обновление репозитория документов для руководств, посвященных устранению неполадок, и публикация неустранимых проблем.

Поскольку система отслеживания происшествий Организации является вариантом выбора, независимого от Lync Server, некоторые инструкции или указатели недоступны. Ознакомьтесь с документацией для системы, выбранной вашей организацией.

</div>

<div>

## <a name="check-iis-logs-and-performance"></a>Проверка журналов и производительности IIS

Еженедельное рецензирование журналов служб IIS и производительность. Дополнительные сведения о мониторинге журналов и производительности IIS можно найти в статье [Обзор ведения журнала событий служб IIS для Windows Server 2003](http://go.microsoft.com/fwlink/?linkid=36077). Проверка должна включать следующее:

  - Счетчики кэша веб-службы для контроля кэша службы WWW.

  - Счетчики ASP-страниц (АСПС) для наблюдения за приложениями, которые выполняются как АСПС.

</div>

## <a name="generate-database-reports"></a>Создание отчетов базы данных

**Создание отчетов в базе данных SQL**

1.  Откройте Lync Server 2013.

2.  В дереве консоли разверните узел леса, разверните **Пулы предприятий**и выберите пул, для которого требуется создать отчет базы данных.

3.  В области сведений откройте вкладку **база данных** .

4.  На вкладке **база данных** выполните указанные ниже действия.
    
    1.  Чтобы просмотреть имя базы данных, разверните раздел **Общие параметры**и просмотрите имя базы данных.
    
    2.  Чтобы получить сводную статистику текущего пользователя для пула, разверните раздел **Сводные отчеты пользователей**, нажмите кнопку **Перейти**и просмотрите результаты.
    
    3.  Чтобы получить текущие данные пользователя для одного пользователя пула, разверните раздел **отчеты для каждого пользователя**, введите URI SIP пользователя, нажмите кнопку **Перейти**и просмотрите результаты.

Чтобы получить текущую сводную статистику для пула, разверните **Сводные отчеты по**конференциям, нажмите кнопку **Перейти**и просмотрите результаты.

</div>

## <a name="check-for-security-and-lync-server-updates"></a>Проверка обновлений для системы безопасности и Lync Server

Определите новые пакеты обновления, исправления и обновления. При необходимости проверьте эти данные в тестовой лаборатории и используйте процедуры управления изменениями для упорядочения развертывания на рабочих серверах. Кроме того, обновления компонентов Lync Server теперь доступны в составе центра обновления Windows. Все обновления компонентов Lync Server необходимо обновлять одновременно на всех серверах, на которых работает Lync Server, для которых применимы обновления.

</div>

## <a name="run-the-lync-server-2013-best-practice-analyzer"></a>Запуск анализатора соответствия рекомендациям для Lync Server 2013

Анализатор соответствия рекомендациям для Lync Server 2013 — средство диагностики, которое собирает сведения о конфигурации и определяет, задана ли конфигурация согласно рекомендациям Microsoft. Документация по этому средству — [анализатор соответствия рекомендациям для Lync Server 2013](lync-server-2013-lync-server-best-practices-analyzer.md).

Это средство сравнивает данные конфигурации развертывания с набором предопределенных правил для Lync Server и сообщает о потенциальных проблемах. Для каждой обнаруженной ошибки средство предоставляет текущую конфигурацию в среде Lync Server и рекомендуемую конфигурацию.

При правильном доступе к сети средство может проанализировать службы AD DS и серверы, на которых работает Lync Server 2013, и выполнить указанные ниже действия.

  - Упреждающее выполнение проверок работоспособности, проверка настройки конфигурации согласно рекомендуемым рекомендациям

  - Создание списка проблем (например, неоптимальных параметров конфигурации или неподдерживаемых или рекомендуемых параметров)

  - Оценить общее состояние системы

  - Помощь в устранении конкретных проблем

  - Запрос на скачивание обновлений (если они доступны)

  - Предоставление сведений об ошибках, связанных с электронной и локальной документацией, и включение советов по устранению неполадок

  - Создание сведений о конфигурации, которые можно записать для последующего просмотра

Убедитесь, что на всех серверах Lync Server 2013 установлен флажок РТКБПА. msi, и создайте еженедельный отчет о работоспособности. Обратите внимание на результаты и, при необходимости, правильность.

## <a name="running-bandwidth-utilization-analyzer"></a>Запуск анализатора использования пропускной способности

Анализатор использования полосы пропускания — это средство, которое создает отчеты о различных аспектах использования полосы пропускания каналов глобальной сети конечными точками объединенных коммуникаций в корпоративной сети. Эти отчеты можно использовать для понимания текущего шаблона потребления для пропускной способности и для планирования пропускной способности по мощности. Оно также позволяет просматривать полосу пропускания, назначенную различным каналам.

Это средство предоставляет следующие возможности:

  - Создание специальных отчетов по использованию звука в сети

  - позволяет более эффективно планировать емкость и распределять полосы пропускания, назначенные различным каналам.

Анализатор использования пропускной способности может создавать графические представления о емкости и отчетах об использовании пропускной способности. Они перечислены ниже.

  - для всех каналов глобальной сети в корпоративной сети;

  - Отфильтровано по выбранным глобальным каналам связи

  - для каналов глобальной сети, емкость которых превышена;

  - Фильтрация по каналам глобальной сети, которые использовали подготовленную пропускную способность

  - Фильтрация по глобальным каналам связи (использование пропускной способности, превышающее 90 процента пропускной способности ссылки WAN)

  - Фильтрация по типу ссылки WAN — ссылки на сайты, межсетевые ссылки и ссылки на сайте

  - с фильтрацией по области сети.

Документация по этому средству можно найти в [документации по средствам для Lync Server 2013 Resource Kit](http://go.microsoft.com/fwlink/?linkid=623245).
