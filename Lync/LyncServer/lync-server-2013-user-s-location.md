# Расположение пользователя в Lync Server 2013

 

_**Дата изменения раздела:** 2013-03-09_

Функция маршрутизации на основе местоположения использует области, сайты и подсети сети, определенные в Lync Server и используемые компонентами E9-1-1, контроля допуска звонков и обхода сервера-посредника для ограничения маршрутизации вызовов и предотвращения обхода платных звонков ТСОП. Местоположение пользователя Lync определяется по IP-подсети конечных точек, с которых он подключается. Каждая IP-подсеть связана с сайтом сети, которые объединяются в области сети, определяемые администратором. Маршрутизация на основе местоположения выполняется исходя из сайта сети данного пользователя.

Правила маршрутизации на основе местоположения устанавливаются для сайтов сети. Это означает, что определенный набор правил действует для всех конечных точек, для которых включена функция маршрутизации на основе местоположения и которые расположены в пределах одного сайта сети. Администраторы могут включать эту функцию для отдельных сайтов.

Политики маршрутизации голосовой связи задаются для отдельных сайтов сети и определяют шлюз ТСОП, через который все пользователи, расположенные в этом сайте, должны звонить на телефонные номера ТСОП. Эти политики имеют приоритет над заданными для отдельных пользователей политиками голосовой связи, если пользователь расположен в сайте, для которого включена маршрутизация на основе местоположения, и предотвращают маршрутизацию звонков через другие шлюзы ТСОП, для которых включена эта функция. Когда пользователь Lync совершает звонок ТСОП, заданная для него политика голосовой связи определяет, имеет ли он право на его совершение. Если у пользователя есть такое разрешение, функция маршрутизации на основе местоположения выбирает шлюз, через который будет совершен этот звонок, исходя из местоположения пользователя.

Местоположение пользователя может относиться к одному из перечисленных ниже типов.

  - Пользователь расположен в известном сайте сети, для которого включена маршрутизация на основе местоположения, и его прямой внутренний номер относится к шлюзу ТСОП, расположенному в том же сайте (например, офисе). Маршрутизация исходящих звонков будет выполняться на основе политики маршрутизации голосовой связи сайта сети, в котором расположен пользователь. Входящие звонки ТСОП данному пользователю маршрутизируются на конечные точки, которые расположены в том же сайте сети, что и шлюз ТСОП.

  - Пользователь расположен в известном сайте сети, отличном от того, в котором расположен шлюз ТСОП (например, пользователь перешел в другой офис того же предприятия). Маршрутизация исходящих звонков будет выполняться на основе политики маршрутизации голосовой связи сайта сети, в котором расположен пользователь. Входящие звонки ТСОП данному пользователю не будут маршрутизироваться на конечные точки, расположенные в отличных от сайта ТСОП сайтах, чтобы избежать обхода платных звонков ТСОП.

  - Если пользователь расположен в сайте сети, который неизвестен системе Lync Server, маршрутизация исходящих звонков будет выполняться на основе назначенной пользователю политики голосовой связи на шлюзы ТСОП, не связанные ограничениями маршрутизации на основе местоположения. Входящие звонки ТСОП не будут маршрутизироваться на конечные точки, расположенные на неизвестных сайтах сети, чтобы предотвратить обход платных звонков ТСОП.
