# <a name="users-location-in-lync-server-2013"></a>Расположение пользователя в Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

_**Последнее изменение темы:** 2013-03-09_

Location-Based маршрутизация использует одни и те же области сети, сайты и подсети, как определено в Lync Server, используемый E9-1-1, CAC и обход сервера-посредника для применения ограничений маршрутизации вызовов для предотвращения бесплатных звонков по сети PSTN. Расположение пользователя определяется IP-подсетью, из которой подключены конечные точки Lync пользователя. Каждая IP-подсеть связана с сетевым сайтом, которые объединяются в сетевые области, определенные администратором. Маршрутизация Location-Based применяется на основе сетевого сайта пользователя.

Location-Based правила маршрутизации применяются для каждого сетевого сайта, что означает, что заданный набор правил будет применен ко всем конечным точкам, включенным для маршрутизации Location-Based, расположенным в пределах одного сетевого сайта. Администраторы могут применять Location-Based маршрутизации к сетевым сайтам, которым она необходима.

Политики маршрутизации голосовой связи можно определить для каждого сетевого сайта, чтобы определить конкретный шлюз PSTN, который будет использоваться всеми пользователями, находящимися на сетевом сайте, для вызова телефонных номеров PSTN. Такие политики маршрутизации голосовых вызовов имеют приоритет перед маршрутизацией, определенной политикой голосовой связи пользователя, когда пользователь находится на сетевом сайте, включенном для маршрутизации Location-Based, и предотвращает маршрутизацию вызовов через другие шлюзы PSTN, для которых включена маршрутизация Location-Based. Когда пользователь Lync помещает Звонок PSTN, политика голосовой связи пользователя определяет, может ли пользователь быть авторизован для совершения звонка. Если политика голосовой связи пользователя разрешает пользователю выполнять вызов, Location-Based маршрутизация определяет шлюз PSTN, из которого будет осуществляться вызов. Маршрутизация Location-Based выполняет это определение в зависимости от расположения пользователя.

Расположение пользователя можно классифицировать следующими способами:

  - Пользователь находится на известном сетевом сайте, включенном для маршрутизации Location-Based, а его номер (прямой вызов) завершается на шлюзе PSTN, размещенном на том же сетевом сайте (например, Office). Маршрутизация исходящих вызовов будет проходить через политику маршрутизации голосовой связи на сетевом сайте, в котором расположен пользователь. Входящие вызовы PSTN направляются пользователю в конечные точки, расположенные на том же сетевом сайте, что и шлюз PSTN.

  - Пользователь находится на известном сетевом сайте, который отличается от сетевого сайта, на котором размещается шлюз PSTN. (то есть пользователь в другом корпоративном офисе). Маршрутизация исходящих вызовов будет использовать политику маршрутизации голосовых вызовов сетевого сайта, в котором расположен пользователь. Входящие звонки PSTN не будут перенаправляться в конечные точки, расположенные на разных сайтах, чем шлюз PSTN для предотвращения обхода бесплатных звонков по сети PSTN.

  - Если пользователь находится на сетевом сайте, неизвестном для развертывания Lync Server, Маршрутизация исходящих вызовов будет основана на политике голосовой связи, назначенной пользователю, на шлюзы PSTN, которые не привязаны к ограничениям маршрутизации Location-Based. Входящие звонки PSTN не будут перенаправляться в конечные точки, расположенные на неизвестных сетевых сайтах, чтобы предотвратить обход звонков по сети PSTN.
