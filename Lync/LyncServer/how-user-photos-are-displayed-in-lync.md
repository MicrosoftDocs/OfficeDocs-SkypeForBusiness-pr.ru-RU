# <a name="how-user-photos-are-displayed-in-lync"></a>Отображение фотографий пользователей в Lync

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Тема последнего изменения:** 2014-08-25_

**Сводка:** Фотографии пользователей, отображаемые в клиенте Lync, могут различаться в зависимости от того, какой компонент Lync вы используете, например в ходе Конференции или в чате.

В Lync 2010 появилась возможность включения фотографии с профилем Lync, который отображается для других пользователей Lync. Вы также можете указать, нужно ли отображать фотографии для контактов в клиенте Lync. В Lync 2013 поддерживается поддержка фотографий высокого разрешения для пользователей. В этой статье объясняется, как клиент Lync получает и отображает фотографии пользователей, в которых хранятся изображения, ограничения для каждого источника изображения и способы использования фотографий пользователей различными службами Lync.

<div>

## <a name="planning-considerations"></a>Вопросы планирования

При планировании реализации поддержки фотографий пользователей следует руководствоваться описанной ниже процедурой.

  - Для поддержки фото в высоком разрешении требуется, чтобы почтовый ящик пользователя находился на сервере Exchange 2013, а учетная запись пользователя Lync — в пуле Lync 2013.

  - Пользовательские фотографии High Definition поддерживаются только в среде, в которой используются как Lync Server 2013, так и Exchange 2013.

  - Пользователи с почтовыми ящиками в Exchange 2010 всегда будут использовать атрибут **Фотоэскиз** из доменных служб Active Directory в качестве источника фотографии пользователя.

  - Пользовательская фотография, сохраненная в качестве атрибута **Фотоэскиз** из доменных служб Active Directory, не будет отображаться для внешних и Федеративных контактов.

  - Если фотографии контактов пользователей хранятся в доменных СЛУЖБах Active Directory, то размер используемого файла изображения ограничен 96 × 96 пикселей и размером файла 100 КБ.

  - Если связь между Lync Server и Exchange Server потеряна, будет отображаться **Фотоэскиз** с низким разрешением пользователя из доменных служб Active Directory и только для внутренних пользователей.

  - Фотографии пользователей с высоким разрешением отображаются в собраниях Lync 2013, когда активный динамик не поддерживает видео. Кроме того, если переместить указатель мыши на фотографию эскиз в галерее, будет отображаться фотография высокого разрешения.

## <a name="user-photos-in-lync-2010"></a>Фотографии пользователей в Lync 2010

В клиенте Lync 2010 вы можете выбрать один из двух вариантов для отображения фотографии вашего профиля, **корпоративного изображения по умолчанию** и **отображения изображения с веб-адреса**.

<div>

## <a name="default-corporate-picture"></a>Корпоративный рисунок по умолчанию

Когда вы выбираете вариант **корпоративного аватара по умолчанию** , Lync получает фотографию, которая отображается для вас из доменных служб Active Directory. Используемое изображение — это изображение, определенное как значение атрибута **Фотоэскиз** в доменных службах Active Directory. Это тот же файл, который используется в Exchange для отображения изображений в Outlook.

Ниже приведены рекомендации по использованию изображений из доменных служб Active Directory.

  - Поддерживаются только изображения с измерениями размером до 96 пикселей на 96 пикселей. Размер файла изображения ограничен до 100 КБ.

  - По умолчанию пользователи имеют возможность изменить изображение, используемое для атрибута **Фотоэскиз** , несмотря на то, что непосредственно с помощью клиента Lync. Вы можете отключить это с помощью доменных служб Active Directory.

  - Изображения, хранящиеся в доменных службах Active Directory, не отображаются для контактов, находящихся вне Организации, даже если они являются федеративными контактами.

  - В крупных организациях хранение и Извлечение изображений для большого количества пользователей может повлиять на размер и производительность базы данных доменных служб Active Directory.

  - Ограниченные размеры изображения и размер файла означают, что могут использоваться только изображения с низким разрешением.

## <a name="how-users-manage-their-user-photos-in-active-directory-domain-services"></a>Управление фотографиями пользователей в доменных службах Active Directory

Пользователь не может изменить изображение, используемое в своем профиле доменных служб Active Directory прямо через клиент Lync 2010. Для этого они могут использовать один из указанных ниже параметров, если он доступен.

  - **Пользователи SharePoint Server**   могут отправить фотографию на страницу "мой сайт" на сервере SharePoint, а затем [настроить синхронизацию профилей в SharePoint](http://go.microsoft.com/fwlink/p/?linkid=507466) , чтобы синхронизировать фотографию с атрибутом **Фотоэскиз** в домене Active Directory. Служб.

  - **Фотография, сохраненная на общедоступном URL-адресе**   пользователи могут настроить свою фотографию пользователя, указав общедоступный URL-адрес для изображения, которое требуется использовать. Изображение должно быть открыто для общего доступа без пароля. Изображение, хранящееся в указанном веб-адресе, передается другим пользователям с помощью категории карточки контакта в сведениях о присутствии. Когда клиент Lync должен отобразить фотографию пользователя, он извлекает изображение с указанного веб-адреса.

  - **Командлеты Exchange 2010 для администраторов Windows PowerShell**   могут выполнять командлет [Import-реЦипиентдатапроперти](http://go.microsoft.com/fwlink/p/?linkid=507468) в командной консоли Exchange 2010 для управления атрибутом **Фотоэскиз** . При импорте изображений с помощью командлетов Exchange 2010 размер файла ограничен 10 КБ.

  - **Сторонние инструменты**   для работы с атрибутом **Фотоэскиз** могут отправлять только свои фотографии.

## <a name="show-a-picture-from-a-web-address"></a>Отображение изображения с веб-адреса

Если вы выбрали вариант **Показать рисунок с помощью веб-адреса** , Lync получает изображение по адресу, который вы ввели, и отображает его для фотографии пользователя в Lync.

Ниже приведены рекомендации по использованию изображений с веб-адресом.

  - Ограничения размера файлов определяются атрибутом **максфотосизекб** в политике клиента, который определяется с помощью командлета [New-CsClientPolicy](http://go.microsoft.com/fwlink/p/?linkid=507463) . По умолчанию предельный размер составляет 30 КБ. Максимальное значение — 100 КБ. Разрешение изображения не ограничивается, но если вы попытаетесь использовать файл изображения, размер которого превышает предельный, он не будет загружен в клиенты Lync. Вы можете установить для этого параметра значение 0, чтобы отключить все фотографии пользователей в Lync.

  - Фотографии пользователей на веб-адресах можно просматривать с помощью внешних федеративных контактов.

## <a name="how-lync-2010-client-gets-photos"></a>Как клиент Lync 2010 получает фотографии

В Lync 2010 пользовательские фотографии на сервере управляются службой адресной книги. Клиент Lync получает фотографии пользователей с помощью предварительного запроса службы веб-запроса адресной книги (АБВК) на сервере, доступ к которому осуществляется с помощью веб-службы расширения списка рассылки. Клиент получает файл изображения, а затем копирует его в кэш пользователя, чтобы избежать загрузки изображения каждый раз, когда необходимо его отобразить. Значения атрибутов, возвращаемые из запроса, также хранятся в записи службы кэшированных адресных книг для пользователя. Служба адресной книги удаляет все кэшированные изображения каждые 24 часа, что означает, что новые образы пользователей будут обновляться в кэше на сервере в течение 24 часов. Вы можете принудительно обновить кэш с помощью командлета [Update-ксаддрессбук](https://docs.microsoft.com/powershell/module/skype/Update-CsAddressBook) .

Фотографии пользователей, включенные в состояние присутствия, также имеют связанное хэш-значение, используемое приложением Lync для определения того, доступно ли более новое изображение. Клиент автоматически сообщает об изменениях файла изображения, используемого в состоянии присутствия.

<div class=" ">


> [!NOTE]  
> Поскольку фотографии не хранятся в базе данных Галконтактс. DB, Загрузка фотографий пользователя не зависит от параметра <STRONG>аддрессбукаваилабилити</STRONG> политики клиента (<A href="http://go.microsoft.com/fwlink/p/?linkid=507508">Set-CsClientPolicy</A>).



</div>

Запрос к службе АБВК включает следующие атрибуты:

  - **** Чтобы определить, изменилась ли текущая фотография, выдается хэш-значение двоичных данных фотографии и используется для определения изменения.   

  - **Фоторелпас**   относительный путь к файлу изображения, хранящемуся на сервере.

  - **** Выберите размер файла изображения (в байтах).   

  - **TimeStamp**   Дата и время последнего скачивания файла изображения с сервера и его копирование в кэш клиента.

Затем, после получения файла изображения, клиент Lync 2010 сравнивает значения атрибутов, возвращенные из запроса, с использованием значений атрибутов, полученных клиентом из встроенной программы подготовки, чтобы выяснить, различаются ли они. Если значения различаются, клиент извлекает файл изображения пользователя, выполнившего вход, с помощью HTTP-запроса GET.

Кроме того, клиент сверяет с сервером каждые 24 часа с момента создания кэшированной версии файла изображения для сравнения значения атрибута "фотохэш" на сервере со значением на **** клиенте. Если значения различаются, клиент знает, что файл изображения изменился. Чтобы получить обновленный файл изображения, клиент снова запрашивает службу АБВК, чтобы обновить файл изображения в кэше клиента с помощью файла изображения на сервере, который также сбрасывает **метку времени** для файла в клиентском кэше.

## <a name="user-photos-in-lync-2013"></a>Фотографии пользователей в Lync 2013

В Lync 2013 добавлена поддержка изображений с высоким разрешением для фотографий пользователей. В Lync 2013 также поддерживается хранение фотографий пользователей в почтовом ящике пользователя на сервере Exchange 2013, в результате чего удаляются разрешения и ограничения размеров, представленные в Lync 2010. Фотографии пользователей в Lync 2013 могут иметь размер до 648 пикселей на 648 пикселов размером до 20 МБ. Фотографии высокого разрешения в Lync 2013 должны находиться в почтовом ящике пользователя на сервере Exchange 2013 и поддерживаются только в клиенте Lync 2013. Интеграция с Exchange использует преимущества новой платформы авторизации, включенной в версии 2013 для Lync, Exchange и SharePoint, именуемые OAuth.

Если в развертывании не используется Exchange 2013, поддержка пользовательских фотографий будет такой же, как в Lync 2010. Однако параметры пользователя для выбора фотографии отличаются в клиенте Lync 2013. В клиенте Lync 2013 пользователи могут выбрать либо **скрыть мою фотографию** , либо **Показать мою фотографию**. Параметр **Показать рисунок с веб-сайта** по умолчанию недоступен, но его можно включить, назначая политику клиента.

Параметры фотографий пользователей находятся в диалоговом окне " **Параметры** " в Lync 2013. Когда вы выбираете команду **скрыть мою**фотографию, в клиенте Lync не отображаются фотографии пользователей, но ваши фото по-прежнему отображаются в карточке контакта и вне Lync.

## <a name="show-a-picture-from-a-website"></a>Показать фотографию с веб-сайта

Параметр **Показать рисунок с веб-сайта** становится доступен в Lync 2013 после того, как политика клиента настроена на ее включение. Версия клиента должна быть более новой, чем 15.0.4535.1002, которая устанавливается вместе с [накопительными обновлениями Lync: ноябрь 2013](http://go.microsoft.com/fwlink/p/?linkid=509908)г. Пользователям может потребоваться выйти из программы, а затем снова войти в систему, чтобы увидеть изменения в клиенте.

Вы можете настроить политику клиента так, чтобы она **отображалась на веб-сайте** , запустив политику [Set-CsClientPolicy](https://docs.microsoft.com/powershell/module/skype/Set-CsClientPolicy) в командной консоли Lync Server Management Shell. В следующем примере командлетов показано, как настроить политику глобально для всех пользователей в развертывании:

   ```
    $pe=New-CsClientPolicyEntry -Name EnablePresencePhotoOptions -Value True
   ```

   ```
    $po=Get-CsClientPolicy -Identity Global
   ```

   ```
    $po.PolicyEntry.Add($pe)
   ```

   ```
    Set-CsClientPolicy -Instance $po
   ```


При загрузке изображения в почтовый ящик пользователя Exchange автоматически создает более низкую версию изображения, которую можно использовать в клиентских приложениях. Фотография пользователя также обновлена в доменных СЛУЖБах Active Directory.

<div class=" ">


> [!NOTE]  
> При обновлении файла изображения в доменных СЛУЖБах Active Directory создается и используется для Фотоэскиз в AD DS изображение пикселя 48 x 48. Все существующие изображения будут заменены. Таким образом, если вы добавили в AD DS изображение 96 x 96, оно будет заменено новым образом 48 x 48. Это важно только в том случае, если у вас есть пользователи в вашей среде с помощью клиентов Lync 2010, так как эти клиенты будут получать фотографии пользователей из доменных служб Active Directory. Вы можете импортировать изображения пикселов 96 x 96 пикселей, чтобы заменить созданные AD DS, если у вас есть клиенты Lync 2010 в своей организации.

## <a name="user-photo-support-in-lync-2013"></a>Поддержка Фото пользователей в Lync 2013

В Lync 2013 для пользовательских фотографий поддерживаются три разрешения изображений, описанные в приведенной ниже таблице. Используемое изображение определяется параметром клиентской политики, назначенным пользователям Lync. Дополнительные сведения можно найти в разделе "Управление фото пользователя с помощью командлетов клиентской политики".

Любой пользователь с включенным в Exchange 2013 почтовым ящиком может загрузить другое изображение, включая фотографии высокого разрешения, с помощью Outlook Web Access или параметров клиента Lync 2013. Для используемых изображений рекомендуются следующие параметры:

  - **Разрешение изображения 648**   на 648 пикселей

  - **Глубина цвета 24**   -разрядная

  - **Размер файла изображения**   до 20 МБ

  - **Формат файла JPEG**   

Типичный 24-битный формат JPEG, 648 пикселей на 648 пикселей, имеет размер файла около 240 КБ, поэтому для каждых 4 фотографий пользователей потребуется 1 МБАЙТ свободного места.
