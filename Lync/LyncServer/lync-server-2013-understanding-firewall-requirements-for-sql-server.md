# Обзор требований к брандмауэру для SQL Server с Lync Server 2013

 

_**Дата изменения раздела:** 2016-12-08_

В случае с развертыванием выпуска Standard Edition исключения брандмауэра создаются автоматически в процессе установки Lync Server 2013. Однако при развертывании выпуска Enterprise Edition исключения брандмауэра нужно настроить вручную на тыловом сервере SQL Server. Протокол TCP/IP допускает использование определенного порта только для одного IP-адреса. Это означает, что для сервера SQL Server вы можете назначить экземпляру базы данных по умолчанию стандартный TCP-порт 1433. Для других экземпляров необходимо назначить уникальные свободные порты с помощью диспетчера конфигурации SQL Server. В этом разделе рассматриваются следующие вопросы:

  - требования к исключению брандмауэра для экземпляра по умолчанию;

  - требования к исключению брандмауэра для службы обозревателя SQL Server;

  - требования к статическим прослушивающим портам при использовании именованных экземпляров.

## Требования к исключению брандмауэра при использовании экземпляра по умолчанию

Если при развертывании Lync Server 2013 вы используете для одной из баз данных экземпляр SQL Server по умолчанию, нужно настроить правило брандмауэра для обеспечения взаимодействия интерфейсного пула с этим экземпляром.

## Требования к статическим прослушивающим портам при использовании именованных экземпляров

При использовании именованных экземпляров SQL Server для баз данных, обслуживающих Lync Server 2013, необходимо настроить статические порты с помощью диспетчера конфигурации SQL Server. Назначив статические порты каждому именованному экземпляру, нужно создать в брандмауэре исключение для каждого из этих портов.
