# Новые функции архивации в Lync Server 2013

 

_**Дата изменения раздела:** 2016-12-08_

При выполнении архивации в Lync Server 2013 могут архивироваться следующие типы контента:

  - одноранговые мгновенные сообщения;

  - конференции (собрания), которые являются обменом мгновенными сообщениями с несколькими участниками;

  - содержимое конференций, включая загруженное содержимого (например, раздаточные материалы), и связанное с событиями содержание (например, сведения о присоединении, выходе, загрузке, общем доступе и изменениях видимости);

Кроме того, архивация в Lync Server 2013 предоставляет новые функции, которые улучшают эффективность развертывания и эксплуатации. Сюда входят следующие функции.

  - **Объединенное архивирование на серверах переднего плана.**    Lync Server 2013 не имеет отдельной роли сервера архивирования. Архивирование является дополнительной функцией, которая доступна на всех серверах переднего плана в развертывании Enterprise Edition и на серверах Standard Edition, которые можно реализовать в конфигурации для пула или сайта.

  - **Интеграция с Microsoft Exchange.**   Во время развертывания архивирования можно интегрировать хранилище данных архивирования с существующим хранилищем Exchange 2013 для всех пользователей, размещенных в Exchange 2013 и чьи почтовые ящики помещены на хранение на месте, чтобы не требовалось развертывать отдельные базы данных SQL Server для архивирования данных Lync. Если развертывание Exchange 2013 отсутствует, или если вы предпочитаете не проводить интеграцию с этой системой, а также если есть какие-либо пользователи Lync 2013, которые не размещены в Exchange 2013, и их почтовые ящики помещены на хранение на месте, можно развернуть отдельные базы данных архивирования с помощью SQL Server для хранения архивных данных из взаимодействий Lync. Можно использовать одновременно интеграцию с Microsoft Exchange и базы данных архивирования Lync Server 2013, если требуется использовать интеграцию с Microsoft Exchange для некоторых, но не для всех пользователей в развертывании. Подробные сведения о хранении на месте см. в статье «Хранение на месте» по адресу [http://go.microsoft.com/fwlink/p/?LinkId=267500](http://go.microsoft.com/fwlink/p/?linkid=267500).

  - **Зеркальное отображение хранилища SQL.**   Во время развертывания архивирования можно включить зеркальное отображение базы данных SQL Server для базы данных архивирования.

  - **Архивирование досок и опросов.**   Архивное содержимое конференций теперь включает доски и опросы, к которым предоставлялся общий доступ во время собрания.

Следующие типы содержимого не могут быть заархивированы:

  - одноранговые передачи файлов;

  - аудио/видео для одноранговых мгновенных сообщений и конференций;

  - общий доступ к приложениям в одноранговом обмене мгновенными сообщениями и конференциях.
