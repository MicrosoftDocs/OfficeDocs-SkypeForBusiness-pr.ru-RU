# Проверка пользовательской репликации выполнена

 

_**Дата изменения раздела:** 2012-09-17_

При выполнении командлета **Move-CsUser** может возникать ошибка, так как пользовательские сведения между доменными службами Active Directory (AD DS) и базами данных Lync Server 2013 не синхронизированы по причине неполноты первоначальной репликации. Время, затрачиваемое на успешное выполнение первоначальной синхронизации служб репликатора пользовательских данных Lync Server 2013, зависит от количества доменных контроллеров, размещенных в лесу Active Directory, в котором размещается пул Lync Server 2013. Процесс первоначальной синхронизации служб репликатора пользовательских данных Lync Server 2013 происходит, когда клиентский сервер Lync Server 2013 запускается в первый раз. После этого синхронизация выполняется с интервалом репликатора пользовательских данных. Чтобы проверить, что репликация пользовательских данных завершена до запуска командлета **Move-CsUser** , выполните следующие действия.

## Чтобы проверить, что репликация пользовательских данных завершена

1.  Войдите на компьютер, где установлен построитель топологий, с использованием учетной записи, входящей в группу администраторов домена и группу RTCUniversalServerAdmins.

2.  Нажмите кнопку **Пуск** и выберите пункт **Выполнить** .

3.  Введите **eventvwr.exe** , затем нажмите кнопку **ОК** .

4.  В окне просмотра событий щелкните **Журналы приложений и служб** , чтобы развернуть окно, а затем выберите "Lync Server".

5.  В области **Действия** щелкните пункт **Фильтровать текущий журнал** .

6.  В списке **Источники событий** щелкните пункт **LS User Replicator** .

7.  В поле **\<Все коды событий\>** введите **30024** , затем нажмите кнопку **ОК** .

8.  В фильтрованном списке событий перейдите на вкладку **Общие** и найдите запись, указывающую, что репликация пользователей успешно завершена.
