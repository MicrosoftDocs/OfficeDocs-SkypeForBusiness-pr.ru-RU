# Получение сведений о местоположении в Lync Server 2013

 

_**Дата изменения раздела:** 2012-06-06_

В развертывании Lync Server 2013 E9-1-1 каждый клиент с внутренним подключением Lync или Lync Phone Edition активно запрашивает собственное местоположение. После регистрации SIP клиент предоставляет все сведения о своем сетевом подключении в запросе местоположения, отправляемом службе информирования о местонахождении, которая является веб-службой, поддерживаемой реплицированной базой данных SQL Server. В каждом пуле центрального сайта имеется служба информирования о местонахождении, которая использует сведения о сети для запроса записей, соответствующих местоположению. Если соответствие существует, служба информирования о местонахождении возвращает местоположение клиенту. Если соответствие отсутствует, пользователю может быть предложено ввести местоположение вручную (в зависимости от параметров политики определения местоположения). Данные о местоположении передаются клиенту в XML-формате стандарта IETF, называемом Presence Information Data Format Location Object (PIDF-LO).

Клиент Lync Server включает данные PIDF-LO в экстренный вызов и эти данные используются поставщиком службы E9-1-1 для определения подходящего пункта PSAP и маршрута вызова этого PSAP, а также правильного ESQK, который позволяет диспетчеру PSAP получить местоположение звонящего.

На следующей схеме показано, как клиент Lync Server запрашивает местоположение (за исключением метода определения местоположения на основе MAC-адреса стороннего клиента).

![Получение расположения клиентом (схема)](images/JJ205110.4438f5fc-f1b2-444b-8565-09035363ed43(OCS.15).jpg "Получение расположения клиентом (схема)")

Чтобы клиент мог получить местоположение, необходимо выполнить следующие действия.

1.  Администратор заполняет базу данных информирования о местонахождении картой географических соответствий сети (таблицами, которые сопоставляют различным типам сетевых адресов соответствующие местоположения аварийно-спасательных служб (ERL)).

2.  Если используется поставщик службы E9-1-1 с магистралью SIP, администратор проверяет физические адреса в ERL по базе данных основного справочника адресов (MSAG), поддерживаемой поставщиком службы E9-1-1. Если используется шлюз ELIN, администратор проверяет, что оператор ТСОП отправляет данные ELIN в базу данных автоматического определения местоположения (ALI).

3.  Во время регистрации или при изменении сети клиент с внутренним подключением отправляет службе информирования о местонахождении запрос местоположения, который содержит обнаруженные сетевые адреса клиента.

4.  Служба информирования о местонахождении ищет местоположение в своих опубликованных записях и, если находит соответствие, возвращает клиенту ERL в формате PIDF-LO.
