# Процесс миграции

 

_**Дата изменения раздела:** 2012-09-24_

Рекомендуемым и поддерживаемым способом миграции для Lync Server 2013 является параллельная миграция. В этом разделе описываются причины для использования параллельной миграции и приводятся сведения о сосуществовании.

## Параллельная миграция

Почти во всех случаях следует использовать параллельную миграцию. При этом способе новый сервер Lync Server 2013 развертывается в дополнение к соответствующему серверу Office Communications Server 2007 R2, после чего операции переносятся на новый сервер. Если необходимо выполнить откат к Office Communications Server 2007 R2, то вам нужно просто перенести операции обратно на исходные серверы. Имейте в виду, что в этом случае проведение новых собраний, запланированных с помощью обновленных клиентов, будет невозможно, и клиенты также потребуется откатить до предыдущей версии.

## Тестирование сосуществования

После параллельного развертывания Lync Server 2013 вместе с Office Communications Server 2007 R2 топология находится в состоянии тестирования сосуществования Lync Server 2013 и Office Communications Server 2007 R2. Пока топология находится в этом состоянии, необходимо убедиться в том, что службы запускаются, все сайты доступны для администрирования и клиенты обеспечивают взаимодействие как с новыми, так и со старыми пользователями. Перед миграцией всех пользователей крайне важно оценить состояние каждого развертывания и убедиться в его работоспособности. Как правило, тестирование сосуществования продолжается в течение всего периода пилотного тестирования Lync Server 2013. Существующие пользователи перемещаются в Lync Server 2013 на некоторое время для проверки совместимости приложений и правильной работы компонентов и функций. По завершении пилотного тестирования пользователи и приложения переносятся в рабочую версию Lync Server 2013, а старые пулы и приложения Office Communications Server 2007 R2 выводятся из эксплуатации.
