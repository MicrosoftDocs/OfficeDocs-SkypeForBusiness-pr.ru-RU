# <a name="e9-1-1-support-in-lync-server-2013"></a>Поддержка E9 – 1 – 1 в Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2012-09-21_

Lync Server 2013 поддерживает расширенные 9-1-1 (E9-1-1) в рамках корпоративного развертывания. E9-1-1 — это компонент экстренного уведомления, который связывает номер телефона вызывающей стороны с городским адресом. Поддержка E9-1-1 доступна только в США.

Для поддержки E9-1-1 в развертывании Lync Server 2013 необходимо получить службу маршрутизации E9-1-1 от сертифицированного поставщика аварийных служб или использовать шлюз с идентификационным номером службы экстренного размещения (ELIN). Поставщик служб экстренного реагирования или перевозчик маршрутизирует экстренные вызовы, полученные из Lync Server 2013, в соответствующую общедоступную точку безопасности (PSAP) на основе сведений о расположении, содержащихся в вызове. Подробные сведения о поддержке E9-1-1 приведены [в статье Планирование экстренных служб (E9-1-1) в Lync Server 2013](lync-server-2013-planning-for-emergency-services-e9-1-1.md) в документации по планированию.
