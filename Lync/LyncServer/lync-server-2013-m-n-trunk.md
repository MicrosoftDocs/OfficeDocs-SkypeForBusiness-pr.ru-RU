<span data-ttu-id="36726-104">Lync Server 2013 обеспечивает большую гибкость в определении магистрали для маршрутизации вызовов из предыдущих выпусков.</span><span class="sxs-lookup"><span data-stu-id="36726-104">Lync Server 2013 supports greater flexibility in the definition of a trunk for call routing purposes from previous releases.</span></span> <span data-ttu-id="36726-105">Магистраль — это логическая связь между сервером-посредником и номером прослушивающего порта с шлюзом и номером прослушивающего порта.</span><span class="sxs-lookup"><span data-stu-id="36726-105">A trunk is a logical association between a Mediation Server and listening port number with a gateway and a listening port number.</span></span> <span data-ttu-id="36726-106">Это подразумевает несколько моментов: сервер-посредник может иметь несколько магистральных линий связи с одним шлюзом; Сервер-посредник может иметь несколько магистральных линий связи между разными шлюзами; напротив, шлюз может иметь несколько магистральных каналов для разных серверов-посредников.</span><span class="sxs-lookup"><span data-stu-id="36726-106">This implies several things: A Mediation Server can have multiple trunks to the same gateway; a Mediation Server can have multiple trunks to different gateways; conversely a gateway can have multiple trunks to different Mediation Servers.</span></span>

<span data-ttu-id="36726-107">Корневая магистраль по-прежнему должна создаваться при добавлении шлюза в топологию Lync с помощью построителя топологий.</span><span class="sxs-lookup"><span <span data-ttu-id="36726-111">При определении маршрута вызовов указываются магистральные линии связи, связанные с этим маршрутом, но не указываются серверы-посредники, связанные с этим маршрутом.</span><span class="sxs-lookup"><span data-stu-id="36726-111">When defining a call route, you specify the trunks associated with that route, but you do not specify which Mediation Servers are associated with that route.</span></span> <span data-ttu-id="36726-112">Вместо этого вы используете построитель топологий для связи магистрали с серверами-посредниками.</span><span class="sxs-lookup"><span data-stu-id="36726-112">Instead, you use Topology Builder to associate trunks with Mediation Servers.</span></span> <span data-ttu-id="36726-113">Другими словами, служба маршрутизации определяет, какая магистральная линия будет использоваться для вызова, а затем сервер-посредник, связанный с этой магистралью, отправляет сигнал для этого вызова.</span><span class="sxs-lookup"><span data-stu-id="36726-113">In other words, routing determines which trunk to use for a call, and, subsequently, the Mediation Server associated with that trunk is sent the signaling for that call.</span></span>

<span data-ttu-id="36726-114">Сервер-посредник можно развернуть как пул; Этот пул можно разместить в пуле переднего плана, или он может быть развернут как изолированный пул.</span><span class="sxs-lookup"><span data-stu-id="36726-114">The Mediation Server can be deployed as a pool; this pool can be collocated with a Front End pool, or it can be deployed as a stand-alone pool.</span></span> <span data-ttu-id="36726-115">Если сервер-посредник размещен в пуле переднего плана, размер пула не может превышать 12 (предельный размер пула регистратора).</span><span class="sxs-lookup"><span data-stu-id="36726-115">When a Mediation Server is collocated with a Front End pool, the pool size can be at most 12 (the limit of the Registrar pool size).</span></span> <span data-ttu-id="36726-116">В совокупности эти новые возможности увеличивают надежность и гибкость развертывания серверов-посредников, но для них требуются связанные возможности в следующих одноранговых сущностях:</span><span class="sxs-lookup"><span data-stu-id="36726-116">Taken together, these new capabilities increase the reliability and deployment flexibility for Mediation Servers, but they require associated capabilities in the following peer entities:</span></span>

  - <span data-ttu-id="36726-119">**Пограничный контроллер сеансов.**</span><span class="sxs-lookup"><span data-stu-id="36726-119">**Session Border Controller.**</span></span> <span data-ttu-id="36726-120">Для магистрали SIP одноранговый объект является пограничным контроллером сеансов (SBC) у поставщика услуг Интернет-телефонии.</span><span class="sxs-lookup"><span
