# Использование пакетов управления Lync Server 2010 в сценариях сосуществования

 

_**Дата изменения раздела:** 2012-10-22_

Многие клиенты используют в своих компаниях программу выгрузки, в которой пользователи прогрессивно переносятся из Microsoft Lync Server 2010 в Lync Server 2013. Администраторов в этих компаниях заботит мониторинг обеих версий Lync Server для обеспечения лучшего качества взаимодействия конечных пользователей. Для этого сценария пакет управления Lync Server 2013 поддерживает параллельную миграцию с пакетом управления Lync Server 2010.

В Lync Server 2010 компьютеры Lync Server можно было обнаружить с помощью документа топологии, который хранился в центральном хранилище управления. В этой конфигурации один компьютер сообщал о существовании всех других компьютеров Lync Server.

Теперь пакеты управления для Lync Server 2013 используют обнаружение на уровне компьютера вместо механизма центрального обнаружения, который применялся в Lync Server 2010. Это значит, что каждый агент System Center обнаруживает себя и сообщает о своем существовании System Center Operations Manager. Обнаружение на уровне компьютера упрощает администрирование инфраструктуры System Center и облегчает сосуществование разных версий пакетов управления Lync Server (например, пакетов управления для Lync Server 2010 и Lync Server 2013).

Для поддержки такой миграции сначала потребуется обновить существующие средства мониторинга Lync Server 2010, чтобы избежать недостаточного покрытия. Для этого выберите существующий компьютер Lync Server 2010 для обслуживания скрипта центрального обнаружения для Lync Server 2010 перед обновлением центрального хранилища управления до Lync Server 2013. Это процесс из четырех шагов:

1.  Обновите пакеты управления Lync Server 2010 до накопительного пакета обновления 7.

2.  Выполните скрипт центрального обнаружения на компьютере Lync Server 2010.

3.  Переопределите кандидат центрального обнаружения в пакете управления Microsoft Lync Server 2010.

4.  Убедитесь, что кандидат центрального обнаружения был обнаружен.

## Выполнение скрипта центрального обнаружения на компьютере Lync Server 2010

Чтобы сделать так, чтобы сервер, не используемый для центрального хранилища управления (например, интерфейсный сервер Lync Server), обрабатывал центральное обнаружение, требуется создать следующий раздел реестра на сервере, не используемом для центрального хранилища управления:

HKLM\\Software\\Microsoft\\Real-Time Communications\\Health\\CentralDiscoveryCandidate

Этот раздел реестра можно создать с помощью следующей процедуры:

1.  В меню **Пуск** щелкните **Выполнить**.

2.  В диалоговом окне **Выполнить** введите **regedit**, затем нажмите ВВОД.

3.  В редакторе реестра разверните **HKEY\_LOCAL\_MACHINE**, затем **SOFTWARE**, разверните **Microsoft**, затем **Real-Time Communications**.

4.  Щелкните правой кнопкой пункт **Health**, выберите **Создать**, затем щелкните **Раздел**. Если раздел **Health** не существует, щелкните правой кнопкой **Real-Time Communications**, выберите пункт **Создать** и щелкните **Раздел**. После создания раздел введите "Health" и нажмите клавишу ВВОД.
    
    После создания нового раздела введите **CentralDiscoveryCandidate** и нажмите клавишу ВВОД, чтобы переименовать раздел.

Для вступления этого изменения в силу может потребоваться несколько часов. Чтобы это произошло незамедлительно остановите и перезапустите службу агента работоспособности. Чтобы перезапустить службу агента работоспособности, выполните следующие действия на компьютере Lync Server 2010:

1.  Нажмите кнопку **Пуск**, последовательно щелкните пункты **Все программы**, **Стандартные**, щелкните правой кнопкой мыши значок **Командная строка** и в контекстном меню выберите пункт **Запуск от имени администратора**.

2.  В окне консоли введите следующую команду и нажмите клавишу ВВОД:
    
        Net stop HealthService

3.  Появится сообщение: "Выполняется остановка службы System Center Management", после чего отображается второе сообщение об остановке службы. После этого в можете перезапустить службу, введя следующую команду и нажав клавишу ВВОД:
    
        Net start HealthService

## Переопределение кандидата центрального обнаружения в пакете управления Lync Server 2010

После настройки компьютера Lync Server 2010 для отчетности по компьютерам Lync Server 2010 вам также потребуется уведомить пакет управления Lync Server 2010 об этом изменения. Для этого необходимо создать переопределение в этом пакете управления. Это можно сделать посредством следующей процедуры:

1.  В консоли Operations Manager щелкните **Создание**.

2.  На вкладке "Создание" разверните узел **Объекты пакета управления**, щелкните **Обнаружения объекта** и выберите **Область**.

3.  В диалоговом окне **Определение области объектов пакета управления** выберите элемент с конечным объектом **Кандидат обнаружения LS** и нажмите кнопку **ОК**. Обратите внимание, что кандидат обнаружения LS Discovery отображается, только если вы установили пакет управления Lync Server 2010.

4.  В консоли Operations Manager щелкните правой кнопкой **Кандидат обнаружения LS** выберите пункт **Переопределения**, наведите указатель на параметр **Переопределить обнаружение объектов** и щелкните **Для всех объектов класса: кандидат обнаружения LS**.

5.  В диалоговом окне **Переопределение свойств** установите флажок **Переопределить** рядом с параметром **WatcherNode Fqdn центрального обнаружения**. Введите полное доменное имя компьютера Lync Server 2010 в полях **Другое значение** и **Действительное значение**. Установите флажок **Принудительный** и нажмите кнопку **ОК**.

После создания переопределения нужно перезапустить службу работоспособности на корневом сервере управления. Для этого выполните следующие действия на корневом сервере управления:

1.  Нажмите кнопку **Пуск**, последовательно щелкните пункты **Все программы**, **Стандартные**, щелкните правой кнопкой мыши значок **Командная строка** и в контекстном меню выберите пункт **Запуск от имени администратора**.

2.  В окне консоли введите следующую команду и нажмите клавишу ВВОД:
    
        Net stop HealthService

3.  Появится сообщение: "Выполняется остановка службы System Center Management", после чего отображается второе сообщение об остановке службы. После этого в можете перезапустить службу, введя следующую команду и нажав клавишу ВВОД:
    
        Net start HealthService

## Проверка того, что кандидат центрального обнаружения был обнаружен

Последний шаг перед обновлением центрального хранилища управления — убедиться, что кандидат центрального обнаружения был найден пакетом управления Lync Server 2010. Для этого откройте консоль Operations Manager и щелкните "Мониторинг". На вкладке "Мониторинг" разверните узел **Работоспособность Microsoft Lync Server 2010**, разверните **Обнаружение топологии**, а затем щелкните **Просмотр состояния обнаружения**. Убедитесь, что в строке есть **путь**, в котором указано полное доменное имя кандидата центрального обнаружения. Вы также должны убедиться, что состояние компьютера указано как **Работоспособен**.
