# Включение мониторинга в Lync Server 2013

 

_**Дата изменения раздела:** 2012-10-17_

Хотя агенты единой системы сбора данных автоматически устанавливаются и активируются на каждом из серверов переднего плана, это не означает, что сбор данных мониторинга начнется автоматически сразу же после установки Microsoft Lync Server 2013. Для этого необходимо выполнить два действия: необходимо связать серверы переднего плана/интерфейсные пулы с базой данных мониторинга, а также разрешить мониторинг регистрации вызовов (CDR) и/или качества взаимодействия (QoE) в глобальной области и/или области сайта.

Пошаговые инструкции по привязке серверов переднего плана или интерфейсных пулов к базе данных мониторинга доступны в разделе [Связь хранилища мониторинга с пулом переднего плана в Lync Server 2013](lync-server-2013-associating-a-monitoring-store-with-a-front-end-pool.md) в руководстве по развертыванию. После выполнения привязки и публикации новой топологии Lync Server, сбор данных мониторинга будет по-прежнему невозможен. Это объясняется тем, что по умолчанию сбор данных о регистрации вызовов и качестве взаимодействия отключен при установке Lync Server 2013.

Чтобы начать сбор данных потребуется активировать мониторинг регистрации вызовов и качества взаимодействия (обратите внимание, что активировать и то, и другое необязательно; при желании можно включить только один тип мониторинга, а второй оставить выключенным). Чтобы включить мониторинг регистрации вызовов в глобальной области, выполните следующую команду из Командная консоль Lync Server:

    Set-CsCdrConfiguration -Identity "global" -EnableCDR $True

Как вариант, можно включить мониторинг регистрации вызовов в панели управления Lync Server 2013. В панели управления Lync Server выполните следующую процедуру:

1.  Щелкните **Мониторинг**.

2.  На вкладке **Регистрация вызовов** дважды щелкните параметр **Глобальные**.

3.  В области **Изменить настройку регистрации вызовов (CDR)** выберите **Разрешить мониторинг регистрации вызовов** и нажмите **Зафиксировать**.

Чтобы разрешить мониторинг качества взаимодействия в глобальной области, выполните эту команду в Командная консоль Lync Server:

    Set-CsQoEConfiguration -Identity "global" -EnableQoE $True

При желании можно также включить мониторинг качества взаимодействия в панели управления Lync Server. На панели управления выполните следующую процедуру:

1.  Щелкните **Мониторинг**.

2.  На вкладке **Данные о качестве взаимодействия** дважды щелкните параметр **Глобальные**.

3.  В области **Изменить настройку качества взаимодействия (CDR)** выберите **Разрешить мониторинг качества взаимодействия** и нажмите **Зафиксировать**.

Как уже отмечено ранее, предыдущие примеры демонстрируют включение мониторинга в глобальной области, то есть, включение мониторинга регистрации вызовов и качества взаимодействия в пределах организации. Как вариант, можно также создать отдельные параметры конфигурации регистрации вызовов и качества взаимодействия в области сайта, после чего выборочно включить или отключить мониторинг для каждого из сайтов. Например, можно включить мониторинг регистрации вызовов для сайта Redmond и отключить при этом мониторинг регистрации вызовов для сайта Dublin. Дополнительные сведения об управлении настройками конфигурации мониторинга см. в разделе [Настройка регистрации вызовов и параметров качества взаимодействия в Lync Server 2013](lync-server-2013-configuring-call-detail-recording-and-quality-of-experience-settings.md) руководства по развертыванию.
