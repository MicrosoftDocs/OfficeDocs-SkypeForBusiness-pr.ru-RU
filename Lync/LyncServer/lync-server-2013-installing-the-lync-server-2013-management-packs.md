# <a name="installing-the-lync-server-2013-management-packs"></a>Установка пакетов управления Lync Server 2013

_**Последнее изменение темы:** 2012-10-22_

У System Center Operations Manager есть возможность отслеживать только небольшую часть операционной системы Windows. Однако вы можете расширить возможности System Center Operations Manager, устанавливая пакеты управления, программное обеспечение, которое определяет, какие элементы будут отслеживаться с помощью System Center Operations Manager, в том числе способ отслеживания этих элементов и способ активации и включения оповещений. Microsoft Lync Server 2013 включает два пакета управления System Center Operations Manager, которые предоставляют следующие возможности:

  - Пакет управления компонентом и пользовательским пакетом (Microsoft.LS.2013.Monitoring.ComponentAndUser.mp) отслеживает проблемы Lync Server, записанные в журналах событий, зарегистрированные счетчиками производительности или зарегистрированные в журналах сведений о вызовах (CDR) или в базах данных качества взаимодействия (QoE). Для критических проблем можно настроить System Center Operations Manager для немедленного уведомления администраторов с помощью электронной почты, обмена мгновенными сообщениями или обмена сообщениями в службе коротких сообщений (SMS). (SMS или Short Message Service — это технология, которая используется для отправки текстовых сообщений с одного мобильного устройства на другое.)

  - Активный пакет мониторинга (Microsoft.LS.2013.Monitoring.ActiveMonitoring.mp) проактивно тестирует ключевые компоненты Lync Server, такие как вход в систему, Обмен мгновенными сообщениями и звонки на телефон, находящийся в телефонной сети общего пользования (PSTN). Эти тесты выполняются с помощью командлетов искусственной транзакции Lync Server. Например, командлет **Test-CsIM** имитирует сеанс обмена мгновенными сообщениями между двумя тестовыми пользователями. Если тестовый сеанс заканчивается с ошибкой, то создается оповещение.

В число двух пакетов управления, включенных в Lync Server 2013, входит большое количество улучшений по сравнению с пакетами управления, используемыми с Microsoft Lync Server 2010. Например, пакет управления Lync Server 2013 Component не ограничивается только мониторингом сервера Lync Server. В дополнение к мониторингу журналов событий и счетчиков производительности для Lync Server пакет управления Components также может отслеживать производительность и выдачу оповещений для критически важных элементов, таких как:

  - **Службы IIS (Internet Information Services)**     Если служба IIS перейдет в автономный режим, будут выдаваться оповещения. Это важно, так как веб-службы Lync Server основываются на службах IIS.

  - **Использование процесса**     Оповещения будут выдаваться в случае нехватки системных ресурсов (например, доступной памяти). Эти оповещения будут выдаваться, даже если Lync Server не отвечает за высокую загрузку системы.

  - **События**     сбоя компьютера Оповещения будут выдаваться в случае аппаратной или программной ситуации, которая может угрожать жизнеспособности сервера. Например, администраторы Lync Server будут уведомлены, если на сервере возникла опасность сбоя жесткого диска.

В новых пакетах управления также были расширены функции создания отчетов. Новые отчеты для Lync Server 2013 включают:

  - **Отчет о**     доступности в сценарии "от" до конца В этом отчете подробно описана доступность и время работы для ключевых служб Lync Server, таких как регистрация или присутствие.

  - **Отчет о емкости**     С помощью данных счетчиков производительности этот отчет показывает тенденции для компонентов системы, таких как доступность памяти и загрузка процессора.

  - **Отчет о**     компонентах В этом отчете представлены топ генераторов оповещений, сгруппированных по компонентам Lync Server.

В дополнение к этим предназначенным отчетам пакеты управления для Lync Server 2013 автоматически отчитываются об оповещениях для надежности (метрики, измеряемые при регистрации вызовов) и состояния QoE (метрики измеряются качеством взаимодействия). Если вы включили запись сведений о вызовах, вы можете просмотреть оповещения о надежности вызовов, выполнив следующую процедуру в консоли System Center Operations Manager:

  - Последовательно разверните узлы **Monitoring**, **Microsoft Lync Server 2013 Health** (Работоспособность Microsoft Lync Server 2013), **Call Reliability and Media Quality** (Стабильность звонков и качество медиаданных) и щелкните **Call Reliability** (Стабильность звонков).

Чтобы просмотреть оповещения о качестве качества взаимодействия, выполните эту процедуру в консоли System Center Operations Manager:

  - Последовательно разверните узлы **Monitoring**, **Microsoft Lync Server 2013 Health**, **Call Reliability and Media Quality** и **Media Quality** (Качество медиаданных).

Пакеты управления для Lync Server 2013 теперь используют обнаружение на уровне компьютера вместо централизованного механизма обнаружения, используемого в Microsoft Lync Server 2010. Это означает, что каждый агент System Center в основном обнаруживает себя и сообщает о существовании сервера центрального управления. Использование функции обнаружения на уровне компьютера упрощает администрирование инфраструктуры System Center, а также разрешает совместное использование различных версий пакетов управления Lync Server (например, пакетов управления для Lync Server 2010 и пакетов управления для Lync Server 2013).
