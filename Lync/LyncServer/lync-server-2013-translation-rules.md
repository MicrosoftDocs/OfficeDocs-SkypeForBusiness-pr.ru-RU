# <a name="translation-rules-in-lync-server-2013"></a>Правила преобразования в Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

_**Последнее изменение темы:** 2012-10-05_

Lync Server 2013 Enterprise Voice требует, чтобы все строки набора номера были нормализованы до формата E. 164 для выполнения обратного просмотра номера (RNL). В Microsoft Lync Server 2010 правила преобразования поддерживаются только для вызываемых номеров. В Microsoft Lync Server 2013 правила преобразования также поддерживаются для номеров звонков. *Магистральный узел* (т.е. соответствующий шлюз, УАТС или канал SIP) может требовать, чтобы номера были в формате местного набора. Чтобы преобразовать номера из формата E.164 в формат местного набора, можно определить правила преобразования для работы с URI запроса перед его направлением в магистральный узел. Например, можно написать правило преобразования, чтобы заменить +44 в начале строки набора на 0144.

Выполняя преобразование исходящего маршрута на сервере, можно уменьшить требования к конфигурации в каждом отдельном магистральном узле для трансляции телефонных номеров в формат местного набора. При планировании того, какие шлюзы и сколько шлюзов необходимо связать с конкретным кластерным сервером-посредником, может быть удобно группировать магистральные одноранговые узлы с одинаковыми локальными требованиями набора номера. Это позволяет уменьшить необходимое количество правил преобразования и сэкономить время, затрачиваемое на их создание.

<div>


> [!IMPORTANT]  
> Связывание одного или нескольких правил преобразования с конфигурацией магистрали корпоративной голосовой связи следует использовать в качестве альтернативы настройке правил преобразования на одноранговом одноранговом узле. Не связывайте правила преобразования с конфигурацией магистрали корпоративной голосовой связи, если вы настроили правила преобразования на одноранговом одноранговом узле, так как эти два правила могут конфликтовать.
