# Подключение пилотного пула к старым пограничным серверам

 

_**Дата изменения раздела:** 2012-10-02_

После развертывания Lync Server 2013 маршрут федерации для этого сайта не настроен. Чтобы использовать федеративный маршрут, который используется Office Communications Server 2007 R2, необходимо настроить Lync Server 2013 для использования этого маршрута.

Чтобы сайт Lync Server 2013 мог использовать директор и пограничный сервер сайта BackCompatSite, настройте сопоставление старого пограничного пула с помощью построителя топологии.

## Связь устаревшего пограничного пула с помощью построителя топологий

1.  Откройте пилотную топологию пула в построителе топологии.

2.  Выберите сайт Lync Server 2013.

3.  В меню **Действие** щелкните **Изменить свойства** .

4.  В разделе **Назначение федеративного маршрута к узлу** выберите параметр **Включить SIP-федерацию**, затем выберите директора Office Communications Server 2007 R2 или пограничный сервер Office Communications Server 2007 R2, если директор отсутствует в списке.
    
    ![Диалоговое окно "Изменение свойств", страница федеративного маршрута](images/JJ205136.bc13014b-3578-4d9e-9ff7-bdd09130b676(OCS.15).jpg "Диалоговое окно \"Изменение свойств\", страница федеративного маршрута")  

5.  Нажмите кнопку **ОК** , чтобы закрыть страницу **Изменение свойств** .

6.  В построителе топологий в узле Lync Server 2013 перейдите к **серверу Standard Edition** или в **интерфейсные пулы Enterprise Edition Front End** , щелкните пул правой кнопкой мыши и выберите команду **Изменить свойства** .

7.  В разделе **Associations** (Связи) установите флажок **Associate Edge pool (for media components)** (Связать пограничный пул (для компонентов медиа)).

8.  Выберите из списка интерфейс пограничного сервера для BackCompatSite.
    
    ![Диалоговое окно "Изменение свойств", страница "Общие"](images/JJ205136.75045212-03ca-4b82-8337-5dacb487094f(OCS.15).jpg "Диалоговое окно \"Изменение свойств\", страница \"Общие\"")  

9.  Нажмите кнопку **ОК** , чтобы закрыть страницу **Изменение свойств** .

10. В **построителе топологий** выберите самый верхний узел **Lync Server**.

11. В меню **Action** (Действие) выберите команду **Publish Topology** (Опубликовать топологию) и нажмите кнопку **Далее** .

12. Когда **мастер публикации** завершит работу, нажмите кнопку **Готово** .
