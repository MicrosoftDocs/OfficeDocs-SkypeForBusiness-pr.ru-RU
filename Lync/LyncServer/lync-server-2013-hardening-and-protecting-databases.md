# <a name="hardening-and-protecting-the-databases-of-lync-server-2013"></a>Усиление защиты и защита баз данных Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2013-12-05_

Microsoft Lync Server 2013 также зависит от баз данных SQL Server для хранения сведений о пользователях, состояния конференций, архивных данных и записей сведений о вызовах (CDRs). Вы можете максимально увеличить доступность данных Lync Server 2013 для серверных баз данных Lync Server путем секционирования данных приложения таким образом, чтобы повысить отказоустойчивость и упростить устранение неполадок. Для достижения этих целей Разбейте данные приложения, выполнив следующие действия:

  - **Использование практических рекомендаций**   по разбиению на разделы сервера для разделения файлов операционной системы, приложений и программ из файлов данных.

  - **Хранение файлов журнала транзакций и файлов**   базы данных в отдельном хранилище для повышения отказоустойчивости и оптимизации восстановления и их хранение на зашифрованном диске или томе.

  - **Использование кластера кластеров серверов**   фоновые серверы для оптимизации доступности системы Lync Server 2013.

  - **Убедитесь, что все резервные копии данных шифруются и правильно обрабатывают**   потерянные, отброшенные или неправильно размещенные носители резервных копий, могут представлять значительную угрозу безопасности данных для развертываний Lync Server 2013

На любом сервере Lync Server 2013, кроме сервера Standard Edition, экземпляр SQL Server Express (экземпляр RTCLOCAL) недоступен для удаленного доступа, а исключения локального брандмауэра не создаются, за исключением SQL Server Express на сервере Standard Edition. На сервере Standard Edition для удаленного доступа к серверной базе данных и центральному хранилищу управления (CMS) можно использовать удаленный доступ. Для защиты баз данных SQL Server можно выполнить следующие действия:

  - Настройте брандмауэр SQL Server Express на серверах Standard Edition, ограничивая область серверов, которые могут осуществлять удаленный доступ к базе данных. По умолчанию любой IP-адрес может получить удаленный доступ к базе данных.

  - С помощью диспетчера конфигураций SQL Server укажите протоколы, IP-адреса и порты для удаленного доступа SQL Server:
    
      - Lync Server 2013 использует протокол TCP/IP. Он поддерживает протокол IP версии 4 (IPv4), но не IP версии 6 (IPv6).
        
        <div>
        

        > [!NOTE]  
        > Lync Server 2013 может работать в сети с включенным двойным стеком IP.

        
        </div>
    
      - Lync Server 2013 поддерживает несколько IP-адресов (сетевых карт с несколькими сетевыми адресами). Можно указать, что SQL Server прослушивает только определенные IP-адреса (индивидуальный адрес или подсеть) и используют только определенные протоколы.
    
      - Lync Server 2013 поддерживает статические и динамические порты SQL Server.

  - Запустите SQL Server на статическом (не по умолчанию) порту и не запускайте браузер SQL Server (поэтому он не может сообщить прослушивающий порт клиенту). Для этого требуется настраиваемая конфигурация для каждого клиента SQL Server, включая серверы переднего плана, сервер мониторинга, сервер архивации и консоль администрирования (с помощью Командная консоль Lync Server, панель управления Lync Server или построитель топологий), а также все остальные серверы, на которых выполняются базы данных Lync Server).

<div>


> [!NOTE]  
> Доступ к базам данных должен быть ограничен только доверенными администраторами баз данных. Злонамеренный администратор базы данных может вставлять или изменять данные в базах данных для получения доступа к серверам Lync Server 2013 или получения конфиденциальной информации от служб, даже если администратору базы данных не предоставлен прямой доступ или Управление серверами Lync Server 2013.



</div>
