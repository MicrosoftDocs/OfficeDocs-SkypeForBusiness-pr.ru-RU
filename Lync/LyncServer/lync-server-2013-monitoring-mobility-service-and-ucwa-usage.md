# <a name="monitoring-mobility-service-and-ucwa-usage-in-lync-server-2013"></a>Мониторинг службы Mobility Service и использования UCWA в Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2013-02-14_

На постоянной основе необходимо отслеживать ресурсы ЦП и памяти, используемые службой Mobility Server Mobility Service (MCX), и веб-API объединенных коммуникаций (UCWA). Для мониторинга использования можно использовать следующие компоненты:

**Для веб-интерфейса API объединенных коммуникаций (UCWA):**

  - Рабочий процесс **линкуква** в ДИСПЕТЧЕРЕ служб IIS. На панели **Рабочие процессы** изучите столбцы **ЦП %** и **Частные байты (КБ)** (память).

  - Счетчики производительности **ЦП** и **Процессор**.

Для большинства развертываний использование ЦП UCWA должно быть менее 15% в среднем. Использование памяти должно попадать в пределы, описанные в разделе [наблюдение за пределами объема памяти сервера в Lync server 2013](lync-server-2013-monitoring-for-server-memory-capacity-limits.md).

В дополнение к счетчикам использования ЦП и памяти можно использовать следующие счетчики производительности, чтобы определить, когда сервер перегружается с запросами.

  - **Ls: Интернет — регулирование и проверка подлинности \\ ВЕБ — общее количество обрабатываемых запросов**, которое указывает количество ожидающих веб-запросов на сервере. Когда этот счетчик достигает 10 000, последующие запросы завершатся неудачей с сообщением об ошибке "503 — Служба недоступна".

  - **ASP.NET \\ Число запросов в очереди** (всегда должно быть равно нулю).

<div>


> [!NOTE]  
> Если вы удовлетворяете или превышаете эти значения, необходимо повторно посетить и повторно вычислить планирование мощности для правильного определения размера ЦП, количества ядер и памяти для компьютеров, на которых размещаются веб-службы.



</div>

**Для службы Mobility Service (MCX):**

  - Рабочие процессы **ксинтмкксапппул** и **Ксекстмкксапппул** в диспетчере служб IIS. На панели **Рабочие процессы** изучите столбцы **ЦП %** и **Частные байты (КБ)** (память).

  - Счетчики производительности **ЦП** и **Процессор**.

Для большинства развертываний загрузка ЦП службы Mobility Service должна быть менее 15% в среднем. Использование памяти должно попадать в пределы, описанные в разделе [наблюдение за пределами объема памяти сервера в Lync server 2013](lync-server-2013-monitoring-for-server-memory-capacity-limits.md).

Помимо счетчиков производительности ЦП и памяти можно использовать следующие счетчики производительности ASP.NET, которые позволят определить степень перегрузки сервера запросами.

  - **ASP.NET v 2.0.50727 \\ Текущее количество запросов**, которое указывает количество ожидающих веб-запросов на сервере. Когда значение этого счетчика достигнет значения 5 000, последующие запросы завершатся неудачей с сообщением об ошибке "503 — Служба недоступна".

  - **ASP.NET \\ Число запросов в очереди** (всегда должно быть равно нулю).

<div>


> [!NOTE]  
> Если вы отвечаете или превышают эти значения, необходимо повторно посетить и повторно вычислить планирование мощности для правильного определения размера ЦП, количества ядер и памяти для компьютеров, на которых размещаются веб-службы.



</div>
