# <a name="requirements-for-location-based-routing-for-conferencing-in-lync-server-2013"></a>Требования к маршрутизации на основе расположения для конференц-связи в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2013-07-19_

Ниже приведены требования, необходимые для установки и настройки приложения для конференц-связи с маршрутизацией на основе расположения.

  - Накопительный пакет обновления 2 (SP2) для Lync Server 2013 должен быть развернут на всех серверах или в пулах в вашей топологии.

<div>


> [!NOTE]  
> Если на сервере Lync или в пуле в вашей топологии не установлен накопительный пакет обновления 2 (SP2) для Lync Server 2013 или более поздней версии, принудительное применение маршрутизации на основе расположения для собраний Lync не гарантируется.



</div>

  - Служба маршрутизации на основе расположения в Lync Server 2013 является предварительным условием для приложения для конференц-связи с маршрутизацией на основе расположения. Чтобы получить дополнительные сведения о настройке маршрутизации на основе расположения Lync Server 2013, обратитесь к разделу [Настройка маршрутизации на основе расположения](lync-server-2013-configuring-location-based-routing.md).

  - Требования к приложению для конференц-связи с маршрутизацией на основе расположения совпадают с требованиями для маршрутизации на основе расположения Lync Server 2013. Для получения дополнительных сведений обратитесь к разделу [планирование маршрутизации на основе расположения](lync-server-2013-planning-for-location-based-routing.md).

## <a name="mediation-server-requirements-for-consultative-call-transfers"></a>Требования к серверу-посреднику при передаче вызовов Консультативного

Приложение для конференц-связи с маршрутизацией на основе расположения требует развертывания изолированных серверов-посредников, чтобы применять ограничения маршрутизации на основе расположения при передаче вызовов Консультативного.

Для принудительной маршрутизации вызовов на основе расположения Консультативного сервер-посредник должен быть связан только с одним одноранговым сервером-посредником (УАТС, шлюзом SIP и т. д.) в областях сети, где требуется маршрутизация на основе расположения. Если дополнительные одноранговые серверы-посредники развернуты в одном регионе сети, узел сервера-посредника должен быть связан с другим сервером-посредником. Ниже приведено описание этого требования.

  - Один сервер-посредник на несколько одноранговых серверов-посредников, когда передача вызовов Консультативного направляется на одноранговый сервер-посредник через сервер-посредник, настроенный на несколько магистральных каналов SIP, на несколько одноранговых узлов (УАТС и шлюзы), Консультативного Передача вызовов блокируется, чтобы предотвратить бесплатный звонок по протоколу PSTN, если передача вызовов Консультативного разрешена через некоторые магистральные линии SIP, но запрещено через другие магистральные линии SIP.
    
    Например, в случае одного сервера-посредника, обслуживающего одноранговый сервер-посредник шлюза PSTN и узел сервера-посредника УАТС, будет наблюдаться следующее поведение:
    
      - Если пользователь Lync с определенного сайта (то есть сайт 1) пытается передать вызов с конечной точкой PSTN на пользователя Lync с другого сайта (то есть сайта 2) с помощью передачи консультативного, вызов будет запрещен для предотвращения бесплатных звонков по сети PSTN.
    
      - Если пользователь Lync с определенного сайта (то есть сайт 1) пытается передать вызов с помощью конечной точки УАТС на том же сайте (сайте 1) на другой сайт (например, сайт 2) с помощью передачи консультативного, вызов будет запрещен даже в том случае, если он не повлечет потенциальную PSTN Тол l обход.

  - Отдельные серверы-посредники для каждого узла серверов-посредников
    
    Если передача Консультативного нацелена на узел сервера-посредника, то передача Консультативного будет оцениваться относительно одного однорангового сервера-посредника, обслуживаемого сервером-посредником. Вызов будет запрещен или запрещен в соответствии с его потенциалом, который может повлечь за собой невзирая на независимые узлы серверов-посредников, независимо от других серверов-посредников, обслуживаемых отдельными серверами-посредниками.
    
    Например, в случае отдельных серверов-посредников, обслуживающих одноранговый сервер-посредник шлюза PSTN и узел сервера-посредника УАТС, будет наблюдаться следующее поведение:
    
      - Если пользователь Lync с определенного сайта (то есть сайт 1) пытается передать вызов с конечной точкой PSTN на пользователя Lync с другого сайта (то есть сайта 2) с помощью передачи консультативного, вызов будет запрещен для предотвращения бесплатных звонков по сети PSTN.
    
      - Если пользователь Lync с определенного сайта (то есть сайт 1) пытается передать вызов с помощью конечной точки УАТС на том же сайте (сайте 1) на другой сайт (например, сайт 2) с помощью передачи консультативного, вызов будет разрешен, так как он не повлечет за собой потенциального беспроблемного звонка. ## <a name="capabilities-not-supported-by-the-location-based-routing-conferencing-application"></a>Возможности, не поддерживаемые приложением для конференц-связи с маршрутизацией на основе расположения

Следующие возможности не поддерживаются приложением для конференц-связи с маршрутизацией на основе расположения:

  - Конференц-связь с телефонным подключением. Маршрутизация на основе расположения не может применяться для конференц-связи с телефонным подключением. Любой запрос на подключение к данной Конференции не будет ограничен маршрутизацией на основе расположения, даже если организатором конференции является пользователь Lync, для которого включена маршрутизация на основе расположения.

  - Не рекомендуется предоставлять номера доступа к конференциям в регионах, где необходимо применять ограничения маршрутизации на основе расположения.
