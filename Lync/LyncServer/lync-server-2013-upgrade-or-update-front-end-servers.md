# <a name="upgrade-or-update-front-end-servers-in-lync-server-2013"></a>Обновление или обновление серверов переднего плана в Lync Server 2013

<div data-xmlns="https://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="https://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

_**Последнее изменение темы:** 2013-06-28_

Серверы переднего плана в пуле Enterprise Edition объединены в *домены обновления*. Такие домены обновления представляют собой подмножество серверов переднего плана в пуле. Домены обновления создаются автоматически с помощью построителя топологий.

При обновлении серверов необходимо сделать это одним доменом обновления за раз. Переведите каждый сервер в один домен для обновления, обновите и перезапустите его перед переходом к другому домену обновления. Следите за тем, какие домены обновления и серверы были обновлены до сих пор. При обновлении каждого сервера используйте следующую блок-схему.

![Обновление серверов переднего плана](images/upgradeupdatefrontendserverslync2013.png)

<div>

## <a name="to-apply-an-upgrade-to-the-front-end-servers-in-a-pool"></a>Применение обновления к серверам переднего плана в пуле

1.  Выполните следующий командлет на сервере переднего плана в пуле:
    
    **Get — CsPoolUpgradeReadinessState**
    
    Если значение *Пулупградестате* **занято**, подождите 10 минут, а затем повторите **Get-CsPoolUpgradeReadinessState** . Если **вы видите по** крайней мере три последовательных интервала, через каждые 10 минут между попытками, или если вы видите результат **инсуффиЦиентактивефронтендс** для **пулупградестате,** то у этого пула возникла ошибка. Если этот пул сопряжен с другим интерфейсным пулом в топологии аварийного восстановления, необходимо аварийно завершить работу пула и перейти в резервный пул, после чего обновить серверы в этом пуле. Подробнее о том, [как отработка отказа для пула в Lync Server 2013](lync-server-2013-failing-over-a-pool.md).
    
    Когда для параметра *PoolUpgradeState* отобразится значение **Готово**, перейдите к шагу 2.

2.  Командлет **Get – CsPoolUpgradeReadinessState** также возвращает сведения о каждом домене обновления в пуле, а также о том, какие серверы переднего плана находятся в каждом домене обновления. Если значение **реадифорупграде** равно **true** для домена обновления, содержащего сервер или серверы, которые требуется обновить, вы можете безопасно обновить эти серверы. Для этого выполните следующие действия:
    
    1.  Остановите новые подключения к серверам переднего плана, которые планируется обновить, с помощью `Stop-CsWindowsService -Graceful -Verbose` командлета.
        
        <div>
        

        > [!NOTE]  
        > При выполнении этих обновлений сервера во время запланированного простоя сервера можно выполнить этот командлет без параметра "–<STRONG>graceful</STRONG>", как показано ниже: <STRONG>Stop — CsWindowsService</STRONG>. Это приведет к немедленному завершению работы служб без ожидания заполнения всех существующих запросов на обслуживание.

        
        </div>
    
    2.  Обновите серверы, связанные с этим доменом обновления.
    
    3.  Перезапустите серверы и убедитесь, что они принимают новые подключения.

3.  Повторите шаги 1 и 2 для всех остальных доменов обновления в пуле до тех пор, пока не будут обновлены все серверы переднего плана.
