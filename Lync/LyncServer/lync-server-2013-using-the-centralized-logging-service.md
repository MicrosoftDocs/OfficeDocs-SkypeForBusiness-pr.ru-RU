# Использование службы централизованного ведения журналов в Lync Server 2013

 

_**Дата изменения раздела:** 2012-11-01_

централизованная служба ведения журнала — это новая возможность в Lync Server 2013. Это улучшенная замена средств **OCSLogger** и **OCSTracer**, представленных в предыдущих версиях. Вы можете использовать централизованная служба ведения журнала для выполнения следующих задач:

  - запуск ведения журнала на одном или нескольких компьютерах и в пулах из одного расположения с помощью одной команды;

  - остановка ведения журнала на одном или нескольких компьютерах и в пулах из одного расположения с помощью одной команды;

  - поиск в журналах на одном или нескольких компьютерах и в пулах из одного расположения с помощью одной команды. Вы можете настроить команду поиска, чтобы возвращать все журналы, записанные и сохраненные на всех компьютерах, или возвращать сокращенные результаты с определенными данными.

  - Настройте сеансы ведения журналов следующим образом:
    
      - Определите **сценарий** или используйте сценарий по умолчанию. *Сценарий* в централизованная служба ведения журнала состоит из области действия (глобальной или области узла), имени сценария для идентификации предназначения и одного или нескольких поставщиков. На одном компьютере одновременно можно запускать два сценария.
    
      - Используйте существующий *поставщик* или создайте новый. *Поставщик* определяет, что собирает сеанс ведения журналов, каков уровень детализации, какие компоненты отслеживаются и какие флаги применяются.
        

        > [!TIP]
        > Если вы знакомы с OCSLogger, термин <EM>поставщики</EM> означает коллекцию <STRONG>компонентов</STRONG> (например, S4, SIPStack), <STRONG>тип ведения журнала</STRONG> (например, WPP, EventLog или IIS logfile), <STRONG>уровень трассировки</STRONG> (например, All, verbose, debug) и <STRONG>флаги</STRONG> (например, TF_COMPONENT, TF_DIAG). Эти элементы определяются в поставщике (переменная Windows PowerShell) и передаются в команде централизованная служба ведения журнала.

    
      - Настройте компьютеры и пулы, из которых вы будете собирать данные.
    
      - Определите узел сеанса ведения журналов в параметрах **Узел** (сбор данных только с компьютеров в этом узле) или **Глобальный** (сбор данных со всех компьютеров в развертывании).

централизованная служба ведения журнала — это очень мощное средство, которое может удовлетворить почти все потребности в устранении неполадок — как большие, так и малые. централизованная служба ведения журнала может быть важным инструментом для администратора, решающего различные задачи — от анализа основной причины до выявления проблем производительности. Все показанные примеры используют Командная консоль Lync Server. Существует компонент централизованная служба ведения журнала для командной строки, **CLSController.exe**. Справка для этого средства предоставлена в самом компоненте. Однако из командной строки можно использовать только ограниченный набор функций. С помощью Командная консоль Lync Server вы получаете доступ гораздо к большему числу функций с более богатыми возможностями настройки. Всегда рассматривайте Командная консоль Lync Server как первый и основной метод при использовании централизованная служба ведения журнала.

В данных разделах описывается использование централизованная служба ведения журнала и приводятся примеры применения многих возможностей этого средства.
