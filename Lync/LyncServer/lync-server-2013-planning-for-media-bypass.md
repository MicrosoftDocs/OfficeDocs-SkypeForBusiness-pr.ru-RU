# Планирование обхода серверов-посредников в Lync Server 2013

 

_**Дата изменения раздела:** 2012-09-21_

Обход сервера-посредника означает исключение сервера-посредника из пути (когда это возможно) для звонков, сигналы которых проходят через сервер-посредник.

Обход сервера-посредника может улучшить качество звука, поскольку уменьшается задержка, потребность в преобразовании, вероятность потери пакетов и число точек вероятного сбоя. Может улучшиться масштабируемость, поскольку устранение обработки мультимедиа для вызовов с обходом сервера-посредника уменьшает нагрузку на сервер-посредник. Это уменьшение нагрузки дополняет возможность сервера- посредника управлять несколькими шлюзами.

Когда сайт филиала без сервера- посредника подключается к центральному сайту с помощью связей WAN с ограниченной пропускной способностью, обход сервера-посредника понижает требования к пропускной способности, позволяя мультимедиа от клиента на сайте филиала проходить напрямую к своему локальному шлюзу без предварительного прохода по связи WAN к серверу- посреднику на центральном сайте и обратно.

Освобождая сервер- посредник от обработки мультимедиа, обход сервера-посредника может уменьшить количество Серверы-посредники, которое требуется для инфраструктуры корпоративной голосовой связи.
