# Прямые SIP-подключения в Lync Server 2013

 

_**Дата изменения раздела:** 2012-08-13_

Вы можете использовать новые *прямые подключения SIP* для подключения Lync Server к одному из следующих компонентов:

  - УАТС на базе протокола IP (подробные сведения см. в статье [Варианты развертывания прямого SIP-подключения в Lync Server 2013](lync-server-2013-direct-sip-deployment-options.md))

  - Шлюз ТСОП (подробные сведения см. в статье [Параметры развертывания шлюза ТСОП в Lync Server 2013](lync-server-2013-pstn-gateway-deployment-options.md))

Чтобы реализовать прямое подключение SIP, необходимо выполнить, по существу, те же действия по развертыванию, что и при реализации магистрали SIP. В обоих случаях реализуется подключение с помощью внешнего интерфейса посредник. Единственное отличие заключается в том, что магистрали SIP подключаются к внешнему объекту, такому как шлюз поставщика услуг Интернет-телефонии, и устанавливаются прямые подключения SIP к внутреннему объекту внутри локальной сети, например к шлюзу телефонной сети общего пользования (ТСОП) или УАТС на базе протокола IP.
