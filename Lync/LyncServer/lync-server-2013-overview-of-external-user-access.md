# <a name="overview-of-external-user-access-in-lync-server-2013"></a>Обзор доступа внешних пользователей в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2013-11-07_

В этой документации мы используем термин *внешний пользователь* , чтобы определить большую категорию пользователей, взаимодействующих с пользователями lync Server 2013 и Lync 2013, которые находятся за пределами брандмауэра. Внешние пользователи, которые могут быть авторизованы для связи Lync Server 2013 с внутренними пользователями (то есть пользователи, которые входят в Lync Server из этого брандмауэра) могут включать следующие компоненты:

  - **Удаленные пользователи**   — пользователи вашей организации, которые входят в Lync Server извне брандмауэра.

  - **Пользователи федеративных пользователей**   с учетной записью с доверенной организацией клиента или партнером, например Lync Server 2010, Lync Server 2013 или Office Communications Server 2007 R2. Федеративные пользователи также могут быть членами определенных партнерских организаций с помощью протокола XMPP (Extensible Messaging and Presence Protocol) с помощью прокси-сервера XMPP на пограничном сервере и шлюза XMPP на сервере переднего плана или в пуле. Определенное отношение доверия, называемое Федерации, не связано с отношением доверия доменных служб Active Directory или зависит от него.
    
    <div>
    

    > [!NOTE]  
    > Дата окончания срока жизни 2014 для AOL и Yahoo! объявлено. Дополнительные сведения см <A href="lync-server-2013-support-for-public-instant-messenger-connectivity.md">в разделе Поддержка общедоступной службы обмена мгновенными сообщениями в Lync Server 2013</A>.

    
    </div>

  - **Общедоступная служба обмена мгновенными сообщениями пользователи**   контакты, которые пользователи устанавливают через общедоступные службы обмена мгновенными сообщениями (Windows Live, Yahoo\! и AOL).

  - **Пользователи мобильных пользователей**   , которые являются участниками вашей организации, которые используют смарт-телефон или планшетный ПК, на котором работает мобильный клиент Lync, вход во внутреннее развертывание и возможность связи с другими классами пользователей. Мобильный пользователь использует службы Mobility Services, которые публикуются с помощью обратного прокси-сервера для доступа к внутреннему развертыванию. Дополнительные сведения о функциях и возможностях, доступных в Lync Mobile, можно найти в [http://go.microsoft.com/fwlink/p/?LinkID=234777](http://go.microsoft.com/fwlink/p/?linkid=234777)таблице сравнения мобильных клиентов по адресу.

  - **Анонимные пользователи**   пользователи, не имеющие учетной записи в доменных службах Active Directory или в поддерживаемом федеративного домене, но получившие приглашения удаленно участвовать в локальной конференции.

Пограничный сервер предоставляет внешний доступ для следующих типов обмена данными:

  - **Обмен мгновенными сообщениями и присутствие**   авторизованных внешних пользователей могут участвовать в беседах и конференциях, а также получать информацию о состоянии присутствия. Пользователи общедоступных служб обмена мгновенными сообщениями могут участвовать в беседах с отдельными пользователями Lync Server в Организации и получать доступ к сведениям о присутствии, но не могут участвовать в многосторонних конференциях для обмена мгновенными сообщениями с помощью Lync Server. Это строго одноранговая связь. Передача файлов не поддерживается для пользователей общедоступных служб обмена мгновенными сообщениями, аудио-и видеоданные в одноранговых подключениях поддерживаются для пользователей Windows Messenger 2011, но не для других пользователей общедоступных поставщиков услуг обмена мгновенными сообщениями.
    
    Поддерживаются оба протокола: SIP и XMPP. Чтобы предоставить услуги для XMPP, обратитесь к разделу [планирование для SIP, Федерации XMPP и общедоступной службы обмена мгновенными сообщениями в Lync Server 2013](lync-server-2013-planning-for-sip-xmpp-federation-and-public-instant-messaging.md).

  - ****   Авторизованные внешние пользователи веб-конференций могут участвовать в конференциях, размещенных в развертывании Lync Server. Удаленные пользователи, Федеративные пользователи и анонимные пользователи могут быть включены для участия в веб-конференциях, но общедоступные пользователи обмена мгновенными сообщениями не могут участвовать в конференциях. В зависимости от выбранных параметров пользователи с включенной поддержкой веб-конференций могут участвовать в совместном доступе к рабочему столу и приложениям и могут выступать в качестве организаторов или докладчиков.

  - **** Прошедшие проверку подлинности аудио-и видеоконференций могут участвовать в аудио-и видеоконференциях, которые размещаются на серверах развертывания Lync Server.    Аудио и видео в одноранговых подключениях поддерживаются для пользователей Windows Messenger 2011, но не для других пользователей общедоступных служб обмена мгновенными сообщениями.

Для управления взаимодействием можно настроить одну или несколько политик, определяющих связь между пользователями в Организации и за ее пределами. Кроме того, можно настроить параметры и применить политики для отдельных внутренних пользователей или для определенных типов внешних пользователей, чтобы управлять связью с внешними пользователями.

Lync Server 2013: роли, которые используются для предоставления внешнего доступа:

**Пограничный сервер**   . пограничный сервер — это сервер или пул серверов, на которых выполняются службы, которые разрешают внешний доступ к IM, сведения о присутствии, конференц-связи, аудио-и видеосвязи, а также другие носители (например, передача файлов). Кроме того, можно настроить пограничный сервер на Федерацию с другими развертываниями Lync Server или Office Communications Server 2007 R2 и другими развертываниями XMPP. Дополнительная возможность подключения к общедоступным системам обмена мгновенными сообщениями включается и настраивается через пограничный сервер.

**Директор**   директор является дополнительным сервером или пулом серверов, на котором работает роль директора Lync Server 2013, которая предварительно проверяет подлинность запросов пользователей и направляет запросы на основной сервер переднего плана или интерфейсный пул пользователей, но не предоставляет никаких учетных записей пользователей.

**Обратный прокси**   обратный прокси — это общий термин для специализированных серверов, где публикуются ресурсы, доступные во внутренней сети, и извлекают сведения о клиентах из опубликованного ресурса. Lync Server 2013 использует обратный прокси-сервер для публикации ряда функций, таких как собрания по конференциям, расположения для присоединения к Конференции, адресная книга, расширение списка рассылки, загрузка контента собраний, обновлений устройств, служб Mobility Services и т. д. Можно использовать любой обратный прокси-сервер, который может удовлетворять требованиям для публикации необходимых расположений ресурсов. Microsoft Forefront Threat Management Gateway (TMG) 2010 используется в качестве примера для демонстрации необходимых правил публикации, но Forefront TMG 2010 не требуется.

<div>


> [!IMPORTANT]  
> Lync Server 2013 поддерживает как IPv4, так и IPv6. Windows Server&nbsp;2008&nbsp;r2, Windows Server 2012 и Windows Server 2012 R2 используют двойной стек, который может одновременно использовать IPv4 и IPv6. Это важно в связи с переходным характером развертывания из IPv4 в IPv6. IPv4 может поддерживаться в некоторых областях, где в других областях развертывания можно использовать IPv6. Это особенно важно, где связаны Интернет и внутренние развертывания. Внешние клиенты должны общаться через обратный прокси-сервер, чтобы использовать такие службы, как мобильность, собрания, Загрузка адресной книги и другие. В настоящее время Forefront Threat Management Gateway 2010 и Internet Security and Acceleration Server 2006 не поддерживают адресацию IPv6 независимо от версии операционной системы, в которой они развернуты. Необходимо спланировать в соответствии с использованием IPv6 и IPv4, так как они связаны с внешними клиентами.
