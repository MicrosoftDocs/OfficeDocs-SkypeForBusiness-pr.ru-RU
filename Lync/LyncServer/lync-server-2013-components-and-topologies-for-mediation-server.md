# Компоненты и топологии для сервера-посредника в Lync Server 2013

 

_**Дата изменения раздела:** 2012-09-21_

В данном разделе описываются компоненты, от которых зависит посредник, и топологии, в которых посредник может быть развернута.

## Зависимости

посредник имеет следующие зависимости:

  - **Регистратор.** Требуется. Регистратор является следующим прыжком для передачи сигналов во время взаимодействия посредник с сетью Lync Server 2013. Обратите внимание, что посредник может быть размещена на переднего плана совместно с регистратором, а также может быть установлена в отдельном пуле, содержащем только Серверы-посредники. Регистратор размещается совместно с посредник и шлюзом ТСОП на для обеспечения связи в филиалах.

  - **Сервер мониторинга.** Необязателен, но настоятельно рекомендуется. мониторинга позволяет посредник записывать измеряемые параметры качества, связаные с сеансами мультимедиа.

  - **Пограничный сервер.** Требуется для поддержки внешних пользователей. сервер позволяет посредник взаимодействовать с пользователями, расположенными за пределами системы трансляции сетевых адресов (NAT) или брандмауэра.

## Топологии

Lync Server 2013, посредник по умолчанию размещается совместно с экземпляром регистратора на Сервер Standard Edition, в переднего плана или для обеспечения связи в филиалах. Все Серверы-посредники в переднего плана должны быть настроены идентично.

В тех случаях, когда производительность является важным фактором, может быть предпочтительно развернуть одну или несколько Серверы-посредники в выделенном отдельном пуле. Либо, если развертывается магистраль SIP, мы рекомендуем развернуть отдельный серверов-посредников.

Если вы развертываете прямые подключения SIP к квалифицированному шлюзу ТСОП, поддерживающему обход сервера-посредника и балансировку нагрузки DNS, отдельный пул серверов-посредников не является необходимым. Отдельный серверов-посредников не является необходимым из-за того, что квалифицированные шлюзы могут производить балансировку нагрузки DNS для пула Серверы-посредники, а также получать трафик из любой посредник в пуле.

Мы также рекомендуем размещать совместно посредник на переднего плана, когда развернуты IP-PBX, подключенные к пограничному контроллеру сеансов (SBC) поставщика услуг интернет-телефонии, и выполняются следующие условия:

  - IP-PBX или SBC настраиваются для получения трафика из любой посредник в пуле и могут маршрутизировать трафик единообразно на все Серверы-посредники в пуле.

  - IP-PBX не поддерживает обход сервера-посредника, но переднего плана, в котором размещается посредник, может обрабатывать перекодировку голосовой связи для вызовов, к которым не применяется обход сервера-посредника.

С помощью средства планирования Microsoft Lync Server 2013 можно оценить, в состоянии ли переднего плана, в котором предполагается совместно размещать посредник, обрабатывать нагрузку. Если ваша среда не может соответствовать этим требованиям, то необходимо развернуть изолированный серверов-посредников.

Дополнительные сведения о том, какую топологию следует развертывать, см. в разделе [Рекомендации по развертыванию для сервера-посредника в Lync Server 2013](lync-server-2013-deployment-guidelines-for-mediation-server.md).
