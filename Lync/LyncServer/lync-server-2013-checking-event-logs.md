# <a name="checking-event-logs-in-lync-server-2013"></a>Проверка журналов событий в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2014-08-06_

С помощью [средства просмотра событий Windows](https://go.microsoft.com/fwlink/p/?linkid=314067) можно просматривать журналы событий и получать сведения об отказах служб, ошибках репликации в ДОМЕНных СЛУЖБАХ Active Directory и предупреждениях о системных ресурсах, таких как виртуальная память и дисковое пространство. Средство просмотра событий входит в состав Windows Server 2008 и 2012.

В средстве ведения журнала Lync Server 2013 при завершении сеанса отладки щелкните **Анализ файлов журнала** для просмотра файлов журнала с помощью средства snooper.

В средстве просмотра событий хранятся журналы событий приложений, безопасности и системных событий компьютера. В журналы событий выводятся предупреждения и сообщения об ошибках в Lync Server 2013 и Windows. Поэтому просмотрите журналы событий ежедневно.

С помощью средства просмотра событий можно выполнять следующие действия:

  - Просмотр журналов событий и управление ими.

  - Получение сведений о неполадках оборудования, программного обеспечения и системы, которые необходимо устранить.

  - Определите тенденции, для которых требуется будущее действие.

Сервер, на котором работает Lync Server, в операционной системе Windows Server записывает события в четырех типах журналов:

  - **Журнал приложений.** журнал приложений содержит события, регистрируемые приложениями или программами.    События, регистрируемые в журнале, определяют разработчики. Например, СУБД может регистрировать в журнале приложений ошибки, возникшие при работе с файлами. Большинство событий, связанных с Lync Server 2013, отображаются в журнале приложений.

  - **Журналы безопасности записи**   журнала безопасности, такие как действительные и недопустимые попытки входа, могут воздействовать в дополнение к событиям, связанным с использованием ресурсов, таких как создание, открытие или удаление файлов или других объектов. Например, если включен аудит входа в систему, попытки входа в систему будут регистрироваться в журнале безопасности.

  - **System — журналы**   системный журнал содержит события, записанные системными компонентами Windows. Например, в системном журнале регистрируются сбои при загрузке драйвера или другого системного компонента. Типы событий, регистрируемых системными компонентами, предопределены сервером.

  - **Lync Server 2013**   средство ведения журнала записывает существенные события, связанные с проверкой подлинности, подключениями и действиями пользователя. После включения сбора данных диагностики можно просмотреть записи журнала в средстве просмотра событий.

<div>


> [!NOTE]  
> Не рекомендуется использовать параметры максимального числа журналов, если вы не хотите делать это в службе поддержки пользователей Майкрософт. Ведение максимального журнала приводит к потере значительных ресурсов и может дать множество ложных срабатываний, то есть ошибки, заносимые в журнал только в течение максимального числа событий, но которые являются ожидаемыми и не являются причиной проблем. Кроме того, рекомендуется не включать функцию сбора данных диагностики без возможности восстановления. Включайте эту функцию только для устранения неполадок.



</div>

В каждом журнале просмотра событий в Lync Server 2013 записывается информация о событиях, предупреждениях и ошибках. Внимательно отслеживайте эти журналы, чтобы отслеживать типы транзакций, выполняемых на серверах Lync Server 2013. Во избежание нехватки свободного места на дисках следует периодически архивировать журналы или использовать функцию их автоматической перезаписи. Так как файлы журналов могут занимать ограниченный объем пространства, увеличьте размер журнала (например, до 50 МБ) и присвойте ему значение overwrite, чтобы сервер Lync Server 2013 мог продолжать создавать новые события.

Кроме того, можно автоматизировать администрирование журнала событий с помощью следующих средств и технологий:

  - System Center Operations Manager отслеживает работоспособность и использование серверов Lync Server 2013. Пакет управления Lync Server 2013 для Operations Manager расширяет Operations Manager, предоставляя специализированный мониторинг для серверов, на которых работает Lync Server 2013.

  - Пакет управления Lync Server 2013 для мониторинга Operations Manager выполняет мониторинг стандартного и корпоративного выпуска Lync Server 2013. Кроме того, этот выпуск включает пакет управления качества взаимодействия (QoE), который ранее был пакетом управления.

Отслеживаемые типы — это записи в журнале событий, счетчики производительности и мониторинг состояния QoE. Эта версия пакета управления только отслеживает Lync Server 2013 и 2010 и не может использоваться для мониторинга Office Communications Server 2007.

Пакет управления предоставляет следующие возможности:

  - Оповещения, указывающие на события, затрагивающие службы

  - Оповещения, указывающие на конфигурацию и другие проблемы, не связанные с обслуживанием

  - Мониторинг состояния служб Lync Server

  - Сведения о продукте: причина и решение обнаруженных проблем

Для получения дополнительных сведений о пакете управления Lync Server 2013 обратитесь к разделу [Monitoring Lync server 2013 с System Center Operations Manager](lync-server-2013-monitoring-lync-server-with-system-center-operations-manager.md).

**Объединение**   событий средство "объединение событий" собирает определенные события из журналов событий нескольких компьютеров в одно центральное расположение. Он позволяет сообщать только о кодах событий или источниках событий, которые он указывает. Дополнительные сведения о событии с объединением событий можно найти на веб-сайте " [Блокировка учетных записей и средства управления](https://go.microsoft.com/fwlink/?linkid=35607) ".

**Триггеры событий в**   Windows Server 2012 вы можете "присоединить задачу к этому событию" в средстве просмотра событий Windows, где администратор может запустить программу, отправить сообщение электронной почты или отобразить сообщение на экране. Дополнительные сведения об этой функции можно найти в разделе Windows Server 2008 R2 [выполнение задачи в ответ на определенное событие](https://technet.microsoft.com/library/cc748900.aspx). Кроме того, с помощью средств командной строки, таких как "EventTrigger. exe", можно создавать журналы событий и запрашивать их, а также связывать программы с определенными событиями в журнале. С помощью Евенттригжерс. exe можно создавать триггеры событий, которые запускают программы при возникновении определенных событий.
