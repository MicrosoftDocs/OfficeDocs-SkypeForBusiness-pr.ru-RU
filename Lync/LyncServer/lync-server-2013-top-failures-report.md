# Отчет по самым частым отказам в Lync Server 2013

 

_**Дата изменения раздела:** 2015-03-09_

Отчет по основным сбоям позволяет узнать о наиболее частых сбоях и характере их изменения с течением времени. Сбои основаны на сочетании двух следующих метрик:

  - **Код диагностики** . Уникальный идентификатор (в виде заголовка ms-diagnostics), прикрепленный к сообщению SIP. Коды диагностики предоставляют информацию, полезную при устранении проблем со звонками.

  - **Код ответа** . Коды ответа используются в сеансах связи SIP для ответы на запросы SIP. Предположим, что пользователь Ken отправляет запрос INVITE пользователю Pilar Ackerman (например, пользователь Ken Myer звонит пользователю Pilar Ackerman). Если Pilar отвечает, ее телефон отправляет код ответа 200 (OK), позволяя телефону Ken узнать об ответе Pilar. Отчет по основным сбоям включает в себя только те коды ответа, которые были отправлены в ответ на сбой звонка; сервер Lync Server не отслеживает все коды ответа, использованные в рамках звонка.

Приводится информация не только об общем числе сеансов, в которых возник сбой, но и об общем числе пользователей, столкнувшихся с этим сбоем.

## Доступ к отчету по основным сбоям

Доступ к отчету по основным сбоям можно получить с домашней страницы отчетов по мониторингу. При щелчке метрики отчетных сеансов открывается [Отчет по распределению отказов в Lync Server 2013](lync-server-2013-failure-distribution-report.md).

## Рекомендации по использованию отчета по основным сбоям

Отчет по основным сбоям имеет одну необычную особенность: он позволяет одновременно выполнять фильтрацию по 5 кодам диагностики. (Обычно одновременно вы можете выполнять фильтрацию только по одному из них, например по SIP-адресу пользователя.) Чтобы выполнить фильтрацию по нескольким кодам диагностики, просто введите каждый код в поле "Diagnostic IDs" (Коды диагностики), отделяя его запятой. (При необходимости вы можете ставить после каждой запятой пробел.) Например:

1011, 2412, 1033, 52116, 1008

Сделайте это, и будут отображаться только неудачные звонки, в которых возник один из данных кодов диагностики.

Если навести указатель на код диагностики, отображается подсказка с описание его значения. Например, если навести указатель на код ответа 486, отображается следующее сообщение:

Busy Here (Занято).

## Фильтры

Фильтры позволяют получать более адресный набор данных или просматривать полученные данные разными способами. Например, в отчете по основным сбоям можно фильтровать возвращенные данные по типу действия (одноранговый сеанс или сеанс конференц-связи) или по коду ответа SIP, выданному для неудачного сеанса. Кроме того, можно выбрать способ группирования данных. В данном случае вызовы использования по часу, дню, неделе или месяцу.
