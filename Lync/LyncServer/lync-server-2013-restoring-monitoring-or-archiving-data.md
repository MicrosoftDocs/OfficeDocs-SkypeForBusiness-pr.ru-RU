# <a name="restoring-monitoring-or-archiving-data-in-lync-server-2013"></a>Восстановление данных мониторинга или архивации в Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2013-02-18_

Восстановление данных мониторинга и архивации не требуется для того, чтобы сделать Lync Server запущенным после сбоя. Тем не менее, если данные мониторинга и архивации важны для вашей организации, необходимо восстановить данные после повторного создания баз данных.

В следующей процедуре описывается, как использовать SQL Server Management Studio для восстановления данных архивации или мониторинга.

<div>

## <a name="to-restore-monitoring-or-archiving-data-from-a-backup-file"></a>Восстановление данных мониторинга или архивации из файла резервной копии

1.  Выполните вход на сервер, который вы восстанавливаете в качестве члена группы "Администраторы" на локальном компьютере или в группе с эквивалентными правами пользователя.

2.  Откройте SQL Server Management Studio: нажмите кнопку **Пуск**, **выберите все программы**, **Microsoft SQL Server 2012** или **Microsoft SQL Server 2008 R2**, а затем щелкните **SQL Server Management Studio**.

3.  В разделе **Подключение к серверу**подключитесь к экземпляру SQL Server, указав по крайней мере имя сервера и сведения о проверке подлинности.

4.  В **обозревателе объектов**щелкните правой кнопкой мыши **базы данных**и выберите команду **восстановить базу данных**.

5.  В разделе **Выбор страницы**щелкните **Общие**, а затем в поле **база данных** выберите имя базы данных следующим образом:
    
      - Для базы данных архивации выберите **LcsLog**.
    
      - Для базы данных регистрации вызовов (CDR) выберите **LcsCDR**.
    
      - Для базы данных качества взаимодействия (QoE) выберите **QoEMetrics**.

6.  Щелкните **с устройства**.

7.  В списке **Выберите резервные наборы данных для восстановления**выберите файл резервной копии и нажмите кнопку **восстановить**.

8.  В разделе **Выбор страницы**щелкните **Параметры**, убедитесь, что путь к файлу данных и путь к журналу находятся в правильной папке, а затем нажмите кнопку **ОК**.

</div>

<div>

## <a name="to-make-sure-that-access-control-lists-acls-are-correct"></a>Проверка правильности списков управления доступом (ACL)

1.  Разверните **узел базы данных, разверните**базу данных архивации или мониторинга, разверните узел **Безопасность**, а затем разверните узел **Пользователи**.

2.  Убедитесь, что группа домена RTCComponentUniversalServices существует в качестве пользователя.

3.  Если RTCComponentUniversalServices не существует в разделе **Пользователи**, выполните следующие действия:
    
    1.  Щелкните правой кнопкой мыши узел **Пользователи** и выберите пункт **Новый пользователь**.
    
    2.  В поле **имя входа**введите недостающее имя группы RTCComponentUniversalServices.
    
    3.  В разделе **членство в роли базы данных**выберите разрешение **serverRole** и нажмите кнопку **ОК**.
    
    <div>
    

    > [!NOTE]  
    > Перезапускать службу архивирования и мониторинга не требуется.
