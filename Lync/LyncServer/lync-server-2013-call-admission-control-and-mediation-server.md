# Контроль допуска звонков и сервер-посредник в Lync Server 2013

 

_**Дата изменения раздела:** 2012-09-21_

Контроль допуска звонков, впервые появившийся в Lync Server 2010, служит для управления созданием сеансов в режиме реального времени в соответствии с доступной пропускной способностью во избежание снижения качества взаимодействия (QoE) в перегруженных сетях. Для поддержки этой возможности сервер-посредник, обеспечивающий передачу сигналов и мультимедийных данных между инфраструктурой корпоративной голосовой связи и шлюзом или поставщиком распределения каналов SIP, управляет пропускной способностью для собственного взаимодействия с Lync Server и шлюзом. При использовании контроля допуска звонков за резервирование пропускной способности отвечает объект-получатель звонка. Одноранговые узлы шлюза (шлюз ТСОП, УАТС с поддержкой IP или пограничный контроллер сеансов), с которыми посредник взаимодействует на стороне шлюза, не поддерживают контроль доступа звонков Lync Server 2013. Таким образом, посредник должен обеспечивать контроль пропускной способности от имени однорангового шлюза. По возможности посредник будет резервировать пропускную способность заблаговременно. Если это невозможно (например, если расположение конечной точки сервера-посредника на стороне шлюза для исходящего звонка на одноранговый шлюз неизвестно), пропускная способность резервируется в момент выполнения звонка. Это может приводить к избыточному выделению полосы пропускания, однако это единственный способ предотвратить ошибочные звонки.

Обход сервера-посредника и резервирование пропускной способности являются взаимоисключающими подходами. Если для звонка используется обход сервера-посредника, контроль допуска звонков не осуществляется. При этом предполагается, что при выполнении звонка отсутствуют каналы с ограниченной пропускной способностью. Если для звонка, задействующего посредник, используется контроль допуска звонков, обход сервера-посредника применяться не может.

Подробные сведения об обходе сервера-посредника и контроле допуска звонков см. в разделах [Планирование обхода серверов-посредников в Lync Server 2013](lync-server-2013-planning-for-media-bypass.md) и [Планирование контроля допуска звонков в Lync Server 2013](lync-server-2013-planning-for-call-admission-control.md) в документации по планированию.
