# <a name="dns-infrastructure-support-in-lync-server-2013"></a>Поддержка инфраструктуры DNS в Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2013-03-08_

Для Lync Server 2013 требуется система доменных имен (DNS) и используется следующими способами:

  - Для обнаружения внутренних серверов или пулов для межсерверной связи.

  - Для предоставления клиентам возможности обнаружения пула переднего плана или сервера Standard Edition, используемого для различных транзакций SIP.

  - Для связи простых URL-адресов для конференций с серверами, на которых размещаются эти конференции.

  - Для предоставления внешним серверам и клиентам возможности подключения к пограничным серверам или обратному прокси-серверу HTTP для обмена мгновенными сообщениями ил конференц-связи.

  - Для предоставления устройствам для объединенных коммуникаций возможности обнаружения пула переднего плана или сервера Standard Edition, на котором функционирует веб-служба обновления устройств, получения обновлений и отправки журналов без необходимости входа.

  - Для предоставления мобильным клиентам возможности автоматического обнаружения ресурсов веб-служб без необходимости ручного ввода URL-адресов в настройках устройств.

  - Для балансировки нагрузки DNS.

<div>


> [!NOTE]  
> Lync Server 2013 не поддерживает международные доменные имена (IDN).



</div>

<div>


> [!IMPORTANT]  
> Указанное имя должно полностью соответствовать имени компьютера, настроенному на сервере. По умолчанию в качестве имени компьютера, который не присоединен к домену, используется короткое имя, а не полное доменное имя (FQDN). Построитель топологии использует полные доменные имена, а не короткие имена. Поэтому необходимо настроить суффикс DNS имени компьютера для развертывания в качестве пограничного сервера, не присоединенного к домену. <STRONG>Используйте только стандартные символы </STRONG> (включая A-Z, a-z, 0-9 и дефисы) при назначении полных доменных имен серверов Lync, пограничных серверов и пулов. Не используйте символы Unicode или подчеркивания. Нестандартные символы, включенные в полное доменное имя, часто не поддерживаются внешними системами DNS и открытыми центрами сертификации (например, когда полное доменное имя необходимо назначить SN в сертификате).
