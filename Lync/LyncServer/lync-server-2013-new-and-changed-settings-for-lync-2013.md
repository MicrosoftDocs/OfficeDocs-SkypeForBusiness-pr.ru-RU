# Новые и измененные параметры для Lync 2013

 

_**Дата изменения раздела:** 2015-03-09_

В этой статье рассматриваются изменения командлетов командной консоли Командная консоль Lync Server, имеющие отношение непосредственно к управлению клиентами. В Lync Server 2013 появилось несколько новых параметров и устарели параметры для функций, которые можно настроить другими способами.

<tr class="odd">
<td><p>TracingLevel</p></td>
<td><p>CsClientPolicy</p></td>
<td><p>При установке значения True будет включена трассировка программного обеспечения в Lync; при установке значения False трассировка программного обеспечения будет отключена. Трассировка программного обеспечения включает сохранение подробной записи всего, что делает программа (в том числе отслеживание вызовов API). Трассировка особенно полезна для разработчиков и сотрудников службы поддержки приложений. Этот параметр эквивалентен параметру &quot;Включить отслеживание программы Communicator&quot; групповой политики Communications Server 2007 R2. Возможны следующие установки:</p>
<ul>
<li><p>* Off — трассировка отключена (пользователь не может изменить этот параметр­);</p></li>
<li><p>* Light — минимальная трассировка (пользователь не может изменить этот параметр);</p></li>
<li><p>* On — подробная трассировка (пользователь не может изменить этот параметр).</p></li>
</ul>
<p>По умолчанию параметр TracingLevel имеет значение NULL. Это означает, что выполняется минимальная трассировка, но пользователь может включить или отключить ее.</p></td>
</tr>
<tr class="even">
<td><p>EnableMediaRedirection</p></td>
<td><p>CsClientPolicy</p></td>
<td><p>Значение True ($True) позволяет отделять видео и аудиопотоки от остального сетевого трафика. Например, это позволяет клиентским устройствам локально кодировать и декодировать видео и звук. Перенаправление мультимедиа позволяет уменьшить используемую полосу пропускания сети, повысить масштабирование сервера и обеспечить более оптимальное взаимодействие с пользователем по сравнению с аналогичными технологиями, такими как удаленное взаимодействие или сжатие с помощью кодека.</p></td>
</tr>
<tr class="odd">
<td><p>AllowLargeMeetings</p></td>
<td><p>CsConferencing</p></td>
<td><p>Если задано значение True, все собрания Lync будут рассматриваться как &quot;большие собрания&quot;. Для &quot;большого собрания&quot; действуют ограничения относительно количества уведомлений, рассылаемых участникам, а также размера списка участников, который передается по умолчанию.</p></td>
</tr>
<tr class="even">
<td><p>DisablePowerPointAnnotations</p></td>
<td><p>CsConferencing</p></td>
<td><p>При установке значения True ($True) пользователи не смогут добавлять аннотации к слайдам PowerPoint, используемым в конференции. Однако (в зависимости от значения свойства AllowAnnotations) у пользователей останется доступ к другим функциям доски. Значение по умолчанию — False, что означает, что аннотирование слайдов PowerPoint разрешено.</p></td>
</tr>
<tr class="odd">
<td><p>AllowSharedNotes</p></td>
<td><p>CsConferencing</p></td>
<td><p>При установке значения True (значение по умолчанию) в открытые записные книжки OneNote, подключенные к конференции, будет автоматически добавлена такая информация, как список участников конференции и сведения об общем содержимом конференции.</p></td>
</tr>
<tr class="even">
<td><p>EnableInviteCustomization</p></td>
<td><p>CsMeetingConfiguration</p></td>
<td><p>Используется наряду с другими новыми параметрами CsMeetingConfiguration для настройки приглашений на собрания, создаваемых надстройкой надстройка собраний по сети для Lync 2013.</p></td>
</tr>

<td><p>LogoURL</p></td>
<td><p>CsMeetingConfiguration</p></td>

</tr>
<tr class="odd">
<td><p>LegalURL</p></td>
<td><p>CsMeetingConfiguration</p></td>
<td><p>Добавляет юридические примечания или текст заявления об отказе во все приглашения, создаваемые надстройкой надстройка собраний по сети для Lync 2013. Вы указываете URL-адрес расположения текста.</p></td>

<td><p>EnableSQMData</p></td>
<td><p>CsClientPolicy</p></td>
<td><p>Параметр EnableSQMData командлета Set-CSClientPolicy, удаленный в Lync Server 2013. Вместо него можно с помощью общего параметра групповой политики для данных управления качеством программного обеспечения (SQM) определить пользовательский интерфейс для параметра &quot;Customer Experience Improvement&quot; (Улучшение качества программного обеспечения) на странице общих параметров клиента Lync:</p>
<p>HKEY_CURRENT_USER\Software\Policies\Microsoft\Office\Common\QMEnable</p>
<p>Значения:</p>
<p>1 — отображать и установить флажок (пользователь может снять этот флажок);</p>
<p>0 — отключить и снять флажок (пользователь не может переопределить это значение);</p>
<p>Null — значение определяется программой установки Office, и флажок отображается пользователям, чтобы они могли установить его по своему выбору.</p></td>
</tr>
<tr class="even">
<td><p>AllowExchangeContactStore</p></td>
<td><p>CsClientPolicy</p></td>
<td><p>Этот параметр был удален. Вместо него при развертывании Lync Server 2013 и публикации топологии для всех пользователей по умолчанию включается единое хранилище контактов. Это означает, что все контакты пользователя сохраняются в Exchange и доступны в Lync, Outlook и Outlook Web Access. Для настройки пользователей, которым доступно единое хранилище контактов, можно использовать командлет Set-CsUserServicesPolicy. Пользователей можно включать глобально, по сайту, по клиенту, а также группами или по-отдельности. Подробные сведения см. в статье <a href="lync-server-2013-enable-users-for-unified-contact-store.md">Включение для пользователей единого хранилища контактов в Lync Server 2013</a>.</p></td>
</tr>
<tr class="odd">
<td><p>MAPIPollInterval</p></td>
<td><p>CsClientPolicy</p></td>
<td><p>В Lync 2013 этот параметр не используется. В предыдущих версиях он определял частоту извлечения клиентом данных MAPI из общедоступных папок Exchange.</p></td>
</tr>
