# Проверка соединений для внешних пользователей в Lync Server 2013

 

_**Дата изменения раздела:** 2012-10-19_

Для проверки возможности подключения внешних пользователей необходимо обеспечить подключение пользователей к серверу и порт для пограничной службы доступа.

Ценным ресурсом подтверждения конфигурации и возможности подключаться, отправлять и принимать допустимые сообщения для сценариев, необходимых для внешнего доступа пользователей, – это сайт анализатора удаленного подключения ( <https://www.testocsconnectivity.com/> ), который управляется и обслуживается службой технической поддержки Microsoft. Чтобы открыть анализатор удаленного подключения, перейдите на указанный веб-сайт в браузере и следуйте инструкциям по выбору сценария.

## Тест подключения внешних пользователей и внешнего доступа

Тесты доступа внешних пользователей должны охватывать все типы внешних пользователей, которые поддерживаются в организации, частично или полностью включая приведенные далее.

  - Пользователи хотя бы из одного федеративного домена, для которых необходимо проверить обмен мгновенными сообщениями, функцию присутствия, аудио- и видеосвязь и общий доступ к рабочим столам.

  - Пользователи каждого общедоступного поставщика услуг обмена мгновенными сообщениями, поддерживаемого в организации (и для которого была выполнена подготовка).

  - Анонимные пользователи.

  - Пользователи в организации, входящие в Lync удаленно, но без использования VPN.

Эти тесты определяют, способен ли ваш пограничный сервер выполнять следующие действия.

  - Прослушивание необходимых портов с помощью клиента Telnet извне вашей сети.
    
      - Пример: telnet sip.contoso.com 443
    
      - Выполнение предыдущего теста в портах, которые используются в пограничном сервере или в пограничном пуле, в зависимости от вашего развертывания.

  - Выполнение точного разрешения внешнего DNS.
    
      - Извне сети проверьте связь для каждого из внешних полных доменных имен вашего пограничного сервера или пограничного пула. Даже если проверка связи выполнится неудачно, вы увидите IP-адреса, которые можно сравнить с назначенными вами.
