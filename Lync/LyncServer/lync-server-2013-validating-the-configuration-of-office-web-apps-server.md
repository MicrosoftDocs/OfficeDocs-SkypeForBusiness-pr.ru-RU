# Проверка конфигурации сервера приложений Office Web Apps в Lync Server 2013

 

_**Дата изменения раздела:** 2014-04-22_

После добавления сервера Office Web Apps в топологию и ее публикации следует просмотреть два новых события в журнале событий Lync Server. Сначала должно быть добавлено событие LS Data MCU (идентификатор события — 41034); это событие уведомляет о том, что был обнаружен сервер Office Web Apps:

**Обнаружен WAC сервера веб-конференций; разрешено содержимое PowerPoint.**

Кроме того, должно отображаться другое событие LS Data MCU (идентификатор события — 41032), которое отправляют URL-адреса сервера Office Web Apps. Это событие может иметь, например, следующий вид:

**Обнаружение WAC сервера веб-конференций успешно выполнено.**

**Внутренняя страница выступающего WAC: https://atl-officewebapps-001.litwareinc.com/m/Presenter.aspx?a=0\&embed=**

**Внутренняя страница участника WAC: https://atl-officewebapps-001.litwareinc.com/m/ParticipantFrame.aspx?a=0\&embed=true&=**

**Внешняя страница выступающего WAC: https://atl-officewebapps-001.litwareinc.com/m/Presenter.aspx?a=0\&embed**

**Внутренняя страница участника WAC: https://atl-officewebapps-001.litwareinc.com/m/ParticipantFrame.aspx?a=0\&embed=true&**

Если событие LS Data MCU отображается с идентификатором события 41033, это означает, что произошел сбой обнаружения сервера Office Web Apps Server. В этом случае Microsoft Lync Server 2013 пытается выполнить столько попыток, сколько нужно для обнаружения недавно настроенного сервера Office Web Apps. Если происходит многократный сбой процесса обнаружения, следует удалить сервер Office Web Apps из документа топологии, опубликовать обновленную топологию, а затем снова попробовать добавить сервер Office Web Apps в топологию после устранения неполадок, связанных с подключением.

Если кажется, что сервер Office Web Apps настроен правильно и был распознан в ходе обнаружения, можно проверить, работает ли сервер Office Web Apps должным образом; для этого предоставьте общий доступ к презентации PowerPoint двум клиентам Microsoft Lync 2013. Если пользователь А может загружать и отображать презентацию PowerPoint, а пользователь Б затем может присоединяться к собранию и просматривать презентацию, сервер Office Web Apps функционирует правильно.

Даже если кажется, что сервер Office Web Apps настроен правильно, при попытке совместного использования презентации PowerPoint существует вероятность возникновения сообщения об ошибке "Некоторые функции общего доступа недоступны из-за проблем с подключением к серверу". В случае отображения этого сообщения об ошибке следует перезапустить серверы переднего плана, сопоставленные с новым сервером Office Web Apps.
