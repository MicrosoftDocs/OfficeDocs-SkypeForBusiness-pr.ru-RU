# <a name="checking-disk-usage-in-lync-server-2013"></a>Проверка использования диска в Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

_**Последнее изменение темы:** 2014-04-30_

Диски с жесткими дисками являются важным компонентом развертывания Lync Server 2013. Без достаточного свободного дискового тома операционная система и базы данных Lync Server 2013 могут работать неправильно. Необходимо ежедневно отслеживать фоновую статистику базы данных Lync Server 2013, чтобы убедиться, что на серверах не хватает дискового пространства, и при необходимости подготовить ресурсы хранилища.

Помимо проверки места на дисках, на которых размещается операционная система, файлы программ, базы данных и журналы транзакций (Lync Server 2013 Back), следует также отслеживать использование файловой системы, в том числе дискового пространства для файловых ресурсов, содержащих следующие важные данные:

  - Содержимое собрания

  - Метаданные контента собраний

  - Журналы соответствия требованиям для собраний

  - Файлы данных приложения (внутренне используются компонентом сервера приложений)

  - Веб-служба сервера группового чата и папки соответствия требованиям (для хранения файлов, отправленных в веб-службу чата)

  - Групповые XML-файлы соответствия групповой беседы (содержащие записи о соответствии группового чата)

  - Обновление файлов (для службы обновления устройств)

  - Файлы адресной книги

Lync Server 2013 требует жесткого диска для хранения баз данных и журналов транзакций в дополнение к файлам в общих файловых ресурсах, перечисленных выше.

Следует регулярно отслеживать дисковое пространство, чтобы обеспечить неблагоприятное влияние развертывания Lync Server 2013 из-за недостатка ресурсов хранилища.

Сравните и настройте статистические сведения о доступном дисковом пространстве на каждом томе Lync Server 2013 и ожидаемом росте баз данных и файлов журналов транзакций. Это помогает при планировании емкости и добавлении хранилища, когда требуются ресурсы хранилища.

Чтобы обеспечить устранение неполадок и аварийное восстановление, рекомендуется, чтобы свободное место на диске было равно или превышать 110 процентов от размера базы данных.

Проверить наличие свободного места на диске можно описанными ниже способами.

1.  **System Center Operations Manager**     System Center Operations Manager можно использовать для предупреждения администраторов о ограничении объема громкости.

2.  **Выполнение скрипта**     Отслеживайте дисковое пространство, выполнив сценарий, который отправляет вам сообщение, если объем свободного места на жестком диске падает ниже 20 процентов. Пример скрипта можно найти в центре сценариев Майкрософт на сайте TechNet, изучите следующее: [https://gallery.technet.microsoft.com/scriptcenter/site/search?query=hard%20disk%20alert\&f%5B0%5D.Value=hard%20disk%20alert\&f%5B0%5D.Type=SearchText\&ac=5](https://gallery.technet.microsoft.com/scriptcenter/site/search?query=hard+disk+alert%26f%5b0%5d.value=hard+disk+alert%26f%5b0%5d.type=searchtext%26ac=5)

3.  **Проводник Windows**     Используйте проводник Windows, чтобы проверить наличие дискового пространства на томах, в которых хранятся журналы и базы данных Lync Server 2013.
