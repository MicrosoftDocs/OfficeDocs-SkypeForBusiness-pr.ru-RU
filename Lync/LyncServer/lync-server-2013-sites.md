# Сайты Lync Server для Lync Server 2013

 

_**Дата изменения раздела:** 2012-10-16_

В Lync Server необходимо определить *сайты* сети, содержащие компоненты Lync Server. Сайт – это набор компьютеров, соединенных с помощью сети с высокой пропускной способностью и низкой задержкой, например локальной сети, или две сети, соединенные с помощью высокоскоростного оптоволоконного канала. Обратите внимание на то, что сайты Lync Server – это не то же самое, что сайты Доменные службы Active Directory и Microsoft Exchange Server. Сайты Lync Server необязательно должны соответствовать сайтам Active Directory.

## Типы сайтов

Каждый сайт представляет собой либо *центральный сайт* , содержащий по крайней мере один переднего плана или Сервер Standard Edition, либо *сайт филиала* . Каждый сайт филиала связан с одним центральным сайтом. Для пользователей филиала большинство возможностей Lync Server предоставляется серверами связанного центрального сайта.

Каждый сайт филиала содержит один из следующих компонентов.

  - *Устройство для обеспечения связи в филиалах (SBA)* , представляющее собой соответствующий отраслевым стандартам сверхкомпактный сервер, на котором размещены Регистратор и сервер-посредник Lync Server под управлением ОС Windows Server. для обеспечения связи в филиалах также содержит шлюз телефонной сети общего пользования (ТСОП). для обеспечения связи в филиалах предназначено для сайтов филиалов с количеством пользователей от 25 до 1000.

  - *Сервер для обеспечения связи в филиалах (SBS)* представляет собой сервер под управлением ОС Windows Server, который соответствует определенным требованиям к оборудованию и на котором установлены Регистратор и сервер-посредник Lync Server. Он должен быть подключен к шлюзу ТСОП или магистрали SIP с поставщиком услуг телефонной связи. Сервер для обеспечения связи в филиалах предназначен для филиалов с количеством пользователей от 1000 до 5000.

  - Шлюз ТСОП и (необязательно) *сервер-посредник* . Сведения об этой и других ролях сервера см. в статье [Роли сервера в Lync Server 2013](lync-server-2013-server-roles.md).

В филиале, подключенном к центральному сайту с помощью отказоустойчивого канала глобальной сети, может использоваться третий вариант (шлюз ТСОП и (необязательно) сервер-посредник). В филиалах с менее отказоустойчивым соединением должно использоваться для обеспечения связи в филиалах или сервер для обеспечения связи в филиалах, который обеспечивает отказоустойчивость в случае сбоев глобальной сети. Например, если на площадке развернуты для обеспечения связи в филиалах или сервер для обеспечения связи в филиалах, пользователи смогут совершать и получать звонки корпоративной голосовой связи, если соединение глобальной сети между филиалом и центральным сайтом отсутствует. Подробные сведения об для обеспечения связи в филиалах, сервере для обеспечения связи в филиалах и отказоустойчивости см. в разделе [Планирование устойчивости корпоративной голосовой связи в Lync Server 2013](lync-server-2013-planning-for-enterprise-voice-resiliency.md) в документации по планированию.

## Топологии сайтов

Развертывание должно содержать, по крайней мере, один центральный сайт и может содержать ни одного или несколько филиалов. Каждый филиал связан с одним центральным узлом. Центральный сайт предоставляет филиалу службы Lync Server, которые не размещены в филиале локально, например сведения о присутствии и конференц-связь.

При использовании нескольких сайтов можно привязать друг к другу интерфейсные пулы на различных сайтах, чтобы предоставить возможности аварийного восстановления. Подробные сведения см. в статье [Поддержка высокой доступности и аварийного восстановления в Lync Server 2013](lync-server-2013-high-availability-and-disaster-recovery-support.md).
