# <a name="setting-up-permissions-for-archiving-in-lync-server-2013"></a>Настройка разрешений для архивации в Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2012-10-01_

В Lync Server 2013 для определенных задач по-прежнему необходимо, чтобы пользователи, выполняющие эти задачи, были членами одной или нескольких определенных групп. Однако вы также можете использовать управление доступом на основе ролей (RBAC), чтобы предоставить привилегии, назначив пользователям предварительно заданные административные роли Lync Server. Перед развертыванием архивации убедитесь, что для этой роли назначены соответствующие права и разрешения пользователя, а также все пользователи, которые вы хотите назначить определенной роли RBAC. Дополнительные сведения о правах, разрешениях и ролях пользователей для развертывания поддержки архивации можно найти в разделе [Контрольный список развертывания для архивации в Lync Server 2013](lync-server-2013-deployment-checklist-for-archiving.md), который доступен в документации по планированию и документации по развертыванию. Подробнее о RBAC можно узнать в статье [Планирование управления доступом на основе ролей в Lync Server 2013](lync-server-2013-planning-for-role-based-access-control.md) в документации по планированию.
