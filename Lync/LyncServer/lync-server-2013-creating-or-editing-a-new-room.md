# Создание или редактирование нового чата в Lync Server 2013

 

_**Дата изменения раздела:** 2015-03-19_

Обычно настройку комнат сохраняемый сеанс беседы выполняют сами пользователи. Администраторы сохраняемый сеанс беседы, как правило, не настраивают комнаты чата и не управляют ими. Командлеты Windows PowerShell, используемые для управления комнатами, доступны только членам группы **CsPersistentChatAdministrator**.

Пользователи, являющиеся членами группы **Создатели** в любой категории, могут создавать комнаты и управлять ими с помощью клиента Lync. Пользователи, назначенные в качестве управляющих определенной комнаты чата, также могут выполнять текущие задачи управления этой комнатой, например изменять ее свойства или параметры членства.


> [!TIP]
> Администраторы сохраняемый сеанс беседы также могут быть создателями, но на них не распространяются ограничения, налагаемые на создателей.



Если вы являетесь администратором сохраняемый сеанс беседы, то для создания комнат чата и управления ими вместо командлетов Windows PowerShell вы можете применять пользовательский интерфейс. Для этого включите поддержку протокола SIP для администратора сохраняемого сеанса беседы и используйте клиент Lync.

Если вы хотите создать настраиваемую рабочую нагрузку для пользователей, задайте свойство **RoomManagementUrl** в конфигурации сохраняемого сеанса беседы для перенаправления пользователей из клиента Lync в ваше настраиваемое решение.

Подробные сведения о настройке комнат чата с помощью командной строки Windows PowerShell см. в подразделе "Управление комнатами" справки [Настройка сервера сохраняемого сеанса беседы с помощью командлетов Windows PowerShell](configuring-persistent-chat-server-by-using-windows-powershell-cmdlets.md).

Дополнительные сведения о настройке чата см. в разделе [Настройка чатов в Lync Server 2013](lync-server-2013-configure-rooms.md) документации по развертыванию.
