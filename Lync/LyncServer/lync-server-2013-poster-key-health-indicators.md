## Что такое ключевые показатели работоспособности?

Ключевые показатели работоспособности – это счетчики производительности с пороговыми значениями, по которым можно определять проблемы взаимодействия с пользователями. Обычно первым шагом в реализации Методики качества вызовов Lync (CQM) является сбор данных ключевых показателей работоспособности (KHI), и этот шаг ориентирован на обеспечение качества звука для пользователей Lync.

Ключевые показатели работоспособности (KHI) используются в дополнение к стандартным решениям Lync для мониторинга (например, System Center Operations Manager, искусственные транзакции, сервер мониторинга), а не вместо этих решений.

Выполните сбор счетчиков производительности по ключевым показателям работоспособности (KHI) и заполните таблицу KHI, поставляемую вместе с Руководством по сетевым подключениям, чтобы получить оценочный лист для определения работоспособности серверов в среде развертывания Lync. Заполненная таблица поможет вам в устранении неисправностей среды, а также позволит получить более подробную информацию для других заинтересованных сторон. Ежемесячно проводите оценку ключевых показателей работоспособности (KHI) и включайте полученные данные в операционные процессы развертывания.

Загрузите документ [Руководство по сетевым подключениям Lync Server](http://go.microsoft.com/fwlink/p/?linkid=390677), чтобы просмотреть полный перечень ключевых показателей работоспособности (KHI) и получить соответствующие таблицы.

## Сбор данных ключевых показателей работоспособности (KHI)

1.  Запустите сценарий KHI из Руководства по сетевым подключениям Lync Server на каждом сервере Lync Server. Внутри монитора производительности (Performance Monitor) будет создан объект «Сборщик данных» (Data Collector). Присвойте ему имя KHI. По умолчанию данные будут опрашиваться каждые 15 секунд.

2.  До начала рабочего дня в вашей компании запустите Сборщик данных KHI на каждом сервере Lync Server.

3.  В конце рабочего дня остановите KHI Data Collector и скопируйте данные на центральный ресурс.

4.  После того, как с помощью Монитора производительности будут заполнены входящие в состав загружаемого Руководства по сетевым подключениям Lync Server таблицы KHI, сравните полученные результаты с рекомендованными целевыми показателями. .

## Процессы исправления для всех серверных ролей

На каждом сервере в среде Lync сначала проверьте работоспособность компонента сервера и убедитесь, что производительность системы не ниже требуемого уровня. Только после этого следует анализировать показатели, имеющие отношение к серверным ролям, во всей среде развертывания Lync.

Начните со сбора данных ключевых показателей работоспособности (KHI) Данные о производительности для всех серверов. Для каждой системной роли (подробнее рассматриваются далее в этом документе) выясните, достигнуты ли рекомендованные целевые показатели для основных компонентов системы. Если эти показатели не достигнуты, выполните исправление неполадок производительности системы, затем выполните сбор данных KHI и обеспечьте необходимую работоспособность системы, прежде чем переходить к изучению метрик, специфичных для той или иной роли сервера в среде развертывания Lync. Работоспособность компонента для всех ролей определяется следующими показателями: :

  - Загрузка ЦП \< 80%

  - Средн. время записи на диск \< 10 мс

  - Средн. время считывания с диска \< 10 мс

  - Available memory  \>20% System Total MB

  - Длина сетевой очереди \< 2

  - Отброшенных пакетов (вх. / исх.) = 0

## Глоссарий

На этом плакате используются следующие термины и акронимы:

AS MCU = Application Sharing Multi-point Control Unit (Многоточечный управляющий узел общего доступа к приложениям)

AV MCU = Audio/Video MCU (Многоточечный управляющий узел аудио/видео)

IM MCU = Instant Messaging MCU (Многоточечный управляющий узел обмена мгновенными сообщениями)

UCWA = Unified Communications Web API (веб-интерфейс API объединенных коммуникаций)

AV Edge = Traversal of audio/video via edge (Обход аудио/видео через границу)

AV Auth = Audio/Video Authentication (Проверка подлинности аудио/видео)

SIP Stack = Содержит базовую реализацию SIP в Lync

Data Proxy = (Прокси для данных) используется для пограничной службы конференций

LySS = Служба Lync Storage Service
