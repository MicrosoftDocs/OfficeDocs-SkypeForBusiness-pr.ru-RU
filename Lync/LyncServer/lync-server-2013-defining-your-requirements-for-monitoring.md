# Определение требований к мониторингу в Lync Server 2013

 

_**Дата изменения раздела:** 2012-09-05_

Упрощение процесса развертывания и установки системы мониторинга в Microsoft Lync Server 2013 также упрощает процессы, связанные с определением требований вашей организации в отношении мониторинга. Тем не менее, по-прежнему существует ряд ключевых проблем, которые необходимо решить, прежде чем приступать к установке и настройке Lync Server 2013:

**Какой тип данных необходимо отслеживать?** Lync Server 2013 позволяет осуществлять мониторинг двух различных типов данных: данных подробной регистрации вызовов (CDR) и данных качества взаимодействия (QoE). Функция подробной регистрации вызовов предоставляет возможность отслеживания использования функций Lync Server, таких как телефонные вызовы по протоколу VoIP, обмен мгновенными сообщениями, передача файлов, аудио- и видеоконференции и сеансы общего доступа к приложениям. Эта информация поможет вам узнать, какие функции Lync Server используются (а какие нет). Кроме того, вы также получаете сведения о том, когда используются эти функции. Показатели качества взаимодействия позволяют осуществлять регистрацию качества аудио- и видеовызовов, выполняемых в организации, включая такие сведения, как число потерянных сетевых пакетов, фоновые помехи и степень «дрожания» (различий в задержке пакетов) .

Если вы выбираете включение мониторинга в Lync Server 2013, можно включить и мониторинг CDR, и контроль QoE, или вы можете включить только один тип мониторинга, оставив другой отключенным. Например, предположим, что пользователи используют только систему обмена мгновенными сообщениями и функции передачи файлов, и не осуществляют аудио- и видеовызовы. В этом случае нет особого смысла в активации мониторинга QoE. Аналогичным образом, Lync Server позволяет с легкостью включать и отключать мониторинг после развертывания этой функции. Например, вы можете выбрать развертывание системы мониторинга, но изначально оставить функцию мониторинга QoE отключенной. Если у пользователей начинают возникать проблемы, связанные с аудио- и видеовызовами, вы можете в последствии включить функцию мониторинга QoE и использовать эти данные для устранения неполадок и решения этих проблем.

Нельзя сказать, что установка системы мониторинга одновременно с установкой Lync Server является более (или менее) выгодной по сравнению с установкой системы мониторинга после установки Lync Server. Однако необходимо иметь в виду, что прежде чем устанавливать систему мониторинга, следует выбрать компьютер для размещения серверного хранилища данных. Кроме того, поддерживаемая версия SQL Server должна быть установлена и настроена на этом же компьютере до того, как компьютер можно будет использовать для мониторинга . Если SQL Server уже установлен на компьютер и компьютер готов к работе, можно установить систему мониторинга одновременно с установкой Lync Server. При отсутствии серверного компьютера, готового к работе, можно продолжить установку Lync Server, а затем развернуть систему мониторинга, когда серверный компьютер будет готов к использованию.

**Когда вы хотите установить систему мониторинга?** Систему мониторинга можно установить и настроить одновременно с установкой и настройкой Lync Server 2013; мастер развертывания Lync Server предоставляет возможность связать пулы переднего плана с базой данных мониторинга в процессе настройки. Кроме того, можно установить систему мониторинга после установки Lync Server; можно использовать построитель топологии для связи пулов переднего плана и серверов с базой данных мониторинга, а затем опубликовать измененную топологию.

**Сколько серверных баз данных мониторинга требуется?** Одна база данных мониторинга может поддерживать десятки тысяч пользователей (для Microsoft Lync Server 2010 было определено, что совместно размещенная база данных мониторинга и архивирования может поддерживать 240 000 пользователей). Кроме того, единая база данных мониторинга может использоваться несколькими пулами переднего плана; если в организации имеются три пула переднего плана, можно связать все эти три пула с одним и тем же серверным хранилищем.

Это просто означает, что для многих организаций объем базы данных не является решающим фактором при определении количества необходимых серверных баз данных мониторинга. Более важным фактором может быть скорость сети. Предположим, у вас имеются три пула переднего плана, но один из этих пулов базируется на сетевом подключении с более низкой скоростью. В этом случае может потребоваться использование двух баз данных мониторинга: одна база данных предназначена для обслуживания двух пулов с высокоскоростным сетевым подключением, а отдельная база данных ? для обслуживания пула с низкоскоростным сетевым подключением.

При планировании инфраструктуры мониторинга следует также принимать во внимание, что должны также принять во внимание, что Lync Server 2013 поддерживает использование зеркальных баз данных. «Зеркалирование баз данных» предоставляет возможность одновременно поддерживать две копии базы данных, причем эти базы данных размещаются на разных серверах. При каждой записи сведений в основную базу данных эти же самые данные также записываются в зеркальную базу данных. Если происходит сбой основной базы данных или она становится недоступной по иной причине, можно «переключиться» на зеркальную базу данных с помощью простой команды Lync Server. Пример:

    Invoke-CsDatabaseFailover -PoolFqdn atl-cs-001.litwareinc.com -DatabaseType "Monitoring" -NewPrincipal "Mirror"

Это имеет большое значение при планировании, так как для использования зеркалирования потребуется дублирование необходимого количества баз данных: в дополнение к каждой основной базе данных вам потребуется вторая база данных, выступающая в качестве зеркала.

**Требуются ли вашим сайтам Lync Server индивидуальные конфигурации мониторинга?** При установке Lync Server 2013 вы также устанавливаете глобальные коллекции CDR и настройки конфигурации QoE; эти глобальные коллекции предоставляют возможность применять одни и те же настройки CDR и QoE для всей организации. Во многих случаях этого достаточно: как правило, необходимо, чтобы, например, мониторинг CDR, был включен для всех пользователей.

Однако в некоторых случаях требуется применение различных настроек для разных сайтов. Например, предположим, вы хотите использовать функции мониторинга CDR и QoE на сайте Redmond, но использовать только мониторинг CDR на сайте Dublin. Аналогичным образом, предположим, что вы хотите хранить данные мониторинга в течение 60 дней на сайте Redmond, однако эти же данные требуется хранить на сайте Dublin в течение всего лишь 30 дней. Lync Server 2013 позволяет создавать отдельные коллекции параметров CDR и QoE на уровне сайта; благодаря этому вы можете управлять всеми сайтами по-разному. (Это подразумевает автономное включение и отключение мониторинга, а также настройку параметров управления, включая длительность хранения данных.)

Обратите внимание, что вы можете принять это решение как перед развертыванием функции мониторинга, так и после ее развертывания. Например, вы можете развернуть систему мониторинга, а затем осуществлять управление всей организацией с помощью глобальных параметров. Если вы впоследствии измените свое мнение, вы сможете создать отдельный набор настроек, например, для сайта Redmond, а затем использовать эти настройки для управления мониторингом сайта Redmond. (Настройки, применяемые на уровне сайта, всегда имеют приоритет над настройками, применяемыми на глобальном уровне.) Если еще раз измените свое решение, можно просто удалить параметры конфигурации, применяемые к сайту Redmond. При удалении набора настроек сайта к этому сайту автоматически применяется глобальный набор параметров.
