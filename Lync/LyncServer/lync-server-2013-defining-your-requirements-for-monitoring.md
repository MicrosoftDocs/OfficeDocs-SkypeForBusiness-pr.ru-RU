# <a name="defining-your-requirements-for-monitoring-in-lync-server-2013"></a>Определение требований для мониторинга в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2012-09-05_

Упрощение развертывания и установки мониторинга в Microsoft Lync Server 2013 также оптимизирует процессы, связанные с определением требований Организации к мониторингу. Тем не менее, прежде чем приступить к установке и настройке Lync Server 2013, необходимо устранить несколько ключевых проблем.

**Какой тип данных требуется отслеживать?** Lync Server 2013 позволяет отслеживать два различных типа данных: сведения о данных (CDR) и данных качества взаимодействия (QoE). Запись сведений о вызовах позволяет отслеживать использование функций Lync Server, таких как телефонные звонки по протоколу VoIP (VoIP). Обмен мгновенными сообщениями (IM); Передача файлов; аудио-и видеоконференции (аудио-и видеоконференций); и сеансов общего доступа к приложениям. Эта информация поможет вам узнать, какие компоненты Lync Server используются (и какие из них не являются), а также сведения о том, когда эти функции используются. С помощью данных качества взаимодействия вы можете поддерживать запись качества звуковых и видеозвонков, выполняемых в Организации, включая такие вопросы, как количество потерянных сетевых пакетов, фоновый шум и степень колебаний (различия в задержке пакетов).

Если вы решили включить мониторинг в Lync Server 2013, вы можете включить мониторинг мониторинга CDR и QoE, а также включить мониторинг одного типа при отключении другого типа. Например, предположим, что пользователи используют только систему обмена мгновенными сообщениями и функции передачи файлов, и не осуществляют аудио- и видеовызовы. В этом случае нет особого смысла в активации мониторинга QoE. Аналогичным образом, Lync Server упрощает включение и отключение мониторинга после развертывания мониторинга. Например, вы можете выбрать развертывание системы мониторинга, но изначально оставить функцию мониторинга QoE отключенной. Если у пользователей начинают возникать проблемы, связанные с аудио- и видеовызовами, вы можете в последствии включить функцию мониторинга QoE и использовать эти данные для устранения неполадок и решения этих проблем.

Не существует определенного преимущества (или недостатков) для установки мониторинга одновременно с установкой Lync Server и установки мониторинга после установки Lync Server. Следует помнить, что перед установкой мониторинга необходимо выбрать компьютер, на котором будет размещено хранилище мониторинга внутреннего сервера, а на компьютере должна быть установлена и настроена поддерживаемая версия SQL Server, прежде чем этот компьютер можно будет использовать для мониторинга. . Если вы уже установили SQL Server на компьютере, и этот компьютер готов к использованию, вы можете установить мониторинг одновременно с установкой Lync Server. Если сервер внутреннего сервера готов, вы можете продолжить установку Lync Server самостоятельно, а затем установить мониторинг всякий раз, когда серверный компьютер готов к использованию.

**Когда вы хотите установить систему мониторинга?** Мониторинг можно устанавливать и настраивать одновременно с установкой и настройкой Lync Server 2013; Мастер развертывания Lync Server предоставит вам возможность связать пулы переднего плана с базой данных мониторинга во время установки. Кроме того, вы можете установить мониторинг после установки Lync Server. Это можно сделать с помощью построителя топологий, чтобы связать интерфейсные пулы и серверы с базой данных мониторинга, а затем опубликовать пересмотренную топологию.

**Сколько серверных баз данных мониторинга требуется?** В одной базе данных мониторинга могут поддерживаться десятки тысяч пользователей (для Microsoft Lync Server 2010, и вы определили, что совместно размещенная база данных для мониторинга и архивации может поддерживать 240 000 пользователей). Кроме того, единая база данных мониторинга может использоваться несколькими пулами переднего плана; если в организации имеются три пула переднего плана, можно связать все эти три пула с одним и тем же серверным хранилищем.

Это просто означает, что для многих организаций объем базы данных не является решающим фактором при определении количества необходимых серверных баз данных мониторинга. Более важным фактором может быть скорость сети. Предположим, у вас имеются три пула переднего плана, но один из этих пулов базируется на сетевом подключении с более низкой скоростью. В этом случае может потребоваться использование двух баз данных мониторинга: одна база данных предназначена для обслуживания двух пулов с высокоскоростным сетевым подключением, а отдельная база данных — для обслуживания пула с низкоскоростным сетевым подключением.

При планировании инфраструктуры мониторинга необходимо также учитывать, что Lync Server 2013 поддерживает использование зеркальных баз данных. «Зеркалирование баз данных» предоставляет возможность одновременно поддерживать две копии базы данных, причем эти базы данных размещаются на разных серверах. При каждой записи сведений в основную базу данных эти же самые данные также записываются в зеркальную базу данных. Если основная база данных завершается с ошибками или становится недоступной в противном случае, можно выполнить команду "отработка отказа" на зеркальную базу данных с помощью простой команды Lync Server PowerShell. Пример:

    Invoke-CsDatabaseFailover -PoolFqdn atl-cs-001.litwareinc.com -DatabaseType "Monitoring" -NewPrincipal "Mirror"

Это имеет большое значение при планировании, так как для использования зеркалирования потребуется дублирование необходимого количества баз данных: в дополнение к каждой основной базе данных вам потребуется вторая база данных, выступающая в качестве зеркала.

**Требуются настраиваемые конфигурации мониторинга для сайтов Lync Server.** При установке Lync Server 2013 Вы также устанавливаете глобальные коллекции параметров конфигурации CDR и QoE; Эти глобальные коллекции предоставляют возможность применять одни и те же параметры CDR и QoE ко всей Организации. Во многих случаях этого достаточно: как правило, необходимо, чтобы, например, мониторинг CDR, был включен для всех пользователей.

Однако в некоторых случаях требуется применение различных настроек для разных сайтов. Например, предположим, вы хотите использовать функции мониторинга CDR и QoE на сайте Redmond, но использовать только мониторинг CDR на сайте Dublin. Аналогичным образом, предположим, что вы хотите хранить данные мониторинга в течение 60 дней на сайте Redmond, однако эти же данные требуется хранить на сайте Dublin в течение всего лишь 30 дней. Lync Server 2013 позволяет создавать отдельные коллекции параметров конфигурации CDR и QoE на уровне сайта; Это позволяет управлять каждым сайтом по-разному. (Это подразумевает автономное включение и отключение мониторинга, а также настройку параметров управления, включая длительность хранения данных.)

Обратите внимание, что вы можете принять это решение как перед развертыванием функции мониторинга, так и после ее развертывания. Например, вы можете развернуть систему мониторинга, а затем осуществлять управление всей организацией с помощью глобальных параметров. Если вы впоследствии измените свое мнение, вы сможете создать отдельный набор настроек, например, для сайта Redmond, а затем использовать эти настройки для управления мониторингом сайта Redmond. (Настройки, применяемые на уровне сайта, всегда имеют приоритет над настройками, применяемыми на глобальном уровне.) Если еще раз измените свое решение, можно просто удалить параметры конфигурации, применяемые к сайту Redmond. При удалении набора настроек сайта к этому сайту автоматически применяется глобальный набор параметров.
