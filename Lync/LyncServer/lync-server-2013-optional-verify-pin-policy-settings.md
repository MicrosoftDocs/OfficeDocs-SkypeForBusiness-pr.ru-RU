# Проверка параметров политики ПИН-кодов в Lync Server 2013 (необязательно)

 

_**Дата изменения раздела:** 2012-06-20_

Пользователи Lync Server 2013, которые обладают учетными данными Доменные службы Active Directory, могут вводить персональный идентификационный номер (ПИН-код) для присоединения к конференциям с телефонными подключениями в качестве прошедших проверку подлинности пользователей. Политика ПИН-кодов определяет правила работы ПИН-кодов конференций с телефонными подключениями.

При развертывании конференц-связи с телефонными подключениями следует убедиться, что глобальная политика ПИН-кодов по умолчанию удовлетворяет вашим требованиям. Если необходимо внести изменения, можно изменить глобальную политику по умолчанию или создать новую политику ПИН-кодов. Можно создать политики ПИН-кодов, которые применяются к определенному узлу, пользователю или группе пользователей.
