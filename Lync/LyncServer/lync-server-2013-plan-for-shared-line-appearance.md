# <a name="plan-for-shared-line-appearance-in-lync-server-2013"></a>Планирование общего внешнего вида линии в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2016-03-21_

В этом разделе рассказывается, как спланировать общий внешний вид линии (SLA) в Lync Server 2013, накопительный пакет обновления от 2016 апреля.

Общий внешний вид линии — это функция в Lync Server 2013, накопительный пакет обновления за Апрель 2016 для обработки нескольких звонков по определенному номеру, называемому общим номером. SLA может настроить любой пользователь Lync для корпоративной голосовой связи как общий номер с несколькими строками для ответа на несколько вызовов. Вызовы на самом деле не приходятся на общий номер, вместо этого они пересылаются пользователям, которые действуют как делегаты для общего номера. Любой из представителей может получить вызов, пока остальные делегаты получат уведомление на своем телефоне о том, кто получил вызов, и какая строка стала занятой в результате. Число строк и делегатов можно настраивать для общего числа в SLA. Кроме того, дополнительные параметры, такие как Бусйоптион (что происходит в ситуации, когда все линии заняты) и Мисседкаллоптион (случаи, когда ни один из представителей не берет вызов), также могут быть настроены для общего номера.

Соглашение об уровне обслуживания поддерживается только на следующих телефонных устройствах (оно не поддерживается для клиентов Lync на компьютерах, мобильных телефонах и других устройствах):

  - Polycom VVX300 с обновлением встроенного по 5.4.1

  - Polycom VVX400 с обновлением встроенного по 5.4.1

  - Polycom VVX500 с обновлением встроенного по 5.4.1

  - Polycom VVX600 с обновлением встроенного по 5.4.1

SLA — это новая функция в Lync Server 2013, накопительный пакет обновления от 2016 апреля.

Сведения о развертывании SLA приведены [в статье Deploy Sharing Line Appearance in Lync Server 2013](lync-server-2013-deploy-shared-line-appearance.md).

<div>

## <a name="feature-list"></a>Список компонентов

Настройка группы SLA позволяет выполнять указанные ниже действия.

  - Все делегаты в группе могут отвечать на входящие звонки с одним и тем же общим номером. Звонки могут быть на основе PSTN или SIP.

  - Представители могут удерживать и принимать звонки.

  - Делегаты могут передавать звонки на номер, находящийся вне группы SLA.

  - Представители могут видеть количество звонков в данный момент, а также просматривать состояние каждого из этих вызовов.

  - Можно настроить максимальное количество одновременных вызовов для общего номера. Вы также можете задать способ обработки дополнительных звонков после достижения этого максимального значения. Лишние звонки могут отклоняться с помощью сигнала "занято", пересылки на другой номер или переадресация на голосовую почту.

  - Вы можете настроить способ обработки пропущенных вызовов (вызовы, не выбранные в течение определенного времени). Если для удостоверения группы включена голосовая почта, пропущенные звонки автоматически переходят на голосовую почту. Если для удостоверения группы (общего номера) не включена поддержка голосовой почты, вы можете отклонить пропущенные звонки с помощью сигнала "занято", пересылаться на другой номер или отключаться.
