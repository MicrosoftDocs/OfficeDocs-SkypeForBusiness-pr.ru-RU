# <a name="configuring-video-example-scenarios-for-lync-server-2013"></a>Настройка примеров видеороликов для Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2012-10-02_

Lync 2013 добавляет новые функции видео для поддержки видеоролика и видеоролика в формате High Definition 1920 x 1080 (HD). Измерения, основанные на данных о клиентах, показывают, что обычная пропускная способность видеосвязи увеличивается только в Lync 2010, но максимальная пропускная способность видеопотока увеличилась в связи с полной поддержкой HD (Дополнительные сведения см. в разделе "использование сети трафика мультимедиа" [требования к пропускной способности сети для трафика мультимедиа в Lync Server 2013](lync-server-2013-network-bandwidth-requirements-for-media-traffic.md)). Поэтому администраторам, возможно, потребуется ограничить полосу пропускания видео для определенных пользователей (например, для сотрудников филиалов, которым требуется меньшее потребление сети) при одновременном сохранении оптимального качества видео для других пользователей (например, для руководителей).

В следующей таблице представлен список рекомендуемых параметров для настройки видео в зависимости от требуемой полосы пропускания сети. Эти параметры позволяют ограничить возможности передачи и приема видео высокого разрешения для отдельных пользователей (см. самый правый столбец). Установка минимального значения приводит к тому, что режим «Развернутый вид» становится недоступным в связи с установкой минимальной полосы пропускания.

Информацию, представленную в предыдущей таблице, можно использовать для развертывания новых функций конференц-связи «Видео в формате HD» и «Развернутый вид» для некоторых пользователей в организации с поддержкой другого разрешения для остальных пользователей.

В следующем примере администратор разворачивает новые функции передачи видео с максимальным качеством только для руководителей. Для сотрудников удаленного филиала с низкой пропускной способностью устанавливается только минимальное значение из приведенной выше таблицы. Для всех других сотрудников устанавливается значение «Хорошее» из приведенной выше таблицы

Чтобы выполнить развертывание новых функций для руководителей, администратор создает политику конференц-связи с именем ExecutiveVideo. Эта политика конференц-связи включает следующие параметры:

  - для параметра VideoBitRateKB задано значение 8000 кбит/с;

  - для параметра TotalReceiveVideoBitRateKB задано значение 8000 кбит/с;

  - для параметра AllowMultiview задано значение True;

  - для параметра EnableMultiviewJoin задано значение True.

Для сотрудников филиала администратор создает политику конференц-связи с именем BranchOfficeVideo. Эта политика конференц-связи включает следующие параметры:

  - для параметра VideoBitRateKB задано значение 350 кбит/с;

  - для параметра TotalReceiveVideoBitRateKB задано значение 350 кбит/с;

  - для параметра AllowMultiview задано значение True;

  - для параметра EnableMultiviewJoin задано значение False.

Для всех остальных сотрудников администратор создает политику конференц-связи с именем StandardVideo. Для этой политики конференц-связи установлены следующие значения:

  - для параметра VideoBitRateKB задано значение 2500 кбит/с;

  - для параметра TotalReceiveVideoBitRateKB задано значение 2500 кбит/с;

  - для параметра AllowMultiview задано значение True;

  - для параметра EnableMultiviewJoin задано значение True.

Администратор назначает политику конференц-связи пользователям следующим образом.

  - Политика конференц-связи ExecutiveVideo назначается руководителям.

  - Политика конференц-связи BranchOfficeVideo назначается всем остальным сотрудникам филиала.

  - Политика конференц-связи StandardVideo назначается всем остальным сотрудникам.

После назначения этих политик для пользователей будут доступны следующие возможности.

  - Все конференции, организованные любым пользователем, поддерживают функцию расширенного вида, но сотрудники филиала не могут ее использовать.

  - Руководители могут передавать видео с разрешением 1920 x 1080 в формате «full HD» (при условии, что его поддерживает оборудование и сетевой канал) для любых двусторонних и многосторонних конференций, если их также поддерживают клиенты других участников.

  - Сотрудники, которые не являются руководителями, просматривают видео с более низким (но все равно хорошим) разрешением, чем руководители, при участии в двусторонних или многосторонних конференциях.

  - Сотрудники, находящиеся в филиалах, будут получать хорошее качество видео в двух вызовах, когда Lync отображает размер окна видео по умолчанию; Однако если окно Lync развернуто на весь экран, разрешение экрана не увеличится. При участии в многосторонних конференциях сотрудники филиала видят только одно активное видео.
