# <a name="new-persistent-chat-server-features-in-lync-server-2013"></a>Новые функции сервера сохраняемого чата в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Тема последнего изменения:** 2012-10-29_

Lync Server 2013, сервер сохраняемого чата позволяет принимать участие в многокомпонентных беседах, которые сохраняются с течением времени. Сервер сохраняемого чата может помочь вашей организации выполнять указанные ниже действия.

  - Улучшена связь между географически распределенными и функциональными группами

  - Расширение возможностей для информирования о информации и участия в них

  - Повышение эффективности общения с помощью расширенной Организации

  - Уменьшить перегрузку данных

  - Улучшенная поддержка информации

  - Повышайте разброса важных знаний и данных

Lync Server 2013, сервер сохраняемого чата недоступен в Microsoft Office 365. В настоящее время она доступна только для локальных пользователей Lync 2013.

В Lync 2013 функция сохраняемого чата встроена в клиент Lync 2013. В результате пользователи смогут получать доступ к голосовой и видеосвязи, а также аудио-и видеосвязи, Конференции и сохраняемый чат в клиенте Lync 2013. Дополнительные сведения о клиенте Lync 2013 можно найти в <http://go.microsoft.com/fwlink/p/?linkid=270877>разделе.

В этой статье описаны изменения функций между новой версией Lync Server 2013, сохраняемым сервером чата и предыдущей версией (Microsoft Lync Server 2010, групповой чат), в том числе:

  - Предоставление административного опыта в панели управления Lync Server и удаление средства администрирования группового чата

  - Интеграция параметров конфигурации для постоянного сервера чата с построителем топологии путем удаления средства настройки группового чата

  - Упрощение миграции и переход с предыдущих версий для сервера сохраняемого чата

  - Предоставление решений о высокой доступности и аварийном восстановлении

Дополнительные сведения о последней версии сохраняемого сервера чатов можно найти в следующих статьях:

  - Справка по сохранению чата <http://go.microsoft.com/fwlink/p/?linkid=270945> , в которой содержится подробный список функций сохраняемого чата, как они работают и как использовать их во время работы с сохраняемым сервером чата.

  - [Сервер планирования в Lync server 2013](lync-server-2013-planning-for-persistent-chat-server.md) в документации по планированию [развертывание сохраняемого сервера чата в Lync Server 2013](lync-server-2013-deploying-persistent-chat-server.md) в документации по развертыванию, [Переход с Lync Server 2010, групповой чат или Office Communications Server 2007 R2 Group Chat to Lync Server 2013, сохраняемый сервер чата](migration-from-lync-server-2010-group-chat-or-office-communications-server-2007-r2-group-chat-to-lync-server-2013-persistent-chat-server.md) в документации по миграции и [Управление Lync Server 2013, сохраняемый сервер чата](managing-lync-server-2013-persistent-chat-server.md) в документации по работе, в которой содержатся инструкции для Настройка сервера сохраняемого чата.

  - MSI-файл документации на сервере (файл установщика Windows) позволяет пользователям получить доступ к полной автономной документации на сервере сохраняемого чата.

## <a name="key-topology-changes-for-persistent-chat-server"></a>Изменения топологии ключей для сервера сохраняемого чата

Следующие высокоуровневые изменения для сервера сохраняемого чата включают:

Теперь сохраняемый сервер чата является ролью сервера. В Microsoft Lync Server 2010 для Microsoft Lync Server 2010 сервер группового чата был сторонним доверенным приложением. Вы можете добавить сохраняемый чат в топологию Lync Server 2013 с помощью построителя топологии. В Lync Server 2013 функции сервера сохраняемого чата реализованы с помощью трех новых ролей сервера:

  - **Персистентчатсервице:** Это роль пользовательского интерфейса для сохраняемого чата. В стандартных развертываниях выпуска роль сервера сохраняемого чата размещается на сервере Standard Edition, развернутом загрузчиком, как и любые другие роли сервера Lync. В развертываниях Enterprise Edition роль службы сохраняемого чата разворачивается на отдельных компьютерах с помощью загрузчика, как и любая другая роль сервера Lync.

  - **Персистентчатсторе:** Сервер, соответствующий базе данных содержимого сохраняемого чата, в которой хранится содержимое чата.

  - **Персистентчаткомплианцесторе:** Серверная роль сервера, соответствующая базе данных соответствия требованиям сохраняемого чата, в которой хранятся все события соответствия требованиям.

Эти роли сервера сохраняемые данные не являются обязательными и устанавливаются только клиентами, которые хотят вести полную функциональность сервера для работы с сохраняемым чат. Роль **персистентчаткомплианцесторе** требуется только в том случае, если вы решили развернуть постоянную подгрупповую беседу.

Роль **персистентчатсервице** выполняет две службы:

  - Служба сохраняемого чата

  - Служба соответствия требованиям сохраняемого чата

Использование этих служб на каждом сервере сохраняемого чата обеспечивает высокий уровень доступности для этих служб в многосерверном пуле сервера сохраняемого чата.

Кроме того, для поддержки отправки и загрузки файлов в сохраняемые комнаты чатов сервер сохраняемого чата включает веб-службу. Ранее эта служба была размещена на сервере сохраняемого чата, сервере переднего плана и необходимых службах IIS для установки в качестве необходимых компонентов. В Lync Server 2013 (сохраняемый сервер) веб-служба отправки и загрузки файлов размещается на сервере переднего плана Lync Server 2013. Как побочный эффект, информационные службы Интернета (IIS) больше не являются обязательным условием для сервера сохраняемого чата. Веб-служба отправки и загрузки файлов указана как **персистентчат** в ДИСПЕТЧЕРЕ служб IIS.

> [!IMPORTANT]  
> Роль <STRONG>персистентчатсервице</STRONG> может выполняться на том же сервере, что и внешний сервер Lync&nbsp;Server 2013, только если этот сервер переднего плана является стандартным&nbsp;внешним сервером выпуска. Роль <STRONG>персистентчатсервице</STRONG> не может выполняться независимо от сервера переднего плана&nbsp;Lync Server 2013. Его можно установить только в контексте развертывания Lync Server 2013.

На сервере сохраняемого чата служба поиска исключена. В Lync Server 2010, групповой чат, служба просмотра была выполнена на каждом внешнем сервере группового чата и выполнена маршрутизация на один из серверов каналов. Lync Server 2013 опирается на маршрутизацию с помощью объектов контактов, в которых каждый серверный пул в каждом из них представляет собой объект-контакт, который используется серверами переднего плана Lync Server для идентификации и маршрутизации запросов к соответствующему пулу серверов для работы с сохраняемым разговором, а также для один из компьютеров, использующих сохраняемый сервер чата в пуле.

В Lync Server 2013 существуют изменения в службе соответствия требованиям.

  - В Lync Server 2010 служба соответствия работала отдельно (не выровненная) и только на одном сервере. Служба соответствия работает на всех серверах переднего плана, работающих на сервере, с помощью службы сохраняемого чата, и таким образом обеспечивает высокий уровень доступности в многосерверном пуле сервера сохраняемого чата. Вы можете настроить один адаптер соответствия требованиям, чтобы извлечь данные из базы данных соответствия и в одну из других систем (XML-файл, архивы Exchange и т. д.). Сервер сохраняемого чата включает адаптер XML.

  - Очередь Message Queuing (также известная как MSMQ), доступная в службе сохраняемого чата, и служба соответствия на каждом внешнем серверном интерфейсе Chat, теперь представляют собой частную очередь, доступную только для двух служб. Все службы соответствия записываются в одну и ту же базу данных соответствия требованиям. Кроме того, они читаются из этой базы данных для отправки данных на их экземпляр адаптера. Сервер обратного соответствия требованиям представлен как новый серверная роль.

На сервере сохраняемого чата один из экземпляров адаптера может обработать данные.

В Lync Server 2013 есть улучшенные возможности восстановления с высокой доступностью и аварийным восстановлением.

  - Улучшения высокого уровня доступности: зеркальное отображение SQL Server используется для обеспечения высокой доступности базы данных контента сервера чата и сохраняемой базы данных соответствия чата в центре обработки данных (на сайте).

  - Улучшенные возможности аварийного восстановления: сервер сохраняемого чата поддерживает архитектуру с растяжением, позволяющую растягивать один серверный пул для разных сайтов (то есть на один логический пул в топологии, на котором физически находятся серверы пула). расположено на двух сайтах). Доставка журналов SQL Server используется для восстановления на основе нескольких сайтов.

Дополнительные сведения о высокой доступности и аварийном восстановлении содержатся в разделе [Настройка сервера сохраняемого чата для обеспечения высокой доступности и аварийного восстановления в Lync Server 2013](lync-server-2013-configuring-persistent-chat-server-for-high-availability-and-disaster-recovery.md) в документации по развертыванию.

## <a name="key-administration-and-management-changes-for-persistent-chat-server"></a>Изменение ключей администрирования и управления для сервера сохраняемого чата

Lync Server 2013 упрощает администрирование и управление сохраняемым сервером чата, предоставляя следующие возможности.

  - Единое администрирование и управление. Lync Server 2013 упрощает управление и администрирование сохраняемого сервера чатов с помощью инструментов, которые уже знакомы администраторам Lync. Сервер сохраняемого чата включает в себя интерфейс административного пользовательского интерфейса, интегрированный с панелью управления Lync Server, который устраняет проблемы с производительностью в предыдущих версиях пользовательского интерфейса сервера группового чата. Кроме того, на сервере сохраняемого чата есть набор командлетов Windows PowerShell для управления категориями серверов сохраняемого чата и управления ими (в том числе удаление комнат и очистку устаревшего содержимого) и надстроек.

  - Упрощенная модель администрирования. В Lync Server 2013 была изменена и упрощена модель сервера "сохраняемый чат", которая устраняет следующие основные требования для пользователей:
    
      - Удалите сложные вложенные иерархии областей и категорий.
    
      - В этой статье приведены сведения о том, как определять списки запрещенных и разрешенных списков (областей) для текущих клиентов Миндалигн, которые планируется перенести на сервер сохраняемого чата.

## <a name="whats-different-about-user-roles-from-previous-group-chat-server-versions"></a>Каковы отличия ролей пользователей от предыдущих версий сервера для группового чата?

Lync Server 2010, групповой чат имел роль администратора пользователей, роль администратора комнаты чата и роль администратора сервера Lync Server, которая может управлять надстройками. сохраняемый сервер чата просто обеспечивает постоянную роль администратора чата (аналогичную другим пользователям Lync). Роли серверного управления доступом (RBAC) на основе ролей сервера. Любой пользователь, являющийся участником этой роли RBAC, может управлять комнатой чата, надстройками и категориями (и, следовательно, получать доступ пользователей по этим категориям) и настройку пула серверов сохраняемого чата.

## <a name="whats-different-about-chat-room-categories-from-previous-group-chat-server-versions"></a>Чем отличаются категории комнаты чата из предыдущих версий сервера для группового чата?

Категории комнаты чата больше не могут быть вложенными, а Корневая категория больше не может быть изменена. Алловедмемберс/Дениедмемберс включает в себя ту область, которая используется в предыдущих версиях группового чата (за исключением того, что она не поддерживала указание запрещенного списка). Области больше не могут быть переопределены, так как не существует вложенных категорий. Администратор постоянного чата в Lync Server 2013 может создавать категории комнат чата и управлять ими. В рамках создания и управления категориями комнаты чата администратор сохраняемого чата может настраивать участников (групп Active Directory, контейнеров и пользователей), которые имеют доступ к участникам или создателям чатов в определенной категории. Администратор сохраняемого чата также может добавить Дениедмемберс в категорию, и это станет явным исключением для списка разрешенных. Участники DeniedMembers переопределяют участников AllowedMembers.

## <a name="whats-different-about-chat-room-properties-from-previous-group-chat-server-versions"></a>Чем отличаются свойства комнаты чата из предыдущих версий сервера для группового чата?

Новая концепция открытых комнат чата в Lync Server 2013, сервере сохраняемого чата. Все разрешенные пользователи могут присоединиться к комнате чата, не имея исключительных членств.

Исключены следующие свойства комнаты чата, включенные в предыдущие версии сервера сохраняемого чата.

  - Тема: Теперь у комнаты есть только описание.

  - Создание нового списка участников: на сервере сохраняемого чата все комнаты чата начинаются с пустого членства (и могут увеличиваться до группы, равной разрешенным пользователям).

  - Отправка файла: используется для помещения в чат, чтобы указать, разрешена ли загрузка и скачивание файлов. Теперь вы задаете только уровень категории и распространяется на все помещения в этой категории.

  - История чатов: используется для настройки для каждой комнаты чата, чтобы управлять включением ведения журнала чата, но теперь оно установлено только на уровне категории и распространяется на все помещения в этой категории.

  - Приглашения: комната всегда наследует параметр "приглашения" для этой категории; Кроме того, она может быть отключена в комнате. Комната не может включить приглашения, если категория была ранее настроена на "мои приглашения".

</div>

<div>

## <a name="whats-different-about-policies-from-previous-group-chat-server-versions"></a>Чем отличаются политики из предыдущих версий сервера группового чата?

На сервере сохраняемого чата включена новая политика Lync с сохраняемым разговором на уровне "на пользователя", в группе/на сайте или в глобальной конфигурации. В клиенте Lync 2013 сохраняемая среда чата доступна только для пользователей, которые поддерживаются политикой для сохраняемого чата (напрямую или с помощью параметра пул/сайт/глобально).

В предыдущих версиях сервера группового чата отсутствовали политики, встроенные в политики сервера Lync Server. Для каждого пользователя, а также по категориям или комнатам с помощью функции " **загрузить файлы** на пользователя" можно сделать пользователя администратором, администратором комнаты чата или настроить возможность отправки файлов. Функция **отправки файлов** на сервере сохраняемого разговора — это только Категория.
