# <a name="creating-or-modifying-a-location-policy-in-lync-server-2013"></a>Создание и изменение политики расположения в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Тема последнего изменения:** 2012-11-01_

В Lync Server 2013 вы можете использовать политику расположения для применения параметров, которые относятся к расширенным функциям 9-1-1 (E9-1-1), а также к параметрам расположения для пользователей и контактов. Политика расположения определяет, разрешено ли пользователю E9-1-1, и в каких случаях может происходить вызов экстренной помощи. Например, вы можете использовать политику расположения, чтобы определить, какой номер является вызовом экстренной помощи (например, 911 в США), следует ли автоматически уведомлять корпоративную безопасность, а также как перенаправлять этот звонок.

Политики местоположений можно настроить в группе **Конфигурация сети** в Lync Server 2013 панели управления. На панели управления Lync Server вы можете просматривать, создавать, изменять и удалять политики расположения. Для создания и изменения политики расположения используйте процедуры, описанные в этом разделе. Подробнее об удалении политик местоположений можно узнать [в разделе Удаление политики расположения в Lync Server 2013](lync-server-2013-deleting-a-location-policy.md).

В Lync Server 2013 вы можете переопределить интервал времени по умолчанию между запросами клиентов на обновление расположения в службе сведений о расположении. Значение по умолчанию — 4 часа. Используйте командлет **Set-кслокатионполици** с параметром локатионрефрешинтервал для переопределения значения по умолчанию.

## <a name="to-create-a-new-location-policy-in-lync-server-control-panel"></a>Создание политики местоположений на панели управления Lync Server

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Lync Server. Дополнительные сведения о различных способах, которые можно использовать для запуска панели управления Lync Server, приведены в разделе [Открытие меню администрирования Lync server 2013](lync-server-2013-open-lync-server-administrative-tools.md).

3.  На панели навигации слева выберите пункт **Настройка сети** , а затем — **Политика расположения**.

4.  На странице " **Политика расположения** " нажмите кнопку **создать** , а затем выберите тип политики, которую вы хотите создать.
    
      - Чтобы создать политику сайта, нажмите кнопку **Политика сайта**. В списке **выберите сайт**выберите сайт, к которому требуется применить политику, и нажмите кнопку **ОК**. На странице " **Создание политики расположения** " в поле " **область** " отображается значение " **сайт**", а в поле " **имя** " — имя выбранного сайта. Ни одно из этих полей изменить нельзя. Политика сайта автоматически применяется ко всем пользователям на указанном сайте и переопределяет глобальную политику для этих пользователей.
    
      - Чтобы создать **политику пользователя**, нажмите кнопку **Политика пользователя**. В **новой политике расположения**поле « **область** » будет содержать значение **User**. Изменить это значение нельзя. В поле **Name (имя** ) введите имя, которое вы хотите назначить этой политике. Политика пользователя не применяется автоматически к каким-либо пользователям. После создания политики пользователя вы должны вручную предоставить политику для пользователей или сайтов сети, к которым вы хотите применить политику.

5.  Заполните остальные поля, как описано ниже.
    
      - **Включение расширенных служб**   экстренной помощи. Установите этот флажок, чтобы включить пользователей, связанных с этой политикой, для E9-1-1. При включении служб экстренной помощи Пользователи Lync Server будут получать сведения о расположении при регистрации и включать эти данные при совершении экстренного звонка.
    
      - **Расположение**   . Укажите одно из следующих значений:
        
          - **Требуется**   . пользователю будет предложено ввести сведения о расположении, когда клиент регистрируется в новом расположении. Пользователь может закрыть приглашение без ввода каких бы то ни было данных. Если введены данные, поставщик услуг может сначала ответить на вызов экстренной помощи, чтобы проверить расположение перед тем, как перенаправляться в оператор ПСАП (оператор 911).
        
          - **Не обязательно**   , пользователю не будет предложено указать расположение. Если при выполнении звонка не указана информация о местонахождении, поставщик услуг службы экстренной помощи будет отвечать на звонок и запрашивать расположение.
        
          - **Заявление**   об отказе. Этот параметр не является **обязательным** , кроме случаев, когда пользователь не может закрыть приглашение без ввода сведений о расположении. Пользователь по-прежнему может выполнить вызов экстренной помощи, но никакие другие звонки нельзя будет завершить, не вводя нужных данных. Кроме того, текст заявления об отказе будет отображаться для пользователя, который может предупредить его последствия для ввода сведений о местоположении. Чтобы настроить текст заявления об отказе, необходимо использовать командную консоль Lync Server Management Shell для запуска командлета **Set-кслокатионполици** или командлета **New-кслокатионполици** с параметром енханцедемерженцисервицедисклаимер. Подробные сведения можно найти в разделе [Set-кслокатионполици](https://docs.microsoft.com/powershell/module/skype/Set-CsLocationPolicy) или [New-кслокатионполици](https://docs.microsoft.com/powershell/module/skype/New-CsLocationPolicy) в документации по среде управления Lync Server.
            
            <div>
            

            > [!NOTE]  
            > В Lync Server 2013 вы можете использовать политику расположения для установки различных отказов на использование разных национальных настроек и разных групп пользователей, в отличие от Lync Server 2010, где вы можете указать только глобальный отказ для всей Организации.

            
            </div>
    
      - **Использовать расположение для экстренных служб**   . Lync может использовать сведения о расположении по различным причинам (например, для уведомления коллег от текущего расположения). Установите этот флажок, чтобы убедиться, что сведения о расположении доступны только для вызова экстренной помощи.
    
      - **PSTN**-использование коммутируемой телефонной сети (PSTN), которая будет использоваться для определения направления голосовых вызовов с клиентов, использующих этот профиль.    Маршрут, связанный с этим использованием, должен указывать на магистраль SIP, выделенный для вызова экстренной помощи, или на шлюз с идентификационным номером для экстренной помощи (Елин), который маршрутизирует экстренные вызовы в ближайшую точку ответа на общедоступную безопасность (ПСАП).
    
      - **Номер для экстренного звонка**   номер, набранный службой экстренных услуг. В США это значение 911. Строка должна состоять из цифр от 0 до 9 и может содержать от 1 до 10 цифр.
    
      - **Маска набора номера**   , который вы хотите переводить на значение номера экстренного набора номера при наборе номера. Например, если ввести значение 212 в это поле, а в поле «номер экстренного набора» — значение 911, то при попытке пользователя выполнить вызов 212 будет произведена 911. Это позволяет использовать дополнительные номера для экстренного реагирования (например, если кто-то из страны или региона с другим номером для экстренного реагирования пытается набрать номер страны или региона, а не номер для страна или регион, в которых они находятся в настоящее время). Вы можете определить несколько масок для экстренного набора номера, разделяя их точками с запятой. Например, 212; 414. Максимальная длина строки — 100 символов. Каждый символ должен быть цифрой от 0 до 9.
        
        <div>
        

        > [!IMPORTANT]  
        > Убедитесь, что указанное значение маски звонка не совпадает с числом в диапазоне на расстоянии по орбите. Маршрутизация на припаркованных звонках имеет приоритет над преобразованием строк в экстренном наборе. Чтобы просмотреть диапазон по орбите, нажмите кнопку <STRONG>функции голоса</STRONG> на левой панели навигации, а затем выберите команду <STRONG>приостановить Звонок</STRONG>. Подробности можно найти <A href="lync-server-2013-configure-phone-number-extensions-for-parking-calls.md">в разделе Настройка расширений номеров телефонов для вызовов парковки в Lync Server 2013</A>.

        
        </div>
    
      - **URL**   -адрес уведомлений один или несколько универсальных кодов ресурсов (URI) SIP, которые будут уведомлены при совершении вызова экстренной помощи. Например, Office для защиты компании может получать уведомления о мгновенных сообщениях во время вызова экстренной помощи. Если расположение вызывающего абонента доступно, оно будет включено в уведомление. В виде списка с разделителями-запятыми можно указывать несколько URI SIP. Например, "SIP: security@litwareinc.com", "SIP: kmyer@litwareinc.com". Поддерживаются списки рассылки. Строка должна содержать от 1 до 256 знаков длиной и начинаться с префикса SIP:. Пример отображения в поле URI уведомлений.
    
      - **Универсальный код ресурса (URI**   ) для Конференции URI-адрес SIP, в данном случае номер телефона третьей стороны, который будет использоваться в любых совершенных звонках в экстренных случаях. Например, Office Security Company может получать звонок при совершении вызова экстренной помощи и принимать участие в этом звонке (в зависимости от значения, заданного в поле **режим конференции** ). Строка должна содержать от 1 до 256 символов и должна начинаться с префикса SIP:. Пример отображается, пока вы не щелкните внутри этого поля.
    
      - **Режим**   конференции если указать значение в поле " **URI конференции** ", **режим конференции** определяет, может ли третья сторона участвовать в звонке или может принимать только прослушивание. Укажите один из следующих параметров:
        
          - **Односторонним**   третьим лицом может быть только прослушивать беседу между вызывающим и оператором ПСАП.
        
          - **Двухстороннее**третье сторона может прослушивать и принимать участие в вызове между вызывающим абонентом и оператором ПСАП.   

6.  Нажмите **Исполнить**.

## <a name="to-modify-a-location-policy-in-lync-server-control-panel"></a>Изменение политики расположения на панели управления Lync Server

1.  Войдите на любой компьютер, находящийся во внутреннем развертывании, с использованием учетной записи, входящей в группу RTCUniversalServerAdmins (или имеющей равнозначные права пользователя) либо назначенной роли CsAdministrator.

2.  Откройте окно браузера и введите URL-адрес администратора, чтобы открыть панель управления Lync Server. Дополнительные сведения о различных способах, которые можно использовать для запуска панели управления Lync Server, приведены в разделе [Открытие меню администрирования Lync server 2013](lync-server-2013-open-lync-server-administrative-tools.md).

3.  На панели навигации слева выберите пункт **Настройка сети** , а затем — **Политика расположения**.

4.  На странице " **Политика расположения** " выберите политику расположения, которую вы хотите изменить.

5.  В меню **Правка** щелкните **Подробнее**.

6.  На странице " **изменение расположения** " внесите необходимые изменения в поля (Дополнительные сведения можно найти в разделе действие 5 в описанных выше процедурах "Создание новой политики местоположений").

7.  Нажмите **Исполнить**.
