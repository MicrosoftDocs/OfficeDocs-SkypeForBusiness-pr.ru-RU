# Новые функции приложения группы ответа в Lync Server 2013

 

_**Дата изменения раздела:** 2012-10-29_

Приложение «Группа ответа» позволяет направлять входящие вызовы определенным пользователям в особых целях или включать их в очередь; в список таких целей входит обслуживание клиентов, обращение во внутреннюю службу поддержки или службу общей телефонной поддержки подразделения.

Следующие функции приложения «Группа ответа» являются новыми в версии Lync Server 2013:

  - **Роль менеджера**
    
    В версию Lync Server 2013 включена новая роль менеджера ответа. Теперь для групп ответа доступны две роли управления: менеджер ответа и администратор ответа. Администраторы ответа могут по-прежнему настраивать любые элементы для любой группы ответа, а менеджеры могут настраивать только определенные элементы и только для принадлежащих им групп ответа.
    
    Это улучшение модели администрирования повышает масштабируемость приложения «Группа ответа», особенно если речь идет о крупных сценариях развертывания.

  - **Высокая доступность**
    
    Поддержка высокой доступности для "Группа ответа" (в форме зеркального отображения SQL Server) предоставляется в рамках общей конфигурации и развертывания высокой доступности для Lync Server 2013. Если вы настроили поддержку высокой доступности, но при этом подключение к основному внутреннему серверу разрывается, на функционирование группы ответа не влияет использование зеркально отраженного внутреннего сервера.
    
    Вы не можете отдельно активировать или настраивать зеркальное отображение SQL Server для "Группа ответа" вне общей конфигурации высокой доступности Lync Server 2013.

  - **Аварийное восстановление**
    
    Поддержка аварийного восстановления для "Группа ответа" предоставляется в рамках конфигурации и развертывания парных пулов переднего плана, которые являются частью общей конфигурации аварийного восстановления Lync Server 2013. Кроме того, командлеты импорта и экспорта группы ответа поддерживают процесс аварийного переключения на резервный пул и восстановления размещения в основном пуле или новом пуле. В случае сбоя в основном пуле группы ответа могут быть переключены на резервный пул, а затем возвращены в основной пул или новый пул после ликвидации сбоя.
