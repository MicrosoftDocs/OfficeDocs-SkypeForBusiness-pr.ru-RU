# <a name="multiple-trunk-support-in-lync-server-2013"></a>Поддержка нескольких магистральных каналов в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2012-11-01_

Lync Server 2013 функциональность поддерживает множественные связи между шлюзами и серверами-посредниками. Эти связи создаются путем определения магистрали, которая является логической связью между пулом серверов-посредника и шлюзом телефонной сети общего пользования (PSTN), пограничным контроллером сеансов (SBC) или IP-УАТС. С помощью построителя топологий свяжите шлюзы с серверами-посредниками (то есть магистральными каналами).

  - Чтобы назначить или удалить магистраль в Lync Server 2013, необходимо сначала определить магистраль в построителе топологий. Магистраль состоит из следующей связи: полное доменное имя сервера-посредника, прослушивающий порт сервера-посредника, полное доменное имя шлюза и прослушивающий порт шлюза.

  - Чтобы настроить несколько магистральных каналов, можно создать несколько связей между одним и тем же шлюзом и сервером-посредником. Это обеспечивает дополнительную устойчивость инфраструктуры корпоративной голосовой связи, которая особенно полезна в сценариях взаимодействия с частными филиалами (УАТС).

После определения магистрали ее необходимо связать с маршрутом. Чтобы связать магистраль с маршрутом, определите простое имя для магистрали в построителе топологий. Это простое имя используется в качестве имени магистрали в панели управления Lync Server, где магистральные линии связи могут быть связаны с маршрутами. Простое имя магистрали используется в качестве имени шлюза в командной консоли Lync Server.

    New-CsVoiceRoute -Identity <RouteId> -NumberPattern <String> -PstnUsages @{add="<UsageString>"} -PstnGatewayList @{add="<TrunkSimpleName>"}

Администратор должен выбрать магистраль по умолчанию, связанный с сервером-посредником. В построителе топологий щелкните правой кнопкой мыши связанный сервер-посредник и выберите пункт **Свойства**. Укажите шлюз по умолчанию для сервера-посредника.

На следующей схеме показаны несколько магистральных линий, определенных для каждого сервера-посредника и шлюза.
