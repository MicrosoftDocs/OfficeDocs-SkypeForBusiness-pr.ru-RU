# <a name="installing-optional-software-in-lync-server-2013"></a>Установка дополнительного программного обеспечения в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2013-02-21_

Microsoft Lync Server 2013, средство планирования предназначено для экспорта в Microsoft Excel и Microsoft Visio. Несмотря на то, что эти приложения не требуются для работы средства планирования, они имеют существенные значения для развертывания и документирования проекта.

<div>

## <a name="optional-software"></a>Дополнительное программное обеспечение

<div>

## <a name="microsoft-excel"></a>Microsoft Excel

При экспорте проекта в Microsoft Excel создается отчет, отображающий семь вкладок в электронной таблице:

  - Сводка — отображает сведения о конфигурации сайта, включая количество пользователей, параметры емкости и сведения о профиле сервера.

  - Profile (профиль оборудования) — отображает отчет о рекомендуемых конфигурациях оборудования для серверов, указанных в топологии, в том числе ЦП, памяти, диска и сетевого интерфейса. Также включается количество и Рекомендуемые характеристики серверных компонентов. Кроме того, каждый сервер определяется сайтом для предоставления полного представления серверных требований по сайту.

  - Требования к портам — отображает отчет обо всех включенных портах и о сопоставлении с балансировкой нагрузки системы доменных имен (DNS) и аппаратной балансировкой нагрузки (HLB). Этот отчет следует использовать для планирования конфигурации брандмауэра и балансировки нагрузки DNS и HLB.

  - Сводный отчет — отображает общую сводку по параметрам, необходимым для настройки сети пограничного сервера.

  - Отчет о сертификатах — отображает имя субъекта и альтернативные имена субъекта, необходимые для сертификатов, необходимых для получения запущенных пограничных серверов.

  - Отчет о брандмауэре — отображает исходные и конечные порты и IP-адреса для внешних и внутренних интерфейсов.

  - DNS Report — отображает полное доменное имя (FQDN) и IP/IP-адрес, необходимые для каждой создаваемой записи DNS.

</div>

<div>

## <a name="microsoft-visio"></a>Microsoft Visio

При экспорте проекта в Microsoft Visio создается схема, используемая в целях вашей документации в настроенной топологии и инфраструктуре. Импортированная схема может быть изменена и упорядочена в соответствии с потребностями в документации. Типичная схема Visio будет включать:

<div>


> [!NOTE]  
> Если ваша структура достаточно велика, чтобы потребовать более трех серверов переднего плана, будет создана дополнительная страница для пула переднего плана, серверов переднего плана, компьютера с SQL Server, IP-адресов и полных доменных имен.



</div>

  - Глобальная топология — схема настроенных сайтов Lync Server 2013.

  - Вкладка "имя сайта" — отображает топологию конфигурации сайта с пограничным сервером, брандмауэром, телефонной сетью общего пользования (PSTN) с шлюзами и внутренним развертыванием сервера. Внутреннее развертывание состоит из настроенных серверов и пулов, в том числе интерфейсных пулов, серверов SQL Server, доменных служб Active Directory, директоров, серверов единой системы обмена сообщениями Exchange, серверов почтовых ящиков Exchange, серверов почтовых ящиков Exchange, серверов Office Web Apps Серверы-посредники и серверы сохраняемого чата.

  - Схема пограничной сети — схема детализации конфигурации пограничного сервера со связанными IP-адресами и полными доменными именами. Также включается балансировка нагрузки на DNS и аппаратные подсистемы балансировки нагрузки. Кроме того, отображаются режиссеры и сервер переднего плана или интерфейсный пул с соответствующей службой DNS фунтов или HLB и назначенным IP-адресом (средство планирования поддерживает как IPv4-, так и IPv6-адреса) и полное доменное имя.
