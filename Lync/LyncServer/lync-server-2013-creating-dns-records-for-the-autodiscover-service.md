# Создание DNS-записей для службы автообнаружения в Lync Server 2013

 

_**Дата изменения раздела:** 2014-06-20_

Для пользователей Lync Mobile необходимо включить функцию автоматического обнаружения. Часть этого включает в себя создание нескольких DNS-записей. В зависимости от потребностей можно выполнить следующие действия:

  - внутренняя DNS-запись для поддержки мобильных пользователей, которые подключаются изнутри сети организации;

  - внешняя или общедоступная запись DNS для поддержки мобильных пользователей, которые подключаются из Интернета.

Почему оба пункта? Необходимо создать внутреннюю и внешнюю DNS-запись для каждого SIP-домена.

Эти DNS-записи могут быть записями A (узла) или записями CNAME. Помощь по созданию этих внутренних и внешних DNS-записей представлена ниже. Дополнительные сведения о требованиях DNS для мобильных пользователей см. в разделе [Технические требования для организации мобильной работы в Lync Server 2013](lync-server-2013-technical-requirements-for-mobility.md).

## Создание внутренней DNS-записи CNAME

1.  Чтобы создать внутреннюю DNS-запись, выполните вход на DNS-сервер вашей сети с помощью доменной учетной записи как член группы администраторов домена или член группы DnsAdmins.

2.  Откройте административную оснастку DNS: нажмите последовательно **Пуск** , **Инструменты администратора** и **DNS** .

3.  В дереве консоли DNS-сервера разверните **Зоны прямого просмотра** для домена Active Directory, в котором установлены Lync Server 2013Директоров и переднего плана (например, contoso.local).

4.  Убедитесь в наличии записи A (AAAA для IPv6) узла Директоров для внутренней DNS-записи. Запись A узла должна существовать для полного доменного имени внутренних веб-служб для Директоров (например, lyncwebdir01.contoso.local). Если она отсутствует, возможно вы используете не Директоров, также может потребоваться использование полного доменного имени для переднего плана или один сервер с FQDN в соответствии с настройкой.

5.  Помня об этом, убедитесь в наличии записи A (AAAA для IPv6) узла переднего плана для внутренней DNS-записи. Запись A (или AAAA) узла должна существовать для полного доменного имени внутренних веб-служб для переднего плана (например, lyncwebpool01.contoso.local). Если она отсутствует, необходимо записать полное доменное имя для переднего плана или Сервер Standard Edition.

6.  После получения сведений о типе записи A (или AAAA) в дереве консоли DNS-сервера разверните **Зоны прямого просмотра** для SIP-домена (например, contoso.com).

7.  Щелкните правой кнопкой название SIP-домена, затем **Новый псевдоним (CNAME)** .

8.  В поле **Имя псевдонима** введите lyncdiscoverinternal в качестве имени узла для URL-адреса внутренней службы автообнаружения.

9.  В поле **Полное доменное имя для целевого узла** введите или перейдите к полному доменному имени внутренних веб-служб для Директоров (например, lyncwebdir01.contoso.local) и нажмите **ОК**.

10. Необходимо создать новую запись CNAME автообнаружения в зоне прямого просмотра каждого SIP-домена, поддерживаемого в вашем окружении Lync Server 2013.

## Создание внешней DNS-записи CNAME

1.  Чтобы создать внешнюю DNS-запись CNAME необходимо подключиться к общедоступному поставщику DNS и следуйте нашим указаниям. Мы не можем точно описать действия в среде поставщика DNS, так как есть разные способы управления внешним DNS, но надеемся, что указанные далее действия помогут.

2.  Войдите в систему внешнего поставщика DNS с помощью учетной запись с возможностью создания в этой среде DNS-записей.

3.  Для этой среды уже должен быть создан SIP-домен. Разверните **Зоны прямого просмотра** для этого SIP-домена или выберите в соответствии с используемым интерфейсом внешнего DNS.

4.  Запись A (AAAA для IPv6) узла уже должна быть видна для Директоров (например, lyncwebexdir01.contoso.com), подтвердите ее наличие. Если нет, возможно вы используете не Директоров. В данном случае потребуется использовать полное доменное имя пула переднего плана или при условии использования этого для одного сервера, сервера переднего плана или Standard Edition.

5.  Также необходимо подтвердить наличие записи A (AAAA для IPv6) узла для полного доменного имени внешних веб-служб пула переднего плана (например, lyncwebextpool01.contoso.com) или полного доменного имени для одного сервера при отсутствии пула переднего плана. Как указано в предыдущем шаге, это может потребоваться далее при отсутствии пула директоров.

6.  Теперь с использованием соответствующего формата для поставщика внешнего DNS выберите параметр для создания **нового псевдонима (CNAME)** (это может быть параметр меню или ссылка в зависимости от формата поставщика DNS).

7.  Должно существовать текстовое поле типа **Имя псевдонима**, так как для внутреннего DNS необходимо указать lyncdiscover в качестве имени узла для внешнего URL-адреса веб-служб системы.

8.  Также должна быть какая-либо форма текстового поля **Полное доменное имя для целевого узла**, так как в него необходимо ввести полное доменное имя внешних веб-служб для Директоров (например, lyncwebexdir01.contoso.com), а затем нажать «ОК» или предпринять соответствующие действия для внешнего DNS в рамках принятия создания записи. Как указано в шаге 4 выше, при отсутствии Директоров может потребоваться использование полного доменного имени пула переднего плана или полного доменного имени одного сервера с соответствующей настройкой.

9.  Необходимо создать новую запись CNAME автообнаружения в зоне прямого просмотра каждого SIP-домена, поддерживаемого в среде Lync 2013.

## Создание записи A внутреннего DNS

1.  Чтобы создать внутреннюю DNS-запись, выполните вход на DNS-сервер вашей сети с помощью доменной учетной записи как член группы администраторов домена или член группы DnsAdmins.

2.  Откройте административную оснастку DNS: нажмите последовательно **Пуск** , **Инструменты администратора** и **DNS** .

3.  Для внутренней DNS-записи в дереве консоли DNS-сервера разверните **Зоны прямого просмотра** для домена Active Directory (например, contoso.local), на котором установлен пул Lync Server 2013Директор и переднего плана.

4.  Убедитесь в наличии записи A (AAAA для IPv6) узла Директоров внутренней DNS-записи. Запись A узла должна существовать для полного доменного имени внутренних веб-служб для Директоров (например, lyncwebdir01.contoso.local). Если она отсутствует, возможно вы используете не Директоров, также может потребоваться использование IP-адреса для пула переднего плана или IP-адреса одного сервера в соответствии с настройкой.

5.  Помня об этом, убедитесь в наличии записи A (AAAA для IPv6) узла переднего плана для внутренней DNS-записи. Запись A (или AAAA) узла должна существовать для полного доменного имени внутренних веб-служб для переднего плана (например, lyncwebpool01.contoso.local). Если она отсутствует, необходимо записать полное доменное имя для переднего плана или Сервер Standard Edition.

6.  После получения сведений о типе записи A (или AAAA) в дереве консоли DNS-сервера разверните **Зоны прямого просмотра** для SIP-домена (например, contoso.com).

7.  Щелкните правой кнопкой имя SIP-домена и нажмите **Новый узел (A или AAAA)** .

8.  В поле **Имя** введите lyncdiscoverinternal в качестве имени узла для URL-адреса внутренней службы автообнаружения.

9.  В поле **IP-адрес** введите IP-адрес внутренних веб-служб Директор (или при использовании балансировщиков нагрузки введите виртуальный IP-адрес (VIP) балансировщика нагрузки Директор). Как указано в шаге 4 выше, если Директор не используется, может потребоваться ввод IP-адреса переднего плана или Сервер Standard Edition, а при использовании балансировщика нагрузки введите VIP балансировщика нагрузки переднего плана.

10. Щелкните **Добавить узел** , а затем нажмите кнопку **ОК** .

11. Чтобы создать дополнительную запись A, повторите шаги с 8 по 10, помня о необходимости создания новых записей A или AAAA автоматического обнаружения в зоне прямого просмотра каждого SIP-домена, поддерживаемого в среде Lync Server 2013.

12. После завершения создания записей A (для IPv6, AAAA) нажмите кнопку **Готово** .

## Создание записи A внешнего DNS

1.  Чтобы создать внешнюю DNS-запись, подключитесь к внешнему поставщику DNS. Мы не можем точно описать действия в среде поставщика DNS, так как есть разные способы управления внешним DNS, но надеемся, что указанные далее действия помогут.

2.  Выполните вход в систему с помощью учетной записи с возможностью создания записей DNS в этой среде.

3.  Для внешней DNS-записи в дереве консоли DNS-сервера разверните **Зоны прямого просмотра** для SIP-домена (например, contoso.com). Для внешней DNS-записи в дереве консоли DNS-сервера разверните элемент **Зоны прямого просмотра** для вашего SIP-домена (например, contoso.com).

4.  Запись A (AAAA для IPv6) узла уже должна быть видна для Директоров (например, lyncwebexdir01.contoso.com), подтвердите ее наличие и IP-адрес. Если нет, возможно вы используете не Директоров. В данном случае потребуется использовать полное доменное имя пула переднего плана или при условии использования этого для одного сервера, сервера переднего плана или Standard Edition. Помните, что серверы могут также находиться за пределами балансировщика нагрузки или использовать обратный прокси-сервер. Запишите IP-адреса для выполнения указанных далее действий.

5.  Также необходимо подтвердить наличие записи A (AAAA для IPv6) узла для полного доменного имени внешних веб-служб пула переднего плана (например, lyncwebextpool01.contoso.com) или IP-адреса для установки одного сервера Lync или при отсутствии пула переднего плана. Как указано в предыдущем шаге, это может потребоваться далее при отсутствии пула директоров.

6.  Теперь с использованием соответствующего формата для поставщика внешнего DNS выберите параметр для создания **нового узла A или AAAA** (это может быть параметр меню или ссылка в зависимости от формата поставщика DNS).

7.  Должно присутствовать поле для **имени**, введите в него lyncdiscover в качестве имени узла для внешнего URL-адреса -службы автоматического обнаружения.

8.  Также должно присутствовать текстовое поле **IP-адрес** для ввода IP-адреса Директоров (например, lyncwebexdir01.contoso.com) или возможного IP-адреса балансировщика нагрузки пула (или IP-адреса обратного прокси-сервера, который указывает на это же), затем нажмите «ОК» или предпримите соответствующие действия для внешнего DNS в рамках принятия создания записи. Как указано в шаге 4 выше, при отсутствии Директоров может потребоваться использование IP-адреса пула переднего плана или IP-адреса одного сервера с соответствующей настройкой.

9.  Необходимо создать новую запись A узла автообнаружения в зоне прямого просмотра для каждого SIP-домена, который поддерживается в среде Lync 2013.
