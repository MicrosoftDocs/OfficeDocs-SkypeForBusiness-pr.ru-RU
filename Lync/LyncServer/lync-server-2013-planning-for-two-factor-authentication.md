# <a name="planning-for-two-factor-authentication-in-lync-server-2013"></a>Планирование двухфакторной проверки подлинности в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2015-04-06_

Ниже приведен список рекомендаций по развертыванию при настройке среды Microsoft Lync Server 2013 для поддержки двухфакторной проверки подлинности.

<div>

## <a name="client-support"></a>Поддержка клиентов

Накопительные пакеты обновления для Lync 2013 для Lync Server 2013: Июль 2013 клиент для настольных ПК и все мобильные клиенты в настоящее время поддерживают двухфакторную проверку подлинности.

</div>

<div>

## <a name="topology-requirements"></a>Требования к топологии

## <a name="lync-service-discovery"></a>Обнаружение службы Lync

Записи DNS, используемые внутренними и/или внешними клиентами для обнаружения служб Lync, должны быть настроены для разрешения на сервер Lync, для которого не включена двухфакторная проверка подлинности. В этой конфигурации пользователи из пулов Lync, для которых не включена двухфакторная проверка подлинности, не должны вводить ПИН-код для проверки подлинности, а пользователи из пулов Lync, для которых включена двухфакторная проверка подлинности, должны ввести ПИН-код для осуществлять.

</div>

<div>

## <a name="exchange-authentication"></a>Проверка подлинности Exchange

Клиенты, которые развернули двухфакторную проверку подлинности для Microsoft Exchange, могут обнаружить, что некоторые функции в клиенте Lync недоступны. В настоящее время он разрабатывается, так как клиент Lync не поддерживает двухфакторную проверку подлинности для функций, зависящих от интеграции Exchange.

</div>

<div>

## <a name="lync-contacts"></a>Контакты Lync

Пользователи Lync, которые настроены для использования единой функции хранилища контактов, смогут обнаружить, что их контакты больше не будут доступны после входа с использованием двухфакторной проверки подлинности.

Следует использовать командлет **Invoke – CsUcsRollback** для удаления существующих контактов пользователей из единого хранилища контактов и их хранения в Lync Server 2013 перед включением двухфакторной проверки подлинности.

</div>

<div>

## <a name="skill-search"></a>Поиск по навыкам

Пользователи, которые настроили функцию поиска по навыкам в среде Lync, смогут обнаружить, что эта функция не работает, если для Lync включена двухфакторная проверка подлинности. Это сделано намеренно, так как Microsoft SharePoint в настоящее время не поддерживает двухфакторную проверку подлинности.

</div>

<div>

## <a name="lync-credentials"></a>Учетные данные Lync

Существует ряд вопросов, связанных с развертыванием учетных данных Lync, которые могут повлиять на пользователей, использующих двухфакторную проверку подлинности.

<div>

## <a name="deleting-saved-credentials"></a>Удаление сохраненных учетных данных

Пользователи настольного клиента должны использовать параметр " **удалить сведения о входе в систему** " в клиенте Lync и удалять свою папку профиля SIP из\\%\\локалаппдата\\%\\Microsoft Office 15,0 Lync, прежде чем пытаться войти в первый раз с использованием двухфакторной проверки подлинности.

</div>

<div>

## <a name="disablentcredentials"></a>DisableNTCredentials

При использовании метода проверки подлинности Kerberos или NTLM учетные данные пользователя Windows используются автоматически для проверки подлинности. В типичном развертывании Lync Server 2013, где для проверки подлинности включена проверка подлинности Kerberos и/или NTLM, пользователям не придется вводить свои учетные данные при каждом входе в систему.

Если пользователям непреднамеренно запрашиваются учетные данные, прежде чем им будет предложено ввести ПИН-код, раздел реестра **DisableNTCredentials** может быть непреднамеренно настроен на клиентских компьютерах, возможно, с помощью групповой политики.

Чтобы не допустить дополнительных запросов учетных данных, создайте следующую запись реестра на локальной рабочей станции или используйте административный шаблон Lync для применения ко всем пользователям для определенного пула с помощью групповой политики:

\_Локальные политики\_\\программного\\обеспечения для\\локального\\компьютера\\\\, Microsoft Office 15,0 Lync

REG\_DWORD: DisableNTCredentials

Значение: 0x0

</div>

<div>

## <a name="savepassword"></a>Сохранение пароля

Когда пользователь первый раз выполняет вход в Lync, пользователю предлагается сохранить пароль. Если этот параметр выбран, этот параметр позволяет хранить сертификат клиента пользователя в хранилище личных сертификатов, а учетные данные пользователя Windows для хранения в диспетчере учетных данных локального компьютера.

Параметр реестра **SavePassword** должен быть отключен при настройке Lync для поддержки двухфакторной проверки подлинности. Чтобы запретить пользователям сохранять свои пароли, измените следующую запись реестра на локальной рабочей станции или используйте административный шаблон Lync для применения ко всем пользователям для определенного пула с помощью групповой политики:

\_Программное\_обеспечение\\\\для текущего\\пользователя\\hKey\\Microsoft Office 15,0 Lync

REG\_DWORD: SavePassword

Значение: 0x0

</div>

</div>

## <a name="ad-fs-20-token-replay"></a>Повторный запуск маркеров AD FS 2,0

AD FS 2,0 предоставляет функцию обнаружения повторяющихся маркеров, с помощью которой можно обнаружить несколько запросов маркеров, использующих один и тот же маркер, а затем удалить их. Если эта функция включена, обнаружение повторяющихся маркеров защищает целостность запросов проверки подлинности в пассивном профиле WS-Federation и в профиле SAML Вебссо, убедившись, что один и тот же маркер никогда не используется более одного раза.

Эту функцию следует включать в тех случаях, когда безопасность очень важна, например, при использовании киосков. Дополнительные сведения об обнаружении повторяющихся маркеров приведены в разделе рекомендации по обеспечению безопасного планирования и развертыванию служб федерации Active [http://go.microsoft.com/fwlink/p/?LinkId=309215](http://go.microsoft.com/fwlink/p/?linkid=309215)Directory 2,0 по адресу.

</div>
