# <a name="frequently-asked-questions-provisioning-lync-server-2013-for-skype-connectivity"></a>Часто задаваемые вопросы: подготовка сервера Lync Server 2013 для подключения Skype

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Тема последнего изменения:** 2019-03-22_

Начиная с апреля 2019, мы прекращаем сбор и хранение контактных данных для клиентов, которые настроены для Федерации Skype с помощью веб-сайта pic.lync.com. Это изменение делается для того, чтобы убедиться в том, что система подготовки pic.lync.com придерживается политик конфиденциальности корпорации Майкрософт. 
 
После того как это изменение станет активным, мы больше не сможете предоставлять обновления по электронной почте для ожидающих изменений в процессе подготовки. Изменения подготовки PIC обычно загружаются в течение 24-48 часов после ввода. Если у вас по-прежнему возникают проблемы с активацией Skype 48 часов после отправки запроса на подготовку, обратитесь в службу технической поддержки Майкрософт, чтобы проанализировать дополнительные сведения.

> [!IMPORTANT]
> В рамках этого изменения все ранее введенные контактные данные будут очищены с нашей системы на конец апреля 2019 г.


**Вопрос: какие функции поддерживаются в Microsoft Lync и Skype?**

**A:** С помощью Skype теперь вы можете получить доступ к новым возможностям, которые позволят вам разнимать сотни миллионов пользователей, использующих Skype, для расширения распространенных ситуаций. Благодаря этой комбинации Пользователи Lync смогут подключаться и сотрудничать с поставщиками, клиентами и партнерами, полагаться на богатую работу Lync и предоставление доступа к Skype.

  - Обмен мгновенными сообщениями, голосовые и видеозвонки и обмен сообщениями между Lync и пользователями Skype

  - Совместное использование сведений о присутствии, Обмен данными о присутствии между федеративными контактами

  - Простое администрирование — параметры и элементы управления для настройки федеративных коммуникаций в соответствии с политиками и стандартами Организации.

**В: как определить, как подключаться к развертыванию Lync с помощью Skype?**

**A:** Лицензировано для связи с абонентом Skype, если вы используете один из указанных ниже способов.

  - Lync Server (2010 или 2013) и клиентские лицензии на Lync Server Standard ("лицензии"; 2010 или 2013) для пользователей и (или) устройств в вашей организации, которые будут подключаться к Skype. 

  - Lync Online (как отдельные лицензии, так и в составе пакета Office 365).Однако эта служба (pic.lync.com) предназначена только для подготовки к развертыванию Lync Server и гибридного развертывания Lync Server и Lync Online.Подготовка к работе с Lync Online PIC осуществляется на панели управления Lync Online (ЛОКП).

**Вопрос: что необходимо для того, чтобы пользователи Lync были подключены к контактам Skype?**

**A:** После активации домена и включения функций администратором Lync Пользователи Lync могут добавлять контакты Skype в свои списки контактов в клиенте Lync.

**Вопрос: что необходимо для того, чтобы пользователи Skype были подключены к контактам Lync?**

**A:** Все пользователи Skype, желающие подключиться к пользователю Lync, должны иметь учетную запись Майкрософт, связанную с идентификаторами Skype, и входить в нее с помощью учетной записи Майкрософт.Это можно включить в клиенте Skype.

**Вопрос: интеграция с Windows Live по-прежнему доступна?**

**A:** Начиная с октября, 2012 Корпорация Майкрософт приступила к переходу на Skype для Windows Live Messenger (ВЛМ), а в конечном итоге выпустили ВЛМ.Lync продолжит поддерживать Федерацию с ВЛМ, пока ВЛМ находится на рынке, но никакие дополнительные активации доменов для Windows Live не будут разрешены.Передвижение ВЛМ пользователей поддерживается Skype 6,0 для Mac и Windows (выпущено 25 октября 2012 г.), что позволяет входить в систему с помощью учетной записи Майкрософт (например, тех же учетных данных, что и ВЛМ). После того как вы просто входите в Skype, ВЛМ контакты автоматически заполняются в Skype, и пользователи могут использовать преимущества расширенных возможностей связи, такие как звонки на стационарные телефоныные и мобильные телефоны, демонстрация экрана, групповые видеозвонки и поддержка широкой различные устройства.Кроме того, Объединенные контакты Lync пользователей ВЛМ подключаются к Skype вместе с другими списками контактов, и обмен мгновенными сообщениями между Skype и Lync для этих абонентов будет возможен немедленно. Пользователям Lync не нужно ничего делать для обеспечения бесперебойности обслуживания.

**Вопрос: интеграция с Yahoo\! или AOL по-прежнему доступна?**

**A:** Нет. Интеграция с Yahoo\! и AOL в зависимости от поддержки Yahoo\! и AOL.Как для Yahoo, так и для\! и AOL, служба заканчивается 30 июня 2014 г. 

**Вопрос: Могу ли я запробовать подключение к Skype перед покупкой Lync?**

**A:** Возможность подключения к Skype не предоставляется для пробной версии. Вместо пробной версии пользователи Lync с подходящими лицензиями должны просто зарегистрироваться в службе для проверки.

**Вопрос: какая информация нужна для подготовки?**

**A:** Чтобы отправить запрос на подготовку в рамках квалифицированной лицензии, вам понадобятся следующие возможности:

  - Номер соглашения (Майкрософт):
    
      - Поддержка по программе корпоративного лицензирования Майкрософт: номер соглашения корпоративного лицензирования Майкрософт
    
      - Сеть партнера Майкрософт: идентификатор партнера в штаб-Квартирае
    
      - Лицензионное соглашение поставщика услуг: первоначальный регистрационный номер
    
      - Большие объемы услуг: номер регистрации продукта

  - Полные доменные имена (FQDN) для службы пограничного доступа.
    
      - Требуется полное доменное имя по крайней мере одной службы пограничного доступа.
    
      - Если в вашей организации более одного сервера, на котором запущена служба пограничного доступа, укажите полные доменные имена для каждой дополнительной службы доступа к доступу. Внимание! если вы ранее указали полное доменное имя для службы Edge Access и хотите изменить его, подготовка к изменению может занять несколько дней, и это может привести к нарушению обслуживания. Чтобы предотвратить прерывание работы службы, рекомендуется сохранить ранее указанное ДОМЕНное имя службы Edge Access.

  - Домены протокола запуска сеансов (SIP). Это доменный суффикс URI SIP, который пользователи в настоящее время используют для обмена мгновенными сообщениями. Если в вашей организации есть несколько доменов SIP, укажите доменные суффиксы для каждого домена, используемого для обмена мгновенными сообщениями. Например, для user1@contoso.com укажите contoso.com для домена SIP. в user1@example.fabrikam.com укажите example.fabrikam.com в качестве домена SIP.
    
    <div>
    

    > [!NOTE]
    > Указывайте только доменные суффиксы для домена SIP. Не указывайте полные доменные имена, включая полное доменное имя для службы EDGE, для домена SIP.

    
    </div>

  - Контактные данные. Укажите адрес электронной почты для администратора каждого из указанных доменов SIP.

**Вопрос: как включить функцию Lync – связь между абонентами Skype в сценарии с разделением доменов?**

**A:** Если у вас есть сценарий Lync Online 2013 и Lync Server с локальным доменом с разделением (для пользователей, подключенных к Интернету и локально с помощью того же домена SIP), включите функцию подключения Lync-Skype, выполнив следующие действия в указанном порядке.

1.  Настройте локальную связь Lync — Skype, как описано в руководстве по подготовке PIC.

2.  Дождитесь появления подтверждения о том, что ваш домен был предоставлен корпорацией Майкрософт.

3.  После появления подтверждения с помощью центра администрирования Lync включите функцию "внешняя связь". Дополнительные сведения можно найти в разделе[http://office.microsoft.com/en-us/support/configure-external-communications-HA102817865.aspx?CTT=5\&origin=HA102817356](http://office.microsoft.com/en-us/support/configure-external-communications-ha102817865.aspx?ctt=5%26origin=ha102817356)

Этот порядок важен.Прежде чем включать связь в Lync Online, необходимо настроить подключение к локальной сети. При обратном порядке данные, введенные для локального использования, не <https://pic.lync.com> будут проходить через. Если вы уже настроили Lync Online для внешней связи с этим доменом, вы должны отключить его, подождать 24 часа, а <https://pic.lync.com> затем включить внешнюю связь для Lync Online.

**Вопрос: можно ли подготовить несколько полных доменных имен службы доступа к сети для подключения к Skype?**

**A:** Вы можете подготовить только одно полное доменное имя для доступа к одному или нескольким доменам. Вы можете подготовить несколько полных доменных имен EDGE, до 10 для одного запроса, если у них есть отдельные домены.

**В: я могу получить список адресов электронной почты Майкрософт, которые вы нашли для Организации, запрашивающей подготовку к работе.**

**A:** Нет. Эти адреса рассматриваются как личные данные и не являются общими.

**Вопрос: как добавить контакт Windows Live Messenger с ИДЕНТИФИКАТОРом, который содержит домен, отличный от того, который поддерживается Windows Live?**

**A:** Если вы добавляете пользователя Windows Live Messenger с учетной записью или идентификатором в домен, не относящийся к Windows Live, введите адрес в следующем \<формате:\>имя\<пользователя (\>доменное имя) @msn. \<com,\> где имя домена — доменное имя в адресе электронной почты пользователя. Например, если вы хотите добавить ted@contoso.com, вы можете использовать следующий формат: \ мои (contoso. com) @msn. com. Список доменов, администрируемых с помощью Windows Live, можно найти в разделе Поддерживаемые домены на странице http://support.microsoft.com/?kbid=897567"известные проблемы, связанные с обменом мгновенными сообщениями после установки пакета обновления 1 (SP1) для Live Communications Server).

**Вопрос: сколько времени займет процесс подготовки?**

**A:** Для подготовки может потребоваться до 30 дней.

**Вопрос: как узнать, когда будет выполнена подготовка?**

**A:** Корпорация Майкрософт отправит уведомление по электронной почте о завершении подготовки.

**Вопрос: как обновить конфигурацию или контактные данные, которые вы отправляете?**

**A:** Вы можете обновить сведения на том же веб-сайте, который использовался для запроса подготовки, после завершения подготовки. Введите номер своего соглашения и нажмите кнопку Обновить службу.
