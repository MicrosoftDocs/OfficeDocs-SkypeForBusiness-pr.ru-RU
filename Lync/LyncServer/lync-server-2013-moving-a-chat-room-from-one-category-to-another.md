# Перемещение чата из одной категории в другую в Lync Server 2013

 

_**Дата изменения раздела:** 2012-11-01_

Не рекомендуется изменять категорию комнаты сохраняемый сеанс беседы после ее создания. Однако если менеджер комнаты чата имеет привилегии **автора** в другой категории, то технически он или могут переместить комнату из одной категорию в другую. Комната не удаляется и воссоздается. Это изменение связи с базой данных.

Изменение категории комнаты должно происходить редко. Категория определяет разрешенное членство в комнате чата, поэтому при перемещении комнаты чата в другую категорию все системные списки управления доступом (SACL), которые более не поддерживаются новой категорией, очищаются. Например, если пользователь был участником комнаты и больше не является **AllowedMember** в новой категории, членство категории будет изменено, а пользователь будет удален из комнаты.

Сведения о перемещении комнаты чата с помощью оболочки командной строки Windows PowerShell см. в разделе "Управление комнатой" статьи [Настройка сервера сохраняемого сеанса беседы с помощью командлетов Windows PowerShell](configuring-persistent-chat-server-by-using-windows-powershell-cmdlets.md).

Дополнительные сведения о настройке чата см. в разделе [Настройка чатов в Lync Server 2013](lync-server-2013-configure-rooms.md) документации по развертыванию.
