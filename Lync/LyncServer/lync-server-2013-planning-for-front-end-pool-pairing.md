# Планирование сопоставления для пула переднего плана в Lync Server 2013

 

_**Дата изменения раздела:** 2012-09-28_

Для обеспечения лучших возможностей восстановления в Lync Server 2013 развертывайте пары интерфейсных пулов в двух географически распределенных узлах. Каждый из них должен содержать интерфейсный пул, сопряженный с соответствующим интерфейсным пулом в другом узле. Оба узла активны, а служба резервного копирования Lync Server выполняет репликацию данных в реальном времени для синхронизации пулов. Служба резервного копирования – это новый компонент Lync Server 2013, предназначенный для поддержки решения аварийного восстановления. Она устанавливается в интерфейсном пуле при соединении его с другим интерфейсным пулом.

Если пул в одном узле становится недоступным, вы можете перевести пользователей из этого пула в другой пул в другом узле, который предоставляет услуги для всех пользователей в обоих пулах. Для планирования емкости каждый пул должен проектироваться таким образом, чтобы обрабатывать рабочие нагрузки всех пользователей в обоих пулах в случае аварии.
