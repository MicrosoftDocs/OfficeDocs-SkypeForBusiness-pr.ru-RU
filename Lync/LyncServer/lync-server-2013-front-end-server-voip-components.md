# VoIP-компоненты сервера переднего плана для Lync Server 2013

 

_**Дата изменения раздела:** 2012-10-01_

На переднего плана расположены следующие компоненты VoIP:

  - Служба перевода

  - компонент маршрутизации входящих вызовов;

  - компонент маршрутизации исходящих вызовов;

  - компонент маршрутизации единой системы обмена сообщениями Exchange;

  - компонент маршрутизации между кластерами.

  - [Компонент сервера-посредника в Lync Server 2013](lync-server-2013-mediation-server-component.md)

## Служба перевода

Служба перевода – это серверный компонент, который отвечает за преобразование набираемого номера в формат E.164 или другой формат согласно правилам нормализации, определенным администратором. Служба перевода может преобразовывать номера и в другие форматы, не только E.164, если ваша организация использует частную систему нумерации или шлюз или УАТС, которые не поддерживают формат E.164.

## Компонент маршрутизации входящих пакетов

Компонент маршрутизации входящих вызовов обрабатывает входящие вызовы в соответствии с настройками, заданными пользователями в своих клиентах корпоративной голосовой связи. Он также осуществляет делегированные и одновременные звонки, если они настроены пользователем. Например, пользователи указывают, переадресуются ли неотвеченные вызовы или просто записываются в журнал. Если переадресация вызовов включена, пользователи могут указать, следует ли перенаправлять неотвеченные вызовы на другой номер или на сервер обмена сообщениями Exchange, настроенный для ответа на звонки. Компонент маршрутизации входящих вызовов устанавливается по умолчанию на всех Сервер Standard Edition и переднего плана.

## Компонент маршрутизации исходящих вызовов

Компонент маршрутизации исходящих вызовов направляет вызовы на УАТС или ТСОП. Он применяет правила авторизации звонков, заданные политикой голосовой связи пользователя, к вызывающим абонентам и определяет оптимальный шлюз ТСОП для маршрутизации каждого звонка. Компонент маршрутизации исходящих вызовов по умолчанию устанавливается на все Сервер Standard Edition и переднего плана.

Логика маршрутизации, используемая компонентом маршрутизации исходящих вызовов, в значительной степени настраивается сетью или администраторами телефонной связи в соответствии с требованиями организации.

## Компонент маршрутизации единой системы обмена сообщениями Exchange

Компонент маршрутизации обмена сообщениями Exchange занимается маршрутизацией между Lync Server и серверами с обмена сообщениями Exchange для интеграции Lync Server с единой системой обмена сообщениями.

Компонент маршрутизации обмена сообщениями Exchange также занимается переадресацией голосовой почты по ТСОП, если серверы обмена сообщениями Exchange недоступны. Если в узлах филиалов есть пользователи корпоративной голосовой связи без надежного канала глобальной сети к центральному узлу, для обеспечения связи в филиалах, развертываемый в узле филиала, обеспечивает надежность голосовой почты для пользователей филиалов при сбое глобальной сети. Если канал глобальной сети недоступен, для обеспечения связи в филиалах выполняет следующие действия:

  - переадресует неотвеченные вызовы по ТСОП на сервер единой системы обмена сообщениями Exchange в центральном узле;

  - обеспечивает пользователю возможность получения сообщений голосовой почты через ТСОП;

  - ставит в очередь уведомления о пропущенных звонках, а затем загружает их на сервер единой системы обмена сообщениями Exchange при восстановлении канала глобальной сети.

Чтобы включить переадресацию голосовой почты, мы рекомендуем администратору Exchange настроить автосекретаря обмена сообщениями Exchange для приема только сообщений.

Дополнительные сведения об этих возможностях см. в разделах [Планирование интеграции единой системы обмена сообщениями Exchange в Lync Server 2013](lync-server-2013-planning-for-exchange-unified-messaging-integration.md) и [Планирование устойчивости корпоративной голосовой связи в Lync Server 2013](lync-server-2013-planning-for-enterprise-voice-resiliency.md) соответственно.

## Компонент маршрутизации между кластерами

Компонент маршрутизации между кластерами отвечает за маршрутизацию вызовов в основном пуле регистраторов вызываемого абонента. Если это недоступно, компонент перенаправляет вызов на резервный пул регистраторов. Если основной и резервный пулы регистраторов недоступны по сети IP, компонент маршрутизации между кластерами переадресует вызов по ТСОП на номер телефона пользователя.

## Другие компоненты интерфейсных серверов, необходимые для VoIP

К другим компонентам, которые размещаются в переднего плана или Директор и обеспечивают поддержку VoIP, но сами не являются компонентами VoIP, относятся следующие системы:

  - **Пользовательские службы.** Выполняют поиск обратного номера для целевого телефонного номера каждого входящего вызова и сопоставляют этот номер с SIP URI конечного пользователя. С помощью этой информации компонент маршрутизации входящих вызовов направляет вызов зарегистрированным конечным точкам SIP пользователя. Пользовательские службы – это основной компонент на всех переднего плана и Директор.

  - **Репликатор пользователей.** Извлекает номера телефонов пользователей из Доменные службы Active Directory и записывает их в таблицы в базе данных RTC, где они доступны для пользовательских служб и сервером адресной книги. Репликатор пользователей – это основной компонент на всех переднего плана.

  - **Сервер адресной книги.** Предоставляет данные списка глобальных адресов от Доменные службы Active Directory клиентам Lync Server. Он также получает данные пользователя и контактные данные из базы данных RTC, записывает их в файлы адресной книги и сохраняет файлы в общей папке, откуда их могут загрузить клиенты Lync. Сервер адресной книги записывает сведения в базу данных RTCAb, которая используется веб-запросов к адресной книге для ответа на поисковые запросы пользователей из Microsoft Lync 2010 Mobile. При необходимости он нормализует корпоративные телефонные номера пользователей, записанные в базу данных RTC, для предоставления контактов пользователей Lync. Служба адресной книги устанавливается по умолчанию на все переднего плана. веб-запросов к адресной книге устанавливается по умолчанию с веб-службами на каждом сервере переднего плана.
