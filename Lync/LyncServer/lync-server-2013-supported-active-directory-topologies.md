# Поддерживаемые топологии Active Directory в Lync Server 2013

 

_**Дата изменения раздела:** 2016-12-08_

Система Lync Server 2013 поддерживает те же топологии доменных служб Доменные службы Active Directory, что и Microsoft Lync Server 2010 и Microsoft Office Communications Server 2007 R2. Поддерживаются следующие топологии:

  - один лес с одним доменом;

  - один лес с одним деревом и несколькими доменами;

  - один лес с несколькими деревьями и несвязанными пространствами имен;

  - несколько лесов в топологии с центральным лесом;

  - несколько лесов в топологии с лесом ресурсов;

  - несколько лесов в топологии с лесом ресурсов Lync с Exchange Online.

На следующем рисунке поясняются значки, используемые на иллюстрациях в данном разделе.

## Одиночный лес, один домен

Самая простая топология Active Directory из поддерживаемых Lync Server; состоит из леса с одним доменом и является самой распространенной.

На следующем рисунке приведена иллюстрация к развертыванию Lync Server в топологии одного домена Active Directory.

## Одиночный лес, несколько доменов

Другая поддерживаемая Lync Server топология Active Directory, представляющая собой одиночный лес из корневого домена и одного или нескольких дочерних доменов. В топологии Active Directory такого типа домен, в котором вы создаете пользователей, может отличаться от домена, в котором вы развертываете Lync Server. Однако в случае развертывания интерфейсного пула все серверы переднего плана следует развернуть в пуле в рамках одного домена. Имеющаяся в Lync Server поддержка универсальных групп администраторов Windows позволяет реализовать междоменное администрирование.

На следующем рисунке приведена иллюстрация к развертыванию в одиночном лесу с несколькими доменами. На этом рисунке значок пользователя обозначает домен, где хранится учетная запись пользователя, а стрелка указывает на домен, где размещается пул Lync Server. Учетные записи пользователей включают в себя следующее:

  - Учетные записи пользователей в том же домене, что и пул Lync Server.

  - Учетные записи пользователей в домене, отличном от домена с пулом Lync Server.

  - Учетные записи пользователей в дочернем домене домена с пулом Lync Server.

## Одиночный лес, несколько деревьев

Топология из леса с несколькими деревьями состоит из двух или более доменов, которые определяют независимые древовидные структуры и раздельные пространства имен Active Directory.

На следующем рисунке приведена иллюстрация к одиночному лесу с несколькими деревьями. На этом рисунке значок пользователя обозначает домен, где хранится учетная запись пользователя, сплошная линия указывает на пул Lync Server, который размещается в том же или другом домене, а пунктирная линия указывает на пул Lync Server, который размещается в другом дереве. Учетные записи пользователей включают в себя следующее:

  - Учетные записи пользователей в том же домене, что и пул Lync Server.

  - Учетные записи пользователей, расположенные в разных доменах, но в одном дереве с пулом Lync Server.

  - Учетные записи пользователей в дереве, отличном от дерева с пулом Lync Server.

## Несколько лесов, центральный лес

Lync Server поддерживает использование нескольких лесов, которые настроены в топологии с центральным лесом. Топологии с центральным лесом используют контактные объекты в центральном лесу, чтобы представлять пользователей в других лесах. Центральный лет также содержит учетные записи для всех пользователей в этом лесу. Продукт для синхронизации службы каталогов, например Microsoft Identity Integration Server (MIIS), Microsoft Forefront Identity Manager (FIM) 2010 или Microsoft Identity Lifecycle Manager (ILM) 2007 с пакетом дополнительных компонентов 1 (FP1), управляет жизненным циклом учетных записей пользователей внутри организации: когда в одном из лесов создается или удаляется учетная запись пользователя, продукт для синхронизации службы каталогов синхронизирует соответствующий контакт в центральном лесу.

Центральный лес предоставляет следующие преимущества:

  - Сервера, на которых запущен Lync Server, централизованы в рамках одного леса.

  - Пользователи могут выполнять поиск пользователей в любом лесу и взаимодействовать с ними.

  - Пользователи могут просматривать состояние присутствия других пользователей в любом лесу.

  - Продукт для синхронизации службы каталогов автоматизирует добавление и удаление контактных объектов в центральном лесу при создании или удалении учетных записей пользователей.

На следующем рисунке приведена иллюстрация к топологии с центральным лесом. На этом рисунке присутствуют двусторонние отношения доверия между доменом, который содержит Lync Server и находится в центральном лесу, и каждым доменом только для пользователей, который находится в отдельном лесу. Расширение схемы в отдельных лесах пользователей не требуется.

## Несколько лесов, лес ресурсов

В топологии с лесом ресурсов один лес выделен для запуска серверных приложений, таких как Microsoft Exchange Server и Lync Server. В лесу ресурсов размещаются серверные приложения и синхронизованное представление активного объекта пользователя, но он не содержит учетные записи пользователей с поддержкой входа. Лес ресурсов выступает в роли среды общих служб для других лесов, где находятся объекты пользователей. Леса пользователей имеют отношение доверия на уровне лесов с лесом ресурсов. Когда вы развертываете Lync Server в такой топологии, вы создаете в лесу ресурсов один отключенный объект пользователя для каждой учетной записи пользователя в лесах пользователей. Если в лесу ресурсов уже развернут продукт Microsoft Exchange, отключенные учетные записи пользователей могут уже существовать. Продукт для синхронизации службы каталогов, например MIIS, Microsoft Forefront Identity Manager (FIM) 2010 или Microsoft Identity Lifecycle Manager (ILM) 2007 с пакетом дополнительных компонентов 1 (FP1), управляет жизненным циклом учетных записей пользователей. Когда в одном из лесов пользователей создается или удаляется учетная запись пользователя, продукт для синхронизации службы каталогов синхронизирует соответствующее представление пользователя в лесу ресурсов.

Эта топология может использоваться, чтобы предоставить общую инфраструктуру для служб в организациях, которые управляют несколькими лесами, или чтобы отделить администрирование объектов Active Directory от остальных задач администрирования. Эту топологию часто выбирают компании, которым требуется изолировать администрирование Active Directory по соображениям безопасности.

Данная топология имеет преимущество, заключающееся в ограничении потребности в расширении схемы Active Directory до отдельного леса (то есть леса ресурсов).

## Несколько лесов в топологии с лесом ресурсов Lync с Exchange Online

В данной топологии один лес или более размещены локально. В них содержатся учетные записи пользователей Active Directory. Лес ресурсов размещен удаленно и управляется сторонним поставщиком услуг размещения. Лес ресурсов содержит только развертывание Lync Server и синхронизированную репликацию учетных записей пользователей из локального(-ых) леса(-ов) учетных записей пользователей. Однако лес ресурсов не содержит учетные записи пользователей с поддержкой входа. Exchange развертывается в локальных лесах учетных записей пользователей, интегрированных с Exchange Online (гибрид). Или службы эл. почты для локальных учетных записей пользователей предоставляются исключительно Exchange Online.

Лес ресурсов служит выступает в роли среды общих служб для локальных лесов Active Directory, где находятся объекты пользователей. Леса учетных записей пользователей имеют односторонние отношения доверия на уровне лесов с лесом ресурсов. При развертывании Lync Server в данном типе топологии вы создаете в лесу ресурсов один отключенный объект пользователя для каждой учетной записи пользователя в лесах пользователей. Продукт для синхронизации службы каталогов, например MIIS, Microsoft Forefront Identity Manager (FIM) 2010 или Microsoft Identity Lifecycle Manager (ILM) 2007 с пакетом дополнительных компонентов 1 (FP1), управляет жизненным циклом учетных записей пользователей. Когда в одном из лесов пользователей создается или удаляется учетная запись пользователя, продукт для синхронизации службы каталогов синхронизирует соответствующее представление пользователя в лесу ресурсов. Дополнительные сведения о конфигурации развертывания с несколькими лесами см. на странице [Развертывание Lync в архитектуре с несколькими лесами (Партнерское размещение Lync с гибридным Exchange)](http://go.microsoft.com/fwlink/p/?linkid=513216).
