# <a name="user-registration-report-in-lync-server-2013"></a>Отчет о регистрации пользователей в Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

_**Последнее изменение темы:** 2012-10-21_

В отчете о регистрации пользователей представлены общие сведения о действиях входа пользователей, в том числе сведения о количестве пользователей, выполнивших вход в Microsoft Lync Server 2013 в течение определенного периода времени (ежечасно, ежедневно, еженедельно, ежемесячно). Следует иметь в виду, что в отчете лишь сообщается, сколько людей входило в систему. В нем не указывается, *какие* люди входили в систему. Отчеты мониторинга не предоставляют сведения о том, какие конкретные пользователи используют Lync Server 2013 (и какие из них не являются). Тем не менее с помощью отчета об активности пользователей можно получить грубую оценку пользовательской информации.

При предоставлении сведений о входах пользователей в систему в отчете о регистрации пользователей делается два важных разграничения. Во-первых, в отчете входы в систему разбиваются на две основные категории: внутренние и внешние входы в систему. Внутренние входы представляют пользователей, вошедших в систему с внутренней стороны брандмауэра организации (то есть, с компьютера, подключенного к корпоративной сети). Внешние входы представляют пользователей, которые входили в систему за пределами брандмауэра через пограничный сервер (например, пользователь, вошедший в Интернет-кафе, считает как внешний вход). Если необходимо узнать, сколько пользователей входит в систему с внешней стороны брандмауэра, эти сведения можно найти в отчете о регистрации пользователей.

Кроме того, в отчете о регистрации пользователей отмечается, сколько *активных* пользователей присутствовало в течение заданного периода времени. Активный пользователь — это пользователь, который выполнил участие в сеансе обмена мгновенными сообщениями, который участвовал в собрании Lync, выполнил или получил телефонный звонок, или в других случаях использует Lync Server в течение этого периода времени. Такой пользователь отличается от пользователя, вошедшего в систему, но не проявлявшего никакой активности в системе.

<div>

## <a name="accessing-the-user-registration-report"></a>Доступ к отчету о регистрации пользователей

Доступ к отчету о регистрации пользователей предоставляется только на домашней странице отчетов мониторинга. Отчет о регистрации пользователей не связан с какими-либо другими отчетами.

## <a name="making-the-best-use-of-the-user-registration-report"></a>Оптимальное использование отчета о регистрации пользователей

Когда вы развернули сервер Lync Server, выполните следующие действия: как узнать, действительно ли пользователи используют эту новую технологию? Хотя в этом отношении есть определенные ограничения, отчет о регистрации пользователей позволяет ответить на этот вопрос. Чтобы определить, используются ли Lync Server пользователями, необходимо выполнить два действия. Во-первых, нужно получить значение показателя уникальных входов пользователей из отчета о регистрации пользователей. Это значение указывает, сколько отдельных участников вошли в Lync Server.

При сравнении метрика общего числа входов показывает общее количество пользователей, выполнивших вход в Lync Server. Например, пусть Кен Myer вошел на Lync Server пять раз в один день. В этом случае для Кен Майера было бы подсчитано пять отдельных сеансов входа в показателе общего числа входов, но лишь один зарегистрированный пользователь в показателе уникальных зарегистрированных пользователей. Аналогично, пользователи нередко входят в систему с нескольких устройств или из нескольких мест. Например, пользователь может войти в систему с помощью настольного компьютера, его портативного компьютера, а также может иметь IP-телефон, который автоматически выполняет вход на Lync Server. В этом примере имеется один уникальный пользователь с тремя входами в систему.

Для дальнейшего пояснения разницы между общим количеством входов и уникальными входами рассмотрим входы в систему для заданного периода времени, приведенные в следующей таблице.

Обратите внимание, что общее количество входов в систему равно пяти; однако существует только два уникальных регистрировавшихся пользователя: Кен Майер (входивший в систему три раза) и Пилар Аккерман (заходивший в систему дважды). В этом разница между входами в систему и уникальными регистрировавшимися пользователями.

Кроме того, чтобы узнать количество уникальных входов в систему, необходимо знать общее количество пользователей, для которых была включена поддержка Lync Server. Это значение можно получить, открыв командную консоль Lync Server 2013 и выполнив следующую команду Windows PowerShell:

    (Get-CsUser).Count

Если предыдущая команда возвращает значение 1 236, то метрика для пользователей с уникальным входом Возвращает среднее значение 667, что означает, что немного больше половины пользователей, которым разрешите Lync, будут на самом деле входить в систему каждый день (то есть 667 на 1 236, то есть приблизительно 54%).

<div>


> [!WARNING]  
> Следует иметь в виду, что показателями входов учитываются пользователи, которые на самом деле входили в систему в указанный период времени. Они не учитывают пользователей, которые уже были зарегистрированы в системе. Например, если показатель количества уникальных регистрировавшихся пользователей равен 667 и имеется 1236 пользователей, это свидетельствует о том, что около половины пользователей входят в систему. Однако предположим, что 300 пользователей уже находились в системе на момент, когда началась проверка данных о регистрации. Это означает, что у вас есть почти 1 000 пользователей, выполнивших вход в Lync Server, что означает, что ближе к 80% пользователей были зарегистрированы.



</div>

Следует также сравнить значение показателя количества уникальных регистрировавшихся пользователей со значением показателя количества уникальных активных пользователей. Метрика уникального активного пользователя указывает, сколько уникальных пользователей использовало Lync Server: они были присоединены к собранию Lync или участвуют в сеансе обмена мгновенными сообщениями. Это полезная информация, так как Microsoft Lync 2013 может быть настроен для автоматического запуска при каждом запуске Windows пользователем. Из-за этого у вас может быть большое количество пользователей, которые автоматически подключаются к Lync при каждом входе в Windows каждый день, но в этом случае никогда не используете Lync Server в течение этого периода времени.

Метрика Unique Active Users также предоставляет более осмысленные данные в Организации, где пользователи обычно не завершают работу Windows в конце дня. Вместо этого они просто блокируют свои компьютеры и не работают с Windows и Lync. В подобной ситуации каждый день может происходить всего несколько входов в систему, так как пользователи зарегистрировались несколько дней назад и с тех пор не выходили из системы. Однако уникальные активные пользователи сообщают, активно ли пользователи с помощью Lync или другого клиента Lync Server.

## <a name="filters"></a>Фильтры

Фильтры позволяют получить более конкретные наборы возвращаемых данных. Например, отчет о регистрации пользователей позволяет просматривать данные для всех пулов регистраторов и пограничных серверов, а также просматривать данные для отдельного пула. Можно также выбрать порядок группирования данных. В этом случае регистрации группируются по времени, дням, неделям или месяцам.
