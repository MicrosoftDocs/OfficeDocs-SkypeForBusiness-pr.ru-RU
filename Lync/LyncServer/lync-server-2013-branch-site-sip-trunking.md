# Распределение каналов SIP для сайта филиала в Lync Server 2013

 

_**Дата изменения раздела:** 2012-09-21_

В некоторых случаях может потребоваться реализация распределенных магистралей SIP в выбранных сайтах филиалов. Чтобы определить, требуется ли магистраль SIP для сайта филиала, просмотрите сведения в статье [Реализация распределения каналов SIP в Lync Server 2013](lync-server-2013-how-do-i-implement-sip-trunking.md).

Сведения о поддерживаемых параметрах топологии для развертывания магистралей SIP в филиалов см. в разделе [Решения устойчивости для сайтов филиалов в Lync Server 2013](lync-server-2013-branch-site-resiliency-solutions.md).

## Пример анализа требований к магистрали SIP для офиса филиала

Когда принимается решение о развертывании магистрали SIP для сайта филиала, необходимо выполнить анализ расходов для конкретного офиса. Например, предприятие, которое имеет сайт сайт в городе Рэдмонд штата Вашингтон и сайт филиала в Нью-Йорке, должно провести анализ для определения необходимости реализации магистрали SIP из офиса в Нью-Йорке до локального поставщика услуг.

Чтобы определить, является ли экономически выгодной распределенная магистраль SIP в Нью-Йорке, определите, какие номера для прямого соединения с внутренними абонентами будет использовать магистраль SIP, и проанализируйте число звонков, совершаемых из офиса в Нью-Йорке в области, не включающие Рэдмонд (425). Можно иметь точку телефонного подключения номеров для прямого соединения с внутренними абонентами для филиала в сайт. Например, сайт Рэдмонда может размещать номера для прямого соединения с внутренними абонентами для филиала Нью-Йорка. Если затраты на реализацию распределенной магистрали SIP меньше, чем затраты на эти звонки, рассмотрите реализацию магистрали SIP в филиала Нью-Йорка.

## Другие требования к магистрали SIP для офиса филиала

Выбор между развертыванием магистрали SIP вместо шлюза основан на разнице тарифов за междугородные звонки в телефонной сети общего пользования в каждом из вариантов. Если производится развертывание магистрали SIP для сайта филиала, необходимо также определить требования к устойчивости и пропускной способности. Если соединение между сайтом филиала и сайт устойчиво и имеет достаточную пропускную способность, можно развернуть магистраль SIP или шлюз. Развертывать для обеспечения связи в филиалах на сайте филиала не требуется. Если соединение между сайтом филиала и сайт неустойчиво, разверните только для обеспечения связи в филиалах или сервер для обеспечения связи в филиалах со шлюзом или магистралью SIP на сайте филиала.
