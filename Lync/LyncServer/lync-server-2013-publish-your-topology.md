# <a name="publish-your-topology-in-lync-server-2013"></a>Публикация топологии в Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Тема последнего изменения:** 2012-09-08_

Каждый раз, когда вы используете Topology Builder для построения топологии, необходимо опубликовать топологию в базе данных в хранилище Центрального управления, чтобы эти данные можно было использовать для развертывания Lync Server 2013. Чтобы опубликовать топологию, выполните указанные ниже действия.

<div>

## <a name="to-publish-the-topology"></a>Публикация топологии

1.  Запустить построитель топологии: нажмите кнопку **Пуск**, выберите пункт **все программы**, а затем — **Microsoft Lync Server 2013**и нажмите кнопку **Построитель топологии Lync Server**.

2.  В построителе топологии в дереве консоли щелкните правой кнопкой мыши **Lync 2013**и выберите команду **топология публикации**.

3.  На странице **приветствия** в мастере нажмите кнопку **Далее**.

4.  На странице **Topology Builder обнаружена страница хранилища CMS** , нажмите кнопку **Далее**.

5.  На странице **создания других баз данных** нажмите кнопку **Далее**.

6.  Когда состояние указывает на то, что создание базы данных прошло успешно, выполните указанные ниже действия.
    
      - Для просмотра журнала нажмите кнопку **Просмотреть журнал**.
    
      - Для завершения работы мастера нажмите кнопку **Готово**.
        
        <div>
        

        > [!IMPORTANT]  
        > Если это новая установка пограничного сервера или пула EDGE, необходимо экспортировать конфигурацию пограничного сервера на существующем сервере переднего плана, пуле переднего плана или стандартном сервере Standard Edition. Для экспорта конфигурации ознакомьтесь со <A href="lync-server-2013-export-your-topology-and-copy-it-to-external-media-for-edge-installation.md">сведениями о том, как экспортировать топологию Lync Server 2013 и скопировать ее на внешние носители для установки пограничного сервера</A>. Вы будете импортировать файл конфигурации из внешнего носителя или сетевого общего доступа на этапе настройки и развертывания пограничных серверов с помощью мастера развертывания Lync Server.<BR>После того как пограничные серверы будут работоспособны, а локальная база данных хранится в локальной конфигурации, последующие обновления конфигурации Lync Server 2013 будут опубликованы и реплицированы на пограничные серверы.
