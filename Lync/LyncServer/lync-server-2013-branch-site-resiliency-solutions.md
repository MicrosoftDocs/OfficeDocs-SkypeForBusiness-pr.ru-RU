# Решения устойчивости для сайтов филиалов в Lync Server 2013

 

_**Дата изменения раздела:** 2015-03-09_

Есть очевидные преимущества в плане обеспечения устойчивости сайта филиала вашей организации. Например, при потере связи с центральным сайтом пользователи сайта филиала смогут продолжать пользоваться службой корпоративной голосовой связи и голосовой почтой, если настроены параметры переадресации голосовой почты. Дополнительные сведения см. в разделе [Требования к устойчивости для сайтов филиала для Lync Server 2013](lync-server-2013-branch-site-resiliency-requirements.md)). Однако для сайтов, где количество пользователей не достигает 25, решение для обеспечения устойчивости может оказаться неэффективным вложением средств.

Доступно три варианта обеспечения устойчивости сайта филиала. Воспользуйтесь следующей таблицей, чтобы определить оптимальный для себя вариант.

## Описание механизма обеспечения связи в филиалах

Механизм обеспечения связи в филиалах Lync Server включает следующие компоненты:

  - Регистратор для проверки подлинности пользователей, регистрации и маршрутизации вызовов

  - Сервер-посредник для обработки сигналов между регистратором и ТСОП-шлюзом

  - ТСОП-шлюз для маршрутизации вызовов в ТСОП в случае отказа глобальной сети

  - SQL Server Express для локального хранения пользовательских данных

для обеспечения связи в филиалах также содержит ТСОП-магистрали, аналоговые порты и Ethernet-адаптер.

Если подключение посредством глобальной сети к центральному сайту становится недоступным на сайте, пользователи в пределах филиала остаются зарегистрированными в регистраторе для обеспечения связи в филиалах и могут по-прежнему непрерывно использовать голосовую службу за счет подключения для обеспечения связи в филиалах к ТСОП. Пользователи сайта филиала, подключающиеся из дома и из других удаленных расположений смогут зарегистрироваться в регистраторе центрального сайта в случае отсутствия подключения к сайту филиала посредством глобальной сети. Эти пользователи имеют доступ ко всем возможностям единой системы обмена данными за одним исключением – входящие вызовы на сайт филиала будут переадресовываться на голосовую почту. После восстановления подключения по глобальной сети пользователям сайта будут снова предоставлены все функциональные возможности. Отработка сбоя для обеспечения связи в филиалах и восстановление служб не требует присутствия ИТ-администратора.

Lync Server поддерживает до двух для обеспечения связи в филиалах в филиала.

## Обзор развертывания механизма обеспечения связи филиала

для обеспечения связи в филиалах изготавливается производителями оборудования - партнерами Microsoft; развертывание выполняется от лица производителей сертифицированными дистрибьюторами. Развертывание должно выполняться только после развертывания Lync Server на центральном сайте при условии наличия подключения к сайту филиала и предоставления пользователям сайта филиала разрешений для работы с корпоративной голосовой связи.

Дополнительные. сведения о данных этапах см. в разделе [Развертывание устройства или сервера обеспечения связи в филиалах с Lync Server 2013](lync-server-2013-deploying-a-survivable-branch-appliance-or-server.md) документации по развертыванию.

## Описание сервера обеспечения связи в филиалах

В топологий создайте сайт филиала, добавьте для обеспечения связи в филиалах на созданный сайт и запустите развертывания Lync Server на компьютере, где необходимо установить роль.
