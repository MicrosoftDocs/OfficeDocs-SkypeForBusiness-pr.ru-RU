# <a name="create-users-and-contacts"></a>Создание пользователей и контактов

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Последнее изменение темы:** 2013-02-22_

Для создания пользователей и контактов в процессе подготовки к нагрузочному тестированию и нагрузочному тестированию необходимо использовать средство подготовки пользователей Lync Server 2013 (UserProvisioningTool.exe).

Ниже приведен список терминов и определений, которые можно использовать при прочтении этой статьи.

  - Организационное подразделение — подразделение доменных служб Active Directory (OU).

  - Федеративный/перекрестный пул — пользователи, которым будет разрешено общаться с пользователями из других служб обмена мгновенными сообщениями, таких как MSN Network of Internet Services, AOL® и Yahoo \! ®.

  - Списки рассылки — объекты в доменных службах Active Directory, содержащие список пользователей доменных служб Active Directory, которые используются для запуска связи с группами людей.

  - Служба сведений о местоположении — служба Lync Server 2013, которая при включении и настройке на телефоне позволяет получать данные о физическом расположении для служб Enhanced 9-1-1 (E9-1-1).

  - Номера телефонов в США — номера телефонов, назначенные пользователям, в дополнение к URI SIP, используемому для маршрутизации входящих и исходящих звонков, а также обратного поиска номеров (RNL).

<div>

## <a name="create-users-and-contacts-by-using-userprovisioningtoolexe"></a>Создание пользователей и контактов с помощью UserProvisioningTool.exe

Для создания пользователей и контактов для моделирования нагрузки необходимо использовать средство подготовки пользователей Lync Server. Средство подготовки пользователей Lync Server устанавливается вместе с пакетом средства нагрузки и производительности Lync Server. Убедитесь, что на сервере переднего плана или сервере Standard Edition запущен установщик пакетов (CapacityPlanningTool.msi). Запустите средство подготовки пользователей Lync Server, выполнив UserProvisioningTool.exe файла (расположенного по адресу% Инсталледдиректори% Линкстрессандперфтул \\ линкстресс) на сервере переднего плана или на сервере Standard Edition.

<div>


> [!IMPORTANT]  
> Для запуска UserProvisioningTool.exe необходимо войти в систему с учетной записью члена группы безопасности "Администраторы домена". Это необходимо для выполнения из этого контекста, так как UserProvisioningTool.exe будет создавать и настраивать новых пользователей доменных служб Active Directory.



</div>

<div>


> [!NOTE]  
> Когда вы создадите большое количество пользователей (10 000 или более), запустите UserProvisioningTool.exe на высокодоступном компьютере. Обратите внимание, что во время создания пользователей контроллер домена также будет работать с высокой загрузкой.



</div>

Когда откроется средство подготовки пользователей Lync Server, щелкните **Конфигурация** и выберите пункт **загрузить конфигурацию**. Чтобы начать настройку пользователей и контактов, загрузите файл по умолчанию, включенный в пакет, SampleData.xml. Это предписывает поля с примерами данных, которые необходимо изменить для системы. Если у вас есть предварительно настроенный файл XML, который уже содержит настроенные параметры, загрузите этот файл. Заполните поля в средстве подготовки пользователей Lync Server, как описано в следующих разделах.

![Вкладка "Создание пользователя".](images/JJ945587.80d3c17b-7482-4818-8381-1eff8717d2fe(OCS.15).jpg "Вкладка "Создание пользователя".")

Чтобы настроить параметры сервера, выполните указанные ниже действия.

1.  В поле полное доменное имя **фонового пула**введите полное доменное имя сервера Standard Edition или интерфейсного пула, в котором будут размещаться пользователи.

2.  В параметре **префикс имени пользователя**введите префикс, который будет использоваться для создания имен пользователей в целях тестирования.

3.  В поле **пароль**укажите пароль, который будет применяться для всех тестовых учетных записей пользователей.

4.  В поле **домен SIP**введите имя домена, которое будет использоваться для тестовых URI SIP (универсальные коды ресурсов) для тестовых пользователей.

5.  В поле **домен учетной записи**введите доменное имя текущего домена доменных служб Active Directory, под которым вы хотите создать тестовых пользователей.

6.  В **поле Подразделение введите**имя подразделения доменных служб Active Directory, в котором необходимо создать объекты пользователя. Если подразделение не существует, оно будет создано.

7.  В поле **код телефонной области**введите код города из трех цифр, который будет использоваться для тестовых учетных записей пользователей. Убедитесь, что телефонный код города не конфликтует с кодами областей других пользователей в доменных службах Active Directory.

8.  Установите флажок **Поддержка голосовой связи** , если необходимо включить тестовых пользователей для корпоративной голосовой связи.

9.  В поле **число пользователей**укажите общее число тестовых пользователей, которое вы хотите создать.

10. В поле **начальный индекс**укажите начальный номер, который будет использоваться в качестве суффикса для префикса имени пользователя.

## <a name="create-users-button"></a>Кнопка "создать пользователей"

При нажатии кнопки Создать пользователей выполняется проверка всех входных параметров.

  - Если имеются ошибки проверки, будет предложено исправить эти входные значения.

  - Если все входные значения верны, будут запущены создание пользователей в доменных службах Active Directory. В нижней части этой формы появится индикатор выполнения. Не рекомендуется закрывать приложение, пока активен индикатор выполнения.

Создание пользователя — медленный процесс. Это может занять несколько минут. Если количество пользователей очень велико, процесс может даже занять несколько часов. Если пользователи уже существуют, они обновляются с учетом изменений. Вы можете проверить, что пользователи были созданы, войдя в систему в качестве одного из пользователей в диапазоне. Используйте префикс пользователя, номер пользователя и @sipDomain в качестве имени пользователя (например, LyncUser10@contoso.net), а также заданный пароль.

## <a name="delete-users-button"></a>Кнопка "удалить пользователей"

При нажатии кнопки удалить пользователей будут проверены все входные параметры.

  - Если имеются ошибки проверки, будет предложено исправить эти входные значения.

  - Если все входные значения верны, будут запущены отключение и удаление пользователей в доменных службах Active Directory. В нижней части этой формы появится индикатор выполнения. Не рекомендуется закрывать приложение, пока активен индикатор выполнения.

<div>


> [!NOTE]  
> <OL>
> <LI>
> <P>Поддерживаются только телефонные номера в формате США. Номера телефонов всегда назначаются пользователям, а все пользователи, созданные UserProvisioningTool.exe, включены для корпоративной голосовой связи. Все сценарии, в которых используется телефонный номер, например автосекретарь конференц-связи или вызовы UC-PSTN, используют этот номер для правильного направления звонков. По этой причине у каждого пользователя должен быть уникальный номер телефона. Если требуется создавать пользователей дважды, команда завершится с ошибками, если не используется другой код города или если предыдущие пользователи отключены с помощью командлета <STRONG>Disable-CsUser</STRONG> .</P>
> <LI>
> <P>Перед созданием контактов необходимо сначала выполнить репликацию пользователя, выполняемую на вкладке Пользователи. Если вы только что создали пользователей, необходимо дождаться завершения репликации Lync Server и заполнения учетных записей пользователей в базе данных. Если репликация пользователей еще не завершена, будет выводиться сообщение об ошибке. Вы узнаете, когда пользователи завершают репликацию при запуске службы внешнего интерфейса Lync Server 2013 или успешно выполнили командлет <STRONG>Get – CsUser</STRONG> для последнего пользователя.</P></LI></OL>

## <a name="contacts-creation-tab"></a>Вкладка "Создание контактов"

Вкладка Создание контактов позволяет указать сведения для контактов пользователей.

![Вкладка "Создание контактов".](images/JJ945587.7508726e-83e6-4878-8edd-114543d9af24(OCS.15).jpg "Вкладка "Создание контактов".")

Чтобы настроить контакты пользователей, выполните указанные ниже действия.

1.  В разделе средние контакты на пользователя укажите среднее число контактов, заполняемых в списках контактов для каждого пользователя.

2.  Установите флажок Фиксированный, если необходимо создать равное количество контактов для каждого пользователя. Если требуется изменить количество контактов, созданных для пользователей, снимите этот флажок.

3.  В разделе средние группы контактов на пользователя укажите количество групп контактов на пользователя. Это значение должно быть меньше среднего числа контактов на пользователя.

4.  В процентах федеративного и кросс-пула укажите число от 0 до 100. Этот процент контактов будет создаваться с федеративными пользователями.

5.  В разделе префикс пользователя федеративного/междоступного пула укажите имя пользователя для федеративных пользователей, которое будет добавлено в списки контактов локальных пользователей.

6.  В домене SIP-сервер федеративного и межсайтового пула укажите имя домена SIP для федеративных пользователей.
    
    <div>
    

    > [!NOTE]  
    > При создании контактов ни один из пользователей не должен быть подписан.

    
    </div>

7.  На вкладке Создание пользователя убедитесь, что параметры указаны правильно. Диапазон пользователей, для которых будут созданы контакты, можно получить на вкладке Создание пользователя.

8.  Нажмите кнопку Создать контакты, чтобы начать создание контакта. Этот процесс может занять несколько минут. После завершения появится диалоговое окно с сообщением "операция успешно завершена". Вы можете проверить контакты, созданные при входе в систему в качестве пользователя, созданного на вкладке Создание пользователя.
    
    <div>
    

    > [!NOTE]  
    > После создания контактов это средство перезапускает все серверы переднего плана в целевом пуле. Для запуска серверов переднего плана может потребоваться больше времени (до 2 часов) в зависимости от того, сколько контактов было создано при выполнении этой операции.

    
    </div>

## <a name="distribution-list"></a>Список рассылки

Одной из функций средства нагрузки и производительности Lync Server 2013 является имитация функции расширения списка рассылки в Lync 2013. Если вы не включаете расширение DL в Усерпровисионингтул, вы можете пропустить этот шаг.

![Вкладка "Создание списка рассылки".](images/JJ945587.0a1d681b-2aea-4724-90d8-efa8a526f600(OCS.15).jpg "Вкладка "Создание списка рассылки".")

На вкладке Список рассылки можно создавать списки рассылки, которые будут использоваться средством "нагрузка и производительность" для расширения списка рассылки. Прежде чем создавать списки рассылки, Lync Server 2013 должен быть уже установлен. Необходимо запустить Lync Server 2013 ForestPrep. В противном случае атрибуты DL не будут существовать в схеме доменных служб Active Directory, и средство не сможет создавать списки рассылки.

Чтобы настроить списки рассылки, выполните указанные ниже действия.

1.  В списке число списков рассылки укажите общее количество списков рассылки, которые необходимо создать. Рекомендуется начать с удвоенного числа пользователей. Они пронумерованы от 0 до n – 1.

2.  В поле префикс списка рассылки укажите префикс, который будут иметь списки рассылки. Например, если вы задаете списки рассылки 100 и префикс Тестдл, списки рассылки будут называться testDL0, testDL1 и т. д. в testDL99.

3.  В списке минимальное число участников в файле Redist. Укажите Минимальное число пользователей, добавляемых в каждый список рассылки.

4.  В списке Максимальное число участников в файле Redist. Укажите максимальное число пользователей, добавляемых в каждый список рассылки.

<div>

## <a name="create-distribution-lists-button"></a>Кнопка "создать списки рассылки"

При нажатии кнопки создать списки рассылки средство запрашивает доменные службы Active Directory, чтобы узнать, существуют ли списки рассылки, удовлетворяющие префиксу и номерам. Средство создаст только те из них, которые еще не существуют. При добавлении участников в эти недавно созданные списки рассылки будут выбираться пользователи из диапазона, указанного на вкладке Создание пользователя.

## <a name="location-info-service-config-tab"></a>Вкладка "Настройка службы сведений о местоположении"

Одной из функций средства нагрузки и производительности Lync Server 2013 является создание фиктивных файлов конфигурации для службы сведений о местоположении. Как правило, служба "сведения о местоположении" не оказывает существенного влияния на производительность серверов.

![Вкладка "Настройка службы сведений о местонахождении".](images/JJ945587.52ea4e9e-d50a-4dc9-982b-31ee5ace4578(OCS.15).jpg "Вкладка "Настройка службы сведений о местонахождении".")

Если вы намерены проверить эту функцию, вы можете ввести значения, указанные в форме, а затем нажать кнопку Создать LIS Config Files. Он будет создавать CSV-файлы с именем LIS \_Subnet.csv, lis \_Switches.csv, LIS \_Ports.csv и LIS \_WAP.csv. Затем вы можете импортировать эти CSV-файлы в базу данных LIS с помощью командлета **Set-кслиссубнет** , командлета Set **-кслиссвитч** , командлета **Set-кслиспорт** и командлета **Set-ксвирелессакцесспоинт** соответственно.
