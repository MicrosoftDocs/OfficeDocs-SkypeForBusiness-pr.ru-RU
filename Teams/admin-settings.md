<a name="admin-settings-for-apps-in-microsoft-teams"></a>Параметры администратора для приложений в Microsoft Teams
==========================================
> [!IMPORTANT]
> [!INCLUDE [new-teams-sfb-admin-center-notice](includes/new-teams-sfb-admin-center-notice.md)]

Приложения предоставляют готовые инструменты для организации, позволяющие использовать все преимущества Teams. Они сочетают функции вкладок, расширений для сообщений, соединителей и ботов, предоставляемых корпорацией Майкрософт, созданных сторонними производителями или разработчиками в организации.

В разделе **Приложения Teams** в Центре администрирования Microsoft Teams вы можете задать политики для управления приложениями в организации. Например, вы можете задать политики, чтобы определять, какие приложения будут доступны пользователям Teams, а также настроить Teams, закрепив наиболее важные приложения для пользователей.

Мы постоянно улучшаем работу с приложениями в Teams и добавляем новые компоненты и функции. С течением времени будут созданы дополнительные возможности управления приложениями, поэтому следите за самыми свежими данными о политиках приложений.

## <a name="app-permission-policies"></a>Политики разрешений приложений

С помощью политик разрешений приложений можно блокировать или разрешать приложения как для всей организации, так и для конкретных пользователей.  При блокировании приложения все взаимодействия с ним будут отключены и приложение не будет отображаться в Teams для пользователей.

Например, с помощью политик разрешений приложений можно:

- Отключить приложение, которое создает угрозу потери данных или разрешений для организации.
- Постепенно развертывать новые сторонние или настраиваемые приложения для конкретных пользователей.
- Упростить взаимодействие с пользователем, особенно при развертывании Teams в организации.

Дополнительные сведения см. в статье [Управление политиками разрешений приложений в Teams](teams-app-permission-policies.md).

## <a name="app-setup-policies"></a>Политики настройки приложений

Политики настройки приложений позволяют настроить работу с приложениями для пользователей. Выберите приложения, которые вы хотите закрепить на панели приложения в клиентах Teams, и порядок их отображения в классических, мобильных и веб-клиентах.

Ниже приведены некоторые примеры использования политик настройки приложений.
- Управляйте поддержкой и принятием основных приложений. Например, закрепите пользовательское приложение для управления набором сотрудников и талантами в группе по управлению персоналом.
- Выборочно закрепите основные функции Teams, такие как "Чат", "Команды" и "Звонки". Это поможет обеспечить выполнение пользователями конкретных действий в Teams.  

Дополнительные сведения см. в статье [Управление политиками настройки приложений в Teams](teams-app-setup-policies.md).

## <a name="custom-app-policies-and-settings"></a>Политики и параметры пользовательских приложений

Служба Teams позволяет разработчикам в организации создавать, тестировать и развертывать пользовательские приложения для других пользователей. Пользовательские приложения можно добавить в Teams, загрузив пакет приложения в ZIP-файле непосредственно в команду или в личном контексте. С помощью политик настройки приложений можно определить, кто в организации может загружать пользовательские приложения. Вы также можете настроить параметры для всей организации, чтобы определить, могут ли пользователи взаимодействовать с конкретными пользовательскими приложениями.

Дополнительные сведения см. в статье [Управление политиками и параметрами пользовательских приложений в Teams](teams-custom-app-policies-and-settings.md).
