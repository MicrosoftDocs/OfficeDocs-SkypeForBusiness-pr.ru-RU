# <a name="use-call-analytics-to-troubleshoot-poor-call-quality"></a>Использование средства аналитики звонков для устранения неудовлетворительного качества связи

В этой статье объясняется, как использовать средство аналитики звонков для устранения некачественных звонков и собраний Microsoft Teams для отдельных пользователей, если вы являетесь администратором Teams или специалистами службы поддержки группового взаимодействия или инженерами-разработчиками.


  
## <a name="call-analytics-permissions"></a>Разрешения на доступ к аналитическим звонкам

В этой статье предполагается, что вы уже настроили средство аналитики звонков. Если вы этого еще не сделали, прочтите [параметр Настройка аналитики звонков для Teams](set-up-call-analytics.md).

## <a name="introduction-to-call-analytics"></a>Общие сведения об аналитических звонках

В средстве аналитики звонков отображаются подробные сведения о звонках и собраниях для каждого пользователя в вашей учетной записи Office 365. Она содержит сведения о устройствах, сетях, подключениях и качестве связи (любые из них могут быть очень факторами для слабого звонка или качества собраний). Если вы отправляете сведения о сборке, сайте и клиенте, эти сведения будут также отображаться для каждого звонка и собрания. С помощью средства аналитики звонков вы можете выяснить, почему у пользователя неудовлетворительный звонок или процесс собрания.

В средстве аналитики звонков вы можете пообщаться с вами по телефону или собранию, например от одного участника до второго участника. Анализируя эти сведения, администратор Teams может изолировать проблемные области и определить основную причину плохого качества.
   
Как администратор Teams, вы получаете полный доступ ко всем данным аналитики звонков для каждого пользователя. Кроме того, вы можете назначать роли Azure Active Directory сотрудникам службы поддержки. Чтобы узнать больше об этих ролях, ознакомьтесь с разделом [предоставление поддержки и сотрудников службы технической](set-up-call-analytics.md#give-permission-to-support-and-helpdesk-staff)поддержки. Не пропустите, [что делает каждая роль в службе поддержки групп?](#what-does-each-teams-support-role-do) ниже.

## <a name="where-to-find-per-user-call-analytics"></a>Где найти аналитический звонок для каждого пользователя

Чтобы просмотреть все сведения о звонках и данные для пользователя, перейдите в [центр администрирования Teams](https://admin.teams.microsoft.com). В разделе **Пользователи**выберите пользователя, а затем откройте вкладку **Журнал звонков** на странице профиля пользователя. Здесь вы найдете все звонки и собрания для этого пользователя за последние 30 дней.

![Снимок экрана: все пользовательские данные аналитики](media/teams-difference-between-call-analytics-and-call-quality-dashboard-image1.png)

Чтобы получить дополнительные сведения о текущем сеансе, включая подробную статистику мультимедиа и сети, щелкните сеанс, чтобы просмотреть сведения.

## <a name="troubleshoot-user-call-quality-problems"></a>Устранение неполадок с качеством связи пользователей 

1. Откройте центр администрирования группы ( https://admin.teams.microsoft.com) и войдите в систему с помощью своей службы поддержки связи или учетных данных администратора Teams).

2. На **панели мониторинга**в области **поиска пользователей**Начните вводить имя или SIP-адрес пользователя, звонки которого вы хотите устранить, или выберите **Просмотр пользователей** , чтобы просмотреть список пользователей.

3. Выберите пользователя из списка.

4. Выберите **Журнал звонков**, а затем выберите звонок или собрание, для которых вы хотите устранить неполадки.
    
5. Перейдите на вкладку **Дополнительно** и найдите желтые и красные элементы, указывающие на качество связи или проблемы с подключением.
    
    В сведениях сеанса для каждого звонка или собрания незначительные проблемы выводятся желтым цветом. Если что-то является желтым, это может быть вызвано за пределами обычного диапазона, но в этом случае, скорее всего, она не является основной причиной проблемы. Если что-то имеет красный цвет, это существенная проблема, и, как правило, основная причина плохого качества звонка для этого сеанса. 
      
В редких случаях данные качества обслуживания не принимаются для сеансов голосовой связи. Часто это связано с тем, что удаленный звонок или соединение с клиентом завершается. В этом случае оценка сеанса **недоступна**.
  
В таблице ниже описаны основные проблемы **, в которых**в сеансах голосовой связи содержатся данные качества взаимодействия (QoE).
  
|**Ошибка**|**Область**|**Описание**|
|:-----|:-----|:-----|
|Настройка звонка  <br/> |Session  <br/> |Код ошибки MS-DIAG 20-29 означает, что при настройке звонка произошел сбой. Пользователю не удалось присоединиться к звонку или собранию.  <br/> |
|Слабый Звонок по голосовой сети  <br/> |Session  <br/> |Обнаружены проблемы, связанные с качеством сети (например, потеря пакетов, колебание, NMOS, RTT или невидимая степень).  <br/> |
|Устройство не работает  <br/> |Device  <br/> | Устройство работает неправильно. Нерабочее отношение устройства: <br/>  DeviceRenderNotFunctioningEventRatio >= 0,005 <br/>  DeviceCaptureNotFunctioningEventRatio >= 0,005 <br/> |
