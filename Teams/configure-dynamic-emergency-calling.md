# <a name="plan-and-configure-dynamic-emergency-calling"></a>Планирование и настройка динамических экстренных вызовов 

Динамические звонки в экстренные службы для планов звонков Майкрософт, операторов Подключение и прямой маршрутики предоставляют возможность настраивать и перенаправить экстренные вызовы, а также уведомлять сотрудников службы безопасности в зависимости от текущего расположения клиента Teams.  

В зависимости от топологии сети (элементов сети, связанных с адресами для экстренного обращения), определенных администратором клиента, клиент Teams предоставляет сведения о сетевом подсети в запросе в службу сведений о расположении (LIS). Если есть совпадение, liS возвращает расположение клиенту.

Клиент Teams включает данные о местоположении в экстренные вызовы. Эти данные затем используются поставщиком экстренных служб для определения соответствующей точки ответа на общедоступные вопросы безопасности (PSAP) и для направления вызова в эту службу PSAP, которая позволяет диспетчеру PSAP получить расположение вызываемой стороны.  

Для динамических экстренных вызовов необходимо следующее:

1. Администратор сети настраивает параметры сети и liS для создания карты сетевого расположения или местоположения для экстренного нахождения.

2. Во время запуска и периодически или при смене сетевого подключения клиент Teams отправляет запрос о расположении, содержащий сведения о сетевом подсети, в параметры сети и в lis.

   - При совпадении сетевых параметров сайта политики экстренных вызовов возвращаются Teams с этого сайта. (Дополнительные сведения о политиках см. в этой [теме.](#configure-emergency-policies)

   - Если имеется совпадение с LIS, в клиент Teams экстренное расположение из сетевого элемента, к Teams подключенного клиента. Совпадение выполняется в следующем порядке, при этом возвращается первый совпаданый результат:
       - WAP
       - Переключение/порт Ethernet
       - Переключатель Ethernet
       - Подсеть

3. Когда клиент Teams звонит на экстренные вызовы, местоположение для экстренного вызова передается в сеть ПСС.

Возможность автоматической маршрутики в соответствующий пункт ответа на общедоступные вопросы безопасности (PSAP) зависит от страны использования Teams пользователя.

Планы звонков и операторы Подключение Майкрософт включают динамические экстренные службы маршрутинга для пользователей в США и Канаде.

Однако при прямой маршрутике требуется дополнительная настройка для маршрутации экстренных вызовов и, возможно, для партнерских подключений. Администратор должен настроить подключение к поставщику службы экстренных маршрутов (ERS)  (США и Канада) или настроить SBC для приложения ELIN для идентификационных номеров экстренных расположений. Сведения о поставщиках ERS см. в [этой](direct-routing-border-controllers.md)ссылке.

## <a name="assign-emergency-addresses"></a>Назначение адресов для экстренного обращения

Вы можете назначить адреса для обращения в экстренных службах следующим образом:

- Для пользователей плана звонков.

- Для оператора Подключение пользователей в зависимости от возможностей, которые были назначены номеру при отправке их в перечень &mdash; клиентов.

Чтобы обеспечить автоматическую маршрутику экстренных звонков на территории США, необходимо убедиться, что местоположения для экстренного вызова, которые назначены сетевым идентификаторам, содержат связанные с ними геопроидеки. (Адреса для экстренного устранения чрезвычайной ситуации без гео кодов невозможно на назначены сетевым идентификаторам, которые требуются для динамических местоположений.)

Azure Карты используется для служб на основе местонахождения. При вводе адреса для обращения в экстренных Microsoft Teams центре администрирования Teams azure Карты адрес.

- Если совпадение найдено, географические коды включаются автоматически.

- Если совпадение не найдено, вы сможете вручную создать адрес для обращения в экстренных службах. Для этого можно использовать функцию перетасковки ПИН-кода. 

> [!NOTE]
> Сетевым идентификаторам не могут быть назначены адреса для экстренного устранения чрезвычайной ситуации, которые находятся более двух лет назад. Вам потребуется повторно создать старые адреса.

Вы можете добавлять и назначать адреса для экстренного Microsoft Teams центре администрирования или с помощью PowerShell. Дополнительные сведения см. ## <a name="configure-network-settings"></a>Настройка параметров сети

Параметры сети используются для определения расположения клиента Teams, а также для динамического получения политик звонков в экстренных службах и местоположения для экстренного вызова. Вы можете настроить параметры сети в соответствии с тем, как ваша организация хочет использовать экстренные вызовы.

Параметры сети включают сайты, которые содержат набор подсетей и используются исключительно для динамического назначения политик пользователям. Например, для сайта Редмонда могут быть назначены политика экстренных вызовов и маршрутная маршрутия экстренных вызовов, чтобы для всех пользователей, перемещающихся из дома или другого расположения Майкрософт, были настроены номера экстренных служб, маршруты и служба безопасности, относясь к Redmond.  

Надежные IP-адреса содержат набор внешних IP-адресов корпоративной сети Интернета и используются для определения того, находится ли конечная точка пользователя в корпоративной сети. Попытка получить динамическую политику или расположение будет выполнена только в том случае, если внешний IP-адрес пользователя соответствует IP-адресу надежного IP-адреса.

Дополнительные сведения об IP-адресах, сетевых регионах, сайтах и адресах подсети см. в настройках сети [для облачных голосовых функций.](cloud-voice-network-settings.md)

Параметры сети настраиваются в центре администрирования Microsoft Teams с помощью PowerShell. Дополнительные см. в управлении топологией [сети для облачных функций голосовой связи.](manage-your-network-topology.md)

Обратите внимание, что некоторые изменения параметров сети (например, нового адреса, сетевого идентификатора и так далее) могут распространяться и быть доступны Teams клиентам в течение некоторого времени (до Teams).  

> [!Note]
> Подсети также можно определить в LIS и могут быть связаны с местоположением для экстренного экстренного ситуация.  Подсети LIS должны быть определены с помощью сетевого ИД, совпадающих с диапазоном IP-адресов подсети, который назначен клиентам. Например, сетевой ИД для IP-адреса клиента или маски 10.10.10.150/25 составляет 10.10.10.128. Дополнительные сведения см. в дополнительных сведениях [о правилах TCP/IP-адресов и подсети.](/troubleshoot/windows-client/networking/tcpip-addressing-and-subnetting)

> [!Important]
> Параметры сети не поддерживаются при развертывании облачных прокси-служб, которые изменяют IP-адреса источников из Teams клиентов.



**Для пользователей планов звонков и операторов Подключение:**

- Если требуется динамическая настройка уведомлений службы безопасности, необходимо настроить как надежные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только надежные IP-адреса. Настройка параметров сети не требуется.

- Если ни одно из них не требуется, настройка параметров сети не требуется. 

**Для пользователей прямой маршрутии:**

- Если требуется динамическое обеспечение экстренных вызовов или динамическая настройка уведомлений службы безопасности, необходимо настроить как надежные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только надежные IP-адреса. Настройка параметров работы с меток не требуется.

- Если ни одно из них не требуется, настройка параметров сети не требуется.


## <a name="configure-location-information-service"></a>Настройка службы сведений о расположении

Клиент Teams получает адреса для экстренного обращения из местоположений, связанных с различными сетевыми идентификаторами. 

Чтобы клиент получил расположение, необходимо заполнить его сетевыми идентификаторами (подсетями, сетевыми сетями, переключателями, портами) и местоположениями для экстренного нахождения. Это можно сделать в центре Microsoft Teams или с помощью PowerShell.

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. В области навигации слева перейдите к пункту **Расположения**  >  **Сетевые & расположения.**
2. Щелкните вкладку сетевого идентификатора, который вы хотите добавить. Например, выберите **подсети**, точки **доступа Wi-Fi,** **Переключатели** или **Порты.** Затем нажмите **кнопку Добавить**.
3. Заполняйте поля, добавьте местоположение для экстренного ситуация и нажмите кнопку **Применить**.

### <a name="using-powershell"></a>С помощью PowerShell

Чтобы добавить порты, коммутаторы, подсети и WAP на ЛИС, воспользуйтесь следующими cmdlets:

- [Get](/powershell/module/skype/get-csonlinelissubnet?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelissubnet?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelissubnet?view=skype-ps) -CsOnlineLisSubnet
- [Get](/powershell/module/skype/get-csonlinelisport?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelisport?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelisport?view=skype-ps) -CsOnlineLisPort
- [Get](/powershell/module/skype/get-csonlineliswirelessaccesspoint?view=skype-ps), [Set](/powershell/module/skype/set-csonlineliswirelessaccesspoint?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlineliswirelessaccesspoint?view=skype-ps) -CsOnlineLisWirelessAccessPoint
- [Get](/powershell/module/skype/get-csonlinelisswitch?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelisswitch?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelisswitch?view=skype-ps) -CsOnlineLisSwitch

>[!Important]
>Если подсеть используется как часть сетевых сайтов, для отображения динамических расположений их необходимо переопрепреть в службе сведений о расположении.

## <a name="configure-emergency-policies"></a>Настройка политик для экстренного ситуация

Для настройки экстренных вызовов используйте следующие политики: Управлять этими политиками можно в Microsoft Teams администрирования или с помощью PowerShell.

- **Политика маршрутинга экстренных вызовов — применяется только к прямой маршрутике.** Эта политика настраивает номера экстренных служб, при желании маскирует число, а также маршрут ПСПС для каждого номера. Эту политику можно назначить пользователям, сетевым сайтам или обоим пользователям. Дополнительные см. в управлении политиками маршрутисти экстренных вызовов [для прямой маршрутизов.](manage-emergency-call-routing-policies.md)  

   (План звонков и оператор Подключение для пользователей автоматически включены экстренные вызовы с номерами экстренных служб из страны в зависимости от местоположения Microsoft 365 или Office 365 звонков.)

- **Политика экстренных вызовов: применяется к планам звонков, Подключение и прямой маршрутике.** Эта политика настраивает службу уведомлений службы безопасности при звонках на экстренные вызовы. Вы можете настроить пользователей, которые должны уведомлять их и каким образом они должны быть уведомлены. Например, чтобы автоматически уведомить сотрудников службы безопасности организации и сделать так, чтобы они прослушивали экстренные звонки.  Эта политика может быть либо назначена пользователям, либо сетевым сайтам, либо обоим. Дополнительные информации см. в [этой](manage-emergency-calling-policies.md)Teams.

## <a name="enable-users-and-sites"></a>Включить пользователей и сайты

Вы можете назначать политики маршрутинга экстренных вызовов и политики экстренных вызовов пользователям и сайтам. Имейте в виду, что политики маршрутинга экстренных вызовов применяются только к прямой маршрутике. (Хотя эту политику можно назначить пользователю плана звонков или оператора Подключение, она не будет действовать.)

Политики назначаются в центре Microsoft Teams или с помощью PowerShell. Дополнительные сведения см. в статьях

- [Управление политиками маршрутинга экстренных вызовов для прямой маршрутизы](manage-emergency-call-routing-policies.md)
- [Управление политиками экстренных вызовов в Teams](manage-emergency-calling-policies.md)

Вот примеры PowerShell:

Чтобы включить уведомление о службе безопасности для определенного пользователя, воспользуйтесь следующей командой:

```PowerShell
Grant-CsTeamsEmergencyCallingPolicy -Identity user1 -PolicyName SecurityDeskNotification
```

Чтобы назначить политику "Политика экстренных вызовов Contoso 1" для сайта 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallingPolicy "Contoso Emergency Calling Policy 1"
```

Чтобы включить прямую маршрутику для конкретного пользователя для экстренных вызовов, используйте следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallRoutingPolicy -Identity user1 -PolicyName UnitedStates
```

Чтобы назначить политику "Маршруты экстренных вызовов Contoso в Нью-Йорке" для сайта 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallRoutingPolicy "Contoso New York Emergency Call Routing"
```

Если политика экстренных вызовов назначена сетевому сайту и пользователю, а этот пользователь находится на этом сайте, политика, назначенная сетевому сайту, переопределит политику, назначенную пользователю.

## <a name="test-emergency-calling"></a>Проверка экстренных вызовов

Некоторые поставщики услуг экстренных маршрутов (ERSP) в США предлагают тестовый бот для экстренного вызова.

- **План звонков** и оператор Подключение пользователи в США или Канаде могут использовать предопределенный тестовый номер 933 для проверки конфигурации экстренных вызовов. Этот номер маршрутируется боту, который затем выводит на экран номер телефона (номер телефонной линии вызова), адрес для экстренного вызова или местоположение, а также то, будет ли звонок автоматически перенаключен на PAP или на экран.

- **Клиенты прямой маршрутики** в США должны координировать действия со своей службой ERSP для тестирования службы.
