# <a name="plan-and-configure-dynamic-emergency-calling"></a>Планирование и настройка динамических экстренных вызовов 

Динамические звонки в экстренные службы для планов звонков Майкрософт и телефонная система прямой маршрутизаации предоставляют возможность настраивать и перенаправить экстренные вызовы, а также уведомлять сотрудников службы безопасности в зависимости от текущего расположения клиента Teams.  

В зависимости от топологии сети, которую определяет администратор клиента, клиент Teams предоставляет сведения о сетевом под подключениях в запросе к службе сведений о расположении (LIS). Если есть совпадение, liS возвращает расположение клиенту. Эти данные о расположении передаются клиенту.  

Клиент Teams включает данные о расположении в экстренные вызовы. Эти данные затем используются поставщиком экстренных служб для определения соответствующей точки ответа на общедоступные вопросы безопасности (PSAP) и для направления вызова в эту службу PSAP, которая позволяет диспетчеру PSAP получить расположение вызываемой стороны.  

Для динамических экстренных вызовов необходимо следующее:

1. Администратор сети настраивает параметры сети и liS для создания карты сетевого расположения или местоположения для экстренного нахождения.

   Для прямой маршрутации требуется дополнительная конфигурация для маршрутации экстренных вызовов и, возможно, для подключения партнеров. Администратор должен настроить подключение к поставщику службы экстренных маршрутов **(США)** или SBC для приложения ELIN.

2. Во время запуска и периодически или при смене сетевого подключения клиент Teams отправляет запрос о расположении, содержащий сведения о сетевом подсети, в параметры сети и liS.

   - При совпадении сетевых параметров сайта политики экстренных вызовов возвращаются Teams с этого сайта. (Дополнительные сведения о политиках см. в этой [странице.](#configure-emergency-policies)

   - Если имеется совпадение с LIS, в клиент Teams экстренное расположение из сетевого элемента, к Teams подключенного клиента. Совпадение выполняется в следующем порядке, при этом возвращается первый совпаданый результат:
       - Wap
       - Переключение/порт Ethernet
       - Переключатель Ethernet
       - Подсеть

3. Когда клиент Teams звонит на экстренные вызовы, местоположение для экстренного вызова передается в сеть ПСС.

   Для прямой маршрутики администратор должен настроить SBC для отправки экстренных вызовов поставщику ERS или настройки приложения ELIN SBC.

## <a name="assign-emergency-addresses"></a>Назначение адресов для экстренного обращения

Вы можете назначать адреса для экстренного вызова как пользователям плана звонков, так и сетевым идентификаторам, которые необходимы для динамического получения расположения. (Поддерживаются subnet и Wi-Fi AP. В настоящее время переключение или порт Ethernet Windows 8.1 и более поздних).

Чтобы обеспечить автоматическую маршрутику экстренных звонков на территории США, необходимо убедиться, что местоположения для экстренного звонков, которые назначены сетевым идентификаторам, содержат связанные с ними геопрод. (Адреса для экстренного устранения чрезвычайной ситуации без гео кодов невозможно на назначены сетевым идентификаторам, которые требуются для динамических местоположений.)

Azure Карты используется для служб на основе местонахождения.  Когда вы вводите адрес для экстренного обращения в Центре администрирования Microsoft Teams, Teams azure Карты адрес:

- Если совпадение найдено, географические коды включаются автоматически.

- Если совпадение не найдено, вы сможете вручную создать адрес для обращения в экстренных службах. Для этого можно использовать функцию перетасковки ПИН-кода. 

Это означает, что если существующее местоположение для экстренного устранения проблем, созданное для назначения пользователям плана звонков, предназначено для динамического расположения, этот же адрес необходимо создать повторно, чтобы включить в него геопродазии. Чтобы различать эти расположения, необходимо у них разное описание. Пользователям, у которых есть старое расположение, можно напометь новое местоположение для экстренного нахождения. При полном переносе старое расположение можно удалить.

Вы можете добавлять и назначать адреса для обращения в экстренных Microsoft Teams центре администрирования или с помощью PowerShell. Дополнительные сведения см. [в](add-change-remove-emergency-location-organization.md) статьи Добавление местоположения для экстренного ситуация для организации и Назначение местоположения для экстренного ситуация [для пользователя.](assign-change-emergency-location-user.md)

## <a name="configure-network-settings"></a>Настройка параметров сети

Параметры сети используются для определения расположения клиента Teams, а также для динамического получения политик звонков в экстренных службах и местоположения для экстренного вызова. Вы можете настроить параметры сети в зависимости от того, как ваша организация хочет использовать экстренные вызовы.

Параметры сети включают сайты, которые содержат набор подсетей и используются исключительно для динамического назначения политик пользователям. Например, для сайта Редмонда могут быть назначены политика для экстренных вызовов и маршрутная маршрутия экстренных вызовов, чтобы для каждого пользователя, перемещаемого из дома или другого расположения Майкрософт, были настроены номера для экстренного вызова, маршруты и служба безопасности, специфичная для Редмонда.  

> [!Note]
> Подсети также можно определить в LIS и могут быть связаны с местоположением для экстренного экстренного ситуация.  Подсети LIS должны быть определены с помощью сетевого ИД, совпадающих с диапазоном IP-адресов подсети, который назначен клиентам. Например, сетевой ИД для IP-адреса клиента или маски 10.10.10.150/25 составляет **10.10.10.128**. Дополнительные сведения см. в дополнительных сведениях [о правилах TCP/IP-адресов и подсети.](/troubleshoot/windows-client/networking/tcpip-addressing-and-subnetting)

> [!Important]
> Параметры сети не поддерживаются при развертывании облачных прокси-служб, которые изменяют IP-адреса источников из Teams клиентов.

Помните о следующих определениях: Дополнительные сведения см. в [настройках сети для облачных функций голосовой связи.](cloud-voice-network-settings.md)

- Надежные IP-адреса содержат набор внешних IP-адресов корпоративной сети Интернета и используются для определения того, находится ли конечная точка пользователя в корпоративной сети. Попытка получить динамическую политику или расположение будет выполнена только в том случае, если внешний IP-адрес пользователя соответствует IP-адресу надежного IP-адреса. Соответствие между IP-адресами IPv4 и IPv6 зависит от формата IP-пакетов, относящихся к сетевым настройкам.  (Если общедоступный IP-адрес имеет как IPv4, так и IPv6, их необходимо добавить в качестве надежных IP-адресов.)

- Область сети содержит коллекцию сетевых сайтов. 

- Сетевой сайт — это расположение, в котором организация имеет физическое значение, например офис, набор зданий или комплекс зданий. Эти сайты определяются как набор IP-подсетей.

- Сетевая подсеть должна быть связана с определенным сетевым сайтом. Расположение клиента определяется на основе сетевой подсети и связанного сетевого сайта.  

Параметры сети настраиваются в центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные см. в управлении топологией [сети для облачных функций голосовой связи.](manage-your-network-topology.md)

Обратите внимание, что некоторые изменения параметров сети (например, нового адреса, сетевого идентификатора и так далее) могут распространяться и быть доступны Teams клиентам.  

**Для пользователей плана звонков:**

- Если требуется динамическая настройка уведомлений службы безопасности, необходимо настроить как надежные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только надежные IP-адреса.

- Если ни одно из них не требуется, настройка параметров сети не требуется. 

**Для пользователей прямой маршрутии:**

- Если требуется динамическое обеспечение экстренных вызовов или динамическая настройка уведомлений службы безопасности, необходимо настроить как надежные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только надежные IP-адреса.

- Если ни одно из них не требуется, настройка параметров сети не требуется.


## <a name="configure-location-information-service"></a>Настройка службы сведений о расположении

Клиент Teams получает адреса для экстренного обращения из местоположений, связанных с различными сетевыми идентификаторами. Поддерживаются как подсети, так и беспроводные точки доступа (WAP). В настоящее время переключение/порт Ethernet Windows 8.1 и более поздних.

Чтобы клиент получил расположение, необходимо заполнить его сетевыми идентификаторами (подсетями, сетевыми сетями, переключателями, портами) и местоположениями для экстренного нахождения. Это можно сделать в центре Microsoft Teams или с помощью PowerShell.

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. В левой области навигации перейдите к пункту **Расположения**  >  **Сетевые & расположения.**
2. Щелкните вкладку сетевого идентификатора, который вы хотите добавить. Например, выберите **подсети**, точки **доступа Wi-Fi,** **Переключатели** или **Порты.** Затем нажмите **кнопку Добавить**.
3. Заполняйте поля, добавьте местоположение для экстренного ситуация и нажмите кнопку **Применить**.

### <a name="using-powershell"></a>С помощью PowerShell

Чтобы добавить порты, коммутаторы, подсети и WAP на ЛИС, воспользуйтесь следующими cmdlets:

- [Get](/powershell/module/skype/get-csonlinelissubnet?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelissubnet?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelissubnet?view=skype-ps) -CsOnlineLisSubnet
- [Get](/powershell/module/skype/get-csonlinelisport?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelisport?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelisport?view=skype-ps) -CsOnlineLisPort
- [Get](/powershell/module/skype/get-csonlineliswirelessaccesspoint?view=skype-ps), [Set](/powershell/module/skype/set-csonlineliswirelessaccesspoint?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlineliswirelessaccesspoint?view=skype-ps) -CsOnlineLisWirelessAccessPoint
- [Get](/powershell/module/skype/get-csonlinelisswitch?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelisswitch?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelisswitch?view=skype-ps) -CsOnlineLisSwitch

>[!Important]
>Если подсеть используется как часть сетевых сайтов, для отображения динамических расположений их необходимо переопрепреть в службе сведений о расположении.

## <a name="configure-emergency-policies"></a>Настройка политик для экстренного срочная ситуация

Для настройки экстренных вызовов используйте следующие политики: Управлять этими политиками можно в центре администрирования Microsoft Teams с помощью PowerShell.

- **Политика маршрутизов экстренных вызовов** — применяется только к прямой маршрутике. Эта политика настраивает номера экстренных служб, маскирует для каждого номера (при желании) и маршрут через ДНР для каждого номера.  Эту политику можно назначить пользователям, сетевым сайтам или обоим пользователям. (Планы звонков Teams для клиентов автоматически включены для экстренных вызовов с номерами экстренных служб из страны в зависимости Microsoft 365 или Office 365 местоположения.)  Подробнее см. в управлении политиками маршрутистики экстренных вызовов [для прямой маршрутизов.](manage-emergency-call-routing-policies.md)

- **Политика экстренных вызовов** — относится к планам звонков и прямой маршрутике. Эта политика настраивает службу уведомлений службы безопасности при звонках на экстренные вызовы. Вы можете настроить пользователей, которые должны уведомлять их и каким образом они должны быть уведомлены. Например, для автоматического уведомления службы безопасности организации и их прослушивания во время экстренных вызовов.  Эта политика может быть либо назначена пользователям, либо сетевым сайтам, либо обоим. Дополнительные информации см. в [этой](manage-emergency-calling-policies.md)Teams.

## <a name="enable-users-and-sites"></a>Включить пользователей и сайты

Вы можете назначать политики маршрутинга экстренных вызовов и политики экстренных вызовов пользователям и сайтам. Имейте в виду, что политики маршрутинга экстренных вызовов применяются только к прямой маршрутике. (Хотя эту политику можно назначить пользователю плана звонков, она не будет действовать.)

Политики назначаются в центре Microsoft Teams или с помощью PowerShell. Дополнительные сведения см. в статьях

- [Управление политиками маршрутинга экстренных вызовов для прямой маршрутизы](manage-emergency-call-routing-policies.md)
- [Управление политиками экстренных вызовов в Teams](manage-emergency-calling-policies.md)

Вот несколько примеров PowerShell.

Чтобы включить уведомление о службе безопасности для определенного пользователя, воспользуйтесь следующей командой:

```PowerShell
Grant-CsTeamsEmergencyCallingPolicy -Identity user1 -PolicyName SecurityDeskNotification
```

Чтобы назначить политику "Политика экстренных вызовов Contoso 1" для сайта 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallingPolicy "Contoso Emergency Calling Policy 1"
```

Чтобы включить прямую маршрутику для конкретного пользователя для экстренных вызовов, используйте следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallRoutingPolicy -Identity user1 -PolicyName UnitedStates
```

Чтобы назначить политику "Маршруты экстренных вызовов Contoso в Нью-Йорке" для сайта 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallRoutingPolicy "Contoso New York Emergency Call Routing"
```

Если политика для экстренных вызовов назначена сетевому сайту и пользователю, а этот пользователь находится на этом сайте, политика, назначенная сетевому сайту, переопределит политику, назначенную пользователю.

## <a name="test-emergency-calling"></a>Проверка экстренных вызовов

Некоторые поставщики услуг экстренных маршрутов (ERSP) в США предлагают тестовый бот для экстренного вызова.

- **Пользователи плана звонков** в США могут использовать предопределенный тестовый номер 933 для проверки конфигурации экстренных вызовов. Этот номер маршрутируется боту, который затем выводит на экран номер телефона (номер телефонной линии вызова), адрес для экстренного вызова или местоположение, а также то, будет ли звонок автоматически перенаключен на PAP или на экран.

- **Клиенты прямой маршрутики** в США должны координировать действия со своей службой ERSP для тестирования службы.
