# <a name="plan-and-configure-dynamic-emergency-calling"></a>Планирование и настройка динамических экстренных вызовов 

Динамические экстренные вызовы для планов звонков Майкрософт и прямой маршрутации телефонной системы предоставляют возможность настраивать и перенаправить экстренные вызовы и уведомлять сотрудников службы безопасности в зависимости от текущего расположения клиента Teams.  

На основе топологии сети, определяемой администратором клиента, клиент Teams предоставляет сведения о сетевом под подключениях в запросе к службе сведений о расположении (LIS). Если есть совпадение, lis возвращает расположение клиенту. Эти данные о расположении передаются клиенту.  

Клиент Teams включает данные о расположении в экстренные вызовы. Затем эти данные используются поставщиком экстренных служб для определения соответствующей точки ответа на общедоступный запрос безопасности (PSAP) и для направления вызова в эту службу PAP, что позволяет диспетчеру PSAP получить местоположение вызываемой службы.  

Для динамических экстренных вызовов необходимо:

1. Администратор сети настраивает параметры сети и lis для создания карты сетевого или экстренного расположения.

   Для прямой маршрутации требуется дополнительная настройка для маршрутации экстренных вызовов и, возможно, для связи с партнером. Администратор должен настроить подключение к поставщику службы экстренных маршрутов  (ERS) (США) или настроить SBC для приложения идентификационного номера для местоположения экстренного местоположения (ELIN).

2. Во время запуска и периодичности работы или при смене сетевого подключения клиент Teams отправляет запрос о расположении, содержащий сведения о сетевом подсети, в параметры сети и на lis.

   - При совпадении сетевых параметров сайта политики экстренных вызовов возвращаются клиенту Teams с этого сайта. (Дополнительные сведения о политиках см. в сведениях [об их настройке.](#configure-emergency-policies)

   - При совпадении с LIS в клиент Teams возвращается местоположение для экстренного экстренного ситуации из сетевого элемента, к какому клиенту Teams подключен клиент Teams. Совпадение выполняется в следующем порядке, при этом возвращается первый совпадает результат:
       - WAP
       - Переключение/порт Ethernet
       - Переключение Ethernet
       - Подсеть

3. Когда клиент Teams звонит в экстренные службы, местоположение для экстренного вызова передается в сеть ННР.

   Для прямой маршрутики администратор должен настроить SBC для отправки экстренных вызовов поставщику ERS или для настройки приложения SBC ELIN.

## <a name="assign-emergency-addresses"></a>Назначение адресов для экстренного устранения проблем

Вы можете назначать адреса для экстренного вызова как пользователям плана звонков, так и сетевым идентификаторам, которые необходимы для динамического получения расположения. (Поддерживаются subnet и Wi-Fi AP. Переключение/порт Ethernet в настоящее время Windows 8.1 и более поздних.

Чтобы обеспечить автоматическую маршрутику экстренных вызовов в США, необходимо убедиться, что местоположения, которые назначены сетевым идентификаторам, содержат связанные с ним гео кода. (Адреса для экстренного экстренного помощи без гео кодов невозможно на указываете сетевые идентификаторы, необходимые для динамических местоположений.)

Карты Azure используются для служб, основанных на расположении.  Когда вы вводите адрес для экстренного помощи с помощью Центра администрирования Microsoft Teams, Teams проверяет наличие адреса в Картах Azure:

- При обнаружении совпадения географические коды включаются автоматически.

- Если совпадение не найдено, вы можете вручную создать адрес для экстренного помощи. Для этого можно использовать функцию перетасковки ПИН-кода. 

Это означает, что если существующее местоположение для экстренного вызова, которое создается для назначения пользователям плана звонков, предназначено для динамического расположения, этот же адрес необходимо повторно создать, чтобы включить в него географические коды. Чтобы различать эти расположения, следует у каждого из них у конфликтующей информации. Для пользователей, у которых есть старое местоположение, можно навести новое местоположение для экстренного экстренного помощи. После полного переноса старое расположение можно удалить.

Вы можете добавлять и назначать адреса для экстренного помощи в Центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения [](add-change-remove-emergency-location-organization.md) см. в дополнительных сведениях о добавлении местоположения для экстренного нахождения организации и назначении местоположения [пользователю.](assign-change-emergency-location-user.md)

## <a name="configure-network-settings"></a>Настройка параметров сети

Параметры сети используются для определения расположения клиента Teams, а также для динамического получения политик экстренных вызовов и местоположения для экстренного вызова. Вы можете настроить параметры сети в зависимости от того, как ваша организация хочет использовать экстренные вызовы.

Параметры сети включают сайты, которые содержат набор подсетей и используются исключительно для динамического назначения политик пользователям. Например, для сайта Редмонда могут быть назначены политика экстренных вызовов и маршрутная политика, чтобы любой пользователь, перемещающийся из дома или другого расположения Майкрософт, мог настроить номера экстренных служб, маршруты и службу безопасности для Редмонда.  

>[!Note]
>Подсети также можно определить в lis и могут быть связаны с местоположением для экстренного экстренного ситуации.  

При этом следует помнить об определениях, которые следующую: Дополнительные сведения см. в [параметрах сети для функций облачного голосового связи.](cloud-voice-network-settings.md)

- Надежные IP-адреса содержат набор внешних IP-адресов корпоративной сети и используются для определения того, находится ли конечная точка пользователя в корпоративной сети. Попытка получить динамическую политику или расположение будет выполнена только в том случае, если внешний IP-адрес пользователя соответствует IP-адресу надежного IP-адреса. Совпадение можно установить с IP-адресами IPv4 или IPv6 и зависит от формата IP-пакета, отправленного в параметры сети.  (Если общедоступный IP-адрес имеет как IPv4, так и IPv6, необходимо добавить оба в качестве надежных IP-адресов.)

- Область сети содержит коллекцию сетевых сайтов. 

- Сетевой сайт — это расположение, в котором организация имеет физическое значение, например офис, набор зданий или много зданий. Эти сайты являются набором IP-подсетей.

- Сетевая подсеть должна быть связана с определенным сетевым сайтом. Расположение клиента определяется на основе подсети сети и связанного с ней сайта сети.  

Параметры сети настраиваются в Центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные узнать см. в топологии облачных голосовых функций управления [сетью.](manage-your-network-topology.md)

Обратите внимание, что некоторые изменения параметров сети (например, новый адрес, идентификатор сети и так далее) распространяются и будут доступны клиентам Teams через некоторое время (до нескольких часов).  

**Для пользователей плана звонков:**

- Если требуется динамическая настройка уведомлений службы безопасности, необходимо настроить как надежные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только надежные IP-адреса.

- Если они не требуются, настройка параметров сети не требуется. 

**Для пользователей прямой маршрутки:**

- Если требуется динамическая настройка экстренных вызовов или динамическая настройка уведомлений службы безопасности, необходимо настроить как надежные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только надежные IP-адреса.

- Если они не требуются, настройка параметров сети не требуется.


## <a name="configure-location-information-service"></a>Настройка службы сведений о расположении

Клиент Teams получает адреса для экстренного помощи из мест, связанных с разными сетевыми идентификаторами. Поддерживаются как подсетеи, так и беспроводные точки доступа (WAP). В настоящее время переключение/порт Ethernet Windows 8.1 и более поздних.

Чтобы клиент получил расположение, необходимо заполнить его идентификаторами сети (подсетями, waP, коммутаторами, портами) и местоположениями для экстренного нахождения. Это можно сделать в Центре администрирования Microsoft Teams или с помощью PowerShell.

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. В области навигации слева перейдите к пунктам **Locations**  >  **Networks & locations.**
2. Щелкните вкладку, представляюную сетевой идентификатор, который вы хотите добавить. Например, выберите "Подсети", "Точки **доступа Wi-Fi",** **"Коммутаторы"** или **"Порты".** Затем нажмите **кнопку "Добавить".**
3. Заполняйте поля, добавьте местоположение для экстренного помощи и нажмите кнопку **"Применить".**

### <a name="using-powershell"></a>С помощью PowerShell

Для добавления портов, коммутаторов, подсетей и waP на lis воспользуйтесь следующими cmdletами:

- [Get](/powershell/module/skype/get-csonlinelissubnet?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelissubnet?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelissubnet?view=skype-ps) -CsOnlineLisSubnet
- [Получить,](/powershell/module/skype/get-csonlinelisport?view=skype-ps) [Установить,](/powershell/module/skype/set-csonlinelisport?view=skype-ps) [Удалить](/powershell/module/skype/remove-csonlinelisport?view=skype-ps) -CsOnlineLisPort
- [Get](/powershell/module/skype/get-csonlineliswirelessaccesspoint?view=skype-ps), [Set](/powershell/module/skype/set-csonlineliswirelessaccesspoint?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlineliswirelessaccesspoint?view=skype-ps) -CsOnlineLisWirelessAccessPoint
- [Получить,](/powershell/module/skype/get-csonlinelisswitch?view=skype-ps) [Установить,](/powershell/module/skype/set-csonlinelisswitch?view=skype-ps) [Удалить](/powershell/module/skype/remove-csonlinelisswitch?view=skype-ps) -CsOnlineLisSw ветвь

>Если подсеть используется как часть сетевых сайтов, для отображения динамических расположений их необходимо переопрепреизировать в службе сведений о расположении.

## <a name="configure-emergency-policies"></a>Настройка политик для экстренного срочная ситуация

Для настройки экстренных вызовов используйте следующие политики: Управлять этими политиками можно в Центре администрирования Microsoft Teams или с помощью PowerShell.

- **Политика маршрутинга экстренных вызовов** — применяется только к прямой маршрутике. Эта политика настраивает номера для экстренного помощи, скрывает за номер при желании и маршрут ННР для каждого номера.  Эту политику можно назначить пользователям, сетевым сайтам или обоим пользователям. (Клиенты Teams планов звонков автоматически получают возможность вызова экстренной помощи с номерами экстренной помощи из страны в зависимости от местоположения Microsoft 365 или Office 365.)  Дополнительные узнать см. в управлении политиками маршрутинга экстренных [вызовов для прямой маршрутинга.](manage-emergency-call-routing-policies.md)

- **Политика для экстренных вызовов** — применяется к планам звонков и прямой маршрутике. Эта политика настраивает уведомления службы безопасности при звонках в экстренные службы. Вы можете настроить, кому и каким образом нужно уведомлять пользователей. Например, для автоматического уведомления сотрудников службы безопасности организации и их прослушивания при вызовах экстренной помощи.  Эту политику можно на назначении пользователям, сетевым сайтам или обоим сайтам. Дополнительные узнать см. в [управлении политиками экстренных вызовов в Teams.](manage-emergency-calling-policies.md)

## <a name="enable-users-and-sites"></a>Включить пользователей и сайты

Вы можете назначать политики маршрутизов экстренных вызовов и политики экстренных вызовов пользователям и сайтам. Имейте в виду, что политики маршрутинга экстренных вызовов применяются только к прямой маршрутике. (Хотя эту политику можно назначить пользователю плана звонков, она не будет действовать.)

Политики назначаются в Центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения см. в статьях

- [Управление политиками маршрутинга экстренных вызовов для прямой маршрутинга](manage-emergency-call-routing-policies.md)
- [Управление политиками экстренных вызовов в Teams](manage-emergency-calling-policies.md)

Вот несколько примеров PowerShell.

Чтобы включить уведомление службы безопасности для определенного пользователя, используйте следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallingPolicy -Identity user1 -PolicyName SecurityDeskNotification
```

Чтобы назначить политику Contoso Emergency Calling Policy 1 сайту 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallingPolicy "Contoso Emergency Calling Policy 1"
```

Чтобы включить для конкретного пользователя прямой маршрутки экстренные вызовы, используйте следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallRoutingPolicy -Identity user1 -PolicyName UnitedStates
```

Чтобы назначить политику contoso New York Emergency Call Routing для сайта 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallRoutingPolicy "Contoso New York Emergency Call Routing"
```

Если политика экстренных вызовов назначена сетевому сайту и пользователю, а этот пользователь находится на этом сайте сети, политика, назначенная сетевому сайту, переопределит политику, назначенную пользователю.

## <a name="test-emergency-calling"></a>Проверка экстренных вызовов

Некоторые поставщики услуг экстренных маршрутов (ERSP) в США предлагают тест-бот для вызова экстренной помощи.

- **Пользователи планов звонков** в США могут проверять конфигурацию экстренных вызовов с помощью предварительно заранее тестового номера 933. Этот номер перенаначируется боту, который затем возвращает номер телефона (номер телефонной линии звонка), адрес для экстренного вызова или местоположение, а также то, будет ли звонок автоматически перенаначен в PAP или перед вызовом.

- **Клиенты, направляющие** прямые маршруты в США, должны согласовать тестовую службу с их ERSP.
