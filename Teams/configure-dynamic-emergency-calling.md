# <a name="plan-and-configure-dynamic-emergency-calling"></a>Планирование и настройка динамических экстренных вызовов 

Динамические экстренные вызовы для планов звонков Майкрософт, Оператор Connect, Teams Phone Mobile и Прямая маршрутизация предоставляют возможность настраивать и маршрутизировать экстренные вызовы, а также уведомлять сотрудников службы безопасности в зависимости от текущего расположения клиента Teams.  

На основе топологии сети (сетевых элементов, связанных с адресами для чрезвычайных ситуаций), определяемой администратором клиента, клиент Teams предоставляет сведения о сетевом подключении в запросе к службе сведений о расположении (LIS). При наличии совпадения LIS возвращает клиенту расположение.

Клиент Teams включает данные о местоположении в рамках экстренного вызова. Затем эти данные используются поставщиком экстренных служб для определения соответствующей точки ответов общественной безопасности (PSAP) и маршрутизации вызова в этот PSAP, что позволяет диспетчеру PSAP получить расположение вызывающего.  

Для динамических экстренных вызовов необходимо выполнить следующее:

1. Администратор сети настраивает параметры сети и LIS для создания карты расположения сети или аварийного реагирования.

2. Во время запуска и периодически после этого или при изменении сетевого подключения клиент Teams отправляет запрос на расположение, содержащий сведения о сетевом подключении, в параметры сети и LIS.

   - Если имеется соответствие сайта параметров сети, политики экстренных вызовов возвращаются клиенту Teams с этого сайта. (Дополнительные сведения о политиках см. в разделе [Настройка политик для чрезвычайных ситуаций](#configure-emergency-policies).

   - При наличии соответствия LIS клиенту Teams возвращается аварийное расположение из сетевого элемента, к которому подключен клиент Teams. Совпадение выполняется в следующем порядке, при этом возвращается первый соответствующий результат:
       - WAP
       - Коммутатор или порт Ethernet
       - Коммутатор Ethernet
       - Подсеть

3. Когда клиент Teams выполняет экстренный вызов, расположение для экстренного реагирования передается в сеть ТСОП.

Возможность автоматической маршрутизации к соответствующей точке ответов общественной безопасности (PSAP) зависит от страны использования пользователя Teams.

Планы звонков Майкрософт, партнеры Operator Connect и Партнеры Teams Phone Mobile включают службы динамической экстренной маршрутизации для пользователей в США и Канаде.

Однако для прямой маршрутизации требуется дополнительная конфигурация для маршрутизации экстренных вызовов и, возможно, для подключения партнеров. Администратор должен убедиться, что шлюз ТСОП, направив экстренный вызов, был настроен для добавления сведений о расположении в исходящий INVITE (путем установки для параметра PidfloSupported значения True в объекте шлюза ТСОП в сети). Кроме того, администратор должен настроить подключение к поставщику службы аварийной маршрутизации (ERS) (США и Канада) **или** настроить пограничный контроллер сеансов (SBC) для приложения ELIN. Сведения о поставщиках ERS см. в разделе [Контроллеры границ сеансов, сертифицированные для прямой маршрутизации](direct-routing-border-controllers.md).

## <a name="supported-clients"></a>Поддерживаемые клиенты

В настоящее время поддерживаются следующие клиенты.  Часто проверяйте обновления этого списка.

- Классический клиент Teams для Microsoft Windows
- Классический клиент Teams для Apple macOS
- Мобильный клиент Teams для клиента Apple iOS версии 1.0.92.2019121004 и App Store версии 1.0.92 и выше
- Мобильный клиент Teams для клиента Android и Google Play версии 1416/1.0.0.2019121201 и более поздних версий
- Телефон Teams версии 1449/1.0.94.2019110802 и выше
- Комнаты Teams версии 4.4.25.0 и выше

> [!NOTE]
> Расположения на основе подсети и Wi-Fi поддерживаются во всех поддерживаемых клиентах Teams. <br><br>
> Ethernet/Switch (LLDP) поддерживается в следующих средах:
> - На данный момент Windows версии 10.0 и более поздних версий.<br>
> - Mac OS, для которой [требуется программное обеспечение для включения LLDP](https://www.microsoft.com/download/details.aspx?id=103383).<br>
> - Телефон Teams с приложением Teams версии 1449/1.0.94.202110101 и более поздних версий.

> [!NOTE]
> Динамические экстренные вызовы, включая уведомление службы безопасности, не поддерживаются в веб-клиенте Teams. Чтобы запретить пользователям использовать веб-клиент Teams для звонков по номерам ТСОП, можно задать политику звонков Teams и отключить параметр **Разрешить звонки по ТСОП** . Дополнительные сведения см. [в разделах Политики звонков в Teams](teams-calling-policy.md) и [Set-CsTeamsCallingPolicy](/powershell/module/skype/set-csteamscallingpolicy?view=skype-ps). 

> [!NOTE]
> Телефоны 3PIP не поддерживают динамические экстренные вызовы. 



## <a name="assign-emergency-addresses"></a>Назначение адресов для экстренного реагирования

Адреса для экстренного реагирования можно назначить следующим образом:

- Пользователям плана звонков.

- Для пользователей Operator Connect и Teams Phone Mobile в зависимости от возможностей&mdash;, назначенных номеру, когда оператор передает их в инвентаризацию клиента.

- К сетевым идентификаторам, необходимым для динамического получения расположения. 

Для поддержки автоматической маршрутизации экстренных вызовов в США необходимо убедиться, что расположения для экстренной помощи, назначенные идентификаторам сети, включают связанные географические коды. (Адреса для экстренного реагирования без географических кодов не могут быть назначены сетевым идентификаторам, необходимым для динамических расположений.)

Azure Maps используется для служб на основе местоположения. Когда вы вводите адрес для экстренного реагирования с помощью Центра администрирования Microsoft Teams, Teams проверяет Azure Maps на адрес:

- Если совпадение найдено, географические коды включаются автоматически.

- Если совпадение не найдено, вы сможете вручную создать адрес для экстренного реагирования. Для этого можно использовать функцию удаления ПИН-кода. 

> [!NOTE]
> Адреса для аварийного реагирования, которым более нескольких лет, не могут быть назначены сетевым идентификаторам. Вам потребуется повторно создать старые адреса.

Вы добавляете и назначаете адреса для чрезвычайных ситуаций в Центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения см. в [разделах Добавление расположения для экстренного реагирования для организации](add-change-remove-emergency-location-organization.md) и [Назначение расположения для экстренного реагирования для пользователя](assign-change-emergency-location-user.md).

## <a name="configure-network-settings"></a>Настройка параметров сети

Параметры сети используются для определения расположения клиента Teams, а также для динамического получения политик экстренных вызовов и расположения для экстренного реагирования. Вы можете настроить параметры сети в соответствии с тем, как ваша организация хочет, чтобы экстренные вызовы функционировали.

Параметры сети включают сайты, которые включают коллекцию подсетей, которые используются исключительно для динамического назначения политик пользователям. Например, политика экстренных вызовов и политика маршрутизации экстренных вызовов могут быть назначены сайту Redmond, чтобы любой пользователь, перемещающийся из дома или из другого расположения Майкрософт, был настроен с номерами экстренных служб, маршрутизацией и службой безопасности, характерными для Редмонда.  

Доверенные IP-адреса содержат коллекцию внешних IP-адресов в Интернете корпоративной сети и используются для определения того, находится ли конечная точка пользователя в корпоративной сети. Попытка получить динамическую политику или расположение будет предпринята только в том случае, если внешний IP-адрес пользователя соответствует IP-адресу в доверенном IP-адресе.

Дополнительные сведения об IP-адресах, регионах сети, сайтах и адресах подсети см. в разделе [Параметры сети для функций облачной голосовой связи](cloud-voice-network-settings.md).

Вы настраиваете параметры сети в Центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения см. в статье [Управление топологией сети для функций облачной голосовой связи](manage-your-network-topology.md).

Обратите внимание, что для распространения и доступности некоторых изменений параметров сети (например, нового адреса, идентификатора сети и т. д.) может потребоваться некоторое время (до нескольких часов).  

> [!Note]
> Подсети также могут быть определены в LIS и могут быть связаны с расположением для экстренного реагирования.  Подсети LIS должны быть определены идентификатором сети, соответствующим диапазону IP-адресов подсети, назначенному клиентам. Например, идентификатор сети для IP-адреса или маски клиента 10.10.10.150/25 — 10.10.128. Дополнительные сведения см. в статье [Основные сведения об адресации TCP/IP и подсети](/troubleshoot/windows-client/networking/tcpip-addressing-and-subnetting).

> [!Important]
> Поиск параметров конфигурации сети не поддерживается для развертываний облачной прокси-службы, которые изменяют исходные IP-адреса из клиентов Teams.



**Для пользователей плана звонков, оператора Connect и Мобильного телефона Teams:**

- Если требуется динамическая настройка уведомлений службы безопасности, необходимо настроить как доверенные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только доверенные IP-адреса; настройка параметров сети не требуется.

- Если ни один из них не требуется, настройка параметров сети не требуется. 

**Для пользователей с прямой маршрутизацией:**

- Если требуется динамическое включение экстренных вызовов или динамическая настройка уведомлений службы безопасности, необходимо настроить как доверенные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только доверенные IP-адреса; настройка параметров сети не требуется.

- Если ни один из них не требуется, настройка параметров сети не требуется.


## <a name="configure-location-information-service"></a>Настройка службы сведений о местоположении

Клиент Teams получает адреса для экстренного реагирования из расположений, связанных с различными сетевыми идентификаторами. 

Чтобы клиент получить расположение, необходимо заполнить LIS идентификаторами сети (подсети, WAP, коммутаторы, порты) и расположениями для аварийного реагирования. Это можно сделать в Центре администрирования Microsoft Teams или с помощью PowerShell.

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. В области навигации слева перейдите в **раздел Сети расположений** > **& расположения**.
2. Перейдите на вкладку, представляющую идентификатор сети, который требуется добавить. Например, щелкните **Подсети**, **точки доступа Wi-Fi**, **Коммутаторы** или **Порты**. Затем нажмите кнопку **Добавить**.
3. Заполните поля, добавьте место для экстренного реагирования и нажмите кнопку **Применить**.

### <a name="using-powershell"></a>С помощью PowerShell

Используйте следующие командлеты, чтобы добавить порты, коммутаторы, подсети и WAP в LIS.

- [Get](/powershell/module/skype/get-csonlinelissubnet?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelissubnet?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelissubnet?view=skype-ps) -CsOnlineLisSubnet
- [Get](/powershell/module/skype/get-csonlinelisport?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelisport?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelisport?view=skype-ps) -CsOnlineLisPort
- [Получение](/powershell/module/skype/get-csonlineliswirelessaccesspoint?view=skype-ps), [установка](/powershell/module/skype/set-csonlineliswirelessaccesspoint?view=skype-ps), [удаление](/powershell/module/skype/remove-csonlineliswirelessaccesspoint?view=skype-ps) -CsOnlineLisWirelessAccessPoint
- [Get](/powershell/module/skype/get-csonlinelisswitch?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelisswitch?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelisswitch?view=skype-ps) -CsOnlineLisSwitch

>[!Important]
>Если подсети используются как часть сетевых сайтов, они должны быть переопределены в службе сведений о расположении для отображения динамических расположений.

## <a name="configure-emergency-policies"></a>Настройка политик для экстренного реагирования

Используйте следующие политики для настройки экстренных вызовов. Управлять этими политиками можно в Центре администрирования Microsoft Teams или с помощью PowerShell.

- **Политика маршрутизации экстренных вызовов — применяется только к прямой маршрутизации**. Эта политика настраивает номера экстренных служб, маски для каждого номера и маршрут ТСОП для каждого номера. Эту политику можно назначить пользователям, сетевым сайтам или обоим. Дополнительные сведения см. в статье [Управление политиками маршрутизации экстренных вызовов для прямой маршрутизации](manage-emergency-call-routing-policies.md).  

   (Пользователи плана звонков, оператора Connect и Teams Phone Mobile автоматически включают экстренные вызовы с номерами экстренных служб из страны в зависимости от расположения использования Microsoft 365 или Office 365.)

- **Политика экстренных вызовов применяется к тарифным планам, оператору Connect, мобильному телефону Teams и прямой маршрутизации.** Эта политика настраивает интерфейс уведомлений службы безопасности при выполнении экстренного вызова. Вы можете указать, кому следует уведомлять и как они уведомляются. Например, чтобы автоматически уведомить службу безопасности вашей организации и заставить их прослушивать экстренные вызовы.  Эту политику можно назначить пользователям или сетевым сайтам или и тем, и другим. Дополнительные сведения см. [в статье Управление политиками экстренных вызовов в Teams](manage-emergency-calling-policies.md).

## <a name="enable-users-and-sites"></a>Включение пользователей и сайтов

Вы можете назначать политики маршрутизации экстренных вызовов и политики экстренных вызовов пользователям и сайтам. Помните, что политики маршрутизации экстренных вызовов применяются только к прямой маршрутизации. (Хотя эту политику можно назначить пользователю плана звонков, оператора Connect или Мобильного телефона Teams, эта политика не будет действовать.)

Политики назначаются в Центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения см. в статьях

- [Управление политиками маршрутизации экстренных вызовов для прямой маршрутизации](manage-emergency-call-routing-policies.md)
- [Управление политиками экстренных вызовов в Teams](manage-emergency-calling-policies.md)

Ниже приведены примеры PowerShell.

Чтобы включить для конкретного пользователя уведомление службы безопасности, используйте следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallingPolicy -Identity user1 -PolicyName SecurityDeskNotification
```

Чтобы назначить сайту 1 политику с именем "Политика экстренных вызовов Contoso 1", используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallingPolicy "Contoso Emergency Calling Policy 1"
```

Чтобы включить определенный пользователь прямой маршрутизации для экстренных вызовов, используйте следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallRoutingPolicy -Identity user1 -PolicyName UnitedStates
```

Чтобы назначить политику "Маршрутизация экстренных вызовов Contoso New York" сайту 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallRoutingPolicy "Contoso New York Emergency Call Routing"
```

Если вы назначили политику экстренных вызовов сетевому сайту и пользователю и если этот пользователь находится на этом сетевом сайте, политика, назначенная сетевому сайту, переопределяет политику, назначенную пользователю.

## <a name="test-emergency-calling"></a>Тестирование экстренных вызовов

Некоторые поставщики служб аварийной маршрутизации (ERSP) в США предлагают тестовый бот для экстренных вызовов.

- **Пользователи плана звонков, оператора Connect и Teams Phone Mobile в США или Канаде** могут использовать предопределенный тестовый номер экстренного реагирования 933 для проверки конфигурации экстренных вызовов. Этот номер перенаправляется боту, который затем возвращает номер телефона звонящего (идентификатор телефонной линии), адрес для экстренного реагирования или расположение, а также указывает, будет ли звонок автоматически перенаправлен в PSAP или экранирован первым.

- **Клиенты прямой маршрутизации в США** должны согласовывать с ERSP для тестовой службы.
