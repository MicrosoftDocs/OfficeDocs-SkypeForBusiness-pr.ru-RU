# <a name="plan-and-configure-dynamic-emergency-calling"></a>Планирование и настройка динамических экстренных вызовов 

Динамические экстренные вызовы для планов звонков Майкрософт, Operator Connect и прямой маршрутизации обеспечивают возможность настройки и маршрутизации экстренных вызовов и уведомления сотрудников службы безопасности в зависимости от текущего расположения клиента Teams.  

На основе топологии сети (сетевых элементов, связанных с аварийными адресами), определенных администратором клиента, клиент Teams предоставляет сведения о сетевом подключении в запросе к службе сведений о расположении (LIS). Если имеется совпадение, LIS возвращает клиенту расположение.

Клиент Teams включает данные о расположении в составе экстренных вызовов. Затем эти данные используются поставщиком экстренных служб для определения соответствующей общедоступной точки ответа (PSAP) и для маршрутизации вызова в этот PSAP, что позволяет диспетчеру PSAP получить расположение вызывающего абонента.  

Для динамических экстренных вызовов должны выполняться следующие действия:

1. Администратор сети настраивает параметры сети и LIS для создания сетевой карты или карты расположения для экстренного реагирования.

2. Во время запуска и периодически после этого или при изменении сетевого подключения клиент Teams отправляет запрос на расположение, содержащий сведения о сетевом подключении к параметрам сети и LIS.

   - Если сайт параметров сети совпадает, политики экстренных вызовов возвращаются клиенту Teams с этого сайта. (Дополнительные сведения о политиках см. в разделе ["Настройка политик экстренного реагирования"](#configure-emergency-policies)).

   - Если имеется соответствие LIS, клиенту Teams возвращается местоположение для экстренного реагирования из сетевого элемента, к котором подключен клиент Teams. Сопоставление выполняется в следующем порядке с возвратом первого сопоставленного результата:
       - WAP
       - Коммутатор Или порт Ethernet
       - Коммутатор Ethernet
       - Подсеть

3. Когда клиент Teams выполняет экстренный вызов, местоположение для экстренного реагирования передается в сеть ТСОП.

Возможность автоматической маршрутизации в соответствующую общедоступную точку ответа (PSAP) зависит от страны использования пользователя Teams.

Партнеры Microsoft Calling Plans и Operator Connect включают динамические службы маршрутизации экстренных служб для пользователей в США и Канаде.

Однако для прямой маршрутизации требуется дополнительная настройка для маршрутизации экстренных вызовов и, возможно, для подключения партнеров. Администратор должен убедиться, что шлюз ТСОП, маршрутизация которого вызывается в экстренных случаях, настроен для добавления сведений о расположении в исходящее invite (путем установки параметра PidfloSupported true в объекте шлюза ТСОП в сети). Кроме того, администратор должен настроить подключение к поставщику службы экстренной маршрутизации (**ERS) (** США и Канаде) или настроить пограничный контроллер сеансов (SBC) для приложения ELIN. Сведения о поставщиках ERS см. в разделе ["Пограничные контроллеры сеансов", сертифицированные для прямой маршрутизации](direct-routing-border-controllers.md).

## <a name="assign-emergency-addresses"></a>Назначение адресов для экстренного реагирования

Адреса для экстренного реагирования можно назначить следующим образом:

- Для пользователей плана звонков.

- Для пользователей Operator Connect&mdash;в зависимости от возможностей, назначенных номеру, когда оператор отправляет их в инвентаризацию клиента.

- К сетевым идентификаторам, необходимым для динамического получения расположения. 

Чтобы обеспечить автоматическую маршрутизацию экстренных вызовов в США, необходимо убедиться, что расположения для экстренного реагирования, назначенные сетевым идентификаторам, включают связанные географические коды. (Адреса для экстренного реагирования без географических кодов не могут быть назначены сетевым идентификаторам, необходимым для динамических расположений.)

Azure Maps используется для служб на основе расположения. При вводе адреса для экстренного реагирования с помощью Центра администрирования Microsoft Teams Teams Azure Maps адрес:

- Если совпадение найдено, географические коды включаются автоматически.

- Если совпадение не найдено, вы сможете вручную создать адрес для экстренного реагирования. Для этого можно использовать функцию удаления ПИН-кода. 

> [!NOTE]
> Адреса для экстренного реагирования, срок действия которых превышает несколько лет, не могут быть назначены сетевым идентификаторам. Вам потребуется повторно создать старые адреса.

Вы добавляете и назначаете адреса для экстренного реагирования в Центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения см. в статье ["Добавление местоположения](add-change-remove-emergency-location-organization.md) для экстренного реагирования для организации" и "Назначение местоположения для экстренного реагирования [для пользователя"](assign-change-emergency-location-user.md).

## <a name="configure-network-settings"></a>Настройка параметров сети

Параметры сети используются для определения расположения клиента Teams, а также динамического получения политик экстренных вызовов и расположения для экстренного реагирования. Вы можете настроить параметры сети в соответствии с тем, как ваша организация хочет, чтобы экстренные вызовы функции.

Параметры сети включают сайты, которые содержат коллекцию подсетей и используются исключительно для динамического назначения политик пользователям. Например, политика экстренных вызовов и политика маршрутизации экстренных вызовов могут быть назначены сайту Redmond, чтобы любой пользователь, перемещающийся из дома или другого расположения Майкрософт, был настроен с помощью номеров экстренных служб, маршрутизации и службы безопасности, относясь к Redmond.  

Доверенные IP-адреса содержат коллекцию внешних IP-адресов в Интернете корпоративной сети и используются для определения того, находится ли конечная точка пользователя в корпоративной сети. Попытка получить динамическую политику или расположение будет предпринята только в том случае, если внешний IP-адрес пользователя соответствует IP-адресу в доверенном IP-адресе.

Дополнительные сведения об IP-адресах, сетевых регионах, сайтах и адресах подсети см. в разделе "Параметры сети" для функций [облачной голосовой связи](cloud-voice-network-settings.md).

Параметры сети настраиваются в Центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения см. в статье ["Управление топологией сети для облачных голосовых функций"](manage-your-network-topology.md).

Обратите внимание, что некоторые изменения параметров сети (например, нового адреса, сетевого идентификатора и т. д.) могут распространяться и быть доступны клиентам Teams через некоторое время (до нескольких часов).  

> [!Note]
> Подсети также можно определить в LIS и связать с аварийным расположением.  Подсети LIS должны быть определены с помощью сетевого идентификатора, соответствующего диапазону IP-адресов подсети, назначенного клиентам. Например, сетевой идентификатор ip/mask клиента 10.10.10.150/25 имеет значение 10.10.10.128. Дополнительные сведения см. [в статье "Основные сведения о адресировании TCP/IP и подсети"](/troubleshoot/windows-client/networking/tcpip-addressing-and-subnetting).

> [!Important]
> Поиск параметров конфигурации сети не поддерживается при развертывании облачной службы прокси-сервера, которые изменяют исходные IP-адреса клиентов Teams.



**Для пользователей тарифного плана и оператора Connect:**

- Если требуется динамическая настройка уведомлений службы безопасности, необходимо настроить как доверенные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только доверенные IP-адреса. Настройка параметров сети не требуется.

- Если ни одно из них не требуется, настройка параметров сети не требуется. 

**Для пользователей прямой маршрутизации:**

- Если требуется динамическое включение экстренных вызовов или динамическая настройка уведомлений службы безопасности, необходимо настроить как надежные IP-адреса, так и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только доверенные IP-адреса. Настройка параметров сети не требуется.

- Если ни одно из них не требуется, настройка параметров сети не требуется.


## <a name="configure-location-information-service"></a>Настройка службы сведений о расположении

Клиент Teams получает адреса для экстренного реагирования из расположений, связанных с разными сетевыми идентификаторами. 

Чтобы клиент получить расположение, необходимо заполнить LIS сетевыми идентификаторами (подсетями, WAP, коммутаторами, портами) и расположениями для экстренного реагирования. Это можно сделать в Центре администрирования Microsoft Teams или с помощью PowerShell.

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. В области навигации слева перейдите в раздел **"Сети** >  расположений **& расположения.**
2. Щелкните вкладку, представляющую сетевой идентификатор, который вы хотите добавить. Например, щелкните **"Подсети**", "Точки доступа **Wi-Fi**", **"Коммутаторы"** или " **Порты"**. Затем нажмите **кнопку "Добавить"**.
3. Заполните поля, добавьте местоположение для экстренного реагирования и нажмите кнопку "Применить **"**.

### <a name="using-powershell"></a>С помощью PowerShell

Используйте следующие командлеты для добавления портов, коммутаторов, подсетей и ВИРТУАЛЬНЫХ IP-адресов в LIS.

- [Get](/powershell/module/skype/get-csonlinelissubnet?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelissubnet?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelissubnet?view=skype-ps) -CsOnlineLisSubnet
- [Get](/powershell/module/skype/get-csonlinelisport?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelisport?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelisport?view=skype-ps) -CsOnlineLisPort
- [Get](/powershell/module/skype/get-csonlineliswirelessaccesspoint?view=skype-ps), [Set](/powershell/module/skype/set-csonlineliswirelessaccesspoint?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlineliswirelessaccesspoint?view=skype-ps) -CsOnlineLisWirelessAccessPoint
- [Get](/powershell/module/skype/get-csonlinelisswitch?view=skype-ps), [Set](/powershell/module/skype/set-csonlinelisswitch?view=skype-ps), [Remove](/powershell/module/skype/remove-csonlinelisswitch?view=skype-ps) -CsOnlineLisSwitch

>[!Important]
>Если подсети используются как часть сетевых сайтов, их необходимо переопределить в службе сведений о расположении для отображения динамических расположений.

## <a name="configure-emergency-policies"></a>Настройка политик экстренного реагирования

Используйте следующие политики для настройки экстренных вызовов. Вы можете управлять этими политиками в Центре администрирования Microsoft Teams или с помощью PowerShell.

- **Политика маршрутизации экстренных вызовов — применяется только к прямой маршрутизации**. Эта политика настраивает номера экстренных служб, маски на число при необходимости и маршрут ТСОП на номер. Эту политику можно назначить пользователям, сетевым сайтам или обоим. Дополнительные сведения см. в статье ["Управление политиками маршрутизации экстренных вызовов для прямой маршрутизации"](manage-emergency-call-routing-policies.md).  

   (Для пользователей тарифного плана и operator Connect автоматически включены экстренные вызовы с номерами экстренных служб из страны в зависимости от расположения использования Microsoft 365 или Office 365).)

- **Политика экстренных вызовов— применяется к тарифным планам, operator Connect и прямой маршрутизации.** Эта политика настраивает интерфейс уведомлений службы безопасности при выполнении экстренных вызовов. Вы можете указать, кому и как они уведомляются. Например, чтобы автоматически уведомлять службу безопасности вашей организации и прослушивать экстренные вызовы.  Эту политику можно назначить пользователям или сетевым сайтам или обоим. Дополнительные сведения см. в статье ["Управление политиками экстренных вызовов" в Teams](manage-emergency-calling-policies.md).

## <a name="enable-users-and-sites"></a>Включение пользователей и сайтов

Вы можете назначить пользователям и сайтам политики маршрутизации экстренных вызовов и экстренных вызовов. Помните, что политики маршрутизации экстренных вызовов применяются только к прямой маршрутизации. (Хотя эту политику можно назначить пользователю плана звонков или оператора Connect, она не будет действовать.)

Политики назначаются в Центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения см. в статьях

- [Управление политиками маршрутизации экстренных вызовов для прямой маршрутизации](manage-emergency-call-routing-policies.md)
- [Управление политиками экстренных вызовов в Teams](manage-emergency-calling-policies.md)

Ниже приведены примеры PowerShell.

Чтобы включить определенного пользователя для уведомлений службы безопасности, используйте следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallingPolicy -Identity user1 -PolicyName SecurityDeskNotification
```

Чтобы назначить политику Contoso Emergency Calling Policy 1 сайту 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallingPolicy "Contoso Emergency Calling Policy 1"
```

Чтобы включить для конкретного пользователя прямой маршрутизации экстренные вызовы, используйте следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallRoutingPolicy -Identity user1 -PolicyName UnitedStates
```

Чтобы назначить политику Contoso New York Emergency Call Routing на сайте 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallRoutingPolicy "Contoso New York Emergency Call Routing"
```

Если политика экстренных вызовов назначена сетевому сайту и пользователю, а также если этот пользователь находится на этом сетевом сайте, политика, назначенная сетевому сайту, переопределяет политику, назначенную пользователю.

## <a name="test-emergency-calling"></a>Тестирование экстренных вызовов

Некоторые поставщики экстренных служб маршрутизации (ERSP) в США предлагают тестовый бот для экстренных вызовов.

- **Пользователи плана звонков и operator Connect** в США или Канаде могут использовать предопределенный тестовый экстренный номер 933 для проверки конфигурации экстренных вызовов. Этот номер направляется боту, который затем возвращает номер телефона вызывающего абонента (идентификатор вызывающей строки), адрес для экстренного реагирования или расположение, а также указывает, будет ли вызов автоматически перенаправляться в PSAP или отображаться первым.

- **Клиенты прямой маршрутизации в США** должны координировать свои действия с ERSP для тестовой службы.
