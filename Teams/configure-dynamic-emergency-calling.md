# <a name="plan-and-configure-dynamic-emergency-calling"></a>Планирование и настройка динамических экстренных вызовов 

Динамическое обращение для экстренных вызовов в планах вызовов Microsoft и прямую маршрутизацию телефонной системы обеспечивает возможность настраивать и перенаправлять экстренные вызовы и уведомлять сотрудников системы безопасности в соответствии с текущим расположением клиента Teams.  

В соответствии с топологией сети, определяемой администратором клиента, клиент Teams предоставляет сведения о сетевом подключении в запросе к службе сведений о расположении (LIS).  Если найдено соответствие, LIS возвращает расположение для клиента. Данные о расположении передаются обратно клиенту.  

Клиент Teams включает данные о местоположении в ходе вызова экстренной помощи. Эти данные затем используются поставщиком услуг экстренной помощи, чтобы определить соответствующую точку ответа на общедоступную безопасность (ПСАП) и перенаправить вызов на этот ПСАП, что позволяет диспетчеру ПСАП получить расположение вызывающего абонента.  

Для динамической службы экстренной помощи должны произойти следующие события:

1. Сетевой администратор настроил сетевые параметры и местонахождение LIS для создания карты расположения сети и экстренной помощи.

   Для прямой маршрутизации требуется дополнительная настройка маршрутизации для экстренных вызовов и, возможно, для подключения партнеров. Администратор должен настроить подключение к поставщику службы экстренной маршрутизации (ЕРС) (США) **или** настроить контроллер границ сеанса (SBC) для приложения с идентификационным номером для экстренной помощи (Елин).

2. При запуске, периодически и при изменении сетевого соединения клиент Teams отправляет запрос на расположение, содержащий сведения о сетевом подключении, в параметры сети и LIS.

   - Если веб-сайт соответствует сетевым параметрам — в клиенте Teams будут возвращаться политики вызова экстренной помощи. (Дополнительные сведения о политиках см. в разделе [Настройка политик экстренного](#configure-emergency-policies)реагирования).

   - Если обнаружено совпадение LIS — место в экстренном случае, в которое подключен клиент Teams, будет возвращено клиенту Teams из сетевого элемента.

3. Когда клиент Teams осуществляет вызов экстренной помощи, расположение для экстренного реагирования передается в сеть PSTN.

   Для прямой маршрутизации администратор должен настроить SBC для отправки экстренных вызовов поставщику ЕРС или настроить приложение SBC Елин.

## <a name="assign-emergency-addresses"></a>Назначение адресов для экстренного реагирования

Вы можете назначать адреса для экстренного реагирования обоим тарифам плана звонков и сетевым идентификаторам, которые необходимы для динамического получения местоположения. (Поддерживаются подсети и ТД AP; ожидание поддержки коммутатора/порта Ethernet.)

Чтобы обеспечить автоматическую маршрутизацию экстренных вызовов в США, необходимо убедиться в том, что местоположения для экстренного реагирования, назначенные сетевым идентификаторам, включают в себя соответствующие географических коды. (Адреса для экстренного реагирования без географических кодов нельзя назначить сетевым идентификаторам, которые необходимы для динамических местоположений.)

Карты Azure используются для служб, основанных на расположении.  Когда вы вводите адрес для экстренного реагирования с помощью центра администрирования Microsoft Teams, Teams проверяет карты Azure для этого адреса.

- Если совпадение найдено, коды Geo включаются автоматически.

- Если совпадение не найдено, вы сможете вручную создать адрес для экстренного реагирования. Для этого можно использовать функцию перетаскивания PIN-кода.   

Это означает, что если расположение для экстренного реагирования, созданное для назначения пользователям плана звонков, предназначено для динамического местоположения, необходимо повторно создать тот же адрес для включения в него географических кодов. Для различения двух местоположений следует добавить другое описание. Новое расположение для экстренного реагирования может назначаться пользователям, у которых есть прежнее расположение. При полной миграции прежнее расположение может быть удалено.

Дополнительные сведения о настройке адресов для экстренного реагирования можно найти в разделе [Добавление расположения для экстренных случаев для Организации](add-change-remove-emergency-location-organization.md) и [Назначение места для экстренного реагирования для пользователя](assign-change-emergency-location-user.md).

## <a name="configure-network-settings"></a>Настройка параметров сети

Параметры сети используются для определения расположения клиента Teams и динамического получения политик вызова экстренной помощи и расположения для экстренного реагирования. Вы можете настроить параметры сети в зависимости от того, как ваша организация предхочет вызвать экстренную функцию.

Сетевые параметры включают сайты, содержащие набор подсетей — они используются исключительно для назначения динамической политики пользователям.  Например, политика Теамсемерженцикаллинг и Теамсемерженцикаллраутинг могут быть назначены "сайту Redmond", чтобы любой пользователь, который перемещается из дома или другого местоположения Майкрософт, настроил номера экстренной службы, маршрутизацию и службу безопасности. конкретно для Редмонде.  

>[!Note]
>Подсеть также может быть определена в LIS и может быть связана с местом для экстренной помощи.  

Учитывайте следующие определения:

- Доверенный IP-адрес: содержит коллекцию внешних адресов Интернета в корпоративной сети и используется для определения того, входит ли конечная точка пользователя в корпоративную сеть. Попытка получить динамическую политику или расположение будет выполнена только в том случае, если внешний IP-адрес пользователя соответствует IP в доверенном IP-адреса. Соответствие можно получить по IP-адресам IPv4 или IPv6 и зависит от формата пакета IP, отправляемого в параметры сети.  (Если у общедоступного IP-адреса есть и IPv4, и IPv6, вам нужно добавить как доверенные IP-адреса.)

- Область сети содержит коллекцию сетевых сайтов. 

- Сетевой сайт — это расположение, в котором Организация имеет физическую стоимость, например Office, набор зданий или кампус. Эти сайты определяются как набор IP-подсетей.

- Сетевая подсеть должна быть связана с определенным сетевым сайтом. Расположение клиента определяется на основе сетевой подсети и связанного сайта сети.  

Дополнительные сведения можно найти в разделе [Параметры сети для функций голосовой связи в облаке](cloud-voice-network-settings.md) и [управлять топологией сети для функций голосовой связи в облаке](manage-your-network-topology.md).

Обратите внимание, что для некоторых изменений сетевых параметров (например, нового адреса, идентификатора сети и т. д.) может потребоваться некоторое время (в течение нескольких часов) для распространения и обеспечения доступности для клиентов Teams.  

**Для пользователей плана звонков:**

- Если требуется динамическая настройка уведомления службы безопасности, необходимо настроить оба доверенных IP-адреса и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только доверенные IP-адреса. 

- Если ни один из них не требуется, Настройка параметров сети не требуется. 

**Для прямого направления пользователей выполните указанные ниже действия.**

- Если требуется динамическое включение вызова экстренной или динамической конфигурации уведомления службы безопасности, необходимо настроить оба доверенных IP-адреса и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только доверенные IP-адреса.

- Если ни один из них не требуется, Настройка параметров сети не требуется.

## <a name="configure-emergency-policies"></a>Настройка политик экстренной помощи

Для настройки вызова экстренной помощи можно использовать следующие политики:

- **Теамсемерженцикаллраутингполици** — относится только к прямой маршрутизации. Этот параметр позволяет настроить номера для экстренного реагирования, маску для каждого номера, а также маршрут PSTN на номер.  Вы можете назначить эту политику пользователям, сетевым сайтам или обоим. (Планы звонков. клиенты Teams автоматически включаются для вызова экстренной помощи с номерами экстренного реагирования на основе места использования Office 365.)  Вы управляете этой политикой с помощью командлетов New-, Set-и Grant-Кстеамсемерженцикаллраутинг. 

- **Теамсемерженцикаллингполици** — относится к плану звонков и прямой маршрутизации. Этот параметр политики позволяет настроить уведомления службы безопасности при проведении экстренного звонка. Вы можете настроить пользователей, которые должны уведомлять и как они уведомлены. Например, чтобы автоматически уведомлять службу безопасности своей организации и послушать ее на экстренных звонках.  Эта политика может быть назначена пользователям либо сетевым сайтам или обоим. Вы управляете этой политикой с помощью командлетов New-, Set-и Grant-Кстеамсемерженцикаллингполици. 

Дополнительные сведения можно найти [в разделе Управление политиками вызова экстренной помощи в Teams](manage-emergency-calling-policies.md) и [Управление политиками маршрутизации вызовов экстренной помощи в прямом маршруте](manage-emergency-call-routing-policies.md).


## <a name="enable-users-and-sites"></a>Включение пользователей и сайтов

Вы можете назначать политики **теамсемерженцикаллинг** и **теамсемерженцикаллраутинг** для пользователей и сайтов.  

Политика Теамсемерженцикаллраутинг применяется только для прямой маршрутизации. (Несмотря на то, что эту политику можно назначить пользователю абонентского плана, политика не будет оказывать никакого воздействия.)

Например, чтобы включить определенного пользователя в уведомление на службу безопасности, используйте следующую команду:

```
Grant-CsTeamsEmergencyCallingPolicy -Identity user1 -PolicyName SecurityDeskNotification
```

Чтобы назначить политику "Contoso срочного вызова 1" для сайта 1, используйте следующую команду:

```
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallingPolicy "Contoso Emergency Calling Policy 1"
```

Чтобы включить для вызова экстренной помощи определенного пользователя, выполните следующую команду:

```
Grant-CsTeamsEmergencyCallRoutingPolicy -Identity user1 -PolicyName UnitedStates
```

Чтобы назначить политику "Маршрутизация экстренных вызовов в Нью-Йорке" для сайта 1, выполните следующую команду:

```
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallRoutingPolicy "Contoso New York Emergency Call Routing"
```

Если вы назначили для сайта сети и пользователя политику вызова экстренной помощи, а если этот пользователь находится на сайте сети, политика, назначенная сетевому сайту, переопределяет политику, назначенную пользователю.


## <a name="test-emergency-calling"></a>Проверка вызова экстренной помощи

Некоторые поставщики услуг для экстренной маршрутизации (Ерспс) в США предлагают Bot-каналы для проверки звонков.

- В **плане звонков пользователи в США** могут использовать предварительно определенный тестовый 933 номер для экстренного реагирования, чтобы проверить конфигурацию для вызова экстренной помощи. Этот номер перенаправляется на Bot, который затем отправляет номер телефона вызывающего абонента (идентификатор строки вызова), адрес для экстренного реагирования или место, а также будет ли этот звонок автоматически перенаправлен на ПСАП или на экране.

- **Прямые маршруты клиентов в США** должны координироваться в соответствии с их ерсп для тестовых служб.
