# <a name="plan-and-configure-dynamic-emergency-calling"></a>Планирование и настройка динамических экстренных вызовов 

Динамическое обращение для экстренных вызовов в планах вызовов Microsoft и прямую маршрутизацию телефонной системы обеспечивает возможность настраивать и перенаправлять экстренные вызовы и уведомлять сотрудников системы безопасности в соответствии с текущим расположением клиента Teams.  

В соответствии с топологией сети, определяемой администратором клиента, клиент Teams предоставляет сведения о сетевом подключении в запросе к службе сведений о расположении (LIS). Если обнаружено соответствие, LIS возвращает расположение для клиента. Данные о расположении передаются обратно клиенту.  

Клиент Teams включает данные о местоположении в ходе вызова экстренной помощи. Эти данные затем используются поставщиком услуг экстренной помощи, чтобы определить соответствующую точку ответа на общедоступную безопасность (PSAP) и перенаправить вызов на этот PSAP, что позволяет диспетчеру PSAP получить расположение вызывающего абонента.  

Для динамической службы экстренной помощи должны произойти следующие события:

1. Сетевой администратор настроил сетевые параметры и местонахождение LIS для создания карты расположения сети и экстренной помощи.

   Для прямой маршрутизации требуется дополнительная настройка маршрутизации для экстренных вызовов и, возможно, для подключения партнеров. Администратор должен настроить подключение к поставщику службы экстренной маршрутизации (ERS) (США) **или** настроить контроллер границ сеанса (SBC) для приложения с идентификационным номером для экстренной помощи (Elin).

2. При запуске, периодически и при изменении сетевого соединения клиент Teams отправляет запрос на расположение, содержащий сведения о сетевом подключении, в параметры сети и LIS.

   - Если на сайте есть параметры сети – в клиенте Teams будут возвращаться политики вызова экстренной помощи. (Дополнительные сведения о политиках см. в разделе [Настройка политик экстренного](#configure-emergency-policies)реагирования).

   - Если обнаружено совпадение LIS — адрес для экстренной помощи из сетевого элемента, к которому подключается клиент Teams, возвращается клиенту Teams.

3. Когда клиент Teams осуществляет вызов экстренной помощи, расположение для экстренного реагирования передается в сеть PSTN.

   Для прямой маршрутизации администратор должен настроить SBC для отправки экстренных вызовов поставщику ERS или настроить приложение SBC ELIN.

## <a name="assign-emergency-addresses"></a>Назначение адресов для экстренного реагирования

Вы можете назначать адреса для экстренного реагирования обоим тарифам плана звонков и сетевым идентификаторам, которые необходимы для динамического получения местоположения. (Поддерживаются подсети и ТД AP; ожидание поддержки коммутатора/порта Ethernet.)

Чтобы обеспечить автоматическую маршрутизацию экстренных вызовов в США, необходимо убедиться в том, что местоположения для экстренного реагирования, назначенные сетевым идентификаторам, включают в себя соответствующие географических коды. (Адреса для экстренного реагирования без географических кодов нельзя назначить сетевым идентификаторам, которые необходимы для динамических местоположений.)

Карты Azure используются для служб, основанных на расположении.  Когда вы вводите адрес для экстренного реагирования с помощью центра администрирования Microsoft Teams, Teams проверяет карты Azure для этого адреса.

- Если совпадение найдено, коды Geo включаются автоматически.

- Если совпадение не найдено, вы сможете вручную создать адрес для экстренного реагирования. Для этого можно использовать функцию перетаскивания PIN-кода. 

Это означает, что если расположение для экстренного реагирования, созданное для назначения пользователям плана звонков, предназначено для динамического местоположения, необходимо повторно создать тот же адрес для включения в него географических кодов. Для различения двух местоположений следует добавить другое описание. Новое расположение для экстренного реагирования может назначаться пользователям, у которых есть прежнее расположение. При полной миграции прежнее расположение может быть удалено.

Вы добавляете и назначаете адреса для экстренного реагирования в центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения можно найти в разделе [Добавление расположения для экстренного реагирования для вашей организации](add-change-remove-emergency-location-organization.md) и [Назначение пользователю места для экстренного](assign-change-emergency-location-user.md)реагирования.

## <a name="configure-network-settings"></a>Настройка параметров сети

Параметры сети используются для определения расположения клиента Teams и динамического получения политик вызова экстренной помощи и расположения для экстренного реагирования. Вы можете настроить параметры сети в зависимости от того, как ваша организация предхочет вызвать экстренную функцию.

Сетевые параметры включают сайты, содержащие набор подсетей, которые используются исключительно для назначения динамической политики пользователям. Например, политика вызова экстренной помощи и политика маршрутизации для экстренной помощи могут быть назначены сайту Redmond, так что для любого пользователя, который перемещается из дома или другого местоположения Майкрософт, настраиваются номера экстренной службы, маршрутизация и служба безопасности, специфичные для Redmond.  

>[!Note]
>Подсеть также может быть определена в LIS и может быть связана с местом для экстренной помощи.  

Учитывайте следующие определения. Дополнительные сведения можно найти в разделе [Параметры сети для функций голосовой связи в облаке](cloud-voice-network-settings.md).

- Доверенные IP-адреса содержат коллекцию внешних IP-адресов Интернета в корпоративной сети и используются для определения того, входит ли конечная точка пользователя в корпоративную сеть. Попытка получить динамическую политику или расположение будет выполнена только в том случае, если внешний IP-адрес пользователя соответствует IP-адресу в доверенном IP-адресе. Соответствие можно получить по IP-адресам IPv4 или IPv6 и зависит от формата пакета IP, отправляемого в параметры сети.  (Если у общедоступного IP-адреса есть и IPv4, и IPv6, вам нужно добавить как доверенные IP-адреса.)

- Область сети содержит коллекцию сетевых сайтов. 

- Сетевой сайт — это расположение, в котором Организация имеет физическую стоимость, например Office, набор зданий или кампус. Эти сайты определяются как набор IP-подсетей.

- Сетевая подсеть должна быть связана с определенным сетевым сайтом. Расположение клиента определяется на основе сетевой подсети и связанного сайта сети.  

Вы настраиваете параметры сети в центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения можно найти в статье [Управление топологией сети для функций голосовой связи в облаке](manage-your-network-topology.md).

Обратите внимание, что для некоторых изменений сетевых параметров (например, нового адреса, идентификатора сети и т. д.) может потребоваться некоторое время (в течение нескольких часов) для распространения и обеспечения доступности для клиентов Teams.  

**Для пользователей плана звонков:**

- Если требуется динамическая настройка уведомления службы безопасности, необходимо настроить оба доверенных IP-адреса и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только доверенные IP-адреса.

- Если ни один из них не требуется, Настройка сетевых параметров не требуется. 

**Для прямого направления пользователей выполните указанные ниже действия.**

- Если требуется динамическое включение вызова экстренной или динамической конфигурации уведомления службы безопасности, необходимо настроить оба доверенных IP-адреса и сетевые сайты.

- Если требуются только динамические расположения, необходимо настроить только доверенные IP-адреса.

- Если ни один из них не требуется, Настройка сетевых параметров не требуется.


## <a name="configure-location-information-service"></a>Настройка службы сведений о расположении

Клиент Teams получает адреса для экстренного реагирования из местоположений, связанных с разными сетевыми идентификаторами. Поддерживаются как подсети, так и точки доступа к беспроводной сети (WAPs). (Ожидается поддержка коммутатора или порта Ethernet.)

Чтобы клиент получил расположение, необходимо заполнить его сетевыми идентификаторами (подсетями, WAPs, коммутаторами, портами) и местами для экстренного реагирования. Это можно сделать в центре администрирования Microsoft Teams или с помощью PowerShell.

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. На панели навигации слева перейдите в раздел **расположения**,  >  **& расположением**.
2. Откройте вкладку, представляющую идентификатор сети, который вы хотите добавить. Например, выберите **подсети**, **точки доступа Wi-Fi**, **переключатели**или **порты**. Затем нажмите кнопку **Добавить**.
3. Заполните поля, добавьте место для экстренного реагирования и нажмите кнопку **Применить**.

### <a name="using-powershell"></a>Использование PowerShell

Используйте следующие командлеты для добавления портов, переключателей, подсетей и WAPs в окно LIS.

- [Get](https://docs.microsoft.com/powershell/module/skype/get-csonlinelissubnet?view=skype-ps), [Set](https://docs.microsoft.com/powershell/module/skype/set-csonlinelissubnet?view=skype-ps), [Remove](https://docs.microsoft.com/powershell/module/skype/remove-csonlinelissubnet?view=skype-ps) -CsOnlineLisSubnet
- [Get](https://docs.microsoft.com/powershell/module/skype/get-csonlinelisport?view=skype-ps), [Set](https://docs.microsoft.com/powershell/module/skype/set-csonlinelisport?view=skype-ps), [Remove](https://docs.microsoft.com/powershell/module/skype/remove-csonlinelisport?view=skype-ps) -CsOnlineLisPort
- [Get](https://docs.microsoft.com/powershell/module/skype/get-csonlineliswirelessaccesspoint?view=skype-ps), [Set](https://docs.microsoft.com/powershell/module/skype/set-csonlineliswirelessaccesspoint?view=skype-ps), [Remove](https://docs.microsoft.com/powershell/module/skype/remove-csonlineliswirelessaccesspoint?view=skype-ps) -CsOnlineLisWirelessAccessPoint
- [Get](https://docs.microsoft.com/powershell/module/skype/get-csonlinelisswitch?view=skype-ps), [Set](https://docs.microsoft.com/powershell/module/skype/set-csonlinelisswitch?view=skype-ps), [Remove](https://docs.microsoft.com/powershell/module/skype/remove-csonlinelisswitch?view=skype-ps) -CsOnlineLisSwitch

>[!Important]
>Если подсети используются как часть сетевых сайтов, они должны быть переопределены в службе сведений о расположении для отображения динамических расположений.

## <a name="configure-emergency-policies"></a>Настройка политик экстренной помощи

Используйте следующие политики для настройки вызова экстренной помощи. Вы можете управлять этими политиками в центре администрирования Microsoft Teams или с помощью PowerShell.

- **Политика маршрутизации вызова экстренной** помощи — применяется только к прямой маршрутизации. Этот параметр позволяет настроить номера для экстренного реагирования, маску для каждого номера, а также маршрут PSTN на номер.  Вы можете назначить эту политику пользователям, сетевым сайтам или обоим. (Планы звонков. клиенты Teams автоматически включаются для вызова экстренной помощи с номерами экстренного реагирования на основе места использования Office 365.)  Дополнительные сведения можно найти в статье [Управление политиками маршрутизации вызова экстренной помощи для прямой маршрутизации](manage-emergency-call-routing-policies.md).

- **Политика вызова экстренной** помощи — относится к тарифным планам и прямой маршрутизации. Этот параметр политики позволяет настроить уведомления службы безопасности при проведении экстренного звонка. Вы можете настроить пользователей, которые должны уведомлять и как они уведомлены. Например, чтобы автоматически уведомлять службу безопасности своей организации и послушать ее на экстренных звонках.  Эта политика может быть назначена пользователям либо сетевым сайтам или обоим. Дополнительные сведения можно найти [в разделе Управление политиками вызова экстренной помощи в Teams](manage-emergency-calling-policies.md).

## <a name="enable-users-and-sites"></a>Включение пользователей и сайтов

Для пользователей и сайтов можно назначать политики маршрутизации вызова экстренной помощи и политики вызова экстренной помощи. Имейте в виду, что политики маршрутизации для экстренных вызовов применяются только для прямой маршрутизации. (Несмотря на то, что эту политику можно назначить пользователю абонентского плана, политика не будет оказывать никакого воздействия.)

Политики назначаются в центре администрирования Microsoft Teams или с помощью PowerShell. Дополнительные сведения:

- [Управление политиками маршрутизации вызова экстренной помощи в прямом маршруте](manage-emergency-call-routing-policies.md)
- [Управление политиками вызова экстренной помощи в Teams](manage-emergency-calling-policies.md)

Вот несколько примеров PowerShell.

Чтобы включить определенного пользователя в уведомление на службу безопасности, используйте следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallingPolicy -Identity user1 -PolicyName SecurityDeskNotification
```

Чтобы назначить политику "Contoso срочного вызова 1" для сайта 1, используйте следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallingPolicy "Contoso Emergency Calling Policy 1"
```

Чтобы включить для вызова экстренной помощи определенного пользователя, выполните следующую команду:

```PowerShell
Grant-CsTeamsEmergencyCallRoutingPolicy -Identity user1 -PolicyName UnitedStates
```

Чтобы назначить политику "Маршрутизация экстренных вызовов в Нью-Йорке" для сайта 1, выполните следующую команду:

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallRoutingPolicy "Contoso New York Emergency Call Routing"
```

Если вы назначили для сайта сети и пользователя политику вызова экстренной помощи, а если этот пользователь находится на сайте сети, политика, назначенная сетевому сайту, переопределяет политику, назначенную пользователю.

## <a name="test-emergency-calling"></a>Проверка вызова экстренной помощи

Некоторые поставщики услуг для экстренной маршрутизации (ERSPs) в США предлагают Bot-каналы для проверки звонков.

- В **плане звонков пользователи в США** могут использовать предварительно определенный тестовый 933 номер для экстренного реагирования, чтобы проверить конфигурацию для вызова экстренной помощи. Этот номер перенаправляется на Bot, который затем отправляет номер телефона вызывающего абонента (идентификатор строки вызова), адрес для экстренного реагирования или место, а также будет ли этот звонок автоматически перенаправлен на PSAP или на экране.

- **Прямые маршруты клиентов в США** должны координироваться в соответствии с их ERSP для тестовых служб.
