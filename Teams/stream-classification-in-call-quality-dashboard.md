## <a name="unclassified-streams"></a><span data-ttu-id="2e708-238">Неклассифицированные потоки</span><span class="sxs-lookup"><span data-stu-id="2e708-238">Unclassified Streams</span></span>

<span data-ttu-id="2e708-239">В CQD поток помечется как неклассифицированный при сбойе подключения к интерактивной установке подключения к интернету (ICE) или при отсутствовке всех метрик, необходимых для вычисления классификации потока. </span><span class="sxs-lookup"><span data-stu-id="2e708-239">In CQD, a stream is marked _Unclassified_ when Interactive Connectivity Establishment (ICE) connectivity fails or when all the metrics required to compute the stream classification are not reported.</span></span>

<span data-ttu-id="2e708-240">Чтобы проверить сбои подключения ICE, изучите размеры «First Connectivity Ice» и «Second Connectivity Ice» для значения «FAILED».</span><span class="sxs-lookup"><span data-stu-id="2e708-240">To check for ICE connectivity failures, examine the dimensions "First Connectivity Ice" and "Second Connectivity Ice" for a "FAILED" value.</span></span> <span data-ttu-id="2e708-241">Если значение указывает на сбой, поток помечется как _неклассифицированный._</span><span class="sxs-lookup"><span data-stu-id="2e708-241">If either value indicates a failure, the stream is marked as _Unclassified_.</span></span>

<span data-ttu-id="2e708-242">Если успешное подключение ICE  для неклассифицированного потока,  скорее всего, он считается неклассифицированным, так как основные показатели потока не илифицированы.</span><span class="sxs-lookup"><span data-stu-id="2e708-242">If ICE connectivity succeeded for an _Unclassified_ stream, the stream is likely considered _Unclassified_ because key stream metrics were not reported.</span></span> <span data-ttu-id="2e708-243">Есть несколько причин, по которым эти показатели могут не сообщаться:</span><span class="sxs-lookup"><span data-stu-id="2e708-243">There are a few reasons these metrics may not be reported:</span></span>
