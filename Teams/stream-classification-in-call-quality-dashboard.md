<span data-ttu-id="bf43b-212">В CQD поток помечается как _неклассифицированный_ при сбое подключения к интерактивному подключению (ICE), а также при отсутствии отчета о всех метриках, необходимых для вычисления классификации потока.</span><span class="sxs-lookup"><span data-stu-id="bf43b-212">In CQD, a stream is marked _Unclassified_ when Interactive Connectivity Establishment (ICE) connectivity fails or when all the metrics required to compute the stream classification are not reported.</span></span>

<span data-ttu-id="bf43b-213">Чтобы проверить сбои подключения ICE, изучите размеры «First Connectivity Ice» и «Second Connectivity Ice» для значения «FAILED».</span><span class="sxs-lookup"><span data-stu-id="bf43b-213">To check for ICE connectivity failures, examine the dimensions "First Connectivity Ice" and "Second Connectivity Ice" for a "FAILED" value.</span></span> <span data-ttu-id="bf43b-214">Если какое-либо из значений указывает на сбой, поток помечается как _неклассифицированный_.</span><span class="sxs-lookup"><span data-stu-id="bf43b-214">If either value indicates a failure, the stream is marked as _Unclassified_.</span></span>

<span data-ttu-id="bf43b-215">Если подключение по протоколу ICE успешно установлено для _неклассифицированного_ потока, поток скорее считается _неклассифицированным_ , так как метрики потока ключей не были включены в отчеты.</span><span class="sxs-lookup"><span data-stu-id="bf43b-215">If ICE connectivity succeeded for an _Unclassified_ stream, the stream is likely considered _Unclassified_ because key stream metrics were not reported.</span></span> <span data-ttu-id="bf43b-216">Есть несколько причин, по которым эти показатели могут не сообщаться:</span><span class="sxs-lookup"><span data-stu-id="bf43b-216">There are a few reasons these metrics may not be reported:</span></span>

- <span data-ttu-id="bf43b-217">**Отчеты QoE не получены** — метрики, используемые для классификации, включаются в отчет QoE, отправленный по завершении звонка.</span><span class="sxs-lookup"><span data-stu-id="bf43b-217">**QoE reports were not received** — The metrics used for classification are reported in a QoE report sent at the end of a call.</span></span> <span data-ttu-id="bf43b-218">Если этот отчет не создается (например, так как некоторые конечные точки сторонних разработчиков не отправляют QoE) или не могут быть отправлены (например, из-за сбоя сети), CQD не может классифицировать поток.</span><span class="sxs-lookup"><span data-stu-id="bf43b-218">If this report is not produced (for example, because some third-party endpoints may not send QoE) or could not be sent (for example, because of a network outage), CQD is unable to classify the stream.</span></span>

> [!TIP]
> <span data-ttu-id="bf43b-219">Измерение «QoE Record Available» может использоваться для определения того, был ли получен отчет QoE для потока.</span><span class="sxs-lookup"><span data-stu-id="bf43b-219">The "QoE Record Available" dimension can be used to determine whether a QoE report was received for a stream.</span></span> <span data-ttu-id="bf43b-220">Обратите внимание, что это измерение будет иметь значение «True», если отчет QoE был получен от любой конечной точки.</span><span class="sxs-lookup"><span data-stu-id="bf43b-220">Note that this dimension will have a value of "True" if a QoE report was received from either endpoint.</span></span> <span data-ttu-id="bf43b-221">Отчет о QoE с обеих конечных точек требуется для наиболее точной отчетности показателей.</span><span class="sxs-lookup"><span data-stu-id="bf43b-221">A QoE report from both endpoints is required for the most accurate reporting of metrics.</span></span>

- <span data-ttu-id="bf43b-222">**Короткие звонки** — в коротких звонках могут быть недостаточно операций мультимедиа для вычисления метрик потока ключей.</span><span class="sxs-lookup"><span data-stu-id="bf43b-222">**Short calls** — Short calls may not have enough media activity to compute key stream metrics.</span></span> <span data-ttu-id="bf43b-223">Без этих показателей CQD не может классифицировать поток.</span><span class="sxs-lookup"><span data-stu-id="bf43b-223">Without these metrics, CQD is unable to classify the stream.</span></span>

> [!TIP]
> <span data-ttu-id="bf43b-224">Для определения продолжительности потока можно использовать размеры «Длительность (секунды)», «Длительность (минуты)», «Длительность 5 секунд или меньше» и «Длительность 60 секунд или более».</span><span class="sxs-lookup"><span data-stu-id="bf43b-224">The dimensions "Duration (Seconds)", "Duration (Minutes)", "Duration 5 seconds or less", and "Duration 60 seconds or more" can be used to determine the duration of a stream.</span></span> <span data-ttu-id="bf43b-225">Измерение «Средняя продолжительность звонка» также может использоваться для вычисления средней продолжительности для набора потоков.</span><span class="sxs-lookup"><span data-stu-id="bf43b-225">The measurement "Avg Call Duration" can also be used to compute the average duration for a set of streams.</span></span>

- <span data-ttu-id="bf43b-226">**Низкое использование пакетов** (например, "короткий Звонок") для вычисления метрик потоков ключей требуется достаточное использование пакетов.</span><span class="sxs-lookup"><span data-stu-id="bf43b-226">**Low packet utilization** — Like the "short call" scenario, sufficient packet utilization is required for computation of key stream metrics.</span></span> <span data-ttu-id="bf43b-227">Без этих показателей CQD не может классифицировать поток.</span><span class="sxs-lookup"><span data-stu-id="bf43b-227">Without these metrics, CQD is unable to classify the stream.</span></span>
  - <span data-ttu-id="bf43b-228">Если участник присоединяется к собранию для прослушивания выступающего, но не говорит (микрофон для большинства звонков), происходит обычное использование слабого сценария.</span><span class="sxs-lookup"><span data-stu-id="bf43b-228">A common low packet utilization scenario occurs when an attendee joins a meeting to listen to the presenter, but never speaks (the microphone is muted for most of the call).</span></span> <span data-ttu-id="bf43b-229">В этом случае поток звука, входящий на клиент, имеет высокое использование пакетов, в то время как поток звука, исходящий от клиента, имеет небольшое количество ресурсов.</span><span class="sxs-lookup"><span data-stu-id="bf43b-229">Here, the audio stream inbound to the client has high packet utilization while the audio stream outbound from the client has little to no packet utilization.</span></span> <span data-ttu-id="bf43b-230">Продолжительность потока может быть не более часа, но при этом загрузка пакетов с клиента на сервер мала, так как микрофон был отключен и _Неклассифицированные_ результаты потока.</span><span class="sxs-lookup"><span data-stu-id="bf43b-230">The duration of the stream may be an hour or longer but the packet utilization on the stream from the client to the server is low since the microphone was muted, and an _Unclassified_ stream results.</span></span>

> [!TIP]
> <span data-ttu-id="bf43b-231">Измерение «Утилизация пакета» и измерение «Среднее использование пакета» могут использоваться для определения активности пакета в потоке.</span><span class="sxs-lookup"><span data-stu-id="bf43b-231">The "Packet Utilization" dimension and "Avg Packet Utilization" measurement can be used to determine the packet activity of a stream.</span></span>
