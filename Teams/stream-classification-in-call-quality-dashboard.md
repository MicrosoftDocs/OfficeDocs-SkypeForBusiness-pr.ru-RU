# <a name="stream-classification-in-call-quality-dashboard-cqd"></a>Классификация потоков на панели мониторинга качества звонка (CQD)

Панель мониторинга качества звонка (ПМКЗ) для Microsoft Teams и Skype для бизнеса Online позволяет получить представление о качестве звонков, сделанных с помощью команд Microsoft и Skype для бизнес-услуг. В этом разделе представлена ​​подробная информация о классификации качества потоков мультимедиа. Дополнительные информацию о CQD и его настройках см. в этой [странице.](turning-on-and-using-call-quality-dashboard.md)

## <a name="classifier-definitions"></a>Определения классификатора

Потоки в CQD классифицируются как "Хорошее", "Плохое" или "Неклассифицировано" на основе значений доступных ключевых показателей качества.  Метрики и условия, используемые для классификации потока, показаны в последующих таблицах. Измерения CQD "Poor Due To" можно использовать для понимания того, какая метрика отвечает за классификацию _Poor._ Дополнительные сведения об этих измерениях см. в теме Измерения и меры, доступные на [панели мониторинга качества звонка.](dimensions-and-measures-available-in-call-quality-dashboard.md)

## <a name="unclassified-streams"></a>Неклассифицированные потоки

В CQD поток помечется как неклассифицированный при сбойе подключения к интерактивному подключению (ICE) или при отсутствовке всех метрик, необходимых для вычисления классификации потока. 

Чтобы проверить сбои подключения ICE, изучите размеры «First Connectivity Ice» и «Second Connectivity Ice» для значения «FAILED». Если хотя бы то или иное значение указывает на сбой, поток помечется как _Неклассифицированный_.

Если для неклассифицированного  потока успешно было подключено  ICE, скорее всего, он считается неклассифицированным, так как ключевые показатели потока не сообщались. Есть несколько причин, по которым эти показатели могут не сообщаться:

- **Отчеты о качестве** оценки качества не поступили— метрики, используемые для классификации, 2010 2016 2016 2016 2016 2016 2016 2016 2016 2016 2016, а также отчет о качестве оценки качества, отсланный в конце звонка. Если этот отчет не был произведен (например, из-за того, что некоторые сторонние конечные точки не отправляют QoE) или его не удалось отправить (например, из-за с отключения сети), CQD не сможет классифицировать поток.

  > [!TIP]
  > Измерение «QoE Record Available» может использоваться для определения того, был ли получен отчет QoE для потока. Обратите внимание, что это измерение будет иметь значение «True», если отчет QoE был получен от любой конечной точки. Отчет о QoE с обеих конечных точек требуется для наиболее точной отчетности показателей.

- **Короткие вызовы:** для вычисления ключевых показателей потока может быть недостаточно действий с мультимедиа. Без этих показателей CQD не может классифицировать поток.

  > [!TIP]
  > Для определения продолжительности потока можно использовать размеры «Длительность (секунды)», «Длительность (минуты)», «Длительность 5 секунд или меньше» и «Длительность 60 секунд или более». Измерение «Средняя продолжительность звонка» также может использоваться для вычисления средней продолжительности для набора потоков.

- **Низкая загруженности пакетов.** Как и в случае "короткого звонка", для вычисления ключевых показателей потока требуется достаточный показатель использования пакетов. Без этих показателей CQD не может классифицировать поток.
  - Обычно низкое использование пакетов происходит, когда участник присоединяется к собранию для прослушивания, но не выступает (микрофон отключен для большинства вызовов). Здесь аудиопоток, входящий в клиент, имеет высокую загруженацию пакетами, а исходящие из клиента аудиопотоки практически не используются. Длительность потока может быть на час или дольше, но использование пакетов на потоке от клиента до  сервера невелико с момента отключения микрофона и неклассифицированного потока.

  > [!TIP]
  > Измерение «Утилизация пакета» и измерение «Среднее использование пакета» могут использоваться для определения активности пакета в потоке.
