# <a name="dimensions-and-measurements-available-in-call-quality-dashboard-cqd"></a>Измерения и измерения, доступные на панели мониторинга качества звонка (CQD)

Панель мониторинга качества звонков (CQD) для Microsoft Teams и Skype для бизнеса Online позволяет лучше понять качество звонков, сделанных в этих службах. В этой теме подробно описаны размеры и измерения, видимые с помощью CQD. Дополнительные информацию о CQD см. [в](quality-of-experience-review-guide.md)этой Microsoft Teams.

## <a name="first-and-second-endpoint-classification"></a>Классификация первых и вторых конечных точек

Многие измерения и измерения в CQD помечены как первые или вторые. Ниже описана логика, согласно которой конечная точка, относящаяся к потоку или звонку, считается первой.

- Конечная точка сервера (AV MCU, сервер-посредник и так далее) считается первой, если в потоке или звонке участвует сервер.
- Конечная точка клиента считается второй, если поток не находится между двумя конечными точками сервера.
- Если обе конечные точки имеют один и тот же тип, первая и вторая устанавливаются на основе внутреннего порядка категории агентов пользователя, чтобы обеспечить согласование порядка.

Например, каждая строка представляет пару агентов пользователя, участвующих в потоке:

|Категория агентов пользователя для вызывающего абонента |Категория агентов пользователя для вызываемого абонента |First Endpoint |Second Endpoint|First Is Caller
|:--- |:--- |:--- |:--- |:--- |
|AV-MCU |OC (клиент Skype для бизнеса) |AV-MCU |OC (клиент Skype для бизнеса) |TRUE |
|OC (клиент Skype для бизнеса) |AV-MCU |AV-MCU |OC (клиент Skype для бизнеса) |FALSE |
|OC (клиент Skype для бизнеса) |OC (клиент Skype для бизнеса) |OC (клиент Skype для бизнеса) |OC (клиент Skype для бизнеса) |FALSE |
|AV-MCU |Сервер-посредник |Сервер-посредник |AV-MCU ||
|Сервер-посредник |AV-MCU |Сервер-посредник |AV-MCU |TRUE |
|OC (клиент Skype для бизнеса) |Телефон OC (IP-телефон Skype для бизнеса) |OC (клиент Skype для бизнеса) |Телефон OC (IP-телефон Skype для бизнеса) |TRUE |
|Телефон OC (IP-телефон Skype для бизнеса) |OC (клиент Skype для бизнеса) |OC (клиент Skype для бизнеса) |Телефон OC (IP-телефон Skype для бизнеса) |FALSE |

> [!NOTE]
> Такая классификация не учитывает, какая конечная точка является вызывающим или вызываемым абонентом. Измерение First Is Caller (Первая точка является вызывающим абонентом) позволяет проще определить принадлежность конечной точки к вызывающему или вызываемому абоненту.

## <a name="dimensions"></a>Измерения

Сведения о измерениях частично основаны на данных, отложенных на портал CQD. Многие значения измерений также можно использовать в качестве фильтров.

### <a name="dimension-data-types-and-units"></a>Типы и единицы измерения

#### <a name="boolean"></a>Boolean

Boolean values are always either True or False. В некоторых случаях истина также может быть представлена как 1, а ложь — как 0.

#### <a name="range"></a>Диапазон

Измерения, выраженные в виде диапазона или группы значений, отображаются в следующем формате.

 _\<sort order string\> [\<lower bound inclusive\> - \<upper bound exclusive\>_

Например, измерение Duration (Minutes) (Длительность в минутах) представляет длительность звонка в секундах, значение которой отображается в виде диапазона.

|Duration (Minutes) |Что означает |
|:--- |:--- |
|062: [0-0) |Длительность потока = 0 минут |
|064: [1-2) |1 минута < = длительность потока < 2 минуты |
|065: [2-3) |2 минуты < = длительность потока < 3 минуты |
|066: [3–4) |3 минуты < = длительность потока < 4 минуты |
|  | |

Строка \<sort order string> служит для управления порядком сортировки при отображении данных и может использоваться для фильтрации. Например, выполнив фильтрацию по параметру Duration (Minutes) < 065, мы отобразим потоки с длительностью меньше 2 минут (0 вначале нужен для правильной работы фильтрации). Фактическое значение указанной строки не имеет значения.

> [!NOTE]
> Вы можете заметить, что диапазоны для заданного измерения кажутся недопустимыми. Примером может быть сила сигнала Wifi, показывающая количество звонков в диапазоне 082: [100 –110), когда 100 является максимальным значением для силы сигнала Wifi. Это связано с присвоением чисел диапазонам в модели данных CQD. Если целое число — 99, оно учитывается в диапазоне 081: [90 – 100). Если это значение 100, оно будет учитываться в диапазоне 082: [100 – 110). Это не означает, что указаны значения силы сигнала Wi-Fi более 100 %.

#### <a name="enumeration-strings"></a>Строки enumeration

Строки, используемые CQD, часто получаются из файлов данных, и они могут быть практически любым сочетанием символов в пределах допустимой длины. Некоторые измерения выглядят как строки, но поскольку они могут быть только одним из кратких списков готовых значений, они являются суммированиями, а не истинными строками. Некоторые строкиumerations также используются в парах.

#### <a name="enumeration-pair"></a>Парное перечисление

Измерения, выраженные в виде парного перечисления, отображаются в следующем формате.

 _\<enumeration value from one end point\> : \<enumeration value from the other endpoint\>_

Порядок значений перечисления является согласованным, но не отражает порядок для первой и второй конечных точек.

Например, Network Connection Detail Pair (Парные сведения о сетевом подключении) показывает значения сведений о сетевом подключении для двух этих конечных точек:

|Network Connection Detail Pair |Что означает |
|:--- |:--- |
|Wired : Wired |Как первая, так и вторая конечные точки использовали проводные подключения Ethernet. |
|Wired : wifi |Первая конечная точка использовала проводное подключение Ethernet, а вторая  подключение Wi-Fi, либо наоборот. |
|: wifi |Первая конечная точка использовала подключение Wi-Fi, а сетевое подключение, использованное второй конечной точкой, неизвестно, либо наоборот. |
| | |

#### <a name="blank-values"></a>Пустые значения

В приведенной ниже таблице перечислены возможные причины, по которым измерение может быть пустым. Многие измерения и измерения будут пустыми, если измерение QoE Record Available (Доступ к записи о QoE) имеет ложный характер. Обычно это происходит, если звонок не удалось установить или клиент не отправил телеметрию в службу.

Многие значения измерений также можно использовать в качестве фильтров. В следующей таблице перечислены измерения, доступные в CQD, в порядке, задаваемом в редакторе запросов.

|Имя меры|Единицы измерения|Описание|
|:---|:---|:---|
|Total Stream Count |Число потоков |Число потоков мультимедиа независимо от типа мультимедиа, включая такие потоки, как надежность и диагностика, которые могут не иметь такого типа. |

|Total Media Failed Stream Count |Число потоков |Число потоков, где не удалось установить канал передачи медиаданных либо он завершался неправильно. |

### <a name="notes-on-measurements"></a>Заметки об измерениях

#### <a name="accuracy-limitations"></a>Ограничения точности
Определенные единицы измерения для подсчета голосовых вызовов и пользователей зависят от того, как выполняется отдельная операция подсчета для набора данных. В настоящее время количество ошибок в 0,2 % зависит от количества строк, в которых выполняется операция. Для максимально точного объема следует использовать показатели подсчета потоков, так как они не зависят от этой отдельной операции подсчета. Фильтрация для уменьшения объема данных может привести к уменьшению ошибки, но не устранят этот источник ошибок при отдельных звонках и подсчетах пользователей. Дополнительные сведения об этом ограничении можно найти здесь: агрегатная функция [dcount.](/azure/data-explorer/kusto/query/dcount-aggfunction)

## <a name="filters"></a>Фильтры

Многие значения измерений и измерений также можно использовать в качестве фильтров. Фильтры в запросе можно использовать для того, чтобы исключить информацию так же, как для добавления или добавления сведений в запрос с помощью измерений или измерений.
