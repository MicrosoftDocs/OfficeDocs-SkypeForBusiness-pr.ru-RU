# <a name="migration-and-interoperability-guidance-for-organizations-using-teams-together-with-skype-for-business"></a>Руководство по миграции и взаимодействию для организаций, использующих Teams вместе со Skype для бизнеса

> [!Tip] 
> Просмотрите следующие сеанса, чтобы узнать о [взаимодействие и совместная работа](https://aka.ms/teams-upgrade-coexistence-interop)

Как принять группами запускает организации с Скайп для бизнеса, администраторы могут управлять взаимодействия с пользователями в своей организации, с помощью концепцию сосуществования «режима», который является свойством TeamsUpgradePolicy. В режиме администраторам управлять взаимодействия и перехода как ими переход к группам Скайп для бизнеса.  Режим пользователя определяет, в которой клиентские входящих чаты и земля вызовы также как и какие новые собрания службы (группы или Скайп для бизнеса) планирования. В будущем режим также будет использоваться для определения поведения клиентских групп с точки зрения, какие функциональные возможности будут доступны. 


## <a name="fundamental-concepts"></a>Основные понятия

1.  *Взаимодействия* : 1-1 обмена данными между Lync/Скайп для корпоративных пользователей и групп пользователей.

2.  *Федерация* : обмена данными между пользователями из различных клиентов.

3.  Все команды, у пользователей есть базового Скайп для бизнеса учетной записи, являющейся «размещенных» либо через Интернет или локальной:
    - Пользователи, уже используется Скайп для бизнеса в Интернет использовать их существующей учетной записи online.
    - Уже используется Скайп для бизнеса/Lync локальных пользователей используйте их существующей локальной учетной записи.
    - Пользователи, для которых не удалось обнаружить существующий Скайп для бизнес-партнер будет иметь Скайп для бизнеса в Интернет учетной записи автоматически подготавливаться при создании группы пользователей.

4.  При наличии локального развертывания либо Скайп для бизнеса или Lync, и эти пользователи должны быть группами пользователей, по крайней мере убедитесь, что подключение Azure AD выполняет синхронизацию msRTCSIP-DeploymentLocator атрибут в AAD, поэтому этой команды/Скайп для бизнеса Online правильно обнаруживает в локальную среду. Кроме того для перемещения всех пользователей в режим только для группы (например, обновление пользователя), *необходимо настроить Скайп для бизнеса гибридный режим*. Для получения дополнительных сведений см [Настройка Azure AD для Скайп для бизнеса и рабочих групп](https://docs.microsoft.com/en-us/SkypeForBusiness/hybrid/configure-azure-ad-connect).

5.  Взаимодействие между рабочими группами и Скайп для бизнес-пользователи — только возможные *Если Интернет-версия расположен пользователь групп в Скайп для бизнеса*. Получатель Скайп для бизнес-пользователь может быть размещен либо локальной (и требуется настройка Скайп для гибридных Business) или в оперативном режиме. Пользователи, которые размещаются в Скайп для бизнеса в локальной можно использовать команды в режиме острова (задать далее в этом документе), но не могут использовать команды для взаимодействия и создать федерацию с других пользователей, использующих Скайп для бизнеса.  

6.  Поведение обновления и взаимодействия определяются на основе режима совместной работы пользователя, приведенную ниже, более поздней версии. Режим управляется TeamsUpgradePolicy. TeamsInteropPolicy больше не Оплаченный и предоставление mode = прежних версий больше не допускается. 

7.  Обновление пользователя в режим TeamsOnly гарантирует, что все входящие чаты и вызовы будут всегда land в клиент группы пользователя, независимо от того, какой клиент его orignated из. Эти пользователи будут также запланировать все новые собрания в группах. Должна находиться в режиме TeamsOnly, пользователь должен быть размещен online в Скайп для бизнеса. Это необходимо для обеспечения взаимодействия, федерации и полного управления группами пользователей. Обновление до TeamsOnly пользователя:
    - Если пользователь является, расположенным в Скайп для бизнеса через Интернет (или никогда не было любой учетной записи Скайп), предоставить им TeamsUpgradePolicy с режимом = TeamsOnly, с помощью экземпляра «UpgradeToTeams», с помощью PowerShell, или выберите режим TeamsOnly с помощью центра администрирования группы.
    - Если пользователь является, размещенные локально, используйте `Move-CsUser` из локальной admin средства для первого move пользователю Скайп для бизнеса в Интернет.  Если у вас есть Скайп Business Server 2019 или CU8 для Скайп для Business Server 2015, можно указать `-MoveToTeams` переключитесь в `Move-CsUser` для перемещения пользователя непосредственно для групп в процессе перемещения online. Этот параметр, также будет переноситься собраний пользователя к группам, (Хотя сейчас собрания миграции функционирует только для клиентов, КОСНИТЕСЬ). Если `-MoveToTeams` не указанный или недоступно, затем после `Move-CsUser` завершается, назначить режим TeamsOnly пользователю с помощью PowerShell или группами центра администрирования. Для получения дополнительных сведений см. [Перемещение пользователей между локальной и облачной](https://docs.microsoft.com/en-us/skypeforbusiness/hybrid/move-users-between-on-premises-and-cloud).  Для получения дополнительных сведений о миграции на собрания просмотрите [с помощью службы миграции собрания (MMS)](https://docs.microsoft.com/en-us/skypeforbusiness/audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms).

8.  Использование функции групп телефонной системой с группами, пользователи должны быть в режиме TeamsOnly (то есть, расположенным в Скайп для бизнеса в Интернет и обновлены для группы) и должен быть настроен либо для телефонной системой Microsoft [Прямой маршрутизации](https://techcommunity.microsoft.com/t5/Microsoft-Teams-Blog/Direct-Routing-is-now-Generally-Available/ba-p/210359#M1277) (который позволяет использовать телефонной системой с помощью собственных магистралях SIP и SBC) или Office 365 вызов плана.   

9.  Планирование собраний группы с аудиоконференции (по телефону с помощью ТСОП или модем) в настоящее время доступен только для пользователей, которые размещаются в Скайп для бизнеса в Интернет. Для группы пользователей с Скайп локальной учетной записи бизнес поддерживается в TAP.


## <a name="coexistence-modes"></a>Режимы сосуществования

Взаимодействие и миграция управляются зависимости «сосуществования режима» с помощью TeamsUpgradePolicy. Режимы совместного существования предоставляют простой, прогнозируемый возможности для конечных пользователей как организаций перехода от Скайп для бизнеса группам. Для перемещения групп организации режим TeamsOnly — конечному получателю для каждого пользователя, хотя не все пользователи должны быть назначены TeamsOnly (или любой режим) в то же время. Пользователей, достигает TeamsOnly режиме организации могут использовать любой из Скайп для бизнеса режимов (SfBOnly, SfBWithTeamsCollab, SfBWithTeamsCollabAndMeetings), чтобы убедиться в прогнозируемый связи между пользователями, которые TeamsOnly и тех, кто еще не.


С точки зрения технические режима пользователя управляет ряд аспектов взаимодействие с пользователем:

- *Маршрутизация входящих*: В какой клиента (группы или Скайп для бизнеса) входящего обсуждений и вызывает Земля? 
- *Публикация сведений о присутствии*: — это состояние присутствия пользователя, которая отображается другим пользователям на основании их активности в группы или Скайп для бизнеса? 
- *Планирование на собрания*: какие служба используется для планирования собраний новые и проверка того, что соответствующие надстройки присутствует в Outlook. Обратите внимание на то, что TeamsUpgradePolicy не влияет на присоединения к собранию. Пользователи могут всегда *присоединиться к* любого собрания, будет ли это Скайп для собраний или группами собрания.
- *Взаимодействия с пользователем*: какие функциональные возможности для клиента Business доступна в группы и/или Скайп? Пользователей можно инициировать вызовы и беседы в группы и Скайп для бизнеса? Доступно, группами & каналы взаимодействия?  Как будет описано далее в этой статье, в этом последнем аспектом режиме запускается доставить.

Для получения дополнительных сведений о маршрутизации и сведения о присутствии поведение в зависимости от режима просмотра [Совместная работа с Скайп для бизнеса](https://docs.microsoft.com/en-us/MicrosoftTeams/coexistence-chat-calls-presence).

Тем не менее с точки зрения взаимодействия режим можно более просто назвать определение работы:
- *Беседы и вызов*: клиента пользователь использует?
- *Планирование собрания*: пользователи планировать нового собрания как группы или Скайп для деловых встреч?
- *Доступность функций совместной работы групп клиента*. Доступно функциональные возможности каналы & групп и файлы во время пользователи по-прежнему имеют Скайп для бизнеса?

Ниже перечислены способы.

TeamsUpgradePolicy предоставляет два основных свойства: режим и NotifySfbUsers.

|NotifySfbUsers|Bool|*Значение false* или true|Указывает, следует ли отображать заголовок в Скайп для клиента Business, уведомляющее о группами скоро заменит Скайп для бизнеса. Это не может быть значение true, если режим = TeamsOnly.|

## <a name="federation-considerations"></a>Сведения о выполнении федерации

Федерации из рабочих групп на другого пользователя, с помощью Скайп для бизнеса требуется пользователей группам быть Интернет-версия размещен в Скайп для бизнеса. В конечном счете группами пользователей, размещенных в Скайп для бизнеса в локальной смогут федерацию с группами только пользователей.

TeamsUpgradePolicy управляет маршрутизации для входящих федеративных чаты и звонки. Федеративные поведение маршрутизации — это так же владельцу сценариев, *за исключением в режиме острова*же.  Когда получатели находятся в режиме о-ва: 
- Чаты и вызовам из групп land SfB Если получатель находится в *федеративных клиента*.
- Чаты и вызовам из Земля группами в группах, если получатель является в *же владельцу*.
- Чаты и вызовам из SfB всегда land в Скайп для бизнеса.

Для получения дополнительных сведений см [Совместная работа с Скайп для бизнеса](https://docs.microsoft.com/en-us/MicrosoftTeams/coexistence-chat-calls-presence).

## <a name="the-intended-client-user-experience-in-teams-when-using-sfb-modes"></a>Предполагаемая клиента взаимодействие с пользователем в группах при использовании режима SfB
Когда пользователь находится в любой из Скайп для режимов Business (SfBOnly, SfBWithTeamsCollab, SfBWithTeamsCollabAndMeetings), все входящие чаты и вызовы направляются Скайп пользователя для клиента Business. Чтобы избежать путаницы конечного пользователя и гарантировать правильную маршрутизацию, функциональные возможности телефонной и чата в клиенте группы — это *должна быть отключена, если пользователь находится в любой из Скайп для режимов бизнеса*. Аналогично планирования собраний в группах — это *должна быть явно отключена, если пользователи находятся в режимах SfBOnly или SfBWithTeamsCollab*и явно включены, если пользователь находится в режиме SfBWithTeamsCollabAndMeetings. 

### <a name="automatic-conformance-of-teams-client-based-on-mode-planned"></a>Автоматическое о соответствии клиентских групп на основе режима (запланировано) 
Функциональность, автоматически отключает чата и вызов функции, а также включить или отключить планирования собраний на основе режима пока недоступна в широком но теперь запуск выгрузка данных для программы TAP. Для получения дополнительных сведений о новых функциональных возможностях видеть [группы взаимодействия с пользователем и соответствия для режима совместной работы](https://docs.microsoft.com/en-us/MicrosoftTeams/teams-client-experience-and-conformance-to-coexistence-modes).

### <a name="manual-configuration-of-workload-policy-settings-prior-to-automatic-conformance"></a>Ручная настройка параметров политики рабочей нагрузки, до автоматического соответствия
До этого решения для автоматического совместимости для режимов доставляется, администраторы могут применять к целям взаимодействия с пользователем режима TeamsUpgradePolicy вручную настроив значения TeamsMessagingPolicy, TeamsCallingPolicy, и TeamsMeetingPolicy. Кроме того, при использовании `Grant-CsTeamsUpgradePolicy` в PowerShell, командлет проверяет конфигурацию соответствующие параметры в TeamsMessagingPolicy, TeamsCallingPolicy и TeamsMeetingPolicy для determmine, если эти параметры совместимы с указанного режима. Если какие-либо не настроена должным образом, предоставление будет выполнена успешно, но будет получено предупреждение, указывающее, какие параметры настроены неправильно. Администратор должен обновлять указанной политики для обеспечения работы совместимые пользователей в группах. Если администратор решил не выполнять никаких действий, из-за предупреждение, пользователи по-прежнему могут иметь доступ к чата, вызов и/или собрания функции планирования в группах в зависимости от значения TeamsMessagingPolicy, TeamsCallingPolicy и TeamsMeetingPolicy, что может привести к путанице работы конечных пользователей.

Для получения дополнительных сведений, на какие политики проверяются параметры при предоставлении TeamsUpgadePolicy видеть [группы взаимодействия с пользователем и соответствия для режима совместной работы](https://docs.microsoft.com/en-us/MicrosoftTeams/teams-client-experience-and-conformance-to-coexistence-modes).


**Примечание:** До доставки автоматического применения поведение клиента, описанных выше каждый из режимов SfB ведут себя практически одинаково. Режимы SfBOnly, SfBWithTeamsCollab и SfBWithTeamsCollabAndMeetings все идентичны в как направлять входящие звонки и беседы. Единственное отличие сейчас находится в включении надстройки Outlook для групп и Скайп для бизнеса. Пока не доставлено experiece дифференцированных клиента только 1 режимы SfB включена в портал администрирования. Но все режимы доступны в PowerShell.


## <a name="teamsinteroppolicy-and-legacy-mode-has-been-retired"></a>TeamsInteropPolicy и режиме прежних версий устарела 

TeamsInteropPolicy вместо нее TeamsUpgradePolicy. Все компоненты, которые ранее соблюдаться TeamsInteropPolicy были обновлены принять TeamsUpgradePolicy.  Microsoft ранее введен режим «Устаревший» в TeamsUpgradePolicy для облегчения перехода с TeamsInteropPolicy на TeamsUpgradePolicy. В режиме прежних версий компоненты маршрутизации, которые поняты TeamsUpgradePolicy будет вернуться к TeamsInteropPolicy. Маршрутизация теперь полностью поддерживает TeamsUpgradePolicy. Режим прежних версий больше не поддерживается и больше не можно предоставить режим прежних версий
