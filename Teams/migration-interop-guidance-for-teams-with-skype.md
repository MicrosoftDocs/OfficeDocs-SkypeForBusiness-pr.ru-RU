# <a name="coexistence-modes---reference"></a>Режимы сосуществования — справочные материалы

> [!Important] 
> - После прекращения использования Skype для бизнеса Online 31 июля 2021 г. невозможно назначить режим сосуществования, отличный от TeamsOnly, пользователю, размещенного в облаке. Как и до выхода из эксплуатации, пользователям, Skype для бизнеса Server локальной среде, можно назначить любой режим, *кроме TeamsOnly*. 

Режимы сосуществования обеспечивают простой и предсказуемый интерфейс для конечных пользователей по мере перехода организаций с Skype для бизнеса на Teams. Для организации, перемещаемой в Teams, режим TeamsOnly является конечным местом назначения для каждого пользователя, хотя не всем пользователям необходимо назначить TeamsOnly (или любой режим) одновременно. До того как пользователи перейдите в режим TeamsOnly, организации могут использовать любой из режимов Skype для бизнеса (SfBOnly, SfBWithTeamsCollab, SfBWithTeamsCollabAndMeetings), чтобы обеспечить предсказуемую связь между пользователями, которые являются TeamsOnly, и пользователями, которые еще не подключены.

С технической точки зрения режим пользователя управляет несколькими аспектами взаимодействия с пользователем:

- *Входящая маршрутизация*: в каком клиенте (Teams или Skype для бизнеса) входящие чаты и звонки будут входящие? 
- *Публикация сведений о* присутствии: отображается ли присутствие пользователя другим пользователям на основе их активности в Teams или Skype для бизнеса? 
- *Планирование собраний*. Какая служба используется для планирования новых собраний и обеспечения на наличие соответствующей надстройки в Outlook? TeamsUpgradePolicy не управляет присоединением к собранию. Пользователи всегда могут *присоединиться к* любому собранию, будь то Skype для бизнеса собрания или собрания Teams.
- *Взаимодействие с* клиентом: какие функции доступны в Teams и (или) Skype для бизнеса клиенте? Могут ли пользователи инициировать звонки и чаты в Teams, Skype для бизнеса или и то, и другое? Доступен ли & каналов Teams?  

Дополнительные сведения о маршрутизации и поведении присутствия на основе режима см. в разделе "Сосуществование [с](./coexistence-chat-calls-presence.md) Skype для бизнеса".

Однако с точки зрения взаимодействия режим можно описать как определяющий интерфейс для:
- *Чат и звонки*: какой клиент использует пользователь?
- *Планирование собраний*. Планирует ли пользователь новые собрания как Teams или Skype для бизнеса собрания?
- *Доступность функций совместной работы в клиенте Teams*. Доступны ли & и файлов Teams, пока пользователи не Skype для бизнеса?

Ниже перечислены режимы. Пользователи облака должны быть пользователями TeamsOnly, а пользователи, размещенные в локальной среде, должны быть в любом режиме, кроме TeamsOnly.

## <a name="using-teamsupgradepolicy"></a>Использование TeamsUpgradePolicy

TeamsUpgradePolicy предоставляет два ключевых свойства: Mode и NotifySfbUsers. 
</br>
</br>

|Параметр|Тип|Допустимые значения</br>(по умолчанию в курсиве)|Описание|
|---|---|---|---|
|Режим|Перечисления|*Острова*</br>Только Teams</br>SfBOnly</br>SfBWithTeamsCollab</br>SfBWithTeamsCollabAndMeetings|Указывает режим, в который должен выполняться клиент.|
|NotifySfbUsers|Bool|*False* или true|Указывает, следует ли отображать баннер в клиенте Skype для бизнеса, информирующее пользователя о том, что Teams скоро заменит Skype для бизнеса. Это не может быть верно, если Mode=TeamsOnly.|
|||||

Teams предоставляет все соответствующие экземпляры TeamsUpgradePolicy с помощью встроенных политик только для чтения. Поэтому доступны только командлеты Get и Grant. Встроенные экземпляры перечислены ниже.
</br>
</br>

|Identity |Режим|NotifySfbUsers|
|---|---|---|
|Острова|Острова|False|
|IslandsWithNotify|Острова|Верно|
|SfBOnly|SfBOnly|False|
|SfBOnlyWithNotify|SfBOnly|Верно|
|SfBWithTeamsCollab|SfBWithTeamsCollab|False|
|SfBWithTeamsCollabWithNotify|SfBWithTeamsCollab|Верно|
|SfBWithTeamsCollabAndMeetings|SfBWithTeamsCollabAndMeetings|False|
|SfBWithTeamsCollabAndMeetingsWithNotify|SfBWithTeamsCollabAndMeetings|Верно|
|UpgradeToTeams|Только Teams|False|
|Глобальная</br>*По умолчанию*.|Острова|False|
||||

Эти экземпляры политики могут быть предоставлены отдельным пользователям или на уровне клиента. Например:
- Чтобы обновить пользователя ($SipAddress) до Teams, предоставьте экземпляр UpgradeToTeams:</br>
`Grant-CsTeamsUpgradePolicy -PolicyName UpgradeToTeams -Identity $SipAddress`
- Чтобы обновить весь клиент, оставьте параметр удостоверения в команде предоставления:</br>
`Grant-CsTeamsUpgradePolicy -PolicyName UpgradeToTeams`

## <a name="the-teams-client-user-experience-when-using-skype-for-business-modes"></a>Взаимодействие с клиентом Teams при использовании Skype для бизнеса режимов

Если пользователь находится в любом из режимов Skype для бизнеса (SfBOnly, SfBWithTeamsCollab, SfBWithTeamsCollabAndMeetings), все входящие чаты и вызовы направляются в клиент Skype для бизнеса пользователя. Чтобы избежать путаницы с конечными пользователями и обеспечить правильную маршрутизацию, вызовы и функции чата в клиенте Teams автоматически отключаются, когда пользователь находится в любом Skype для бизнеса режиме. Аналогичным образом планирование собраний в Teams автоматически отключается, когда пользователи находятся в режиме SfBOnly или SfBWithTeamsCollab, и автоматически включается, когда пользователь находится в режиме SfBWithTeamsCollabAndMeetings. Дополнительные сведения см. в [статье о клиентском интерфейсе Teams и соответствии режимам сосуществования](./teams-client-experience-and-conformance-to-coexistence-modes.md).

> [!Note] 
> - До доставки автоматического применения Teams и каналов режимы SfbOnly и SfBWithTeamsCollab ведут себя одинаково.
