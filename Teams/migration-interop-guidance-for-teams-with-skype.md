# <a name="migration-and-interoperability-guidance-for-organizations-using-teams-together-with-skype-for-business"></a>Руководство по миграции и взаимодействия для организаций, с помощью команды Скайп для бизнеса

В апреле 2018 Microsoft уточнен его руководство по миграции групп из Скайп для бизнеса и как эти два клиенты могут одновременно использоваться для данного пользователя. В этот момент мы объявили об изменении запланированных изменений для упрощения управляемости и повышения уровня удовлетворенности конечных пользователей. Затем Microsoft доставке обновление admin взаимодействия с пользователем, схожих с этого плана. Эти изменения отображаются инструкции в этом документе.

Как уже объявлена, не поддерживается TeamsInteropPolicy. В TeamsUpgradePolicy объединена его функциональные возможности. Взаимодействие и миграция осуществляется с помощью «сосуществования режима» согласно TeamsUpgradePolicy. Выбор режима пользователя управляет маршрутизации входящих звонков и чатов и ли пользователь планирует собраний в группы или Скайп для бизнеса.  В ближайшее время в сочетании с предстоящие TeamsAppPermissionsPolicy режим также преимущественную силу в которой клиентские пользователь может запустить чаты и звонки. 

Настройка TeamsInteropPolicy больше не требуется. Не применяется только при наличии TeamsUpgradePolicy mode = прежних версий.  Теперь, когда поддержка TeamsUpgradePolicy завершен, клиентам следует обновить их конфигурации для использования режима, отличного от прежних версий.


</br>


## <a name="fundamental-concepts"></a>Основные понятия

1.  *Взаимодействия* : 1-1 обмена данными между Lync/Скайп для корпоративных пользователей и групп пользователей.

2.  *Федерация* : обмена данными между пользователями из различных клиентов.

3.  Все команды, у пользователей есть базового Скайп для бизнеса учетной записи, являющейся «размещенных» либо через Интернет или локальной:
    - Пользователи, уже используется Скайп для бизнеса в Интернет использовать их существующей учетной записи online.
    - Уже используется Скайп для бизнеса/Lync локальных пользователей используйте их существующей локальной учетной записи.
    - Пользователи, для которых не удалось обнаружить существующий Скайп для бизнес-партнер будет иметь Скайп для бизнеса в Интернет учетной записи автоматически подготавливаться при создании группы пользователей. Не Скайп для бизнеса лицензии не требуется.

4.  При наличии локального развертывания либо Скайп для бизнеса или Lync, и эти пользователи должны быть группами пользователей, по крайней мере убедитесь, что подключение Azure AD выполняет синхронизацию msRTCSIP-DeploymentLocator атрибут в AAD, поэтому этой команды/Скайп для бизнеса Online правильно обнаруживает в локальную среду. Кроме того для перемещения всех пользователей в режим только для группы (например, обновление пользователя), *необходимо настроить Скайп для бизнеса гибридный режим*.

5.  Взаимодействие между рабочими группами и Скайп для бизнес-пользователи — только возможные *Если Интернет-версия расположен пользователь групп в Скайп для бизнеса*. Скайп для бизнес-пользователь может быть размещен либо локальной (и требуется настройка Скайп для гибридных Business) или в оперативном режиме. Пользователи, которые размещаются в Скайп для бизнеса в локальной можно использовать команды в режиме острова (задать далее в этом документе), но не могут использовать команды для взаимодействия и создать федерацию с других пользователей, использующих Скайп для бизнеса.  

6.  Для обновления пользователя к группам (то есть, предоставить им TeamsUpgradePolicy с режимом = TeamsOnly), пользователь должен быть размещен в Скайп для бизнеса через Интернет. Это необходимо для обеспечения взаимодействия, федерации и полного управления группами пользователей. Чтобы обновить пользователи, размещенные локально, используйте `Move-CsUser` из локальной admin средства для первого move пользователю Скайп для бизнеса в Интернет. Затем предоставить TeamsUpgradePolicy и TeamsInteropPolicy online пользователю или с помощью современных портала можно назначать TeamsOnly режим. Один раз CU8 Скайп для входящего Business Server 2015, клиент может просто воспользоваться нового `-MoveToTeams` переключитесь в `Move-CsUser` который объединяет эти шаги 2 в 1.

7.  Основные политики для управления обновлением и взаимодействия — TeamsUpgradePolicy. TeamsInteropPolicy больше не используется за исключением случаев использования режима TeamsUpgradePolicy = устаревшим и клиентов, использующих режим = прежних версий следует обновить их конфигурации TeamsUpgradePolicy для использования в другой режим.  

8.  Использование групп телефонной системой функций приложения, пользователи должны быть в режиме TeamsOnly (то есть, расположенным в Скайп для бизнеса в Интернет и обновлены для группы) и должен быть настроен либо для телефонной системой Microsoft [Прямой маршрутизации](https://techcommunity.microsoft.com/t5/Microsoft-Teams-Blog/Direct-Routing-is-now-Generally-Available/ba-p/210359#M1277) (позволяет использовать телефонной системой с вашей владельцем магистралях SIP и SBC) или Office 365 вызов плана.   

9.  Планирование собраний группы с аудиоконференции (по телефону с помощью ТСОП или модем) в настоящее время доступен только для пользователей, которые размещаются в Скайп для бизнеса в Интернет. Для группы пользователей с Скайп локальной учетной записи бизнес поддерживается в TAP.


## <a name="coexistence-modes"></a>Режимы сосуществования

Чтобы упростить управление и повысить удовлетворенность конечного пользователя, взаимодействия и перехода теперь управляются зависимости «сосуществования режима» с помощью TeamsUpgradePolicy. Определяет режим пользователя:

- *Маршрутизация входящих* : В какой клиента (группы или Скайп для бизнеса) входящего обсуждений и вызывает Земля? 
- *Планирование на собрания* : какие служба используется для планирования собраний новые и проверка того, что соответствующие надстройки присутствует в Outlook. Обратите внимание на то, что TeamsUpgradePolicy не влияет на присоединения к собранию. Пользователи могут всегда *присоединиться к* любого собрания, будет ли это Скайп для собраний или группами собрания.
- *Взаимодействия с пользователем* : какие функциональные возможности для клиента Business доступна в группы и/или Скайп? Этот метод реализован для режима TeamsOnly. Поддержка для других режимов, зависит от предстоящие TeamsAppPermissionsPolicy. При этом новой политики будет готова, TeamsUpgradePolicy, имеют зависимость ее, чтобы убедиться, что команды надлежащим образом настроена для нужный режим.

Ниже перечислены плановых режимах. SfBWithTeamsCollab и SfBWithTeamsCollabAndMeetings позволит смешанного использования обоих клиентов, но не перекрывающиеся функции. О-ва режим позволяет об использовании обоих клиентов, но с перекрывающимися функциональные возможности. Например в режиме острова, пользователь может начать разговор в любом Скайп для бизнеса или группами, но в SfBWithTeamsCollab, они могут только беседа в Скайп для бизнеса. Обратите внимание на то, что не все режимы еще, полностью доступны.

## <a name="teamsinteroppolicy-being-retired"></a>TeamsInteropPolicy не поддерживается 

Как было сказано ранее, TeamsInteropPolicy вместо нее TeamsUpgradePolicy. Все компоненты, которые ранее соблюдаться TeamsInteropPolicy были обновлены принять TeamsUpgradePolicy. 

Microsoft ранее введен режим «Устаревший» для облегчения перехода с TeamsInteropPolicy на TeamsUpgradePolicy в стандартном режиме, компоненты маршрутизации, которые поняты TeamsUpgradePolicy будет вернуться к TeamsInteropPolicy. Маршрутизация полностью поддерживает TeamsUpgradePolicy и больше не осталось требуется использовать режим прежних версий. Пользователям следует обновить их конфигурации TeamsUpgradePolicy от режима прежних версий.

Клиенты, которые используют режим прежних версий следует обновить их конфигурации, чтобы воспользоваться одним из других режимов. Пользователи по-прежнему используется режим прежних версий получают уведомление, поддерживаемые только три конкретные экземпляры TeamsInteropPolicy, перечисленных ниже. В обоих случаях значение CallingDefaultClient совпадает со значением ChatDefaultClient и AllowEndUserClientOverride всегда имеет значение false. 
</br>
</br>

|---|---|---|---|

## <a name="federation-considerations"></a>Сведения о выполнении федерации

Федерации из рабочих групп на другого пользователя, с помощью Скайп для бизнеса требуется пользователей группам быть Интернет-версия размещен в Скайп для бизнеса. В конечном счете пользователей, размещенных в Скайп для бизнеса в локальной группы смогут федерацию с другими группами пользователей.

TeamsUpgradePolicy управляет маршрутизации для входящих федеративных чаты и звонки. Чтобы облегчить других организаций, инициализация федеративного обмена данными с пользователями в вашей организации, рекомендуется выбрать режим, в частности маршрутизирует либо для Скайп для бизнеса или группами, а не о-ва.

## <a name="completing-the-transition-to-mode-management"></a>Завершение работы перехода к разделу Управление режим

В этом году, корпорация Майкрософт планирует вводить новый тип политики, TeamsAppPermissionsPolicy, чтобы контролировать, какие части клиента команды включены (например, обмен мгновенными Сообщениями, собрания, беседы, каналы). По мере поступления новой политики для включения и отключения рабочих нагрузок в группах TeamsUpgradePolicy будут обновлены таким образом, когда администратор пытается предоставить пользователю экземпляр TeamsUpgradePolicy, сначала проверяет, убедитесь, что TeamsAppPolicy должным образом настроен нужный режим. В противном случае grant завершится с ошибкой, о том, как другие политики сначала должно быть задано. 

Пока TeamsAppPolicy станет доступным, TeamsUpgradePolicy по сути управляет маршрутизации вызовов и беседы, а также планирования собраний (на которых предоставляется через надстроек Outlook). Так как поведение клиента групп еще не на месте, не все режимы включены в современном портала. С точки зрения маршрутизации режимы SfBOnly, SfBWithTeamsCollab и SfBWithTeamsCollabAndMeetings, совпадают. 

## <a name="action-required-for-organizations-that-have-already-used-teamsinteroppolicy"></a>Действия, необходимые для организации, которые уже использовали TeamsInteropPolicy

По-прежнему используется режим прежних версий клиентов необходимо выполнить следующие:

1. Убедитесь, что пользователи с TeamsInteropPolicy назначаются только один из этих трех встроенных экземпляры какие CallingDefaultClient = ChatDefaultClient и для каких AllowEndUserClientOverride = false. Эти экземпляры являются:
</br>
</br>
   |Identity |AllowEndUserClientOverride |CallingDefaultClient|ChatDefaultClient|
   |---|---|---|---|
   |`DisallowOverrideCallingDefaultChatDefault`|False|"Default" (По умолчанию)|"Default" (По умолчанию)|
   |`DisallowOverrideCallingSfbChatSfb`|False|Sfb|Sfb|
   |`DisallowOverrideCallingTeamsChatTeams`|False|Teams|Teams|
   |||||

    Используйте следующий синтаксис командлета, где $policy — это один из приведенных выше значений удостоверений:

    `Grant-CsTeamsInteropPolicy -PolicyName $policy -Identity $SipAddress`

    **Microsoft запросов, что обновление их политик с 30 июня 2018.** После этого приходится Microsoft удаление других экземпляров TeamsInteropPolicy.</br> 
    ***Организации, которые нечасто обновляют к одной из этих экземплярах со временем будут иметь своим пользователям автоматически обновляется, один из этих экземпляров. Очевидно, что желательно, что клиенты этого, чтобы вы могли выбрать, что лучше всего подходит для пользователей.***

2. Если настройка встроенных глобальную политику, отменить это. Глобальная политика должна иметь следующие значения:
</br>
</br>
    |Параметр|Значение|
    |---|---|
    |`AllowEndUserClientOverride`|False|
    |`CallingDefaultClient`|"Default" (По умолчанию)|
    |`ChatDefaultClient`|"Default" (По умолчанию)|
    |||

    Если любое из значений, отличаются от выше, выполните следующие действия, чтобы удалить все настройки, зависящие от клиента.

    `Grant-CsTeamsInteropPolicy -PolicyName $null`

|---|---|
|**О-ва**|Один пользователь запускает оба Скайп для бизнеса и групп-одновременно. Этому пользователю:</br><ul><li>Можно инициировать чаты и вызывает VOIP в любой Скайп для клиента Business или группами. Примечание: Пользователей с Скайп для бизнеса, размещенных в локальной нельзя инициировать из рабочих групп для связи с другой Скайп для корпоративных пользователей.<li>Получает чаты & звонки по протоколу VOIP, инициированная другим пользователем в их Скайп для клиента Business в Скайп для бизнеса.<li>Получает VOIP звонки, инициированная другим пользователем в клиента по группам в группах.<li>Получает бесед, инициированная другим пользователем в в группах:<ul><li>Скайп для бизнеса предоставляются получателя размещаются в Интернете и никогда не войти в систему для групп<li>Команды во всех остальных случаях.</ul><li>Содержит ТСОП функциональные возможности, как указано ниже:<ul><li>Если пользователь, размещенный в Скайп для бизнеса в локальной, звонки по ТСОП инициируется и получаемых с Скайп для бизнеса.<li>Интернет-версия расположен пользователь, есть ли у пользователя телефонной системой, этом случае пользователя:<ul><li>Инициирует и получает вызовы в ТСОП в группах, если пользователь настроен прямой маршрутизации<li>Инициирует и получает вызовы в ТСОП в Скайп для бизнеса, если пользователь имеет MS вызов плана или подключения к Телефонной сети общего пользования с помощью любого из Скайп для соединителя Cloud Business Edition или локальное развертывание Скайп для Business Server (гибридной голосовой связи)</ul></ul><li>Можно планировать собрания в группы или Скайп для бизнеса (и будут видеть оба модули по умолчанию).<li>Для присоединения к любой Скайп для бизнеса или группами собрания; собрания будут открываться в соответствующих клиента.</ul>|
|**SfBOnly**|Один пользователь запускает только Скайп для бизнеса. Этому пользователю:</br><ul><li>Можно инициировать чаты и вызовы только от Скайп для бизнеса.<li>Получает все чата/вызов в их Скайп для клиента Business вне зависимости от где инициировал, пока инициатор — это группы пользователя с помощью Скайп для бизнеса, размещенных в локальной. * <li>Можно запланировать только Скайп для собраний бизнеса, но могут присоединиться к Скайп для бизнеса или группами собраний. </br> *Острова использование режима с локальными пользователями не рекомендуется в сочетании с другими пользователями в режиме SfBOnly. Если пользователь группами с Скайп для бизнеса, размещенный локальной инициирует звонок или пообщаться пользователю SfBOnly, пользователь SfBOnly недоступен и получает email.* пропущенные беседы/звонка|
|**SfBWithTeamsCollab**|Один пользователь запускает оба Скайп для бизнеса и групп-одновременно. Этому пользователю:</br><ul><li>Имеет функциональные возможности пользователя в режиме SfBOnly.<li>Команды, включено только для совместной работы групп (канала); планирования чата/вызова/собраний отключены.</ul>|
|**SfBWithTeamsCollab</br>AndMeetings**</br>(запланировано)|Один пользователь запускает оба Скайп для бизнеса и групп-одновременно. Этому пользователю:<ul><li>В режиме SfBOnly имеет чата и функциями вызова пользователя.<li>Включил групп для совместной работы групп (канала); беседы и вызова отключены.<li>Можно назначать только группы собрания, но могут присоединиться к Скайп для бизнеса или группами собраний.</ul>|
|**TeamsOnly**</br>(требуется SfB Online Домашняя страница)|Один пользователь запускает только группы. Этому пользователю:<ul><li>Получает все беседы и вызывает в клиента по группам, независимо от того, где инициировал.<li>Можно инициировать чаты и звонки только из рабочих групп.<li>Можно планировать собрания только в группах, но могут присоединиться к Скайп для бизнеса или группами собраний.<li>Можно продолжить использование Скайп для бизнеса IP-телефонов.</ul> |
|**Прежних версий**</br>(по умолчанию)|В этом режиме использовался во время перехода TeamsInteropPolicy для TeamsUpgradePolicy чтобы обеспечить согласованную работу как программное обеспечение изменения для развертывания. Теперь, когда TeamsUpgradePolicy полностью поддерживаются этот режим больше не требуется. Клиенты, которые используют режим = прежних версий следует обновить их конфигурации для использования на других режимов.|
