# <a name="migration-and-interoperability-guidance-for-organizations-using-teams-together-with-skype-for-business"></a>Руководство по миграции и взаимодействию для организаций, использующих Teams вместе со Skype для бизнеса

> [!Tip] 
> Просмотрите следующий сеанс, чтобы узнать о [сосуществовании и взаимодействии](https://aka.ms/teams-upgrade-coexistence-interop)

Поскольку в компании Skype для бизнеса начинают принимать команды, администраторы могут управлять работой пользователей в своей организации с помощью концепции режима сосуществования, которая является свойством Теамсупградеполици. В режиме "режим" Администраторы управляют взаимодействием и миграцией, так как они управляют переходом с Skype для бизнеса на Teams.  Режим пользователя определяет, в каких клиентах входящие разговоры и звонки, а также о том, в какой службе (Teams или Skype для бизнеса) запланированы новые собрания. Кроме того, он определяет функции, доступные в клиенте Teams. 


## <a name="fundamental-concepts"></a>Основные понятия

1.  *Взаимодействие* : 1 – 1 связь между пользователями Lync и Skype для бизнеса и пользователем Teams.

2.  *Федерация* : обмен данными между пользователями из разных клиентов.

3.  У всех пользователей Teams есть учетная запись Skype для бизнеса, которая находится в сети или в локальной среде:
    - Пользователи, уже использующие Skype для бизнеса Online, используют существующую учетную запись в Интернете.
    - Пользователи, уже использующие Skype для бизнеса или Lync, используют свою локальную учетную запись.
    - Пользователи, для которых не удается обнаружить существующую учетную запись Skype для бизнеса, будут автоматически настроены учетная запись Skype для бизнеса Online при создании пользователя Teams.

4.  Если у вас есть локальное развертывание Skype для бизнеса или Lync и вы хотите, чтобы эти пользователи были пользователями Teams, вам необходимо по крайней мере убедиться в том, что Azure AD Connect синхронизирует атрибут msRTCSIP-Деплойментлокатор в AAD, поэтому в Teams и Skype для бизнеса Сеть правильно обнаруживает локальную среду. Кроме того, для перемещения пользователей в режим "только для Teams" (например, обновление пользователя) *необходимо предварительно настроить гибридный режим Skype для бизнеса*. Дополнительные сведения можно найти в разделе [Настройка Azure AD Connect для Skype для бизнеса и Teams](https://docs.microsoft.com/en-us/SkypeForBusiness/hybrid/configure-azure-ad-connect).

5.  Взаимодействие между группами и пользователями Skype для бизнеса возможно только *в том случае, если пользователь Teams подключен к сети в Skype для бизнеса*. Пользователь Skype для бизнеса может находиться в локальной сети (и требовать настройки гибридного развертывания Skype для бизнеса) или в Интернете. Пользователи, размещенные в локальной версии Skype для бизнеса, могут использовать Teams в режиме острова (определенный позже в этом документе), но они не смогут работать с другими пользователями Skype для бизнеса с помощью команд взаимодействия или Федерации.  

6.  Поведение при обновлении и взаимодействии определяется в соответствии с режимом сосуществования пользователя, описанным ниже. Режим управляется Теамсупградеполици. 

7.  Обновление пользователя до режима Теамсонли гарантирует, что все входящие разговоры и звонки будут всегда поступать в клиенте Teams пользователя, независимо от того, откуда он был создан. Эти пользователи также будут планировать все новые собрания в Teams. В режиме Теамсонли пользователь должен быть подключен к Интернету в Skype для бизнеса. Это необходимо для того, чтобы обеспечить взаимодействие, Федерацию и полное администрирование пользователя Teams. Чтобы обновить пользователя до Теамсонли, выполните указанные ниже действия.
    - Если пользователь входит в Skype для бизнеса Online (или никогда не имел ни одной учетной записи в Skype), предоставьте им Теамсупградеполици в режиме "Теамсонли" с помощью PowerShell, а затем в центре администрирования Teams выберите режим Теамсонли.
    - Если пользователь подключен к локальной сети, вы можете сначала `Move-CsUser` переместить пользователя в Skype для бизнеса Online с помощью локальных средств администрирования.  Если у вас есть Skype для бизнеса Server 2019 или обновления HF1 для Skype для бизнеса Server 2015, вы можете указать `-MoveToTeams` переключатель, `Move-CsUser` чтобы переместить пользователя прямо в Teams в рамках перемещения в сети. Этот параметр также позволяет перенести собрания пользователя в Teams. Если `-MoveToTeams` он не указан или недоступен, после `Move-CsUser` завершения назначьте режим теамсонли для этого пользователя с помощью PowerShell или центра администрирования Teams. Дополнительные сведения можно найти [в разделе Перемещение пользователей между локальным и облаком](https://docs.microsoft.com/skypeforbusiness/hybrid/move-users-between-on-premises-and-cloud).  Дополнительные сведения о миграции собраний можно найти в разделе [Использование службы миграции собраний (MMS)](https://docs.microsoft.com/skypeforbusiness/audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms).

8.  Чтобы использовать телефонную систему Microsoft с командами, пользователи должны находиться в Теамсонли режиме (например, в Skype для бизнеса Online и обновлены до Teams), и они должны быть настроены для [прямой маршрутизации](https://techcommunity.microsoft.com/t5/Microsoft-Teams-Blog/Direct-Routing-is-now-Generally-Available/ba-p/210359#M1277) Microsoft Phone System (позволяющей использовать телефонную систему с собственные магистральные каналы SIP и SBC) или план звонков по Office 365. Прямая маршрутизация Microsoft Phone System в режиме "острова" не поддерживается.    

9.  Планирование собраний в Teams с помощью голосовой конференции (коммутируемое или коммутируемое подключение через КТСОП) осуществляется независимо от того, где находится пользователь: в Skype для бизнеса Online или Skype для бизнеса. 


## <a name="coexistence-modes"></a>Режимы сосуществования

Взаимодействие и миграция управляются на основе "режима косуществования" с помощью Теамсупградеполици. Режим совместного существования обеспечивает простой и предсказуемый опыт для конечных пользователей в том случае, если Организация переходит из Skype для бизнеса в Teams. Для Организации, перемещающейся в Teams, режим Теамсонли является конечным пунктом для каждого пользователя, хотя ему не нужно назначать Теамсонли (или любой режим) одновременно. Перед тем как пользователи приходили к режиму Теамсонли, организации могут использовать любой из режимов Skype для бизнеса (Сфбонли, Сфбвистеамсколлаб, Сфбвистеамсколлабандмитингс), чтобы обеспечить предсказуемую связь между пользователями, которые еще не являются Теамсонли.


С технической точки зрения пользователя в пользовательском режиме определяются некоторые аспекты взаимодействия с пользователем.

- *Входящая маршрутизация*: в каких клиентах (Teams или Skype для бизнеса) входящие разговоры и звонки на землю? 
- *Публикация присутствия*: является ли присутствие пользователя видимым для других пользователей на основе их активности в Teams или Skype для бизнеса? 
- *Планирование собраний*: какая служба используется для планирования новых собраний и обеспечения наличия в Outlook соответствующей надстройки? Обратите внимание, что Теамсупградеполици не управляет присоединением к собранию. Пользователи всегда могут ** присоединиться к собранию, будь это собрание Skype для бизнеса или собрание Teams.
- *Взаимодействие с клиентом*: какие функции доступны в Teams и в клиенте Skype для бизнеса? Могут ли пользователи инициировать звонки и разговоры в Teams, Skype для бизнеса или и то, и другое? Доступны ли Teams & каналами?  

Дополнительные сведения о маршрутизации и возможностях присутствия, основанных на режиме, приведены в разделе [сосуществование в Skype для бизнеса](https://docs.microsoft.com/en-us/MicrosoftTeams/coexistence-chat-calls-presence).

Однако с точки зрения удобства режим работы может быть более прост в описании, как определение возможностей:
- *Чат и Звонок*: какой клиент использует пользователь?
- *Планирование собраний*: планирование новых собраний в группах или собраниях Skype для бизнеса
- *Доступность функций совместной работы в клиенте Teams*. Доступны ли Teams & каналами и файлов, когда пользователи по-прежнему работают в Skype для бизнеса?

## <a name="federation-considerations"></a>Вопросы по интеграции

Для интеграции из Teams с другим пользователем, использующим Skype для бизнеса, необходимо, чтобы пользователь Teams был подключен к Интернету в Skype для бизнеса. В конечном итоге пользователи Teams, размещенные в локальной среде Skype для бизнеса, смогут выполнять Федерацию только для пользователей Teams.

Теамсупградеполици управляет маршрутизацией входящих федеративных чатов и звонков. Режим федеративного маршрутизации аналогичен сценариям для одного и того же клиента, за исключением случаев, когда он находится *в режиме "острова*".  Если получатели находятся в режиме острова, сделайте следующее: 
- Разговоры и звонки, инициированные из Teams Land в SfB, если получатель находится в *Федеративной клиенте*.
- Разговоры и звонки, инициированные из Teams в Teams, если получатель находится в том *же клиенте*.
- Разговоры и звонки, инициированные из SfB, всегда помещаются в Skype для бизнеса.

Дополнительные сведения можно найти в разделе "сосуществование" [в Skype для бизнеса](https://docs.microsoft.com/en-us/MicrosoftTeams/coexistence-chat-calls-presence).

## <a name="the-teams-client-user-experience-when-using-sfb-modes"></a>Взаимодействие с пользователем клиента Teams при использовании режима SfB
Когда пользователь входит в любой из режимов Skype для бизнеса (Сфбонли, Сфбвистеамсколлаб, Сфбвистеамсколлабандмитингс), все входящие разговоры и звонки перенаправляются клиенту Skype для бизнеса пользователя. Чтобы избежать путаницы и удостовериться в том, что функция маршрутизации, вызов и чат в клиенте Teams автоматически отключается, когда пользователь входит в какой-либо из режимов Skype для бизнеса. Кроме того, планирование собраний в Teams автоматически отключается, если пользователи находятся в режимах Сфбонли или Сфбвистеамсколлаб и автоматически включаются, когда пользователь находится в режиме Сфбвистеамсколлабандмитингс. Дополнительные сведения можно найти [в разделе взаимодействие с клиентами и их соответствие режимам](https://docs.microsoft.com/en-us/MicrosoftTeams/teams-client-experience-and-conformance-to-coexistence-modes)сосуществования.

> [!Note] 
> - Перед отправкой автоматического применения групп и каналов, режимы Сфбонли и Сфбвистеамсколлаб работают одинаково.
