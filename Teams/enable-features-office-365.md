<a name="enable-microsoft-teams-features-in-your-office-365-organization"></a>Включение функций Microsoft Teams в организации Office 365
======================================================

Система Microsoft Teams содержит несколько параметров, которые можно включить или отключить на уровне клиента. Когда система Microsoft Teams активна для клиента, любой пользователь, включенный для Microsoft Teams, наследует эти параметры уровня клиента.

Ниже представлен список функций, которые администратор Office 365 может включить или отключить в Microsoft Teams.

Если не указано иное, по умолчанию для параметра установлено значение "Включено".

Примечание. Администратор Office 365 в любое время может отключить Microsoft Teams с помощью Центра администрирования Office 365. Помните о том, что пользователи с активными лицензиями на Microsoft Teams продолжат видеть плитку приложения Teams, даже если вы отключите Teams. Дополнительные сведения об удалении лицензий пользователей см. в статье "Управление доступом пользователей к лицензиям Microsoft Teams". После отключения Teams доступ из клиента Microsoft Teams блокируется, но данные, предоставляемые через другие клиенты и службы, остаются доступными, например файлы в SharePoint и OneDrive. Все данные остаются на месте, если только не производится явное удаление команд.

<a name="tenant-wide-settings"></a>Параметры на уровне клиента 
---------------------

В **области параметров на уровне клиента** можно включить или отключить общие параметры, параметры для интеграции электронной почты, приложений и настраиваемого облачного хранилища.

### <a name="general"></a>Общие

В разделе "Общие" можно настроить для своей организации следующие параметры:

> ![Снимок экрана с общим разделом в параметрах на уровне клиента.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image1.png)

-   **Show organizational chart in personal profile (Показать организационную диаграмму в личном профиле).** Когда этот параметр включен, в карточке контакта пользователя отображается значок организационной диаграммы. При его выборе отображается подробная организационная диаграмма.

    ![Снимок экрана со значком организационной диаграммы в карточке контакта пользователя.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image2.png)

    ![Снимок экрана с организационной диаграммой.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image3.png)

-   **Use Skype for Business for recipients who don’t have Microsoft Teams (Использовать Skype для бизнеса для получателей без Microsoft Teams).** Когда этот параметр включен, пользователям Microsoft Teams разрешено общаться с другими пользователями в организации, для которых не была включена поддержка Microsoft Teams через Skype для бизнеса.

-   **Allow T-bot proactive help messages (Разрешить справку П-бота).** Когда этот параметр включен, П-бот открывает с пользователями сеанс приватного чата, чтобы помочь им с использованием Microsoft Teams.

    ![Снимок экрана с разделом П-бота в интерфейсе Microsoft Teams.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image4.png)

<a name="email-integration"></a>Интеграция электронной почты
-----------------

Включите параметр "Email integration" (Интеграция электронной почты), чтобы пользователи могли отправить сообщение электронной почты по адресу канала в Microsoft Teams. Подобную операцию пользователи могут выполнить для любого канала, относящегося к принадлежащей им команде. Кроме того, пользователи могут отправлять сообщения электронной почты в любой канал в команде, где для участников включены соединители добавления. И даже если у пользователя нет разрешений на создание адреса электронной почты для канала, когда кто-либо с подобными правами создает такой адрес, пользователь может обратиться к нему с помощью меню \<значка "Дополнительно"\> этого канала.

В разделе "Email integration" (Интеграция электронной почты) можно настроить для своей организации следующие параметры:

   ![Снимок экрана с разделом интеграции электронной почты в параметрах на уровне клиента.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image5.png)

-   **Allow users to send emails to channels (Разрешить пользователям отправлять сообщения электронной почты в каналы).** Когда этот параметр включен, активируются обработчики почты, и пользователи могут публиковать сообщения в канале, отправляя сообщения электронной почты на адрес канала Microsoft Teams.

> Чтобы найти адрес электронной почты канала, щелкните элемент **Дополнительные параметры** рядом с именем канала и выберите **Получить адрес эл. почты**.

-   **Restricted Senders List (Ограниченный список отправителей).** Параметр позволяет дополнительно ограничить домены отправителей, разрешив отправку сообщений электронной почты в каналы Microsoft Teams только из доменов SMTP.

<a name="apps"></a>Приложения
----

Приложения в Microsoft Teams отлично подходят для интеграции средств и служб, необходимых вашей команде, прямо в любой канал или чат.

В разделе **Приложения** можно настроить для своей организации следующие параметры:

![Снимок экрана с разделом приложений.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image6.png)

-   **Allow external apps in Microsoft Teams (Разрешить внешние приложения в Microsoft Teams).** Когда этот параметр включен, пользователи могут добавлять доступные вкладки и боты для клиента Office 365.
![Снимок экрана с параметром "Allow external apps control" (Разрешить управление внешними приложениями) в разделе приложений.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image6.2.png)

-   **Allow sideloading of external apps (Разрешить загрузку неопубликованных внешних приложений).** Когда этот параметр включен, пользователи могут устанавливать и активировать настраиваемые вкладки и боты.

<a name="custom-cloud-storage"></a>Настраиваемое облачное хранилище
--------------------

Сейчас параметры облачного хранилища в Microsoft Teams включают Box, Dropbox, Google Диск и ShareFile. Пользователи могут отправлять и совместно использовать файлы из служб облачного хранилища в каналах и чатах Microsoft Teams. Щелкните или нажмите переключатель рядом с поставщиками облачных хранилищ, которые нужно использовать в вашей организации.

![Снимок экрана с разделом настраиваемого облачного хранилища.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image7.png)

<a name="user-settings-by-license"></a>Параметры пользователей по лицензии
------------------------

В области **User settings by license** (Параметры пользователей по лицензии) вы можете включить или отключить параметры для команд и каналов, звонков и собраний, а также обмена сообщениями.

<a name="teams-and-channels"></a>Команды и каналы
------------------

Команда предназначена для объединения группы людей, которые тесно сотрудничают друг с другом для выполнения поставленных задач. Команды могут динамически формироваться под конкретные проекты (например, выпуск продукта на рынок или организация цифрового конференц-зала). Кроме того, команды могут быть оперативными для отражения внутренней структуры организации.

Являясь администратором, вы можете управлять владельцами и участниками команд с помощью панели мониторинга "Группы" на портале Центра администрирования Office 365. В разделе "Команды и каналы" щелкните ссылку **Use the Groups dashboard in the Office 365 admin center to manage teams** (Использовать панель мониторинга "Группы" Центре администрирования Office 365 для управления командами).

Вы можете контролировать, кто из пользователей организации может создавать команды в Microsoft Teams. Определенные в группах Office 365 параметры создания применяются и к Microsoft Teams. Дополнительные сведения об управлении группами Office 365 см. в статьях [Создание групп Office 365](https://support.office.com/en-us/article/Create-Office-365-groups-74a1ef8b-3844-4d08-9980-9f8f7a36000f) и [Управление правами на создание групп Office 365](https://support.office.com/en-us/article/Control-who-can-create-Office-365-Groups-4c46c8cb-17d0-44b5-9776-005fced8e618).

Примечание. Вы не можете создавать команды из панели мониторинга "Группы". Для этого нужно использовать классический или веб-клиент Microsoft Teams.

По умолчанию создать команду или группу может любой пользователь. Пользователи могут создавать команды, выбрав пункт "Команды" в левой части клиента Microsoft Teams (классического или веб-приложения) и затем пункт "Создать команду" в нижней части окна, под списком команд.

Сейчас максимальное число команд, которые может иметь клиент Office 365, по умолчанию равно 500 000. Глобальный администратор может создать любое число команд. Пользователь может создать 250 команд. Владелец команды может добавить в нее 2500 участников.

![Снимок экрана с разделом параметров пользователей по лицензии.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image8.png)

Каналы являются подкатегориями команд. Любой участник команды может добавить канал и участвовать в беседах канала. Канал можно создать для действия или для отдела. Беседы, файлы и вики-страницы относятся к конкретному каналу, но просматривать их могут все участники команды.

### <a name="calls-and-meetings"></a>Звонки и собрания

Каналы являются подкатегориями команд. Любой участник команды может добавить канал и участвовать в беседах канала. Канал можно создать для действия или для отдела. Беседы, файлы и вики-страницы относятся к конкретному каналу, но просматривать их могут все участники команды.

В разделе **Звонки и собрания** можно настроить для своей организации следующие параметры:

> ![Снимок экрана с разделом звонков и собраний.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image9.png)

-   **Allow scheduling for private meetings (Разрешить планирование для частных собраний):** Когда параметр включен, пользователи могут планировать частные собрания, не указанные ни в одном из каналов.

-   **Allow ad-hoc channel meetup (Разрешить специальную встречу в канале).**

-   **Allow scheduling for channel meetings (Разрешить планирование для собраний канала).** Когда параметр включен, пользователи могут планировать собрание для канала, к которому все участники этого канала могут присоединиться одним щелчком.

-   **Allow videos in meetings (Разрешить видео на собраниях).** Указывает, разрешено ли использовать видео на собраниях.

-   **Allow screen sharing in meetings (Разрешить демонстрацию экрана на собраниях).** Указывает, разрешено ли использовать демонстрацию экрана на собраниях.

-   **Allow private calling (Разрешить частные звонки).** Когда параметр включен, пользователи могут совершать частные звонки.

Максимальное число людей на собрании равно 80. В приватном чате могут участвовать 20 человек, включая создавшего его пользователя.

### <a name="messaging"></a>Обмен сообщениями 

В разделе "Обмен сообщениями" можно настроить для своей организации следующие параметры:

![Снимок экрана с разделом сообщений.](media/Enable_Microsoft_Teams_features_in_your_Office_365_organization_image10.png)

-   **Enable Giphy so users can add gifs to conversations (Включить Giphy, чтобы пользователи могли добавлять в беседы GIF-файлы).** Когда параметр включен, пользователи могут добавлять рисунки с анимацией во время бесед.

    -   **Content Rating (Рейтинг контента).** Если включены рисунки с анимацией, можно применить рейтинг контента для ограничения типов подобных изображений, которые разрешено отображать в беседах. Доступны следующие варианты рейтинга контента:

        -   Без ограничений

        -   Средний (значение по умолчанию)

        -   Строгий

-   **Enable memes that users can edit and add to conversations (Включить мемы, которые пользователи могут изменять и добавлять в беседы).** Когда параметр включен, можно использовать интернет-мемы для придания публикациям шутливого оттенка.

-   **Enable stickers that users can edit and add to conversations (Включить стикеры, которые пользователи могут изменять и добавлять в беседы).** Когда параметр включен, пользователи могут публиковать изображения с редактируемым текстом для привлечения внимания участников канала.

-   **Allow owners to delete all messages (Разрешить владельцам удалить все изображения).** Когда параметр включен, владелец канала может удалить все сообщения в нем.

-   **Allow users to edit their own messages (Разрешить пользователям редактировать свои сообщения).** Когда параметр включен, пользователи могут редактировать свои сообщения.

-   **Allow users to delete their own messages (Разрешить пользователям удалять свои сообщения).** Когда параметр включен, пользователи могут удалять свои сообщения.

-   **Allow users to chat privately (Разрешить пользователям приватный чат).** Когда параметр включен, пользователи могут открывать приватный чат, видимый только его участникам, а не всем людям в команде.
