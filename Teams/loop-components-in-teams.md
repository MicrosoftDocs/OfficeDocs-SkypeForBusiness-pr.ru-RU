# <a name="overview-of-loop-components-in-teams"></a>Обзор компонентов цикла в Teams

Компоненты цикла в чате Teams предлагают новый способ совместного создания идей, создания и принятия решений. Отправьте компонент, например таблицу, список задач или абзац, где все пользователи чата могут изменять встроенные изменения и просматривать изменения по мере их внесения. 

> [!Note]
> Компоненты цикла — это первая функция [приложения Microsoft Loop](https://www.microsoft.com/en-us/microsoft-loop), которая станет доступной в Teams. 

**Ускоряйте выполнение задач.** Вы можете истоки повестки дня, отслеживать действия группы или совместно делать заметки. Это всего лишь несколько сценариев, которые упрощаются с помощью компонентов цикла.

**Общий доступ к компонентам.** В этом выпуске вы можете совместно использовать компоненты цикла в разных чатах Teams. Получатели могут редактировать, где бы они ни находились, и мгновенно просматривать обновления независимо от того, где были внесены изменения.

**Начните в чате, выполните сборку оттуда.** Каждый компонент, создаваемый из чата Teams, автоматически сохраняется в файл в OneDrive. Таким образом, вы можете начать совместную работу в чате, а затем перейти к файлу на Office.com, где у вас есть больше визуального пространства для редактирования и вы можете добавить любое количество компонентов.

Сведения о параметрах администрирования для компонентов цикла в Teams см. [в разделе Управление компонентами цикла в SharePoint](/sharepoint/manage-loop-components).

## <a name="clients-and-platforms"></a>Клиенты и платформы

Доступно в приложениях Teams для Windows, Mac, Linux, iOS и Android.

## <a name="loop-components-and-fluid-files"></a>Компоненты цикла и файлы .fluid

Компоненты цикла, созданные в Teams, поддерживаются файлом .fluid (будет изменен на .loop в ближайшем будущем), хранящимся в OneDrive создателя. Файл в OneDrive означает, что пользователи могут создавать, обнаруживать и управлять компонентами цикла (флюид-файлами) так же легко, как и любой документ Office. 

## <a name="how-are-fluid-files-stored"></a>Как хранятся fluid-файлы?

Fluid-файлы отображаются в Office.com и OneDrive, например в разделах Последние и Рекомендуемые. Пользователи могут искать содержимое в fluid-файлах из Office.com и OneDrive. Файлы с расширением .fluid можно восстановить в предыдущих версиях из OneDrive. Для создания компонентов цикла участники чата должны иметь учетную запись OneDrive. Без действительной учетной записи OneDrive участники чата по-прежнему могут работать над компонентом, созданным другими пользователями, у которых есть действительная учетная запись OneDrive, но не могут создать собственную. 

Перемещение жидкостного файла из OneDrive на сайт SharePoint приведет к тому, что динамический компонент не загрузится в чате Teams.

## <a name="what-happens-if-the-owner-of-the-file-leaves-the-company"></a>Что произойдет, если владелец файла покинет компанию?

Политики хранения OneDrive применяются к fluid-файлам так же, как и к другому содержимому, созданному пользователем.

## <a name="how-are-fluid-files-shared"></a>Как предоставляются общие сведения о файлах .fluid?

Компоненты цикла можно вставлять в чат Teams или копировать из одного чата в другой. (Компоненты цикла пока не поддерживаются в каналах.) Они по умолчанию имеют существующие разрешения организации, но пользователи могут изменить разрешения перед отправкой, чтобы убедиться, что у всех есть доступ.

При открытии компонентов из чата Teams в Office.com в верхней части окна предоставляется общий доступ, аналогичный параметрам, предлагаемым для других документов Office.

## <a name="what-if-a-fluid-file-becomes-corrupted-or-damaged"></a>Что делать, если файл .fluid будет поврежден или поврежден?

Журнал версий позволяет просматривать, восстанавливать или копировать предыдущие версии файла.

## <a name="what-apps-can-open-and-edit-fluid-files"></a>Какие приложения могут открывать и редактировать fluid-файлы?

Fluid-файлы можно открывать только как ссылки в браузере, например Office.com, и как компоненты цикла в чате Teams. При скачивании их невозможно открыть повторно, не отправив их обратно в OneDrive или SharePoint.

## <a name="does-fluid-files-support-ediscovery"></a>Поддерживают ли fluid-файлы обнаружение электронных данных?

Fluid-файлы доступны для обнаружения, но имеют ограниченную поддержку рабочих процессов обнаружения электронных данных. В настоящее время fluid-файлы хранятся в OneDrive создателя и доступны для поиска и сбора как в eDiscovery (стандартный), так и в eDiscovery (премиум). Однако они не отображаются в предварительной версии, и формат экспорта для проверки не используется существующими средствами. Чтобы просмотреть экспортируемую информацию, отправьте его в любой oneDrive. При желании можно временно отключить эти возможности, как описано в разделе [Управление параметрами](/sharepoint/manage-loop-components#settings-management) .

## <a name="if-loop-is-disabled-from-the-admin-switch-what-will-the-user-experience-be"></a>Если параметр "Цикл" отключен в переключении администратора, то каким будет взаимодействие с пользователем?

Если отключить эти возможности, как описано в разделе [Управление параметрами, будут применены](/sharepoint/manage-loop-components#settings-management) следующие изменения интерфейса:

- Точка входа создания и вставки в сообщениях Teams будет скрыта. Пользователи не смогут создавать новые fluid-файлы.
- Существующие сообщения, которые ранее отображались в виде интерактивного компонента цикла, вместо этого будут отображаться в виде гиперссылки "Компонент цикла". Интерактивное содержимое в Teams не будет отображаться.
- Когда пользователь щелкает гиперссылку "Компонент цикла" или переходит к файлу .fluid в OneDrive для бизнеса и щелкает, чтобы открыть его, он откроет файл в отдельной вкладке браузера. Конечные пользователи по-прежнему смогут редактировать файл.

## <a name="known-issues"></a>Известные проблемы

- Если для разрешений файла клиента по умолчанию задано значение *Конкретные люди* (только те люди, которые указывает пользователь), для копирования ссылки на компонент Цикл и ее вставки в другой чат отправителю необходимо использовать диалоговое окно разрешений и добавить получателей в параметр Определенные люди, чтобы предоставить доступ должным образом.
- Если для разрешений на файл клиента по умолчанию задано значение *Конкретные люди* (только те люди, которых указывает пользователь), для создания динамического компонента в групповом чате с более чем 20 участниками отправителю потребуется вручную выбрать параметры разрешений для компонента.
- Поиск компонентов цикла в поиске Teams вернет ссылку на компонент в office.com, а не само сообщение чата.
- Компоненты цикла отключены в федеративных чатах.
- Гости не смогут совместно работать над динамическим компонентом, к которому им предоставлен общий доступ через ссылку Company Share Link. Задайте разрешения для **Люди в настоящее время в этом чате**, чтобы поделиться компонентами с гостями.
- Компоненты цикла не поддерживаются в каналах Teams.
- Компоненты цикла в чате не загружаются только в том случае, если файл был перемещен в другую библиотеку. Если файл перемещен в другую папку, он будет продолжать загружаться в чате.
