# <a name="what-are-cloud-auto-attendants"></a>Что представляют собой облачные автосекретари?

Телефонная система предоставляет автоматические ассистенты, которые можно использовать, чтобы разрешить внешним и внутренним вызывающим абонентам перемещаться по системе меню для поиска и размещения или передачи звонков пользователям или подразделениям в Организации.
  
Автосекретарь чаще всего является узлом системы, предоставляя вызывающему абоненту серию голосовых запросов или звуковых файлов, а не оператора человека. При обращении пользователей к номеру, связанному с автосекретарем, вы можете перенаправить вызов пользователю или найти другого пользователя в вашей организации, а затем подключиться к нему. Они могут выражать варианты выбора и взаимодействовать с системой меню с помощью телефонной клавиатуры (DTMF) или распознавания речи. Варианты, которые они делают, могут также перенаправлять вызов на другой автоматический секретарь или в очередь звонков.
  
Чтобы настроить автосекретарь для телефонной системы, перейдите в раздел Настройка автосекретаря в [облаке](create-a-phone-system-auto-attendant.md).
  
У автосекретаря в облаке есть следующие возможности:
  
- Воспроизведение корпоративных или информационных приветствий.
- Поддержка настраиваемых корпоративных меню. Вы можете настроить эти меню, чтобы они имели несколько уровней.
- Он обеспечивает поиск в каталоге, позволяющий пользователям, которые намерены, вызываемым для поиска имени в каталоге Организации.
- Она позволяет любому человеку, который будет звонить, или оставлять сообщение для человека в вашей организации.
- Она поддерживает несколько языков для запросов, преобразования текста в речь и распознавания речи.
- Она поддерживает указание праздников и рабочих часов.
- Он поддерживает передачу вызова оператору, другим пользователям, очереди звонков и автосекретарей.
- Она поддерживает общую голосовую почту для вызывающих абонентов для выхода из сообщения в Организации.

> [!NOTE]
> Эта статья относится как в Microsoft Teams, так и в Skype для бизнеса Online.

## <a name="getting-started"></a>Начало работы

Перед началом работы с функцией автосекретаря необходимо запомнить следующие моменты.

- Для связи с учетной записью ресурса необходимо наличие автосекретаря. Дополнительные сведения об учетных записях ресурсов содержатся [в разделе Управление учетными записями ресурсов в Teams](manage-resource-accounts.md) . <!-- You can either use an existing resource account or create a new resource account as you set up your auto attendant. -->
- При назначении автосекретаря номера телефона строго говорят, что он назначается учетной записи ресурса, связанной с этим автосекретарем. Это позволяет получить доступ к автосекретарь нескольким номерам телефонов. Чаще всего для учетной записи ресурса используется лицензия на виртуальную систему пользователей с бесплатной платной телефонной системой. Эта лицензия обеспечивает возможности телефонной системы на телефонные номера на уровне Организации, а также позволяет создавать автоматические ассистенты и очереди звонков.

> [!NOTE]
> Номера служб прямой маршрутизации для автосекретаря и очередей звонков поддерживаются только для пользователей Microsoft Teams и для агентов звонков.

   > [!TIP]
   > Чтобы перенаправить вызов оператору или меню, которое является сетевым пользователем с лицензией на **телефонную систему** , вам потребуется включить ее учетную запись для корпоративного голосовой связи или назначить для них планы звонков. Ознакомьтесь [с Разназначением лицензий на надстройки Microsoft Teams](teams-add-on-licensing/assign-teams-add-on-licenses.md). Вы также можете использовать Windows PowerShell. Например, выполните указанные ниже действия.  `Set-CsUser -identity "Amos Marble" -EnterpriseVoiceEnabled $true`
  
- Для получения и использования бесплатных номеров служб для автосекретарей необходимо настроить кредиты на связь. Чтобы сделать это, посмотрите, [что такое](what-are-communications-credits.md) баллы для обмена информацией и хотите [настроить кредиты на связь в Организации](set-up-communications-credits-for-your-organization.md).

    > [!IMPORTANT]
    > Номера телефонов пользователей (абонентов) нельзя назначить автосекретарям  можно использовать платные или бесплатные номера телефонов службы.

- Полная автоматическая система автосекретаря обычно включает несколько автосекретарей.
- Для автосекретарей на уровне ввода можно применить более одного номера телефона.
- Номера автосекретарей и очередей звонков в системе целиком должны быть только в том случае, если они будут совершать исходящие звонки PSTN.
  
## <a name="feature-overview"></a>Обзор функций

### <a name="searching-for-users"></a>Поиск пользователей

Имя набора номера — это функция автосекретаря, которая также известна как поиск в каталоге. Она позволяет людям, вызывающим автосекретарь, использовать голосовую почту (распознавание речи) или отвечать на них, чтобы ввести полное или частичное имя для поиска в каталоге компании, найти нужного пользователя, а затем передать ему звонок. Пользователи, которых вы хотите набрать, **не обязаны иметь номер телефона или назначенные им планы звонков, но у них должны быть лицензии на телефонную систему, если они подключены к Интернету, или Корпоративная голосовая связь включена для пользователей Skype для бизнеса Server**. Звонить по имени даже смогут находить и переносить звонки пользователям Microsoft Teams, которые размещаются в разных странах или регионах для работы с несколькими национальными организациями. Имея необходимые предварительные требования, вы явным образом включаете функцию набрать имя в автоматическом ассистенте.

Расширение набора номера — это функция автосекретаря, которая также является частью поиска в каталоге. Она позволяет людям, которые вызывают автосекретарь, использовать голосовую почту (распознавание речи) или отклики на клавиатуре телефона (DTMF) для ввода расширения телефона пользователя, которому они пытаются получить доступ, а затем передавали им звонок. Пользователи, которым нужно  **присвоить доступ, не обязаны иметь номер телефона или назначенные им планы звонков, но они должны иметь лицензию на телефонную систему, если они подключены к Интернету, или Корпоративная голосовая связь включена для пользователей Skype для бизнеса Server**. Кроме того, вам потребуется настроить для пользователей соответствующую абонентскую схему. С помощью добавочного номера вы даже сможете находить и передавать звонки пользователям Microsoft Teams, которые размещаются в разных странах или регионах для нескольких национальных организаций. Если вы предоставили необходимые условия, вы явным образом включаете режим набора номера в автосекретарях.

#### <a name="maximum-directory-size"></a>Максимальный размер справочника

Количество пользователей Active Directory, исходящих по имени и набору номера, может быть недоступно при поиске с помощью вызывающего абонента для определенного человека. Вызывающий объект может вводить частичные или полные имена (FirstName + LastName, а также LastName + FirstName), но для них требуется полный добавочный номер. Максимально допустимый размер списка имен, поддерживаемый одним автосекретарем, — 80 000 пользователей.
  
|Тип ввода|Формат поиска|Максимальное число пользователей в организации|
|:-----|:-----|:-----|
|DTMF (ввод с клавиатурной панели) |Частичный  <br/> Имя + фамилия  <br/> Фамилия + Имя |Без ограничений  |
|Распознавание речи (голосовой ввод) |FirstName  <br/> Полю  <br/> Имя + фамилия  <br/> Фамилия + Имя  | Пользователи 80 000 |

> [!NOTE]
> Если вы используете функцию "набрать номер" с распознаванием речи, но служба каталогов Active Directory вашей организации больше 80 000 пользователей и вы не ограничили область набора номера с помощью телефонной клавиатуры, для вызывающих абонентов по-прежнему будет использоваться абонентская клавиатура, и для других сценариев будут доступны голосовые входные данные. С помощью функции "область набора номера" можно сузить список имен, чтобы они были доступны, изменяя область набора номера по имени для определенного автоматического ассистента.
  
### <a name="dial-by-name---keypad-dtmf-entry"></a>Звонок по имени  ввод с клавиатурной панели телефона (DTMF)
Абоненты в Skype могут использовать для доступа к ним имя пользователя, указав полное или частичное имя человека, которому он пытается связаться. Существуют различные форматы, которые можно использовать при вводе имени.

При поиске в каталоге организации пользователи могут использовать клавишу "0" (нуль) для обозначения места между именем и фамилией или фамилией. После ввода имени вам будет предложено завершить ввод с клавиатуры с помощью клавиши #. Например, после ввода имени человека, которому вы пытаетесь обратиться, нажмите клавишу #. Если найдено несколько имен, вызывающему абоненту будет предоставлен список имен для выбора.
  
Поддерживаются следующие форматы поиска имен в организации с использованием клавиатурной панели телефона.
  
|Формат имени|Тип поиска|Пример|Результат поиска|
|:-----|:-----|:-----|:-----|
|Имя + фамилия |Полный  |Иван0Макаренко# |Иван Макаренко |
|Фамилия + Имя |Полный |Макаренко0Иван#  |Иван Макаренко |
|FirstName  |Полный   |Иван#   |Нажмите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите 2 для связи с пользователем "Иван Марченко" |
|Полю |Полный |Макаренко#  |Нажмите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите 2 для связи с пользователем "Мария Макаренко" |
|Имя или фамилия |Частичный |Ма# |Нажмите 1 для связи с пользователем "Мария Макаренко"  <br/> Нажмите 2 для связи с пользователем "Мария Петрова"  <br/> Нажмите 3 для связи с пользователем "Иван Марченко" |
|Имя + Фамилия |Частичный |Amos0Mar # |Нажмите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите 2 для связи с пользователем "Иван Марченко" |
|Фамилия + Имя |Частичный |Ма0Ив# |Нажмите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите 2 для связи с пользователем "Иван Марченко" |

Есть несколько специальных символов, которые используются для поиска людей с помощью телефонной клавиатуры. Например, пользователю будет предложено использовать клавишу решетки (#), а с помощью клавиши нуль (0) — для пробела между именами. Нажатие клавиши со звездочкой (*) будет повторять список совпадающих имен с пользователем.
  
|Специальный знак на клавиатуре телефона|Описание|
|:-----|:-----|
|#   |Завершение ввода имени. |
|до   |Пробел между именами. |
|*    |Повторное воспроизведение списка имен пользователей, соответствующих условиям поиска. |

#### <a name="dial-by-name---name-recognition-with-speech"></a>Звонок по имени  распознавание имен при голосовом вводе

Пользователи могут искать другие пользователи в своей организации с помощью голосовых команд (распознавание речи). Кроме того, они могут обращаться к любому пользователю в службе каталогов Active Directory, указав имя человека, которого они пытаются найти. Использование голосового ввода может распознавать имена в разных форматах, включая имя, фамилия, имя + фамилия или фамилию и имя.
  
Вы можете включить функцию распознавания речи для автосекретаря, но ввод телефонной клавиатуры (DTMF) не отключен. Клавиша на телефонную клавиатуру можно использовать в любое время даже в том случае, если для автосекретаря включена функция распознавания речи.
  
Как и в случае с телефонной клавиатурой, если найдено несколько имен, абонент звонит на список имен для выбора.
  
Вызывающие имена могут называться в следующих форматах:
  
|Имя с голосовым распознаванием|Тип поиска|Пример|Результат поиска|
|:-----|:-----|:-----|:-----|
|Имя + фамилия |Полный |Иван Макаренко |Иван Макаренко |
|Фамилия + Имя |Полный  |Макаренко Иван |Иван Макаренко |
|FirstName |Полный |Иван |Нажмите или произнесите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите или произнесите 2 для связи с пользователем "Иван Петров" |
|Полю |Полный |Макаренко |Нажмите или произнесите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите или произнесите 2 для связи с пользователем "Сергей Макаренко" |

> [!NOTE]
> Для нового пользователя может потребоваться до 36 часов, чтобы в каталоге было указано имя для набора номера с помощью функции распознавания речи из-за задержки репликации Active Directory.

### <a name="the-operator-option"></a>Параметр оператора

При необходимости автосекретарь можно настроить так, чтобы вызывающий абонент мог поговорить с человеком.
  
Клавиша 0 и голосовая команда "оператор" направляет вызов назначенному оператору по умолчанию. Это так для всех языков, поддерживаемых для распознавания речи. Вы также можете использовать **Параметры меню "Настройка** " для задания настраиваемого значения оператора.
  
Для оператора можно установить следующие значения:
  
- Пользователь Microsoft Teams или сервер Skype для бизнеса, на котором включена поддержка корпоративной голосовой связи.
- Другой настроенный в организации автосекретарь.
- Все существующие очереди звонков, настроенные в вашей организации. Чтобы узнать больше о очередях звонков, ознакомьтесь со статьей [Создание очереди звонков в облаке](/SkypeForBusiness/what-is-phone-system-in-office-365/create-a-phone-system-call-queue).

### <a name="business-hours-and-call-handling"></a>Рабочие часы и обработка звонков

Рабочие часы можно настроить для каждого автосекретаря. Если рабочие часы не настроены, по умолчанию устанавливается круглосуточный график работы без выходных дней. Рабочие часы можно настроить с разбивкой на время в течение дня, а все часы, не заданные в качестве рабочих часов, рассматриваются после-часов. Вы можете настроить различные параметры обработки входящих звонков и различные приветствия (это необязательно) для рабочих часов и в течение часа.
  
У каждого автосекретаря есть несколько возможных вариантов обработки звонков.
  
- Звонок можно отключить после воспроизведения приветствия.
- Также вы можете:
  - Перенаправить вызов пользователю Microsoft Teams, у которого есть лицензия на **телефонную систему** , для которой включена Корпоративная голосовая связь или абоненты, которым они назначены. Ее можно настроить так, что звонки вызывающего абонента будут отправляться в голосовую почту. Для этого выберите **Сотрудник вашей компании** и установите для этого абонента режим переадресации звонков непосредственно на голосовую почту.

  - Перенаправить звонок в очередь звонков. Чтобы узнать больше о очередях звонков, ознакомьтесь со статьей [Создание очереди звонков в облаке](/SkypeForBusiness/what-is-phone-system-in-office-365/create-a-phone-system-call-queue).

  - Перенаправить звонок на другой автоматический секретарь.

Эти параметры выражаются автоматическим ассистентом при воспроизведении приглашений меню. Например: "нажмите 1 для продажи, нажмите 2 для обслуживания. Чтобы проговаривать с оператором, в любое время нажмите клавишу 0.

### <a name="set-menu-options"></a>Настройка параметров меню

С помощью автосекретарей в облаке вы можете создавать запросы меню ("нажмите 1 для продаж, нажмите 2 для обслуживания) и настройте параметры меню для направления звонков на основе выбранных пользователем параметров. Параметры меню для автосекретаря. Вы можете предоставить организациям ряд вариантов, направленных вызывающим абонентам быстрее, без полагаться на оператор человека для обработки входящих звонков. Вывод меню может быть создан с помощью функции преобразования текста в речь или при загрузке записанного звукового файла. Распознавание речи принимает голосовые команды для бесплатной навигации, но абоненты также могут использовать клавиатуру телефона для навигации по меню.
  
Клавиши от 0 до 9 могут быть назначены для клавиш с набором номера в автосекретарях с помощью центра администрирования Skype для бизнеса. Вы можете создать различные наборы параметров меню для рабочих часов и после часов, а также включить или отключить набор номера по имени в **параметрах меню**. Ключи можно сопоставить для передачи звонков:
  
- Оператор (по умолчанию назначается клавиша 0). Однако его можно переназначить другим ключом или удалить из меню.
- Очередь звонков.
- Другой автосекретарь. Многоуровневые меню можно настроить, указав **параметр меню** в одном автоматическом секретарь для другого автосекретаря с собственным набором параметров меню, которые называются автосекретарем "вложенный".
- Пользователь Microsoft Teams, у которого есть лицензия на **телефонную систему** , для которой включена Корпоративная голосовая связь или абоненты, которым они назначены. Ее можно настроить так, что звонки вызывающего абонента будут отправляться в голосовую почту. Для этого выберите **Сотрудник вашей компании** и установите для этого абонента режим переадресации звонков непосредственно на голосовую почту.
  
Если функция распознавания речи включена, имя каждого пункта меню становится ключевым словом распознавания речи. Например, вызывающие объекты могут сказать "один", чтобы выбрать параметр меню, сопоставленный с клавишей 1, или указать "продажи", чтобы выбрать тот же вариант меню "продажи".
  
Чтобы настроить автосекретарь и параметры меню, перейдите [в раздел Настройка автоматического секретаря облачных](create-a-phone-system-auto-attendant.md)функций.
  
### <a name="assigning-phone-numbers-for-an-auto-attendant"></a>Назначение телефонных номеров для автосекретаря

Вы можете назначить номер Услуги Microsoft, прямой номер маршрутизации или гибридный номер для учетной записи связанного ресурса автоматического ассистента (или нескольких учетных записей ресурсов, если требуется более одного номера телефона). Дополнительные сведения приведены в разделе [планирование прямой маршрутизации](direct-routing-plan.md) .

Чтобы получить доступ к вашим услугам, ознакомьтесь со статьей [получение телефонных номеров служб для Skype для бизнеса и Microsoft Teams](/microsoftteams/getting-service-phone-numbers) или, если вы хотите перенести и существующий номер Услуги, в разделе [Перенос номеров телефонов в Teams](phone-number-calling-plans/transfer-phone-numbers-to-teams.md).
