# <a name="implement-quality-of-service-qos-in-microsoft-teams-clients"></a>Реализация качества обслуживания (QoS) в клиентах Microsoft Teams

Вы можете использовать качество обслуживания на основе политик (QoS) в групповая политика, чтобы задать диапазон исходных портов для предопределенного значения DSCP в клиенте Teams. Диапазоны портов, указанные в следующей таблице, являются отправной точкой для создания политики для каждой рабочей нагрузки.

*Таблица 1. Рекомендуемые начальные диапазоны портов*

|Тип медиатрафика| Диапазон портов источника клиента  |Протокол|Значение DSCP|Класс DSCP|
|:--- |:--- |:--- |:--- |:--- |
|Звук| 50,000–50,019|TCP/UDP|46|Беспрепятственная переадресация (EF)|
|Видео| 50,020–50,039|TCP/UDP|34|Гарантированная переадресация (AF41)|
|Приложение / Разделение экрана| 50,040–50,059|TCP/UDP|18|Гарантированная пересылка (AF21)|
| | | | | |

По возможности настройте параметры качества обслуживания на основе политик в групповая политика объекта. Следующие действия очень похожи на настройку диапазонов портов и политику качества обслуживания для клиентов на [Skype для бизнеса Server, которая](/SkypeForBusiness/manage/network-management/qos/configuring-port-ranges-for-your-skype-clients#configure-quality-of-service-policies-for-clients-running-on-windows-10) содержит некоторые дополнительные сведения, которые могут не потребоваться.

Чтобы создать политику качества звука для присоединенных к домену Windows 10 компьютеров, сначала войдите на компьютер, на котором групповая политика management. Откройте групповая политика управления (нажмите кнопку "Пуск", выберите пункт "Администрирование", а затем нажмите кнопку групповая политика Management), а затем выполните следующие действия:

1. В групповая политика management найдите контейнер, в котором должна быть создана новая политика. Например, если все клиентские компьютеры находятся в подразделении с именем **Clients**, в подразделении клиентов должна быть создана новая политика.

1. Щелкните правой кнопкой мыши соответствующий контейнер и выберите команду **"Создать объект групповой политики" в** этом домене и свяжите его здесь.

1. В **диалоговом** окне "Новый объект групповой политики" введите имя нового объекта групповая политика в поле "Имя", а затем нажмите кнопку "ОК **"**.

1. Щелкните правой кнопкой мыши созданную политику и выберите команду "Изменить **"**.

1. В редакторе групповая политика управления разверните "Конфигурация компьютера **",** разверните "Параметры **Windows**", щелкните правой кнопкой мыши **QoS** на основе политик и выберите команду "Создать **новую политику"**.

1. В **диалоговом окне "Качество** обслуживания на основе политик" на открывающей странице введите имя новой политики в поле **"Имя** ". Выберите **"Указать значение DSCP"** и задайте значение **46**. Не **устанавливайте** флажок "Указать исходящий коэффициент регулирования", а затем нажмите кнопку **"Далее"**.

1. На следующей странице выберите только  приложения с этим именем исполняемого файла и **введите** Teams.exeи нажмите кнопку **"Далее"**. Этот параметр указывает политике определять приоритеты только соответствующего трафика от клиента Teams.

1. На третьей странице убедитесь, что выбраны любой исходный **и** конечный **IP-адреса** , а затем нажмите кнопку **"Далее"**. Эти два параметра гарантируют, что управление пакетами будет выполняться независимо от того, какой компьютер (IP-адрес) отправил пакеты и какой компьютер (IP-адрес) получит эти пакеты.

1. На четвертой странице выберите **TCP и UDP** в раскрывающемся списке выберите протокол, к которому применяется эта политика качества обслуживания. TCP (протокол управления передачей) и UDP (протокол датаграммы пользователя) — это два наиболее часто используемых сетевых протокола.

1. Под заголовком **"Укажите номер исходного порта"** выберите **"Из этого исходного порта или диапазона"**. В соответствующем текстовом поле введите диапазон портов, зарезервированный для передачи звука. Например, если вы зарезервированы порты от 50 000 до портов 50019 для аудиопотока, введите диапазон портов в следующем формате: **50 000:50019**. Нажмите **Готово**.

1. Повторите шаги 5–10, чтобы создать политики для общего доступа к видео и приложениям и рабочим столам, заменив соответствующие значения на шагах 6 и 10.

Созданные политики вступает в силу только после обновления групповая политика на клиентских компьютерах. Хотя групповая политика периодически обновляется самостоятельно, вы можете принудительно выполнить немедленное обновление, выполнив следующие действия.

1. На каждом компьютере, для которого требуется обновить групповая политика, откройте командную строку от имени администратора (*запуск от имени администратора*).

1. В командной строке введите

   ```console
   gpupdate /force
   ```

## <a name="verify-dscp-markings-in-the-group-policy-object"></a>Проверка разметки DSCP в групповая политика объекта

Чтобы убедиться, что значения из групповая политика заданы, выполните следующие действия.

1. Откройте командную строку от имени администратора (*запуск от имени администратора*).

1. В командной строке введите

   ```console
   gpresult /R > gp.txt
   ```

   При этом будет создан отчет о примененных объектах групповой политики и он будет отправляться в текстовый файл *с именемgp.txt*.

   Для более удобочитаемого HTML-отчета *с именемgp.html* введите следующую команду:

   ```console
   gpresult /H gp.html
   ```

1. В созданном файле найдите заголовок **Applied групповая политика Objects** и убедитесь, что имена объектов групповая политика, созданных ранее, находятся в списке примененных политик.

1. Откройте редактор реестра и перейдите к

   HKEY\_LOCAL\_MACHINE\\Software\\Policies\\Microsoft\\Windows\\QoS

   Проверьте значения для записей реестра, перечисленных в таблице 2.

   *Таблица 2. Значения для записей реестра Windows для QoS*

   |          Имя          |  Тип  |    Данные     |
   |         :---:          | :---:  |    :---:    |
   |    Application Name    | REG_SZ |  Teams.exe  |
   |       DSCP Value       | REG_SZ |     46      |
   |        Local IP        | REG_SZ |     \*      |
   | Local IP Prefix Length | REG_SZ |     \*      |
   |       Local Port       | REG_SZ | 50000-50019 |
   |        Протокол        | REG_SZ |     \*      |
   |       Remote IP        | REG_SZ |     \*      |
   |    Префикс удаленного IP-адреса    | REG_SZ |     \*      |
   |      Remote Port       | REG_SZ |     \*      |
   |     Throttle Rate      | REG_SZ |     -1      |
   | | | |

1. Убедитесь, что для имени приложения указано правильное значение для клиента, который вы используете, и убедитесь, что значения DSCP и локальный порт отражают параметры в объекте групповая политика.
