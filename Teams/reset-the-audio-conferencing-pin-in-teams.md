# <a name="reset-the-audio-conferencing-pin-in-microsoft-teams"></a>Сброс ПИН-кода аудиоконференций в Microsoft Teams

ПИН-код — это число, которое создается для каждого пользователя Microsoft Teams, которому включена аудиоконференция. ПИН-коды аудиоконференций используют организаторы собраний для подтверждения статуса организатора собрания и запуска собрания по телефону. Если для начала собрания используется приложение Microsoft Teams, ПИН-код не требуется. Если пользователь забыл свой ПИН-код, и ему не удается найти сообщение электронной почты, отправленное ему при предоставлении возможности участия в аудиоконференциях, администратор может выполнить сброс его ПИН-кода, или же он может изменить свой ПИН-код самостоятельно.
  
Собрания можно начать, когда пользователь, аутентификация, присоединяется с помощью приложения Microsoft Teams или когда организатор присоединяется с помощью ПИН-кода по телефону. Если для участия в телефонном собрании требуется ПИН-код, то все пользователи, подключившиеся к этому собранию по телефону, перенаправляются в зал ожидания и могут прослушивать музыку до начала собрания. Если организатор собрания не запрашивает ПИН-код при открытии собрания по телефону, то при подключении к собранию участникам не потребуется вводить ПИН-код.

## <a name="reset-a-users-pin"></a>Сброс ПИН-кода пользователя

![Значок с логотипом Microsoft Teams](media/teams-logo-30x30.png) **Использование центра администрирования Microsoft Teams**

1. В области навигации слева **щелкните**"Пользователи" и выберите пользователя в списке доступных пользователей.

2. Нажмите **кнопку "Изменить".**

3. В **области "Аудиоконференция" нажмите** кнопку **"Сбросить ПИН-код".**

4. Нажмите **кнопку "Сброс".**

## <a name="what-else-should-you-know-about-pins"></a>Что еще необходимо знать о ПИН-кодах?

- В целях безопасности ПИН-код отображается администратору только один раз при сбросе ПИН-кода. После сброса ПИН-кода он будет указан в списке **********.
    
- По умолчанию автоматическое отправка сообщений электронной почты пользователям включено, и пользователи получают электронное письмо со своим ПИН-кодом, когда им будет включена аудиоконференция или сброс ПИН-кода. Но если вы отключили автоматическое отправку электронных писем, пользователю не будет отправлено электронное письмо с сбросом ПИН-кода, и вам придется вручную отправлять пользователю сведения о ПИН-коде.
    
- Когда собрание начинается, к нему автоматически присоединяются все пользователи в "вестибюле". Например, если два участника попытаются присоединиться к собранию до его начала, они будут помещаются в "вестибюль" и будут прослушивать музыку в удержании, а когда организатор собрания присоединяется по телефону с помощью ПИН-кода, начинается собрание, и к нему присоединяются участники из "ветвя".
    
- По умолчанию анонимные звонки не могут начать собрание.
    
- Если пользователю включена аудиоконференция, по умолчанию ему отправляются сообщения электронной почты, включающие данные для аудиоконференции и ПИН-код. У пользователя должен быть почтовый ящик Microsoft 365 или Office 365, так как при сбросе ПИН-кода новый ПИН-код отправляется пользователю по электронной почте на его основной SMTP-адрес (псевдоним), установленный для пользователя.
    
- При настройке аудиоконференций вы задаете цифры, которые должны быть включены в ПИН-коды в вашей организации. ПИН-коды могут иметь от 4 до 12 цифр . Значение по умолчанию — 5. При изменении длины ПИН-кода новое значение применяется только к новым ПИН-кодам и не применяется к ПИН-кодам для существующих пользователей, которым разрешено участие в аудиоконференциях. См. [Настройка длины PIN-кода для собраний с аудиоконференцией](Set-the-PIN-length-for-Audio-Conferencing-meetings-in-teams.md).
    
- По умолчанию для электронной почты устанавливается основной SMTP-адрес Microsoft 365 или Office 365 пользователя. Вы можете отправить сообщение электронной почты на адрес, который не является Microsoft 365 или office 365, кроме Office 365, например Hotmail или MSN-адрес. Используемый по умолчанию адрес электронной почты можно изменить с помощью Windows PowerShell. Это полезно, если у пользователей нет почтового ящика Exchange в Microsoft 365 или Office 365.
