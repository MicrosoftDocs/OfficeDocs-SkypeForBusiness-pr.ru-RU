|----|-----------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1  | Убедитесь, что ваша среда включает все необходимые компоненты Teams | Teams зависит от других платформ для создания сквозного решения для совместной работы. Обратитесь к ИТ-командам, чтобы убедиться, что вы развернули и правильно настроите Exchange, SharePoint Online и OneDrive для бизнеса. | | [Взаимодействие SharePoint Online и OneDrive для бизнеса с Microsoft Teams](sharepoint-onedrive-interact.md) <br/><br/>[Взаимодействие Microsoft Teams и Exchange](exchange-teams-interact.md) |
| 2  | Проверка включения Teams для клиента | Teams включен по умолчанию для всех организаций. Проверьте **страницу &** служб на странице Центр администрирования Microsoft 365, чтобы убедиться, что Teams включен для вашей организации, и при необходимости включите ее. | | [Настройка Microsoft Teams в Microsoft 365 или Office 365](office-365-set-up.md) |
| 3  | Настройка ролей и разрешений | Teams поддерживает два типа ролей: "Участник" и "Владелец". <br/><br/>После добавления участника в команду владелец также может повысить уровень участника до роли владельца. Рекомендуется назначить каждой команде по крайней мере двух владельцев. <br/><br/>По умолчанию все сотрудники организации, у которых есть почтовый ящик, размещенный в Exchange Online, могут создать команду. Пользователю, создавшего новую команду, автоматически предоставляется роль владельца для этой команды. <br/><br/>При необходимости можно настроить параметры группы Microsoft 365, чтобы разрешить только определенным пользователям создавать новые команды. | | [Назначение ролей и разрешений в Microsoft Teams](assign-roles-permissions.md) <br/><br/>[Группы Microsoft 365 и Microsoft Teams](office-365-groups.md) <br/><br/>[Управление пользователями, которые могут создавать Группы Microsoft 365](https://support.office.com/article/Manage-who-can-create-Office-365-Groups-4c46c8cb-17d0-44b5-9776-005fced8e618) |
| 4  | Настройка параметров Teams на уровне клиента | Некоторые параметры Teams можно настроить на уровне клиента. Пользователи, которые включены в Teams, наследуют эти параметры из конфигурации клиента:<ul><li>Общий</li><li>Интеграция электронной почты</li><li>Приложения</li><li>Настраиваемое облачное хранилище</li><li>Звонки и собрания</li><li>Обмен сообщениями</li></ul>| | [Управление параметрами Microsoft Teams в организации](enable-features-office-365.md) |
| 5  | НЕОБЯЗАТЕЛЬНО. Настройка гостевого доступа | Гостевой доступ в Teams используется для совместной работы с пользователями за пределами организации, предоставив им доступ к командам и каналам. Гостевой доступ — это параметр на уровне клиента в Teams. Она по умолчанию отключена. <br/>Включите гостевой доступ и настройте гостевые параметры на уровне клиента, если ваша организация планирует использовать эту функцию. | | [Гостевой доступ в Microsoft Teams](guest-access.md) |
| 6  | НЕОБЯЗАТЕЛЬНО. Настройка политики именования Teams | Teams использует политики именования для Группы Microsoft 365 при создании или изменении имен команд. <br/><br/>По умолчанию при создании команды пользователем ограничения именования не применяются. <br/><br/>Если необходимо применить правила для имен команд, настройте Группы Microsoft 365 именования, которые применяются к вашей организации. Можно задать обязательные префиксы и суффиксы, а также указать запрещенные слова. | | [Планирование групп Microsoft 365 при создании команд в Microsoft Teams](plan-office-365-groups.md) <br/><br/>[Группы Microsoft 365 именования](https://support.office.com/article/Office-365-Groups-naming-policy-6ceca4d3-cad1-4532-9f0f-d469dfbbb552) |
| 7  | Настройка Exchange для домена SMTP Teams | Teams использует Exchange Online для отправки уведомлений участникам команды с помощью домена SMTP email.teams.microsoft.com , когда они были добавлены или удалены. <br/><br/>Не забудьте добавить этот домен SMTP в список принятых доменов в инфраструктуре Exchange. | | [Создание списков надежных отправителей в Exchange](/microsoft-365/security/office-365-security/create-safe-sender-lists-in-office-365?view=o365-worldwide) |

| 9  | Назначение лицензий пользователям | Назначение лицензий пользователям для таких функций, как аудиоконференции, телефонная система и планы звонков | | [Назначение лицензий на надстройки Microsoft Teams](teams-add-on-licensing/assign-teams-add-on-licenses.md)|
