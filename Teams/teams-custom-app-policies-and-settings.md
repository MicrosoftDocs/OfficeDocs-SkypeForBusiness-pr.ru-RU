# <a name="manage-custom-app-policies-and-settings-in-microsoft-teams"></a>Управление пользовательскими политиками и параметрами приложений в Microsoft Teams

> [!INCLUDE [feature coming soon](includes/new-feature-coming-soon-article.md)]

Как администратора можно использовать пользовательское приложение политики и параметры, чтобы контролировать, кто в организации настраиваемые приложения можно публиковать для групп Майкрософт. "Администраторы" решить, какие пользователи могут загружать настраиваемые приложения и рабочая группа администраторов и владельцев можно указать, разрешено ли определенными группами в организации настраиваемые приложения для добавления к ним.  

## <a name="overview-of-custom-apps"></a>Общие сведения о настраиваемых приложениях

Пользователи могут добавлять пользовательское приложение для групп, загрузив пакет приложения (в ZIP-файл) непосредственно к группе или в контексте личные. Это отличается от как приложения, добавленных с помощью команды хранилище приложений. Добавление настраиваемого приложения путем загрузки пакета приложения, также известной как sideloading позволяет тестирования приложения после его разработки, прежде чем его можно быть широко распространены. Это также позволяет создать приложение только для внутреннего использования и вместе со ваша группа без отправки в каталог приложений группы в хранилище приложений группами.

![Загрузка настраиваемые приложения](media/teams-custom-app-policy-and-settings-upload-app.png)

## <a name="custom-app-policy-and-settings"></a>Пользовательское приложение политики и параметры

Три компонента определить, является ли пользователь отправлять пользовательское приложение для группы, обеспечивая настраиваемый контроль пользователей, которые можно добавлять настраиваемые приложения группы. и который групп настраиваемые приложения можно добавить в:

- [Пользовательское приложение политики пользователя](#user-custom-app-policy)
- [Параметр пользовательское приложение группы](#team-custom-app-setting)
- [Параметр пользовательское приложение масштабе организации](#org-wide-custom-app-setting)

Эти параметры не влияют на возможность блокировать приложения сторонних производителей.  

### <a name="user-custom-app-policy"></a>Пользовательское приложение политики пользователя

В рамках [программы установки политики в отношении приложений](teams-app-setup-policies.md)администраторы могут использовать параметр политики, **Разрешить отправку настраиваемые приложения**, для управления ли пользователь, настраиваемые приложения можно публиковать группами.
 
Если этот параметр отключен:

- Пользователь не может загрузить пользовательское приложение любой группы в вашей организации или в личный контекст.
- Пользователь может взаимодействовать с настраиваемые приложения, в зависимости от параметра пользовательское приложение масштабе организации.

Если этот параметр включен:

- Пользователя можно передать настраиваемые приложения, группами, которые позволяют и группам, для которых они являются владельцев, в зависимости от параметра пользовательское приложение масштабе организации.
- Пользователь отправлял настраиваемые приложения личный контекст. 
- Пользователь может взаимодействовать с настраиваемые приложения, в зависимости от параметра пользовательское приложение масштабе организации.

Можно изменить параметры в политике установки глобальные приложения, чтобы включить приложений, которые будут. Если вы хотите настроить группы для разных групп пользователей в вашей организации, создайте и назначьте одну или несколько политик программы установки пользовательское приложение.

#### <a name="set-a-user-custom-app-policy"></a>Задать настраиваемые приложения политику уровня пользователя

1. В левой области переходов центра администрирования группами Майкрософт, перейдите к **группам приложений** > **Настройка политик**.
2. Выберите **новую политику**.
3. Включить или отключить **Разрешить отправку настраиваемые приложения**.
4. Выберите другие параметры, которые необходимо для политики.
5. Нажмите кнопку **Сохранить**.

### <a name="team-custom-app-setting"></a>Параметр пользовательское приложение группы

Рабочая группа администраторов и владельцев можно управлять, позволяет ли группы для настраиваемых приложений для добавления в нее. Этот параметр, **Разрешить членам для загрузки пользовательских приложений**вместе с политике пользовательское приложение определяет пользователей, которые можно добавить настраиваемые приложения конкретной команде.
 
Если этот параметр отключен:

- Если эта политика пользовательское приложение позволяет владельцев группы можно добавить настраиваемые приложения.
- Участники группы, которые не владельцев группы нельзя добавлять настраиваемые приложения в группу.

Если этот параметр включен:

- Если эта политика пользовательское приложение позволяет это владельцев группы можно добавить настраиваемые приложения.
- Если эта политика пользовательское приложение позволяет это участников группы, которые не владельцев группы можно добавить настраиваемые приложения.

#### <a name="configure-the-team-custom-app-setting"></a>Настройка параметра пользовательское приложение группы

1. В группах, перейдите в группу, нажмите кнопку **Дополнительные параметры ˙˙˙** > **Управление группы**.
2. Нажмите кнопку **Параметры**и затем разверните узел **разрешения участников**.
3. Установите или снимите флажок **Разрешить участникам загрузки пользовательских приложений** .

    ![параметр пользовательское приложение группы](media/teams-custom-app-policy-and-settings-team.png)

### <a name="org-wide-custom-app-setting"></a>Параметр пользовательское приложение масштабе организации

Параметр пользовательское приложение масштабе организации **Разрешить взаимодействие с настраиваемые приложения**применяется ко всем пользователям в вашей организации и управляет ли они передача или взаимодействия с настраиваемые приложения. Этот параметр переопределяет пользователей и групп пользовательское приложение политики и установки. Он предназначен для использования в качестве главной переключатель во время события безопасности.

#### <a name="configure-the-org-wide-custom-app-setting"></a>Настройка параметра пользовательское приложение масштабе организации

1. В левой области переходов центра администрирования группами Майкрософт, перейдите к **группам приложений** > **политики разрешений**.
2. Нажмите кнопку **Параметры приложения в масштабе организации**.
3. В разделе **Custom apps**включить или отключить **Разрешить взаимодействие с настраиваемые приложения**.

    ![Параметр пользовательское приложение масштабе организации](media/teams-custom-app-policy-and-settings-org-wide.png)
