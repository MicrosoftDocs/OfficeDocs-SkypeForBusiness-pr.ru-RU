## <a name="project-vision"></a>Концепции проекта

Ваше представление — «общую картину» или окончательным конечного состояния, которая отвечает на вопрос «Почему делается этого проекта?» Идеальный концепции решает вашей организации бизнес-факторов и пользователь полезный перспектив, как показано в следующих примерах:

- **Организация бизнес-фактор**: стандартизация с группами Майкрософт коммерческим наших преобразования цифровых рабочего места и позволяет нам диска эффективность операций, исключить избыточные решения и сохранять USD5 млн.
- **Добавьте значение пользователя**: Microsoft групп (1) сохраняет времени с указанием в одном месте для проекта заметок, документов Office, участники группы, бесед и собраний; (2) упрощает связь с помощью централизованного список контактов и отслеживание сохраняемого чата для ошибочного найти, вложения электронной почты потеряны, хранение и доступ к файлами в одном месте позволяет быстро получить доступ к беседы и (3).

Помните о следующем обсуждений для уточнения ваше представление:

- Описание текущей бизнес-процесса

- Проблемы, связанные с с существующей бизнес-процесса

## <a name="project-goals"></a>Цели проекта

Выполнения задач определить результат должен и предоставляют возможность эффективной проекта. Цели может быть определен как _цели и основные результаты_ (OKRs) и меры успеха проекта может быть определен как _успеха ключевых индикаторов_ (KSIs). Очень важно Получение полного участия из заинтересованных лиц проекта в определение OKRs и KSIs, чтобы обеспечить надлежащую они считают смысле владения и выровнять эти показатели успеха определенный проект задачи. Цели должна включать сочетание успеха технические и ориентированных на пользователя.

- **OKRs** содержат целей, заданную вами в начале проекта и основные результаты измерения на определенном ритмичности (например, ежемесячно или ежеквартально). Просмотрев основные результаты, гарантирует вашей конечных результатов проекта, по расписанию, или определение и устранять проблемы, чтобы получить проект из отслеживания. OKRs обычно классифицируются как «достигнуть» или «не достигнуто.»
- **KSIs** показатель качества и успешности основные результаты и дополнения двоичные природе OKRs в деталях хороший и/или недопустимый результатов. При определении KSIs, рекомендуется использовать «конкретных, измерить, может быть назначен, реалистичной, связанные со временем» (смарт-) критериям:
  - Конкретных: определенной области для улучшения предназначенных
  - Измерить: количественной оценки или хотя бы предложить индикатор хода выполнения,
  - Может быть назначен: Укажите пользователей, которые будут выполнять
  - Реалистичные: состояний, что в реальной можно результаты достигнуть имеются доступные ресурсы
  - Связанные с времени: Укажите, когда было достигнуто за результаты

## <a name="timeline"></a>Временная шкала

Как область обновления пути, не забудьте установить временной шкалы для основных этапов (например, включение группы наряду с Скайп для бизнеса для всех пользователей) в дополнение к дате завершения. Определенный временной шкалы помогает дисковод team project направить согласованные конечное состояние и информирует прямо рабочих обратной расписания, помогая обеспечить соответствие плану. Рассмотрите временной шкалы, не слишком ускорения (где задач может быть пропущен) или слишком удаленных (где momentum могут быть потеряны). Учетные записи идеальная временной шкалы для:

- **Готовность продукта для соответствия требованиям и сценарий требования пользователей**: обращение к [Схема продуктов](https://aka.ms/O365Roadmap) для измерения при команды будет готов для вашей организации.
- **Обновление группы**: Определяет, вы будете ли позволяя группам или обновление пользователей путем обновления группы, которые могут повлиять на шкале общими сведениями по обновлению пути.
- **Факторы организации, такие как изменение закрепить, end финансовый год, жизненные циклы развертывания**: обсуждение и учетной записи для внутренних процессов, которые могут повлиять на ваше обновления временной шкалы.
- **Другие изменения, которые происходят в или в одно время**: объединение изменений необходимо учитывать или разделитель их в работе для облегчения положительное пользователя и взаимодействия с минимизировать воздействие на производительность.
- **Услуг**: подтверждение выделения ресурсов проекта заинтересованным сторонам для убедитесь, что вы собрали группы проекта недостаточно пропускной способности для выполнения всех необходимых задач.

Качестве опорной точки образец временной шкалы предоставляется для подготовки к обновлению, обновление и после обновления этапов [обновления специалистов пути](https://aka.ms/UpgradePro), который мы рекомендуем вам скорректируйте в соответствии с требованиями вашей организации.

После выполнения действий, описанных выше, необходимо обеспечить надежную основу для проекта. Продолжите [технической подготовки](https://aka.ms/SkypeToTeams-TechnicalReadiness) и [готовности организации](https://aka.ms/SkypeToTeams-UserReadiness) планирования.
