# <a name="cloud-video-interop-for-microsoft-teams"></a>Взаимодействие Microsoft Teams с облачными видеослужбами

Cloud Video Interop (CVI) — это решение корпорации Майкрософт, которое позволяет сторонним комнатам собраний (телепреенции) и личным видео устройствам (VTCs) присоединяться к собраниям Microsoft Teams собраниям.
 
Благодаря Microsoft Teams вы получаете доступ к содержимому из Интернета на собраниях, которые включают аудио- и видеосвязь, а также общий доступ к содержимому. Эта возможность может быть как на компьютере, так и в веб-клиенте, а также на многих партнерских устройствах, которые интегрируются с Microsoft Teams. Однако многие клиенты уже сделали инвестиции в видеоконференции и личные видеосоединения, обновление которых может стоить дороже. Cloud Video Interop — это простое решение, которое позволяет продолжать использовать существующие решения до тех пор, пока вы не будете готовы к обновлению.

Благодаря облачному видеоконферентиву Microsoft Teams обеспечивает родной для всех участников — в комнатах собраний или в Teams клиентах.

### <a name="is-cloud-video-interop-for-me"></a>Есть ли у меня облачное видео с interop?

Cloud Video Interop предоставляет промежуточную службу при переходе на полное Microsoft Teams решение с Teams конечными точками. Предоставляемые службы должны быть частью пути миграции.

Cloud Video Interop предназначен для клиентов, которые соответствуют следующим условиям:

- Развертывание крупных устройств в комнате для собраний и личных видео устройств (более 50 устройств), которые не имеют возможности прямой интеграции с Microsoft Teams
- Поддерживаются одним из наших партнеров Cloud Video Interop
- Хотите сохранить стоимость инвестиций в их текущие устройства комнаты собраний и личные видео устройства во время миграции на Microsoft Teams решения

Хотя Cloud Video Interop — это отличное промежуточное решение, мы рекомендуем нашим клиентам в долгосрочной перспективе иконки Teams собраний, например системы комнат Teams комнат. 

### <a name="office-365-us-government-and-third-party-services"></a>Office 365 Государственные и сторонние службы

Office 365 позволяет интегрировать сторонние приложения с сайтами SharePoint Online, Skype для бизнеса, Teams, Office, приложениями, включенными в Приложения Microsoft 365 для предприятий (например, Word, Excel, PowerPoint и Outlook) и Outlook Web App. Кроме того, Office 365 поддерживает интеграцию со сторонними поставщиками услуг. Эти сторонние приложения и службы могут включать хранение, передачу и обработку данных клиентов организации в сторонних системах, которые не находятся в инфраструктуре Office 365 и поэтому не обрабатываются Office 365 соответствия требованиям и защите данных. **При оценке надлежащего использования этих служб в организации рекомендуется и просмотреть заявления о конфиденциальности и соответствии требованиям, предоставленные третьими лицами.**

### <a name="cloud-video-interop-overview"></a>Обзор cloud Video Interop

Cloud Video Interop — это стороннее решение, которое предлагается нашими партнерами для обеспечения взаимодействия между существующими решениями для видеоконференций и личных видеоконферентов локально и Microsoft Teams.

Решения, предлагаемые нашими партнерами, состоят из компонентов, которые можно развернуть полностью в облаке или частично или полностью локально.

## <a name="deploy-cloud-video-interop"></a>Развертывание cloud Video Interop

При развертывании решения Cloud Video Interop важно понимать, что вы развертываете партнерского решения. На следующей схеме перечислены общие действия, которые необходимо предпринять для развертывания Cloud Video Interop.

![Схема, описывающие развертывание CVI в организации.](media/deploying-cvi.png)

### <a name="plan"></a>Планирование

На этапе планирования следует определить устройства, которые не будут заменяться на Teams, и найти партнера Cloud Video Interop, который поддерживает эти устройства.  

Кроме того, важно понимать, что для каждого пользователя, который будет планировать собрания, к которому будет присоединяться устройство Cloud Video Interop, потребуется лицензия. Обратите внимание, что точные требования к лицензированию можно получить у партнера Cloud Video Interop. Прежде чем приступить к развертыванию, убедитесь, что это понятно.

### <a name="configure"></a>Настройка

Партнер, выбранный для развертывания CVI, предоставит вам полный документ о развертывании, состоящий из всех шагов, необходимых для успешного развертывания в организации. Это будут порты брандмауэра и диапазоны IP-адресов, изменения конфигурации ваших устройств и другие параметры, которые нужно изменить.

### <a name="provision"></a>Предоставление  

На этапе подготовка вы назначите лицензии соответствующим пользователям в соответствии с руководством по настройке партнера. Вам также потребуется пройти процесс согласия Azure, чтобы предоставить партнеру доступ к вашей Teams среде. Дополнительные сведения о процессе согласия Azure [см.](/azure/active-directory/develop/v2-permissions-and-consent) в платформа удостоверений Майкрософт и согласия в конечной точке Azure.

### <a name="schedule"></a>Расписание

После включения облачной видеосвязи все собрания, запланированные с помощью надстройки Teams Meeting Interop для Outlook или клиента Teams, будут автоматически добавлены в собрание Teams, чтобы устройства, совместимые с Cloud Video Interop, могли присоединяться к этим собраниям.

### <a name="join"></a>Join

В зависимости от партнерских решений существует несколько способов присоединиться к собранию Cloud Video Interop. Точные сценарии перейти к собранию предоставляет ваш партнер Cloud Video Interop. Ниже приведены некоторые примеры.

- IVR (интерактивный голосовой ответ) 
  - Вы можете позвонить в IVR партнера с помощью tenantkey@domain.
  - Когда вы находитесь в партнерской организации IVR, вам будет предложено ввести VTC conferenceId, который затем подключит вас к Teams собранию.
- Прямой набор номера 
  - Вы можете напрямую позвонить на Teams, не взаимодействуя с IVR партнера, используя функцию прямого набора номера, используя полную строку клиента. VTC ConferenceId@domain.
- Телефонная связь с одним касанием 
  - Если у вас встроенная Teams, вы можете использовать функции набора одним касанием, предлагаемые партнером (без необходимости ввести строку набора).

## <a name="manage-cloud-video-interop"></a>Управление облачным видео interop

После развертывания Cloud Video Interop вы можете управлять устройствами с помощью решений, предоставленных нашими партнерами. Каждый партнер предоставит вам административный интерфейс, который будет включать как управление лицензией, так и устройствами. 

Отчеты также доступны непосредственно в интерфейсе администрирования партнера. Для получения дополнительных сведений о возможностях отчетов обратитесь к вашему партнеру. 

### <a name="troubleshooting-cloud-video-interop"></a>Устранение неполадок между облачными видеосвязи

Cloud Video Interop — это служба, предоставляемая партнером. Если у вас возникают проблемы, сначала подключите устройство, на котором установлен клиент Teams, и подключите его к сегменту устройства Cloud Video Interop, который вызывает проблемы. 

Если Teams в этом сегменте и вы также придерживались всех рекомендаций по сети и настройке, предоставленных партнером, вам потребуется обратиться к партнеру для дальнейшего устранения неполадок. 

## <a name="powershell-for-cloud-video-interop"></a>PowerShell для cloud Video Interop

Чтобы (частично) автоматизировать развертывание Cloud Video Interop, можно (частично) автоматизировать развертывание PowerShell.

- **Get-CsTeamsVideoInteropServicepolicy**: Корпорация Майкрософт предоставляет предварительно заранее сконструированы политики для каждого из поддерживаемых партнеров, которые позволяют назначать партнеров, которые будут использовать для cloud Video Interop.<br>С помощью этого cmdlet можно определить предварительно заранее сконструированы политики, которые можно использовать в вашей организации. Эту политику можно назначить одному или несколько пользователям с помощью Grant-CsTeamsVideoInteropServicePolicy управления.
- **Grant-CsTeamsVideoInteropServicePolicy:** этот cmdlet позволяет назначить предварительно построенную политику для использования в организации или назначить ее определенным пользователям.
- **New-CsVideoInteropServiceProvider**: используйте этот cmdlet, чтобы указать сведения о поддерживаемом партнере CVI, который будет использовать ваша организация.
- **Set-CsVideoInteropServiceProvider**: используйте этот cmdlet для обновления сведений о поддерживаемом партнере CVI, которую использует ваша организация.
- **Get-CsVideoInteropServiceProvider**: Используйте этот cmdlet, чтобы получить все поставщики, настроенные для использования в организации.
- **Remove-CsVideoInteropServiceProvider**: используйте этот cmdlet, чтобы удалить все сведения о поставщике, который больше не используется в вашей организации.
