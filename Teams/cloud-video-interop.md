# <a name="cloud-video-interop-for-microsoft-teams"></a>Взаимодействие Microsoft Teams с облачными видеослужбами

Cloud Video Interop (CVI) — это стороннее решение майкрософт, которое позволяет сторонним конференц-залам (телеприсутствия) и личным видеоустройствам (VTC) присоединяться к собраниям Microsoft Teams.
 
С помощью Microsoft Teams вы получаете широкие возможности для совместной работы с контентом в Интернете на собраниях, в том числе аудио, видео и общий доступ к содержимому. Этим можно воспользоваться через настольный и веб-клиент, а также через множество партнерских устройств, которые изначально интегрируются с Microsoft Teams. Тем не менее, многие клиенты уже инвестировали в видеотелеконференции и личные устройства видеосвязи, которые могут быть дорогостоящими для обновления. Cloud Video Interop — это простое решение, позволяющее использовать существующие решения до тех пор, пока не будете готовы к обновлению.

Благодаря облачному видеообязаяния Microsoft Teams обеспечивает собственный интерфейс собраний для всех участников — в комнатах для собраний или в клиентах Teams.

### <a name="is-cloud-video-interop-for-me"></a>Это облачное видео-взаимодействие для меня?

Cloud Video Interop предоставляет промежуточную службу при переходе на полное собственное решение Microsoft Teams с использованием конечных точек Teams. Предоставленная служба должна быть частью пути миграции.

Cloud Video Interop предназначен для клиентов, которые соответствуют следующим критериям:

- Большое развертывание устройств конференц-залов и личных видеоустройств (более 50 устройств), которые не могут быть напрямую интегрированы с Microsoft Teams.
- Поддерживаются одним из наших партнеров по облачному видео-взаимодействия
- Хотите сохранить ценность своих инвестиций в текущие устройства комнаты собраний и личные видеоустройства во время миграции на собственное решение Microsoft Teams

Хотя облачное видео-взаимодействие предоставляет отличное промежуточное решение, мы рекомендуем нашим клиентам в долгосрочной перспективе ознакомиться с нашими собственными решениями для собраний Teams, такими как Системы комнат Teams. 

### <a name="office-365-us-government-and-third-party-services"></a>Office 365 государственных и сторонних служб США

Office 365 позволяет интегрировать сторонние приложения в сайты SharePoint Online, Skype для бизнеса, Teams, приложения Office, включенные в Приложения Microsoft 365 для предприятий (например, Word, Excel, PowerPoint и Outlook), а также Outlook Web App. Кроме того, Office 365 поддерживает интеграцию со сторонними поставщиками услуг. Эти сторонние приложения и службы могут включать хранение, передачу и обработку данных клиентов вашей организации в сторонних системах, которые находятся за пределами инфраструктуры Office 365 и, следовательно, не охватываются обязательствами Office 365 по обеспечению соответствия и защите данных. **Рекомендуется ознакомиться с заявлениями о конфиденциальности и соответствии, предоставленными третьими лицами, при оценке надлежащего использования этих служб в вашей организации.**

### <a name="cloud-video-interop-overview"></a>Общие сведения о облачном видеообзоре

Cloud Video Interop — это сторонняя служба, которая предоставляется нашими партнерами для обеспечения взаимодействия между существующими решениями видеоконференций и личных видеоустройств в локальной среде и Microsoft Teams.

Решения, предлагаемые нашими партнерами, состоят из компонентов, которые могут быть развернуты либо полностью на основе облака, либо частично или полностью локально.

## <a name="deploy-cloud-video-interop"></a>Развертывание облачного видео-взаимодействия

При развертывании решения облачного видеообязаимодейза важно понимать, что вы развертываете партнерское решение. На следующей схеме приведены общие действия, которые необходимо предпринять для развертывания облачного видеообязаимода.

![Схема, описывающая развертывание CVI в организации.](media/deploying-cvi.png)

### <a name="plan"></a>Планирование

На этапе планирования следует определить устройства, которые вы не будете заменять собственным устройством Teams, и найти партнера по облачному видеообязаяву, который может поддерживать эти устройства.  

Кроме того, важно понимать, что вам потребуется лицензия для каждого пользователя, который будет планировать собрания, к которым требуется присоединиться устройство с поддержкой облачного видеообъединений. Обратите внимание, что точные требования к лицензированию можно получить у партнера по облачному видеообязаяниям. Убедитесь, что это ясно перед началом развертывания.

### <a name="configure"></a>Настройка

Партнер, выбранный для развертывания CVI, предоставит вам полный документ о развертывании, который содержит все шаги, необходимые для успешного развертывания в организации. Сюда входят порты и диапазоны IP-адресов брандмауэра, изменения конфигурации для устройств и другие параметры, которые необходимо изменить.

### <a name="provision"></a>Предоставление  

На этапе подготовки вы назначите лицензии соответствующим пользователям в соответствии с руководством по настройке партнера. Вам также потребуется пройти процесс предоставления согласия Azure, чтобы предоставить партнеру доступ к вашей среде Teams. Дополнительные сведения о процессе согласия Azure см. [в разделе Разрешения и согласие в конечной точке платформа удостоверений Майкрософт](/azure/active-directory/develop/v2-permissions-and-consent).

### <a name="schedule"></a>Расписание

После включения для пользователя облачного видеообъединений все собрания, запланированные с помощью надстройки для собраний Teams для Outlook или клиента Teams, будут автоматически добавлять в собрание Teams соответствующие дополнительные сведения, чтобы устройства, совместимые с облачным видеоизобращением, могли присоединиться к этим собраниям.

### <a name="join"></a>Join

В зависимости от решения партнера существует несколько способов присоединиться к собранию с поддержкой облачного видеообязаимода. Точные сценарии присоединения к собранию будут предоставлены вашим партнером по облачному видеообязаяву. Ниже приведены некоторые примеры.

- IVR (интерактивный голосовой ответ) 
  - С помощью tenantkey@domain вы можете набирать номер IVR партнера.
  - Когда вы находитесь в партнерском IVR, вам будет предложено ввести идентификатор конференции VTC, который затем подключит вас к собранию Teams.
- Прямой набор 
  - Вы можете напрямую войти в собрание Teams, не взаимодействуя с IVR партнера с помощью функции прямого набора, используя полную строку tenantkey. ConferenceId@domain VTC.
- Набор одним касанием 
  - Если у вас есть интегрированная комната Teams, вы можете использовать возможности набора номера одним касанием, предлагаемые вашим партнером (без необходимости вводить строку набора).

## <a name="manage-cloud-video-interop"></a>Управление взаимодействием с облачным видео

После развертывания Cloud Video Interop вы можете управлять устройствами с помощью решений, предоставляемых нашими партнерами. Каждый партнер предоставит вам административный интерфейс, который будет включать управление лицензиями и устройствами. 

Отчеты также доступны непосредственно из интерфейса администрирования партнера. Чтобы получить дополнительные сведения о возможностях создания отчетов, обратитесь к партнеру по своему выбору. 

### <a name="troubleshooting-cloud-video-interop"></a>Устранение неполадок с облачным видео-взаимодействием

Cloud Video Interop — это предоставляемая партнером служба. Если возникают проблемы, первым шагом является подключение устройства, на котором установлен клиент Teams, и его подключение к тому же сегменту, что и устройство облачного видеообъединения, которое вызывает проблемы. 

Если Teams работает правильно в этом сегменте и вы также выполнили все рекомендации по настройке сети и конфигурации, предоставленные партнером, вам потребуется обратиться к партнеру для дальнейшего устранения неполадок. 

## <a name="powershell-for-cloud-video-interop"></a>Взаимодействие видео в PowerShell для облака

Следующие командлеты PowerShell доступны для (частично) автоматизации развертывания облачного видеообязаимода.

- **Get-CsTeamsVideoInteropServicepolicy**. Корпорация Майкрософт предоставляет предварительно созданные политики для каждого из наших поддерживаемых партнеров, которые позволяют указать, какие партнеры следует использовать для облачного видеообзора.<br>Этот командлет позволяет определить предварительно созданные политики, которые можно использовать в организации. Эту политику можно назначить одному или нескольким пользователям, используя командлет Grant-CsTeamsVideoInteropServicePolicy.
- **Grant-CsTeamsVideoInteropServicePolicy**. Этот командлет позволяет назначить предварительно сконструированную политику для использования в организации или назначить ее определенным пользователям.
- **New-CsVideoInteropServiceProvider**: используйте этот командлет, чтобы указать сведения о поддерживаемом партнере CVI, который ваша организация хотела бы использовать.
- **Set-CsVideoInteropServiceProvider**: используйте этот командлет для обновления сведений о поддерживаемом партнере CVI, используемом вашей организацией.
- **Get-CsVideoInteropServiceProvider**: используйте этот командлет, чтобы получить все поставщики, настроенные для использования в организации.
- **Remove-CsVideoInteropServiceProvider**: используйте этот командлет, чтобы удалить все сведения о поставщике, который ваша организация больше не использует.
