# <a name="teams-approvals-app-availability"></a>Доступность приложения "Утверждения Teams"

[!INCLUDE [preview-feature](includes/preview-feature.md)]

Приложение "Утверждения" доступно как личное приложение для всех пользователей Microsoft Teams.
Приложение "Утверждения" представляет собой простой способ обеспечить аудит, соответствие требованиям, ответственность и рабочий процесс для структурированных и неструктурированных утверждений в Teams.

 ![приложение "Утверждения"](media/approvals-selection.png)

Пользователи могут закрепить приложение "Утверждения", чтобы сохранить его в меню.

 ![приложение "Утверждения" с параметром закрепления](media/approvalApp-pin.png)

Первое утверждение, созданное с помощью приложения "Утверждения", активирует его подготовка в среде cdS по умолчанию. Утверждения, созданные с помощью приложения "Утверждения", будут храниться в среде CDS по умолчанию.

В этой статье описаны требования и роли приложений "Утверждение".

## <a name="required-permissions-and-licenses"></a>Необходимые разрешения и лицензии

Для использования приложения "Утверждения" необходимо разрешение на следующие элементы:

- Разрешения на создание базы данных CDS (Майкрософт).

- Учетная запись на [flow.microsoft.com](https://flow.microsoft.com/)

- Роль администратора в целевой среде.

- Лицензия на [Power Automate,](https://docs.microsoft.com/power-automate/get-started-approvals)Office 365 или Dynamics 365.

## <a name="storage-with-cds"></a>Хранилище с компакт-дисками

Общая модель данных (CDM) — это язык общих данных, используемый в бизнес-приложениях и аналитических приложениях на cdS. Он содержит набор стандартизированных, extensible data schemas, опубликованных корпорацией Майкрософт и нашими партнерами, которые обеспечивают согласованность данных и их значение в приложениях и бизнес-процессах. Узнайте больше об общей [модели данных microsoft Power Platform.](https://docs.microsoft.com/power-automate/get-started-approvals)

Подробнее о рабочий [процессе "Утверждение".](https://docs.microsoft.com/power-automate/modern-approvals)

## <a name="approvals-teams-app-permissions"></a>Разрешения приложений Teams утверждения

Приложение Approvals Teams позволяет получить доступ к следующим возможностям:

- Получение сообщений и данных, которые вы предоставляете ему.

- Отправка вам сообщений и уведомлений.

- Отрисуйте личные приложения и диалоговое окно без заглавной области, предоставленной Teams.

- Доступ к данным профиля, таким как ваше имя, адрес электронной почты, название компании и предпочтительный язык.

- Получение сообщений и данных, которые участники группы предоставляют в канале.

- Отправка сообщений и уведомлений в канале.

- Доступ к сведениям о группе:
  - название команды
  - список каналов
  - список членов группы (имена и адреса электронной почты участника группы).

- Используйте сведения группы, чтобы связаться с ним.

## <a name="disable-the-approvals-app"></a>Отключение приложения "Утверждения"

Приложение "Утверждения" доступно по умолчанию. Вы можете отключить приложение в Центре администрирования Teams.

  1. Во sign in to the Teams admin center.

  2. Раз развернуть **приложения Teams** и выбрать **"Управление приложениями".**

  3. Найди приложение "Утверждения".

![Навигация в Центре администрирования с выделенной командой > "Управление приложениями"](media/manage-approval-apps.png)

  4. Выберите "Утверждения".

  5. Чтобы отключить приложение для организации, выберите его.

![подробные сведения о приложении "Утверждения"](media/approvals-details.png)

## <a name="retention-policy"></a>Политика хранения

Утверждения, созданные с помощью приложения "Утверждения", хранятся в среде CDS по умолчанию, которая в настоящее время не поддерживает резервное копирование. Узнайте больше о том, как восстанавливать и восстанавливать [среды — Microsoft \| Docs для Power Platform.](https://docs.microsoft.com/power-platform/admin/backup-restore-environments)

## <a name="auditing"></a>Аудит

Приложение Approvals регистрет события аудита в Центре безопасности и соответствия требованиям Microsoft 365. Вы можете просмотреть журнал аудита.

1. Перейдите на сайт соответствия требованиям M365.

2. Выберите раздел **"Аудит".**

3. Поиск действий в **рамках действий утверждения в Microsoft Teams.**

Вы можете найти следующие действия:

- Создание нового запроса на утверждение

- Просмотр сведений о запросе утверждения

- Утвержденный запрос на утверждение

- Отклоненный запрос на утверждение

- Отменен запрос на утверждение

- Общий запрос на утверждение

- Файл, вложенный в запрос на утверждение

- Повторно назначен запрос на утверждение

- В запрос на утверждение добавлена электронная подпись

Чтобы получить доступ к дополнительным утверждениям аудита в потоке, встроите и настройте аудит в среде по умолчанию для основных запросов утверждения, запросов на утверждение и ответов на утверждение. Операции создания, обновления и удаления являются событиями аудита для записей утверждения. Дополнительные сведения о данных аудита и действиях пользователей для обеспечения безопасности и соответствия требованиям [— Microsoft \| Docs power Platform.](https://docs.microsoft.com/power-platform/admin/audit-data-user-activity)

Аудит можно дополнительно настроить в Центре безопасности и соответствия требованиям [Microsoft 365.](https://support.office.com/article/go-to-the-office-365-security-compliance-center-7e696a40-b86b-4a20-afcc-559218b7b1b8?ui=en-US&rs=en-US&ad=US)

1. Чтобы использовать предварительно заранее задав их, вопишите в службу безопасности и соответствия требованиям Office 365.

2. Выберите **"Поиск & исследования".**

3. В журнале аудита выберите вкладку **действий Dynamics 365.**

Дополнительные сведения о регистрации действий в приложениях на основе модели и microsoft Dataverse — [Power Platform.](https://docs.microsoft.com/power-platform/admin/enable-use-comprehensive-auditing)

## <a name="security"></a>Безопасность

В приложении "Утверждение Teams" пользователи могут создавать новые утверждения и просматривать отправленные и полученные утверждения. Пользователи не смогут получить доступ к утверждениям, созданным другими пользователями, если они не являются либо ответив, либо просматривают запрос.

> [!Note]
> Пользователю будет предоставлена роль просмотра запроса, если он является частью чата или канала, где было создано утверждение. Они не могут выполнять действия с запросом, если им не была предоставлена эта роль при утверждении.
