# <a name="pstn-considerations-for-upgrading-to-teams-from-skype-for-business-on-premises"></a>Вопросы обновления локального приложения Skype для бизнеса до Teams через ПС

В этой статье описаны вопросы, которые следует учитывать при переходе на Teams.   


Кроме того, в следующих статьях описаны важные понятия обновления и принципы сосуществования.

- [Совместное сосуществование Teams и Skype для бизнеса](upgrade-to-teams-on-prem-coexistence.md)
- [Режимы сосуществования — ссылки](migration-interop-guidance-for-teams-with-skype.md)
- [Взаимодействие с клиентом Teams и соответствие режимам сосуществования](teams-client-experience-and-conformance-to-coexistence-modes.md)


 > [!NOTE]
 > - Использование телефонной системы с Teams поддерживается только в том случае, если пользователь работает в режиме TeamsOnly.  Если пользователь работает в режиме "Острова", телефонная система поддерживается только в Skype для бизнеса. 
 > - Любые параметры переадселирования звонков, группы звонков и делегирования из Skype для бизнеса не переносились, и их необходимо воссоздать для Teams.
 > - Общие сведения о облачных голосовых службах Microsoft Teams и о том, как решить, какое из решений Microsoft правильно для вашей организации, см. в справке по планированию голосового [решения Teams.](cloud-voice-landing-page.md)


## <a name="pstn-calling-scenarios"></a>Сценарии вызовов через ПС

При переходе в режим TeamsOnly возможны четыре сценария звонков:

- [Пользователь Skype для бизнеса Online с планом звонков Майкрософт.](#from-skype-for-business-online-with-microsoft-calling-plans) После обновления у этого пользователя сохранится план Microsoft Calling.

- [Пользователь в Skype для бизнеса Online с](#from-skype-for-business-online-with-on-premises-voice) функцией локального голосового подключения через локальное приложение Skype для бизнеса или Cloud Connector Edition. Обновление пользователя до Teams должно быть согласовано с переносом пользователя на Прямую маршрутику, чтобы обеспечить пользователям TeamsOnly функциональность ННР.

- [Пользователь в локальной](#from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing)службе Skype для Корпоративная голосовая связь, который будет передвигаться на веб-сайт и хранить локальное подключение по STN.  Для переноса этого пользователя в Teams требуется переместить его локальное приложение Skype для бизнеса в облако и координировать переход с миграции пользователя на Direct Routing. 

- [Пользователь в локальной](#from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan)службе Skype для Корпоративная голосовая связь, который будет передвигаться на веб-сайт и использовать план Microsoft Calling.  Для переноса этого пользователя в Teams требуется переместить его локальное приложение Skype для бизнеса в облако и координировать перенос с помощью A) переноса номера телефона этого пользователя в план звонков Майкрософт или Б), назначая новый абонентский номер из доступных регионов.

Эта статья содержит общие сведения. Дополнительные сведения см. в прямой [маршрутке](direct-routing-landing-page.md) и планах звонков для [телефонной системы.](calling-plan-landing-page.md) 

## <a name="from-skype-for-business-online-with-microsoft-calling-plans"></a>Из Skype для бизнеса Online с планами звонков Майкрософт 

Это самый простой сценарий обновления, в котором участвуют голосовые голоса. 

1. Убедитесь, что пользователям назначена лицензия на Teams. По умолчанию при назначении лицензии Microsoft 365 или Office 365 в Teams включена, поэтому если вы ранее не отключили лицензию Teams, никаких действий не требуется.

2.  Если у пользователей уже есть план звонков Майкрософт с номером телефона, необходимо только назначить пользователю режим TeamsOnly в TeamsUpgradePolicy.  Перед назначением режима TeamsOnly входящие вызовы по STN будут переназначаться в клиент Skype для бизнеса пользователя. После обновления до режима TeamsOnly входящие вызовы по STN будут понижимы в клиенте Teams пользователя.  

## <a name="from-skype-for-business-online-with-on-premises-voice"></a>Из Skype для бизнеса Online с локальной голосовой связи

В этом сценарии пользователь уже находится в Skype для бизнеса Online, но его подключение по СТАНД является своим подключением к интернету: с помощью Skype для бизнеса Server в гибридном режиме или Cloud Connector Edition. Миграция этих пользователей в режим TeamsOnly с функциями STN означает включение для них прямой маршрутации, при которой линии связи STN подключаются напрямую к службе прямой маршрутации в облаке через ваш собственный пограничный контроллер сеанса (SBC).

Ниже перечислены основные действия.  Шаги 1–4 указаны в списке предлагаемых последовательностей, но их можно сделать в любом порядке. Важно, чтобы все эти процессы были выполнены до шага 5.

1. Если вы задаете политику для клиента в одном из режимов Skype для бизнеса, обязательно укоройте существующих пользователей, явно назначив им режим "Острова", как описано выше.

2. Настройте клиент для прямой маршрутинга. См. [сводку по настройке direct Routing](#summary-of-per-tenant-configuration-of-direct-routing)для каждого клиента.

3. При желании настройте различные политики Teams для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и т. д.). Это можно сделать в любое время, но если вы хотите убедиться, что у пользователей должна быть правильная конфигурация при обновлении, лучше сделать это до обновления пользователя до режима TeamsOnly.

4. Подготовка выбранных пользователей к голосовой миграции: 
   - При необходимости назначьте лицензию Teams.  Предположим, что пользователь уже работает в локальной голосовой системе Skype для бизнеса Online, у него уже есть план 2 Skype для бизнеса и телефонная система Майкрософт. Оставьте оба этих плана включенными, включая лицензию Skype для бизнеса Online (план 2).  
   - Назначьте нужный OnlineVoiceRoutingPolicy. 

5. Обновление пользователя: эти действия должны быть согласованы. 

   - В Microsoft 365 или Office 365 обновите пользователя до режима TeamsOnly (Grant-CsTeamsUpgradePolicy).
   - В SBC настройте голосовую маршрутику, чтобы включить входящие звонки, отправив звонки на прямую маршрутацию, а не на сервер-посредник на локальном сервере-посреднике.


## <a name="from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing"></a>От локального сервера Skype для бизнеса с Корпоративная голосовая связь до прямой маршрутинга

В этом сценарии пользователь все еще находится в локальной службе Skype для бизнеса, а его подключение по ННР также является локальной. Миграция этих пользователей в режим TeamsOnly с функциями STN означает включение прямой маршрутации и перемещение пользователя в облако. 
 
Ниже перечислены основные действия.  Шаги 1–5 указаны в списке предлагаемых последовательностей, но их можно сделать в любом порядке. Важно, чтобы все эти процессы были выполнены до шага 6.

1. Если вы назначите политику для клиента одному из режимов Skype для бизнеса, не забудьте охарактеризовить существующих пользователей Островов, явно назначив им режим "Острова", как описано выше.

2. Если вы еще не сделали этого, настройте гибридную конфигурацию Skype для бизнеса в [организации.](https://docs.microsoft.com/SkypeForBusiness/hybrid/configure-hybrid-connectivity)

3. Настройте клиент для прямой маршрутинга. См. [сводку по настройке direct Routing](#summary-of-per-tenant-configuration-of-direct-routing)для каждого клиента.

4. При желании настройте различные политики Teams для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и т. д.). Это можно сделать в любое время, но если вы хотите убедиться, что у пользователей должна быть правильная конфигурация при обновлении, лучше сделать это до обновления до TeamsOnly.

5. При необходимости назначьте лицензии на Microsoft 365 или Office 365.  У пользователя должны быть как Teams, так и Skype для бизнеса Online (план 2), а также телефонная система. Если skype для бизнеса Online (план 2) отключен, снова включим его.  

6. Обновление пользователя: эти действия должны быть согласованы. 

   - Используя локальное приложение Skype для бизнеса, запустите Move-CsUser с переключателем -MoveToTeams. Если вы используете версию Skype для бизнеса Server, которая не поддерживает переключатель -MoveToTeams, сначала запустите Move-CsUser а затем назначьте режим TeamsOnly в клиентской удаленной консоли PowerShell или консоли администратора Teams.

   - В SBC настройте голосовую маршрутику, чтобы включить входящие звонки, отправив звонки на прямую маршрутацию, а не на сервер-посредник на локальном сервере-посреднике. 

   - В Microsoft 365 или Office 365: назначьте соответствующий OnlineVoiceRoutingPolicy, чтобы включить исходяющие звонки. 


## <a name="from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan"></a>От локального сервера Skype для бизнеса и Корпоративная голосовая связь до плана звонков Майкрософт

В этом сценарии пользователь все еще находится в локальной службе Skype для бизнеса, а его подключение по ННР также является локальной. Перенос этих пользователей в режим TeamsOnly с функциями STN означает перемещение пользователя в облако и перенос номера из старого оператора в план Microsoft Calling или назначение пользователю нового номера. 

Ниже перечислены основные действия.Шаги 1–5 указаны в списке предлагаемых последовательностей, но их можно сделать в любом порядке. Важно, чтобы все эти процессы были выполнены до шага 6. 

1. Если вы назначите политику для клиента одному из режимов Skype для бизнеса, не забудьте охарактеризовить существующих пользователей Островов, явно назначив им режим "Острова", как описано выше. 

2. Если вы еще не сделали этого, настройте гибридную конфигурацию Skype для бизнеса в [организации.](https://docs.microsoft.com/SkypeForBusiness/hybrid/configure-hybrid-connectivity) 

3. При желании настройте различные политики Teams для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и т. д.). Это можно сделать в любое время, но если вы хотите убедиться, что у пользователей должна быть правильная конфигурация при обновлении, лучше сделать это до обновления до TeamsOnly. 

4. При необходимости назначьте лицензии на Microsoft 365 или Office 365.У пользователя должны быть как Teams, так и Skype для бизнеса Online (план 2), а также телефонная система. Если skype для бизнеса Online (план 2) отключен, снова включим его.  

5. Получите номера телефонов для своих пользователей. (Подробные сведения [см. в подмногании "Управление номерами телефонов для организации".)](https://docs.microsoft.com/MicrosoftTeams/manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization)

   - Если вы будете повторно использовать эти номера, отправьте запрос на перенос номеров вашему оператору.  
   - Кроме того, вы можете получить новые номера непосредственно у Корпорации Майкрософт. 

6. Обновите пользователя и при необходимости назначьте LineUri. Используя локальное приложение Skype для бизнеса, запустите Move-CsUser с переключателем -MoveToTeams.  

    - При переносе номеров в Корпорацию Майкрософт необходимо координировать сроки этой операции. 

    - Если вы используете новые номера от Майкрософт, необходимо изменить Для пользователя LineUri. Это необходимо сделать после того, как пользователь переместит пользователя в Интернет с помощью Set-CsOnlineVoiceUser.  

## <a name="summary-of-per-tenant-configuration-of-direct-routing"></a>Сводка настройки direct Routing для каждого клиента 

1. Убедитесь, что ваш граничный контроллер сеанса (SBC) поддерживается прямой маршрутией, просмотрев [этот список.](direct-routing-border-controllers.md) Кроме того, необходимо убедиться в том, что у вас правильная версия поквартовки.  

2. Соедините свою локальное SBC-связь со службой прямой маршрутинга Teams. Подробные сведения [см. в теме "Связь SBC с службой прямой маршрутки телефонной системы".](direct-routing-configure.md) 

3. Эта конфигурация фактически является зеркальным отражением локальной конфигурации. Конфигурация в Сети состоит из: 
   - OnlineVoiceRoutingPolicy (на основе локальной политики VoiceRoutingPolicy, если перенос пользователей из Skype для бизнеса Online, и VoicePolicy при переносе пользователей из локальной сети с Корпоративная голосовая связь).
   - Объекты OnlinePSTNUsage (с учетом использования локальной ННОП). 
   - Объекты OnlineVoiceRoute (на основе локального voiceRoute). 

Дополнительные сведения см. [в теме "Настройка прямой маршрутки".](direct-routing-configure.md) 

## <a name="manage-enterprisevoiceenabled-property-during-migration"></a>Управление свойством EnterpriseVoiceEnabled во время миграции 

Для использования прямой маршрутики или плана Microsoft Calling пользователь должен иметь EnterpriseVoiceEnabled=true в Azure AD, чтобы он мог пользоваться функциями ННР.  EnterpriseVoiceEnabled ("с поддержкой ОО") — это свойство (а не политика), которое существует как в локальном каталоге, так и в облаке. Значение в облаке имеет значение для Teams.  Точная логика того, как эта возможность становится истинной, зависит от следующего сценария: 

- Если пользователь работает с ОО в локальной системе Skype для бизнеса и ему назначена лицензия на телефонную систему, перед тем как переместить пользователя в облако с помощью Move-CsUser, он будет иметь доступ к учетной записи EV-enabled=true. 

- Если существующему пользователю TeamsOnly или Skype для бизнеса Online назначена лицензия на телефонную систему, по умолчанию не включена возможность использовать ОО.  Это также происходит, если пользователь из локальной сети перемещается в облако перед назначением лицензии на телефонную систему. В обоих случаях администратор должен указать следующий cmdlet: 

  ```PowerShell
  Set-CsUser -EnterpriseVoiceEnabled $True 
  ```
