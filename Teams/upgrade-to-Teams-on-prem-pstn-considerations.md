## <a name="pstn-calling-scenarios"></a>Сценарии КОММУТИРУЕМой телефонной сети

При переходе в режим TeamsOnly возможны четыре варианта звонка:

- [Пользователь в Skype для бизнеса Online с планом вызова Microsoft](#from-skype-for-business-online-with-microsoft-calling-plans). После обновления у этого пользователя будет по-прежнему содержаться план звонков по Microsoft.

- [Пользователь в Skype для бизнеса Online с локальной функцией голосовой связи](#from-skype-for-business-online-with-on-premises-voice) через Skype для бизнеса в локальной или облачной версии. Переход пользователя на Teams необходимо координировать с помощью миграции пользователя в направлении прямой маршрутизации, чтобы убедиться в том, что у пользователя TeamsOnly есть функция КТСОП.

- [Пользователь в локальной среде Skype для бизнеса с корпоративной голосовой](#from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing)связью, который будет переходить на Интернет и поддерживать локальную сеть PSTN.  Для миграции этого пользователя в Teams требуется переместить локальную учетную запись пользователя Skype для бизнеса в облако, а также перенести его с помощью миграции пользователей на прямую переадресацию. 

- [Пользователь в локальной среде Skype для бизнеса с корпоративной голосовой](#from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan)связью, который будет переноситься в Интернет и с помощью плана звонков Microsoft.  Для миграции этого пользователя в Teams требуется переместить локальную учетную запись пользователя Skype для бизнеса в облако, а также прикрепить ее с помощью A) к номеру телефона этого пользователя в плане Microsoft о звонке или B) назначить новый номер абонента из доступных регионов.

В этой статье приводятся общие сведения только о высоком уровне. Дополнительные сведения можно найти в разделе планы прямой маршрутизации и [звонков](calling-plan-landing-page.md)на [телефонную систему](direct-routing-landing-page.md) . 

## <a name="from-skype-for-business-online-with-microsoft-calling-plans"></a>Из Skype для бизнеса Online с тарифами Майкрософт 

Это простейший сценарий обновления, включающий голосовую связь. 

1. Убедитесь, что пользователям назначена лицензия на команды Teams. По умолчанию, когда вы назначаете лицензию Microsoft 365 или Office 365, команды включаются, поэтому, если вы ранее не отключили лицензию Teams, никаких действий не требуется.

2.  Если у пользователей уже есть план звонков Microsoft с номером телефона, необходимо изменить его только в том случае, если вы применяете режим TeamsOnly пользователя в TeamsUpgradePolicy.  Перед назначением режима TeamsOnly входящие звонки PSTN будут поступать в клиенте Skype для бизнеса пользователя. После перехода на режим TeamsOnly входящие звонки PSTN будут помещаются на клиенте Teams пользователя.  

## <a name="from-skype-for-business-online-with-on-premises-voice"></a>Из Skype для бизнеса Online с использованием локального голоса

В этом сценарии пользователь уже входит в Skype для бизнеса Online, но подключение по протоколу PSTN является локальным либо с помощью Skype для бизнеса Server в гибридном режиме, либо из облачного соединителя Edition. Миграция этих пользователей в режим TeamsOnly с помощью функции PSTN означает, что она позволяет осуществлять прямую маршрутизацию, в которой КОММУТИРУЕМые магистрали подключаются непосредственно к службе прямой маршрутизации в облаке с помощью локального контроллера (SBC).

Ниже приведены основные шаги.  Шаги 1-4 перечислены в предлагаемой последовательности, но их можно выполнить в любом порядке. Ключ состоит в том, что все эти действия должны быть выполнены перед шагом 5.

1. Если вы задаете политике на уровне клиента один из режимов Skype для бизнеса, не забудьте grandfather все существующие пользователи, явным образом назначив им режим "острова", как описано выше.

2. Настройте клиент для прямой маршрутизации. Ознакомьтесь [со сведениями о конфигурации "прямой маршрут" для каждого клиента](#summary-of-per-tenant-configuration-of-direct-routing).

3. При необходимости настройте различные политики групп для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и т. д.). Это можно сделать в любое время, но если вы хотите, чтобы пользователи правильно настроили конфигурацию, лучше сделать это перед тем, как пользователь будет обновлен до TeamsOnly Mode.

4. Подготовьте выбор пользователей для голосовой миграции. 
   - При необходимости назначьте лицензию Teams.  Предполагается, что пользователь уже работает в локальной голосовой системе Skype для бизнеса Online, у пользователя уже есть Skype для бизнеса (план 2), а также телефонная система Microsoft. Оставьте эти планы включенными, в том числе лицензией Skype для бизнеса Online 2.  
   - Назначьте нужное OnlineVoiceRoutingPolicy. 

5. Обновите пользователя: эти действия должны быть скоординированы. 

   - В Microsoft 365 или Office 365 обновите пользователя до режима TeamsOnly (Grant-CsTeamsUpgradePolicy).
   - На SBC Настройте голосовую почту, чтобы включить входящие звонки, отправив вызовы на прямой маршрут, а не на сервер локальных исправлений.


## <a name="from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing"></a>Из Skype для бизнеса Server локально с корпоративной голосовой связью для прямой маршрутизации

В этом сценарии пользователь по-прежнему находится в локальной среде Skype для бизнеса, и их подключение по протоколу PSTN также является локальным. Миграция этих пользователей в режим TeamsOnly с помощью функций PSTN означает, что они используются для прямой маршрутизации и последующего перемещения пользователя в облако. 
 
Ниже приведены основные шаги.  Шаги 1-5 перечислены в предлагаемой последовательности, но их можно выполнить в любом порядке. Ключ состоит в том, что все эти действия должны быть выполнены перед шагом 6.

1. Если вы задаете политике на уровне клиента один из режимов Skype для бизнеса, не забудьте grandfather существующих пользователей, явным образом назначив им режим, как описано выше.

2. Если вы еще не сделали этого, [Настройте структуру Организации для гибридного развертывания Skype для бизнеса](https://docs.microsoft.com/SkypeForBusiness/hybrid/configure-hybrid-connectivity).

3. Настройте клиент для прямой маршрутизации. Ознакомьтесь [со сведениями о конфигурации "прямой маршрут" для каждого клиента](#summary-of-per-tenant-configuration-of-direct-routing).

4. При необходимости настройте различные политики групп для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и т. д.). Это можно сделать в любое время, но если вы хотите, чтобы пользователи правильно настроили конфигурацию, лучше сделать это перед тем, как пользователь будет обновлен до TeamsOnly.

5. При необходимости назначьте лицензии Microsoft 365 или Office 365.  У пользователя должны быть как команды, так и Skype для бизнеса Online (план 2), а также телефонная система. Если вы отключили Skype для бизнеса Online (план 2), включите его заново.  

6. Обновите пользователя: эти действия должны быть скоординированы. 

   - С помощью локальных средств Skype для бизнеса выполните команду Move-CsUser с параметром-MoveToTeams. Если вы используете версию Skype для бизнеса Server, не поддерживающую параметр-MoveToTeams, сначала запустите Remove-CsUser и назначьте режим TeamsOnly в клиенте удаленной оболочки PowerShell или консоли администрирования Teams.

   - На SBC Настройте голосовую почту, чтобы включить входящие звонки, отправив вызовы на прямой маршрут, а не на сервер локальных исправлений. 

   - В Microsoft 365 или Office 365: Назначьте необходимые OnlineVoiceRoutingPolicy для исходящих звонков. 


## <a name="from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan"></a>С локального сервера Skype для бизнеса, с корпоративной голосовой связью в план звонков по Microsoft

В этом сценарии пользователь по-прежнему находится в локальной среде Skype для бизнеса, и их подключение по протоколу PSTN также является локальным. Миграция пользователей в режим TeamsOnly с помощью функции PSTN означает, что пользователь перемещается в облако и переносит его из старой перевозчика в план звонков Microsoft или назначает пользователю новый номер. 

Ниже приведены основные шаги.Шаги 1-5 перечислены в предлагаемой последовательности, но их можно выполнить в любом порядке. Ключ состоит в том, что все эти действия должны быть выполнены перед шагом 6. 

1. Если вы задаете политике на уровне клиента один из режимов Skype для бизнеса, не забудьте grandfather существующих пользователей, явным образом назначив им режим, как описано выше. 

2. Если вы еще не сделали этого, [Настройте структуру Организации для гибридного развертывания Skype для бизнеса](https://docs.microsoft.com/SkypeForBusiness/hybrid/configure-hybrid-connectivity). 

3. При необходимости настройте различные политики групп для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и т. д.). Это можно сделать в любое время, но если вы хотите, чтобы пользователи правильно настроили конфигурацию, лучше сделать это перед тем, как пользователь будет обновлен до TeamsOnly. 

4. При необходимости назначьте лицензии Microsoft 365 или Office 365.У пользователя должны быть как команды, так и Skype для бизнеса Online (план 2), а также телефонная система. Если вы отключили Skype для бизнеса Online (план 2), включите его заново.  

5. Получение телефонных номеров для пользователей. (Дополнительные сведения см. [в разделе Управление номерами телефонов для Организации](https://docs.microsoft.com/MicrosoftTeams/manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization).)

   - Если вы будете использовать номера заново, отправьте заявку на перенос на свой перевозчик.  
   - Кроме того, вы можете получать новые номера прямо из Microsoft. 

6. Обновите пользователя и при необходимости назначьте LineUri. С помощью локальных средств Skype для бизнеса выполните команду Move-CsUser с параметром-MoveToTeams.  

    - Если вы переносите номера в корпорацию Майкрософт, вы должны координировать время выполнения этой операции, когда порт будет происходить. 

    - Если вы используете новые номера от корпорации Майкрософт, вам нужно будет изменить LineUri для этого пользователя. Это необходимо сделать после перемещения пользователя через Интернет с помощью Set-CsOnlineVoiceUser.  

## <a name="summary-of-per-tenant-configuration-of-direct-routing"></a>Сводка по конфигурации "прямой маршрут" для каждого клиента 

1. Убедитесь, что ваш контроллер границ вашего сеанса (SBC) поддерживается прямой маршрутизацией, просмотрев [этот список](direct-routing-border-controllers.md). Кроме того, необходимо убедиться, что у вас установлена правильная версия встроенного по.  

2. Свяжите локальный одноранговый SBC с помощью службы прямой маршрутизации Teams. Подробности можно найти [в разделе Связывание SBC с прямой маршрутизацией в телефонной системе](direct-routing-configure.md). 

3. Эта конфигурация по сути является зеркалом локальной конфигурации. Сетевая конфигурация состоит из следующих элементов: 
   - OnlineVoiceRoutingPolicy (в зависимости от локального VoiceRoutingPolicy, если вы переносите пользователей из Skype для бизнеса Online, и в зависимости от того, VoicePolicy ли вы миграция пользователей из локальной сети в корпоративную голосовую почту).
   - OnlinePSTNUsage объекты (в соответствии с использованием локальной сети PSTN). 
   - OnlineVoiceRoute объекты (на основе локальных VoiceRoute). 

Дополнительные сведения можно найти в разделе [Настройка прямого маршрута](direct-routing-configure.md). 

## <a name="manage-enterprisevoiceenabled-property-during-migration"></a>Управление свойством EnterpriseVoiceEnabled во время миграции 

У пользователя должно быть EnterpriseVoiceEnabled = true в Azure AD, чтобы пользователь имел возможность использовать функцию КТСОП.  EnterpriseVoiceEnabled ("EV-Enabled") — это свойство (не политика), которое существует как в локальном, так и в облачном каталогах. Значение в облаке — это то, что важно для Teams.  Точная логика использования режима EV определяется значением true зависит от следующего сценария: 

- Если пользователь поддерживает функцию EV в локальной версии Skype для бизнеса и пользователю назначена лицензия на телефонную систему до того, как вы перейдете в облако с помощью функции Move-CsUser, пользователь в сети будет поддерживаться с параметром EV-enabled = true. 

- Если существующему пользователю TeamsOnly или Skype для бизнеса Online назначена лицензия на телефонную систему, по умолчанию для EV включена не задано значение "true".  Это происходит также в том случае, если локальный пользователь перемещается в облако до назначения лицензии на телефонную систему. В любом случае администратор должен указать следующий командлет: 

  ```PowerShell
  Set-CsUser -EnterpriseVoiceEnabled $True 
  ```
