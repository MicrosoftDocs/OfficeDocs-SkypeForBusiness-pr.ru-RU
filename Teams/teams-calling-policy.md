# <a name="calling-and-call-forwarding-in-teams"></a>Звонки и переадресация звонков в Teams

В Microsoft Teams политики звонков определяют, какие функции звонков и переадресации звонков доступны пользователям. Политики звонков определяют, может ли пользователь совершать частные звонки, использовать переадресацию или одновременный звонок другим пользователям или внешним номерам телефонов, маршрутизировать звонки на голосовую почту, отправлять звонки в группы звонков, использовать делегирование для входящих и исходящих звонков и т. д.

Вы можете использовать глобальную политику (по умолчанию для всей организации), которая создается автоматически, или создавать и назначать пользовательские политики.

## <a name="create-a-custom-calling-policy"></a>Создание настраиваемой политики вызовов

Чтобы создать настраиваемую политику вызовов, выполните следующие действия.

1. В левой области навигации Центра администрирования Microsoft Teams перейдите в **раздел Политики** **голосовых** >  звонков.
2. Нажмите **Добавить**.
3. Включите или отключите функции, которые вы хотите использовать в политике звонков.
    - Например, чтобы определить, могут ли пользователи направлять входящие вызовы в голосовую почту, выберите **Включено** или **Управляется пользователем**. Чтобы запретить маршрутизацию к голосовой почте, выберите **Не включено**.
4. Нажмите **Сохранить**.

## <a name="edit-a-calling-policy"></a>Изменение политики вызовов

Чтобы изменить существующую политику вызовов, выполните следующие действия.

1. В левой области навигации Центра администрирования Microsoft Teams выберите Политики **голосовых** > **звонков**.
2. Щелкните рядом с политикой, которую нужно изменить, а затем выберите **Изменить**.
3. Внесите необходимые изменения и нажмите кнопку **Сохранить**.

## <a name="calling-policy-settings"></a>Параметры политики вызова

Ниже приведены параметры, которые можно настроить для политик вызова.

### <a name="make-private-calls"></a>Совершение приватных звонков

Этот параметр управляет всеми возможностями вызовов в Teams. Отключите эту функцию, чтобы отключить все функции вызовов в Teams.

### <a name="cloud-recording-for-calling"></a>Облачная запись для звонков

Этот параметр определяет, могут ли пользователи записывать вызовы. По умолчанию этот параметр отключен.

### <a name="transcription"></a>Транскрипции

Этот параметр определяет, доступна ли транскрибирование вызовов для пользователей. По умолчанию этот параметр отключен.

### <a name="call-forwarding-and-simultaneous-ringing-to-people-in-your-organization"></a>Переадресация звонков и одновременный звонок людям в организации

Этот параметр определяет, можно ли переадресовать входящие звонки другим пользователям или одновременно позвонить другому пользователю в вашей организации. Этот параметр включен по умолчанию.

### <a name="call-forwarding-and-simultaneous-ringing-to-external-phone-numbers"></a>Переадресация звонков и одновременный звонок на внешние номера телефонов

Этот параметр определяет, можно ли переадресовать входящие вызовы на внешний номер или одновременно позвонить на внешний номер. Этот параметр включен по умолчанию.

### <a name="voicemail-is-available-for-routing-inbound-calls"></a>Голосовая почта доступна для маршрутизации входящих вызовов

Этот параметр позволяет отправлять входящие вызовы на голосовую почту. По умолчанию используется параметр **User controlled**. Допустимые варианты:

- **Включен** Голосовая почта всегда доступна для входящих вызовов.
- **Не включено**  Голосовая почта недоступна для входящих вызовов.
- **Управление пользователем** Пользователи могут определить, должна ли быть доступна голосовая почта.

### <a name="inbound-calls-can-be-routed-to-call-groups"></a>Входящие вызовы можно перенаправить в группы вызовов

Этот параметр определяет, можно ли переадресовать входящие вызовы в группу звонков. Этот параметр включен по умолчанию.

### <a name="delegation-for-inbound-and-outbound-calls"></a>Делегирование для входящих и исходящих вызовов

Этот параметр позволяет маршрутизировать входящие вызовы делегатам, позволяя делегатам совершать исходящие вызовы от имени пользователей, которым они делегировали разрешения. Этот параметр включен по умолчанию. Дополнительные сведения см. в разделе [Предоставление делегату общего доступа к телефонной линии](https://support.office.com/article/share-a-phone-line-with-a-delegate-16307929-a51f-43fc-8323-3b1bf115e5a8).

### <a name="prevent-toll-bypass-and-send-calls-through-the-pstn"></a>Предотвращение обхода платных сборов и отправки вызовов через ТСОП

Если установить для этого параметра значение **Включено** , вызовы будут отправляться через ТСОП и взиматься плата, а не по сети и обхода сборов. Этот параметр по умолчанию отключен.

### <a name="music-on-hold-for-pstn-calls"></a>Музыка при удержании для вызовов ТСОП

Этот параметр позволяет включать или отключать музыку при удержании вызывающего ТСОП. Он включен по умолчанию. Этот параметр не применяется к функциям парковки вызовов и делегирования босса. Узнайте больше о [настройке пользовательской музыки](music-on-hold.md).

### <a name="busy-on-busy-when-in-a-call"></a>Занят при выполнении звонка

Занят при выполнении звонка (также называемые параметрами занятости) позволяет настроить обработку входящих вызовов, когда пользователь уже находится в звонке или конференции или находится на удержании звонка. Новые или входящие вызовы могут отклоняться с сигналом о занятости или маршрутизироваться соответствующим образом в соответствии с параметрами пользователя, не отвеченными. Независимо от того, как настроены параметры занятости, пользователям, выполняющим звонок или конференцию, или пользователям, которые находятся на удержании звонка, не запрещается инициировать новые звонки или конференции. По умолчанию для этого параметра задано значение **Не включено** .

- **Не включено** Параметр занятости не включен, и новые или входящие вызовы могут по-прежнему отправляться пользователю, пока пользователь уже находится в вызове.
- **Включен** Новые или входящие вызовы будут отклонены с сигналом о занятости.
- **Неотвеченные** Будут использоваться параметры пользователя, не отвеченные, например маршрутизация на голосовую почту или переадресация другому пользователю.

### <a name="web-pstn-calling"></a>Вызовы через ТСОП

Этот параметр позволяет пользователям вызывать номера ТСОП с помощью веб-клиента Teams. Этот параметр включен по умолчанию.

### <a name="real-time-captions-in-teams-calls"></a>Субтитры в режиме реального времени в вызовах Teams

Этот параметр определяет, доступны ли субтитры в вызовах Teams в режиме реального времени для пользователей. Этот параметр включен по умолчанию.

### <a name="automatically-answer-incoming-meeting-invites"></a>Автоматически отвечать на приглашения на входящие собрания

Этот параметр определяет, будут ли автоматически отвечать на приглашения на входящие собрания. Она по умолчанию отключена. Помните, что этот параметр применяется только к входящим приглашениям на собрание. Он не применяется к другим типам вызовов.

### <a name="spam-filtering"></a>Фильтрация нежелательной почты

Этот параметр позволяет управлять типом фильтрации нежелательной почты, доступной для входящих вызовов. Можно выполнять проверки как basic, так и Captcha Interactive Voice (IVR). Этот параметр включен по умолчанию.

### <a name="sip-devices-can-be-used-for-calls"></a>Устройства SIP можно использовать для звонков

Этот параметр позволяет пользователям использовать SIP-устройство для совершения и приема вызовов. По умолчанию это значение отключено.

### <a name="open-apps-in-browser-for-incoming-pstn-calls"></a>Открытие приложений в браузере для входящих вызовов ТСОП

Этот параметр определяет, будут ли приложения автоматически открываться в браузере для входящих вызовов ТСОП для пользователей. Его можно использовать для передачи номера телефона входящего абонента в приложение для поиска связанной записи клиента во время звонка. Этот параметр по умолчанию отключен.

Если этот параметр включен, в URL-адресе необходимо будет указать ссылку на приложение, **чтобы открыть приложения в браузере для входящих вызовов ТСОП** . Заполнитель {phone} можно использовать для передачи номера телефона (в формате E.164) в указанный URL-адрес. Кроме того, можно указать универсальный URL-адрес без заполнителя. При этом будет просто запущен указанный URL-адрес.
