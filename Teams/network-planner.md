# <a name="use-the-network-planner-for-microsoft-teams"></a>Использование планировщика сети для Microsoft Teams

Планировщик сети — это новое средство, доступное в центре Teams администрирования. Его можно найти, посетив **планировщик** сети  >  **планирования**. С помощью сетевого планировщика можно всего за несколько шагов определить и упорядочести требования к сети для Microsoft Teams пользователей в организации. Когда вы предоставляете сведения о сети и об использовании Teams, планировщик сети вычисляет требования к сети для развертывания Teams и облачных решений голосовой связи в физических расположениях вашей организации.

![Снимок экрана: планировщик сети.](media/network-planner.png)

Планировщик сети позволяет:

- Создавайте представления своей организации с помощью сайтов и рекомендуемых Майкрософт лиц (офисных сотрудников, удаленных сотрудников и Teams комнат).

    > [!NOTE]
    > Рекомендуемые лица были разработаны на основе данных из Teams сценариев использования и типичных шаблонов использования. Однако вы можете создать до трех пользовательских людей в дополнение к трем рекомендуемых.

- Создание отчетов и расчет требований к пропускной способности для Teams использования.

Для использования планировщика сети необходимо быть глобальным администратором, администратором Teams или администратором Teams связи.

## <a name="create-a-custom-persona"></a>Создание пользовательского лица

Чтобы создать настраиваемый пользователь, выполните указанные здесь действия.

1. Перейдите в планировщик сети в центре Microsoft Teams администрирования.

2. На **вкладке "Люди"** нажмите **кнопку + Пользовательская.** 

3. В **области Новые пользовательские persona** (Новые пользовательские persona) добавьте имя и описание нового человека.

4. Выберите разрешения, которые этот человек будет использовать в организации.

5. Нажмите кнопку **Сохранить**.

## <a name="build-your-plan"></a>Создание плана

Чтобы приступить к построению сетевого плана, выполните указанные здесь действия.

1. Перейдите в планировщик сети в центре Microsoft Teams администрирования.

2. На **вкладке План** сети нажмите **кнопку Добавить план сети**.

3. Введите имя и описание сетевого плана. План сети появится в списке доступных планов.

4. Щелкните название плана, чтобы выбрать новый план.

5. Добавьте сайты, чтобы создать представление сетевой настройки организации.

    В зависимости от сети организации вы можете использовать сайты для представления здания, расположения офиса или другого объекта. Сайты могут быть подключены по сети WAN, чтобы разрешить общий доступ к Интернету и/или подключениям по STN. Для лучших результатов создавайте сайты с локальными подключениями, прежде чем создавать сайты, которые будут удаленно подключаться к Интернету или STN.

    Чтобы создать сайт:

    1. Добавьте имя и описание сайта.

    2. В **области Параметры сети** добавьте количество пользователей сети на этом сайте (обязательно).

    3. Добавьте сведения о сети: пропускная способность по сети, пропускная способность по сети WAN, доступ к Интернету **(локальный** или удаленный) и регрессия через ДНР (нет, локальный или удаленный).

      > [!NOTE]
      > Для получения определенных рекомендаций по пропускной способности при создание отчета необходимо добавить номера пропускной способности сети WAN и Интернета.

    4. Нажмите кнопку **Сохранить**.

## <a name="create-a-report"></a>Создание отчета

После добавления всех сайтов вы можете создать отчет следующим образом:

1. На **вкладке Отчеты** нажмите **кнопку Начать отчет**.

2. Для каждого создаемого сайта распределяйте по всем доступным пользователям число пользователей. Если вы используете рекомендуемые Корпорацией Майкрософт люди, они будут распределяться автоматически (80 % сотрудников и 20 % удаленных сотрудников).

3. После завершения распространения нажмите кнопку **Создать отчет**.

    В сгенерированном отчете будут демонстрироваться требования к пропускной способности в нескольких различных представлениях, что позволит четко понять выходные данные.
    - В таблице с отдельными вычислениями будут отображаться требования к пропускной способности для каждого разрешенного действия.
    - В дополнительном представлении будут демонстрироваться общие потребности в пропускной способности с рекомендациями.

4. Нажмите кнопку **Сохранить**. Отчет будет доступен в списке отчетов для просмотра в дальнейшем.
