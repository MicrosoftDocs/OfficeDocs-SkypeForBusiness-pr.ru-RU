# <a name="use-the-network-planner-for-microsoft-teams"></a>Использование планировщика сети для Microsoft Teams

Планировщик сети — это новое средство, доступное в Teams администрирования. Его можно найти, перейдя в **планировщик** > **планирования сети**. Всего за несколько шагов планировщик сети поможет определить и упорядочить требования к сети для подключения Microsoft Teams пользователей в организации. Когда вы предоставляете сведения о сети и об использовании Teams, планировщик сети вычисляет требования к сети для развертывания Teams и облачных решений голосовой связи в физических расположениях вашей организации.

![Снимок экрана: планировщик сети.](media/network-planner.png)

Планировщик сети позволяет выполнять следующие задачи:

- Создавайте представления организации с помощью сайтов и рекомендуемых сотрудников Майкрософт (офисных сотрудников, удаленных сотрудников и Teams комнат).

    > [!NOTE]
    > Рекомендуемые пользователи были разработаны на основе данных из Teams и типичных шаблонов использования. Однако в дополнение к трем рекомендованным пользователям можно создать до трех пользовательских пользователей.

- Создание отчетов и вычисление требований к пропускной способности для Teams использования.

Чтобы использовать планировщик сети, необходимо быть глобальным администратором, администратором Teams или администратором Teams связи.

## <a name="create-a-custom-persona"></a>Создание пользовательского пользователя

Выполните следующие действия, чтобы создать настраиваемого пользователя:

1. Перейдите к планировщику сети в Microsoft Teams администрирования.

2. На **вкладке "Personas** " щелкните **"+ Пользовательская личность"**. 

3. В области **"Создание настраиваемого** пользователя" добавьте имя и описание нового пользователя.

4. Выберите разрешения, которые этот пользователь будет использовать в организации.

5. Нажмите кнопку **Сохранить**.

## <a name="build-your-plan"></a>Создание плана

Выполните следующие действия, чтобы приступить к созданию сетевого плана:

1. Перейдите к планировщику сети в Microsoft Teams администрирования.

2. На **вкладке "План** сети" щелкните **"Добавить сетевой план"**.

3. Введите имя и описание плана сети. План сети появится в списке доступных планов.

4. Щелкните имя плана, чтобы выбрать новый план.

5. Добавьте сайты, чтобы создать представление конфигурации сети вашей организации.

    В зависимости от сети организации вы можете использовать сайты для представления здания, расположения офиса или другого объекта. Сайты могут быть подключены глобальной сетью, чтобы разрешить общий доступ к подключениям к Интернету и (или) ТСОП. Для получения наилучших результатов создайте сайты с локальными подключениями, прежде чем создавать сайты, которые удаленно подключаются к Интернету или ТСОП.

    Чтобы создать сайт:

    1. Добавьте имя и описание сайта.

    2. В **разделе "Параметры сети**" добавьте количество пользователей сети на этом сайте (обязательно).

    3. Добавьте сведения о сети: с поддержкой глобальной сети, емкость глобальной сети, исходящий **интернет (** локальный или удаленный **) и** исходящий трафик ТСОП (нет, локальный или удаленный).

      > [!NOTE]
      > Чтобы просмотреть конкретные рекомендации по пропускной способности при создании отчета, необходимо добавить номера емкости глобальной сети и Интернета.

    4. Нажмите кнопку **Сохранить**.

## <a name="create-a-report"></a>Создание отчета

После добавления всех сайтов можно создать отчет следующим образом.

1. На **вкладке "Отчеты** " нажмите **кнопку "Запустить отчет"**.

2. Для каждого создадимого сайта распределите количество пользователей среди доступных пользователей. Если вы используете рекомендуемых сотрудников Майкрософт, это число будет распространяться автоматически (80 % сотрудников office и 20 % удаленных сотрудников).

3. После завершения распространения нажмите кнопку **"Создать отчет"**.

    Созданный отчет будет отображать требования к пропускной способности в нескольких разных представлениях, чтобы вы могли четко понять выходные данные:
    - В таблице с отдельными вычислениями будут отображаться требования к пропускной способности для каждого разрешенного действия.
    - В дополнительном представлении будут показаны общие потребности в пропускной способности с рекомендациями.

4. Нажмите кнопку **Сохранить**. Отчет будет доступен в списке отчетов для последующего просмотра.
