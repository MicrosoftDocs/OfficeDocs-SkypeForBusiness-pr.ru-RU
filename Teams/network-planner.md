# <a name="use-the-network-planner-for-microsoft-teams"></a>Использование планировщика сети для Microsoft Teams

Планировщик сетей — это новый инструмент, доступный в Центре администрирования Teams. Его можно найти в **планировщике сети**  >  **планирования.** С помощью Планировщика сетей можно всего несколькими шагами определить и упорядочеть требования к сети для подключения пользователей Microsoft Teams в организации. При предоставлении сведений о сети и использовании Teams планировщик сети рассчитывает требования к сети для развертывания Teams и облачного голосового связи в физических расположениях организации.

![Снимок экрана: планировщик сети](media/network-planner.png)

Планировщик сети позволяет:

- Создавайте представления организации с помощью сайтов и рекомендуемых Майкрософт людей (офисных сотрудников, удаленных сотрудников и системы комнат Teams).

    > [!NOTE]
    > Рекомендуемые типовые группы были разработаны на основе данных из сценариев использования Teams и типичных шаблонов использования. Однако вы можете создать до трех пользовательских людей в дополнение к трем рекомендуемых.

- Создание отчетов и расчет требований к пропускной способности для использования Teams.

Для использования планировщика сети необходимо быть глобальным администратором, администратором служб Teams или администратором связи Teams.

## <a name="create-a-custom-persona"></a>Создание пользовательского лица

Чтобы создать пользовательское лицо, выполните указанные здесь действия.

1. Перейдите в планировщик сетей в Центре администрирования Microsoft Teams.

2. На **вкладке "Люди"** нажмите кнопку **+Пользовательский пользователь.** 

3. В области **"Новый пользователь"** добавьте имя и описание нового человека.

4. Выберите разрешения, которые этот человек будет использовать в организации.

5. Щелкните **Сохранить**.

## <a name="build-your-plan"></a>Создание плана

Чтобы приступить к построению сетевого плана, выполните указанные здесь действия.

1. Перейдите в планировщик сетей в Центре администрирования Microsoft Teams.

2. На **вкладке "План сети"** нажмите кнопку **"Добавить план сети".**

3. Введите имя и описание сетевого плана. План сети появится в списке доступных планов.

4. Щелкните имя плана, чтобы выбрать новый план.

5. Добавьте сайты, чтобы создать представление настройки сети организации.

    В зависимости от сети организации вы можете использовать сайты для представления здания, расположения офиса или другого объекта. Сайты могут быть подключены через сеть WAN, чтобы разрешить совместный доступ к Интернету и (или) подключениям по ННР. Для лучших результатов создайте сайты с локальными подключениями, прежде чем создавать сайты, которые будут удаленно подключаться к Интернету или STN.

    Чтобы создать сайт:

    1. Добавьте имя и описание сайта.

    2. В **параметрах сети** добавьте количество пользователей сети на этом сайте (обязательно).

    3. Добавьте сведения о сети: пропускную способность для WAN, пропускную способность по интернету **(локальный** или удаленный) и ДНР (отсутствует, локальный или удаленный).

      > [!NOTE]
      > Для получения определенных рекомендаций по пропускной способности при созданием отчета необходимо добавить номера пропускной способности WAN и Интернета.

    4. Щелкните **Сохранить**.

## <a name="create-a-report"></a>Создание отчета

После добавления всех сайтов можно создать отчет следующим образом:

1. На **вкладке "Отчеты"** нажмите кнопку **"Начать отчет".**

2. Для каждого созда создаемого сайта распределяйте по всем доступным пользователям количество пользователей. Если вы используете рекомендуемые корпорацией Майкрософт номера сотрудников, они будут распределяться автоматически (80 % сотрудников офиса и 20 % удаленных сотрудников).

3. После завершения распространения нажмите кнопку **"Создать отчет".**

    В сгенерированном отчете будут демонстрироваться требования к пропускной способности в нескольких различных представлениях, что позволит понять выходные данные.
    - В таблице с индивидуальными вычислениями будут отображаться требования к пропускной способности для каждого разрешенного действия.
    - В дополнительном представлении будут демонстрироваться общие потребности в пропускной способности с рекомендациями.

4. Щелкните **Сохранить**. Отчет будет доступен в списке отчетов для просмотра в дальнейшем.
