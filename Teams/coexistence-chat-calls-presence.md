# <a name="coexistence-with-skype-for-business"></a>Сосуществование со Skype для бизнеса

Совместная работа и взаимодействие между клиентами и пользователями групп Skype для бизнеса и Teams определяются режимами Теамсупграде, описанными в [статье Руководство по миграции и взаимодействию для организаций, использующих команды совместно с Skype для бизнеса](migration-interop-guidance-for-teams-with-skype.md).

Каждому пользователю всегда будет назначен режим Теамсупграде либо по умолчанию, либо явным образом администратором. Значение по умолчанию — *острова*. Пользователи, обновленные до Teams, имеют режим *теамсонли*. *Сфбонли*, *сфбвистеамсколлаб*и *сфбвистеамсколлабандмитингс* также являются допустимыми режимами.


## <a name="routing-parameters"></a>Параметры маршрутизации

Режим Теамсупграде получателя является ключом в определении поведения чатов, звонков и присутствия в рамках клиента и на федеративных клиентах.

Если отправитель использует группы, решение по маршрутизации создается при создании нового обсуждения. Существующие потоки беседы в Teams всегда сохраняют метод маршрутизации, определенный при создании потока: teams поддерживает устойчивые потоки.

 Методы маршрутизации потоков:  

- *машинный код* для беседы в Teams с участниками-клиентами
- *взаимодействие* между участниками в Skype для бизнеса (в том же клиенте)
- *федеративное* для Федеративной беседы между клиентами

Ниже перечислены параметры, которые определяют метод маршрутизации потока.

- Режим Теамсупграде получателя
- Клиент, используемый отправителем
- Указывает, является ли беседа новой или входит в существующий поток.
- Указывает, находится ли беседа в клиенте или Федеративной
- Возможен ли разговор
    - Взаимодействие с клиентами требует, чтобы клиент либо ни Интернет, либо ни Skype для бизнеса *в* гибридной среде. Для обеспечения взаимодействия на стороне клиента не может быть достаточно локальных клиентов.
    - Для *Федерации с несколькими клиентами* всегда требуется правильная конфигурация Федерации Skype для бизнеса, а также правильная конфигурация Федерации Teams для обоих клиентов. Гибридная версия Skype для бизнеса не требуется ни одному из клиентов.
    - Если учетная запись Skype для бизнеса для инициатора размещена локально, этот пользователь не может использовать клиент Teams для взаимодействия с клиентами или для Федерации. Этот пользователь может использовать только клиент Skype для бизнеса для взаимодействия и интеграции.
    - Взаимодействие Teams с Teams всегда возможно в клиенте.

> [!NOTE]
> В настоящее время все Федерации с участием групп используют конвейер Федерации Skype для бизнеса и Teams – взаимодействие с Skype для бизнеса. Мы планируем собственные Teams – Федерацию Teams. Настоящий документ будет обновлен после выпуска собственной Федерации.

## <a name="chats-and-calls-from-pre-existing-threads"></a>Разговоры и звонки из уже существующих потоков

### <a name="from-teams"></a>Из Teams

Звонки и разговоры, запущенные из существующего постоянного потока в Teams, будут маршрутизироваться таким же образом, как и этот поток, если этот параметр маршрутизации по-прежнему доступен.

Если уже существующий постоянный поток в Teams является собственным потоком (например, перенаправленным в Teams), дополнительные сообщения чата и звонки из этого потока будут перенаправлены в Teams. Если это поток взаимодействия (то есть пересылается в Skype для бизнеса), дополнительные сообщения и звонки будут передаваться в Skype для бизнеса (в этом случае появятся варианты маршрутизации).

> [!NOTE]
> Уже существующие потоки в Teams больше нельзя будет перенаследовать, например, когда поток был потоком взаимодействия с пользователем, который теперь обновлен до Teams. Так как он был создан в качестве потока взаимодействия, поток будет направляться в Skype для бизнеса, но этот пользователь больше не сможет использовать Skype для бизнеса для общения и звонков. В этом случае поток будет отключен и дальнейшая передача данных не разрешается.

### <a name="from-skype-for-business"></a>Из Skype для бизнеса

Потоки Skype для бизнеса не сохраняются за тайм-аут сеанса SIP 10 мин. Разговоры и звонки из существующего потока в Skype для бизнеса до истечения срока действия сеанса SIP будут маршрутизироваться так же, как и поток. Звонки и разговоры из существующего потока в Skype для бизнеса за пределами таймаута сеанса SIP будут перенаправляться в Skype для бизнеса удаленной стороны, независимо от того, на какой стороне клиента получен первоначальный поток.

## <a name="availability"></a>Официального

Доступны как описанные выше поведения из клиента, так и Федеративной, со следующими ограничениями:

- Внешние участники, клиенты которых находятся в разных Голокал развернутых или географических элементах, не отображаются в чате с помощью "федеративного" собрания.
- Интеграция и взаимодействие между клиентами O365 и изолированной облака не поддерживаются

# <a name="presence"></a>Присутствие

Если у вас есть ситуация, когда некоторые пользователи используют клиента Teams и другие по-прежнему используют клиента Skype для бизнеса, возможно, у вас есть несколько пользователей, использующих оба клиента. Вы по-прежнему хотите предоставить доступ к состояниям присутствия всем пользователям независимо от того, какой клиент есть у отдельного пользователя. Если общий доступ предоставляется в рамках всей Организации, пользователи смогут лучше определить, нужно ли вам начать чат или позвонить.

Например, если чат или звонок инициатора должен располагаться на клиенте Skype для бизнеса, это значит, что клиент Skype для бизнеса должен быть видимым для создателя. Если он должен располагаться на целевом клиенте Teams, это должен быть элемент клиента Teams, который нужно отобразить.

Чтобы узнать, какое поведение следует ожидать, вам нужно понимать, что оно доступно на основании режима сосуществования пользователя.

* Если пользователь находится в режиме Теамсонли, любой другой пользователь (в Teams или Skype для бизнеса) может видеть, что пользователи в Teams Теамсонли
* Если пользователь находится в любом из режимов SfB\* (Сфбонли, Сфбвистеамсколлаб, сфбвистеамсколлабандмитингс), другие пользователи (в Teams или Skype для бизнеса) увидят, что SfB\* пользователя в Skype для бизнеса
* Если пользователь находится в режиме острова (или устаревшем), присутствие в Teams и присутствии в Skype для бизнеса независимы (значения не совпадают), а другие пользователи увидят одно или другое присутствие пользователя острова в зависимости от того, находятся ли они в одном и том же клиенте или в Федерации.     * В Teams у любого другого пользователя в Федеративной клиенте будут видны сведения о присутствии пользователя в Skype для бизнеса. Это выровнено с таблицей Федеративной маршрутизации, приведенной выше
    * В Skype для бизнеса любые другие пользователи будут видеть присутствие в Skype для бизнеса пользователей, которые имеют в своем мире (как в клиенте, так и в Федеративной). оно выравнивается по таблицам маршрутизации выше.


## <a name="in-tenant-presence"></a>Присутствие в клиенте

Сообщения, отправляемые пользователям Теамсонли, всегда помещаются в Teams. ## <a name="presence-in-pre-existing-threads"></a>Присутствие в уже существующих потоках

Для выравнивания присутствия и достижимости в уже существующих потоках присутствие целевого объекта, предоставляемого в этом потоке, должен быть выровнен с маршрутизацией потока, предполагая, что маршрутизация возможна.

В частности, если получатель, у которого ранее существовала постоянная цепочка взаимодействия между собой, был обновлен до Teams, этот поток больше не будет показывать точное присутствие и больше не будет поддерживать маршрутизацию. Начните новый поток.
