# <a name="coexistence-with-skype-for-business"></a>Сосуществование со Skype для бизнеса

Сосуществование и взаимодействие между клиентами Skype для бизнеса и Teams управляются режимами сосуществования. Дополнительные сведения см. в руководстве по миграции и взаимодействию для организаций, использующих [Teams вместе с Skype для бизнеса](migration-interop-guidance-for-teams-with-skype.md). После прекращения поддержки Skype для бизнеса Online 31 июля 2021 г. пользователи, размещенные в облаке, всегда будут пользователями TeamsOnly. Пользователю в Сети больше нельзя назначить режим сосуществования, отличный от TeamsOnly. Режимы сосуществования, отличные от TeamsOnly, актуальны только для организаций с локальными развертываниями Skype для бизнеса Server или Lync Server 2013. В этой статье любая ссылка на "Skype для бизнеса Server" также относится к Lync Server 2013.


## <a name="determining-a-users-coexistence-mode"></a>Определение режима сосуществования пользователя
Все пользователи в организациях без локального развертывания Skype для бизнеса Server находятся в режиме TeamsOnly, а режим действия клиента также — TeamsOnly. Это можно проверить, просмотрев свойство TeamsUpgradeEffectiveMode в клиенте или пользователя с помощью Teams PowerShell.   До прекращения использования Skype для бизнеса Online 31 июля 2021 г. организации имели возможность изменить режим сосуществования для пользователя или клиента. Это невозможно, за исключением организаций с локальным развертыванием Skype для бизнеса Server, которые не должны иметь режим TeamsOnly на уровне клиента. Вы можете убедиться, что режим сосуществования больше не может быть изменен, если TeamsUpgradePolicyIsReadOnly = "ModeAndNotifications" на пользователе или клиенте.  (TeamsUpgradePolicyIsReadOnly для любого пользователя будет иметь то же значение, что и значение клиента.)  


 ```powershell
 //Check if Tenant is TeamsOnly and if mode is read only.
$t=Get-CsTenant
$t|fl TeamsUpgradeEffectiveMode, TeamsUpgradePolicyIsReadOnly

TeamsUpgradeEffectiveMode  : TeamsOnly
TeamsUpgradePolicyIsReadOnly: ModeAndNotifications

 //Check if user is TeamsOnly and if mode is read only.
$u=Get-CsOnlineUser
$u|fl TeamsUpgradeEffectiveMode, TeamsUpgradePolicyIsReadOnly

TeamsUpgradeEffectiveMode  : TeamsOnly
TeamsUpgradePolicyIsReadOnly: ModeAndNotifications
 ```

В организации с локальным развертыванием Skype для бизнеса Server глобальная политика клиента для TeamsUpgradePolicy может иметь любой режим, отличный от *TeamsOnly*. Допустимые режимы: *SfBOnly*, *SfBWithTeamsCollab* и *SfBWithTeamsCollabAndMeetings*. Пользователям также может быть напрямую назначен экземпляр TeamsUpgradePolicy, который заменяет глобальную политику клиента.  Пользователи, размещенные в облаке, должны быть teamsOnly, а пользователи, размещенные в локальной среде, должны быть в любом режиме, кроме TeamsOnly.  Если пользователю не назначен экземпляр TeamsUpgradePolicy, он получает значение из глобальной политики клиента. 

## <a name="routing-parameters"></a>Параметры маршрутизации

Режим сосуществования получателя определяет поведение чатов, звонков и присутствия как в клиенте, так и в федеративных клиентах. Если отправитель использует Teams, решение о маршрутизации принимается при создании нового потока беседы. После создания потока беседы его маршрутизация не изменяется, и он сохраняет метод маршрутизации, определенный при создании потока.

Методы маршрутизации потоков:

- *Собственный для* беседы Teams в Teams в клиенте
- *взаимодействие для* Teams для Skype для бизнеса беседы в клиенте
- *собственная федеративная* для федеративной беседы между клиентами, если оба пользователя имеют режим TeamsOnly. 
- *федеративное* взаимодействие для федеративной беседы между клиентами, которая зависит от взаимодействия между Skype для бизнеса и Teams.

> [!NOTE]
> - Собственные беседы, как в одном клиенте, так и в федеративных сценариях, происходят, когда получатель и отправитель имеют режим TeamsOnly. Беседа будет иметь собственный интерфейс чата, который включает все широкие возможности обмена сообщениями и звонков. Дополнительные сведения см. в [разделе "Собственный чат" для внешних (федеративных) пользователей в Teams](native-chat-for-external-users.md). 
> - Если у любого из участников беседы нет режима TeamsOnly, беседа представляет собой взаимодействие с текстовыми сообщениями.
> - Федеративные связи между пользователями TeamsOnly в мультитенантных облаках и специальных облачных средах (например, облаках для государственных организаций) будут отображаться как федеративные чаты взаимодействия.


При создании новой беседы определяются следующие факторы, определяющие способ маршрутизации потока.

- Режим сосуществования получателя
- Клиент, используемый отправитором
- Является ли беседа в клиенте или федеративной
- Возможна ли беседа. Если у пользователя есть локальная учетная запись Skype для бизнеса, он не может использовать клиент Teams для взаимодействия в клиенте или для федерации. Этот пользователь может использовать только клиент Skype для бизнеса для взаимодействия и федерации. Обратите внимание, что взаимодействие Teams с Teams всегда возможно в клиенте.

## <a name="chats-and-calls-from-pre-existing-threads"></a>Чаты и вызовы из существующих потоков

### <a name="from-teams"></a>Из Teams

Звонки или чаты, запущенные из существующего потока беседы в Teams, направляются так же, как и этот поток. Если ранее существующий поток в Teams был собственным потоком (т. е. перенаправленным в Teams), дополнительные сообщения чата и вызовы из этого потока будут отправляться в Teams. Если это был поток взаимодействия (т. е. направляемый в Skype для бизнеса), дополнительные сообщения чата и вызовы будут отправляться в Skype для бизнеса (при условии, что доступны параметры маршрутизации).

> [!NOTE]
> Существующие потоки в Teams могут быть недоступны для маршрутизации, например, когда поток был потоком взаимодействия с пользователем, который с тех пор был обновлен до Teams. Так как он был создан как поток взаимодействия, поток будет маршрутизироваться в Skype для бизнеса, но этот пользователь больше не может использовать Skype для бизнеса для чата и звонков. В этом случае поток будет отключен и не разрешит дальнейшее взаимодействие.

### <a name="from-skype-for-business"></a>Из Skype для бизнеса

Skype для бизнеса потоки не сохраняются после 10-минутного времени ожидания сеанса SIP. Чаты и вызовы из существующего потока в Skype для бизнеса до истечения срока действия сеанса SIP будут направляться так же, как поток. Вызовы и чаты из существующего потока в Skype для бизнеса за пределами времени ожидания сеанса SIP будут направляться на сервер удаленной Skype для бизнеса независимо от того, какой клиент был получен исходным потоком на стороне другой стороны.

## <a name="presence"></a>Присутствие

В ситуациях, когда некоторые пользователи используют клиент Teams, а другие используют клиент Skype для бизнеса, возможно, некоторые из этих пользователей используют оба клиента. Важно понимать, что сведения о присутствии публикуются в режиме сосуществования пользователя. Например, если чат или вызов инициатора должны находиться на клиенте целевого Skype для бизнеса, то это присутствие Skype для бизнеса клиента, которое должно быть показано инициатору. Если он должен находиться на клиенте Teams целевого объекта, следует отобразить его присутствие.

Присутствие является общим на основе режима сосуществования пользователя, как описано ниже:

- Если пользователь находится в режиме TeamsOnly, любой другой пользователь (в Teams или Skype для бизнеса) будет видеть, что пользователь TeamsOnly находится в Teams.
- Если пользователь находится в любом из режимов Skype для бизнеса (SfbOnly, SfbWithTeamsCollab, SfbWithTeamsCollabAndMeetings), то любой другой пользователь (в Teams или Skype для бизнеса) будет видеть Skype для бизнеса пользователя Skype для бизнеса.
- Если пользователь находится в режиме Islands, присутствие в Teams и присутствие в Skype для бизнеса являются независимыми (значения не должны совпадать), а другие пользователи будут видеть одно или другое присутствие пользователя Islands, в зависимости от того, находится ли он в том же клиенте или в федеративном клиенте и какой клиент они используют
  - В Teams любой другой пользователь в том же клиенте будет видеть сведения о присутствии Пользователя Islands в Teams. соответствует приведенной выше таблице маршрутизации в клиенте.
  - В Teams любой другой пользователь в федеративном клиенте будет видеть сведения о присутствии Skype для бизнеса Islands; это соответствует таблице федеративной маршрутизации, приведенной выше.
  - Из Skype для бизнеса любой другой пользователь будет видеть сведения о присутствии пользователя Islands Skype для бизнеса (как в клиенте, так и в федеративном режиме). Это соответствует таблицам маршрутизации, указанным выше.

### <a name="presence-in-pre-existing-threads"></a>Присутствие в существующих потоках

Чтобы обеспечить согласование присутствия и доступности в существующих потоках, присутствие целевого объекта, предоставляемое в этом потоке, должно соответствовать маршрутизации потока при условии, что маршрутизация возможна.  В частности, если получатель, с которого ранее был выполнен постоянный поток общения, был обновлен до Teams, этот поток больше не будет отражать точное присутствие и больше не будет маршрутизации. Необходимо запустить новый поток.

### <a name="federation-and-interop-with-office-365-operated-by-21vianet"></a>Федерация и взаимодействие с Office 365 21Vianet
