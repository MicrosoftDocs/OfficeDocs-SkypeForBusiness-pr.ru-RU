# <a name="coexistence-with-skype-for-business"></a>Сосуществование со Skype для бизнеса

Сосуществование и взаимодействия между клиентами Skype для бизнеса и Teams управляются режимами сосуществования. Дополнительные сведения см. в [руководстве](migration-interop-guidance-for-teams-with-skype.md)по миграции и совместному использованию данных для организаций, использующих Teams вместе с Skype для бизнеса. После выхода из Skype для бизнеса Online 31 июля 2021 г. пользователи, которые находятся в облаке, всегда являются пользователями TeamsOnly. Более невозможно назначить пользователю в сети режим сосуществования, кроме TeamsOnly. Режимы сосуществования, кроме TeamsOnly, релевантны только для организаций с Skype для бизнеса Server или Lync Server 2013. В этой статье любая ссылка на Skype для бизнеса Server относится и к Lync Server 2013.


## <a name="determining-a-users-coexistence-mode"></a>Определение режима совместной работы пользователя
Все пользователи в организациях без локального развертывания Skype для бизнеса Server работают в режиме TeamsOnly, а в клиенте — в режиме TeamsOnly. Это можно подтвердить с помощью свойства TeamsUpgradeEffectiveMode на клиенте или пользователя с помощью Teams PowerShell.   До того как Skype для бизнеса Online 31 июля 2021 г., организации имели возможность изменить режим сосуществования для пользователя или клиента. Эта возможность больше не возможна, за исключением организаций с локальной развертыванием Skype для бизнеса Server, для которых не должен быть развернут клиент в режиме TeamsOnly. Вы можете подтвердить, что режим сосуществования больше нельзя изменить, если TeamsUpgradePolicyIsReadOnly = "ModeAndNotifications" для пользователя или клиента.  (TeamsUpgradePolicyIsReadOnly для любого пользователя будет иметь то же значение, что и значение клиента.)  


 ```powershell
 //Check if Tenant is TeamsOnly and if mode is read only.
$t=Get-CsTenant
$t|fl TeamsUpgradeEffectiveMode, TeamsUpgradePolicyIsReadOnly

TeamsUpgradeEffectiveMode  : TeamsOnly
TeamsUpgradePolicyIsReadOnly: ModeAndNotifications

 //Check if user is TeamsOnly and if mode is read only.
$u=Get-CsOnlineUser
$u|fl TeamsUpgradeEffectiveMode, TeamsUpgradePolicyIsReadOnly

TeamsUpgradeEffectiveMode  : TeamsOnly
TeamsUpgradePolicyIsReadOnly: ModeAndNotifications
 ```

В организации с локальной развертыванием Skype для бизнеса Server, глобальная политика клиента для TeamsUpgradePolicy может иметь любой режим, кроме *TeamsOnly.* Допустимые режимы: *SfBOnly,* *SfBWithTeamsCollab* и *SfBWithTeamsCollabAndMeetings.* Кроме того, пользователям может быть напрямую назначен экземпляр TeamsUpgradePolicy, который следует за глобальной политикой клиента.  Пользователями, которые работают в облаке, должны быть TeamsOnly, а пользователями, которые работают локально, должен быть любой режим, кроме TeamsOnly.  Если пользователю не назначен экземпляр TeamsUpgradePolicy, он получает значение из глобальной политики клиента. 

## <a name="routing-parameters"></a>Параметры маршрутов

Режим сосуществования получателя определяет поведение чатов, звонков и присутствия как в клиенте, так и в федерационных клиентах. Если отправитель использует Teams, при создании новой беседы принимается решение о маршрутике. После создания цепочки беседы ее маршрутия не меняется и сохраняется способ маршрутинга, определяющий, когда была создана цепочка.

Маршруты цепочки можно использовать в:

- *native* for a Teams to Teams conversation in-tenant
- *во время* беседы Teams Skype для бизнеса в клиенте
- *федеративна* для федеративной беседы в разных клиентах, если оба пользователя работают в режиме TeamsOnly. 
- *федерален* для федераированной беседы в разных клиентах, которая зависит от взаимодействия между Skype для бизнеса и Teams.

> [!NOTE]
> - Беседы, как в одном клиенте, так и в федеративном сценарии, возникают, когда и отправитель, и отправитель работают в режиме TeamsOnly. Беседа будет иметь свой опыт чата, который включает все возможности обмена сообщениями и звонков. Чтобы узнать больше, ознакомьтесь со чатом на языке внешних [(федеративных) пользователей](native-chat-for-external-users.md)в Teams. 
> - Если у одного из участников беседы нет режима TeamsOnly, беседа будет работать с текстовыми сообщениями.
> - Федеративная связь между пользователями TeamsOnly в облаках с несколькими пользователями и специальных облачных средах (например, правительственных облаках) будет отображаться как взаимодействие федеративных чатов.


При создании новой беседы определяются факторы, влияющие на маршрутику цепочки.

- Режим сосуществования получателя
- Клиент, используемый отправивщиком
- Является ли беседа клиентской или федераательной
- Возможна ли беседа. Если у пользователя есть Skype для бизнеса локальной учетной записи, он не может использовать клиент Teams для локального клиентского интерфейса или для федерации. Этот пользователь может использовать клиент Skype для бизнеса только для обеспечения взаимосвязи и федерации. Обратите внимание Teams что Teams взаимодействие всегда возможно в клиенте.

## <a name="chats-and-calls-from-pre-existing-threads"></a>Чаты и звонки из существующих бесед

### <a name="from-teams"></a>Из Teams

Звонки и чаты, начатые из предварительно существующей беседы в Teams маршруты маршрутются так же, как и в эту цепочку. Если ранее существующая цепочка в Teams была нитью (например, на Teams), дополнительные сообщения чата и звонки из этой цепочки будут отправляться в Teams. Если это была переписка (например, маршрутная в Skype для бизнеса), дополнительные сообщения чата и звонки будут отправляться в Skype для бизнеса (при условии, что доступны параметры маршрутинга).

> [!NOTE]
> Существующие цепочки в Teams могут перестать быть маршрутизируемыми, например, когда цепочка была цепочкой для пользователя, который был обновлен до Teams. Так как она была создана в качестве беседы, поток будет маршрутизовываться в Skype для бизнеса, но этот пользователь больше не сможет использовать Skype для бизнеса чата и звонков. В этом случае цепочка будет отключена и не будет разрешать дальнейшую связь.

### <a name="from-skype-for-business"></a>Из Skype для бизнеса

Skype для бизнеса цепочки не сохраняются после 10-минутного времени сеанса SIP. Чаты и звонки из существующей цепочки в Skype для бизнеса до истечения срока действия сеанса SIP будут маршрутизовываться так же, как цепочка. Звонки и чаты из существующей цепочки в Skype для бизнеса за пределами времени времени сеанса SIP будут переназначаться удаленной стороне Skype для бизнеса независимо от того, какой клиент исходная цепочка поступила на стороне другой стороны.

## <a name="presence"></a>Присутствие

Если некоторые пользователи используют клиент Teams, а другие — клиент Skype для бизнеса, возможно, некоторые из них используют оба клиента. Важно понимать, что публикация данных о присутствии основана на режиме совместной работы пользователя. Например, если чат или звонок источника должен перенаться в клиент Skype для бизнеса целевого клиента, это значит, что Skype для бизнеса должен быть показан его источнику. Если она должна быть нацелена на Teams целевого клиента, следует показать его Teams клиента.

Доступ к присутствию основан на режиме совместной работы пользователя, как описано ниже.

- Если пользователь находится в режиме TeamsOnly, то любой другой пользователь (как в Teams, так и в Skype для бизнеса) увидит, что Teams TeamsOnly.
- Если пользователь находится в любом из режимов Skype для бизнеса (SfbOnly, SfbWithTeamsCollab, SfbWithTeamsCollabAndMeetings), то любой другой пользователь (как в Teams, так и Skype для бизнеса) увидит, что Skype для бизнеса пользователя Skype для бизнеса.
- Если пользователь находится в режиме "Острова", присутствие в Teams и присутствие в Skype для бизнеса являются независимыми (значения не должны совпадать), а другие пользователи будут видеть те или иные данные о присутствии пользователя "Острова" в зависимости от того, в том же клиенте или федераированном клиенте и какой клиент он использует.
  - В Teams другие пользователи в том же клиенте будут видеть данные о присутствии пользователя Teams о-ва "Острова"; они будут согласованы с приведенной выше таблицей маршрутов в клиенте.
  - Из Teams, любой другой пользователь в федератовском клиенте будет видеть данные о присутствии пользователя Skype для бизнеса о-ва (он соответствует таблице федераированной маршрутии выше).
  - В Skype для бизнеса другие пользователи будут видеть данные о присутствии пользователя Skype для бизнеса (как в клиенте, так и в федераированном расположении); эти данные соответствуют таблицам маршрутинга выше.
