# <a name="coexistence-with-skype-for-business"></a>Совместная работа с Скайп для бизнеса

Взаимодействие между Скайп для бизнеса и рабочих групп и сосуществование определяется TeamsUpgrade режима, описанных в [миграции и руководство по взаимодействию для организаций с помощью команды Скайп для бизнеса](migration-interop-guidance-for-teams-with-skype.md).

Любого пользователя всегда назначается режим TeamsUpgrade, по умолчанию, или явным образом администратором. Значение по умолчанию — *о-ва*. Пользователи, обновленных до группы имеют режим *TeamsOnly*.

> [!NOTE]
> Режим *прежних версий* устарел и; Пользователи в режиме *прежних версий* будут преобразованы в режиме *острова* после 15 ноября 2018.

## <a name="routing-parameters"></a>Параметры маршрутизации

Режим TeamsUpgrade получателя — ключ для определения поведения бесед, звонки и сведения о присутствии, как клиента, так и федеративных клиентов.

Если отправитель использование команды, при создании нового потока беседы принято решение маршрутизации. Существующие потоки беседы в группах всегда сохранять метод маршрутизации, определяются при создании потока. Команды поддерживает постоянные потоков.

 Методы маршрутизации потока являются:  
* *собственные* группы для групп беседы в клиента
* *взаимодействие* группы для Скайп для бизнеса беседы в клиента
* *федеративные* для федеративного беседы клиентов.

Используются параметры, определяющие метод маршрутизации потока.
* Режим TeamsUpgrade получателя
* Клиент, используемого отправителем
* Является ли беседе нового или существующего обсуждения в рамках
* Является ли беседу в клиента или федеративным
* Можно ли беседу
    * Федерация из рабочих групп и взаимодействия в клиент требует этого отправитель клиента либо чисто online или Скайп для гибридных бизнеса. Клиенты исключительно на месте не может содержать взаимодействия в клиентов или федерации для групп.
    * Если Скайп для учетной записи Business инициатора размещенные локально, этого пользователя нельзя использовать команды клиента для взаимодействия клиентов, а также для федерации. Этому пользователю необходимо использовать Скайп для клиента Business по взаимодействию и федерации.
    * Команды для передачи данных групп всегда является возможные в клиента.

## <a name="chats-and-calls-from-pre-existing-threads"></a>Обсуждений и вызывает с уже существующей потоков

### <a name="from-teams"></a>Из рабочих групп

Звонки или чаты работы с уже существующий, постоянного потока в группах будут направляться в так же, как этот поток, если маршрутизации может по-прежнему. 

Если уже существующей постоянного потока в группах собственного потока (то есть перенаправленные группы), дополнительные мгновенных сообщений и вызовов из этого потока будут поступать на группы. Если он был поток взаимодействия (то есть перенаправленные Скайп для бизнеса), дополнительные мгновенных сообщений и звонки будут поступать на Скайп для бизнеса (еще раз предполагается, что доступные параметры маршрутизации).

> [!NOTE]
> Уже существующей потоков в группах больше не может быть поддержки маршрутизации. Это может произойти, например если поток был взаимодействия поток для пользователя, который был обновлен для групп. Поскольку взаимодействия поток, поток будет перенаправлять Скайп для бизнеса, но этого пользователя больше не может использоваться Скайп для бизнеса для чата и телефонной. В этом случае потока будут отключены и не разрешать дальнейшей связи.

### <a name="from-skype-for-business"></a>Из Скайп для бизнеса

Скайп для бизнеса не имеет сохраняемость потока за пределы время ожидания сеанса SIP (10 мин). Чаты и вызовов из существующего потока в Скайп для бизнеса до истечения срока действия сеанса SIP будут направляться в так же, как поток. Звонки и беседы из существующего обсуждения в Скайп для бизнеса за пределы время ожидания сеанса SIP будут направляться Скайп удаленного пользователя для бизнеса, независимо от того, какой компьютер исходный поток поступил от на стороне другой стороны.

# <a name="presence"></a>Присутствие

При наличии ситуации, где некоторым пользователям с помощью команды клиента и другие пользователи по-прежнему используется Скайп для клиента Business может иметь количество пользователей, использующих обоих этих клиентах. По-прежнему требуется состояния присутствия для общего доступа всех пользователей вне зависимости от того, какой клиент имеет отдельного пользователя. Когда это общего доступа в организации, пользователи могут лучше определить, подходит ли она для запуска беседы или звонок.

Например если автором чата или звонка необходимо правильному целевому Скайп целевого объекта для клиента Business, ее будет Скайп для клиента Business сведения о присутствии, которое будет отображаться для этого отправитель. Если он должен land на целевой группы клиента, является клиентских групп присутствия, которое должно отображаться.

Чтобы узнать, какие поведение следует ожидать, то необходимо понять, что сведения о присутствии распределено зависимости от режима совместной работы пользователя:

* Если пользователь находится в режиме TeamsOnly, любой пользователь, (ли в группы или Скайп для бизнеса) будут видеть сведения о присутствии пользователя TeamsOnly групп
* Если пользователь находится в любой из SfB\* будут видеть режимы (SfbOnly, SfbWithTeamsCollab, SfbWithTeamsCollabAndMeetings), а затем другой пользователь (ли в группы или Скайп для бизнеса), SfB\* Скайп пользователя для бизнеса сведений о присутствии
* Если пользователь является острова (устаревший) режим, сведения о присутствии в группах и сведения о присутствии в Скайп для бизнеса, независимый от (значения нет) и других пользователей будут видеть одну или другие сведения о присутствии в зависимости от того, являются ли они тому же владельцу или в federat пользователя о-ва Ed клиента и который они используют клиента
    * Из рабочих групп любой другой пользователь в пределах того же клиента будут видеть сведения о присутствии пользователей о-ва групп; Это согласуется с предыдущей таблицы маршрутизации в клиента
    * Из группы других пользователей в федеративных клиентов будут видеть Скайп пользователя о-ва для бизнеса сведения о присутствии; Это согласуется с федеративным из предыдущей таблицы маршрутизации
    * Из Скайп для бизнеса других пользователей будут видеть Скайп пользователя о-ва для бизнеса присутствия (в клиента и федеративных); Это согласуется с выше таблицы маршрутизации

> [!NOTE]
> Это последнее изменение от предыдущей реализации (называемые объединенные сведения о присутствии) которого показывают объединенный сводный наличие целевой группы и Скайп пользователей. Чтобы выровнять сведения о присутствии и возможности доступа в уже существующей потоки, целевой присутствия, предоставляемые в том, что поток должен синхронизируется с маршрутизации в потоке, предполагается применение маршрутизации невозможно.
