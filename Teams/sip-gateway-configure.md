# <a name="configure-sip-gateway"></a>Настройка шлюза SIP

В этой статье объясняется, как настроить шлюз SIP, чтобы ваша организация могли использовать совместимые устройства SIP с Microsoft Teams. Сведения о том, что шлюз SIP может сделать для вашей организации и какое оборудование, программное обеспечение и лицензии требуются вашей организации, см. в описании [плана для шлюза SIP](sip-gateway-plan.md).

Перед настройкой шлюза SIP выполните следующие действия.

- **Сбросьте SIP-устройства до заводских параметров по умолчанию.** Вы или пользователи вашей организации должны сбросить параметры по умолчанию для каждого устройства SIP, используемого со шлюзом SIP. Сведения о том, как это сделать, см. в инструкциях производителя.

- **Откройте брандмауэр в Microsoft 365 и Teams.** Откройте брандмауэр сети для трафика Microsoft 365 и Teams, как описано в Office 365 [URL-адресах и диапазонах IP-адресов](/microsoft-365/enterprise/urls-and-ip-address-ranges). Правила брандмауэра необходимы только для исходящего трафика.

- **Убедитесь, что устройства SIP не находятся за прокси-сервером.** Убедитесь, что трафик http/s обходит любой корпоративный прокси-сервер HTTP/s.

- **Откройте UDP-порт.** Откройте UDP-порт в диапазоне от 49152 до 53247 для диапазонов IP-адресов 52.112.0.0/14 и 52.120.0.0/14.

- **Откройте TCP-порт.** Откройте TCP-порт 5061 для диапазонов IP-адресов 52.112.0.0/14 и 52.120.0.0/14.

- **Откройте следующие конечные точки HTTPS (IP-адреса и URL-адреса):**

  - 13.75.175.145
  - 52.189.219.201
  - 51.124.34.164
  - 13.74.250.91
  - 13.83.55.36
  - 23.96.103.40
  - https://blobsdgapac.blob.core.windows.net
  - https://blobsdgemea.blob.core.windows.net
  - https://blobsdgnoam.blob.core.windows.net
  - https://httpblobsdgapac.blob.core.windows.net
  - https://httpblobsdgemea.blob.core.windows.net
  - https://httpblobsdgnoam.blob.core.windows.net

## <a name="verify-that-sip-gateway-is-available-for-your-organization"></a>Проверка доступности шлюза SIP для вашей организации

1. Войдите в Центр [администрирования Teams](https://admin.teams.microsoft.com/).

2. Слева выберите устройства **Teams** и посмотрите, отображается ли вкладка **устройств SIP** . Если это так, служба шлюза SIP включена для вашей организации.

## <a name="enable-sip-gateway-for-the-users-in-your-organization"></a>Включение шлюза SIP для пользователей в организации

Шлюз SIP для организации можно включить двумя способами: с помощью Центра администрирования Teams или с помощью командлета PowerShell.

### <a name="by-using-teams-admin-center"></a>Использование Центра администрирования Teams

Чтобы включить шлюз SIP в Центре администрирования Teams, выполните следующие действия.

1. Перейдите в [Центр администрирования Teams](https://admin.teams.microsoft.com/)

2. Слева в разделе **"Голосовая связь**" выберите **политики звонков**.

3. Справа в разделе **"Управление политиками**" выберите соответствующую политику звонков, назначенную пользователям, или при необходимости создайте политику звонков и назначьте ее нужным пользователям.

4. Выберите **"Управление политиками**", выберите политику и нажмите кнопку "Изменить **"**.

5. Включите параметр для **устройств SIP, которые можно использовать для вызовов**, а затем нажмите кнопку **"Сохранить"**.


### <a name="by-using-powershell"></a>С помощью PowerShell

Шлюз SIP также можно включить с помощью командлета [PowerShell Set-CsTeamsCallingPolicy](/powershell/module/skype/set-csteamscallingpolicy?view=skype-ps) . Чтобы включить пользователей для устройств SIP, выберите политику и задайте для атрибута `-AllowSIPDevicesCalling` значение `True`. Значение по умолчанию — `False`это то, что пользователи не смогут использовать свои устройства SIP, если вы не включите их.

> [!NOTE]
> - Распространение политики может занять до 24 часов.

## <a name="set-the-sip-gateway-provisioning-server-url"></a>Задание URL-адреса сервера подготовки шлюза SIP

URL-адрес сервера подготовки шлюза SIP можно задать на DHCP-сервере. Пользователи, которые работают удаленно, должны настроить его вручную.

### <a name="using-dhcp"></a>Использование DHCP

Для каждого устройства SIP задайте один из следующих URL-адресов сервера подготовки шлюза SIP: 

- EMEA: `http://emea.ipp.sdg.teams.microsoft.com`
- Америкас: `http://noam.ipp.sdg.teams.microsoft.com`
- APAC: `http://apac.ipp.sdg.teams.microsoft.com`

Добавьте устройства SIP в организацию Teams, настроив указанный выше URL-адрес сервера подготовки шлюза SIP на DHCP-сервере. Дополнительные сведения о DHCP-сервере см. в статье ["Развертывание DHCP и управление ими"](/training/modules/deploy-manage-dynamic-host-configuration-protocol). Кроме того, с помощью параметра DHCP 42 можно указать NTP-сервер, а параметр DHCP 2 — смещение от времени в формате UTC в секундах. Устройства в вашей организации будут перенаправлены на сервер подготовки шлюза SIP. Для успешно подготовленных телефонов SIP отобразится логотип Teams и кнопка для входа.

Убедитесь, что устройства SIP имеют минимальную поддерживаемую версию встроенного ПО для подключения. Во время подключения шлюз SIP отправит пользовательский интерфейс конфигурации и проверки подлинности по умолчанию на устройство. Сведения о требуемой версии встроенного ПО для устройств SIP см. в разделе [Plan for SIP Gateway](sip-gateway-plan.md).

### <a name="manually"></a>Вручную

Пользователи, которые работают удаленно, должны вручную настроить URL-адрес сервера подготовки на своем устройстве SIP, выполнив следующие действия.

1. Откройте **параметры** на устройстве и получите IP-адрес устройства.

2. Откройте окно браузера, введите IP-адрес устройства, выполните вход (при необходимости) и настройте URL-адрес сервера подготовки в веб-программе устройства.

3. В **разделе "Параметры****" или** "Дополнительные параметры" веб-программы введите URL-адрес сервера подготовки, показанный выше.

> [!NOTE]
> - К шлюзу SIP можно подключить только совместимые устройства SIP. 
> - Перед подключением ip-телефонов Cisco необходимо включить встроенное ПО с несколькими платформами. Дополнительные сведения см. в руководстве [по преобразованию встроенного ПО Cisco](https://www.cisco.com/c/en/us/products/collateral/collaboration-endpoints/unified-ip-phone-7800-series/guide-c07-742786.html).
> - Для телефонов Yealink используйте вариант 66.
> - Для телефонов Cisco, Poly и AudioCode используйте вариант 160. 
> - Для устройств Cisco добавьте **/$PSN.xml** URL-адрес сервера подготовки.


## <a name="configure-conditional-access"></a>Настройка условного доступа

Условный доступ — это функция Azure Active Directory (Azure AD), которая помогает обеспечить правильное управление и безопасность устройств, которые имеют доступ к ресурсам Microsoft 365. Шлюз SIP проверяет подлинность устройств SIP с помощью Azure AD, поэтому если ваша организация использует условный доступ для устройств в корпоративной сети, следует исключить следующие IP-адреса:

- Северная Америка:
    - Восточная часть США: 52.170.38.140
    - Западная часть США: 40.112.144.212
-   Регион EMEA:
    - Северная часть ЕС: 40.112.71.149
    - Западная часть ЕС: 40.113.112.67
-   Регион APAC:
    - Восточная Австралия: 20.92.120.71
    - Юго-Восточная Австралия: 13.73.115.90

Дополнительные сведения см. в [разделе диапазонов IP-адресов](/azure/active-directory/conditional-access/location-condition#ip-address-ranges).

## <a name="provision-and-enroll-sip-devices-as-common-area-phones"></a>Подготовка и регистрация устройств SIP в качестве телефонов общего пользования

> [!NOTE]
> Устройство SIP должно быть подключено к шлюзу SIP, прежде чем его можно будет зарегистрировать.

Чтобы упростить задачи, можно зарегистрировать устройства SIP в Центре администрирования Teams по одному или пакетно. Вот как это сделать:

1. Войдите в Центр [**администрирования Teams**](https://admin.teams.microsoft.com).

2. Выберите **устройства Teams** > **SIP**.

3. В правом верхнем углу выберите " **Actions** > **Provision devices" (** Подготовка действий) и выполните одно из следующих действий:

  - **Чтобы подготовить одно устройство:**

     а) В **разделе "Ожидание активации**" выберите " **Добавить"**.

     б) На панели **"Добавление MAC-адресов**" введите **MAC-адрес** и расположение устройства, а затем нажмите кнопку "Применить **"**.
     
     в. В **разделе "Ожидание активации**" выберите только что добавленное устройство, а затем выберите " **Создать код проверки"**.
     
     г. На панели **"Подготовка устройств** " в разделе " **Код** проверки" запишите код проверки для устройства SIP.

   - **Чтобы подготовить множество устройств:**

     а) В **разделе "Ожидание активации**" справа выберите " **Экспорт** " (значок Microsoft Excel).
     
     б) На панели **"Подготовка устройств** " в разделе **"Отправка нескольких MAC-адресов**" выберите " **Скачать шаблон"**.
     
     в. **СохранитеTemplate_Provisioning.csv** на компьютере и заполните **поля MAC-идентификатора** и **расположения**.
    
     г. На панели **"Подготовка устройств** " выберите " **Отправить несколько MAC-адресов"**. 

     E. Справа на панели отправки **MAC-адресов** выберите "Выбрать файл **"** иTemplate_Provisioning.csvфайл, содержащий ваши данные.

     F. На панели **"Подготовка устройств**" в разделе "Ожидание активации" выберите  **устройство, а** затем выберите "Создать код проверки", чтобы создать однофакторный код проверки для каждого подготовленного устройства. Запишите код проверки для каждого устройства SIP.

4. На устройстве SIP найдите код компонента регистрации, за которым следует код проверки. На устройстве SIP \*найдите код функции регистрации 55* (используется шлюзом SIP для проверки кода однофакторной проверки регистрации), за которым следует код проверки, созданный в Teams Администратор Center для этого конкретного устройства. Например, если код проверки 123456, \*наведите 123456 55\*, чтобы зарегистрировать устройство.

5.  На панели **"Подготовка устройств** " в разделе **"Ожидание входа**" выберите **"Выход"**.

6. В **диалоговом окне входа** пользователя скопируйте или запишите код связывания устройства SIP.

7. Перейдите в [https://microsoft.com/devicelogin](https://microsoft.com/devicelogin)раздел "Ввод **кода"**, введите код связывания устройства SIP и нажмите кнопку **"Далее"**.

8. На **странице входа Майкрософт** в поле **Email или** телефона введите адрес электронной почты для устройства SIP, а затем нажмите кнопку **"Далее"**.

9. На странице **"Пароль** " введите пароль для адреса электронной почты для устройства SIP, а затем выберите **"Войти"**.

10. На странице **шлюза устройств SIP Teams** выберите " **Продолжить"**.

## <a name="how-to-sign-in-and-sign-out"></a>Вход и выход

Для личных устройств пользователей поддерживается только локальный вход. Чтобы выйти устройства из центра Администратор, выполните следующие действия.

1. Войдите в Центр [**администрирования Teams**](https://admin.teams.microsoft.com).

2. Выберите **устройства Teams** > **SIP**.

3. Справа выберите устройство SIP, а затем выберите **"Выйти"**.


### <a name="user-pairing-and-sign-in"></a>Связывание пользователей и вход

Чтобы связать устройство SIP после того, как пользователь выполнит проверку подлинности с использованием корпоративных учетных данных, пользователь должен:

1. Нажмите **клавишу SIGN-in** на телефоне SIP, чтобы отобразить URL-адрес проверки подлинности и код связывания. Код связывания учитывает время. Если срок действия истекает, пользователь должен нажать кнопку **"** Назад" на телефоне и снова запустить процесс входа.

2. Перейдите по URL-адресу проверки подлинности на рабочем столе или мобильном браузере пользователя и используйте корпоративные учетные данные для входа.

3. Введите код связывания, отображаемый на телефоне SIP, в веб-приложение проверки подлинности, чтобы связать телефон SIP с учетной записью пользователя. При успешном входе, который может занять некоторое время, на телефоне SIP отобразится номер телефона и имя пользователя, если устройство его поддерживает.

> [!NOTE]
> Расположение устройства, отображаемого в приложении веб-проверки подлинности Azure Active Directory, — это центр обработки данных шлюза SIP, к которому подключено устройство. Телефоны SIP в области не поддерживают OAuth, поэтому шлюз SIP выполняет проверку подлинности пользователя через веб-приложение проверки подлинности, а затем сопошает устройство с учетными данными пользователя. Дополнительные сведения см[. платформа удостоверений Майкрософт и поток предоставления авторизации устройств OAuth 2.0](/azure/active-directory/develop/v2-oauth2-device-code).

### <a name="sign-out"></a>Выход

Чтобы выйти из системы, пользователь устройства может:

- Нажмите **клавишу Sign-Out** на устройстве SIP и выполните действия, описанные на устройстве. 

Чтобы выйти из устройства в Центре администрирования Teams, выполните следующие действия.

1. Войдите в Центр [**администрирования Teams**](https://admin.teams.microsoft.com).

2. Выберите **устройства Teams** > **SIP**.

3. В области устройств **SIP** справа выберите устройство.

4.  В области сведений **устройства выберите** вкладку "Сведения", а в правом верхнем углу меню "Действия"  выберите **"Выйти"**. 

## <a name="view-and-monitor-sip-devices"></a>Просмотр и мониторинг устройств SIP

Вы можете просматривать и отслеживать данные инвентаризации устройств SIP в Центре администрирования Teams после того, как пользователи устройств войдите в систему по крайней мере один раз. Вот как это сделать:

1. Войдите в Центр [администрирования Teams](https://admin.teams.microsoft.com/).

2. Выберите **устройства Teams** > **SIP**. Все устройства SIP, выполнив вход, перечислены справа.

## <a name="restart-a-sip-device"></a>Перезапуск устройства SIP

1. Войдите в Центр [администрирования Teams](https://admin.teams.microsoft.com).

2. Выберите **устройства Teams** > **SIP**. 

3. Справа выберите устройство SIP, которое требуется перезапустить, а затем нажмите кнопку " **Перезапустить"**.


> [!NOTE]
> - Удаление устройства SIP из клиента в настоящее время недоступно в Центре администрирования Teams. 
> - Выполнение команды зависит от доступности устройства и может не соответствовать статусу выполнения, показанному в Центре администрирования Teams. При попытке включить шлюз SIP на устройстве, которое его не поддерживает, команда не будет выполнена.

## <a name="sync-policy-changes-to-sip-devices-to-enforce-policies"></a>Изменения политики синхронизации на устройствах SIP для принудительного применения политик

Сведения о пользователе и политики будут извлекаться на устройства SIP при входе пользователей. Любые изменения политики для пользователей, выполнившего вход, будут синхронизированы с устройством в течение одного часа. На устройствах должна периодически обновляться регистрация с помощью шлюза SIP. Телефоны SIP зависят от перенаправления вызовов, поэтому администратор должен задать `AllowCallRedirect` для атрибута `Set-CsTeamsCallingPolicy` значение `Enabled`.


## <a name="set-a-sip-devices-ui-language"></a>Настройка языка пользовательского интерфейса устройства SIP

Устройство SIP обычно может отображать информацию на многих языках. Настройка языка пользовательского интерфейса влияет на его интерфейс, включая обратимые клавиши и системные сообщения о входе и выходе. Настройка языка пользовательского интерфейса выполняется на сервере подготовки, с помощью DHCP-сервера или вручную путем добавления строки кода в URL-адрес, как показано в следующих примерах.

Как задать немецкий язык для телефонов Polycom, AudioCodes и Yealink:
- `http://emea.ipp.sdg.teams.microsoft.com/lang_de`

Настройка японского языка для телефонов Cisco:
- `http://emea.ipp.sdg.teams.microsoft.com/lang_ja/$PSN.xml`

## <a name="microsoft-teams-and-ipv6"></a>Microsoft Teams и IPv6

Шлюз SIP поддерживает только протокол IPv4. Служба и клиент Microsoft Teams поддерживают протоколы IPv4 и IPv6. Если вы хотите управлять обменом данными с Microsoft Teams, используйте диапазоны IP-адресов в URL-адресах [и диапазонах IP-адресов Microsoft 365](/microsoft-365/enterprise/urls-and-ip-address-ranges).

## <a name="emergency-calling"></a>Экстренные вызовы

Шлюз SIP поддерживает динамические экстренные вызовы (динамические E911) для совместимых устройств SIP, которые используют сетевые атрибуты по сети. Эти атрибуты подготавливаются в Центре администрирования Teams и могут быть сочетанием локального IP-адреса и длины подсети, идентификатора корпуса и номера сетевого порта. Для устройств, которые не используют атрибуты расположения или если расположение не разрешается динамически по какой-либо причине, шлюз SIP будет продолжать поддерживать экстренные вызовы на основе зарегистрированных адресов. В настоящее время зарегистрированные адреса не поддерживаются для сценариев прямой маршрутизации. Дополнительные сведения о экстренных вызовах см. в статье ["Планирование экстренных](/microsoftteams/what-are-emergency-locations-addresses-and-call-routing) вызовов и управление ими".
