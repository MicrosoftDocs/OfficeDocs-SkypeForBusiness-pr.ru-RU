# <a name="configure-sip-gateway"></a>Настройка шлюза SIP

В этой статье объясняется, как настроить шлюз SIP таким образом, чтобы ваша организация может использовать совместимые устройства SIP с Microsoft Teams. Чтобы узнать, что шлюз SIP может сделать для вашей организации, какое оборудование, программное обеспечение и лицензии необходимы для него, ознакомьтесь со статьей Планирование [шлюза SIP](sip-gateway-plan.md).

Перед настройкой шлюза SIP сделайте следующее:

- **Перезагрудите устройства SIP до заводских настроек по умолчанию.** Вы или пользователи вашей организации должны восстановить заводские параметры по умолчанию для каждого устройства SIP, используемой со шлюзом SIP. Чтобы узнать, как это сделать, см. инструкции производителя.

- **Откройте брандмауэр, чтобы Microsoft 365 и Teams.** Откройте брандмауэр своей сети для Microsoft 365 и Teams трафика, как описано в Office 365 [URL-адреса и диапазоны IP-адресов](/microsoft-365/enterprise/urls-and-ip-address-ranges).

- **Убедитесь, что устройства SIP не отстают от прокси-сервера.** Убедитесь, что трафик http/s обошел любой корпоративный http/s-прокси.

- **Откройте порт UDP.** Откройте порт UDP в диапазоне от 49152 до 53247 для IP-адресов от 52.112.0.0/14 до 52.120.0.0/14.

- **Откройте порт TCP.** Откройте порт TCP 5061 для IP-адресов от 52.112.0.0/14 до 52.120.0.0/14.

- **Откройте следующие конечные точки https (IP-адреса и URL-адреса):**

  - 13.75.175.145
  - 52.189.219.201
  - 51.124.34.164
  - 13.74.250.91
  - 13.83.55.36
  - 23.96.103.40
  - https://blobsdgapac.blob.core.windows.net
  - https://blobsdgemea.blob.core.windows.net
  - https://blobsdgnoam.blob.core.windows.net
  - https://httpblobsdgapac.blob.core.windows.net
  - https://httpblobsdgemea.blob.core.windows.net
  - https://httpblobsdgnoam.blob.core.windows.net

## <a name="verify-that-sip-gateway-is-available-for-your-organization"></a>Проверка того, что шлюз SIP доступен для вашей организации

1. Во sign in to the [Teams admin center](https://admin.teams.microsoft.com/).

2. В левой области выберите Teams **устройств и** посмотрите, отображается ли вкладка **Устройства SIP**. Если это так, служба шлюза SIP включена для вашей организации.

## <a name="enable-sip-gateway-for-the-users-in-your-organization"></a>Включить шлюз SIP для пользователей в организации

Шлюз SIP для организации можно включить двумя способами: с помощью Teams центра администрирования или с помощью powerShell.

### <a name="by-using-teams-admin-center"></a>Использование центра Teams администрирования

Чтобы включить шлюз SIP в Центре Teams администрирования, выполните следующие действия:

1. Перейдите в [центр Teams администрирования](https://admin.teams.microsoft.com/)

2. В левой области в **области Голосовая почта** выберите **Политики звонков**.

3. В правой области **Управление** политиками выберите назначенную пользователям политику звонков или при необходимости создайте новую политику звонков и назначьте ее необходимым пользователям.

4. Выберите **Управление политиками**, выберите политику, а затем выберите **Изменить**.

5. Включите параметр для **устройств SIP, которые можно использовать для звонков**, а затем выберите **Сохранить**.


### <a name="by-using-powershell"></a>С помощью PowerShell

Вы также можете включить шлюз SIP с помощью [cmdlet Set-CsTeamsCallingPolicy](/powershell/module/skype/set-csteamscallingpolicy?view=skype-ps) PowerShell. Чтобы включить пользователей для устройств SIP, выберите политику и установите для атрибута `-AllowSIPDevicesCalling` параметр `True`. По умолчанию используется значение `False`, поэтому пользователи не смогут использовать свои устройства SIP, пока вы не ввяжете их.


> [!NOTE]
> - Распространение политики может занять до 24 часов.

## <a name="set-the-sip-gateway-provisioning-server-url"></a>Настройка URL-адреса сервера подготовка шлюза SIP

ВЫ можете настроить URL-адрес сервера подготовка шлюза SIP на сервере DHCP. Пользователи, которые работают удаленно, должны настроить его вручную.

### <a name="using-dhcp"></a>Использование DHCP

Для каждого устройства SIP за установите один из следующих URL-адресов серверов для подстроки шлюза SIP: 

- EMEA: `http://emea.ipp.sdg.teams.microsoft.com`
- Америкас: `http://noam.ipp.sdg.teams.microsoft.com`
- APAC: `http://apac.ipp.sdg.teams.microsoft.com`

Добавьте устройства SIP в организацию Teams, настроив на сервере DHCP url-адрес сервера для подстроки шлюза SIP выше. Дополнительные информацию о DHCP-сервере см. в этой [теме](/learn/modules/deploy-manage-dynamic-host-configuration-protocol). Кроме того, с помощью параметра DHCP 42 можно указать сервер NTP, а параметр DHCP 2 — смещение от UTC за секунды. Устройства в вашей организации будут перенаправлиться на сервер подготовка шлюза SIP. На телефонах SIP, которые успешно Teams логотип, и мягкие кнопки для регистрации.

Убедитесь, что устройства SIP находятся в минимально поддерживаемой версии подержитее для вбора в новую версию. Во время входа шлюз SIP подавляет пользовательский интерфейс настройки и проверки подлинности по умолчанию на устройство. Чтобы узнать необходимую версию программного обеспечения для устройств SIP, см. планирование [шлюза SIP](sip-gateway-plan.md).

### <a name="manually"></a>Вручную

Пользователи, которые работают удаленно, должны вручную настроить URL-адрес сервера для подготовка на своем устройстве SIP с помощью следующих действий:

1. Откройте **Параметры** на устройстве и получите ЕГО IP-адрес.

2. Откройте окно браузера, введите IP-адрес устройства, войдите в систему (при необходимости) и настройте URL-адрес сервера регистрации в веб-службе устройства.

3. В **Параметры** или **Дополнительные** параметры веб-приложения введите URL-адрес сервера, показанный выше.

> [!NOTE]
> - К шлюзу SIP можно использовать только совместимые устройства SIP. 
> - Ip-телефоны Cisco должны мигать в многоплатформовом ПО, прежде чем их можно будет использовать. Чтобы узнать, как это сделать, см. [руководство по преобразованию программного обеспечения Cisco](https://www.cisco.com/c/en/us/products/collateral/collaboration-endpoints/unified-ip-phone-7800-series/guide-c07-742786.html).
> - Для телефонов Yealink используйте вариант 66.
> - Для телефонов Cisco, Poly и AudioCode используйте вариант 160. 
> - Для устройств Cisco в **URL-адрес сервераPSN.xml/$** .

## <a name="provision-and-enroll-sip-devices-as-common-area-phones"></a>Подготовка и регистрация устройств SIP в качестве общих телефонов в области
> [!NOTE]
> Устройство SIP должно быть вовсю вовсю в шлюз SIP, прежде чем его можно будет зарегистрировать.

Чтобы упростить задачи, вы можете зарегистрировать устройства SIP в центре администрирования Teams по одному или группами. Вот как это сделать:

1. Войдите в центр [**Teams администрирования**](https://admin.teams.microsoft.com).

2. Выберите **Teams** **устройствSIP** > .

3. В правом верхнем нижнем конце выберите **Устройства** **ActionsProvision** >  и выполните одно из указанных выше действий.

  - **Чтобы получить одно устройство:**

     в. В **области Ожидание активации** выберите устройство, которое вы только что добавили, и выберите Создать **код проверки**.
     
     г. В области **Подготовка устройств** в области **Код проверки** обратите внимание на код проверки для устройства SIP.

   - **Чтобы получить несколько устройств:**

     а) В **области Ожидание активации** справа выберите **Экспорт** (значок Microsoft Excel).
     
     б) В области **Подготовка устройств** в Upload **адресов MAC** выберите **скачать шаблон**.
     
     в. **СохранитеTemplate_Provisioning.csv** на компьютере и заполните поля **"Mac id**" и **"Расположение**".
    
     г. В области **Подготовка устройств** выберите Upload **адреса MAC**. 

     E. В правой области адресов **Upload Mac** выберите Выберите **файл и выберите** файлTemplate_Provisioning.csv, содержащий ваши данные.****

     F. В области **Подготовка устройств** в области Ожидание активации выберите устройство, а затем выберите Создать код  проверки, чтобы создать разовую код проверки для каждого из них. Обратите внимание на код проверки для каждого устройства SIP.

На устройстве SIP наберите код функции регистрации 55* (используется шлюзом SIP для регистрации для проверки кода одновратной проверки), а затем код проверки, \*который создается в Центре администрирования Teams для этого устройства. ## <a name="how-to-sign-in-and-sign-out"></a>Вход и выход

Для личных устройств пользователей поддерживается только локальный вход. Чтобы выйти устройства из Центра администрирования, выполните следующие действия:

1. Войдите в центр [**Teams администрирования**](https://admin.teams.microsoft.com).

2. Выберите **Teams** **устройствSIP** > .

3. В правой области выберите устройство SIP, а затем выберите **Выйти**.


### <a name="user-pairing-and-sign-in"></a>Сопряжение пользователей и вход в нее

Чтобы сопряжение устройства SIP после аутентификации пользователя с использованием корпоративных учетных данных, пользователю необходимо:

1. Нажмите **кнопку Войти** на телефоне SIP, чтобы отобразить URL-адрес проверки подлинности и код сопряжения. Код сопряжения является конфиденциальным с емким временем. Если срок действия истекает, пользователю необходимо  нажать кнопку Назад на телефоне и снова начать вход.

2. Перейдите по URL-адресу проверки подлинности в браузере на компьютере или мобильном устройстве пользователя и используйте корпоративные учетные данные для входа.

3. Введите код сопряжения, отображающийся на телефоне SIP, в веб-приложении для проверки подлинности, чтобы привяжеть телефон SIP к учетной записи пользователя. При успешном входе, который может занять некоторое время, на телефоне SIP будут отображаться номер телефона и имя пользователя, если устройство поддерживает его.

> [!NOTE]
> Расположение устройства, показанного в веб-приложении Azure Active Directory проверки подлинности, — это центр обработки данных шлюза SIP, к которому подключено устройство. Телефоны SIP в области действия не имеют OAuth, поэтому шлюз SIP проходит проверку подлинности пользователя через веб-приложение для проверки подлинности, а затем сопряжение устройства с учетными данными пользователя. Подробнее об этом можно узнать платформа удостоверений Майкрософт и о предоставлении авторизации устройств [в OAuth 2.0](/azure/active-directory/develop/v2-oauth2-device-code).

### <a name="sign-out"></a>Выход

Чтобы выйти из устройства в Центре администрирования Teams:

1. Войдите в центр [**Teams администрирования**](https://admin.teams.microsoft.com).

2. Выберите **Teams** **устройствSIP** > .

3. В правой области устройств **SIP** выберите устройство.

4. На устройстве в области **Сведения** выберите вкладку Сведения,  а затем в правом верхнем меню Действия выберите **Выйти**.

## <a name="sync-policy-changes-to-sip-devices-to-enforce-policies"></a>Изменение политики синхронизации устройств SIP для применения политик

Сведения о пользователях и политики будут получаться на устройствах SIP при входе пользователей. Все изменения политик, внесенные после этого для пользователей, во внесенных в нее, будут синхронизированы с устройством в течение часа. Регистрация устройств должна периодически обновляться с помощью шлюза SIP. Телефоны SIP зависят от перенаправления звонка, поэтому администратор должен установить `AllowCallRedirect` для атрибута `Set-CsTeamsCallingPolicy` в .`Enabled`


## <a name="set-a-sip-devices-ui-language"></a>Настройка языка пользовательского интерфейса устройства SIP

Как правило, устройство SIP может отображать информацию на разных языках. Настройка языка пользовательского интерфейса влияет на его интерфейс, включая сжатие и системные сообщения для входов и выходов. Настройка языка пользовательского интерфейса делается на сервере подготовка, с помощью DHCP-сервера или вручную путем в строки кода в URL-адрес, как по примеру в следующих примерах.

Настройка немецкого языка для телефонов Polycom, AudioCodes и Yealink:
- `http://emea.ipp.sdg.teams.microsoft.com/lang_de`

Настройка японского языка для телефонов Cisco:
- `http://emea.ipp.sdg.teams.microsoft.com/lang_ja/$PSN.xml` 


### <a name="supported-languages"></a>Поддерживаемые языки

|Название языка|Код языка]
|-------------|-------------|
|Английский (по умолчанию)|En       |
|Испанский      |Es           |
|Японский     |Ja           |
|Немецкий       |Де           |
|Французский       |Fr           |
|Португальский   |Pt           |

> [!Note]
> - Японский язык не поддерживается в Yealink и частично поддерживается Polycom VVX.
> - Если выбранный язык не поддерживается конечной точкой SIP, система по умолчанию выбирает английский.
> - Если параметр **lang_xx** не задан с помощью URL-адреса для подстроки, в качестве языка по умолчанию используется английский.
> - Если  текст войти в службу для экстренного вызова не переведен на другие языки, сокращенная версия на английском языке будет представлена при нажатии нажатие войти в следующих моделях IP-телефонов из-за ограничений на размер экрана:
>   - Poly VVX 150, VVX 201
>   - Cisco CP-6821, CP-7811, CP-7821, CP-7841, CP-7861
>   - Метка со сжатием голосовой почты обозначена текстом **VM** на всех языках poly VVX из-за ограничения длины строки.

## <a name="microsoft-teams-and-ipv6"></a>Microsoft Teams и IPv6

Шлюз SIP поддерживает только IPv4. Microsoft Teams службы и клиенты поддерживают IPv4 и IPv6. Если вы хотите управлять связью для Microsoft Teams, используйте диапазоны IP-Microsoft 365 [URL-адресов и диапазонов IP-адресов](/microsoft-365/enterprise/urls-and-ip-address-ranges).

## <a name="emergency-calling"></a>Экстренные вызовы

Шлюз SIP поддерживает только статические (также называемые зарегистрированными) адресами для экстренного обращения. В настоящее время зарегистрированные адреса не поддерживаются при прямой маршрутике. Дополнительные сведения о вызовах экстренных служб см. в [этой теме](/microsoftteams/what-are-emergency-locations-addresses-and-call-routing).
