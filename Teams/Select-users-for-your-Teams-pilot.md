<a name="select-users-for-your-teams-pilot"></a>Выберите пользователей для пилотной группы 
==================================

Успешные пилотного проекта проверяет как группами Майкрософт будет использоваться на ежедневной основе пользователями. Это помогает понять, являются пользователи, как они общения и совместной работы, и какие функции помогут оптимизировать их производительность.

Пилотного проекта должно содержать сотрудников в разных отделов, которые представляют различные роли. В то же время его следует включить разумное количество пользователей, чтобы ресурсы по управлению проектами может обрабатывать объем свои отзывы и предложения, которые будет создавать пилотного проекта.

Существует три основных атрибутов выбора оптимальной пилотных пользователей.

-   **Пользователей («,»):** Классификация пользователей, которые выгоднее сценариев совместной работы, поддерживаемые команды. С помощью оценки пользователя, можно включить хороший одни из различных ролей в организации для представления пользователей в пилотного развертывания.

-   **Варианты использования и сценарии («как»):** С помощью функции во время повседневной работы. Соответствующие сценарии можно использовать для создания плана пилотного тестирования.

-   **Функциональные возможности («что»):** Отдельные компоненты в группах. Можно оценить плану группами для определения вашего пилотных этапов на основе доступности функций, которые уже включены в группах или будет представлено позднее.

<a name="perform-user-persona-assessment"></a>Оценка пользователя пользователя
-------------------------------

Оценки пользователя пользователей, созданные на основе восемь Основные пользователи, часто встречающиеся в организациях, помогает распределить по категориям в соответствии с их требованиями обмена данными и совместной работы пользователей. Это позволяет сузить список пользователей, которым будет предложено выбрать для пилотного проекта во время планирования для широкого развертывания групп по времени.

<a name="map-use-cases-to-your-personas"></a>Сопоставление варианты использования вашей действующие лица
------------------------------

Когда речь идет о связи и совместной работы, существует четыре распространенных типов взаимодействия: бесед, собраний, совместной работы групп и проекта и коммуникации в масштабе. Во время оценки необходимо определить, как ваша организация сопоставляет к каждому из них, по отношению к вашей реализация из Скайп для бизнеса в группы.

![Четыре распространенных типов взаимодействия: бесед, собраний, совместной работы групп и проекта и коммуникации с горизонтальным] (./media/Select-users-for-your-Teams-pilot-image2.png "Четыре распространенных типов взаимодействия: бесед, собраний, совместной работы групп и проекта и коммуникации с горизонтальным")

Примеры сценариев для каждого типа взаимодействия приведены ниже. Используйте эти примеры в качестве отправной точки и сопоставьте варианты использования вашей организации эти сценарии для вашего пути к группам.

### <a name="scenario-01--conversations-ad-hoc"></a>Сценарий 01 – бесед (компьютер)
**Темы сценарий:** Ускорение связи для поддержки project momentum и взаимопонимание.

**Варианты использования примера:**

![Пример варианты использования для сценария 01: Ad hoc бесед] (./media/Select-users-for-your-Teams-pilot-image3.png "Пример варианты использования для сценария 01: Ad hoc бесед")

**Общие характеристики:**
-   Один на одном или нескольких участников
-   Компьютер компьютер, при необходимости
-   Краткое и интерактивные
-   На основе чата или – управляемых телефонного звонка

### <a name="scenario-02meetings"></a>Сценарий 02 — собраний
**Темы сценарий:** Упростите обсуждения и распространения информации через интерактивные беседы и мозгового.

**Варианты использования примера:**

![Пример варианты использования для сценария 02: собраний] (./media/Select-users-for-your-Teams-pilot-image4.png "Пример варианты использования для сценария 02: собраний")

**Общие характеристики:**
-   В момент собрания с помощью заданных повесток
-   Синхронная связь
-   Определенные аудитории может быть внутренним и внешним
-   Запланировано заранее
-   В режиме реального времени общий доступ к содержимому
-   Несколько докладчиков и участников

### <a name="scenario-03--collaboration"></a>Сценарий 03 – совместной работы
**Темы сценарий:** Ускорить общий доступ к информации и упростить специальные сведения для работы группы диска и повышения производительности.

**Варианты использования примера:**

![Пример варианты использования для сценария 03: совместная работа] (./media/Select-users-for-your-Teams-pilot-image5.png "Пример варианты использования для сценария 03: совместная работа")

**Общие характеристики:**
-   Связанными с проектом или командные усилия
-   Определенных аудиторий, локальные и удаленные vTeams (по всему миру)
-   Добавление новых участников со временем, необходимо быстро встроенного
-   Сохраняемый доступ к файлам, беседы и приложений
-   Синхронными и асинхронными связи (бесед, собрания)

### <a name="scenario-04--at-scale-communications"></a>Сценарий 04-коммуникаций в масштабе
**Темы сценарий:** Отправить сообщение в виде структурированных и профессиональный для большой группы участников.

**Варианты использования примера:**

![Пример примеры использования сценария 04: коммуникаций в масштаба] (./media/Select-users-for-your-Teams-pilot-image6.png "Пример примеры использования сценария 04: коммуникаций в масштаба")

**Общие характеристики:**
-   Заданные или open аудитории
-   Структурированный формат
-   Запланировано заранее
-   Односторонняя распространения информации
-   Управляемые участников (ответы на вопросы, аудио)
