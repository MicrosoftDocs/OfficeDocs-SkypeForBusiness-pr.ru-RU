# <a name="plan-your-teams-voice-solution"></a>Планирование голосового решения Teams 

Эта статья поможет вам выбрать решение Для голосовой связи Майкрософт, которое лучше всего для вашей организации. После этого в статье содержится план для контента, который позволит вам реализовать выбранное решение.

> [!NOTE]
> Рекомендации по планированию голосового решения Teams в рамках общего плана перехода на Teams с Skype для бизнеса Server см. в руководстве по [STN](upgrade-to-teams-on-prem-pstn-considerations.md)при переходе с локального решения Skype для бизнеса на Teams.

Вам может потребоваться простейшее решение &mdash; с телефонной системой с планом звонков. Это полностью облачное решение Майкрософт, которое предоставляет функции частного филиала Exchange (PBX) и звонки на телефонную сеть общего звонков (STN), как показано на приведенной ниже схеме. В этом случае корпорация Майкрософт является вашим оператором ННП.

![Схема 1. Телефонная система с планом звонков](media/voice-solutions-simple.png)

Если вы отвечаете "да" на одно из следующих решений, то вам может быть наиболее правильным решением будет телефонная система с планом звонков.

- План звонков доступен в вашем регионе.
- Не требуется сохранять текущий оператор ННР.
- Вы хотите использовать доступ к STN под управлением Майкрософт.

Однако ваша ситуация может быть более сложной. Например, у вас могут быть офисы в расположениях, где план звонков не доступен. Или вам может потребоваться комбинированное решение, которое поддерживает сложное развертывание на нескольких уровнях с разными требованиями для разных географических регионов. Корпорация Майкрософт поддерживает несколько решений: 

- Телефонная система с планом звонков
- Телефонная система со своим оператором ННР с прямой маршрутией
- Комбинированное решение, использующее как телефонную систему с планом звонков, так и телефонную систему с прямой маршрутией

## <a name="phone-system"></a>Телефонная система

Телефонная система — это технология корпорации Майкрософт, позволяющая управлять звонками и возможности Private Branch Exchange (PBX) в облаке Microsoft 365 или Office 365 с помощью Microsoft Teams.

Телефонная система работает с клиентами Teams или Skype для бизнеса и сертифицированными устройствами. Телефонная система позволяет заменить существующую систему УАКС набором функций, непосредственно доставленных из Microsoft 365 или Office 365. 

Звонки между пользователями в организации обрабатываются внутри телефонной системы и никогда не перенаправяются на телефонную сеть общего звонков (ПС). Это относится к звонкам между пользователями в организации, расположенных в разных географических областях, что позволяет снизить затраты на междугородние и внутренние звонки.

В этой статье рассматриваются следующие ключевые функции и функции телефонной системы, а также решения по развертыванию, которые необходимо учесть:

- [Удостоверение для звонков](#calling-identity)

Если используются вместе, автозаполнеющие и очереди вызовов могут без труда перена могут маршрутизации вызывающего вызова до соответствующего человека или отдела в организации.

### <a name="cloud-voicemail"></a>Облачная голосовая почта

Облачная голосовая почта на платформе служб голосовой почты Azure поддерживает только вклады голосовой почты в почтовые ящики Exchange. Она не поддерживает сторонние почтовые системы. 

Голосовая облачная почта включает функцию расшифровки голосовых сообщений, которая по умолчанию включена для всех пользователей в организации. В ваших деловых потребностях может потребоваться отключить транскрибцию голосовой почты для определенных пользователей или всех пользователей в организации.

Облачная голосовая почта автоматически настроена и подготовка для пользователей, которые получают лицензию на телефонную систему. Пользователям телефонной системы с почтовым ящиком Exchange потребуется выполнить дополнительные действия по настройке. 

Дополнительные сведения о облачной голосовой почте и ее конфигурации см. в следующих статьях:

### <a name="calling-identity"></a>Удостоверение для звонков

По умолчанию все исходящие звонки используют номер назначенного телефона в качестве идентификации вызывающего. Получатель звонка может быстро идентифицировать звонящего и решить, следует ли принять или отклонить вызов. Сведения о настройке и изменении или блокировании ИД вызываемой вызовы см. в сведениях о настройке ИД вызываемой вызовы [для пользователя.](set-the-caller-id-for-a-user.md) 

## <a name="public-switched-telephone-network-connectivity-options"></a>Параметры подключения к телефонной сети общего перейти

Телефонная система предоставляет все возможности УАКС для вашей организации. Однако чтобы пользователи могли звонить за пределы организации, необходимо подключить телефонную систему к телефонной сети общего звонков (ННР). Чтобы подключить телефонную систему к STN, можно выбрать один из следующих вариантов:

- [**Телефонная система с планом звонков.**](#phone-system-with-calling-plan) Облачное решение с Майкрософт в качестве оператора связи STN.

- [**Телефонная система со своим оператором STN**](#phone-system-with-own-pstn-carrier-with-direct-routing) с использованием прямой маршрутинга для подключения локальной среды к Teams.

Вы также можете выбрать сочетание параметров, которое позволяет создать решение для сложной среды или управлять многоэтапной миграцией (подробнее о переносе позже).

### <a name="phone-system-with-calling-plan"></a>Телефонная система с планом звонков 

Как было описано выше в этой статье, телефонная система с планом звонков — это облачное решение Майкрософт для пользователей Teams. Это простейший способ подключения телефонной системы Майкрософт к телефонной сети общего звонков (STN), позволяющий звонить на стационарные и мобильные телефоны по всему миру. В этом случае корпорация Майкрософт предоставляет вашей организации функции Private Branch Exchange (PBX) и выступает в качестве оператора STN, как показано на следующей схеме:

Если вы отвечаете "да" на одно из следующих решений, то вам может быть наиболее правильным решением будет телефонная система с планом звонков.

- План звонков доступен в вашем регионе.
- Не требуется сохранять текущий оператор ННР.
- Вы хотите использовать доступ к STN под управлением Майкрософт.

С помощью этого параметра: 

- Вы получаете телефонную систему Майкрософт с добавленными планами внутренних или международных звонков, которые позволяют звонить на телефоны по всему миру (в зависимости от уровня лицензии службы).

- Развертывание или обслуживание локального развертывания не требуется, так как план звонков выполняется из &mdash; Microsoft 365 или Office 365.

- Примечание. При необходимости вы можете подключить поддерживаемый пограничный контроллер сеанса (SBC) с помощью прямой маршрутировки для обеспечения связи со сторонними УАКС, аналоговыми устройствами и другими телефонными устройствами сторонних операторов, поддерживаемых SBC.

Для этого требуется непрерывное подключение к Microsoft 365 или Office 365.

### <a name="phone-system-with-own-pstn-carrier-with-direct-routing"></a>Телефонная система с оператором ННР с прямой маршрутией

Этот параметр подключает Телефонную систему Майкрософт к телефонной сети с помощью прямой маршрутии, как показано на следующей схеме: 

![Схема 5: телефонная система с прямой маршрутией](media/voice-solution-with-direct-routing.png)

Если вы отвечаете "да" на следующие вопросы, то для вас наиболее правильной будет телефонная система с прямой маршрутией.

- Вы хотите использовать Teams с телефонной системой.
- Необходимо сохранить текущего оператора связи ДНР.
- Вы хотите комбировать маршруты, когда некоторые звонки проходят через план звонков, некоторые через оператора.
- Вам необходимо работать со сторонними УАКС и оборудованием, например накладными страницами, аналоговыми устройствами и так далее.

С помощью этого параметра:

- Вы подключали свою телефонную систему СКА к телефонной системе Майкрософт без необходимости в дополнительном локальном программном обеспечении.

- С Microsoft Phone System можно использовать практически любой оператор телефонии.

- Вы можете настроить этот параметр и управлять им, а также настроить его и управлять им может оператор или партнер (спросите, предоставляет ли ваш оператор или партнер эту возможность).

- Вы можете настроить взаимодействия между телефонным оборудованием, таким как стороннее УАПС и аналоговые устройства и &mdash; &mdash; телефонная система Майкрософт.


Для этого параметра необходимо следующее:

- Непрерывное подключение к Microsoft 365 или Office 365.

- Развертывание и обслуживание поддерживаемой SBC.

- Контракт со сторонним оператором связи.
  (Если не развернуто подключение к сторонним УАП, аналоговым устройствам или другому оборудованию телефонной системы для пользователей телефонной системы с планом звонков.)

## <a name="phone-numbers-from-microsoft"></a>Номера телефонов от Майкрософт

Корпорация Майкрософт имеет два типа доступных номеров телефонов: номера абонентов (пользователей), которые могут быть назначены пользователям в вашей организации, и номера служб, доступные в качестве платных и бесплатных номеров служб.   Номера служб имеют более высокую пропускную способность по сравнению с абонентами и могут быть назначены таким службам, как аудиоконференция, автослужбу или очереди вызовов.

## <a name="migrate-your-existing-voice-solution-to-teams"></a>Перенос существующего голосового решения в Teams

В организации, обновляемой до Teams, конечная цель — переместить всех пользователей в режим TeamsOnly. Использование телефонной системы с Teams поддерживается только в том случае, если пользователь работает в режиме TeamsOnly. Если вам нужны основные сведения об обновлении до Teams, начните отсюда:

- [Начало перехода на Microsoft Teams](upgrade-start-here.md)
- [Схема перехода](upgrade-framework.md)
- [Стратегии обновления для ИТ-администраторов](upgrade-to-teams-on-prem-implement.md)

При переносе голосового решения в режим TeamsOnly возможны четыре сценария звонков:

- [**Пользователь Skype для бизнеса Online с планом звонков Майкрософт.**](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-online-with-microsoft-calling-plans) После обновления у этого пользователя сохранится план звонков Майкрософт.

- **[Пользователь в Skype для бизнеса Online с](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-online-with-on-premises-voice)** функцией локального голосового подключения через локальное приложение Skype для бизнеса или Cloud Connector Edition. Обновление пользователя до Teams должно быть согласовано с его переносом на Прямую маршрутику, чтобы обеспечить пользователям TeamsOnly функциональные возможности ННР.

- Пользователь в локальной службе Корпоративная голосовая связь Skype для бизнеса, который будет передвигаться на **[веб-сайт](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing)** и хранить локальное подключение по STN. Для переноса этого пользователя в Teams требуется переместить его локальное приложение Skype для бизнеса в облако и координировать переход с миграцией пользователя на Direct Routing. 

- Пользователь в локальной службе Skype для бизнеса с Корпоративная голосовая связь, который переходит на **[веб-сайт](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan)** и использует план Microsoft Calling.  Для переноса этого пользователя в Teams требуется переместить его локальное приложение Skype для бизнеса в облако и координировать перенос с помощью A) переноса номера телефона этого пользователя в план звонков Майкрософт или Б), назначая новый абонентский номер из доступных регионов.
