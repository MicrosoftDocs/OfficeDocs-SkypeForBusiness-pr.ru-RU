# <a name="plan-your-teams-voice-solution"></a>Планирование решения Teams голосовой почты 

В этой статье вы можете определить, какое голосовое решение Майкрософт лучше всего выбрать для вашей организации. После этого в этой статье содержится план для содержимого, которое позволит вам реализовать выбранное решение.

Возможно, вы захотите найти простейшее &mdash; решение телефонная система с планом звонков. Это полностью облачное решение Корпорации Майкрософт, которое предоставляет функции частных филиалов Exchange (УАЦ) и звонки на телефонную сеть общего звонков ,как показано на приведенной ниже схеме. С помощью этого решения корпорация Майкрософт является вашим оператором ПСПС.

![На схеме 1 показаны телефонная система с планом звонков.](media/voice-solutions-simple.png)

Если вы отвечаете да, то телефонная система с помощью плана звонков — это правильное решение.

- План звонков доступен в вашем регионе.
- Не требуется хранить текущий оператор ПСПС.
- Вы хотите использовать доступ к STN, управляемый Майкрософт.

Однако ваша ситуация может быть более сложной. Например, у вас могут быть офисы в расположениях, где план звонков не доступен. Или вам может потребоваться комбинированное решение, которое поддерживает сложное развертывание на нескольких уровнях с различными требованиями для разных географических местоположений. Корпорация Майкрософт поддерживает сочетание решений: 

- телефонная система с планом звонков
- телефонная система оператора связи через ПСОП с оператором Подключение
- телефонная система с собственным оператором ПСПС с прямой маршрутией
- Сочетание решений, в телефонная система плана звонков, телефонная система с операторами Подключение и /или телефонная система с прямой маршрутией

## <a name="phone-system"></a>Телефонная система

телефонная система технологией Майкрософт для включения управления звонками и личных ветвей Exchange (УАПС) в облаке Microsoft 365 с Microsoft Teams.

телефонная система работает с Teams или Skype для бизнеса клиентами и сертифицированными устройствами. телефонная система вы можете заменить существующую систему УАКС набором функций, непосредственно поставляемой из Microsoft 365. 

Звонки между пользователями в организации обрабатываются внутри организации телефонная система и никогда не перенаправяются на телефонную сеть общего звонков (STN). Это относится к звонкам между пользователями в организации, расположенных в разных географических областях, что позволяет снизить затраты на междугородние и внутренние вызовы.

### <a name="cloud-voicemail"></a>Облачная голосовая почта

облачная голосовая почта на платформе служб голосовой почты Azure, поддерживает депозиты голосовой почты только Exchange почтовым ящикам. Она не поддерживает сторонние почтовые системы. 

Голосовая облачная почта включает функцию расшифровки голосовых сообщений, которая по умолчанию включена для всех пользователей в организации. Для бизнес-потребностей может потребоваться отключить транскрибцию голосовой почты для определенных пользователей или всех пользователей в организации.

Для пользователей, которые не облачная голосовая почта, автоматически устанавливается и подготовка для пользователей после телефонная система лицензии. Для телефонная система пользователей с почтовым Exchange почтовым ящиком потребуется выполнить дополнительные действия по настройке. 

Дополнительные сведения о облачная голосовая почта конфигурации см. в следующих статьях:

Получатель звонка может быстро идентифицировать звонящего и решить, следует ли принять или отклонить вызов. ## <a name="public-switched-telephone-network-connectivity-options"></a>Параметры подключения к телефонной сети общего перейти

телефонная система предоставляет полный доступ к УАС для вашей организации. Однако для того чтобы пользователи могли звонить за пределы организации, необходимо подключить телефонная система к телефонной сети общего звонков (STN). Чтобы подключить телефонная система к STN, можно выбрать один из следующих вариантов:

- [**телефонная система с планом звонков**](pstn-connectivity.md#phone-system-with-calling-plan). Облачное решение, в качестве оператора связи по STN — Майкрософт.

- [**телефонная система с собственным оператором ПСОП с**](operator-connect-plan.md)помощью оператора Подключение. Операторы Подключение, если ваш оператор является участником программы microsoft Подключение, они могут управлять службой для перенаправки звонков по ПСОП Teams. Сведения о преимуществах и требованиях оператора Подключение см. в [Подключение.](operator-connect-plan.md)

- [**телефонная система с собственным оператором ННР,**](pstn-connectivity.md#phone-system-with-direct-routing) используя прямую маршрутику для подключения локальной среды к Teams.

Вы также можете выбрать сочетание параметров, которое позволяет создать решение для сложной среды или управлять многошаговой миграцией (подробнее о миграции позже).

Большинство телефонная система функций одинаковы независимо от того, какой способ подключения вы выбрали. Однако существуют некоторые отличия в функциональных возможностях, которые влияют на настройку некоторых функций телефонная система, таких как маршруты звонков и экстренные вызовы. Дополнительные сведения о вариантах подключения к ПСС и этих настройках конфигурации см. в этой [теме.](pstn-connectivity.md)
