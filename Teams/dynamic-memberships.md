# <a name="overview-of-dynamic-membership-for-teams"></a>Общие сведения о динамическом членстве в командах

Microsoft Teams поддерживает группы, связанные с группами Microsoft 365 с помощью *динамического участия.* Динамическое членство позволяет определять членство в группе с помощью одного или двух правил, которые проверяют определенные атрибуты пользователей в Azure Active Directory (Azure AD). Пользователи автоматически добавляются или удаляются в правильные группы по мере изменения их атрибутов или перейти в клиент и выйти из нее.

С динамическим членством вы можете настроить команды для определенных групп пользователей в организации. Возможные сценарии:
- В больнице можно создавать отдельные команды для медсестр, врача и врача для трансляции коммуникаций. Это особенно важно, если в больнице работают сотрудники временных служб.
- Университет может создать команду для всех преподавателей в конкретном вузе, включая сотрудников, которые часто меняются.
- Авиакомпании хотят создать группу для каждого рейса (например, во вторник после полудня без остановки из Чикаго в Чикаго в Чикаго) и при необходимости автоматически получить или удалить из нее часто меняющийся авиарейс.

При использовании этой функции участники группы обновляются автоматически на основе определенного набора критериев, а не вручную управляют членством. Для этого администратор клиента Azure AD Premium P1 лицензии и [](/azure/active-directory/users-groups-roles/groups-dynamic-membership) членство в группах для любых свойств Azure AD пользователя, если у вас есть клиент и учетная запись администратора.

Microsoft Teams от нескольких минут до 2 часов, чтобы отражать динамические изменения состава после их вступления в группу Microsoft 365 группы.

При использовании команд с динамическими группами:

- Правила определяют, кто является участником команды, но не владелец команды.
- Владельцы не смогут добавлять или удалять пользователей в качестве участников команды, так как участники определяются динамическими правилами группы.
- Teams не разрешает управление членами команды. Параметры добавления участников, изменения ролей участников, отправки и утверждения запросов на присоединиться и оставить команду скрытыми.

Чтобы создать команду с динамическим [](/azure/active-directory/users-groups-roles/groups-create-rule) членством, сначала создайте динамическую группу Microsoft 365, а затем создайте [группу из нее.](https://support.microsoft.com/en-us/office/create-a-team-from-an-existing-group-24ec428e-40d7-4a1a-ab87-29be7d145865)

Вы можете изменить существующую команду на динамическое членство. Сведения см. в Azure Active Directory [статических Azure Active Directory](/azure/active-directory/users-groups-roles/groups-change-type) группах.
