# <a name="authorize-guest-access-in-microsoft-teams"></a>Авторизация гостевого доступа в Microsoft Teams

В соответствии с требованиями вашей организации вы можете управлять функциями и возможностями гостевого доступа для Teams с четырьмя разными уровнями авторизации. Все уровни авторизации применяются к вашей организации Microsoft 365. Каждый уровень авторизации управляет взаимодействием с гостями, как показано ниже:

- **Azure Active Directory** : гостевой доступ в Teams использует ПЛАТФОРМУ Azure AD для бизнеса (B2B). Этот уровень авторизации управляет взаимодействием с гостями на уровне каталога, клиента и приложения.
- **Teams** : контролирует работу гостя только в Teams.
- **Группы microsoft 365** : управляет возможностями гостевых групп в группах и командах Microsoft 365.
- **SharePoint и onedrive** : управляет возможностями гостя в SharePoint, OneDrive, Microsoft 365 Groups и Teams.

Все эти уровни авторизации дают вам гибкость в настройке гостевого доступа для организации. Например, если вы не хотите разрешать гостевым пользователям в Teams, но хотите разрешить ее для всей Организации, просто отключите гостевой доступ в Teams. Еще один пример: вы можете включить гостевой доступ на уровнях Azure AD, Teams и Groups, но затем [Отключить добавление гостевых пользователей для выбранных команд, удовлетворяющих одному или нескольким условиям, таким как классификация данных, равно конфиденциальной](https://docs.microsoft.com/microsoft-365/compliance/sensitivity-labels-teams-groups-sites). У SharePoint и OneDrive есть собственные параметры гостевой доступа, которые не основываются на группах Microsoft 365.

Инструкции по настройке для гостевого доступа можно найти [в разделе совместная работа в группе "гости](https://docs.microsoft.com/microsoft-365/solutions/collaborate-as-team)".

> [!NOTE]
> На гостей распространяются ограничения служб, описанные в статьях [Описание служб Microsoft 365 и Office 365](https://go.microsoft.com/fwlink/p/?linkid=282347) и [Ограничения службы совместной работы Azure AD B2B](https://docs.microsoft.com/azure/active-directory/external-identities/current-limitations).

## <a name="control-guest-access-in-azure-active-directory"></a>Управление гостевым доступом в Azure Active Directory

Используйте Azure AD, чтобы определить, можно ли приглашать внешних участников в свой клиент в качестве гостей, а также способы приглашения. Дополнительные сведения о гостевом доступе Azure B2B см. в статье [Что представляет собой гостевой доступ в службе совместной работы Azure Active Directory B2B?](https://docs.microsoft.com/azure/active-directory/b2b/what-is-b2b) Сведения о ролях Azure AD см. в статье [Предоставление пользователям разрешений от партнерских организаций в клиенте Azure Active Directory](https://docs.microsoft.com/azure/active-directory/b2b/add-guest-to-role).

Настройки приглашений применяются на уровне Организации и управляют гостевой взаимодействию на уровне каталогов и приложений. Эти параметры можно настроить во [внешних параметрах совместной работы](https://aad.portal.azure.com/#blade/Microsoft_AAD_IAM/CompanyRelationshipsMenuBlade/Settings).

В Azure AD есть следующие параметры для настройки внешних пользователей:

- [Ограничения доступа пользователей гостя](https://docs.microsoft.com/azure/active-directory/users-groups-roles/users-restrict-guest-permissions)

- **Администраторы и пользователи с ролью приглашающего гостей могут приглашать других пользователей**. **Да** означает, что администраторы и пользователи с ролью приглашающего гостей смогут приглашать гостей в клиент. **Нет** означает, что администраторы и пользователи не могут приглашать гостей в клиент.
- **Участники могут приглашать других пользователей**. Чтобы разрешить пользователям, не являющимся администраторами каталога, приглашать гостей, присвойте этой политике значение **Да** (рекомендуется). Если вы хотите, чтобы только администраторы могли добавлять гостей, можно присвоить этой политике значение **Нет**. Имейте в виду, что присвоение значения **Нет** ограничит гостевое взаимодействие для владельцев команд, не являющихся администраторами. Они смогут добавлять только тех гостей Teams, которые уже добавлены администратором в AAD.
- **Гости могут приглашать других пользователей**. **Да** означает, что гости в вашем каталоге смогут приглашать других гостей для совместной работы с ресурсами, защищенными службой Azure AD, например сайтами SharePoint или ресурсами Azure. **Нет** означает, что гости не могут приглашать других гостей для совместной работы с вашей организацией. Даже если выбрано значение **"Да"** , гость не может пригласить других гостей в Teams.
 
Дополнительные сведения об управлении тем, кто может приглашать гостей, можно найти в разделе [Включение внешней совместной работы в сотрудничестве и управление пользователями, которые могут приглашать гостей](https://docs.microsoft.com/azure/active-directory/b2b/delegate-invitations).

> [!NOTE]
> Вы также можете управлять тем, какие домены могут быть приглашены в ваш клиент в качестве гостей. Просмотр [приглашений и запрещенных для пользователей B2B конкретных организаций](https://docs.microsoft.com/azure/active-directory/external-identities/allow-deny-list).

Не требуется вручную добавлять гостевую учетную запись в Azure AD B2B, так как она будет добавлена в каталог автоматически при добавлении гостя в Teams.

### <a name="licensing-for-guest-access"></a>Лицензии для гостевого доступа

В рамках лицензирования гостевой доступа используются цены внешних удостоверений Azure AD и они основываются на ежемесячных активных гостям. Дополнительные сведения см. в разделе [модель выставления счетов для внешних удостоверений Azure AD](https://docs.microsoft.com/azure/active-directory/external-identities/external-identities-pricing) .

> [!NOTE]
> Пользователей из вашей организации, у которых есть только отдельные планы подписки на Office 365, например на Exchange Online (план 2), нельзя пригласить в качестве гостей в организацию, так как Teams считает, что они относятся к одной организации. Чтобы использовать Teams для этих пользователей, им должна быть назначена подписка на Microsoft 365 бизнес стандарт, Office 365 корпоративный или Office 365 для образования.
