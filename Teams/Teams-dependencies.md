# <a name="authorize-guest-access-in-microsoft-teams"></a>Авторизация гостевого доступа в Microsoft Teams

Чтобы выполнить требования организации, можно управлять функциями и возможностями гостевого доступа Teams с помощью четырех разных уровней авторизации. К вашей организации Microsoft 365 применяются все уровни авторизации. Каждый уровень авторизации управляет взаимодействием с гостями, как показано ниже:

- **Azure Active Directory**: Гостевой доступ в Teams опирается на платформу Azure AD B2B. Этот уровень авторизации управляет взаимодействием с гостями на уровне каталога, клиента и приложения.
- **Teams**: Управляет взаимодействием с гостями только в Teams.
- **Группы Microsoft 365**: Управляет взаимодействием с гостями в группах Microsoft 365 и Teams.
- **SharePoint и OneDrive**: Управляет взаимодействием с гостями в SharePoint, OneDrive, группах Microsoft 365 и Teams.

Все эти уровни авторизации дают вам гибкость в настройке гостевого доступа для организации. Например, если вы хотите запретить гостей в Teams, но разрешить их в своей организации, просто отключите гостевой доступ в Teams. Другой пример. Вы можете включить гостевой доступ на уровнях Azure AD, Teams и групп, но затем [отключить добавление гостей в выбранных группах, соответствующих одному критерию или нескольким, например, если данные классифицируются как конфиденциальная информация](/microsoft-365/compliance/sensitivity-labels-teams-groups-sites). SharePoint и OneDrive имеют собственные параметры гостевого доступа, которые не связаны с группами Microsoft 365.

Инструкции по настройке сквозного гостевого доступа см. в разделе[Совместная работа с гостями в команде](/microsoft-365/solutions/collaborate-as-team).

> [!NOTE]
> На гостей распространяются ограничения служб, описанные в статьях [Описание служб Microsoft 365 и Office 365](/office365/servicedescriptions/office-365-service-descriptions-technet-library) и [Ограничения службы совместной работы Azure AD B2B](/azure/active-directory/external-identities/current-limitations).

## <a name="control-guest-access-in-azure-active-directory"></a>Управление гостевым доступом в Azure Active Directory

Используйте Azure AD, чтобы определить, можно ли приглашать внешних участников в свой клиент в качестве гостей, а также способы приглашения. Дополнительные сведения о гостевом доступе Azure B2B см. в статье [Что представляет собой гостевой доступ в службе совместной работы Azure Active Directory B2B?](/azure/active-directory/b2b/what-is-b2b) Сведения о ролях Azure AD см. в статье [Предоставление пользователям разрешений от партнерских организаций в клиенте Azure Active Directory](/azure/active-directory/b2b/add-guest-to-role).

Параметры приглашений применяются на уровне организации и управляют гостевым взаимодействием на уровне каталога, организации и приложения. Настроить эти параметры можно в разделе [Настройки внешнего взаимодействия](https://aad.portal.azure.com/#blade/Microsoft_AAD_IAM/CompanyRelationshipsMenuBlade/Settings).

В Azure AD есть следующие параметры для настройки внешних пользователей:

- [Ограничения доступа гостевых пользователей](/azure/active-directory/users-groups-roles/users-restrict-guest-permissions)

- **Администраторы и пользователи с ролью приглашающего гостей могут приглашать других пользователей**. **Да** означает, что администраторы и пользователи с ролью приглашающего гостей смогут приглашать гостей в клиент. **Нет** означает, что администраторы и пользователи не могут приглашать гостей в клиент.
- **Участники могут приглашать других пользователей**. Чтобы разрешить пользователям, не являющимся администраторами каталога, приглашать гостей, присвойте этой политике значение **Да** (рекомендуется). Если вы хотите, чтобы только администраторы могли добавлять гостей, можно присвоить этой политике значение **Нет**. Имейте в виду, что присвоение значения **Нет** ограничит гостевое взаимодействие для владельцев команд, не являющихся администраторами. Они смогут добавлять только тех гостей Teams, которые уже добавлены администратором в AAD.
- **Гости могут приглашать других пользователей**. **Да** означает, что гости в вашем каталоге смогут приглашать других гостей для совместной работы с ресурсами, защищенными службой Azure AD, например сайтами SharePoint или ресурсами Azure. **Нет** означает, что гости не могут приглашать других гостей для совместной работы с вашей организацией. Даже если установлено значение **Да**, гость не может приглашать других гостей в Teams.
 
Дополнительные сведения об управлении приглашениями гостей см. в разделе [Включение внешней совместной работы B2B и управление ролями приглашающих](/azure/active-directory/b2b/delegate-invitations).

> [!NOTE]
> Вы также можете управлять тем, какие домены могут быть приглашены в ваш клиент в качестве гостей. См. раздел [Разрешение или блокирование приглашений B2B-пользователям от определенных организаций.](/azure/active-directory/external-identities/allow-deny-list).

Не требуется вручную добавлять гостевую учетную запись в Azure AD B2B, так как она будет добавлена в каталог автоматически при добавлении гостя в Teams.

### <a name="licensing-for-guest-access"></a>Лицензии для гостевого доступа

При лицензировании гостевого доступа используются цены Azure Active Directory для внешних удостоверений и количество активных гостей за месяц. Дополнительные сведения см. в разделе [Модель выставления счетов для внешних удостоверений Azure AD](/azure/active-directory/external-identities/external-identities-pricing).

> [!NOTE]
> Пользователей из вашей организации, у которых есть только отдельные планы подписки на Office 365, например на Exchange Online (план 2), нельзя пригласить в качестве гостей в организацию, так как Teams считает, что они относятся к одной организации. Чтобы использовать Teams для этих пользователей, им должна быть назначена подписка на Microsoft 365 бизнес стандарт, Office 365 корпоративный или Office 365 для образования.
