# <a name="microsoft-365-app-compliance-program-for-security-compliance-and-privacy-of-third-party-apps"></a>Программа соответствия требованиям для обеспечения безопасности приложений и конфиденциальности приложений Microsoft 365 и сторонних приложений

В рамках программы соответствия требованиям Майкрософт приложение проверяется на соответствие мерам контроля, принятым в ведущих отраслевых стандартных платформах, а для защиты данных клиентов применяются надежные методы обеспечения безопасности и соответствия требованиям. Программа состоит из двух этапов:

* Аттестация издателя
* Сертификация Microsoft 365

## <a name="publisher-attestation"></a>Аттестация издателя

Аттестация издателя — это следующий уровень программы соответствия приложений. Приложения, прошедшие аттестационную проверку издателем, дают администраторам больше уверенности в мерах безопасности и соответствия требованиям, а также экономят время, необходимое для проверки приложения. Аттестация будет отражать безопасность приложения, методы обработки данных и соответствия требованиям в отношении более 80 факторов риска, выявленных с помощью[Microsoft Cloud App Security](https://www.microsoft.com/microsoft-365/enterprise-mobility-security/cloud-app-security).

Разработчикам приложений рекомендуется выполнять самостоятельную оценку, включающую часто задаваемые клиентами и ИТ-администраторами вопросы, для проверки безопасности и соответствия требованиям приложений. Затем корпорация Майкрософт публикует эту информацию для упрощения и ускорения оценки. Подробнее см. [Справочник по сертификации](/microsoft-365-app-certification/docs/enterprise-app-attestation-guide).

Администраторы могут быстро проверить опубликованные сертифицированные приложения тремя различными способами.

* В Интернете при сборе дополнительных сведений о приложении см. сведения о конкретном приложении по его ссылке на странице [Безопасность приложений Microsoft Teams и их соответствие требованиям](/microsoft-365-app-certification/teams/teams-apps). Кроме того, можно щелкнуть ссылку "Аттестация издателем" в центре администрирования Teams.

  :::image type="content" source="media/attested-app-tac3.png" alt-text="В центре администрирования Teams щелкните ссылку аттестации издателя, чтобы просмотреть сведения об аттестации приложения.":::

* В центре администрирования Teams при проверке сведений о приложении **на странице "Управление приложением"** вы увидите значок аттестации издателя в приложении-баннере страницы сведений о приложении.

  :::image type="content" source="media/attested-app-tac1.png" alt-text="В центре администрирования Teams значок аттестации издателя отображается во всех аттестованных приложениях.":::

* В центре администрирования Teams при предоставлении разрешений приложению синяя галочка перед именем приложения указывает, что это либо приложение, сертифицированное издателем, либо приложение, сертифицированное Microsoft 365.

   :::image type="content" source="media/attested-app-tac2.png" alt-text="В центре администрирования Teams в диалоговом окне предоставления разрешений администраторы могут проверить наличие синей галочки, чтобы убедиться, что приложение сертифицировано издателем.":::

На странице сведений об аттестации и сертификации приложений вы можете найти следующее.

:::image type="content" source="media/attested-app-doc-details.png" alt-text="Темы, по которым предоставляется подробная информация для сертифицированных приложений.":::

## <a name="microsoft-365-certification"></a>Сертификация Microsoft 365

Для сертификации приложений квалифицированный аналитик всесторонней оценки рассматривает и утверждает приложения, фокусируясь на их безопасности и соответствии платформам, процессам и процедурам.  Приложение проверяется на соответствие ряду требований безопасности, основанных на ведущих отраслевых стандартах.

Сертификат демонстрирует наличие надежных методов обеспечения безопасности и соответствия требованиям для защиты данных клиентов. Дополнительные сведения о том, почему сертификация Microsoft 365 полезна для администраторов и конечных пользователей, можно найти в обзоре [программы соответствия требованиям для приложений Microsoft 365](/microsoft-365-app-certification/docs/enterprise-app-certification-guide).

Администраторы могут быстро проверить приложения, сертифицированные для Microsoft 365, следующими способами.

* При сборе дополнительных сведений о приложении в Интернете найдите значок щита в списке приложений в документации по безопасности и обеспечению соответствия требованиям для приложений Microsoft Teams или тот же значок на странице документации для конкретного приложения.

  :::image type="content" source="media/attested-app-doc-details.png" alt-text="Просмотрите сведения о сертификации Microsoft 365 в подробной справочной статье о безопасности и соответствии требованиям для приложения.":::

* При проверке приложения в Центре администрирования Teams сортируйте список приложений с помощью столбца "Сертификация". Найдите значок и при желании щелкните ссылку, чтобы получить доступ к странице конкретного приложения, упомянутой выше.

  :::image type="content" source="media/m365cert-apps-list1.png" alt-text="Просмотрите состояние сертификации приложения Microsoft 365 в Центре администрирования Teams." lightbox="media/m365cert-apps-list2.png":::

* При просмотре сведений о приложении найдите значок сертификации Microsoft 365 на баннере приложения.

  :::image type="content" source="media/m365cert-app-details-banner.png" alt-text="Просмотрите сведения о сертификации Microsoft 365 на баннере приложения при управлении определенным приложением в Центре администрирования Teams":::

* В центре администрирования Teams при предоставлении разрешений приложению синяя галочка перед именем приложения указывает, что это либо приложение, сертифицированное издателем, либо приложение, сертифицированное Microsoft 365.

   :::image type="content" source="media/attested-app-tac2.png" alt-text="В центре администрирования Teams в диалоговом окне для предоставления разрешений администраторы могут установить синюю галочку, чтобы убедиться, что приложение сертифицировано Microsoft 365.":::

## <a name="know-details-of-third-party-apps"></a>Сведения о сторонних приложениях

Как администратор вы можете просматривать сведения, связанные с безопасностью, соответствием требованиям, конфиденциальностью и другие сведения о любых сторонних приложениях.

### <a name="view-security-compliance-and-privacy-information-in-microsoft-documentation"></a>Просмотр сведений о безопасности, соответствии требованиям и конфиденциальности в документации Майкрософт

Сведения о безопасности, конфиденциальности, соответствии требованиям и другие сведения о сертифицированном или аттестованном приложении перечислены в статьях справки для конкретного приложения по ссылке из раздела [Безопасность и соответствие требованиям для приложений Teams](/microsoft-365-app-certification/teams/teams-apps).

:::image type="content" source="media/attested-app-doc-details.png" alt-text="Темы, о которых для проходящих программу соответствия требованиям Майкрософт приложений представлены подробные сведения.":::

### <a name="view-the-granted-graph-permissions-in-azure-portal"></a>Просмотр предоставленных разрешений Graph на портале Azure

Администраторы могут предоставить приложению разрешение от имени всех пользователей организации. Это помогает избежать запроса разрешений по отдельности для каждого пользователя. Разрешения, предоставленные администратором, на [портале Azure](https://aad.portal.azure.com/) называются делегированными разрешениями.

Прежде чем предоставлять приложению какие-либо разрешения, просмотрите список запрашиваемых разрешений в разделе [Управление приложениями](https://admin.teams.microsoft.com/policies/manage-apps) в Центре администрирования Teams.

:::image type="content" source="media/attested-app-tac2.png" alt-text="В Центре администрирования Teams в диалоговом окне для предоставления разрешений администраторы могут проверить разрешения, запрашиваемые приложением.":::

После предоставления приложению разрешений для всей организации администраторы могут просмотреть разрешения Graph на портале Azure.

:::image type="content" source="media/tac-perms-in-aad-after-granting1.png" alt-text="Администраторы могут просматривать все разрешения приложения, предоставленные пользователями и администраторами на портале Azure Portal." lightbox="media/tac-perms-in-aad-after-granting2.png":::

### <a name="view-privacy-policy-and-terms-of-use-of-an-app"></a>Просмотр политики конфиденциальности и условий использования приложения

В Центре администрирования Teams каждая страница с подробными сведениями о приложении содержит ссылку на заявление о конфиденциальности и условия использования, добавленные издателем приложения.

:::image type="content" source="media/tac-app-tou-privacy-info1.png" alt-text="С помощью Центра администрирования Teams администраторы могут получить доступ к ссылке на политику конфиденциальности и условия использования для каждого приложения." lightbox="media/tac-app-tou-privacy-info2.png":::
