# <a name="supervised-chats-for-non-educational-tenants"></a>Защищенные чаты для клиентов, не относящихся к образованию

Защищенный чат позволяет назначенным руководителям инициировать чаты с любым пользователем в своей организации и блокирует запуск новых чатов пользователям с ограниченным доступом, если отсутствует соответствующий руководитель. Если включен контроль чата, руководители не могут покидать чаты, а другие участники не могут удалять их, гарантируя, что чаты с участием ограниченных пользователей должным образом контролируются.

Эти ограничения применяются только к новым приватным чатам, созданным после полного включения защищенного чата. Они не применяются к существующим частным чатам, чатам собраний или каналам.

Защищенный чат предназначен для образовательных учреждений, но он также доступен для клиентов, не являющихся образовательными.

> [!NOTE]
> Защищенный чат защищает новые чаты, созданные после применения этой функции. Он не защищает существующие чаты.

## <a name="enable-supervised-chat"></a>Включение защищенного чата

> [!NOTE]
> Убедитесь, что вы настроили роли разрешений чата и политики разрешений чата на основе ролей, прежде чем включать чат для вашего учреждения, чтобы избежать нежелательного ограниченного доступа пользователей к неконтролируемым чатам.

**Определите роли разрешений чата для каждого пользователя в вашей среде**:

Чтобы защищенный чат работал должным образом, каждому пользователю в вашей среде необходимо назначить правильную роль разрешений чата. Существует три роли, которые пользователь может назначить:

- Полные разрешения. Эта роль должна быть назначена руководителям чата в вашей среде. Они могут начать чаты с любым пользователем в вашей среде. Ожидается, что пользователи с полными разрешениями будут контролировать чаты, в которых они участвуют. Они не могут покинуть или быть удалены из чатов, которые они запускают, или чатов, которые они контролируют в федеративных клиентах.

- Ограниченные разрешения. Эта роль идеально подходит для сотрудников, которые должны иметь контролируемый доступ только для ограниченных пользователей. Они могут запускать чаты с любыми полными или ограниченными пользователями, но не могут запускать чаты с ограниченными пользователями. Если пользователь с полными разрешениями начинает чат с ограниченным пользователем, в беседу могут быть введены ограниченные пользователи. Этот доступ происходит из-за того, что пользователь с полными разрешениями присутствует для контроля совместной работы между ограниченными и ограниченными пользователями.

- Ограниченные разрешения. Эта роль идеально подходит для пользователей, которым требуется контроль. Они могут начинать чаты только с пользователями с полными разрешениями. Они могут участвовать в любой беседе, в которую их приглашает пользователь с полными разрешениями. В федеративных чатах ограниченные пользователи могут быть добавлены в чаты только пользователем с полными разрешениями, который входит в клиент ограниченного пользователя.

Чтобы задать роль разрешений для пользователей в чате, используйте политику роли **разрешения чата** , найденную в параметрах политики обмена сообщениями на портале администрирования Teams. PowerShell можно использовать для определения ролей с помощью политики ChatPermissionRole со значениями Full, Limited или Restricted. Эта политика находится в разделе [CsTeamsMessagingPolicy](/powershell/module/skype/set-csteamsmessagingpolicy).

Роли не могут быть назначены гостям в клиенте. Гостям назначается ограниченная роль.

## <a name="allow-supervised-chat"></a>Разрешить защищенный чат

Защищенный чат отключен по умолчанию для вашего клиента. После настройки ролей разрешений чата для пользователей можно включить защищенный чат в клиенте, перейдя в **Параметры** **Teams** \> и задав для политики **разрешений чата на основе ролей** значение **Включено**. Вы также можете использовать PowerShell для включения защищенного чата, задав для параметра AllowRoleBasedChatPermissions значение True. Этот командлет находится в разделе [CsTeamsClientConfiguration](/powershell/module/skype/set-csteamsclientconfiguration).

Защищенный чат должен быть включен для всех пользователей в клиенте и не может быть включен только для части пользователей.

**Включение чата**:

Включите чат для всех пользователей с помощью существующей политики чата, доступной в Центре администрирования Teams.

**Ведение защищенных чатов**:

После первоначального включения защищенного чата необходимо выполнить несколько действий, чтобы чаты в вашей среде оставались под контролем:

- Назначьте соответствующие роли всем новым пользователям, которые присоединяются к вашему клиенту. По умолчанию пользователям будет назначена ограниченная роль.

- Если пользователь с полными разрешениями покидает клиент или удаляется из него, чаты, которые он контролировал, будут оставлены без присмотра. Перед удалением исходного пользователя убедитесь, что в эти беседы добавлен другой пользователь с полными разрешениями, чтобы чат оставался защищенным. После удаления исходного руководителя новые участники не могут быть добавлены в беседу, но текущие участники могут продолжать общаться.
