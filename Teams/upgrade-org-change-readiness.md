## <a name="assessing-teamwork-scenarios"></a>Оценка сценариев командной работы

Группами Майкрософт расширяет возможности Скайп для бизнеса, предлагает комплексного решения обмена данными и совместной работы. При разработке плана для обновления до группами пользователей Скайп для бизнеса, рассмотрите возможность использования возможности совместной работы командную excite пользователей и упростить их применение Скайп для бизнеса групп.

По сути командную — как пользователи работают вместе для выполнения таких задач. Это о выравнивание нужных людей с соответствующих инструментальных средств, чтобы их работу. Команды объединяет людей, бесед, файлы и проекты в одной рабочей области, создание значение true, сервер-концентратор для работы в вашей организации.

Общие сведения о как работы пользователей и как они могут совместно может помочь пользователям визуализации, как можно использовать команды, упрощение shift вне Скайп для бизнеса. Существует два способа основные сценарии работу в команде, учитывайте следующее: ориентированный на людей и ориентированный на проект.

- **Ориентированный на проект командную** центры по определенному проекту или по проекту (например, планирование событие запуска продукта). Из области действия планирования для управления бюджета для маркетинговых кампаний ориентированный на проект командную помогает выровнять рабочих задач проекта, сохранив все уведомляться.
- **Ориентированный на людей командную** способном людей с аналогичными обязанности или атрибуты, может работать по группам (например, чемпионатов peer) или из группы (например, для отдела продаж, мобильных сотрудников). Из разговор с коллегой, чтобы получить ответ на вопрос на участие в собрании группы, ориентированный на людей действия включают общий доступ к информации и построение взаимопонимания.

Для получения максимально эффективному использованию групп построение стратегии работу в команде, которая включает в себя оба сценария, как помочь пользователям понять, как их работы команды.

### <a name="tips-for-identifying-a-good-fit-for-your-teamwork-strategy"></a>Советы для идентификации «подходит» для вашей стратегии сотрудничество

Это может быть легко увлекайтесь формирование командную стратегии для каждого пользователя, для любой группы и каждого проекта. Не каждый проект или рабочая группа необходимо полностью определенные командную стратегии. Ниже приведены некоторые рекомендации по работе с Приступая к работе.

- Рассмотрите возможность между проектами, только начинает работать, или предстоящие.
  - Планирование мероприятий — координация из нескольких групп (бюджета, логистические операции, презентаций и т. п.)
  - Пилотное новый продукт - обмениваться информацией, сбор отзывов в управляемой среде

- Определение основных рабочих групп и пользователей пользователя, подключение различные способы их взаимодействия друг с другом в группы

  - Команды/регионы сбыта – мобильных сотрудников, простой доступ к ресурсам, параметр Включить быстрый вокруг ответы
  - Ч – стандартизованной процессов на уровне всех подразделений, последовательного подхода найма
  - Работники Firstline – подключение к одноранговых узлов и управление ими, доступ к процедурам, NEO

- Помните о следующих:

  - Запустите небольшой. Сохраните больше, более сложных проектов для после проверки таких модели работу в команде.
  - Специализация для новых проектов. Не изменяйте существующего проекта, который хорошо стадии реализации, как это может нарушить рабочего процесса.
  - Рассмотрите возможность синхронизации. Развертывание новой стратегии командную для продаж в конце финансовый год может не являются идеальным решением.
  - Выполните итерацию. Нет необходимости изобретать велосипед. Найдя командную структуру, которая работает, повторите его вместе с другими рабочих групп и проектов. Определить шаблон группы и назначьте рекомендации, помогающие пользователям быстро их использования в группы и обеспечивают согласованность.

Используйте сведения, собранные выше для оповещения [пользователя план](https://aka.ms/UserReadinessPlan).
