## <a name="allow-meet-now-in-channels"></a>Разрешить возможность встретиться сейчас в каналах

Это политика "на пользователя", которая применяется до начала собрания. Этот параметр управляет возможностью пользователя начать собрание в канале Teams. Если этот режим включить, пользователи  смогут нажать кнопку "Собрание", чтобы начать собрание или запланировать собрание на канале. По умолчанию используется значение True.

[![Снимок экрана: значок "Выполнить сейчас" под сообщением ](media/meeting-policies-meet-now.png)](media/meeting-policies-meet-now.png#lightbox)

## <a name="allow-the-outlook-add-in"></a>Разрешение надстройки Outlook

Это политика "на пользователя", применяемая до начала собрания. Этот параметр управляет тем, можно ли запланть собрания Teams из Outlook (Windows, Mac, Интернета и мобильной связи).

![Снимок экрана: возможность запланировать новое собрание](media/meeting-policies-outlook-add-in.png)

Если отключить эту функцию, пользователи не смогут планировать собрания Teams при создании нового собрания в Outlook. Например, в Outlook для Windows параметр "Новое собрание **Teams"** не будет показываться на ленте.

## <a name="allow-channel-meeting-scheduling"></a>Разрешение планирования собраний в канале

Используйте существующую политику AllowChannelMeetingScheduling для управления типами событий, которые могут быть созданы в календарях каналов группы. Это политика "на пользователя", которая применяется до начала собрания. Этот параметр управляет возможностью пользователей запланировать собрание в канале Teams. По умолчанию этот параметр включен. 

Если эта политика отключена, пользователи не смогут создавать собрания канала. Однако существующие собрания канала может редактировать организатор события.

Запланировать собрание будет отключено.

![Снимок экрана: параметр "Запланировать собрание" в Teams](media/schedule-meeting-option.png)

Выбор канала отключен.

[![Снимок экрана: параметр календаря для выбора канала, в котором вы хотите запланировать собрание. ](media/meeting-policies-select-a-channel-to-meet-in.png)](media/meeting-policies-select-a-channel-to-meet-in.png#lightbox)

На странице записей канала будут отключены следующие пунктов:

- **Кнопка "Запланировать** собрание" в окне составить ответ на канал.
  ![Снимок экрана: параметр календаря для выбора канала, в котором вы хотите запланировать собрание.](media/schedule-meeting-disabled-in-chat2.png)
  
- **Кнопка "Запланировать** собрание" в заглавной области канала.
  ![Снимок экрана: параметр календаря для выбора канала, на котором вы хотите запланировать собрание.](media/schedule-now-in-header.png)

В календаре канала:

- **Кнопка добавления нового** события в заглавном календаре канала будет отключена.
  ![Снимок экрана: параметр календаря для выбора канала, который позволит вам запланировать собрание.](media/add-new-event-disabled.png)

- Пользователи не смогут перетаскивать и выбирать блок времени в календаре канала, чтобы создать собрание канала.

- Пользователи не могут использовать сочетания клавиш для создания собрания в календаре канала.

В Центре администрирования:

Приложение "Календарь канала" будет от которое вы сможете демонстрировать в разделе **приложений Майкрософт** на странице политик разрешений приложений.

![Снимок экрана: политика разрешений для приложений в Центре администрирования Teams.](media/manage-microsoft-apps-policy.png)

## <a name="allow-scheduling-private-meetings"></a>Разрешить планирование частных собраний

Это политика "на пользователя", которая применяется до начала собрания. Этот параметр управляет возможностью пользователей планировать частные собрания в Teams. Собрание является частным, если оно не опубликовано в канале группы.

Обратите внимание, что если вы отключите "Разрешить планирование  частных собраний"  и "Разрешить планирование собраний канала", параметры "Добавить необходимых участников" и "Добавить канал" будут отключены для пользователей в Teams.   По умолчанию этот параметр включен.

## <a name="allow-meet-now-in-private-meetings"></a>Разрешить собрание на частных собраниях

Это политика "на пользователя", которая применяется до начала собрания. Этот параметр управляет возможностью пользователя начинать частные собрания.  По умолчанию этот параметр включен.

## <a name="designated-presenter-role-mode"></a>Назначенный режим роли в режиме роли

Это политика для каждого пользователя. Этот параметр позволяет изменить значение по умолчанию для параметра **"Кто может представить"?** в параметрах собрания **в** клиенте Teams. Этот параметр политики влияет на все собрания, включая собрания "Собрание сейчас".

Параметр **"Кто может представлять"?** Позволяет организаторам собраний выбирать, кто может быть на собрании в качестве presenters. Дополнительные параметры см. в [настройках](https://support.microsoft.com/article/change-participant-settings-for-a-teams-meeting-53261366-dbd5-45f9-aae9-a70e6354f88e) участников для собрания Teams и ролей [в собрании Teams.](https://support.microsoft.com/article/roles-in-a-teams-meeting-c16fa7d0-1666-4dde-8686-0a0bfe16e019)

В настоящее время для настройки этого параметра политики можно использовать только PowerShell. Вы можете изменить существующую политику собраний Teams с помощью командлета [Set-CsTeamsMeetingPolicy.](https://docs.microsoft.com/powershell/module/skype/set-csteamsmeetingpolicy) Или создайте политику собраний Teams с помощью командлета [New-CsTeamsMeetingPolicy](https://docs.microsoft.com/powershell/module/skype/new-csteamsmeetingpolicy) и назначьте ее пользователям.

Чтобы указать значение по умолчанию параметра **"Кто может присутствовать?"** в Teams, задате для параметра **DesignatedPresenterRoleMode** одно из следующих значений:

- **EveryoneUserOverride:** все участники собрания могут быть в том же октете. Это значение по умолчанию. Этот параметр соответствует параметру **"Все"** в Teams.
- **EveryoneInCompanyUserOverride:** пользователи, которые пройдет проверку подлинности в организации, включая гостевых пользователей, могут быть в качестве участников-участников. Этот параметр соответствует параметрам **"Люди" в моей организации** в Teams.
- **OrganizerOnlyUserOverride:** только организатор собрания может быть в качестве участника, а все участники собрания назначаются участниками. Этот параметр соответствует параметру **"Только я"** в Teams.

Имейте в виду, что после установки значения по умолчанию организаторы собраний могут изменить этот параметр в Teams и выбрать, кто может выведет свои мысли на собраниях.

## <a name="meeting-attendance-report"></a>Отчет об участие в собрании

Это политика для каждого пользователя. Этот параметр управляет тем, могут ли организаторы собраний загружать отчет об участие в [собрании.](teams-analytics-and-reports/meeting-attendance-report.md)

В настоящее время для настройки этого параметра политики можно использовать только PowerShell. Вы можете изменить существующую политику собраний Teams с помощью командлета [Set-CsTeamsMeetingPolicy.](https://docs.microsoft.com/powershell/module/skype/set-csteamsmeetingpolicy) Или создайте политику собраний Teams с помощью командлета [New-CsTeamsMeetingPolicy](https://docs.microsoft.com/powershell/module/skype/new-csteamsmeetingpolicy) и назначьте ее пользователям.

Чтобы включить организатора собрания для загрузки отчета об посещение, установите для параметра **AllowEngagementReport** параметр **Enabled.** Если этот параметр включен, на области "Участники" отображается параметр для **скачивания** отчета.

Чтобы запретить организатору собрания загружать отчет, установите для параметра **отключение.** По умолчанию этот параметр отключен, и скачать отчет нельзя.

## <a name="meeting-provider-for-islands-mode"></a>Режим поставщика собраний для островов

Это политика для каждого пользователя. Этот параметр управляет тем, какая надстройка собрания Outlook используется для *пользователей, которые находятся в режиме "О-ва".* Укажите, могут ли пользователи в режиме островов использовать только надстройку "Собрание Teams" или обе надстройки "Собрание Teams" и "Собрание Skype для бизнеса" для планирования собраний в Outlook.

Эту политику можно применять только к пользователям, которые находятся в режиме островов и установили для параметра **AllowOutlookAddIn** значение **True** в политике собраний Teams.

В настоящее время для этого можно использовать только PowerShell. Вы можете изменить существующую политику собраний Teams с помощью командлета [Set-CsTeamsMeetingPolicy.](https://docs.microsoft.com/powershell/module/skype/set-csteamsmeetingpolicy) Или создайте политику собраний Teams с помощью командлета [New-CsTeamsMeetingPolicy](https://docs.microsoft.com/powershell/module/skype/new-csteamsmeetingpolicy) и назначьте ее пользователям.

Чтобы указать, какая надстройка собрания должна быть доступна пользователям, задайте параметр **PreferredMeetingProviderForIslandsMode** следующим образом:

- Задайте параметр **TeamsAndSfB,** чтобы включить в Outlook надстройки "Собрание Teams" и Skype для бизнеса. Это значение по умолчанию.
- Установите параметр в **Teams,** чтобы включить только надстройка "Собрание Teams" в Outlook. Этот параметр политики гарантирует, что все последующие собрания будут иметь ссылку на собрание Teams. Существующие ссылки на собрания Skype для бизнеса не переноситься в Teams. Этот параметр политики не влияет на функции присутствия, чата, звонков по ТСТС и любых других возможностей Skype для бизнеса. Это означает, что пользователи по-прежнему будут использовать Skype для бизнеса.

  Если вы установили параметр в **Teams,** а затем возвращались в **TeamsAndSfB,** обе надстройки собраний будут включены. Обратите внимание, что существующие ссылки на ссылки на собрания Teams не будут перенесены в Skype для бизнеса. Только для собраний Skype для бизнеса, запланированных после изменения, будет ссылка на собрание Skype для бизнеса.

## <a name="meeting-reactions"></a>Реакции на собрания

Параметр AllowMeetingReactions можно применять только с помощью PowerShell. В Центре администрирования Teams нет возможности отключить параметр AllowMeetingReactions.

Реакции на собрания по умолчанию отключены. Отключение реакций для пользователя не означает, что пользователь не может использовать реакции на собраниях, которые он планит. Организатор собрания по-прежнему может включить реакции на странице параметров собрания независимо от настройки по умолчанию.
