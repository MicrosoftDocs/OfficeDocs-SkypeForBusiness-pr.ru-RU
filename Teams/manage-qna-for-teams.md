# <a name="manage-qa-in-teams-meetings"></a>Управление Q&A в собраниях Teams

Q&A — это интегрированный интерфейс для собраний, который позволяет участникам официально задавать вопросы и получать ответы на них, а также участвовать в структурированных обсуждениях. Этот интерфейс рекомендуется использовать для собраний с более чем 10 участниками.

В этой статье описывается, как управлять политиками Q&A и на уровне пользователя, которые определяют, может ли организатор включить Teams Q&A в собраниях.

> [!NOTE]
> Эта функция в настоящее время находится в закрытой предварительной версии и не является общедоступной. Если вы хотите принять участие в программе закрытой предварительной версии, зарегистрируйтесь [здесь.](https://m365crmedu.powerappsportals.com/LMSSignup/)


## <a name="prerequisites"></a>Необходимые компоненты

- Убедитесь, что вы не блокируют доступ к [IP-адресам и URL-адресам Yammer.](/microsoft-365/enterprise/urls-and-ip-address-ranges)
- Чтобы разрешить пользователям в организации добавлять собрания Q&A в Teams, необходимо убедиться, что входы в службу Office 365 Yammer включены в Azure Active Directory. Выполните следующие действия, чтобы убедиться, что входы включены:
  - Перейдите в **центр администрирования** >  Azure AD **"Все службы** > **корпоративных** >  приложений **Office 365 Yammer** > **"**.
  - Для **параметра "Включено для входа пользователей"** **выберите "Да** ", если это необходимо.

## <a name="who-can-use-qa"></a>Кто может использовать Q&A

Q&A может использоваться следующими типами пользователей:

- Обычный пользователь — пользователь с учетными данными Microsoft 365 в клиенте.
- Федеративный пользователь — пользователь с учетными данными Microsoft 365 для другого клиента.
- Гостевой пользователь — все гости, добавленные в Microsoft Teams, SharePoint или Azure Active Directory.

> [!NOTE]
> Q&A в данный момент не поддерживает анонимных пользователей.

Когда администраторы включит Q&A, пользователи с ролью организатора могут включить Q&A при создании или обновлении собраний. С помощью параметров собраний Teams и Outlook организаторы также могут удалить Q&A из собраний, где он был добавлен ранее, чтобы запретить участникам использовать эту функцию.

## <a name="use-the-teams-admin-center-to-manage-qa"></a>Использование Центра администрирования Teams для управления Q&A

> [!NOTE]
> Управление Q&A в Центре администрирования недоступно в общедоступной предварительной версии. Используйте PowerShell для настройки Q&A для пользователей и управления ими.

У вашей организации могут быть требования к ограничению того, какие организаторы могут включить Q&A. Вы можете использовать Центр администрирования Teams для управления тем, какие организаторы могут включать Q&A на больших собраниях.
Выполните следующие действия, чтобы определить, какие организаторы могут использовать Q&A:

1. В Центре администрирования Teams перейдите к **политикам собраний** > [**собраний**](/meeting-policies-participants-and-guests)
2. Выберите существующую политику или создайте новую и присвойте ему имя, например "No Q&A for these organizers" (Нет Q&A для этих организаторов).
3. Прокрутите до раздела **"Участники & гостей"**, выберите "Отключить" для параметра "Взаимодействие & **ответов"** и сохраните политику.
4. Назначьте политику определенным группам M365, конечным пользователям или подпискам, которые вы хотите запретить настройку Q&A

## <a name="use-powershell-to-manage-qa"></a>Использование PowerShell для управления Q&A

У вашей организации могут быть требования к ограничению того, какие организаторы могут включить Q&A. С помощью PowerShell можно управлять тем, какие организаторы могут включать Q&A на больших собраниях.

Пример команды PowerShell для включения Q&A:

```PowerShell
Set-CsTeamsMeetingPolicy -Identity Global -QnAEngagementMode Enabled
```

## <a name="delete-an-individual-message-from-qa-in-teams"></a>Удаление отдельного сообщения из Q&A в Teams

Чтобы удалить вопрос или ответ, опубликованный в Q&A, выполните следующие действия.

1. Войдите в Центр exchange Администратор с правами глобального администратора.
2. Перейдите **в почтовые** > **ящики получателей** и выполните поиск по имени пользователя, устроившего собрание.
3. Выберите организатор собрания и щелкните " **Управление делегированием почтовых ящиков"**. В разделе **"Чтение и управление** " выберите " **Изменить** > **добавить разрешения"**.
4. Добавьте себя в качестве делегата организатора собрания и нажмите кнопку "Сохранить".
5. Откройте Календарь Outlook в Outlook Web App (не на рабочем столе) и выберите "Добавить **календарь**" и **"Добавить из каталога"**.
6. Найдите организатора собрания и добавьте его календарь в **"Мои календари"**. Собрания для выбранного пользователя теперь будут отображаться в календаре.
7. В календаре найдите собрание, для которого хотите удалить содержимое, откройте запись собрания и выберите **"Чат с участниками"**. При выборе чата с участниками откроется чат собрания в Teams.
8. Перейдите к Q&a application in the Teams app bar.
9. Найдите любые вопросы или ответы, которые вы хотите удалить, и нажмите кнопку "Удалить".
10. Завершив удаление содержимого, вернитесь в Центр exchange Администратор и удалите себя в качестве делегата организатора собрания.

> [!NOTE]
> Если удалить вопрос перед удалением ответов, первым ответом на него станет вопрос.
>
> Чтобы избежать этого, выполните следующие действия по порядку:
>
> 1. Определите вопрос, который вы хотите удалить.
> 2. Удаление ответов на вопрос
> 3. Удаление вопроса

## <a name="available-languages-for-yammer-versus-teams"></a>Доступные языки для Yammer и Teams

Q&A будет по умолчанию использовать язык пользователя для Teams. Если языки, доступные для Teams и Yammer, отличаются, по умолчанию будут использоваться следующие языки:

- Ближайший основной язык — Yammer по умолчанию будет использовать ближайший первичный язык, если он доступен. Например, Yammer не предоставляет версию французского языка (fr-CA), поэтому вместо нее будет отображаться содержимое на французском языке (fr-FR).
- Неподдерживаемый язык— если язык не предоставлен Yammer, Yammer по умолчанию будет использовать английский (США).

## <a name="ediscovery"></a>Обнаружение электронных данных

Обнаружение электронных данных для Q&A будет работать так же, как обнаружение электронных данных для любого другого содержимого Yammer.

- Если вы используете Teams Q&A в приложении Teams клиента, это содержимое будет доступно в службе обнаружения электронных данных независимо от конфигурации или существования сети Yammer. Чтобы использовать обнаружение электронных данных для стандартного содержимого Yammer, сеть Yammer должна находиться в собственном [режиме](/yammer/configure-your-yammer-network/overview-native-mode).
- При обнаружении электронных данных можно определить, были ли сообщения созданы в Yammer или с помощью Q&A в Teams. В разделе "Метаданные файла" эти сведения можно найти в поле "Класс элемента".
- Если ваша организация использует Teams Q&A на основе Yammer, содержимое, созданное Q&A, считается содержимым Yammer и будет обнаруживаемым. Дополнительные сведения об обнаружении электронных данных в приложениях Microsoft 365 см. в статьях о решениях [eDiscovery в Microsoft 365.](/microsoft-365/compliance/ediscovery)

## <a name="data-storage"></a>Хранилище данных

Вопрос&сообщения, созданные в рамках собрания, хранятся в виде сообщений Yammer. Эти сообщения хранятся в Yammer и приняты для обнаружения электронных данных.
Файлы всегда хранятся в SharePoint, которая обрабатывает все политики и расположения для хранения данных.

В следующем руководстве объясняется, как хранятся данные обмена сообщениями:

- Вопрос&, что содержимое A доступно для поиска с помощью обнаружения электронных данных и Advanced eDiscovery на уровне пользователя.
- Данные сообщений (включая содержимое сообщений) по умолчанию будут храниться в Северная Америка или ЕС (в зависимости от расположения сети Yammer клиента).
- Для клиентов, у которых нет существующей сети Yammer, Q&сеть Yammer будет создана в США Yammer. Сообщения вашей организации для Q&A всегда будут храниться в США.
- Как и вкладки OneNote, при удалении Q&A из собрания данные собрания не удаляются. Удаление Q&A из собрания только удаляет доступ к данным из собрания. Если добавить вкладку Q&A, вы снова увидите все беседы о собрании.

## <a name="gdpr-for-qa-in-teams"></a>GDPR для Q&A в Teams

Когда вы завершаете запрос субъекта данных через центр Microsoft 365 Admin, он автоматически удаляет контактные данные пользователей из всего содержимого в Q&A.

## <a name="data-lifecycle-for-qa-in-teams"></a>Жизненный цикл данных для Q&A в Teams

Жизненный цикл данных, созданных Q&A в Teams, зависит от параметров хранения данных Yammer в Центре соответствия требованиям. Если вы жестко удалите всех пользователей в собрании, которые получили копию сообщений Q&A в сегменте подстроки через Azure Active Directory, Yammer удалит свою копию Q&содержимого для этого собрания в течение 30 дней после удаления пользователя.

## <a name="retention-and-deletion"></a>Хранение и удаление

Хранение содержимого следует политикам хранения, задаваемых для Yammer, независимо от того, настроены ли для Yammer и Teams разные политики.

> [!NOTE]
> Если сеть Yammer не находится в собственном режиме, созданные здесь политики будут применяться только к teams Q&A. В собственном режиме все пользователи Yammer находятся в Azure Active Directory (Azure AD), все группы являются группами Microsoft 365, а все файлы хранятся в SharePoint Online.
