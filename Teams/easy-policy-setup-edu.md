# <a name="use-the-teams-for-education-policy-wizard-to-easily-apply-policies-for-a-safe-learning-environment"></a>Использование мастера Teams для образования политики для простого применения политик для безопасной среды обучения

## <a name="overview"></a>Обзор

Мастер Microsoft Teams для образования упрощает управление политиками для учащихся и преподавателей. Используйте его для простого и быстрого применения наиболее важного набора политик, относящихся к созданию безопасного и продуктивного обучения.

Политики в Teams позволяют управлять поведением Teams в вашей среде и доступными пользователями функциями. Например, существуют политики звонков, политики собраний и политики обмена сообщениями, чтобы указать несколько имен, и каждую область политики можно настроить в соответствии с потребностями вашей организации.

Чтобы обеспечить безопасную и ориентированную среду обучения, важно задать политики для управления действиями учащихся в Teams. Например, политики можно использовать для управления тем, кто может использовать частный чат и частные звонки, кто может планировать собрания и какие типы контента можно совместно использовать. Вы также можете использовать политики для включения функций Teams, которые расширяют возможности обучения.

Политики должны быть скорректированы как для учащихся, так и для преподавателей, чтобы обеспечить безопасность обучения. Политики для учащихся должны быть более строгими, чтобы снизить риск получения недопустимых уровней доступа. Преподавателям и сотрудникам требуется отдельный набор политик, которые могут быть более разрешительными, чтобы обеспечить их успешное выполнение. Например, разрешите преподавателям планировать собрания и запретить учащимся делать это.

:::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="Снимок экрана мастера.":::

В этой статье описывается, как запустить мастер.

> [!IMPORTANT]
> Политики, применяемые мастером, удовлетворяют потребностям большинства Teams для образования клиентов. Мастер настраивает глобальное определение (по умолчанию для всей организации) основного набора политик с параметрами, которые мы рекомендуем использовать для безопасности учащихся, и применяет его к учащимся. Мастер также создает и назначает набор настраиваемых политик преподавателям и сотрудникам. Большинство Teams для образования клиентов не будут использовать другие методы назначения политики после запуска этого мастера. Используйте другие методы *назначения политики только* в том случае, если вы хотите вручную создавать политики для учащихся, преподавателей и сотрудников и управлять ими.

## <a name="teams-for-education-policy-wizard"></a>Teams для образования политики

<a name="polwiz_intro"> </a>

Мастер применяет набор основных определений политик для учащихся и отдельный набор основных определений политик для преподавателей и сотрудников с параметрами, подходящими для каждого из них. Вот что происходит при запуске мастера.

Мастер настраивает политики на основе типа учебного заведения (**основное, дополнительное** или **учебное заведение**). Вы выбираете тип учебного заведения, и мастер выполняет следующие действия:

- **Учащиеся**. Мастер настраивает определение глобальной политики (по умолчанию для всей организации) для каждой области политики, подходящей для мастера, с помощью новых параметров по умолчанию, которые подходят для обеспечения безопасности учащихся. Это гарантирует, что текущие учащиеся и все новые учащиеся получают наиболее строгий набор политик.
- **Преподаватели и сотрудники**. Мастер создает набор пользовательских определений политик для каждой области политики, охватываемой мастером, с параметрами, адаптированными для потребностей преподавателей и сотрудников. Затем он назначает определения политики группе преподавателей и сотрудников, которых вы выберете. Таким образом, преподаватели и сотрудники получают более строгий набор политик, чтобы обеспечить их успешное выполнение.

Мастер необходимо запустить только один раз. Новые учащиеся автоматически получают определения глобальной политики (по умолчанию для всей организации), применяемые мастером, и новым сотрудникам, добавляемым в выбранную группу, автоматически назначаются настраиваемые политики.

Кроме того, при каждом добавлении новой функции в Teams соответствующее значение политики по умолчанию, соответствующее EDU для этой функции, автоматически добавляется в глобальный (по умолчанию для всей организации) без вмешательства администратора. Это гарантирует, что для обеспечения безопасности и вовлечения учащихся будут перечислены правильные политики.

> [!NOTE]
> [Подробный список определений](#policies-applied-by-the-wizard) политик, применяемых мастером, см. в разделе "Политики, применяемые мастером".

Теперь приступим к работе!

## <a name="run-the-wizard"></a>Запуск мастера

<a name="polwiz_run"> </a>

Выполните следующие действия, чтобы запустить мастер.

1. Если вы не знакомы с Teams, мастер запускается автоматически. В противном случае мастер можно запустить в любое время с панели мониторинга. В левой области навигации Центра администрирования Microsoft Teams перейдите на домашнюю страницу **, а** затем  в разделе "Простая настройка политики" для плитки безопасной среды обучения выберите "**Быстрая настройка"**.

    :::image type="content" source="media/easy-policy-setup-quick-setup.png" alt-text="Снимок экрана: мастер на панели мониторинга.":::

2. Выберите тип учебного заведения (**основное, дополнительное** или **дополнительное**), а затем нажмите кнопку **"Далее"**.

    :::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="Снимок экрана: страница мастера для выбора типа учреждения.":::

3. Найдите и выберите группы, содержащие преподавателей и сотрудников, а затем нажмите кнопку **"Далее"**. Если у вас еще нет групп для преподавателей и сотрудников [,](/microsoft-365/admin/create-groups/create-groups) создайте группу, а затем повторно запустите мастер. <br/><br/>Можно выбрать до трех групп. Преподавателям и сотрудникам выбранных групп будет назначен набор настраиваемых [](#policies-applied-by-the-wizard) политик, адаптированных для их потребностей. Помните, что этот набор политик отделен от политик, применяемых к учащимся.

    :::image type="content" source="media/edu-policy-wizard-add-3-groups.png" alt-text="Снимок экрана: страница мастера для выбора групп преподавателей и сотрудников.":::

4. Проверьте выбранные параметры.

    :::image type="content" source="media/edu-policy-wizard-3-groups-review.png" alt-text="Снимок экрана: страница мастера для просмотра выбранных элементов.":::

5. Нажмите **кнопку "** Применить", чтобы применить изменения. Это может занять несколько минут.<br/><br/>Определения глобальной политики (по умолчанию для всей организации) немедленно применяются к учащимся. Для преподавателей и сотрудников может потребоваться несколько часов, чтобы настраиваемые политики назначались каждому участнику выбранных групп в зависимости от размера групп. Это происходит в фоновом режиме после успешного завершения этого шага.
6. Вы уже на пути, но еще не сделали этого! Необходимо учитывать еще несколько моментов. Затем ознакомьтесь с действиями, описанными в разделе "Что [делать после запуска](#what-to-do-after-running-the-wizard) мастера" этой статьи.

    :::image type="content" source="media/easy-policy-setup-on-way.png" alt-text="Снимок экрана: страница мастера для выполнения следующих действий.":::

## <a name="what-to-do-after-running-the-wizard"></a>Что делать после запуска мастера

<a name="polwiz_remove"> </a>

### <a name="step-1-remove-existing-policy-assignments-that-conflict-with-policies-applied-by-the-wizard"></a>Шаг 1. Удаление существующих назначений политик, конфликтующих с политиками, применяемыми мастером

> [!IMPORTANT]
> **Выполните этот шаг,** только если у вас есть политики, назначенные учащимся, преподавателям и сотрудникам перед запуском мастера. Если вы еще не знакомы с Teams и у вас нет политик, отличных от политик, созданных мастером, пропустите этот шаг и перейдите к шагу 2.

В Teams для заданной области политики политику можно применить к пользователю следующими способами:

- Прямое назначение пользователю
- Назначение группе, членом
- Если пользователю не назначена политика напрямую или он не является членом какой-либо группы, назначаемой политикой, пользователь автоматически получает глобальную политику (по умолчанию для всей организации).

Если для пользователя существует несколько назначений политики, Teams использует следующий порядок, чтобы определить, какое назначение политики вступает в силу. Дополнительные сведения см. в [разделе о том, какая политика имеет приоритет](policy-assignment-overview.md#which-policy-takes-precedence) или [правила приоритета для групп](assign-policies-users-and-groups.md#precedence-rules).

|Назначения политик пользователя|Политика, которая вступает в силу|
|---|---|
|Политика, назначенная группе: Нет <p> Политика, назначенная пользователю напрямую: нет|Глобальная (в пределах организации) политика по умолчанию|
|Политика, назначенная группе: Нет <p> Политика, назначенная пользователю напрямую: Да|Политика, назначенная пользователю напрямую|
|Политика, назначенная группе: Да <p> Политика, назначенная пользователю напрямую: Да|Политика, назначенная пользователю напрямую|
|Политика, назначенная группе: Да <p> Политика, назначенная пользователю напрямую: нет|Политика, назначенная группе <p> Если пользователь является членом нескольких групп и каждой группе назначается политика одной области политики, политика с наивысшим ранжированием назначений [групп](assign-policies-users-and-groups.md#group-assignment-ranking) вступает в силу.|

Из-за этого порядка политики, созданные мастером, не вступает в силу, если у пользователя есть прямые назначения или назначения групп. Это означает, что вам придется удалить существующие назначения политик у пользователя, чтобы политика, применяемая мастером, вступает в силу.

Для каждой [области политики, применяемой мастером](#policies-applied-by-the-wizard), сделайте следующее:

- Удалите все существующие прямые задания и групповые задания из учащихся, чтобы определение глобальной политики (по умолчанию для всей организации), применяемое мастером, вступает в силу.
- Удалите все конфликтующее прямое назначение для преподавателей и сотрудников, чтобы определение настраиваемой политики, созданное мастером, вступает в силу. Используйте приведенную выше таблицу, чтобы определить применимые сценарии. <p> Имейте в виду, что мастер назначает политики преподавателям и группе сотрудников, используя ранжирование назначения группы 1, что является самым высоким рейтингом.[](assign-policies-users-and-groups.md#group-assignment-ranking) Если вашим преподавателям и группе сотрудников назначена политика той же области политики, она перемещается в более низкий рейтинг и политика, назначенная мастером, вступает в силу.

[Узнайте больше](batch-group-policy-assignment-edu.md#remove-a-policy-that-was-directly-assigned-to-users) о том, как удалить политики, назначенные пользователям напрямую.

Например, вы назначите политику собраний непосредственно преподавателям, а учащиеся имеют глобальную (по умолчанию для всей организации) политику собраний. В этом сценарии удалите политику собраний, назначенную непосредственно преподавателям, чтобы определение настраиваемой политики для политики собраний, созданной мастером, вступает в силу. Вам не нужно ничего делать с существующей глобальной политикой собраний (по умолчанию для всей организации) для учащихся, так как с ней не конфликтуют другие политики собраний.

### <a name="step-3-check-message-center-for-policy-updates"></a>Шаг 3. Проверка центра сообщений на наличие обновлений политики

В настоящее время мастер применяет рекомендуемые политики при его запуске. Важно знать, что по мере того, как новые политики становятся доступными в Teams, мастер автоматически обновляет глобальные (по умолчанию для всей организации) параметры безопасности учащихся.

Но регулярно проверяйте Центр сообщений [(в](https://admin.microsoft.com/AdminPortal/Home?#/MessageCenter) Центр администрирования Microsoft 365), чтобы быть в курсе новых функций, их политик и параметров политики в Teams.

## <a name="make-changes-in-the-wizard"></a>Внесение изменений в мастер

<a name="polwiz_change"> </a>

Если после запуска мастера необходимо внести изменения, вы можете повторно запустить его и изменить выбранные параметры.

1. В левой области навигации Центра администрирования Microsoft Teams перейдите на домашнюю страницу **, а** затем  в разделе "Простая настройка политики" для плитки безопасной среды обучения выберите "Изменить **"**.
2. Перейдите на каждую страницу мастера, чтобы внести изменения. Вы можете изменить тип учебного заведения, группы преподавателей и сотрудников, которым вы хотите назначить политики, или и то, и другое.

В следующей таблице перечислены действия, которые происходят при внесении изменений в мастере.

|Тип изменения|Поведение политики|
|---|---|
|Изменение типа учебного заведения, преподавателей и групп сотрудников|<ul><li>**Учащиеся**: определения глобальной политики (по умолчанию для всей организации) на основе нового типа образовательного учреждения применяются к учащимся.</li><li>**Преподаватели и сотрудники**: создается набор пользовательских определений политик на основе нового типа образовательного учреждения, который назначается новым преподавателям и группам персонала. Предыдущие определения настраиваемой политики удаляются из предыдущих преподавателей и групп персонала.</li></ul>|
|Изменение только типа учебного заведения|<ul><li>**Учащиеся**: определения глобальной политики (по умолчанию для всей организации) на основе нового типа образовательного учреждения применяются к учащимся.</li><li>**Преподаватели и сотрудники**: набор определений настраиваемых политик на основе нового типа учебного заведения создается и назначается преподавателям и группам персонала. Определения настраиваемой политики, созданные для предыдущего типа учебного заведения, удаляются из преподавателей и групп персонала.</li></ul>|
|Изменение только преподавателей и групп персонала|<ul><li>**Учащиеся**: определения глобальной политики (по умолчанию для всей организации) не применяются к учащимся.</li><li>**Преподаватели и сотрудники**: пользовательские определения политик назначаются новым преподавателям и группам персонала и удаляются из предыдущих групп преподавателей и сотрудников.</li></ul>|
