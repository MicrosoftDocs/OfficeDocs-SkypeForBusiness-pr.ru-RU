# <a name="use-the-teams-for-education-policy-wizard-to-easily-apply-policies-for-a-safe-learning-environment"></a>Использование мастера политики обучения для упрощения применения политик в безопасной среде обучения

[!INCLUDE [preview-feature](includes/preview-feature.md)]

## <a name="overview"></a>Обзор

Мастер политики Microsoft Teams для образовательных учреждений упрощает управление политиками для учащихся и преподавателей. С его помощью вы сможете легко и быстро применять самый важный набор политик, необходимых для создания надежного и производительного обучения.

Политики в Teams позволяют управлять поведением групп в вашей среде, а также тем, какие функции доступны для пользователей. Например, существуют политики вызова, политики собраний и политики обмена сообщениями, чтобы присвоить имя нескольким, и каждая область политики может быть настроена в соответствии с потребностями Организации.

Для поддержки безопасной и письменной среды обучения важно настроить политики для управления тем, что учащийся может сделать в Teams. Например, политики можно использовать для управления тем, кто может использовать частный чат и частный звонок, кто может планировать собрания и какие типы контента можно предоставлять для общего доступа. Вы также можете использовать политики для включения функций Teams, которые улучшают обучение.

Политики должны быть скорректированы как для учащихся, так и для преподавателей, чтобы обеспечить безопасность обучающего процесса. Политики для учащихся должны быть более строгими, чтобы снизить риск получения недопустимых уровней доступа. Преподавателям и сотрудникам нужен отдельный набор политик, которые могут быть более строгими, чтобы их можно было успешно использовать. Например, разрешить преподавателям планировать собрания и ограничивать их выполнение.

:::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="Снимок экрана: мастер":::

В этой статье объясняется, как запустить мастер.

> [!IMPORTANT]
> Политики, примененные мастером, удовлетворяют требованиям большинства Teams для образовательных учреждений. Мастер настраивает глобальное определение основного набора политик с помощью параметров, которые мы рекомендуем использовать для учащихся, и применять их к студентам. Мастер также создает и назначает набор настраиваемых политик для преподавателей и персонала. Большинству специалистов по обучению не требуется использовать другие методы назначения политик после запуска этого мастера. Другие методы назначения политики следует использовать *только* в том случае, если вы хотите вручную создавать политики для учащихся, преподавателей и сотрудников и управлять ими.

## <a name="teams-for-education-policy-wizard"></a>Мастер настройки политики в Teams для образовательных учреждений

<a name="polwiz_intro"> </a>

Мастер применяет набор основных определений политик к студентам и отдельным наборам определений основной политики для преподавателей и сотрудников, с параметрами, подходящими для каждого из них. Ниже показано, что происходит при запуске мастера.

Мастер настраивает политики на основе учебного заведения (**основного или дополнительного** или **более более образования**). Вы выбираете тип учебного заведения, и мастер выполнит указанные ниже действия.

- **Students**: мастер настраивает глобальное определение политики (по умолчанию для всей организации) для каждой области политики, которая раскрывается мастером, с новыми параметрами по умолчанию, которые подходят для защиты учащихся. Это гарантирует получение наиболее строгого набора политик для текущих учащихся и всех новых учащихся.
- **Преподаватели и сотрудники**: мастер создает набор настраиваемых определений политик для каждой области политики, покрываемой мастером, с настройками, адаптированными для нужд преподавателей и персонала. Затем он назначает определения политики группе преподавателей и сотрудников, которые вы выбираете. Таким образом, ваши преподаватели и сотрудники получают более разрешающий набор политик, чтобы их можно было успешно использовать.

Запускать мастер нужно только один раз. Новые учащиеся автоматически получают глобальные политики (параметры по умолчанию в масштабах организации), примененные мастером, и новые сотрудники, добавленные в выбранную вами группу, автоматически назначаются пользовательским политикам.

> [!NOTE]
> Просмотрите список [политик, примененных мастером](#policies-applied-by-the-wizard) , для подробного списка определений политик, примененных мастером.

Теперь приступим!

## <a name="run-the-wizard"></a>Запуск мастера

<a name="polwiz_run"> </a>

Выполните указанные ниже действия, чтобы запустить мастер.

1. Если вы не знакомы с Teams, мастер автоматически запускается. В противном случае вы можете запустить мастер в любое время на панели мониторинга. В левой области навигации центра администрирования Microsoft Teams перейдите на **панель мониторинга**, а затем в окне **Простая настройка политики для плитки безопасной среды обучения** выберите пункт Экспресс- **Настройка**.

    :::image type="content" source="media/easy-policy-setup-quick-setup.png" alt-text="Снимок экрана: мастер на панели мониторинга":::

2. Выберите тип учебного заведения (**основного или дополнительного** или **более более образования**) и нажмите кнопку **Далее**.

    :::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="Снимок экрана: страница в мастере для выбора типа учебного заведения":::

3. Найдите и выберите группу, которая включает в себя ваших преподавателей и сотрудников, и нажмите кнопку **Далее**. Если вы еще не настроили ни одной из групп для преподавателей и сотрудников, [Создайте группу](https://docs.microsoft.com/microsoft-365/admin/create-groups/create-groups), а затем снова запустите мастер. <br/><br/>В настоящее время вы можете выбрать только одну группу. Для преподавателей и персонала в группе, которую вы выбрали, будет назначен [набор настраиваемых политик](#policies-applied-by-the-wizard) , адаптированных к их нуждам. Помните, что этот набор политик отделен от политик, применяемых к студентам.

    :::image type="content" source="media/easy-policy-setup-group.png" alt-text="Снимок экрана: страница мастера для выбора преподавателей и группы сотрудников":::

4. Просмотр выбранных параметров.

    :::image type="content" source="media/easy-policy-setup-review-selections.png" alt-text="Снимок экрана: страница в мастере для просмотра выбранных параметров":::

5. Нажмите кнопку **Применить** , чтобы применить изменения. Это может занять несколько минут.<br/><br/>Глобальные определения политик, используемые по умолчанию, будут немедленно применены для учащихся. Для преподавателей и персонала может потребоваться до 48 часов, после которого пользовательские политики будут назначены каждому участнику выбранной вами группы в зависимости от размера группы. Это происходит в фоновом режиме, после того как вы успешно завершите этот шаг.
6. Ты еще не закончится! Вот еще несколько вещей, которые следует обратить на вопрос. Далее ознакомьтесь с инструкциями, приведенными в разделе [что нужно сделать после запуска мастера](#what-to-do-after-running-the-wizard) в этой статье.

    :::image type="content" source="media/easy-policy-setup-on-way.png" alt-text="Снимок экрана: страница в мастере для дальнейших действий":::

## <a name="what-to-do-after-running-the-wizard"></a>Что делать после запуска мастера

<a name="polwiz_remove"> </a>

### <a name="step-1-remove-existing-policy-assignments-that-conflict-with-policies-applied-by-the-wizard"></a>Шаг 1: удаление существующих назначений политики, которые конфликтуют с политиками, примененными мастером

> [!IMPORTANT]
> **Заполните этот шаг только в том случае, если у вас есть политики, назначенные студентам, преподавателям и сотрудникам *до* запуска мастера**. Если вы новичок в Teams и у вас нет политик, которые не созданы мастером, пропустите это действие и перейдите к шагу 2.

В Teams для заданной области политики политика может быть применена к пользователю следующими способами:

- Прямое назначение пользователю
- Назначение группе, в которую входит пользователь
- Если пользователь не назначил политику или не является членом какой-либо группы, которой назначена политика, пользователь автоматически получает глобальную политику (по умолчанию на уровне Организации).

Если у пользователя есть более одного назначения политики, в Teams для определения политики применяется следующий порядок. Дополнительные сведения о том, [какая политика имеет приоритет?](assign-policies.md#which-policy-takes-precedence) и [правила приоритетов](assign-policies.md#precedence-rules).

|Назначение политики для пользователя|Политика, которая вступает в силу |
|---------|---------|
|Политика, назначенная группе: нет<br/>Политика, назначенная пользователю напрямую: нет    |Глобальная политика по умолчанию (в масштабах организации)      |
|Политика, назначенная группе: нет<br/>Политика, назначенная непосредственно пользователю: Да    |Политика, назначенная пользователю напрямую         |
|Политика, назначенная группе: Да<br/>Политика, назначенная непосредственно пользователю: Да     |Политика, назначенная пользователю напрямую         |
|Политика, назначенная группе: Да<br/>Политика, назначенная пользователю напрямую: нет     |Политика, назначенная группе<br/><br/>Если пользователь является членом нескольких групп, а каждая из них назначается для политики той же области политики, действует политика, которая [имеет наивысший приоритет.](assign-policies.md#group-assignment-ranking)       |

Из-за этого порядка политики, созданные мастером, не вступают в силу, если пользователь имеет действующие прямые назначения или назначения групп. Это означает, что вы должны удалить существующие назначения политики от пользователя, чтобы политика, примененная этим мастером, вступит в силу.

Для каждой [области политики, примененной мастером](#policies-applied-by-the-wizard), выполните указанные ниже действия.

- Удалите все существующие прямые назначения и групповые назначения из учащихся, чтобы применить определение политики, применяемой этим мастером, в силу.
- Удалите все конфликтующие прямые назначения для преподавателей и сотрудников, чтобы определение настраиваемой политики, созданное мастером, вступит в силу. С помощью приведенной выше таблицы определите применимые сценарии. <br/><br/>Имейте в виду, что мастер назначает политики вашим преподавателям и сотрудникам с помощью [ранжирования группового назначения](assign-policies.md#group-assignment-ranking) 1, что является наивысшим рейтингом. Если к вашим преподавателям и сотрудникам применена политика, существующая в той же области политики, она будет перемещена в более низкую ранжирование и будет применена политика, назначенная мастером.

[Узнайте больше](batch-group-policy-assignment-edu.md#remove-a-policy-that-was-directly-assigned-to-users) о том, как удалить политики, непосредственно назначенные пользователям.

Например, вы назначили политику собраний непосредственно для преподавателей, а у учащихся — Глобальная политика собраний (по умолчанию на уровне Организации). В этом случае удалите политику собраний, назначенную для преподавателей, чтобы определение особой политики для политики собраний, созданной мастером, вступит в силу. Для учащихся не нужно ничего делать с помощью существующей политики собраний по умолчанию на уровне Организации, так как другие политики не конфликтуют с ним.

<a name="polwiz_addmeasures"> </a>

### <a name="step-2-check-for-additional-measures-that-you-can-take-for-student-safety"></a>Шаг 2: Проверка наличия дополнительных мер, которые можно предпринять для безопасности учащихся

Мастер автоматически настраивает и применяет [эти политики](#policies-applied-by-the-wizard). Есть несколько дополнительных мер, которые можно использовать в зависимости от потребностей вашего учреждения, чтобы оставаться в курсе безопасных.

Узнайте о том, [как сохранить учащихся в безопасном режиме, используя Teams для обучения](https://support.microsoft.com/office/keeping-students-safe-while-using-teams-for-distance-learning-f00fa399-0473-4d31-ab72-644c137e11c8#ID0EBBAAA) дополнительным рекомендациям по безопасности.

<a name="polwiz_mc"> </a>

### <a name="step-3-check-message-center-for-policy-updates"></a>Шаг 3: Проверка обновлений политики в центре сообщений

В настоящее время мастер применяет рекомендуемые политики при запуске. Важно знать, что при доступе к новым политикам в Teams автоматически не добавляются глобальные параметры безопасности для учащихся (по умолчанию на уровне Организации). Эта возможность будет доступна в будущем выпуске.

До тех пор пока эта возможность не будет доступна, проверьте в [центре сообщений](https://admin.microsoft.com/AdminPortal/Home?#/MessageCenter) (в центре администрирования Microsoft 365), что вы можете следить за новыми политиками и параметрами политики в Teams. По мере появления новых функций может потребоваться вручную обновить политики, чтобы обеспечить безопасность вашей среды обучения.

## <a name="make-changes-in-the-wizard"></a>Внесение изменений в мастере

<a name="polwiz_change"> </a>

Если после запуска мастера вам нужно внести изменения, вы можете запустить его повторно и изменить параметры.

1. В левой области навигации центра администрирования Microsoft Teams перейдите на **панель мониторинга**, а затем в окне **Простая настройка политики для плитки безопасной среды обучения** нажмите кнопку **изменить**.
2. Начиная с этой страницы, продолжайте пользоваться всеми страницами мастера, чтобы внести изменения. Вы можете изменить тип учебного заведения, группу преподавателей и сотрудников, которым нужно назначить политики, или и то, и другое.

В следующей таблице приведены сведения о том, что происходит при внесении изменений в мастере.

|Тип изменения  |Поведение политики  |
|---------|---------|
|Изменение типа учебного заведения и преподавателей и группы сотрудников    |<ul><li>**Учащиеся**: глобальные определения политики (по умолчанию на уровне Организации), основанные на новом типе учебного заведения, применяются к студентам.</li><li>**Преподаватели и сотрудники**: набор определений настраиваемой политики, основанный на новом типе учебного заведения, создается и назначается новой группе преподавателей и персонала. Предыдущие пользовательские определения политики удаляются из предыдущих преподавателей и из группы staff.</li></ul>    |
|Изменение только типа учебного заведения    |<ul><li>**Учащиеся**: глобальные определения политики (по умолчанию на уровне Организации), основанные на новом типе учебного заведения, применяются к студентам.</li><li>**Преподаватели и сотрудники**: набор определений настраиваемой политики, основанный на новом типе учебного заведения, создается и назначается для преподавателей и группы staff. Пользовательские определения политик, созданные для предыдущего учебного учреждения, удаляются из преподавателей и из группы staff.</li></ul>         |
|Изменение только преподавателей и группы сотрудников   |<ul><li>**Учащиеся**: изменения глобальных политик (параметры по умолчанию на уровне Организации), примененные к студентам, не изменяются.</li><li>**Преподаватели и сотрудники**: пользовательские определения политик назначаются новым преподавателям и группам сотрудников и удаляются из предыдущих преподавателей и из группы staff.</li></ul>         |
