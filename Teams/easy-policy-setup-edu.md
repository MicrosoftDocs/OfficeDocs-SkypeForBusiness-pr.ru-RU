# <a name="use-the-teams-for-education-policy-wizard-to-easily-apply-policies-for-a-safe-learning-environment"></a>Использование мастера Teams для образования политики для простого применения политик для безопасной среды обучения

## <a name="overview"></a>Обзор

Мастер Microsoft Teams для образования упрощает управление политиками для учащихся и преподавателей. Используйте его для быстрого и простого применения наиболее важного набора политик, необходимых для создания безопасного и продуктивного обучения.

Политики в Teams могут контролировать поведение Teams в вашей среде и доступные пользователям функции. Например, существуют политики звонков, политики собраний и политики обмена сообщениями, которые можно назвать несколькими, и каждую область политики можно настроить в зависимости от потребностей вашей организации.

Чтобы обеспечить безопасную и уделяемую обучению среду, важно настроить политики для управления возможностью учащихся в Teams. Например, с помощью политик можно управлять тем, кто может использовать частный чат и частные вызовы, кто может планировать собрания и какие типы контента можно использовать совместно. Вы также можете использовать политики для Teams функций, которые о обогащении возможностей обучения.

Политики необходимо скорректировать как для учащихся, так и для преподавателей, чтобы сделать процесс обучения безопасным. Политики для учащихся должны быть более строгими, чтобы снизить риск получения несоответствующего уровня доступа. Преподавателям и сотрудникам необходим отдельный набор политик, которые позволят им добиться успеха. Например, разрешить преподавателям планировать собрания и запретить учащимся делать это.

:::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="Снимок экрана: мастер.":::

В этой статье вы можете узнать, как запустить мастер.

> [!IMPORTANT]
> Политики, применяемые мастером, будут удовлетворять потребностям большинства Teams для образования клиентов. Мастер корректирует глобальное определение (по умолчанию в организации) основного набора политик с помощью параметров, рекомендуемых для безопасности учащихся, и применяет его для учащихся. Мастер также создает и назначает набор настраиваемой политики преподавателям и сотрудникам. Большинству Teams для образования клиентам не придется использовать другие методы назначения политик после запуска этого мастера. Используйте другие методы назначения *политики* только в том случае, если вы хотите вручную создать политики для учащихся, преподавателей и сотрудников и управлять ими.

## <a name="teams-for-education-policy-wizard"></a>Teams для образования Мастер политик

<a name="polwiz_intro"> </a>

Мастер применяет набор основных определений политик для учащихся и отдельный набор основных определений политики для преподавателей и сотрудников с подходящими для каждого из них настройками. Вот что происходит при запуске мастера.

Мастер настраивает политики на основе типа образовательного учреждения **(основной** или дополнительный или **высший учебный).** Вы выбираете тип учебного заведения, и мастер делает следующее:

- **Учащиеся.** Мастер настраивает определение глобальной политики (по умолчанию в организации) для каждой области политики, в которую он распространяется, с помощью новых параметров по умолчанию, которые подходят для безопасности учащихся. Это гарантирует, что текущие учащиеся и все новые учащиеся получат самый строгий набор политик.
- **Преподаватели** и сотрудники. Мастер создает набор настраиваемых определений политик для каждой области политики, влияемой в мастере, с настройками, разработанными для преподавателей и сотрудников. Затем определения политики назначаются группе преподавателей и сотрудников, которые вы выбрали. Таким образом, преподаватели и сотрудники получат более разрешительный набор политик, позволяющих им добиться успеха.

Мастер нужно запустить только один раз. Новые учащиеся автоматически получают определения глобальной политики (по умолчанию в организации), применяемые мастером, и новым сотрудникам, добавленным в выбранную группу, автоматически назначены настраиваемые политики.

Кроме того, при добавлении новой функции в Teams соответствующее значение по умолчанию для соответствующего EDU значения политики для этой функции автоматически добавляются в глобальный (по умолчанию в организации) без вмешательства администратора. Это гарантирует, что будут добиться правильных политик, чтобы обеспечить безопасность и вовлечение учащихся.

> [!NOTE]
> Подробный [список определений](#policies-applied-by-the-wizard) политик, применяемых мастером, см. в описании политик, применяемых мастером.

Теперь давайте начнем!

## <a name="run-the-wizard"></a>Запуск мастера

<a name="polwiz_run"> </a>

Выполните эти действия, чтобы запустить мастер.

1. Если вы еще не Teams, мастер запускается автоматически. В противном случае вы можете запустить мастер в любое время с панели мониторинга. В левой области навигации Центра администрирования Microsoft Teams перейдите на  главная **,** а затем на плитке Простая настройка политики для безопасной среды обучения выберите **Быстрая настройка**.

    :::image type="content" source="media/easy-policy-setup-quick-setup.png" alt-text="Снимок экрана: мастер на панели мониторинга.":::

2. Выберите тип учебного заведения (основной **или** дополнительный или высший), а затем выберите **Далее**.

    :::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="Снимок экрана: страница мастера для выбора типа учебного заведения.":::

3. Найщите и выберите группу, которая содержит преподавателей и сотрудников, а затем выберите **Далее**. Если у вас еще нет групп для преподавателей и [](/microsoft-365/admin/create-groups/create-groups)сотрудников, создайте группу и снова запустите мастер. <br/><br/>В настоящее время можно выбрать только одну группу. Преподавателям и сотрудникам выбранной группы [](#policies-applied-by-the-wizard) будет назначен набор настраиваемой политики, соответствующий их потребностям. Помните, что этот набор политик не является политикой, примененной к учащимся.

    :::image type="content" source="media/easy-policy-setup-group.png" alt-text="Снимок экрана: страница в мастере для выбора преподавателей и групп сотрудников.":::

4. Просмотрите выбор.

    :::image type="content" source="media/easy-policy-setup-review-selections.png" alt-text="Снимок экрана: страница в мастере для проверки выбора.":::

5. Чтобы **применить изменения,** выберите применить. Это может занять несколько минут.<br/><br/>Определения глобальной (по умолчанию в организации) политики немедленно применяются к учащимся. Для преподавателей и сотрудников настраиваемые политики могут быть назначены каждому участнику выбранной группы в зависимости от ее размера. Это происходит в фоновом режиме после успешного выполнения этого шага.
6. Вы уже на пути, но еще не сделали этого! Следует учесть еще несколько моментов. Затем ознакомьтесь с действиями в разделе Что делать после [запуска мастера](#what-to-do-after-running-the-wizard) этой статьи.

    :::image type="content" source="media/easy-policy-setup-on-way.png" alt-text="Снимок экрана: страница в мастере для следующих действий.":::

## <a name="what-to-do-after-running-the-wizard"></a>Что необходимо сделать после запуска мастера

<a name="polwiz_remove"> </a>

### <a name="step-1-remove-existing-policy-assignments-that-conflict-with-policies-applied-by-the-wizard"></a>Шаг 1. Удаление существующих назначений политик, которые конфликтуют с политиками, применяемыми мастером

> [!IMPORTANT]
> **Выполните это  только в том** случае, если перед началом работы мастера вам назначены существующие политики для учащихся, преподавателей и сотрудников. Если вы раньше не Teams и у вас нет политик, кроме политик, созданных мастером, пропустите этот шаг и перейдите к шагу 2.

В Teams для заданной области политики к пользователю можно применить политику следующим образом:

- Прямое назначение пользователю
- Назначение группе, участником группы является пользователь
- Если пользователю не назначена политика напрямую или он не является участником какой-либо группы, ему автоматически предоставляется глобальная политика (по умолчанию в организации).

Если для пользователя существует несколько назначений политики, Teams определяет, какое назначение политики будет действовать, в следующем порядке. Дополнительные сведения см. в [том, какая политика](policy-assignment-overview.md#which-policy-takes-precedence) имеет приоритет или [правила приоритета для групп.](assign-policies-users-and-groups.md#precedence-rules)

|Назначения политик пользователю|Политика, которая вступает в силу |
|---------|---------|
|Политика, назначенная группе: Нет<br/>Политика, назначенная пользователю напрямую: Нет    |Стандартная политика глобальной (для всей организации)      |
|Политика, назначенная группе: Нет<br/>Политика, назначенная пользователю напрямую: Да    |Политика, назначенная пользователю напрямую         |
|Политика, назначенная группе: Да<br/>Политика, назначенная пользователю напрямую: Да     |Политика, назначенная пользователю напрямую         |
|Политика, назначенная группе: Да<br/>Политика, назначенная пользователю напрямую: Нет     |Политика, назначенная группе<br/><br/>Если пользователь входит в несколько групп и каждой группе назначена политика одной и той же [](assign-policies-users-and-groups.md#group-assignment-ranking) области политики, вступает в силу политика с наивысшим ранжированием назначения группы.       |

Из-за этого порядка политики, созданные мастером, не будут действовать, если у пользователя уже есть прямые назначения или назначения группы. Это означает, что вам придется удалить существующие назначения политики для пользователя, чтобы политика, примененная мастером, вступила в силу.

Для каждой [области политики, применяемой мастером,](#policies-applied-by-the-wizard)сделайте следующее:

- Удалите все существующие прямые задания и групповые задания учащихся, чтобы в действие вступает в силу определение глобальной политики (по умолчанию в организации), применяемого мастером.
- Удалите все конфликтующие прямые назначения для преподавателей и сотрудников, чтобы настраиваемые определения политики, созданные мастером, вступили в силу. Используйте таблицу выше для определения возможных сценариев. <br/><br/>Имейте в виду, что мастер назначает политики вашим преподавателям и группе сотрудников с помощью ранжирования заданий группы из 1 — самого высокого ранжирования. [](assign-policies-users-and-groups.md#group-assignment-ranking) Если вашим преподавателям и группе сотрудников назначена существующая политика той же области политики, она перемещается в более низкий рейтинг, а политика, назначенная мастером, вступает в силу.

[Узнайте больше](batch-group-policy-assignment-edu.md#remove-a-policy-that-was-directly-assigned-to-users) о том, как удалить политики, которые непосредственно назначены пользователям.

Например, вы написали политику собраний напрямую преподавателям, а учащимся назначена глобальная (по умолчанию в организации) политика собраний. В этом случае удалите назначенную преподавателям политику собраний, чтобы настраиваемые определения политики собраний, созданные мастером, вступили в силу. Вам не нужно ничего делать с существующей глобальной (по умолчанию в организации) политикой собраний для учащихся, так как другие политики собраний не конфликтуют с ней.
