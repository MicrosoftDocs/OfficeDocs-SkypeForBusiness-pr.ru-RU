# <a name="using-the-cqd-pstn-direct-routing-report"></a>Использование отчета о прямой маршрутизации CQD PSTN

Новая версия в марте 2020 добавлена в [шаблоны запросов на Power BI для CQD](https://github.com/MicrosoftDocs/OfficeDocs-SkypeForBusiness/blob/live/Teams/downloads/CQD-Power-BI-query-templates.zip?raw=true), CQD КТСОП Direct Routing. 


Отчет о маршрутизации CQD PSTN помогает пользователям понять принципы использования и качество обслуживания своих служб PSTN, а также сведения о том, что такое служба телефонной связи, параметры сети и отношение к эффективности сети и использование сервиса. Эти сведения помогут вам определить проблемы, в том числе причину прерванных звонков. Например, вы будете знать, когда громкость снижается, сколько звонков повлияет на все причины.

В отчете о маршрутизации CQD PSTN есть четыре раздела:

  - [Общие сведения о PSTN](#pstn-overview)

  - [Сведения о службе](#service-details)

  - [Коэффициент эффективности сети](#network-effectiveness-ratio)

  - [Сетевые параметры](#network-parameters)

## <a name="pstn-overview"></a>Общие сведения о PSTN

В отчете о маршрутизации CQD PSTN указаны следующие сведения, относящиеся к общей работоспособности службы за последние 180 дней.
![Снимок экрана: отчет о PSTN CQD](media/CQD-PSTN-report1.png)

Например, если вы заинтересованы в общем использовании и работоспособности всех входящих звонков, которые проходит через SBC abc.bca.adatum.biz с нами в качестве внутренней страны, сделайте следующее:

| **Вызовите** | **Описание**                                                                                                                                                 |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1            | Вы можете использовать фильтры в верхней части экрана для детализации и выбрать Бйотин в качестве типа звонка, abc.bca.contoso.com в качестве контроллера сеанса и в качестве внутренней страны. |
| 2            | Тенденция использования за последние 180 дней. Вы можете найти подробный отчет об использовании на странице сведений о службе.                                                                     |
| 3            | Задержка набора номера, задержка, колебание и тенденция потери пакетов за последние 180 дней. Вы можете найти подробный отчет на странице Параметры сети.                           |
| 4            | Одновременный звонок и дневная активная тенденция пользователей за последние 180 дней. Эта диаграмма поможет вам понять, как это максимальный объем услуг.                            |
| 5            | Причина, по которой заканчивается качество обслуживания по верхнему краю за последние 180 дней. Вы можете найти сведения о работоспособности службы на странице коэффициент эффективности сети (Нер).                    |

## <a name="service-details"></a>Сведения о службе

Эта страница обеспечивает тенденции использования служб за день и отзыв пользователей по географическим подразделениям.

  - **Всего попыток вызова:** Общее количество одновременных звонков в указанном диапазоне времени, включая успешные и неудачные вызовы

  - **Общее количество подключенных звонков-** Общее количество подключенных вызовов в указанном диапазоне времени

  - **Всего минут –** Общее использование минут в указанном диапазоне времени

  - **Ежедневные активные пользователи (ДАУ) —** Количество активных пользователей, которые выполнили по крайней мере один подключенный Звонок в этот день.

  - **Одновременные звонки:** Максимум одновременных активных звонков за минуту

  - **Отзывы пользователей:** "Оценить баллы о звонке", поступает от пользователя. 3-5 считается хорошим звонком. 1-2 считается неправильным звонком.

![Снимок экрана: отчет о PSTN CQD](media/CQD-PSTN-report2.png)

Например:

1.  Если вы видите среднюю длительность звонка в 0 в 02/14/2020, вы можете сначала проверить, выглядит ли громкость звонка обычно, и узнать, есть ли существенные отличия между общим соединением и общим количеством одновременных звонков. Затем перейдите на страницу коэффициента эффективности сети, чтобы присоединиться к причине ошибки звонка.

2.  Если вы видите более красные пятна на карте отзывов пользователей, вы можете перейти на страницу коэффициента эффективности сети и сетевой параметр, чтобы проверить, есть ли какие-либо аномалии, и вы можете создать билет с помощью службы поддержки Майкрософт.

## <a name="network-effectiveness-ratio"></a>Коэффициент эффективности сети

Это та же метрика, которая отображается на общей информационной панели мониторинга работоспособности. Вы можете проверить почасовое Нер число с помощью сведений о вызываемых звонках для обоих направлений звонка (входящий/исходящий трафик) на почасовом соотношении эффективности сети и конечной причины звонка ниже.

  - **Нер** — возможность (%) сети, чтобы доставлять звонки, измеряя количество отправленных звонков и количество звонков, доставленных получателю.

  - **Код ответа SIP**— трехзначный целочисленный код ответа показывает состояние вызова.

  - **Код ответа Microsoft**— код ответа, отправленный из компонента Microsoft.

  - **Описание** — этап причины, соответствующий коду ответа SIP и коду ответа Microsoft.

  - **Количество затронутых звонков** — общее количество звонков, которые были затронуты в течение выбранного интервала времени.

> ![Снимок экрана: отчет о PSTN CQD](media/CQD-PSTN-report3.png)
> 
Например:

![Снимок экрана: отчет о PSTN CQD](media/CQD-PSTN-report4.png)

Если для ежедневного Нер установлен DIP-значение в 02/05/2020, вы можете щелкнуть дату, и другие диаграммы будут масштабироваться до указанной даты.

![Снимок экрана: отчет о PSTN CQD](media/CQD-PSTN-report5.png)

С почасовой НЕРой процентной тенденции вы можете найти DIP, выступающей около 21:00. Затем щелкните еще раз, чтобы перейти к часу 21 и проверить сведения о вызываемом звонке, чтобы узнать, сколько произошел сбой звонков в течение этого часа и каковы причины, по которым нужно закончить звонок. Вы можете приступить к устранению проблем, связанных с SBC, или сообщить в службу поддержки, если проблема не связана с SBC.

## <a name="network-parameters"></a>Сетевые параметры

Все сетевые параметры измеряются от интерфейса Direct Routing на контроллере границ сеанса. Дополнительные сведения о рекомендуемых значениях можно найти в [статье Подготовка сети организации для Microsoft Teams](prepare-network.md)и ознакомьтесь со статьей "клиент" в разделе "Рекомендуемые значения Microsoft Edge".

  - **Колебание** — это величина, которая измеряется в миллисекундах времени задержки распространения сети, вычисленной между двумя конечными точками с помощью Рткп (протокол управления RTP).

  - **Потери пакетов** — это мера пакета, который не удалось получить; Она вычисляется между двумя конечными точками.

  - **Задержка** (также известная как время кругового поездки) — продолжительность отправки сигнала и время, необходимое для подтверждения подлинности этого сигнала. Это время задержки состоит из времени распространения между двумя точками сигнала.

> ![Снимок экрана: отчет о PSTN CQD](media/CQD-PSTN-report6.png)

Например:

Если вы видите копилку на любой из четырех диаграмм (задержка, колебание, частота потерь пакетов, задержка набора номера) для определенной даты, например задержка в 02/14/2020, щелкните точку даты. А диаграмма почасового тренда внизу будет обновлена, чтобы показать почасовое число. Вы можете проверить SBCs или создать билет с помощью службы MS Service Desk.

![Снимок экрана: отчет о PSTN CQD](media/CQD-PSTN-report7.png)
