# <a name="use-advisor-for-teams-to-help-you-roll-out-microsoft-teams"></a>Использование помощника по Teams для развертывания Microsoft Teams

[!INCLUDE [preview-feature](includes/preview-feature.md)]

Помощник по Teams (предварительная версия) поможет вам выполнить все необходимые действия для развертывания Microsoft Teams. Он оценивает вашу среду клиента Office 365 и определяет самые распространенные конфигурации, которые, возможно, потребуется обновить или изменить, чтобы успешно выполнить развертывание Teams. Затем помощник по Teams создает команду развертывания (в Teams) и каналы для каждой рабочей нагрузки, которую нужно развернуть. К каждой рабочей нагрузке в команде развертывания прилагается комплексный план Планировщика, включающий все задачи развертывания для каждой рабочей нагрузки.  Используя такой план Планировщика, вы назначаете задачи сотрудникам, отвечающим за каждый этап развертывания, в том числе руководителю проекта, администраторам Teams и Office 365, а также специалистам по поддержке, внедрению и подготовке пользователей. Каждая задача развертывания содержит все инструкции и ресурсы, необходимые для успешного выполнения такой задачи.

Помощник по Teams является компонентом [Центра администрирования Teams](https://admin.teams.microsoft.com). Чтобы использовать помощник по Teams впервые, нажмите кнопку **Пуск** в мини-приложении **Развертывание рабочей нагрузки** на информационной панели. Или перейдите в **помощник по** > **планированию**.

> [!IMPORTANT]
> Помощник по Teams недоступен для развертываний Microsoft 365 для государственных организаций — GCC High или DoD.

## <a name="using-advisor-for-teams-preview"></a>Использование помощника по Teams (предварительная версия)

Вам не обязательно быть администратором Teams, чтобы использовать помощник по Teams, — его может использовать любой пользователь в вашей организации. Мы настроили особые разрешения, поэтому пользователи, не являющиеся администраторами, могут получить доступ к помощнику по Teams, несмотря на то, что он находится в Центре администрирования Teams. Тем не менее, вам необходимо быть администратором Teams, администратором службы Teams или глобальным администратором, чтобы открыть оценку готовности клиента.

Когда вы приступаете к использованию помощника по Teams, для вас в Teams создается команда развертывания. Помощник добавляет каналы для каждой рабочей нагрузки, которую нужно развернуть. 


## <a name="available-advisor-for-teams-plans"></a>Доступные планы помощника по Teams

Пока доступна предварительная версия помощника по Teams, мы предоставляем два указанных ниже плана.

1. Чат, команды, каналы и приложения
    - Оценка клиента
    - План Планировщика, включая задачи по внедрению
    - Опрос пользователей Forms
1. Собрания и конференции
    - Оценка клиента
    - План Планировщика, включая задачи по внедрению
    - Опрос пользователей Forms

Рекомендуем начать с плана "Чат, команды, каналы и приложения". После того как вы закончите развертывание этой рабочей нагрузки, вернитесь к помощнику и щелкните **Добавить канал**, чтобы приступить к следующей рабочей нагрузке. 

## <a name="tenant-assessment"></a>Оценка клиента
Каждый план включает оценку готовности клиента, которую можно использовать для выявления и устранения любых недостатков в вашей среде, прежде чем развернуть Teams. Ниже приводится описание того, что проверяется в ходе каждой проверки.

### <a name="advisor-bot"></a>Бот помощника
После того как помощник создаст команду развертывания, бот помощника отобразит приведенное ниже сообщение.

>**Добро пожаловать в команду развертывания Microsoft Teams!**
>  
>Цель этой команды — пошаговое выполнение развертывания Teams в вашей организации с предоставлением всех нужных вам ресурсов и пространства для совместной работы проектной группы. Каждый канал, созданный с помощью помощника по Teams, включает пошаговый план Планировщика и другие ресурсы, такие как опрос пользователей Forms, который можно использовать в ходе развертывания. В любой момент вы можете вернуться к оценке готовности клиента или добавить дополнительные планы рабочей нагрузки, используя Центр администрирования Teams. 
> 
>**Что нужно сделать?** 
>- Если вы еще не работали с Teams или Планировщиком, ознакомьтесь с [пошаговым руководством по Teams](https://teamsdemo.office.com/) и посмотрите [видео с краткими руководствами по началу работы с Планировщиком.](https://support.office.com/article/microsoft-planner-video-training-4d71390f-08d8-4db0-84ea-92fb078687c7). 
>- Перейдите в команду развертывания в Teams. Выберите канал своей рабочей нагрузки (например, "Чат, группы, каналы и приложения"), а затем откройте вкладку **Планировщик**, чтобы начать работу.
> 
>Дополнительные сведения о помощнике по Teams см. в статье [Использование помощника по Teams для развертывания Microsoft Teams](use-advisor-teams-roll-out.md).
>
> [!IMPORTANT]
> Бот помощника по Teams используется только для отправления приветственного сообщения вашей команде развертывания. Дополнительные данные не собираются.

> [!IMPORTANT]
> Бот помощника по Teams включен по умолчанию. Не отключайте его, если вы используете или планируете использовать помощник по Teams.


## <a name="frequently-asked-questions"></a>Вопросы и ответы
### <a name="what-are-the-licensing-requirements-for-advisor-for-teams"></a>Каковы требования к лицензированию для помощника по Teams?
Дополнительных требований к лицензированию, кроме лицензирования для Teams, нет.

### <a name="can-i-delete-the-deployment-team"></a>Можно ли удалить команду развертывания?
После того как помощник по Teams создаст команду развертывания, этой командой можно управлять так же, как и любой другой командой, включая возможность удаления. Имейте в виду, если вы не удалите команду с помощью Центра администрирования Teams, отображаемые сведения будут указывать на то, что команда существует.

### <a name="can-i-add-or-remove-channels-in-the-deployment-team"></a>Можно ли добавлять и удалять каналы в команде развертывания?
Да. После создания команды развертывания вы можете управлять каналами так же, как и в любой другой команде.

### <a name="can-i-add-or-remove-project-team-members-in-the-deployment-team"></a>Можно ли добавлять и удалять участников команды проекта в команде развертывания?
Да. После создания команды развертывания ею можно управлять так же, как и любой другой командой.

### <a name="can-i-modify-the-planner-plans"></a>Можно ли изменить планы Планировщика?
Да. После того как помощник по Teams создаст команду развертывания, нужно обновить план Планировщика, чтобы он наилучшим образом поддерживал развертывание Teams. Вы можете изменять что угодно: сегменты, задачи и сведения о задачах, — как и любые другие планы Планировщика.


### <a name="can-i-modify-the-forms-survey"></a>Можно ли изменить опрос Forms?
Да. После того как помощник по Teams создаст команду развертывания, можно будет изменить опрос Forms согласно требованиям.

### <a name="what-information-is-advisor-for-teams-collecting-about-my-organization"></a>Какие сведения помощник по Teams собирает о моей организации?
Помощник по Teams запрашивает ваше согласие на сбор данных, не являющихся личными сведениями пользователей (EUII). Осуществляется сбор данных телеметрии, что позволяет корпорации Майкрософт определить, насколько успешные результаты дает использование помощника по Teams и в каких случаях их можно улучшить. Эти же сведения используются для определения возможностей своевременного взаимодействия корпорации Майкрософт с вашей организацией с целью предоставления поддержки в развертывании.

### <a name="can-i-use-advisor-for-teams-with-fasttrack"></a>Можно ли использовать помощник по Teams вместе с FastTrack?
Да. FastTrack использует помощник по Teams для всех клиентов, желающих развернуть Teams. Предоставляется помощь в первоначальной настройке команды развертывания с использованием помощника по Teams (если это необходимо), а также поддержка по определенным вопросам развертывания Teams при их возникновении.

### <a name="can-i-use-advisor-for-teams-with-a-partner"></a>Можно ли использовать помощник по Teams вместе с партнером?
Да. Помощник по Teams можно использовать одновременно с партнером по развертыванию для Teams. Если ваш партнер является поставщиком облачных служб (CSP) и управляет вашим клиентом от вашего имени, он может использовать помощник по Teams, чтобы создать команду развертывания и помочь вам выполнить весь проект. Кроме того, вы можете работать с любым партнером, добавляя таких пользователей в свою команду развертывания в качестве гостей, чтобы они могли принимать участие в команде общего проекта.
