# <a name="use-advisor-for-teams-to-help-you-roll-out-microsoft-teams"></a>Использование помощника для Teams для развертывания Microsoft Teams

Помощник для Teams поможет вам выполнить все необходимые действия для развертывания Microsoft Teams. Он оценивает среду вашей организации Microsoft 365 и определяет самые распространенные конфигурации, которые, возможно, потребуется обновить или изменить, чтобы успешно выполнить развертывание Teams. Затем помощник для Teams создает команду развертывания (в Teams) и каналы для каждой рабочей нагрузки, которую нужно развернуть. К каждой рабочей нагрузке в команде развертывания прилагается комплексный план Планировщика, включающий все задачи развертывания для каждой рабочей нагрузки.  Используя такой план Планировщика, вы назначаете задачи сотрудникам, отвечающим за каждый этап развертывания, в том числе руководителю проекта, администраторам службы Teams, а также специалистам по поддержке, внедрению и подготовке пользователей. Каждая задача развертывания содержит все инструкции и ресурсы, необходимые для успешного выполнения такой задачи.

Помощник для Teams является компонентом [Центра администрирования Teams](https://admin.teams.microsoft.com). Как минимум, у вас должна быть лицензия на Microsoft 365 бизнес базовый, чтобы вы могли воспользоваться интеграцией помощника для Teams с Forms и Планировщиком. Чтобы начать использовать помощника для Teams, нажмите кнопку **Начать** в мини-приложении **Развертывание рабочей нагрузки Teams** на информационной панели. Или выберите **Планирование** > **Помощник для Teams**.

> [!IMPORTANT]
> Помощник для Teams недоступен для развертываний Microsoft 365 для государственных организаций (GCC High или DoD).

Интерактивный обзор интерфейса помощника для Teams см. в видео Microsoft Mechanics [Развертывание и настройка Microsoft Teams](https://youtu.be/o2mlsUubIO4?t=50).

## <a name="using-advisor-for-teams"></a>Использование помощника для Teams

**Чтобы использовать помощника для Teams, требуются лицензии Teams, Forms и Планировщика.** Но вам не обязательно быть администратором Teams, чтобы использовать помощника для Teams, — его может использовать любой пользователь в вашей организации. Мы настроили особые разрешения, поэтому пользователи, не являющиеся администраторами, могут получить доступ к помощнику для Teams, несмотря на то, что он находится в Центре администрирования Teams. Чтобы открыть оценки готовности клиентов, вы ДОЛЖНЫ быть администратором Teams, администратором службы Teams или глобальным администратором (так как у особых ролей, не являющихся администраторами, нет доступа к API Microsoft Graph, на основе которых выполняются оценки).

> [!IMPORTANT]
> Если пункт **Помощник для Teams** отсутствует в разделе **Планирование** в Центре администрирования Teams, значит у пользователя нет лицензии на Teams.

Когда вы приступаете к использованию помощника для Teams, для вас в Teams создается команда развертывания. Она добавляет канал для каждой выбранной рабочей нагрузки.

> [!IMPORTANT]
> Если команда развертывания уже создана и другой пользователь пытается создать ее, возникнет ошибка, сообщающая ему о необходимости обратиться в службу поддержки. Это предотвращает непреднамеренное раскрытие сведений Teams о существующих командах и их участниках. Попросите владельца команды развертывания добавить вас или обратитесь за помощью к специалисту службы поддержки.

## <a name="available-advisor-for-teams-plans"></a>Доступные планы помощника для Teams

В настоящее время помощник для Teams предоставляет следующие планы:

1. Чат, команды, каналы и приложения
    - Оценка клиента
    - План Планировщика, включая задачи по внедрению
    - Опрос пользователей Forms
    - Бот помощника для Teams
1. Собрания и конференции
    - Оценка клиента
    - План Планировщика, включая задачи по внедрению
    - Опрос пользователей Forms
    - Бот помощника для Teams
1. Обновление Skype для бизнеса
    - Оценка клиента
    - План Планировщика, включая задачи по внедрению
    - Опрос пользователей Forms
    - Бот помощника для Teams
    - План обновления Skype для бизнеса, разработанный для клиентов, которые в настоящее время используют локальные среды Skype для бизнеса Online или Skype для бизнеса, поможет вам угадать догадки в процессе обновления. Используя проверенные рамки успеха для внедрения изменений, план проведет вас через пошаговый процесс, независимо от того, начинаете ли вы работать с Teams, уже используете Teams вместе со Skype для бизнеса или готовы к обновлению. План также соединит вас с [онлайн-руководством и лучшими практиками](./upgrade-start-here.md), [загружаемыми ресурсами](https://aka.ms/UpgradeSuccessKit), [действующим 1: много семинаров по планированию](./upgrade-workshops-landing-page.yml) и дополнительными ресурсами для поддержки вашего успеха.

Рекомендуем начать с плана "Чат, команды, каналы и приложения". После развертывания этой рабочей нагрузки вернитесь к помощнику для Teams и выберите **Добавить канал**, чтобы приступить к следующей рабочей нагрузке.

## <a name="tenant-assessment"></a>Оценка клиента

Каждый план включает оценку готовности клиента, которую можно использовать для быстрого выявления аспектов вашей среды, которые может потребоваться исправить до развертывания Teams. Оценки включают предварительные требования и рекомендации. Каждый тест оценки содержит зеленую галочку или оранжевый треугольник предупреждения.

- <sub><img src="media/use-advisor-teams-roll-out-image2.png" alt="Green check mark"/></img></sub>Зеленая галочка означает, что ваш клиент прошел конкретный тест.
- <sub><img src="media/use-advisor-teams-roll-out-image1.png" alt="Yellow alert mark"/></img></sub>Оранжевый треугольник предупреждения означает, что мы рекомендуем выполнять отслеживание для определения необходимости каких-либо действий (например, политика срока действия группы Microsoft 365 является рекомендуемой, но не обязательной).

> [!IMPORTANT]
> После того как пользователь с ролью администратора запустит помощника для Teams, все оценки выполняются в фоновом режиме. Если вы выполняете обновление или исправление, оно может не отражаться в ваших оценках в течение 24 часов.

В разделах ниже описаны все оценки с указанием того, какие из них относятся к предварительным требованиям или рекомендациям, какие проверки выполняет каждая оценка и почему, а также представлено руководство по исправлению в случае необходимости.

### <a name="advisor-for-teams-bot"></a>Бот помощника для Teams

После того как помощник для Teams создаст команду развертывания, бот помощника отобразит следующее сообщение в канале "Общий":

>**Добро пожаловать в команду развертывания Microsoft Teams!**
>  
>Цель этой команды — пошаговое выполнение развертывания Teams в вашей организации с предоставлением всех нужных вам ресурсов и пространства для совместной работы проектной группы. Каждый канал, созданный с помощью помощника для Teams, включает пошаговый план Планировщика и другие ресурсы, такие как опрос пользователей Forms, который можно использовать в ходе развертывания. В любой момент вы можете вернуться к оценке готовности клиента или добавить дополнительные планы рабочей нагрузки, используя Центр администрирования Teams.
>
>**Что нужно сделать?**
>
>- Если вы еще не работали с Teams или Планировщиком, ознакомьтесь с [пошаговым руководством для Teams](https://teamsdemo.office.com/) и посмотрите [видео с краткими руководствами по началу работы с Планировщиком.](https://support.office.com/article/microsoft-planner-video-training-4d71390f-08d8-4db0-84ea-92fb078687c7).
>- Перейдите в команду развертывания в Teams. Выберите канал своей рабочей нагрузки (например, "Чат, группы, каналы и приложения"), а затем откройте вкладку **Планировщик**, чтобы начать работу.
>
>Дополнительные сведения о помощнике для Teams см. в статье [Использование помощника для Teams для развертывания Microsoft Teams](use-advisor-teams-roll-out.md).
>

> [!IMPORTANT]
> Бот помощника для Teams используется только для отправления приветственного сообщения вашей команде развертывания. Дополнительные данные не собираются.

> [!IMPORTANT]
> Бот помощника для Teams включен по умолчанию. Не отключайте его, если вы используете или планируете использовать помощника для Teams.

## <a name="advisor-for-teams-and-microsoft-365-learning-pathways"></a>Помощник для Teams и схемы обучения Microsoft 365

[Схемы обучения Microsoft 365](/office365/customlearning/) — это решение для обучения по запросу, которое можно настроить для обучения пользователей и расширения использования и внедрения Teams в организации. Используйте схемы обучения вместе с помощником для Teams, чтобы быстро подготовить пользователей и обеспечить внедрение.

Схемы обучения предоставляют вам шаблон сайта SharePoint Online и возможность легко создать сайт обучения для пользователей. Вы можете настроить учебный портал схем обучения, чтобы включить содержимое обучения и поддержки, предназначенное специально для нужд ваших пользователей. Используйте списки воспроизведения Teams из сетевого каталога Майкрософт и добавляйте собственные.

Вы можете создать сайт обучения в схемах обучения и добавить его в виде вкладки в канал Teams для быстрого и удобного доступа пользователей.

Например, используйте помощника для Teams вместе со схемами обучения, чтобы подготовить службу технической поддержки и лидеров, а затем с помощью схем обучения учить ваших пользователей. Создайте сайт обучения для подключения службы технической поддержки и лидеров к Teams, а затем добавьте его в качестве вкладки в каждый канал рабочих нагрузок, которые вы разворачиваете. Ваша служба технической поддержки и лидеры смогут создать страницу поддержки на учебном портале схем обучения со ссылками и настраиваемыми списками воспроизведения для поддержки ваших пользователей в Teams. Эту страницу поддержки можно добавить в канал любой команды, чтобы обучать ваших пользователей.

Ниже представлен обзор способа использования помощника для Teams вместе со схемами обучения.

### <a name="get-started-in-learning-pathways"></a>Начало работы в схемах обучения

Чтобы приступить к работе со схемами обучения, ознакомьтесь с разделом [Начало работы со схемами обучения](/office365/customlearning/).

Чтобы настроить новое решение схем обучения в своей среде, ознакомьтесь со статьей [Подготовка нового решения для схем обучения](/office365/customlearning/custom_provision).

### <a name="create-a-learning-plan"></a>Создание плана обучения

#### <a name="plan-your-learning-content"></a>Планирование учебного содержимого

Прежде чем создать сайт со схемами обучения, уделите время анализу и сбору ресурсов, а также возможностей обучения, доступных для вас. В схемах обучения вы можете использовать содержимое со страницы обучения Microsoft 365 и добавить содержимое, создаваемое для настройки сайта под ваши уникальные потребности.

Дополнительные сведения см. в статьях [Планирование содержимого для схем обучения](/office365/customlearning/custom_plancontent) и [Ресурсы для поддержки удаленных сотрудников](/office365/customlearning/custom_plancontent_remoteresources).

#### <a name="explore-teams-content-in-learning-pathways"></a>Изучение содержимого Teams в схемах обучения

Схемы обучения предоставляют сайт SharePoint с веб-частью, подключенной к сетевому каталогу. Страница обучения Microsoft 365, на которой размещается веб-часть, содержит все учебные курсы, доступные в схемах обучения. Осмотритесь, чтобы ознакомиться с доступными возможностями и способом организации содержимого.

[Перейдите на свой сайт схем обучения](/office365/customlearning/custom_goto), выберите **Обучение Microsoft 365** и нажмите **Microsoft Teams**, чтобы просмотреть все учебные списки воспроизведения Teams в сетевом каталоге. Выберите список воспроизведения и нажимайте кнопки **Далее** и **Назад**, чтобы перемещаться в нем. Вы также можете щелкнуть стрелку вниз, чтобы просмотреть содержимое списка воспроизведения и перейти к определенной теме.

#### <a name="take-an-inventory-of-teams-learning-resources-in-your-organization"></a>Составление перечня учебных ресурсов Teams в организации

Просмотрите учебное содержимое Teams, которое уже доступно в вашей организации. Например, вы могли уже разработать собственное содержимое подключения, обучения или поддержки для Teams. Ваши существующие ресурсы SharePoint могут быть объединены с содержимым Майкрософт в списках воспроизведения для создания целевого списка воспроизведения для организации.

#### <a name="build-your-site-in-learning-pathways"></a>Создание сайта в схемах обучения

[Центр успеха администраторов](/office365/customlearning/custom_successcenter) в схемах обучения содержит инструкции и ресурсы для планирования и настройки схем обучения в организации. Узнайте, как [настроить сайт](/office365/customlearning/custom_overview), отображать и скрывать содержимое, создавать собственные списки воспроизведения и т. д.

Чтобы открыть Центр успеха администраторов, на домашней странице схем обучения выберите **Центр успеха администраторов**.

#### <a name="add-your-site-to-teams"></a>Добавление сайта в Teams

Когда ваш сайт будет готов, добавьте его в канал любой команды для быстрого и удобного доступа.

1. В Teams перейдите в команду и выберите канал.
2. В верхней части страницы канала нажмите **+** (**Добавление вкладки**).
3. Выберите **Страницы SharePoint** и нажмите **Добавьте страницу с любого сайта SharePoint**.
4. Вставьте URL-адрес своего сайта схем обучения и нажмите **Сохранить**.

Дополнительные сведения см. в статье [Добавление страницы или списка SharePoint в канал в Teams](https://support.microsoft.com/office/add-a-sharepoint-page-or-list-to-a-channel-in-teams-131edef1-455f-4c67-a8ce-efa2ebf25f0b).

### <a name="train-your-support-team"></a>Подготовка группы технической поддержки

Используйте ресурсы на своем сайте схем обучения, чтобы подключить свою службу технической поддержки и лидеров к Teams. Подготовьте их с использованием инструментов и сведений, которые нужны им для поддержки ваших пользователей в Teams.

Инструкции и ресурсы по подготовке службы технической поддержки и лидеров к Teams см. в разделах [Подготовка организации](https://adoption.microsoft.com/microsoft-teams/#train-your-org) и [Создание лидеров](https://adoption.microsoft.com/microsoft-teams/#build-champions).

При возникновении у пользователей вопросов, первыми к кому они обращаются, — это ваша служба технической поддержки и лидеры, которые могут использовать сайт схем обучения для подготовки пользователей и в качестве альтернативы созданию запросов в службу поддержки. Рекомендуйте своей службе технической поддержки и лидерам [настроить сайт схем обучения](/office365/customlearning/) путем создания страницы обучения и поддержки, а затем [добавить его в качестве вкладки в канал](#add-your-site-to-teams) команды для самостоятельного применения пользователями.

### <a name="drive-adoption"></a>Внедрение

После настройки сайта и подготовки планов обучения рассмотрите способ ознакомления с ними ваших пользователей, чтобы подтолкнуть их к использованию схем обучения для постоянной учебы.

Используйте каналы коммуникации для продвижения сайта и уведомления о нем. Например, добавьте стандартное уведомление "Ознакомьтесь с нашим сайтом обучения и поддержки с инструкциями по повышению производительности в Teams" при общении с пользователями.

Поддерживайте пользователей путем описания способов совместной работы в Teams и направляйте их прямо на сайт схем обучения с соответствующими инструкциями.

Ознакомьтесь со следующими ресурсами, включающими инструкции, наборы внедрения, рекомендации и многое другое, для реализации плана успешного развертывания и внедрения.

## <a name="frequently-asked-questions"></a>Вопросы и ответы

### <a name="what-are-the-licensing-requirements-for-advisor-for-teams"></a>Каковы требования лицензирования к помощнику для Teams

Как минимум, у вас должен быть Microsoft 365 бизнес базовый, чтобы вы могли воспользоваться интеграцией помощника для Teams с Forms и Планировщиком.

### <a name="can-i-delete-the-deployment-team"></a>Можно ли удалить команду развертывания

После того как помощник для Teams создаст команду развертывания, этой командой можно управлять так же, как и любой другой командой, включая возможность удаления. Имейте в виду, если вы не удалите команду с помощью Центра администрирования Teams, в нем будет указано, что команда по-прежнему существует.

### <a name="can-i-add-or-remove-channels-in-the-deployment-team"></a>Можно ли добавлять и удалять каналы в команде развертывания

Да. После создания команды развертывания вы можете управлять каналами так же, как и в любой другой команде.

### <a name="can-i-add-or-remove-project-team-members-in-the-deployment-team"></a>Можно ли добавлять и удалять участников команды проекта в команде развертывания

Да. После создания команды развертывания ею можно управлять так же, как и любой другой командой.

### <a name="can-i-modify-the-planner-plans"></a>Можно ли изменить планы Планировщика

Да. После того как помощник для Teams создаст команду развертывания, нужно обновить план Планировщика, чтобы он наилучшим образом поддерживал развертывание Teams. Вы можете изменять что угодно: сегменты, задачи и сведения о задачах, — как и любые другие планы Планировщика.

### <a name="can-i-modify-the-forms-survey"></a>Можно ли изменить опрос Forms

Да. После того как помощник для Teams создаст команду развертывания, можно будет изменить опрос Forms согласно требованиям.

### <a name="are-there-any-differences-between-advisor-for-teams-in-gcc"></a>Есть ли различия между помощником для Teams в GCC

Да, формы опроса пользователей созданы, но не закреплены в каналах плана, поскольку приложение «Формы Teams» в настоящее время недоступно в GCC.

### <a name="what-information-is-advisor-for-teams-collecting-about-my-organization"></a>Какие сведения помощник для Teams собирает о моей организации

Помощник для Teams запрашивает ваше согласие на сбор данных, не являющихся личными сведениями пользователей (EUII). Осуществляется сбор данных телеметрии, что позволяет корпорации Майкрософт определить, насколько успешные результаты дает использование помощника для Teams и в каких случаях их можно улучшить. Эти же сведения используются для определения возможностей своевременного взаимодействия корпорации Майкрософт с вашей организацией с целью предоставления поддержки в развертывании.

### <a name="can-i-use-advisor-for-teams-with-fasttrack"></a>Можно ли использовать помощника для Teams вместе с FastTrack

Да. FastTrack использует помощник для Teams для всех клиентов, желающих развернуть Teams. Предоставляется помощь в первоначальной настройке команды развертывания с использованием помощника для Teams (если это необходимо), а также поддержка по определенным вопросам развертывания Teams при их возникновении.

### <a name="can-i-use-advisor-for-teams-with-a-partner"></a>Можно ли использовать помощника для Teams вместе с партнером

Да. Помощник для Teams можно использовать одновременно с партнером по развертыванию для Teams. Если ваш партнер является поставщиком облачных служб (CSP) и управляет вашим клиентом от вашего имени, он может использовать помощник для Teams, чтобы создать команду развертывания и помочь вам выполнить весь проект. Кроме того, вы можете работать с любым партнером, добавляя таких пользователей в свою команду развертывания в качестве гостей, чтобы они могли принимать участие в команде общего проекта.

### <a name="how-do-i-use-planner"></a>Как использовать Планировщик

Ознакомьтесь с разделом [Справка по Планировщику (Майкрософт)](https://support.office.com/article/Microsoft-Planner-help-4a9a13c6-3adf-4a60-a6fc-15c0b15e16fc) и посмотрите [видео с краткими инструкциями по началу работы с Планировщиком](https://support.office.com/article/microsoft-planner-video-training-4d71390f-08d8-4db0-84ea-92fb078687c7).
