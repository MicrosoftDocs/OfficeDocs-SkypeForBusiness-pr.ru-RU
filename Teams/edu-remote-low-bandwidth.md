# <a name="troubleshoot-low-bandwidth-scenarios-for-teams"></a>Устранение неполадок в сценариях с низкой пропускной способностью для Teams

В этой статье ИТ-администраторам даны рекомендации по устранению проблем с низкой пропускной способностью в Teams.

Большое количество сетевых элементов может повлиять на производительность при работе с Microsoft Teams.

- Низкоскоростное подключение к Интернету для учебного заведения.
- Низкоскоростное подключение к Интернету для одного или нескольких учащихся.
- Время суток, когда пропускная способность низка из-за использования сети в регионе.
- Сбои не являются локальными для учебного заведения или учащихся, но влияют на производительность.
- Проблемы с оборудованием, которые вызывают проблемы с низкой пропускной способностью.

> [!IMPORTANT]
> Узнайте [, как Microsoft Teams использует память](teams-memory-usage-perf.md) для ограничений ресурсов на устройствах.
>
>Руководство по сети Teams см. в разделе ["Подготовка сети организации для Microsoft Teams"](prepare-network.md).

## <a name="resolving-low-bandwidth-issues-for-it-admins"></a>Устранение проблем с низкой пропускной способностью для ИТ-администраторов

Некоторые проблемы могут быть устранены только с помощью узкого фокуса на уровне отдельных пользователей.

Если проблемы с пропускной способностью возникают у многих пользователей или действия, выполненные на уровне пользователя, не были полезны, следующим шагом является действие на уровне учебного заведения.

> [!NOTE]
> Вы также можете ознакомиться [с руководством по проверке качества взаимодействия](quality-of-experience-review-guide.md). Он не является конкретным элементом EDU, но содержит ценную информацию.

### <a name="meetings-and-video"></a>Виртуальные собрания и видео

При устранении проблем с собранием, связанных с низкой пропускной способностью сети, учитывайте приведенные ниже действия.

#### <a name="meeting-video-policies"></a>Политики видео собраний

Teams автоматически масштабирует качество собраний до обнаруженной пользователем пропускной способности. Однако можно задать дополнительные ограничения для сохранения пропускной способности.

Ниже перечислены некоторые ограничения, которые можно задать с помощью политики.

- Полностью отключите видео, чтобы никто не сможет использовать видео.
- Ограничение скорости передачи мультимедиа, которая задается для каждого пользователя.

Чтобы настроить дополнительные политики для собраний и видео, ознакомьтесь с параметрами политики собраний [в Teams: аудио и видео](meeting-policies-audio-and-video.md).

#### <a name="screen-sharing-policies"></a>Правила совместного использования экрана

В Teams пользователи могут предоставлять общий доступ ко всему экрану или отдельным окнам.

Общий доступ ко всему экрану использует больше пропускной способности, чем просто совместное использование окна.

- Запретить пользователям предоставлять общий доступ ко всему экрану с помощью политики.
- Попросите преподавателей предоставлять общий доступ только к приложениям, а не ко всему экрану.

Сведения о политиках общего доступа к экранам в [параметрах политики собраний в Teams: аудио и видео](meeting-policies-audio-and-video.md).

#### <a name="dial-in-number-for-meetings"></a>Телефонный номер для встреч

Некоторым учащимся может быть проще вламыться в сеансы аудитории.

- Укажите номер для телефонного подключения для собраний Teams в качестве альтернативы приглашению на видеосхему.

Дополнительные сведения см. в разделе ["Задание номеров телефонов, включенных в приглашения в Microsoft Teams"](set-the-phone-numbers-included-on-invites-in-teams.md).

## <a name="low-bandwidth-scenarios-as-an-educator"></a>Сценарии низкой пропускной способности для преподавателей

Наличие преподавателей для устранения проблем с пропускной способностью может быть лучшим выбором, чем ИТ-действие в следующих ситуациях:

- Проблема возникает периодически.
- Существует определенное время дня, в течение которого можно прогнозировать возникновение проблемы.

Инструкции, которые преподаватель может выполнить для устранения проблем с пропускной способностью, см. в статье "Использование Teams для работы в учебном заведении при низкой [пропускной способности"](https://support.office.com/article/use-teams-for-schoolwork-when-bandwidth-is-low-5c5675f7-1b55-471a-9daa-ec1e6df38262).

## <a name="low-bandwidth-scenarios-as-a-parent-or-student"></a>Сценарии низкой пропускной способности со стороны родителя или ученика

Иногда проблема с пропускной способностью находится на стороне учащегося.

- Домашняя папка может не иметь доступа к высокой пропускной способности.
- У них может быть много людей в непосредственной области, которые также используют пропускную способность.
- Может возникнуть нестабильное подключение к Интернету.

Мы совместим руководство по использованию [Teams](https://support.office.com/article/use-teams-for-schoolwork-when-bandwidth-is-low-5c5675f7-1b55-471a-9daa-ec1e6df38262) для работы в учебном заведении, если для родителей и учащихся низкая пропускная способность.
