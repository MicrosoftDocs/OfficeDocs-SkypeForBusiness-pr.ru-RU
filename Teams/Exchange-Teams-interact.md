# <a name="how-exchange-and-microsoft-teams-interact"></a>Взаимодействие Exchange и Microsoft Teams

> [!Tip]
> Посмотрите следующий сеанс, чтобы узнать, как Teams взаимодействует с Azure Active Directory (AAD), Группами Microsoft 365, Exchange, SharePoint и OneDrive для бизнеса: основы [Microsoft Teams](https://aka.ms/teams-foundations)

Для полноценного интерфейса Teams для каждого пользователя должна быть включена возможность создания групп Exchange Online, SharePoint Online и Microsoft 365.

Почтовые ящики пользователей Exchange могут размещаться как в сети, так и локально.

Пользователям, размещенным в Exchange Online или Exchange Dedicated vNext, доступны все функции Teams. Они могут создавать команды и каналы и присоединяться к ним, создавать и просматривать собрания, звонить и общаться, изменять изображения профилей пользователей (если это разрешает политика Outlook в веб-почтовых ящиках), а также добавлять и настраивать соединители, вкладки и боты. Более полный список доступных функций см. в таблице ниже.

Пользователи, которые работают на платформе Exchange Online Dedicated (старые), должны быть синхронизированы с Azure Active Directory в Microsoft 365 или Office 365. Они могут создавать команды и каналы и присоединяться к ним, добавлять и настраивать вкладки и боты, а также использовать возможности чата и звонков. Однако они не могут изменять изображения профилей, управлять собраниями, получать доступ к контактам Outlook и управлять соединитетелями.

> [!IMPORTANT]
> Для интеграции с локальной версией настоятельно рекомендуется использовать полное классическое гибридное развертывание Exchange с Exchange Server 2016 или более поздней версией. Поддержка современной гибридной системы ограничена ограничением занятости и не обеспечивает интеграцию календаря из Teams с почтовыми ящиками локально. Дополнительные сведения о настройке гибридного развертывания см. в Exchange Server [развертываниях.](https://docs.microsoft.com/exchange/exchange-hybrid)

Пользователи с почтовыми ящиками, которые были локально, должны быть синхронизированы с Azure Active Directory. Они могут использовать все функции, перечисленные в выше сценарии, но также управлять [](#requirements-to-create-and-view-meetings-for-mailboxes-hosted-on-premises) собраниями, если будут выполнены требования, перечисленные в разделе "Требования к почтовым ящикам,которые находятся в локальном разделе".

## <a name="requirements-to-get-the-most-out-of-microsoft-teams"></a>Требования для того, чтобы в Microsoft Teams было все необходимое

Microsoft Teams работает с несколькими службами Microsoft 365 и Office 365, чтобы предоставить пользователям богатый опыт работы. Для поддержки этой функции необходимо включить определенные функции или службы и назначить лицензии.

- Пользователям должна быть назначена лицензия на Exchange Online.

- Для хранения и совместного использования файлов в беседах команды требуется SharePoint Online. Microsoft Teams не поддерживает локальную среду SharePoint.

- Чтобы пользователи могли делиться файлами в чатах, им должна быть назначена лицензия На SharePoint Online. Если пользователям не назначены и не включены лицензии На SharePoint Online, у них нет хранилища OneDrive для бизнеса в Microsoft 365 или Office 365. Общий доступ к файлам продолжит работать в каналах, но пользователи не смогут делиться файлами в чатах без хранилища OneDrive для бизнеса в Microsoft 365 или Office 365.

- Для создания команд в Microsoft Teams у пользователей должна быть включена возможность создания групп Microsoft 365.

  > [!IMPORTANT]
  > Если удалить клиент Skype для бизнеса после перемещения пользователя в режим "Только **Teams",** присутствие может перестать работать в Outlook и других приложениях Office. Присутствие в Teams отображается нормально. Чтобы устранить эту проблему, выберите изображение своего профиля в правом верхнем углу Microsoft Teams и выберите **"Параметры".** На **вкладке "Общие"** в **приложении** выберите "Зарегистрировать Группы как приложение **чата для Office" (требуется** перезапуск приложений Office). После выбора этого параметра закроем и снова откройте все приложения Office, включая Outlook. После того как вы откроете Outlook, сведения о присутствии будут доступны.

## <a name="requirements-to-create-and-view-meetings-for-mailboxes-hosted-on-premises"></a>Требования для создания и просмотра собраний для почтовых ящиков, которые были организованы локально

Если почтовые ящики находятся локально, для создания и просмотра собраний должны быть выполнены следующие требования:

- Для синхронизированного пользователя Azure Active Directory должна быть назначена необходимая лицензия Teams.

- Пользователи должны быть синхронизированы с Azure Active Directory. Сведения об использовании Azure AD Connect для синхронизации с Azure Active Directory см. в документации [по гибридным удостоверениям.](https://docs.microsoft.com/azure/active-directory/hybrid/)

- Почтовые ящики находятся Exchange Server накопительном обновлении за 2016 г. 3 или более поздней версии.

- Служба автообнаружия и веб-службы Exchange публикуются внешними данными.

- Проверка подлинности OAuth настраивается предпочтительно с помощью мастера гибридной конфигурации Exchange с полной гибридной конфигурацией (классической или современной). Если вы не можете использовать мастер гибридной конфигурации, настройте OAuth так, как описано в описании в окне "Настройка проверки подлинности [OAuth между организациями Exchange и Exchange Online".](https://docs.microsoft.com/exchange/configure-oauth-authentication-between-exchange-and-exchange-online-organizations-exchange-2013-help)

  > [!NOTE]
  > Exchange доверяет маркеру OAuth из службы Teams, которая называется EvoSTS. Шага 1 должно быть достаточно, но только EvoSTS; AcS используется для подметок "Свободное/занятое" в календаре.

- Установленный контроль для функции гибридного развертывания Exchange в Azure AD Connect.

- Для поддержки календаря и Teams Outlook Add-In для Mac URL-адреса веб-служб Exchange должны быть настроены как SPNS в клиенте Azure AD для основного клиента службы Exchange. Это делается в мастере гибридной конфигурации или при настройке гибридной современной проверки [подлинности вручную.](https://docs.microsoft.com/microsoft-365/enterprise/configure-exchange-server-for-hybrid-modern-authentication#add-on-premises-web-service-urls-as-spns-in-azure-ad)

Чтобы включить делегирования доступа к календарю для этих пользователей:

- Необходимо также выполнить действия 2–3, описанные в настройках интеграции и [OAuth](https://docs.microsoft.com/skypeforbusiness/deploy/integrate-with-exchange-server/oauth-with-online-and-on-premises)между Skype для бизнеса Online и Exchange Server; эти действия предоставляют приложению для планирования Teams необходимые разрешения для подтверждения разрешений представителя.
 
  > [!NOTE]
  > Шаг 2 включает назначение ролей для ArchiveApplication, которое не требуется для делегирования.

- Надстройка Teams Для планирования собрания от имени другого пользователя требуется Exchange 2013 CU19 или более поздней. Это необходимо для того, чтобы служба проверила разрешения представителя для почтового ящика делегирования. Представителем и представителем может быть Exchange 2013 или более поздней или Exchange Online, но для автообнаружия необходимо разрешить доступ к Exchange 2013 с cu19 или более поздней.

## <a name="additional-considerations"></a>Дополнительные факторы

При внедрении Microsoft Teams в организации необходимо думать некоторые дополнительные вещи.

- Функции обеспечения безопасности и соответствия в Microsoft Teams, такие как обнаружение электронных данных, поиск контента, архивация и удержание по юридическим причинам, лучше всего работают в средах Exchange Online и SharePoint Online. Во время бесед в каналах сообщения передаются в почтовый ящик группы Exchange Online, где они доступны для обнаружения электронных данных. Если пользователи организации имеют доступ к SharePoint Online и OneDrive для бизнеса (с помощью учебной или рабочей учетной записи), эти функции по обеспечению соответствия также доступны для всех файлах в Teams.

- Контроль и защита конфигурации политик соответствия требованиям в Teams и Exchange с помощью условного доступа. Дополнительные сведения см. [в том, как](security-compliance-overview.md#how-conditional-access-policies-work-for-teams) работают политики условного доступа для Teams?

- Если в вашей организации есть требования к обеспечению соответствия требованиям для обнаружения всех обсуждений собрания, отключите частные собрания, если у организатора есть почтовый ящик Exchange. Дополнительные сведения см. в [теме "Разрешение планирования частных собраний".](https://docs.microsoft.com/microsoftteams/meeting-policies-in-teams#allow-scheduling-private-meetings)

- В гибридном развертывании Exchange содержимое сообщений чата можно искать независимо от того, есть ли у участников облачного или локального почтового ящика. Для получения дополнительных данных ознакомьтесь с поиском в облачных почтовых ящиках для пользователей [локальной платформы.](https://docs.microsoft.com/office365/securitycompliance/search-cloud-based-mailboxes-for-on-premises-users) Чтобы узнать о поиске контента в Teams, ознакомьтесь с поиском контента в Центре соответствия требованиям [Microsoft 365.](https://docs.microsoft.com/Office365/SecurityCompliance/content-search#searching-microsoft-teams-and-office-365-groups)

- Для состояния присутствия Microsoft Teams должен проверить, находится ли почтовый ящик в Exchange Online или локально. Затем служба решает, куда нужно получить доступ к почтовому ящику. Чтобы включить службу Teams для проверки расположения почтового ящика с помощью звонка REST API в службу Exchange Online, [](https://docs.microsoft.com/exchange/hybrid-deployment/deploy-hybrid)необходимо развернуть гибридную среду Exchange с помощью мастера гибридной конфигурации Exchange, как описано в описании мастера гибридной конфигурации.
