## <a name="call-routing-overview"></a>Общие сведения о маршрутизации вызовов

В телефонной системе Майкрософт есть механизм маршрутизации, который позволяет отправлять вызов определенному пограничному контроллеру сеанса (SBC) на основе следующих параметров: 

- Шаблон вызываемого числа 
- Шаблон вызываемого номера плюс конкретный пользователь, который выполняет вызов
 
Контроллеры SBC можно назначить активными и резервными копиями. Если SBC, настроенный как активный, недоступен для определенного маршрута вызова, вызов будет перенаправлен в резервный SBC.
 
Маршрутизация вызовов состоит из следующих элементов: 

- **Политика маршрутизации вызовов —** также называется политикой маршрутизации голосовой связи. Контейнер для использования ТСОП, который можно назначить пользователю или нескольким пользователям. 

- **Использование ТСОП** — контейнер для голосовых маршрутов и использования ТСОП, который можно совместно использовать в разных политиках маршрутизации голосовой связи. 

- **Маршруты голосовой** связи — шаблон номера и набор сетевых шлюзов ТСОП, которые будут использовать для звонков, где вызывающий номер соответствует шаблону.

- Сетевой шлюз **ТСОП** — указатель на SBC, в котором также хранится конфигурация, которая применяется при вызове через SBC, например пересылка P-Asserted-Identity (PAI) или preferred Codecs; можно добавить в голосовые маршруты.

## <a name="voice-routing-policy-considerations"></a>Рекомендации по политике маршрутизации голосовой связи

Если у пользователя есть лицензия на план звонков, исходящие звонки этого пользователя автоматически направляются через инфраструктуру ТСОП плана звонков Майкрософт. Если вы настроите и назначите политику маршрутизации голосовой связи по сети пользователю плана звонков, исходящие звонки этого пользователя проверяются, чтобы определить, соответствует ли номер шаблону номера, определенному в политике маршрутизации голосовой связи в Сети. Если есть совпадение, вызов направляется через магистраль прямой маршрутизации. Если совпадений нет, вызов направляется через инфраструктуру ТСОП плана звонков.

> [!CAUTION]
> Если вы настроите и примените глобальную политику сетевой маршрутизации голосовой связи (по умолчанию для всей организации), все пользователи с поддержкой голосовой связи в вашей организации наследуют эту политику, что может привести к случайному перенаправлению звонков из плана звонков и оператора Connect в магистраль прямой маршрутизации. Если вы не хотите, чтобы все пользователи могли использовать глобальную политику маршрутизации голосовой связи в Интернете, настройте настраиваемую политику маршрутизации голосовой связи в Интернете и назначьте ее отдельным пользователям с поддержкой голосовой связи.

## <a name="example-1-voice-routing-with-one-pstn-usage"></a>Пример 1. Маршрутизация голосовой связи с использованием одного ТСОП

На следующей схеме показаны два примера политик маршрутизации голосовой связи в потоке вызовов.

**Поток вызовов 1 (слева):** Если пользователь вызывает +1 425 XXX XX XX или +1 206 XXX XX XX, вызов направляется в SBC sbc1.contoso.biz или sbc2.contoso.biz. Если ни sbc1.contoso.biz, ни sbc2.contoso.biz недоступны, вызов удаляется. 

**Поток вызовов 2 (справа):** Если пользователь вызывает +1 425 XXX XX XX или +1 206 XXX XX XX, вызов сначала направляется в SBC sbc1.contoso.biz или sbc2.contoso.biz. Если ни один из SBC не доступен, будет предпринята попытка маршрутов с более низким приоритетом (sbc3.contoso.biz и sbc4.contoso.biz). Если ни один из контроллеров SBC не доступен, вызов удаляется. 

![Примеры политики маршрутизации голосовой связи.](media/ConfigDirectRouting-VoiceRoutingPolicyExamples.png)

В обоих примерах, когда маршруту голосовой связи назначены приоритеты, SBCs в маршрутах выполняются в случайном порядке.

  > [!NOTE]
  > Если у пользователя нет лицензии плана звонков Майкрософт, вызовы любых номеров, кроме номеров, соответствующих шаблонам +1 425 XXX XX XX или +1 206 XXX XX XX XX в примере конфигурации, удаляются. Если у пользователя есть лицензия на план звонков, вызов автоматически направляется в соответствии с политиками плана звонков Майкрософт. План звонков Майкрософт применяется автоматически как последний маршрут для всех пользователей с лицензией плана звонков Майкрософт и не требует дополнительной конфигурации маршрутизации звонков.

В примере, показанном на следующей схеме, добавляется голосовой маршрут для отправки вызовов на все остальные номера США и Канады (вызовы, которые отправляются в шаблон номера +1 XXX XXX XX XX XX).

![Показывает политику маршрутизации голосовой связи с третьим маршрутом.](media/ConfigDirectRouting-VoiceRoutingPolicywith3rdroute.png)

Для всех остальных звонков, если у пользователя есть обе лицензии (телефонная система Майкрософт и план звонков Майкрософт), используется автоматический маршрут. Если ничего не соответствует шаблонам номеров в сетевых голосовых маршрутах, созданных администратором, вызов направляется через план звонков Майкрософт. Если у пользователя есть только телефонная система Майкрософт, вызов удаляется, так как отсутствуют соответствующие правила.

  > [!NOTE]
  > Значение приоритета для маршрута "Other +1" в данном случае не имеет значения, так как существует только один маршрут, соответствующий шаблону +1 XXX XXX XX XX XX. Если пользователь вызывает +1 324 567 89 89 и sbc5.contoso.biz и sbc6.contoso.biz недоступны, вызов удаляется.

### <a name="using-powershell"></a>С помощью PowerShell
<a name="powershellexample1"></a>


#### <a name="step-1-create-the-us-and-canada-pstn-usage"></a>Шаг 1. Создание ТСОП "США и Канада"

В удаленном сеансе PowerShell в Skype для бизнеса Online введите:

```PowerShell
Set-CsOnlinePstnUsage -Identity Global -Usage @{Add="US and Canada"}
```

Убедитесь, что использование создано, введя:

```PowerShell
Get-CSOnlinePSTNUsage
``` 

Возвращает список имен, которые могут быть усечены:

```console
Identity    : Global
Usage        : {testusage, US and Canada, International, karlUsage. . .}
```

В следующем примере показан результат выполнения `(Get-CSOnlinePSTNUsage).usage` команды PowerShell для отображения полных имен (без усечения):

```console
 testusage
 US and Canada
 International
 karlUsage
 New test env
 Tallinn Lab Sonus
 karlUsage2
 Unrestricted
 Two trunks
```

#### <a name="step-2-create-three-voice-routes-redmond-1-redmond-2-and-other-1"></a>Шаг 2. Создание трех голосовых маршрутов (Redmond 1, Redmond 2 и Other +1)

Чтобы создать маршрут Redmond 1, в сеансе PowerShell в Skype для бизнеса Online введите:

```PowerShell
New-CsOnlineVoiceRoute -Identity "Redmond 1" -NumberPattern "^\+1(425|206)
(\d{7})$" -OnlinePstnGatewayList sbc1.contoso.biz, sbc2.contoso.biz -Priority 1 -OnlinePstnUsages "US and Canada"
```

Возвращает следующее:

```console
Identity                : Redmond 1
Priority                : 1
Description             :
NumberPattern           : ^\+1(425|206) (\d{7})$
OnlinePstnUsages        : {US and Canada}
OnlinePstnGatewayList   : {sbc1.contoso.biz, sbc2.contoso.biz}
Name                    : Redmond 1
```

Чтобы создать маршрут Redmond 2, введите:

```PowerShell
New-CsOnlineVoiceRoute -Identity "Redmond 2" -NumberPattern "^\+1(425|206)
(\d{7})$" -OnlinePstnGatewayList sbc3.contoso.biz, sbc4.contoso.biz -Priority 2 -OnlinePstnUsages "US and Canada"
```

Чтобы создать другой маршрут +1, введите:

```PowerShell
New-CsOnlineVoiceRoute -Identity "Other +1" -NumberPattern "^\+1(\d{10})$"
-OnlinePstnGatewayList sbc5.contoso.biz, sbc6.contoso.biz -OnlinePstnUsages "US and Canada"
```

  > [!CAUTION]
  > Убедитесь, что регулярное выражение в атрибуте NumberPattern является допустимым выражением. Его можно протестировать с помощью этого веб-сайта: [https://www.regexpal.com](https://www.regexpal.com)

В некоторых случаях необходимо перенаправить все вызовы в один и тот же SBC. use -NumberPattern ".*"

Перенаправите все вызовы к одному ИТ-серверу.

```PowerShell
Set-CsOnlineVoiceRoute -id "Redmond 1" -NumberPattern ".*" -OnlinePstnGatewayList sbc1.contoso.biz
```

Убедитесь, что маршрут `Get-CSOnlineVoiceRoute` настроен правильно, выполнив команду PowerShell с помощью параметров, как показано ниже.

```PowerShell
Get-CsOnlineVoiceRoute | Where-Object {($_.priority -eq 1) -or ($_.priority -eq 2) or ($_.priority -eq 4) -Identity "Redmond 1" -NumberPattern "^\+1(425|206) (\d{7})$" -OnlinePstnGatewayList sbc1.contoso.biz, sbc2.contoso.biz -Priority 1 -OnlinePstnUsages "US and Canada"
```
Что должно возвращаться:

```console
Identity            : Redmond 1 
Priority               : 1
Description         : 
NumberPattern         : ^\+1(425|206) (\d{7})$
OnlinePstnUsages     : {US and Canada}     
OnlinePstnGatewayList    : {sbc1.contoso.biz, sbc2.contoso.biz}
Name             : Redmond 1
Identity        : Redmond 2 
Priority               : 2
Description         : 
NumberPattern         : ^\+1(425|206) (\d{7})$
OnlinePstnUsages     : {US and Canada}     
OnlinePstnGatewayList    : {sbc3.contoso.biz, sbc4.contoso.biz}
Name             : Redmond 2
    
Identity        : Other +1 
Priority               : 4
Description         : 
NumberPattern         : ^\+1(\d{10})$
OnlinePstnUsages     : {US and Canada}     
OnlinePstnGatewayList    : {sbc5.contoso.biz, sbc6.contoso.biz}
Name             : Other +1
```

В этом примере маршруту "Other +1" был автоматически назначен приоритет 4. 

#### <a name="step-3-create-a-voice-routing-policy-named-us-only-and-add-the-us-and-canada-pstn-usage-to-the-policy"></a>Шаг 3. Создайте политику маршрутизации голосовой связи с именем "Только ДЛЯ США" и добавьте в нее использование ТСОП "США и Канада"

В сеансе PowerShell в Skype для бизнеса Online введите:

```PowerShell
New-CsOnlineVoiceRoutingPolicy "US Only" -OnlinePstnUsages "US and Canada"
```

В этом примере показан результат:

```console
Identity            : Tag:US only
OnlinePstnUsages    : {US and Canada}
Description         :
RouteType           : BYOT
```

#### <a name="step-4-assign-the-voice-routing-policy-to-a-user-named-spencer-low"></a>Шаг 4. Назначение политики маршрутизации голосовой связи пользователю с именем Low

В сеансе PowerShell в Skype для бизнеса Online введите:

```PowerShell
Grant-CsOnlineVoiceRoutingPolicy -Identity "Spencer Low" -PolicyName "US Only"
```

Проверьте назначение политики, введя следующую команду:

```PowerShell
Get-CsOnlineUser "Spencer Low" | select OnlineVoiceRoutingPolicy
```

Команда возвращает следующее:

```console
OnlineVoiceRoutingPolicy
---------------------
US Only
```

## <a name="example-2-voice-routing-with-multiple-pstn-usages"></a>Пример 2. Маршрутизация голосовой связи с использованием нескольких ТСОП

Политика маршрутизации голосовых вызовов, созданная в примере 1, разрешает звонки только на номера телефонов в США и Канаде, если пользователю не назначена лицензия плана звонков Майкрософт.

В следующем примере можно создать политику маршрутизации голосовой связи "Без ограничений". Политика повторно использует использование ТСОП "США и Канада", созданное в примере 1, а также новое "международное" использование ТСОП. Эта политика направляет все остальные вызовы к контроллерам sbc2.contoso.biz и sbc5.contoso.biz.

В приведенных примерах политика "Только ДЛЯ США" назначается пользователю Суэлу Ниу и политике "Нет ограничений" пользователю Джону Джону ,чтобы маршрутизация велись следующим образом:

- Low (Сша) — политика только для США.  Звонки разрешены только на номера США и Канады. При вызове диапазона номеров Redmond необходимо использовать определенный набор контроллеров SBC. Номера, отличные от США, не будут маршрутизироваться, если пользователю не назначена лицензия плана звонков.

- Джон Куа (John Ivan), международный политик.  Вызовы разрешены на любой номер. При вызове диапазона номеров Redmond необходимо использовать определенный набор контроллеров SBC. Номера, отличные от США, будут маршрутизироваться с sbc2.contoso.biz и sbc5.contoso.biz.

![Показывает политику маршрутизации голосовой связи, назначенную пользователю Low.](media/ConfigDirectRouting-VoiceRoutingPolicyAssignedtoSpencerLow.png)

Для всех остальных звонков, если у пользователя есть обе лицензии (телефонная система Майкрософт и план звонков Майкрософт), используется автоматический маршрут. Если ничего не соответствует шаблонам номеров в сетевых голосовых маршрутах, созданных администратором, вызов направляется с помощью плана звонков Майкрософт.  Если у пользователя есть только телефонная система Майкрософт, вызов удаляется, так как отсутствуют соответствующие правила.

![Отображает политику маршрутизации голосовых вызовов, назначенную пользователю John Ivan.](media/ConfigDirectRouting-VoiceRoutingPolicyAssignedtoJohnWoods.png)

В следующей таблице перечислены определения использования политики маршрутизации "Без ограничений" и маршруты голосовой связи. 

| Использование ТСОП | Маршрут голосовой связи | Шаблон номера | Priority (Приоритет) | Sbc | Описание |
|:-----|:-----|:-----|:-----|:-----|:-----|
|США и Канада|"Redmond 1"|^\\+1(425\|206)(\d{7})$|1|sbc1.contoso.biz<br/>sbc2.contoso.biz|Активный маршрут для вызываемых номеров +1 425 XXX XX XX или +1 206 XXX XX XX|
|США и Канада|"Redmond 2"|^\\+1(425\|206)(\d{7})$|2|sbc3.contoso.biz<br/>sbc4.contoso.biz|Маршрут резервного копирования для вызываемых номеров +1 425 XXX XX XX или +1 206 XXX XX XX|
|США и Канада|"Другое +1"|^\\+1(\d{10})$|3|sbc5.contoso.biz<br/>sbc6.contoso.biz|Маршрут для вызываемых номеров +1 XXX XXX XX XX (за исключением +1 425 XXX XX XX или +1 206 XXX XX XX)|
|International|International|\d+|4|sbc2.contoso.biz<br/>sbc5.contoso.biz|Маршрут для любого числового шаблона |

  > [!NOTE]
  > - Очень важен порядок использования ТСОП в политиках маршрутизации голосовой связи. Использование применяется по порядку, и если совпадение найдено в первом использовании, то другие данные об использовании никогда не оцениваются. "Международное" использование ТСОП должно быть размещено после использования ТСОП "США и Канада". Чтобы изменить порядок использования ТСОП, выполните команду `Set-CSOnlineVoiceRoutingPolicy` . <br/>Например, чтобы изменить порядок с "США и Канада" сначала и "Международный" на обратный порядок выполнения:<br/> `Set-CsOnlineVoiceRoutingPolicy -id tag:"no Restrictions" -OnlinePstnUsages @{Replace="International", "US and Canada"}`
 > - Приоритет для голосовых маршрутов "Другие +1" и "Международные" назначается автоматически. Они не имеют значения, если имеют более низкие приоритеты, чем Redmond 1 и Redmond 2.

## <a name="example-2-configuration-steps"></a>Пример 2. Этапы настройки

В следующем примере показано, как:

1. Создайте новое использование ТСОП с именем International.
2. Создайте маршрут голосовой связи с именем International.
3. Создайте политику маршрутизации голосовой связи с именем "Нет ограничений".
4. Назначьте политику пользователю Джону Джону.

Для выполнения этих [действий можно использовать](#admincenterexample2) Центр администрирования Microsoft Teams или [PowerShell](#powershellexample2) .

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams
<a name="admincenterexample2"></a>

#### <a name="step-1-create-the-international-pstn-usage"></a>Шаг 1. Создание "международного" использования ТСОП

1. В левой области навигации Центра администрирования Microsoft Teams перейдите к маршрутизации **Voice** > **Direct**, а затем в правом верхнем углу выберите "Управление записями использования **ТСОП"**.
2. Нажмите **кнопку "Добавить**", введите **"Международный**", а затем нажмите кнопку " **Применить"**.

#### <a name="step-2-create-the-international-voice-route"></a>Шаг 2. Создание "международного" голосового маршрута

1. В левой области навигации Центра администрирования Microsoft Teams перейдите к **маршрутизации Voice** > **Direct**, а затем выберите вкладку **"Маршруты голосовой связи** ".
2. Нажмите **кнопку "** Добавить", введите "International" в качестве имени и добавьте описание.
3. Задайте для приоритета значение 4, а затем задайте для шаблона набора номеров значение \d+.
4. В **разделе "Зарегистрированные контроллеры SBC**( необязательно)" щелкните "Добавить **SBCs**", выберите sbc2.contoso.biz и sbc5.contoso.biz, а затем нажмите кнопку "Применить **"**.
5. В **разделе записей** об использовании ТСОП (необязательно) щелкните "Добавить использование ТСОП **", выберите** запись об использовании ТСОП "Международный" и нажмите кнопку "Применить **"**.
6. Нажмите кнопку **Сохранить**.

#### <a name="step-3-create-a-voice-routing-policy-named-no-restrictions-and-add-the-us-and-canada-and-international-pstn-usages-to-the-policy"></a>Шаг 3. Создание политики маршрутизации голосовой связи с именем "Без ограничений" и добавление в нее ТСОП "США и Канада" и "Международный" ТСОП

Использование ТСОП "США и Канада" повторно используется в этой политике маршрутизации голосовых вызовов, чтобы сохранить специальную обработку для вызовов номеров "+1 425 XXX XX XX" и "+1 206 XXX XX XX" в качестве локальных или локальных звонков.

1. В левой области навигации Центра администрирования Microsoft Teams перейдите  >  к политикам маршрутизации голосовой **связи и нажмите** кнопку **"Добавить"**.
2. **Введите "Нет ограничений**" в качестве имени и добавьте описание.
3. В **разделе** "Записи об использовании ТСОП" щелкните "Добавить использование ТСОП **", выберите** запись об использовании ТСОП "США и Канада", а затем выберите запись об использовании ТСОП "Международный". Нажмите кнопку **Применить**.

    Запишите порядок использования ТСОП:

    - Если вызов с номером "+1 425 XXX XX XX" с настроенным использованием, как в этом примере, вызов следует за набором маршрутов в использовании "США и Канада", и применяется специальная логика маршрутизации. То есть вызов сначала направляется sbc1.contoso.biz sbc2.contoso.biz, а sbc3.contoso.biz и sbc4.contoso.biz в качестве маршрутов резервного копирования.

    - Если "Международный" ТСОП используется до "США и Канада", вызовы +1 425 XXX XX XX направляются в sbc2.contoso.biz и sbc5.contoso.biz как часть логики маршрутизации.

4. Нажмите кнопку **Сохранить**.

Дополнительные сведения см. в [статье "Управление политиками маршрутизации голосовой связи"](manage-voice-routing-policies.md).

#### <a name="step-4-assign-the-voice-routing-policy-to-a-user-named-john-woods"></a>Шаг 4. Назначение политики маршрутизации голосовой связи пользователю с именем John Ivan

1. В Центре администрирования Microsoft Teams в области навигации слева перейдите в раздел **Пользователи**, затем щелкните пользователя.
2. **Щелкните "Политики**", а затем рядом с назначенными **политиками** нажмите кнопку "Изменить **"**.
3. В **разделе "Политика маршрутизации голосовой** связи" выберите политику "Нет ограничений" и нажмите кнопку " **Сохранить"**.

В результате политика голосовой связи, применяемая к звонкам Ивана Иванова, является неограниченной и будет соответствовать логике маршрутизации звонков, доступной для звонков из США, Канады и международного звонков.

### <a name="using-powershell"></a>С помощью PowerShell
<a name="powershellexample2"></a>

#### <a name="step-1-create-the-international-pstn-usage"></a>Шаг 1. Создание "международного" использования ТСОП

В удаленном сеансе PowerShell в Skype для бизнеса Online введите:

```PowerShell
Set-CsOnlinePstnUsage -Identity Global -Usage @{Add="International"}
```

#### <a name="step-2--create-a-new-voice-route-named-international"></a>Шаг 2. Создание маршрута голосовой связи с именем International

```PowerShell
New-CsOnlineVoiceRoute -Identity "International" -NumberPattern ".*" -OnlinePstnGatewayList sbc2.contoso.biz, sbc5.contoso.biz -OnlinePstnUsages "International"
```

Возвращает следующее:

```console
Identity                  : International
Priority                  : 5
Description               :
NumberPattern             : .*
OnlinePstnUsages          : {International}
OnlinePstnGatewayList     : {sbc2.contoso.biz, sbc5.contoso.biz}
Name                      : International
```

#### <a name="step-3-create-a-voice-routing-policy-named-no-restrictions"></a>Шаг 3. Создание политики маршрутизации голосовой связи с именем "Без ограничений"

Использование ТСОП "Redmond 1" и "Redmond" повторно используется в этой политике маршрутизации голосовых вызовов, чтобы сохранить специальную обработку для вызовов номеров "+1 425 XXX XX XX" и "+1 206 XXX XX XX XX" в качестве локальных или локальных звонков.

  ```PowerShell
  New-CsOnlineVoiceRoutingPolicy "No Restrictions" -OnlinePstnUsages "US and Canada", "International"
  ```

Запишите порядок использования ТСОП:

  - Если вызов с номером "+1 425 XXX XX XX" с настроенным использованием, как в следующем примере, вызов следует за набором маршрутов в использовании "США и Канада", и применяется специальная логика маршрутизации. То есть вызов сначала направляется sbc1.contoso.biz sbc2.contoso.biz, а sbc3.contoso.biz и sbc4.contoso.biz в качестве маршрутов резервного копирования.

  - Если "Международный" ТСОП используется до "США и Канада", вызовы +1 425 XXX XX XX направляются в sbc2.contoso.biz и sbc5.contoso.biz как часть логики маршрутизации. Введите команду:

  ```PowerShell
  New-CsOnlineVoiceRoutingPolicy "No Restrictions" -OnlinePstnUsages "US and Canada", "International"
  ```

Возвращает следующее:

```console
    Identity              : International 
    OnlinePstnUsages : {US and Canada, International}     
    Description         :  
    RouteType               : BYOT
```

#### <a name="step-4-assign-the-voice-routing-policy-to-the-user-named-john-woods"></a>Шаг 4. Назначение политики маршрутизации голосовой связи пользователю с именем John Ivan

```PowerShell
Grant-CsOnlineVoiceRoutingPolicy -Identity "John Woods" -PolicyName "No Restrictions"
```

Затем проверьте назначение с помощью команды: 

```PowerShell
Get-CsOnlineUser "John Woods" | Select OnlineVoiceRoutingPolicy
```

Возвращает следующее:

```console
OnlineVoiceRoutingPolicy
------------------------
No Restrictions
```

В результате политика голосовой связи, применяемая к звонкам Ивана Иванова, является неограниченной и будет соответствовать логике маршрутизации звонков, доступной для звонков в США, Канаде и международных вызовах.
