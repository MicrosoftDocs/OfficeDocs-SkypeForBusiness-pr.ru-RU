# <a name="set-up-a-cloud-auto-attendant"></a>Настройка облачного автосекретаря

С помощью автосекретарей пользователи могут позвонить в вашу организацию и переходить в систему меню для общения с нужным Отделом, очередью звонков, лицом или оператором. Вы можете создавать автоматические ассистенты в Организации с помощью центра администрирования Microsoft Teams или с помощью PowerShell. Чтобы создать автосекретарь, на панели навигации слева выберите пункт **Голосовая связь** , а затем — **Auto attendants**  >  **Добавить новые**ассистенты.

Если вы хотите узнать больше об автосекретарях, посмотрите, [что такое автосекретарь облака?](/microsoftteams/what-are-phone-system-auto-attendants)

> [!NOTE]
> Эта статья относится как в Microsoft Teams, так и в Skype для бизнеса Online.

Номера телефонов не назначаются автосекретаря напрямую, а также [учетной записи ресурса](manage-resource-accounts.md) , связанной с автосекретарем.

В реализации автосекретаря часто задействованы несколько автосекретарей. Автоматический секретарь *первого уровня* обычно имеет учетную запись ресурса с назначенным номером телефона. Вложенный автосекретарь используется как меню второго уровня, которое подключается автосекретарем *первого уровня* . Для учетной записи ресурса *вложенному* автосекретарь не требуется присвоение номера телефона.

## <a name="step-1--get-started"></a>Этап 1 — Начало работы

Для связи с учетной записью ресурса необходимо наличие автосекретаря. Дополнительные сведения об учетных записях ресурсов и всех необходимых лицензиях смотрите [в разделе Управление учетными записями ресурсов в Teams](manage-resource-accounts.md) .

##### <a name="call-flow-for-after-hours"></a>Поток звонков за часы

По умолчанию для рабочего времени автосекретаря задано значение 9am-5pm, понедельник — пятница  <!--24/7-->и параметры потока звонков для вызовов *после часа* отключаются, так как все часы рассматриваются как *Рабочие часы*.
