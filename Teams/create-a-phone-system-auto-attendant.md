# <a name="set-up-an-auto-attendant"></a>Настройка автозавода

Автозаверяющие могут звонить в вашу организацию и переходить в систему меню для связи с нужным отделом, очередью вызовов, человеком или оператором. Вы можете создавать автозаводы для своей организации в Центре администрирования Microsoft Teams или с помощью PowerShell.

Прежде чем выполнять действия, которые вы выполните в [](plan-auto-attendant-call-queue.md#getting-started) этой статье, убедитесь в том, что вы прочитали "План" для автоотетаря [Teams](plan-auto-attendant-call-queue.md) и очередей вызовов, а затем следуйте этим шагам.

Автоответчики могут перенаправять звонки в одно из следующих мест на основе данных, вводимых вызывающими вызовы: <a name="call-routing-options" ></a>

- **Оператор** — оператор, задавающий для автозавода. Определение оператора не является обязательным. Оператор можно определить как любой другой пункт назначения в этом списке.
- **Человек в организации —** это человек в вашей организации, который может принимать голосовые звонки. Это может быть пользователь в Сети или пользователь, который использует сервер Skype для бизнеса Server, который является пользователем локальной сети.
- **Приложение голосовой** почты — другой автозавод или очередь вызовов. (Выберите учетную запись ресурса, связанную с автостраницой или очередью вызовов, при выборе этого назначения.)
- **Голосовая почта** — это почтовый ящик голосовой почты, связанный с группой Microsoft 365, которую вы указали.
- **Внешний номер телефона** — любой номер телефона. (См. [технические сведения о внешнем передаче).](create-a-phone-system-auto-attendant.md#external-phone-number-transfers---technical-details)
- **Объявление** — воспроизведения звукового файла. Добавленное вами сообщение с объявлением, сохраненное как звуковое. WAV, . MP3 или . Формат WMA. Размер записи не должен быть больше 5 МБ. Система воспроизводит объявление, а затем возвращается в меню автозапуска.
- **Объявление.** Введите сообщение. Текст, который должна читать система. Можно ввести до 1000 символов. Система воспроизводит объявление, а затем возвращается в меню автозапуска.

Вам будет предложено выбрать один из этих параметров на разных этапах настройки автозаказа.

Чтобы настроить автостраницу, в Центре администрирования Teams развите список **"Голосовая** команда", выберите "Автозапись" и выберите **"Добавить".**

## <a name="general-info"></a>Общие сведения

![Снимок экрана: параметры автозавода для имени, оператора, часового пояса, языка и голосового ввода](media/auto-attendant-general-info-page-new.png)

1. Введите имя автозавода в поле в верхней части окна.

2. Если вы хотите назначить оператор, укажите место назначения для звонков оператору. Это необязательно (но рекомендуется). Параметр **"Оператор"** позволяет вызывателям выйти из меню и поговорить с назначенным человеком.

3. Укажите часовой пояс для этого автозавода. Часовой пояс используется для расчета рабочих часов, если вы создаете отдельный поток [зовов для нечасов.](#call-flow-for-after-hours)

4. Укажите [поддерживаемый язык](create-a-phone-system-auto-attendant-languages.md) для этого автозавода. Этот язык будет использоваться для системных голосовых подсказок.

5. Выберите, нужно ли включить голосовой ввод. Если этот параметр включен, название каждого параметра меню становится ключевым словом распознавания речи. Например, можно сказать "Один" для выбора параметра меню, назначенного клавише 1, или "Продажи", чтобы выбрать пункт меню "Продажи".

> [!NOTE]
> Если на шаге 4 вы выберете язык, который не поддерживает голосовой ввод, этот параметр будет отключен.

6. Выберите **"Далее".**

## <a name="call-flow"></a>Поток вызовов

![Снимок экрана: параметры приветствия сообщения](media/auto-attendant-call-flow-greeting-message.png)

Выберите, хотите ли вы воспроизведения приветствия, когда автоответ отвечает на звонок.

Выбрав **"Воспроизведения звукового** файла",  вы можете добавить записанное приветствие, сохраненное как звуковое сообщение, с помощью кнопки "Отправить файл". WAV, . MP3 или . Формат WMA. Размер записи не должен быть больше 5 МБ.

Если вы **выберете** "Введите приветствие", система прочитает текст, который вы введите (до 1000 символов), когда автостраница отвечает на звонок.

![Снимок экрана: параметры маршрутки вызовов](media/auto-attendant-call-flow-route-call-message.png)

Выберите, как вы хотите перена маршрутизовыть звонок.

Если вы выберете **"Отключить",** автоотетарь повесит звонок.

При выборе пункта **"Перенаправление** звонка" можно выбрать одно из назначений маршрутиации.

Если **выбраны** параметры меню "Воспроизведения", можно  выбрать "Воспроизведения звукового файла" или "Ввести приветствие", а затем выбрать параметры меню и поиск по каталогу. 

### <a name="menu-options"></a>Параметры меню

![Снимок экрана: параметры набора номера](media/auto-attendant-call-flow-menu-options-complete.png)

Для настройки параметров набора можно назначить 0–9 клавиш на телефонной клавиатуре одному из назначений маршрутки. (Клавиши \* (Повтор) и (Назад) зарезервированы системой и не \# могут быть перенаписаны.)

Сопоставление ключей не должно быть непрерывным. Например, можно создать меню с клавишами 0, 1 и 3, относясь к параметрам, хотя они не используются.

Если вы его настроили, мы рекомендуем соендуть с оператором ноль. Если оператор не назначен никакой клавише, голосовая команда "Оператор" также отключена.

Для каждого параметра меню укажите следующие параметры:

- **Клавиша набора** номера — клавиша на телефонной клавиатуре для доступа к этому параметру. Если голосовой ввод доступен, вызываемая звонок также может говорить на этот номер, чтобы получить доступ к параметру.

- **Голосовая** команда — определяет голосовую команду, которую может предоставить вызываемая команда для доступа к этому параметру, если включен голосовой ввод. Оно может содержать несколько слов, например "Обслуживание клиентов" или "Действия и причины". Например, звонок может нажать 2, сказать "два" или сказать "Продажи", чтобы выбрать параметр, выбранный с двумя клавишами. Этот текст также отрисовывается текстом в речь при запросе на подтверждение службы, например", например "Перена передача звонка в отдел продаж".

- **Redirect to** - the call routing destination used when callers choose this option. Если вы перенаправляете звонок к автоотправлению или очереди вызовов, выберите связанную с ним учетную запись ресурса.

### <a name="directory-search"></a>Поиск по каталогу

При назначении клавиш набора номера назначениям рекомендуется выбрать вариант **"Нет"** для поиска **по каталогу.** Если звоняющий пытается набрать имя или расширение с помощью клавиш, которые назначены определенным назначениям, он может быть неожиданно перенаправлен в назначение до того, как закончит вводить имя или расширение. Рекомендуется создать отдельный автозавод для поиска по каталогу и создать ссылку на него с помощью клавиши набора номера.

Если вы не назначили наберите клавиши, выберите параметр поиска **по каталогу.**

**Звонок по имени.** Если этот параметр заданной опцией, звоня люди, звонят по имени пользователя или введите его на телефонной клавиатуре. Любой пользователь в Сети или любой пользователь, использующий локальное приложение Skype для бизнеса Server, является подходящим пользователем, и его можно найти с помощью набора номера по имени. (На странице "Область набора номера" можно установить, кто является и не включен в [каталог.)](#dial-scope)

**Подключаться по расширению—** если этот параметр не включить, пользователи смогут связываться с пользователями в организации, набрав их расширения для телефона. Любой пользователь в Сети или любой пользователь, который находится локально с помощью Skype для бизнеса Server, является подходящим пользователем, и его можно найти с помощью телефонного **набора по расширению.** (На странице "Область набора номера" можно установить, кто является и не включен в [каталог.)](#dial-scope)

Пользователи, доступ к которые вы хотите сделать доступными для расширения "Набор по", должны иметь расширение, указанное как [](https://docs.microsoft.com/microsoft-365/admin/add-users/add-users) часть одного из следующих атрибутов телефонов, определенных в Active Directory или Azure Active Directory (дополнительные сведения см. в дополнительных сведениях о добавлении пользователей по одному или массово).)

- OfficePhone
- HomePhone
- Mobile/MobilePhone
- TelephoneNumber/PhoneNumber
- OtherTelephone

Необходимый формат для ввода расширения в поле номера телефона пользователя:

- *+\<phone number>;ext=\<extension>*
- *+\<phone number>x\<extension>*
- *x\<extension>*

- Пример 1. Set-MsolUser -UserPrincipalName usern@domain.com -Phonenumber "+1555555678;ext=5678"
- Пример 2. Set-MsolUser -UserPrincipalName usern@domain.com -Phonenumber "+1555555678x5678"
- Пример 3. Set-MsolUser -UserPrincipalName usern@domain.com -Phonenumber "x5678"

Расширение можно установить в Центре администрирования [Microsoft 365](https://admin.microsoft.com/) или [Центре администрирования Azure Active Directory.](https://aad.portal.azure.com) Для того чтобы изменения были доступны автоотводам и очередям вызовов, может занять до 12 часов.

> [!NOTE]
> Если вы хотите использовать  функции  "Набрать по имени" и "Набрать по расширению", назначьте ему клавишу набора для основного автозавода, чтобы связаться с автозаводом, включенным для функции "Набрать по **имени".** Вы можете назначить клавишу 1 (не связанную с ней буквами), чтобы связаться с автозаводом по набору. 

Выбрав параметр поиска по **каталогу, выберите** "Далее". 

## <a name="call-flow-for-after-hours"></a>Поток вызовов в нечасовом времени

![Снимок экрана: параметры дня и времени после окончания дня](media/auto-attendant-business-hours.png)

Для каждого автозавода можно настроить часы работы. Если рабочие часы не настроены, по умолчанию устанавливается круглосуточный график работы без выходных дней. Для работы можно установить перерывы в течение дня, а все часы, которые не считаются часами, считаются неавтными. Вы можете настроить другие параметры обработки входящих параметров обработки параметров и приветствий в течение часа.

В зависимости от настройки автозаполнения и очередей звонков может потребоваться настроить маршрутику звонков только для автозаполнения с прямыми номерами телефонов.

Если вам нужна отдельная маршрутия для вызывающих телефонных вызовов в нечасовом времени, укажите часы работы для каждого дня. Выберите **"Добавить новое время",** чтобы указать несколько часов для заданного дня, например, чтобы указать перерыв на обед.

После того как вы указали часы работы, выберите параметры маршрутизов для неавных часов. Доступны те же параметры, что и для маршрутизов в часы, которые вы указали выше.

Когда **все будет** готово, выберите "Далее".

## <a name="call-flows-during-holidays"></a>Потоки вызовов во время праздников

![Снимок экрана: параметры поздравительных и праздников](media/auto-attendant-holiday-greeting.png)

Для каждого настроенного праздника автозавод может быть настроен поток [звонка.](set-up-holidays-in-teams.md) Вы можете добавить до 20 запланированных праздников для каждого автосекретаря.

1. На странице параметров звонка "Праздник" выберите **"Добавить".**

2. Введите название этого параметра праздников.

3. В **выпадаемом** окнах "Праздники" выберите праздник, который вы хотите использовать.

4. Выберите нужный тип приветствия.

    ![Снимок экрана: параметры действия по праздничным звонкам](media/auto-attendant-holiday-actions.png)

5. Выберите, хотите ли вы **отключить или** **перенаправить** звонок.

6. Если вы перенаправили звонок, выберите пункт назначения маршрутиации для звонка.

7. Выберите **"Сохранить".**

![Снимок экрана: параметры праздников со списком праздников](media/auto-attendant-holiday-call-settings.png)

Повторите процедуру для каждого дополнительного праздника.

После того как вы добавите все праздники, выберите **"Далее".**

## <a name="dial-scope"></a>Область набора номера

![Снимок экрана: область набора номера с вариантами "Включить" и "Исключить"](media/auto-attendant-dial-scope.png)

Область *набора определяет* пользователей, доступных в каталоге, если звоня по имени или по телефону. По умолчанию **в** число всех сетевых пользователей входят все пользователи в вашей организации, которые являются пользователями Online или находятся на локальном сервере Skype для бизнеса Server.

Вы можете включить или исключить  определенных  пользователей, выбрав пользовательскую группу в группе "Включить" или **"Исключить"** и выбрав одну или несколько групп Microsoft 365, списки рассылки или группы безопасности. Например, может потребоваться исключить руководителей организации из каталога набора номера. (Если пользователь есть в обоих списках, он будет исключен из каталога.)

> [!NOTE]
> Для того чтобы новый пользователь указал свое имя в каталоге, может потребоваться до 36 часов.

После настройки области набора номера выберите **"Далее".**

## <a name="resource-accounts"></a>Учетные записи ресурсов

У всех автосуммников должна быть связанная учетная запись ресурса.  Автослужбу первого уровня потребуется по крайней мере одна учетная запись ресурса с связанным номером службы. При желании вы можете назначить автозаводу несколько учетных записей ресурсов с отдельным номером службы.

![Снимок экрана: панель добавления учетных записей для учетной записи ресурса](media/auto-attendant-add-resource-account.png)

Чтобы добавить учетную  запись ресурса, выберите "Добавить учетную запись" и на выберите учетную запись, которую вы хотите добавить. Выберите **"Добавить",** а затем выберите **"Добавить".**

![Снимок экрана: список учетных записей ресурсов с учетной записью ресурса с номером назначенной услуги](media/auto-attendant-resource-account-assigned.png)

Завершив добавление учетных записей  службы, выберите "Отправить", чтобы завершить настройку автоотчета.

## <a name="external-phone-number-transfers---technical-details"></a>Передача внешних номеров телефонов — технические сведения

Чтобы разрешить [автоответам](plan-auto-attendant-call-queue.md#prerequisites) переводить звонки внешним пользователям, обратитесь к необходимым условиям.  В дополнение:

- Для учетной записи ресурса с номером плана звонков номер телефона для внешнего переноса должен быть введен в формате E.164 (+[код страны][код города][номер телефона]). [](calling-plans-for-office-365.md)

- Для учетной записи ресурса с прямым маршрутным номером внешний [](direct-routing-connect-the-sbc.md) формат номера телефона для передачи зависит от параметров граничного контроллера сеанса.

Отображаемая исходящие телефонные номера определяется следующим образом:

  - Для номеров планов звонков отображается исходный номер телефона звоня.
  - Для номеров прямой маршрутификации количество отправленных номеров основано на параметре P-Утвердилось-Identity (PAI) на SBC:
    - Если установлено отключение, отображается исходный номер телефона звоня. Это значение задается по умолчанию и рекомендуется.
    - Если установлено "Включено", отображается номер телефона учетной записи ресурса.

В гибридной среде Skype для бизнеса создайте нового локального пользователя с настроенной для переадстройки звонков номером ПС. Пользователю должно быть включено Корпоративная голосовая связь и ему назначена голосовая политика. Дополнительные узнать см. в подзагонке "Перенастройка вызовов [автостраницы на ДНР".](https://docs.microsoft.com/SkypeForBusiness/plan/exchange-unified-messaging-online-migration-support#auto-attendant-call-transfer-to-pstn)
