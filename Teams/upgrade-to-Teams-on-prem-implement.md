## <a name="overlapping-capabilities-upgrade-using-islands-mode"></a>Обновление перекрывающихся возможностей (в режиме "Острова")

Для варианта обновления перекрывающихся возможностей:

- Рассмотрите этот вариант, если вы можете выполнить быстрое обновление для всей организации.  Так как существует потенциальный риск путаницы для конечных пользователей с запуском обоих клиентов, лучше всего свести к минимуму период времени, в течение которого пользователи должны запускать оба клиента. Убедитесь, что пользователи знают, как запускать оба клиента.

- Этот вариант является обычной моделью и не требует действий администратора для начала работы с Teams кроме назначения Microsoft 365 или Office 365 лицензии. Если у пользователей уже есть Skype для бизнеса Online, возможно, вы уже используете эту модель.

- Выход из режима перекрывающихся возможностей и переход на TeamsOnly может оказаться сложной задачей. Так как обновленные пользователи обмениваются данными только через Teams, любой другой пользователь в организации, взаимодействуя с этим пользователем, должен использовать Teams.  Если у вас есть пользователи, которые не начали использовать Teams, они будут подвержены отсутствующим сообщениям. Кроме того, они не будут видеть пользователей TeamsOnly в интернете в Skype для бизнеса. Некоторые организации решили выполнить обновление на уровне клиента, используя глобальную политику клиента, чтобы избежать этого, однако это требует предварительного планирования и ожидания, пока все пользователи будут готовы к обновлению.

## <a name="a-select-capabilities-upgrade-for-an-organization-that-has-not-yet-started-using-teams"></a>Обновление выбранных возможностей для организации, которая еще не начала использовать Teams

Если в вашей организации еще нет активных пользователей в Teams, первым шагом является установка политики на уровне клиента по умолчанию для TeamsUpgradePolicy в один из режимов Skype для бизнеса, например SfbWithTeamsCollab.  Пользователи, которые еще не начали использовать Teams, не заметят никаких различий в поведении. Однако установка этой политики на уровне клиента позволяет начать обновление пользователей до режима TeamsOnly и гарантирует, что обновленные пользователи по-прежнему смогут взаимодействовать с пользователями, не обновленными.  После идентификации пилотных пользователей их можно обновить до TeamsOnly.  Если они находятся в локальной среде, используйте Move-CsUser. Если они подключены к сети, просто назначьте им режим TeamsOnly с помощью Grant-CsTeamsUpgradePolicy. По умолчанию все Skype для бизнеса, запланированные этими пользователями, будут перенесены в Teams.

Ниже приведены основные команды.

1. Установите режим SfbWithTeamsCollab на уровне клиента по умолчанию следующим образом:

   ```PowerShell
   Grant-CsTeamsUpgradePolicy -PolicyName SfbWithTeamsCollab -Global
   ```

2. Обновите пилотных пользователей до TeamsOnly следующим образом:

   - Для пользователя, который находится в сети:

     ```PowerShell
     Grant-CsTeamsUpgradePolicy -PolicyName UpgradeToTeams -Identity $username
     ```

   - Для локального пользователя:

     ```PowerShell
     Move-CsUser -identity $user -Target sipfed.online.lync.com -MoveToTeams -credential $cred
     ```

**Примечания**:

- Вместо того чтобы задавать для политики на уровне клиента значение SfbWithTeamsCollab, можно задать для нее значение SfbWithTeamsCollabAndMeetings. Это приводит к тому, что все пользователи будут планировать все новые собрания в Teams.
- По умолчанию Skype для бизнеса переносятся в Teams при обновлении до режима TeamsOnly или при назначении режима SfbWithTeamsCollabAndMeetings.

> [!NOTE]
> В рамках подготовки к предстоящему прекращению Skype для бизнеса Online корпорация Майкрософт упростила переход организаций на Teams. Вам больше не нужно указывать `-MoveToTeams` `Move-CsUser` параметр для перемещения пользователей непосредственно из локальной среды в TeamsOnly. Ранее, если этот параметр не был указан, пользователи переходят с локального Skype для бизнеса Server на Skype для бизнеса Online, а их режим не изменился. `Move-CsUser`Теперь при перемещении пользователя из локальной среды в облако с помощью этого параметра пользователям автоматически назначается режим TeamsOnly, а их собрания из локальной среды автоматически преобразуются в Teams собрания, `-MoveToTeams switch had been specified`как если бы переключатель был указан на самом деле. Это поведение доступно во всех версиях Skype For Business Server и Lync Server 2013 (которые никогда не поддерживаются`-MoveToTeams`).

На схеме ниже показаны концептуальные этапы обновления выбранных возможностей для организации без предварительного использования Teams. Высота столбцов представляет количество пользователей. На любом этапе обновления все пользователи могут взаимодействовать друг с другом.  Skype для бизнеса взаимодействуют с пользователями TeamsOnly с помощью взаимодействия и наоборот. Пользователи в режиме "Острова" должны обязательно запускать оба клиента.

## <a name="a-select-capabilities-upgrade-for-an-organization-that-is-already-using-teams-in-islands-mode"></a>Обновление выбранных возможностей для организации, которая уже использует Teams в режиме "Острова".

Если некоторые пользователи в вашей организации активно используют Teams в режиме "Острова", вы, вероятно, не захотите удалять функциональные возможности существующих пользователей. Поэтому перед изменением политики на уровне клиента необходимо выполнить дополнительный шаг. Решением является "перенастройка" существующих активных Teams пользователей в режим "Острова", прежде чем назначить для политики на уровне клиента значение SfbWithTeamsCollab.  После этого можно продолжить развертывание, как описано выше, однако у вас будет две группы пользователей, которые переходят на TeamsOnly: пользователи, которые были активны в Teams, будут работать в режиме Islands, а остальные пользователи — в режиме SfbWithTeamsCollab. Вы можете постепенно перемещать этих пользователей в режим TeamsOnly.

1. Найдите пользователей, активных в Teams следующим образом:

   1. В Центр администрирования Microsoft 365 навигации слева перейдите к разделу "Отчеты", а затем — "Использование".
   2. В раскрывающемся списке "Выбор отчета" выберите Microsoft Teams и действие пользователя. Это позволит предоставить экспортируемую таблицу пользователей, которые были активны в Teams.
   3. Нажмите кнопку "Экспорт", Excel и отфильтруйте, чтобы отобразить только пользователей, активных в Teams.

2. Для каждого активного Teams, найденного на шаге 1, назначьте ему режим "Острова" в удаленной версии PowerShell. Это позволяет перейти к следующему шагу и не изменять взаимодействие с пользователем.

   ```PowerShell
   $users=get-content "C:\MyPath\users.txt"
    foreach ($user in $users){
    Grant-CsTeamsUpgradePolicy -identity $user -PolicyName Islands}
   ```

3. Задайте для политики на уровне клиента значение SfbWithTeamsCollab:

   ```PowerShell
   Grant-CsTeamsUpgradePolicy -Global -PolicyName SfbWithTeamsCollab
   ```

4. Обновите выбранных пользователей до режима TeamsOnly. Вы можете обновить пользователей в режиме Islands или SfbWithTeamsCollab, хотя вам может потребоваться сначала установить приоритеты обновления пользователей в режиме "Острова", чтобы свести к минимуму вероятность путаницы, которая может возникнуть, когда пользователи находятся в режиме "Острова".

   Для пользователей, которые размещены в Skype для бизнеса Online:

   ```PowerShell
   Grant-CsTeamsUpgradePolicy -Identity $user -PolicyName UpgradeToTeams
   ```

   Для пользователей, Skype для бизнеса Server локально:

   ```PowerShell
   Move-CsUser -Identity $user -Target sipfed.online.lync.com -MoveToTeams -credential $cred
   ```

На любом этапе обновления все пользователи могут взаимодействовать друг с другом.  Skype для бизнеса взаимодействуют с пользователями TeamsOnly с помощью взаимодействия и наоборот.
