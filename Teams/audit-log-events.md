# <a name="search-the-audit-log-for-events-in-microsoft-teams"></a>Поиск событий Microsoft Teams в журнале аудита

> [!IMPORTANT]
> [!INCLUDE [new-teams-sfb-admin-center-notice](includes/new-teams-sfb-admin-center-notice.md)]

Журнал аудита поможет вам изучить определенные действия в службах Microsoft 365. Вот некоторые из действий, которые проверяются в Microsoft Teams:

- Создание команды
- Удаление команды
- Добавление канала
- Изменение параметра

Полный список действий Teams, которые проверяются в режиме аудита, см. в действиях [Teams](#teams-activities) и "Смены" [(в предварительной версии).](#shifts-in-teams-activities)

> [!NOTE]
> События аудита из частных каналов также регистрируются так же, как и для стандартных и командных каналов.

## <a name="turn-on-auditing-in-teams"></a>Включение аудита в Teams

Чтобы можно было посмотреть данные аудита, необходимо сначала включить аудит в Центре & [безопасности.](https://protection.office.com) Справку о включите аудит, прочитав статью "Включение и отключение поиска [в журнале аудита".](https://support.office.com/article/Turn-Office-365-audit-log-search-on-or-off-e893b19a-660c-41f2-9074-d3631c95a014)

> [!IMPORTANT]
> Данные аудита доступны только с того момента, когда вы включили аудит.

## <a name="retrieve-teams-data-from-the-audit-log"></a>Извлечение данных Teams из журнала аудита

1. Для извлечения журналов аудита войдите в [Центр безопасности и соответствия требованиям](https://go.microsoft.com/fwlink/?linkid=855775). В **поле "Поиск"** **выберите "Поиск в журнале аудита".**

2. Используйте **Search** (Поиск) для фильтрации по действиям, датам и пользователям, по которым необходим аудит.

3. Для дальнейшего анализа экспортируйте результаты в Excel.

> [!IMPORTANT]
> Данные аудита видны в журнале аудита только в том случае, если он включен.

Продолжительность сохранения записи аудита и возможность поиска в журнале аудита зависит от вашей подписки на Microsoft 365 или Office 365, а именно от типа лицензии, которая назначена пользователям. Дополнительные инструкции см. в [описании службы & безопасности и соответствия требованиям.](/office365/servicedescriptions/office-365-platform-service-description/office-365-securitycompliance-center)

## <a name="tips-for-searching-the-audit-log"></a>Советы по поиску в журнале аудита

Вот несколько советов по поиску действий Teams в журнале аудита.

![Снимок экрана: страница поиска в журнале аудита](media/audit-log-search-page.png)

- Вы можете выбрать действия, которые нужно найти, щелкнув их имя. Вы также можете найти все действия в группе (например, действия с файлами и папками), щелкнув ее имя. Если действие выбрано, щелкните его, чтобы отменить выбор. В поле поиска также можно отобразить действия, содержащие ключевое слово, которое вы введите.

  ![Снимок экрана: поиск по журналу аудита](media/audit-log-search.png)

- Чтобы отобразить события для действий, которые запускаются с помощью cmdlets, выберите "Показать результаты **по всем действиям"** в списке **"Действия".** Если вы знаете название операции для этих действий, поищите все действия, а затем отфильтруем результаты, введя имя операции в поле в столбце **"Действия".** Дополнительные узнать [см. в шаге 3 . Фильтрация результатов поиска.](/microsoft-365/compliance/search-the-audit-log-in-security-and-compliance?view=o365-worldwide#step-3-filter-the-search-results)

- Чтобы очистить текущие условия поиска, нажмите кнопку **"Очистить".** Диапазон дат возвращается по умолчанию (последние семь дней). Вы также можете нажать кнопку **"Очистить все",** чтобы показать результаты по всем действиям, чтобы отменить все выбранные действия.

- Если найдено 5000 результатов, можно предположить, что условиям поиска соответствует более 5000 событий. Вы можете уточнить условия поиска и повторно использовать поиск, чтобы получить меньше результатов, или экспортировать все результаты поиска, выбрав "Скачать все результаты  >  **экспорта".**

Посмотрите это [видео об](https://www.youtube.com/embed/UBxaRySAxyE) использовании поиска в журнале аудио. Присоединитесь к Управляющему программой Для Teams, который демонстрирует, как искать Teams в журнале аудита.

## <a name="use-cloud-app-security-to-set-activity-policies"></a>Настройка политик действий с помощью Cloud App Security

С [помощью интеграции Microsoft Cloud App Security](/cloud-app-security/what-is-cloud-app-security) можно настроить политики действий для применения широкого спектра автоматизированных процессов с помощью API поставщика приложений. [](/cloud-app-security/user-activity-policies) Эти политики позволяют отслеживать определенные действия, которые выполняли разные пользователи, или отслеживать неожиданный высокий уровень активности определенного типа.

После того как вы настроите политику обнаружения действий, она начнет создавать оповещения. Оповещения создаются только для действий, которые происходят после создания политики. Вот несколько примеров сценариев использования политик действий в Cloud App Security для отслеживания действий Teams.

### <a name="external-user-scenario"></a>Сценарий для внешнего пользователя

Один из сценариев, на который вы, возможно, захотите следить с точки зрения бизнеса, — это добавление внешних пользователей в среду Teams. Если внешние пользователи включены, лучше следить за их присутствием.  Вы можете использовать [Cloud App Security](/cloud-app-security/what-is-cloud-app-security) для выявления потенциальных угроз.

![Снимок экрана: список событий, инициированных массовым удалением](media/TeamsExternalUserAddPolicy.png)

Снимок экрана с этой политикой для отслеживания добавления внешних пользователей позволяет присвоть ей имя, определить серьезность в соответствии с потребностями организации, установить для нее одно действие, а затем установить параметры, которые будут специально отслеживать добавление внешних пользователей, и ограничить это действие Teams.

Результаты этой политики можно просмотреть в журнале действий.

![Снимок экрана: список событий, инициированных массовым удалением](media/TeamsExternalUserList.png)

Здесь вы можете просмотреть совпадения с настроенной политикой и внести необходимые изменения или экспортировать результаты для использования в другом месте.

### <a name="mass-delete-scenario"></a>Сценарий массового удаления

Как упоминалось выше, вы можете отслеживать сценарии удаления. Можно создать политику, которая будет отслеживать массовое удаление сайтов Teams. В этом примере политика на основе оповещения настроена для обнаружения массового удаления команд в течение 30 минут.

![Снимок экрана: страница создания политики с настройкой политики обнаружения массового удаления группы](media/TeamsMassDeletePolicy.png)

Как показано на снимке экрана, вы можете настроить для этой политики множество различных параметров для отслеживания удаления Teams, включая серьезность, однократные или повторяющие действия, а также параметры, ограничивающие это удалением Teams и сайтов. Это можно сделать независимо от шаблона или на основе этой политики в зависимости от потребностей организации.

После создания политики, которая работает для вашей организации, вы можете просмотреть результаты в журнале действий по мере запуска событий:

![Снимок экрана: список событий, инициированных массовым удалением](media/TeamsMassDeleteList.png)

Вы можете отфильтровать результаты до политики, настроенной вами. Если результаты, которые вы получаете в журнале действий, не являются удовлетворительными (например, вы видите много результатов или вообще ничего не видите), это может помочь вам настроить запрос, чтобы он был более релевантным для его выполнения.

### <a name="alert-and-governance-scenario"></a>Сценарий оповещения и управления

Вы можете настроить оповещения и отправлять сообщения электронной почты администраторам и другим пользователям при запуске политики действий. Вы можете настроить автоматические действия управления, такие как приостановка пользователя или повторная вход пользователя автоматически. В этом примере показано, как учетную запись пользователя можно приостановить при запуске политики действий и определить, что пользователь удалил несколько групп за 30 минут.

![Снимок экрана: оповещения и действия управления для политики действий](media/audit-log-governance.png)

## <a name="use-cloud-app-security-to-set-anomaly-detection-policies"></a>Настройка политик обнаружения аномалий с помощью Cloud App Security

Политики [обнаружения](/cloud-app-security/anomaly-detection-policy) аномалий в Cloud App Security обеспечивают пользовательскую и entity behavioral analytics (UEBA) и машинное обучение (ML), чтобы можно было немедленно запустить расширенные возможности обнаружения угроз в облачной среде. Так как они включены автоматически, новые политики обнаружения аномалий предоставляют мгновенные результаты, обеспечивая немедленное обнаружение, выявляя множество аномалий поведения на ваших компьютерах и устройствах, подключенных к сети. Кроме того, новые политики выдвигают больше данных из средства обнаружения Cloud App Security, что помогает ускорить исследование и выявить угрозы.

Мы работаем над интеграцией событий Teams с политиками обнаружения аномалий. В настоящее время вы можете настроить политики обнаружения аномалий для других продуктов Office и принять меры с пользователями, которые соответствуют этим политикам.

## <a name="attribution-in-teams-audit-logs"></a>Атрибуция в журналах аудита Teams

Изменения в членстве в Teams (например, добавленные или удаленные пользователи), внесенные через Azure Active Directory (Azure AD), портал администрирования Microsoft 365 или API Microsoft 365 Groups Graph, будут отображаться в сообщениях аудита Teams и в канале "Общее" с привяжителем к существующему владельцу команды, а не фактическому инициатору действия. В этих сценариях для получения необходимой информации обратитесь в журналы аудита Azure AD или [группы Microsoft 365.](/microsoft-365/compliance/search-the-audit-log-in-security-and-compliance)
