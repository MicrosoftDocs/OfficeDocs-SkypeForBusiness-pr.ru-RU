# <a name="search-the-audit-log-for-events-in-microsoft-teams"></a>Поиск событий Microsoft Teams в журнале аудита

> [!IMPORTANT]
> [!INCLUDE [new-teams-sfb-admin-center-notice](includes/new-teams-sfb-admin-center-notice.md)]

Журнал аудита поможет вам изучить конкретные действия в Microsoft 365 службах. Вот Microsoft Teams действий, которые проверяются.

- Создание команды
- Удаление команды
- Добавление канала
- Удаленный канал
- Изменен параметр канала

Полный список действий, Teams, которые проверяются, [](#teams-activities) см. в Teams действиях и смены в Teams [действиях.](#shifts-in-teams-activities)

> [!NOTE]
> События аудита из частных каналов также регистрируются так же, как и для команд и стандартных каналов.

## <a name="turn-on-auditing-in-teams"></a>Включение аудита в Teams

Чтобы можно было посмотреть данные аудита, необходимо сначала включить аудит в Центр соответствия требованиям Microsoft 365. Дополнительные сведения [см.](/microsoft-365/compliance/turn-audit-log-search-on-or-off)в том, как включить или отключить аудит.

> [!IMPORTANT]
> Данные аудита доступны только с того момента, когда вы включили аудит.

## <a name="retrieve-teams-data-from-the-audit-log"></a>Извлечение данных Teams из журнала аудита

1. Чтобы получить журналы аудита для Teams действий, перейдите к <https://compliance.microsoft.com> и выберите **Аудит**.

2. На странице **Поиск** отфильтруются действия, даты и пользователи, для аудита.

3. Для дальнейшего анализа экспортируйте результаты в Excel.

Пошаговые инструкции см. в журнале [аудита в Центре соответствия требованиям.](/microsoft-365/compliance/search-the-audit-log-in-security-and-compliance#search-the-audit-log)

> [!IMPORTANT]
> Данные аудита видны в журнале только в том случае, если включен аудит.

Время сохранения записи аудита и возможность поиска в журнале аудита зависит от вашей подписки на Microsoft 365 или Office 365, а именно от типа лицензии, которая назначена пользователям. Дополнительные инструкции см. в описании [службы Центра & безопасности.](/office365/servicedescriptions/office-365-platform-service-description/office-365-securitycompliance-center)

## <a name="tips-for-searching-the-audit-log"></a>Советы поиска по журналу аудита

Вот несколько советов по поиску Teams действий в журнале аудита.

:::image type="content" alt-text="Снимок экрана: страница поиска по журналу аудита в Центре соответствия требованиям" source="media/audit-log-search-page.png" lightbox="media/audit-log-search-page.png":::

- Вы можете выбрать определенные действия, которые нужно найти, нажав на один или несколько действий. Если выбрано действие, вы можете щелкнуть его, чтобы отменить выбор. В поле поиска также можно отобразить действия, содержащие введите ключевое слово.

  ![Снимок экрана: список действий на странице поиска в журнале аудита](media/audit-log-search.png)

- Чтобы отобразить события для действий, которые будут происходить с помощью этих действий, выберите в списке Действия показывать результаты **по** **всем** действиям. Если вы знаете название операции для этих действий, введите ее в поле поиска, чтобы отобразить действие, а затем выберите его.

- Чтобы очистить текущие условия поиска, нажмите **кнопку Очистить все**. Диапазон дат возвращается к значениям по умолчанию за последние семь дней.

- Если найдено 5000 результатов, можно предположить, что условиям поиска соответствует более 5000 событий. Вы можете уточнить условия поиска и повторно использовать его, чтобы получить меньше результатов, или экспортировать все результаты поиска, выбрав Экспорт  >  **Скачать все результаты**. Пошаговую инструкцию по экспорту журналов аудита см. в записи Экспорт [результатов поиска в файл.](/microsoft-365/compliance/search-the-audit-log-in-security-and-compliance#step-3-export-the-search-results-to-a-file)

Посмотрите это [видео об](https://www.youtube.com/embed/UBxaRySAxyE) использовании поиска в журнале звукозаписей. Присоединитесь к Ансуману Ахарию, руководителю программы для Teams, так как он демонстрирует, как искать данные в журнале Teams.

## <a name="office-365-management-activity-api"></a>Office 365 API действий управления

Вы можете использовать API Office 365 действий управления для получения сведений о Teams событиях. Дополнительные информацию о схеме API действий управления для Teams см. в Teams [схеме.](/office/office-365-management-api/office-365-management-activity-api-schema#microsoft-teams-schema)

## <a name="attribution-in-teams-audit-logs"></a>Атрибуция в Teams журналах аудита

Членство изменяется на Teams (например, добавленных или удаленных пользователей), внесенных с помощью Azure Active Directory (Azure AD), портала администрирования Microsoft 365 или Microsoft 365 групп Graph API будет отображаться в Teams  сообщения аудита и в канале "Общее" с приписацией к существующему владельцу команды, а не к фактическому инициатору действия. В этих сценариях обратитесь к Azure AD [или Microsoft 365 журналы](/microsoft-365/compliance/search-the-audit-log-in-security-and-compliance) аудита группы, чтобы увидеть соответствующие сведения.

## <a name="use-defender-for-cloud-apps-to-set-activity-policies"></a>Настройка политик действий с помощью Защитника облачных приложений

С [помощью интеграции с Microsoft](/cloud-app-security/what-is-cloud-app-security) [](/cloud-app-security/user-activity-policies) Defender для облачных приложений вы можете настроить политики действий, чтобы применять разнообразные автоматизированные процессы с помощью API поставщиков приложений. Эти политики позволяют отслеживать определенные действия, которые выполняли разные пользователи, или отслеживать неожиданные высокие показатели для определенного типа деятельности.

После того как вы настроите политику обнаружения действий, она начнет создавать оповещения. Оповещения создаются только для действий, которые происходят после создания политики. Вот несколько сценариев, в которые можно использовать политики действий в Защитнике облачных приложений для отслеживания Teams действий.

### <a name="external-user-scenario"></a>Сценарий внешнего пользователя

Если внешние пользователи включены, лучше следить за их присутствием.  Защитник для [облачных приложений можно использовать](/cloud-app-security/what-is-cloud-app-security) для определения потенциальных угроз.

Как упоминалось выше, вы можете отслеживать сценарии удаления. Можно создать политику, которая будет отслеживать массовое удаление Teams сайтов. В этом примере политика на основе оповещений настроена для обнаружения массового удаления команд в течение 30 минут.

## <a name="use-defender-for-cloud-apps-to-set-anomaly-detection-policies"></a>Настройка политик обнаружения аномалий с помощью Защитника облачных приложений

Политики [обнаружения](/cloud-app-security/anomaly-detection-policy) аномалий в Защитнике для облачных приложений предоставляют обн. пользовательский и юридический аналитики поведения (UEBA) и машинного обучения (ML), что позволяет немедленно запускать расширенные средства обнаружения угроз в облачной среде. Так как они включены автоматически, новые политики обнаружения аномалий обеспечивают немедленные результаты, обеспечивая немедленное обнаружение, выявляя различные аномалии поведения на компьютерах и устройствах, подключенных к сети. Кроме того, новые политики содержат дополнительные данные с помощью средства обнаружения "Защитник для облачных приложений", что помогает ускорить процесс исследования и выявлять угрозы.

Мы работаем над интеграцией Teams с политиками обнаружения аномалий. В настоящее время вы можете настроить политики обнаружения аномалий для других продуктов Office и принять меры для пользователей, которые соответствуют этим политикам.
