# <a name="understand-teams-and-channels-in-microsoft-teams"></a>Знакомство с командами и каналами в Microsoft Teams

Концепции команд и каналов являются фундаментальными для работоспособной и продуктивной реализации Microsoft Teams. 

- Команда — это коллекция людей, содержимого и средств, которые работают вместе для получения бизнес-результата для вашей компании. Команды Teams создаются на основе групп Microsoft 365, и любые изменения участия в группе Microsoft 365 синхронизируются в команде.

- Каналы — это пространства для совместной работы в команде, в которой выполняется фактическая работа. 

Дополнительные сведения о командах и каналах см. в обзоре команд и каналов [в Microsoft Teams](teams-channels-overview.md). Полное руководство по внедрению Teams см. в [руководстве по внедрению Teams](https://aka.ms/teamstoolkit). Дополнительные сведения о Teams и Группы Microsoft 365 см. Группы Microsoft 365 [Microsoft Teams](office-365-groups.md) и дополнительные сведения о [Группы Microsoft 365.](https://support.office.com/article/Learn-about-Office-365-groups-b565caa1-5c40-40ef-9915-60fdb2d97fa2)

Типы доступа к команде определяют, кто может присоединиться к команде:

- В *закрытые* команды могут вступать только участники, одобренные их владельцами. Это типичный вариант для команд по проектам и виртуальных команд в крупной организации.

- *Общедоступные* команды открыты для всех сотрудников организации, и пользователи могут присоединяться к ним напрямую. Открытые команды полезны для совместной работы по темам, интересующих людей из разных отделов или работающих над разными проектами. Это хороший вариант по умолчанию для небольших организаций.

## <a name="use-teams-to-drive-cross-organization-collaboration"></a>Использование Teams для совместной работы между организацией

Рекомендуется использовать команды для улучшения совместной работы в разных организациях, и многие проекты подходят для этой модели. Кроме того, руководителям или подразделениям внутри организации может потребоваться выделенная команда для своих сотрудников. Рассмотрим модели организации и проектов, показанные на следующем рисунке.

![Модели организации и проекта.](media/teams-adoption-organization-project.png)

В рабочей группе организации (показана слева) могут быть предоставлены основные сведения о том, как организация выполняет свою бизнес-задачу, события группы, основные стратегии и другие операционные сведения, такие как бизнес-проверки. Для команды, показанной справа, сведения о округе разбивается на каналы, которые представляют категории работы, выполняемые этой командой. Члены этой команды могут или не могут сообщать одному и тем же пользователю, но все они вовлечены в управление результатами для округа.
