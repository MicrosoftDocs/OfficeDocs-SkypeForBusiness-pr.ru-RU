# <a name="use-microsoft-teams-administrator-roles-to-manage-teams"></a>Управление Microsoft Teams с помощью ролей администраторов

Используя Azure Active Directory (Azure AD), вы можете назначать администраторов, которым необходимы разные уровни доступа для управления Microsoft Teams. Администраторы могут управлять всей рабочей Teams или иметь делегирование разрешений на устранение неполадок с качеством звонка или управление потребностями вашей организации в телефонии.

## <a name="teams-roles-and-capabilities"></a>Teams ролей и возможностей

Существует несколько ролей Teams администраторов: администратор Teams, администратор связи Teams, специалист службы Teams связи, инженер службы поддержки Teams связи и администратор Teams устройства. В таблице ниже поймем, что может делать каждая роль и какие средства администратор может использовать в Microsoft Teams администрирования и PowerShell.

Для этого вы должны быть администратором. Инструкции по получении разрешений указаны в этой статье.

## <a name="assign-users-to-each-role"></a>Назначение пользователей каждой роли

Вы можете назначать пользователям эти роли в Azure AD. Чтобы узнать, как назначать роли администратора пользователю в Azure AD, см. назначение роли администратора [в Azure Active Directory.](/azure/active-directory/fundamentals/active-directory-users-assign-role-azure-portal)

## <a name="cmdlets-available-for-each-role"></a>Доступные для каждой роли cmdlets

Большинство средств PowerShell для этих ролей администраторов доступны в модуле Teams PowerShell, и важно отметить, что некоторые командлеты, которым они доступны, имеют доступ к общим настройкам, которые также используются для Skype для бизнеса Online. 

Чтобы просмотреть полный список cmdlets:

```powershell
Get-Command -Module MicrosoftTeams
 ```
