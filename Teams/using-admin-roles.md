# <a name="use-microsoft-teams-administrator-roles-to-manage-teams"></a>Управление Microsoft Teams с помощью ролей администраторов

Используя Azure Active Directory (Azure AD), вы можете назначать администраторов, которым требуются разные уровни доступа для управления Microsoft Teams. Администраторы могут управлять всей рабочей нагрузкой Teams или иметь делегирование разрешений на устранение неполадок с качеством звонка или управление потребностями организации по телефону.

## <a name="teams-roles-and-capabilities"></a>Роли и возможности Teams

Доступно несколько ролей администратора Teams: администратор службы Teams, администратор связи Teams, специалист по поддержке связи Teams, специалист службы поддержки связи Teams и администратор устройства Teams. В таблице ниже поймут, что может делать каждая роль и какие средства может использовать администратор в Центре администрирования Microsoft Teams и PowerShell.

Для этого вы должны быть администратором. Инструкции по получении разрешений указаны в этой статье.

## <a name="assign-users-to-each-role"></a>Назначение пользователей для каждой роли

Эти роли можно назначать пользователям в Azure AD. Чтобы узнать, как назначать административные роли пользователю в Azure AD, см. процедуру назначения роли администратора пользователю [в Azure Active Directory.](/azure/active-directory/fundamentals/active-directory-users-assign-role-azure-portal)

## <a name="cmdlets-available-for-each-role"></a>Для каждой роли доступны cmdlets

Большинство средств PowerShell для этих ролей администраторов находятся в модуле Teams PowerShell, и важно отметить, что некоторые командлеты, которым эти роли администратора, доступны для управления общими настройками, которые также используются в Skype для бизнеса Online. 

Чтобы просмотреть полный список cmdlets:

```powershell
Get-Command -Module MicrosoftTeams
 ```
