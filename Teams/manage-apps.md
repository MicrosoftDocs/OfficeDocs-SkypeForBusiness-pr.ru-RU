<a name="manage-your-apps-in-the-microsoft-teams-admin-center"></a>Управление приложениями в Центре администрирования Microsoft Teams
======================================================

Как администратор, на странице "Управление приложениями" в Центре администрирования Microsoft Teams можно просматривать и управлять всеми приложениями Teams для организации. Здесь вы можете просмотреть состояние и свойства приложений на уровне организации, утвердить или добавить новые пользовательские приложения в магазин приложений организации, заблокировать или разрешить приложения на уровне организации, добавить приложения в группы, приобрести службы для сторонних приложений, просмотреть разрешения, запрашиваемые приложениями, предоставить администратору согласие на использование приложений и управлять настройками приложений в масштабе организации.

На странице "Управление приложениями" вы можете просмотреть все доступные приложения, чтобы определить, какие приложения следует разрешить или заблокировать в организации. Затем вы можете использовать политики разрешений [приложений,](teams-app-permission-policies.md)политики настройки [приложений,](teams-app-setup-policies.md)а также настраиваемые политики и параметры приложений для настройки работы с приложениями для определенных пользователей в организации. [](teams-custom-app-policies-and-settings.md)

В левой области навигации Центра администрирования Microsoft Teams перейдите в приложение **Teams**  >  **"Управление приложениями".** Для доступа к странице необходимо быть глобальным администратором или администратором служб Teams.

> [!NOTE]
> Страница "Управление приложениями" пока недоступна в развертываниях Teams в microsoft 365 Government Community Cloud (GCC).

## <a name="view-apps"></a>Просмотр приложений

Вы можете просмотреть все приложения, включая следующие сведения о каждом приложении.

![Снимок экрана: страница "Управляемые приложения"](media/manage-apps.png)

- **Name**: имя приложения. Щелкните имя приложения, чтобы перейти на страницу сведений о приложении, чтобы увидеть дополнительные сведения о нем. Оно содержит описание разрешенного или заблокированного приложения, версию, политику конфиденциальности, условия использования, категории, применимые к приложению, состояние сертификации, поддерживаемые возможности и ИД приложения. Ниже приводится пример.

  ![Снимок экрана: страница сведений о приложениях](media/manage-apps-app-details.png)
  
- **Сертификация:** если приложение прошло сертификацию, вы увидите либо **сертифицированную Microsoft 365,** либо **проверку Publisher.** Щелкните ссылку, чтобы просмотреть сведения о сертификации для приложения. Если вы видите **--** ", у нас нет сведений о сертификации для приложения. Дополнительные информацию о сертифицированных приложениях в Teams можно найти в программе сертификации [приложений Microsoft 365.](https://docs.microsoft.com/teams-app-certification/all-apps)  
- **Publisher:** имя издателя.
- **Состояние публикации:** состояние публикации пользовательских приложений.
- **Состояние:** состояние приложения на уровне организации, которое может быть одним из следующих:

    - **Разрешено:** приложение доступно для всех пользователей в организации.
    
    - **Заблокировано:** приложение заблокировано и не доступно ни для одного пользователя в организации.
    
    - **Заблокирована на всей организации:** приложение блокируется в параметрах приложения для всей организации.
    
      Важно знать, что в этом столбце представлено разрешенное и заблокированное состояние  приложений, которые ранее находились на области параметров всей организации. Теперь на странице "Управление приложениями" можно просматривать, блокировать и разрешать приложения во всей **организации.** 
- **Лицензии**: указывает, предоставляет ли приложение подписку на программное обеспечение как услугу (SaaS) для приобретения. Этот столбец относится только к приложениям сторонних приложений. Для каждого приложения сторонних разработчиков будет доступно одно из следующих значений:
    - **Приобрести сейчас:** приложение предоставляет подписку на SaaS и доступно для приобретения.  
    - **Приобретено:** приложение предоставляет подписку на SaaS, и вы приобрели для него лицензии.
    - **- -**. Приложение не предлагает подписку на SaaS.
- **Пользовательское** приложение: является ли оно настраиваемой.
- **Разрешения:** указывает, имеет ли стороннее или пользовательское приложение, зарегистрированное в Azure Active Directory (Azure AD), разрешения, для получения согласия. Вы увидите одно из следующих значений:
    - **Просмотр сведений:** у приложения есть разрешения, которые требуют разрешения, чтобы приложение может получить доступ к данным. 
    - **- -**. У приложения нет разрешений, которые требуют разрешения.
- **Категории:** категории, которые относятся к приложению.
- **Версия:** версия приложения.

Чтобы увидеть в таблице нужные сведения, нажмите кнопку "Изменить столбец" в правом верхнем углу, чтобы добавить столбцы в таблицу или удалить их из нее. 

## <a name="publish-a-custom-app-to-your-organizations-app-store"></a>Публикация пользовательского приложения в магазине приложений организации

На странице "Управление приложениями" можно публиковать приложения, разработанные специально для вашей организации. После публикации пользовательского приложения оно будет доступно пользователям в магазине приложений вашей организации. Опубликовать пользовательское приложение в магазине приложений организации можно двумя способами. Способ использования зависит от того, как вы получаете приложение.

- [Утверждение пользовательского приложения.](#approve-a-custom-app)Используйте этот способ, если разработчик передает приложение непосредственно на страницу "Управление приложениями" с помощью API отправки приложения Teams. После этого вы можете просмотреть и опубликовать (или отклонить) приложение прямо на странице сведений о приложении.
- [Отправка пакета приложения:](#upload-an-app-package)используйте этот способ, если разработчик отправляет вам пакет приложения в формате ZIP. Вы публикуете приложение, загрузив пакет приложения.

###  <a name="approve-a-custom-app"></a>Утверждение пользовательского приложения

**Мини-приложение "Ожидающих** утверждения" на странице "Управление приложениями" сообщает о том, что разработчик подает приложение с помощью API отправки приложения Teams. В списке будет указано  новое отправленное приложение с состоянием публикации **"Отправлено"** **и** состоянием **блокировки.** Перейдите на страницу сведений о приложении, чтобы увидеть дополнительные сведения о приложении, а затем опубликовать его, задав для состояния публикации **состояние** **"Опубликовать".**

Вы также будете уведомлены, когда разработчик подает обновление в пользовательское приложение. Затем вы можете просмотреть и опубликовать (или отклонить) обновление на странице сведений о приложении. Для обновленного приложения сохраняются все политики разрешений и настройки приложений.

Дополнительные данные [см. в документе "Публикация](submit-approve-custom-apps.md)пользовательского приложения, отправленного с помощью API отправки приложения Teams".

### <a name="upload-an-app-package"></a>Отправка пакета приложения

Разработчик создает пакет приложения Teams с помощью [Teams App Studio,](https://docs.microsoft.com/microsoftteams/platform/get-started/get-started-app-studio)а затем отправляет его вам в zip-формате. Если у вас есть пакет приложения, вы можете отправить его в магазин приложений своей организации.

Чтобы добавить новое пользовательское приложение, выберите **"Отправить",** чтобы добавить пакет приложения. После отправки приложение не выделяется, поэтому вам нужно будет найти его в списке приложений на странице "Управление приложениями".

Чтобы обновить приложение после его отправки, в списке приложений на странице "Управление приложениями" щелкните имя приложения и нажмите кнопку **"Обновить".** Это заменяет существующее приложение, а все политики разрешений и настройки приложений остаются принудительно примененными для обновленного приложения.

Дополнительные данные [см. в статьи "Публикация пользовательского приложения путем отправки пакета приложения".](upload-custom-apps.md)

## <a name="allow-and-block-apps"></a>Разрешение и блокирование приложений

На странице "Управление приложениями" можно разрешить или заблокировать отдельные приложения на уровне организации. Здесь показаны все доступные приложения и их текущее состояние на уровне организации. (Блокировка и разрешение приложений на уровне  организации перемещены из области параметров приложения для всей организации.)

Чтобы разрешить или заблокировать приложение, выберите его и нажмите кнопку **"Разрешить"** или **"Заблокировать".** Когда вы блокируете приложение, все его взаимодействия с ним отключаются и приложение не появляется в Teams для пользователей в вашей организации.

Если вы блокируете или разрешаете приложение на странице "Управление приложениями", это приложение блокируется или разрешено для всех пользователей в организации.  Если вы блокируете или разрешаете приложение в политике разрешений приложений Teams, оно блокируется или разрешено для пользователей, которым назначена эта политика. Чтобы пользователь мог устанавливать приложения и работать с ними, необходимо разрешить это приложение на уровне организации на странице "Управление приложениями" и в назначенной пользователю политике разрешений.

 > [!NOTE]
 > Чтобы удалить приложение, щелкните его правой кнопкой  мыши и выберите пункт  "Удалить" или воспользуйтесь меню "Дополнительные приложения" в левой части.

## <a name="add-an-app-to-a-team"></a>Добавление приложения в команду

Кнопка **"Добавить в команду"** используется для установки приложения в команду. Имейте в виду, что это касается только приложений, которые можно установить в рамках группы. Кнопка **"Добавить в команду"** недоступна для приложений, которые можно установить только в личной области.

![Снимок экрана: кнопка "Добавить в команду"](media/manage-apps-add-app-team.png)

1. Найдите нужное приложение, а затем выберите его, щелкнув слева от его имени.
2. Выберите **"Добавить в команду".**
3. В области **"Добавить в команду"** на выберите команду, в нее нужно добавить приложение, а затем выберите команду "Применить". 

## <a name="purchase-services-for-third-party-apps"></a>Приобретение служб для сторонних приложений

Вы можете искать и приобретать лицензии на службы, предлагаемые сторонними приложениями для пользователей в организации, непосредственно на странице "Управление приложениями". Столбец **"Лицензии"** в таблице указывает, предоставляет ли приложение платную подписку на SaaS. Нажмите **кнопку "Приобрести",** чтобы просмотреть сведения о планах и ценах, а также приобрести лицензии для пользователей. Подробнее см. в центре администрирования Microsoft Teams, где можно приобрести службы для [сторонних приложений.](purchase-third-party-apps.md)

## <a name="grant-admin-consent-to-apps"></a>Предоставление администратору разрешения на доступ к приложениям

Вы можете просмотреть и предоставить согласие приложениям, запрашивающим разрешения от имени всех пользователей в организации. Это необходимо для того, чтобы пользователям не было нужно при запуске приложения пересматривать и принимать разрешения, запрашиваемые приложением. В **столбце** "Разрешения" указывается, есть ли у приложения разрешения, для получения согласия. Вы увидите ссылку "Просмотреть **сведения"** для каждого приложения, зарегистрированного в Azure AD, с разрешениями, которые требуют согласия. Дополнительные узнать см. в центре администрирования Microsoft Teams, чтобы просмотреть разрешения приложений и предоставить разрешение [администратора.](app-permissions-admin-center.md)

## <a name="view-resource-specific-consent-permissions"></a>Просмотр разрешений на доступ к ресурсам

Разрешения для конкретного ресурса (RSC) дает владельцу группы согласие на доступ к данным группы и их изменение. Разрешения RSC — это детализация , особые разрешения Teams, которые определяют, что приложение может делать в определенной команде. Вы можете просмотреть разрешения RSC на вкладке **"Разрешения"** страницы сведений о приложении. Дополнительные узнать см. в центре администрирования Microsoft Teams, чтобы просмотреть разрешения приложений и предоставить согласие [администратора.](app-permissions-admin-center.md)

## <a name="manage-org-wide-app-settings"></a>Управление настройками приложений для всей организации

Используйте параметры приложений для всей организации, чтобы управлять возможностью установки сторонних приложений и возможностью отправки пользовательских приложений в организации и взаимодействия с ними. Параметры приложения в пределах организации определяют поведение всех пользователей и переопределяют любые другие политики разрешений для приложений, назначенных пользователям. Их можно использовать для управления вредоносными или проблемными приложениями.

> [!NOTE]
> Чтобы узнать, как использовать параметры приложений для всей организации в microsoft 365 для государственных организаций — GCC развертываниях Teams, см. управление политиками разрешений приложений [в Teams.](teams-app-permission-policies.md)

1. На странице "Управление приложениями" выберите параметры приложений **для всей организации.** Затем можно настроить необходимые параметры на панели.

    ![Снимок экрана параметров приложения в пределах организации](media/manage-apps-org-wide-app-settings.png)
    
2. В разделе **Сторонние приложения**, отключите и включите эти параметры для управления доступом к сторонним программам:

    - **Разрешить приложениям сторонних разработчиков**: Определяет, могут ли пользователи использовать приложения сторонних разработчиков. Если отключить этот параметр, пользователи не смогут устанавливать и использовать приложения сторонних разработчиков, а их  состояние отображается в таблице как заблокированное в масштабе организации.

        > [!NOTE]
        > Когда **разрешение сторонних приложений** отключено, исходящая веб-часть отключается, то есть пользователи не могут их создавать. [](https://docs.microsoft.com/microsoftteams/platform/webhooks-and-connectors/what-are-webhooks-and-connectors) Если этот параметр включен, исходяющие веб-интерфейсы включены для всех пользователей, и вы можете управлять ими на уровне пользователей, разрешив или блокируя приложение Outgoing Webhook с помощью политик разрешений [приложений.](teams-app-permission-policies.md) <br><br>Обратите внимание, что [](teams-app-permission-policies.md) если у  вас есть существующие политики разрешений для приложений Майкрософт, которые используют параметр "Разрешить определенные приложения" и заблокируют все остальные параметры, и вы хотите включить исходяющие веб-функции для пользователей, добавьте в список приложение Outgoing Webhook. 
    - **Разрешить любые новые сторонние приложения, опубликованные в магазине по умолчанию**: Определяет, будут ли новые сторонние приложения, опубликованные в магазине приложений Teams, автоматически доступны в Teams. Этот параметр доступен только, если разрешено использование приложений сторонних разработчиков.

3. В **области "Пользовательские** приложения" отключите или включите разрешение **взаимодействия с пользовательскими приложениями.** Этот параметр управляет возможностью взаимодействия пользователей с пользовательскими приложениями. Дополнительные сведения см. в статье [Управление политиками и параметрами пользовательских приложений в Teams](teams-custom-app-policies-and-settings.md).
4. Чтобы параметры приложения в пределах организации вступили в силу, выберите пункт **Сохранить**.
