# <a name="overview-of-app-management-and-governance-in-teams-admin-center"></a>Обзор управления приложениями и управления в Центре администрирования Teams

Управление приложениями для организации осуществляется на странице **Приложения Teams** на портале Центра администрирования Microsoft Teams. Страница «Управление приложениями» позволяет просматривать все приложения Teams в каталоге приложений вашей организации и управлять ими, обслуживать основные сценарии управления приложениями, определять доступ к приложениям с помощью политик и т. д.

:::image type="content" source="media/manage-apps.png" alt-text="Снимок экрана: страница «Управление приложениями»." lightbox="media/manage-apps.png":::

Для управления приложениями используются политики, которые позволяют управлять разрешениями для пользователей, установкой приложений и отправкой пользовательских приложений, созданных в организации. Сведения о политиках предоставлены в разделе [Общие сведения о политиках приложений](app-policies.md).

Чтобы использовать Центр администрирования Teams, необходимо иметь роль глобального администратора или администратора Teams. Дополнительные сведения см. в статьях о [ролях администратора Teams](./using-admin-roles.md) и [ролях администратора Microsoft 365](/microsoft-365/admin/add-users/about-admin-roles).

> [!NOTE]
> Страница «Управление приложениями» недоступна в развертываниях Teams в Microsoft 365 для облака сообщества для государственных организаций с высоким уровнем защиты (GCCH) или Министерства обороны (DoD).

## <a name="allow-and-block-apps"></a>Разрешение и блокирование приложений

Как администратор, вы управляете доступом ко всем типам приложений, которые используются во всем контексте всеми пользователями. Teams предоставляет детализированные элементы управления для настройки доступа для каждого приложения и для каждого пользователя.

Чтобы разрешить приложение, необходимо выполнить все следующие параметры. Чтобы заблокировать приложение, заблокируйте его с помощью любого из следующих параметров.

* [Параметры приложений для всей организации](manage-apps.md#manage-org-wide-app-settings). Используйте этот параметр, чтобы разрешить использование приложений в организации. Вы решаете, какие именно приложения используются.
* [Разрешить отдельное приложение](manage-apps.md#allow-and-block-apps). Используйте этот параметр, чтобы разрешить определенное приложение в вашей организации. Вы решаете, какие пользователи могут использовать приложение.
* [Политика разрешений приложений](teams-app-permission-policies.md). Используйте политики, чтобы разрешить или разрешить определенным пользователям использовать приложение. Вы решаете доступ на уровне пользователей и приложений.

На странице «Управление приложениями» можно разрешить или заблокировать отдельные приложения на уровне организации. На странице отображаются все доступные приложения и их текущее состояние на уровне организации. Чтобы разрешить или заблокировать приложение, выполните следующие действия.

1. Вход в Центр администрирования Teams и доступ к **приложениям** >  Teams **[Управление приложениями](https://admin.teams.microsoft.com/policies/manage-apps)**
1. Найдите приложение и выберите его.
1. Выберите параметр **Разрешить** или **Блокировать**.

Сведения о разрешении приложения для определенных пользователей см. в разделе [Политики разрешений приложений](teams-app-permission-policies.md).

Когда разработчик публикует приложение в магазине Teams, некоторым приложениям может потребоваться администратор для настройки приложения. Прежде чем администратор разрешает такое приложение, оно отображается как `Blocked by publisher` в Центре администрирования. Выполнив инструкции издателя по настройке приложения, вы можете сделать его доступным для пользователей, разрешив его.

## <a name="manage-org-wide-app-settings"></a>Управление параметрами приложения в пределах организации

Используйте параметры приложения для всей организации, чтобы контролировать, получают ли пользователи с [лицензией F](https://www.microsoft.com/microsoft-365/enterprise/frontline#office-SKUChooser-0dbn8nt) специализированный интерфейс приложения для сотрудников первой линии, могут ли пользователи устанавливать сторонние приложения и могут ли пользователи передавать пользовательские приложения в вашу организацию или взаимодействовать с ними.

> [!NOTE]
> Сведения об использовании параметров приложений на уровне организации в развертываниях Teams в Microsoft 365 для государственных организаций: облака сообщества для государственных организаций с высоким уровнем защиты (GCCH) и Министерства обороны (DoD) — см. в статье [Управление политиками разрешений приложений в Teams](teams-app-permission-policies.md).

1. На странице **[Управление приложениями](https://admin.teams.microsoft.com/policies/manage-apps)** выберите **Параметры приложений для всей организации**. Затем вы можете настроить необходимые параметры на панели.

    :::image type="content" source="media/manage-apps-org-wide-app-settings.png" alt-text="Снимок экрана: панель параметров приложения в пределах организации на странице «Управление приложениями»":::

1. В разделе **Специализированные приложения** отключите или включите параметр **Показать специализированные приложения**. Если этот параметр включен, пользователи с [лицензией F](https://www.microsoft.com/microsoft-365/enterprise/frontline#office-SKUChooser-0dbn8nt) получают специализированный интерфейс приложения для сотрудников первой линии. Этот интерфейс закрепляет наиболее релевантные приложения в Teams для сотрудников первой линии. Дополнительные сведения см. в статье [Настройка приложений Teams для сотрудников первой линии](pin-teams-apps-based-on-license.md).

    Эта функция доступна для лицензий F. Другие типы лицензий будут поддерживаться в будущем.
1. В разделе **Сторонние приложения**, отключите и включите эти параметры для управления доступом к сторонним программам:

    * **Разрешить приложениям сторонних разработчиков**: Определяет, могут ли пользователи использовать приложения сторонних разработчиков. Если отключить этот параметр, пользователи не смогут устанавливать или использовать сторонние приложения, а их состояние будет отображаться в таблице как **Заблокированное для всей организации**.

        > [!NOTE]
        > Если параметр **Разрешить сторонние приложения** отключен, [исходящие веб-перехватчики](/microsoftteams/platform/webhooks-and-connectors/what-are-webhooks-and-connectors) по-прежнему включены для всех пользователей, но вы можете управлять ими на уровне пользователя, разрешая или блокируя приложение исходящего веб-перехватчика с помощью [политик разрешений приложений](teams-app-permission-policies.md). Обратите внимание, что если у вас есть существующие [политики разрешений](teams-app-permission-policies.md) для **приложений Майкрософт**, которые используют параметр **Разрешить определенные приложения и блокировать все остальные**, и вы хотите включить исходящие веб-перехватчики для пользователей, добавьте в список приложение исходящего веб-перехватчика.

        > [!NOTE]
        > Пользователи Teams могут добавлять приложения, когда они проводят собрания или чаты с пользователями из других организаций. Они также могут использовать приложения, предоставленные пользователями из других организаций, когда они присоединяются к собраниям или чатам, проводимым в этих организациях. Применяются политики данных организации пользователя узла, а также методы совместного использования данных сторонних приложений, предоставленных организацией этого пользователя.

    * **Разрешить любые новые сторонние приложения, опубликованные в магазине по умолчанию**: Определяет, будут ли новые сторонние приложения, опубликованные в магазине приложений Teams, автоматически доступны в Teams. Этот параметр доступен только, если разрешено использование приложений сторонних разработчиков.

1. В разделе **Настраиваемые приложения** отключите или включите параметр **Разрешить взаимодействие с настраиваемыми приложениями**. Этот параметр определяет, могут ли пользователи взаимодействовать с настраиваемыми приложениями. Дополнительные сведения см. в статье [Управление политиками и параметрами для пользовательских приложений](teams-custom-app-policies-and-settings.md).

1. Нажмите **Сохранить**. Параметры вступают в силу через несколько часов.

## <a name="manage-org-wide-app-settings-for-microsoft-365-government"></a>Управление параметрами приложений в пределах организации в Microsoft 365 для государственных организаций  

При развертывании Teams в Microsoft 365 для государственных организаций (GCC, GCCH и DoD) все сторонние приложения по умолчанию блокируются. В облаках GCCH и DoD сторонние приложения недоступны. Кроме того, в GCC на странице политик разрешений приложений в Центре администрирования Microsoft Teams отобразится следующее примечание об управлении сторонними приложениями.

:::image type="content" source="media/app-permission-policies-gcc.png" alt-text="Снимок экрана: политика разрешений приложений в GCCH и DoD.":::

Используйте параметры приложений в пределах организации, чтобы указать, могут ли пользователи устанавливать приложения сторонних разработчиков. Параметры приложения в пределах организации определяют поведение всех пользователей и переопределяют любые другие политики разрешений для приложений, назначенных пользователям.

### <a name="for-gcc-clouds"></a>Для облаков GCC

1. На странице **Приложения** >  Teams **[Управление приложениями](https://admin.teams.microsoft.com/policies/manage-apps)** выберите **Параметры приложений для всей организации**. Затем можно настроить необходимые параметры на панели.

   :::image type="content" source="media/app-permission-policies-gcc-org-wide.png" alt-text="Снимок экрана: параметры приложения для всей организации в GCC.":::

1. В разделе **Сторонние приложения**, отключите и включите эти параметры для управления доступом к сторонним программам:

    * **Разрешить сторонние приложения**. Этот параметр определяет, могут ли пользователи использовать сторонние приложения. Если отключить этот параметр, пользователи не смогут устанавливать или использовать приложения сторонних разработчиков. При развертывании Teams в Microsoft 365 для государственных организаций (GCCH и DoD) этот параметр по умолчанию отключен.
    * **Разрешить любые новые сторонние приложения, опубликованные в магазине по умолчанию**. Этот параметр определяет, будут ли новые сторонние приложения, опубликованные в магазине приложений Teams, автоматически доступны в Teams. Этот параметр доступен только, если разрешено использование приложений сторонних разработчиков.

1. В разделе **Заблокированные приложения** добавьте приложения, которые необходимо заблокировать в организации. При развертывании Teams в Microsoft 365 для государственных организаций (GCCH и DoD) все сторонние приложения по умолчанию добавляются в этот список. Чтобы разрешить стороннее приложение в вашей организации, удалите его из списка заблокированных приложений. При блокировке приложения в пределах организации оно автоматически блокируется для всех пользователей, независимо от того, разрешено ли это приложение в каких-либо политиках разрешений приложений.

1. Чтобы параметры приложения в пределах организации вступили в силу, нажмите **Сохранить**.

Чтобы разрешить сторонние приложения, измените и используйте глобальную политику (по умолчанию для всей организации) или создайте и назначьте политику, созданную администратором.

### <a name="for-gcch-and-dod-clouds"></a>Для облаков GCCH и DoD

1. Войдите в Центр администрирования Teams и получите доступ к **[политикам разрешений](https://admin.teams.microsoft.com/policies/app-permission)** **для приложений** >  Teams.

1. Выберите **Параметры приложения для всей организации**. В разделе **Заблокированные приложения** добавьте приложения, которые необходимо заблокировать в организации. При развертывании Teams в Microsoft 365 для государственных организаций (GCCH и DoD) все сторонние приложения по умолчанию добавляются в этот список.
