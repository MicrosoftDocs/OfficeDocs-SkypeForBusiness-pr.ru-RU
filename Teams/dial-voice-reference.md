# <a name="auto-attendant-and-call-queue-dialing-and-voice-recognition-reference"></a>Справочник по автосекретарю и очереди звонков и распознаванию речи

Dial by Name or Extension — это функция автосекретаря, которая позволяет вызывающим абонентам связаться Teams пользователям в вашей организации. С помощью голосовой или телефонной клавиатуры вызывающие абоненты могут произнести или ввести полное или частичное имя или расширение пользователя, к которому они хотели бы обратиться. Автосекретарь выполнит поиск в каталоге компании, найдите человека и передайте вызывающего абонента.  Параметры dial by Name или Dial by Extension настраиваются при настройке параметров потока вызовов [в автосекретаре](create-a-phone-system-auto-attendant.md?tabs=call-flow).

## <a name="searching-for-users"></a>Поиск пользователей

Teams пользователей, к которым можно получить доступ с помощью dial by Name, не обязательно иметь номер телефона или назначать им планы звонков, но они должны быть Корпоративная голосовая связь для Skype для бизнеса Server **пользователей**. Для нескольких национальных организаций абоненты по имени будут находить и передавать вызывающих абонентов Microsoft Teams пользователям из разных стран или регионов.

Teams пользователей, к которым можно получить доступ с помощью dial by Extension, не должны иметь номер телефона или назначать им планы звонков, но они должны быть Корпоративная голосовая связь для Skype для бизнеса Server **пользователей**. Вам также потребуется соответствующая настроенная абонентской группы для пользователей. Для нескольких национальных организаций функция dial by Extension находит и передает вызывающих абонентов Microsoft Teams пользователям из разных стран или регионов.

Учитывая необходимые условия, набор номера по имени или расширению должен быть явно включен при настройке автосекретаря.

### <a name="maximum-directory-size"></a>Максимальный размер справочника

Количество пользователей Active Directory, на которые можно назначить номера по имени и по расширению, не ограничивается, если вызывающий объект выполняет поиск определенного человека. Вызывающий объект может ввести частичные или полные имена (FirstName + LastName, а также LastName + FirstName), но требует полного номера расширения. Максимальный размер списка имен, поддерживаемый одним автосекретарем с помощью распознавания речи, составляет 80 000 пользователей.
  
|Тип ввода|Формат поиска|Максимальное число пользователей в организации|
|:-----|:-----|:-----|
|DTMF (ввод с клавиатурной панели) |Частичный  <br/> Имя + фамилия  <br/> Фамилия + Имя |Без ограничений  |
|Распознавание речи (голосовой ввод) |Firstname  <br/> Lastname  <br/> Имя + фамилия  <br/> Фамилия + Имя  | 80 000 пользователей |

> [!NOTE]
> Если вы используете dial by Name с распознаванием речи, но Active Directory вашей организации превышает 80 000 пользователей и вы не ограничиваете область набора по имени с помощью функции "Dial by [Name",](create-a-phone-system-auto-attendant.md?tabs=dial-scope) dial by Name по-прежнему будет работать для вызывающих абонентов с помощью клавиатуры телефона, а голосовой ввод будет доступен для всех других сценариев. С помощью функции "Область набора номера" можно сузить имена, которые доступны, изменив область набора по имени для конкретного автосекретаря.

### <a name="search-considerations"></a>Рекомендации по поиску

Сначала выполняется поиск по имени в каталоге всей организации, прежде чем применять настроенные списки включения или исключения области набора. Если при первоначальном поиске по всему каталогу возвращается более 100 пользователей, списки области набора не будут применены, поиск завершится ошибкой, и вызывающего абонента будет сказано, что найдено слишком много имен.

## <a name="dial-by-name---keypad-dtmf-entry"></a>Звонок по имени  ввод с клавиатурной панели телефона (DTMF)

Пользователи, вызывающие абонентов, могут использовать dial by Name для доступа к пользователям, указав полное или частичное имя пользователя, к которому они пытаются связаться. Существуют различные форматы, которые можно использовать при указании имени.

При поиске в каталоге организации пользователи могут использовать клавишу "0" (ноль), чтобы указать пробел между именем, фамилией или фамилией и первым. При вводе имени ему будет предложено завершить запись клавиатуры с помощью ключа #. Например, "После ввода имени пользователя, которого вы пытаетесь связаться, нажмите #". Если найдено несколько имен, вызывающему пользователю будет предоставлен список имен для выбора.

> [!NOTE]
> Если после применения списков включения или исключения области набора остается более 5 имен, поиск завершится ошибкой, и вызывающей стороне будет предложено найти слишком много имен.
  
Поддерживаются следующие форматы поиска имен в организации с использованием клавиатурной панели телефона.
  
|Формат имени|Тип поиска|Пример|Результат поиска|
|:-----|:-----|:-----|:-----|
|Имя + фамилия |Полный  |Иван0Макаренко# |Иван Макаренко |
|Фамилия + Имя |Полный |Макаренко0Иван#  |Иван Макаренко |
|Firstname  |Полный   |Иван#   |Нажмите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите 2 для связи с пользователем "Иван Марченко" |
|Lastname |Полный |Макаренко#  |Нажмите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите 2 для связи с пользователем "Мария Макаренко" |
|Имя или фамилия |Частичный |Ма# |Нажмите 1 для связи с пользователем "Мария Макаренко"  <br/> Нажмите 2 для связи с пользователем "Мария Петрова"  <br/> Нажмите 3 для связи с пользователем "Иван Марченко" |
|Имя + Фамилия |Частичный |Amos0Mar # |Нажмите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите 2 для связи с пользователем "Иван Марченко" |
|Фамилия + Имя |Частичный |Ма0Ив# |Нажмите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите 2 для связи с пользователем "Иван Марченко" |

Существует несколько специальных символов, которые используются при поиске людей, использующих клавиатуру телефона. Например, пользователю будет предложено использовать ключ решетки (#), а для пробела между именами используется ноль (0). Нажатие клавиши star (*) повторит список совпадающих имен с пользователем.
  
|Специальный знак на клавиатуре телефона|Описание|
|:-----|:-----|
|#   |Завершение ввода имени. |
|0   |Пробел между именами. |
|*    |Повторное воспроизведение списка имен пользователей, соответствующих условиям поиска. |

### <a name="dial-by-name---name-recognition-with-speech"></a>Звонок по имени  распознавание имен при голосовом вводе

Пользователи могут искать других пользователей в своей организации с помощью голоса (распознавания речи). Они также могут связаться с любым пользователем в Active Directory, выполнив полное или частичное имя человека, которого он пытается найти. С помощью голосовых входных данных можно распознавать имена в различных форматах, включая FirstName, LastName, FirstName + LastName или LastName + FirstName.
  
Вы можете включить распознавание речи для автосекретаря, но запись на клавиатуре телефона (DTMF) не отключена. Телефон клавиатуры можно использовать в любое время, даже если в автосекретаре включено распознавание речи.
  
Как и в случае с записью клавиатуры телефона, при обнаружении нескольких имен вызывающий пользователь услышит список имен для выбора.

> [!NOTE]
> Если после применения списков включения или исключения области набора остается более 5 имен, поиск завершится ошибкой, и вызывающей стороне будет предложено найти слишком много имен.
  
Вызывающие абоненты могут произнести имена в следующих форматах:
  
|Имя с речью|Тип поиска|Пример|Результат поиска|
|:-----|:-----|:-----|:-----|
|Имя + фамилия |Полный |Иван Макаренко |Иван Макаренко |
|Фамилия + Имя |Полный  |Макаренко Иван |Иван Макаренко |
|Firstname |Полный |Иван |Нажмите или произнесите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите или произнесите 2 для связи с пользователем "Иван Петров" |
|Lastname |Полный |Макаренко |Нажмите или произнесите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите или произнесите 2 для связи с пользователем "Сергей Макаренко" |
|Имя или фамилия |Частичный |Мар |Нажмите или скажите 1 для Марии Шарик  <br/> Нажмите или скажите "2" для Mary Jones  <br/> Нажмите или скажите "3" для Амоса Макла (Amos Close) |
|Имя + Фамилия |Частичный |Амос Мар (Amos Mar) |Нажмите или произнесите 1 для связи с пользователем "Иван Макаренко"  <br/> Нажмите или скажите 2 для Амоса Муамя |

> [!NOTE]
> Для того чтобы имя нового пользователя было указано в каталоге dial by Name с распознаванием речи из-за задержки репликации Active Directory, может потребоваться до 36 часов.

### <a name="dial-by-extension"></a>Набор по расширению

Пользователи, которых вы хотите сделать доступными для расширения **Dial By,** должны иметь расширение, указанное как часть одного из следующих атрибутов телефона, определенных в Active Directory (и синхронизированных с помощью Azure AD Подключение) или Azure Active Directory. ( [Дополнительные сведения см](/microsoft-365/admin/add-users/add-users) . в разделе "Добавление пользователей по отдельности или массово".)

- OfficePhone/PhoneNumber (AD и Azure AD)
- HomePhone (AD)
- Mobile/MobilePhone (AD и Azure AD)
- OtherTelephone (AD)

Для ввода дополнительного номера в поле номера телефона пользователя может требоваться один из следующих форматов.

- *+\<phone number>;ext=\<extension>*
- *+\<phone number>x\<extension>*
- *x\<extension>*

- Пример 1: Set-MsolUser -UserPrincipalName usern@domain.com -Phonenumber "+15555555678;ext=5678"
- Пример 2: Set-MsolUser -UserPrincipalName usern@domain.com -Phonenumber "+15555555678x5678"
- Пример 3: Set-MsolUser -UserPrincipalName usern@domain.com -Phonenumber "x5678"

Вы также можете настроить дополнительный номер в [Центре администрирования Microsoft 365](https://admin.microsoft.com/) или [Центре администрирования Azure Active Directory](https://aad.portal.azure.com). Может пройти до 12 часов, прежде чем изменения станут доступны автосекретарям и очередям вызовов.
