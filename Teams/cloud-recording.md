# <a name="teams-cloud-meeting-recording"></a>Запись облачного собрания в Teams

В Microsoft Teams пользователи могут записывать свои собрания Teams и групповые звонки. Можно записывать звук, видео и действия демонстрации экрана. Существует возможность записи с автоматической расшифровкой аудиозаписей, чтобы пользователи могли воспроизводить записи собрания с субтитрами и просматривать важные элементы обсуждения в расшифровке. Запись происходит в облаке и сохраняется в Microsoft OneDrive для бизнеса и Microsoft SharePoint Online, поэтому пользователи могут безопасно предоставлять общий доступ к ней в своей организации.

При записи собрания автоматически выполняются следующие действия:

- отправка в OneDrive для бизнеса или SharePoint Online;
- предоставление разрешений пользователям, приглашенным на собрание;
- размещение ссылки в чате собрания;
- отображение на вкладке "Записи и расшифровки" собрания в календаре Teams;
- добавление в различные списки файлов в Microsoft 365: "Мне предоставлен доступ", office.com, "Рекомендуемые", "Недавние";
- индексация для поиска Microsoft 365.

Связанные: [документация конечного пользователя записи собрания Teams](https://support.microsoft.com/en-us/office/record-a-meeting-in-teams-34dfbe7f-b07d-4a27-b4c6-de62f1348c24)

>[!Note]
> Замена классической версии Microsoft Stream на OneDrive для бизнеса и SharePoint Online в качестве инструмента для записи собраний произойдет автоматически в августе 2021 года. Подробнее: [Использование OneDrive для бизнеса, SharePoint Online или Stream для записи собраний](tmr-meeting-recording-change.md)

> [!NOTE]
> Сведения об использовании ролей в собраниях Teams и их изменении см. в статье [Роли в собрании Teams](https://support.microsoft.com/office/roles-in-a-teams-meeting-c16fa7d0-1666-4dde-8686-0a0bfe16e019). Параметры записи трансляций см. в статье [Политики записи трансляций в Teams](teams-live-events/live-events-recording-policies.md).

## <a name="prerequisites-for-teams-cloud-meeting-recording"></a>Предварительные условия для записи облачных собраний Teams

Для записи собраний пользователя Teams для клиента необходимо включить OneDrive для бизнеса и SharePoint Online. Кроме того, организатор собрания и лицо, инициирующее запись, должны отвечать следующим условиям:

- В OneDrive для бизнеса пользователя достаточно места для сохранения записей собраний без канала.

- В SharePoint Online канала Teams достаточно места для сохранения записей собраний.

- Пользователь установил для параметра `CsTeamsMeetingPolicy -AllowCloudRecording` значение true, чтобы записывать собрания и групповые звонки.

- Пользователь установил для параметра `CsTeamsCallingPolicy -AllowCloudRecordingForCalls` значение true, чтобы записывать частные звонки.

- Пользователь не является анонимным, гостевым или федеративным пользователем на собрании.

- Чтобы включить транскрибирование, в назначенной пользователю политике собраний Teams для параметра `-AllowTranscription` должно быть установлено значение true.

- Чтобы записи собраний канала сохранялись, но участники канала не могли их редактировать или скачивать, для параметра `CSTeamsMeetingPolicy -ChannelRecordingDownload` необходимо установить значение Block.

> [!IMPORTANT]
>
> OneDrive для бизнеса или SharePoint Online не нужно включать, если вы хотите, чтобы пользователи только записывали и скачивали записи. В таком случае записи будут храниться не в OneDrive для бизнеса или SharePoint Online, а во временном хранилище Teams на протяжении 21 дня, а затем будут удалены. В этом случае администратор не может ни управлять записями, ни удалять их.
>
> [Подробнее о том, как работает временное хранилище записей собраний](#temp-storage), читайте ниже.  

## <a name="set-up-teams-cloud-meeting-recording-for-users-in-your-organization"></a>Настройка записи облачных собраний Teams для пользователей в вашей организации

В этом разделе объясняется, как вы можете настроить и запланировать запись собраний Teams через [политики Teams](policy-assignment-overview.md).

### <a name="turn-on-or-turn-off-cloud-recording"></a>Включить или отключить облачную запись

Вы можете использовать центр администрирования Microsoft Teams или PowerShell, чтобы установить политику собраний команд, чтобы контролировать возможность записи собраний пользователей.

В Центре администрирования Microsoft Teams включите или выключите параметр **Запись в облаке** в политике собраний. Подробнее: [Параметры политики собраний для аудио- и видеофайлов](meetings-policies-recording-and-transcription.md#allow-cloud-recording).

Используя PowerShell, вы настраиваете параметр AllowCloudRecording в TeamsMeetingPolicy. Чтобы узнать больше, см. [New-CsTeamsMeetingPolicy](/powershell/module/skype/new-csteamsmeetingpolicy)и [Set-CsTeamsMeetingPolicy](/powershell/module/skype/set-csteamsmeetingpolicy).

Как организатор собрания, так и инициатор записи должны иметь разрешения на запись для записи собрания. Если пользователь не назначил пользовательскую политику, пользователи получают глобальную политику, в которой AllowCloudRecording включен по умолчанию.

> [!NOTE]
> Дополнительные сведения об использовании ролей Teams для настройки разрешений для записи собрания см. в статье [Роли в собрании Teams](https://support.microsoft.com/office/roles-in-a-teams-meeting-c16fa7d0-1666-4dde-8686-0a0bfe16e019).

Чтобы пользователь мог вернуться к глобальной политике, используйте следующий командлет, чтобы удалить конкретное назначение политики для пользователя:

```powershell
Grant-CsTeamsMeetingPolicy -Identity {user} -PolicyName $null -Verbose
```

Чтобы изменить значение AllowCloudRecording в Глобальной политике, используйте следующий командлет:

```powershell
Set-CsTeamsMeetingPolicy -Identity Global -AllowCloudRecording $true
```

### <a name="block-or-allow-download-of-channel-meeting-recordings"></a>Как запретить или разрешить скачивание записей собраний канала

Этот параметр определяет, сохраняются ли собрания канала в папке "Recordings" или в папке "Recordings\View only" в канале. Этот параметр применяется к политике пользователя, который выбирает запись для собрания канала.

Возможные значения этого параметра:

- **Allow** (по умолчанию) — сохраняет записи собраний канала в папку "Recordings". Разрешения для файлов записи определяются с учетом разрешений SharePoint Online канала. Это же правило применяется ко всем другим отправляемым файлам.

- **Block** — сохраняет записи собраний канала в папку "Recordings\View only". У владельцев канала будут полные права на записи в этой папке, а у участников — доступ для чтения без возможности скачивания.

Используя PowerShell, вы настраиваете параметр ChannelRecordingDownload в TeamsMeetingPolicy. Подробнее: [New-CsTeamsMeetingPolicy](/powershell/module/skype/new-csteamsmeetingpolicy)и [Set-CsTeamsMeetingPolicy](/powershell/module/skype/set-csteamsmeetingpolicy).

Если вы не назначите особую политику, пользователи получат глобальную политику, в которой для параметра ChannelRecordingDownload по умолчанию установлено значение Allow.

Чтобы пользователь мог вернуться к глобальной политике, используйте следующий командлет, чтобы удалить конкретное назначение политики:

```powershell
Grant-CsTeamsMeetingPolicy -Identity {user} -PolicyName $null -Verbose
```

Чтобы изменить значение ChannelRecordingDownload в глобальной политике, используйте следующий командлет:

```powershell
Set-CsTeamsMeetingPolicy -Identity Global -ChannelRecordingDownload Block
```

> [!NOTE]
> Параметр ChannelRecordingDownload доступен только в модуле Teams PowerShell версии 2.4.1-preview или более поздней. Чтобы скачать последнюю версию модуля, воспользуйтесь этой командой:
>
>```powershell
>Install-Module -Name MicrosoftTeams -Force -AllowClobber -AllowPrerelease
>```

### <a name="turn-on-or-turn-off-recording-transcription"></a>Включить или отключить запись транскрипции
Этот параметр определяет, доступны ли функции субтитров и транскрипции во время воспроизведения записей собрания. Этот параметр должен быть включен у пользователя, который начал запись, чтобы эти функции поддерживались в его записи.
  
При включении этого параметра создается копия расшифровки, которая хранится с записью собрания, что включает функции **поиска**, **копии** и **расшифровки** для записи собрания.

## <a name="permissions-and-storage"></a>Разрешения и хранение

Записи собраний хранятся в облачном хранилище OneDrive для бизнеса и SharePoint Online. Расположение и разрешения зависят от типа собрания и роли пользователя. Ниже перечислены разрешения по умолчанию, применяемые к записи. Пользователи, обладающие полными правами на редактирование файла видеозаписи, могут изменять разрешения и предоставлять общий доступ другим пользователям.

### <a name="non-channel-meetings"></a>Собрания без канала

- Запись хранится в папке **Recordings** в OneDrive для бизнеса пользователя, запустившего запись. 

  Пример: <i>OneDrive для бизнеса</i>/**Recordings**

- Всем приглашенным на собрание, за исключением внешних пользователей, автоматически будет предоставлено разрешение на доступ к файлу записи без возможности скачивания.

- Организатор собрания и пользователь, запустивший запись, получат полный доступ для редактирования с возможностью изменять разрешения и предоставлять общий доступ другим пользователям.

### <a name="channel-meetings"></a>Собрания канала

Если для параметра `Set-CsTeamsMeetingPolicy -ChannelRecordingDownload` установлено значение Allow (по умолчанию):

- Запись хранится в библиотеке документации сайта Teams в папке **Recordings**.

  Пример: <i>Название Teams - Название канала</i>/**Documents**/**Recordings**

- Участник, запустивший запись, имеет права на редактирование записи.

- Разрешения остальных участников основаны на разрешениях SharePoint Online канала.

Если для параметра `Set-CsTeamsMeetingPolicy -ChannelRecordingDownload` установлено значение Block:

- Запись хранится в библиотеке документации сайта Teams в папке **Recordings/View only**. 

  Пример: <i>Название Teams - Название канала</i>/**Documents/Recordings/View only**

- Владельцы канала будут иметь полные права на редактирование и скачивание записей в этой папке.

- У участников канала будет только доступ для просмотра без возможности скачивания.

<a name="temp-storage"></a>
### <a name="temporary-storage-when-unable-to-upload-to-onedrive-for-business-and-sharepoint-online"></a>Временное хранилище, если отправка в OneDrive для бизнеса и SharePoint Online невозможна

Если запись собрания не удается отправить в OneDrive для бизнеса и SharePoint Online, она будет временно доступна для скачивания из Teams в течение 21 дня, а затем удалена. В этом случае администратор не может ни управлять записями, ни удалять их.

Записи собраний могут оказаться в этом временном хранилище по следующим причинам:

- Собрания без канала — если учетная запись OneDrive для бизнеса не настроена или исчерпана квота хранилища
- Собрания канала — если исчерпана квота хранилища на сайте SharePoint Online или сайт не готов
- Если политики OneDrive для бизнеса и SharePoint Online разрешают пользователям отправлять файлы только из определенных диапазонов IP-адресов и т. д.

Хранение записи во временном хранилище зависит от самого сообщения чата. Таким образом, удаление исходного сообщения чата не позволит пользователям получить доступ к записи. На это могут повлиять два сценария:

- **Пользователь вручную удаляет сообщение чата.** В этом сценарии, поскольку исходное сообщение исчезло, пользователи больше не смогут получить доступ к записи, и дальнейшие скачивания будут невозможны. Однако сама запись может храниться во внутренних системах Microsoft в течение некоторого времени (не более первоначального 21-дневного периода).

- **Запись сообщения чата удаляется политикой хранения.** Записи временного хранилища напрямую связаны с политикой хранения чата. Таким образом, хотя записи во временном хранилище Teams по умолчанию хранятся 21 день до удаления, если сообщение чата будет удалено раньше, из-за политики хранения сообщений чата запись также будет удалена. После этого восстановить запись будет невозможно.

### <a name="planning-for-storage"></a>Планирование хранения

Размер 1-часовой записи составляет 400 МБ. Определите емкость, необходимую для записанных файлов, и убедитесь, что в OneDrive для бизнеса и SharePoint Online достаточно места для хранения.  О базовом объеме, включенном в подписку, и приобретении дополнительного объема читайте в статьях [Настройка стандартного размера хранилища OneDrive для бизнеса](/onedrive/set-default-storage-space) и [Управление ограничениями хранилища для сайтов SharePoint Online](/sharepoint/manage-site-collection-storage-limits).

 <a name="auto-expiration"></a>
### <a name="auto-expiration-of-teams-meeting-recordings"></a>Автоматическое окончание срока действия записей собраний Teams

Дополнительные сведения об изменениях, касающихся администратора, можно получить [здесь](meeting-expiration.md#changes-to-meeting-expiration).

Дополнительные сведения о том, как конечные пользователи могут управлять сроком действия собраний, см. [здесь](https://support.microsoft.com/office/record-a-meeting-in-teams-34dfbe7f-b07d-4a27-b4c6-de62f1348c24#bkmk_view_change_expiration_date).
  
Ознакомьтесь с часто задаваемыми вопросами для администраторов и конечных пользователей, чтобы получить представление о том, как будет работать автоматическое истечение срока действия записей собраний Teams, какие действия можно предпринять сейчас и какие действия можно предпринять после запуска функции.
  
## <a name="frequently-asked-questions"></a>Вопросы и ответы

**Что изменилось?**
  
Мы вводим 60-дневный срок действия по умолчанию для всех вновь созданных записей собраний Teams (TMR). Это означает, что по умолчанию все TMRs, созданные после включения этой функции, будут удалены через 60 дней после даты их создания. Если администраторы хотят, чтобы срок действия записей собраний истекал раньше или позже, чем указано по умолчанию, они могут изменить параметр срока действия. Системы OneDrive и SharePoint будут отслеживать дату окончания срока действия, установленную для всех записей собраний, и автоматически перемещать их в корзину по истечении срока их действия.

**Кого это коснется?**
  
Любой, кто хранит запись собрания Teams (неканальное, канальное или специальное собрание) в OneDrive или SharePoint.

**Зачем использовать эту функцию?**
  
Эту функцию следует использовать для ограничения хранилища OneDrive или SharePoint, используемого записями собраний Teams (примечание: они обычно используют около 400 МБ на час записи).
  
**Зачем мы вносим это изменение?**
  
Клиенты предоставили огромное количество отзывов о том, что им нужны дополнительные элементы управления, чтобы уменьшить беспорядок в хранилище, создаваемый записями собраний Teams, 99% из которых, в среднем, никогда не пересматриваются через 60 дней.
  
**Почему этот параметр включен по умолчанию?**
  
Мы полагаем, что почти все клиенты выиграют от уменьшения нагрузки на хранилище для своих клиентов, удалив записи, которые, скорее всего, никогда не будут пересмотрены через 60 дней. Наша цель — обеспечить максимально удобные возможности работы для всех клиентов по умолчанию.
  
**Произойдет ли автоматическое удаление, даже если к данным получили доступ или скачали их?**
  
При доступе к файлу дата окончания срока действия не меняется.
  
**Отображается ли дата окончания срока действия в виде столбца в списке?**

Пользователи с доступом на просмотр записи увидят красный значок рядом с файлом в папке OneDrive или SharePoint за 14 дней до истечения срока действия файла. В настоящее время добавить столбец в список с датой окончания срока действия нельзя.
  
**Как рассчитывается дата окончания срока действия?**
  
Дата окончания срока действия рассчитывается как день создания записи собрания плюс количество дней по умолчанию, установленное администратором в параметрах Teams.
  
**Можно ли изменить дату истечения срока действия для каждого TMR, например, дата истечения срока действия данных А составляет 30 дней, а дата истечения срока действия данных Б — 60 дней?**

Да, срок действия задается для каждого файла отдельно. Изменить срок действия можно в области сведений выбранного файла в OneDrive или SharePoint.

**Как администратор может изменить дату окончания срока действия?**
  
Администраторы могут изменить стандартный срок действия в PowerShell или Центре администрирования Teams до выпуска этой функции. Изменение параметров срока действия повлияет только на вновь созданные записи собраний Teams с этого момента. Это не повлияет на записи, созданные до этой даты. Срок действия новых записей не будет истекать автоматически до выпуска функции несмотря на то, что вы можете настроить атрибут политики до ее выпуска.

Количество дней срока действия можно установить указанным ниже образом.
  
- Допустимое значение: от 1 до 9 999.
- Также может применяться значение -1. В этом случае срок действия TMR никогда не истечет. 
 
Администраторы не могут изменить дату окончания срока действия существующих записей собрания Teams, уже загруженных в OneDrive или SharePoint, до выпуска этой функции. Это защищает намерение пользователя, который владеет записью собрания Teams.
  
Чтобы изменить поведение автоматического окончания срока действия по умолчанию для клиента, измените следующий атрибут в PowerShell. В этом примере параметр по умолчанию изменен на значение 50 дней.
 
Set-CsTeamsMeetingPolicy -Identity Global -**New** MeetingRecordingExpirationDays 50

Возможность изменения параметра по умолчанию в Центре администрирования Teams будет развернута позже (не менее чем за 30 дней до включения функции автоматического окончания срока действия по умолчанию).
  
**Может ли администратор задать неограниченный срок действия для TMR?**
  
 Да, администраторы могут задать неограниченный срок действия для TMR.
  
**Меняет ли воспроизведение записи дату окончания срока действия?**

Нет, воспроизведение не влияет на дату окончания срока действия.
  
**Что произойдет с датой окончания срока действия, если TMR будет загружен и повторно загружен?**

Дата истечения срока действия будет удалена при повторной загрузке независимо от SKU пользователя.
  
**Что произойдет, если скопировать или переместить TMR в другое расположение или сайт?**

Дата сохраняется только для перемещенного TMR-файла. Скопированные файлы не будут иметь дату окончания срока действия, как у повторно загруженных TMR.
  

**Каковы возможности управления политикой администратора?**
  
Как собраниями, так и звонками будет управлять один и тот же параметр `CsTeamsMeetingPolicy`, `MeetingRecordingExpirationDays`. 
  
**Как конечные пользователи могут изменить дату окончания срока действия определенного файла записи собрания Teams?**
  
Любой, у кого есть разрешения на изменение и удаление записей собрания Teams, может изменить дату истечения срока действия на панели сведений о файле в OneDrive или SharePoint.

Пользователь может отложить истечение срока действия на 14, 30 или 60 дней, или они могут выбрать конкретную дату в будущем, или они могут выбрать, что срок действия файла никогда не истечет.
  
**Должны ли администраторы использовать эту функцию для обеспечения строгой безопасности и соблюдения нормативных требований?**
  
Нет, администраторам не следует полагаться на эту функцию для юридической защиты, так как конечные пользователи могут изменять дату окончания срока действия любых записей, которыми они управляют.
  
**Будет ли эта функция обеспечивать сохранение файлов?**
  
Нет, файлы не будут сохранены из-за этой функции или ее параметров. Если пользователь с разрешениями на удаление пытается удалить запись собрания Teams, для которого установлен срок действия, будет выполнено действие удаления этого пользователя.

**Будет ли политика хранения и /или удаления, заданная в Центре безопасности и соответствия требованиям (S+C), переопределять параметр срока действия записи собрания Teams?**
  
Да, все политики, установленные в центре S+C, имеют полный приоритет. Например:
  
- Если у вас есть политика, согласно которой все файлы на сайте должны храниться в течение 100 дней, а срок действия записей собрания Teams составляет 30 дней, то файл записи будет храниться в течение полных 100 дней.  
- Если в политике удаления говорится, что все записи собрания Teams будут удалены через 5 дней, а для файла записи установлено ограничение в 30 дней, этот файл будет удален через пять дней.

**Что происходит после истечения срока действия записи собрания Teams?**
  
По истечении срока действия записи собрания Teams перемещается в корзину OneDrive или SharePoint, а поле даты истечения срока действия очищается. Это действие системы точно такое же, как при удалении файла пользователем. В дальнейшем жизненный цикл корзины будет следовать обычному пути. Если пользователь восстанавливает запись собрания Teams из корзины, запись собрания Teams не будет снова удалена этой функцией, поскольку дата истечения срока действия была очищена, если конечный пользователь не установит новую дату окончания срока действия файла.
  
**Как я узнаю об окончании срока действия файла?**
  
Все, у кого есть доступ к представлению, увидят уведомление о дате окончания срока действия в командной ленте в окне чата Teams.
  
Все, у кого есть доступ к представлению, увидят красный значок рядом с файлом в вашей папке OneDrive или SharePoint за 14 дней до истечения срока действия файла.
  
Владелец файла получит уведомление по электронной почте по истечении срока действия записи собрания Teams и будет направлен в корзину для восстановления записи собрания Teams, если он этого хочет.
  
**Какие номера SKU требуются для этой функции?**
  
Эта функция будет по умолчанию у всех номеров SKU. Для пользователей A1 по умолчанию будет установлен 30-дневный срок действия.
  
**Является ли истечение срока действия файла управляемый событием, и я смогу увидеть его в своих журналах аудита?**
  
Да, срок действия файлов будет отображаться как события удаления системы в журнале аудита.
  
**Что, если я хочу, чтобы администратор имел полный контроль над жизненным циклом записи собрания Teams и не хочу давать конечным пользователям возможность изменять дату окончания срока действия?**
  
Рекомендуем использовать политики сохранения и удаления S+C, доступные как часть SKU соответствия требованиям E5. Это предложение предназначено для решения сложных юридических проблем, связанных с политикой и соглашением об уровне обслуживания.

Эта функция предназначена исключительно в качестве простого вспомогательного механизма для уменьшения беспорядка в хранилище, создаваемого холодными записями собраний Teams.
  
**Когда файл будет удален?**
  
Файл будет удален в течение 5 дней с даты окончания срока действия, хотя это не является строгой гарантией.
  
**Будет ли в будущих записях собраний Teams, перенесенных из классической потоковой передачи после выпуска этой функции, применяться автоматическое окончание срока действия?**
  
Нет, для перенесенных записей собраний Teams не будет установлен срок действия. Вместо этого мы рекомендуем администраторам переносить только те записи собраний Teams, которые они хотят сохранить. Дополнительные сведения будут предоставлены в документации по миграции.
  
## <a name="manage-meeting-recordings"></a>Управление записями собраний

Записи собраний хранятся в виде видеофайлов в OneDrive для бизнеса и SharePoint Online, поэтому для управления ими используются процессы, доступные на этих платформах. Подробнее: [Обзор управления SharePoint Online](/sharepoint/governance-overview), [OneDrive для бизнеса: руководство для предприятий](/onedrive/plan-onedrive-enterprise) или [OneDrive для бизнеса: руководство для малых предприятий](/onedrive/one-drive-quickstart-small-business).

Записи собраний без канала хранятся в OneDrive для бизнеса записывающего сотрудника, поэтому к правам владельца и хранению после его увольнения применяются действующие [в OneDrive для бизнеса и SharePoint Online правила](/onedrive/retention-and-deletion#the-onedrive-deletion-process).

## <a name="closed-captions-for-recordings"></a>Субтитры для записей

Субтитры для записей собраний Teams будут доступны во время воспроизведения, только если во время записи у пользователя было включено транскрибирование. Администраторы должны [включить транскрибирование записей с помощью политики](#turn-on-or-turn-off-recording-transcription), чтобы пользователи могли записывать собрания с транскрибированием.

Субтитры помогают создавать инклюзивные материалы для всех зрителей. Владелец может скрыть субтитры в записи собрания, хотя расшифровка собрания по-прежнему будет доступна в Teams, если вы не удалите ее там.

Сегодня субтитры для видеофайла записи связаны с расшифровкой собрания Teams. В большинстве случаев эта связь сохраняется на протяжении всего времени существования файла, но может быть удалена, если видеофайл будет скопирован на том же сайте OneDrive для бизнеса или SharePoint Online, что приведет к исчезновению субтитров из скопированного видеофайла.

Разъяснения любых будущих изменений связи между расшифровкой в Teams и записью будут приведены здесь и в уведомлениях центра сообщений. В случае внесения каких-либо изменений в будущем, мы позаботимся о том, чтобы в записях, сделанных не позднее 60 дней назад, отображалась расшифровка собрания в виде субтитров.

> [!NOTE]
> Расшифровка собраний пока недоступна в GCC.

## <a name="ediscovery-and-compliance-for-meeting-recordings"></a>Обнаружение электронных данных и соответствие требованиям для записей собраний

### <a name="ediscovery"></a>Обнаружение электронных данных

Записи собраний хранятся в OneDrive для бизнеса и SharePoint Online, которые совместимы с Microsoft 365 и Office 365 Tier-D. Для поддержки запросов e-Discovery для администраторов соответствия, которые заинтересованы в записях собраний или вызовов, сообщение о завершении записи доступно в функции поиска контента соответствия для Microsoft Teams. Администраторы соответствия могут искать ключевое слово «запись» в строке темы элемента в предварительном просмотре поиска содержимого соответствия и обнаруживать записи встреч и вызовов в организации.

Кроме того, видеофайл записи собрания можно найти с помощью поиска файлов в SharePoint Online и OneDrive для бизнеса.

Дополнительную информацию об обнаружении электронных данных см. в [этой статье](/microsoft-365/compliance/ediscovery)

### <a name="retention-policies"></a>Политики хранения

С помощью свойства ProgID можно применять автоматические метки хранения только к видеофайлам записи собраний Teams. Дополнительную информацию см. в [этой статье](/microsoft-365/compliance/apply-retention-labels-automatically#microsoft-teams-meeting-recordings).

### <a name="data-loss-prevention-dlp-policies"></a>Политики защиты от потери данных (DLP)

С помощью свойства ProgID к файлам записи собраний также можно применять политики DLP. В правиле DLP для файлов в SharePoint Online и OneDrive для бизнеса установите условия:

- Свойство документа = *ProgID*
- Значение = *Media.Meeting*

Дополнительные сведения о DLP см. в [этой статье](/microsoft-365/compliance/dlp-learn-about-dlp)

## <a name="meeting-recording-diagnostic-tools"></a>Средства диагностики записей собраний
  ### <a name="user-cannot-record-meetings"></a>Пользователь не может записывать собрания

Если вы администратор, вы можете использовать следующее средство диагностики, чтобы проверить, правильно ли настроен пользователь для записи собрания в Teams:

1. Выберите **Выполнить тесты** ниже, чтобы заполнить диагностику в Центре администрирования Microsoft 365. 

   > [!div class="nextstepaction"]
   > [Выполнение тестов: запись собрания](https://aka.ms/MeetingRecordingDiag)

2. В области диагностики "Выполнение" введите адрес электронной почты пользователя, который не может записывать собрания в поле **Имя пользователя или адрес электронной почты** и выберите **Выполнить тесты**.

3. Тесты возвращают наилучшие дальнейшие действия с любыми конфигурациями клиента или политики, чтобы проверить, правильно ли настроен пользователь для записи собрания в Teams.
  
  ### <a name="meeting-record-is-missing"></a>Отсутствует запись собрания

Если вы администратор, вы можете использовать следующее средство диагностики для проверки успешного завершения записи собрания и ее отправки в Stream или OneDrive на основе ИД собрания и времени начала записи:

1. Выберите **Выполнить тесты** ниже, чтобы заполнить диагностику в Центре администрирования Microsoft 365. 

   > [!div class="nextstepaction"]
   > [Выполнение тестов: отсутствует запись собрания](https://aka.ms/MissingRecordingDiag)

2. В области диагностики "Выполнить" введите URL-адрес собрания в поле **URL-адрес записанного собрания** (обычно находится в приглашении на собрание), а также дату собрания в поле **Когда собрание было записано?** и выберите **Выполнить тесты**.

3. В тестах будет проверено, успешно ли завершена запись собрания и была ли она загружена в Stream или OneDrive.
