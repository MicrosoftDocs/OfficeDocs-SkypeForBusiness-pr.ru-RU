# <a name="teams-cloud-meeting-recording"></a>Запись собрания Teams в облаке

В Microsoft Teams пользователи могут записывать собрания команд и групповые звонки для захвата активности звука, видео и демонстрации экрана. Существует возможность записи с автоматической расшифровкой аудиозаписей, чтобы пользователи могли воспроизводить записи собрания с субтитрами и искать важные элементы обсуждения в расшифровке. Запись выполняется в облаке и сохраняется в [Microsoft Stream](https://docs.microsoft.com/stream/), так что пользователи могут безопасно поделиться ею в рамках своей организации.

Дополнительные действия: [Документация для конечных пользователей по записи собраний Teams](https://aka.ms/recordmeeting)

## <a name="prerequisites-for-teams-cloud-meeting-recording"></a>Предварительные требования для записи облачных собраний Teams

Для заносимых собраний пользователей Teams необходимо включить Microsoft Stream для клиента. Кроме того, для организатора собрания и человека, инициирующего запись, необходимы следующие предварительные требования:

- У пользователя есть Office 365 E1, E3, в ячейках, a1, A3, A5, M365 Business или Business Premium
- Пользователь должен быть лицензирован для Microsoft Stream<sup>1</sup> 
- У пользователя есть разрешения на отправку видео из Microsoft Stream
- Пользователь был в соответствии с рекомендациями компании, если он настроен администратором
- У пользователя есть достаточно места для сохранения записей в Microsoft Stream.
- Для параметра User (Теамсмитингполици-Алловклаудрекординг) задано значение true
- Пользователь не является анонимным, гостевым или федеративным пользователем в собрании

> [!NOTE]
> Кроме того, чтобы разрешить пользователю, запускающему запись, указать, нужно ли автоматически затем переносите запись, параметр Теамсмитингполици-Алловтранскриптион пользователя должен иметь значение true.

<sup>1</sup> Пользователь должен быть лицензирован для отправки и скачивания собраний в Microsoft Stream, но ему не требуется лицензия на запись собрания. Если вы хотите запретить пользователю запись собрания Microsoft Teams, вы должны предоставить Теамсмитингполици, у которого Алловклаудрекординг установлен на $False.

## <a name="set-up-teams-cloud-meeting-recording-for-users-in-your-organization"></a>Настройка записи облачных собраний Teams для пользователей в Организации

В этом разделе объясняется, как настраивать и планировать собрания групп.

### <a name="enable-microsoft-stream-for-users-in-the-organization"></a>Включение Microsoft Stream для пользователей в Организации

Microsoft Stream доступен как часть подходящих подписок на Office 365 или как самостоятельная служба.  Дополнительные сведения о [лицензировании Stream](https://docs.microsoft.com/stream/license-overview) см.  Microsoft Stream теперь включен в Microsoft 365 Business, Office 365 Business Premium и Office 365 Business Essentials.

Узнайте больше о том, как [назначать лицензии пользователям в Office 365](https://support.office.com/article/Assign-licenses-to-users-in-Office-365-for-business-997596B5-4173-4627-B915-36ABAC6786DC) , чтобы пользователи могли получать доступ к Microsoft Stream. Убедитесь, что приложение Microsoft Stream не заблокировано для пользователей, как описано в [этой статье](https://docs.microsoft.com/stream/disable-user-organization).

### <a name="ensure-that-users-have-upload-video-permissions-in-microsoft-stream"></a>Убедитесь, что у пользователей есть разрешения на передачу видео в Microsoft Stream.

По умолчанию все пользователи в организации могут создавать содержимое в потоке, после включения потока и назначения ему лицензии. Администратор Microsoft Stream может [ограничивать сотрудников для создания содержимого](https://docs.microsoft.com/stream/restrict-uploaders) в потоке. Пользователи, которые находятся в этом списке ограниченных, не смогут записывать собрания.

### <a name="notify-employees-to-consent-to-company-guidelines-in-microsoft-stream"></a>Уведомление сотрудников о согласии с рекомендациями компании в Microsoft Stream

Если администратор Microsoft Stream [настроил политику "Руководство компании](https://docs.microsoft.com/stream/company-policy-and-consent) " и перед сохранением контента вам потребуется, чтобы они были одобрены сотрудниками, перед записью в Microsoft Teams необходимо выполнить эту политику. Перед развертыванием функции записи в организации убедитесь, что пользователи отправили эту политику.

### <a name="turn-on-or-turn-off-cloud-recording"></a>Включение и отключение записи в облаке

Используйте параметр Алловклаудрекординг в Теамсмитингполици в Teams PowerShell, чтобы указать, разрешено ли записывать собрания пользователя или нет. Дополнительные сведения об управлении Теамсмитингполици с помощью приложения Office 365 PowerShell можно найти [здесь](https://docs.microsoft.com/office365/enterprise/powershell/manage-skype-for-business-online-with-office-365-powershell).

Обратите внимание, что для записи собрания у организатора собрания и инициатора записи должны быть разрешения на запись. Если вы не назначили пользователям специальную политику, пользователи получат глобальную политику, которая по умолчанию Алловтранскриптион отключилась.

Чтобы пользователь вернулся на глобальную политику, удалите определенное назначение политики для пользователя с помощью следующего командлета:

`Grant-CsTeamsMeetingPolicy -Identity {user} -PolicyName $null -Verbose`

Чтобы изменить значение Алловклаудрекординг в глобальной политике, используйте следующий командлет:

`Set-CsTeamsMeetingPolicy -Identity Global -AllowCloudRecording $false`

### <a name="turn-on-or-turn-off-recording-transcription"></a>Включение и отключение транскрипции записи

Когда пользователи записывают собрания Teams, они могут проверить, следует ли автоматически создавать запись после того, как собрание будет записано. Если администраторы отключили возможность транскрипции для организатора собрания и инициатора записи, инициатор записи не сможет получить затем переносите записи собрания.

С помощью параметра Алловтранскриптион в Теамсмитингполици в Teams PowerShell можно управлять тем, может ли инициатор записи выбрать для затем переносите записи собрания. Дополнительные сведения об управлении Теамсмитингполици с помощью приложения Office 365 PowerShell можно найти [здесь](https://docs.microsoft.com/office365/enterprise/powershell/manage-skype-for-business-online-with-office-365-powershell).

Если вы не назначили пользователям специальную политику, они получают глобальную политику, которая по умолчанию отключена Алловтранскриптион.

Чтобы пользователь вернулся на глобальную политику, удалите определенное назначение политики для пользователя с помощью следующего командлета:

Чтобы изменить значение Алловклаудрекординг в глобальной политике, используйте следующий командлет:

`Set-CsTeamsMeetingPolicy -Identity Global -AllowTranscription $false`

### <a name="planning-for-storage"></a>Планирование хранилища

Размер записи в 1 час составляет 400 МБАЙТ. Убедитесь, что вы понимаете емкость, требуемую для записанных файлов, и достаточно места для хранения в Microsoft Stream.  В [этой статье](https://docs.microsoft.com/stream/license-overview) объясняется, как получить сведения о базовом хранилище подписки и о том, как приобрести дополнительное хранилище.

## <a name="manage-meeting-recordings"></a>Управление записями собраний
Запись собрания считается контентом, принадлежащем клиенту. Если владелец записи выходит из компании, администратор может открыть URL-адрес записи в Microsoft Stream в режиме администратора. Администратор может удалить запись, обновить любые метаданные записи или изменить разрешения для видео для записи. Дополнительные сведения о [возможностях администратора в потоке](https://docs.microsoft.com/stream/manage-content-permissions).

## <a name="compliance-and-ediscovery-for-meeting-recordings"></a>Соответствие требованиям и обнаружение электронных данных для записей собраний
Записи собраний хранятся в Microsoft Stream, совместимом с уровнем Office 365 (уровень C). Для поддержки запросов на обнаружение пользователей для администраторов соответствия требованиям, которые заинтересованы в записях собрания или звонков в Microsoft Teams, сообщение о завершении записи будет доступно в функции поиска контента соответствия требованиям для Майкрософт. Администраторы соответствия требованиям могут найти ключевое слово "Recording" в строке темы элемента в предварительной версии поиска по содержимому, а также искать собрания и записи звонков в Организации. Предварительным требованием для просмотра всех записей является то, что их необходимо настроить в Microsoft Stream с правами администратора. Подробнее о [назначении разрешений администратора в потоке](https://docs.microsoft.com/stream/assign-administrator-user-role).
