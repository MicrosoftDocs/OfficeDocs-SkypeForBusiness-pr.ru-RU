# <a name="about-apps-in-microsoft-teams"></a>О приложениях в Microsoft Teams

Приложения — это отличный способ объединить средства и службы на рабочем месте и совместно работать с другими пользователями. Приложения помогают конечным пользователям работать более продуктивно, совместно и эффективно выполнять повседневные задачи. Организации используют приложения для подключения к своим клиентам, предоставления услуг и предоставления сведений. Приложения позволяют пользователям эффективнее Teams чатах, собраниях и каналах. Примерами являются пользователи, использующие закрепленный календарь в Teams для быстрой совместной работы с другими пользователями, приложение с функциями ботов, информирующее пользователей о QoS веб-службы в канале Teams, и приложение для совместного использования и назначения задач различным конечным пользователям в канале.

Наш широкий выбор проверенных и безопасных приложений в Магазине предоставляет конечным пользователям доступ к средствам и службам, которые требуются вашей организации каждый день. Microsoft Teams приложения — это веб-приложения SaaS, которые не нужно развертывать. Конечные пользователи могут использовать приложения в Teams только на основе предоставленных вами разрешений. Как администратор вы просто утверждаете или блокируете использование любого приложения для пользователей вашей организации. Вы управляете доступностью приложений для всех пользователей в собраниях, чатах и каналах.

Чтобы предоставить конечным пользователям необходимые приложения, прочтите дополнительные сведения о типах приложений и о том, где пользователи имеют доступ к этим приложениям. Дополнительные сведения об использовании приложений см. в статье ["Общие сведения о приложениях для конечных пользователей"](https://support.office.com/article/overview-of-apps-in-teams-747492ee-7cdd-4115-a993-8c7e7f98a3d0).

## <a name="custom-apps"></a>Пользовательские приложения

Приложения, созданные разработчиками в организации, называются пользовательскими приложениями. Разработка такого приложения выполняется в соответствии с конкретными требованиями вашей организации, и у вас есть элементы управления для разрешения или запрета таких приложений. Разработчики в вашей организации могут быстро создавать пользовательские решения с низким кодом, Teams интеграцию с [Microsoft Power Platform](/microsoftteams/platform/samples/teams-low-code-solutions).

После того как администратор разрешит использовать пользовательские приложения, конечные пользователи смогут найти такие приложения,  щелкнув "Встроено для вашей организации" в левой области навигации Teams магазине.

### <a name="understand-sideloading-of-custom-apps"></a>Общие сведения о загрузке неопубликованных пользовательских приложений

При разработке пользовательских приложений и их распространении для конечных пользователей разработчики тестируете приложения, добавляя их в магазин для тестирования самостоятельно или с помощью команды, в которой они перегружают приложение. Этот метод называется загрузкой неопубликованных приложений и применяется только к пользовательским приложениям.

Разработчики могут загрузить неопубликованное приложение, чтобы сделать его доступным для участников определенной команды, как правило, для тестирования приложения, на которое выполняется разработка. Для этого не требуется утверждение администратора, если разрешена загрузка неопубликованных приложений. Администратор может запретить загрузку неопубликованных приложений для любого разработчика.

Если запретить загрузку неопубликованных приложений, разработчики по-прежнему смогут тестировать свои приложения в [тестовом клиенте](/microsoftteams/platform/concepts/build-and-test/prepare-your-o365-tenant). После завершения разработки пользовательского приложения разработчики запрашивают у администраторов распространение пользовательского приложения среди конечных пользователей. Дополнительные сведения см. [в статье о публикации пользовательского приложения](/microsoftteams/upload-custom-apps). Администратор может разрешить или запретить использование пользовательского приложения для определенных пользователей.

## <a name="understand-app-capabilities"></a>Общие сведения о возможностях приложения

Чтобы предоставить широкие возможности, позволяющие конечным пользователям работать внутри Teams, разработчики приложений используют следующие возможности приложения. Расширения обмена сообщениями позволяют пользователям взаимодействовать с веб-службой Teams клиентом. Они выполняют поиск или запускают действия во внешней системе. Результат взаимодействия можно отправить клиенту Teams в виде карточки с форматированием. Приложения расширяемости собраний интегрируются с приложениями разработчика в рамках собраний и обеспечивают адаптивный интерфейс для собраний.

Боты также называются чат-ботами или чат-ботами. Это приложение, которое выполняет простые и повторяющиеся задачи. Взаимодействие с ботом может быть быстрым вопросом и ответом или сложным диалогом, предоставляющим доступ к службам или помощи. Пользователи могут общаться в чате один на один или в канале. Например, вы можете использовать приложение Polly для создания быстрых опросов, получения отзывов и проверки пульса.

Вкладки Teams веб-страницы, закрепленные в верхней части канала или чата. Вкладки позволяют взаимодействовать с содержимым и службами с помощью веб-интерфейса. Вкладки можно добавлять как часть канала в команде, групповом чате или личном приложении для отдельного пользователя.

Веб-перехватчики и соединители доставлять содержимое и обновления из служб, которые пользователи часто используют (например, Jira Cloud и Bitbucket), непосредственно в беседу по каналу. Приложения, использующие эту возможность, могут взаимодействовать с внешними приложениями и отправлять или получать уведомления и сообщения от внешней службы.

Расширения для обмена сообщениями — это ярлыки для вставки содержимого приложения или выполнения действий с сообщением без необходимости перехода конечных пользователей из беседы. Расширения обмена сообщениями могут иметь команды поиска для конечных пользователей, чтобы быстро находить внешнее содержимое и вставлять его в сообщения или команды действий.

Чтобы просмотреть распространенные варианты использования, сопоставленные с Teams, см. статью "Сопоставление вариантов использования с Teams [приложениями"](/microsoftteams/platform/concepts/design/map-use-cases).
