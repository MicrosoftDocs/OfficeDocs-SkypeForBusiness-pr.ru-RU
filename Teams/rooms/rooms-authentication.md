# <a name="authentication-in-microsoft-teams-rooms"></a>Проверка подлинности в Комнаты Microsoft Teams

Управление учетными записями Комнаты Microsoft Teams выполняется на уровне приложения. Приложение подключается к Microsoft Teams, Skype для бизнеса и Exchange, чтобы получить ресурсы для учетной записи ресурса для звонков и собраний. Комнаты Teams использует выделенную учетную запись ресурса, чтобы обеспечить постоянное использование возможностей, сценарии вызовов (для устройств, настроенных с тарифным планом) и пользовательские механизмы блокировки. Это означает, что проверка подлинности Комнаты Teams происходит не так, как для устройств конечных пользователей.  

Современная проверка подлинности рекомендуется для всех клиентов, использующих Комнаты Microsoft Teams с Microsoft 365 или Office 365. Если у вас есть локальное развертывание сервера Exchange или сервера Skype для бизнеса, настройте гибридную современную проверку подлинности [](/office365/enterprise/hybrid-modern-auth-overview) с помощью Azure Active Directory (Azure AD), чтобы включить современную проверку подлинности.

Современная проверка подлинности поддерживается Комнаты Microsoft Teams версии 4.4.25.0 и более поздних.

## <a name="modern-authentication"></a>Современная проверка подлинности

При использовании современной проверки подлинности с приложением Комнаты Microsoft Teams библиотека проверки подлинности Active Directory (ADAL) используется для подключения к Microsoft Teams, Exchange и Skype для бизнеса. Современный механизм проверки подлинности [](/azure/active-directory/develop/v2-oauth-ropc) использует тип предоставления авторизации для учетных данных владельца ресурса в OAuth 2.0, который не требует вмешательства пользователя. Это одно из ключевых различий между современной проверкой подлинности для учетных записей пользователей и учетных записей ресурсов, используемых Комнаты Microsoft Teams. Из-за этого учетные записи Комнаты Microsoft Teams ресурсов не должны быть настроены для использования многофакторной проверки подлинности (MFA), проверки подлинности смарт-карты или проверки подлинности на основе сертификата клиента (все они доступны для конечных пользователей).

Еще одно ключевое различие между современной проверкой подлинности на устройствах Комнаты Microsoft Teams и конечных пользователей заключается в том, что вы не можете использовать учетную запись ресурса для применения политик условного доступа на уровне устройства в Azure Active Directory и Endpoint Manager так как сведения об устройстве не передаются при использовании этого типа предоставления. Вместо этого можно зарегистрировать устройство в Microsoft Endpoint Manager применить политики соответствия. [Дополнительные сведения см. в статье Intune условного доступа и соответствия Комнаты Microsoft Teams](conditional-access-and-compliance-for-devices.md).

## <a name="enable-modern-authentication-on-microsoft-teams-rooms"></a>Включение современной проверки подлинности Комнаты Microsoft Teams

Чтобы Комнаты Microsoft Teams использовать современную проверку подлинности с Skype для бизнеса и Exchange, включите параметр на стороне клиента для современной проверки подлинности на Комнаты Microsoft Teams. Это можно сделать в параметрах устройства или в XML-файле конфигурации.

> [!NOTE]
> Прежде чем включить параметр на стороне клиента для современной проверки подлинности, убедитесь, что среда настроена правильно для использования современной проверки подлинности.

### <a name="using-device-settings"></a>Использование параметров устройства

1. На Комнаты Microsoft Teams перейдите к разделу **"Дополнительно**" (**...**).
    
2. Выберите **Параметры**, а затем введите имя пользователя и пароль администратора устройства.
3. Перейдите на **вкладку "Учетная** запись", включите **современную** проверку подлинности, а затем выберите " **Сохранить и выйти"**.

### <a name="using-the-xml-config-file"></a>Использование XML-файла конфигурации

В SkypeSettings.xml задайте для XML-элемента современной проверки подлинности значение **True**, как показано ниже.

```XML
<ModernAuthEnabled>True</ModernAuthEnabled>
```

Чтобы применить этот параметр, см. раздел ["Удаленное управление Комнаты Microsoft Teams консоли с помощью XML-файла конфигурации"](xml-config-file.md).

### <a name="enable-modern-authentication-in-microsoft-365-or-office-365"></a>Включение современной проверки подлинности в Microsoft 365 или Office 365

Чтобы включить современную проверку подлинности для Exchange Online, см. статью "Включение современной проверки подлинности [в](/exchange/clients-and-mobile-in-exchange-online/enable-or-disable-modern-authentication-in-exchange-online) Exchange Online".

Мы рекомендуем не удалять базовые политики проверки подлинности для Exchange Online или отключать обычную проверку подлинности для клиента, пока не убедитесь, что Комнаты Microsoft Teams устройства могут успешно войти с помощью Exchange Online и Teams.

Дополнительные сведения об отключении обычной проверки подлинности в Exchange Online см. в разделе "Отключение обычной проверки подлинности [в Exchange Online"](/exchange/clients-and-mobile-in-exchange-online/disable-basic-authentication-in-exchange-online).

## <a name="hybrid-modern-authentication"></a>Гибридная современная проверка подлинности

Чтобы обеспечить успешную проверку подлинности на локальном Exchange-сервере и (или) Skype для бизнеса-сервере, необходимо убедиться, что учетная запись ресурса, используемая с Комнаты Microsoft Teams, настроена для получения авторизации из Azure AD.

Комнаты Teams проверки подлинности зависят от конфигурации проверки подлинности. Для клиентов, использующих управляемый домен, Комнаты Teams использует учетные данные пароля владельца ресурса [OAuth 2.0](/azure/active-directory/develop/v2-oauth-ropc) с Azure Active Directory. Однако для клиентов, использующих федеративный домен, используется утверждение носителя [SAML Flow OAuth 2.0](/azure/active-directory/develop/v2-saml-bearer-assertion).

> [!NOTE]
> Поставщику удостоверений могут потребоваться определенные конфигурации или параметры для интеграции с Azure Active Directory или Office 365. Обратитесь к поставщику удостоверений, если вам нужна помощь по настройке проверки подлинности с помощью Комнаты Teams.


### <a name="prerequisites-specific-to-microsoft-teams-rooms"></a>Предварительные требования, относящиеся к Комнаты Microsoft Teams

Предварительные требования для включения современной проверки подлинности в гибридной топологии рассматриваются в обзоре гибридной современной проверки подлинности и предварительных требованиях для ее использования с локальными Skype для бизнеса и [Exchange серверами](/office365/enterprise/hybrid-modern-auth-overview). Применяются все предварительные требования, описанные в этой статье.

Однако поскольку Комнаты Microsoft Teams использует авторизацию учетных [](https://tools.ietf.org/html/rfc6749#section-1.3.3) данных владельца ресурса и базовые ИНТЕРФЕЙСы REST API для современной проверки подлинности, следует учитывать следующие важные отличия, относящиеся к Комнаты Microsoft Teams.

- Необходимо иметь накопительный пакет Exchange Server обновления 8 (CU8) 2016 или более поздней версии или Exchange Server накопительный пакет обновления 1 (CU1) 2019 или более поздней версии.
- Вам потребуется накопительный пакет Skype для бизнеса Server обновления 5 (CU5) 2015 или более поздней версии Skype для бизнеса Server 2019 или более поздней версии.
- MFA не поддерживается независимо от используемой топологии.
- Комнаты Microsoft Teams не поддерживает несоответствие SIP и UPN. Для работы необходимо создать учетную запись Комнаты Microsoft Teams с тем же именем участника-пользователя и SIP.
- При использовании стороннего поставщика проверки подлинности, поддерживаемого Azure AD, он должен поддерживать активный поток проверки подлинности через WS-Trust.
- Не используйте политики условного доступа на уровне устройства для учетной записи ресурса, настроенной с приложением. Это приведет к сбоям при входе. Вместо этого зарегистрируйте устройство в Microsoft Intune и примените политики соответствия. [Дополнительные сведения см. в статье Intune условного доступа и соответствия Комнаты Microsoft Teams](conditional-access-and-compliance-for-devices.md).
