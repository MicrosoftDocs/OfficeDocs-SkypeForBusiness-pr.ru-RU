# <a name="prepare-your-environment"></a>Подготовка среды

В этом разделе содержится обзор действий, необходимых для подготовки среды, чтобы можно было использовать все функции Комнаты Microsoft Teams.
  
1. Подготовьте учетную запись ресурса для каждой Комнаты Microsoft Teams консоли. [Дополнительные сведения см. в Комнаты Microsoft Teams](rooms-deploy.md) развертывания.
    
2. Проверьте наличие рабочего подключения к сети или Интернету, которое будет использовать устройство. 
  
3. Корпорация Майкрософт осуществляет сбор данных с целью улучшения своих продуктов. Чтобы разрешить корпорации Майкрософт собирать данные, разрешите следующие сайты:

   - Конечная точка клиента телеметрии: `https://vortex.data.microsoft.com/`
   - Конечная точка параметров телеметрии:` https://settings.data.microsoft.com/`
    
### <a name="create-and-test-a-resource-account"></a>Создание и тестирование учетной записи ресурса

*Учетная запись ресурса* — это учетная запись, которую клиент Комнаты Microsoft Teams для доступа к функциям из Exchange, например календаря, и для подключения к Microsoft Teams. [Дополнительные сведения см. в Комнаты Microsoft Teams](rooms-deploy.md) развертывания.
  
### <a name="check-network-availability"></a>Проверка доступности сети

Чтобы обеспечить правильную работу, Комнаты Microsoft Teams должен иметь доступ к проводной сети, которая соответствует следующим требованиям:
  
- Доступ к экземпляру Active Directory или Azure Active Directory (Azure AD), а также microsoft Exchange и Microsoft Teams.

- Доступ к серверу, предоставляющему IP-адреса через DHCP. Комнаты Microsoft Teams нельзя настроить со статическим IP-адресом при первом запуске единицы.

- Доступ к HTTP-портам 80 и 443.

- Порты TCP и UDP, настроенные в соответствии [](/skypeforbusiness/plan-your-deployment/network-requirements/ports-and-protocols) с требованиями к портам и протоколам для серверов для локальных реализаций Skype для бизнеса Server, а также URL-адреса Microsoft 365 и [Office 365 и диапазоны IP-адресов](https://support.office.com/article/Office-365-URLs-and-IP-address-ranges-8548a211-3fe7-47cb-abb1-355ea5aa88a2?ui=en-US&amp;rs=en-US&amp;ad=US) для Microsoft Teams.

Если в вашей сети работает прокси-сервер, вам также потребуются его адрес и данные скрипта.
    
> [!IMPORTANT]
> Комнаты Microsoft Teams не поддерживает проверку подлинности прокси-сервера, так как она может помешать выполнению обычных операций в помещении. Перед переходом в рабочую среду убедитесь, что Комнаты Microsoft Teams были исключены из проверки подлинности прокси-сервера.

> [!IMPORTANT]
> Для обеспечения достаточной пропускной способности используйте проводное сетевое подключение стандарта 1 Гбит/с. 

> [!NOTE]
> Обновления программного обеспечения Комнаты Microsoft Teams автоматически скачиваются с Microsoft Store для бизнеса. [Ознакомьтесь с предварительными требованиями для Microsoft Store для бизнеса](/microsoft-store/prerequisites-microsoft-store-for-business) и образования, чтобы убедиться, что консоль комнаты сможет получить доступ к магазину и самостоятельно обновиться.
  
### <a name="certificates"></a>Сертификаты

Устройство Комнаты Microsoft Teams использует сертификаты для Exchange веб-служб, Microsoft Teams или Skype для бизнеса, использования сети и проверки подлинности. Если связанные серверы используют общедоступные сертификаты, что является вариантом для сетевых и некоторых локальных развертываний, администратор не должен выполнять никаких дополнительных действий для установки сертификатов. Если, с другой стороны, центр сертификации является частным ЦС, устройство должно доверять этому ЦС. Это означает, что на устройстве установлены сертификаты цепочки ЦС и ЦС. При добавлении устройства в домен эта задача может быть выполнена автоматически.
  
Сертификаты устанавливаются так же, как для любого другого клиента Windows. 

> [!IMPORTANT]
> Если прокси-сервер использует внутренние подписанные сертификаты, необходимо установить внутреннюю цепочку сертификатов, включая корневой ЦС, на Комнаты Microsoft Teams устройстве.
  
> [!NOTE]
> Сертификаты могут потребоваться для того, чтобы Комнаты Microsoft Teams использовать Skype для бизнеса Server.
  
### <a name="proxy"></a>Прокси-сервер

Комнаты Microsoft Teams предназначен для наследования параметров прокси-сервера от Windows ОС. Для доступа к параметрам ОС Windows выполните следующие действия.
  
1. В пользовательском Комнаты Microsoft Teams щелкните значок шестеренки Параметры, где вам будет предложено ввести пароль локального администратора на устройстве (пароль по умолчанию **— sfb**).
2. Коснитесь Параметры, а затем нажмите кнопку **"** Перейти к Windows", а затем нажмите кнопку **"** Перейти к Администратор", а затем нажмите кнопку "Администратор"  (если компьютер присоединен к домену, выберите "Другой пользователь"**,** а затем в качестве имени пользователя используйте файл .\admin).
3. В поле **поиска Windows** левый нижний тип в regedit (либо длинное нажатие экрана, либо щелчок правой кнопкой мыши и выбор команды **"Запуск от имени администратора"**).
4. Щелкните папку HKEY_USERS (вы увидите список идентификаторов SID пользователей компьютера) и убедитесь в том, что выбрана корневая папка HKEY_USERS.
       
5. Щелкните "Файл", а затем выберите **"Загрузить Hive".**
6. Перейдите в **папку C:\Users\Skype** и введите в поле имени файла NTUSER.dat и нажмите кнопку "Открыть".

7. Вам будет предложено ввести имя ключа для загруженного Hive. введите Skype (теперь вы увидите параметры реестра для Skype пользователя).
 
8. Откройте ключ Skype и перейдите к HKEY_USERS\Skype\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings убедитесь, что вводятся следующие параметры: 
    
    ```console
    [HKEY_USERS\Skype\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings]
    "MigrateProxy"=dword:00000001
    "ProxyEnable"=dword:00000001
    "ProxyServer"="xx.xx.xx.xx:8080"
    ```
    
    Если параметр ProxyServer не существует, его может потребоваться добавить в качестве строкового параметра. Замените xx.xx.xx.xx:8080 на IP-адрес или имя узла и порт вашего прокси-сервера.
 
    Если клиент использует PAC-файл, конфигурация будет выглядеть следующим образом:

     ```console
    [HKEY_USERS\Skype\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings]
    "MigrateProxy"=dword:00000001
    "ProxyEnable"=dword:00000001
    "AutoConfigURL"=http://contosoproxy.corp.net/proxy.pac
    ```
    
9. Завершив внесение изменений, выделите раздел Skype User (корневую папку Skype), а затем в меню "Файл" редактора реестра выберите пункт "Выгрузить куст" (появится запрос на подтверждение — выберите **Да**).
    
10. Теперь можно закрыть редактор реестра и ввести logoff в поле "Поиск в Windows".
    
11. Вернувшись на экран входа, выберите пользователя **Skype**. Если все предыдущие шаги были выполнены успешно, Комнаты Microsoft Teams устройство успешно выполнит вход.
    
Подробные [сведения о](./security.md#network-security) полных доменных именах, портах и диапазонах IP-адресов, необходимых для Комнаты Microsoft Teams, см. в статье "Безопасность сети".
  
### <a name="admin-group-management"></a>Управление группой администраторов

Если вы решили присоединиться к домену (Azure Active Directory или Active Directory), можно использовать Microsoft Endpoint Manager, групповая политика или локальное управление компьютерами, чтобы задать группу безопасности в качестве локального администратора так же, как для компьютера Windows в вашем домене. Любой участник этой группы безопасности сможет ввести свои учетные данные и разблокировать раздел "Параметры".
  
> [!NOTE]
> Если ваше устройство с приложением "Комнаты Microsoft Teams" утрачивает отношение доверия с доменом (например, если вы удалите "Комнаты Microsoft Teams" из домена после присоединения к нему), вы не сможете пройти проверку подлинности на устройстве и открыть раздел "Параметры". В качестве обходного решения войдите с помощью локальной учетной записи администратора. 
  
## <a name="local-accounts"></a>Локальные учетные записи

### <a name="microsoft-teams-rooms-local-user-account"></a>Комнаты Microsoft Teams учетной записи локального пользователя

Комнаты Teams включает локальную учетную запись без пароля с именем Skype. Эта учетная запись используется для входа Windows запуска Комнаты Teams приложения. Применение пароля к этой учетной записи не поддерживается. [Дополнительные сведения см. Комнаты Microsoft Teams security](security.md).
  
### <a name="admin---local-administrator-account"></a>Admin — учетная запись локального администратора

Комнаты Microsoft Teams по умолчанию задано значение "sfb". Пароль можно изменить локально в Администратор или в файле AutoUnattend.xml (используйте диспетчер системных образов Windows из ADK, чтобы внести изменения в XML-файл).
  
> [!CAUTION]
> Обязательно измените пароль Комнаты Microsoft Teams как можно скорее. 
  
Пароль локального администратора нельзя выбрать во время настройки.

Дополнительные сведения об учетной записи Администратор см. в Комнаты Microsoft Teams [security](security.md).
  
### <a name="machine-account"></a>Учетная запись компьютера

Как и Windows устройства,  \> имя компьютера можно переименовать, щелкнув правой кнопкой мыши в Параметры **"О** \> **переименовании компьютера"**.
  
Если вы хотите переименовать компьютер после присоединения к домену, используйте команду [Rename-Computer](/powershell/module/microsoft.powershell.management/rename-computer), команду PowerShell, а затем новое имя компьютера.
