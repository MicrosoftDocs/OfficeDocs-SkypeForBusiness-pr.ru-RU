# <a name="prepare-your-environment"></a>Подготовка среды

В этом разделе содержатся общие сведения о действиях, необходимых для подготовки среды к использованию всех функций Комнаты Microsoft Teams.
  
1. Подготовка учетной записи ресурса для Комнаты Microsoft Teams консоли. Дополнительные [сведения см. в Комнаты Microsoft Teams](rooms-deploy.md) развертывания.
    
2. Проверьте наличие рабочего подключения к сети или Интернету, которое будет использовать устройство. 
  
3. Корпорация Майкрософт осуществляет сбор данных с целью улучшения своих продуктов. Чтобы разрешить корпорации Майкрософт собирать данные, разрешим этим сайтам:

   - Конечная точка клиента телеметрии: https://vortex.data.microsoft.com/
   - Конечная точка параметров телеметрии: https://settings.data.microsoft.com/
    
### <a name="create-and-test-a-resource-account"></a>Создание и проверка учетной записи ресурса

*Учетная запись ресурса* — это учетная запись, которую клиент Комнаты Microsoft Teams для доступа к функциям из Exchange, например календаря, и для подключения к Microsoft Teams. Дополнительные [сведения см. в Комнаты Microsoft Teams](rooms-deploy.md) развертывания.
  
### <a name="check-network-availability"></a>Проверка доступности сети

Для правильной работы Комнаты Microsoft Teams должен быть доступ к проводной сети, которая соответствует этим требованиям:
  
- Доступ к экземпляру Active Directory или Azure Active Directory (Azure AD), а также microsoft Exchange и Microsoft Teams.

- Доступ к серверу, предоставляющему IP-адреса через DHCP. Комнаты Microsoft Teams не удается настроить статический IP-адрес при первом запуске.

- Доступ к HTTP-портам 80 и 443.

- Порты TCP и UDP настроены так, как [](/skypeforbusiness/plan-your-deployment/network-requirements/ports-and-protocols) описано в требованиях к портам и протоколам для серверов для локальной реализации Skype для бизнеса Server, а также URL-адреса и диапазоны [IP-адресов](https://support.office.com/article/Office-365-URLs-and-IP-address-ranges-8548a211-3fe7-47cb-abb1-355ea5aa88a2?ui=en-US&amp;rs=en-US&amp;ad=US) Microsoft 365 и Office 365 для Microsoft Teams.

Если в вашей сети работает прокси-сервер, вам также потребуются его адрес и данные скрипта.
    
> [!IMPORTANT]
> Комнаты Microsoft Teams не поддерживает проверку подлинности прокси-сервера, так как она может мешать обычной работе в комнате. Перед началом Комнаты Microsoft Teams исключены из проверки подлинности прокси-сервера.

> [!IMPORTANT]
> Для обеспечения достаточной пропускной способности используйте проводное сетевое подключение стандарта 1 Гбит/с. 

> [!NOTE]
> Обновления программного обеспечения Комнаты Microsoft Teams автоматически загружаются из Microsoft Store для бизнеса. Чтобы [убедиться,](/microsoft-store/prerequisites-microsoft-store-for-business) что консоль комнаты сможет получить доступ к магазину и самообновка, см. статью Предварительные условия для Microsoft Store для бизнеса и Образования.
  
### <a name="certificates"></a>Сертификаты

Ваше устройство Комнаты Microsoft Teams использует сертификаты для веб-служб Exchange, Microsoft Teams или Skype для бизнеса, использования сети и проверки подлинности. Если на связанных серверах используются общедоступные сертификаты ,что относится к сетевым и некоторым локальному развертываниям, никаких дополнительных действий со стороны администратора для установки сертификатов не требуется. Если же это частный ЦС, устройство должно доверять этому ЦС. Это означает, что на устройстве установлены сертификаты ca + CA chain. При добавлении устройства в домен эта задача может быть выполнена автоматически.
  
Сертификаты устанавливаются так же, как для любого другого клиента Windows.  
  
> [!NOTE]
> Сертификаты могут потребоваться для использования Комнаты Microsoft Teams Skype для бизнеса Server.
  
### <a name="proxy"></a>Прокси-сервер

Комнаты Microsoft Teams предназначен для наследования параметров прокси-сервера от Windows ОС. Для доступа к параметрам ОС Windows выполните следующие действия.
  
1. В пользовательском интерфейсе Комнаты Microsoft Teams щелкните значок шестеренки Параметры, где вам будет предложено вводить локальный пароль администратора на устройстве (по умолчанию это **SFB**).
2. Коснитесь Параметры, а затем нажмите кнопку Перейти к Windows, а затем коснитесь кнопки Перейти к  входу администратора и нажмите кнопку Администратор (если  компьютер является доменом, выберите Другой пользователь **,** а затем в качестве имени пользователя используйте .\admin). 
3. В поле **поиск Windows** regedit (длинное нажатие экрана или щелкните правой кнопкой мыши и выберите **запуск от администратора**).
4. Щелкните папку HKEY_USERS (вы увидите список идентификаторов SID пользователей компьютера) и убедитесь в том, что выбрана корневая папка HKEY_USERS.
       
5. Щелкните Файл и выберите **загрузить Файл.**
6. Перейдите к **папке C:\Users\Skype** и введите в поле Имя файла N ДЕОСТЕР.dat и нажмите кнопку открытия

7. Вам будет предложено в качестве имени ключа для загруженного человека. введите Skype (теперь вы увидите параметры реестра для Skype пользователя).
 
8. Откройте клавишу Skype и перейдите к HKEY_USERS\Skype\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings убедитесь, что ввели эти параметры: 
    
    ```console
    [HKEY_USERS\Skype\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings]
    "MigrateProxy"=dword:00000001
    "ProxyEnable"=dword:00000001
    "ProxyServer"="xx.xx.xx.xx:8080"
    ```
    
    Если параметр ProxyServer не существует, его может потребоваться добавить в качестве строкового параметра. Замените xx.xx.xx.xx:8080 на IP-адрес или имя узла и порт вашего прокси-сервера.
 
    Если клиент использует PAC-файл, конфигурация будет выглядеть так, как по образцу ниже:

     ```console
    [HKEY_USERS\Skype\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings]
    "MigrateProxy"=dword:00000001
    "ProxyEnable"=dword:00000001
    "AutoConfigURL"=http://contosoproxy.corp.net/proxy.pac
    ```
    
9. Завершив внесение изменений, выделите раздел Skype User (корневую папку Skype), а затем в меню "Файл" редактора реестра выберите пункт "Выгрузить куст" (появится запрос на подтверждение — выберите **Да**).
    
10. Теперь можно закрыть редактор реестра и ввести logoff в поле "Поиск в Windows".
    
11. Вернувшись на экран входа, выберите пользователя **Skype**. Если все предыдущие шаги были успешными, Комнаты Microsoft Teams устройство успешно войму.
    
Подробные [сведения о полных](./security.md#network-security) полных и полных полных версиях, портах и диапазонах IP-адресов, необходимых для Комнаты Microsoft Teams, см. в статье Безопасность сети.
  
### <a name="admin-group-management"></a>Управление группой администраторов

Если вы решите присоединиться к домену (Azure Active Directory или Active Directory), вы можете использовать Microsoft Endpoint Manager, групповую политику или управление локальным компьютером, чтобы настроить группу безопасности в качестве локального администратора так же, как Windows компьютера в вашем домене. Любой участник этой группы безопасности сможет ввести свои учетные данные и разблокировать раздел "Параметры".
  
> [!NOTE]
> Если ваше устройство с приложением "Комнаты Microsoft Teams" утрачивает отношение доверия с доменом (например, если вы удалите "Комнаты Microsoft Teams" из домена после присоединения к нему), вы не сможете пройти проверку подлинности на устройстве и открыть раздел "Параметры". В качестве обходного решения войдите с помощью локальной учетной записи администратора. 
  
## <a name="local-accounts"></a>Локальные учетные записи

### <a name="microsoft-teams-rooms-local-user-account"></a>Комнаты Microsoft Teams учетной записи локального пользователя

Комнаты Teams включает локализованную учетную запись с именем "Skype". Эта учетная запись используется для Windows запуска Комнаты Teams приложения. Применение пароля к этой учетной записи не поддерживается. [Дополнительные Комнаты Microsoft Teams см. в Комнаты Microsoft Teams](security.md) Security.
  
### <a name="admin---local-administrator-account"></a>Admin — учетная запись локального администратора

Комнаты Microsoft Teams по умолчанию установлено значение sfb. Пароль можно изменить локально в режиме администрирования или в файле AutoUnattend.xml (внести изменения в XML-файл с помощью Windows System Image Manager из ADK).
  
> [!CAUTION]
> Обязательно измените пароль Комнаты Microsoft Teams как можно скорее. 
  
Пароль локального администратора нельзя выбрать во время настройки.

Дополнительные информацию об учетной записи администратора можно найти в статье [Комнаты Microsoft Teams безопасности](security.md).
  
### <a name="machine-account"></a>Учетная запись компьютера

Как и Windows, имя компьютера можно переименовать,  \>  \> щелкнув правой кнопкой мыши в Параметры О **переименовании компьютера**.
  
Если после присоединения к домену вы хотите переименовать компьютер, воспользуйтесь командой [Переименовать-Компьютер](/powershell/module/microsoft.powershell.management/rename-computer?view=powershell-7.2) , командой PowerShell и новым именем компьютера.
