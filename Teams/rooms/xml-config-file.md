# <a name="manage-a-microsoft-teams-rooms-console-settings-remotely-with-an-xml-configuration-file"></a>Удаленное управление настройками консоли комнат Microsoft Teams с помощью XML-файла конфигурации

В этой статье рассмотрено удаленное управление настройками по умолчанию, используемыми устройством комнат Microsoft Teams, включая применение пользовательской темы. В нем обсуждается создание файла параметров, а также ссылки на обсуждения того, как разместить их на устройствах, управляемых удаленно.
  
Вы можете изменить параметры по умолчанию для устройств с удаленным управлением, обновив этот XML-файл и отправив копии на управляемые консоли. Вы также можете реализовать настраиваемые темы на консолях комнат Microsoft Teams с XML-файлами конфигурации.
  
## <a name="create-an-xml-configuration-file"></a>Создание XML-файла конфигурации

Для создания файла параметров можно использовать любой текстовый редактор. В **таблице XML Elements** (Элементы XML) объясняется, SkypeSettings.xml примере файла конфигурации (обязательное имя файла).
  
```XML
<SkypeSettings>
    <AutoScreenShare>true</AutoScreenShare>
    <HideMeetingName>true</HideMeetingName>
    <UserAccount>
        <SkypeSignInAddress>RanierConf@contoso.com</SkypeSignInAddress>
        <ExchangeAddress>RanierConf@contoso.com</ExchangeAddress>
        <ModernAuthEnabled>false</ModernAuthEnabled>
        <DomainUsername>Seattle\RanierConf</DomainUsername>
        <Password>password</Password>
        <ConfigureDomain>domain1, domain2</ConfigureDomain>
    </UserAccount>
    <IsTeamsDefaultClient>false</IsTeamsDefaultClient>
    <BluetoothAdvertisementEnabled>true</BluetoothAdvertisementEnabled>
    <AutoAcceptProximateMeetingInvitations>false</AutoAcceptProximateMeetingInvitations>
    <AutoExitMeetingEnabled>true</AutoExitMeetingEnabled>
    <SkypeMeetingsEnabled>false</SkypeMeetingsEnabled>
    <TeamsMeetingsEnabled>true</TeamsMeetingsEnabled>
    <WebExMeetingsEnabled>true</WebExMeetingsEnabled>
    <ZoomMeetingsEnabled>true</ZoomMeetingsEnabled>
    <UseCustomInfoForThirdPartyMeetings>true</UseCustomInfoForThirdPartyMeetings>
    <CustomDisplayNameForThirdPartyMeetings>Rainier Conference Room</CustomDisplayNameForThirdPartyMeetings>
    <CustomDisplayEmailForThirdPartyMeetings>RanierConf@contoso.com</CustomDisplayEmailForThirdPartyMeetings> 
    <DualScreenMode>true</DualScreenMode>
    <DuplicateIngestDefault>false</DuplicateIngestDefault>
    <DisableTeamsAudioSharing>false</DisableTeamsAudioSharing>
    <CortanaEnabled>false</CortanaEnabled>
    <CortanaWakewordEnabled>false</CortanaWakewordEnabled>
    <SendLogs>
        <EmailAddressForLogsAndFeedback>RanierConf@contoso.com</EmailAddressForLogsAndFeedback>
        <SendLogsAndFeedback>true</SendLogsAndFeedback>
    </SendLogs>
    <Devices>
        <MicrophoneForCommunication>Microsoft LifeChat LX-6000</MicrophoneForCommunication>
        <SpeakerForCommunication>Realtek High Definition Audio</SpeakerForCommunication>
        <DefaultSpeaker>Polycom CX5100</DefaultSpeaker>
        <ContentCameraId>USB\VID_046D&amp;PID_0843&amp;MI_00\7&amp;17446CF2&amp;0&amp;0000</ContentCameraId>
        <ContentCameraInverted>false</ContentCameraInverted>
        <ContentCameraEnhancement>true</ContentCameraEnhancement>
    </Devices>
    <Theming>
        <ThemeName>Custom</ThemeName>
        <CustomThemeImageUrl>file name</CustomThemeImageUrl>
        <CustomThemeColor>
            <RedComponent>100</RedComponent>
            <GreenComponent>100</GreenComponent>
            <BlueComponent>100</BlueComponent>
        </CustomThemeColor>
    </Theming>
    <CoordinatedMeetings enabled="true"> 
        <TrustedAccounts>room@contoso.com</TrustedAccounts>
        <Settings> 
            <Audio default="false" enabled="false" />
            <Video default="false" enabled="true" /> 
            <Whiteboard default="false" enabled="false"/>
        </Settings> 
    </CoordinatedMeetings>
</SkypeSettings>
```

Если переменное значение имеет неправильный тип, элементы имеют неправильный порядок, элементы не печатаются или обнаружена другая ошибка, XML-файл *формируется неправильно.* При обработке XML-файла формируется плохо сформированный XML-файл, параметры до того места, где возникает ошибка, применяются, а остальной файл игнорируется. Любые неизвестные элементы XML-файла также пропускаются. Соответствующие пропущенным элементам параметры устройств остаются без изменений. Если значение параметра является недопустимым, его предыдущее значение не изменяется.

## <a name="manage-console-settings-with-an-xml-configuration-file"></a>Управление параметрами консоли с помощью XML-файла конфигурации

Если консоль комнат Microsoft Teams находит XML-файл с именем SkypeSettings.xml расположен, при запуске применяются параметры конфигурации, указанные в XML-файле, а затем `C:\Users\Skype\AppData\Local\Packages\Microsoft.SkypeRoomSystem_8wekyb3d8bbwe\LocalState` удаляется XML-файл.
  
В зависимости от того, сколько устройств комнат Microsoft Teams установлено в организации и как вы решили настроить их, существует несколько способов разместить XML-файл конфигурации. После принудительной передачи файла на консоль необходимо перезапустить ее для обработки изменений. После успешной обработки XML-файл конфигурации удаляется. Методы управления, предлагаемые для устройств с комнатами Microsoft Teams, рассмотрены в
  
- [Настройка групповой политики для комнат Microsoft Teams](rooms-operations.md#GroupPolicy)
- [Удаленное управление с помощью PowerShell](rooms-operations.md#RemotePS) [и настройка элемента файла](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc772536(v=ws.11))

Вы можете использовать любой из этих способов при условии, что он позволяет передавать файлы и перезапускать устройство консоли. Учетная запись локального пользователя устройства должна иметь возможность чтения, записи и удаления. Желательно, чтобы оно принадлежит этому пользователю и имеет все права, предоставленные этому пользователю. Если разрешения для файла настроены неправильно, программное обеспечение может не применить параметры, не удалить файл после успешной обработки и даже привести к сбою.
  
## <a name="custom-theme-images"></a>Изображения настраиваемых тем

<a name="Themes"> </a>

Файл пользовательского изображения темы должен быть помещен в `C:\Users\Skype\AppData\Local\Packages\Microsoft.SkypeRoomSystem_8wekyb3d8bbwe\LocalState` папку. Введите имя файла и расширение в \<CustomThemeImageUrl\> переменной.
  
Файл изображения должен иметь размер 3840X1080 пикселей и иметь один из следующих форматов: JPG, JPEG, PNG и BMP. Если вашей организации нужно использовать пользовательское изображение, графический конструктор может использовать шаблон [Custom Theme Photoshop.](../downloads/ThemingTemplateMicrosoftTeamsRooms_v2.1.psd) Она содержит более подробные сведения о том, где различные элементы пользовательского интерфейса находятся относительно остальной части изображения темы и какие области отображаются на консолях и на дисплеях.
  
Изображение темы будет распознаваться только после того, как XML-файл конфигурации обновится при запуске устройства. После обработки и удаления нового XML-файла графический файл темы удаляется из каталога.
  
## <a name="locate-the-content-camera-usb-instance-path"></a>Найдите путь экземпляра USB-камеры содержимого

Чтобы найти путь экземпляра:

1. Перейдите в параметры Windows на консоли комнат Microsoft Teams.
2. Введите пароль администратора.
3. В командной подсказке введите запрос, `devmgmt.msc` чтобы отвести диспетчер устройств.
4. В **диспетчере устройств** посмотрите на узел **"Устройства обработки изображений"** и найдите камеру содержимого.
5. Щелкните камеру правой кнопкой мыши и откройте **"Свойства".**
6. Перейдите на **вкладку** "Сведения" и найдите свойство **"Путь экземпляра устройства"** в down-down.
7. Показанное значение представляет собой путь экземпляра устройства, заданная в XML-файле конфигурации. При указании пути в XML замените амперсанд (&) на `&amp;` .
