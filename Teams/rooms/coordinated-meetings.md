# <a name="set-up-coordinated-meetings-with-microsoft-teams-rooms-and-surface-hub"></a>Настройка скоординированных собраний с помощью Комнаты Microsoft Teams и Surface Hub

Если у вас есть одно или Комнаты Microsoft Teams устройств или Устройств Surface Hub в комнате для собраний, вы можете настроить координированное собрание. Координированное собрание позволяет настроить устройства Комнаты Teams и Surface Hub таким образом, чтобы при присоединении к собранию на одном устройстве другие устройства в комнате также присоединялись к одному собранию. Вы можете настроить камеры, динамики и микрофоны таким образом, чтобы те, которые обеспечивают наилучшее взаимодействие с участниками, были включены, а другие отключены. Это позволяет избежать проблем с эхом и шумом обратной связи, которые могут происходить при добавлении нескольких устройств на собрание.

Чтобы настроить координированные собрания, необходимо убедиться, что устройства Комнаты Teams и Surface Hub уже правильно настроены для участия в собраниях. Самое важное, что каждое устройство должно иметь собственный почтовый ящик комнаты Exchange. Сведения о том, как их настроить, см. в следующих статьях:

- [Развертывание комнат Microsoft Teams](../rooms/rooms-deploy.md)
- [Создание учетной записи устройства Surface Hub 2S](/surface-hub/surface-hub-2s-account)

Убедився, что ваши устройства Комнаты Teams и Surface Hub могут автоматически принимать собрания и присоединяться к ним успешно, можно настроить координированные собрания.

Для каждой комнаты для собраний необходимо выполнить следующие действия отдельно.

## <a name="step-1-plan-your-coordinated-meeting-experience"></a>Шаг 1. Планирование координированного собрания

Прежде чем вносить изменения в конфигурацию, необходимо решить, какие устройства будут выполнять действия в каждой комнате для собраний. То есть для конкретной комнаты для собраний необходимо решить, на каком устройстве будет активен микрофон, камера и доска. Настройка устройств зависит от конкретной среды, но ниже приведены некоторые общие рекомендации для начала.

- **Устройство Комнаты Teams** микрофона
- **Устройство** Комнаты Teams (включено по умолчанию) и Surface Hub (отключено по умолчанию, но разрешено участникам)
- **Доски** Surface Hub

> [!IMPORTANT]
> Убедитесь, что микрофон включен только на одном устройстве. Если включить его на нескольких устройствах, вы получите звуковое эхо и обратную связь.

## <a name="step-2-get-your-devices-upns"></a>Шаг 2. Получение имен участника-пользователя устройств

При настройке функции "Координированное собрание" в комнате для собраний необходимо сообщить Комнаты Teams устройствам и устройствам Surface Hub в этой комнате, с которыми следует координировать действия. Это делается путем добавления имени участника-пользователя (UPN) устройств, с которых он должен координироваться, в конфигурацию. Если вы не знаете имена участников-пользователей для каждого устройства, которое вы хотите настроить для скоординированных собраний, их можно найти с помощью Центр администрирования Microsoft 365. 

Вам должна быть назначена роль администратора для доступа к Центр администрирования Microsoft 365. Дополнительные сведения см. в [разделе о ролях администратора](/microsoft-365/admin/add-users/about-admin-roles).

Чтобы получить имена участника-пользователя Комнаты Teams устройств и Устройств Surface Hub, выполните следующие действия.

1. Войдите в Центр администрирования Microsoft 365, посетив https://admin.microsoft.com.
2. Перейдите в **раздел "Активные** > **пользователи"**.
3. Найдите имя устройства Комнаты Teams surface Hub в столбце "Отображаемое имя" (поле поиска можно использовать при наличии  большого количества пользователей).
4. Найдите имя участника-пользователя в столбце **"** Имя пользователя" (оно будет выглядеть примерно так alias@contoso.com или alias@contoso.onmicrosoft.com).
5. Повторите это действие для каждого устройства, которое будет участвовать в координированном собрании.

## <a name="step-3-create-a-deployment-worksheet"></a>Шаг 3. Создание листа развертывания

Лист развертывания поможет вам визуализировать конфигурацию, которую необходимо задать на всех устройствах, что позволит проверить выбранные параметры и проверить наличие ошибок.

В приложении электронной таблицы добавьте строки для следующих элементов в первом столбце:

Это можно сделать на `true` любых других устройствах участников мероприятия, которые хотят поделиться разными перспективами видео (например, если участник использует доску Surface Hub). Если вы не хотите, чтобы участники включили или отключили камеру на устройстве, установите для этого параметра значение `false`.<p> Если **по умолчанию** задано значение "Видео" `true`, этот параметр игнорируется, и участники могут включать или отключать камеру.         |

В последующих столбцах добавьте каждое из Комнаты Teams устройств и Устройств Surface Hub. В каждом столбце заполните значения, соответствующие интерфейсу комнаты для собраний. Ниже приведен пример с одним устройством Комнаты Teams и одним Surface Hub:

## <a name="step-4-configure-teams-rooms-device"></a>Шаг 4. Настройка Комнаты Teams устройства

Вы можете настроить координированное собрание на устройстве Комнаты Teams с помощью сенсорного экрана устройства или, если необходимо настроить множество устройств и сделать это из центрального расположения, можно использовать XML-файл конфигурации.

Используйте лист, созданный на предыдущем шаге, чтобы помочь вам настроить устройства.

### <a name="use-the-teams-rooms-devices-touch-screen"></a>Использование сенсорного Комнаты Teams устройства

Чтобы настроить координированное собрание на устройстве, сделайте следующее:

1. Выберите **... Дополнительные** > **параметры**.
2. Введите пароль администратора и нажмите кнопку **"Да"**.
3. Выберите **"Координированное собрание"**.
4. В **разделе "Параметры**" **задайте параметр "Координированное** _собрание"_.
5. Если **на листе** по `true`умолчанию установлено значение "Звук", включите микрофон этого устройства, в противном случае оставьте _его отключенным_.
6. Если **на листе** `true`включен звук, выберите "Разрешить пользователям  включать при присоединении к собранию" в разделе **"Включить микрофон этого устройства"**. Этот параметр нельзя отключить, если включен микрофон этого  устройства.
7. Если **видео по** умолчанию на `true`листе установлено, включите камеру этого устройства, в противном случае оставьте _ее отключенной_.
8. Если **на листе** `true`включено видео, выберите "Разрешить пользователям включать при присоединении к собранию **" в** разделе **"Включить камеру этого устройства"**. Этот параметр нельзя отключить, если включена камера  этого _устройства_.
9. Если **доска по** `true`умолчанию на листе установлена, включите доску на этом  устройстве _, в_ противном случае оставьте _ее отключенной_.
10. В **разделе "Доверенные учетные записи устройств**" введите каждое имя участника-пользователя, указанное в списке доверенных учетных записей на листе. Разделите несколько имен участника-пользователей запятыми.
11. На доверенном устройстве отключите удаленное расположение и помещение. 
12. Нажмите **кнопку "Сохранить" и "Выйти"**.

После выбора команды **"Сохранить и выйти**" устройство перезапустится, и оно будет готово к участия в координированном собрании.

### <a name="use-the-teams-rooms-xml-configuration-file"></a>Использование Комнаты Teams XML-файла конфигурации

Координированное собрание можно настроить с Комнаты Teams `SkypeSettings.xml` XML-файла конфигурации устройства. Файл `SkypeSettings.xml` не является статическим. При запуске Комнаты Teams устройство выполняет проверку `C:\Users\Skype\AppData\Local\Packages\Microsoft.SkypeRoomSystem_8wekyb3d8bbwe\LocalState` на наличие файла с именем `SkypeSettings.xml`. Если файл существует, устройство считывает и применяет конфигурацию, указанную в файле. После применения конфигурации файл удаляется. Дополнительные сведения о файле см `SkypeSettings.xml` . в разделе ["Управление параметрами консоли с помощью XML-файла конфигурации"](../rooms/xml-config-file.md#manage-console-settings-with-an-xml-configuration-file).

Ниже приведен синтаксис параметров координированного собрания в файле конфигурации:

```xml
<CoordinatedMeetings enabled="true">
    <Settings>
        <Audio default="true" enabled="true"/>
        <Video default="true" enabled="true"/>
        <Whiteboard default="false" enabled="false"/>
    </Settings>
    <TrustedAccounts>hub@contoso.com</TrustedAccounts>
</CoordinatedMeetings>
```

Чтобы настроить координированное собрание на устройстве, сделайте следующее:

1. В редакторе текстовых файлов, например Visual Studio Code или Блокноте, вставьте приведенный выше XML-файл в новый файл.

2. Задайте для каждого XML-элемента соответствующее `true` значение `false` или значение в электронной таблице. Например, если по умолчанию **используется audio**`true`, задайте значение .`<Audio default="true">`

3. Обязательно измените список `TrustedAccounts` имен участника-пользователей.

4. Сохраните файл с именем `SkypeSettings.xml`.

5. Поместите файл в папку Комнаты Teams устройства`C:\Users\Skype\AppData\Local\Packages\Microsoft.SkypeRoomSystem_8wekyb3d8bbwe\LocalState`. Это можно сделать несколькими способами:

    - **Скопируйте файл на устройство Комнаты Teams** необходимо включить общий доступ к файлам и создать сетевую папку, прежде чем можно будет скопировать файлы на устройство. После этого можно подключиться к сетевой папке и скопировать файл на устройство. Дополнительные сведения см[. в Комнаты Microsoft Teams обслуживания и операций](../rooms/rooms-operations.md).
    - **Используйте групповая политика** создать групповую политику для копирования файла на устройство. Дополнительные сведения см[. в групповая политика обзоре](/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/hh831791(v=ws.11)).
    - **Скачайте файл на** Комнаты Teams устройства. Вы можете войти на устройство с помощью Администратор, а затем скопировать файл на устройство из сетевой папки или USB-накопителя. Дополнительные сведения см. в разделе ["Переключение Администратор режиме"](../rooms/rooms-operations.md#switching-to-admin-mode-and-back-when-the-microsoft-teams-rooms-app-is-running).
    
6. Перезапустите устройство. Это можно сделать несколькими способами:

    - **Удаленная служба PowerShell** Вы можете выполнить команду Shutdown на устройстве с помощью удаленной оболочки PowerShell. Дополнительные сведения см. в [разделе "Удаленное управление с помощью PowerShell"](../rooms/rooms-operations.md).
    - **Запуск restart-computer** Вы можете запустить командлет `Restart-Computer` на локальном компьютере и указать имя компьютера устройства, которое требуется перезапустить. Дополнительные сведения см. в [разделе Restart-Computer](/powershell/module/microsoft.powershell.management/restart-computer?view=powershell-7).

## <a name="step-5-configure-surface-hub"></a>Шаг 5. Настройка Surface Hub

Конструктор конфигураций Windows можно использовать для создания пакета подготовки, который можно использовать для применения параметров координирования собраний к устройствам Surface Hub. Чтобы создать пакет подготовки, вставьте созданный ранее XML-файл в конструктор конфигураций Windows.

### <a name="create-coordinated-meetings-xml-configuration-file-for-surface-hub"></a>Создание XML-файла конфигурации координированного собрания для Surface Hub

Конструктор конфигураций Windows и Microsoft Intune используются для применения конфигурации "Координированное собрание" к устройствам Surface Hub. Конфигурация определяется с помощью XML. Прежде чем продолжить, необходимо создать XML-код, который будет применен.

Ниже приведен синтаксис XML-файла конфигурации координированного собрания.

```xml
<SurfaceHubSettings>
    <BluetoothAdvertisementEnabled>true</BluetoothAdvertisementEnabled>
    <AutoAcceptProximateMeetingInvitations>true</AutoAcceptProximateMeetingInvitations>
    <CoordinatedMeetings enabled="true"> 
        <TrustedAccounts>room@contoso.com</TrustedAccounts>
        <Settings> 
            <Audio default="false" enabled="false" />
            <Video default="false" enabled="true" /> 
        </Settings> 
    </CoordinatedMeetings>
</SurfaceHubSettings>
```

Выполните следующие действия, чтобы подготовить XML-код для конструктора конфигураций Windows или Microsoft Intune:

1. В редакторе текстовых файлов, например Visual Studio Code или Блокноте, вставьте приведенный выше XML-файл в новый файл.

2. Задайте для каждого XML-элемента соответствующее `true` значение `false` или значение в электронной таблице. Например, если по умолчанию **используется audio**`true`, задайте значение .`<Audio default="true">`

3. Обязательно измените список `TrustedAccounts` имен участника-пользователей.

4. Конструктор конфигураций Windows требует, чтобы XML-код был в одной строке. Удалите все разрывы строк между строками, чтобы XML-код выглядел следующим образом:

    ```xml
    <SurfaceHubSettings><BluetoothAdvertisementEnabled>true</BluetoothAdvertisementEnabled>...
    ```

5. Сохраните файл на компьютере.

После создания XML-файла конфигурации выполните действия, описанные в разделе "Управление параметрами [Microsoft Teams" на Surface Hub](surface-hub-manage-config.md) , чтобы применить его к устройствам Surface Hub.
