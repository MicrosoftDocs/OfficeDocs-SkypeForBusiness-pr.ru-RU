# <a name="set-up-coordinated-meetings-with-microsoft-teams-rooms-and-surface-hub"></a>Настройка согласованных собраний с помощью комнат и Surface Hub Microsoft Teams

Если у вас есть одна или несколько устройств Microsoft Teams или Surface Hub в комнате для собраний, вы можете настроить согласованные собрания. С помощью координированных собраний вы можете настроить устройства и Surface Hub для групп, чтобы при присоединении к собранию на одном устройстве другие устройства в комнате были также присоединены к тому же собранию. Вы можете настроить камеры, динамики и микрофоны таким образом, чтобы пользователи, которые применяют собеседников, были включены, пока другие отключены. Таким образом, при добавлении нескольких устройств к собранию могут возникать проблемы с участием в Dreaded эха и обратной связи.

Чтобы настроить согласованные собрания, необходимо убедиться в том, что на устройствах и Surface Hub уже настроены для участия в собраниях. Важнее всего, что для каждого устройства требуется собственный почтовый ящик в комнате Exchange. Сведения о том, как их настраивать, можно найти в следующих статьях:

- [Развертывание комнат Microsoft Teams](../rooms/rooms-deploy.md)
- [Создание Surface Hub 2S учетная запись устройства](https://docs.microsoft.com/surface-hub/surface-hub-2s-account)

После подтверждения того, что ваши устройства и Surface Hub смогут автоматически присоединиться к собраниям и получать к ним успешные собрания, вы можете настроить согласование собраний.

Следующие действия необходимо выполнить отдельно для каждой комнаты для собраний. Устройства в одной комнате для собраний не следует настраивать для согласованных собраний с устройствами в других помещениях для собраний.

## <a name="step-1-plan-your-coordinated-meeting-experience"></a>Шаг 1: планирование согласованного процесса собраний

Прежде чем вносить изменения в конфигурацию, необходимо решить, какие устройства будут выполнять эти действия в каждой комнате для собраний. Для заданной комнаты для собраний вам нужно решить, на каком устройстве будет находиться активный микрофон, Камера и доска. Способ настройки устройств зависит от конкретной среды, но ниже приведены некоторые общие рекомендации по началу работы.

- **Microphone (микрофон** ) Устройство "комнаты рабочих групп"
- **Камера** Устройства Teams (по умолчанию) и Surface Hub (по умолчанию отключены, но разрешены для включения участниками)
- **Доска** Surface HUB

> [!IMPORTANT]
> Убедитесь, что микрофон включен только на одном устройстве. Если вы включите ее на нескольких устройствах, вы получите звуковой эхо и отзыв.

## <a name="step-2-get-your-devices-upns"></a>Действие 2: получите UPN устройства.

Когда вы настраиваете координированный процесс собрания в комнате для собраний, вам нужно сообщить устройствам и Surface Hub в этой комнате, с какими устройствами они должны координироваться. Это можно сделать, добавив имя участника-пользователя (UPN) устройств, которое оно должно координировать с его конфигурацией. Если вы не знаете UPN для каждого из устройств, которые вы хотите настроить для согласованных собраний, вы можете найти их с помощью центра администрирования Microsoft 365. 

Чтобы получить доступ к центру администрирования Microsoft 365, вам необходимо назначить роль администратора. Дополнительные сведения можно найти в разделе [о ролях администратора](https://docs.microsoft.com/microsoft-365/admin/add-users/about-admin-roles).

Чтобы получить UPN для устройств и Surface Hub в Teams, выполните указанные ниже действия.

1. Войдите в центр администрирования Microsoft 365, посетив https://admin.microsoft.com
2. Перейти к **Users**  >  **активным пользователям** пользователей
3. Найдите имя устройства Teams и Surface Hub в столбце **Display Name** (вы можете использовать поле **поиска** , если у вас много пользователей).
4. Найдите имя участника-пользователя в столбце **username** (оно будет выглядеть так, как alias@contoso.com или alias@contoso.onmicrosoft.com)
5. Повторите эти действия для каждого устройства, которое будет принимать участие в согласованных собраниях.

## <a name="step-3-create-a-deployment-worksheet"></a>Шаг 3: Создание листа развертывания

После того как вы запланировали согласованный процесс собрания и собрали список UPN ваших устройств, рекомендуется создать лист развертывания. Лист развертывания поможет вам визуализировать конфигурацию, которую вы хотите установить на всех устройствах, позволяя проверить выбранные параметры и проверить наличие ошибок.

В приложении для электронной таблицы добавьте в первый столбец строки, указанные ниже.

| Параметр                | Описание      |
|------------------------|-----------------|
| **Звук по умолчанию**      | Определение того, на каком устройстве будет активен микрофон при запуске собрания. Это поле может иметь только одно устройство (обычно — комнаты рабочих помещений), `true` так как в остальных устройствах должно быть установлено это поле, чтобы `false` избежать появления звукового эха и обратной связи.          |
| **Включено звуковое сопровождение**      | Определяет, могут ли участники собрания включать или отключать микрофон. Для устройств, для которых **по умолчанию** задано значение " `false` должен быть настроен `false` , чтобы участники не могли случайно включить микрофон, а также привести к голосовой эхо или отзыву.<p>Если задано значение **по умолчанию** `true` , этот параметр игнорируется, а участники могут отключить или включить микрофон.          |
| **Видео по умолчанию**      | Определяет, на каком устройстве камера будет активна при запуске собрания. Для оптимальной работы мы рекомендуем использовать только устройство в комнатах Teams, `true` пока установлены все другие устройства `false` .          |
| **Видео включено**      | Определяет, могут ли участники собрания включать или отключать камеру. Вы можете настроить этот параметр `true` на любые другие устройства в участниках событий, которым нужно поделиться различными видеороликами (например, если участник использует доску Surface HUB). Если вы не хотите, чтобы участники отключили или отключили камеру на устройстве, установите для этого параметра значение `false` .<p> Если для видео установлено значение **по умолчанию** `true` , этот параметр игнорируется, а участники могут включать и отключать камеру.         |

Рекомендуем установить этот `false` флажок, если у вас есть Surface Hub, и `true` Если у вас его нет. Этот параметр не влияет на концентраторы Surface Hub. Surface Hub всегда будут показывать доску, доступную для участников собрания.         |
| **Доверенные учетные записи**   | Это разделенный запятыми список UPN для каждого устройства комнаты команд или Surface Hub, на котором устройство должно принимать запросы на присоединение к собранию или на которые должны отправляться запросы на присоединение к собранию. |

В последующих столбцах добавьте все устройства и Surface Hub, расположенные в рабочих центрах. В каждом столбце введите значения, соответствующие требуемому варианту для комнаты для собраний. Ниже приведен пример с одним устройством, комнатными группами, и одним Surface Hub:

  - При запуске собрания видео **отключается** . Участники **могут** включать и отключать видео.

## <a name="step-4-configure-teams-rooms-device"></a>Действие 4: Настройка комнаты на устройстве для рабочих групп

Вы можете настроить координированные собрания на устройстве рабочих групп, используя сенсорный экран устройства, или, если вам нужно настроить большое количество устройств и сделать это из центрального, можно использовать XML-файл конфигурации.

С помощью листа, созданного на предыдущем шаге, вы сможете настроить устройства.

### <a name="use-the-teams-rooms-devices-touch-screen"></a>Использование сенсорного экрана устройства "комнаты для команд"

Чтобы настроить согласованные собрания на устройстве, выполните указанные ниже действия.

1. SELECT **... Дополнительные**  >  **Параметры**
2. Введите пароль администратора и нажмите **кнопку Да** .
3. Выбор **согласованных собраний**
4. В разделе **Параметры**установите для параметра **согласованное собрание** значение включено.
5. Если на листе задано **значение по умолчанию** `true` , **включите микрофон для этого устройства** , в противном случае оставьте его недоступным.
6. Если на листе **включено звуковое сопровождение** `true` , выберите параметр Разрешить **пользователям включать при присоединении к собранию** в разделе **включить микрофон этого устройства**. Этот параметр нельзя отключить, если **флажок включить микрофон на этом устройстве** установлен на включен.
7. Если на листе используется **видео по умолчанию** `true` , установите **флажок Включить камеру на устройстве** , в противном случае оставить ее выключенной.
8. Если на листе **включено видео** `true` , выберите параметр Разрешить **пользователям включать при присоединении к собранию** в разделе **включить камеру этого устройства**. Этот параметр нельзя отключить, если включена **Камера на устройстве** с установленным флажком вкл.
9. Если на листе используется **значение по умолчанию** `true` , установите **флажок Включить доску на устройстве** , в противном случае оставить ее выключенной.
10. В разделе **учетные записи доверенных устройств**введите каждое имя участника-пользователя в списке **доверенные учетные записи** на листе. Разделяйте несколько UPN запятыми.
11. Нажмите кнопку **сохранить и выйти** .

После того как вы выберете команду **сохранить и выйти**, устройство будет перезапущено, и оно будет готово к участию в согласованных собраниях.

### <a name="use-the-teams-rooms-xml-configuration-file"></a>Использование XML-файла конфигурации комнат рабочих групп

Согласованное собрание можно настроить с помощью `SkypeSettings.xml` XML-файла конфигурации устройства Teams в комнатах. `SkypeSettings.xml`Файл не является статическим файлом. При запуске команды запускается устройство, в котором выполняется `C:\Users\Skype\AppData\Local\Packages\Microsoft.SkypeRoomSystem_8wekyb3d8bbwe\LocalState` Поиск файла с именем `SkypeSettings.xml` . Если файл существует, устройство считывает и применяет конфигурацию, указанную в файле. После того как вы закончите применение конфигурации, файл будет удален. Дополнительные сведения о `SkypeSettings.xml` файле можно найти в разделе [Управление параметрами консоли с помощью XML-файла конфигурации](../rooms/xml-config-file.md#manage-console-settings-with-an-xml-configuration-file).

Ниже приведен синтаксис параметров координированных собраний в файле конфигурации.

```xml
<CoordinatedMeetings enabled="true">
    <Settings>
        <Audio default="true" enabled="true"/>
        <Video default="true" enabled="true"/>
        <Whiteboard default="false"/>
    </Settings>
    <TrustedAccounts>hub@contoso.com</TrustedAccounts>
</CoordinatedMeetings>
```

Чтобы настроить согласованные собрания на устройстве, выполните указанные ниже действия.

1. В редакторе текстового файла, например в коде Visual Studio или в блокноте, вставьте вышеприведенные XML-файлы в новый файл
2. Задайте для каждого из элементов XML соответствующие `true` `false` значения в электронной таблице. Например, если для звука задано значение **по умолчанию** `true` , установлено `<Audio default="true">` .
3. Не забудьте изменить `TrustedAccounts` список UPN
4. Сохранение файла с именем `SkypeSettings.xml`
5. Поместите файл в папку устройства "комнаты для Teams" `C:\Users\Skype\AppData\Local\Packages\Microsoft.SkypeRoomSystem_8wekyb3d8bbwe\LocalState` . Это можно сделать несколькими способами.
    - **Копирование файла в комнаты рабочих групп** Чтобы можно было копировать файлы на устройство, необходимо включить общий доступ к файлам и создать общую сетевую сеть. После этого вы сможете подключиться к общему сетевому доступу и скопировать файл на устройство. Дополнительные сведения можно найти в разделе [обслуживание и эксплуатация комнат Microsoft Teams](../rooms/rooms-operations.md).
    - **Использование групповой политики** Создайте групповую политику, чтобы скопировать файл на устройство. Дополнительные сведения можно найти в разделе [Обзор групповой политики](https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/hh831791(v=ws.11)).
    - **Скачивание файла на устройстве "комнаты рабочих групп"** Вы можете войти в систему с помощью режима администратора, а затем скопировать файл на устройство с сетевого диска или USB-накопителя. Дополнительные сведения можно найти [в разделе переключение в режим администратора](../rooms/rooms-operations.md#switching-to-admin-mode-and-back-when-the-microsoft-teams-rooms-app-is-running).
6. Перезапустите устройство. Это можно сделать двумя способами:
    - **Удаленная оболочка PowerShell** Вы можете выполнить команду shutdown на устройстве с помощью удаленной оболочки PowerShell. Дополнительные сведения можно найти [в разделе Удаленное управление с помощью PowerShell](../rooms/rooms-operations.md).
    - **Запуск перезапуска компьютера** Вы можете запустить `Restart-Computer` командлет на локальном компьютере и указать имя компьютера, на котором вы хотите перезапустить устройство. Дополнительные сведения можно найти в разделе [перезапуск компьютера](https://docs.microsoft.com/powershell/module/microsoft.powershell.management/restart-computer?view=powershell-7).

## <a name="step-5-configure-surface-hub"></a>Шаг 5: Настройка Surface HUB

Вы можете использовать конструктор конфигураций Windows для создания пакета подготовки, который можно использовать для применения параметров согласования собраний с Surface Hub. Чтобы создать пакет подготовки, вы можете вставить созданный ранее XML-файл в конструктор конфигураций Windows.

### <a name="create-coordinated-meetings-xml-configuration-file-for-surface-hub"></a>Создание XML-файла конфигурации согласованных собраний для Surface HUB

Как конструктор конфигураций Windows, так и Microsoft Intune используются для применения конфигурации согласованных собраний к концентраторам Surface Hub. Конфигурация определяется с помощью XML. Прежде чем продолжить, необходимо создать XML-файл, который будет применен.

Ниже приведен синтаксис файла конфигурации XML для собраний.

```xml
<SurfaceHubSettings>
    <BluetoothAdvertisementEnabled>true</BluetoothAdvertisementEnabled>
    <AutoAcceptProximateMeetingInvitations>true</AutoAcceptProximateMeetingInvitations>
    <CoordinatedMeetings enabled="true"> 
        <TrustedAccounts>room@contoso.com</TrustedAccounts>
        <Settings> 
            <Audio default="false" enabled="false" />
            <Video default="false" enabled="true" /> 
        </Settings> 
    </CoordinatedMeetings>
</SurfaceHubSettings>
```

Выполните описанные ниже действия, чтобы подготовить конструктор конфигураций XML для Windows или Microsoft Intune.

1. В редакторе текстового файла, например в коде Visual Studio или в блокноте, вставьте вышеприведенные XML-файлы в новый файл
2. Задайте для каждого из элементов XML соответствующие `true` `false` значения в электронной таблице. Например, если для звука задано значение **по умолчанию** `true` , установлено `<Audio default="true">` .
3. Не забудьте изменить `TrustedAccounts` список UPN
4. Конструктор конфигураций Windows требует, чтобы XML настоять из одной строки. Удалите все разрывы строк между строками, чтобы XML-код выглядел следующим образом:

    ```xml
    <SurfaceHubSettings><BluetoothAdvertisementEnabled>true</BluetoothAdvertisementEnabled>...
    ```

5. Сохранение файла на компьютере

После создания XML-файла конфигурации выполните действия, описанные в статье [Управление параметрами Microsoft Teams на Surface Hub](surface-hub-manage-config.md) , чтобы применить ее к концентраторам поверхностей.
