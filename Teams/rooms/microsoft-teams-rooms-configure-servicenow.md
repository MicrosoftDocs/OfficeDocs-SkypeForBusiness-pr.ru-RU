# <a name="configure-servicenow-for-teams-rooms"></a>Настройка ServiceNow для Комнаты Teams

В этой статье описаны предварительные требования и действия по настройке среды ServiceNow на портале управления Комнаты Teams Pro.

## <a name="watch-microsoft-teams-rooms-pro-management--service-now-integration"></a>Просмотр: управление Комнаты Microsoft Teams Pro — интеграция службы

> [!VIDEO https://www.microsoft.com/en-us/videoplayer/embed/RE4ZK4B]


### <a name="teams-rooms-prerequisites"></a>предварительные требования Комнаты Teams

- Вам должна быть назначена роль администратора служб. Дополнительные сведения см. в статье [Управление доступом на основе ролей с помощью управления Комнаты Microsoft Teams Pro](rooms-pro-rbac.md).

### <a name="servicenow-prerequisites"></a>Предварительные требования к ServiceNow

- Вход с обычной авторизацией или вход [OAuth](https://docs.servicenow.com/bundle/rome-platform-administration/page/administer/security/concept/c_OAuthApplications.html) . Дополнительные сведения см. в разделе [Создание учетных данных](https://developer.servicenow.com/dev.do#!/learn/learning-plans/rome/servicenow_application_developer/app_store_learnv2_rest_rome_creating_credentials) в ServiceNow.
- Экземпляр ServiceNow и его имя узла экземпляра и URI API
- Роль incident_manager или выше
- Программная версия ServiceNow, поддерживающая API таблиц.

## <a name="configure-your-environment"></a>Настройка среды

Настройка среды зависит от потребностей вашей организации. Ниже описано, как скопировать существующую конфигурацию в ServiceNow на портал управления Комнаты Teams Pro.

1. Откройте экземпляр ServiceNow, который требуется скопировать. Вам потребуется ссылаться на это при заполнении формы конфигурации на портале управления Комнаты Teams Pro.
2. На новой вкладке браузера перейдите на [портал управления Комнаты Teams Pro](https://portal.rooms.microsoft.com/) и **перейдите в раздел Параметры**. Затем выберите **ServiceNow** в меню навигации слева, чтобы открыть форму конфигурации.
3. Выберите метод проверки подлинности для входа и введите узел экземпляра ServiceNow и URI API.
4. Все необходимые элементы в столбце Поле ServiceNow раздела Сопоставление полей должны быть предварительно заполнены. В приведенной ниже таблице содержатся каждое поле ServiceNow и соответствующее Комнаты Microsoft Teams поле. Выполните действие для каждой строки раздела Сопоставление полей. Определения каждого поля ServiceNow см. в разделе [Определения полей ServiceNow](#servicenow-field-definitions).

| Поле ServiceNow | поле Комнаты Microsoft Teams | Действие |
| --- | --- | --- |
| short_description | Описание инцидента | Никаких действий не требуется. Поле Комнаты Teams заполняется автоматически. |
| Описание | Первое сообщение | Никаких действий не требуется. Поле Комнаты Teams заполняется автоматически. |
| assignment_group | Группа комнат | Скопируйте значение assignment_group в экземпляре ServiceNow и вставьте его в поле Значение ServiceNow в форме конфигурации. Если у вас несколько assignment_group, выберите **Добавить группу комнат** для каждого дополнительного настраиваемого значения. |
| Тяжести | Кольца | Серьезность — это пользовательское значение в ServiceNow. Это четвертый элемент во втором столбце экземпляра ServiceNow. Скопируйте значение и вставьте его в поле ServiceNow value в форме конфигурации. Если у вас несколько значений серьезности, выберите **Добавить кольцо** для каждого дополнительного настраиваемого значения. |
| Комментарии (необязательно) | Пользовательское значение* | Чтобы добавить поле примечаний в форму конфигурации, выберите **Добавить** в верхней части раздела сопоставления полей. Скопируйте значение комментария в экземпляре ServiceNow и вставьте его в поле ServiceNow в форме конфигурации. Назначьте ему поле Комнаты Microsoft Teams из раскрывающегося меню, а затем скопируйте и вставьте значение ServiceNow. |
| состояние (разрешено) | Пользовательское значение* | Скопируйте состояние разрешения из экземпляра ServiceNow и вставьте его в поле Значение ServiceNow в форме конфигурации. |
| close_code | Пользовательское значение* | На вкладке **Сведения о разрешении** экземпляра ServiceNow скопируйте код закрытия и вставьте его в поле Значение ServiceNow в форме конфигурации. |

*Выберите пользовательские значения в раскрывающемся меню

>[!NOTE]
>Если вам не удается найти пользовательские значения, обратитесь к администратору ServiceNow.

Чтобы добавить дополнительные обязательные поля в раздел Разрешение инцидента, нажмите кнопку **Добавить**.

## <a name="test-and-enable"></a>Тестирование и включение

После завершения формы конфигурации выберите **Тест** в нижней части страницы. Для отправки конфигурации требуется тестирование.

После успешного прохождения теста нажмите **кнопку Отправить** , чтобы сохранить изменения. Когда вы будете готовы включить ServiceNow для своей организации, установите **переключатель Включить ServiceNow?** в **положение Вкл**.

## <a name="servicenow-field-definitions"></a>Определения полей ServiceNow

- **short_description**. Краткое поле описания в ServiceNow представляет собой краткое 160-значное буквенно-цифровое значение, которое содержит сводку инцидента. Краткое описание эквивалентно описанию инцидента на портале управления Комнаты Teams Pro.

- **описание**: поле описания в ServiceNow является первым значением в журнале бесед инцидента ServiceNow. Описание эквивалентно первому сообщению на портале управления Комнаты Teams Pro.

- **assignment_group**. Поле группы назначений в ServiceNow используется для организации инцидентов. Группы назначений эквивалентны группам комнат на портале управления Комнаты Teams Pro. По умолчанию существует одна группа помещений, и можно добавить другие. Вы решаете, сколько групп и как группировать инциденты. Например, вы можете упорядочить инциденты по расположению.

- **серьезность**. Поле серьезности в ServiceNow используется для упорядочения инцидентов по приоритету. Значения, обозначающие приоритет, можно настроить. Серьезность эквивалентна полю Ring на портале управления Комнаты Teams Pro. Чтобы настроить круги на портале управления Комнаты Teams Pro, перейдите к **Обновления** в меню навигации слева. Затем перейдите на вкладку **Кольца** и выберите **Добавить кольцо**.

- **комментарии**. Примечания — это необязательное поле в ServiceNow, которое используется для включения настраиваемых обязательных полей из экземпляра ServiceNow в конфигурацию портала управления Комнаты Teams Pro. Эквивалент комментариев — это пользовательское значение на портале управления Комнаты Teams Pro.

- **state (resolved)**: поле состояние (разрешено) в ServiceNow используется для обозначения способа разрешения инцидента и требуется для закрытия инцидента. Значение состояния (разрешено) настраивается. Эквивалент состояния (разрешено) — это настраиваемое значение на портале управления Комнаты Teams Pro.

- **close_code**. После полного устранения инцидента необходимо назначить код закрытия. Это значение можно настроить в ServiceNow. Эквивалентом кода закрытия является пользовательское значение на портале управления Комнаты Teams Pro.
