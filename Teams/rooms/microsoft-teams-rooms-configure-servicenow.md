# <a name="configure-servicenow-for-teams-rooms"></a>Настройка ServiceNow для Комнаты Teams

В этой статье описаны предварительные условия и действия по настройке среды ServiceNow на Комнаты Teams Premium портале.

## <a name="before-you-begin"></a>Подготовка к работе

### <a name="teams-rooms-prerequisites"></a>Комнаты Teams необходимые условия

- Вам должна быть назначена роль администратора службы. Дополнительные сведения см. в теме Управление доступом на основе ролей [с помощью Комнаты Microsoft Teams управляемых служб.](microsoft-teams-rooms-premium-rbac.md)

### <a name="servicenow-prerequisites"></a>ServiceNow prerequisites

- Вход в базовую авторизацию или вход в OAuth. Дополнительные сведения см. [в теме Создание учетных данных](https://developer.servicenow.com/dev.do#!/learn/learning-plans/rome/servicenow_application_developer/app_store_learnv2_rest_rome_creating_credentials) в serviceNow.
- Экземпляр ServiceNow и имя хоста экземпляра и URI API
- Роль incident_manager более высокого
- Программная версия ServiceNow, которая поддерживает API таблиц

## <a name="set-up-servicenow-configuration-to-teams-rooms"></a>Настройка конфигурации ServiceNow для Комнаты Teams

Конфигурация среды в значительной степени настраивается и зависит от потребностей организации. Ниже вы можете скопировать существующую конфигурацию в ServiceNow на Комнаты Teams Premium портал.

1. Откройте экземпляр ServiceNow, который нужно скопировать. По мере завершения формы конфигурации на портале необходимо будет Комнаты Teams Premium этой ссылке.
2. На новой вкладке браузера перейдите на Комнаты Teams Premium [и](https://portal.rooms.microsoft.com/) перейдите **в** Параметры . Затем выберите **ServiceNow в** меню навигации слева, чтобы открыть форму конфигурации.
3. Выберите способ проверки подлинности, чтобы войти в службу и ввести свой хост экземпляра ServiceNow и URI API.
4. Все необходимые элементы в столбце Поле ServiceNow раздела Сопоставление полей должны быть заранее заполнены. В приведенной ниже таблице содержатся каждое поле ServiceNow и соответствующее Комнаты Microsoft Teams поля. Выполните действие для каждой строки раздела Сопоставление полей. Определения каждого поля ServiceNow см. в определении поля [ServiceNow.](#servicenow-field-definitions)

| **Поле ServiceNow** | **Комнаты Microsoft Teams поле** | **Описание** |
| --- | --- | --- |
| short_description | Описание инцидента | Никаких действий не требуется. Поле Комнаты Teams заполнено автоматически. |
| Описание | Первое сообщение | Никаких действий не требуется. Поле Комнаты Teams заполнено автоматически. |
| assignment_group | Группа комнат | Скопируйте assignment_group в экземпляре ServiceNow и в виде конфигурации в поле Значение ServiceNow. Если у вас несколько assignment_group, выберите Добавить группу **комнат** для каждого дополнительного пользовательского значения. |
| Тяжести | Кольца | Серьезность — это пользовательское значение в ServiceNow. Это четвертый элемент во втором столбце экземпляра ServiceNow. Скопируйте значение и вкопируйте его в поле ServiceNow в форме конфигурации. Если у вас несколько значений серьезности, выберите **Добавить звонок** для каждого дополнительного настраиваемого значения. |
| При комментарии (необязательно) | Пользовательское значение* | Чтобы добавить поле комментариев в форму конфигурации, выберите **Добавить** в верхней части раздела сопоставления полей. Скопируйте значение комментария в экземпляре ServiceNow и в поле ServiceNow в форме конфигурации. Назначьте поле Microsoft Teams комнаты из меню и скопируйте и вкопируйте значение ServiceNow. |
| состояние (разрешено) | Пользовательское значение* | Скопируйте состояние разрешения из экземпляра ServiceNow и в поле "Значение ServiceNow" в форме конфигурации. |
| close_code | Пользовательское значение* | На **вкладке Сведения** о решении экземпляра ServiceNow скопируйте код закрытия и вкопируйте его в поле "Значение ServiceNow" в форму конфигурации. |

* Выберите настраиваемые значения в меню

>[!NOTE]
>Если вам не удалось найти пользовательские значения, обратитесь к администратору ServiceNow.

Чтобы добавить дополнительные необходимые поля в раздел Устранить инцидент, выберите **Добавить**.

## <a name="test-and-enable-your-servicenow-configuration"></a>Проверка и включить конфигурацию ServiceNow

Заполнив форму конфигурации, выберите **Проверить** в нижней части страницы. Для отправки конфигурации необходимо тестирование.

После успешного тестирования выберите Отправить, **чтобы** сохранить изменения. Когда вы будете готовы включить ServiceNow для своей организации, переключите переключатель Включить **ServiceNow?** в **переключатель Вкл.**.

## <a name="servicenow-field-definitions"></a>Определения полей ServiceNow

- **short_description:** поле краткого описания в поле ServiceNow представляет собой краткое 160-символное букватическое число, которое представляет собой сводку по инциденту. Краткое описание эквивалентно описанию инцидента на Комнаты Teams Premium портале.

- **Описание:** поле описания в поле ServiceNow является первым значением в истории бесед инцидента ServiceNow. Описание эквивалентно первому сообщению на Комнаты Teams Premium портале.

- **assignment_group:** поле группы назначений в ServiceNow используется для у упорядочения инцидентов. Группа заданий эквивалентна группам комнат на Комнаты Teams Premium портале. По умолчанию существует одна группа комнат, и можно добавить и другие комнаты. Вы решаете, сколько групп в них и как сгруппть свои инциденты. Например, вы можете упорядочить инциденты по расположению.

- **серьезность.** Поле серьезности в поле ServiceNow используется для уорганизовывай инцидентов по приоритету. Значения, которые обозначают приоритет, настраиваются. Серьезность эквивалентна полю "Звонок" на Комнаты Teams Premium портале. Чтобы настроить круги на Комнаты Teams Premium, перейдите к пункту Обновления **в** меню навигации слева. Затем перейдите на **вкладку Кольца** и выберите **Добавить круг**.

- **комментарии.** Комментарии — это необязательное поле в serviceNow, которое используется для того, чтобы включить настраиваемые поля из экземпляра ServiceNow в конфигурацию Комнаты Teams Premium портала. Эквивалентом комментариев является пользовательское значение на Комнаты Teams Premium портале.

- **состояние (разрешено):** поле "Состояние" (разрешено) в поле ServiceNow используется для разрешения инцидента и его необходимо закрыть. Значение состояния (разрешено) настраивается. Эквивалентом состояния (разрешено) является пользовательское значение на Комнаты Teams Premium портале.

- **close_code:** инциденту после его полного решения должен быть назначен код закрытия. Это значение можно настроить в ServiceNow. Эквивалентом кода закрытия является пользовательское значение на Комнаты Teams Premium портале.
