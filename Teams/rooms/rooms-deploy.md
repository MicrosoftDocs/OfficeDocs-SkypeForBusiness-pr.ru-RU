# <a name="deployment-overview"></a>Обзор развертывания

Развертывание Комнаты Microsoft Teams фактически разбивается на этапы:

- Подтверждение того, что расположения (пробелы) соответствуют зависимостей развертывания
- Создание Microsoft Teams или Skype для бизнеса и Exchange учетных записей и назначение их Комнаты Teams (см. [Комнаты Microsoft Teams)](rooms-configure-accounts.md)
- (Необязательно) Настройка монитора Azure для систем (см. Комнаты Microsoft Teams [развертывания с помощью Azure Monitor.](azure-monitor-deploy.md)
- Настройка Комнаты Teams собраний и подключение необходимых периферийных устройств (см. документацию OEM для набора устройств)

## <a name="site-readiness"></a>Готовность сайта 

Когда заказаные устройства доставляются в организацию, работайте с сетевыми группами, помещениями и группами av, чтобы убедиться в том, что зависимости развертывания выполнены, и каждый сайт и пространство готовы к работе с точки зрения питания, сетевого обслуживания и отображения. Кроме того, убедитесь, что выполнены физические требования к установке. В вопросах физической установки обратитесь к поставщику и используйте возможности вашей av team при установке и установке экранов и под управлением кабелей.

Дополнительные информацию об этих зависимостей см. в указанных ниже ссылках на рекомендации по планированию.

-   [Проверка доступности сети](rooms-prep.md#check-network-availability)
-   [Сертификаты](rooms-prep.md#certificates)
-   [Прокси-сервер](rooms-prep.md#proxy)

**Pro совет.** Если для предоставления доступа к Teams необходимо использовать прокси-серверы, сначала [просмотрите эту статью.](../proxy-servers-for-skype-for-business-online.md) Если речь идет о Microsoft Teams трафике мультимедиа в режиме реального времени через прокси-серверы, мы рекомендуем полностью обойти прокси-серверы. Microsoft Teams трафик уже зашифрован, поэтому прокси-серверы не делают его более безопасным и добавляют задержку к трафику в режиме реального времени. В рамках более широкой развертывания мы рекомендуем [](../prepare-network.md) следовать указаниям из Teams подготовки сети к пропускной способности для планирования пропускной способности и оценки пригодности сети для трафика в режиме реального времени.

## <a name="service-readiness"></a>Готовность к службе

Чтобы подготовиться к развертыванию Комнаты Microsoft Teams, сделайте следующее:

-   Определите Комнаты Microsoft Teams ресурсов.
-   Если вы Teams для Azure Active Directory, подготовьте группу Azure AD с динамическим членством для удержания всех учетных записей Комнаты Teams ресурсов. Это упростит управление в будущем, например применение политик условного доступа. Чтобы проще всего использовать динамические группы Azure AD, определите соглашение об именово окном, которое однозначно определит Комнаты Teams ресурсов.
-   Присоединившись к Teams Room to Active Directory, подготовьте подразделение и группу Active Directory для удержания учетных записей компьютера и ресурсов Комнаты Microsoft Teams и (при желании) подготовьте объекты групповой политики (GPOS), чтобы включить перезапев.

### <a name="prepare-to-host-microsoft-teams-rooms-and-resource-accounts-optional"></a>Подготовка к выбору учетных Комнаты Microsoft Teams ресурсов и ресурсов (необязательно)

Чтобы можно было управлять учетными Комнаты Microsoft Teams ресурсами и управлять ими, подготовьте свои локальное каталоги Active Directory Azure Active Directory (Azure AD). 

Определите локальное каталоге или группу Azure Active Directory, в которые нужно добавить Комнаты Microsoft Teams ресурсов. При использовании Azure Active Directory использовать динамическую группу для автоматического добавления и удаления учетных записей ресурсов из группы.

Определите одну организационную единицу в локальной иерархии Active Directory для удержания всех учетных записей Комнаты Microsoft Teams компьютера (если они соединены с доменом) и одного подразделения для всех учетных записей Комнаты Microsoft Teams пользователей. Отключать наследование групповой политики, чтобы гарантировать применение только политик, которые должны применяться к доменным Комнаты Microsoft Teams.

Создайте объект групповой политики, который назначен блоку организации, содержа Комнаты Microsoft Teams учетным Комнаты Microsoft Teams компьютера. Используйте его для: 

-   [Настройка параметров питания и локальной учетной записи.](rooms-operations.md#configuring-group-policy-for-microsoft-teams-rooms)
-   В Windows обновления.
-   Включить переумноирование PowerShell. Вы можете настроить сценарий запуска для запуска сценария: Enable-PSRemoting -Force

С помощью PowerShell можно выполнять несколько действий по удаленному управлению, включая получение и настройку сведений о конфигурации. Чтобы можно было управлять удаленной службой PowerShell, необходимо включить ее в процессе развертывания или настроить с помощью групповой политики.  Дополнительные сведения об этих возможностях и их включаемой функции см. в сведениях об обслуживании [и операциях.](rooms-operations.md#remote-management-using-powershell)

Планирование конфигурации и развертывания охватывает следующие ключевые области:

-    Выполнение тестов

### <a name="resource-account-provisioning"></a>Подготовка учетной записи ресурса 

Для каждого Комнаты Microsoft Teams требуется специальная и уникальная учетная запись ресурса, которая должна быть включена как для Microsoft Teams, так и для Skype для бизнеса и Exchange. Эта учетная запись должна иметь почтовый ящик помещения, который будет Exchange. Обработка календаря должна быть настроена таким образом, что устройство может автоматически принимать входящие запросы на собрания. Дополнительные сведения о создании таких учетных записей см. в [Комнаты Microsoft Teams.](rooms-configure-accounts.md) 

**Pro совет.** Каждый Комнаты Microsoft Teams должен иметь допустимое и уникальное имя компьютера в сети. Во многих системах мониторинга и оповещений имя компьютера отображается в качестве ключевого идентификатора, поэтому важно разработать соглашение об именовке для развертывания Комнаты Microsoft Teams, которое позволит сотрудникам службы поддержки легко находить Комнаты Microsoft Teams, помеченные как требующие действий. В качестве примера можно использовать шаблон MTR-*Site* - *Room Name* (MTR-LON-CURIE).
