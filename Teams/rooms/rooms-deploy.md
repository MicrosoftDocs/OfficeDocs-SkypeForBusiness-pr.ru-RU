# <a name="deployment-overview"></a>Обзор развертывания

Развертывание комнат Microsoft Teams фактически разбивается на этапы:

- Подтверждение того, что расположения (помещения) соответствуют зависимостей развертывания
- Создание учетных записей Microsoft Teams или Skype для бизнеса и Exchange и назначение их устройствам консоли (см. "Настройка учетных записей для комнат [Microsoft Teams")](rooms-configure-accounts.md)
- Перенастройка планшетов Microsoft Surface для работы в качестве консолей комнат Microsoft Teams (см. настройку консоли комнат [Microsoft Teams](console.md) или руководство по развертыванию [помещений Microsoft Teams)](rooms-scale.md)
- (Необязательно) Настройка набора microsoft Operations Management Suite для систем (см. развертывание управления комнатами [Microsoft Teams с помощью Azure Monitor)](azure-monitor-deploy.md)
- Настройка консолей в помещениях для собраний и подключение необходимых периферийных устройств (см. документацию OEM для вашего набора устройств)

Для последней задачи можно использовать технические технологии, но ИТ-отдел вашей организации должен будет сделать остальные части процесса. 


## <a name="site-readiness"></a>Готовность сайта 

Во время доставки заказаных устройств в организацию работайте со своими сетевыми сетями, помещениями и группами av teams, чтобы обеспечить возможность применения зависимостей развертывания и готовность каждого сайта и помещения к работе с точки зрения питания, сети и отображения. Кроме того, убедитесь, что физические требования к установке выполнены. Что касается физической установки, посетите сайт поставщика и используйте возможности av team при установке, установке и установке экранов и запуске кабели.

## <a name="service-readiness"></a>Готовность к службе

### <a name="prepare-to-host-microsoft-teams-rooms-machine-and-service-accounts-optional"></a>Подготовка к помещению Microsoft Teams на компьютере и в учетной записи службы (необязательно)

Чтобы вы могли управлять комнатами и службами Microsoft Teams на компьютере и в учетных записях служб и управлять ими, подготовьте свои учетные записи локальной службы Active Directory или Azure Active Directory (Azure AD). 

Определите локальное развертывание Active Directory или группу Azure AD, в которые нужно добавить все учетные записи службы комнат Microsoft Teams (пользователей), а затем создайте отчеты об использовании с помощью командлета Get-CSUserSession PowerShell в развертывании комнат Microsoft Teams. Например, создайте группу с именем SkypeRoomSystemsv2-Service-Accounts. 


Определите один организационный блок в локальной иерархии Active Directory или Azure AD, чтобы вместить все учетные записи машин microsoft Teams Rooms (если они присоединились к домену) и одну организационную единицу для помещения всех учетных записей пользователей в комнатах Microsoft Teams. Если вы создадим подразделение для учетных записей компьютера Microsoft Teams Rooms, отключив наследование, чтобы применить только политики, которые должны применяться к комнатам Microsoft Teams, которые присоединились к домену. 

Создайте объект групповой политики, который назначен блоку организации, которая содержит учетные записи компьютеров в комнатах Microsoft Teams. Используется для: 

-   [Настройка параметров питания и локальной учетной записи.](rooms-operations.md#configuring-group-policy-for-microsoft-teams-rooms)
-   "Включить Обновление Windows".
-   Включить переемку PowerShell. Вы можете настроить запуск сценария: Enable-PSRemoting -Force

С помощью PowerShell можно выполнять несколько действий по удаленному управлению, в том числе получать и настраивать сведения о конфигурации. Перед тем как дистанционное  управление PowerShell станет возможным, его следует включить в процесс развертывания PowerShell или настроить с помощью групповой политики. Дополнительные сведения об этих возможностях и их включаемом см. в [техническом обслуживании и операциях.](rooms-operations.md#remote-management-using-powershell)

Планирование настройки и развертывания охватывает следующие основные области:

-   Подготовка учетной записи
-   Установка программного обеспечения для устройств

-   Конфигурация приложения комнат Microsoft Teams и периферийных устройств
-    Выполнение тестов
-   Управление активами

### <a name="account-provisioning"></a>Подготовка учетной записи 

Для каждого устройства комнат Microsoft Teams требуется специальная и уникальная учетная запись ресурса, которая должна быть включена как для Microsoft Teams, так и для Skype для бизнеса и Exchange. Эта учетная запись должна иметь почтовый ящик помещения, размещенного в Exchange, и быть включена в качестве комнаты для собрания в развертывании Teams или Skype для бизнеса. На стороне Exchange должна быть настроена обработка календаря, чтобы устройство автоматически принимать входящие запросы на собрания. Дополнительные сведения о создании таких учетных записей см. в сведениях о настройке учетных записей [для комнат Microsoft Teams.](rooms-configure-accounts.md) 

**Совет.** Делайте имена этих учетных записей понятными и понятными. Это имена, которые пользователи будут видеть при поиске и добавлении систем комнат Microsoft Teams к собраниям. В некоторых организациях используется конференц-зал (максимальный объем помещений)-RS, поэтому, например, у Климова ( конференц-зал в Лондоне, вместимый 12 человек), может отображаться имя  - LON-CURIE(12)-RS.

**Совет.** Все комнаты Microsoft Teams должны иметь допустимое и уникальное имя компьютера в сети. Многие системы мониторинга и оповещения отображают имя компьютера в качестве ключевого идентификатора, поэтому важно разработать соглашение об именовании для развертывания комнат Microsoft Teams, которое позволяет сотрудникам службы поддержки легко находить комнаты Microsoft Teams, помеченные как требующие действия. Примером может быть использование шаблона MTR -*Site* - *Room Name* (MTR-LON-CURIE). 

В рамках развертывания также необходимо продумыть стратегию управления локальными [](https://docs.microsoft.com/skypeforbusiness/plan-your-deployment/clients-and-devices/skype-room-systems-v2-0#local-accounts) учетными записями, созданными с помощью установщика приложений комнат Microsoft Teams.

Мы предоставляем инструкции по использованию монитора [Microsoft Azure для](https://docs.microsoft.com/skypeforbusiness/plan-your-deployment/clients-and-devices/azure-monitor) отслеживания развертывания комнат Microsoft Teams Rooms и отчета о доступности, ошибках оборудования и программного обеспечения, а также версии приложения комнат Microsoft Teams. Если вы решили использовать набор Microsoft Operations Management Suite, установите агент Operations Management Suite в рамках процесса установки программного обеспечения и настройте сведения о подстановке рабочей области для рабочей области. 

Дополнительным фактором является то, будут ли комнаты Microsoft Teams соединены с доменом. Сведения о преимуществах присоединения к домену можно найти в вопросах присоединения к домену [системы комнат Skype.](domain-joining-considerations.md)

### <a name="microsoft-teams-rooms-application-and-peripheral-device-configuration"></a>Конфигурация приложения комнат Microsoft Teams и периферийных устройств 

После физического развертывания каждой системы комнат Microsoft Teams и подключения поддерживаемых периферийных устройств необходимо настроить приложение "Комнаты Microsoft Teams" для назначения учетной записи ресурсов и пароля, созданных ранее, для того чтобы система комнат Microsoft Teams стала вводить учетные записи Microsoft Teams Rooms в Microsoft Teams или Skype для бизнеса и Exchange. Важно использовать сертифицированные USB-аудио- и видео периферийные устройства, связанные с другими устройствами в документе. Это может привести к непредсказуемым результатам. 

Каждую систему комнат Microsoft Teams можно настроить вручную. Кроме того, вы можете использовать XML-файл конфигурации для централизованного хранения данных на каждый XML-файл комнат Microsoft Teams для управления настройками приложения и использования запуска сценария GPO для повторного применения нужной конфигурации каждый раз, когда система Microsoft Teams Rooms сапог. 

Дополнительные сведения о том, как использовать XML-файл конфигурации, см. в файле конфигурации XML для удаленного управления настройками комнат [Microsoft Teams.](xml-config-file.md) 

Вы можете воспользоваться [удаленной командой PowerShell,](rooms-operations.md#remote-management-using-powershell) чтобы использовать конфигурацию комнат Microsoft Teams для отчетов.

### <a name="testing"></a> Выполнение тестов

После развертывания системы комнат Microsoft Teams необходимо протестировать ее. Убедитесь, что возможности, перечисленные в [справке по комнатам Microsoft Teams,](https://support.office.com/article/Skype-Room-Systems-version-2-help-e667f40e-5aab-40c1-bd68-611fe0002ba2) работают на развернутом устройстве. Настоятельно рекомендуем группе развертывания убедиться, что комнаты Microsoft Teams используют для входа в набор Microsoft Operations Management Suite (если он используется). Кроме того, важно сделать ряд тестовых звонков и собраний для проверки качества. Дополнительные сведения см. в этом [полезном контрольный список развертывания.](console.md#microsoft-teams-rooms-deployment-checklist)

Мы рекомендуем в рамках общего выпуска Teams или Skype для бизнеса настроить создание файлов для панели мониторинга качества звонков, отслеживать тенденции качества и принимать участие в процессе проверки качества. Дополнительные сведения см. в дополнительных сведениях об улучшении [и отслеживании качества вызова в Teams.](../monitor-call-quality-qos.md)
