# <a name="deployment-overview"></a>Обзор развертывания

Развертывание Комнаты Microsoft Teams по сути разбивает на этапы:

- Подтверждение соответствия расположений развертывания (пробелов) зависимостям развертывания
- Создание Microsoft Teams или Skype для бизнеса и Exchange и их назначение Комнаты Teams (см. раздел "Настройка учетных записей [для](rooms-configure-accounts.md) Комнаты Microsoft Teams)
- (Необязательно) Настройка Azure Monitor для систем (см. раздел ["Развертывание Комнаты Microsoft Teams с помощью Azure Monitor"](azure-monitor-deploy.md)
- Настройка Комнаты Teams в пространствах собраний и подключение необходимых периферийных устройств (см. документацию изготовителя оборудования для набора устройств)

## <a name="site-readiness"></a>Готовность сайта 

Пока упорядоченные устройства доставляются в вашу организацию, работайте с сетями, средствами и группами av, чтобы обеспечить выполнение зависимостей развертывания и готовность каждого сайта и пространства с точки зрения питания, сети и отображения. Кроме того, убедитесь, что выполнены требования к физической установке. Чтобы получить сведения о физической установке, обратитесь к поставщику и используйте опыт команды av при установке и подключении экранов и запуске кабелей.

Дополнительные сведения об этих зависимостях см. в следующих руководствах по планированию:

-   [Проверка доступности сети](rooms-prep.md#check-network-availability)
-   [Сертификаты](rooms-prep.md#certificates)
-   [Прокси-сервер](rooms-prep.md#proxy)

**Pro совет**. Если для предоставления доступа к Teams необходимо использовать прокси-серверы, сначала [ознакомьтесь с этой статьей](../proxy-servers-for-skype-for-business-online.md). Когда речь идет о Microsoft Teams передачи мультимедиа через прокси-серверы в режиме реального времени, мы рекомендуем полностью обойти прокси-серверы. Microsoft Teams трафик уже зашифрован, поэтому прокси-серверы не делают его более безопасным и добавляют задержку в трафик в режиме реального времени. В рамках более широкого развертывания рекомендуется следовать указаниям в разделе "Подготовка сети [к](../prepare-network.md) Teams планированию пропускной способности и оценке пригодности сети для трафика в режиме реального времени".

## <a name="service-readiness"></a>Готовность к службе

Чтобы подготовиться к развертыванию Комнаты Microsoft Teams, выполните следующие ключевые, центральные задачи:

-   Определите Комнаты Microsoft Teams ресурсов.
-   При присоединении Teams Room Azure Active Directory подготовьте группу Azure AD с динамическим членством для хранения всех учетных записей Комнаты Teams ресурсов. Это упростит управление в будущем, например применение политик условного доступа. Чтобы с легкостью использовать динамические группы Azure AD, определите соглашение об именовании, которое будет уникальным образом идентифицировать учетные записи Комнаты Teams ресурсов.
-   При присоединении Teams Room к Active Directory подготовьте подразделение и группу Active Directory для хранения учетных записей компьютера и ресурсов Комнаты Microsoft Teams и (при необходимости) подготовьте объекты групповая политика (GPO) для включения удаленного взаимодействия PowerShell.

### <a name="prepare-to-host-microsoft-teams-rooms-and-resource-accounts-optional"></a>Подготовка к размещению Комнаты Microsoft Teams и учетных записей ресурсов (необязательно)

Чтобы вы можете управлять учетными записями Комнаты Microsoft Teams ресурсов и управлять ими, подготовьте локальная служба Active Directory или Azure Active Directory (Azure AD). 

Определите локальная служба Active Directory или Azure Active Directory для добавления всех учетных записей Комнаты Microsoft Teams ресурсов. При использовании Azure Active Directory рекомендуется использовать динамическую группу для автоматического добавления и удаления учетных записей ресурсов из группы.

Определите одно подразделение в иерархии локальная служба Active Directory, в котором будут храниться все учетные записи Комнаты Microsoft Teams компьютера (если они присоединены к домену), и одно подразделение для хранения всех Комнаты Microsoft Teams учетных записей пользователей. Отключите групповая политика, чтобы обеспечить применение только политик, которые вы хотите применить к присоединенному к домену Комнаты Microsoft Teams.

Создайте групповая политика, назначенный подразделению, который содержит Комнаты Microsoft Teams компьютеров. Используйте его для: 

-   [Задайте параметры питания и локальной учетной записи](rooms-operations.md#configuring-group-policy-for-microsoft-teams-rooms).
-   Включите клиентский компонент Центра обновления Windows.
-   Включите удаленное взаимодействие PowerShell. Вы можете настроить сценарий запуска для запуска скрипта: Enable-PSRemoting -Force

PowerShell можно использовать для выполнения нескольких действий удаленного управления, включая получение и настройку сведений о конфигурации. Удаленное взаимодействие PowerShell должно быть включено до того, как можно будет выполнять удаленное управление PowerShell. Его следует рассматривать как часть процессов развертывания или настроить с помощью групповая политика. Дополнительные сведения об этих возможностях и их включении см. в разделе ["Обслуживание и операции"](rooms-operations.md#remote-management-using-powershell). 


## <a name="configuration-and-deployment"></a>Настройка и развертывание 

Планирование конфигурации и развертывания охватывает следующие ключевые области:

-   Подготовка учетной записи ресурса
-   Установка программного обеспечения устройства
-   Развертывание устройства
-   Комнаты Microsoft Teams конфигурации приложения и периферийного устройства
-    Выполнение тестов
-   Управление активами

### <a name="resource-account-provisioning"></a>Подготовка учетной записи ресурса 

Для Комнаты Microsoft Teams устройства требуется выделенная уникальная учетная запись ресурса, которая должна быть включена как для Microsoft Teams, так и для Skype для бизнеса и Exchange. Эта учетная запись должна содержать почтовый ящик помещения, размещенный в Exchange. Необходимо настроить обработку календаря, чтобы устройство автоматически принимать входящие приглашения на собрания. Дополнительные сведения о создании этих учетных записей см. в разделе "Настройка учетных записей [для](rooms-configure-accounts.md) Комнаты Microsoft Teams". 

**Pro совет**. Каждый Комнаты Microsoft Teams должен иметь допустимое и уникальное имя компьютера в сети. Во многих системах мониторинга и оповещений имя компьютера отображается как идентификатор ключа, поэтому важно разработать соглашение об именовании для развертываний Комнаты Microsoft Teams, которое позволяет сотрудникам службы поддержки легко находить Комнаты Microsoft Teams, помеченные как требующие действия. Примером может быть использование шаблона *MTR-SiteRoom*- *Name* (MTR-LON-CURIE).

### <a name="testing"></a> Выполнение тестов

После Комнаты Teams развертывания необходимо протестировать его. Убедитесь, что возможности, перечисленные в [Комнаты Microsoft Teams справке](https://support.microsoft.com/en-us/office/microsoft-teams-rooms-help-e667f40e-5aab-40c1-bd68-611fe0002ba2?ui=en-us&rs=en-us&ad=us), работают на развернутом устройстве. Настоятельно рекомендуется убедиться, что команда развертывания Комнаты Microsoft Teams в Teams администрирования. Кроме того, важно выполнить ряд тестовых звонков и собраний для проверки качества. Дополнительные сведения см. в этом [полезном контрольный список развертывания](console.md#microsoft-teams-rooms-deployment-checklist).

Мы рекомендуем в рамках общего развертывания Teams или Skype для бизнеса настроить сборку файлов для панели мониторинга качества звонков (CQD), отслеживать тенденции качества и участвовать в процессе проверки качества взаимодействия. Дополнительные сведения см. в разделе ["Улучшение и мониторинг качества звонков для Teams](../monitor-call-quality-qos.md)".
