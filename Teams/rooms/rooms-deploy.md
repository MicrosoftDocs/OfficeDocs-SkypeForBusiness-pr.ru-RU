# <a name="deployment-overview"></a>Обзор развертывания

Развертывание Комнаты Microsoft Teams фактически разбивается на этапы:

- Подтверждение того, что расположения (помещения) соответствуют зависимостей развертывания
- Создание Microsoft Teams или Skype для бизнеса и Exchange и назначение их на устройствах консоли (см. настройка учетных записей для [Комнаты Microsoft Teams)](rooms-configure-accounts.md)
- Повторное использование планшетов Microsoft Surface в качестве консолей Комнаты Microsoft Teams (см. настройки консоли Комнаты Microsoft Teams или Развертывание Комнаты Microsoft Teams [развертывания)](rooms-scale.md) [](console.md)
- (Необязательно) Настройка набора microsoft Operations Management Suite для систем (см. Комнаты Microsoft Teams развертывания с [помощью Azure Monitor.](azure-monitor-deploy.md)
- Настройка консолей в комнатах собраний и подключение необходимых периферийных устройств (см. документацию OEM для набора устройств)

Для последней задачи можно использовать технические технологии, но ИТ-отделу вашей организации потребуется сделать другие части процесса. 


## <a name="site-readiness"></a>Готовность сайта 

Во время доставки заказаных устройств в организацию работайте с сетевыми сетями, помещениями и группами av teams, чтобы убедиться в том, что зависимости развертывания выполнены, и каждый сайт и помещение готовы к работе с точки зрения питания, сетевого обслуживания и отображения. Кроме того, убедитесь, что выполнены физические требования к установке. Для физической установки посетите сайт поставщика и используйте возможности вашей av team при установке и установке экранов и под управлением кабелей.

Дополнительные информацию об этих зависимостей см. в указанных ниже ссылках на рекомендации по планированию.

-   [Проверка доступности сети](rooms-prep.md#check-network-availability)
-   [Сертификаты](rooms-prep.md#certificates)

Skype для бизнеса трафик уже зашифрован, поэтому прокси-серверы не делают его более безопасным. ### <a name="prepare-to-host-microsoft-teams-rooms-machine-and-service-accounts-optional"></a>Подготовка к Комнаты Microsoft Teams компьютеров и служб (необязательно)

Чтобы вы могли управлять учетными записями компьютера и службы Комнаты Microsoft Teams, подготовьте к этому свою службу active Directory или службу Azure Active Directory (Azure AD). 

Определите локальное развертывание Active Directory или Azure AD, чтобы добавить в нее все учетные записи службы Комнаты Microsoft Teams (пользователей), а затем создайте отчеты об использовании с помощью Get-CSUserSession PowerShell в развертывании Комнаты Microsoft Teams. Например, создайте группу с именем SkypeRoomSystemsv2-Service-Accounts. 


Определите одну организационную единицу в локальной иерархии Active Directory или Azure AD для удержания всех учетных записей компьютера Комнаты Microsoft Teams (если они соединены с доменом) и одного подразделения для удержания всех учетных записей Комнаты Microsoft Teams пользователей. Если вы создали подразделение для учетных записей Комнаты Microsoft Teams компьютера, отключать наследование, чтобы гарантировать применение только политик, которые должны применяться к доменным Комнаты Microsoft Teams. 

Создайте объект групповой политики, который назначен блоку организации, содержа Комнаты Microsoft Teams учетным Комнаты Microsoft Teams компьютера. Используйте его для: 

-   [Настройка параметров питания и локальной учетной записи.](rooms-operations.md#configuring-group-policy-for-microsoft-teams-rooms)
-   В Windows обновления.
-   Включить переумноирование PowerShell. Вы можете настроить сценарий запуска для запуска сценария: Enable-PSRemoting -Force

С помощью PowerShell можно выполнять несколько действий удаленного управления, включая получение и настройку сведений о конфигурации. Перед началом удаленного управления  PowerShell должно быть включено перенаправление PowerShell. Его следует рассматривать как часть процессов развертывания или настроить с помощью групповой политики. Дополнительные сведения об этих возможностях и их включаемой функции см. в сведениях об обслуживании [и операциях.](rooms-operations.md#remote-management-using-powershell)

### <a name="account-provisioning"></a>Подготовка учетной записи 

Для каждого Комнаты Microsoft Teams требуется специальная и уникальная учетная запись ресурса, которая должна быть включена как для Microsoft Teams, так и для Skype для бизнеса и Exchange. Эта учетная запись должна иметь почтовый ящик комнаты Exchange и быть включена в качестве комнаты для Teams или Skype для бизнеса развертывания. На стороне Exchange настроить обработку календаря, чтобы устройство автоматически принимать входящие запросы на собрания. Дополнительные сведения о создании таких учетных записей см. в [Комнаты Microsoft Teams.](rooms-configure-accounts.md) 

**Pro совет.** Сделайте отображательные имена этих учетных записей понятными и понятными. Это имена, которые пользователи будут видеть при поиске и добавлении систем Комнаты Microsoft Teams к собраниям. В некоторых организациях используется соглашение Site Room Name (Max Room Room Room )-RS, поэтому, например, у Curie , 12-х конференц-зала в Москве, может быть отображаемая фамилия - LON-CURIE(12)-RS.

**Pro совет.** Каждый Комнаты Microsoft Teams должен иметь допустимое и уникальное имя компьютера в сети. Во многих системах мониторинга и оповещений имя компьютера отображается в качестве ключевого идентификатора, поэтому важно разработать соглашение об именовке для развертывания Комнаты Microsoft Teams, которое позволит сотрудникам службы поддержки легко находить Комнаты Microsoft Teams, помеченные как требующие действий. В рамках развертывания также необходимо продумыть стратегию управления локальными [](/skypeforbusiness/plan-your-deployment/clients-and-devices/skype-room-systems-v2-0#local-accounts) учетной записью, созданной с помощью установщика Комнаты Microsoft Teams приложений.

Если вы решили использовать набор microsoft Operations Management Suite, установите агент Operations Management Suite в рамках процесса установки программного обеспечения и настройте сведения о подстановке рабочей области для рабочей области.

### <a name="microsoft-teams-rooms-application-and-peripheral-device-configuration"></a>Комнаты Microsoft Teams приложения и периферийного устройства 

После физического развертывания каждой системы Комнаты Microsoft Teams и подключения поддерживаемых периферийных устройств необходимо настроить приложение Комнаты Microsoft Teams, чтобы назначить учетную запись ресурса Комнаты Microsoft Teams и пароль, созданные ранее, чтобы система Комнаты Microsoft Teams вошел в Microsoft Teams или Skype для бизнеса и Exchange. Это важно для использования сертифицированных usb-аудио- и видео периферийных устройств, связанных в другом месте документа. Это может привести к непредсказуемым результатам. 

Вы можете вручную настроить каждую Комнаты Microsoft Teams системы. Кроме того, вы можете использовать централизованно хранимый XML-файл конфигурации per-Комнаты Microsoft Teams для управления настройками приложений и повторного применения нужной конфигурации с помощью сценария GPO Комнаты Microsoft Teams системной конфигурации. 

Дополнительные сведения об использовании XML-файла конфигурации см. в Комнаты Microsoft Teams консоли управления файлом [конфигурации XML.](xml-config-file.md) 

Вы можете использовать [удаленную powerShell](rooms-operations.md#remote-management-using-powershell) для настройки Комнаты Microsoft Teams отчетов.
