# <a name="microsoft-teams-rooms-security"></a>Безопасность комнат Microsoft Teams

Корпорация Майкрософт сотрудничает с нашими партнерами в отношении безопасного решения, которое не требует дополнительных действий для обеспечения безопасности комнат Microsoft Teams. В этой статье рассмотрены многие функции безопасности, которые есть в комнатах Teams.

> [!NOTE]
> Комнаты Microsoft Teams не должны рассматриваться как обычная рабочий станция для конечных пользователей. При этом различные случаи использования значительно отличаются, но и профили безопасности по умолчанию.

Ограниченные данные пользователя хранятся в комнатах Teams. Данные пользователей могут храниться в файлах журнала только для устранения неполадок и поддержки. Участники собрания ни при каких моментах не смогут скопировать файлы на жесткий диск или войти как сами. Никакие данные пользователя не передаются или доступны с помощью устройства комнат Microsoft Teams.

Хотя конечным пользователям нельзя помещает файлы на жесткий диск комнат Teams, Защитник Майкрософт по-прежнему включен. Производительность комнат Teams протестирована с помощью Microsoft Defender. Отключение этого программного обеспечения или добавление программного обеспечения безопасности конечных точек может привести к непредсказуемым результатам и потенциальному ухудшению системы.

## <a name="hardware-security"></a>Безопасность оборудования

В среде комнат Teams есть централизованный вычислительный модуль, который запускает выпуск Windows 10 IoT Enterprise. Каждый сертифицированный вычислительный модуль должен иметь решение для безопасного подключения, разъем для блокировки системы безопасности (например, блокировка Кузнецова) и меры безопасности доступа к портам I/O, предотвращающие подключение неавторизованных устройств. Вы также можете отключить определенные порты с помощью единой конфигурации extensible Firmware Interface (UEFI).

Каждый сертифицированный вычислительный модуль должен быть включен по умолчанию с технологией, которая соответствует доверенного платформенного модуля (TPM) 2.0. Для шифрования данных для учетной записи ресурсов комнат Teams используется TPM.

По умолчанию безопасная загрузка включена. Безопасная загрузка — это стандарт безопасности, разработанный участниками отрасли компьютеров, который обеспечивает безопасность загрузки устройств с использованием только программного обеспечения, доверенного изготовителем исходного оборудования (OEM). Когда компьютер запускается, программное обеспечение проверяет подпись каждого фрагмента программного обеспечения загрузки, включая драйверы UEFI (также известные как OPTION ROMs), приложения EFI и операционную систему. Если подписи действительны, валенки компьютеров и программы управления операционной системой. Дополнительные сведения см. в [подмноге "Безопасная загрузка".](https://docs.microsoft.com/windows-hardware/design/device-experiences/oem-secure-boot)

Доступ к настройкам UEFI возможен только с помощью физической клавиатуры и мыши. Это предотвращает доступ к UEFI через консоль с поддержкой сенсорного управления Teams Rooms, а также любые другие сенсорные дисплеи, вложенные в комнаты Teams.

Защита прямого доступа к памяти ядра (DMA) — это параметр Windows 10, который включен в комнатах Teams. С помощью этой функции ОС и системное ПО защищают систему от вредоносных и нежелательных атак DMA на всех устройствах с возможностью DMA:

- Во время загрузки.

- От вредоносных DMA-устройств, подключенных к легко доступу к внутренним и внешним портам с подключением к DMA, таким как разъемы PCIE m.2 и Thunderbolt 3, во время работы ОС.

В комнатах Teams также обеспечивается целостность кода, защищенного гипервидителем (HVCI). В HVCI есть одна из функций Credential Guard. Credential Guard имеет следующие преимущества:

- **Безопасность оборудования** В NTLM, Kerberos и диспетчере учетных данных для защиты учетных данных есть ряд функций безопасности платформы, включая безопасную загрузку и виртуализацию.

- **Безопасность на основе виртуализации** Полученные учетные данные Windows NTLM и Kerberos и другие секреты запускаются в защищенной среде, изолированной от операционной системы.

- **Более постоянная защита от сложных сохраняемой угрозы** Когда учетные данные диспетчера учетных данных, NTLM и Kerberos защищаются с помощью системы безопасности на основе виртуализации, методы и средства кражи учетных данных, используемые во многих целевых атаках, блокируются. Вредоносные программы, работающие в операционной системе с административными привилегиями, не могут извлечь секрет, защищенный безопасностью на основе виртуализации.

## <a name="software-security"></a>Безопасность программного обеспечения

После загрузки Microsoft Windows комнаты Teams автоматически включались в локализованную учетную запись пользователя Windows с именем Skype. У учетной записи Skype нет пароля. Чтобы обеспечить безопасность сеанса учетной записи Skype, сделайте следующее:

> [!IMPORTANT]
> Не изменяйте пароль или учетную запись локального пользователя Skype. Это может предотвратить автоматический вход в комнаты Teams.

Приложение "Комнаты Microsoft Teams" работает с функцией "Назначенное доступ", доступной в Windows 10 1903 и более поздних версиях. Assigned Access is a feature in Windows 10 that limits the application entry points exposed to the user. Это позволяет использовать одно приложении в режиме киоска. С помощью shell Launcher комнаты Teams настраиваются в качестве киоска, которое запускает настольное приложение Windows в качестве пользовательского интерфейса. Приложение "Комнаты Microsoft Teams" заменяет стандартную оболочку (explorer.exe), которая обычно выполняется при входе пользователя. Другими словами, традиционная оболочка проводника не запускаться. Это значительно снижает уязвимость комнат Microsoft Teams в Windows. Дополнительные сведения см. [в настройках киосков и цифровых знаков в классических версиях Windows.](https://docs.microsoft.com/windows/configuration/kiosk-methods)

Если вы решили запустить проверку безопасности или производительности Центра безопасности Интернета (CIS) в комнатах Teams, сканирование можно выполнить только в контексте учетной записи локального администратора, так как учетная запись пользователя Skype не поддерживает запущенные приложения, кроме приложения "Комнаты Teams". Многие функции безопасности, примененные к контексту пользователя Skype, не применяются к другим локальным пользователям, поэтому такие проверки безопасности не распространяются на полный блок блокировки, применяемый к учетной записи Skype. Поэтому не рекомендуется запускать локализованную проверку в комнатах Teams. Однако при желании вы можете выполнить внешние тесты на внешние наружу. В связи с этим мы рекомендуем запускать внешние тесты на наружу на устройствах комнат Teams, а не на локальные проверки.

Кроме того, политики блокировки применяются для ограничения использования функций, не сфере администрирования. Фильтр клавиатуры позволяет перехватывать и блокировать потенциально комбинации клавиатуры, не соответствующие политикам Access. Только пользователи с правами администратора локального домена или домена имеют право на вход в Windows для управления комнатами Teams. Эти и другие политики, применяемые к Windows на устройствах комнат Microsoft Teams, постоянно оцениваются и тестются в течение жизненного цикла продукта.

## <a name="account-security"></a>Безопасность учетной записи

На устройствах комнат Teams есть учетная запись администратора с паролем по умолчанию. Мы настоятельно рекомендуем вам как можно скорее сменить пароль по умолчанию после завершения настройки.

Учетная запись администратора не требуется для правильной работы устройств с комнатами Teams и может быть переименована или даже удалена. Однако перед удалением учетной записи администратора необходимо настроить альтернативную учетную запись локального администратора, прежде чем удалять учетную запись, которая вместе с устройствами teams Rooms. Дополнительные сведения о том, как изменить пароль локальной учетной записи Windows с помощью встроенных средств Windows или PowerShell, см. в следующих местах:

- [Изменение или сброс пароля Windows](https://support.microsoft.com/windows/change-or-reset-your-windows-password-8271d17c-9f9e-443f-835a-8318c8f68b9c)
- [Set-LocalUser](https://docs.microsoft.com/powershell/module/microsoft.powershell.localaccounts/set-localuser?view=powershell-5.1#example-2--change-the-password-on-an-account)

Вы также можете импортировать учетные записи домена в локализованную группу администраторов Windows. Это можно сделать для учетных записей Azure AD с помощью Intune. Дополнительные сведения см. в [политике CSP – RestrictedGroups.](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-restrictedgroups)

> [!CAUTION]
> Если вы удалите или отключите учетную запись администратора, прежде чем предоставлять локальные разрешения администратора другой локальной или доменной учетной записи, вы можете потерять возможность администрирования устройства комнат Teams. В этом случае вам потребуется восстановить исходные параметры устройства и снова выполнить настройку.
>
> Не предоставлять учетной записи пользователя Skype разрешения локального администратора.

Конструктор конфигураций Windows можно использовать для создания пакетов подготовка Windows 10. Вы можете не только изменить пароль локального администратора, но и изменить имя компьютера и зарегистрироваться в Azure Active Directory. Дополнительные сведения о создании пакета подготовка Конструктора конфигурации Windows см. в подстановок [для Windows 10.](https://docs.microsoft.com/windows/configuration/provisioning-packages/provisioning-packages)

Для этого необходимо создать учетную запись ресурса для каждого устройства комнат Teams. Использовать двухфакторную или многофакторную проверку подлинности с этой учетной записью нельзя. Требование второго фактора помешает учетной записи автоматически войти в приложение "Комнаты Teams" после перезагрузки. Однако вы можете включить современную проверку подлинности для дополнительной защиты этой учетной записи. Кроме того, для учетной записи ресурса можно развернуть политики условного доступа на основе расположения, чтобы предотвратить вход в учетную запись из неподтверченного расположения. Дополнительные сведения см. в [условии "Использование условия расположения" в политике условного доступа](https://docs.microsoft.com/azure/active-directory/conditional-access/location-condition)

По возможности рекомендуется создать учетную запись ресурса в Azure AD. Хотя синхронизированная учетная запись может работать с комнатами Teams в гибридных развертываниях, этим синхронизированным учетным записям часто сложно войди в комнаты Teams, и их может быть сложно устранить. Если для проверки подлинности учетных данных для учетной записи ресурса используется сторонная служба федерации, убедитесь, что в ответе на IDP сторонних данных задав `wsTrustResponse` `urn:oasis:names:tc:SAML:1.0:assertion` атрибут.

## <a name="network-security"></a>Безопасность сети

Как правило, к помещениям Teams предъявляются те же требования к сети, что и к любому клиенту Microsoft Teams. Доступ через брандмауэры и другие устройства безопасности для комнат Teams такой же, как и для других клиентов Microsoft Teams. Категории, которые указаны как "необходимые" для Teams, должны быть открыты в брандмауэре. Комнаты Teams также должны иметь доступ к Windows Update, Microsoft Store и Microsoft Intune (если вы управляете устройствами с помощью Microsoft Intune). Полный список URL-адресов и URL-адресов, необходимых для комнат Microsoft Teams, см. в:

- **URL-адреса** и диапазоны IP-адресов Microsoft Teams [для Office 365](https://docs.microsoft.com/microsoft-365/enterprise/urls-and-ip-address-ranges?view=o365-worldwide#skype-for-business-online-and-microsoft-teams)
- Настройка [WSUS в](https://docs.microsoft.com/windows-server/administration/windows-server-update-services/deploy/2-configure-wsus#211-connection-from-the-wsus-server-to-the-internet) **Обновлении Windows**
- **Необходимые** [условия для Microsoft Store для бизнеса и образования](https://docs.microsoft.com/microsoft-store/prerequisites-microsoft-store-for-business#proxy-configuration) в Microsoft Store
- **Microsoft Intune** [Network Enpoints для Microsoft Intune](https://docs.microsoft.com/mem/intune/fundamentals/intune-endpoints)

Если вы используете компонент служб, управляемых комнатами Microsoft Teams, в Microsoft Teams Rooms Premium, необходимо также убедиться, что комнаты Teams имеют доступ к следующим URL-адресам:

- global.azure-devices-provisioning.net
- gj3ftstorage.blob.core.windows.net
- gj3ft-hub.azure-devices.net
- iothubsgagwt5wgvwg6.azure-devices.net
- blobssgagwt5wgvwg6.blob.core.windows.net
- prod-48.westus.logic.azure.com
- prod-08.westus.logic.azure.com
- prod-62.westus.logic.azure.com
- prod-65.westus.logic.azure.com
- prod-05.westus.logic.azure.com

Комнаты Teams настроены для автоматического обновления с учетом последних обновлений Windows, включая обновления для системы безопасности. В комнатах Teams все ожидающих обновления устанавливаются каждый день, начиная с 14:00, с использованием предварительно заранее задайте локализованную политику. Нет необходимости использовать дополнительные средства для развертывания и применения Обновлений Windows. Использование дополнительных средств для развертывания и применения обновлений может задержать установку исправлений Windows, что приведет к менее безопасному развертыванию. Приложение "Комнаты Teams" развертывается с помощью Microsoft Store. Если на ваших устройствах установлена лицензия на Microsoft Teams Rooms Standard, все новые версии приложения устанавливаются автоматически во время ночного исправления. Если ваши устройства имеют лицензию на Microsoft Teams Rooms Premium и зарегистрированы в управляемой службе Майкрософт, новые версии приложения "Комнаты Teams" устанавливаются в установленном плане.

Устройства комнат Teams работают с большинством протоколов безопасности на основе 802.1X или других сетевых протоколов безопасности. Однако мы не можем протестировать комнаты Teams на все возможные конфигурации безопасности сети. Поэтому, если возникают проблемы с производительностью, которые могут быть отследить до проблем с производительностью сети, может потребоваться отключить эти протоколы, если они настроены в вашей организации.

Для оптимальной производительности мультимедиа в режиме реального времени настоятельно рекомендуем настроить трафик мультимедиа Teams в обход прокси-серверов и других сетевых устройств безопасности. Мультимедиа в режиме реального времени очень чувствителен к задержкам, а прокси-серверы и устройства безопасности сети могут значительно повысить качество видео и звука пользователей. Кроме того, поскольку мультимедиа Teams уже зашифрованы, нет неохобходимой выгоды в передаче трафика через прокси-сервер. Дополнительные сведения см. в подсети [(в облаке)](https://docs.microsoft.com/microsoft-365/solutions/networking-design-principles?view=o365-worldwide) — по рекомендации одного архитектора, в котором обсуждается рекомендации по сети для повышения производительности мультимедиа с помощью Microsoft Teams и комнат Microsoft Teams.

> [!IMPORTANT]
> Комнаты Teams не поддерживают прокси-серверы, аутентификацию.

Устройства комнат Teams не нужно подключаться к внутренней локальной сети. Комнаты Teams можно разместить в безопасном сегменте сети с прямым доступом к Интернету. При компрометации внутренней локальной сети возможности вектора атак на комнаты Teams будут уменьшены.

Мы настоятельно рекомендуем подключить устройства комнат Teams к проводной сети. Использовать беспроводные сети на устройствах комнат Teams не рекомендуется и не рекомендуется. Некоторые функции подключения, например Wi-Fi Sense, по умолчанию отключены.

Присоединитесь к близости и другие функции комнат Teams, которые зависят от Bluetooth. Однако внедрение Bluetooth на устройствах комнат Teams не разрешает подключение внешнего устройства к устройству комнат Teams. Bluetooth технология, используемая на устройствах комнат Teams, в настоящее время ограничена рекламными маяками и запросами на подключение. Тип `ADV_NONCONN_INT` блока данных протокола (PDU) используется в рекламного маяка. Этот тип PDU предназначен для непривимемых к подключению устройств, которые рекламизют информацию на устройстве прослушивания. Такое сопряжение Bluetooth не существует. Дополнительные сведения о протоколах Bluetooth можно найти на веб-сайте Bluetooth [SIG.](https://www.bluetooth.com/blog/bluetooth-low-energy-it-starts-with-advertising/)
