# <a name="configure-a-microsoft-teams-rooms-console"></a>Настройка консоли комнат Microsoft Teams

В этой статье описано, как настроить консоль комнат Microsoft Teams и ее периферийные устройства.
  
Эти действия следует выполнять только в том случае, если необходимые учетные записи Microsoft Teams, Skype для бизнеса и Exchange и Exchange уже созданы и проверены, как описано в описании развертывания [комнат Microsoft Teams.](rooms-deploy.md) Вам потребуется оборудование и программное обеспечение, описанные в требованиях [к комнатам Microsoft Teams.](requirements.md) В этой статье содержатся следующие разделы.
  
- [Подготовка установного носителю](console.md#Prep_Media)
- [Установка частного сертификата ЦС на консоли](console.md#Certs)
- [Установка Windows 10 и приложения консоли для комнат Microsoft Teams](console.md#Reimage)
- [Начальная настройка консоли](console.md#Initial)
- [Контрольный список развертывания комнат Microsoft Teams](console.md#Checklist)

> [!NOTE]
> Комнаты Microsoft Teams будут работать только в правильно настроенной среде Microsoft Teams или Skype для бизнеса, где учетные записи устройств настроены правильно, как описано в описании в развертывании [комнат Microsoft Teams.](rooms-deploy.md)
  
## <a name="prepare-the-installation-media"></a>Подготовка установного носителю
<a name="Prep_Media"> </a>

Для установки консоли Microsoft Teams Rooms требуется USB-накопитель емкостью не менее 32 ГБ. На устройстве не должно быть других файлов; все существующие файлы в USB-накопителе будут потеряны.
  
> [!NOTE]
> Непредвиденное поведение при установке мультимедиа в комнатах Microsoft Teams согласно этим инструкциям.

> [!NOTE]
> Ниже приведен процесс создания установок мультимедиа для создания образа новых устройств комнат Microsoft Teams. Существующие устройства по умолчанию автоматически обновляются из Windows Update и Магазина Windows.

> [!IMPORTANT]
> Компьютер с Windows 10, используемый для создания установок Microsoft Teams Rooms, должен быть в той же или более поздней версии Windows, что и целевой установный носитчик.
  
1. Скачайте [сценарийCreateSrsMedia.ps1.](https://go.microsoft.com/fwlink/?linkid=867842)
2. Запустите сценарий CreateSrsMedia.ps1 из командной строки с повышенными привилегиями на компьютере с ОС Windows 10.
3. Следуйте инструкциям сценария, чтобы создать USB-диск microsoft Teams Rooms.


> [!TIP]
> При каждом CreateSrsMedia.ps1 сценария на экране выводит имя файла журнала или запись разговоров для сеанса. При наличии проблем с запуском сценария убедитесь, что при запросе поддержки доступна копия этого расшифровки. 

Сценарий CreateSrsMedia.ps1 автоматизирует следующие задачи:

1. Скачайте последнюю версию установщика MSI для комнат Microsoft Teams.
2. Определите сборку Windows, которую должен предоставить пользователь. Последние версии могут не тестироваться и не поддерживаться для использования с устройствами комнат Microsoft Teams.
3. Скачайте необходимые вспомогательные компоненты.
4. Соберите необходимые компоненты на установимом носите.

Необходима определенная версия Windows 10, которая доступна только клиентам корпоративного лицензирования.  Вы можете получить копию из [Volume Licensing Service Center.](https://www.microsoft.com/Licensing/servicecenter/)

По завершению удалите USB-диск с компьютера и переперейти к установке [Windows 10 и приложения](console.md#Reimage)консоли microsoft Teams Rooms.

    
## <a name="install-windows-10-and-the-microsoft-teams-rooms-console-app"></a>Установка Windows 10 и приложения консоли для комнат Microsoft Teams
<a name="Reimage"> </a>

Теперь нужно применить созданный установ установный носител. Целевое устройство будет работать как устройство, а по умолчанию будет настроено только на запуск консоли microsoft Teams Rooms.

1. Если целевое устройство будет установлено на док-станции (например, Surface Pro), отключите его от док-станции.

2. Убедитесь, что целевое устройство не подключено к сети.

3. Убедитесь, что целевое устройство подключено к электроу питания.

4. Подключите usb-накопитель к целевому устройству.

5. Загрузка на установочный диск USB. Следуйте инструкциям производителя. Если целевым устройством является Surface Pro, для загрузки USB-диска используйте следующие действия:

    а) Нажмите и удерживайте кнопку громкости (-).

    б) Нажмите и отпустите кнопку питания.

    в. После загрузки установки Windows отпустите кнопку уменьшения громкости (–).

8. Система закроется после завершения установки.
    
После того как система выключится, вы можете удалить usb-накопитель установки. На этом этапе вы можете разместить целевое устройство на своей док-станции (при использовании док-станции), прикрепить периферийные устройства, необходимые для вашей комнаты собрания, и подключиться к сети. Следуйте инструкциям производителя.

> [!NOTE]
> Обновления программного обеспечения для комнат Microsoft Teams автоматически загружаются из Microsoft Store для бизнеса. См. [статью "Предварительные](https://docs.microsoft.com/microsoft-store/prerequisites-microsoft-store-for-business) условия для Microsoft Store для бизнеса и образования", чтобы убедиться в том, что консоль комнаты может получить доступ к магазину и для самостоятельного обновления.  

### <a name="selecting-a-language"></a>Выбор языка 

В Обновлении для создателя необходимо использовать сценарий ApplyCurrentRegionAndLanguage.ps1 в сценариях, когда неявный выбор языка не дает пользователю нужного языка приложения (например, он хочет, чтобы приложение консоли было доступно на французском языке, но будет доступно на английском).
  
> [!NOTE]
> Эти инструкции работают только для консолей, созданных с помощью Обновления Windows. Устаревшие и рыночные системы, которые не были настроены на использование мультимедиа в новой системе предварительной установки, не смогут использовать эти инструкции, но не должны страдать от первоначальной проблемы, которая требует вмешательства вручную (Anniversary Edition позволяет выбрать язык приложения явным образом в рамках настройки).
  
### <a name="to-apply-your-desired-language"></a>Применение необходимого языка

1. Переключитесь в режим администрирования.
    
2. Откройте меню "Пуск".
    
3. Щелкните значок шестеренки, чтобы запустить приложение **Параметры**.
    
4. Выберите **язык &amp; времени.**
    
5. Выберите **язык &amp; "Регион".**
    
6. Выберите **Добавить язык**.
    
7. Выберите язык, который требуется добавить.
    
8. Выберите язык, который вы только что добавили в список "Языки".
    
9. Выберите **По умолчанию**.
    
10. Удаление языков
    
    а. Выберите язык, который требуется удалить.
    
    б. Выберите **Удалить**.
    
11. Запустите командную строку с повышенными привилегиями.
    
12. Выполните следующую команду.  
    ```PowerShell
    powershell -executionpolicy unrestricted c:\Rigel\x64\scripts\provisioning\scriptlaunch.ps1 ApplyCurrentRegionAndLanguage.ps1
    ```
    
13. Перезапустите систему.
    
Теперь нужное вам язык будет применено к консоли комнат Microsoft Teams.
## <a name="initial-set-up-of-the-console"></a>Начальная настройка консоли
<a name="Initial"> </a>

После установки Windows консоль приложения "Комнаты Microsoft Teams" начнет перейти в процесс начальной настройки при следующем его начале или при выбранном варианте /reboot.
  
1. Появится экран "Учетная запись пользователя". Введите адрес для входа в Skype (user@domain формате) учетной записи комнаты, которая будет использоваться с консолью.
    
2. Введите пароль для учетной записи комнаты и подтвердите его.
    
3. В области "Настройка домена" установите FQDN для сервера Skype для бизнеса. Если домен SIP Skype для бизнеса отличается от домена Exchange пользователя, введите в этом поле домен Exchange.
    
4. Нажмите кнопку **Далее**.
    
5. Выберите указанные устройства на экране "Возможности" и нажмите кнопку **"Далее".** По умолчанию автоматическая демонстрация экрана включена, а скрытие имен во время собрания отключено. Для выбора доступны следующие устройства:
    
   - Микрофон для конференций. Микрофон, который используется по умолчанию в этом конференц-зале.
    
   - Динамик для конференций. Динамик, который используется по умолчанию для проведения конференций.  
    
   - Динамик по умолчанию. Динамик, который используется для воспроизведения звука со входа HDMI.
    
     Для каждого элемента есть меню параметров, которые можно выбрать. Необходимо выбрать его для каждого устройства.
    
6. Нажмите **Готово**.
    
Приложение консоли комнат Microsoft Teams должно сразу же начать вход в Skype для бизнеса Server с учетными данными, которые введены выше, и синхронизировать календарь с Exchange с помощью этих же учетных данных. Подробные сведения об использовании приложения консоли можно найти в справке [по комнатам Microsoft Teams.](https://support.office.com/article/Skype-Room-Systems-version-2-help-e667f40e-5aab-40c1-bd68-611fe0002ba2)
  
> [!IMPORTANT]
> В комнатах Microsoft Teams есть сертифицированное консольное оборудование. Даже правильно созданное изображение с консолью "Комнаты Microsoft Teams" не будет загрузиться после начальной процедуры настройки, если оборудование консоли не будет обнаружено. Для решения Surface Pro для проверки необходимо подключение Surface Pro к прилагаемой док-станции.
  
> [!NOTE]
> Если на сенсорной клавиатуре не поддерживаются символы, некоторым пользователям на других языках может потребоваться физическая клавиатура, подключенная к консоли.
  
### <a name="install-a-private-ca-certificate-on-the-console"></a>Установка частного сертификата ЦС на консоли
<a name="Certs"> </a>

Консоли комнат Microsoft Teams необходимо доверять сертификатам, используемым серверами, к которые она подключается. Для O365 это происходит автоматически, так как эти серверы используют общедоступные органы сертификации, и Windows 10 автоматически доверяет их. Если сертификат является частным, например при локальном развертывании с Active Directory и сертификатом Windows, вы можете добавить сертификат на консоль комнат Microsoft Teams несколькими способами:
  
- Вы можете присоединиться к консоли в Active Directory, и она автоматически добавит необходимые сертификаты с учетом публикации центра сертификации в Active Directory (обычный вариант развертывания).
    
- После обработки изображений сертификат можно установить вручную. Прежде чем это сделать, необходимо завершить начальное [настройка консоли.](console.md#Initial)
    
### <a name="to-manually-install-the-certificate"></a>Установка сертификата вручную

1. Скачайте сертификат ЦС на компьютер и сохраните его в "C:\Skype Room Systems\x64\Scripts\Provisioning\CAcertificate.cer".
    
2. Поместите консоль в режим администрирования (см. [режим администрирования и управление устройствами).](rooms-operations.md#AdminMode)
    
3. Выполните следующую команду.
    
   ```PowerShell
   certutil -addstore -f -enterprise root "C:\Skype Room Systems\x64\Scripts\Provisioning\CAcertificate.cer"
   ```

### <a name="join-an-active-directory-domain-optional"></a>Присоединиться к домену Active Directory (необязательно)
<a name="Certs"> </a>

Вы можете присоединиться к консолям комнат Microsoft Teams в вашем домене. Консоли комнат Microsoft Teams следует размещать в отдельном обл. и на рабочих станциях компьютера, так как многие политики рабочих станции несовместимы с комнатами Microsoft Teams. Распространенным примером являются политики обеспечения паролей, которые предотвращают автоматический запуск комнат Microsoft Teams. Сведения об управлении настройками GPO можно найти в сведениях об управлении комнатами [Microsoft Teams.](rooms-operations.md)
  
### <a name="to-join-microsoft-teams-rooms-to-a-domain"></a>Чтобы присоединиться к комнатам Microsoft Teams в домене

1. Во войти в консоль можно из учетной записи администратора (см. [режим администрирования и управление устройствами).](rooms-operations.md#AdminMode)
    
2. Запустите командную подсказку Powershell с повышенными уровнями.
    
3. Введите следующую команду в Powershell:
    
   ```PowerShell
   Add-Computer -DomainName <Fully qualified domain> -OUPath "OU=<Child OU>, … ,OU=<Top level OU>,DC=<child domain>,…,DC=<top level domain>"
   ```

Например, если ваш полное доменное имя redmond.corp.microsoft.com и вы хотите, чтобы консоли комнат Microsoft Teams были в "комнатах Microsoft Teams", который является потомком "Ресурсов", для этой команды будет:
  
```PowerShell
Add-Computer -DomainName redmond.corp.microsoft.com -OUPath "OU=Microsoft_Teams_Rooms,OU=Resources,DC=redmond,DC=corp,DC=microsoft,DC=com"
```

 Если вы хотите переименовать компьютер при присоединении к домену, используйте флаг -NewName и новое имя компьютера.
