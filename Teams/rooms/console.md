# <a name="build-a-microsoft-teams-rooms-image"></a>Создание образа Комнаты Microsoft Teams

В этой статье описывается создание образа Комнаты Microsoft Teams для массового развертывания Комнаты Teams.

> [!NOTE]
> Следующие шаги следует использовать только при создании образа на основе [WIM для](/windows-hardware/manufacture/desktop/capture-and-apply-an-image) массового развертывания. Если вы восстанавливаете отдельные устройства, обратитесь за поддержкой к изготовителю оборудования.

Эти действия следует выполнять только в том случае, если необходимые учетные записи Microsoft Teams или Skype для бизнеса и Exchange уже созданы и протестированные, как описано в статье [Комнаты Microsoft Teams](rooms-deploy.md). Вам потребуется оборудование и программное обеспечение, описанные [в Комнаты Microsoft Teams требованиях](requirements.md). В этой статье содержатся следующие разделы.
  
- [Подготовка установного носителя](console.md#Prep_Media)
- [Установка сертификата частного ЦС на консоли](console.md#Certs)
- [Установка Windows 10 и консольного Комнаты Microsoft Teams приложения](console.md#Reimage)
- [Начальная настройка консоли](console.md#Initial)
- [Комнаты Microsoft Teams развертывания](console.md#Checklist)

## <a name="prepare-the-installation-media"></a>Подготовка установного носителя
<a name="Prep_Media"> </a>

Для установки консольного Комнаты Microsoft Teams требуется USB-устройство хранения с емкостью не менее 32 ГБ. На устройстве не должно быть других файлов. Все существующие файлы в USB-хранилище будут потеряны.
  
> [!NOTE]
> Если не создать Комнаты Microsoft Teams установок в соответствии с этими инструкциями, скорее всего, это приведет к неожиданному поведению.

> [!NOTE]
> Ниже описан процесс создания установного носителя для создания образа Комнаты Microsoft Teams устройств. Существующие устройства по умолчанию обновляются автоматически из клиентский компонент Центра обновления Windows Магазина Windows.

> [!IMPORTANT]
> Компьютер Windows 10, используемый для создания Комнаты Microsoft Teams установки, должен находиться в той же или более поздней версии Windows, что и целевой установный носитель.
  
1. Скачайте [CreateSrsMedia.ps1 скрипта](https://go.microsoft.com/fwlink/?linkid=867842).
2. Запустите сценарий CreateSrsMedia.ps1 из командной строки с повышенными привилегиями на компьютере с ОС Windows 10.
3. Следуйте инструкциям сценария, чтобы создать Комнаты Microsoft Teams USB-накопителя.


> [!TIP]
> При каждом запуске CreateSrsMedia.ps1 сценарий в выходных данных экрана будет содержаться имя файла журнала или расшифровки сеанса. При возникновении проблем с выполнением скрипта убедитесь, что копия расшифровки доступна при запросе поддержки. 

Скрипт CreateSrsMedia.ps1 автоматизирует следующие задачи:

1. Скачайте последнюю версию установщика MSI для Комнаты Microsoft Teams.
2. Определите сборку Windows, которую должен предоставить пользователь. Последние выпущенные версии могут быть или не протестироваться и поддерживаться для использования с Комнаты Microsoft Teams устройствами.
3. Скачайте необходимые вспомогательные компоненты.
4. Соберите необходимые компоненты на установяемом носителе.

> [!NOTE]
Требуется определенная Windows 10, и эта версия доступна только клиентам корпоративного лицензирования.  Вы можете получить копию из [Центра обслуживания корпоративного лицензирования](https://www.microsoft.com/Licensing/servicecenter/).

По завершении удалите USB-диск с компьютера и перейдите к Windows 10 [и Комнаты Microsoft Teams консольного приложения](console.md#Reimage).

    
## <a name="install-windows-10-and-the-microsoft-teams-rooms-console-app"></a>Установка Windows 10 и консольного Комнаты Microsoft Teams приложения
<a name="Reimage"> </a>

Теперь необходимо применить созданный вами носитель установки. Целевое устройство будет работать как устройство, а пользователь по умолчанию будет настроен на запуск только Комнаты Microsoft Teams приложения.

1. Если целевое устройство будет установлено на док-станции (например, Surface Pro), отключите его от док-станции.

2. Убедитесь, что целевое устройство не подключено к сети.

3. Убедитесь, что целевое устройство подключено к питания ac.

4. Подключите диск установки USB к целевому устройству.

5. Загрузка на диск установки USB. Следуйте инструкциям производителя. Если целевое устройство является Surface Pro, выполните следующие действия для загрузки на диск установки USB:

    а) Нажмите и продолжайте удерживать кнопку уменьшения громкости (-).

    б) Нажмите и отпустите кнопку питания.

    в. После загрузки установки Windows отпустите кнопку уменьшения громкости (–).

8. После завершения установки система завершит работу.
    
После завершения работы системы можно безопасно удалить диск установки USB. На этом этапе целевое устройство можно поместить на док-станцию (при использовании док-станции), подключить периферийные устройства, необходимые для комнаты собраний, и подключиться к сети. Следуйте инструкциям производителя.

> [!NOTE]
> Обновления программного обеспечения Комнаты Microsoft Teams автоматически скачиваются с Microsoft Store для бизнеса. [Ознакомьтесь с предварительными требованиями для Microsoft Store для бизнеса](/microsoft-store/prerequisites-microsoft-store-for-business) и образования, чтобы убедиться, что консоль комнаты сможет получить доступ к магазину и самостоятельно обновиться.  

### <a name="selecting-a-language"></a>Выбор языка 

В Обновлении создателя необходимо использовать сценарий ApplyCurrentRegionAndLanguage.ps1 в сценариях, где неявный выбор языка не предоставляет пользователю нужный язык приложения (например, консольное приложение должно отображаться на французском языке, но оно будет отображаться на английском языке).
  
> [!NOTE]
> Приведенные ниже инструкции работают только для консолей, созданных с помощью Обновления создателя Windows (Windows 10 20H1) или более поздней версии.
  
### <a name="to-apply-your-desired-language"></a>Применение необходимого языка

1. Переключитесь в режим администрирования.
    
2. Откройте меню "Пуск".
    
3. Щелкните значок шестеренки, чтобы запустить приложение **Параметры**.
    
4. Выберите **язык времени&amp;**.
    
5. Выберите **язык**.
    
6. Выберите **Добавить язык**.
    
7. Выберите язык, который требуется добавить.
    
8. Установка языковых функций.
    
9. Не проверяйте set as my Windows display language. (Не проверяйте set as my Windows display language).
    
10. Выберите **"Установить"**.
    
11. Выберите язык, который вы только что добавили в список "Языки".
    
12. Задайте значение по умолчанию — стрелка вверх, чтобы задать значение по умолчанию

13. Удаление языков
    
    а. Выберите язык, который требуется удалить.
    
    б) Выберите Удалить.

14. Запустите командную строку с повышенными привилегиями.

15. Выполните следующую команду.  
    ```PowerShell
    powershell -executionpolicy unrestricted c:\Rigel\x64\scripts\provisioning\scriptlaunch.ps1 ApplyCurrentRegionAndLanguage.ps1
    ```
    
16. Перезапустите систему.
    
Требуемый язык теперь применяется к Комнаты Microsoft Teams консоли.
## <a name="initial-set-up-of-the-console"></a>Начальная настройка консоли
<a name="Initial"> </a>

После установки Windows приложение Комнаты Microsoft Teams в процесс начальной установки.
  
1. Появится экран "Учетная запись пользователя". Введите адрес входа учетной записи ресурса Microsoft Exchange (в user@domain формате) учетной записи комнаты, которая будет использоваться с консолью.
    
2. Введите пароль для учетной записи комнаты и подтвердите его.
   
3. Выберите поддерживаемый режим собрания — только Microsoft Teams, Skype для бизнеса только или один из двух параметров смешанного режима. При необходимости включите современную проверку подлинности.

4. Нажмите **Далее**.
    
5. Если используется Skype для бизнеса и если Skype для бизнеса SIP отличается от домена Exchange пользователя, задайте полное доменное имя Skype для бизнеса Server в разделе "Дополнительно". Если вы не используете Skype для бизнеса домен SIP соответствует домену Exchange, оставьте этот раздел пустым.
6. Нажмите **Далее**.
    
7. Нажмите **кнопку "Готово"**.
    
Приложение Комнаты Microsoft Teams войти в Microsoft Teams или Skype для бизнеса Server с учетными данными, введенными выше, а также начать синхронизацию календаря с Exchange с использованием тех же учетных данных. Дополнительные сведения об использовании Комнаты Teams см. в Комнаты Microsoft Teams [справке](https://support.office.com/article/Skype-Room-Systems-version-2-help-e667f40e-5aab-40c1-bd68-611fe0002ba2).
  
> [!IMPORTANT]
> Комнаты Microsoft Teams зависит от наличия сертифицированного консольного оборудования. Даже правильно созданный образ, содержащий Комнаты Microsoft Teams консольного приложения, не будет выполнять загрузку после начальной процедуры установки, если не будет обнаружено оборудование консоли. Для Surface Pro решений необходимо подключить Surface Pro к соответствующему оборудованию док-станции, чтобы пройти эту проверку.
  
> [!NOTE]
> Некоторым пользователям, не использующим английский язык, может потребоваться физическая клавиатура, подключенная к консоли во время начальной настройки в случае, если символы не поддерживаются на сенсорной клавиатуре.
  
### <a name="install-a-private-ca-certificate-on-the-console"></a>Установка сертификата частного ЦС на консоли
<a name="Certs"> </a>
> [!NOTE]
> Следующее применимо только при подключении Комнаты Teams к Skype для бизнеса.

Комнаты Microsoft Teams необходимо доверять сертификатам, используемым серверами, к которые они подключаются. В случае, если центр сертификации является частным, например при локальном развертывании с Active Directory и центром сертификации Windows, можно добавить сертификат в Комнаты Microsoft Teams несколькими способами:
  
- Вы можете присоединить консоль к Active Directory, которая автоматически добавит необходимые сертификаты при публикации центра сертификации в Active Directory (обычный вариант развертывания).
    
- Сертификат можно установить вручную после процесса создания образа. Перед этим необходимо завершить начальную [настройку консоли](console.md#Initial).
    
### <a name="to-manually-install-the-certificate"></a>Установка сертификата вручную

1. Скачайте сертификат ЦС на компьютер и сохраните его в папке C:\Skype Room Systems\x64\Scripts\Provisioning\CAcertificate.cer.
    
2. Поместите консоль в режим администрирования (см[. Администратор режиме и управлении устройствами](rooms-operations.md#AdminMode)).
    
3. Выполните следующую команду.
    
   ```PowerShell
   certutil -addstore -f -enterprise root "C:\Skype Room Systems\x64\Scripts\Provisioning\CAcertificate.cer"
   ```

### <a name="join-an-active-directory-domain-optional"></a>Присоединение к домену Active Directory (необязательно)
<a name="Certs"> </a>

Вы можете присоединить Комнаты Microsoft Teams к домену. Комнаты Microsoft Teams следует поместить в отдельное подразделение от рабочих станций пк, так как многие политики рабочих станций несовместимы с Комнаты Microsoft Teams. Распространенным примером является политика принудительного применения паролей, которая Комнаты Microsoft Teams запускаться автоматически. Сведения об управлении параметрами объектов групповой политики см. в разделе ["Управление Комнаты Microsoft Teams](rooms-operations.md)".
  
### <a name="to-join-microsoft-teams-rooms-to-a-domain"></a>Присоединение Комнаты Microsoft Teams к домену

1. Войдите в консоль из учетной записи администратора (см. Администратор [режиме и управлении устройствами](rooms-operations.md#AdminMode)).
    
2. Запустите командную строку PowerShell с повышенными привилегиями.
    
3. Введите следующую команду в PowerShell:
    
   ```PowerShell
   Add-Computer -DomainName <Fully qualified domain> -OUPath "OU=<Child OU>, ... ,OU=<Top level OU>,DC=<child domain>,...,DC=<top level domain>"
   ```

Например, если ваш полный домен является redmond.corp.microsoft.com и вы хотите, чтобы консоли Комнаты Microsoft Teams были в подразделении Комнаты Microsoft Teams, которое является дочерним для подразделения Resources, команда будет:
  
```PowerShell
Add-Computer -DomainName redmond.corp.microsoft.com -OUPath "OU=Microsoft_Teams_Rooms,OU=Resources,DC=redmond,DC=corp,DC=microsoft,DC=com"
```

 Если вы хотите переименовать компьютер при присоединении к домену, используйте флаг -NewName, за которым следует новое имя компьютера.
