# <a name="assignments-in-teams-for-education"></a>Задания в Teams для образовательных учреждений

Функции "Задания" и "Оценки" в Teams для образования позволяют преподавателям назначать своим учащимся задачи, задания или тесты. Преподаватели могут управлять временными шкалами заданий, инструкциями, добавлять ресурсы для списания заданий, оценки с помощью рубрик и т. д. Они также могут отслеживать ход занятий и отдельных учащихся на вкладке Оценки.

[Подробнее о заданиях и оценках можно узнать в Teams для образования.](https://support.office.com/article/microsoft-teams-5aa4431a-8a3c-4aa5-87a6-b6401abea114?ui=en-US&rs=en-IE&ad=IE#ID0EAABAAA=Assignments)

> [!Note]
> Подробные сведения о Teams на разных платформах см. в Teams [по платформам.](https://support.microsoft.com/office/teams-features-by-platform-debe7ff4-7db4-4138-b7d0-fcc276f392d3)

## <a name="assignments-integrations-in-the-microsoft-teams-admin-center"></a>Интеграция заданий в Центре Microsoft Teams администрирования

С помощью параметров администратора в Центре Microsoft Teams вы можете включить или отключить функции для преподавателей в организации и их учащихся. Ниже параметров, связанных с заданиями:

<a name="#bkemaildigest"> </a>

### <a name="weekly-guardian-email-digest"></a>Еженедельный электронный дайджест для опекунов

Письма опекунов отправляются родителям или опекунам каждые выходные. Сообщение электронной почты содержит сведения о заданиях за предыдущую и предстоящую неделю. Синхронизацию для родителей и опекунов можно настроить с помощью [Синхронизация сведений о школе.](/schooldatasync/parent-contact-sync)

1. Импортируйте контактные данные родителей с помощью синхронизации для родителей и опекунов в SDS. Инструкции по включению синхронизации для родителей и опекунов см. в этой [ссылке.](/schooldatasync/parent-contact-sync#enabling-parent-and-guardian-sync)

2. Включите параметр Опекун в центре Microsoft Teams, так как этот параметр отключен по умолчанию. Это позволит преподавателям отправлять еженедельный дайджест.

   > [!NOTE]
   > Преподаватели могут отказаться от дайджеста, отобрав этот параметр в своей личной группе класса (в Параметры > для родителей и **опекунов).**

Чтобы убедиться, что родители получат сообщение, должны быть истинными следующие три пункта:

- Адрес электронной почты, вложенный в профиль учащегося в SDS и помечен как _родительский или_ _опекун._ Подробные сведения см. в теме Формат файлов синхронизации для родителей [и опекунов.](/schooldatasync/parent-contact-sync-file-format)

- Учащиеся относятся хотя бы к одному классу, в котором преподаватели не отключили электронную почту в [параметрах задания.](https://support.microsoft.com/office/adjust-assignment-settings-in-your-class-team-05bb3b89-1cdf-415a-b6c7-44add0376a77)

- Сообщения электронной почты будут содержать сведения о заданиях, срок выполнения которых был от предыдущей недели или предстоящую неделю.

По умолчанию для этой функции заданная функция **отключена.**

<a name="bkmakecode"> </a>

### <a name="makecode"></a>MakeCode

Microsoft MakeCode — это платформа программирования с блокировкой, которая позволяет информатиковать всех учащихся.

MakeCode — это продукт Корпорации [](https://go.microsoft.com/fwlink/?LinkID=206977) Майкрософт, на который налагаются условия использования и политики [конфиденциальности](https://go.microsoft.com/fwlink/?LinkId=521839) корпорации Майкрософт.

По умолчанию для этой функции заданная функция **отключена.**

Чтобы включить задания MakeCode в Teams, перейдите в Центр администрирования  **Teams**, перейдите в раздел Задания и включите параметр MakeCode вкл. . Нажмите кнопку **Сохранить**. Чтобы эти параметры вступили в силу, в течение нескольких часов.

Дополнительные сведения о том, как работает эта функция, можно посмотреть в этом [видео.](https://makecode.com/blog/teams/teams-assignments)

[Подробнее о MakeCode](https://aka.ms/makecode).

### <a name="turnitin"></a>Turnitin

[Turnitin](https://www.turnitin.com/) — это служба целостности знаний. Это сторонние службы, на которые лагаются собственные условия и политика конфиденциальности. Вы несете ответственность за использование сторонних продуктов и служб.

Значение по умолчанию для этой функции — **Отключено**..

Чтобы включить Turnitin для организации, вам потребуется подписка на Turnitin. Затем вы можете ввести следующие сведения, которые можно найти в консоли администратора Turnitin:

В консоли администратора можно создать интеграцию и ключ API, связанный с ней.

Выберите **Интеграции** в боковом меню, а затем выберите **Добавить интеграцию** и придайте интеграции имя.

![Снимок экрана: добавление новой интеграции.](./educationImages/Assignments_mopo_turnitin2.png)

**TurnitinApiKey** будет предоставлен вам после того, как вы следуйте этим запросам.
Скопируйте ключ API и вкопируйте его в Microsoft Teams администрирования.  Это единственный раз, когда можно просмотреть ключ.

![Снимок экрана: копирование ключа API.](./educationImages/Assignments_mopo_turnitin3.png)

## <a name="assignments-data"></a>Данные о назначениях

В заданиях хранится информация, сгенерированная преподавателями и учащимися. Все данные совместно делиться между преподавателем и конкретным учеником, для которого они предназначены в классе. Это два магазина: SharePoint и за пределами SharePoint.

>[!NOTE]
>Эти же правила применяются и к интеграции сторон, таким как ход чтения.

### <a name="assignments-data-in-sharepoint-document-libraries"></a>Данные назначений в SharePoint документа

Файлы учащихся, связанные с отправкой задания, хранятся в библиотеке документов (она называется Работы *учащихся).* Файлы, связанные с заданиями, созданными преподавателями и доступными учащимися, хранятся в другой библиотеке документов (она называется *Файлы* занятий) на соответствующем сайте группы SharePoint занятий. Интеграция с первой стороной также может хранить данные "Задания" на соответствующем сайте группы SharePoint занятий (название задания *+ отметка времени).*

#### <a name="files-associated-with-the-student"></a>Файлы, связанные с учеником

ИТ-администраторы могут использовать средство поиска контента для поиска файлов учащихся *(работ* учащихся, файлов классов или других файлов интеграции, связанных с отправкой заданий и файлами, связанными с заданиями). Например, администратор может выполнять поиск на всех сайтах SharePoint в организации и использовать имя учащегося, а также имя класса или задания в поисковом запросе для поиска данных, соответствующих запросу на передачу данных (DSR).

#### <a name="files-associated-with-the-teacher"></a>Файлы, связанные с преподавателем

ИТ-администраторы могут использовать средство поиска контента для поиска файлов преподавателей *(работ* учащихся, файлов занятий или других 1-х сторонних файлов интеграции), связанных с заданиями, а также файлов, которые распространяются среди учащихся в классе по заданиям. Например, администратор может выполнять поиск на всех SharePoint в организации и использовать имя преподавателя, а также имя занятия или задания в поисковом запросе для поиска данных, соответствующих DSR.

### <a name="assignments-data-outside-of-sharepoint-document-libraries"></a>Данные назначений за пределами SharePoint документов

Некоторые данные, связанные с заданиями, не хранятся на сайте группы класса SharePoint, что означает, что их невозможно найти с помощью поиска контента. К ним относятся:

- Оценки учащихся и отзывы преподавателя
- Список документов, отправленных каждым учащемуся для задания
- Сведения о задании, такие как дата выполнения и т. д.
- Данные об интеграции, такие как отрывки о ходе чтения или данные о произношении учащихся

Для этого типа данных ИТ-администратору или владельцу данных, например преподавателю, может потребоваться перейти к заданию в группе класса, чтобы найти данные, соответствующие DSR. Администратор может добавить себя в класс в качестве владельца и просмотреть все задания для группы класса.

>[!NOTE]
>Если учащийся больше не является частью класса, его данные могут по-прежнему присутствовать в классе, так как *он больше не зарегистрирован*. Учащемуся необходимо предоставить администратору клиента список классов, в которых он когда-либо участвовал.

### <a name="bulk-export-assignment-data-outside-of-sharepoint-document-libraries"></a>Массовый экспорт данных назначений за SharePoint библиотек документов

#### <a name="for-a-student"></a>Для учащегося

Чтобы выполнить массовый экспорт данных одного учащегося, перед удалением учащегося из [классов,](/microsoft-365/education/deploy/configure-assignments-for-teams) в которые он входит, запустите сценарий и предо.. ``userId`` Если учащегося удалили с сайта, администратор может добавить учащегося обратно в класс перед запуском сценария или предоставить ему и его ``userId`` ``classId`` часть.

Данные о отправленных учащихся будут экспортироваться.

#### <a name="for-a-teacher"></a>Для преподавателя

Данные о задании для массового экспорта работают одинаково, но все отправленные сообщения, к которые у преподавателя есть доступ, будут экспортироваться.

### <a name="bulk-delete-assignment-data-outside-of-sharepoint-document-libraries"></a>Массовое удаление данных назначений за SharePoint библиотек документов

#### <a name="for-a-student"></a>Для учащегося

Чтобы массово удалить данные одного учащегося, перед удалением учащегося [](/microsoft-365/education/deploy/configure-assignments-for-teams) из занятий, в которые он входит, запустите сценарий и укалите ``userId`` . Если учащегося удалили с сайта, администратор может добавить учащегося обратно в класс перед запуском сценария или предоставить ему и его ``userId`` ``classId`` часть.

Если у вас есть разрешение, администратор сможет удалять только ``ClassId`` сведения об учащихся из определенного класса.

#### <a name="for-a-teacher"></a>Для преподавателя

Так как данные задания для преподавателя делиться в классе, возможность массового удаления не существует. Вместо этого администратор может добавить себя в класс, перейти в приложение и удалить задание.

Дополнительные сведения см. [в этой](/microsoft-365/education/deploy/configure-assignments-for-teams)Teams.

## <a name="removing-assignments-and-grades"></a>Удаление заданий и о заданий

Вы также можете использовать Teams, чтобы удалить задания и оценки для определенного пользователя или для всего клиента.

Чтобы удалить задания и оценки для отдельного пользователя, перейдите в Центр администрирования Teams и перейдите в Teams приложений **>** политики разрешений, чтобы создать новое определение политики разрешений приложений.   При создании определения политики установите в политике  приложений **Майкрософт** задание Блокировать  определенные приложения и разрешить всем остальным пользователям добавлять задания в список заблокированных приложений. После с сохраненного определения политики назначьте его соответствующим пользователям.

Чтобы удалить задания и оценки для всего клиента, перейдите в Центр администрирования **Teams**, перейдите к Teams  приложениям **> Управление** приложениями и найдите и выберите Задания в списке приложений. Измените параметр состояния на странице параметров приложения назначения на _Заблокировано._
