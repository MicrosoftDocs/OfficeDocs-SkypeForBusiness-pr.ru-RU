# <a name="assignments-in-teams-for-education"></a>Задания в Teams для образовательных учреждений

Функции заданий и оценок в Teams для образования позволяют преподавателям назначать задачи, трудовую работу или тесты своим учащимся. Преподаватели могут управлять временными шкалами заданий, инструкциями, добавлять ресурсы для с проверки, оценки с помощью rubrics и т. д. Они также могут отслеживать ход выполнения занятий и отдельных учащихся на вкладке "Оценки".

[Дополнительные сведения о заданиях и оценках см. в Teams для образования](https://support.office.com/article/microsoft-teams-5aa4431a-8a3c-4aa5-87a6-b6401abea114?ui=en-US&rs=en-IE&ad=IE#ID0EAABAAA=Assignments).

> [!Note]
> Дополнительные сведения о Teams на разных платформах см. [Teams по платформам](https://support.microsoft.com/office/teams-features-by-platform-debe7ff4-7db4-4138-b7d0-fcc276f392d3).

## <a name="assignments-integrations-in-the-microsoft-teams-admin-center"></a>Интеграция назначений в центре Microsoft Teams администрирования

Используя параметры администратора в Microsoft Teams администрирования, вы можете включить или отключить функции для преподавателей в организации и их учащихся. Ниже приведены параметры, связанные с назначениями.

<a name="#bkemaildigest"> </a>

### <a name="weekly-guardian-email-digest"></a>Дайджест электронной почты еженедельной защиты

Сообщения электронной почты защиты отправляются каждые выходные для родителей или опекунов. Сообщение электронной почты содержит сведения о назначениях за предыдущую неделю и на неделю. Синхронизацию родительских и опекунов можно настроить с помощью [Синхронизация сведений о школе](/schooldatasync/parent-contact-sync).

1. Импорт родительских контактных данных с помощью родительской синхронизации и синхронизации защиты в SDS. Инструкции по включению синхронизации родительских и опекунов см. в разделе ["Включение синхронизации родительских и опекунов"](/schooldatasync/parent-contact-sync#enabling-parent-and-guardian-sync).

2. Включите параметр защиты в центре администрирования Microsoft Teams, так как этот параметр отключен по умолчанию. Это позволит преподавателям отправлять еженедельный дайджест.

   > [!NOTE]
   > Преподаватели могут отказаться от дайджеста, отменив выбор параметра в собственной команде личных занятий (назначение Параметры > **родительской или опекунской электронной почты**).

Чтобы убедиться, что родительские элементы будут получать сообщение электронной почты, должны быть истинными следующие три элемента:

- Адрес электронной почты, подключенный к профилю учащегося в SDS помечен как _родительский_ _или опекун._ Дополнительные сведения см [. в разделе "Формат файла синхронизации родительских элементов и опекунов"](/schooldatasync/parent-contact-sync-file-format).

- Учащиеся относятся по крайней мере к одному классу, в котором электронная почта не отключена преподавателем в [параметрах задания](https://support.microsoft.com/office/adjust-assignment-settings-in-your-class-team-05bb3b89-1cdf-415a-b6c7-44add0376a77).

- Сообщения электронной почты будут содержать сведения о назначениях с датой выполнения за предыдущую неделю или на предстоящей неделе.

Значение по умолчанию для этой функции — **"Выкл."**.

### <a name="makecode"></a>MakeCode

Microsoft MakeCode — это платформа программирования на основе блоков, которая обеспечивает информатику для всех учащихся.

MakeCode — это продукт Майкрософт, на который распространяются условия [](https://go.microsoft.com/fwlink/?LinkID=206977) использования и [политики конфиденциальности](https://go.microsoft.com/fwlink/?LinkId=521839) корпорации Майкрософт.

Значение по умолчанию для этой функции — **"Выкл."**.

Чтобы включить назначения MakeCode в Teams, перейдите в центр **Teams Администратор, перейдите** к разделу "Назначения" и  включите переключатель MakeCode **.** Нажмите **Сохранить**. В течение нескольких часов эти параметры вступает в силу.

Дополнительные сведения о том, как работает эта функция, см. в этом [видеоролике](https://makecode.com/blog/teams/teams-assignments).

[Дополнительные сведения о MakeCode](https://aka.ms/makecode).

### <a name="turnitin"></a>Turnitin

[Turnitin](https://www.turnitin.com/) — это служба целостности для образовательных учреждений. Это сторонняя служба, на которую распространяются собственные условия и политика конфиденциальности. Вы несете ответственность за использование любых сторонних продуктов и служб.

Значение по умолчанию для этой функции — **"Выкл."**.

Чтобы включить Turnitin для организации, вам потребуется подписка Turnitin. Затем можно ввести следующие сведения, которые можно найти в консоли администрирования Turnitin:

- **TurnitinApiKey**: это 32-символьный GUID, найденный в консоли администрирования в разделе Integrations.
- **TurnitinApiUrl**: ЭТО URL-адрес HTTPS консоли администрирования Turnitin.

Ниже приведены некоторые инструкции, которые помогут вам получить эти сведения.

**TurnitinApiUrl** — это адрес узла консоли администрирования.
Пример: `https://your-tenant-name.turnitin.com`

Консоль администрирования — это место, где можно создать интеграцию и ключ API, связанный с интеграцией.

Выберите **"Интеграции** " в боковом меню, а затем выберите **"Добавить интеграцию** " и присвойте интеграции имя.

![Снимок экрана: добавление новой интеграции.](./educationImages/Assignments_mopo_turnitin2.png)

**TurnitinApiKey** будет предоставлен вам после выполнения запросов.
Скопируйте ключ API и вставьте его в Microsoft Teams центра администрирования.  Это единственный раз, когда можно просмотреть ключ.

![Снимок экрана: копирование ключа API.](./educationImages/Assignments_mopo_turnitin3.png)

Нажав кнопку **"** Сохранить" в Центре администрирования для этого параметра, введите эти параметры в силу через несколько часов.

## <a name="assignments-data"></a>Данные назначений

В заданиях хранятся сведения, создаваемые преподавателями и учащимися. Все данные совместно используются преподавателем и конкретным учеником, для которого информация предназначена в классе. Эти данные хранятся в двух хранилищах, SharePoint и за пределами SharePoint.

>[!NOTE]
>Те же правила применяются и к интеграции от первой стороны, например к ходу чтения.

### <a name="assignments-data-in-sharepoint-document-libraries"></a>Назначение данных в SharePoint документов

Файлы учащихся, связанные с отправкой для задания, хранятся в библиотеке документов (с именем *Student Work*). Файлы, связанные с заданиями, созданными преподавателями и доступными учащимся, хранятся в другой библиотеке документов (с именем "*Файлы* классов") на соответствующем сайте SharePoint класса. Интеграция с первой стороной также может хранить данные назначений на том же соответствующем сайте SharePoint класса (с именем: *Assignments title + time stamp*).

#### <a name="files-associated-with-the-student"></a>Файлы, связанные с учащегося

ИТ-администраторы могут использовать средство поиска контента для поиска файлов учащихся (student *Work*, *Class Files* или других файлов интеграции 1-й стороны), связанных с отправками заданий и файлами, связанными с заданиями. Например, администратор может выполнить поиск SharePoint всех сайтов в организации и использовать имя учащегося, класс или назначение в поисковом запросе для поиска данных, относящихся к запросу субъекта данных (DSR).

#### <a name="files-associated-with-the-teacher"></a>Файлы, связанные с преподавателем

ИТ-администраторы могут использовать средство "Поиск контента" для поиска файлов преподавателей (student *Work*, *Class Files* или других файлов интеграции 1-й стороны), связанных с заданиями и файлами, которые преподаватели в классе распространяют на задания. Например, администратор может выполнять поиск SharePoint всех сайтов в организации и использовать имя преподавателя, класс или назначение в поисковом запросе для поиска данных, относящихся к DSR.

### <a name="assignments-data-outside-of-sharepoint-document-libraries"></a>Назначение данных за пределами SharePoint библиотек документов

Некоторые данные, связанные с заданиями, не хранятся на сайте SharePoint класса, что означает, что они недоступны для поиска контента. К ним относятся:

- Оценки учащихся и отзывы преподавателя
- Список документов, отправленных для задания каждым учащегося
- Сведения о назначении, такие как дата выполнения и т. д.
- Данные интеграции от первой стороны, такие как чтение записей о ходе выполнения или произношение учащихся

Для этого типа данных ИТ-администратору или владельцу данных, например преподавателю, может потребоваться перейти к заданию в команде класса, чтобы найти данные, относящиеся к DSR. Администратор может добавить себя в качестве владельца класса и просмотреть все назначения для этой команды класса.

>[!NOTE]
>Если учащийся больше не является частью класса, его данные могут по-прежнему присутствовать в классе, *так как они больше не зарегистрированы*. Учащемуся необходимо предоставить администратору клиента список таких классов, в которые он когда-либо влиял.

### <a name="bulk-export-assignment-data-outside-of-sharepoint-document-libraries"></a>Массовый экспорт данных назначения за пределами SharePoint библиотек документов

#### <a name="for-a-student"></a>Для учащегося

Чтобы выполнить массовый экспорт данных одного учащегося, перед удалением учащегося из [классов, в](/microsoft-365/education/deploy/configure-assignments-for-teams) которые он входит, запустите скрипт и укажите ``userId``. Если учащийся был удален с сайта, администратор может добавить учащегося обратно в класс перед выполнением скрипта, ``userId`` ``classId`` или администратор может предоставить и то, с чем учащийся когда-либо был частью.

Данные об отправках учащихся будут экспортированы.

#### <a name="for-a-teacher"></a>Для преподавателя

Данные задания массового экспорта работают одинаково для учащегося, но все отправки, к которых преподаватель имеет доступ, будут экспортированы.

### <a name="bulk-delete-assignment-data-outside-of-sharepoint-document-libraries"></a>Массовое удаление данных назначения за пределами SharePoint библиотек документов

#### <a name="for-a-student"></a>Для учащегося

Чтобы массово удалить данные одного учащегося, перед удалением учащегося из [классов, в](/microsoft-365/education/deploy/configure-assignments-for-teams) которые он входит, запустите скрипт и укажите ``userId``. Если учащийся был удален с сайта, администратор может добавить учащегося обратно в класс перед выполнением скрипта, ``userId`` ``classId`` или администратор может предоставить и то, с чем учащийся когда-либо был частью.

Предоставление разрешения ``ClassId`` позволяет администратору удалять только сведения о учащемся из определенного класса.

#### <a name="for-a-teacher"></a>Для преподавателя

Так как данные задания для преподавателя совместно используются в классе, возможность массового удаления отсутствует. Вместо этого администратор может добавить себя в класс, перейти в приложение и удалить назначение.

Дополнительные сведения см. в [разделе "Настройка назначений для Teams](/microsoft-365/education/deploy/configure-assignments-for-teams)".

## <a name="removing-assignments-and-grades"></a>Удаление заданий и оценок

Вы также можете использовать Teams для удаления заданий и оценок для определенного пользователя или для всего клиента.

Чтобы удалить назначения и оценки для отдельного пользователя, перейдите в центр **Teams Администратор и** перейдите к Teams приложениям **>** политики разрешений для создания нового определения политики разрешений приложений.  При создании нового определения политики задайте для политики приложений **Майкрософт** значение _"_ Блокировать определенные приложения", разрешите всем  остальным и добавьте назначения в список заблокированных приложений. После сохранения нового определения политики назначьте его соответствующим пользователям.

Чтобы удалить задания и оценки для всего клиента, перейдите в **Teams Администратор Center**, перейдите к Teams приложениям **> Управление** приложениями, найдите и выберите "Задания" в списке приложений. Измените значение параметра состояния на странице параметров приложения назначения на _"Заблокировано"_.

## <a name="assignments-diagnostic-tool-for-users"></a>Средство диагностики назначений для пользователей

служба поддержки Майкрософт средство для сбора диагностических данных для команды инженеров Майкрософт для изучения проблем, связанных с функцией назначений.

Доступ к этому средству можно получить в разделе "Назначения" на любом экране, с которым у пользователей возникает проблема.

Чтобы получить средство диагностики в Teams, пользователи могут:

- **На рабочем столе и в Интернете:**
  - Нажмите клавиши CTRL+/.
- **На мобильных устройствах:**
  - Касание экрана двумя пальцами и поворот пальцами на 45 градусов или
  - Коснитесь экрана тремя пальцами в течение 15 секунд

Когда средство диагностики отобразится, пользователи будут видеть список данных, которые могут потребоваться службой технической поддержки Майкрософт.

Извлеченные данные могут включать:

- Идентификатор группы
- Идентификатор клиента
- Идентификатор сеанса
- Идентификатор назначения
- Идентификатор отправки
- Идентификатор пользователя

Эти данные не отправляются в корпорацию Майкрософт автоматически. Пользователи должны скопировать и вставить данные в агент службы поддержки Майкрософт в отношении запроса в службу поддержки.

Если пользователь извлекет средство диагностики, а затем закрывает его, данные не отправляются.

Когда данные отправляются в агент поддержки Майкрософт, они обрабатываются как данные поддержки в соответствии с соглашениями Microsoft 365 службы.

Инструкции по использованию этого средства диагностики, которые можно предоставлять преподавателям и учащимся, см. в статье "Получение диагностических данных для устранения [неполадок заданий"](https://support.microsoft.com/topic/b40793f5-dbae-4c8a-841a-6baa7f232e2e).
