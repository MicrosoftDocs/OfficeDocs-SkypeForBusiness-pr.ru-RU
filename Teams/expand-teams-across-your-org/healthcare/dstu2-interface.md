# <a name="dstu2-interface-specification"></a>Спецификации интерфейса DSTU2

[!INCLUDE [preview-feature](../../includes/preview-feature.md)]

Настройка или перенастройка сервер FHIR для работы с приложением Microsoft группами пострадавших необходимо понимание того, какие данные требуется доступ к приложению. FHIR сервер должен поддерживать запросы POST, с помощью пакета для со следующими ресурсами:

- [Пострадавший](#patient)
- [Наблюдения](#observation)
- [Условие](#condition)
- [Встретиться](#encounter)
- [Intolerance аллергических реакций](#allergyintolerance)
- [Покрытия](#coverage)
- [Порядок лечения](#medication-order)
- [Расположение](#location)

> [!NOTE]
> Пациента ресурсов — это единственная обязательная ресурсов, (без которой приложение не будет загружен вообще. Тем не менее рекомендуется, что партнера реализации поддержки выше ресурсов на спецификации представлены ниже для рекомендации для конечных пользователей с группами пострадавших Microsoft App.

Запросы из приложения пострадавших группами Майкрософт для нескольких ресурсов Публикация пакета (пакет) запросов для URL-адрес сервера FHIR. Сервер обрабатывает каждого запроса и возвращает набор ресурсов, совпадающих с каждого запроса. Дополнительные сведения и примеры, в разделе [https://www.hl7.org/fhir/DSTU2/http.html#transaction](https://www.hl7.org/fhir/DSTU2/http.html#transaction).

Следующие ресурсы FHIR должен быть доступен с помощью ссылки на прямое ресурсов.

## <a name="conformance-minimum-required-field-set"></a>Задайте о соответствии минимальным обязательное поле

 Сервер FHIR необходимо реализовать заявление о соответствии для нас иметь факты Сводка его возможности. Мы рассчитываем ниже параметров в сервере FHIR DSTU2:

1. REST
   1. Режим
   2. Взаимодействие
   3. Ресурс: тип
   4. Безопасность: [расширения для URI OAuth](http://hl7.org/fhir/extension-oauth-uris.html)
2. FhirVersion (код требует, чтобы она понять, какие версии, мы должны вводить несколько как мы поддерживают различные версии.)

Просмотреть [https://www.hl7.org/fhir/dstu2/conformance.html](https://www.hl7.org/fhir/dstu2/conformance.html) другие сведения в этом поле значение.

## <a name="patient"></a>Пострадавший

Далее представлены минимальные обязательные поля, которые представляют собой подмножество полей [Argonaut пациента профилей](http://www.fhir.org/guides/argonaut/r2/StructureDefinition-argo-patient.html) :

1. Name.Family
2. Name.Given
3. Пол
4. Дата рождения
5. MRN (идентификатор)

Помимо полей Argonaut для удобную работу пользователя приложения пострадавших также считывает следующие поля:

1. Name.Use
2. Name.Prefix
3. CareProvider (этой ссылки на пациента ресурса должна включать отображения полей, отображаемых в следующем примере).

* * *

    Запрос: Получение <fhir-server>/пострадавший/<patient-id>
    
    Ответ: {«типа ресурса»: «Пациента», «id»: «<patient-id>».
      .
      .
      «Имя»: [{«используйте»: «официальный», «префикс»: [«Mr»], «семья»: [«Chau»], «присвоенное»: [«Хью»]}], «идентификатор»: [{«используйте»: «официальный», «тип»: {«создание кода»: [{«система»: "http://hl7.org/fhir/v2/0203«, «код»: «MR»}]}, «значение»: «1234567»}], «Пол»: «штекер», «Дата рождения»: «1957-06-05 «, «careProvider»: [{«отображение»: «Иван Петров»}],}

* * *

Поиск ресурсов использует метод POST на /Patient/_search and следующие параметры:

1. Идентификатор
2. Семейство: содержит = (выполняет поиск всех пострадавших семейства, имя которого содержит значение).
3. Данный =\<substring>
4. имя =\<substring>
5. Дата рождения =(exact match)
6. \_Count (максимальное количество результатов, которые должны быть возвращены) <br> Ответ должен содержать общее число записей, возвращаемых в результате поиска, и \_count будут использовать PatientsApp для ограничения числа возвращаемых записей.
7. Идентификатор =\<mrn>

Цель — должны иметь возможность поиска и фильтров для пострадавший следующими службами:

- ID — Это идентификатор ресурсов, у каждого ресурса FHIR.
- MRN: Это фактический идентификатор пациент, знать клинических персонала. Мы понимаем, это MRN, основанные на тип идентификатора внутри идентификатор ресурса в FHIR
- Имя
- Дата рождения

Следующий пример вызова см.

* * *

    Запрос: <fhir-server>/пострадавший/_search запросить запроса POST: заданного = hugh&family = chau
    
    Ответ: {«типа ресурса»: «Объединяют», «id»: «<bundle-id>».
      .
      .
      «запись»: [{«ресурс»: {«типа ресурса»: «Пациента», «id»: «<patient id>», «имя»: [{«текст»: «Хью Chau», «семья»: [«Chau»], «присвоенное»: [«Хью»]}], «Пол»: «штекер», «Дата рождения»: «1957-06-05»}, «поиск»: {«режима»: «match»}}]}

* * *

Просмотреть [https://www.hl7.org/fhir/DSTU2/Patient.html](https://www.hl7.org/fhir/DSTU2/Patient.html) другие сведения в этом поле значение.

## <a name="observation"></a>Наблюдения

Далее представлены минимальные обязательные поля, которые представляют собой подмножество профилей Argonaut основные показатели:

 1. Эффективного (дата, время или периода)
 2. Code.Coding.Code
 3. ValueQuantity.Value

Помимо полей Argonaut для удобную работу пользователя приложения пострадавших также считывает следующие поля:

 1. Code.Coding.Display

## <a name="encounter"></a>Встретиться

Далее представлены минимальные обязательные поля, которые представляют собой подмножество полей "мне Нравится" встретиться основных «должен иметь» профиль:

1. Состояние
2. Тип [0]. Создание кода [0]. Отображение

Кроме того следующие поля из профиля "мне Нравится" встретиться Core «должна поддерживать» поля

1. Period.Start
2. Расположение [0]. Location.Display

Поиск ресурсов используется метод GET и следующие параметры:

1. пострадавший =\<пациента id>
2. _sort:DESC =\<поля ex. date>
3. _count =\<максимальной results>

Цель — должны иметь возможность получения последнего заданного расположения пациента. Каждый обнаружения ссылается на расположение ресурса. Справочные материалы по должен также включать отображаемое поле расположение. Следующий пример вызова см.
* * *

    Запрос: Получение <fhir-server>/обнаружения? пострадавший = <patient-id>&_sort:desc = date&_count = 1
