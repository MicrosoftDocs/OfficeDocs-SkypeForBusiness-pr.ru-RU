# <a name="stu3-interface-specification"></a>Спецификации интерфейса STU3

[!INCLUDE [preview-feature](../../includes/preview-feature.md)]

Настройка или перенастройка сервер FHIR для работы с приложением Microsoft группами пострадавших необходимо понимание того, какие данные требуется доступ к приложению. FHIR сервер должен поддерживать запросы POST, с помощью пакета для со следующими ресурсами:

- [Пострадавший](#patient)
- [Наблюдения](#observation)
- [Условие](#condition)
- [Встретиться](#encounter)
- [Intolerance аллергических реакций](#allergyintolerance)
- [Покрытия](#coverage)
- [Оператор лечения](#medication-request) (заменяет MedicationOrder в версии DSTU2 PatientsApp)
- Расположение (сведения о необходимости из этот ресурс может быть включен в обнаружения)

> [!NOTE]
> Пациента ресурсов — это единственная обязательная ресурсов, (без которой приложение не будут загружаться на всех); Тем не менее рекомендуется, что партнера реализации поддержки выше ресурсов на спецификации представлены ниже для рекомендации для конечных пользователей с группами пострадавших Microsoft App.

Запросы из приложения пострадавших группами Майкрософт для нескольких ресурсов должна учитывать пакета (пакет) запросов на URL-адрес сервера FHIR. Сервер должен обработать каждого запроса и вернуть набор ресурсов, совпадающих с каждого запроса. Дополнительные сведения и примеры, в разделе [https://www.hl7.org/fhir/STU3/http.html#transaction](https://www.hl7.org/fhir/STU3/http.html#transaction).

Далее представлены минимальные обязательные поля:

Просмотреть [https://www.hl7.org/fhir/stu3/capabilitystatement.html](https://www.hl7.org/fhir/stu3/capabilitystatement.html) другие сведения в этом поле значение.

## <a name="patient"></a>Пострадавший

Ниже приведены минимальные обязательные поля, которые представляют собой подмножество полей Argonaut пациента профилей.

1. Name.Given
2. Name.Family
3. Пол
4. Дата рождения
5. MRN (идентификатор)

Помимо [полей Argonaut](http://www.fhir.org/guides/argonaut/r2/StructureDefinition-argo-patient.html)удобную работу пользователя приложения пострадавших также содержатся следующие поля:

1. Name.Use
2. Name.Prefix
3. [GeneralPractitioner] - GeneralPractitioner ссылку должно быть включено в пациента ресурсов (только отображаемое поле)

Поиск ресурсов использует метод POST на /Patient/_search and следующие параметры:

1. Идентификатор
2. Семейство = (поиск всех пострадавших семейства, имя которого содержит значение)
3. Данный =\<substring>
4. Дата рождения =(exact match)
5. Пол = (значения из административных Пол)
6. \_Count (максимальное количество результатов, которые должны быть возвращены) <br> Ответ должен содержать общее число записей, возвращаемых в результате поиска и \_count будут использовать PatientsApp для ограничения числа возвращаемых записей.
7. Идентификатор =\<mrn>

Цель — должны иметь возможность поиска и фильтров для пострадавший следующими службами:

- ID — Это идентификатор ресурсов, у каждого ресурса FHIR.
- MRN: Это фактический идентификатор пациент, знать клинических персонала. Мы понимаем, это MRN, основанные на тип идентификатора внутри идентификатор ресурса в FHIR.
- Имя
- Дата рождения

## <a name="encounter"></a>Встретиться

Далее представлены минимальные обязательные поля, которые представляют собой подмножество полей «должен иметь» [профиля возникать ядра в США](http://hl7.org/fhir/us/core/2018Jan/StructureDefinition-us-core-encounter.html) ).

1. Состояние
2. Тип [0]. Создание кода [0]. Отображение

Цель — должны иметь возможность получения последнего заданного расположения пациента. Каждый обнаружения ссылается на расположение ресурса. Справочные материалы по должен также включать отображаемое поле расположение.
