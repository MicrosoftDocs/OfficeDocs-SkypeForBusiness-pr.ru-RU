# <a name="audit-logs-for-patients-app"></a>Журналы аудита приложения для пациентов

> [!NOTE]
> С 30 октября 2020 г. приложение "Пациенты" устарело и заменено приложением [Lists](https://support.microsoft.com/office/get-started-with-lists-in-teams-c971e46b-b36c-491b-9c35-efeddd0297db) в Teams. Данные приложения "Пациенты" хранятся в почтовом ящике группы Office 365, на которой основана команда. Все данные, связанные с приложением "Пациенты", сохраняются в этой группе, но больше недоступны в пользовательском интерфейсе. Пользователи могут повторно создать свои списки с помощью приложения [Lists](https://support.microsoft.com/office/get-started-with-lists-in-teams-c971e46b-b36c-491b-9c35-efeddd0297db).
>
>С помощью приложения Lists медицинские бригады вашей организации здравоохранения могут создавать списки пациентов для различных сценариев: от обходов и собраний междисциплинарных команд до общего наблюдения за пациентами. Ознакомьтесь с шаблоном "Пациенты" в приложении Lists, чтобы начать работу. Дополнительные сведения об управлении приложением Lists в организации см. в статье [Управление приложением Lists](../../manage-lists-app.md).

Журнал аудита действий в приложении "Пациенты" позволяет группам реагирования после инцидента проверять изменения в электронных медицинских записях пациентов (EMR) или сведениях о медицинском учреждении пациентов (ФИ) и определять, необходимы ли изменения или улучшения в политике или процедурах доступа к ФИ в инструментах повышения производительности. События журнала аудита охватывают действия, выполняемые с помощью пользовательского интерфейса приложения "Пациенты".

## <a name="meet-hipaa-requirements"></a>Соответствует требованиям HIPAA

Согласно рекомендациям hipAA поставщики услуг здравоохранения должны хранить записи о доступе ко всем данным, чтобы изменения можно было проверять. Корпорация Майкрософт стремится к своим корпоративным клиентам, использующим Microsoft Teams, и помогает им соответствовать требованиям и средствам контроля HIPAA. Доступ к функции ФИ с помощью приложения "Пациенты" полностью отслеживается, а журналы доступны в Центре соответствия требованиям Microsoft 365, как описано в статье Поиск в журнале [аудита.](/microsoft-365/compliance/search-the-audit-log-in-security-and-compliance)

> [!IMPORTANT]
> Грусти за обеспечение конфиденциальности пациентов ложится на поставщика услуг здравоохранения по закону. Закон дает пациенту право на конфиденциальность и требует, чтобы ИТ-администратор или геймпад HIPAA мог легко определять, к какой из медицинских услуг, медицинских учреждений или социальных работников были доступны или изменены записи пациентов. Одним из наиболее распространенных примеров нарушения доступа к PHI является доступ к пациенту, которые являются пациентами с статусом VIP. Функции журнала аудита необходимы для проведения исследований по любому нарушению доступа с PHI и для того, чтобы соответствовать требованиям HIPAA.

## <a name="enable-audit-logs-for-the-patients-app"></a>Включить журналы аудита для приложения "Пациенты"

Аудит зависит от нескольких предыдущих конфигураций:

1. Администратору необходимо будет работать со своим поставщиком услуг FHIR, чтобы использовать EMR в формате, используемом приложением "Пациенты". 
2. Администратору службы здравоохранения нужно будет включить приложение "Пациенты" Teams Центре администрирования. Дополнительные сведения см. в статье [Управление политиками Microsoft Teams](../../teams-app-setup-policies.md) приложениями и связанными статьями.
3. Администратор должен будет включить аудит действий так же, как и любой аудит [](/microsoft-365/compliance/search-the-audit-log-in-security-and-compliance#before-you-begin) журнала действий, как описано в действиях Перед началом и Отключение поиска в журнале [аудита](/office365/securitycompliance/turn-audit-log-search-on-or-off#turn-on-audit-log-search). Если ведение журнала аудита уже есть, для приложения "Пациенты" ничего специального не требуется. Каждый раз, когда поставщик медицинских услуг устанавливает и запускает приложение в команде, журналы аудита регистрирует их действия с помощью PHI.
4. После этого администратору потребуется сообщить о доступности приложения "Пациенты", а сотрудникам сферы здравоохранения придется начать создавать действия, которые будут включены в аудит.

## <a name="logged-activities-for-patients-app"></a>Зарегистрированные действия для приложения "Пациенты"

В приложении "Пациенты" есть собственные зарегистрированные действия, перечисленные в следующей таблице:

|Удобное имя | Операция | Описание|
|:---|:---|:---|
| Просмотр списка пациентов | PatientListView | Пользователь просматривал список пациентов.|
| Список удаленных пациентов | PatientListDelete | Пользователь удалил список пациентов.|
| Добавлен пациент в список | PatientListAddPatient | Пациент был добавлен в список пациентов. |
| Добавлена заметка для пациента | PatientNoteAdd | В запись пациента добавлена заметка. |
| Создана схема пациентов | PatientSchemaCreate | Был создан набор столбцов, используемых в записи пациента. |
| Пользователь инициировал экспорт | ExportInitiation | Данные пациентов были экспортироваться из приложения "Пациенты" в Excel файл. Файл будет сохранен на сайте группы SharePoint. |
| Создан список пациентов | PatientListCreate | Пользователь создал список пациентов.|
| Настройка списка пациентов по умолчанию| PatientListDefaultSet| Пользователь настроит определенный список в качестве списка по умолчанию.|
| Удален пациент из списка| PatientListRemovePatient | Пациент был удален из списка пациентов. |
| Ищите пациента | PatientSearch | Поиск записи пациента в службе EHR. |
| Обновленная схема пациентов | PatientSchemaUpdate  | Обновлен набор столбцов, используемых в записи пациента. |<!-- | Перемещен пациент в другой список| PatientMoved | Запись пациента была перемещена из одного списка в другой. |-->
| Переименован список пациентов | PatientListRename | Список пациентов был переименован. |
| Измененные столбцы в списке пациентов | PatientListEditColumns | Столбец в списке пациентов был изменен (добавлен или удален). |
| Просмотр сведений о пациенте | PatientView | Пользователь просматривал запись пациента.|
| Измененные сведения о пациенте | PatientDetailsEdit | Подробные данные в записи пациентов были изменены. |
| Настройка подключения eHR | Набор EHRConnectionSet | Установите URL-адрес, используемый для подключения к службе EHR FHIR. Пример: https://<span>api-v8-dstu2.hspconsortium.org/ContosoHospital/open</span>  |

При необходимости вы можете настроить аудит, чтобы найти или отфильтровать любые из этих зарегистрированных действий.
