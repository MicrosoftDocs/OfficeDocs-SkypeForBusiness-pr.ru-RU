# <a name="connect-the-patients-app-to-azure-api-for-fhir"></a>Подключение приложения для пациентов к Azure API для FHIR

> [!NOTE]
> С 30 октября 2020 г. приложение "Пациенты" устарело и заменено приложением [Lists](https://support.microsoft.com/office/get-started-with-lists-in-teams-c971e46b-b36c-491b-9c35-efeddd0297db) в Teams. Данные приложения "Пациенты" хранятся в почтовом ящике группы Office 365, на которой основана команда. Все данные, связанные с приложением "Пациенты", сохраняются в этой группе, но больше недоступны в пользовательском интерфейсе. Пользователи могут повторно создать свои списки с помощью приложения [Lists](https://support.microsoft.com/office/get-started-with-lists-in-teams-c971e46b-b36c-491b-9c35-efeddd0297db).
>
>С помощью приложения Lists медицинские бригады вашей организации здравоохранения могут создавать списки пациентов для различных сценариев: от обходов и собраний междисциплинарных команд до общего наблюдения за пациентами. Ознакомьтесь с шаблоном "Пациенты" в приложении Lists, чтобы начать работу. Дополнительные сведения об управлении приложением Lists в организации см. в статье [Управление приложением Lists](../../manage-lists-app.md).

Чтобы разрешить приложению "Пациенты" в Microsoft Teams доступ к экземпляру Azure API для FHIR, выполните указанные здесь действия. В этой статье предполагается, что в вашем клиенте настроен экземпляр Azure API для [FHIR.](https://azure.microsoft.com/services/azure-api-for-fhir/)  Если вы еще не создали экземпляр Azure API для FHIR в своем клиенте, см. краткое краткое видео: развертывание Azure API для [FHIR с помощью портала Azure.](/azure/healthcare-apis/fhir-paas-portal-quickstart)

1. Щелкните [здесь,](https://login.microsoftonline.com/common/adminConsent?client_id=4aee3506-b263-43e0-ba31-1468fa7b2806) чтобы предоставить администратору разрешение для приложения "Пациенты". При запросе войдите с учетными данными администратора  клиента или глобального администратора, а затем нажмите кнопку "Принять", чтобы предоставить необходимые разрешения.

    ![Снимок экрана: запрос разрешений для приложения "Пациенты"](../../media/patients-app-permissions-request.png)

    После того как вы примете приглашение, закроем окно. Вы увидите страницу, которая может выглядеть так: Сообщение об ошибке можно игнорировать на странице. Это не означает, что вы получили согласие. (Мы работаем над более удобной страницей для этого URL-адреса. Следите за обновлениями!)

    ![Снимок экрана: запрос разрешений для приложения "Пациенты"](../../media/patients-app-permissions-request-granted.png)

2. Войте на портал [Azure с](https://portal.azure.com) учетными данными администратора.

3. В области навигации слева выберите **Azure Active Directory,** а затем — **"Корпоративные приложения".**

    Найди строку "Пациенты" **и** скопируйте  значение в столбце "ИД объекта" в буфер обмена.

    ![Снимок экрана: строка "Пациенты" на портале Azure](../../media/patients-app-azure-portal-object-id.png)

4. Перейдите к экземпляру ресурса Azure API для FHIR, к которому вы хотите подключить приложение "Пациенты" (посредством поиска или просмотра ресурсов), а затем откройте параметры этого экземпляра.

    ![Снимок экрана: параметры экземпляра Azure API для FHIR на портале Azure](../../media/patients-app-azure-portal-instance-settings.png)

5. Нажмите **кнопку "Проверка** подлинности" и в поле "Разрешенные  ИД объектов" в поле "Разрешенные" в поле "ИД объекта", скопированные на шаге 3. Это позволит приложению "Пациенты" получать доступ к серверу FHIR. После вметки ИД объекта Azure Active Directory проверит его, и рядом с его иным объектом появится зеленая метка.

    ![Снимок экрана: параметры проверки подлинности на портале Azure](../../media/patients-app-azure-portal-authentication.png)

6. Нажмите кнопку **Сохранить**. При этом экземпляр будет раздвоен, что может занять несколько минут.

7. Щелкните **"Обзор"** и скопируйте URL-адрес с конечной точки **метаданных FHIR.** Удалите тег метаданных, чтобы получить URL-адрес сервера FHIR. Например, `https://test02-teamshealth.azurehealthcareapis.com/` .

    ![конечная точка метаданных на портале Azure](../../media/patients-app-azure-portal-metadata-endpoint.png)

8. В Teams перейдите в экземпляр приложения "Пациенты", загруженный в группу,  щелкните "Параметры", а затем в поле "Ссылка" введите URL-адрес конечной точки сервера FHIR. Затем нажмите **кнопку** "Подключиться", чтобы установить подключение, а затем найдите и добавьте пациентов в список.  

    ![ Параметры приложения "Пациенты" в Teams](../../media/patients-app-teams.png)

    Если на этом этапе вы получаете сообщение об ошибке при подключении к Teams, отправьте подробный снимок экрана, журналы из [Fiddler](https://www.telerik.com/download/fiddler) и другие действия повторного действия в сообщении электронной почты с строкой темы "Приложение для пациентов — устранение неполадок в режиме EMR" в [teamsforhealthcare@service.microsoft.com.](mailto:teamsforhealthcare@service.microsoft.com)
