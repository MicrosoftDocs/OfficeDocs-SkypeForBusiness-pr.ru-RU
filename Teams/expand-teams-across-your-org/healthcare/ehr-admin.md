# <a name="virtual-visits-with-teams---integration-into-ehr"></a>Виртуальные посещения с помощью Teams — интеграция с EHR

Соединителей электронной медицинских записей Microsoft Teams (EHR) можно легко запустить виртуальное посещение пациентов или обратиться к другому поставщику в Teams непосредственно из системы EHR. Microsoft Teams, встроенный в облако Microsoft 365, обеспечивает простую и безопасную совместную работу и общение с чатом, видеосвязь, голосовой связью и медицинскими инструментами в едином центре, который поддерживает соответствие с сертификацией HIPAA, HITECH и другими программами.
Платформа для коммуникаций и совместной работы Teams позволяет клиникам легко перегромождать негроможденные системы и тратить время на предоставление наилучшей помощи. Соединитер электронной записи здравоохранения Microsoft Teams (EHR) может:
- Запускать виртуальные визиты Teams как с порталов поставщика услуг, так и с порталов пациентов.
- Записывая запись в метаданные EHR при подключении и отключении событий, чтобы включить автоматическое ведение аудита и сохраняемую запись.
- Интегрируются в существующие рабочий процессы клиники и пациентов, позволяя им использовать Microsoft Teams.

  Посмотрите видео о том, как управлять виртуальными посещениями с портала EHR.

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4HAtn]

## <a name="before-you-begin"></a>До начала работы

Перед интеграцией соединитела EHR необходимо убедиться в том, что у вас есть следующие необходимые условия:

- Access to use to use to the Microsoft Teams app in [Market's App Marketplace.](https://apporchard.epic.com/Gallery?id=6153)

- Активная подписка на Microsoft Cloud для здравоохранения или подписка на автономные предложения Microsoft Teams EHR Connector (которые применяются только во время тестирования в производственной сфере).

- У пользователей должна быть соответствующая лицензия Microsoft 365 или Office 365, которая включает собрания Microsoft Teams.

- Microsoft Teams следует принятия и использования в организации.

- У организаций должна быть версия с ноябрь 2018 г. или более поздней.

- Ваши системы должны соответствовать всем требованиям [к программному обеспечению и браузеру.](https://docs.microsoft.com/microsoftteams/hardware-requirements-for-the-teams-app)

Вам также потребуется информация от следующих людей в вашей организации:

- Администратор Microsoft 365

- Веха аналитика клиентов

> [!Note]
> Запросите своего технических специалиста по Epic-Microsoft интеграции с телетайлами Teams, доступное в Marketplace.

### <a name="launch-the-ehr-connector-configuration-portal"></a>[Запуск портала настройки соединителя EHR](#launch-the-ehr-connector-configuration-portal)

Настройка организации здравоохранения для запуска виртуальных посещений с помощью Microsoft Teams начинается с запуска портала конфигурации EHR Connector. Для тестирования интеграции необходимо настроить одну или несколько организаций. Настройте тестовый и производственный URL-адрес на портале конфигурации. Протестировать интеграцию из тестовой среды Сергея перед переходом на производственную среду.
  
- URL-адрес конфигурации соединителя EHR: [https://ehrconnector.teams.microsoft.com](https://ehrconnector.teams.microsoft.com)

Администратор Microsoft 365 и клиентский аналитик Из вашей организации должны выполнить действия по настройке и сведениям на портале конфигурации. Обратитесь к ресурсу технических специалистов, назначенного вашей организации.

### <a name="configuration-information"></a>[Сведения о конфигурации](#configuration-information)

Это этап должен завершить администратор **Microsoft 365.** Администратор Microsoft 365 должен запустить портал конфигурации соединителя и войти с учетными данными Майкрософт, чтобы начать настройку.

Для выполнения этого действия администратор Microsoft 365 должен получить действительный базовый URL-адрес службы FHIR от вашего специалиста по технической подготовке Ит и имя пользователя клиентского аналитика, который будет утверждать конфигурацию. Администратор Microsoft 365 должен запустить страницу конфигурации соединителя и войти с учетными данными Майкрософт, чтобы начать настройку.

- Базовый URL-адрес FHIR — это статический адрес, соответствующий конечной точке API FHIR сервера. Пример `https://lamnahealthcare.org/fihr/auth/connect-ocurprd-oauth/api/FHDST` URL-адреса:

- Имя оголившего конфигурации — это имя клиентского аналитика Висячего, который будет отвечать за утверждения конфигурации на следующем этапе. Клиентский аналитик Висяка — это человек в вашей организации, у которого есть доступ для регистрации к Ним.

  ![Имя утверждения конфигурации выбирается из списка в соединитеке EHR.](../../media/teams-ehr-connector.png)

### <a name="approve-or-view-configuration"></a>[Утверждение или просмотр конфигурации](#approve-or-view-configuration)

Клиентский аналитик вашей организации, добавленный в качестве утверждения, должен использовать тот же URL-адрес соединитора EHR, что и при предыдущем шаге, для входа с учетными данными Microsoft 365. После успешной проверки проверяемого попросят войти с учетными данными "Сергей" для проверки организации.

> [!Note]
> Администратор Microsoft 365 и клиентский аналитик в вашей организации Может быть один и тот же человек. В этом случае добавьте свое имя пользователя в качестве утвержденного. Для проверки доступа вам все равно потребуется войти в Учетную подпись. Он используется только для проверки базового URL-адреса СЛФ. Корпорация Майкрософт не будет хранить учетные данные или получать доступ к данным EHR с помощью этого входа.

  ![Проверьте и утвердить конфигурацию учетных данных.](../../media/approve-view-configuration.png)

После успешного войти в Процесс аналитик Клиента Должен **утвердить** конфигурацию. Если конфигурация неправилена, администратор Microsoft 365 сможет изменить исходные конфигурации, повторно вовшись на портал соединители microsoft EHR. 

![Подтвердив настройку соединитела EHR и параметр для изменения конфигурации.](../../media/ehc-approve-3.png)

### <a name="review-and-finish-the-configuration"></a>[Просмотр и завершить настройку](#review-and-finish-the-configuration)

Эти записи необходимы для завершения настройки виртуального посещения в Будет. Дополнительные сведения можно найти в Epic-Microsoft по интеграции с телетайлами Teams.

## <a name="launch-teams-virtual-visits"></a>Запуск виртуальных посещений Teams

После выполнения действий по подключению eHR и настройке Учетной записи Майкрософт ваша организация готова поддерживать видеосегменты с помощью Microsoft Teams.

### <a name="virtual-visit-prerequisites"></a>Предварительные условия виртуального посещения

- Ваши системы должны соответствовать всем требованиям [к программному обеспечению и браузеру.](https://docs.microsoft.com/microsoftteams/hardware-requirements-for-the-teams-app)

- Организации здравоохранения должны завершить настройку между организацией "Организация Скайла" и Microsoft 365.

### <a name="provider-experience"></a>Опыт работы с поставщиком

Поставщики здравоохранения из вашей организации также могут присоединяться к виртуальным посещениям Microsoft Teams из приложений поставщика услуг-поставщиков услуг Майкрософт (Hyperspace, Haiku, Canto). Кнопка **"Начать виртуальное** посещение" внедрена в поток поставщика услуг.

Основные функции работы поставщика:

- Поставщики могут присоединяться к виртуальным посещениям с помощью поддерживаемых браузеров или приложения Microsoft Teams.

- Поставщики должны войти в свою учетную запись Microsoft 365 один раз при первом присоединении к виртуальному посещению.

- После одно разового входов поставщик будет сразу перенаправься на виртуальную встречу в Microsoft Teams. (Поставщик должен быть вписан в Microsoft Teams.

- Поставщик может в режиме реального времени видеть обновления участников, подключаться к назначенной встрече и отключать ее. Поставщик может видеть, когда пациент подключен к виртуальному визиту.

  ![Виртуальный визит пациента с поставщиком услуг](../../media/ehc-provider-experience-6.png)

### <a name="patient-experience"></a>Patient experience

Соединитестор поддерживает пациентов, которые присоединяются к виртуальным посещениям через MyChart Web и мобильные устройства. Во время встречи пациенты могут начать виртуальный визит из myChart с помощью кнопки "Начать **виртуальный визит".**

Основные функции пациентов:

- Пациенты могут присоединиться к виртуальным посещениям современных веб-браузеров на компьютере и мобильном устройстве без установки приложения.

- Пациенты могут присоединиться к виртуальным посещениям одним щелчком мыши, для этого не требуется другая учетная запись или вход в учетную запись.

- Для запуска виртуального посещения пациенту не требуется создавать учетную запись Майкрософт или войди в нее.

- Пациенты будут помещаются в "вестибюль", пока поставщик услуг здравоохранения не присоединится к встрече и не допустит их на виртуальный визит.

- Тестирование видео и микрофона доступно в "вестибюле" перед присоединением к виртуальному посещению.

  ![Patient experience of the virtual visit](../../media/ehc-virtual-visit-5.png)

> [!Note]
> Также, MyChart, Haiku и Canto являются товарными знаками корпорации Systems.

### <a name="privacy-and-location-of-data"></a>Конфиденциальность и расположение данных

Интеграция Teams с системами EHR оптимизирует объем данных, которые используются и хранятся во время интеграции и потоков виртуального посещения. Решение должно следовать общим принципам конфиденциальности и управления данными Teams, а также рекомендациям, описанным в этой области.

Соединиттель EHR для Microsoft Teams не хранит и не передает из системы EHR личные данные и медицинские записи пациентов и медицинских учреждений. Единственными данными, которые хранятся соединитетелем EHR, является уникальный ИД пользователя EHR, который используется при настройке собрания Teams. Уникальный ИД пользователя EHR хранится в одном из трех географических регионов, описанных в месте хранения данных клиента [Microsoft 365.](https://docs.microsoft.com/microsoft-365/enterprise/o365-data-locations?view=o365-worldwide#data-center-geographies) Все данные чата, записи и другие данные, вносяые участниками собрания в Teams, хранятся в соответствии с существующими политиками хранения. Если вы хотите узнать больше о расположении данных в Microsoft Teams, посетите [веб-сайт с расположениями данных в Teams.](https://docs.microsoft.com/microsoftteams/location-of-data-in-teams)
