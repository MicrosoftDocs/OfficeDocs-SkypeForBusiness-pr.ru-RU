# <a name="virtual-visits-with-teams---integration-into-ehr"></a>Виртуальные посещения с помощью Teams — интеграция с EHR

Соединителей электронной медицинских записей Microsoft Teams (EHR) можно легко запустить виртуальное посещение пациентов или обратиться к другому поставщику в Teams непосредственно из системы EHR. Microsoft Teams, встроенный в облако Microsoft 365, обеспечивает простую и безопасную совместную работу и общение с чатом, видеосвязь, голосовой связью и медицинскими инструментами в едином центре, который поддерживает соответствие с сертификацией HIPAA, HITECH и другими программами.

Платформа для коммуникаций и совместной работы Teams позволяет клиникам легко перегромождать негроможденные системы и тратить время на предоставление наилучшей помощи. Соединитер электронной записи здравоохранения Microsoft Teams (EHR) может:

- Запускать виртуальные визиты Teams как с порталов поставщика услуг, так и с порталов пациентов.

- Записывая запись в метаданные EHR при подключении и отключении событий, чтобы включить автоматическое ведение аудита и сохраняемую запись.

- Интегрируются в существующие рабочий процессы клиники и пациентов, позволяя им использовать Microsoft Teams.

  Посмотрите видео о том, как управлять виртуальными посещениями с портала EHR.

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4HAtn]

## <a name="before-you-begin"></a>До начала работы

Перед интеграцией соединитела EHR необходимо убедиться в том, что у вас есть следующие необходимые условия:

- Активная подписка на Microsoft Cloud для здравоохранения или подписка на автономный продукт Microsoft Teams EHR Connector.

- У пользователей должна быть соответствующая лицензия Microsoft 365 или Office 365, которая включает собрания Microsoft Teams.

- Microsoft Teams следует принятия и использования в организации.

- У организаций должна быть версия с ноябрь 2018 г. или более поздней.

- Ваши системы должны соответствовать всем требованиям [к программному обеспечению и браузеру.](https://docs.microsoft.com/microsoftteams/hardware-requirements-for-the-teams-app)

Вам также потребуется информация от следующих людей в вашей организации:

- Администратор Microsoft 365

- Администратор

> [!Note]
> Запросите у администратора Вашей учетной Epic-Microsoft интеграции с Telehealth Teams, доступное в Marketplace.

### <a name="launch-the-ehr-connector-configuration-portal"></a>[Запуск портала настройки соединителя EHR](#launch-the-ehr-connector-configuration-portal)

Настройка организации здравоохранения для запуска виртуальных посещений с помощью Microsoft Teams начинается с запуска портала конфигурации EHR Connector. Используйте тестовый URL-адрес для настройки соединителя в тестовой среде Test Environment. Используйте производственный URL-адрес, когда вы будете готовы включить свою среду Production.
  
- Тестовая среда [https://ehrconnector-ppe.teams.microsoft.com](https://ehrconnector-ppe.teams.microsoft.com)
- Производственная среда [https://ehrconnector.teams.microsoft.com](https://ehrconnector.teams.microsoft.com)

Администратор Microsoft 365 и Администратор Microsoft 365 в вашей организации должны выполнить действия по настройке и сведениям на портале конфигурации. Обратитесь к ресурсу "Переупоход", назначенного вашей организации.

### <a name="configure-provider-organization-information"></a>[Настройка сведений о компании-поставщике](#configure-provider-organization-information)

Это этап должен быть выполнен администратором Microsoft 365. Администратор Microsoft 365 должен запустить портал конфигурации соединителя и войти с учетными данными Майкрософт, чтобы начать настройку.

Для выполнения этого действия администратор Microsoft 365 должен получить действительный базовый URL-адрес FHIR от администратора Microsoft 365 и имя пользователя, который будет утверждать конфигурацию. Администратор Microsoft 365 должен запустить страницу конфигурации соединителя и войти с учетными данными Майкрософт, чтобы начать настройку.

- Базовый URL-адрес FHIR — это статический адрес, соответствующий конечной точке API FHIR сервера. Пример `https://lamnahealthcare.org/fihr/auth/connect-ocurprd-oauth/api/FHDST` URL-адреса:

- Имя управляющего конфигурацией — это имя системного администратора ,который отвечает за утверждения конфигурации.

  ![Имя утверждения конфигурации выбирается из списка в соединитеке EHR.](../../media/ehc-provider-1.png)

### <a name="verify-and-approve-the-configuration"></a>[Проверка и утверждение конфигурации](#verify-and-approve-the-configuration)

Администратор Сайпа в вашей организации здравоохранения, добавленный в качестве утвержденного администратора, должен использовать для входа в службу с учетными данными Microsoft 365 тот же URL-адрес соединитора EHR, который использовался на предыдущем этапе. После успешной проверки проверяемого попросят войти с учетными данными "Сергей" для проверки организации.

> [!Note]
> АдминистраторАми Microsoft 365 и Admin 365 в организациях могут быть один и тот же человек. В этом случае на первом этапе добавьте свое имя пользователя в качестве утвержденного. Для проверки доступа вам все равно потребуется войти в Учетную подпись. Он используется только для проверки базового URL-адреса СЛФ. Корпорация Майкрософт не будет хранить учетные данные или получать доступ к данным EHR при этом входе.

  ![Проверьте и утвердить конфигурацию учетных данных.](../../media/ehc-provider-2.png)

После успешного входе в Агом администратор **должен** утвердить конфигурацию. Если конфигурация неправилена, администратор Microsoft 365 сможет изменить исходные конфигурации, повторно вовшись на портал Microsoft EHR Connector.

![Подтвердив настройку соединитела EHR и параметр для изменения конфигурации.](../../media/ehc-approve-3.png)

## <a name="launch-teams-virtual-visits"></a>Запуск виртуальных посещений Teams

После выполнения действий по подключению eHR и настройке Учетной записи Майкрософт ваша организация готова поддерживать видеосегменты с помощью Microsoft Teams.

### <a name="virtual-visit-prerequisites"></a>Предварительные условия виртуального посещения

- Ваши системы должны соответствовать всем требованиям [к программному обеспечению и браузеру.](https://docs.microsoft.com/microsoftteams/hardware-requirements-for-the-teams-app)

- Организации здравоохранения должны завершить настройку между организацией "Организация Скайла" и Microsoft 365.

### <a name="provider-experience"></a>Опыт работы с поставщиком

Поставщики здравоохранения из вашей организации также могут присоединяться к виртуальным посещениям Microsoft Teams из приложений поставщика услуг-поставщиков услуг Майкрософт (Hyperspace, Haiku, Canto). Кнопка **"Начать виртуальное** посещение" внедрена в поток поставщика услуг.

Основные функции работы поставщика:

- Поставщики могут присоединяться к виртуальным посещениям с помощью поддерживаемых браузеров или приложения Microsoft Teams.

- Поставщики должны один раз войти в свою учетную запись Microsoft 365 при первом присоединении к виртуальному посещению.

- После одновского входов поставщик будет сразу перенаправься на виртуальную встречу в Microsoft Teams. (Поставщик должен быть вписан в Microsoft Teams.

- Поставщик может в режиме реального времени видеть обновления участников, подключаться к назначенной встрече и отключать ее. Поставщик может видеть, когда пациент подключен к виртуальному посещению.

  ![Впечатления от виртуального визита с пациентом](../../media/ehc-provider-experience-6.png)

### <a name="patient-experience"></a>Patient experience

Соединитестор поддерживает пациентов, которые присоединяются к виртуальным посещениям с помощью Веб-приложения MyChart и мобильных устройств. Во время встречи пациенты могут начать виртуальный визит из myChart с помощью кнопки "Начать **виртуальный визит".**

Основные функции пациентов:

- Пациенты могут присоединиться к виртуальным посещениям современных веб-браузеров на компьютере и мобильном устройстве без установки приложения.

- Пациенты могут присоединиться к виртуальным посещениям одним щелчком мыши, а дополнительная учетная запись не требуется или вход не требуется.

- Для запуска виртуального посещения пациенту не требуется создавать учетную запись Майкрософт или войди в нее.

- Пациенты будут помещаются в "вестибюль", пока поставщик услуг здравоохранения не присоединится к встрече и не допустит их на виртуальный визит.

- Тестирование видео и микрофона доступно в "вестибюле" перед присоединением к виртуальному посещению.

  ![Patient experience of the virtual visit](../../media/ehc-virtual-visit-5.png)

### <a name="privacy-and-location-of-data"></a>Конфиденциальность и расположение данных

Интеграция Teams с системами EHR оптимизирует объем данных, которые используются и хранятся во время интеграции и потоков виртуального посещения. Решение должно следовать общим принципам конфиденциальности и управления данными Teams, а также рекомендациям, описанным в этой области.

Соединиттель EHR для Microsoft Teams не хранит и не передает из системы EHR личные данные и медицинские записи пациентов и медицинских учреждений. Единственными данными, которые хранятся соединитетелем EHR, является уникальный ИД пользователя EHR, который используется при настройке собрания Teams. Уникальный ИД пользователя EHR хранится в одном из трех географических регионов, описанных в статье о месте хранения данных клиента [Microsoft 365.](https://docs.microsoft.com/microsoft-365/enterprise/o365-data-locations?view=o365-worldwide#data-center-geographies) Все данные чата, записи и другие данные, вносяые участниками собрания в Teams, хранятся в соответствии с существующими политиками хранения. Если вы хотите узнать больше о расположении данных в Microsoft Teams, посетите [веб-сайт с расположениями данных в Teams.](https://docs.microsoft.com/microsoftteams/location-of-data-in-teams)
