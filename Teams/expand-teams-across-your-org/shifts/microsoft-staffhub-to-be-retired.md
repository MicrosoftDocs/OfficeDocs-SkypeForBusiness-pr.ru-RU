# <a name="microsoft-staffhub-to-be-retired"></a>Прекращение поддержки Microsoft StaffHub

Действующий 30 июня 2020 г. Корпорация Microsoft StaffHub будет прекращена. Мы создаем возможности StaffHub в Microsoft Teams. Сегодня команды включают в себя приложение смен для управления планированием и дополнительные возможности, которые будут вычислены с течением времени.

Эти изменения являются частью нашей продолжающейся работы, чтобы предоставить каждому сотруднику Microsoft 365. Благодаря возможностям для Firstlineных сотрудников в Teams у каждого сотрудника в вашей организации будет возможность использовать Teams для упрощения рабочего дня, совместной работы с коллегами и получения информации и опыта для обеспечения лучшей работы.

## <a name="frequently-asked-questions"></a>Вопросы и ответы

### <a name="about-staffhub-retirement"></a>О StaffHub выбытиях

#### <a name="when-will-staffhub-be-retired"></a>Когда StaffHub будет прекращена?

Начиная с апреля 2019, Microsoft StaffHub больше не будет доступен для новых клиентов, и вы больше не сможете получить доступ к приложению из средства запуска приложений Microsoft 365 или Office 365 в существующих клиентах. Текущие пользователи смогут использовать эту службу до 31 декабря 2019, однако никакие дополнительные функции будут недоступны.

#### <a name="what-will-happen-once-staffhub-is-retired"></a>Что произойдет после прекращения StaffHubа?

Microsoft StaffHub перестанет работать для всех пользователей 30 июня 2020 г. Каждый, кто пытается открыть приложение, будет показывать сообщение, направленное на загрузку групп. Если у пользователя есть активная лицензия, включающая в себя команды и предпринимала соответствующие инструкции для включения групп, их данные и основные функции StaffHub будут доступны для них при переходе в Teams.

С 2019 апреля и июнь 2020 пользователи StaffHub смогут получать уведомления о приложении, чтобы они могли использовать команды.

#### <a name="will-teams-offer-all-the-functionality-currently-offered-in-staffhub"></a>Будут ли доступны все возможности, которые в настоящее время предлагаются в StaffHub?

В январе 2019 Teams начали предлагать смены, которые ранее были доступны в Microsoft StaffHub. Эта функция позволяет руководителям планировать и распределять сменные расписания и членам групп просматривать их расписание и вносить запросы на изменение в расписании с мобильных устройств. Дополнительные сведения можно найти [в разделе Управление приложением смен для Организации в Teams](manage-the-shifts-app-for-your-organization-in-teams.md).

Мы стремимся приступить к работе с более широкими возможностями пользователей и StaffHub в Teams уникальные возможности. Дополнительные сведения о функциях, которые будут доступны в Teams, можно найти в разделе [функции рабочих процессов Firstline, поступающие в Teams](firstline-worker-features-coming-to-teams.md).

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Что нужно сделать, чтобы подготовиться к этому изменению?

Чтобы получить сведения о том, как подготовиться к переходу с StaffHub на смену в Teams, перейдите к разделу [Подготовка к перемещению StaffHubных групп на смену](move-staffhub-teams-to-shifts-in-teams.md#prepare).

#### <a name="why-cant-users-in-my-organization-access-their-staffhub-team-from-shifts-in-teams"></a>Почему пользователи в Организации не могут получить доступ к своей команде StaffHub из смен в Teams?

Команды, созданные с помощью StaffHub, можно просматривать и управлять только в StaffHub. После перемещения группы StaffHub в Teams пользователи смогут просматривать их расписания в сменах. Мы рекомендуем вам своевременно сообщать эти изменения и ознакомиться с существующими пользователями с помощью Teams.

### <a name="about-shifts"></a>Смена

#### <a name="what-is-shifts"></a>Что такое смена?

Приложение смены в Teams — это средство управления планированием, которое обеспечивает подключение Firstlineных сотрудников и синхронизацию. Менеджеры создают, обновляют и управляют расписаниями смен для групп, а участники групп могут просматривать их расписания, видеть, кто в данный момент запланирован на этот день, отвечать на запросы на замену или предлагать смену, а также запрашивать время отключения с мобильных устройств.  

#### <a name="which-plans-is-shifts-available-in"></a>В каких планах доступны сменные планы?

Смены доступны во всех SKU предприятий, где доступны команды.

#### <a name="how-do-i-get-shifts"></a>Как получить смену? 

Если вы уже подключены к плану, включающему смену, вы можете использовать смены прямо сейчас в рамках Teams. Ознакомьтесь со [статьями "Начало работы" в сменах](https://support.office.com/article/get-started-in-shifts-5f3e30d8-1821-4904-be26-c3cd25a497d6).

#### <a name="which-devices-or-platforms-is-shifts-available-on"></a>На каких устройствах и платформах доступны смены?

Смена доступна в веб-клиенте Teams, клиентском компьютере Teams и клиентах Teams для мобильных устройств (iOS и Android).
 
#### <a name="are-there-differences-between-the-shifts-desktop-and-web-apps-and-the-mobile-ios-and-android-apps"></a>Существуют ли различия между классической версией и веб-приложениями и приложениями для мобильных устройств (iOS и Android)?

Для руководителей классическое приложение и веб-приложения оптимизируются для создания групп, добавления участников, построения и публикации расписаний и предоставления общего доступа к содержимому участникам команды. Руководители обычно используют мобильное приложение для утверждения запросов, отправки мгновенных сообщений группе, вызова участника команды и совместной работы в пути. 

Для участников команды приложение для мобильных устройств оптимизировано для просмотра своего расписания, просмотра сведений о том, с кем работают каждый день, часы и время ухода, Настройка доступности, Загрузка и возможностях запросов, а также переключения и открытия смен. Существуют различия в том, что пользователи могут выполнять в классическом и веб-приложениях, и в мобильных приложениях. Мы работаем над контролем четности функций.

#### <a name="are-there-differences-between-the-shifts-ios-app-and-the-android-app"></a>Существуют ли различия между приложением для iOS и приложением Android?

Основные функциональные возможности одинаковы для приложений для iOS и Android. Существуют небольшие различия пользовательского интерфейса между двумя приложениями, которые нужно выровнять с рекомендованными шаблонами разработки.

#### <a name="what-browsers-does-shifts-work-on"></a>В каких браузерах работают смены?

Смена действует для всех браузеров, которые поддерживаются в Teams, включая последние версии Microsoft EDGE, Internet Explorer, Google Chrome и Mozilla Firefox. Дополнительные сведения можно найти в разделе [получение клиентов для Teams](../../get-clients.md).

#### <a name="what-languages-does-shifts-support"></a>На каких языках поддерживается Смена языков?

Смена доступна на следующих языках: Китайский (упрощенное письмо), английский, китайский (США), испанский, Русский, японский, немецкий, португальский (Бразилия), французский, корейский, итальянский, португальский (традиционный, нидерландский, шведский, норвежский). Мы планируем добавить поддержку дополнительных языков в будущем.  

#### <a name="where-is-shifts-data-stored"></a>Куда перемещаются данные?

Сменные данные хранятся в Azure в центрах обработки данных в Северной Америке и Западной Европе. В контенте SharePoint используются группы Microsoft 365, а также хранение и поддержка.

Дополнительные сведения можно найти в разделе [Расположение смен данных](manage-the-shifts-app-for-your-organization-in-teams.md#location-of-shifts-data).

#### <a name="where-can-i-learn-more-about-shifts"></a>Где можно узнать больше о сменах?

Сведения о продукте можно найти по адресу [https://products.office.com/microsoft-teams/staff-scheduling-software](https://products.office.com/microsoft-teams/staff-scheduling-software) .

Инструкции для администраторов в разделе [Управление приложением смен для Организации в Teams](manage-the-shifts-app-for-your-organization-in-teams.md)

Для получения справки для конечных пользователей переходите к разделу " [Смена Firstline работников](https://support.office.com/article/apps-and-services-cc1fba57-9900-4634-8306-2360a40c665b)".

### <a name="switch-from-staffhub-to-shifts-in-teams"></a>Переход с StaffHub на смену в Teams

#### <a name="what-licenses-do-i-need-for-teams"></a>Какие лицензии нужны для Teams?

Каждый пользователь должен иметь действующую лицензию Microsoft 365 или Office 365 из [подходящего плана](#which-plans-is-shifts-available-in) , и ему должна быть назначена лицензия Teams. Дополнительные сведения можно найти в разделе [Перемещение StaffHub Teams to Shifts в Teams](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="how-do-i-move-my-organizations-staffhub-teams-to-teams"></a>Как переместить группы StaffHub Организации в Teams?

Подробное руководство по переходу с StaffHub в Teams можно найти в [статье перевод Microsoft StaffHub Teams для смены в Teams](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="how-do-i-enable-teams-in-my-organization"></a>Как включить команды в моей организации?

По умолчанию группы включены для всех организаций. Вы назначаете пользователям лицензии на управление индивидуальными участниками группы. Дополнительные сведения можно найти [в разделе Включение поддержки групп в Организации](../../Office-365-set-up.md) и [Управление доступом пользователей к Teams](../../user-access.md).

#### <a name="what-if-i-currently-have-skype-for-business-enabled-in-my-organization-but-not-teams"></a>Что делать, если в настоящее время Skype для бизнеса включен в моей организации, но не в Teams?

Teams поддерживает сосуществование с помощью Skype для бизнеса. Дополнительные сведения можно найти в статье [Знакомство с сосуществованием групп и Skype для бизнеса](../../teams-and-skypeforbusiness-coexistence-and-interoperability.md) , а также руководство по взаимодействию и взаимодействию между [организациями с помощью групп в Skype для бизнеса](../../migration-interop-guidance-for-teams-with-skype.md).

#### <a name="does-each-user-need-to-have-a-microsoft-365-or-office-365-account-or-azure-active-directory-azure-ad-account"></a>Требуется ли каждому пользователю иметь учетную запись Microsoft 365 или Office 365 или учетную запись Azure Active Directory (Azure AD)?

Да, каждый руководитель и участник группы, работающие с группами и сменами в Teams, должны иметь учетную запись Microsoft 365 или Office 365 (Azure AD), которой необходимо назначить действительную лицензию.
 
#### <a name="in-staffhub-i-could-add-a-special-prefix-to-all-microsoft-365-groups-that-were-created-by-staffhub-teams-can-i-do-the-same-in-teams"></a>В StaffHub я могу добавить особый префикс ко всем группам Microsoft 365, созданным группами StaffHub Teams. Можно ли сделать то же самое в Teams?

При переходе с StaffHub на Teams вы не будете иметь такой же возможности. Если необходимо применить согласованность имен ко всем группам Microsoft 365, вы можете использовать [политику именования групп microsoft 365](https://docs.microsoft.com/office365/admin/create-groups/groups-naming-policy).

#### <a name="will-team-names-be-the-same-in-teams"></a>Будут ли названия групп одинаковыми в Teams?

Возможно, имя группы будет другим. Имена групп в Teams используют имя группы Microsoft 365. Если название группы StaffHub отличается от имени соответствующей группы Microsoft 365, вы увидите другое имя при переходе на Teams.

#### <a name="what-happens-to-team-membership-when-a-staffhub-team-is-moved-to-teams"></a>Что происходит с членством в группе, когда группа StaffHub перемещается в Teams?

Участие в группах сохраняется при перемещении группы StaffHub в Teams. Номера телефонов участников группы не отображаются в сменах. Если номер телефона StaffHub совпадает с номером телефона в Azure AD, он будет храниться в Azure AD для этой учетной записи пользователя. 

Мы рекомендуем вам просмотреть список существующих команд в StaffHub и по возможности выравнивать номера телефонов для соответствия с удостоверениями Azure AD.
 
#### <a name="what-happens-to-staffhub-team-members-who-arent-provisioned-or-whose-status-is-inactive"></a>Что происходит с неподготовленными участниками группы StaffHub или состоянием, чей статус неактивен?

Расписание пользователя будет перемещено на смену, но пользователь не будет членом группы или Microsoft 365.

Для неактивных учетных записей, не связанных с учетной записью пользователя Azure AD, руководитель может убедиться, что пользователь добавлен в группу в качестве участника, добавив допустимый адрес электронной почты Microsoft 365 или Office 365 на странице Участники группы StaffHub, чтобы пользователь мог связать его с правильной учетной записью Azure AD в своей организации.

Кроме того, можно использовать командлеты **Add-StaffHubMember** и **Remove-StaffHubUser** для удаления неподготовленной учетной записи из группы StaffHub и повторного добавления пользователя в группу с его UPN-именем.

Для минимизации перерывов и доступа к службам Корпорация Майкрософт рекомендует убедиться в том, что все пользователи организации обсуждаются с соответствующей лицензией.
 
#### <a name="what-happens-to-chats-in-staffhub"></a>Что происходит с разговорами в StaffHub?

Чтобы помочь пользователям переходов, мы планируем перенести историю чата с данными в StaffHub в Teams. Объем истории чатов, который будет доступен в Teams, зависит от того, как скоро вы настраиваете параметры организации для перехода. Мы рекомендуем использовать не позднее 30 июня 2019, чтобы предоставить достаточное время для перехода на 90 дней с даты до 30 июня 2020 г.

Перед началом перехода убедитесь, что в центре администрирования 365 Microsoft 365 включена поддержка групп, так как это необходимо для того, чтобы мы могли переместить данные чата StaffHub в Teams. Если вы не можете включить эту службу до 30 июня 2019 г., она получит более позднюю дату миграции или меньшее количество копируемых данных чата.

#### <a name="what-happens-to-files-that-have-been-uploaded-to-staffhub"></a>Что происходит с файлами, загруженными в StaffHub?

Вы можете переместить файлы, необходимые вашим группам, из StaffHub в Teams. Перемещенные файлы хранятся в папке "Общие" сайта группы в SharePoint Online. В Teams пользователи могут получить доступ к файлам на вкладке " **файлы** " в канале "Общее". Дополнительные сведения можно найти в [статье перевод Microsoft StaffHub Teams для смены в Teams](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="can-i-export-shift-schedules-from-shifts"></a>Можно ли экспортировать сменные расписания из смен?
  
Вы можете экспортировать сведения о расписании в файл Excel из настольных систем и веб-приложений. Затем вы можете импортировать эти данные в системы и инструменты для своей организации.

#### <a name="how-can-i-recover-a-schedule-if-i-accidentally-deleted-a-team"></a>Как восстановить расписание в случае случайного удаления команды?

Удаление расписания связано с удалением группы. Если вы восстановите команду, расписание не будет восстановлено.

#### <a name="what-powers-the-file-sharing-capability-in-teams-where-are-files-stored"></a>Что обеспечивает возможность общего использования файлов в Teams? Где хранятся файлы?

Возможность совместного использования файлов, способность руководителя передавать файлы участникам группы, будет включена на сервер SharePoint Online. При создании команды группа Microsoft 365 автоматически размещается для этой группы в фоновом режиме.

Чтобы узнать больше, посмотрите, [как SharePoint Online и OneDrive для бизнеса могут взаимодействовать с группами](../../SharePoint-OneDrive-interact.md).
 
#### <a name="is-there-integration-between-shifts-and-the-outlook-calendar-can-i-export-a-schedule-to-a-calendar"></a>Есть ли интеграция между сменами и календарем Outlook? Можно ли экспортировать расписание в календарь?
 
Firstline сотрудники смогут быстро просматривать все трудозатраты на всем рабочем графике в сменах. Мы будем поддерживать встроенную интеграцию с календарем Outlook в рамках нашего плана. На данный момент используйте Microsoft Flow для настройки интеграции с сменами в Teams и календарях Outlook и Google.

#### <a name="do-i-need-to-train-users-on-how-to-use-shifts-if-theyre-already-familiar-with-staffhub"></a>Нужно ли обучать пользователям использовать сменные, если они уже знакомы с StaffHub?

Дополнительное обучение не требуется. Они могут использовать смену аналогично тому, как они использовали StaffHub. 
 
#### <a name="what-admin-controls-are-available-to-manage-whether-users-in-my-organization-can-use-shifts"></a>Какие элементы управления доступны для управления, могут ли пользователи в Организации использовать смену?  
  
Вы можете включить или отключить смену организационного уровня для всех пользователей в вашей организации. В центре администрирования Microsoft Teams можно также использовать политику настройки приложения Firstline Worker для настройки рабочих групп для Firstlineных сотрудников. Настройка политики осуществляется с помощью действий, смен, чата и вызова приложений. Назначение политики перенаправить эти приложения на панель приложения в Teams для быстрого и удобного доступа пользователей. 

Дополнительные сведения можно найти [в разделе Управление приложением смен для Организации в Teams](manage-the-shifts-app-for-your-organization-in-teams.md).

#### <a name="what-powershell-controls-are-available-for-teams"></a>Какие элементы управления PowerShell доступны для Teams?

Управление Teams осуществляется с помощью модуля Microsoft Teams PowerShell и модуля PowerShell Skype для бизнеса. Дополнительные сведения можно найти в разделе [Общие сведения о Teams PowerShell](../../teams-powershell-overview.md). В настоящее время нет специальных командлетов PowerShell, которые можно использовать для управления сменами.

#### <a name="what-will-happen-with-the-staffhub-api-which-is-currently-in-private-preview"></a>Что произойдет с API StaffHub, который в настоящее время находится в частном предварительной версии?

Если вы создали интеграцию с API StaffHub, эти API будут устаревшими. Мы выпустили API Microsoft Graph для поддержки интеграции с решениями для управления персоналом, платой или ресурсами. Дополнительные сведения можно найти в [статье Использование API Microsoft Graph для работы с Teams](https://docs.microsoft.com/graph/api/resources/teams-api-overview).

#### <a name="is-reporting-available-to-view-shifts-usage-across-my-organization"></a>Доступны ли отчеты для просмотра используемых смен в моей организации?

Поскольку смена является приложением в Teams, вы можете просматривать использование с помощью отчетов Teams. Дополнительные сведения можно найти [в разделе Отчеты о командах в центре администрирования Microsoft Teams](../../teams-analytics-and-reports/teams-reporting-reference.md) и в [отчетах об активности Teams в центре администрирования Microsoft 365](../../teams-activity-reports.md).

#### <a name="where-can-i-provide-feedback"></a>Где можно оставить отзыв?
  
В мобильном приложении коснитесь значка **Дополнительно** в левом верхнем углу экрана, а затем выберите **Параметры**  >  **Отправить отзыв** или **сообщить о проблеме**. Аналогичным образом в классической версии веб-приложений в левой верхней части **экрана можно**  >  **поделиться идеями** или **сообщить о проблеме**.
