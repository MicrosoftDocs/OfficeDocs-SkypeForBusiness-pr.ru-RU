# <a name="microsoft-staffhub-to-be-retired"></a>Прекращение поддержки Microsoft StaffHub

Начиная с 1 октября 2019, Microsoft StaffHub будет из эксплуатации. Мы создаем StaffHub возможностей в группами Майкрософт. В настоящее время группы включает в себя приложение смены для управление планированием и дополнительные возможности будут развернуты со временем.

Эти изменения являются частью нашей продолжением работы предоставить всех сотрудников с Microsoft 365. С возможностями для работников Firstline теперь в группах каждого сотрудника в вашей организации могут использовать команды для упрощения их рабочего дня, совместно работать с коллегами и сведения о доступе и опыт, помогая им эффективную работу.

## <a name="frequently-asked-questions"></a>Вопросы и ответы

### <a name="about-staffhub-retirement"></a>О пенсионного обеспечения StaffHub

#### <a name="when-will-staffhub-be-retired"></a>Когда будет StaffHub из эксплуатации?

Запуск середине апреля 2019 Microsoft StaffHub больше не будут доступны для новых клиентов и вы больше не смогут получить доступ к приложению из запуска приложения для Office 365. Текущий пользователи по-прежнему смогут использовать службу до 1 октября 2019, однако не дополнительные функции будут доступны.

#### <a name="what-will-happen-once-staffhub-is-retired"></a>Что произойдет, если не поддерживается StaffHub?

Microsoft StaffHub перестанет работать для всех пользователей на 1 октября 2019. Всем пользователям, которые пытается открыть приложение будет выведено сообщение, предлагая, чтобы загрузить групп. Если у пользователя есть активная лицензия, которая включает в себя группы и выполнения соответствующие действия, чтобы включить командами, их данные организации и основные функциональные возможности из StaffHub можно будет их при их переходе на группы.

Между апреля 2019 и 2019 октября StaffHub пользователи будут получать уведомления в приложении стимулирования их использование групп.

#### <a name="will-teams-offer-all-the-functionality-currently-offered-in-staffhub"></a>Все возможности, предоставляемые в настоящее время в StaffHub предлагает групп?

В 2019 января группами начали предлагать смены которых был ранее доступен в Microsoft StaffHub. Эта функция менеджеры для планирования и распространения shift расписания и участниками возможность просмотреть их расписания и внесите изменения расписания запросами с помощью своих мобильных устройств. Для получения дополнительных сведений см [приложение смены для вашей организации в группах](manage-the-shifts-app-for-your-organization-in-teams.md).

Мы поддерживаем доставка функциональные возможности и внедрение уникальные возможности StaffHub в группы. Для получения дополнительных сведений, на котором функции будут доступны в группах см [Firstline работников готовится к выпуску для групп](firstline-worker-features-coming-to-teams.md).

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>Что нужно сделать, чтобы подготовиться к этому изменению?

В этой статье описывается Подготовка для перехода от StaffHub для смены в группах, перейдите к [подготовить для перемещения группам StaffHub смены](move-staffhub-teams-to-shifts-in-teams.md#prepare).

#### <a name="why-cant-users-in-my-organization-access-their-staffhub-team-from-shifts-in-teams"></a>Почему не удается пользователей в организации доступ к своей группы StaffHub из смен в группах?

Команды, которые были созданы с помощью StaffHub только можно просматривать и осуществляется в StaffHub. После StaffHub группы перемещаются командами, пользователи смогут просматривать свои расписания в смены. Мы рекомендуем вам общаться это изменение раннее и ознакомьтесь с группами существующих пользователей.

### <a name="about-shifts"></a>О смены

#### <a name="what-is-shifts"></a>Что такое смены?

Приложение смен в группы — это средство управления расписание, которое обеспечивает Firstline работников подключенными и синхронизации. Менеджеры создание, обновление и Управление расписаниями shift для групп и участники группы могут просматривать свои расписания, видеть, кто еще запланированного за день, запрос на переключение или предлагают shift и запросить нерабочее время, все из своих мобильных устройств.  

#### <a name="which-plans-is-shifts-available-in"></a>Планы которого является смены, доступные в?

Смены доступна как часть команды в следующем планы без любые дополнительные расходы:

- Office 365 бизнес премиум 
- Office 365 корпоративный F1, E1, E3, E5
- Office 365 для образовательных заведений
- Microsoft 365 F1 E1, E3, E5

Для получения дополнительных сведений см [Доступности смены](manage-the-shifts-app-for-your-organization-in-teams.md#availability-of-shifts).

#### <a name="how-do-i-get-shifts"></a>Как получить смены? 

Если вы уже план, который включает в себя смены, можно начать использование смены сегодня в рамках рабочих групп. Извлечение [Начало работы в смены](https://support.office.com/en-us/article/get-started-in-shifts-5f3e30d8-1821-4904-be26-c3cd25a497d6).

#### <a name="which-devices-or-platforms-is-shifts-available-on"></a>Какие устройства или платформ доступен смены?

Смены доступна на групп веб-клиент, клиент рабочего стола групп и групп мобильных клиентов (операций ввода-вывода и Android (en)).
 
#### <a name="are-there-differences-between-the-shifts-desktop-and-web-apps-and-the-mobile-ios-and-android-apps"></a>Существуют ли различия между смены рабочего стола и веб-приложений и приложений mobile (операций ввода-вывода и Android)?

Для руководителей рабочего стола и веб-приложений, оптимизированных для создания групп, добавить элементы, создание и публикация расписания и общий доступ к содержимому с группой. Мобильное приложение обычно используется руководителями утверждение запросов, отправлять быстрого сообщения в группу, вызовите участником группы и совместно работать в дороге. 

Участники рабочей группы мобильного приложения оптимизирован для просмотра их расписания, видеть пользователей, которые вы работаете в день, часов и выход, задайте доступности, время запроса из системы, файл подкачки или предлагают смены и откройте смены. Существуют некоторые отличия в доступны следующие действия в рабочий стол и веб-приложений и мобильных приложений. Мы работаете равенства функциональности.

#### <a name="are-there-differences-between-the-shifts-ios-app-and-the-android-app"></a>Существуют ли различия между смены операций ввода-вывода приложения и приложения для Android?

Основные функциональные возможности аналогична между смены iOS и Android приложения. Существуют небольшие отличия пользовательского интерфейса между двумя приложениями для выравнивания схем их рекомендуется применять.

#### <a name="what-browsers-does-shifts-work-on"></a>Какие браузеры смены работать на?

Смены работает во всех браузерах, поддерживаемых групп это включает последние версии пограничного сервера Microsoft Internet Explorer, Google Chrome и Mozilla Firefox. Дополнительные сведения можно [получить клиентов для групп](../../get-clients.md).

#### <a name="what-languages-does-shifts-support"></a>Какие языки поддерживает смены?

Смены доступна на следующих языках: китайский упрощенный английский (США), испанского, русского, японского языка, французский, португальский (Бразилия), немецкий, корейский, итальянский, китайский (традиционное письмо), голландский, турецкий, шведский, датский и норвежский. Планируется добавить поддержку для дополнительных языков в будущем.  

#### <a name="where-is-shifts-data-stored"></a>Где хранится смены данных?

Смены данные хранятся в Azure в центрах обработки данных в Северной Америке и Западная Европа. Контент SharePoint использует группы Office 365 и используется поддержка и хранилища.

Для получения дополнительных сведений см [расположение смены данных](manage-the-shifts-app-for-your-organization-in-teams.md#location-of-shifts-data).

### <a name="switch-from-staffhub-to-shifts-in-teams"></a>Переключитесь из StaffHub смены в группах

Каждый пользователь должны иметь active Microsoft 365 или Office 365 лицензии с [подходящими план](#which-plans-is-shifts-available-in) и должны быть назначены лицензии группами. Дополнительные сведения можно [перемещать группам StaffHub смены в группах](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="how-do-i-move-my-organizations-staffhub-teams-to-teams"></a>Как переместить Моя организация StaffHub группами групп?

Подробное руководство по переход от StaffHub групп, в разделе [Перемещение группам Microsoft StaffHub смены в группах](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="how-do-i-enable-teams-in-my-organization"></a>Как включить группами в организации?

По умолчанию группы включена для всей организации. Назначение групп лицензий для пользователей для управления доступом к отдельным группам. Дополнительные сведения можно [Включить группы в вашей организации](../../Office-365-set-up.md) и [Управление доступом пользователей к группам](../../user-access.md).

#### <a name="what-if-i-currently-have-skype-for-business-enabled-in-my-organization-but-not-teams"></a>Что делать, если в настоящее время имеют Скайп для бизнеса, включенных в моей организации, но не групп?

Совместная работа с Скайп поддерживает группами для бизнеса. Дополнительные сведения можно [понять, групп и Скайп для бизнеса сосуществования и взаимодействия](../../teams-and-skypeforbusiness-coexistence-and-interoperability.md) и [переноса и руководство по взаимодействию для организаций, с помощью команды Скайп для бизнеса](../../migration-interop-guidance-for-teams-with-skype.md).

#### <a name="does-each-user-need-to-have-an-office-365-account-or-azure-active-directory-azure-ad-account"></a>Необходимо иметь учетную запись Office 365 или учетная запись Azure Active Directory (Azure AD) каждого пользователя?

Да, каждый член manager и группы с помощью групп и смены внутри групп необходимо настроить учетную запись Office 365 (Azure AD) и необходимо присвоить действительной лицензии.
 
#### <a name="in-staffhub-i-could-add-a-special-prefix-to-all-office-365-groups-that-were-created-by-staffhub-teams-can-i-do-the-same-in-teams"></a>В StaffHub я добавьте специальный префикс все группы Office 365, которые были созданы только StaffHub группами. Можно ли сделать это в группах

При перемещении из StaffHub группы не будут иметь одну возможность. Если требуется применить согласованности имени все группы Office 365, можно использовать [политики именования групп Office 365](https://docs.microsoft.com/office365/admin/create-groups/groups-naming-policy).

#### <a name="will-team-names-be-the-same-in-teams"></a>Названия групп будет же в группах?

Это возможно, имя группы будет отличаться. Названия групп в группах используйте имя группы Office 365. Если имя группы StaffHub отличается от имени соответствующей группе Office 365, вы увидите другое имя при переходе группам.

#### <a name="what-happens-to-team-membership-when-a-staffhub-team-is-moved-to-teams"></a>При перемещении группы StaffHub группам, что происходит с членства группы?

Быть членом группы сохраняется при перемещении группы StaffHub по группам. Номера телефонов членов группы не будут отображаться на смены. Если номер телефона StaffHub сопоставляет номер телефона в Azure AD, он будет храниться в Azure AD для этой учетной записи пользователя. 

Мы рекомендуем вам просмотрите вашей существующие списки групп на StaffHub и где это возможно выровнять телефонных номеров в соответствующих Azure AD удостоверения.
 
#### <a name="what-happens-to-staffhub-team-members-who-arent-provisioned-or-whose-status-is-inactive"></a>Что происходит с StaffHub участники группы пользователей, которые не подготовлены или, состояние которого неактивные?

Расписание пользователя будут перемещены на смены, тем не менее, пользователь не будет входить в группу или группу Office 365.

Для неактивных учетных записей, которые не связаны учетной записи пользователя в Azure AD, руководителя можно обеспечить пользователя добавляется в качестве члена группы, добавив допустимый адрес электронной почты Office 365 на странице StaffHub группы участников, чтобы пользователь связан с правильный Azure AD учетной записи инфракрасного организации.

Другая возможность — для администраторов выполнить **Add StaffHubMember** и **Remove-StaffHubUser** командлетов для удаления учетной записи не подготовлен из группы StaffHub и добавьте пользователя обратно группы с помощью своих имени участника-пользователя.

Мы рекомендуем, убедитесь, что все пользователи в вашей организации освещается с соответствующей лицензии для сведения к минимуму нарушения нормального функционирования и доступ к службам.
 
#### <a name="what-happens-to-chats-in-staffhub"></a>Что происходит с бесед в StaffHub?

Чтобы помочь пользователям перехода, планирование для переноса данных журнал чата в StaffHub группами. Объем журнал чата, которые будут доступны в группах зависит от того, насколько быстро Настройка перехода в вашей организации. Мы рекомендуем не позднее 30 июня 2019 для предоставления достаточно времени для перехода в течение 90 дней данных с 1 октября 2019 действия.

Убедитесь, что создание группы Office 365 включена в центре администрирования Microsoft 365 перед выполнением перехода, это важны для нас возможность перемещения данных чата StaffHub в группы. Сбой, связанный с включить эту службу, 30 июня 2019, приведет к в нужный момент миграции или меньше чата данные копируются по.

#### <a name="can-i-export-shift-schedules-from-shifts"></a>Можно экспортировать shift расписания из смены?
  
Сведения о расписании shift можно экспортировать в файле Excel из смены рабочего стола и веб-приложения. Затем можно импортировать эти сведения для вашей организации систем и средства.

#### <a name="how-can-i-recover-a-schedule-if-i-accidentally-deleted-a-team"></a>Как восстановить расписание Если группа удалена?

Удаление расписания связан с удалением группы. При восстановлении группа не будет восстанавливать расписание.

#### <a name="what-powers-the-file-sharing-capability-in-teams-where-are-files-stored"></a>Что выключает возможность в группах совместного использования файлов? Где хранятся файлы?

Поддержка, для диспетчера распределяются файлы для участников группы, совместного использования файлов — это работает от серверного компонента SharePoint Online. При создании группы для группы Office 365 автоматически подготавливаться для этой группы в фоновом режиме.

Чтобы получить дополнительные сведения, обратитесь к разделу [как в SharePoint Online и OneDrive для бизнеса взаимодействовать с группами](../../SharePoint-OneDrive-interact.md).
 
#### <a name="is-there-integration-between-shifts-and-the-outlook-calendar-can-i-export-a-schedule-to-a-calendar"></a>Существует ли интеграцию смен и календаря Outlook? Можно экспортировать расписания в календарь?
 
Firstline работники могут видеть их всей рабочий график с первого взгляда в смены. Мы будем поддержка собственную интеграцию с календаря Outlook как часть наших схема. В это время используйте поток Microsoft, чтобы настроить интеграцию с смен в группы и Outlook и Google календарей.

#### <a name="do-i-need-to-train-users-on-how-to-use-shifts-if-theyre-already-familiar-with-staffhub"></a>Требуется для обучения пользователей на использование смены, если они уже знакомы StaffHub?

Необходим без дополнительного обучения. Они могут использовать смены аналогично как их использовать StaffHub. 
 
#### <a name="what-admin-controls-are-available-to-manage-whether-users-in-my-organization-can-use-shifts"></a>Какие элементы управления администрирования доступны для пользователей в организации можно использовать смены?  
  
Можно включить или отключить смены масштабе организации для всех пользователей в организации. В центре администрирования группами Майкрософт также можно использовать политики рабочих Firstline app программы установки для настройки групп для своих сотрудников Firstline. Политики приложений активности, смены, беседы и вызова. Назначение политики ПИН этих приложений на панели приложения в группах для быстрого и легкого доступа пользователей. 

Для получения дополнительных сведений см [приложение смены для вашей организации в группах](manage-the-shifts-app-for-your-organization-in-teams.md).

#### <a name="what-will-happen-with-the-staffhub-api-which-is-currently-in-private-preview"></a>Что произойдет с помощью API StaffHub, являющийся в настоящее время в частной предварительной версии?

При построении любого интеграции с помощью интерфейсов API StaffHub эти интерфейсы API является устаревшим. Выпущен Microsoft Graph API-интерфейсы для поддержки интеграции с персоналом, о начислении заработной платы или решений по управлению ресурсами. Для получения дополнительных сведений см [Microsoft Graph API для работы с группами](https://docs.microsoft.com/graph/api/resources/teams-api-overview).

#### <a name="is-reporting-available-to-view-shifts-usage-across-my-organization"></a>Сообщает о доступных для просмотра смены использования в организации?

Поскольку смены приложения в группах, можно просмотреть использования по группам отчетов. Для получения дополнительных сведений извлекать [групп отчетов в центре администрирования группами Майкрософт](../../teams-analytics-and-reports/teams-reporting-reference.md) и [отчеты об активности групп в центре администрирования Microsoft 365](../../teams-activity-reports.md).
