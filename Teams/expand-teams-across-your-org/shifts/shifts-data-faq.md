# <a name="shifts-data-faq"></a>Часто задаваемые вопросы о данных о сменах

В этой статье рассматриваются часто задаваемые вопросы о данных Shifts, в том числе о хранении данных Shifts, хранении, извлечении и шифровании данных.

## <a name="where-is-shifts-data-stored"></a>Где хранятся данные shifts?

Данные о сменах хранятся в одном из трех географических регионов: Азиатско-Тихоокеанский регион (APAC), Европейский союз (ЕС) или США. В каждом географическом регионе данные хранятся по крайней мере в двух регионах центра обработки данных Azure для обеспечения высокого уровня доступности (HA) и аварийного восстановления (DR). Сегодня геоцентры США и Северная Америка используют центры обработки данных в центрально-северной и южно-центральной США. Дополнительные сведения см. в разделе [Где хранятся данные клиентов Microsoft 365](/microsoft-365/enterprise/o365-data-locations).

В настоящее время shifts предлагает размещение данных в Австралии, Канаде, Франции, Японии и Соединенном Королевстве. Мы активно работаем над расширением поддержки в других расположениях.

## <a name="can-i-choose-where-shifts-data-is-stored"></a>Можно ли выбрать место хранения данных shifts?

При первой настройке Teams вы выбираете страну или регион, который устанавливается на уровне подписки. Смены учитывают этот выбор и используют языковой стандарт и регион, заданные в Teams, если мы поддерживаем этот регион. Если мы еще не в этом регионе, мы храним данные в соседнем регионе, который мы поддерживаем. В будущем мы планируем перенести существующие данные, если они хранятся в соседнем регионе, в регион, подготовленный в Teams.

## <a name="can-i-access-and-export-or-delete-a-users-personal-data-in-shifts"></a>Можно ли получить доступ к персональным данным пользователя, экспортировать или удалить их в сменах?

Смены соответствуют Общему регламенту по защите данных (GDPR). Официальный запрос лица (известного как субъект данных) на принятие мер в отношении своих персональных данных называется запросом субъекта данных (DSR). Вы можете находить персональные данные и действовать в разделе Смены в ответ на DSR.

Вы можете использовать средство обнаружения электронных данных поиска контента в Портал соответствия требованиям Microsoft Purview для поиска и экспорта данных расписания и часов времени в Excel. Для всех остальных данных Shifts можно делать снимки экрана данных.

Дополнительные сведения см[. в разделе Office 365 запросов субъектов данных для GDPR и CCPA](/microsoft-365/compliance/gdpr-dsr-office365).

## <a name="what-happens-to-shifts-data-if-i-turn-off-shifts-for-my-organization"></a>Что произойдет с данными смен, если я отключаю смены для своей организации?

Отключение смен в организации *не приводит к удалению* данных. Если отключить смены, а затем включить смены, данные о сменах будут по-прежнему доступны.

При удалении клиента все данные shifts удаляются по истечении срока хранения.

Нет возможности удалить только данные shifts. При удалении команды в Teams данные расписания смен, связанные с ней, удаляются после окончания срока хранения. Дополнительные сведения см [. в статье Хранение, удаление и уничтожение данных в Microsoft 365](/compliance/assurance/assurance-data-retention-deletion-and-destruction-overview).

## <a name="can-i-recover-a-shifts-schedule-that-was-deleted"></a>Можно ли восстановить удаленное расписание смен?

Вы можете восстановить удаленное расписание, если группа Microsoft 365, которая его поддерживает (или команда в Teams), восстановлена.

По умолчанию удаленная группа Microsoft 365 хранится в течение 30 дней. Этот 30-дневный период называется обратимым удалением, так как вы по-прежнему можете восстановить группу. Дополнительные сведения см. в статье [Восстановление удаленной группы Microsoft 365](/microsoft-365/admin/create-groups/restore-deleted-group?tabs=admin-center).

## <a name="can-i-use-custom-retention-policies-for-shifts-data"></a>Можно ли использовать пользовательские политики хранения для данных смен?

В настоящее время shifts не поддерживает пользовательские политики хранения.

Дополнительные сведения о политиках хранения в Teams см [. в разделах Сведения о хранении для Teams](/microsoft-365/compliance/retention-policies-teams) и [Управление политиками хранения для Teams](../../retention-policies.md).

## <a name="can-i-retrieve-shifts-data-for-a-user-whose-license-was-revoked"></a>Можно ли получить данные shifts для пользователя, лицензия которого была отозвана?

Сегодня мы не предлагаем возможность получения данных для пользователя, лицензия которого была отозвана. Мы работаем над этой возможностью.

## <a name="what-type-of-encryption-does-shifts-use-for-data-at-rest-and-in-transit"></a>Какой тип шифрования использует shifts для неактивных и передаваемых данных?

Неактивные данные сдвигов шифруются в Azure Cosmos DB и службе хранилища Azure. Дополнительные сведения см [. в разделах Шифрование неактивных данных Azure](/azure/security/fundamentals/encryption-atrest) и [Шифрование данных в Azure Cosmos DB](/azure/cosmos-db/database-encryption-at-rest).

Смены соответствуют рекомендациям Microsoft 365 по шифрованию передаваемых данных. Дополнительные сведения см. [в статье Шифрование передаваемых данных](/compliance/assurance/assurance-encryption-in-transit).

Смены шифрования неактивных и передаваемых данных ежегодно проверяются аудитом соответствия ТРЕБОВАНИЯМ SOC2.

## <a name="can-i-access-immutable-copies-of-shifts-data"></a>Можно ли получить доступ к неизменяемым копиям данных Shifts?

Мы не храним неизменяемые копии данных Shifts. Например, руководитель может вносить изменения в расписание, например добавлять заметки, изменять время смены, назначать задачи и т. д.

## <a name="can-shifts-data-be-edited"></a>Можно ли изменять данные shifts?

Существуют определенные аспекты смен, которые нельзя изменить, и некоторые аспекты, которые можно изменить. Например, сведения о сменах, такие как заметки и цвета, можно изменить так же, как их можно изменить в приложении "Смены". Запросы на смену нельзя изменить, если запрос не будет снят.

Чтобы узнать, какие поля были изменены, можно найти события смен в журнале аудита Microsoft 365. Дополнительные сведения о событиях, которые регистрируются для действий Shifts в журнале аудита Microsoft 365, см. [в статье Смены в действиях Teams](../../audit-log-events.md#shifts-in-teams-activities).

## <a name="my-organization-uses-a-workforce-management-system-for-scheduling-can-we-integrate-with-and-access-shifts-data"></a>Моя организация использует систему управления персоналом для планирования. Можем ли мы интегрироваться с данными Shifts и получить доступ к ним?

API-интерфейсы Graph смен позволяют интегрировать данные Shifts с внешними системами управления персоналом (WFM). Дополнительные сведения см. в разделе [API-интерфейсы Графа смен](/graph/api/resources/shift).

Мы также предлагаем соединители управляемых смен. С помощью этих соединителей вы можете интегрировать систему WFM непосредственно с shifts. Дополнительные сведения о соединителях Shifts и поддерживаемых WFM системах см. в разделе [Соединители смен](/microsoft-365/frontline/shifts-connectors).

## <a name="can-shifts-data-be-deleted-permanently-after-a-specified-period-of-time"></a>Можно ли удалить данные shifts безвозвратно по истечении указанного периода времени?

Сегодня мы вообще не удаляем ваши данные shifts. С помощью [API Graph shifts](/graph/api/resources/shift) можно [создать приложение с помощью Power Apps](/powerapps/maker/) для хранения данных в течение указанного периода времени. Однако мы не поддерживаем это изначально.

## <a name="can-shifts-data-be-moved-in-a-tenant-to-tenant-migration"></a>Можно ли перемещать данные shifts при миграции между клиентами?

Чтобы перенести существующие данные Shifts в другой клиент, необходимо экспортировать расписания смен для диапазона дат, изменить имена пользователей (при необходимости), а затем импортировать расписания в целевой клиент. За раз можно экспортировать данные смен до 100 дней. Диапазон дат может находиться в прошлом или будущем. Если вам нужно экспортировать данные в течение более 100 дней, повторите процесс.

Перед переносом данных смен убедитесь, что выполнены следующие требования:

- Домен целевого клиента, команды и участники команды уже должны существовать. Миграция не создает команды или не изменяет членство в команде или права собственности.
- Приложение Shifts должно быть настроено в командах в целевом клиенте и иметь пустое расписание. Помните, что миграция не заменяет или не удаляет существующие данные. Это означает, что если у команды есть существующее расписание, пользователи могут увидеть повторяющиеся или конфликтующие смены, которые необходимо устранить вручную.
- Открытые запросы (например, запросы на переключение и отгул), ожидающие утверждения, не переносятся. Перед переносом данных рекомендуется закрыть все открытые запросы.

Ниже приведен обзор действий по переносу данных Shifts в другой клиент.

1. В исходном клиенте для каждой команды экспортируйте расписание смен:
    1. В разделе Смены на странице **Расписание** перейдите в раздел **Дополнительные параметры (...)** >  **Расписание экспорта**.
    1. Выберите диапазон дат.
    1. Включите **параметр Экспорт в формате, который можно импортировать**, а затем выберите **Экспорт**. Данные расписания сдвигов экспортируются в файл Excel.
1. В рамках миграции, если кто-либо из участников команды переключает свой домен электронной почты, вручную обновите файл Excel, чтобы изменить имя участника-пользователя (UPN) этих пользователей на домен целевого клиента.
1. В целевом клиенте импортируйте расписание в каждую команду.
    1. В разделе Смены на странице **Расписание** перейдите в раздел **Дополнительные параметры (...)** >  **Расписание импорта**.
    1. Выберите **Отправить файл**, перейдите к файлу Excel для этой команды и нажмите кнопку **Открыть**.

Дополнительные сведения см. в разделе [Шаблон Excel для смен](https://support.microsoft.com/office/the-excel-template-for-shifts-6fc6a206-e7cc-4907-87b8-a296bae84ce3).
