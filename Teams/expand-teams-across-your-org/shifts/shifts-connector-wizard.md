# <a name="use-the-shifts-connector-wizard-to-connect-shifts-to-blue-yonder-workforce-management"></a>Использование мастера соединителя "Смены" для подключения shifts к управлению ресурсами "Синий Yonder"

## <a name="overview"></a>Обзор

Мастер соединители "Смены" в Центр администрирования Microsoft 365 позволяет интегрировать приложение "Смены" в Microsoft Teams с системой управления трудовыми ресурсами (WFM). После того как вы настроите подключение, сотрудники, работающие с телефоном, смогут без проблем просматривать свои расписания в WFM-системе и управлять ими в сменах.

Мастер настроит соединителе Shifts, создаст подключение к вашей системе WFM и примежет параметры синхронизации и сопоставления команд, которые вы выберете. Параметры синхронизации определяют сведения о расписании, синхронизируются между WFM-системой и сменами. Сопоставления групп определяют отношение синхронизации между сайтами WFM и группами в Teams. Вы можете соеоставить карту с существующими и новыми командами.

Вы можете настроить несколько подключений с разными настройками синхронизации. Например, если в вашей организации есть несколько расположений с различными требованиями к расписанию, создайте подключение с уникальными настройками синхронизации для каждого расположения. Имейте в виду, что сайт WFM можно соотнося только с одной командой в любое время. Если сайт WFM уже соотнося с командой, его нельзя соотнося с другой командой.

Благодаря системе WFM в качестве системы записи сотрудники, работающие с телефоном, могут видеть и менять смены, управлять доступностью и запрашивать отработки отработки в сменах на своих устройствах. Руководители frontline могут по-прежнему использовать вашу систему WFM для написания расписанию.

## <a name="integrate-shifts-with-blue-yonder-workforce-management"></a>Интеграция смен с управлением ресурсами "Синий Уодер"

В настоящее время мастер поддерживает соединителя [Microsoft Teams Смены для синего желтодера](shifts-connectors.md#microsoft-teams-shifts-connector-for-blue-yonder). Этот соединитетель позволяет интегрировать смены с управлением ресурсами "Синий Yonder" (Blue Yonder WFM), чтобы управлять расписаниями и поддерживать их в курсе. В этой статье мы поймем, как запустить мастер, чтобы настроить подключение к WFM "Синий yonder" с помощью соединителя.

> [!NOTE]
> Вы также можете использовать PowerShell для интеграции Shifts с WFM "Синий Уодер". Дополнительные информации см. в этой теме. В этой области вы можете подключить смены к управлению трудовыми [ресурсами "Синий Yonder"](shifts-connector-blue-yonder-powershell-setup.md).

## <a name="before-you-begin"></a>Подготовка к работе

Для запуска мастера Microsoft 365 быть глобальным администратором.

### <a name="prerequisites"></a>Необходимые компоненты
<a name="prerequisites"> </a>
[!INCLUDE [shifts-connector-prerequisites](../../includes/shifts-connector-prerequisites.md)]

- У групп, которые вы хотите спланировать, нет расписания. Если у группы есть существующее [расписание,](#remove-schedules-from-teams-you-want-to-map) удалите его из нее, прежде чем соотнося с ней сайт WFM "Синий yonder". В противном случае вы увидите повторяющиеся смены.

## <a name="remove-schedules-from-teams-you-want-to-map"></a>Удаление расписанию из групп, которые вы хотите спланировать
<a name="remove_schedules"> </a>

> [!NOTE]
> Выполните это шаг, если вы соотношали сайты WFM "Синий Yonder" с существующими командами с расписаниями. Если вы сопоставлены с командами, у которые нет расписания, или создаете новые группы для сопоставления, вы можете пропустить этот шаг.

Удалите расписания из команд с помощью PowerShell.

1. Сначала необходимо установить модули PowerShell и настроить их. Выполните действия, [чтобы настроить среду](shifts-connector-powershell-manage.md#set-up-your-environment).
1. Выполните следующую команду.

    ```powershell
    Remove-CsTeamsShiftsScheduleRecord -TeamId <Teams team ID> -DateRangeStartDate <start time> -DateRangeEndDate <end time> -ClearSchedulingGroup:$false -EntityType <the scenario entities that you want to remove, the format is @(scenario1, scenario2, ...)> -DesignatedActorId <Teams team owner ID>
    ```

    Чтобы получить список сценариев для `EntityType` параметра, запустите [Get-CsTeamsShiftsConnectionConnector](/powershell/module/teams/get-csteamsshiftsconnectionconnector?view=teams-ps). Данные расписания будут удалены для за указанного вами диапазона дат и времени.

Дополнительные информации см. [в этой теме](/powershell/module/teams/remove-csteamsshiftsschedulerecord?view=teams-ps).

## <a name="run-the-wizard"></a>Запуск мастера

### <a name="get-started"></a>Начало работы

1. В левой области навигации [Центр администрирования Microsoft 365 настройки](https://admin.microsoft.com/)**, а** затем перейдите в раздел Приложения **и электронная** почта.
1. Выберите **Подключение систему управления трудовыми ресурсами**. Здесь вы можете узнать больше о соединителах Смены, а также о работе сотрудников и руководителей при подключении shifts к системе WFM.
    :::image type="content" source="../../media/shifts-connector-wizard-get-started.png" alt-text="Снимок экрана: страница сведений для мастера соединитеща "Смены" в Центр администрирования Microsoft 365." lightbox="../../media/shifts-connector-wizard-get-started.png":::
1. Когда все будет готово **, выберите начало работы**.
1. Чтобы **создать подключение** WFM", выберите далее.

### <a name="enter-connection-details"></a>Введите сведения о под соединении
<a name="connection_details"> </a>

1. На странице Сведения о подсети придайте вашему подключению уникальное имя. Оно не может быть длиннее 128 символов и не должно иметь специальных символов.
    :::image type="content" source="../../media/shifts-connector-wizard-connection-details.png" alt-text="Снимок экрана: страница сведений о подсети мастера, на которой показаны параметры подключения." lightbox="../../media/shifts-connector-wizard-connection-details.png":::
1. Введите имя учетной записи службы WFM Blue Yonder, пароль и URL-адреса службы.
1. Когда все будет готово, выберите Далее, чтобы проверить подключение с за введенными настройками.

### <a name="choose-sync-settings"></a>Выбор параметров синхронизации
<a name="sync"> </a>

На странице Параметры синхронизации вы можете выбрать данные для синхронизации с blue Yonder WFM на Shifts, частоту синхронизации и возможность изменения данных пользователями Shifts.

1. Введите Microsoft 365 системную учетную запись.
    :::image type="content" source="../../media/shifts-connector-wizard-sync-settings.png" alt-text="Снимок экрана: страница параметров синхронизации мастера, на которой показаны параметры синхронизации." lightbox="../../media/shifts-connector-wizard-sync-settings.png":::
<a name="email"> </a>
1. В **области Получатели уведомлений электронной** почты выберите, кто получает уведомления по электронной почте об этом подмыве. Вы можете добавлять отдельных пользователей и группы. Уведомления по электронной почте содержат сведения о состоянии настройки подключения, а также о проблемах и ошибках, которые могут возникнуть после настройки подключения.
1. Выберите параметры синхронизации:
    1. В **области Расписание и смены** выберите данные WFM "Синий yonder", которые могут видеть или изменять пользователи Shifts, а затем задай частоту синхронизации.
    2. В **области Запросы** выберите типы запросов, которые могут видеть и создавать пользователи Shifts.

    > [!IMPORTANT]
    > Если вы выбрали один из следующих вариантов, чтобы отключить открытые смены, открытые запросы на смену, запросы на обмен или запросы на отступ, вам нужно сделать еще один шаг, чтобы скрыть эту возможность в shifts.
    >
    > - Open shifts: **Shifts users will not see Blue Yonder WFM data**
    > - Запросы на обмен: **функция отключена для всех пользователей**
    > - Запросы на отключительно: функция **отключена для всех пользователей**
    >
    > После запуска мастера убедитесь, что вы выполните действия, которые см. в разделе Отключение открытых смен, открытые запросы на смены [,](#disable-open-shifts-open-shifts-requests-swap-requests-and-time-off-requests) запросы на обмен и запросы на отрезки далее в этой статье.
 
1. После выбора параметров выберите Создать **подключение**.

### <a name="map-blue-yonder-workforce-management-sites-to-teams"></a>Карта сайтов управления ресурсами "Синий Yonder" с группами
<a name="sites"> </a>

Выберите сайты WFM "Синий yonder", которые вы хотите подключить к shifts. Вы можете выбрать до 100 сайтов.

:::image type="content" source="../../media/shifts-connector-wizard-sites.png" alt-text="Снимок экрана: мастер со списком сайтов WFM "Синий угон"." lightbox="../../media/shifts-connector-wizard-sites.png":::
<a name="mapping"> </a>
<a name="search_teams"> </a> Затем соотберем каждый выбранный вами сайт WFM "Синий Teams. Вы можете соедивести сайт с существующей командой или создать новую команду.
:::image type="content" source="../../media/shifts-connector-wizard-search-team.png" alt-text="Снимок экрана: на снимке экрана показана команда поиска и команда создания команды." lightbox="../../media/shifts-connector-wizard-search-team.png":::
#### <a name="to-map-a-site-to-an-existing-team"></a>Чтобы соеоставить сайт с существующей командой

1. Выберите имя сайта.
2. В области найщите команду и выберите ее. Имейте в виду, что группы, уже подключенные к сайту в этом соединении, не будут в поиске.
3. Выберите часовой пояс и ближайший город.
4. Выберите **Сохранить**, а затем **далее**.

#### <a name="to-map-a-site-to-a-new-team"></a>Чтобы соеоставить сайт с новой командой

1. Выберите имя сайта.
2. В области выберите Создать **команду**. Вы получите новую вкладку в браузере, где сможете создать команду в Центр администрирования Microsoft 365.
    1. Введите имя и необязательное описание группы.
    1. Добавьте одного или несколько владельцев команд. Убедитесь, что вы добавили Microsoft 365 системную учетную запись в качестве владельца.
    1. Добавьте участников группы.
    1. Добавьте адрес электронной почты группы и выберите параметр конфиденциальности.
    1. Просмотрите параметры и выберите команду **Добавить команду**. Когда ваша команда будет создана, выберите **Закрыть**.
3. Назад мастер, найщите и выберите созданную команду.
4. Выберите часовой пояс и ближайший город.
5. Выберите **Сохранить**, а затем **далее**.

### <a name="review-and-finish"></a>Просмотр и завершение

Просмотрите параметры. Если вам нужно внести изменения в сопоставление группы, выберите **изменить** . Когда все будет готово, выберите **Готово**.

:::image type="content" source="../../media/shifts-connector-wizard-review.png" alt-text="Снимок экрана: страница "Проверка" мастера с сопоставлениями." lightbox="../../media/shifts-connector-wizard-review.png":::

Вы увидите сообщение с подтверждением того, что мы получили ваш запрос вместе с идом операции. Заметьте ид операции. Оно потребуется для проверки состояния настройки подключения.

:::image type="content" source="../../media/shifts-connector-wizard-operation-id.png" alt-text="Снимок экрана: страница мастера с подтверждением и ИД операции." lightbox="../../media/shifts-connector-wizard-operation-id.png":::

Мастер начнет процесс, чтобы настроить подключение и соедеть сайты с выбранными командами. Этот процесс может занять некоторое время. Выбранные получатели получат по электронной почте уведомления о состоянии настройки.

Чтобы **выйти** из мастера, выберите готово.

Вы уже на пути, но еще не сделали этого! Проверьте адрес электронной почты. Вы получите подтверждение о том, что мы получили ваш запрос, а также [](shifts-connector-powershell-manage.md#check-connection-setup-status) ссылку на проверку состояния настройки.

> [!NOTE]
> Если в подключении возникает проблема или ошибка после ее настроки, вы получите уведомление по электронной почте. Чтобы устранить проблему, следуйте инструкциям в сообщении электронной почты.

## <a name="disable-open-shifts-open-shifts-requests-swap-requests-and-time-off-requests"></a>Отключение открытых смен, открытых запросов на смены, запросов на обмен и отработан

> [!IMPORTANT]
> Выполните эти действия, только если вы выбрали один из следующих вариантов, чтобы отключить в мастере открытые смены, запросы на смену, запросы на обмен или отменить. Выполнение этого шага скрывает возможности в сменах.
>
> - Open shifts: **Shifts users will not see Blue Yonder WFM data**
> - Запросы на обмен: **функция отключена для всех пользователей**
> - Запросы на отключительно: функция **отключена для всех пользователей**
>
> Без этого второго шага пользователи по-прежнему будут видеть функцию в shifts и получат сообщение об ошибке "Неподсвеченная операция" при попытке использовать ее.

Чтобы скрыть открытые смены, запросы на обмен и запросы на отключки в shifts [](/graph/api/resources/schedule?view=graph-rest-1.0) ```false```, используйте тип ресурса API Graph, чтобы настроить следующие параметры для каждой группы, назначенной сайту WFM "Синий Уодер":

- Открытие смен: ```openShiftsEnabled```
- Запросы на обмен:  ```swapShiftsRequestsEnabled```
- Запросы на отключки: ```timeOffRequestsEnabled```

Чтобы скрыть запросы на открытие смен в сменах, перейдите  Параметры в "Смены" и отключите параметр **"Открыть смены**".
