# <a name="quality-of-service-qos-in-microsoft-teams"></a>Качество обслуживания (QoS) в Microsoft Teams

В этой статье, помогут подготовиться сети организации для качества обслуживания (QoS) в группах Майкрософт.
QoS — это механизм, используемого для создания приоритетов определенных типов сетевого трафика. Определение приоритетов трафик для службы в режиме реального времени, такие как команды важна для предоставления бизнес марки пользовательского интерфейса. Для качества обслуживания для эффективной действительно необходимо настроить поддержкой качества обслуживания подключения от конечных конец (ПК, сетевые коммутаторы и маршрутизаторы в облако), так как любой части пути, который завершается сбоем для поддержки качества обслуживания может привести к снижению качества всего звонка.

_На рисунке 1. Связь между сетями для организации, а также служб Office 365_


В большинстве случаев сетевых соединений будет подключение к Интернету неуправляемые сети. Одним из вариантов с учетом качества обслуживания начала до конца — [Azure ExpressRoute](https://azure.microsoft.com/documentation/articles/expressroute-introduction/). По-прежнему рекомендуется реализовать качества обслуживания на части сети обеспечивает контроль над, а именно в локальной сети. Это улучшение качества рабочих нагрузок общение в реальном времени во всем развертывании и развеять сомнения chokepoints в существующее развертывание. 


## <a name="prioritize-teams-network-traffic-for-qos"></a>Определение приоритетов группы сетевого трафика для качества обслуживания 

В этой статье рассказывается о определять приоритеты трафика в режиме реального времени группы, а именно, аудио- или видеобеседы. Также можно определять приоритеты других типов трафика, в зависимости от потребностей.

Приоритизировать трафик можно по-разному, но чаще всего применяются метки DSCP (Differentiated Services Code Point — точка кода дифференцированных услуг). Они могут применяться («с тегами») на основании диапазоны портов и также с помощью объектов групповой политики. Мы обсудим них в этой статье. Мы рекомендуем использовать тегов, основанные на диапазоны портов, поскольку оно будет работать для всех устройств, не только те присоединен к домену.

Управление разметки DSCP с помощью объектов групповой политики гарантирует, что присоединенный к домену компьютерам получать правильные параметры и, что только администратор может управлять им.

Важно понимать, что QoS работает только при реализации на все ссылки, которые подключаются вызывающего абонента вызываемому. При использовании QoS во внутренней сети и пользователь входит в с удаленного компьютера, можно только определять приоритеты внутренним, управляемой сети. Несмотря на то, что удаленных расположений может получать управляемого подключения виртуальная частная сеть (VPN), рекомендуется избегать под управлением трафика в режиме реального времени через VPN.

> [!NOTE]
> Рекомендуется реализовать разделенное туннелирование для VPN-подключения удаленных пользователей, для обеспечения максимальной качества взаимодействия с пользователем. Загрузите документ [Туннель Split развернуть руководство по виртуальной частной сети](https://myadvisor.fasttrack.microsoft.com/CloudVoice/Downloads?SelectedIDs=5_1_0_9 ) с MyAdvisor для получения дополнительных сведений.

В международной организации с помощью управляемой ссылки, охватывающих разных странах качества обслуживания настоятельно рекомендуется, поскольку пропускной способности для этих ссылок не может превышать по сравнению с локальной сети.

## <a name="qos-queues"></a>Качество обслуживания очереди

Для обеспечения гарантированной уровня службы для приложения в сети, базовым сетевых устройств должно иметь возможность для классификации различных типов трафика. Если ваша организация хочет приоритетность трафика голосовой связи по другому трафику, маршрутизатор (например) необходимо иметь возможность различать голосовой связи и обычной трафика просмотра веб страниц. 

Разнообразных услуг (DiffServ) предоставляет платформу, в котором трафика задается приоритет различных сетевых устройств, на основе типа службы (ToS) поля в заголовке пакета IPv4 и IPv6. Шесть наиболее важные поля DiffServ биты точка кода разнообразных услуг или DSCP. С помощью этой платформы, трафика можно рассматривать как трафика (например, голосовой связи) определенного типа и затем помечены (101110 или 46 в десятичном формате для передачи голосовых данных), таким образом, когда сетевых устройств обработать пометки, может быть трафика их приоритета соответствующим образом () Срочные пересылки, в данном примере).

Когда сетевой трафик поступает маршрутизатором, трафика ставится в очередь, если нет на месте без QoS, по сути существует только одна очередь и данные обрабатываются как первое на first-out. Это означает, что передачи голосовых данных (который является очень чувствительны к задержкам) может войти за трафика из Интернета потоковой передачи. При реализации качества обслуживания, можно определить несколько очередей с помощью различных перегрузки функции управления (например, Cisco приоритет очереди и на основе класса Средневзвешенное обычное очереди [CBWFQ]) и функции предотвращения перегрузки (например, взвешенными в случайном порядке первых [обнаружения [[[WRED]).

После этих частей возможность доставить прогнозируемый QoS, так как базовым управляемой сети теперь имеет представление как классифицировать, пометить и определять приоритеты трафика. С точки зрения групп — это самый важный этап настройки, классификации и маркировки пакетов, но для начала до конца качества обслуживания для успешной также необходимо тщательно выровнять конфигурации приложения с базовым сетевой конфигурации.

## <a name="teams-qos-scenarios"></a>Сценарии QoS групп

Руководство по внедрению качества обслуживания для групп основано на четыре сценария:

*  **Сценарий 1:** Развернутый, или развертывание, групп и планирования на внедрение QoS через порт based тегов. На основе порта тегов — это самый надежный метод, так как он работает универсального во всем всех платформ и является простым для реализации.  

*  **Сценарий 2:** Развернутый, или развертывание, групп и планирования для реализации QoS с помощью тегов объекта групповой политики. Этот сценарий иногда применяется в сочетании со сценарием 1. Несмотря на то, что этот сценарий полностью допустим, важно понимать, что он работает только для клиентов, присоединенный к домену Windows. Любое устройство, не присоединенный к домену клиента Windows не будет включена поддержка DSCP тегов.

*  **Сценарий 3:** Развернутый Скайп для бизнеса в Интернет, включая теги качества обслуживания и теперь выполняется развертывание групп. В этом случае Teams сохранит текущие настройки и будет использовать те же диапазоны портов и маркировку, что и клиент Skype для бизнеса. В большинстве случаев будет необходимо управлять дополнительной настройки. 

    > [!NOTE]
    > Если вы используете маркировки с помощью групповой политики QoS имя приложения, необходимо добавить Teams.exe в качестве имени приложения.

*  **Сценарий 4:** Когда развертывание или выполняется развертывание групп и требуется реализовать качества обслуживания через порт based тегов с использованием настраиваемых портов диапазона.

## <a name="recommended-dscp-markings"></a>Рекомендуемые маркировку DSCP

Первым шагом является для классификации потоки трафика (например, звук и видео), присваивая значения DSCP диапазоны портов уникальный, без перекрытия. Значением DSCP, назначенные здесь определяет приоритет трафика, перейдя по сети, в зависимости от конфигурации сети. Каждой рабочей нагрузки получает собственный значением DSCP, хотя не обязательно уникальное значение — некоторые клиенты могут задать такое же значение для голосовой и видеосвязи рабочих нагрузок, предоставляя им одинаковый приоритет в сети.  

Значение DSCP для использования зависит от того, как требуется определить приоритеты рабочей нагрузки. Например, бизнес-требований могут быть причиной, что вам нужно предоставить приложения общего доступа к одинаковый приоритет как видео (и поэтому помечать с тем же значением DSCP, как видео). Других средах может быть существующей стратегии качества обслуживания на месте, которая поможет вам определить приоритет рабочие нагрузки сети. 

В таблице 1 приведены маркировку DSCP, необходимые при использовании групп с ExpressRoute. Пометки можно настроить в качестве хорошей отправной точкой для пользователей, которые не уверены, что следует использовать в своих собственных средах. Дополнительные сведения см. в статье [Требования к качеству обслуживания для ExpressRoute](https://docs.microsoft.com/azure/expressroute/expressroute-qos).


_В таблице 1. Маркировку DSCP_

| Диапазон портов источника клиента |Протокол|Категория медиаданных|DSCP Value|Класс DSCP|
|---------|---------|---------|---------|---------|
| 50 000 – 50,019|TCP/UDP|Аудио|46|Беспрепятственная переадресация (EF)|
| 50,020 – 50,039|TCP/UDP|Видео|34|Гарантированная переадресация (AF41)|

-  Если планируется реализовать ExpressRoute в будущем и еще не реализован качества обслуживания, рекомендуется следовать указаниям в таблице 1, чтобы значения DSCP отличаются от отправителя получателю. 

-  Все клиенты, включая мобильных клиентов и устройств команды будет использовать данные диапазоны портов и затрагиваются политикой DSCP, реализацией, которая использует эти диапазоны портов источника. Только клиентов, которые будут использовать динамические порты — это веб-клиентов (то есть эти клиенты, которые позволяют участникам присоединяться к собраниям с помощью обозревателей).

-  Несмотря на то, что клиент Mac использовать же диапазоны портов, клиент Mac также использует жестко значения для (EF) аудио и видео (AF41). Эти значения не настраиваются.

## <a name="setting-dscp-markings"></a>Настройка пометок DSCP

Существует несколько подходов к установке правильного маркировку DSCP для классификации.

-  **Разметки DSCP в конечной точке:** Обычно это предпочтительным, поскольку конечной точки самого обеспечивает правильное маркировку. В настоящее время это можно сделать с помощью объекта групповой политики, но его можно использовать только на клиентах Windows присоединенный к домену. Мобильные клиенты не предоставляют механизм, позволяющий пометить как трафика с помощью значения DSCP. Несмотря на то, что нельзя настроить не&ndash;присоединенный к домену клиенты Windows для трафика тег, клиенты, например, Mac OS жестко теги и всегда тега трафика, как описано выше.

-  **На основе порта DSCP тегов с помощью списки управления доступом (ACL) на маршрутизаторы:** Это очень часто параметр в разнородных средах Windows и Mac. В этом сценарии сети группы помечает трафик в маршрутизаторы входящих и исходящих (обычно находится в глобальной сети) на основании диапазоны портов источника, определенных для каждой модальности. Несмотря на то, что это работает на платформах, только помечает трафика на границе глобальной сети — не на клиентском компьютере и поэтому снижает производительность управления.

-  **Сочетание маркировку DSCP в конечную точку и на основе порта списки управления доступом на маршрутизаторы:** Рекомендуется это сочетание, если это возможно в вашей среде. Используйте объект групповой политики для перехвата большинства клиентов, а также на основе порта DSCP теги, убедитесь, что, мобильный, Mac и другие клиенты будут по-прежнему получить лечения качества обслуживания (по крайней мере, частично).

QoS на основе политики в рамках групповой политики можно использовать для задания значения DSCP для диапазона портов предварительно заданных источника в клиентских групп. Чтобы создать политику для каждой рабочей нагрузки используйте диапазоны портов, указанных в таблице 3.

_В таблице 3. Диапазоны портов по типу трафика_

| Тип трафика клиента|Начало диапазона портов|Конец диапазона портов|DSCP Value|
|---------|---------|---------|--------|
| Аудио|50000|50019|46|
| Видео|50020|50039|34|
| Общий доступ к приложениям|50040|50059|18|

> [!NOTE]
> Набор команд диапазонов портов нельзя изменить. Просмотрите [в этой статье поддержка](https://support.microsoft.com/kb/2409256) для получения последних сведений. 

Определив диапазоны портов, следующим шагом является настройка параметров QoS на основе политики в объект групповой политики. В [этой статье TechNet](https://technet.microsoft.com/library/jj205371(v=ocs.15).aspx)можно найти дополнительные сведения об этих действиях. 

Новые политики, созданному вами вступят в силу только после обновления групповой политики на клиентских компьютерах. Несмотря на то, что на собственный периодически обновляет групповой политики, вы можете принудительно немедленного обновления.

### <a name="force-group-policy-refresh"></a>Обновления групповой политики.

1. На каждом компьютере, для которого требуется обновить групповой политики откройте командную консоль. Убедитесь, что командной консоли для запуска от имени администратора.

2. В командной строке введите
   ```
    gpudate.exe /force
   ```

## <a name="verify-dscp-markings-in-the-group-policy-object"></a>Проверка маркировку DSCP в объекте групповой политики

Чтобы убедиться в том, что были заданы значения из объекта групповой политики, выполните следующие действия.

1. Откройте командную консоль. Убедитесь, что командной консоли для запуска от имени администратора.

2. В командной строке введите 
   ```
   gpresult /R >gp.txt
   ```

   Это будет создать отчет и отправьте его в текстовый файл с именем gp.txt. Кроме того можно ввести следующую команду, чтобы создать те же данные в формате HTML отчет с именем gp.html:
   ```
   gpresult /H >gp.html
   ```

   ![Снимок экрана окна консоли, выполнив команду gpresult.] (media/Qos-in-Teams-Image3.png "Снимок экрана окна консоли, выполнив команду gpresult.")

3. В созданном файле найдите заголовок **Применения объектов групповой политики** и убедитесь, что имена объектов групповой политики, созданной ранее в списке примененные политики. 

4. Откройте редактор реестра и перейдите к:

   HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Windows\QoS\

   Проверка значения реестра, перечисленных в таблице 3.

   _В таблице 3. Значения для записи реестра Windows для качества обслуживания_


   |          Имя          |  Тип  |    Значение     |
   |------------------------|--------|-------------|

   | Local IP Prefix Length | REG_SZ |     \*      |
   |       Local Port       | REG_SZ | 50000-50019 |

5. Проверьте правильность значение для записи в имя приложения для клиента, которую вы используете и убедитесь, что локальный порт и значением DSCP записи отражают параметров в объекте групповой политики.

## <a name="validate-qos-by-analyzing-teams-traffic-on-the-network"></a>Проверка качества обслуживания анализ групп трафика в сети

Значение DSCP, установленное объект групповой политики должен присутствовать от вызывающего абонента вызываемому в порядке для качества обслуживания для эффективной. Просмотрев трафика, созданных функцией клиентских групп, вы можете проверить, что значение DSCP не будет изменен или включаются при трафика рабочая нагрузка групп проходят через сеть. 

Желательно, чтобы записать трафик в выходной точке сети. Чтобы помочь в этом, можно использовать зеркальное отображение порт коммутатора или маршрутизатора.

### <a name="use-network-monitor-to-verify-dscp-values"></a>Использование сетевого монитора для проверки значения DSCP

Сетевой монитор — это средство, вы можете [загрузить корпорацией Майкрософт](https://www.microsoft.com/download/4865) для анализа сетевого трафика.

1.  На Компьютере под управлением сетевой монитор подключитесь к порт, который был настроен для порта зеркального отображения и Пуск записи пакетов. 

2.  Звонок с помощью Скайп для клиента Business. Убедитесь, что мультимедиа было установлено до разъединения вызова. 

3.  Остановите запись.

4. В поле **Фильтр отображения** используйте IP-адрес источника ПК, выполнившем вызов и уточнить фильтра, определение значением DSCP 46 (шестнадцатеричный 0xb8) в качестве условия поиска, как показано в следующем примере:

    Source == "192.168.137.201" AND IPv4.DifferentiatedServicesField == 0xb8 

    ![Снимок экрана диалогового окна фильтра отображения в сетевом мониторе, отражающая фильтры, применяемые.] (media/Qos-in-Teams-Image4.png "Снимок экрана диалогового окна фильтра отображения в сетевом мониторе, отражающая фильтры, применяемые.")

5.  Выберите **Применить** , чтобы активировать фильтр.

6.  В окне **Сводка Frame** установите первого пакета UDP-ПОРТ.

7.  В окне **Сведений о кадре** разверните элемент списка IPv4 и обратите внимание на значение в конце строки, которая начинается с **DSCP**. 

    ![Снимок экрана диалогового окна сведений о кадре в сетевом мониторе, выделение параметры DSCP.] (media/Qos-in-Teams-Image5.png "Снимок экрана диалогового окна сведений о кадре в сетевом мониторе, выделение параметры DSCP.")

В этом примере значение DSCP значение 46. Это правильно, так как исходный порт, используемый 50019, который означает, что это рабочей нагрузки голосовой связи. 

Повторите проверку для каждой рабочей нагрузки, которая маркируется при помощи GPO.
