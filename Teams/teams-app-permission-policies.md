# <a name="manage-app-permission-policies-in-microsoft-teams"></a>Управление политиками разрешений для приложений в Microsoft Teams

[!INCLUDE [preview-feature](includes/preview-feature.md)]

Как администратора чтобы контролировать, какие приложения будут доступны пользователям группами Майкрософт в организации можно использовать политики разрешений приложения. Можно разрешить или заблокировать все приложения или определенных приложений, опубликованные корпорацией Майкрософт, сторонние производители и вашей организации. При блокировании приложения, пользователи не могут установить его из хранилища приложения группы.

Для управления политиками разрешений приложения в центре администрирования группами Майкрософт. Можно применить параметры всей организации, с помощью глобальной политики (по умолчанию в масштабах организации), создание и назначение настраиваемых политик для отдельных пользователей или пользователей в группу.  

![Снимок экрана приложения политику разрешений](media/app-permission-policies.png)

> [!NOTE]
> Если не создавать и назначить пользовательскую политику пользователей в вашей организации автоматически получают глобальную политику. Настройки приложения масштабе организации переопределения глобальную политику и настраиваемых политик, создайте и назначьте для пользователей.

Предположим, например, нужно адресов все приложения сторонних производителей и конкретные приложения от корпорации Майкрософт для управления Персоналом группы в вашей организации. Будет создать пользовательскую политику с именем политики разрешений приложения отдела Кадров, настройте его на адресов и приложений, которые будут и назначить пользователям в группе отдела Кадров.

## <a name="manage-org-wide-app-settings"></a>Управление параметрами приложения масштабе организации

Использование параметров приложения в масштабе организации ли контролировать, какие приложения, доступны в организации. Параметры приложения в масштабе организации управляют поведением для всех пользователей и переопределить любые разрешение политики приложения, назначенные пользователям. Настройки приложения масштабе организации вступили в силу немедленно, и их можно использовать для управления вредоносные или инспектором приложений.

1. В левой области переходов центра администрирования группами Майкрософт, перейдите к **группам приложения** > **политики разрешений**.
2. Выберите **Параметры на уровне организации**. Затем можно настроить параметры, которые требуется в панели. 
![Снимок экрана параметров приложения в масштабе организации](media/app-permission-policies-org-wide-settings.png)
3. В разделе **приложения сторонних производителей**отключить или включить эти параметры для управления доступом к приложениям сторонних производителей:

    - **Разрешить приложений сторонних производителей в группах**: Определяет, могут ли пользователи с помощью приложения сторонних производителей.
    - **Разрешить любой новых приложений сторонних производителей, опубликован в хранилище по умолчанию**: в этом ли новые приложения сторонних производителей, которые опубликованы в приложения группы хранения автоматически доступны в группах элементов управления. Этот параметр можно задать только в том случае, если вы включили приложений сторонних производителей.

4. В разделе **Custom apps**отключить или включить **Разрешить взаимодействие с настраиваемые приложения**. Этот параметр определяет, будет ли пользователи могут работать с приложениями custom (sideloaded). Имейте в виду, что это отличается от позволяя пользователям *загружать* настраиваемые приложения.
5. В разделе **приложения заблокирован**искать и добавлять приложения, которые необходимо заблокировать внутри организации. Приложения можно выбрать из каталога приложений клиента или в хранилище приложений группами.
6. Нажмите кнопку **Сохранить** для настройки приложения масштабе организации вступили в силу.

## <a name="create-a-custom-app-permission-policy"></a>Создайте политику разрешений пользовательское приложение

Если вы хотите управлять приложения, доступные для различных групп пользователей в вашей организации, создайте и назначьте одну или несколько политик разрешений пользовательское приложение. Можно создать и назначить отдельные настраиваемых политик на основании ли приложения публикуются корпорацией Майкрософт, сторонние производители или вашей организации. Важно знать, что после создания пользовательскую политику, изменить его нельзя в масштабе организации параметры при отключении приложений сторонних производителей. 

1. В левой области переходов центра администрирования группами Майкрософт, перейдите к **группам приложения** > **политики разрешений**.
2. Выберите **новую политику**.
    ![Снимок экрана новую политику разрешений приложения](media/app-permission-policies-new-policy.png)
3. Введите описательное имя для политики.
4. В разделе **приложений Microsoft**, **приложений сторонних производителей**и **приложения клиента**выберите один из следующих:

    - **Разрешить всем приложениям**
    - **Разрешать конкретных приложений и заблокировать все остальные**
    - **Блокировать определенных приложений и разрешить всем другим пользователям**
    - **Блокировать все приложения**

5. Если выбран вариант **разрешить конкретные приложения и блокировка других пользователей**, добавьте приложения, которые необходимо разрешить:

    1. Установите флажок **Разрешить приложений**.
    1. Поиск для приложений, которые необходимо разрешить, а затем нажмите кнопку **Добавить**. Результаты поиска фильтруются для издателя приложения (**приложений Microsoft**, **приложений сторонних производителей**или **приложения клиента**).
    1. Если вы выбрали список приложений, нажмите кнопку **Разрешить**.

6. Аналогично Если вы выбрали **адресов конкретные приложения и все остальные**, поиск и добавлять приложения, которые нужно заблокировать.
7. Нажмите кнопку **Сохранить**.

## <a name="edit-an-app-permission-policy"></a>Изменение политики разрешений приложения

Можно использовать Центр администрирования группами Майкрософт для изменения политики, включая глобальной политики (по умолчанию в масштабах организации) и настраиваемых политик, которые вы создаете. 

1. В левой области переходов центра администрирования группами Майкрософт, перейдите к **группам приложения** > **политики разрешений**.
2. Выберите политику, которую требуется изменить.
3. Здесь внесите изменения, которые будут. Можно управлять параметрами на основании издателя приложения и добавлять и удалять приложения на основе параметра разрешенных и запрещенных.
4. Нажмите кнопку **Сохранить**.

## <a name="assign-a-custom-app-permission-policy-to-users"></a>Назначение политики разрешений пользовательское приложение для пользователей

Можно использовать Центр администрирования группами Майкрософт назначение настраиваемая политика для отдельных пользователей или Скайп для модуль Business PowerShell, чтобы назначить пользовательскую политику к нескольким пользователям, например всех пользователей в группу безопасности или группу рассылки.

> [!IMPORTANT]
> Рекомендуется только для назначения политик для пользователей, используя PowerShell. Использование центра администрирования группами Майкрософт для создания, редактирования и управление политиками.

### <a name="assign-a-custom-app-permission-policy-to-individual-users"></a>Назначение политики разрешений пользовательское приложение для отдельных пользователей

1. В левой области переходов центра администрирования группами Майкрософт перейдите к **пользователям**и нажмите кнопку пользователя.
2. Рядом с пунктом **назначения политик**и выберите **Изменить**.
3. В разделе **Политика разрешений приложения**выберите политику разрешений приложения, которые необходимо назначить и нажмите кнопку **Сохранить**.

    ![приложение-программы установки разрешение assign-policy.png](media/app-permission-policies-assign-policy.png)

Можно также назначить политику разрешений приложения к одному или нескольким пользователям следующим образом:

1. Перейдите в **Центр администрирования группами Майкрософт** > **группами приложений** > **политики разрешений**.
2. Выберите политику, нажав кнопку слева от имени политики.
3. Выберите **Управление пользователями**.
4. В области **Управление пользователями** поиска для пользователя по отображаемому имени или по имени пользователя, выберите имя и выберите команду **Добавить**. Повторите этот шаг для каждого пользователя, который вы хотите добавить.
5. Завершив добавление пользователей, выберите команду **Сохранить**.
 

### <a name="assign-a-custom-app-permission-policy-to-users-in-a-group"></a>Назначение политики разрешений пользовательское приложение для пользователей в группы

Можно назначить политику разрешений пользовательское приложение к нескольким пользователям, которые уже определен. Например можно назначить политики для всех пользователей в группу безопасности. Это можно сделать, подключение к Azure Active Directory PowerShell для модуля график и Скайп для модуль PowerShell бизнеса. Дополнительные сведения об использовании PowerShell для управления группами см [Команды PowerShell](teams-powershell-overview.md).

В этом примере назначить политику разрешений пользовательское приложение, называется политика разрешений приложения отдела Кадров для всех пользователей в группу проекта Contoso Pharmaceuticals отдела Кадров.  

> [!NOTE]
> Убедитесь, что подключение сначала для Azure Active Directory PowerShell для модуля график и Скайп для модуль Business PowerShell, выполнив действия, описанные в [подключиться ко всем службам Office 365 в одном окне Windows PowerShell](https://docs.microsoft.com/office365/enterprise/powershell/connect-to-all-office-365-services-in-a-single-windows-powershell-window).

Получите GroupObjectId конкретной группы.
```
$group = Get-AzureADGroup -SearchString "Contoso Pharmaceuticals HR Project"
```
Получение элементов из указанной группы.
```
$members = Get-AzureADGroupMember -ObjectId $group.ObjectId -All $true | Where-Object {$_.ObjectType -eq "User"}
```
Назначьте всем пользователям в группе политика разрешений определенного приложения. В этом примере это политика разрешений приложения отдела Кадров.
```
$members | ForEach-Object { Grant-CsTeamsAppPermissionPolicy -PolicyName "HR App Permission Policy" -Identity $_.EmailAddress}
``` 
В зависимости от числа членов группы эта команда может занять несколько минут для выполнения.

## <a name="faq"></a>Вопросы и ответы

### <a name="working-with-app-permission-policies"></a>Работа с политиками разрешений приложения

#### <a name="can-i-control-line-of-business-lob-apps"></a>Можно управлять бизнес-приложения

Да, можно использовать политики разрешений приложения для контроля развертывания и распространения пользовательских приложений (LOB).

#### <a name="how-do-app-permission-policies-relate-to-pinned-apps-and-app-setup-policies"></a>Как связаны политики разрешений приложения в закрепленных приложения и политик настройки приложения?

Можно использовать политики в отношении установки приложений вместе с политиками разрешений приложения. Предварительно закрепленных приложений выбираются из набора включенных приложений для пользователя. Кроме того Если пользователь имеет политику разрешений приложения, который блокирует приложения в политику установки приложения, это приложение не будет отображаться в группах.

#### <a name="can-i-use-app-permission-policies-to-restrict-uploading-custom-apps-also-known-as-sideloading"></a>Можно ли использовать политики разрешений приложения запретить отправку настраиваемые приложения (также известной как sideloading)?

Для получения дополнительных сведений об ограничении загрузки настраиваемые приложения, просмотрите [Управление пользовательское приложение политики и параметры в группах](teams-custom-app-policies-and-settings.md).

#### <a name="how-long-does-it-take-for-policy-changes-to-take-effect"></a>Сколько времени для изменения политики вступили в силу?

Изменить глобальную политику или назначение политики для пользователей, может потребоваться до 24 часов для изменения вступили в силу. Параметры приложения в масштабе организации вступили в силу немедленно.

#### <a name="does-blocking-an-app-apply-to-teams-mobile-clients"></a>Блокировка приложения применяется к группам мобильных клиентов?

Да, при блокировании приложения это приложение блокируется для всех групп клиентов.  

### <a name="user-experience"></a>Взаимодействие с пользователем

#### <a name="what-does-a-user-experience-when-an-app-is-blocked"></a>Что взаимодействие с пользователем при блокировании приложения?

Пользователи не могут взаимодействовать с заблокированные приложения или свои возможности, такие программы-роботы, вкладок и расширения системы обмена сообщениями. В общего контекста, такие как группы или группы чата программы-роботы по-прежнему могут отправлять сообщения всем участникам этого контекста. Группы указывает пользователя при блокировании приложения. 

Например приложения заблокирована, пользователям не удается одной из следующих:

- Добавить приложение, раскрывающие личность или беседы или группы
- Отправлять сообщения в приложении программа-робот
- Выполнение действия кнопки, которые отправляют данные обратно в приложения, например предусматривающей действие сообщения  
- Вкладка приложения
- Настройка соединителей для получения уведомлений
- Использование расширения приложения обмена сообщениями
