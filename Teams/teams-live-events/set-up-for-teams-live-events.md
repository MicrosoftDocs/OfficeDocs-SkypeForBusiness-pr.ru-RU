# <a name="set-up-for-live-events-in-microsoft-teams"></a>Подготовка к трансляциям в Microsoft Teams

При настройке для событий Live Events необходимо выполнить несколько действий.

## <a name="step-1-set-up-your-network-for-live-events-in-teams"></a>Шаг 1: Настройка сети для мероприятий Live Events в Teams
События в реальном времени, созданные в Teams, требуют [подготовки сети организации для Teams](https://docs.microsoft.com/microsoftteams/prepare-network).  

## <a name="step-2-get-and-assign-licenses"></a>Шаг 2. Покупка и назначение лицензий
Убедитесь, что у вас есть правильные назначения лицензий для [тех, кто может создавать и планировать](plan-for-teams-live-events.md#who-can-create-and-schedule-live-events) мероприятия в реальном времени, а [также кто может смотреть события в реальном времени](plan-for-teams-live-events.md#who-can-watch-live-events).

## <a name="step-3-set-up-live-events-policies"></a>Шаг 3: Настройка политик Live Events
Политики Live Events используются для управления тем, кто в Организации может включать в себя события в реальном времени и возможности, доступные в создаваемых ими событиях. Вы можете использовать политику по умолчанию или создать одну или несколько настраиваемых политик живых событий. После создания настраиваемой политики назначьте ее пользователю или группе пользователей в Организации.

> [!NOTE]
> Пользователи в вашей организации получат глобальную политику, если вы не создадите и не назначаете пользовательскую политику. По умолчанию в глобальной политике планирование событий включено для пользователей Teams, живых подписей и субтитров (транскрипция) отключено, все пользователи в организации могут присоединяться к событиям в реальном времени, а параметру записи будет присвоена постоянная запись. 

### <a name="create-or-edit-a-live-events-policy"></a>Создание и изменение политики живых событий

2. Выполните одно из следующих действий:
- Если вы хотите изменить существующую политику по умолчанию, выберите пункт **Глобальная (по умолчанию)**. 
- Если вы хотите создать новую настраиваемую политику, нажмите кнопку **создать политику**. 
- Если вы хотите изменить настраиваемую политику, выберите ее и нажмите кнопку **изменить**.

### <a name="assign-a-live-events-policy-to-users"></a>Назначение пользователям политики Live Events 

Если вы создали специальную политику событий Live Events, назначьте ее пользователям, чтобы политика была активна. 

![Значок, показывающий логотип Microsoft Teams](../media/teams-logo-30x30.png) Использование центра администрирования Microsoft Teams

1. На панели навигации слева перейдите в раздел **Пользователи**и выберите пользователя.
2. Рядом с пунктом **назначенные политики**нажмите кнопку **изменить**. 
3. Выберите политику живых событий, которую вы хотите назначить, и нажмите кнопку **сохранить**. 

Вы также можете назначить политику интерактивных событий для одного или нескольких пользователей, выполнив указанные ниже действия.

![Значок, показывающий логотип Microsoft Teams](../media/teams-logo-30x30.png) Использование центра администрирования Microsoft Teams

1. Перейдите к разделу политики **собраний** > **Live Events**.
2. Выберите политику, щелкнув слева от имени политики.
3. Выберите **Управление пользователями**.
4. В области **Управление пользователями** найдите пользователя по отображаемому имени или по имени пользователя, выберите имя и нажмите кнопку **Добавить**. Повторите этот шаг для каждого пользователя, которого вы хотите добавить.
5. Завершив добавление пользователей, нажмите кнопку **сохранить**.
 

### <a name="enable-users-to-schedule-events-that-were-produced-with-an-external-app-or-device"></a>Предоставление пользователям возможности планировать события, созданные с помощью внешнего приложения или устройства

Для планирования событий, возникающих при работе с внешним приложением или устройством, необходимо также выполнить указанные ниже действия.

1. Включение Microsoft Stream для пользователей в Организации. Поток доступен как часть подходящих подписок на Office 365 или как самостоятельная служба. Поток не включен в планы бизнеса или бизнес премиум. Дополнительные сведения о [лицензировании Stream](https://docs.microsoft.com/stream/license-overview) см.

      Узнайте больше о том, как [назначать лицензии пользователям в Office 365](https://support.office.com/article/Assign-licenses-to-users-in-Office-365-for-business-997596B5-4173-4627-B915-36ABAC6786DC) , чтобы пользователи могли получать доступ к потоку. Убедитесь, что поток не заблокирован для пользователей, описанных в [этой статье](https://docs.microsoft.com/stream/disable-user-organization).

2. Убедитесь, что у пользователей есть разрешение на создание событий в потоковом режиме. По умолчанию администраторы могут создавать события с помощью внешнего приложения или устройства. Администратор Stream может [включить дополнительные пользователи для создания событий в реальном времени](https://docs.microsoft.com/stream/live-event-administration#enabling-and-restricting-users-to-creating) в потоке.  

3. Убедитесь в том, что организаторов событий в реальном времени проводился в политику компании, установленную администратором потока. Если администратор потока настроил [политику руководств компании](https://docs.microsoft.com/stream/company-policy-and-consent) и требует, чтобы сотрудники принимали эту политику перед сохранением содержимого, необходимо сделать это перед созданием живого мероприятия (с внешним приложением или устройством) в Teams. Перед развертыванием функции Live Events в организации убедитесь в том, что пользователи, которые будут создавать эти события в реальном времени, были отправлены в политику. 

## <a name="step-4-set-up-a-video-distribution-solution-for-live-events-in-teams"></a>Шаг 4: Настройка решения для рассылки сообщений в реальном времени в Teams
Воспроизведение видеороликов в реальном времени использует адаптивную скорость потока (ГРАНИЧНЫЙ канал), но это одноадресный поток, что означает, что каждое средство просмотра получает собственный видеопоток из Интернета. Для мероприятий и видеороликов, отправленных в большие части организации, может быть получено значительное количество пропускной способности Интернета. Для организаций, которые хотят уменьшить этот Интернет-трафик для событий в реальном времени, решения для служб Live Events интегрируются с партнерами по доставке доверенных видеофайлов Майкрософт, предлагающего программные сети (СДНС) или корпоративные сети доставки содержимого (Екднс). Эти платформы SDN/Екдн позволяют организациям оптимизировать пропускную способность сети без ущерба для конечных пользователей, просматривающих опытные пользователи. Наши партнеры помогут вам повысить масштабируемость и эффективное распространение видео в корпоративной сети.

**Приобретение и настройка решения за** пределами Teams Получите экспертную помощь по масштабированию доставки видео, используя доверенные партнеры Майкрософт по доставке. Прежде чем вы сможете использовать службу доставки видео в Teams, вы должны приобрести и настроить решение SDN/Екдн вне и отдельно от Teams.

Следующие решения SDN/Екдн предварительно интегрированы и могут быть настроены для использования с потоком.

- **Потоки кустов** предоставляют простое и мощное решение для интерактивного и оперативного распространения видео по сети. Hive — это программное решение, которое не требует дополнительного оборудования или пропускной способности и обеспечивает безопасный способ включения нескольких одновременных средств просмотра видео без воздействия на сеть. Для пользователей, стремящихся понять, какое влияние на видео в сети до приобретения решения SDN/Екдн, поток Hive также предоставляет решение для анализа на основе браузера для пользователей Microsoft. [Подробнее](https://www.hivestreaming.com/partners/integration-partners/microsoft/).
 
- **Коллективе** — это облачная платформа для распространения на основе облаков, использующая существующую сетевую инфраструктуру для предоставления доступа к содержимому, во многих формах (видеороликах, видеороликах, обновлениях программного обеспечения, исправлениях для системы безопасности и т. д.) быстрее, больше надежно и с меньшим объемом пропускной способности. Наша надежная платформа является надежной для больших финансовых учреждений мира и не имеет дополнительного оборудования, настройки и обслуживания. [Подробнее](http://www.kollective.com).
 
- **Пандус омникаче** обеспечивает сетевое распространение нового поколения и обеспечивает плавную доставку видеосодержимого по глобальным глобальным сетям, помогая сотрудникам оптимизировать пропускную способность сети и поддерживать успешные трансляции событий и по запросу. материалы. Поддержка пандус Омникаче для событий Live, созданных в Teams, ожидается в ближайшее время. [Подробнее](http://www.ramp.com). 
 
> [!NOTE] 
> Выбранное решение SDN или Екдн зависит от **условий обслуживания и положения о конфиденциальности**, которые будут управлять использованием решения поставщика. Использование решения поставщика не распространяется на условия лицензионного соглашения на доступ к корпоративным лицензиям Microsoft или онлайн-служб. Если вы не согласны с условиями стороннего **поставщика**, не включайте решение в Teams. 

После настройки решения SDN или Екдн вы можете настроить поставщик для событий Live Events в Teams.
