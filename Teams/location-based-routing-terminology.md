# <a name="location-based-routing-terminology"></a>Терминология маршрутизации на основе расположения

> [!INCLUDE [Preview customer token](includes/preview-feature.md)] 

Ниже приведено несколько терминов и концепций, которые используются во время работы с документацией на основе местоположения. Перед тем как приступить к работе с документацией, рекомендуется ознакомиться с этими положениями и концепциями.

|Термин  |Описание  |
|---------|---------|
|Регионы сети     | Область сети содержит коллекцию сетевых сайтов. Например, если ваша организация содержит много сайтов в Индии, вы можете указать "Индия" в качестве сетевого региона.        |
|Сетевые сайты    | Сетевой сайт — это расположение, в котором Организация имеет физическое место проведения, например Office, набор зданий или кампус. Сетевые сайты определяются как набор IP-подсетей. Лучшим вариантом для маршрутизации на основе местоположения является создание отдельного сайта для каждого местоположения с уникальной связью по протоколу PSTN.  Каждый сетевой сайт должен быть связан с сетевым регионом. Вы можете создать сайт, для которого включена маршрутизация на основе местоположения, или сайт, для которого не включена маршрутизация на основе местоположения. Например, вы можете создать сайт, для которого не включена маршрутизация на основе расположения, чтобы предоставить пользователям, которые разрешено использовать маршрутизацию на основе местоположения, звонить по коммутируемой телефонной связи, когда они перемещаются на этот сайт. Обратите внимание, что сетевые сайты также могут использоваться для включения и настройки вызова экстренной помощи.        |
|Сетевые подсети     |IP-подсети в расположении, в котором конечные точки Teams могут подключаться к сети, должны быть определены и связаны с определенной сетью для обеспечения бесплатного обхода. Несколько подсетей могут быть связаны с одним и тем же сетевым сайтом, но несколько сайтов не могут быть связаны с одной и той же подсетью. Эта ассоциация подсетей позволяет расположить конечные точки на основе местоположения, чтобы определить, следует ли разрешить данный звонок по протоколу PSTN. Поддерживаются обе подсети IPv6 и IPv4. При определении того, находится ли конечная точка Teams на сайте, маршрутизация на основе местоположения сначала проверяет соответствующий IPv6-адрес. Если IPv6-адрес отсутствует, маршрутизация на основе местоположения проверяет адрес IPv4. <br><br>
|Доверенные внешние IP-адреса    |Доверенные внешние IP-адреса — это внешние IP-адреса в сети Организации. Они определяют, входит ли конечная точка пользователя в корпоративную сеть, прежде чем проверять соответствие определенного сайта. Если внешний IP-адрес пользователя соответствует указанному в списке надежных, IP-адреса, для определения внутренней подсети, в которой находится конечная точка пользователя, проверяется маршрутизация на основе местоположения. Если внешний IP-адрес пользователя не соответствует ни одному IP-адресу, определенному в списке доверенных, конечная точка классифицируется как неизвестная, и любые вызовы по КТСОП для пользователя, для которого включена маршрутизация на основе местоположения, заблокированы.          |
