# <a name="choose-your-upgrade-journey-from-skype-for-business-to-teams"></a>Выберите путь обновления от Skype для бизнеса до Teams

![Схема пути обновления с акцентом на Project определения.](media/upgrade-banner-project-definition.png "Этапы пути обновления с акцентом на этапе Project определения")

Эта статья является частью этапа определения Project пути обновления. Прежде чем при этом подтверждать, что вы выполнили следующие действия:

- [Привлечение заинтересованных лиц по проекту](upgrade-enlist-stakeholders.md)
- [Определение области проекта](./upgrade-define-project-scope.md)
- [Понятное сосуществование и совместное Skype для бизнеса и Teams](./teams-and-skypeforbusiness-coexistence-and-interoperability.md)

Полный переход Skype для бизнеса на Teams может занять некоторое время. Однако вы можете начать понимать значение Teams, позволяя пользователям использовать Teams вместе с Skype для бизнеса. Поскольку между двумя приложениями есть некоторые перекрывающиеся функции, рекомендуем ознакомиться с доступными режимами сосуществования и обновления, чтобы определить, какой путь будет правильным для вашей организации. Например, вы можете включить все рабочие нагрузки для обоих решений без возможности обеспечения одновременной работы. Вы также можете управлять пользовательским интерфейсом, постепенно Teams возможности или путем выбора групп пользователей, пока ваша организация не будет готова обновить их до Teams. Используйте результаты пилотного проекта, чтобы оценить путь обновления для вашей организации.

> [!IMPORTANT]
> Поддержка Skype для бизнеса Online будет прекращена 31 июля 2021 г., после чего эта служба больше не будет доступна. Чтобы максимально расширить возможности преимуществ и у вашей организации есть надлежащее время на реализацию обновления, мы рекомендуем начать свой путь к Microsoft Teams сегодня.

В этой статье описаны различные режимы, позволяющие управлять тем, Skype для бизнеса и Teams доступны для пользователей. Как и при любом развертывании, мы настоятельно рекомендуем вам пилотный план с выбранной группой пользователей перед обновлением организации до Teams. [](pilot-essentials.md) Помните, что внедрение новых технологий может нарушить работу пользователей. У вас есть время на оценку готовности пользователей и реализацию плана коммуникации и обучения перед внедрением любого из описанных здесь режимов.

## <a name="upgrade-journey-building-blocks"></a>Обновленные блоки пути

Чтобы официально подготовить организацию к Teams, необходимо приступить к планированию сценариев обновления, которые в конечном итоге позволит вашей организации полностью Teams в качестве единого решения для коммуникации и совместной работы.

Чтобы помочь в принятии решений, ознакомьтесь с различными режимами, понятиями и терминами, которые необходимы для перехода с Skype для бизнеса до Teams. Дополнительные сведения см. в Microsoft Teams и Skype для бизнеса сосуществования и [совместной работы.](./teams-and-skypeforbusiness-coexistence-and-interoperability.md)

> [!NOTE]
> Также рассмотрите сценарии переноса голосовой почты. телефонная система технологией Майкрософт для включения управления звонками и личных ветвей Exchange (УАПС) в Microsoft 365 или Office 365 облаке. Для телефонная система телефонной сети общего звонков (STN), чтобы пользователи могли звонить по всему миру, у вас есть параметры, основанные на деловых потребности. Дополнительные сведения о параметрах подключения телефонная система ДНР и ДНР см. в телефонная система голосовой связи и подключения через [STN.](cloud-voice-landing-page.md)

Пользователь, перенесенный в Teams больше не использует клиент Skype для бизнеса, кроме присоединиться к собранию, которое Skype для бизнеса. Все входящие чаты и звонки отправляются в клиент Teams пользователя, независимо от того, Teams или Skype для бизнеса. Все новые собрания, организованные обновленным пользователем, будут запланированы как Teams собрания. Если пользователь пытается использовать клиент Skype для бизнеса, начало чатов и звонков блокируется<sup>1</sup>. Однако пользователь может (и должен) по-прежнему использовать клиент Skype для бизнеса, чтобы присоединяться к собраниям, на которые он приглашен.

Администраторы управляют переходом на Teams с помощью концепции [режима](migration-interop-guidance-for-teams-with-skype.md), которое является свойством [TeamsUpgradePolicy.](/powershell/module/skype/grant-csteamsupgradepolicy?view=skype-ps&preserve-view=true) Пользователь, перенесенный в Teams как описано выше, находится в режиме TeamsOnly. В организации, которая переходить на Teams, конечная цель — переместить всех пользователей в режим TeamsOnly.

Существует два способа переноса существующей организации с Skype для бизнеса (как локальной, так и локальной) в Teams:

- **Метод перекрывающихся** возможностей (в режиме "О-ва"): пользователи существующей организации Skype для бизнеса впервые используют Teams, чтобы они могли использовать оба клиента одновременно во время поэтапного этапа. В течение этого периода для них доступна большая часть функций Teams, но не все. Режим этой конфигурации называется "Острова", и этот режим по умолчанию заданная для всех организаций, в которых есть Skype для бизнеса. Когда организация будет готова, администратор перемещает пользователей в режим TeamsOnly.

- Метод выбора возможностей **(с** помощью одного или Skype для бизнеса режимов): администратор управляет переходом (с Skype для бизнеса на Teams) чата, звонков и планирования собраний для пользователей в организации. Каждая из этих функций доступна в Skype для бизнеса или Teams, но не обе. Администраторы используют TeamsUpgradePolicy для управления переносом этой функции Teams пользователями. Пользователи, которые еще не используют режим TeamsOnly, по-прежнему используют Skype для бизнеса чата и звонков, и эти два набора пользователей могут общаться посредством взаимодействия. Администраторы управляют переходом, постепенно перенося больше пользователей в режим TeamsOnly.

<sup>1</sup> Старые Skype для бизнеса, которые были отправлены до 2017 г., не взимали на TeamsUpgradePolicy. Убедитесь, что вы используете последнюю версию клиента Skype для бизнеса, доступную в Office канале.

После того как вы поймете и выберите способ обновления, вы сможете узнать о средствах для управления обновлением до [Teams.](upgrade-to-teams-on-prem-tools.md)

## <a name="overlapping-capabilities-method-using-islands-mode"></a>Метод перекрывающихся возможностей (в режиме "Острова")

При использовании перекрывающихся возможностей пользователи могут использовать как клиенты Teams, так и Skype для бизнеса чата, вызовы VoIP и собрания. При этом способе чат и вызовы VOIP в Teams фокусируется на внутренней организации, а Skype для бизнеса позволяет общаться с внешними организациями и звонить по voIP/ПСN (при их настройке). Собрания могут быть запланированы и посещаться в обоих продуктах.

При использовании метода перекрывающихся возможностей коммуникационный трафик для Skype для бизнеса и Teams остается самостоятельным (даже для одного пользователя), а два разных клиента никогда не общаются друг с другом (для пользователей в одной организации). Пользовательские интерфейсы основаны на конфигурации получателя. Например, предположим, что пользователь А находится в режиме "О-ва":

- Общение, иници Skype для бизнеса клиенте другого пользователя, всегда будет в клиенте Skype для бизнеса пользователя A.

- Общение, иници Teams клиента в  той же организации, всегда будет по-Teams пользователя A.

- Взаимодействие, иници Teams клиента  из внешней организации, всегда будет по-Skype для бизнеса пользователя A.

Если вы наизировали лицензию Microsoft 365 или Office 365 пользователям, это будет стандартный режим обновления для вашей организации. При назначении лицензии Microsoft 365 или Office 365 лицензии на Teams и Skype для бизнеса Online назначаются по умолчанию. <sup>2</sup>

Чтобы этот способ работал эффективно, все пользователи должны запускать оба клиента одновременно. Входящие чаты и звонки из организации пользователю в режиме "Острова" могут перенаться в клиент Skype для бизнеса или Teams, и это не контролируется получателем. Это зависит от того, какой клиент использует отправитель для начала связи. Если отправитель и получатель находятся в разных организациях, входящие звонки и чаты пользователю в режиме "Острова" всегда находятся в Skype для бизнеса клиенте.

Например, если получатель из режима "О-ва" в Skype для бизнеса, но не Teams, но получил сообщение от Teams, получатель в режиме "Острова" не увидит сообщение (но в конечном итоге получит сообщение о пропущенном сообщении в Teams). Аналогично, если пользователь работает в Teams, но не Skype для бизнеса, а кто-то Skype для бизнеса, он не увидит этот чат. Поведение в каждом из этих случаев аналогично звонкам. Если пользователи не запускают оба клиента, это может привести к проблеме.

При использовании этого метода Teams и Skype для бизнеса функции присутствия. Это означает, что другие пользователи могут видеть разные состояния присутствия для пользователя A в зависимости от того, какой клиент они используют. Дополнительные сведения см. в [сведениях о присутствии.](teams-and-skypeforbusiness-coexistence-and-interoperability.md#presence)

- Другие пользователи, использующие Teams, будут видеть данные о присутствии в зависимости от действий пользователя А в Teams.

- Другие пользователи, использующие Skype для бизнеса, будут видеть присутствие в зависимости от действий пользователя A в Skype для бизнеса.

Когда вы будете готовы перейти в режим TeamsOnly, вы можете обновить пользователей по отдельности или сразу весь клиент с помощью политики для всего<sup>клиента 3.</sup> После обновления пользователя до режима TeamsOnly он получает все входящие чаты и звонки в Teams. (Обратите внимание, что перенос Skype для бизнеса собраний в Teams запускается только при применении TeamsUpgradePolicy к отдельным пользователям, а не на уровне клиента. Подробные [сведения см. в](upgrade-to-teams-on-prem-tools.md#meeting-migration) переносе собраний.)

Однако получатели, не обновленные в режиме "Острова", могут продолжать получать чаты и звонки от пользователя TeamsOnly в своих клиентах Skype для бизнеса или Teams. В существующих беседах пользователь TeamsOnly отвечает клиенту, от кого отправитель инициировал чат или звонок. 

Для новых бесед с точки зрения пользователя TeamsOnly чат или звонок всегда будут перенаходить в режиме "Teams". Это потому, что клиент Teams ведет отдельные беседы для Teams общения Teams и Teams-связи Skype для бизнеса даже для одного пользователя. Дополнительные информации см. в Teams [Беседы — interop и native threads](teams-and-skypeforbusiness-coexistence-and-interoperability.md#interoperability).

## <a name="select-capabilities-method-using-skype-for-business-modes"></a>Выбор метода возможностей (Skype для бизнеса режимов)

В некоторых организациях по мере перехода с Skype для бизнеса на Teams пользователи могут сделать работу пользователей более предсказуемой. В этой модели ИТ-администраторы используют один из режимов Skype для бизнеса в TeamsUpgradePolicy, чтобы явно узначить, какие возможности остаются в Skype для бизнеса до перехода в режим TeamsOnly. Когда они готовы переключить выбранные возможности в режим TeamsOnly, администратор обновляет этот режим для этих пользователей до TeamsOnly. Во время перехода:

- Администраторы могут включить определенные возможности Teams пользователям, сохраняя возможности чата и звонков в Skype для бизнеса до того, как они будут переходить на teamsOnly. Администрирование может Teams возможности совместной работы или Teams собрания и возможности совместной работы.

- Пользователи по-прежнему Skype для бизнеса получать все входящие чаты и звонки в клиенте Skype для бизнеса, независимо от того, было ли сообщение Teams или Skype для бизнеса клиента другого пользователя. Кроме того, для Skype для бизнеса пользователей функции звонков и чата в клиенте Teams отключены, чтобы избежать путаницы с конечными пользователями, а также обеспечить правильную маршрутику и присутствие.

- Пользователи в режиме TeamsOnly получают все входящие чаты и звонки в клиенте Teams, а их присутствие предоставляется Teams, независимо от того, откуда было взаимодействие: Teams, Skype для бизнеса или федеративный пользователь.

В отличие от метода перекрывающихся возможностей (Islands), при выборе метода возможностей пользователи, использующие Skype для бизнеса могут общаться с пользователями в TeamsOnly. Взаимодействие между Skype для бизнеса и Teams называется взаимодействием [или](teams-and-skypeforbusiness-coexistence-and-interoperability.md#interoperability) взаимодействием. Взаимодействие между пользователем в Skype для бизнеса и другим пользователем в Teams возможно на индивидуальной основе. Кроме того, приглашенные пользователи всегда могут присоединиться к собранию Skype для бизнеса или Teams, но им необходимо использовать клиент, соответствующий типу собрания. Дополнительные сведения см. в [теме Собрания.](teams-and-skypeforbusiness-coexistence-and-interoperability.md#meetings)

Для пользователей, которые используют метод перехода с выбранных возможностей, присутствие для пользователя является согласованным независимо от того, какой клиент используется другим пользователем. Если пользователь находится в одном из режимов Skype для бизнеса, все остальные пользователи видят данные о присутствии в зависимости от его действий в Skype для бизнеса. Аналогичным образом, если пользователь находится в режиме TeamsOnly, все остальные пользователи видят данные о присутствии в зависимости от его действий в Teams. Подробные сведения см. в [сведениях о присутствии.](teams-and-skypeforbusiness-coexistence-and-interoperability.md#presence)

Если организация еще не начала использовать Teams, администратор должен изменить режим клиента с "Острова" на SfbWithTeamsCollab. (Для организаций, у которых уже есть некоторые Teams, администратор должен "задействовать" пользователей, уже Teams, чтобы это изменение не применялась к ним. Дополнительные сведения см. в примере Метод выбора возможностей для организации, которая уже использует Teams [в режиме "Острова".)](upgrade-to-teams-on-prem-implement.md#a-select-capabilities-upgrade-for-an-organization-that-is-already-using-teams-in-islands-mode) 

Пользовательские интерфейсы гостей будут соответствовать режиму совместной работы, назначенном для клиента. Если вы настроите Skype для бизнеса на уровне клиента, гости не смогут общаться, звонить и показывать свои присутствии. Дополнительные сведения см. в статье [Гостевой доступ в Teams](guest-access.md).

Когда режим изменится с "О-ва" на SfbWithTeamsCollab, пользователь, который никогда не использовал Teams, не увидит различий в способах Skype для бизнеса. Однако если пользователь начнет использовать Teams, он будет работать только с такими функциями, как Teams & Channel и Files. Чат, вызовы и планирование собраний будут недоступны в Teams, так как администратор назначил Skype для бизнеса клиентом для этих функций.

> [!NOTE]
> Когда пользователь A изменяется с Islands на один из режимов Skype для бизнеса, клиент Teams другого пользователя, который общается с пользователем A, должен знать, что режим пользователя А изменен, чтобы он перенаналил связь с соответствующим клиентом для пользователя A. Для всех пользователей, которые уже наладили чаты Teams-к-Teams с пользователем A, клиентам Teams этих пользователей может быть известно о том, что режим изменится с "Острова" на любой режим Skype для бизнеса. В то же время изменения, внесенные существующим пользователем в режим TeamsOnly, будут выявлены другими клиентами в течение 2 часов.<br>
> Когда администраторы будут готовы, они могут перенести чат, вызовы и планирование собраний для определенного пользователя на Teams сразу, обновив его режим до TeamsOnly.

Кроме того, администратор может сначала переключить только планирование собрания на Teams, покидая функции чата и звонков в Skype для бизнеса в режиме SfBWithTeamsCollabAndMeetings. Этот режим позволяет организациям перейти Teams к собраниям, если пользователи еще не готовы к переходу в режим TeamsOnly (например, если вы еще не готовы перенести существующие функции STN). Этот сценарий называется Первым [собранием.](meetings-first.md)
