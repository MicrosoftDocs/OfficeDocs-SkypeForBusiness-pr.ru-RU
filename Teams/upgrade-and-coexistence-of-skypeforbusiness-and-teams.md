# <a name="choose-your-upgrade-journey-from-skype-for-business-to-teams"></a>Выбор обновления в Skype для бизнеса в Teams

Microsoft Teams в конечном итоге заменит Skype для бизнеса Online, поэтому вам нужно будет планировать работу команд так, чтобы вы были готовы к работе.

Если вы уже зарегистрированы в Skype для бизнеса, ваш переход на Teams может занять некоторое время. Тем не менее, вы можете приступить к реализации рабочих групп прямо сейчас, разрешив пользователям использовать Teams вместе с Skype для бизнеса. С учетом того, что между двумя приложениями есть несколько перекрывающихся функциональных возможностей, мы рекомендуем проверить доступные режимы сосуществования и обновления, чтобы определить, какой путь является наиболее подходящего для вашей организации. Например, вы можете отказаться от использования всех рабочих нагрузок в обоих решениях без взаимодействия. Кроме того, вы можете настроить взаимодействие с пользователем, постепенно вводя возможности Teams или нацеливания на группы пользователей для выбора возможностей, пока ваша организация не будет готова к обновлению до Teams. Используй свой пилотный проект, чтобы оценить подходящего пути обновления для вашей организации.

> [!IMPORTANT]
> В этой статье описываются различные режимы, с помощью которых вы можете управлять тем, какие модальностей в Skype для бизнеса и Teams доступны для пользователей. Как и в случае с развертыванием, мы настоятельно рекомендуем вам предварительно выпилотовать [запланированный план](pilot-essentials.md) с выбранной группой пользователей перед обновлением вашей организации до Teams. Помните о том, что введение новых технологий может привести к нарушению работы пользователей. Ознакомьтесь со временем о том, как оценить готовность пользователей и внедрить план общения и обучения, прежде чем приступать к реализации описанных выше способов.

## <a name="upgrade-journey-building-blocks"></a>Обновление стандартных блоков для поездки

Для формальной подготовки Организации к его путешествием в Teams необходимо приступить к планированию сценариев обновления, которые в конечном итоге позволят вашей организации полностью поддерживать группы в рамках собственного решения для связи и совместной работы.

Для упрощения процесса принятия решений ознакомьтесь с различными режимами, концепциями и терминологией, относящимися к переходу с Skype для бизнеса на Teams. Дополнительные сведения можно найти в разделе [Microsoft Teams и сосуществование и взаимодействие Skype для бизнеса](https://aka.ms/SkypeToTeams-Coexist)

Когда некоторые пользователи будут готовы к использованию только групп для повседневных задач общения и совместной работы, вы можете приступить к обновлению этих пользователей в Teams, включив для них режим " **только** для Teams".

Если вы не хотите, чтобы ваша организация могла перейти на Teams, вы можете начать работу с Teams в режиме совместного существования в Skype для бизнеса. **** Как дополнительные режимы сосуществования (например, **Skype для бизнеса с совместная работа в Teams** и **Skype для бизнеса с совместная работа и собрания**в Teams) будут последовательно доступны в течение ближайших месяцев, вы также можете начать работу полностью Сначала разработайте группы в качестве решения для совместной работы с группами, сохранив Skype для бизнеса в качестве решения для единой системы обмена сообщениями в Организации. Это рекомендуемый вариант для пользователей, использующих Skype для бизнеса Server (локальный или гибридный), и пользователи с значительной сложностью, тражектори в Teams, в состав которой входит длительный период сосуществования.

## <a name="upgrade-journeys"></a>Переход на новую версию

Вы можете использовать несколько подходов к обновлению из Skype для бизнеса, как в Интернете, так и локально в teams:

- При прямом переходе вы сначала разместите группы вместе со Skype для бизнеса в режиме " **острова** " в рамках оценки и раннего внедрения, а затем обновите свои пользователи до режима " **только** для Teams", чтобы получить возможность быстрого снятия Skype для бизнеса с сайта среда для всех пользователей в Организации. Это рекомендуемое путешествие для пользователей Skype Online, если только они не связаны с тем, что их пользователи не будут запутаться, если у них есть два средства для выполнения одного и того же действия (чат, Звонок, планирование собраний).
- Плавные переходы между процессами доставляют конкретный режим сосуществования и обновления для определенной группы пользователей (также называемых *кохорт*) в зависимости от требований к общению и совместной работе. Со временем вся Организация может присоединиться только к Teams и в конечном итоге заменить Skype для бизнеса. Тем не менее, если ваша организация имеет привлекательные бизнес-причины для поддержания Skype для бизнеса (например, зависимость от управляемого интерфейса API УКМА), которая интегрируется с бизнес-приложениями, или решением безупречной стены в настоящее время поддерживается только для Skype для бизнеса или сложное корпоративное развертывание, которое может занимать только время перехода на **Teams**(только для тех, кто **** хранит пользователи Skype для бизнеса) в одной из режимы сосуществования для части пользовательского населения. Плавный переход на новую версию — это рекомендуемый подход для локальных (и гибридных) пользователей, начинающихся с Skype для бизнеса с помощью режима совместной работы в Teams и переход из него в режим "только в Teams", если требование для пользователей будет выполнено (возможно, с помощью Skype для бизнеса с совместной работой в Teams и режимом сосуществования собраний.

> [!IMPORTANT]
> Если в настоящее время в вашей организации есть только локальное развертывание Skype для бизнеса, вам нужно сначала приступить к планированию внедрения гибридной версии Skype для бизнеса перед обновлением пользователей в режиме " **только** для Teams". Это также поможет вам облегчить взаимодействие с Teams.

> [!NOTE]
> Режим " **только** для Teams" требует, чтобы пользователи, являющиеся частью кохортс, были размещены в Skype для бизнеса Online, а Гибридная связь между локальным развертыванием Skype для бизнеса и вашим клиентом Skype для бизнеса Online — требуется для упрощения взаимодействие между Skype для бизнеса и рабочими группами. Переход в Skype для бизнеса Online должен быть выполнен для пользователей, которые являются частью кохортс, прежде чем они будут обновлены до режима " **только для Team** ". Skype для бизнеса Server 2019 и Skype для бизнеса Server 2015 с помощью обновления HF1 Update позволяет упростить модернизацию локальных пользователей до Teams, управляя переходом в Skype для бизнеса Online и обновляя пользователей в режиме "только для **Teams** " за один шаг. .

### <a name="direct-upgrade-journey"></a>Прямое обновление

Прямое переходное путешествие показано на следующей схеме.

![Снимок экрана: прямая модернизация] (media/upgrade_journey_direct_upgrade.png "Снимок экрана: путешествие прямо по обновлению. Все пользователи изначально используют Teams в режиме \"острова\", а затем переход в режим \"только для Teams\", в котором конечное состояние всей Организации было обновлено до Teams.")

Teams разворачивается для всех пользователей в Организации и настраивается в режиме " **острова** ". Если ваша организация определяет готовность Teams к удовлетворению ваших потребностей в общении и совместной работе, извещать пользователей и обновите их до режима " **только для Team** ". В этот момент Skype для бизнеса может быть закрыт из среды.

### <a name="gradual-upgrade-journey"></a>Плавный переход

Пример постепенного перехода на обновленный объект показан на следующей схеме.

![Пример плавных переходов] (media/upgrade_journey_gradual_upgrade.png "При постепенном переходе на обновление кохортс пользователей изначально использует Teams в режиме острова для оценки, а затем в различных режимах обновления для раннего внедрения и параллельной работы в Skype для бизнеса. Некоторые кохортс переходы в режим \"только для Teams\", в то время как одна группа пользователей остается в Skype для бизнеса с помощью режима совместной работы и собраний Teams.")

Teams разворачивается в Организации в режиме **острова** для оценки, а затем переходите к различным режимам сосуществования и обновления для разных групп пользователей. Например, группа пользователей может быть включена в режим " **острова** ", другой включенный для **Skype для бизнеса с режимом совместной работы и собраний** , а третья группа пользователей — изначально разрешена для **Skype для бизнеса с помощью Teams. режим "только совместная работа** ".

С течением времени группы пользователей можно обновить только в режиме " **только** для Teams", а затем в оставшейся части Организации. В конечном итоге вся Организация будет готова к изъятию Skype для бизнеса и использовать только команды для общения и совместной работы, а также — если бизнес-требования определяют, что Skype для бизнеса будет храниться для определенной группы — большинство пользователей в Организация может использовать только Teams. <br><br>
