# <a name="choose-your-upgrade-journey-from-skype-for-business-to-teams"></a>Выбрать обновления реализация Скайп для бизнеса для групп

Как существующие Скайп для клиента Business завершения перехода к группам может занять некоторое время. Тем не менее можно начать в настоящее время осознавать значение группам, что позволяет пользователям использовать команды наряду с Скайп для бизнеса. Что есть некоторые функциональные возможности перекрывающиеся между двумя приложениями, рекомендуется просмотреть доступные сосуществования и обновление режимы, чтобы определить, какой путь для своей организации. К примеру может выбрать для включения всех рабочих нагрузок на обоих решения без взаимодействия. Или можно управлять удобство работы пользователей с постепенно краткие сведения о возможности групп или ориентация групп пользователей для выбора возможностей, пока не будет готово к обновлению все команды вашей организации. Используйте результат в пилотную среду для оценки прямо обновления пути для вашей организации.

> [!IMPORTANT]
> В этой статье описываются различные режимы, которые позволяют управлять какие модальностей Скайп для бизнеса и команды доступны для пользователей. Как и в любом развертывании мы настоятельно рекомендуем вам для [Пилотный проект плана предполагаемая](pilot-essentials.md) с выбранной группы пользователей перед обновлением вашей организации по группам. Помните, что краткие сведения о новых технологий может быть аварийные для пользователей. Потратьте время оценить готовность пользователей и реализовывать обмена данными и плана обучения до внедрения какие-либо из режимов, описанные здесь.
 
## <a name="upgrade-journey-building-blocks"></a>Стандартные блоки пути обновления
Чтобы официально подготовить пути к группам в вашей организации, необходимо начать планирование сценарии обновления, которые со временем позволит вашей организации полностью принять группами как единственным коммуникаций и совместной работы решения.

Для помощи в процесс принятия решений, ознакомьтесь с различные режимы, принципы и терминологию, предназначенных для обновления Скайп для бизнеса до группами. Для получения дополнительных сведений см [Microsoft групп и Скайп для бизнеса сосуществования и взаимодействия](https://aka.ms/SkypeToTeams-Coexist)

Если некоторые пользователи все готово для использования только группы их повседневного общения и совместной работы потребностям, можно начать обновление этих пользователей к группам, включив режим **группами только** для них.

Если это не возможно в организации для перемещения по группам, можно начать пилотной группы наряду с Скайп для бизнеса в режиме сосуществования **острова** . Когда доступны дополнительные сосуществования режима, такие как **Скайп для бизнеса с совместной работы групп** и **Скайп для бизнеса с собраний и совместной работы групп**, также можно начать можно полностью принять группами как совместной работы групп решения для коммуникаций единой решения сначала сохраняя Скайп для бизнеса в вашей организации. Другая возможность — это переместить собрания групп, в дополнение к краткие сведения о возможности совместной работы группы команд, сохраняя остальные возможности объединенных коммуникаций в Скайп для бизнеса.

## <a name="upgrade-journeys"></a>Обновление поездок
Вы можете воспользоваться различные подходы к обновлению Скайп для бизнеса, либо через Интернет или локальной, к группам:
- В прямого пути обновления, сначала развернуть группы наряду с Скайп для бизнеса в режиме **острова** как часть оценки и раннее внедрение, а затем обновите пользователей в режим **только группы** с целью быстро закончится Скайп для бизнеса из среда для всех пользователей в организации.
- Постепенного обновления реализация доставляет конкретных сосуществования и обновления режим определенные группы пользователей (также называемая *cohort*), в зависимости от их коммуникаций и требованиям к совместной работе. Со временем во всей организации могут для достижения в только с помощью команды и в конечном счете замените Скайп для бизнеса. Тем не менее если в вашей организации есть убедительных причин business хранения Скайп для бизнеса, такие как зависимость от решения на основе Unified Communications Managed API UCMA, интегрированного с бизнес приложения или решения на базе корпоративными стандартами в настоящее время недоступен для Скайп для бизнеса только — большинство пользователей можно выполнить обновление до режима **только группы** сохраняя Скайп для бизнес-пользователей в одном из режимов сосуществования части остальных пользователей.

> [!IMPORTANT]
> Для обоих типов пути обновления Если вашей организации в настоящий момент Скайп Business локальной только для развертывания, необходимо начать планирование для реализации Скайп для бизнеса гибридной среды перед обновлением до режима **только групп** пользователей. Это также поможет улучшения взаимодействия с группами.
> Используйте [MyAdvisor](https://myadvisor.fasttrack.microsoft.com/) для вашей Скайп для реализации гибридных бизнеса.

> [!NOTE]
> **Только группы** режима требуется, что пользователи, которые входят в состав последователей быть размещен в Скайп для бизнеса в Интернет и гибридного отношение между вашей Скайп для локального развертывания Business и вашей Скайп для бизнеса интерактивного клиента является обязательным для облегчения взаимодействие между Скайп для бизнеса и рабочих групп. Должен быть выполнен переход на Скайп для бизнеса в Интернет для пользователей, которые входят в состав последователей до обновления в режим **только для групп** . Скайп для Business Server 2019, а также будущих накопительный пакет обновлений за Скайп для Business Server 2015, планирование для упрощения механизм обновление локальных пользователей группам, управление миграции для Скайп для бизнеса в Интернет и обновление до **групп пользователей только** режим в один шаг.

### <a name="direct-upgrade-journey"></a>Прямое пути обновления
На следующей схеме показано прямого пути обновления.

![Снимок экрана прямого пути обновления. Все пользователи изначально использовать команды в режиме острова, а затем переход в режим только для групп, с состоянием конечных во всей организации, обновленных до группами.](media/upgrade_journey_direct_upgrade.png)

Группы развернуты для всех пользователей в организации и настроен в режиме **острова** . Когда ваша организация определяет, что команды готова для выполнения всех коммуникаций и совместной работы потребностям, все пользователи будут обновлены в режим **только для групп** . На этом этапе можно не поддерживается Скайп для бизнеса из среды.

### <a name="gradual-upgrade-journey"></a>Реализация постепенного обновления
На следующей схеме показан пример постепенного обновления пути.

![Постепенное обновление результат последователей пользователей изначально используется группами в режиме острова для оценки, а затем в различные режимы обновления для Раннее внедрение рядом с Скайп для бизнеса. Некоторые последователей переход в режим только для групп, а по одной группе пользователей остается Скайп для бизнеса с помощью режима собраний и совместной работы групп.](media/upgrade_journey_gradual_upgrade.png)

Группы развертывания в организации в режиме **острова** для вычислений и затем перетащить в разных сосуществования и обновите режимов для разных групп пользователей. Например группы пользователей, которые могут быть включены для режима **острова** , другой включена для режима **Скайп для бизнеса с собраний и совместной работы групп** , пока третий группы пользователей изначально может быть включена для **Скайп для бизнеса с группами Совместная работа только** режим.

Со временем в режим **только группы** , а затем остальной части организации может быть обновлена групп пользователей. В конечном итоге во всей организации будет готов к удаление Скайп для бизнеса и использовать только группы коммуникаций и совместной работы, или, если бизнес-требований диктуют необходимость сохранения Скайп для бизнеса определенной группе — большинство пользователей в Организация может использовать только группы. <br><br>
