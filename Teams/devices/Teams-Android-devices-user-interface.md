# <a name="set-microsoft-teams-android-devices-user-interface"></a>Настройка пользовательского интерфейса для устройств с Android в Microsoft Teams

Microsoft Teams Устройства Android могут отображать определенный пользовательский интерфейс в зависимости от типа лицензии, назначенной учетной записи, подписанной. Вы можете переопрепредить это поведение и контролировать, какой интерфейс показан. В этой статье подробно по умолчанию выбран пользовательский интерфейс и как можно изменить интерфейс с помощью политики Powershell.

Существует три типа пользовательских интерфейсов на устройствах Teams Android:

1. User
2. Общая область
3. Собрание

Если [](/microsoftteams/user-access) назначить учетной записи лицензию пользователя, например лицензию E3 или E5, устройство Teams отображает интерфейс конечных пользователей по умолчанию, который полностью показан для большинства сценариев пользователей. Однако если устройство выполняет определенную функцию, например общий телефон или зал собраний, для этих пользования существуют определенные пользовательские интерфейсы.

На следующих трех изображениях покажите, как меняется пользовательский интерфейс на основе лицензии, назначенной учетной записи пользователя. На первом изображении учетной записи пользователя назначена лицензия E5. Это лицензия пользователя, поэтому устройство показывает интерфейс конечных пользователей по умолчанию:

:::image type="content" source="../media/TeamsAndroidDevices-UserMode1.jpg" alt-text="Интерфейс пользовательского режима":::

На этом изображении учетной записи пользователя была назначена общая лицензия [телефона области](/microsoftteams/set-up-common-area-phones). Общие телефоны области в основном используются для принятия и получения телефонных звонков. Таким образом, на дисплее отображается панель набора:

:::image type="content" source="../media/TeamsAndroidDevices-CAP1.jpg" alt-text="Интерфейс телефона общей области":::

Наконец, на этом изображении показана учетная запись пользователя с Комнаты Microsoft Teams [стандартной](/MicrosoftTeams/rooms/rooms-licensing) лицензией. Комнаты Teams лицензии предназначены для использования в залах собраний или в общих пространствах, поэтому пользовательский интерфейс изменяется, чтобы легко присоединиться к собранию, показывая представление календаря:

:::image type="content" source="../media/TeamsAndroidDevices-Meeting.jpg" alt-text="Интерфейс собрания":::

> [!NOTE]
> Изменение пользовательского интерфейса не влияет на возможность использования других лицензированных функций. Например, несмотря на то, что представление по умолчанию лицензии Team Rooms — это представление календаря, вы все равно можете звонить и получать телефонные вызовы общедоступных коммутаторов (PSTN), если учетная запись правильно лицензирована и настроена.

> [!IMPORTANT]
> Существуют другие элементы интерфейса, которые изменяются. Например, чтобы не допустить выхода конечных пользователей из общего телефона или устройства комнаты собраний, параметр "Выйти" на этих устройствах перемещается в часть меню параметров, которая требует разрешений администратора для доступа.

## <a name="override-automatic-user-interface-detection"></a>Переопределять автоматическое обнаружение пользовательского интерфейса

В некоторых случаях можно назначить лицензию учетной записи, которая не соответствует ее предназначению. Например, можно назначить пользовательской лицензии учетной записи, предназначенной для Комнаты Teams на Android. По умолчанию вместо интерфейса комнаты собраний будет видеться интерфейс конечного пользователя. Чтобы переопредить интерфейс по умолчанию, создайте новый Teams [ip-Телефон и](/powershell/module/skype/new-csteamsipphonepolicy?view=skype-ps) применяйте его к этой учетной записи.

> [!NOTE]
> Лицензия, назначенная учетной записи пользователя, должна иметь по крайней мере те же права лицензии, что и нужный пользовательский интерфейс. Лицензия common Area Телефон позволяет использовать только пользовательский интерфейс телефона общей области. Лицензия комнаты собраний позволяет использовать пользовательские интерфейсы телефонов общего зоны и комнаты собраний. Лицензия E3 или E5 поддерживает все режимы входов.

Ниже приводится пример переопределения автоматического обнаружения лицензии. В этом примере предположим, что учетной записи ресурса комнаты собраний с именем conf-adams@contoso.com была назначена лицензия E3. Когда эта учетная запись будет подписана, необходимо, чтобы пользователи увидели пользовательский интерфейс комнаты собраний.

### <a name="create-a-new-policy-and-assign-to-user"></a>Создание новой политики и назначение пользователю

1. Запустите сеанс удаленной Windows PowerShell и подключите Microsoft Teams с помощью следующего cmdlet:

    ``` Powershell
    Connect-MicrosoftTeams
    ```

2. Создайте новую политику Teams IP Телефон и установите режим входов в "MeetingSignIn":

   ``` Powershell
   New-CsTeamsIPPhonePolicy –Identity 'Meeting Sign in' –Description 'Meeting Sign In Phone Policy' -SignInMode 'MeetingSignIn'

   ```

3. Теперь эту новую политику можно назначить учетной записи ресурса комнаты собраний:

   ``` Powershell
   Grant-CsTeamsIPPhonePolicy –Identity 'conf-adams@contoso.com' –PolicyName 'Meeting Sign In'
   ```

После предоставления политики в учетную запись ресурса зала собраний необходимо дождаться репликации назначения политики. Кроме того, необходимо выйти из устройства и войти обратно.
