# <a name="set-microsoft-teams-android-devices-user-interface"></a>Настройка пользовательского интерфейса устройств Android в Microsoft Teams

Устройства Microsoft Teams android могут отображать определенный пользовательский интерфейс в зависимости от типа лицензии, назначенной учетной записи для входа. Вы можете переопределить это поведение и управлять отображаемым интерфейсом. В этой статье подробно описано, как выбирается пользовательский интерфейс по умолчанию и как его можно изменить с помощью политики PowerShell.

На устройствах Android Teams существует три типа пользовательских интерфейсов:

1. Пользователь
2. Общая область
3. Собрание

При [назначении](/microsoftteams/user-access) пользовательской лицензии учетной записи, например лицензии E3 или E5, устройство Teams отобразит пользовательский интерфейс по умолчанию, который полностью поддерживается в большинстве пользовательских сценариев. Однако если устройство выполняет определенную функцию, например телефон общего содержимого или конференц-зал, существуют определенные пользовательские интерфейсы для этих операций использования.

На следующих трех изображениях показано, как изменяется пользовательский интерфейс на основе лицензии, назначенной учетной записи пользователя. 

## <a name="end-user-interface"></a>Интерфейс конечного пользователя 

Учетной записи пользователя назначена лицензия E5. Это лицензия пользователя, поэтому на устройстве отображается пользовательский интерфейс по умолчанию:

:::image type="content" source="../media/teams-android-devices-usermode1.jpg" alt-text="Интерфейс пользовательского режима.":::

## <a name="common-area-interface"></a>Интерфейс общей области

На этом изображении учетной записи пользователя назначена [лицензия common Area Phone](/microsoftteams/set-up-common-area-phones). Телефоны общего пользования в основном используются для звонков и приема звонков. Таким образом, панель набора номера отображается на дисплее:

:::image type="content" source="../media/teams-android-devices-cap1.jpg" alt-text="Интерфейс телефона общей области.":::

## <a name="meeting-interface"></a>Интерфейс собрания

На этом изображении показана учетная запись пользователя с назначенной Комнаты Microsoft Teams [лицензией](/MicrosoftTeams/rooms/rooms-licensing). Комнаты Teams лицензии предназначены для использования в комнатах для собраний или в общих пространствах, поэтому пользовательский интерфейс изменяется, чтобы упростить присоединение к собранию с помощью представления календаря:

:::image type="content" source="../media/teams-android-devices-meeting.jpg" alt-text="Интерфейс собрания.":::

> [!NOTE]
> Изменение пользовательского интерфейса не влияет на возможность использования других лицензированных функций. Например, несмотря на то, что представлением по умолчанию лицензии Team Rooms является представление календаря, вы по-прежнему можете совершать и принимать телефонные звонки по телефонной сети общего пользования (ТСОП), если учетная запись имеет правильную лицензию и настроена.

> [!IMPORTANT]
> Существуют другие элементы интерфейса, которые изменяются. Например, чтобы пользователи не могли выйти из общего телефона или устройства комнаты для собраний, параметр "Выйти" на этих устройствах перемещается в часть меню параметров, для доступа к которой требуются разрешения администратора.

## <a name="override-automatic-user-interface-detection"></a>Переопределение автоматического обнаружения пользовательского интерфейса

В некоторых случаях вы можете назначить лицензию учетной записи, которая не соответствует предполагаемому использованию. Например, вы можете назначить лицензию пользователя учетной записи, предназначенной для входа Комнаты Teams Android. По умолчанию вместо интерфейса комнаты для собраний будет отображаться интерфейс конечного пользователя. Чтобы переопределить интерфейс по умолчанию, создайте политику [IP-телефонов Teams](/powershell/module/skype/new-csteamsipphonepolicy) и примените ее к этой учетной записи.

> [!NOTE]
> Лицензия, назначенная учетной записи пользователя, должна иметь по крайней мере те же права лицензии, что и требуемый пользовательский интерфейс. Лицензия common Area Phone позволяет использовать только пользовательский интерфейс телефона общего зоны. Лицензия на конференц-зал позволяет использовать пользовательские интерфейсы конференц-залов и телефонов общего зоны. Лицензия E3 или E5 поддерживает все режимы входа.

Ниже приведен пример переопределения автоматического обнаружения лицензий. В этом примере предположим, что учетной записи ресурса комнаты conf-adams@contoso.com назначена лицензия E3. При входе в эту учетную запись необходимо, чтобы пользователи могли видеть пользовательский интерфейс комнаты собраний.

### <a name="create-a-new-policy-and-assign-to-user"></a>Создание политики и назначение пользователю

1. Запустите сеанс удаленного Windows PowerShell и подключитесь к Microsoft Teams с помощью следующего командлета:

    ``` Powershell
    Connect-MicrosoftTeams
    ```

2. Создайте политику IP-телефона Teams и задайте для режима входа значение MeetingSignIn:

   ``` Powershell
   New-CsTeamsIPPhonePolicy –Identity 'Meeting Sign in' –Description 'Meeting Sign In Phone Policy' -SignInMode 'MeetingSignIn'

   ```

3. Теперь эту новую политику можно назначить учетной записи ресурса комнаты собраний:

   ``` Powershell
   Grant-CsTeamsIPPhonePolicy –Identity 'conf-adams@contoso.com' –PolicyName 'Meeting Sign In'
   ```

После предоставления политики учетной записи ресурса комнаты собраний необходимо дождаться репликации назначения политики. Вам также потребуется выйти из устройства и войти обратно.

## <a name="impact-on-microsoft-teams-admin-center"></a>Влияние на Центр администрирования Microsoft Teams

Центр администрирования Microsoft Teams позволяет управлять устройствами Microsoft Teams. Дополнительные сведения об управлении устройствами с помощью Центра администрирования Teams см. в статье "Управление устройствами [в Microsoft Teams"](device-management.md).


Центр администрирования Teams предоставляет возможность управления телефонами Teams. Телефоны фильтруются на одной из трех вкладок в зависимости от их функции: телефонов пользователей, телефонов общего пользования и конференц-связи. 

 :::image type="content" source="../media/teams-admin-center-phones-header.png" alt-text="Заголовок телефонов в Центре администрирования Teams.":::

Как и при обнаружении пользовательского интерфейса, телефоны Teams классифицируются на основе лицензии, назначенной учетной записи для входа на телефон. Например, если учетная запись, для которого назначена лицензия на телефон общего пользования, входит на телефон, этот телефон будет отображаться как в разделе "Все телефоны по умолчанию", так и в разделе "Телефоны общего **пользования".**

Если вы хотите, чтобы телефон отображался в другом разделе, можно назначить ему другую лицензию или создать и назначить политику IP-телефонов Teams, как описано [выше](#override-automatic-user-interface-detection).
