# <a name="microsoft-teams-analytics-and-reporting"></a>Аналитика и отчеты Microsoft Teams

Новый интерфейс аналитики и отчетов для Microsoft Teams доступен в Центре администрирования Microsoft Teams. Вы можете запускать различные отчеты, чтобы получить аналитические сведения о том, как пользователи в вашей организации используют Teams. Например, можно увидеть, сколько пользователей общаются через сообщения канала и чата, а также типы устройств, которые они используют для подключения к Teams. Ваша организация может использовать информацию из отчетов, чтобы лучше понять закономерности использования, помочь принимать бизнес-решения, а также информировать об обучении и коммуникации.

## <a name="how-to-access-the-reports"></a>Доступ к отчетам

Для доступа к отчетам необходимо назначить одну из следующих ролей:

- глобальный администратор.
- Teams или администратор Skype для бизнеса.
- Глобальное средство чтения (только агрегат на уровне клиента и без данных для каждого пользователя или команды).

Дополнительные сведения о ролях администраторов Teams и отчетах, к которым может получить доступ каждая роль администратора, см. в статье [Использование ролей администратора Teams для управления Teams](../using-admin-roles.md).

Перейдите в Центр администрирования Microsoft Teams, в области навигации слева выберите **Аналитика & отчетов**, а затем в разделе **Просмотр отчетов** выберите отчет, который нужно запустить.

> [!NOTE]
> Отчеты в Центре администрирования Microsoft Teams отделены от отчетов о действиях для Teams, которые являются частью отчетов Microsoft 365 в Центр администрирования Microsoft 365. Дополнительные сведения об отчетах о действиях в Центр администрирования Microsoft 365 см. [в разделе Отчеты Microsoft 365 в Центре администрирования](/microsoft-365/admin/activity-reports/activity-reports).

## <a name="make-the-user-specific-data-anonymous"></a>Сделать пользовательские данные анонимными

Чтобы сделать идентифицируемую информацию в отчетах об использовании анонимной, необходимо быть глобальный администратор. Глобальный администратор может скрывать идентифицируемые сведения (с помощью хэшей MD5), такие как отображаемое имя, имя группы, адрес электронной почты и идентификатор AAD, в отчете и его экспорте.

1. В Центр администрирования Microsoft 365 перейдите в **раздел Параметры** \> **организации** и на вкладке **Службы** выберите **Отчеты**.
    
2. Выберите **Отчеты**, а затем **— Отображение скрытых имен пользователей, групп и сайтов во всех отчетах**. Этот параметр применяется как к отчетам об использовании в Центр администрирования Microsoft 365, так и к Центру администрирования Teams.
  
3. Выберите **Сохранить изменения**.

> [!NOTE]
> Включение этого параметра приведет к отмене идентификации пользователей, групп и имени сайта в [отчете о действиях пользователей Teams](user-activity-report.md), [отчете об использовании устройств Teams](device-usage-report.md) и [отчете об использовании Teams](teams-usage-report.md). Начиная с 1 сентября 2021 г., этот параметр включен по умолчанию для всех в рамках нашего постоянного обязательства по защите важной информации и предоставления компаниям поддержки своих местных законов о конфиденциальности. 
>
>Этот параметр также применяется к отчетам об использовании Microsoft 365 в Центр администрирования Microsoft 365, Microsoft Graph и Power BI.
