# <a name="microsoft-teams-information-protection-license-report"></a>Microsoft Teams лицензии на защиту информации


Teams лицензии на защиту информации предоставляет аналитические сведения об использовании API microsoft [Graph API, которые имеют требования к лицензиям и платежам](/graph/teams-licenses). В этом отчете выделены все приложения, использующие эти API в [model=A](/graph/teams-licenses#modela-requirements). При использовании API в [режиме model=B](/graph/teams-licenses#modelb-requirements) или режиме оценки не отображается [использование](/graph/teams-licenses#evaluation-mode-default-requirements). 


## <a name="view-the-information-protection-license-report"></a>Просмотр отчета о лицензии на защиту информации

Вы должны быть администратором службы Teams, чтобы вносить эти изменения. Сведения о получении ролей и разрешений администратора см. в статье [Управление Teams с помощью ролей администратора Teams](../using-admin-roles.md).

1. В левой области навигации центра администрирования Microsoft Teams **выберите "Аналитика & отчетов** >  об **использовании"**. На **вкладке "Просмотр отчетов**" в разделе "Отчет **" выберите Information Protection лицензии**.
2. В **разделе "Диапазон дат**" выберите диапазон.
3. В **разделе "** Приложения" выберите приложение и нажмите кнопку **"Запустить отчет"**.

    ![Снимок экрана: раскрывающийся список Teams лицензии на защиту информации в центре администрирования Teams с выноски.](../media/teams-info-protection-license-report-dropdown-with-callouts.png "Снимок экрана: раскрывающийся список Teams лицензии на защиту информации в центре администрирования Teams с выноски.")

4. Для каждой категории можно просмотреть метрики пользователей с лицензией, пользователей без лицензии, заполненной емкости и используемой емкости. 

    ![Снимок экрана: сводная диаграмма Teams лицензии на защиту информации в центре администрирования Teams уведомлений об изменениях с выноской.](../media/teams-info-protection-license-report-chart-with-callouts.png "Снимок экрана: сводная диаграмма Teams лицензии на защиту информации в центре администрирования Teams уведомлений об изменениях с выноской.")

5. Данные можно экспортировать, нажав кнопку экспорта. Данные таблицы можно переключить, щелкнув вкладки для API уведомлений об изменениях Teams, API исправлений Teams, API экспорта Teams (чат) и Teams API экспорта (teams). 

    ![Снимок экрана: различные вкладки Teams лицензии на защиту информации в центре администрирования Teams вкладок с выносками.](../media/teams-info-protection-license-report-legend-tabs-with-callouts.png "Снимок экрана: различные вкладки Teams лицензии на защиту информации в центре администрирования Teams вкладок с выносками.")

    ![Снимок экрана: вкладка уведомления об изменениях Teams лицензии на защиту информации в центре администрирования Teams с выносками.](../media/teams-info-protection-license-report-change-notification-with-callouts.png "Снимок экрана: вкладка уведомления об изменениях Teams лицензии на защиту информации в центре администрирования Teams с выносками.")


## <a name="interpret-the-report"></a>Толкование отчета

|Выноска |Описание  |
|--------|-------------|
|**1**   |В отчете о лицензиях на защиту информации можно просмотреть тенденции за последние три месяца. |
|**2**   |В имени приложения отобразится список всех приложений, которые использовали [Microsoft API Graph](/graph/teams-licenses) с требованиями к лицензии и оплате в течение выбранного периода времени.|
|**3**   |Количество пользователей с [действительными лицензиями](/graph/teams-licenses#required-licenses-for-modela).  |
|**4**   |Число пользователей, у которых нет действительной [лицензии](/graph/teams-licenses#required-licenses-for-modela).  |
|**5**   |Общий объем вызовов API, разрешенный приложению, после которого плата за потребление за вызов API будет взиматься с приложения. |
|**6**   |Общий объем вызовов API, используемый приложением для заданного диапазона дат. |
|**7**   |Экспорт отчета в CSV-файл для автономного анализа. Выберите **"Экспорт в Excel**" и откройте вкладку "Загрузки". Выберите  "Скачать", чтобы скачать отчет, когда он будет готов. Этот экспорт будет экспортировать только выбранную вкладку.|
|**8**   |Выберите любую конкретную метку, чтобы отобразить или скрыть ее на диаграмме. |
|**9**   |На каждой вкладке отображается использование определенного набора API, а также уведомления об изменениях [,](/graph/api/resources/webhooks) [экспорт API](/microsoftteams/export-teams-content) и [API обновления сообщений](/graph/api/message-update). Выберите вкладку, чтобы просмотреть сведения об использовании этого API. |
|**10**   |**Использование API уведомлений об изменениях**<li>**Отображаемое** имя — отображаемое имя пользователя, для которого активируется уведомление об изменении.</li><li>**Требуемая лицензия** — указывает, имеет ли данный пользователь необходимую лицензию для успешной отправки уведомления об изменениях в приложение.</li><li>**Число попыток** — общее количество уведомлений об изменениях, активированных из-за этого пользователя.</li><li>**Уведомление об изменении** — общее количество уведомлений об изменениях, отправленных в приложение. Это значение будет меньше общего количества уведомлений об изменениях, активированных, если у пользователя нет действительной лицензии.</li>|
|**11**|**API исправлений**<li>**Отображаемое** имя — отображаемое имя пользователя, в котором была предпринята попытка обновить сообщение.</li> <li>**Требуемая лицензия** — указывает, имеет ли данный пользователь необходимую лицензию для успешного обновления сообщения.</li><li>**Число попыток** — общее число попыток обновления сообщений.</li><li>**Исправлено сообщение —** общее количество сообщений, которые были успешно обновлены.</li>|
|**12**|**Использование API экспорта чата**<li>**Отображаемое** имя — отображаемое имя пользователя, в котором была выполнена попытка экспорта сообщения чата пользователя.</li><li>**Требуемая лицензия** — указывает, имеет ли данный пользователь необходимую лицензию для успешного экспорта сообщения.</li><li>**Число попыток** — всего попыток экспорта сообщений чата.</li><li>**Экспортированное сообщение** — общее число попыток успешного экспорта сообщения чата.</li> |

## <a name="make-the-user-specific-data-anonymous"></a>Сделать данные конкретного пользователя анонимными

Чтобы сделать данные в отчете Teams действия пользователя анонимными, необходимо быть глобальным администратором. Это позволит скрыть идентифицируемые сведения, такие как отображаемое имя, электронная почта и Azure AD в отчетах и их экспорте.

1. В Центр администрирования Microsoft 365 перейдите **Параметры** \> **org Параметры** и на вкладке "Службы" выберите **"Отчеты"**.
    
2. Выберите **"Отчеты**", а затем выберите " **Отобразить анонимные идентификаторы"**. Этот параметр применяется как к отчетам об использовании в Центр администрирования Microsoft 365, так и к Teams центра администрирования.
  
3. Нажмите **кнопку "Сохранить изменения"**.
