<a name="manage-external-access-in-microsoft-teams"></a>Управление внешним доступом в Microsoft Teams
======================================================

Внешний доступ — это способ поиска, звонков, общения и настройки собраний в Teams для внешних пользователей групп из всего домена. Вы также можете использовать внешний доступ для общения с внешними пользователями Skype для бизнеса (в Интернете и в локальной системе) и Skype (на ранних этапах 2020).

Если внешним пользователям нужно предоставить доступ к командам и каналам, удобнее использовать гостевой доступ. Дополнительные сведения о различиях между внешним доступом и гостевым доступом приведены в статье [Сравнение внешних и гостевой доступа](communicate-with-users-from-other-organizations.md#compare-external-and-guest-access). 

Внешний доступ используется в следующих случаях:
  
- У вас есть пользователи в разных доменах, которым требуется совместная работа. Например, Rob@contoso.com и Ann@northwindtraders.com работают над проектом вместе с другими пользователями в доменах contoso.com и northwindtraders.com.

- Вы хотите, чтобы сотрудники вашей организации использовали Teams для связи с сотрудниками из других организаций.

- Вы хотите, чтобы любой пользователь Teams по всему миру мог вас найти и с вами связаться по адресу электронной почты. 




> [!IMPORTANT]
> В настоящее время, чтобы создать Федерацию в приложении Microsoft Teams с внешним пользователем не из вашей организации, который не является гостем или клиентом в вашей Azure Active Directory (Azure AD), необходимо правильно настроить гибридное развертывание и перейти в Skype для бизнеса Online. На 25 февраля 2019 Teams не поддерживает собственную федерацию без использования профиля SIP в Skype для бизнеса Online. Дополнительные сведения о настройке учетной записи для гибридного развертывания и о переходе на Teams см. в статье [Переход от гибридного развертывания Skype для бизнеса на Teams](https://docs.microsoft.com/microsoftteams/upgrade-to-teams-execute-skypeforbusinesshybrid).






## <a name="plan-for-external-access"></a>Планирование внешнего доступа

По умолчанию в Teams включена функция External Access, которая означает, что ваша организация может взаимодействовать со всеми внешними доменами. Если вы добавите блокируемые домены, все остальные домены будут разрешены; Если вы добавите разрешенные домены, все остальные домены будут заблокированы. Существует три сценария настройки внешнего доступа в центре администрирования Teams ( > **внешний доступ к****параметрам в масштабах Организации**).

- **Открыть Федерацию**: это параметр по умолчанию в Teams, который позволяет сотрудникам вашей организации находить, набирать, общаться и настраивать собрания с участием пользователей, которые являются внешними для вашей организации в любых доменах.

    В этом случае пользователи смогут общаться со всеми внешними доменами, работающими под управлением Teams или Skype для бизнеса, и использовать открытую Федерацию или добавить ваш домен в список разрешенных.

- **Разрешить конкретные домены**: Добавление доменов в список **разрешений** Ограничьте внешний доступ только для разрешенных доменов. После настройки списка разрешенных доменов все другие домены будут заблокированы. Чтобы разрешить конкретные домены, щелкните **Добавить домен**, добавьте имя домена, нажмите кнопку **действие, чтобы принять участие в этом домене**, а затем выберите **разрешено**.

- **Блокировать конкретные домены** — добавляя домены в **черный** список, вы можете общаться со всеми внешними доменами, *Кроме* блокируемых. Чтобы заблокировать конкретные домены, щелкните **Добавить домен**, добавьте имя домена, нажмите кнопку **действие, чтобы принять участие в этом домене**, а затем выберите пункт **заблокировано**. После настройки списка блокируемых доменов будут разрешены все другие домены.

## <a name="allow-or-block-domains"></a>Разрешение и блокирование доменов

### <a name="step-1---enable-your-organization-to-communicate-with-another-teams-organization"></a>Шаг 1. Предоставление своей организации возможности общаться с другой организацией Teams

![Значок с логотипом Microsoft Teams](media/teams-logo-30x30.png) **Использование центра администрирования Microsoft Teams**

1. В области навигации слева выберите **Параметры на уровне организации** > **Внешний доступ**.

2. Переведите переключатель **Разрешить взаимодействие с пользователями Skype для бизнеса и Teams** в положение **Вкл**.

     ![Снимок экрана: переключатель внешнего доступа в положении Вкл.](media/manage-external-access-2.png).

3. Если вы хотите, чтобы все организации Teams могли взаимодействовать с пользователями в вашей организации, перейдите к шагу 5.

4. Чтобы ограничить количество организаций, которые могут взаимодействовать с пользователями в вашей организации, вы можете разрешить все домены, кроме некоторых, или разрешить только определенные домены. 

    - Чтобы разрешить все домены, кроме некоторых, добавьте домены, которые вы хотите заблокировать, нажав кнопку **Добавить домен**. В области **Добавить домен** введите доменное имя, нажмите кнопку **Заблокирован**, а затем кликните **Готово**. 
    - Чтобы взаимодействовать только с определенными организациями, добавьте их домены в список со статусом **Разрешен**. Когда вы добавили домен в список разрешенных, взаимодействие будет ограничено только теми организациями, чьи домены находятся в списке разрешенных. 

5. Нажмите кнопку **Сохранить**.

6. Администратор в другой организации Teams обязательно должен произвести те же действия. Например, если другая организация ограничивает число организаций, которые могут взаимодействовать с ее пользователями, то их администратор должен внести домен вашей организации в список **разрешенных доменов**.

### <a name="step-2---test-it"></a>Шаг 2. Проверка

Чтобы проверить настройки, вам потребуется пользователь Teams, которого не блокирует ваш брандмауэр.
  
1. После того как вы и администратор организации изменили параметры **внешнего доступа**, можно начинать.

2. В приложении Teams найдите пользователя по адресу электронной почты и отправьте запрос на чат.

3. Попросите свой контакт в Teams отправить вам запрос на чат. Если вы не получили его запрос, проблема в настройках вашего брандмауэра (предполагается, что он подтвердил корректность настроек своего брандмауэра).

4. Еще один способ проверить, является ли проблема брандмауэром, – это перейти в расположение WiFi, не защищенное брандмауэром. например, кафе, и используйте Teams для отправки запроса вашему контакту на чат. Если сообщение пройдет через Wi-Fi, но не работает в сети, вы знаете, что проблема связана с брандмауэром.

> [!NOTE]
> Если вы и другой пользователь включите внешний доступ и разрешите другим доменам, это будет работать. Если это не поможет, необходимо убедиться, что ваш домен не заблокирован другим пользователем.

## <a name="communicate-with-users-in-a-skype-for-business-online-organization"></a>Общение с пользователями в организации, использующей Skype для бизнеса Online

Если вы настраиваете внешний доступ, чтобы пользователи Teams могли находить и общаться с пользователями в Организации Skype для бизнеса, которые ограничивают круг лиц, которые могут обращаться к своим пользователям, выполните указанные ниже действия, чтобы настроить внешний доступ из вашего домена к домену другой организации. Затем попросите администратора из другой организации выполнить указанные ниже действия, чтобы настроить внешний доступ к Skype для бизнеса Online. 

Конкретные рекомендации по распространению сценариев для Skype для бизнеса Online можно найти в разделе [Общие сценарии внешнего доступа](#common-external-access-scenarios) ниже.

![Значок с логотипом Skype для бизнеса](media/sfb-logo-30x30.png) **Использование центра администрирования Skype для бизнеса**

Пошаговые инструкции для администратора из этой организации.

1. В Центре администрирования Microsoft 365 перейдите в раздел **Центры администрирования** > **Teams и Skype** > **Портал прежней версии**.
  
2. В **Центре администрирования Skype для бизнеса** выберите пункт **Организация** > **Внешний обмен данными**.

3. Чтобы настроить обмен данными с конкретной компанией или с пользователями из другого домена, в раскрывающемся списке выберите **Включить только для разрешенных доменов**.

    Кроме того, если они хотят разрешить обмен данными с любыми пользователями, у которых политики Skype для бизнеса открыты, выберите **Включить для всех, кроме заблокированных доменов**. Это настройка по умолчанию.

4. В разделе **Заблокированные или разрешенные домены** выберите **+** и добавьте имя домена, который хотите разрешить.
