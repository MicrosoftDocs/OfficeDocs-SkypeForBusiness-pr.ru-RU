<a name="manage-external-access-in-microsoft-teams"></a>Управление внешним доступом в Microsoft Teams
======================================================

При использовании внешнего доступа Microsoft Teams пользователи Teams из других доменов могут участвовать в ваших беседах и звонках. Вы также можете разрешить доступ внешним пользователям, которые еще работают в Skype для бизнеса Online, Skype для бизнеса в локальной системе или даже в Skype.

Выполните действия, описанные в этой статье, в следующих случаях.
  
- В вашей организации пользователи используют разные домены, например Rob@contoso.com и Ann@northwindtraders.com.

- Вы хотите, чтобы сотрудники вашей организации использовали Teams для связи с сотрудниками из других организаций.

- Вы хотите, чтобы любой пользователь Teams по всему миру мог вас найти и с вами связаться по адресу электронной почты. Чтобы всё работало правильно, вы и другой пользователь должны включить внешний доступ и разрешить домены друг друга. Если что-то не работает, другой пользователь должен проверить, не заблокирован ли ваш домен в его настройках.

Внешний доступ позволяет внешним пользователям находить вас, звонить и отправлять мгновенные сообщения, а также назначать вам встречи. Если же вы хотите дать внешним пользователям доступ к командам и каналам, то удобнее использовать гостевой доступ. Подробнее о различиях между внешним доступом и гостевым доступом см. ниже в статье [Сравнение внешнего доступа и гостевого доступа](#external-access-vs-guest-access). Чтобы включить гостевой доступ, см. статью [Включение гостевого доступа](set-up-guests.md), так пользователи смогут общаться.

> [!IMPORTANT]
> В настоящее время, чтобы создать Федерацию в приложении Microsoft Teams с внешним пользователем не из вашей организации, который не является гостем или клиентом в вашей Azure Active Directory (Azure AD), необходимо правильно настроить гибридное развертывание и перейти в Skype для бизнеса Online. По состоянию на 25/02/2019, Teams не поддерживает собственную Федерацию, если пользователь профиля SIP не подключен к Skype для бизнеса Online.  Дополнительные сведения о настройке учетной записи для гибридного развертывания и о переходе на Teams см. в статье [Переход от гибридного развертывания Skype для бизнеса на Teams](https://docs.microsoft.com/ru-RU/microsoftteams/upgrade-to-teams-execute-skypeforbusinesshybrid).

## <a name="external-access-vs-guest-access"></a>Сравнение внешнего доступа и гостевого доступа

Различия между внешним доступом (Федерация) и гостевым доступом.

- Гостевой доступ позволяет получить доступ к отдельному сотруднику. Внешний доступ дает доступ ко всему домену.

- Гостевой доступ, предоставляемый владельцем команды, позволяет гостю [получить доступ к ресурсам](guest-experience.md), таким как обсуждения и файлы в каналах, для определенной команды, и общаться с другими пользователями в той команде, в которую он приглашен. При использовании внешнего доступа (федеративного чата) у участников внешнего чата нет доступа к командам или ресурсам команд в пригласившей их организации. Они могут участвовать только в приватном федеративном чате. Администраторы клиента могут выбрать один из двух вариантов общения, в зависимости от уровня взаимодействия, необходимого для внешней стороны. Администраторы могут выбрать один из подходов или оба, в зависимости от потребностей организации. Мы рекомендуем включать гостевой доступ. Это дает больше функций и возможностей для совместной работы в Teams. 

В таблице ниже сравниваются функции внешнего и гостевого доступа.

| Функция | Пользователи внешнего доступа | Пользователи гостевого доступа |
|---------|-----------------------|--------------------|
| Пользователь может общаться с кем-либо в другой компании | Да |Да |
| Пользователь может позвонить кому-либо в другой компании | Да | Да |
| Пользователь может видеть, доступен ли кто-то из другой компании для звонков или общения | Да | Да<sup>1</sup> |
| Пользователь может искать пользователей среди внешних клиентов | Да<sup>2</sup> | Нет |
| Пользователь может предоставлять доступ к файлам | Нет | Да |
| У пользователя есть доступ к ресурсам Teams | Нет | Да |
| Возможность добавления пользователя в групповой чат | Нет | Да |
| Возможность добавления пользователя на собрание | Да | Да |
| Возможность добавления дополнительных пользователей в чат с внешним пользователем | Нет<sup>3</sup> | Недоступно |
| Идентификация пользователя как внешней стороны | Да | Да |
| Отображается присутствие | Да | Да |
| Отображается сообщение об отсутствии на рабочем месте | Нет | Да |
| Можно заблокировать отдельных пользователей | Нет | Да |
| Поддерживаются @mentions | Нет | Да |
| Совершение приватных звонков | Да | Да |
| Разрешить видео по IP | Да | Да |
| Режим демонстрации экрана | Нет | Да |
| Разрешить функцию "Начать собрание" | Нет | Да |
| Редактирование отправленных сообщений | Нет | Да |
| Удаление отправленных сообщений | Нет | Да |
| Использование в беседе гиф-эмотиконов | Нет | Да |
| Использование в беседе мемов | Нет | Да |
| Использование в беседе стикеров | Нет | Да |
||||

<sup>1</sup> Если пользователь добавлен в качестве гостя и выполнил вход в качестве гостя для гостевого клиента.<br>
<sup>2</sup> Только по электронной почте или по протоколу SIP.<br>
<sup>3</sup> Внешний (федеративный) чат — только 1:1.

Дополнительные сведения о функциях гостевого доступа и о работе в режиме гостевого доступа см. в статьях [Включение и отключение гостевого доступа в Microsoft Teams](https://docs.microsoft.com/microsoftteams/set-up-guests) и [Возможности работы в режиме гостевого доступа](https://docs.microsoft.com/microsoftteams/guest-experience).

Дополнительные сведения о бесплатной версии Teams и о ее работе с функциями внешнего доступа, см. в статье [Различия между Microsoft Teams и Microsoft Teams (бесплатная версия)](https://support.office.com/article/differences-between-microsoft-teams-and-microsoft-teams-free-0b69cf39-eb52-49af-b255-60d46fdf8a9c?ui=en-US&rs=en-US&ad=US).

## <a name="let-your-teams-users-chat-and-communicate-with-users-in-another-organization"></a>Предоставьте своим пользователям Teams возможность общаться в чате и взаимодействовать с пользователями из другой организации.

Благодаря внешнему доступу ваши пользователи Teams и Skype для бизнеса могут общаться с другими пользователями за пределами вашей организации. Ваша организация по умолчанию может взаимодействовать со всеми внешними доменами. При добавлении заблокированных доменов будут разрешены все другие домены, но при добавлении разрешенных доменов все остальные домены будут заблокированы. Вы легко можете настроить внешний доступ для своей организации. Возможны три сценария настройки:

- **Сценарий 1**. Вы можете использовать **ОТКРЫТУЮ ФЕДЕРАЦИЮ**. Эта настройка используется по умолчанию и позволяет сотрудникам вашей организации находить, звонить и отправлять мгновенные сообщения, а также назначать встречи с внешними пользователями из других организаций.

    При использовании этой настройки ваши пользователи могут общаться со всеми внешними доменами, на которых работают Teams или Skype для бизнеса, а также используется Open Federation, или с теми, у которых ваш домен добавлен в список разрешений.

- **Сценарий 2**. Вы можете добавить один или несколько доменов в список **РАЗРЕШИТЬ**. Для этого нажмите кнопку **Добавить домен**, добавьте доменное имя, нажмите **Действие с этим доменом**, а затем выберите **Разрешен**. Имейте в виду, что в этом случае все остальные домены будут **БЛОКИРОВАТЬСЯ**.

- **Сценарий 3**. Вы можете добавить один или несколько доменов в список **БЛОКИРОВКИ**. Для этого нажмите кнопку **Добавить домен**, добавьте доменное имя, нажмите **Действие с этим доменом**, а затем выберите **Заблокирован**. Имейте в виду, что в этом случае все остальные домены будут **РАЗРЕШЕНЫ**.

Чтобы разрешить или запретить доступ к доменам, выполните указанные ниже действия.

### <a name="step-1---enable-your-organization-to-communicate-with-another-teams-organization"></a>Шаг 1. Предоставление своей организации возможности общаться с другой организацией Teams

![Значок с логотипом Microsoft Teams](media/teams-logo-30x30.png) **Использование центра администрирования Microsoft Teams**

1. В области навигации слева выберите **Параметры на уровне организации** > **Внешний доступ**.

2. Переведите переключатель **Разрешить взаимодействие с пользователями Skype для бизнеса и Teams** в положение **Вкл**.

     ![Снимок экрана: переключатель внешнего доступа в положении Вкл.](media/manage-external-access-2.png).

3. Если вы хотите, чтобы все организации Teams могли взаимодействовать с пользователями в вашей организации, перейдите к шагу 5.

4. Чтобы ограничить количество организаций, которые могут взаимодействовать с пользователями в вашей организации, вы можете разрешить все домены, кроме некоторых, или разрешить только определенные домены. 

    - Чтобы разрешить все домены, кроме некоторых, добавьте домены, которые вы хотите заблокировать, нажав кнопку **Добавить домен**. В области **Добавить домен** введите доменное имя, нажмите кнопку **Заблокирован**, а затем кликните **Готово**. 
    - Чтобы взаимодействовать только с определенными организациями, добавьте их домены в список со статусом **Разрешен**. Когда вы добавили домен в список разрешенных, взаимодействие будет ограничено только теми организациями, чьи домены находятся в списке разрешенных. 

5. Нажмите кнопку **Сохранить**.

6. Администратор в другой организации Teams обязательно должен произвести те же действия. Например, если другая организация ограничивает число организаций, которые могут взаимодействовать с ее пользователями, то их администратор должен внести домен вашей организации в список **разрешенных доменов**.

### <a name="step-2---test-it"></a>Шаг 2. Проверка

Чтобы проверить настройки, вам потребуется пользователь Teams, которого не блокирует ваш брандмауэр.
  
1. После того как вы и администратор организации изменили параметры **внешнего доступа**, можно начинать.

2. В приложении Teams найдите пользователя по адресу электронной почты и отправьте запрос на чат.

3. Попросите свой контакт в Teams отправить вам запрос на чат. Если вы не получили его запрос, проблема в настройках вашего брандмауэра (предполагается, что он подтвердил корректность настроек своего брандмауэра).

4. Еще один способ проверить, не является ли брандмауэр источником проблемы, — подключиться к WiFi, не защищенному вашим брандмауэром,  например в кафе, и отправить запрос на чат своему контакту в Teams. Если при подключении к этому Wi-Fi сообщение проходит, а на рабочем месте — нет, то ясно, что проблема в вашем брандмауэре.

## <a name="communicate-with-users-in-a-skype-for-business-online-organization"></a>Общение с пользователями в организации, использующей Skype для бизнеса Online

Вы настраиваете внешний доступ, чтобы ваши пользователи Teams могли находить пользователей из организации, использующей Skype для бизнеса, и с ними общаться. Если эта организация ограничивает круг лиц, которые могут общаться с ее пользователями, следуйте инструкции по настройке внешнего доступа из своего домена к домену другой организации. Затем попросите администратора из другой организации выполнить указанные ниже действия, чтобы настроить внешний доступ к Skype для бизнеса Online.

![Значок с логотипом Skype для бизнеса](media/sfb-logo-30x30.png) **Использование центра администрирования Skype для бизнеса**

Пошаговые инструкции для администратора из этой организации.

1. В Центре администрирования Microsoft 365 перейдите в раздел **Центры администрирования** > **Teams и Skype** > **Портал прежней версии**.
  
2. В **Центре администрирования Skype для бизнеса** выберите пункт **Организация** > **Внешний обмен данными**.

3. Чтобы настроить обмен данными с конкретной компанией или с пользователями из другого домена, в раскрывающемся списке выберите **Включить только для разрешенных доменов**.

    Кроме того, если они хотят разрешить обмен данными с любыми пользователями, у которых политики Skype для бизнеса открыты, выберите **Включить для всех, кроме заблокированных доменов**. Это настройка по умолчанию.

4. В разделе **Заблокированные или разрешенные домены** выберите **+** и добавьте имя домена, который хотите разрешить.
