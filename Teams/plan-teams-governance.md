# <a name="plan-for-governance-in-teams"></a>Планирование управления в группах

Команды предоставляет набор средств для реализации любой функциональные возможности управления, которые может потребоваться вашей организации. В данной статье описывается ИТ-специалистов, чтобы задать вопросы вправо для определения их требования для управления, а также в соответствии с их. 

## <a name="group-and-team-creation-naming-classification-and-guest-access"></a>Создание группы и группы, именования, классификации и доступ в качестве гостя

Ваша организация может потребоваться реализации строго контролировать как группы с именем и классификации, гости могут добавляться как участники групп и пользователей, которые можно создавать группы. Каждая из этих областей можно настроить с помощью Azure Active Directory (Azure AD).

## <a name="group-and-team-expiration-retention-and-archiving"></a>Истечение срока действия группы и группы, хранения и архивации

Ваша организация может иметь дополнительные требования к настройке политик для прекращения действия, хранения, и архивирование групп и групп данных. Можно настроить истечение срока действия групповых политик для управления жизненным циклом политики группы и хранения для сохранения или удаления сведения при необходимости автоматически, а можно архивировать групп (задать их в режим только для чтения), чтобы сохранить представление в момент команды на больше не активно.

| ![](media/audio_conferencing_image9.png)<br/>Дальнейшие действия|<ul><li>Документирование требований организации для прекращения действия группы, срок хранения данных и архивации.</li><li>Планирование для реализации этих требований как часть процесса развертывания групп.</li><li>Связи и публикация политик для оповещения пользователей группам поведение, которое они могут ожидать.</li></ul>|

Используйте следующую таблицу для записи требований к вашей организации.

|---------|---------|---------|---------|
|Политика прекращения действия |Управление жизненным циклом групп Office 365, задав политики срока действия. |P1 |ПОДЛЕЖИТ ОПРЕДЕЛЕНИЮ|
|Политики хранения |Сохранить или удалить данные (группам сообщения и файлы канала) для конкретного периода времени, задав политики хранения для групп в центре безопасности и соблюдения правовых норм. **Примечание**: это средство требует лицензирования Office 365 для предприятий E3 или выше. |Нет |ПОДЛЕЖИТ ОПРЕДЕЛЕНИЮ |

## <a name="security-and-compliance"></a>Безопасность и соответствие требованиям

Команды создан на основе повышенной безопасности и обеспечения соответствия требованиям Office 365 и поддерживает аудита и отчеты, контента поиска соответствия требованиям, обнаружение электронных данных, юридическое удержание и политики хранения.

> Если вашей организации есть требования к безопасности и соответствия требованиям, ознакомьтесь с исчерпывающими материалы по этому разделу в статье [Обзор безопасности и соответствия требованиям в группах Майкрософт](security-compliance-overview.md).
