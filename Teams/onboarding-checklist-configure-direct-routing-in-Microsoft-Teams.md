|1|Определить, какие подключение к ТСОП будет развертываться для вашей организации расположений|Корпорация Майкрософт предоставляет альтернативных методов, чтобы предоставить подключение ТСОП для пользователей с помощью Office 365 телефонной системой.<ul><li>Телефон системы с Тарифные планы («Тарифные планы»): Скайп для бизнеса в Интернете и рабочих групп<li>Телефон системы прямой маршрутизации («Прямая отправка»): Только групп<li>Телефон системы с локального подключения к ТСОП: Скайп для бизнеса в Интернет только<li>Скайп облаке соединитель для выпуска для бизнеса: Скайп для бизнеса только Online</ul>Прямое маршрутизации дает организаций тех преимуществ, которые вызов планы, за исключением того, что упростить подключение к ТСОП с помощью стороннего поставщика, а не корпорации Майкрософт. Это позволяет для развертывания в странах, где вызов планы недоступны или в развертываниях, где существующего контракта поставщика службы PSTN должна поддерживаться или взаимодействие с системами определенных локальной является обязательным.<br><br>Определите, какие параметры наиболее подходящий для вашей организации. | |[Доступность аудиоконференций и планов звонков в различных странах и регионах](country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans.md)<br><br>[Адаптация новых сотрудников контрольные списки для конференц-связи аудио и вызов планов](https://docs.microsoft.com/MicrosoftTeams/onboarding-checklist-configure-cloud-voice-workloads-in-Microsoft-Teams) |
|2|Проверьте список пользователей и ритмичности развертывания для включения прямой маршрутизации|Проверьте список бизнес-единицы или сайты в области для прямой маршрутизации с группами. Использование целей и модель основные результаты, работают в работе пользователей, которые будут добавлены в область для прямой маршрутизации. Мы рекомендуем работать на основе узла, поэтому могут быть направлены ресурсов.<br><br>Как часть плана включение, определение пользователей, которые можно будет включить, запрошенные (пилотного проекта, узле 1, 2 сайта и т. п.).||[Прямое подключение формирования представления маршрутизации](2-envision-make-my-service-decisions-direct-routing.md)|
|3|Планирование и получить лицензии|Пользователи прямой маршрутизации должны иметь следующие лицензии, назначенные в Office 365:<ul><li>Skype для бизнеса Online (план 2)<li>Microsoft телефонной системой<li>Microsoft Teams<li>Конференц-связи Microsoft звука</ul>Прямое маршрутизации также поддерживает пользователей, которые лицензируются для вызова планов. Система телефон с вызова планы могут перенаправлять некоторые вызовы с использованием интерфейса прямой маршрутизации.<br><br>Аудиоконференции лицензия требуется для добавления внешних участников к запланированным собраниям по выходному им или с указанием телефонного номера.||[Прямое подключение лицензирования маршрутизации](https://docs.microsoft.com/microsoftteams/direct-routing-plan#licensing-and-other-requirements)|
|4|Планирование сеанса границы контроллера (SBC) доменных имен|Имя домена SBC должно быть от одного из указанных имен, зарегистрированных в «Домены» клиента.<br><br>**Примечание:** Нельзя использовать *. onmicrosoft.com для полное доменное имя (FQDN) SBC.<br><br>Имя домена SBC важно также в планировании сертификаты, необходимые для каждой SBC.||[SBC доменных имен](https://docs.microsoft.com/microsoftteams/direct-routing-plan#sbc-domain-names)|
|5|План для сертификатов|Мы настоятельно рекомендуем запросить сертификат для SBC, создав сертификата (CSR) запрос подписи.<br><br>Сертификат должен иметь полное доменное имя пограничного контроллера Сеансов в теме, общее имя или поля альтернативное имя субъекта. Кроме того прямой маршрутизации поддерживает подстановочные знаки в общее имя или альтернативное имя субъекта.<br><br>Сведения о создании запроса CSR для SBC обратитесь к документации, поставляемой с поставщиком SBC.<br><br>Статьи в столбце **Дополнительные сведения** приведены поддерживаемые корневые центры сертификации.||[Общедоступный доверенный сертификат для SBC](https://docs.microsoft.com/microsoftteams/direct-routing-plan#public-trusted-certificate-for-the-sbc)|
|6|Планирование и настройка порты брандмауэра|Точек подключения для прямой маршрутизации являются следующие три полных доменных имен:<ul><li>SIP.pstnhub.Microsoft.com<li>sip2.pstnhub.Microsoft.com<li>sip3.pstnhub.Microsoft.com</ul>Должен быть разрешен трафик между этих точек подключения и вашей их изготовителей на корпоративные брандмауэры. При настройке SBC на SBC определяется TCP-порт.<br><br>— Это трафика мультимедиа на UDP-ПОРТ. Этот тип трафик направляется в или из компонентов процессора мультимедиа. Двунаправленный трафика между SBC и процессоров мультимедиа также должны быть включены на брандмауэре.<br><br>**Примечание:** Мультимедийные процессоры имеют динамические IP-адреса и статический IP-адрес будет доступна позднее. Важно, что позволяет любого IP-адреса, перечисленные в [Диапазоны IP-адресов центра обработки данных Azure](https://www.microsoft.com/download/details.aspx?id=41653).||[Передача сигналов SIP: Полные доменные имена и порты брандмауэра](https://docs.microsoft.com/microsoftteams/direct-routing-plan#sip-signaling-fqdns-and-firewall-ports)<br><br>[Трафика мультимедиа: IP-адреса и порта для диапазона](https://docs.microsoft.com/microsoftteams/direct-routing-plan#media-traffic-port-ranges)<br><br>[Диапазоны IP-адресов центра обработки данных Azure](https://www.microsoft.com/download/details.aspx?id=41653)|
|7|Настройка тем|Майкрософт поддерживает только сертифицированные SBC связать с прямой маршрутизации.<br><br>Настройте их изготовителей с помощью специализированное руководство и инструкциям в статье в столбце **Дополнительные сведения** .||[Поддерживаемые пограничные контроллеры сеансов (SBC)](https://docs.microsoft.com/microsoftteams/direct-routing-plan#supported-session-border-controllers-sbcs)|
|8|Пара SBC при прямой маршрутизации|SBC на каждом сайте должен соответствовать прямой маршрутизации для предоставления набора номера и возможности вызовов ТСОП с помощью собственного магистральных линий связи.<br><br>Проверка SBC этого конкретного сайта уже сопряжено с прямой маршрутизации или настройка пары, если не выполнены перед.<br><br>Майкрософт поддерживает только сертифицированные SBC связать с прямой маршрутизации. Проверка сертифицированный SBC на этом сайте.||[Пары SBC для направления маршрутизации службы телефонной системой](https://docs.microsoft.com/microsoftteams/direct-routing-configure#pair-the-sbc-to-direct-routing-service-of-phone-system)|
|9|Проверка SBC связывания|Запустите `Get-CsOnlinePSTNGateway` командлет для каждого SBC пару для определенного сайта и убедитесь, что параметр **Enabled** значение **True**.<br><br>Убедитесь, что SBC получает с помощью интерфейса управления SBC **200 «ОК»** ответов на параметры исходящих SIP.|||
