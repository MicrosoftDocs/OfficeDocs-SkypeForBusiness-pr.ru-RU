# <a name="setting-voicemail-policies-in-your-organization"></a>Настройка политик голосовой почты в организации

> [!WARNING]
> Для Skype для бизнеса пользователей отключение голосовой почты с помощью политики Microsoft Teams вызовов также может отключить службу голосовой почты для Skype для бизнеса пользователей.

Политики голосовой почты можно использовать для управления различными функциями, связанными с облачная голосовая почта.

## <a name="voicemail-organization-defaults-for-all-users"></a>Организации голосовой почты по умолчанию для всех пользователей
- Транскрибирование голосовой почты включено.
- Преобразование транскрибирования голосовой почты включено.
- Маскировка ненормативной лексики при транскрибировании голосовой почты отключена.
- Максимальная длительность записи — пять минут.
- Изменение правил ответов на вызовы включено.
- Для основного и дополнительного языков системных запросов задано значение NULL, и используется параметр языка голосовой почты пользователя.

Вы можете использовать глобальную политику (по умолчанию для всей организации), которая создается автоматически, или создавать и назначать настраиваемые политики.

## <a name="create-a-custom-voicemail-policy"></a>Создание пользовательской политики голосовой почты

Выполните следующие действия, чтобы создать пользовательскую политику голосовой почты.

1. В левой области навигации центра администрирования Microsoft Teams перейдите к политикам **VoiceVoicemail** > .
2. Нажмите **Добавить**.
3. Включите или отключите функции, которые вы хотите использовать в политике голосовой почты.
4. Нажмите **Сохранить**.

## <a name="edit-a-voicemail-policy"></a>Изменение политики голосовой почты

Выполните следующие действия, чтобы изменить существующую политику голосовой почты.

1. В левой области навигации центра администрирования Microsoft Teams выберите политики **VoiceVoicemail** > .
2. Щелкните рядом с политикой, которую нужно изменить, а затем выберите "Изменить **"**.
3. Внесите необходимые изменения и нажмите кнопку " **Сохранить"**.

> [!IMPORTANT]
> Невозможно изменить или удалить предварительно настроенные экземпляры политики с именами TranscriptionDisabled и TranscriptionProfanityMaskingEnabled.

## <a name="voicemail-policy-settings"></a>Параметры политики голосовой почты
  
### <a name="enable-transcription"></a>Включение транскрибирования

Этот параметр определяет, будет ли служба облачная голосовая почта создавать текстовую транспликацию записанной голосовой почты и включать ее в сообщение голосовой почты. Транскрибирование будет выполняться на основе языка, обнаруженного в записанной голосовой почте.

### <a name="transcription-translation"></a>Транскрибирование

Этот параметр определяет, будет ли служба облачная голосовая почта переводит транскрибирование записанной голосовой почты. Попытка перевода будет выполнена на предпочитаемый язык получателя голосовой почты.

### <a name="transcription-profanity-masking"></a>Маскирование ненормативной лексики транскрибирования

Этот параметр определяет, будет ли служба облачная голосовая почта маскировать ненормативную лексику, обнаруженную в транскрибировании голосовой почты.

### <a name="maximum-recording-duration"></a>Максимальная длительность записи

Максимальная длина записи определяет максимальное время записи голосовой почты. Значение по умолчанию — 5 минут.

### <a name="call-answering-rules"></a>Правила ответа на вызовы

Этот параметр определяет, разрешено ли пользователю настраивать правила ответа на вызовы голосовой почты в Microsoft Teams.

### <a name="dual-language-system-prompts"></a>Запросы к двойной языковой системе

По умолчанию системные запросы голосовой почты отображаются вызывающим абонентам на языке, выбранном пользователем при настройке голосовой почты. Если существует бизнес-требование, чтобы системные запросы голосовой почты отображались на двух языках, можно задать основной и дополнительный языки, и они могут быть не одинаковыми.

### <a name="share-data-for-service-improvements"></a>Совместное использование данных для улучшения службы

Указывает, используются ли данные голосовой почты и транскрибирования в службе для обучения и повышения точности. Если задано значение false, данные голосовой почты не будут совместно использоваться независимо от выбора пользователя.


> [!IMPORTANT]
> Служба голосовой почты Microsoft 365 и Office 365 кэширует политики голосовой почты и обновляет кэш каждые 6 часов. Поэтому внесение изменений в политику может занять до 6 часов.
