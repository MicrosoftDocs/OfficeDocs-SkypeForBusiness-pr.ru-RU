# <a name="use-real-time-telemetry-to-troubleshoot-poor-meeting-quality"></a>Использование телеметрии в режиме реального времени для устранения неполадок с качеством собраний

В этой статье объясняется, как использовать Real-Time Analytics (RTA) для устранения неполадок с качеством собраний Microsoft Teams для отдельных пользователей. Вы можете получить доступ Real-Time Analytics, если у вас есть одна из следующих ролей:

- Администратор Teams
- Специалист службы поддержки связи Teams
- Инженер поддержки по коммуникациям Teams

Дополнительные сведения о ролях администраторов Teams см. в статье "Использование ролей администратора [Microsoft Teams для управления Teams"](/MicrosoftTeams/using-admin-roles).

Real-Time Analytics позволяет ИТ-администраторам просматривать запланированные собрания важных пользователей и просматривать звук, видео, общий доступ к содержимому и проблемы, связанные с сетью. Администратор может использовать эти данные телеметрии для изучения этих проблем во время собраний и устранения неполадок в режиме реального времени.

## <a name="what-is-real-time-analytics"></a>Что такое Real-Time Analytics?

В настоящее время администраторам Teams доступно устранение неполадок с отдельными собраниями с помощью [аналитики](use-call-analytics-to-troubleshoot-poor-call-quality.md) звонков после окончания собрания. Real-Time аналитики позволяет администраторам устранять неполадки запланированных собраний во время их выполнения.

Real-Time Analytics отображает подробные сведения о собраниях Teams для каждого пользователя в учетной записи Office 365, обновленной в режиме реального времени. Он содержит сведения об устройствах, сети, подключении, звуке, видео и совместном использовании содержимого, что поможет администраторам эффективнее устранять неполадки с качеством звонков.

Администратор Teams получает полный доступ ко всем данным телеметрии в режиме реального времени для каждого пользователя. Кроме того, вы можете назначить роли Azure Active Directory для сотрудников службы поддержки. Дополнительные сведения об этих ролях см. в разделе ["Предоставление разрешений для сотрудников службы поддержки](set-up-call-analytics.md#give-permission-to-support-and-helpdesk-staff) и поддержки".

## <a name="where-to-find-per-user-real-time-troubleshooting-telemetry"></a>Где найти данные телеметрии для каждого пользователя в режиме реального времени

Чтобы просмотреть все сведения о собрании и данные для пользователя, перейдите в Центр [администрирования Teams](https://admin.teams.microsoft.com). В **разделе "** > **Управление пользователями**" выберите пользователя и откройте вкладку "Собрания **& вызовов** " на странице профиля пользователя. В **разделе** "Недавние собрания" вы увидите список собраний, на которых пользователь присутствовал в течение последних 24 часов, для которых доступна телеметрия в режиме реального *времени, включая* все собрания, в которых он выполняется. Если собрание не выполняется или нет данных телеметрии в режиме реального времени, оно будет отображаться на **прошлых собраниях**.

:::image type="content" alt-text="Снимок экрана: таблица последних собраний." source="media/recent-meetings.png" lightbox="media/recent-meetings.png":::

Чтобы получить дополнительные сведения о участниках собрания, которое выполняется, включая статистику устройства, сети и звука, найдите собрание в разделе "Недавние  собрания" и щелкните ссылку в столбце **"Участники**".

:::image type="content" alt-text="Снимок экрана: таблица сведений о участнике." source="media/participant-details-edit.png" lightbox="media/participant-details-edit.png":::

Чтобы просмотреть данные телеметрии заданного пользователя для выполнения собрания, включая сведения об устройстве, сети, звуке, видео и сведениях о совместном использовании содержимого, выберите идентификатор **собрания**.

:::image type="content" alt-text="Снимок экрана: данные сеанса пользователя аналитики звонков." source="media/real-time-telemetry-edit.png" lightbox="media/real-time-telemetry-edit.png":::

|:---|:---|:---|
| Звук | &bull; Незамыщенные, говорить с участниками <br/> &bull; Не включено, не произносит речи <br/> &bull; Приглушенные | Указывает поведение пользователя для звуковой части вызова.  |

## <a name="limitations"></a>Ограничения

- Телеметрия в режиме реального времени доступна только для запланированных собраний и собраний. Для вызовов ТСОП, звонков 1:1 и групповых звонков телеметрия в режиме реального времени недоступна.
- Телеметрия в режиме реального времени доступна только для докладчиков запланированного трансляции. В настоящее время он недоступен для участников трансляции.
- Данные телеметрии в режиме реального времени доступны для  собрания в разделе "Последние собрания" в течение 24 часов после завершения собрания. По истечении 24 часов вы не сможете получить доступ к данным, и собрание перейдет на **прошлые собрания**. Если время собрания превышает 3 часа, данные телеметрии в режиме реального времени будут доступны только за *последние 3 часа*.
- Телеметрия недоступна в режиме реального времени при использовании более ранних версий Teams. Если телеметрия недоступна, попробуйте обновить клиент.
- Если внешние или анонимные пользователи присоединяются к собранию, их отображаемое  имя будет отображаться как недоступное для сохранения конфиденциальности между клиентами.

> [!NOTE]
> В рамках общедоступной предварительной версии с ограниченным временем данные телеметрии в реальном времени доступны в течение **7** дней после завершения собрания. По завершении предварительной версии только клиенты с лицензией на надстройку Advanced Communications будут иметь доступ к телеметрии в течение 7 дней. На все остальные клиенты распространяются указанные выше ограничения.
