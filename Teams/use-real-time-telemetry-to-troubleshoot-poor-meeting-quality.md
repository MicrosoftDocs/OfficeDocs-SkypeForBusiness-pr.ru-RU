# <a name="use-real-time-telemetry-to-troubleshoot-poor-meeting-quality"></a>Использование телеметрии в режиме реального времени для устранения неполадок с качеством собраний

В этой статье объясняется, как использовать Real-Time Analytics (RTA) для устранения неполадок с плохим качеством собраний Майкрософт Teams для отдельных пользователей. Вы можете получить доступ к Real-Time Analytics, если у вас есть одна из следующих ролей:

- Администратор Teams
- Специалист по поддержке коммуникаций Teams
- Инженер поддержки по коммуникациям Teams

Дополнительные сведения о ролях администраторов Teams см. в статье [Использование Майкрософт ролей администраторов Teams для управления Teams](/MicrosoftTeams/using-admin-roles).

Real-Time Analytics позволяет ИТ-администраторам просматривать запланированные собрания важных пользователей и просматривать аудио, видео, общий доступ к содержимому и проблемы, связанные с сетью. Администратор может использовать эту телеметрию для изучения этих проблем во время собраний и устранения неполадок в режиме реального времени.

## <a name="what-is-real-time-analytics"></a>Что такое аналитика Real-Time?

Сегодня после завершения собрания администраторы Teams могут устранять неполадки с отдельными собраниями с помощью [аналитики звонков](use-call-analytics-to-troubleshoot-poor-call-quality.md) . Real-Time Analytics позволяет администраторам устранять неполадки с запланированными собраниями, пока они выполняются.

Real-Time Analytics отображает подробные сведения о собраниях Teams для каждого пользователя в вашей учетной записи Office 365, обновленные в режиме реального времени. Она содержит сведения об устройствах, сети, подключениях, проблемах со звуком, видео и общим доступом к содержимому, что поможет администраторам более эффективно устранять неполадки с качеством звонков.

Администратор Teams получает полный доступ ко всем данным телеметрии в режиме реального времени для каждого пользователя. Кроме того, вы можете назначить роли Azure Active Directory сотрудникам службы поддержки. Дополнительные сведения об этих ролях см. в статье [Предоставление разрешения сотрудникам службы поддержки и службы поддержки](set-up-call-analytics.md#give-permission-to-support-and-helpdesk-staff).

## <a name="where-to-find-per-user-real-time-troubleshooting-telemetry"></a>Где найти данные телеметрии для отдельных пользователей в режиме реального времени

Чтобы просмотреть все сведения о собраниях и данные пользователя, перейдите в [Центр администрирования Teams](https://admin.teams.microsoft.com). В разделе **Пользователи** > **Управление пользователями** выберите пользователя и откройте вкладку **Собрания & вызовы** на странице профиля пользователя. В разделе **Последние собрания** вы увидите список собраний, на которых пользователь присутствовал в течение последних 24 часов, *для которых доступна телеметрия в режиме реального времени*, включая все собрания, которые выполняется. Если собрание не выполняется или не содержит данных телеметрии в режиме реального времени, оно будет отображаться в **разделе Прошлые собрания**.

:::image type="content" alt-text="Снимок экрана: таблица недавних собраний." source="media/recent-meetings.png" lightbox="media/recent-meetings.png":::

> [!NOTE]
> Чтобы собрание отображалось в разделе "Последние собрания", администратор Teams должен щелкнуть собрание в Real-Time Analytics во время собрания, чтобы начать поток телеметрии клиента в режиме реального времени.


Чтобы получить дополнительные сведения об участниках собрания, которое выполняется, включая статистику устройств, сети и звука, найдите собрание в разделе **Последние собрания** и щелкните ссылку в столбце **Участники** .

:::image type="content" alt-text="Снимок экрана: таблица сведений об участниках." source="media/participant-details-edit.png" lightbox="media/participant-details-edit.png":::

Чтобы просмотреть данные телеметрии определенного пользователя для проведения собрания, включая сведения об устройстве, сети, аудио, видео и общий доступ к содержимому, выберите **идентификатор собрания**.

:::image type="content" alt-text="Снимок экрана: данные сеанса пользователя аналитики вызовов." source="media/real-time-telemetry-edit.png" lightbox="media/real-time-telemetry-edit.png":::

## <a name="limitations"></a>Ограничения

- Подписка на данные телеметрии в режиме реального времени не является автоматической для всех собраний и должна быть запущена администратором Teams во время собрания.
- Данные телеметрии в режиме реального времени будут доступны только для поддерживаемых конечных точек собрания с момента, когда администратор впервые щелкнул собрание в Real-Time Analytics.
- Данные телеметрии в режиме реального времени доступны только для запланированных собраний и собраний. Для вызовов ТСОП, вызовов 1:1 и групповых вызовов телеметрия в режиме реального времени недоступна.
- Телеметрия в режиме реального времени доступна только для докладчиков запланированных трансляций. В настоящее время он недоступен для участников трансляции.
- Данные телеметрии в режиме реального времени доступны для собрания в разделе **Последние собрания** в течение 24 часов после завершения собрания. Через 24 часа вы не сможете получить доступ к данным, и собрание переместится в **прошлое собрание**. Если собрание длится более 3 часов, данные телеметрии в режиме реального времени будут доступны только за *последние 3 часа*.
- Данные телеметрии недоступны в режиме реального времени при использовании более старых версий Teams. Если данные телеметрии недоступны, попробуйте обновить клиент.
- Если к собранию присоединяются внешние участники или анонимные пользователи, их отображаемое имя будет **отображаться как недоступное** для сохранения конфиденциальности между клиентами.

> [!NOTE]
> В рамках общедоступной предварительной версии с ограниченным временем данные телеметрии в режиме реального времени в настоящее время доступны в течение **7 дней** после завершения собрания. После завершения предварительной версии только у клиентов с лицензированием надстройки Advanced Communications будут доступны данные телеметрии за продленный 7-дневный период. На всех остальных арендаторов распространяются указанные выше ограничения.
