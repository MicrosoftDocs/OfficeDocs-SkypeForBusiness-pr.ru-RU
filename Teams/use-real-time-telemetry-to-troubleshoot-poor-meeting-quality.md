# <a name="use-real-time-telemetry-to-troubleshoot-poor-meeting-quality"></a>Использование телеметрии в режиме реального времени для устранения неполадок с качеством собраний

В этой статье объясняется, как использовать Real-Time Analytics (RTA) для устранения неполадок с Microsoft Teams качеством собраний для отдельных пользователей. Вы можете получить доступ Real-Time Analytics, если у вас есть одна из следующих ролей:

- Администратор Teams
- Teams службы поддержки связи
- Инженер поддержки по коммуникациям Teams

Дополнительные сведения о Teams администраторов см. в статье [Microsoft Teams](/MicrosoftTeams/using-admin-roles) роли администратора для управления Teams.

Real-Time Analytics позволяет ИТ-администраторам просматривать запланированные собрания важных пользователей и просматривать звук, видео, общий доступ к содержимому и проблемы, связанные с сетью. Администратор может использовать эти данные телеметрии для изучения этих проблем во время собраний и устранения неполадок в режиме реального времени.

## <a name="what-is-real-time-analytics"></a>Что такое Real-Time Analytics?

В настоящее время устранение неполадок с отдельными собраниями доступно Teams администраторам через [аналитику](use-call-analytics-to-troubleshoot-poor-call-quality.md) звонков после завершения собрания. Real-Time аналитики позволяет администраторам устранять неполадки запланированных собраний во время их выполнения.

Real-Time Analytics отображает подробные сведения о собраниях Teams для каждого пользователя в учетной записи Office 365, обновленной в режиме реального времени. Он содержит сведения об устройствах, сети, подключении, звуке, видео и совместном использовании содержимого, что поможет администраторам эффективнее устранять неполадки с качеством звонков.

Как администратор Teams вы получаете полный доступ ко всем данным телеметрии в режиме реального времени для каждого пользователя. Кроме того, вы можете назначить Azure Active Directory для сотрудников службы поддержки. Дополнительные сведения об этих ролях см. в разделе ["Предоставление разрешений для сотрудников службы поддержки](set-up-call-analytics.md#give-permission-to-support-and-helpdesk-staff) и поддержки".

## <a name="where-to-find-per-user-real-time-troubleshooting-telemetry"></a>Где найти данные телеметрии для каждого пользователя в режиме реального времени

Чтобы просмотреть все сведения о собрании и данные для пользователя, перейдите в центр [Teams администрирования](https://admin.teams.microsoft.com). В **разделе "Пользователи** **UsersManage** > " выберите пользователя и откройте вкладку "Собрания **& вызовов**" на странице профиля пользователя. В **разделе** "Недавние собрания" вы увидите список собраний, на которых пользователь присутствовал в течение последних 24 часов, для которых доступна телеметрия в режиме реального *времени, включая* все собрания, в которых он выполняется. Если собрание не выполняется или нет данных телеметрии в режиме реального времени, оно будет отображаться на **прошлых собраниях**.

:::image type="content" alt-text="Снимок экрана: таблица последних собраний." source="media/recent-meetings.png" lightbox="media/recent-meetings.png":::

Чтобы получить дополнительные сведения о участниках собрания, которое выполняется, включая статистику устройства, сети и звука, найдите собрание в разделе "Недавние  собрания" и щелкните ссылку в столбце **"Участники**".

:::image type="content" alt-text="Снимок экрана: таблица сведений о участнике." source="media/participant-details.png" lightbox="media/participant-details.png":::

Чтобы просмотреть данные телеметрии заданного пользователя для выполнения собрания, включая сведения об устройстве, сети, звуке, видео и сведениях о совместном использовании содержимого, выберите идентификатор **собрания**.

:::image type="content" alt-text="Снимок экрана: данные сеанса пользователя аналитики звонков." source="media/real-time-telemetry.png" lightbox="media/real-time-telemetry.png":::

## <a name="measures-available-in-real-time-analytics"></a>Меры, доступные в Real-Time Analytics

### <a name="audio"></a>Звук
|Имя меры |Единицы измерения |Хороший порог |Описание |
|:---|:---|:---|:---|
|Дрожание |Миллисекунды |Менее 30 мс |Дрожание — это мера изменения задержки пакета для потока данных. Если это слишком высокое значение, звук может стать нестрогой. | 
|Потеря пакетов |Процент |Менее 5 % |Потеря пакетов возникает, когда пакеты данных не достигают места назначения. Процент потерянных пакетов зависит от общего количества отправленных пакетов. |
|Время кругового пути |Миллисекунды |Менее 500 мс |Время кругового пути — это время, необходимое для перемещения одного пакета от клиента к удаленной конечной точке и обратно клиенту. Большое время кругового пути может привести к задержкам при воспроизведении потока. Примером этого может быть то, что два человека на собрании непреднамерно говорят друг над другом из-за задержки. |
|Битрейт |Килобит в секунду (Кбит/с) |Больше 24 Кбит/с |Пропускная способность аудиопотока, выраженная в килобитах в секунду. |

## <a name="client-platforms-supported-for-real-time-telemetry"></a>Клиентские платформы, поддерживаемые для телеметрии в режиме реального времени

- Windows
- macOS
- Linux
- Android
- iOS

> [!NOTE]
> Teams веб-клиент (включая VDI) не поддерживает доставку телеметрии в режиме реального времени.

## <a name="teams-devices-with-support-for-real-time-telemetry"></a>Teams устройств с поддержкой телеметрии в режиме реального времени

- MTR — Surface Hub
- MTR — Teams отображения
- MTR — панель совместной работы
- IP-Телефон устройств

> [!NOTE]
> Устройства, которые присоединились к собранию с помощью решений cloud Video Interop (CVI), не поддерживаются в Real-Time Analytics.


## <a name="limitations"></a>Ограничения

- Телеметрия в режиме реального времени доступна только для запланированных собраний. Для нерегламентированных собраний, таких как "Собрание сейчас", ТСОП, звонков 1:1 и групповых звонков, телеметрия в режиме реального времени недоступна.
- Телеметрия в режиме реального времени доступна только для докладчиков запланированного трансляции. В настоящее время он недоступен для участников трансляции.
- Данные телеметрии в режиме реального времени доступны для  собрания в разделе "Последние собрания" в течение 24 часов после завершения собрания. По истечении 24 часов вы не сможете получить доступ к данным, и собрание перейдет на **прошлые собрания**. Если время собрания превышает 3 часа, данные телеметрии в режиме реального времени будут доступны только за *последние 3 часа*.
- Телеметрия недоступна в режиме реального времени при использовании более старых версий Teams. Если телеметрия недоступна, попробуйте обновить клиент.
- Если внешние или анонимные пользователи присоединяются к собранию, их отображаемое  имя будет отображаться как недоступное для сохранения конфиденциальности между клиентами.
