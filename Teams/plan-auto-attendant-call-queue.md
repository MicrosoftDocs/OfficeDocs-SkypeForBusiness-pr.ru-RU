# <a name="plan-for-teams-auto-attendants-and-call-queues"></a>Планирование автоотетарей и очередей вызовов в Teams

Автоответы позволяют настроить параметры меню для перенастройки звонков на основе входных данных вызывающего пользователя. Параметры меню, например "Для продаж, нажмите 1.  Для служб нажмите 2", чтобы автоответчик предоставил организации ряд вариантов, которые могут помочь вызывающим вызовам быстро перенаправить их, не влияя на оператора человека для обработки входящих звонков.

Очереди вызовов находятся в очередях для пунктов ожидания. В ситуациях, когда вызывателям требуется связаться с определенными сотрудниками, например с отделом продаж или службой, а не с определенным человеком, вы можете использовать очереди звонка для подключения вызывающих к группе агентов, которые могут оказать им помощь. Вызыватели будут поставлены на удержание, пока не будет доступен агент, который назначен в очереди, чтобы принять свой звонок.

Если используется вместе, автозаполнеющие и очереди вызовов могут без труда перена могут маршрутизации вызовов соответствующему лицу или отделу в вашей организации.

## <a name="auto-attendants"></a>Автосекретари

Основное назначение автозаверха — перенаправить вызывающего пользователя к соответствующему человеку или отделу на основе его ввода в предоставленные параметры меню. Вызываемого звонок можно перенаправить на определенных людей в вашей организации, чтобы звонить в очереди, где они будут ждать, чтобы поговорить со следующим доступным агентом, или к голосовой почте. Для деловых часов, неавтных часов и праздников можно у указывается несколько параметров маршрутизов.

Для создания подсказок меню можно использовать функцию "текст в речь" (системные запросы) или путем отправки записанного звукового файла. Распознавание речи принимает голосовые команды для беззвучной навигации, но звоня люди также могут перемещаться по меню с помощью клавиатурной панели телефона.

У каждого автозавода есть определенный язык и часовой пояс. Если вы разные языки или разные части мира, вы можете создать столько автозаверх, сколько потребуется для размещения вызывающего.

Для каждого автофигуры можно настроить оператор. Хотя вы можете настроить перенаправление звонков с помощью операторов в различные страны, функция операторов предназначена для того, чтобы дать возможность вызывателям общаться с определенным человеком в вашей организации, который может помочь им.

Можно настроить автозаверхов так, чтобы разрешить вызывающим пользователям искать в каталоге организации по имени или по расширению. Вы можете указать, кто доступен для поиска по каталогу, выбрав группы пользователей, которых нужно включить в поиск или исключить из него. (Это называется областью *набора номера.)*

Вызываемые звонки могут связаться с автозаверчиком по прямому номеру телефона, если он настроен, перенаправлен из другого автозаверх или из очереди звонков.

## <a name="call-queues"></a>Очередь звонков

Очереди звонка сх желтую с номером ожидания в физическом здании. Во время перенастройки звонков агентам, которые находятся в очереди, вызываемая на удержание, звонят на удержание. Очереди звонка обычно используются для функций продаж и обслуживания. Однако очереди звонков можно использовать в любой ситуации, когда количество звонков превышает внутреннюю мощность, например при приемной в загруженном помещении.

Очереди звонков позволяют использовать определенную маршрутию звонков в тех случаях, когда общее количество вызывающих пользователей в очереди или время ожидания превышает за определенные ограничения. Звонки можно перена маршрутизовыть определенным людям, голосовой почте, другим очередям звонков или автоответчикам.

Как и для автоотетарей, для каждой очереди параметров за настраивается язык. Если вы пользуется несколькими языками, вы можете использовать разные очереди вызовов. Агенты могут быть участниками нескольких очередей, если они многоязычные.

Для каждой очереди звонков вы можете указать, могут ли агенты из очереди отказаться от приема звонков и должны ли они перена маршрутизировать звонки в зависимости от их присутствия в Teams.

Вы можете назначить номер для очереди звонков, но очереди звонков не обеспечивают отдельную маршрутику для нечасов и праздников. Если в очереди звонков нет сотрудников круглосуточно, рекомендуется назначить номер телефона автозагрузщику, который перенаправляет звонок в очередь звонков в часы работы.

## <a name="prerequisites"></a>Необходимые компоненты

Для настройки автозагодеров и очередей вызовов необходимы следующие ресурсы:

- Учетная запись ресурса для каждого автозачета и каждой очереди вызовов
- Бесплатная телефонная система — лицензия виртуального пользователя для каждой учетной записи ресурса
- По крайней мере [один номер службы](getting-service-phone-numbers.md)Майкрософт, прямой маршрутный номер или гибридный номер для каждой учетной записи ресурса, которую вы хотите использовать для прямого набора
 - Номер службы может быть платным или бесплатным

Агенты, которые получают звонки из очередей звонков, должны Корпоративная голосовая связь включена как для сетевых, так и для пользователей локальной сети. 

Если ваши агенты используют приложение Microsoft Teams для звонков в очереди звонков, они должны быть в режиме TeamsOnly.

При перенастройке звонков на внешний номер телефона учетная запись ресурса, которая выполняет передачу (то есть связанную с автоответом или очередью звонков), должна иметь номер телефона и лицензию виртуального пользователя телефонной системы Microsoft 365. Кроме того:

- Для учетной записи ресурса с номером плана звонков назначьте [лицензию на план звонков.](calling-plans-for-office-365.md)
- Для учетной записи ресурса с прямым маршрутным номером назначьте политику маршрутинга голосовой [связи через Интернет.](manage-voice-routing-policies.md)

> [!NOTE]
> Прямой маршрутизинг номеров служб для автозагрузки и очередей вызовов поддерживается только для пользователей Microsoft Teams и агентов по звонкам.<br>
> Передачи между линиями связи плана звонков и прямой маршрутизов не поддерживаются.

## <a name="business-decisions"></a>Деловые решения

Перед тем как настроить автостраницу и очереди вызовов, следует принять решение о том, как использовать эти функции в вашей компании. Эти решения определяют параметры, которые вы выбираете при настройке автоотетаря и очередей вызовов.

Задокумировать ответы на эти вопросы и предоставить сведения администратору, который занимается настройкой.

- Какие языки вам нужны? Где нужны эти языки : отдел или группа?
- Хотите разрешить голосовой ввод от вызывающих или только набирающих?
- Нужна ли отдельная маршрутная маршрутка для неавных часов или праздников? Что такое часы и праздники?
- Вы хотите разрешить агентам в очереди звонков отказаться от приема звонков?
- Хотите, чтобы у агентов из очередей звонка или у вашего оператора был определенный номер вызываемой связи, если они звоните?
- Вы хотите включить [парковку](call-park-and-retrieve.md) вызовов и инкамента в своей организации для помощи в раздаточных звонках между людьми или отделами?
- Хотите записывать голосовые данные самостоятельно или использовать системный голос? (Системный голос можно легко обновить.)

## <a name="technical-decisions"></a>Технические решения

При использовании автозаверх и очередей вызовов для подключения вызывающего персонала к людям в организации необходимо принять ряд технических решений, прежде чем начать настройку.

Агентов можно добавлять в очереди вызовов следующими способами:

- Отдельные пользователи
- Списки рассылки
- Группы безопасности, включая группы безопасности с поддержкой электронной почты
- Группы и Teams в Microsoft 365

При необходимости вы можете использовать сочетание этих параметров для каждой очереди. Группы с адресами электронной почты могут использоваться для голосовой почты. Используя Teams, вы можете воспользоваться различными преимуществами, включая общее хранилище файлов и общение между агентами, общий почтовый ящик, в котором можно получить голосовую почту, и надстройку, которая может включать интеграцию с бизнес-приложениями или приложениями Power Apps.

Перед началом настройки рекомендуется выбрать стратегию добавления агентов вызовов в очереди.

Если у вас уже есть автоотетарь и инфраструктура очереди звонков, и вы переносите звонки в Teams, вам потребуется спланировать перенос существующих номеров телефонов новым автозаводщикам и очередям звонков. Для переноса [номеров](phone-number-calling-plans/port-order-overview.md) от других поставщиков может потребоваться создать заказ на перенос. Мы рекомендуем временно приобрести один или несколько новых номеров телефонов и протестировать функцию автоотетаря и потоки очереди звонков, прежде чем переходить на номера, которые сейчас находятся в службе.

*Режим конференции* — это параметр в очередях звонков, значительно уменьшая время, необходимое для подключения вызовов VoIP и звонков teams к агенту. Чтобы можно было работать в режиме конференции, агенты из очереди звонка должны использовать один из следующих клиентов:

- Последняя версия клиента Microsoft Teams для настольных пк, приложения Android или приложения для iOS
  - Телефон Microsoft Teams версии 1449/1.0.94.2020051601 или более поздней версии
  
Настройка учетных записей Teams агентами в режиме только teams. Агенты, не отвечающие требованиям, не включаются в список маршрутов вызовов.

Мы рекомендуем использовать режим конференции для очередей звонка, если все ваши агенты используют совместимые клиенты.

> [!NOTE]
> Заняты в режиме занятости не поддерживаются в режиме конференции. Агенты, которые находятся в очереди звонков, могут по-прежнему получить вызов в очереди звонков, если маршрутия на основе присутствия не включена.

## <a name="plan-your-call-routing-flow"></a>Планирование потока маршрутов вызовов

В процессе планирования рекомендуется отработать маршрутику вызовов для организации на схеме. Схема помогает определить наиболее эффективную маршрутику для людей, которые звонят в вашу организацию. Вы также можете использовать схему для определения автоотетарей и очередей вызовов, которые необходимо создать, а также связанных требований, таких как номера служб, лицензии и учетные записи ресурсов.

Давайте посмотрим, как автоответы и очереди звонков перена маршрутизят вызовы.

Автоответы перена маршрутизят все звонки одним из следующих способов:

- **Перенаправление** немедленно — звонки можно перенаправлять в одно из мест маршрутации звонков (перечисленные ниже) сразу после ответа на первоначальное приветствие или после него.
- **Перенаправление** на основе параметров набора номера: можно перенаправлять пользователей на выбор параметров, которые назначены номерам на телефонной клавиатуре (0–9). Каждому набираемой клавише можно на назначенное место назначения маршрутизов.
- **Набор номера** по имени или расширению : можно перенаправить вызывающих пользователей на номер расширения для вызываемого человека в каталоге организации или по орфографии имени.
- **Отключение** — автозавод может повесить звонок.

> [!NOTE]
> Один автозаместить может поддерживать только один способ телефонного набора.  Чтобы разрешить звонку по имени и номеру, необходимо создать автозаверяющий, который может звонить по имени, а другой — по расширению.  Каждый из этих параметров будет маршрутизироваться в отдельные автозаводы, настроенные для таких сценариев "телефонного набора".

При переадресации звонков автостраницы или очереди звонков вы можете выбрать один из следующих назначений маршрутизации звонков:

- **Человек в организации —** это человек в вашей организации, который может принимать голосовые звонки. Это может быть пользователь в Сети или пользователь, который использует сервер Skype для бизнеса Server, который является пользователем локальной сети.
- **Приложение голосовой** почты — другой автозавод или очередь вызовов. Выберите учетную запись ресурса, связанную с назначением.
- **Внешний номер телефона** — любой номер телефона. (См. [технические сведения о внешнем передаче).](create-a-phone-system-auto-attendant.md#external-phone-number-transfers---technical-details)
- **Голосовая почта** — это почтовый ящик голосовой почты, связанный с группой Microsoft 365, которую вы указали.
- **Оператор** (только для автоотетаря) — оператор, задавающийся для автозавода. Определение оператора не является обязательным. Оператором может быть любой из других назначений в этом списке.

Автоответы предлагают отдельные параметры маршрутиза вызовов для звонков, которые будут получены в не в нее в неявные часы и в праздники. Без перенаправления вызовов можно использовать все параметры, указанные выше, в то время как маршрутка праздников позволяет перенаправлять и отключать только звонок, но не набирать клавишу.

В очередях вызовов вызываемая звонок перенастройка на удержание до тех пор, пока не будет доступен агент, который назначен в очереди. Существует две ситуации, в которых звонок может быть направлен из очереди:

- **Переполнение звонков** — если количество звонков в очереди превышает за установленный предел, то новые звонки перенаправляются из очереди.
- **Время ожидания звонка** — если звонок отошел в очереди дольше установленного параметра времени ожидания, он будет перенаправлен из очереди.

Звонки, перенаправленные из очереди, могут быть отправлены в любую из указанных выше маршрутов звонков, за исключением оператора. (В очередях вызовов нет операторов, но вы можете перенаправить вызывающего оператора в ту же точку, что и для оператора, настроенного для автоотправления.)

В примере ниже показан пример маршрутиза вызовов с использованием автоотетаря и очередей вызовов.

![Схема маршрутиза вызовов с использованием автоотетаря и очередей вызовов](media/attendant-and-queue-call-routing.png)

В примере выше:

- Ноль (0) клавиша перенаправляет вызывающих пользователей на оператора. Оператор для этого автофигуры настроен как "Сотрудник **организации".**
- Один (1) ключа перенаправляет вызывающих пользователей в очередь слева. Эта очередь вызовов подключена к команде, которая содержит назначенную в ней отдел продаж.
- Два (2) ключа перенаправляют вызывающих пользователей в очередь вызовов службы поддержки. Эта очередь вызовов подключена к группе, которая содержит службу поддержки, назначенную группе.
- В очереди звонков службы поддержки есть прямой номер телефона, который может вовсюю звонить автоотправка. Если автоотвека отвечает на строку поддержки, это позволит использовать отдельные часы и маршрутку праздников.
- Три (3) ключа перенаправляют пользователей на другого автозаправщика для каталога организации. Автозаверчик позволяет вызывающим пользователям звонить отдельным пользователям в организации, набирая их имена или расширения.

Мы рекомендуем создать одну или несколько схем, похожих на схему выше, чтобы составить маршрутику вызовов. Не забудьте включить в схему следующую или сопутствующей документацию:

- Какие автоотправицы будут иметь прямой доступ по номерам телефонов?
- Каковы требования к внечасовой и праздничным маршрутизам для каждого автоотетаря?
- Состав участников для каждой очереди вызовов. (Вы можете добавлять пользователей по отдельности или со карте очереди с различными группами. Сопоставление очереди с командой — самый универсальный инструмент.)

Ниже 2010 г.

- Взгляните на существующую систему звонков и проанализируйте типы и частоту входящих звонков. Используйте эти сведения для информирования автозавода и структуры очереди вызовов.
- Чтобы перенапорять вызовы как можно быстрее, вставьте наиболее распространенные варианты в начале меню.
- Избегайте прямого подключения номеров служб к очередям звонка, если очереди доступны без 07.02.2017. Очереди вызовов не позволяют обрабатывать отдельные вызовы в не праздничные часы или часы. Если вы хотите, чтобы в очереди был прямой номер, назначьте номер автозаправителю, который автоматически перенаправляет в очередь в часы работы.
- Если вы получаете множество звонков с запросом основных сведений о вашей компании, таких как рабочие часы, местонахождение или адрес веб-сайта, можно создать автоответ, чтобы отвечать на эти вопросы с записанными сообщениями.
- В списке элементов меню должно быть не более пяти элементов. При этом может быть трудно запомнить более пяти параметров. Используйте вложенные автоотетары, если вам нужны дополнительные параметры для правильного перенаводки звонка.
- Сначала опишите службу, а затем вариант нажать (например, для sales нажмите 1), а не наоборот (например. Нажмите 1 для связи с отделом продаж.
- Термины, понятные вызывателям, а не внутренние термины.
- Избегайте частых обновлений маршрутов вызовов. Если вы измените параметры меню для автозапуска в будущем, вызовите его в голосовых подсказках в течение первых 30 дней.

## <a name="getting-started"></a>Начало работы

Завершив задачи планирования, указанные в этой статье, выполните указанные здесь действия, чтобы настроить функцию автоотетаря и очередей вызовов.

1. Получите номера служб, необходимые для автозаполнения и очередей вызовов, которые вы хотите сделать доступными, прямой набор номера из-за пределов организации. Это может быть [перенос номеров от другого поставщика или](phone-number-calling-plans/transfer-phone-numbers-to-teams.md) запрос новых [номеров служб.](getting-service-phone-numbers.md)

2. Получите [телефонную систему — лицензию виртуального пользователя](teams-add-on-licensing/virtual-user.md) для каждой учетной записи ресурса, которую вы планируете создать. Эти лицензии бесплатны, поэтому мы рекомендуем получить еще несколько на случай, если вы решите внести изменения в учетные записи ресурсов в будущем.

3. [Создайте учетную запись ресурса](manage-resource-accounts.md) для каждого автозавода и очереди вызовов, которые вы хотите создать. Назначьте каждой учетной записи телефонную систему — лицензию виртуального пользователя и, при желании, номер службы.

4. [Создайте праздники,](set-up-holidays-in-teams.md) для которых нужно создать отдельную маршрутику вызовов в автостранице.

5. При желании [вы можете настроить](call-park-and-retrieve.md) приозвать и инкаментировать вызовы, если вы хотите использовать эту функцию для перенастройки.

6. Создайте группы, которые должны содержать агентов по звонкам в очередях вызовов.

7. Если вы планируете разрешить набор по расширению, убедитесь, что добавите добавок пользователей в их профиль Azure Active Directory.

Вы можете создавать автозаводы и очереди вызовов, как только вы завершите все действия, которые вы выполнили выше. Так как автоответы и очереди звонков могут перенаправлять звонки друг другу, обратитесь к созданной схеме рабочего процесса, чтобы определить, какую очередь звонков следует создать. В приведеном выше примере перед созданием основного автозавершника Contoso необходимо создать очереди вызовов продаж и поддержки, так как основной автозавершник должен направлять вызывающих клиентов в очереди вызовов и поддержку.
