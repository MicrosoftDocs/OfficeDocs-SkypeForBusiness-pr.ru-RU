# <a name="plan-for-teams-auto-attendants-and-call-queues"></a>Планирование авто Teams и очередей вызовов

Автоответы позволяют настроить параметры меню для перенастройки звонков на основе входных данных вызывающего пользователя. Параметры меню, например "Для продаж, нажмите 1.  Для служб нажмите 2", чтобы автоответчик предоставил организации ряд вариантов, которые обеспечивают быстрое перенаправление вызывающего звонка по месту назначения без использования оператора человека для обработки входящих звонков.

Очереди вызовов находятся в очередях ожидания для вызывающих. В ситуациях, когда вызывателям нужно связаться не с конкретным человеком, а с определенным специалистом, например с отделом продаж или службой, вы можете использовать очереди вызовов для подключения вызывающих лиц к группе агентов, которые могут им помочь. Вызыватели будут поставлены на удержание, пока не будет доступен агент, который назначен в очередь, чтобы принять свой звонок.

В этом режиме автоотводчики и очереди вызовов могут легко перенау числеть вызывающего персонала соответствующему человеку или отделу в вашей организации.

## <a name="auto-attendants"></a>Автосекретари

Основное назначение функции автозавершего — перенаправить вызывающего пользователя к соответствующему человеку или отделу с учетом его входных данных в предоставленные параметры меню. Вызывающих можно перенаправить на определенных людей в вашей организации, чтобы звонить в очереди, где они будут ждать, чтобы поговорить со следующим доступным агентом, или на голосовую почту. Для деловых часов, неавных часов и праздников можно укаварить различные параметры маршрутизов параметров параметров.

Для создания подсказок меню можно использовать функцию "Текст в речь" (системные запросы) или путем отправки записанного звукового файла. Распознавание речи принимает голосовые команды для свободной навигации с помощью голосовых команд, но для навигации по меню также можно использовать клавиатурную панель телефона.

У каждого автозавода есть определенный язык и часовой пояс. Если вы делите свою деятельность на нескольких языках или в разных частях мира, вы можете создать столько автозаверх, сколько потребуется для размещения вызывающего персонала.

Для каждого автозавода можно настроить оператор. Хотя вы можете настроить перенаправление звонков по различным назначениям, функция операторов позволяет вызывателям общаться с определенным человеком в вашей организации, который может им помочь.

Для автозаверх можно разрешить вызывающим пользователям искать в каталоге организации поиск по имени или по расширению номера. Вы можете указать, кто доступен для поиска по каталогу, выбрав группы пользователей, которых нужно включить или исключить. (Это называется *областью набора* номера.)

Звоня из другого автозаверха или очереди звонков, вызываемые вызываемые участники могут связаться с автозаправщиком по прямому номеру телефона(если он настроен) или перенаправлен.

## <a name="call-queues"></a>Очередь звонков

Очередь вызовов аналогична ожиданиям в физическом здании. При перенастройки звонков на агентов в очереди вызыватели находятся на удержании. Очереди вызовов обычно используются для функций продаж и обслуживания. Однако очереди звонков можно использовать в любой ситуации, когда количество звонков превышает внутреннюю пропускную способность, например для администраторов в загруженном помещении.

Очереди звонков позволяют использовать определенную маршрутику звонков в тех случаях, когда общее количество вызывающих или время ожидания превышает за определенные ограничения. Звонки можно перенаадвать определенным людям, голосовой почте, другим очередям звонков или автоответчикам.

Как и для автоотетарей, в очередях вызовов есть языковой параметр. Вы можете использовать разные очереди вызовов, если вы используете несколько языков. Агенты могут быть участниками нескольких очередей, если они многоязычны.

Для каждой очереди звонков вы можете указать, могут ли агенты из очереди отказаться от приема звонков и должны ли они перена должны быть перена которые должны перенаться с учетом их указаний наличия в Teams.

Вы можете назначить номер телефона очереди звонков, но очереди звонков не обеспечивают отдельную маршрутику звонков в не часы и праздники. Если в очереди звонков нет персонала 24/7, мы рекомендуем назначить номер телефона автозаправщику, который будет перенаправляться в очередь звонков в часы работы.

## <a name="prerequisites"></a>Необходимые компоненты

Для настройки автоотетаря и очередей вызовов необходимы следующие ресурсы:

- Учетная запись ресурса для каждого автозавода и каждой очереди вызовов
- Бесплатная телефонная система — лицензия виртуального пользователя для каждой учетной записи ресурса
- По крайней мере один номер [службы](getting-service-phone-numbers.md)Майкрософт, номер прямой маршрутки или гибридный номер для каждой учетной записи ресурса, которую вы хотите использовать для прямого набора
 - Номер службы может быть платным или бесплатным

Для агентов, которые получают звонки из очередей звонков, Корпоративная голосовая связь в сети или локально. Кроме того, если в очередях звонков есть номера прямой маршрутики, агентам, которым требуется конференц-связь или перенаправка звонков, также требуются:

- Политика маршрутации голосовой почты в Интернете, назначенная, если в очереди вызовов используется режим передачи.
- Лицензия на аудиоконференцию или политика маршрутизации голосовой сети, назначенная, если в очереди вызовов используется режим конференции

Если ваши агенты используют приложение Microsoft Teams для звонков в очереди звонков, они должны быть в режиме TeamsOnly.

При перенастройке звонков на внешний номер телефона учетная запись ресурса, которая выполняет передачу (то есть связанную с автоответом или очередью звонков), должна иметь лицензию виртуального пользователя Microsoft 365 телефонная система и одну из следующих лицензий:

- Лицензия [на план звонков](calling-plans-for-office-365.md) и номер телефона, который назначен
- Политика [маршрутинга голосовой связи](manage-voice-routing-policies.md) в Интернете (назначение номеров телефонов не является обязательным при использовании прямой маршрутии)

> [!NOTE]
> Номера служб прямой маршрутики для автозагрузки и очередей вызовов поддерживаются только для Microsoft Teams пользователей и агентов вызовов.<br>
> Переводы между линиями связи плана звонков и прямой маршрутизов не поддерживаются.<br>
> В гибридном сценарии учетная запись ресурса должна быть создана локально. Дополнительные сведения см. в [теме Планирование очередей вызовов в облаке.](/skypeforbusiness/hybrid/plan-call-queue)

## <a name="business-decisions"></a>Деловые решения

Перед тем как настроить функции автозавода и очередей вызовов, необходимо принять некоторые решения о том, как использовать эти функции в вашей компании. Эти решения определяют параметры, которые вы выбираете при настройке автоотехов и очередей вызовов.

Задокумировать ответы на эти вопросы и предоставить сведения администратору, который занимается настройкой.

- Какие языки вам нужны? Где нужны эти языки — какой отдел или группа?
- Хотите ли вы разрешить голосовой ввод от вызывающих или только набрав входные данные?
- Требуется ли отдельная маршрутизовая маршрутия для неавных часов или праздников? Часы и праздники
- Хотите разрешить агентам из очереди звонков отказаться от приема звонков?
- Хотите, чтобы у агентов в очередях вызовов или у вашего оператора был определенный ИД звонив при выходе из службы?
- Вы хотите включить [в](call-park-and-retrieve.md) организации автостоянку и сбор вызовов для помощи в раздаточных звонках между людьми или отделами?
- Хотите записать собственный голос или использовать системный голос? (Системный голос можно легко обновить.)

## <a name="technical-decisions"></a>Технические решения

При использовании автозаверх и очередей вызовов для подключения вызывающего персонала к людям в организации необходимо принять ряд технических решений перед началом настройки.

Агентов можно добавлять в очереди вызовов следующими способами:

- Отдельные пользователи
- Списки рассылки
- Группы безопасности, включая группы безопасности с поддержкой электронной почты
- Microsoft 365 Группы или Teams

При необходимости эти параметры можно использовать для каждой очереди. Группы с адресами электронной почты можно использовать для голосовой почты. Использование Teams дает ряд преимуществ, включая совместное хранение файлов и общение между агентами, общий почтовый ящик, в котором можно получить голосовую почту, и надежную платформу, которая может включать интеграцию с бизнес-приложениями или Power Apps.

Рекомендуем выбрать стратегию добавления агентов вызовов в очереди перед началом настройки.

Если у вас уже есть инфраструктура автоотетаря и очереди звонков и вы переносите их в Teams, вам потребуется план переноса существующих номеров телефонов новым автозаводщикам и очередям звонков. Для переноса номеров от других поставщиков может потребоваться создать заказ на перенос номеров. [](phone-number-calling-plans/port-order-overview.md) Мы рекомендуем временно приобрести один или несколько новых номеров телефонов и протестировать потоки очереди автослужб и звонков перед их переключение на номера, которые в настоящее время находятся в службе.

*Режим конференции* — это параметр в очередях звонков, который значительно сокращает время, необходимое для подключения к Teams вызовов VOIP и звонков по ПСПУ агенту. Чтобы режим конференции работал, агенты из очереди зовов должны использовать один из следующих клиентов:

- Последняя версия клиента Microsoft Teams, приложения Для Android или iOS
  - Microsoft Teams телефон версии 1449/1.0.94.2020051601 или более поздней
  
Установите для Teams агентов режим Teams только для агентов. Агенты, не отвечающие требованиям, не включаются в список маршрутов зовов.

Мы рекомендуем использовать режим конференции для очередей вызовов, если все ваши агенты используют совместимые клиенты.

## <a name="plan-your-call-routing-flow"></a>Планирование потока маршрутизов вызовов

В процессе планирования рекомендуется отработать маршрутику вызовов для организации на схеме. Схема помогает определить наиболее эффективную маршрутику для людей, которые звонят в вашу организацию. Вы также можете использовать схему для определения автоотчетов и очередей вызовов, которые необходимо создать, а также связанных требований, таких как номера служб, лицензии и учетные записи ресурсов.

Давайте посмотрим, как автоответы и очереди звонков перена могут перенанастраить звонки.

Автоответы маршрутизят все звонки одним из следующих способов:

- **Перенаправление** немедленно — звонки можно перенаправлять в одно из назначений маршрутизации звонков (перечисленные ниже) сразу после ответа или после первоначального приветствия.
- **Перенаправление** с учетом параметров набора номера. Можно перенаправлять вызывающих пользователей для выбора параметров, которые назначены номерам на телефонной клавиатуре (0–9). Каждому клавише набора номера можно на назначено назначение маршрутизов пунктов перенаправления зовов.
- **Набирайте** номера пользователей по имени или расширению: можно перенаправить вызывающих пользователей, чтобы набрать номер расширения для человека, которого они пытаются найти, в каталоге организации или внести в него орфографию.
- **Отключить** — автозаводщик может повесить трубку.

> [!NOTE]
> Отдельный автозамест может поддерживать только один способ набора номера.  Чтобы разрешить вызывающим звонить по имени и номеру, необходимо создать автоотводчик, у которого есть возможность набора номера по имени, а для другого — по имени.  Каждый из этих параметров будет маршрутизироваться в отдельные автозаводцы, настроенные для таких сценариев набора номера.

При переадресации звонков автозаправщиком или очередью звонков вы можете выбрать один из следующих вариантов:

- **Человек в организации —** это человек в вашей организации, который может принимать голосовые звонки. Это может быть пользователь в Интернете или пользователь, который является пользователем локальной Skype для бизнеса Server.
- **Голосовое приложение** — другой автосекретарь или очередь вызовов. Выберите учетную запись ресурса, связанную с назначением.
- **Внешний номер телефона** — любой номер телефона. (См. [технические сведения о внешней передаче](create-a-phone-system-auto-attendant.md#external-phone-number-transfers---technical-details).)
- **Голосовая почта** — почтовый ящик голосовой почты, связанный с группой Microsoft 365, которую вы указали. Вы можете выбрать, требуются ли расшифровки голосовой почты и системное приглашение "Оставьте сообщение после сигнала" .
- **Оператор** (только автоотекадр) — оператор, определенный для автозавода. Определение оператора является необязательным. Оператором может быть любое из других назначений в этом списке.

Автоответы предлагают отдельные параметры маршрутиза вызовов для звонков, полученных не в часы работы и праздники. Маршрутизовы в не часах позволяют использовать все указанные выше параметры, в то время как маршрутия праздников позволяет перенаправлять и отключать только звонок, но не набирать клавишу.

В очередях вызовов вызываемая звонок будет удержан до тех пор, пока не будет доступен агент, который назначен в очередь, чтобы принять звонок. Существует две ситуации, в которых вызываемого человека могут перенаправить из очереди:

- **Переполнение звонков:** если количество звонков в очереди превышает установленный предел, новые звонки перенаправляются из очереди.
- **Время ожидания звонка:** если вызываемый звонок был в очереди дольше установленного времени ожидания, он будет перенаправлен из очереди.

Звонки, перенаправленные из очереди, можно перенаправлять в любое из указанных выше маршрутов звонков, за исключением оператора. (В очередях вызовов нет операторов, но вы можете перенаправить вызывающего оператора в ту же точку, что и оператор, настроенный для автоотправления.)

В примере ниже показан пример маршрутиза вызовов с использованием автоотехов и очередей вызовов.

![Схема маршрутиза вызовов с использованием автоотехов и очередей вызовов.](media/attendant-and-queue-call-routing.png)

В примере выше:

- Ноль (0) перенаправляет вызывающих на оператора. Оператор для этого автозавода настроен как сотрудник **организации.**
- Один (1) ключ перенаправляет вызывающих пользователей в очередь звонка с продаж. Эта очередь вызовов подключена к команде, которая содержит отдел продаж, который назначен в очередь.
- Два (2) ключа перенаправляют вызывающих в очередь вызовов службы поддержки. Эта очередь вызовов подключена к команде, которая содержит команду поддержки, назначенную группе.
- В очереди звонков службы поддержки есть прямой номер телефона через автозаправщик. Если на строку поддержки отвечает автозаводщик, это позволяет использовать отдельные часы и маршруты праздничных зовов.
- Три (3) ключа перенаправляют пользователей к другому автозаправщику для каталога организации. Автоотводчик каталога организации позволяет вызывающим звоня людям в организации, набрав их имя или расширение.

Мы рекомендуем создать одну или несколько схем, похожих на приведенную выше, чтобы составить маршрутику зова. Не забудьте включить в схему или сопутствующие документы:

- Какие автоследники будут иметь прямой доступ по номерам телефонов?
- Каковы требования к не часам и праздничным маршрутиям для каждого автоотвода?
- Участие в каждой очереди вызовов. (Вы можете добавить пользователей по отдельности или соеоставить очередь с различными типами групп. Сопоставление очереди с командой обеспечивает самый универсальный возможности.)

## <a name="getting-started"></a>Начало работы

После выполнения задач планирования, указанных в этой статье, выполните указанные в этой статье действия, чтобы настроить автоотехожники и очереди вызовов.

1. Получите номера служб, необходимые для автозаполнения и очередей вызовов, которые должны быть доступны при прямом наборе номера из-за пределов организации. Это может [быть перенос номеров от другого поставщика](phone-number-calling-plans/transfer-phone-numbers-to-teams.md) или запрос новых [номеров служб.](getting-service-phone-numbers.md)

2. Получите [лицензию телефонная система — виртуальный пользователь](teams-add-on-licensing/virtual-user.md) для каждой учетной записи ресурса, которую вы планируете создать. Эти лицензии бесплатны, поэтому мы рекомендуем получить дополнительные лицензии на случай, если вы решите внести изменения в учетные записи ресурсов в будущем.

3. [Создайте учетную запись ресурса](manage-resource-accounts.md) для каждого автозавода и очереди вызовов, которые вы хотите создать. Назначьте каждой учетной телефонная система — лицензию виртуального пользователя и (при желании) номер службы.

4. [Создайте праздники,](set-up-holidays-in-teams.md) для которых нужно создать отдельную маршрутику зовов в автоотехниках.

5. Если вы хотите [использовать](call-park-and-retrieve.md) эту функцию для передачи вызовов, при желании вы можете настроить приозвать и найти их.

6. Создайте группы, которые должны содержать агентов вызовов для очередей вызовов.

7. Если вы планируете разрешить набор номеров по расширению, убедитесь, что вы добавили добавимый номер Azure Active Directory профиля.

После завершения действий, которые вы выполнили выше, вы можете создать автоспутники и очереди вызовов. Так как автоответы и очереди звонков могут перенаправлять вызовы друг другу, обратитесь к схеме рабочего процесса, которую вы создали, чтобы определить, какой автозаправщик или очередь звонков должна быть создана в первую очередь. В приведеном выше примере перед созданием главного автозавершника Contoso необходимо создать очереди зовов продаж и поддержки, так как основной автозавершник должен направлять вызывающего персонала в очереди вызовов продаж и поддержки.

- [Создание очереди вызовов](create-a-phone-system-call-queue.md)
