# <a name="manage-meeting-policies-in-teams"></a>Управление политиками собрания в группах

::: zone target="docs"
Политик собраний используются для управления функциональные возможности, доступные для участников собраний, запланированных пользователями в вашей организации собрания. После создания политики и внесите изменения, затем можно назначить пользователям в политике. Управление политиками собрания в центре администрирования группами Майкрософт или с [помощью PowerShell](teams-powershell-overview.md).

Вы можете реализовать политики следующими способами, влияющей на собрание работы пользователей перед собранием запускает, во время собрания или собрания. 

|Тип реализации  |Описание  |
|---------|---------|
|Организатора    |При реализации политики для-Организатор всем участникам собрания наследуют политику организатора. К примеру **Автоматически одобрять людей** — это политики для-Организатор и элементов управления ли пользователям присоединиться к собранию непосредственно или Ожидание в зале ожидания для собраний, запланированных пользователем, который назначается политика.          |
|На пользователя    |При реализации политики уровня пользователя только политики уровня пользователя применяется для ограничения определенные функциональные возможности для участников организатора и/или собрания. Например **Разрешить начать собрание** будет политики уровня пользователя.     |
|Организатора и уровня пользователя     |При реализации сочетание-организатора и индивидуальной политики определенные функциональные возможности ограничены для участников на основе их политику и политики организатора собрания. К примеру **облако разрешить запись** —-организатора и индивидуальной политики. Включите этот параметр, чтобы разрешить Организатор собрания и участников для запуска и остановки записи. 

По умолчанию создается политика, с именем Global (по умолчанию в масштабах организации). Все пользователи в вашей организации будет назначен эта политика собрания по умолчанию. Можно внести изменения в этой политике или создание одного или нескольких настраиваемых политик и назначение пользователям. При создании пользовательскую политику, можно разрешить или запретить определенные функциональные возможности доступными для пользователей и назначьте его одному или нескольким пользователям, которые будут иметь параметров, примененных к ним. 

## <a name="change-or-create-a-meeting-policy"></a>Изменение или создание политики собрания

Чтобы изменить или создать политику собрания, перейдите к > Центр администрирования группами Майкрософт **собраний** > **политик собраний**. Выберите соответствующую политику в списке или выберите **новую политику**. При создании новой политики, добавьте имя и описание. Имя не может содержать специальные символы или превышать 64 символов. Выберите параметры, а затем выберите **Сохранить**.

Допустим, у вас есть несколько пользователей и для ограничения пропускной способности, который требуется их собрания. Вам нужно создать новый пользовательскую политику с именем «Ограниченной пропускной способностью» и отключить следующие параметры:

В разделе **& звук видео**:
- Отключить облачных записи
- Отключить видео разрешенных IP-адресов

В разделе **общий доступ к содержимому**:
- Отключение режима совместного использования экрана
- Отключить доски
- Отключить общие заметки

Назначьте политику для пользователей.

> [!NOTE] 
> Пользователь может быть назначен только один собрания политики за раз. 

## <a name="assign-a-meeting-policy-to-users"></a>Назначение политики собрания пользователям

Если для одного пользователя в случае применения политики, выберите **пользователей** на левой панели переходов и нажмите кнопку отображаемое имя пользователя. На странице пользователя, рядом с пунктом **назначения политики**выберите команду **Изменить**. В области **Изменение политик пользователей** в разделе **Политика собрания**, выберите политика собрания из раскрывающегося списка и выберите команду **Сохранить**. В списке пользователей можно также назначить политики. Для этого выберите пользователя, нажав кнопку слева от отображаемое имя пользователя. Выберите **Изменить параметры**. В области **Изменение параметров** в разделе **Политика собрания**, выберите параметр из раскрывающегося списка и выберите команду **Сохранить**. 
 
Если политика применяется к нескольким пользователям, выберите **пользователей** на панели навигации слева выберите каждого пользователя, нажав кнопку слева от имени пользователя и нажмите кнопку **Изменить параметры**. В области **Изменить параметры** в разделе **Политика собрания**выберите параметр из раскрывающегося списка и выберите команду **Сохранить**.
 
Можно также назначить политику собрания для одного или нескольких пользователей следующим образом:

1. Перейдите в **Центр администрирования группами Майкрософт** > **собраний** > **политик собраний**.
2. Выберите политику, нажав кнопку слева от имени политики.
3. Выберите **Управление пользователями**.
4. В области **Управление пользователями** поиска для пользователя по отображаемому имени или по имени пользователя, выберите имя и выберите команду **Добавить**. Повторите этот шаг для каждого пользователя, который вы хотите добавить.
5. По завершении добавления пользователей нажмите кнопку **Сохранить**.
 
> [!NOTE] 
> Не удается удалить политику, если пользователи назначаются. Сначала необходимо назначить разные политики для всех заинтересованных пользователей и удалите исходное политики.

### <a name="allow-transcription"></a>Разрешить транскрибирования

Это сочетание-организатора и индивидуальной политики. Этот параметр управляет подписи и транскрибирования функции доступны ли при воспроизведении записи приглашений на собрания. Если отключить этот режим, параметры **поиска** и **копия** не будут доступны при воспроизведении запись собрания. Пользователя, начавшего записи должно этот параметр включен, чтобы записи также включает в себя транскрибирования. 

Обратите внимание, что транскрибирования для записанных собраний в данный момент поддерживается только для пользователей, имеющих язык в группах, английский и когда английский разговорный участниками собрания.

![собрания политик transcription.png](media/meeting-policies-transcription.png)

### <a name="allow-cloud-recording"></a>Разрешить облачных записи

Это сочетание-организатора и индивидуальной политики. Этот параметр определяет, могут быть записаны собрания этого пользователя. Запись может запускаться с организатором собрания или другой участник собрания, если параметр политики включен для участника, и они прошедшего проверку пользователя из той же организации.

Люди за пределами вашей организации, такие как федеративным и анонимным пользователям, не может начать запись. Гости не удается запустить или остановить запись. 

![собрания политик recording.png](media/meeting-policies-recording.png)

Рассмотрим следующий пример.

|Пользователь |Политика собрания  |Разрешить облачных записи |
|---------|---------|---------|
|Daniela | Глобальный   | False |
|Аманда | Location1MeetingPolicy | True|
|Джон (внешних пользователей) | Неприменимо | Неприменимо|

Не удается записать организованы Daniela и Аманда, который имеет включенные параметры политик, не удается записать организованы Daniela. Можно записать организованы Аманда, однако Daniela, имеющий отключен параметр политики и John, являющийся внешних пользователей, не удается записать организованы Аманда.

Для получения дополнительных сведений об облаке запись собрания, видеть [группами в облаке запись собрания](cloud-recording.md).

### <a name="allow-ip-video"></a>Разрешить IP-адрес видео

Это сочетание-организатора и индивидуальной политики. Видео — это основной компонент на собрания. В некоторых организациях администраторов может оказаться более контроль над собрания пользователей, которые получают видео. Этот параметр определяет ли видео может быть включен в собраниях, размещенные пользователем и в 1:1 вызовов и вызовов групповой работы пользователя. Организованы пользователь, имеющий этот параметр включен, вносили видео в собрании с участниками собрания, если участники собрания также политикой. Участники собрания, у которых нет любые назначенные политики (например, анонимные и федеративных участникам) наследуют политику организатора собрания.

![конференц политик аудио видео settings.png](media/meeting-policies-audio-video-settings.png)

Рассмотрим следующий пример.

|Пользователь |Политика собрания  |Разрешить видео IP-адресов |
|---------|---------|---------|
|Daniela   | Глобальный   | True        |
|Аманда    | Location1MeetingPolicy        | False      |

Собрания, размещенного в Daniela разрешение видео включена. Daniela могут присоединиться к собранию и включите видео. Аманда нельзя включить видео в Daniela на собрание, поскольку его Аманда политика установлена равной не разрешать видео. Аманда можно просмотреть видео, общий для других участников собрания.

В собраниях, размещенного в Аманда ни один можно включить видео, вне зависимости от политики видео, назначенные им. Это означает, что Daniela не удается включить видео в собраниях по Аманда.  

Если Daniela вызывает Аманда с видео, Аманда можно ответить на звонок с помощью только со звуком.  При соединения, Аманда могут просматривать видео Daniela, но не может включить видео. Если Аманда вызывает Daniela, Daniela можно ответить на звонок, аудио и видео. Когда соединения, Daniela можно включить или отключить ее видео при необходимости.

### <a name="media-bit-rate-kbs"></a>Скорость передачи мультимедиа (КБ)

Это-организатора политики. Этот параметр определяет скорость потока мультимедиа для аудио, видео и на основе видео приложения, общий доступ к передач в вызовах и встречи пользователя. Оно применяется для исходящей связи и downlink обхода мультимедиа для пользователей в звонок или собрания. Этот параметр позволяет четко контролировать управление пропускной способности в вашей организации. В зависимости от сценариев собраний, необходимые для пользователей рекомендуется использовать недостаточно пропускной способности на месте для высокого качества взаимодействия. Минимальное значение — 30 Кбит/с, максимальное значение зависит от сценария собрания. Чтобы узнать больше о рекомендуемого пропускной способности для высокого качества собрания, звонки и live события в группах, просмотрите [требования к пропускной способности](prepare-network.md#bandwidth-requirements).

Если недостаточно пропускной способности для собрания, участники получат сообщение, указывающее, неполадки сетевого подключения.

Для собраний, которые должны наибольший видеосвязи качества, такие как CEO платы собраний и группами live события, мы рекомендуем пропускная способность равным 10 Мбит/с. Даже в том случае, если задано максимальное качества, стек мультимедиа группами преобразует в условиях низкой пропускной способности сети при обнаружении определенных условиях сети, в зависимости от сценария.

### <a name="screen-sharing-mode"></a>Полноэкранный режим совместного доступа

Это сочетание-организатора и индивидуальной политики. Этот параметр определяет, будет ли настольных систем и/или окне общий доступ к приложениям во время собрания пользователя. Участников, которые не имеют политики, назначенные (для пример анонимных, гостя, B2B и федеративных участников) наследуют политику организатора собрания.

|Значение параметра |Поведение  |
|---------|---------|
|**Весь экран**    | Полноэкранный режим общего доступа и общий доступ к приложениям разрешено на собрании |
|**Одно приложение**   | Общий доступ к приложениям приложениям во время собрания        |
|**Отключено**     |Совместное использование рабочего стола и общий доступ к приложениям отключена на собрании.       |

Рассмотрим следующий пример.

|Пользователь |Политика собрания |Полноэкранный режим совместного доступа |
|---------|---------|---------|
|Daniela  | Глобальный   | Весь экран |
|Аманда   | Location1MeetingPolicy  | Отключено |

Собрания, размещенного в Daniela разрешить участникам собрания для совместного использования их весь экран или конкретного приложения. Если Аманда присоединяется Daniela's собрания, Аманда не могут совместно использовать свой экран или отдельного приложения как установил параметр отключен. В собраниях, размещенного в Аманда не может совместно использовать свой экран или одно приложение, вне зависимости от совместного использования режима политики, назначенные им экрана. Это означает, что Daniela не могут совместно использовать свой экран или отдельного приложения в собраниях по Аманда.  

В настоящее время пользователи не могут воспроизвести видео и совместно использовать экран в ходе собрания групп, если при использовании Google Chrome.

### <a name="allow-a-participant-to-give-or-request-control"></a>Разрешить участника для предоставления или запросить управление

Это политики уровня пользователя. Этот параметр определяет, может ли пользователь передать управления общим рабочим столом или окна другим участникам собрания. Чтобы предоставить управление, наведите указатель мыши на верхней части экрана. 

Если этот параметр включен для пользователя, параметр **Предоставить управление** отображается в верхней панели в сеансе совместного доступа. 

![собрания политик присвойте control.png](media/meeting-policies-give-control.png)

Если параметры отключена для пользователя, параметр **Предоставить управление** недоступен.

![Meeting-policies-give-Control-not-Available.PNG](media/meeting-policies-give-control-not-available.png)

Рассмотрим следующий пример.

|Пользователь |Политика собрания  |Разрешить участника предоставить или запросить управление |
|---------|---------|---------|
|Daniela   | Глобальный   | True       |
|Babek    | Location1MeetingPolicy        | False   |

Daniela можно передать управление общим рабочим столом или окно другим участникам собрания, упорядоченные по Babek, тогда как Babek нельзя передать управление другим участникам.

### <a name="allow-an-external-participant-to-give-or-request-control"></a>Разрешить внешних участников предоставить или запросить управление

Это политики уровня пользователя. Этот параметр определяет, будет ли внешних участников в ходе собрания можно передать управления общим рабочим столом или окно другим участникам собрания. Внешние участники в собраниях групп можно классифицировать следующим образом:  

   - Анонимный пользователь
   - Гости  
   - B2B пользователя
   - Федеративный пользователь  

Является ли федеративные пользователи могут предоставить элемент управления для внешних пользователей во время общего доступа управляет параметр **Разрешить внешних участников предоставить или запросить управление** в своей организации.

Даже в том случае, если собрание упорядочено по Аманда Daniela могут совместно использовать наборов слайдов PowerPoint слайда. Аманда можно просмотреть слайд наборов слайдов PowerPoint, совместно с другими пользователями в собрании, несмотря на то, что она не могут совместно использовать наборов слайдов PowerPoint слайда.
