# <a name="about-the-upgrade-framework"></a>Схема перехода

Чтобы помочь избавят вас от обновления пути, мы применил проверенных платформу для реализации изменений. Как показано ниже, каждый шаг в framework создает на предыдущем шаге и для получения оптимальных результатов рекомендуется следующие действия в указанном порядке.  

Начнем с собрать правом заинтересованные стороны и определение плана обновления (например, FTP. область, цели и временной шкалы). План на месте, подтвердите среды технического и конечных пользователей все готово для групп. Затем реализации обновления поэтапно, перемещение из пилотной обновлением всей организации, когда все будет готово. Как только вашей организации в группах, установите оперативный план, который отслеживает качество и ускоряет адаптацию пользователей.

## <a name="pre-upgrade"></a>Подготовка к обновлению

**Подготовка групп в вашей организации**. Для обеспечения успешного обновления к группам, важно выделить достаточное количество времени для подготовки. Не только будет организации иметь возможность быстро начать осознавать значение групп, вы сможете ускорить процесс обновления Скайп для бизнеса сразу же команды будет готов для вас. Если уже было включено группами наряду с Скайп для бизнеса, используйте эти перед обновлением действия для контроля для проверки готовности организации перед обновлением до группам пользователей.

> Загрузить [Комплект успеха](https://aka.ms/UpgradeSuccessKit) для шаблона пользователя проверки готовности к источников коммуникаций и опросов пользователей, в дополнение к пример плана проекта обновления и пилотного проекта план тестирования. ## <a name="upgrade"></a>Upgrade 

**Официальная переход к группам**. При обновлении пользователей вы переместите их в режим только группы. Становится их основного приложения для разговора, собраний, вызова и совместной работы групп и доступ к Скайп для бизнес-приложения будет отключен. Несмотря на то, что технические аспекты этого этапа все довольно просто, необходимо Учитывайте влияние изменения могут иметь на взаимодействие с пользователем и разрешить пользователям официально переноса их действий из Скайп для бизнеса группам время. Для уменьшения пользователей с другой каждый раз с помощью различных клиентов, попробуйте ограничить обновления окно начала до конца 45 дней или так.
