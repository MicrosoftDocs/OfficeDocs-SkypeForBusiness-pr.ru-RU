Перед началом работы убедитесь, что у вас есть следующие необходимые условия:

- Blue Yonder WFM версии 2020.3, 2021.1 или 2021.2.

    > [!NOTE]
    > Если у вас есть blue Yonder WFM 2020.3 или 2021.1, применим исправление 2020.3.0.4 или 2021.1.0.3. Это исправление исправит проблему, из-за которой пользователи получают сохраняемое сообщение об ошибке в shifts. Кроме того, исправлена проблема, которая не позволяет пользователям обновлять свою доступность в сменах.

- Имя, пароль и URL-адреса службЫ WFM службы Blue Yonder:

    - URL-адрес федераированной проверки подлинности
    - URL-адрес проверки подлинности cookie
    - URL-адрес самообслуживаемого сотрудника
    - URL-адрес API Розничного веб-сайта
    - URL-адрес API диспетчера сайтов
    - URL-адрес API администрирования

    Если у вас нет этой информации, обратитесь в службу поддержки "Синий Yonder". Учетная запись создается на корневом уровне предприятия администратором предприятия "Синий Yonder". Она должна иметь доступ к API, администратору клиента и диспетчеру магазинов. Для создания подключения необходимы учетная запись и пароль.
- Федерационная проверка подлинности SSO включена в среде WFM "Синий Yonder". Обратитесь в службу поддержки "Синий Yonder", чтобы убедиться в том, что федератерский SSO включен. Им потребуется следующая информация:

    - federatedSSOValidationService: `https://wfmconnector.teams.microsoft.com/api/v1/fedauth/{tenantId}/6A51B888-FF44-4FEA-82E1-839401E9CD74/authorize` где {tenantId} — это ваш tenantId
     - proxyHeader: X-MS-AuthToken

- Хотя бы одна команда настроена в Teams.
- Вы добавили свою учетную запись Microsoft 365 в качестве владельца команды во все команды, которые вы хотите составить.</br> [Создайте эту учетную запись в Microsoft 365](/microsoft-365/admin/add-users/add-users) и назначьте Microsoft 365 лицензию. Затем добавьте учетную запись в качестве владельца команды во все команды, которые вы хотите составить. Соединителя "Смены" использует эту учетную запись при синхронизации изменений shifts из WFM "Синий Уодер".

    Мы рекомендуем создавать учетные записи специально для этого, а не использовать учетную запись пользователя.
