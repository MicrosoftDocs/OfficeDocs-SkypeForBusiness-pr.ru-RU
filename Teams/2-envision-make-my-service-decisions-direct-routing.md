# <a name="make-my-service-decisions"></a>Принятие моих решений на обслуживание

Для планирования технической реализации прямой маршрутизации на телефонную систему ("прямая маршрутизация") вы должны заранее решить ряд решений, чтобы лучше подготовить свою организацию к реализации решения, удовлетворяющего требованиям, которые вы определили для бизнеса.

## <a name="calling-in-teams"></a>Звонки в Teams

С помощью Microsoft Teams ваши пользователи могут звонить и принимать телефонные звонки в коммутируемой телефонной сети (КТСОП). Пользователи могут использовать собственные специальные телефонные номера для помещения и приема внутренних и международных звонков (включая голосовую почту) в клиентском приложении Teams.

## <a name="direct-routing"></a>Прямая маршрутизация

Чтобы пользователи Teams могли размещать и принимать звонки PSTN, они должны быть включены для телефонной системы — функции в Office 365.

Чтобы обеспечить возможность подключения к КТСОП, вы можете использовать прямую маршрутизацию для предоставления сотрудникам Организации возможности совершать и принимать телефонные звонки за пределами Организации по протоколу PSTN.

Благодаря прямой маршрутизации подключение по протоколу PSTN обеспечивается сторонним поставщиком услуг, что дает возможность продолжать использовать существующие магистральные КОММУТИРУЕМые линии, предоставляемые поставщиком служб PSTN. Это позволяет развертывать в странах, в которых телефонная система с планами звонков (планы звонков) недоступна, а также в тех случаях, когда необходимо поддерживать или взаимодействовать с определенными локальными системами контракта поставщика услуг PSTN. Обязательно.

## <a name="voicemail"></a>Голосовая почта

Облачная Голосовая почта, основанная на службах голосовой почты Azure, поддерживает депозиты голосовой почты только для почтовых ящиков Exchange и не поддерживает сторонние системы электронной почты.

Если в вашей организации разрешено ведение транскрипции голосовой почты, необходимо также учитывать, нужно ли включать маску ненормативной лексики. Дополнительные сведения приведены [в разделе Настройка политик голосовой почты в Организации](set-up-phone-system-voicemail.md) .

Дополнительные сведения о голосовой почте в реализации телефонной системы можно найти в разделе [Настройка голосовой почты](set-up-phone-system-voicemail.md)в облаке.

## <a name="licensing-for-direct-routing"></a>Лицензирование для прямой маршрутизации

Если ваша организация планирует использовать прямую переадресацию, вам нужно получить необходимые лицензии. Пользователям прямой маршрутизации должны быть назначены следующие лицензии в Office 365:

-   Телефонная система Майкрософт

-   Microsoft Teams

-   Аудиоконференции

Лицензия на голосовую конференцию требуется для добавления внешних участников к запланированным собраниям либо путем исходящих звонков, либо путем предоставления номера для телефонного подключения. Когда внешний участник набирает номер, Служба голосовой связи помещает звонок, используя возможности онлайн-звонков. Лицензия на голосовую конференцию необязательна и нужна только для тех пользователей, которые будут представлять Конференции групп, включающие голосовую конференцию.


> [!NOTE]
> Чтобы предоставить бесплатные телефонные номера для моста конференц-связи с телефонным подключением для звонков на международные номера, вы должны настроить [кредиты на связь](what-are-communications-credits.md) в Организации.

Голосовой Конференц-связь и телефонная система могут быть лицензированы отдельно в качестве служб надстроек для существующих пользователей, которые имеют планы подписки на Office 365 E3 или E1; они уже включены как часть плана подписки на Office 365.

> [!TIP]
> Кроме того, вы можете использовать прямую маршрутизацию для пользователей, которые работают с тарифными планами при маршрутизации звонков в сторонние АТС. Дополнительные сведения можно найти в разделе [Лицензирование и другие требования прямой маршрутизации](direct-routing-plan.md#licensing-and-other-requirements).

## <a name="office-365-considerations"></a>Вопросы о Office 365

Любой пользователь, которому будет разрешена прямая маршрутизация, должен быть расположен в Office 365. Для гибридных развертываний с локальным сервером Skype для бизнеса Server или Lync Server необходимо перенести пользователей в Skype для бизнеса Online, прежде чем настраивать их для использования прямой маршрутизации с помощью Teams.

Все пользователи, которые находятся в области реализации прямых маршрутов, должны быть включены для Teams.

Клиент Office 365 должен быть включен в один или несколько доменов, так как домен Default \*. onmicrosoft.com нельзя использовать с прямой маршрутизацией. Дополнительные сведения о доменах и клиентах Office 365 можно найти в статье [вопросы и ответы](https://support.office.com/article/Domains-FAQ-1272bad0-4bd4-4796-8005-67d6fb3afc5a)по доменам.

## <a name="sbc-considerations"></a>Особенности SBC

Необходимо использовать сертифицированные и поддерживаемые контроллеры границ сеансов (SBCs), которые должны быть парными с прямой маршрутизацией, чтобы обеспечить подключение к сети PSTN для пользователей. Список сертифицированных SBCs-данных можно найти в разделе [Поддерживаемые контроллеры границ для сеансов](direct-routing-plan.md#supported-session-border-controllers-sbcs).

В зависимости от среды, количества местоположений и требований к маршрутизации голосовой связи может потребоваться развертывание нескольких однобайтовых данных для поддержки базы пользователей.

### <a name="sbc-domain-names"></a>Доменные имена SBC

У каждого SBC-элемента, который вы хотите связать с прямой маршрутизацией, должно быть уникальное имя DNS. DNS-имена SBC должны находиться в одном из доменных имен, зарегистрированных в клиенте Office 365. Если вашему клиенту назначено несколько доменных имен, вы можете использовать любые из этих доменных имен при планировании DNS-имен SBCs.

> [!IMPORTANT]
> Использование "*. onmicrosoft.com" для DNS-имени в SBC не разрешено.

### <a name="certificates"></a>Сертификаты

Для каждого SBC, развернутого с прямой маршрутизацией, требуется сертификат, полученный в списке поддерживаемых центров сертификации. Сертификат должен содержать имя DNS для SBC в поле Тема, альтернативное имя темы или поля общего имени.

> [!NOTE]
> Также поддерживается использование сертификатов с подстановочными знаками с SBCs.

Дополнительные сведения и список поддерживаемых центров сертификации можно найти в разделе общедоступный [доверенный сертификат для SBC](direct-routing-plan.md#public-trusted-certificate-for-the-sbc).


### <a name="sbc-ip-addresses-and-ports"></a>IP-адреса и порты SBC

Каждый из SBC должен быть настроен с помощью общедоступного IP-адреса, доступного в центрах обработки данных Office 365. Вы также можете настроить трансляцию сетевых адресов (NAT) для публикации SBCs в центрах обработки данных Office 365.

SBCs требуется двунаправленное соединение для связи с облачными службами для сигнализации и мультимедиа. Передача сигналов обрабатывается компонентом, именуемым *прокси-сервером SIP*, а носители обрабатываются с помощью *процессоров мультимедиа*.

Необходимо определить конкретные номера портов для каждого SBC для сигналов SIP и мультимедиа, а также настроить брандмауэры так, чтобы они разработали двунаправленный трафик для этих портов и соответствующих им IP-адресов.

Дополнительные сведения можно найти в разделе [сигнализация SIP: полные доменные имена](direct-routing-plan.md#sip-signaling-fqdns) и [трафик мультимедиа: диапазоны портов](direct-routing-plan.md#media-traffic-port-ranges).


> [!NOTE]
> Мы настоятельно рекомендуем задать максимальное количество одновременных сеансов для каждого SBC, основываясь на модели SBC. Этот предел затем может инициировать уведомление, когда SBC выполняется на уровне или рядом с его емкостью.

## <a name="voice-routing"></a>Маршрутизация голоса

Необходимо настроить телефонную систему Microsoft, чтобы перенаправлять звонки на конкретный SBCs для прямой маршрутизации. Вы можете настроить голосовые маршруты на основе:

-   Вызываемый шаблон номера.

-   Название "шаблон номера" + пользователь, который выполняет звонок.


Маршрутизация звонков состоит из следующих элементов:

-   Политика маршрутизации голосовой связи — контейнер использования КТСОП; может назначаться пользователю или нескольким пользователям

-   Использование PSTN — контейнер для голосовых маршрутов и использование PSTN; могут совместно использоваться в разных политиках голосовой маршрутизации

-   Маршруты голосовой связи — шаблон номера и набор шлюзов в Интернет-шлюзах для звонков, где номер звонка соответствует указанному шаблону

-   Сетевой шлюз PSTN — указатель на SBC также хранит конфигурацию, которая применяется при помещении вызова через SBC, например переадресация с подлинным идентификатором P-утвержденных (паи) или предпочтительных кодеков. можно добавлять в Голосовые маршруты

Вы можете настроить маршруты голосовой связи с прямой маршрутизацией для совместного использования с планами звонков. Эта конфигурация позволяет тарифным планам перенаправлять некоторые вызовы в определенную SBCs с помощью прямой маршрутизации.

> [!TIP]
> SBCs можно назначить *активными* и *Архивировать*. Это означает, что если объект SBC, настроенный как активный для этого шаблона номера (или шаблон номера + определенного пользователя), недоступен, звонки будут перенаправляться в одноэлементный объект SBC.

## <a name="calling-and-interop-policies"></a>Политики вызовов и взаимодействия

Прямая маршрутизация поддерживается только в Microsoft Teams. Для того чтобы звонить или принимать звонки PSTN через прямую маршрутизацию, необходимо настроить необходимые политики для обеспечения приема входящих звонков в Teams.

> [!NOTE]
> Пользователи, которым разрешена прямая маршрутизация, не могут размещать или принимать прямые маршрутные звонки с помощью Skype для бизнеса и, следовательно, должны быть развернуты в клиенте Teams.

Вы можете настроить пользователей так, чтобы они выстроили группы в качестве предпочтительного клиента для звонков одним из указанных ниже способов.

-   Настройка пользователя для режима "только для Teams"

-   Настройте группы в качестве предпочтительного клиента, назначив Теамскаллингполици и Теамсинтеропполици.

Дополнительные сведения можно найти [в разделе Настройка Microsoft Teams в качестве предпочтительного телефонного клиента для пользователей](direct-routing-configure.md#set-microsoft-teams-as-the-preferred-calling-client-for-users).
