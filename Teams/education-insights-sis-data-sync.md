# <a name="sync-student-information-system-sis-data-with-education-insights"></a>Синхронизация сведений системы данных об учащихся (SIS) с помощью Аналитики образования
Чем больше данных вводится в [Аналитику образования](class-insights.md), тем лучше преподаватели могут поддерживать своих учеников, а руководители образовательных учреждений поддерживать преподавателей.

Чтобы обеспечить аналитику на уровне организации, нужно использовать [Синхронизацию сведений о школе (SDS)](/SchoolDataSync) для подключения к системе данных учащихся (SIS), чтобы Аналитика правильно сопоставила иерархическую структуру системы образования. 

Для просмотра аналитических данных на уровне класса в качестве преподавателя эта синхронизация *не требуется*, так как используется структура классов и разрешений Teams.

## <a name="plan-your-school-data-sync-integration"></a>Планирование интеграции Синхронизации сведений о школе
Синхронизации сведений о школе от Майкрософт (SDS) предоставляет сведения системы данных об учащихся (SIS), а также иерархическую структуру образовательной системы и сопоставляет назначения пользователей, а также предоставляет дополнительные данные об учащихся и организационной иерархии.

Insights лучше всего работает при использовании [формата файла SDS V2](/schooldatasync/sds-v2-csv-file-format) и более поздней версии, но также поддерживает [формат файлов SDS V1](/schooldatasync/school-data-sync-format-csv-files-for-sds) *с ограниченной функциональностью*.


### <a name="differences-between-sds-v1-and-v2-file-formats"></a>Различия между форматами файлов SDS V1 и V2

Для оптимального использования аналитики рекомендуется использовать формат файлов v2 или v2.1 (ожидается в ближайшее время)

| Тип данных | V1 | V2 |
|:--- |:--- |:--- |
| **Пользователи** | Формат V1 содержит **только преподавателей**, поэтому, чтобы установить разрешения на уровне организации для руководителей образовательных учреждений, необходимо определить разрешения для каждого из них вручную. | Формат V2 содержит **все роли**, что позволяет назначать разрешения на основе ролей. |
| **Организации** | Формат V1 содержит **только школы**, поэтому отображается только один уровень агрегирования (все ваши школы). Можно увеличить масштаб до конкретной школы, используя плоский список, но в этом списке может быть большое количество школ или разные типы школ, которые трудно сравнивать (например, начальная школа со средней или школа естествознания с художественной).<br/><br/> Если иерархия уже существует, можно создавать обоснованные уровни, такие как отдел науки или искусства.| Формат V2 содержит **полную иерархию вашего округа или учреждения**, включая университеты, колледжи, факультеты, кампусы, области, программы и т. д.<br/><br/> С помощью иерархии можно видеть соответствующую агрегацию по каждому уровню иерархии, быстро сравнивать единицы организации на каждом уровне, назначать разрешения для определенных уровней, устанавливать цели на уровне организации и т. д.|

> [!NOTE]
> С 15 июля 2021 г. клиенты не смогут подключать формат файла v2. Им потребуется использовать формат v2.1. Все будущие обновления и новые возможности будут реализованы в формате v2.1 и будут полностью обратно совместимы с форматом файла v1.

## <a name="best-practices"></a>Рекомендации
Точное сопоставление иерархии и места каждого в этой иерархии позволяет Аналитике предоставить точные данные и более точную и релевантную аналитику для различных типов руководителей образовательных учреждений.

Чем больше сведений вы предоставите, тем лучше и точнее будут отчеты и сведения.

Ниже приведены рекомендации по обеспечению плавного развертывания SDS, чтобы пользователи могли максимально эффективно использовать Insights.

### <a name="file-format-version-to-ue"></a>Версия формата файла для использования
*   Используйте формат файла V2.1 (ожидается в ближайшее время) и синхронизируйте необязательные данных, применяемые службой Insights для образования.

### <a name="users-and-roles"></a>Пользователи и роли
*   Убедитесь, что **все пользователи указаны в** предоставленных и синхронизированных файлах. Это касается всех учащихся и сотрудников, которые должны видеть данные для своих организационных подразделений.
    Если в настоящее время в SIS указаны только преподаватели, добавьте других пользователей вручную перед загрузкой файлов в SDS и синхронизацией данных.
    Служба Insights собирает статистику только о зарегистрированных учащихся. Если некоторые учащиеся отсутствуют, данные и выводы будут неверными.
    
*   Не забудьте **указать имя и фамилию каждого пользователя**. В противном случае на них будут ссылаться по адресу электронной почты, что менее удобно в отчетах и карточках важного (карточки с аналитикой действий или производительности учащихся).

*   Класс или год обучения должен быть основан на этом [списке сопоставления](#supported-grade-level-values). 

*   Важно **добавить класс или год обучения для всех учащихся**, чтобы по нему можно было агрегировать и фильтровать данные о действиях.    

*   Не забудьте **назначить каждому пользователю соответствующее подразделение**. Благодаря этому Insights для образования не будет показывать неправильные данные в карточках важного.

    *   Учащийся может быть связан с несколькими подразделениями, например учащиеся, зарегистрированные в специальной программе или на двух факультетах. Если у учащегося есть несколько подразделений, предоставьте строку для каждого из них в файле пользователей для этого учащегося.
    
    *   ИТ-администратор может предоставлять разрешения на основе подразделения для сотрудников. **Убедитесь, что сотрудники связаны с правильным уровнем подразделения**, чтобы они получали необходимые разрешения. Например, консультант, назначенный для четырех учебных заведений, должен видеть все классы в этих учебных заведениях. Директору необходимо видеть все классы в своем учебном заведении. 
    
*   **Эта роль очень важна**. Хотя этот список закрыт, попробуйте сопоставить роль из [списка](/schooldatasync/sds-v2-csv-file-format#enumerated-values-enum-supported) с реальной ролью каждого добавленного пользователя. Это позволит назначать соответствующие разрешения на основе ролей. Например, предоставьте разрешения директоров, чтобы они могли видеть их учебные заведения, или для всех профессоров, чтобы они видели свои факультеты. 

### <a name="organizations"></a>Организации

* Убедитесь, что **отображается реальная и полная иерархия организации**. Это можно сделать добавив файл вручную. В ряде случаев иерархия не отражается в SIS. Тем не менее, может понадобиться увидеть соответствующее обобщение по каждому уровню иерархии, назначить разрешения для определенных уровней, установить цели по уровням организации и т. д. 

* Убедитесь, что **все подразделения дерева организации включают учащихся и классы**, чтобы обобщить данные учащихся для них. Рекомендуется, чтобы учащиеся были на наинизшей ветви дерева.

> [!NOTE]
> Дополнительные сведения о развертывании SDS см. в статье [Планирование SDS](/schooldatasync/planning-school-data-sync).

## <a name="integrate-sis-data-using-sds"></a>Интеграция данных SIS с помощью SDS

Синхронизация сведений о школе (SDS) предоставляется вместе с Office 365 для образовательных учреждений. SDS считывает данные из системы данных об учащихся (SIS) образовательного учреждения и встраивает их в приложения Майкрософт, например Teams, чтобы обеспечить автоматическое создание онлайн-классов и пользователей.

При этом данные SIS синхронизируются с Insights.

В качестве ИТ-администратора вы можете выбрать следующие варианты: использовать SDS только для подготовки, только Insights или оба решения.

Чтобы синхронизировать данные SIS с Insights для образования, следуйте инструкциям в статье [Как развернуть SDS для Insights](/schooldatasync/how-to-deploy-sds-for-insights).

### <a name="deploy-sds"></a>Развертывание SDS
**Если SDS уже используется**, рекомендуется следовать [лучшим практикам](#best-practices). 

**Если вы еще не используете SDS**, необходимо [развернуть ее](/schooldatasync/deploying-school-data-sync).

В процессе развертывания можно решить, нужно ли использовать SDS для подготовки пользователей и классов в Teams или только для предоставления пользователя и иерархии организации в Insights.

> [!NOTE]
> Если сейчас середина года и вы уже создали команды вручную, используйте SDS только для предоставления пользователя и данных иерархии организации в Insights, а в следующем году рассмотрите возможность использования SDS для подготовки пользователей и классов в Teams.

### <a name="verify-the-sync-process"></a>Проверка процесса синхронизации
Чтобы проверить ход выполнения синхронизации, следуйте инструкциям из статьи [SDS для состояния данных Insights и отслеживания](/schooldatasync/sds-for-insights-data-health-and-monitoring).

> [!IMPORTANT]
> Если у вас возникли проблемы, [служба поддержки клиентов](https://aka.ms/edusupport) готова помочь.
