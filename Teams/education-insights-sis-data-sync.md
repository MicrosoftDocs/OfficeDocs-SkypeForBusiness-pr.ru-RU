# <a name="sync-student-information-system-sis-data-with-education-insights"></a>Синхронизация сведений системы данных об учащихся (SIS) с помощью Аналитики образования
Чем больше данных вводится в [Аналитику образования](class-insights.md), тем лучше преподаватели могут поддерживать своих учеников, а руководители образовательных учреждений поддерживать преподавателей.

Чтобы обеспечить аналитику на уровне организации, нужно использовать [Синхронизацию сведений о школе (SDS)](/SchoolDataSync) для подключения к системе данных учащихся (SIS), чтобы Аналитика правильно сопоставила иерархическую структуру системы образования. 

Для просмотра аналитических данных на уровне класса в качестве преподавателя эта синхронизация *не требуется*, так как используется структура классов и разрешений Teams.

## <a name="plan-your-sis-integration"></a>Планирование интеграции SIS
Данные SIS предоставляют иерархическую структуру системы образования и показывают куда назначен пользователь.

Аналитика лучше всего работает при использовании [формата файла SDS V2](/schooldatasync/sds-v2-csv-file-format), но также поддерживает [формат файлов SDS V1](/schooldatasync/school-data-sync-format-csv-files-for-sds) с *ограниченной* функциональностью.

### <a name="differences-between-sds-v1-and-v2-file-formats"></a>Различия между форматами файлов SDS V1 и V2

| Тип данных |   V1 | V2 (рекомендуется для новых пользователей) |
|:--- |:--- |:--- |
| **Пользователи** | Формат V1 содержит **только преподавателей**, поэтому, чтобы установить разрешения на уровне организации для руководителей образовательных учреждений, необходимо найти их и определить разрешения для каждого вручную. | Формат V2 содержит **все роли**, что позволяет назначать разрешения на основе ролей. |
| **Организации** | Формат V1 содержит **только школы**, поэтому отображается только один уровень агрегирования (все ваши школы). Можно увеличить масштаб до конкретной школы, используя плоский список, но в этом списке может быть большое количество школ или разные типы школ, которые трудно сравнивать (например, начальная школа со средней или школа естествознания с художественной).<br/><br/> Если иерархия уже существует, можно создавать обоснованные уровни, такие как отдел науки или искусства.| Формат V2 содержит **полную иерархию вашего округа или учреждения**, включая университеты, колледжи, факультеты, кампусы, области, программы и т. д.<br/><br/> С помощью иерархии можно видеть соответствующую агрегацию по каждому уровню иерархии, быстро сравнивать единицы организации на каждом уровне, назначать разрешения для определенных уровней, устанавливать цели на уровне организации и т. д.|

### <a name="type-of-data-required"></a>Обязательный тип данных
В таблице ниже указаны типы данных, необходимые для эффективного использования Аналитики.

| Тип данных | Примеры того, что нужно указать|Почему это важно?|
|:--- |:--- |:--- |
| **Пользователи** |   Роль (например, ученик)<br/> [Класс или год обучения](#supported-grade-level-values) (например, 10)<br/> Организация (имя) | При правильном назначении роли каждого человека, класса или года обучения, а также организации можно обеспечить правильность сводов и агрегаций.|
| **Организации** | Тип организации (например, колледж) |   Эта иерархия важна. Например, учебные заведения могут относиться к своему округу, и этот округ может принадлежать штату.<br/> Если руководителю образовательных учреждений округа разрешено просматривать данные, это касается только школ этого округа.|
| **Классы** | Название (например, Информатика 101) | В этой таблице приведены сведения о занятиях, которые проводятся в организации. В этой таблице должны использоваться правильные соответствия, чтобы можно было назначить учащегося нужному классу. |
| **Регистрация** | Роль (например, ученик) | Эта таблица предназначена для учащихся и преподавателей и позволяет определить, в каком классе они зарегистрированы. |

> [!NOTE]
> В процессе развертывания можно решить, нужно ли использовать SDS для подготовки пользователей и классов в Teams или только для предоставления данных для Аналитики.

## <a name="best-practices"></a>Рекомендации
Точное сопоставление иерархии и места каждого в этой иерархии позволяет Аналитике предоставить точные данные и более точную и релевантную аналитику для различных типов руководителей образовательных учреждений.

Чем больше сведений вы предоставляете, тем лучше и точнее будут отчеты и сведения.
Ниже приведены рекомендации по обеспечению плавного развертывания SDS, чтобы пользователи могли максимально эффективно использовать Аналитику.

### <a name="users"></a>Пользователи
*   Убедитесь, что *все пользователи* указаны в предоставленных и синхронизированных файлах. Это касается всех учащихся и сотрудников, которые должны видеть данные для своих организационных подразделений.

    Если в настоящее время в SIS указаны только преподаватели, добавьте других пользователей вручную перед загрузкой файлов в SIS и синхронизацией данных.

    Если некоторые учащиеся отсутствуют, результаты Аналитики собираются только у зарегистрированных студентов, что приводит к неверным данным и выводам.
    
*   Не забудьте *указать имя и фамилию каждого пользователя*. В противном случае на них будут ссылаться по адресу электронной почты, что приводит к менее положительному результату в отчетах и полезных сведениях (карточки с аналитикой действий или производительности учащихся).

*   *Класс или год обучения должен быть введен как 2 цифры* (например, 07 для 7 класса). Ознакомьтесь со [списком сопоставлений](#supported-grade-level-values). 

*   Важно *добавить класс или год обучения для всех учащихся*, чтобы они могли фильтровать данные.    

*   Не забудьте *назначить каждому пользователю соответствующее подразделение*. Таким образом, в полезных сведениях не будут отображаться неверные данные на основе обобщенных данных для каждого подразделения.

    *   Студент может быть связан с несколькими подразделениями, например учащиеся, зарегистрированные в специальной программе или на двух факультетах. В таком случае предоставьте две строки в файле пользователей для каждого учащегося — по одному для каждой организации.
    
    *   С учетом подразделения для сотрудников можно определять соответствующие разрешения. Убедитесь, что они связаны с правильным уровнем подразделения, чтобы они получали необходимые разрешения. Например, консультант, назначенный на четыре школы, должен увидеть все классы в этих школах. Директору необходимо видеть все классы в своей школе. 
    
*   Эта роль очень важна. Хотя этот список закрыт, попробуйте сопоставить роль из [списка](/schooldatasync/sds-v2-csv-file-format#enumerated-values-enum-supported), с реальной ролью каждого добавленного пользователя. Так можно назначать разрешения на уровне роли соответствующим образом. Например, предоставьте разрешения директоров, чтобы они могли видеть их учебные заведения, или для всех профессоров, чтобы они видели свои факультеты. 

### <a name="organizations"></a>Организации

* Убедитесь, что *отображается реальная иерархия организации*. Это можно сделать добавив файл вручную. В ряде случаев иерархия не отражается в SIS. Тем не менее, может быть необходимо увидеть соответствующую агрегацию по каждому уровню иерархии, назначить разрешения для определенных уровней, установить цели по уровням организации и т. д. 

* Убедитесь, что *все подразделения дерева организации включают учащихся и классы*, чтобы собрать данные о студентах для них. Рекомендуется, чтобы учащиеся были на наинизшей ветви дерева.

> [!NOTE]
> Дополнительные сведения о развертывании SDS см. в статье [Планирование SDS](/schooldatasync/planning-school-data-sync).

## <a name="integrate-sis-using-sds"></a>Интеграция SIS с помощью SDS

Синхронизация сведений о школе (SDS) предоставляется вместе с Office 365 для образовательных учреждений. SDS считывает данные из системы данных об учащихся (SIS) образовательного учреждения и встраивает их в Teams, чтобы обеспечить автоматическое создание онлайн-классов и пользователей.

При этом данные SIS будут синхронизироваться с Аналитикой.

### <a name="sync-with-insights"></a>Синхронизация с Аналитикой

Сначала необходимо включить функцию "Аналитика", чтобы начать синхронизацию.

* На [**портале SDS**](https://sds.microsoft.com) перейдите в **Параметры**, прокрутите страницу вниз до параметра **Сбор данных для Аналитики** и убедитесь в том, что он включен (он *включен* по умолчанию).

* Прокрутите вниз до следующего переключателя **Синхронизация данных организации из SDS (предварительная версия)** и включите.

Если параметр *Синхронизация данных организации из SDS (предварительная версия)* не отображается на странице настроек, перейдите на [страницу регистрации](https://aka.ms/insights/join), чтобы предоставить свою информацию, и член команды свяжется с вами.

:::image type="content" source="media/insights-sds-settings.png" alt-text="Синхронизация с переключателями аналитики":::

### <a name="deploy-sds"></a>Развертывание SDS
**Если SDS уже используется**, рекомендуется следовать [лучшим практикам](#best-practices). 

Чтобы синхронизировать текущие профили с Insights, перейдите в **Профили синхронизации**, нажмите **Изменить** и выберите **Синхронизация с Insights**. При первоначальной синхронизации рекомендуется подождать 24 часа, чтобы отчеты стали доступны после обновления данных в SIS.  

:::image type="content" source="media/insights-sds-profile-sync.png" alt-text="Переключатели синхронизации профиля с Insights":::

**Если вы еще не используете SDS**, необходимо [развернуть ее](/schooldatasync/deploying-school-data-sync).

В процессе развертывания можно решить, нужно ли использовать SDS для подготовки пользователей и классов в Teams или только для предоставления данных для Аналитики.

> [!NOTE]
> Если сейчас середина года и вы уже создали группы вручную, используйте SDS только для того, чтобы предоставить Аналитике данные, а в следующем году используйте SDS для подготовки пользователей и классов в Teams.

### <a name="verify-the-sync-process"></a>Проверка процесса синхронизации
Рядом с полем "Синхронизация данных организации из SDS (предварительная версия)" на странице "Параметры" отобразится новая область состояния.
 
*   Если состояние — **В процессе**, подождите до 24 часов после развертывания профиля SDS.

*   Если состояние – **Выполнено**, можно просмотреть аналитику на уровне организации и перейти к следующему этапу.

*   Если состояние – **Выполнено с ошибками**, **Выполнено с предупреждениями** или **Прервано**, скачайте файл журнала, в котором содержатся ошибки и предупреждения для последней синхронизации, и проверьте, можно ли их устранить. 

> [!IMPORTANT]
> Если у вас возникли проблемы, [служба поддержки клиентов](https://aka.ms/edusupport) готова помочь.
