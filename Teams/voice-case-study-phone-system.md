# <a name="contoso-case-study-phone-system-for-a-multi-national-corporation"></a>Пример: телефонная система корпорации Contoso

В зависимости от географического расположения и других факторов в contoso были офисы, использующие следующие решения для телефонии:

- Тип сайта A: Skype для бизнеса Корпоративная голосовая связь

- Тип сайта B. Традиционные устаревшие системы телефонии

- Тип сайта C: сочетание Skype для бизнеса Корпоративная голосовая связь и традиционных устаревших систем телефонии


Чтобы внедрить решение Телефон (Майкрософт) для всей организации, contoso должен был определить для каждого типа сайта, какие из следующих параметров будут использоваться с телефонная система для подключения к ОКП. &mdash; &mdash;

- телефонная система с планом звонков 

- телефонная система с собственным оператором STN через прямую маршрутику 

- Сочетание телефонная система с планом звонков и телефонная система с собственным оператором ЗВОНКОВ через прямую маршрутику
 
Чтобы определить правильное решение для своей организации, Contoso использовал решения для телефонии [Майкрософт](/SkypeForBusiness/hybrid/msft-telephony-solutions) и сеанс Ignite 2019 [Calling в Microsoft Teams.](https://myignite.techcommunity.microsoft.com/sessions/83170?source=sessions)  

## <a name="site-type-a-skype-for-business-enterprise-voice"></a>Тип сайта A: Skype для бизнеса Корпоративная голосовая связь 

Contoso Skype для бизнеса Корпоративная голосовая связь была настроена как центр и ора. В регионе поддерживался шлюз ННР, который обеспечивает подключение к STN для Skype для бизнеса Корпоративная голосовая связь пользователей в стране. Часто эти спутниковые офисы не имеют собственного доступа к Интернету. Номера этих пользователей находились на соединительных связях SIP с существующим SBC. 

Чтобы определить, сертифицирован ли SBC для прямого обхода маршрутов и мультимедиа, Contoso проверил список контроллеров границ сеанса, сертифицированный для прямой [маршрутирования.](direct-routing-border-controllers.md)  

При наборе номера пользователю нужно было набрать номер в устаревшей телефонной системе с помощью расширения, даже если у пользователя есть клиент Skype для бизнеса для одноранговой связи. 

Решение Contoso основано на следующих вопросах:

- Q. Нужно ли сохранять функции, предоставляемые в локальном развертывании?<br>
  A. Нет 

- Q. Нужно ли во время связи со сторонними системами УАКС и другим телефонным оборудованием?<br>
  A. Нет 

- Q. Нужно ли сохранить текущий сторонний оператор?<br> A. Да (регулируемые страны) и Нет 

- Q. Нужно ли развертывать рентабельность инвестиций на SBCs?<br> A. Да и нет  

- Q. Доступны ли планы звонков по ЗВОНКОВ майкрософт в этом регионе?<br> A. Да и нет 

На основе ответов на их вопросы компания Contoso решила:

- Перемещение пользователей, расположенных в регионе, где планы звонков по ЗВОНКОВ доступны для телефонная система планов звонков. 

- Перемещение пользователей, не расположенных в регионе, где доступны планы звонков по ЗВОНКОВ, пользователей на сайте, где рентабельность инвестиций на БСК еще не выполнены, а пользователей, проживающих в стране, в которой есть правила телефонии, телефонная система прямой маршрутизов. 

На следующей схеме показано Skype для бизнеса Корпоративная голосовая связь развертывания и его переноса в планы звонков Майкрософт и прямую маршрутику.

## <a name="site-type-b-traditional-legacy-telephony-systems"></a>Тип сайта B. Традиционные устаревшие системы телефонии

В Contoso было много офисов, в которые использовались устаревшие системы телефонии. Некоторые пользователи имели номер телефона E1.64, тогда как у других были только расширения. Эти номера находились на магистрали TDM для шлюза STN. Настройка набора внутри сайта с помощью кода сайта перед расширением для определения направления звонка. Пользователям нужно было набирать номера по расширению.   

Решение Contoso основано на следующих вопросах:

- Q. Нужно ли сохранять функции, предоставляемые в локальном развертывании?<br>
  A. Нет 

- Q. Нужно ли во время связи со сторонними системами УАКС и другим телефонным оборудованием?<br> A. Да

- Q. Нужно ли сохранить текущий сторонний оператор?<br> A. Нет 

- Q. Доступен ли в нашем регионе план звонков по ННР Майкрософт?<br> A. Да и нет 

На основе ответов на их вопросы компания Contoso решила: 

- Перемещение пользователей, расположенных в регионе, где планы звонков по ЗВОНКОВ доступны для телефонная система планов звонков. 

- Перемещение пользователей, которые не находятся в регионе, где планы звонков по ЗВОНКОВ телефонная система с прямой маршрутикой. 

- Поддерживать подключение через STN к критически важным аналоговым устройствам.

## <a name="site-type-c-combination-of-skype-for-business-enterprise-voice-and-traditional-legacy-telephony-systems"></a>Тип сайта C: сочетание Skype для бизнеса Корпоративная голосовая связь и традиционных устаревших телефонных систем

Contoso Skype для бизнеса Корпоративная голосовая связь номера пользователей находятся на телефоне SIP в SBC от оператора связи. Номера традиционных систем телефонии находились на tDM-канале до шлюза STN.   

Решение Contoso основано на следующих вопросах:

- Q. Нужно ли сохранять функции, предоставляемые в локальном развертывании?<br>
  A. Нет 

- Q. Нужно ли во время связи со сторонними системами УАКС и другим телефонным оборудованием?<br> A. Нет 

- Q. Нужно ли сохранить текущий сторонний оператор?<br> A. Нет 

- Q. Нужно ли развертывать рентабельность инвестиций на SBCs?<br> A. Да и нет  

- Q. Доступен ли план звонков по ЗВОНКОВ по ЗВОНКОВ майкрософт в этом регионе?<br> A. Нет 

На основе ответов на их вопросы Компания Contoso решила следующее: 

- Для пользователей устаревших телефоний, которые будут использовать прямую маршрутику, Contoso перенаправил номера из TDM-линии на линию SIP для SBC, так как SBC сертифицирован для прямой маршрутии. 

- Чтобы обеспечить поддержку подмножество пользователей, переехав на телефонная система и разрешив продолжить маршрутику по устаревшей системе, устаревшая телефонная система была настроена в качестве следующего перехода к SBC.   

- Кроме того, чтобы способствовать изменению поведения пользователей и удалить зависимость от набора расширений между сайтами и внутри сайта, Contoso предоставляет рекомендации по использованию Teams для всех внутренних звонков.

## <a name="configuration"></a>Конфигурация

### <a name="calling-plans-sites"></a>Сайты планов звонков

Чтобы получить лицензии и назначить номера телефонов пользователям, Компания Contoso, как было поручено в области Настройка [планов звонков,](set-up-calling-plans.md)придерживается действий. 

Из-за количества пользователей, которые должны быть назначены телефонные номера, Contoso решил использовать PowerShell для назначения номеров телефонов. Чтобы узнать, как назначать числа с помощью PowerShell в дополнение к другим настройкам, Contoso использует Teams &mdash; &mdash; [PowerShell Overview](teams-powershell-overview.md).  

### <a name="direct-routing-sites"></a>Сайты прямой маршрутии

Чтобы подключить к Microsoft Teams инфраструктуру локальной телефонной связи Contoso, администратор Contoso, [](direct-routing-configure.md) вы следовать указаниям в окне Настройка прямой маршрутации, а также просмотрел видео Direct [Routing](https://www.youtube.com/watch?v=1ASftX_Msb8&index=10&list=PLaSOUojkSiGnKuE30ckcjnDVkMNqDv0Vl) в Microsoft Teams.  Contoso также ссылается на документацию сертифицированного поставщика SBC о прямой маршрутистике. 

После настройки прямой маршрутиации между SBC и Телефон (Майкрософт) System необходимо было проверить конфигурацию contoso. Для этого администраторы Contoso использовали клиент SIP Tester, который обсуждается в сеансе Обновления для прямой маршрутии на [ignite 2019.](https://myignite.techcommunity.microsoft.com/sessions/83178?source=sessions) Сценарий клиента тестировщика SIP и документация были загружены из сценария PowerShell для тестирования подключений к контроллеру границы прямого сеанса маршрутирования.   


### <a name="local-media-optimization"></a>Оптимизация локальных мультимедиа

Contoso видит возможность использовать оптимизацию локальных мультимедиа в разных регионах по всему миру. Поддерживаемые сценарии для Contoso описаны в local [Media Optimization для прямой маршрутизации.](direct-routing-media-optimization.md) Настройка оптимизации локальных мультимедиа была завершена с помощью инструкций поставщика SBC и Корпорации Майкрософт. Действия по настройке для оптимизации локальных мультимедиа: 

- Настройка сайтов пользователя и SBC 

- Настройте SBC в соответствии с спецификацией поставщика SBC, 

- Добавление внешних доверенных IP-адресов на каждый сайт, используемый для оптимизации локальных мультимедиа    

- Определение топологии сети 

- Определение топологии виртуальной сети 

- Определение режима: Всегда обходить или Только для локальных пользователей 

## <a name="networking-considerations"></a>Сетевые аспекты

В Contoso было несколько пользователей, которым требовалось работать удаленно в течение длительного периода времени после включения телефонная система. Пользователи использовали VPN для доступа к определенным бизнес-приложениям. В VPN качество телефонная система ухудшалось. 

Чтобы устранить проблему с качеством, компания Contoso реализовала раздельное тунтенинг VPN, что позволило Office 365 трафику через Интернет, а подключение к внутренним приложениям оставалось на VPN. Чтобы внедрить раздельный vpn- tunneling, Contoso следовать указаниям в реализации [vpn split tunneling для](/office365/enterprise/office-365-vpn-implement-split-tunnel)Office 365.
