# <a name="contoso-case-study-phone-system"></a>Исследование успеха Contoso: телефонная система

В зависимости от географического местоположения и других факторов компания Contoso содержала офисы, использующие следующие решения для телефонной связи:

- Тип сайта а: голосовой связи Skype для бизнеса (корпоративный)

- Тип сайта B: традиционные старые системы телефонии

- Тип сайта C: сочетание голосовой связи в Skype для бизнеса для предприятий и традиционных стандартных телефонных систем


Для реализации решения Microsoft Phone System для всей Организации компания Contoso приходила определить &mdash; для каждого из типов сайтов, &mdash; какие из указанных ниже вариантов будут использоваться в телефонной сети для подключения к общественной коммутируемой телефонной линии.

- Телефонная система с планом звонков 

- Телефонная система с собственной несущей для КОММУТИРУЕМой связи через прямую маршрутизацию 

- Сочетание телефонной системы с планом звонков и телефонной системой с помощью прямой маршрутизации
 
Чтобы определить подходящего решения для своей организации, Contoso использует [решения Microsoft для телефонной связи](https://docs.microsoft.com/SkypeForBusiness/hybrid/msft-telephony-solutions) и сеанс Ignite 2019 [в Microsoft Teams](https://myignite.techcommunity.microsoft.com/sessions/83170?source=sessions).  

## <a name="site-type-a-skype-for-business-enterprise-voice"></a>Тип сайта а: голосовой связи Skype для бизнеса (корпоративный) 

Голосовая связь Contoso Skype для бизнеса Enterprise была настроена в качестве основного и резервного. В регионе, который обеспечивает подключение к сети PSTN для пользователей голосовой связи Skype для бизнеса в стране, есть центральное место, в котором поддерживается шлюз PSTN. Часто в этих вспомогательных офисах нет собственного выхода в Интернет. Номера для этих пользователей находились на магистральной магистрали SIP, соединяющей с существующим SBC. 

Чтобы определить, является ли уже развернутый SBC, сертифицированный для прямой маршрутизации и пропуска мультимедиа, компания Contoso проверила [список контроллеров границ сеанса, сертифицированных для прямого маршрутизации](direct-routing-border-controllers.md).  

Пользователь должен набрать пользователя в устаревшей системе телефонной связи с помощью расширения, даже если у пользователя есть клиент Skype для бизнеса, доступный для однорангового звука. 

В соответствии со своими решениями для Contoso на следующие вопросы:

- Стоит. Нужно ли сохранять функциональные возможности, предоставляемые нашим локальным развертыванием?<br>
  Помощью. Нет 

- Стоит. Требуется ли взаимодействие со сторонними системами УАТС и другим оборудованием телефонной связи?<br>
  Помощью. Нет 

- Стоит. Нужно ли хранить текущую стороннюю несущую версию?<br> A. Да (регулируемые страны) и без 

- Стоит. Требуется ли получить коэффициент окупаемости инвестиций для развернутого SBCs?<br> A. Да и нет  

- Стоит. Доступны ли в этом регионе планы звонков по Microsoft PSTN?<br> A. Да и нет 

В соответствии с ответами на вопросы компания Contoso решила:

- Перемещайте пользователей, которые находятся в регионе, где планы звонков по КОММУТИРУЕМой телефонной линии доступны для телефонных систем с тарифными планами. 

- Перемещайте пользователей, которых нет в регионе, где доступны планы по КОММУТИРУЕМой телефонной сети, пользователи, находящиеся на сайте, где коэффициент окупаемости не соблюдается, и пользователи, которые находились в стране, в которой есть правила для телефонной системы с прямой маршрутизацией. 

На приведенной ниже схеме показана первоначальная настройка голосовой связи в Skype для бизнеса Enterprise и способ переноса этого развертывания на планы звонков и прямую маршрутизацию.

## <a name="site-type-b-traditional-legacy-telephony-systems"></a>Тип сайта B: традиционные старые системы телефонии

В компании Contoso уже много офисов, использующих старые системы телефонной связи. У меня есть подмножество пользователей с 1.64ным номером телефона, а у других — только расширение. Эти номера находились на магистральной магистрали TDM с шлюзом PSTN. Внутренний набор номера находился с помощью кода сайта перед расширением, чтобы определить, куда перенаправляться звонок. Пользователи должны набрать номер, чтобы звонить по расширению.   

В соответствии со своими решениями для Contoso на следующие вопросы:

- Стоит. Нужно ли сохранять функциональные возможности, предоставляемые нашим локальным развертыванием?<br>
  Помощью. Нет 

- Стоит. Требуется ли взаимодействие со сторонними системами УАТС и другим оборудованием телефонной связи?<br> О. Да

- Стоит. Нужно ли хранить текущую стороннюю несущую версию?<br> А. 

- Стоит. Доступно ли в нашем регионе план звонков по Microsoft PSTN?<br> A. Да и нет 

В соответствии с ответами на вопросы компания Contoso решила: 

- Перемещайте пользователей, которые находятся в регионе, где планы звонков по КОММУТИРУЕМой телефонной линии доступны для телефонных систем с тарифными планами. 

- Переместить пользователей, которые не находятся в регионе, где планы звонков по КОММУТИРУЕМой телефонной линии доступны для прямой маршрутизации. 

- Поддерживать PSTN-связь с важнейшими аналоговыми устройствами для бизнеса.

На приведенных ниже схемах показано первоначальное развертывание старой системы с удаленными сайтами и миграция на прямую подписку с помощью локальной оптимизации файлов мультимедиа.

## <a name="site-type-c-combination-of-skype-for-business-enterprise-voice-and-traditional-legacy-telephony-systems"></a>Тип сайта C: сочетание голосовой связи в Skype для бизнеса (корпоративный) и традиционных стандартных телефонных систем

Номера пользователей Skype для бизнеса для предприятий голосовой связи (Contoso) размещаются на магистралье SIP на SBC из несущей частоты. Номера традиционных телефонных систем, находились на магистральной магистрали TDM на шлюзе PSTN.   

В соответствии со своими решениями для Contoso на следующие вопросы:

- Стоит. Нужно ли сохранять функциональные возможности, предоставляемые нашим локальным развертыванием?<br>
  Помощью. Нет 

- Стоит. Требуется ли взаимодействие со сторонними системами УАТС и другим оборудованием телефонной связи?<br> А. 

- Стоит. Нужно ли хранить текущую стороннюю несущую версию?<br> А. 

- Стоит. Требуется ли получить коэффициент окупаемости инвестиций для развернутого SBCs?<br> A. Да и нет  

- Стоит. Доступно ли в этом регионе тарифный план по КОММУТИРУЕМой телефонной связи Майкрософт?<br> А. 

В соответствии с ответами на вопросы компания Contoso решила следующее: 

- Для устаревших пользователей телефонной связи, которые будут включены для прямой маршрутизации, компания Contoso переносит номера из магистральной TDM на магистраль SIP для SBC, так как SBC сертифицирован для прямой маршрутизации. 

- Для поддержки подмножества пользователей, переходящих в телефонную систему и возможной непрерывной маршрутизации в устаревшей системе, устаревшая система телефонии была настроена как следующий прыжок для SBC.   

- Кроме того, для включения режима поведения пользователей и удаления зависимости для набора номера между сайтами, предоставленного компанией Contoso, для использования Teams во всех внутренних звонках.  

На приведенных ниже схемах показана исходная версия Skype для бизнеса Enterprise и устаревшая система телефонной связи, а также миграция в смешанное развертывание с помощью Direct Routing.

## <a name="configuration"></a>Конфигурация

### <a name="calling-plans-sites"></a>Сайты планов звонков

Чтобы получить лицензии и назначить номера телефонов пользователям, компания Contoso продемонстрирует действия, описанные в разделе [Настройка планов звонков](set-up-calling-plans.md). 

Из-за количества пользователей, которым необходимо назначить номера телефонов, компания Contoso решила использовать PowerShell для назначения номеров телефонов. Чтобы научиться назначать номера с помощью PowerShell, &mdash; а также других параметров &mdash; компания Contoso использовала [Общие сведения о Teams PowerShell](teams-powershell-overview.md).  

### <a name="direct-routing-sites"></a>Прямые сайты маршрутизации

Чтобы подключить локальную инфраструктуру телефонной связи Contoso к Microsoft Teams, администратор Contoso проверил инструкции по [настройке Direct Routing](direct-routing-configure.md) и проверки маршрутизации видеорассылки [в Microsoft Teams](https://www.youtube.com/watch?v=1ASftX_Msb8&index=10&list=PLaSOUojkSiGnKuE30ckcjnDVkMNqDv0Vl) .  Компания Contoso также ссылалась на документацию по развертыванию прямой маршрутизации, предоставленную поставщиком SBC. 

После того, как настроена прямая маршрутизация между SBC и телефонной системой Microsoft, для Contoso требуется проверка конфигурации. Для этого администраторы Contoso использовали клиент тестеров SIP, который был описан в разделе [обновления для прямого маршрутизации сеансов на Ignite 2019](https://myignite.techcommunity.microsoft.com/sessions/83178?source=sessions). Клиентский сценарий и документация тестера SIP скачаны из сценария PowerShell, чтобы протестировать соединения с контроллером границ для однонаправленных сеансов маршрутизации.   


### <a name="local-media-optimization"></a>Оптимизация локальных файлов мультимедиа

Компания Contoso высмотрела возможность использовать локальную оптимизацию мультимедиа в разных регионах земного шара. Поддерживаемые сценарии для Contoso описаны в разделе [Оптимизация локальных файлов мультимедиа для прямой маршрутизации](direct-routing-media-optimization.md). Настройка оптимизации локальных файлов мультимедиа была выполнена с помощью руководства, полученного от поставщика SBC и от корпорации Майкрософт. Ниже приведены этапы настройки для локальной оптимизации мультимедиа. 

- Настройка пользователей и сайтов SBC 

- Настройка SBC в соответствии с спецификацией поставщика SBC 

- Добавление внешних надежных IP-адресов к каждому сайту, используемому для оптимизации локального мультимедиа    

- Определение топологии сети 

- Определение топологии виртуальной сети 

- Определение режима: всегда обходить или только для локальных пользователей 

## <a name="networking-considerations"></a>Вопросы работы в сети

В компании Contoso существовало несколько пользователей, которым требовалось работать в течение длительного периода времени после включения телефонной системы. Пользователи, использующие VPN для доступа к определенным строкам бизнес-приложений. В сети VPN пользователи телефонной системы попытались снизить качество связи. 

Для устранения проблемы с качеством компания Contoso реализовала разделение VPN, которая разрешила трафик Office 365 для прохождения через Интернет, пока подключение к внутренним приложениям осталось в сети VPN. Для реализации разделения с разделением VPN компания Contoso применяет руководство по [реализации разделения VPN-туннелей для Office 365](https://docs.microsoft.com/office365/enterprise/office-365-vpn-implement-split-tunnel).
