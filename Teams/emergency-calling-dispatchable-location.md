# <a name="emergency-addresses-for-remote-locations"></a>Адреса для экстренного реагирования для удаленных расположений

В этой статье описывается поддержка корпорацией Майкрософт сведений о расположении экстренных вызовов 911 в США. Эта поддержка гарантирует, что для пользователей Teams, выполняющих экстренные вызовы, предоставляются наиболее точные сведения о расположении для диспетчеризации. Независимо от расположения вызывающего объекта ( на месте или работы из дома) — сведения о расположении вызывающего объекта, отправляемые в общедоступную точку ответа на безопасность (PSAP), должны быть точными.

В этой статье содержатся сведения о соответствии корпорации Майкрософт требованиям к Акту RAY BAUM для многострочного телефонного обеспечения (MLTS). Закон RAY BAUM расширяет требования закона Кари, которые вступают в силу в начале 2021 года. Дополнительные сведения о законе RAY BAUM и Законе Кари см. в разделе "Расположение диспетчеризации для [911](https://www.fcc.gov/911-dispatchable-location) звонков и многостроковых телефонных систем" — Требования к прямому набору номеров, уведомлениям и расположению, указанным в Законе Кари и законе [RAY BAUM 911](https://www.fcc.gov/mlts-911-requirements). 

Теперь пользователи, работающие дома, могут задать собственные адреса для экстренного реагирования, если это применимо. В этой статье описывается, как настроить политики пользователей, чтобы конечные пользователи могли задавать адреса для экстренного реагирования.

Хотя эта информация относится к экстренным вызовам 911 в США, введенные пользователем расположения также будут переданы в центр проверки в Канаде.

## <a name="support-for-emergency-calling-location-information"></a>Поддержка сведений о расположении экстренных вызовов

Для поддержки этих требований Teams использует службы расположения, предоставляемые соответствующей операционной системой, чтобы предложить адрес ( если администратор или пользователь предоставил разрешение). Конечный пользователь может подтвердить расположение предложенного адреса, изменить его или вручную ввести новый адрес. Затем подтвержденный, измененный или введенный вручную адрес сохраняется в клиенте Teams, чтобы при подключении клиента к этой сети автоматически использовался подтвержденный пользователем адрес. Сохраненные пользователем адреса автоматически очищаются при выходе клиента Teams.


## <a name="location-precedence"></a>Приоритет расположения

Адреса для экстренного реагирования для Teams могут быть классифицируются по разным типам. В следующем списке показан приоритет расположения, используемый при наборе номера для экстренного реагирования:

1. Динамически полученный адрес, определенный администратором клиента в службе сведений о расположении.

2. Адрес, подтвержденный, измененный или введенный пользователем вручную, связанный с локальной сетью, к которой подключен клиент Teams.

3. Адрес, автоматически предлагаемый операционной системой.

4. Адрес, который администратор статически назначает пользователю.


## <a name="emergency-address-classification-and-routing"></a>Классификация и маршрутизация экстренных адресов

В следующей таблице показаны типы адресов для экстренного реагирования и связанные методы маршрутизации для каждого типа: выполняется ли автоматический перенаправление вызова в обслуживающий PSAP или проверка точности перед передачей в обслуживающий PSAP. Такое поведение маршрутизации поддерживается в США для всех пользователей плана звонков, партнеров Operator Connect и сертифицированных поставщиков услуг прямой маршрутизации для экстренных вызовов.


| Тип адреса для экстренного реагирования | Метод экстренной маршрутизации |
| :------------| :-------|
| Динамически полученный адрес для экстренного реагирования, определенный администратором. | Прямо в PSAP. |
| Адрес для экстренного реагирования, полученный из операционной системы без подтверждения **точности** пользователем. | Экран и перенос в PSAP. |
| Адрес для экстренного реагирования, полученный из операционной системы с **подтверждением** точности пользователем.| Прямо в PSAP. |
| Адрес для экстренного реагирования, полученный из операционной системы, измененный и подтвержденный пользователем. | Экран и перенос в PSAP. |
| Адрес для экстренного реагирования, введенный и подтвержденный пользователем. | Экран и перенос в PSAP. |
| Адрес для экстренного реагирования статически назначается пользователю или номеру. | Экран и перенос в PSAP. |
| Null | Экран и перенос в PSAP. |


## <a name="enable-end-users-to-configure-their-emergency-address"></a>Разрешить конечным пользователям настраивать адреса для экстренного реагирования

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. В левой области навигации Центра администрирования Microsoft Teams перейдите к политикам **голосовой** > **связи для экстренного реагирования**.
2. Нажмите **Добавить**.
3. Введите имя политики экстренных вызовов, например E911WFH.
4. **Включите режим поиска внешних расположений**.
5. Нажмите **Применить**.

### <a name="using-powershell"></a>С помощью PowerShell

Чтобы включить эту функцию для конечных пользователей, используйте командлет New-CsTeamsEmergencyCallingPolicy PowerShell и задайте для параметра ExternalLocationLookupMode значение Enabled. См. следующий пример: 


``` PowerShell
New-CsTeamsEmergencyCallingPolicy -Identity E911WFH -ExternalLocationLookupMode Enabled
```

```PowerShell
Grant-CsTeamsEmergencyCallingPolicy -PolicyName E911WFH -Identity user@contoso.com
```

После включения этой функции для конечных пользователей на вкладке "Вызовы" пользователь может добавить, изменить или подтвердить адрес для экстренного реагирования и отобразить адрес после его настройки. Дополнительные сведения о том, как конечные пользователи могут устанавливать службы расположения, см. в разделе "Работа из дома для экстренного реагирования [911: включение служб определения местоположения"](https://support.microsoft.com/office/work-from-home-emergency-911-enable-location-services-583dd649-87fc-4b23-aed6-f4e2279297f9?storagetype=live).

В Windows вы можете управлять службой расположения Windows, а также доступом приложений к расположению с помощью групповой политики или управления мобильными [устройствами (MDM).](/windows/client-management/mdm/policy-csp-privacy#privacy-letappsaccesslocation)

Дополнительные сведения о службе расположения Windows см. в статье о [службе расположения и конфиденциальности Windows](https://support.microsoft.com/windows/windows-location-service-and-privacy-3a8eee0a-5b0b-dc07-eede-2a5ca1c49088).



## <a name="notes-and-restrictions"></a>Примечания и ограничения

Учитывайте следующее.

- Описанная работа на домашних компьютерах предназначена для настольных компьютеров Teams в Windows и Mac.

- Телефоны Teams не поддерживают работу из дома.

- Teams Mobile поддерживает автоматическое обнаружение расположения, но не описанное пользователем взаимодействие.

- Параметры конфиденциальности могут конфликтовать с автоматическим обнаружением расположения. Управление устройствами мобильные системы.
