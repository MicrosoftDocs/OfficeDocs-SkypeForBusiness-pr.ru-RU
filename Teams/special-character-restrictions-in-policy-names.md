# <a name="what-are-the-special-character-restrictions-in-teams-policies"></a>Какие существуют ограничения специального символа в политиках групп?

**Хотя специальных символов можно использовать для создания политики групп с помощью PowerShell, который будет ограничен при управлении эти политики.  Таким образом мы настоятельно рекомендуем имена политики не включать специальные символы.**

Имена политики, созданные с помощью PowerShell для собраний и чата в группах может иметь специальные символы, таких как @, #, $. Тем не менее если которым требуется для изменения политики в Microsoft групп и Скайп по центру администрирования бизнеса, не будут иметь возможность. Необходимо использовать Windows PowerShell и командлет правильный политики для внесения изменений.

Если у вас есть объект политики со специальными знаками и удалить специальных символов, чтобы лучше управлять политики в Microsoft групп и Скайп по центру администрирования бизнес, вам потребуется использовать ниже процедуры. 

Примечание: В процедуре определена здесь используется пример политики системы обмена сообщениями.  Тот же самый процесс используется для другого типа политики (собрания для примера) subsituting соответствующих командлетов. 

1.) вызов Get-CSMessagingPolicy-identity < old_policy_name > и запись данных выходные данные политики.
2.) вызова Set-CSMessagingPolicy-identity < new_policy_name >, чтобы создать новую политику с такой же конфигурацией, что исходной политики, но без специальные символы в имени.
3.) вызова Delete-CSMessagingPolicy-identity < old_policy_name > Удалить политику.  Если эта команда выполнена успешно, вы все готово и можно приступить к назначение пользователей в новую политику, используя либо PowerShell, или группами Майкрософт и Скайп по центру администрирования бизнес.
4.) Если выше команда завершится неудачно, он является, поскольку старая политика назначена пользователю.  Запуск отменить командлет, чтобы отменить политику от пользователя, назначение новой политики, а затем снова запустите dwlete.
