# <a name="conduct-an-ediscovery-investigation-of-content-in-microsoft-teams"></a>Обнаружение электронных данных в Microsoft Teams

Крупные предприятия часто становятся в судебных разбирательствах по запросу отправки всех хранимых в электронном виде данных (ESI). Microsoft Teams контент можно искать и использовать во время исследований eDiscovery.

## <a name="overview"></a>Обзор

Все Microsoft Teams чаты (1:1) и групповые чаты проходят журнал в почтовые ящики соответствующих пользователей. Все стандартные сообщения канала проходят журнал в почтовый ящик группы, представляющий группу. Файлы, добавленные в стандартные каналы, охватываются функциями eDiscovery для SharePoint Online и OneDrive для бизнеса.

EDiscovery сообщений и файлов в закрытых [каналах](private-channels.md) работает не так, как в стандартных каналах. Дополнительные узнать об этом можно в этой [теме.](#ediscovery-of-private-channels)

## <a name="ediscovery-of-private-channels"></a>EDiscovery частных каналов

Записи для сообщений, отправленных по частному каналу, доставляются в почтовый ящик всех участников частного канала, а не в групповой почтовый ящик. Названия записей форматируются, чтобы указать, с какого частного канала они были отправлены.

Так как каждый частный канал имеет собственный SharePoint сайт, который не является родительским сайтом группы, управление файлами в частном канале не является родительским.

Teams не поддерживает поиск по обнаружению электронных данным по одному каналу в группе, поэтому поиск необходимо для всей группы. Чтобы выполнить поиск контента с помощью eDiscovery в частном канале, выполните поиск по всей команде, по семейому веб-сайту, связанному с частным каналом (чтобы включить файлы), а также по почтовым ящикам участников частного канала (чтобы включить сообщения).

Чтобы идентифицировать файлы и сообщения в частном канале, чтобы включить их в поиск при обнаружении электронных данных, с помощью следующих действий:

### <a name="include-private-channel-files-in-an-ediscovery-search"></a>Включить файлы частного канала в поиск eDiscovery

Перед выполнением этих действий установите SharePoint Online Management Shell и [подключите его к SharePoint Online.](/powershell/sharepoint/sharepoint-online/connect-sharepoint-online)

1. Чтобы получить список всех веб-SharePoint, связанных с частными каналами в группе, запустите следующую команду:

    ```PowerShell
    Get-SPOSite
    ```

2. Запустите следующий сценарий PowerShell, чтобы получить список URL-адресов всех SharePoint, связанных с частными каналами в группе, и их ИД родительской группы.

    ```PowerShell
    $sites = get-sposite -template "teamchannel#0"
    foreach ($site in $sites) {$x= get-sposite -identity $site.url -detail; $x.relatedgroupID; $x.url}
    ```

3. Для каждой группы или ИД группы запустите следующий сценарий PowerShell, чтобы определить все релевантные сайты частных каналов, где $groupID — это ИД группы.

    ```PowerShell
    $sites = get-sposite -template "teamchannel#0"
    $groupID = "e8195240-4a70-4830-9106-80193cf717cb"
    foreach ($site in $sites) {$x= Get-SpoSite -Identity $site.url -Detail; if ($x.RelatedGroupId -eq $groupID) {$x.RelatedGroupId;$x.url}}
    ```

### <a name="include-private-channel-messages-in-an-ediscovery-search"></a>Включить сообщения частного канала в поиск eDiscovery

Прежде чем выполнять эти действия, убедитесь, что у вас установлена последняя версия Teams [PowerShell.](teams-powershell-overview.md)

1. Чтобы получить список частных каналов в команде, запустите следующую команду:

    ```PowerShell
    Get-TeamChannel -GroupId <GroupID> -MembershipType Private
    ```

2. Чтобы получить список участников частного канала, запустите следующую команду:

    ```PowerShell
    Get-TeamChannelUser -GroupId <GroupID> -DisplayName "Engineering" -Role Member
    ```

3. Включите в поисковый запрос [eDiscovery](/microsoft-365/compliance/search-for-content-in-core-ediscovery)почтовые ящики всех участников с каждого частного канала в команду.

## <a name="search-for-content-for-guest-users"></a>Поиск контента для гостевых пользователей

Средства eDiscovery можно использовать для поиска Teams, связанных с гостевых пользователей в организации. Teams содержимого чата, связанного с гостевым пользователем, сохраняется в облачном хранилище, и его можно искать с помощью eDiscovery. Это включает поиск контента в чатах 1:1 и 1:N, в которых гостевая пользователь является участником с другими пользователями в вашей организации. Вы также можете искать сообщения частного канала, в которых гостевом пользователем является участник, и искать содержимое в беседах гостевого чата, в которых единственными участниками являются гостевых пользователей. 

Чтобы найти контент для гостевых пользователей:

1. Подключение в Azure AD PowerShell. Инструкции см. в разделе "Подключение с Azure Active Directory PowerShell" в Подключение для Microsoft 365 [с помощью PowerShell.](/microsoft-365/enterprise/connect-to-microsoft-365-powershell#connect-with-the-azure-active-directory-powershell-for-graph-module) Не забудьте выполнить этапы 1 и 2 в предыдущей теме.

2. После успешного подключения к Azure AD PowerShell запустите следующую команду, чтобы отобразить имя директора-пользователя (UPN) для всех гостевых пользователей в организации. При создании поиска на шаге 4 необходимо использовать имя-имя гостя.

   ```powershell
   Get-AzureADUser -Filter "userType eq 'Guest'" -All $true | FL UserPrincipalName
   ```

   > [!TIP]
   > Вместо отображения списка имен участников-пользователей на экране компьютера можно перенаправить результат команды в текстовый файл. Для этого можно использовать предыдущую `> filename.txt` команду. Текстовый файл с именами основных пользователей будет сохранен в текущей папке.

3. В другом окне Windows PowerShell к PowerShell центра безопасности & соответствия требованиям. Инструкции см. в [Подключение PowerShell](/powershell/exchange/connect-to-scc-powershell)центра & безопасности. Вы можете подключаться с помощью многофакторной проверки подлинности или без нее.

4. Создайте поиск контента, в котором выполняется поиск всего содержимого (например, сообщений чата и сообщений электронной почты), в котором указанный гость был участником:

   ```powershell
   New-ComplianceSearch <search name> -ExchangeLocation <guest user UPN>  -AllowNotFoundExchangeLocationsEnabled $true -IncludeUserAppContent $true
   ```

   Например, чтобы найти содержимое, связанное с гостевых пользователей Е.Ермолаевых, нужно выполнить следующую команду:

   ```powershell
   New-ComplianceSearch "Sara Davis Guest User" -ExchangeLocation "sara.davis_hotmail.com#EXT#@contoso.onmicrosoft.com" -AllowNotFoundExchangeLocationsEnabled $true -IncludeUserAppContent $true
   ```

    Дополнительные сведения об использовании PowerShell для создания поиска контента см. в [new-ComplianceSearch.](/powershell/module/exchange/new-compliancesearch)

5. Чтобы начать поиск контента, созданного в шаге 4, запустите следующую команду:

   ```powershell
   Start-ComplianceSearch <search name>
   ```

6. Перейдите к [https://compliance.microsoft.com](https://compliance.microsoft.com) кнопке **Показать весь поиск**  >  **контента.**

7. В списке поисковых запросов выберите поиск, созданный на шаге 4, чтобы отобразить вылетную страницу.

8. На странице вы можете сделать следующее:

   - Нажмите **кнопку Просмотреть** результаты, чтобы просмотреть результаты поиска и просмотреть содержимое.

   - Рядом с **полем Query (Запрос)** **щелкните Edit (Изменить),** чтобы изменить, а затем повторно войдите в поиск. Например, вы можете добавить поисковый запрос, чтобы сузить результаты.

   - Нажмите **кнопку Экспортировать** результаты, чтобы экспортировать и скачать результаты поиска.

## <a name="search-for-card-content"></a>Поиск содержимого карт

Содержимое карточных карт, созданных приложениями в Teams каналах, 1:1 чатах и чатах 1xN, сохраняется в почтовых ящиках и может быть искомо. *Карточка* — это контейнер пользовательского интерфейса для коротких фрагментов содержимого. Карточки могут иметь несколько свойств и вложений, а также кнопки, которые могут запускать действия с карточками. Дополнительные сведения см. в [карточках](/microsoftteams/platform/task-modules-and-cards/what-are-cards)

Как и Teams, содержимое карточки хранится в зависимости от места использования карточки. Содержимое карточек, используемых Teams канале, хранится в почтовом Teams группы. Содержимое карток чатов 1:1 и 1xN сохраняется в почтовых ящиках участников чата.

Для поиска содержимого карточки можно использовать условия или `kind:microsoftteams` `itemclass:IPM.SkypeTeams.Message` условия поиска. При проверке результатов поиска содержимое карточки, сгенерированное ботами в канале Teams, имеет свойство **Отправитель/Автор** сообщения электронной почты как , где имя приложения, сгенер которого было сгенерировано `<appname>@teams.microsoft.com` содержимое `appname` карточки. Если содержимое карточки было сгенерировано пользователем, его определяет значение **Отправитель или** Автор.

При просмотре содержимого карточки в результатах поиска контента оно отображается как вложение в сообщение. Вложение называется , где находится имя приложения, сгенер которого `appname.html` `appname` было создано содержимое карточки.
