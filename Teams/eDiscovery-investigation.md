# <a name="conduct-an-ediscovery-investigation-of-content-in-microsoft-teams"></a>Обнаружение электронных данных в Microsoft Teams

Крупные предприятия часто становятся открыты для юридической процедуры, в связи с чем требуется отправка всех хранимых в электронном виде данных (ESI). Контент Microsoft Teams можно искать и использовать во время исследования eDiscovery.

## <a name="overview"></a>Обзор

Все чаты Microsoft Teams 1:1 или группы проходят журнал по почтовым ящикам соответствующих пользователей. Все стандартные сообщения канала проходят журнал в почтовый ящик группы, представляющий группу. Файлы, добавленные в стандартные каналы, охватываются функциями eDiscovery для SharePoint Online и OneDrive для бизнеса.

EDiscovery сообщений и [](private-channels.md) файлов в закрытых каналах работает не так, как в стандартных каналах. Дополнительные узнать см. [в eDiscovery частных каналов.](#ediscovery-of-private-channels)

## <a name="ediscovery-of-private-channels"></a>EDiscovery частных каналов

Записи для сообщений, отправленных по частному каналу, доставляются в почтовый ящик всех участников частного канала, а не в групповой почтовый ящик. Названия записей форматируются, чтобы указать, с какого частного канала они были отправлены.

Так как у каждого частного канала есть собственный сайт SharePoint, который отдельно от родительского сайта группы, управление файлами в частном канале не является родительским.

Teams не поддерживает поиск eDiscovery в одном канале в команде, поэтому необходимо искать вся команда. Чтобы выполнить поиск контента с помощью eDiscovery в частном канале, выполните поиск в группе, в семейсе веб-сайтов, связанном с закрытым каналом (для включаемого файлов), а также в почтовых ящиках участников частного канала (включая сообщения).

Чтобы определить файлы и сообщения в частном канале и включить их в поиск при обнаружении электронных данных, с помощью следующих действий:

### <a name="include-private-channel-files-in-an-ediscovery-search"></a>Включить файлы частных каналов в поиск eDiscovery

Перед выполнением этих действий установите управляющую оболочку [SharePoint Online и подключитесь к SharePoint Online.](https://docs.microsoft.com/powershell/sharepoint/sharepoint-online/connect-sharepoint-online)

1. Чтобы получить список всех коллекций веб-сайтов SharePoint, связанных с частными каналами в группе, запустите следующую команду:

    ```PowerShell
    Get-SPOSite
    ```

2. Запустите следующий сценарий PowerShell, чтобы получить список URL-адресов всех сайтов sharePoint, связанных с частными каналами в группе, и их ИД родительской группы.

    ```PowerShell
    $sites = get-sposite -template "teamchannel#0"
    foreach ($site in $sites) {$x= get-sposite -identity $site.url -detail; $x.relatedgroupID; $x.url}
    ```

3. Для каждой группы или группы запустите следующий сценарий PowerShell, чтобы определить все соответствующие сайты частных каналов, где $groupID — это ИД группы.

    ```PowerShell
    $sites = get-sposite -template "teamchannel#0"
    $groupID = "e8195240-4a70-4830-9106-80193cf717cb"
    foreach ($site in $sites) {$x= Get-SpoSite -Identity $site.url -Detail; if ($x.RelatedGroupId -eq $groupID) {$x.RelatedGroupId;$x.url}}
    ```

### <a name="include-private-channel-messages-in-an-ediscovery-search"></a>Включить сообщения частного канала в поиск eDiscovery

Перед выполнением этих действий убедитесь, что установлена последняя версия [модуля Teams PowerShell.](teams-powershell-overview.md)

1. Чтобы получить список частных каналов в команде, запустите следующую команду:

    ```PowerShell
    Get-TeamChannel -GroupId <GroupID> -MembershipType Private
    ```

2. Чтобы получить список участников частного канала, запустите следующую команду:

    ```PowerShell
    Get-TeamChannelUser -GroupId <GroupID> -DisplayName "Engineering" -Role Member
    ```

3. Включите в запрос поиска [eDiscovery](https://docs.microsoft.com/microsoft-365/compliance/search-for-content-in-core-ediscovery)почтовые ящики всех участников каждого частного канала в команду.

## <a name="search-for-content-for-guest-users"></a>Поиск контента для гостевых пользователей

Вы можете использовать средства eDiscovery для поиска контента Teams, связанного с гостевых пользователей в вашей организации. Содержимое чата Teams, связанное с гостевым пользователем, сохраняется в облачном хранилище, и его можно искать для использования eDiscovery. Это включает поиск контента в чатах 1:1 и 1:N, в которых гостевой пользователь является участником с другими пользователями в организации. Вы также можете искать сообщения в частном канале, в которых  гость является участником, и искать содержимое в беседах гостевого чата, где единственными участниками являются гостевых пользователей.

Чтобы найти контент для гостевых пользователей:

1. Подключите Azure AD PowerShell. Инструкции см. в разделе "Подключение с помощью Azure Active Directory PowerShell" статьи "Подключение к [Microsoft 365 с помощью PowerShell".](https://docs.microsoft.com/microsoft-365/enterprise/connect-to-microsoft-365-powershell#connect-with-the-azure-active-directory-powershell-for-graph-module) Не забудьте выполнить этапы 1 и 2 в предыдущей теме.

2. После успешного подключения к Azure AD PowerShell запустите следующую команду, чтобы отобразить имя директора-пользователя для всех гостевых пользователей в организации: Имя-имя гостя необходимо использовать при создании поиска на шаге 4.

   ```powershell
   Get-AzureADUser -Filter "userType eq 'Guest'" -All $true | FL UserPrincipalName
   ```

   > [!TIP]
   > Вместо отображения списка имен участников-пользователей на экране компьютера можно перенаправить выходные данные команды в текстовый файл. Для этого можно использовать предыдущую `> filename.txt` команду. Текстовый файл с именами основных пользователей будет сохранен в текущей папке.

3. В другом окне Windows PowerShell подключились к PowerShell в Центре & безопасности и соответствия требованиям. Инструкции см. в [powerShell Центра & безопасности](https://docs.microsoft.com/powershell/exchange/connect-to-scc-powershell)и соответствия требованиям. Вы можете подключаться с помощью многофакторной проверки подлинности или без нее.

4. Создайте поиск контента, в котором выполняется поиск всего контента (например, сообщений чата и сообщений электронной почты), в котором указанный гостевой пользователь был участником:

   ```powershell
   New-ComplianceSearch <search name> -ExchangeLocation <guest user UPN>  -AllowNotFoundExchangeLocationsEnabled $true -IncludeUserAppContent $true
   ```

   Например, чтобы найти содержимое, связанное с гостевых пользователей Sara Davis, необходимо выполнить следующую команду:

   ```powershell
   New-ComplianceSearch "Sara Davis Guest User" -ExchangeLocation "sara.davis_hotmail.com#EXT#@contoso.onmicrosoft.com" -AllowNotFoundExchangeLocationsEnabled $true -IncludeUserAppContent $true
   ```

    Дополнительные сведения о создании поиска контента с помощью PowerShell см. в [new-ComplianceSearch.](https://docs.microsoft.com/powershell/module/exchange/new-compliancesearch)

5. Чтобы начать поиск контента, созданный на шаге 4, запустите следующую команду:

   ```powershell
   Start-ComplianceSearch <search name>
   ```

6. Перейдите к [https://compliance.microsoft.com](https://compliance.microsoft.com) кнопке **"Показать весь**  >  **поиск контента" и выберите ее.**

7. В списке поисковых запросов выберите поиск, созданный на шаге 4, чтобы отобразить листовку.

8. Во flyout page вы можете сделать следующее:

   - Нажмите **кнопку "Просмотреть** результаты", чтобы просмотреть результаты поиска и просмотреть содержимое.

   - Рядом с **полем "Запрос"** нажмите **кнопку** "Изменить", чтобы изменить и повторно запросить результат поиска. Например, вы можете добавить поисковый запрос, чтобы сузить результаты.

   - Нажмите **кнопку "Экспортировать** результаты", чтобы экспортировать и скачать результаты поиска.

## <a name="advanced-ediscovery"></a>Advanced eDiscovery

Некоторые материалы Microsoft Teams также можно искать и сохранять с помощью рабочего процесса [Advanced eDiscovery.](https://docs.microsoft.com/microsoft-365/compliance/overview-ediscovery-20) Так как eDiscovery предоставляет широкий диапазон функций поиска, удержания и экспорта, расширенные функции eDiscovery предоставляют администраторам соответствия требованиям дополнительные средства для определения источников данных и анализа их содержимого.

### <a name="advanced-ediscovery-custodian-workflow-for-teams-content"></a>Рабочий процесс Advanced eDiscovery для контента Teams

Хранители могут быть членами разных команд. Вы можете захватывать контент Teams, который относится к этим хранителям. Инструкции по работе хранителей см. в инструкциях по добавлению хранителей в дело [Advanced eDiscovery.](https://docs.microsoft.com/microsoft-365/compliance/add-custodians-to-case)

Добавив хранителя, нажмите **кнопку** "Далее", а затем **кнопку "Добавить".** После этого отобразит окно, в котором будет предложено выбрать дополнительные расположения, в которых будут показаны все члены хранителя и соответствующие расположения сайта SharePoint для их данных. Во всех этих источниках данных и в командах можно выбрать контент, который вы хотите использовать для eDiscovery, а затем разместить этого пользователя и все источники данных, которые вы определили, на удержание.

Вы можете выбрать, следует ли включать контент Exchange, содержимое OneDrive или и то, и другое. Содержимое Exchange содержит все содержимое приложения в почтовых ящиках пользователя, например его электронную почту, содержимое Teams, которое хранится в почтовом ящике, и другие данные. Содержимое OneDrive содержит не только содержимое пользователя, но и все содержимое Teams, которое хранится в OneDrive, например чаты (1:1), чаты 1:N и файлы, к совместному доступу в чатах.

Вы также можете связать любую команду, в которую входит хранителя, чтобы включить в нее сообщения чата и файлы, к ним имеет доступ хранителя. Кроме того, любую другую команду можно связывать с хранителями.

> [!NOTE]
> EDiscovery сообщений и [](private-channels.md) файлов в закрытых каналах работает не так, как в стандартных каналах. Дополнительные узнать см. [в eDiscovery частных каналов.](#ediscovery-of-private-channels)

### <a name="placing-a-data-source-on-hold"></a>Помещение источника данных на удержание

Если конкретный пользователь не назначен хранителями, вы можете поместите на удержание весь источник данных. Дополнительные сведения об управлении удерживами см. в дополнительных сведениях об управлении [удерживами при advanced eDiscovery.](https://docs.microsoft.com/microsoft-365/compliance/managing-holds)

При создании удержания для контента Teams вы можете выбрать все расположения, которые хотите включить в удержание. Даже если пользователи удаляют или изменяют контент, при удержании будут сохраняться копии всех предыдущих версий этого контента.

Вы также можете использовать необязательный запрос, чтобы установить условия для удержания на основе ключевых слов, диапазона дат, автора и многих других условий. Если не указать ключевые слова, все данные из этого источника данных будут на удержание.

### <a name="advanced-ediscovery-searches"></a>Расширенные поисковые запросы eDiscovery

Кроме того, можно искать контент Teams. Дополнительные сведения о поиске см. в сведениях о сборе данных по делу [в Advanced eDiscovery.](https://docs.microsoft.com/microsoft-365/compliance/collecting-data-for-ediscovery) Если поисковому запросу соответствует хотя бы одно сообщение, будет возвращена вся беседа.

При создании поискового запроса можно выбрать хранителей, чтобы поиск был по всем уже выбранным источникам. Вы также можете искать данные из других источников, таких как сайт Teams, не соединый с пользователем. Кроме того, с помощью необязательных запросов можно сузить поиск в содержимом Teams.

После создания и выбора поискового запроса в окне отображаются дополнительные сведения и действия, которые можно использовать для выбранного поиска. Нажав  кнопку "Статистика", вы можете просмотреть статистику поиска, в том числе разбивку по типам расположения, исходному источнику контента, а также по расположению контента в почтовом ящике группы, почтовом ящике отдельного пользователя или на сайте SharePoint. Таким образом, вы можете увидеть разбивку источников, которые повествуют о результатах поиска. Кроме того, доступно представление "Запросы", в котором можно узнать, какие отдельные ключевые слова повествуют о результатах поиска. 

Завершив поиск, вы можете  нажать кнопку "Добавить результаты для проверки набора" и добавить ее в набор проверки. Дополнительные сведения о наборах проверки см. далее в этой статье в статье "Управление наборами проверки в расширенных наборах [eDiscovery](https://docs.microsoft.com/microsoft-365/compliance/managing-review-sets) и [наборов](#review-sets-workflow) проверки".

#### <a name="normal-review-sets-and-conversation-review-sets"></a>Обычные наборы проверки и наборы проверки бесед

При добавлении поиска в набор отзывов можно выбрать обычный набор отзывов или набор для проверки беседы.

Набор проверки похож на экспорт; В ней представлены `.msg` отдельные файлы для контента Teams, а содержимое представлено в базовом представлении. Обычно при использовании других программных средств для повторной проверки файлов обычно используется обычный набор рецензирощений.

Набор отзывов бесед обеспечивает более интуитивно понятное и многопоточной представление бесед; соответствующие сообщения отображаются в правильном порядке.

> [!div class="mx-imgBorder"]
> ![Снимок экрана: набор проверки беседы](media/conversationOptions2.png)

Такие функции, как redaction, доступны в обоих типах наборов проверки. Дополнительные сведения о наборах проверки см. в дополнительных сведениях о проверке бесед [в advanced eDiscovery.](https://docs.microsoft.com/microsoft-365/compliance/conversation-review-sets)

#### <a name="collection-options"></a>Параметры коллекции

При добавлении в набор отзывов в разделе "Параметры коллекции" окна есть  несколько параметров, включая параметры инициировать инициировать и параметры иконки "Беседы **Teams".**  Если вы встроите эти параметры, все отдельные сообщения Teams, в составе вашего набора отзывов, будут также показаны вместе с дополнительными сообщениями, окружающими их в контексте. Например, если запрос конкретный и в результате возвращается только одно сообщение, включение этих параметров также возвратит несколько сообщений, которые являются его результатом.

Множество логических критериев используются для определения того, предоставляют ли дополнительные сообщения контекст для сообщений, которые соответствуют запросу. Например, для содержимого Teams при включите эти параметры, чтобы получить родительское и все сообщения ребенка из-за цепочки сообщений.

Также будут проверяться отметки времени сообщений. Если сообщение соответствует запросу, то соседние сообщения, предшествующие ему в течение 4 часов или следующие за ним в течение 4 часов, считаются частью беседы и также включаются в результаты.

Если вы должны быть уверены, какие контекстные сообщения будут возвращены с совпадениями с поисковым запросом, использовать эти параметры не нужно. Вы можете собрать все содержимое или расширить диапазон дат поиска, чтобы больше сообщений возвращалось в результате запроса.

### <a name="review-sets-workflow"></a>Рабочий процесс "Наборы проверки"

Вы можете просмотреть существующие наборы проверки или создать новые на вкладке **"Наборы рецензировок".** Дополнительные сведения о наборах проверки см. в [документе "Управление наборами проверки в Advanced eDiscovery".](https://docs.microsoft.com/microsoft-365/compliance/managing-review-sets)

Кроме документов, в набор отзывов можно добавлять сообщения электронной почты, сообщения Teams, сообщения Yammer и другое содержимое. В наборе проверки также можно выполнять многие из тех же операций, которые можно выполнять в других контекстах, таких как поиск контента и создание настраиваемого запроса. Эти операции относятся только к тем пунктам, которые были добавлены в набор проверки.

Кнопка **"Управление наборами** проверки" предоставляет дополнительные параметры, такие как аналитика, сводные отчеты, количество добавленных наборов нагрузки и так далее.

Чтобы получить доступ к визуализациям  и диаграммам с данными, щелкните представление профиля поиска по индивидуальным результатам \>  в правом верхнем поле. Вы можете щелкнуть на этих диаграммах нужный тип контента, чтобы выбрать его в интерактивном режиме. Например, можно выбрать запрос только для контента Teams. Эти запросы также можно сохранить точно так же, как и запросы, написание вручную.

#### <a name="summary-view-text-view-and-annotate-view"></a>Представление "Сводка", "Текст" и представление "Примечать"

Если щелкнуть беседу Teams в наборе рецензиров, отобразится представление сводки, в котором вся беседа Teams отображается как список сообщений, с которыми можно взаимодействовать по отдельности. Щелкните стрелку вниз справа от сообщения, чтобы отобразить контекстное меню, в котором можно просмотреть сведения о сообщении или скачать отдельный `.msg` файл. Щелкнув сведения о сообщении, вы отведет сводку метаданных или полные метаданные сообщения.

Чтобы скачать PDF-файл, нажмите кнопку скачивания в правом верхнем части представления сводки.

Щелкните **вкладку "Текстовый",** чтобы отобразить обычный текст извлеченного текста беседы Teams. Это содержимое в виде обычного текста подходит для экспорта, и вы можете легко работать с ним с помощью других программ.

Для доступа к **функциям представления "Примечания"** щелкните вкладку представления "Примечания". На этой вкладке содержимое отображается в формате, который напоминает беседу Teams, но есть и дополнительные параметры редактирования. Существует инструмент карандаша, который можно использовать для создания заметок, создания рисунков в сообщении или создания тонких заметок для целей redaction. Кроме **того,** существует инструмент для начертания прямоугольника, который затемнение области и пометит его как "Красный".

Вот пример файла, который был красный для цепочки бесед между пользователями.

> [!div class="mx-imgBorder"]
> ![Снимок экрана: красный файл](media/RedactedFileExample.png)

В нижней  части вкладки представления  "Примечание" находится кнопка "Пометить документы", которая отображает панель пометки. В этой области вы можете применить тег во все сообщения в беседе Teams. Беседу можно пометить как адаптивную или нечувствительную, привилегированную или неисключимую, содержаную в ней информацию "Интересные элементы", сведения о том, следует ли включить ее в экспорт и требуется ли дополнительная проверка. Вы также можете управлять настраиваемыми тегами и применять их.

#### <a name="action-menu"></a>Меню действий

В окне наборов проверки вы можете экспортировать содержимое, нажав кнопку **"Экспорт** \> **действий".** Существует множество вариантов экспорта.

Чтобы экспортировать файл, содержащий все метаданные для всех сообщений Teams, щелкните, чтобы нажать кнопку **"Загрузить** файл". Чтобы включить в файл все теги, примененные к содержимому, щелкните, чтобы выбрать его. 

Используйте параметр **"Собственные файлы"** для экспорта файлов в их родном формате. Вы можете экспортировать беседу как один файл или все отдельные сообщения чата в отдельные файлы.

Параметр **"Текстовые** файлы" позволяет сохранять версии содержимого в виде обычного текста. Дополнительные сведения о том, как получить представление обычного текста бесед Teams в наборе рецензировок, см. выше в представлении сводки, текстовом представлении и представлении [с примеча ru-RU.](#summary-view-text-view-and-annotate-view)

Если вы применили какие-либо удаления к содержимому, как описано в разделе "Сводка", "Текст" и "Примечаний" выше, вы можете выбрать параметр "Заменить старые документы преобразованными **PDF-файлами",** чтобы заменить их преобразованными копиями в ФОРМАТе PDF. [](#summary-view-text-view-and-annotate-view)

Вы можете экспортировать данные в контейнер хранилища BLOB-устройств Azure, предоставленный Майкрософт, или предоставить собственный контейнер хранилища BLOB-устройств Azure.

Когда вы будете готовы начать экспорт, нажмите кнопку **"Экспорт".** Дополнительные [сведения о том,](https://docs.microsoft.com/microsoft-365/compliance/download-export-jobs) как получить доступ к контейнеру хранилища BLOB-устройств Azure и скачать экспортируемую информацию после экспорта, см. в этой ссылке.

> [!NOTE]
> Экспорт может занять много времени. Чтобы отслеживать состояние экспорта, зайдите  на вкладку "Наборы проверки" и перейдите **на вкладку "Экспорт".**
