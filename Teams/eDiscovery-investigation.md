# <a name="conduct-an-ediscovery-investigation-of-content-in-microsoft-teams"></a>Обнаружение электронных данных в Microsoft Teams

Крупные предприятия часто становятся в судебных разбирательствах по запросу отправки всех хранимых в электронном виде данных (ESI). Microsoft Teams контент можно искать и использовать во время исследования eDiscovery.

## <a name="overview"></a>Обзор

Все Microsoft Teams чаты (1:1) и групповые чаты проходят журнал по почтовым ящикам соответствующих пользователей. Все стандартные сообщения канала пересылаются в почтовый ящик группы, представляющий группу. Файлы, добавленные в стандартные каналы, охватываются функциями eDiscovery для SharePoint Online и OneDrive для бизнеса.

EDiscovery сообщений и файлов в закрытых каналах работает не так, как в стандартных каналах.[](private-channels.md) Дополнительные узнать об этом можно в этой [теме.](#ediscovery-of-private-channels)

## <a name="ediscovery-of-private-channels"></a>EDiscovery частных каналов

Копии сообщений, отправленных в частном канале, доставляются в почтовый ящик всех участников частного канала, а не в почтовый ящик группы. Названия копий соответствия требованиям отформатированы так, чтобы указать, из каких частных каналов они были отправлены.

Так как каждый частный канал имеет собственный SharePoint сайт, который не является родительским сайтом группы, управление файлами в частном канале не является родительским.

Teams не поддерживает поиск по обнаружению электронных данным по одному каналу в группе, поэтому поиск необходимо всем ее командам. Чтобы выполнить поиск контента с помощью eDiscovery в частном канале, выполните поиск по всей команде, по семейому веб-сайту, связанному с частным каналом (чтобы включить файлы), а также по почтовым ящикам участников частного канала (чтобы включить сообщения).

Чтобы идентифицировать файлы и сообщения в частном канале, чтобы включить их в поиск при обнаружении электронных данных, с помощью следующих действий:

### <a name="include-private-channel-files-in-an-ediscovery-search"></a>Включить файлы частного канала в поиск eDiscovery

Перед выполнением этих действий установите SharePoint [Online Management Shell и подключите его к SharePoint Online](/powershell/sharepoint/sharepoint-online/connect-sharepoint-online).

1. Чтобы получить список всех веб-SharePoint, связанных с частными каналами в группе, запустите следующую команду:

    ```PowerShell
    Get-SPOSite
    ```

2. Запустите следующий сценарий PowerShell, чтобы получить список URL SharePoint url-адресов всех сайтов, связанных с частными каналами в группе и ид родительской группы.

    ```PowerShell
    $sites = get-sposite -template "teamchannel#0"
    foreach ($site in $sites) {$x= get-sposite -identity $site.url -detail; $x.relatedgroupID; $x.url}
    ```

3. Для каждой группы или группы запустите следующий сценарий PowerShell, чтобы определить все релевантные сайты частных каналов, `$groupID` где находится ИД группы.

    ```PowerShell
    $sites = get-sposite -template "teamchannel#0"
    $groupID = "e8195240-4a70-4830-9106-80193cf717cb"
    foreach ($site in $sites) {$x= Get-SpoSite -Identity $site.url -Detail; if ($x.RelatedGroupId -eq $groupID) {$x.RelatedGroupId;$x.url}}
    ```
> [!NOTE]
> SharePoint личных каналов, созданных после 28 июня 2021 `teamchannel#1` г., будут использовать значение для пользовательского кода шаблона. Поэтому для закрытых каналов, созданных после этой даты, `teamchannel#1` используйте значение при запуске двух предыдущих сценариев.

### <a name="include-private-channel-messages-in-an-ediscovery-search"></a>Включить сообщения частного канала в поиск eDiscovery

Прежде чем выполнять эти действия, убедитесь, что у вас установлена последняя версия Teams [PowerShell](teams-powershell-overview.md).

1. Чтобы получить ид группы, которая содержит частные каналы, в которые вы хотите выполнить поиск, запустите следующую команду:

   ```powershell
   Get-Team -MailNickName <mail alias of the associated Office 365 Group>
   ```

   > [!TIP]
   > Запустите **командлет Get-Team** без параметров, чтобы отобразить список всех Teams организации. Список содержит код группы и MailNickName для каждой команды.

2. Чтобы получить список частных каналов в команде, запустите следующую команду: Используйте ИД группы, полученный в шаге 1.

    ```PowerShell
    Get-TeamChannel -GroupId <GroupId> -MembershipType Private
    ```

3. Чтобы получить список владельцев и участников частного канала, запустите следующую команду:

    ```PowerShell
    Get-TeamChannelUser -GroupId <GroupId> -DisplayName "Engineering" 
    ```

4. Включите в поисковый запрос [eDiscovery](/microsoft-365/compliance/search-for-content-in-core-ediscovery) почтовые ящики всех владельцев и участников каждого частного канала в команду.

## <a name="search-for-content-for-guest-users"></a>Поиск контента для гостевых пользователей

Средства eDiscovery можно использовать для поиска Teams, связанных с гостевых пользователей в организации. Teams содержимого чата, связанного с гостевым пользователем, сохраняется в облачном хранилище, и его можно искать с помощью eDiscovery. Это включает поиск контента в чатах 1:1 и 1:N, в которых гостевая пользователь является участником с другими пользователями в вашей организации. Вы также можете искать сообщения частного канала, в которых гостевом пользователем является участник, и искать содержимое  в беседах гостевого чата, в которых единственными участниками являются гостевых пользователей.

Чтобы найти контент для гостевых пользователей:

1. Подключение в Azure AD PowerShell. Инструкции см. в разделе "Подключение с Azure Active Directory PowerShell" Подключение для Microsoft 365 [PowerShell](/microsoft-365/enterprise/connect-to-microsoft-365-powershell#connect-with-the-azure-active-directory-powershell-for-graph-module). Не забудьте выполнить этапы 1 и 2 в предыдущей теме.

2. После успешного подключения к Azure AD PowerShell запустите следующую команду, чтобы отобразить имя директора-пользователя (UPN) для всех гостевых пользователей в организации. При создании поиска на шаге 4 необходимо использовать имя-имя гостя.

   ```powershell
   Get-AzureADUser -Filter "userType eq 'Guest'" -All $true | FL UserPrincipalName
   ```

   > [!TIP]
   > Вместо отображения списка имен участников-пользователей на экране компьютера можно перенаправить выходные данные команды в текстовый файл. Для этого можно использовать предыдущую `> filename.txt` команду. Текстовый файл с именами основных пользователей будет сохранен в текущей папке.

3. В другом окне Windows PowerShell к PowerShell центра & безопасности. Инструкции см. в Подключение [PowerShell центра & безопасности](/powershell/exchange/connect-to-scc-powershell). Вы можете подключаться с помощью многофакторной проверки подлинности или без нее.

4. Создайте поиск контента, в котором выполняется поиск всего контента (например, сообщений чата и сообщений электронной почты), в котором указанный гость был участником:

   ```powershell
   New-ComplianceSearch <search name> -ExchangeLocation <guest user UPN>  -AllowNotFoundExchangeLocationsEnabled $true -IncludeUserAppContent $true
   ```

   Например, чтобы найти содержимое, связанное с гостевых пользователей Е.Ермолаевых, нужно выполнить следующую команду:

   ```powershell
   New-ComplianceSearch "Sara Davis Guest User" -ExchangeLocation "sara.davis_hotmail.com#EXT#@contoso.onmicrosoft.com" -AllowNotFoundExchangeLocationsEnabled $true -IncludeUserAppContent $true
   ```

    Дополнительные сведения об использовании PowerShell для создания поиска контента см. в [new-ComplianceSearch](/powershell/module/exchange/new-compliancesearch).

5. Чтобы начать поиск контента, созданного в шаге 4, запустите следующую команду:

   ```powershell
   Start-ComplianceSearch <search name>
   ```

6. Перейдите к [https://compliance.microsoft.com](https://compliance.microsoft.com) полю **и нажмите кнопку Показать** **всеКонтементный** >  поиск.

7. В списке поисковых запросов выберите поиск, созданный на шаге 4, чтобы отобразить вылетную страницу.

8. На странице вы можете сделать следующее:

   - Нажмите **кнопку Просмотреть** результаты, чтобы просмотреть результаты поиска и просмотреть содержимое.

   - Рядом с **полем Query (Запрос** ) **щелкните Edit** (Изменить), чтобы изменить и повторно запросить результаты поиска. Например, вы можете добавить поисковый запрос, чтобы сузить результаты.

   - Нажмите **кнопку Экспортировать** результаты, чтобы экспортировать и скачать результаты поиска.

## <a name="search-for-card-content"></a>Поиск содержимого карт

Содержимое карточных карт, созданных приложениями в Teams каналах, чаты 1:1 и чаты 1xN, сохраняется в почтовых ящиках и может быть поиском. *Карточка* — это контейнер пользовательского интерфейса для коротких фрагментов содержимого. Карточки могут иметь несколько свойств и вложений, а также кнопки, которые могут запускать действия с карточками. Дополнительные сведения см. в [карточках](/microsoftteams/platform/task-modules-and-cards/what-are-cards)

Как и Teams, содержимое карточки хранится в зависимости от того, где она использовалась. Содержимое карточек, используемых Teams канале, хранится в почтовом Teams группы. Содержимое карток чатов 1:1 и 1xN сохраняется в почтовых ящиках участников чата.

Для поиска содержимого карточки можно использовать условия `kind:microsoftteams` `itemclass:IPM.SkypeTeams.Message` или условия поиска. При проверке результатов поиска содержимое карточки, сгенерированное ботами в канале Teams, имеет свойство Отправитель **/**`<appname>@teams.microsoft.com`Автор сообщения электронной почты как , `appname` где находится имя приложения, сгенер которого было сгенерировано содержимое карточки. Если содержимое карточки было сгенерировано пользователем, его определяет значение **Отправитель или** Автор.

При просмотре содержимого карточки в результатах поиска контента оно отображается как вложение в сообщение. Вложение называется `appname.html`, где `appname` находится имя приложения, сгенер которого было создано содержимое карточки.
