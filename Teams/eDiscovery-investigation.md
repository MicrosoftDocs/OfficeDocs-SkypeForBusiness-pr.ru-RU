# <a name="conduct-an-ediscovery-investigation-of-content-in-microsoft-teams"></a>Обнаружение электронных данных в Microsoft Teams

Крупные предприятия часто становятся в судебных разбирательствах по запросу отправки всей хранимой в электронном виде информации (ESI). Microsoft Teams контент можно искать и использовать во время исследования eDiscovery.

## <a name="overview"></a>Обзор

Все Microsoft Teams чаты 1:1 или группы проходят журнал в почтовые ящики соответствующих пользователей. Все стандартные сообщения каналов пересылаются в почтовый ящик группы, представляющий группу. Файлы, добавленные в стандартные каналы, охватываются функциями eDiscovery для SharePoint Online и OneDrive для бизнеса.

EDiscovery сообщений и файлов в закрытых [каналах](private-channels.md) работает не так, как в стандартных каналах. Дополнительные узнать см. в этой [ссылке.](#ediscovery-of-private-channels)

## <a name="ediscovery-of-private-channels"></a>EDiscovery частных каналов

Записи для сообщений, отправленных по частному каналу, доставляются в почтовый ящик всех участников частного канала, а не в групповой почтовый ящик. Названия записей форматируются, чтобы указать, с какого частного канала они были отправлены.

Так как у каждого закрытого канала есть собственный SharePoint сайт, который не является родительским сайтом группы, управление файлами в частном канале не будет действовать независимо от родительской группы.

Teams не поддерживает поиск по обнаружению электронных данным по одному каналу в группе, поэтому поиск необходимо для всей группы. Чтобы выполнить поиск контента с помощью eDiscovery в частном канале, выполните поиск по всей команде, по семейому веб-сайту, связанному с закрытым каналом (чтобы включить файлы) и почтовым ящикам участников частного канала (чтобы включить сообщения).

Чтобы идентифицировать файлы и сообщения в частном канале, чтобы включить их в поиск при обнаружении электронных данных, с помощью следующих действий:

### <a name="include-private-channel-files-in-an-ediscovery-search"></a>Включить файлы частных каналов в поиск eDiscovery

Перед выполнением этих действий установите SharePoint Online Management Shell и [подключите его к SharePoint Online.](/powershell/sharepoint/sharepoint-online/connect-sharepoint-online)

1. Чтобы получить список всех веб-SharePoint, связанных с частными каналами в группе, запустите следующую команду:

    ```PowerShell
    Get-SPOSite
    ```

2. Запустите следующий сценарий PowerShell, чтобы получить список URL-SharePoint сайтов, связанных с частными каналами в группе и ид родительской группы.

    ```PowerShell
    $sites = get-sposite -template "teamchannel#0"
    foreach ($site in $sites) {$x= get-sposite -identity $site.url -detail; $x.relatedgroupID; $x.url}
    ```

3. Для каждой группы или ИД группы запустите следующий сценарий PowerShell, чтобы определить все соответствующие сайты частных каналов, где $groupID — это ИД группы.

    ```PowerShell
    $sites = get-sposite -template "teamchannel#0"
    $groupID = "e8195240-4a70-4830-9106-80193cf717cb"
    foreach ($site in $sites) {$x= Get-SpoSite -Identity $site.url -Detail; if ($x.RelatedGroupId -eq $groupID) {$x.RelatedGroupId;$x.url}}
    ```

### <a name="include-private-channel-messages-in-an-ediscovery-search"></a>Включить сообщения частного канала в поиск eDiscovery

Перед выполнением этих действий убедитесь, что у вас установлена последняя версия Teams [PowerShell.](teams-powershell-overview.md)

1. Чтобы получить список частных каналов в команде, запустите следующую команду:

    ```PowerShell
    Get-TeamChannel -GroupId <GroupID> -MembershipType Private
    ```

2. Чтобы получить список участников частного канала, запустите следующую команду:

    ```PowerShell
    Get-TeamChannelUser -GroupId <GroupID> -DisplayName "Engineering" -Role Member
    ```

3. Включите в поисковый запрос [eDiscovery](/microsoft-365/compliance/search-for-content-in-core-ediscovery)почтовые ящики всех участников из каждого частного канала в команду.

## <a name="search-for-content-for-guest-users"></a>Поиск контента для гостевых пользователей

Средства eDiscovery можно использовать для поиска Teams, связанных с гостевых пользователей в организации. Teams содержимого чата, связанного с гостевым пользователем, сохраняется в облачном хранилище, и его можно искать с помощью eDiscovery. Это включает поиск контента в чатах 1:1 и 1:N, в которых гостевая пользователь является участником с другими пользователями в организации. Вы также можете искать сообщения в частном канале, в которых  гостевом пользователем является участник, и искать содержимое в беседах гостевого чата, в которых единственными участниками являются гостевых пользователей.

Чтобы найти контент для гостевых пользователей:

1. Подключение в Azure AD PowerShell. Инструкции см. в разделе "Подключение с Azure Active Directory PowerShell" в [Подключение-Microsoft 365 PowerShell.](/microsoft-365/enterprise/connect-to-microsoft-365-powershell#connect-with-the-azure-active-directory-powershell-for-graph-module) Не забудьте выполнить этапы 1 и 2 в предыдущей теме.

2. После успешного подключения к Azure AD PowerShell запустите следующую команду, чтобы отобразить имя директора-пользователя (UPN) для всех гостевых пользователей в организации. При создании поиска на шаге 4 необходимо использовать имя-имя гостя.

   ```powershell
   Get-AzureADUser -Filter "userType eq 'Guest'" -All $true | FL UserPrincipalName
   ```

   > [!TIP]
   > Вместо отображения списка имен участников-пользователей на экране компьютера можно перенаправить результат команды в текстовый файл. Для этого можно использовать предыдущую `> filename.txt` команду. Текстовый файл с именами основных пользователей будет сохранен в текущей папке.

3. В другом окне Windows PowerShell к PowerShell Центра & безопасности. Инструкции см. в Подключение [PowerShell центра](/powershell/exchange/connect-to-scc-powershell)& безопасности. Вы можете подключаться с помощью многофакторной проверки подлинности или без нее.

4. Создайте поиск контента для поиска всего контента (например, сообщений чата и сообщений электронной почты), в котором указанный гость был участником:

   ```powershell
   New-ComplianceSearch <search name> -ExchangeLocation <guest user UPN>  -AllowNotFoundExchangeLocationsEnabled $true -IncludeUserAppContent $true
   ```

   Например, чтобы найти содержимое, связанное с гостевых пользователей Е.Ермолаевых, нужно выполнить следующую команду:

   ```powershell
   New-ComplianceSearch "Sara Davis Guest User" -ExchangeLocation "sara.davis_hotmail.com#EXT#@contoso.onmicrosoft.com" -AllowNotFoundExchangeLocationsEnabled $true -IncludeUserAppContent $true
   ```

    Дополнительные сведения об использовании PowerShell для создания поиска контента см. в [new-ComplianceSearch.](/powershell/module/exchange/new-compliancesearch)

5. Чтобы начать поиск контента, созданного в шаге 4, запустите следующую команду:

   ```powershell
   Start-ComplianceSearch <search name>
   ```

6. Перейдите к [https://compliance.microsoft.com](https://compliance.microsoft.com) кнопке **Показать весь поиск**  >  **контента.**

7. В списке поисковых запросов выберите поиск, созданный на шаге 4, чтобы отобразить вылетную страницу.

8. На странице вы можете сделать следующее:

   - Нажмите **кнопку Просмотреть** результаты, чтобы просмотреть результаты поиска и просмотреть содержимое.

   - Рядом с **полем Query (Запрос)** **щелкните Edit (Изменить),** чтобы изменить, а затем повторно нажмите кнопку Search (Запрос). Например, вы можете добавить поисковый запрос, чтобы сузить результаты.

   - Нажмите **кнопку Экспортировать** результаты, чтобы экспортировать и скачать результаты поиска.

## <a name="search-for-card-content"></a>Поиск содержимого карт

Содержимое карток, сгенерированное приложениями в Teams каналах, 1:1 чатах и 1xN-чатах, сохраняется в почтовых ящиках и может искаться. *Карточка* — это контейнер пользовательского интерфейса для коротких фрагментов содержимого. Карточки могут иметь несколько свойств и вложений, а также кнопки, которые могут запускать действия с карточками. Дополнительные сведения см. в [карточках](/microsoftteams/platform/task-modules-and-cards/what-are-cards)

Как и Teams, содержимое карточки хранится в зависимости от места использования карточки. Содержимое карточек, используемых Teams канале, хранится в почтовом Teams группы. Содержимое карток для чатов 1:1 и 1xN сохраняется в почтовых ящиках участников чата.

Для поиска содержимого карточки можно использовать условия или `kind:microsoftteams` `itemclass:IPM.SkypeTeams.Message` условия поиска. При проверке результатов поиска содержимое карточки, сгенерированное ботами в канале Teams, имеет свойство **Отправитель/Автор** сообщения электронной почты как , где имя приложения, сгенер которого было сгенерировано `<appname>@teams.microsoft.com` содержимое `appname` карточки. Если содержимое карточки было сгенерировано пользователем, его определяет значение **Отправитель или** Автор.

При просмотре содержимого карточки в результатах поиска контента оно отображается как вложение в сообщение. Вложение называется , где находится имя приложения, сгенер которого `appname.html` `appname` было создано содержимое карточки. ## <a name="advanced-ediscovery"></a>Advanced eDiscovery

Некоторые Microsoft Teams контента также можно искать и сохранять с помощью [Advanced eDiscovery рабочего процесса](/microsoft-365/compliance/overview-ediscovery-20). Хотя eDiscovery предоставляет диапазон функций поиска, хранения и экспорта, Advanced eDiscovery предоставляет администраторам соответствия требованиям дополнительные средства для определения источников данных и анализа их содержимого.

### <a name="advanced-ediscovery-custodian-workflow-for-teams-content"></a>Advanced eDiscovery хранителя для Teams контента

Хранители могут быть членами разных команд. Вы можете Teams содержимое, которое относится к хранителям. Инструкции по работе хранителей см. в этой Advanced eDiscovery [дела.](/microsoft-365/compliance/add-custodians-to-case)

Добавив хранителя, нажмите **кнопку** Далее, а затем **кнопку** Добавить. После этого в окне будет предложено выбрать дополнительные расположения, в которых будут показаны все члены хранителя и соответствующие расположения SharePoint сайта для их данных. Во всех этих источниках данных и командах вы можете выбрать контент, который вы хотите использовать для eDiscovery, а затем поместите этого пользователя и все источники данных, которые вы определили, на удержание.

Вы можете выбрать, следует ли включать Exchange контент, OneDrive контент или оба этих контента. Exchange содержимое включает все содержимое приложения в почтовых ящиках пользователя, например его электронную почту, содержимое Teams, храняное в его почтовом ящике, и так далее. Содержимое OneDrive содержит не только содержимое пользователя, но и все содержимое Teams, которое хранится в OneDrive, например чаты 1:1, чаты 1:N и файлы, к совместно хранимым в чатах.

Вы также можете связать любую команду, в которую входит хранителя, чтобы включить в нее сообщения чата и файлы, к которые есть у хранителя. Кроме того, с хранителями можно связывать любые другие группы.

> [!NOTE]
> EDiscovery сообщений и файлов в закрытых [каналах](private-channels.md) работает не так, как в стандартных каналах. Дополнительные узнать см. в этой [ссылке.](#ediscovery-of-private-channels)

### <a name="placing-a-data-source-on-hold"></a>Помещение источника данных на удержание

Если конкретный пользователь не назначен хранителями, вы можете разместить на удержании весь источник данных. Дополнительные сведения об управлении удерживами см. [в](/microsoft-365/compliance/managing-holds)Advanced eDiscovery.

При создании удержания для Teams вы можете выбрать все расположения, которые вы хотите включить в удержание. Даже если пользователи удаляют или изменяют содержимое, удержание будет поддерживать копии всех предыдущих версий этого контента.

Вы также можете использовать необязательный запрос для определения условий удержания на основе ключевых слов, диапазона дат, автора и многих других условий. Если не указать ни одного ключевого слова, все данные из этого источника данных будут подлежать удержанию.

### <a name="advanced-ediscovery-searches"></a>Advanced eDiscovery поиска

Teams контент также можно искать. Дополнительные сведения о поиске см. в [этой](/microsoft-365/compliance/collecting-data-for-ediscovery)Advanced eDiscovery. При поиске будет возвращена вся беседа, если поисковому запросу соответствует хотя бы одно сообщение.

При создании поискового запроса можно выбрать хранителей, чтобы поиск был по всем выбранным источникам. Вы также можете искать данные из других источников, таких как Teams, не соедиавли с пользователем. Кроме того, с помощью необязательных запросов можно сузить поиск в Teams контенте.

После создания и выбора поискового запроса в окне отображаются дополнительные сведения и действия, которые можно использовать для выбранного поиска. Если нажать  кнопку Статистика, вы сможете просмотреть статистику поиска, включая разбивку по типам расположения, исходному источнику контента, а также по расположению контента в почтовом ящике группы, почтовом ящике отдельного пользователя или SharePoint сайте. Таким образом, вы можете увидеть разбивку источников, которые повествуют о результатах поиска. Кроме того, доступно **представление Запросы,** в котором можно увидеть, какие отдельные ключевые слова повествуют о результатах.

Завершив поиск, вы можете нажать кнопку Добавить результаты для проверки **и** добавить ее в набор для проверки. Дополнительные сведения о наборах проверки см. в статьях Управление наборами проверки в Advanced eDiscovery [и](/microsoft-365/compliance/managing-review-sets) Просмотр наборов [далее](#review-sets-workflow) в этой статье.

#### <a name="normal-review-sets-and-conversation-review-sets"></a>Обычные наборы проверки и наборы проверки бесед

При добавлении поиска в набор проверки можно выбрать обычный набор или набор для проверки беседы.

Обычный набор рецензий похож на экспорт; она содержит отдельные файлы для Teams контента и представляет `.msg` его в базовом представлении. Обычно для повторной проверки файлов используется обычный набор.

Набор рецензий бесед обеспечивает более интуитивно понятное и многопоточный просмотр бесед; соответствующие сообщения отображаются в правильном порядке.

> [!div class="mx-imgBorder"]
> ![Снимок экрана: набор проверки беседы](media/conversationOptions2.png)

Такие функции, как redaction, доступны в обоих типах наборов проверки. Дополнительные сведения о наборах проверки см. в документе [Просмотр бесед в advanced eDiscovery.](/microsoft-365/compliance/conversation-review-sets)

#### <a name="collection-options"></a>Параметры коллекции

При добавлении в набор проверки в разделе Параметры коллекции окна доступны несколько  параметров, включая Параметры иконки и Teams **беседы.**  Если включить эти параметры, Teams сообщения, которые являются частью вашего набора для проверки, также будут показаны с дополнительными сообщениями, окружающими их для контекста. Например, если запрос является конкретным и в результате возвращается только одно сообщение, включение этих параметров также возвратит несколько сообщений, которые соответствуют запросу.

Множество логических критериев используются для определения того, обеспечивают ли дополнительные сообщения контекст для сообщений, которые соответствуют вашему запросу. Например, если Teams контента, при включите эти параметры будут извлекать родительское сообщение и все сообщения ребенка из-за цепочки сообщений.

Также будут проверяться отметки времени сообщений. Если сообщение соответствует вашему запросу, то соседние сообщения, которые предшествуют ему в течение 4 часов или следуют за ним в течение 4 часов, считаются частью беседы и также включаются в результаты.

Если вы должны быть уверены, какие контекстные сообщения будут возвращены с совпадениями с поисковым запросом, использовать эти параметры не нужно. Вы можете собрать весь контент или расширить диапазон дат поиска, чтобы больше сообщений возвращалось в результате запроса.

### <a name="review-sets-workflow"></a>Рабочий процесс "Проверка наборов"

Вы можете просмотреть существующие наборы проверки или создать новые на **вкладке Наборы проверки.** Дополнительные сведения о наборах проверки см. [в](/microsoft-365/compliance/managing-review-sets)Advanced eDiscovery.

Помимо документов, в набор проверки можно добавлять сообщения электронной почты, Teams сообщения, Yammer сообщения и другое содержимое. В наборе проверки можно выполнять многие операции, которые можно выполнять в других контекстах, например поиск контента и создание пользовательских запросов. Эти операции относятся только к тем пунктам, которые были добавлены в набор проверки.

Кнопка **Управление наборами** проверки предоставляет дополнительные параметры, такие как аналитика, сводные отчеты, количество добавленных наборов загрузки и так далее.

Чтобы получить доступ к визуализациям  и диаграммам с данными, щелкните Отдельные результаты \> **Просмотр профиля поиска** в правом верхнем поле. Чтобы выбрать тип контента для запроса, щелкните на этих диаграммах. Например, можно выбрать запрос только для Teams контента. Эти запросы также можно сохранить так же, как и запросы, которые вы пишете вручную.

#### <a name="summary-view-text-view-and-annotate-view"></a>Представление "Сводка", "Текст" и представление "Примечать"

Если щелкнуть беседу Teams в наборе рецензировки, отобразится представление Сводка **,** в котором вся беседа Teams в качестве списка сообщений, с которыми можно взаимодействовать по отдельности. Щелкните стрелку вниз справа от сообщения, чтобы отобразить контекстное меню, в котором можно просмотреть сведения о сообщении или скачать отдельный `.msg` файл. Щелкнув сведения о сообщении, вы сможете отобрать сводку метаданных или полные метаданные сообщения.

Чтобы скачать PDF-файл, нажмите кнопку скачивания в правом верхнем верхнем конце представления сводки.

Щелкните **вкладку Текстовый** вид, чтобы отобразить обычный текст извлеченного текста Teams беседы. Этот обычный текст подходит для экспорта, и вы можете легко работать с ним с помощью других программных средств.

Щелкните **вкладку Представление Примечания,** чтобы получить доступ к функциям примечания. На этой вкладке содержимое отображается в формате, напоминаемом Teams беседе, но есть и дополнительные параметры редактирования. Существует инструмент карандаша, который можно использовать для создания заметок, создания набросков в сообщении или создания точной разметки в целях redaction. Кроме того, с помощью инструмента **redaction** области можно нарисовать прямоугольник, который затемнение области и пометит его как "Красный".

Вот пример отредакт доступного файла для цепочки бесед между пользователями.

> [!div class="mx-imgBorder"]
> ![Снимок экрана: красный файл](media/RedactedFileExample.png)

В нижней части вкладки Представление  **примечания** находится кнопка Документы тегов, которая отображает панель маркировки. На этой панели можно применить тег для всех сообщений в Teams беседе. Вы можете пометить беседу как реагировать или не реагировать, иметь привилегии или нет привилегий, включать ли беседу в экспорт, включать ли ее в экспорт и требует ли она дальнейшего анализа. Вы также можете управлять и применять другие настраиваемые теги.

#### <a name="action-menu"></a>Меню действий

В окне наборов проверки вы можете экспортировать содержимое, нажав кнопку **Экспорт** \> **действий.** Существует множество вариантов экспорта.

Чтобы экспортировать файл, содержащий все метаданные для всех Teams сообщений, щелкните, чтобы выбрать файл **загрузки.** Чтобы включить в файл все теги, примененные к содержимому, щелкните, чтобы выбрать **теги.**

Используйте параметр **Собственные файлы** для экспорта файлов в их родном формате. Вы можете экспортировать беседу как один файл или все отдельные сообщения чата в отдельные файлы.

Параметр **Текстовые** файлы позволяет сохранять версии содержимого в виде обычного текста. Дополнительные сведения о [том,](#summary-view-text-view-and-annotate-view) как получить обычный текст для Teams бесед в наборе рецензировок, см. в вышесказанном представлении сводки, текстовом представлении и представлении сносок.

Если вы применили какие-либо отредакции к содержимому, как описано в разделе [Сводка,](#summary-view-text-view-and-annotate-view) текстовое представление и представление примечаний выше, вы можете выбрать параметр Заменить старые форматы преобразованными PDF-файлами, чтобы заменить их преобразованными копиями в PDF. 

Вы можете экспортировать данные в контейнер хранилища BLOB-данных Azure, предоставленный Майкрософт, или предоставить собственный контейнер хранилища BLOB-хранилищ Azure.

Когда вы будете готовы начать экспорт, нажмите **кнопку** Экспорт. Дополнительные [сведения о том,](/microsoft-365/compliance/download-export-jobs) как получить доступ к контейнеру хранилища BLOB-содержимого Azure и скачать экспортируемую информацию после завершения экспорта, см. в этой статьи.

> [!NOTE]
> Экспорт может занять продолжительное время. Чтобы отслеживать состояние экспорта, зайдите на **вкладку** Наборы проверки и щелкните **вкладку Экспорты.**
