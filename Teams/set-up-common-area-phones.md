# <a name="deploy-common-area-phones-for-microsoft-teams"></a>Развертывание телефонов общего пользования для Microsoft Teams

Телефон общего содержимого обычно размещается в таких областях, как "зал ожидания" или другой регион, доступный многим людям для звонка; например, зона приема, зал ожидания или конференц-телефон. Телефоны общего пользования выполняют вход с учетными записями, привязанными к лицензии common Area Phone.

В этой статье приводятся общие сведения о развертывании и настройке телефонов Teams в качестве телефонов общего пользования для общих пространств. Для более полного взаимодействия с конференц-залами, включая аудиоконференции, рассмотрите возможность приобретения лицензии на выделенную комнату для собраний с помощью устройства комнаты для собраний.

## <a name="overview"></a>Обзор

Лицензия common Area Phone поддерживает:

|                                           | Телефон общего пользования                                 |
|-------------------------------------------|---------------------------------------------------|
| **Microsoft Teams**                       | &#x2714;                                          |
| **Teams Phone &**  sup1;                   | &#x2714;                                          |
| **Аудиоконференции**                    | &#x2718; &sup2;                                   |
| **Microsoft Intune**                      | &#x2714;                                          |
| **Azure Active Directory Premium план 1** | &#x2714;                                          |
| **Exchange Online плана 2**                | &#x2714;  &sup3;                                  |
| **Доступность по всему миру**                | &#x2714;                                          |
| **Доступность канала**                  | EA, EAS, EES, CSP, Web Direct, GCC, GCC-High, DoD |

&sup1; Ранее известная как телефонная система.

&sup2; Телефоны общего пользования могут присоединяться к аудиоконференции по номеру телефонного подключения, предоставленного организатором собрания.  

&sup3; Только облачные возможности голосовой почты.

>[!NOTE]
> Учетные записи для объектов телефонов общего пользования, созданных в Skype для бизнеса Server не могут быть перенесены в Microsoft Teams. Выполните действия, описанные в этой статье, чтобы повторно создать эти учетные записи для Teams и при необходимости перенести подключение по ТСОП.

## <a name="step-1---buy-the-licenses"></a>Шаг 1. Приобретение лицензий

Сначала необходимо приобрести лицензию на телефон общего содержимого (CAP) и убедиться, что у вас есть сертифицированный телефон. Чтобы найти и узнать больше о сертифицированных телефонах, перейдите на [устройства Microsoft Teams](https://products.office.com/microsoft-teams/across-devices?ms.url=officecomteamsdevices&rtc=1).

1. В Центр администрирования Microsoft 365 перейдите к **службам покупки** > **выставления счетов**. 

2. Если раздел "Просмотр по **категориям** " еще не отображается, перейдите к разделу "Приобретение у корпорации **Майкрософт**" и выберите " **Просмотреть продукты"**. Затем выберите " **Совместная работа и взаимодействие"**.  

3. В списке продуктов найдите **телефон общего содержимого и** выберите " **Сведения"**.

4. Введите необходимое количество лицензий и нажмите кнопку **"Купить"**.

>[!NOTE]
>Если вы используете Intune в своей среде и имеете правила условного доступа, которые требуют соответствия устройств, вам потребуется назначить лицензию Azure Active Directory Premium (план 1) и Intune учетной записи устройства для телефона общего содержимого.
>
>На телефоны общего пользования могут влиять правила условного доступа и другие конфигурации удостоверений, такие как Многофакторная идентификация. Дополнительные сведения см. в рекомендациях по проверке подлинности [для устройств Android в Teams](devices/authentication-best-practices-for-android-devices.md) .

## <a name="step-2---create-a-new-user-account-and-assign-licenses"></a>Шаг 2. Создание учетной записи пользователя и назначение лицензий

### <a name="using-the-microsoft-365-admin-center"></a>Использование Центр администрирования Microsoft 365

Если вы развертываете несколько телефонов общего пользования одновременно, узнайте, как создавать учетные записи и назначать лицензии [с помощью PowerShell](#using-powershell).

При развертывании одного устройства:

1. В Центр администрирования Microsoft 365 перейдите к **разделу "Пользователи** > **— активные пользователи" "** > **Добавить пользователя"**.

2. Введите имя пользователя, например Main, в качестве имени и "Прием" для второго имени.

3. Введите отображаемое имя, если оно не создается автоматически, например "Основной прием".

4. Введите имя пользователя, например MainReception или Mainlobby.

5. Задайте пароль для телефона общего содержимого вручную. Для этого снимите флажок **"Автоматически создать** пароль" и требовать от этого пользователя изменить пароль при **первом входе**.  

    >[!Important]
    > Настоятельно рекомендуется вручную задать пароль для телефонов общего пользования, чтобы предотвратить проблемы со входом для конечных пользователей.

6. Выберите расположение использования устройства и назначьте учетной записи лицензию common Area Phone. Если требуются какие-либо другие лицензии, например планы звонков, назначьте их.

>[!NOTE]
> Вам не нужно добавлять лицензию на телефонную систему. Она включена в лицензию Телефона общего пользования.
>
>Если вы не используете прямую маршрутизацию или подключение к телефонной системе Майкрософт, вам может потребоваться добавить лицензии на планы звонков. Дополнительные сведения о лицензиях см. в разделе ["Лицензирование надстроек Microsoft Teams"](./teams-add-on-licensing/microsoft-teams-add-on-licensing.md).

### <a name="using-powershell"></a>С помощью PowerShell

Используйте PowerShell, если вы хотите создать и назначить лицензии для нескольких учетных записей пользователей одновременно. Дополнительные сведения см. в статье "Создание учетных записей пользователей [Microsoft 365 с помощью PowerShell](/microsoft-365/enterprise/create-user-accounts-with-microsoft-365-powershell?view=o365-worldwide&preserve-view=true) и назначение лицензий [Microsoft 365](/microsoft-365/enterprise/assign-licenses-to-user-accounts-with-microsoft-365-powershell?view=o365-worldwide&preserve-view=true) учетным записям пользователей с помощью PowerShell".

## <a name="step-3---set-policies-for-common-area-phones"></a>Шаг 3. Настройка политик для телефонов общего пользования

Используйте политики для управления функциями, доступными пользователям на телефонах общего пользования.

>[!NOTE]
>Назначив политику, выйдите из телефона и войдите обратно. Чтобы назначение политики вступает в силу, может потребоваться до часа.

### <a name="ip-phone-policies"></a>Политики IP-телефонов

На телефонах, выполнив вход с помощью учетных записей, которым назначена лицензия common Area Phone, будет отображаться общее взаимодействие с пользователем.

Если вы хотите переопределить интерфейс телефона по умолчанию, рассмотрите возможность создания [политики IP-телефонов](/powershell/module/skype/new-csteamsipphonepolicy?view=skype-ps&preserve-view=true). Например, если телефон общего содержимого используется в общедоступной области, задайте политику IP-телефонов, чтобы ограничить поиск в глобальной адресной книге вашей организации и заблокировать горячее обслуживание. Дополнительные сведения см. в разделе ["Настройка пользовательского интерфейса устройств Android для Teams](devices/Teams-Android-devices-user-interface.md) ".

### <a name="calling-policies"></a>Политики звонков

Используйте политики звонков, чтобы включить частные звонки, используя переадресацию звонков или одновременный звонок на телефонах общего пользования. Дополнительные сведения см. в разделе "Звонки и [переадресация звонков в Teams](teams-calling-policy.md) ".

По умолчанию парковка звонков не включена для телефонов общего пользования. Чтобы включить политику, необходимо создать ее. [Дополнительные сведения см. в](call-park-and-retrieve.md) разделе о парковке звонков и извлечении в Microsoft Teams.

## <a name="step-4---acquire-and-assign-phone-numbers"></a>Шаг 4. Получение и назначение номеров телефонов

Сведения [о том](manage-phone-numbers-landing-page.md) , как получить и назначить номера телефонов в зависимости от варианта подключения к ТСОП, см. в статье "Управление телефонными номерами для вашей организации".

## <a name="step-5---sign-in"></a>Шаг 5. Вход

После создания и настройки учетной записи пользователя можно войти на телефон. В зависимости от количества развертываемых телефонов у вас есть три варианта входа:

- [Локальный вход](#local-sign-in)
- [Вход с другого устройства](#sign-in-from-another-device)
- [Вход с помощью Центра администрирования Teams](#sign-in-using-the-teams-admin-center)

### <a name="local-sign-in"></a>Локальный вход

Чтобы выполнить локальный вход с помощью имени пользователя и пароля: 

1. Включение телефона общего содержимого

2. Выберите **"Войти" на этом устройстве**

3. Следуйте инструкциям по входу на устройстве. После входа на телефоне отобразится общее взаимодействие с пользователем на телефоне.

> [!NOTE]
> Если вы используете настраиваемую политику установки, которая открепит вызывающее приложение, панель набора номера не будет отображаться на телефоне общего содержимого. Дополнительные сведения о политиках установки Teams см. в статье "Управление политиками установки [приложений в Microsoft Teams"](/microsoftteams/teams-app-setup-policies).

### <a name="sign-in-from-another-device"></a>Вход с другого устройства

Вы также можете войти на телефон общего содержимого с другого устройства с помощью кода. При входе таким образом вы введете имя пользователя и пароль на другом устройстве, а не на самом телефоне.

1. Сначала на телефоне общего содержимого найдите код, отображаемый на экране входа.

2. На другом устройстве перейдите к .https://www.microsoft.com/devicelogin

3. Введите код и следуйте инструкциям по выполнению входа.

### <a name="sign-in-using-the-teams-admin-center"></a>Вход с помощью Центра администрирования Teams

Администратор может удаленно подготавливать и входить на телефоны общего пользования из Центра администрирования Teams. Это наиболее эффективный метод входа при одновременном развертывании большого количества телефонов. Дополнительные сведения см. в разделе "Удаленная [подготовка и вход на устройствах Android в Teams](devices/remote-provision-remote-login.md) ".
