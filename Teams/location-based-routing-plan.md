# <a name="plan-location-based-routing-for-direct-routing"></a>Планирование маршрутизации на основе расположения для прямой маршрутизации

> [!INCLUDE [Preview customer token](includes/preview-feature.md)]

## <a name="overview-of-location-based-routing"></a>Обзор маршрутизации на основе местоположения

В некоторых странах и регионах не следует обходиться поставщиком услуг телефонной сети общего пользования (PSTN), чтобы снизить стоимость звонков на междугородную связь. В этой статье описано, как использовать маршрутизацию на основе местоположения для ограничения платных пользователей Microsoft Teams в соответствии с их географическим расположением. Эта статья относится только к прямой маршрутизации телефонной системы.

Здесь вы найдете общие сведения о маршрутизации на основе местоположения и руководствах, которые помогут вам спланировать ее. Когда вы будете готовы к применению и включению маршрутизации на основе местоположения, ознакомьтесь с разделами:
- [Развертывание сетевых параметров для маршрутизации на основе местоположения](location-based-routing-configure-network-settings.md)
- [Включение функции "Маршрутизация на основе расположения" для прямой маршрутизации](location-based-routing-enable.md)

Маршрутизация на основе местоположения — это функция, которая позволяет ограничивать платный звонок на основе политики и географического положения пользователя во время входящего или исходящего звонка по коммутируемой телефонной сети. 

Если для маршрутизации на основе расположения включена пользователь Teams, применяются следующие сведения.
- Чтобы выполнить исходящий звонок PSTN, необходимо выполнить одно из следующих условий:
    - Конечная точка пользователя находится на сетевом сайте, для которого включена маршрутизация на основе местоположения и звонки с помощью соответствующего шлюза, для которого включена маршрутизация на основе местоположения. 
    - Конечная точка пользователя находится на сетевом сайте, для которого не включена маршрутизация на основе местоположения и звонки с помощью шлюза, для которого не включена маршрутизация на основе местоположения.

    Исходящие звонки не разрешены в других сценариях.

- Чтобы получить входящий звонок по коммутируемой телефонной сети, конечная точка пользователя должна находиться на том же сайте, на котором он ингрессес через шлюз, который включен для маршрутизации на основе местоположения. В любом другом сценарии, например, если пользователь является перемещаемым, Звонок не разрешен и перенаправляется в параметры переадресации звонков пользователя (обычно голосовой почте).
- Чтобы передать КОММУТИРУЕМый Звонок на другой пользователь Teams, конечная точка конечного пользователя должна располагаться на том же сайте сети, что и пользователь, который инициировал передачу. В других сценариях передача не разрешена. 
- Для передачи другим пользователям Teams сети PSTN необходимо передать вызов через шлюз с поддержкой маршрутизации, расположенный на том же сетевом сайте, что и начальный вызывающий объект. В других сценариях передача не разрешена.

Маршрутизация на основе местоположения использует те же определения сетевого региона, сайта и подсети, которые используются в Skype для бизнеса Server. Если для местоположения не задано значение "платный звонок", Администратор связывает каждую подсеть IP и каждый шлюз PSTN для этого расположения на сайте сети. Расположение пользователя определяется IP-подсетью, к которой подключены конечные точки пользователя, на момент звонка по протоколу PSTN. Пользователь может иметь несколько клиентов Teams, расположенных на разных сайтах, и в этом случае маршрутизация на основе местоположения принудительно обеспечивает каждый клиентский маршрут отдельно в зависимости от расположения ее конечной точки. 

Чтобы узнать, как ознакомиться с некоторыми из терминологии, используемыми в этой статье, ознакомьтесь со статьей [терминология маршрутизации на основе местоположения](location-based-routing-terminology.md).

## <a name="apply-location-based-routing"></a>Применение маршрутизации на основе местоположения

Необходимо применить маршрутизацию на основе местоположения для пользователей, сетевых сайтов и шлюзов PSTN.  

### <a name="apply-location-based-routing-at-the-user-location"></a>Применение маршрутизации на основе местоположения в местоположении пользователя

Как упоминалось ранее, маршрутизация на основе местоположения применяется только для пользователей, которые настроены на прямую маршрутизацию. Маршрутизация на основе местоположения не распространяется на пользователей, которые настроены на план звонков. Пользователи должны быть включены для маршрутизации с учетом расположения, если они находятся в ограничении пропускной связи, что определяет условия, при которых они могут совершать и принимать звонки PSTN, а также шлюз PSTN, который можно использовать. Если пользователь, для которого включена маршрутизация на основе местоположения, находится на сайте, для которого включена маршрутизация на основе местоположения, пользователь должен выполнить звонки через шлюз, поддерживающий маршрутизацию на основе местоположения, подключенный к сайту. 

Маршрутизация на основе местоположения позволяет определить текущее расположение пользователя на основе IP-адреса конечной точки Teams пользователя и применяет эти правила соответствующим образом. Расположение пользователя, для которого включена маршрутизация на основе местоположения, можно разделить следующими способами: 
- **Пользователь расположен на том же сайте, поддерживающем маршрутизацию на основе местоположения, связанном с шлюзом PSTN, которому они назначены.**<br>В этом сценарии пользователь находится на известном сетевом сайте, для которого включена маршрутизация на основе местоположения, а номер прямого набора номера (-а) пользователя завершается на шлюзе PSTN, находящимся на том же сайте сети. Например, пользователь находится на своем офисе. 
- **Пользователь находится на другом сайте, поддерживающем маршрутизацию на основе местоположения, не связанном с шлюзом PSTN, где они были назначены.**<br>В этом сценарии пользователь находится на известном сетевом сайте, для которого включена маршрутизация на основе местоположения, и этот сайт не связан с шлюзом PSTN, в котором назначен номер пользователя. Например, пользователь перемещается на другой офис.  
- **Пользователь находится на внутреннем сайте, для которого не включена маршрутизация на основе местоположения.** <br>В этом сценарии пользователь находится на известном сайте внутренней сети, для которого не включена маршрутизация на основе местоположения. 
- **Пользователь находится на неизвестном сайте.** 
    - Пользователь находится в внутренней сети, которая не определена как сетевой сайт. 
    - Пользователь находится за пределами внутренней сети. Например, пользователь находится в Интернете на домашних компьютерах или в кафе. 

### <a name="apply-location-based-routing-at-the-network-site"></a>Применение маршрутизации на основе местоположения на сайте сети 
Сетевые сайты должны быть включены для маршрутизации с учетом расположения, чтобы определить, какие шлюзы должны быть включены в маршрут на основе местоположения при роуминге. Если пользователь, для которого включена маршрутизация на основе местоположения, перемещается на сайт, для которого включена маршрутизация на основе местоположения, для исходящих звонков можно использовать только шлюз PSTN, включенный для маршрутизации на основе местоположения на этом сайте. Если пользователь, для которого включена маршрутизация на основе местоположения, перемещается на сайт, для которого не включена маршрутизация на основе местоположения, для исходящих звонков можно использовать любой шлюз, для которого не включена маршрутизация на основе местоположения.  

### <a name="apply-location-based-routing-at-the-pstn-gateway"></a>Применение маршрутизации на основе местоположения на шлюзе PSTN 

Шлюзы связаны с сайтами, чтобы определить, где можно найти пользователя, поддерживающего маршрутизацию на основе местоположения, при совершении и получении звонка по сети PSTN. Шлюзы должны быть включены для маршрутизации с учетом расположения, чтобы убедиться, что она находится в ограничениях обходных ограничений и не может использоваться пользователями, которые не поддерживают маршрутизацию на основе местоположения. Один и тот же шлюз может быть связан с несколькими сайтами, и его можно настроить для маршрутизации на основе местоположения или не включать для маршрутизации на основе местоположения в зависимости от сайта.

### <a name="teams-user-transfers-or-forwards-call-to-another-teams-user"></a>Пользователи Teams пересылают или пересылает вызов другому пользователю Teams.

При использовании конечной точки КТСОП маршрутизация на основе местоположения анализирует, разрешены ли для маршрутизации на основе местоположения один или оба пользователя, и определяет, следует ли передавать или пересылать звонки в зависимости от расположения обеих конечных точек. 
 
Для передачи вызова требуется, чтобы вызывающий пользователь Выбери звонок, а переадресация звонков не требует ответа на первый звонок. Это означает, что звонки можно переадресовывать, даже если Пользователь1 не находится в местоположении для приема входящих звонков (вы можете обратиться к таблице в разделе [Получение входящего звонка из раздела PSTN](#teams-user-receives-an-inbound-call-from-the-pstn) ), и звонки не могут быть переданы, если пользователь User1 не может получить входящий звонок. 

#### <a name="user-not-enabled-for-location-based-routing"></a>Пользователь не включен для маршрутизации с учетом расположения

Пользователь, не поддерживающий маршрутизацию на основе местоположения, может передавать или переадресовывать звонки PSTN другим пользователям, которые не поддерживают маршрутизацию на основе местоположения. Обычно пользователю не разрешается передавать или пересылать КОММУТИРУЕМые звонки на пользователей, которые поддерживают маршрутизацию на основе местоположения, так как пользователи с включенной маршрутизацией на основе местоположения, как правило, размещаются только на шлюзах с поддержкой маршрутизации на основе местоположения для КТСОП. междугородн. Исключением является то, что пользователь, поддерживающий маршрутизацию на основе местоположения, перемещается на сайт, для которого не включена маршрутизация на основе местоположения. В этом случае передача звонка разрешена.  

Кроме того, пользователь, не поддерживающий маршрутизацию на основе местоположения, может получать звонок только от другого пользователя, для которого не включена маршрутизация на основе местоположения.

### <a name="simultaneous-ringing"></a>Одновременный звонок

Если пользователь, использующий для маршрутизации на основе местоположения, получает вызов и поддерживает одновременные звонки, маршрутизация на основе местоположения анализирует расположение вызывающего абонента и конечных точек вызываемого абонента, чтобы определить, следует ли маршрутизировать звонок. Входящие звонки PSTN из шлюза с разрешенной маршрутизацией разрешено подключаться к автосекретарьу или списку звонков. Пользователи, для которых включена маршрутизация на основе местоположения, могут получать входящие звонки из этих приложений, когда они находятся на том же сайте, из которого поступил входящий звонок КТСОП. 
 
Переадресация звонков и одновременные звонки между пользователями и PSTN разрешены для передачи голосовых приложений. Для выполнения звонка в целевую папку действуют те же правила маршрутизации, описанные выше.  
 
Также разрешена переадресация в голосовую почту.  

### <a name="delegation"></a>Делегирование

Пользователи Teams могут выбирать делегатов, которые могут совершать и принимать звонки от их имени. Возможности делегирования в Teams повлияют на маршрутизацию на основе местоположения, как описано ниже. 
- Для исходящих звонков с делегата, поддерживающего маршрутизацию на основе местоположения, от имени представителя, действуют те же правила. Маршрутизация звонков основывается на политике авторизации звонков, политике голосовой маршрутизации и расположении. Дополнительные сведения можно найти в разделе [Пользователи Teams помещают исходящий звонок в КТСОП](#teams-user-places-an-outbound-call-to-the-pstn). 
- Для входящих звонков по коммутируемой телефонной сети с помощью этого же правила маршрутизации, применяемые для переадресации звонков и одновременного звонка другим пользователям, также применяются к делегатам. Дополнительные сведения можно найти в разделе [передача или переадресация звонков другим пользователям Teams](#teams-user-transfers-or-forwards-call-to-another-teams-user), [команды пользователи Teams перемещаются или пересылают звонки в конечную точку PSTN](#teams-user-transfers-or-forwards-call-to-pstn-endpoint)и [одновременные звонки](#simultaneous-ringing). Когда представитель задает конечную точку PSTN в качестве одновременных абонентов, для направления звонка в КТСОП используется политика маршрутизации голосовой связи делегата. 
- Для делегирования рекомендуется, чтобы представитель и связанные с ним делегаты находились на одном сайте сети. 

## <a name="other-planning-considerations"></a>Другие факторы, которые следует учесть при планировании

### <a name="changes-from-an-on-premises-location-based-routing-deployment"></a>Изменения локального развертывания маршрутизации на основе местоположения

Сетевая политика маршрутизации голосовых сообщений больше не используется. Вместо этого мы используем политику голосовой маршрутизации пользователя. Это означает, что для перемещения пользователей на другие сайты Политика маршрутизации голосовой связи должна включать шлюзы перемещаемых сайтов. 

### <a name="technical-considerations-for-location-based-routing"></a>Технические рекомендации для маршрутизации на основе расположения

Поддерживаются подсети IPv4 и IPv6, однако IPv6 имеет приоритет при поиске соответствия.

### <a name="client-support-for-location-based-routing"></a>Поддержка маршрутизации с учетом расположения в клиенте

Поддерживаются следующие клиенты teams:
- Настольные клиенты Teams (Windows и Mac)
- Клиенты Teams для мобильных устройств (iOS и Android)
- IP-телефоны Teams

Веб-клиент Teams и клиенты Skype для бизнеса не поддерживаются.

### <a name="capabilities-not-supported-by-location-based-routing"></a>Функции, не поддерживаемые маршрутизацией на основе расположения

Маршрутизация на основе местоположения не распространяется на следующие типы взаимодействий. Маршрутизация на основе местоположения не применяется при взаимодействии конечных точек Teams с конечными точками PSTN в следующих сценариях: 
- Парковка вызовов или преобразование вызовов ТСОП с помощью функции парковки вызовов 
- Локальный пользователь Skype для бизнеса или пользователь Skype для бизнеса Online вызывает пользователя Teams.  

### <a name="location-based-routing-for-conferencing"></a>Маршрутизация на основе местоположения для конференций

Пользователь с поддержкой маршрутизации на основе местоположения, входящий в КОММУТИРУЕМый звонок, не может начать конференцию с другим пользователем или PSTN-телефоном. Подключение к автосекретарям и очередям звонков разрешено. Если у пользователя есть лицензия на Конференц-связь, пользователь должен начать конференцию с соответствующими пользователями и позвонить через мост конференции, чтобы начать конференцию.
