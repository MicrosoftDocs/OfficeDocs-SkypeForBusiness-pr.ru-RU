# <a name="plan-location-based-routing-for-direct-routing"></a>Планирование маршрутизации на основе расположения для прямой маршрутизации

## <a name="overview-of-location-based-routing"></a>Обзор Location-Based маршрутов

В некоторых странах и регионах обходить поставщика услуг телефонной сети общего звонков (STN) для снижения затрат на междугородние звонки является неправомерным. В этой статье описано, как Location-Based маршруты для ограничения обхода платных Microsoft Teams пользователей в зависимости от их географического положения. Эта статья относится только к телефонная система прямой маршрутии.

Здесь вы найдете обзор Location-Based маршрутов и инструкций, которые помогут вам спланировать их. Когда вы будете готовы применить и включить Location-Based маршруты, см.:

- [Развертывание параметров сети для Location-Based маршрутов](location-based-routing-configure-network-settings.md)
- [Включение функции "Маршрутизация на основе расположения" для прямой маршрутизации](location-based-routing-enable.md)

> [!NOTE]
> Location-Based маршруты недоступны в Microsoft 365 облако сообщества для государственных организаций (GCC) high или DoD.

Location-Based маршруты — это функция, которая позволяет ограничить обход платных номеров на основе политики и географического расположения пользователя во время входящий или исходящие вызовы по ПСС. Location-Based маршруты предназначены для предотвращения обхода платных номеров. Его не следует использовать в качестве механизма динамической маршрутиации звонков по ННР в зависимости от расположения пользователя или непредвиденных последствий.

Если Teams включена Location-Based маршрутии, применяется следующее:

- Чтобы сделать исходящие вызовы через ОКП, необходимо сделать следующее:
    - Конечная точка пользователя находится на сетевом сайте, который включен для Location-Based маршрутов и вызывает выход через соответствующий шлюз, для Location-Based маршрут. 
    - Конечная точка пользователя находится на сетевом сайте, который не включен для Location-Based маршрутиза и вызывает выход через шлюз, для Location-Based маршруты не Location-Based вход.

    В других случаях исходящие звонки запрещены.

- Для получения входящий звонок через ПСПУ конечная точка ответа пользователя должна быть расположена на том же сетевом сайте, где звонок проходит через шлюз, для Location-Based маршрутиза. В любом другом сценарии, например при роуминге пользователя, звонок не разрешен и перенаночен в параметры его переадминизора (как правило, голосовая почта).
- Чтобы перевести звонок через ОКП на Teams другому пользователю, конечная точка целевого пользователя должна быть расположена на том же сетевом сайте, что и пользователь, инициирует передачу. Передача не разрешена ни в каких других сценариях. 
- Чтобы перевести другого Teams пользователя на ПСПС, звонок должен быть передан через Location-Based, расположенный на том же сетевом сайте, что и первоначальный пользователь. Передача не разрешена ни в каких других сценариях.

Location-Based маршрутизации используются те же определения сети, сайта и подсети, Skype для бизнеса Server используются. Если для расположения ограничен обход платных номеров, администратор связывает каждую ip-подсеть и каждый шлюз ТСТС для этого расположения с сетевым сайтом. Расположение пользователя определяется подсетью IP Teams к Teams в момент звонка по ДНР. У пользователя может быть несколько Teams, расположенных на разных сайтах. В этом случае Location-Based маршруты будут выполняться отдельно в зависимости от расположения конечной точки. 

Чтобы ознакомиться с некоторыми сетевыми терминами, используемыми в этой статье, см. статью Параметры сети для облачных функций голосовой связи в [Teams.](cloud-voice-network-settings.md)

## <a name="apply-location-based-routing"></a>Применение Location-Based маршрутов

Необходимо применить Location-Based маршрутику к пользователям, сетевым сайтам и шлюзам ННП.  

### <a name="apply-location-based-routing-at-the-user-location"></a>Применение Location-Based маршрутов по расположению пользователя

Как упоминалось выше, Location-Based маршруты применяются только к пользователям, для которых настроена прямая маршрутная маршрутия. Location-Based маршруты не применяются к пользователям, для которых настроен план звонков. Если для них действуют ограничения на обход платных номеров, для пользователей должна быть включена Location-Based, которая управляет условиями, при которых они могут звонить и принимать звонки по ДНР, а также шлюз ПСN, который можно использовать. Если пользователь с включенной маршрутией Location-Based находится на сайте с включенной маршрутией Location-Based, он должен звонить через Location-Based шлюз с включенной маршрутией, подключенный к сайту. 

Location-Based маршруты определяет текущее расположение пользователя на основе IP-адреса его Teams конечной точки и соответствующим образом применяет правила. Расположение пользователя, у которого включена Location-Based маршрутизации, можно классифицизировать следующим образом: 
- **Пользователь находится на том же сайте Location-Based, связанном с шлюзом STN, где ему назначено did.**<br>В этом сценарии пользователь находится на известном сетевом сайте, на который включена маршрутная маршрутия Location-Based, а на шлюзе, который находится на том же сетевом сайте, заканчивается его номер прямого телефонного номера (DID). Например, пользователь находится в офисе. 
- **Пользователь находится на другом сайте Location-Based с включенной маршрутией, не связанном со шлюзом ННП, где ему назначено did.**<br>В этом сценарии пользователь находится на известном сетевом сайте, на котором включена маршрутная Location-Based, и этот сайт не связан с шлюзом ННР, где назначен номер DID пользователя. Например, пользователь путешествует в другой офис.  
- **Пользователь находится на внутреннем сайте, на который не включена Location-Based маршрут.** <br>В этом сценарии пользователь находится на известном внутреннем сетевом сайте, который не Location-Based маршрутии. 
- **Пользователь находится на неизвестном сайте.** 
    - Пользователь находится во внутренней сети, которая не является сетевым сайтом. 
    - Пользователь находится за пределами внутренней сети. Например, пользователь находится в Интернете дома или в кафе. 

### <a name="apply-location-based-routing-at-the-network-site"></a>Применение Location-Based маршрутов на сайте сети 
Сетевые сайты должны быть включены для Location-Based маршрутов, чтобы определить, какие шлюзы нужно Location-Based маршрутии пользователей при роуминге. Если пользователь с включенной маршрутией Location-Based перемещается на сайт, для которого включена маршрутная маршрутка Location-Based, для исходяющих звонков можно использовать только шлюз ННР, для которого включена маршрутная маршрутия Location-Based на этом сайте. Если пользователь с включенной Location-Based маршрутизов перемещается на сайт, для которого не включена маршрутная маршрутка Location-Based, для исходящие вызовы можно использовать любой шлюз, не включенный для Location-Based маршрутизов.  

### <a name="apply-location-based-routing-at-the-pstn-gateway"></a>Применение Location-Based маршрутов к шлюзу ДНР 

Шлюзы связываются с сайтами, чтобы определить, где может быть Location-Based, когда он звонит или принимает звонок по ДНР. Шлюзы должны быть включены для Location-Based маршрутов, чтобы убедиться, что они находятся в ограничениях на обход платных номеров и не могут использоваться пользователями, у которых нет Location-Based маршрут. Один и тот же шлюз может быть связан с несколькими сайтами, и его можно включить для Location-Based маршрутов или Location-Based для него (в зависимости от сайта).

При переададанте звонка пользователь, который инициировал звонок, не должен отвечать на первоначальный звонок. Это [означает,](#teams-user-receives-an-inbound-call-from-the-pstn) что звонки можно переадпорять даже в том случае, если пользователь1 не находится в расположении для получения входящие звонков (см. таблицу в разделе Teams пользователь получает входящий звонок из раздела ПСДН), а звонки не могут быть переданы, если пользователь1 не может принимать входящий звонок. 

#### <a name="user-not-enabled-for-location-based-routing"></a>Пользователь не включен для Location-Based маршрутов

Пользователь, у которого не включена Location-Based маршрутизов, может переадпорять или переадпорять звонки по ПСС другим пользователям, у которых Location-Based маршрут. Как правилоLocation-Based пользователю не разрешается переадпоряторам звонков по ПСС, так как Location-Based пользователям с включенной маршрутией разрешается совместное местонахождение только в шлюзах Location-Based, в которых включена маршрутная маршрутия для звонков по ЗВОНКОВ по ДНР. Исключением является Location-Based, когда пользователь с включенной маршрутией перемещается на сайт, для Location-Based маршрутинга. В этом случае перенаправленный звонок разрешен.  

Кроме того, пользователь, у которого не включена маршрутная Location-Based маршрутизов, может получать звонок только от другого пользователя, у которого не включена Location-Based маршрутизов.

### <a name="teams-user-transfers-or-forwards-call-to-pstn-endpoint"></a>Teams переадпора через конечную точку ПСС

#### <a name="user-not-enabled-for-location-based-routing"></a>Пользователь не включен для Location-Based маршрутов

- Разрешается переадпорять и переадпорять звонок через ПСПС на другой номер. 
- При передаче и переадваровке входящий вызов VOIP в ДНР необходимо соблюдать ограничения платного обхода для вызывающего.

## <a name="other-planning-considerations"></a>Другие факторы, которые следует учесть при планировании

### <a name="changes-from-an-on-premises-location-based-routing-deployment"></a>Изменения в локальном развертывании Location-Based маршрутии

Политика маршрутинга голосовой связи на сетевом сайте больше не используется. Вместо этого мы используем политику голосовой маршрутии пользователя. Это означает, что для того, чтобы разрешить пользователям перемещаться на другие сайты, политика маршрутинга голосовой связи должна включать шлюзы перемещаемого сайта. 

### <a name="technical-considerations-for-location-based-routing"></a>Технические рекомендации для маршрутизации на основе расположения

Подсети IPv4 и IPv6 поддерживаются, но IPv6 имеет приоритет при проверке совпадения.

### <a name="client-support-for-location-based-routing"></a>Поддержка клиентов для Location-Based маршрутов

Поддерживаются Teams следующие клиенты:
- Teams классических клиентов (Windows и Mac)
- Teams мобильных клиентов (iOS и Android)
- Teams IP-телефоны

Веб-Teams и Skype для бизнеса не поддерживаются.

### <a name="capabilities-not-supported-by-location-based-routing"></a>Функции, не поддерживаемые маршрутизацией на основе расположения

Location-Based маршруты не применяются к следующим типам взаимодействия. Location-Based маршруты не применяются, Teams конечные точки взаимодействуют с конечными точками ННР в следующих сценариях: 
- Парковка вызовов или преобразование вызовов ТСОП с помощью функции парковки вызовов 
- Пользователь локальной Skype для бизнеса или Skype для бизнеса Online звонит Teams пользователю  

### <a name="location-based-routing-for-conferencing"></a>Location-Based маршруты для conferencing

Пользователю Location-Based, включенной маршрутизовой во время звонка по ННР, не разрешено начинать конференцию с другим пользователем или номером ДНР. Разрешено подключение к автоотводщикам или очередям вызовов. Если у пользователя есть лицензия на конференц-связи, он должен начать конференцию с соответствующими пользователями и позвонить по STN через мост конференции, чтобы начать конференцию.  

### <a name="media-bypass-requirement-for-location-based-routing"></a>Требования об обходе мультимедиа для Location-Based маршрутии

При развертывании Location-Based маршрутов в Индии необходимо также настроить обход мультимедиа. Дополнительные см. в дополнительных информациях В планах обхода мультимедиа с помощью [прямой маршрутизации](direct-routing-plan-media-bypass.md) и Оптимизация локальных мультимедиа [для прямой маршрутизации.](direct-routing-media-optimization.md)

### <a name="direct-voice-over-ip-voip"></a>Прямая голосовая связь по IP-адресу (VoIP)

Прямая голосовая связь по IP-адресу (VoIP) не должна быть развернута с телефонным оборудованием в Индии.
