# <a name="plan-location-based-routing-for-direct-routing"></a>Планирование зависимостью от расположения маршрутизации для непосредственного

## <a name="overview-of-location-based-routing"></a>Общие сведения о маршрутизации на основе расположения

В некоторых странах и регионах недопустимо пропускать поставщика общедоступной переключения телефонной сети общего пользования (PSTN) чтобы снизить расходы на междугородние и международные вызывающего. В этой статье описывается использование маршрутизации на основе расположения для ограничения международную сервера-посредника для групп Майкрософт пользователей на основе географического расположения. Эта статья относится только к телефонной системе прямой маршрутизации.

Здесь вы получите Обзор маршрутизации на основе расположения и руководство для помощи при планировании для него. Когда вы будете готовы для применения и Включение маршрутизации на основе местоположения, см.
- [Развертывание параметров сети для маршрутизации на основе расположения](location-based-routing-configure-network-settings.md)
- [Включение зависимостью от расположения маршрутизации для непосредственного](location-based-routing-enable.md)

Маршрутизация на основе расположения — функция, которая позволяет ограничить международную сервера-посредника на основе политики и географического расположения пользователя во время входящих или исходящих вызовов ТСОП. 

Если группам пользователей включена для маршрутизации на основе местоположения, применяются следующие условия:
- Чтобы сделать исходящих вызовов PSTN, должны выполняться одно из следующих:
    - Конечная точка пользователя находится в области сети, поддерживающей зависимостью от расположения маршрутизации и вызовов исходящего трафика через соответствующий шлюз, который включен для маршрутизации на основе местоположения. 
    - Конечная точка пользователя находится в области сети, которая не включен для маршрутизации на основе расположения и вызывает исходящего трафика через шлюз, который не включен для маршрутизации на основе местоположения.

    Исходящие вызовы не разрешается в других сценариях.

- Для получения входящих звонков ТСОП, ответа на конечной точки пользователя должен находиться в том же сайте сети где ingresses звонок через шлюз, который включен для маршрутизации на основе местоположения. В других сценариях например, если пользователь перемещения, вызов не является допустимым и перенаправляется в настройках (как правило, голосовая почта) переадресации звонков пользователя.
- Переключение звонка PSTN другому пользователю групп, целевой конечной точки пользователя должна размещаться на том же сайте сети, как и пользователь, инициирующий переключения. Передача не разрешается в других сценариях. 
- Чтобы переключить другим группам пользователей в ТСОП, необходимо перенести вызова через шлюз маршрутизации на основе расположения включены, расположенный на том же сайте сети как начальное вызывающего абонента. Передача не разрешается в других сценариях.

Маршрутизация на основе местоположения использует же сетевой регион, сайтов и подсетей определения, которые использует Скайп для Business Server. При ограниченных расположения сервера-посредника международную администратор связывает каждого IP-подсеть и каждого шлюза ТСОП для данного расположения с сетевым узлом. Расположение пользователя, определяется IP-подсеть, конечные точки группы пользователя, подключенные к во время звонка PSTN. Пользователь может иметь несколько групп клиентов, расположенных в разных местах, в противном случае принудительного применения маршрутизации на основе расположения каждого клиента отдельно маршрутизации в зависимости от расположения конечной точки. 

Ознакомиться с некоторыми сети термины, используемые в этой статье, просмотрите [терминология маршрутизации на основе местоположения](location-based-routing-terminology.md).

## <a name="apply-location-based-routing"></a>Применение маршрутизация на основе расположения

Необходимо применить маршрутизации на основе местоположения для пользователей, сетевые узлы и шлюзов ТСОП.  

### <a name="apply-location-based-routing-at-the-user-location"></a>Применение маршрутизации на основе расположения в расположении пользователя

Как уже было сказано ранее, маршрутизации на основе местоположения применяется только для пользователей, которые настроены для прямой маршрутизации. Маршрутизация на основе расположения не применяется для пользователей, которые настроены для вызова планирование. Пользователей должна быть включена для маршрутизации на основе расположения, если они находятся в разделе ограничения сервера-посредника международную, который управляет условия, в которых они выполнение и прием звонков ТСОП и шлюз ТСОП, который может использоваться. Когда пользователь, который включен для маршрутизации на основе местоположения находится на сайте, который включен для маршрутизации на основе местоположения, пользователь должен выполнять вызовы через шлюз маршрутизации на основе расположения включены, подключенные к сайту. 

Маршрутизация на основе местоположения работает путем определения текущее расположение пользователя на основании IP-адрес конечной точки для группы пользователей и применяет правила соответствующим образом. Расположение пользователь, который включен для маршрутизации на основе расположения можно классифицировать следующим образом: 
- **Пользователь находится на том же маршрутизации на основе расположения включены сайте связанный со шлюзом ТСОП которой назначается их DID.**<br>В этом сценарии пользователь находится в известных сетевым узлом, который включен для маршрутизации на основе расположения и пользователя прямого входящего вызова по (DID) номер завершает на шлюз ТСОП, который находится в том же сайте сети. Например пользователь является их офиса. 
- **Пользователь находится на другом маршрутизации на основе расположения включены сайте не связанный со шлюзом ТСОП которой назначается их DID.**<br>В этом сценарии пользователь находится в известных сетевым узлом, который включен для маршрутизации на основе расположения и этого сайта, не сопоставленный с шлюзом ТСОП которой назначен номер DID пользователя. Например он перемещается на другой.  
- **Пользователь находится на внутренний узел, который не включен для маршрутизации на основе местоположения.** <br>В этом сценарии пользователь находится в известных внутреннего сетевого узла, который не включен для маршрутизации на основе местоположения. 
- **Пользователь находится в неизвестный сайта.** 
    - Пользователь находится в пределах внутренней сети, который не определен как с сетевым узлом. 
    - Пользователь находится за пределами внутренней сети. Например пользователь находится в Интернете дома или в кафе. 

### <a name="apply-location-based-routing-at-the-network-site"></a>Применение маршрутизации на основе расположения на узле сети 
Сетевые узлы должен быть включен для маршрутизации на основе местоположения можно определить, какие шлюзов для маршрутизации маршрутизации на основе расположения поддержкой пользователей после перемещения. Если пользователь, который включен для маршрутизации на основе местоположения перемещается на сайт, который включен для маршрутизации на основе местоположения, шлюз ТСОП, который включен для маршрутизации на основе расположения этого сайта можно использовать для исходящих звонков. Если пользователь, который включен для маршрутизации на основе местоположения перемещается на сайт, который не включен для маршрутизации на основе местоположения, любой шлюза, который не включен для маршрутизации на основе расположения можно использовать для исходящих звонков.  

### <a name="apply-location-based-routing-at-the-pstn-gateway"></a>Применение зависимостью от расположения маршрутизации на шлюз ТСОП 

Шлюзы связаны сайты, чтобы определить, где пользователь, который включен для маршрутизации на основе расположения можно найти, когда они сделать или получить вызов ТСОП. Шлюзы должен быть включен для маршрутизации на основе расположения для убедитесь, что он в разделе ограничения сервера-посредника международную и не может использоваться с пользователями, не включенных в маршрутизации на основе местоположения. Же шлюз может быть сопоставлен с нескольких сайтов и настройки необходимо включить для маршрутизации на основе расположения или не включен для маршрутизации на основе расположения, в зависимости от сайта.

Пользователь, который не включен для маршрутизации на основе расположения можно выполнять исходящие вызовы с помощью любого шлюза на любой сайт, который не включен для маршрутизации на основе местоположения через политику маршрутизации назначенный голосовой связи. Если пользователь переходит на сайт, который включен для маршрутизации на основе местоположения, их можно выполнять только вызовы через обычный маршрутизации шлюзов, которые не включены для маршрутизации на основе местоположения.

### <a name="teams-user-transfers-or-forwards-call-to-another-teams-user"></a>Группы пользователей передает или перенаправляет звонок на другого пользователя группы

Для конечной точки PSTN, зависимостью от расположения маршрутизации служит для анализа одного или обоих пользователей включены ли для маршрутизации на основе расположения и определяет перенесены или перенаправлять в зависимости от расположения обе конечные точки вызова. 
 
Перевод звонка требует, чтобы принять звонок во время переадресации не требует первого вызова ответа на вызывающей стороны пользователь. Это означает, что можно перенаправлять неотвеченные звонки, даже в том случае, если пользователь User1 не в расположение для получения входящих звонков (см в разделе [группы пользователь получает входящий вызов из ТСОП](#teams-user-receives-an-inbound-call-from-the-pstn) ) и вызовы не могут быть перенесены User1 не может принять входящий звонок. 

#### <a name="user-not-enabled-for-location-based-routing"></a>Пользователь не включен для маршрутизации на основе расположения

Пользователь, который не включен для маршрутизации на основе расположения можно перенести или вызовы прямого PSTN другим пользователям, не включенных в маршрутизации на основе местоположения. Пользователь обычно не разрешены для передачи или переадресовывать звонок через ТСОП пользователю, который включен для маршрутизации на основе местоположения, так как пользователей с включенной поддержкой маршрутизации на основе местоположения обычно только могут размещаться совместно на шлюзах маршрутизации на основе расположения включены для ТСОП звонки. Исключение при включении маршрутизации на основе расположения перемещается на сайт, который не включен для маршрутизации на основе местоположения. В этом сценарии перенесенной разрешено.  

Аналогичным образом пользователь, который не включен для маршрутизации на основе расположения может получать только перемещение или переадресацию звонков ТСОП из другого пользователя, который не включен для маршрутизации на основе местоположения.

### <a name="delegation"></a>Делегирование

Группы пользователей может выбрать делегатов, которые можно сделать и принимать звонки от его имени. Возможности делегирования в группах, следующим образом влияют на основе расположения маршрутизации: 
- Для исходящих звонков из делегата маршрутизации на основе расположения включены от имени сотрудника применяются те же правила. Маршрутизация звонков основано на политику авторизации вызовов делегата, политику маршрутизации голосовой связи и расположение. Для получения дополнительных сведений см [группы пользователь совершает исходящие вызовы в ТСОП](#teams-user-places-an-outbound-call-to-the-pstn). 
- Для входящих звонков ТСОП сотрудника переадресация звонков те же правила маршрутизации на основе расположения, которые применяются для или одновременный вызов другим пользователям, также применимы к делегатам. Для получения дополнительных сведений см [группами пользователей передает или перенаправляет звонок на другого пользователя группам](#teams-user-transfers-or-forwards-call-to-another-teams-user), [группам пользователей передает или перенаправляет вызов к конечной точке ТСОП](#teams-user-transfers-or-forwards-call-to-pstn-endpoint)и [Simultaneous звонков](#simultaneous-ringing). Если делегат устанавливает конечной точки ТСОП в качестве целевого одновременных звонков, политику маршрутизации голосовых вызовов делегата используется для маршрутизации вызовов в ТСОП. 
- Для делегирования рекомендуется представитель и связанные делегаты находиться на том же сайте сети. 

## <a name="other-planning-considerations"></a>Другие факторы, которые следует учесть при планировании

### <a name="changes-from-an-on-premises-location-based-routing-deployment"></a>Изменения из локальной маршрутизации на основе расположения развертывания

Политики маршрутизации голосовой связи сайтов сети больше не используется. Вместо этого мы используем политики маршрутизации голосовых данных пользователя. Это значит, чтобы разрешить пользователям перемещаться на другие сайты, политику маршрутизации голосовой связи необходимо включить шлюзы roamed сайтов. 

### <a name="technical-considerations-for-location-based-routing"></a>Технические рекомендации для маршрутизации на основе расположения

Поддерживаемые подсетей IPv4 и IPv6, однако IPv6 приоритет при проверке соответствия. Поддержка IPv6 в настоящее время находится в стадии разработки и будет доступна с общие доступности Предоставлена для маршрутизации на основе местоположения. 

### <a name="client-support-for-location-based-routing"></a>Поддержка клиентов для маршрутизации на основе расположения

Поддерживаются следующие клиенты групп:
- Команды настольных клиентов (Windows и Mac)
- Группы мобильных клиентов (операций ввода-вывода и Android (en))
- Команды IP-телефонов

Скайп пользователей не поддерживаются.

### <a name="capabilities-not-supported-by-location-based-routing"></a>Функции, не поддерживаемые маршрутизацией на основе расположения

Маршрутизация на основе расположения не применяется к следующих типов взаимодействия. Маршрутизация на основе расположения не применяется, если значение группы конечных точек взаимодействия с конечными точками ТСОП в следующих сценариях: 
- Парковка вызовов или преобразование вызовов ТСОП с помощью функции парковки вызовов 
- Локальный Скайп для бизнес-пользователя или Скайп для бизнеса в Интернет пользователя вызывает групп пользователей  

### <a name="location-based-routing-for-conferencing"></a>Маршрутизация конференц-связи на основе расположения

Не может начать конференцию с другим пользователем маршрутизации на основе расположения включен на звонок через ТСОП пользователь или номер PSTN. Разрешено подключение к автосекретари или звонок очереди. Если у пользователя есть лицензии конференц-связи, пользователь должен начать конференцию с соответствующим пользователям и вызовов ТСОП через мост конференции, чтобы начать конференцию.
