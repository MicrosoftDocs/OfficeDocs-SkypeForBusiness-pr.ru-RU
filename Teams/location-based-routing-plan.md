# <a name="plan-location-based-routing-for-direct-routing"></a>Планирование маршрутизации на основе расположения для прямой маршрутизации

## <a name="overview-of-location-based-routing"></a>Обзор Location-Based маршрутов

В некоторых странах и регионах в обход поставщика услуг телефонной сети общего звонков (STN) для снижения затрат на междугородние звонки запрещено. В этой статье описано, как использовать Location-Based маршруты для ограничения платного обхода для пользователей Microsoft Teams в зависимости от их географического положения. Эта статья относится только к прямой маршрутике телефонной системы.

Здесь вы найдете общие сведения о Location-Based маршруты и инструкции по планированию. Когда вы будете готовы применить и включить Location-Based маршрутов, см. статью:

- [Развертывание параметров сети для Location-Based маршрутов](location-based-routing-configure-network-settings.md)
- [Включение функции "Маршрутизация на основе расположения" для прямой маршрутизации](location-based-routing-enable.md)

> [!NOTE]
> Location-Based маршруты недоступны в развертываниях Microsoft 365 Government Community Cloud (GCC) High или DoD.

Location-Based маршруты — это функция, которая позволяет ограничивать платные обходы на основе политики и географического расположения пользователя во время входящий или исходящие вызовы по ПС. Location-Based маршруты обеспечивают механизм предотвращения обхода платных номеров. Ее не следует использовать в качестве механизма для динамической маршрутации звонков по ДНР в зависимости от расположения пользователя или непредвиденных последствий.

Если для пользователя Teams включена Location-Based маршрутов, применяется следующее:

- Чтобы сделать исходящие вызовы по ННР, должно быть истинно одно из следующих:
    - Конечная точка пользователя находится на сетевом сайте, для Location-Based маршрутинга и вызова выходного шлюза через соответствующий шлюз, для Location-Based маршрутинга. 
    - Конечная точка пользователя находится на сетевом сайте, для Location-Based маршрутинга и вызова выходного шлюза, для Location-Based маршрутинга не включена.

    Исходящие звонки не разрешены ни в каких других сценариях.

- Чтобы принимать входящий звонок по ПС, конечная точка ответа пользователя должна быть расположена на том же сетевом сайте, где звонок проходит через шлюз, для Location-Based маршрутинга. В любом другом сценарии, например если пользователь находится в роуминге, звонок не разрешен и перенаночен в параметры переадминизовки (как правило, в голосовую почту).
- Чтобы перенести звонок по ННР другому пользователю Teams, конечная точка целевого пользователя должна быть расположена на том же сетевом сайте, что и пользователь, инициирует передачу. Передача не разрешена ни в каких других сценариях. 
- Чтобы перевести другого пользователя Teams на ПС, звонок должен быть передан через Location-Based шлюз с поддержкой маршрутинга, расположенный на том же сетевом сайте, что и первоначальный звонок. Передача не разрешена ни в каких других сценариях.

Location-Based маршрутизации используются те же определения сетевого региона, сайта и подсети, что и в Skype для бизнеса Server. Если в местоположении ограничен обход платных номеров, администратор связывает каждую IP-подсеть и каждый шлюз ТСТС с сетевым сайтом. Расположение пользователя определяется IP-подсетей, к которую подключены его конечные точки Teams во время звонка по ДНР. У пользователя может быть несколько клиентов Teams на разных сайтах. В этом случае Location-Based маршрутия будет применяться к каждому клиенту отдельно в зависимости от расположения конечной точки. 

Чтобы ознакомиться с некоторыми сетевыми терминами, используемыми в этой статье, см. статью "Параметры сети" для функций облачного голосового [связи в Teams.](cloud-voice-network-settings.md)

## <a name="apply-location-based-routing"></a>Применение Location-Based маршрутов

Необходимо применять Location-Based маршрутику к пользователям, сетевым сайтам и шлюзам ННР.  

### <a name="apply-location-based-routing-at-the-user-location"></a>Применение Location-Based маршрутов к расположению пользователя

Как упоминалось выше, Location-Based маршруты применяются только к пользователям, которые настроены на прямую маршрутику. Location-Based маршруты не применяются к пользователям, для которых настроен план звонков. В случае настроив Location-Based обхода, пользователи должны иметь доступ к Location-Based, которая контролирует условия, при которых они могут звонить и принимать вызовы по ННР, а также шлюз ННР. Если пользователь с включенной Location-Based маршрутизов находится на сайте, для которого включена маршрутная маршрутка Location-Based, он должен звонить через шлюз Location-Based, подключенный к сайту. 

Location-Based Routing работает путем определения текущего расположения пользователя на основе IP-адреса конечной точки Teams пользователя и соответствующим образом применяет правила. Расположение пользователя, для которого включена Location-Based маршрутизации, можно классифицизировать следующим образом: 
- **Пользователь находится на том же сайте Location-Based, связанном с шлюзом STN, где ему назначена надстройка DID.**<br>В этом сценарии пользователь находится на известном сетевом сайте, на который включена маршрутная Location-Based, а его номер для прямого набора номера "внутрь" (DID) прекращается на шлюзе ННР, который находится на том же сетевом сайте. Например, пользователь находится в офисе. 
- **Пользователь находится на другом сайте с поддержкой Location-Based, не связанном с шлюзом ННР, где ему назначена надстройка DID.**<br>В этом сценарии пользователь находится на известном сетевом сайте, на котором включена маршрутная Location-Based, и этот сайт не связан с шлюзом ННР, где назначен номер DID пользователя. Например, пользователь путешествует в другой офис.  
- **Пользователь находится на внутреннем сайте, на который не включена Location-Based маршрутов.** <br>В этом сценарии пользователь находится на известном внутреннем сетевом сайте, для Location-Based маршрутинга не включена. 
- **Пользователь находится на неизвестном сайте.** 
    - Пользователь находится во внутренней сети, которая не определена как сетевой сайт. 
    - Пользователь находится за пределами внутренней сети. Например, пользователь находится в Интернете дома или в кафе. 

### <a name="apply-location-based-routing-at-the-network-site"></a>Применение Location-Based маршрутов на сетевом сайте 
Сетевые сайты должны быть включены для Location-Based маршрутов, чтобы определить шлюзы для маршрутов Location-Based пользователей с включенной маршрутии во время роуминга. Если пользователь, для которого включена Location-Based маршрутия, перемещается на сайт, для которого включена маршрутная маршрутка Location-Based, для исходящие вызовы можно использовать только шлюз ННР, для которого включена маршрутная маршрутия Location-Based на этом сайте. Если пользователь, для которого включена Location-Based маршрутия, перемещается на сайт, для которого не включена маршрутная маршрутка Location-Based, для исходящие вызовы можно использовать шлюз, для которого не включена Location-Based маршрутия.  

### <a name="apply-location-based-routing-at-the-pstn-gateway"></a>Применение Location-Based маршрутов к шлюзу ННР 

Шлюзы связываются с сайтами, чтобы определить, где может быть расположен пользователь, для которого включена маршрутная Location-Based при звонке по ННР. Для маршрутной Location-Based должны быть включены шлюзы, чтобы убедиться, что они находятся в платных обходных ограничениях и не могут использоваться пользователями, у которых не включена Location-Based маршрут. Один и тот же шлюз может быть связан с несколькими сайтами, и его можно включить Location-Based для Location-Based, в зависимости от сайта.

### <a name="teams-user-transfers-or-forwards-call-to-another-teams-user"></a>Пользователь Teams передает или перенаадваровет звонок другому пользователю Teams

При вовлечении конечной точки ННР Location-Based маршрутизирует, включена ли маршрутка Location-Based для одного или обоих пользователей, и определяет, следует ли перенаправить звонок, в зависимости от расположения обеих конечных точек. 
 
При переададанте звонка пользователь, инициющий вызов, не должен отвечать на первоначальный звонок. Это означает, что звонки можно переадпорять, даже если пользователь1 не находится в этом расположении для получения входящий звонок (см. таблицу в разделе "Пользователи [Teams"](#teams-user-receives-an-inbound-call-from-the-pstn) получает входящий звонок из раздела ПС), а переадпорять звонки невозможно, если пользователь1 не может принять входящий звонок. 

#### <a name="user-not-enabled-for-location-based-routing"></a>Пользователю не включена Location-Based маршрутов

Пользователь, у которого не включена Location-Based маршрутизов, может переносить или переадпорять звонки по ННР другим пользователям, у которых не включена Location-Based маршрут. Как правило, пользователю не разрешено переносить или переадпорять звонок по ННР пользователю, у которого включена маршрутная маршрутка Location-Based, так как пользователи, включенные маршрутизов Location-Based, обычно могут совместно расположению только в шлюзах Location-Based, включенных маршрутизовом, для звонков по ОКП. Исключением является то, Location-Based пользователя с включенной маршрутией перемещается на сайт, для Location-Based маршрутинга не включена. В этом случае перенаправленный звонок разрешен.  

Кроме того, пользователь, у которого не включена Location-Based маршрутия, может принимать только переадренный или перенастройный звонок по ННР от другого пользователя, у которого не включена Location-Based маршрутизов. 

#### <a name="user-enabled-for-location-based-routing"></a>Включена маршрутная Location-Based пользователей

Как правило, переключение и переадваровка входящие вызовы через ПС из шлюза, для Location-Based маршрутизов разрешена только в том случае, если целевой пользователь включен для маршрутной Location-Based и находится на том же сайте. В противном случае переадваровка звонков будет не разрешена.

### <a name="teams-user-transfers-or-forwards-call-to-pstn-endpoint"></a>Пользователи Teams перена могут переадпорять вызовы в конечную точку ННР

#### <a name="user-not-enabled-for-location-based-routing"></a>Пользователю не включена Location-Based маршрутов

- Разрешается переадпорять звонок по ННР на другой номер. 
- Переадпоряция входящий вызов VOIP на телефонную телефонную службу должна соблюдать ограничения платного обхода вызывающего. 
    - Если для вызываемой службы не включена Location-Based маршрутизов, их можно перенести на любой шлюз ННР, для Location-Based перенаправления.
    - Если для вызываемой Location-Based маршрутов включена маршрутная маршрутия, их можно перенести только на Location-Based шлюз с поддержкой маршрутинга, расположенный на том же сетевом сайте. 

#### <a name="user-enabled-for-location-based-routing"></a>Включена маршрутная Location-Based пользователей

- Переключение и переададратка входящий звонок по ДНР на другой номер ДНР должны быть маршрутизовы для того же шлюза Location-Based, в который был входящий звонок. 
- Переадпоряция входящий вызов VOIP на телефонную телефонную службу должна входить в учет как вызывающего, так и вызываемого обзвонного обхода.

### <a name="simultaneous-ringing"></a>Одновременный звонок

Когда пользователь, для которого включена Location-Based маршрутия, принимает звонок и одновременное звонок включен, Location-Based Маршрутия анализирует расположение вызываемой стороны и конечные точки вызываемой стороны, чтобы определить, следует ли переназначение звонка. Одновременный звонок следует за тем же Location-Based, что и при переадных звонках.

Входящие звонки по ПС из Location-Based шлюза с поддержкой маршрутизинга разрешены для подключения к автоответам или очереди звонков. Пользователи, Location-Based маршрутизов могут принимать входящие переключение звонка из этих приложений, только если они находятся на том же сайте, откуда поступает входящий звонок по ПС. 
 
Переадпорять и одновременно звонить пользователям и ДНР можно для передачи голосовых приложений. Выполнение звонка целевому объекту регламентируется правилами Location-Based, перечисленными выше.  
 
Также разрешена переадваровка голосовой почты.  

### <a name="delegation"></a>Делегирование

Пользователь Teams может выбирать делегатов, которые могут звонить и принимать звонки от их имени. На функции делегирования в Teams влияет Location-Based маршруты следующим образом: 
- Для исходящие звонки от Location-Based представителя, включенного маршрутизов, от имени представителя, применяются те же правила. Маршрутизация вызовов основана на политике авторизации вызовов делегата, политике маршрутизации голосовой маршрутизации и местоположении делегата. Дополнительные сведения см. в том, как пользователь Teams звонит по [STN.](#teams-user-places-an-outbound-call-to-the-pstn) 
- Для входящие вызовы через ОКП к делегату также применяются те же правила Location-Based маршрутации, которые применяются к переадреации звонков или одновременным звонкам для других пользователей. Дополнительные сведения см. в записях о переадрении и переадрении зовов другим пользователям [Teams,](#teams-user-transfers-or-forwards-call-to-another-teams-user)а также о переадрении вызовов на конечную точку [STN](#teams-user-transfers-or-forwards-call-to-pstn-endpoint)и одновременных [звонках.](#simultaneous-ringing) Когда представитель задает конечную точку ННР в качестве целевой для одновременного звонка, для направления вызова на ПС используется политика маршрутирования голосовой почты представителя. 
- Для делегирования рекомендуется, чтобы представитель и связанные с ним делегаты находились на одном сетевом сайте. 

## <a name="other-planning-considerations"></a>Другие факторы, которые следует учесть при планировании

### <a name="changes-from-an-on-premises-location-based-routing-deployment"></a>Изменения в локальном развертывании Location-Based маршрутов

Политика маршрутинга голосовой связи на сетевом сайте больше не используется. Вместо этого мы используем политику маршрутинга голосовой почты пользователя. Это означает, что для того чтобы разрешить пользователям перемещаться на другие сайты, политика маршрутинга голосовой связи должна включать шлюзы перемещаемых сайтов. 

### <a name="technical-considerations-for-location-based-routing"></a>Технические рекомендации для маршрутизации на основе расположения

Подсетеи IPv4 и IPv6 поддерживаются, но при проверке совпадения приоритет имеет IPv6.

### <a name="client-support-for-location-based-routing"></a>Поддержка клиентов для Location-Based маршрутов

Поддерживаются следующие клиенты Teams:
- Классические клиенты Teams (Windows и Mac)
- Мобильные клиенты Teams (iOS и Android)
- IP-телефоны Teams

Веб-клиент Teams и клиенты Skype для бизнеса не поддерживаются.

### <a name="capabilities-not-supported-by-location-based-routing"></a>Функции, не поддерживаемые маршрутизацией на основе расположения

Location-Based маршруты не применяются к следующим типам взаимодействия. Location-Based маршруты не применяются, когда конечные точки Teams взаимодействуют с конечными точками ННР в следующих сценариях: 
- Парковка вызовов или преобразование вызовов ТСОП с помощью функции парковки вызовов 
- Пользователь локальной учетной записи Skype для бизнеса или Skype для бизнеса Online звонит пользователю Teams  

### <a name="location-based-routing-for-conferencing"></a>Location-Based маршруты для conferencing

Пользователю Location-Based, включенной маршрутизовом, при звонке по ННП запрещено начинать конференцию с другим пользователем или номером ННР. Разрешено подключение к автоотводам или очередям вызовов. Если у пользователя есть лицензия на конференц-связи, он должен начать конференцию с соответствующими пользователями и позвонить по STN через мост конференции, чтобы начать конференцию.  

### <a name="media-bypass-requirement-for-location-based-routing"></a>Требования обхода мультимедиа для Location-Based маршрутии

При развертывании маршрутов Location-Based Индии необходимо также настроить обход мультимедиа. Дополнительные узнать см. в дополнительных сообщениях об обходе мультимедиа с помощью [direct Routing](direct-routing-plan-media-bypass.md) и [local Media Optimization для Direct Routing.](direct-routing-media-optimization.md)

### <a name="direct-voice-over-ip-voip"></a>Прямая голосовая связь по IP-адресу (VoIP)

Развертывание Direct Voice Over IP (VoIP) не должно быть развернуто с любым телефонным оборудованием в Индии.
