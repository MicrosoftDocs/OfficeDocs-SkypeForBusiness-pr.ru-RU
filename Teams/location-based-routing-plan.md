# <a name="plan-location-based-routing-for-direct-routing"></a>Планирование маршрутизации на основе расположения для прямой маршрутизации

## <a name="overview-of-location-based-routing"></a>Обзор Location-Based маршрутов

В некоторых странах и регионах обходить поставщика услуг телефонной сети общего звонков (STN) для снижения затрат на междугородние звонки является неправомерным. В этой статье описано, как Location-Based маршруты для ограничения обхода платных Microsoft Teams пользователей в зависимости от их географического положения. Эта статья относится только к телефонная система прямой маршрутии.

Здесь вы найдете обзор Location-Based маршрутов и инструкций, которые помогут вам спланировать их. Когда вы будете готовы применить и включить Location-Based маршруты, см.:

- [Развертывание параметров сети для Location-Based маршрутов](location-based-routing-configure-network-settings.md)
- [Включение функции "Маршрутизация на основе расположения" для прямой маршрутизации](location-based-routing-enable.md)

> [!NOTE]
> Location-Based маршруты недоступны в Microsoft 365 облако сообщества для государственных организаций (GCC) high или DoD.

Location-Based маршруты — это функция, которая позволяет ограничить обход платных номеров в зависимости от политики и географического расположения пользователя во время входящий или исходящие вызовы по ОКП. Location-Based маршруты предназначены для предотвращения обхода платных номеров. Его не следует использовать в качестве механизма динамической маршрутиации звонков по ННР с учетом расположения пользователя или непредвиденных последствий.

Если Teams включена Location-Based маршрутии, применяется следующее:

- Чтобы сделать исходящие вызовы через ОКП, необходимо сделать следующее:
    - Конечная точка пользователя находится на сетевом сайте, который включен для Location-Based маршрутов и вызывает выход через соответствующий шлюз, для Location-Based маршрут. 
    - Конечная точка пользователя находится на сетевом сайте, который не включен для Location-Based маршрутов и вызывает выход через шлюз, для Location-Based маршруты.

    В других случаях исходящие звонки запрещены.

- Для получения входящие вызовы через ПСПУ конечная точка ответа пользователя должна быть расположена на том же сетевом сайте, где звонок проходит через шлюз, для Location-Based маршрут. В любом другом сценарии, например при роуминге пользователя, звонок не разрешен и перенаночен в параметры его переадминизора (как правило, голосовая почта).
- Чтобы перевести звонок через ДНР на другого Teams, конечная точка целевого пользователя должна быть расположена на том же сетевом сайте, что и пользователь, инициирует передачу. Передача не разрешена ни в каких других сценариях. 
- Чтобы перевести другого Teams пользователя на ДНР, звонок должен быть передан через Location-Based, расположенный на том же сетевом сайте, что и первоначальный пользователь. Передача не разрешена ни в каких других сценариях.

Location-Based маршрутизации используются те же определения сети, сайта и подсети, Skype для бизнеса Server используются. Если для расположения ограничен обход платных номеров, администратор связывает каждую подсеть IP-адресов и каждый шлюз ТСТС для этого расположения с сетевым сайтом. Расположение пользователя определяется подсетью IP, к Teams к конечным точкам пользователя во время звонка по ДНР. У пользователя может быть несколько Teams, расположенных на разных сайтах, Location-Based в этом случае маршрутия каждого клиента будет применяться отдельно в зависимости от расположения конечной точки. 

Чтобы ознакомиться с некоторыми сетевыми терминами, используемыми в этой статье, см. статью Параметры сети для облачных функций голосовой связи [в Teams.](cloud-voice-network-settings.md)

## <a name="apply-location-based-routing"></a>Применение Location-Based маршрутов

Необходимо применить Location-Based маршрутику к пользователям, сетевым сайтам и шлюзам ННП.  

### <a name="apply-location-based-routing-at-the-user-location"></a>Применение Location-Based маршрутов по расположению пользователя

Как упоминалось выше, Location-Based маршруты применяются только к пользователям, для которых настроена прямая маршрутная маршрутия. Location-Based маршруты не применяются к пользователям, для которых настроен план звонков. Если для них действуют ограничения на Location-Based обхода платных номеров, пользователи должны иметь доступ к маршруту, который контролирует условия, при которых они могут звонить и принимать звонки по ПСN, а также шлюз ПСN, который можно использовать. Если пользователь с включенной маршрутией Location-Based находится на сайте, для которого включена маршрутная Location-Based, он должен звонить через Location-Based шлюз с включенной маршрутией, подключенный к сайту. 

Location-Based routing работает путем определения текущего расположения пользователя на основе IP-адреса его Teams конечной точки и соответствующим образом применяет правила. Расположение пользователя, у которого включена Location-Based маршрутизации, можно классифицизировать следующим образом: 
- **Пользователь находится на том же сайте Location-Based, связанном с шлюзом STN, где ему назначено did.**<br>В этом сценарии пользователь находится на известном сетевом сайте, на который включена маршрутная маршрутия Location-Based, а на шлюзе сети, который находится на том же сетевом сайте, заканчивается его номер прямого телефонного номера (DID). Например, пользователь находится в офисе. 
- **Пользователь находится на другом сайте Location-Based с включенной маршрутией, не связанной со шлюзом ННП, для которых назначено его имя DID.**<br>В этом сценарии пользователь находится на известном сетевом сайте, на котором включена маршрутная Location-Based, и этот сайт не связан с шлюзом ННР, где назначен номер DID пользователя. Например, пользователь путешествует в другой офис.  
- **Пользователь находится на внутреннем сайте, на который не включена Location-Based маршрут.** <br>В этом сценарии пользователь находится на известном внутреннем сетевом сайте, который не Location-Based маршрутии. 
- **Пользователь находится на неизвестном сайте.** 
    - Пользователь находится во внутренней сети, которая не является сетевым сайтом. 
    - Пользователь находится за пределами внутренней сети. Например, пользователь находится в Интернете дома или в кафе. 

### <a name="apply-location-based-routing-at-the-network-site"></a>Применение Location-Based маршрутов на сайте сети 
Сетевые сайты должны быть включены для Location-Based маршрутов, чтобы определить, какие шлюзы Location-Based маршруты пользователей во время роуминга. Если пользователь с включенной маршрутией Location-Based перемещается на сайт, для которого включена маршрутная маршрутка Location-Based, для исходяющих звонков может использоваться только шлюз ПСПС, для которого включена маршрутная маршрутия Location-Based на этом сайте. Если пользователь с включенной Location-Based маршрутизов перемещается на сайт, для которого не включена маршрутная маршрутная маршрутия Location-Based, для исходящие вызовы можно использовать все шлюзы, для которых не включена маршрутная Location-Based.  

### <a name="apply-location-based-routing-at-the-pstn-gateway"></a>Применение Location-Based маршрутов к шлюзу STN 

Шлюзы связываются с сайтами, чтобы определить, где может быть Location-Based при звонке или приеме звонка по ННП. Шлюзы должны быть включены для Location-Based маршрутов, чтобы убедиться, что они находятся в ограничениях на обход платных номеров и не могут использоваться пользователями, которые не могут использовать Location-Based маршрут. Один и тот же шлюз может быть связан с несколькими сайтами, и его можно включить для Location-Based маршрутов или Location-Based для него (в зависимости от сайта).

### <a name="teams-user-transfers-or-forwards-call-to-another-teams-user"></a>Teams или переадваровка звонка другому Teams пользователю

При вовлечении конечной точки STN Location-Based Routing анализирует, включена ли маршрутизовая Location-Based для одного или обоих пользователей, и определяет, следует ли переадпорять или переадпорять звонок в зависимости от расположения обеих конечных точек. 
 
При переададанте звонка пользователь, который инициировал звонок, не должен отвечать на первоначальный звонок. Это [означает,](#teams-user-receives-an-inbound-call-from-the-pstn) что звонки можно переадпорять даже в том случае, если пользователь1 не находится в расположении для получения входящие звонков (см. таблицу в таблице Teams пользователь получает входящий звонок из раздела ПСN), а звонки не могут быть переданы, если пользователь1 не может принимать входящий звонок. 

#### <a name="user-not-enabled-for-location-based-routing"></a>Пользователь не включен для Location-Based маршрутов

Пользователь, у которого не включена Location-Based маршрутизов, может переадпорять или переадпорять вызовы через ОКП другим пользователям, у которых Location-Based маршрут. Как правило, пользователю не разрешается передавать или переадпорять вызовы по ПСС пользователям, у которых включена маршрутия Location-Based, так как Location-Based пользователям с включенной маршрутией, как правило, разрешено совместное местонахождение только в шлюзах Location-Based, включенных маршрутизов для звонков по ЗВОНКОВ по ДНР. Исключением является Location-Based, когда пользователь с включенной маршрутией перемещается на сайт, для Location-Based маршрутов. В этом случае перенаправленный звонок разрешен.  

Кроме того, пользователь, у которого не включена маршрутная Location-Based маршрутизов, может получать только переадпоратовку или переадваровку звонка по ДНР от другого пользователя, у которого нет Location-Based маршрутизов.

Как правило, переключение и переадваровка входящие вызовы через ОКП со шлюза, который включен для Location-Based Маршруты разрешено только в том случае, если целевой пользователь включен для Location-Based Маршруты и находится на том же сайте. В противном случае переадваровка звонков будет не разрешена.

### <a name="teams-user-transfers-or-forwards-call-to-pstn-endpoint"></a>Teams переадпора через конечную точку ПСС

#### <a name="user-not-enabled-for-location-based-routing"></a>Пользователь не включен для Location-Based маршрутов

- Разрешается переадваровка звонка через ПСПС на другой номер. 
- При передаче и переадваровке входящий вызов VOIP в ДНР необходимо соблюдать ограничения платного обхода для вызывающего. 
    - Если вызывающий не может использовать Location-Based маршрутизов, его можно перенести на любой шлюз ННР, для Location-Based маршрутизов.
    - Если вызывающий звонок Location-Based маршрутизов, его можно перенести только на Location-Based, расположенный на том же сетевом сайте. 

#### <a name="user-enabled-for-location-based-routing"></a>Пользователь с включенной Location-Based маршрутии

- Переадпорядующий входящий звонок через ДНР на другой номер ПСС должен быть маршрутен из того же шлюзаLocation-Based который включен маршрутизовы, на который поступил входящий звонок. 
- Переадпоряцию входящий вызов VOIP в ДНР следует соблюдать как для вызывающего, так и для вызывающего пользователя ограничения на обход платных номеров. 
    - Если вызывающий не может использовать Location-Based маршрутизов, его можно перенести на любой шлюз ННР, для Location-Based маршрутизов.
    - Если вызывающий Location-Based маршрут, его можно перенести только на Location-Based, расположенный на том же сетевом сайте.

### <a name="simultaneous-ringing"></a>Одновременный звонок

Если пользователь с включенной маршрутиали Location-Based получает звонок и одновременно включил звонок, Location-Based Routing анализирует расположение вызываемой стороны и конечные точки вызываемой стороны, чтобы определить, следует ли переназначение звонка. Одновременный звонок следует тем же правилам Location-Based, что и Location-Based переадваровки звонка.

Входящие звонки через ПСОП с Location-Based шлюза с включенной маршрутизаключаемой маршрутизакона разрешается подключаться к автозаводу или очереди звонков. Пользователи, Location-Based маршрутизов могут принимать входящие вызовы только из этих приложений, если они находятся на том же сайте, откуда поступает входящий звонок через ОКП. 
 
Для передачи голосовых приложений разрешена переадстройка вызовов и одновременные вызовы пользователям и STN. Выполнение звонка на целевой сайт регламентируется правилами Location-Based маршрутизов, перечисленными выше.  
 
Также разрешена переадваровка голосовой почты.  

### <a name="delegation"></a>Делегирование

Пользователь Teams может выбрать делегатов, которые могут звонить и принимать звонки от своего имени. На возможности делегирования в Teams влияет Location-Based маршруты: 
- Для исходящие звонки от Location-Based делегирования маршрутизации, включенной маршрутизов от имени представителя, применяются те же правила. Маршрутизация вызовов основана на политике авторизации звонка представителя, политике маршрутизации голосовой маршрутизации и расположении делегата. Дополнительные сведения см. в Teams о том, как пользователь звонит в [ОКП.](#teams-user-places-an-outbound-call-to-the-pstn) 
- Для входящие вызовы через ОКП к делегату применяются те же Location-Based маршрутизов, которые применяются к переадреации звонков или одновременным звонкам другим пользователям. Дополнительные сведения см. в Teams переад Teams другому пользователю [](#teams-user-transfers-or-forwards-call-to-pstn-endpoint) [Teams,](#teams-user-transfers-or-forwards-call-to-another-teams-user)Teams переназначение звонка на конечную точку ОКП и Одновременный [звонок.](#simultaneous-ringing) Когда представитель устанавливает конечную точку STN в качестве целевой аудитории для одновременного звонка, для направления вызова в ПСС используется политика маршрутирования голосовой маршрутии делегата. 
- Для делегирования рекомендуется, чтобы делегатор и связанные делегаты находились на одном сетевом сайте. 

## <a name="other-planning-considerations"></a>Другие факторы, которые следует учесть при планировании

### <a name="changes-from-an-on-premises-location-based-routing-deployment"></a>Изменения в локальном развертывании Location-Based маршрутии

Политика маршрутинга голосовой связи на сетевом сайте больше не используется. Вместо этого мы используем политику голосовой маршрутии пользователя. Это означает, что для того, чтобы разрешить пользователям перемещаться на другие сайты, политика маршрутинга голосовой связи должна включать шлюзы перемещаемого сайта. 

### <a name="technical-considerations-for-location-based-routing"></a>Технические рекомендации для маршрутизации на основе расположения

Подсети IPv4 и IPv6 поддерживаются, но IPv6 имеет приоритет при проверке совпадения.

### <a name="client-support-for-location-based-routing"></a>Поддержка клиентов для Location-Based маршрутов

Поддерживаются Teams следующие клиенты:
- Teams классических клиентов (Windows и Mac)
- Teams мобильных клиентов (iOS и Android)
- Teams IP-телефоны

Веб-Teams и Skype для бизнеса не поддерживаются.

### <a name="capabilities-not-supported-by-location-based-routing"></a>Функции, не поддерживаемые маршрутизацией на основе расположения

Location-Based маршруты не применяются к следующим типам взаимодействия. Location-Based маршруты не применяются, Teams конечные точки взаимодействуют с конечными точками ННР в следующих сценариях: 
- Парковка вызовов или преобразование вызовов ТСОП с помощью функции парковки вызовов 
- Пользователь локальной Skype для бизнеса или Skype для бизнеса Online звонит Teams пользователю  

### <a name="location-based-routing-for-conferencing"></a>Location-Based маршруты для conferencing

ПользователюLocation-Based включенной маршрутии при звонке по ПСПС, не разрешено начинать конференцию с другим пользователем или номером ДНР. Разрешено подключение к автозаводщикам или очередям вызовов. Если у пользователя есть лицензия на конференц-связи, он должен начать конференцию с соответствующими пользователями и позвонить по STN через мост конференции, чтобы начать конференцию.  

### <a name="media-bypass-requirement-for-location-based-routing"></a>Требования об обходе мультимедиа для Location-Based маршрутии

При развертывании Location-Based маршрутии в Индии необходимо также настроить обход мультимедиа. Дополнительные см. в дополнительных информациях В планах обхода мультимедиа с помощью [прямой маршрутизации](direct-routing-plan-media-bypass.md) и Оптимизация локальных мультимедиа [для прямой маршрутизации.](direct-routing-media-optimization.md)

### <a name="direct-voice-over-ip-voip"></a>Прямая голосовая связь по IP-адресу (VoIP)

Прямая голосовая связь по IP-адресу (VoIP) не должна быть развернута с телефонным оборудованием в Индии.
