# <a name="plan-location-based-routing-for-direct-routing"></a>Планирование маршрутизации на основе расположения для прямой маршрутизации

В некоторых странах и регионах обход поставщика телефонной сети общего пользования (ТСОП) является незаконным, чтобы снизить расходы на междугородние звонки. 

В этой статье описывается, что необходимо знать, чтобы использовать маршрутизацию Location-Based для ограничения обхода платных сборов для пользователей Microsoft Teams в зависимости от их географического расположения. Эта статья относится только к прямой маршрутизации. Location-Based маршрутизация не применяется к плану звонков или оператору Connect.

Когда вы будете готовы включить маршрутизацию Location-Based, см. следующие статьи:

- [Настройка параметров сети для маршрутизации на основе расположения](location-based-routing-configure-network-settings.md)
- [Включение функции "Маршрутизация на основе расположения" для прямой маршрутизации](location-based-routing-enable.md)

> [!NOTE]
> Не следует использовать маршрутизацию Location-Based для динамической маршрутизации вызовов ТСОП на основе расположения пользователя. Это может привести к непредвиденным результатам.

## <a name="overview"></a>Обзор

Location-Based маршрутизация позволяет ограничить обход платных сборов для пользователя на основе политики и географического расположения пользователя во время входящего или исходящего вызова ТСОП. 

Location-Based маршрутизация использует топологию сети, определяемую для региона сети, сайта и подсети. Если обход платных сборов ограничен для расположения, вы связываете каждую IP-подсеть и каждый шлюз ТСОП для этого расположения с сетевым сайтом. 

Во время вызова ТСОП расположение пользователя определяется IP-подсетью, к которому подключены конечные точки Teams пользователя. Если у пользователя есть несколько клиентов Teams, расположенных на разных сайтах, Location-Based маршрутизация применяет маршрутизацию каждого клиента отдельно в зависимости от расположения конечных точек Teams.

Дополнительные сведения о параметрах сети см [. в разделе Параметры сети для функций облачной голосовой связи в Teams](cloud-voice-network-settings.md).

В этой статье предполагается, что сетевой сайт может находиться в одном из следующих состояний:

- **Включено** — сайт, настроенный с использованием подсетей и сайтов сети клиента и включенный для Location-Based маршрутизации.

- **Не включен** — сайт, настроенный с использованием подсетей и сайтов сети клиента, но не включенный для Location-Based маршрутизации.

- **Unknown** — сайт, не настроенный с использованием подсетей и сайтов сети клиента. Как правило, такие сайты являются либо внутренними для корпоративной сети, но по проекту не настроены, либо являются внешними для корпоративной сети. В любом случае для этих сайтов не включена маршрутизация Location-Based. 

### <a name="toll-bypass-evaluation-and-outcome"></a>Оценка и результат обхода платных сборов

При использовании Location-Based маршрутизации оценивается вызов между пользователем Teams и ТСОП, чтобы определить, ограничен ли обход платных сборов. В зависимости от результатов вызов будет завершен или не будет завершен. 

Если для пользователя включена маршрутизация Location-Based и пользователь находится на сайте, где действуют ограничения маршрутизации Location-Based, обход платных сборов ограничен для этого пользователя. Teams использует следующие сведения, чтобы определить, ограничен ли обход платных сборов: 

- Включена ли для пользователя Teams маршрутизация Location-Based, как определено в политике звонков Teams пользователя.

- Расположение сетевого сайта конечной точки пользователя Teams и включена ли для сайта маршрутизация Location-Based.

- Расположение сетевого сайта шлюза ТСОП, используемого вызовом.

- Включен ли шлюз ТСОП, используемый вызовом, для маршрутизации Location-Based.

- В сценариях передачи маршрут вызова ТСОП основан на параметрах маршрутизации пользователя, передавшего вызов, и на параметрах маршрутизации Location-Based пользователя Teams, которому передается вызов.  

- Для сценариев конференц-связи и групповых вызовов независимо от того, является ли пользователь Teams, для которого ограничен обход платных сборов, является или был частью звонка.

Если не удается выполнить вызов, пользователь Teams получает уведомление следующим образом:

- Для исходящих вызовов по ТСОП в окне вызова появляется следующее сообщение: Вызов не разрешен из-за параметров вашей организации.

- Для входящих вызовов ТСОП вызов перенаправляется на основе параметров безотвеченной переадресации вызовов пользователя Teams, как правило, на голосовую почту. Если для пользователя Teams не настроены параметры вызова без ответа, вызов будет отключен.

## <a name="apply-location-based-routing"></a>Применение маршрутизации Location-Based

Необходимо применить Location-Based маршрутизации к следующему:

- [Пользователи](#apply-location-based-routing-at-the-user-location)
- [Сетевые сайты](#apply-location-based-routing-at-the-network-site)
- [Шлюзы ТСОП](#apply-location-based-routing-at-the-pstn-gateway)

Учитывайте следующие рекомендации.

- Шлюз ТСОП и сетевой сайт, связанные со шлюзом, должны быть включены для Location-Based маршрутизации.

- Чтобы выполнять вызовы через шлюз ТСОП, который включен для маршрутизации Location-Based, пользователи также должны быть включены для Location-Based маршрутизации.

- Чтобы разрешить пользователям, для которых включена маршрутизация Location-Based, выполнять исходящие вызовы ТСОП с неизвестного сетевого сайта, должно выполняться следующее:

  - Вызов должен быть исходящим из шлюза ТСОП, который включен для Location-Based маршрутизации.
  - Шлюз ТСОП должен быть настроен с флагом GatewayLbrEnabledUserOverride, равным True.


### <a name="apply-location-based-routing-at-the-user-location"></a>Применение маршрутизации Location-Based в расположении пользователя

Ограничение обхода платных сборов определяет условия, в которых пользователь может совершать и принимать вызовы по ТСОП, а также шлюз ТСОП, который можно использовать. 

Если пользователь находится под ограничением обхода платных сборов, этот пользователь должен быть включен для Location-Based маршрутизации. Если включенный пользователь находится на сайте, где включена маршрутизация Location-Based, он должен совершать вызовы через шлюз, подключенный к сайту и включенный для маршрутизации Location-Based. 

Location-Based маршрутизация определяет текущее расположение пользователя на основе IP-адреса конечной точки Teams пользователя и применяет соответствующие правила. Расположение пользователя, для которого включена маршрутизация Location-Based, можно классифицировать следующим образом: 

- **Пользователь находится на том же Location-Based сайте с поддержкой маршрутизации, связанном со шлюзом ТСОП, где назначен его DID.**<br>В этом сценарии пользователь находится на настроенном сетевом сайте, на котором включена маршрутизация Location-Based, а номер прямого входящего набора (DID) пользователя завершается в шлюзе ТСОП, который находится на том же сетевом сайте. Например, пользователь находится в своем офисе. 

- **Пользователь находится на другом сайте с поддержкой маршрутизации Location-Based, не связанном со шлюзом ТСОП, где назначен его DID.**<br>В этом сценарии пользователь находится на настроенном сетевом сайте, на котором включена маршрутизация Location-Based, и этот сайт не связан со шлюзом ТСОП, где назначен номер DID пользователя. Например, пользователь отправляется в другой офис.  

- **Пользователь находится на внутреннем сайте, на который не включена маршрутизация Location-Based.** <br>В этом сценарии пользователь находится на настроенном сетевом сайте, на котором не включена маршрутизация Location-Based. 

- **Пользователь находится на неизвестном сайте.** 
    - Пользователь находится во внутренней сети, которая не определена как сетевой сайт. 
    - Пользователь находится за пределами внутренней сети. Например, пользователь находится в Интернете дома или в кафе. 

### <a name="apply-location-based-routing-at-the-network-site"></a>Применение маршрутизации Location-Based на сетевом сайте 

Если пользователи, для которых включена маршрутизация Location-Based, перемещаются в роуминге, сетевые сайты, включенные для Location-Based маршрутизации, помогут определить, какие шлюзы следует использовать. Например:

- Если пользователь, для которого включена маршрутизация Location-Based, перемещается на сайт, на который включена маршрутизация Location-Based, для исходящих вызовов можно использовать только шлюз ТСОП, для которого включена маршрутизация Location-Based на этом сайте. 

- Если пользователь, для которого включена маршрутизация Location-Based, перемещается на сайт, на который не включена маршрутизация Location-Based, для исходящих вызовов можно использовать любой шлюз, для которого не включена маршрутизация Location-Based.  

### <a name="apply-location-based-routing-at-the-pstn-gateway"></a>Применение маршрутизации Location-Based в шлюзе ТСОП  

Чтобы применить маршрутизацию Location-Based в шлюзе ТСОП, необходимо выполнить следующие действия.

- Включите шлюз для маршрутизации Location-Based. (Шлюзы должны быть включены для Location-Based маршрутизации, чтобы гарантировать, что они не могут использоваться пользователями, которые не включены для маршрутизации Location-Based.)

- Назначьте сетевой сайт шлюзу.

Затем система определяет, разрешено ли данному пользователю на данном сайте использовать шлюз. 

Кроме того, если для параметра GatewayLbrEnabledUserOverride задано значение True, пользователи с поддержкой маршрутизации на основе расположения на неизвестных сайтах(например, пользователи, работающие на домашних компьютерах), могут совершать исходящие вызовы по ТСОП.


## <a name="restriction-rules"></a>Правила ограничения

Правила ограничения зависят от того, включена ли для пользователя Teams маршрутизация Location-Based.

### <a name="user-is-enabled-for-location-based-routing"></a>Для пользователя включена маршрутизация Location-Based

Если для пользователя включена Location-Based маршрутизация, применяется следующее:

- **Чтобы выполнить исходящий вызов ТСОП**, необходимо выполнить одно из следующих действий:

  - Конечная точка пользователя находится на сайте, где включена маршрутизация Location-Based и вызывает исходящий трафик через шлюз ТСОП, который включен для Location-Based маршрутизации на том же сайте.  

  - Конечная точка пользователя находится на неизвестном сайте и вызывает исходящий трафик через шлюз ТСОП, который включен для Location-Based маршрутизации. Для шлюза ТСОП для параметра GatewayLbrEnabledUserOverride задано значение True.

  - Конечная точка пользователя находится на сайте, на который не включена маршрутизация Location-Based и вызовы исходящего трафика через шлюз ТСОП, который не включен для маршрутизации Location-Based.

- **Чтобы получить входящий вызов ТСОП**, необходимо выполнить одно из следующих действий: 

   - Конечная точка ответа пользователя и шлюз ТСОП, через который осуществляется входящий вызов, должны находиться на том же сайте, где включена маршрутизация Location-Based. Шлюз ТСОП должен быть включен для маршрутизации Location-Based.

   - Конечная точка ответа пользователя и шлюз ТСОП, через который осуществляется входящий вызов, должны находиться на том же сайте, где не включена маршрутизация Location-Based. Шлюз ТСОП не должен быть включен для маршрутизации Location-Based.  (Этот сценарий включает в себя повторную маршрутизацию входящего звонка ТСОП на входящий трафик, но другой шлюз ТСОП, чем тот, который обычно используется для входящих звонков на номер телефона пользователя.)

   - В любом другом сценарии, например при перемещении пользователя, вызов не разрешен и перенаправлен на параметры переадресации вызовов без ответа пользователя (как правило, голосовая почта).  
   
- **Для вызова VoIP в формате 1:1 Teams и передачи в ТСОП** обратите внимание на следующее:

  - Маршрутизация вызова, то есть шлюза ТСОП для исходящего вызова по адресу, основана на параметрах маршрутизации пользователя, передающего вызов.

  - Будет ли разрешена передача, зависит от следующего:
  
    - Location-Based параметры маршрутизации пользователя, передаваемого в ТСОП.
    - Расположение сетевого сайта конечной точки.
    - Включено ли расположение для маршрутизации Location-Based.

    Передача будет разрешена, если переносимый пользователь сможет выполнить этот вызов по ТСОП в своем текущем расположении с помощью того же шлюза ТСОП.

- **Для входящего или исходящего вызова ТСОП и передачи другому пользователю Teams** разрешена ли передача зависит от следующих условий:

   - Параметры маршрутизации пользователя, получающего переданный вызов. 
   - Расположение сетевого сайта конечной точки.
   - Включено ли расположение для маршрутизации Location-Based.

   Передача будет разрешена, если пользователь, получающий переданный вызов, сможет выполнить или принять этот вызов по ТСОП в своем текущем расположении с помощью шлюза ТСОП, используемого текущим вызовом по ТСОП.


### <a name="user-is-not-enabled-for-location-based-routing"></a>Для пользователя не включена маршрутизация Location-Based

Если для пользователя Teams не включена маршрутизация Location-Based, все вызовы этого пользователя должны направляться через шлюз ТСОП, который не включен для маршрутизации Location-Based. Входящий вызов такого пользователя, маршрутизируемый через шлюз ТСОП, включенный для Location-Based Маршрутизация, будет направляться в параметры переадресации вызовов пользователя без ответа (обычно голосовая почта).

### <a name="teams-user-places-an-outbound-call-to-the-pstn"></a>Пользователь Teams размещает исходящий вызов к ТСОП

#### <a name="user-not-enabled-for-location-based-routing"></a>Для пользователя не включена маршрутизация Location-Based

Пользователь, для которого не включена маршрутизация Location-Based, может совершать исходящие вызовы с помощью любого шлюза на любом сайте, для которого не включена маршрутизация Location-Based с помощью назначенной политики маршрутизации голосовой связи. Однако если шлюз включен для маршрутизации Location-Based, пользователь не сможет совершать исходящие вызовы через шлюз, даже если он назначен его политике маршрутизации голосовой связи. Если пользователь перемещается к сайту, на который включена маршрутизация Location-Based, он может совершать вызовы только через обычные шлюзы маршрутизации, для которых не включена маршрутизация Location-Based.
 
#### <a name="user-enabled-for-location-based-routing"></a>Для пользователя включена маршрутизация Location-Based

Для сравнения маршрутизация исходящих вызовов для пользователей, для которых включена маршрутизация Location-Based, зависит от сетевого расположения конечной точки пользователя. В следующей таблице показано, как маршрутизация Location-Based влияет на маршрутизацию исходящих вызовов User1 в зависимости от расположения User1.

### <a name="teams-user-transfers-or-forwards-call-to-another-teams-user"></a>Пользователь Teams передает или пересылает вызов другому пользователю Teams

При использовании конечной точки ТСОП Location-Based маршрутизация анализирует, включен ли один или оба пользователя для маршрутизации Location-Based, и определяет, следует ли передавать или перенаправить вызов в зависимости от расположения обеих конечных точек. 
 
Для передачи звонка инициирующий пользователь должен принять вызов, а для переадресации звонков не требуется ответ на первоначальный вызов. Вызовы можно переадресовать, даже если пользователь User1 не находится в расположении для приема входящих вызовов (см. таблицу в разделе [Пользователь Teams получает входящий звонок из раздела ТСОП](#teams-user-receives-an-inbound-call-from-the-pstn) ), а вызовы не могут быть переданы, если Пользователь User1 не может получить входящий вызов. 

#### <a name="user-not-enabled-for-location-based-routing"></a>Для пользователя не включена маршрутизация Location-Based

Пользователь, для которого не включена маршрутизация Location-Based, может передавать или пересылать вызовы ТСОП другим пользователям, для которых не включена маршрутизация Location-Based. Пользователи, для которых включена маршрутизация Location-Based, как правило, совместно размещаются в шлюзах с поддержкой маршрутизации Location-Based для вызовов ТСОП. Следовательно, пользователи, которые не включены, не могут передавать или пересылать вызов ТСОП пользователю, для которого включена маршрутизация Location-Based. Исключением является то, что пользователь с поддержкой Location-Based маршрутизации перемещается на сайт, на который не включена маршрутизация Location-Based. В этом сценарии перенос вызова разрешен.  

Аналогичным образом, пользователь, которому не включена маршрутизация Location-Based, может принимать только переадресованный или переадресованный вызов ТСОП от другого пользователя, для которого не включена маршрутизация Location-Based.

### <a name="delegation"></a>Делегирование

Пользователь Teams может выбрать делегатов, которые могут совершать и принимать звонки от его имени. На возможности делегирования в Teams влияет маршрутизация Location-Based следующим образом: 

- Для исходящих вызовов от делегата с поддержкой маршрутизации Location-Based от имени делегатора применяются те же правила. Маршрутизация звонков основана на политике авторизации звонков делегата, политике маршрутизации голосовой связи и расположении. Дополнительные сведения см. в статье [Пользователь Teams размещает исходящий вызов в ТСОП](#teams-user-places-an-outbound-call-to-the-pstn). 

- Для входящих вызовов ТСОП к делегированию к делегатам также применяются те же правила Location-Based маршрутизации, которые применяются для переадресации вызовов или одновременного звонка другим пользователям. Дополнительные сведения см. в [разделах Пользователь Teams передает или пересылает вызов другому пользователю Teams](#teams-user-transfers-or-forwards-call-to-another-teams-user), [пользователь Teams передает или перенаправляет вызов в конечную точку ТСОП и Одновременный](#teams-user-transfers-or-forwards-call-to-pstn-endpoint) [звонок](#simultaneous-ringing). Когда делегат задает конечную точку ТСОП в качестве целевого объекта одновременного звонка, для маршрутизации вызова в ТСОП используется политика маршрутизации голосовой связи делегата. 

- Для делегирования корпорация Майкрософт рекомендует, чтобы делегатор и связанные делегаты находились на одном сетевом сайте. 

## <a name="other-planning-considerations"></a>Другие факторы, которые следует учесть при планировании

### <a name="changes-from-an-on-premises-location-based-routing-deployment"></a>Изменения локального развертывания маршрутизации Location-Based

Политика маршрутизации голосовой связи на сайте сети больше не используется. Вместо этого мы используем политику маршрутизации голосовой связи пользователя. Чтобы разрешить пользователям перемещаться по другим сайтам, политика маршрутизации голосовой связи должна включать шлюзы перемещаемых сайтов. 

### <a name="technical-considerations-for-location-based-routing"></a>Технические рекомендации для маршрутизации на основе расположения

Поддерживаются подсети IPv4 и IPv6, однако IPv6 имеет приоритет при проверке соответствия.

### <a name="client-support-for-location-based-routing"></a>Поддержка клиентом маршрутизации Location-Based

Поддерживаются следующие клиенты Teams:
- Настольные клиенты Teams (Windows и Mac)
- Мобильные клиенты Teams (iOS и Android)
- IP-телефоны Teams

Веб-клиент Teams и клиенты Skype для бизнеса не поддерживаются.

### <a name="capabilities-not-supported-by-location-based-routing"></a>Функции, не поддерживаемые маршрутизацией на основе расположения

Location-Based маршрутизация не применяется к следующим типам взаимодействий. маршрутизация Location-Based не применяется, когда конечные точки Teams взаимодействуют с конечными точками ТСОП в следующих сценариях: 

- Парковка вызовов или преобразование вызовов ТСОП с помощью функции парковки вызовов 

- Локальный пользователь Skype для бизнеса или пользователь Skype для бизнеса Online вызывает пользователя Teams  

### <a name="location-based-routing-for-conferencing"></a>маршрутизация Location-Based для конференц-связи

Пользователю с поддержкой маршрутизации Location-Based без лицензии на аудиоконференции при вызове по ТСОП запрещено начинать конференцию с другим пользователем или номером ТСОП. Подключение к автосекретарям или очередям вызовов разрешено.

Если у пользователя есть лицензия на аудиоконференции, он должен начать конференцию с соответствующими пользователями и вызвать ТСОП через мост конференции, чтобы начать конференцию. Если пользователь уже находится на вызове по ТСОП, он может добавить другого пользователя или номер ТСОП в вызов путем эскалации звонка с помощью моста конференц-связи для набора номера.

В конференц-связи, начатой пользователем без лицензии на аудиоконференции, добавление участников ТСОП запрещено, если в конференции есть или был хотя бы один пользователь с поддержкой маршрутизации Location-Based. Если хотя бы один участник ТСОП является или был частью такого конференц-звонка до того, как к нему были приглашены участники с поддержкой маршрутизации Location-Based, такие участники Location-Based с поддержкой маршрутизации не могут быть добавлены в вызов.

Если пользователь с поддержкой маршрутизации Location-Based присоединяется к конференции с внутреннего сайта, на который не включена маршрутизация Location-Based, ограничения, указанные в приведенном выше абзаце, не применяются. 

Конференц-связь по сети для аудиоконференций не должна развертываться с любым телефонным оборудованием в Индии.

Пользователю Location-Based маршрутизации в вызове ТСОП запрещено объединять этот вызов с другим вызовом. Следующие функции не поддерживаются: запись вызова по ТСОП и запись соответствия для вызова ТСОП.

### <a name="media-bypass-requirement-for-location-based-routing"></a>Требование обхода мультимедиа для маршрутизации Location-Based

Дополнительные сведения см. в [разделах Планирование обхода мультимедиа с помощью прямой маршрутизации](direct-routing-plan-media-bypass.md) и [оптимизации локального носителя для прямой маршрутизации](direct-routing-media-optimization.md).

### <a name="direct-voice-over-ip-voip"></a>Прямая голосовая связь по IP-адресу (VoIP)

Прямая голосовая связь по IP-адресу (VoIP) не должна развертываться с телефонным оборудованием в Индии.
