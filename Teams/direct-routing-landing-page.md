# <a name="direct-routing"></a>Прямая маршрутизация

Вы выполнили задачу [Начало работы](get-started-with-teams-quick-start.md). Вы развернули Teams с [чатом, командами, каналами и приложениями](deploy-chat-teams-channels-microsoft-teams-landing-page.md) в пределах всей организации. Возможно, вы развернули собрания [& конференц-связи](deploy-meetings-microsoft-teams-landing-page.md). Теперь все готово к добавлению голосовых рабочих нагрузок, и вы решили использовать собственный оператор телефонии для подключения к общедоступной телефонной сети (ТСОП) с помощью телефонной системы с прямой маршрутизаторией. Благодаря прямой маршрутизации вы можете использовать телефонную систему практически с любым оператором телефонной связи.

В этой статье описываются основные решения по развертыванию для прямой маршрутизации, а также дополнительные рекомендации, которые могут потребоваться учитывать в зависимости от потребностей вашей организации. Вы также должны [прочитать "Планирование голосового решения"](cloud-voice-landing-page.md) , чтобы получить дополнительные сведения о предложениях голосовой связи Майкрософт.

### <a name="ensure-incoming-calls-land-in-the-teams-client-using-teamsupgradepolicy"></a>Убедитесь, что входящие вызовы попадать в клиент Teams с помощью TeamsUpgradePolicy

Прямая маршрутизация поддерживается только в Microsoft Teams. Чтобы принимать вызовы ТСОП через прямую маршрутизацию, необходимо настроить TeamsUpgradePolicy, чтобы обеспечить прием входящих звонков в Teams. Пользователи должны находиться в режиме TeamsOnly, что можно сделать, назначив им экземпляр UpgradeToTeams TeamsUpgradePolicy. 

|Задайте себе вопрос|Действие |
|:------------|:-------|
|Что означает режим TeamsOnly? | Дополнительные сведения см. в руководстве по миграции и взаимодействию для организаций, использующих [Teams вместе с Skype для бизнеса](./migration-interop-guidance-for-teams-with-skype.md).|
