# <a name="direct-routing"></a>Прямая маршрутия

Вы выполнили задачу [Начало работы](get-started-with-teams-quick-start.md). Вы развернули Teams с [чатом, командами, каналами и приложениями](deploy-chat-teams-channels-microsoft-teams-landing-page.md) в пределах всей организации. Возможно, вы развернули [& собрания](deploy-meetings-microsoft-teams-landing-page.md). Теперь вы готовы добавить рабочие нагрузки голосовой связи и решили использовать собственный оператор телефонии для подключения к телефонной сети общего пользования (STN) с помощью телефонная система прямой маршрутии. Благодаря прямой маршрутизации вы можете использовать телефонную систему практически с любым оператором телефонной связи.

В этой статье описаны основные решения по развертыванию прямой маршрутизации, а также дополнительные факторы, которые следует учитывать в зависимости от потребностей организации. Дополнительные [сведения о](cloud-voice-landing-page.md) голосовых предложениях Майкрософт можно найти в этой публикации.

### <a name="ensure-incoming-calls-land-in-the-teams-client-using-teamsupgradepolicy"></a>Убедитесь, что входящие звонки в клиент Teams с помощью TeamsUpgradePolicy

Прямая маршрутная маршрутия поддерживается только в Microsoft Teams. Для получения звонков по ЗВОНКОВ через прямую маршрутику необходимо настроить TeamsUpgradePolicy, чтобы обеспечить прием входящих звонков Teams. Пользователи должны работать в режиме TeamsOnly, что можно сделать, назначив им экземпляр UpgradeToTeams TeamsUpgradePolicy.
