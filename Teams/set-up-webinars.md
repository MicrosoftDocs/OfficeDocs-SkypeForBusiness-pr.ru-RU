# <a name="set-up-webinars-in-microsoft-teams"></a>Настройка вебинаров в Майкрософт Teams

> [!NOTE]
> В этой статье описываются некоторые функции вебинаров, которые находятся в предварительной версии и требуют лицензии Teams Premium.

Майкрософт теперь предлагает новый вебинар. В этой статье описывается, как обновить параметры для использования этих функций.

Если вы планируете использовать вебинары, рекомендуется использовать новый интерфейс вебинара.

Регистрация собраний включает основные функции вебинара, возможность требовать регистрацию для собраний и отчет о посещаемости. Включив новый вебинар, вы можете использовать регистрацию собраний и множество новых функций вебинара, таких как:

- Выделенное событие и страница регистрации для вебинара
- Соорганизаторы
- Bios докладчика на странице события
- Общие сведения о состоянии регистрации и управление ими

Дополнительные сведения о новых функциях, доступных для конечных пользователей, см. в статье [Начало работы с вебинары Teams](https://support.microsoft.com/office/42f3f874-22dc-4289-b53f-bbc1a69013e3).

Если в вашей организации включена регистрация собраний, все созданные вебинары будут иметь новый интерфейс. Запланированные ранее вебинары будут использовать предыдущий вебинар. В новом интерфейсе используется TeamsEventsPolicy. Если вебинары отключены, они останутся отключенными по мере развертывания нового интерфейса.

В настоящее время базовый вебинар управляется регистрацией собрания с помощью политики собраний Teams (Set-CsTeamsMeetingPolicy). В будущем параметр регистрации собрания не будет управлять вебинарами; вебинары переходят на управление политикой событий Teams (Set-CsTeamsEventsPolicy).

Новый интерфейс вебинара настраивается в PowerShell. См. примеры настройки [нового вебинара](#set-up-new-webinar-experience).

Дополнительные сведения о различиях между собраниями, вебинарами и трансляциями см. в разделе [Собрания, вебинары и трансляции](quick-start-meetings-live-events.md).

> [!IMPORTANT]
> Чтобы пользователи могли настраивать вебинары, необходимо настроить Microsoft Списки в SharePoint, включив создание личных списков для целей обнаружения электронных данных. Дополнительные сведения см. в разделе [Параметры управления для Microsoft Списки](/sharepoint/control-lists).

## <a name="set-up-new-webinar-experience"></a>Настройка нового вебинара

Для настройки нового вебинара в организации необходимо использовать PowerShell. Возможность настройки нового вебинара в Центре администрирования Teams пока недоступна.

Регистрация собрания должна быть включена, чтобы использовать новый интерфейс вебинара.

### <a name="configure-the-new-webinar-experience-with-powershell"></a>Настройка нового интерфейса вебинара с помощью PowerShell

Чтобы настроить новый интерфейс вебинара, используйте следующие атрибуты в командлете Windows PowerShell **Set-CsTeamsEventsPolicy**.

|Параметр|Настройка по умолчанию|Описание|
|---------|-----------|---------------|
|AllowWebinars|Включено|Этот параметр определяет, может ли пользователь создавать вебинары.|
|EventAccessType|Все|Этот параметр определяет, какие пользователи могут получить доступ к странице регистрации события или сайту событий для регистрации, а также к какому типу пользователя разрешено присоединяться к сеансам в событии.|

Перед выполнением этих командлетов необходимо подключиться к Майкрософт Teams PowerShell. Дополнительные сведения см. в статье [Управление Teams с помощью Майкрософт Teams PowerShell](/microsoftteams/teams-powershell-managing-teams).

1. Включите регистрацию собрания.

    ```powershell
    Set-CsTeamsMeetingPolicy -Identity <policy name> -AllowMeetingRegistration $True
    ```

1. Активируйте новый интерфейс вебинара.

    ```powershell
    Set-CsTeamsEventsPolicy -Identity <policy name> -AllowWebinars Enabled
    ```

1. Настройка пользователей, которые могут зарегистрироваться для вебинаров и собраний.

    - **Разрешить **_только_* _ пользователям в вашей организации регистрироваться для участия в вебинаре и meetings_*

        ```powershell
        Set-CsTeamsEventsPolicy -Identity <policy name> -EventAccessType EveryoneInCompanyExcludingGuests
        ```

    - **Разрешить всем, включая анонимных пользователей, регистрироваться для участия в вебинаре и собраниях**

        ```powershell
        Set-CsTeamsEventsPolicy -Identity <policy name> -EventAccessType Everyone
        ```

> [!IMPORTANT]
> Если **анонимные пользователи могут присоединиться к собранию** в **параметрах собрания**, анонимные пользователи не смогут присоединяться к вебинарам. Дополнительные сведения и включение этого параметра см. [в разделе Параметры собрания в Teams](meeting-settings-in-teams.md).

## <a name="configure-meeting-registration"></a>Настройка регистрации собраний

Если вы хотите использовать вебинары, регистрация собраний должна быть включена.

Центр администрирования Teams можно использовать в разделе **Политики собраний** >  для настройки регистрации собраний и вебинаров собраний.

### <a name="meeting-registration"></a>Регистрация собрания

Если включить **регистрацию собраний**, пользователи в вашей организации могут запланировать вебинары и собрания, требующие регистрации. По умолчанию этот параметр включен. Если вы хотите отключить регистрацию собраний и вебинары, установите для этой политики значение **Выкл**.

Чтобы регистрация собрания работала, необходимо включить **планирование приватных собраний**. Узнайте больше о [планировании частных собраний](meeting-policies-in-teams-general.md).

Для учащихся в арендаторах для образовательных учреждений эта политика отключена по умолчанию. Дополнительные сведения о том, как включить планирование частных собраний для учащихся, см[. в разделе Teams для образования политик и пакетов политик](policy-packages-edu.md).

#### <a name="who-can-register"></a>Кто может зарегистрироваться

> [!NOTE]
> Эта политика не применяется к новому интерфейсу вебинара. Чтобы настроить пользователей, которые могут зарегистрироваться для участия в новом вебинаре, используйте `Set-CsTeamsEventsPolicy -EventAccessType`, как показано в [разделе Настройка нового интерфейса вебинара с помощью PowerShell](#configure-the-new-webinar-experience-with-powershell).

Эта политика определяет, какие пользователи могут регистрироваться и посещать вебинары только с регистрацией собраний. Эта политика имеет два варианта, которые доступны только в том случае, если **регистрация собрания** включена. По умолчанию **параметр Кто может зарегистрировать** имеет значение **Все**.

Если выбрать **Все**, все пользователи, включая анонимных пользователей, смогут зарегистрироваться и принять участие в вебинаре. Если выбрать **Все в организации**, только пользователи в вашей организации смогут зарегистрироваться и принять участие в вебинаре. Если регистрация собрания отключена, параметр **Кто может зарегистрировать** будет недоступен, и никто не сможет зарегистрироваться для участия в вебинаре.

Значение по умолчанию для **параметра Кто может зарегистрировать** — **Все в организации** в арендаторах для образовательных учреждений. Дополнительные сведения см. в разделе [Мастер политик Teams для образования](easy-policy-setup-edu.md).

## <a name="collect-webinar-and-meeting-registration-attendance"></a>Сбор сведений о посещаемости вебинара и регистрации собраний

Вы можете использовать Центр администрирования Teams в разделе **Политики собраний** > **,** чтобы включить **отчет о взаимодействии**.

Если этот параметр включен, организаторы могут просматривать отчеты о том, кто зарегистрировал и принял участие в вебинаре или собраниях, которые они настроили. Эта политика включена по умолчанию. Дополнительные сведения см. [в статье Политики собраний в Teams — отчет о взаимодействии](meeting-policies-in-teams-general.md#engagement-report). Сведения о взаимодействии с конечными пользователями см. в разделе [Просмотр и скачивание отчетов о посещаемости собраний](https://support.microsoft.com/office/ae7cf170-530c-47d3-84c1-3aedac74d310).

В PowerShell для включения этого параметра можно использовать параметр **AllowEngagementReport** . Эта политика включена по умолчанию. Чтобы отключить его, выполните следующую команду в PowerShell:

```powershell
Set-CsTeamsMeetingPolicy -Identity <policy name> -AllowEngagementReport Disabled
```

Дополнительные сведения о командлете см. в статье [Set-CsTeamsMeetingPolicy](/powershell/module/skype/set-csteamsmeetingpolicy) .

## <a name="turn-off-webinars"></a>Отключение вебинаров

Вебинары можно отключить с помощью PowerShell. Это приведет к отключению нового вебинара, а также вебинаров только с регистрацией собраний.

Чтобы отключить вебинары, используйте следующий сценарий PowerShell:

```powershell
Set-CsTeamsMeetingPolicy -Identity <policy name> -AllowMeetingRegistration $False
Set-CSTeamsEventsPolicy -Identity <policy name> -AllowWebinars Disabled
```
