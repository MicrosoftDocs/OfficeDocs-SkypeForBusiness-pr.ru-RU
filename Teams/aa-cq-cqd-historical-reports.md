# <a name="auto-attendant--call-queue-historical-report"></a>Auto Attendant & Call Queue Historical Report

Шаблон CQD Teams auto Attendant & истории Power BI очереди вызовов содержит следующие три отчета:

- Автоответ: отображение аналитики для звонков, которые будут приходить в автоответы.
- Очередь звонков— отображение аналитики для звонков, которые будут приходить в очереди звонков.
- Временная шкала агента— отображение временной шкалы, в которой агенты активны при звонках в очереди звонков.

В этих отчетах [](CQD-Power-BI-query-templates.md) используются данные из магазина данных "Панель мониторинга качества звонков", что позволяет организациям сообщать о количестве звонков, обрабатываемых автоответами и очередями звонков, а также о производительности агентов в очередях звонков.

## <a name="what-are-the-requirements"></a>Каковы требования? 

Необходимо установить Power BI Desktop. Вы можете установить его из [Microsoft Windows Store.](https://aka.ms/pbidesktopstore)

Вы можете использовать бесплатную версию Power BI Desktop. Минимальная совместимая версия: 2.85.681.0 (сентябрь 2020 г.).

## <a name="permissions-to-access-the-cqd-pipeline"></a>Разрешения на доступ к каналу CQD

Учетная запись, используемая для просмотра отчета AA & CQ Analytics, должна иметь разрешения на доступ к каналу данных CQD. Дополнительные сведения можно найти в роли "Доступ к [CQD".](./turning-on-and-using-call-quality-dashboard.md#assign-admin-roles-for-access-to-cqd)

## <a name="installation"></a>Установка 

В следующих действиях предполагается, что Power BI Desktop на компьютере уже установлены и что у вашей учетной записи есть необходимые разрешения на доступ к каналу данных CQD.

Выполните следующие действия:

- Скачайте [шаблоны запросов Power BI CQD и](https://www.microsoft.com/download/details.aspx?id=102291) сохраните ZIP-файл в каталоге на компьютере.

- Дважды щелкните ZIP-файл, чтобы открыть его.

- Дважды щелкните файл шаблона "CQ и AA combined Analytics 20201105.pbit" и Power BI Desktop запустится.

- Вам будет предложено выбрать область CQD-канал данных. Выберите регион, в котором находится ваш клиент.

  :::image type="content" source="media/cqd-teams-aa-cq-historical-report-01.png" alt-text="Снимок экрана: выбор области CQD-каналов данных":::

 - Регион можно увидеть с помощью Skype для бизнеса PowerShell (Get-CsTenant). Вывод ServiceInstance. 
 В этом примере регион будет отображаться после /like:

   Microsoftcommunicationsonline/noam-4a-s7, где регионом является noam.
 
 - Отчет будет запущен с образцом данных.
 
 - Чтобы увидеть собственные данные, нажмите **кнопку** Обновить на вкладке Главная в области Запросы Power BI Desktop.

   :::image type="content" source="media/cqd-teams-aa-cq-historical-report-02.png" alt-text="Снимок экрана: выбор параметра обновления":::

- После этого вам будет предложено войти в нее. Выберите **Учетная запись организации,** а затем выберите **Войти**.

  :::image type="content" source="media/cqd-teams-aa-cq-historical-report-03.png" alt-text="Снимок экрана: вход":::

- Выберите **Подключение** и просмотрите обновление данных.

## <a name="data-latency-and-aa--cq-analytics"></a>Задержка данных и аналитика CQ & AA

Данные будут доступны в канале данных CQD в течение 30 минут.

Чтобы увидеть новые аналитические данные, необходимо обновить данные. 

## <a name="customization"></a>Настройки 

Вы можете настроить определенные аспекты визуализации отчетов, например добавить или удалить поля, которые будут показаны в различных визуализациях, изменить тип диаграммы и т. д.

Добавить дополнительные поля данных, кроме полей отчета, невозможно.

## <a name="known-issues"></a>Известные проблемы

- В очереди вызовов и автоотехниках вместо имен очереди вызовов и автоотехников отображается ИД учетной записи ресурса.  Чтобы отобрать весь трафик для автозавода или очереди вызовов, необходимо выбрать все учетные записи ресурсов, которые назначены автозаводщику или очереди вызовов.

- На панели мониторинга доступен только 28-й день истории, так как данные очереди вызовов и автоотображаемых считаются идентифицируемыми данными конечного пользователя и подотменяются политиками хранения конфиденциальности данных.

- <sup>1</sup> **Источник входящих** вызовов на графиках автозавода и очереди вызовов показывает конечный источник, а не исходный источник зова. Например, если автоотекатарь принимает внешний звонок и переносит звонок другому автозаводщику или очереди вызовов, источник входящих вызовов будет называться Внутренний.
