# <a name="meeting-policy-settings---participants--guests"></a>Параметры политик собраний — участники и гости

<a name="bkmeetingparticipants"> </a>

Эти параметры определяют, какие участники собрания будут находиться в "зале ожидания" перед допуском в собрание, а также разрешенный для них уровень участия в собрании.

- [Разрешать анонимным пользователям начинать собрания](#let-anonymous-people-start-a-meeting)
- [Автоматически допускать пользователей](#automatically-admit-people)
- [Разрешить пользователям с телефонным подключением обходить "зал ожидания"](#allow-dial-in-users-to-bypass-the-lobby)
- [Включить автоматические субтитры](#enable-live-captions)
- [Разрешить чат в собраниях](#allow-chat-in-meetings)

> [!NOTE]
>Варианты присоединения к собранию зависят от параметров каждой группы Teams и метода подключения. Если ваша группа поддерживает аудиоконференции и использует их для подключения, см. раздел [Аудиоконференции](/microsoftteams/audio-conferencing-in-office-365). Если в вашей группе Teams нет аудиоконференций, см. раздел [Присоединение к собранию в Teams](https://support.office.com/article/join-a-meeting-in-teams-1613bb53-f3fa-431e-85a9-d6a91e3468c9).

## <a name="let-anonymous-people-start-a-meeting"></a>Разрешать анонимным пользователям начинать собрания

Этот параметр является политикой "на каждого организатора", которая позволяет использовать собрания с безудеративным набором телефонных номеров. Этот параметр позволяет контролировать, могут ли пользователи с телефонным доступом присоединяться к собранию без участия пользователя, который проходит проверку подлинности организации. По умолчанию этот параметр отключен, то есть пользователи, которые присоединяются к собранию по телефонной связи, будут ждать в "ожидании" в "ожидании", пока пользователь из организации, у которого есть проверка подлинности, не присоединится к собранию.

> [!NOTE]
> Если этот параметр отключен, а пользователь, входящего в телефонное окно, присоединяется к собранию первым и помещается в " "в", пользователь организации должен присоединиться к собранию с помощью клиента Teams, чтобы допустить пользователя из "Teams". Для пользователей, подключающихся по телефону, недоступны элементы управления "зала ожидания".

## <a name="automatically-admit-people"></a>Автоматически допускать пользователей

Это политика для организатора. Этот параметр определяет, присоединяются ли пользователи к собранию напрямую или помещаются в "зал ожидания", пока их не допустит авторизованный пользователь. Этот параметр не применяется к пользователям с телефонным подключением.

![Снимок экрана: собрание с пользователем в "вестибюле".](media/meeting-policies-lobby.png)

 Организаторы собраний могут щелкнуть **Параметры** собрания в приглашении на собрание, чтобы изменить этот параметр для каждого собрания, на который они запланировать.

> [!NOTE]
> В параметрах собрания этот параметр называется "Кто может миновать "зал ожидания"?". Если изменить параметр по умолчанию для любого пользователя, этот параметр будет применяться ко всем новым собраниям, организованным этим пользователем, и к любым предыдущим собраниям, в которых пользователь не изменял параметры.
  
|Значение параметра  |Поведение при присоединении |
|---------|---------|
|**Все**   |Все участники собрания присоединяются к собранию напрямую, без помещения в "зал ожидания". К ним относятся авторизованные пользователи, внешние пользователи из доверенных организаций (федеративные), гости и анонимные пользователи.     |
|**Люди в моей организации и гости**     |Пользователи, которые проходит проверку подлинности в организации, в том числе гостевых, присоединяются к собранию напрямую, не дожидаясь ожидания в зале ожидания. Пользователи из надежных организаций и анонимные пользователи будут ждать в "ожидании". Это параметр по умолчанию.    |
|**Люди в моей организации, доверенные организации и гости**     |Авторизованные пользователи организации, включая гостевых пользователей и пользователей из доверенных организаций, присоединяются к собранию напрямую, без помещения в "зал ожидания".  Анонимные пользователи помещаются в "зал ожидания".   |
|**Люди в моей организации**    |Пользователи, которые проходит проверку подлинности из организации, присоединяются к собранию напрямую, не дожидаясь в зале ожидания.  Пользователи из надежных организаций, гостевых и анонимных пользователей будут ждать в "ожидании".          |
|**Только организатор**    |Только организаторы собрания могут присоединяться к собранию напрямую, без помещения в "зал ожидания". Все остальные пользователи, включая пользователей, которые пройти проверку подлинности в организации, гостевых пользователей, пользователей из надежных организаций и анонимных пользователей, должны ждать в "ожидании". На странице параметров Teams клиента отображается только я.          |
|**Только приглашенные пользователи**    |Только приглашенные пользователи и организаторы собраний могут присоединиться к собранию напрямую, не дожидаясь в зале ожидания. Все остальные пользователи, включая пользователей, которые пройти проверку подлинности в организации, гостевых пользователей, пользователей из надежных организаций и анонимных пользователей, должны ждать в "ожидании". На странице Teams параметров клиентского собрания отображается "Приглашенные люди".          |

## <a name="allow-dial-in-users-to-bypass-the-lobby"></a>Разрешить пользователям с телефонным подключением обходить "зал ожидания"

Это политика для организатора. Этот параметр определяет, могут ли пользователи присоединяться непосредственно к собранию по телефону или будут попадать в "зал ожидания" независимо от параметра **Автоматически допускать пользователей**. По умолчанию этот параметр отключен. Если этот параметр отключен, пользователи с телефонным подключением будут находиться в "зале ожидания", пока пользователь организации не присоединится к собранию в клиенте Teams и не допустит их. Если этот параметр включен, пользователи с телефонным подключением автоматически присоединяются к собранию, когда к нему присоединяется пользователь организации.

> [!NOTE]
> Если пользователь с телефонным подключением присоединяется к собранию до пользователя организации, он будет помещен в "зал ожидания" до тех пор, пока пользователь организации не присоединится к собранию с помощью клиента Teams и не допустит его. Если вы измените параметр по умолчанию для любого пользователя, он будет применяться ко всем новым собраниям, организуемым этим пользователем, и к любым предыдущим собраниям, в которых пользователь не изменил параметры.

## <a name="enable-live-captions"></a>Включить автоматические субтитры

Этот параметр является политикой "на пользователя" и применяется во время собрания. Этот параметр определяет, доступен ли параметр **Включить автоматические субтитры** для пользователя, чтобы включать и отключать автоматические субтитры в собраниях, в которые он участвует.  

![Снимок экрана: параметр "Включить субтитры в прямом эфире".](media/meeting-policies-live-captions.png)

|Значение параметра |Поведение  |
|---------|---------|
|**Отключено, но пользователь может переопределить**     | Автоматические субтитры не включены для пользователя во время собрания. Пользователь видит параметр **Включить субтитры** в меню **(...)**, где может их включить. Это параметр по умолчанию. |
|**Отключено**     | Автоматические субтитры отключены для пользователя во время собрания. У пользователя нет возможности включить их.          |
