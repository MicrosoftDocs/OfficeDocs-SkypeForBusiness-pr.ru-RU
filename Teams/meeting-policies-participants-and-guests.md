# <a name="meeting-policy-settings---participants--guests"></a>Параметры политик собраний — участники и гости

<a name="bkmeetingparticipants"> </a>

Эти параметры определяют, какие участники собрания ожидают в зале ожидания, прежде чем они будут допущены к собранию, и уровень участия в собрании.

- [Разрешать анонимным пользователям начинать собрания](#let-anonymous-people-start-a-meeting)
- [Кто может присутствовать на собраниях](#who-can-present-in-meetings)
- [Автоматически допускать пользователей](#automatically-admit-people)
- [Пользователи с телефонным подключением могут обойти лобби](#dial-in-users-can-bypass-the-lobby)
- [Встречайте сейчас на закрытых собраниях](#meet-now-in-private-meetings)
- [Субтитры в режиме реального времени](#live-captions)
- [Чат на собраниях](#chat-in-meetings)
- [Teams Q&A](#teams-qa)
- [Реакции на собрания](#meeting-reactions)

Эти параметры находятся в Центре администрирования Teams в разделе **Политики собраний**  >  в разделе **Участники & гостей**.

> [!NOTE]
> Варианты присоединения к собранию зависят от параметров каждой группы Teams и метода подключения. Если ваша группа поддерживает аудиоконференции и использует их для подключения, см. раздел [Аудиоконференции](/microsoftteams/audio-conferencing-in-office-365). Если в вашей группе Teams нет аудиоконференций, см. раздел [Присоединение к собранию в Teams](https://support.office.com/article/join-a-meeting-in-teams-1613bb53-f3fa-431e-85a9-d6a91e3468c9).

## <a name="let-anonymous-people-join-a-meeting"></a>Предоставление анонимным пользователям присоединения к собранию

Этот параметр для каждого организатора позволяет любому пользователю присоединяться к собраниям в качестве анонимного пользователя, щелкнув ссылку в приглашении на собрание. Чтобы узнать больше, см. [Присоединение к собранию без учетной записи Teams](https://support.microsoft.com/office/c6efc38f-4e03-4e79-b28f-e65a4c039508). Возможность присоединения анонимных пользователей к собраниям также контролируется на уровне организации. Более строгий параметр будет действовать. Дополнительные сведения см [. в статье Использование Центра администрирования Microsoft Teams для настройки политики на уровне организации](meeting-settings-in-teams.md#allow-anonymous-users-to-join-meetings).

## <a name="let-anonymous-people-start-a-meeting"></a>Разрешать анонимным пользователям начинать собрания

Этот параметр является политикой для каждого организатора, которая позволяет проводить конференц-связь с телефонным подключением без лидера. Этот параметр определяет, могут ли пользователи с телефонным подключением присоединяться к собранию без участия пользователя, прошедшего проверку подлинности из организации. По умолчанию этот параметр отключен, что означает, что пользователи с телефонным подключением будут ждать в вестибюле, пока пользователь из организации, прошедший проверку подлинности, присоединится к собранию.

> [!NOTE]
> Если этот параметр отключен и пользователь с телефонным подключением присоединяется к собранию первым и размещается в зале ожидания, пользователь организации должен присоединиться к собранию с клиентом Teams, чтобы принять пользователя из лобби. Для пользователей, подключающихся по телефону, недоступны элементы управления "зала ожидания".

## <a name="who-can-present-in-meetings"></a>Кто может присутствовать на собраниях

Это политика для пользователя. Этот параметр позволяет изменить значение по умолчанию параметра **Кто будет выступать?** в разделе **Параметры собрания** в клиенте Teams. Этот параметр политики влияет на все собрания, включая быстрые незапланированные собрания.

Параметр **Кто будет выступать?** позволяет организаторам собрания выбирать, кто может выступать на собрании. Дополнительные сведения см. в статье [Изменение параметров участников для собраний Teams](https://support.microsoft.com/office/53261366-dbd5-45f9-aae9-a70e6354f88e) и [Роли в собрании Teams](https://support.microsoft.com/office/c16fa7d0-1666-4dde-8686-0a0bfe16e019).

Чтобы указать значение по умолчанию для параметра **Кто может представить?** в Teams, задайте в политике один из следующих параметров:

- **Организаторы, но пользователи могут переопределить**: докладчиком может быть только организатор собрания, и все участники собрания назначаются в качестве участников. Этот параметр соответствует параметру **Только я** в Teams.
- **Все в организации, но пользователь может переопределить**: пользователи, прошедшие проверку подлинности в организации, включая гостевых пользователей, могут быть выступающими. Этот параметр соответствует **Люди в параметре моей организации** в Teams.
- **Все, кроме пользователя, могут переопределить**: все участники собрания могут быть выступающими. Это значение используется по умолчанию. Этот параметр соответствует параметру **Все** в Teams.

Имейте в виду, что после установки значения по умолчанию организаторы собраний по-прежнему могут изменить этот параметр в Teams и выбрать, кто может выступать на собраниях, которые они планируют.

## <a name="automatically-admit-people"></a>Автоматически допускать пользователей

Это политика для организатора. Этот параметр определяет, присоединяются ли пользователи к собранию напрямую или ждут в зале ожидания, пока они не будут допущены пользователем, прошедшим проверку подлинности. Этот параметр не применяется к пользователям с телефонным подключением.

![Снимок экрана: собрание с пользователем в "зале ожидания".](media/meeting-policies-lobby.png)

 Организаторы собрания могут щелкнуть **Параметры собрания** в приглашении на собрание, чтобы изменить этот параметр для каждого запланированного собрания.

> [!NOTE]
> In the meeting options the setting is labeled "Who can bypass the lobby". If you change the default setting for any user, it will apply to all new meetings organized by that user and any prior meetings where the user didn't modify Meeting options.
  
|Значение параметра  |Поведение при присоединении |
|---------|---------|
|**Все**   |Все участники собрания присоединяются к собранию напрямую, без помещения в "зал ожидания". Сюда входят пользователи, прошедшие проверку подлинности, пользователи из доверенных организаций, гости и анонимные пользователи.     |
|**Люди в моей организации и гости**     |Прошедшие проверку подлинности пользователи в организации, включая гостей, присоединяются к собранию напрямую, не дожидаясь в зале ожидания. Пользователи из доверенных организаций и анонимные пользователи помещаются в "зал ожидания". Это параметр по умолчанию.    |
|**Люди в моей организации, доверенные организации и гости**     |Пользователи, прошедшие проверку подлинности в организации, включая гостей и пользователей из доверенных организаций, присоединяются к собранию напрямую, не ожидая в зале ожидания.  Анонимные пользователи помещаются в "зал ожидания".   |
|**Люди в моей организации**    |Прошедшие проверку подлинности пользователи из организации присоединяются к собранию напрямую, не дожидаясь в зале ожидания.  Пользователи из доверенных организаций, гости и анонимные пользователи ждут в зале ожидания.          |
|**Только организатор**    |Только организаторы собрания могут присоединяться к собранию напрямую, без помещения в "зал ожидания". Все остальные, включая прошедших проверку подлинности пользователей в организации, гостей, пользователей из доверенных организаций и анонимных пользователей, должны ждать в зале ожидания. На странице Параметры собрания клиента Teams отображается значение "Только я".          |
|**Только приглашенные пользователи**    |Только приглашенные пользователи и организаторы собраний могут присоединиться к собранию напрямую, не дожидаясь в зале ожидания. Все остальные, включая прошедших проверку подлинности пользователей в организации, гостей, пользователей из доверенных организаций и анонимных пользователей, должны ждать в зале ожидания. На странице параметров собрания клиента Teams он отображается как "Люди я приглашаю". Пользователям, добавленным в группу рассылки, придется пройти через лобби.      |

 > [!NOTE]
> Доверенные организации — это домены, с которыми разрешено федеративное взаимодействие в Teams. Если включить **параметр Разрешить все внешние домены** для внешнего доступа в Центре администрирования Teams, любой пользователь, прошедший проверку подлинности в любой организации Teams, будет доверенным. Если вы решили указать разрешенные внешние домены и заблокировать все остальные, разрешенные домены станут доверенными организациями. Любой заблокированный домен не считается доверенной организацией.

## <a name="dial-in-users-can-bypass-the-lobby"></a>Пользователи с телефонным подключением могут обойти лобби

Это политика для организатора. Этот параметр определяет, могут ли пользователи присоединяться непосредственно к собранию по телефону или будут попадать в "зал ожидания" независимо от параметра **Автоматически допускать пользователей**. По умолчанию этот параметр отключен. Если этот параметр отключен, пользователи с телефонным подключением будут находиться в "зале ожидания", пока пользователь организации не присоединится к собранию в клиенте Teams и не допустит их. Если этот параметр включен, пользователи с телефонным подключением автоматически присоединяются к собранию, когда пользователь организации присоединяется к собранию с клиентом Teams.

> [!NOTE]
> Если пользователь с телефонным подключением присоединяется к собранию до пользователя организации, он будет помещен в "зал ожидания" до тех пор, пока пользователь организации не присоединится к собранию с помощью клиента Teams и не допустит его. Если вы измените параметр по умолчанию для любого пользователя, он будет применяться ко всем новым собраниям, организуемым этим пользователем, и к любым предыдущим собраниям, в которых пользователь не изменил параметры.

## <a name="meet-now-in-private-meetings"></a>Встречайте сейчас на закрытых собраниях

Это политика для пользователя, которая применяется перед началом собрания. Этот параметр определяет, может ли пользователь начать незапланированное личное собрание. По умолчанию этот параметр включен.

## <a name="live-captions"></a>Субтитры в режиме реального времени

Этот параметр является политикой для каждого пользователя и применяется во время собрания. Этот параметр определяет, доступен ли параметр **Включить автоматические субтитры** для пользователя, чтобы включать и отключать автоматические субтитры в собраниях, в которые он участвует.  

![Снимок экрана: параметр "Включить автоматические субтитры".](media/meeting-policies-live-captions.png)

|Значение параметра |Поведение  |
|---------|---------|
|**Не включено, но пользователь может переопределить**     | Автоматические субтитры не включены для пользователя во время собрания. Пользователь видит параметр **Включить субтитры** в меню **(...)**, где может их включить. Это параметр по умолчанию. |
|**Не включено**     | Автоматические субтитры отключены для пользователя во время собрания. У пользователя нет возможности включить их.          |

## <a name="chat-in-meetings"></a>Чат на собраниях

Это политика для каждого пользователя и организатора. Этот параметр определяет, разрешен ли чат собрания во время собрания пользователя. Этот параметр не применяется к собраниям каналов.

|Значение параметра |Поведение  |
|---------|---------|
|**Включить его для всех**     | Все участники могут писать и просматривать сообщения чата. |
|**Отключить его для всех**     | Чат собрания отключен для всех участников.  |
|**Включите его для всех, кроме анонимных пользователей**     | Доступ на запись в чате собрания отключен только для анонимных участников.  |

После применения этой политики **чата в собраниях** к пользователям организатор не сможет переопределить эту политику с помощью **параметров собрания**.

Политика, применяемая к организатору собрания, может повлиять на других пользователей собрания. Например:

- Если у организатора есть параметр **"Чат на собраниях** ", для которого задано значение **Включить для всех** или **Включить его для всех, кроме анонимных пользователей**, будет применена отдельная политика пользователя, и все пользователи с установленным параметром **"Отключить для всех** " не смогут общаться в чате на собрании.
- Если у организатора есть параметр **"Чат на собраниях** ", для которого задано значение **Отключить его для всех**, применяется политика организатора, и никто не сможет общаться в чате на собрании.

## <a name="teams-qa"></a>Teams Q&A

Это политика для организатора. Этот параметр включает или отключает интерфейс "Вопросы & ответы" (Q&A).

Этот параметр применяется при создании или обновлении собрания организаторами. По умолчанию этот параметр отключен. Mer informasjon о [Q&A в собраниях Teams](/manage-qna-for-teams).

Параметр QnAEngagementMode управляет этой политикой в PowerShell. Q&A также можно изменить в Центре администрирования Teams.

|Значение параметра |Поведение  |
|---------|---------|
|**Вкл**     | Организаторы могут добавлять Q&A при создании собраний. |
|**Выкл**     | Организаторы не могут добавлять Q&A при создании собраний.  |

## <a name="meeting-reactions"></a>Реакции на собрания

По умолчанию реакции на собрания включены. Отключение реакций для пользователя не означает, что пользователь не сможет использовать реакции на собраниях, которые он планирует. Организатор собрания по-прежнему может включить реакции на странице параметров собрания независимо от параметра по умолчанию.

## <a name="enable-meeting-policy-settings"></a>Включение параметров политики собраний

Чтобы включить параметры политики собраний, можно использовать [Центр администрирования Teams](https://admin.teams.microsoft.com/policies/meetings) (**Политики собраний** > **Изменение политики участников** > **& гостей**) или командлет [Set-CsTeamsMeetingPolicy](/powershell/module/skype/set-csteamsmeetingpolicy) в Teams PowerShell. 

В этом примере мы используем PowerShell для изменения глобальной политики собраний, чтобы разрешить любому пользователю начинать собрание или присоединяться к нему.

```powershell
Set-CsTeamsMeetingPolicy -Identity Global -AutoAdmittedUsers "Everyone" -AllowAnonymousUsersToStartMeeting $True -AllowPSTNUsersToBypassLobby $True
```

После настройки политики необходимо применить ее к пользователям. Если вы изменили глобальную политику (по умолчанию для всей организации), она будет автоматически применяться к пользователям. Чтобы изменения политики вступили в силу, необходимо подождать не менее 4 часов, но это может занять до 24 часов.
