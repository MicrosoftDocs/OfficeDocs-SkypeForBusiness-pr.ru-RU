# <a name="meeting-policy-settings---participants--guests"></a>Параметры политики собраний: участники могут & гостей

<a name="bkmeetingparticipants"> </a>

Эти параметры контролируют, какие участники собрания должны ждать в "вестибюле", прежде чем их допустят к собранию, и уровень участия в нем.

- [Позволить анонимным пользователям начинать собрание](#let-anonymous-people-start-a-meeting)
- [Автоматически допустить людей](#automatically-admit-people)
- [Разрешить пользователям телефонного дозвона минуть "вести".](#allow-dial-in-users-to-bypass-the-lobby)
- [Включить субтитры в прямом эфире](#enable-live-captions)
- [Разрешение чата на собраниях](#allow-chat-in-meetings)

> [!NOTE]
>Параметры присоединиться к собранию зависят от настроек для каждой группы Teams и способа подключения. Если в вашей группе есть аудиоконференция и она используется для подключения, см. [эту видеоконференцию.](https://docs.microsoft.com/microsoftteams/audio-conferencing-in-office-365) Если в вашей группе Teams нет аудиоконференции, обратитесь к команде ["Присоединиться к собранию в Teams".](https://support.office.com/article/join-a-meeting-in-teams-1613bb53-f3fa-431e-85a9-d6a91e3468c9)

## <a name="let-anonymous-people-start-a-meeting"></a>Позволить анонимным пользователям начинать собрание

Этот параметр является политикой на каждого организатора, которая позволяет использовать собрания с бездережным телефонным и видеоконференцией. Этот параметр управляет возможностью присоединиться к собранию пользователям с телефонным доступом без участия пользователя, который проходит проверку подлинности из организации. По умолчанию этот параметр отключен, то есть пользователи, которые будут присоединяться к собранию по телефонной связи, будут ждать в "ожидании" до тех пор, пока к собранию не присоединится авторификация пользователя из организации.

> [!NOTE]
> Если этот параметр отключен и пользователь, присоединяюсь к собранию сначала с помощью телефонного номера, а затем помещается в ""вести", пользователь организации должен присоединиться к собранию с помощью клиента Teams, чтобы допустить пользователя из "вести". Для пользователей с набраным телефонным номером не доступны средства контроля "в".

## <a name="automatically-admit-people"></a>Автоматически допустить людей

Это политика на каждого организатора. Этот параметр управляет тем, будут ли пользователи присоединяться к собранию напрямую или ждать в "ожидании", пока они не будут допущены пользователем, который проходит проверку подлинности. Этот параметр не применяется к пользователям с телефонным доступом.

![Снимок экрана: собрание с пользователем в "вестибюле"](media/meeting-policies-lobby.png)

 Организаторы собраний могут щелкнуть **"Параметры** собрания" в приглашении на собрание, чтобы изменить этот параметр для каждого собрания, на который они запланют.

> [!NOTE]
> В параметрах собрания этот параметр имеет метку "Кто может минуть "вести". Если изменить параметр по умолчанию для любого пользователя, он будет применяться во всех новых собраниях, организованных этим пользователем, и во всех предыдущих собраниях, где пользователь не внося изменения в параметры собрания.
  
|Значение параметра  |Поведение при подмылеве |
|---------|---------|
|**Все**   |Все участники собрания присоединяются к собранию напрямую, не дожидаясь в зале ожидания. К ним относятся пользователи, проверенные подлинности, внешние пользователи из доверенных организаций (федератов), гости и анонимные пользователи.     |
|**Все в организации и федератированные организации**     |Пользователи, проверенные на подлинности в организации, включая гостевых пользователей и пользователей из надежных организаций, могут присоединиться к собранию прямо в зале ожидания.  Анонимные пользователи будут ждать в "ожидании".   |
|**Все в организации**    |Пользователи, которые проходит проверку подлинности в организации, в том числе гостевых, могут присоединиться к собранию прямо в зале ожидания.  Пользователи из надежных организаций и анонимные пользователи ждут в "ожидании". Это настройка по умолчанию.           |
|**Только организатор**    |Только организаторы собраний могут присоединиться к собранию напрямую, не дожидаясь в зале ожидания. Все остальные пользователи, включая пользователей, пройти проверку подлинности в организации, гостевых пользователей, пользователей из надежных организаций и анонимных пользователей, должны подождать в "ожидании".           |

## <a name="allow-dial-in-users-to-bypass-the-lobby"></a>Разрешить пользователям телефонного дозвона минуть "вести".

Это политика на каждого организатора. Этот параметр управляет тем, могут ли люди, которые присоединяются к собранию по телефону, напрямую или ждать в "ожидании" независимо от настройки параметра "Автоматически допустить **людей".** По умолчанию этот параметр отключен. Если этот параметр отключен, пользователи, которые присоединяются к собранию с клиентом Teams, будут ждать в "ожидании", пока пользователь организации не присоединится к собранию с клиентом Teams и не допустит их. Если этот параметр включен, пользователи, которые присоединяются к собранию по телефонной связи, автоматически присоединяются к собранию, когда пользователь организации присоединяется к собранию.

> [!NOTE]
> Если пользователь, присоединяясь к собранию с помощью телефонного дозвона, присоединяется к собранию до того, как пользователь организации присоединится к собранию, он будет помещиться в ""вести", пока пользователь организации не присоединится к собранию с помощью клиента Teams и не допустит его. Если изменить параметр по умолчанию для любого пользователя, он будет применяться во всех новых собраниях, организованных этим пользователем, и во всех предыдущих собраниях, где пользователь не внося изменения в параметры собрания.

## <a name="enable-live-captions"></a>Включить субтитры в прямом эфире

Этот параметр настраивается как политика "на пользователя", которая применяется во время собрания. Этот параметр управляет  возможностью включить субтитры в прямом эфире для пользователя, чтобы включить или отключить субтитры в трансляции собраний, которые посещает пользователь.  

![Снимок экрана: параметр "Включить живые субтитры"](media/meeting-policies-live-captions.png)

|Значение параметра |Поведение  |
|---------|---------|
|**Отключено, но пользователь может переопределять его**     | Автоматические субтитры не включались пользователем во время собрания. Пользователь видит параметр **"Включить живые субтитры"** в меню переполнения **(...),** чтобы включить их. Это настройка по умолчанию. |
|**Отключено**     | Субтитры в прямом эфире отключены для пользователя во время собрания. У пользователя нет возможности включить его.          |
