# <a name="set-up-parent-connection-in-microsoft-teams-for-education"></a>Настройка родительского подключения в Microsoft Teams для образования

Родительское подключение в Teams для образования помогает преподавателям безопасно общаться и взаимодействовать с родителями и опекунами учащихся в их группах класса с помощью Teams.

В этой статье содержатся рекомендации для ИТ-специалистов в области образования о требованиях и настройке родительского подключения.

## <a name="share-guardian-and-educator-resources"></a>Общий доступ к ресурсам опекуна и преподавателя

Ниже приведены некоторые ресурсы, которыми ИТ-администраторы могут поделиться с опекунами и преподавателями о том, как приступить к работе с родительским подключением.

- Инструкции по настройке опекунов см. [в статье Подключение с преподавателями в Teams](https://support.microsoft.com/topic/connect-with-educators-in-teams-ec2430c3-952a-4ba4-9891-1d1cab577960).
- Инструкции по настройке преподавателей см. [в статье Взаимодействие с опекунами в Microsoft Teams](https://support.microsoft.com/topic/communicate-with-guardians-in-microsoft-teams-01471ecd-eb5d-4eda-9c5d-0064d672960e?ui=en-us&rs=en-us&ad=us).

## <a name="benefits-of-parent-connection"></a>Преимущества родительского подключения

Родительское подключение позволяет преподавателям и опекунам общаться, отправлять сообщения электронной почты и звонить с помощью Teams.

- Преподаватели могут инициировать чаты с опекунами.
  - Если у опекуна нет учетной записи потребителя Teams или он еще не присоединился к Teams, он получит сообщение от преподавателя вместе с приглашением перейти в Teams по электронной почте. Это применимо только в тех случаях, когда ограничения приглашений не были достигнуты, а чат является новым или существующим чатом, который был повторно введен из родительского подключения.
- Он работает с защищенным чатом. Дополнительные сведения см. [в статье Использование защищенных чатов в Microsoft Teams](supervise-chats-edu.md).
  - По умолчанию опекуны имеют ограниченные разрешения, поэтому они не могут общаться с учащимися или удалять пользователей из чатов.
  - Этот параметр может быть изменен администратором клиента.
- Преподаватели могут щелкнуть адрес электронной почты опекуна, чтобы отправить им электронное письмо с помощью собственного почтового клиента.
- Преподаватели могут щелкнуть номер телефона опекуна, чтобы позвонить им в Teams.

> [!IMPORTANT]
> Чтобы щелкнуть, чтобы вызвать функциональные возможности в Teams, вашему клиенту необходимо:
>
> - Возможности УАТС.
> - Подключение к ТСОП.
>
> Планы Microsoft 365 A1 и A3 не включают возможности УАТС и подключения по ТСОП. Для [каждой из них можно приобрести лицензии на надстройки](/microsoftteams/teams-add-on-licensing/microsoft-teams-add-on-licensing).
>
> Microsoft 365 A5 планы включают только возможности УАТС с помощью телефонной системы Teams. Вам [по-прежнему потребуется приобрести план звонков Teams или использовать стороннее решение](pstn-connectivity.md) для подключения к внешним номерам в ТСОП.
>
> Дополнительные сведения обо всех вариантах подключения по ТСОП см. в разделе [Параметры подключения по ТСОП](pstn-connectivity.md).
>
> Дополнительные сведения о лицензировании звонков в Teams см. в разделе [Параметры лицензирования надстроек Teams](/microsoftteams/teams-add-on-licensing/microsoft-teams-add-on-licensing).

## <a name="requirements"></a>Требования

Вам необходимо использовать Microsoft Graph или school Data Sync (SDS) для заполнения контактных данных, связанных с родителями и опекунами каждого учащегося.

### <a name="graph-api"></a>API Graph

Если вы уже используете [пакет SDK Для Microsoft Graph PowerShell](/powershell/microsoftgraph/overview) для создания удостоверений учащихся, вы можете легко включить [тип ресурса relatedContact](/graph/api/resources/relatedcontact).

### <a name="school-data-sync"></a>Синхронизация данных учебного заведения

Контактные данные опекуна Teams остаются актуальными в SIS с помощью school data sync (SDS), когда SDS настроена на регулярную синхронизацию.

Если опекун удален из записей *учащегося* , все существующие чаты с их участием будут содержать баннер, видимый владельцу чата. Этот баннер сделает владельца чата осведомленным об изменениях с просьбой удалить опекуна из чата. Корпорация Майкрософт не будет автоматически обновлять членство в чате, чтобы удалить опекуна.

- Для заполнения **контактных данных, связанных с** родителями и опекунами каждого учащегося, требуется school Data Sync (SDS).
  - [Развертывание SDS](/schooldatasync/parents-and-guardians-in-sds)

- Если вам нужна помощь в настройке SDS и заполнении контактов, **связанных с** родителями и опекунами, для учащихся в клиенте, обратитесь в группу EDU Customer Success по следующим причинам:
  - Завершение процесса RFA в [FastTrack](https://www.microsoft.com/fasttrack?rtc=1).
  - Открытие запроса в [службу поддержки](https://aka.ms/sdssupport).

- В настоящее время SDS поддерживает только прием данных на основе CSV для родительских контактов. Однако вы можете использовать [PowerSchool API Sync](/schooldatasync/how-to-deploy-school-data-sync-by-using-powerschool-sync) или [OneRoster API Sync](/schooldatasync/how-to-deploy-school-data-sync-by-using-oneroster-sync) для всех данных списка, а также просто добавить родительские контакты с помощью CSV-файла.
  - Создайте второй профиль синхронизации с помощью [формата синхронизации SDS v1 CSV](/schooldatasync/school-data-sync-format-csv-files-for-sds).
  - Извлеките два заполненных [родительских файла](/schooldatasync/parent-contact-sync-file-format) с остальными файлами версии 1 пустыми (только заголовки).
    - User.csv
    - Guardianrelationship.csv
      - Значение *Роли* должно быть заполнено для каждого родителя и опекуна, чтобы указать, являются ли они *родителем* или *опекуном*.
  - Чтобы просмотреть пример набора CSV-файлов версии 1, см. статью [Минимальные обязательные атрибуты файлов GitHub](https://github.com/OfficeDev/O365-EDU-Tools/tree/master/CSV%20Samples/SDS%20Format/Min%20Required%20Attributes).
  - Если вы хотите автоматизировать извлечение CSV-файлов после начальной синхронизации, прочитайте [наш документ Синхронизация файлов автоматизации CSV](/schooldatasync/csv-file-sync-automation).
  - Чтобы получить помощь по настройке синхронизации данных SDS, обратитесь к [нашей команде успешного выполнения](https://www.microsoft.com/fasttrack?rtc=1) или [отправьте запрос в службу поддержки](https://edusupport.microsoft.com/support?product_id=data_sync).

### <a name="teams-admin-center-policies"></a>Политики Центра администрирования Teams

- У владельцев команд класса должен быть включен чат Teams.
- Владельцы команд класса должны иметь внешний доступ с **учетными записями Teams, которые не управляются включенной организацией** .
  - Это необходимо включить на уровне клиента и пользователя. Параметр уровня клиента можно найти в разделе **Пользователи > внешний доступ** в Центре администрирования Teams. Доступ к этому параметру также можно получить через PowerShell. Доступ к политикам внешнего доступа на уровне пользователя можно получить только через PowerShell. Дополнительные рекомендации см. в приведенных ниже командах PowerShell.

#### <a name="parent-and-guardian-restrictions"></a>Ограничения для родителей и опекунов

Родители и опекуны классифицируются как *внешние пользователи* в родительском подключении, что означает, что у них нет полных прав клиента. У них есть доступ только к чату или чатам, в которые они входят, а также к файлам, изображениям и другому содержимому, к которым предоставлен общий доступ в чате.

Для внешних чатов внутренние и внешние пользователи могут добавлять пользователей в чат. Дополнительные сведения о внешнем чате см. в статье [Управление внешними собраниями и чатами в Microsoft Teams](manage-external-access.md).

Кроме того, внешние пользователи могут видеть сведения о присутствии (в автономном режиме, доступности, занятости и т. д.) пользователей вашей организации, но это можно отключить с помощью PowerShell для защиты конфиденциальности пользователей. В PowerShell используйте [Set-CsPrivacyConfiguration](/powershell/module/skype/set-csprivacyconfiguration) и задайте .``EnablePrivacyMode=true``

Несмотря на то, что родители и опекуны являются внешними пользователями, их вклад в чаты можно обнаружить. Узнайте, как провести исследование обнаружения электронных данных Teams, см. статью [Исследование содержимого в Microsoft Teams](ediscovery-investigation.md).

> [!IMPORTANT]
> ИТ-администраторы должны ознакомить всех владельцев классов с рекомендациями по обмену информацией об учащихся в чате, включая риски для конфиденциальности учащихся.

#### <a name="blocking-a-parent-or-guardian-in-a-chat"></a>Блокировка родителя или опекуна в чате

Преподаватели могут заблокировать опекуна в чате, инициированном в родительском подключении.

Владелец класса может:

1. Откройте карточку профиля опекуна, выберите многоточие и **заблокировать пользователя**.
2. Затем удалите опекуна из чата.

Заблокированный пользователь не сможет начать дополнительные чаты с владельцем класса.

## <a name="allow-external-access-with-teams-accounts-not-managed-by-an-organization"></a>Разрешить внешний доступ с помощью учетных записей Teams, не управляемых организацией

Чтобы позволить преподавателям общаться с родителями и опекунами в Teams, ИТ-администратору клиента для образовательных учреждений необходимо обновить политики клиента, чтобы разрешить внешний доступ для учетных записей Teams за пределами клиента. Дополнительные сведения об управлении внешним доступом см. [в статье Управление внешним доступом в Microsoft Teams](manage-external-access.md).

Ниже описано, как включить внешний доступ для родителей и опекунов.

1. Установите последнюю предварительную версию модуля Microsoft Teams PowerShell.

    ```powershell
    Install-Module -Name PowerShellGet -Force -AllowClobber
    Install-Module -Name MicrosoftTeams -AllowPrerelease -Force –AllowClobber
    ```

2. Используя учетные данные с правами администратора, выполните следующие команды:

    ```powershell
    $credential = Get-Credential
    Connect-MicrosoftTeams -Credential $credential
    ```

    Параметр политики, который включает внешний доступ с учетными записями Teams, не управляемыми организацией на уровне пользователя (`EnableTeamsConsumerAccess`), по умолчанию включен для всех политик внешнего доступа на уровне пользователя. Чтобы у пользователя был внешний доступ к учетным записям Teams, не управляемым организацией, необходимо включить параметры политики на уровне клиента и политики на уровне пользователя. Если вы не хотите, чтобы этот доступ был включен для всех пользователей в клиенте, убедитесь, что параметр на уровне клиента отключен, обновите назначенные пользователям политики внешнего доступа на уровне пользователя, а затем включите параметр на уровне клиента.

    Чтобы проверить, какие политики внешнего доступа на уровне пользователя существуют и кому они назначены, выполните следующие действия.

3. Убедитесь, что существуют политики внешнего доступа на уровне пользователя.

    ```powershell
    Get-CsExternalAccessPolicy
    ```

4. Для каждой политики, отличной от политики "Глобальная", проверьте, каким пользователям назначена политика.

   > [!NOTE]
   > Все пользователи, которым не назначена определенная политика, будут возвращаться к глобальной политике. Всем новым пользователям, добавленным в клиент, будет назначена глобальная политика.

    ```powershell
    Get-CsOnlineUser -Filter {ExternalAccessPolicy -eq "<PolicyName>"} | Select-Object DisplayName,ObjectId,UserPrincipalName
    ```

Так как все политики внешнего доступа на уровне пользователя по умолчанию имеют `EnableTeamsConsumerAccess` значение true, если вы хотите настроить `EnableTeamsConsumerAccess` параметр для определенных пользователей, можно создать или изменить существующие политики внешнего доступа с измененными параметрами и (или) переназначить пользователей на новые или существующие политики с помощью следующих командлетов PowerShell:

- Создание политики внешнего доступа: [New-CsExternalAccessPolicy](/powershell/module/skype/new-csexternalaccesspolicy)

- Настройка существующей политики внешнего доступа (включая глобальную политику): [Set-CsExternalAccessPolicy](/powershell/module/skype/set-csexternalaccesspolicy)

> [!NOTE]
> Нельзя изменить следующие политики подписки по умолчанию: "FederationAndPICDefault", "FederationOnly", "NoFederationAndPIC". Политика FederationAndPICDefault была назначена всем пользователям по умолчанию, однако теперь новым пользователям назначается политика "Глобальная" по умолчанию. Если вам нужно изменить параметры политики для пользователей, которым назначены эти политики подписки по умолчанию, назначьте этим пользователям различные политики с правильными параметрами.

- Назначение политики внешнего доступа одному пользователю: [Grant-CsExternalAccessPolicy](/powershell/module/skype/grant-csexternalaccesspolicy)

- Назначение политики набору пользователей: [New-CsBatchPolicyAssignmentOperation](/powershell/module/teams/new-csbatchpolicyassignmentoperation)

После правильной настройки политик внешнего доступа на уровне пользователя для пользователей в клиенте можно включить параметр уровня клиента (`AllowTeamsConsumer`) для клиента с помощью следующего командлета:

- Настройка параметров конфигурации федерации для клиента: [Set-CsTenantFederationConfiguration](/powershell/module/skype/set-cstenantfederationconfiguration)

## <a name="turn-on-the-parents-app-in-the-teams-admin-center"></a>Включение приложения "Родители" в Центре администрирования Teams

Приложение "Родители" по умолчанию отключено, поэтому владельцы команд класса не будут видеть его в своих группах классов, пока оно не будет разрешено через Центр администрирования Teams. Приложение "Родители" включается в Центре администрирования Teams с помощью [параметра Разрешить приложения, заблокированные разработчиками](manage-apps.md#allow-and-block-apps).

В любое время приложение можно отключить на уровне клиента с помощью [параметра Разрешить и заблокировать приложения](manage-apps.md#allow-and-block-apps) в Центре администрирования Teams. Если она отключена на уровне клиента, она будет заблокирована для всех пользователей, даже если включены разрешения уровня пользователя.

Приложение "Родители" также можно отключить на уровне пользователя с помощью [управления политиками разрешений приложений в Microsoft Teams](teams-app-permission-policies.md).
