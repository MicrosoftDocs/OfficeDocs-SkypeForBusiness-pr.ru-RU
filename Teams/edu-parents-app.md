# <a name="set-up-parent-connection-in-microsoft-teams-for-education"></a>Настройка родительского подключения в Microsoft Teams для образования

Родительское подключение в Teams для образования помогает преподавателям безопасно подключаться и взаимодействовать с родителями и защитниками учащихся в своих командах класса с помощью Teams.

В этой статье содержатся рекомендации для ИТ-специалистов по образовательным технологиям по требованиям и настройке родительского подключения.

## <a name="share-guardian-and-educator-resources"></a>Предоставление общего доступа к ресурсам защитника и преподавателя

Ниже приведены некоторые ресурсы, которые ИТ-администраторы могут предоставлять защитникам и преподавателям, чтобы узнать, как приступить к работе с родительским подключением.

- Инструкции по настройке защитников см. Подключение [с преподавателями в Teams](https://support.microsoft.com/topic/connect-with-educators-in-teams-ec2430c3-952a-4ba4-9891-1d1cab577960).
- Инструкции по настройке преподавателей см. в разделе "Общение с защитниками[" Microsoft Teams](https://support.microsoft.com/topic/communicate-with-guardians-in-microsoft-teams-01471ecd-eb5d-4eda-9c5d-0064d672960e?ui=en-us&rs=en-us&ad=us).

## <a name="benefits-of-parent-connection"></a>Преимущества родительского подключения

Подключение "Родители" позволяет преподавателям и опекунам общаться в чате, отправлять сообщения электронной почты и вызывать их с помощью Teams.

- Teams контактных данных защиты остаются актуальными в SIS с помощью Синхронизация сведений о школе (SDS).
- Он работает с защищенным чатом. Дополнительные сведения см. в статье ["Использование защищенных чатов в Microsoft Teams](supervise-chats-edu.md)".
  - По умолчанию у защитников есть ограниченные разрешения, поэтому они не могут общаться со учащимися или удалять пользователей из чатов.
  - Этот параметр может изменить администратор клиента.
- Преподаватели могут инициировать беседы с защитниками.
  - Если у защитника нет учетной записи Teams потребителя, он получит начальное сообщение от преподавателя и приглашение по электронной почте перейти Teams.
- Преподаватели могут щелкнуть сообщение электронной почты защитника, чтобы отправить его по электронной почте с помощью собственного почтового клиента.
- Преподаватели могут щелкнуть номер телефона опекуна, чтобы позвонить ему в Teams.

> [!IMPORTANT]
> Чтобы щелкнуть функцию вызова в Teams, клиенту требуется:
>
> - Возможности общедоступной Exchange (УАТС).
> - Подключение к ТСОП.
>
> Microsoft 365 A1 и планы A3 не включают возможности УАТС и подключения по ТСОП. Вы можете [приобрести лицензии на надстройки для каждой из них](/teams-add-on-licensing/microsoft-teams-add-on-licensing).
>
> Microsoft 365 A5 планы включают только возможности УАТС, использующие Teams телефонная система. Вам по-прежнему потребуется приобрести Teams звонков или использовать стороннее решение для подключения к внешним номерам в ТСОП.[](pstn-connectivity.md)
>
> Дополнительные сведения о всех вариантах подключения по ТСОП см. в разделе "Параметры подключения по [ТСОП"](pstn-connectivity.md).
>
> Дополнительные сведения о Teams лицензировании вызовов см. [Teams варианты лицензирования надстроек](/teams-add-on-licensing/microsoft-teams-add-on-licensing).

- Владельцы команд классов должны включить Teams чате.
- Владельцы команд классов должны иметь внешний доступ **Teams учетных записей, не управляемых организацией**.
  - Этот параметр должен быть включен на уровне клиента и пользователя. Параметр уровня клиента можно найти в разделе "Пользователи **> внешнего доступа**" Teams центре администрирования. Доступ к этому параметру также можно получить с помощью PowerShell. Доступ к политикам внешнего доступа на уровне пользователя можно получить только с помощью PowerShell. Дополнительные рекомендации см. в приведенных ниже командах PowerShell.

#### <a name="parent-and-guardian-restrictions"></a>Ограничения родительских и опекунов
Родители и опекуны классифицируются как внешние пользователи в связи с родительскими элементами, то есть у них нет полных прав клиента. У них есть доступ только к чату или чатам, в которые они добавлены, а также к файлам, изображениям и другому содержимому, к котором они предоставили общий доступ в чате.

Кроме того, внешние пользователи могут видеть сведения о присутствии (автономных, доступных, занятых и т. д.) пользователей вашей организации, но их можно отключить с помощью PowerShell для защиты конфиденциальности пользователей. В PowerShell используйте [Set-CsPrivacyConfiguration](/powershell/module/skype/set-csprivacyconfiguration) и задайте .``EnablePrivacyMode=true``

Несмотря на то что родители и опекуны являются внешними пользователями, их участие в чатах можно обнаружить. Узнайте, как провести Teams обнаружения электронных данных, прочитав статью "Проведение исследования [eDiscovery содержимого в](ediscovery-investigation.md) Microsoft Teams".

#### <a name="blocking-a-parent-or-guardian-in-a-chat"></a>Блокировка родительского или опекуна в чате
Преподаватели могут блокировать защитника в чате, инициированном в родительском соединении.

Владелец класса может:

1. Откройте карточку профиля защиты, нажмите кнопку с многоточием и **заблокировать пользователя**.
2. Затем удалите защитника из чата.

Заблокированный пользователь не сможет начать дополнительные чаты с владельцем класса.

## <a name="allow-external-access-with-teams-accounts-not-managed-by-an-organization"></a>Разрешение внешнего доступа с Teams учетными записями, не управляемыми организацией

Чтобы разрешить преподавателям взаимодействовать с родителями и опекунами в Teams, ИТ-администратору клиента для образовательных учреждений необходимо обновить политики клиента, чтобы разрешить внешний доступ Teams учетным записям за пределами клиента. Дополнительные сведения об управлении внешним доступом см. в разделе "Управление внешним [доступом" Microsoft Teams](manage-external-access.md).

Ниже приведены шаги по включению внешнего доступа для родителей и опекунов.

1. Установите последнюю версию Microsoft Teams PowerShell.

    ```powershell
    Install-Module -Name PowerShellGet -Force -AllowClobber
    Install-Module -Name MicrosoftTeams -AllowPrerelease -Force –AllowClobber
    ```

2. Используя учетные данные с правами администратора, выполните следующие команды:

    ```powershell
    $credential = Get-Credential
    Connect-MicrosoftTeams -Credential $credential
    ```

    Параметр политики, который включает внешний доступ с Teams учетными записями, не управляемыми организацией на уровне пользователя (`EnableTeamsConsumerAccess`), по умолчанию включен для всех политик внешнего доступа на уровне пользователя. Чтобы Teams у пользователя был внешний доступ с учетными записями, не управляемыми организацией, необходимо включить параметры уровня клиента и политики на уровне пользователя. Если вы не хотите, чтобы этот доступ был включен для каждого пользователя в клиенте, убедитесь, что параметр уровня клиента отключен, обновите политики внешнего доступа на уровне пользователя, назначенные пользователям, а затем включите параметр на уровне клиента.

    Чтобы проверить, какие политики внешнего доступа на уровне пользователя существуют и кому они назначены, выполните следующие действия.

3. Убедитесь, что существуют политики внешнего доступа на уровне пользователя.

    ```powershell
    Get-CsExternalAccessPolicy
    ```

4. Для каждой политики, отличной от глобальной, проверьте, какие пользователи назначены политике.

   > [!NOTE]
   > Все пользователи, которым не назначена определенная политика, будут использовать глобальную политику. Новым пользователям, добавленным в клиент, будет назначена глобальная политика.

    ```powershell
    Get-CsOnlineUser -Filter {ExternalAccessPolicy -eq "<PolicyName>"} | Select-Object DisplayName,ObjectId,UserPrincipalName
    ```

`EnableTeamsConsumerAccess` Так как все политики внешнего доступа на уровне пользователя по умолчанию имеют значение true, `EnableTeamsConsumerAccess` если вы хотите настроить параметр для конкретных пользователей, можно создать или изменить существующие политики внешнего доступа с помощью скорректированных параметров и (или) переназначить пользователей новым или существующим политикам с помощью следующих командлетов PowerShell:

- Создание политики внешнего доступа [: New-CsExternalAccessPolicy](/powershell/module/skype/new-csexternalaccesspolicy)

- Настройка существующей внешней политики доступа (включая глобальную): [Set-CsExternalAccessPolicy](/powershell/module/skype/set-csexternalaccesspolicy)

> [!NOTE]
> Нельзя изменить следующие политики подписки по умолчанию: FederationAndPICDefault, FederationOnly, NoFederationAndPIC. Политика FederationAndPICDefault была назначена всем пользователям по умолчанию, однако новым пользователям теперь по умолчанию назначается глобальная политика. Если необходимо изменить параметры политики для пользователей, которым назначены эти политики подписки по умолчанию, назначьте этим пользователям различные политики с правильными параметрами.

- Назначение политики внешнего доступа одному пользователю: [Grant-CsExternalAccessPolicy](/powershell/module/skype/grant-csexternalaccesspolicy)

- Назначение политики набору пользователей: [New-CsBatchPolicyAssignmentOperation](/powershell/module/teams/new-csbatchpolicyassignmentoperation)

После правильной настройки политик внешнего доступа на уровне пользователя для пользователей в клиенте можно включить параметр уровня клиента (`AllowTeamsConsumer`) с помощью следующего командлета:

- Настройка параметров конфигурации федерации для клиента: [Set-CsTenantFederationConfiguration](/powershell/module/skype/set-cstenantfederationconfiguration)

## <a name="turn-on-the-parents-app-in-the-teams-admin-center"></a>Включение приложения "Родители" в центре Teams администрирования

Приложение "Родители" по умолчанию отключено, поэтому владельцы группы классов не будут видеть его в своих командах класса, пока это не будет разрешено через Teams центре администрирования. Приложение "Родители" включено в центре администрирования Teams с помощью разрешения [приложений, заблокированных издателями](manage-apps.md#apps-blocked-by-publishers).

В любое время приложение можно отключить на уровне клиента с помощью разрешения и блокировки [](manage-apps.md#allow-and-block-apps) приложений в Teams администрирования. Если она отключена на уровне клиента, она будет заблокирована для всех пользователей, даже если разрешения на уровне пользователя включены.

Приложение "Родители" также можно отключить на уровне пользователя с помощью политик управления разрешениями приложений [в Microsoft Teams](teams-app-permission-policies.md).
