# <a name="set-up-parent-connection-in-microsoft-teams-for-education"></a>Настройка родительского подключения в Microsoft Teams для образования

Родительское подключение в Teams для образования помогает преподавателям безопасно подключаться и взаимодействовать с родителями и опекунами учащихся в своих командах класса с помощью Teams.

В этой статье содержатся рекомендации для ИТ-специалистов по образовательным технологиям по требованиям и настройке родительского подключения.

## <a name="share-guardian-and-educator-resources"></a>Предоставление общего доступа к ресурсам защитника и преподавателя

Ниже приведены некоторые ресурсы, которые ИТ-администраторы могут предоставлять защитникам и преподавателям, чтобы узнать, как приступить к работе с родительским подключением.

- Инструкции по настройке защитников см. в статье ["Подключение к преподавателям в Teams"](https://support.microsoft.com/topic/connect-with-educators-in-teams-ec2430c3-952a-4ba4-9891-1d1cab577960).
- Инструкции по настройке преподавателей см. в статье ["Общение с защитниками" в Microsoft Teams](https://support.microsoft.com/topic/communicate-with-guardians-in-microsoft-teams-01471ecd-eb5d-4eda-9c5d-0064d672960e?ui=en-us&rs=en-us&ad=us).

## <a name="benefits-of-parent-connection"></a>Преимущества родительского подключения

Подключение "Родители" позволяет преподавателям и опекунам общаться в чате, отправлять сообщения электронной почты и вызывать их с помощью Teams.

- Преподаватели могут инициировать беседы с защитниками.
  - Если у защитника нет учетной записи потребителя Teams или он еще не присоединился к Teams, он получит сообщение от преподавателя вместе с приглашением по электронной почте перейти в Teams. Это применимо только в тех случаях, когда ограничения на приглашение не были достигнуты, а чат — это новый чат или существующий чат, который был повторно введен из родительского подключения.
- Он работает с защищенным чатом. Дополнительные сведения см. в статье ["Использование защищенных чатов в Microsoft Teams"](supervise-chats-edu.md).
  - По умолчанию у защитников есть ограниченные разрешения, поэтому они не могут общаться со учащимися или удалять пользователей из чатов.
  - Этот параметр может изменить администратор клиента.
- Преподаватели могут щелкнуть сообщение электронной почты защитника, чтобы отправить его по электронной почте с помощью собственного почтового клиента.
- Преподаватели могут щелкнуть номер телефона опекуна, чтобы позвонить ему в Teams.

> [!IMPORTANT]
> Чтобы щелкнуть функцию вызова в Teams, клиенту необходимо:
>
> - Возможности обмена общедоступными филиалами (PBX).
> - Подключение к ТСОП.
>
> Microsoft 365 A1 и планы A3 не включают возможности УАТС и подключения по ТСОП. Вы можете [приобрести лицензии на надстройки для каждой из них](/microsoftteams/teams-add-on-licensing/microsoft-teams-add-on-licensing).
>
> Microsoft 365 A5 планы включают только возможности УАТС с помощью телефонной системы Teams. Вам по-прежнему потребуется приобрести план звонков [Teams](pstn-connectivity.md) или использовать стороннее решение для подключения к внешним номерам в ТСОП.
>
> Дополнительные сведения о всех вариантах подключения по ТСОП см. в разделе "Параметры подключения по [ТСОП"](pstn-connectivity.md).
>
> Дополнительные сведения о лицензировании вызовов Teams см. в разделе о вариантах лицензирования [надстроек Teams](/microsoftteams/teams-add-on-licensing/microsoft-teams-add-on-licensing).

## <a name="requirements"></a>Требования

Для заполнения контактных данных, связанных с родительским и опекуном каждого учащегося, необходимо использовать Microsoft Graph или School Data Sync (SDS).

### <a name="graph-api"></a>API Graph

Если вы уже используете [пакет SDK Microsoft Graph PowerShell](/powershell/microsoftgraph/overview) для создания удостоверений учащихся, вы можете легко включить [тип ресурса relatedContact](/graph/api/resources/relatedcontact).

### <a name="school-data-sync"></a>School Data Sync

Контактные данные защитника Teams остаются актуальными в SIS с помощью School Data Sync (SDS), если SDS настроен на регулярное синхронизацию.

Если защитник удален из записей учащегося, все существующие чаты, включаемые в них, будут содержать баннер, видимый владельцу чата. С помощью этого баннера владелец чата узнает об изменении и запросит удалить защитника из чата. Корпорация Майкрософт не будет автоматически обновлять членство в чате, чтобы удалить защитника.

- Для заполнения контактных данных, связанных с родительским и опекуном каждого учащегося,  требуется school Data Sync (SDS).
  - [Развертывание SDS](/schooldatasync/parents-and-guardians-in-sds)

- Если вам нужна помощь в настройке служб SDS и заполнении родительских контактов и контактов, связанных с опекунами, для учащихся в клиенте, обратитесь в службу поддержки клиентов EDU, выполнив следующие действия:
  - Завершение процесса RFA в [FastTrack](https://www.microsoft.com/fasttrack?rtc=1).
  - Открытие запроса в [службу поддержки](https://aka.ms/sdssupport).

- В настоящее время SDS поддерживает только прием данных на основе CSV для родительских контактов. Однако для всех данных списка можно использовать синхронизацию [API PowerSchool](/schooldatasync/how-to-deploy-school-data-sync-by-using-powerschool-sync) или [OneRoster](/schooldatasync/how-to-deploy-school-data-sync-by-using-oneroster-sync) , а также просто добавить родительские контакты с помощью CSV.
  - Создайте второй профиль синхронизации, используя формат [синхронизации CSV-файлов SDS версии 1](/schooldatasync/school-data-sync-format-csv-files-for-sds).
  - Извлеките два заполненных [родительских файла](/schooldatasync/parent-contact-sync-file-format) с пустыми остальными файлами версии 1 (только заголовками).
    - User.csv
    - Guardianrelationship.csv
  - Чтобы просмотреть пример набора CSV-файлов версии 1, ознакомьтесь с файлами [GitHub](https://github.com/OfficeDev/O365-EDU-Tools/tree/master/CSV%20Samples/SDS%20Format/Min%20Required%20Attributes) с минимальными обязательными атрибутами.
  - Если вы хотите автоматизировать извлечение CSV-файлов после начальной синхронизации, ознакомьтесь с документом Синхронизация файлов [службы автоматизации](/schooldatasync/csv-file-sync-automation).
  - Чтобы получить помощь в настройке синхронизации данных SDS, обратитесь [](https://www.microsoft.com/fasttrack?rtc=1) в службу поддержки клиентов или [откройте запрос в службу поддержки](https://edusupport.microsoft.com/support?product_id=data_sync).

### <a name="teams-admin-center-policies"></a>Политики Центра администрирования Teams

- Владельцы команд классов должны включить чат Teams.
- Владельцы команд классов должны иметь внешний доступ к учетным записям **Teams, не управляемым организацией** .
  - Этот параметр должен быть включен на уровне клиента и пользователя. Параметр уровня клиента можно найти в разделе "Пользователи **> внешнего доступа"** в Центре администрирования Teams. Доступ к этому параметру также можно получить с помощью PowerShell. Доступ к политикам внешнего доступа на уровне пользователя можно получить только с помощью PowerShell. Дополнительные рекомендации см. в приведенных ниже командах PowerShell.

#### <a name="parent-and-guardian-restrictions"></a>Ограничения родительских и опекунов

Родители и опекуны классифицируются как внешние *пользователи* в связи с родительскими элементами, то есть у них нет полных прав клиента. У них есть доступ только к чату или чатам, в которые они в составе, а также к файлам, изображениям и другому содержимому, совместно используемым в чате.

Для внешних чатов внутренние и внешние пользователи могут добавлять пользователей в чат. Дополнительные сведения о внешнем чате см. в статье "Управление внешними собраниями [и чатами в Microsoft Teams"](manage-external-access.md).

Кроме того, внешние пользователи могут видеть сведения о присутствии (в автономном режиме, доступности, занятости и т. д.) пользователей вашей организации, но их можно отключить с помощью PowerShell для защиты конфиденциальности пользователей. В PowerShell используйте [Set-CsPrivacyConfiguration](/powershell/module/skype/set-csprivacyconfiguration) и задайте .``EnablePrivacyMode=true``

Несмотря на то, что родители и опекуны являются внешними пользователями, их вклад в чаты можно обнаружить. Узнайте, как провести исследование обнаружения электронных данных Teams, прочитав статью "Проведение исследования электронных данных" о содержимом [в Microsoft Teams](ediscovery-investigation.md).

> [!IMPORTANT]
> ИТ-администраторы должны информировать всех владельцев классов о рекомендациях по обмену сведениями об учащихся через чат, включая риски для конфиденциальности учащихся.

#### <a name="blocking-a-parent-or-guardian-in-a-chat"></a>Блокировка родительского или опекуна в чате

Преподаватели могут блокировать защитника в чате, инициированном в родительском соединении.

Владелец класса может:

1. Откройте карточку профиля защиты, нажмите кнопку с многоточием и **заблокировать пользователя**.
2. Затем удалите защитника из чата.

Заблокированный пользователь не сможет начать дополнительные чаты с владельцем класса.

## <a name="allow-external-access-with-teams-accounts-not-managed-by-an-organization"></a>Разрешение внешнего доступа с учетными записями Teams, не управляемыми организацией

Чтобы разрешить преподавателям взаимодействовать с родителями и опекунами в Teams, ИТ-администратору клиента для образовательных учреждений необходимо обновить политики клиента, чтобы разрешить внешний доступ к учетным записям Teams за пределами клиента. Дополнительные сведения об управлении внешним доступом см. в разделе ["Управление внешним доступом в Microsoft Teams"](manage-external-access.md).

Ниже приведены шаги по включению внешнего доступа для родителей и опекунов.

1. Установите последнюю предварительную версию модуля Microsoft Teams PowerShell.

    ```powershell
    Install-Module -Name PowerShellGet -Force -AllowClobber
    Install-Module -Name MicrosoftTeams -AllowPrerelease -Force –AllowClobber
    ```

2. Используя учетные данные с правами администратора, выполните следующие команды:

    ```powershell
    $credential = Get-Credential
    Connect-MicrosoftTeams -Credential $credential
    ```

    Параметр политики, который включает внешний доступ к учетным записям Teams, не управляемым организацией на уровне пользователя (`EnableTeamsConsumerAccess`), по умолчанию включен для всех политик внешнего доступа на уровне пользователя. Чтобы у пользователя был внешний доступ к учетным записям Teams, не управляемым организацией, необходимо включить параметры уровня клиента и политики на уровне пользователя. Если вы не хотите, чтобы этот доступ был включен для каждого пользователя в клиенте, убедитесь, что параметр уровня клиента отключен, обновите политики внешнего доступа на уровне пользователя, назначенные пользователям, а затем включите параметр на уровне клиента.

    Чтобы проверить, какие политики внешнего доступа на уровне пользователя существуют и кому они назначены, выполните следующие действия.

3. Убедитесь, что существуют политики внешнего доступа на уровне пользователя.

    ```powershell
    Get-CsExternalAccessPolicy
    ```

4. Для каждой политики, отличной от глобальной, проверьте, какие пользователи назначены политике.

   > [!NOTE]
   > Все пользователи, которым не назначена определенная политика, будут использовать глобальную политику. Новым пользователям, добавленным в клиент, будет назначена глобальная политика.

    ```powershell
    Get-CsOnlineUser -Filter {ExternalAccessPolicy -eq "<PolicyName>"} | Select-Object DisplayName,ObjectId,UserPrincipalName
    ```

`EnableTeamsConsumerAccess` Так как все политики внешнего доступа на уровне пользователя по умолчанию имеют значение true, `EnableTeamsConsumerAccess` если вы хотите настроить параметр для конкретных пользователей, можно создать или изменить существующие политики внешнего доступа с помощью скорректированных параметров и (или) переназначить пользователей новым или существующим политикам с помощью следующих командлетов PowerShell:

- Создание политики внешнего доступа [: New-CsExternalAccessPolicy](/powershell/module/skype/new-csexternalaccesspolicy)

- Настройка существующей внешней политики доступа (включая глобальную): [Set-CsExternalAccessPolicy](/powershell/module/skype/set-csexternalaccesspolicy)

> [!NOTE]
> Нельзя изменить следующие политики подписки по умолчанию: FederationAndPICDefault, FederationOnly, NoFederationAndPIC. Политика FederationAndPICDefault была назначена всем пользователям по умолчанию, однако новым пользователям теперь по умолчанию назначается глобальная политика. Если необходимо изменить параметры политики для пользователей, которым назначены эти политики подписки по умолчанию, назначьте этим пользователям различные политики с правильными параметрами.

- Назначение политики внешнего доступа одному пользователю: [Grant-CsExternalAccessPolicy](/powershell/module/skype/grant-csexternalaccesspolicy)

- Назначение политики набору пользователей: [New-CsBatchPolicyAssignmentOperation](/powershell/module/teams/new-csbatchpolicyassignmentoperation)

После правильной настройки политик внешнего доступа на уровне пользователя для пользователей в клиенте можно включить параметр уровня клиента (`AllowTeamsConsumer`) с помощью следующего командлета:

- Настройка параметров конфигурации федерации для клиента: [Set-CsTenantFederationConfiguration](/powershell/module/skype/set-cstenantfederationconfiguration)

## <a name="turn-on-the-parents-app-in-the-teams-admin-center"></a>Включение приложения "Родители" в Центре администрирования Teams

Приложение "Родители" по умолчанию отключено, поэтому владельцы групп классов не будут видеть его в своих командах класса, пока это не будет разрешено через Центр администрирования Teams. Приложение "Родители" включено в Центре администрирования Teams с помощью разрешения [приложений, заблокированных разработчиками](manage-apps.md#allow-the-apps-that-are-blocked-by-the-developers).

В любое время приложение можно отключить на уровне клиента с помощью функции "Разрешить [и блокировать приложения](manage-apps.md#allow-and-block-apps) " в Центре администрирования Teams. Если она отключена на уровне клиента, она будет заблокирована для всех пользователей, даже если разрешения на уровне пользователя включены.

Приложение "Родители" также можно отключить на уровне пользователя с помощью политик управления разрешениями приложений [в Microsoft Teams](teams-app-permission-policies.md).
