# <a name="conduct-a-user-pilot"></a>Проведение пилотных пользователей

Развертывание новых технологий, организации могут реализовать экономическую выгоду, такие как экономии затрат, соответствия требованиям безопасности, удовлетворенности сотрудников и эффективность операций, но может также повлиять на производительность пользователей и организации инфраструктуры ( к сети). Перед включением новую технологию внутри организации, Проведение пилотного официальную пользователя. Так же, как бы рисовать небольшой исправление цвета на стену перед Рисование всей комнаты, будет протестируйте широкий выгрузка данных по размеру шкале, Проведение пилотного проекта для проверки технические и готовность пользователей, определить и устранять проблемы и обеспечить успешное Реализация всей организации.

Для достижения реалистичные результатов пилотного проекта должны включать реальных пользователей, внешнему как они общаться и совместно работать и проверьте технические и пользователей. Ли вашей организации учет под управлением Скайп для бизнеса и рабочих групп рядом друг с другом, обновление до группами в будущем или развертывание новые функциональные возможности телефонной или конференц-связи, пилотной может помочь определить правильный путь вперед для вашей организации. В некоторых случаях этап 1 развертывания, идеальная пилотного проекта использует подготовительных действий, которые вы уже начали и реализует определенный план с целевой группы пользователей.

## <a name="1-outline-pilot-logistics"></a>1. Пилотное логистика структуры

Успешные пилотного проекта определил start и конечную даты и [четко определенные цели](upgrade-define-project-scope.md#project-goals) для измерения успеха. Этих целей должен синхронизироваться с область более широкой проекта, как это описано при [определенных области проекта](upgrade-define-project-scope.md)и будет использоваться для оповещения path вперед после реализации пилотного проекта. Также необходимо убедиться, что будут указаны правом заинтересованных лиц во время выполнения проекта. Может потребоваться необходимо убедиться, что в достаточно времени для выполнения пилотного проекта и оценить влияние: мы рекомендуем не менее 30 дней.

Запустите небольшой и добавьте в пилотную среду соответствующим образом — ли путем добавления рабочих нагрузок или компонентов или дополнительных пользователей — Отображение времени для оценки результатов и настроить пилотную среду, как вы выполните итерацию. Даже могут выбрать для запуска последующих Пилотные в качестве новых функций группам газа выделяется на плану.

## <a name="2-select-your-pilot-participants-and-test-scenarios"></a>2. Выберите пилотного участников и тестовые сценарии

Одной из наиболее важных задач при планировании пилотного проекта является отбор участников. Помните, что команды оптимизирован для работы, поэтому для выбора участников пилотного проекта не только на основе роли или пользователей, но также на основе их project и работа нескольких групп. Прекрасной начальной запрашивает заинтересованные лица и руководители отдела проектах, в которых можно проверить в группах. Пример проекта на основе ролей следует использовать команды организации продаж, чтобы убедиться, что поле представители можно легко получить доступ к ресурсы, они должны и совместное использование обмена мнениями с другими участниками поля. Пример работы на основе проекта может Координирование событие запуска продукта с отдела маркетинга, обучение, связей с общественностью и событий, планирование групп. Независимо от выбранного сценариев, можно выбрать, следует расширить пилотного проекта для сотрудников в ИТ, обучение и служба поддержки, поэтому можно тщательной проверки решения при полностью оптимизации ресурсы по управлению проектами.

> [!Tip]
> После выбора ваших команд пилотный проект участников группы, не забудьте включить наиболее активных пользователей Скайп для бизнеса. Обратитесь к пользователям понять, как их использовать Скайп для бизнеса сегодня, а затем создать план тестирования, чтобы убедиться, что команды можно их потребностям текущей.

## <a name="3-design-your-test-plan-and-feedback-survey"></a>3. Разработка тестовой план и отзыв опроса

Для успешной пилотной программой предоставьте участников четко определенные задачи для выполнения, а также способ их для совместного использования их отзывы. Сгруппировать задачи вместе, чтобы предлагать реальных сценариях для пользователей, демонстрирующие релевантности для своих повседневных задач. Разрешить использование, планирование случаев, определенных в руководстве по [Оценка организационной изменение проверки готовности к](https://aka.ms/OrgReadiness) теста.

Ваша организация может выбрать пилотный проект все функциональные возможности одновременно или постепенного подход — например пилотный проект совместной работы, а затем собраний, затем чата и вызов. Убедитесь, что канал open свои отзывы и предложения для отслеживания хода выполнения и оценить результаты. Используйте предопределенные опроса как простой способ захвата и оценка пилотного результатов; Разработка опроса должен быть основан сценарии и функции в плане тестирования.

## <a name="4-create-your-communications-plan"></a>4. Создание плана взаимодействия

Очень важно для успеха в пилотную среду, ознакомить участников пилотного проекта на то, что происходит, когда и почему, и что от них ожидается. Для управления волнение и максимальная степень участия, обязательно включите значение пользователя системы обмена сообщениями, а также ссылки на обучающие и поддержка которых пользователи могут получить дополнительные сведения о мере продвижения по пилотного проекта. Вот некоторые примеры ресурсы, которые приступить к работе с плана пилотного взаимодействия.

- [Ресурсы пилотного проекта](https://aka.ms/UpgradeSuccessKit), включая шаблоны электронной почты и вопросы опроса пример свои отзывы и предложения
- [Переключатель, позволяющий групп из Скайп для бизнеса](https://support.office.com/article/Switch-to-Teams-from-Skype-for-Business-6295a0ae-4e8e-4bba-a100-64cc951cc964), краткое руководство, призванные помочь Скайп для бизнес-пользователям начало работы с группами

## <a name="5-conduct-your-pilot"></a>5. проведение реализации пилотного проекта

С помощью всех логистика на месте теперь вы готовы приступить к реализации пилотного проекта. Проведение пилотной включает в себя обмена данными с пользователями, мониторинг сети и об использовании, чтобы повысить производительность сети и Нормальное качество звонка, получая отзывы от участников, и просмотра билетов службы поддержки по вопросам, связанные с Группы.

### <a name="tips-for-pilot-success"></a>Советы по пилотного успеха

Следующие советы могут помочь обеспечить успешность реализации пилотного проекта:

- Перед началом работы над пилотной средой, убедитесь, что все участники пилотного включены для соответствующей [режим совместимости]
- (https://aka.ms/SkypeToTeams-SetCoexistence) требуется проверить.
- Еженедельно в пилотном проекте, соответствуют заинтересованным сторонам проект для просмотра отзывов пользователей, данные об использовании, сетевых данных и билетов службы поддержки, чтобы убедиться, что бесперебойную пилотную среду. При необходимости внесите изменения.

### <a name="suggested-timeline"></a>Предлагаемые временной шкалы

Вот предлагаемая временная шкала для 30-дневного пилотного проекта.

- За неделю до пилотного вводное: Отправка начального соединения пилотных пользователей.
- День 1: Отправьте связи вводное пилотных пользователей.
- День 7: Проведение первого еженедельно группы контрольной точки собрания по проекту.
- День 14: Отправить среднюю точку связи пилотных пользователей, удерживая еженедельное собрание контрольной точки team project.
- 21 день: Удержание еженедельное собрание контрольной точки team project.
- 30-дневную: Отправьте окончательный связи пилотных пользователей.
- 31 – 45 дней: оценки результатов пилотного развертывания и планирование для следующих действий.

## <a name="6-assess-learnings-and-evaluate-your-go-forward-plan"></a>6. оценки знаниями и Оценка плана перейти вперед

После завершения работы над пилотной средой, настало время необходимо собрать все опросы свои отзывы и предложения, stats окончательный сети и поддержка билетов для анализа для выполнения задач и определить, реализуется ли ваш план перейти вперед. Может оказаться, что ваша организация готова для широкомасштабного развертывания необходимо расширить пилотную среду нескольких пользователей или необходимо пересмотреть параметры пилотного проекта в нужный момент после любого проблем, которые вы определили, устранены. Имейте в виду, что в пилотную среду отличный способ для прогнозирования технические и результатов пользователей в среде с _управлением_ ; быть продуманной о переходе издержек слишком быстро.

Если указать результатов:

- **Достигли выполнения пилотного задач (например, удовлетворенности пользователей и качества сети)**, должен быть готов перейти к следующему этапу процесса развертывания. В зависимости от цели проекта это может быть один из следующих:
  - Функция пилотного проекта на других участников
  - [Включение команд наряду с Скайп для бизнеса (режим**о-ва** ) для некоторых или всех вашей организации](https://aka.ms/SkypeToTeams-SetCoexistence)
  - [Обновление пользователей из Скайп для бизнеса в группы (**групп только** в режиме) для некоторых или всех вашей организации](https://aka.ms/SkypeToTeams-SetCoexistence)
- **Пилотной не достижения результатов, которые вы хотите, чтобы (для примера, удовлетворенности пользователей и качества сети)**, потратьте время, чтобы внести соответствующие изменения в плане и вернитесь в пилотную среду.

> [!Tip]
> Включить в список участников вашей пилотного проекта как чемпионатов peer помогут Пропаганда и встроенного новых пользователей в группы. Чемпионатов Peer можно легко которые относятся к другим пользователям, общий доступ к собственные каждый раз и знаниями и предложение поддержка и рекомендации для своих коллег. Дополнительные сведения о [чемпионатов](https://go.microsoft.com/fwlink/?linkid=859068) и как можно использовать их в свои собственные выгрузка данных.
