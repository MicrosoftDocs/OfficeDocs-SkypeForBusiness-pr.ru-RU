# <a name="view-app-permissions-and-grant-admin-consent-in-the-microsoft-teams-admin-center"></a>Просмотр разрешений приложения и предоставление согласия администратора в центре Microsoft Teams администрирования

На [странице "Управление приложениями](manage-apps.md)" в Microsoft Teams администрирования можно просматривать и управлять всеми приложениями Teams вашей организации. Например, вы можете просмотреть состояние и свойства приложений на уровне организации, утвердить или отправить новые пользовательские приложения в магазин приложений организации, заблокировать или разрешить приложения на уровне организации, а также управлять параметрами приложений на уровне организации.

Здесь вы также можете предоставить согласие администратора на уровне организации приложениям, которые запрашивают разрешения на доступ к данным и просмотр разрешений на согласие для конкретных ресурсов (RSC) для приложений.

## <a name="grant-org-wide-admin-consent-to-an-app"></a>Предоставление согласия администратора на уровне организации для приложения

Если вы являетесь глобальным администратором, вы можете просмотреть и предоставить согласие приложениям, которые запрашивают разрешения от имени всех пользователей в организации. Это необходимо для того, чтобы пользователям не нужно было просматривать и принимать разрешения, запрашиваемые приложением при запуске приложения. Кроме того, в зависимости от параметров согласия пользователя в [](/azure/active-directory/manage-apps/configure-user-consent) Azure Active Directory (Azure AD), некоторым пользователям может быть запрещено предоставлять согласие приложениям, которые имеют доступ к корпоративным данным.

Примеры разрешений, запрашиваемых приложениями, включают возможность чтения сведений, хранящихся в команде, чтения профиля пользователя и отправки сообщения электронной почты от имени пользователей. Дополнительные сведения см. [в разделе "Разрешения и согласие" платформа удостоверений Майкрософт конечной точке](/azure/active-directory/develop/v2-permissions-and-consent).

В **столбце "** Разрешения" указывается, имеет ли приложение разрешения, для которых требуется согласие. Вы увидите ссылку "**Просмотр сведений**" для каждого приложения, зарегистрированного в Azure AD с разрешениями, для которых требуется согласие. Помните, что это относится только к пользовательским и сторонним приложениям. Ссылка недоступна для приложений, предоставляемых корпорацией Майкрософт. Кроме того, администраторам не нужно предоставлять согласие для таких приложений.

Чтобы предоставить приложению согласие на уровне организации, выполните следующие действия.

1. В левой области навигации центра администрирования Microsoft Teams перейдите в раздел Teams **"Управление** > **[приложениями"](https://admin.teams.microsoft.com/policies/manage-apps)**.

1. Выполните одно из следующих действий.
    * Найдите нужное приложение, щелкните имя приложения, чтобы перейти на страницу сведений о приложении, а затем выберите **вкладку "Разрешения** ".
    * **Отсортируйте столбец** "Разрешения" в порядке убывания, чтобы найти приложение, а затем выберите "**Просмотреть сведения"**. При этом вы перейдете на **вкладку "Разрешения** " на странице сведений о приложении.

1. В **разделе разрешений для всей организации выберите** " **Проверить разрешения и согласие"**.

    :::image type="content" source="media/app-perm-admin-center-org-wide.png" alt-text="Снимок экрана: разрешения для всей организации на вкладке &quot;Разрешения&quot; для приложения.":::

    Для этого необходимо быть глобальным администратором. Этот параметр недоступен для администраторов Teams служб.

1. На **вкладке "** Разрешения" просмотрите разрешения, запрашиваемые приложением.

    :::image type="content" source="media/app-perm-admin-center-org-wide-permissions.png" alt-text="Снимок экрана: разрешения, запрашиваемые приложением.":::

    > [!IMPORTANT]
    > Предоставление согласия для приложения на уровне организации позволяет приложению получать доступ к данным вашей организации. Внимательно проверьте разрешения, запрашиваемые приложением, прежде чем предоставлять согласие.

1. Если вы согласны с разрешениями, запрашиваемые приложением, нажмите кнопку **"Принять"** , чтобы предоставить согласие. В верхней части страницы временно появляется баннер с уведомлением о том, что для приложения предоставлены запрашиваемые разрешения. Теперь приложение имеет доступ к указанным ресурсам для всех пользователей в организации, и больше никто не будет получать запрос на проверку разрешений.

После принятия разрешений на странице сведений о приложении вы увидите сообщение  в разделе разрешений на уровне организации, чтобы узнать, что согласие было предоставлено. Чтобы просмотреть сведения о разрешениях приложения, щелкните ссылку  Azure Active Directory, чтобы перейти на страницу приложения на Azure AD портале.

:::image type="content" source="media/app-perm-admin-center-org-wide-accepted-new.png" alt-text="Снимок экрана: сообщение, отображаемое при предоставлении согласия.":::

Если пользователям в вашей организации разрешено предоставлять согласие и один или несколько пользователей предоставили согласие определенному приложению, вы также увидите то же сообщение, чтобы узнать, что согласие было предоставлено, и ссылку Azure Active Directory на страницу приложения на портале Azure AD.

> [!NOTE]
> Хотя параметр проверки разрешений и согласия недоступен для администраторов службы Teams и они не могут предоставить согласие администратора на уровне организации для приложений, администраторы Teams могут просматривать содержимое на вкладке "Разрешения" для приложения.  Например, администратор Teams службы может щелкнуть ссылку Azure Active Directory, чтобы просмотреть сведения о разрешениях приложения на Azure AD портале.

## <a name="view-resource-specific-consent-permissions-of-an-app"></a>Просмотр разрешений на согласие для конкретного ресурса приложения

Разрешения RSC позволяют владельцам команд предоставлять приложению согласие на доступ и изменение данных команды. Разрешения RSC детализированные, Teams, которые определяют, что приложение может делать в определенной команде. Примеры разрешений RSC включают возможность создания и удаления каналов, получения параметров для команды, а также создания и удаления вкладок каналов.

Разрешения RSC определяются в манифесте приложения, а не в Azure AD. Вы предоставляете согласие на разрешения RSC при добавлении приложения в команду. Дополнительные сведения см. [в разделе о согласии для конкретного ресурса (RSC).](/microsoftteams/platform/graph-api/rsc/resource-specific-consent)

Глобальные администраторы и Teams могут просматривать разрешения RSC для приложения на вкладке **"** Разрешения" страницы сведений о приложении.

Чтобы просмотреть разрешения RSC для приложения, выполните следующие действия.

1. В левой панели навигации Центра администрирования Microsoft Teams выберите **Приложения Teams** > **Управление приложениями**.
1. Найдите нужное приложение, щелкните имя приложения, чтобы перейти на страницу сведений о приложении, а затем выберите **вкладку "Разрешения** ".
1. В разделе microsoft Graph разрешения на согласие для конкретных ресурсов **(RSC)** просмотрите разрешения RSC, запрашиваемые приложением.

    :::image type="content" source="media/app-perm-admin-center-rsc-new.png" alt-text="Снимок экрана: разрешения RSC для приложения.":::

## <a name="known-issues"></a>Известные проблемы

### <a name="the-view-details-link-isnt-displayed-in-the-permissions-column-for-some-third-party-apps-that-request-permissions"></a>Ссылка "Просмотр сведений" не отображается в столбце "Разрешения" для некоторых сторонних приложений, запрашивающей разрешения

В настоящее время возможность просматривать разрешения и предоставлять согласие доступна не для всех сторонних приложений, зарегистрированных в Azure AD запроса разрешений. Вместо **ссылки "Просмотр сведений** " вы увидите **--** столбец **"Разрешения** ". Мы работаем с независимыми поставщиками программного обеспечения, чтобы включить эту функцию для своих приложений.
