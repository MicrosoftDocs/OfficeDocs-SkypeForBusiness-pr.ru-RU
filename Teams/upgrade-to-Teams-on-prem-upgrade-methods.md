## <a name="upgrade-methods"></a>Способы обновления

Существует два способа обновления существующей организации с помощью Skype для бизнеса (как в Интернете, так и локально) для teams: перекрытие методов и метод Select capabilities. В этой статье объясняется, как выбрать нужный метод для Организации, обменяя оба способа и выполнив для них достоинства и недостатки каждого из них. 

- [Метод перекрытия возможностей (с использованием режима «острова»)](#overlapping-capabilities-method-using-islands-mode)

   Пользователи в существующей организации Skype для бизнеса вводятся в Teams, чтобы они могли использовать оба клиента на переходном этапе. В течение этого периода вы можете использовать для них не все возможности Teams. Режим для этой конфигурации называется «острова», и этот режим используется по умолчанию для любой организации, в которой установлен Skype для бизнеса. После того как Организация будет готова, администратор перейдет пользователей в режим TeamsOnly.

- [Выберите способ выбора возможностей (с помощью одного или нескольких режимов Skype для бизнеса).](#select-capabilities-method-using-skype-for-business-modes)

   Администратор управляет переходом (от Skype для бизнеса в Teams), а также с помощью функций планирования чата, звонков и собраний для пользователей в своей организации.  Каждая из этих функций доступна либо в Skype для бизнеса, либо в Teams, но не на обоих. Администраторы используют TeamsUpgradePolicy, чтобы управлять тем, когда вы перемещаете эту функцию в Teams для пользователей. Пользователи, которые еще не в режиме TeamsOnly, продолжают использовать Skype для бизнеса для общения и звонков, а два набора пользователей могут взаимодействовать с помощью функций взаимодействия. Администраторы могут управлять переходом, изменяя способно переводя пользователей в режим TeamsOnly.  

После того как вы понимаете и выберете метод обновления, вы можете изучить [инструменты для управления обновлением вашей организации до Teams](upgrade-to-teams-on-prem-tools.md).

## <a name="overlapping-capabilities-method-using-islands-mode"></a>Метод перекрытия возможностей (с использованием режима «острова»)

С помощью перекрывающихся возможностей пользователи могут использовать как команды, так и клиенты Skype для бизнеса для общения, звонков по VoIP и собраний. Это состояние называется режимом "острова", поскольку трафик для взаимодействия в Skype для бизнеса и Teams остается отдельным (даже для одного пользователя), а два разных клиента не взаимодействуют друг с другом (для пользователей в пределах одной организации). Например, предположим, что пользователь A находится в режиме "острова".

- Связь, которая инициируется клиентом Skype для бизнеса другого пользователя, всегда будет в клиенте "Skype для бизнеса".

- Связь, инициированная клиентом группы другого пользователя, всегда будет находиться в клиенте Teams для пользователя, *Если другой пользователь находится в той же организации*. 

- Связь, инициированная из клиента группы другого пользователя, всегда будет в клиенте "Skype для бизнеса *", если другой пользователь — в федеративной организации*.

Режим "острова" — режим по умолчанию TeamsUpgradePolicy для любой существующей организации, которая еще не обновлена до TeamsOnly. Когда вы назначаете лицензию Microsoft 365 или Office 365, по умолчанию назначаются лицензии для Teams и Skype для бизнеса Online. (Это справедливо даже в том случае, если пользователь подключен к локальной сети в Skype для бизнеса Server. Если пользователь является локальным или подключенным к Интернету, оставьте лицензию на использование Skype для бизнеса Online включенной, так как она в настоящее время нужна для полных функций Teams.) На самом деле, если вы не предоставили какие – либо инструкции по изменению конфигурации по умолчанию, возможно, у вас уже есть существенные способы использования Teams в Организации.  Это один из преимуществ подхода к перекрытию возможностей. Она позволяет быстро и ориентированно на конечную пользователей переходить в пределах организации.

Для эффективного использования этого метода требуется, чтобы все пользователи одновременно работали на обоих клиентах. Входящие разговоры и звонки из Организации в режим "острова" могут быть доступны либо в Skype для бизнеса, либо в клиенте Teams, и это не является элементом управления получателя. Если отправитель и получатель находятся в одной организации, это зависит от того, какой клиент использует отправитель для инициирования обмена данными. Если отправитель и получатель находятся в разных организациях, входящие звонки и разговоры в режиме острова всегда помещаются в клиенте Skype для бизнеса.  

Например, если получатель режима "острова деятельности" работает в Skype для бизнеса, но не в Teams, а кто-то в той же организации в той же почте отправляет сообщение в группе "острова", получатель не увидит его (но, в конечном итоге, получит сообщение электронной почты о том, что он пропущен в Teams). Аналогичным образом, если пользователь работает с группами, но не Skype для бизнеса, а кто-то поделился этим пользователем в Skype для бизнеса, он не будет видеть этот чат.  Сообщение о том, что оно было пропущено, будет получено по электронной почте. Поведение в каждом из этих случаев похоже на вызов. Если пользователи не работают на обоих клиентах, это может привести к неудовлетворенности.

Когда пользователь A находится в режиме "острова", для пользователей в Teams, а также в Skype для бизнеса независимым является возможность присутствия.

- Другие пользователи, работающие с группами, увидят состояние присутствия на основе действий пользователя а в Teams. 
- Другие пользователи при использовании Skype для бизнеса увидят состояние присутствия на основе действий пользователя а в Skype для бизнеса. 

Это означает, что другие пользователи могут видеть различные состояния присутствия для пользователя а в зависимости от того, какой клиент они используют. Дополнительные сведения можно найти в разделе сведения о [присутствии](upgrade-to-teams-on-prem-coexistence.md#presence).

После того как вы будете готовы перейти к обновлению режима TeamsOnly, вы можете обновить пользователей по отдельности или обновить весь клиент за один раз с помощью политики на уровне клиента. После того как пользователь будет обновлен до TeamsOnly режима, он получит все входящие разговоры и звонки в Teams. (Обратите внимание, что миграция собраний Skype для бизнеса в собрания Teams инициируется только при применении TeamsUpgradePolicy к отдельным пользователям, а не по отдельности для каждого клиента. Подробные сведения о [миграции собраний содержатся в разделе Миграция собрания](upgrade-to-teams-on-prem-tools.md#meeting-migration) .

Однако необновляемые получатели в режиме "острова" могут по-прежнему принимать разговоры и звонки от TeamsOnly пользователя в Skype для бизнеса или клиентах Teams.  Это связано с тем, что клиент Teams поддерживает отдельные потоки беседы для обмена сообщениями между группами и группами в Skype для бизнеса, даже для одного и того же пользователя.  (См. [: беседы в группах — взаимодействие и потоки взаимодействия и машинного кода](upgrade-to-teams-on-prem-coexistence.md#teams-conversations---interop-versus-native-threads).)  Например, предположим, что пользователь A использует команды для сообщения TeamsOnly User B. Когда пользователь B отправляет ответы на этот чат, Обмен данными осуществляется в клиенте Teams для пользователей. Теперь предполагаю, что пользователь A использует клиент Skype для бизнеса для обмена сообщениями TeamsOnly пользователя B. пользователь B получит собеседника в Teams, но это будет отдельной беседа в клиенте Teams для пользователей по сравнению с другой беседой. Если пользователь B отвечает на этот чат с пользователем A, он будет находиться в клиенте "Skype для бизнеса" пользователя "A".

## <a name="select-capabilities-method-using-skype-for-business-modes"></a>Выберите способ выбора возможностей (в режимах Skype для бизнеса)

В некоторых организациях предпочтительнее предоставить конечным пользователям более простой и предсказуемый опыт в переходе между организациями из Skype для бизнеса в Teams. В этой модели ИТ-администраторы используют один из режимов Skype для бизнеса в TeamsUpgradePolicy, чтобы явным образом определять, какие пользователи остаются в Skype для бизнеса перед переходом в режим TeamsOnly. После того как они будут готовы к смене выбранных пользователей в режиме TeamsOnly, администратор обновляет режим для этих пользователей на TeamsOnly. По мере выполнения развертывания новые пользователи переходят из Skype для бизнеса в режим TeamsOnly.  Во время этого перехода:

- Пользователи Skype для бизнеса смогут получать все входящие разговоры и звонки в клиенте Skype для бизнеса независимо от того, откуда они поступили: от других пользователей Teams или Skype для бизнеса. Кроме того, для пользователей Skype для бизнеса возможности звонков и чата в клиенте Teams отключаются, чтобы предотвратить путаницу с пользователями и убедиться в правильности маршрутизации. 

- Пользователи в режиме TeamsOnly получают все входящие разговоры и звонки в своем клиенте Teams, независимо от того, откуда поступило соединение: teams, Skype для бизнеса или любой другой федеративный пользователь. 

В отличие от метода "острова", в разделе "выбор возможностей" пользователи Skype для бизнеса и пользователи TeamsOnly могут общаться друг с другом. Связь между пользователями Skype для бизнеса и пользователями Teams называется взаимодействием или взаимодействием. Взаимодействие между абонентами в Skype для бизнеса и другим пользователем в Teams может осуществляться с помощью взаимодействия. Однако некоторые дополнительные функции могут быть недоступны. (См. раздел [взаимодействие](upgrade-to-teams-on-prem-coexistence.md#interoperability).) Кроме того, приглашенные пользователи всегда могут присоединиться к собранию Skype для бизнеса или Teams, однако они должны использовать клиент, соответствующий типу собрания. Дополнительные сведения можно найти в разделе [собрания](upgrade-to-teams-on-prem-coexistence.md#meetings).

Так как пользователи в переходе на выбор возможностей обычно не находятся в режиме острова, присутствие пользователя является постоянным, независимо от того, какой клиент используется другим пользователем. Если пользователь входит в один из режимов Skype для бизнеса, другие пользователи видят сведения о присутствии на основе активности пользователя в Skype для бизнеса. Точно так же, если пользователь находится в режиме TeamsOnly, все другие пользователи видят сведения о присутствии на основе активности этого пользователя в Teams. Подробнее: Просмотр сведений о [присутствии](upgrade-to-teams-on-prem-coexistence.md#presence).

Для Организации, которая еще не начиналась с помощью Teams, администратор должен изменить режим клиента с островов на SfbWithTeamsCollab. (В организациях, у которых уже есть использование некоторых групп, администратор должен "grandfather" пользователей, которые уже активны в Teams, чтобы не применять это изменение. Дополнительные сведения можно найти в описании [метода выбора возможностей для Организации, в которой уже используется Teams в режиме "острова](upgrade-to-teams-on-prem-implement.md#a-select-capabilities-upgrade-for-an-organization-that-is-already-using-teams-in-islands-mode)".)

Если режим изменится с островов на SfbWithTeamsCollab, пользователь, который никогда не использовался в Teams, не получит никаких различий в том, как они используют Skype для бизнеса. Тем не менее, если пользователь начал использовать Teams, они будут доступны только для таких функций, как команды & канал и файлы. Skype, звонки и собрания в Teams будут недоступны, так как администратор (сейчас) заблокировал (-а) в качестве нужного клиента для этих функций.  

Примечание. когда пользователь а изменяются из островов в один из режимов Skype для бизнеса, клиент Teams любого другого пользователя, который взаимодействует с пользователем, должен знать, что пользователь изменил режим, чтобы он мог перенаправлять связь на соответствующий клиент для пользователя A.  Для пользователей, которые уже установили сеансы в Skype для бизнеса с пользователем A, может потребоваться до 36 часов для того, чтобы клиенты Teams могли учитывать изменения в режимах, отличных от островов и до любого из режимов, в которых вы можете работать в любом из них.   Напротив, изменения существующего пользователя в режим TeamsOnly будут обнаружены другими клиентами в течение 2 часов.

После того как администраторы будут готовы, они смогут одновременно перемещаться между абонентом, вызовом и планированием собраний для определенного пользователя в Teams, обновляя режим пользователя на TeamsOnly.  

Кроме того, администратор может сначала сдвинуть только планирование собраний в Teams, в то время как в Skype для бизнеса не покидая функций чата и звонков, используя режим SfBWithTeamsCollabAndMeetings. Этот режим позволяет организациям переходить в Teams для собраний — если пользователи пока еще не готовы к переходу в режим TeamsOnly (как правило, это может потребоваться для миграции существующих функций PSTN). Этот переходный сценарий в [первую очередь называется собранием](meetings-first.md).

В приведенной ниже таблице перечислены достоинства и недостатки использования режимов Skype для бизнеса в качестве переходного этапа к режиму TeamsOnly.


| Специалистов     |       Недостатк |

| Прогнозируемый маршрут для конечного пользователя. В приведенной ниже таблице перечислены способы обновления:

| Перекрывающиеся возможности (с использованием режима острова)     |      Выбор возможностей (в режимах Skype для бизнеса) |
| :------------------ | :---------------- |
| Перед обновлением до TeamsOnly пользователи должны одновременно запускать оба клиента, так как входящие разговоры и звонки могут находиться на любом клиенте.   | Разговоры и звонки только на один клиент в зависимости от режима получателя. Необновленные пользователи могут запускать оба клиента, но в Teams не существует функционального перекрытия (звонки и чат недоступны).  Администраторы также могут управлять планированием собраний в Teams или Skype для бизнеса.   |
| Пользователи могут одновременно использовать Skype для бизнеса и Teams для работы с одной и той же функциональностью.   | Позволяет администраторам создавать сетевые новые возможности Teams для конечных пользователей (рабочих групп и каналов), не предоставляя возможности, которые также есть в Skype для бизнеса.   |
|Взаимодействие между Skype для бизнеса и Teams не существует, пока оба пользователя находятся в режиме острова. После того как некоторые пользователи будут обновлены до TeamsOnly, взаимодействие между ними может происходить между пользователями и другими пользователями в режиме "острова". Тем не менее, пользователи могут выбрать использование Teams и избежать беседы о взаимодействии. | Для взаимодействия между пользователями Skype для бизнеса и Teams требуется взаимодействие.   |
