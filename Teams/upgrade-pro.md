# <a name="upgrade-pro"></a>Профессиональная модернизация

Разработан для крупных организаций или для сложных развертываний Skype для бизнеса (например, гибридных и облачных голосовых развертываний), обновления учетных записей Pro для более продолжительного жизненного цикла и с различными сценариями обновления, включая запуск Microsoft Команды Skype для бизнеса до тех пор, пока не будет готова ваша организация.

Разбивается на три фазы, переход на новую версию Pro:

|**[Перед обновлением](#pre-upgrade)** |**[Обновление](#upgrade)** |**[После обновления](#post-upgrade)** |
|---|---|---|
|Реализуйте значения в Teams<ul><li>Готовность среды</li><li>Принятие команд для совместной работы, собраний и/или звонков</li><li>Планирование обновления Skype для бизнеса в Teams с течением времени</li></ul> |Перенос пользователей из Skype для бизнеса<ul><li>Уведомление и готовность пользователей</li><li>Перемещение пользователей в режиме "только для Teams"</li><li>Отслеживание хода использования для целей</li></ul> | Расширяют свой коэффициент окупаемости инвестиций <ul><li>Наблюдение за работоспособностью и качеством сети</li><li>Поддержка пользователей</li><li>Планирование существующих командных инноваций</li></ul>|

> [!NOTE]
> Мы понимаем, что ваше путешествие в Teams может включать в себя использование нескольких [режимов](https://aka.ms/skypetoteams-coexist) и обновление групп пользователей в разное время, что позволяет управлять пользовательским интерфейсом, сохраняя при этом время работы с Teams. <br><br>
Чтобы продемонстрировать, как может унфолд переход на новую версию, мы предоставили примерный план, который определяет путешествие из Skype для бизнеса в режим " **острова** " только в Teams. Кроме того, примерный план содержит структуру, в которой пользователи делятся на четыре группы обновления или _кохортс_. Используя этот шаблон в качестве шаблона, вы можете настроить его в соответствии с конкретным путешествием для Teams, включив в него различные [режимы](https://aka.ms/skypetoteams-coexist) и количество групп обновления, на которые вы собираетесь побить пользователей.

## <a name="pre-upgrade"></a>Перед обновлением

**Подготовьте свою организацию для Teams.** Чтобы обеспечить успешный переход на Teams, важно выделить достаточное время для подготовки. Ваша организация сможет быстро приступить к внедрению значений Teams, поэтому вы сможете ускорить переход с Skype для бизнеса, как только они будут готовы к вам. Отслеживайте [планы](https://aka.ms/O365Roadmap) улучшений, которые запланированы для Teams; Это поможет вам определить подправную временную шкалу обновления для Организации. Если вы уже включили команды в Skype для бизнеса, используйте эти действия перед обновлением в качестве контрольной точки для проверки готовности Организации перед обновлением пользователей до Teams.

> [!Tip]
> Загрузи [комплект обновления](https://aka.ms/UpgradeSuccessKit) для подготовки к обновлению в соответствии с шаблоном "сведения о готовности пользователей", например о связи и опросах пользователей, в дополнение к образцу плана проекта на обновление и пробного тестирования. Элементы, доступные в комплекте, помечаются звездочкой (\*) в списках ниже.

## <a name="upgrade"></a>Upgrade

**Сделайте официальный переход на Teams.** При обновлении пользователей они перемещаются в **Teams только** в режиме совместного существования. Teams становится основным приложением для общения, собраний, звонков и совместной работы, а также доступа к приложению Skype для бизнеса. Несмотря на то, что технические аспекты этого этапа довольно просты, рассматривайте влияние изменения на взаимодействие с пользователем и разрешите пользователям официально переходить от Skype для бизнеса в Teams. Чтобы уменьшить объем работы пользователей с разными клиентами, попробуйте ограничить окно сквозного обновления до 45 дней и т. д.
