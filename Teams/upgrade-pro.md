# <a name="upgrade-pro"></a>Обновление специалистов

Предназначена для крупных организаций или с сложных развертываний Скайп для бизнеса (например, гибридного или облака. развертывания голосовой связи), обновление специалистов учетных записей для более жизненный цикл обновления а также различные сценарии обновления, включая под управлением Microsoft Группы наряду с Скайп для бизнеса, пока команды будет готов для вашей организации. 

Сегментированные в три этапа обновления специалистов рассматриваются:

-   **Подготовка к обновлению**: запустите осознавать значение команды, при подготовке к обновлению по времени. Рассмотрим эти действия обновления условием, чтобы обеспечить надлежащую положительное взаимодействия при переходе к группам, когда он будет готов для вас. Если ваша организация уже выполняется группами наряду с Скайп для бизнеса, используйте эти действия перед обновлением для контроля проверки для проверки готовности к вашей организации для обновления пользователей по группам.
-    **Обновление**: Если ваша организация готова для пользователей прекратить использование Скайп для бизнеса, необходимые для обеспечения у них есть достаточно уведомление о Начало обучения и начать использовать команды (если они еще не установлен). Может также потребоваться выполнять действия, такие как перемещение их Скайп для собраний бизнес группами для обеспечения оптимального взаимодействия.
-    **После завершения обновления**: после всей организации на группы, реализация плана для обеспечения максимальной значение. Это включает в себя мониторинг сети для контроля качества, планирование для расширения функциональных возможностей как команды, стратегия продолжает увеличиваться и продвижение внедрения группам будут недоступны при предоставлении, Скайп для использования Business — это обмотка вниз.
 
> [!NOTE]
> Мы понимаем, что поездке групп, может потребоваться использование нескольких [режимах](https://aka.ms/skypetoteams-coexist) и обновление групп пользователей в разное время, которые позволяют управлять обновлением удобство работы пользователей без снижения momentum с группами. Чтобы продемонстрировать как может unfold обновления пути, мы подготовили что пример плана ниже, который определяет пути, переход с Скайп для бизнеса на **острова** только в режиме группам. Кроме того пример плана содержат организации, которая разделены своим пользователям четыре обновления группы или _последователей_. Использование это в качестве шаблона, настройте планирования охватывать поездке определенным группам интеграции сведений о различных [режимах](https://aka.ms/skypetoteams-coexist) работы и число обновления групп, которые будут сегмент пользователей в.

## <a name="pre-upgrade"></a>Подготовка к обновлению

**Подготовка групп в вашей организации.**  Для обеспечения успешного обновления к группам, важно выделить достаточное количество времени для подготовки. Не только будет организации иметь возможность быстро начать осознавать значение групп, вы сможете ускорить процесс обновления Скайп для бизнеса сразу же команды будет готов для вас. Мониторинг [стратегия](https://aka.ms/skype2teamsroadmap) для улучшения, запланированных для групп; Это поможет вам определить прямо обновления временной шкалы для вашей организации. Если уже было включено группами наряду с Скайп для бизнеса, используйте эти перед обновлением действия для контроля для проверки готовности организации перед обновлением до группам пользователей.

> Загрузите [Комплект успешности обновления](https://aka.ms/UpgradeSuccessKit) для шаблона пользователя проверки готовности к источников коммуникаций и опросов пользователей, в дополнение к пример плана проекта обновления и плана пилотного тестирования. ## <a name="upgrade"></a>Upgrade

**Официальная откройте для себя группы.** При обновлении пользователей вы переместите их в режиме сосуществования **только группы** . Становится их основного приложения для разговора, собраний, вызова и совместной работы групп и доступ к Скайп для бизнес-приложения будет отключен. Несмотря на то, что технические аспекты этого этапа все довольно просто, необходимо Учитывайте влияние изменения могут иметь на взаимодействие с пользователем и разрешить пользователям официально переноса их действий из Скайп для бизнеса группам время. Для уменьшения пользователей с другой каждый раз с помощью различных клиентов, попробуйте ограничить обновления окно начала до конца 45 дней или так.

Дней 1&ndash;2: Убедитесь, что ваша организация готова к обновлению.
-   Убедитесь, что вы выполнили перед обновлением действий, описанных выше.
-   Сегментация базы пользователей на четыре группы обновления.
-   Отправьте уведомление об обновлении для всех пользователей.\*
-   Установите [Уведомить Скайп для бизнес-пользователей, что команды доступен для обновления](https://aka.ms/SkypeToTeams-SetCoexistence) для всех пользователей.

_7-45 дней: реализация обновления Скайп для бизнеса в группы._

7 дней
-   Отправка уведомлений обновления группа 1, их обновления начнется через две недели.\*
-   Мониторинг сетевых вызовов работоспособности и служба технической поддержки для поиска ошибок обновления интерфейса пользователя или технические и устранять при необходимости.

14 дней
-   Отправьте напоминание, чтобы обновить группа 1, их обновления начнется в неделю.\*
-   Мониторинг сетевых вызовов работоспособности и служба технической поддержки для поиска ошибок обновления интерфейса пользователя или технические и устранять при необходимости.

21 день
-   [Задать режим сосуществования для обновления группы 1, чтобы **Только группы**](https://aka.ms/SkypeToTeams-SetCoexistence).
-   Отправьте уведомление «В приветствия для групп» обновление группы 1.\*

14 дней&ndash;42
-   Повторите предыдущих действий обновления для обновления оставшихся групп на повторяющегося цикла. Например начать уведомлений обновления группы 2 на день 14, группа 3 уведомления об обновлении на 21 день и группа 4 уведомления об обновлении в день 28. Это должна поддерживать для всех обновлений в качестве завершена, 42 дня.

45 дней
-   Отправьте [свои отзывы и предложения опросы, выполняемые после обновления](https://aka.ms/UpgradeSuccessKit) для всех пользователей.\*

## <a name="post-upgrade"></a>После завершения обновления

**Ценность бизнеса с группами.** После полного обновления вашей организации групп потратьте время для оценки успешность для выполнения задач и реализация плана для продолжения прямого momentum.

Дней 1&ndash;14: измерения как успешного обновления была.
-   Оценка начальной обновления успеха поставленных целей, установленная на стадии подготовки к обновлению.
-   Реализация плана смягчение цели, которых нет в соответствии с графиком.

Постоянная: Обслуживание momentum и действующие работоспособность развертывания групп.
-   Мониторинг сетевых вызовов работоспособности и служба технической поддержки для поиска ошибок обновления интерфейса пользователя или технические и устранять при необходимости.
-   [Диск momentum пользователя с помощью электронной почты, бюллетени или другие средства связи, который сосредоточиться на предстоящие новых функций, истории успеха и т. д](https://aka.ms/successwithteams).
-   Планирование инициативы готовности технические сведения и пользователя для подготовки новых возможностей и оптимизации Microsoft 365. <!-- (point to blog??)-->
