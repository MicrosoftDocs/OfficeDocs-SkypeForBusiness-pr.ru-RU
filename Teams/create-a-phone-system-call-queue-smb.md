# <a name="create-a-call-queue"></a>Создание очереди вызовов

Очереди звонков направляют вызывающих абонентов пользователям в вашей организации, которые могут помочь с определенной проблемой или вопросом. Звонки распространяются по одному людям в очереди, которые называются *агентами*.

Очереди вызовов предоставляют:

- Приветствие.
- Музыка, пока люди ожидают удержания в очереди.
- Маршрутизация вызовов ( в порядке " *Первый вход", "* Первый выход" (FIFO) — агентам.
- Параметры обработки переполнения очереди и времени ожидания.

Перед выполнением процедур, описанных в этой статье, убедитесь, что вы прочитали план для автосекретарей и очередей звонков [Teams](plan-auto-attendant-call-queue.md) , а также выполнили действия по [началу работы](plan-auto-attendant-call-queue.md#getting-started).

## <a name="whats-new-for-call-queues-in-the-past-6-months"></a>Новые возможности очередей звонков за последние 6 месяцев

- Августа
  - **Теперь для основного приветствия** очереди вызовов поддерживается добавление приветственного сообщения (текст в речь (TTS).
  - **Пропуск элементов управления системным** сообщением голосовой почты теперь предоставляется при маршрутизации к общей голосовой почте, что также применяется к добавлению **приветственного сообщения** .

## <a name="steps-to-create-a-call-queue"></a>Действия по созданию очереди вызовов

Ниже приведены шаги по настройке очереди вызовов.

1. Настройка общих сведений
1. Настройка приветствия и музыки
1. Настройка ответов на вызовы
1. Выбор и назначение агентов
1. Настройка обработки переполнения вызовов
1. Настройка обработки времени ожидания вызова

Действия, описанные в статье, создают очереди вызовов с помощью Центра администрирования Teams. Инструкции по созданию очередей вызовов с помощью PowerShell см. в статье "Создание очередей вызовов с помощью командлетов [PowerShell"](create-a-phone-system-call-queue-via-cmdlets.md).

## <a name="step-2-add-a-greeting-and-on-hold-music"></a>Шаг 2. Добавление приветствия и музыки на удержание

Укажите, нужно ли воспроизводить *приветствие* вызывающим абонентам при их поступлении в очередь.

- При выборе **параметра "** Воспроизведение звукового файла" необходимо отправить файл MP3, WAV или WMA, содержащий приветствие, которое вы хотите воспроизвести. Загруженная запись не может превышать 5 МБ.

- Музыка по умолчанию, предоставляемая в очередях звонков Teams, не оплачивается вашей организацией.
- Если вы хотите воспроизвести определенный звуковой файл, выберите  "Воспроизвести звуковой файл" и отправьте файл MP3, WAV или WMA.

### <a name="conference-mode"></a>Режим конференции.

**Режим конференции** сокращает время, необходимое для подключения вызывающего абонента к агенту после принятия вызова агентом. Для работы режима конференции агенты в очереди вызовов должны использовать один из следующих клиентов:

- Последняя версия настольного клиента Microsoft Teams, приложения Android или приложения iOS
- Телефонная система Microsoft Teams версии 1449/1.0.94.2020051601 или более поздней
  
Учетные записи Teams агентов должны быть настроены в режиме TeamsOnly. Агенты, которые не соответствуют требованиям, не включаются в список маршрутизации вызовов. Мы рекомендуем включить режим конференции для очередей звонков, если агенты используют совместимые клиенты.

> [!NOTE]
> Режим конференции не поддерживается, если телефонные звонки направляются в очередь из шлюза прямой маршрутизации, который включен для маршрутизации на основе расположения.
>
> Режим конференции не поддерживается, если телефонные звонки направляются в очередь из Skype для бизнеса Server.
> 
> Режим конференции требуется, если пользователям Teams необходимо обращаться к звонкам и передавать их с помощью очередей звонков.
>
> При первом присоединении к вызову агенты могут слышать настроенную музыку при удержании в очереди в течение 2 секунд.


> [!TIP]
> Рекомендуется **установить режим конференции** **в режим "** В сети".

Выбрав параметры ответа на вызовы, нажмите кнопку **"** Далее" в нижней части страницы **"Добавление очереди вызовов** ".

# <a name="step-4-agent-routing"></a>[Шаг 4. Маршрутизация агента](#tab/agent-routing)

## <a name="step-4-select-your-agent-routing-options"></a>Шаг 4. Выбор параметров маршрутизации агента

**Метод маршрутизации определяет** порядок, в котором агенты получают вызовы из очереди.

Выберите один из следующих вариантов:

- **Маршрутизация помощника ко** всем агентам в очереди одновременно. Первый агент вызова, который выбирает вызов, получает вызов.

- **Последовательная маршрутизация** вызывает все агенты вызовов по одному в порядке, указанном в списке **агентов** вызовов. Если агент закрывает или не выбирает вызов, вызов вызывает следующий агент. Это будет повторяться до тех пор, пока не будет установлен или не истекло время ожидания вызова.

- **Циклический перебор** распределяет маршрутизацию входящих вызовов таким образом, чтобы каждый агент вызова получает одинаковое количество вызовов из очереди. Этот метод маршрутизации может быть желательно в среде входящих продаж, чтобы обеспечить равные возможности для всех агентов вызовов.

- **Самый длинный бездействующий** маршрут направляет каждый вызов агенту, который был неактивным в течение самого длительного времени. Агент считается неактивным, если его состояние присутствия доступно. Агенты, состояние присутствия которых недоступно, не смогут принимать звонки, пока не будут изменены на "Доступно".

Рекомендуется задать для метода **маршрутизации** значение **"Циклический перебор** " или **"Самый длинный бездействующий"**.

> [!NOTE]
> Если [на агентах](teams-recording-policy.md) включена запись соответствия, **сочетание режима конференции** и маршрутизации помощника не поддерживается. Если необходимо использовать режим конференции **, выберите** в качестве метода маршрутизации последовательную маршрутизацию **, циклический** перебор или самое длинное **бездействие**. Если необходимо использовать маршрутизацию **помощника**, задайте режим **конференции** **"Выкл."**.
>
> При использовании **самого длительного** простоя и меньшего числа вызовов в очереди, чем доступных агентов, вызовы из очереди будут отображаться только для первых двух самых длинных неактивных агентов.
>
> При использовании самого длительного простоя могут возникать моменты, когда агент получает вызов из очереди вскоре после того, как он становится недоступным, или кратковременная задержка при получении вызова из очереди после того, как она станет доступной.
>
> Представление вызовов очереди вызовов для агентов может конфликтовать с ограничениями маршрутизации на основе расположения. В этом случае агент получит всплывающее уведомление о вызове, но не сможет ответить на звонок. Это условие будет продолжаться до тех пор, пока другой агент не будет доступен для ответа на вызов, вызывающий объект зависает или не произойдет состояние ожидания очереди вызовов.  

### <a name="presence-based-call-routing"></a>Маршрутизация вызовов на основе присутствия

**Маршрутизация вызовов** на основе присутствия использует состояние доступности агентов вызовов, чтобы определить, должен ли агент быть включен в список маршрутизации вызовов для выбранного метода маршрутизации.

Агенты вызовов, состояние доступности которых имеет значение **"** Доступно", включены в список маршрутизации звонков и могут принимать звонки. Агенты, состояние доступности которых имеет любое другое состояние, исключаются из списка маршрутизации звонков и не будут получать вызовы, пока их состояние доступности не изменится на **"Доступно"**.

Маршрутизацию **вызовов на основе** присутствия можно включить с помощью любого из методов маршрутизации.

Если агент отключит получение звонков, он не будет включен в список маршрутизации звонков независимо от состояния доступности.

Мы рекомендуем включить **маршрутизацию на основе присутствия**.

> [!NOTE]
> Если  в качестве метода маршрутизации выбран самый длинный режим простоя, маршрутизация на основе присутствия является обязательной и автоматически включена, даже если переключатель маршрутизации на основе присутствия будет **отключен** и неактивен.
>
> Если маршрутизация на основе присутствия не включена и в очереди несколько вызовов, система будет одновременно представлять эти вызовы агентам независимо от их состояния присутствия. Это действие приведет к нескольким уведомлениям об звонках агентам, особенно если некоторые агенты не отвечают на исходный вызов, представленный им.
>
> При использовании маршрутизации на основе присутствия могут возникать моменты **, когда** агент получает вызов из очереди вскоре после того, как он становится недоступным, или кратковременная задержка при получении вызова из очереди после его получения.
>
> Агенты, использующие Skype для бизнеса клиента, не включаются в список маршрутизации вызовов, если включена маршрутизация на основе присутствия. Если у вас есть агенты, Skype для бизнеса, не включайте маршрутизацию вызовов на основе присутствия.

### <a name="call-agents-can-opt-out-of-taking-calls"></a>Агенты вызовов могут отказаться от звонков

Можно указать, могут ли агенты вызовов отказаться от звонков.

Мы рекомендуем включить **агенты вызовов, чтобы отказаться от звонков**.

### <a name="agent-alert-time"></a>Время оповещения агента

**Время оповещения агента** указывает, как долго будет звонить телефон агента, прежде чем очередь перенаправит вызов следующему агенту.

Мы рекомендуем задать для **оповещений агента** **значение 20 секунд**.

Выбрав параметры маршрутизации вызовов агента, нажмите кнопку "Далее"  в нижней части страницы "Добавление **очереди вызовов**".

# <a name="step-5-call-overflow"></a>[Шаг 5. Переполнение вызовов](#tab/call-overflow)

## <a name="step-5-set-how-to-handle-call-overflow"></a>Шаг 5. Настройка способа обработки переполнения вызовов

**Максимальное число вызовов в очереди** указывает максимальное количество вызовов, которые могут ожидать в очереди в любой момент времени.

Значение по умолчанию — 50, но может варьироваться от 0 до 200.

При превышении этого ограничения вызов обрабатывается, как указано в параметре "Когда достигается максимальное количество **вызовов** ".

Вы можете отключить **вызов или** перенаправить  его в любое из назначений маршрутизации параметров.

Например, вызывающий абонент может оставить голосовую почту для агентов в очереди.

*Новое— **теперь** при маршрутизации к общей голосовой почте доступны элементы управления "Пропустить системные сообщения голосовой почты", которые также применяются к запросам на добавление **приветственного сообщения** .*

Сведения о внешних передачах см. [в разделе "Предварительные](./plan-auto-attendant-call-queue.md#prerequisites) требования" и "Передача внешних номеров телефонов" — [технические](create-a-phone-system-auto-attendant.md?tabs=additional-resources) сведения о форматировании номеров.

> [!NOTE]
> Если максимальное число вызовов равно 0, приветствие не воспроизводится.

Выбрав параметры обработки переполнения звонков, нажмите кнопку "Далее" в  нижней части страницы "Добавление **очереди вызовов**".

# <a name="step-6-call-timeout"></a>[Шаг 6. Время ожидания звонка](#tab/call-timeout)

## <a name="step-6-set-how-to-handle-call-timeouts"></a>Шаг 6. Настройка обработки времени ожидания зова

**Время ожидания вызова: максимальное** время ожидания, задающее максимальное время удержания вызова в очереди перед перенаправлением или отключением.

Можно указать значение от 0 секунд до 45 минут.

Вы можете отключить **вызов или** перенаправить  его в одно из назначений маршрутизации вызовов.

Например, вызывающий абонент может оставить голосовую почту для агентов в очереди.

*Новое— **теперь** при маршрутизации к общей голосовой почте доступны элементы управления "Пропустить системные сообщения голосовой почты", которые также применяются к запросам на добавление **приветственного сообщения** .*

Сведения о внешних передачах см. [в разделе "Предварительные](./plan-auto-attendant-call-queue.md#prerequisites) требования" и "Передача внешних номеров телефонов" — [технические](create-a-phone-system-auto-attendant.md?tabs=additional-resources) сведения о форматировании номеров.

Выбрав параметры обработки времени ожидания звонков, нажмите кнопку **"Отправить** " в нижней части страницы "Добавление **очереди вызовов** ".
