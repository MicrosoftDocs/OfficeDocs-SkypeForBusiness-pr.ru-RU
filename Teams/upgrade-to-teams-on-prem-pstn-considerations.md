# <a name="pstn-considerations-for-upgrading-to-teams-from-skype-for-business-on-premises"></a>Вопросы, которые следует рассмотреть при переходе с локального приложения Skype для бизнеса на Teams

В этой статье описаны вопросы, которые следует учитывать при переходе на Teams.

 > [!NOTE]
 > - Использование телефонной системы в Teams поддерживается только в том случае, если учетной записи пользователя назначена политика обновления Teams в режиме только Teams.  
 > - Использование телефонной системы в Skype для бизнеса поддерживается только в том случае, если учетной записи пользователя назначена политика обновления Teams в режиме SfB. 
 > - Телефонная система не поддерживается, если учетной записи пользователя назначена политика обновления Teams в режиме "Острова".
 > - Любые параметры переадваровки звонков, группы звонков и делегирования из Skype для бизнеса не переносились, и их нужно будет повторно воссоздать для Teams.
 > - Общие сведения о облачных голосовых функций Microsoft Teams и о том, как определить, какое из голосовых решений Майкрософт должно быть правильной для вашей организации, см. в справке по планированию голосового [решения Teams.](cloud-voice-landing-page.md)


## <a name="pstn-calling-scenarios"></a>Сценарии звонков по ПСС

При переходе в режим TeamsOnly возможны четыре сценария звонков:

- [Пользователь Skype для бизнеса Online с планом звонков Майкрософт](#from-skype-for-business-online-with-microsoft-calling-plans). После обновления у этого пользователя сохранится план Microsoft Calling.

- [Пользователь в Skype для бизнеса Online с](#from-skype-for-business-online-with-on-premises-voice) функцией локального голосового подключения через локальное приложение Skype для бизнеса или Cloud Connector Edition. Обновление пользователя до Teams должно быть согласовано с миграцией пользователя на прямую маршрутику, чтобы гарантировать, что у пользователя TeamsOnly есть функциональность STN.

- [Пользователь локальной службы Skype](#from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing)для бизнеса с корпоративной голосовой связью , который переходит на интернет-службу и будет поддерживать локальное подключение к ОКП.  Чтобы перенести этого пользователя в Teams, необходимо переместить его локальное приложение Skype для бизнеса в облако и координировать его миграцию на Прямую маршрутию. 

- [Пользователь локальной службы Skype](#from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan)для бизнеса с корпоративной голосовой сетью , который переходит в интернет-службу и использует план Microsoft Calling.  Для переноса этого пользователя в Teams требуется переместить его локальное приложение Skype для бизнеса в облако и координировать перенос с помощью А) переноса номера телефона этого пользователя в план звонков Майкрософт или Б), назначая новый номер абонента из доступных регионов.

В этой статье представлен только общий обзор. Дополнительные сведения см. в сведениях о [прямой маршрутике](direct-routing-landing-page.md) и [планах звонков телефонной системы.](calling-plan-landing-page.md) 

## <a name="from-skype-for-business-online-with-microsoft-calling-plans"></a>Из Skype для бизнеса Online с планами звонков Майкрософт 

Это самый простой сценарий обновления, связанный с голосовым голосом. 

1. Убедитесь, что пользователям назначена лицензия на Teams. По умолчанию при назначении лицензии Microsoft 365 или Office 365 teams включена, поэтому если вы ранее не отключили лицензию Teams, никаких действий не требуется.

2.  Если у пользователей уже есть план звонков Майкрософт с номером телефона, необходимо лишь назначить пользователю режим TeamsOnly в TeamsUpgradePolicy.  Перед назначением режима TeamsOnly входящие звонки по STN будут переназначаться в клиент Skype для бизнеса пользователя. После обновления до режима TeamsOnly входящие звонки по STN будут перенаться в клиент Teams пользователя.  

## <a name="from-skype-for-business-online-with-on-premises-voice"></a>Из Skype для бизнеса Online с локальной голосовой связи

В этом сценарии пользователь уже подключен к Skype для бизнеса Online, но его подключение по ПСС является локальной, используя Skype для бизнеса Server в гибридном режиме или Cloud Connector Edition. Миграция этих пользователей в режим TeamsOnly с функциями STN означает включение прямой маршрутии, при которой линии связи STN подключаются напрямую к службе прямой маршрутации в облаке через ваш пограничный контроллер сеанса (SBC).

Ниже перечислены основные действия.  Шаги 1–4 указаны в предложенной последовательности, но их можно сделать в любом порядке. Важно, чтобы все эти этапы были выполнены до шага 5.

1. При настройке политики для всего клиента в одном из режимов Skype для бизнеса убедитесь, что все существующие пользователи островов явно назначили им режим "Острова", как описано выше.

2. Настройте для клиента прямую маршрутику. См. [сводку конфигурации direct Routing](#summary-of-per-tenant-configuration-of-direct-routing)для каждого клиента.

3. При желании настройте различные политики Teams для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и т. д.). Это можно сделать в любое время, но если вы хотите убедиться, что у пользователей должна быть правильная конфигурация при обновлении, лучше всего сделать это до обновления пользователя до режима TeamsOnly.

4. Подготовка выбранных пользователей к переносу голосовой почты 
   - При необходимости назначьте лицензию Teams.  Предположим, что пользователь уже работает в локальной голосовой системе Skype для бизнеса Online, у пользователя уже есть план 2 Skype для бизнеса, а также телефонная система Майкрософт. Оставьте оба этих плана включенными, включая лицензию Skype для бизнеса Online (план 2).  
   - Назначьте нужный OnlineVoiceRoutingPolicy. 

5. Обновление пользователя. Эти действия должны быть согласованы. 

   - В Microsoft 365 или Office 365 обновите пользователя до режима TeamsOnly (Grant-CsTeamsUpgradePolicy).
   - В SBC настройте голосовую маршрутику, чтобы включить входящие звонки, направив звонки на прямую маршрутику, а не на локального сервера-посредника.


## <a name="from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing"></a>От локального сервера Skype для бизнеса с корпоративной голосовой связью до прямой маршрутиза

В этом сценарии пользователь по-прежнему находится в локальной службе Skype для бизнеса, а его подключение по STN — к локальной сети. Миграция этих пользователей в режим TeamsOnly с функциями STN означает включение прямой маршрутации и перемещение пользователя в облако. 
 
Ниже перечислены основные действия.  Шаги 1–5 указаны в предложенной последовательности, но их можно сделать в любом порядке. Важно, чтобы все эти этапы были выполнены до шага 6.

1. Если вы назначите политику для всего клиента одному из режимов Skype для бизнеса, убедитесь, что существующие пользователи островов были совмедомы, явно назначив им режим "Острова", как описано выше.

2. Если вы еще не сделали этого, настройте гибридную конфигурацию Skype для бизнеса в [организации.](/SkypeForBusiness/hybrid/configure-hybrid-connectivity)

3. Настройте для клиента прямую маршрутику. См. [сводку конфигурации direct Routing](#summary-of-per-tenant-configuration-of-direct-routing)для каждого клиента.

4. При желании настройте различные политики Teams для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и т. д.). Это можно сделать в любое время, но если вы хотите убедиться, что у пользователей должна быть правильная конфигурация при обновлении, лучше сделать это до обновления пользователя до TeamsOnly.

5. При необходимости назначьте лицензии Microsoft 365 или Office 365.  У пользователя должны быть как Teams, так и Skype для бизнеса Online (план 2), а также телефонная система. Если skype для бизнеса Online (план 2) отключен, снова включить его.  

6. Обновление пользователя. Эти действия должны быть согласованы. 

   - Используя локальное приложение Skype для бизнеса, запустите Move-CsUser с переключателем -MoveToTeams. Если вы используете версию Skype для бизнеса Server, которая не поддерживает переключатель -MoveToTeams, сначала запустите Move-CsUser а затем назначьте режим TeamsOnly в клиентской удаленной консоли PowerShell или консоли администратора Teams.

   - В SBC настройте голосовую маршрутику, чтобы включить входящие звонки, направив звонки на прямую маршрутику, а не на локального сервера-посредника. 

   - В Microsoft 365 или Office 365: назначьте соответствующий OnlineVoiceRoutingPolicy, чтобы включить исходяющие звонки. 


## <a name="from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan"></a>От локального сервера Skype для бизнеса с корпоративной голосовой связи до плана звонков Майкрософт

В этом сценарии пользователь по-прежнему находится в локальной службе Skype для бизнеса, а его подключение по STN — к локальной сети. Перенос этих пользователей в режим TeamsOnly с функциями STN означает перемещение пользователя в облако и перенос номера из старого оператора в план Microsoft Calling или назначение пользователю нового номера. 

Ниже перечислены основные действия.Шаги 1–5 указаны в предложенной последовательности, но их можно сделать в любом порядке. Важно, чтобы все эти этапы были выполнены до шага 6. 

1. Если вы назначите политику для всего клиента одному из режимов Skype для бизнеса, убедитесь, что существующие пользователи островов были совмедомы, явно назначив им режим "Острова", как описано выше. 

2. Если вы еще не сделали этого, настройте гибридную конфигурацию Skype для бизнеса в [организации.](/SkypeForBusiness/hybrid/configure-hybrid-connectivity) 

3. При желании настройте различные политики Teams для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и т. д.). Это можно сделать в любое время, но если вы хотите убедиться, что у пользователей должна быть правильная конфигурация при обновлении, лучше сделать это до обновления пользователя до TeamsOnly. 

4. При необходимости назначьте лицензии Microsoft 365 или Office 365.У пользователя должны быть как Teams, так и Skype для бизнеса Online (план 2), а также телефонная система. Если skype для бизнеса Online (план 2) отключен, снова включить его.  

5. Получите номера телефонов для пользователей. (Подробные сведения [см. в управлении номерами телефонов для организации.)](./manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization.md)

   - Если вы будете повторно использовать номера, отправьте запрос на перенос номеров вашему оператору связи.  
   - Кроме того, вы можете получить новые номера непосредственно у корпорации Майкрософт. 

6. Обновите пользователя и при необходимости назначьте LineUri. Используя локальное приложение Skype для бизнеса, запустите Move-CsUser с переключателем -MoveToTeams.  

    - При переносе номеров в корпорацию Майкрософт необходимо координировать время переноса. 

    - Если вы используете новые номера от Майкрософт, вам потребуется изменить LineUri для пользователя. Это необходимо сделать после того, как пользователь переместит пользователя в Интернет с помощью Set-CsOnlineVoiceUser.  

## <a name="summary-of-per-tenant-configuration-of-direct-routing"></a>Сводка конфигурации для клиента прямой маршрутации 

1. Убедитесь, что ваш контроллер границы сеанса (SBC) поддерживает прямую маршрутику, просмотрев [этот список.](direct-routing-border-controllers.md) Кроме того, необходимо убедиться, что у вас правильная версия погона.  

2. Привяжи локальное SBC к службе прямой маршрутии Teams. Подробные сведения см. в теме Сопряжение SBC с службой [прямой маршрутии телефонной системы.](direct-routing-configure.md) 

3. Эта конфигурация фактически является зеркальным отражением локальной конфигурации. Конфигурация сети включает в себя: 
   - OnlineVoiceRoutingPolicy (на основе локальной voiceRoutingPolicy при миграции пользователей из Skype для бизнеса Online и VoicePolicy при переносе пользователей из локальной сети Корпоративная голосовая связь).
   - Объекты OnlinePSTNUsage (с учетом использования локального ПСОП). 
   - Объекты OnlineVoiceRoute (на основе локальной voiceRoute). 

Дополнительные сведения см. [в этой ссылке.](direct-routing-configure.md) 

## <a name="manage-enterprisevoiceenabled-property-during-migration"></a>Управление свойством EnterpriseVoiceEnabled во время миграции 

Для использования прямой маршрутики или плана звонков Майкрософт пользователь должен иметь EnterpriseVoiceEnabled=true в Azure AD, чтобы у пользователя были функции ЗВОНКОВ.  EnterpriseVoiceEnabled ("с поддержкой ОО") — это свойство (а не политика), которое существует как в локальном каталоге, так и в облаке. Значение в облаке очень важно для Teams.  Точная логика того, как с поддержкой ОО получается истина, зависит от следующего сценария: 

- Если пользователь включен в локальной сети Skype для бизнеса Server и ему назначена лицензия на телефонная система, перед перемещением пользователя в облако с помощью Move-CsUser, для пользователя в Сети будет назначена лицензия на EV-enabled=true. 

- Если существующему пользователю TeamsOnly или Skype для бизнеса Online назначена лицензия телефонная система, то по умолчанию не включена ОО.  Это также происходит, если пользователь из локальной сети перемещается в облако перед назначением телефонная система лицензии. В обоих случаях администратор должен указать следующий cmdlet: 

  ```PowerShell
  Set-CsUser -EnterpriseVoiceEnabled $True 
  ```
