# <a name="pstn-considerations-for-upgrading-to-teams-from-skype-for-business-on-premises"></a>Факторы, которые следует учесть при обновлении ПС TEAMS из локальной Skype для бизнеса службы

В этой статье описаны вопросы, которые следует учитывать при обновлении до Teams.

[!INCLUDE [sfbo-retirement-skype](../Skype/Hub/includes/sfbo-retirement.md)]

В следующих статьях описаны важные понятия обновления и принципы совместной использования.

- [Совместное Teams Skype для бизнеса](teams-and-skypeforbusiness-coexistence-and-interoperability.md)
- [Режимы сосуществования — справка](migration-interop-guidance-for-teams-with-skype.md)
- [Взаимодействие с клиентом Teams и соответствие режимам сосуществования](teams-client-experience-and-conformance-to-coexistence-modes.md)


 > [!NOTE]
 > - Использование телефонная система с Teams поддерживается только в том случае, если учетной записи пользователя назначена политика Teams обновления в режиме Teams только для пользователей.  
 > - Использование телефонная система с Skype для бизнеса поддерживается только в том случае, если учетной записи пользователя назначена политика Teams обновления в режиме SFB. 
 > - телефонная система не поддерживается, если учетной записи пользователя назначена политика Teams в режиме "Острова".
 > - Любые параметры переад вызовов, группового звонка и делегирования из Skype для бизнеса не переносились, и их потребуется повторно воссоздать для Teams.
 > - Общие сведения о Microsoft Teams голосовых функций в облаке и о том, как определить, какое из решений Голосовой поддержки Майкрософт будет правильной для вашей организации, см. в Teams [голосового решения.](cloud-voice-landing-page.md)


## <a name="pstn-calling-scenarios"></a>Сценарии звонков по ННП

При переходе в режим TeamsOnly возможны четыре сценария звонков:

- [Пользователь в Skype для бизнеса Online с планом звонков Майкрософт](#from-skype-for-business-online-with-microsoft-calling-plans). После обновления у этого пользователя сохранится план Microsoft Calling.

- [Пользователь в Skype для бизнеса Online с](#from-skype-for-business-online-with-on-premises-voice) функцией локального голосового подключения через Skype для бизнеса или Cloud Connector Edition. Чтобы гарантировать, что у пользователя TeamsOnly есть функциональность STN, необходимо координировать его обновление до Teams с переходом пользователя на прямую маршрутику.

- [Пользователь в Skype для бизнеса локальной](#from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing)службе Корпоративная голосовая связь , который будет передвигаться на веб-сайт и поддерживать локальное подключение к ОКП.  Чтобы перенести этого пользователя в Teams, необходимо переместить его локальное Skype для бизнеса учетную запись в облако и координировать его с миграцией на Прямую маршрутию. 

- [Пользователь в Skype для бизнеса локальной](#from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan)службе Корпоративная голосовая связь , который переходит в интернет-службу и использует план Microsoft Calling.  Чтобы перенести этого пользователя в Teams, необходимо переместить его Skype для бизнеса в облако. Необходимо координировать перемещение с помощью А) переноса номера телефона этого пользователя в план звонков Майкрософт или Б), назначая новый номер абонента из доступных регионов.

В этой статье представлен только общий обзор. Дополнительные сведения см. [в телефонная система маршрутизов и](direct-routing-landing-page.md) планов [звонков.](calling-plan-landing-page.md) 

## <a name="from-skype-for-business-online-with-microsoft-calling-plans"></a>Из Skype для бизнеса Online с планами звонков Майкрософт 

Это самый простой сценарий обновления с голосовым голосом. 

1. Убедитесь, что пользователям назначена Teams лицензия. По умолчанию при назначении лицензии Microsoft 365 или Office 365 лицензия Teams включена. Если вы ранее не отключили Teams лицензию, никаких действий не требуется.

2.  Если у пользователей уже есть план звонков Майкрософт с номером телефона, необходимо лишь назначить пользователю режим TeamsOnly в TeamsUpgradePolicy. Перед назначением режима TeamsOnly входящие звонки по STN будут переназначаться в клиент Skype для бизнеса пользователя. После обновления до режима TeamsOnly входящие звонки по STN будут перенаться в клиент Teams пользователя.  

## <a name="from-skype-for-business-online-with-on-premises-voice"></a>Из Skype для бизнеса Online с локальной голосовой почтой

В этом случае пользователь уже находится в Skype для бизнеса Online. Подключение по STN пользователя является локальной версией, Skype для бизнеса Server гибридном режиме или Cloud Connector Edition. Чтобы перенести этих пользователей в режим TeamsOnly с функциями STN, необходимо включить для них прямую маршрутику. При прямой маршрутике линии связи STN подключаются напрямую к службе прямой маршрутии через ваш локальной пограничный контроллер сеанса (SBC).

Ниже перечислены основные действия.  Шаги 1–4 указаны в предложенной последовательности, но их можно сделать в любом порядке. Эти действия следует завершить до шага 5.

1. Если вы задаете политику для всего клиента в одном из режимов Skype для бизнеса, не забудьте закорестить существующих пользователей Островов, явно назначив им режим "Острова", как описано выше.

2. Настройте для клиента прямую маршрутику. См. [сводку конфигурации direct Routing](#summary-of-per-tenant-configuration-of-direct-routing)для каждого клиента.

3. При желании настройте различные политики Teams для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и другие). Вы можете настроить пуази в любое время. Однако если вы хотите убедиться, что у пользователей должна быть правильная конфигурация при обновлении, настройте эти политики перед обновлением пользователя до режима TeamsOnly.

4. Подготовка выбранных пользователей к переносу голосовой почты 
   - При необходимости назначьте лицензию Teams лицензии.  Предположим, что пользователь уже работает в локальной Skype для бизнеса Online, у него уже есть Skype для бизнеса план 2 и Телефон (Майкрософт) Система. Оставьте оба этих плана включенными, включая Skype для бизнеса Online (план 2).  
   - Назначьте нужный OnlineVoiceRoutingPolicy. 

5. Обновление пользователя. Эти действия должны быть согласованы. 

   - В Microsoft 365 или Office 365 до режима TeamsOnly (Grant-CsTeamsUpgradePolicy).
   - На SBC настройте голосовую маршрутику, чтобы включить входящие звонки, направив звонки на прямую маршрутику, а не на локального сервера-посредника.


## <a name="from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing"></a>От Skype для бизнеса Server локальной и Корпоративная голосовая связь к прямой маршрутике

В этом случае пользователь по-прежнему будет Skype для бизнеса локально. Подключение по STN пользователя также является локальной. Чтобы перенести этого пользователя в режим TeamsOnly с функциональными возможностями STN, необходимо включить для пользователя прямую маршрутику, а затем переместить пользователя в облако. 
 
Ниже перечислены основные действия. Шаги 1–5 указаны в предложенной последовательности, но их можно сделать в любом порядке. Перед шагом 6 необходимо выполнить шаги 1–5.

1. Если вы назначите политику для всего клиента одному из режимов Skype для бизнеса, убедитесь, что существующие пользователи островов были совмедомы, явно назначив им режим "Острова", как описано выше.

2. Если вы еще не сделали этого, настройте в организации гибридное [Skype для бизнеса](/SkypeForBusiness/hybrid/configure-hybrid-connectivity).

3. Настройте для клиента прямую маршрутику. См. [сводку конфигурации direct Routing](#summary-of-per-tenant-configuration-of-direct-routing)для каждого клиента.

4. При желании настройте различные политики Teams для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy). Вы можете настроить политики в любое время. Однако если вы хотите убедиться, что у пользователей должна быть правильная конфигурация при обновлении, настройте эти политики перед обновлением пользователя до TeamsOnly.

5. При необходимости Microsoft 365 лицензии Office 365 лицензии.  Пользователю должны быть как Teams, так и Skype для бизнеса Online (план 2) и телефонная система. Если Skype для бизнеса Online (план 2) отключен, снова включим его.  

6. Обновление пользователя. Эти действия должны быть согласованы. 

   - Используя инструменты локального Skype для бизнеса, запустите Move-CsUser с переключателем -MoveToTeams. Если вы используете версию Skype для бизнеса Server, которая не поддерживает переключатель -MoveToTeams, сначала запустите Move-CsUser, а затем назначьте режим TeamsOnly в клиентской удаленной консоли PowerShell или Teams администратора.

   - На SBC настройте голосовую маршрутику, чтобы включить входящие звонки, направив звонки на прямую маршрутику, а не на локального сервера-посредника. 

   - В Microsoft 365 или Office 365: назначьте соответствующий OnlineVoiceRoutingPolicy, чтобы включить исходяющие звонки. 


## <a name="from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan"></a>От Skype для бизнеса Server локальной службы с Корпоративная голосовая связь до плана звонков Майкрософт

В этом случае пользователь по-прежнему будет Skype для бизнеса локально. Подключение по STN пользователя также является локальной. Чтобы перенести этого пользователя в режим TeamsOnly с функциями STN, необходимо переместить пользователя в облако, а также перенести его номер из старого оператора в план Microsoft Calling или назначить новый номер пользователю. 

Ниже перечислены основные действия.Шаги 1–5 указаны в предложенной последовательности, но их можно сделать в любом порядке. Перед шагом 6 необходимо выполнить шаги 1–5. 

1. Если вы назначите политику для всего клиента одному из режимов Skype для бизнеса, убедитесь, что существующие пользователи островов были совмедомы, явно назначив им режим "Острова", как описано выше. 

2. Если вы еще не сделали этого, настройте в организации гибридное [Skype для бизнеса](/SkypeForBusiness/hybrid/configure-hybrid-connectivity). 

3. При желании настройте различные политики Teams для этих пользователей (например, TeamsMessagingPolicy, TeamsMeetingPolicy и другие). Вы можете настроить политики в любое время. Однако если вы хотите убедиться, что у пользователей должна быть правильная конфигурация при обновлении, настройте эти политики перед обновлением пользователя до TeamsOnly. 

4. При необходимости Microsoft 365 лицензии Office 365 лицензии.Пользователю должны быть как Teams, так и Skype для бизнеса Online (план 2) и телефонная система. Если Skype для бизнеса Online (план 2) отключен, снова включим его.  

5. Получите номера телефонов для пользователей. (Подробные сведения [см. в управлении номерами телефонов для организации.)](./manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization.md)

   - Если вы будете повторно использовать номера, отправьте запрос на перенос номеров оператору связи.  
   - Кроме того, вы можете получить новые номера непосредственно у корпорации Майкрософт. 

6. Обновите пользователя и при необходимости назначьте LineUri. Используя инструменты локального Skype для бизнеса, запустите Move-CsUser с переключателем -MoveToTeams.  

    - При переносе номеров в корпорацию Майкрософт необходимо согласовать время переноса. 

    - Если вы используете новые номера от Майкрософт, вам потребуется изменить LineUri для пользователя после его перемещении в Интернет с помощью Set-CsPhoneNumberAssignment.  

## <a name="summary-of-per-tenant-configuration-of-direct-routing"></a>Сводка конфигурации для клиента прямой маршрутации 

1. Убедитесь, что ваш контроллер границы сеанса (SBC) поддерживает прямую маршрутику, просмотрев [этот список.](direct-routing-border-controllers.md) Кроме того, убедитесь, что у вас правильная версия программного обеспечения.  

2. Привяжи свою локальное SBC к службе Teams Direct Routing. Подробные сведения см. в теме Связь SBC с службой прямой [маршрутии телефонная система.](direct-routing-configure.md) 

3. Эта конфигурация фактически является зеркальным отражением локальной конфигурации. Конфигурация сети включает в себя: 
   - OnlineVoiceRoutingPolicy (на основе локальной voiceRoutingPolicy при миграции пользователей из Skype для бизнеса Online и VoicePolicy при переносе пользователей из локальной сети с помощью Корпоративная голосовая связь).
   - Объекты OnlinePSTNUsage (с учетом использования локального ПСОП). 
   - Объекты OnlineVoiceRoute (на основе локальной voiceRoute). 

Дополнительные сведения см. [в теме Настройка прямой маршрутии.](direct-routing-configure.md) 

## <a name="manage-enterprisevoiceenabled-property-during-migration"></a>Управление свойством EnterpriseVoiceEnabled во время миграции 

Для использования прямой маршрутики или плана Microsoft Calling пользователь должен иметь EnterpriseVoiceEnabled=true в Azure AD, чтобы пользователь мог пользоваться функциями ПСС.  EnterpriseVoiceEnabled ("с поддержкой ОО") — это свойство (а не политика), которое существует как в локальном каталоге, так и в облаке. Значение в облаке очень важно для Teams.  Точная логика того, как с поддержкой ОО получается истина, зависит от следующего сценария: 

- Если пользователь включил ОО в локальной Skype для бизнеса Server и ему назначена лицензия телефонная система до перемещения пользователя в облако с помощью Move-CsUser, ему будет назначена лицензия на EV-enabled=true. 

- Если существующему пользователю TeamsOnly или Skype для бизнеса Online назначена лицензия телефонная система, то по умолчанию не включена оО. Это также происходит, если пользователь из локальной сети перемещается в облако перед назначением телефонная система лицензии. В обоих случаях администратор должен указать следующий cmdlet: 

  ```PowerShell
  Set-CsPhoneNumberAssignment -EnterpriseVoiceEnabled $True 
  ```
