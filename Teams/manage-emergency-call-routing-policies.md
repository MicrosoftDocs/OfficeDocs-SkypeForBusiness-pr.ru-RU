# <a name="manage-emergency-call-routing-policies-in-microsoft-teams"></a>Управление политиками маршрутизов экстренных вызовов в Microsoft Teams

Если в вашей [](direct-routing-landing-page.md) организации развернута прямая маршрутия телефонной системы, вы можете использовать политики маршрутизации экстренных вызовов в Microsoft Teams, чтобы настроить номера для экстренного вызова и настроить маршруты экстренных вызовов. Политика маршрутинга экстренных вызовов определяет, включены ли улучшенные экстренные службы для пользователей, которым назначена политика, номера, используемые для вызова экстренных служб (например, 911 в США), а также как перена могут перенапределяться звонки в экстренные службы.

Вы управляете политиками маршрутизов экстренных вызовов, переходить к политикам голосового экстренного вызова в Центре администрирования Microsoft Teams или с помощью   >   Windows PowerShell. Политики могут быть назначены пользователям и [сетевым сайтам.](cloud-voice-network-settings.md)

Для пользователей можно использовать глобальную (по умолчанию в организации) политику или создавать и назначать настраиваемые политики. Пользователи будут автоматически получать глобальную политику, если вы не создайте и не назначите ее. Помните, что вы можете изменить параметры глобальной политики, но не можете переименовать или удалить ее. Для сетевых сайтов создаются и назначаются настраиваемые политики.

Если политика маршрутинга экстренных вызовов назначена сетевому сайту и пользователю, а этот пользователь находится на этом сайте сети, политика, назначенная сетевому сайту, переопределит политику, назначенную пользователю.

## <a name="create-a-custom-emergency-call-routing-policy"></a>Создание настраиваемой политики маршрутизов экстренных вызовов

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. В левой области навигации Центра администрирования Microsoft Teams перейдите к политикам голосового экстренного вызова, а затем щелкните вкладку "Политики  >   **маршрутинга экстренных вызовов".**
2. Нажмите **Добавить**.
3. Введите имя и описание для политики.
4. Чтобы включить динамические экстренные вызовы, включив **динамические экстренные вызовы.** Если динамические экстренные вызовы включены, Teams извлекает сведения о политике и расположении из службы и включает их в экстренный вызов.
5. Определить один или несколько номеров для экстренного срочная ситуация. Для этого в категории **"Номера для экстренного помощи"** **нажмите** кнопку "Добавить" и сделайте следующее:
    1. **Строка набора номера для экстренного помощи:** введите строку набора экстренных служб. Эта строка набора указывает на то, что звонок является экстренным.
        > [!NOTE]
        > Для прямой маршрутинга мы переходить от клиентов Teams, которые отправляют экстренные вызовы со строкой "+" перед строкой вызова экстренной помощи. До завершения перехода образец голосового пути для обеспечения совпадения со строкой экстренного набора номера должен обеспечить соответствие строкам, которые имеют или не имеют предыдущего "+", например 911 и +911. Например, ^ \\ +?911 или .*.
    2. **Маска набора номера** для экстренного помощи: для каждого номера можно задать ноль или несколько масок для экстренного набора номера. Маска набора номера — это номер, который вы хотите перевести на значение в строке набора экстренных служб. Это позволяет звонить на другие номера экстренных служб и по-прежнему может звонить в экстренные службы. <br>Например, вы можете добавить в качестве маски набора номера экстренных служб номер 112 — номер для экстренных служб для большинства стран Европы, а номер 911 — в качестве строки экстренного набора. Пользователь Teams из Европы может не знать, что 911 — это номер для экстренного вызова в США, а при наборе номера 112 звоните по номеру 911. Чтобы определить несколько масок набора номера, разделите каждое значение за semicolon. Например, 112:212.
    3. **Запись использования ДНР:** выберите запись использования телефонной сети общего пользования (ННР). Запись использования ДНР используется для определения маршрутов, используемых для маршрутизации экстренных вызовов от пользователей, которым разрешено их использовать. Маршрут, связанный с этим использованием, должен указать на магистраль SIP, выделенную для экстренных вызовов, или на шлюз ELIN, который перенаправляет экстренные вызовы в ближайший КПЭ.

    > [!NOTE]
    > Строки набора и маски набора должны быть уникальными в политике. Это означает, что для политики можно определить несколько номеров для экстренного помощи и настроить несколько масок набора для строки набора номера, но каждую из них необходимо использовать только один раз.

6. Щелкните **Сохранить**.

## <a name="edit-an-emergency-call-routing-policy"></a>Изменение политики маршрутизов экстренных вызовов

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

Вы можете изменить глобальную политику или любые настраиваемые политики, которые вы создаете.

1. В левой области навигации Центра администрирования Microsoft Teams перейдите к политикам голосового экстренного вызова, а затем щелкните вкладку "Политики  >   **маршрутинга экстренных вызовов".**
2. Выберите политику, щелкнув слева от ее имени и нажав кнопку **"Изменить".**
3. Внести нужные изменения и нажмите кнопку **"Сохранить".**

### <a name="using-powershell"></a>С помощью PowerShell

См. [set-CsTeamsEmergencyCallRoutingPolicy.](https://docs.microsoft.com/powershell/module/skype/set-csteamsemergencycallroutingpolicy)

## <a name="assign-a-custom-emergency-call-routing-policy-to-a-network-site"></a>Назначение настраиваемой политики маршрутизов экстренных вызовов на сетевой сайт

Используйте [командлет Set-CsTenantNetworkSite,](https://docs.microsoft.com/powershell/module/skype/set-cstenantnetworksite) чтобы назначить политику маршрутизации экстренных вызовов на сетевой сайт.

В этом примере показано, как назначить политику под названием "Маршрутная политика для экстренных вызовов 1" сайту "Сайт1".

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallRoutingPolicy "Emergency Call Routing Policy 1"
```
