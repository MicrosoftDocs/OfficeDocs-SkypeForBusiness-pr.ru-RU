# <a name="manage-emergency-call-routing-policies-for-direct-routing"></a>Управление политиками маршрутизации экстренных вызовов для прямой маршрутизации

Если вы развернули прямую [](direct-routing-landing-page.md) маршрутизацию в организации, вы можете использовать политики маршрутизации экстренных вызовов в Microsoft Teams, чтобы настроить номера экстренных служб и указать способ маршрутизации экстренных вызовов. Политика маршрутизации экстренных вызовов определяет, включены ли расширенные экстренные службы для пользователей, которым назначена политика, какие номера используются для вызова экстренных служб (например, 911 в США) и как направляются вызовы к экстренным службам. 

> [!Note]
> **Обратите внимание, что эти политики маршрутизации звонков применяются только к прямой маршрутизации— они не применяются к тарифным планам или Operator Connect.**

Вы управляете политиками маршрутизации экстренных вызовов, перейдя к политикам **VoiceEmergency**  >  в центре администрирования Microsoft Teams или с помощью Windows PowerShell. Политики можно назначить пользователям и [сетевым сайтам](cloud-voice-network-settings.md).

Для пользователей можно использовать глобальную политику (по умолчанию для всей организации) или создавать и назначать пользовательские политики. Пользователи будут автоматически получать глобальную политику, если вы не создадим и не назначите пользовательскую политику. Помните, что вы можете изменить параметры глобальной политики, но не можете переименовать или удалить их. Для сетевых сайтов создаются и назначаются настраиваемые политики.

Если вы назначите политику маршрутизации экстренных вызовов сетевому сайту и пользователю и если этот пользователь находится на этом сетевом сайте, политика, назначенная сетевому сайту, переопределяет политику, назначенную пользователю.

## <a name="create-a-custom-emergency-call-routing-policy"></a>Создание настраиваемой политики маршрутизации экстренных вызовов

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. В левой области навигации центра администрирования Microsoft Teams перейдите к политикам **VoiceEmergency** >  и перейдите на вкладку "Политики маршрутизации вызовов".
2. Нажмите **Добавить**.
3. Введите имя и описание для политики.
4. Чтобы включить динамические экстренные вызовы, включите **динамические экстренные вызовы**. Если динамический экстренный вызов включен, Teams извлекает сведения о политике и расположении из службы и включает эти сведения в составе экстренных вызовов.
5. Определите один или несколько номеров экстренных служб. Для этого в разделе **"Номера экстренных служб**" нажмите кнопку **"** Добавить" и выполните следующие действия:
    1. **Строка набора номера для экстренного реагирования**: введите строку набора номера для экстренного реагирования. Эта строка набора указывает, что вызов является экстренным вызовом, а шаблон маршрута должен точно соответствовать этой строке набора. 
        > [!NOTE]
        > **Для прямой маршрутизации Teams клиенты больше не отправляют экстренные вызовы с "+" перед строкой набора экстренных служб. Убедитесь, что шаблон маршрута голосовой связи, соответствующий строке набора экстренных служб, отражает это изменение.**
    2. **Маска набора экстренных служб**. Для каждого номера экстренных служб можно указать ноль или более масок набора экстренных служб. Маска набора номера — это номер, который требуется перевести в значение строки набора экстренных служб. Это позволяет вызывать альтернативные номера экстренных служб и по-прежнему обращаться к экстренным службам. <br>Например, вы добавляете 112 в качестве маски набора номера для экстренного реагирования, которая является номером службы экстренного реагирования для большей части Европы, и 911 в качестве строки набора номера для экстренного реагирования. Пользователь Teams из Европы может не знать, что 911 — это номер для экстренного реагирования в США, а при вызове номера 112 вызов выполняется по номеру 911. Чтобы определить несколько масок набора, разделите каждое значение точкой с запятой. Например, 112;212.
    3. **Запись об использовании ТСОП**: выберите запись об использовании телефонной сети общего пользования (ТСОП). Запись об использовании ТСОП используется для определения маршрута, используемого для маршрутизации экстренных вызовов от пользователей, имеющих право на их использование. Маршрут, связанный с этим использованием, должен указывать на магистраль протокола SIP, выделенную для экстренных вызовов, или на шлюз ELIN, который направляет экстренные вызовы к ближайшей общедоступной точке ответа (PSAP).

    > [!NOTE]
    > Строки набора и маски набора должны быть уникальными в политике. Это означает, что для политики можно определить несколько экстренных номеров и задать несколько масок набора для строки набора, но каждая строка набора и маска набора должны использоваться только один раз.

6. Нажмите кнопку **Сохранить**.

## <a name="edit-an-emergency-call-routing-policy"></a>Изменение политики маршрутизации экстренных вызовов

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

Вы можете изменить глобальную политику или любые настраиваемые политики, которые вы создаете.

1. В левой области навигации центра администрирования Microsoft Teams перейдите к политикам **VoiceEmergency** >  и перейдите на вкладку "Политики маршрутизации вызовов".
2. Выберите политику, щелкнув слева от ее имени, а затем нажмите **Изменить**.
3. Внесите необходимые изменения и нажмите кнопку " **Сохранить"**.

### <a name="using-powershell"></a>С помощью PowerShell

См [. раздел Set-CsTeamsEmergencyCallRoutingPolicy](/powershell/module/skype/set-csteamsemergencycallroutingpolicy).

## <a name="assign-a-custom-emergency-call-routing-policy-to-a-network-site"></a>Назначение пользовательской политики маршрутизации экстренных вызовов сетевому сайту

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

Вы можете назначить глобальную политику или любые настраиваемые политики, которые вы создаете.

1. В левой области навигации центра администрирования Microsoft Teams выберите топологию **LocationsNetwork** >  и перейдите на вкладку **"Сетевые сайты**".
2. Выберите сайт, щелкнув слева от имени, а затем нажмите кнопку "Изменить **"**.
3. В **разделе "Политика маршрутизации экстренных вызовов**" выберите политику и нажмите кнопку " **Сохранить"**.

### <a name="using-powershell"></a>С помощью PowerShell

Используйте [командлет Set-CsTenantNetworkSite](/powershell/module/skype/set-cstenantnetworksite) , чтобы назначить политику маршрутизации экстренных вызовов сетевому сайту.

В этом примере показано, как назначить политику с именем "Политика маршрутизации экстренных вызовов 1" сайту Site1.

```PowerShell
Set-CsTenantNetworkSite -identity "site1" -EmergencyCallRoutingPolicy "Emergency Call Routing Policy 1"
```
