# <a name="define-my-success"></a>Определение показателей успеха

В этой статье приводятся общие сведения о требованиях к определению успеха при развертывании телефонной системы с планами звонков для вашей организации. Правильно определив, что будет выглядеть успешно, вы можете оценить результаты по ходу выполнения развертывания и убедиться в том, что нужные результаты являются нужными.

## <a name="define-business-use-cases-for-phone-system-with-calling-plans"></a>Определение вариантов использования для бизнеса в телефонной системе с помощью планов звонков

Организации могут использовать телефонную систему с тарифными планами, чтобы Улучшите представление на свое рабочее место, позволяя пользователям совершать звонки по телефонной связи с компьютеров и мобильных устройств.

Модернизатион на рабочем месте может быть частью любого количества сценариев — рабочей реализации на основе действий, крупного перемещения Office, обновления Office, а также отключать старые решения для обмена данными филиалов, а также для прекращения использования общедоступного коммутируемого телефона. контракт от поставщика услуг сети (КТСОП) и т. д.

Для начала работы с заинтересованными лицами в проекте необходимо определить варианты использования для бизнеса, которые поддерживают реализацию телефонной системы с планами звонков.

Варианты использования бизнеса предназначены для определения и документирования ожидаемых, измеримых результатов бизнеса и включения следующих параметров:

-   Описание текущего бизнес-процесса

-   Проблемы с существующим бизнес-процессом

-   Каким образом технология позволяет преодолеть эти трудности

-   Измеримые бизнес-результаты, ожидаемые в случае преодоления этих трудностей

## <a name="define-okrs-ksis-and-risks"></a>Определение Окрс, КСИС и рисков

С помощью собранных заинтересованных лиц проекта можно преобразовывать варианты использования для бизнеса, организационную область и временные шкалы проекта в цели и ключевые результаты (Окрс), а также показатели успешности проекта могут быть определены как список индикаторов успеха (КСИС).

Полное участие заинтересованных лиц из проекта в определении Окрс и КСИС важно для того, чтобы убедиться в том, что они являются собственностью и выравнивают эти показатели успеха в бизнес-требования Организации.

Окрс содержит цели, заданные в начале проекта, и вы определяете измеряемые ключевые результаты на квартальной основе. Вы просматриваете ключевые результаты ежемесячно, чтобы отслеживать состояние всего проекта, а также — на основании хода выполнения — вы можете настроить квартальные планы по мере необходимости.

КСИС измерение качество и успешное достижение основных результатов и дополнение к двоичной природе Окрс (достижение или не достигается) за счет подробных и/или неудачных результатов.

При определении КСИС мы рекомендуем использовать особые, измеримые, назначаемые, реалистичные условия, связанные со временем (SMART).

-   Определено: ориентирован на определенную область для улучшения

-   Измеримые: количественное или хотя бы предложите индикатор "ход выполнения"

-   Назначаемые возможности: укажите, кто будет делать это

-   Реалистичные: определение результатов, которые могут быть достигнуты с учетом доступных ресурсов

-   Относящихся к времени: определяет, когда можно достичь результатов

## <a name="establish-a-steering-committee"></a>Создание Комитета для управления

Комитет по управлению — это управляющая группа ключевых заинтересованных лиц и руководителей проектов, которые были объединены для продвижения проекта или программы в соответствии с определенными бизнес-результатами. Комиссия по управления непосредственным образом несет ответственность за способ доставки проекта, а *также о* том, *как* он доставляется в бизнес.

Для каждого проекта требуется согласованное видение и Устав. Для доставки нужных результатов из проекта концепция должна быть четко определена, и ее необходимо отслеживать и поддерживать. Это становится обязанностью Комиссии по контролю: для принятия решений по обоснованию, рекомендации, предоставления стратегического анализа для использования в качестве ответственности за инициативы проекта и, при необходимости, для удаления блокируемых.

Ваша организация должна быть важна в формировании комитетов по управления. Комиссия обязана удостовериться, что проект достиг целей бизнеса, которые вы определили для изменения во всей Организации, регулярно соблюдайте вопросы, чтобы обсудить текущий период проекта, а также устранить препятствия, обнаруженные на одинаков.

Комитет должен определить свою Устав, чтобы включить некоторые ключевые цели:

-   Обеспечьте строгое выравнивание между группой проекта и спонсором руководителя или руководителем лидерства.

-   Соответственен за состояние проекта для вашего спонсора или руководящего лидера.

-   Разрешите руководителям и руководителям по лидерам по проекту предоставлять направление и ввод для проекта, а также обеспечивать соответствие требованиям бизнес-целей, изменяя планы проектов, важные результаты (Окрс) и другие действия проекта.

Комиссия по управлениям отвечает за период времени существования проекта для обеспечения выравнивания между организационным руководителем и командой проекта. Это критическое собрание означает, что в проекте есть полная техническая поддержка и все отзывы, предоставленные руководством лидера за успешное выполнение проекта. Комиссия использует эти собрания, чтобы получить информацию о состоянии проекта, а также:

-   Согласитесь с деловыми результатами, которые выравниваются в соответствии с деловым случаем и гарантируют, что проект будет доставляться в соответствии с этими результатами.

-   Проверка и утверждение проекта на точность и соответствие требованиям для деловой ситуации.

-   Просматривайте и проверяйте изменения, внесенные в бизнес-сценарий, которые могут повлиять на определенные результаты.

-   Стратегические решения в отношении приоритетов конечных результатов проекта и утверждения промежуточных конечных результатов.

-   Определять, отслеживать и устранять перерывы, риски и проблемы, требующие от Комитета дополнительные возможности.

-   Соберите поддержку в рамках проекта руководителя или руководителя лидерства за проблемы, требующие эскалации, приоритизации и устранения конфликтов между деловыми подразделениями заинтересованных лиц. 

-   Предоставьте формальное мнение и рекомендации для руководства руководителя, информационная доска изменений, а также других заинтересованных лиц для бизнеса и ИТ.
