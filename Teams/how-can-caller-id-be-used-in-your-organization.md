# <a name="how-can-caller-id-be-used-in-your-organization"></a>Как можно использовать идентификатор звонящего в организации

ИД звонящая состоит из двух идентифицируемых пользователем фрагментов информации:

- Номер телефона (обычно он называется CLID или ИД строки звонка). Это общедоступный переключимся телефонный номер (ПСП), который представлен в качестве идентификации вызывающего.

- Имя вызываемой стороны (обычно называется CNAM). 
  
Функция "ИД вызываемого звоня" доступна всем телефонная система пользователям независимо от варианта подключения к ДНР:

- Планы звонков Майкрософт 

- Прямая маршрутизация телефонной системы 
  
Вы можете управлять ИД вызываемого звонка как для входящие, так и для исходящие вызовы с помощью политики CallingLineIdentity. Дополнительные сведения см. в [дополнительных сведениях об ИД строки звонков и названии вызываемой стороны.](more-about-calling-line-id-and-calling-party-name.md)

  
## <a name="outbound-pstn-caller-id"></a>ИД вызываемой службы для исходящие вызовы через ПСС

Для исходящие ИД вызываемой службы ПСС доступны следующие параметры: 
  
- Номер телефона, назначенный пользователю, который задан по умолчанию.

- Анонимный доступ, который можно получить, удалив презентацию номера ЗВОНКОВ пользователя. 

- Замените номер телефона, который может быть:

  - Номер телефона, классифицированный как услуга, и бесплатный номер в перечне номеров телефонов планов звонков. Как правило, она назначена авто Teams или очереди вызовов.

  - Номер локального телефона в рамках прямой маршрутиза путем перенаправки, который назначен учетной записи ресурса, используемой Teams автозаправщиком или очередью вызовов. 

- Имя вызываемой стороны или CNAM, задаваемая в исходящие вызовы по ПСN.  
    
Дополнительные сведения см. в [этой](./set-the-caller-id-for-a-user.md)записи.
  
### <a name="end-user-control-of-outbound-caller-id"></a>Управление конечным пользователем для ид исходящие вызовы

Пользователи могут изменить параметр "Анонимный" в параметре "Анонимный",  задав атрибут EnableUserOverride. 

Если для исходячего звонка задан анонимный ИД, EnableUserOverride не действует, а для него всегда установлено анонимное. По умолчанию EnableUserOverride имеет значение False.

Конечные пользователи могут сделать свой ИД звонячего анонимным, выбрав Параметры > **Вызовы**, а затем в области ИД звонив **выберите** Скрыть номер телефона и данные профиля для всех **звонков**.

### <a name="notes"></a>Notes

Учитывайте следующее.

- Для исходящие номера телефонов нельзя назначать следующие типы номеров:

  - Любые телефонные номера, классифицированные как пользовательские, в перечне номеров телефонов планов звонков.

  - Любой номер локального телефона через прямую маршрутику, который назначен пользователю.

  - Номер Skype для бизнеса Server локального телефона.

- Подстановка номеров телефонов для учетных записей ресурсов работает только для Teams пользователей. Замена номеров телефонов службы работает как для Skype для бизнеса Online, так и Teams пользователей.

- Имя вызываемой стороны отправляется только при звонках, когда ИД звонящая заменяется номером телефона LineUri, номером телефона учетной записи службы или ресурса, а также при Teams пользователем.

- Имя вызываемой стороны может иметь не более 200 символов, но система, ходящий вниз, может поддерживать меньше символов.

- При прямой маршрутике подстановка номеров телефонов и имя вызываемой стороны отправляются в заглавной области FROM SIP. Если соответствующая веб-часть OnlinePstnGateway настроена с помощью ForwardPai = True, заглавная карточка SIP P-НАСТОЛ-IDENTITY будет содержать реально вызывающего пользователя.

- EnableUserOverride имеет приоритет над другими настройками в политике, если подстановка не задана для параметра Анонимно. Например, предположим, что экземпляр политики имеет подстановку с помощью учетной записи ресурса, а enableUserOverride задан и включен пользователем. В этом случае ИД исходящие звонки будут заблокированы и будут использоваться анонимные. Если для экземпляра политики задана подстановка Анонимный и EnableUserOverride, то исходящие звонки всегда будут анонимными независимо от настроек конечного пользователя.

   
## <a name="inbound-caller-id"></a>ИД входящие звонящие

телефонная система номер входящих внешних телефонов в качестве ИД звоня. Если номер связан с пользователем или контактом в Azure AD или личном контакте, клиенты Skype для бизнеса и Teams будут показывать ИД вызываемого пользователя на основе этой информации. Если номер телефона не находится в Azure AD или личном контакте, отображается отображаемая телефонная связь, если она доступна.

Атрибут BlockIncomingCallerID обеспечивает блокирование идентификатора абонента на входящие вызовы в сети ТСОП. Можно задать этот атрибут, но он не доступен вашим пользователям на странице параметров пользователей. Если этот параметр включен, входящий звонок по STN будет отображаться как исходящая от анонимного.
  
Чтобы заблокировать ИД входящие вызовы, см. настройка ИД звоня для [пользователя.](./set-the-caller-id-for-a-user.md)
