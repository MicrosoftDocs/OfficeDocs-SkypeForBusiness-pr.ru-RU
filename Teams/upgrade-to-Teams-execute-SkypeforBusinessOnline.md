# <a name="upgrade-from-skype-for-business-online-to-teams"></a>Переход со Skype для бизнеса Online на Teams

![Схема перехода на более поздние версии с акцентом на развертывание и реализацию.](media/upgrade-banner-deployment.png "Этапы пути обновления с акцентом на этап развертывания и реализации")

Эта статья является частью этапа развертывания и реализации процесса обновления. Прежде чем продолжить, убедитесь, что вы выполнили следующие действия:

- [Прикрепление заинтересованных лиц проекта](upgrade-enlist-stakeholders.md)
- [Определение области проекта](./upgrade-define-project-scope.md)
- [Понимание сосуществования и взаимодействия Skype для бизнеса и Teams](./teams-and-skypeforbusiness-coexistence-and-interoperability.md)
- [Выбор пути обновления](upgrade-and-coexistence-of-skypeforbusiness-and-teams.md)
- [Подготовка среды](./upgrade-prepare-environment.md)
- [Подготовка организации](./upgrade-prepare-organization.md)
- [Проводит пилотный проект](./pilot-essentials.md)

Следуйте указаниям в этой статье, если вы развернули Skype для бизнеса Online и хотите обновить пользователей с Skype для бизнеса до Teams. Вы можете выполнить выборочное или все обновление пользователей в зависимости от выбранного в вашей организации пути обновления, назначив пользователям соответствующий режим сосуществования и обновления.

> [!IMPORTANT]
> Skype для бизнеса Online прекращена 31 июля 2021 г. Чтобы максимально эффективно реализовать преимущества и убедиться, что у вашей организации есть соответствующее время для реализации обновления, мы рекомендуем начать работу Microsoft Teams сегодня. Помните, что успешное обновление соответствует технической и пользовательской готовности, поэтому не забудьте использовать рекомендации по переходу к Microsoft Teams.

## <a name="assign-the-coexistence-and-upgrade-mode"></a>Назначение режима сосуществования и обновления

Вы можете обновить пользователей до режима TeamsOnly, назначив экземпляр UpgradeToTeams TeamsUpgradePolicy, который можно выполнить с помощью центра администрирования Microsoft Teams или удаленного сеанса Skype для бизнеса Windows PowerShell. Это можно сделать как для каждого пользователя, так и для всего клиента, если вы хотите обновить весь клиент за один шаг. 

Дополнительные сведения см. в разделе ["](./setting-your-coexistence-and-upgrade-settings.md) Настройка параметров сосуществования и обновления" и ["TeamsUpgradePolicy:](upgrade-to-teams-on-prem-tools.md) управление миграцией и сосуществованием".

## <a name="upgrade-all-users-to-teams-at-one-time"></a>Обновление всех пользователей до Teams одновременно

Выполните следующие действия, чтобы обновить всех пользователей до Teams одновременно.

### <a name="step-1-notify-the-users-of-the-change-optional"></a>Шаг 1. Уведомление пользователей об изменении (необязательно)

1. В Microsoft Teams администрирования **выберите Teams** >  **Teams параметры обновления**.
2. В **режиме сосуществования** измените уведомление Skype для бизнеса **о** том, что обновление до Teams доступно для перехода на **"Включаем"**.

### <a name="step-2-set-the-coexistence-mode-to-teamsonly-for-the-organization"></a>Шаг 2. Установка режима сосуществования в TeamsOnly для организации

1. В Microsoft Teams администрирования **выберите Teams** >  **Teams параметры обновления**.
2. Выберите **Teams режим "** Только" в раскрывающемся списке режима сосуществования.

## <a name="upgrade-users-in-stages"></a>Поэтапное обновление пользователей

Выполните следующие действия, если вы хотите постепенно обновить пользователей до TeamsOnly.

### <a name="step-1-identify-groups-of-users-for-upgrade"></a>Шаг 1. Определение групп пользователей для обновления

Часто организации могут обновлять свои организации в волне успешных попыток пользователей.  Сначала необходимо определить этих пользователей, чтобы их можно было легко найти в Microsoft Teams администрирования. Кроме того, для более эффективного выполнения этой задачи можно использовать PowerShell. После того как вы определили набор пользователей для данной волны обновления, выполните оставшиеся действия.

### <a name="step-2-set-notification-for-the-users-in-the-current-upgrade-wave-optional"></a>Шаг 2. Настройка уведомлений для пользователей на текущей волне обновления (необязательно)

Если вы используете Microsoft Teams администрирования, можно настроить TeamsUpgradePolicy одновременно для 20 пользователей:
1. В центре Microsoft Teams выберите "Пользователи **",** найдите и установите флажок до 20 пользователей, которые должны быть обновлены. 
2. Выберите **"Изменить параметры** " в левом верхнем углу представления списка. 
3. На панели **"Изменение параметров**" справа в **разделе Teams обновления** измените уведомление Skype для бизнеса пользователя на "**В сети"**. Примечание. Если для режима сосуществования задано значение "Использовать параметры на уровне организации", этот параметр не отображается, поэтому сначала необходимо явно задать для этих пользователей режим сосуществования независимо от значения по умолчанию для организации.

Кроме того, вы можете упростить включение уведомлений для групп пользователей одновременно с помощью PowerShell. 

### <a name="step-3-set-the-coexistence-mode-for-users-to-teams-only"></a>Шаг 3. Настройка режима сосуществования для пользователей Teams только

Когда вы будете готовы обновить пользователей в текущей волне, чтобы использовать Teams в качестве их единственным приложением, задайте режим сосуществования для пользователей Teams только.

Если вы используете Microsoft Teams администрирования, можно настроить TeamsUpgradePolicy одновременно для 20 пользователей:
1. В Microsoft Teams центре администрирования **выберите "Пользователи**", а затем установите флажок до 20 пользователей.
2. Выберите **"Изменить параметры** " в левом верхнем углу представления списка.
3. В области **"** Изменение параметров" справа в разделе **Teams** обновления задайте режим сосуществования Teams только в раскрывающемся списке.

Кроме того, вы можете упростить обновление групп пользователей одновременно с помощью PowerShell. 

### <a name="step-4-repeat-steps-1-3-for-successive-waves-of-users"></a>Шаг 4. Повторите шаги 1–3 для последовательных волн пользователей

После проверки обновления до Teams режим "Только" и готовности к развертыванию повторите предыдущие шаги, чтобы применить TeamsOnly к большему число пользователей.  


## <a name="phone-system-and-pstn-connectivity-options"></a>телефонная система подключения и ТСОП

телефонная система с Teams поддерживается после того, как пользователь находится в режиме TeamsOnly. (Если пользователь находится в режиме "Острова", телефонная система поддерживается только Skype для бизнеса.)  

### <a name="pstn-connectivity-options"></a>Параметры подключения ТСОП

При рассмотрении вариантов подключения к общедоступной телефонной сети (ТСОП) существует два возможных сценария перехода с Skype для бизнеса Online на режим TeamsOnly:

- Пользователь в Skype для бизнеса Online с планом звонков Майкрософт. После обновления у этого пользователя по-прежнему будет план звонков Майкрософт. Это самый простой сценарий, требующий всего нескольких шагов. Дополнительные сведения см. в [Skype для бизнеса Online с планами звонков Майкрософт](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-online-with-microsoft-calling-plans).

- Пользователь в Skype для бизнеса Online с локальными функциями голосовой связи через Skype для бизнеса или Cloud Connector Edition. Обновление пользователя до Teams должно быть согласовано с миграцией пользователя на прямую маршрутизацию, чтобы убедиться, что пользователь TeamsOnly имеет функции ТСОП.  Дополнительные сведения см[. в Skype для бизнеса Online с локальным голосом](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-online-with-on-premises-voice).
