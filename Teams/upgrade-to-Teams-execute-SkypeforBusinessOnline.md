# <a name="upgrade-from-skype-for-business-online-to-teams"></a>Переход со Skype для бизнеса Online на Teams

Следуйте указаниям в этой статье, если полностью развернут Скайп для бизнеса в Интернет и требуется обновление пользователей с Скайп для бизнеса в группы. Можно обновить пользователей выборочно или файловый, основанным на обновление journey, ваша организация решила, назначив соответствующие сосуществования и обновления режима для пользователей.

> [!IMPORTANT]
> [!INCLUDE [upgrade-disclaimer](includes/upgrade-disclaimer.md)]

## <a name="assign-the-coexistence-and-upgrade-mode"></a>Назначение режима совместимости и обновления

Пользователей можно обновить до режима TeamsOnly с назначением UpgradeToTeams экземпляр TeamsUpgradePolicy, в которой может выполняться с помощью центра администрирования группами Майкрософт или Скайп для бизнеса удаленного сеанса Windows Powershell. Это возможно на уровне пользователя или на уровне клиента Если вы хотите ugprade всей клиента в одном шаге. 

Для получения дополнительных сведений см [Задание вашей сосуществования и параметры обновления](https://aka.ms/SkypeToTeams-SetCoexistence) и [TeamsUpgradePolicy: управление миграция и сосуществование](migration-interop-guidance-for-teams-with-skype.md#teamsupgradepolicy-managing-migration-and-co-existence).

## <a name="upgrade-all-users-to-teams-at-one-time"></a>Обновление всех пользователей к группам за один раз

Выполните следующие действия для обновления всех пользователей к группам за один раз.

### <a name="step-1-notify-the-users-of-the-change-optional"></a>Шаг 1: Уведомить пользователей об изменениях (необязательно)

1. В центре администрирования группами Майкрософт, выберите пункт **Параметры масштабе организации** > **Обновление группы**.
2. В разделе **режим совместимости**измените переключатель **Уведомить Скайп для бизнес-пользователей, что обновление группам доступен** **на**.

### <a name="step-2-set-the-coexistence-mode-to-teamsonly-for-the-organization"></a>Шаг 2: Задайте режим сосуществования TeamsOnly для организации

1. В центре администрирования группами Майкрософт выберите **Параметры на уровне организации**.
2. Выберите **Команды только** в режиме **сосуществования режим** раскрывающегося списка.

## <a name="upgrade-users-in-stages"></a>Обновление пользователей на стадии

Для постепенного обновления пользователей до TeamsOnly, выполните следующие действия.

### <a name="step-1-identify-groups-of-users-for-upgrade"></a>Этап 1: Определение групп пользователей к обновлению

Часто организациям может потребоваться обновить своей организации в этапами успеха пользователей.  Необходимо сначала определить эти пользователи, поэтому можно легко найти их в центре администрирования группами Майкрософт. Кроме того можно сделать это более эффективно с помощью PowerShell. После определения группы пользователей для данного обновления звукового файла выполните остальные действия.

### <a name="step-2-set-notification-for-the-users-in-the-current-ugprade-wave-optional"></a>Шаг 2: Настройка уведомлений для пользователей в текущем волне ugprade (необязательно)

Если с помощью центра администрирования группами Майкрософт, можно настроить TeamsUpgradePolicy для пользователя до 20 одновременно:
1. В центре администрирования группами Майкрософт флажок **пользователей**и поиск и выбор нескольких для до 20 пользователей, которые должны быть обновлены. 
2. Выберите **Изменить параметры** в верхнем левом углу списка автофильтра. 
3. В области **Изменение параметров** в правой части окна, в разделе **группы обновления**измените переключатель **Уведомлять Скайп для корпоративных пользователей** **на**. Примечание: Если значение режиме сосуществования «масштабе организации использовать параметры», не будут отображаться этот переключатель, вам потребуется сначала явно режим сосуществования для этих пользователей независимо от существующих значение по умолчанию — для ориентацию.

Кроме того может быть проще Включение уведомлений для групп пользователей, с помощью PowerShell. 

### <a name="step-3-set-the-coexistence-mode-for-users-to-teams-only"></a>Шаг 3: Установите режим совместимости для пользователей к группам только

Режим совместной работы для пользователей к группам только, когда все будет готово к обновлению пользователей в текущем звукового файла для использования в качестве их только приложения группы.

Если с помощью центра администрирования группами Майкрософт, можно настроить TeamsUpgradePolicy для пользователя до 20 одновременно:
1. В центре администрирования группами Майкрософт выберите **пользователей**и затем установите флажок для до 20 пользователями.
2. Выберите **Изменить параметры** в верхнем левом углу списка автофильтра.
3. В области **Изменение параметров** в правой части окна, в разделе **Обновление группы** режим совместной работы **Групп** только в раскрывающемся списке.

Кроме того может быть проще обновление групп пользователей, с помощью PowerShell. 

### <a name="step-4-repeat-steps-1-3-for-successive-waves-of-users"></a>Шаг 4: Повторите шаги 1-3 для последующими этапами пользователей

Как проверить вашего обновления до групп только в режиме и все готово к разверните узел, повторите предыдущие действия, чтобы применить TeamsOnly для других пользователей.  


## <a name="phone-system-and-teams-upgrade"></a>Обновление телефонной системой и рабочих групп

Если ваше Скайп для бизнеса в Интернет развертывания включает в себя телефонной системы с помощью вызова планы и Microsoft — это ваш поставщик телефонной сети (общего пользования PSTN), обновление пользователей группам автоматический переход входящих PSTN, вызов к группам.

Если ваше Скайп для бизнеса в Интернет развертывания включает системы Phone Edition соединителя облако с, см.в [Дополнительные вопросы для прямой маршрутизации телефонной системы](2-envision-make-my-service-decisions-direct-routing.md).
