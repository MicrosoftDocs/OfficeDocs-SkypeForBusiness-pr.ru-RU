# <a name="understand-microsoft-teams-and-skype-for-business-coexistence-and-interoperability"></a>Общие сведения о сосуществовании и взаимодействии Microsoft Teams и Skype для бизнеса

Если в вашей организации используется Skype для бизнеса сегодня и вы планируете приступить к работе с группами в Skype для бизнеса — или вы хотите начать переход на Teams, важно понять, как совместно сосуществование двух приложений, когда и как они взаимодействуют и как Управляйте миграцией пользователей на весь рабочий процесс с помощью Skype для бизнеса и Teams.

## <a name="coexistence-of-teams-and-skype-for-business"></a>Сосуществование Teams и Skype для бизнеса

В дополнение к возможностям совместной работы Teams предоставляет возможности для общения, звонков и собраний. В зависимости от того, как вы выбираете развертывание команд, эти возможности могут перекрываться с возможностями, предоставляемыми Skype для бизнеса для определенного пользователя. Режим по умолчанию позволяет запускать команды вместе с Skype для бизнеса с перекрытием возможностей; Однако пользователь может назначить один из нескольких режимов сосуществования, разработанных для обеспечения того, что эти возможности не перекрываются для этого пользователя (в этом случае будет доступно взаимодействие между Teams и Skype для бизнеса).

Мы рекомендуем ознакомиться с режимами сосуществования, описанными ниже, чтобы определить, какой путь является подходящую для вашей организации.

> [!Important]
> Познакомьтесь с новыми технологиями или вносите изменения в уже имеющуюся привычную среду Skype для бизнеса, обеспечивая очень много новых возможностей для бизнеса, и это может привести к нарушениям работы пользователей. Ознакомьтесь со временем о том, как оценить готовность пользователей и внедрить план общения и обучения, прежде чем приступать к реализации изменений, описанных в этой статье. Кроме того, мы настоятельно рекомендуем вам провести пробную версию плана с выбранной группой пользователей, прежде чем внедрять ее в своей организации.

### <a name="islands-mode"></a>Режим "острова"

По умолчанию пользователи могут запускать команды вместе со Skype для бизнеса в виде двух отдельных решений, которые обеспечивают аналогичные и перекрывающиеся возможности, такие как присутствие, чат, звонки и собрания. Пользователи Teams также могут использовать новые возможности совместной работы, такие как команды и каналы, доступ к файлам в Office 365 и приложения.

В режиме совместного существования под названием " **острова**" все клиентские приложения работают как отдельные острова. Skype для бизнеса взаимоотношения с Skype для бизнеса, а команды – для общения с группами. Пользователи работают на обоих клиентах в любое время и могут взаимодействовать с ними в клиенте, с которого была инициирована связь. Таким образом, взаимодействие в режиме **островов** не требуется.

Чтобы избежать путаницы или регрессии в Skype для бизнеса, внешние (Федеративные) коммуникации, голосовые службы PSTN и голосовые приложения, интеграция с Office и несколько других интеграций продолжают обрабатывать Skype для бизнеса.

> [!Important]
> В режиме " **острова** " все сообщения от федеративных пользователей (пользователей за пределами вашей организации) доставляются в Skype для бизнеса. После переключения в режим " **только** для Teams" все сообщения за пределами вашей организации будут доставляться в Teams.

> [!Tip]
> Пользователи Skype для бизнеса Online рекомендованный путь — это начало работы с режимом **острова** по умолчанию, а также быстрое переключение в режим "только для **Teams** " в Организации. Для локальных и гибридных пользователей может потребоваться развертывание **Skype для бизнеса с использованием режима совместной работы** в качестве начальной точки, а не задач в **Skype для бизнеса с использованием режима совместной работы и собраний** в Teams При необходимости и в режиме " **только** для Teams", если организация готова принимать команды.

### <a name="skype-for-business-only"></a>Только Skype для бизнеса

В режиме совместного существования пользователи остаются в Skype для бизнеса, а не в Teams — для возможностей для общения, собраний и звонков, а также для работы с группами и каналами. Этот режим доступен сегодня; Однако в текущей реализации команды и каналы не отключаются автоматически для пользователя. Это можно сделать с помощью политики "разрешения приложений", чтобы скрыть команды и каналы.

### <a name="teams-only"></a>Только для Teams

Пользователи Teams могут получить доступ ко всем возможностям в Teams **только** в том случае, если у вас есть *обновленный* пользователь. Клиенты Skype для бизнеса могут присоединиться к собраниям в Skype для бизнеса, упорядоченных необновленными пользователями или внешними сторонами. Обновленный пользователь может взаимодействовать с другими пользователями в Организации, которые по-прежнему используют Skype для бизнеса, с помощью возможностей взаимодействия между Teams и Skype для бизнеса (при условии, что эти пользователи Skype для бизнеса не находятся в режиме "острова") ; Тем не менее, обновленный пользователь не может инициировать чат, Звонок или собрание Skype для бизнеса.

После того как ваша организация готова для некоторых или всех пользователей к использованию Teams в качестве единственного средства связи и совместной работы, вы можете обновить эти пользователи в режиме " **только** для Teams". Если вы обновляете режим от "острова", мы рекомендуем вам предварительно загрузить настройку команд во всей Организации, прежде чем начинать процесс обновления. Это позволяет избежать неработающих сценариев связи, так как в режиме "острова" не обеспечивается взаимодействие.

Дополнительные сведения о переходе в режим "только в Teams" можно найти в разделе [вопросы, касающиеся режима только Teams](teams-only-mode-considerations.md).

### <a name="skype-for-business-with-teams-collaboration"></a>Skype для бизнеса с совместная работа в Teams

Этот режим используется для введения команд в вашу среду, пока вы продолжаете использовать существующие инвестиции в Skype для бизнеса. В этом режиме вы оставляете Skype для бизнеса неизменным для функций для общения, звонков и собраний, а можно добавлять возможности совместной работы в Teams — команды и каналы, получать доступ к файлам в Office 365 и приложениям. Возможности коммуникации в Teams (частный чат, звонки и планирование собраний) по умолчанию отключены в этом режиме. Организации с начальной точкой Skype для бизнеса Server в локальной или гибридной среде должны рассматривать этот режим как альтернативный режим, если требуется предоставить пользователям возможность взаимодействия и предсказуемости для их взаимодействия.

### <a name="skype-for-business-with-teams-collaboration-and-meetings"></a>Skype для бизнеса с совместной работой и собраниями в Teams

Этот режим позволяет ускорить доступность возможностей собрания Teams в Организации, а также возможности совместной работы, позволяющие пользователям использовать преимущества более высокого качества собраний в Teams – отличное качество. инновационные возможности, такие как транскрипция и перевод или фоновое размытие, а также удобство работы пользователей на всех платформах, включая мобильные устройства и браузеры.

Кроме того, с помощью Teams для Teams и каналов, основанных на каналах, в этом режиме пользователи будут использовать Teams для планирования и проведения собраний. Закрытый чат и звонки остаются в Skype для бизнеса. Teams и Skype для бизнеса – это преимущество в различных возможностях, таких как выверка присутствия, автоматическое удержание/неудержание и поддержка HID-устройств для обоих приложений. 

Этот режим совместного существования особенно полезен для пользователей в локальных развертываниях Skype для бизнеса с корпоративной голосовой связью, которые, скорее всего, будут обновляться до Teams, и вам будет удобнее работать с другими собраниями Teams, как только это станет возможным.

> [!Note]
> При развертывании в конкретных режимах совместного существования участники Teams и Skype для бизнеса могут [взаимодействовать](#interoperability-of-teams-and-skype-for-business)друг с другом, позволяя пользователям общаться с другими пользователями, а также обеспечивать связь между ними во время путешествия по переходу на Teams. Режимы сосуществования управляют взаимодействием. Режим сосуществования получателя определяет, будет ли доступно взаимодействие. Например, если получатель находится в режиме, в котором чат доступен только в одном клиенте (скажем, Teams), взаимодействие с чат обычно будет доступно в том случае, если инициатор использует другой клиент (в данном случае — Skype для бизнеса) для начала чата. С другой стороны, если получатель находится в режиме, в котором чат доступен на обоих клиентах (в режиме "острова"), взаимодействие не будет доступно для этого чата. Сообщение будет получено получателем в том же клиенте, в котором инициатор запустил чат. Таким образом, для правильного общения в режиме острова требуется насыщенность внедрения команды. Это значит, что все пользователи активно используют и отслеживают обоих клиентов.

Дополнительные сведения о режимах сосуществования, требованиях и управлении можно найти в [статье Руководство по миграции и взаимодействию для организаций, использующих Teams совместно со Skype для бизнеса](https://aka.ms/SkypeToTeams-Interop) и [настройку параметров сосуществования и обновления](https://aka.ms/SkypeToTeams-SetCoexistence).

## <a name="interoperability-of-teams-and-skype-for-business"></a>Взаимодействие Teams и Skype для бизнеса

Взаимодействие — это возможность для пользователей Teams и Skype для бизнеса в той же организации общаться в разных группах и Skype для бизнеса.

### <a name="native-interop-and-interop-escalation"></a>Распространение собственного взаимодействия и взаимодействия

Существует два типа взаимодействия: распространение собственного и взаимодействующих функций.

- В клиенте, который используется в данный момент, происходит _взаимодействие с собственным_ интерфейсом. Один пользователь будет находиться в клиенте Skype для бизнеса, а также в Teams. Встроенные функции взаимодействия не перемещаются другим клиентам для общения, поэтому пользователи смогут проводить свои беседы в клиенте, которые в настоящее время используются. Собственный интерфейс взаимодействия — это чат и звонок.
- Использование _эскалации взаимодействия_ означает, что в ходе упрощения работы пользователей (например, совместного использования рабочего стола) клиент упрощает создание собраний, к которым пользователи смогут присоединиться, чтобы продолжить работу на этом собрании. Собрание создается на платформе инициатора действия. Пользователи, которых нет на этой платформе, получают ссылку присоединиться к собранию. По мере щелчка этой ссылки они присоединяются к собранию в совместимом клиенте (браузер, веб-приложение или на весь клиент) в зависимости от конфигурации. Для эскалации взаимодействия из Skype для бизнеса требуется последний клиент. В ближайшее время взаимодействие между участниками из групп становится более подходящим.

### <a name="native-interop-experiences"></a>Взаимодействие с машинным интерфейсом

В зависимости от режима сосуществования, назначенного пользователям (как описано выше), доступны следующие встроенные возможности взаимодействия:

- Пользователи Skype для бизнеса могут использовать один и тот же чат с пользователями Teams и наоборот. Чат взаимодействия должен пройти через шлюз взаимодействия, который входит в состав облачных служб Teams (и, следовательно, только в Интернете). Беседы взаимодействия — это обычный текст: форматированный текст и смайлики не поддерживаются. Пользователи в Teams и Skype для бизнеса получат уведомление о том, что беседа является обсуждением взаимодействия.

- Пользователи Skype для бизнеса могут выполнять голосовые и видеозвонки между пользователями Teams и наоборот.

    ![Снимок экрана с интерфейсом вызова взаимодействия из Teams] (media/Interop_calling_experience_from_Teams.png "Взаимодействие между абонентами из Teams")

> [!Important]
> Для взаимодействия с локальным развертыванием Skype для бизнеса необходимо, чтобы локальная среда была в гибридном режиме с Office 365 Skype для бизнеса. Подробности можно найти в [статье Руководство по миграции и взаимодействию](https://aka.ms/SkypeToTeams-Interop).

Это взаимодействие между пользователями, у которых есть один из следующих вариантов сосуществования: **Skype для бизнеса с совместная**работа в Teams, **Skype для бизнеса с совместной работой и собраниями**в Teams, **Skype для **Только для бизнеса или **только для рабочих групп**. Взаимодействие с пользователями в режиме "острова" не поддерживается.

### <a name="native-interop-experience-limitations"></a>Ограничения взаимодействия с собственными интерфейсами

Некоторые функции недоступны для чата взаимодействия и взаимодействия между Teams и Skype для бизнеса:

- Markdown, форматированный текст и полный набор смайликов не поддерживаются ни в Teams, ни в Skype для бизнеса. Другие собственные функции в окне создания сообщения в чате Teams не поддерживаются.
- Демонстрация экрана (общий доступ к рабочему столу или приложению) между группами и Skype для бизнеса не поддерживается.
- Групповые разговоры (многосторонние беседы) в Teams могут включать только участников, использующих Teams.
- Многосторонние текстовые беседы (чат групп) в Skype для бизнеса могут включать только те участники, которые используют Skype для бизнеса.
- Эскалация одноранговых голосовых и видеозвонков с участием многочисленных абонентов и пользователей Skype для бизнеса не поддерживается.
- Передача файлов для двусторонних чатов или вложенных файлов в групповых беседах из групп в Skype для бизнеса и наоборот — не поддерживаются.
- Вы не можете взаимодействовать с сохраняемым чат в Skype для бизнеса.

Для всех этих ограничений (за исключением сохраняемого чата) один из возможных способов решения этой проблемы состоит в том, чтобы один пользователь начал собрание и пригласить другого пользователя присоединиться к нему. Это временное решение является основой для распространения взаимодействия.
