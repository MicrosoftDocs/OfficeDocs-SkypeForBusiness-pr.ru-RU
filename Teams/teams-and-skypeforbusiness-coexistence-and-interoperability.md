## <a name="coexistence-of-teams-and-skype-for-business"></a>Сосуществование Teams и Skype для бизнеса

В дополнение к возможностям совместной работы Teams предоставляет возможности для общения, звонков и собраний. В зависимости от того, как вы выбираете развертывание команд, эти возможности будут перекрываться с возможностями, предоставляемыми Skype для бизнеса для определенного пользователя. Режим по умолчанию позволяет запускать команды вместе с Skype для бизнеса; Однако пользователю может быть назначен один из нескольких режимов сосуществования, разработанных для обеспечения того, что эти возможности не перекрываются для этого пользователя.

Мы рекомендуем ознакомиться с режимами сосуществования, описанными ниже, чтобы определить, какой путь является подходящую для вашей организации.

> [!Important]
> Познакомьтесь с новыми технологиями или вносите изменения в уже имеющуюся привычную среду Skype для бизнеса, обеспечивая очень много новых возможностей для бизнеса, и это может привести к нарушениям работы пользователей. Ознакомьтесь со временем о том, как оценить готовность пользователей и внедрить план общения и обучения, прежде чем приступать к реализации изменений, описанных в этой статье. Кроме того, мы настоятельно рекомендуем вам провести пробную версию плана с выбранной группой пользователей, прежде чем внедрять ее в своей организации.

### <a name="islands-mode"></a>Режим "острова"

По умолчанию пользователи могут запускать команды вместе со Skype для бизнеса в виде двух отдельных решений, которые обеспечивают аналогичные и перекрывающиеся возможности, такие как чат, звонки и собрания. Пользователи Teams также могут использовать возможности совместной работы, такие как команды и каналы, доступ к файлам в Office 365 и приложения.

В режиме совместного существования под названием " **острова**" все клиентские приложения работают как отдельные острова. Skype для бизнеса взаимоотношения с Skype для бизнеса, а команды – для общения с группами. Пользователи выполняют оба клиента и могут общаться с ним в клиенте, с которого была инициирована связь. Таким образом, взаимодействие в режиме **островов** не требуется.

> [!Tip]
> Рекомендуемый путь для клиентов Skype для бизнеса Online — начало работы с режимом " **острова** по умолчанию", а также в Организации, а затем быстро переходите в режим " **только Teams** ". Для локальных и гибридных пользователей вам может потребоваться развертывание предстоящей среды **Skype для бизнеса с использованием режима совместной работы** в качестве начальной точки, а не задач в режиме " **только** для Teams", если организация готова к переходу Teams.

### <a name="skype-for-business-only"></a>Только Skype для бизнеса

В режиме совместного существования пользователи остаются в Skype для бизнеса, а не в Teams — для возможности общения, собраний и звонков, а также для работы с группами и каналами. Этот режим доступен сегодня; Однако в текущих реализациях Teams модальностей не отключены для пользователя автоматически. Это предстоящие возможности. В предварительной версии администраторы могут удалить лицензию Teams для всех пользователей, которым нужно оставаться в Skype для бизнеса в качестве их единственной коммуникационной программы.

### <a name="teams-only"></a>Только для Teams

С **** помощью клиента Skype для бизнеса можно присоединиться к существующим собраниям Skype для бизнеса или собраниям в Skype для бизнеса, которые были организованы необновленными пользователями или внешними лицами. Обновленный пользователь может взаимодействовать с другими пользователями в Организации, которые по-прежнему используют Skype для бизнеса, с помощью возможностей взаимодействия между Teams и Skype для бизнеса; Тем не менее, обновленный пользователь не может инициировать чат, Звонок или собрание Skype для бизнеса.

После того как ваша организация готова для некоторых или всех пользователей к использованию Teams в качестве единственного средства связи и совместной работы, вы можете обновить эти пользователи в режиме " **только** для Teams".

Дополнительные сведения о переходе в режим "только в Teams" можно найти в разделе [вопросы, касающиеся режима только Teams](teams-only-mode-considerations.md).

### <a name="skype-for-business-with-teams-collaboration-this-mode-is-upcoming"></a>Skype для бизнеса с совместной работой в Teams (этот режим является предстоящим)

Этот режим используется для введения команд в вашу среду, пока вы продолжаете использовать существующие инвестиции в Skype для бизнеса. В этом режиме вы оставляете Skype для бизнеса неизменным с помощью функций чата, звонков и собраний, а можно добавлять возможности совместной работы в Teams — команды и каналы, доступ к файлам в Office 365 и приложения. Организации с начальной точкой Skype для бизнеса Server в локальной или гибридной версии должны использовать этот режим, а не режим "острова".

### <a name="skype-for-business-with-teams-collaboration-and-meetings-this-mode-is-upcoming"></a>Skype для бизнеса с совместной работой и собраниями в Teams (этот режим является предстоящим)

Этот режим позволяет ускорить доступность возможностей собрания Teams в Организации, а также возможности совместной работы, позволяющие пользователям использовать преимущества высокого качества, новые возможности, такие как транскрипция и перевод и поддержка собраний в браузерах.

Кроме того, с помощью Teams для Teams и каналов, основанных на каналах, в этом режиме пользователи могут приступить к работе с Teams для планирования и проведения собраний. Закрытые разговоры, голосовые и видеозвонки остаются в Skype для бизнеса. Этот режим совместного существования особенно полезен для пользователей в локальных развертываниях Skype для бизнеса, которые имеют корпоративный голосовую почту, что может занять некоторое время для обновления до Teams, но вы хотите воспользоваться преимуществами самых крупных собраний Teams.

> [!Note]
> При развертывании в конкретных режимах совместного существования участники Teams и Skype для бизнеса могут [взаимодействовать](#interoperability-of-teams-and-skype-for-business)друг с другом, позволяя пользователям общаться с другими пользователями, а также обеспечивать связь между ними во время путешествия по переходу на Teams. Режимы сосуществования управляют взаимодействием. Режим сосуществования получателя определяет, будет ли доступно взаимодействие. Например, если получатель находится в режиме, в котором чат доступен только в одном клиенте (скажем, Teams), взаимодействие с чат обычно будет доступно в том случае, если инициатор использует другой клиент (в данном случае — Skype для бизнеса) для начала чата. С другой стороны, если получатель находится в режиме, в котором чат доступен на обоих клиентах, взаимодействие не будет доступно для этого чата, и сообщение будет получено получателем в том же клиенте, в котором инициатор запустил чат.

Дополнительные сведения о режимах сосуществования, требованиях и управлении можно найти в [статье Руководство по миграции и взаимодействию для организаций, использующих Teams совместно со Skype для бизнеса](https://aka.ms/SkypeToTeams-Interop) и [настройку параметров сосуществования и обновления](https://aka.ms/SkypeToTeams-SetCoexistence).

## <a name="interoperability-of-teams-and-skype-for-business"></a>Взаимодействие Teams и Skype для бизнеса

Взаимодействие — это возможность для пользователей Teams и Skype для бизнеса в той же организации общаться в разных группах и Skype для бизнеса.

### <a name="native-interop-and-interop-escalation"></a>Распространение собственного взаимодействия и взаимодействия

Существует два типа взаимодействия: распространение собственного и взаимодействующих функций.

- В клиенте, который используется в данный момент, происходит _взаимодействие с собственным_ интерфейсом. Один пользователь будет находиться в клиенте Skype для бизнеса, а также в Teams. Встроенные функции взаимодействия не перемещаются другим клиентам для общения, поэтому пользователи смогут проводить свои беседы в клиенте, которые в настоящее время используются. Собственный интерфейс взаимодействия — это чат и звонок.
- _Расширение взаимодействия_ — это то, что в ходе упрощения работы пользователей (например, совместного использования рабочего стола) служба может упростить создание собрания и продолжить работу с ним на собрании. Собрание создается на платформе инициатора действия. Пользователи, которых нет на этой платформе, получают координаты присоединения к собранию и присоединитесь к собранию (после переключения клиентов).

### <a name="native-interop-experiences"></a>Взаимодействие с машинным интерфейсом

В зависимости от режима сосуществования, назначенного пользователям (как описано выше), доступны следующие встроенные возможности взаимодействия:

- Пользователи Skype для бизнеса могут использовать один и тот же чат с пользователями Teams и наоборот. Чат взаимодействия должен пройти через шлюз взаимодействия, который входит в состав облачных служб Teams (и, следовательно, только в Интернете). Беседы взаимодействия — это обычный текст: форматированный текст и смайлики не поддерживаются. Пользователи в Teams получат уведомление о том, что разговор является обсуждением взаимодействия; похожее уведомление для пользователей Skype для бизнеса скоро будет предложено.

Взаимодействие ![с чат в чате из Teams] Взаимодействие (media/Interop_chat_experience_from_Teams.png "с чат в чате из Teams")

- Пользователи Skype для бизнеса могут выполнять голосовые и видеозвонки между пользователями Teams и наоборот.

> [!Important]
> Для взаимодействия с локальным развертыванием Skype для бизнеса необходимо, чтобы локальная среда была в гибридном режиме с Office 365 Skype для бизнеса. Подробности можно найти в [статье Руководство по миграции и взаимодействию](https://aka.ms/SkypeToTeams-Interop).

Это взаимодействие между пользователями, у которых есть один из указанных ниже режимов взаимосуществования: **Skype для бизнеса с совместной**работой в Teams, Skype **для бизнеса с совместной работой и собраниями**в Teams, **Skype для **Только для бизнеса или **только для рабочих групп**. Взаимодействие с пользователями в режиме "острова" не поддерживается.

### <a name="native-interop-experience-limitations"></a>Ограничения взаимодействия с собственными интерфейсами

Некоторые функции недоступны для чата взаимодействия и взаимодействия между Teams и Skype для бизнеса:

- Markdown, форматированный текст и полный набор смайликов не поддерживаются ни в Teams, ни в Skype для бизнеса. Другие собственные функции в окне создания сообщения в чате Teams не поддерживаются.
- Демонстрация экрана (общий доступ к рабочему столу или приложению) между группами и Skype для бизнеса не поддерживается.
- Групповые разговоры (многосторонние беседы) в Teams могут включать только участников, использующих Teams.
- Многосторонние текстовые беседы (чат групп) в Skype для бизнеса могут включать только те участники, которые используют Skype для бизнеса.
- Эскалация одноранговых голосовых и видеозвонков с участием многочисленных абонентов и пользователей Skype для бизнеса не поддерживается.
- Передача файлов для двусторонних чатов или вложенных файлов в групповых беседах из групп в Skype для бизнеса и наоборот — не поддерживаются.
- Вы не можете взаимодействовать с сохраняемым чат в Skype для бизнеса.

Для всех этих ограничений (за исключением сохраняемого чата) один из возможных способов решения этой проблемы состоит в том, чтобы один пользователь начал собрание и пригласить другого пользователя присоединиться к нему. Это временное решение является основой для распространения взаимодействия.

> [!Important]
> Что начинается в виде простого чата (IM), может быстро переключаться между вызовом или рекламным собранием. Мы понимаем, что эти сценарии важны для удобства использования и взаимодействия с пользователем, и мы постоянно собираем взаимодействие между пользователями Skype для бизнеса и Teams.
