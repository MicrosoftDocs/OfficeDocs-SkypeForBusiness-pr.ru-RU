# <a name="understand-microsoft-teams-and-skype-for-business-coexistence-and-interoperability"></a>Понятие о группами Майкрософт и Скайп для бизнеса сосуществования и взаимодействия

Если организация использует Скайп для бизнеса сегодня и требуется запустить с помощью команды наряду с Скайп для бизнеса, или планируется начать обновление до групп — важно понимать, как сосуществования двух приложений, когда и как их взаимодействие и как для Управление миграции пользователей до их окончательным обновления в Скайп для бизнеса в группы.

## <a name="coexistence-of-teams-and-skype-for-business"></a>Совместная работа групп и Скайп для бизнеса

В дополнение к возможности совместной работы групп доставляет чата, звонков и возможности проведения собраний. В зависимости от способа развертывания групп эти возможности будут перекрывать возможности, представленные с Скайп для бизнеса для данного пользователя. Режим по умолчанию должна выполняться команды наряду с Скайп для бизнеса; Тем не менее пользователь может задать один из нескольких режимах сосуществования, которые были созданы, чтобы убедиться, что эти возможности не перекрывались для этого пользователя.

Мы рекомендуем рассмотреть режимы сосуществования, описывается ниже, чтобы определить, какой путь для своей организации.

> [!Important]
> Краткие сведения о новой технологии или внесение изменений в вашей существующей, знакомых Скайп для бизнес-среде доставки замечательных новых преимущества для бизнеса, может быть аварийные для пользователей. Потратьте время оценить готовность пользователей и реализовывать обмена данными и план обучения, перед реализацией какие-либо изменения, описанные в этой статье. Кроме того мы настоятельно рекомендуем вам пилотный проект плана с выбранной группы пользователей, прежде чем внедрять в организации.

### <a name="islands-mode"></a>Режим острова

По умолчанию пользователи могут запускать команды наряду с Скайп для бизнеса в двух отдельных решения, которые предоставляют аналогичное и перекрывающиеся возможности, такие как чата, звонков и собраний. Группы пользователей также можно воспользоваться преимуществами возможности совместной работы, такие как команды и каналы, доступ к файлам в Office 365 и приложений.

В этом режиме сосуществования называется **острова**каждого из клиентских приложений действует как отдельные остров. Обменивается данными только Скайп для бизнеса кругом Скайп для бизнеса и рабочих групп с группами. Пользователи выполните обоих этих клиентах и обмениваться данными изначально в клиенте, из которого была инициирована передача данных. Таким образом нет необходимости для взаимодействия в режиме **острова** .

> [!Tip]
> Начинать с режимом **острова** по умолчанию, диск насыщенность внедрения в организации и быстро перемещение **Групп только** в режиме — Скайп для бизнеса в Интернет рекомендуемые пути клиентов. На локального и гибридного клиентов может использовать преимущества развертывание предстоящие режим **Скайп для бизнеса с совместной работы групп** в качестве отправной точки, а не о-ва и о ходе выполнения оттуда **Групп только** в режиме при организации готов принять Группы.

### <a name="skype-for-business-only"></a>Скайп для бизнеса только

В этом режиме сосуществования пользователей остаются в Скайп для бизнеса — не групп — для чата, собрания и вызывающий возможности и они не используйте команды для групп и каналы. Этот режим доступен в настоящее время; Тем не менее в текущей реализации модальностей команды не отключаются для пользователя автоматически. Это предстоящие. До этого времени "Администраторы" можно удалить лицензии групп пользователей, которые должны оставаться в Скайп для бизнеса, как их только приложение коммуникаций.

### <a name="teams-only"></a>Только группы

Чтобы присоединиться к существующей Скайп для деловых встреч или собраний на Скайп для бизнеса, организованы необновленной пользователями или внешними участниками **только группам** пользователей можно использовать только Скайп для клиента Business. Обновленный пользователь может продолжить взаимодействие с других пользователей в организации, по-прежнему используется Скайп для бизнеса с помощью возможности взаимодействия между рабочими группами и Скайп для бизнеса; Тем не менее обновленный пользователь не может инициировать Скайп для бизнеса чата, звонок или собрания.

Как только ваша организация готова для некоторых или всех пользователей для использования в качестве их единственным средством коммуникаций и совместной работы групп, этих пользователей можно выполнить обновление до **Групп только** в режиме.

Дополнительные вопросы о перемещении групп только в режиме содержатся в разделе [группы только режим](teams-only-mode-considerations.md).

![Скайп для клиента Business в специальных режиме после обновления учетной записью пользователя, только для групп пользователей] (media/teams-and-skypeforbusiness-coexistence-and-interop-image1.png "Скайп для клиента Business в специальных режиме после обновления учетной записью пользователя, только для групп пользователей")

### <a name="skype-for-business-with-teams-collaboration-this-mode-is-upcoming"></a>Скайп для бизнеса с совместной работы групп (этот режим предстоящие)

Используйте этот режим для знакомства группами в вашей среде во время использования существующих инвестиций в Скайп для бизнеса. В этом режиме оставьте Скайп для бизнеса, без изменений с помощью чата, звонков и возможности проведения собраний и добавлять возможности совместной работы групп, групп и каналы, доступ к файлам в Office 365 и приложений. Организациям начальной точки Скайп для Business server на локальном или гибридного с следует использовать этот режим вместо режима острова.

### <a name="skype-for-business-with-teams-collaboration-and-meetings-this-mode-is-upcoming"></a>Скайп для бизнеса с группами собраний и совместной работы (этот режим предстоящие)

Используйте этот режим сосуществования для ускорения доступность групп возможности в вашей организации, в дополнение к возможности совместной работы на собрания, что позволяет обеспечить, чтобы воспользоваться преимуществами высокого качества, новые возможности, такие как транскрибирования и перевод и поддержки для собраний в браузерах.

Наряду с помощью команды для групп и на основе каналы бесед в этом режиме, пользователям начать работу с группами для планирования и проводить их. Закрытый чаты и голосовую и видеосвязь, остаются на Скайп для бизнеса. В этом режиме сосуществования особенно полезен для пользователей в Скайп для бизнеса локальные развертывания корпоративной голосовой связи, которые, скорее всего, потратьте некоторое время обновления до групп, но нужно извлечь пользу из собрания руководителя группы.

> [!Note]
> При развертывании в режимах конкретных сосуществования, групп и Скайп для бизнеса можно [будет взаимодействовать](#interoperability-of-teams-and-skype-for-business), предоставление пользователям возможности общаться с и вызвать друг с другом и проверка того, что communications остаются уровень жидкости внутри организации во время вашего обновления пути к группам. Режимы сосуществования управляют взаимодействия. Режим сосуществования приемника определяет, будет ли взаимодействие доступен. К примеру Если получатель находится в режиме, в котором чата доступен только в один клиент (скажем, группы), взаимодействие чата обычно будут доступны в случае, если инициатор использует другие клиент (в данном случае Скайп для бизнеса) чтобы начать разговор. С другой стороны, если получатель находится в режиме, в котором чата доступна в обоих этих клиентах, взаимодействие будет доступен для разговора, и сообщение будет получено на стороне получателя в одного клиента, в котором инициатор работы чата.

Для получения дополнительных сведений о режимах сосуществования, необходимые условия и управления видеть [миграции и руководство по взаимодействию для организаций, с помощью команды Скайп для бизнеса](https://aka.ms/SkypeToTeams-Interop) , а [параметру вашей сосуществования и параметры обновления](https://aka.ms/SkypeToTeams-SetCoexistence).

## <a name="interoperability-of-teams-and-skype-for-business"></a>Совместимость рабочих групп и Скайп для бизнеса

Взаимодействие является возможность для групп и Скайп для бизнес-пользователей в той же организации для связи по командам и Скайп для бизнеса.

### <a name="native-interop-and-interop-escalation"></a>Собственная эскалации взаимодействия и взаимодействия

Существует два типа взаимодействия каждый раз: эскалации собственный и взаимодействия.

- В клиенте, который пользователь в настоящее время с помощью происходит с программой _встроенного взаимодействия_ . Один пользователь будет находиться в Скайп для клиента Business, другое в группах. Собственная качества взаимодействия вступят их к другим клиентом для связи, пользователи смогут провести их беседы в клиенте, которые в настоящее время используется. Собственная взаимодействия каждый раз, один к одному чата и вызова.
- Обеспечения взаимодействия _взаимодействия эскалации_ означает, что как часть как помочь пользователям выполнить дополнительные действия (например, общий доступ к рабочим столам), служба может облегчают создание собрания и продолжить работу в этого собрания. Собрания создается на платформе инициатор действие. Одного или нескольких пользователей, которые не на этой платформе получать собрания присоединиться к координаты и присоединиться к собранию (после переключения клиентов).

### <a name="native-interop-experiences"></a>Собственная взаимодействия каждый раз

В зависимости от режима совместной работы, назначить пользователям (как описано выше) доступны следующие интерфейсы собственный взаимодействия:

- Скайп для бизнес-пользователи one-on-one общаться с пользователями, группами и наоборот. Взаимодействия чата должно пройти через шлюз взаимодействия (en), который является частью группы облачных служб (и поэтому существует только Интернет-версия). Взаимодействия чаты являются обычного текста: форматированный текст и значки настроения не поддерживаются. Уведомления пользователей в группы, которые беседу — это взаимодействия беседы; скоро будет предоставляться аналогичные уведомления о Скайп для коммерческих пользователей.

![Взаимодействие чата взаимодействия из рабочих групп] (media/Interop_chat_experience_from_Teams.png "Взаимодействие чата взаимодействия из рабочих групп")

- Скайп для бизнес-пользователей можно сделать индивидуального голосовых и видеозвонков для группы пользователей и наоборот.

![Взаимодействие вызов качества из рабочих групп] (media/Interop_calling_experience_from_Teams.png "Взаимодействие вызов качества из рабочих групп")

> [!Important]
> Каждый раз взаимодействия с развертыванием локальной Скайп для бизнеса требуют локальной среды в режиме гибридного с Скайп Office 365 для бизнеса. Дополнительные сведения см [миграции и рекомендации взаимодействия](https://aka.ms/SkypeToTeams-Interop).

Все это взаимодействия доступны для и от пользователей, имеющих один из следующих режимов сосуществования назначенных: **Скайп для бизнеса с совместной работы групп**, **Скайп для бизнеса с собраний и совместной работы групп**, **Скайп для Только Business**, или **группами только**. Нет взаимодействие для пользователей в режиме острова.

### <a name="native-interop-experience-limitations"></a>Ограничения собственный качества взаимодействия (en)

Некоторые функции не доступны для взаимодействия чата и взаимодействия вызова взаимодействия между рабочими группами и Скайп для бизнеса:

- Наценки, форматированного текста, а также полный смайлик не поддерживаются из групп или Скайп для бизнеса. Другие собственные функции создания поля в беседах группы не поддерживаются.
- Не поддерживается совместного (рабочий стол или совместный доступ к приложениям) между рабочими группами и Скайп для бизнеса.
- Группируйте чаты (несколько сторонних производителей бесед) в группах могут включать только участников, у которых выполняется с помощью команды.
- Несколько сторонних производителей мгновенные сообщения (группа чаты) в Скайп для бизнеса могут включать только участников, у которых с помощью Скайп для бизнеса.
- Переход текущего peer-to-peer voice или видеозвонок вызову несколько сторонних производителей, включающие использование групп и Скайп для бизнес-пользователи не поддерживается.
- Передача для двусторонних чаты файлов или файлов вложений в чаты группы, из рабочих групп для Скайп для бизнеса и наоборот, не поддерживаются.
- Нет взаимодействие с Скайп для сохраняемого чата бизнеса.

Для всех этих ограничений (за исключением Persistent Chat) один возможное решение этой проблемы — для одного пользователя для запуска собрания и приглашать других пользователей присоединиться к собранию. Такое решение является основой для взаимодействия эскалации.

> [!Important]
> Что запускается как простой чата обмена Мгновенными сообщениями могут быстро переведите ее в звонка или на совещании ad-hoc. Мы понимаем, что эти сценарии необходимы для удобства использования и пользовательский интерфейс и мы постоянно меняется каждый раз взаимодействия между Скайп для бизнеса и группам пользователей. Советуем последних сведений.
