## <a name="coexistence-of-teams-and-skype-for-business-overview"></a>Общие сведения о сосуществовании Teams и Skype для бизнеса

В следующих разделах описываются режимы сосуществования, доступные при обновлении до Teams, и возможности, доступные для каждого режима. Кроме того, мы описываем взаимодействие между пользователями в клиентах Skype для бизнеса и пользователей на клиентах Teams, а также то, как на взаимодействие влияет выбранный режим сосуществования.

Teams предоставляет возможности совместной работы, чата, звонков и собраний. В зависимости от способа развертывания Teams эти возможности могут перекрываться с возможностями, предоставляемыми Skype для бизнеса для конкретного пользователя. Режим по умолчанию — запуск Teams вместе с Skype для бизнеса с перекрывающимися возможностями. Однако пользователю можно назначить один из нескольких режимов сосуществования (также называемых режимами обновления), которые предназначены для обеспечения того, чтобы эти возможности не перекрывались для этого пользователя (в этом случае доступно взаимодействие между Teams и Skype для бизнеса). Например, если у вас есть Skype для бизнеса Server локальные ресурсы со сложным развертыванием Корпоративная голосовая связь, но вы хотите, чтобы пользователи могли как можно быстрее пользоваться современными собраниями, вам может потребоваться оценить "Собрания сначала" в качестве [](meetings-first.md) альтернативного пути.

Мы рекомендуем ознакомиться со следующими режимами сосуществования, чтобы определить, какой путь является правильным для вашей организации.

> [!Important]
> Режимы сосуществования по-прежнему существуют после прекращения использования Skype для бизнеса Online, однако пользователи, подключенные к Сети, могут использовать только режим TeamsOnly. Пользователю, размещенного в Сети, больше нельзя назначить режим, отличный от TeamsOnly.  Как и до прекращения использования Skype для бизнеса Online, пользователям, размещенным в локальной среде, можно назначить любой режим, кроме *TeamsOnly*.

### <a name="islands-mode"></a>Режим "Острова"

По умолчанию пользователи могут запускать Teams вместе с Skype для бизнеса как два отдельных решения, которые предоставляют похожие и перекрывающиеся возможности. К ним относятся сведения о присутствии, чате, звонках и собраниях. Пользователи Teams также могут использовать новые возможности совместной работы, такие как команды и каналы, доступ к файлам в Microsoft 365 и приложения.

В этом режиме сосуществования, называемом **Islands**, каждое клиентское приложение работает как отдельный остров. Skype для бизнеса взаимодействует с Skype для бизнеса, а Teams — с Teams. Пользователи должны запускать оба клиента в любое время и могут взаимодействовать в собственном коде на клиенте, с которого было запущено взаимодействие. Таким образом, нет необходимости в взаимодействии в режиме **"Острова** ".

Чтобы избежать путаницы или регрессии Skype для бизнеса, Skype для бизнеса обрабатывает следующие интеграции, которые не обрабатываются в режиме Teams **Islands**:

- Внешние (федеративные) коммуникации.
- Голосовые службы ТСОП и голосовые приложения, интеграция с Office.
- Элементы управления HID для USB-устройств.
- Несколько других интеграций.

Телефонная система не поддерживается в Teams в **режиме "Острова** ". **Режим "** Острова" не поддерживает Корпоративная голосовая связь клиент Skype для бизнеса.

> [!Important]
> В **режиме "Острова**" все сообщения и звонки от федеративных пользователей (людей за пределами организации) доставляются Skype для бизнеса. После обновления до **режима "Только Teams** " все сообщения и звонки из-за пределов организации доставляются в Teams.

### <a name="teams-only"></a>Только Teams

Пользователь **Teams Only** (также *называемый обновленным* пользователем) имеет доступ ко всем возможностям в Teams. Они могут сохранить клиент Skype для бизнеса для присоединения к собраниям Skype для бизнеса, которые были организованы не обновленными пользователями или внешними сторонами. Обновленный пользователь может продолжать взаимодействовать с другими пользователями в организации, которые по-прежнему используют Skype для бизнеса, используя возможности взаимодействия между Teams и Skype для бизнеса (при условии, что Skype для бизнеса пользователи не находятся в режиме "**Острова").** Однако обновленный пользователь не может инициировать Skype для бизнеса чат, звонок или собрание.

Как только ваша организация будет готова для некоторых или всех пользователей использовать Teams в качестве своего средства обмена данными и совместной работы, обновите этих пользователей до режима **"Только Teams** ". Если вы выполняете обновление с режима **"** Острова", мы рекомендуем сначала ускорить внедрение Teams в организации перед началом процесса обновления. Такое внедрение позволяет избежать неработающих сценариев связи из-за того, что режим " **Острова" не** обеспечивает взаимодействие.

В **режиме "Только Teams** " Teams является приложением по умолчанию для протокола SIP/Tel. Ссылки в карточке контакта пользователя в Outlook для звонков или чата будут обрабатываться Teams.

Дополнительные сведения о переходе в режим **"Только Teams** " см. в разделе " [Рекомендации по режиму "Только Teams"](teams-only-mode-considerations.md).

### <a name="skype-for-business-only"></a>Только Skype для бизнеса

В этом режиме сосуществования пользователи остаются в Skype для бизнеса, а не Teams для чата, собраний и звонков и не используют Teams для команд и каналов. Этот режим доступен уже сегодня. однако в текущей реализации команды и каналы не отключаются автоматически для пользователя. Это можно сделать с помощью политики установки приложений, чтобы скрыть команды и файлы.

Этот режим можно использовать перед запуском управляемого развертывания Teams, чтобы запретить пользователям использовать Teams до того, как они будут иметь встроенную готовность. Этот режим также позволяет включить участие в собраниях Teams с проверкой подлинности для Skype для бизнеса пользователей при условии, что у пользователей есть лицензия на Teams.

### <a name="skype-for-business-with-teams-collaboration"></a>Skype для бизнеса совместной работы Teams

Используйте этот режим, чтобы ввести Teams в свою среду, пока вы продолжаете использовать существующие инвестиции в Skype для бизнеса. Оставьте Skype для бизнеса без изменений для чата, звонков и собраний. Добавьте возможности совместной работы Teams:

- Teams и каналы.
- Доступ к файлам в Microsoft 365 или Office 365.
- Приложений. Возможности связи Teams— частный чат, звонки и планирование собраний.

В этом режиме закрытый чат Teams, звонки и планирование собраний отключены по умолчанию.

Организации с начальной точкой Skype для бизнеса Server локальной или гибридной среды должны рассматривать этот режим как альтернативу режиму  "Острова", если они хотят предоставить своим пользователям возможность взаимодействия и предсказуемости для своих коммуникаций, а также иметь прогнозируемую временную шкалу для обновления до Teams (в отличие от использования насыщенности внедрения в режиме "**Острова").**

### <a name="skype-for-business-with-teams-collaboration-and-meetings-also-known-as-meetings-first"></a>Skype для бизнеса совместной работы и собраний Teams, также известных как "Собрания в первую очередь"

Используйте этот режим сосуществования, чтобы ускорить доступность возможностей собраний и совместной работы Teams в вашей организации. Режим сосуществования позволяет пользователям воспользоваться преимуществами улучшенных возможностей собраний Teams:

- Отличное качество.
- Транскрибирование и перевод.
- Размытие фона.
- Улучшенный пользовательский интерфейс на всех платформах, включая мобильные устройства и браузеры.

Наряду с использованием Teams для бесед на основе команд и каналов в этом режиме пользователи будут использовать Teams для планирования и проведения собраний. Частный чат и звонки остаются Skype для бизнеса. Teams и Skype для бизнеса воспользоваться преимуществами ряда возможностей "лучше вместе", таких как выверка сведений о присутствии, автоматическое удержание и отмена удержания и поддержка устройств HID в обоих приложениях. При необходимости можно скрыть команды и каналы с помощью политики установки приложений.

Этот режим сосуществования особенно полезен для организаций с Skype для бизнеса локальными развертываниями с Корпоративная голосовая связь. Эти организации, скорее всего, потребуется некоторое время, чтобы перейти на Teams и как можно скорее воспользоваться преимуществами улучшенных собраний Teams.

### <a name="interoperability-of-teams-and-skype-for-business"></a>Взаимодействие Teams и Skype для бизнеса

Взаимодействие — это возможность teams и Skype для бизнеса пользователей в одной организации обмениваться данными между Teams и Skype для бизнеса.

Взаимодействие регулируется режимом сосуществования (также известным как режим обновления) получателя. Взаимодействие отсутствует, если получатель находится в режиме **"Острова** ".

> [!Note]
> При развертывании в любом режиме сосуществования, кроме **Islands**, Teams и Skype для бизнеса могут взаимодействовать [,](#interoperability-of-teams-and-skype-for-business) позволяя пользователям общаться друг с другом и вызывать друг друга, а также обеспечивать плавность обмена данными в организации во время перехода на Teams. Режимы сосуществования управляют взаимодействием. Режим сосуществования получателя определяет, будет ли доступно взаимодействие. Например, если получатель находится в режиме, в котором чат доступен только в одном клиенте (например, Teams), взаимодействие чата обычно будет доступно, если инициатор использует другой клиент (в данном случае Skype для бизнеса) для запуска чата. С другой стороны, если получатель находится в режиме, в котором чат доступен в обоих клиентах (в режиме "Острова"), взаимодействие для чата будет недоступно. Сообщение будет получено получателем в том же клиенте, в котором инициатор запустил чат. Поэтому для правильного обмена данными в **режиме "Острова** " требуется насыщенность внедрения Teams. то есть все пользователи активно используют и отслеживают оба клиента.

> [!Note]
> **Чтобы получить последнюю версию сосуществования, клиентская версия должна быть последней доступной версией клиента в канале развертывания Office пользователя.**

#### <a name="native-interop-and-interop-escalation"></a>Укрупнение собственного взаимодействия и взаимодействия

Существует два типа взаимодействия: собственное взаимодействие и эскалация взаимодействия.

- В _клиенте,_ который в настоящее время используется пользователем, происходит собственное взаимодействие. Один пользователь будет в клиенте Skype для бизнеса, а другой — в Teams. Собственный интерфейс взаимодействия не будет передавать их другому клиенту для обмена данными. Пользователи смогут проводить беседы в клиенте, который они используют в настоящее время. Взаимодействие в машинном коде — это чат "один к одному" и вызов.
- Эскалация _взаимодействия_ означает, что в рамках помощи пользователям в выполнении дополнительных действий (например, предоставления общего доступа к рабочему столу) клиент упрощает создание собрания, к которому пользователи могут присоединиться, чтобы продолжить работу на этом собрании. Собрание создается на платформе инициатора действия. Пользователь или пользователи, не использующие эту платформу, получают ссылку на присоединение к собранию. Щелкнув эту ссылку, они присоединяются к собранию в совместимом клиенте (браузере, веб-приложении или полном клиенте в зависимости от конфигурации). Для эскалации взаимодействия Skype для бизнеса требуется последний клиент. Теперь доступна эскалация взаимодействия из Teams. Оба варианта поддерживаются в интерфейсах взаимодействия в клиенте и для федеративного взаимодействия между клиентами.

#### <a name="native-interop-experiences"></a>Встроенные возможности взаимодействия

В зависимости от режимов сосуществования, назначенных пользователям (как описано выше), доступны следующие собственные возможности взаимодействия:

Skype для бизнеса пользователи могут общаться в чате один на один с пользователями Teams и наоборот. Чат взаимодействия должен проходить через шлюз взаимодействия, который входит в состав облачных служб Teams (и, следовательно, существует только в Сети). Чаты взаимодействия — это обычный текст: форматированный текст и смайлики не поддерживаются. Пользователи в Teams и Skype для бизнеса получают уведомления о том, что беседа является беседой взаимодействия.

<!--![Screen shot of Interop chat experience from Teams.](media/Interop_chat_experience_from_Teams.png "Interop chat experience from Teams")-->

Skype для бизнеса пользователи могут совершать голосовые и видеозвонки для пользователей Teams один к одному, а пользователи Teams могут делать то же самое.

<!--![Screen shot of Interop calling experience from Teams.](media/Interop_calling_experience_from_Teams.png "Interop calling experience from Teams")-->

> [!Important]
> Взаимодействие с локальным развертыванием Skype для бизнеса требует, чтобы локальная среда была в гибридном режиме с Teams. Дополнительные сведения см. в статье о настройке [гибридного подключения между Skype для бизнеса Server и Microsoft 365 или Office 365](/SkypeForBusiness/hybrid/configure-hybrid-connectivity).

Эти возможности взаимодействия доступны пользователям, которым назначен один из следующих режимов сосуществования: **Skype для бизнеса** совместной работы Teams, **Skype для бизнеса** совместной работы и собраний Teams, **только Skype для бизнеса или** **Teams**. Взаимодействие с пользователями в режиме " **Острова" отсутствует** .

#### <a name="native-interop-experience-limitations"></a>Ограничения взаимодействия в машинном коде

Из-за различий в протоколах и технологиях невозможно поддерживать все возможности в собственном коде. В частности, недоступны следующие возможности:

- Markdown, форматированный текст и полный набор смайликов не поддерживаются ни в Teams, ни Skype для бизнеса. Другие собственные функции поля создания в чатах Teams не поддерживаются.
- Общий доступ к экрану (общий доступ к рабочему столу или приложению) между Teams и Skype для бизнеса не поддерживается в собственном коде. Однако это поддерживается посредством эскалации взаимодействия.
- Групповые чаты (много сторонние беседы) в Teams могут включать только участников, использующих Teams.
- В нескольких чатах (групповых чатах) в Skype для бизнеса могут быть только участники, использующие Skype для бизнеса. Однако эскалация взаимодействия для нескольких сторон доступна из Skype для бизнеса.
- Эскалация текущего однорангового голосового или видеозвонка к много стороннему вызову, включающему Как Teams, так и Skype для бизнеса пользователей, не поддерживается.
- Передача файлов для двух сторонних чатов или вложений файлов в групповых чатах из Teams в Skype для бизнеса и наоборот не поддерживается.
- Взаимодействие с Skype для бизнеса сохраняемого чата отсутствует.

Для всех этих ограничений (за исключением сохраняемого чата) одним из возможных обходных решений является запуск собрания и приглашение другого пользователя присоединиться к собранию.

Это решение является основой для эскалации взаимодействия. В частности, общий доступ к экрану и эскалация для многопользовательских функций недоступны в собственном коде, но они поддерживаются посредством эскалации взаимодействия.

#### <a name="interop-escalation-experiences"></a>Возможности эскалации взаимодействия

Эскалация взаимодействия состоит в дополнении собственных возможностей взаимодействия управляемыми эскалациями к собраниям. Собрания предоставляют широкие возможности, доступные любому пользователю, независимо от того, какой клиент у них есть.

Когда пользователь Teams активирует эскалацию взаимодействия, создается собрание Teams. Когда он активируется пользователем Skype для бизнеса, создается Skype для бизнеса собрания. В обоих случаях созданное собрание является собранием **"** Провести собрание", которое не отражается в календаре пользователя.

Другая сторона получает ссылку на присоединение к собранию через чат взаимодействия и присоединяется, щелкнув эту ссылку. Если у Skype для бизнеса есть учетная запись Teams и он приглашен пользователем Teams, он присоединится к собранию, прошедшему проверку подлинности. В противном случае они присоединятся как анонимный участник. И наоборот, пользователи Teams почти всегда имеют учетную запись Skype для бизнеса и клиент Skype для бизнеса, которые они могут использовать для присоединения к собранию Skype для бизнеса в качестве участника, прошедшего проверку подлинности, но они также могут присоединиться как анонимный участник, например с помощью приложения для собраний Skype.

После присоединения к собранию стороны могут выполнять любые действия, поддерживаемые в собраниях, такие как общий доступ к рабочему столу или содержимому, общий доступ к файлам или передача файлов, добавление других участников и т. д.

### <a name="meetings"></a>Собрания

Независимо от режима, пользователи всегда могут присоединяться к собраниям любого типа, на которые они приглашены, независимо от того, Skype для бизнеса или Teams.  Однако пользователи должны присоединиться к собранию с соответствующим клиентом, который соответствует типу собрания:

- Если собрание является собранием Teams, все участники (будь то Пользователи TeamsOnly, Islands или Skype для бизнеса) используют клиент Teams для присоединения к собранию. Если Teams не установлен, пользователь будет перенаправлен в Интернет при попытке присоединиться к собранию.

- Если собрание является Skype для бизнеса собранием, все участники (будь то Пользователи TeamsOnly, Islands или Skype для бизнеса) используют клиент Skype для бизнеса для присоединения к собранию. Если клиент Skype для бизнеса не установлен, пользователь будет перенаправлен в Интернет для присоединения через приложение для собраний Skype. 

  В некоторых случаях участники в режиме TeamsOnly смогут присоединяться к собраниям Skype для бизнеса только с помощью Skype для бизнеса Web App или приложения "Собрания Skype" в качестве анонимного пользователя. В конечном итоге это будет справедливо для всех пользователей в режиме TeamsOnly. Дополнительные сведения см[. в Skype для бизнеса online](skype-for-business-online-retirement.md#what-to-expect-post-retirement).

При организации собраний тип запланированного собрания зависит от режима организатора, как показано в следующей таблице:

| Режим организатора    |      Поведение |
| :------------------ | :---------------- |
| TeamsOnly, SfbWithTeamsCollabAndMeetings | Все собрания, запланированные в Teams. Skype для бизнеса надстройка недоступна в Outlook. |
| SfbWithTeamsCollab, SfbOnly | Все собрания, запланированные в Skype для бизнеса. Надстройка Teams недоступна в Outlook. |
| Острова | По умолчанию собрания можно планировать как в Skype для бизнеса, так и в Teams. Обе надстройки доступны в Outlook. Однако при необходимости можно требовать, чтобы пользователи в Islands всегда назначали собрания в Teams, назначая им экземпляр TeamsMeetingPolicy с preferredMeetingProviderForIslandsMode=Teams.|

### <a name="interoperability"></a>Взаимодействия

Как описано выше в статье о взаимодействии [Teams и Skype для бизнеса](#interoperability-of-teams-and-skype-for-business), Teams поддерживает взаимодействие с Skype для бизнеса в определенных сценариях. Взаимодействие — это чат или звонок между пользователем Skype для бизнеса и пользователем Teams.  Взаимодействие возможно только между двумя пользователями. многопользовательский чат, вызов или добавление дополнительных пользователей не поддерживаются.

Чат взаимодействия или звонок между двумя пользователями создается, когда выполняется каждое из следующих условий:

- Один пользователь использует Teams, а другой — Skype для бизнеса.

- Если оба пользователя находятся в одной и той же организации, получателем исходного сообщения является NOT Islands (в противном случае обмен данными будет выполняться в одном клиенте). В федеративных сценариях отправляющий пользователь использует Teams, а получатель не находится в режиме TeamsOnly.

- У пользователя Teams нет учетной записи Skype для бизнеса, размещенной локально.

В рамках взаимодействия чат предназначен только для обычного текста. Кроме того, общий доступ к файлам и демонстрация экрана невозможно в *самом чате взаимодействия*. Однако пользователи в беседе взаимодействия могут легко добиться общего доступа к файлам и (или) экрану, создав собрание по запросу из чата взаимодействия, как описано ниже:

- Если пользователь Teams пытается поделиться своим экраном, собрание Teams по запросу создается автоматически, а ссылка на приглашение на это собрание отправляется клиенту Skype для бизнеса пользователя. Щелкнув ссылку, Skype для бизнеса откроет Teams и присоединится к собранию. Оба пользователя теперь находятся на собрании Teams и могут предоставлять общий доступ по мере необходимости.

- Если пользователь Skype для бизнеса использует клиент 2018 или более поздней версии и пытается поделиться любым содержимым, по запросу создается собрание Skype для бизнеса по запросу и ссылка на приглашение на это собрание отправляется клиенту пользователя Teams. Щелкнув ссылку, пользователь Teams попытается присоединиться к Skype для бизнеса собрания. Если у пользователя Teams установлен Skype для бизнеса клиент, он откроется и пользователю будет предложено выполнить вход (если он еще не выполнил вход).  Если у пользователя Teams не установлен Skype для бизнеса клиента, пользователю будет предложено использовать веб-версию. После входа оба пользователя находятся в Skype для бизнеса и могут предоставлять общий доступ по мере необходимости.

- Изменение значка целевого пользователя, отображающее "S" для Skype для бизнеса.

### <a name="presence"></a>Присутствие

Присутствие для данного пользователя зависит от действий пользователя в службе через клиент. Затем сведения о присутствии публикуются для просмотра другими пользователями.  Skype для бизнеса Teams — это отдельные службы с отдельными клиентами, поэтому каждая служба имеет собственное состояние присутствия для пользователя.   Также выполняется синхронизация между службами присутствия в Teams и Skype для бизнеса Online.  Это позволяет одной службе при необходимости публиковать сведения о присутствии пользователя из другой службы.

Поведение публикации сведений о присутствии зависит от режима пользователя. Существует три основных варианта:

- Если пользователь находится в режиме TeamsOnly, все остальные пользователи видят присутствие Teams для этого пользователя, независимо от того, какой клиент он использует.

- Если пользователь находится в любом из режимов Skype для бизнеса, все остальные пользователи видят Skype для бизнеса для этого пользователя, независимо от того, какой клиент он использует.

- Если пользователь находится в режиме "Острова", сведения о присутствии, опубликованные в Skype для бизнеса и Teams, независимы, поэтому присутствие пользователей в той же организации будет зависеть от клиента другого пользователя. Пользователи федеративных организаций будут видеть присутствие этого пользователя на основе их Skype для бизнеса, так как федеративный трафик к пользователю в режиме "Острова" попадает в Skype для бизнеса.

Например, предположим, что пользователь А находится в режиме "Острова". Если пользователь А активен в Teams, но не выполнил вход в Skype для бизнеса, другие пользователи будут видеть пользователя A активным из своего клиента Teams, но в Skype для бизнеса клиенте они будут видеть пользователя A как автономный. Это по умолчанию, так как пользователь A не может получить доступ, если он не выполняет клиент.


### <a name="federation"></a>Федерация

Для федерации из Teams с другим пользователем, использующим Skype для бизнеса, требуется, чтобы пользователь Teams был размещен в интернете в Skype для бизнеса. TeamsUpgradePolicy управляет маршрутизативом для входящих федеративных чатов и звонков. Поведение федеративной маршрутизации такое же, как в сценариях с одним клиентом, за исключением режима "Острова". Когда получатели находятся в режиме "Острова":

- Чаты и звонки, инициированные из Teams, Skype для бизнеса, если получатель находится в федеративном клиенте.
- Чаты и вызовы, инициированные из Teams, попадать в Teams, если получатель находится в одном клиенте.
- Чаты и вызовы, инициируемые Skype для бизнеса всегда попадать в Skype для бизнеса.

Федеративный чат может быть собственным потоком или потоком взаимодействия. Ознакомьтесь [с взаимодействием и собственными потоками беседы](#interop-versus-native-conversation-threads).

- Если получатель и отправитель находятся в режиме обновления TeamsOnly, беседа будет иметь собственный чат, который включает все широкие возможности обмена сообщениями и звонков. Дополнительные сведения см. в [разделе "Собственный чат" для внешних (федеративных) пользователей в Teams](native-chat-for-external-users.md).

- Если один из участников беседы не находится в режиме обновления TeamsOnly, беседа остается взаимодействием с текстовыми сообщениями. Пользовательский интерфейс предоставляет федеративные чаты аналогично однотенантным потокам взаимодействия, за исключением того, что пользователь является внешним.

Дополнительные сведения см. в статье "Управление внешним доступом в [Microsoft Teams](manage-external-access.md) и собственный чат для внешних [(федеративных) пользователей в Teams"](native-chat-for-external-users.md).

### <a name="contacts"></a>Контакты

Команды и Skype для бизнеса имеют отдельные списки контактов. Это означает, что добавление, удаление и изменение контактов, внесенные в одну систему, не синхронизируются с другой системой. Однако контакты из Skype для бизнеса автоматически копируются в Teams при возникновении одного из двух определенных событий:

- Для любого Skype для бизнеса Online при первом входе в Teams контакты из Skype для бизнеса будут скопированы в Teams.  Такое поведение недоступно для пользователей с локальной учетной записью в Skype для бизнеса Server.

- После обновления пользователя до TeamsOnly (путем назначения TeamsUpgradePolicy или через Move-CsUser -MoveToTeams) при следующем входе пользователя в Teams существующие контакты в Skype для бизнеса будут объединены с существующими контактами, уже существующими в Teams. Это происходит независимо от того, был ли пользователь перемещен в TeamsOnly из локальной среды или из Интернета.

В обоих случаях передача контактов из Skype для бизнеса в Teams выполняется асинхронно, поэтому их отображение в Teams может занять несколько минут. Два указанных выше события инициируют копирование.
