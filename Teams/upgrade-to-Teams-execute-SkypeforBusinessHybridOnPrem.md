# <a name="upgrade-from-skype-for-business-on-premises-to-teams"></a>Обновление с Skype для бизнеса локальной среды до Teams

![Схема перехода на более поздние версии с акцентом на развертывание и реализацию.](media/upgrade-banner-deployment.png "Этапы пути обновления с акцентом на этап развертывания и реализации")

Эта статья является частью этапа развертывания и реализации процесса обновления. Прежде чем продолжить, убедитесь, что вы выполнили следующие действия:

- [Прикрепление заинтересованных лиц проекта](upgrade-enlist-stakeholders.md)
- [Определение области проекта](./upgrade-define-project-scope.md)
- [Понимание сосуществования и взаимодействия Skype для бизнеса и Teams](./teams-and-skypeforbusiness-coexistence-and-interoperability.md)
- [Выбор пути обновления](upgrade-and-coexistence-of-skypeforbusiness-and-teams.md)
- [Подготовка среды](./upgrade-prepare-environment.md)
- [Подготовка организации](./upgrade-prepare-organization.md)
- [Проводит пилотный проект](./pilot-essentials.md)

Если вы развернули Skype для бизнеса Server или Microsoft Lync Server в локальной среде и ваша организация хочет выполнить обновление до Teams, следуйте инструкциям в этой статье. Необходимо настроить гибридное подключение к организации Microsoft 365, как описано в разделе "Планирование гибридного подключения между Skype для бизнеса Server [и Teams](/skypeforbusiness/hybrid/plan-hybrid-connectivity).

Прежде чем приступить к работе, ознакомьтесь с важными рекомендациями для организаций с Skype для бизнеса Server локальной среде далее в этой статье.[](#important-considerations-for-organizations-with-skype-for-business-server-on-premises)

> [!IMPORTANT]
> Skype для бизнеса Online прекращена 31 июля 2021 г. Чтобы максимально эффективно реализовать преимущества и убедиться, что у вашей организации есть соответствующее время для реализации обновления, мы рекомендуем начать работу Microsoft Teams сегодня. Помните, что успешное обновление соответствует технической и пользовательской готовности, поэтому не забудьте использовать это руководство при переходе к Microsoft Teams.

## <a name="step-1-configure-hybrid-connectivity"></a>Шаг 1. Настройка гибридного подключения

Ключевым предварительным требованием для обновления локальных пользователей до Teams является настройка гибридного подключения Skype для бизнеса Server локального развертывания.

Начните с чтения [плана гибридного подключения](/SkypeForBusiness/hybrid/plan-hybrid-connectivity?toc=%2fSkypeForBusiness%2fsfbhybridtoc%2ftoc.json), а затем выполните задачи, описанные в разделе ["Настройка гибридного подключения"](/skypeforbusiness/skype-for-business-hybrid-solutions/deploy-hybrid-connectivity/deploy-hybrid-connectivity).

## <a name="step-2-set-transitional-coexistence-mode-optional"></a>Шаг 2. Настройка режима переходного сосуществования (необязательно)

Сосуществование и взаимодействие между Skype для бизнеса и Teams и пользователями определяются режимами [сосуществования](migration-interop-guidance-for-teams-with-skype.md). Гибридные организации по умолчанию находятся в режиме "Острова". Режим "Острова" позволяет пользователям использовать Teams и Skype для бизнеса клиенты параллельно. Организации могут при необходимости использовать другие режимы сосуществования, чтобы обеспечить предсказуемый интерфейс для конечных пользователей по мере перехода от Skype для бизнеса к Teams. Независимо от режима, который организация использует для локальных пользователей, при перемещении этих пользователей из локальной среды в облако они становятся TeamsOnly (и не могут иметь какой-либо другой режим).  Пока пользователи по-прежнему используют Skype для бизнеса Server, им можно назначить любой режим, кроме TeamsOnly. При необходимости пользователей TeamsOnly можно переместить обратно в локальную среду, что приведет к получении глобальной политики TeamsUpgradePolicy клиента.

Когда пользователь находится в любом Skype для бизнеса, все входящие чаты и вызовы направляются в клиент Skype для бизнеса пользователя. Чтобы избежать путаницы с конечным пользователем и обеспечить правильную маршрутизацию, вызовы и функции чата в клиенте Teams отключены для пользователя, которому назначен любой из Skype для бизнеса *режимов*. Планирование собраний в Teams отключено, когда пользователи находятся в режиме SfBOnly или SfBWithTeamsCollab и включены, когда  пользователь находится в режиме SfBWithTeamsCollabAndMeetings.

В зависимости от требований можно назначить соответствующий режим сосуществования в зависимости от выбранного организацией пути обновления. Дополнительные сведения см. [](migration-interop-guidance-for-teams-with-skype.md) в руководстве по миграции и взаимодействию для организаций, использующих Teams вместе с Skype для бизнеса и настройкой параметров сосуществования и [обновления](./setting-your-coexistence-and-upgrade-settings.md).

## <a name="step-3-move-users-from-skype-for-business-on-premises-to-teams-only"></a>Шаг 3. Перемещение пользователей из локальной Skype для бизнеса в Teams только

Корпорация Майкрософт недавно упростила процесс перемещения пользователей в TeamsOnly. Этот процесс теперь является одним шагом независимо от используемой Skype для бизнеса Server или Lync Server 2013. Дополнительные сведения см. [](/SkypeForBusiness/hybrid/move-users-between-on-premises-and-cloud) в статье "Перемещение пользователей между локальной средой и облаком и перемещение пользователей из локальной среды [в](/SkypeForBusiness/hybrid/move-users-from-on-premises-to-teams) Teams".

## <a name="step-4-complete-your-migration-to-the-cloud-by-disabling-hybrid-and-decommissioning-on-premises-skype-for-business"></a>Шаг 4. Завершение миграции в облако путем отключения гибридной среды и вывода из эксплуатации локальных Skype для бизнеса

После перемещения всех пользователей из локальной среды в облако вы можете списать локальное развертывание Skype для бизнеса развертывания. Дополнительные сведения см. в разделе "Вывод из эксплуатации локальной [Skype для бизнеса среды"](/skypeforbusiness/hybrid/decommission-on-prem-overview).

## <a name="phone-system-and-pstn-connectivity-options"></a>телефонная система подключения и ТСОП

телефонная система с Teams поддерживается после того, как пользователь находится в режиме TeamsOnly. (Если пользователь находится в режиме "Острова", телефонная система поддерживается только Skype для бизнеса.)

### <a name="pstn-connectivity-options"></a>Параметры подключения ТСОП

При рассмотрении вариантов подключения к общедоступной телефонной сети (ТСОП) существует два возможных сценария перехода с локального Skype для бизнеса на режим TeamsOnly:

- Пользователь в локальной Skype для бизнеса с Корпоративная голосовая связь, который будет переключяться в Интернет и использовать план звонков Майкрософт. Миграция этого пользователя в Teams требует перемещения локальной учетной записи пользователя Skype для бизнеса в облако и координации перемещения с помощью A) порта номера телефона этого пользователя в план звонков Майкрософт или B), назначая новый номер подписчика из доступных регионов.  Дополнительные сведения см. в Skype для бизнеса Server локальной [службе с Корпоративная голосовая связь в план звонков Майкрософт](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan).

- Пользователь в локальной Skype для бизнеса с Корпоративная голосовая связь, который будет переключяться в режим "в сети" и поддерживать локальное подключение к ТСОП. Для переноса этого пользователя в Teams требуется переместить локальную учетную запись пользователя Skype для бизнеса в облако и координировать этот переход с миграцией пользователя на прямую маршрутизацию. Дополнительные сведения см. в Skype для бизнеса Server локальной среде [с Корпоративная голосовая связь прямой маршрутизации](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing).

## <a name="important-considerations-for-organizations-with-skype-for-business-server-on-premises"></a>Важные рекомендации для организаций с Skype для бизнеса Server локальной среде

- Настройка гибридного Skype для бизнеса является необходимым условием для перехода в режим TeamsOnly. Локальные пользователи могут использовать Skype для бизнеса Server в режиме Teams без гибридного использования. Однако переход в режим TeamsOnly невозможно выполнить без перемещения пользователя в облако с помощью [Move-CsUser](/SkypeForBusiness/hybrid/move-users-between-on-premises-and-cloud), для которого требуется гибридное подключение. Дополнительные сведения см. в [разделе "Настройка гибридного подключения"](/skypeforbusiness/hybrid/configure-hybrid-connectivity). Предстоящее прекращение поддержки Skype для бизнеса Online не изменит это требование. Чтобы организации могли перейти Skype для бизнеса Server на Teams, они по-прежнему должны настраивать и настраивать гибридную среду с помощью того же набора инструментов, как и до прекращения *поддержки*.

- Чтобы переместить локального пользователя в облако, `Move-CsUser` используйте локальные средства администрирования. Вам больше не нужно указывать параметр `-MoveToTeams` для перемещения пользователей непосредственно из локальной среды в TeamsOnly. Пользователям автоматически назначается режим TeamsOnly`-MoveToTeams`, а локальные собрания автоматически преобразуются в Teams собраний независимо от того, указан ли переключатель.

- Если в вашей организации Skype для бизнеса Server и вы не настроите гибридное подключение, но хотите использовать Teams, для администрирования Teams необходимо использовать учетную запись администратора с доменом .onmicrosoft.com. Без гибридного подключения средства администрирования не распознают локальные домены.

- Teams пользователей с локальной учетной записью Skype для бизнеса (то есть они еще не были перемещены в облако с помощью Move-CsUser) не могут взаимодействовать с любыми Skype для бизнеса пользователями и не могут выполнять федерацию с внешними пользователями. Эта функция доступна только после того, как пользователи перемещаются в облако и являются пользователями TeamsOnly.

- Если у вас есть пользователи с Skype для бизнеса учетными записями в локальной среде или у вас по-прежнему есть запись DNS lyncdiscover для локального развертывания, вы не сможете назначить режим TeamsOnly на уровне клиента. Сначала необходимо переместить всех пользователей с локальными учетными записями Skype для бизнеса в облако `Move-CsUser`с помощью . Затем выполните действия, описанные в разделе ["](/skypeforbusiness/hybrid/cloud-consolidation-disabling-hybrid)Отключение гибридной среды", чтобы завершить миграцию в облако, включая удаление записей DNS. `Grant-CsTeamsUpgradePolicy -PolicyName UpgradeToTeams`не будет работать на уровне клиента, если обнаружена запись DNS lyncdiscover, которая указывает на расположение, отличное от Office 365.

- Необходимо убедиться, что пользователи правильно синхронизированы с Azure AD с правильными Skype для бизнеса атрибутами. Все эти атрибуты являются префиксами с msRTCSIP-. Если пользователи не синхронизируются с Azure AD, средства управления в Teams не смогут управлять этими пользователями. (Например, вы не сможете назначать локальным пользователям политики Teams, если вы не синхронизируете эти атрибуты должным образом.) Дополнительные сведения см. в разделе [Azure AD Подключение для Teams и Skype для бизнеса](/SkypeForBusiness/hybrid/configure-azure-ad-connect).

- Чтобы создать TeamsOnly в гибридной организации, необходимо сначала включить пользователя в локальной среде *Skype для бизнеса Server*, а затем переместить пользователя из локальной среды в облако с помощью Move-CsUser.  Создание пользователя в локальной среде сначала гарантирует, что все остальные локальные Skype для бизнеса пользователи смогут перенаправить его только что созданному пользователю. После *перемещения* всех пользователей в интернет больше не нужно сначала включить пользователей в локальной среде.

- Если вы хотите отображать уведомления в клиенте Skype для бизнеса для локальных пользователей, необходимо использовать TeamsUpgradePolicy в локальном наборе инструментов. Для локальных пользователей применим только параметр NotifySfbUsers.  Локальные пользователи получают свой режим от сетевых экземпляров TeamsUpgradePolicy. См. заметки [в grant-CsTeamsUpgradePolicy](/powershell/module/skype/grant-csteamsupgradepolicy).

> [!NOTE]
> Все новые клиенты, созданные после 3 сентября 2019 г., создаются как клиенты TeamsOnly, если в организации еще нет локального развертывания Skype для бизнеса Server. Корпорация Майкрософт использует записи DNS для идентификации локальных Skype для бизнеса Server организаций. Дополнительные сведения см. в статье [о последствиях DNS для локальных организаций, которые становятся гибридными](/SkypeForBusiness/hybrid/configure-hybrid-connectivity#dns-implications-for-on-premises-organizations-that-become-hybrid).
