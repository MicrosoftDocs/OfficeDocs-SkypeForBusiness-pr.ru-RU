# <a name="upgrade-from-skype-for-business-on-premises-to-teams"></a>Обновив Skype для бизнеса локально до Teams

![Схема пути обновления с акцентом на развертывании и внедрении.](media/upgrade-banner-deployment.png "Этапы пути обновления с акцентом на этапе развертывания и реализации")

Эта статья является частью этапа развертывания и реализации, на этапе обновления. Прежде чем при этом подтверждать, что вы выполнили следующие действия:

-   [Привлечение заинтересованных лиц по проекту](upgrade-enlist-stakeholders.md)
-   [Определение области проекта](./upgrade-define-project-scope.md)
-   [Понятное сосуществование и совместное Skype для бизнеса и Teams](./teams-and-skypeforbusiness-coexistence-and-interoperability.md)
-   [Выбор пути обновления](upgrade-and-coexistence-of-skypeforbusiness-and-teams.md)
-   [Подготовить среду](./upgrade-prepare-environment.md)
-   [Подготовленные организации](./upgrade-prepare-organization.md)
-   [Пилотный проект](./pilot-essentials.md)

Если вы развернули Skype для бизнеса Server или Microsoft Lync и хотите перейти на Teams, следуйте указаниям в этой статье. Необходимо настроить гибридное подключение к организации Microsoft 365 или Office 365, а также определить требования к совместному Teams пользователям.

Прежде чем начать, [ИТ-специалисты](#important-considerations-for-organizations-with-skype-for-business-server-on-premises) и администраторы должны и далее в этой статье рассмотреть важные аспекты для Skype для бизнеса Server локальной работы.

> [!IMPORTANT]
> Skype для бизнеса Online была отменена 31 июля 2021 г. Чтобы максимально увеличить преимущества и убедиться, что у вашей организации есть надлежащее время на реализацию обновления, мы рекомендуем начать свой путь к Microsoft Teams сегодня. Помните, что успешное обновление соответствует технической и пользовательской готовности, поэтому не забудьте использовать это руководство при переходе к Microsoft Teams.

## <a name="step-1-configure-hybrid-connectivity"></a>Шаг 1. Настройка гибридного подключения 

Для обновления локального развертывания пользователей на Teams необходимо настроить гибридное подключение для Skype для бизнеса Server локального развертывания. 

Для начала прочитайте [статью Планирование](/SkypeForBusiness/hybrid/plan-hybrid-connectivity?toc=%2fSkypeForBusiness%2fsfbhybridtoc%2ftoc.json)гибридного подключения, а затем выполните задачи, описанные в [области Настройка гибридного подключения.](/skypeforbusiness/skype-for-business-hybrid-solutions/deploy-hybrid-connectivity/deploy-hybrid-connectivity)


## <a name="step-2-set-transitional-coexistence-mode-optional"></a>Шаг 2. Настройка режима совместной работы (необязательно)

Сосуществование и взаимодействия между клиентами Skype для бизнеса и Teams и пользователями определяются режимами Teams обновления.  Гибридные организации, а также все организации, созданные до 3 сентября 2019 г., по умолчанию находятся в режиме "Острова". Режим "Острова" позволяет пользователям одновременно Teams и Skype для бизнеса клиентах. Для организации, которая переходит на Teams, режим TeamsOnly является конечным пунктом назначения для каждого пользователя, хотя не всем пользователям необходимо одновременно на назначение TeamsOnly (или любого другого режима).

Прежде чем пользователи будут переначлены в режим TeamsOnly, организации могут при желании использовать любой из режимов Skype для бизнеса, чтобы обеспечить предсказуемую связь между пользователями, которые работают в режиме TeamsOnly, и пользователями, которые еще не работают.  Режимы сосуществования Skype для бизнеса (SfBOnly, SfBWithTeamsCollab, SfBWithTeamsCollabAndMeetings) обеспечивают простой и предсказуемый опыт для конечных пользователей по мере перехода организаций с Skype для бизнеса на Teams. Пользователю, который Skype для бизнеса Server локальной сети, можно на назначенном режиме, кроме TeamsOnly. После того как пользователь перемещается в облако, он получает teamsOnly.  Пользователям, которые работают только в облаке, не может быть назначен режим, кроме TeamsOnly. (Однако их можно снова переместить в локальное решение, что приведет к получении глобальной политики клиента TeamsUpgradePolicy.)

Когда пользователь находится в любом из режимов Skype для бизнеса, все входящие чаты и звонки перенанаются его Skype для бизнеса клиенту. Чтобы избежать путаницы с конечными пользователями и обеспечить правильную маршрутику, вызовы и чаты в клиенте Teams отключены, когда пользователь Skype для бизнеса режиме. Точно так же планирование собраний в Teams отключено, если пользователи находятся в режиме SfBOnly или SfBWithTeamsCollab и явно включены, когда пользователь находится в режиме SfBWithTeamsCollabAndMeetings.

В зависимости от ваших требований вы можете назначить соответствующий режим совместной работы на основе выбранного в организации пути обновления. Дополнительные сведения [](migration-interop-guidance-for-teams-with-skype.md) см. в инструкциях по миграции и совместному использованию данных для организаций, использующих Teams вместе с Skype для бизнеса и Настройка параметров сосуществования и [обновления.](./setting-your-coexistence-and-upgrade-settings.md)


## <a name="step-3-move-users-from-skype-for-business-on-premises-to-teams-only"></a>Шаг 3. Перемещение пользователей из локальной Skype для бизнеса в Teams-

Корпорация Майкрософт недавно упростила процесс перемещения пользователей в TeamsOnly. Этот процесс теперь является одним этапом независимо от используемой версии Skype для бизнеса Server или Lync Server 2013. Дополнительные сведения [](/SkypeForBusiness/hybrid/move-users-between-on-premises-and-cloud) см. в [Teams.](/SkypeForBusiness/hybrid/move-users-from-on-premises-to-teams) 

## <a name="step-4-disable-hybrid-to-complete-your-migration-to-the-cloud"></a>Шаг 4. Отключение гибридного решения для завершения миграции в облако

После того как вы переместите всех пользователей из локальной службы в облако, вы можете списание локального Skype для бизнеса развертывания. Дополнительные сведения см. в [теме Отключение гибридной системы для завершения миграции в облако.](/skypeforbusiness/hybrid/cloud-consolidation-disabling-hybrid)


## <a name="phone-system-and-pstn-connectivity-options"></a>телефонная система подключения к ОКП и ДНР

телефонная система с Teams поддерживается после того, как пользователь находится в режиме TeamsOnly. (Если пользователь находится в режиме "Острова", телефонная система поддерживается только Skype для бизнеса.) 

### <a name="pstn-connectivity-options"></a>Параметры подключения через ДНР

При рассмотрении параметров подключения к телефонной сети общего перейти на Skype для бизнеса teamsOnly можно в двух сценариях:

- Пользователь в Skype для бизнеса локальной службе Корпоративная голосовая связь, который переходит на веб-сайт и использует план Microsoft Calling. Для переноса этого пользователя в Teams требуется переместить его локальной учетной записи Skype для бизнеса в облако и координировать перенос с помощью А) переноса номера телефона этого пользователя в план звонков Майкрософт или Б), назначив новый номер абонента из доступных регионов.  Дополнительные сведения см. в [Skype для бизнеса Server локальной службы с Корпоративная голосовая связь плана звонков Майкрософт.](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-server-on-premises-with-enterprise-voice-to-microsoft-calling-plan)

- Пользователь в Skype для бизнеса локальной службе Корпоративная голосовая связь, который будет передвигаться на веб-сайт и поддерживать локальное подключение к ОКП. Для Teams миграции пользователя в облако требуется переместить его Skype для бизнеса учетную запись пользователя в облако и координировать его миграцию на прямую маршрутику. Дополнительные сведения см. в Skype для бизнеса Server [локальной и Корпоративная голосовая связь прямой маршрут.](upgrade-to-teams-on-prem-pstn-considerations.md#from-skype-for-business-server-on-premises-with-enterprise-voice-to-direct-routing)


## <a name="important-considerations-for-organizations-with-skype-for-business-server-on-premises"></a>Важные аспекты для организаций с Skype для бизнеса Server локальной

- В следующих статьях описаны важные понятия обновления и принципы совместной использования.
    - [Совместное Teams Skype для бизнеса](teams-and-skypeforbusiness-coexistence-and-interoperability.md)
    - [Режимы сосуществования — справка](migration-interop-guidance-for-teams-with-skype.md)
    - [Взаимодействие с клиентом Teams и соответствие режимам сосуществования](teams-client-experience-and-conformance-to-coexistence-modes.md)

- Настройка гибридной Skype для бизнеса является необходимым условием для перехода в режим TeamsOnly. Хотя пользователи локальной сети Skype для бизнеса Server могут использовать Teams в режиме "Острова" без гибридного решения, переход в режим TeamsOnly невозможно без перемещения пользователя в облако с помощью [Move-CsUser,](/SkypeForBusiness/hybrid/move-users-between-on-premises-and-cloud)для которого требуется гибридное подключение. Дополнительные сведения см. [в теме Настройка гибридного подключения.](/skypeforbusiness/hybrid/configure-hybrid-connectivity) Кроме того, предстоящее Skype для бизнеса Online не изменит это требование. Чтобы организации переходили Skype для бизнеса Server на Teams, им по-прежнему необходимо настроить и настроить гибридную конфигурацию с помощью того же набора инструментов, что и до *выхода на пенсию.*

- Чтобы переместить локального пользователя в облако, используйте средства локального `Move-CsUser` администрирования. Для перемещения пользователей непосредственно из локальной сети в TeamsOnly больше не требуется указывать `-MoveToTeams` переключатель. При перемещении пользователя из локальной службы в облако с помощью , пользователям автоматически назначен режим TeamsOnly, а их собрания из локальной службы автоматически преобразуются в собрания Teams, как если бы переключатель не был указан. `Move-CsUser` `-MoveToTeams switch had been specified`

- Если в вашей организации Skype для бизнеса Server и вы не настроили гибридное подключение, но все равно хотите использовать Teams, для администрирования Teams функций необходимо использовать учетную запись администратора с доменом onmicrosoft.com. Без гибридного подключения средства администрирования не распознают ваши домены. 

- Teams пользователей, у которых есть Skype для бизнеса учетная запись локальной учетной записи (то есть они еще не были перемещены в облако с помощью Move-CsUser), не могут пересекаться с другими пользователями Skype для бизнеса, а также федерироваться с внешними пользователями. Эта функция доступна только после того, как пользователи перемещаются в облако и являются пользователями TeamsOnly. 

- Если у вас есть пользователи с локальной Skype для бизнеса учетными записями или у вас по-прежнему есть запись DNS lyncdiscover для локального развертывания, назначить режим TeamsOnly на уровне клиента невозможно. Для завершения миграции в облако, которая включает удаление записей DNS, необходимо сначала переместить всех пользователей с локальной учетной записью Skype для бизнеса в облако, а затем выполнить действия, описанные в описании статьи Отключение гибридного `Move-CsUser` решения. [](/skypeforbusiness/hybrid/cloud-consolidation-disabling-hybrid)  `Grant-CsTeamsUpgradePolicy -PolicyName UpgradeToTeams`не будет работать на уровне клиента, если обнаружена запись DNS lyncdiscover, которая указывает на другое расположение, кроме Office 365.

- Убедитесь, что пользователи правильно синхронизированы с Azure AD с правильными Skype для бизнеса атрибутами. Все эти атрибуты являются префиксами с префиксами msRTCSIP-. Если пользователи не синхронизируются с Azure AD надлежащим образом, средства управления в Teams не смогут управлять этими пользователями. (Например, вы не сможете назначать политики Teams пользователям локальной сети, если не синхронизируете эти атрибуты надлежащим образом.) Дополнительные сведения см. в Подключение [Azure AD Teams и Skype для бизнеса.](/SkypeForBusiness/hybrid/configure-azure-ad-connect)

- Чтобы создать Новую версию TeamsOnly в гибридной организации, необходимо сначала включить пользователя в *локальной* Skype для бизнеса Server, а затем переместить его из локальной в облако с помощью Move-CsUser.  При создании пользователя в локальной сети все остальные оставшиеся Skype для бизнеса смогут перенапость созданному пользователю. После *того* как все пользователи будут перемещены через Интернет, их больше не нужно будет сначала включить в локальной сети.

- Если вы хотите, чтобы в клиенте Skype для бизнеса отображались уведомления для пользователей локальной сети, необходимо использовать TeamsUpgradePolicy в локальном средстве. Для локального пользователя релевантны только параметр NotifySfbUsers.  Пользователи локальной сети получают свой режим из интернет-экземпляров TeamsUpgradePolicy. См. примечания [в grant-CsTeamsUpgradePolicy.](/powershell/module/skype/grant-csteamsupgradepolicy?view=skype-ps) 

>[!NOTE]
> Все новые клиенты, созданные после 3 сентября 2019 г., создаются как клиенты TeamsOnly, если в организации еще не развернуто локальное Skype для бизнеса Server. Корпорация Майкрософт использует записи DNS для идентификации Skype для бизнеса Server организации. Если в вашей организации есть локальное Skype для бизнеса Server без общедоступных записей DNS, вам потребуется позвонить в службу поддержки Майкрософт, чтобы ваш новый клиент был более наднадеван.
