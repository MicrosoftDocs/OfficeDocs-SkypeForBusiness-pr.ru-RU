# <a name="audio-conferencing-with-direct-routing-for-gcc-high-and-dod"></a>Аудиоконференции с прямой маршрутизацией для GCC High и DoD

Аудиоконференция с прямой маршрутизацией для GCC High и DoD позволяет участникам присоединяться к собраниям Teams в организации GCC High или DoD с помощью телефонного устройства. Участники собрания могут использовать телефонное устройство для присоединения к собраниям Teams в таких сценариях, как ограничение подключения к Интернету или когда пользователи находятся в пути и не имеют доступа к Teams. Участники могут присоединиться к собранию, набрав для организации телефонный номер для телефонного звонка или набрав на него телефонный звонок.

Благодаря аудиоконференции с прямой маршрутизацией для GCC High и DoD ваша организация использует собственные номера телефонов в качестве номеров телефонов для телефонного звонка, а все телефонные звонки на телефонные устройства перенаправляются через прямую маршрутизацию. Чтобы включить службу, организациям необходимо настроить прямую маршрутику и настроить номера телефонов, которые могут использоваться в качестве номеров телефонов для телефонного звонка. Требования к прямой маршрутизаке отличаются от службы аудиоконференций, предлагаемых для организаций, не в которых используется GCC High, и без doD, где телефонные номера телефонов для телефонного звонка предоставлены корпорацией Майкрософт.

## <a name="deploy-audio-conferencing-with-direct-routing-for-gcc-high-and-dod"></a>Развертывание аудиоконференций с прямой маршрутизацией для GCC High и DoD

### <a name="step-1-get-audio-conferencing-with-direct-routing-for-gcc-high-or-dod-licenses"></a>Шаг 1. Аудиоконференция с прямой маршрутизацией для лицензий GCC High или DoD 

Чтобы использовать аудиоконференцию в GCC High или DoD, вашей организации и пользователям необходимо иметь лицензию на аудиоконференцию с прямой маршрутизацией. Ниже писались лицензии, необходимые для того, чтобы включить аудиоконференцию с прямой маршрутизацией для GCC High или DoD.

- GCC High: лицензия на аудиоконференцию с высоким клиентом GCC для вашей организации и лицензии на аудиоконференцию с GCC High для пользователей.

- DoD: аудиоконференция — лицензия клиента DoD для вашей организации и аудиоконференция — лицензии doD для пользователей.

Чтобы включить службу, требуются лицензия клиента и хотя бы одна лицензия пользователя. Службу нельзя включить только с лицензией клиента или только с пользовательскими лицензиями. Чтобы получить лицензии на обслуживание для клиента и пользователей в организации, обратитесь в свою команду учетной записи.

> [!IMPORTANT]
> Для пользователей нельзя включить аудиоконференцию с прямой маршрутизацией, пока не будут настроены номера телефонов для телефонного звонка и у пользователей есть рабочая панель набора номера в клиенте Teams. Мы не рекомендуем назначать пользователям аудиоконференцию с прямой маршрутизацией лицензий GCC High или DoD, пока не настроите номера телефонов для телефонного звонка, как описано в этой статье.

### <a name="step-3-set-up-dial-in-phone-numbers"></a>Шаг 3. Настройка номеров телефонов для телефонного звонка

Номера телефонов для телефонного звонка — это номера телефонов, связанные с мостом аудиоконференции. Эти номера используются участниками для присоединиться к собраниям, запланированным пользователями в вашей организации. Эти номера также включаются в приглашения на собрания для пользователей, планвших собрания в вашей организации и на странице "Поиск локального номера".

#### <a name="define-service-phone-numbers-in-your-tenant"></a>Определение номеров телефонов служб в клиенте

Для определения номеров телефонов службы в клиенте, которые можно использовать для перенаправки звонков в службу аудиоконференции с помощью Прямой маршрутики, можно использовать новый для csHybridTelephoneNumber PowerShell. 

  ```PowerShell
  New-csHybridTelephoneNumber -TelephoneNumber <Phone number in E.164 format>
  ```

Например:
  ```PowerShell
  New-csHybridTelephoneNumber -TelephoneNumber "+14250000000"
  ```

#### <a name="assign-the-service-phone-numbers-to-the-audio-conferencing-bridge-of-your-organization"></a>Назначение номеров телефонов службы мосту аудиоконференции вашей организации

Вы можете назначить номера телефонов службы мосту аудиоконференции вашей организации с помощью cmdlet Register-csOnlineDialInConferencingServiceNumber PowerShell.

  ```PowerShell
  Register-csOnlineDialInConferencingServiceNumber -identity <Telephone number in E.164 format> -BridgeId <Identity of the audio conferencing bridge>
  ```

Вы можете узнать ИД моста аудиоконференции с помощью Get-CsOnlineDialInConferencingBridge. Например:

  ```PowerShell
  $b= Get-CsOnlineDialInConferencingBridge
  Register-csOnlineDialInConferencingServiceNumber -identity 14257048060 -BridgeId $b.identity
  ```


### <a name="step-4-define-a-global-voice-routing-policy-to-enable-the-routing-of-outbound-calls-from-meetings"></a>Шаг 4. Определение глобальной политики маршрутинга голосовой почты для обеспечения маршрутинга исходяющих звонков из собраний

Маршрутка исходящие вызовы по STN из собраний, организованных пользователями в организации, определяется глобальной политикой маршрутизации голосовой почты организации. Если в вашей организации определена глобальная политика маршрутизации голосовой почты, убедитесь, что глобальная политика маршрутизации голосовой почты позволяет исходящие звонки по STN, которые должны инициироваться пользователями в организации. Если в вашей организации не определена глобальная политика маршрутизации голосовой почты, необходимо определить ее, чтобы включить маршрутику исходяющих звонков по STN из собраний, организованных пользователями в организации. Обратите внимание, что глобальная политика маршрутизации голосовой почты вашей организации также применяется к звонкам ЗВОНКОВ по ПС, сделанным пользователями в вашей организации. Если для пользователей в вашей организации включены звонков по ННР, убедитесь, что глобальная политика маршрутизации голосовой почты соответствует требованиям организации для обоих типов звонков. 

> [!NOTE]
> Location-Based маршруты недоступны в развертываниях Microsoft 365 Government Community Cloud (GCC) High или DoD. Включив аудиоконференцию, убедитесь, что для пользователей аудиоконференции в среде GCC High или DoD не включена Location-Based маршрутизацию.

#### <a name="defining-a-global-voice-routing-policy"></a>Определение глобальной политики маршрутинга голосовой почты

Глобальную политику маршрутизации голосовой почты можно определить, определив использование ННР, голосовой маршрут, политику маршрутизации голосовой маршрутизации и назначив новую политику голосовой маршрутизации в качестве глобальной политики маршрутизации голосовой почты в организации.

Ниже описано, как определить новую глобальную политику маршрутизации голосовой почты для организации без нее. Если в вашей организации уже определены политики маршрутизации голосовой связи, убедитесь, что следующая конфигурация не конфликтует с существующими политиками маршрутизации голосовой связи в организации.

Чтобы создать новую функцию использования ННР в удаленном сеансе PowerShell в Skype для бизнеса Online, используйте следующую команду:

  ```PowerShell
  Set-CsOnlinePstnUsage -Identity Global -Usage @{Add="International"}
  ```

Дополнительные сведения [см. в set-CsOnlinePstnUsage.](https://docs.microsoft.com/powershell/module/skype/set-csonlinepstnusage)

Чтобы создать новый голосовой маршрут, воспользуйтесь следующей командой:

  ```PowerShell
  New-CsOnlineVoiceRoute -Identity "International" -NumberPattern ".*" -OnlinePstnGatewayList sbc1.contoso.biz -OnlinePstnUsages "International"
  ```

При определении нового голосового пути для организации укажите один или несколько шлюзов ННР, определенных для организации в ходе настройки прямой маршрутизации. 

Шаблон номера определяет, какие звонки будут перенаправлены через указанный список шлюзов на основе номера конечного телефона звонка. В примере выше звонки в любую точку мира будут соответствовать голосовой маршрут. Если вы хотите ограничить набор телефонных номеров для собраний пользователей в вашей организации, вы можете изменить шаблон номера таким образом, чтобы голосовой маршрут совпадал только с шаблонами номеров в разрешенных назначениях. Обратите внимание, что если не существует голосовых маршрутов, которые соответствуют шаблону номеров телефонов назначения для данного звонка, перенаправление звонка не будет.

Дополнительные сведения [см. в new-CsOnlineVoiceRoute.](https://docs.microsoft.com/powershell/module/skype/new-csonlinevoiceroute)

Чтобы создать политику маршрутинга голосовой почты, используйте следующую команду:

  ```PowerShell
  New-CsOnlineVoiceRoutingPolicy "InternationalVoiceRoutingPolicy" -OnlinePstnUsages "International"
  ```

Если в политике голосовой маршрутки определено несколько вариантов использования ННР, они будут оцениваться в том порядке, в котором они определены. Рекомендуется определить использование телефонной сети общего пользования в порядке определения наиболее общих с точки зрения числовых шаблонов голосовых маршрутов, связанных с использованием ПС ДНР. Например, если для перенастройки звонков в США определена использование ЗВОНКОВ по ТЕЛЕФОННОЙ СЕТИ, а для перенастройки звонков в другие страны мира — еще одна, в политике перенастройки голосовой маршрутации должны быть указаны данные об использовании ЗВОНКОВ по ПС ННР для перенастройки звонков в другие страны мира.

Дополнительные сведения [см. в new-CsOnlineVoiceRoutingPolicy.](https://docs.microsoft.com/powershell/module/skype/new-csonlinevoiceroutingpolicy)

Чтобы назначить новый голосовой маршрут глобальной политике маршрутизации голосовой почты в организации, используйте следующую команду:

  ```PowerShell
  Grant-CsOnlineVoiceRoutingPolicy -PolicyName "InternationalVoiceRoutingPolicy" -Global
  ```

Дополнительные сведения [см. в окне Grant-CsOnlineVoiceRoutingPolicy.](https://docs.microsoft.com/powershell/module/skype/grant-csonlinevoiceroutingpolicy)

После определения глобальной политики маршрутизации голосовой связи все исходящие звонки, сделанные пользователями в организации, будут оцениваться с использованием голосовых маршрутов, связанных с использованием глобальной политики маршрутизации голосовой связи по ПС. Исходящие звонки будут маршрутизовы в соответствии с первым голосовым маршрутом, соответствующим шаблону набра номера телефона.

## <a name="audio-conferencing-capabilities-not-supported-in-audio-conferencing-with-direct-routing-for-gcc-high-and-dod"></a>Возможности аудиоконференции не поддерживаются в аудиоконференции с прямой маршрутизацией для GCC High и DoD

Ниже следующую возможность аудиоконференции не поддерживают аудиоконференцию с прямой маршрутизацией для GCC High и DoD:

- Уведомления о входе и выходе с помощью записи имени. Для аудиоконференций с прямой маршрутизацией уведомления о входе и выходе будут играть на собрании в тонах.

- Политики ограничений на исходящие вызовы для аудиоконференций. Элементы управления на уровне пользователей, ограничивающие исходящие звонки, не применимы к звонкам на собрания, перенаправимым с помощью прямой маршрутинга.

- Отключать использование бесплатных номеров для определенного организатора собраний. Элементы управления на уровне пользователей, ограничивающие использование бесплатных номеров для звонков в организации, не применимы к звонкам, перенаправимым с помощью прямой маршрутизации.

- Отправка уведомлений пользователям при изменении их параметров. Сообщения электронной почты с уведомлениями об аудиоконференции не поддерживаются для аудиоконференций с прямой маршрутизацией для GCC High и DoD.
