# <a name="audio-conferencing-with-direct-routing-for-gcc-high-and-dod"></a>Аудиоконференции с прямой маршрутизацией для GCC High и DoD

Аудиоконференция с прямой маршрутизацией для GCC High и DoD позволяет участникам присоединяться к собраниям Teams в организации GCC High или DoD с помощью телефонного устройства. Участники собрания могут использовать телефонное устройство для присоединения к собраниям Teams в таких случаях, как ограниченное подключение к Интернету или когда пользователи находятся в пути и не имеют доступа к Teams. Участники могут присоединяться к собраниям, набрав для вашей организации телефонный номер для телефонного звонка или набрав для собрания телефонный звонок.

При аудиоконференции с прямой маршрутизацией для GCC High и DoD в организации в качестве номеров телефонов для телефонного звонка используются собственные номера, а все телефонные звонки на телефонные устройства перенаправлены по прямой маршрутизации. Чтобы включить службу, организациям необходимо настроить прямую маршрутику и номера телефонов, которые могут использоваться в качестве номеров телефонов для телефонного звонка. Требования к прямой маршрутизационной маршрутике отличаются от службы аудиоконференций, которая предлагается для организаций, не в которых GCC высокая и не является doD, где телефонные номера для телефонного звонка предоставлены корпорацией Майкрософт.

## <a name="deploy-audio-conferencing-with-direct-routing-for-gcc-high-and-dod"></a>Развертывание аудиоконференций с прямой маршрутизацией для GCC высокая и doD

### <a name="step-1-get-audio-conferencing-with-direct-routing-for-gcc-high-or-dod-licenses"></a>Шаг 1. Получить аудиоконференцию с прямой маршрутизацией для лицензий GCC высокая или DoD 

Чтобы использовать аудиоконференцию в GCC высокая или DoD, вашей организации и пользователям в организации должна быть назначена лицензия на аудиоконференцию с прямой маршрутизацией. Вот лицензии, необходимые для того, чтобы включить аудиоконференцию с прямой маршрутизацией для GCC Высокая или DoD.

- GCC Высокая: аудиоконференция — GCC высокого клиента для организации и аудиоконференция — GCC лицензии для пользователей.

- DoD: аудиоконференция — лицензия клиента DoD для вашей организации и аудиоконференция — лицензии на DoD для пользователей.

Чтобы включить службу, требуются лицензия клиента и хотя бы одна лицензия пользователя. Вы не можете включить службу только с лицензией клиента или только с пользовательскими лицензиями. Чтобы получить лицензии на обслуживание для клиента и пользователей в организации, обратитесь в свою учетную запись.

> [!IMPORTANT]
> Для пользователей нельзя включить аудиоконференцию с прямой маршрутизацией, пока не будут настроены телефонные номера для телефонного звонка. Мы не рекомендуем назначать пользователям аудиоконференцию с прямой маршрутизацией для лицензий на GCC high или DoD, пока не настроите номера телефонов для телефонного звонка, как описано в этой статье.  Невыполнение инструкций может привести к тому, что панель набора номера будет полностью Teams клиенте.

### <a name="step-3-set-up-dial-in-phone-numbers"></a>Шаг 3. Настройка номеров телефонов для телефонного звонка

Номера телефонов для телефонного звонка — это номера телефонов, связанные с мостом аудиоконференции. Эти числа используются участниками для присоединиться к собраниям, запланированным пользователями в вашей организации. Эти номера также включаются в приглашения на собрания пользователей, план которых будут планировать собрания в вашей организации, и на странице "Поиск местного номера".

#### <a name="define-service-phone-numbers-in-your-tenant"></a>Определение номеров телефонов службы в клиенте

С помощью нового-csHybridTelephoneNumber PowerShell вы можете определить номера телефонов служб в клиенте, которые можно использовать для маршрутинга звонков в службу аудиоконференций с помощью прямой маршрутики. 

  ```PowerShell
  New-csHybridTelephoneNumber -TelephoneNumber <Phone number in E.164 format>
  ```

Например:
  ```PowerShell
  New-csHybridTelephoneNumber -TelephoneNumber "+14250000000"
  ```

#### <a name="assign-the-service-phone-numbers-to-the-audio-conferencing-bridge-of-your-organization"></a>Назначение номеров телефонов службы мосту аудиоконференций организации

Вы можете назначить номера телефонов службы мосту аудиоконференции вашей организации с помощью cmdlet Register-csOnlineDialInConferencingServiceNumber PowerShell.

  ```PowerShell
  Register-csOnlineDialInConferencingServiceNumber -identity <Telephone number in E.164 format> -BridgeId <Identity of the audio conferencing bridge>
  ```

Вы можете узнать ИД моста аудиоконференции с помощью Get-CsOnlineDialInConferencingBridge. Например:

  ```PowerShell
  $b= Get-CsOnlineDialInConferencingBridge
  Register-csOnlineDialInConferencingServiceNumber -identity 14257048060 -BridgeId $b.identity
  ```


### <a name="step-4-define-a-global-voice-routing-policy-to-enable-the-routing-of-outbound-calls-from-meetings"></a>Шаг 4. Определение глобальной политики маршрутинга голосовой почты, которая позволяет маршрутизовы исходящие звонки из собраний

Маршрутизировка исходящие вызовы через STN из собраний, организованных пользователями в организации, определяется глобальной политикой маршрутизации голосовой почты вашей организации. Если в вашей организации определена глобальная политика маршрутизации голосовой почты, убедитесь в том, что глобальная политика маршрутизации голосовой почты разрешает исходящие звонки на STN, которые должны инициироваться на собраниях, организованных пользователями в вашей организации. Если в вашей организации не определена глобальная политика маршрутизации голосовой почты, необходимо определить ее, чтобы включить маршрутику исходяющих звонков по STN из собраний, организованных пользователями в организации. Обратите внимание на то, что глобальная политика маршрутизации голосовой почты вашей организации также применяется к звонкам 1-го на один, сделанным пользователями вашей организации в ОКП. Если для пользователей в вашей организации включена возможность звонков по ОКП один-к-одному, убедитесь, что глобальная политика маршрутизации голосовой почты соответствует потребностям организации для обоих типов звонков. 

> [!NOTE]
> Location-Based маршруты недоступны в Microsoft 365 облако сообщества для государственных организаций (GCC) high или DoD. Включив аудиоконференцию, убедитесь, что для GCC Высокая или DoD не включена Location-Based аудиоконференцию.

#### <a name="defining-a-global-voice-routing-policy"></a>Определение глобальной политики маршрутинга голосовой почты

Глобальную политику маршрутизации голосовой почты можно определить, определив использование STN, голосовой маршрут, политику голосовой маршрутизации и назначив новую политику маршрутизации голосовой почты глобальной политикой маршрутизации голосовой почты вашей организации.

Ниже описано, как определить новую глобальную политику маршрутизации голосовой почты для организации без нее. Если в вашей организации уже определены политики маршрутизации голосовой связи, убедитесь, что следующая конфигурация не конфликтует с существующими политиками маршрутизации голосовой связи в организации.

Чтобы создать новую функцию использования ОКП в удаленном сеансе PowerShell в Skype для бизнеса Online, используйте следующую команду:

  ```PowerShell
  Set-CsOnlinePstnUsage -Identity Global -Usage @{Add="International"}
  ```

Дополнительные сведения [см. в настройках Set-CsOnlinePstnUsage.](/powershell/module/skype/set-csonlinepstnusage)

Чтобы создать новый голосовой маршрут, используйте следующую команду:

  ```PowerShell
  New-CsOnlineVoiceRoute -Identity "International" -NumberPattern ".*" -OnlinePstnGatewayList sbc1.contoso.biz -OnlinePstnUsages "International"
  ```

При определении нового голосового маршрута для организации укажите один или несколько шлюзов ПСДН, определенных для организации при настройке прямой маршрутизации. 

Шаблон номера определяет, какие звонки будут перенаправлены через указанный список шлюзов в зависимости от номера конечного телефона. В примере выше звонки в любую точку мира будут соответствовать голосовой маршруту. Если вы хотите ограничить набор телефонных номеров с собраний пользователей в организации, вы можете изменить шаблон номера таким образом, чтобы голосовой маршрут совпадал только с шаблонами номеров в разрешенных назначениях. Обратите внимание, что если не существует голосовых маршрутов, которые соответствуют шаблону номера конечного телефона для данного звонка, звонок не будет перенаправлен.

Дополнительные сведения см. [в new-CsOnlineVoiceRoute.](/powershell/module/skype/new-csonlinevoiceroute)

Чтобы создать политику маршрутинга голосовой почты, используйте следующую команду:

  ```PowerShell
  New-CsOnlineVoiceRoutingPolicy "InternationalVoiceRoutingPolicy" -OnlinePstnUsages "International"
  ```

Если в политике перенастройки голосовой маршрутии определено несколько функций ОКП, они оцениваются в том порядке, в котором они определены. Рекомендуется определить использование услуг STN в порядке определения наиболее общих с точки зрения шаблонов номеров голосовых маршрутов, связанных с использованием ОКП. Например, если для перенастройки звонков в США определено использование ЗВОНКОВ через ОКП, а для перенастройки звонков в любое другое место в мире — другое, в политику голосовой маршрутиации перед использованием ЗВОНКОВ по ННП следует у перечислить использование ЗВОНКОВ в США.

Дополнительные сведения см. [в new-CsOnlineVoiceRoutingPolicy.](/powershell/module/skype/new-csonlinevoiceroutingpolicy)

Чтобы назначить новый голосовой маршрут глобальной политике маршрутизации голосовой почты вашей организации, используйте следующую команду:

  ```PowerShell
  Grant-CsOnlineVoiceRoutingPolicy -PolicyName "InternationalVoiceRoutingPolicy" -Global
  ```

Дополнительные сведения [см. в окне Grant-CsOnlineVoiceRoutingPolicy.](/powershell/module/skype/grant-csonlinevoiceroutingpolicy)

После определения глобальной политики маршрутизации голосовой связи все исходящие звонки, исходящие из собраний, организованных пользователями в вашей организации, будут оцениваться в зависимости от голосовых маршрутов, связанных с использованием услуг ЗВОНКОВ по STN глобальной политики маршрутизации голосовой связи. Исходящие звонки будут маршрутизовы в соответствии с первым голосовым маршрутом, который соответствует шаблону номера набра номера.

## <a name="audio-conferencing-capabilities-not-supported-in-audio-conferencing-with-direct-routing-for-gcc-high-and-dod"></a>Возможности аудиоконференций, не поддерживаемые в аудиоконференциях с прямой маршрутизацией для GCC High и DoD

Ниже ются возможности аудиоконференций, которые не поддерживаются в аудиоконференциях с прямой маршрутизацией для GCC High and DoD:

- Уведомления о входе и выходе с помощью записи имени. Для аудиоконференации с прямой маршрутизацией уведомления о входе и выходе на собрание будут заимещены в тонах.

- Отключать использование бесплатных номеров для определенных организаторов собраний. Элементы управления на уровне пользователей, ограничивающие использование бесплатных номеров для звонков в организации, не применимы к звонкам, перенаправимым по прямой маршрутизации.

- Отправка уведомлений пользователям при изменении их параметров. Сообщения электронной почты с уведомлениями о аудиоконференции не поддерживаются для аудиоконференций с прямой маршрутизацией для GCC Высокая и DoD.
