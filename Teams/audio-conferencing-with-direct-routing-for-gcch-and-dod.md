# <a name="audio-conferencing-with-direct-routing-for-gcc-high-and-dod"></a>Аудиоконференции с прямой маршрутизацией для GCC High и DoD

Аудиоконференции с прямой маршрутизаций для GCC High и DoD позволяют участникам присоединяться к собраниям Teams в организации GCC High или DoD с помощью телефонного устройства. Участники собрания могут использовать телефонное устройство для присоединения к собраниям Teams в таких сценариях, как ограниченное подключение к Интернету или когда пользователи находятся в пути и не имеют доступа к Teams. Участники могут присоединяться к собраниям, используя телефонный номер для вашей организации с телефонным подключением или на телефонное устройство.

Благодаря аудиоконференции с прямой маршрутизаций для GCC High и DoD ваша организация использует собственные номера телефонов для телефонов с телефонным подключением, а все телефонные подключения к собраниям направляются через прямую маршрутизацию. Чтобы включить службу, организациям необходимо настроить прямую маршрутизацию и номера телефонов, которые можно использовать в качестве номеров телефонов с телефонным подключением. Требование использовать прямую маршрутизацию отличается от службы аудиоконференций, которая предлагается организациям без GCC High и сторонним организациям, где номера телефонов с телефонным подключением предоставляются корпорацией Майкрософт.

## <a name="deploy-audio-conferencing-with-direct-routing-for-gcc-high-and-dod"></a>Развертывание аудиоконференций с прямой маршрутизаций для GCC High и DoD

### <a name="step-1-get-audio-conferencing-with-direct-routing-for-gcc-high-or-dod-licenses"></a>Шаг 1. Получение аудиоконференций с помощью прямой маршрутизации для лицензий GCC High или DoD

Чтобы использовать аудиоконференции в GCC High или DoD, вашей организации и пользователям в организации должна быть назначена лицензия на прямую маршрутизацию. Ниже приведены лицензии, необходимые для включения аудиоконференций с прямой маршрутизаций для GCC High или DoD.

- GCC High: аудиоконференции — лицензия GCC High Tenant для вашей организации и аудиоконференции — лицензии GCC High для пользователей.

- DoD: аудиоконференции — лицензия клиента DoD для вашей организации и аудиоконференции — лицензии DoD для пользователей.

Для включения службы требуется лицензия клиента и по крайней мере одна пользователь. Вы не можете включить службу только с лицензией клиента или только с пользовательскими лицензиями. Чтобы получить лицензии на службу для клиента и пользователей в организации, обратитесь в службу поддержки учетных записей.

> [!IMPORTANT]
> Пользователи не могут быть включены для аудиоконференций с прямой маршрутизаций, пока не будут настроены номера телефонов с телефонным подключением. Мы рекомендуем не назначать пользователям аудиоконференции с прямой маршрутизаций для GCC High или DoD, пока не настроите номера телефонов с телефонным подключением, как описано в этой статье.  Если не выполнить это руководство, панель набора номера в клиенте Teams может полностью отсутствовать.

### <a name="step-3-set-up-dial-in-phone-numbers"></a>Шаг 3. Настройка номеров телефонов с телефонным вызовом

Номера телефонов с телефонным подключением — это номера телефонов, связанные с мостом аудиоконференций. Эти номера используются участниками для присоединения к собраниям, запланированным пользователями в организации. Эти номера также включаются в приглашения на собрания пользователей, планировщик которых планирует собрания в вашей организации, и на странице "Поиск локального номера".

#### <a name="define-service-phone-numbers-in-your-tenant"></a>Определение номеров телефонов службы в клиенте

Командлет PowerShell New-csHybridTelephoneNumber можно использовать для определения номеров телефонов службы в клиенте, которые можно использовать для маршрутизации вызовов к службе аудиоконференций через прямую маршрутизацию.

  ```PowerShell
  New-csHybridTelephoneNumber -TelephoneNumber <Phone number in E.164 format>
  ```

Например:

  ```PowerShell
  New-csHybridTelephoneNumber -TelephoneNumber "+14250000000"
  ```

#### <a name="assign-the-service-phone-numbers-to-the-audio-conferencing-bridge-of-your-organization"></a>Назначение номеров телефонов службы мосту аудиоконференций организации

Номера телефонов службы можно назначить мосту аудиоконференций организации с помощью командлета Register-csOnlineDialInConferencingServiceNumber PowerShell.

  ```PowerShell
  Register-csOnlineDialInConferencingServiceNumber -identity <Telephone number in E.164 format> -BridgeId <Identity of the audio conferencing bridge>
  ```

Идентификатор моста аудиоконференций можно просмотреть с помощью Get-CsOnlineDialInConferencingBridge. Например:

  ```PowerShell
  $b= Get-CsOnlineDialInConferencingBridge
  Register-csOnlineDialInConferencingServiceNumber -identity 14257048060 -BridgeId $b.identity
  ```

### <a name="step-4-define-a-global-voice-routing-policy-to-enable-the-routing-of-outbound-calls-from-meetings"></a>Шаг 4. Определение глобальной политики маршрутизации голосовой связи для включения маршрутизации исходящих звонков из собраний

Маршрутизация исходящих звонков, которые выполняются в ТСОП, из собраний, организованных пользователями в организации, определяется глобальной политикой маршрутизации голосовой связи вашей организации. Если в вашей организации определена глобальная политика маршрутизации голосовой связи, убедитесь, что глобальная политика маршрутизации голосовой связи разрешает исходящие вызовы ТСОП, которые должны инициироваться из собраний, организованных пользователями в организации. Если в вашей организации не определена глобальная политика маршрутизации голосовой связи, необходимо определить ее, чтобы включить маршрутизацию исходящих вызовов к ТСОП из собраний, организованных пользователями в организации. Обратите внимание, что глобальная политика маршрутизации голосовой связи в вашей организации также применяется к вызовам типа "один к одному", сделанным пользователями в вашей организации по ТСОП. Если для пользователей в организации включены вызовы ТСОП "один к одному", убедитесь, что глобальная политика маршрутизации голосовых вызовов соответствует потребностям вашей организации для обоих типов звонков.

> [!NOTE]
> Location-Based маршрутизации недоступны в развертываниях Microsoft 365 Government Community Cloud (GCC) High или DoD. При включении аудиоконференций убедитесь, что пользователи аудиоконференций в средах GCC High или DoD не включены для Location-Based маршрутизации.

#### <a name="defining-a-global-voice-routing-policy"></a>Определение глобальной политики маршрутизации голосовой связи

Глобальную политику маршрутизации голосовой связи можно определить, определив использование ТСОП, маршрут голосовой связи, политику маршрутизации голосовой связи и назначив новую политику маршрутизации голосовой связи в качестве глобальной политики маршрутизации голосовой связи вашей организации.

Ниже описано, как определить новую глобальную политику маршрутизации голосовой связи для организации без нее. Если в вашей организации уже определены политики маршрутизации голосовой связи, убедитесь, что приведенная ниже конфигурация не конфликтует с существующими политиками маршрутизации голосовой связи в вашей организации.

Чтобы создать новое использование ТСОП в удаленном сеансе PowerShell в Teams, используйте следующую команду:

  ```PowerShell
  Set-CsOnlinePstnUsage -Identity Global -Usage @{Add="International"}
  ```

Дополнительные сведения см [. в разделе Set-CsOnlinePstnUsage](/powershell/module/skype/set-csonlinepstnusage).

Чтобы создать маршрут голосовой связи, используйте следующую команду:

  ```PowerShell
  New-CsOnlineVoiceRoute -Identity "International" -NumberPattern ".*" -OnlinePstnGatewayList sbc1.contoso.biz -OnlinePstnUsages "International"
  ```

При определении нового голосового маршрута для организации укажите один или несколько сетевых шлюзов ТСОП, определенных для вашей организации во время настройки прямой маршрутизации.

Шаблон номера указывает, какие вызовы будут перенаправляться через указанный список шлюзов на основе целевого номера телефона звонка. В приведенном выше примере вызовы к любым местам назначения в мире будут соответствовать маршруту голосовой связи. Если вы хотите ограничить номера телефонов, которые могут направляться на собрания пользователей в вашей организации, вы можете изменить шаблон номера, чтобы голосовой маршрут соответствовал только шаблонам номеров разрешенных назначений. Обратите внимание, что при отсутствии голосовых маршрутов, соответствующих шаблону номера целевого номера телефона заданного звонка, вызов не будет перенаправлен.

Дополнительные сведения см. в [разделе New-CsOnlineVoiceRoute](/powershell/module/skype/new-csonlinevoiceroute).

Чтобы создать политику маршрутизации голосовой связи, используйте следующую команду:

  ```PowerShell
  New-CsOnlineVoiceRoutingPolicy "InternationalVoiceRoutingPolicy" -OnlinePstnUsages "International"
  ```

Если в политике маршрутизации голосовой связи определено несколько ТСОП, они будут оцениваться в том порядке, в котором они определены. Рекомендуется определить использование ТСОП в порядке, наиболее конкретном для более универсального с точки зрения числового шаблона голосовых маршрутов, связанных с использованием ТСОП. Например, если было определено использование ТСОП для маршрутизации вызовов в США, а другое использование ТСОП было определено для маршрутизации вызовов в любое другое расположение в мире, то использование ТСОП для вызовов к США должно быть указано в политике маршрутизации голосовой связи перед использованием ТСОП для маршрутизации вызовов в любое другое расположение в мире.

Дополнительные сведения см. [в разделе New-CsOnlineVoiceRoutingPolicy](/powershell/module/skype/new-csonlinevoiceroutingpolicy).

Чтобы назначить новый маршрут голосовой связи глобальной политике маршрутизации голосовой связи вашей организации, используйте следующую команду:

  ```PowerShell
  Grant-CsOnlineVoiceRoutingPolicy -PolicyName "InternationalVoiceRoutingPolicy" -Global
  ```

Дополнительные сведения см [. в разделе Grant-CsOnlineVoiceRoutingPolicy](/powershell/module/skype/grant-csonlinevoiceroutingpolicy).

После определения глобальной политики маршрутизации голосовой связи любые исходящие звонки из собраний, организованных пользователями в вашей организации, будут оцениваться с учетом маршрутов голосовой связи, связанных с использованием ТСОП глобальной политики маршрутизации голосовой связи. Исходящие звонки будут направляться в соответствии с первым голосовым маршрутом, который соответствует шаблону номера на телефоне.

## <a name="audio-conferencing-capabilities-not-supported-in-audio-conferencing-with-direct-routing-for-gcc-high-and-dod"></a>Возможности аудиоконференций не поддерживаются в аудиоконференции с прямой маршрутизаций для GCC High и DoD

Ниже приведены возможности аудиоконференций, которые не поддерживаются в аудиоконференции с прямой маршрутизаций для GCC High и DoD:

- Уведомления о входе и выходе с помощью записи имени. Для аудиоконференций с прямой маршрутизаций уведомления о входе и выходе воспроизводится на собрании в виде звукового сигнала.

- Отключите использование бесплатных номеров для организаторов собраний. Элементы управления уровня пользователя, ограничивающие использование бесплатных номеров для присоединения к собраниям вашей организации, неприменимы к звонкам, перенаправленным через прямую маршрутизацию.

- Отправка уведомлений по электронной почте пользователям при изменении их параметров. Сообщения электронной почты с уведомлениями о аудиоконференции не поддерживаются для аудиоконференций с прямой маршрутизаций для GCC High и DoD.
