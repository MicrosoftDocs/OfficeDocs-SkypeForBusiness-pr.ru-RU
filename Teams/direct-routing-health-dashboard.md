# <a name="health-dashboard-for-direct-routing"></a>Панель мониторинга состояния для прямой маршрутки

Информационная панель health for Direct Routing позволяет отслеживать соединение между контроллером границы сеанса (SBC) и интерфейсом прямой маршрутки.  С помощью панели мониторинга здравоохранения можно отслеживать сведения о SBC, телефонной службе и сетевых параметрах между SBC и интерфейсом прямой маршрутки. Эти сведения помогут вам выявить проблемы, в том числе причину переключеки звонков. Например, SBC может прекратить отправку звонков, если истек срок действия сертификата на SBC или имеются проблемы с сетью. Узнайте о [ролях администраторов,](using-admin-roles.md) чтобы узнать, у кого есть доступ к панели мониторинга состояния.

На информационной панели здравоохранения отслеживается два уровня информации:

- Общее состояние подключенных SBCs
- Подробные сведения о подключенных SBCs

Вы можете просмотреть панель мониторинга здравоохранения в Центре администрирования Microsoft Teams и Skype для бизнеса.

## <a name="overall-health"></a>Общее состояние

На панели мониторинга состояния можно найти следующие сведения, связанные с общим состоянием подключенных SBCs:

 ![Статистика панели мониторинга здравоохранения](media/direct-routing-dashboard-stats1.png)

- **Сводка по прямой маршрутике** — общее количество SBCs, зарегистрированных в системе. Регистрация означает, что администратор клиента добавил SBC с помощью команды New-CsOnlinePSTNGateway клиента. Если SBC был добавлен в PowerShell, но не подключен, панель мониторинга состояния показывает его как неработоспособное.

- **SBC** — FQDN для сопряженного SBC.

- **Коэффициент эффективности сети (NER). NER** измеряет возможность сети доставки звонков, измерив количество звонков в сравнении с числом звонков, доставленных получателю.  

   NER измеряет возможности сетей для доставки звонков на конечный терминал, исключая действия пользователей, которые вызывают отклонение звонков.  Если получатель отклонил звонок или отправил его на голосовую почту, звонок считается успешной доставкой. Это означает, что ответное сообщение, сигнал занятости или звонок без ответа считаются успешными звонками.
  
   Например, предположим, что прямой маршрутизации был отправлен звонок на SBC, а код SBC возвращает SIP "504 Server Time-out - The server attempted to access another server in attempting to process the request and did not receive a prompt response". Этот ответ указывает на проблему на стороне SBC, и в этом случае NER на информационной панели здравоохранения для этого SBC снизится.
  
   Так как действия, которые вы принимаете, зависят от количества звонков, на информационной панели health можно узнать, сколько звонков было проанализировано для вычисления параметра. Если количество звонков меньше 100, то NER может быть довольно низким, но по-прежнему обычным.

   Для вычисления NER используется формула:

   NER = 100 x (answered calls + User Busy + Ring no Answer + Terminal Reject Ветвей)/Total Calls

- **Средняя продолжительность звонков:** сведения о средней длительности звонков помогут вам контролировать качество звонков. Средняя длительность звонка по ПС 1:1 составляет от четырех до пяти минут.  Однако для каждой компании это среднее значение может различаться.  Корпорация Майкрософт рекомендует определить базовый план средней длительности звонка для вашей организации. Если этот параметр значительно ниже базового плана, это может означать, что у пользователей возникли проблемы с качеством и надежностью звонка и вызов идет раньше обычного. Если вы начинаете видеть очень низкую среднюю длительность звонка, например 15 секунд, возможно, вызов выполняется из-за низкой надежности работы службы.

   Так как действия, которые вы принимаете, зависят от количества звонков, на информационной панели health можно узнать, сколько звонков было проанализировано для вычисления параметра.

- **Состояние подключения TLS:** подключение TLS (TLS— безопасность уровня транспорта) показывает состояние подключений TLS между Direct Routing и SBC. Панель мониторинга состояния также анализирует дату окончания срока действия сертификата и предупреждает о том, что срок действия сертификата истекает в течение 30 дней, чтобы администраторы могли продлить его до нарушения службы.

   Щелкнув предупреждающее сообщение, вы увидите подробное описание проблемы во всплывающее окно справа и рекомендации по ее устранению.

- **Состояние параметров SIP —** по умолчанию SBC каждую минуту отправляет параметры сообщений. Эта конфигурация может отличаться для разных поставщиков SBC. Прямая маршрутная маршрутия предупредит вас, если параметры SIP не отправлены или не настроены. Дополнительные сведения о мониторинге параметров SIP и условиях, когда SBC можно пометить как нефункциональный, см. в статью "Мониторинг и устранение неполадок с прямой [маршрутией".](direct-routing-monitor-and-troubleshoot.md)

- Подробное состояние параметров **SIP.** Помимо того, что имеется проблема с потоком параметров SIP, панель мониторинга состояния также содержит подробное описание ошибок. Чтобы получить доступ к описанию, щелкните сообщение "Предупреждение". Во всплываемом окне справа покажите подробное описание ошибки.

   Возможные значения для сообщений о состоянии параметров SIP:

    - Активна — активна SBC: служба прямой маршрутки Майкрософт видит параметры, которые вытекают из регулярного интервала.

    - Предупреждение, параметры SIP не доступны: в базе данных есть контроллер границы сеанса (его создал администратор с помощью команды New-CsOnlinePSTNGateway). Она настроена для отправки параметров SIP, но служба прямой маршрутинга никогда не видели параметры SIP, исходяющие из этого SBC.

    - Предупреждение, сообщения SIP не настроены: мониторинг магистрали с помощью параметров SIP не включен. Система звонков Майкрософт использует параметры SIP и функцию мониторинга безопасности TLS для определения состояния подключенных граничных контроллеров сеанса (SBCs) на уровне приложения. Если сделать это на уровне сети (с помощью PING), но срок действия сертификата истек или Стек SIP не работает. Чтобы на ранней стадии выявлять такие проблемы, корпорация Майкрософт рекомендует отправлять параметры SIP. Настройте параметры отправки SIP в документации производителя SBC.

- **Пропускная способность** для одновременного звонка: вы можете указать ограничение на количество одновременно обрабатываемых SBC звонков с помощью команды New- или Set-CsOnlinePSTNGateway с параметром -MaxConcurrentSessions. Этот параметр подсчитывает количество звонков, отправленных или полученных с помощью прямой маршрутии с помощью определенной SBC, и сравнивает его с установленным ограничением. Примечание. Если SBC также обрабатывает вызовы на разные УАЦ-связи, этот номер не будет фактически одновременно обрабатываться.

## <a name="detailed-information-for-each-sbc"></a>Подробные сведения для каждого SBC

Вы также можете просмотреть подробные сведения о конкретном SBC, как показано на снимке экрана:

![Сведения о SBC панели мониторинга здравоохранения](media/direct-routing-dashboard-SBC-detail1.png)

В подробном представлении показаны следующие дополнительные параметры:

- **Состояние подключения TLS —** это та же метрика, что и на странице "Общее состояние";

- **Последнее состояние подключения TLS** — время, когда SBC подключался по TLS к службе Direct Routing;

- **Состояние параметров SIP** — та же метрика, что и на странице "Общее состояние".

- **Параметры SIP последний раз проверялись** — время последней проверки параметров SIP.

- **Состояние SBC —** общее состояние SBC на основе всех отслеживаемого параметров.

- **Одновременное звонок**— показывает, сколько звонков обрабатывает SBC одновременно. Эти сведения полезны для предсказания количества необходимых одновременно каналов и для прогнозирования тенденции. Вы можете сдвиньте данные по количеству дней и направлению звонка (входящие, исходящие и все потоки).

- **Параметры сети** — все параметры сети отмеряются от прямого интерфейса маршрутинга до граничного геймпада сеанса. Сведения о рекомендуемых значениях см. в подготовке сети организации для [Microsoft Teams](./prepare-network.md)и рекомендуемых значениях в microsoft Edge.

   - Jitter — это миллисекундный показатель разброса времени распространения по сети, вычисляемого между двумя конечными точками с помощью RTCP (протокола управления RTP).

   - Packet Loss — это показатель пакета, который не удалось получить; вычисляется между двумя конечными точками.

   - Задержка (или время кругового пути) — это время, необходимое для сигнала, а также время его подтверждения. Это время задержки представляет собой время распространения между двумя точками сигнала.

   Вы можете сдвиньте данные по количеству дней и направлению звонка (входящие, исходящие и все потоки).

**Отношение эффективности сети** — это тот же параметр, который отображается на панели мониторинга "Общее состояние", но с возможностью анализа данных по рядам времени или направлению звонка.
