# <a name="manage-emergency-calling"></a>Управление экстренными звонками

В этой статье&mdash;описываются основные понятия, которые необходимо знать для управления экстренными вызовами, в том числе сведения об адресах для экстренного реагирования, динамических адресах экстренных служб и маршрутизации экстренных вызовов. В этой статье используется следующую терминологию:

- **Адрес для экстренного** реагирования — адрес для&mdash;физического или улицного адреса организации.

  Например, адрес *12345 North Main Street, Redmond, WA 98052* используется для маршрутизации экстренных вызовов в соответствующие диспетчерские центры и для помощи в поиске вызывающего экстренных служб.

- **Место** — обычно это этаж, здание, wing или номер офиса. Место связано с адресом для экстренного реагирования, чтобы указать более точное расположение в здании. У вас может быть неограниченное количество мест, связанных с адресом для экстренного реагирования. Например, если в организации несколько зданий, может потребоваться включить сведения о месте для каждого здания и каждого этажа в каждом здании.

- **Местоположение для экстренного** реагирования — это общедоступный адрес&mdash;с необязательным местом. Если в вашей организации несколько физических расположений, скорее всего, потребуется несколько экстренных расположений.

  При создании адреса для экстренного реагирования для этого адреса автоматически создается уникальный идентификатор расположения. Например,&mdash;если вы добавляете место в адрес для экстренного реагирования,&mdash;то при добавлении этажа в адрес здания создается идентификатор расположения для сочетания адреса и места для экстренного реагирования.  В этом примере будет два идентификатора расположения: один для адреса для граждан. один для присоединенного адреса и связанного с ним места.

  При назначении местоположения для экстренного реагирования пользователю или сайту это уникальный идентификатор расположения, связанный с пользователем или сайтом.

- **Зарегистрированный адрес** — адрес для экстренного реагирования, назначенный каждому пользователю. Зарегистрированный адрес иногда называют статическим адресом для экстренного реагирования или адресом записи. (В настоящее время зарегистрированные адреса не поддерживаются для прямой маршрутизации. Проверьте наличие обновлений в ближайшее время.)

>[!Note]
>Существуют некоторые различия в управлении экстренными звонками в зависимости от того, используете ли вы планы звонков Майкрософт, operator Connect, Teams Phone Mobile или прямую маршрутизацию для подключения к [ТСОП](pstn-connectivity.md). Эти рекомендации описаны в этой статье.

## <a name="emergency-address-validation"></a>Проверка адреса для экстренного реагирования

Чтобы назначить адрес для экстренного реагирования пользователю или сетевому идентификатору, необходимо убедиться, что адрес для экстренного реагирования помечен как "проверенный". Проверка адреса гарантирует, что адрес является допустимым и его нельзя изменить после назначения.

Если вы определяете адрес для экстренного реагирования с помощью функции поиска карты адресов в Центре администрирования Teams, адрес автоматически помечается как проверенный. Так как вы не можете&mdash;изменить проверенный адрес для экстренного реагирования при изменении формата или представления адреса, необходимо создать новый адрес с обновленным форматом.

## <a name="emergency-address-geo-codes"></a>Географические коды адресов для экстренного реагирования

С каждым адресом для экстренного реагирования может быть связан географический код (широта и долгота). Эти географические коды используются в некоторых странах для маршрутизации экстренных вызовов с динамическими расположениями.

Если вы определяете адрес для экстренного реагирования с помощью функции поиска карты адресов в Центре администрирования Teams, географический код автоматически сопоставляется с адресом для экстренного реагирования. Вы также можете связать географические коды с адресом, если вы определяете адрес с помощью PowerShell.

Корпорация Майкрософт рекомендует создавать адреса для экстренного реагирования с помощью функции поиска карт в Центре администрирования Teams, которая гарантирует форматирование, проверку адресов и соответствующие географические коды.

>[!Important]
>Чтобы назначить местоположение для экстренного реагирования сетевому идентификатору для динамических экстренных вызовов, адрес для экстренного реагирования должен содержать соответствующий географический код.

## <a name="considerations-for-calling-plans"></a>Рекомендации по тарифным планам

В следующих разделах описывается управление экстренными звонками для пользователей плана звонков Майкрософт. Чтобы узнать, являются ли планы звонков Майкрософт правильным решением для вашего бизнеса, ознакомьтесь с вариантами подключения [по ТСОП](pstn-connectivity.md).

### <a name="emergency-call-enablement-for-calling-plans"></a>Включение экстренных вызовов для планов звонков

Каждый пользователь плана звонков автоматически включает экстренные вызовы и должен иметь зарегистрированный адрес для экстренного реагирования, связанный с назначенным им телефонным номером.

Если расположение связано с телефонным номером, зависит от страны или региона:

- Например, в США и Канаде требуется местоположение для экстренного реагирования, когда пользователю назначается номер.

- &mdash;Для других стран, таких как Европа, Ближний Восток и Африка (EMEA),&mdash; требуется местоположение для экстренного реагирования, когда вы получаете номер телефона из Microsoft 365 или когда он передается от другого поставщика услуг или оператора.

### <a name="dynamic-emergency-calling-for-calling-plans"></a>Динамические экстренные вызовы для планов звонков

Динамические экстренные вызовы для планов звонков обеспечивают возможность настройки и маршрутизации экстренных вызовов в зависимости от текущего расположения клиента Teams. Возможность автоматической маршрутизации в соответствующую общедоступную точку ответа (PSAP) или уведомления сотрудников службы безопасности зависит от страны использования пользователя Teams.

Динамическое расположение для маршрутизации экстренных вызовов поддерживается в США ниже.

- Если клиент Teams для плана звонков США динамически получает адрес для экстренного реагирования в США, этот адрес используется для экстренной маршрутизации, а не зарегистрированного адреса, и вызов будет автоматически перенаправлен в PSAP в обслуживаемой области адреса.

- Если клиент Teams для США плана звонков не получает динамический адрес для экстренного реагирования в США, зарегистрированный адрес для экстренного реагирования используется для экранирования справки и маршрутизации звонка. Однако вызов будет отсвечен, чтобы определить, требуется ли обновленный адрес перед подключением вызывающего объекта к соответствующему PSAP.

Динамическое расположение для маршрутизации экстренных вызовов поддерживается в Канаде так же, как и в США, за исключением следующего исключения: все экстренные вызовы будут отсвечены на национальной основе перед передачей в PSAP.

Дополнительные сведения см. в [статье "Планирование и настройка динамических экстренных вызовов"](configure-dynamic-emergency-calling.md).

### <a name="emergency-call-routing-for-calling-plans"></a>Маршрутизация экстренных вызовов для планов звонков

Когда пользователь плана звонков Teams назовет номер для экстренного реагирования, способ маршрутизации звонка в PSAP зависит от следующего:

- Определяет ли клиент Teams динамически адрес для экстренного реагирования.

- Указывает, является ли адрес для экстренного реагирования зарегистрированным адресом, связанным с номером телефона пользователя.

- Сеть экстренных вызовов в этой стране.

Например:

**В США:**

- Если клиент Teams находится в определенном клиентом динамическом расположении для экстренного реагирования, экстренные вызовы от этого клиента автоматически направляются в PSAP, обслуживающий это географическое расположение.

- Если клиент Teams не находится в динамическом расположении для экстренного реагирования, определенном клиентом, экстренные вызовы от этого клиента отображаются в центре обработки вызовов национального центра обработки вызовов перед передачей вызова в PSAP, обслуживающий это географическое расположение.

- Если вызывающий экстренный вызов не может обновить свое местоположение для экстренного реагирования в центре проверки, вызов будет передан в PSAP, обслуживающий зарегистрированный адрес вызывающего абонента.

**В Канаде, Ирландии** и Соединенном Королевстве экстренные вызовы сначала отображаются для определения текущего расположения пользователя перед подключением вызова к соответствующему центру диспетчеризации.

**Во Франции, Германии** и Испании экстренные вызовы направляются непосредственно в PSAP, обслуживающий адрес для экстренного реагирования, связанный с номером, независимо от расположения вызывающего абонента.

**В Нидерланды экстренные** вызовы направляются непосредственно в PSAP для кода номера в локальной области независимо от расположения вызывающего абонента.

**В Австралии** адреса для экстренного реагирования настраиваются и маршрутизируются партнером-оператором.

**В Японии** экстренные вызовы не поддерживаются.

- Если клиент Teams для пользователя США динамически получает адрес для экстренного реагирования в США, этот адрес используется для маршрутизации экстренных служб вместо зарегистрированного адреса, и вызов будет автоматически перенаправлен в PSAP в обслуживаемой области адреса.

- Если клиент Teams для США не получает динамический адрес для экстренного реагирования в США, зарегистрированный адрес для экстренного реагирования используется для экранирования справки и маршрутизации звонка. Однако вызов будет отсвечен, чтобы определить, требуется ли обновленный адрес перед подключением вызывающего объекта к соответствующему PSAP.

Дополнительные сведения см. в [статье "Планирование и настройка динамических экстренных вызовов"](configure-dynamic-emergency-calling.md).

### <a name="emergency-call-routing-for-operator-connect"></a>Маршрутизация экстренных вызовов для Operator Connect

Когда пользователь Teams Operator Connect назовет номер для экстренного реагирования, способ перенаправления вызова в PSAP зависит от следующего:

- Если клиент Teams находится в определенном клиентом динамическом расположении для экстренного реагирования:

  - В Канаде все экстренные вызовы будут отсвечены национальным центром обработки вызовов перед передачей вызова в PSAP, обслуживающий это географическое расположение.

- Если клиент Teams для пользователя США динамически получает адрес для экстренного реагирования в США, этот адрес используется для маршрутизации экстренных служб вместо зарегистрированного адреса, и вызов будет автоматически перенаправлен в PSAP в обслуживаемой области адреса.

- Если клиент Teams для США не получает динамический адрес для экстренного реагирования в США, зарегистрированный адрес для экстренного реагирования используется для экранирования справки и маршрутизации звонка. Однако вызов будет отсвечен, чтобы определить, требуется ли обновленный адрес перед подключением вызывающего объекта к соответствующему PSAP.

Динамическое расположение для маршрутизации экстренных вызовов поддерживается в Канаде так же, как в США со следующими исключениями: все экстренные вызовы будут отсвечены на национальной основе перед передачей в PSAP.

Дополнительные сведения см. в [статье "Планирование и настройка динамических экстренных вызовов"](configure-dynamic-emergency-calling.md).

### <a name="emergency-call-routing-for-teams-phone-mobile"></a>Маршрутизация экстренных вызовов для Teams Phone Mobile

Когда пользователь Teams Teams Phone Mobile назовет номер для экстренного реагирования с помощью клиента Microsoft Teams, способ маршрутизации звонка в PSAP зависит от следующего:

- Определяет ли клиент Teams динамически адрес для экстренного реагирования.

- Указывает, является ли адрес для экстренного реагирования зарегистрированным адресом, связанным с номером телефона пользователя.

- Сеть экстренных вызовов в этой стране.

- В США и Канаде динамическая маршрутизация является частью службы оператора. Вам не нужно приобретать эту службу у другого поставщика услуг.

- Если клиент Teams находится в определенном клиентом динамическом расположении для экстренного реагирования:
  - В США, экстренные вызовы от этого клиента автоматически направляются в PSAP, обслуживающий это географическое расположение.
  - В Канаде все экстренные вызовы будут отсвечены национальным центром обработки вызовов перед передачей вызова в PSAP, обслуживающий это географическое расположение.

- Если клиент Teams не находится в динамическом расположении для экстренного реагирования, определенном клиентом, экстренные вызовы от этого клиента отображаются в центре обработки вызовов национального центра обработки вызовов перед передачей вызова в PSAP, обслуживающий это географическое расположение.

- Если вызывающий экстренный вызов не может обновить свое местоположение для экстренного реагирования в центре проверки, вызов будет передан в PSAP, обслуживающий зарегистрированный адрес вызывающего абонента.

Мобильный оператор управляет всеми экстренными вызовами, которые выполняются с помощью собственного набора номера смартфона SIM-Enabled, и может использовать несколько технологий, чтобы приблизительно определить местоположение для экстренного реагирования, например географические координаты или ячейки, которые обрабатывают звонок и т. д. За дополнительными сведениями обратитесь к оператору.


## <a name="considerations-for-direct-routing"></a>Рекомендации по прямой маршрутизации

В следующих разделах описывается управление экстренными вызовами для пользователей прямой маршрутизации. Чтобы узнать, является ли прямая маршрутизация правильным решением для вашего бизнеса, см. параметры подключения [по ТСОП](pstn-connectivity.md).

### <a name="emergency-call-enablement-for-direct-routing"></a>Включение экстренных вызовов для прямой маршрутизации

Для прямой маршрутизации необходимо определить политики экстренных вызовов для пользователей с помощью политики маршрутизации экстренных вызовов [Teams](manage-emergency-call-routing-policies.md) , чтобы определить номера экстренных служб и связанное с ними назначение маршрутизации. (В настоящее время зарегистрированные расположения для экстренного реагирования не поддерживаются для пользователей прямой маршрутизации.)

Политику маршрутизации экстренных вызовов можно назначить учетной записи пользователя прямой маршрутизации, сетевому сайту или обоим. Когда клиент Teams запускает или изменяет сетевое подключение, Teams выполняет поиск сетевого сайта, на котором находится клиент, следующим образом:

- Если с сайтом связана политика маршрутизации экстренных вызовов, политика сайта используется для настройки экстренных вызовов.

- Если с сайтом не связана политика маршрутизации экстренных вызовов, если клиент подключен на неопределенном сайте или если номер набора не соответствует ни одному из номеров экстренных служб, определенных в политике маршрутизации экстренных вызовов, связанной с сайтом, для настройки экстренных вызовов используется политика маршрутизации экстренных вызовов, связанная с учетной записью пользователя.

- Если клиенту Teams не удается получить политику маршрутизации экстренных вызовов, пользователь не может вызывать экстренные вызовы.

### <a name="dynamic-emergency-calling-for-direct-routing"></a>Динамические экстренные вызовы для прямой маршрутизации

Динамические экстренные вызовы для прямой маршрутизации обеспечивают возможность настройки и маршрутизации экстренных вызовов в зависимости от текущего расположения клиента Teams. Возможность автоматической маршрутизации в соответствующую общедоступную точку ответа (PSAP) или уведомления сотрудников службы безопасности зависит от страны использования пользователя Teams.

Для пользователей прямой маршрутизации динамическое расположение для маршрутизации экстренных вызовов поддерживается только в США следующим образом:

- Если клиент Teams для пользователя прямой маршрутизации США динамически получает адрес для экстренного реагирования в США, этот адрес используется для маршрутизации в экстренных случаях и вызов будет автоматически перенаправлен в PSAP в обслуживаемой области адреса.

- Если клиент Teams для пользователя прямой маршрутизации США не получает динамический адрес для экстренного реагирования в США, вызов будет отсвечен, чтобы определить, требуется ли обновленный адрес перед подключением вызывающего абонента к соответствующему PSAP.

Динамическое расположение для маршрутизации экстренных вызовов поддерживается в Канаде так же, как и в США, за исключением следующего исключения: все экстренные вызовы будут отсвечены на национальной основе перед передачей в PSAP.

Дополнительные сведения см. в [разделе "Настройка динамических экстренных вызовов"](configure-dynamic-emergency-calling.md).

### <a name="emergency-call-routing-for-direct-routing"></a>Маршрутизация экстренных вызовов для прямой маршрутизации

Политика маршрутизации экстренных вызовов для прямой маршрутизации ссылается на сетевое использование ТСОП, которое должно иметь соответствующую конфигурацию прямой маршрутизации для правильной маршрутизации экстренных вызовов на соответствующие шлюзы ТСОП. В частности, необходимо убедиться в наличии OnlineVoiceRoute для строки набора экстренных служб. Дополнительные сведения см. в [разделе "Настройка прямой маршрутизации"](direct-routing-configure.md).

> [!NOTE]
> Клиенты Teams больше не добавляют знак "+" перед номерами экстренных служб. То есть +911. Следовательно, экстренные вызовы Teams больше не будут отправлять "+" перед номером 911. Убедитесь, что шаблоны маршрутов голосовой связи отражают это изменение.

Возможность динамической маршрутизации экстренных вызовов для пользователей прямой маршрутизации зависит от сети экстренных вызовов в конкретной стране. Доступно два решения:

- [Поставщики экстренных служб маршрутизации (только для США)](#emergency-routing-service-providers)
- [Приложения для идентификации местоположения для экстренного реагирования](#emergency-location-identification-number-applications)

#### <a name="emergency-routing-service-providers"></a>Поставщики экстренных служб маршрутизации

В США есть множество сертифицированных поставщиков услуг аварийной маршрутизации (ERSP), которые могут автоматически маршрутизировать экстренные вызовы в зависимости от расположения вызывающего абонента.

- Если поставщик службы экстренной маршрутизации интегрирован в развертывание прямой маршрутизации, экстренные вызовы с динамически полученным расположением будут автоматически направляться в точку ответа общедоступной безопасности (PSAP), обслуживающий это расположение.

- Экстренные вызовы без динамически полученных расположений сначала отображаются для определения текущего расположения пользователя перед подключением вызова к соответствующему центру диспетчеризации на основе обновленного расположения.

Дополнительные сведения см. в [разделе "Пограничные контроллеры сеансов", сертифицированные для прямой маршрутизации](direct-routing-border-controllers.md).

#### <a name="emergency-location-identification-number-applications"></a>Приложения для идентификации местоположения для экстренного реагирования

Пограничные контроллеры сеансов (SBC) могут включать приложения ELIN. Если приложение SBC ELIN интегрировано в развертывание прямой маршрутизации, необходимо настроить адреса для экстренного реагирования и связанные с ним телефонные номера в приложении ELIN, а затем отправить записи ELIN в базу данных экстренных вызовов в соответствующем ТСОП. Расположения для экстренного реагирования Teams с идентификатором ELIN должны соответствовать расположениям в приложении ELIN.

Когда экстренный вызов с динамически полученным расположением направляется в соответствующий SBC, приложение ELIN:

- Анализирует местоположение вызывающего абонента в экстренных службах.
- Соответствует расположению записи ELIN.
- Заменяет номер звонющего в экстренные службы номером телефона ELIN.
- Направляет вызов в PSAP, обслуживающий это расположение, а затем диспетчеры получают расположение из отправленной записи ELIN.

После обратного вызова на номер для экстренного реагирования приложение ELIN выполнит обратное подстановку номера исходного вызывающего экстренную службу.

Дополнительные сведения см. в [разделе "Пограничные контроллеры сеансов", сертифицированные для прямой маршрутизации](direct-routing-border-controllers.md).

## <a name="security-desk-notification"></a>Уведомление службы безопасности

Уведомления службы безопасности доступны в планах звонков Майкрософт, Operator Connect и прямой маршрутизации.

Вы используете политику экстренных вызовов Teams (TeamsEmergencyCallingPolicy), чтобы настроить, кто должен получать уведомления во время экстренных звонков и как они получают уведомления: только в чате, с конференцией и отключены или конференц-связи и отключены, но с возможностью включения звука. Вы также можете указать внешний номер ТСОП пользователя или группы для вызова и присоединения к экстренным вызовам. Обратите внимание, что стороне ТСОП не разрешено включить включение.

Политику экстренных вызовов можно предоставить учетной записи пользователя Teams, назначенной сетевому сайту, или и то, и другое.  Когда клиент Teams запускает или изменяет сетевое подключение, Teams выполняет поиск сетевого сайта, на котором находится клиент:

- Если политика экстренных вызовов связана с сетевым сайтом, политика сайта используется для настройки уведомлений службы безопасности.

- Если с сайтом не связана политика экстренных вызовов или клиент подключен на неопределенном сайте, то политика экстренных вызовов, связанная с учетной записью пользователя, используется для настройки уведомлений службы безопасности.

- Если клиенту Teams не удается получить политику экстренных вызовов, пользователь не сможет получать уведомления службы безопасности.

Во время экстренных вызовов служба безопасности с конференцией в звонит, а взаимодействие пользователя службы безопасности контролируется на основе политики экстренных вызовов Teams. Групповой чат запускается с каждым участником службы безопасности, а расположение вызывающего экстренных служб используется через уведомление о важном сообщении.  Если в рамках политики настроен параметр конференции, каждый пользователь службы безопасности также вызывается в рамках конференции.

### <a name="custom-emergency-disclaimer"></a>Заявление об отказе от пользовательских экстренных служб

Администраторы могут добавлять настраиваемый баннер в клиент, чтобы пользователи могли включить E911. Пользователи могут закрыть баннер, когда подтвердит свой адрес, и он снова появится при перезапуске Teams. Чтобы включить эту функцию, необходимо задать заявление  об отказе от ответственности службы экстренного реагирования в рамках политики экстренных вызовов Teams и ввести строковое сообщение, которое будет отображаться для пользователей. Это поле является необязательным при настройке настраиваемой политики, а строковое поле ограничено 250 символами.
