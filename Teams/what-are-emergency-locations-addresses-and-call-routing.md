# <a name="manage-emergency-calling"></a>Управление звонками экстренных служб

В этой статье описаны основные понятия, необходимые для управления звонками в экстренные службы, в том числе сведения об адресах для экстренного вызова, динамических адресах для экстренного вызова и маршрутике &mdash; экстренных вызовов. В этой статье используются следующие термины:

- **Адрес для экстренного** обращения: физический или улицный адрес места работы вашей &mdash; организации.

  Например, адрес  *12345 North Main Street, Redmond, WA 98052* используется для направления экстренных вызовов в соответствующие диспетчерские органы и для оказания помощи в поиске вызывающего.

- **Место.** Обычно это этаж, здание, этаж или номер офиса. Место связано с адресом для экстренного устранения чрезвычайной ситуации, чтобы более точно у вас было место в здании. С адресом для экстренного обращения может быть связано неограниченное количество мест. Например, если в организации несколько зданий, может потребоваться включить сведения о месте для каждого здания и для каждого этажа в каждом из них.  

- **Местоположение для экстренного** решения. Местоположение — это городской адрес &mdash; с необязательным местом. Если в вашей компании несколько физических расположений, скорее всего, вам потребуется несколько местоположений для экстренного экстренного нахождения. 

  При создании адреса для экстренного обращения для этого адреса автоматически создается уникальный ИД местоположения.  Например, при добавлении места в адрес для экстренного устранения чрезвычайной ситуации при добавлении этажа в адрес здания создается ИД местоположения для сочетания адреса для экстренного обращения и &mdash; &mdash; места.  В этом примере будут два ИД расположения: один для городского адреса; один для присоединились к городскому адресу и связанному месту.

  Когда вы назначаете местоположение для экстренного ситуация пользователю или сайту, это уникальный ИД местоположения, связанный с пользователем или сайтом.

- **Зарегистрированный адрес:** адрес для экстренного вызова, который назначен каждому пользователю плана звонков; его иногда называют статическим адресом для экстренного обращения или адресом записи.  (Зарегистрированные адреса не относятся к прямой маршрутике пользователей.)

Адреса для экстренного вызова создаются для пользователей планов звонков с Teams центре администрирования.  

>[!Note]
>Существуют некоторые различия в управлении звонками экстренных служб в зависимости от того, используете ли вы планы телефонная система или прямую маршрутику телефонная система для подключения к ПС. Эти факторы описаны в этой статье.

## <a name="emergency-address-validation"></a>Проверка адреса для экстренного устранения

Чтобы назначить адрес для экстренного обращения пользователю или сетевому идентификатору, необходимо убедиться, что адрес для экстренного обращения помечен как "проверенный".  Проверка адреса гарантирует, что адрес является допустимым и его невозможно изменить после его присвоения. 

Если адрес для экстренного устранения проблемы определяется с помощью функции поиска адресной карты в центре администрирования Teams, он автоматически помечется как проверенный. Проверенный адрес для экстренного устранения проблемы изменить нельзя. Поэтому при изменениях формата или представления адреса необходимо создать новый адрес с обновленным форматом.


## <a name="emergency-address-geo-codes"></a>Геопроидеи адресов для экстренного устранения проблем

С каждым адресом для экстренного устранения чрезвычайной ситуации может быть связан географический код (широта и долгота). Эти географические коды используются в некоторых странах для маршрутации экстренных вызовов с динамическими расположениями. 

Если адрес для экстренного устранения проблемы определяется с помощью функции поиска адресной карты в центре администрирования Teams, геопрод автоматически связан с адресом для экстренного обращения. Если адрес определен с помощью PowerShell, с его адресом также можно связать географические коды. Однако корпорация Майкрософт рекомендует создавать адреса для экстренного вызова с помощью функции поиска карт в Центре администрирования Teams, которая гарантирует, что адреса отформатированы, проверены и имеют соответствующие геокоманду.  

>[!Important]
>Чтобы назначить местоположение для экстренного вызова сетевому идентификатору для динамических экстренных вызовов, адрес для экстренного вызова должен содержать соответствующий географический код.

### <a name="emergency-call-enablement"></a>Включить экстренные вызовы

Каждый пользователь плана звонков автоматически может звонить на экстренные вызовы и должен иметь зарегистрированный адрес для экстренного вызова, связанный с его номером телефона. 

Когда расположение должно быть связано с телефонным номером, зависит от страны или региона:

- Например, в США и Канаде при назначении пользователю номера требуется местоположение для экстренного ситуация.

- В других странах, например в Европе, На Ближнем Востоке и в Африка (EMEA), местоположение для экстренного экстренного обслуживания требуется, когда вы получаете номер телефона в Microsoft 365 или Office 365, а также при его переносе от другого поставщика услуг или оператора связи.

### <a name="dynamic-emergency-calling"></a>Динамические экстренные вызовы

Динамические звонки в экстренные службы для планов звонков Майкрософт обеспечивают возможность настройки и направления экстренных вызовов в зависимости от текущего расположения клиента Teams клиента. Возможность автоматической маршрутики в соответствующий пункт ответа на общедоступные вопросы безопасности (PSAP) или оповещать сотрудников службы безопасности зависит от страны использования Teams пользователя.  

Для пользователей плана звонков динамическое расположение для маршрутации экстренных звонков поддерживается только в США. (Сведения о динамических вызовах экстренных служб и прямой маршрутике см. в этой [теме.](#considerations-for-direct-routing)

- Если клиент Teams для плана звонков США динамически получает адрес для экстренного вызова на территории США, этот адрес используется для маршрутировки на экстренных службах, а не для зарегистрированного адреса, и звонок автоматически перенанося в ПСПС в обслуживаемой области адреса.

- Если клиент Teams для пользователя плана звонков в США не получает динамический адрес для экстренного вызова в США, зарегистрированный адрес для обращения в экстренных службах используется для звонков на экране справки и для маршрутизация звонка. Однако перед подключением вызываемой почты к соответствующей psAP будет отсвещаться экран, чтобы определить, требуется ли обновить адрес.

В США необходимо настроить городской адрес, который является частью местоположений для экстренного обращения, которые назначены сетевым идентификаторам, и включить связанные &mdash; с ними геопроидеки. Дополнительные сведения см. в [теме Планирование и настройка динамических экстренных вызовов.](configure-dynamic-emergency-calling.md)


### <a name="emergency-call-routing"></a>Маршрутия экстренных вызовов

Когда пользователь Teams плана звонков звонит на экстренный номер, то то, как звонок передается на PAP, зависит от следующего:

- Определяется ли адрес для экстренного обращения динамически клиентом Teams.

- Является ли адрес для экстренного обращения зарегистрированным адресом, связанным с номером телефона пользователя.

- Сеть для экстренного вызова в этой стране.

  **В США:**

  - Если клиент Teams расположен в динамическом расположении, определяемом клиентом, экстренные вызовы из этого клиента автоматически перенанося в PAP, обслуживающий это географическое расположение. 

  - Если клиент Teams не находится в динамическом расположении, определяемом клиентом, экстренные вызовы из этого клиента будут отсвечены в национальном центре обработки звонков, чтобы определить расположение вызывателя перед переадранием звонка в ПСAP, обслуживающий это географическое расположение.

  - Если вызываемой службе не удается обновить местоположение для экстренного вызова до центра экстренного обслуживания, звонок будет перенаправлен на PAP, обслуживающий зарегистрированный адрес вызываемого.

  **В Канаде, Ирландии и Соединенном Королевстве** экстренные вызовы сначала по экрану определяют текущее расположение пользователя, прежде чем подключаться к соответствующему диспетчеру. 

  **Во Франции, Германии** и Испании экстренные вызовы перена маршрутируются непосредственно в PAP, который обслуживает адрес для экстренного вызова, связанный с номером независимо от расположения вызываемого.

  **В Нидерландских регионах** экстренные вызовы перенаются непосредственно в ПСAP для кода местной области номера независимо от расположения вызываемого.

  **В Австралии** адреса для экстренного устранения чрезвычайной ситуации настраиваются и перенанастраиются партнером-оператором.

  **В Японии** экстренные вызовы не поддерживаются.

### <a name="emergency-call-enablement-and-configuration"></a>Включить и конфигурация экстренных вызовов

Вы должны определить политики экстренных вызовов для пользователей прямой маршрутизации с помощью политики маршрутизации экстренных вызовов Teams (TeamsEmergencyCallRoutingPolicy), чтобы определить номера экстренных служб и их место назначения маршрутизации. (Обратите внимание на то, что зарегистрированные местоположения для экстренного ситуация не поддерживаются для пользователей прямой маршрутации.)

Вы можете назначить политику маршрутинга экстренных вызовов учетной записи Teams прямой маршрутии, сетевому сайту или обоим пользователям. Когда клиент Teams запускает или изменяет сетевое подключение, Teams выполняет подстановку сетевого сайта, на котором он находится:

- Если с сайтом связана политика маршрутинга экстренных звонков, она используется для настройки экстренных вызовов.

- Если с сайтом не связана политика маршрутирования экстренных вызовов или клиент подключен к неопределенным сайтам, для настройки экстренных вызовов используется политика маршрутирования экстренных вызовов, связанная с учетной записью пользователя. 

- Если Teams не может получить политику маршрутизов экстренных вызовов, пользователю не будут включены экстренные вызовы.

### <a name="dynamic-emergency-calling"></a>Динамические экстренные вызовы

Teams клиенты для пользователей прямой маршрутации могут получить динамический адрес для экстренного вызова, который можно использовать для динамической маршрутизаации звонков в зависимости от расположения вызываемого пользователя. Дополнительные сведения см. [в настройках динамических экстренных вызовов.](configure-dynamic-emergency-calling.md)

### <a name="emergency-call-routing"></a>Маршрутия экстренных вызовов

Политика маршрутации экстренных вызовов ссылается на использование ЗВОНКОВ по сети, для правильного перенаправки экстренных вызовов на соответствующие шлюзы ПСС необходимо иметь соответствующую конфигурацию прямой маршрутации. В частности, необходимо убедиться в том, что для строки набора экстренных служб есть OnlineVoiceRoute. Дополнительные сведения см. [в этой ссылке.](direct-routing-configure.md) 

(Примечание. Teams- и более новых клиентов перед номерами экстренных служб предопластили знак "+", как Skype для бизнеса клиент, то есть +911. В ближайшие месяцы это поведение будет изменено, Teams экстренных вызовов больше не будет отправляться "+" перед номером; то есть 911.)

Возможность динамического перенаправки экстренных вызовов для пользователей прямой маршрутки зависит от сети экстренных вызовов в пределах данной страны. Существует два решения:

- Экстренные поставщики маршрутов (только в США) 
- Приложения шлюза ELIN для экстренного определения местоположения

#### <a name="emergency-routing-service-providers"></a>Поставщики услуг для экстренных маршрутов

В США существует множество сертифицированных поставщиков услуг маршрутации экстренных служб (ERSP), которые могут автоматически перенау числеть экстренные вызовы в зависимости от расположения вызываемого.

- Если поставщик услуг экстренных маршрутов интегрирован в развертывание Direct Routing, экстренные вызовы с динамически приобретенным местоположением автоматически перенаправляются в точку ответа на общедоступный доступ (PSAP), обслуживающий это расположение.

-  Экстренные вызовы без динамического расположения сначала по экрану определяют текущее расположение пользователя, прежде чем подключаться к соответствующему диспетчеру на основе обновленного расположения.

Дополнительные сведения см. в [теме Контроллеры границ сеанса, удостоверенные для прямой маршрутии.](direct-routing-border-controllers.md)


#### <a name="emergency-location-identification-number-elin-applications"></a>Приложения для идентификации местоположения для экстренного нахождения (ELIN)

Контроллеры границ сеанса (SBCs) могут включать приложения ELIN для идентификации местоположения в экстренных службах. Если приложение ELIN SBC интегрировано с развертыванием Direct Routing, необходимо настроить адреса для экстренного вызова и связанные с ним телефонные номера в приложении ELIN, а затем отправить записи ELIN в базу данных для экстренных вызовов в соответствующей базе данных ЗВОНКОВ.  Teams местоположения для экстренного ситуация с идентификатором ELIN должны соответствовать расположениям в приложении ELIN.

Если экстренный звонок с динамически приобретенным местоположением перенанается в соответствующую службу SBC, приложение ELIN:

- Просмотр местоположения вызываемого в экстренных службах.
- Соответствует расположению записи ELIN.
- Заменяет номер экстренных вызовов номером телефона ELIN.
- Перенаправит звонок в ПСОП, обслуживающий это расположение, и диспетчеры получат расположение из отправленной записи ELIN.

При обратном звонке на номер для экстренного вызова приложение ELIN будет заменять номер исходного вызываемого экстренных служб. 

Дополнительные сведения см. в [теме Контроллеры границ сеанса, удостоверенные для прямой маршрутии.](direct-routing-border-controllers.md)


## <a name="security-desk-notification"></a>Уведомление службы безопасности

Уведомление службы безопасности доступно как для планов звонков Майкрософт, так и для телефонная система прямой маршрутики.

С помощью политики Teams экстренного вызова (TeamsEmergencyCallingPolicy) можно настроить, кто должен быть уведомлен во время экстренного звонка и как они будут уведомлены: только чат, конференц-вызов, отключен или конференц-вызов, а также отключен звук, но с возможностью его отключения.  Вы также можете указать внешний номер ДНР пользователя или группы, чтобы позвонить и присоединиться к экстренным звонкам. 

Политику экстренных вызовов можно предоставить учетной записи Teams, сетевому сайту или обоим пользователям.  Когда клиент Teams запускает или изменяет сетевое подключение, Teams выполняет подстановку сетевого сайта, на котором он находится:

- Если политика экстренных вызовов связана с сетевым сайтом, она используется для настройки уведомлений службы безопасности.

- Если с сайтом не связана политика экстренных вызовов или клиент подключен к неопределенным сайтам, для настройки уведомлений службы безопасности используется политика для экстренного вызова, связанная с учетной записью пользователя.  

- Если Teams не может получить политику экстренных вызовов, пользователь не получает уведомление службы безопасности.

Во время звонка в экстренные службы конференц-зал безопасности, а пользовательский интерфейс пользователя службы безопасности управляется на основе политики Teams экстренных вызовов. С каждым сотрудником службы безопасности начинается групповой чат, и к расположению вызываемого на экстренную помощь будет отправляться уведомление о важном сообщении.  Если конференц-зал настроен в рамках политики, каждый пользователь службы безопасности также называется частью конференции.
