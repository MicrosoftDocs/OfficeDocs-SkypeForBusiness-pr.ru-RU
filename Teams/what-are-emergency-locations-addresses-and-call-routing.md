# <a name="manage-emergency-calling"></a>Управление звонками экстренных служб

В этой статье&mdash; описаны основные понятия, необходимые для управления звонками экстренных служб. В ней содержатся сведения об адресах для экстренного вызова, динамических адресах для экстренного вызова и маршрутизах экстренных вызовов. В этой статье используются следующие термины:

- **Адрес для экстренного** обращения: физический&mdash; или городской адрес места работы вашей организации.

  Например, адрес *12345 North Main Street, Redmond, WA 98052* используется для направления экстренных вызовов в соответствующие диспетчерские органы и для оказания помощи в поиске вызывающего.

- **Место** . Обычно это этаж, здание, этаж или номер офиса. Место связано с адресом для экстренного устранения чрезвычайной ситуации, чтобы получить более точное расположение в здании. С адресом для экстренного обращения может быть связано неограниченное количество мест. Например, если в организации несколько зданий, может потребоваться включить сведения о каждом здании и каждом этаже в каждом из них.  

- **Местоположение для экстренного** устранения проблем. Местоположение — это городской&mdash; адрес с необязательным местом. Если в вашей компании несколько физических расположений, скорее всего, вам потребуется несколько местоположений для экстренного экстренного нахождения. 

  При создании адреса для экстренного обращения для этого адреса автоматически создается уникальный ИД местоположения. &mdash;Например,&mdash; при добавлении места в адрес для экстренного устранения чрезвычайной ситуации, при добавлении этажа в ИД местоположения здания для сочетания адреса для экстренного обращения и места создается этаж.  В этом примере будут два ИД расположения: один для городского адреса; один для присоединились к городскому адресу и связанному месту.

  Когда вы назначаете местоположение для экстренного ситуация пользователю или сайту, это уникальный ИД местоположения, связанный с пользователем или сайтом.

- **Зарегистрированный адрес** : адрес для экстренного устранения чрезвычайной ситуации, который назначен каждому пользователю. Зарегистрированный адрес иногда называют статическим адресом для экстренного обращения или адресом записи. (В настоящее время зарегистрированные адреса не поддерживаются при прямой маршрутике. Проверьте, нет ли обновлений.)

>[!Note]
>Существуют некоторые различия в управлении звонками экстренных служб в зависимости от того, используете ли вы планы звонков Майкрософт, операторы Подключение или прямую маршрутику для подключения к [ПСОП](pstn-connectivity.md). Эти факторы описаны в этой статье.

## <a name="emergency-address-validation"></a>Проверка адреса для экстренного устранения

Чтобы назначить адрес для экстренного обращения пользователю или сетевому идентификатору, необходимо убедиться, что адрес для экстренного обращения помечен как "проверенный". Проверка адреса гарантирует, что адрес является допустимым и его невозможно изменить после его присвоения. 

Если адрес для экстренного устранения проблемы определяется с помощью функции поиска адресной карты в центре администрирования Teams, адрес автоматически помечется как проверенный. Поскольку вы не можете изменить&mdash; проверенный адрес для экстренного обращения, если формат или представление адреса изменяется, необходимо создать новый адрес с обновленным форматом.


## <a name="emergency-address-geo-codes"></a>Геопроидеи адресов для экстренного устранения проблем

С каждым адресом экстренных служб может быть связан географический код (широта и долгота). Эти географические коды используются в некоторых странах для маршрутации экстренных вызовов с динамическими расположениями. 

Если адрес для экстренного устранения проблемы определяется с помощью функции поиска адресной карты в центре администрирования Teams, геопрод автоматически связан с адресом для экстренного обращения. Если адрес определен с помощью PowerShell, с его адресом также можно связать географические коды. 

Корпорация Майкрософт рекомендует создавать адреса для экстренного обращения с помощью функции поиска карт в центре администрирования Teams, которая гарантирует, что адреса отформатированы, проверены и имеют соответствующие геокоманду. 

>[!Important]
>Чтобы назначить местоположение для экстренного вызова сетевому идентификатору для динамических экстренных вызовов, адрес для экстренного вызова должен содержать соответствующий географический код.


## <a name="considerations-for-calling-plans"></a>Аспекты планов звонков

В следующих разделах описано, как управлять звонками экстренных служб для пользователей плана звонков Майкрософт. Чтобы узнать, правильные ли планы звонков Майкрософт являются правильным решением для вашей компании, см. параметры подключения через [ЗВОНКОВ по ННР](pstn-connectivity.md).


### <a name="emergency-call-enablement-for-calling-plans"></a>Включить экстренные вызовы для планов звонков

Каждый пользователь плана звонков автоматически может звонить на экстренные вызовы и должен иметь зарегистрированный адрес для экстренного вызова, связанный с назначенным номером телефона. 

Когда расположение связано с телефонным номером, зависит от страны или региона:

- Например, в США и Канаде при назначении пользователю номера требуется местоположение для экстренного ситуация.

- &mdash;Для других стран, например Европы, Среднего Востока и Африка (EMEA),&mdash; требуется местоположение для экстренного экстренного обслуживания, когда вы получаете номер Microsoft 365 или при его переносе от другого поставщика услуг или оператора.


### <a name="dynamic-emergency-calling-for-calling-plans"></a>Динамические экстренные вызовы для планов звонков

Динамические звонки в экстренные службы для планов звонков предоставляют возможность настраивать и маршрутировать экстренные вызовы в зависимости от текущего расположения клиента Teams экстренного вызова. Возможность автоматической маршрутики в соответствующий пункт ответа на общедоступные вопросы безопасности (PSAP) или оповещать сотрудников службы безопасности зависит от страны использования Teams пользователя.  

Динамическое расположение для маршрутизации экстренных вызовов поддерживается в США следующим образом. 

- Если клиент Teams для плана звонков США динамически получает адрес для экстренного обращения в США, этот адрес используется для маршрутировки в экстренных службах, а не для зарегистрированного адреса, и звонок будет автоматически перенаначен в ПСПС в обслуживаемой области адреса.

- Если клиент Teams для пользователя плана звонков в США не получает динамический адрес для экстренного вызова в США, зарегистрированный адрес для обращения в экстренных службах используется для обращения в экстренных службах и маршрутизация звонка. Однако перед подключением вызываемой почты к соответствующей psAP будет отсвещаться экран, чтобы определить, требуется ли обновить адрес.

Динамическое расположение для маршрутизация экстренных вызовов поддерживается в Канаде так же, как и в США, за исключением следующего: все экстренные вызовы будут отсвечены на национальном уровне перед их перенаправлением в PAP.

Дополнительные сведения см. в [теме Планирование и настройка динамических экстренных вызовов](configure-dynamic-emergency-calling.md).

### <a name="emergency-call-routing-for-calling-plans"></a>Маршрутия экстренных вызовов для планов звонков

Когда пользователь Teams плана звонков звонит на экстренный номер, то то, как звонок передается на PAP, зависит от следующего:

- Определяется ли адрес для экстренного обращения динамически клиентом Teams.

- Является ли адрес для экстренного обращения зарегистрированным адресом, связанным с номером телефона пользователя.

- Сеть для экстренного вызова в этой стране.

Например:

**В США:**

- Если клиент Teams расположен в динамическом расположении, определяемом клиентом, экстренные вызовы из этого клиента автоматически перенанося в PAP, обслуживающий это географическое расположение.

- Если клиент Teams не находится в динамическом расположении, определяемом клиентом, экстренные вызовы из этого клиента будут отсвечены в национальном центре обработки звонков, чтобы определить расположение вызывателя перед переададционным вызовом в PAP, обслуживающий это географическое расположение.

- Если вызываемой службе не удается обновить местоположение для экстренного вызова до центра экстренного обслуживания, звонок будет перенаправлен на PAP, обслуживающий зарегистрированный адрес вызываемого.

**В Канаде, Ирландии и Соединенном** Королевстве для определения текущего местоположения пользователя перед подключением звонка в соответствующий диспетчерский центр сначала высвеяются экстренные вызовы.

**Во Франции, Германии** и Испании экстренные вызовы перена маршрутируются непосредственно в PAP, обслуживающий адрес для экстренного вызова независимо от расположения вызываемого.

**В Нидерландских регионах** экстренные вызовы перенаются непосредственно в ПСAP для кода местной области номера независимо от расположения вызываемого.

**В Австралии** адреса для экстренного устранения чрезвычайной ситуации настраиваются и перенанастраиются партнером- оператором.

**В Японии** экстренные вызовы не поддерживаются.

## <a name="considerations-for-operator-connect"></a>Факторы, которые следует учитывать при Подключение

В следующих разделах описано, как управлять экстренным звонком для Подключение пользователей. Чтобы узнать, является ли Подключение "Оператор" правильным решением для вашей компании, см. параметры подключения через [ПСОП](pstn-connectivity.md).

### <a name="emergency-call-enablement-for-operator-connect"></a>Включить экстренные вызовы для операторов Подключение

Каждый Подключение автоматически включен для экстренных вызовов. Экстренные вызовы автоматически перена маршрутируются оператору Подключение оператора на заданный номер.

Возможность администратора клиента настроить зарегистрированный адрес для оператора Подключение пользователя будет зависеть от возможностей, которые назначены номеру при отправке их в перечень клиентов. На основе этого параметра&mdash;&mdash; администратор клиента может или не иметь права на установку, изменение или удаление местоположения пользователя для экстренного ситуация. 

### <a name="dynamic-emergency-calling-for-operator-connect"></a>Динамические экстренные вызовы для Подключение

Динамические экстренные вызовы для Подключение операторов предоставляют возможность настраивать и маршрутировать экстренные вызовы в зависимости от текущего расположения клиента Teams экстренного вызова. Возможность автоматической маршрутики в соответствующий пункт ответа на общедоступные вопросы безопасности (PSAP) или оповещать сотрудников службы безопасности зависит от страны использования Teams пользователя. 

Динамическое расположение для маршрутизации экстренных вызовов поддерживается в США следующим образом. 

- Если клиент Teams для пользователя из США динамически получает адрес для экстренного устранения чрезвычайной ситуации в США, этот адрес используется для маршрутировки в экстренных службах, а не для зарегистрированного адреса, и звонок автоматически передается в ПСПС в обслуживаемой области адреса.

- Если клиент Teams для пользователя из США не получает динамический адрес для обращения в экстренных службах в США, зарегистрированный адрес для экстренного устранения экстренных ситуаций используется для обращения в экстренные службы и маршрутизация звонка. Однако перед подключением вызываемой почты к соответствующей psAP будет отсвещаться экран, чтобы определить, требуется ли обновить адрес.

Динамическое расположение для маршрутизация экстренных вызовов поддерживается в Канаде так же, как и в США, за исключением следующих исключений: перед перенаправлением в PAP все экстренные вызовы будут отсвечены на все страны.

Дополнительные сведения см. в [теме Планирование и настройка динамических экстренных вызовов](configure-dynamic-emergency-calling.md).

### <a name="emergency-call-routing-for-operator-connect"></a>Маршрутия экстренных вызовов для операторов Подключение

Когда оператор Teams Подключение наберет номер для экстренного вызова, то то, как звонок будет перенаначен на PAP, зависит от следующего:

- Определяется ли адрес для экстренного обращения динамически клиентом Teams.

- Является ли адрес для экстренного обращения зарегистрированным адресом, связанным с номером телефона пользователя.

- Сеть для экстренного вызова в этой стране.

- В США и Канаде динамическая маршрутия является частью службы оператора связи. Вам не нужно приобретать эту услугу у другого поставщика услуг.

- Если клиент Teams расположен в динамическом расположении, определяемом клиентом:

   - В США экстренные вызовы от этого клиента автоматически перенанаются в ПСПС, обслуживающий это географическое расположение.
   - В Канаде все экстренные вызовы будут отсвещаться в национальном центре обработки вызовов, прежде чем перенаводить звонок в PAP, обслуживающий это географическое расположение.

- Если клиент Teams не находится в динамическом расположении, определяемом клиентом, экстренные вызовы из этого клиента будут отсвечены в национальном центре обработки звонков, чтобы определить расположение вызывателя перед переададционным вызовом в PAP, обслуживающий это географическое расположение.

- Если вызываемой службе не удается обновить местоположение для экстренного вызова до центра экстренного обслуживания, звонок будет перенаправлен на PAP, обслуживающий зарегистрированный адрес вызываемого.


## <a name="considerations-for-direct-routing"></a>Факторы, которые следует учитывать при прямой маршрутике

В следующих разделах описано, как управлять звонками в экстренные службы для пользователей прямой маршрутки. Чтобы узнать, является ли Прямая маршрутная маршрутия правильным решением для вашей компании, см. параметры подключения через [STN](pstn-connectivity.md).

### <a name="emergency-call-enablement-for-direct-routing"></a>Включить экстренные вызовы для прямой маршрутии

Чтобы определить прямую маршрутику, необходимо определить политики экстренных вызовов для пользователей с [](manage-emergency-call-routing-policies.md) помощью Teams для определения номеров экстренных служб и их назначения маршрутинга. (В настоящее время зарегистрированные местоположения для экстренного ситуация не поддерживаются для пользователей прямой маршрутации.)

Вы можете назначить политику маршрутинга экстренных вызовов учетной записи пользователя прямой маршрутки, сетевому сайту или обоим из них. Когда клиент Teams запускает или изменяет сетевое подключение, Teams выполняет подстановку сетевого сайта, на котором он находится:

- Если с сайтом связана политика маршрутинга экстренных звонков, она используется для настройки экстренных вызовов.

- Если с сайтом не связана политика маршрутирования экстренных звонков, если клиент подключен на неопределеном сайте или этот номер не соответствует ни одной из номеров, определенных в политике маршрутирования экстренных звонков, связанной с сайтом, то для настройки экстренных вызовов используется политика маршрутирования экстренных вызовов, связанная с учетной записью пользователя. 

- Если Teams не может получить политику маршрутизов экстренных вызовов, пользователю не будут включены экстренные вызовы.


### <a name="dynamic-emergency-calling-for-direct-routing"></a>Динамические экстренные вызовы для прямой маршрутизы

Динамические звонки в экстренные службы для прямой маршрутации предоставляют возможность настраивать и перенаправить экстренные вызовы в зависимости от текущего расположения клиента Teams. Возможность автоматической маршрутики в соответствующий пункт ответа на общедоступные вопросы безопасности (PSAP) или оповещать сотрудников службы безопасности зависит от страны использования Teams пользователя.

Для пользователей прямой маршрутации динамическое расположение для маршрутизов в экстренных службах поддерживается только в США:

-   Если клиент Teams для пользователя прямой маршрутировки в США динамически получает адрес для экстренного обращения в США, этот адрес используется для маршрутировки в экстренных службах, и звонок будет автоматически перенаправляться в ПСПС в области подачи адреса.

-   Если клиент Teams для пользователя прямой маршрутизация в СОЕДИНЕННЫх Штатах не получает динамический адрес для экстренного вызова в США, то перед подключением вызываемого пользователя к соответствующей psap будет по экрану отсвечен звонок, чтобы определить, требуется ли для него обновленный адрес.

Динамическое расположение для маршрутизация экстренных вызовов поддерживается в Канаде так же, как и в США, за исключением следующего: все экстренные вызовы будут отсвечены на национальном уровне перед их перенаправлением в PAP.

Дополнительные сведения см. [в настройках динамических экстренных вызовов](configure-dynamic-emergency-calling.md).

### <a name="emergency-call-routing-for-direct-routing"></a>Маршрутия экстренных вызовов для прямой маршрутизы

Политика маршрутации экстренных вызовов для прямой маршрутиации ссылается на использование ЗВОНКОВ через Интернет, для правильного перенаправки экстренных вызовов на соответствующие шлюзы ЗВОНКОВ через Интернет. В частности, необходимо убедиться в том, что для строки набора экстренных служб есть OnlineVoiceRoute. Дополнительные сведения см. [в этой ссылке](direct-routing-configure.md). 

> [!NOTE]
> Teams клиенты больше не предопластили знак "+" перед номерами для экстренного помощи, то есть +911. Поэтому Teams экстренных вызовов больше не будет отправляться "+" перед номером 911. Убедитесь, что изменения отражаются в шаблонах голосового маршрута.

Возможность динамического перенаправки экстренных вызовов для пользователей прямой маршрутии зависит от сети экстренных вызовов в пределах данной страны. Существует два решения:

- [Поставщики услуг для экстренных маршрутов (только в США)](#emergency-routing-service-providers)
- [Приложения для идентификации местоположения для экстренного местоположения](#emergency-location-identification-number-applications)

#### <a name="emergency-routing-service-providers"></a>Поставщики услуг экстренных маршрутов

В США существует множество сертифицированных поставщиков услуг маршрутации экстренных служб (ERSP), которые могут автоматически перенау числеть экстренные вызовы в зависимости от расположения вызываемого.

- Если поставщик услуг экстренных маршрутов интегрирован в развертывание Direct Routing, экстренные звонки с динамически приобретенным местоположением автоматически перенаправляются в точку ответа на общедоступные вопросы безопасности (PSAP), обслуживающие это расположение.

-  Экстренные вызовы без динамического расположения сначала по экрану определяют текущее расположение пользователя, прежде чем подключаться к соответствующему диспетчеру на основе обновленного расположения.

Дополнительные сведения см. в [теме Контроллеры границ сеанса, удостоверенные для прямой маршрутии](direct-routing-border-controllers.md).


#### <a name="emergency-location-identification-number-applications"></a>Приложения для идентификации местоположения для экстренного местоположения

Контроллеры границ сеанса (SBCs) могут включать приложения ELIN для экстренного определения местоположения. Если приложение ELIN SBC интегрировано в развертывание Direct Routing, необходимо настроить адреса для экстренного вызова и связанные с ним телефонные номера в приложении ELIN, а затем отправить записи ELIN в базу данных для экстренных вызовов в соответствующей базе данных ЗВОНКОВ. Teams местоположения для экстренного ситуация с идентификатором ELIN должны соответствовать расположениям в приложении ELIN.

Если экстренный звонок с динамически приобретенным местоположением перенанается в соответствующую службу SBC, приложение ELIN:

- Просмотр местоположения для экстренного вызова.
- Соответствует расположению записи ELIN.
- Заменяет номер экстренных вызовов номером телефона ELIN.
- Перенаправит звонок в ПСОП, обслуживающий это расположение, и диспетчеры получат расположение из отправленной записи ELIN.

При обратном звонке на номер для экстренного вызова приложение ELIN отменит подстановку номера для первоначального вызываемого экстренных служб. 

Дополнительные сведения см. в [теме Контроллеры границ сеанса, удостоверенные для прямой маршрутии](direct-routing-border-controllers.md).


## <a name="security-desk-notification"></a>Уведомление службы безопасности

Уведомление службы безопасности доступно как для планов звонков Майкрософт, так и Подключение и прямой маршрутики.

С помощью политики Teams экстренных вызовов (TeamsEmergencyCallingPolicy) можно настроить, кто должен быть уведомлен во время звонка и как они будут уведомлены: только чат, конференц-вызов, отключен или конференц-вызов с отключенным звуком, но с возможностью его отключения. Вы также можете указать внешний номер ДНР пользователя или группы, чтобы позвонить и присоединиться к экстренным звонкам. Обратите внимание на то, что стороне ПСП не разрешено в звуках.

Политику экстренных вызовов можно предоставить учетной записи Teams, сетевому сайту или обоим пользователям.  Когда клиент Teams запускает или изменяет сетевое подключение, Teams выполняет подстановку сетевого сайта, на котором он находится:

- Если политика экстренных вызовов связана с сетевым сайтом, она используется для настройки уведомлений службы безопасности.

- Если с сайтом не связана политика экстренных вызовов или клиент подключен к неопределенным сайтам, для настройки уведомлений службы безопасности используется политика экстренных вызовов, связанная с учетной записью пользователя.  

- Если Teams не может получить политику экстренных вызовов, это значит, что пользователь не получает уведомление службы безопасности.

Во время звонка в экстренные службы к вызову конференц-зал безопасности, а пользовательский интерфейс пользователя службы безопасности управляется на основе политики Teams экстренных вызовов. С каждым сотрудником службы безопасности начинается групповой чат, и о расположении вызываемого экстренных служб можно получить уведомление о важном сообщении.  Если конференц-зал настроен в рамках политики, каждый пользователь службы безопасности также называется частью конференции.
