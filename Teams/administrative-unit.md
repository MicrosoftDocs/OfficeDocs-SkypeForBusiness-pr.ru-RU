# <a name="administrative-unit-functionality-for-device-management-in-teams"></a>Функции администрирования для управления устройствами в Teams

Более детализированный доступ на основе ролей для управления устройствами с помощью Microsoft Teams администрирования. Мы реализовали концепцию администрирования для управления устройствами через центр администрирования Teams администрирования.

Благодаря концепции администрирования вы обеспечиваете доступ к определенному набору ресурсов специальному администратору. Административный блок ограничивает доступ ко всем ресурсам. Вы можете расширить функции управления Teams устройств.

> [!NOTE]
> Понятие административного блока доступно только для роли администратора Teams устройств.

Например, в Contoso есть операции в разных регионах. Юда является глобальным ИТ-администратором в Москве, а Прашант — ИТ-администратором Индии. Сегодня, когда Prashant в Teams с ролью Администратор устройства, он увидит устройства по всему миру. Он хочет ограничить доступ Prashant только к устройствам, которые находятся в Индии. Эту проблему можно решить с помощью концепции административных единиц. Подробнее о [концепции административных единиц.](/azure/active-directory/roles/administrative-units)

## <a name="creation-of-administrative-units"></a>Создание административных единиц

Создавайте административные единицы на портале Azure и назначайте администраторов соответствующим административным единицам. Дополнительные информацию о назначении административных единиц можно узнать в [этой теме.](/azure/active-directory/roles/admin-units-manage)

![пример единиц администрирования компании](media/au-example.png)

После создания глобальный ИТ-администратор может добавить пользователей устройств, соответствующих этому административному блоку.

![Пример компании с предварительным просмотром пользователей](media/au-example2.png)

Назначение роли можно выполнять с помощью PowerShell с помощью [cmdlet add-AzureADMSScopedRoleMembership.](/powershell/module/azuread/add-azureadmsscopedrolemembership?view=azureadps-2.0)

После того как вы направили роли для пользователей административных единиц, пользователям необходимо войти в Teams администрирования, чтобы начать управлять устройствами с областью действия.

## <a name="experience-for-administrative-unit-admin"></a>Опыт администрирования административных единиц

Если одному и тем же администраторам назначены обязанности нескольких административных единиц, они могут переключаться между административными единицами, не выходя с портала. В измененной административной единице будут видеться только те устройства, которые связаны с новым административным блоком.
