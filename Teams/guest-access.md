# <a name="guest-access-in-microsoft-teams"></a>Гостевой доступ в Microsoft Teams

Благодаря гостевому доступу вы можете предоставлять доступ к командам, документам в каналах, ресурсам, чатам и приложениям людям за пределами организации, при этом сохраняя контроль над корпоративными данными.

Гость — это человек, не являющийся сотрудником, учащимся или участником вашей организации. У него нет учебной или рабочей учетной записи, предоставленной вашей организацией. Например, к гостям могут относиться партнеры, продавцы, поставщики или консультанты. Любого пользователя, не состоящего в вашей организации, можно добавить в качестве гостя в Teams. Это означает, что любой человек с бизнес-учетной записью (учетной записью Azure Active Directory) или учетной записью электронной почты потребителей (с Outlook.com, Gmail.com или другими) может принимать участие в качестве гостя в Teams с доступом к командам и каналам.

На гостей в Teams распространяется та же защита соответствия требованиям и аудита, что и на остальной части системы Microsoft 365, и управление гостями в Azure AD. На гостевой доступ распространяются ограничения служб Azure AD и Microsoft 365 или Office 365.

Гостевое взаимодействие предусматривает ограничения. Полный список возможностей, которые может и не может делать гость в Teams, см. в сравнении возможностей участника команды и [гостя.](guest-experience.md#comparison-of-team-member-and-guest-capabilities)

> [!IMPORTANT]
> Гости могут перейти в режим сосуществования в режиме обновления Teams на все организации. Изменить их невозможно.

О том, как настроить гостевой доступ, см. в совместной работе с [гостями в команде.](https://docs.microsoft.com/microsoft-365/solutions/collaborate-as-team) 

Чтобы сравнить внешний доступ (федерацию) с гостевым доступом (и выбрать нужный), ознакомьтесь со статьей [Общение с пользователями из других организаций в Teams](communicate-with-users-from-other-organizations.md).

## <a name="set-up-guest-access"></a>Настройка гостевого доступа

Для гостевого доступа в Teams необходимо настроить другие параметры Microsoft 365, включая параметры в Azure AD, Группах Microsoft 365 и SharePoint. Если вы готовы пригласить гостей в команды, прочитайте одно из следующих сообщений:

- Чтобы настроить гостевой доступ для Teams для общего использования, см. совместное использование с гостями [в команде.](https://docs.microsoft.com/microsoft-365/solutions/collaborate-as-team)
- Для совместной работы с партнерской организацией, которая использует Azure Active Directory и позволяет гостям самостоятельно зарегистрироваться для доступа к команде, см. дополнительные данные о создании [экстрасети B2B](https://docs.microsoft.com/microsoft-365/solutions/b2b-extranet)с управляемыми гостями.

Гостевой доступ в Teams настраивается на всей организации и по умолчанию отключен. Вы можете управлять гостевом доступом к отдельным командам с помощью [меток конфиденциальности.](https://docs.microsoft.com/microsoft-365/compliance/sensitivity-labels-teams-groups-sites)

## <a name="how-a-guest-becomes-a-member-of-a-team"></a>Как гость может стать участником команды

1. Владелец команды или администратор Microsoft 365 добавляет [гостя в команду.](https://support.office.com/article/add-guests-to-a-team-fccb4fa6-f864-4508-bdde-256e7384a14f)
2. Гость получает приветственное электронное сообщение от владельца группы, содержащее сведения о группе и о том, чего следует ожидать, став ее участником.
3. Гость принимает приглашение.
  Гости, у которых есть учетная запись для работы или учебного заведения в Azure Active Directory, могут принять приглашение и подтвердить свою подлинность напрямую. Другим пользователям отправляется разовая проверка подлинности для подтверждения личности (требуется разовая проверка[подлинности).](https://docs.microsoft.com/azure/active-directory/external-identities/one-time-passcode)
4. Приняв приглашение, гость сможет [использовать команды и каналы](https://support.office.com/article/df38ae23-8f85-46d3-b071-cb11b9de5499), просматривать сообщения каналов и отвечать на них, [получать доступ к файлам в каналах](https://support.office.com/article/access-files-in-channels-c593c78a-27c4-4661-a598-682baa30ca7e), участвовать в чатах, присоединяться к собраниям, вести совместную работу над документами и т. д. 

Гости в Teams четко определены. Имя гостя включает в себя метку "Гость", а в канале есть значок, который указывает на то, что в команде есть гости. Дополнительные сведения см. в разделе [Взаимодействие с гостями](guest-experience.md).
  
Находясь внутри Teams, гости могут выходить из команды в любое время. Дополнительные сведения см. в разделе [Как покинуть команду?](https://support.office.com/article/leave-a-team-e481005d-3ec6-4694-b300-375472ba4076)

> [!NOTE]
> Выход из команды не удаляет гостевую учетную запись из каталога организации. Это должен сделать глобальный администратор Microsoft 365 или администратор Azure AD.

## <a name="licensing-for-guest-access"></a>Лицензии для гостевого доступа

Гостевой доступ входит во все подписки на Microsoft 365 бизнес стандартный, Microsoft 365 корпоративный и Microsoft 365 для образования. Дополнительная лицензия На Microsoft 365 не требуется. В Teams не ограничивается число гостей, которых можно добавить. Однако общее число гостей, которых можно добавить в клиент, может быть ограничено платными функциями Azure AD. Дополнительные сведения см. в модели [вы выставления счета для внешних](https://docs.microsoft.com/azure/active-directory/b2b/licensing-guidance)удостоверений Azure AD.

> [!NOTE]
> Пользователей в организации, у которых есть только автономные планы подписки на Microsoft 365, например Exchange Online (план 2), невозможно пригласить в качестве гостей, так как Teams считает, что эти пользователи относятся к одной и той же организации. Чтобы использовать Teams для этих пользователей, им должна быть назначена подписка на Microsoft 365 бизнес стандарт, Office 365 корпоративный или Office 365 для образования. 

## <a name="guest-access-reviews"></a>Отзывы о гостевом доступе

С помощью Azure AD можно создать отзыв о доступе для участников группы или пользователей, которые назначены приложению. Создание повторяющихся проверок доступа поможет сэкономить время. Если вам нужно регулярно пересматривать пользователей, у которых есть доступ к приложению, команде или членам группы, вы можете определить частоту этих проверок. 

Вы можете самостоятельно просмотреть гостевой доступ, попросить гостей просмотреть их собственные членство или попросить владельца приложения или бизнес-решения выполнить проверку доступа. Для выполнения проверок гостевого доступа используйте портал Azure. Дополнительные сведения см. в обзоре гостевого [доступа Azure AD.](https://docs.microsoft.com/azure/active-directory/governance/manage-guest-access-with-access-reviews)
