В Microsoft Teams есть новый способ сохранения записей собраний. В качестве первого этапа перехода от классической программы Microsoft Stream к новому потоку этот метод сохраняет записи в Microsoft OneDrive для бизнеса и SharePoint в Microsoft 365 и предоставляет множество преимуществ. [](/stream/streamnew/new-stream)

> [!NOTE]
> Если запись собрания Teams не удается отправить в OneDrive или SharePoint, она будет временно сохранена в службах мультимедиа Azure (AMS). Если запись сохранена в amS, попытка автоматической отправки записи в OneDrive, SharePoint или Stream не будет повториться.

Записи собраний, хранимые в AMS, доступны в течение 21 дней перед автоматическим удалением. Пользователи могут скачать видео из amS, если им нужно сохранить копию.

Преимущества использования OneDrive для бизнеса и SharePoint для хранения записей:

- Политики хранения для записи собраний Teams (TMR) (автоматические метки S+C E5)
- Преимущества управления сведениями в OneDrive для бизнеса и SharePoint
- Простое настройка разрешений и общего доступа
- Делиться записями только с гостями (внешними пользователями) с явным доступом
- Запрос потока доступа
- Предоставление общих ссылок в OneDrive для бизнеса и SharePoint
- Записи собраний доступны быстрее
- **Перейти в службу поддержки локального** клиента
- Поддержка с несколькими регионами: записи хранятся в конкретном регионе для этого пользователя
- Поддержка собственного ключа (BYOK)

См. полный список [функций, доступных сегодня, и о том, чего ожидать с течением времени.](https://docs.microsoft.com/stream/streamnew/features-new-version-stream)

## <a name="set-up-the-meeting-recording-option-for-onedrive-for-business-and-sharepoint"></a>Настройка параметра записи собрания в OneDrive для бизнеса и SharePoint

Параметр записи собрания — это параметр на уровне политики Teams. В следующем примере показано, как настроить глобальную политику. Убедитесь, что вы настроите параметр записи собрания для политик, которые вы настроите для пользователей.

> [!Note]
> Распространение изменений в политике собраний Teams займет некоторое время. Повторите попытку через несколько часов после его настройки, а затем выйди из системы и войди в приложение Teams Desktop еще раз или просто перезагрузите компьютер.

1. Установите Teams PowerShell PowerShell.

   > [!NOTE]
   > Skype для бизнеса Online Connector в настоящее время является частью последнего модуля Teams PowerShell. Если вы используете последний общедоступный выпуск Teams PowerShell, вам не нужно устанавливать Соединитель Skype для бизнеса Online. См. [управление Skype для бизнеса Online с помощью PowerShell.](/microsoft-365/enterprise/manage-skype-for-business-online-with-microsoft-365-powershell?preserve-view=true&view=o365-worldwide)

2. Запустите PowerShell в качестве администратора.

3. Установите [модуль Teams PowerShell](./teams-powershell-install.md).

4. Импорт модуля MicrosoftTeams и вход в качестве администратора Teams.


   ```powershell
   # When using Teams PowerShell Module
   
   Import-Module MicrosoftTeams
   $credential = Get-Credential
   Connect-MicrosoftTeams -Credential $credential
   ```

5. Используйте [Set-CsTeamsMeetingPolicy,](/powershell/module/skype/set-csteamsmeetingpolicy) чтобы настроить политику собраний Teams для перехода с хранилища Stream на OneDrive для бизнеса и SharePoint.

   ```powershell
   Set-CsTeamsMeetingPolicy -Identity Global -RecordingStorageMode "OneDriveForBusiness"
   ```

> [!Note]
> Если некоторым из ваших пользователей назначена политика "на пользователя" или "на пользователя", необходимо установить этот параметр для этой политики, если вы хотите, чтобы они также могли хранить записи собраний в OneDrive для бизнеса и SharePoint. Дополнительные сведения см. в [управлении политиками собраний в Teams.](meeting-policies-in-teams.md)

## <a name="opt-out-of-onedrive-for-business-and-sharepoint-to-continue-using-stream"></a>Отказ от использования OneDrive для бизнеса и SharePoint для продолжения использования Stream

Даже если политика говорит, что для нее установлено **"Stream",** возможно, она не настроена. Как правило, если политика не настроена, по умолчанию используется **параметр Stream.** Однако при этом изменении, если вы хотите отказаться от использования SharePoint или OneDrive для бизнеса, необходимо сбросить политику на **Stream,** чтобы гарантировать, что по умолчанию используется **Stream.**

```PowerShell
Set-CsTeamsMeetingPolicy -Identity Global -RecordingStorageMode "Stream"
```

## <a name="frequently-asked-questions"></a>Вопросы и ответы

**Где будет храниться запись собрания?**

- Для собраний без канала запись хранится в папке "Записи", которая находится на верхнем уровне OneDrive для бизнеса, принадлежащее человеку, который начал запись собрания.  Пример.

  <i>OneDrive для бизнеса</i> / для записи **Записи**

- Для собраний канала запись хранится в библиотеке документации сайта Teams в папке **"Записи".** Пример.

  <i>Название группы — название канала</i> / **Документы** / **Записи**

**Как это сделать, если файлы Stream (например, записи) хранятся в SharePoint/OneDrive? У администратора есть возможность изменить место ее вехи?**

По умолчанию все файлы записи будут перенаходить в учетную запись OneDrive пользователя, набиравшего **запись.** Для собраний канала запись всегда будет перенаходить на сайт SharePoint канала. Администратор не может изменить место хранения записи.

**Как обрабатывать записи бывших сотрудников?**

Так как видео похожи на другие файлы в OneDrive для бизнеса и SharePoint, обработка владения и хранения после того, как сотрудник уедет, будет следовать обычному процессу OneDrive для [бизнеса и SharePoint.]( https://docs.microsoft.com/onedrive/retention-and-deletion#the-onedrive-deletion-process)

**У кого есть разрешения на просмотр записи собрания?**

- Для собраний без канала все приглашенные, за исключением внешних пользователей, будут автоматически получать личную ссылку. Внешние пользователи должны быть явно добавлены в общий список организатором собрания или пользователем, который запустил запись собрания.

- Для собраний канала разрешения наследуются от списка владельцев и участников канала.

> [!NOTE]
> После сохранения записи вы не получите сообщение электронной почты, но запись появится в чате собрания по ее завершению. Это произойдет намного быстрее, чем в Stream.
> Вы можете управлять тем, кому вы делитесь записью, но не сможете заблокировать загрузку записи людям с общим доступом.  

**Как управлять субтитрами?**

Субтитры для записей собраний Teams будут доступны во время воспроизведения только в том случае, если на момент записи у пользователя была включена транскрибация. Администраторам необходимо [включить запись транскрибации]( https://docs.microsoft.com/microsoftteams/cloud-recording#turn-on-or-turn-off-recording-transcription) с помощью политики, чтобы обеспечить пользователям возможность записи собраний с транскрибцией.

Субтитры помогают создавать инклюзивные материалы для всех пользователей. Владелец может скрыть подписи в записи собрания, хотя расшифровка собрания будет доступна в Teams до тех пор, пока вы не удалите ее. 

Субтитры поддерживаются для записей собраний Teams в течение 60 дней после записи собрания.

Субтитры не полностью поддерживаются, если запись собрания Teams перемещается или копируется из исходного расположения в OneDrive для бизнеса или SharePoint.

> [!NOTE]
> Субтитры будут доступны только на английском языке (транскрибация собрания пока недоступна в GCC).

**Как повлияет квота хранилища?**

Файлы записи собраний Teams находятся в OneDrive для бизнеса и SharePoint и включаются в квоту этих служб. См. [раздел Квота SharePoint](/sharepoint/sites/plan-site-maintenance-and-management#quotas) и [Квота OneDrive для бизнеса.](/onedrive/set-default-storage-space)

Благодаря [OneDrive](/office365/servicedescriptions/sharepoint-online-service-description/sharepoint-online-limits) для бизнеса вы получаете больше места для хранения данных, чем Stream, и более интересные хранилища с SharePoint.

**Как записать собрание Teams?**

В зависимости от того, где вы имеете доступ к файлу, видео будет воспроизведения в видеоиграмме OneDrive для бизнеса или SharePoint.

**Если вы планируете не использовать возможность добавления в Stream, будут ли существующие видео оставаться в том же и в течение долгого времени?**

В ближайшем будущем поток как платформа не будет отрешен. Видео, которые в настоящее время находятся в Stream, останутся там, пока мы не начнем миграцию. После миграции эти видео также будут перенесены в OneDrive для бизнеса или SharePoint. Дополнительные [сведения можно получить в классической](/stream/streamnew/classic-migration) миграции Stream.

См. [автоматическое применение метки хранения.](/microsoft-365/compliance/apply-retention-labels-automatically?view=o365-worldwide#microsoft-teams-meeting-recordings)

См. [также, какая политика имеет приоритет?](./assign-policies.md#which-policy-takes-precedence).

**Куда идет запись, если у пользователя нет OneDrive для бизнеса или SharePoint?**

Запись будет расположена во временном хранилище, где она будет проводиться в течение 21 дня. В это время организатор должен скачать запись. Если запись не была загружена в течение 21 дня, она будет удалена.
