Microsoft Teams есть новый способ сохранения записей собраний. В качестве первого этапа перехода от классической программы Microsoft Stream к новому потоку этот метод сохраняет записи в Microsoft OneDrive для бизнеса и SharePoint в Microsoft 365 и предоставляет множество преимуществ. [](/stream/streamnew/new-stream)

> [!NOTE]
> Если запись Teams не удается загрузить в OneDrive/SharePoint, запись будет временно сохранена в Службы мультимедиа Azure AMS. После хранения в AMS повторные попытки автоматической отправки записи в OneDrive/SharePoint или Stream.

Записи собраний, хранимые в AMS, доступны в течение 21 дней перед автоматическим удалением. Пользователи могут скачать видео из amS, если им нужно сохранить копию.

Преимущества использования OneDrive для бизнеса и SharePoint для хранения записей:

- Политики хранения для Teams собраний (TMR) (автоматические метки S+C E5)
- Преимущества OneDrive для бизнеса и SharePoint управления сведениями
- Простое настройка разрешений и общего доступа
- Поделиться записями с гостями (внешними пользователями) только с явным доступом
- Запрос потока доступа
- Предоставление OneDrive для бизнеса и SharePoint ссылки
- Записи собраний доступны быстрее
- **Перейти в службу поддержки локального** клиента
- Поддержка нескольких регионов: записи хранятся в конкретном регионе для этого пользователя
- Поддержка собственного ключа (BYOK)

См. полный список [функций, доступных сегодня, и о том, чего ожидать с течением времени.](/stream/streamnew/features-new-version-stream) 

Дополнительные сведения можно получить в Microsoft Teams "Новые возможности записи собраний".

## <a name="set-up-the-meeting-recording-option-for-onedrive-for-business-and-sharepoint"></a>Настройка параметра записи собрания для OneDrive для бизнеса и SharePoint

Параметр записи собрания находится на уровне политики Teams собрания. В следующем примере показано, как настроить глобальную политику. Убедитесь, что вы настроите параметр записи собрания для политик, которые вы настроите для пользователей.

> [!Note]
> Teams изменения политики собраний распространяются через некоторое время. Повторите попытку через несколько часов после его настройки, а затем вы выходите из системы и снова во Teams или просто перезагрузите компьютер.

1. Установите Teams PowerShell.

   > [!NOTE]
   > Skype для бизнеса Online Connector в настоящее время является частью последней версии Teams PowerShell. Если вы используете последнюю версию Teams PowerShell, вам не нужно устанавливать Skype для бизнеса Online Connector. См. [управление Skype для бизнеса Online с помощью PowerShell.](/microsoft-365/enterprise/manage-skype-for-business-online-with-microsoft-365-powershell?preserve-view=true&view=o365-worldwide)

2. Запустите PowerShell в качестве администратора.

3. Установите [Teams PowerShell](./teams-powershell-install.md).

4. Импорт модуля MicrosoftTeams и вход в Teams администратора.


   ```powershell
   # When using Teams PowerShell Module
   
   Import-Module MicrosoftTeams
   Connect-MicrosoftTeams
   ```

5. Используйте [Set-CsTeamsMeetingPolicy,](/powershell/module/skype/set-csteamsmeetingpolicy) чтобы настроить политику Teams собраний для перехода с хранилища Stream на OneDrive для бизнеса и SharePoint.

   ```powershell
   Set-CsTeamsMeetingPolicy -Identity Global -RecordingStorageMode "OneDriveForBusiness"
   ```

> [!Note]
> Если некоторым из ваших пользователей назначена политика "на пользователя" или "на пользователя", необходимо установить этот параметр для этой политики, если вы хотите, чтобы они также могли хранить записи собраний в OneDrive для бизнеса и SharePoint. Дополнительные сведения см. в [этой](meeting-policies-in-teams.md)Teams.

## <a name="opt-out-of-onedrive-for-business-and-sharepoint-to-continue-using-stream"></a>Отказ от использования OneDrive для бизнеса и SharePoint для использования Stream

Даже если политика говорит, что для нее установлено **"Stream",** возможно, она не настроена. Как правило, если политика не настроена, по умолчанию используется **параметр Stream.** Однако при этом новом изменении, если вы хотите отказаться от SharePoint или OneDrive для бизнеса, необходимо сбросить политику на **Stream,** чтобы гарантировать, что по умолчанию используется **Stream.**

```PowerShell
Set-CsTeamsMeetingPolicy -Identity Global -RecordingStorageMode "Stream"
```

## <a name="frequently-asked-questions"></a>Вопросы и ответы

**Где будет храниться запись собрания?**

- Для собраний, не в каналах, запись хранится в папке "Записи", которая находится на верхнем уровне записи OneDrive для бизнеса, которая принадлежит человеку, который начал запись собрания.  Пример.

  <i>запись OneDrive для бизнеса</i> / **Записи**

- Для собраний канала запись хранится в библиотеке Teams сайта в папке **"Записи".** Пример.

  <i>Teams — название канала</i> / **Документы** / **Записи**

**Как это сделать, если файлы Stream (например, записи) хранятся в SharePoint/OneDrive, как это сделать? У администратора есть возможность изменить место ее вехи?**

По умолчанию все файлы записи будут OneDrive учетную запись пользователя, который выбрал **запись.** Для собраний канала запись всегда будет SharePoint сайте канала. Администратор не может изменить место хранения записи.

**Как обрабатывать записи бывших сотрудников?**

Так как видео похожи на другие файлы в OneDrive для бизнеса и SharePoint, обработка владения и хранения после того, как сотрудник уедет, будет выполняться обычный OneDrive для бизнеса и [SharePoint.](/onedrive/retention-and-deletion#the-onedrive-deletion-process)

**Кто есть разрешения на просмотр записи собрания?**

- Для собраний без канала все приглашенные, за исключением внешних пользователей, автоматически получают личную ссылку. Внешние пользователи должны быть явно добавлены в общий список организатором собрания или пользователем, который запустил запись собрания.

- Для собраний канала разрешения наследуются от списка владельцев и участников канала.

> [!NOTE]
> После сохранения записи вы не получите сообщение электронной почты, но запись появится в чате собрания по ее завершению. Это произойдет намного быстрее, чем в Stream.

**Как управлять субтитрами?**

Субтитры для Teams собраний будут доступны во время воспроизведения только в том случае, если на момент записи у пользователя была включена транскрибация. Администраторам необходимо [включить запись транскрибации](/microsoftteams/cloud-recording#turn-on-or-turn-off-recording-transcription) с помощью политики, чтобы обеспечить пользователям возможность записи собраний с транскрибцией.

Субтитры помогают создавать инклюзивные материалы для всех пользователей. Владелец может скрыть субтитры в записи собрания, но расшифровка собрания будет доступна в Teams пока вы не удалите ее. 

Субтитры поддерживаются для Teams собраний в течение 60 дней после записи собрания.

Субтитры не полностью поддерживаются, если Teams запись собрания перемещается или копируется из исходного расположения на OneDrive для бизнеса или SharePoint.

> [!NOTE]
> Субтитры будут доступны только на английском языке (запись разговоров пока недоступна в GCC).

**Как повлияет квота хранилища?**

Teams записи собраний в OneDrive для бизнеса и SharePoint и включаются в квоту этих служб. См. [SharePoint квоты и](/sharepoint/sites/plan-site-maintenance-and-management#quotas) [OneDrive для бизнеса квоты.](/onedrive/set-default-storage-space)

Вы получите больше места для [хранения](/office365/servicedescriptions/sharepoint-online-service-description/sharepoint-online-limits) данных OneDrive для бизнеса чем Stream, и более интересные хранилища с SharePoint.

**Как записать Teams собрания?**

Ваше видео будет играть в видеоигравте OneDrive для бизнеса или SharePoint в зависимости от того, где вы имеете доступ к файлу.

**Если вы планируете не использовать возможность добавления в Stream, будут ли существующие видео оставаться в том же и в течение долгого времени?**

В ближайшем будущем поток как платформа не будет отрешен. Видео, которые в настоящее время находятся в Stream, останутся там, пока мы не начнем миграцию. После миграции эти видео также будут перенесены в OneDrive для бизнеса или SharePoint. Дополнительные [сведения можно получить в классической](/stream/streamnew/classic-migration) миграции Stream.

См. [автоматическое применение метки хранения.](/microsoft-365/compliance/apply-retention-labels-automatically?view=o365-worldwide#microsoft-teams-meeting-recordings)

**Как назначить политики пользователям в Microsoft Teams и какие политики имеют приоритет?**

См. [также, какая политика имеет приоритет?](./assign-policies.md#which-policy-takes-precedence).

**Куда идет запись, если у пользователя нет записи OneDrive для бизнеса или SharePoint или квота хранилища заполнена?**

Запись будет расположена во временном хранилище, где она будет проводиться в течение 21 дня. В это время организатор должен скачать запись. Если запись не была скачина в течение 21 дня, она будет удалена.
