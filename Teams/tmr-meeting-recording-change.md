В Microsoft Teams существует новый метод сохранения записей собраний. В качестве первого этапа перехода от классической программы Microsoft Stream к новому потоку этот метод сохраняет записи в Microsoft OneDrive для бизнеса и SharePoint в Microsoft 365 и предоставляет множество преимуществ. [](/stream/streamnew/new-stream)

> [!NOTE]
> Если запись Teams не удается загрузить в OneDrive/SharePoint, появится сообщение об ошибке "Запись неожиданно завершена", а запись будет временно сохранена в Службы мультимедиа Azure (AMS). После хранения в AMS повторные попытки автоматической отправки записи в OneDrive/SharePoint или Stream.

Записи собраний, сохраненные в AMS, доступны в течение 21 дня, а затем автоматически удаляются. При необходимости сохранить копию пользователи могут скачать видео из AMS.

Преимущества использования OneDrive для бизнеса и SharePoint для хранения записей:

- Политики хранения для записи собраний Teams (TMR) (метки автоматического хранения S+C E5)
- Преимущества OneDrive для бизнеса и SharePoint информации
- Простота настройки разрешений и общего доступа
- Делитесь с гостями (внешними пользователями) записями, только явно предоставляя общий доступ
- Поток запроса доступа
- Предоставление OneDrive для бизнеса и SharePoint ссылки
- Записи собраний доступны быстрее
- Поддержка **локального** клиента
- Поддержка нескольких регионов — записи хранятся в регионе, определенном для этого пользователя
- Поддержка создания собственных ключей (BYOK)

См. полный список [возможностей, доступных в настоящее время и ожидаемых в будущем](/stream/streamnew/features-new-version-stream).

Дополнительные сведения см. в статье "Новые возможности записей собраний в Microsoft Teams".

## <a name="set-up-the-meeting-recording-option-for-onedrive-for-business-and-sharepoint"></a>Настройка параметра записи собрания для OneDrive для бизнеса и SharePoint

Параметр записи собраний — это параметр на уровне политики Teams. В приведенном ниже примере показано, как задать глобальную политику. Убедитесь, что вы задаете параметр записи собраний для политики или политик, которые вы назначили пользователям.

> [!Note]
> Распространение изменений политики собраний Teams займет некоторое время. Выполните проверку через несколько часов после настройки, а затем выйдите из системы и повторно войдите в классическое приложение Teams или просто перезагрузите компьютер.

1. Установка PowerShell в Teams PowerShell

   > [!NOTE]
   > Соединитель Skype для бизнеса Online в настоящее время является частью последнего модуля Teams PowerShell. Если вы используете последний общедоступный выпуск Teams PowerShell, вам не нужно устанавливать соединитель Skype для бизнеса Online. См. статью [Управление Skype для бизнеса Online с помощью PowerShell](/microsoft-365/enterprise/manage-skype-for-business-online-with-microsoft-365-powershell?preserve-view=true&view=o365-worldwide)

2. Запустите PowerShell от имени администратора.

3. Установите модуль [Teams PowerShell](./teams-powershell-install.md).

4. Импортируйте модуль MicrosoftTeams и выполните вход в качестве администратора Teams.

   ```powershell
   # When using Teams PowerShell Module
   
   Import-Module MicrosoftTeams
   $credential = Get-Credential
   Connect-MicrosoftTeams -Credential $credential
   ```

5. Используйте [Set-CsTeamsMeetingPolicy,](/powershell/module/skype/set-csteamsmeetingpolicy) чтобы настроить политику Teams собраний для перехода из хранилища Stream в OneDrive для бизнеса и SharePoint.

   ```powershell
   Set-CsTeamsMeetingPolicy -Identity Global -RecordingStorageMode "OneDriveForBusiness"
   ```

> [!Note]
> Если некоторым из ваших пользователей назначена политика "на пользователя" или "на пользователя", необходимо установить этот параметр для этой политики, если вы хотите, чтобы они также могли хранить записи собраний в OneDrive для бизнеса и SharePoint. Дополнительные сведения см. в статье [Управление политиками собраний в Teams](meeting-policies-in-teams.md).

## <a name="opt-out-of-onedrive-for-business-and-sharepoint-to-continue-using-stream"></a>Отказ от использования OneDrive для бизнеса и SharePoint для использования Stream

Даже если политика говорит, что для нее установлено **"Stream",** возможно, она не настроена. Как правило, если политика не настроена, по умолчанию используется **параметр Stream.** Однако при этом новом изменении, если вы хотите отказаться от SharePoint или OneDrive для бизнеса, необходимо сбросить политику в **Stream,** чтобы гарантировать, что по умолчанию используется **Stream.**

```PowerShell
Set-CsTeamsMeetingPolicy -Identity Global -RecordingStorageMode "Stream"
```

## <a name="frequently-asked-questions"></a>Вопросы и ответы

**Где будет храниться запись собрания?**

- Для собраний, не в каналах, запись хранится в папке "Записи", которая находится на верхнем уровне OneDrive для бизнеса записи, которая принадлежит человеку, который начал запись собрания.  Пример.

  <i>запись OneDrive для бизнеса</i> / **Записи**

- Для собраний канала запись хранится в библиотеке Teams сайта в папке **Записи.** Пример.

  <i>Teams — название канала</i> / **Документы** / **Записи**

**Как это сделать, если файлы Stream (например, записи) хранятся в SharePoint/OneDrive, как это сделать? У администратора есть возможность изменить место ее вехи?**

По умолчанию все файлы записи будут OneDrive учетную запись пользователя, который выбрал **запись.** Для собраний канала запись всегда будет SharePoint сайте канала. Администратор не может изменить место хранения записи.

**Как обрабатывать записи бывших сотрудников?**

Так как видео похожи на другие файлы в OneDrive для бизнеса и SharePoint, обработка владения и хранения после того, как сотрудник уедет, будет выполняться обычный OneDrive для бизнеса и [SharePoint.](/onedrive/retention-and-deletion#the-onedrive-deletion-process)

**Кто есть разрешения на просмотр записи собрания?**

- Для собраний без канала все приглашенные, за исключением внешних пользователей, автоматически получают личную ссылку. Внешние пользователи должны быть явно добавлены в общий список организатором собрания или пользователем, который запустил запись собрания.

- Для собраний канала разрешения наследуются от списка владельцев и участников канала.

> [!NOTE]
> После сохранения записи вы не получите сообщение электронной почты, но запись появится в чате собрания по ее завершению. Это произойдет намного быстрее, чем в Stream.

**Как управлять субтитрами?**

Субтитры для записей собраний Teams будут доступны во время воспроизведения, только если во время записи у пользователя было включено транскрибирование. Администраторы должны [включить транскрибирование записей с помощью политики](/microsoftteams/cloud-recording#turn-on-or-turn-off-recording-transcription), чтобы пользователи могли записывать собрания с транскрибированием.

Субтитры помогают создавать инклюзивные материалы для всех зрителей. Владелец может скрыть субтитры в записи собрания, хотя расшифровка собрания по-прежнему будет доступна в Teams, если вы не удалите ее там.

Субтитры поддерживаются для Teams собраний в течение 60 дней после записи собрания.

Субтитры не полностью поддерживаются, если Teams запись собрания перемещается или копируется из исходного расположения на OneDrive для бизнеса или SharePoint.

> [!NOTE]
> Субтитры доступны только на английском языке (транскрибирование собраний пока недоступно в GCC).

**Как повлияет квота хранилища?**

Teams записи собраний в OneDrive для бизнеса и SharePoint и включаются в квоту этих служб. См. [SharePoint квоты](/sharepoint/sites/plan-site-maintenance-and-management#quotas) [и OneDrive для бизнеса квоты.](/onedrive/set-default-storage-space)

Вы получаете больше места в [хранилище, OneDrive для бизнеса](/office365/servicedescriptions/sharepoint-online-service-description/sharepoint-online-limits) чем Stream, и более интересные хранилища с SharePoint.

**Как записать Teams собрания?**

Видео будет играть в видеоигравте OneDrive для бизнеса или SharePoint в зависимости от того, где вы имеете доступ к файлу.

**Если вы планируете не использовать возможность добавления в Stream, будут ли существующие видео оставаться в том же и в течение долгого времени?**

В ближайшем будущем поток как платформа не будет отрешен. Видео, которые в настоящее время находятся в Stream, останутся там, пока мы не начнем миграцию. После миграции эти видео также будут перенесены в OneDrive для бизнеса или SharePoint. Дополнительные [сведения можно получить в классической](/stream/streamnew/classic-migration) миграции Stream.

**Как применить метку хранения к Microsoft Teams собраний?**

См. [автоматическое применение метки хранения.](/microsoft-365/compliance/apply-retention-labels-automatically?view=o365-worldwide#microsoft-teams-meeting-recordings)

**Как назначить политики пользователям в Microsoft Teams и какие политики имеют приоритет?**

См. [также, какая политика имеет приоритет?](./assign-policies.md#which-policy-takes-precedence).

**Куда идет запись, если у пользователя нет записи OneDrive для бизнеса или SharePoint или квота хранилища заполнена?**

Запись будет расположена во временном хранилище, где она будет проводиться в течение 21 дня. В это время организатор должен скачать запись. Если запись не была загружена в течение 21 дня, она будет удалена.
