# <a name="make-my-service-decisions"></a>Мои проведения службы

Чтобы спланировать технического внедрения телефонной системы с помощью вызова планы, необходимо принять ряд службы решений, чтобы лучше подготовить вашей организации для реализации решения, отвечающую определенных бизнес-требования.

## <a name="calling-in-teams"></a>Звонки в Teams

С группами Майкрософт пользователей можно поместить и принимать телефонные звонки или из телефонной сети общего пользования (PSTN). Пользователей можно использовать собственные выделенные номера телефонов для выполнения и приема внутренних и международных телефонных звонков из группы клиентских приложений с помощью расширенных функций, включая голосовую почту.

> [!NOTE]
> Последняя версия плана группами для идентификации групп телефонной системы с использованием вызова планирование функций в области действия для развертывания, можно найти в <https://aka.ms/skype2teamsroadmap>.

## <a name="phone-system-in-teams"></a>Телефонной системой в группах

Для группы Пользователи должны иметь возможность выполнять и принимать звонки по ТСОП они должны быть включена поддержка телефонной системой, компонент в Office 365.

Чтобы включить возможность подключения к ТСОП, вашей организации можно использовать Microsoft как его поставщика телекоммуникационных услуг. В конечном счете вам также придется параметр «создайте свои собственные» поставщика телекоммуникационных услуг, чтобы включить возможность подключения к ТСОП для телефонной системой.

> [!IMPORTANT]
> Возможность использования поставщиком услуг телекоммуникации для телефонной системой с развертыванием группы будут доступны при прямой маршрутизации телефонной системы. Для получения дополнительных сведений о прямой маршрутизации, просмотрите [руководство прямой маршрутизации](2-envision-make-my-service-decisions-direct-routing.md).

## <a name="phone-system-with-calling-plans"></a>Телефонная система с планами звонков

Чтобы использовать Microsoft как вашей области телекоммуникационных услуг поставщика услуг, необходимо получить вызов планирование лицензии и назначьте их пользователям телефонной системой.

Существует два основных типа Тарифные планы.

-   Внутренние телефонной

-   Внутреннее и международное телефонной

Каждый тип телефонной выделяет определенное количество минут звонков в месяц для каждого пользователя, которому назначена лицензия. При закончится распределения минут звонок, пользователь не сможет для исходящих звонков, за исключением экстренных вызовов — до следующего месяца выставления счета цикла. Если требуется, чтобы пользователи могли продолжать для исходящего вызова даже после они исчерпано их выделении минут звонок или сообщите пользователям, имеющим внутренних телефонной международные звонки, можно настроить кредитов коммуникаций для вашей организации.

## <a name="phone-numbers-and-emergency-locations"></a>Номера телефонов и местоположения для экстренного реагирования

Просмотрите [Управление облачных голосовой связи телефонных номеров](https://docs.microsoft.com/MicrosoftTeams/2-envision-make-my-service-decisions-phone-system#manage-cloud-voice-telephone-numbers) для планирования приобретения номер телефона для реализации вызова планы.

При настройке номера телефонов для вызова планы, необходимо назначить экстренных адрес для каждого номера телефона перед номером назначить пользователю. Это требуется сделать для поддержки экстренных вызовов. Чтобы убедиться, что она находится в правильном формате, используемый службами действия в чрезвычайных ситуациях необходимо проверить экстренных адрес.

> [!IMPORTANT]
> Экстренные службы вызов работает по-разному в службе вызов планы чем в традиционных телефонией. Важно понять различия и сообщения о них заинтересованным ко всем пользователям. Для получения дополнительных сведений см [аварийного вызов сроками и условиями](https://docs.microsoft.com/skypeforbusiness/legal-and-regulatory/emergency-calling-terms-and-conditions) .

Таким местоположением может быть номер здания, этаж, крыло здания или номер офиса, где работает пользователь.

Для получения дополнительных сведений о местоположении для экстренных служб при использовании вызова планы, ознакомьтесь со следующими статьями:

## <a name="voicemail"></a>Голосовая почта

Система голосовой почты телефона, на базе служб Azure голосовой почты, поддерживает вклады голосовой почты в почтовые ящики Exchange только и не поддерживает систем электронной почты сторонних производителей.

По умолчанию телефонной системы голосовой почты для работы с Exchange Online; Тем не менее имеет минимальные поддерживаемые Exchange в локальной версии и развертывания модели, чтобы разрешить доставки сообщений голосовой почты для почтовых ящиков пользователей в локальном развертывании Exchange.

Голосовая почта Телефонной системы включает функцию расшифровки голосовых сообщений, которая по умолчанию включена для всех пользователей в организации. Бизнес-требованиям может потребоваться отключить транскрибирования голосовой почты для определенных пользователей или всем пользователям в организации. Если ваша организация решение сохранить транскрибирования голосовая почта включена, необходимо также принять во внимание необходимо включить режим маски сквернословие транскрибирования голосовой почты. Для получения дополнительных сведений см [настройке политик голосовой почты в вашей организации](https://docs.microsoft.com/SkypeForBusiness/what-is-phone-system-in-office-365/phone-system-voicemail/set-up-phone-system-voicemail) .

>[!NOTE]
> Был реализован резервный механизм, чтобы служба голосовой почты Телефонной системы могла повторно отправлять сообщения по протоколу SMTP. Это значит, что пользователи с почтовым ящиком в сторонней почтовой системе смогут получать адресованные им сообщения голосовой почты. Этот механизм не содержит время работы гарантированной службы или других функций голосовой почты, такие как изменение приветствия голосовой почты.

Дополнительные сведения о голосовой почты в реализации телефонной системы [голосовой почты УАТС Azure поддержки для Exchange Server](https://docs.microsoft.com/MicrosoftTeams/phone-system-with-calling-plans#licensing-for-calling-plans)см.

## <a name="licensing-for-cloud-voice-capabilities"></a>Лицензирование для облачных возможности голосовой связи

Аудио- и телефонной системы конференц-связи являются компонентами Office 365. Они могут иметь лицензию отдельно как дополнительный компонент службы существующих пользователям Office 365 E3 или планы подписки E1; они уже включены в план подписки Office 365 E5.

Тарифные планы дополнительного компонента к телефонной системой функции в Office 365, поэтому необходимо иметь телефонной системы с корпоративным лицензированием разрешено использовать вызов планов.

Для поддержки для конференц-связи дополнительные аудио и вызов планы использование случаях (Международные конференции по телефону, внешний вызов после исчерпано вызов планирование minute выделений и т. д.), кредитов коммуникации можно настроить для вашей организации.

## <a name="licensing-for-calling-plans"></a>Лицензирование для планов звонков

Если организация планирует на использование Microsoft как поставщика телекоммуникационных услуг, необходимо получить вызов планирование надстройки подходят для пользователей бизнес-требований. Как правило не всем пользователям в организации необходимо поместить международные звонки, поэтому вы можете подготовить большинство пользователей с внутренних лицензий на вызов планирование.

Существует два типа лицензий для планов звонков:

-   план для местных звонков;

-   план для международных и местных звонков.

> [!NOTE]
> Местные звонки определяются назначенным пользователю расположением использования Office 365.

Каждый тип плана звонков назначает пользователям определенное количество минут в месяц для совершения местных или международных вызовов. Планирование внутренний вызов расходы, которые меньше по сравнению с International и планирование внутренний вызов.

Гибкость подписки и назначение наиболее подходящий тип вызова планирование для отдельных пользователей бизнес-требований поможет вашей организации расходы его реализация вызов планы.

Для каждого клиента Office 365 общее количество минут звонков объединяется по стране или региону и по типу плана звонков. При достижении клиентом месячного лимита минут действие службы планов звонков (кроме экстренных вызовов) будет приостановлено на оставшуюся часть месяца. Перезапуск службы вызов планы будут автоматически с первого дня следующего месяца календаря.

Можно настроить Communications кредит для вашей организации, чтобы разрешить пользователям выполнять исходящие вызовы после распределения вызова минут закончится без ожидания до следующего месяца выставления счетов цикла. Кроме того, кредитов коммуникаций позволяют пользователям назначить внутренний вызов планирование возможность международных звонков, которые затем оплачиваемые с помощью модели «оплаты в минуту».

## <a name="communications-credits"></a>Кредиты на связь

Использование кредитов коммуникаций, пользователи могут исходящих звонков на собрании аудиоконференции для добавления другого пользователя из в мире (за пределами отправляющего страны Организатор собрания). ## <a name="transfer-existing-telephone-numbers"></a>Перенос существующих телефонных номеров

## <a name="dial-plans"></a>Абонентские группы

Абонентская группа в компоненте телефонной системой Office 365 — это набор правил нормализации, переводить набора телефонных номеров в альтернативный формат (обычно формат E.164) для маршрутизации вызовов и проверка подлинности вызовов. Служба конференц-связи звук использует те же возможности, используемые с телефонной системой для перевода набранного номера телефонов в сценариях конференции по телефону (например, пригласить участников через PSTN и снова, вызова функции «позвонить мне»).

В компоненте телефонной системой Office 365 существует два типа абонентских групп:

-   **Службы единой системы обмена сообщениями:** Это значение по умолчанию абонентской группы, которая применяется для пользователей на основе Office 365 использования расположения и его нельзя изменить.

-   **Клиента абонентская группа:** Это настраиваемый абонентской группы клиента, который разделить на два типа:

    -   **Глобального клиента абонентской группы:** Единой системы обмена сообщениями, которое применяется ко всем пользователям в клиента.

    -   **Абонентской группы пользователей клиента:** Единой системы обмена сообщениями, которое применяется только к определенным пользователям.

Эффективный абонентскую группу назначены пользователям представляет собой комбинацию абонентской службы (на основе пользователя Office 365 использования местоположения) и клиента абонентская группа (который может быть глобального клиента абонентской группы или абонентской группы пользователей клиента).
