# <a name="create-your-first-teams-in-microsoft-teams"></a>Создание первых команд в Microsoft Teams

Лучший способ внедрения Teams — использовать продукт. Мы рекомендуем управлять развертыванием Microsoft 365 или Office 365 с помощью Teams. Кроме того, следует создать сообщество сотрудников, которые ориентированы на командную работу и производительность. (Эти начальные команды со временем могут меняться.) Ниже перечислены некоторые рекомендуемые команды.

| Название команды | Предложенные имена каналов |
| --------- | ---------------------- |
| Знакомство с Teams | Общий</br> Поздороваться</br> Разделы справки?</br>Отзывы </br> Идеи для будущих команд </br> Поддержка |
| Реализация Microsoft Teams | Общий <br/> Осведомленность и внедрение <br/> Взаимодействие с бизнесом <br/> Программа ранних последователей <br/> Отзывы и аналитические сведения <br/> Безопасность и соответствие требованиям <br/> Стратегия и планирование <br/> Работоспособность служб и инциденты <br/> Учебный курс <br/> Чат Watercooler |
| Лидеры командной работы | Общий <br/> Рекомендации, обучение и практическое руководство <br/> Угол лидеров <br/> Отзывы и поддержка <br/> Потенциальные клиенты программы <br/> Центр социальных сетей |

В Microsoft Teams библиотека вкладок предоставляет доступ ко всем приложениям Microsoft 365 или Office 365 и сторонним приложениям, которые были включены администратором.[](/microsoftteams/platform/concepts/tabs/tabs-overview) Чтобы получить максимальное значение от вашего продукта, мы рекомендуем закрепить ключевые ресурсы в качестве вкладок в каждом из соответствующих каналов. См. пример ниже.

![Снимок экрана: стандартные и настраиваемые вкладки.](media/teams-adoption-tab-example.png)

В начальных командах мы рекомендуем закрепить следующие элементы, чтобы легко ссылаться на них в каналах группы. Эти ресурсы являются дополнением к разделам записной книжки OneNote. Если вы работаете с нашей командой FastTrack, они помогут вам настроить эти первые команды. 

|Имя команды или канала | Закрепленная вкладка |
|----------------- | ---------- |
| **Развертывание Microsoft Teams** ||
| Общий | Веб-вкладка: [документация по Teams](./index.yml) |
| Осведомленность и внедрение | [Центр внедрения Teams](https://aka.ms/DriveTeamsAdoption)<br/>[Комплект для успешной работы клиентов Microsoft Teams](https://aka.ms/TeamsCustomerSuccess)|
| Взаимодействие с бизнесом | Вики-сайт — руководство по задействованию<br/>Форма — запрос на консультацию в Teams<br/>List — Engagement requests |
|Ранние последователей | Закрепленное сообщество отзывов <br/> OneNote для часто задаваемых вопросов и распространенных тем |
| Отзывы и отчеты | Закрепленный отчет об использовании Microsoft 365 или Office 365 (только для администраторов Microsoft 365 или Office 365) |
| Безопасность и соответствие требованиям | Портал Microsoft Trust <br/> [Документация по безопасности и соответствию требованиям](/office365/securitycompliance/index)<br/> ["Roadmap" (Дорожная карта)](/office365/securitycompliance/security-roadmap) |
| Стратегия и планирование | Совместная работа в North Star PowerPoint <br/> План проекта внедрения службы |
| Учебный курс | Персонализированное обучение обучения <br/> [Онлайн-обучение Teams](https://aka.ms/TeamsTraining) |
| **Лидеры командной работы**|  |
| Общий | Использование для объявлений о программе <br/> Общие сведения о программе закрепления лидеров |
| Рекомендации | Рекомендации по командам, каналам и чатам PowerPoint <br/> Жизненный цикл команды |
| Угол лидеров | Заметки к собранию и презентации |
| Центр социальных сетей | Соединитель RSS: блог Teams и блог по внедрению |
| Обучение и практическое руководство | Ссылка на портал обучения |

## <a name="making-the-switch-to-microsoft-teams"></a>Переход на Microsoft Teams

Когда вы начнете использовать Teams, вам и остальным участникам команды проекта будет важно зафиксировать использование чата, каналов и ресурсов Teams для запуска проекта. Ваше использование Teams имеет решающее значение для качества проекта внедрения. Рекомендуется не фрагментировать проект, продолжая беседы с основной командой проекта по электронной почте. Перейдите в Teams, чтобы моделировки поведения, которые в конечном итоге должны принять ваши сотрудники. Задайте тон, сначала перейдите и начните пользоваться информацией о проекте в одном месте!
