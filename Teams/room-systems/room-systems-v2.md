# <a name="deployment-overview"></a>Обзор развертывания

Развертывание комнат группами Майкрософт по сути разделяет этапов:

- Убедившись, что ваше развертывание расположения (комнат) соответствуют зависимости развертывания
- Создание учетных записей Exchange и бизнес-группами Майкрософт или Скайп и назначение их на консоли устройства (см. [Настройка учетных записей для комнат группами Майкрософт](room-systems-v2-configure-accounts.md))
- Осуществлении Microsoft Surface планшетных компьютеров для работы в качестве консоли комнат группами Майкрософт (см. [Настройка консоли комнат группами Майкрософт](console.md) или [Развернуть комнат группами Майкрософт массовое руководство по развертыванию](room-systems-scale.md))
- (Необязательно) Настройка Microsoft Operations Management Suite (см. [Управление развертывание комнат группы Microsoft Azure монитор](azure-monitor-deploy.md) систем
- Настройка консоли в конференц-залы и подключение периферийных устройств необходимо (обратитесь к документации ПВТ для набора устройств)

Полное аудио/видео можно использовать для последнего задач, но вашей организации необходимо сделать другие части процесса ИТ-подразделения.

### <a name="prepare-to-host-microsoft-teams-rooms-machine-and-service-accounts-optional"></a>Подготовка к узла компьютера комнат группами Майкрософт и (необязательно) учетные записи служб

Чтобы включить управление и создание отчетов по вашей комнат группами Майкрософт компьютера и служебных учетных записях, Подготовка своей локальной службы каталогов Active Directory или Azure Active Directory (Azure AD). 

Определения локальной группы Active Directory или Azure AD для добавления всех учетных записей служб (пользователь) комнат группами Майкрософт для, а затем создайте отчеты об использовании с помощью командлета Get-CSUserSession PowerShell в развертывании комнат группами Майкрософт. Например можно создайте группу с именем учетных записей служб SkypeRoomSystemsv2. 


Определение одного подразделения в локальной иерархии Active Directory или Azure AD для хранения всех учетных записей компьютеров комнат группами Майкрософт (если они будет присоединен к домену) и одного подразделения для хранения всех учетных записей пользователей комнат группами Майкрософт. При создании подразделение для учетных записей компьютеров комнат группами Майкрософт, вы можете отключить наследования, чтобы убедиться, что применяются только политик, которые предназначены для применения к комнат присоединенный к домену группы Microsoft. 

Создайте объект групповой политики, назначенные подразделение, содержащее учетные записи компьютера комнат группами Майкрософт. Используется для: 

-   [Настройка power и параметры локальной учетной записи](room-systems-v2-operations.md#configuring-group-policy-for-microsoft-teams-rooms).
-   Включите Центр обновления Windows.
-   Включение удаленного взаимодействия PowerShell. Можно настроить при запуске сценария простого скрипта: Enable-PSRemoting - Force

PowerShell можно использовать для выполнения ряда операции удаленного управления, включая считывание и запись сведений о конфигурации. Об удаленном взаимодействии PowerShell должна быть включена *перед* любой PowerShell может занять от удаленного управления поместите и следует рассматривать как части процессов развертывания или настроен с помощью групповой политики. Дополнительные сведения об этих возможностей, а также включение их можно [обслуживание и операции](room-systems-v2-operations.md#remote-management-using-powershell).

### <a name="account-provisioning"></a>Учетная запись подготовки 

Каждое устройство комнат группами Майкрософт требуется учетная запись выделенного и уникальных ресурса, который должен быть включен для групп Майкрософт или Скайп для бизнеса и Exchange. Эта учетная запись должна иметь почтовый ящик помещения, размещенных на сервере Exchange и включить как конференц-зала в группы или Скайп по развертыванию. Обработка календаря на стороне Exchange, необходимо настроить, чтобы устройство автоматически принимает входящие приглашения. Дополнительные сведения о создании этих учетных записей, содержатся в разделе [Настройка учетных записей для комнат группами Майкрософт](room-systems-v2-configure-accounts.md). 

**Pro совет** – следует отображение имен для этих учетных записей описательное и понятный. Далее представлены имена, видят пользователи при поиск и добавление систем Microsoft группами комнат на собрания. В некоторых организациях используется соглашение о *сайте*-*Имени комнаты*(*Max комнаты емкость*)-RS, например Curie — 12 человека конференц-зала в Лондон — может иметь Встроенный CURIE (12)-RS отображаемое имя. 

Если в организации имеется много конференц-залы, которые требуют нескольких, подготовленные учетные записи, можно использовать [Сценарии подготовки Скайп комнаты систем учетных записей](https://myadvisor.fasttrack.microsoft.com/CloudVoice/Downloads?SelectedIDs=5_2_0_4,5_2_0_5) для массового provision несколько учетных записей службы в автоматическом режиме.

В процессе развертывания необходимо также рассмотрим стратегии для управления и настройки [локальных учетных записей](https://docs.microsoft.com/skypeforbusiness/plan-your-deployment/clients-and-devices/skype-room-systems-v2-0#local-accounts) , созданных программой установки приложения комнат группами Майкрософт.

Мы рекомендации по использованию [Microsoft Azure Monitor](https://docs.microsoft.com/skypeforbusiness/plan-your-deployment/clients-and-devices/azure-monitor) для мониторинга развертывания Microsoft группами комнат и отчет о доступности, ошибки оборудования и программного обеспечения и версия приложения комнат группами Майкрософт.
