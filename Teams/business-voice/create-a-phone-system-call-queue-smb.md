# <a name="create-a-call-queue---small-business-tutorial"></a>Создание очереди вызовов — учебник для малого бизнеса

Очереди вызовов предоставляют способ маршрутизации вызывающих вызовов для людей в организации, которые могут помочь с определенной проблемой или вопросом. Звонки распределяются по одному людям в очереди (которые называются *агентами).* 

Очереди вызовов предоставляют:

- Приветствие.

- Музыка, пока люди находятся в очереди на удержание.

- Маршрутка вызовов ( *в порядке first In, First Out* (FIFO) для агентов.

- Параметры обработки для переполнения очереди и времени ожидания.

#### <a name="before-you-begin"></a>До начала работы

Получите [телефонную систему — лицензии](../teams-add-on-licensing/virtual-user.md) виртуальных пользователей, если у вас их еще нет. Получите по одному для каждой очереди вызовов и автозавода, которые вы планируете настроить. Эти лицензии бесплатны, поэтому мы рекомендуем получить дополнительные лицензии на случай, если вы решите внести изменения в настройку в будущем.

Так как агенты из очереди звонков могут звонить и возвращать клиентский звонок, вы можете установить для своих агентов, вызывающего звонок, основной номер телефона или номер соответствующего автосбережа. Дополнительные сведения см. в сведениях об управлении политиками ИД вызываемой [службы в Microsoft Teams.](../caller-id-policies.md)

#### <a name="follow-these-steps-to-set-up-your-call-queue"></a>Выполните эти действия, чтобы настроить очередь вызовов

# <a name="step-1brcreate-a-team"></a>[Шаг <br> 1. Создание команды](#tab/create-team)

При создании очереди вызовов вы можете добавить отдельных пользователей в очередь или использовать существующую группу безопасности, группу Microsoft 365 или команду Microsoft Teams. Рекомендуем использовать команду. Благодаря этому участники очереди могут общаться друг с другом, обмениваться идеями, создавать документы и другие ресурсы для помощи своим клиентам. Группа также предоставляет вызывателям голосовой почты почтовый ящик, в который можно отправлять сообщения в не часах или в том случае, если очередь достигает максимальной емкости.

Создание команды

1. Сначала щелкните **Teams** в левой части приложения, а затем нажмите кнопку **"Присоединиться"** или "Создать команду" в нижней части списка команд.

2. Затем нажмите **кнопку "Создать команду"** (первая карточка, левый верхний угол).

3. Выберите **"Создать команду с нуля".**

4. Затем выберите, нужна ли вам частная или гостевая команда. Мы рекомендуем **сделать** очередь личных, чтобы избежать непреднамеренного вхождения людей в очередь путем присоединения к команде.

5. Привязайте имя команды и добавьте необязательное описание.

6. Когда все будет готово, нажмите кнопку **"Создать".**

8. Введите имена людей, которых вы хотите добавить в очередь вызовов, и нажмите кнопку **"Добавить".**

9. Нажмите кнопку **Закрыть**. Люди, которых вы добавите в команду, получат по электронной почте сообщение с сообщением о том, что они теперь являются членами вашей команды, и группа от пока не будет добавлена в их список команд.

# <a name="step-2brresource-accounts"></a>[Шаг <br> 2. Учетные записи ресурсов](#tab/resource-account)

Для каждой создаемой очереди вызовов требуется учетная запись ресурса. Эта учетная запись аналогична учетной записи пользователя, за исключением того, что она связана с автоотчетом или очередью вызовов, а не с человеком. На этом шаге мы создадим учетную запись, назначим ее телефонной системе *Microsoft 365 —* лицензию виртуального пользователя, а затем создадим очередь звонков с ее помощью.

### <a name="create-a-resource-account"></a>Создание учетной записи ресурса

Учетную запись ресурса можно создать в Центре администрирования Teams.

1. В Центре администрирования Teams раз щелкните **"Учетные** записи ресурсов", чтобы развернуть параметры для всей **организации.**

2. Нажмите **Добавить**.

3. В области **"Добавление учетной** записи ресурса" введите  отображаемого **имени,** имя пользователя и выберите "Очередь зовов" для типа учетной **записи ресурса.**

    ![Снимок экрана: добавление учетной записи ресурса в пользовательский интерфейс](../media/resource-account-add-cq.png)

4. Щелкните **Сохранить**.

Новая учетная запись появится в списке учетных записей.

![Снимок экрана: список учетных записей ресурсов](../media/resource-accounts-page.png)

### <a name="assign-a-license"></a>Назначение лицензии

Учетной записи ресурса необходимо назначить *лицензию "Телефонная система Microsoft 365 —* виртуальный пользователь".

1. В Центре администрирования Microsoft 365 выберите учетную запись ресурса, для которой вы хотите назначить лицензию.

2. На **вкладке "Лицензии и** приложения" в **области "Лицензии"** выберите **"Телефонная система Microsoft 365 — виртуальный пользователь".**

3. Нажмите **кнопку "Сохранить изменения".**

    ![Снимок экрана: пользовательский интерфейс назначения лицензий в Центре администрирования Microsoft 365](../media/resource-account-assign-virtual-user-license.png)

### <a name="create-a-call-queue"></a>Создание очереди вызовов

Затем мы начнем создавать новую очередь зовов и назначаем учетную запись ресурса.

1. В Центре администрирования Teams раззовите **"Голосовая команда",** **выберите**"Очереди зовов" и нажмите кнопку **"Добавить".**

1. Введите имя очереди звонка. Это имя будет видно агентам при входящих вызовах из очереди.

2. Нажмите **кнопку "Добавить учетные** записи", найдите учетную запись ресурса, которую вы хотите использовать с этой очередью вызовов, нажмите кнопку "Добавить", а затем нажмите кнопку **"Добавить".**

3. Выберите язык. Этот язык будет использоваться для системных голосовых подсказок и транскрибации голосовой почты (если вы их включаете).

    ![Снимок экрана: учетная запись ресурса и языковые параметры](../media/call-queue-name-language.png)

4. Укажите, хотите ли вы воспроизведения приветствия вызывателям при их поступления в очередь. Необходимо отправить файл MP3, WAV или WMA, содержащий приветствие, которое вы хотите воспроизведения.

5. Teams предоставляет музыку по умолчанию вызывателям, пока они находятся на удержании в очереди. Если вы хотите воспроизведения определенного  звукового файла, выберите "Воспроизведения звукового файла" и загрузите файл MP3, WAV или WMA.

> [!NOTE]
> Размер добавленной записи не должен быть больше 5 МБ.
> По умолчанию музыка, которая предоставляется в очередях вызовов Teams, не предоставляется вашей организацией.

# <a name="step-4brcall-routing"></a>[Шаг <br> 4. Маршрутка вызовов](#tab/call-routing)

Выберите нужный способ маршрутизов.

1. Установите режим  **конференции автоматическим.**

2. Выберите **нужный способ** маршрутинга. Это определяет порядок, в котором агенты получают звонки из очереди. Рекомендуем **серийный маршрут или Округлевую** **юлеву.** Выберите один из этих вариантов:

    - **Маршрутка с помощником** одновременно звонит всем агентам в очереди. Звонок получает первый агент, который звонит.

    - **В последовательной маршрутке** все агенты звонят по одному. Если агент отклоает или не берет звонок, звонок будет звонить следующему агенту и будет пытаться все агенты до тех пор, пока он не будет разозвать или невызовется.

    - **Round balances** the routing of incoming calls so each call agent gets the same number of calls from the queue. Это может быть нежелательно в среде входящие продаж, чтобы гарантировать равные возможности для всех агентов по звонкам.

    - **Наиболее длинные** неавтомитные маршруты каждый звонок агенту, который простаивает самый длительный срок. (Агенты, состояние присутствия которых было "Нет на сети" более 10 минут, не включаются.)

    ![Снимок экрана: параметры режима конференции и способа маршрутации](../media/call-queue-conference-mode-routing-method.png)

3. **Включив маршрутную маршрутику на основе присутствия.** В этом случае звонки будут маршрутизовы агентам со статусом "В **сети".**

4. Выберите, нужно ли разрешить агентам отказаться от звонков.

5. Укажите **время оповещения** агента, чтобы указать, как долго телефон агента будет звонить до того, как очередь перенаправит звонок следующему агенту.

    ![Снимок экрана: параметры времени маршрутинга, отказа от работы и оповещений](../media/call-queue-presence-agents-time.png)

# <a name="step-5brcall-overflow"></a>[Шаг <br> 5. Переполнение звонка](#tab/call-overflow)

Выберите, как обрабатывать вызовы, превышают максимальное значение в очереди.

1. Установите **максимальное число звонков в очереди.**

2. Выберите, что вы хотите сделать, когда достигается максимальное количество звонков. Вы можете отключить звонок или перенаправить его. Мы рекомендуем перенаправить звонок на одно из следующих назначений:
    - **Человек в организации —** человек в организации, который может принимать голосовые звонки
    - **Приложение голосовой** почты — автозавод или другая очередь вызовов. (Выберите учетную запись ресурса, связанную с автостраницой или очередью вызовов, при выборе этого назначения.)
    - **Внешний номер телефона** — любой номер телефона. Используйте такой формат: +[код страны][код города][номер телефона]
    - **Голосовая** почта — вы можете использовать почтовый ящик созданной вами группы голосовой почты.

    ![Снимок экрана: параметры переполнения звонка](../media/call-queue-overflow-handling.png)

# <a name="step-6brcall-timeout"></a>[Шаг <br> 6. Время вызова](#tab/call-timeout)

Выберите, что должно происходить, когда звонки будут ждать в очереди слишком долго.

1. Настройка времени **ожидания звонка: максимальное время ожидания.**

2. Выберите, что вы хотите делать, когда разоберется время звонка. Вы можете отключить звонок или перенаправить его. Мы рекомендуем перенаправить звонок на одно из следующих назначений:
    - **Человек в организации —** человек в организации, который может принимать голосовые звонки
    - **Приложение голосовой** почты — автозавод или другая очередь вызовов. (Выберите учетную запись ресурса, связанную с автостраницой или очередью вызовов, при выборе этого назначения.)
    - **Внешний номер телефона** — любой номер телефона. Используйте такой формат: +[код страны][код города][номер телефона]
    - **Голосовая** почта — вы можете использовать почтовый ящик созданной вами группы голосовой почты.

    ![Снимок экрана: параметры времени и времени вызова](../media/call-queue-timeout-handling.png)

3. Щелкните **Сохранить**.

В этом случае настройка очереди звонка будет завершена. Затем может потребоваться настроить [автозаполнеть.](create-a-phone-system-auto-attendant-smb.md)
