# <a name="microsoft-education-governance-faq-for-admins"></a>Управление решениями Майкрософт для образования: вопросы и ответы для администраторов

> [!Tip]
> В следующем сеансе вы узнаете больше об управлении в Microsoft Teams: управление, управление и [жизненный цикл в Microsoft Teams](https://aka.ms/teams-governance)

## <a name="how-do-i-control-team-creation-im-worried-students-are-going-to-create-inappropriate-teams"></a>Как управлять созданием команд? Я беспокоится, что учащиеся будут создавать неприемлемые команды.

Чтобы избежать недопустимых или вводящих в заблуждение имен или просто обеспечить более структурированную структуру имен групп, вы можете использовать политику именования групп Microsoft 365 групп (в настоящее время она предварительной версии):

-   **Политика именования префиксов и суффиксов** Вы можете использовать префиксы или суффиксы для определения соглашения об именовожении команд (групп), например GRP_US_My Group_Engineering **.** Префиксы и суффиксы могут быть фиксированными строками или атрибутами пользователя (например, **[Department]),** которые добавляются в имя на основе пользователя, создавшего команду.
-   **Настраиваемые заблокированные слова** Вы можете добавить набор слов, которые пользователи в определенной организации не могут использовать в именах команд, которые они создают. Например, термины "Директор", "Заработная  оплата" и "Отдел кадров" можно использовать в названиях групп, к которых они не относятся. 
-   **Классификация** Вы можете создавать классификации, которые пользователи в организации могут устанавливать при создании Microsoft 365 групп. 

> [!IMPORTANT]
> Для Microsoft 365 именования групп требуется Azure Active Directory Premium P1 лицензии или лицензии Azure AD Basic EDU для каждого уникального пользователя, который является участником одной или Microsoft 365 групп.

Подробные инструкции см. [в Office именования групп.](https://support.office.com/article/office-365-groups-naming-policy-6ceca4d3-cad1-4532-9f0f-d469dfbbb552)

> [!Note]
> Если команды создаются автоматически с помощью данных из другой системы (например, Синхронизация сведений о школе), убедитесь, что входные данные соответствуют настроенной политике именования. Если этого не сделать, создание команды не удастся.

## <a name="can-i-see-who-created-a-team"></a>Можно ли посмотреть, кто создал команду?

Чтобы узнать, кто создал определенную команду, см. поиск событий в журнале аудита в [Microsoft Teams.](audit-log-events.md)

## <a name="can-i-control-who-can-create-teams"></a>Можно ли управлять тем, кто может создавать команды?

В общем, мы рекомендуем запретить всем создавать команды. Если все могут создавать команды, Teams, скорее всего, будут широко приняты. Преподаватели, преподаватели и учащиеся могут использовать Teams для создания учебных групп или специальных групп по интересам. Это поможет Teams принять приглашение как в аудитории, так и за ее пределами.

По нашему опыту, обучение пользователей помогает обеспечить Teams использование. Как только пользователи поймут, что создание команд не является анонимным, они понимают последствия их неосторожного создания и стараются не использовать средство.

Если вы уверены, что хотите управлять тем, кто может создавать команды, см. статью Управление возможностью Microsoft 365 [групп.](https://support.office.com/article/manage-who-can-create-office-365-groups-4c46c8cb-17d0-44b5-9776-005fced8e618)

## <a name="how-do-i-automatically-create-a-team-for-each-course-at-the-beginning-of-the-semester-or-quarter"></a>Как автоматически создать команду для каждого курса в начале семестра или квартала?

В начале каждого семестра или квартала вам потребуется несколько новых команд. Возможно, имеет смысл автоматически создавать такие команды автоматически, заполнять их нужными пользователями и устанавливать нужные разрешения.

-   Синхронизация сведений о школе можно создавать группы Microsoft 365 для Exchange Online и SharePoint Online, группы классов для Microsoft Teams и OneNote для занятий, учебные группы для Intune для образования и интеграция регистрации и единого входов (SSO) для многих других сторонних приложений. Дополнительные сведения можно узнать [в Синхронизация сведений о школе.](/schooldatasync/overview-of-school-data-sync)
-   С помощью PowerShell можно создавать команды и каналы, а также настраивать параметры автоматически. Дополнительные [Microsoft Teams PowerShell.](/powershell/module/teams/?view=teams-ps)
-   С помощью API microsoft Graph (в настоящее время она находится в бета-версии) можно создавать, настраивать, клонировать и архивировать команды. Дополнительные сведения см. в Graph [Microsoft Microsoft Teams API.](/graph/api/resources/teams-api-overview)

> [!TIP]
> Синхронизация сведений о школе создает группу Microsoft 365 для каждого класса, которая [](https://techcommunity.microsoft.com/t5/School-Data-Sync/HiddenGroupMembershipEnabled-SDS-setting/td-p/159945) синхронизируется и включает скрытое членство в группах, чтобы только преподаватели и учащиеся в классе могли видеть участников этого класса. Если для создания групп классов используется другой процесс, используйте параметр HiddenGroupMembershipEnabled New-UnifiedGroup, чтобы соответствовать тем же требованиям к конфиденциальности.

## <a name="how-do-i-deal-with-teams-when-the-semester-or-quarter-ends"></a>Как работать с командами после окончания семестра или квартала?

Мы рекомендуем сначала подумать о том, как вы хотите работать с данными Teams после окончания учебного семестра или квартала: следует ли их удалить или оставить доступными для учащихся даже после завершения курса. Вам нужно сохранить календарь учебного заведения, чтобы политики, которые вы установили, не конфликтуют с праздниками. Для реализации стратегии можно использовать следующие средства:

-   **Политика хранения:** Это позволяет удалить все данные старше определенного вами возраста, чтобы удалить старые данные из чатов (для всех или некоторых пользователей) и каналов. Вы также можете настроить Teams сохранить содержимое, чтобы его нельзя было удалить. Дополнительные сведения см. в [Microsoft Teams.](https://techcommunity.microsoft.com/t5/Microsoft-Teams-Blog/Retention-policies-for-Microsoft-Teams/ba-p/178011)
-   **Политика истечения срока действия:** Настройте срок действия групп по истечении определенного количества дней. За 30 дней до истечения срока действия все владельцы команды будут уведомлены о том, что их команду необходимо продлить, в противном случае она будет удалена (хотя администратор может восстановить удаленные команды в течение дополнительных 30 дней). Этот параметр очень удобен, если неиспользованые команды находятся на закате. Подробнее об этом можно узнать [в Microsoft 365 срока действия группы](https://support.office.com/article/office-365-group-expiration-policy-8d253fe5-0e09-4b3c-8b5e-f48def064733).

-   **Архивировать команду:** Этот параметр выводит команды в режим только для чтения. Их по-прежнему можно просматривать и искать, но никто не может добавлять новые записи. [В статье Архивировать или восстанавливать команды](https://support.office.com/article/archive-or-restore-a-team-dc161cfd-b328-440f-974b-5da5bd98b5a7) описывается, как владельцы могут архивировать команду; Владельцы команд также могут [Graph API (бета-версии)](/graph/api/resources/teams-api-overview) для архива или восстановления команды.
 
> [!IMPORTANT]
> Для Microsoft 365 групп требуется Azure Active Directory Premium P1 лицензий для каждого уникального пользователя, в который входит одна или Microsoft 365 групп.

## <a name="are-there-team-templates-for-my-faculty-members-to-use-when-creating-a-team"></a>Есть ли шаблоны для преподавателей, которые можно использовать при создании команды?

Да. Пользователи могут  выбрать создать команду из существующего шаблона, а владельцы Teams могут использовать API Graph [(бета-версию)](/graph/api/resources/teams-api-overview) для создания команды из доступных шаблонов.

## <a name="what-tasks-can-i-automate-via-powershell-or-graph"></a>Какие задачи можно автоматизировать с помощью PowerShell или Graph?

API [Graph Майкрософт (бета-версия)](/graph/api/resources/teams-api-overview) может:

-   Создайте команду.
-   Добавьте участников и владельцев.
-   Добавление каналов.
-   "Добавить приложения".
-   С помощью этих действий можно клонирования существующей команды и получить вкладки.
-   Предо в качестве пользователя ссылку на только что созданную команду.
-   Удаляйте участников, владельцев, каналы и приложения, если они вам больше не нужны.
-   Архивировать команду, если она больше не активна. 
-   Удалите команду.
-   Создание цепочки каналов

[PowerShell](/powershell/module/teams/?view=teams-ps) может сделать следующее:

-   Создайте команду.
-   Добавьте участников и владельцев.
-   Добавление каналов.
-   Удаляйте участников, владельцев и каналы, если они вам больше не нужны.
-   Удалите команду.

> [!TIP]
> С Graph API и PowerShell постоянно добавляются новые функции. Для улучшения функций часто проверяйте API Graph [Microsoft (бета-версия)](/graph/api/resources/teams-api-overview) и [PowerShell.](/powershell/module/teams/?view=teams-ps)  


## <a name="can-i-control-what-teams-features-my-faculty-and-students-have-access-to"></a>Могу ли я управлять Teams возможностями, к которые имеют доступ преподаватели и учащиеся?

Да. С помощью политик можно контролировать определенные функции обмена сообщениями, собраний, звонков и трансляций, к которые имеют доступ пользователи. С помощью параметров клиента можно применить одинаковые параметры для всех или при необходимости применить политики на уровне пользователей. 

Дополнительные сведения о политиках Teams см. в Microsoft Teams параметров управления политиками [для организации.](enable-features-office-365.md)
 
## <a name="can-i-control-what-external-parties-my-faculty-and-students-collaborate-with"></a>Могу ли я управлять тем, с какими внешними участниками работают преподаватели и учащиеся?

Вы можете использовать гостевой доступ, чтобы приглашать пользователей за пределами клиента, что может быть полезно при совместной работе или гостевых лекциях.

-   Используйте список разрешаний для домена, чтобы разрешить или заблокировать гостей на основе их домена.
-   Включите и отключите гостевой доступ для Microsoft 365 групп и групп, чтобы контролировать, какие команды могут (и не могут) приглашать гостей.
-   Чтобы узнать, какие оповещения были отправлены приглашенным гостям, используйте журнал аудита.

Дополнительные сведения см. в этой Microsoft 365 [группах.](https://support.office.com/article/Guest-access-in-Office-365-Groups-bfc7a840-868f-4fd6-a390-f347bf51aff6#PickTab=Manage)
