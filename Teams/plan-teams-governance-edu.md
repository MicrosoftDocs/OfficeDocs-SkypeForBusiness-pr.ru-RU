## <a name="how-do-i-control-team-creation-im-worried-students-are-going-to-create-inappropriate-teams"></a>Как контролировать создание группы? Сомнения по студентов будет создан неподходящие группы.

Во избежание неверные или вводящие в заблуждение ответы имена или просто обеспечивают дополнительные структуру именах групп, можно использовать группы Office 365, политика именования (в настоящее время предварительной версии):

-   **Политика именования префикс суффикс** Можно использовать префиксы или суффиксы определить соглашение об именовании рабочих групп (группы), например, **GRP_US_My Group_Engineering**. Префиксы и суффиксы устранения строк или атрибутов пользователя (например, **[отдел]**), которые добавляются к имени, на основе пользователя, создающего группу.
-   **Настраиваемые заблокированные слова** Можно отправить набор слов блокируется, что использование в именах групп, они создают пользователей в конкретной организации. К примеру можно заблокировать условия **CEO**, **о начислении заработной платы**и **управления Персоналом** используется в именах групп для групп, которые они не применимы к.
-   **Классификация** Вы можете создать классификации строк, которые пользователи в вашей организации можно задать при создании группы с Office 365. 

> [!IMPORTANT]
> С помощью политики именования групп Office 365 требует Azure Active Directory Premium P1 лицензий или Azure AD базовой EDU лицензии для каждого уникального пользователя, который является членом одной или нескольких групп Office 365.

Подробные сведения содержатся в разделе [политика именования групп Office](https://support.office.com/article/office-365-groups-naming-policy-6ceca4d3-cad1-4532-9f0f-d469dfbbb552).

> [!Note]
> Если группы создаются автоматически с помощью входные данные из другой системы (например, синхронизация данных School), следует проверьте соответствие входные данные политики назначения имен, который был настроен; Если нет, группы создания завершится с ошибкой.

## <a name="can-i-control-who-can-create-teams"></a>Можно контролировать, кто может создавать группы?

В общем мы не рекомендуем запретить всем пользователям создавать группы. Если все, можно создать группы, группы чаще всего будет широко. Факультета преподавателей и учащихся можно использовать команды для создания групп внедрения или группы. Это позволит команды будет принято внутри и за ее пределами аудитории.

Опыт обучение пользователей помогает обеспечить ответственность использования групп. Как только пользователи понять, что создание группы не будет анонимных, они осознаю последствия высока их создания и приводят к Если от неправильного использования средства.

Если вы уверены, требуется ли контролировать, кто может создавать группы, см [пользователей, которые можно создавать группы Office 365](https://support.office.com/article/manage-who-can-create-office-365-groups-4c46c8cb-17d0-44b5-9776-005fced8e618).

## <a name="how-do-i-automatically-create-a-team-for-each-course-at-the-beginning-of-the-semester-or-quarter"></a>Автоматическое создание группы для каждого курса в начале режим или квартала?

В начале каждого режим или квартала вам понадобится несколько новых рабочих групп. Смысл автоматизированных подход для создания таких групп автоматически, внесите их в правом пользователей и настройки соответствующих разрешений:

-   Синхронизация данных School можно создать группы Office 365 для Exchange Online и SharePoint Online, группы конструкторов класса записных книжек группами Майкрософт и класс OneNote, школа групп для Intune для образовательных учреждений и rostering и единого входа (SSO) интеграции для многих других приложения сторонних производителей. Дополнительные [Overview of школа данных синхронизации](https://docs.microsoft.com/schooldatasync/overview-of-school-data-sync).
-   С помощью PowerShell, можно создать группы и каналы и настроить параметры автоматически. Для получения дополнительных сведений в разделе [Microsoft команды PowerShell](https://docs.microsoft.com/powershell/module/teams/?view=teams-ps) .
-   Можно использовать API-Интерфейс Microsoft Graph (в настоящее время в бета-версия) для создания, настройки, клонирование и архивировать группами. Показано [Использование Microsoft Graph API для работы с группами Майкрософт](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview) для получения дополнительных сведений.

> [!TIP]
> Синхронизация данных School создает группу Office 365 для каждого класса синхронизирован и [позволяет скрытое членство](https://techcommunity.microsoft.com/t5/School-Data-Sync/HiddenGroupMembershipEnabled-SDS-setting/td-p/159945) , чтобы только преподавателей и учащихся в этом классе могут видеть члены этого класса. Если используется другой процесс для создания групп класс параметр HiddenGroupMembershipEnabled командлета New-UnifiedGroup в соответствии с теми же требованиями конфиденциальности.

## <a name="how-do-i-deal-with-teams-when-the-semester-or-quarter-ends"></a>Что делать с группами по окончании режим или квартала?

Мы рекомендуем сначала выбирая способ обработки данных групп при режим школа или квартала на: чтобы удалить или оставить его недоступны для учащихся даже после их прохождения курса. Может потребоваться следует помнить о календаре школа, любые политики, заданную вами не конфликтует с праздничные дни. Для реализации стратегии, можно использовать следующие средства:

-   **Политика хранения:** Используется для удаления всех данных старше возраст указываемые убедитесь в том, что старые данные удаляются из беседы (для пользователей всех или некоторых) и каналы. Можно также настроить группы, чтобы сохранить содержимое, поэтому его нельзя удалить. Для получения дополнительных сведений см [политики хранения для групп Майкрософт](https://techcommunity.microsoft.com/t5/Microsoft-Teams-Blog/Retention-policies-for-Microsoft-Teams/ba-p/178011).
-   **Политикой истечения срока действия:** Настройка групп истекает после определенного количества дней. 30 дней до окончания срока действия, всех владельцев группы будут уведомлены, что группа требует обновления, в противном случае — оно будет удалено (хотя администратора можно использовать для восстановления удаленных групп для дополнительных 30 дней). Этот параметр является очень позволяет убедиться, что неиспользуемые команды являются sunsetted. Дополнительные сведения по [Истечение срока действия групповой политики для Office 365](https://support.office.com/article/office-365-group-expiration-policy-8d253fe5-0e09-4b3c-8b5e-f48def064733).

-   **Архива группы:** Этот параметр помещает группы в режиме только для чтения. Их можно по-прежнему просматривать и поиск, но не могут добавлять всех новых сообщений. [Восстановление группы или архива](https://support.office.com/article/archive-or-restore-a-team-dc161cfd-b328-440f-974b-5da5bd98b5a7) описывает, как владельцев группы можно архивировать группы; Группа владельцев можно также использовать [API график (бета-версия)](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview) для архивации и восстановления группы.
 
> [!IMPORTANT]
> С помощью политики срока действия группы Office 365 требует Azure Active Directory Premium P1 лицензий для каждого уникального пользователя, который является членом одной или нескольких групп Office 365.

## <a name="are-there-team-templates-for-my-faculty-members-to-use-when-creating-a-team"></a>Какие шаблоны группы для личных факультета, использованный при создании группы?

Да. Пользователи могут выбрать **Создание группы на основе существующего шаблона** , при создании новой группы и владельцев группы можно также использовать [API график (бета-версия)](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview) для создания новой группы из доступных шаблонов.

## <a name="what-tasks-can-i-automate-via-powershell-or-graph"></a>Какие задачи можно автоматизировать с помощью PowerShell или графике

[Microsoft Graph API (бета-версия)](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview) можно выполнить следующие:

-   Создание группы.
-   Добавление участников и владельцев.
-   Добавьте каналы.
-   Добавление приложений.
-   Ярлык на соответствующие шаги путем копирования существующей группы и получить его вкладки слишком.
-   Предоставляются ссылки для группы, которую вы только что создали.
-   Удаление членов, владельцев, каналов и приложений, связанных с они больше не нужны.
-   Архивируйте группа, когда он больше не активен. 
-   Удалите группу.
-   Создание канала потока

[PowerShell](https://docs.microsoft.com/powershell/module/teams/?view=teams-ps) можно выполнить следующие:

-   Создание группы.
-   Добавление участников и владельцев.
-   Добавьте каналы.
-   Удаление участников, владельцев и каналы, когда они больше не нужны.
-   Удалите группу.

> [!TIP]
> Командлеты графике API и PowerShell постоянно добавляются функциональные возможности. Убедитесь в том, что проверка статьи [Microsoft Graph API (бета-версия)](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview) и [PowerShell](https://docs.microsoft.com/powershell/module/teams/?view=teams-ps) часто для улучшения компонента.
