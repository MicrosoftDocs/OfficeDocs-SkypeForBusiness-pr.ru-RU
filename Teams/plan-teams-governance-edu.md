# <a name="microsoft-education-governance-faq-for-admins"></a>Управление решениями Майкрософт для образования: вопросы и ответы для администраторов

> [!Tip]
> Просмотрите следующий сеанс, чтобы узнать больше об управлении в Microsoft Teams: [управление, управление](https://aka.ms/teams-governance) и жизненный цикл в Microsoft Teams

## <a name="how-do-i-control-team-creation-im-worried-students-are-going-to-create-inappropriate-teams"></a>Как управлять созданием команды? Я волнуюсь, что учащиеся создадут несоответствуют требованиям.

Чтобы избежать недопустимых или вводящих в заблуждение имен, а также для более точной структуры именования групп, используйте политику именования групп Microsoft 365 (в настоящее время в предварительной версии):

-   Политика именования префиксов **и суффиксов** С помощью префиксов и суффиксов можно определить соглашение об именовожении команд (групп), например **GRP_US_My Group_Engineering.** Префиксами и суффиксами могут быть фиксированные строки или атрибуты пользователя (например, **[Department]),** которые добавляются к имени в зависимости от того, кто создает команду.
-   **Настраиваемые заблокированные слова** Вы можете добавить набор слов, которые не могут использовать пользователи в определенных организациях в именах команд, которые они создают. Например, можно заблокировать термины "Директор",  "Заработная оплата" и "Кадры" в названиях групп, к которых они не относятся.  
-   **Классификация** Вы можете создавать классификации, которые пользователи организации могут устанавливать при создании группы Microsoft 365. 

> [!IMPORTANT]
> Для использования политики именования групп Microsoft 365 требуются лицензии Azure Active Directory Premium P1 или лицензии azure AD Basic EDU для каждого уникального пользователя, в который входит одна или несколько групп Microsoft 365.

Подробные инструкции см. в [политике именования групп Office.](https://support.office.com/article/office-365-groups-naming-policy-6ceca4d3-cad1-4532-9f0f-d469dfbbb552)

> [!Note]
> Если команды создаются автоматически с помощью входных данных из другой системы (например, School Data Sync), убедитесь, что входные данные соответствуют настроенной вами политике именования. если этого не сделать, создание команды не удастся.

Чтобы узнать, кто создал определенную команду, см. поиск событий в Microsoft Teams в журнале [аудита.](audit-log-events.md)

## <a name="can-i-control-who-can-create-teams"></a>Могу ли я управлять тем, кто может создавать команды?

В целом, мы рекомендуем запретить всем создавать команды. Если все могут создавать команды, Вероятность того, что Teams будет широко распространена, будет полнее. Преподаватели и учащиеся могут использовать Teams для создания учебных групп или специальных групп по интересам. Это поможет принимать Teams как в аудитории, так и за ее пределами.

По нашему опыту, образование пользователей помогает обеспечить ответственное использование Teams. Как только пользователи поймут, что создание команд не является анонимным, они понимают последствия неосторожного создания и стараются не стесняться, не стесняясь при неправильном доступе к инструменту.

Если вы уверены, что хотите управлять тем, кто может создавать команды, см. статью "Управление тем, кто может создавать группы [Microsoft 365".](https://support.office.com/article/manage-who-can-create-office-365-groups-4c46c8cb-17d0-44b5-9776-005fced8e618)

## <a name="how-do-i-automatically-create-a-team-for-each-course-at-the-beginning-of-the-semester-or-quarter"></a>Как автоматически создать команду для каждого курса в начале семестра или квартала?

В начале каждого семестра или квартала вам потребуется несколько новых команд. Возможно, имеет смысл автоматически создавать такие команды, заполнять их нужными пользователями и устанавливать нужные разрешения.

-   School Data Sync может создавать группы Microsoft 365 для Exchange Online и SharePoint Online, учебные группы для Записных книжок Microsoft Teams и OneNote для занятий, учебные группы для Intune для образовательных систем, а также интеграцию со службой единого входов (SSO) для многих других сторонних приложений. Дополнительные сведения о [School Data Sync.](https://docs.microsoft.com/schooldatasync/overview-of-school-data-sync)
-   С помощью PowerShell можно создавать команды и каналы, а также настраивать параметры автоматически. Дополнительные [сведения см. в Microsoft Teams PowerShell.](https://docs.microsoft.com/powershell/module/teams/?view=teams-ps)
-   С помощью API Microsoft Graph (в настоящее время она находится в бета-версии) можно создавать, настраивать, клонировать и архивировать группы. Дополнительные [сведения см. в API Microsoft Graph для работы с Microsoft Teams.](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview)

> [!TIP]
> School Data Sync создает группу Microsoft 365 [](https://techcommunity.microsoft.com/t5/School-Data-Sync/HiddenGroupMembershipEnabled-SDS-setting/td-p/159945) для каждого класса, синхронизируются и включает скрытое членство в группах, чтобы только преподаватели и учащиеся в классе могли видеть участников этого класса. Если для создания групп класса используется другой процесс, используйте параметр HiddenGroupMembershipEnabled для New-UnifiedGroup, чтобы соответствовать тем же требованиям к конфиденциальности.

## <a name="how-do-i-deal-with-teams-when-the-semester-or-quarter-ends"></a>Как работать с командами после окончания семестра или квартала?

Мы рекомендуем сначала подумать о том, как обрабатывать данные Teams после окончания семестра или квартала: удалить их или сделать доступными для учащихся даже после завершения курса. Необходимо помнить о том, что календарь учебного заведения не конфликтует с политиками, установленными вами. Для реализации стратегии можно использовать следующие средства:

-   **Политика хранения:** Используется для удаления всех данных старше определенного возраста, чтобы удалить старые данные из чатов (для всех или некоторых пользователей) и каналов. Вы также можете настроить Teams для сохранения содержимого, чтобы его нельзя было удалить. Дополнительные сведения см. [в политиках хранения для Microsoft Teams.](https://techcommunity.microsoft.com/t5/Microsoft-Teams-Blog/Retention-policies-for-Microsoft-Teams/ba-p/178011)
-   **Политика истечения срока действия:** Настройте срок действия команд через определенное количество дней. За 30 дней до истечения срока действия все владельцы команды будут уведомлены о том, что их команду необходимо продлить, в противном случае она будет удалена (хотя администратор может восстановить удаленные команды в течение дополнительных 30 дней). Этот параметр очень удобен, когда неиспользовано команды на закате. Подробнее об этом можно узнать в политике срока действия групп [Microsoft 365.](https://support.office.com/article/office-365-group-expiration-policy-8d253fe5-0e09-4b3c-8b5e-f48def064733)

-   **Команда архива:** Этот параметр перенастит команды в режим только для чтения. Их по-прежнему можно просматривать и искать, но никто не может добавлять новые записи. [Архив или восстановление команды описывает,](https://support.office.com/article/archive-or-restore-a-team-dc161cfd-b328-440f-974b-5da5bd98b5a7) как владельцы могут архивировать команду; Владельцы команд также могут использовать [API Graph (бета-версию)](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview) для архива или восстановления команды.
 
> [!IMPORTANT]
> Для использования политики срока действия групп Microsoft 365 требуются лицензии Azure Active Directory Premium P1 для каждого уникального пользователя, в который входит одна или несколько групп Microsoft 365.

## <a name="are-there-team-templates-for-my-faculty-members-to-use-when-creating-a-team"></a>Существуют ли шаблоны для преподавателей, которые можно использовать при создании команды?

Да. Пользователи могут выбрать команду **"Создать** команду" из существующего шаблона при создании новой команды, а владельцы Teams также могут использовать [API Graph (бета-версию)](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview) для создания команды из доступных шаблонов.

## <a name="what-tasks-can-i-automate-via-powershell-or-graph"></a>Какие задачи можно автоматизировать с помощью PowerShell или Graph?

API [Microsoft Graph (бета-версия)](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview) может делать следующее:

-   Создайте команду.
-   Добавьте участников и владельцев.
-   "Добавить каналы".
-   "Добавить приложения".
-   Захотеть эти действия, клонирование существующей команды и получить вкладки.
-   Предо дайте пользователю ссылку на только что созданную команду.
-   Удаляйте участников, владельцев, каналы и приложения, когда они больше не нужны.
-   Архивировать команду, если она больше не активна. 
-   Удалите команду.
-   Создание цепочки каналов

[PowerShell](https://docs.microsoft.com/powershell/module/teams/?view=teams-ps) может делать следующее:

-   Создайте команду.
-   Добавьте участников и владельцев.
-   "Добавить каналы".
-   Удаляйте участников, владельцев и каналы, когда они больше не нужны.
-   Удалите команду.

> [!TIP]
> Постоянно добавляются функциональные возможности с помощью API Graph и PowerShell. Для улучшения функций часто проверяйте [API Microsoft Graph (бета-версия)](https://developer.microsoft.com/graph/docs/api-reference/beta/resources/teams_api_overview) и [powerShell.](https://docs.microsoft.com/powershell/module/teams/?view=teams-ps)  


## <a name="can-i-control-what-teams-features-my-faculty-and-students-have-access-to"></a>Могу ли я управлять возможностями Teams, к каким функциям Teams имеют доступ преподаватели и учащиеся?

Да. С помощью политик можно контролировать определенные функции обмена сообщениями, собраний, звонков и трансляций, к которые имеют доступ пользователи. С помощью параметров клиента можно применить одинаковые параметры для всех или при необходимости применить политики на уровне пользователей. 

Дополнительные сведения о политиках Teams см. в параметрах [Microsoft Teams для вашей организации.](enable-features-office-365.md)
 
## <a name="can-i-control-what-external-parties-my-faculty-and-students-collaborate-with"></a>Могу ли я контролировать, с какими внешними лицами работают преподаватели и учащиеся?

Вы можете использовать гостевой доступ, чтобы приглашать пользователей из-за пределов клиента, что может быть полезно для исследования совместной работы или гостевой лекции.

-   Используйте список доменов, чтобы разрешить или заблокировать гостей на основе их домена.
-   Включите и отключите гостевой доступ для определенных групп и команд Microsoft 365, чтобы контролировать, какие команды могут (и не могут) приглашать гостей.
-   Чтобы узнать, какие оповещения были отправлены приглашенным гостям, используйте журнал аудита.

Дополнительные сведения [см. в гостевом доступе в группах Microsoft 365.](https://support.office.com/article/Guest-access-in-Office-365-Groups-bfc7a840-868f-4fd6-a390-f347bf51aff6#PickTab=Manage)
