# <a name="microsoft-education-governance-faq-for-admins"></a>Управление решениями Майкрософт для образования: вопросы и ответы для администраторов

> [!Tip]
> Просмотрите следующий сеанс, чтобы узнать больше об управлении в Microsoft Teams: управление [, управление и жизненный цикл в Microsoft Teams](https://aka.ms/teams-governance)

## <a name="how-do-i-control-team-creation-im-worried-students-are-going-to-create-inappropriate-teams"></a>Разделы справки команд управления? Я волнуюсь, что учащиеся будут создавать недопустимые команды.

Чтобы избежать недопустимых или неверных имен или просто предоставить дополнительную структуру имен команд, можно использовать политику именования Группы Microsoft 365 (в настоящее время доступна предварительная версия):

-   **Политика именования префиксов и суффиксов** Вы можете использовать префиксы или суффиксы для определения соглашения об именовании команд (групп), например **GRP_US_My Group_Engineering.** Префиксы и суффиксы могут быть фиксированными строками или атрибутами пользователя (например, **[Отдел]**), которые добавляются в имя на основе пользователя, создавшего команду.
-   **Настраиваемые заблокированные слова** Вы можете отправить набор слов, которые пользователи в определенной организации не могут использовать в именах команд, которые они создают. Например, вы можете заблокировать использование терминов **"** главный директор **", "** Заработная плата" и **"Кадровая** система" в именах групп, к которым они не применяются.
-   **Классификации** Вы можете создавать классификации, которые пользователи в организации могут задавать при создании группы Microsoft 365. 

> [!IMPORTANT]
> Для использования Группы Microsoft 365 именования требуются Azure Active Directory Premium P1 лицензии или лицензии Azure AD Basic EDU для каждого уникального пользователя, который является членом одной или нескольких групп Microsoft 365.

Подробные инструкции см. в [разделе "Политика именования групп Office"](https://support.office.com/article/office-365-groups-naming-policy-6ceca4d3-cad1-4532-9f0f-d469dfbbb552).

> [!Note]
> Если команды создаются автоматически с помощью входных данных из другой системы (например, School Data Sync), убедитесь, что входные данные соответствуют настроенной политике именования. Если это не так, создание команды завершится ошибкой.

## <a name="can-i-control-who-can-create-teams"></a>Можно ли управлять тем, кто может создавать команды?

Как правило, мы не рекомендуем запретить всем создавать команды. Если все могут создавать команды, скорее всего, Teams будет широко внедрен. Преподаватели, преподаватели или учащиеся могут использовать Teams для создания учебных групп или специальных групп интересов. Это поможет принять Teams внутри и за пределами аудитории.

По нашему опыту, обучение пользователей помогает обеспечить ответственное использование Teams. Как только пользователи поймют, что создание команд не является анонимным, они понимают последствия их неоправронного создания и, как правило, не используются неправильно.

Если вы уверены, что хотите контролировать, кто может создавать команды, см. статью "Управление пользователями, которые [могут создавать Группы Microsoft 365](https://support.office.com/article/manage-who-can-create-office-365-groups-4c46c8cb-17d0-44b5-9776-005fced8e618)".

## <a name="how-do-i-automatically-create-a-team-for-each-course-at-the-beginning-of-the-semester-or-quarter"></a>Разделы справки вы автоматически создаете команду для каждого курса в начале квартала или полугодия?

В начале каждого полугодия или квартала вам потребуется несколько новых команд. Возможно, имеет смысл использовать автоматизированный подход для автоматического создания этих команд, заполнения их нужными пользователями и задания необходимых разрешений:

-   School Data Sync может создать Группы Microsoft 365 для Exchange Online и SharePoint Online, группы классов для Microsoft Teams и записных книжек OneNote Для занятий, учебные группы для Intune для образовательных учреждений, а также интеграцию с реестром и единым входом (SSO) для многих других сторонних приложений. Дополнительные сведения [см. в статье "Общие сведения о синхронизации учебных данных"](/schooldatasync/overview-of-school-data-sync).
-   С помощью PowerShell можно создавать команды и каналы и настраивать параметры автоматически. [Дополнительные сведения см. в статье о Microsoft Teams PowerShell](/powershell/module/teams/?view=teams-ps).
-   Вы можете использовать microsoft API Graph (в настоящее время в бета-версии) для создания, настройки, клонирования и архивации команд. [Дополнительные сведения см. в](/graph/api/resources/teams-api-overview) API Graph Microsoft Teams.

> [!TIP]
> School Data Sync создает группу Microsoft 365 для каждого синхронизированных классов [](https://techcommunity.microsoft.com/t5/School-Data-Sync/HiddenGroupMembershipEnabled-SDS-setting/td-p/159945) и включает скрытое членство в группах, чтобы только преподаватели и учащиеся в классе могли видеть участников этого класса. При использовании другого процесса для создания групп классов используйте параметр HiddenGroupMembershipEnabled командлета New-UnifiedGroup для удовлетворения тех же требований к конфиденциальности.

## <a name="how-do-i-deal-with-teams-when-the-semester-or-quarter-ends"></a>Разделы справки работать с командами по завершении квартала или полугодия?

Мы рекомендуем сначала подумать о том, как вы хотите обрабатывать данные Teams по истечении учебного года или квартала: следует ли удалить их или оставить доступными для учащихся даже после завершения курса. Вам нужно учитывать календарь учебного заведения, чтобы политики, задаемые вами, не конфликтовать с праздниками. Для реализации стратегии можно использовать следующие средства:

-   **Политика хранения:** Используйте его для удаления всех данных старше указанного возраста, чтобы удалить старые данные из чатов (для всех или некоторых пользователей) и каналов. Вы также можете настроить Teams на сохранение содержимого, чтобы его нельзя было удалить. Дополнительные сведения см [. в разделе "Политики хранения для Microsoft Teams"](https://techcommunity.microsoft.com/t5/Microsoft-Teams-Blog/Retention-policies-for-Microsoft-Teams/ba-p/178011).
-   **Политика истечения срока действия:** Настройте срок действия команд по истечении определенного количества дней. За 30 дней до истечения срока действия все владельцы команды получают уведомление о необходимости продления команды, в противном случае она будет удалена (хотя администратор может восстановить удаленные команды еще на 30 дней). Этот параметр очень полезен для того, чтобы неиспользуемые команды были удалены. Дополнительные сведения см [. в политике срока действия группы Microsoft 365](https://support.office.com/article/office-365-group-expiration-policy-8d253fe5-0e09-4b3c-8b5e-f48def064733).

-   **Команда архивации:** Этот параметр переводит команды в режим только для чтения. Их по-прежнему можно просматривать и искать, но никто не может добавлять новые записи. [Архивация или восстановление команды описывает](https://support.office.com/article/archive-or-restore-a-team-dc161cfd-b328-440f-974b-5da5bd98b5a7), как владельцы команды могут архивировать команду; Владельцы команд также могут использовать API Graph [(бета-версия)](/graph/api/resources/teams-api-overview) для архивации или восстановления команды.
 
> [!IMPORTANT]
> Для использования Группы Microsoft 365 срока действия Azure Active Directory Premium P1 лицензий для каждого уникального пользователя, который является членом одной или нескольких групп Microsoft 365.

## <a name="are-there-team-templates-for-my-faculty-members-to-use-when-creating-a-team"></a>Существуют ли шаблоны команд для преподавателей, которые будут использовать при создании команды?

Да. Пользователи **могут выбрать "** Создать команду" из существующего шаблона при создании новой команды, а владельцы Teams также могут использовать API Graph (бета-версия [)](/graph/api/resources/teams-api-overview) для создания новой команды на основе доступных шаблонов.

## <a name="what-tasks-can-i-automate-via-powershell-or-graph"></a>Какие задачи можно автоматизировать с помощью PowerShell или Graph?

Microsoft [API Graph (бета-версия)](/graph/api/resources/teams-api-overview) может выполнять следующие действия:

-   Создайте команду.
-   Добавьте участников и владельцев.
-   Добавление каналов.
-   Добавление приложений.
-   Перейдите к этим шагам, клонировав существующую команду, и получите ее вкладки.
-   Предоставьте пользователю ссылку на только что созданную команду.
-   Удалите участников, владельцев, каналы и приложения, когда они больше не нужны.
-   Архивируйте команду, когда она больше не активна. 
-   Удалите команду.
-   Создание потока канала

[PowerShell](/powershell/module/teams/?view=teams-ps) может выполнять следующие действия:

-   Создайте команду.
-   Добавьте участников и владельцев.
-   Добавление каналов.
-   Удалите участников, владельцев и каналы, когда они больше не нужны.
-   Удалите команду.

> [!TIP]
> Командлеты API Graph и PowerShell постоянно добавляют функциональные возможности. Обязательно ознакомьтесь со статьями [microsoft API Graph (](/graph/api/resources/teams-api-overview)бета-версия) и [PowerShell](/powershell/module/teams/?view=teams-ps), чтобы узнать об улучшениях функций.  


## <a name="can-i-control-what-teams-features-my-faculty-and-students-have-access-to"></a>Можно ли управлять функциями Teams, к каким функциям имеют доступ преподаватели и учащиеся?

Да. Политики можно использовать для управления определенными функциями обмена сообщениями, собраниями, звонками и трансляцией, к которым пользователи имеют доступ. Вы можете использовать параметры на уровне клиента, чтобы применить одинаковые параметры ко всем или при необходимости применить политики на уровне пользователя. 

Дополнительные сведения о политиках Teams см. в статье ["Управление параметрами Microsoft Teams для вашей организации"](enable-features-office-365.md).
 
## <a name="can-i-control-what-external-parties-my-faculty-and-students-collaborate-with"></a>Можно ли управлять внешними сторонами, с которыми работают мои преподаватели и учащиеся?

Вы можете использовать гостевой доступ для приглашения пользователей за пределами клиента, что может быть полезно для совместной работы и гостевых лекций:

-   Используйте список разрешений домена, чтобы разрешить или заблокировать гостей в зависимости от их домена.
-   Включите и отключите гостевой доступ для определенных Группы Microsoft 365 и команд, чтобы контролировать, какие команды могут (и не могут) приглашать гостей.
-   Используйте журнал аудита, чтобы узнать, какие оповещения были отправлены приглашенным гостям.

Дополнительные сведения см[. в статье о гостевом доступе Группы Microsoft 365](https://support.office.com/article/Guest-access-in-Office-365-Groups-bfc7a840-868f-4fd6-a390-f347bf51aff6#PickTab=Manage).
