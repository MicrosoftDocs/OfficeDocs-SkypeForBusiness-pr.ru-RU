# <a name="contact-center-integrations-for-microsoft-teams"></a>Интеграция Центра контактов для Microsoft Teams

Интеграция популярных решений Центра контактов с Microsoft Teams часто необходима клиентам, которые развертывают возможности звонков Teams.  В этой статье представлен обзор интеграции решений центра контактов с Microsoft Teams и дополнительных сведений о поставщиках решений, участвующих в программе сертификации Подключенный центр контактов Microsoft Teams.

## <a name="what-is-a-contact-center-integration-for-microsoft-teams"></a>Что такое интеграция Центра контактов с Microsoft Teams?

Современные центры контактов предоставляют не только поддержку, но и выступают в качестве одного из основных средств взаимодействия и неотлитных отзывов о фирменом опыте клиента. Из-за широты каналов, которые предпочитают клиенты в настоящее время ( телефон, электронная почта, текст, социальные сети), а также в связи с расширенным объемом точек, связанных с процессами приобретения на сегодняшний день, во многих организациях реализованы два дополнительных клиента:

1. Каждый сотрудник организации может быть вовлечен непосредственно в взаимодействие с клиентом, поэтому его необходимо обучать соответствующими средствами.

2. В такой расширенной области взаимодействия с клиентами требуются средства, которые помогают обеспечить согласованность, постоянное улучшение и масштаб.

Microsoft Teams поддерживает потоки взаимодействия с клиентами, выступая в качестве центра для внутреннего и внешнего взаимодействия с клиентами в различных режимах общения, включая чат, видеособрации и вызовы. В некоторых компаниях облачные голосовые возможности [](./what-are-phone-system-auto-attendants.md) Microsoft [Teams,](./cloud-voice-landing-page.md)включая функции автоотвода и очереди вызовов, [](./create-a-phone-system-call-queue.md)предоставляют необходимые функции и конфигурацию.

Microsoft Teams также интегрируется с некоторыми ведущими поставщиками решений CCaaS, чтобы интегрировать интегрированные решения с бизнес-инструментами и бизнес-процессами.

### <a name="connect"></a>Подключения

Эта модель связывает поставщиков решений CCaaS с инфраструктурой телефонной системы Microsoft Teams, включая улучшенную маршрутику, конфигурацию и аналитику системы. 

Агенты, использующие решения, встроенные в модель подключения, могут собирать сведения, & сведения и при необходимости передавать звонки экспертам напрямую, используя сведения о присутствии SME в Teams для обеспечения их доступности.

Организации могут обеспечить перенастройку звонков агенту, настроив автоматических виртуальных помощников и очереди маршрутов на основе навыков.

**Основные моменты функций:**

Хотя ниже не полный список возможностей этой модели интеграции, в число областей, на которые она направлена, относятся следующие:

  - Office 365 authN для агентов, чтобы разрешить агентам подключаться к их клиенту Майкрософт из интегрированного клиента CCaaS 

  - Указание присутствия от пользователей Teams 

  - Поддержка переводов и групповых звонков для пользователей Teams 

  - API Teams Graph и API облачных коммуникаций для интеграции с Teams 

  - Поддержка связи SIP с несколькими клиентами в SBC поставщика решений.  

  - Поставщики решений для использования [ <span class="underline">сертифицированного контроллера границы сеанса (SBC) Майкрософт</span>](./direct-routing-border-controllers.md) 

### <a name="extend"></a>Расширить

Эта модель расширяет взаимодействие сотрудников и агентов центра контактов путем интеграции с клиентом Teams с помощью клиентской платформы [Teams](/microsoftteams/platform/overview), [API Graph Teams Graph](/graph/api/resources/teams-api-overview?view=graph-rest-1.0) и [API](/graph/api/resources/communications-api-overview?view=graph-rest-1.0) облачных коммуникаций в Microsoft Graph и использует телефонную систему Teams для всех звонков в центре контактов и управления звонками. В этой модели поставщик решений Центра контактов выступает в качестве оператора телефонии вместе с Microsoft 365.

С помощью решений Connect и Extend-based агенты могут использовать динамические контекстные примечания, соотносяющие данные из нескольких систем перед началом взаимодействия и избежая затратного переключения контекста, работая в Teams как для внутренней совместной работы, так и для внешней связи.

Организации могут проектировать рабочий процесс и расширенные конфигурации маршрутов в зависимости от конкретного пользователя и измерять качество системы и взаимодействия.

**Основные моменты функций:**

Хотя ниже не полный список возможностей этой модели интеграции, в нем выделены основные области:

  - API Teams Graph и API облачных коммуникаций для интеграции с Teams 

  - Приложение teams для работы с агентами 

  - Teams в качестве основной конечной точки для звонков агентов 

  - Вызов клиента Teams для всех элементов управления звонками

  - Приложение "Опыт работы агента" также должно работать в веб-и мобильных клиентах Teams

  - Аналитика, управление рабочего процессами, ролевая работа агентов в приложении CCaaS в Teams

  - Взаимодействие и взаимодействие, интегрированные с клиентами Teams 

  - Сохранение производительности и качества работы клиентов Teams во всех приложениях  

### <a name="power"></a>Мощность

Эта модель позволяет поставщикам решений создавать голосовые приложения на базе Azure, используя инфраструктуру звонков Teams и клиентскую платформу для предоставления современных интеллектуальных решений для взаимодействия с клиентами и агентами. Цель Расширения и Power — повысить творческие способности разработчиков и повысить производительность клиентов.

Непосредственно работая в Azure, поставщики решений могут быстро развертывать и подстраить решение во все регионы и регионы Teams, извлекая выгоду из общей глобальной сети связи, при этом пользуясь преимуществами хранилища, вычислительной и аналитики Azure & когнитивными службами.

С помощью модели расширения и интеграции с Power поставщики решений могут предоставлять агентам Центра контактов взаимодействие по каналам, в то же время включая средства искусственного интеллекта для настройки вовлечения участников (или других служб) в вызов, применяющий API облачных коммуникаций в [Microsoft Graph.](/graph/api/resources/communications-api-overview?view=graph-rest-1.0)

**Основные моменты функций:**

Ниже представлен не полный список возможностей этой модели интеграции, но в дополнение к возможностям, предоставляемым моделью Подключения и расширения, эти области выделены.

  - Формальные возможности агентов, которые включены для взаимодействия с каналом через Teams SDK 

  - Использование служб совместной работы Teams для совместной работы агентов и взаимодействия с клиентами  

  - Быстрая подготовка облачных служб и развертывание в любом месте 

  - Прямое управление беседами и взаимодействие с пользователями во время бесед Teams
