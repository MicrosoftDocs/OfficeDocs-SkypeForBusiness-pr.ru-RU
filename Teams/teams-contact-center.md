# <a name="contact-center-integrations-for-microsoft-teams"></a>Интеграция Центра контактов с Microsoft Teams

Интеграция популярных решений Центра контактов с Microsoft Teams — распространенная необходимость для клиентов, которые развертывают возможности звонков Teams.  В этой статье приводится обзор интеграции решений Центра контактов с Microsoft Teams и дополнительные сведения о партнерских решениях, участвующих в программе сертификации Microsoft Teams Connected Contact Center.

## <a name="what-is-a-contact-center-integration-for-microsoft-teams"></a>Что такое интеграция Центра контактов с Microsoft Teams?

Современные центры контактов предоставляют не только поддержку, но и выступают в качестве одного из основных средств взаимодействия и неотмеченных отзывов о работе клиентов с фирменным брендом. Из-за широты каналов, которые клиенты предпочитают использовать по телефону, электронной почте, тексту, социальной сети, а также к расширенному объему сенсорных точек, связанных с процессами приобретения сегодня, во многих организациях реализованы два дополнительных функции:

1. Каждый сотрудник организации может быть вовлечен напрямую в взаимодействие с клиентом, поэтому его нужно поучащать соответствующими средствами.

2. В такой расширенной области взаимодействия с клиентом требуются средства, которые помогают обеспечить согласованность, постоянное улучшение и масштаб.

Microsoft Teams поддерживает потоки взаимодействия с клиентом, выступая в качестве центра для внутреннего и внешнего взаимодействия с клиентами в различных режимах общения, включая чат, видеособрации и вызовы. В некоторых компаниях функции облачного голосового управления Microsoft [Teams,](https://docs.microsoft.com/microsoftteams/cloud-voice-landing-page)включая функции автоотетаря и очереди вызовов, предоставляют необходимые функции и конфигурацию. [](https://docs.microsoft.com/microsoftteams/what-are-phone-system-auto-attendants) [](https://docs.microsoft.com/microsoftteams/create-a-phone-system-call-queue)

Для тех, кто хочет интегрировать интегрированные решения с бизнес-инструментами и бизнес-процессами, чтобы принять участие клиентов, Microsoft Teams также интегрируется с некоторыми ведущими в отрасли поставщиками решений Contact Center as a Service (CCaaS).

### <a name="connect"></a>Подключение

Эта модель связывает партнеров CCaaS с инфраструктурой телефонной системы Microsoft Teams, позволяя улучшить маршрутизаацию, конфигурацию и аналитику системы. 

Агенты, использующие решения, использующие модель подключения, могут собирать информацию & анализа и при необходимости передавать звонки непосредственно экспертам, используя присутствие SME в Teams для обеспечения их доступности.

Организации могут обеспечить маршруты звонков сотруднику, оптимальному агенту, настроив автоматизированные виртуальные помощники и очереди маршрутов на основе навыков.

**Основные возможности**

Ниже содержится не полный список возможностей этой модели интеграции, но в число ее пунктов следует включить следующие области:

  - AuthN Office 365 для агентов, чтобы разрешить агентам подключаться к клиенту Майкрософт из интегрированного клиента CCaaS 

  - Сведения о присутствии от пользователей Teams 

  - Поддержка передач и групповых звонков с пользователями Teams 

  - API Teams Graph и API облачных коммуникаций для интеграции с Teams 

  - Поддержка многоэтапной службы SIP для поддержки нескольких клиентов на SBC партнера.  

  - Партнеры для использования [ <span class="underline">сертифицированного пограничного контроллера сеанса Майкрософт (SBC)</span>](https://docs.microsoft.com/MicrosoftTeams/direct-routing-border-controllers) 

### <a name="connect-and-extend"></a>Подключение и расширение

Эта модель расширяет взаимодействие сотрудников и агентов центра контактов за счет интеграции с клиентом [Teams](https://docs.microsoft.com/microsoftteams/platform/overview)с клиентской платформой [Teams, API Teams Graph](https://docs.microsoft.com/graph/api/resources/teams-api-overview?view=graph-rest-1.0) и API Cloud [Communications](https://docs.microsoft.com/graph/api/resources/communications-api-overview?view=graph-rest-1.0) в Microsoft Graph и использует телефонную систему Teams для всех звонков и управления звонками в Центре контактов. В этой модели партнер Центра контактов выступает в качестве оператора телефонной связи вместе с Microsoft 365.

Используя решения Connect и Extend-based, агенты могут использовать динамические контекстные заметки для корреляции данных из нескольких систем перед запуском взаимодействия и избегайте затратного переключения контекста путем внутренней работы внутри Teams для внутренней совместной работы и внешней связи.

Организации могут проектировать рабочий процесс и дополнительные конфигурации маршрутов для отдельных людей и измерять качество системы и взаимодействия.

**Основные возможности**

Хотя ниже не полный список возможностей этой модели интеграции, в нем выделяются основные важные области:

  - API Teams Graph и API облачных коммуникаций для интеграции с Teams 

  - Приложение teams для работы с агентом 

  - Teams в качестве основной конечной точки для звонков агентов 

  - Клиентские вызовы Teams для всех элементов управления звонками

  - Приложение для работы с агентом также должно работать в веб-и мобильном клиенте Teams

  - Аналитика, управление рабочего процессами, ролями агентов в приложении CCaaS в Teams

  - Взаимодействие и чат, интегрированные с клиентами Teams 

  - Сохранение производительности и качества работы клиентов Teams во всех приложениях  

### <a name="extend-and-power"></a>Расширение и питание

Эта модель позволяет партнерам создавать голосовые приложения на основе Azure, используя инфраструктуру звонков Teams и клиентскую платформу для предоставления современных интеллектуальных решений для совместной работы с клиентами и агентами. Цель расширения и эффективности Power — повысить творческие способности разработчиков и повысить производительность клиентов.

Непосредственно работая в Azure, партнеры могут быстро развертывать и подстраить свое решение во все регионы и регионы Teams, извлекая выгоду из общей глобальной сети связи, а также пользуясь преимуществами хранилища, вычислительной и аналитики Azure & когнитивными службами.

С помощью модели интеграции с расширением и питанием партнеры могут предоставлять агентам Центра контактов взаимодействие по каналу, в то же время включая средства искусственного интеллекта для настройки взаимодействия участников (или других служб) с помощью API облачных коммуникаций в [Microsoft Graph.](https://docs.microsoft.com/graph/api/resources/communications-api-overview?view=graph-rest-1.0)

**Основные возможности**

Ниже представлен не полный список возможностей этой модели интеграции, но в дополнение к возможностям модели подключения и расширения эти функции выделены.

  - Формальный опыт взаимодействия с агентом через Teams SDK 

  - Использование служб совместной работы Teams для совместной работы агентов и взаимодействия с клиентами  

  - Быстрая подготовка облачных служб и развертывание в любом месте 

  - Управление беседами и взаимодействие с пользователями во время бесед Teams
