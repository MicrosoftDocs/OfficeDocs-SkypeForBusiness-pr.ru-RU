# <a name="contact-center-integrations-for-microsoft-teams"></a>Интеграция Центра контактов с Microsoft Teams

Интеграция популярных решений Центра контактов с Microsoft Teams — распространенная необходимость для клиентов, которые развертывают возможности звонков Teams.  В этой статье приводится обзор интеграции решений Центра контактов с Microsoft Teams, а также дополнительные сведения о поставщиках решений, участвующих в программе сертификации Центра контактов Microsoft Teams Connected.

## <a name="what-is-a-contact-center-integration-for-microsoft-teams"></a>Что такое интеграция Центра контактов с Microsoft Teams?

Современные центры контактов предоставляют не только поддержку, но и выступают в качестве одного из основных средств взаимодействия и неотстимиловых отзывов о работе клиентов с фирменным брендом. Из-за широты каналов, которые клиенты предпочитают использовать по телефону, электронной почте, тексту, социальной сети, а также к расширенному объему сенсорных точек, связанных с процессами приобретения сегодня, во многих организациях реализованы два дополнительных функции:

1. Каждый сотрудник организации может быть вовлечен напрямую в взаимодействие с клиентом, поэтому его необходимо обучать соответствующими средствами.

2. В такой расширенной области взаимодействия с клиентом требуются средства, которые помогают обеспечить согласованность, постоянное улучшение и масштаб.

Microsoft Teams поддерживает потоки взаимодействия с клиентом, выступая в качестве центра для внутреннего и внешнего взаимодействия с клиентами в различных режимах общения, включая чат, видеособрации и вызовы. В некоторых компаниях функции облачного голосового управления Microsoft [Teams,](./cloud-voice-landing-page.md)включая функции автоотетаря и очереди вызовов, предоставляют необходимые функции и конфигурацию. [](./what-are-phone-system-auto-attendants.md) [](./create-a-phone-system-call-queue.md)

Для тех, кто хочет интегрировать интегрированные решения с бизнес-инструментами и бизнес-процессами, чтобы удлинять путь клиентов, Microsoft Teams также интегрируется с некоторыми ведущими в отрасли поставщиками решений Contact Center as a Service (CCaaS).

### <a name="connect"></a>Подключение

Эта модель подключает поставщиков решений CCaaS к инфраструктуре телефонной системы Microsoft Teams, позволяя улучшить маршрутизаацию, конфигурацию и аналитику системы. 

Агенты, использующие решения, использующие модель подключения, могут собирать информацию & анализа и при необходимости передавать звонки непосредственно экспертам, используя присутствие SME в Teams для обеспечения их доступности.

Организации могут обеспечить маршруты звонков сотруднику, оптимальному агенту, настроив автоматизированные виртуальные помощники и очереди маршрутов на основе навыков.

**Основные возможности**

Ниже содержится не полный список возможностей этой модели интеграции, но в ее число входят следующие области:

  - AuthN Office 365 для агентов, чтобы разрешить агентам подключаться к клиенту Майкрософт из интегрированного клиента CCaaS 

  - Сведения о присутствии от пользователей Teams 

  - Поддержка передач и групповых звонков пользователям Teams 

  - API Teams Graph и API облачных коммуникаций для интеграции с Teams 

  - Поддержка связи SIP с несколькими клиентами для нескольких клиентов на SBC поставщика решений.  

  - Поставщики решений для использования пограничного контроллера сеанса Майкрософт, сертифицированного [ <span class="underline">для сеанса (SBC)</span>](./direct-routing-border-controllers.md) 

### <a name="connect-and-extend"></a>Подключение и расширение

Эта модель расширяет взаимодействие сотрудников и агентов центра контактов за счет интеграции с клиентом [Teams](/microsoftteams/platform/overview)с клиентской платформой [Teams, API Teams Graph](/graph/api/resources/teams-api-overview?view=graph-rest-1.0) и API cloud [Communications](/graph/api/resources/communications-api-overview?view=graph-rest-1.0) в Microsoft Graph и использует телефонную систему Teams для всех звонков и управления звонками в Центре контактов. В этой модели поставщик решений Центра контактов выступает в качестве оператора телефонии наряду с Microsoft 365.

Используя решения Connect и Extend-based, агенты могут использовать динамические контекстные заметки для корреляции данных из нескольких систем перед запуском взаимодействия, а затем избежать затратного переключения контекста путем внутренней работы в Teams для внутренней совместной работы и внешней связи.

Организации могут проектировать рабочий процесс и дополнительные конфигурации маршрутов для отдельных людей и измерять качество системы и взаимодействия.

**Основные возможности**

Хотя ниже не полный список возможностей этой модели интеграции, в нем выделяются основные важные области:

  - API Teams Graph и API облачных коммуникаций для интеграции с Teams 

  - Приложение teams для работы с агентом 

  - Teams в качестве основной конечной точки для звонков агентов 

  - Клиентские вызовы Teams для всех элементов управления звонками

  - Приложение для работы с агентом также должно работать в веб-и мобильном клиенте Teams

  - Аналитика, управление рабочего процессами, ролями агентов в приложении CCaaS в Teams

  - Взаимодействие и чат, интегрированные с клиентами Teams 

  - Сохранение производительности и качества работы клиентов Teams во всех приложениях  

### <a name="extend-and-power"></a>Расширение и питание

Эта модель позволяет поставщикам решений создавать голосовые приложения на основе Azure, используя инфраструктуру звонков Teams и клиентскую платформу для предоставления современных интеллектуальных решений для совместной работы с клиентами и агентами. Цель расширения и эффективности Power — повысить творческие способности разработчиков и повысить производительность клиентов.

Непосредственно работая в Azure, поставщики решений могут быстро развертывать и подстраить свое решение во все регионы и регионы Teams, извлекая выгоду из нашей общей глобальной сети связи, одновременно пользуясь преимуществами хранилища Azure, вычислительных и аналитических & когнитивными службами.

С помощью модели интеграции с расширением и питанием поставщики решений могут предоставлять агентам Центра контактов взаимодействие по каналам, в то же время включая средства искусственного интеллекта, чтобы настроить способ и время вовлечения участников (или других служб) в вызов с помощью [API Cloud Communications в Microsoft Graph.](/graph/api/resources/communications-api-overview?view=graph-rest-1.0)

**Основные возможности**

Ниже представлен не полный список возможностей этой модели интеграции, но в дополнение к возможностям модели подключения и расширения эти функции выделены.

  - Формальный опыт взаимодействия с агентом через Teams SDK 

  - Использование служб совместной работы Teams для совместной работы агентов и взаимодействия с клиентами  

  - Быстрая подготовка облачных служб и развертывание в любом месте 

  - Управление беседами и взаимодействие с пользователями во время бесед Teams
