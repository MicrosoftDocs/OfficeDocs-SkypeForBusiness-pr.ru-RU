# <a name="monitor-and-improve-call-quality-for-microsoft-teams"></a>Мониторинг и повышение качества звонка для Microsoft Teams

В этой статье 3 ключевых средства, которые можно использовать для отслеживания, устранения неполадок, управления и улучшения качества звонка в Microsoft Teams. 

- **Панель мониторинга** качества звонка : для анализа тенденций или проблем в масштабе организации необходимо повысить производительность.

- **Аналитика вызовов:** анализ качества вызовов и собраний для отдельных пользователей

- **Качество обслуживания (QoS):** приоритет важного сетевого трафика



## <a name="monitor-and-troubleshoot-call-quality"></a>Отслеживание и устранение неполадок с качеством звонка
Для поиска и устранения неполадок, которые возникают в ходе текущей работы, используются средства аналитики вызовов и панели мониторинга качества звонка для каждого пользователя.   Это позволяет улучшить производительность сети. Оба этих средства находятся в Teams администрирования.

 - **В средстве аналитики** звонков подробные сведения об **** устройствах, сетях и подключениях, связанных с конкретными звонками и собраниями, для каждого пользователя Teams. Teams администратор и агенты службы справки будут использовать эти сведения для устранения неполадок с качеством звонка и проблем с подключением во время определенного звонка. Подробнее об этом можно узнать в настройках средства аналитики [](set-up-call-analytics.md) вызовов и Устранении неполадок с качеством звонка с помощью средства аналитики [вызовов.](use-call-analytics-to-troubleshoot-poor-call-quality.md)
 
 - Панель мониторинга качества звонка **(CQD)** позволяет просматривать качество звонка в масштабе всей организации. **** Используйте сведения CQD для выявления и устранения проблем. [Во-первых, настройка CQD](turning-on-and-using-call-quality-dashboard.md). Прочитайте [статью Управление качеством звонка и собрания в Teams.](quality-of-experience-review-guide.md)

 Аналитика вызовов и CQD запускаются параллельно и могут использоваться независимо или вместе. Например, если специалист службы поддержки связи определяет, что ему нужна дополнительная помощь в устранении неполадок с звонком пользователя, он передает звонок специалисту службы поддержки связи, у которого есть доступ к дополнительным сведениям о звонке. В свою очередь, инженер службы поддержки связи оповещает сетевого инженера о возможной проблеме, связанной с сайтом, которые он заметил в средстве аналитики вызовов. Сетевой инженер проверяет CQD, чтобы узнать, не является ли общая проблема, связанная с сайтом, одной из причин возникновения проблемы со звонком пользователя.


## <a name="prioritize-important-network-traffic-using-qos"></a>Расставить приоритеты для важного сетевого трафика с помощью QoS
Когда пользователи начинают использовать Teams для звонков и собраний, у них может возникнуть нарушение голосовой разгона или вырезание звонка или собрания. Общее видео может зависать или провисать в пикселях, а также в целом сбой. Это связано с тем, что IP-пакеты, которые представляют трафик голосовой и видеосвязи, которые сталкиваются с перегрузкой сети и вылетом из последовательности или вообще не выгрузются. В этом случае (или, в первую очередь, для предотвращения этого) используйте **QoS .** 

С помощью QoS приоритеты для сетевого трафика, необходимого для задержки (например, голосового или видеопотока), позволяют ему "вырезать строку" перед менее конфиденциальным трафиком (например, скачиванием нового приложения, где дополнительная секунда для скачивания не так велика). QoS позволяет выявлять и пометить все пакеты в потоках в режиме реального времени с помощью объектов групповой политики Windows и функции маршрутизация под названием Списки управления на основе портов доступа, которая позволяет вашей сети передавать голосовые и видеосвязи, а также делиться экранами с собственной выделенной пропускной способностью сети.

Лучше всего внедрить QoS во внутренней сети, когда будете готовы к Teams, но это можно сделать в любое время. Если у вас достаточно маленький размер, возможно, QoS не нужен.

Когда все будет готово, см. статью Реализация качества обслуживания [(QoS) в Microsoft Teams.](QoS-in-Teams.md)

Чтобы использовать QoS для управления трафиком собраний, прочитайте статью Настройка обработки трафика мультимедиа в режиме реального времени для Teams [собраний.](meeting-settings-in-teams.md#set-how-you-want-to-handle-real-time-media-traffic-for-teams-meetings)
