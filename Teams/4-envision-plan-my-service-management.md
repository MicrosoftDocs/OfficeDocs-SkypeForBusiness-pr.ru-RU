# <a name="plan-my-service-management"></a>Планирование управления Мои службы

В этой статье дается обзор требования, необходимые для доставки и обслуживать развертывание групп Майкрософт высокого качества. Можно обеспечить успешное развертывание по планированию для управления службой и качество во время фазы Предвидения перед первой Развертывание пилотной или производственной.

## <a name="service-management-for-teams"></a>Управление службой для групп

Управление службой является широкий раздела, который охватывает повседневной эксплуатации службы группами Майкрософт после развертывания и доступна для пользователей. Служба группы включает в себя Microsoft Office 365 и компонентов инфраструктуры, развернутые локально (например, к сети).

Общее представление о службе управления не вероятнее всего новые концепции для большинства организаций. Возможно, уже был реализован процессов и задач, связанных с существующими службами. С другой стороны, возможно, можно расширить, что у вас есть на месте при планировании для управления службой сегодня для поддержки Microsoft команд в будущем.

Служба управления включает все действия и процессов, участвующих в управлении группами Майкрософт сквозного. Некоторые компоненты службы управления — компоненты инфраструктуры, состоит из самой службы Office 365 — являются ответственность корпорации Майкрософт, тогда как клиента отвечает его пользователям управлять различных аспектов командами, сети, и конечные точки, которые они предоставляют.
Полное обсуждение ответственности клиента для службы управления группами и его связь со основные компоненты, которые обеспечивают высокое качество работы пользователя в разделе [Планирование службы управления и контроля качества](https://docs.microsoft.com/MicrosoftTeams/envision-planning-for-service-management-and-quality-complete-guide).

![Схема три компонента качества--службы Office 365, сети и конечные точки, и как служба управления перекрывается все три.] (media/plan-my-service-management-image1.png "Схема три компонента качества--службы Office 365, сети и конечные точки, и как служба управления перекрывается все три.")

## <a name="introduction-to-the-operations-guide"></a>Введение в руководство пользователя 

**Какие**, **кто**и **как** являются три важные вопросы, которые нужно ответить, когда речь идет о управления службой.

[Руководство пользователя](https://docs.microsoft.com/MicrosoftTeams/1-drive-value-operate-my-service) можно использовать для решения всех трех из этих вопросов. Руководство по предоставляет список действий, выполняемых на основе ежедневно, еженедельно, ежемесячно и по необходимости. Эти действия и задачи важны для развертывании группами высокое качество. Определение пользователей, которые будут отвечать для выполнения определенных действий в службе управления является важных аспектов планирования, которое необходимо выполнить на раннем этапе Предвидения для обеспечения успешного развертывания. После поняли задачи и действия, нужные для понимания и следуют групп или пользователей, которые нужно назначить их. Руководство пользователя содержит базы знаний и рекомендации по выполнять все задачи и/или ссылки на содержимое, находящиеся за пределами организации.

## <a name="plan-for-operational-role-mapping"></a>План для сопоставления действующие роли

На раннем планирования для управления службой является важных вех, поскольку на этапе операций начинается при включении первого пилотных пользователей. Группы проекта необходимо с на задачи и просмотрите и действия, необходимые, Идентификация группы, который несет ответственность за каждой задачи действующие и затем получите подтверждение и подпись из всех отделов.

По завершении подписи ответственной группе следует запустить операционализации этих ролей и обязанностей. Сюда может входить обучения и подготовки к обновления кадров модели или проверка того, что внешние партнеры все готово для доставки.

Сопоставление ролей на раннем этапе Предвидения включает все группы для запуска их рабочих задачах во время пилотного проекта и расширение путь операции и убедитесь в том, что все готова после того как запустится развертывания.

Руководство пользователя предоставляет список наиболее распространенных задач, сопоставленные с типичные роли, которые должны быть допустимый в большинстве случаев. Необходимо настроить эти обязанности для работы для вашей организации.

## <a name="the-quality-champion-role"></a>Роль известный качества

Группы или отдельного должно быть ответственность за качество во всех организациях.
Это наиболее важные роли управления службой. Известный качества — это роль клиента, назначенная для человека или группы, являющийся отношусь взаимодействия своих пользователей. Эта роль требуются навыки для идентификации тенденции в среде и финансирования для работы с другими группами для обновлений.
Лучший вариант для известный качества обычно является владельцем службы клиента, — в зависимости от размера организации и сложности, может быть любой пользователь или группа, являющийся отношусь взаимодействие с пользователем.

Известный качества использует имеющихся средств и задокументировать процессов, таких как вызов качества панели мониторинга (CQD) и качества взаимодействия руководство по решению, отслеживать взаимодействие с пользователем, определение тенденций качества, а также исправление при необходимости. Известный качества для работы с соответствующих групп на диске исправлению действия, отчетность для исполнительного комитета по их о ходе выполнения и открытых вопросов.

Задачи и действия, связанные с ролью приведен в руководстве по эксплуатации. Эта роль должна быть назначена на раннем этапе Предвидения. Это важный шаг операционализации роль известный качества при подключении знаний, необходимых для роли и проверка того что необходимого программного обеспечения на месте для доставки на задачи. Ключевые задачи для этой роли работает регулярного анализа качества взаимодействия.

Руководство по решению взаимодействия качество имеет ряд действий, которые оценки и даются рекомендации по исправлению в ключевых областях, которые имеют наибольшее влияние для улучшения пользовательского интерфейса, как показано на следующем рисунке.

![Ключевые области для проверки во время качества взаимодействия review: аудио-, надежности и результатах опроса пользователя.] (media/plan-my-service-management-image2.png "Ключевые области для проверки во время качества взаимодействия review: аудио-, надежности и результатах опроса пользователя.")

Постоянно вызов и устранению области, описанные в этом документе, позволит сократить их потенциально может отрицательно сказаться на взаимодействие с пользователем. Большинство взаимодействие с пользователем проблем, возникающих при развертывании, можно разбить на следующие категории:

-   Неполный конфигурации брандмауэр или прокси-сервера

-   Низкий покрытия Wi-Fi

-   Недостаточно пропускной способности

-   VPN

-   Использование неоптимизированном или встроенных звуковых устройств

-   Инспектором подсетей и сетевых устройств

Оптимизация внесенные в сети для улучшения качества звука также непосредственно преобразует для улучшения в общий доступ к рабочему столу и видео.
