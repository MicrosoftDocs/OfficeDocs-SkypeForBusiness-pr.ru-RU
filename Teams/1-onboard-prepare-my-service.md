# <a name="prepare-my-service"></a>Подготовка службы

В этой статье приводятся общие сведения о требованиях для подготовки облачных служб голосовой связи для Организации. Выполнив подготовку, вы можете убедиться, что вы готовы предоставить возможности голосовой связи с облаком в Организации.

## <a name="onboarding-checklists-for-microsoft-teams-voice-workloads"></a>Контрольные списки для подготовки голосовых загрузок Microsoft Teams

Приведенные ниже контрольные списки пошаговые инструкции по применению функций голосовой видеосвязи, телефонной системы с планами звонков (планы звонков) и прямой маршрутизации по телефонной системе ("прямая маршрутизация") в Microsoft Teams.

*  [Подготовка Office 365 для Teams](onboarding-checklist-enable-office-365.md)

*  [Настройка основных возможностей Teams](onboarding-checklist-configure-microsoft-teams-core-capabilities.md)

*  [Настройка сети](onboarding-checklist-configure-networking.md)

*  [Настройка рабочих нагрузок облачной голосовой связи в Teams](onboarding-checklist-configure-cloud-voice-workloads-in-Microsoft-Teams.md)

*  [Настройка прямой маршрутизации в Teams](onboarding-checklist-configure-direct-routing-in-Microsoft-Teams.md)

Задачи и действия в этих контрольных списках — это основные элементы "задача", которые применяются к каждому развертыванию возможностей голосовой связи в облаке с помощью Teams. Вы можете настроить контрольные списки, чтобы включить в них мероприятия и задачи, которые относятся к своему своему расприятию в Teams.

>[!NOTE]
>Это руководство сосредоточено исключительно на тарифных планах, голосовых конференциях и прямом маршруте. Если вы не знакомы с Teams, ознакомьтесь с [обзором Microsoft Teams](teams-overview.md). Общее руководство по планированию развертывания команд начинается с [развертывания чата, рабочих групп, каналов и приложений в Microsoft Teams](deploy-chat-teams-channels-microsoft-teams-landing-page.md).

С помощью приведенных контрольных списков вы можете отслеживать состояние каждого отдельного мероприятия и задачи, а также не пропускать важные этапы. Каждое действие включает подробное описание необходимых действий и ссылки на дополнительные сведения, которые можно использовать для выполнения этого действия.

Несмотря на то, что мы рекомендуем выполнять контрольные списки по порядку, Точная последовательность будет зависеть от области развертывания и конфигурации и сложности вашей среды. Они организованы для поддержки развертывания в среде "гринфиелд" (в том случае, если в Skype для бизнеса Online отсутствует прежнее состояние) или переход с Skype для бизнеса Online на Teams. Если вы выполняете переход с Skype для бизнеса Online, возможно, вы уже выполнили некоторые из этих действий и можете проигнорировать их прямо сейчас.

При планировании пользователей на уровне сайта мы настоятельно рекомендуем использовать [Плайбук включения для голоса (плайбук)](https://github.com/MicrosoftDocs/OfficeDocs-SkypeForBusiness/blob/live/Teams/downloads/site-enablement-playbook-for-voice-(playbook).xlsx?raw=true) в качестве дополнительного руководства для этих контрольных списков.

>[!NOTE]
>Большая часть параметров конфигурации обычно используется в Teams и Skype для бизнеса Online. Настройте эти параметры с помощью центра администрирования Microsoft 365 и центра администрирования Microsoft Teams.

На следующем этапе следует использовать [Плайбук включения для голоса (плайбук)](https://github.com/MicrosoftDocs/OfficeDocs-SkypeForBusiness/blob/live/Teams/downloads/site-enablement-playbook-for-voice-(playbook).xlsx?raw=true) , чтобы помочь пользователям на каждом сайте, а также убедиться, что вы планируете и выполняете важные действия, связанные с конкретными сайтами.

-   Готовый план выпуска для сайта

-   Установка процесса управления службами

-   Выполнение тестирования и исправления

## <a name="test-cloud-voice-workloads-in-teams"></a>Проверка рабочих нагрузок в облаке в Teams

После того как вы определили и задокументированы планы успеха и технической реализации облачных голосов для бизнеса в составе этапа создания и настроили в центре администрирования требуемую конфигурацию, необходимо проверить, что в вашей организации ожидание и требования выполняются с помощью функций, функций и удобства использования. Этот шаг проверки следует выполнить перед развертыванием пилотного или финального развертывания в рабочей среде.

Вы можете использовать план успеха бизнеса, который вы определили на этапе представления, чтобы послужить основой для определения действий, ожиданий, функций, тестовых случаев и общей области, которую нужно вычислить на этапе тестирования.

## <a name="define-your-testing-approach"></a>Определение подхода к тестированию

В самой простой форме подход к тестированию основывается на проверке возможностей голосовой конференции, планах звонков или прямой маршрутизации и разработке плана тестирования для проверки соблюдения требований к функциональным возможностям для пользователей в области. Ниже приведен пример тестового плана для встроенной фазы реализации голосовой конференции.


| Проверка возможности голосовой конференции | Сводка по результатам | Дополнительные заметки |
|------------|-----------------|------------------|
| Планирование собраний с помощью рекламных подходящих команд, которые содержат данные для подключения к голосовой конференции | Передача или сбой   | РИСУНКА |
| Использование телефона для звукового сопровождения при наборе номера для собрания по сети PSTN с помощью предоставленных данных для подключения | Передача или сбой | РИСУНКА |
| Присоединение пользователей к существующему собранию с помощью сети PSTN | Передача или сбой | РИСУНКА |



| Планы звонков или прямая маршрутизация для тестирования | Сводка по результатам | Дополнительные заметки |
|----------------------------------------------------|-----------------|------------------|
| Совершение коммутируемого звонка посредством набора номера PSTN       | Передача или сбой       | РИСУНКА |
| Получение коммутируемого звонка из внешней линии (мобильный, стационарные) | Передача или сбой | РИСУНКА|
| Передача вызова по коммутируемой телефонной сети от одного пользователя группы к другому | Передача или сбой | РИСУНКА |


>[!TIP]
>Чтобы облегчить создание тестового случая как отправную точку, ознакомьтесь со списком руководств пользователя, доступных на [собраниях и звонках Teams](https://support.office.com/article/Meetings-and-calls-d92432d5-dd0f-4d17-8f69-06096b6b48a8#bkmk_havingmeetings).

### <a name="execute-the-test-plan"></a>Выполнение плана тестирования

[//]: # (Изменить? "Пользователь" — это немного неоднозначная возможность.)
После того как пользовательская среда и служба настроены, последний этап тестирования включает выполнение плана тестирования, в котором фокус находится на проверке функций и функциональных возможностей. 

**Проверка соблюдения требований и предположения для пользователей и сайтов в области:**

-   Определение варианта использования для бизнес-конференций для службы голосовой связи завершено.

-   Лицензирование, необходимое для голосовой конференции, доступно и назначено.

-   Идентифицирован список организационных сайтов и групп пользователей.

-   Указан и настроен список выделенных и общих звуковых конференций с использованием языковых и других параметров.

-   [Кредиты на связь](what-are-communications-credits.md) для вашей организации (если это необходимо) настроено (при необходимости).

-   Параметры моста конференц-связи с голосовой связью заданы и настроены (уведомление о длине, ввод-вывод, Настройка уведомлений о включении).

-   Политики конференц-связи с клиентами и параметры телефонной схемы, поддерживающие сценарии исходящих звонков для голосовой связи, определены, настроены и применены.

-   Определение и Настройка требований к голосовой конференции на соответствие требованиям.

**Планы звонков тестирование предварительных требований и допущения для пользователей и сайтов в области:**

-   Определение варианта использования для бизнеса для службы планов звонков завершено.

-   Лицензирование, требуемое для планов звонков, доступно и назначено.

-   Идентифицирован список организационных сайтов и групп пользователей.

-   Номера телефонов, которые должны быть назначены пользователям, были приобретены или перенесены в корпорацию Майкрософт и доступны на портале клиента.

-   [Кредиты на связь](what-are-communications-credits.md) для вашей организации (если это необходимо) настроено (при необходимости).

-   Политики пользователей клиента и параметры телефонной группы, поддерживающие сценарии планов вызовов, определены, настроены и применены.

-   Планы звонков, требования соответствия требованиям определены и настроены.

**Прямое тестирование и допущения для пользователей и сайтов в области:**

-   Определение варианта использования для бизнеса для службы прямой маршрутизации завершено.

-   Лицензирование, требуемое для прямой маршрутизации, доступно и назначено.

-   Идентифицирован список организационных сайтов и групп пользователей.

-   [Контроллер границ сертифицированного сеанса (SBC)](https://docs.microsoft.com/microsoftteams/direct-routing-plan#supported-session-border-controllers-sbcs) был развернут, настроен и связан с телефонной системой.

-   Корпоративная голосовая связь включена, и номера телефонов назначены.

-   Политики маршрутизации голосовой связи определены, настроены и назначены.

-   Microsoft Teams установлен в качестве предпочтительного абонентского клиента для пользователей в области.
 
-   Были идентифицированы и настроены требования к соответствия требованиям Direct Routing.
