# <a name="create-a-call-queue"></a>Создание очереди вызовов

Очереди звонков предоставляют метод маршрутизации вызывающих абонентов для людей в организации, которые могут помочь в определенной проблеме или вопросе. Звонки распространяются по одному людям в очереди (которые называются *агентами*). 

> [!TIP]
> Эта статья предназначена для крупных организаций. Если ваша организация — малый бизнес, ознакомьтесь с руководством по созданию очереди звонков для [малого](/microsoftteams/business-voice/create-a-phone-system-call-queue-smb) бизнеса.

Очереди вызовов предоставляют:

- Приветствие

- Музыка, пока люди ожидают удержания в очереди

- Маршрутизация вызовов — в порядке *"Первый вход", "* Первый выход" (FIFO) — агентам

- Параметры обработки переполнения очереди и времени ожидания

Перед выполнением процедур, описанных в этой статье, убедитесь, что [](plan-auto-attendant-call-queue.md) у вас есть план Teams автосекретарей и очередей звонков, а также следуйте инструкциям [по началу работы](plan-auto-attendant-call-queue.md#getting-started).

**Дополнительные сведения [см](#call-queue-feature-compatibility) . в таблице совместимости функций очереди вызовов ниже.**

Введите имя очереди вызовов.

## <a name="call-routing"></a>Маршрутизация вызовов

![Снимок экрана: режим конференции и параметры метода маршрутизации.](media/call-queue-conference-mode-routing-method.png)

**Режим конференции** сокращает время, необходимое для подключения вызывающего абонента к агенту после принятия вызова агентом. Для работы режима конференции агенты в очереди вызовов должны использовать один из следующих клиентов:

  - Последняя версия настольного Microsoft Teams, Android или iOS приложения
  - Телефонная система Microsoft Teams версии 1449/1.0.94.2020051601 или более поздней
  
Учетные Teams агентов должны быть настроены в режиме TeamsOnly. Агенты, которые не соответствуют требованиям, не включаются в список маршрутизации вызовов. Мы рекомендуем включить режим конференции для очередей звонков, если агенты используют совместимые клиенты.

> [!NOTE]
> Режим конференции не поддерживается, если телефонные звонки направляются в очередь из шлюза прямой маршрутизации, который включен для маршрутизации на основе расположения.
>
> Режим конференции не поддерживается, если телефонные звонки направляются в очередь из Skype для бизнеса Server.
> 
> Режим конференции требуется, если Teams должны обращаться к звонкам или передавать их с помощью очередей звонков.

> [!TIP]
> Рекомендуется **установить режим конференции** **в режим "** В сети".

**Метод маршрутизации определяет** порядок, в котором агенты получают вызовы из очереди. Выберите один из следующих вариантов:

- **Маршрутизация помощника ко** всем агентам в очереди одновременно. Первый агент вызова, который выбирает вызов, получает вызов.

- **Последовательная маршрутизация** вызывает все агенты вызовов по одному в порядке, указанном в списке **агентов** вызовов. Если агент закрывает или не выбирает вызов, вызов вызывает следующий агент. Это будет повторяться до тех пор, пока не будет установлен или не истекло время ожидания вызова.

- **Циклический перебор** распределяет маршрутизацию входящих вызовов таким образом, чтобы каждый агент вызова получает одинаковое количество вызовов из очереди. Этот метод маршрутизации может быть желательно в среде входящих продаж, чтобы обеспечить равные возможности для всех агентов вызовов.

- **Самый длинный бездействующий** маршрут направляет каждый вызов агенту, который был неактивным в течение самого длительного времени. Агент считается неактивным, если его состояние присутствия доступно. Агенты, состояние присутствия которых недоступно, не смогут принимать звонки, пока не будут изменены на "Доступно". 

> [!TIP]
> Рекомендуется **установить для метода маршрутизации** значение **"Циклический** перебор" или **"** Самый длинный бездействующий".

> [!NOTE]
> Если [на агентах](teams-recording-policy.md) включена запись соответствия, **сочетание режима конференции** и маршрутизации помощника не поддерживается. Если необходимо использовать режим конференции **, выберите** в качестве метода маршрутизации последовательную маршрутизацию **, циклический** перебор или самое длинное **бездействие**. Если необходимо использовать маршрутизацию **помощника**, задайте режим **конференции** **"Выкл."**.
> 
> При использовании **самого длительного** простоя и меньшего числа вызовов в очереди, чем доступных агентов, вызовы из очереди будут отображаться только для первых двух самых длинных неактивных агентов.
> 
> При использовании самого длительного простоя могут возникать моменты, когда агент получает вызов из очереди вскоре после того, как он становится недоступным, или кратковременная задержка при получении вызова из очереди после того, как она станет доступной.
> 
> Представление вызовов очереди вызовов для агентов может конфликтовать с ограничениями маршрутизации на основе расположения. В этом случае агент получит всплывающее уведомление о вызове, но не сможет ответить на звонок. Это условие будет продолжаться до тех пор, пока другой агент не будет доступен для ответа на вызов, вызывающий объект зависает или не произойдет состояние ожидания очереди вызовов.  


![Снимок экрана: параметры маршрутизации, отказа и времени генерации оповещений.](media/call-queue-presence-agents-time.png)

**Маршрутизация на** основе присутствия использует состояние доступности агентов вызовов, чтобы определить, должен ли агент быть включен в список маршрутизации вызовов для выбранного метода маршрутизации. Агенты вызовов, состояние доступности которых имеет значение **"** Доступно", включены в список маршрутизации звонков и могут принимать звонки. Агенты, состояние доступности которых имеет любое другое состояние, исключаются из списка маршрутизации звонков и не будут получать вызовы, пока их состояние доступности не изменится на **"Доступно"**. 

Маршрутизацию вызовов на основе присутствия можно включить с помощью любого из методов маршрутизации.

Если агент отключит получение звонков, он не будет включен в список маршрутизации звонков независимо от состояния доступности. 

> [!NOTE]
> Если  в качестве метода маршрутизации выбран самый длинный режим простоя, маршрутизация на основе присутствия является обязательной и автоматически включена, даже если переключатель маршрутизации на основе присутствия будет **отключен** и неактивен.
>
> Если маршрутизация на основе присутствия не включена и в очереди несколько вызовов, система будет одновременно представлять эти вызовы агентам независимо от их состояния присутствия. Это действие приведет к нескольким уведомлениям об звонках агентам, особенно если некоторые агенты не отвечают на исходный вызов, представленный им.
>
> При использовании маршрутизации на основе присутствия могут возникать моменты **, когда** агент получает вызов из очереди вскоре после того, как он становится недоступным, или кратковременная задержка при получении вызова из очереди после его получения.
> 
> Агенты, использующие Skype для бизнеса клиента, не включаются в список маршрутизации вызовов, если включена маршрутизация на основе присутствия. Если у вас есть агенты, Skype для бизнеса, не включайте маршрутизацию вызовов на основе присутствия.

> [!TIP]
> Рекомендуется **установить для маршрутизации на основе присутствия** значение **"** В сети".

**Время оповещения агента** указывает, как долго будет звонить телефон агента, прежде чем очередь перенаправит вызов следующему агенту.

> [!TIP]
> Рекомендуется **задать для оповещений** агента **значение 20** секунд.

> [!NOTE]
> Параметр [обработки времени ожидания вызова](#call-timeout-handling) имеет приоритет над временем оповещения агента. Если достигнуто максимальное время ожидания в очереди, настроенное для обработки времени ожидания вызова, вызов будет извлечен из агентов, даже если не достигнут предел времени оповещения агента.

## <a name="call-overflow-handling"></a>Обработка переполнения вызовов

![Снимок экрана: параметры переполнения вызовов.](media/call-queue-overflow-handling.png)

**Максимальное число вызовов в очереди** указывает максимальное количество вызовов, которые могут ожидать в очереди в любой момент времени. Значение по умолчанию — 50, но может варьироваться от 0 до 200. При превышении этого ограничения вызов обрабатывается, как указано в параметре "Когда достигается максимальное количество **вызовов** ".

Вы можете отключить вызов или перенаправить его в любое из назначений маршрутизации параметров. Например, вызывающий абонент может оставить голосовую почту для агентов в очереди. Сведения о внешних передачах см. [в разделе "Предварительные](plan-auto-attendant-call-queue.md#prerequisites) требования" и "Передача внешних номеров телефонов" — [технические](create-a-phone-system-auto-attendant.md#external-phone-number-transfers---technical-details) сведения о форматировании номеров.

> [!NOTE]
> Если максимальное число вызовов равно 0, приветствие не воспроизводится.
>
> При перенаправлении на общую голосовую почту убедитесь, что пользователи за пределами этой организации могут отправлять электронное письмо этой команде в группу или группу в Microsoft 365 Admin Center.


## <a name="call-timeout-handling"></a>Обработка времени ожидания вызова

![Снимок экрана: параметры времени ожидания вызова.](media/call-queue-timeout-handling.png)

**Время ожидания вызова: максимальное** время ожидания, задающее максимальное время удержания вызова в очереди перед перенаправлением или отключением. Можно указать значение от 0 секунд до 45 минут.

Вы можете отключить вызов или перенаправить его в одно из назначений маршрутизации вызовов. Например, вызывающий абонент может оставить голосовую почту для агентов в очереди. Сведения о внешних передачах см. в разделе " [Предварительные](plan-auto-attendant-call-queue.md#prerequisites) требования" и "Передача внешних номеров телефонов" — [технические](create-a-phone-system-auto-attendant.md#external-phone-number-transfers---technical-details) сведения о форматировании номеров.

> [!NOTE]
> При перенаправлении на общую голосовую почту убедитесь, что пользователи за пределами этой организации могут отправлять электронное письмо этой команде в группу или группу в Microsoft 365 Admin Center.

Выбрав параметры времени ожидания звонка, нажмите кнопку " **Сохранить"**.

## <a name="summary-of-recommended-call-queue-settings"></a>Сводка рекомендуемых параметров очереди вызовов

Рекомендуется использовать следующие параметры:

- **Режим конференции включен** 
- **Метод маршрутизации в** **циклический перебор или** **самое длинное бездействие**
- **Маршрутизация на основе присутствия в** **включенную**
- **Время оповещения агента:** **до 20 секунд**

  - Все модели IP-телефонов, поддерживаемые для Microsoft Teams. См[. раздел "Получение телефонов для Skype для бизнеса Online"](/skypeforbusiness/what-is-phone-system-in-office-365/getting-phones-for-skype-for-business-online/getting-phones-for-skype-for-business-online).

  - Клиент Android Skype для бизнеса (версия 6.16.0.9 и более поздние)
  - Клиент iPhone Skype для бизнеса (версия 6.16.0 и более поздние)
  - Клиент iPad Skype для бизнеса (версия 6.16.0 и более поздние)
  - Microsoft Teams Windows клиента (32-разрядная и 64-разрядная версии)
  - Клиент Microsoft Teams Mac
  - Microsoft Teams инфраструктуре [виртуализированных](/microsoftteams/teams-for-vdi) рабочих столов (Windows Виртуальный рабочий стол, Citrix и VMware)

  - Microsoft Teams Android приложения

## <a name="call-queue-diagnostic-tool"></a>Средство диагностики очереди вызовов

Если вы являетесь администратором, вы можете использовать следующее средство диагностики, чтобы убедиться, что очередь вызовов может принимать вызовы:

1. Выберите **Выполнить тесты** ниже, чтобы заполнить диагностику в Центре администрирования Microsoft 365. 

   > [!div class="nextstepaction"]
   > [Выполнение тестов: Teams очереди вызовов](https://aka.ms/TeamsCallQueueDiag)

2. В области "Запуск диагностики" введите учетную запись ресурса в поле **"** Имя пользователя" или "Адрес электронной почты", а затем выберите " **Выполнить тесты"**.

3. Тесты возвратит лучшие дальнейшие действия для решения любых конфигураций клиента, политики и учетной записи ресурса, чтобы убедиться, что очередь вызовов может принимать вызовы.
