# <a name="create-a-call-queue"></a>Создание очереди вызовов

Очереди вызовов предоставляют способ маршрутизации вызывающих вызовов для людей в организации, которые могут помочь с определенной проблемой или вопросом. Звонки распределяются по одному людям в очереди (которые называются *агентами).* 

Очереди вызовов предоставляют:

- Приветствие.

- Музыка, пока люди находятся в очереди на удержание.

- Маршрутка вызовов ( *в порядке first In, First Out* (FIFO) для агентов.

- Параметры обработки для переполнения очереди и времени ожидания.

Прежде чем выполнять действия, которые вы выполните в [](plan-auto-attendant-call-queue.md#getting-started) этой статье, ознакомьтесь с планом для автоотетаря и очередей вызовов [Teams](plan-auto-attendant-call-queue.md) и следуйте этим шагам.

Чтобы настроить очередь вызовов, в Центре администрирования Teams раззовите "Голосовая команда", выберите "Очереди зовов" и нажмите кнопку  "Добавить". 

## <a name="resource-account-and-language"></a>Учетная запись ресурса и язык

![Снимок экрана: учетная запись ресурса и языковые параметры](media/call-queue-name-language.png)

1. Введите имя очереди звонка. Это имя будет видно агентам при входящих вызовах из очереди.

2. Нажмите **кнопку "Добавить учетные** записи", найдите учетную запись ресурса, которую вы хотите использовать с этой очередью вызовов, нажмите кнопку "Добавить", а затем нажмите кнопку **"Добавить".**

3. Выберите язык. Этот язык будет использоваться для системных голосовых подсказок и транскрибации голосовой почты (если вы их включаете).

## <a name="greetings-and-music-on-hold-in-queue"></a>Приветствия и музыка в очереди

Укажите, хотите ли вы воспроизведения приветствия вызывателям при их поступления в очередь. Необходимо отправить файл MP3, WAV или WMA, содержащий приветствие, которое вы хотите воспроизведения.

Teams предоставляет музыку по умолчанию вызывателям, пока они находятся на удержании в очереди. Если вы хотите воспроизведения определенного  звукового файла, выберите "Воспроизведения звукового файла" и загрузите файл MP3, WAV или WMA.

> [!NOTE]
> Размер добавленной записи не должен быть больше 5 МБ.
> По умолчанию музыка, которая предоставляется в очередях вызовов Teams, не предоставляется вашей организацией. 

## <a name="call-agents"></a>Агенты вызовов

Чтобы добавить [агентов](plan-auto-attendant-call-queue.md#prerequisites) в очередь звонка, обратитесь к условиям.

![Снимок экрана: параметры пользователей и групп для очередей вызовов](media/call-queue-users-groups.png)

Вы можете добавить до 20 агентов по отдельности и до 200 агентов с помощью групп.

Чтобы добавить пользователя в очередь, нажмите кнопку "Добавить пользователей", найдите нужного пользователя, нажмите кнопку "Добавить" и выберите "Добавить". 

Чтобы добавить группу в очередь, нажмите кнопку "Добавить **группы",** найдите группу, нажмите кнопку "Добавить" и выберите "Добавить".  Вы можете использовать списки рассылки, группы безопасности, группы Microsoft 365 или команды Microsoft Teams.

> [!NOTE]
> Для первого вызова новых пользователей, добавленных в группу, может быть до восьми часов.

## <a name="call-routing"></a>Маршрутизация вызовов

![Снимок экрана: режим конференции и параметры метода маршрутации](media/call-queue-conference-mode-routing-method.png)

**Режим конференц-связи** значительно сокращает время, необходимое для связи с агентом после того, как агент принимает звонок. Чтобы можно было работать в режиме конференции, агенты из очереди звонка должны использовать один из следующих клиентов:

  - Последняя версия клиента Microsoft Teams для настольных ПК, приложения Android или приложения для iOS
  - Телефон Microsoft Teams версии 1449/1.0.94.2020051601 или более поздней версии
  
Учетные записи Агентов в Teams должны быть настроены в режиме "Только teams". Агенты, не отвечающие требованиям, не включаются в список маршрутов вызовов. Мы рекомендуем использовать режим конференции для очередей звонка, если все ваши агенты используют совместимые клиенты.

> [!NOTE]
> Заняты в режиме занятости не поддерживается режимом конференции. Агенты, которые находятся в очереди звонков, могут по-прежнему получить вызов в очереди звонков, если не включена маршрутия на основе присутствия.

**Способ маршрутики определяет** порядок, в котором агенты получают звонки из очереди. Выберите один из этих вариантов:

- **Маршрутка с помощником** одновременно звонит всем агентам в очереди. Звонок получает первый агент, который звонит.

- **В последовательной маршрутке** все агенты звонят по одному в порядке, указанном в списке агентов **по звонкам.** Если агент отклоает или не берет звонок, звонок будет звонить следующему агенту и будет пытаться все агенты до тех пор, пока он не будет разозвать или невызовется.

- **Round balances** the routing of incoming calls so each call agent gets the same number of calls from the queue. Это может быть нежелательно в среде входящие продаж, чтобы гарантировать равные возможности для всех агентов по звонкам.

- **Наиболее длинные** неавтомитные маршруты каждый звонок агенту, который простаивает самый длительный срок. Агент считается неающим, если его состояние присутствия "В сети" или состояние присутствия "Нет на сети" менее 10 минут. Агенты, состояние присутствия которых отсутствует более 10 минут, не считаются неавными и не смогут получать звонки, пока не изменят свое состояние присутствия на "В сети". 

![Снимок экрана: параметры времени маршрутинга, отказа от работы и оповещений](media/call-queue-presence-agents-time.png)


**В маршрутике** на основе присутствия используется состояние доступности агентов для определения того, следует ли включить агента в список маршрутов для выбранного метода маршрутинга. Call agents whose availability status is **set to Available are** included in the call routing list and can receive calls. Агенты с любым другим состоянием доступности исключаются из списка маршрутов звонков и не будут получать звонки, пока их состояние не изменится на "В **наличии".** 

Вы можете включить маршрутную маршрутику на основе присутствия с помощью любого из ее способов.

Если агент отключит получение звонков, он не будет включен в список маршрутов звонков независимо от задайте для него состояние доступности. 

> [!NOTE]
> Агенты, которые используют клиент Skype для бизнеса, не включаются в список маршрутов звонков, если включена маршрутка на основе присутствия. Если у вас есть агенты, которые используют Skype для бизнеса, не в включаете маршруты звонков на основе присутствия.

**Время оповещения агента** определяет, как долго телефон агента будет звонить до того, как очередь перенаправит звонок следующему агенту.

Для очередей с высоким уровнем громкости рекомендуем использовать следующие параметры:

- **Режим конференции с** **автоматическим**
- **Способ маршрутинга для** **маршрутов Attendant**
- **Маршрут с учетом присутствия в "В** **сети"**
- **Время оповещения агента:** **до 20 секунд**

## <a name="call-overflow-handling"></a>Обработка переполнения вызовов

![Снимок экрана: параметры переполнения вызовов](media/call-queue-overflow-handling.png)

**Максимальное число звонков** в очереди определяет максимальное количество звонков, которые могут быть в очереди в любое время. Значение по умолчанию — 50, но может быть в диапазоне от 0 до 200. По такому ограничению звонок обрабатывается в том случае, если установлено максимальное число **звонков.**

Вы можете отключить звонок или перенаправить его в любую из маршрутизов. Например, вы можете оставить голосовое сообщение для агентов в очереди. Сведения о внешних [](plan-auto-attendant-call-queue.md#prerequisites) денежных переводах: предварительные условия и передача внешних номеров телефонов [—](create-a-phone-system-auto-attendant.md#external-phone-number-transfers---technical-details) технические сведения о формате номеров.

> [!NOTE]
> Если максимальное число звонков — 0, приветствие не будет воспроизведения.

## <a name="call-timeout-handling"></a>Обработка времени вызова

![Снимок экрана: параметры времени и времени вызова](media/call-queue-timeout-handling.png)

**Время ожидания звонка: максимальное** время ожидания определяет максимальное время удержания звонка в очереди перед его перенаправлением или отключением. Можно указать значение от 0 до 45 минут.

Вы можете отключить звонок или перенаправить его в одно из маршрутов. Например, вы можете оставить голосовое сообщение для агентов в очереди. Сведения о внешних [](plan-auto-attendant-call-queue.md#prerequisites) денежных переводах: предварительные условия и передача внешних номеров телефонов [—](create-a-phone-system-auto-attendant.md#external-phone-number-transfers---technical-details) технические сведения о формате номеров.

Выбрав параметры времени и времени и времени вызова, нажмите кнопку **"Сохранить".**

## <a name="caller-id-for-outbound-calls"></a>ИД вызываемой звонков для исходящие вызовы

Дополнительные сведения см. в сведениях об управлении политиками ИД вызываемой [службы в Microsoft Teams.](caller-id-policies.md)

## <a name="supported-clients"></a>Поддерживаемые клиенты

Для агентов, которые находятся в очереди звонка, поддерживаются следующие клиенты:

  - Клиент Skype для бизнеса для настольных ПК 2016 (32- и 64-битные версии)
  - Клиент Lync для настольных пк 2013 (32- и 64-битные версии)
  - Все модели IP-телефонов, поддерживаемые в Microsoft Teams. См. ["Получение телефонов для Skype для бизнеса Online".](/skypeforbusiness/what-is-phone-system-in-office-365/getting-phones-for-skype-for-business-online/getting-phones-for-skype-for-business-online)
  - Клиент Mac Skype для бизнеса (версия 16.8.196 и более поздние)
  - Клиент Android Skype для бизнеса (версия 6.16.0.9 и более поздние)
  - Клиент iPhone Skype для бизнеса (версия 6.16.0 и более поздние)
  - Клиент iPad Skype для бизнеса (версия 6.16.0 и более поздние)
  - Клиент Microsoft Teams для Windows (32- и 64-битные версии)
  - Клиент Microsoft Teams Mac
  - Приложение Microsoft Teams для iPhone
  - Приложение Microsoft Teams для Android

    > [!NOTE]
    > Очереди звонков, которые имеют прямой маршрутный номер, не поддерживают в качестве агентов клиенты Skype для бизнеса, клиенты Lync и IP-телефоны Skype для бизнеса.
