# <a name="create-a-cloud-call-queue"></a>Создание облачной очереди вызовов

Очереди облачных вызовов предоставляют следующие возможности.

- Сообщение приветствия.
- Воспроизведение музыки во время удержания звонка.
- Переадресация звонков операторам с использованием почтовых списков рассылки и групп безопасности.
- Задать различные параметры, такие как максимальный размер, таймаут и параметры обработки звонков.

Вы должны связать номер телефона с очередью звонков с помощью [учетной записи ресурса](manage-resource-accounts.md). Вы можете набрать очередь звонков напрямую или получить доступ к ней с помощью выбранного элемента автосекретаря.

Вызывающий абонент слышит музыку, пока они находятся на удержании, и Звонок подключается к агентам звонков в порядке " *первым вошел — первым вышел* " (FIFO).

Все звонки в очереди отправляются агентам одним из следующих способов:

- При использовании маршрутизации помощника первый вызов в очереди одновременно колец все агенты.
- При использовании последовательной маршрутизации первый вызов из очереди звонит по каждому из них по одному.
- При циклическом перераспределении Маршрутизация входящих звонков сбалансирована таким образом, что каждый агент вызова получает одинаковое количество звонков из очереди.

    > [!NOTE]
    > Агенты звонков, которые находятся в **автономном режиме**, имеют состояние " **не беспокоить** " или не переносятся в очередь звонков, не будут получать звонки.

- Только одно уведомление о входящем звонке (для звонка в заголовке очереди) в каждый момент времени отправляется агентам звонков.
- После того, как оператор принимает звонок, всем операторам поступает следующий входящий звонок из очереди.

> [!NOTE]
> Эта статья относится как в Microsoft Teams, так и в Skype для бизнеса Online.

**Метод маршрутизации** Вы можете выбрать **участника**, **последовательного**или **циклического** перераспределения для метода распространения очереди звонков. - **** Подставляются маршруты для входящих звонков, так что каждый агент вызывает одинаковое количество звонков из очереди. Это может быть предпочтительнее в входящей среде продаж для обеспечения возможной возможности между всеми агентами звонков.

**Максимальное количество вызовов в очереди** Используйте этот параметр для настройки максимального количества звонков, которые могут ожидать в очереди. Значение по умолчанию — 50, но может принимать значения от 0 до 200. По достижении этого предела Звонок обрабатывается так, как вы настроили, **когда достигнуто максимальное количество звонков** .

* * *

![Значок числа 2 с ссылкой на выноску на предыдущем снимке экрана](media/sfbcallout2.png)

**При достижении максимально допустимого количества звонков** Когда очередь звонков достигает максимального размера (с использованием **максимального количества звонков в** параметрах очереди), вы можете выбрать, что произойдет для новых входящих звонков.

- **Отключение** Звонок прерван.
- **Перенаправить на** Выбрав это, выберите один из указанных ниже вариантов.

  - **Сотрудники компании**. Пользователи, подключенные к сети, обладающие лицензией на **телефонную систему** и имеющие доступ к корпоративной голосовой связи, либо тарифный план. Вы можете настроить его таким образом, чтобы вызывающий абонент мог отправлять голосовую почту. Для этого выберите **пользователя в компании** и настройте для него звонки, переадресованные непосредственно в голосовую почту.

  Сведения о лицензиях, необходимых для голосовой почты, можно найти в статье [Настройка голосовой почты в облаке](set-up-phone-system-voicemail.md).

  - **Голосовое приложение** Выберите имя учетной записи ресурса, связанной с уже созданной очередью звонков или автоматическим ассистентом.

**Таймаут вызова: максимальное время ожидания** Кроме того, можно выбрать время, в течение которого звонок может быть задержан в очереди, прежде чем истекает время ожидания, и его необходимо переадресовать или отключить. Там, где он перенаправляется, зависит от того, как вы задаете время истечения **времени вызова** . Вы можете установить время в интервале от 0 до 45 минут.

Значение тайм-аута можно указать в секундах, с интервалом 15 секунд. Это позволяет управлять потоком вызовов с более высоким уровнем детализации. Например, вы можете указать, что любой звонок, не отвеченный агентом в течение 30 секунд, будет открывать автоматический секретарь поиска в каталоге.

- **Отключение** Звонок прерван.
- **Перенаправлять этот звонок** Выбрав этот вариант, вы можете использовать следующие параметры:
  - **Сотрудники компании**. Пользователи, подключенные к сети, обладающие лицензией на **телефонную систему**  и имеющие доступ к корпоративной голосовой связи, либо имеющие тарифный план. Ее можно настроить так, что звонки вызывающего абонента будут отправляться в голосовую почту. Для этого выберите **пользователя в компании** и настройте для него звонки, переадресованные непосредственно в голосовую почту.

  Сведения о лицензировании, требуемом для голосовой почты, можно найти в разделе [Настройка голосовой почты в облаке](set-up-phone-system-voicemail.md).

  - **Голосовое приложение** Выберите имя учетной записи ресурса, связанной с уже созданной очередью звонков или автоматическим ассистентом.

## <a name="change-a-users-caller-id-for-outbound-calls"></a>Изменение идентификатора звонящего пользователя для исходящих звонков

Вы можете защитить удостоверение пользователя, изменив его идентификатор вызывающего абонента для исходящих звонков в очередь звонков, автосекретарь или какой-либо номер службы вместо этого с помощью командлета **New-кскаллинглинеидентити** .

Для этого выполните команду:

``` Powershell
New-CsCallingLineIdentity -Identity "UKSalesQueue" -CallingIdSubstitute "Service" -ServiceNumber 14258828080 -EnableUserOverride $False -Verbose
```

Затем примените политику к пользователю, воспользовавшись командлетом **Grant-CallingLineIdentity**. Для этого выполните команду:

``` Powershell
Grant-CsCallingLineIdentity -PolicyName UKSalesQueue -Identity "AmosMarble@contoso.com"
```

Дополнительные сведения о том, как настроить параметры идентификатора вызывающего абонента в Организации, можно найти в статье [как можно использовать идентификатор звонящего в вашей организации](/microsoftteams/how-can-caller-id-be-used-in-your-organization).
