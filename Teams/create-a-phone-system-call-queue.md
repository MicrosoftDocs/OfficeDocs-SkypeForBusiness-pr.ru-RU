# <a name="create-a-cloud-call-queue"></a>Создание облачной очереди вызовов

Очереди облачных вызовов предоставляют следующие возможности.

- Сообщение приветствия.
- Воспроизведение музыки во время удержания звонка.
- Переадресация звонков операторам с использованием почтовых списков рассылки и групп безопасности.
- Задать различные параметры, такие как максимальный размер, таймаут и параметры обработки звонков.
- Общая Голосовая почта для вызывающих абонентов для выхода из сообщения для Организации.

Вы не можете напрямую связать номер телефона с очередью звонков, а номер телефона связан с [учетной записью ресурса](manage-resource-accounts.md). Вы можете набрать очередь звонков напрямую или получить доступ к ней с помощью выбранного элемента автосекретаря.

Вызывающий абонент слышит музыку, пока они находятся на удержании, и Звонок подключается к агентам звонков в порядке " *первым вошел — первым вышел* " (FIFO).

Все звонки в очереди отправляются агентам одним из следующих способов:

- При использовании маршрутизации помощника первый вызов в очереди одновременно колец все агенты.
- При использовании последовательной маршрутизации первый вызов из очереди звонит по каждому из них по одному.
- При циклическом перераспределении Маршрутизация входящих звонков сбалансирована таким образом, что каждый агент вызова получает одинаковое количество звонков из очереди.

Вы можете настроить параметры обработки звонков, такие как включение или отказ от агента, маршрутизация на основе присутствия, время ожидания вызова и время вызова для любого из описанных выше методов.

Только одно уведомление о входящем звонке (для звонка в заголовке очереди) в каждый момент времени отправляется агентам звонков. После того, как оператор принимает звонок, всем операторам поступает следующий входящий звонок из очереди.

> [!NOTE]
> Эта статья относится как в Microsoft Teams, так и в Skype для бизнеса Online.

## <a name="step-1--get-started"></a>Этап 1 — Начало работы

Перед началом работы с очередями звонков необходимо запомнить следующие моменты.

- Для связи с учетной записью ресурса требуется очередь звонков. Дополнительные сведения об учетных записях ресурсов содержатся [в разделе Управление учетными записями ресурсов в Teams](manage-resource-accounts.md) .

Телефонная система позволяет использовать номера телефонов на уровне Организации для автосекретаря по низким тарифам и для службы очередей звонков.

> [!NOTE]
> Номера услуг прямой маршрутизации для очередей звонков поддерживаются только для пользователей и агентов Microsoft Teams.

Чтобы получить номера услуг, ознакомьтесь со статьей [получение телефонных номеров служб](getting-service-phone-numbers.md) или перенесите существующий номер Услуги в разделе [Перенос номеров телефонов в Office 365](phone-number-calling-plans/transfer-phone-numbers-to-teams.md). Бесплатные телефонные номера будут перечислены в списке с **типом** **услуги (** бесплатный номер телефона).

> [!NOTE]
> За пределами США нельзя использовать центр администрирования Microsoft Teams для получения номеров служб. Чтобы узнать, как сделать это из за пределами США, перейдите в раздел [Управление телефонными номерами в Организации](manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization.md) .

При настройке нескольких автосекретарей обычно назначается номер телефона основной учетной записи автоматического ассистента. Для учетных записей ресурсов, связанных с вложенными автосекретарем или очередями звонков, часто не нужны номера телефонов. Этот автоматический секретарь может перенаправлять абонентов в очереди звонков или на вложенные автоматические ассистенты, даже если у них нет номера телефона. В этих случаях вы можете создать все автосекретарей и очереди звонков в системе без назначения параметров панель набора номера и затем изменить параметры позже. Чтобы настроить ее как параметр меню, должна существовать очередь звонков или автосекретарь.

Агенты, которые не входят в список маршрутизации звонков, не будут получать звонки. Если у вас есть агенты, использующие Skype для бизнеса, не включайте маршрутизацию звонков на основе присутствия.

Для получения доступа к параметру отказа агенты могут:

 3. На странице Параметры пользователя нажмите кнопку **очереди звонков**, а затем снимите флажки, чтобы отказаться от использования очередей.

    >
    > Если агенты находятся в настольных клиентах Microsoft Teams, они могут отказаться от использования параметров звонка.

## <a name="change-caller-id-for-outbound-calls"></a>Изменение идентификатора вызывающего абонента для исходящих звонков

Чтобы защитить удостоверение агента звонков, измените идентификатор вызывающего абонента для исходящих звонков на очередь звонков, автосекретарь или любой номер службы с помощью командлета **New-CsCallingLineIdentity** , как показано в следующем примере:

``` Powershell
New-CsCallingLineIdentity -Identity "UKSalesQueue" -CallingIdSubstitute "Service" -ServiceNumber 14258828080 -EnableUserOverride $False -Verbose
```

Затем примените политику к пользователю с помощью командлета **Grant-CallingLineIdentity** , как показано в следующем примере: 

``` Powershell
Grant-CsCallingLineIdentity -PolicyName UKSalesQueue -Identity "AmosMarble@contoso.com"
```

Дополнительные сведения о том, [как можно использовать идентификатор вызывающего абонента в вашей организации](/microsoftteams/how-can-caller-id-be-used-in-your-organization).
