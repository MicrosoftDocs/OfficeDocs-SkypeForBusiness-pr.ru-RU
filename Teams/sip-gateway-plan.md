# <a name="plan-for-sip-gateway"></a>Планирование шлюза SIP

Шлюз SIP позволяет вашей организации использовать любое совместимое SIP-устройство с Майкрософт Teams для сохранения инвестиций в SIP-устройства. Теперь вы можете войти в Teams с корпоративными учетными данными, а также совершать и принимать звонки с помощью совместимого SIP-устройства. Совместимые устройства могут быть Skype для бизнеса IP-телефонов со стандартным встроенным ПО SIP, IP-телефонов Cisco с встроенным ПО SIP с несколькими платформами или SIP-устройств от таких поставщиков, как Poly, Yealink и AudioCodes. Сведения о настройке sip-устройств для шлюза SIP см. в статье [Настройка шлюза SIP](sip-gateway-configure.md).

## <a name="benefits-of-sip-gateway"></a>Преимущества шлюза SIP

Шлюз SIP подключает совместимые sip-устройства к Teams, чтобы помочь пользователям легко перейти на телефонию Teams. С помощью шлюза SIP пользователи могут выполнять все указанные ниже действия.

- **Совершать звонки:** Пользователи устройств SIP могут совершать звонки в телефонную сеть общего пользования (ТСОП), на другие устройства SIP, а также в Teams и Skype для бизнеса пользователей. Пользователи устройств SIP могут звонить только пользователям с номерами телефонов.
- **Прием вызовов:** Пользователи устройств SIP могут принимать вызовы из ТСОП, из Teams или Skype для бизнеса пользователей с устройствами SIP, а также из Teams и Skype для бизнеса клиентских приложений. Устройство SIP выступает в качестве конечной точки Teams. Входящие вызовы также будут вилкой на SIP-устройство пользователя.
- **Несколько одновременных вызовов:** Пользователь SIP-устройства в вызове может поставить вызов на удержание, чтобы совершать или принимать другие вызовы. Пользователь устройства SIP также может выполнять две конференции.
- **Не беспокоить:** Пользователь sip-устройства может настроить параметр "Не беспокоить" на устройстве, чтобы устройство не звонило для входящих вызовов. Это не влияет на состояние пользователя во всех остальных конечных точках Teams.
- **Удержание, возобновление и отключение звука или включение звука:** Пользователь sip-устройства может удерживать и возобновлять или отключать и включить вызов с помощью функций для этих действий на устройстве.
- **Голосовую почту:** Пользователи устройств SIP могут прослушивать электронные голосовые сообщения, которые звонящие оставляют для них.
- **Индикатор ожидания сообщения:** Пользователи устройств SIP могут получать уведомления, предупреждающие их о новых сообщениях голосовой почты.
- **Вход и выход:** Пользователи sip-устройств могут входить и выходить из Teams на устройстве.
- **Двухцветная многочастотная:** Пользователи устройств SIP могут нажать числовые клавиши для ввода данных во время интерактивных вызовов голосового ответа.
- **Собрания Teams:** Пользователь устройства SIP может присоединиться к собранию Teams, набрав номер доступа к собранию. Участники собрания могут добавить пользователя sip-устройства в собрание, набрав номер телефона пользователя или просто добавив участника, щелкнув "Запрос на присоединение", также оповещает sip-устройство пользователя. Гостевые пользователи из другой организации могут быть добавлены в собрание Teams участником, который набирает номер гостевого пользователя, чтобы включить этого гостя.
- **Передача звонков:** Пользователи устройств SIP могут передавать вызовы. Шлюз SIP поддерживает как передачу вслепую, так и передачу консультаций.
- **Локальная переадресация звонков:** Пользователь sip-устройства может задать правила переадресации (всегда, по истечении времени ожидания и занятости) для устройства. Если устройство подключено к шлюзу SIP, вызов будет перенаправляться на целевой адрес на основе правила, установленного пользователем устройства. Чтобы обеспечить работу локальной переадресации вызовов, администратор должен задать `AllowCallRedirect` атрибуту в `Set-CsTeamsCallingPolicy` значение `Enabled`.
- **Отключенные устаревшие устройства:** Шлюз SIP поддерживает автоматическое отключение устаревших устройств, подготовленных для клиента. Сопряженные (входные) устройства будут отключены, если они не подключены в течение 30 дней, а непарные (выписанные) устройства будут отключены через 14 дней. Отключенное устройство можно повторно подключить после сброса до заводских настроек.

## <a name="requirements-to-use-sip-gateway"></a>Требования к использованию шлюза SIP

Для использования SIP-шлюза у пользователей Teams должен быть номер телефона с включенным вызовом по ТСОП.

> [!NOTE]
> Шлюз SIP недоступен для сред государственных организаций (GCC, GCC High и DoD).

### <a name="hardware-software-and-licenses"></a>Оборудование, программное обеспечение и лицензии

Если у вас есть устройство 3PIP или SIP, необходимо:

- Лицензия на Майкрософт Teams, Skype для бизнеса online plan 2 и Майкрософт 365 Phone System (через E5 или автономные лицензии)
- Включение ТСОП (т. е. номер телефона) через план звонков Майкрософт Teams, прямую маршрутизацию или подключение оператора
- Лицензия **на общие устройства Teams Майкрософт** для любых устройств с общим доступом
