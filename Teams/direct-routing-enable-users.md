# <a name="enable-users-for-direct-routing"></a>Включение прямой маршрутизации для пользователей

В этой статье описывается, как разрешить пользователям прямую маршрутизацию. Это шаг 2 из следующих действий по настройке прямой маршрутизации:

- Шаг 1. [Подключение SBC с помощью телефонная система и проверьте подключение.](direct-routing-connect-the-sbc.md) 
- **Шаг 2. Включение прямой маршрутизации**   для пользователей (в этой статье)
- Шаг 3. [Настройка маршрутизации голосовой связи](direct-routing-voice-routing.md)
- Шаг 4. [Перевод чисел в альтернативный формат](direct-routing-translate-numbers.md) 


Сведения о всех шагах, необходимых для настройки прямой маршрутизации, см. в разделе ["Настройка прямой маршрутизации"](direct-routing-configure.md).

Когда вы будете готовы включить прямую маршрутизацию для пользователей, выполните следующие действия. 

1. Создайте пользователя в Microsoft 365 и назначьте телефонная система лицензию.  
2. Убедитесь, что пользователь размещен в Интернете.
3. Настройте номер телефона и включите корпоративную голосовую связь. 
4. Назначьте Teams только пользователям.

## <a name="create-a-user-and-assign-the-license"></a>Создание пользователя и назначение лицензии

Существует два варианта создания нового пользователя в Microsoft 365. Однако корпорация Майкрософт рекомендует вашей организации выбрать один из вариантов, чтобы избежать проблем с маршрутиза службой. 

- Создайте пользователя в локальная служба Active Directory синхронизации пользователя с облаком. См[. статью "Интеграция локальных каталогов с Azure Active Directory](/azure/active-directory/connect/active-directory-aadconnect)".
- Создайте пользователя непосредственно в Центр администрирования Microsoft 365. См[. раздел "Добавление пользователей по отдельности или в пакетном режиме Microsoft 365 или Office 365 справку администратора](https://support.office.com/article/Add-users-individually-or-in-bulk-to-Office-365-Admin-Help-1970f7d6-03b5-442f-b385-5880b9c256ec). 

Если развертывание Skype для бизнеса Online сосуществоват с локальной версией Skype для бизнеса 2015 или Lync 2010 или 2013, единственным поддерживаемым вариантом является создание пользователя в локальная служба Active Directory и синхронизация пользователя с облаком (вариант 1). 

Сведения о требованиях к лицензиям см [. в разделе о лицензировании](direct-routing-plan.md#licensing-and-other-requirements) и других требованиях в [плане прямой маршрутизации](direct-routing-plan.md).

## <a name="ensure-that-the-user-is-homed-online"></a>Убедитесь, что пользователь размещен в Сети 

Этот шаг относится к Skype для бизнеса Server Корпоративная голосовая связь пользователей, которые переносятся Teams прямой маршрутизации.

Для прямой маршрутизации требуется, чтобы пользователь был размещен в Сети. Вы можете проверить параметр RegistrarPool, который должен иметь значение в infra.lync.com домене. Корпорация Майкрософт рекомендует, но не требует, чтобы при переносе пользователей на прямую маршрутизацию Teams lineURI из локальной среды в интернет.) 

1. Подключение сеанс Microsoft Teams PowerShell.

2. Выполните команду: 

    ```PowerShell
    Get-CsOnlineUser -Identity "<User name>" | fl RegistrarPool,OnPremLineUri,LineUri
    ``` 
    Если onPremLineUri заполняется номером <E.164>, номер телефона был назначен локально и синхронизирован с Microsoft 365. Если вы хотите управлять номером телефона в Сети, очистите параметр с помощью локальной командной консоли Skype для бизнеса и синхронизируются с Microsoft 365 перед настройкой номера телефона с помощью Teams PowerShell. 

1. В Skype для бизнеса Management Shell выполните следующую команду: 

   ```PowerShell
   Set-CsUser -Identity "<User name>" -LineUri $null
    ``` 
 > [!NOTE]
 > Не задайте Для EnterpriseVoiceEnabled значение False, так как это не требуется, и это может привести к проблеме нормализации абонентской группы, если используются устаревшие телефоны Skype для бизнеса и гибридная конфигурация клиента задана с помощью UseOnPremDialPlan $True. 
    
   После синхронизации изменений с Microsoft 365 ожидаемые выходные данные`Get-CsOnlineUser -Identity "<User name>" | fl RegistrarPool,OnPremLineUri,LineUri`:

   ```console
   RegistrarPool                        : pool.infra.lync.com
   OnPremLineURI                        : 
   LineURI                              : 
   ```
 > [!NOTE]
 > Все атрибуты телефона пользователя должны управляться через Интернет, прежде чем вы деконструируете локальную Skype для бизнеса [среду](/skypeforbusiness/hybrid/decommission-on-prem-overview). 

## <a name="configure-the-phone-number-and-enable-enterprise-voice"></a>Настройка номера телефона и включение корпоративной голосовой связи 

После создания пользователя и назначения лицензии необходимо настроить параметры телефона пользователя в Сети. Настройка облачная голосовая почта для пользователя выполняется автоматически; других действий по настройке не требуется.

Номер телефона можно настроить с помощью Teams центра администрирования или Teams PowerShell.

### <a name="use-teams-admin-center"></a>Использование Teams центра администрирования

1. Перейдите к **пользователям** **UsersManage** -> .

2. Выберите пользователя.

2. В **разделе "Общие сведения** **об учетной записи**" выберите " **Изменить"**.

3. В **разделе "Назначить номер** телефона **" в раскрывающемся Телефон типа** номера выберите прямую **маршрутизацию**.

4. Введите назначенный номер телефона и расширение номера телефона, если это применимо.

5. Нажмите **кнопку "Применить".**

Общие сведения об учетной записи теперь отображают назначенный номер телефона и прямую маршрутизацию в качестве типа номера телефона.


### <a name="use-powershell"></a>Использование PowerShell

1. Подключение к Microsoft Teams PowerShell. 

2. Дальнейшие действия зависят от того, управляете ли вы номером телефона пользователя в локальной или сетевой среде. Если вы управляете номером телефона в локальной среде, необходимо использовать локальную командную консоль Skype для бизнеса, панель управления или один из методов, описанных в разделе "Решение об управлении атрибутами после вывода из эксплуатации[".](/skypeforbusiness/hybrid/cloud-consolidation-managing-attributes)

   - Если вы управляете номером телефона пользователя в локальной среде, убедитесь, что пользователь Корпоративная голосовая связь в сети с помощью следующей команды:

       ```PowerShell
       Set-CsPhoneNumberAssignment -Identity "<User name>" -EnterpriseVoiceEnabled $true
       ```
       
   - Если вы управляете номером телефона пользователя в Интернете, необходимо назначить его пользователю с помощью следующей команды в Teams PowerShell. Пользователь автоматически Корпоративная голосовая связь с помощью команды: 
 
       ```PowerShell
       Set-CsPhoneNumberAssignment -Identity "<User name>" -PhoneNumber <phone number> -PhoneNumberType DirectRouting
       ```
    
       Например, чтобы добавить номер телефона для пользователя "Low Low", введите следующее: 

       ```PowerShell
       Set-CsPhoneNumberAssignment -Identity "spencer.low@contoso.com" -PhoneNumber "+14255388797" -PhoneNumberType DirectRouting
       ```
       Если пользователи "Low Low" и "Stacy Quinn" совместно используют одно и то же базовое число с уникальными расширениями, введите следующее:
    
       ```PowerShell
       Set-CsPhoneNumberAssignment -Identity "spencer.low@contoso.com" -PhoneNumber "+14255388701;ext=1001" -PhoneNumberType DirectRouting
       Set-CsPhoneNumberAssignment -Identity "stacy.quinn@contoso.com" -PhoneNumber "+14255388701;ext=1002" -PhoneNumberType DirectRouting
       ```

    Корпорация Майкрософт рекомендует, но не требует, чтобы номер телефона был настроен как полный номер телефона E.164 с кодом страны. Номера телефонов можно настроить с помощью расширений. Эти расширения будут использоваться для поиска пользователей, когда поиск по базовому номеру возвращает несколько результатов. Эта функция позволяет компаниям настраивать номера телефонов с одинаковым базовым номером и уникальными расширениями. Чтобы поиск был успешным, приглашение должно содержать полный номер с расширением следующим образом:
    
    ```PowerShell
    To: <sip:+14255388701;ext=1001@sbc1.adatum.biz
    ```


## <a name="configure-sending-calls-directly-to-voicemail"></a>Настройка отправки звонков непосредственно в голосовую почту

Прямая маршрутизация позволяет завершить звонок пользователю и отправить его непосредственно в голосовую почту пользователя. Если вы хотите отправить звонок непосредственно в голосовую почту, присоедините opaque=app:voicemail к заголовку URI запроса. Например, sip:user@yourdomain.com;opaque=app:voicemail. Пользователь Teams не получит уведомление о вызове. Вместо этого вызов будет напрямую подключен к голосовой почте пользователя.

## <a name="assign-teams-only-mode-to-users-to-ensure-calls-land-in-microsoft-teams"></a>Назначьте Teams только пользователям, чтобы гарантировать, что вызовы будут Microsoft Teams

Для прямой маршрутизации необходимо, чтобы пользователи Teams режиме только для того, чтобы входящие вызовы Teams клиенте. Чтобы перевести пользователей в Teams", назначьте им экземпляр UpgradeToTeams TeamsUpgradePolicy. Дополнительные сведения см. в [статье "Стратегии обновления для ИТ-администраторов"](upgrade-to-teams-on-prem-implement.md). Если ваша организация использует Skype для бизнеса Server, см. сведения о взаимодействии между Skype и [Teams:](migration-interop-guidance-for-teams-with-skype.md) миграция и взаимодействие с Skype для бизнеса.
