# <a name="information-barriers-in-microsoft-teams-preview"></a>Информационные барьеры в предварительной версии Microsoft Teams

> [!INCLUDE [Preview feature](includes/preview-feature.md)]

Сведения о препятствующих, политик, которые администратор может настроить для предотвращения отдельных пользователей или групп обмениваться данными друг с другом. Это полезно, если, например одного подразделения обрабатывает сведения, которые не следует использовать совместно с другими отделы или группу необходимо запрещено обмениваться данными с любыми за пределами контакты.

> [!NOTE]
> - Сведения о барьер группы не могут создаваться клиентов.
> - С помощью программы-роботы для добавления пользователей не поддерживается в версии 1.
> - Сведения о препятствующих версии 1 сюда не включены поддержка для SharePoint и OneDrive для бизнеса. Мы работа на включение компонента в SharePoint и будет обмениваться данными, когда он доступен.

Сведения о политиках барьер также запретить поиска и обнаружения. Это означает, что при попытке общаться с кем вы должны не взаимодействуют с не сможете найти этого пользователя в средстве выбора людей.

## <a name="background"></a>Общие сведения

Основным средством для препятствующих данные, поступающие из финансовых служб. Финансовые отрасли нормативные центра сертификации ([FINRA]( http://www.finra.org/)) рассматриваются препятствующих сведения и конфликты интересов внутри фирмы члена, а также приводятся рекомендации о том, как управлять такие конфликты (FINRA 2241, [Долг Research соответствию стандартам 15-31](http://www.finra.org/sites/default/files/Regulatory-Notice-15-31_0.pdf).  

## <a name="when-should-i-use-information-barriers"></a>Когда следует использовать препятствующих сведения?

Можно использовать препятствующих сведения в подобных ситуациях:

- Группа должен быть запрещен связи или общий доступ к данным с определенным других групп.
- Команды не должен связи или общего доступа к данным с кем-либо вне группы.

Сведения о барьер политики оценки служба определяет, соответствует ли связи политики барьер сведениями.

## <a name="when-are-information-barrier-policies-checked"></a>Когда проверяется политик барьер сведения?

Сведения о политиках барьер проверяется при возникновении следующих событий групп:

- **Элементы добавляются в группу** - при добавлении пользователя в группу, политики пользователя для его вычисления политики барьер сведениями из других участников группы. После успешного добавления пользователя, пользователь может выполнять все функции из группы без дальнейшей проверки. Если политика пользователя блокирует их добавляются в группу, пользователь не будет отображаться в поиска.
- **Запрашивается новый чата** - каждый раз нового чата запрашивается между двумя или несколькими пользователями чата вычисляется для убедитесь в том, что оно не будет нарушения любые политики барьер сведения. Если беседа нарушает политики барьер сведениями, затем сеанс не будет начат и отображается сообщение об ошибке.
- **Пригласить пользователя на присоединение к собранию** — Если пользователь приглашен для присоединения к собранию политике сравнивается политик другими участниками группы, и если нарушение, пользователь не смогут присоединиться к собранию и появится сообщение об ошибке.
- **Экран распределено между двумя или несколькими пользователями** - любое время, на экране будут общими у этих двух или нескольких пользователей, на экране общий ресурс должен быть равен убедитесь в том, что она не нарушает политики барьер сведения о других пользователей. Если нарушается политики барьер сведениями, не будут разрешены к общей папке экрана и будет выведено сообщение об ошибке.
- **Пользователь помещает телефонного звонка (VOIP) в группах** - каждый раз, когда запускается голосовой звонок на другого пользователя или группы пользователей, вызов пользователем вычисляется убедитесь в том, что она не нарушает политики барьер сведениями oher членов группы. Если все нарушения, блокируется голосовой звонок.

## <a name="what-happens-to-existing-chat-threads-when-a-policy-is-changed"></a>Что происходит с существующей чата потоков при изменении политики?

Когда политики администрирования барьер сведения вносятся изменения в политике или изменение политики включается в силу из-за изменения задания пользователя или аналогично, служба оценки политики барьер сведения автоматически выполняет поиск участников, чтобы убедиться, что Участники группы не нарушают любые политики. 

Если имеется существующий чата или другие связи между пользователями и задать политику или изменить существующую политику, службу оценивает существующие связи убедитесь в том, что они не «как поврежденные использующим» (больше не разрешены): 

- **1:1 чата** - Если обмена данными между двумя пользователями больше не разрешается (если для одного или обоих пользователей применяется политика блокировки связи), дальнейшей заблокирован обмена данными и чата беседы будут доступны только для чтения.
- **Групповой чат** - от одного пользователя в группу больше не разрешен (например, если пользователь изменяет заданий), пользователя наряду с другими пользователями, нарушает политики могут быть удалены из групповой беседы и дальнейший связь с группой не будет допускается. Пользователь по-прежнему могут видеть старые беседы (которые будут доступны только для чтения), но не сможет увидеть или вести любой новой беседы с группой. Если новые или измененные политики, препятствующие связи применяется к нескольким пользователям, пользователи, затрагиваемые политики могут быть удалены из групповой беседы. Они по-прежнему могут просматривать старые беседы. 
- **Группа** - пользователей, которые были удалены из группы, удаляются из группы и не сможет увидеть или участвовать в беседах существующего или нового.
