# <a name="set-up-cloud-voicemail"></a>Настройка облачной голосовой почты

Эта статья предназначена для администраторов Microsoft 365, которые хотят настроить облачная голосовая почта для своих пользователей.

облачная голосовая почта помещает сообщения голосовой почты в почтовый ящик Пользователя Exchange. облачная голосовая почта не поддерживает сторонние почтовые системы. Требования к лицензированию Exchange Online см. [в описании службы Exchange Online](/office365/servicedescriptions/exchange-online-service-description/exchange-online-service-description#features-available-to-all-plans). Дополнительные сведения о ролях администраторов см. в разделе [Сведения о ролях администратора](/microsoft-365/admin/add-users/about-admin-roles).

## <a name="cloud-voicemail-provisioning"></a>подготовка облачная голосовая почта

Для пользователей Teams облачная голосовая почта настраивается и подготавливается автоматически. *Лицензия Телефонная система Microsoft Teams не требуется для облачная голосовая почта.*

Подготовка пользователей Teams не совпадает с подготовкой пользователей Skype для бизнеса Online. Для пользователей Skype для бизнеса Online облачная голосовая почта была автоматически настроена и подготовлена, когда пользователям была назначена лицензия телефонной системы и Корпоративная голосовая связь включена система подготовки.

Для Skype для бизнеса Server локальных пользователей облачная голосовая почта настраивается и подготавливается автоматически. Однако необходимо настроить среду Skype для бизнеса Server для маршрутизации вызовов в облачная голосовая почта. Дополнительные сведения см. в статье [Планирование службы облачная голосовая почта для локальных пользователей](/skypeforbusiness/hybrid/plan-cloud-voicemail).

## <a name="cloud-voicemail-storage"></a>хранилище облачная голосовая почта

Сообщения голосовой почты, созданные облачная голосовая почта, доставляются и хранятся в почтовом ящике Exchange пользователя в Exchange Online или в Exchange Server. Для голосовой почты нет специального или дополнительного хранилища. Клиенты, обращаюющиеся к голосовой почте (например, Microsoft Outlook, Microsoft Teams или Skype для бизнеса), делают это через почтовый ящик Exchange и предоставленные API.

Сообщение голосовой почты содержит вложенный звуковой файл с голосовыми сообщениями и транскрибированием голосовой почты (если включено).

В почтовом ящике Exchange пользователя хранятся все пользовательские записанные приветствия. Эти приветствия извлекаются облачная голосовая почта по мере необходимости во время входящего звонка.

## <a name="cloud-voicemail-processing"></a>обработка облачная голосовая почта

Запись и транскрибирование облачная голосовая почта начинается в Microsoft 365 с начала вызова, направляемого в облачная голосовая почта. Затем сообщение доставляется в почтовый ящик Exchange пользователя.

Например, если звонок поступает к недоступному пользователю прямой маршрутизации через пограничный контроллер сеансов (SBC) в Европе, запись и транскрибирование голосовой почты выполняются в Европе. Затем сообщение доставляется в почтовый ящик Exchange пользователя. В качестве другого примера предположим, что пользователь Teams в Северная Америка вызывает недоступного пользователя Teams в Европе. В этом случае вызов начинается в Северная Америка, обработка выполняется в Северная Америка, а затем голосовая почта доставляется в почтовый ящик пользователя Exchange в Европе.

Доставка голосовой почты в почтовый ящик Exchange выполняется по протоколу SMTP, как и любая другая электронная почта.

## <a name="manage-cloud-voicemail-for-users"></a>Управление облачная голосовая почта для пользователей

Вы можете управлять облачная голосовая почта для пользователей, указав политики голосовой почты и настроив параметры голосовой почты.  

- **Политики голосовой почты** позволяют управлять функциями для групп пользователей. Вы можете настроить и назначить существующие или новые политики голосовой почты для таких функций, как правила ответа на вызовы, транскрибирование голосовой почты, маскирование ненормативной лексики транскрибирования, перевод транскрибирования и язык системных запросов. Сведения об управлении политиками голосовой почты см. в разделе [Управление политиками голосовой почты](manage-voicemail-policies.md).

- **Параметры голосовой почты** позволяют настраивать параметры для отдельных пользователей. Вы можете настроить такие параметры, как правила ответа на вызовы, язык запросов, текст в речь по умолчанию и приветствия с отпуском. Сведения о настройке параметров для отдельных пользователей см. в разделе [Управление параметрами голосовой почты](manage-voicemail-settings.md). Обратите внимание, что пользователи также могут настроить эти параметры в клиенте Teams, перейдя в **раздел Параметры -> Вызовы -> Настройка голосовой почты**.

## <a name="control-routing-of-calls-to-cloud-voicemail"></a>Управление маршрутизацией вызовов в облачная голосовая почта

Параметр по умолчанию для всех пользователей, подготовленных для облачная голосовая почта, — разрешить маршрутизацию вызовов в облачная голосовая почта и разрешить пользователям переадресовывать вызовы в облачная голосовая почта.

Вы можете контролировать, разрешена ли маршрутизация вызовов к облачная голосовая почта пользователям Teams, с помощью Центра администрирования Teams или с помощью PowerShell. 

- Чтобы использовать Центр администрирования Teams, перейдите в раздел Политики **голосовых** -> **звонков** - > добавьте новую или измените существующую политику > **Голосовая почта доступна для маршрутизации входящих вызовов**.  

- В PowerShell используйте командлет Set-CsTeamsCallingPolicy с параметром AllowVoicemail. Дополнительные сведения см. в разделе [Set-CsTeamsCallingPolicy](/powershell/module/skype/set-csteamscallingpolicy).

  - Если для параметра AllowVoicemail задано значение AlwaysDisabled, вызовы никогда не перенаправляются в голосовую почту, независимо от параметров переадресации или безответного вызова для пользователя. Голосовая почта недоступна как параметр переадресации звонков или без ответа в Teams.

  - Если для параметра AllowVoicemail задано значение AlwaysEnabled, вызовы всегда перенаправляются в голосовую почту без ответа после звонка в течение тридцати секунд независимо от параметра перенаправления вызовов без ответа для пользователя.

  - Если для параметра AllowVoicemail задано значение UserOverride, звонки перенаправляются в голосовую почту на основе параметров переадресации звонков и (или) без ответа для пользователя.

## <a name="set-up-cloud-voicemail-to-work-with-on-premises-users"></a>Настройка облачная голосовая почта для работы с локальными пользователями

Вы можете использовать облачная голосовая почта для предоставления функций голосовой почты пользователям, у которых есть почтовые ящики на серверах Exchange Server, или для пользователей, использующих Skype для бизнеса Server.

В этом разделе описывается настройка облачная голосовая почта для пользователей почтовых ящиков Exchange Server. Сведения о настройке облачная голосовая почта для пользователей Skype для бизнеса Server см. в статье [Планирование службы облачная голосовая почта для локальных пользователей](/skypeforbusiness/hybrid/plan-cloud-voicemail).

### <a name="set-up-cloud-voicemail-for-exchange-server-mailbox-users"></a>Настройка облачная голосовая почта для пользователей почтовых ящиков Exchange Server

Ниже приведены сведения о настройке облачная голосовая почта для работы с пользователями Teams, у которых есть почтовый ящик на Exchange Server.

1. Сообщения голосовой почты доставляются в почтовый ящик Пользователя Exchange через SMTP,перенаправленные через Exchange Online Protection. Чтобы обеспечить успешную доставку этих сообщений, убедитесь, что соединители Exchange правильно настроены между серверами Exchange и Exchange Online Protection. Дополнительные сведения см. в разделе [Использование соединителей для настройки потока обработки почты](/exchange/mail-flow-best-practices/use-connectors-to-configure-mail-flow/use-connectors-to-configure-mail-flow).

2. Чтобы включить функции голосовой почты, такие как настройка приветствий и визуальной голосовой почты, в клиентах Teams необходимо настроить подключение между Microsoft 365 и почтовым ящиком Exchange Server. Чтобы включить это подключение, необходимо настроить новый протокол проверки подлинности Oauth Exchange, описанный в статье [Настройка проверки подлинности OAuth между Exchange и Exchange Online организациями](/exchange/configure-oauth-authentication-between-exchange-and-exchange-online-organizations-exchange-2013-help), или запустить мастер гибридного развертывания Exchange из Exchange 2013 с накопительным пакетом обновления 5 (CU5) или более поздней версии. Необходимо также настроить интеграцию и Oauth между Teams и Exchange Server описано в разделе [Настройка интеграции и OAuth между Teams и Exchange Server](/skypeforbusiness/deploy/integrate-with-exchange-server/oauth-with-online-and-on-premises).

## <a name="enable-protected-voicemail-in-your-organization"></a>Включение защищенной голосовой почты в организации

Когда кто-то оставляет сообщение голосовой почты для пользователя в вашей организации, голосовая почта доставляется в почтовый ящик пользователя в виде вложения сообщения электронной почты.

С помощью Защита информации Microsoft Purview можно зашифровать сообщения голосовой почты, оставленные как внутренними, так и внешними вызывающими. Вы также можете запретить пользователю пересылать эти сообщения. Эта функция поддерживается для пользователей с Exchange Online почтовыми ящиками.

Чтобы зашифровать сообщение голосовой почты, можно создать метку конфиденциальности. С помощью функции автоматической маркировки вы можете убедиться, что метка будет автоматически применена к входящим сообщениям голосовой почты.

При включении защищенной голосовой почты пользователи могут прослушивать защищенные сообщения голосовой почты, открыв их в Outlook, Outlook в Интернете, Outlook для Android или iOS. Защищенные сообщения голосовой почты нельзя открыть в Microsoft Teams или Skype для бизнеса.

Сведения о создании метки конфиденциальности для голосовой почты см. в статье [Использование меток конфиденциальности](/microsoft-365/compliance/encryption-sensitivity-labels#let-users-assign-permissions). В разделе **Шифрование** выберите **Разрешить пользователям назначать разрешения при применении метки**. Выберите **В Outlook, примените одно из следующих ограничений**, а затем выберите параметр **Не пересылать** .

Сведения о том, как создать политику автоматической маркировки для применения метки конфиденциальности к голосовой почте, см. в статье [Настройка политик автоматической маркировки](/microsoft-365/compliance/apply-sensitivity-label-automatically#how-to-configure-auto-labeling-policies-for-sharepoint-onedrive-and-exchange) и указание следующих параметров:

- В поле **Выберите сведения, к которым применяется эта метка**, выберите **Пользовательская политика**.

- В **поле Выберите расположения, к которым требуется применить метку**, выберите **Расположения: Exchange для всех пользователей**.

- В  **разделе Настройка общих или расширенных правил** выберите **Дополнительные правила**.

- Правила exchange:
  - Условия:
    - **Заголовок соответствует шаблону**: Content-Class = Voice-CA
    - **IP-адрес отправителя**: 13.107.64.0/18, 52.112.0.0/14, 52.122.0.0/15, 52.238.119.141/32, 52.244.160.207/322

- В поле **Выберите метку для автоматического применения** выберите метку конфиденциальности, созданную для голосовой почты на предыдущем шаге.

- Для **получения дополнительных параметров электронной почты** выберите **Применить шифрование к электронной почте, полученной из-за пределов организации**, и укажите владельца Rights Management.

Диапазоны IP-адресов версии 4, указанные в поле IP-адрес отправителя, основаны на списке с идентификатором 12 в [Office 365 URL-адресах и диапазонах IP-адресов](/microsoft-365/enterprise/urls-and-ip-address-ranges#skype-for-business-online-and-microsoft-teams).

Дополнительные сведения о шифровании сообщений см. в статье [Определение правил потока обработки почты для шифрования сообщений электронной почты](/microsoft-365/compliance/define-mail-flow-rules-to-encrypt-email).
