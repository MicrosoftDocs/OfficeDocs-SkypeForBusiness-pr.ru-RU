# <a name="set-up-cloud-voicemail"></a>Настройка облачной голосовой почты

Эта статья для администраторов Microsoft 365, которые хотят настроить облачная голосовая почта для своих пользователей. 

облачная голосовая почта вклады в почтовый ящик пользователя сообщения голосовой почты Exchange вклады. облачная голосовая почта не поддерживает сторонние почтовые системы. Инструкции Exchange Online лицензирования см. в Exchange Online [описание службы](/office365/servicedescriptions/exchange-online-service-description/exchange-online-service-description#features-available-to-all-plans). Дополнительные сведения о ролях администратора см. в [сведениях о ролях администратора](/microsoft-365/admin/add-users/about-admin-roles).

## <a name="cloud-voicemail-provisioning"></a>облачная голосовая почта подготовка

Для Teams пользователей облачная голосовая почта автоматически настроена и подготовка. *Лицензия Microsoft Teams Телефон лицензия не требуется для облачная голосовая почта.*

Подготовка для Teams пользователей — это не то же самое, что для пользователей Skype для бизнеса Online. Для Skype для бизнеса Online облачная голосовая почта была автоматически настроена и подготовка, когда пользователям была назначена лицензия телефонная система и они Корпоративная голосовая связь включены системой ее подготовка.

Для Skype для бизнеса Server пользователей локальной облачная голосовая почта настроен и подготовка. Однако необходимо настроить среду Skype для бизнеса Server маршрутизов на облачная голосовая почта. Дополнительные сведения см. [в облачная голосовая почта службы для локального пользователя](/skypeforbusiness/hybrid/plan-cloud-voicemail.md). 

## <a name="cloud-voicemail-storage"></a>облачная голосовая почта хранилища

Сообщения голосовой почты, созданные облачная голосовая почта, доставляются в почтовый Exchange пользователя и хранятся в Exchange Online или в Exchange Server. Для голосовой почты не существует специального или дополнительного хранилища. Клиенты, которые имеют доступ к голосовой почте (например, Microsoft Outlook, Microsoft Teams или Skype для бизнеса), делают это с помощью Exchange почтового ящика и API. 

Сообщение голосовой почты содержит вложенный звуковой файл с голосовым сообщением и транскрибцией голосовой почты (если она включена). 

В Exchange почтового ящика пользователя хранится любое пользовательское записанное приветствие. Эти приветствия извлекаются облачная голосовая почта во время входящих зовов. 

## <a name="cloud-voicemail-processing"></a>облачная голосовая почта обработки 

Запись и транскрибация облачная голосовая почта начинается Microsoft 365 начале звонка, перенанося на облачная голосовая почта. Сообщение будет доставлено в почтовый ящик Exchange пользователя. 

Например, если звонок поступает недоступному пользователю прямой маршрутировки через контроллер границы сеанса (SBC) в Европе, запись и транскрибация голосовой почты будут проходить в Европе. Сообщение будет доставлено в почтовый ящик Exchange пользователя. Например, предположим, что Teams в Северной Америке звонит недоступному Teams в Европе. В этом случае звонок начинается в Северной Америке, обработка происходит в Северной Америке, а затем голосовая почта доставляется в почтовый ящик Exchange пользователя в Европе. 

Для доставки голосовой почты в почтовый Exchange почты используется протокол SMTP, как и для других сообщений электронной почты. 

## <a name="manage-cloud-voicemail-for-users"></a>Управление облачная голосовая почта пользователями 

Чтобы управлять облачная голосовая почта пользователями, воспользуйтесь модулем Teams PowerShell. 

Для управления облачная голосовая почта функций для групп пользователей используйте [новый csOnlineVoicemailPolicy](/powershell/module/skype/new-csonlinevoicemailpolicy). Вы можете настроить и назначить существующие или новые политики голосовой почты для таких функций, как правила ответа на звонки, транскрибирование голосовой почты, маскирование транскрибации, перевод разговоров и язык системных подсказок. Дополнительные сведения см. [в new-CsOnlineVoicemailPolicy](/powershell/module/skype/new-csonlinevoicemailpolicy).

Для управления облачная голосовая почта отдельными пользователями используйте для этого cmdlet [Set-CsOnlineVoicemailUserSettings](/powershell/module/skype/set-csonlinevoicemailusersettings). облачная голосовая почта параметры, которые можно применять к отдельным пользователям, включают правила ответа на звонки, язык подсказки, текст по умолчанию для речи и приветствия в отпуске. Дополнительные сведения см [. в записи Set-CsOnlineVoicemailUserSettings](/powershell/module/skype/set-csonlinevoicemailusersettings).
(Обратите внимание, что пользователи также могут настроить эти параметры в клиенте Teams, нажав **Параметры** ->  **CallsConfigure** ->  Voicemail.)

Вы также можете облачная голосовая почта параметр VoicemailEnabled для пользователя с помощью $false [Set-CsOnlineVoicemailUserSettings](/powershell/module/skype/set-csonlinevoicemailusersettings). Этот параметр гарантирует, облачная голосовая почта больше не сможет записывать голосовую почту для пользователя.



## <a name="control-routing-of-calls-to-cloud-voicemail"></a>Управление маршрутизами звонков на облачная голосовая почта 

По умолчанию для всех пользователей, облачная голосовая почта, можно разрешить маршрутику звонков облачная голосовая почта, а также разрешить им переадваровку звонков на облачная голосовая почта. 

С помощью Teams с параметром AllowVoicemail можно разрешить облачная голосовая почта маршрутизов на Set-CsTeamsCallingPolicy почты. Дополнительные сведения см [. в полеSet-CsTeamsCallingPolicy](/powershell/module/skype/set-csteamscallingpolicy.md). 

- Если для параметра AllowVoicemail задан параметр AlwaysDisabled, звонки никогда не перенанося в голосовую почту независимо от параметров переад сообщения или неотвеченных параметров для пользователя. Голосовая почта недоступна в качестве параметра переад/ или неотвеченного ответа в Teams.  

- Если для параметра AllowVoicemail задан параметр AlwaysEnabled, звонки всегда переадрены на голосовую почту в неотвеченном ответе после 30 секунд независимо от параметра неотвеченной переадстройки для пользователя.  

- Если для параметра AllowVoicemail задан параметр UserOverride, звонки переадрены на голосовую почту в зависимости от параметров переад сообщения и(или) неотвеченных параметров для пользователя. 



## <a name="set-up-cloud-voicemail-to-work-with-on-premises-users"></a>Настройка облачная голосовая почта для работы с пользователями локальной сети

С помощью облачная голосовая почта можно предоставлять функции голосовой почты для пользователей, у которых есть почтовые ящики на серверах Exchange, или для пользователей, использующих Skype для бизнеса Server. 

В этом разделе описано, как настроить облачная голосовая почта для Exchange Server почтовых ящиков. Сведения о том, как настроить облачная голосовая почта для Skype для бизнеса Server пользователей, см. в облачная голосовая почта для [локального пользователя](/skypeforbusiness/hybrid/plan-cloud-voicemail).

### <a name="set-up-cloud-voicemail-for-exchange-server-mailbox-users"></a>Настройка облачная голосовая почта для Exchange Server почтовых ящиков

Ниже данная информация о настройке облачная голосовая почта для работы с Teams пользователями, у которых есть почтовый ящик Exchange Server.

1. Сообщения голосовой почты доставляются в почтовый ящик Exchange пользователя через SMTP, маршрутный через Exchange Online Protection. Чтобы обеспечить успешную доставку этих сообщений, убедитесь, Exchange соединители правильно настроены между серверами Exchange и Exchange Online Protection. Дополнительные сведения см. в [этой](/exchange/mail-flow-best-practices/use-connectors-to-configure-mail-flow/use-connectors-to-configure-mail-flow) Flow.

2. Чтобы включить функции голосовой почты, такие как настройка приветствий и визуальной голосовой почты в Teams клиентах, необходимо настроить подключение между Microsoft 365 и почтовым Exchange Server почтой. Чтобы включить это подключение, необходимо настроить новый протокол проверки подлинности Exchange Oauth, описанный в окне Настройка проверки подлинности [OAuth](/exchange/configure-oauth-authentication-between-exchange-and-exchange-online-organizations-exchange-2013-help) между организациями Exchange и Exchange Online, или запустить мастер гибридной конфигурации Exchange в Exchange 2013 CU5 или более новой. Кроме того, необходимо настроить интеграцию и [Oauth](/skypeforbusiness/deploy/integrate-with-exchange-server/oauth-with-online-and-on-premises) между Teams и Exchange Server, описанными в Teams и Exchange Server.


## <a name="enable-protected-voicemail-in-your-organization"></a>Включить защищенную голосовую почту в организации

Когда кто-то покидает сообщение голосовой почты для пользователя в вашей организации, голосовая почта доставляется в почтовый ящик пользователя в виде вложения сообщения электронной почты. Используя правила потока почты для применения шифрования сообщений, вы можете запретить их пересылку другим получателям. Если включить защищенную голосовую почту, пользователи смогут прослушивать защищенные сообщения голосовой почты, позвонив в свой почтовый ящик голосовой почты или открыв их в Outlook, Outlook в Интернете или Outlook для Android или iOS. Защищенные сообщения голосовой почты нельзя открывать в Skype для бизнеса или Microsoft Teams.

Дополнительные сведения о шифровании сообщений см. в [теме Определение правил потока почты для шифрования сообщений электронной почты](/microsoft-365/compliance/define-mail-flow-rules-to-encrypt-email).
