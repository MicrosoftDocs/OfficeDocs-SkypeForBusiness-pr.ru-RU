# <a name="document-my-success-plan"></a>Документирование плана по достижению успеха

В этой статье приводятся общие сведения о требованиях для правильного документирования развертывания облачных голосов. Определяя и заполняя все точки принятия решений и дальнейшие действия при планировании развертывания облачных голосов, вы можете убедиться, что все заинтересованные лица и участники группы проекта будут выровнены по поставке успешных результатов. 

## <a name="execution-planning"></a>Планирование выполнения 

Определив способ реализации голосовой конференции или телефонной системы с помощью решения план звонков в вашей организации, вам необходимо планировать выполнение проекта реализации.

Если у вашей организации только один или два сайта, вам может потребоваться выполнить все описанные в этой статье сведения, но вы должны прочитать их, чтобы приступать к использованию.

<!--ENDOFSECTION-->

## <a name="deployment-model"></a>Модель развертывания 

Как и в случае с любой реализацией технологии, которая преобразует способы работы сотрудников в вашей организации, выбор верного пути для развертывания будет сильно влиять на успешность реализации облачной голосовой связи.

Ниже перечислены возможные модели развертывания.

-   **На сайт:** Эта модель подходит для случаев, когда Организация географически распределена, а в ветвях есть большое количество сотрудников. Однако эта модель развертывания может привести к нарушению связи в разных подразделениях, где сотрудники отдела распределены по нескольким расположениям.

-   **На каждом подразделении**: Эта модель обычно является более подходящим вариантом для компаний среднего размера и обеспечивает одинаковый интерфейс для задействованных отделов.

-   **Вся компания за один раз:** Эта модель обычно является наилучшим вариантом для малых компаний, где все сотрудники получают одинаковый опыт на одном из развертываний.

## <a name="raci-modeling"></a>Моделирование RACI

## <a name="quarterly-execution-plan"></a>Квартальный план выполнения

Чтобы выполнить облачное развертывание по облаку в управляемых фрагментах работы, мы рекомендуем создавать план квартального выполнения, основываясь на результатах основных результатов (OKRs), выбранной модели развертывания и возможности выполнения проекта в вашей организации.

Таким образом, вы можете отслеживать ход выполнения за квартал, изменяйте план при необходимости и пропускайте возможности облачной голосовой связи в зависимости от производительности организации, которую нужно выполнить.

Если в вашей организации только один или два сайта, вам может потребоваться квартальный план выполнения, так как ожидается, что развертывание будет выполнено в течение короткого промежутка времени.

## <a name="finalize-my-success-plan"></a>Завершение работы плана успеха

План успеха — это сводка документации, созданной на этапе предвидения.

План успеха предоставляет проектную группу (которая может включать FastTrack или партнера по развертыванию), достаточную информацию для реализации целей Организации, благодаря внедрению голосовой конференции или телефонной системы с помощью службы план звонков.

Как правило, план успеха содержит следующие основные разделы, многие из которых вы проработали на этапе предвидения.

-   Экономическое обоснование

-   Готовность к службе

-   Решения для служб

-   План выполнения

-   План внедрения

-   План работ

### <a name="business-case"></a>Экономическое обоснование

Варианты использования для бизнеса, список заинтересованных лиц, OKRs и KSIs, регистры риска и временные шкалы проекта обычно составляют массу информации, необходимой для деловой ситуации. Вы должны документировать эти данные в рамках плана успеха.

### <a name="service-readiness"></a>Готовность к службе

Ваша оценка среды предоставляет начальные сведения, необходимые для того, чтобы определить техническую готовность Организации к внедрению голосовой конференции и/или телефонной системы с помощью плана звонков.

В этой статье описана оценка готовности службы и планирование для областей, которые должны быть исправлены с помощью оценки среды.

### <a name="service-decisions"></a>Решения для служб

Документирование запланированной голосовой конференции или телефонной системы с помощью технического обслуживания плана звонков для вашей организации.

### <a name="execution-plan"></a>План выполнения

Документ, в котором запланировано выполнение проекта для реализации решения во всей Организации.

### <a name="adoption-plan"></a>План внедрения

После того как вы пройдете оценку готовности к внедрению, команде проекта потребуется получить полный набор планов связи, план обучения и планы действий по переходу на предварительный запуск, запуск и после запуска.

Определение ресурсов для поддержки операций по внедрению, таких как объявления, приветственные сообщения и учебные материалы, а также все настройки, которые необходимо выполнить в соответствии с требованиями вашей организации.

Скачайте шаблоны для действий по внедрению в [комплект поставки клиентов Microsoft Teams](https://www.microsoft.com/download/details.aspx?id=54244).

### <a name="operational-plan"></a>План работ

При сопоставлении рабочих ролей устанавливаются роли и обязанности, а также группы, назначенные каждой работающей роли, которые должны поддерживать реализацию голосовой конференции.

Вам нужно выполнить его и включить план работ в состав плана по достижению успеха, чтобы обеспечить готовность к работе с решением.
