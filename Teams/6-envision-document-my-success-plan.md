# <a name="document-my-success-plan"></a>Документирование плана по достижению успеха

В этой статье дается обзор требований к должным образом документирования развертывания облака голосовой связи. Определение и документирование все точки принятия решений и последующие действия при планировании развертывания облака голосовой связи, могут видеть все заинтересованные стороны и членов группы проекта выровненные на обеспечение успешные результаты. 

## <a name="execution-planning"></a>Выполнение планирования 

После определения, как реализуется аудиоконференции или телефонной системой с вызове плана решения в вашей организации, вам нужно запланировать выполнение реализации проекта.

Если в организации имеется только один или два сайтов, могут не нужно выполнять все сведения, приведенные в этой статье, но следует прочитать его в качестве руководства подхода.

## <a name="deployment-model"></a>Модель развертывания 

Как с любой реализации технология, которая преобразует способ работы в организации, Выбор оптимального способа предпринять развертывания будет значительно влияют на успешность внедрения облачных голосовой связи.

Потенциальные модели развертывания относятся следующие:

-   **Каждого сайта:** Эта модель подходит для случаев, когда географически распределенное вашей организации и ветви имеют значительное число сотрудников. Тем не менее эта модель развертывания потенциально может нарушить связь в отделах, в которых сотрудники отдела распространяются в нескольких подразделениях.

-   **Каждого подразделения**: в этой модели обычно является лучшим вариантом для организаций среднего размера и гарантирует, что отделы участвующие тот же интерфейс.

-   **За один раз для всей компании:** В этой модели обычно имеет лучший вариант для небольших организаций, где все сотрудники получить тот же интерфейс с первого дня развертывания.

## <a name="quarterly-execution-plan"></a>План квартальное выполнения

Для выполнения развертывания облака голосовой связи в управляемые фрагменты работы, рекомендуется создать квартальное план на основе результатов целевые ключевые (OKRs), выбранный модели развертывания и возможность выполнения проекта вашей организации.

Таким образом, можно отслеживать ход выполнения ежеквартально, измените план при необходимости и развернуть возможности голосовой связи облака зависимости от степени вашей организации для выполнения.

Если ваша организация имеет только одного или двух сайтов, не могут потребоваться квартальное план поскольку предполагается полностью развертываться в короткий промежуток времени.

## <a name="finalize-my-success-plan"></a>Заключительный этап план успеха

План успеха — это сводка документации, созданной на этапе Предвидения.

Планирование успеха дает группы проекта — которого может включать эту или партнера по развертыванию — достаточно сведений для экономии цели вашей организации с внедрением аудиоконференции или телефонной системы со службой вызов плана.

В общем случае успеха плана содержит следующие разделы, многие какие будут работал на этап Предвидения:

-   Экономическое обоснование

-   Готовность к службе

-   Службы решения

-   План выполнения

-   План внедрения

-   План работ

### <a name="business-case"></a>Экономическое обоснование

Варианты использования бизнеса, список заинтересованных лиц, OKRs и KSIs, регистрирует риска и временных шкал проекта обычно составляют большую часть сведения, необходимые для экономическое обоснование. Следует документировать их как часть плана успеха.

### <a name="service-readiness"></a>Готовность к службе

Окружающей среды оценки предоставляет начальные сведения, необходимые для определения технические готовность своей организации для реализации аудиоконференции и/или телефонной системы с помощью вызова планирование.

Здесь включена оценки готовности к службе и планирование на адрес области, требующие исправления, обнаруженные в результате окружающей среды оценки.

### <a name="service-decisions"></a>Службы решения

Документ, как запланированного аудиоконференции или телефонной системы с помощью вызова планирование технической реализации службы для вашей организации.

### <a name="execution-plan"></a>План выполнения

Документ, как запланированного выполнения проекта для реализации решения в вашей организации.

### <a name="adoption-plan"></a>План внедрения

После выполнения оценки готовности к внедрения группы проекта должно для обеспечиваемые полный набор планов взаимодействия, план обучения и планы по предварительного выпуска, запуска и после запуска внедрения действий.

Определите ресурсы для поддержки внедрения деятельности, такой как рекламные листовки, Добро пожаловать по электронной почте и обучающие материалы, а также все настройки, необходимые для удовлетворения требований организации.

Загрузите шаблоны для внедрения действий с [Microsoft группами Customer Success Kit](https://www.microsoft.com/download/details.aspx?id=54244).

### <a name="operational-plan"></a>План работ

Упражнения сопоставления ролей установит ролей и обязанностей или группам, каждый действующие роль, необходимого для поддержки реализации звук конференц-связи.

Вам нужно выполнить его и включить план работ в состав плана по достижению успеха, чтобы обеспечить готовность к работе с решением.
