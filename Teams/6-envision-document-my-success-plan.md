# <a name="document-my-success-plan"></a>Документирование плана по достижению успеха

В этой статье дается обзор требований для правильной документации облачного голосового развертывания. Определив и документируя все точки принятия решений и дальнейшие действия при планировании облачного голосового развертывания, вы можете обеспечить согласование всех заинтересованных лиц и участников группы проектов с конечными результатами. 

## <a name="execution-planning"></a>Планирование выполнения 

Определив способ реализации решения для аудиоконференции или телефонной системы с планом звонков в организации, необходимо спланировать реализацию проекта по внедрению.

Если в вашей организации есть только один или два сайта, возможно, вам не потребуется заполнять все сведения, предоставленные в этой статье, но вам следует ознакомиться с ней, чтобы помочь вам в этом.

<!--ENDOFSECTION-->

## <a name="deployment-model"></a>Модель развертывания 

Как и в случае с любой технологией, которая преобразует процесс работы людей в организации, выбор правильного способа внедрения развертывания значительно повлияет на успех облачной реализации голосовой связи.

Возможные модели развертывания включают следующие:

-   **Для каждого сайта:** Эта модель подходит для случаев географических рассредовлений организации, в которых филиалы имеют большое количество сотрудников. Однако эта модель развертывания может нарушить взаимодействие внутри отделов, сотрудники которых находятся в разных местах.

-   **Для каждого** подразделения: эта модель обычно лучше всего используется для компаний среднего размера и гарантирует, что у всех отделов одинаковый опыт работы.

-   **Вся компания одновременно:** Эта модель обычно лучше всего используется в небольших компаниях, где все сотрудники получают одинаковые возможности со дня первого развертывания.

## <a name="raci-modeling"></a>Моделирование RACI

## <a name="quarterly-execution-plan"></a>План квартального выполнения

Чтобы использовать облачное голосовое развертывание на управляемых фрагментах работы, рекомендуем создать план квартального выполнения на основе выбранных вами целей и модели развертывания, а также возможностей выполнения проекта в вашей организации.

Таким образом вы сможете квартально отслеживать ход выполнения, при необходимости пересматривать план и развертывать облачные голосовые возможности с учетом производительности организации.

Если в организации есть только один или два сайта, план квартального выполнения может не понадобиться, так как предполагается, что его полное развертывание будет развернуто за короткий период времени.

## <a name="finalize-my-success-plan"></a>Завершив свой план успешной работы

План успеха — это сводка документации, созданной на этапе "Создание проекта".

План успешного развертывания предоставляет проектной группе, которая может включать FastTrack или партнера по развертыванию, достаточно информации для реализации целей вашей организации по внедрению аудиоконференций или телефонной системы со службой плана звонков.

В целом план успеха содержит следующие основные разделы, над которыми вы работали на этапе "Создание проекта":

-   Экономическое обоснование

-   Готовность к службе

-   Решения об обслуживании

-   План выполнения

-   План внедрения

-   План работ

### <a name="business-case"></a>Экономическое обоснование

Дела использования бизнесом, список заинтересованных лиц, okRs и KSIs, регистры рисков и временные шкалы проектов обычно составляют основную часть информации, необходимой для бизнес-дела. Документируйте их в рамках плана по успеху.

### <a name="service-readiness"></a>Готовность к службе

Оценка, заданная по среде, содержит начальную информацию, необходимую для определения технической готовности организации к внедрению аудиоконференций и телефонной системы с планом звонков.

Здесь вы можете найти оценку готовности службы и план по устранению проблем, которые были выявлены при оценке среды.

### <a name="service-decisions"></a>Решения об обслуживании

Документируйте план реализации службы аудиоконференций или телефонной системы с технической реализацией плана звонков для вашей организации.

### <a name="execution-plan"></a>План выполнения

Запланировать выполнение проекта для реализации решения в организации.

### <a name="adoption-plan"></a>План внедрения

После оценки готовности к внедрению группа проекта должна получить полный набор планов коммуникаций, плана обучения и планов действий по предварительному запуску, запуску и внедрению после запуска.

Определите ресурсы для поддержки действий по внедрению, таких как листовки, приветствия электронной почты и учебные материалы, а также настройки, необходимые для обеспечения требований вашей организации.

Скачайте шаблоны для действий по внедрению из [набора microsoft Teams Customer Success Kit.](https://aka.ms/TeamsCustomerSuccess)

### <a name="operational-plan"></a>План работ

При сопоставлении рабочих ролей будут назначены роли и обязанности, а группам, которые назначены каждой из рабочих ролей, потребуется помощь в внедрении аудиоконференций.

Вам нужно выполнить его и включить план работ в состав плана по достижению успеха, чтобы обеспечить готовность к работе с решением.
