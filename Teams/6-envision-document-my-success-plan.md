# <a name="document-my-success-plan"></a>Документирование плана по достижению успеха

В этой статье дается обзор требований для правильной документации облачного голосового развертывания. Определив и документируя все точки принятия решений и дальнейшие действия при планировании облачного голосового развертывания, вы можете обеспечить согласование всех заинтересованных лиц и участников группы проектов с конечными результатами. 

## <a name="execution-planning"></a>Планирование выполнения 

Определив способ реализации аудиоконференции или телефонная система с помощью решения плана звонков в организации, необходимо спланировать выполнение проекта по внедрению.

Если в вашей организации есть только один или два сайта, возможно, вам не придется заполнять все сведения, предоставленные в этой статье, но вам следует ознакомиться с ней, чтобы помочь вам в этом.

<!--ENDOFSECTION-->

## <a name="deployment-model"></a>Модель развертывания 

Как и любое внедрение технологий, которое преобразует способ работы людей в вашей организации, выбор правильного способа развертывания значительно повлияет на успех облачной реализации голосовой связи.

Возможные модели развертывания:

-   **Для каждого сайта:** Эта модель подходит для случаев, когда организация расположена географически, а филиалы имеют большое количество сотрудников. Однако эта модель развертывания может нарушить взаимодействие между отделами, в которых сотрудники разных отделов находятся в разных местах.

-   **Отдел.** Как правило, эта модель лучше всего используется для компаний среднего размера и гарантирует, что у участвующих отделов одинаковый опыт работы.

-   **Вся компания одновременно:** Эта модель обычно лучше всего используется для небольших компаний, где все сотрудники получают одинаковый опыт с первого дня развертывания.

## <a name="quarterly-execution-plan"></a>План квартального выполнения

## <a name="finalize-my-success-plan"></a>Окончательный план успеха

План успеха — это сводка документации, созданной на этапе "Создание проектной стадии".

План успеха предоставляет группе проекта, в состав которой входит FastTrack или партнер по развертыванию, достаточно информации для реализации целей организации по внедрению аудиоконференций или телефонная система с помощью службы плана звонков.

Как правило, план успеха содержит следующие основные разделы, многие из которых вы должны были проработать на этапе "Создание проектной стадии":

-   Экономическое обоснование

-   Готовность к службе

-   Решения об обслуживании

-   План выполнения

-   План внедрения

-   План работ

### <a name="business-case"></a>Экономическое обоснование

Дела о бизнес-использовании, список заинтересованных лиц, okrs и KSIs, регистры рисков и временные шкалы проектов обычно составляют основную часть сведений, необходимых для бизнес-дела. Документируйте их в рамках плана успеха.

### <a name="service-readiness"></a>Готовность к службе

Оценка среды содержит первоначальные сведения, необходимые для определения технической готовности организации к внедрению аудиоконференций и /или телефонная система с планом звонков.

Здесь вы можете найти оценку готовности к обслуживанию и план по устранению областей, которые требуют исправлений, выявленных в рамках оценки среды.

### <a name="service-decisions"></a>Решения об обслуживании

Документируйте план аудиоконференций или телефонная система с помощью технической реализации плана звонков для вашей организации.

### <a name="execution-plan"></a>План выполнения

За документом о том, как вы запланировали выполнение проекта, чтобы реализовать решение во всей организации.

### <a name="adoption-plan"></a>План внедрения

После оценки готовности к внедрению проектной группе необходимо создать полный набор планов связи, плана обучения и планов для предварительного запуска, запуска и после запуска мероприятий по внедрению.

Определите ресурсы для поддержки действий по внедрению, таких как листовки, приветствия электронной почты и учебные материалы, а также настройки, необходимые для обеспечения требований организации.

Скачайте шаблоны для действий по внедрению из Microsoft Teams [клиентского успеха](https://aka.ms/TeamsCustomerSuccess).

### <a name="operational-plan"></a>План работ

При сопоставлении операционных ролей устанавливаются роли и обязанности, а группы, которые назначены каждой операционной роли, необходимо поддерживать внедрение аудиоконференций.

Вам нужно выполнить его и включить план работ в состав плана по достижению успеха, чтобы обеспечить готовность к работе с решением.
