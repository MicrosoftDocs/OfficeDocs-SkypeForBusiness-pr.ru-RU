## <a name="set-up-an-operator"></a>Настройка оператора

Вы можете настроить, изменить и удалить операторы в Центре администрирования Teams. В области навигации слева перейдите к **оператору голосовой >.**

Чтобы настроить оператор:

1. **Выберите оператор.** На **вкладке "Все операторы** " отфильтруйте доступные операторы по регионам или службам, чтобы найти нужный оператор для ваших голосовых потребностей. Затем выберите оператор, который вы хотите использовать. Для Operator Connect Conferencing убедитесь, что у оператора есть **конференц-связь**, указанная как доступный продукт.

2. **Выберите страны.** В **разделе "Параметры оператора**" выберите страны, которые вы хотите включить с помощью выбранного оператора.

3. **Предоставление контактных данных** Ваши контактные данные, включая полное имя и адрес электронной почты, будут предоставлены оператору. Эти сведения можно изменить позже. Кроме того, необходимо указать размер компании с возможностью указать номер телефона. Операторы используют эти сведения, чтобы связаться с вами с дополнительными сведениями о Operator Connect Conferencing.

4. Примите уведомление о передаче данных.

5. **Добавьте оператор.** Выберите **"Добавить как мой оператор" для** сохранения.

## <a name="acquire-numbers-for-your-audio-conferencing-bridge"></a>Получение номеров для моста аудиоконференций

Мост аудиоконференций вашей организации включает телефонные номера, доступные всем пользователям в организации для присоединения к собраниям Microsoft Teams с помощью номеров телефонов ТСОП. Номера телефонов, связанные с мостом аудиоконференций вашей организации, можно просмотреть в Центре Администратор Teams в разделе "Собрания **> мостах конференций"**.

Чтобы получить номера телефонов для моста аудиоконференций, выполните следующие действия.

1. **Подписка на аудиоконференции уровня "Стандартный" или Operator Connect Conferencing лицензии.** Пользователи, которым Operator Connect Conferencing номера для присоединения к собраниям, которые они упорядочивают, должны иметь лицензию на подписку на аудиоконференции уровня "Стандартный" или Operator Connect Conferencing лицензию на аудиоконференции. Дополнительные сведения см. в [разделе "Планирование Operator Connect Conferencing](operator-connect-conferencing-plan.md)".

2. **Получение чисел.** Перейдите на веб-сайт оператора, чтобы заказать и получить номера телефонов. Список веб-сайтов операторов см. в [каталоге Microsoft 365 Operator Connect](https://cloudpartners.transform.microsoft.com/practices/microsoft-365-for-operators/directory). Необходимо указать идентификатор клиента. Если вы не знаете свой идентификатор клиента, см. раздел ["Поиск идентификатора клиента Microsoft 365"](/onedrive/find-your-office-365-tenant-id). После того как оператор завершит заказ, он загрузит номера в клиент. Вы можете просмотреть номера и поставщика в Центре администрирования Teams, перейдя **> номера телефонов**.

3. **Назначьте номера мосту аудиоконференций.** Вы можете назначить номера мосту аудиоконференций из Центра администрирования Teams в разделе "Собрания > мосты **> добавить"**. Дополнительные сведения см [. в статье "Изменение номеров телефонов в мосте аудиоконференций"](change-the-phone-numbers-on-your-audio-conferencing-bridge.md).

>[!NOTE]
>Нельзя назначать Operator Connect Conferencing как номера моста по умолчанию. После назначения номера телефона мосту аудиоконференций этот номер будет указан на странице "Поиск локального номера",  включенной в приглашения на собрания пользователей в организации с лицензией на аудиоконференции или лицензией Operator Connect Conferencing.
>
>Сведения о маршрутизации исходящих вызовов через оператора см. в разделе "Отправка исходящих звонков из собраний [**Teams**](#sending-outbound-calls-from-microsoft-teams-meetings-through-an-operator) с помощью раздела оператора" далее в этой статье.

## <a name="change-the-default-phone-numbers-that-are-included-in-the-meeting-invites-of-users"></a>Изменение номеров телефонов по умолчанию, включенных в приглашения пользователей на собрания

 Этот шаг является необязательным.

Номера телефонов пользователя по умолчанию включаются в приглашения на собрания при планировании собрания. Вы можете обновить номера телефонов, включенные в приглашения пользователей на собрания в Центре Администратор Teams в разделе "Пользователи **> Управление пользователями"**. Чтобы обновить номера телефонов, включенные в приглашения пользователей на собрание, выберите пользователя и нажмите кнопку **"** Изменить" в разделе "Аудиоконференции **"** .

После применения изменений новые приглашения организаторов на собрания будут включать новые номера телефонов по умолчанию. Однако существующие приглашения на собрания, запланированные до изменения, будут сохранять исходные номера по умолчанию.

## <a name="sending-outbound-calls-from-microsoft-teams-meetings-through-an-operator"></a>Отправка исходящих звонков из собраний Microsoft Teams с помощью оператора

Если у вас есть подписка Microsoft Audio Conferencing Standard или лицензия Operator Connect Conferencing, вы можете настроить службу аудиоконференций для маршрутизации всех или нескольких исходящих звонков из собраний Teams через оператора, настроив политику маршрутизации аудиоконференций.

>[!NOTE]
>При Operator Connect Conferencing лицензии исходящие вызовы могут проходить только через ваш оператор. Если у вас Operator Connect Conferencing лицензии, необходимо настроить службу, как описано ниже, чтобы разрешить исходящие звонки из собраний Teams на телефонные номера.

Вы можете применить политики маршрутизации аудиоконференций на уровне пользователя, то есть оператор направляет только исходящие вызовы из собраний Teams, организованных пользователями с помощью соответствующей политики. Вы также можете применить эти политики на глобальном уровне, то есть оператор направляет исходящие звонки из собраний Teams, организованных всеми пользователями в вашей организации.

С помощью PowerShell создайте новую политику маршрутизации аудиоконференций в организации. Чтобы указать оператора в качестве основного маршрута для исходящих вызовов из собраний Teams, выполните следующие действия, используя указанные команды PowerShell:

1. [Шаг 1. Добавление новой строки в политику использования ТСОП в сети](#step-1-add-a-new-string-to-the-online-pstn-usage-policy)
2. [Шаг 2. Создание политики маршрутизации голосовой связи в Сети](#step-2-create-a-new-online-voice-route-policy)
3. [Шаг 3. Создание политики маршрутизации аудиоконференций в Интернете](#step-3-create-a-new-online-audio-conferencing-routing-policy)
4. [Шаг 4. Назначение новой политики пользователям](#step-4-assign-the-new-policy-to-users)

### <a name="step-1-add-a-new-string-to-the-online-pstn-usage-policy"></a>Шаг 1. Добавление новой строки в политику использования ТСОП в сети

Дополнительные сведения об использовании этого командлета см. в разделе [Set-CsOnlinePstnUsage](/powershell/module/skype/set-csonlinepstnusage) .

```powershell
Set-CsOnlinePstnUsage -Identity Global -Usage @{Add="International"}
```

### <a name="step-2-create-a-new-online-voice-route-policy"></a>Шаг 2. Создание политики маршрутизации голосовой связи в Сети

Дополнительные сведения об использовании этого командлета см. в разделе [Set-CsOnlineVoiceRoute](/powershell/module/skype/set-csonlinevoiceroute) .

```powershell
New-CsOnlineVoiceRoute -Identity "International" -NumberPattern "\d+" -OnlinePstnUsages "International" -BridgeSourcePhoneNumber <an Operator Connect Conferencing number assigned to your Audio Conferencing bridge>
```

### <a name="step-3-create-a-new-online-audio-conferencing-routing-policy"></a>Шаг 3. Создание политики маршрутизации аудиоконференций в Интернете

```powershell
New-CsOnlineAudioConferencingRoutingPolicy "International Policy" -OnlinePstnUsages "International"
```

### <a name="step-4-assign-the-new-policy-to-users"></a>Шаг 4. Назначение новой политики пользователям

```powershell
Grant-CsOnlineAudioConferencingRoutingPolicy -Identity <identity of the organizer of the meeting> -PolicyName "International Policy"
```

>[!NOTE]
>Чтобы задать глобальную политику маршрутизации аудиоконференций и применить ее ко всем пользователям в организации, ``-Global`` вместо параметра можно использовать этот ``-Identity`` параметр. Например, ``Grant-CsOnlineAudioConferencingRoutingPolicy -Global -PolicyName "International Policy"``.

При создании политики маршрутизации аудиоконференций и ее применении к пользователю оператор, предоставляющий номер телефона, указанный в параметре, ``BridgeSourcePhoneNumber`` направляет исходящие вызовы Teams на номера телефонов ТСОП. Кроме того, параметр ``BridgeSourcePhoneNumber`` указывает номер телефона, который будет использовать в качестве номера телефона для идентификации исходящих звонков по номерам ТСОП.

Шаблон, указанный в ``NumberPattern`` форме регулярного выражения, и указывает, какие вызовы маршрутизируются через оператор. Шаблон ``"\d+"`` в приведенном выше примере соответствует всем исходящим звонкам из собраний Teams. Можно также задать параметр NumberPattern  ``"^\+1(425|206)(\d{7})$"``, соответствующий наборам номеров в следующих форматах: +1 425 XXX XX XX или +1 206 XXX XX XX XX, ``"^\+1(\d{10})$"``или , который соответствует наборам номеров в следующем формате: +1 425 XXX XX XX XX.

После назначения пользователю политики маршрутизации аудиоконференций все звонки из собраний на номер телефона, соответствующий регулярному выражению, указанному в политике маршрутизации аудиоконференций, направляются через оператора, включая звонки на звонки и звонки, инициированные с помощью  параметра "Пригласить кого-то" или на **номер собрания.**

## <a name="manage-your-operators"></a>Управление операторами

На **вкладке "Мои операторы** " можно просмотреть операторы, их состояние и внести следующие изменения в выбранные параметры:

- Управление службами операторов по странам
- Приостановка оператора
- Удаление оператора

>[!NOTE]
>Перед удалением оператора из организации или страны необходимо удалить все номера телефонов, назначенные пользователям и мосту аудиоконференций, а затем связаться с оператором, чтобы освободить эти номера.
