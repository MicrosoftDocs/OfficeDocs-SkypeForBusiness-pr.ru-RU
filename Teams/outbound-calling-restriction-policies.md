# <a name="outbound-calling-restriction-policies-for-audio-conferencing-and-user-pstn-calls"></a>Политики ограничений для исходящих звонков аудиоконференций и пользовательских звонков по ТСОП

Администраторы могут использовать элементы управления исходящие звонки для ограничения типа аудиоконференций и звонков по телефонной сети общего пользования (СТСК), которые могут делать пользователи в вашей организации.

Элементы управления исходящие звонки можно применять для каждого пользователя или клиента, а также предоставлять следующие два средства управления для независимого ограничения каждого типа исходящие вызовы. По умолчанию оба элемента управления разрешают международные и внутренние исходящие вызовы.

|Управления|Описание|Параметры элемента управления|
|:-----|:-----|:-----|
|Вызовы аудиоконференции по ТСОП|Ограничивает тип исходящих </br>вызовов, которые разрешено выполнять </br>во время собраний, организованных пользователем.|Назначение (по умолчанию)</br>В той же стране или регионе, где был организатор </br> [Только страны или регионы зоны A](audio-conferencing-zones.md) </br>Не разрешайте|
|Звонки через ПСС для конечных пользователей|Ограничивает тип вызовов, </br>доступных пользователю.|Международные и внутренние (по умолчанию)</br>Внутренний</br>Нет|

Чтобы узнать, какие страны и регионы считаются зонами A, см. зоны страны и региона для [аудиоконференции.](audio-conferencing-zones.md)

   > [!NOTE]
   > Звонок считается внутренним, если набраный номер находится в той же стране, где настроена Microsoft 365 или Office 365 для организатора собрания (в случае аудиоконференции) или конечного пользователя (в случае звонков по ННП для конечного пользователя).

## <a name="restrict-audio-conferencing-outbound-calls"></a>Ограничение исходящих вызовов аудиоконференций

![логотип Microsoft Teams.](media/teams-logo-30x30.png) **С помощью Центра администрирования Microsoft Teams**

1. В левой области навигации выберите **Пользователи**, а затем выберите отображаемого имени пользователя в списке доступных пользователей.

3. Рядом с **аудиоконференцией** выберите **Изменить**.

4. В **диалоговом окте**"Из собрания" выберите нужный вариант ограничения на набор номеров.

5. Нажмите **Сохранить**.

![Значок с логотипом Skype для бизнеса.](media/sfb-logo-30x30.png) **Использование центра Skype для бизнеса администрирования**

1. В **Skype для бизнеса** администрирования на левой навигации перейдите в меню Пользователи аудиоконференции , а затем выберите пользователя в списке  >  доступных пользователей.

2. В области действий выберите **Изменить**.

3.  В разделе **Ограничение телефонных вызовов во время собраний с этим пользователем** выберите нужный параметр ограничения телефонных вызовов.

      ![Ограничения для параметров телефонного набора.](media/restrictions-to-dial-outs.png)

4. Нажмите **Сохранить**.

**Использование PowerShell**

Ограничения исходящие вызовы контролируются одной политикой Под названием OnlineDialOutPolicy, которая имеет атрибут ограничения для каждого из них. Эту политику нельзя изменить в соответствии с потребностями пользователя, но можно использовать предварительно заданные экземпляры политики для каждого сочетания параметров.

С помощью командлета Get-CSOnlineDialOutPolicy вы можете просмотреть политики исходящие вызовы и использовать для настройки следующую команду:

**Задать политику на уровне "на пользователя", выдав** ее, можно с помощью следующего cmdlet ( На уровне пользователя). (В нем нет слова "Online", как в get.)

```
Grant-CsDialoutPolicy -Identity <username> -PolicyName <policy name>    
```

**Задать политику на уровне клиента с помощью следующего cmdlet**:

```
Grant-CsDialoutPolicy  -Tenant <guid> -PolicyName <policy name>  -Global 
```

Эта политика будет назначена всем пользователям клиента, у которых нет назначенной политики. Другие пользователи остаются с текущей политикой.
