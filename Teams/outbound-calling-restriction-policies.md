# <a name="outbound-calling-restriction-policies-for-audio-conferencing-and-user-pstn-calls"></a>Политики ограничений для исходящих звонков аудиоконференций и пользовательских звонков по ТСОП

Администратор может использовать элементы управления исходящими звонками, чтобы ограничить тип аудиоконференций и звонков по ТСОП, которые могут выполнять пользователи в вашей организации.

Элементы управления исходящими звонками могут применяться для каждого пользователя или клиента и предоставлять следующие два элемента управления для независимого ограничения каждого типа исходящих вызовов. По умолчанию оба элемента управления разрешают международные и внутренние исходящие вызовы.

|Управления|Описание|Параметры элемента управления|
|:-----|:-----|:-----|
|Вызовы аудиоконференции по ТСОП|Ограничивает тип исходящих </br>вызовов, которые разрешено выполнять </br>во время собраний, организованных пользователем.|Любое назначение (по умолчанию)</br>В той же стране или регионе, что и организатор </br> [Только страны или регионы зоны A](audio-conferencing-zones.md) </br>Не разрешать|
|Вызовы ТСОП конечных пользователей|Ограничивает тип вызовов, </br>доступных пользователю.|Международные и внутренние (по умолчанию)</br>Внутренний</br>Нет|

Сведения о том, какие страны и регионы считаются зонами A[](audio-conferencing-zones.md), см. в разделе "Страна и регионы" для аудиоконференций.

   > [!NOTE]
   > Звонок считается внутренним, если номер на телефоне находится в той же стране, где microsoft 365 или Office 365 настроен для организатора собрания (в случае аудиоконференций) или конечного пользователя (в случае вызовов ТСОП конечным пользователем).

## <a name="restrict-audio-conferencing-outbound-calls"></a>Ограничение исходящих вызовов аудиоконференций

### <a name="using-the-microsoft-teams-admin-center"></a>С помощью Центра администрирования Microsoft Teams

1. В области навигации слева выберите **"** Пользователи", а затем в списке доступных пользователей выберите отображаемое имя пользователя.

2. Затем перейдите к **аудиоконференции** и нажмите кнопку **"Изменить"**.

3. В **разделе "Исходящий доступ из собраний**" выберите нужный параметр ограничения для исходящего подключения.

4. Нажмите **Сохранить**.

### <a name="using-powershell"></a>С помощью PowerShell

Ограничения исходящих вызовов управляются одной политикой с именем OnlineDialOutPolicy, которая имеет атрибут ограничения для каждого из них. Эту политику нельзя изменить в соответствии с потребностями пользователя, но можно использовать предварительно заданные экземпляры политики для каждого сочетания параметров.

С помощью Get-CSOnlineDialOutPolicy командлета можно просмотреть политики исходящих вызовов и использовать следующую команду для установки.

**Задайте политику на уровне пользователя с помощью следующего командлета**. (Командлет Grant не содержит слово "Online", как это делает командлет Get.)

```powershell
Grant-CsDialoutPolicy -Identity <username> -PolicyName <policy name>    
```

**Задайте политику на уровне клиента с помощью следующего командлета**.

```powershell
Grant-CsDialoutPolicy -PolicyName <policy name>  -Global 
```

Эта политика будет использоваться для всех пользователей клиента, которым не назначена политика диалога. Другие пользователи остаются с текущей политикой.

**Проверьте текущую политику на уровне клиента с помощью следующего командлета**.

```powershell
Get-CSOnlineDialOutPolicy -Identity Global
```
