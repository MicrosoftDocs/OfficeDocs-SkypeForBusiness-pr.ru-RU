<a name="microsoft-365-groups-and-microsoft-teams"></a>Группы Microsoft 365 и Microsoft Teams
=====================================

> [!Tip]
> В следующем сеансе показано, как взаимодействуют Teams с Azure Active Directory (Azure AD), группами Microsoft 365, Exchange, SharePoint и OneDrive для бизнеса: [основы Microsoft Teams](https://aka.ms/teams-foundations)

Microsoft 365 Groups — это служба членства в разных приложениях в Office 365. На базовом уровне группа Microsoft 365 — это объект в Azure Active Directory со списком пользователей и слабой присоединением к связанным рабочим нагрузкам, включая сайт группы SharePoint, группу Yammer, общие ресурсы почтового ящика Exchange, планировщик, Power BI и OneNote. Вы можете добавлять и удалять пользователей в группе так же, как и любые другие объекты безопасности на основе группы в Active Directory.

Администратор Office 365 может определять группу Microsoft 365, добавлять участников и использовать преимущества таких функций, как общий почтовый ящик Exchange, библиотеку документов SharePoint, группу Yammer и т. д. Дополнительные сведения о группах Microsoft 365 можно найти в [статье сведения о группах microsoft 365](https://support.office.com/article/Learn-about-Office-365-groups-b565caa1-5c40-40ef-9915-60fdb2d97fa2).

Не пропустите группы плакатов [в Microsoft 365 для ИТ архитекторов](teams-architecture-solutions-posters.md#groups-in-microsoft-365).

<a name="how-microsoft-365-groups-work"></a>Принципы работы групп Microsoft 365
--------------------------

Когда вы создаете команду на сервере, вы создаете группу Microsoft 365 и связанную библиотеку документов SharePoint и записную книжку OneNote вместе с связями с другими облачными приложениями Office 365. Если пользователь, создавший группу, является владельцем существующей или общедоступной группы Office 365, они могут добавлять в нее возможности Teams, если у нее менее 5000 человек и они еще не были добавлены в Teams. При этом создается **Общий** канал по умолчанию, в котором хранятся сообщения чата, документы, OneNote и другие объекты. При просмотре библиотеки документов для канала откроется **Общая** папка, представляющая канал в команде. Что особенно важно, когда вы создаете собственную структуру папок в библиотеке документов, **она не распространяется** в Teams в виде канала. Сейчас она просто передается из Teams в SharePoint.

> [!NOTE]
> С учетом отзывов пользователей новые группы Microsoft 365, созданные в результате создания группы в клиенте Microsoft Teams, больше не будут отображаться в Outlook по умолчанию. Чтобы включить или отключить отображение групп в Outlook, используйте командлет [Set-UnifiedGroup](https://docs.microsoft.com/powershell/module/exchange/users-and-groups/set-unifiedgroup) с параметром **HiddenFromExchangeClientsEnabled** . Группы, созданные в Outlook, а затем включенные для Teams, будут по прежнему отображаться в Outlook и в Teams. 

> [!NOTE]
> Удаление группы Microsoft 365 приведет к удалению псевдонима почтового ящика для постоянных разговоров Outlook/OWA и приглашений на собрание Teams и пометке сайта SharePoint для удаления. Удаление команды и ее воздействия на Outlook занимает около 20 минут. Удаление команды из клиента Teams немедленно удалит ее из представления всем пользователям, которые являются участниками команды. Если удалить пользователей из группы Microsoft 365, на которых включены функции Teams, может быть задержка около двух часов перед тем, как команда будет удалена из представления в клиенте Teams для уязвимых пользователей, которые были удалены.
>
>Ознакомьтесь [со сведениями](https://support.office.com/article/Restore-a-deleted-Office-365-Group-b7c66b59-657a-4e1a-8aa0-8163b1f4eb54) о восстановлении удаленной группы Microsoft 365.

<a name="group-membership"></a>Членство в группах
----------------

Группировка функций и возможностей для пользователей зависит от того, откуда вы разработаны в группах. Например, если вы удалите участника команды, он также будет удален из группы Microsoft 365. Удаление из группы немедленно удаляет группу и каналы из клиента Teams. Если вы удалили человека из группы с помощью центра администрирования Microsoft 365, у него больше не будет доступа к другим аспектам совместной работы, таким как библиотека документов SharePoint Online, Группа Yammer или общая программа OneNote. Однако они по-прежнему смогут получать доступ к функциям чата для участников группы в течение примерно двух часов.

В качестве рекомендации по управлению участниками группы добавьте и удалите участников из клиента Teams, чтобы убедиться в том, что применен правильный каскадный элемент управления доступом к другим зависимым облачным приложениям. Кроме того, это позволит избежать путаницы, когда пользователи думают, что могут и дальше использовать привычные им ресурсы (пока доступ к конкретному компоненту не будет запрещен и разрешен при следующем цикле синхронизации). Если вы добавляете или удаляйте участников группы за пределами клиента Teams (с помощью центра администрирования Microsoft 365, Azure AD или Exchange Online PowerShell), это может занять до 24 часов (в некоторых случаях), чтобы изменения были отражены в Teams.

<a name="ability-to-add-group-as-attendee-while-scheduling-meetings"></a>Возможность добавления групп в качестве участников при планировании собраний
----------------------------------------------------------

Начиная с мая 2020, вы можете пригласить группу к запланированному собранию, выполнив следующие предупреждения:
1. Все существующие группы Microsoft 365 и группы, созданные из существующих групп 365, будут доступны для поиска и могут быть добавлены к собранию. Тем не менее участники собрания будут получать приглашение на собрание по своей подписке на группу.
2. Команды, созданные с нуля до мая 2018, будут также доступна для поиска, но участники собрания не получат приглашение на собрание из-за подписки на групповую подписку по умолчанию, которая является "единственными ответами". Это можно сделать с помощью Outlook, изменив параметры группы
3. Команды, созданные с нуля после того, как могут 2018 не будут доступны для поиска и скрыты с помощью свойства "HiddenFromAddressListsEnabled". Это параметр, контролируемый администратором, который может изменить администратор.
