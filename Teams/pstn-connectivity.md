# <a name="pstn-connectivity-options"></a>Параметры подключения через ДНР

Корпорация Майкрософт предоставляет полный доступ к Exchange УАПС для вашей организации с помощью телефонная система. Однако для того чтобы пользователи могли звонить за пределы организации, необходимо подключить телефонная система к телефонной сети общего звонков (STN).

В этой статье данная статья посвящена вариантам подключения через ДНР. Дополнительные сведения о голосовых решениях Майкрософт, в том числе о телефонная система функций, см. в Teams [голосового решения](cloud-voice-landing-page.md).

Чтобы подключить телефонная система к STN, можно выбрать один из следующих вариантов:

- [**План звонков**](#phone-system-with-calling-plan). Облачное решение, в качестве оператора связи по STN — Майкрософт.

- [**Оператор Подключение**](#phone-system-with-operator-connect). Операторы Подключение, если ваш оператор принимает участие в программе microsoft Подключение, они могут управлять звонками по ЗВОНКОВ по ЗВОНКОВ через ДНР и пограничными контроллерами сеансов. 

- [**Прямая маршрутная маршрутия**](#phone-system-with-direct-routing), с помощью которой вы можете использовать собственный оператор STN, подключив контроллер границы сеанса (SBC) к телефонная система.


Вы также можете выбрать сочетание параметров, которое позволяет создать решение для сложной среды или управлять многошаговой миграцией.

Эти параметры влияют на телефонная система функций. Дополнительные сведения см. в [статье Сведения о конфигурации далее](#configuration-considerations) в этой статье.


## <a name="phone-system-with-calling-plan"></a>телефонная система с планом звонков 

телефонная система с планом звонков — это облачное решение Майкрософт для голосовых Teams пользователей. Это решение является простейшим вариантом, который подключает телефонная система к STN. При этом корпорация Майкрософт выступает в качестве оператора связи по ННР, как показано на схеме ниже.

![На схеме 1 показаны телефонная система с планом звонков.](media/voice-solutions-simple.png)

Если вы отвечаете да, то телефонная система с помощью плана звонков — это правильное решение.

- План звонков доступен в вашем регионе.
- Не нужно хранить текущего оператора связи через ДНР.
- Вы хотите использовать доступ к STN, управляемый Майкрософт.

С помощью этого параметра: 

- Вы получаете телефонная система с добавленными планами внутренних или международных звонков, которые позволяют звонить на телефоны по всему миру (в зависимости от уровня обслуживания).

- Развертывание или обслуживание локального&mdash; развертывания с использованием плана звонков выполняется не Microsoft 365.

- Примечание. Вы можете подключить поддерживаемый пограничный контроллер сеанса (SBC) с помощью прямой маршрутировки для обеспечения связи со сторонними УАКС, аналоговыми устройствами и другим телефонным оборудованием, поддерживаемым SBC.

Для этого параметра требуется непрерывное подключение к Microsoft 365.

## <a name="phone-system-with-operator-connect"></a>телефонная система с оператором Подключение

Операторы Подключение, если ваш оператор участвует в программе Подключение Майкрософт, они могут управлять службой для звонков по ЗВОНКОВ по ПСОП Teams. Ваш оператор связи управляет службами звонков по ЗВОНКОВ по ЖКХ и пограничными контроллерами сеансов( SBCs), что позволяет экономить на приобретении оборудования и управлении им.

Оператор Подключение может быть правильным решением для вашей организации, если:

- План звонков Майкрософт не доступен в вашем географическом расположении.
- Вашим предпочтительным оператором является участник программы microsoft Operator Подключение.
- Вы хотите найти нового оператора связи для звонков в Teams.

Сведения о преимуществах и требованиях оператора Подключение, а также список операторов, участвующих в этой программе, см. в [Подключение.](operator-connect-plan.md) Сведения о настройке оператора Подключение см. в [Подключение](operator-connect-configure.md).


## <a name="phone-system-with-direct-routing"></a>телефонная система с прямой маршрутией

Этот параметр подключается телефонная система к телефонной сети с помощью прямой маршрутии, как показано на следующей схеме: 

![На схеме 5 телефонная система с прямой маршрутией.](media/voice-solution-with-direct-routing.png)

Если вы отвечаете да на следующие вопросы, то телефонная система с прямой маршрутией— это правильное решение.

- Вы хотите использовать Teams с телефонная система.
- Необходимо сохранить текущий оператор ПСПС.
- Вы хотите комбировать маршрутику с некоторыми звонками через план звонков, некоторые из них через вашего оператора связи.
- Необходимо работать со сторонними УАКС и оборудованием, например накладными страницами, аналоговыми устройствами и так далее.

С помощью этого параметра:

- Вы подключали собственный поддерживаемый контроллер границы сеанса (SBC) к телефонная система без необходимости в дополнительном локальном программном обеспечении.

- Вы можете использовать практически любой оператор телефонии с телефонная система.

- Вы можете настроить этот параметр и управлять им, а также настроить его оператором или партнером (спросите, предоставляет ли этот параметр ваш оператор или партнер).

- Вы можете настроить взаимодействия между&mdash;&mdash; оборудованием телефонии как стороннее УАПС и аналоговые устройства и использовать телефонная система.

Для этого параметра требуется следующее:

- Непрерывное подключение к Microsoft 365.

- Развертывание и обслуживание поддерживаемых SBC.

- Контракт со сторонним оператором.
  (Если не развернуто как подключение к сторонним УАП, аналоговым устройствам или другому телефонном оборудованию, пользователям, которые находятся в телефонная система с планом звонков.)

### <a name="phone-number-management"></a>Телефон управления номерами

У корпорации Майкрософт есть два типа доступных номеров телефонов: номера абонентов (пользователей), которые могут быть назначены пользователям в вашей организации, и номера служб, доступные в качестве платных и бесплатных номеров служб. Номера служб имеют более высокую пропускную способность при одновременном звонке, чем номера абонентов, и могут быть назначены таким службам, как аудиоконференция, автоотединение или очереди  вызовов.

Необходимо решить:

- В каких расположениях нужны новые телефонные номера от Майкрософт?
- Какой тип номера телефона (абонент или служба) мне нужен?
- Как перенос существующих номеров телефонов в Teams?

Способ получения номеров телефонов и управление ими зависит от варианта подключения к ДНР.

- Сведения об управлении номерами телефонов для плана звонков см. в управлении [номерами телефонов для организации](manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization.md).

- Сведения об управлении номерами телефонов с помощью оператора Подключение см. в этой [Подключение.](operator-connect-configure.md#set-up-phone-numbers)

- Сведения об управлении номерами телефонов для прямой маршрутки см. в этой ссылке[.](direct-routing-enable-users.md#configure-the-phone-number-and-enable-enterprise-voice)

### <a name="emergency-calling"></a>Экстренные вызовы

Способ настройки экстренных вызовов зависит от варианта подключения к ННП.

- Для плана звонков каждый пользователь автоматически получает экстренные вызовы. У этого пользователя должен быть зарегистрирован адрес для экстренного устранения чрезвычайной ситуации, связанный с номером назначенного им телефона. Поддерживаются динамические экстренные вызовы (в зависимости Teams клиента).  

- Для операторов Подключение для каждого пользователя автоматически включена служба экстренных вызовов. У пользователя должен быть зарегистрирован адрес для экстренного устранения чрезвычайной ситуации, связанный с назначенной телефонией,r, но адрес может установить только партнер оператора связи. Поддерживаются динамические экстренные вызовы (в зависимости Teams клиента).

- Для прямой маршрутизации необходимо определить политики экстренных вызовов для пользователей с помощью Teams маршрутизации экстренных вызовов (TeamsEmergencyCallRoutingPolicy). Политика определяет номера экстренных служб и связанные с ними маршруты. Зарегистрированные местоположения для экстренного ситуация не поддерживаются пользователями прямой маршрутации. Для динамических экстренных вызовов требуется дополнительная настройка для маршрутации экстренных вызовов и, возможно, для подключения партнеров.

Дополнительные сведения о понятиях и терминологии экстренных вызовов, а также о настройке экстренных и динамических вызовов экстренных служб см. в следующих статьях:
