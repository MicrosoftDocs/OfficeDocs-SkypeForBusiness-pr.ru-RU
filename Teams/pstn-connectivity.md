# <a name="pstn-connectivity-options"></a>Варианты подключения ТСОП

Корпорация Майкрософт предоставляет полный Exchange частных филиалов (PBX) для вашей организации через телефонная система. Однако для того, чтобы пользователи могли звонить за пределами организации, необходимо подключить телефонная система к открытой телефонной сети (PSTN).

В этой статье основное внимание уделяется вариантам подключения к PSTN. Дополнительные сведения о голосовых решениях Майкрософт, вклюая сведения о телефонная система, см. в Teams [голосового решения.](cloud-voice-landing-page.md)

Чтобы подключить телефонная система к PSTN, вы можете выбрать из следующих параметров:

- [**План вызова**](#phone-system-with-calling-plan). Облачное решение с Microsoft в качестве носителя PSTN.

- [**Оператор Подключение,**](#phone-system-with-operator-connect)который в настоящее время доступен только в **общедоступных предварительных просмотрах.**  С помощью Подключение операторов, если существующий оператор является участником программы microsoft Operator Подключение, они могут управлять вызовами ПСОП и контроллерами пограничной связи сеансов (SBCs). 

- [**Прямая маршрутная маршрутная**](#phone-system-with-direct-routing)маршрутия, которая позволяет использовать собственный переносчик PSTN, подключив контроллер границы сеанса (SBC) к телефонная система.


Вы также можете выбрать сочетание параметров, которое позволяет разработать решение для сложной среды или управлять многошаговой миграцией.

Следует помнить, что параметр или параметры, которые вы выбираете, влияют на настройку некоторых телефонная система функций. Дополнительные сведения см. в [статье Configuration considerations](#configuration-considerations) later in this article.


## <a name="phone-system-with-calling-plan"></a>Телефонная система с тарифным планом 

телефонная система с помощью плана вызовов — это голосовое решение Microsoft для всех пользователей Teams облачным решением. Это самый простой вариант, который соединяет телефонная система с PSTN. С помощью этого параметра Корпорация Майкрософт выступает в качестве носителя PSTN, как показано на следующей схеме:

![На диаграмме 1 телефонная система с планом вызова](media/voice-solutions-simple.png)

Если вы отвечаете "да" на следующее, телефонная система с помощью плана вызовов является правильным решением для вас:

- В вашем регионе доступен план вызова.
- Вам не нужно сохранять текущий переносчик PSTN.
- Вы хотите использовать управляемый Microsoft доступ к PSTN.

Используя этот вариант, вы получаете указанные ниже возможности. 

- Вы получаете Телефонную систему Майкрософт с добавленными Планами звонков для совершения вызовов внутри страны или международных вызовов, что позволяет вам звонить на телефоны по всему миру (в зависимости от лицензированного уровня обслуживания).

- Развертывание или обслуживание локального развертывания не требуются, так как план вызова работает &mdash; Microsoft 365.

- Примечание. При необходимости можно подключить поддерживаемый пограничный контроллер сеанса (SBC) с помощью прямой маршрутировки для обеспечения связи с сторонними PBX, аналоговыми устройствами и другим сторонним оборудованием телефонии, поддерживаемым SBC.

Чтобы использовать этот вариант, требуется непрерывное подключение к Microsoft 365.

## <a name="phone-system-with-operator-connect"></a>телефонная система с оператором Подключение

С помощью Подключение оператора, который в настоящее время находится в режиме предварительного просмотра, если существующий оператор является участником программы Microsoft Operator Подключение, они могут управлять службой для привлечения вызовов PSTN в Teams. Ваш оператор управляет службами вызова PSTN и диспетчерами пограничной связи сеансов (SBCs), что позволяет экономить на покупке оборудования и управлении.

Оператор Подключение может быть правильным решением для вашей организации, если:

- Microsoft Calling Plan недоступна в вашем географическом расположении.
- Предпочтительный оператор — участник программы microsoft Operator Подключение.
- Вы хотите найти нового оператора, чтобы включить вызов в Teams.

Сведения о преимуществах и требованиях оператора Подключение и списке перевозчиков, участвующих в этой программе, см. в Подключение [.](operator-connect-plan.md) Сведения о настройке оператора Подключение см. в [Подключение.](operator-connect-configure.md)


## <a name="phone-system-with-direct-routing"></a>телефонная система с прямой маршрутией

Этот параметр подключает телефонная система к сети телефонии с помощью прямого маршрутинга, как показано на следующей схеме: 

![Схема 5 телефонная система с прямым маршрутигом](media/voice-solution-with-direct-routing.png)

Если вы ответите "да" на указанные ниже вопросы, то, возможно, телефонная система с прямой маршрутизацией — правильный выбор для вас.

- Вы хотите использовать Teams с Телефонной системой.
- Вам необходимо продолжать использовать вашего текущего оператора ТСОП.
- Вы хотите использовать смешанную маршрутизацию, при которой одна часть звонков будет совершаться через тарифный план, а вторая — через вашего оператора.
- Вам необходимо скооперироваться с сторонними PBX и/или оборудованием, такими как накладные пейджеры, аналоговые устройства и так далее.

Используя этот вариант, вы получаете указанные ниже возможности.

- Вы подключите собственный поддерживаемый пограничный контроллер сеанса (SBC) к телефонная система без необходимости дополнительного локального программного обеспечения.

- Вы можете использовать практически любой оператор телефонии с телефонная система.

- Вы можете настроить этот вариант и управлять им. Кроме того, настроить его и управлять им могут оператор или партнер (спросите своего оператора или партнера, поддерживают ли они такой вариант).

- Можно настроить возможность взаимодействия между телефонным оборудованием, например сторонним PBX, аналоговыми устройствами &mdash; &mdash; и телефонная система.

Для этого варианта требуются указанные ниже элементы.

- Непрерывное подключение к Microsoft 365.

- Развертывание и обслуживание поддерживаемого пограничного контроллера сеанса.

- Контракт со сторонним оператором. (За исключением случаев развертывания для подключения к сторонним УАТС, аналоговым устройствам или прочему телефонному оборудованию для пользователей, применяющих телефонную систему с тарифным планом.)

### <a name="phone-number-management"></a>Телефон управления номерами

Корпорация Майкрософт имеет два типа доступных телефонных номеров: абонентские (пользовательские) номера, которые могут быть назначены пользователям в вашей организации, и номера служб, доступные в качестве платных и бесплатных номеров служб. Номера служб имеют более высокую пропускную способность по сравнению с номерами абонентов и могут быть назначены службам, таким как аудиоконференция, автовласти или очереди на вызовы.

Необходимо решить:

- В каких расположениях пользователей требуются новые номера телефонов от Microsoft?
- Какой тип номера телефона (абонента или службы) мне нужен?
- Как портить существующие номера телефонов на Teams?

Способ приобретения и управления номерами телефонов зависит от параметра подключения к PSTN.

- Сведения об управлении номерами телефонов для плана звонков см. в этой [информации.](manage-phone-numbers-for-your-organization/manage-phone-numbers-for-your-organization.md)

- Сведения об управлении номерами телефонов с оператором Подключение см. в ссылке Настройка номеров телефонов в [операторе Подключение.](operator-connect-configure.md#set-up-phone-numbers)

- Сведения об управлении номерами телефонов для прямой маршрутной маршрутики см. в ссылке Настройка номера телефона и включить голосовую [и голосовую почту предприятия.](direct-routing-enable-users.md#configure-the-phone-number-and-enable-enterprise-voice-and-voicemail-online)

Вы настраивает планы набора пользователей для перевода номеров для авторизации вызовов и маршрутизации вызовов. Дополнительные сведения см. [в дополнительных сведениях о планах набора?](what-are-dial-plans.md).

### <a name="emergency-calling"></a>Экстренные вызовы

Настройка вызова экстренных служб отличается в зависимости от параметра подключения к PSTN.

- Для плана вызова каждый пользователь автоматически включен для экстренного вызова и должен иметь зарегистрированный аварийный адрес, связанный с назначенным номером телефона. Поддерживаются динамические вызовы экстренных служб (в зависимости Teams клиента).  

- Для оператора Подключение, каждый пользователь автоматически включен для экстренного вызова и должен иметь зарегистрированный аварийный адрес, связанный с назначенным им номером телефона, но может быть установлен только партнером-перевозчиком. Поддерживаются динамические вызовы экстренных служб (в зависимости Teams клиента).

- Для прямой маршрутизации необходимо определить политики вызова экстренных служб для пользователей с помощью политики маршрутизации Teams экстренных вызовов (TeamsEmergencyCallRoutingPolicy), чтобы определить номера экстренных служб и связанные с ними маршрутизации. Зарегистрированные расположения экстренных служб не поддерживаются пользователями прямой маршрутной маршрутики. Для динамических вызовов экстренных служб требуется дополнительная конфигурация для маршрутизов экстренных вызовов и, возможно, для подключения партнеров.
