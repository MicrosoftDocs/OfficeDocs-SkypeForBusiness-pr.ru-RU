# <a name="meetings-first"></a>Вначале собрания

Задача "Сначала собрания" оптимизирована для организаций, Skype для бизнеса Server с локальной Корпоративная голосовая связь, которые хотят как можно скорее начать использовать Teams собрания. Для этих организаций режим "Собрания  сначала" является альтернативой использованию режима "Острова", который Teams собраниям.

## <a name="what-is-meetings-first"></a>Что такое "Собрания в первую очередь"?

First meetings is based on the **SfBWithTeamsCollabAndMeetings** coexistence mode. Собрания Во-первых, это не продукт или функция— это конфигурация, которая использует возможности и функции Teams и Skype для бизнеса для обеспечения уникального сосуществования.

На первом этапе собрания пользователи создают собрания в Teams, продолжая использовать Skype для бизнеса чата, звонков и присутствия. Перекрытие модальных режимов между Teams и Skype для бизнеса. Чат, вызовы и присутствие Skype для бизнеса в Teams. Это позволяет использовать уникальные сценарии "лучше вместе" между Skype для бизнеса и Teams, которые улучшают работу пользователей во время совместной  работы, а также сценарии взаимодействия с Teams только пользователями.

![Снимок экрана: лучший сценарий совместной работы с Teams и Skype для бизнеса.](media/meetings-first-meeting-in-meeting.png)

> [!Important]
> Для организаций, в которых нет активных Teams чатов, лучше всего сое Teams собрания. Активные Teams чата не следует переключаться в режим "Собрания в первый", так как они потеряют возможность общаться в Teams и получать доступ к истории чатов. Эти пользователи должны быть  перенаданы в режиме "О-ва", а "Собрания сначала" предоставлены только пользователям, которые еще не активны в чате в Teams.

## <a name="who-should-consider-meetings-first"></a>Кто сначала следует рассмотреть вопрос "Собрания"?

Первое собрание предназначено для организаций, которые используют Skype для бизнеса Server с Корпоративная голосовая связь, которые хотят ускорить переход на собрания Teams, особенно если ит-дисциплина и нужна управличенная детерминированная путь обновления Teams.

В сложных или крупных организациях перенос голосовой связи обычно происходит по сайту и может занять много времени, а это может занять несколько лет, что приводит к расширенным сценариям совместной работы. Если такое сосуществование находится  в режиме "Острова", пользователи всегда смогут выбрать два решения для собраний (Skype для бизнеса и Teams), которые могут привести к путанице или подопчему. В отличие от переноса голосовой почты, перенос собраний обычно может быть выполнен во всей компании за короткий промежуток времени. Организациям, которые хотят как можно скорее Teams собрания (не дожидаясь завершения переноса голосовой почты), следует сначала перейти на собрания.

Собрания. Первый из них может быть не полезен для организаций, у которых нет Корпоративная голосовая связь пользователей. Эти организации смогут перейти на Teams **только** после принятия Teams собраний. Им следует сначала пропустить собрания.

Кроме того, функцию Собрания в первую очередь удобно использовать для организаций, область действия которых является решением для проведения собраний в чистом виде, например при выдаче запроса только на собрания.

- Упрощенный пользовательский интерфейс в Teams, центр Teams собраниях, командах и каналах (которые при желании могут быть скрыты с помощью политики разрешений [приложений);](teams-app-permission-policies.md) [Teams](teams-client-experience-and-conformance-to-coexistence-modes.md) личных чатов, звонков и самосовершенство не личные беседы в первую очередь, что позволяет полностью сконцентрироваться на собраниях при развертывании и внедрении.
- Выше [Teams собрания.](tutorial-meetings-in-teams.yml)
- "Лучше вместе" Teams и Skype для бизнеса: 
  - Автоматическое удержание. Когда вы Teams в собрании, во время звонка в Skype для бизнеса, собрание Teams на удержание и наоборот. Это не позволит пользователям перенагрещать участников собраний своими личными звонками.
    ![Снимок экрана: лучший сценарий совместной работы с Teams и Skype для бизнеса.](media/meetings-first-better-together-hold.png)
  - Выверка присутствия. Teams в сети отражается на присутствии пользователя, что является Skype для бизнеса присутствия, так как чат и вызовы находятся в Skype для бизнеса. В частности, при первом собрании Teams собрания их присутствие будет обновлено с учетом этого. Когда они показывают свой экран, его присутствие будет обновлено с учетом их параметров в Skype для бизнеса).
  - Выверка управления USB-устройством HID (также доступна на Mac): элементы управления HID Teams собраниях Teams, а также Skype для бизнеса других случаях.
  - Если не указано иное, в настоящее время для улучшения совместной работы требуется Windows клиентах для настольных пк.

## <a name="prerequisites-for-meetings-first"></a>Предварительные условия для проведения собраний

Требования только к собраниям в первую очередь такие же, как и для Teams с локальной службой Active Directory и локальной Skype для бизнеса развертывания:

- [Общие требования для Teams,](upgrade-plan-journey-prerequisites.md)включая
- [Удостоверения и проверка подлинности в Teams](identify-models-authentication.md) и
- [Настройте Azure Active Directory для Teams и Skype для бизнеса](/skypeforbusiness/hybrid/configure-azure-ad-connect).

Не [Skype для бизнеса гибридной топологии,](/skypeforbusiness/hybrid/configure-federation-with-skype-for-business-online) но рекомендуется. Некоторые возможности, такие как служба миграции собраний и возможность интероперируемости, зависят от этой топологии.

Собрания. В первую очередь поддерживается любая версия Skype для бизнеса Server (и она работает с сервером Lync Server, который больше не поддерживается). Она поддерживается любым поддерживаемым клиентом Skype для бизнеса, но для улучшения совместной работы требуется последний клиент.

После того как эти требования будут выполнены (а не раньше), пользователи смогут получить лицензии на Microsoft 365 или Office 365 [и Teams.](/office365/enterprise/assign-licenses-to-user-accounts)

Для лучшего опыта проведения собраний в первую очередь пользователям следует включить [Exchange Online,](exchange-teams-interact.md) [SharePoint Online,](sharepoint-onedrive-interact.md)OneDrive для бизнеса и Microsoft 365 групп. Собрания В первую очередь поддерживается для пользователей, почтовые ящики которых Exchange локально, у которых нет SharePoint Online, OneDrive для бизнеса или Microsoft 365 групп. Однако их опыт будет менее полным. В частности, для организаций, использующих Exchange Server локально, могут быть (в зависимости от версии Exchange Server) некоторые ограничения на создание и просмотр собраний с помощью клиента Teams, а также в отношении возможностей обеспечения соответствия требованиям.

Как минимум у пользователей должна быть [лицензия на Teams.](/microsoft-365/admin/manage/assign-licenses-to-users) Кроме того, при необходимости [](set-up-audio-conferencing-in-teams.md)они могут быть лицензированы на аудиоконференцию.

Мы рекомендуем на момент лицензирования пользователей использовать по умолчанию [ **SfBOnly** или **SfBWithTeamsCollab**](/powershell/module/skype/grant-csteamsupgradepolicy?view=skype-ps) в качестве клиента. Это гарантирует, что пользователи не начнут использовать Teams в  режиме "Острова" по умолчанию до того, как вы будете готовы к запуску собрания.

Собрания в первую очередь поддерживаются в классических клиентах (Windows и Mac), в клиентах браузера и мобильных клиентах. Он также совместим с [Комнаты Microsoft Teams](/microsoftteams/room-systems/). Для улучшения совместной работы требуется полный клиент для настольных компьютеров.
