# <a name="meetings-first"></a>Вначале собрания

Задача "Сначала собрания" оптимизирована для организаций, Skype для бизнеса Server с локальной Корпоративная голосовая связь, которые хотят как можно скорее начать Teams собрания. Для этих организаций режим "Собрания  сначала" является альтернативой использованию режима "Острова", который Teams собраниям.

## <a name="what-is-meetings-first"></a>Что такое "Собрания в первую очередь"?

First meetings is based on the **SfBWithTeamsCollabAndMeetings** coexistence mode. Собрания Во-первых, это не продукт или функция— это конфигурация, которая использует возможности и функции Teams и Skype для бизнеса для обеспечения уникального сосуществования.

На первом этапе собрания пользователи создают собрания в Teams, продолжая использовать Skype для бизнеса чата, звонков и присутствия. Перекрытие модалий между Teams и Skype для бизнеса. Чат, вызовы и присутствие Skype для бизнеса в Teams. Это позволяет использовать уникальные сценарии "лучше вместе" Skype для бизнеса и Teams, которые улучшают работу пользователей во время совместной работы, а также сценарии взаимодействия с Teams пользователями. 

![Снимок экрана: лучший сценарий совместной работы с Teams и Skype для бизнеса](media/meetings-first-meeting-in-meeting.png)

> [!Important]
> Первым собраниям лучше соответствовать организациям, у которых нет активных Teams чатов. Активные Teams чата не следует переключаться в режим "Собрания в первый", так как они потеряют возможность общаться в чате Teams доступ к истории чатов. Эти пользователи должны быть  заданы в режиме "О-ва", а "Собрания сначала" предоставлены только пользователям, которые еще не активны в чате в Teams.

## <a name="who-should-consider-meetings-first"></a>Кто сначала следует рассмотреть вопрос "Собрания"?

Первое собрание предназначено для организаций, которые используют Skype для бизнеса Server с Корпоративная голосовая связь, которые хотят ускорить переход на собрания Teams, особенно если ит-дисциплина и нужна управличенная детерминированная путя обновления Teams.

В сложных или крупных организациях перенос голосовой связи обычно происходит по сайту и может занять много времени, а это может занять несколько лет, что приводит к расширенным сценариям совместной работы. Если такое сосуществование находится  в режиме "О-ва", пользователи всегда смогут выбрать два решения для собраний (Skype для бизнеса и Teams), которые могут привести к путанице или подразрядным ситуациям. В отличие от переноса голосовой почты, перенос собраний обычно может быть выполнен во всей компании за короткий промежуток времени. Организациям, желающим полностью перейти Teams собрания как можно быстрее (и не дожидаясь завершения голосовой миграции), следует сначала рассмотреть вариант Собрания.

Собрания. Первый из них может быть не полезен для организаций, у которых нет Корпоративная голосовая связь пользователей. Эти организации смогут перейти на Teams **только** после принятия Teams собраний. Им следует сначала пропустить собрания.

Кроме того, функции Собрания в первую очередь полезны организациям, область действия которых является решением для проведения собраний в чистом виде, например при выдаче запроса только на собрания.

- Упрощенный пользовательский интерфейс в Teams, в центре Teams собраний, групп и каналов (которые при желании могут быть скрыты с помощью политики разрешений [приложений);](teams-app-permission-policies.md) [Teams](teams-client-experience-and-conformance-to-coexistence-modes.md) личных чатов, звонков и самосовершенство не личные беседы в первую очередь, что позволяет полностью сконцентрироваться на собраниях при развертывании и внедрении.
- [Вышестоя Teams собрания](tutorial-meetings-in-teams.yml).
- "Лучше вместе" Teams и Skype для бизнеса: 
  - Автоматическое удержание. Когда вы Teams в собрании, звоня в Skype для бизнеса, собрание Teams на удержание и наоборот. Это не позволит пользователям перенагрещать участников собраний своими личными звонками.
    ![Снимок экрана: лучший сценарий совместной работы с Teams и Skype для бизнеса](media/meetings-first-better-together-hold.png)
  - Выверка присутствия. Teams в сети отражается на присутствии пользователя, что является Skype для бизнеса присутствия, так как чат и Skype для бизнеса. В частности, при первом собрании Teams собрания их присутствие будет обновлено с учетом этого. Когда они показывают свой экран, его присутствие будет обновлено с учетом их параметров в Skype для бизнеса).
  - Выверка управления USB-устройством HID (также доступна на Mac): элементы управления HID Teams в собраниях Teams собраниях и Skype для бизнеса при любых других обстоятельствах.
  - Если не указано иное, в настоящее время для улучшения совместной работы требуются Windows клиенты для настольных пк.

## <a name="prerequisites-for-meetings-first"></a>Предварительные условия для проведения собраний

Требования только к собраниям в первую очередь те же, что и для Teams с локальной службой Active Directory и Skype для бизнеса локального развертывания:

- [Общие требования для Teams,](upgrade-plan-journey-prerequisites.md)включая
- [Удостоверения и проверка подлинности в Teams](identify-models-authentication.md) и
- [Настройте Azure Active Directory для Teams и Skype для бизнеса](/skypeforbusiness/hybrid/configure-azure-ad-connect).

Не [Skype для бизнеса гибридной топологии,](/skypeforbusiness/hybrid/configure-federation-with-skype-for-business-online) но рекомендуется. Некоторые возможности, такие как служба миграции собраний и возможность интероперируемости, зависят от этой топологии.

Собрания. В первую очередь поддерживается любая версия Skype для бизнеса Server (которая, как известно, работает с сервером Lync Server, который больше не поддерживается). Она поддерживается любым поддерживаемым клиентом Skype для бизнеса, но для улучшения возможностей совместной работы требуется последний клиент.

После того как эти требования будут выполнены (а не раньше), пользователи смогут получить лицензии на Microsoft 365 или Office 365 [и Teams.](/office365/enterprise/assign-licenses-to-user-accounts)

Для лучшего опыта проведения собраний в первую очередь пользователям следует включить [Exchange Online,](exchange-teams-interact.md) [SharePoint Online,](sharepoint-onedrive-interact.md)OneDrive для бизнеса и Microsoft 365 групп. Собрания. В первую очередь поддерживается для пользователей, почтовые ящики которых Exchange локально, у которых нет SharePoint Online или OneDrive для бизнеса, а также Microsoft 365 групп. Однако их опыт будет менее полным. В частности, для организаций, использующих Exchange Server локально, могут быть (в зависимости от версии Exchange Server) некоторые ограничения на создание и просмотр собраний с помощью клиента Teams, а также в отношении возможностей обеспечения соответствия требованиям.

Как минимум у пользователей должна быть [лицензия на Teams.](/microsoft-365/admin/manage/assign-licenses-to-users) Кроме того, при необходимости [](set-up-audio-conferencing-in-teams.md)они могут быть лицензированы на аудиоконференцию.

Мы рекомендуем на момент лицензирования пользователей использовать по умолчанию [ **SfBOnly** или **SfBWithTeamsCollab**](/powershell/module/skype/grant-csteamsupgradepolicy?view=skype-ps) в качестве клиента. Это гарантирует, что пользователи не начнут использовать Teams  в режиме "Острова" по умолчанию до того, как вы будете готовы к запуску собрания в первую очередь.

Собрания в первую очередь поддерживаются в классических клиентах (Windows и Mac), в клиентах браузера и мобильных клиентах. Он также совместим с [Комнаты Microsoft Teams](/microsoftteams/room-systems/). Для улучшения совместной работы требуется полный клиент для настольных компьютеров.
