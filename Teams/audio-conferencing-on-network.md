# <a name="on-network-conferencing-for-audio-conferencing"></a>Аудиоконференция по сети

С помощью этой функции организации могут отправлять входящие и исходящие звонки на телефонные номера Майкрософт по прямой маршрутизации. Эта возможность не предназначена для расширения поддержки аудиоконференций на номера сторонних телефонных телефонов. Аудиоконференция по сети не поддерживается, если она используется для направления входящие вызовы в службу аудиоконференции через телефонные номера сторонних пользователей или исходящие вызовы через мост аудиоконференции Майкрософт.

В этой статье описаны необходимые условия и этапы настройки, необходимые для того, чтобы включить в организации интернет-конференцию.

> [!IMPORTANT]
> С телефонным оборудованием в Индии не должно быть развернуто сетевое оборудование.
  
## <a name="prerequisites"></a>Необходимые компоненты

Прежде чем настраивать интернет-конференцию, убедитесь, что ваша организация соответствует следующим требованиям:

- Убедитесь, что все пользователи в организации, у которых включена или будет включена аудиоконференция, используют Teams для всех собраний. Маршрутизацию входящие и исходящие звонки аудиоконференции через интернет-конференцию поддерживаются только для Teams собраний.

- Назначьте лицензии на аудиоконференцию всем пользователям, которые будут использовать эту функцию.

- Настройка службы аудиоконференций. Дополнительные сведения см. в [этой](set-up-audio-conferencing-in-teams.md)Microsoft Teams.

- Настройка контроллера границы сеанса (SBC) для прямой маршрутии. Дополнительные сведения [см.](direct-routing-plan.md) в настройках прямой маршрутной маршрутии и Планировании [прямой маршрутии.](direct-routing-configure.md)

  Если прямая маршрутия настраивается только для аудиоконференций, для этой цели необходимо выполнить только "Шаг 1. Подключение вашего SBC".
  
## <a name="enable-the-routing-of-dial-in-calls-to-microsoft-audio-conferencing-through-direct-routing"></a>Включить маршрутику звонков с телефонным доступом на аудиоконференцию Майкрософт через прямую маршрутику

Чтобы перенаправить звонки с телефонным доступом, сделанные локально пользователями, в службу аудиоконференций с помощью прямой маршрутики, необходимо настроить соответствующие правила маршрутики для служб SBCs и private Branch Exchange(PBX).

Необходимо настроить телефонное оборудование сайтов для маршрутизации звонков на любой номер моста конференц-связи вашей организации с помощью линии прямой маршрутизации.

Номера служб можно найти в Центре администрирования Teams в статье Мосты для собраний **>** или с помощью Skype для бизнеса Online PowerShell Get-CsOnlineDialInConferencingBridge. Дополнительные сведения см. в списке номеров аудиоконференций в [Microsoft Teams.](see-a-list-of-audio-conferencing-numbers-in-teams.md)

> [!NOTE]
> Эта функция недоступна пользователям с лицензией на аудиоконференцию с поминутной оплатой.

## <a name="enable-the-routing-of-teams-meeting-dial-out-calls-through-direct-routing"></a>Включить маршрутику звонков Teams с помощью прямой маршрутии

Teams вызовы телефонного звонка инициировали из собрания в организации на номера ЗВОНКОВ по ДНР, включая звонки по телефону и вызовы для привлечения новых участников к собранию.

Чтобы включить Teams маршрутизации собраний с помощью прямой маршрутизации пользователям в сети, необходимо создать и назначить политику маршрутизации аудиоконференции OnlineAudioConferencingRoutingPolicy.

Политика OnlineAudioConferencingRoutingPolicy эквивалентна политике CsOnlineVoiceRoutingPolicy для звонков 1:1 по ОКП по прямой маршрутизации. Управлять политикой OnlineAudioConferencingRoutingPolicy можно с помощью следующих cmdlets:

-   New-CsOnlineAudioConferencingRoutingPolicy
- Set-CsOnlineAudioConferencingRoutingPolicy
- Get-CsOnlineAudioConferencingRoutingPolicy
- Grant-CsOnlineAudioConferencingRoutingPolicy
- Remove-CsOnlineAudioConferencingRoutingPolicy

Дополнительные сведения о маршрутике для прямой маршрутии см. в этой [ссылке.](direct-routing-voice-routing.md)

Чтобы включить маршрутику звонков с телефонным вызовом через Прямую маршрутику, необходимо:

- Настройка политик маршрутизации аудиоконференций
- Настройка маршрутизации на телефонном оборудовании организации
- (Необязательно) Настройка набора номера

Исходят звонки из Teams будут исходят из номера службы по умолчанию на мосте конференции. Дополнительные сведения о номере службы по умолчанию для моста аудиоконференций см. в этой [теме.](change-the-phone-numbers-on-your-audio-conferencing-bridge.md)

### <a name="configure-audio-conferencing-routing-policies"></a>Настройка политик маршрутизации аудиоконференций

Политика маршрутизации аудиоконференции OnlineAudioConferencingRoutingPolicy определяет, какие звонки на собрания перенаправляются на линии прямой маршрутизации. Если вы знакомы с политикой CsOnlineVoiceRoutingPolicy, она работает почти так же.

Чтобы настроить политики маршрутинга аудиоконференций, необходимо сделать следующее:

1. Создание использования ОКП
1. Настройка голосовых маршрутов
1. Создание политик голосовой маршрутизации аудиоконференций
1. Назначение политики пользователям

#### <a name="create-pstn-usages"></a>Создание использования ОКП

Использование услуг STN — это наборы голосовых маршрутов. При инициирующем звонке из собрания организатора голосовые маршруты будут использоваться для определения пути маршрутизов на основе использования услуг STN, связанных с пользователем с помощью политики маршрутизов голосовой связи пользователя.

Вы можете создать использование ПСОП с помощью "Set-CsOnlinePstnUsage". Например:

```powershell
Set-CsOnlinePstnUsage -Identity Global -Usage @{Add="US and Canada"}
```

#### <a name="configure-voice-routes"></a>Настройка голосовых маршрутов

Голосовые маршруты определяют шлюз ЗВОНКОВ, который должен использоваться для перенастройки звонка на основе номера телефона, набраного из Teams собрания. Voice routes determine the PSTN gateway that should be used to route a given call by matching the phone number dialed from a Teams with a regex pattern. При создании голосового маршрута он должен быть связан с одним или нескольком использованием ОКП.

Вы можете создать голосовой маршрут и определить regex и шлюзы, которые будут связаны с голосовым маршрутом, с помощью "New-CsOnlineVoiceRoute". Например:

```powershell
New-CsOnlineVoiceRoute -Identity "Redmond 1" -NumberPattern "^\+1(425|206)(\d{7})$" -OnlinePstnGatewayList sbc1.contoso.biz, sbc2.contoso.biz -Priority 1 -OnlinePstnUsages "US and Canada"
```

#### <a name="create-audio-conferencing-voice-routing-policies"></a>Создание политик голосовой маршрутизации аудиоконференций

Политики маршрутизации голосовой связи для аудиоконференций определяют возможные маршруты, которые могут использоваться для перенастройки звонка, исходя из собраний организатора, на основе целевого номера телефона в телефонном звонке. Политики перенастройки голосовой маршрутизации аудиоконференций связаны с одним или несколькою вариантами использования услуг ЗВОНКОВ по STN, которые, в свою очередь, определяют возможные маршруты, которые будут использоваться для звонков с телефонным вызовом организаторов, связанных с политикой.

Политику маршрутизации голосовой маршрутизации аудиоконференции можно создать с помощью "New-CsOnlineAudioConferencingRoutingPolicy". Например:

```powershell
New-CsOnlineAudioConferencingRoutingPolicy "Policy 1" -OnlinePstnUsages "US and Canada"
```

После того как политика будет назначена пользователю и когда из одного из собраний будет инициировался звонок с телефонного звонка, будут оцениваться голосовые маршруты в рамках использования услуг STN, связанных с организатором с помощью политики маршрутизов голосовой связи пользователя. Если место назначения для телефонного звонка для собрания совпадает срегесом в одном из голосовых маршрутов, связанных с организатором, звонок будет перенаправлен на шлюз STN, определенный в голосовом маршруте. Если место назначения для телефонного звонка не соответствует ни одной из голосовых маршрутов, связанных с организатором, звонок будет перенаправлен корпорацией Майкрософт.

#### <a name="assign-a-policy-to-your-users"></a>Назначение политики пользователям

После определения политик маршрутинга аудиоконференций вы можете назначить их пользователям. После того как политикам будут назначены политики, для определения пути их маршрутки будут оцениваться звонки с телефонным вызовом на собрание. Политики маршрутизации аудиоконференций всегда оцениваются на основе организатора собрания независимо от пользователя в собрании, который инициирует звонок с телефонного звонка.

Политику маршрутизации голосовой сети аудиоконференции можно назначить пользователю с помощью "Grant-CsOnlineAudioConferencingRoutingPolicy". Например:

```powershell
Grant-CsOnlineAudioConferencingRoutingPolicy -Identity "<User Identity>" -PolicyName "Policy 1”
```

### <a name="configure-routing-on-the-telephony-equipment-of-your-organization"></a>Настройка маршрутизации на телефонном оборудовании организации

На телефонном оборудовании организации необходимо обеспечить маршрутизовы телефонного звонка, перенаправленные по прямой маршрутизации, в предполагаемый сетевой маршрут.

### <a name="optional-configure-a-dial-plan"></a>(Необязательно) Настройка набора номера

Телефонная плана — это набор правил нормализации, которые переводят номера телефонов отдельных пользователей в альтернативный формат (обычно E.164) для авторизации звонков и маршрутизации звонков.

По умолчанию Teams к номерам ДНР в формате E.164, т. е. + \<country code\> \<number\> . Однако можно использовать планы набора, чтобы разрешить пользователям набирать номера телефонов в других форматах, например в четырехзначных расширениях.

Если вы хотите включить телефонную сеть с телефонным набором с учетом расширения, вы можете настроить наборные планы так, чтобы они совпадали с диапазонами телефонных номеров вашей организации. Чтобы настроить наборные планы, см. создание планов набора номера [и управление ими.](create-and-manage-dial-plans.md)
