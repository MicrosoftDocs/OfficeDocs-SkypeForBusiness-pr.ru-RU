# <a name="open-preview-of-on-network-conferencing-for-audio-conferencing"></a>Открытие предварительного просмотра интернет-аудиоконференции

Открытая предварительная версия интернет-конференции доступна для всех клиентов. С помощью сетевой аудиоконференции организации могут отправлять входящие и исходящие звонки на телефонные номера Майкрософт через прямую маршрутизацию. Эта возможность не предназначена для расширения поддержки аудиоконференций на номера сторонних телефонных номеров. Интернет-аудиоконференция не поддерживается, если она используется для направления входящие вызовы в службу аудиоконференций через телефонные номера сторонних пользователей.

В этой статье описаны предварительные условия и действия по настройке, необходимые для того, чтобы включить в организации интернет-конференцию.

> [!IMPORTANT]
> Сетевое оборудование для телефонной связи в Индии не должно быть развернуто.
  
## <a name="prerequisites"></a>Необходимые компоненты

Перед настройкой интернет-связи убедитесь, что ваша организация соответствует следующим требованиям: 

- Убедитесь, что все пользователи в организации, у которых включена или включена аудиоконференция, используют Teams для всех собраний. Маршрутия входящие и исходящие вызовы аудиоконференции с помощью сетевой аудиоконференции поддерживается только для собраний Teams.

- Назначьте лицензии на аудиоконференцию всем пользователям, которые будут использовать эту функцию.

- Настройка службы аудиоконференций. Дополнительные сведения см. в сведениях о настройках аудиоконференций [для Microsoft Teams.](set-up-audio-conferencing-in-teams.md)

- Настройка граничного контроллера сеанса (SBC) для прямой маршрутки. Дополнительные сведения см. в планах ["Прямая маршрутия"](direct-routing-plan.md) и ["Настройка прямой маршрутинга".](direct-routing-configure.md) 

  Если прямая маршрутия настраивается только для аудиоконференции, необходимо выполнить только шаг 1 . Подключение SBC к сетевой аудиоконференции.
  
## <a name="enable-the-routing-of-dial-in-calls-to-microsoft-audio-conferencing-through-direct-routing"></a>Включить маршрутизацию звонков с телефонным доступом на аудиоконференцию Майкрософт через прямую маршрутизацию 

Чтобы перенаправлять звонки с телефонным доступом, сделанные пользователями локальной сети, в службу аудиоконференций с помощью прямой маршрутики, необходимо настроить соответствующие правила маршрутики для SBCs и частных филиалов Exchange.

Необходимо настроить телефонное оборудование ваших сайтов для перенаправки звонков на любой номер службы моста конференц-связи вашей организации через линию прямой маршрутизации.

Номера служб можно найти в Центре администрирования Teams на веб-сайте **"Собрания>** мосты для собраний" или с помощью командлета Get-CsOnlineDialInConferencingBridge в Skype для бизнеса Online. Дополнительные сведения см. в списке номеров для аудиоконференций [в Microsoft Teams.](see-a-list-of-audio-conferencing-numbers-in-teams.md)

> [!NOTE]
> Эта функция недоступна пользователям с лицензией на аудиоконференцию с поминутной оплатой.

## <a name="enable-the-routing-of-teams-meeting-dial-out-calls-through-direct-routing"></a>Включить маршрутику звонков с телефонным вызовом на собрании Teams с помощью прямой маршрутинга

Вызовы телефонного звонка в Teams начаты из собрания вашей организации на номера ЗВОНКОВ по ДНР, включая звонки на мой номер и вызовы для привлечения новых участников к собранию. 

Чтобы включить маршрутизация для собраний Teams с помощью прямой маршрутизации сетевым пользователям, необходимо создать и назначить политику маршрутизации аудиоконференции под названием "OnlineAudioConferencingRoutingPolicy". 

Политика OnlineAudioConferencingRoutingPolicy эквивалентна политике CsOnlineVoiceRoutingPolicy для звонков по 1:1 через direct Routing. Управлять политикой OnlineAudioConferencingRoutingPolicy можно с помощью следующих cmdlets:

-   New-CsOnlineAudioConferencingRoutingPolicy
- Set-CsOnlineAudioConferencingRoutingPolicy
- Get-CsOnlineAudioConferencingRoutingPolicy
- Grant-CsOnlineAudioConferencingRoutingPolicy
- Remove-CsOnlineAudioConferencingRoutingPolicy

Дополнительные сведения о маршрутинге для прямой маршрутки см. в подстройке "Настройка голосовой маршрутии [для прямой маршрутинга".](direct-routing-voice-routing.md)


Чтобы включить маршрутику звонков для звонков с прямой маршрутии, необходимо: 

- Настройка политик маршрутизации аудиоконференций
- Настройка маршрутизации на телефонном оборудовании организации
- (Необязательно) Настройка набора номера

Исходят звонки из собраний Teams будут приходить с номера службы по умолчанию на мосте конференц-зала. Дополнительные сведения о номере службы по умолчанию для моста аудиоконференций см. в сведениях об изменении номеров телефонов для моста [аудиоконференции.](change-the-phone-numbers-on-your-audio-conferencing-bridge.md)

### <a name="configure-audio-conferencing-routing-policies"></a>Настройка политик маршрутизации аудиоконференций

Политика маршрутизации аудиоконференции OnlineAudioConferencingRoutingPolicy определяет, какие звонки с телефонного звонка на собрания перенаправляются на линии прямой маршрутизации. Если вы знакомы с политикой CsOnlineVoiceRoutingPolicy, эта политика работает почти так же.

Чтобы настроить политики маршрутинга аудиоконференций, необходимо сделать следующее:
1.  Создание использования ННР
2.  Настройка голосовых маршрутов
3.  Создание политик маршрутизации голосовой маршрутки для аудиоконференций
4.  Назначение политики пользователям


#### <a name="create-pstn-usages"></a>Создание использования ННР

Использование услуг ННР — это наборы голосовых маршрутов. При инициирующем звонке из собрания организатора голосовые маршруты используются для определения пути маршрутизов звонка на основе использования телефонной сети ННР, связанных с пользователем с помощью политики маршрутинга голосовой связи пользователя.

Вы можете создать использование ННР с помощью cmdlet "Set-CsOnlinePstnUsage". Например:

```powershell
Set-CsOnlinePstnUsage -Identity Global -Usage @{Add="US and Canada"}
```

#### <a name="configure-voice-routes"></a>Настройка голосовых маршрутов

Voice routes determine the PSTN gateway that should be used to route a call based on the phone number that is dialed from a Teams meeting. Voice Routes determine the PSTN gateway that should be used to route a call by matching the phone number dialed from a Teams meeting with a regex pattern. При создании голосового маршрута он должен быть связан с одним или несколько использованием услуг ННР.

Вы можете создать голосовой маршрут и определить regex и шлюзы, которые будут связаны с голосовым маршрутом, с помощью cmdlet "New-CsOnlineVoiceRoute". Например:

```powershell
New-CsOnlineVoiceRoute -Identity "Redmond 1" -NumberPattern "^\+1(425|206)(\d{7})$" -OnlinePstnGatewayList sbc1.contoso.biz, sbc2.contoso.biz -Priority 1 -OnlinePstnUsages "US and Canada"
```

#### <a name="create-audio-conferencing-voice-routing-policies"></a>Создание политик маршрутизации голосовой маршрутки для аудиоконференций

Политики маршрутизации голосовой связи аудиоконференций определяют, какие маршруты можно использовать для перенастройки звонка из собраний организатора на основе конечного номера телефона в телефонном звонке собрания. Политики голосовой маршрутизации аудиоконференций связаны с одним или нескольком использованием услуг ЗВОНКОВ по ДНР, которые, в свою очередь, определяют возможные маршруты, которые будут использоваться для звонков с телефонным вызовом организаторов, связанных с политикой.

Политику маршрутизации голоса для аудиоконференции можно создать с помощью cmdlet "New- CsOnlineAudioConferencingRoutingPolicy". Например:

```powershell
New-CsOnlineAudioConferencingRoutingPolicy "Policy 1" -OnlinePstnUsages "US and Canada"
```

После того как политика назначена пользователю и когда для одного из собраний происходит звонок с телефонного звонка, голосовая маршрутия в ОКП, связанные с организатором с помощью политики маршрутинга голосовой связи пользователя, оцениваются. Если место назначения для звонка по телефону совпадает с агрегатом в одном из голосовых маршрутов, связанных с организатором, звонок будет перенаправлен на шлюз ННР, определенный в голосовом маршруте. Если место назначения для телефонного звонка не соответствует ни одной из голосовых маршрутов, связанных с организатором, звонок будет перенаправлен корпорацией Майкрософт.

#### <a name="assign-a-policy-to-your-users"></a>Назначение политики пользователям

После определения политик маршрутинга аудиоконференций вы можете назначать их пользователям. После того как им будут назначены политики, вызовы телефонного звонка к собранию будут оцениваться с ее целью для определения пути маршрутизов. Политики маршрутинга аудиоконференций всегда оцениваются на основе организатора собрания независимо от пользователя в собрании, который инициирует исходяющий звонок.

Политику маршрутизации голосовых аудиоконференции можно назначить пользователю с помощью cmdlet Grant-CsOnlineAudioConferencingRoutingPolicy. Например:

```powershell
Grant-CsOnlineAudioConferencingRoutingPolicy -Identity "<User Identity>" -PolicyName "Policy 1”
```


### <a name="configure-routing-on-the-telephony-equipment-of-your-organization"></a>Настройка маршрутизации для телефонного оборудования организации

На телефонном оборудовании вашей организации необходимо обеспечить маршрутизации звонков телефонного звонка, перенаправленных по прямой маршрутизации в пункт назначения в сети.


### <a name="optional-configure-a-dial-plan"></a>(Необязательно) Настройка набора номера

Телефонная плана — это набор правил нормализации, который преобразует номера телефонов конкретного пользователя в альтернативный формат (обычно E.164) для авторизации звонков и маршрутизации звонков.

По умолчанию пользователи Teams могут звонить на номера ННР в формате E.164, то есть + \<country code\> \<number\> . Однако можно использовать такие планы, чтобы разрешить пользователям набирать номера телефонов в других форматах, например в четырехзначных расширениях.

Если вы хотите включить телефонную сеть с учетом расширения, вы можете настроить наборные планы так, чтобы они совпадали с диапазонами телефонных номеров вашей организации. О том, как настроить наборные планы, см. в теме "Создание телефонных планов [и управление ими".](create-and-manage-dial-plans.md)


## <a name="known-issues-in-open-preview"></a>Известные проблемы в режиме предварительного просмотра Open Preview

Ниже представлен список известных проблем, которые в настоящее время присутствуют в выпуске Open Preview для интернет-связи. Корпорация Майкрософт работает над решением этих проблем.

| Ошибка | Обходное решение |
| :--- | :--- |
| К собранию нельзя подключаться к звонкам с анонимными или скрытыми ИД звонив через сетевую связь. | По возможности задайте в УАЦ или SBC конфигурацию, чтобы всегда отправлять ИД вызываемого звонка для звонков, перенаправляемых через сетевую веб-конференцию.|
| В некоторых случаях приветствие, которое пользователи слышат при наборе номера в службу ("Добро пожаловать в службу аудиоконференций..."), убирается, и пользователи не слышат слова "Добро пожаловать".| В настоящее время обходного пути для этой проблемы нет. |
