# <a name="on-network-conferencing-for-audio-conferencing"></a>Конференц-связь по сети для аудиоконференций

Конференц-связь по сети позволяет организациям отправлять входящие и исходящие вызовы аудиоконференций на телефонные номера Майкрософт через прямую маршрутизацию. Эта возможность не предназначена для расширения поддержки аудиоконференций на номера сторонних абонентов. Конференц-связь по сети не поддерживается, если она используется для маршрутизации входящих вызовов в службу аудиоконференций через телефонные номера сторонних абонентов или исходящие вызовы к ТСОП из моста аудиоконференций Майкрософт.

В этой статье описываются предварительные требования и действия по настройке, необходимые для включения конференц-связи в сети для вашей организации.

> [!IMPORTANT]
> Конференц-связь по сети не должна развертываться с телефонным оборудованием в Индии.

## <a name="prerequisites"></a>Необходимые компоненты

Перед настройкой конференц-связи в сети убедитесь, что ваша организация соответствует следующим предварительным требованиям:

- Убедитесь, что все пользователи в вашей организации, которые включены или будут включены для аудиоконференций, используют Teams для всех собраний. Маршрутизация входящих и исходящих аудиоконференций с помощью конференц-связи по сети поддерживается только для собраний Teams.

- Назначьте лицензии на аудиоконференции всем пользователям, которые будут использовать конференц-связь в сети.

- Настройка службы аудиоконференций. Дополнительные сведения см. в разделе "Настройка аудиоконференций [для Microsoft Teams"](set-up-audio-conferencing-in-teams.md).

- Настройка пограничного контроллера сеанса (SBC) для прямой маршрутизации. Дополнительные сведения см [. в разделе "Планирование прямой маршрутизации](direct-routing-plan.md) и настройка [прямой маршрутизации"](direct-routing-configure.md).

  Если вы настраивали прямую маршрутизацию только для аудиоконференций, вам потребуется выполнить только шаг 1. Подключение SBC для конференц-связи в сети.

## <a name="enable-the-routing-of-dial-in-calls-to-microsoft-audio-conferencing-through-direct-routing"></a>Включение маршрутизации звонков с телефонным подключением к аудиоконференции Майкрософт с помощью прямой маршрутизации

Чтобы маршрутизировать вызовы с телефонным подключением, сделанные локальными пользователями, в службу аудиоконференций через прямую маршрутизацию, необходимо настроить соответствующие правила маршрутизации для контроллеров SBC и обмена сообщениями частной ветви (УАТС).

Необходимо настроить телефонное оборудование ваших сайтов для маршрутизации вызовов на любой номер службы моста конференций вашей организации через магистраль прямой маршрутизации.

Номера служб можно найти в Центре администрирования Teams в разделе "Собрания **>** Мосты конференц-связи" или с помощью командлета Teams PowerShell Get-CsOnlineDialInConferencingBridge. Дополнительные сведения см. в списке номеров аудиоконференций [в Microsoft Teams](see-a-list-of-audio-conferencing-numbers-in-teams.md).

> [!NOTE]
> Эта функция недоступна пользователям с лицензией на аудиоконференции с оплатой поминутной оплаты.

## <a name="enable-the-routing-of-teams-meeting-dial-out-calls-through-direct-routing"></a>Включение маршрутизации телефонных звонков на собрания Teams с помощью прямой маршрутизации

Звонки с телефонным подключением к собраниям Teams инициируются из собрания в вашей организации на номера ТСОП, включая звонки с телефонным подключением и вызовы, чтобы привести новых участников к собранию.

Чтобы включить маршрутизацию конференц-связи Teams через прямую маршрутизацию для пользователей в сети, необходимо создать и назначить политику маршрутизации аудиоконференций с именем OnlineAudioConferencingRoutingPolicy.

Политика OnlineAudioConferencingRoutingPolicy эквивалентна политике CsOnlineVoiceRoutingPolicy для вызовов ТСОП 1:1 через прямую маршрутизацию. Управлять политикой OnlineAudioConferencingRoutingPolicy можно с помощью следующих командлетов:

- New-CsOnlineAudioConferencingRoutingPolicy
- Set-CsOnlineAudioConferencingRoutingPolicy
- Get-CsOnlineAudioConferencingRoutingPolicy
- Grant-CsOnlineAudioConferencingRoutingPolicy
- Remove-CsOnlineAudioConferencingRoutingPolicy

Дополнительные сведения о маршрутизации для прямой маршрутизации см. в разделе "Настройка [голосовой маршрутизации для прямой маршрутизации"](direct-routing-voice-routing.md).

Чтобы включить маршрутизацию телефонных звонков собрания через прямую маршрутизацию, необходимо выполнить следующие действия:

- Настройка политик маршрутизации аудиоконференций
- Настройка маршрутизации на телефонное оборудование организации
- (Необязательно) Настройка абонентской группы

Исходящие звонки из собраний Teams поступают с номера службы по умолчанию на мосте конференций. Дополнительные сведения о номере службы по умолчанию моста аудиоконференций см. в разделе "Изменение номеров телефонов в мосте аудиоконференций ["](change-the-phone-numbers-on-your-audio-conferencing-bridge.md).

### <a name="configure-audio-conferencing-routing-policies"></a>Настройка политик маршрутизации аудиоконференций

Политика маршрутизации аудиоконференций OnlineAudioConferencingRoutingPolicy определяет, какие телефонные вызовы собраний направляются в магистрали прямой маршрутизации. Если вы знакомы с политикой CsOnlineVoiceRoutingPolicy, эта политика работает очень похожим образом.

Чтобы настроить политики маршрутизации аудиоконференций, необходимо выполнить следующие действия:

1. Создание сведений об использовании ТСОП
1. Настройка голосовых маршрутов
1. Создание политик маршрутизации голосовой связи для аудиоконференций
1. Назначение политики пользователям

#### <a name="create-pstn-usages"></a>Создание сведений об использовании ТСОП

Использование ТСОП — это коллекции голосовых маршрутов. При инициировании исходящего звонка из собрания заданного организатора голосовые маршруты будут использоваться для определения пути маршрутизации звонка на основе использования ТСОП, связанных с пользователем с помощью политики маршрутизации голосовой связи пользователя.

Использование ТСОП можно создать с помощью командлета Set-CsOnlinePstnUsage. Например:

```powershell
Set-CsOnlinePstnUsage -Identity Global -Usage @{Add="US and Canada"}
```

#### <a name="configure-voice-routes"></a>Настройка голосовых маршрутов

Маршруты голосовых вызовов определяют шлюз ТСОП, который должен использоваться для маршрутизации звонка на основе номера телефона, который назовется из собрания Teams. Голосовые маршруты определяют шлюз ТСОП, который должен использоваться для маршрутизации заданного звонка, путем сопоставления номера телефона, наданного на собрании Teams, с использованием шаблона регулярного выражения. При создании голосового маршрута маршрут должен быть связан с одним или несколькими вариантами использования ТСОП.

С помощью командлета New-CsOnlineVoiceRoute можно создать голосовой маршрут и определить регулярное выражение и шлюзы, которые будут связаны с голосовым маршрутом. Например:

```powershell
New-CsOnlineVoiceRoute -Identity "Redmond 1" -NumberPattern "^\+1(425|206)(\d{7})$" -OnlinePstnGatewayList sbc1.contoso.biz, sbc2.contoso.biz -Priority 1 -OnlinePstnUsages "US and Canada"
```

#### <a name="create-audio-conferencing-voice-routing-policies"></a>Создание политик маршрутизации голосовой связи для аудиоконференций

Политики маршрутизации голосовой связи для аудиоконференций определяют возможные маршруты, которые можно использовать для маршрутизации звонка, исходящего из собраний организатора, на основе целевого номера телефона телефонного звонка. Политики маршрутизации голосовой связи аудиоконференций связаны с одним или несколькими параметрами использования ТСОП, которые, в свою очередь, определяют возможные маршруты, которые будут использоваться для звонков организаторов, связанных с политикой.

Политику маршрутизации голосовой связи для аудиоконференций можно создать с помощью командлета New- CsOnlineAudioConferencingRoutingPolicy. Например:

```powershell
New-CsOnlineAudioConferencingRoutingPolicy "Policy 1" -OnlinePstnUsages "US and Canada"
```

После назначения политики пользователю и инициации телефонного звонка из одного из собраний пользователя голосовая связь маршрутизируется в параметрах использования ТСОП, связанных с организатором, с помощью политики маршрутизации голосовой связи пользователя. Если место назначения телефонного вызова собрания соответствует регулярному выражению в одном из голосовых маршрутов, связанных с организатором, звонок с обратным вызовом собрания будет направляться в шлюз ТСОП, определенный в голосовом маршруте. Если назначение телефонного звонка собрания не соответствует ни одному из маршрутов голосовых вызовов, связанных с организатором, звонок с обратным вызовом будет перенаправлен корпорацией Майкрософт.

#### <a name="assign-a-policy-to-your-users"></a>Назначение политики пользователям

После определения политик маршрутизации аудиоконференций их можно назначить пользователям. После назначения им политик для определения пути маршрутизации будут оцениваться исходящие вызовы собрания. Политики маршрутизации аудиоконференций всегда оцениваются на основе организатора собрания независимо от пользователя в собрании, который инициирует телефонный звонок.

Политику маршрутизации голосовой связи аудиоконференций можно назначить пользователю с помощью командлета Grant-CsOnlineAudioConferencingRoutingPolicy. Например:

```powershell
Grant-CsOnlineAudioConferencingRoutingPolicy -Identity "<User Identity>" -PolicyName "Policy 1"
```

### <a name="configure-routing-on-the-telephony-equipment-of-your-organization"></a>Настройка маршрутизации на телефонное оборудование организации

На телефонных устройствах организации необходимо убедиться, что телефонные вызовы собрания, перенаправленные через прямую маршрутизацию, направляются в целевое расположение в сети.

### <a name="optional-configure-a-dial-plan"></a>(Необязательно) Настройка абонентской группы

Абонентский план — это набор правил нормализации, которые преобразовывают номера телефонов отдельного пользователя в альтернативный формат (обычно E.164) для авторизации звонков и маршрутизации звонков.

По умолчанию пользователи Teams могут использовать номера ТСОП в формате E.164, то есть +\<country code\>\<number\>. Однако абонентские группы можно использовать, чтобы разрешить пользователям на номера телефонов в других форматах, например в формате 4 цифр.

Если вы хотите включить набор номеров на основе расширений с помощью конференц-связи по сети, можно настроить абонентские группы, соответствующие шаблону набора расширений диапазонам номеров телефонов в организации. Сведения о настройке абонентской группы см. в статье ["Создание абонентской группы и управление ими"](create-and-manage-dial-plans.md).
